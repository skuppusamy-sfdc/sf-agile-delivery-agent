# /knowledge/sprints

Drop JIRA sprint exports here.

## Convention

```
sprints/
  Sprint 1/
    sprint1-export.html           # Original JIRA export (monolithic)
    stories/                      # Auto-generated per-story files
      STORY-101.md
      STORY-102.md
      ...
  Sprint 2/
    sprint2-export.html
    stories/
      ...
```

The folder name pattern is configurable via `workspace.config.yaml > sprints.folder_pattern` (default `Sprint {n}`).

The `stories/` subdirectories are generated by `scripts/split-sprint-stories.py`. Each `.md` file contains the full metadata, description, acceptance criteria, solution, and linked issues for a single story. This enables precise RAG retrieval — the AI fetches one 2-30KB story file instead of parsing a 1-4.5MB sprint HTML.

## Workflow

1. Export the sprint from JIRA as HTML
2. Drop into `Sprint N/`
3. Run `python scripts/split-sprint-stories.py --sprint "Sprint N"` to generate per-story markdown files
4. Run `python scripts/parse-sprint-html.py` to (re)build `SPRINT-INDEX.md` and `MASTER-STORY-INDEX.md`
5. Run `python scripts/create-ac-index.py` and `create-solution-index.py` to refresh the AC/Solution indexes

## Anti-patterns

- ❌ Don't commit huge HTML files without being intentional — they bloat the repo. Consider Git LFS for files >5MB.
- ❌ Don't hand-edit HTML exports — re-export from JIRA and replace the file.
- ❌ Don't read raw HTML in Cursor when an index would answer the question (see `_cursor/rules/token-optimization.md`).

## Description

As a Claims Operations Manager, I need a Coordination of Benefits (COB) summary panel on the Policy__c record page that displays primary and secondary coverage details, COB determination status, and any open COB investigation tasks so that adjusters can quickly verify coverage order before adjudicating claims and avoid overpayment errors.

COB data is stored across Policy__c (primary), a related list of COB_Coverage__c records (secondary/tertiary carriers), and open Task records flagged with Task.Type = 'COB Investigation'.

## Acceptance Criteria

Given a Claims adjuster views a Policy__c record page,
When the COB Summary LWC panel renders,
Then the panel displays Policy__c.COB_Status__c, Policy__c.Primary_Carrier__c, Policy__c.Primary_Policy_Number__c, and Policy__c.Primary_Effective_Date__c in the 'Primary Coverage' section.

Given the policy has one or more COB_Coverage__c child records,
When the panel renders the 'Other Coverage' section,
Then all COB_Coverage__c records are listed showing: COB_Coverage__c.Carrier_Name__c, COB_Coverage__c.Policy_Number__c, COB_Coverage__c.Coverage_Order__c (Primary/Secondary/Tertiary), COB_Coverage__c.Effective_Date__c, COB_Coverage__c.Termination_Date__c.

Given Task records exist where Task.WhatId = Policy__c.Id AND Task.Type = 'COB Investigation' AND Task.Status != 'Completed',
When the panel renders the 'Open COB Investigations' section,
Then each open Task is displayed with Task.Subject, Task.ActivityDate, Task.OwnerId (owner name), and a hyperlink to the Task record.

Given Policy__c.COB_Status__c = 'COB Confirmed — Secondary Payer',
When the panel header renders,
Then a blue informational badge 'COB Active' is displayed adjacent to the policy number.

Given Policy__c.COB_Status__c = 'Pending Investigation',
When the panel header renders,
Then an amber warning badge 'COB Pending' is displayed and a 'Create COB Task' quick action button is visible.

-Given the panel loads, display the Medicare Secondary Payer (MSP) questionnaire completion date from Policy__c.MSP_Questionnaire_Date__c- (descoped to Phase 3 per product backlog grooming 2024-11-08)

## Solution

Build a Lightning Web Component (cob_SummaryPanel_LWC) using @wire(getRecord) to fetch Policy__c fields and a @wire(getRelatedListRecords) adapter for COB_Coverage__c child records. Open COB Tasks fetched via @wire(getRelatedListRecords) with filter on Task.Type and Task.Status. Conditional badge rendering via getter computed properties returning CSS class strings. 'Create COB Task' quick action invokes standard Salesforce quick action 'NewTask' with predefined field values via NavigationMixin. LWC added to Policy Record Page via Lightning App Builder, visible to Claims_Adjuster_PS and Claims_Manager_PS permission sets.

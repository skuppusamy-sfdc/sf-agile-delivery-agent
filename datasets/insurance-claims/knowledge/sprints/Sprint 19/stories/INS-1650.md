## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to proactively surface a recommended SIU (Special Investigations Unit) referral flag on Claim__c records when it detects anomaly patterns indicative of potential fraud, waste, or abuse so that SIU analysts are engaged earlier in the claims lifecycle, reducing fraudulent payment leakage.

Anomaly patterns to detect include: duplicate billing (same CPT code, same provider, same member within 30 days), billing for services on dates of inpatient confinement (unbundling), provider NPI not matching Claim__c.Facility__c geography (>100 mile radius), and billed amount exceeding 3x the Fee_Schedule__c.Reimbursement_Rate__c for the CPT code.

## Acceptance Criteria

Given a Claim__c record is created or updated with Status__c = 'Pending Adjudication',
When the Agentforce Claims AI action (AgentAction_SIUReferralEvaluation) is invoked via the Agentforce Agent Topic,
Then the action evaluates all four anomaly pattern rules against the claim and its Claim_Line__c children and returns a structured JSON recommendation payload.

Given the AI action detects one or more anomaly patterns,
When the recommendation payload is returned,
Then Claim__c.SIU_Referral_Recommended__c (checkbox) = true, Claim__c.SIU_Anomaly_Flags__c (long text) is populated with a pipe-delimited list of triggered rule names and supporting evidence (e.g., 'DUPLICATE_BILLING | CPT 99213 billed by NPI 1234567890 on 2024-11-01 and 2024-11-15 for Member M-00441'), and Claim__c.SIU_Confidence_Score__c (percent) reflects the Einstein scoring confidence.

Given Claim__c.SIU_Referral_Recommended__c = true,
When the record is saved post-evaluation,
Then a Record-Triggered Flow creates a Case record with: Case.RecordType = 'SIU Referral', Case.AccountId = Claim__c.Member__c, Case.Subject = 'AI-Recommended SIU Referral — Claim [Claim__c.Name]', Case.Description = Claim__c.SIU_Anomaly_Flags__c, Case.Priority = 'High', Case.Origin = 'Agentforce', and the Case is assigned to the SIU Queue via Assignment Rule.

Given the SIU referral Case is created,
When the Claims Operations Manager views the Claim__c record,
Then an LWC alert banner ('SIU Referral Recommended by Agentforce — View Case') is displayed at the top of the record page with a hyperlink to the created Case record.

Given the anomaly evaluation runs and no patterns are detected,
When the action completes,
Then Claim__c.SIU_Referral_Recommended__c = false, Claim__c.SIU_Anomaly_Flags__c = null, and no Case is created.

Given the Agentforce action is invoked,
When the evaluation completes (pass or fail),
Then Claim__c.SIU_Last_Evaluated__c (DateTime) is stamped with the current timestamp for audit trail purposes.

## Solution

Define an Agentforce Agent Topic 'ClaimsFraudDetection' in the Einstein Copilot Studio with an Agent Action (AgentAction_SIUReferralEvaluation) backed by an Invocable Apex class (SIUReferralEvaluationAction). The Apex class implements four rule evaluators as Strategy pattern classes (DuplicateBillingRule, UnbundlingRule, GeographyMismatchRule, BilledAmountAnomalyRule), each returning a RuleResult DTO. Geography distance calculation uses a utility class leveraging Google Maps API via Named Credential. Einstein Scoring integrated via Einstein Prediction Builder model 'SIU_Fraud_Likelihood' trained on historical SIU referral outcomes. LWC 'siuReferralBanner' uses @wire(getRecord) to reactively show/hide based on SIU_Referral_Recommended__c. Record-Triggered Flow 'RTF_SIUCaseCreation' handles Case creation with fault path logging. Permission Set Group 'SIUAnalyst_PSG' grants access to SIU-related fields.

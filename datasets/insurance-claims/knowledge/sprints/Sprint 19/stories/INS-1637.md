## Description

**Bug Report — Regression introduced in Sprint 18 deploy (2024-11-14)**

After the ERA inbound integration payload handler was refactored in INS-1601, Claim_Line__c records are no longer transitioning from 'Pending Repricing' to 'Adjudicated' when the downstream Fee Schedule engine returns an approved repriced amount.

**Steps to Reproduce:**
1. Create a Claim__c record with LOB = 'Medical' and Status__c = 'In Adjudication'.
2. Add at least two Claim_Line__c records with CPT codes 99213 and 93000; set Claim_Line__c.Status__c = 'Pending Repricing' on both.
3. Trigger the ERA inbound Platform Event (ERA_Inbound__e) with a valid 835 transaction payload containing matching NPI and TIN.
4. Observe Claim_Line__c.Status__c and Claim_Line__c.Repriced_Amount__c after the Platform Event fires.

**Expected Result:**
Claim_Line__c.Status__c transitions to 'Adjudicated' and Claim_Line__c.Repriced_Amount__c is populated with the value from the ERA payload.

**Actual Result:**
Claim_Line__c.Status__c remains 'Pending Repricing'. Claim_Line__c.Repriced_Amount__c is null. No error appears in the debug log; the Platform Event subscriber Apex trigger (ERA_InboundEventTrigger) fires but the DML update is silently skipped due to a null-check regression on Fee_Schedule__c lookup.

**Root Cause (identified):**
In ERA_InboundEventHandler.cls, the conditional block at line 142 checks `if (feeSchedule != null && feeSchedule.Effective_Date__c <= today)` but the Sprint 18 refactor introduced a variable shadowing issue — a local `feeSchedule` variable declared inside the for-loop shadows the outer query result, always evaluating as null.

**Affected Environments:** Full Sandbox (FS-01), UAT
**Regression Introduced:** INS-1601

## Acceptance Criteria

as per Description — fix verified when:
- Claim_Line__c.Status__c = 'Adjudicated' after ERA_Inbound__e fires with matching NPI/TIN
- Claim_Line__c.Repriced_Amount__c populated with correct dollar value from 835 payload
- Parent Claim__c.Status__c rolls up correctly to 'Ready for Payment' when all child Claim_Line__c records are 'Adjudicated'
- No regression on single-line claims (covered by existing test class ERA_InboundEventHandlerTest)
- Code coverage for ERA_InboundEventHandler.cls remains >= 85%

## Solution

Fix variable shadowing bug in ERA_InboundEventHandler.cls by renaming the inner loop variable from `feeSchedule` to `lineFeeSchedule`. Re-run all ClaimsProcessing Apex test classes. Deploy fix via change set to FS-01 and UAT. Add a dedicated unit test method `testMultiLineERAWithFeeScheduleLookup()` to ERA_InboundEventHandlerTest to prevent future regression. No Flow or LWC changes required.

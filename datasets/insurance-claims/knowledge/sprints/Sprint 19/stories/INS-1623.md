## Description

**Bug Report — Regression introduced in Sprint 18**

When an ERA (Electronic Remittance Advice) file is ingested for a Claim__c that has a secondary payer flagged via COB (Coordination of Benefits), the associated Claim_Line__c records fail to transition their Status__c field from 'Pending Payment' to 'Adjudicated'. The Payment__c record IS created correctly, but the Claim_Line__c records remain stale.

**Steps to Reproduce:**
1. Locate or create a Claim__c record with COB_Indicator__c = true and Secondary_Payer__c populated.
2. Ensure at least 2 Claim_Line__c records exist with Status__c = 'Pending Payment'.
3. Trigger ERA ingest via the DataIntegration middleware (CMS 835 file format).
4. Observe Payment__c record — created successfully with correct Paid_Amount__c.
5. Navigate to related Claim_Line__c records — Status__c remains 'Pending Payment'.

**Expected Behavior:** All Claim_Line__c.Status__c values should update to 'Adjudicated' and Claim_Line__c.Adjudication_Date__c should be stamped with the ERA processing timestamp.

**Actual Behavior:** Claim_Line__c.Status__c = 'Pending Payment' (unchanged). Claim_Line__c.Adjudication_Date__c = null.

**Root Cause (Preliminary):** The Record-Triggered Flow on Payment__c that performs the child Claim_Line__c update appears to be skipping execution when Payment__c.COB_Sequence__c = 'Secondary', likely due to an entry condition added in INS-1598 that was not scoped for secondary COB payments.

as per Description

## Solution

1. Identify the Record-Triggered Flow 'Payment_Adjudicate_ClaimLines_Flow' on Payment__c — review the entry condition filtering on COB_Sequence__c.
2. Update the flow entry criteria to include COB_Sequence__c IN ('Primary', 'Secondary', null) so secondary COB payments are no longer excluded.
3. Add a loop element to iterate over related Claim_Line__c records via Payment__c.Claim__r relationship and perform a bulk update of Status__c = 'Adjudicated' and Adjudication_Date__c = NOW().
4. Deploy a fix to the Apex invocable method 'ClaimLineAdjudicationService.updateLinesFromPayment()' to handle null COB_Sequence__c gracefully.
5. Add unit test coverage for COB secondary scenario (target: 90%+ coverage on affected class).
6. Validate in full regression suite against ERA ingest test harness.

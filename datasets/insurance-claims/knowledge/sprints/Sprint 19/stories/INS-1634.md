## Description

As a Claims Operations Manager, I need an Agentforce AI recommendation panel embedded in the FNOL intake screen so that coverage gap risks and potential SIU referral flags are surfaced automatically during first notice of loss capture, reducing manual review time and improving adjudication accuracy.

This is a net-new capability for Phase 2. The panel should invoke an Agentforce Agent Action that evaluates the incoming Claim__c against the linked Policy__c coverage terms and historical claim patterns.

## Acceptance Criteria

Given a Claims Operations Manager is completing the FNOL OmniScript for a new Claim__c:
When the 'Policy Lookup' step is completed and Policy__c.Policy_Status__c = 'Active':
Then the Agentforce recommendation panel renders inline within the OmniScript displaying at minimum: Coverage_Gap_Risk_Score (0–100), Recommended_Action (string), and Top_3_Risk_Factors (list).

Given the AI model evaluates Coverage_Gap_Risk_Score >= 75:
When the panel renders:
Then a prominent warning banner displays: 'High Coverage Gap Risk — Verify Policy__c.Coverage_Type__c and Policy__c.Deductible_Amount__c before proceeding' and the 'Submit FNOL' button is disabled until the Claims Ops Manager checks an acknowledgment checkbox.

Given the AI model returns a SIU_Referral_Flag = TRUE:
When the FNOL is submitted:
Then Claim__c.SIU_Referral_Required__c is automatically set to TRUE and a Task is created assigned to the SIU queue with Subject = 'AI-Flagged SIU Review — ' + Claim__c.Claim_Number__c.

Given Policy__c.LOB__c = 'Workers Compensation':
When the Agentforce panel evaluates coverage:
Then ICD-10 code validation is included in the risk assessment using Claim__c.Primary_ICD10_Code__c against the policy's covered diagnosis list.

Given the Agentforce API call fails or times out:
When the panel attempts to load:
Then a non-blocking inline message displays: 'AI recommendations temporarily unavailable — manual review required' and FNOL submission is NOT blocked.

-Given the panel displays a 'Auto-Adjudicate' button for low-risk claims (score < 25)-
-This feature descoped to INS-1660 pending AI model certification-

## Solution

1. Configure an Agentforce Agent `Claims_FNOL_Coverage_Advisor` with a custom Agent Action `Evaluate_Coverage_Gap_Risk` backed by an Apex class `CoverageGapRiskEvaluator` that queries Policy__c coverage terms and invokes the Einstein Prediction Service model `Coverage_Gap_Risk_Model_v2`.
2. Build an LWC `agentforceFnolPanel` that calls the Agent Action via the Agentforce API and renders the risk score, recommended action, and risk factors using a lightning-card layout.
3. Embed the LWC into the FNOL OmniScript as a Custom LWC step after the Policy Lookup step, passing Policy__c.Id and Claim__c draft fields as input parameters.
4. Implement conditional OmniScript navigation logic: if Coverage_Gap_Risk_Score >= 75, inject an acknowledgment Boolean element that must equal TRUE before the 'Submit FNOL' navigation action is enabled.
5. Create a Record-Triggered Flow `RTF_Claim_SIU_Flag_On_Submit` on Claim__c AFTER INSERT to check SIU_Referral_Required__c = TRUE and auto-create the SIU Task assigned to the SIU_Review_Queue.
6. Add a Custom Permission `CP_Agentforce_FNOL_Panel` and include in the Claims_Operations_Manager Permission Set Group to gate the LWC rendering.

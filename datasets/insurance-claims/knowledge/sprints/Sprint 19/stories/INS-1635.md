## Description

As a Broker Relations Coordinator, I need the system to automatically activate a Group_Service_Agreement__c and generate associated Policy__c records for each enrolled member when the DocuSign envelope for the GSA is completed and returned so that group onboarding is fully automated and eliminates the current 2–3 day manual activation lag.

This story covers the DocuSign webhook-to-Salesforce automation leg. Member Policy__c bulk creation is handled by a downstream batch job invoked from this flow.

## Acceptance Criteria

Given a Group_Service_Agreement__c.DocuSign_Envelope_Status__c transitions to 'Completed' via the DocuSign Connect webhook:
When the Platform Event `DocuSign_Status_Event__e` is received and processed:
Then Group_Service_Agreement__c.GSA_Status__c is updated from 'Pending Signature' to 'Active' and GSA_Effective_Date__c is set to the envelope completion timestamp.

Given Group_Service_Agreement__c.GSA_Status__c = 'Active' and Group_Service_Agreement__c.Enrolled_Member_Count__c > 0:
When the activation flow completes:
Then the Apex batch job `GSAMemberPolicyGenerationBatch` is enqueued with the GSA Id as a parameter to create Policy__c records for each enrolled member.

Given the batch job runs successfully:
When all Policy__c records are created:
Then each Policy__c.Policy_Status__c = 'Active', Policy__c.Group_Service_Agreement__c = GSA Id, Policy__c.LOB__c = Group_Service_Agreement__c.LOB__c, and Policy__c.Effective_Date__c = GSA_Effective_Date__c.

Given the DocuSign envelope is 'Declined' or 'Voided':
When the Platform Event is processed:
Then Group_Service_Agreement__c.GSA_Status__c = 'Signature Declined' and a Task is created for the Broker Relations Coordinator with Subject = 'GSA Signature Declined — Action Required: ' + GSA.Agreement_Number__c.

Given the GSA activation completes:
When Policy__c records are generated:
Then an automated notification email is sent to the Group Contact (Group_Service_Agreement__c.Primary_Contact__c) using Email Template `ET_GSA_Activation_Confirmation`.

## Solution

1. Configure DocuSign Connect webhook to fire a Platform Event `DocuSign_Status_Event__e` with fields: Envelope_Id, Status, Completion_Timestamp, Related_Record_Id (mapped to GSA Id via DocuSign merge field).
2. Build a Platform Event-Triggered Flow `PETF_DocuSign_GSA_Activation` that subscribes to `DocuSign_Status_Event__e`, retrieves the GSA record, and branches on envelope status: 'Completed' → update GSA fields; 'Declined'/'Voided' → set 'Signature Declined' and create Task.
3. In the 'Completed' branch, use a Flow Action to invoke Apex method `GSAActivationService.enqueueMemberPolicyBatch(gsaId)` which calls `Database.executeBatch(new GSAMemberPolicyGenerationBatch(gsaId), 50)`.
4. `GSAMemberPolicyGenerationBatch` queries GSA_Member__c junction records linked to the GSA and performs bulk insert of Policy__c records with mapped fields.
5. Add an Email Alert `EA_GSA_Activation_Confirmation` on Group_Service_Agreement__c triggered by GSA_Status__c = 'Active', using Email Template `ET_GSA_Activation_Confirmation` sent to Primary_Contact__c.
6. Create an Integration_Error_Log__c entry in the Platform Event flow's fault path to capture any processing failures with Error_Source__c = 'DocuSign GSA Activation'.

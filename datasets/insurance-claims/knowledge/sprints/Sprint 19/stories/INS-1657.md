## Description

As a Claims Operations Manager, I need a Policyholder 360 dashboard LWC embedded on the Policy__c record page so that I can view a policyholder's active and recent claims, payment disbursement history, and linked EOB documents in a single consolidated view, reducing the time spent navigating related lists during member inquiries. Extends INS-1621 (Policyholder 360 foundation component).

## Acceptance Criteria

Given a Claims Operations Manager opens a Policy__c record page,
When the Policyholder 360 LWC loads,
Then it displays all Claim__c records where Claim__c.Policy__c = current Policy__c.Id and Claim__c.Status__c IN ('Open', 'Pending Review', 'Adjudicated') in a data table sorted by Claim__c.Date_of_Service__c descending.

Given the policyholder has Payment__c records linked to their claims,
When the Payment History tab is selected within the LWC,
Then Payment__c records are displayed showing Payment__c.Amount__c, Payment__c.Payment_Date__c, Payment__c.Payment_Method__c, and Payment__c.Status__c for the last 24 months.

Given an EOB document exists on a Claim__c record (Explanation_of_Benefits__c linked via EOB.Claim__c),
When the EOB Documents tab is selected,
Then all Explanation_of_Benefits__c records are listed with EOB.Document_Date__c, EOB.Claim__c (hyperlinked), and a 'Download' action that opens the associated ContentVersion file.

Given the policy has no active claims,
When the Claims tab renders,
Then an empty state message 'No active claims on this policy' is displayed with a 'File New Claim (FNOL)' quick action button.

Given a user without the 'Claims_Read' Permission Set,
When they access the Policy__c record page,
Then the Policyholder 360 LWC is hidden via Permission Set Group visibility and replaced with a 'Insufficient Access' placeholder.

Given the LWC is loaded on a Policy__c with more than 200 related Claim__c records,
When the claims data table renders,
Then results are paginated at 25 records per page with next/previous navigation controls and total record count displayed.

## Solution

Build 'policyholderDashboard' LWC with three child components: 'claimsDataTable', 'paymentHistoryTable', and 'eobDocumentList'. Parent component uses @wire(getRecord) to fetch Policy__c context. Child components use @wire(getRelatedListRecords) for Claim__c and Payment__c, and a custom Apex controller ('PolicyholderDashboardController') with @AuraEnabled(cacheable=true) methods for EOB document retrieval including ContentVersion SOQL. Pagination implemented client-side with LWC reactive properties for currentPage and pageSize. 'File New Claim' quick action invokes the existing FNOL OmniScript via NavigationMixin. Permission Set Group 'Claims_Read_PSG' controls LWC visibility via targeted App Builder audience configuration. Component targets flexipage:recordPage on Policy__c object. Extend INS-1621 base CSS design tokens for visual consistency.

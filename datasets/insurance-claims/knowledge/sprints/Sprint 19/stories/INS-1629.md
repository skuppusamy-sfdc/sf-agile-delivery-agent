## Description

As a Claims Operations Manager, I need an Agentforce-powered FNOL (First Notice of Loss) intake OmniScript that guides claimants and claims representatives through structured loss capture while performing an AI-assisted real-time eligibility pre-check against the claimant's Policy__c, so that incomplete FNOL submissions are reduced and invalid claims against lapsed or excluded coverages are flagged before a Claim__c record is created.

## Acceptance Criteria

Given a Claims Representative launches the 'FNOL Intake' OmniScript
When they enter a Policy__c.Policy_Number__c on the first step
Then the OmniScript calls an Integration Procedure to validate: Policy__c.Status__c = 'Active', Policy__c.Effective_Date__c <= TODAY(), Policy__c.Expiration_Date__c >= TODAY(), and displays the policyholder name and coverage summary in a read-only review card.

Given Policy__c.Status__c = 'Lapsed' or Policy__c.Expiration_Date__c < TODAY()
When the policy lookup returns
Then the OmniScript displays a blocking error message: 'Policy is not active. FNOL cannot be submitted.' and prevents progression to the next step.

Given a valid active policy
When the Claims Representative enters the loss date, loss type, and loss description
Then Agentforce AI analyzes the loss description text and recommends: a Claim__c.Claim_Type__c value, up to 3 probable ICD-10 codes (for medical LOBs), and a preliminary SIU referral flag (Claim__c.SIU_Referral_Flag__c) if fraud indicators are detected with confidence > 0.75.

Given the Agentforce AI SIU flag recommendation
When the Claims Representative reviews the AI suggestions on the summary step
Then they can accept or override each recommendation, and any override is logged to Claim__c.AI_Override_Log__c with the representative's user ID and timestamp.

Given the FNOL OmniScript reaches the final confirmation step
When the Claims Representative submits
Then a Claim__c record is created with Status__c = 'FNOL Received', all AI-accepted fields populated, and a confirmation number (Claim__c.Claim_Number__c, auto-generated) is displayed to the representative.

Given any step in the OmniScript is left incomplete for > 15 minutes
When the session timeout triggers
Then OmniScript state is saved as a Draft_FNOL__c record linked to the Policy__c, allowing resume without data loss.

## Solution

1. Build a multi-step OmniScript 'FNOL_Intake_OS' with steps: Policy Lookup, Loss Details, Coverage Pre-Check, AI Recommendations Review, Confirm & Submit.
2. Create Integration Procedure 'IP_FNOL_PolicyEligibilityCheck' using DataRaptor Extract on Policy__c with validation logic in Set Values elements.
3. Integrate Agentforce Einstein AI action 'FNOL_ClaimClassifier' via an Agentforce Action step in the OmniScript, passing loss description and LOB as input, receiving Claim_Type, ICD10 suggestions, and SIU confidence score.
4. Configure the Agentforce topic 'Claims FNOL Assistant' in the Agentforce Agent Builder with grounding on Policy__c and Claim__c object schemas.
5. Build a DataRaptor Transform to map OmniScript output to Claim__c field API names for the Create Records action on submit.
6. Implement a 15-minute inactivity timer using OmniScript Script Actions calling an Apex method 'DraftFNOLService.saveDraft()' that upserts a Draft_FNOL__c record.
7. Add AI_Override_Log__c as a Long Text Area field on Claim__c and populate via DataRaptor Turbo Action on override events.
8. Deploy OmniScript to the 'Claims Representative' app page and configure Permission Set Group access.

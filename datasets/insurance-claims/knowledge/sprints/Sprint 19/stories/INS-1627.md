## Description

As a Claims Operations Manager, I need an active claims summary tile on the Policy__c record page that displays the count of open claims, total incurred amount, and total paid-to-date amount for the policy, so that I can quickly assess financial exposure during policyholder inquiries without running a separate report.

## Acceptance Criteria

Given a Policy__c record with one or more associated Claim__c records where Claim__c.Status__c IN ('Open', 'In Review', 'Pending Payment')
When the Claims Summary tile loads on the Policy__c record page
Then the tile displays: 'Open Claims: [count]', 'Total Incurred: $[SUM of Claim__c.Total_Incurred_Amount__c]', 'Total Paid: $[SUM of Claim__c.Total_Paid_Amount__c]', formatted as currency with two decimal places.

Given a Policy__c with NO open claims
When the tile loads
Then the tile displays 'No open claims on this policy.' with a green status indicator.

Given a Policy__c where Claim__c.Total_Incurred_Amount__c exceeds Policy__c.Coverage_Limit__c
When the tile renders
Then a yellow warning banner displays: 'Incurred amount approaching or exceeding coverage limit.'

Given a Claims Operations Manager user profile
When they click 'View All Claims' in the tile
Then a filtered list view of Claim__c records opens, pre-filtered to the current Policy__c.Id.

Given the tile is rendered on a Policy__c record belonging to a LOB of 'Workers Compensation'
When the tile loads
Then an additional row 'Indemnity Paid: $[SUM of Claim__c.Indemnity_Paid__c]' is displayed below Total Paid.

## Solution

1. Build an LWC 'policyClaimsSummaryTile' with an Apex controller method 'PolicyClaimsSummaryController.getClaimsSummary(String policyId)' that runs an aggregate SOQL query: SELECT COUNT(Id), SUM(Total_Incurred_Amount__c), SUM(Total_Paid_Amount__c), SUM(Indemnity_Paid__c) FROM Claim__c WHERE Policy__c = :policyId AND Status__c IN ('Open','In Review','Pending Payment').
2. Use @wire with refreshApex to keep data current.
3. Implement conditional rendering in the LWC template using if:true/if:false directives for empty state, coverage limit warning, and LOB-specific indemnity row.
4. Expose the LWC as a Lightning App Builder component with a 'policyId' design attribute.
5. Add the tile to the 'Policy 360 - Claims Ops' Lightning Record Page via App Builder, restricted to 'Claims Operations' Permission Set Group.
6. Cache the aggregate result with a 5-minute TTL using Platform Cache (Org partition) to reduce SOQL load on high-traffic pages.

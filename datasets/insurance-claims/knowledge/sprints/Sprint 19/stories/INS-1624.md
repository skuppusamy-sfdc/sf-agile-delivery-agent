## Description

As a Provider Relations Specialist, I need the Provider 360 FlexCard on the Provider__c (Person Account) record page to display active Provider_Network_Agreement__c (PNA) details — including network tier, effective dates, and associated Fee_Schedule__c — so that I can quickly assess a provider's contracting status without navigating to multiple related lists during a live call.

Extends INS-1589 (Provider 360 base FlexCard implementation).

## Acceptance Criteria

Given a Provider__c record where at least one Provider_Network_Agreement__c exists with PNA__c.Status__c = 'Active' and PNA__c.Effective_Date__c <= TODAY() and PNA__c.Termination_Date__c >= TODAY()
When the Provider 360 FlexCard loads on the Provider__c record page
Then the FlexCard displays a 'Network Agreements' section showing: PNA__c.Network_Name__c, PNA__c.Network_Tier__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, and the linked Fee_Schedule__c.Schedule_Name__c.

Given a Provider__c with multiple active PNA records
When the FlexCard loads
Then all active PNAs are displayed in a scrollable card list sorted by PNA__c.Effective_Date__c descending, with a visible count badge (e.g., '3 Active Agreements').

Given a Provider__c with NO active PNA records
When the FlexCard loads
Then the 'Network Agreements' section displays an empty-state message: 'No active network agreements found.' and a button labeled 'Initiate New PNA' that navigates to the PNA creation OmniScript.

-Given a Provider__c with a pending PNA (Status__c = 'Pending Signature'), display the pending agreement in a yellow warning banner- (deprecated — pending PNAs handled in INS-1601)

Given a Provider Relations Specialist user
When they click the Fee_Schedule__c.Schedule_Name__c link within the FlexCard
Then the Fee Schedule detail page opens in a new tab without navigating away from the Provider 360 page.

Given a Provider__c record where PNA__c.Termination_Date__c is within 30 days of TODAY()
When the FlexCard renders the agreement row
Then the row displays a red 'Expiring Soon' badge next to PNA__c.Termination_Date__c.

## Solution

1. Build a DataRaptor Extract 'DR_Extract_ActivePNA_ByProvider' querying Provider_Network_Agreement__c WHERE Provider__c = :recordId AND Status__c = 'Active' AND Effective_Date__c <= TODAY AND Termination_Date__c >= TODAY, joining to Fee_Schedule__c.
2. Create an Integration Procedure 'IP_Provider360_PNADetails' that calls the DataRaptor and applies the 30-day expiry logic using a Set Values element.
3. Update the existing 'Provider360_FlexCard' (OmniStudio FlexCard) to add a new 'Network Agreements' flyout card block bound to the Integration Procedure output.
4. Implement conditional rendering using FlexCard state conditions for empty-state display.
5. Use the FlexCard Action 'NavigateToURL' with target='_blank' for the Fee Schedule link.
6. Apply the 'Expiring_Soon' CSS class conditionally when days-to-termination < 30 (computed in Integration Procedure).
7. Assign FlexCard to the Provider__c Lightning Record Page via App Builder, visible to 'Provider Relations' Permission Set Group.

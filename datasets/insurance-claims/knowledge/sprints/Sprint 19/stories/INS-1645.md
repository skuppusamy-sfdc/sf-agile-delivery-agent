## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces active Provider_Network_Agreement__c (PNA) contract details, network tier classification, and effective date ranges so that I can quickly validate in-network status during real-time provider inquiries without navigating to multiple related lists.

Extends INS-1618 (Provider 360 base FlexCard scaffold).

## Acceptance Criteria

Given a Provider__c record with one or more related Provider_Network_Agreement__c records,
When the Provider 360 FlexCard loads on the record page,
Then the FlexCard DataRaptor Extract retrieves all PNA records where PNA__c.Status__c = 'Active' AND PNA__c.Effective_Date__c <= TODAY() AND PNA__c.Termination_Date__c >= TODAY() and displays them in a tabular child card section.

Given an active PNA exists,
When the FlexCard renders the PNA section,
Then the following fields are displayed: PNA__c.Contract_Number__c, PNA__c.Network_Tier__c, PNA__c.LOB__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, and a derived badge showing 'In-Network' (green) or 'Out-of-Network' (red) based on current date evaluation.

Given no active PNA records exist for the provider,
When the FlexCard PNA section renders,
Then a contextual message 'No Active Network Agreements Found — Contact Network Contracting' is displayed and the in-network badge shows 'Out-of-Network' (red).

Given a PNA with PNA__c.Termination_Date__c within 30 days of TODAY(),
When the FlexCard renders,
Then a yellow warning badge 'Expiring Soon' is displayed adjacent to the contract row with the exact termination date.

-Given the FlexCard includes a 'Request Contract Renewal' quick action button linking to Conga CLM template ID CT-2201- (deprecated — renewal workflow moved to INS-1659)

Given a Provider Relations Specialist views the FlexCard,
When they click the NPI hyperlink in the PNA row,
Then a modal displays the NPPES integration lookup result for that NPI, showing Provider__c.NPI__c validation status from the last NPPES sync.

## Solution

Build OmniStudio FlexCard 'ProviderNetworkAgreementCard' as a child card nested within the existing Provider360FlexCard (INS-1618). Use a DataRaptor Extract (DRE_ActivePNAByProvider) with a SOQL source querying Provider_Network_Agreement__c filtered by Provider__c Id, Status__c, Effective_Date__c, and Termination_Date__c. Add a DataRaptor Transform to compute the 'Expiring Soon' flag (DATEDIFF logic). Deploy FlexCard via OmniStudio package to the Provider__c Lightning Record Page using App Builder. NPPES modal uses a separate LWC (nppesLookupModal) wired to a named credential callout. Permission Set Group 'ProviderRelations_PSG' grants FlexCard visibility.

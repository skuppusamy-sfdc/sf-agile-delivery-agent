## Description

As a Credentialing Specialist, I need the new Credentialing Case intake OmniScript to automatically query the CAQH ProView API using the provider's NPI so that attestation data — including board certifications, malpractice history, and DEA license status — is pre-populated on the Credentialing_Case__c record, eliminating manual data entry and reducing credentialing cycle time.

## Acceptance Criteria

Given a Credentialing Specialist creates a new Credentialing_Case__c via the OmniScript and enters a valid Provider__c.NPI__c,
When they advance past Step 1 ('Provider Identification'),
Then an Integration Procedure call is made to the CAQH ProView API and the response populates: Credentialing_Case__c.Board_Certification__c, Credentialing_Case__c.DEA_License_Status__c, Credentialing_Case__c.Malpractice_Claims_Count__c, and Credentialing_Case__c.CAQH_Profile_Status__c.

Given the CAQH API returns a profile status of 'Not Authorized',
When the OmniScript processes the response,
Then an inline error message 'Provider has not authorized CAQH data release. Manual credentialing required.' is displayed and Credentialing_Case__c.Manual_Review_Required__c is set to true.

Given the CAQH API call succeeds and data is pre-populated,
When the Credentialing Specialist reviews the pre-filled fields,
Then each auto-populated field displays a 'Source: CAQH' badge and Credentialing_Case__c.Data_Source__c = 'CAQH Auto-Verified'.

Given the CAQH API is unreachable (timeout or HTTP 5xx),
When the integration step executes,
Then the OmniScript logs the failure to Integration_Error_Log__c with fields Error_Source__c = 'CAQH', Error_Code__c, and Timestamp__c, and allows the Credentialing Specialist to proceed with manual entry.

-Given the provider's CAQH attestation is older than 180 days, block case creation and require re-attestation.- (Requirement removed — re-attestation enforcement deferred to INS-1679 per Credentialing Ops team decision)

Given a Credentialing_Case__c is saved with CAQH-sourced data,
When the record is created,
Then Credentialing_Case__c.CAQH_Verification_Date__c is stamped with the current date and Credentialing_Case__c.Status__c = 'Data Verified – Pending Review'.

## Solution

Build an Integration Procedure ('IP_CAQH_ProView_Lookup') using OmniStudio Integration Procedures with a Remote Action step calling the CAQH ProView REST API via a Named Credential ('CAQH_ProView_NC'). The IP accepts NPI as input, constructs the API request, and maps response JSON fields to Credentialing_Case__c field API names using a DataRaptor Transform. Embed IP call in the FNOL OmniScript ('Credentialing_Case_Intake_v1') as an Action step triggered on NPI field blur event. Error handling uses an OmniScript Conditional View to display inline messages. Integration_Error_Log__c inserts handled via a separate Apex class ('CAQHErrorLogger') invoked from the IP's error path. Named Credential configured with OAuth 2.0 using CAQH sandbox credentials in lower environments and production credentials managed via Salesforce Secrets.

## Description

**Bug Report — Performance Regression**

The nightly batch adjudication job 'BatchClaimLineAdjudication' is throwing System.LimitException: Too many query rows: 50001 in the Production-equivalent Full Sandbox during Sprint 19 regression testing. This regression was introduced after the Fee Schedule Engine refactor in Sprint 17.

**Steps to Reproduce:**
1. Load Full Sandbox with ≥ 48,000 Claim_Line__c records in Status__c = 'Pending Adjudication'.
2. Execute BatchClaimLineAdjudication via Anonymous Apex: Database.executeBatch(new BatchClaimLineAdjudication(), 200);
3. Monitor Apex Jobs — job fails at the execute() method with LimitException on batch chunk 4 or later.
4. Review debug logs — SOQL in execute() fetches related Fee_Schedule__c, PNA, and Claim__c parent records without scope filtering, causing row bloat.

**Expected:** Batch processes all Claim_Line__c records in scope without hitting SOQL row limits regardless of org data volume.
**Actual:** Job fails after ~40,000 rows processed; remaining lines are not adjudicated, causing payment delays.

**Root Cause (identified):** The execute() method re-queries Fee_Schedule__c and Provider_Network_Agreement__c records for each batch chunk without leveraging the start() scope — pulling full table datasets per chunk rather than scoped lookups.

## Solution

Refactored BatchClaimLineAdjudication.execute() to eliminate unscoped SOQL queries: (1) Extracted all unique Provider__c and Fee_Schedule__c IDs from the batch scope List<Claim_Line__c> using a Set, then performed a single IN-clause SOQL query per related object per chunk. (2) Built in-memory Maps (Map<Id, Fee_Schedule__c> and Map<Id, Provider_Network_Agreement__c>) within execute() for O(1) lookups during line processing. (3) Reduced default batch size from 200 to 150 as an additional guard. (4) Added a Custom Setting 'Batch_Adjudication_Config__c' with a field Max_Chunk_Size__c to allow ops team to tune batch size without code deployment. (5) Added @TestVisible annotation to internal maps for unit test assertions. Full regression suite passed in Full Sandbox at 95,000 Claim_Line__c records without LimitException.

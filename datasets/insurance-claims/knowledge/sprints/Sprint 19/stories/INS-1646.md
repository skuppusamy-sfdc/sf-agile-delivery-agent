## Description

As a Credentialing Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record and notify the Provider Network team when a Credentialing_Case__c transitions to a status of 'Lapsed' so that downstream network agreements can be reviewed for suspension and regulatory compliance timelines are met (CMS 30-day notification requirement).

## Acceptance Criteria

Given a Credentialing_Case__c record with Status__c changing from any prior value to 'Lapsed',
When the record is saved,
Then a Record-Triggered Flow fires after save and creates a new Provider_Change_Notification__c with: PCN__c.Type__c = 'Credentialing Lapse', PCN__c.Provider__c = Credentialing_Case__c.Provider__c, PCN__c.Effective_Date__c = TODAY(), PCN__c.Status__c = 'Pending Review', PCN__c.Source_Case__c = Credentialing_Case__c.Id.

Given the PCN record is created,
When PCN__c.Status__c = 'Pending Review',
Then a Platform Event (CredentialingStatusChange__e) is published with payload fields: Provider_Id__c, Case_Id__c, Change_Type__c = 'Lapsed', Notification_Date__c so that subscribing processes (network suspension flow) can act asynchronously.

Given the Platform Event is published,
When the subscribing Record-Triggered Flow (RTF_NetworkSuspensionOnLapse) receives the event,
Then all related Provider_Network_Agreement__c records where PNA__c.Provider__c matches and PNA__c.Status__c = 'Active' are updated to PNA__c.Status__c = 'Under Review' with PNA__c.Review_Reason__c = 'Credentialing Lapse — Auto-Flagged'.

Given the PCN is created,
When the creation timestamp is recorded,
Then an automated email notification is sent to the assigned Provider Network Specialist (via Flow Email Action using template 'PCN_CredentialingLapse_Alert') within 15 minutes of the status change.

Given a Credentialing_Case__c transitions to 'Lapsed' but a PCN with PCN__c.Type__c = 'Credentialing Lapse' already exists for the same provider within the last 90 days,
When the flow evaluates duplicate prevention logic,
Then no duplicate PCN is created and a Chatter post is added to the existing PCN record noting the re-lapse event.

-Given the PCN triggers an automatic network termination case- (deprecated — auto-termination requires manual approval per compliance review INS-1640)

## Solution

Implement Record-Triggered Flow 'RTF_PCNOnCredentialingLapse' set to run after save on Credentialing_Case__c when Status__c is changed to 'Lapsed'. Flow includes a Get Records element to check for existing PCN duplicates within 90-day window. Publish Platform Event CredentialingStatusChange__e via Flow Publish action. Separate subscriber Flow 'RTF_NetworkSuspensionOnLapse' handles PNA bulk update via Collection Loop. Email template stored in Org-Wide Email Template folder 'Credentialing Alerts'. Apex invocable method used for Chatter post on duplicate scenario. All flows deployed via unlocked package. New Custom Permission 'Bypass_PCN_Duplicate_Check' added to Admin Permission Set for testing overrides.

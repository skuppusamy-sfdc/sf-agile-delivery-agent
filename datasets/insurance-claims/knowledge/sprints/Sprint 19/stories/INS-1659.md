## Description

**Bug Report — Regression from INS-1629**

After ERA batch posting completes, the Provider 360 FlexCard displaying Claim_Line__c adjudication status does not refresh to reflect updated values. The stale data persists until a full page reload is performed manually.

**Steps to Reproduce:**
1. Navigate to a Provider (Person Account) record with at least one associated Claim__c in 'Adjudicated' status.
2. Open the Provider 360 FlexCard panel — note Claim_Line__c.Adjudication_Status__c shows 'Pending'.
3. Trigger an ERA batch job via the PaymentEngine integration (or simulate via Developer Console).
4. Observe the FlexCard after batch completes — status still shows 'Pending'.
5. Manually reload the page — status now correctly shows 'Adjudicated'.

**Expected:** FlexCard automatically refreshes Claim_Line__c.Adjudication_Status__c to 'Adjudicated' upon ERA posting without manual page reload.
**Actual:** FlexCard displays stale 'Pending' status; real-time refresh does not occur.

**Environment:** Full Sandbox | Salesforce API v59.0 | OmniStudio Winter '24 package

## Solution

Root cause identified: the OmniStudio DataRaptor Transform feeding the Provider 360 FlexCard was not subscribed to the ERA_Posted__e Platform Event. Fix involves: (1) Adding a Platform Event subscriber in the FlexCard JSON to listen for ERA_Posted__e and invoke a DataRaptor Extract refresh on Claim_Line__c filtered by Provider__c. (2) Updating the DataRaptor Extract 'DR_Get_ClaimLines_ByProvider' to include Claim_Line__c.Adjudication_Status__c, Claim_Line__c.Paid_Amount__c, and Claim_Line__c.ERA_Reference__c fields. (3) Deploying updated FlexCard version 3.2 to Provider 360 Lightning App Page. Regression test against INS-1629 acceptance criteria confirmed passing.

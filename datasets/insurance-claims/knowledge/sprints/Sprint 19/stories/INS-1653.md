## Description

As a Broker Relations Coordinator, I need automated reminder tasks and email notifications generated at 90, 60, and 30 days before a Group_Service_Agreement__c expiration date so that I can proactively initiate renewal conversations with employer groups and avoid coverage lapses for enrolled members.

## Acceptance Criteria

Given a Group_Service_Agreement__c record where GSA.Status__c = 'Active' and GSA.Expiration_Date__c is 90 days from today,
When the nightly scheduled flow runs,
Then a Task is created with Subject = 'GSA Renewal Reminder – 90 Days', assigned to the GSA.Broker_Relations_Coordinator__c user, with ActivityDate = today and Priority = 'Normal'.

Given the same GSA reaches 60 days before expiration,
When the scheduled flow runs,
Then a second Task is created with Subject = 'GSA Renewal Reminder – 60 Days' and Priority = 'High', and an email alert is sent to GSA.Broker_Relations_Coordinator__c using the 'GSA_60Day_Renewal_Alert' email template.

Given the GSA reaches 30 days before expiration,
When the scheduled flow runs,
Then a Task with Priority = 'High' and Subject = 'GSA Renewal Reminder – 30 Days' is created AND GSA.Renewal_Stage__c is updated to 'Renewal Imminent'.

Given a GSA where GSA.Status__c = 'Expired' or GSA.Status__c = 'Terminated',
When the scheduled flow evaluates the record,
Then no reminder tasks are created for that GSA.

Given a renewal reminder task is created,
When the Broker Relations Coordinator completes the task,
Then GSA.Last_Renewal_Contact_Date__c is updated to the task completion date via a Record-Triggered Flow.

-Given GSA.Auto_Renew__c = true, suppress all reminder tasks.- (Deprecated — auto-renew suppression logic removed per compliance team request INS-1638)

## Solution

Implement a Scheduled Flow ('GSA_Renewal_Reminder_Scheduler') configured to run daily at 6:00 AM. Flow queries Group_Service_Agreement__c where Status__c = 'Active' and Expiration_Date__c IN (TODAY+90, TODAY+60, TODAY+30) using Get Records elements with separate branches per milestone. Each branch creates a Task record via Create Records element and, for the 60-day milestone, invokes an Apex Action (SendTemplatedEmailAction) referencing the 'GSA_60Day_Renewal_Alert' email template stored in Salesforce. A separate Record-Triggered Flow ('GSA_Task_Completion_Tracker') fires on Task update where Status changes to 'Completed' and Task.WhatId references a Group_Service_Agreement__c, updating GSA.Last_Renewal_Contact_Date__c. Custom metadata type 'GSA_Reminder_Config__mdt' stores milestone day values to allow admin configuration without code changes.

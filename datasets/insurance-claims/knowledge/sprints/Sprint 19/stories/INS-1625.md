## Description

As a Credentialing Specialist, I need a Provider_Change_Notification__c (PCN) record to be automatically created and routed to the Provider Network team when a Credentialing_Case__c transitions to Status__c = 'Approved', so that downstream network onboarding and system updates (CAQH sync, NPI validation via NPPES) are triggered without manual handoff delays.

## Acceptance Criteria

Given a Credentialing_Case__c record where Status__c is updated to 'Approved'
When the record save completes
Then a new Provider_Change_Notification__c is created with: PCN__c.Type__c = 'Credentialing Approval', PCN__c.Provider__c = Credentialing_Case__c.Provider__c, PCN__c.Effective_Date__c = TODAY(), PCN__c.Status__c = 'Pending Review', PCN__c.Source_Case__c = Credentialing_Case__c.Id.

Given the PCN is created
When PCN__c.Status__c = 'Pending Review'
Then an Omni-Channel work item is pushed to the 'Provider Network Queue' and the assigned Provider Network Specialist receives a Bell notification with the PCN record link.

Given a Credentialing_Case__c that transitions to 'Approved' but Provider__c.NPI__c is null
When the automation runs
Then the PCN is created with PCN__c.Flag_NPI_Missing__c = true and a Task is created assigned to the Credentialing Specialist with Subject = 'NPI required before PCN can be processed'.

Given a Credentialing_Case__c that is re-opened and re-approved (second approval cycle)
When the automation runs
Then a NEW PCN is created (duplicate not blocked) and the previous PCN linked to the same Credentialing_Case__c is updated to PCN__c.Status__c = 'Superseded'.

-Given PCN creation, also trigger CAQH outbound callout synchronously- (deprecated — CAQH sync moved to async Platform Event in INS-1612)

## Solution

1. Implement a Record-Triggered Flow 'Credentialing_Case_Approval_Flow' on Credentialing_Case__c, triggering AFTER UPDATE when Status__c changes to 'Approved'.
2. Flow creates Provider_Change_Notification__c via Create Records element with mapped fields.
3. Add a Decision element to check Provider__c.NPI__c — if null, create a Task record and set PCN__c.Flag_NPI_Missing__c = true.
4. Add a Get Records element to find existing open PCNs for the same Credentialing_Case__c and a Loop + Update Records to set those to 'Superseded'.
5. Publish a Platform Event 'PCN_Created__e' with PCN Id payload to trigger the Omni-Channel routing logic via a separate Platform Event-Triggered Flow.
6. Unit test: Apex test class 'CredentialingApprovalFlowTest' covering happy path, null NPI, and re-approval scenarios.

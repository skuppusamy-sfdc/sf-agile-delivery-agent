## Description

**Bug Report — Performance Regression**

The Explanation_of_Benefits__c PDF generation process (invoked via Conga CLM template 'EOB_Standard_v3') is timing out for claims with 20 or more Claim_Line__c child records. This regression was introduced after the Claim_Line__c object received 4 new formula fields in Sprint 17 (INS-1572).

**Steps to Reproduce:**
1. Identify or create a Claim__c record with exactly 22 Claim_Line__c child records, each with CPT codes, ICD-10 diagnosis codes, and Repriced_Amount__c populated.
2. Navigate to the Claim__c record and click 'Generate EOB' quick action.
3. Monitor the Conga merge job and Apex CPU time in the debug log.

**Expected Result:**
EOB PDF is generated and attached to Explanation_of_Benefits__c.Document_URL__c within 30 seconds. Apex CPU time < 10,000ms.

**Actual Result:**
Conga merge job fails after ~12 minutes with 'Apex CPU time limit exceeded'. Debug log shows the EOB_GenerationHelper.cls method `buildLineItemMatrix()` consuming 9,800–11,200ms due to nested SOQL loops iterating over Claim_Line__c records and re-querying Fee_Schedule__c and Provider__c for each line.

**Impact:**
- Approximately 340 claims per day exceed the 20-line threshold (per Claims Ops Manager report)
- Manual workaround: Claims Operations team splitting claims before EOB generation — adding ~15 min per claim

**Root Cause (suspected):**
The four new formula fields added in INS-1572 reference cross-object lookups (Claim_Line__c → Fee_Schedule__c → Provider_Network_Agreement__c), which appear to be forcing additional SOQL queries inside the `buildLineItemMatrix()` loop.

## Acceptance Criteria

as per Description — fix verified when:
- EOB PDF generation completes in <= 30 seconds for claims with up to 50 Claim_Line__c records
- Apex CPU time for EOB_GenerationHelper.cls `buildLineItemMatrix()` method < 5,000ms for 50-line test claim
- Explanation_of_Benefits__c.Document_URL__c is populated after successful generation
- Explanation_of_Benefits__c.Generation_Status__c = 'Completed' (not 'Failed')
- No regression on claims with 1–5 Claim_Line__c records
- Load test with 100 concurrent EOB generation requests passes in Full Sandbox

## Solution

Refactor EOB_GenerationHelper.cls `buildLineItemMatrix()` to eliminate nested SOQL by pre-fetching all required Claim_Line__c records with a single SOQL query including cross-object fields (Fee_Schedule__r.Rate__c, Fee_Schedule__r.PNA__r.Network_Tier__c) in the SELECT clause. Replace formula field cross-object lookups on Claim_Line__c (INS-1572 fields) with stored Apex-calculated values populated via a Record-Triggered Flow on Claim_Line__c after-save, eliminating runtime formula evaluation overhead during PDF merge. Update Conga template data mapping to reference the new stored fields. Performance benchmarked in Full Sandbox before UAT promotion.

## Description

As a Provider Network Specialist, I need an automated workflow triggered when a Network_Termination_Case__c is moved to 'Approved' status so that the affected provider is notified via DocuSign termination letter, all associated Provider_Network_Agreement__c records are updated to 'Terminated' status, and a compliance audit record is generated — ensuring we meet CMS-mandated provider termination notification timelines and maintain a complete audit trail.

## Acceptance Criteria

Given a Network_Termination_Case__c.Status__c is updated to 'Approved' by a user with the 'Network_Termination_Approver' Permission Set,
When the Record-Triggered Flow fires,
Then a DocuSign envelope is sent to Provider__c.Email (sourced via Network_Termination_Case__c.Provider__c lookup) using the 'Provider_Termination_Letter_v2' DocuSign template within 15 minutes of status change.

Given the DocuSign envelope is sent,
When the DocuSign Platform Event (DocuSign__DocuSign_Status__e) fires with status 'Sent',
Then Network_Termination_Case__c.Notification_Sent_Date__c is stamped with the envelope send timestamp and Network_Termination_Case__c.Notification_Method__c = 'DocuSign'.

Given the Network_Termination_Case__c is approved,
When the automation runs,
Then all Provider_Network_Agreement__c records where PNA.Provider__c = Network_Termination_Case__c.Provider__c AND PNA.Status__c = 'Active' are bulk-updated to PNA.Status__c = 'Terminated' and PNA.Termination_Date__c = Network_Termination_Case__c.Effective_Termination_Date__c.

Given PNA records are terminated,
When the status update completes,
Then a Compliance_Audit_Record__c is created with fields: Audit_Type__c = 'Provider Termination', Related_Case__c = Network_Termination_Case__c.Id, Provider__c = termination case provider, Termination_Effective_Date__c, Affected_PNA_Count__c = count of terminated PNAs, Initiated_By__c = running user, and Audit_Timestamp__c = NOW().

Given the Effective_Termination_Date__c is fewer than 30 days from the approval date,
When the workflow evaluates the timeline,
Then a Warning Chatter post is made on the Network_Termination_Case__c record tagging the assigned Compliance_Officer__c user with message: 'Termination timeline is below 30-day CMS minimum. Manual compliance review required.'

-Given the provider has open Claim__c records, block PNA termination until claims are resolved.- (Deprecated — claims hold logic decoupled to separate process per INS-1643 decision log)

Given the DocuSign send fails (envelope error),
When the failure Platform Event is received,
Then Network_Termination_Case__c.Status__c is reverted to 'Notification Failed', an error Task is created assigned to the owning Provider Network Specialist, and Integration_Error_Log__c is updated with Error_Source__c = 'DocuSign', Error_Message__c, and Case_Id__c.

## Solution

Implement a Record-Triggered Flow ('NTC_Approval_Automation') on Network_Termination_Case__c, entry condition: Status__c changes to 'Approved'. Flow path 1: invoke DocuSign Flow Action ('Send_DocuSign_Envelope') with template ID sourced from Custom Metadata 'DocuSign_Template_Config__mdt'. Path 2: Apex-invocable action ('TerminatePNABatch') triggers a Queueable Apex class to bulk-update PNA records, respecting governor limits for large provider networks. Path 3: Create Records element creates Compliance_Audit_Record__c. Path 4: decision element checks timeline; if < 30 days, ConnectApi.ChatterFeeds Apex action posts Chatter. DocuSign Platform Event subscriber Flow ('DocuSign_Status_Handler') handles send confirmation and failure callbacks, updating Network_Termination_Case__c accordingly. All Apex classes covered by minimum 85% test coverage. Custom Permission 'Network_Termination_Approver' validated in Flow entry condition via $Permission global variable to prevent unauthorized status transitions.

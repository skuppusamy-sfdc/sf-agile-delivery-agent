## Description

As a Claims Operations Manager, I need an Agentforce AI assistant embedded in the Claims Intake console that can parse a FNOL conversation transcript (from voice-to-text or chat) and automatically pre-populate key Claim__c fields, so that claims intake agents can reduce manual data entry time by at least 40% and decrease FNOL-to-claim-creation cycle time from an average of 12 minutes to under 7 minutes.

The assistant should extract: claimant identity (matched to Policy__c), incident date/time, loss description, LOB classification, and initial reserve estimate, and present them to the agent for confirmation before saving.

## Acceptance Criteria

Given an agent pastes or uploads a FNOL transcript into the Agentforce Claims Intake panel,
When the AI assistant processes the transcript,
Then it returns a structured JSON payload populating: Claim__c.Policy__c (matched via Policy__c.Policy_Number__c extracted from transcript), Claim__c.Incident_Date__c, Claim__c.Loss_Description__c, Claim__c.LOB__c, and Claim__c.Initial_Reserve__c.

Given the AI identifies a Policy_Number__c in the transcript,
When the number matches exactly one active Policy__c record (Policy__c.Status__c = 'Active'),
Then Claim__c.Policy__c is pre-populated and the matched policy name is displayed in a confirmation banner.

Given the AI cannot confidently match a policy number (confidence score < 0.85),
When the pre-population result is returned,
Then Claim__c.Policy__c is left blank, a warning banner reads 'Policy match uncertain — please verify manually', and the raw extracted text is shown in a side panel.

Given the AI classifies LOB from the transcript,
When the classification is complete,
Then Claim__c.LOB__c is pre-populated with one of the picklist values: 'Commercial', 'Medicare Advantage', 'Medicaid', 'Workers Comp', 'Auto' — no free-text LOB values are accepted.

Given the agent reviews the pre-populated fields,
When the agent clicks 'Confirm & Create Claim',
Then a new Claim__c record is created with Status__c = 'FNOL Received' and FNOL_Source__c = 'Agentforce AI', and the original transcript is attached as a ContentDocument linked to the Claim__c record.

Given a transcript contains potential fraud indicators (e.g., keywords matching SIU watchlist terms),
When the AI processes the transcript,
Then Claim__c.SIU_Flag__c is set to TRUE and a task is auto-created and assigned to the SIU review queue.

## Solution

1. Configure an Agentforce Agent (Einstein Copilot) with a custom Agent Action 'Parse FNOL Transcript' backed by an Apex Action class (FNOLTranscriptParserAction.cls) that calls the Einstein LLM API via ConnectApi.EinsteinLLM with a structured prompt template stored in a custom Prompt Template (FNOL_Intake_Prompt__mdt).
2. The Apex Action returns a parsed DTO object; a Lightning Web Component (fnolIntakeAssistant) renders the confirmation UI with editable pre-populated fields and a 'Confirm & Create Claim' button.
3. On confirmation, an Invocable Apex method (ClaimCreationService.createFromFNOL()) creates the Claim__c record, sets FNOL_Source__c = 'Agentforce AI', and calls ContentDocumentLink insert to attach the transcript.
4. SIU keyword detection uses a Custom Metadata Type (SIU_Keyword_Config__mdt) list evaluated within the Apex Action; if matched, sets SIU_Flag__c = TRUE and creates a Task via a sub-flow called from the LWC post-save.
5. Store confidence threshold (0.85) in a Custom Setting (AgentforceConfig__c.FNOL_Confidence_Threshold__c) for runtime tunability without deployment.
6. Instrument with Salesforce Shield Event Monitoring to log all AI-assisted claim creations for audit trail compliance.

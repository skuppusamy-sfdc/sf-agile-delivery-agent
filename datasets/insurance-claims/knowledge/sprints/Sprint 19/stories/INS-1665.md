## Description

As a Provider Relations Specialist, I need an automated workflow on the Network_Termination_Case__c object that generates and sends a Provider Change Notification (PCN) letter to the affected provider via DocuSign, captures their acknowledgement signature, and updates the termination case status accordingly so that we meet the contractually mandated 90-day termination notice requirement and maintain an auditable compliance record without manual document handling.

Extends INS-1641 (Provider Termination Case object model and status picklist build).

## Acceptance Criteria

Given a Network_Termination_Case__c record where Status__c is updated to 'Termination Notice Issued' by a Provider Relations Specialist or Compliance Officer,
When the status change is saved,
Then a Record-Triggered Flow fires and initiates a DocuSign envelope using the 'PCN_Termination_Letter_v2' template, addressed to Provider__r.Email (primary) with Provider__r.NPI__c and Provider_Network_Agreement__c.Contract_Number__c merged into the template.

Given the DocuSign envelope is successfully sent,
When the envelope send confirmation is received,
Then Network_Termination_Case__c.PCN_Sent_Date__c is stamped with today's date, PCN_DocuSign_Envelope_Id__c is populated, and Termination_Notice_Deadline__c is calculated as PCN_Sent_Date__c + 90 days.

Given the provider completes and signs the DocuSign PCN envelope,
When the DocuSign callback webhook fires (via Connected App),
Then Network_Termination_Case__c.PCN_Acknowledged__c is set to TRUE, PCN_Acknowledgement_Date__c is stamped, Status__c transitions to 'Acknowledgement Received', and the signed PDF is saved as a Salesforce File (ContentDocument) linked to the Network_Termination_Case__c record.

Given the DocuSign envelope has been sent but not signed within 15 days,
When the nightly reminder scheduled flow runs,
Then an automated follow-up email is sent to Provider__r.Email using template 'PCN_Reminder_15Day' and a Task is created on the Network_Termination_Case__c assigned to the OwnerId with Subject = 'PCN Acknowledgement Overdue — Follow Up Required'.

Given Network_Termination_Case__c.Termination_Notice_Deadline__c is reached and PCN_Acknowledged__c = FALSE,
When the nightly scheduled flow evaluates the record,
Then Status__c is set to 'Escalated - No Acknowledgement', a high-priority Case is auto-created and routed to the Compliance Officer queue, and a Provider_Change_Notification__c (PCN) child record is created with PCN.Status__c = 'Unacknowledged - Escalated'.

Given any DocuSign API failure during envelope initiation,
When the flow encounters the error,
Then the error is logged to a DocuSign_Error_Log__c record with fields: Error_Message__c, Envelope_Template__c, Related_Case__c, Timestamp__c, and the Network_Termination_Case__c.Status__c is rolled back to its prior value with an error banner surfaced to the initiating user.

Given a Network_Termination_Case__c where Termination_Type__c = 'For Cause',
When the PCN workflow initiates,
Then the DocuSign template used is 'PCN_ForCause_Letter_v1' (separate from standard termination) and the notice period is set to 30 days instead of 90 days per contractual policy.

## Solution

Implement a Record-Triggered Flow on Network_Termination_Case__c (fires After Save, when Status__c changes to 'Termination Notice Issued'). Flow calls an Invocable Apex action 'DocuSignPCNEnvelopeAction' that uses the DocuSign eSignature REST API (via named credential 'DocuSign_Prod') to create and send the envelope. Template selection and notice period calculation are driven by Custom Metadata Type PCN_Config__mdt keyed on Termination_Type__c. DocuSign callback is handled by a public REST Apex endpoint (without sharing, IP-restricted) that processes the webhook payload, updates case fields, and saves the signed document via ContentVersion insert. A separate Scheduled Flow runs nightly at 06:00 AM to evaluate overdue acknowledgements and deadline breaches. All DocuSign API interactions are logged to DocuSign_Error_Log__c via a dedicated error-handling subflow. Permission Set 'PCN_Workflow_Admin' grants Provider Relations Specialists and Compliance Officers access to Network_Termination_Case__c and PCN fields. Full integration tested in Partial Sandbox with DocuSign Developer account.

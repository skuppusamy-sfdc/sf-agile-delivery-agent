## Description

As a Claims Operations Manager, I need an Agentforce-powered FNOL triage assistant embedded in the claim intake OmniScript so that when a new FNOL is submitted (via phone-to-digital or self-service portal), the AI agent automatically pre-classifies the claim type (Medical, Dental, Vision, Behavioral Health), assigns an initial severity score (Low / Medium / High / Critical), and recommends the appropriate adjudication pathway — reducing average intake time by at least 40% and ensuring high-severity claims are escalated to senior adjusters immediately.

The Agentforce agent will be grounded on Claim__c historical data (last 24 months) and will use Einstein Prompt Builder templates for classification and severity scoring. The agent must explain its reasoning in plain language visible to the adjuster.

## Acceptance Criteria

Given a new FNOL is submitted via the FNOL_Intake OmniScript,
When the adjuster completes the 'Incident Description' step and clicks 'Analyze with AI',
Then the Agentforce agent returns a claim type classification within 5 seconds, populated into Claim__c.LOB__c (pre-fill, editable), with a confidence score displayed as a percentage badge.

Given the Agentforce agent classifies the claim,
When the severity scoring model evaluates incident description, claimant history, and provider type,
Then Claim__c.Initial_Severity__c is pre-filled with one of: 'Low', 'Medium', 'High', 'Critical', and the agent displays a 2–3 sentence plain-language rationale visible in the OmniScript UI panel 'AI Triage Summary'.

Given Claim__c.Initial_Severity__c = 'Critical' (AI-assigned),
When the FNOL is saved,
Then a Record-Triggered Flow immediately creates a Task assigned to the 'Senior_Adjuster_Queue' with Subject = 'CRITICAL FNOL — Immediate Review Required' and sets Claim__c.Priority__c = 'Critical'.

Given the adjuster disagrees with the AI classification,
When the adjuster overrides Claim__c.LOB__c or Claim__c.Initial_Severity__c manually,
Then Claim__c.AI_Override__c = TRUE, Claim__c.AI_Override_Reason__c (required text field) must be populated before save, and the override is logged to an AI_Audit_Log__c record with original AI values preserved.

Given the Agentforce agent is unavailable (timeout or service error),
When the OmniScript 'Analyze with AI' action fails,
Then a graceful fallback message is displayed: 'AI triage temporarily unavailable — please classify manually' and the FNOL can be saved without AI classification (manual fields required).

Given a completed FNOL with AI classification,
When the Claim__c record is saved,
Then Claim__c.AI_Triage_Timestamp__c is populated with the datetime of AI analysis and Claim__c.AI_Model_Version__c is populated from the Einstein Prompt Builder template version metadata.

## Solution

Configure Agentforce Agent in Salesforce with two Agent Actions: (1) 'ClassifyClaimType' — Einstein Prompt Builder template grounded on Claim__c historical records via Retrieval Augmented Generation (RAG) with a custom Apex Tool calling a SOQL-based semantic search; (2) 'ScoreClaimSeverity' — a second Prompt Builder template using structured input from FNOL fields. Agent embedded in FNOL_Intake OmniScript via the Agentforce OmniScript component. Confidence score extracted from prompt response JSON and displayed via DataRaptor Transform. Override logic handled by a Record-Triggered Flow on Claim__c checking for field changes on LOB__c and Initial_Severity__c. AI_Audit_Log__c custom object created to store pre-override AI values. Permission Set 'Agentforce_Claims_PS' grants agent access to Claims_Adjuster profiles. Fallback handled by OmniScript error handling step with conditional navigation.

## Description

As a Provider Network Specialist, I need an automated termination workflow on the Network_Termination_Case__c object that — upon case approval — sends a termination notice via DocuSign to the provider's address of record, updates the associated Provider_Network_Agreement__c status to 'Pending Termination', and auto-generates a Provider_Change_Notification__c (PCN) record so that terminations are processed consistently, within regulatory timelines (state-mandated 60-day notice minimum), and without manual document preparation.

Termination reasons in scope: Voluntary Withdrawal, Credentialing Failure, Contract Non-Renewal, Cause (SIU Referral).

Extends INS-1614 (Network_Termination_Case__c object and approval process build).

## Acceptance Criteria

Given a Network_Termination_Case__c record enters the 'Approved' stage via the multi-step approval process,
When the final approver clicks 'Approve',
Then Network_Termination_Case__c.Status__c transitions to 'Approved — Pending Notice' and a DocuSign envelope is sent to the email address stored in Provider__c.Provider_Email__c using template 'Network_Termination_Notice_v2'.

Given the DocuSign envelope is sent,
When the envelope status callback is received (status = 'sent'),
Then Network_Termination_Case__c.DocuSign_Sent_Date__c is populated with today's date and Network_Termination_Case__c.Notice_Deadline__c is set to DocuSign_Sent_Date__c + 60 days.

Given the DocuSign envelope is delivered and the notice period begins,
When Network_Termination_Case__c.Status__c = 'Approved — Pending Notice',
Then the related Provider_Network_Agreement__c.Status__c is updated to 'Pending Termination' and PNA.Termination_Date__c = Network_Termination_Case__c.Notice_Deadline__c.

Given the status transitions to 'Approved — Pending Notice',
When the automation fires,
Then a Provider_Change_Notification__c record is auto-created with: PCN.Type__c = 'Termination', PCN.Provider__c = Network_Termination_Case__c.Provider__c, PCN.Effective_Date__c = Network_Termination_Case__c.Notice_Deadline__c, PCN.Reason__c = Network_Termination_Case__c.Termination_Reason__c, PCN.Status__c = 'Draft'.

Given Network_Termination_Case__c.Termination_Reason__c = 'Cause (SIU Referral)',
When the PCN is auto-created,
Then PCN.Requires_Compliance_Review__c = TRUE and PCN.OwnerId = Compliance_Review_Queue.Id.

Given the provider's address of record (Provider__c.Provider_Email__c) is blank,
When the workflow attempts to send the DocuSign envelope,
Then the envelope send is suppressed, Network_Termination_Case__c.Status__c is set to 'Action Required — Missing Contact', and a Task is created assigned to the case owner with Subject = 'Missing provider email — termination notice blocked'.

Given the DocuSign envelope status callback returns 'declined' or 'voided',
When the callback is processed,
Then Network_Termination_Case__c.Status__c = 'Notice Declined — Manual Follow-up Required' and an alert email is sent to the assigned Provider Network Specialist.

## Solution

Record-Triggered Flow on Network_Termination_Case__c fires on Status__c change to 'Approved — Pending Notice'. Flow calls Apex action (DocuSign_EnvelopeSender.cls) that uses DocuSign eSignature API via Named Credential 'DocuSign_NC' to send envelope with template mapping. DocuSign status callbacks handled by a REST endpoint (DocuSign_CallbackHandler.cls) that publishes a Platform Event (DocuSign_Status__e); a second Record-Triggered Flow subscribes to this event to update Network_Termination_Case__c fields. PNA status update and PCN creation handled in the same after-save Flow via related record update and create elements. SIU routing logic implemented as a Decision element in the Flow. Missing email guard implemented as a Flow fault path. All DocuSign interactions logged to a custom DocuSign_Audit_Log__c object for compliance traceability.

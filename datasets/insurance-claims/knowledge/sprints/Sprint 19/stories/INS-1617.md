## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces the provider's active Provider_Network_Agreement__c contract tier, associated Fee_Schedule__c effective and expiration dates, and current credentialing status from Credentialing_Case__c, so that I can answer provider inquiries about contract terms and payment rates without navigating across multiple objects.

Extends INS-1586 (Provider 360 base FlexCard scaffold).

## Acceptance Criteria

Given a Provider__c record with at least one active Provider_Network_Agreement__c (PNA__c.Status__c = 'Active'),
When the Provider Relations Specialist opens the Provider 360 FlexCard on that record,
Then the FlexCard displays PNA__c.Contract_Tier__c, PNA__c.Effective_Date__c, and PNA__c.Expiration_Date__c in a clearly labeled section titled 'Active Network Agreement'.

Given the PNA is linked to a Fee_Schedule__c record,
When the FlexCard renders,
Then Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.Expiration_Date__c are displayed in a 'Fee Schedule' subsection below the PNA section.

Given a provider has multiple PNA records (e.g., one Active, one Expired),
When the FlexCard loads,
Then only the record where PNA__c.Status__c = 'Active' AND PNA__c.Effective_Date__c <= TODAY() is shown; expired agreements are suppressed from the primary view.

-Given a provider has a pending PNA in DocuSign, display the DocuSign envelope status inline on the FlexCard.- (Superseded — DocuSign envelope widget deferred to INS-1641)

Given the provider has an open Credentialing_Case__c (Status__c != 'Closed'),
When the FlexCard renders,
Then a banner displays 'Credentialing In Progress' with Credentialing_Case__c.Case_Number__c and Credentialing_Case__c.Expected_Completion_Date__c.

Given a Provider__c record with NO active PNA,
When the FlexCard renders,
Then the 'Active Network Agreement' section displays 'No Active Contract on File' and a Quick Action button labeled 'Initiate PNA' is surfaced.

Given any user without the 'Provider_Network_Read' Permission Set Group,
When they attempt to load the FlexCard,
Then the PNA and Fee Schedule sections are hidden and replaced with 'Access Restricted — Contact Provider Network Team'.

## Solution

1. Build an OmniStudio FlexCard 'Provider360_ContractTier_FC' using a DataRaptor Extract source querying Provider_Network_Agreement__c, Fee_Schedule__c, and Credentialing_Case__c with relationship traversal from the Provider__c context record.
2. DataRaptor (DR_Extract_ProviderPNA) uses a SOQL filter: PNA__c.Status__c = 'Active' AND PNA__c.Provider__c = :recordId, ordered by Effective_Date__c DESC LIMIT 1.
3. Implement a Conditional State on the FlexCard to toggle between 'Active Contract' and 'No Contract' card states based on the DataRaptor response payload.
4. Apply Field-Level Security enforcement via the DataRaptor's 'Respect FLS' setting and add a FlexCard Visibility Rule tied to the 'Provider_Network_Read' Permission Set Group using a custom label check.
5. Surface the 'Initiate PNA' Quick Action via a FlexCard Action of type 'Navigation' pointing to the PNA record creation screen (OmniScript: PNA_Initiation_OS).
6. Add the FlexCard to the Provider__c Lightning Record Page via App Builder, positioned in the right-hand sidebar panel below the standard Highlights Panel.

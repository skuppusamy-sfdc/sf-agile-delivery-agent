## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can immediately assess a provider's credentialing state, active PNA participation, open Provider Change Notifications (PCNs), and any in-flight Network Termination Cases without navigating across multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page,
When the Provider 360 FlexCard loads,
Then it displays the provider's NPI, TIN, and primary specialty pulled from Provider__c.NPI__c, Provider__c.TIN__c, and Provider__c.Primary_Specialty__c respectively.

Given the provider has an active Credentialing_Case__c record,
When the FlexCard renders the Credentialing Status tile,
Then Credentialing_Case__c.Status__c and Credentialing_Case__c.Expiration_Date__c are shown with a red indicator if expiration is within 30 days.

Given the provider is associated with one or more Provider_Network_Agreement__c records,
When the Network Participation section renders,
Then all PNAs where PNA.Status__c = 'Active' are listed with PNA.Network_Name__c, PNA.Effective_Date__c, and PNA.Termination_Date__c.

Given the provider has open Provider_Change_Notification__c records,
When the PCN indicator renders,
Then a badge count displays the number of PCNs where PCN.Status__c IN ('Open', 'Pending Review') and clicking the badge navigates to a filtered PCN list view.

Given the provider has an open Network_Termination_Case__c,
When the FlexCard loads,
Then a prominent warning banner displays with Network_Termination_Case__c.Termination_Reason__c and Network_Termination_Case__c.Effective_Termination_Date__c.

-Given the provider is flagged for SIU review, display SIU flag in the header tile.- (Superseded — SIU tile moved to INS-1671)

Given a provider has no active PNAs, no open PCNs, and no termination cases,
When the FlexCard renders,
Then a 'No Active Network Issues' status message is displayed in the network participation section.

## Solution

Build using OmniStudio FlexCard with a parent DataRaptor Extract sourcing data from Provider__c, with child DataRaptor Turbo Extracts for Credentialing_Case__c (filtered by Provider__c lookup and Status__c != 'Closed'), Provider_Network_Agreement__c (filtered by Status__c = 'Active'), Provider_Change_Notification__c (filtered by Status__c IN ('Open','Pending Review')), and Network_Termination_Case__c (filtered by Status__c != 'Completed'). Use FlexCard conditional rendering states for warning banners and badge counts. Embed FlexCard on Provider__c Lightning Record Page via App Builder. Apply Permission Set Group 'Provider_Relations_Standard' to control visibility. Cache DataRaptor results for 60 seconds using OmniStudio cache settings to reduce SOQL load given high page-view volume.

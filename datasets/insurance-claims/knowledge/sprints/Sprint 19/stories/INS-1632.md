## Description

As a Credentialing Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record when a Credentialing_Case__c transitions to Status__c = 'Approved' so that downstream network contracting and claims teams are immediately notified of newly credentialed providers without manual handoff steps.

This supports compliance with internal SLA requiring PCN issuance within 2 business hours of credentialing approval.

## Acceptance Criteria

Given a Credentialing_Case__c record has Status__c changed to 'Approved':
When the record is saved:
Then a new Provider_Change_Notification__c is created with:
  - PCN_Type__c = 'New Credential Approval'
  - Related_Provider__c = Credentialing_Case__c.Provider__c
  - Effective_Date__c = TODAY()
  - PCN_Status__c = 'Pending Distribution'
  - Source_Case__c = Credentialing_Case__c.Id
  - Created_By_Automation__c = TRUE.

Given the Credentialing_Case__c.Credentialing_Type__c = 'Re-credentialing':
When the PCN is auto-created:
Then PCN_Type__c = 'Re-credentialing Approval' (not 'New Credential Approval').

Given a Credentialing_Case__c transitions to 'Approved' but a PCN with Source_Case__c = same case Id already exists with PCN_Status__c != 'Cancelled':
When the automation runs:
Then NO duplicate PCN is created and a warning is logged to the Credentialing_Case__c chatter feed: 'PCN already exists for this credentialing case — duplicate suppressed.'

Given the PCN is successfully created:
When PCN_Status__c = 'Pending Distribution':
Then an automated email notification is dispatched to the Provider Relations queue (Queue: Provider_Relations_PCN_Queue) using Email Alert `EA_PCN_New_Credential_Notification`.

Given the automation fails due to a validation error on Provider_Change_Notification__c:
When the exception is caught:
Then the error is logged to a custom object `Integration_Error_Log__c` with Error_Source__c = 'PCN Automation' and the Credentialing_Case__c is NOT rolled back.

## Solution

1. Implement a Record-Triggered Flow `RTF_CredCase_Create_PCN` on Credentialing_Case__c, triggered AFTER SAVE on field change: Status__c = 'Approved'.
2. Add a Get Records element to check for existing non-cancelled PCN records with Source_Case__c = triggering record Id.
3. Use a Decision element to branch: if duplicate found → post Chatter via Post to Chatter action; if no duplicate → Create Records element to insert Provider_Change_Notification__c with mapped fields.
4. Add a separate Fault Path subflow `SF_Log_Integration_Error` that writes to Integration_Error_Log__c using a before-save context to avoid rollback of parent transaction.
5. Configure the Email Alert `EA_PCN_New_Credential_Notification` on Provider_Change_Notification__c object targeting the Provider_Relations_PCN_Queue.
6. Add a Sharing Rule to ensure PCN records created by automation are visible to Provider_Relations_PCN_Queue members.

## Description

As a Policyholder accessing the Acme Insurance self-service Experience Cloud portal, I need a consolidated Claims Summary dashboard on my Policyholder 360 page that displays all active and recently closed claims with a direct download link for each associated Explanation_of_Benefits__c (EOB) document so that I can track my claims status and access EOB documentation without calling the customer service center, reducing inbound call volume and improving member satisfaction scores.

## Acceptance Criteria

Given an authenticated Policyholder user on the Experience Cloud portal,
When they navigate to the 'My Claims' tab on the Policyholder 360 page,
Then the LWC 'claimsSummaryDashboard' displays all Claim__c records where Policy__c.Policyholder__c = current user's Contact and Claim__c.Status__c IN ('Open', 'Under Review', 'Adjudicated', 'Closed') within the past 24 months.

Given a Claim__c record is displayed in the dashboard,
When the policyholder views the claim row,
Then the following fields are visible: Claim__c.Name (Claim Number), Claim__c.Date_of_Service__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, Claim__c.Total_Paid_Amount__c, Claim__c.Provider__r.Name, and an EOB download icon (if EOB exists).

Given a Claim__c has one or more associated Explanation_of_Benefits__c records (Explanation_of_Benefits__c.Claim__c = Claim__c.Id),
When the policyholder clicks the EOB download icon,
Then the most recent EOB document (by EOB.EOB_Generated_Date__c DESC) is downloaded as a PDF from Salesforce Files (ContentDocument linked to the EOB record) with filename format: 'EOB_[ClaimNumber]_[YYYYMMDD].pdf'.

Given a Claim__c with no linked Explanation_of_Benefits__c record,
When the claim row is displayed,
Then the EOB icon is replaced with the text 'EOB Pending' styled in muted gray, and no download action is available.

Given the dashboard loads for a policyholder with more than 20 claims in scope,
When the component initializes,
Then results are paginated with 10 claims per page and pagination controls (Previous / Next / Page X of Y) are displayed.

-Given Claim__c.LOB__c = 'Dental', exclude from the portal dashboard view pending dental portal build.-

Given a policyholder with no claims in the past 24 months,
When the dashboard loads,
Then an empty state illustration is displayed with message: 'No claims found in the past 24 months. If you believe this is incorrect, please contact Member Services.'

Given the component is accessed on a mobile device (viewport < 768px),
When the dashboard renders,
Then the layout collapses to a card-based stacked view with all required fields visible without horizontal scrolling.

## Solution

Build a custom LWC 'claimsSummaryDashboard' deployed to the Experience Cloud Digital Experience using a Guest/Authenticated User profile with appropriate Object and Field-level security via Permission Set Group 'Policyholder_Portal_Access'. The component uses @wire with a custom Apex controller 'ClaimsSummaryController' (with 'with sharing' enforcement) that queries Claim__c and related Explanation_of_Benefits__c and ContentDocumentLink records in a single method returning a wrapper class list. Pagination handled client-side in the LWC JavaScript controller using a sliced array and tracked currentPage variable. EOB PDF download uses NavigationMixin to a ContentDocument URL resolved in the Apex controller. SLDS responsive grid classes handle mobile layout. A separate Apex test class achieves ≥ 85% coverage including no-claims, paginated, and mobile scenarios. Content Security Policy headers reviewed with Security team for file download compliance.

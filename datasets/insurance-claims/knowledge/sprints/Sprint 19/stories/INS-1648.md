## Description

As a Claims Operations Manager, I need the Fee Schedule Engine to support tiered reimbursement rates on Fee_Schedule__c records differentiated by Provider_Network_Agreement__c.Network_Tier__c and Claim__c.LOB__c so that claims are automatically repriced at the correct contractual rate for Tier 1, Tier 2, and Tier 3 network providers across Commercial, Medicare Advantage, and Medicaid LOBs without manual rate lookup.

Builds on Fee Schedule foundation from INS-1589.

## Acceptance Criteria

Given a Fee_Schedule__c record,
When it is created or edited,
Then the record must support the following fields: Fee_Schedule__c.Network_Tier__c (picklist: Tier 1, Tier 2, Tier 3, Out-of-Network), Fee_Schedule__c.LOB__c (picklist: Commercial, Medicare Advantage, Medicaid, Self-Funded), Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Reimbursement_Rate__c (currency), Fee_Schedule__c.Rate_Type__c (picklist: Percent of Billed, Flat Fee, Percent of Medicare), Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.Expiration_Date__c.

Given a Claim_Line__c record is submitted for adjudication,
When the repricing engine resolves the applicable fee schedule,
Then it queries Fee_Schedule__c where CPT_Code__c matches Claim_Line__c.CPT_Code__c AND Network_Tier__c matches the provider's PNA__c.Network_Tier__c AND LOB__c matches Claim__c.LOB__c AND Effective_Date__c <= Claim__c.Service_Date__c AND Expiration_Date__c >= Claim__c.Service_Date__c, selecting the most specific match using a priority scoring algorithm (LOB match > Tier match > CPT match).

Given a matching Fee_Schedule__c is found with Rate_Type__c = 'Percent of Medicare',
When Claim_Line__c.Allowed_Amount__c is calculated,
Then the engine retrieves the current Medicare rate from the CMS integration (CMSRateService) for the CPT code and applies Fee_Schedule__c.Reimbursement_Rate__c as a percentage multiplier, storing the result in Claim_Line__c.Allowed_Amount__c and Claim_Line__c.Fee_Schedule_Applied__c = the matched Fee_Schedule__c.Id.

Given no matching Fee_Schedule__c is found for the LOB and Tier combination,
When the engine fails to match,
Then Claim_Line__c.Repricing_Status__c = 'No Rate Found', Claim_Line__c.Allowed_Amount__c = 0, and the parent Claim__c is routed to a 'Manual Pricing Review' queue via Assignment Rule.

-Given Fee_Schedule__c supports a 'Default Rate' fallback field for unmatched tiers- (deprecated — default rate fallback removed per actuarial team request, manual review queue is the approved path)

Given a Fee_Schedule__c record's Expiration_Date__c is within 14 days of TODAY(),
When the daily scheduled flow (SF_FeeScheduleExpiryAlert) runs,
Then a Task is created assigned to the owning Claims Operations Manager with Subject = 'Fee Schedule Expiring: [CPT Code] - [LOB] - [Tier]' and Due Date = Fee_Schedule__c.Expiration_Date__c.

## Solution

Extend Fee_Schedule__c object with new custom fields (Network_Tier__c, LOB__c, Rate_Type__c) via metadata deployment. Build Apex service class FeeScheduleResolutionService with method resolveRate(Claim_Line__c line, Provider_Network_Agreement__c pna) implementing priority scoring logic using a weighted integer score per match dimension. Integrate with existing CMSRateService Apex class (HTTP callout to CMS OPPS/PFS API via Named Credential 'CMS_API'). Update ClaimRepricingBatch to invoke FeeScheduleResolutionService per line in bulkified pattern. Scheduled Flow 'SF_FeeScheduleExpiryAlert' runs daily at 6AM via Schedule-Triggered Flow. Custom index added on Fee_Schedule__c (CPT_Code__c, LOB__c, Network_Tier__c) for query performance.

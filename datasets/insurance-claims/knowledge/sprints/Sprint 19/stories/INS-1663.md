## Description

As a Credentialing Specialist, I need the system to automatically monitor CAQH attestation expiration dates synced from the CAQH ProView integration and close or escalate open Credentialing_Case__c records when a provider's attestation lapses without re-attestation so that we maintain continuous compliance with NCQA credentialing standards and avoid credentialing gaps that could expose the organization to regulatory risk.

## Acceptance Criteria

Given a Credentialing_Case__c record with Status__c = 'Active Credentialing' and a linked Provider__c (Person Account) where Provider__c.CAQH_Attestation_Expiry__c is populated,
When the daily scheduled job 'CAQH_Attestation_Monitor' runs,
Then the job identifies all Provider__c records where CAQH_Attestation_Expiry__c is within 30 days of today and Credentialing_Case__c.CAQH_Renewal_Notified__c = FALSE.

Given a Provider__c with CAQH attestation expiring within 30 days,
When the monitor job identifies the record,
Then an automated email notification is sent to the provider's primary contact email (Provider__c.Email) and the assigned Credentialing Specialist (Credentialing_Case__c.OwnerId) using Email Template 'CAQH_Expiry_30Day_Warning', and Credentialing_Case__c.CAQH_Renewal_Notified__c is set to TRUE.

Given a Provider__c where CAQH_Attestation_Expiry__c has passed (date < today) and no updated attestation date has been received from the CAQH integration,
When the monitor job runs,
Then Credentialing_Case__c.Status__c is updated to 'Suspended - Attestation Lapsed' and a Case Comment is appended: 'CAQH attestation expired [date]. Awaiting re-attestation. Auto-suspended per compliance policy.'

Given Credentialing_Case__c.Status__c = 'Suspended - Attestation Lapsed',
When Provider__c.CAQH_Attestation_Expiry__c is updated (via CAQH integration sync) to a future date,
Then Credentialing_Case__c.Status__c automatically transitions back to 'Active Credentialing' and a Task is created for the assigned Credentialing Specialist to verify re-attestation documentation.

-Given CAQH integration is unavailable, mark all open cases as 'Pending CAQH Verification' and send a bulk alert to the Credentialing Ops team.-

Given a Credentialing_Case__c where the provider's NPI is flagged as 'Deactivated' in the NPPES integration,
When the monitor job runs,
Then the case is immediately escalated to Status__c = 'Escalated - NPI Issue' regardless of CAQH attestation status, and a high-priority Task is created for the Compliance Officer.

Given the monitor job completes each daily run,
When execution finishes,
Then a Credentialing_Monitor_Run__c log record is created capturing: run timestamp, records evaluated count, notifications sent count, suspensions triggered count, and any error messages.

## Solution

Implement a Schedulable Apex class 'CAQH_Attestation_MonitorScheduler' that invokes a Batchable Apex class 'CAQH_Attestation_MonitorBatch' (batch size 100). The batch start() method queries Credentialing_Case__c records with Status__c IN ('Active Credentialing','Suspended - Attestation Lapsed') with related Provider__c CAQH and NPPES fields via a relationship query. Execute() method applies date comparison logic and updates case status, creates Tasks, and queues email sends via Messaging.SingleEmailMessage list (bulk sent post-loop). CAQH field updates arrive via existing DataIntegration middleware (MuleSoft) writing to Provider__c.CAQH_Attestation_Expiry__c — a separate Record-Triggered Flow on Provider__c handles the re-attestation reinstatement path to keep batch and real-time logic decoupled. Credentialing_Monitor_Run__c log written in finish() method. Scheduled daily at 02:00 AM org time via Setup > Scheduled Jobs.

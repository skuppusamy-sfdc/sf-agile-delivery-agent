## Description

Bug reported by Claims Operations Manager users in UAT. The 'My Team's Open Claims' list view on Claim__c is timing out for users in the 'Claims_Operations_Manager' profile when the queue contains more than approximately 10,000 records. Browser console shows a Salesforce SOQL timeout error. The list view uses a filter on Claim__c.Status__c IN ('Open','Pending Review','Under Investigation') combined with Claim__c.Assigned_Queue__c = current user's queue.

**Steps to Reproduce:**
1. Log in as a user with 'Claims_Operations_Manager' profile.
2. Navigate to the Claim__c tab and select the 'My Team's Open Claims' list view.
3. Observe loading behavior with >10,000 qualifying records in the org.

**Expected:** List view loads within 5 seconds, displaying paginated results.

**Actual:** List view times out after ~30 seconds with a generic Salesforce error. Confirmed via Salesforce debug logs that the underlying SOQL lacks a selective filter, performing a full table scan on Claim__c (~2.1M records in UAT).

as per Description

## Solution

Root cause: 'My Team's Open Claims' list view SOQL filter on Claim__c.Status__c is non-selective given the high cardinality of open claims. Fix: (1) Add a custom index on Claim__c.Status__c and Claim__c.Assigned_Queue__c by submitting an index request via Salesforce Support — confirmed approved. (2) Add Claim__c.Date_of_Service__c >= LAST_N_DAYS:365 as an additional list view filter to reduce result set to a selective range, coordinated with Claims Operations Manager stakeholders who confirmed <1-year-old claims are operationally relevant. (3) Enable 'Enable Search Optimization Service' for the Claim__c object in Setup. Validate fix in UAT by running the list view with 2.1M record dataset and confirming sub-5-second load time. Update list view filter criteria via Setup > Object Manager.

## Description

As a Provider Network Specialist, I need the system to automatically validate that a linked Fee_Schedule__c record has an effective date that falls within the renewed Provider_Network_Agreement__c (PNA) contract term so that claim adjudication does not fail due to misaligned fee schedule coverage periods during the renewal window.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record with Status__c = 'Renewal In Progress' and a linked Fee_Schedule__c,
When the PNA Contract_End_Date__c is updated and saved,
Then the system shall compare Fee_Schedule__c.Effective_Start_Date__c and Fee_Schedule__c.Effective_End_Date__c against PNA.Contract_Start_Date__c and PNA.Contract_End_Date__c.

Given the Fee_Schedule__c.Effective_End_Date__c is earlier than the PNA.Contract_End_Date__c,
When the user attempts to save the PNA record,
Then a blocking validation error shall display: 'Fee Schedule [Fee_Schedule__c.Name] expires before the contract term ends. Please update the Fee Schedule or adjust the contract end date.'

Given a PNA with multiple linked Fee_Schedule__c records (LOB-specific),
When any single Fee_Schedule__c has a coverage gap relative to the PNA term,
Then each offending Fee Schedule shall be listed individually in the validation error message.

-Given the PNA LOB__c = 'Medicare Advantage', skip Fee Schedule date validation and route to Compliance Officer queue for manual review.-

Given all linked Fee_Schedule__c records have effective dates fully encompassing the PNA contract term,
When the user saves the PNA,
Then the record saves successfully and PNA.Fee_Schedule_Validated__c (checkbox) is set to TRUE.

Given PNA.Fee_Schedule_Validated__c = TRUE,
When the PNA Status__c transitions to 'Active',
Then a Conga CLM document generation job is triggered automatically to produce the renewal contract PDF.

## Solution

Implement a Record-Triggered Flow on Provider_Network_Agreement__c, firing on Update where Status__c changes to 'Renewal In Progress' or Contract_End_Date__c is modified. Flow uses a Get Records element to retrieve all related Fee_Schedule__c records via the PNA_Fee_Schedules__r relationship. A loop with decision elements compares date fields and collects error messages into a text collection variable. If the collection is non-empty, a Custom Error element surfaces the concatenated message. On clean validation, a separate flow path sets Fee_Schedule_Validated__c = TRUE via a direct field update. Conga CLM trigger invoked via Apex callout from an invocable action called at flow terminus. All logic unit-tested with @IsTest coverage ≥ 85%.

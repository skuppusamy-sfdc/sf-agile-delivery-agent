## Description

As a Credentialing Specialist, I need a proactive alert system within the Credentialing dashboard that identifies providers whose CAQH profile attestation is expiring within 30, 60, or 90 days, and automatically creates a re-attestation task on the associated Credentialing_Case__c record, so that I can manage credentialing renewal pipelines efficiently and avoid lapses that would trigger provider network termination under our PNA compliance obligations.

CAQH profile expiration dates are synced nightly from the CAQH ProView API into Provider__c.CAQH_Attestation_Expiry__c.

## Acceptance Criteria

Given a Provider__c record where CAQH_Attestation_Expiry__c is within 90 days of today,
When the nightly scheduled job runs,
Then a Task is created on the associated open Credentialing_Case__c with: Subject = 'CAQH Re-Attestation Required — [Provider Name]', ActivityDate = CAQH_Attestation_Expiry__c - 14 days, Priority = 'High', OwnerId = the Credentialing_Case__c.OwnerId, and a custom field Task.Alert_Tier__c populated as '90-Day', '60-Day', or '30-Day' based on remaining days.

Given a Task with Alert_Tier__c = '30-Day' is created,
When the task is created,
Then an additional Custom Notification is sent to the Credentialing Specialist via the 'Credentialing_Alerts' Notification Type with message: 'URGENT: [Provider NPI] CAQH attestation expires in 30 days. Immediate action required.'

Given a provider's CAQH attestation has already expired (CAQH_Attestation_Expiry__c < TODAY()),
When the scheduled job runs,
Then Credentialing_Case__c.Status__c is automatically set to 'Attestation Lapsed' and Provider__c.Credentialing_Status__c is updated to 'Lapsed — Pending Re-Attestation', and a separate escalation task is created and assigned to the Credentialing Specialist's manager (resolved via the Credentialing_Case__c.Supervisor__c lookup field).

Given a Task for a given Alert_Tier__c already exists on the Credentialing_Case__c for a provider,
When the nightly job runs again,
Then NO duplicate task is created — the job checks for existing open Tasks with matching Subject and Alert_Tier__c before inserting.

Given the Credentialing dashboard list view,
When a Credentialing Specialist views the 'CAQH Expiry Alerts' list view on Credentialing_Case__c,
Then records are sorted by CAQH_Attestation_Expiry__c ASC and color-coded using a formula field Expiry_Alert_Color__c: Red (≤30 days), Amber (31–60 days), Yellow (61–90 days).

## Solution

1. Build a Scheduled Apex class (CAQHExpiryAlertScheduler.cls) running nightly at 06:00 UTC, querying Provider__c records WHERE CAQH_Attestation_Expiry__c <= TODAY() + 90 AND Credentialing_Status__c != 'Lapsed — Pending Re-Attestation'.
2. Implement deduplication logic using a SOQL query for existing Tasks: SELECT Id FROM Task WHERE WhatId = :caseId AND Alert_Tier__c = :tier AND IsClosed = false LIMIT 1 before inserting.
3. Create a formula field Expiry_Alert_Color__c (Text) on Credentialing_Case__c using nested IF() to return 'Red', 'Amber', or 'Yellow' based on days remaining; use this in a list view with a conditional highlight rule via a Lightning List View with Conditional Formatting (pilot feature enabled in org).
4. Custom Notifications sent via Messaging.CustomNotification with the 'Credentialing_Alerts' notification type configured in Setup.
5. Manager escalation resolved by traversing Credentialing_Case__c.Supervisor__c lookup; if null, default to the 'Credentialing_Escalation_Queue' as a fallback OwnerId.
6. Schedule the class via a Custom Metadata Type (ScheduledJobConfig__mdt) to allow time adjustment without code redeployment.

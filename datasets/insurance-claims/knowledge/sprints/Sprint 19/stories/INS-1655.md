## Description

As a Claims Operations Manager, I need an Agentforce-powered suggested next-best-action panel embedded within the FNOL intake OmniScript so that claims handlers are guided toward the optimal triage path — including SIU referral, COB investigation, or straight-through processing — based on real-time claim attributes and historical fraud signals, reducing average FNOL handle time and improving triage accuracy.

## Acceptance Criteria

Given a claims handler is on Step 3 ('Incident Details') of the FNOL OmniScript,
When Claim__c.LOB__c, Claim__c.Incident_Type__c, and Policy__c.Coverage_Type__c are populated,
Then the Agentforce NBA panel renders within the OmniScript sidebar with at least one ranked action suggestion.

Given the Agentforce model evaluates the in-progress claim data,
When the fraud propensity score returned by the Einstein model exceeds the threshold defined in Custom Metadata 'Agentforce_Config__mdt.SIU_Referral_Threshold__c',
Then the top suggested action is 'Refer to SIU' with a confidence score displayed as a percentage and a rationale summary generated by the Einstein Generative AI prompt template 'FNOL_SIU_Rationale'.

Given the policy has an active COB flag (Policy__c.COB_Active__c = true),
When the NBA panel loads,
Then 'Initiate COB Investigation' appears as a suggested action with a link to the COB OmniScript.

Given the claim attributes indicate low complexity (Claim__c.Estimated_Amount__c < 5000 AND Claim__c.Incident_Type__c = 'Routine Medical'),
When the NBA panel evaluates the claim,
Then 'Approve for Straight-Through Processing' is the top-ranked suggestion.

Given a claims handler accepts a suggested action,
When they click 'Apply Suggestion',
Then the OmniScript auto-populates the relevant fields (e.g., Claim__c.Triage_Path__c) and logs the accepted suggestion to Agentforce_Action_Log__c with fields: Action_Type__c, Confidence_Score__c, Handler_Accepted__c = true, Timestamp__c.

Given a claims handler dismisses a suggested action,
When they click 'Dismiss',
Then Agentforce_Action_Log__c records Handler_Accepted__c = false and Dismissal_Reason__c captured from a mandatory picklist.

Given the Agentforce service is unavailable,
When the NBA panel attempts to load,
Then a graceful fallback message 'AI suggestions temporarily unavailable' is displayed and the OmniScript remains fully functional without the panel.

## Solution

Integrate Agentforce Einstein Next Best Action (NBA) with the FNOL OmniScript ('FNOL_Intake_v2') built in OmniStudio. Create an Einstein NBA Strategy ('FNOL_Triage_Strategy') using the Strategy Builder, connecting to a custom Einstein Prediction Service model trained on historical Claim__c data with SIU referral outcomes as the target variable. Embed the NBA component as a custom LWC ('fnolAgentforcePanel') within the OmniScript using an OmniScript Custom LWC step. The LWC calls the Einstein NBA API via a named credential ('Agentforce_API') using an Apex controller ('AgentforceNBAController'). Prompt template 'FNOL_SIU_Rationale' configured in Einstein Generative AI Prompt Builder using Claim__c merge fields. Log accepted/dismissed actions via a Record-Triggered Flow on Agentforce_Action_Log__c insert. Threshold values stored in Agentforce_Config__mdt to allow admin tuning without deployment.

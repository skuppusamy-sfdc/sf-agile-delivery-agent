## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces the provider's active Provider_Network_Agreement__c details, associated Fee_Schedule__c tier, and current credentialing status so that I can resolve provider inquiries without navigating across 4–5 separate objects.

Extends INS-1601 (Provider 360 base FlexCard shell). This story adds the contracting and fee schedule data layer on top of the existing demographics panel.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record with at least one active PNA:
When the Provider 360 FlexCard loads:
Then the 'Contracting' panel displays Provider_Network_Agreement__c.Contract_Status__c, Provider_Network_Agreement__c.Effective_Date__c, Provider_Network_Agreement__c.Expiration_Date__c, and Provider_Network_Agreement__c.Network_Tier__c.

Given a Provider__c is linked to a Fee_Schedule__c via the active PNA:
When the FlexCard contracting panel renders:
Then Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.LOB__c, and Fee_Schedule__c.Reimbursement_Rate_Percentage__c are displayed in a read-only tile labeled 'Fee Schedule'.

Given a Provider__c has NO active PNA (Contract_Status__c != 'Active'):
When the FlexCard loads:
Then the Contracting panel displays an inline message: 'No active network agreement on file' and the Fee Schedule tile is hidden.

Given the FlexCard is viewed by a user without the 'Provider_Network_Specialist' Permission Set:
When the contracting panel attempts to render:
Then Fee_Schedule__c.Reimbursement_Rate_Percentage__c is masked and displayed as '—' per data sensitivity rules.

-Given the FlexCard includes a 'Renegotiate Contract' quick action button launching an OmniScript-
-This requirement moved to INS-1645-

Given a Provider__c has multiple historical PNAs:
When the contracting panel renders:
Then only the PNA with Contract_Status__c = 'Active' AND the most recent Effective_Date__c is displayed; historical records are accessible via a 'View History' expand toggle.

## Solution

1. Build a DataRaptor Extract `DR_Get_Provider_PNA_FeeSchedule` that queries Provider_Network_Agreement__c and related Fee_Schedule__c using a LEFT OUTER JOIN on Provider__c Id, filtered by Contract_Status__c = 'Active'.
2. Create an Integration Procedure `IP_Provider360_ContractingPanel` that invokes the DataRaptor, applies conditional logic to mask Fee_Schedule__c.Reimbursement_Rate_Percentage__c based on running user's Permission Set Group membership (`PSG_Provider_Network_Specialist`).
3. Update the existing `FlexCard_Provider360` OmniStudio FlexCard (from INS-1601) to add a new 'Contracting' state panel bound to the Integration Procedure output.
4. Use FlexCard conditional visibility rules to toggle Fee Schedule tile visibility based on IP response flag `hasActivePNA`.
5. Deploy Permission Set adjustments to restrict Fee Schedule rate field via Field-Level Security on Fee_Schedule__c.Reimbursement_Rate_Percentage__c.

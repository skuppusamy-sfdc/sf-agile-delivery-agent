## Description

As a Broker Relations Coordinator, I need a Group_Service_Agreement__c (GSA) document to be automatically generated through Conga CLM and sent for e-signature via DocuSign when a group enrollment record reaches Status__c = 'Enrollment Complete', so that the manual document preparation step is eliminated and time-to-activation for employer groups is reduced from an average of 5 business days to same-day.

Extends INS-1605 (Group Enrollment Workflow Automation).

## Acceptance Criteria

Given a Group_Service_Agreement__c record where GSA__c.Status__c transitions to 'Enrollment Complete' and GSA__c.Group_Size__c > 0
When the automation triggers
Then Conga CLM is invoked via outbound callout to generate a GSA document pre-populated with: GSA__c.Group_Name__c, GSA__c.Effective_Date__c, GSA__c.Renewal_Date__c, GSA__c.Broker__c (linked Broker contact name and NPN), GSA__c.Plan_Type__c, GSA__c.Monthly_Premium__c.

Given the Conga CLM document is generated successfully
When the document is ready
Then a DocuSign envelope is automatically created and sent to the contacts listed in GSA__c.Signatory_Contacts__c (multi-select relationship field) with a 5-business-day expiration on the envelope.

Given the DocuSign envelope is completed (all signatories signed)
When the completion webhook fires
Then GSA__c.Status__c is updated to 'Active', GSA__c.Executed_Date__c = signature completion timestamp, and the executed PDF is stored as a Salesforce File attached to the GSA__c record.

Given a GSA where GSA__c.Group_Size__c >= 100 (large group threshold)
When the automation triggers
Then the document generation uses the 'Large_Group_GSA_Template' in Conga CLM instead of the standard template, and GSA__c.Underwriting_Review_Required__c is set to true, pausing DocuSign dispatch until an Underwriting Analyst approves.

-Given GSA generation failure, send email alert to generic claims-ops@acmeinsurance.com- (deprecated — replaced by Slack integration alert per INS-1618)

Given a Conga CLM callout failure (HTTP 4xx/5xx)
When the error is caught
Then GSA__c.Status__c is set to 'Document Generation Failed', a Case is auto-created with Subject = 'GSA Document Generation Error - [GSA Name]' assigned to the 'Broker Relations Queue', and the error response body is logged to GSA__c.Last_Error_Message__c.

## Solution

1. Build a Record-Triggered Flow 'GSA_Enrollment_Complete_Flow' on Group_Service_Agreement__c, triggering AFTER UPDATE when Status__c changes to 'Enrollment Complete'.
2. Add a Decision element to branch on Group_Size__c >= 100 for large group logic, setting Underwriting_Review_Required__c and halting DocuSign dispatch.
3. Invoke an Apex action 'CongaCLMService.generateGSADocument(gsaId, templateName)' that performs the Conga CLM REST callout using Named Credential 'Conga_CLM_Prod'.
4. On success, invoke 'DocuSignService.createAndSendEnvelope(gsaId, documentId, signatoryList)' with envelope expiration set to 5 business days (calculated using Custom Metadata business calendar).
5. Build an inbound REST endpoint 'DocuSignWebhookController' to handle DocuSign Connect callbacks, updating GSA__c fields and attaching the executed PDF as ContentVersion linked to the GSA__c record.
6. Implement Apex try/catch in CongaCLMService with error logging to GSA__c.Last_Error_Message__c and Case creation via Database.insert.
7. Configure Remote Site Settings and Named Credentials for both Conga CLM and DocuSign endpoints.

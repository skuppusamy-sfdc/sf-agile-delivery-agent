## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces active Provider_Network_Agreement__c contract terms, associated Group_Service_Agreement__c affiliations, and current credentialing status so that I can answer provider inquiries without navigating across five separate related lists.

Extends INS-1589 (Provider 360 base FlexCard scaffold).

Key data points required:
- Active PNA: PNA effective/termination dates, contracted LOB(s), reimbursement methodology, network tier
- GSA affiliations: GSA name, employer group, effective date, participation status
- Credentialing_Case__c: most recent case status, expiration date, CAQH profile ID
- NPI and TIN display (masked TIN — last 4 only)
- Quick-action button to launch 'Initiate PCN' OmniScript if provider has an expiring PNA (<=90 days to termination)

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the FlexCard displays all Provider_Network_Agreement__c records where PNA.Status__c = 'Active' and PNA.Provider__c = current record Id, showing fields: PNA.Effective_Date__c, PNA.Termination_Date__c, PNA.LOB__c, PNA.Reimbursement_Methodology__c, PNA.Network_Tier__c.

Given a provider has one or more GSA affiliations,
When the FlexCard GSA section renders,
Then all related Group_Service_Agreement__c records where GSA.Provider__c = current record Id and GSA.Participation_Status__c = 'Active' are listed showing GSA.Name, GSA.Employer_Group__c, GSA.Effective_Date__c.

Given the provider has a Credentialing_Case__c record,
When the credentialing panel renders,
Then the most recent Credentialing_Case__c (ordered by CreatedDate DESC, limit 1) is shown with fields: Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, Credentialing_Case__c.CAQH_Profile_ID__c.

Given the provider's TIN is stored in Provider__c.TIN__c,
When the FlexCard header renders,
Then TIN is displayed as '***-**-XXXX' (last 4 digits only) and NPI is displayed unmasked from Provider__c.NPI__c.

Given a provider has an active PNA where PNA.Termination_Date__c is within 90 calendar days of today,
When the FlexCard PNA row renders,
Then an amber warning badge 'Expiring Soon' is shown and an 'Initiate PCN' action button is visible that launches the Provider_Change_Notification OmniScript pre-seeded with Provider__c.Id and PNA.Id.

Given a provider has NO active PNA records,
When the FlexCard PNA section renders,
Then an informational message 'No active contracts on file' is displayed and the 'Initiate PCN' button is hidden.

-Given the FlexCard loads, a Chatter feed snapshot of the last 3 posts is displayed in a side panel- (descoped — moved to INS-1651)

## Solution

Build using OmniStudio FlexCard with two child DataRaptor Extract actions: (1) DR_GetActivePNAsByProvider querying Provider_Network_Agreement__c with filter on Status__c and Provider__c; (2) DR_GetGSAAndCredentialingByProvider joining Group_Service_Agreement__c and Credentialing_Case__c. TIN masking implemented via a DataRaptor Transform formula field. Conditional 'Expiring Soon' badge uses FlexCard conditional rendering with a date-diff calculation against today's date. 'Initiate PCN' button invokes OmniScript 'ProviderChangeNotification_OS' via FlexCard action with input JSON mapping PNA.Id and Provider__c.Id. FlexCard deployed to Provider Record Page via Lightning App Builder. Profile/Permission Set visibility restricted to Provider_Relations_PS and Network_Admin_PS.

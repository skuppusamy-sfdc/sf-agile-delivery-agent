## Description

As a Claims Operations Manager, I need an Agentforce AI action embedded in the Claim__c record page that analyzes claim anomaly signals and recommends SIU referral with a confidence score so that investigators are engaged earlier in the claims lifecycle, reducing fraudulent payouts and improving compliance posture.

The action should evaluate: duplicate CPT code patterns on Claim_Line__c, provider billing frequency outliers (via Provider__c history), ICD-10 / CPT mismatch flags, Claim__c.Total_Billed_Amount__c thresholds by LOB, and claimant COB discrepancies on Policy__c.

## Acceptance Criteria

Given a Claim__c record with Status__c = 'Under Review',
When a Claims Operations Manager invokes the Agentforce 'Triage Claim' action,
Then the AI action returns a SIU_Referral_Recommended__c flag (TRUE/FALSE) and a Fraud_Confidence_Score__c (0.00–1.00) within 5 seconds.

Given Fraud_Confidence_Score__c >= 0.75,
When the action completes,
Then the system automatically creates a linked Task with Subject = 'SIU Referral Review', assigned to the SIU queue, and sets Claim__c.SIU_Referral_Status__c = 'Pending Review'.

Given Fraud_Confidence_Score__c between 0.40 and 0.74,
When the action completes,
Then a non-blocking alert banner appears on the Claim__c record page: 'Moderate anomaly signals detected. Manual review recommended.' No automatic Task is created.

Given Fraud_Confidence_Score__c < 0.40,
When the action completes,
Then no SIU action is taken and Claim__c.AI_Triage_Completed__c is stamped TRUE with a datestamp.

Given the Agentforce action is invoked on a Claim__c where LOB__c = 'Workers Compensation',
When scoring completes,
Then the scoring model applies the WC-specific threshold configuration stored in Custom Metadata Type Agentforce_LOB_Config__mdt.

Given any invocation of the Agentforce action,
When the action completes (any score),
Then an Agentforce_Triage_Log__c record is created capturing: Claim__c lookup, score, contributing signal breakdown (JSON blob in Triage_Signal_Detail__c), invoking user, and timestamp.

Given a network timeout or Einstein API failure during scoring,
When the action is invoked,
Then a graceful error message is displayed: 'AI triage temporarily unavailable. Please retry or escalate manually.' and no Claim__c fields are modified.

## Solution

Implement an Agentforce Custom Action backed by an Apex invocable method 'ClaimsTriageAIAction'. The Apex class calls the Einstein Platform Services scoring endpoint (or a configured External Service registered in Salesforce) passing a JSON payload constructed from Claim__c, Claim_Line__c, Policy__c COB fields, and Provider__c billing history aggregates. LOB-specific thresholds are read from Agentforce_LOB_Config__mdt to avoid hardcoding. Response handling uses a wrapper class to parse confidence score and signal breakdown. Conditional logic in the Apex method drives Task creation and field updates. All writes wrapped in a try/catch with a custom exception handler that invokes a Platform Event (Triage_Error__e) for monitoring. LWC toast component surfaces user-facing messages. Agentforce_Triage_Log__c insert is performed in a future method to avoid governor limit conflicts.

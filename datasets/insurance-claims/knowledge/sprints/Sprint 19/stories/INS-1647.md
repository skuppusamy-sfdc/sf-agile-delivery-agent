## Description

**Bug Report**

**Steps to Reproduce:**
1. Locate or create a Claim__c record with 51 or more related Claim_Line__c child records (test claim ID: CLM-2024-88341 in UAT).
2. Navigate to the Explanation_of_Benefits__c record linked to the claim: EOB__c.Claim__c = CLM-2024-88341.
3. Click the 'Generate EOB PDF' button (Conga Composer button on EOB__c layout).
4. Observe the result.

**Expected Result:** A complete EOB PDF is generated containing all Claim_Line__c records with CPT codes, ICD-10 diagnosis codes, billed amounts, allowed amounts, and member responsibility amounts, delivered to the policyholder via DocuSign envelope.

**Actual Result:** Conga Composer returns a timeout error: 'Apex CPU time limit exceeded' and no PDF is generated. The EOB__c.Generation_Status__c field remains 'Pending' and no DocuSign envelope is created. Error logged in Salesforce debug logs: 'System.LimitException: Apex CPU time limit exceeded' originating in CongaComposerController.generateEOBDocument().

**Root Cause Hypothesis:** The Conga template SOQL query for Claim_Line__c is executing a nested loop with per-line Fee_Schedule__c lookups inside the controller, causing O(n) SOQL queries per claim line. For 50+ lines this breaches CPU limits.

**Frequency:** Affects approximately 12% of institutional claims (facility billing) which routinely have 50-200 line items. High-severity for hospital claims processing.

## Acceptance Criteria

as per Description

## Solution

Refactor CongaComposerController.generateEOBDocument() to pre-fetch all related Claim_Line__c records and their associated Fee_Schedule__c lookups in a single bulkified SOQL query using a Map<Id, Fee_Schedule__c> keyed by Fee_Schedule__c.Id before entering any iteration logic. Replace per-line SOQL calls with Map.get() lookups. For claims exceeding 100 lines, introduce a Queueable Apex job (EOBGenerationQueueable) that generates the PDF asynchronously and updates EOB__c.Generation_Status__c = 'In Progress' immediately, then 'Generated' on completion, triggering DocuSign envelope via platform event. Add a custom exception handler to log failures to a new Error_Log__c object. Update Conga template queries to use template parameters instead of inline SOQL where possible.

## Description

As a Provider Network Specialist, I need Fee_Schedule__c records to automatically transition to 'Superseded' status when a new rate version becomes effective, so that Claim_Line__c adjudication always references the correct reimbursement rates without manual intervention.

Background: Currently, analysts manually update Fee_Schedule__c.Status__c when a rate change cycle begins. This has caused adjudication errors on 3 occasions in the past two sprints where stale fee schedules were applied to Claim_Line__c records submitted after the new effective date.

## Acceptance Criteria

Given a Fee_Schedule__c record with Status__c = 'Active' and a successor record where Effective_Start_Date__c equals today's date,
When the scheduled nightly job executes at 00:01 CST,
Then the predecessor Fee_Schedule__c.Status__c transitions to 'Superseded' and Fee_Schedule__c.Superseded_Date__c is stamped with today's date.

Given a Fee_Schedule__c record transitions to 'Superseded',
When the transition occurs,
Then a Platform Event (Fee_Schedule_Transitioned__e) is published containing Fee_Schedule__c.Id, Fee_Schedule__c.LOB__c, and Fee_Schedule__c.Provider_Network_Agreement__c lookup.

Given the Fee_Schedule_Transitioned__e platform event is received,
When the subscribing Record-Triggered Flow processes the event,
Then all associated Provider_Network_Agreement__c records where PNA.Fee_Schedule__c = superseded record Id are updated with the new Fee_Schedule__c.Id and PNA.Rate_Version__c is incremented.

Given a Claim_Line__c record is submitted with a Service_Date__c that falls within a gap between two fee schedule effective periods,
When adjudication attempts to resolve the applicable fee schedule,
Then the system logs a Fee_Schedule_Gap_Exception__c record and routes the Claim_Line__c to Status__c = 'Pending Manual Review' rather than applying an incorrect rate.

-Given the transition job runs, a chatter post is created on the Fee_Schedule__c record- (deprecated — replaced by Platform Event notification pattern above)

Given a user with the 'Network_Contracting_Admin' Permission Set Group,
When they view the Fee_Schedule__c record after transition,
Then the record detail page displays a 'Rate History' FlexCard showing all prior versions linked via Fee_Schedule__c.Predecessor_Schedule__c lookup chain.

## Solution

1. Apex Schedulable class (FeeScheduleTransitionJob) runs nightly via CronTrigger. Queries Fee_Schedule__c WHERE Status__c = 'Active' AND Successor_Schedule__c != null AND Successor_Schedule__r.Effective_Start_Date__c <= TODAY.
2. Batch Apex (FeeScheduleTransitionBatch) processes records in chunks of 200, updating Status__c and publishing Fee_Schedule_Transitioned__e Platform Events via EventBus.publish().
3. Record-Triggered Flow on Provider_Network_Agreement__c subscribes to the platform event and performs the PNA update via a scheduled-path flow action.
4. Fee_Schedule_Gap_Exception__c custom object created with lookup to Claim__c and Claim_Line__c; populated via Apex Trigger on Claim_Line__c during adjudication rate resolution.
5. FlexCard 'Rate History Card' built in OmniStudio, powered by a DataRaptor Extract querying the predecessor chain. Surfaced on Fee_Schedule__c record page via Lightning App Builder.
6. Permission Set Group 'Network_Contracting_Admin' updated to include CRUD on Fee_Schedule_Gap_Exception__c.

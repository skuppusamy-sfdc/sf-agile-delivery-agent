## Description

As a Provider Network Specialist, I need a FlexCard on the Provider 360 page that displays the provider's current and upcoming Fee_Schedule__c rates grouped by CPT code and LOB so that I can quickly review rate history and proactively communicate upcoming changes to providers without navigating to multiple related lists.

## Acceptance Criteria

Given a Provider__c (Person Account) record page is loaded,
When the 'Fee Schedule Rates' FlexCard is rendered,
Then all Fee_Schedule__c records linked to the provider via active Provider_Network_Agreement__c must be displayed, filtered to Status__c IN ('Active', 'Published') and grouped by LOB__c.

Given the FlexCard is loaded,
When the current date falls within Fee_Schedule__c.Effective_Date__c and Fee_Schedule__c.Expiration_Date__c,
Then the row must display a green 'Active' badge; if Effective_Date__c is in the future, display an amber 'Upcoming' badge.

Given the FlexCard displays rate rows,
When a Provider Network Specialist clicks 'View Details' on a rate row,
Then an OmniScript modal must open showing CPT code, ICD-10 qualifier, allowed amount, rate type, and the linked PNA record name.

Given more than 10 CPT code rows exist,
When the FlexCard renders,
Then pagination must be applied showing 10 rows per page with a CPT code search/filter input field.

## Solution

Build OmniStudio FlexCard 'Provider_Fee_Schedule_FC' using a DataRaptor Extract as the data source, querying Fee_Schedule__c joined to Provider_Network_Agreement__c filtered by Provider__c.Id (context record). Conditional styling actions on the FlexCard handle badge color logic. 'View Details' action launches an OmniScript 'Fee_Schedule_Detail_OS' in a modal. FlexCard embedded on the Provider 360 Lightning Record Page via App Builder. DataRaptor caches results for 5 minutes using a custom cache key on Provider__c.Id to reduce SOQL volume on high-traffic pages.

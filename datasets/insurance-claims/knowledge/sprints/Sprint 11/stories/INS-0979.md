## Description

As a Provider Network Specialist, I need a Fee Schedule FlexCard embedded on the Provider 360 Lightning page (Person Account / Provider__c layout) that displays the provider's currently active Fee_Schedule__c records, upcoming rate changes, and any open Provider_Change_Notification__c items, so that I have a single consolidated view of rate information without navigating to multiple related lists.

Design mockups approved in Confluence: ACM-FEE-CARD-v2. Depends on INS-0977 (Fee Schedule effective date enforcement) being deployed to staging.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c (Person Account) record page,
When the Fee Schedule FlexCard loads,
Then it must display a tabbed interface with tabs: 'Active Rates', 'Pending Rates', and 'Notifications'.

Given the 'Active Rates' tab is selected,
When the DataRaptor retrieves Fee_Schedule__c records,
Then only records where Fee_Schedule__c.Status__c = 'Active' AND Fee_Schedule__c.Effective_Date__c <= TODAY() AND Fee_Schedule__c.Expiration_Date__c >= TODAY() AND the provider NPI matches must be displayed, showing columns: Fee_Schedule__c.Name, LOB__c, Effective_Date__c, Expiration_Date__c, and a count of child Fee_Schedule_Line__c records.

Given the 'Pending Rates' tab is selected,
When the DataRaptor fetches upcoming schedules,
Then Fee_Schedule__c records with Status__c = 'Approved' AND Effective_Date__c > TODAY() must be listed with a visual badge color = amber (#F59E0B) to distinguish from active records.

Given the 'Notifications' tab is selected,
When Provider_Change_Notification__c records are fetched for the provider,
Then PCN records with PCN.Status__c IN ('Pending Delivery', 'Delivered') must appear, sorted by PCN.Notice_Date__c descending, showing PCN.Type__c, PCN.Required_Notice_By__c, PCN.Status__c, and a quick-action button 'View PCN' that navigates to the PCN record.

Given the FlexCard is rendered on a Provider__c record where no Fee_Schedule__c records exist,
When the DataRaptor returns zero results,
Then an empty-state message 'No fee schedules on file. Contact Network Contracting to initiate.' must be displayed in place of the table.

Given a user without the 'Provider_Network_Specialist' Permission Set Group,
When they access the Provider 360 page,
Then the Fee Schedule FlexCard must render in read-only mode with the 'View PCN' action button hidden.

## Solution

Build OmniStudio FlexCard 'ProviderFeeScheduleCard' with three DataRaptor Extract sources: DR_GetActiveFeeSchedules, DR_GetPendingFeeSchedules, DR_GetProviderPCNs. Each DataRaptor queries its respective object filtered by the Provider__c.NPI__c context variable passed from the record page. Tab navigation handled natively within the FlexCard layout designer. Conditional styling for amber badge implemented via FlexCard State configuration with condition Fee_Schedule__c.Status__c = 'Approved'. Read-only enforcement uses FlexCard visibility conditions bound to UserContext.permissionSets. FlexCard deployed to the Provider__c Lightning Record Page via App Builder, positioned in the 'Rates & Agreements' accordion section. No Apex required; all data access through DataRaptor to maintain OmniStudio governor limit best practices.

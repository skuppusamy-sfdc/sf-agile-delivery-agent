## Description

As an Underwriting Analyst, I need the claims adjudication engine to recognize when a Claim__c is associated with a self-funded employer group that has a Group_Service_Agreement__c (GSA) containing negotiated rate overrides, so that those GSA-specific rates take precedence over the standard Fee_Schedule__c and TPA clients receive accurate repricing per their contract terms.

## Acceptance Criteria

Given a Claim__c record has Policy__c.Group_Service_Agreement__c populated and GSA.Status__c = 'Active',
When adjudication begins,
Then the system checks for GSA_Rate_Override__c child records on the GSA before consulting the standard Fee_Schedule__c lookup.

Given a matching GSA_Rate_Override__c exists for the Claim_Line__c.CPT_Code__c,
When the allowed amount is calculated,
Then Claim_Line__c.Allowed_Amount__c uses the GSA_Rate_Override__c.Override_Rate__c value and Claim_Line__c.Rate_Source__c = 'GSA Override'.

Given no GSA_Rate_Override__c matches the CPT code,
When the adjudication engine falls through the hierarchy,
Then it applies the standard Fee_Schedule__c logic (as per INS-0954) and Claim_Line__c.Rate_Source__c = 'Standard Fee Schedule'.

Given a GSA has GSA.TPA_Administered__c = TRUE,
When the EOB is generated,
Then EOB.Payer_Name__c = GSA.TPA_Name__c and the ERA file header reflects the TPA's NPI stored in GSA.TPA_NPI__c.

Given a GSA_Rate_Override__c record has an Expiration_Date__c in the past,
When the override is evaluated,
Then it is excluded from rate resolution and treated as if no override exists for that CPT code.

-Given GSA override rates are loaded via manual CSV upload by the Underwriting team- (replaced by automated Workday integration — see INS-0962 backlog item)

Given a Claim__c is adjudicated using a GSA override rate,
When the Payment__c record is generated,
Then Payment__c.Rate_Basis__c = 'GSA' and Payment__c.GSA__c is populated with the parent GSA record ID for reconciliation reporting.

## Solution

Extend the FeeScheduleResolutionService Apex class with a new method 'resolveGSAOverride()' called before the standard fee schedule lookup. Implement a rate hierarchy resolver using a Strategy design pattern with a custom metadata type (Rate_Hierarchy_Config__mdt) defining resolution order: GSA Override → Standard Fee Schedule → Medicare Percentage. Update the Claim_Line__c Record-Triggered Flow to pass GSA context to the invocable. Add GSA__c lookup field to Payment__c if not present (schema change requires Salesforce admin deployment). Create a Sharing Rule to ensure Underwriting Analyst profile can read Group_Service_Agreement__c records owned by the Contracting team.

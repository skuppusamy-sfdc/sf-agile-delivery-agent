## Description

**Bug Report**

**Summary:** When a Provider_Change_Notification__c email is sent for a rate change on an active PNA, the 'Rate_Change_Notification' Email Template incorrectly merges PNA.Contract_Start_Date__c and PNA.Contract_End_Date__c from a previously terminated Provider_Network_Agreement__c record instead of the currently active one, when the provider has both active and terminated PNA records on file.

**Steps to Reproduce:**
1. Create Provider__c (Person Account) with NPI = '1234567890'.
2. Create PNA-1 linked to Provider__c with PNA.Contract_Status__c = 'Terminated', PNA.Contract_Start_Date__c = 01/01/2022, PNA.Contract_End_Date__c = 12/31/2023.
3. Create PNA-2 linked to Provider__c with PNA.Contract_Status__c = 'Active', PNA.Contract_Start_Date__c = 01/01/2024, PNA.Contract_End_Date__c = 12/31/2025.
4. Update a Fee_Schedule_Line__c.Allowed_Amount__c on the Fee_Schedule__c linked to PNA-2.
5. Observe the PCN email generated and sent to the provider.

**Expected Result:** Email body shows Contract Effective: 01/01/2024 – 12/31/2025 (PNA-2 dates).

**Actual Result:** Email body shows Contract Effective: 01/01/2022 – 12/31/2023 (PNA-1 terminated dates).

**Environment:** Acme Insurance - Full Sandbox (FS1)
**Frequency:** Consistent/reproducible
**First Observed:** 2024-11-06 during INS-0924 UAT
**Related Story:** INS-0924

## Acceptance Criteria

As per Description. The email template merge must resolve PNA contract dates exclusively from the PNA record with Contract_Status__c = 'Active' that directly triggered the rate change notification.

## Solution

Root cause: PCNGeneratorService.createRateChangePCN() passes Provider__c.Id to the email template context rather than the specific PNA.Id, causing the template's merge field {!Provider_Network_Agreement__c.Contract_Start_Date__c} to resolve against the first PNA returned by an unordered query. Fix: update the Messaging.SingleEmailMessage to use setWhatId(pnaId) with the triggering PNA record Id explicitly, and update the Email Template merge fields to reference the PNA object directly. Add query filter PNA.Contract_Status__c = 'Active' as a safeguard in the service layer. Regression test added for multi-PNA provider scenario.

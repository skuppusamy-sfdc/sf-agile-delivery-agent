## Description

**Bug Report**

**Summary:** When a provider declines a DocuSign envelope sent for a rate change PCN, the DocuSign webhook fires correctly but the Provider_Change_Notification__c.PCN_Status__c field is not updated to 'Declined'. The record remains stuck in 'Sent - Awaiting Acknowledgment', causing the 15-day reminder automation (built in INS-0884) to continue sending reminders to providers who have already actively declined.

**Steps to Reproduce:**
1. Navigate to a Provider_Change_Notification__c record with PCN_Status__c = 'Sent - Awaiting Acknowledgment' and a valid DocuSign_Envelope_ID__c.
2. In the DocuSign test environment, trigger a 'Declined' event for the corresponding envelope.
3. Confirm the webhook POST is received in Salesforce (verify via DocuSign_Webhook_Log__c — event IS logged correctly).
4. Navigate back to the Provider_Change_Notification__c record.
5. Observe PCN_Status__c.

**Expected Result:** PCN_Status__c = 'Declined' and a Task is created for the Provider Relations Specialist to follow up.

**Actual Result:** PCN_Status__c remains 'Sent - Awaiting Acknowledgment'. No Task is created.

**Root Cause (Suspected):** The DocuSign webhook handler Apex class (DocuSignWebhookHandler.cls) maps 'completed' and 'voided' envelope statuses but is missing a case block for 'declined'.

**Environment:** Acme Insurance Full Sandbox (ACME-FULL)
**Severity:** High — causing provider relations SLA breaches.

## Acceptance Criteria

As per Description.

Additional: After fix, confirm that PCN_Status__c = 'Declined' triggers a Task creation with Subject = 'PCN Declined - Provider Follow-Up Required' assigned to the Provider Relations Specialist on the PCN record, and that the 15-day reminder Flow correctly excludes records with PCN_Status__c IN ('Declined','Acknowledged','Voided').

## Solution

1. Update DocuSignWebhookHandler.cls to add a 'declined' case in the switch statement that sets PCN_Status__c = 'Declined'.
2. Add Task creation logic within the 'declined' handler branch.
3. Update the Scheduled Flow (from INS-0884) entry criteria to exclude PCN_Status__c IN ('Declined','Acknowledged','Voided').
4. Add a new Apex test method in DocuSignWebhookHandlerTest to simulate a declined envelope payload.
5. Deploy and validate in ACME-FULL sandbox against live DocuSign test account.

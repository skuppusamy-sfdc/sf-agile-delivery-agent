## Description

As a Compliance Officer, I need a Salesforce dashboard and underlying report set that identifies all active Provider_Network_Agreement__c records that either have no associated Fee_Schedule__c or whose linked Fee_Schedule__c has expired or is expiring within 30 days so that the Provider Network team can proactively remediate coverage gaps before they result in adjudication errors or regulatory findings.

## Acceptance Criteria

Given the Compliance Officer opens the 'Fee Schedule Coverage' dashboard,
When the dashboard loads,
Then it must display four report-driven components: (1) PNAs with No Fee Schedule, (2) PNAs with Expired Fee Schedule (Expiration_Date__c < TODAY()), (3) PNAs with Fee Schedule Expiring in 0-30 Days, (4) PNAs with Fee Schedule Expiring in 31-60 Days.

Given the 'PNAs with No Fee Schedule' report component is rendered,
When data is displayed,
Then each row must include: Provider_Network_Agreement__c.Name, PNA.Provider__c (provider name), PNA.LOB__c, PNA.Effective_Date__c, PNA.Status__c, and PNA.Assigned_Network_Specialist__c.

Given any report component shows records,
When the Compliance Officer clicks a row,
Then the user must be navigated directly to the Provider_Network_Agreement__c record detail page.

Given the dashboard is refreshed,
When data is recalculated,
Then all reports must use real-time data (no scheduled refresh caching) and the dashboard header must display the Last_Refreshed_Datetime.

Given the 'PNAs Expiring in 0-30 Days' component shows 1 or more records,
When the nightly scheduled job runs,
Then an automated email digest must be sent to the Compliance Officer and all active Provider Relations Specialists listing the expiring PNA names, provider NPIs, and Fee_Schedule__c expiration dates.

Given a PNA record is remediated (a new active Fee_Schedule__c is linked),
When the dashboard is next refreshed,
Then the remediated PNA must no longer appear in any gap report component.

## Solution

Create four custom Report Types in Salesforce: (1) PNA with Fee Schedule (outer join to show nulls), (2-4) filtered variants by expiration date ranges. Build reports using the 'Provider_Network_Agreements with Fee Schedules' custom report type that joins PNA to Fee_Schedule__c via a lookup relationship. Assemble a Lightning Dashboard with four report chart components. For the email digest, implement a Scheduled Flow running nightly at 6AM that queries the same report data via a Report Run Apex class (Reports.ReportManager) and sends an email using an Apex EmailService with a Visualforce email template rendering the PNA list as an HTML table. Add a Permission Set 'Compliance_Dashboard_Access' granting View access to the dashboard folder and Read access to Fee_Schedule__c and Provider_Network_Agreement__c objects for Compliance Officers.

## Description

As a Claims Operations Manager, I need the system to automatically resolve the correct Fee_Schedule__c record based on the Claim_Line__c service date and LOB so that adjudicated amounts always reflect the contractually valid rate in effect at the time of service, eliminating manual lookups and reducing overpayment risk.

## Acceptance Criteria

Given a Claim_Line__c record is saved with Status__c = 'Pending Adjudication',
When the Record-Triggered Flow evaluates Fee_Schedule__c records linked to the Provider_Network_Agreement__c,
Then the system selects the Fee_Schedule__c where Effective_Date__c <= Claim_Line__c.Date_of_Service__c AND Expiration_Date__c >= Claim_Line__c.Date_of_Service__c AND LOB__c = Claim__c.Line_of_Business__c.

Given a matching Fee_Schedule__c record is found,
When the adjudication engine processes the CPT code on the Claim_Line__c,
Then Claim_Line__c.Allowed_Amount__c is populated from Fee_Schedule_Line__c.Rate__c where CPT_Code__c matches Claim_Line__c.CPT_Code__c.

Given no matching Fee_Schedule__c record exists for the service date,
When the flow attempts rate resolution,
Then Claim_Line__c.Adjudication_Hold_Reason__c is set to 'No Active Fee Schedule' and Claim_Line__c.Status__c is set to 'On Hold'.

Given a Fee_Schedule__c record has Fee_Schedule_Type__c = 'Medicare-Based' ,
When the rate is resolved,
Then Claim_Line__c.Allowed_Amount__c = Fee_Schedule_Line__c.Medicare_Percentage__c * CMS_Reference_Rate__c (sourced from CMS integration).

-Given manual override is attempted by a user without 'Fee Schedule Override' Permission Set, the system should silently ignore the change- (deprecated — override validation moved to INS-0902)

Given the adjudication completes successfully,
When Claim_Line__c.Status__c transitions to 'Adjudicated',
Then a Platform Event (Fee_Schedule_Applied__e) is published with Fee_Schedule__c.Id, Claim_Line__c.Id, and Applied_Rate__c payload for downstream EOB generation.

## Solution

Implement a Record-Triggered Flow on Claim_Line__c (after save, async) that queries Fee_Schedule__c and Fee_Schedule_Line__c using indexed fields (Effective_Date__c, Expiration_Date__c, CPT_Code__c, LOB__c). For Medicare-based schedules, invoke an Apex action (FeeScheduleCMSRateResolver) that calls the CMS integration via a Named Credential. Publish Fee_Schedule_Applied__e Platform Event via Apex after successful rate assignment. Add a custom index on Fee_Schedule_Line__c.CPT_Code__c for query performance. Deploy associated Permission Set Group updates for Claims Adjudicators.

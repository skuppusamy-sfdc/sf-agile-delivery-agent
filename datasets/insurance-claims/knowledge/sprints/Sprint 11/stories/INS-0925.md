## Description

As a Provider Network Specialist, I need the Provider 360 FlexCard on the Provider__c (Person Account) record page to display a consolidated view of all active Fee_Schedule__c records linked through Provider_Network_Agreement__c, including a visual indicator for schedules expiring within 60 days, so that I can proactively initiate rate renewal conversations and avoid coverage gaps that result in adjudication pends.

## Acceptance Criteria

Given a Provider__c record page is loaded by a user with the 'Provider_Network_Specialist' Permission Set Group,
When the Fee Schedules section of the FlexCard renders,
Then all Fee_Schedule__c records where Fee_Schedule__c.Effective_End_Date__c >= TODAY() linked via active PNA records are displayed in a datatable showing: Fee_Schedule__c.Name, Fee_Schedule__c.LOB__c, Fee_Schedule__c.Effective_Start_Date__c, Fee_Schedule__c.Effective_End_Date__c, and PNA.Contract_Status__c.

Given a Fee_Schedule__c.Effective_End_Date__c is within 60 days of TODAY(),
When the row is rendered,
Then a yellow warning badge labeled 'Expiring Soon' appears on that row.

Given a Fee_Schedule__c.Effective_End_Date__c is within 14 days of TODAY(),
When the row is rendered,
Then the badge color changes to red and the label reads 'Urgent Renewal Required'.

Given no active Fee_Schedule__c records exist for the provider,
When the section renders,
Then the message 'No active fee schedules on file. Please review PNA status.' is displayed.

-Given the FlexCard also displays historical expired schedules in the same table- (deprecated — moved to separate tabbed component per UX review)

Given a user clicks the 'Initiate Renewal' action on a row,
When the action is triggered,
Then an OmniScript 'FeeScheduleRenewal_OS' is launched in a modal, pre-populated with the selected Fee_Schedule__c.Id and linked PNA.Id.

## Solution

OmniStudio FlexCard 'ProviderFeeSchedules_FC' built with a DataRaptor Extract source querying Fee_Schedule__c via PNA junction. Conditional styling rules on Effective_End_Date__c using DAYS_BETWEEN formula to drive badge color. 'Initiate Renewal' action card button launches OmniScript via NavigationMixin. FlexCard embedded in the Provider__c Lightning Record Page via App Builder in the 'Fee Schedule & Contracting' tab. DataRaptor caching set to 5-minute TTL to balance performance with data freshness.

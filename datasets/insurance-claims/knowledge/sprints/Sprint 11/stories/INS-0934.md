## Description

**Bug Report – Fee Schedule Rate Resolution Failure on Modified CPT Codes**

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record with Claim__c.LOB__c = 'Commercial Health'.
2. Create a new Claim_Line__c with Claim_Line__c.CPT_Code__c = '99213-25' (CPT code with modifier '-25' appended).
3. Set Claim_Line__c.Service_Date__c to a date within an active Fee_Schedule__c effective range.
4. Set Claim_Line__c.Adjudication_Status__c = 'In Review' to trigger the fee schedule resolution flow.
5. Observe Claim_Line__c.Allowed_Amount__c and Claim_Line__c.Rate_Source__c after save.

**Expected Behavior:**
The system should strip the modifier suffix ('-25') from '99213-25', resolve the base CPT code '99213' against Fee_Schedule_Line__c.CPT_Code__c, populate Claim_Line__c.Allowed_Amount__c with the matched rate, and set Claim_Line__c.Rate_Source__c = 'Fee Schedule'.

**Actual Behavior:**
The system performs an exact string match of '99213-25' against Fee_Schedule_Line__c.CPT_Code__c. No match is found. Claim_Line__c.Adjudication_Status__c is set to 'Pended' and a manual review Task is created. Claim_Line__c.Allowed_Amount__c remains NULL. This is causing ~12% of commercial health claim lines to pend unnecessarily, per Claims Operations Manager report dated Sprint 10 retrospective.

**Environment:** Production sandbox (Acme-Full-Copy-S11)
**Regression:** Introduced in INS-0931 deployment (Sprint 11 Day 2)

## Acceptance Criteria

As per Description. Additionally: after fix deployment, regression test must confirm that Claim_Line__c records with CPT codes containing standard CMS modifiers (-25, -59, -76, -77, -91, -GT) correctly resolve to their base CPT rate. Modifier value must be stored separately in Claim_Line__c.CPT_Modifier__c for audit purposes.

## Solution

Fix the 'FeeScheduleResolutionService' Apex class introduced in INS-0931 to extract the base CPT code before performing the Fee_Schedule_Line__c lookup. Implement a utility method 'CPTCodeParser.stripModifier(String cptWithModifier)' that splits on '-' and returns the first segment. Store the modifier in Claim_Line__c.CPT_Modifier__c (new field, 10-char text). Update the SOQL query in FeeScheduleResolutionService to use the parsed base code. Add a new custom field CPT_Modifier__c to Claim_Line__c object. Deploy hotfix to Acme-Full-Copy-S11 sandbox for validation before production push. Update unit tests in FeeScheduleResolutionServiceTest to cover modifier scenarios.

## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to surface contextual resolution recommendations when a Claim__c is pended due to a fee schedule discrepancy so that claims examiners can resolve pended claims faster, reduce manual research time, and improve SLA compliance on fee schedule-related pend queues.

## Acceptance Criteria

Given a Claim__c.Status__c = 'Pended – Fee Schedule Review' and the claim examiner opens the record,
When the Agentforce Claims AI panel loads,
Then the AI must display at least one recommended action from the set: ['Apply Nearest Active Fee Schedule', 'Request Manual Rate Authorization', 'Route to Provider Relations for GSA Review', 'Escalate to SIU'] based on claim attributes.

Given the AI recommendation panel is displayed,
When the examiner selects 'Apply Nearest Active Fee Schedule',
Then Agentforce must invoke the FeeScheduleLookupService via an Agent Action and pre-populate Claim_Line__c.Allowed_Amount__c for examiner confirmation before committing.

Given the AI generates a recommendation,
When the recommendation is displayed,
Then the panel must show a confidence score (0–100%) and cite the data points used (e.g., DOS, CPT code, provider NPI, matched Fee_Schedule__c record name).

Given an examiner accepts or rejects an AI recommendation,
When the action is recorded,
Then an AI_Interaction_Log__c record must be created capturing Claim__c.Id, recommended action, confidence score, examiner decision, and timestamp for model feedback purposes.

## Solution

Configure Agentforce Agent with a custom Agent Topic 'Claims Fee Schedule Resolution' and define Agent Actions mapped to Apex invocable methods in FeeScheduleLookupService.cls and ClaimResolutionActionService.cls. Prompt template built in Einstein Prompt Builder referencing Claim__c, Claim_Line__c, Fee_Schedule__c, and Provider__c fields as grounding context. AI_Interaction_Log__c custom object created to capture feedback loop data for future model fine-tuning. Agentforce panel embedded on Claim__c Lightning Record Page via a custom LWC wrapper (agentforceClaimsPanel) that passes record context. Permission Set Group 'Claims_AI_User_PSG' controls access to AI recommendations panel.

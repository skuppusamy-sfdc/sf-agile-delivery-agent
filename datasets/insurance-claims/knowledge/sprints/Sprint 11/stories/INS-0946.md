## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to surface a recommended Fee_Schedule__c record during FNOL intake based on the provider's NPI, the reported CPT codes, and the policy's LOB, so that claims adjusters can pre-validate the applicable rate at intake and reduce downstream adjudication pend rates by at least 20%.

## Acceptance Criteria

Given a claims adjuster is completing FNOL intake on a new Claim__c record,
When the Agentforce AI panel is active and Claim__c.Provider_NPI__c and at least one Claim_Line__c.CPT_Code__c are populated,
Then the AI assistant surfaces up to 3 ranked Fee_Schedule__c recommendations with fields: PNA name, LOB, Rate__c, Effective_Date__c, Expiration_Date__c, and a confidence score.

Given the adjuster accepts a recommended Fee_Schedule__c,
When they click 'Apply Recommendation' in the Agentforce panel,
Then Claim_Line__c.Fee_Schedule__c lookup is populated with the selected record and Claim_Line__c.AI_Recommendation_Accepted__c = true for reporting purposes.

Given the Agentforce AI cannot find a matching Fee_Schedule__c (e.g., provider not in network, CPT code not contracted),
When the recommendation engine returns no results,
Then the AI panel displays: 'No contracted fee schedule found for this provider and CPT combination. Consider out-of-network adjudication pathway.' and sets Claim__c.Network_Status__c = 'Out of Network - Pending Review'.

Given the AI recommendation is applied,
When the Claim__c is saved and moves to adjudication,
Then the standard Fee Schedule validation from INS-0939 still executes as a secondary confirmation gate.

-Given the AI panel is used, bypass manual Fee Schedule lookup fields for adjusters in the 'Tier 1 Adjuster' profile- (deprecated — security review determined all profiles must retain lookup access)

Given a Claims Operations Manager reviews AI recommendation adoption,
When they access the 'Agentforce Fee Schedule Adoption' dashboard,
Then they can see: total FNOL intakes, % where AI recommendation was offered, % accepted, % that passed downstream adjudication without pend, broken down by LOB and date range.

## Solution

Configure an Agentforce Agent Action using a custom Apex Action class 'FeeScheduleRecommendationAction' that accepts NPI, CPT codes array, and LOB as inputs, queries Fee_Schedule__c with relevance scoring (exact LOB match weighted higher than 'All' LOB, recency of effective date as secondary sort), and returns a ranked list. Expose the action in the Agentforce Agent Builder under the 'Claims FNOL' topic. Build the Agentforce panel as an LWC sidebar component embedded in the Claim__c Lightning Record Page via App Builder, communicating with the Agentforce API via Lightning Message Service. Add custom fields AI_Recommendation_Accepted__c (Checkbox) and AI_Confidence_Score__c (Percent) to Claim_Line__c. Build the adoption dashboard in CRM Analytics (Tableau CRM) using a dataset joining Claim__c, Claim_Line__c, and Fee_Schedule__c. Gate the feature with a Feature Flag in Custom Metadata 'Agentforce_Feature_Flags__mdt' for phased rollout.

## Description

**Bug Report**

**Summary:** Fee_Schedule__c records associated with an expired Provider_Network_Agreement__c are incorrectly remaining in Status__c = 'Pending Activation' after their Effective_Date__c has passed, instead of transitioning to 'Active' or being flagged as 'Ineligible'. This causes Claim_Line__c adjudication to fail to resolve a valid rate, resulting in claims being stuck in 'Pended' status.

**Steps to Reproduce:**
1. Locate a Provider_Network_Agreement__c record with End_Date__c < TODAY (i.e., expired).
2. Create a Fee_Schedule__c record linked to that PNA with Effective_Date__c = yesterday and Status__c = 'Pending Activation'.
3. Run the FeeScheduleTransitionBatch manually via Developer Console.
4. Observe the Fee_Schedule__c record — Status__c remains 'Pending Activation'.
5. Attempt to adjudicate a Claim_Line__c with Service_Date__c = yesterday referencing the same PNA.
6. Observe Claim_Line__c.Status__c = 'Pended' with Pend_Reason__c = 'No Active Fee Schedule Found'.

**Expected Behavior:** The batch should either (a) transition the record to 'Active' regardless of PNA expiry status and flag it with a warning, OR (b) transition it to a new Status__c = 'Ineligible - Expired PNA' and trigger a Case for Provider Network review. Business rule to be confirmed with Claims Operations Manager.

**Actual Behavior:** The batch silently skips records where PNA.End_Date__c < TODAY due to an overly restrictive SOQL WHERE clause filter: `AND PNA__r.End_Date__c >= TODAY`.

**Environment:** Acme Insurance Full Sandbox (FS1)
**Regression:** Introduced in INS-0969 deployment on Sprint 11 Day 3.

## Acceptance Criteria

as per Description

## Solution

Remove the restrictive `AND PNA__r.End_Date__c >= TODAY` filter from the FeeScheduleTransitionBatch SOQL query. Introduce a post-query conditional branch: if PNA.End_Date__c < TODAY, set Fee_Schedule__c.Status__c = 'Ineligible - Expired PNA' and publish a Fee_Schedule_Transition_Error__e Platform Event with Error_Type__c = 'Expired PNA'. Update the batch unit tests in FeeScheduleTransitionBatch_Test to cover the expired PNA scenario. Deploy fix to FS1 for regression validation before Sprint 11 release.

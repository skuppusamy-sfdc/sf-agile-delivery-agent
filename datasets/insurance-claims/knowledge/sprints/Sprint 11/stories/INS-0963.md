## Description

As a Provider Relations Specialist, I need Provider_Change_Notification__c (PCN) records to be automatically generated and dispatched to affected providers whenever a Fee_Schedule__c rate change is published, so that we remain compliant with contractual 30-day advance notice obligations under each Provider_Network_Agreement__c.

Extends INS-0962 — relies on Fee_Schedule_Transitioned__e platform event introduced in that story.

## Acceptance Criteria

Given a Fee_Schedule_Transitioned__e platform event is received with a valid Fee_Schedule__c.Id,
When the PCN generation flow processes the event,
Then one Provider_Change_Notification__c record is created per unique Provider__c (Person Account) linked to the affected Provider_Network_Agreement__c records, with PCN.Type__c = 'Rate Change', PCN.Status__c = 'Pending Dispatch', and PCN.Effective_Date__c = Fee_Schedule__c.Effective_Start_Date__c.

Given a PCN record is created with Status__c = 'Pending Dispatch',
When the PCN.Effective_Date__c is more than 30 days in the future,
Then PCN.Compliance_Status__c = 'Compliant'; when 30 days or fewer remain, PCN.Compliance_Status__c = 'At Risk' and a High-priority Task is created and assigned to the owning Provider Relations Specialist.

Given a PCN batch is ready for dispatch,
When the Provider Relations Specialist clicks 'Send Notifications' on the PCN List View,
Then a Conga CLM merge job is triggered using the 'Rate_Change_Notice_v2' template, generating a PDF attached to each PCN record, and PCN.Status__c transitions to 'Dispatched' with PCN.Dispatched_Date__c = NOW().

-Given a PCN is dispatched, an automatic DocuSign envelope is sent requiring provider e-signature acknowledging receipt- (deprecated — legal confirmed acknowledgement is not required for rate notices; dispatch record is sufficient)

Given a provider's preferred contact method on Provider__c.Preferred_Contact_Method__c = 'Portal',
When a PCN is dispatched,
Then the PCN record is surfaced in the Provider 360 FlexCard under the 'Notifications' tab and Provider__c.Pending_PCN_Count__c (formula roll-up) is incremented.

Given a PCN remains in Status__c = 'Dispatched' for more than 14 days without a provider acknowledgement event,
When the daily follow-up batch runs,
Then an automated follow-up Task is created on the Provider__c record with Subject = 'PCN Follow-Up Required – [PCN Number]'.

## Solution

1. Platform Event trigger (after-insert Apex trigger on Fee_Schedule_Transitioned__e) invokes a Queueable Apex class (PCNGenerationQueueable) to fan out PCN record creation across all linked providers — avoids governor limits on large networks.
2. Record-Triggered Flow on Provider_Change_Notification__c (after insert) evaluates compliance date logic and creates Tasks via Create Records flow element.
3. Conga CLM integration called via Apex HttpCallout from an Invocable Action exposed as a Flow Action — template ID 'Rate_Change_Notice_v2' parameterized via Custom Metadata Type (PCN_Template_Config__mdt).
4. OmniStudio FlexCard 'Provider_360_Notifications' updated with a new DataRaptor Extract step querying Provider_Change_Notification__c WHERE Provider__c = :recordId AND Status__c != 'Closed'.
5. Scheduled Apex (PCNFollowUpBatch) runs daily to detect stale dispatched PCNs and generate follow-up Tasks.
6. Formula field Provider__c.Pending_PCN_Count__c replaced with a Roll-Up Summary equivalent via a custom Apex-maintained counter field due to Person Account architecture constraints.

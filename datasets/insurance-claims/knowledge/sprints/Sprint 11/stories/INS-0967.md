## Description

As a Compliance Officer, I need a dedicated Lightning Dashboard that surfaces fee schedule rate change events, PCN dispatch compliance status, and any at-risk notifications, so that I can demonstrate regulatory compliance with state-mandated provider notice requirements during audits and internal reviews.

This story depends on PCN records generated by INS-0963 and the Fee_Schedule_Transitioned__e events from INS-0962.

## Acceptance Criteria

Given the Compliance Officer navigates to the 'Fee Schedule Compliance' Lightning Dashboard,
When the dashboard loads,
Then it displays the following report components:
  (1) 'Rate Changes – Last 90 Days' (count of Fee_Schedule__c records where Superseded_Date__c >= LAST_90_DAYS)
  (2) 'PCN Compliance Status' (donut chart of Provider_Change_Notification__c grouped by Compliance_Status__c: Compliant / At Risk / Overdue)
  (3) 'At-Risk Notifications' (list of PCN records where Compliance_Status__c = 'At Risk' or 'Overdue', showing PCN.Name, Provider__c.Name, PCN.Effective_Date__c, PCN.Dispatched_Date__c)
  (4) 'Average Days Notice Given' (summary report on PCN records calculating average of PCN.Effective_Date__c - PCN.Dispatched_Date__c).

Given the dashboard is accessed by a user without the 'Compliance_Officer' Permission Set,
When they attempt to navigate to the dashboard,
Then they receive a 'Insufficient Privileges' error and are redirected to the home page — dashboard is not accessible via direct URL sharing.

Given a PCN record transitions to Compliance_Status__c = 'Overdue' (Effective_Date__c has passed without dispatch),
When the Compliance Officer views the dashboard,
Then the 'Overdue' segment of the donut chart is highlighted in red and clicking it drills into the filtered PCN list view.

Given the Compliance Officer needs to export audit evidence,
When they click 'Export to CSV' on the 'At-Risk Notifications' report,
Then a CSV file is generated containing all visible columns including PCN.Name, Provider__c.NPI__c, Provider__c.TIN__c, PCN.Effective_Date__c, PCN.Dispatched_Date__c, PCN.Compliance_Status__c.

## Solution

1. Create four custom Salesforce Reports using Report Builder: Fee_Schedule_Changes_90Days (Fee_Schedule__c object, filter Superseded_Date__c), PCN_Compliance_Donut (Provider_Change_Notification__c, grouped by Compliance_Status__c), PCN_At_Risk_List (Provider_Change_Notification__c, filtered At Risk/Overdue, with Provider NPI and TIN columns via cross-object fields), PCN_Avg_Notice_Days (Summary report with custom Summary Formula: AVG(PCN.Effective_Date__c - PCN.Dispatched_Date__c)).
2. Lightning Dashboard 'Fee Schedule Compliance' created in the Compliance Officers app, assembling the four report components.
3. Dashboard folder sharing restricted to 'Compliance_Officers' Public Group. Permission Set 'Compliance_Officer_PS' grants View access to the folder.
4. Overdue status logic: Scheduled Flow runs nightly, queries PCN WHERE Compliance_Status__c = 'At Risk' AND Effective_Date__c < TODAY, updates Compliance_Status__c to 'Overdue'.
5. CSV export uses native Salesforce Report Export functionality — no custom development required.

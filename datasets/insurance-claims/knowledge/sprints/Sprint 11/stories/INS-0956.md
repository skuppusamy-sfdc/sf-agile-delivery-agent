## Description

As a Provider Relations Specialist, I need a FlexCard on the Provider__c (Person Account) record page that surfaces the provider's currently active Fee_Schedule__c rates grouped by CPT code and LOB, so that I can answer provider inquiries about allowed amounts without navigating multiple related lists or running manual reports.

## Acceptance Criteria

Given I am on a Provider__c record page with an active PNA,
When the 'Active Fee Schedules' FlexCard loads,
Then it displays a filterable table showing Fee_Schedule__c.Name, Fee_Schedule__c.LOB__c, CPT code, Allowed_Amount__c, Rate_Type__c, and Effective_Date__c for all records where today's date falls within the effective range.

Given the FlexCard table is rendered,
When I filter by LOB using the dropdown (values sourced from Fee_Schedule__c.LOB__c picklist),
Then only rows matching the selected LOB are shown without a full page reload.

Given a Fee_Schedule__c record has Status__c = 'Pending Publication',
When the FlexCard renders,
Then pending records are shown in a separate 'Upcoming Rates' section with a distinct amber badge and the Effective_Date__c displayed prominently.

Given the provider has no active Fee_Schedule__c records linked via their PNA,
When the FlexCard loads,
Then an empty-state message 'No active fee schedules on file — contact Network Contracting' is displayed with a quick-action button to create a Case.

## Solution

Build OmniStudio FlexCard 'ProviderActiveFeeSchedules' with a DataRaptor Extract source querying Fee_Schedule__c via the PNA junction. Use FlexCard conditional rendering states for empty, active, and pending sections. LOB filter implemented as a FlexCard input block bound to a DataRaptor filter parameter. Deploy FlexCard to Provider__c Lightning Record Page via App Builder. Assign visibility rule: Profile IN ('Provider_Relations', 'Claims_Operations'). DataRaptor uses a parent-child SOQL across Provider_Network_Agreement__c → Fee_Schedule__c.

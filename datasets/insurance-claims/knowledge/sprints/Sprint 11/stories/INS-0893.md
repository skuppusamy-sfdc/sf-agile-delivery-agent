## Description

As a Provider Relations Specialist, I need a FlexCard on the Provider 360 page that displays all active and pending Fee_Schedule__c records associated with the provider's PNA, grouped by CPT code, so that I can quickly review current reimbursement rates and upcoming changes during provider inquiries without navigating to multiple related lists.

The FlexCard should surface both active rates (Fee_Schedule__c.Status__c = 'Active') and pending transitions (Fee_Schedule__c.Status__c = 'Pending') with clear visual differentiation.

## Acceptance Criteria

Given a Provider__c record page with the Fee Schedule FlexCard component,
When the page loads,
Then the FlexCard must display a grouped list of Fee_Schedule__c records filtered by the provider's active PNA lookup (Fee_Schedule__c.PNA__c), showing CPT_Code__c, Rate__c, Pending_Rate__c, Rate_Effective_Date__c, and Status__c.

Given a Fee_Schedule__c record with Status__c = 'Pending',
When rendered in the FlexCard,
Then the row must display a yellow badge labeled 'Rate Change Pending' and show both the current Rate__c and the incoming Pending_Rate__c side by side.

Given a provider has more than 50 active CPT codes in their fee schedule,
When the FlexCard loads,
Then pagination must be applied showing 20 records per page with a search filter input for CPT_Code__c.

Given the user has the 'Provider_ReadOnly' Permission Set,
When the FlexCard renders,
Then Rate__c and Pending_Rate__c fields must be masked as '$---.--' and a tooltip must read 'Contact your manager for rate details'.

-Given the FlexCard is embedded on the Facility__c page layout, it should display the same fee schedule data.- (Deprecated: Facility fee schedule view deferred to Phase 3)

## Solution

Build an OmniStudio FlexCard (ProviderFeeScheduleCard) with a DataRaptor Extract as the primary data source, querying Fee_Schedule__c WHERE PNA__r.Provider__c = :recordId AND Status__c IN ('Active','Pending') ORDER BY CPT_Code__c. Use a Conditional Format block in the FlexCard to apply yellow styling when Status__c = 'Pending'. Implement a Child Card for pagination using the FlexCard 'List' template with pageSize: 20 and a Search state action bound to CPT_Code__c. Rate masking handled via a Custom LWC child component (feeRateMaskField) that reads the running user's Permission Set via @wire(getPermissionSetAssignments) and conditionally renders masked vs. actual values. Embed FlexCard on Provider__c Lightning Record Page via App Builder.

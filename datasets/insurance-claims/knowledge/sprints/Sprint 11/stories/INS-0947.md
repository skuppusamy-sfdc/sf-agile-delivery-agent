## Description

As a Claims Operations Manager, I need the system to automatically transition Fee_Schedule__c records to the correct effective version when a CPT code rate change takes effect, so that Claim_Line__c adjudication always applies the contractually correct allowed amount without manual intervention.

Background: Currently, rate transitions require a manual update to Fee_Schedule__c.Status__c, which causes adjudication errors when staff miss the effective date cutover. This story automates the transition using a scheduled process keyed on Fee_Schedule__c.Effective_Date__c and Fee_Schedule__c.Termination_Date__c.

Extends INS-0917 (base Fee Schedule object model).

## Acceptance Criteria

Given a Fee_Schedule__c record with Status__c = 'Pending' and Effective_Date__c = today
When the nightly scheduled Apex job runs at 00:05 AM system time
Then Fee_Schedule__c.Status__c must be updated to 'Active' and Fee_Schedule__c.Activated_Datetime__c must be stamped with the current timestamp

Given an existing Fee_Schedule__c record with Status__c = 'Active' and Termination_Date__c = today
When the nightly scheduled Apex job runs
Then Fee_Schedule__c.Status__c must be set to 'Expired' and the previously 'Pending' successor record for the same CPT_Code__c and Provider_Network_Agreement__c lookup must be activated in the same transaction

Given a Claim_Line__c record where Adjudication_Status__c = 'Pending' and Service_Date__c falls within a newly activated Fee_Schedule__c effective range
When adjudication is triggered
Then Claim_Line__c.Allowed_Amount__c must be populated from the newly active Fee_Schedule__c.Allowed_Amount__c and Claim_Line__c.Fee_Schedule__c lookup must reference the correct active record

Given a CPT code that has no active or pending Fee_Schedule__c record on its effective date
When adjudication attempts to price the Claim_Line__c
Then Claim_Line__c.Adjudication_Status__c must be set to 'Pended - No Fee Schedule' and a Task must be created assigned to the Claims Operations queue with Subject = 'Missing Fee Schedule: [CPT Code]'

Given the scheduled job completes processing
When more than 0 Fee_Schedule__c records were transitioned
Then a Platform Event Fee_Schedule_Transition__e must be published with fields: Transition_Count__c, Job_Run_Datetime__c, and Status__c = 'Completed' for downstream audit logging

-Given a manual override flag Fee_Schedule__c.Manual_Lock__c = true, the scheduled job must skip that record and send an email to the assigned Provider Relations Specialist- (deprecated: replaced by Permission-based lock in INS-0951)

## Solution

1. Schedulable Apex class 'FeeScheduleTransitionBatch' implementing Database.Batchable and Schedulable interfaces, scheduled via Setup > Scheduled Jobs at 00:05 daily.
2. Batch query scopes Fee_Schedule__c records where (Status__c = 'Pending' AND Effective_Date__c <= TODAY) OR (Status__c = 'Active' AND Termination_Date__c <= TODAY).
3. Execute method performs status transitions in a single DML operation per batch chunk (200 records), with savepoint/rollback on partial failure.
4. Platform Event Fee_Schedule_Transition__e published in finish() method via EventBus.publish().
5. Record-Triggered Flow on Claim_Line__c (before insert/update) invokes an Apex Action 'GetActiveFeeSchedule' to resolve correct Fee_Schedule__c lookup based on CPT_Code__c, Service_Date__c, and PNA.
6. Custom metadata FeeSchedule_Config__mdt stores batch size and schedule parameters for environment portability.

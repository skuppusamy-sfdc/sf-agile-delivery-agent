## Description

As a Provider Relations Specialist, I need an automated workflow that generates a Provider_Change_Notification__c (PCN) record and sends a DocuSign envelope to the provider whenever a Fee_Schedule__c linked to an active Provider_Network_Agreement__c (PNA) is updated with a new rate tier so that providers receive timely, documented notification of rate changes and we maintain contractual compliance with state-mandated 90-day notice requirements.

## Acceptance Criteria

Given a Fee_Schedule__c record with Fee_Schedule__c.Status__c = 'Active' is updated and Fee_Schedule__c.Rate_Tier__c value changes,
When the record is saved,
Then a Provider_Change_Notification__c record must be created with PCN.Type__c = 'Rate Change', PCN.Effective_Date__c = Fee_Schedule__c.Effective_Date__c, PCN.Related_Fee_Schedule__c = Fee_Schedule__c.Id, and PCN.Status__c = 'Pending Acknowledgment'.

Given a PCN record is created for a rate change,
When PCN.Status__c = 'Pending Acknowledgment',
Then a DocuSign envelope must be sent to Provider__c.Email__c (sourced from the linked PNA.Provider__c Person Account) within 15 minutes of PCN creation.

Given the DocuSign envelope is sent,
When the provider signs the acknowledgment,
Then PCN.Status__c must update to 'Acknowledged', PCN.Acknowledged_Date__c must be stamped with the signature timestamp, and PNA.Rate_Acknowledgment_Status__c must be set to 'Current'.

Given the DocuSign envelope remains unsigned after 30 days,
When the scheduled batch job runs daily,
Then PCN.Status__c must be set to 'Overdue' and an escalation Task must be assigned to the Provider Relations queue with Priority = 'High'.

Given a PNA has PNA.Network_Status__c = 'Terminated' or 'Pending Termination',
When a rate change Fee_Schedule__c update is processed,
Then no PCN record should be created and the event must be logged in Claim__c.Adjudication_Notes__c on any open claims referencing that provider.

Given a bulk Fee_Schedule__c update affects more than 50 linked PNA records,
When the workflow triggers,
Then PCN records must be created via a Queueable Apex job to avoid governor limit violations, and a summary notification must be sent to the Claims Operations Manager.

## Solution

Record-Triggered Flow on Fee_Schedule__c (after save, async path) evaluates Rate_Tier__c change using $Record vs $Record__Prior. Flow calls Apex Action 'PCNGenerationService' which bulk-creates Provider_Change_Notification__c records and enqueues a Queueable Apex job for DocuSign envelope dispatch via DocuSign eSignature for Salesforce managed package (DocuSign.createEnvelopeFromTemplate). A separate Scheduled Flow runs nightly to identify overdue PCNs. Platform Event (PCN_Created__e) triggers downstream notification to Claims Operations Manager via Custom Notification. Extends the PCN data model introduced in INS-0901.

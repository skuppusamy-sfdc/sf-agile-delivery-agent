## Description

As a Provider Relations Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record and trigger a DocuSign envelope whenever a Fee_Schedule__c rate amendment is approved on a Provider_Network_Agreement__c, so that providers receive timely, compliant rate change notifications and our audit trail is complete. Extends INS-0909.

## Acceptance Criteria

Given a Fee_Schedule__c record linked to a Provider_Network_Agreement__c has its Status__c changed to 'Approved' by an authorized user,
When the approval is saved,
Then a Provider_Change_Notification__c record is auto-created with PCN.Type__c = 'Rate Change', PCN.Effective_Date__c = Fee_Schedule__c.Effective_Date__c, and PCN.Related_PNA__c populated.

Given the PCN record is created,
When PCN.Status__c = 'Pending Distribution',
Then a DocuSign envelope is dispatched to the provider's primary email (Provider__c.Email on the linked Person Account) within 15 minutes via Platform Event 'PCN_Distribution__e'.

Given the DocuSign envelope is sent,
When the provider signs and returns the amendment,
Then PCN.Status__c transitions to 'Acknowledged' and PNA.Amendment_Acknowledged_Date__c is stamped with the signature timestamp.

Given the provider does not acknowledge within 30 days,
When the scheduled nightly batch runs,
Then PCN.Status__c = 'Overdue - Escalated' and a Case is created and assigned to the Provider Relations Specialist queue with Priority = 'High'.

-Given a PCN is generated for a TPA-managed provider, route acknowledgment to TPA contact instead- (deprecated — TPA routing handled in GSA workflow, not PCN)

Given a PCN is generated,
When a Compliance Officer views the PCN record,
Then a FlexCard on the PCN layout displays the rate delta (old vs. new Fee_Schedule__c.Rate__c), effective date, impacted CPT codes, and DocuSign envelope status.

## Solution

Use an Approval Process on Fee_Schedule__c with a final-approval action publishing Platform Event 'PCN_Distribution__e'. An Apex Platform Event trigger subscriber creates the Provider_Change_Notification__c and invokes DocuSign eSignature REST API via named credential 'DocuSign_Prod'. Scheduled Apex batch 'PCNEscalationBatch' runs nightly to identify overdue PCNs and create escalation Cases. Build a FlexCard (OmniStudio) on PCN layout consuming a DataRaptor Extract to surface rate delta comparison. Store DocuSign template IDs in Custom Metadata 'DocuSign_Template_Config__mdt' keyed by LOB and notification type.

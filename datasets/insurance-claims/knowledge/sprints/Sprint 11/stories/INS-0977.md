## Description

As a Claims Operations Manager, I need the system to automatically resolve and apply the correct Fee_Schedule__c record to each Claim_Line__c during adjudication based on the service date, provider NPI, and LOB, so that reimbursement rates are always consistent with the contractually active fee schedule at the time of service and manual rate lookups are eliminated.

Extends INS-0947 (Provider Network Agreement rate binding).

## Acceptance Criteria

Given a Claim_Line__c record is submitted for adjudication with a populated Service_Date__c, Provider_NPI__c, and LOB__c,
When the adjudication engine processes the line,
Then the system must query Fee_Schedule__c records where Fee_Schedule__c.Effective_Date__c <= Claim_Line__c.Service_Date__c AND Fee_Schedule__c.Expiration_Date__c >= Claim_Line__c.Service_Date__c AND Fee_Schedule__c.LOB__c = Claim_Line__c.LOB__c AND Fee_Schedule__c.Status__c = 'Active'.

Given a matching Fee_Schedule__c record is found,
When the CPT code on Claim_Line__c.CPT_Code__c matches a Fee_Schedule_Line__c entry,
Then Claim_Line__c.Allowed_Amount__c must be populated with Fee_Schedule_Line__c.Allowed_Rate__c and Claim_Line__c.Fee_Schedule_Applied__c (lookup) must reference the resolved Fee_Schedule__c record ID.

Given multiple overlapping Fee_Schedule__c records exist for the same NPI, LOB, and service date,
When the engine resolves the applicable schedule,
Then the record with the most recent Fee_Schedule__c.Last_Modified_Date__c AND highest Fee_Schedule__c.Priority__c value must be selected, and a warning entry must be written to Fee_Schedule_Conflict_Log__c with Resolution_Method__c = 'Priority Override'.

Given no active Fee_Schedule__c record is found for the provider and service date combination,
When adjudication attempts to process the line,
Then Claim_Line__c.Status__c must be set to 'Pended' and Claim_Line__c.Pend_Reason__c = 'No Active Fee Schedule', and a Task must be created and assigned to the Provider Network Specialist queue with Subject = 'Fee Schedule Gap – Manual Review Required'.

-Given a Fee_Schedule__c with Status__c = 'Draft' exists for the matching criteria, the system should apply it as a fallback.- (Deprecated: Draft schedules must never be applied to live adjudication per Compliance ruling CR-2024-114.)

Given the fee schedule is successfully applied,
When the Claim_Line__c record is saved,
Then Claim__c.Fee_Schedule_Resolved__c (checkbox) must be set to TRUE and Claim__c.Last_Adjudication_Date__c must be stamped with the current datetime.

## Solution

Implement an Apex invocable method 'FeeScheduleResolver' called from a Record-Triggered Flow on Claim_Line__c (trigger: before update, entry condition: Status__c changes to 'In Adjudication'). The Apex class performs a SOQL query with date-range filtering and priority ordering on Fee_Schedule__c and Fee_Schedule_Line__c. Conflict detection logic writes to Fee_Schedule_Conflict_Log__c via a separate Apex DML operation in a try/catch block. Pend logic fires a Platform Event (Fee_Schedule_Pend__e) consumed by a downstream Flow that creates the Task in the Provider Network Specialist queue. Unit test coverage target: 90%+. Deploy via unlocked package component: FeeScheduleEngine v2.3.

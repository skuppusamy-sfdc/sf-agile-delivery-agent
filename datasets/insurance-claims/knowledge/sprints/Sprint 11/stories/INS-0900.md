## Description

As a Provider Relations Specialist, I need an automated notification workflow that creates a Provider_Change_Notification__c (PCN) record and alerts the assigned contract manager when a Provider_Network_Agreement__c Fee_Schedule__c is within 60 days of expiration so that renegotiations are initiated proactively and provider rate continuity is maintained.

## Acceptance Criteria

Given a Fee_Schedule__c record has Expiration_Date__c within 60 calendar days of today and Status__c = 'Active',
When the nightly scheduled Apex job (FeeScheduleExpirationScanner) executes,
Then a Provider_Change_Notification__c is created with PCN_Type__c = 'Rate Expiration Warning', Related_PNA__c = Provider_Network_Agreement__c.Id, and Due_Date__c = Fee_Schedule__c.Expiration_Date__c - 30 days.

Given the PCN record is created,
When the record is inserted,
Then an email notification is sent to the User referenced in Provider_Network_Agreement__c.Contract_Manager__c using an Org-Wide Email Address and a custom Email Template ('Rate_Expiration_Warning_60Day').

Given a PCN already exists for the same PNA with PCN_Type__c = 'Rate Expiration Warning' and Status__c != 'Closed',
When the scheduler runs again,
Then no duplicate PCN is created (idempotency check on Related_PNA__c + PCN_Type__c + Status__c).

Given Expiration_Date__c is within 15 days and PCN Status__c is still 'Open',
When the scheduler evaluates the record,
Then PCN Priority__c is escalated to 'Critical' and a Chatter post is made on the PNA record tagging the Contract_Manager__c user.

-Given the PNA is associated with a GSA, the GSA Contract Owner should also be notified- (deprecated — GSA notification chain handled separately in INS-0904)

## Solution

Build a Schedulable Apex class (FeeScheduleExpirationScanner) that runs nightly via a Scheduled Job. Query Fee_Schedule__c with SOQL filtering on Expiration_Date__c range and Status__c. Use a Set of existing open PCN Related_PNA__c IDs to enforce idempotency before DML insert. Leverage Messaging.SingleEmailMessage for contract manager alerts referencing a custom EmailTemplate. Implement a secondary Record-Triggered Flow on Provider_Change_Notification__c for the 15-day escalation path, posting to Chatter via ConnectApi. Add a custom report type 'Fee Schedules Nearing Expiration' for the Provider Relations dashboard.

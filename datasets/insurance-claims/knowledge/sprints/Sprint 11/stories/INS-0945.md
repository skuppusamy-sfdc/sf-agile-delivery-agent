## Description

**Bug Report**

**Summary:** When a Provider_Network_Agreement__c has more than one Contact Role defined (e.g., Primary Signatory and Secondary Signatory), the DocuSign envelope generated by the PCN distribution Platform Event (introduced in INS-0940) is sent to the Secondary Signatory instead of the Primary Signatory.

**Steps to Reproduce:**
1. Open any Provider_Network_Agreement__c record with two or more OpportunityContactRole-equivalent PNA Contact Roles where Role = 'Primary Signatory' and Role = 'Secondary Signatory'.
2. Approve a linked Fee_Schedule__c record to trigger PCN generation (see INS-0940 flow).
3. Observe the DocuSign envelope recipient in the DocuSign admin console and on the PCN record (PCN.DocuSign_Recipient_Email__c).

**Expected Result:** The DocuSign envelope is addressed to the Contact with PNA Contact Role = 'Primary Signatory' and PCN.DocuSign_Recipient_Email__c matches that contact's email.

**Actual Result:** The envelope is sent to the last Contact Role record returned by the query (Secondary Signatory), because the Apex subscriber query on PNA_Contact_Role__c lacks an ORDER BY or role filter.

**Environment:** Acme Insurance - Full Sandbox (CS47)
**Frequency:** Reproducible whenever PNA has multiple contact roles.
**Impact:** Rate change notifications are reaching non-authorized signatories, creating compliance and legal risk.

## Acceptance Criteria

As per Description.

## Solution

Fix the Apex Platform Event subscriber class 'PCNDistributionEventHandler': update the SOQL query on PNA_Contact_Role__c to include WHERE Role__c = 'Primary Signatory' LIMIT 1. Add a null-check fallback: if no Primary Signatory is found, set PCN.Status__c = 'Error - No Primary Signatory' and create a Task for the Provider Relations Specialist queue rather than sending to an unverified contact. Add unit test methods covering single-role, multi-role, and no-role scenarios. Validate fix in CS47 sandbox with existing PCN test data before production deployment.

## Description

**Bug Report**

**Summary:** Fee_Schedule__c records whose Expiration_Date__c has passed are not being automatically transitioned to Status__c = 'Expired', causing the adjudication engine to continue applying stale rates to new Claim_Line__c records.

**Steps to Reproduce:**
1. Create a Fee_Schedule__c record with Status__c = 'Published' and Expiration_Date__c = yesterday's date.
2. Wait for the nightly scheduled Apex job 'FeeScheduleExpirationBatch' to execute (or manually run via Developer Console).
3. Open the Fee_Schedule__c record and observe Status__c.
4. Create a new Claim_Line__c with Date_of_Service__c = today and a CPT code covered by the expired schedule.
5. Trigger adjudication and observe Claim_Line__c.Applied_Fee_Schedule__c.

**Expected Behavior:** After the batch runs, Fee_Schedule__c.Status__c should equal 'Expired'. New Claim_Line__c adjudication should not resolve to the expired schedule.

**Actual Behavior:** Fee_Schedule__c.Status__c remains 'Published'. The adjudication engine resolves Claim_Line__c.Applied_Fee_Schedule__c to the expired schedule and applies the outdated Unit_Rate__c.

**Environment:** Full Sandbox (FS-01)
**Frequency:** 100% reproducible
**Impact:** High — incorrect reimbursement rates applied to all claims adjudicated after a schedule's expiration date.

## Acceptance Criteria

As per Description. 

Additionally: Given the 'FeeScheduleExpirationBatch' completes a run, all Fee_Schedule__c records where Expiration_Date__c < TODAY() and Status__c = 'Published' must have Status__c updated to 'Expired'. Claim_Line__c adjudication must return 'Pended - No Rate Found' for CPT codes that only exist on expired schedules.

## Solution

Root cause identified: 'FeeScheduleExpirationBatch' Apex class has a WHERE clause bug — the SOQL filter uses 'Expiration_Date__c <= TODAY()' but TODAY() in Apex returns a Date type while Expiration_Date__c is a DateTime field, causing the comparison to silently fail. Fix: cast TODAY() to DateTime using DateTime.newInstance(Date.today(), Time.newInstance(0,0,0,0)) in the SOQL bind variable. Add a unit test asserting batch correctly expires records. Deploy fix to FS-01 for regression, then promote to production. Add monitoring via a Scheduled Flow that creates a daily Case if any Fee_Schedule__c records with Expiration_Date__c < TODAY() remain in 'Published' status.

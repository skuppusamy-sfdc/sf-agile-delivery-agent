## Description

As a Provider Relations Specialist, I need a FlexCard embedded on the Provider__c (Person Account) record page that displays the provider's current and historical Fee_Schedule__c rates grouped by CPT code and LOB so that I can quickly assess rate history during provider inquiries, dispute resolution, and contract renewal negotiations without navigating multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page,
When the Fee Schedule FlexCard loads,
Then it must display all Fee_Schedule__c records linked via the provider's active PNA where Fee_Schedule__c.Effective_Date__c is within the last 24 months, sorted by Fee_Schedule__c.Effective_Date__c descending.

Given the FlexCard is rendered,
When the data is displayed,
Then each row must show: Fee_Schedule__c.Name, Fee_Schedule__c.LOB__c, Fee_Schedule__c.Rate_Tier__c, Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.Termination_Date__c, and Fee_Schedule__c.Status__c with a color-coded badge (green = Active, yellow = Pending, grey = Expired).

Given a provider has more than 10 Fee_Schedule__c records in the 24-month window,
When the FlexCard renders,
Then pagination must be applied showing 10 records per page with a 'Load More' action.

Given the FlexCard is loaded by a user without the 'Provider_Fee_Schedule_View' Permission Set,
When the component attempts to render fee rate data,
Then rate amount fields (Fee_Schedule_Line__c.Allowed_Rate__c) must be masked and a tooltip must display 'Rate details require elevated access'.

-Given the FlexCard includes an inline edit action for Fee_Schedule__c.Rate_Tier__c- (removed from scope: rate edits must go through formal change workflow per INS-0932).

## Solution

Build OmniStudio FlexCard 'ProviderFeeScheduleCard' with a DataRaptor Extract source querying Fee_Schedule__c and Fee_Schedule_Line__c with a 24-month date filter and PNA relationship traversal. Use FlexCard conditional styling (State configuration) for Status badge color coding. Pagination implemented via DataRaptor offset/limit parameters bound to FlexCard state variables. Permission-based field masking implemented using FlexCard Conditional Visibility on rate amount fields, driven by a Custom Permission 'Provider_Fee_Schedule_View'. Embed FlexCard on Provider__c Lightning Record Page via App Builder. DataRaptor performance tuned with indexed filter on Fee_Schedule__c.Effective_Date__c.

## Description

As a Claims Operations Manager, I need a REST API endpoint exposed from Salesforce that allows our external TPA adjudication system to query the current allowed amount for a given CPT code, NPI, service date, and LOB combination in real time, so that the TPA can apply Acme's contracted fee schedule rates during their adjudication pass without requiring a nightly flat-file extract and reducing rate discrepancy errors on Claim_Line__c records imported from the TPA.

## Acceptance Criteria

Given the TPA system sends a POST request to /services/apexrest/v1/feeschedule/lookup with payload {npi, tin, cpt_code, service_date, lob}, When the Apex REST handler processes the request, Then it returns HTTP 200 with a JSON response containing: allowed_amount, fee_schedule_id, effective_date, expiry_date, pna_id, and rate_tier.

Given the NPI provided does not match an active Provider__c record with a valid PNA in the requested LOB, When the lookup executes, Then the API returns HTTP 404 with error_code = 'PROVIDER_NOT_IN_NETWORK' and an empty allowed_amount.

Given no active Fee_Schedule__c record exists for the CPT + LOB + PNA combination on the requested service_date, When the lookup executes, Then the API returns HTTP 200 with allowed_amount = null and error_code = 'NO_ACTIVE_FEE_SCHEDULE'.

Given a valid lookup request is received, When the API response is generated, Then the response time must be under 500ms for 95th percentile requests (to be validated via load test in staging).

Given the TPA authenticates using a Named Credential / Connected App with a dedicated Permission Set Group 'TPA_FeeSchedule_API_Access', When the request arrives without a valid OAuth token, Then the API returns HTTP 401.

Given any API call is made, When the request is processed (success or failure), Then an API_Audit_Log__c record is created with: Request_Payload__c, Response_Code__c, Response_Time_ms__c, Requesting_System__c = 'TPA', and Timestamp__c.

-Given the API supports SOAP/WSDL format for legacy TPA compatibility- (descoped — TPA confirmed REST-only)

## Solution

Build an Apex REST class 'FeeScheduleLookupAPI' annotated with @RestResource(urlMapping='/v1/feeschedule/lookup'). Implement @HttpPost method that accepts a custom inner class DTO for deserialization. Rate resolution logic is extracted into a reusable service class 'FeeScheduleResolutionService' (also used by the adjudication flow in INS-0969) to avoid duplication. Use SOQL with indexed fields (NPI__c on Provider__c, CPT_Code__c + Effective_Date__c on Fee_Schedule__c) for performance. Implement Apex caching via Platform Cache (org partition, TTL = 300s) for high-frequency CPT lookups. Create a Connected App 'TPA_Adjudication_Integration' with OAuth 2.0 client credentials flow. Assign Permission Set Group 'TPA_FeeSchedule_API_Access' with object-level read access to Fee_Schedule__c, Provider_Network_Agreement__c, and Provider__c. API_Audit_Log__c inserts use a @future method to avoid adding latency to the synchronous response.

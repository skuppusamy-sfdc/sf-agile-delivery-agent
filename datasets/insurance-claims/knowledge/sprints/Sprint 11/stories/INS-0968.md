# INS-0968: Agentforce Claims AI — suggest applicable Fee Schedule and expected reimbursement during FNOL intake

## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to surface the expected allowed amount and applicable fee schedule during FNOL intake, so that claims intake agents can set accurate payment expectations with claimants and flag potential cost outliers before adjudication begins.

The AI suggestion should be presented as an inline recommendation within the FNOL OmniScript, not as a blocking validation, to preserve agent autonomy.

## Acceptance Criteria

Given a claims intake agent is completing the FNOL OmniScript and has entered Claim__c.Rendering_Provider__c, Claim__c.LOB__c, and at least one Claim_Line__c.CPT_Code__c,
When the agent proceeds past the 'Service Details' step,
Then the Agentforce AI panel displays a recommendation card showing: Estimated Allowed Amount (derived from current Fee_Schedule__c rate for the entered CPT code), applicable Fee_Schedule__c.Name, and confidence level (High / Medium / Low based on data completeness).

Given the Agentforce recommendation is displayed,
When the estimated allowed amount exceeds the Claim_Line__c.Billed_Amount__c by more than 20%,
Then the recommendation card displays a yellow caution banner: 'Billed amount may be below contracted rate — verify with provider before proceeding.'

Given the rendering provider has no active PNA on file (Provider_Network_Agreement__c.Status__c != 'Active'),
When Agentforce attempts to resolve the fee schedule,
Then the AI panel displays: 'Provider is out-of-network or has no active agreement. Default out-of-network rates will apply.' and Claim__c.Network_Status__c is pre-populated as 'Out of Network'.

Given the Agentforce suggestion is displayed,
When the agent accepts the suggestion by clicking 'Apply Estimate',
Then Claim_Line__c.AI_Suggested_Allowed_Amount__c is stamped with the recommended value and Claim_Line__c.AI_Suggestion_Accepted__c = true for post-hoc accuracy reporting.

Given Agentforce is unavailable (API timeout or service degradation),
When the FNOL OmniScript reaches the 'Service Details' step,
Then the AI panel displays a graceful fallback message 'AI estimate unavailable — proceed with manual entry' and the OmniScript does not block progression.

## Solution

1. Agentforce Agent Action created: 'Resolve_Fee_Schedule_Estimate' — invokes an Apex Invocable Action (FeeScheduleEstimateAction.cls) that accepts CPT_Code__c, LOB__c, and Provider__c Id, executes the two-pass fee schedule lookup from INS-0965, and returns estimated allowed amount + confidence score.
2. Agentforce Topic configured: 'FNOL Fee Estimation' with instructions referencing the Resolve_Fee_Schedule_Estimate action and grounding it with Fee_Schedule__c object data via Einstein Data Library.
3. OmniScript 'FNOL_Intake_v3' updated to embed the Agentforce panel LWC component (agentforcePanelHost) after the 'Service Details' step. Integration Point step calls the Apex action asynchronously.
4. New fields on Claim_Line__c: AI_Suggested_Allowed_Amount__c (Currency), AI_Suggestion_Accepted__c (Checkbox), AI_Confidence_Level__c (Picklist: High/Medium/Low).
5. Graceful degradation handled via try-catch in Apex action with a custom exception class (FeeScheduleEstimateException) returning a structured fallback response object.
6. Agentforce audit logging: each AI recommendation event recorded to a custom AI_Recommendation_Log__c object for accuracy tracking and model improvement feedback loop.

## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to surface a suggested applicable Fee_Schedule__c rate and LOB classification during the FNOL intake OmniScript so that claims intake specialists can validate the expected payment range at the point of claim creation, reducing downstream adjudication pends caused by rate ambiguity and improving first-pass resolution rates.

## Acceptance Criteria

Given a claims intake specialist is completing the FNOL OmniScript and has entered Claim__c.Provider__c (NPI) and Claim_Line__c.CPT_Code__c,
When the specialist reaches the 'Rate Preview' step of the OmniScript,
Then the Agentforce AI action must query the active Fee_Schedule__c for the provider's linked PNA and return a suggested Allowed_Amount__c range (min/max across applicable rate tiers) displayed as an informational panel.

Given the Agentforce action returns a fee schedule suggestion,
When the suggestion is displayed,
Then the panel must show: Suggested Rate, Fee_Schedule__c.Name, Fee_Schedule__c.LOB__c, Fee_Schedule__c.Rate_Tier__c, and a confidence indicator (High/Medium/Low) based on whether an exact CPT match, a CPT family match, or a default rate was used.

Given the Agentforce suggestion confidence is 'Low' (default rate applied),
When the suggestion panel renders,
Then a warning banner must display: 'Rate suggestion based on default schedule. Verify CPT code accuracy before proceeding.'

Given the specialist accepts the suggested rate,
When they click 'Apply Suggested Rate',
Then Claim_Line__c.Expected_Allowed_Amount__c must be pre-populated and Claim_Line__c.Rate_Suggestion_Source__c must be set to 'Agentforce AI'.

Given the Agentforce action cannot resolve a rate (no active Fee_Schedule__c found),
When the OmniScript rate preview step loads,
Then the panel must display 'No fee schedule found for this provider and service type' and the specialist must be able to proceed without a pre-populated rate.

## Solution

Configure an Agentforce Agent Action 'SuggestFeeScheduleRate' using a custom Apex Action class that accepts NPI, CPT code, and service date as inputs and returns rate suggestion payload (JSON). The Apex class reuses the FeeScheduleResolutionService (INS-0931) resolution logic in read-only mode. Confidence scoring: 'High' = exact CPT match, 'Medium' = CPT family match (first 3 digits), 'Low' = default rate. Agentforce action invoked from the FNOL OmniScript (Step: 'Rate Preview') via an Integration Procedure that calls the Apex action and maps output to OmniScript state variables. New Claim_Line__c fields: Expected_Allowed_Amount__c (Currency), Rate_Suggestion_Source__c (Picklist). Agentforce Agent Topic configured with grounding instructions referencing Fee_Schedule__c and PNA objects. Ensure Agentforce Action respects FLS on Fee_Schedule_Line__c.Allowed_Rate__c via Permission Set Group 'Claims_Intake_Agent'.

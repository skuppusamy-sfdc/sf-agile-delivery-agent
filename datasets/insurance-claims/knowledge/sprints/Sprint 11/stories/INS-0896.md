## Description

**Bug Report — INS-0896**

**Summary:** The FeeScheduleTransitionBatch (delivered in INS-0891) uses a SOQL query filtering on `Rate_Effective_Date__c = TODAY`. When a rate effective date falls on a Saturday or Sunday, the daily batch job still runs (Monday through Friday only per the configured schedule), causing those Fee_Schedule__c records to be skipped permanently. The records remain in 'Pending' status indefinitely and are never transitioned, resulting in stale rates being applied to Claim_Line__c adjudication the following week.

**Steps to Reproduce:**
1. Create a Fee_Schedule__c record with Rate_Effective_Date__c set to the upcoming Saturday and Status__c = 'Pending'.
2. Allow the Friday night batch job to run.
3. On Monday, observe that the Fee_Schedule__c record still has Status__c = 'Pending' and Rate__c has not been updated to Pending_Rate__c.
4. Run the Monday batch job — the record is skipped because Rate_Effective_Date__c (Saturday) != TODAY (Monday).
5. Submit a Claim_Line__c on Monday referencing this Fee_Schedule__c — the old rate is applied.

**Expected Behavior:** Fee_Schedule__c records with Rate_Effective_Date__c falling on a weekend should be transitioned on the next scheduled business day (Monday), or the batch schedule should be extended to run 7 days a week.

**Actual Behavior:** Records with weekend Rate_Effective_Date__c values are permanently skipped and never transitioned without manual intervention.

**Environment:** Production sandbox (Partial Copy). Reproducible in Developer sandbox. First observed 2024-11-02 (Saturday rate date).

**Impact:** HIGH — affects all providers with weekend-dated quarterly rate changes. Estimated 47 Fee_Schedule__c records currently stuck in 'Pending' status.

## Acceptance Criteria

As per Description. Fix must ensure Fee_Schedule__c records with Rate_Effective_Date__c on a Saturday or Sunday are transitioned no later than the following Monday. SOQL query in FeeScheduleTransitionBatch must be updated to use Rate_Effective_Date__c <= TODAY AND Status__c = 'Pending' with appropriate governor limit safeguards. Regression test must confirm records with future dates (tomorrow or later) are NOT inadvertently transitioned.

## Solution

Update FeeScheduleTransitionBatch SOQL query from `WHERE Rate_Effective_Date__c = TODAY` to `WHERE Rate_Effective_Date__c <= TODAY AND Status__c = 'Pending'`. Add a scope guard in the execute() method to log a warning if batch scope exceeds 500 records (indicating a large backlog). Update the Scheduled Job to run 7 days a week to eliminate the root cause. Add a unit test (FeeScheduleTransitionBatchTest) with a @testSetup scenario creating records with past-weekend effective dates and asserting Status__c = 'Active' post-batch. Deploy fix via change set to UAT for QA validation before production push.

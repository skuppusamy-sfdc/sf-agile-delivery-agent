## Description

As a Provider Network Specialist, I need a self-service bulk import tool that allows me to upload a CSV file containing CPT code rate tables and have the system create or update Fee_Schedule__c records in bulk, so that annual rate renegotiations affecting hundreds of CPT codes across multiple PNAs can be processed in hours rather than days of manual data entry.

The tool must handle validation, duplicate detection, and error reporting without requiring Salesforce Admin involvement.

## Acceptance Criteria

Given a Provider Network Specialist accesses the Fee Schedule Import OmniScript,
When they upload a CSV file,
Then the system must validate the file structure against a required column schema: [NPI, TIN, CPT_Code, New_Rate, Effective_Date, LOB] and reject any file missing mandatory columns with a descriptive error message displayed inline.

Given a valid CSV is uploaded containing 500 rows,
When the import is initiated,
Then the system must process records asynchronously via a Queueable Apex chain, display a progress indicator in the OmniScript, and send an email notification to the uploading user upon completion with a summary of: records created, records updated, records failed.

Given a CSV row references an NPI that does not match any Provider__c.NPI__c in the system,
When that row is processed,
Then the row must be skipped, logged in Import_Error_Log__c with Import_Error_Log__c.Error_Type__c = 'Provider Not Found', and included in the failure summary report.

Given a CSV row references a CPT code that already has an active Fee_Schedule__c for the same PNA,
When the import processes that row,
Then the existing Fee_Schedule__c.Pending_Rate__c and Fee_Schedule__c.Rate_Effective_Date__c must be updated (upsert behavior) rather than creating a duplicate record, and Fee_Schedule__c.Last_Import_Date__c must be stamped.

Given the import completes with one or more failures,
When the summary screen is shown,
Then a downloadable error report CSV must be available listing all failed rows with their original data and the specific error reason.

Given the CSV contains an Effective_Date value less than 30 days from today,
When the row is validated,
Then a warning flag must be appended to the row in the preview screen: 'Short-notice rate change — compliance review required', and Fee_Schedule__c.Compliance_Review_Required__c must be set to TRUE on save.

## Solution

Build an OmniScript (FeeScheduleImport) with a File Upload step feeding a custom Apex REST endpoint (FeeScheduleImportController) that parses the CSV using a lightweight CSV parser utility class. Validation logic runs synchronously in the controller before enqueuing a Queueable Apex job (FeeScheduleImportQueueable) that chains in batches of 200. Upsert performed using Database.upsert() with a custom External ID field Fee_Schedule__c.CPT_PNA_External_Id__c (composite of CPT_Code__c + PNA__c). Errors written to Import_Error_Log__c via a separate DML list. Error CSV generation uses a ContentVersion insert with a dynamically built CSV string. OmniScript progress polling implemented via a timed SetValues action calling a DataRaptor that queries the Queueable job status via AsyncApexJob SOQL. Permission gated by 'Provider_Network_Specialist' Permission Set Group.

## Description

**Bug Report – PCN DocuSign Envelope Delivered to Stale Provider Email**

**Steps to Reproduce:**
1. Open an existing Provider__c (Person Account) record where Provider__c.PersonEmail is 'old.provider@example.com'.
2. Update Provider__c.PersonEmail to 'new.provider@example.com' and save the record.
3. Within the same business day, trigger a Fee_Schedule__c Rate_Tier__c change on a linked active PNA to initiate the PCN workflow (per INS-0932).
4. Observe the DocuSign envelope recipient email on the generated Provider_Change_Notification__c record and in the DocuSign admin console.

**Expected Behavior:**
The PCN workflow should resolve Provider__c.PersonEmail at the time of DocuSign envelope dispatch (asynchronous Queueable Apex job execution), reading the current 'new.provider@example.com' value.

**Actual Behavior:**
The DocuSign envelope is sent to 'old.provider@example.com'. Investigation reveals the Queueable Apex job 'PCNDocuSignDispatchJob' stores Provider__c.PersonEmail in a String variable at the time the job is enqueued (synchronous path of the Record-Triggered Flow), before the NPPES nightly sync (DataIntegration job) or manual update is committed to the Person Account. The stale email value is then passed directly to the DocuSign envelope API call without a re-query.

**Impact:** 7 PCN envelopes confirmed sent to stale addresses in Sprint 10 production deployment window. Compliance Officer flagged as potential state regulatory notice violation.

**Environment:** Production (Acme-PROD) and Full Copy Sandbox (Acme-Full-Copy-S11)
**Regression:** Introduced in INS-0932 deployment.

## Acceptance Criteria

As per Description. Fix must ensure the Queueable Apex job 're-queries' Provider__c.PersonEmail immediately before constructing the DocuSign envelope API call using the Provider__c.Id (not a cached email string). Post-fix validation: create a test scenario in Acme-Full-Copy-S11 where Provider email is updated and a PCN is triggered within the same transaction window; confirm envelope is delivered to the updated email address. PCN_DocuSign_Dispatch audit log must record both the email address used and a timestamp of the re-query.

## Solution

Fix 'PCNDocuSignDispatchJob' Queueable Apex class to store only Provider__c.Id (not PersonEmail) in the job's instance variables at enqueue time. In the execute() method, perform a fresh SOQL query: SELECT Id, PersonEmail FROM Account WHERE Id = :providerId immediately before the DocuSign.createEnvelopeFromTemplate() call. This ensures the most current email is used regardless of any updates between enqueue and execution. Add PCN_DocuSign_Audit__c child record on Provider_Change_Notification__c capturing: Email_Used__c, Email_Queried_At__c, Envelope_Id__c. Update the Record-Triggered Flow (INS-0932) to pass only the Provider__c.Id to the Apex Action rather than the full email string. Add a unit test in PCNDocuSignDispatchJobTest that mocks a Provider email update between enqueue and execute phases. Deploy as hotfix to production after sandbox validation sign-off by Compliance Officer.

## Description

As a Compliance Officer, I need the system to automatically generate Provider_Change_Notification__c (PCN) records whenever a Fee_Schedule__c rate change of greater than 15% (increase or decrease) is detected for a provider's active PNA, so that the organization meets state-mandated provider notification timelines and maintains an auditable record of all material rate change communications.

## Acceptance Criteria

Given a Fee_Schedule__c record transitions from 'Pending Activation' to 'Active' and the computed Variance_Percent__c exceeds 15% in absolute value, When the Record-Triggered Flow fires on Fee_Schedule__c.Status__c change, Then a Provider_Change_Notification__c record is created with: PCN_Type__c = 'Rate Change', Related_Fee_Schedule__c = the triggering record ID, Provider__c = resolved from PNA__r.Provider__c, Notification_Due_Date__c = TODAY + 30, and Status__c = 'Pending Send'.

Given the PCN record is created, When PCN_Type__c = 'Rate Change', Then PCN.Affected_CPT_Codes__c is populated with a semicolon-delimited list of all CPT_Code__c values from the same Import_Batch_ID__c that also exceeded the 15% threshold, to consolidate notifications per provider per batch.

Given a PCN record with Status__c = 'Pending Send' exists and Notification_Due_Date__c = TODAY + 7, When the daily reminder flow runs, Then an email alert is sent to the assigned Provider Relations Specialist and a Task is created on the PCN record with Subject = 'PCN Due in 7 Days - Action Required'.

Given a PCN record Status__c is manually set to 'Sent', When the record is saved, Then PCN.Sent_Date__c is auto-stamped with the current datetime and PCN.Sent_By__c is populated with the running user's ID.

Given a provider has multiple active PNA records across different LOBs, When rate changes trigger PCN creation, Then separate PCN records are created per LOB__c to support LOB-specific notification tracking.

## Solution

Implement a Record-Triggered Flow on Fee_Schedule__c that fires After update when Status__c changes to 'Active'. In the flow, compute Variance_Percent__c using a formula resource and branch on ABS(Variance_Percent__c) > 15. Use a Get Records element to find sibling Fee_Schedule__c records with the same Import_Batch_ID__c also exceeding the threshold. Call an Apex invocable method 'PCNConsolidationService.buildAffectedCPTList' to generate the semicolon-delimited CPT string (avoiding flow loop DML limits). Create Provider_Change_Notification__c via a Create Records flow element. Build a separate Scheduled Flow for the 7-day reminder. Add PCN.Sent_Date__c and PCN.Sent_By__c auto-population via a before-save Record-Triggered Flow on PCN itself.

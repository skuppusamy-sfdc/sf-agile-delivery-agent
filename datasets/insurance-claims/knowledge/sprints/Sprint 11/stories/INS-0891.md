## Description

As a Claims Operations Manager, I need the system to automatically transition Fee_Schedule__c records to the new effective rate when a rate change date is reached, so that Claim_Line__c adjudication always applies the correct CPT-based reimbursement rate without manual intervention.

Background: Currently, Provider Relations Specialists must manually update Fee_Schedule__c.Effective_Date__c and Fee_Schedule__c.Rate__c fields each time a rate change cycle occurs (typically quarterly). This has caused 14 adjudication errors in Sprint 10 due to stale rates being applied to Claim_Line__c records. This story extends INS-0861 (Fee Schedule versioning foundation).

## Acceptance Criteria

Given a Fee_Schedule__c record with a Pending_Rate__c value and a Rate_Effective_Date__c set to a future date,
When the system date equals or exceeds Rate_Effective_Date__c,
Then Fee_Schedule__c.Rate__c must be updated to equal Fee_Schedule__c.Pending_Rate__c, Fee_Schedule__c.Status__c must transition from 'Pending' to 'Active', and the prior rate must be archived to Fee_Schedule_History__c with a Archived_Date__c timestamp.

Given a Claim_Line__c record in 'Pending Adjudication' status referencing a Fee_Schedule__c that has just transitioned,
When the adjudication engine evaluates the Claim_Line__c,
Then Claim_Line__c.Allowed_Amount__c must be calculated using the newly active Fee_Schedule__c.Rate__c and not the archived rate.

Given multiple Fee_Schedule__c records for the same CPT code across different Provider_Network_Agreement__c (PNA) tiers,
When a rate transition occurs,
Then only the Fee_Schedule__c records matching the specific PNA__c lookup on the Claim_Line__c are applied, and no cross-PNA rate contamination occurs.

Given a Fee_Schedule__c rate transition has completed,
When the transition is logged,
Then a Provider_Change_Notification__c (PCN) record must be auto-created with PCN.Notification_Type__c = 'Rate Change', PCN.Effective_Date__c populated, and PCN.Status__c = 'Pending Delivery'.

-Given a manual override flag Fee_Schedule__c.Manual_Override__c = TRUE, the scheduled transition should still fire and clear the override flag.-
(Deprecated: manual override behavior moved to INS-0895)

Given the scheduled job runs at 00:01 UTC daily,
When no Fee_Schedule__c records have a Rate_Effective_Date__c matching the current date,
Then no DML operations are performed and the job completes with a 'No transitions required' log entry in Fee_Schedule_Transition_Log__c.

## Solution

Implement a Schedulable Apex class (FeeScheduleTransitionBatch) extending Database.Batchable<SObject> that queries Fee_Schedule__c WHERE Rate_Effective_Date__c = TODAY AND Status__c = 'Pending'. Batch size set to 200. On execute(), use a Record-Triggered Flow (pre-save) as a secondary guard to prevent duplicate transitions. Publish a Platform Event (Fee_Schedule_Transitioned__e) with payload including Fee_Schedule_Id__c, Old_Rate__c, New_Rate__c, and CPT_Code__c. A separate Platform Event-triggered Flow subscribes to Fee_Schedule_Transitioned__e to auto-create PCN records. Fee_Schedule_History__c inserts handled via an Apex Trigger (after update) on Fee_Schedule__c. Schedule the batch via Setup > Scheduled Jobs at 00:01 UTC. Add custom Permission Set 'Fee_Schedule_Admin' to control access to Rate__c and Pending_Rate__c fields.

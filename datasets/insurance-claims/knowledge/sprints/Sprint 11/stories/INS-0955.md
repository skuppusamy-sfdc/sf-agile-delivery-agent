## Description

As a Provider Relations Specialist, I need an automated Provider_Change_Notification__c (PCN) record created and distributed whenever a Fee_Schedule__c record is published or its rates are materially changed, so that affected providers receive timely written notice in compliance with our PNA contractual obligations and state regulatory requirements.

## Acceptance Criteria

Given a Fee_Schedule__c record's Status__c transitions to 'Published',
When the record-triggered automation fires,
Then a Provider_Change_Notification__c record is created with PCN.Type__c = 'Rate Change', PCN.Effective_Date__c = Fee_Schedule__c.Effective_Date__c, PCN.Fee_Schedule__c = the parent record ID, and PCN.Status__c = 'Pending Distribution'.

Given the PCN record is created,
When the system identifies affected providers,
Then all Provider__c (Person Account) records linked via Provider_Network_Agreement__c where PNA.Fee_Schedule__c = the updated record AND PNA.Status__c = 'Active' are added as PCN_Recipient__c junction records.

Given PCN_Recipient__c records are populated,
When the distribution job runs,
Then a Conga CLM template 'Rate_Change_Notice_v3' is merged and delivered via provider's preferred contact method stored in Provider__c.Preferred_Contact_Method__c (Email or Portal).

Given Fee_Schedule__c.Rate_Change_Percentage__c < 5,
When the PCN type is evaluated,
Then PCN.Notice_Period_Days__c = 30; when Rate_Change_Percentage__c >= 5, PCN.Notice_Period_Days__c = 60 per contractual SLA.

-Given provider has opted out of email, send paper notice via postal mail integration- (deferred to Phase 3 — postal API not yet available)

Given a PCN has been distributed,
When PCN.Distribution_Date__c is populated,
Then PCN.Status__c updates to 'Distributed' and an audit entry is written to PCN.Audit_Log__c (rich text field).

## Solution

Record-Triggered Flow on Fee_Schedule__c fires on Status__c change to 'Published', calls Apex invocable 'PCNGenerationService.createPCN()' which creates the PCN and queries related PNA records. A second scheduled Flow runs daily to process 'Pending Distribution' PCNs, invoking Conga CLM via REST callout (Named Credential: CongaCLM_Prod). Provider portal delivery uses a Platform Event (PCN_Distributed__e) consumed by the Provider Portal LWC notification bell component. Permission Set Group 'Provider_Relations_PSG' grants edit on Provider_Change_Notification__c.

## Description

As a Provider Relations Specialist, I need an automated notification process to alert affected providers when a Fee_Schedule__c record is updated or a new version is activated, so that providers receive timely rate change communications in compliance with contractual 30-day advance notice requirements embedded in Provider_Network_Agreement__c records.

This story depends on the Fee Schedule versioning model delivered in INS-0871.

## Acceptance Criteria

Given a Fee_Schedule__c record is updated with a future Effective_Date__c (at least 30 days from today) and Fee_Schedule__c.Status__c is set to 'Pending Activation',
When the record is saved,
Then the system must identify all Provider_Network_Agreement__c records where PNA.Fee_Schedule__c = the updated Fee_Schedule__c.Id AND PNA.Status__c = 'Active'.

Given the list of affected PNA records is resolved,
When the notification process executes,
Then a Provider_Change_Notification__c (PCN) record must be created for each unique Provider__c (NPI) linked to an affected PNA, with PCN.Notification_Type__c = 'Rate Change', PCN.Effective_Date__c = Fee_Schedule__c.Effective_Date__c, PCN.Status__c = 'Pending Send', and PCN.Fee_Schedule__c stamped with the source record ID.

Given PCN records are in 'Pending Send' status,
When the nightly batch process runs (scheduled for 02:00 local time),
Then each PCN must generate a DocuSign envelope via the DocuSign Apex Toolkit containing the rate change letter template (Template_ID: DS-RATE-CHG-001) pre-populated with provider NPI, TIN, effective date, and a summary of CPT code rate changes exceeding 5% variance.

Given a DocuSign envelope is sent successfully,
When the envelope status callback is received,
Then PCN.Status__c must be updated to 'Sent' and PCN.Sent_Date__c must be stamped with the current datetime.

Given a Fee_Schedule__c.Effective_Date__c is fewer than 30 days from today,
When an attempt is made to set Status__c to 'Pending Activation',
Then a validation error must be displayed: 'Rate change effective date must be at least 30 days in the future to satisfy contractual notice requirements (PNA Section 4.2).'

Given a provider has PCN.Opt_Out_Electronic_Notice__c = true,
When PCN records are processed,
Then a printed mail task (Task.Subject = 'Rate Change Letter - Mail Required') must be created and assigned to the Provider Relations queue instead of a DocuSign envelope.

-Given a GSA is linked to the Fee Schedule, GSA administrators must also receive notification.- (Deprecated: GSA notification moved to separate story INS-0882)

## Solution

Record-Triggered Flow on Fee_Schedule__c fires on update when Status__c changes to 'Pending Activation'. Flow calls an Apex invocable action (PCNGenerationService.cls) that queries affected PNA records and bulk-creates Provider_Change_Notification__c records. A scheduled Apex batch job (PCNDispatchBatch.cls, runs nightly) processes PCNs in 'Pending Send' status, invokes DocuSign Apex Toolkit for envelope creation, and handles opt-out routing via Task creation. DocuSign callback handled by a @RestResource endpoint updating PCN status. Custom metadata (Rate_Change_Config__mdt) stores notice period thresholds and DocuSign template IDs per LOB to avoid hardcoding.

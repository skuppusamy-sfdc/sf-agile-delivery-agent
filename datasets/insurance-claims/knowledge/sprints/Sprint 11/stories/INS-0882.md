## Description

As a Broker Relations Coordinator, I need the system to automatically notify Group Service Agreement (GSA) administrators and associated broker contacts when a linked Fee_Schedule__c is updated with a new rate version, so that employer group plan administrators and TPA partners receive timely, compliant advance notice of rate changes affecting their plan members, satisfying the 45-day notice requirement stipulated in GSA contractual terms.

Note: Provider-level rate change notifications are handled separately in INS-0877. This story covers the employer/group administrator notification path.

## Acceptance Criteria

Given a Fee_Schedule__c record transitions to Status__c = 'Pending Activation' and one or more Group_Service_Agreement__c records reference this fee schedule (GSA.Fee_Schedule__c = updated record ID),
When the activation event fires,
Then the system must identify all active GSAs (GSA.Status__c = 'Active') linked to the fee schedule and retrieve the GSA.Administrator_Contact__c (Contact lookup) and GSA.Broker_of_Record__c (Contact lookup) for each.

Given GSA administrator and broker contacts are identified,
When the notification batch executes,
Then a Conga CLM document generation request must be submitted for each GSA using template 'GSA-RATE-NOTICE-v3', merging fields: GSA.Name, GSA.Group_ID__c, Fee_Schedule__c.Effective_Date__c, a table of CPT codes with old vs. new rates (sourced from Fee_Schedule_Rate__c delta), and the Broker_of_Record__c.Name.

Given the Conga document is generated,
When the document is ready,
Then it must be attached to the GSA record as a ContentDocument and a DocuSign envelope must be sent to GSA.Administrator_Contact__c.Email for acknowledgement signature, with GSA.Broker_of_Record__c.Email on CC.

Given Fee_Schedule__c.Effective_Date__c is fewer than 45 days from today,
When the system attempts to link a new fee schedule version to an active GSA,
Then a validation rule on GSA must prevent the save and display: 'Group plan fee schedule changes require 45-day advance notice. Effective date must be on or after [today + 45 days].'

Given a GSA has GSA.Notification_Preference__c = 'Portal Only' (no DocuSign),
When the notification process runs,
Then a record must be created in GSA_Notification_Log__c with Status__c = 'Pending Portal Delivery' and a notification must be surfaced via the Agent Portal (Community) bell notification for the administrator's portal user.

## Solution

Subscribe a Platform Event Trigger (Apex) to the Fee_Schedule_Applied__e event published by the fee schedule activation flow (INS-0876/INS-0877). The trigger invokes a GSANotificationService Apex class that queries linked active GSAs and spawns a Queueable Apex chain: (1) Conga CLM REST API call for document generation with merge field JSON payload; (2) ContentDocument attachment via ConnectAPI; (3) DocuSign envelope creation via DocuSign Apex Toolkit. Portal-only preference handled by creating a Custom Notification (CustomNotificationType: 'GSA_Rate_Change') delivered via Salesforce Bell Notifications to the portal Community User. GSA_Notification_Log__c custom object tracks all outbound notifications with delivery status, timestamps, and document IDs. Validation rule implemented as a Formula-based rule on Group_Service_Agreement__c referencing Fee_Schedule__c.Effective_Date__c via cross-object formula.

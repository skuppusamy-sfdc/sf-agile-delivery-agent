## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to automatically suggest the correct Fee_Schedule__c record to apply to a Claim_Line__c during adjudication, based on the provider's NPI, the submitted CPT code, and the active PNA, so that claims examiners can adjudicate complex multi-line claims faster with AI-guided rate lookups reducing manual lookup time by an estimated 40%.

This story is part of the Agentforce Claims AI epic and builds on the Fee Schedule data model solidified in INS-0891 and INS-0894.

## Acceptance Criteria

Given a claims examiner opens a Claim__c record in the Agentforce Claims AI panel,
When the examiner asks 'What fee schedule applies to this claim?',
Then the AI must query Fee_Schedule__c records matching the Claim__c's primary provider NPI (via Provider__c.NPI__c) and each Claim_Line__c.CPT_Code__c, returning the top matching Fee_Schedule__c.Id, CPT_Code__c, Rate__c, and PNA__c name in a structured response within 3 seconds.

Given the AI returns a fee schedule suggestion,
When the examiner accepts the suggestion via the 'Apply' button in the AI panel,
Then Claim_Line__c.Fee_Schedule__c lookup must be populated with the suggested record ID and Claim_Line__c.AI_Assisted__c must be set to TRUE for audit reporting.

Given the provider has multiple active PNAs with different fee schedules for the same CPT code,
When the AI suggests a fee schedule,
Then it must present all matching options ranked by PNA__c.Priority__c (ascending) and clearly indicate the recommended option with a 'Best Match' label.

Given a CPT code on the Claim_Line__c has no matching active Fee_Schedule__c for the provider's NPI,
When the AI is queried,
Then it must respond with 'No active fee schedule found for CPT [code] under NPI [NPI]. Manual review required.' and create a Task assigned to the claim examiner with Task.Subject = 'Fee Schedule Gap — Manual Rate Assignment Required'.

Given a claims examiner rejects the AI suggestion,
When they select 'Dismiss',
Then an AI_Feedback__c record must be created with AI_Feedback__c.Feedback_Type__c = 'Rejected', AI_Feedback__c.Claim_Line__c populated, and AI_Feedback__c.Suggested_Fee_Schedule__c populated, to feed model improvement telemetry.

## Solution

Configure an Agentforce Agent Action (FeeScheduleSuggestionAction) as a custom Apex-backed Agent Action class implementing the AgentAction interface. The action performs a SOQL query on Fee_Schedule__c with filters on Provider__c.NPI__c (via PNA relationship), CPT_Code__c, Status__c = 'Active', and ordered by PNA__c.Priority__c. Results are serialized as a structured JSON response consumed by the Einstein Copilot prompt template (FeeScheduleSuggestionPrompt). The 'Apply' button is an LWC (agentFeeScheduleApply) embedded in the Agentforce side panel that calls an Apex method to update Claim_Line__c fields. AI_Feedback__c record creation handled by a Record-Triggered Flow on a custom 'AI Suggestion Dismissed' Platform Event. All Agentforce interactions logged in Einstein Activity Capture for audit. Requires Einstein for Service license and Agentforce configuration in Setup.

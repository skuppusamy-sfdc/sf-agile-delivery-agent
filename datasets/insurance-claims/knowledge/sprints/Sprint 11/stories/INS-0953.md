## Description

**Bug Summary:** Provider_Change_Notification__c.PCN_Status__c is not updating to 'Delivered' when the DocuSign envelope delivery webhook callback is received in the Production org. The callback returns HTTP 200 but the PCN record is not updated. The issue does not reproduce in Full Sandbox (CS47).

**Steps to Reproduce (Production):**
1. Approve a Provider_Network_Agreement__c amendment (triggers PCN creation per INS-0948).
2. Confirm Provider_Change_Notification__c is created with PCN_Status__c = 'Pending Delivery'.
3. DocuSign dispatches the envelope and sends a delivery status webhook to the Salesforce REST endpoint /services/apexrest/docusign/pcn/callback.
4. Observe the DocuSignPCNCallbackController response in DocuSign Connect logs — HTTP 200 returned.
5. Check Provider_Change_Notification__c.PCN_Status__c — still shows 'Pending Delivery'.

**Expected Result:** PCN_Status__c = 'Delivered' and Delivery_Timestamp__c populated.

**Actual Result:** PCN_Status__c remains 'Pending Delivery'. No error logs in Salesforce Debug Logs or Apex Exception Email.

**Root Cause (Identified):** The @RestResource Apex class 'DocuSignPCNCallbackController' performs a SOQL query filtering on Provider_Change_Notification__c.DocuSign_Envelope_Id__c. In Production, the DocuSign Connect configuration is sending the Envelope ID with uppercase prefix 'ENV-' prepended (e.g., 'ENV-abc123') whereas the stored value in Salesforce omits the prefix ('abc123'). The SOQL WHERE clause returns 0 rows, DML is skipped silently, and HTTP 200 is still returned.

**Environment:** Production. Sandbox unaffected because Sandbox DocuSign Connect config does not prepend 'ENV-' prefix.

## Acceptance Criteria

As per Description.

Given the fix is deployed to Production
When the DocuSign webhook sends an Envelope ID with 'ENV-' prefix
Then DocuSignPCNCallbackController must normalize the Envelope ID by stripping the 'ENV-' prefix before SOQL lookup and Provider_Change_Notification__c.PCN_Status__c must update to 'Delivered'

Given the normalization fix is in place
When an Envelope ID without the 'ENV-' prefix is received (Sandbox behavior)
Then the callback must continue to function correctly without regression

## Solution

1. Updated DocuSignPCNCallbackController.cls: added String normalization logic — envelopeId = envelopeId.startsWith('ENV-') ? envelopeId.substringAfter('ENV-') : envelopeId — before the SOQL WHERE clause.
2. Added explicit error logging: if SOQL returns 0 results, insert an Application_Error_Log__c record with Error_Context__c = 'PCN Callback - Envelope ID Not Found' and the raw payload for ops visibility.
3. Updated unit test DocuSignPCNCallbackControllerTest.cls to cover both prefixed and non-prefixed envelope ID scenarios.
4. Deployed hotfix to Production via change set 'CS-Sprint11-INS0953-Hotfix'.
5. Coordinated with DocuSign admin to align Production Connect configuration to remove 'ENV-' prefix as permanent fix; Salesforce normalization retained as defensive coding.
6. Backfilled 12 Production PCN records that were stuck in 'Pending Delivery' via anonymous Apex execution during maintenance window.

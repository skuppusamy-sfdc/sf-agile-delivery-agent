## Description

As a Provider Relations Specialist, I need a FlexCard on the Provider__c (Person Account) and Facility__c (Business Account) record pages that displays the provider's currently active Fee_Schedule__c rates and highlights any upcoming rate changes within the next 60 days, so that I can proactively manage provider communications and avoid rate dispute escalations.

The FlexCard should aggregate data across all active Provider_Network_Agreement__c records linked to the provider.

## Acceptance Criteria

Given a Provider__c record with one or more active Provider_Network_Agreement__c records,
When the Provider 360 Fee Schedule FlexCard loads,
Then it displays a table of active Fee_Schedule__c entries with columns: CPT_Code__c, LOB__c, Rate__c, Effective_Start_Date__c, Effective_End_Date__c, and PNA_Name__c.

Given a Fee_Schedule__c record where Effective_End_Date__c is within 60 days of today,
When the FlexCard renders,
Then the row is highlighted in amber and a badge 'Rate Change Pending' is displayed.

Given a Fee_Schedule__c record where a successor record exists with Effective_Start_Date__c within 60 days,
When the FlexCard renders,
Then an inline expansion shows the upcoming Rate__c and the delta percentage.

Given a Provider__c with no active PNA,
When the FlexCard loads,
Then it displays the message 'No active network agreements found' and a quick action button 'Initiate PNA' is surfaced.

-Given the FlexCard is viewed on mobile Salesforce app, a condensed 2-column layout should render-
(Mobile layout deferred to Phase 3)

Given a Provider Relations Specialist clicks 'View Full Fee Schedule',
When the action fires,
Then an OmniScript launches displaying the complete Fee_Schedule__c list with filter options for LOB__c and CPT_Code__c.

## Solution

1. DataRaptor Turbo Extract fetches Fee_Schedule__c records via PNA junction, filtering on Status__c IN ('Active','Pending') and Provider__c = :recordId.
2. FlexCard 'ProviderFeeSchedule360' built in OmniStudio with a child card for upcoming rate changes using a conditional state (amber highlight via CSS class binding).
3. Integration Procedure orchestrates the DataRaptor call and calculates Rate_Delta_Percentage using a formula action.
4. OmniScript 'FeeScheduleDetailViewer' launched via FlexCard action button with DataRaptor Extract for full schedule data.
5. FlexCard deployed to Provider__c and Facility__c Lightning record pages via App Builder.

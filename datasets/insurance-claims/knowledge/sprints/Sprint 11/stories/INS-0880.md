## Description

As a Claims Operations Manager, I need the adjudication engine to evaluate Group_Service_Agreement__c (GSA) rate tiers when processing claims for employer group policyholders, so that group-specific negotiated rates (which may differ from standard Fee_Schedule__c rates) are correctly applied, ensuring accurate EOB generation and downstream ERA file production for TPA-administered plans.

Group plans represent 38% of Acme's commercial book of business and currently require manual rate override by senior adjusters, introducing processing delays averaging 2.3 days per claim.

## Acceptance Criteria

Given a Claim__c is submitted where Policy__c.Policy_Type__c = 'Group' and Policy__c.Group_Service_Agreement__c is populated,
When the adjudication engine resolves the applicable rate,
Then it must first check for a matching GSA rate tier before falling back to the standard Fee_Schedule__c lookup hierarchy.

Given a GSA is found (GSA.Status__c = 'Active' AND GSA.Effective_Date__c <= Claim_Line__c.Date_of_Service__c AND GSA.Expiration_Date__c >= Claim_Line__c.Date_of_Service__c),
When the CPT code on Claim_Line__c.CPT_Code__c is evaluated,
Then the system must look up GSA_Rate_Tier__c records where GSA_Rate_Tier__c.GSA__c = GSA.Id AND GSA_Rate_Tier__c.CPT_Code__c = Claim_Line__c.CPT_Code__c AND GSA_Rate_Tier__c.Service_Category__c matches the claim line's ICD-10 diagnosis category.

Given a GSA rate tier match is found,
When Claim_Line__c.Allowed_Amount__c is set,
Then the value must equal GSA_Rate_Tier__c.Negotiated_Rate__c and Claim_Line__c.Rate_Source__c must be stamped as 'GSA' (not 'Fee Schedule') to ensure correct EOB line-item attribution.

Given no GSA rate tier matches the CPT code,
When adjudication falls back to the standard Fee_Schedule__c,
Then Claim_Line__c.Rate_Source__c = 'Fee Schedule' and Claim_Line__c.GSA_Fallback__c (checkbox) = true, triggering a notification Task to the TPA liaison queue.

Given a GSA has GSA.TPA_Administered__c = true,
When adjudication completes for all claim lines,
Then an ERA (835 transaction) file generation request must be queued via Platform Event (ERA_Generation_Request__e) with the TPA's EDI endpoint ID from GSA.TPA_EDI_Endpoint__c.

-Given GSA rate tiers support a 'carve-out' specialty flag, carve-out CPT codes must bypass GSA pricing and use the specialty Fee_Schedule__c.- (Deprecated: carve-out logic deferred to Phase 3 per INS-0880 scope reduction agreed in Sprint 10 planning)

## Solution

Extend the FeeScheduleResolver Apex class (from INS-0876) with a new GSARateLookup strategy implementing a Strategy design pattern. The resolver checks Policy__c.Group_Service_Agreement__c on the parent Claim__c before executing standard fee schedule logic. GSA_Rate_Tier__c records are cached using a Platform Cache Org partition (key: GSA_ID + CPT_Code) with a 4-hour TTL to reduce SOQL consumption on high-volume adjudication batches. Platform Event (ERA_Generation_Request__e) published via an after-insert Apex trigger on Payment__c when TPA flag is true. Rate_Source__c field added to Claim_Line__c as a picklist (Fee Schedule, GSA, Manual Override). Permission Set Group 'Claims_Adjudicator_PSG' updated to include access to GSA_Rate_Tier__c object.

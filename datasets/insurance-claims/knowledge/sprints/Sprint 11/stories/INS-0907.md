## Description

As a Claims Operations Manager, I need the system to automatically resolve the correct Fee_Schedule__c record based on the date of service on each Claim_Line__c so that reimbursement rates are always applied from the fee schedule that was active on the date services were rendered, eliminating manual rate lookups and reducing payment errors.

## Acceptance Criteria

Given a Claim_Line__c record is saved with a Date_of_Service__c value,
When the adjudication engine evaluates the line item,
Then the system must query Fee_Schedule__c where Effective_Date__c <= Date_of_Service__c AND (Expiration_Date__c >= Date_of_Service__c OR Expiration_Date__c = null) AND LOB__c matches Claim__c.Line_of_Business__c.

Given multiple overlapping Fee_Schedule__c records exist for the same LOB,
When the system resolves the applicable schedule,
Then it must select the record with the most recent Effective_Date__c and log the resolved Fee_Schedule__c.Id into Claim_Line__c.Applied_Fee_Schedule__c.

Given a Claim_Line__c CPT code is not found in the resolved Fee_Schedule__c line items,
When adjudication attempts to apply a rate,
Then Claim_Line__c.Adjudication_Status__c must be set to 'Pended - No Rate Found' and a Task must be created for the assigned Claims Operations Manager.

Given a Fee_Schedule__c record has been superseded by a rate change amendment,
When the prior schedule's Expiration_Date__c is reached,
Then no new Claim_Line__c records may reference the expired schedule and Claim_Line__c.Applied_Fee_Schedule__c must reflect the successor record.

-Given Fee_Schedule__c.Override_Flag__c = true, the engine applies a flat rate regardless of CPT code.- (Deprecated: override logic moved to INS-0912)

Given adjudication completes successfully,
When Claim_Line__c.Adjudication_Status__c is set to 'Adjudicated',
Then Claim_Line__c.Allowed_Amount__c must equal Fee_Schedule__c line item unit rate multiplied by Claim_Line__c.Units__c.

## Solution

Implement an Apex invocable method 'FeeScheduleResolver' called from a Record-Triggered Flow on Claim_Line__c (after save, async). The resolver performs a SOQL query against Fee_Schedule__c and related Fee_Schedule_Line__c junction object using indexed fields (Effective_Date__c, Expiration_Date__c, LOB__c, CPT_Code__c). Result is written back via a second flow element updating Applied_Fee_Schedule__c and Allowed_Amount__c. Platform Event 'Fee_Schedule_Resolution_Event__e' is published for downstream EOB generation. Add a custom index on Fee_Schedule__c.Effective_Date__c and Fee_Schedule__c.LOB__c to support query performance at volume.

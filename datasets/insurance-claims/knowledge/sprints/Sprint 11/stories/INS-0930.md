## Description

As a Claims Operations Manager, I need Agentforce to assist claims examiners during manual Claim_Line__c entry by surfacing the expected allowed amount from the applicable Fee_Schedule__c and flagging billed amounts that deviate by more than a configurable threshold, so that examiners can identify potential overbilling, data entry errors, or SIU referral candidates before the claim is submitted for adjudication. This capability extends the Agentforce Claims AI framework established in the platform readiness phase.

## Acceptance Criteria

Given a claims examiner is entering or editing a Claim_Line__c record with a valid CPT code and the parent Claim__c has a linked Provider__c with an active PNA,
When the examiner populates Claim_Line__c.CPT_Code__c and Claim_Line__c.Billed_Amount__c,
Then Agentforce displays an inline suggestion panel showing: Expected Allowed Amount (from Fee_Schedule__c), the applicable Fee_Schedule__c.Name, PNA.Contract_Type__c, and a confidence score.

Given the Claim_Line__c.Billed_Amount__c exceeds the Fee_Schedule__c expected allowed amount by more than the threshold defined in Custom Metadata 'AI_Anomaly_Threshold__mdt.Overbill_Percentage__c' (default: 25%),
When Agentforce evaluates the entry,
Then a yellow anomaly banner reads: 'Billed amount exceeds expected rate by [X]%. Review before submission.' and Claim_Line__c.AI_Anomaly_Flag__c = true is staged (not yet saved).

Given the deviation exceeds 50%,
When the anomaly is detected,
Then the banner color escalates to red and the message reads: 'Potential overbilling detected. Consider SIU referral.' with a one-click 'Flag for SIU' action that pre-populates a SIU referral Case.

Given the examiner accepts the Agentforce suggested allowed amount,
When they click 'Apply Suggestion',
Then Claim_Line__c.Allowed_Amount__c is pre-filled with the suggested value and Claim_Line__c.AI_Suggestion_Accepted__c = true is logged for model feedback telemetry.

Given Agentforce cannot resolve a Fee_Schedule__c for the entered CPT code (no active schedule on file),
When the suggestion panel renders,
Then it displays: 'No fee schedule found for this provider and CPT code. Manual review required.' and no anomaly flag is set.

Given AI_Suggestion_Accepted__c and AI_Anomaly_Flag__c are populated,
When the Claim_Line__c is saved and adjudication completes,
Then these fields are preserved on the record for downstream Compliance Reporting and model retraining data exports.

## Solution

Agentforce Agent Action built as an Apex invocable 'FeeScheduleAISuggestion.getSuggestion()' that accepts CPT code, Provider NPI, LOB, and Date of Service as inputs and returns suggested allowed amount with confidence metadata. Action registered in Agentforce Agent Builder under the 'Claims Examiner Assistant' agent topic. LWC 'claimsLineAISuggestion' renders the inline panel on the Claim_Line__c record page, communicating with the Agentforce action via @wire adapter and Einstein Generative AI API. Custom Metadata Type 'AI_Anomaly_Threshold__mdt' stores configurable thresholds per LOB. New fields AI_Anomaly_Flag__c (Checkbox), AI_Suggestion_Accepted__c (Checkbox), and AI_Confidence_Score__c (Percent) added to Claim_Line__c. SIU referral pre-population handled via NavigationMixin to new Case record with RecordType = 'SIU_Referral' and pre-mapped fields from Claim_Line__c context.

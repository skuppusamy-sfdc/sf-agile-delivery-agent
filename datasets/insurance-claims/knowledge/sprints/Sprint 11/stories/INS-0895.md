## Description

As a Claims Operations Manager, I need the ability to manually override a Fee_Schedule__c rate for a specific Claim_Line__c without altering the master fee schedule record, and have that override fully audited, so that legitimate exception payments (e.g., contracted carve-outs, settlement agreements) can be processed compliantly without corrupting the base rate table.

Note: This story inherits the deprecated override behavior originally scoped in INS-0891 AC 5.

## Acceptance Criteria

Given a Claims Operations Manager opens a Claim_Line__c record in 'Pending Adjudication' status,
When they activate the 'Override Rate' action button,
Then a modal must appear prompting for: Override_Rate__c (currency), Override_Reason__c (picklist: Settlement | Carve-Out | System Error | Other), and Override_Justification__c (text, required if reason = 'Other').

Given the override is submitted,
When saved,
Then Claim_Line__c.Rate_Override_Active__c must be set to TRUE, Claim_Line__c.Override_Rate__c must store the entered value, Claim_Line__c.Allowed_Amount__c must recalculate using Override_Rate__c instead of the linked Fee_Schedule__c.Rate__c, and a Fee_Schedule_Override_Audit__c child record must be created capturing: original rate, override rate, reason, justification, overriding user, and timestamp.

Given a Claim_Line__c has Rate_Override_Active__c = TRUE,
When the claim is submitted for payment,
Then Payment__c.Amount__c must reflect the overridden Allowed_Amount__c and Payment__c.Override_Flag__c must be set to TRUE for ERA/EOB downstream flagging.

Given a user without the 'Claims_Override_Authority' Permission Set attempts to use the Override Rate action,
When they click the button,
Then the action must be hidden from their view via a Lightning App Builder visibility filter on Profile/Permission Set, and any direct API attempt must be blocked by an Apex Trigger validation rule on Claim_Line__c.

## Solution

Create a custom Quick Action ('Override Rate') on Claim_Line__c that launches an LWC modal (claimLineRateOverride). The LWC calls an Apex method (ClaimLineOverrideController.applyOverride()) that performs all DML in a single transaction: updates Claim_Line__c fields and inserts Fee_Schedule_Override_Audit__c. Allowed_Amount__c recalculation implemented in a Before-Save Record-Triggered Flow triggered when Rate_Override_Active__c changes to TRUE or Override_Rate__c changes. Payment__c.Override_Flag__c set via a separate After-Save Flow on Claim_Line__c that finds related Payment__c records in 'Draft' status. Visibility controlled by a Lightning App Builder Component Visibility rule checking Permission Set 'Claims_Override_Authority'. Audit records surfaced on a related list on both Claim_Line__c and a Compliance Dashboard report.

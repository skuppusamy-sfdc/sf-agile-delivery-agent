## Description

**Bug Report – PCN Batch Silent Failure on Null Email**

**Environment:** Acme Insurance Full Copy Sandbox
**Related Story:** INS-0916

**Steps to Reproduce:**
1. Ensure at least one Provider__c record exists with NPI_Email__c = null and an active Provider_Network_Agreement__c.
2. Publish a Fee_Schedule__c record (Status__c = 'Published') that triggers PCN generation for the affected provider pool.
3. Allow the PCN notification batch job (PCNGenerationBatch.cls) to execute.
4. Review batch job logs and PCN_Provider_Junction__c records for the provider with null email.

**Expected Behavior:** The batch job should catch the null email condition, set PCN_Provider_Junction__c.Delivery_Status__c = 'Failed – No Email on File', log an error to the Batch_Error_Log__c object, and continue processing remaining providers without aborting.

**Actual Behavior:** When Messaging.SingleEmailMessage.setToAddresses() receives a null value, an unhandled NullPointerException is thrown in PCNGenerationBatch.cls execute() method. The entire batch chunk fails silently — no exception is logged, PCN_Provider_Junction__c.Delivery_Status__c is not updated, and subsequent providers in the same chunk are not processed.

**Root Cause:** Missing null check on Provider__c.NPI_Email__c before constructing the SingleEmailMessage object. The try/catch block in the execute() method does not log caught exceptions to Batch_Error_Log__c.

**Impact:** Providers without a registered NPI email silently drop out of rate change notifications, creating compliance exposure under PNA notification obligations.

## Acceptance Criteria

As per Description.

## Solution

Add null check for Provider__c.NPI_Email__c before constructing Messaging.SingleEmailMessage in PCNGenerationBatch.cls. If null: set PCN_Provider_Junction__c.Delivery_Status__c = 'Failed – No Email on File' and insert a Batch_Error_Log__c record with error type, provider Id, and batch job Id. Update catch block to explicitly log all caught exceptions to Batch_Error_Log__c using a utility class (BatchErrorLogger.cls). Add unit test `testNullEmailHandling()` in PCNGenerationBatchTest.cls. Validate fix with a test batch run against sandbox data containing 5 providers with null NPI_Email__c values.

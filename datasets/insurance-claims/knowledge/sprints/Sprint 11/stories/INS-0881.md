## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to proactively surface recommended resolution actions when a Claim__c is pended due to a fee schedule discrepancy, so that claims adjusters can resolve pended claims faster without manually researching applicable fee schedules, reducing average pend-resolution time from the current 3.1 days to a target of under 8 hours.

The AI assistant should leverage the Einstein for Service agent framework configured in Sprint 10 (INS-0868) and be grounded on Fee_Schedule__c, PNA, and GSA data via Retrieval Augmented Generation (RAG) over the Data Cloud unified data model.

## Acceptance Criteria

Given a Claim__c.Status__c = 'Pended' AND Claim__c.Pend_Reason__c = 'No Active Fee Schedule',
When an adjuster opens the Claim__c record and the Agentforce panel loads,
Then the AI assistant must display a suggested action card with at minimum: (a) the nearest expiring/upcoming Fee_Schedule__c for the same LOB and provider NPI, (b) a 'Apply Fee Schedule' quick action button that pre-populates the Fee_Schedule__c lookup on the Claim__c, and (c) a confidence score (%) for the suggestion.

Given a Claim__c.Pend_Reason__c = 'Fee Schedule Rate Mismatch' (billed amount exceeds allowed amount by > 20%),
When the Agentforce assistant evaluates the claim,
Then it must display the historical average allowed amount for the same CPT code and provider from the last 12 months (sourced from Data Cloud segment 'Provider_CPT_Avg_Rates') alongside the current Fee_Schedule__c rate.

Given the adjuster accepts a suggestion from the Agentforce panel,
When the 'Apply' action is triggered,
Then Claim__c.Status__c must transition from 'Pended' to 'In Adjudication', an Agentforce_Action_Log__c record must be created capturing the suggestion accepted, the adjuster's user ID, and the timestamp.

Given the Agentforce suggestion confidence score is below 60%,
When the card renders,
Then the card must display a disclaimer: 'Low confidence — manual review recommended. Contact Provider Relations for rate verification.' and the 'Apply' button must be disabled.

Given the feature is accessed by a user without the 'Agentforce_Claims_User' Permission Set,
When the Agentforce panel loads on a Claim__c record,
Then the panel must not render and no AI inference calls must be made.

## Solution

Configure an Agentforce Agent Topic ('Fee Schedule Discrepancy Resolution') with custom Actions defined as Apex invocable methods: (1) GetActiveFeeScheduleSuggestion — queries Fee_Schedule__c with LOB and NPI filters; (2) GetHistoricalCPTAverage — calls Data Cloud Query API against the Provider_CPT_Avg_Rates segment. Agent grounding uses a custom Data Cloud Data Space connecting Fee_Schedule__c, PNA, and Claim__c objects via the Salesforce CRM connector. Confidence scoring is implemented as a calculated field in the Apex action based on match specificity (exact CPT + LOB match = high, LOB-only match = low). Agentforce_Action_Log__c is a custom object with a Master-Detail to Claim__c, written via an Apex after-action callback. Permission control enforced through the 'Agentforce_Claims_User' Permission Set scoped to the agent's connected app.

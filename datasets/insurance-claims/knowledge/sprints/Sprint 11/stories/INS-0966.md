## Description

As a Provider Relations Specialist, I need the Provider 360 FlexCard on the Provider__c (Person Account) record page to display the provider's currently active Fee_Schedule__c details and any pending rate changes, so that I can answer provider inquiries about reimbursement rates without navigating to multiple related records.

This story is part of the broader Provider 360 experience initiative and complements the PCN notification work in INS-0963.

## Acceptance Criteria

Given a Provider__c record with at least one active Provider_Network_Agreement__c,
When the Provider Relations Specialist opens the Provider 360 page,
Then the 'Fee Schedule Summary' card displays: Fee_Schedule__c.Name, Fee_Schedule__c.Effective_Start_Date__c, Fee_Schedule__c.Effective_End_Date__c, Fee_Schedule__c.LOB__c, and Fee_Schedule__c.Status__c for each active PNA.

Given a Fee_Schedule__c with a successor record where Effective_Start_Date__c is within 60 days,
When the FlexCard renders,
Then an 'Upcoming Rate Change' badge is displayed in amber with the text 'New rates effective [date]' and a link to the successor Fee_Schedule__c.Name.

Given a Provider__c record with no active PNA records,
When the FlexCard renders the Fee Schedule Summary section,
Then a placeholder message 'No active network agreements on file' is displayed and the card does not throw a null pointer exception.

-Given the FlexCard, a 'Download Rate Card' button triggers a Conga merge of the full CPT rate table- (deprecated — rate card download moved to a separate self-service portal story in backlog)

Given the FlexCard loads fee schedule data,
When more than 5 PNA-linked fee schedules exist,
Then the card displays the 5 most recently effective schedules and provides a 'View All' link navigating to a filtered Fee_Schedule__c list view.

## Solution

1. OmniStudio DataRaptor Extract (DR_GetProviderFeeSchedules) queries Provider_Network_Agreement__c WHERE Provider__c = :recordId AND Status__c = 'Active', then traverses PNA.Fee_Schedule__c and PNA.Fee_Schedule__r.Successor_Schedule__c.
2. FlexCard 'Provider_Fee_Schedule_Card' built with a repeater block iterating over PNA results. Conditional state applied for 'Upcoming Rate Change' badge using FlexCard State configuration (condition: Successor_Schedule__r.Effective_Start_Date__c <= TODAY + 60).
3. Empty state block configured for null PNA scenario.
4. FlexCard embedded on Provider__c (Person Account) Lightning Record Page via Lightning App Builder, positioned within the existing 'Provider 360' tab using a Tab component.
5. DataRaptor response capped at 5 records via Extract limit configuration; 'View All' button uses a Navigate action to a pre-filtered Fee_Schedule__c list view filtered by PNA.Provider__c.

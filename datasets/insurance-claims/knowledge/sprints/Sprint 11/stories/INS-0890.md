## Description

As a Provider Network Specialist, I need a guided OmniScript-based mass upload tool that allows me to import Fee_Schedule__c records from a CSV file, validate CPT codes against the CMS code set, check for duplicate or overlapping effective date ranges, and preview the impact before committing, so that annual rate table updates can be processed accurately without requiring Salesforce admin involvement or risking data integrity issues.

The tool must support uploads of up to 5,000 Fee_Schedule__c rows per session and provide a downloadable validation error report.

## Acceptance Criteria

Given a Provider Network Specialist uploads a CSV file via the OmniScript 'Fee Schedule Mass Upload' tool,
When the file is parsed,
Then the system validates each row for required fields: CPT_Code__c, LOB__c, Rate__c, Effective_Start_Date__c, Effective_End_Date__c, and Provider_Network_Agreement__c ID, and rows with missing required fields are flagged with error code 'MISSING_REQUIRED_FIELD'.

Given a CPT_Code__c value in the uploaded file,
When validation runs,
Then the code is checked against the CMS CPT Code Reference custom metadata (CMS_CPT_Code__mdt) and invalid codes are flagged with error code 'INVALID_CPT_CODE'.

Given a row where Effective_Start_Date__c and Effective_End_Date__c overlap with an existing active Fee_Schedule__c record for the same CPT_Code__c, LOB__c, and PNA,
When validation runs,
Then the row is flagged with error code 'DATE_OVERLAP' and the conflicting Fee_Schedule__c record ID is included in the error message.

Given validation completes with one or more errors,
When the specialist reviews the preview screen,
Then valid rows are shown in green and errored rows in red, with a 'Download Error Report' button that generates a CSV of failed rows with error codes and messages.

Given all rows pass validation,
When the specialist clicks 'Commit Upload',
Then Fee_Schedule__c records are created in bulk (up to 5,000 rows) using Apex Bulk API integration, Fee_Schedule__c.Upload_Batch_ID__c is stamped with a unique batch GUID, and a confirmation screen displays total records created and a link to the Batch Summary Report.

Given a batch upload exceeds 5,000 rows,
When the file is submitted,
Then the system rejects the upload with error message 'Upload limit exceeded. Maximum 5,000 rows per session. Please split your file and retry.'

-Given the upload tool is accessible from the Provider Network Agreement related list, a mini inline upload widget should be available-

Given a completed upload batch,
When the Compliance Officer accesses the Fee Schedule Audit Report,
Then the batch is listed with Upload_Batch_ID__c, Uploaded_By__c, Upload_Timestamp__c, Row_Count__c, and Error_Count__c.

## Solution

1. OmniScript 'FeeScheduleMassUpload' with steps: File Upload (LWC file input component), Validation Preview (DataRaptor Transform for CSV parsing + Apex validation action), Commit Confirmation.
2. Apex class FeeScheduleUploadValidator handles CPT code validation against CMS_CPT_Code__mdt custom metadata and overlap detection via SOQL aggregate query.
3. Apex class FeeScheduleUploadCommitter uses Database.insertImmediate (Bulk API 2.0 via REST callout from Apex) for 5,000-row inserts with partial success handling.
4. DataRaptor Load creates Fee_Schedule_Upload_Batch__c header record for audit trail.
5. LWC 'ErrorReportDownloader' generates client-side CSV from validation results JSON.
6. Custom Report Type 'Fee Schedule Upload Audit' built on Fee_Schedule_Upload_Batch__c for Compliance Officer visibility.
7. Permission Set 'FeeSchedule_BulkUpload' grants OmniScript access to Provider Network Specialists only.

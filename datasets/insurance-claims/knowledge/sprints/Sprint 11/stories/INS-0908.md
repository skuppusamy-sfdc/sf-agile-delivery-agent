## Description

As a Provider Relations Specialist, I need an automated Provider_Change_Notification__c (PCN) record to be created and distributed to affected providers whenever a Fee_Schedule__c record is published or amended so that providers receive timely, accurate notice of rate changes as required by state mandate and PNA contractual obligations. Extends INS-0877.

## Acceptance Criteria

Given a Fee_Schedule__c record's Status__c is changed from 'Draft' to 'Published',
When the record is saved,
Then a Provider_Change_Notification__c record must be created for each active Provider_Network_Agreement__c where PNA.Fee_Schedule__c = the updated schedule's Id and PNA.Status__c = 'Active'.

Given the PCN record is created,
When Provider_Change_Notification__c.Notification_Type__c = 'Rate Change',
Then PCN.Effective_Date__c must equal Fee_Schedule__c.Effective_Date__c and PCN.Provider__c must be populated from PNA.Provider__c.

Given a PCN record is created,
When PCN.Delivery_Method__c = 'Email',
Then an email alert must be dispatched to Provider__c.NPI_Email__c within 24 hours using an org-wide email address designated for provider communications.

Given a provider's PNA covers multiple LOBs impacted by the same fee schedule update,
When PCN records are generated,
Then one consolidated PCN must be created per provider (not one per LOB) with PCN.Affected_LOBs__c populated as a multi-select picklist.

Given the PCN batch completes,
When all notifications are dispatched,
Then Fee_Schedule__c.Notification_Status__c must be updated to 'Notifications Sent' and Fee_Schedule__c.Notification_Sent_Date__c stamped with the current datetime.

## Solution

Record-Triggered Flow on Fee_Schedule__c fires on Status__c change to 'Published', calling an Apex queueable class 'PCNGeneratorQueueable'. The queueable queries affected PNA records, deduplicates by Provider__c, and bulk-inserts Provider_Change_Notification__c records. A second Apex class 'PCNEmailDispatcher' is scheduled to run hourly and processes PCN records where Delivery_Status__c = 'Pending'. Email templates are managed via Salesforce Email Templates with merge fields referencing PCN and Fee_Schedule__c. Notification_Status__c on Fee_Schedule__c is updated via a final Platform Event subscriber flow.

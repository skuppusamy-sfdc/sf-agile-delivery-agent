## Description

As a Provider Relations Specialist, I need a FlexCard on the Provider 360 page that displays all active Fee_Schedule__c records associated with the provider's Provider_Network_Agreement__c records, grouped by Line of Business (LOB) and CPT code range, so that I can quickly validate contracted rates during provider calls without navigating multiple related lists.

The FlexCard must support filtering by LOB__c and display a visual indicator when any rate is within 30 days of its Termination_Date__c.

## Acceptance Criteria

Given a Provider (Person Account) record page with an active PNA
When the Provider 360 page loads
Then the Fee Schedule FlexCard must display all Fee_Schedule__c records where Status__c = 'Active' and Related_PNA__r.Provider__c = the current Provider Id, grouped by Fee_Schedule__c.LOB__c

Given the FlexCard is rendered
When a Fee_Schedule__c.Termination_Date__c is within 30 calendar days of today
Then that row must display an amber warning badge with label 'Expiring Soon' and the Termination_Date__c value

Given the LOB filter dropdown on the FlexCard
When a user selects a specific LOB value (e.g., 'Commercial', 'Medicare Advantage', 'Medicaid')
Then the FlexCard must re-render showing only Fee_Schedule__c records matching the selected Fee_Schedule__c.LOB__c value without a full page reload

Given a provider with no active Fee_Schedule__c records
When the FlexCard renders
Then an empty state message 'No active fee schedules found for this provider' must be displayed with a button linking to New Fee_Schedule__c creation

Given the FlexCard
When a user clicks on a CPT_Code__c row
Then a modal must open displaying Fee_Schedule__c detail fields: Allowed_Amount__c, Modifier__c, Billing_Code_Type__c, Effective_Date__c, Termination_Date__c, and the related PNA name

## Solution

1. OmniStudio FlexCard 'ProviderFeeScheduleCard' built with DataRaptor Extract source querying Fee_Schedule__c with filters on Status__c = 'Active' and PNA provider lookup.
2. DataRaptor 'DR_GetProviderActiveFeeSchedules' uses SOQL with relationship traversal: SELECT Id, CPT_Code__c, Allowed_Amount__c, LOB__c, Effective_Date__c, Termination_Date__c, Modifier__c, Billing_Code_Type__c FROM Fee_Schedule__c WHERE Related_PNA__r.Provider__c = :recordId AND Status__c = 'Active'.
3. FlexCard conditional styling on Termination_Date__c using a Formula field comparing to TODAY() + 30.
4. LOB filter implemented as a FlexCard State Action updating a filter parameter fed back into the DataRaptor input.
5. Row-click action triggers a child FlexCard flyout panel rendering Fee_Schedule__c detail fields.
6. FlexCard embedded on Provider (Person Account) Lightning Record Page via App Builder, wrapped in a Tab component alongside existing provider panels.
7. DataRaptor and FlexCard deployed via OmniStudio migration tool (JSON export/import) across sandboxes.

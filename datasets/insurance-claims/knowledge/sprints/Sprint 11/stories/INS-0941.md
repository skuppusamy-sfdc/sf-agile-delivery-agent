## Description

As a Provider Network Specialist, I need a guided OmniScript interface to bulk-import annual CPT rate updates into Fee_Schedule__c records from a CSV file, so that the manual data entry burden during annual rate negotiations is eliminated and import errors are caught before records are committed.

## Acceptance Criteria

Given a Provider Network Specialist navigates to the Fee Schedule Import OmniScript,
When they upload a CSV file containing columns [CPT_Code, Rate, Effective_Date, Expiration_Date, LOB, NPI],
Then the OmniScript validates the CSV schema before proceeding and displays a row-count summary.

Given the CSV is validated,
When the DataRaptor Transform processes each row,
Then Fee_Schedule__c records are upserted using an external ID field Fee_Schedule__c.External_Rate_Key__c (composite of NPI + CPT_Code + Effective_Date).

Given any row fails validation (e.g., invalid CPT code format, missing NPI, overlapping effective dates),
When the import completes,
Then a downloadable error report is presented listing row number, field name, and error reason; failed rows are not committed.

Given the import succeeds for all rows,
When Fee_Schedule__c records are created/updated,
Then Fee_Schedule__c.Status__c = 'Pending Approval' and the existing approval process is triggered automatically.

Given the import contains more than 5,000 rows,
When the OmniScript submits the job,
Then processing is handed off to an Apex Queueable job and the user receives an email notification upon completion.

## Solution

Build an OmniScript (Type: Fee Schedule, Sub-type: Bulk Import) with a File Upload step, a DataRaptor Transform for schema validation and field mapping, and a branching Integration Procedure that calls an Apex invocable method 'FeeScheduleBulkUpsertService' for large payloads (>5,000 rows) using Queueable Apex. For smaller payloads, use DataRaptor Load directly. Error reporting is generated via a DataRaptor Extract writing to a ContentDocument linked to the running user. Custom Metadata 'CPT_Validation_Rules__mdt' drives CPT format checks without hardcoding.

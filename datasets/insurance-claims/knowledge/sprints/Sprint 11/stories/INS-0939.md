## Description

As a Claims Operations Manager, I need the system to automatically validate that the Fee_Schedule__c record applied to each Claim_Line__c falls within its effective date range at the time of service, so that adjudication errors due to expired or future-dated fee schedules are eliminated and payment accuracy is improved across all LOBs.

## Acceptance Criteria

Given a Claim_Line__c record enters adjudication (Claim_Line__c.Status__c = 'Pending Adjudication'),
When the system evaluates the linked Fee_Schedule__c record,
Then Fee_Schedule__c.Effective_Date__c must be <= Claim_Line__c.Date_of_Service__c AND Fee_Schedule__c.Expiration_Date__c must be >= Claim_Line__c.Date_of_Service__c.

Given a Fee_Schedule__c record is expired or not yet effective for the date of service,
When the validation fails,
Then Claim_Line__c.Status__c must be set to 'Pended - Fee Schedule Mismatch' and a Task is auto-created and assigned to the Provider Network Specialist queue.

Given a valid Fee_Schedule__c is found but the CPT code on Claim_Line__c.CPT_Code__c is not listed in the fee schedule,
When adjudication is attempted,
Then Claim_Line__c.Adjudication_Error_Code__c = 'CPT_NOT_IN_SCHEDULE' and the claim line is pended.

Given multiple Fee_Schedule__c records exist for the same Provider_Network_Agreement__c and LOB,
When selecting the applicable schedule,
Then the system must apply the most recently effective schedule that covers the date of service (no duplicate application).

-Given a manual override flag Fee_Schedule__c.Override_Flag__c = true, bypass all validations- (deprecated — override logic moved to INS-0944)

Given validation passes successfully,
When the Claim_Line__c proceeds to payment calculation,
Then Claim_Line__c.Allowed_Amount__c is populated from Fee_Schedule__c.Rate__c and Claim_Line__c.Adjudication_Status__c = 'Fee Schedule Applied'.

## Solution

Implement a Record-Triggered Flow on Claim_Line__c firing on create/update when Status__c changes to 'Pending Adjudication'. Flow calls an invocable Apex class 'FeeScheduleValidationService' that queries Fee_Schedule__c with SOQL filtering on Effective_Date__c, Expiration_Date__c, and related PNA. Apex returns a validation result wrapper consumed by the Flow to branch into success/failure paths. On failure, Flow creates a Task via standard action and updates Claim_Line__c fields. On success, Flow sets Allowed_Amount__c. Use Custom Metadata Type 'Fee_Schedule_Config__mdt' to store LOB-specific validation rules without code changes. Deploy with associated unit tests targeting 90%+ coverage.

## Description

As a Provider Relations Specialist, I need a FlexCard on the Provider 360 console that surfaces all upcoming and recent Fee_Schedule__c rate changes associated with a provider's active Provider_Network_Agreement__c records, so that I can proactively communicate changes to providers before they take effect and reduce inbound provider calls regarding unexpected reimbursement amounts. Extends INS-0955 (Provider 360 base FlexCard framework).

## Acceptance Criteria

Given a Provider__c (Person Account) record is open in the console, When the Rate Changes FlexCard loads, Then it displays all Fee_Schedule__c records linked through PNA__c WHERE Effective_Date__c >= TODAY - 30 AND Effective_Date__c <= TODAY + 90, ordered by Effective_Date__c ascending.

Given the FlexCard data is loaded, When a rate change row is rendered, Then it displays: CPT_Code__c, Service_Description__c, Current_Rate__c (from Active record), New_Rate__c (from Pending Activation record), Effective_Date__c, LOB__c, and a calculated Variance_Percent__c field.

Given Variance_Percent__c is greater than 10% (increase or decrease), When the row is rendered, Then a visual indicator (amber badge for >10%, red badge for >25%) is applied using FlexCard state actions.

Given no upcoming rate changes exist within the 90-day window, When the FlexCard loads, Then a 'No upcoming rate changes' empty state message is displayed.

Given the Provider Relations Specialist clicks 'Send Notification', When the action fires, Then an OmniScript is launched pre-populated with the provider NPI, TIN, affected CPT codes, and Effective_Date__c values to generate a Provider_Change_Notification__c record.

## Solution

Build OmniStudio FlexCard 'Provider_Rate_Changes_FC' using a DataRaptor Extract as the data source, joining Fee_Schedule__c to Provider_Network_Agreement__c to Account. Use a child DataRaptor Transform to compute Variance_Percent__c = ((New_Rate__c - Current_Rate__c) / Current_Rate__c) * 100. Apply FlexCard Conditional State Actions for badge color logic using state thresholds. The 'Send Notification' action button invokes the existing PCN_Generation_OS OmniScript (built in INS-0955) via a FlexCard Action of type OmniScript. Deploy the FlexCard as an Aura-wrapped LWC embedded in the Provider__c Lightning Record Page using App Builder.

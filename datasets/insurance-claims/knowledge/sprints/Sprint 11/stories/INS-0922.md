# INS-0922: Create compliance report for fee schedule rate changes delivered within PNA-required notice window

## Description

As a Compliance Officer, I need a Salesforce report and dashboard component that tracks whether Provider_Change_Notification__c records for fee schedule rate changes were delivered within the contractually required advance notice window defined on each Provider_Network_Agreement__c so that the organization can demonstrate regulatory compliance, identify at-risk notifications, and respond to state insurance department audits with accurate delivery evidence.

## Acceptance Criteria

Given the compliance dashboard is accessed by a user with the 'Compliance_Officer' Permission Set,
When the 'Fee Schedule Notice Compliance' report is loaded,
Then it must display all PCN records where PCN.Type__c = 'Rate Change' created in the selected date range, grouped by LOB and Provider_Network_Agreement__c.

Given a PCN record is included in the report,
When the compliance window calculation runs,
Then the report must compute Notice_Days_Delivered__c = Fee_Schedule__c.Effective_Date__c - PCN.Delivery_Timestamp__c (in calendar days) and compare against PNA.Required_Notice_Days__c.

Given the notice window calculation is complete,
When the report renders each PCN row,
Then rows where Notice_Days_Delivered__c >= PNA.Required_Notice_Days__c must display a 'Compliant' indicator; rows where Notice_Days_Delivered__c < PNA.Required_Notice_Days__c must display 'Non-Compliant' in red.

Given the dashboard component is rendered,
When data is aggregated,
Then a summary tile must show: total PCNs issued, % compliant, % non-compliant, and count of PCNs with Delivery_Status__c = 'Failed – No Email on File'.

-Given non-compliant PCNs exist, auto-generate a Salesforce Case for each.- (Deferred to Sprint 12 per Product Owner decision; manual follow-up process remains in place.)

Given the report is run,
When a Compliance Officer exports data,
Then the export must include PCN.Id, Provider__c.NPI__c, Provider__c.TIN__c, PNA.Agreement_Number__c, Fee_Schedule__c.Name, Delivery_Timestamp__c, Effective_Date__c, Notice_Days_Delivered__c, and compliance status for audit package preparation.

## Solution

Create a custom report type 'Provider Change Notifications with PNA and Fee Schedule' joining Provider_Change_Notification__c → Provider_Network_Agreement__c → Fee_Schedule__c. Add a formula field Notice_Days_Delivered__c (Number) on PCN object: `Fee_Schedule__c.Effective_Date__c - PCN.Delivery_Timestamp__c`. Add a formula field Compliance_Status__c (Text) using IF logic against PNA.Required_Notice_Days__c. Build a Salesforce Report (tabular with summary) and embed as a Dashboard component using a vertical bar chart for compliant vs. non-compliant counts by LOB. Apply Report Folder sharing to restrict access to Compliance_Officer Permission Set Group. Schedule report refresh daily at 06:00 UTC via Report Subscription.

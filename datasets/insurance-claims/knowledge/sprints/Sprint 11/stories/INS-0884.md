## Description

As a Provider Relations Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record and trigger a DocuSign envelope whenever a Provider_Network_Agreement__c (PNA) is amended with new Fee_Schedule__c rates, so that providers receive timely, compliant rate change notices within the contractually required 30-day window.

This story covers the PCN generation logic, DocuSign template binding, and delivery tracking. Integration with Conga CLM for amendment document assembly is in scope.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where Amendment_Status__c changes to 'Rate Amendment Pending' and a linked Fee_Schedule__c record has Rate_Change_Percentage__c != 0,
When the record is saved,
Then a Provider_Change_Notification__c is created with PCN_Type__c = 'Rate Change', Effective_Date__c = Fee_Schedule__c.Effective_Start_Date__c, and PCN_Status__c = 'Draft'.

Given a PCN record in 'Draft' status,
When PCN_Status__c is updated to 'Approved for Delivery',
Then a DocuSign envelope is sent to the Provider__c (Person Account) email address on file with the Conga-assembled rate amendment PDF attached and DocuSign_Envelope_ID__c is stamped on the PCN record.

Given a DocuSign envelope is sent,
When the provider signs the acknowledgment,
Then PCN_Status__c updates to 'Acknowledged', Provider_Network_Agreement__c.Amendment_Status__c updates to 'Executed', and the signed document is stored in the PCN record's Files section.

Given a PCN with Effective_Date__c within 30 days of creation,
When PCN_Status__c remains 'Sent - Awaiting Acknowledgment' for more than 15 days,
Then an automated reminder email is sent to the provider and a Task is created for the assigned Provider Relations Specialist.

Given a Facility__c (Business Account) on a Group_Service_Agreement__c (GSA) rather than an individual PNA,
When a rate amendment is processed,
Then PCN records are bulk-generated for all active Provider__c records linked to that Facility__c with PCN_Type__c = 'GSA Rate Change'.

-Given PCN delivery via fax, the system should log a Fax_Confirmation__c child record-
(Fax delivery channel deprecated in Phase 2; email and portal delivery only)

## Solution

1. Record-Triggered Flow on Provider_Network_Agreement__c (after-save) detects Amendment_Status__c = 'Rate Amendment Pending' and invokes an Apex action (PCNGenerationService) to create PCN records.
2. PCNGenerationService handles both individual PNA and GSA bulk-generation scenarios using a loop with chunked DML.
3. Conga CLM API callout assembles the rate amendment PDF using a pre-built template bound to Fee_Schedule__c merge fields.
4. DocuSign eSignature Apex Toolkit sends the envelope and stores Envelope ID via callback.
5. A Scheduled Flow checks for overdue PCNs daily and creates follow-up Tasks.
6. OmniScript 'PCN Review & Approve' provides the Provider Relations Specialist a guided UI for PCN review before dispatch.

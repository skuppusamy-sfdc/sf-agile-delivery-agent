## Description

As a Compliance Officer, I need a governed approval process and immutable audit trail whenever a Claims Adjuster manually overrides a Claim_Line__c.Allowed_Amount__c that deviates from the resolved Fee_Schedule__c rate so that all manual rate adjustments are justified, approved by a Claims Operations Manager, and fully auditable for regulatory review. Extends INS-0899.

## Acceptance Criteria

Given a Claims Adjuster edits Claim_Line__c.Allowed_Amount__c to a value different from Claim_Line__c.Fee_Schedule_Rate__c,
When the record is saved,
Then Claim_Line__c.Override_Requested__c is set to TRUE, Claim_Line__c.Override_Justification__c is required (validation rule fires if blank), and Claim_Line__c.Status__c is set to 'Pending Override Approval'.

Given Claim_Line__c.Override_Requested__c = TRUE,
When the record save completes,
Then a Salesforce Approval Process ('Fee_Schedule_Override_Approval') is submitted automatically, routing to the queue 'Claims_Override_Approvers'.

Given the approval is rejected,
When the Approval Process final rejection step executes,
Then Claim_Line__c.Allowed_Amount__c is reverted to Claim_Line__c.Fee_Schedule_Rate__c via a Flow-triggered field update, Override_Requested__c is set to FALSE, and a task is created for the adjuster.

Given the approval is approved,
When the Approval Process final approval step executes,
Then Claim_Line__c.Status__c transitions back to 'Adjudicated', Override_Approved_By__c = approving User.Id, and Override_Approved_Date__c = System.today().

Given any override action (submit, approve, reject),
When the action occurs,
Then a Fee_Schedule_Override_Audit__c record is created capturing Claim_Line__c.Id, Old_Amount__c, New_Amount__c, Action__c, Acting_User__c, and Timestamp__c — this record must be non-deletable for all profiles.

Given a user with 'Fee Schedule Override' Permission Set attempts to save an override exceeding 20% variance from Fee_Schedule_Rate__c,
When the validation fires,
Then an error message is returned: 'Override amount exceeds 20% threshold. Escalation approval required.' and the record is not saved.

-Given the override variance is less than 5%, auto-approval should be granted without manager review- (deprecated — all overrides require explicit approval per compliance directive CR-2024-117)

## Solution

Implement a Validation Rule on Claim_Line__c enforcing Override_Justification__c when Allowed_Amount__c != Fee_Schedule_Rate__c. Build a Record-Triggered Flow (before save) to set Override_Requested__c and status. Configure a standard Salesforce Approval Process with entry criteria on Override_Requested__c = TRUE, routing to a public group. Use an Apex-invocable action within the approval process final steps to handle amount reversion and Fee_Schedule_Override_Audit__c record creation. Set the Fee_Schedule_Override_Audit__c object's 'Allow Activities' and restrict Delete via a Profile-level object setting and Apex trigger blocking delete DML. Add a custom Permission Set 'Fee_Schedule_Override' and include in the Claims_Adjuster_Senior Permission Set Group.

## Description

As a Provider Relations Specialist, I need a FlexCard on the Provider__c (Person Account) record page that displays the provider's currently active Fee Schedule, key contracted rate highlights, and any pending rate changes, so that I can quickly assess a provider's compensation structure without navigating to multiple related records during provider inquiries.

UX mockups approved in Figma (Acme-FeeCard-v2). Extends the Provider 360 dashboard introduced in INS-0854.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page,
When the 'Fee Schedule & Rates' FlexCard loads,
Then it must display the active Fee_Schedule__c.Name, Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.Expiration_Date__c, and Fee_Schedule__c.Schedule_Type__c for each active PNA linked to the provider.

Given the provider has more than one active PNA across different LOBs,
When the FlexCard renders,
Then rates must be grouped by PNA.Line_of_Business__c with a collapsible section per LOB showing the top 10 CPT codes by claim volume (sourced from a pre-aggregated custom object Fee_Schedule_Summary__c refreshed nightly).

Given a pending rate change exists (Provider_Change_Notification__c.Status__c IN ('Pending Send', 'Sent') for this provider),
When the FlexCard renders,
Then a yellow warning banner must display: 'Rate change pending — effective [PCN.Effective_Date__c]' with a link to the PCN record.

Given the FlexCard is viewed by a user without the 'Fee_Schedule_View' Permission Set,
When the component loads,
Then rate amounts must be masked and a message displayed: 'Contact your administrator to view contracted rates.'

Given the provider has no active PNA,
When the FlexCard loads,
Then the card must display 'No active network agreement on file' with a button to initiate a new PNA via OmniScript (OS-PNA-CREATE-001).

## Solution

Build an OmniStudio FlexCard (FC-ProviderFeeSchedule) backed by a DataRaptor Extract that queries Fee_Schedule__c, PNA, and Fee_Schedule_Summary__c with the Provider__c Account ID as the input key. Conditional visibility states handle the warning banner and masked-rate scenarios using FlexCard state actions. Permission masking implemented via a custom Apex DataRaptor Turbo Extract action that checks the running user's Permission Set assignments before returning rate fields. The 'Initiate PNA' button invokes OmniScript OS-PNA-CREATE-001 in a modal. FlexCard deployed via OmniStudio App Designer and added to the Provider__c Lightning Record Page using a dynamic component visibility rule (Profile: Provider Relations).

## Description

As a Claims Operations Manager, I need the system to automatically transition Fee_Schedule__c records to the new effective rate when a scheduled rate change date is reached, so that Claim_Line__c adjudication always applies the correct CPT-to-reimbursement rate without manual intervention from the Provider Network team.

## Acceptance Criteria

Given a Fee_Schedule__c record with Effective_Date__c = today and Status__c = 'Pending Activation', When the scheduled nightly batch job executes, Then the record Status__c transitions to 'Active' and the previously Active record for the same CPT_Code__c and LOB__c combination is set to Status__c = 'Superseded'.

Given a Claim_Line__c record entering adjudication with Service_Date__c falling within a Fee_Schedule__c effective window, When the FeeScheduleEngine resolves the reimbursement rate, Then Claim_Line__c.Applied_Fee_Schedule__c is stamped with the correct Fee_Schedule__c record ID and Claim_Line__c.Allowed_Amount__c reflects the rate from that schedule.

Given multiple Fee_Schedule__c records exist for the same CPT_Code__c, LOB__c, and Provider_Network_Agreement__c, When rate resolution runs, Then the record with the most recent Effective_Date__c that is less than or equal to Service_Date__c is selected.

Given a Fee_Schedule__c transition fails due to a data conflict, When the batch encounters the error, Then a Platform Event Fee_Schedule_Transition_Error__e is published and a Case is auto-created and assigned to the Provider Network Specialist queue with Priority = 'High'.

-Given manual override is allowed on Claim_Line__c.Allowed_Amount__c without audit logging- (superseded by INS-0941 audit requirement)

Given the batch completes successfully, When all transitions are processed, Then a summary notification is sent to the Claims Operations Manager via Custom Notification with counts of Activated, Superseded, and Failed records.

## Solution

Implement a Scheduled Apex job (FeeScheduleTransitionBatch) running nightly at 00:30 UTC that queries Fee_Schedule__c records WHERE Effective_Date__c = TODAY AND Status__c = 'Pending Activation'. Use Database.Stateful to accumulate transition counts. For each record, perform a SOQL query to locate the currently Active sibling record on the same CPT_Code__c + LOB__c + PNA__c composite key and update it to 'Superseded' in the same DML transaction using partial success (Database.update with allOrNone=false). Publish Fee_Schedule_Transition_Error__e Platform Events for failures. Claim_Line__c rate resolution is handled by an invocable Apex method called from the Adjudication Record-Triggered Flow. Add a custom index on Fee_Schedule__c.CPT_Code__c + Fee_Schedule__c.Effective_Date__c to support performant lookups.

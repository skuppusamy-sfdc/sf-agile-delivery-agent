## Description

As a Claims Operations Manager, I need a scheduled batch process that scans all active Provider_Network_Agreement__c records nightly and identifies any PNAs that lack a valid, non-expired Fee_Schedule__c for each contracted LOB, so that the Provider Network team receives a daily exception report and can address gaps before they cause adjudication pends during the business day.

## Acceptance Criteria

Given the nightly batch job 'FeeScheduleGapAudit_Batch' runs at 11:00 PM local time,
When it processes all PNA records where PNA.Contract_Status__c = 'Active',
Then for each PNA it checks whether at least one Fee_Schedule__c exists with Effective_Start_Date__c <= TODAY() AND (Effective_End_Date__c >= TODAY() OR Effective_End_Date__c = null) AND Fee_Schedule__c.Active__c = true.

Given a gap is detected on a PNA,
When the batch completes processing that record,
Then a Fee_Schedule_Gap__c exception record is created (or updated if one exists from a prior run) with Gap.PNA__c, Gap.Provider__c, Gap.LOB__c, Gap.Gap_Detected_Date__c = TODAY(), and Gap.Status__c = 'Open'.

Given the batch job completes its full run,
When all records have been processed,
Then a summary Platform Event 'BatchJobComplete__e' is published and triggers an automated email to the 'Provider_Network_Ops' public group with: total PNAs scanned, total gaps found, and a link to the Fee Schedule Gaps list view.

Given a Fee_Schedule_Gap__c.Status__c = 'Open' is resolved (a valid schedule is added),
When the next nightly batch runs and no gap is detected for that PNA,
Then the existing Fee_Schedule_Gap__c record is updated to Status__c = 'Resolved' and Gap.Resolved_Date__c = TODAY().

Given the batch processes more than 10,000 PNA records,
When executed,
Then the job must complete within the Salesforce governor limits using chunked batch size of 200 and must not produce any SOQL governor limit exceptions.

-Given results are also written to a custom report snapshot object for BI export- (deferred to Phase 3 per scope review)

## Solution

Apex Batch class 'FeeScheduleGapAudit_Batch' implementing Database.Batchable<SObject> and Database.Stateful. Query scope: all active PNA records. Execute method performs aggregate SOQL to detect missing valid Fee_Schedule__c per PNA/LOB combination using a subquery. Upsert on Fee_Schedule_Gap__c using PNA_LOB_ExternalId__c (formula-based external ID: PNA.Id + '_' + LOB) to prevent duplicate gap records. Platform Event published in finish() method. Scheduler registered via System.schedule() in a one-time anonymous Apex execution. Batch size set to 200 to optimize heap usage.

## Description

As a Provider Network Specialist, I need a scheduled batch process that identifies all Fee_Schedule__c records expiring within the next 90 days and generates automated renewal alerts so that the network contracting team can proactively initiate rate negotiations before schedule gaps occur, preventing claim adjudication failures and potential provider payment delays.

## Acceptance Criteria

Given the scheduled batch job runs daily at 6:00 AM system time,
When Fee_Schedule__c.Termination_Date__c is within TODAY() + 90 days AND Fee_Schedule__c.Status__c = 'Active' AND Fee_Schedule__c.Renewal_Alert_Sent__c = false,
Then a Task must be created for each qualifying record with Subject = 'Fee Schedule Expiring – Renewal Action Required', Due Date = Fee_Schedule__c.Termination_Date__c - 30 days, assigned to the owner of the linked PNA.OwnerId.

Given a Task is created for an expiring Fee_Schedule__c,
When the Task is created,
Then Fee_Schedule__c.Renewal_Alert_Sent__c must be set to true and Fee_Schedule__c.Alert_Sent_Date__c must be stamped with the current date to prevent duplicate alerts.

Given a Fee_Schedule__c record enters the 30-day expiry window (Termination_Date__c <= TODAY() + 30),
When the daily batch runs,
Then an escalation Custom Notification must be sent to the Provider Network Specialist's Salesforce notification bell and email with the message 'URGENT: Fee Schedule [Fee_Schedule__c.Name] expires in [X] days. Immediate action required.'

Given the batch job completes its run,
When all records are processed,
Then a summary report record (Batch_Job_Log__c) must be created with fields: Batch_Job_Log__c.Job_Name__c = 'FeeScheduleExpiryBatch', Batch_Job_Log__c.Records_Processed__c, Batch_Job_Log__c.Alerts_Generated__c, Batch_Job_Log__c.Run_Date__c, Batch_Job_Log__c.Status__c.

-Given a Fee_Schedule__c is linked to a terminated PNA, the batch must still generate renewal alerts- (removed: terminated PNA fee schedules are excluded from renewal processing per compliance guidance).

Given a Fee_Schedule__c has Fee_Schedule__c.Auto_Renew__c = true,
When it enters the 90-day window,
Then instead of a renewal Task, the batch must create a new Fee_Schedule__c record cloned from the expiring record with a new Effective_Date__c = expiring record's Termination_Date__c + 1 day, Status__c = 'Draft', and a Task assigned to the PNA owner to review and activate the cloned record.

## Solution

Implement Apex Batchable class 'FeeScheduleExpiryBatch' implementing Database.Batchable<SObject> and Database.Stateful. Batch size set to 200. SOQL start query filters on Termination_Date__c range and Status__c = 'Active'. Execute method processes expiry logic with separate handling for 90-day alerts, 30-day escalations, and auto-renew cloning. Custom Notification sent via Messaging.CustomNotification API using a custom Notification Type 'Fee_Schedule_Expiry_Alert'. Batch scheduled via a Scheduled Apex class 'FeeScheduleExpiryBatchScheduler' configured to run at 0600 daily. New custom fields on Fee_Schedule__c: Renewal_Alert_Sent__c (Checkbox), Alert_Sent_Date__c (Date), Auto_Renew__c (Checkbox). New custom object Batch_Job_Log__c with fields as specified. Deployment includes a post-install script to backfill Renewal_Alert_Sent__c = false on all existing active Fee_Schedule__c records.

## Description

As a Claims Operations Manager, I need the system to automatically transition Fee_Schedule__c records to a new effective rate when a CPT code rate change becomes active, so that Claim_Line__c adjudication always applies the correct allowed amount without manual intervention.

Background: Currently, rate transitions require a nightly batch job and manual override. This story automates the cutover logic using a scheduled Platform Event and Record-Triggered Flow. Extends INS-0853.

## Acceptance Criteria

Given a Fee_Schedule__c record with Effective_End_Date__c = TODAY and a successor record with Effective_Start_Date__c = TOMORROW for the same CPT_Code__c and LOB__c,
When the scheduled automation runs at 00:01 system time,
Then Fee_Schedule__c.Status__c transitions from 'Active' to 'Expired' on the outgoing record and 'Pending' to 'Active' on the incoming record.

Given a Claim_Line__c with Service_Date__c falling within the new Fee_Schedule__c effective date range,
When adjudication is triggered,
Then Claim_Line__c.Allowed_Amount__c is populated using the new Fee_Schedule__c.Rate__c and Claim_Line__c.Fee_Schedule_Applied__c is stamped with the new Fee_Schedule__c record ID.

Given a Fee_Schedule__c rate change that affects more than 500 active Claim_Line__c records in 'Pending Adjudication' status,
When the transition fires,
Then a Platform Event (Fee_Schedule_Transition__e) is published and a Case is auto-created with Subject = 'Bulk Rate Change Impact Review' assigned to the Claims Operations Manager queue.

-Given manual override flag Fee_Schedule__c.Force_Active__c = TRUE, the scheduled transition should be skipped entirely-
(Superseded: force-active logic moved to INS-0891)

Given a Fee_Schedule__c record with no successor record defined,
When Effective_End_Date__c = TODAY,
Then Fee_Schedule__c.Status__c remains 'Active' and a warning notification is sent to the Provider Network Specialist assigned to the related Provider_Network_Agreement__c.

## Solution

1. Scheduled Apex job (FeeScheduleTransitionBatch) runs daily at 00:01 UTC, queries Fee_Schedule__c records where Effective_End_Date__c = TODAY and Status__c = 'Active'.
2. Batch publishes Fee_Schedule_Transition__e Platform Events per affected record.
3. Record-Triggered Flow on Fee_Schedule__c (after-save) handles Status__c field transitions.
4. A second Record-Triggered Flow on Claim_Line__c (before-save) re-evaluates Fee_Schedule_Applied__c lookup when Status__c = 'Pending Adjudication'.
5. Custom Metadata Type (Fee_Schedule_Config__mdt) stores bulk-impact threshold (default 500).
6. Permission Set Group 'FeeSchedule_Admin' gates manual override capability.

## Description

As a Claims Operations Manager, I need the system to automatically validate that the active Fee_Schedule__c record aligns with the Claim_Line__c service date of service (DOS) during adjudication so that reimbursement rates are always drawn from the correct effective fee schedule and overpayments due to stale rate lookups are eliminated.

## Acceptance Criteria

Given a Claim_Line__c record enters adjudication status,
When the system evaluates the applicable Fee_Schedule__c,
Then Fee_Schedule__c.Effective_Date__c must be <= Claim_Line__c.Date_of_Service__c AND Fee_Schedule__c.Expiration_Date__c must be >= Claim_Line__c.Date_of_Service__c.

Given a matching Fee_Schedule__c record is found for the CPT code and LOB,
When the allowed amount is calculated,
Then Claim_Line__c.Allowed_Amount__c must be populated with Fee_Schedule__c.Allowed_Amount__c and Claim_Line__c.Fee_Schedule_Lookup_Status__c must be set to 'Matched'.

Given no active Fee_Schedule__c record exists for the given CPT code, NPI, and DOS,
When the adjudication engine attempts the lookup,
Then Claim_Line__c.Fee_Schedule_Lookup_Status__c must be set to 'No Match Found' and Claim__c.Status__c must be updated to 'Pended – Fee Schedule Review'.

Given multiple overlapping Fee_Schedule__c records exist for the same CPT code,
When the system selects the applicable rate,
Then the record with the most recent Fee_Schedule__c.Effective_Date__c that still falls on or before the DOS must be selected (most-specific-wins logic).

-Given Fee_Schedule__c.Rate_Type__c = 'Percentage of Billed', the system shall apply billed charges multiplied by the configured percentage.- (Superseded by flat-rate-only requirement for Sprint 11 scope; percentage logic deferred to INS-0931.)

Given the fee schedule lookup completes successfully,
When Claim__c.Status__c transitions to 'Adjudicated',
Then an audit record must be written to Fee_Schedule_Audit_Log__c capturing Fee_Schedule__c.Id, Claim_Line__c.Id, matched CPT code, allowed amount, and timestamp.

## Solution

Implement a Record-Triggered Flow on Claim_Line__c firing on create/update when Status__c changes to 'In Adjudication'. Flow calls an Apex invocable method (FeeScheduleLookupService.cls) that queries Fee_Schedule__c using CPT_Code__c, LOB__c, and Date_of_Service__c with effective/expiration date range filters. Apex returns matched Fee_Schedule__c.Id and Allowed_Amount__c back to Flow for field population. A separate Apex trigger (ClaimLineAfterUpdate) writes to Fee_Schedule_Audit_Log__c via a queueable job to avoid DML conflicts. Custom metadata type (Fee_Schedule_Config__mdt) controls fallback behavior and pend reason codes. Deploy Permission Set Group 'Claims_Adjudication_PSG' to restrict direct edit of Claim_Line__c.Allowed_Amount__c.

## Description

As a Compliance Officer, I need a governed manual override mechanism for Fee_Schedule__c rate application on individual Claim_Line__c records that requires dual approval (Claims Operations Manager + Compliance Officer), so that exceptional rate adjustments are fully audited and compliant with state regulatory requirements. This story implements the override logic deprecated from INS-0939.

## Acceptance Criteria

Given a Claims Operations Manager identifies a Claim_Line__c requiring a manual fee schedule override,
When they set Claim_Line__c.Override_Requested__c = true and populate Claim_Line__c.Override_Reason__c (required picklist: ['Provider Dispute', 'Contractual Exception', 'Regulatory Adjustment', 'Other']),
Then an Approval Process is triggered requiring sequential approval from (1) Claims Operations Manager supervisor and (2) Compliance Officer.

Given the dual approval is completed,
When both approvers have approved,
Then Claim_Line__c.Override_Approved__c = true, Claim_Line__c.Override_Approved_Date__c = NOW(), and an immutable Audit_Log__c record is created capturing both approver names, timestamps, and the original vs. override rate.

Given either approver rejects the override request,
When the rejection is submitted,
Then Claim_Line__c.Override_Requested__c is reset to false, Override_Reason__c is cleared, and the submitting user receives an email notification with the rejection reason.

Given an override-approved Claim_Line__c proceeds to payment,
When Payment__c is generated,
Then Payment__c.Override_Flag__c = true and EOB__c includes a remark code 'MA_OVERRIDE' to flag the adjustment for ERA output.

Given a Compliance Officer runs the monthly override audit report,
When they access the 'Fee Schedule Override Audit' report folder,
Then the report lists all Claim_Line__c records where Override_Approved__c = true in the period, including policy number, provider NPI, original rate, override rate, and both approver names.

## Solution

Add custom fields Override_Requested__c (Checkbox), Override_Reason__c (Picklist), Override_Approved__c (Checkbox), Override_Approved_Date__c (DateTime), and Override_Rate__c (Currency) to Claim_Line__c. Configure a Sequential Approval Process on Claim_Line__c triggered when Override_Requested__c = true. Use Approval Process entry criteria and step assignments referencing Role hierarchy. Create Audit_Log__c as a child object with a Master-Detail to Claim_Line__c; populate via Record-Triggered Flow on approval completion. Add Override_Flag__c to Payment__c and EOB__c objects. Build the audit report in Salesforce Reports using a custom report type joining Claim_Line__c and Audit_Log__c. Restrict override field editing via Field-Level Security tied to 'Claims Override Requestor' Permission Set.

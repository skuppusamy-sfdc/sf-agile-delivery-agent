## Description

As a Compliance Officer, I need a Salesforce report and dashboard component that displays a complete history of all Fee_Schedule__c rate changes per Provider_Network_Agreement__c over a configurable date range, so that I can fulfill regulatory audit requests and internal compliance reviews demonstrating that all rate change notifications were delivered within the contractually required 30-day advance notice window.

The report must draw from Fee_Schedule_History__c (archived rates from INS-0891) and Provider_Change_Notification__c (PCN delivery records from INS-0892).

## Acceptance Criteria

Given a Compliance Officer accesses the 'PNA Rate Change Compliance' report,
When they apply a date range filter on Fee_Schedule_History__c.Archived_Date__c,
Then the report must return rows grouped by Provider_Network_Agreement__c showing: PNA Name, Provider NPI, CPT Code, Old Rate, New Rate, Rate Effective Date, PCN Delivered Date, Days Notice Given, and a calculated 'Compliant (Y/N)' column where compliant = Days Notice Given >= 30.

Given the report contains rows where Days Notice Given < 30,
When the report renders,
Then those rows must be highlighted in red via a conditional formatting rule and the count of non-compliant notifications must appear in a summary metric at the top of the report.

Given the Compliance Officer needs to export the report for an external audit,
When they click 'Export',
Then the report must be exportable as both CSV and PDF with Acme Insurance branding via a Conga Composer template (RateChangeComplianceReport_v1).

Given the dashboard is configured with a 'Rate Change Compliance Score' gauge chart,
When the current period has 100% compliant notifications,
Then the gauge must display green (90-100%), amber for 75-89%, and red for below 75%, refreshing on a 24-hour schedule.

Given a Compliance Officer filters the report by LOB (e.g., 'Medicare Advantage'),
When the filter is applied,
Then only PNA records with PNA__c.LOB__c matching the selected value must be returned, and the CMS_Reporting_Required__c flag must be surfaced as an additional column for Medicare Advantage rows.

## Solution

Create a custom Report Type joining Provider_Network_Agreement__c (primary), Fee_Schedule_History__c (child via PNA__c lookup), and Provider_Change_Notification__c (child via PNA__c lookup). Add a custom formula field on Fee_Schedule_History__c: Days_Notice_Given__c = Fee_Schedule__c.Rate_Effective_Date__c - PCN.Delivered_Date__c (cross-object formula approximated via a Roll-Up Summary or Apex-calculated field populated on PCN delivery). Compliant_Flag__c (formula: IF(Days_Notice_Given__c >= 30, 'Yes', 'No')) added to Fee_Schedule_History__c. Build the report in Salesforce Reports with row-level conditional formatting on Days_Notice_Given__c < 30. Dashboard built in Lightning Dashboard with a Gauge chart component using report summary metrics. Conga Composer integration configured via a Conga Button on the Report page using the pre-built RateChangeComplianceReport_v1 template stored in Salesforce Files. Report and dashboard access gated by 'Compliance_Officer' Permission Set Group.

## Description

As a Provider Network Specialist, I need a version lock mechanism on Fee_Schedule__c records that are currently under amendment review, so that concurrent edits by multiple specialists do not overwrite each other's changes and corrupt contracted rate data.

This story introduces a Fee_Schedule__c.Lock_Status__c field and associated Permission-based controls. Note: This story supersedes the manual lock flag approach deprecated in INS-0947.

## Acceptance Criteria

Given a Fee_Schedule__c record where Amendment_Review_Status__c = 'In Review'
When any user without the Permission Set 'Fee_Schedule_Unlock' attempts to edit Fee_Schedule__c.Allowed_Amount__c, Effective_Date__c, or Termination_Date__c
Then a Validation Rule must block the save and display the error: 'This fee schedule is locked for amendment review. Contact your Provider Network Specialist to request an unlock.'

Given a Fee_Schedule__c record with Lock_Status__c = 'Locked'
When a user with Permission Set 'Fee_Schedule_Unlock' sets Lock_Status__c = 'Unlocked'
Then a Fee_Schedule_Audit__c child record must be created capturing: Unlocked_By__c, Unlock_Timestamp__c, Unlock_Reason__c (required text field), and Previous_Status__c = 'Locked'

Given a Fee_Schedule__c record where Amendment_Review_Status__c transitions from 'In Review' to 'Approved' or 'Rejected'
When the record is saved
Then Lock_Status__c must automatically be set to 'Unlocked' via a Record-Triggered Flow

Given the Fee Schedule list view used by Provider Network Specialists
When Lock_Status__c = 'Locked'
Then a lock icon indicator must appear in the list view column and the inline edit pencil icon must be suppressed for restricted fields

## Solution

1. New picklist field Fee_Schedule__c.Lock_Status__c with values: 'Locked', 'Unlocked' (default).
2. New picklist field Fee_Schedule__c.Amendment_Review_Status__c with values: 'Not Started', 'In Review', 'Approved', 'Rejected'.
3. Validation Rule 'Enforce_Fee_Schedule_Lock' on Fee_Schedule__c: AND(Lock_Status__c = 'Locked', NOT($Permission.Fee_Schedule_Unlock), OR(ISCHANGED(Allowed_Amount__c), ISCHANGED(Effective_Date__c), ISCHANGED(Termination_Date__c))).
4. Record-Triggered Flow 'FeeSchedule_AutoUnlock' on Amendment_Review_Status__c change to 'Approved' or 'Rejected' updates Lock_Status__c = 'Unlocked'.
5. Apex Trigger 'FeeScheduleAuditTrigger' on Fee_Schedule__c after update captures unlock events and inserts Fee_Schedule_Audit__c records.
6. Permission Set 'Fee_Schedule_Unlock' created with custom permission; assigned to Claims Operations Manager and Provider Network Specialist profiles.
7. List View updated with Lock_Status__c column; conditional formatting applied via List View settings.

## Description

As an Underwriting Analyst, I need the claims adjudication engine to honor Group_Service_Agreement__c (GSA)-level Fee Schedule overrides when a Claim__c is associated with a group policy, so that employer groups negotiating custom reimbursement rates outside the standard PNA schedule are adjudicated at their contracted amounts rather than the default network rates. This story extends the fee schedule framework introduced in INS-0923.

## Acceptance Criteria

Given a Claim__c.Policy__c is linked to a Policy__c that has an associated Group_Service_Agreement__c,
When adjudication begins,
Then the engine must first check for a GSA-level Fee_Schedule__c override before falling back to the PNA-level schedule (GSA takes precedence).

Given a GSA_Fee_Schedule__c override exists with GSA_Fee_Schedule__c.Active__c = true,
When the CPT code on Claim_Line__c matches a GSA_Fee_Schedule_Line__c.CPT_Code__c,
Then Claim_Line__c.Allowed_Amount__c is set from GSA_Fee_Schedule_Line__c.Allowed_Amount__c and Claim_Line__c.Rate_Source__c = 'GSA Override'.

Given a GSA override does not cover the submitted CPT code,
When the engine falls back to PNA-level schedule,
Then Claim_Line__c.Rate_Source__c = 'PNA Standard' and the standard fee schedule logic from INS-0923 applies.

Given a GSA_Fee_Schedule__c.Effective_End_Date__c has passed,
When the engine evaluates the override,
Then the expired GSA schedule is ignored and PNA-level rates are applied; a warning is logged to Claim__c.Adjudication_Notes__c.

Given Claim_Line__c.Rate_Source__c is populated after adjudication,
When the EOB is generated,
Then Explanation_of_Benefits__c.Rate_Source_Label__c reflects either 'Group Agreement Rate' or 'Network Standard Rate' in the member-facing EOB document.

Given a user with 'Underwriting_Analyst' Permission Set Group views the GSA record,
When reviewing Group_Service_Agreement__c detail,
Then a related list of all linked GSA Fee Schedule overrides is visible with columns: CPT Code, Allowed Amount, Effective Dates, Status.

## Solution

Extend FeeScheduleValidationService with a new method 'resolveGSAOverride()' that accepts Claim__c.Policy__c -> Policy__c.GSA__c lookup chain. GSA_Fee_Schedule__c is a new custom object (child of Group_Service_Agreement__c) with GSA_Fee_Schedule_Line__c as line items. Adjudication resolution order encoded as a Custom Metadata Type 'Fee_Schedule_Priority__mdt' allowing admin-configurable hierarchy without code changes. Claim_Line__c.Rate_Source__c picklist field added. EOB generation Apex class updated to map Rate_Source__c to display label. Integration test scenarios added for group vs. individual policy adjudication paths.

## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to analyze Claim_Line__c billing patterns against active Fee_Schedule__c rates and automatically flag anomalies for SIU referral, so that potential fee schedule fraud (e.g., unbundling, upcoding) is detected earlier and investigated without requiring manual claim review queues.

The AI model should leverage historical adjudication data and CPT code grouping rules to identify statistical outliers. This story covers the Agentforce Action configuration, SIU case creation logic, and the Claims Adjudicator-facing explanation panel.

## Acceptance Criteria

Given a Claim__c with 5 or more Claim_Line__c records where Billed_Amount__c exceeds Fee_Schedule__c.Rate__c by more than 40% on 3 or more lines,
When Agentforce anomaly detection runs during adjudication,
Then a SIU_Referral__c record is created with Referral_Reason__c = 'Fee Schedule Anomaly', linked Claim__c, and SIU_Status__c = 'Pending Review'.

Given an Agentforce recommendation is generated,
When displayed in the Claims Adjudicator console,
Then a natural language explanation is shown in the 'AI Insights' panel with the specific CPT codes flagged, the variance percentage, and a confidence score between 0.00 and 1.00.

Given a Claims Adjudicator reviews the Agentforce recommendation,
When they select 'Accept Referral' or 'Dismiss',
Then Claim__c.AI_Recommendation_Disposition__c is stamped with the action and timestamp, and Agentforce feedback is logged for model retraining.

Given a provider with more than 3 SIU referrals in a rolling 90-day window,
When a new SIU_Referral__c is created,
Then Provider_Network_Agreement__c.PNA_Review_Flag__c is set to TRUE and a Task is assigned to the Provider Relations Specialist.

Given the Agentforce model is invoked,
When the Fee_Schedule__c lookup returns no matching rate for a CPT_Code__c,
Then the anomaly check is skipped for that line and Claim_Line__c.AI_Skip_Reason__c = 'No Fee Schedule Match'.

## Solution

1. Agentforce Agent Action 'DetectFeeScheduleAnomalies' configured with an Apex invocable method (FeeScheduleAnomalyDetector) that scores each Claim_Line__c against Fee_Schedule__c rates.
2. Einstein Analytics (CRM Analytics) dataset pre-aggregates historical billing patterns per NPI and CPT code for baseline comparison.
3. Record-Triggered Flow on Claim__c (after-save, when Status__c = 'Ready for Adjudication') invokes the Agentforce Action.
4. LWC 'AgentforceClaimsInsightsPanel' embedded in the Claims Adjudicator console displays AI explanation and action buttons.
5. Apex trigger on SIU_Referral__c handles the rolling 90-day provider pattern check using aggregate SOQL.
6. Custom Permission 'Agentforce_Claims_AI_User' gates access to the AI panel via Permission Set Group.

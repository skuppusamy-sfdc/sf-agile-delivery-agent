## Description

As a Provider Relations Specialist, I need a FlexCard on the Provider__c (Person Account) record page that displays all active Fee_Schedule__c records linked through the provider's Provider_Network_Agreement__c records, along with visual indicators for schedules expiring within 90 days, so that I can proactively manage rate renewals without navigating multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Fee Schedule Summary FlexCard loads,
Then it displays a list of all Fee_Schedule__c records where Status__c = 'Approved' and Expiration_Date__c >= TODAY(), grouped by PNA and LOB.

Given a Fee_Schedule__c.Expiration_Date__c is within 90 days of today,
When the FlexCard renders that row,
Then the row is highlighted in amber with an expiration countdown label (e.g., 'Expires in 45 days').

Given a Fee_Schedule__c.Expiration_Date__c is within 30 days of today,
When the FlexCard renders that row,
Then the row is highlighted in red and a 'Renew Now' action button is displayed that launches the Fee Schedule renewal OmniScript.

Given the provider has no active Fee_Schedule__c records,
When the FlexCard loads,
Then an empty state message reads: 'No active fee schedules found. Please review Provider Network Agreement status.'

Given the FlexCard is loaded,
When a user clicks a Fee Schedule row,
Then the record detail flyout shows Fee_Schedule__c.Rate__c, CPT code range, Effective_Date__c, Expiration_Date__c, and the linked PNA name.

## Solution

Build an OmniStudio FlexCard (Card Name: 'ProviderFeeScheduleSummary') using a DataRaptor Extract as the data source, querying Fee_Schedule__c via Provider_Network_Agreement__c.Provider__c relationship. Use conditional styling states in the FlexCard designer for amber (<=90 days) and red (<=30 days) thresholds driven by a formula field Fee_Schedule__c.Days_Until_Expiration__c (formula: Expiration_Date__c - TODAY()). The 'Renew Now' action invokes the existing Fee Schedule OmniScript via a Navigate action. Add the FlexCard to the Provider__c Lightning Record Page via App Builder, visible only to the 'Provider Relations' Permission Set Group.

## Description

As a Broker Relations Coordinator, I need the system to automatically cascade a rate change notification to all employer groups covered under a Group_Service_Agreement__c when the associated Fee_Schedule__c is updated or replaced so that brokers and employer group administrators receive timely, accurate communication about upcoming rate impacts and can update their benefit plan documentation accordingly.

## Acceptance Criteria

Given a Fee_Schedule__c record linked to a Group_Service_Agreement__c has its Status__c changed to 'Superseded' or Expiration_Date__c is updated,
When the Record-Triggered Flow on Fee_Schedule__c fires (after update),
Then a Platform Event (GSA_Rate_Change__e) is published with GSA__c.Id, Old_Expiration_Date__c, New_Expiration_Date__c, and Changed_By__c.

Given the GSA_Rate_Change__e Platform Event is received,
When the subscribing Flow processes the event,
Then a Provider_Change_Notification__c is created for each Account (employer group) in Group_Service_Agreement__c.Member_Groups__r with PCN_Type__c = 'GSA Rate Change', Related_GSA__c = GSA__c.Id, and Effective_Change_Date__c = new Expiration_Date__c.

Given the PCN records are created for employer groups,
When PCN Status__c = 'Open',
Then an automated email is sent to each employer group's primary contact (Account.Primary_Benefits_Contact__c) using the 'GSA_Rate_Change_Employer_Notification' Email Template within 1 business hour.

Given the GSA has more than 50 member employer groups,
When the cascade is triggered,
Then PCN creation is processed via a Queueable Apex chain to avoid governor limit violations on DML rows.

Given a member employer group has opted out of automated notifications (Account.Notification_Opt_Out__c = TRUE),
When the PCN cascade runs,
Then no PCN or email is generated for that account, and a suppression log entry is written to Notification_Suppression_Log__c.

## Solution

Implement a Record-Triggered Flow on Fee_Schedule__c publishing a GSA_Rate_Change__e Platform Event via an Apex action. Build a Platform Event-triggered Flow to handle the cascade logic, delegating bulk PCN creation to a Queueable Apex class (GSARateChangePCNCascade) that processes member groups in chunks of 50 using iterator pattern. Use Messaging.renderStoredEmailTemplate for email generation. Create Notification_Suppression_Log__c as a custom object with a Master-Detail to Account. Add a custom report 'GSA Rate Change Notification Log' to the Broker Relations dashboard.

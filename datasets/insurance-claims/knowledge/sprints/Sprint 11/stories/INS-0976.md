## Description

**Bug Report**

**Summary:** The Record-Triggered Flow introduced in INS-0973 calculates Provider_Change_Notification__c.Notification_Due_Date__c as TODAY + 30 universally, but state regulations in CA, NY, and TX require a 60-day advance notification for material rate changes. Providers with Provider__c.BillingState = 'CA', 'NY', or 'TX' are receiving PCN records with a 30-day due date, creating a compliance risk.

**Steps to Reproduce:**
1. Ensure a Provider__c (Person Account) record exists with BillingState = 'CA'.
2. Trigger a Fee_Schedule__c Status__c change to 'Active' with Variance_Percent__c > 15% on a PNA linked to the CA provider.
3. Observe the created Provider_Change_Notification__c record.
4. Check Notification_Due_Date__c — it shows TODAY + 30.

**Expected Behavior:** For providers with BillingState IN ('CA', 'NY', 'TX'), Notification_Due_Date__c should be TODAY + 60. For all other states, TODAY + 30 is correct.

**Actual Behavior:** Notification_Due_Date__c = TODAY + 30 for all providers regardless of BillingState.

**Compliance Impact:** HIGH — CA Department of Managed Health Care requires 60-day notice. Failure to comply may result in regulatory penalties.

**Environment:** Acme Insurance Full Sandbox (FS1) and Production (verified by Compliance Officer)
**Regression:** Present since INS-0973 initial deployment.

## Solution

Update the Record-Triggered Flow 'Fee_Schedule_PCN_Creation_Flow' (built in INS-0973) to add a Decision element after the Get Records step that resolves Provider__c.BillingState from the PNA. Branch on BillingState IN ['CA', 'NY', 'TX'] to set a flow variable varNoticeDays = 60, otherwise varNoticeDays = 30. Update the Create Records element for Provider_Change_Notification__c to use {!varNoticeDays} in the Notification_Due_Date__c formula: TODAY + varNoticeDays. Consider refactoring the notice day logic into a custom metadata type 'State_Notice_Requirement__mdt' with State__c and Required_Notice_Days__c fields to support future state additions without flow redeployment. Add unit test coverage in the associated Flow test suite for CA, NY, TX, and a default state (e.g., OH). Coordinate with Compliance Officer to confirm the full list of states requiring extended notice periods before deploying to Production.

## Description

As a Provider Relations Specialist, I need an automated workflow that delivers rate change notifications to affected providers via their preferred contact channel when a Fee Schedule rate change is finalized, so that providers receive timely, accurate PCN communications compliant with our 30-day advance notice contractual obligation.

This story consumes the Platform Event published by INS-0891 and extends the PCN delivery framework established in the Provider Network Contracting epic.

## Acceptance Criteria

Given a Provider_Change_Notification__c record with PCN.Status__c = 'Pending Delivery' and PCN.Notification_Type__c = 'Rate Change',
When the PCN delivery workflow is triggered,
Then the system must evaluate Provider__c.Preferred_Contact_Method__c and route delivery accordingly: 'Email' triggers an automated email via OrgWideEmailAddress, 'Portal' creates a Provider_Portal_Message__c record, and 'Mail' sets PCN.Delivery_Method__c = 'Physical Mail' and PCN.Status__c = 'Awaiting Print Queue'.

Given a PCN is routed for email delivery,
When the email is sent successfully,
Then PCN.Status__c must update to 'Delivered', PCN.Delivered_Date__c must be stamped with the current datetime, and PCN.Delivery_Attempt_Count__c must increment by 1.

Given a PCN email delivery fails (bounce or invalid address),
When the failure is detected,
Then PCN.Status__c must be set to 'Delivery Failed', a Task must be created and assigned to the Provider Relations Specialist owning the Provider__c record, and PCN.Delivery_Attempt_Count__c must not exceed 3 before escalating to PCN.Status__c = 'Manual Intervention Required'.

Given a PCN record has PCN.Effective_Date__c less than 30 days from today,
When the PCN is created,
Then a Compliance_Alert__c record must be created with Compliance_Alert__c.Alert_Type__c = 'Late Rate Notice' and linked to the PCN via lookup, and the owning Provider Relations Specialist must receive an in-app Salesforce notification.

Given a Provider_Network_Agreement__c (PNA) has PNA.LOB__c = 'Medicare Advantage',
When a rate change PCN is generated for that PNA,
Then PCN.Regulatory_Flag__c must be set to TRUE and PCN.CMS_Reporting_Required__c must be set to TRUE to trigger downstream CMS integration.

## Solution

Build a Record-Triggered Flow on Provider_Change_Notification__c (after insert, when Status__c = 'Pending Delivery') that evaluates Provider__c.Preferred_Contact_Method__c via a Decision element. For email path: invoke an Apex Action (PCNEmailDispatcher.cls) using a custom EmailTemplate keyed by PCN.Notification_Type__c. For portal path: create Provider_Portal_Message__c via Flow Create Records element. For mail path: update PCN fields only. Email bounce handling via an EmailMessage trigger that checks RelatedToId against PCN records. Compliance check implemented as a separate Before-Save Record-Triggered Flow on PCN insert that calculates days between CreatedDate and Effective_Date__c. CMS flag logic embedded in same before-save flow using a Get Records element to inspect PNA.LOB__c. All delivery attempts logged in PCN_Delivery_Log__c child object.

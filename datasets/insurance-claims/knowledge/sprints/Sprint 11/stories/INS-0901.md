## Description

As a Provider Relations Specialist, I need a FlexCard on the Provider__c (Person Account) record page that displays a consolidated summary of the provider's active Fee_Schedule__c records, expiration timelines, and any open Provider_Change_Notification__c items so that I can assess contractual rate status without navigating multiple related lists.

## Acceptance Criteria

Given I am viewing a Provider__c record page with at least one active Provider_Network_Agreement__c,
When the 'Fee Schedule Summary' FlexCard loads,
Then it displays a list of all Fee_Schedule__c records where Status__c = 'Active' and the related PNA's Provider__c = current record Id, showing Fee_Schedule__c.Name, Effective_Date__c, Expiration_Date__c, LOB__c, and Fee_Schedule_Type__c.

Given a Fee_Schedule__c Expiration_Date__c is within 60 days,
When the FlexCard renders that row,
Then the expiration date cell is highlighted in amber and a warning icon is displayed.

Given a Fee_Schedule__c Expiration_Date__c is within 15 days,
When the FlexCard renders that row,
Then the expiration date cell is highlighted in red and PCN_Count__c (count of open PCNs) is displayed inline.

Given there are no active Fee_Schedule__c records for the provider,
When the FlexCard loads,
Then an empty-state message 'No Active Fee Schedules Found' is displayed with a quick action button to create a new PNA.

Given I click the 'View PCNs' action on a Fee Schedule row,
When the action executes,
Then an OmniScript modal opens pre-filtered to Provider_Change_Notification__c records related to that Fee_Schedule__c's PNA.

## Solution

Build an OmniStudio FlexCard using a DataRaptor Turbo Extract as the data source, joining Provider_Network_Agreement__c, Fee_Schedule__c, and Provider_Change_Notification__c via a multi-object DataRaptor. Implement conditional styling in the FlexCard JSON definition using state expressions on Expiration_Date__c delta calculations. The 'View PCNs' action launches an OmniScript (PCN_Review_OS) with input parameters passed from the FlexCard card state. Deploy the FlexCard to the Provider__c Lightning Record Page via App Builder, restricted to the 'Provider Relations' Permission Set Group.

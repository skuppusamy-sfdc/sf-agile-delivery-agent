## Description

As a Provider Network Specialist, I need the system to proactively alert me and create renewal Tasks when a Fee_Schedule__c record is approaching its Expiration_Date__c, so that I can initiate contract renegotiation in time to prevent rate gaps that cause claim pend queues and provider payment disputes. Extends INS-0954 fee schedule enforcement framework.

## Acceptance Criteria

Given a Fee_Schedule__c record has Status__c = 'Published' and Expiration_Date__c is within 90 days of today,
When the nightly scheduled automation runs,
Then a Task is created with Subject = 'Fee Schedule Renewal Due: [Fee_Schedule__c.Name]', ActivityDate = Fee_Schedule__c.Expiration_Date__c - 30, and assigned to the Provider Network Specialist listed in Fee_Schedule__c.Owner.

Given the same Fee_Schedule__c record reaches 30 days before Expiration_Date__c with no successor record (a new Fee_Schedule__c with Effective_Date__c = Expiration_Date__c + 1),
When the nightly job evaluates the record,
Then Fee_Schedule__c.Renewal_Risk__c picklist is set to 'High' and an in-app notification is sent to the record owner via Salesforce Bell Notification.

Given Fee_Schedule__c.Renewal_Risk__c = 'High',
When the record is viewed by a Claims Operations Manager,
Then a warning banner is displayed on the Fee_Schedule__c record page via a dynamic Lightning Web Component reading the Renewal_Risk__c field value.

Given a successor Fee_Schedule__c record is published before the expiration date,
When the nightly job next runs,
Then any open renewal Tasks linked to the expiring record are automatically closed with Status = 'Completed' and Comments = 'Successor rate schedule published — auto-resolved.'

## Solution

Build a Scheduled Flow (daily at 06:00 UTC) that queries Fee_Schedule__c records approaching expiration using a relative date filter. The Flow calls an Apex invocable 'FeeScheduleRenewalAlertService.evaluate()' which handles Task creation, successor record lookup, and Renewal_Risk__c field updates in a single transaction. In-app Bell Notifications use the Salesforce Notification Builder with a custom notification type 'Fee_Schedule_Expiry_Alert'. The warning banner LWC ('feeScheduleRenewalBanner') uses @wire(getRecord) to reactively read Renewal_Risk__c and conditionally renders using an if:true/if:false directive. A custom report type 'Fee Schedules by Expiry Risk' is added to the Provider Network dashboard for operational visibility.

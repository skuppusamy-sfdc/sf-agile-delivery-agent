# INS-0912: Fee Schedule override logic for out-of-network providers on Claim_Line__c adjudication

## Description

As a Claims Operations Manager, I need the adjudication engine to apply a configurable out-of-network (OON) reimbursement override rate to Claim_Line__c records where the rendering provider does not have an active Provider_Network_Agreement__c so that OON claims are reimbursed at the correct policy-defined percentage of the Medicare fee schedule rather than being incorrectly pended or denied. This story implements the override logic that was deprecated from INS-0907.

## Acceptance Criteria

Given a Claim_Line__c is being adjudicated and the rendering provider (Claim__c.Rendering_Provider__c) has no active PNA with Status__c = 'Active',
When the FeeScheduleResolver evaluates the line,
Then Claim_Line__c.Network_Status__c must be set to 'Out-of-Network'.

Given Claim_Line__c.Network_Status__c = 'Out-of-Network',
When the override rate is calculated,
Then the system must retrieve the applicable CMS Medicare fee schedule rate for the CPT code from the CMS_Rate__c object and apply the OON reimbursement percentage defined on Policy__c.OON_Reimbursement_Pct__c.

Given Policy__c.OON_Reimbursement_Pct__c is null or zero,
When adjudication attempts to apply the OON override,
Then Claim_Line__c.Adjudication_Status__c must be set to 'Pended - OON Config Missing' and a high-priority Task assigned to the Claims Operations Manager queue.

Given the OON override rate is successfully applied,
When Claim_Line__c.Allowed_Amount__c is calculated,
Then Claim_Line__c.Override_Applied__c must be set to true and Claim_Line__c.Override_Reason__c = 'OON Provider - CMS Rate Applied'.

Given Claim__c.LOB__c = 'Workers Comp',
When OON adjudication logic evaluates the claim,
Then the system must bypass OON override and instead set Claim_Line__c.Adjudication_Status__c = 'Pended - Manual Review Required' as Workers Comp OON rules differ by jurisdiction.

## Solution

Extend the existing Apex 'FeeScheduleResolver' invocable method (INS-0907) with an OON evaluation branch. When no active PNA is found, the resolver queries CMS_Rate__c using CPT_Code__c and the current CMS rate year (derived from Date_of_Service__c). Allowed_Amount__c is computed as CMS_Rate__c.Medicare_Rate__c * (Policy__c.OON_Reimbursement_Pct__c / 100). Workers Comp LOB bypass is implemented as a custom metadata type 'OON_LOB_Exclusion__mdt' to allow configuration without code changes. Override_Applied__c and Override_Reason__c are updated via the same Record-Triggered Flow post-processing step. Add Compliance_Flag__c = true on the Claim__c parent when any child Claim_Line__c has Override_Applied__c = true to route to ComplianceReporting dashboard.

## Description

**Bug Report**

**Summary:** When a Provider__c has two or more active Provider_Network_Agreement__c records sharing the same TIN (Provider__c.Tax_Identification_Number__c) but belonging to different LOBs, the Fee Schedule lookup during Claim_Line__c adjudication intermittently applies the wrong LOB's rate, resulting in incorrect Claim_Line__c.Allowed_Amount__c values.

**Steps to Reproduce:**
1. Create a Provider__c record with Tax_Identification_Number__c = '12-3456789'.
2. Create PNA-A with PNA.LOB__c = 'Commercial' and link Fee_Schedule__c-A with Allowed_Amount = $150.00 for CPT 99213.
3. Create PNA-B with PNA.LOB__c = 'Medicare Advantage' and link Fee_Schedule__c-B with Allowed_Amount = $98.50 for CPT 99213.
4. Submit a Claim__c with Claim__c.LOB__c = 'Medicare Advantage' and a Claim_Line__c for CPT 99213.
5. Trigger adjudication.

**Expected Result:** Claim_Line__c.Allowed_Amount__c = $98.50 (Medicare Advantage rate applied).

**Actual Result:** Claim_Line__c.Allowed_Amount__c = $150.00 (Commercial rate incorrectly applied ~60% of the time).

**Environment:** Acme Insurance - Full Sandbox (FS1)
**Frequency:** Intermittent (~60% occurrence rate in regression testing)
**First Observed:** Sprint 11 regression cycle, 2024-11-04
**Related Story:** INS-0923

## Acceptance Criteria

As per Description. Fee schedule lookup must filter by both Provider__c.Tax_Identification_Number__c AND Claim__c.LOB__c matching PNA.LOB__c to ensure the correct rate is returned in all cases.

## Solution

Root cause identified in FeeScheduleValidationService.validateAndApply() — the SOQL query filters on TIN only without including PNA.LOB__c in the WHERE clause, causing non-deterministic record selection when multiple PNAs share a TIN. Fix: add AND PNA.LOB__c = :claim.LOB__c filter to the query. Additionally, add a LIMIT 1 with ORDER BY Fee_Schedule__c.Effective_Start_Date__c DESC as a safety net. Deploy updated Apex class with revised test methods covering multi-PNA/same-TIN scenarios. Regression test suite to be expanded in QA.

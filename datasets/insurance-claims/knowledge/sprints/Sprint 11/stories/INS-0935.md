## Description

As a Claims Operations Manager, I need the adjudication engine to check for a Group_Service_Agreement__c (GSA) rate override before applying the standard Fee_Schedule__c rate on a Claim_Line__c so that employer group members with negotiated custom rates receive accurate payment calculations that honor contractual GSA terms rather than defaulting to the standard network fee schedule.

## Acceptance Criteria

Given a Claim_Line__c is being adjudicated and the parent Claim__c.Policy__c has a linked Group_Service_Agreement__c,
When the fee schedule resolution engine runs,
Then it must first query GSA.GSA_Rate_Override__c records before querying the standard Fee_Schedule__c.

Given a GSA_Rate_Override__c record exists where GSA_Rate_Override__c.CPT_Code__c = Claim_Line__c.CPT_Code__c AND GSA_Rate_Override__c.Effective_Date__c <= Claim_Line__c.Service_Date__c AND GSA_Rate_Override__c.Expiration_Date__c >= Claim_Line__c.Service_Date__c,
When the override is found,
Then Claim_Line__c.Allowed_Amount__c must be set to GSA_Rate_Override__c.Override_Rate__c, Claim_Line__c.Rate_Source__c must be set to 'GSA Override', and Claim_Line__c.GSA_Reference__c must be populated with GSA.Id.

Given a GSA rate override is applied,
When the EOB is generated,
Then Explanation_of_Benefits__c.Rate_Basis__c must display 'Group Agreement Rate' and Explanation_of_Benefits__c.GSA_Number__c must reflect the GSA.Agreement_Number__c.

Given no active GSA_Rate_Override__c is found for the CPT code,
When the resolution engine falls through,
Then the standard Fee_Schedule__c lookup must proceed as defined in INS-0931.

Given a GSA has GSA.Status__c = 'Expired' or 'Terminated',
When the adjudication engine queries for overrides,
Then GSA rate overrides from that agreement must be excluded regardless of GSA_Rate_Override__c date fields.

Given a Claim_Line__c.Rate_Source__c = 'GSA Override' after adjudication,
When a Claims Operations Manager reviews the claim,
Then a read-only 'GSA Rate Details' section must be visible on the Claim_Line__c Lightning record page showing GSA.Agreement_Number__c, GSA.Group_Name__c, and GSA_Rate_Override__c.Override_Rate__c.

Given bulk adjudication processes more than 200 Claim_Line__c records in a single transaction,
When GSA override lookup is performed,
Then SOQL queries must be bulkified and must not exceed 100 SOQL queries per transaction (enforced via Apex test assertion on Limits.getQueries()).

## Solution

Extend the 'FeeScheduleResolutionService' Apex class (INS-0931) with a new method 'resolveGSARateOverride(List<Claim_Line__c> claimLines)' that executes prior to standard fee schedule lookup. Method performs a single bulkified SOQL query joining Policy__c → Group_Service_Agreement__c → GSA_Rate_Override__c using a Map<Id, GSA_Rate_Override__c> keyed by CPT code. Resolution priority chain: (1) Active GSA Override → (2) Standard Fee Schedule → (3) Pend for manual review. New custom fields added: Claim_Line__c.GSA_Reference__c (Lookup to GSA), Claim_Line__c.Rate_Source__c extended picklist value 'GSA Override', EOB__c.Rate_Basis__c picklist, EOB__c.GSA_Number__c text. Lightning Record Page updated with a dynamic 'GSA Rate Details' component (LWC) that shows conditionally based on Claim_Line__c.Rate_Source__c = 'GSA Override'. Permission Set Group 'Claims_Adjudication_Full' updated to include new field-level security.

## Description

As a Provider Network Specialist, I need a guided OmniScript wizard that accepts a CSV upload of annual CPT rate tables and bulk-creates Fee_Schedule__c records with Status__c = 'Pending Activation', so that the team can onboard updated CMS fee schedules and GSA-negotiated rates each plan year without relying on manual data entry or IT-run data loader jobs.

## Acceptance Criteria

Given the Provider Network Specialist navigates to the Fee Schedule Import OmniScript, When they upload a CSV file, Then the system validates required columns: CPT_Code__c, LOB__c, New_Rate__c, Effective_Date__c, PNA_ID__c and surfaces a row-level error report for any missing or malformed values before processing.

Given the CSV passes header validation, When the DataRaptor Load processes rows, Then each valid row creates a Fee_Schedule__c record with Status__c = 'Pending Activation', Created_By_Import__c = true, and Import_Batch_ID__c stamped with a UUID generated at the start of the OmniScript session.

Given a CPT_Code__c in the import file does not exist in the CPT_Code__mdt custom metadata table, When validation runs, Then that row is flagged as 'Invalid CPT Code' and excluded from the load, and the error is included in the post-import summary.

Given the import completes, When all rows are processed, Then a summary screen displays: Total Rows Submitted, Records Created, Rows Skipped (with reasons), and a Download Error Log button that generates a CSV of failed rows.

Given a duplicate Fee_Schedule__c record would be created (same CPT_Code__c + LOB__c + PNA__c + Effective_Date__c), When the DataRaptor Load encounters the duplicate, Then it updates the existing Pending Activation record's New_Rate__c rather than creating a new record, and logs the upsert action in the summary.

-Given the import supports XLS format in addition to CSV- (descoped to Phase 3)

## Solution

Build OmniScript 'Fee_Schedule_Bulk_Import_OS' with a File Upload step leveraging OmniStudio's native file parsing capability. Use a DataRaptor Transform to map and validate CSV columns against Fee_Schedule__c field schema and CPT_Code__mdt metadata. Implement a DataRaptor Load with upsert logic using an External ID field Import_Key__c (formula: CPT_Code__c + LOB__c + PNA_ID__c + Effective_Date__c). Use Integration Procedures to orchestrate validation, load, and summary generation steps. Store the Import_Batch_ID__c as a UUID generated via Apex REST action invoked from the OmniScript. Error log CSV generation uses a Visualforce PDF/CSV controller invoked via a URL action on the summary step.

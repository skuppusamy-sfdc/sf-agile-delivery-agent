## Description

As a Claims Operations Manager, I need the adjudication engine to check for an active Group_Service_Agreement__c (GSA) rate override before applying the standard Fee_Schedule__c rate on Facility (Business Account) claims so that negotiated GSA rates are honored for employer group members, ensuring contractual compliance and accurate payment to facilities.

## Acceptance Criteria

Given a Claim__c is submitted where Claim__c.Claim_Type__c = 'Facility' and the rendering provider is a Facility__c (Business Account),
When adjudication begins,
Then the system must first query Group_Service_Agreement__c records linked to the Policy__c.Group_Account__c where GSA.Status__c = 'Active' and GSA.Effective_Date__c <= DOS <= GSA.Expiration_Date__c before consulting the standard Fee_Schedule__c.

Given an active GSA is found with a matching CPT code override,
When the allowed amount is calculated,
Then Claim_Line__c.Allowed_Amount__c must use GSA.Override_Allowed_Amount__c and Claim_Line__c.Rate_Source__c must be set to 'GSA Override'.

Given no active GSA override exists for the CPT code,
When the adjudication engine falls through,
Then standard Fee_Schedule__c lookup logic (per INS-0915) must apply and Claim_Line__c.Rate_Source__c must be set to 'Fee Schedule'.

Given both a GSA override and a Fee_Schedule__c record exist for the same CPT code,
When the adjudication engine evaluates priority,
Then GSA override must always take precedence over the standard fee schedule (GSA wins).

Given Claim_Line__c.Rate_Source__c = 'GSA Override',
When the EOB is generated,
Then Explanation_of_Benefits__c.Rate_Override_Indicator__c must be set to true and EOB must display the GSA reference number (GSA.Agreement_Number__c).

-Given multiple active GSAs exist for the same group and CPT, apply the lowest allowed amount.- (Requirement updated: apply the GSA with the most recent Effective_Date__c per legal review 2024-10-15.)

## Solution

Extend FeeScheduleLookupService.cls to add a new method `resolveGSAOverride(claimLineId, groupAccountId, dos, cptCode)` that queries Group_Service_Agreement__c and GSA_Line_Item__c (CPT-level child object) before the standard fee schedule query. Return a RateResolutionResult wrapper class containing source type, allowed amount, and GSA reference. Update the Record-Triggered Flow on Claim_Line__c to pass Policy__c.Group_Account__c as an additional parameter to the invocable. Add Claim_Line__c.Rate_Source__c picklist field (GSA Override, Fee Schedule, Manual Override) via field metadata deployment. EOB generation Apex class (EOBGenerationService.cls) updated to conditionally populate Rate_Override_Indicator__c and Agreement_Number__c. Custom metadata (GSA_Priority_Config__mdt) stores tie-breaking rules.

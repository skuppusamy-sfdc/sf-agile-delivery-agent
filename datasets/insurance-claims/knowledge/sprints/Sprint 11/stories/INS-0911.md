## Description

As a Provider Network Specialist, I need a guided OmniScript interface that allows me to upload a CSV file of CPT codes and unit rates and have the system create or update Fee_Schedule_Line__c records in bulk so that annual rate table updates — which often contain thousands of CPT codes — can be processed in hours rather than days of manual data entry.

## Acceptance Criteria

Given the Provider Network Specialist launches the 'Fee Schedule Import' OmniScript from a Fee_Schedule__c record,
When the first step is rendered,
Then the user must be able to upload a CSV file up to 5MB containing columns: CPT_Code__c, Description__c, Unit_Rate__c, Modifier__c, LOB__c.

Given the CSV is uploaded,
When the system parses the file,
Then a DataRaptor Transform must validate that: CPT_Code__c is a 5-character alphanumeric string, Unit_Rate__c is a positive decimal, and LOB__c matches an active picklist value on Fee_Schedule__c.LOB__c.

Given validation passes,
When the import is confirmed by the user,
Then a batch Apex job must upsert Fee_Schedule_Line__c records using CPT_Code__c + Fee_Schedule__c.Id as the composite external Id key.

Given the batch job completes,
When results are available,
Then the OmniScript final step must display: total records processed, records created, records updated, and records failed with reason codes.

Given one or more rows fail validation,
When the error summary is displayed,
Then the user must be able to download a CSV error report listing the failed row number, CPT_Code__c value, and error message.

Given the import is for a Fee_Schedule__c with Status__c = 'Published',
When the OmniScript is launched,
Then a warning modal must inform the user that importing to a published schedule will trigger PCN generation per INS-0908 logic.

-Given the import file exceeds 5MB, the system must reject the upload with an inline error.- (Deprecated: limit raised to 10MB after infrastructure review per architect sign-off 2024-11-14)

## Solution

Build OmniScript 'FeeScheduleImportOS' with steps: (1) File Upload element storing base64 CSV in integration procedure context, (2) DataRaptor Transform 'DR_ParseFeeScheduleCSV' to parse and validate CSV rows, (3) Review step displaying parsed preview table via a child FlexCard, (4) Confirmation step calling an Integration Procedure that invokes Apex 'FeeScheduleImportBatch' via @InvocableMethod. The batch uses Database.upsert with allOrNone=false and collects SaveResult errors. Results are written to a transient Fee_Schedule_Import_Log__c object. A ContentVersion record is created for the error CSV download using Apex ContentVersion insert. File size limit enforced client-side in the OmniScript File Upload element configuration (maxFileSize: 10240KB).

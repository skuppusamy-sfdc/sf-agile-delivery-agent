## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to surface a recommended Fee_Schedule__c record during FNOL intake when a provider NPI and service type are entered so that claims intake agents receive proactive rate guidance at the point of first notice, reducing downstream adjudication corrections and improving first-pass resolution rates.

## Acceptance Criteria

Given an agent is completing an FNOL OmniScript and has entered a valid Provider NPI in the Provider_NPI__c field,
When the NPI is validated against the NPPES integration and a matching Provider__c record is found,
Then the Agentforce AI panel displays a 'Fee Schedule Recommendation' card showing up to 3 ranked Fee_Schedule__c options ordered by LOB relevance and Effective_Date__c recency.

Given the agent selects a recommended Fee_Schedule__c from the AI panel,
When the selection is confirmed,
Then Claim__c.Recommended_Fee_Schedule__c is populated with the selected Fee_Schedule__c.Id and a confidence score (AI_Confidence_Score__c) is recorded on the Claim__c record.

Given the AI cannot identify a matching Fee_Schedule__c with confidence score >= 0.75,
When the recommendation engine returns results,
Then the AI panel displays 'No High-Confidence Fee Schedule Found — Manual Selection Required' and Claim__c.Fee_Schedule_Resolution_Method__c is set to 'Manual'.

Given a Fee Schedule recommendation is accepted or dismissed,
When the agent action is recorded,
Then an AI_Recommendation_Feedback__c record is created with fields: Claim__c.Id, Recommended_Fee_Schedule__c.Id, Agent_Action__c ('Accepted'|'Dismissed'), and Timestamp__c — for model feedback loop.

-Given the provider is out-of-network, the AI should recommend a UCR (Usual, Customary, and Reasonable) rate schedule- (deprecated — UCR recommendation logic deferred to Phase 3 per product decision)

## Solution

Configure an Agentforce Agent Action backed by an Apex invocable method (FeeScheduleRecommendationAction) that accepts NPI and LOB as inputs, queries Provider_Network_Agreement__c and Fee_Schedule__c, and returns a ranked list using a scoring algorithm weighting LOB match (60%), date recency (30%), and CPT coverage breadth (10%). Expose the action in the Agentforce Agent Builder as a 'Fee Schedule Recommendation' topic. Build an LWC panel (feeScheduleRecommendationPanel) embedded in the FNOL OmniScript to render AI results. Store feedback in AI_Recommendation_Feedback__c via a Record-Triggered Flow on Claim__c. Ensure all AI outputs are logged to a custom AI_Audit_Log__c object for compliance.

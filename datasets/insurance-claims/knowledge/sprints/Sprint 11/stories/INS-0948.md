## Description

As a Provider Relations Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record and deliver a notification to the affected provider whenever a Fee_Schedule__c rate amendment is approved on their Provider_Network_Agreement__c, so that we remain compliant with state-mandated 90-day advance notice requirements for rate changes.

This story covers the PCN record creation, DocuSign envelope dispatch for provider acknowledgment, and status tracking through the PCN lifecycle.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where Amendment_Status__c transitions to 'Approved' and the amendment type includes a Fee_Schedule__c rate change
When the record is saved
Then a Provider_Change_Notification__c record must be created with: PCN_Type__c = 'Rate Change', PCN_Status__c = 'Pending Delivery', Effective_Date__c = Fee_Schedule__c.Effective_Date__c, Related_PNA__c = the triggering PNA Id, and Notice_Required_By__c = Effective_Date__c minus 90 days

Given a Provider_Change_Notification__c record with PCN_Status__c = 'Pending Delivery'
When the record is created
Then a DocuSign envelope must be dispatched to the provider's primary email (Provider__r.PersonEmail) containing the rate amendment summary PDF generated via Conga CLM template 'PCN_Rate_Amendment_v2'

Given the DocuSign envelope is sent successfully
When the envelope status callback is received
Then Provider_Change_Notification__c.PCN_Status__c must update to 'Delivered' and Delivery_Timestamp__c must be stamped

Given Provider_Change_Notification__c.Notice_Required_By__c is within 5 calendar days and PCN_Status__c != 'Acknowledged'
When the daily compliance check process runs
Then a High priority Case must be created linked to the PCN with Subject = 'PCN Acknowledgment Overdue: [Provider NPI]' and assigned to the Provider Network Specialist queue

Given the provider signs the DocuSign envelope
When the signed callback is received
Then Provider_Change_Notification__c.PCN_Status__c = 'Acknowledged', Acknowledged_Date__c is stamped, and the related Provider_Network_Agreement__c.Rate_Change_Acknowledged__c checkbox is set to true

-Given PCN_Type__c = 'Rate Change', a physical mail fallback must be triggered if no valid email exists- (deprecated: physical mail handled by external vendor integration outside Salesforce scope per architecture decision ADR-044)

## Solution

1. Record-Triggered Flow on Provider_Network_Agreement__c fires on update when Amendment_Status__c changes to 'Approved'; uses Decision element to check for fee schedule rate change amendment type.
2. Flow creates Provider_Change_Notification__c via Create Records element and invokes Apex Action 'DispatchPCNDocuSign' which calls the DocuSign eSignature REST API via Named Credential.
3. Conga CLM template merge triggered via Conga API callout within the Apex Action, passing Fee_Schedule__c rate delta fields as merge parameters.
4. DocuSign status webhook handled by a @RestResource Apex class 'DocuSignPCNCallbackController' that updates PCN_Status__c and stamps timestamps.
5. Scheduled Flow 'PCN_Compliance_Check' runs daily, queries overdue PCNs using SOQL filter on Notice_Required_By__c and PCN_Status__c, and creates Cases via Create Records element.
6. Permission Set 'PCN_Manager' grants Provider Relations Specialists CRUD on Provider_Change_Notification__c.

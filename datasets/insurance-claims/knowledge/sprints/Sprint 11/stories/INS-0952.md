## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to surface relevant Fee Schedule information and rate change history when a claims adjudicator is working on a Claim__c record, so that adjusters can make faster and more accurate adjudication decisions without leaving the claim record.

The AI assistant should be able to answer natural language questions such as 'What is the current contracted rate for CPT 99213 for this provider?' and 'Has this provider's fee schedule changed in the last 90 days?' by retrieving live Salesforce data via Agent Actions.

## Acceptance Criteria

Given an adjudicator has a Claim__c record open and the Agentforce panel is active
When the adjudicator types 'What is the contracted rate for [CPT code] for this provider?'
Then the Agentforce agent must invoke the 'GetActiveFeeSchedule' Agent Action, retrieve Fee_Schedule__c.Allowed_Amount__c for the matching CPT_Code__c and PNA, and return a natural language response including the rate, effective date, and LOB within 5 seconds

Given the Agentforce agent retrieves Fee Schedule data
When the Fee_Schedule__c record was updated within the last 90 days (based on LastModifiedDate)
Then the agent response must include a proactive notice: 'Note: This fee schedule was updated on [date]. A rate change notification (PCN) was sent on [PCN Delivery_Timestamp__c].'

Given the adjudicator asks about rate history
When the agent queries Fee_Schedule_Audit__c records for the relevant PNA
Then the agent must summarize the last 3 rate changes in a bulleted list format including old rate, new rate, and effective date

Given the Agentforce agent cannot find an active Fee_Schedule__c for the queried CPT code
When the agent responds
Then it must state 'No active fee schedule found for CPT [code] under this provider's network agreement' and suggest the adjudicator pend the claim using Claim__c.Status__c = 'Pended - No Fee Schedule'

Given the agent action is invoked
When the response is generated
Then all data returned must be sourced exclusively from Salesforce objects (no hallucination) and the agent must cite the Fee_Schedule__c record Id in its response for traceability

## Solution

1. Agentforce Agent configured in Setup > Einstein > Agents with custom 'Claims Adjudication Assistant' agent topic scoped to Claim__c record pages.
2. Agent Action 'GetActiveFeeSchedule' built as an Apex-defined action calling FeeScheduleService.getApplicableRate() (fixed per INS-0950) and returning a structured response object.
3. Agent Action 'GetFeeScheduleHistory' queries Fee_Schedule_Audit__c ordered by Unlock_Timestamp__c DESC LIMIT 3 for the relevant PNA.
4. Agent Instructions (system prompt) configured to enforce data grounding: 'Always cite the Salesforce record Id when referencing fee schedule data. Never infer or estimate rates.'
5. Einstein Trust Layer configured with data masking for TIN fields to prevent PII exposure in agent responses.
6. Agent embedded on Claim__c Lightning Record Page via Agentforce for Service component in App Builder.
7. Agent topic instructions include fallback behavior definition for missing fee schedule scenario mapping to Claim__c status pend action.

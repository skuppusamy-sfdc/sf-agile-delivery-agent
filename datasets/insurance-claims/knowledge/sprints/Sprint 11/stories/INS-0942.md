## Description

**Bug Report**

**Summary:** When a Provider_Network_Agreement__c has more than one active Fee_Schedule__c record covering overlapping LOBs, the Claim_Line__c.Allowed_Amount__c remains null after adjudication instead of being populated with the correct rate.

**Steps to Reproduce:**
1. Navigate to a Provider_Network_Agreement__c record (e.g., PNA-00412) that has two active Fee_Schedule__c records: one for LOB = 'Commercial' and one for LOB = 'All'.
2. Create a Claim__c with Claim.LOB__c = 'Commercial' and add a Claim_Line__c with CPT_Code__c = '99213' and Date_of_Service__c = today.
3. Set Claim_Line__c.Status__c = 'Pending Adjudication' and save.
4. Observe Claim_Line__c.Allowed_Amount__c.

**Expected Result:** Claim_Line__c.Allowed_Amount__c is populated with the rate from the 'Commercial' LOB-specific Fee_Schedule__c record (more specific LOB takes precedence over 'All').

**Actual Result:** Claim_Line__c.Allowed_Amount__c = null. No adjudication error code is set either, leaving the claim line in a silent failure state.

**Environment:** Acme Insurance - Full Sandbox (CS47)
**Frequency:** 100% reproducible when PNA has overlapping LOB fee schedules.
**Severity:** P1 — affects all multi-LOB providers currently in adjudication queue (~340 claim lines impacted).

## Acceptance Criteria

As per Description. Additionally: after fix, regression tests must confirm that LOB-specific Fee_Schedule__c records always take precedence over LOB = 'All' schedules, and Claim_Line__c.Allowed_Amount__c is correctly populated in all LOB combinations.

## Solution

Root cause: the SOQL query in 'FeeScheduleValidationService' Apex class uses ORDER BY CreatedDate ASC with LIMIT 1, which can return the 'All' LOB record before the specific LOB record depending on creation order. Fix: update SOQL to add a CASE-based ordering via a custom sort field or apply post-query sorting in Apex to prioritize exact LOB match over 'All'. Add a new method 'resolveBestMatchSchedule()' with explicit precedence logic: (1) exact LOB match, (2) LOB = 'All'. Update unit tests to cover multi-schedule scenarios. Deploy hotfix to Full Sandbox for QA validation before production push.

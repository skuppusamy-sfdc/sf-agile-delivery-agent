## Description

As a Claims Operations Manager, I need Payment__c records to be automatically created when a Claim__c reaches 'Adjudicated' status so that disbursement workflows are triggered without manual intervention, reducing processing lag and human error in payment initiation.

## Acceptance Criteria

Given a Claim__c record where Status__c transitions to 'Adjudicated'
When all Claim_Line__c records under the parent Claim__c have Line_Status__c = 'Approved' or 'Partially Approved'
Then a Payment__c record is auto-created with Payment__c.Status__c = 'Pending', Payment__c.Claim__c populated with the parent Claim__c Id, and Payment__c.Payment_Amount__c = sum of Claim_Line__c.Allowed_Amount__c

Given a Payment__c record is created
When Payment__c.Payment_Method__c is evaluated
Then if Provider__c.Preferred_Payment_Method__c = 'EFT', Payment__c.Payment_Method__c is set to 'EFT'; otherwise default to 'Check'

Given a Claim__c with COB (Coordination of Benefits) flag Claim__c.COB_Applicable__c = true
When the Payment__c record is created
Then Payment__c.COB_Adjustment_Amount__c must be populated and Payment__c.Payment_Amount__c must reflect the post-COB net payable amount

Given a Payment__c record is created
When the record is saved
Then a Platform Event (Payment_Initiated__e) is published with fields: claimId, paymentId, paymentAmount, paymentMethod, and timestamp

-Given Claim__c.LOB__c = 'Workers Comp', a separate WC_Payment__c object should be used instead of Payment__c- (deprecated per INS-0571 architecture decision; Payment__c handles all LOBs via LOB__c field)

Given a Claim__c has Status__c = 'Denied'
When the record is saved
Then no Payment__c record is created and a Claim_Note__c is logged with Note_Type__c = 'System' and Note_Body__c = 'Claim denied — payment suppressed'

## Solution

Implement a Record-Triggered Flow on Claim__c firing on Status__c update to 'Adjudicated'. Flow uses a Roll-Up Summary or invoked Apex to sum Claim_Line__c.Allowed_Amount__c. A Decision element branches on COB_Applicable__c. Payment__c record is created via Flow Create Records element. An Apex Action within the Flow publishes the Payment_Initiated__e Platform Event. Apex trigger on Payment__c handles EFT vs Check defaulting by querying Provider__c.Preferred_Payment_Method__c. Unit tests cover COB and non-COB paths, denial suppression, and platform event publish verification.

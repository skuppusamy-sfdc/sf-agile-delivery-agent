## Description

As a Compliance Officer, I need claims flagged by the Special Investigations Unit (SIU) to automatically have their associated Payment__c records placed on hold, and I need a controlled manual release workflow requiring dual approval so that fraudulent or suspicious disbursements are prevented until SIU investigation is complete. SIU flags originate from Claim__c.SIU_Referred__c = true, set either manually or by the fraud scoring integration.

## Acceptance Criteria

Given a Claim__c record where SIU_Referred__c is set to true
When the Record-Triggered Flow on Claim__c fires
Then all related Payment__c records with Payment_Status__c IN ('Pending', 'Approved') are updated to Payment_Status__c = 'On Hold - SIU'
And Payment__c.SIU_Hold_Date__c = TODAY()
And a Task is created assigned to the SIU Investigations queue with Subject = 'SIU Payment Hold Applied - Review Required' and linked to the Claim__c

Given Payment__c.Payment_Status__c = 'On Hold - SIU'
When a Compliance Officer or SIU Investigator initiates a 'Release SIU Hold' action
Then an Approval Process ('SIU_Hold_Release_Approval') is submitted requiring two sequential approvers: (1) SIU Lead and (2) Compliance Officer
And Payment__c.SIU_Release_Requested_By__c = current user
And Payment__c.SIU_Release_Request_Date__c = TODAY()

Given the dual approval is fully approved
When both approvers have approved
Then Payment__c.Payment_Status__c = 'Pending'
And Payment__c.SIU_Hold_Released__c = true
And Claim__c.SIU_Hold_Released__c = true
And an audit record is created in Payment_Audit_Log__c with Action__c = 'SIU Hold Released', Performed_By__c, and Timestamp__c

Given either approver rejects the release request
When a rejection is submitted
Then Payment__c.Payment_Status__c remains 'On Hold - SIU'
And a notification is sent to the requestor via Salesforce In-App Notification
And Payment__c.SIU_Release_Rejection_Reason__c = approver's comments

Given Claim__c.SIU_Referred__c is set back to false (SIU case closed with no fraud finding)
When the Record-Triggered Flow evaluates
Then Payment__c.Payment_Status__c = 'Pending' is restored automatically without requiring the manual release approval process
And Payment_Audit_Log__c records Action__c = 'SIU Hold Auto-Released - Case Closed Clean'

## Solution

Record-Triggered Flow on Claim__c (After Save) handles SIU_Referred__c = true scenario, bulk-updating related Payment__c records via Flow Loop + Update Records. Salesforce Approval Process 'SIU_Hold_Release_Approval' configured with two sequential approval steps using 'Related User' criteria for SIU Lead (Role hierarchy) and Compliance Officer (Permission Set). Approval process entry criteria: Payment__c.Payment_Status__c = 'On Hold - SIU'. Payment_Audit_Log__c records created via Apex trigger on Payment__c (after update) to capture all status transitions for SOX compliance audit trail. Custom Permission 'SIU_Hold_Release_Initiator' controls visibility of the release action button on Payment__c page layout.

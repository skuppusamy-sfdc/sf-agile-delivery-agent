## Description

As a Claims Operations Manager, I need an Explanation_of_Benefits__c (EOB) record and associated PDF document to be automatically generated when a Payment__c record reaches 'Disbursed' status so that policyholders receive timely, accurate benefit summaries compliant with state-mandated EOB disclosure requirements. Extends INS-0529 payment automation workflow.

## Acceptance Criteria

Given a Payment__c record where Payment__c.Status__c transitions to 'Disbursed'
When the Record-Triggered Flow fires
Then an Explanation_of_Benefits__c record is created with EOB__c.Claim__c = Payment__c.Claim__c, EOB__c.Payment__c = Payment__c.Id, EOB__c.EOB_Date__c = TODAY(), EOB__c.Status__c = 'Generated'

Given the EOB__c record is created
When Conga CLM template merge is invoked
Then a PDF is generated using EOB template 'ACME_EOB_v3' and attached to EOB__c as a ContentDocument with ContentVersion.Title = 'EOB_[ClaimNumber]_[EOBDate]'

Given Claim_Line__c records associated with the Claim__c
When the EOB PDF is rendered
Then each Claim_Line__c is listed with: CPT code (Claim_Line__c.CPT_Code__c), billed amount (Claim_Line__c.Billed_Amount__c), allowed amount (Claim_Line__c.Allowed_Amount__c), member responsibility (Claim_Line__c.Member_Responsibility__c), and denial reason code if Claim_Line__c.Status__c = 'Denied'

Given EOB__c.Status__c = 'Generated'
When generation is confirmed
Then an automated email is dispatched to Policy__c.Policyholder_Email__c using Email Template 'EOB_Member_Notification' and EOB__c.Delivery_Method__c is set to 'Email'

-Given paper EOB delivery preference, a print job API call to third-party print vendor is triggered-

Given a Payment__c with Payment__c.COB_Applied__c = TRUE
When EOB is generated
Then EOB__c.COB_Indicator__c = TRUE and a secondary payer section is rendered in the PDF showing Claim__c.Secondary_Payer_Amount__c

## Solution

Record-Triggered Flow on Payment__c AFTER UPDATE, entry condition: Status__c changed to 'Disbursed'. Create EOB__c record, then invoke Conga CLM via an Apex callout action (CongaMergeService.generateEOB()) passing EOB__c Id and template ID. Store resulting PDF via ContentVersion insert linked to EOB__c. Send email via Messaging.sendEmail() in Apex. COB logic handled by a separate Apex helper class (COBEOBRenderer) that appends secondary payer data to the Conga template data map. Add EOB__c to the 'Claims_Payment_Ops' and 'Member_Services' Permission Set Groups.

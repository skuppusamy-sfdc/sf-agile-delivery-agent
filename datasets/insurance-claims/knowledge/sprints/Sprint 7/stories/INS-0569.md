## Description

As a Claims Operations Manager, I need the system to automatically generate an Explanation_of_Benefits__c (EOB) record and corresponding PDF document when a Claim__c is moved to 'Adjudicated' status, so that policyholders and providers receive timely, accurate payment explanations compliant with state and CMS disclosure requirements.

## Acceptance Criteria

Given a Claim__c record with Status__c = 'Adjudicated' and all Claim_Line__c records in a terminal state (Paid, Denied, or Partially Paid),
When the Record-Triggered Flow fires on Claim__c.Status__c update,
Then an Explanation_of_Benefits__c record is created with EOB_Status__c = 'Generated', linked via Claim__c lookup, and populated with Member_Name__c, Member_ID__c, Date_of_Service__c, Total_Billed__c, Total_Allowed__c, Plan_Paid__c, Member_Responsibility__c, and Denial_Reason_Code__c (if applicable).

Given the EOB record is created,
When Conga CLM template merge is triggered via Apex invocable action,
Then a PDF is generated using the 'EOB_Standard_v2' Conga template and attached to the EOB record as a ContentVersion with ContentDocument linked, and EOB__c.Document_Generated__c = true.

Given the EOB PDF is attached,
When EOB_Status__c transitions to 'Generated',
Then a Platform Event (EOB_Ready__e) is published containing Claim_ID__c, Member_Email__c, and Provider_NPI__c so downstream notification services can dispatch member and provider copies.

Given a Claim__c with COB (Coordination of Benefits) flag COB_Active__c = true,
When the EOB is generated,
Then EOB.COB_Primary_Payer__c, COB_Primary_Paid__c, and COB_Secondary_Responsibility__c fields are populated from the linked COB_Detail__c child records.

-Given EOB generation, a fax transmission to the provider should be triggered automatically via the legacy fax integration.- (Deprecated: replaced by ERA electronic transmission per INS-0541)

Given any EOB generation failure (Conga timeout or missing required field),
When the Apex invocable action catches an exception,
Then Claim__c.EOB_Generation_Status__c = 'Failed', a Salesforce Task is created assigned to the Claims Operations queue, and an error entry is written to the Integration_Error_Log__c object with Error_Type__c = 'EOB_Generation_Failure'.

## Solution

Implement a Record-Triggered Flow on Claim__c firing on Status__c update to 'Adjudicated'. Flow invokes an Apex invocable action (EOBGenerationService.cls) that queries all child Claim_Line__c records, aggregates financial totals, creates the Explanation_of_Benefits__c record, and calls the Conga CLM REST API to merge the EOB_Standard_v2 template. On success, publish EOB_Ready__e Platform Event via EventBus.publish(). On failure, update Claim__c.EOB_Generation_Status__c and insert Integration_Error_Log__c via a separate savepoint. Use Custom Metadata (EOB_Config__mdt) to store Conga template IDs and endpoint URLs per LOB. Deploy with Permission Set Group 'Claims_Processor_PSG' granting FLS on all new EOB fields.

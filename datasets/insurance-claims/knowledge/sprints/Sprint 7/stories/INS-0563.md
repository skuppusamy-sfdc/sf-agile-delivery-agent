## Description

As a Claims Operations Manager, I need the system to generate an ANSI X12 835 Electronic Remittance Advice (ERA) file for each approved EFT payment batch so that providers can automatically reconcile payments against their claims in their practice management systems. The ERA file must be transmitted to the provider's designated clearinghouse endpoint stored on the Provider__c record.

## Acceptance Criteria

Given one or more Payment__c records with Payment__c.Status__c = 'Approved' and Payment__c.Payment_Method__c = 'EFT' within the same payment batch
When the nightly scheduled Apex batch job (ERAGenerationBatch.cls) runs at 02:00 UTC
Then an ANSI X12 835 file is generated grouping all qualifying Payment__c records by Provider__c.TIN__c

Given the 835 file is generated
When the file is assembled
Then each claim loop (CLP segment) must include Claim__c.External_Claim_Number__c, Payment__c.Payment_Amount__c, Claim__c.Billed_Amount__c, and all associated Claim_Line__c service line adjustments (SVC/CAS segments)

Given the 835 file is ready for transmission
When Provider__c.Clearinghouse_Endpoint__c is populated
Then the file is transmitted via SFTP using the stored credentials in Salesforce Named Credentials ('Clearinghouse_SFTP') and ERA_Transmission__c record is created with ERA_Transmission__c.Status__c = 'Sent'

Given a transmission failure (SFTP timeout or authentication error)
When the Apex batch encounters an exception
Then ERA_Transmission__c.Status__c = 'Failed', ERA_Transmission__c.Failure_Reason__c is populated, and a retry is scheduled for 60 minutes later (max 3 retries)

Given all Payment__c records in a batch are included in the 835 file
When transmission succeeds
Then each Payment__c.ERA_Sent__c = TRUE and Payment__c.ERA_Transmission_Date__c = TODAY()

## Solution

Scheduled Apex batch (ERAGenerationBatch.cls) queries Payment__c records meeting criteria. An Apex utility (ANSI835Builder.cls) constructs the X12 835 string using a template-based segment builder pattern, iterating over related Claim__c and Claim_Line__c records via SOQL. The assembled file is stored temporarily as a ContentDocument. SFTP transmission is handled via an Apex callout to an MuleSoft middleware layer using Named Credentials. ERA_Transmission__c records are created via DML post-transmission. Retry logic uses a custom Schedulable Apex class (ERARetryScheduler.cls) triggered on failure. A custom Permission Set 'ERA_Operations' controls access to ERA_Transmission__c.

## Description

**Bug Report**

**Summary:** When a Claim__c contains a mix of approved and denied Claim_Line__c records, the Payment__c.Amount__c is being calculated as the sum of ALL Claim_Line__c.Allowed_Amount__c values including denied lines, resulting in overpayment.

**Steps to Reproduce:**
1. Create a Claim__c record with Status__c = 'In Adjudication'
2. Add 3 Claim_Line__c records: Line 1 (Allowed_Amount__c = $150, Status__c = 'Approved'), Line 2 (Allowed_Amount__c = $200, Status__c = 'Denied', Denial_Reason__c = 'Not Medically Necessary'), Line 3 (Allowed_Amount__c = $75, Status__c = 'Approved')
3. Transition Claim__c.Status__c to 'Adjudicated'
4. Observe the auto-created Payment__c record

**Expected Result:** Payment__c.Amount__c = $225.00 (sum of approved lines only: $150 + $75)

**Actual Result:** Payment__c.Amount__c = $425.00 (sum of all lines including denied: $150 + $200 + $75)

**Environment:** Acme Insurance Full Sandbox
**Introduced in:** INS-0577 implementation
**Severity:** Critical — active overpayments being disbursed in staging

## Acceptance Criteria

As per Description. Payment__c.Amount__c must equal the sum of Claim_Line__c.Allowed_Amount__c WHERE Claim_Line__c.Status__c = 'Approved' only. Denied, Pending, and Voided lines must be excluded from the payment calculation.

## Solution

Root cause identified in the Record-Triggered Flow (INS-0577) where the Get Records element retrieving Claim_Line__c records for amount rollup does not filter by Claim_Line__c.Status__c. Fix: add filter condition Status__c = 'Approved' to the Get Records element in the flow. Additionally, update the Apex invocable method used for COB adjustment to apply the same filter in its SOQL query: SELECT SUM(Allowed_Amount__c) FROM Claim_Line__c WHERE Claim__c = :claimId AND Status__c = 'Approved'. Deploy fix to sandbox, execute regression test suite covering mixed-status claim scenarios, then promote to production via change set.

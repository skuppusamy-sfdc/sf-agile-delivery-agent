## Description

As a Claims Operations Manager, I need each Claim_Line__c to automatically look up the applicable Fee_Schedule__c rate based on the provider's Provider_Network_Agreement__c (PNA), CPT code, and service date so that approved amounts are calculated consistently and in compliance with contracted rates rather than relying on manual adjuster lookups.

## Acceptance Criteria

Given a Claim_Line__c record is saved with CPT_Code__c, Service_Date__c, and a parent Claim__c linked to a Provider__c with an active PNA
When the Apex trigger (ClaimLineAdjudicationTrigger) fires
Then the system queries Fee_Schedule__c where Fee_Schedule__c.PNA__c = Provider_Network_Agreement__c.Id AND Fee_Schedule__c.CPT_Code__c = Claim_Line__c.CPT_Code__c AND Fee_Schedule__c.Effective_Date__c <= Service_Date__c AND Fee_Schedule__c.Expiration_Date__c >= Service_Date__c

Given a matching Fee_Schedule__c record is found
When the lookup succeeds
Then Claim_Line__c.Approved_Amount__c = MIN(Claim_Line__c.Billed_Amount__c, Fee_Schedule__c.Allowed_Amount__c) and Claim_Line__c.Fee_Schedule__c lookup field is populated

Given no matching Fee_Schedule__c record exists for the CPT/PNA/date combination
When the lookup returns zero results
Then Claim_Line__c.Adjudication_Status__c = 'Pending_Review', Claim_Line__c.Adjudication_Note__c = 'No active fee schedule found for CPT ' + CPT_Code__c, and the claim is routed to the manual adjudication queue

Given the Provider__c has no active PNA (Provider_Network_Agreement__c.Status__c != 'Active')
When adjudication is attempted
Then Claim_Line__c is flagged with Out_of_Network__c = true and Approved_Amount__c is calculated using the default OON rate from a system-level Custom Metadata Type (OON_Rate_Config__mdt)

-Given the GSA overrides the PNA rate, GSA rate should take precedence- (deprecated: GSA rate override logic deferred to Phase 2)

## Solution

Apex Trigger (ClaimLineAdjudicationTrigger.cls) on Claim_Line__c (before insert, before update) invokes FeeScheduleLookupService.cls. The service performs a SOQL query against Fee_Schedule__c with CPT, PNA, and date range filters. OON fallback rates are stored in Custom Metadata Type OON_Rate_Config__mdt keyed by LOB__c. Results are bulk-safe, processing up to 200 lines per transaction. FeeScheduleLookupService is unit-tested with 90%+ coverage including boundary date cases. A custom permission 'Bypass_Fee_Schedule_Engine' is available for SIU adjuster override scenarios.

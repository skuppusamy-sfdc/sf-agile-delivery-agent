## Description

**Bug Report**

**Summary:** When a Claim__c contains one or more Claim_Line__c records with Allowed_Amount__c = 0.00 (e.g., fully denied lines or contractual zero-dollar adjustments), the Payment__c record created by the PaymentOrchestration Platform Event subscriber is inserted with Amount__c = null instead of $0.00 or the correct aggregate of non-zero lines.

**Steps to Reproduce:**
1. Create a Claim__c record with Status__c = 'Adjudicated'.
2. Add two Claim_Line__c records: one with Allowed_Amount__c = 150.00 and one with Allowed_Amount__c = 0.00 (Denial_Reason_Code__c = 'CO-45').
3. Trigger EOB generation (Status__c update to 'Adjudicated').
4. Observe the EOB_Ready__e Platform Event fires and PaymentOrchestration subscriber runs.
5. Open the resulting Payment__c record.
6. Observe Amount__c = null.

**Expected Behavior:** Payment__c.Amount__c = 150.00 (sum of non-zero Claim_Line__c.Allowed_Amount__c values where Line_Status__c = 'Paid').

**Actual Behavior:** Payment__c.Amount__c = null, causing downstream EFT batch to skip the record and log a false 'missing amount' error in Integration_Error_Log__c.

**Environment:** Full Sandbox (ACM-FULL-SBX1)
**Frequency:** 100% reproducible when any Claim_Line__c has Allowed_Amount__c = 0.00
**Introduced by:** INS-0570 deployment on Sprint 7 Day 2

## Acceptance Criteria

As per Description. Payment__c.Amount__c must never be null after creation; it should reflect the sum of Claim_Line__c.Allowed_Amount__c where Line_Status__c IN ('Paid', 'Partially_Paid') and default to 0.00 when all lines are denied.

## Solution

Fix in EOBGenerationService.cls: update the SOQL aggregate query that sums Claim_Line__c.Allowed_Amount__c to use COALESCE equivalent — wrap the SUM result with a null-check in Apex (amount = (aggResult != null) ? aggResult : 0.00). Add a validation rule on Payment__c: Amount__c must not be null (ISBLANK(Amount__c) returns error). Add unit test in EOBGenerationServiceTest.cls covering the zero-dollar line scenario with System.assertNotEquals(null, payment.Amount__c). Deploy hotfix to ACM-FULL-SBX1 for QA validation before promoting to UAT.

## Description

As a Claims Operations Manager, I need an Agentforce AI assistant embedded in the Claims console that can answer payment status questions and guide agents through dispute initiation workflows, so that agents can resolve member and provider payment inquiries faster without manually querying multiple related objects or escalating to senior staff.

## Acceptance Criteria

Given an agent opens a Claim__c record in the Claims Service Console,
When the Agentforce 'Claims Payment Assistant' panel is activated,
Then the assistant greets the agent with a summary of the claim's current payment status, pulling from Payment__c.Payment_Status__c, Payment__c.Amount__c, and EOB__c.EOB_Status__c in real time.

Given the agent asks 'Why was this claim underpaid?' in natural language,
When the Agentforce topic action 'ExplainPaymentVariance' is triggered,
Then the assistant returns a human-readable explanation derived from Claim_Line__c.Adjustment_Reason_Code__c, Fee_Schedule__c.Allowed_Amount__c vs Claim_Line__c.Billed_Amount__c, and PNA contractual terms from Provider_Network_Agreement__c.Discount_Percentage__c.

Given the agent asks the assistant to initiate a payment dispute,
When the 'InitiatePaymentDispute' topic action is invoked,
Then the Agentforce assistant collects Dispute_Reason__c, Supporting_Documentation (file upload prompt), and confirms Claim__c.Claim_Number__c before creating a Payment_Dispute__c record with Status__c = 'Open' and assigning to the 'Payment_Disputes' queue.

Given the agent asks about provider banking details during an EFT failure inquiry,
When the assistant processes the query,
Then it explicitly declines to display Provider__c.Bank_Account__c or Provider__c.Bank_Routing__c fields, returns a message 'Banking details are restricted for security purposes', and logs the inquiry to Security_Audit_Log__c with Inquiry_Type__c = 'Sensitive_Field_Request'.

Given any Agentforce topic action that performs DML (e.g., creating Payment_Dispute__c),
When the action completes,
Then the agent receives a confirmation message with the new record's Name and a clickable deep link, and the action is recorded in Agentforce_Audit__c with Agent_User__c, Action_Name__c, and Timestamp__c.

## Solution

Configure Agentforce Agent in Salesforce (Einstein Copilot framework) with a custom 'Claims Payment Assistant' agent persona. Define three custom Topics: 'PaymentStatusInquiry', 'ExplainPaymentVariance', and 'InitiatePaymentDispute'. Each topic backed by Apex Action classes (AgentforcePaymentStatusAction.cls, AgentforceVarianceExplainerAction.cls, AgentforceDisputeInitiationAction.cls) annotated with @InvocableMethod. Use Retrieval Augmented Generation (RAG) grounding on Fee_Schedule__c and PNA records via a custom Einstein Data Library index. Sensitive field masking enforced in AgentforceVarianceExplainerAction via field-level check before returning payload. Agentforce_Audit__c writes in @future to avoid governor limit conflicts. Assign Agentforce agent to Claims_Processor_PSG via Agent Access Policy.

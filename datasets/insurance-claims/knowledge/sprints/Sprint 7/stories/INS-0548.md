## Description

As a Claims Operations Manager, I need the claims adjudication engine to apply COB (Coordination of Benefits) rules when a Claim__c is linked to a policyholder with multiple active Policy__c records so that the primary and secondary payer obligations are correctly calculated, preventing overpayment and ensuring regulatory compliance with state COB mandates.

## Acceptance Criteria

Given a Claim__c where the related Contact has two or more Policy__c records with Policy__c.Status__c = 'Active' and Policy__c.COB_Eligible__c = true
When the claim enters adjudication (Claim__c.Status__c = 'In Adjudication')
Then the system must identify the primary policy using Policy__c.COB_Priority__c = 'Primary' and secondary using Policy__c.COB_Priority__c = 'Secondary' and populate Claim__c.Primary_Policy__c and Claim__c.Secondary_Policy__c accordingly

Given primary and secondary policies are identified
When COB calculation executes
Then Claim__c.Primary_Payer_Liability__c = (Claim__c.Billed_Amount__c * Primary_Policy__c.Coverage_Percentage__c / 100) capped at Claim__c.Billed_Amount__c, and Claim__c.Secondary_Payer_Liability__c = MAX(0, Claim__c.Billed_Amount__c - Claim__c.Primary_Payer_Liability__c - Claim__c.Member_Responsibility__c)

Given COB calculation is complete
When Payment__c records are generated (per INS-0545)
Then two Payment__c records must be created: one linked to Primary_Policy__c with Payment__c.Amount__c = Claim__c.Primary_Payer_Liability__c and one linked to Secondary_Policy__c with Payment__c.Amount__c = Claim__c.Secondary_Payer_Liability__c, each with Payment__c.COB_Role__c = 'Primary' and 'Secondary' respectively

Given Claim__c.COB_Override__c = true (manually flagged by Claims Operations Manager)
When the adjudication flow evaluates COB logic
Then the automated COB calculation must be bypassed and Claim__c.COB_Notes__c must contain a non-null value, enforced by a validation rule

## Solution

Apex invocable method (COBAdjudicationService.cls) called from the existing claims adjudication Flow. Service queries related Policy__c records via SOQL on the policyholder Contact. COB priority resolution logic encapsulated in COBPriorityResolver inner class. Calculation results written back to Claim__c fields via Flow assignment elements post-invocable return. Validation rule on Claim__c enforces COB_Notes__c requirement when COB_Override__c = true. Custom Metadata (COB_Rules__mdt) stores state-specific COB rule overrides keyed by Policy__c.State__c.

## Description

**Bug Report**

**Summary:** For policyholders with Policyholder__c.Delivery_Preference__c = 'Paper', the Explanation_of_Benefits__c.EOB_Status__c field remains stuck at 'Generated' indefinitely. The expected transition to 'Delivered' (triggered when the print vendor confirms mailing) is not occurring, causing compliance reporting dashboards to show inflated 'Generated' counts and masking actual delivery SLA metrics.

**Steps to Reproduce:**
1. Ensure a Policyholder__c record exists with Delivery_Preference__c = 'Paper' and a valid Mailing_Address__c.
2. Adjudicate a linked Claim__c (Status__c = 'Adjudicated').
3. Confirm EOB is generated: Explanation_of_Benefits__c.EOB_Status__c = 'Generated', Document_Generated__c = true.
4. Wait for the nightly print vendor callback batch to run (or manually trigger PrintVendorCallbackBatch.cls in Execute Anonymous).
5. Query Explanation_of_Benefits__c.EOB_Status__c.
6. Observe: EOB_Status__c = 'Generated' (unchanged).

**Expected Behavior:** After print vendor confirms mailing via inbound REST callback, EOB_Status__c should update to 'Delivered' and Delivered_Timestamp__c should be populated.

**Root Cause (Suspected):** The PrintVendorCallbackHandler REST endpoint is filtering on EOB__c.Delivery_Channel__c = 'Electronic' (hardcoded incorrect filter) instead of 'Paper', so paper EOB records are never matched and updated.

**Environment:** Full Sandbox (ACM-FULL-SBX1) and UAT Sandbox (ACM-UAT-SBX2)
**Frequency:** 100% reproducible for all Paper delivery preference members since Sprint 6 deployment.
**Introduced by:** Suspected regression from INS-0555 (Delivery Channel field introduction).

## Acceptance Criteria

As per Description. EOB__c.EOB_Status__c must transition to 'Delivered' and EOB__c.Delivered_Timestamp__c must be populated within one processing cycle after the print vendor callback is received for all records where EOB__c.Delivery_Channel__c = 'Paper'.

## Solution

Fix the hardcoded filter in PrintVendorCallbackHandler.cls: change the SOQL WHERE clause from Delivery_Channel__c = 'Electronic' to Delivery_Channel__c = 'Paper' for the paper callback processing branch. Add a unit test (PrintVendorCallbackHandlerTest.cls) with explicit coverage of both 'Paper' and 'Electronic' delivery channel scenarios asserting correct EOB_Status__c transitions. Run data fix script in both sandboxes to retroactively update stuck 'Generated' EOB records where Delivery_Channel__c = 'Paper' and a vendor confirmation exists in the callback log. Add a monitoring SOQL query to the Compliance Reporting dashboard alerting when EOB_Status__c = 'Generated' age exceeds 5 business days.

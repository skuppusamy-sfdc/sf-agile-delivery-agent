## Description

As a Claims Operations Manager, I need the system to generate ANSI X12 835 ERA files for all EFT-based Payment__c disbursements to providers so that Provider__c (Person Account) and Facility__c (Business Account) entities can reconcile payments against their submitted claims in their practice management systems. ERA files must be transmitted securely to each provider's designated clearinghouse endpoint stored on the Provider_Network_Agreement__c record.

## Acceptance Criteria

Given one or more Payment__c records with Payment__c.Payment_Method__c = 'EFT' and Payment__c.Status__c = 'Issued' within a processing batch window,
When the nightly scheduled Apex batch job (ERA_Generation_Batch) executes at 11:00 PM EST,
Then a single X12 835 ERA file is generated per Provider NPI (Provider__c.NPI__c or Facility__c.NPI__c) grouping all qualifying Payment__c records for that run.

Given the ERA file is generated,
When the file is validated against X12 835 schema,
Then ISA, GS, ST, BPR, TRN, CLP, SVC segments are correctly populated with Payment__c.Amount__c, Claim__c.Claim_Number__c, Claim_Line__c.CPT_Code__c, and Claim_Line__c.Allowed_Amount__c values.

Given Provider_Network_Agreement__c.ERA_Delivery_Endpoint__c is populated,
When the ERA file passes validation,
Then it is transmitted via SFTP to PNA.ERA_Delivery_Endpoint__c and ERA_Transmission_Log__c is created with transmission status, timestamp, and file name.

Given PNA.ERA_Delivery_Endpoint__c is NULL or PNA.Status__c != 'Active',
When ERA generation is attempted,
Then the file is held in ERA_Transmission_Log__c.Status__c = 'Pending Manual Review' and a Task is assigned to the Provider Relations Specialist.

Given ERA transmission fails (SFTP timeout or authentication error),
When the batch encounters the failure,
Then up to 3 retry attempts are made at 15-minute intervals, after which ERA_Transmission_Log__c.Status__c = 'Failed' and a High-priority Case is auto-created on the Provider__c or Facility__c account.

## Solution

Implement Apex Schedulable + Batchable class (ERA_Generation_Batch implements Schedulable, Database.Batchable) scoped to Payment__c records processed in current batch window. X12 835 file construction handled by ERA_835_Builder Apex utility class using StringBuilder pattern per segment. File transmitted via Named Credential-secured SFTP callout using HttpRequest to middleware integration layer (MuleSoft). ERA_Transmission_Log__c records created via DML in batch finish() method. Retry logic implemented via Queueable Apex chain (ERA_Retry_Queueable) with exponential backoff using Custom Metadata Type ERA_Config__mdt storing retry intervals and max attempts. Case auto-creation via Apex leveraging RecordType 'Provider_ERA_Issue'.

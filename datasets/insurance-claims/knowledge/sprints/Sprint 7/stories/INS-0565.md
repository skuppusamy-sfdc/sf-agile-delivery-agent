## Description

**Bug Report**

**Summary:** When a Claim__c has Claim__c.COB_Flag__c = TRUE and more than one COB adjustment record exists on the claim, the Payment__c.Payment_Amount__c is being populated with Claim__c.Billed_Amount__c instead of Claim__c.Approved_Amount__c (post-COB net amount), resulting in overpayment disbursements.

**Steps to Reproduce:**
1. Create a Claim__c record with Claim__c.COB_Flag__c = TRUE and Claim__c.Billed_Amount__c = $1,200.00
2. Add two COB_Adjustment__c child records: first adjustment of -$400.00 (primary payer), second adjustment of -$200.00 (secondary payer)
3. Claim__c.Approved_Amount__c should resolve to $600.00
4. Transition Claim__c.Status__c to 'Adjudicated'
5. Observe Payment__c record created by the Record-Triggered Flow (INS-0561)

**Expected Result:** Payment__c.Payment_Amount__c = $600.00 (Claim__c.Approved_Amount__c)

**Actual Result:** Payment__c.Payment_Amount__c = $1,200.00 (Claim__c.Billed_Amount__c)

**Root Cause Analysis:** The Record-Triggered Flow formula resource 'frmPaymentAmount' references {!Claim__c.Billed_Amount__c} instead of {!Claim__c.Approved_Amount__c}. This appears to be a misconfiguration introduced during the INS-0561 flow build. Single COB adjustment claims are unaffected because a separate formula path was used.

**Environment:** Full Sandbox (ACME-FS1)
**Severity:** P1 — Financial impact risk

## Acceptance Criteria

as per Description

## Solution

Correct the Flow formula resource 'frmPaymentAmount' in the Record-Triggered Flow 'Claim_Adjudication_Payment_Creation' to reference {!$Record.Approved_Amount__c} instead of {!$Record.Billed_Amount__c}. Add a Flow unit test scenario in the associated test class (ClaimPaymentFlowTest.cls) covering COB claims with 2+ adjustment records. Validate fix in Full Sandbox against regression dataset. Coordinate with QA (Rachel Torres) for UAT sign-off before promoting to Staging.

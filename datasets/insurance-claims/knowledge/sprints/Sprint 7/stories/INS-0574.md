## Description

As a Provider (external user on the Acme Provider Portal), I need to view my ERA documents and payment details within the portal, so that I can download remittance files, reconcile claims payments, and review line-level EOB data without contacting Provider Relations. This story covers the Experience Cloud portal surface; ERA file generation is handled in INS-0571.

## Acceptance Criteria

Given a Provider user logged into the Acme Provider Portal (Experience Cloud site) with Profile 'Provider_Community_User',
When they navigate to the 'Payments & Remittance' page,
Then they see a list of ERA_Batch__c records where ERA_Batch__c.Provider__c = the logged-in user's Contact.AccountId, filtered to ERA_Batch__c.Status__c = 'Delivered', sorted by Delivery_Timestamp__c descending.

Given the ERA list is displayed,
When the provider clicks 'Download ERA' on a row,
Then the ContentVersion file attached to that ERA_Batch__c is downloaded as an X12 835 .txt file via a secure Apex Controller method that enforces record-level sharing.

Given the provider clicks on an ERA_Batch__c row to expand details,
When the detail panel opens,
Then Payment__c records linked to that ERA batch are displayed with Claim__c.Claim_Number__c, Date_of_Service__c, Billed_Amount__c, Allowed_Amount__c, Plan_Paid__c, and Adjustment_Reason_Code__c for each Claim_Line__c.

Given a provider whose Provider__c.ERA_Enrollment__c = false,
When they access the Payments & Remittance page,
Then ERA download buttons are hidden and a banner message reads: 'You are not currently enrolled in electronic remittance. Contact Provider Relations to enroll.' with a 'Contact Us' button launching a pre-populated Case creation form.

Given any attempt to access ERA_Batch__c or Payment__c records belonging to a different provider,
When the Apex controller executes a SOQL query,
Then the query enforces 'WITH USER_MODE' (or WITH SECURITY_ENFORCED) and returns no records, logging an unauthorized access attempt to Security_Audit_Log__c.

## Solution

Build LWC 'providerEraDownload' for the Experience Cloud page, backed by an Apex controller (ProviderEraController.cls) using 'with sharing' and WITH USER_MODE in all SOQL. ContentVersion download served via a custom Apex REST endpoint that validates the requesting user's AccountId against ERA_Batch__c.Provider__c before returning a base64-encoded file stream. Guest profile and Provider_Community_User profile restricted via OWD (Private on ERA_Batch__c) with a Sharing Rule granting access only when ERA_Batch__c.Provider__c = Community User's Account. Security_Audit_Log__c writes handled in a separate @future method to avoid mixed DML. Deploy Experience Cloud page via a custom Lightning page in the Provider Portal site.

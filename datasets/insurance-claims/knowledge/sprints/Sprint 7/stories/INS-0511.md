## Description

As a Claims Operations Manager, I need an Agentforce AI assistant embedded on the Claim__c record page that can answer natural language queries about the claim's current payment status, EOB generation state, and any disbursement holds, so that I can quickly triage claim issues without navigating across multiple related records and tabs during high-volume processing periods.

## Acceptance Criteria

Given the Claims Operations Manager opens a Claim__c record page,
When the Agentforce panel is visible in the utility bar,
Then the agent is pre-contextualized with the current Claim__c.Id and displays a greeting referencing Claim__c.Claim_Number__c.

Given the user asks 'What is the payment status for this claim?',
When Agentforce processes the query,
Then the response includes Payment__c.Status__c, Payment__c.Amount__c, Payment__c.Payment_Method__c, and Payment__c.Disbursement_Date__c (if populated) for the linked Payment__c record(s).

Given the user asks 'Has the EOB been sent to the member?',
When Agentforce processes the query,
Then the response includes EOB__c.Status__c, EOB__c.Portal_Delivery_Status__c or EOB__c.Mail_Delivery_Status__c, and EOB__c.Generated_Date__c from the linked Explanation_of_Benefits__c record.

Given Payment__c.Status__c = 'On Hold',
When the agent surfaces the payment status,
Then the response must include Payment__c.Hold_Reason__c and suggest the recommended next action based on the hold reason (e.g., 'Missing EFT banking details — contact Provider Relations').

Given the Claim__c has no linked Payment__c record,
When the user queries payment status,
Then the agent responds: 'No payment record has been generated for Claim [Claim_Number__c]. The claim may still be in adjudication or pending approval.' and does not hallucinate a payment status.

## Solution

Configure Agentforce Agent (Einstein Copilot) with a custom Agent Action backed by an Apex invocable method (ClaimsAIContextService.getClaimPaymentContext()) that retrieves Payment__c and EOB__c data for a given Claim__c.Id. The invocable returns a structured ClaimPaymentContext wrapper object. A custom Prompt Template ('Claim_Payment_Status_Prompt') is authored in the Prompt Builder using the context wrapper and grounding instructions to prevent hallucination. The Agentforce panel is added to the Claim__c Lightning Record Page via the Agentforce utility bar component. Data access governed by the 'Agentforce_Claims_Reader' Permission Set scoped to Payment__c and EOB__c read access only.

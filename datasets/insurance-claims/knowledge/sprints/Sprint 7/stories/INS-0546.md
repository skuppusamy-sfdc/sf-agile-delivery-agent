## Description

As a Claims Operations Manager, I need an Explanation_of_Benefits__c (EOB) record and corresponding PDF document to be generated automatically when a Payment__c record transitions to 'Disbursed' status so that policyholders receive timely, compliant EOB documentation per state regulatory requirements. Extends INS-0545.

## Acceptance Criteria

Given a Payment__c.Status__c transitions to 'Disbursed'
When the Record-Triggered Flow on Payment__c fires
Then an Explanation_of_Benefits__c record is created with EOB__c.Payment__c = Payment__c.Id, EOB__c.Claim__c = Payment__c.Claim__c, EOB__c.Policy__c = Claim__c.Policy__c, EOB__c.EOB_Date__c = TODAY(), and EOB__c.Status__c = 'Pending Generation'

Given the Explanation_of_Benefits__c record is created
When the Conga CLM API callout is initiated from an Apex Queueable job
Then the EOB PDF template (Template_Code__c = 'EOB_STANDARD_V3') is merged with claim, payment, and policyholder data and the resulting document is attached to the EOB__c record as a ContentDocument with ContentDocument.Title = 'EOB_[ClaimNumber]_[EOBDate].pdf'

Given the PDF is successfully attached
When attachment is confirmed
Then EOB__c.Status__c must update to 'Generated' and EOB__c.Generated_Datetime__c must be stamped

Given the policyholder has Policy__c.EOB_Delivery_Preference__c = 'Email'
When EOB__c.Status__c = 'Generated'
Then an automated email must be dispatched via an Email Alert using the 'EOB_Policyholder_Notification' template, addressed to Policy__c.Policyholder_Email__c, with the PDF attached

Given the policyholder has Policy__c.EOB_Delivery_Preference__c = 'Portal'
When EOB__c.Status__c = 'Generated'
Then the ContentDocument must be shared to the policyholder's Experience Cloud community via a ContentDocumentLink with ShareType = 'V' and Visibility = 'AllUsers'

-Given the LOB = 'Medicare Advantage', apply CMS-specific EOB template Template_Code__c = 'EOB_CMS_MA_V1'- (superseded: CMS MA template deferred to Phase 2 per INS-0531)

Given the Conga CLM callout fails after 3 retry attempts
When the Queueable job exhausts retries
Then EOB__c.Status__c must be set to 'Generation Failed', a Case record must be created with Subject = 'EOB Generation Failure - [ClaimNumber]' and assigned to the Claims Operations queue

## Solution

Record-Triggered Flow on Payment__c (After Update, Status__c = 'Disbursed') creates the EOB__c record and enqueues an Apex Queueable class (EOBGenerationQueueable.cls) that performs the Conga CLM REST API callout using Named Credential 'Conga_CLM_Prod'. Retry logic implemented via a counter field on EOB__c (Generation_Attempt_Count__c). Delivery branching handled by a separate Record-Triggered Flow on EOB__c firing on Status__c = 'Generated'. ContentDocumentLink insertion performed in Apex to ensure proper sharing context. Custom Label stores Conga template codes per LOB driven by Custom Metadata (EOB_Template_Config__mdt).

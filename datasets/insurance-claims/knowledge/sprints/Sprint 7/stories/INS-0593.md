## Description

As a Claims Operations Manager, I need the system to automatically generate an Explanation_of_Benefits__c record and a formatted PDF document when a claim payment is finalized, so that policyholders receive timely, accurate benefit summaries in compliance with state mandates and ACA requirements.

The EOB must summarize all Claim_Line__c records associated with the parent Claim__c, reflect COB adjustments where applicable, and be delivered via the policyholder's preferred communication channel (mail or portal). This story extends INS-0571 (Payment__c record creation workflow).

## Acceptance Criteria

Given a Claim__c record where Claim__c.Status__c transitions to 'Adjudicated' and a linked Payment__c record is created with Payment__c.Status__c = 'Issued',
When the Record-Triggered Flow fires on the Payment__c object,
Then an Explanation_of_Benefits__c record is created with EOB__c.Claim__c populated, EOB__c.Policy__c populated from Claim__c.Policy__c, EOB__c.Generated_Date__c = TODAY(), and EOB__c.Status__c = 'Pending Distribution'.

Given the EOB record is created,
When all associated Claim_Line__c records are retrieved (Claim_Line__c.Claim__c = Claim__c.Id),
Then each line is reflected on the EOB with fields: EOB_Line__c.CPT_Code__c, EOB_Line__c.Billed_Amount__c, EOB_Line__c.Allowed_Amount__c, EOB_Line__c.Plan_Paid__c, EOB_Line__c.Member_Responsibility__c, and EOB_Line__c.Adjustment_Reason_Code__c.

Given COB is applicable (Claim__c.COB_Applicable__c = true),
When the EOB is generated,
Then EOB__c.COB_Savings__c is populated with the calculated COB offset and a COB detail section is included in the PDF.

Given the EOB record Status__c = 'Pending Distribution',
When the policyholder's preferred channel (Policy__c.Communication_Preference__c) is evaluated,
Then if preference = 'Portal', EOB__c.Portal_Delivery__c = true and a ContentVersion PDF is attached; if preference = 'Mail', EOB__c.Mail_Delivery_Requested__c = true and a print job Platform Event (EOB_Print_Request__e) is published.

-Given the EOB is generated, When the provider is out-of-network, Then a separate OON penalty line is appended to the EOB PDF footer.- (Deprecated: OON penalty display moved to INS-0601)

Given the EOB PDF is generated via Conga Composer,
When the template merge completes,
Then EOB__c.Status__c transitions to 'Distributed' and EOB__c.Distribution_Timestamp__c is stamped with the current datetime.

## Solution

Implement a Record-Triggered Flow on Payment__c (fires After Insert/Update where Payment__c.Status__c = 'Issued') that invokes an Apex invocable method (EOBGenerationService.generateEOB()) to create the Explanation_of_Benefits__c and child EOB line records via a bulk-safe DML operation. A separate Apex class (CongaEOBComposer) calls the Conga Composer REST API to merge the EOB PDF template and attaches the output as a ContentVersion linked to the EOB record. A Platform Event (EOB_Print_Request__e) is published for mail-preference policyholders, consumed by a downstream print-fulfillment integration via a Trigger-on-Event Apex handler. COB logic is encapsulated in a reusable Apex utility class (COBCalculationUtility) shared with the adjudication engine. Permission Set Group 'Claims_Ops_Standard' grants read access to EOB records for Claims Operations staff.

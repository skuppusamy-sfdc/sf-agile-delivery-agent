## Description

As a Claims Operations Manager, I need the system to generate an 835 HIPAA-compliant ERA file for each batch of provider payments processed on a given business day so that providers and their billing systems can automatically reconcile received payments against submitted claims without manual rework.

## Acceptance Criteria

Given one or more Payment__c records with Status__c = 'Disbursed' and Payment__c.ERA_Included__c = FALSE exist for a given Provider__c (Person Account) on the current business date
When the nightly Scheduled Apex batch job runs at 20:00 local time
Then an ERA file in HIPAA 835 format is generated grouping all qualifying Payment__c records by Provider__c.NPI__c and Provider__c.TIN__c

Given the ERA file is generated
When the file is finalized
Then each Payment__c.ERA_Included__c is set to TRUE, Payment__c.ERA_File_Reference__c is populated with the ERA batch ID, and the file is stored as a ContentVersion linked to the Provider__c record

Given a Claim_Line__c has Claim_Line__c.Adjustment_Reason_Code__c populated
When the ERA file is built
Then the CAS segment in the 835 file includes the correct adjustment reason code and amount per X12 835 specification

Given the ERA batch includes payments for a Provider__c associated with a Fee_Schedule__c where Fee_Schedule__c.Effective_End_Date__c < TODAY()
When the ERA is generated
Then a warning flag ERA_Warning__c = TRUE is set on the Payment__c and the line is annotated in the ERA file with remark code MA130

Given the ERA generation batch completes
When all files are produced
Then a summary report is emailed to the Claims Operations Manager distribution list detailing: total ERA files generated, total payment amount covered, count of flagged records

## Solution

Apex Batchable class (ERA_GenerationBatch) queries Payment__c records meeting criteria using SOQL with aggregate grouping by Provider__c. For each provider group, a separate Apex class (ERA_835Builder) constructs the 835 file string per X12 specification, handling ISA/GS/ST/CLP/SVC/CAS/AMT/SE/GE/IEA segments. The generated file string is serialized and saved as ContentVersion via ContentDocumentLink to the Provider__c record. Batch uses Database.Stateful to accumulate summary metrics for the completion email sent via Messaging.sendEmail. Scheduled via a Scheduled Apex class registered in Setup. ERA_Included__c and ERA_File_Reference__c fields updated in batch finish() method via DML update.

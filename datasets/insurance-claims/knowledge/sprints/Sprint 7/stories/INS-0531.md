## Description

As a Claims Operations Manager, I need the system to generate an ANSI X12 835 (ERA) file for each Provider payment batch so that Providers and their billing systems can automatically reconcile claim payments without manual remittance entry. The ERA file must reference Provider__c (NPI, TIN), Facility__c, and all associated Claim_Line__c adjudication details.

## Acceptance Criteria

Given a Payment__c batch where Payment__c.Payment_Type__c = 'Provider' and Payment__c.Status__c = 'Approved_for_Disbursement'
When the nightly scheduled Apex batch job (ERA_GenerationBatch) runs at 02:00 UTC
Then an 835 ERA file is produced per Provider__c.NPI__c grouping containing all qualifying Payment__c records from the prior business day

Given the ERA file structure
When the 835 is generated
Then the file must include: ISA/GS envelope segments with Acme payer ID, BPR segment with Payment__c.Payment_Amount__c, CLP loop per Claim__c.Claim_Number__c, SVC loop per Claim_Line__c with CPT code and adjudicated amounts, and CAS segments for any Claim_Line__c.Adjustment_Reason_Code__c values

Given the ERA file is generated
When the file is ready
Then it is stored as a ContentDocument attached to a new ERA_Batch__c record and an outbound SFTP push is triggered via a named credential endpoint 'ERA_SFTP_Provider'

Given a Provider__c with Provider__c.ERA_Delivery_Preference__c = 'Portal'
When the ERA is generated
Then the file is also surfaced on the Provider 360 FlexCard under the 'Remittance' tab with download capability

Given a generation failure (SFTP timeout or file validation error)
When the batch encounters an exception
Then ERA_Batch__c.Status__c = 'Failed', a Case is auto-created with Subject = 'ERA Generation Failure — [BatchDate]' and assigned to the 'Claims_Ops_Queue'

## Solution

Implement ERA_GenerationBatch as a Schedulable/Batchable Apex class. Use a custom ERA835Builder Apex utility to construct X12 835 segments from Payment__c and Claim_Line__c data. Group by Provider__c.NPI__c using AggregateResult queries. Write file content to ContentVersion and link to ERA_Batch__c. SFTP push via HttpCallout using Named Credential 'ERA_SFTP_Provider'. Provider Portal FlexCard updated via DataRaptor Extract querying ERA_Batch__c filtered by Provider__c. Error handling uses Platform Events (ERA_Error__e) consumed by a Flow to create Cases.

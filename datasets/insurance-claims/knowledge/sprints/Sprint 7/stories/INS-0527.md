## Description

As a Claims Operations Manager, I need a scheduled batch process to identify all Payment__c records with Payment__c.Payment_Method__c = 'Check' and Payment__c.Status__c = 'Pending' and transmit a structured print file to the third-party check print vendor (Deluxe Financial Services) so that physical checks are printed and mailed to policyholders and providers within the SLA window of 3 business days from payment approval. Referenced by INS-0522 (deprecates the inline check print trigger).

## Acceptance Criteria

Given Payment__c records where Payment__c.Payment_Method__c = 'Check' AND Payment__c.Status__c = 'Pending' AND Payment__c.Approved_Date__c <= TODAY(),
When the Check_Print_Batch Apex job runs at 6:00 AM EST Monday through Friday,
Then all qualifying records are included in a delimited flat file with fields: Payment__c.Id, Payment__c.Amount__c, Payment__c.Payee_Name__c, Payment__c.Payee_Address__c (multi-line), Payment__c.Check_Memo__c = Claim__c.Claim_Number__c.

Given the print file is generated,
When it is transmitted to Deluxe Financial Services via Named Credential SFTP endpoint,
Then Payment__c.Status__c is updated to 'Check Print Submitted', Payment__c.Print_Submission_Date__c = TODAY(), and a Check_Print_Batch_Log__c record is created with file name, record count, and transmission timestamp.

Given the vendor returns a confirmation acknowledgment file,
When the inbound file is processed by the scheduled Apex job (Check_Print_Ack_Processor),
Then Payment__c.Status__c = 'Issued', Payment__c.Check_Number__c is populated from the acknowledgment file, and Payment__c.Issued_Date__c = acknowledgment date.

Given a Payment__c.Payee_Address__c is blank or invalid (fails USPS validation),
When the batch encounters the record,
Then that record is excluded from the print file, Payment__c.Status__c = 'Check Print Hold - Invalid Address', and a Task is assigned to the Claims Operations Manager.

Given the total batch file contains more than 5,000 records,
When the file is generated,
Then it is split into chunks of 5,000 records each with sequential file naming convention 'ACM_CHECK_YYYYMMDD_001.txt', 'ACM_CHECK_YYYYMMDD_002.txt', etc.

## Solution

Implement Apex Schedulable + Batchable class (Check_Print_Batch) with batch size of 200. File generation uses StringBuilder in finish() method aggregating results from batch execute() scope. SFTP transmission via HttpRequest through Named Credential 'Deluxe_SFTP_NC'. Inbound acknowledgment processing via separate Schedulable Apex (Check_Print_Ack_Processor) polling SFTP inbound folder. Address validation integrated via USPS Address Validation API callout in a pre-filter Apex method before batch scope query. File chunking logic in finish() method splits List<String> rows into sublists of 5,000. All batch runs logged in Check_Print_Batch_Log__c with ContentVersion attachment of the generated file for audit trail.

## Description

As a Claims Operations Manager, I need the system to automatically generate a Payment__c record when a Claim__c transitions to 'Adjudicated' status so that disbursement processing is initiated without manual intervention, reducing cycle time and human error in the payment pipeline.

## Acceptance Criteria

Given a Claim__c record where Status__c transitions to 'Adjudicated' and Claim__c.Payment_Method__c is populated
When the Record-Triggered Flow fires on the Claim__c update
Then a Payment__c record is created with Payment__c.Claim__c = the triggering Claim__c.Id, Payment__c.Amount__c = Claim__c.Approved_Amount__c, Payment__c.Status__c = 'Pending', Payment__c.Payment_Method__c = Claim__c.Payment_Method__c, and Payment__c.Due_Date__c = TODAY() + 5 business days

Given a Payment__c record is created successfully
When the record is saved
Then a Platform Event (Payment_Initiated__e) is published with fields: Claim_Id__c, Provider_NPI__c, Payment_Amount__c, and LOB__c

Given the Claim__c.LOB__c = 'Workers Comp'
When the payment record is created
Then Payment__c.Requires_TPA_Approval__c = TRUE and the record is assigned to the TPA Review queue

-Given Payment__c.Payment_Method__c = 'Paper Check', print job is triggered automatically-

Given the Claim__c has an associated COB record where COB.Primary_Payer_Paid__c > 0
When Payment__c is created
Then Payment__c.COB_Adjusted_Amount__c = Claim__c.Approved_Amount__c - COB.Primary_Payer_Paid__c and Payment__c.Amount__c reflects the adjusted value

Given a Claim__c.Status__c transitions to 'Adjudicated' but Claim__c.Provider__c is NULL
When the flow attempts to create a Payment__c
Then the flow halts, logs a ClaimsProcessing_Error__c record with Error_Code__c = 'MISSING_PROVIDER', and sends an email alert to the Claims Operations Manager queue

## Solution

Implement a Record-Triggered Flow on Claim__c (fires After Update) that evaluates the Status__c transition to 'Adjudicated'. Flow uses a Decision element to branch by LOB__c and COB presence. Payment__c record creation handled via Flow Create Records element. Platform Event (Payment_Initiated__e) published via Flow Publish Message element to decouple downstream ERA and EOB generation processes. An Apex invocable method handles business-day calculation for Payment__c.Due_Date__c. Error handling branch creates ClaimsProcessing_Error__c and invokes a custom notification via Bell Notification. All logic covered by Apex test class with 90%+ coverage targeting the invocable method.

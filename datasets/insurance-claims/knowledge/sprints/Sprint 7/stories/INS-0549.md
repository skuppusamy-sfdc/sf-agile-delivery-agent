## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page that displays the provider's active Provider_Network_Agreement__c, linked Fee_Schedule__c details, and a rolling 90-day Payment__c summary so that I can quickly assess provider standing, reimbursement rates, and payment history without navigating multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page
When the Provider 360 FlexCard renders
Then the card must display: Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Credentialing_Status__c, and the count of active Credentialing_Case__c records where Credentialing_Case__c.Status__c IN ('In Review', 'Pending Documentation')

Given the provider has one or more Provider_Network_Agreement__c records
When the FlexCard PNA section renders
Then only PNA records where Provider_Network_Agreement__c.Status__c = 'Active' and Provider_Network_Agreement__c.Effective_Date__c <= TODAY() and Provider_Network_Agreement__c.Termination_Date__c >= TODAY() (or is null) are displayed, showing PNA Name, Network__c, and Contract_Type__c

Given an active PNA exists with a linked Fee_Schedule__c
When the Fee Schedule section renders
Then Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.LOB__c, and the count of Fee_Schedule__c child line items (Fee_Schedule_Line__c) are displayed

Given Payment__c records linked to the provider exist
When the Payment Summary section renders
Then the card must display: total Payment__c.Amount__c sum for the rolling 90 days (Payment__c.Disbursed_Date__c >= LAST_90_DAYS), count of payments by Payment__c.Status__c ('Disbursed', 'Pending Disbursement', 'Failed'), and the most recent Payment__c.Disbursed_Date__c

-Given the provider is flagged under SIU investigation (Provider__c.SIU_Flag__c = true), display a red warning banner on the FlexCard- (deferred to INS-0561 per SIU team request)

Given the user's Profile does not include the 'Provider_Financial_Data_Access' Permission Set
When the FlexCard renders
Then the Payment Summary section must be hidden and replaced with 'Access Restricted' messaging

## Solution

OmniStudio FlexCard (Provider_360_Card) with child cards for PNA, Fee Schedule, and Payment sections. DataRaptors (DR_Get_Provider_PNA, DR_Get_Fee_Schedule, DR_Get_Payment_Summary) retrieve data via SOQL with appropriate WHERE clauses. Payment summary aggregation handled by an Apex REST action (ProviderPaymentSummaryAction.cls) to support the rolling 90-day aggregate calculation. Conditional visibility on Payment section driven by a Custom Permission (Provider_Financial_Data_Access) checked via $Permission merge field in FlexCard state. FlexCard deployed to Provider__c Lightning Record Page via App Builder.

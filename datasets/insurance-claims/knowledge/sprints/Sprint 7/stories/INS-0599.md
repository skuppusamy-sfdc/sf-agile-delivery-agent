## Description

As a Claims Operations Manager, I need an Agentforce AI assistant embedded on the Claims workspace that can answer natural language questions about payment status, disbursement timelines, and EOB availability for a given claim, so that I can resolve policyholder inquiries faster without manually navigating multiple related records.

Example queries the assistant must handle:
- 'What is the payment status for claim CLM-0042891?'
- 'Has the EOB been sent to the member for this claim?'
- 'Why was payment voided on claim CLM-0039210?'
- 'What is the total plan-paid amount across all lines for this claim?'

## Acceptance Criteria

Given a Claims Operations Manager opens the Agentforce panel on a Claim__c record page,
When they type or speak 'What is the payment status for this claim?',
Then the Agentforce agent responds with the current Payment__c.Status__c value(s), Payment__c.Payment_Date__c, Payment__c.Payment_Amount__c, and Payment__c.Reference_Number__c for all linked payments.

Given the user asks about EOB delivery status,
When the agent processes the query,
Then it retrieves EOB__c.Status__c, EOB__c.Distribution_Timestamp__c, and EOB__c.Portal_Delivery__c or EOB__c.Mail_Delivery_Requested__c and presents a human-readable summary.

Given the user asks 'Why was payment voided?',
When a voided Payment__c exists (Payment__c.Status__c = 'Voided'),
Then the agent retrieves Payment__c.Void_Reason__c and Payment__c.Void_Date__c from the most recent voided payment and summarizes the reason in plain language.

Given a query that cannot be resolved from Claim__c, Payment__c, or EOB__c data,
When the agent cannot find a confident answer,
Then the agent responds with 'I was unable to find that information. Would you like me to create a follow-up task?' and offers a one-click task creation action.

Given the Agentforce assistant is active,
When any query is processed,
Then all interactions are logged to an Agent_Interaction__c custom object with fields: Agent_Interaction__c.Query_Text__c, Agent_Interaction__c.Response_Text__c, Agent_Interaction__c.Claim__c, Agent_Interaction__c.Timestamp__c for audit and model improvement purposes.

## Solution

Configure an Agentforce Agent (Einstein Copilot) with a custom Agent Topic scoped to Claims Payment & Disbursement. Define Agent Actions backed by Apex invocable methods: (1) GetClaimPaymentStatus — queries Payment__c by Claim__c.Id; (2) GetEOBStatus — queries Explanation_of_Benefits__c by Claim__c.Id; (3) GetVoidReason — retrieves voided Payment__c details. Each action is registered as an Agentforce Action with natural language descriptions to guide the LLM's function-calling. A custom Agent_Interaction__c object with a Record-Triggered Flow on agent response events captures the interaction log. The agent is embedded on the Claim__c Lightning Record Page via the Einstein Copilot LWC component. Prompt templates are stored in Salesforce Prompt Builder with merge fields referencing Claim__c.Name and Claim__c.Status__c for context grounding. Permission Set Group 'Claims_Ops_Standard' includes Agentforce User permission.

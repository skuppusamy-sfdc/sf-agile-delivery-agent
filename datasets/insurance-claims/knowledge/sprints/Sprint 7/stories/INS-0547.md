## Description

As a Claims Operations Manager, I need the system to generate an 835 EDI-formatted ERA file for each batch of disbursed payments and transmit it via SFTP to the relevant provider billing system so that providers receive timely electronic remittance data aligned with their Provider_Network_Agreement__c (PNA) transmission preferences, reducing manual reconciliation effort.

## Acceptance Criteria

Given one or more Payment__c records with Status__c = 'Disbursed' exist for a given Provider__c within the current processing date
When the nightly Scheduled Apex job (ERABatchProcessor.cls) executes at 23:00 UTC
Then all eligible payments are grouped by Provider__c.NPI__c and a single ERA batch record (ERA_Batch__c) is created per provider with ERA_Batch__c.Provider__c, ERA_Batch__c.Total_Payment_Amount__c, ERA_Batch__c.Payment_Count__c, and ERA_Batch__c.Batch_Date__c = TODAY()

Given an ERA_Batch__c record is created
When the 835 EDI file is generated by the ERAFileGenerator Apex class
Then the file must conform to the ANSI X12 835 v5010 standard, include ISA/GS envelope segments with Provider__c.TIN__c in the payer ID field, and contain one CLP segment per Payment__c and one SVC segment per associated Claim_Line__c.CPT_Code__c

Given the ERA file is generated
When the SFTP transmission is initiated
Then the file must be delivered to the SFTP endpoint defined in Provider_Network_Agreement__c.ERA_SFTP_Endpoint__c using credentials stored in the 'Provider_SFTP_[NPI]' Named Credential, and ERA_Batch__c.Transmission_Status__c must be set to 'Transmitted' with ERA_Batch__c.Transmitted_Datetime__c stamped

Given the SFTP transmission fails
When the callout returns a non-200 response or times out
Then ERA_Batch__c.Transmission_Status__c = 'Failed', ERA_Batch__c.Failure_Reason__c is populated, and a Task is created on the Provider__c record assigned to the Provider Relations Specialist queue with Subject = 'ERA Transmission Failure - [NPI] - [BatchDate]'

Given Provider_Network_Agreement__c.ERA_Delivery_Method__c = 'Portal' instead of 'SFTP'
When the batch processor evaluates delivery method
Then the ERA file must be attached to the ERA_Batch__c record as a ContentDocument and shared to the provider's Experience Cloud portal rather than transmitted via SFTP

## Solution

Scheduled Apex (ERABatchProcessor implements Schedulable, Database.Batchable) queries disbursed Payment__c records. ERAFileGenerator.cls constructs the 835 EDI string using a StringBuilder pattern driven by Custom Metadata (EDI_Segment_Config__mdt). SFTP transmission uses an Apex HTTP callout to an internal MuleSoft middleware layer (Named Credential: MuleSoft_SFTP_Gateway) which handles actual SFTP protocol. Delivery method branching driven by PNA field. Platform Event (ERA_Transmission_Complete__e) published post-transmission for downstream audit logging.

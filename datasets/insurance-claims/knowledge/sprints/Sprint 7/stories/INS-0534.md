## Description

**Bug Report**

**Summary:** Payment__c.Status__c is not updating to 'Disbursed' after the EFT payment processor sends a successful disbursement confirmation webhook to the Salesforce REST endpoint.

**Steps to Reproduce:**
1. Create a Payment__c record with Payment__c.Status__c = 'Approved_for_Disbursement' and Payment__c.Payment_Method__c = 'EFT'.
2. Trigger the EFT batch job (EFT_DisbursementBatch) manually or wait for scheduled run.
3. Observe the outbound EFT file is sent successfully (confirmed via SFTP logs).
4. EFT processor sends webhook POST to /services/apexrest/payment/eft/confirm with payload: { 'paymentId': '[Payment__c.Id]', 'status': 'SUCCESS', 'transactionRef': 'TXN-XXXXX' }.
5. Observe Payment__c.Status__c — it remains 'Approved_for_Disbursement' instead of updating to 'Disbursed'.
6. Payment__c.EFT_Transaction_Reference__c also remains NULL.

**Expected Behavior:** Payment__c.Status__c = 'Disbursed', Payment__c.EFT_Transaction_Reference__c = 'TXN-XXXXX', Payment__c.Disbursement_Date__c = webhook receipt date.

**Actual Behavior:** Payment__c.Status__c remains 'Approved_for_Disbursement'. No field updates occur. No error is logged in the debug logs.

**Root Cause (Identified):** The EFT confirmation REST resource (EFT_ConfirmationResource.cls) performs a SOQL query using Payment__c.External_Payment_Id__c to locate the record, but the EFT batch job was updated in INS-0529 to populate Payment__c.Id (Salesforce record ID) in the outbound file instead of Payment__c.External_Payment_Id__c. The webhook payload 'paymentId' field now contains the Salesforce record ID, but the REST resource still queries by External_Payment_Id__c, returning zero results and silently exiting.

**Environment:** Partial Sandbox (PS1)
**Severity:** High — all EFT payments post-INS-0529 deployment are not being marked Disbursed, blocking EOB generation (INS-0530).

## Acceptance Criteria

As per Description. Fix verified when Payment__c.Status__c = 'Disbursed' and Payment__c.EFT_Transaction_Reference__c is populated after webhook receipt in PS1 sandbox regression test.

## Solution

Update EFT_ConfirmationResource.cls to query Payment__c by Id (Salesforce record ID) when the inbound paymentId value matches the 18-character Salesforce ID format (use regex check). Retain External_Payment_Id__c fallback query for backward compatibility with legacy payment records. Add null-check guard before DML update. Add error logging via a custom EFT_Error_Log__c insert on exception. Deploy fix to PS1 and run regression suite covering INS-0529 and INS-0530 dependent flows.

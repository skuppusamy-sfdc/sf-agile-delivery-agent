## Description

As a Policyholder (Member), I need to view my claims payment history and download EOB documents directly from the Member Portal so that I have transparent, self-service access to my benefits information without needing to call the member services helpline. This story extends the Policyholder 360 record page introduced in INS-0548.

## Acceptance Criteria

Given an authenticated Policyholder accesses the Member Portal Experience Cloud site
When they navigate to the 'Claims & Payments' tab
Then a FlexCard displays a list of their Claim__c records filtered by Policy__c.Policyholder__c = current user's Contact Id, showing: Claim__c.Claim_Number__c, Claim__c.Service_Date__c, Claim__c.Status__c, and associated Payment__c.Payment_Amount__c

Given the Policyholder selects a specific Claim__c record
When the claim detail view loads
Then a nested FlexCard displays all Claim_Line__c child records with: CPT_Code__c, Service_Description__c, Billed_Amount__c, Allowed_Amount__c, Member_Responsibility__c, and Line_Status__c

Given a Claim__c has one or more associated EOB__c records with Status__c = 'Delivered'
When the Policyholder views the claim detail
Then a 'Download EOB' button is displayed for each EOB__c record, triggering a secure download of the ContentDocument PDF via a Guest User sharing rule scoped to the authenticated member's Policy__c

Given the Policyholder portal page loads
When there are no Claim__c records associated with the Policy__c
Then an empty state message displays: 'No claims found for your policy. If you believe this is an error, please contact Member Services at 1-800-ACME-INS'

-Given the portal must display TPA-administered claims separately under a 'TPA Claims' tab- (descoped; TPA portal is a separate Phase 2 workstream)

Given a Policyholder attempts to access another member's Claim__c record via direct URL manipulation
When the page loads
Then the record is not displayed and an 'Access Denied' message is shown; Salesforce sharing rules and Guest User profile restrictions enforce data isolation at the object level

## Solution

Build OmniStudio FlexCards (ClaimSummaryCard, ClaimDetailCard, EOBDownloadCard) consuming DataRaptors that query Claim__c, Claim_Line__c, Payment__c, and EOB__c filtered by the authenticated user's Contact Id via $User.ContactId. Experience Cloud site uses a Customer Community Plus license profile. Sharing Rules on Claim__c grant Read access where Claim__c.Policyholder__c = current user Contact. EOB download uses a Visualforce PDF controller or ContentDistribution link scoped to authenticated users only. Guest User profile explicitly denies access to Claim__c and EOB__c objects. OmniScript wraps the full Claims & Payments tab layout, deployed to the Experience Cloud page via the OmniStudio component in Experience Builder.

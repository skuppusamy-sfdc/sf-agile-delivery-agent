## Description

As a Provider Relations Specialist, I need a consolidated Payment and EOB history panel on the Provider__c (Person Account) record page so that I can quickly review recent disbursements, outstanding payments, and EOB delivery status during provider inquiries without navigating to multiple related lists or running reports.

## Acceptance Criteria

Given a Provider__c record is opened by a user with the 'Provider_Relations' Permission Set Group
When the Provider 360 FlexCard loads
Then the Payment History panel displays the 10 most recent Payment__c records related to the provider, sorted by Payment__c.Disbursement_Date__c descending, showing: Payment__c.Name, Payment__c.Amount__c, Payment__c.Status__c, Payment__c.ERA_File_Reference__c, and Payment__c.Disbursement_Date__c

Given the FlexCard Payment History panel is rendered
When Payment__c.Status__c = 'Pending'
Then the row is highlighted in amber and a 'Follow Up' action button is displayed inline

Given the FlexCard loads
When the EOB History tab is selected
Then all Explanation_of_Benefits__c records linked to the provider's claims are displayed showing EOB.Name, EOB.Status__c, EOB.Portal_Published__c, and a 'Download PDF' action that retrieves the linked ContentVersion

Given a Provider__c has no Payment__c records in the last 12 months
When the FlexCard loads
Then an empty state message reads: 'No payment activity found in the last 12 months' and a 'Submit Inquiry' action is available

-Given Provider__c.Network_Status__c = 'Terminated', payment history is hidden from Provider Relations Specialist view-

Given the FlexCard is loaded on a mobile device via Salesforce Mobile App
When the screen width is below 480px
Then the panel collapses to a summary card showing total payments YTD and most recent EOB status only

## Solution

OmniStudio FlexCard built on the Provider__c object using two child DataRaptor Extract actions: (1) DR_GetProviderPayments retrieving Payment__c records filtered by Provider__c = :recordId and Disbursement_Date__c >= LAST_N_MONTHS:12, limited to 10 records; (2) DR_GetProviderEOBs retrieving EOB records via Claim__c relationship. FlexCard uses conditional state styling for amber highlighting on Pending payments. 'Download PDF' action triggers a DataRaptor Extract to retrieve ContentVersion download URL. 'Follow Up' action launches a sub-FlexCard with a quick action to create a Case. FlexCard is surfaced via Lightning App Builder on the Provider__c record page, restricted by Permission Set Group 'Provider_Relations'.

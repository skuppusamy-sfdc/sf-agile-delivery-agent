## Description

As a Claims Operations Manager, I need an Agentforce AI action embedded on the Claim__c record page that generates a plain-language summary of the adjudication rationale, payment breakdown, and any denial justifications, so that claims staff can quickly understand complex adjudication decisions and communicate outcomes to members and providers without manually interpreting raw claim data.

The AI summary must reference Claim_Line__c adjudication details, Fee_Schedule__c applied rates, PNA terms, and any SIU flags. The generated summary should be storable as a Claim__c internal note for audit trail purposes.

## Acceptance Criteria

Given a Claim__c record with Status__c = 'Adjudicated'
When a claims staff member clicks the 'Generate AI Summary' Agentforce action
Then an Agentforce prompt template (PT_ClaimAdjudicationSummary) is invoked with Claim__c.Id as context and a summary is rendered in the action panel within 5 seconds

Given the Agentforce summary is generated
When the summary is displayed
Then it includes: total billed vs allowed amounts, per-line CPT_Code__c adjudication outcomes, applied Fee_Schedule__c.Schedule_Name__c, denial reason codes with plain-language explanations, and member responsibility breakdown

Given Claim__c.SIU_Referral__c = true
When the AI summary is generated
Then the summary includes a prominent SIU flag notice and omits payment amount details (to avoid tipping off the member prematurely)

Given the user clicks 'Save Summary as Note'
When the action is confirmed
Then a ContentNote is created linked to the Claim__c record with Note_Type__c = 'AI Adjudication Summary' and Created_By__c stamped with the running user

Given the Agentforce action fails due to LLM timeout
When the error is caught
Then a user-friendly error toast is displayed: 'AI summary unavailable. Please try again or contact your system administrator.'

## Solution

Configure an Agentforce Agent Action using the Einstein Generative AI framework. Build a Prompt Template (PT_ClaimAdjudicationSummary) using Flex Template type with merge fields pulling from Claim__c, Claim_Line__c, Fee_Schedule__c, and PNA records via a custom Apex DataProvider class (ClaimSummaryDataProvider). The 'Save Summary as Note' action is implemented as an Agentforce Action invoking an Apex invocable method (ClaimNoteService.saveAISummary()). SIU masking logic is enforced in the DataProvider class before data is passed to the prompt. Permission to invoke the Agentforce action is controlled via Permission Set Group (PSG_Claims_AI_Users). All AI prompt invocations are logged in AI_Interaction_Log__c for compliance auditing.

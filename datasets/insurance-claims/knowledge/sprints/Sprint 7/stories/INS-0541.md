## Description

As a Claims Operations Manager, I need Payment__c records to be automatically routed to either ACH direct deposit or physical check disbursement based on the provider's configured payment preferences on Provider__c so that payment disbursement is accurate, timely, and eliminates manual routing decisions. This story handles the disbursement routing previously noted as deprecated in INS-0537.

## Acceptance Criteria

Given a Payment__c record where Payment_Status__c = 'Pending' and Payment_Method__c = 'ACH'
When the disbursement routing flow executes
Then the system validates that Provider__c.ACH_Routing_Number__c and Provider__c.ACH_Account_Number__c are both populated and Provider__c.ACH_Verified__c = true before proceeding

Given ACH details are valid
When the ACH routing flow proceeds
Then a callout is made to the external ACH payment processor API (endpoint stored in Named Credential 'ACH_Processor') with Payment_Amount__c, Provider__c.TIN__c, Provider__c.ACH_Routing_Number__c, and Provider__c.ACH_Account_Number__c, and Payment__c.ACH_Transaction_Reference__c is populated upon success

Given Payment_Method__c = 'Check'
When the disbursement routing flow executes
Then a Check_Print_Request__c record is created linked to Payment__c with Check_Payee__c = Provider__c.Name, Check_Mailing_Address__c = Provider__c.Mailing_Address__c, and Check_Amount__c = Payment__c.Payment_Amount__c, and Payment__c.Payment_Status__c = 'Check_Queued'

Given an ACH callout failure (HTTP 4xx or 5xx response)
When the error is caught
Then Payment__c.Payment_Status__c = 'Disbursement_Failed', Payment__c.Disbursement_Error_Message__c is populated with the error response body, and a Task is created assigned to the Claims Operations queue with Subject = 'ACH Disbursement Failed — ' + Payment__c.Name

Given Payment_Method__c is null or unrecognized
When routing logic executes
Then Payment__c is flagged with Payment_Status__c = 'Requires_Manual_Review' and routed to the Claims Operations Manager queue

Given a TPA-administered claim where Claim__c.TPA_Administered__c = true
When disbursement routing executes
Then payment is directed to the TPA's ACH details stored on Group_Service_Agreement__c.TPA_ACH_Routing__c rather than the provider's ACH details

## Solution

Implement a scheduled Flow (runs every 15 minutes) querying Payment__c where Payment_Status__c = 'Pending'. Flow branches on Payment_Method__c. ACH path invokes Apex action (ACHDisbursementService.cls) which performs an async callout using Continuation or Queueable Apex to the ACH processor Named Credential endpoint. Check path invokes CheckPrintRequestFactory.cls to create Check_Print_Request__c records. TPA override logic in ACHDisbursementService checks Claim__c.TPA_Administered__c and queries GSA for TPA banking details. All callout responses handled with try/catch and logged to a custom Payment_Audit_Log__c object. Permission Set 'Payment_Disbursement_Admin' controls access to override fields.

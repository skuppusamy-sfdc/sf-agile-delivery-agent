## Description

As a Provider Relations Specialist, I need to capture and validate provider payment preferences (EFT banking details, check mailing address, TIN) directly on the Provider__c (Person Account) record so that the Payment Engine can accurately route disbursements and reduce returned payments and EFT rejections.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record
When the 'Payment Preferences' section is visible on the Provider_360 Lightning page
Then the following fields are displayed: Preferred_Payment_Method__c (picklist: EFT, Check), Bank_Account_Number__c (encrypted), Bank_Routing_Number__c (encrypted), Bank_Account_Type__c (picklist: Checking, Savings), Check_Mailing_Address__c (text area), TIN__c, and EFT_Effective_Date__c

Given Preferred_Payment_Method__c = 'EFT'
When the record is saved
Then Bank_Account_Number__c, Bank_Routing_Number__c, and Bank_Account_Type__c are required fields; if any are blank, a validation error displays: 'EFT payment method requires complete banking information'

Given Bank_Routing_Number__c is entered
When the field is validated on save
Then the routing number must be exactly 9 numeric digits; if not, display: 'Routing number must be a 9-digit numeric value'

Given TIN__c is entered or updated
When the record is saved
Then TIN__c must match the format XX-XXXXXXX (EIN) or XXX-XX-XXXX (SSN); a validation rule enforces this and displays 'Invalid TIN format'

Given Bank_Account_Number__c and Bank_Routing_Number__c are populated
When the record is saved
Then both fields are stored using Salesforce Shield Platform Encryption (deterministic encryption policy) and are masked in the UI to show only the last 4 digits

Given a Provider__c record where Preferred_Payment_Method__c changes from 'EFT' to 'Check'
When the record is saved
Then a Provider_Change_Notification__c (PCN) record is auto-created with PCN__c.Change_Type__c = 'Payment Method Change' and PCN__c.Effective_Date__c = TODAY()

## Solution

Add custom fields to Provider__c (Person Account) via Field Manager. Implement Salesforce Shield Platform Encryption on Bank_Account_Number__c and Bank_Routing_Number__c using deterministic encryption. Build validation rules for EFT required fields, routing number format (REGEX), and TIN format. Record-Triggered Flow on Provider__c fires on Preferred_Payment_Method__c change to auto-create PCN__c. Lightning page updated with a new 'Payment Preferences' collapsible section using Lightning App Builder. Field-level security restricts Bank_Account_Number__c and Bank_Routing_Number__c to 'Provider_Payment_Admin' Permission Set only.

## Description

As a Claims Operations Manager, I need the system to generate an ANSI X12 835 ERA file for each approved Payment__c batch and transmit it to the associated Provider or Facility so that providers receive standardized electronic remittance data aligned with their TPA or clearinghouse configuration, reducing manual reconciliation effort.

## Acceptance Criteria

Given one or more Payment__c records with Status__c = 'Approved' and Payment__c.Payment_Method__c = 'EFT'
When the nightly ERA batch job runs (scheduled at 11:00 PM ET)
Then all eligible Payment__c records are grouped by Provider__c.NPI__c and TIN__c and a single ERA_File__c record is created per provider group with ERA_File__c.Status__c = 'Generated'

Given an ERA_File__c record is created
When the X12 835 file is built
Then the file must include: ISA/GS envelope segments, CLP loop per Claim__c, SVC loop per Claim_Line__c with CPT_Code__c and Allowed_Amount__c, and PLB segment for any payment adjustments

Given ERA_File__c.Status__c = 'Generated'
When Provider__c.ERA_Delivery_Method__c = 'SFTP'
Then the file is transmitted to the provider's configured SFTP endpoint stored in Provider__c.SFTP_Endpoint__c and ERA_File__c.Status__c updated to 'Transmitted'

Given ERA_File__c.Status__c = 'Generated'
When Provider__c.ERA_Delivery_Method__c = 'Clearinghouse'
Then the file is routed via the registered clearinghouse API and ERA_File__c.Clearinghouse_Tracking_Id__c is populated upon acknowledgment

Given an ERA transmission failure
When the outbound call returns a non-200 HTTP response or SFTP error
Then ERA_File__c.Status__c = 'Transmission Failed', ERA_File__c.Failure_Reason__c is populated, and a retry is scheduled for the following business day

Given ERA_File__c.Status__c = 'Transmitted'
When the record is updated
Then each associated Payment__c.ERA_Transmitted__c = true and Payment__c.ERA_File__c lookup is populated

## Solution

Apex Batch class (ERA_GenerationBatch) executes nightly via Scheduled Apex. Batch queries Payment__c grouped by NPI/TIN, constructs X12 835 string using a dedicated ERA_835Builder Apex utility class, and creates ERA_File__c records with the file stored as a ContentVersion attachment. An Apex Callout class (ERA_TransmissionService) handles SFTP via Named Credential and clearinghouse REST API via a separate Named Credential. Platform Events (ERA_Transmission__e) log transmission outcomes. Custom Metadata (ERA_Config__mdt) stores clearinghouse endpoint URLs and SFTP paths per provider segment. Retry logic uses a Queueable Apex chain.

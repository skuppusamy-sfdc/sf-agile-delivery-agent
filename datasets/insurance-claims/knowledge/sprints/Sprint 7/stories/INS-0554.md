## Description

As a Claims Operations Manager, I need Explanation_of_Benefits__c (EOB) documents to be automatically generated as PDFs and delivered to the policyholder via DocuSign upon claim payment disbursement so that regulatory EOB delivery requirements are met within the mandated 30-day window. The EOB must include itemized Claim_Line__c detail, allowed amounts, member responsibility, and denial reason codes where applicable.

## Acceptance Criteria

Given a Payment__c record where Payment_Status__c transitions to 'Disbursed'
When the Record-Triggered Flow on Payment__c fires
Then an Explanation_of_Benefits__c record is created with:
  - EOB__c.Claim__c = Payment__c.Claim__c
  - EOB__c.Payment__c = Payment__c.Id
  - EOB__c.Member__c = Claim__c.Member__c
  - EOB__c.EOB_Status__c = 'Generated'
  - EOB__c.Generation_Date__c = TODAY()
  - EOB__c.Delivery_Deadline__c = TODAY() + 30

Given the EOB__c record is created
When the Conga Composer template 'EOB_Standard_v3' is invoked
Then a PDF is generated containing all related Claim_Line__c records with fields: CPT_Code__c, ICD10_Code__c, Billed_Amount__c, Allowed_Amount__c, Plan_Paid__c, Member_Responsibility__c, and Denial_Reason_Code__c

Given the PDF is generated successfully
When DocuSign envelope is dispatched
Then the envelope is sent to Policy__c.Primary_Member_Email__c with DocuSign template 'EOB_Delivery_No_Signature_Required'
And EOB__c.Delivery_Method__c = 'Email - DocuSign'
And EOB__c.EOB_Status__c = 'Sent'

Given a member with Member_Portal_Enrolled__c = true
When EOB is generated
Then EOB__c record is also shared to the member's Experience Cloud portal via Sharing Rule on EOB__c object (Member__c = $User.ContactId)

Given DocuSign delivery fails with a bounce or error
When the DocuSign callback webhook is received
Then EOB__c.Delivery_Failed__c = true, EOB__c.EOB_Status__c = 'Delivery Failed', and a Task is created assigned to the Claims Operations Manager queue with Subject = 'EOB Delivery Failure - Manual Action Required'

-EOB to be sent via postal mail as fallback using vendor API- (deprecated: postal mail integration descoped to Phase 2 per stakeholder decision 2024-11-15)

Given EOB__c.Delivery_Deadline__c is within 5 days and EOB_Status__c != 'Sent'
When a scheduled daily Flow runs
Then a High priority Case is created linked to EOB__c with Subject = 'EOB Delivery SLA At Risk'

## Solution

Record-Triggered Flow on Payment__c (After Save) triggers EOB__c record creation. Apex invocable action wraps Conga Composer REST API call to generate PDF using template 'EOB_Standard_v3', with a related list query pulling all Claim_Line__c children. DocuSign for Salesforce managed package used for envelope dispatch; envelope status callbacks update EOB__c via a custom DocuSign Connect endpoint (Apex REST resource). Experience Cloud sharing handled via Criteria-Based Sharing Rule on EOB__c. Scheduled Flow runs nightly for SLA breach detection. Conga template managed in Salesforce Files linked to ContentVersion. Integration errors logged to Integration_Error_Log__c.

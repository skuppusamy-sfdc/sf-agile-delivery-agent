## Description

As a Claims Operations Manager, I need the Agentforce AI assistant to proactively surface next-best action recommendations for Payment__c records that have exceeded the 5-business-day disbursement SLA so that my team can prioritize resolution actions, reduce payment aging, and maintain compliance with state prompt-pay regulations without manually reviewing aged payment queues.

## Acceptance Criteria

Given a Payment__c record where Payment__c.Status__c = 'Pending' and Payment__c.Due_Date__c < TODAY()
When the Agentforce Claims AI topic evaluates the record
Then the AI surfaces a recommendation panel on the Payment__c record page with at minimum one next-best action from: ['Escalate to TPA', 'Request Missing Documentation', 'Initiate Manual Review', 'Contact Provider']

Given the Agentforce recommendation is displayed
When the Claims Operations Manager selects 'Escalate to TPA'
Then a Task is created with Subject = 'TPA Escalation - [Payment Name]', assigned to the TPA Review queue, with ActivityDate = TODAY() + 1, and Payment__c.Escalated_to_TPA__c = TRUE

Given a Payment__c has been stalled for more than 10 business days
When Agentforce evaluates the record
Then the recommendation confidence score displayed is >= 85% and the SIU Referral option is included in the action list if Claim__c.SIU_Flag__c = TRUE

Given the AI recommendation is acted upon
When the action is completed
Then an Agentforce_Interaction__c record is logged with fields: Recommended_Action__c, Action_Taken__c, User__c, Timestamp__c, and Payment__c lookup populated

Given a user without the 'Agentforce_Claims_User' Permission Set
When they open a Payment__c record
Then the Agentforce recommendation panel is not visible

## Solution

Configure Agentforce Agent with a custom Topic ('Stalled Payment Resolution') and associated Actions mapped to Apex invocable methods. The Topic prompt template is configured to query Payment__c.Due_Date__c, Claim__c.SIU_Flag__c, and Claim__c.LOB__c to contextualize recommendations. Next-best action options are defined as Agent Actions invoking Record-Triggered Flows or Apex methods (e.g., TPA escalation flow reused from INS-0577). Agentforce_Interaction__c custom object created to log all AI interactions with a lookup to Payment__c. Einstein Next Best Action strategy configured as a fallback for non-AI-licensed orgs. LWC component surfaces the recommendation panel on the Payment__c Lightning record page, gated by Permission Set 'Agentforce_Claims_User'.

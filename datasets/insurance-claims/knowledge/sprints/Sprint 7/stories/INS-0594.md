## Description

As a Claims Operations Manager, I need the system to automatically generate an ANSI X12 835 ERA file for each finalized Payment__c batch and transmit it to the receiving provider or clearinghouse, so that providers can reconcile payments electronically without manual remittance notices.

ERA files must be grouped by Provider__c (NPI) and TIN, include all Claim_Line__c adjustments and remark codes, and be transmitted via the provider's configured EDI endpoint stored on the Provider_Network_Agreement__c record.

## Acceptance Criteria

Given one or more Payment__c records with Payment__c.Status__c = 'Issued' and Payment__c.Payment_Batch__c populated,
When the nightly scheduled Apex batch job (ERAGenerationBatch) executes,
Then an ERA_File__c record is created per unique Provider__c + TIN combination with ERA_File__c.Format__c = 'X12-835', ERA_File__c.Batch_ID__c populated, and ERA_File__c.Status__c = 'Generated'.

Given the ERA_File__c record is in 'Generated' status,
When the EDI transmission service is invoked,
Then the file is sent to the endpoint defined in Provider_Network_Agreement__c.EDI_Endpoint_URL__c and ERA_File__c.Transmission_Status__c is set to 'Sent' with ERA_File__c.Transmitted_At__c stamped.

Given a transmission failure (HTTP response code != 200 or timeout),
When the retry logic executes (max 3 attempts with exponential backoff),
Then after 3 failures ERA_File__c.Transmission_Status__c = 'Failed' and a Case is auto-created with Subject = 'ERA Transmission Failure - [Provider NPI]' assigned to the Provider Relations queue.

Given ERA_File__c.Transmission_Status__c = 'Sent',
When the provider's clearinghouse returns a 999 acknowledgement,
Then ERA_File__c.Acknowledgement_Status__c = 'Accepted' and ERA_File__c.Acknowledged_At__c is populated.

Given a provider whose PNA has Provider_Network_Agreement__c.ERA_Opted_Out__c = true,
When the ERA batch processes that provider's payments,
Then no ERA file is generated and Payment__c.ERA_Suppressed__c = true is stamped on each associated payment record.

## Solution

Scheduled Apex Batch class (ERAGenerationBatch, implements Database.Batchable, Database.Schedulable) runs nightly at 02:00 AM local time. Batch scope queries Payment__c records joined to Claim__c and Claim_Line__c. An Apex utility (X12835Builder) constructs the ISA/GS/ST envelope and CLP/SVC segments per HIPAA 5010 spec. The generated file content is stored as a ContentVersion attached to ERA_File__c. Transmission is handled via a named credential (NC_EDI_Clearinghouse) in an Apex callout class (EDITransmissionService) with retry logic using a custom metadata type (ERA_Retry_Config__mdt) for configurable backoff intervals. Inbound 999 acknowledgements are received via a REST endpoint (ERACallbackController) exposed through Salesforce Experience Cloud. A Custom Label stores the max retry count for environment-specific overrides.

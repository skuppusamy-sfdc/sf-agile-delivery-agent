## Description

As a Claims Operations Manager, I need the system to generate HIPAA-compliant 835 ERA transaction files for all EFT-based provider payments so that providers can automatically reconcile payments against their submitted claims within their practice management systems. ERA files must be grouped by Provider NPI and TIN, covering all Payment__c records in 'Disbursed' status within the processing batch window.

## Acceptance Criteria

Given one or more Payment__c records with Payment_Status__c = 'Disbursed', Payment_Method__c = 'EFT', and ERA_Generated__c = false
When the nightly ERA batch job executes
Then an ERA file (HIPAA 835 format) is generated per unique combination of Provider__c.NPI__c and Provider__c.TIN__c

Given the 835 ERA file is generated
When file content is validated
Then it must contain: ISA/GS envelope segments, BPR segment with Payment_Amount__c, NM1 segments for payer and payee (using NPI__c and TIN__c), CLP segments per Claim__c, and SVC segments per Claim_Line__c with CPT_Code__c and Allowed_Amount__c

Given ERA file generation completes successfully
When the file is stored
Then a ContentVersion record is created linked to the Provider__c (Person Account) record
And Payment__c.ERA_Generated__c = true
And Payment__c.ERA_File_Date__c = TODAY()

Given the ERA file is generated
When Provider__c.ERA_Delivery_Preference__c = 'SFTP'
Then the file is transmitted to the provider's registered SFTP endpoint stored in Provider__c.SFTP_Endpoint__c
And ERA_Transmission_Status__c = 'Transmitted'

Given an SFTP transmission failure
When the file cannot be delivered
Then ERA_Transmission_Status__c = 'Transmission Failed' and a High priority Case is created in the Provider Relations queue

Given a provider with ERA_Delivery_Preference__c = 'Portal'
When the ERA file is generated
Then the ContentVersion is shared to the Provider Portal (Experience Cloud) via manual share on ContentDocument

## Solution

Apex Batch class (ERA_GenerationBatch) queries Payment__c records meeting criteria, groups by Provider NPI/TIN using a Map, and invokes an Apex service class (ERA_835Builder) that constructs the 835 flat file string per HIPAA X12 835 spec. File stored as ContentVersion via Apex ContentDocumentLink insert. SFTP transmission handled via an Apex callout to internal MuleSoft integration layer (named credential 'SFTP_Gateway'). Platform Event (ERA_Transmission__e) published post-transmission for audit logging. Batch scheduled via Schedulable Apex to run at 02:00 AM nightly. Custom Metadata Type (ERA_Config__mdt) stores payer ISA/GS header values per LOB.

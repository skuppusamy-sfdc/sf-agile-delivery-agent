## Description

As a Claims Operations Manager, I need all Payment__c records exceeding a configurable dollar threshold to be routed through a multi-step approval process before disbursement is authorized so that financial controls and audit requirements are met for high-value claim payments across all LOBs. The threshold value must be stored in a Custom Metadata Type to allow configuration without code deployment.

## Acceptance Criteria

Given a Payment__c record is created with Payment__c.Payment_Amount__c greater than the value stored in Claims_Payment_Config__mdt.High_Value_Threshold__c
When the Record-Triggered Flow evaluates the threshold condition
Then the Payment__c record is submitted to the 'High_Value_Payment_Approval' Approval Process and Payment__c.Status__c is set to 'Pending Approval'

Given a Payment__c is in 'Pending Approval' status
When the Step 1 approver (Claims Operations Manager role) approves
Then the record advances to Step 2 (Underwriting Analyst role) and an email notification is sent to the Step 2 approver queue

Given both approval steps are completed with 'Approved' decisions
When the final approval action fires
Then Payment__c.Status__c = 'Approved', Payment__c.Approval_Completed_Date__c = TODAY(), and the ClaimPaymentInitiated__e Platform Event is published

Given any approver rejects the Payment__c at any step
When the rejection action fires
Then Payment__c.Status__c = 'Rejected', Payment__c.Rejection_Reason__c is populated from the approver's comment, and the originating Claims processor is notified via Salesforce in-app notification

Given Claims_Payment_Config__mdt.High_Value_Threshold__c is updated by a System Administrator
When the next Payment__c is evaluated
Then the new threshold value is applied without requiring any code or Flow redeployment

Given a Payment__c below the threshold
When the Flow evaluates
Then no approval process is triggered and Payment__c.Status__c remains 'Pending' proceeding to standard disbursement

## Solution

Custom Metadata Type Claims_Payment_Config__mdt stores the High_Value_Threshold__c field. Record-Triggered Flow on Payment__c (after-insert) reads the CMT value via a Get Records element and uses a Decision element to branch high-value payments into a Submit for Approval action element. Salesforce standard Approval Process 'High_Value_Payment_Approval' configured with two sequential approver steps mapped to Role hierarchy. Final approval/rejection Apex approval actions update Payment__c.Status__c and publish Platform Event via Apex (PaymentApprovalActions.cls). In-app notifications sent via Salesforce Notification Builder. Approval history visible on Payment__c record page via standard Approval History related list.

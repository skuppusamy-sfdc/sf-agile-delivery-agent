## Description

As a Claims Operations Manager, I need an Agentforce (Einstein Copilot) custom action surfaced on the Claim__c record page that can summarize the current payment and EOB status, flag anomalies (e.g., stalled payments, missing EOBs), and recommend next-best actions so that claims staff can resolve bottlenecks faster and reduce average claim closure time.

## Acceptance Criteria

Given a Claims Operations Manager opens a Claim__c record and invokes the 'Claim Payment Status' Copilot action
When the action executes
Then the Copilot response must include: current Claim__c.Status__c, Payment__c.Status__c of the most recent linked payment, EOB__c.Status__c of the most recent linked EOB, and days elapsed since Claim__c.Adjudicated_Date__c

Given Claim__c.Status__c = 'Adjudicated' and no linked Payment__c record exists
When the Copilot action evaluates the claim
Then the response must flag 'Missing Payment Record' as an anomaly and recommend the action 'Trigger Payment Creation' with a link to the relevant Flow

Given a Payment__c record exists with Payment__c.Status__c = 'Pending Disbursement' and Payment__c.Created_Date older than 5 business days
When the Copilot action evaluates the payment
Then the response must flag 'Stalled Payment' and recommend escalation to the Claims Operations Manager queue

Given EOB__c.Status__c = 'Generation Failed'
When the Copilot action evaluates the EOB
Then the response must surface the EOB__c.Failure_Reason__c value and recommend 'Retry EOB Generation' as a next-best action

Given the user does not have the 'Agentforce_Claims_User' Permission Set assigned
When the Copilot action is invoked
Then the action must return a graceful 'Access Denied' message and log the attempt in Agentforce_Action_Audit__c

## Solution

Create a custom Copilot Action (Einstein Copilot Action Builder) of type 'Apex' backed by ClaimCopilotActionService.cls which accepts Claim__c.Id as input and returns a structured ActionResult wrapper containing status summary, anomaly flags, and recommended actions as a JSON string. Apex class queries Claim__c, Payment__c, and EOB__c with appropriate SOQL. Business day calculation for stalled payment detection uses a Custom Metadata holiday calendar (Business_Calendar__mdt). Action registered in Copilot Action library with natural language description for intent matching. Permission check implemented via FeatureManagement.checkPermission('Agentforce_Claims_User'). Audit logging via Platform Event (Agentforce_Audit__e).

## Description

As a Claims Operations Manager, I need Payment__c records to be automatically created and an ERA file triggered when a Claim__c reaches 'Adjudicated' status so that provider reimbursement is initiated without manual intervention and downstream ERA 835 transactions are dispatched to the clearinghouse on the same business day.

## Acceptance Criteria

Given a Claim__c record where Status__c transitions to 'Adjudicated' and all Claim_Line__c records have Adjudication_Status__c = 'Approved' or 'Partial'
When the Record-Triggered Flow fires on the Claim__c object
Then a Payment__c record is created with Payment_Status__c = 'Pending', Payment_Method__c defaulting to Provider__c.Preferred_Payment_Method__c, Payment_Amount__c = SUM(Claim_Line__c.Approved_Amount__c), and Payment__c.Claim__c populated with the originating Claim__c Id

Given the Payment__c record is created successfully
When Payment_Status__c = 'Pending'
Then a Platform Event (Payment_Initiated__e) is published containing Payment__c.Id, Claim__c.Id, Provider__c.NPI__c, and Payment_Amount__c to trigger the ERA 835 generation microservice via the DataIntegration component

Given the ERA file is generated by the external clearinghouse service
When the acknowledgment webhook is received
Then Payment__c.ERA_Transaction_ID__c is populated and Payment_Status__c transitions to 'Issued'

Given a Claim__c with COB flag COB_Applicable__c = true
When adjudication completes
Then Payment__c.COB_Coordination_Flag__c = true and Payment_Amount__c reflects the post-COB approved amount from Claim__c.COB_Adjusted_Amount__c

-Given Payment_Method__c = 'Check', a physical check record is auto-generated in the legacy print queue- (deprecated: check disbursement moved to INS-0541)

Given any Claim_Line__c with Adjudication_Status__c = 'Denied'
When the parent Claim__c is adjudicated
Then those denied lines are excluded from Payment_Amount__c and flagged on the EOB via EOB_Line_Denial_Code__c

## Solution

Implement a Record-Triggered Flow on Claim__c (after-save, fires on Status__c change to 'Adjudicated'). Flow invokes an Apex invocable action (PaymentRecordFactory.cls) to aggregate Claim_Line__c approved amounts and create the Payment__c record. On Payment__c insert, a second Record-Triggered Flow publishes the Payment_Initiated__e Platform Event. A CometD subscriber (PaymentERASubscriber.cls) listens for the event and calls the ERA generation REST endpoint. Inbound ERA acknowledgment is handled via a named credential-secured REST endpoint in PaymentERACallbackController.cls. COB logic handled via a separate Apex utility (COBCalculator.cls) invoked within the flow before payment amount is set.

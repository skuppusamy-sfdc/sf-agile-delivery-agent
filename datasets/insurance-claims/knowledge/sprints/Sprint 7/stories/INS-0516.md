## Description

As a Claims Operations Manager, I need the system to automatically create Payment__c records and initiate disbursement workflows for adjudicated claims — including split-payment scenarios where amounts are distributed between provider, facility, and member — so that all payees receive accurate and timely payments without manual payment entry.

Split-payment logic must evaluate Claim__c.Assignment_of_Benefits__c, Provider_Network_Agreement__c.Payment_Routing__c, and any applicable GSA terms to determine payee allocation. Payments must be dispatched to Workday Financials via integration for check or ACH processing.

## Acceptance Criteria

Given Claim__c.Status__c transitions to 'Adjudicated' and Claim__c.Total_Approved_Amount__c > 0
When Assignment_of_Benefits__c = true
Then a Payment__c record is created with Payee_Type__c = 'Provider', Payment_Amount__c = Claim__c.Total_Approved_Amount__c, and Payment_Status__c = 'Pending Approval'

Given Assignment_of_Benefits__c = false
When the adjudication completes
Then Payment__c is created with Payee_Type__c = 'Member' and linked to Policy__c.Policyholder_Contact__c

Given Claim__c.Split_Payment_Indicator__c = true
When split payment rules are evaluated
Then multiple Payment__c records are created — one per payee — with Payment__c.Split_Sequence__c populated (1, 2, 3...) and the sum of all Payment__c.Payment_Amount__c values equals Claim__c.Total_Approved_Amount__c

Given a Payment__c record has Payment_Status__c = 'Approved'
When the Workday integration callout succeeds
Then Payment__c.Payment_Status__c = 'Disbursed', Payment__c.Disbursement_Date__c = today(), and Payment__c.External_Payment_Reference__c is populated with the Workday transaction ID

Given a payment requires manager approval per Payment__c.Payment_Amount__c > $10,000
When the record is created
Then an Approval Process is launched and Payment__c.Payment_Status__c = 'Pending Approval' until approval action is taken

Given the Workday disbursement callout fails
When the error is returned
Then Payment__c.Payment_Status__c = 'Disbursement Failed' and Payment__c.Failure_Details__c logs the HTTP status and response body

## Solution

Record-Triggered Flow on Claim__c evaluates Assignment_of_Benefits__c and Split_Payment_Indicator__c logic, then calls an Apex invocable action (PaymentCreationService.createPayments()) to handle complex split-payment calculations referencing PNA and GSA records. Payment__c records exceeding $10,000 are routed through a standard Salesforce Approval Process with the Claims Operations Manager as approver. Upon approval, a second Flow calls a Workday REST API via Named Credential (NC-WORKDAY-FINANCE) using an Apex HttpCallout wrapper. Payment results update Payment__c fields via callback endpoint exposed as a Salesforce Site Guest User REST resource. All payment audit events are captured in Payment_Audit_Log__c via an Apex Trigger on Payment__c.

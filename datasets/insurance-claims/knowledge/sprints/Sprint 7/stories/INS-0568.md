## Description

**Bug Report**

**Summary:** EOB__c records are not being generated when Payment__c.Status__c is updated to 'Approved' by the nightly ERAGenerationBatch.cls scheduled job (INS-0563). EOB generation works correctly when Payment__c approval is performed interactively by a user during business hours. Investigation indicates the Record-Triggered Flow responsible for EOB creation (INS-0562) is firing but the downstream Apex invocable action (EOBGenerationService.cls) is silently failing due to a null Session ID when attempting to call the Conga CLM REST API from an asynchronous batch context.

**Steps to Reproduce:**
1. Set up a Payment__c record in 'Pending' status linked to an adjudicated Claim__c
2. Execute ERAGenerationBatch.cls manually via Anonymous Apex (or wait for 02:00 UTC scheduled run)
3. Batch updates Payment__c.Status__c to 'Approved' as part of batch processing
4. Check for EOB__c record creation — record is NOT created
5. Review Apex Debug Logs — observe 'CalloutException: You have uncommitted work pending' or 'Session ID null in batch context' in EOBGenerationService.cls

**Expected Result:** EOB__c record is created and Conga PDF generation is queued regardless of whether Payment__c approval occurs interactively or via batch job.

**Actual Result:** EOB__c record is not created when Payment__c is approved by batch. No error surfaced to users. ClaimsErrorLog__c entry is also missing, indicating the catch block is also not executing in this context.

**Root Cause:** EOBGenerationService.cls makes a synchronous Conga CLM callout inside a method invoked from a Record-Triggered Flow running in batch context. Callouts from batch-triggered flows are not supported in this execution context. The Queueable enqueue inside the catch block also fails because a DML operation precedes it in the same transaction.

**Environment:** Full Sandbox (ACME-FS1), reproducible 100%
**Severity:** P2 — EOB delivery SLA risk

## Solution

Refactor EOBGenerationService.cls to always enqueue EOB generation via CongaEOBQueueable.cls rather than making a synchronous callout. The Record-Triggered Flow should create the EOB__c record in 'Draft' status via a Create Records element (no Apex callout in flow), then publish an EOBGenerationRequested__e Platform Event. A Platform Event-Triggered Flow subscribes to EOBGenerationRequested__e and enqueues the CongaEOBQueueable, which is a valid pattern from both batch and interactive contexts. Update the catch block in CongaEOBQueueable.cls to use a separate Queueable for ClaimsErrorLog__c DML to avoid the uncommitted work issue. Add test coverage in EOBGenerationServiceTest.cls simulating batch execution context using Test.startTest()/stopTest() with a mock HttpCalloutMock. Regression test INS-0562 acceptance criteria post-fix.

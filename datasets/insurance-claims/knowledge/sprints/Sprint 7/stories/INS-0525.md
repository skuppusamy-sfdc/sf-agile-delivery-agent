## Description

As a Provider Relations Specialist, I need a real-time Payment Disbursement Status FlexCard embedded in the Provider 360 console (on both Provider__c Person Account and Facility__c Business Account layouts) so that I can quickly view outstanding, issued, and failed payments for a given provider without navigating away from the account record, reducing call handle time during provider inquiries.

## Acceptance Criteria

Given a Provider Relations Specialist is viewing a Provider__c (Person Account) or Facility__c (Business Account) record page,
When the Payment Disbursement Status FlexCard loads,
Then it displays all Payment__c records linked via Payment__c.Provider__c or Payment__c.Facility__c where Payment__c.Created_Date__c >= LAST_90_DAYS, showing columns: Payment__c.Name, Payment__c.Amount__c, Payment__c.Status__c, Payment__c.Payment_Method__c, Payment__c.Due_Date__c, and associated Claim__c.Claim_Number__c.

Given the FlexCard is loaded,
When Payment__c.Status__c = 'Failed',
Then the row is highlighted in red and a 'Retry Payment' action button is rendered inline, invoking the Retry_Payment OmniScript on click.

Given the FlexCard is loaded,
When Payment__c.Status__c = 'Issued',
Then a 'View ERA' action button is rendered, which navigates to the associated ERA_Transmission_Log__c record.

Given the provider has no Payment__c records in the last 90 days,
When the FlexCard loads,
Then an empty-state message 'No disbursements found in the last 90 days' is displayed with a 'View All Payments' link to a filtered list view.

Given the FlexCard is displayed,
When the data is fetched,
Then the DataRaptor Extract (DR_Get_Provider_Payments) response time must be under 2 seconds for up to 200 Payment__c records.

## Solution

Build OmniStudio FlexCard (FC_Provider_Payment_Disbursement) with DataRaptor Extract data source (DR_Get_Provider_Payments) querying Payment__c with polymorphic provider lookup filter. Card states configured for 'Failed' (red highlight, Retry action), 'Issued' (View ERA action), and empty state. Retry action invokes OmniScript (OS_Retry_Payment) via FlexCard action. FlexCard embedded on Provider__c and Facility__c Lightning Record Pages via App Builder using the OmniStudio FlexCard standard component. DataRaptor uses JSON output with child relationship traversal to Claim__c. Caching configured at 60-second TTL via OmniStudio Cache settings to reduce SOQL load.

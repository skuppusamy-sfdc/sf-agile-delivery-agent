## Description

As a Claims Operations Manager, I need a unified Claims Payment 360 FlexCard embedded on the Claim__c record page that consolidates payment status, EOB delivery status, and ERA transaction details so that adjusters can assess the full disbursement lifecycle at a glance without navigating to multiple related lists or external systems.

## Acceptance Criteria

Given a Claims Adjuster opens a Claim__c record in Salesforce
When the Claims Payment 360 FlexCard loads
Then the card displays Payment__c.Payment_Status__c, Payment__c.Payment_Amount__c, Payment__c.Payment_Method__c, Payment__c.ACH_Transaction_Reference__c, and Payment__c.ERA_Transaction_ID__c sourced via a DataRaptor Extract

Given the Claim__c has an associated Explanation_of_Benefits__c record
When the FlexCard loads
Then EOB_Status__c, EOB_Generated_Date__c, EOB_Delivery_Timestamp__c, and a direct link to the EOB PDF (ContentDocumentLink) are displayed in a dedicated EOB section

Given Payment__c.Payment_Status__c = 'Disbursement_Failed'
When the FlexCard renders
Then the payment status badge renders in red with the Disbursement_Error_Message__c displayed inline and a 'Retry Disbursement' action button visible to users with Permission Set 'Payment_Disbursement_Admin'

Given the Claim__c has no Payment__c record yet
When the FlexCard loads
Then a placeholder message 'Payment not yet initiated — Claim pending adjudication' is displayed and the EOB section is hidden

Given a user without the 'Payment_Disbursement_Admin' Permission Set
When the FlexCard renders with a failed payment
Then the 'Retry Disbursement' action button is hidden (not merely disabled)

## Solution

Build OmniStudio FlexCard 'ClaimsPayment360Card' with two child cards: PaymentStatusCard and EOBStatusCard. DataRaptor Extract 'DR_Extract_ClaimsPayment360' queries Payment__c, Explanation_of_Benefits__c, and ContentDocumentLink in a single structured extraction keyed on Claim__c.Id. Conditional rendering uses FlexCard state conditions on Payment_Status__c value. 'Retry Disbursement' action button invokes an Integration Procedure 'IP_RetryACHDisbursement' which calls ACHDisbursementService. Permission check on button visibility uses a DataRaptor-sourced custom permission flag. FlexCard embedded on Claim__c Lightning Record Page via App Builder.

## Description

As a Claims Operations Manager, I need the claims adjudication engine to correctly apply COB rules at the Claim_Line__c level when a policyholder carries dual insurance coverage so that Acme Insurance pays only its correct portion and avoids overpayment, ensuring compliance with COB regulations and the Group_Service_Agreement__c terms.

## Acceptance Criteria

Given a Claim__c where Policy__c.COB_Active__c = TRUE and COB.Primary_Payer_Paid__c is populated on one or more Claim_Line__c records
When the adjudication flow processes the claim
Then each Claim_Line__c.Allowed_Amount__c is calculated as MIN(Claim_Line__c.Billed_Amount__c - COB.Primary_Payer_Paid__c, Fee_Schedule__c.Allowed_Amount__c) and Claim_Line__c.COB_Applied__c = TRUE

Given Claim_Line__c.COB_Applied__c = TRUE
When Payment__c is created for the parent Claim__c
Then Payment__c.COB_Adjusted_Amount__c is populated and Payment__c.Amount__c does not exceed the GSA.Max_COB_Liability__c for the associated Group_Service_Agreement__c

Given a Claim_Line__c where COB.Primary_Payer_Paid__c >= Claim_Line__c.Billed_Amount__c
When adjudication runs
Then Claim_Line__c.Allowed_Amount__c = 0.00, Claim_Line__c.Status__c = 'Zero Pay COB', and no Payment__c is generated for that line

Given a Claim__c with mixed COB and non-COB Claim_Line__c records
When adjudication completes
Then COB logic is applied only to lines where COB_Active__c = TRUE and non-COB lines are adjudicated using standard Fee_Schedule__c logic independently

-Given COB.Secondary_Payer__c is populated, trigger subrogation workflow automatically-

## Solution

OmniScript-driven adjudication engine invokes a DataRaptor Extract to retrieve all Claim_Line__c records and associated COB data for a given Claim__c. An Apex Integration Procedure applies COB calculation logic per line, referencing Fee_Schedule__c via a SOQL lookup on CPT code and Provider_Network_Agreement__c. Results are written back via DataRaptor Turbo Action updating Claim_Line__c fields in bulk. A separate Apex method validates against GSA.Max_COB_Liability__c before finalizing Payment__c.Amount__c. FlexCard on the Claim__c record displays COB summary to Claims Operations Manager with color-coded indicators for zero-pay lines.

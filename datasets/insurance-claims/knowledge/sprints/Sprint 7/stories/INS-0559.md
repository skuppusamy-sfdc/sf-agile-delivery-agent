# INS-0559: EOB__c Generation_Date__c field not populated when claim has zero-dollar approved amount

## Description

**Bug Description:**
EOB__c records created for zero-dollar payment claims (e.g., fully denied lines or member-responsibility-only claims) are missing the Generation_Date__c value, causing SLA monitoring and compliance reporting queries to fail with null reference errors.

**Steps to Reproduce:**
1. Create a Claim__c with Adjudication_Outcome__c = 'Approved' and Approved_Amount__c = 0.00 (all lines denied or applied to deductible).
2. Set Claim__c.Status__c = 'Adjudicated'.
3. Observe the auto-created EOB__c record (created per INS-0554 flow).
4. Check EOB__c.Generation_Date__c.

**Expected Result:**
EOB__c.Generation_Date__c = date of EOB creation (TODAY() at time of record creation).

**Actual Result:**
EOB__c.Generation_Date__c = null. The Record-Triggered Flow on Payment__c that sets Generation_Date__c is never reached because no Payment__c record is created for zero-dollar claims (Payment__c creation is correctly suppressed per INS-0553 logic), leaving the EOB generation path incomplete for this scenario.

**Impact:** Low-Medium — zero-pay EOBs are still required for regulatory delivery. Approximately 12% of claims result in zero-dollar approved amounts. Compliance reporting dashboard (built on EOB__c.Generation_Date__c) shows inaccurate SLA metrics.

**Root Cause:** EOB__c creation was implemented as a child process of Payment__c disbursement (INS-0554) and does not account for the zero-pay scenario where no Payment__c is generated.

## Acceptance Criteria

As per Description. EOB__c.Generation_Date__c must be populated for all adjudicated claims regardless of Approved_Amount__c value. Regression: existing non-zero payment EOB generation must remain unaffected.

## Solution

Add a separate decision branch in the Claim__c Record-Triggered Flow (or create a new subordinate flow 'EOB_ZeroPay_Generation_Flow') that handles the zero-dollar scenario: when Claim__c.Status__c = 'Adjudicated' AND Approved_Amount__c = 0, directly create an EOB__c record with Generation_Date__c = TODAY(), EOB_Status__c = 'Generated', and EOB__c.Zero_Pay_EOB__c = true (new checkbox field). This bypasses the Payment__c dependency. Update the Conga EOB template to render a 'No Payment Issued' section when Zero_Pay_EOB__c = true. Ensure Delivery_Deadline__c = TODAY() + 30 is still set for SLA tracking.

## Description

As a Claims Operations Manager, I need the system to automatically generate a compliant Explanation_of_Benefits__c (EOB) record and corresponding PDF document when a Payment__c record transitions to 'Approved' status, so that policyholders receive timely, accurate EOB communications via the member portal and physical mail in accordance with state regulatory timelines. Extends INS-0476 (Claim Line adjudication summary).

## Acceptance Criteria

Given a Payment__c record where Payment__c.Status__c transitions to 'Approved',
When the Record-Triggered Flow fires,
Then an Explanation_of_Benefits__c record is created with EOB__c.Claim__c, EOB__c.Policy__c, EOB__c.Payment__c, EOB__c.Generated_Date__c = TODAY(), and EOB__c.Status__c = 'Draft'.

Given the EOB__c record is in 'Draft' status,
When the Conga CLM template merge is triggered via Apex,
Then a PDF is generated using the 'EOB_Standard_Template_v3' Conga template, attached to EOB__c as a ContentDocument, and EOB__c.Status__c updated to 'Generated'.

Given EOB__c.Status__c = 'Generated' and the linked Policy__c.Portal_Enrollment__c = true,
When the delivery process runs,
Then EOB__c.Portal_Delivery_Status__c = 'Delivered' and the document is surfaced on the PolicyholderExperience portal within 1 business day; EOB__c.Portal_Delivery_Date__c is stamped.

Given EOB__c.Status__c = 'Generated' and Policy__c.Portal_Enrollment__c = false,
When the delivery process runs,
Then EOB__c.Mail_Delivery_Status__c = 'Queued' and a physical mail fulfillment Platform Event (EOB_Mail_Request__e) is published with policyholder mailing address from Policy__c.PolicyholderAddress__c.

-Given EOB delivery via fax when Provider__c.Fax_Number__c is populated — DEPRECATED: fax delivery removed from Phase 1 scope per compliance review 2024-11-02-

Given any Claim_Line__c records associated with the Claim__c,
When the EOB PDF is generated,
Then each Claim_Line__c must appear as a line item in the EOB showing: Claim_Line__c.CPT_Code__c, Claim_Line__c.Billed_Amount__c, Claim_Line__c.Allowed_Amount__c, Claim_Line__c.Member_Responsibility__c, and Claim_Line__c.Denial_Reason__c (if applicable).

## Solution

Record-Triggered Flow on Payment__c (After Update, entry condition: Status__c = 'Approved') creates EOB__c record and invokes Apex class (EOBGenerationService) which calls Conga CLM REST API asynchronously via a Queueable Apex job to merge the EOB_Standard_Template_v3 template. Upon PDF callback, ContentVersion is inserted and linked via ContentDocumentLink to EOB__c. A second Queueable handles delivery routing: checks Policy__c.Portal_Enrollment__c and either updates portal visibility via a custom LWC (eobPortalViewer) or publishes EOB_Mail_Request__e Platform Event consumed by the print-and-mail integration. Conga template maps Claim_Line__c child records via a related list DataSource.

## Description

As a Claims Operations Manager, I need an EOB document to be automatically generated and delivered to the Policyholder when a Payment__c record is approved so that members receive timely, compliant benefit explanations and Acme meets state-mandated EOB delivery SLAs. Extends INS-0585 payment automation.

## Acceptance Criteria

Given a Payment__c record where Status__c transitions to 'Approved'
When the Record-Triggered Flow fires
Then an Explanation_of_Benefits__c (EOB) record is created with EOB__c.Claim__c, EOB__c.Payment__c, EOB__c.Policy__c, EOB__c.Generated_Date__c = TODAY(), and EOB__c.Status__c = 'Pending Generation'

Given EOB__c.Status__c = 'Pending Generation'
When the Conga CLM merge job runs
Then a PDF is generated using the 'EOB_Standard_Template_v3' Conga template and attached to the EOB__c record as a Salesforce File (ContentDocument)

Given the EOB PDF is successfully attached
When EOB__c.Delivery_Method__c = 'Email'
Then an automated email is sent to Policy__c.Policyholder_Email__c with the EOB PDF attached and EOB__c.Status__c is updated to 'Delivered'

Given EOB__c.Delivery_Method__c = 'Mail'
When the PDF is generated
Then EOB__c.Status__c = 'Queued for Print' and a Print_Job__c record is created referencing the EOB__c Id

Given the EOB includes line-level detail
When the Conga template renders
Then each Claim_Line__c associated with the Claim__c is listed with: CPT_Code__c, Service_Date__c, Billed_Amount__c, Allowed_Amount__c, Member_Responsibility__c, and Denial_Reason__c (if applicable)

-Given EOB__c.LOB__c = 'Medicare Advantage', use a separate MA-specific template- (consolidated into unified template with LOB conditional sections per design review 2024-11-14)

Given EOB generation fails due to a Conga API timeout
When the error is caught
Then EOB__c.Status__c = 'Generation Failed', a Case is auto-created with Subject = 'EOB Generation Failure - [EOB Id]', and assigned to the Claims Operations queue

## Solution

Record-Triggered Flow on Payment__c fires on Status__c = 'Approved'. Flow creates EOB__c record and invokes a Conga Composer API call via an Apex invocable action (CongaMergeService.generateEOB()). The Apex class handles template selection, PDF attachment via ContentVersion insert, and delivery routing logic. A separate Scheduled Flow retries failed EOBs (Status__c = 'Generation Failed') up to 3 times within a 2-hour window. LWC on the Claim__c record page displays EOB history via a child FlexCard. Permission Set Group 'Claims_Ops_Standard' grants Read on EOB__c.

## Description

As a Claims Operations Manager, I need the system to automatically generate and deliver a compliant Explanation of Benefits (EOB) document to the policyholder whenever a claim payment is disbursed, so that members receive timely, accurate, and regulatory-compliant communication detailing how their claim was adjudicated and what amounts were paid, adjusted, or denied.

Background: Currently, EOB generation is a manual process handled outside of Salesforce, resulting in delays, inconsistencies, and compliance risk. This story automates EOB creation tied directly to the Payment__c record and delivers it via the policyholder's preferred communication channel. This extends work completed in INS-0571 (Payment__c disbursement engine) and INS-0583 (Policyholder communication preferences).

Key Data Points to Include on EOB:
- Claim__c.Claim_Number__c
- Claim__c.Date_of_Service__c
- Claim_Line__c details (CPT code, ICD-10 diagnosis, billed amount, allowed amount, plan paid amount, member responsibility)
- Provider__c (NPI, name, TIN)
- Payment__c.Payment_Amount__c, Payment__c.Payment_Date__c, Payment__c.Payment_Method__c
- COB adjustments if applicable
- Denial reason codes per Claim_Line__c.Denial_Reason_Code__c
- Member deductible, copay, and coinsurance breakdown
- Policyholder grievance/appeal rights language (regulatory requirement)

## Acceptance Criteria

Given a Payment__c record is created with Payment__c.Status__c = 'Disbursed' and Payment__c.Payment_Type__c = 'Claim Payment',
When the Record-Triggered Flow fires on Payment__c insert/update,
Then a new Explanation_of_Benefits__c record must be created and linked via Explanation_of_Benefits__c.Payment__c (lookup) and Explanation_of_Benefits__c.Claim__c (lookup) within 2 minutes of disbursement.

Given the Explanation_of_Benefits__c record is created,
When the EOB generation Apex service is invoked,
Then all associated Claim_Line__c records for the parent Claim__c must be queried and their fields (Claim_Line__c.CPT_Code__c, Claim_Line__c.ICD10_Primary__c, Claim_Line__c.Billed_Amount__c, Claim_Line__c.Allowed_Amount__c, Claim_Line__c.Plan_Paid_Amount__c, Claim_Line__c.Member_Responsibility__c, Claim_Line__c.Denial_Reason_Code__c) must be accurately reflected in the EOB document body.

Given the EOB document is being assembled via Conga Composer template,
When the policyholder's LOB is 'Commercial' or 'Medicare Advantage',
Then the correct regulatory appeals language block must be conditionally rendered based on Claim__c.Line_of_Business__c field value, with Medicare Advantage EOBs including CMS-mandated grievance language.

Given the Explanation_of_Benefits__c.Status__c = 'Generated',
When the policyholder's preferred delivery method (Policy__c.EOB_Delivery_Preference__c) is evaluated,
Then:
  - If value = 'Email', trigger an automated email via Salesforce OmniSend with the EOB PDF attached
  - If value = 'Portal', post the document to the Policyholder Experience portal and set Explanation_of_Benefits__c.Portal_Available__c = TRUE
  - If value = 'Mail', create a Print_Queue__c record with status 'Pending Print' for downstream fulfillment vendor handoff
  - -If value = 'Fax', send via integrated fax service- (deprecated — fax delivery no longer supported per INS-0591 decision)

Given a Claim__c has COB (Coordination of Benefits) flag set (Claim__c.COB_Applicable__c = TRUE),
When the EOB is generated,
Then Explanation_of_Benefits__c.COB_Primary_Payer_Amount__c and Explanation_of_Benefits__c.COB_Secondary_Payer_Amount__c must be populated from the COB adjudication results and displayed in a dedicated COB section on the EOB document.

Given the EOB has been successfully delivered (email sent, portal posted, or print queue record created),
When delivery confirmation is received or portal post succeeds,
Then Explanation_of_Benefits__c.Status__c must be updated to 'Delivered', Explanation_of_Benefits__c.Delivery_Date__c must be stamped with the current datetime, and a Chatter post must be added to the parent Claim__c record: 'EOB successfully delivered to member via [delivery method] on [date]'.

Given an EOB generation or delivery failure occurs (Apex exception, Conga error, or email bounce),
When the error is caught,
Then Explanation_of_Benefits__c.Status__c must be set to 'Failed', Explanation_of_Benefits__c.Failure_Reason__c must capture the error message, a Platform Event (EOB_Failure__e) must be published to alert the Claims Operations queue, and a Case must be auto-created with Case.Subject = 'EOB Delivery Failure - [Claim Number]' and Case.Priority = 'High' assigned to the Claims Operations queue.

Given a batch reprocessing scenario where Payment__c.Reprocess_EOB__c is manually set to TRUE by a Claims Operations Manager,
When the batch Apex job (EOBReprocessBatch) runs nightly,
Then previously failed or missing EOBs must be regenerated and redelivered, and Explanation_of_Benefits__c.Reprocessed__c must be set to TRUE with an audit trail entry created.

-Given the EOB is generated for a TPA-administered plan, the TPA branding logo must be dynamically injected into the Conga template header- (descoped to Phase 2 — tracked in INS-0614)

Given any EOB record is created or updated,
When a Compliance Officer runs the EOB Compliance Dashboard,
Then all Explanation_of_Benefits__c records must be queryable by Explanation_of_Benefits__c.Status__c, Explanation_of_Benefits__c.Delivery_Date__c, Claim__c.Line_of_Business__c, and Payment__c.Payment_Date__c to support regulatory audit reporting.

## Solution

Technical implementation approach:

1. **Record-Triggered Flow (Payment__c):** Create a Record-Triggered Flow on Payment__c that fires After Save on insert and update where Payment__c.Status__c = 'Disbursed' AND Payment__c.Payment_Type__c = 'Claim Payment'. Flow invokes an Apex action (EOBOrchestrationService.initiateEOBGeneration()) passing the Payment__c Id.

2. **Apex Service Class (EOBOrchestrationService):** Invocable Apex class that:
   - Creates the Explanation_of_Benefits__c record with all header fields populated
   - Queries related Claim_Line__c records via SOQL
   - Calls CongaComposerCalloutService to merge data into the pre-built EOB Conga template (template ID stored in Custom Metadata Type: EOB_Template_Config__mdt)
   - Stores generated PDF as a ContentDocument linked to both Explanation_of_Benefits__c and Claim__c
   - Evaluates Policy__c.EOB_Delivery_Preference__c and routes to appropriate delivery handler method

3. **Delivery Handlers:**
   - Email: OmniSend API callout via named credential; attach ContentDocument PDF
   - Portal: Update Explanation_of_Benefits__c.Portal_Available__c = TRUE; LWC FlexCard on Policyholder Experience portal surfaces the document via ContentDocumentLink query
   - Mail: Apex creates Print_Queue__c record; separate nightly batch job packages print queue records for fulfillment vendor SFTP delivery

4. **Platform Event (EOB_Failure__e):** Published on any caught exception. A separate Platform Event-triggered Flow subscribes to EOB_Failure__e, creates the high-priority Case, and sends an in-app notification to the Claims Operations queue using Custom Notification.

5. **Batch Apex (EOBReprocessBatch):** Scheduled nightly at 2:00 AM EST. Queries Explanation_of_Benefits__c where Status__c = 'Failed' OR (Payment__c.Status__c = 'Disbursed' AND Explanation_of_Benefits__c = null) and Reprocess_EOB__c = TRUE. Calls EOBOrchestrationService in batch context with a scope of 50.

6. **Permission Set Group (PSG_Claims_EOB_Manager):** Grants Claims Operations Manager CRUD on Explanation_of_Benefits__c, Print_Queue__c, and read access to ContentDocument. Compliance Officer Permission Set grants read-only access to all EOB records across all accounts via Sharing Rule (criteria-based: all Explanation_of_Benefits__c records shared to Compliance Officer public group).

7. **Custom Metadata Types:** EOB_Template_Config__mdt stores Conga template IDs per LOB; EOB_Regulatory_Language__mdt stores appeals language blocks keyed by LOB and state for dynamic injection.

8. **Conga Composer Template:** Separate templates for Commercial, Medicare Advantage, and Medicaid LOBs. COB section conditionally rendered using Conga IF statements based on Claim__c.COB_Applicable__c. Claim_Line__c data rendered via Conga repeating table.

9. **Testing:** Apex test class (EOBOrchestrationServiceTest) with minimum 85% code coverage; test scenarios include happy path, COB claim, delivery failure, and batch reprocess. UAT scripts provided for Claims Operations Manager and Compliance Officer personas.

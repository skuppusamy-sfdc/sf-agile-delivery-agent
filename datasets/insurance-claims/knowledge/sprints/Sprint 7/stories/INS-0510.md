## Description

**Bug Report**

**Summary:** When a Claim__c record with COB_Applied__c = true is adjudicated, the auto-created Payment__c record is populated with Claim__c.Billed_Amount__c instead of Claim__c.COB_Adjusted_Amount__c, resulting in overpayment to providers.

**Steps to Reproduce:**
1. Create a Claim__c record with Status__c = 'In Review', Billed_Amount__c = 1500.00, Approved_Amount__c = 900.00, COB_Applied__c = true, COB_Adjusted_Amount__c = 650.00.
2. Update Claim__c.Status__c to 'Adjudicated'.
3. Navigate to the auto-created Payment__c record linked to the Claim.
4. Observe Payment__c.Amount__c.

**Expected Result:** Payment__c.Amount__c = 650.00 (COB_Adjusted_Amount__c)

**Actual Result:** Payment__c.Amount__c = 1500.00 (Billed_Amount__c)

**Environment:** Acme Insurance Full Sandbox (AIS-FULL)
**First Observed:** Sprint 7 regression testing, 2024-11-14
**Severity:** High — financial impact; overpayment to providers if deployed to production
**Related Story:** INS-0506 (Payment__c creation flow)

## Acceptance Criteria

As per Description.

Given a Claim__c with COB_Applied__c = true and COB_Adjusted_Amount__c = 650.00,
When Claim__c.Status__c is set to 'Adjudicated',
Then Payment__c.Amount__c must equal 650.00, not Billed_Amount__c or Approved_Amount__c.

## Solution

Root cause identified in PaymentCreationService.createPayment() Apex method: the COB conditional branch (lines 47-52) references Claim__c.Billed_Amount__c due to a copy-paste error from an earlier method. Fix: update the field reference in the COB branch from 'claim.Billed_Amount__c' to 'claim.COB_Adjusted_Amount__c'. Add a unit test in PaymentCreationServiceTest covering COB = true scenario asserting Payment__c.Amount__c = COB_Adjusted_Amount__c. Deploy fix to AIS-FULL sandbox for regression, then to UAT. Update the Record-Triggered Flow decision element documentation to clarify COB logic path.

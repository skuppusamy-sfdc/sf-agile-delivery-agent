## Description

As a Claims Operations Manager, I need the adjudication engine to automatically look up and apply the correct Fee_Schedule__c rates to each Claim_Line__c based on the claim's Line of Business (LOB), the rendering provider's NPI, the service date, and the CPT code, so that allowed amounts are calculated accurately and consistently in compliance with contracted rates defined in the Provider_Network_Agreement__c.

The fee schedule lookup must handle: multiple LOBs (Commercial, Medicare Advantage, Medicaid MCO), effective date ranges on Fee_Schedule__c, provider-specific rate overrides on PNA, and fallback to default schedule when no provider-specific rate exists.

## Acceptance Criteria

Given a Claim_Line__c record is submitted for adjudication
When the Fee Schedule lookup process runs
Then the engine queries Fee_Schedule__c records filtered by: LOB__c matching Claim__c.Line_of_Business__c, Effective_Start_Date__c <= Claim__c.Service_Date__c <= Effective_End_Date__c, and CPT_Code__c = Claim_Line__c.Procedure_Code__c

Given a matching Fee_Schedule__c record is found
When the provider has a PNA with a rate override for the CPT code
Then Claim_Line__c.Allowed_Amount__c = Provider_Network_Agreement__c.Override_Rate__c and Claim_Line__c.Rate_Source__c = 'PNA Override'

Given no PNA rate override exists
When a standard Fee_Schedule__c match is found
Then Claim_Line__c.Allowed_Amount__c = Fee_Schedule__c.Allowed_Amount__c and Claim_Line__c.Rate_Source__c = 'Fee Schedule'

Given no Fee_Schedule__c record matches the LOB, CPT, and service date combination
When the fallback logic executes
Then Claim_Line__c.Allowed_Amount__c = Claim_Line__c.Billed_Amount__c * Fee_Schedule__c.Default_Reimbursement_Percentage__c / 100 using the LOB default schedule, and Claim_Line__c.Rate_Source__c = 'Default Percentage'

Given Claim__c.Line_of_Business__c = 'Medicare Advantage'
When the fee schedule lookup runs
Then the engine additionally validates against CMS published rates via the CMS integration endpoint and logs any variance > 5% in Fee_Schedule_Variance_Log__c

Given the Claim_Line__c.Procedure_Code__c is not a valid CPT code per the current CPT code master
When adjudication runs
Then Claim_Line__c.Adjudication_Status__c = 'Pended' and Claim_Line__c.Pend_Reason__c = 'Invalid CPT Code — Manual Review Required'

-Given fee schedule lookup uses a legacy stored procedure via Heroku Connect- (deprecated: replaced with native Salesforce SOQL lookup per platform consolidation initiative)

## Solution

Implement FeeScheduleLookupService Apex class called from the main AdjudicationEngine Apex class. The lookup uses a priority-ordered query chain: (1) PNA rate override lookup joining Provider_Network_Agreement__c and Fee_Schedule_Line__c, (2) standard Fee_Schedule__c lookup with date range filter, (3) default percentage fallback. All queries use custom indexes on CPT_Code__c, LOB__c, and Effective_Start_Date__c for performance. CMS rate validation for Medicare Advantage is handled via a Named Credential REST callout (NC-CMS-RATES) with results cached in a Platform Cache partition (FeeScheduleCache) with 24-hour TTL to minimize API calls. CPT code validation references a custom metadata type (CPT_Code_Master__mdt). All rate decisions are written to Fee_Schedule_Application_Log__c for audit trail.

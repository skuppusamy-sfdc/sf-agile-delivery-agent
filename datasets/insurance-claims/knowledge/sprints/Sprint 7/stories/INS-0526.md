## Description

As a Claims Operations Manager, I need the system to automatically apply Coordination of Benefits (COB) adjustment logic at the Claim_Line__c level when a Claim__c is flagged as a COB claim so that payment amounts correctly reflect primary and secondary payer responsibilities, preventing overpayment and ensuring compliance with CMS COB guidelines. This story extends the adjudication engine established in INS-0501.

## Acceptance Criteria

Given Claim__c.COB_Indicator__c = true and Claim__c.COB_Type__c = 'Secondary Payer',
When the COB Adjudication Flow processes the claim,
Then for each Claim_Line__c, Claim_Line__c.COB_Adjustment_Amount__c = Claim_Line__c.Billed_Amount__c - Claim_Line__c.Primary_Payer_Paid__c - Claim_Line__c.Member_Responsibility__c, and Claim_Line__c.Allowed_Amount__c is capped at this adjusted value.

Given Claim_Line__c.COB_Adjustment_Amount__c is calculated,
When the resulting value is <= 0,
Then Claim_Line__c.Status__c = 'Denied - COB Zero Balance' and Claim_Line__c.Denial_Reason_Code__c = 'COB-001'.

Given Claim__c.COB_Type__c = 'Primary Payer',
When COB logic is applied,
Then standard Fee_Schedule__c adjudication rules apply without COB adjustment and Claim__c.COB_Primary_Amount__c is recorded for downstream secondary payer reference.

Given COB processing is complete for all Claim_Line__c records,
When Claim__c.Total_Approved_Amount__c is recalculated,
Then it equals the SUM of all Claim_Line__c.COB_Adjustment_Amount__c where Claim_Line__c.Status__c != 'Denied - COB Zero Balance'.

-Given COB claims require manual supervisor approval for amounts > $10,000- (Deprecated: auto-approval threshold raised to $25,000 per policy update POL-2024-08)

Given COB processing is complete,
When Claim__c.Total_Approved_Amount__c > 0,
Then Claim__c.Status__c = 'Adjudicated' and the standard Payment__c auto-creation flow (INS-0522) is triggered.

Given any COB calculation error (e.g., missing Claim_Line__c.Primary_Payer_Paid__c),
When the flow encounters incomplete COB data,
Then Claim__c.Status__c = 'Pending COB Data' and a Task is created for the Claims Operations Manager with Subject = 'COB Data Incomplete - Action Required'.

## Solution

Implement a sub-flow (SF_COB_Adjudication_Logic) invoked from the main Claims Adjudication Record-Triggered Flow via the 'Run a Subflow' element. Sub-flow iterates over Claim_Line__c collection using Loop element, applies COB formula logic via Assignment elements, and updates each Claim_Line__c. Aggregate recalculation of Claim__c.Total_Approved_Amount__c performed via Roll-Up Summary field override using Apex trigger (ClaimLineAfterUpdate) to handle complex filtering. COB_Type__c picklist values managed via Custom Metadata Type COB_Rules__mdt for configurability. Denial reason codes stored in COB_Denial_Code__mdt Custom Metadata. Integration with CMS COB contractor data via scheduled DataRaptor integration procedure for primary payer verification.

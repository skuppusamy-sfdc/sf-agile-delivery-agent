## Description

As a Claims Operations Manager, I need an EOB document to be automatically generated and delivered to the policyholder when a claim reaches 'Adjudicated' status, so that members receive timely and accurate benefit explanations in compliance with state mandates and plan requirements.

The EOB must summarize each Claim_Line__c with adjudicated amounts, denial reasons (where applicable), and patient responsibility breakdowns. The document should be delivered via DocuSign envelope to the member's preferred contact method on file.

## Acceptance Criteria

Given a Claim__c record transitions to Status__c = 'Adjudicated'
When all associated Claim_Line__c records have Adjudication_Status__c = 'Processed' or 'Denied'
Then an Explanation_of_Benefits__c record is created with EOB_Status__c = 'Generated' and linked via Claim__c lookup

Given the EOB record is created
When the EOB generation process runs
Then each Claim_Line__c is reflected as a child row on the EOB with fields: CPT_Code__c, Billed_Amount__c, Allowed_Amount__c, Plan_Paid_Amount__c, Member_Responsibility__c, and Denial_Reason_Code__c (if applicable)

Given the EOB document is generated via Conga CLM template
When the member's Contact.Preferred_Contact_Method__c = 'Email'
Then a DocuSign envelope is dispatched to Contact.Email with the EOB PDF attached and Explanation_of_Benefits__c.Delivery_Status__c updated to 'Sent'

Given the EOB delivery fails due to invalid email
When DocuSign returns a bounce callback
Then Explanation_of_Benefits__c.Delivery_Status__c = 'Failed' and a Task is created assigned to the owning Claims Operations queue

-Given EOB generation is triggered on Claim_Line__c update- (deprecated: trigger point moved to Claim__c status transition per INS-0498)

Given a COB scenario where Claim__c.COB_Indicator__c = true
When the EOB is generated
Then the COB_Coordination_Details__c section is populated with primary and secondary payer amounts and Explanation_of_Benefits__c.COB_Applied__c = true

## Solution

Implement a Record-Triggered Flow on Claim__c firing on Status__c change to 'Adjudicated'. Flow invokes an Apex invocable method (EOBGenerationService.generateEOB()) that queries all child Claim_Line__c records and constructs the Explanation_of_Benefits__c header and line data. Conga CLM REST API is called asynchronously via a Queueable Apex class to render the PDF template (Template ID: TMPL-EOB-001). Upon PDF generation, a DocuSign envelope is created via DocuSign eSignature API. Platform Event (EOB_Generated__e) is published to notify downstream ERA processing. A separate Flow subscribes to the Platform Event to update Claim__c.EOB_Sent__c = true.

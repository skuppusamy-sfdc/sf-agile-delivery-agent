## Description

As a Claims Operations Manager, I need a Payment Summary FlexCard embedded on the Claim__c record page that displays real-time payment status, disbursement details, and EOB delivery status so that I can assess the full payment lifecycle of a claim without navigating to multiple related records. This component should surface Payment__c, EOB__c, and ERA transmission data in a consolidated, actionable view.

## Acceptance Criteria

Given a Claims Operations Manager views a Claim__c record page
When the Payment Summary FlexCard loads
Then it displays the following data sourced via DataRaptor:
  - Payment__c.Payment_Amount__c
  - Payment__c.Payment_Status__c (with color-coded badge: green='Disbursed', yellow='Pending', red='On Hold*')
  - Payment__c.Payment_Method__c
  - Payment__c.ERA_Generated__c and Payment__c.ERA_File_Date__c
  - EOB__c.EOB_Status__c
  - EOB__c.Delivery_Deadline__c (highlighted red if within 5 days and status != 'Sent')

Given a Claim__c with multiple Payment__c records (e.g., COB split payments)
When the FlexCard renders
Then all associated Payment__c records are displayed in a child data table within the card, sorted by Payment__c.CreatedDate descending

Given Payment__c.Payment_Status__c = 'On Hold - COB Review'
When the FlexCard renders
Then an action button 'Release COB Hold' is visible to users with Permission Set 'Claims_COB_Adjuster'
And clicking the button launches the 'COB_Hold_Release' OmniScript

Given the FlexCard is loading
When the DataRaptor Extract query returns no Payment__c records
Then the card displays an empty state message: 'No payment records found for this claim.'

-FlexCard to include a Refund Initiation button visible to all Claims staff- (deprecated: refund initiation scoped to separate story INS-0562 with stricter permission controls)

## Solution

Build OmniStudio FlexCard 'Claim_Payment_Summary_FC' with a DataRaptor Extract (DRE) 'Get_Claim_Payment_Summary' that queries Payment__c and EOB__c via SOQL with Claim__c.Id as input parameter. Child data table built using FlexCard Child Element with repeat block on Payment__c list. Color-coded badges implemented via FlexCard conditional styling on Payment_Status__c field value. 'Release COB Hold' action button conditionally rendered using FlexCard State/Condition on user Permission Set membership (checked via custom UserInfo Apex action). Button click navigates to OmniScript 'COB_Hold_Release' in modal. FlexCard deployed to Claim__c Lightning Record Page via App Builder.

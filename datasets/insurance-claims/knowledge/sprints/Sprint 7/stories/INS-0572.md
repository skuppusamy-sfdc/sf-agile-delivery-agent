## Description

As a Provider Relations Specialist, I need a FlexCard on the Provider 360 page (Provider__c / Person Account) that displays a summarized payment history and outstanding claim balances, so that I can quickly assess a provider's payment standing during calls and network agreement reviews without navigating to multiple related lists.

## Acceptance Criteria

Given a Provider__c record is open in the Provider 360 Lightning page,
When the 'Payment History' FlexCard loads,
Then it displays the last 12 months of Payment__c records related to the provider via Payment__c.Provider__c, showing columns: Payment__c.Name, Amount__c, Payment_Status__c, Payment_Date__c, and Claim__c.Claim_Number__c.

Given the FlexCard is rendered,
When Payment__c.Payment_Status__c = 'Failed',
Then the row is highlighted in red with an alert icon and a 'Resolve' action button that launches the PaymentResolution OmniScript.

Given the FlexCard summary section,
When the DataRaptor Extract aggregates Payment__c data,
Then summary tiles display: Total_Paid_YTD__c (sum of cleared payments this calendar year), Outstanding_Balance__c (sum of Pending + Submitted payments), and Failed_Payment_Count__c.

Given a provider with no Payment__c records,
When the FlexCard loads,
Then an empty state message 'No payment history available' is displayed and summary tiles show $0.00.

Given the FlexCard is viewed by a user with Profile 'Provider_Relations_User',
When the page loads,
Then only Payment__c records where Payment__c.LOB__c matches the user's assigned LOB__c (via Custom Permission or User field) are displayed, enforcing data segregation.

## Solution

Build OmniStudio FlexCard 'ProviderPaymentHistory_FC' with a DataRaptor Extract as the primary data source querying Payment__c with filter on Provider__c = {recordId} and Payment_Date__c = LAST_N_MONTHS:12. Add a second DataRaptor for aggregate summary tiles using aggregate SOQL. Conditional styling on Payment_Status__c = 'Failed' using FlexCard State configuration. 'Resolve' action button invokes PaymentResolution OmniScript (built in INS-0570 scope). LOB filtering implemented via a DataRaptor input parameter sourced from UserInfo custom field via Integration Procedure. Add FlexCard to the Provider_360_Lightning_Page via Lightning App Builder. Assign to Provider_Relations_PSG Permission Set Group.

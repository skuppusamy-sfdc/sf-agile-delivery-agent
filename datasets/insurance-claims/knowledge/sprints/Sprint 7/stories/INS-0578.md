## Description

As a Claims Operations Manager, I need an Explanation_of_Benefits__c (EOB) record to be automatically created and a PDF rendered using Conga CLM when a Payment__c.Status__c transitions to 'Disbursed' so that policyholders and providers receive accurate, regulation-compliant EOB documentation within mandated timelines. Extends INS-0577.

## Acceptance Criteria

Given a Payment__c.Status__c transitions from 'Pending' to 'Disbursed'
When the Record-Triggered Flow on Payment__c fires
Then an Explanation_of_Benefits__c record is created with EOB.Claim__c = Payment__c.Claim__c, EOB.Payment__c = the triggering Payment__c.Id, EOB.Policyholder__c = Claim__c.Policy__c.Insured__c, EOB.Total_Billed__c = sum of all Claim_Line__c.Billed_Amount__c, EOB.Total_Allowed__c = sum of Claim_Line__c.Allowed_Amount__c, EOB.Total_Paid__c = Payment__c.Amount__c, EOB.Status__c = 'Generated'

Given the EOB record is created
When Conga CLM is invoked via Apex callout
Then a PDF is rendered using the EOB_Standard_Template and attached to the EOB record as a ContentVersion with Title = 'EOB_[ClaimNumber]_[YYYY-MM-DD].pdf'

Given the EOB PDF is attached
When EOB.Delivery_Preference__c = 'Email'
Then an automated email is dispatched to Policy__c.Insured_Email__c with the EOB PDF as an attachment within 1 hour of disbursement

Given EOB.Delivery_Preference__c = 'Portal'
When the PDF is generated
Then the ContentVersion is shared to the Policyholder Experience community and EOB.Portal_Published__c = TRUE

-Given EOB.Delivery_Preference__c = 'Fax', fax is sent via integrated fax service-

Given any Claim_Line__c on the parent Claim__c has Claim_Line__c.Denial_Reason__c populated
When the EOB is generated
Then the denial reason codes and descriptions are included in the EOB PDF line-item section per ICD-10 and CPT mapping

Given the EOB generation fails due to a Conga CLM callout timeout
When the error is caught
Then EOB.Status__c = 'Generation Failed', a ClaimsProcessing_Error__c is logged, and a retry is scheduled via Scheduled Flow within 30 minutes

## Solution

Record-Triggered Flow on Payment__c (After Update) detects Status__c = 'Disbursed' and calls an Apex invocable method that (1) queries related Claim_Line__c records for financial rollups, (2) creates the EOB record, and (3) makes a synchronous Conga CLM REST API callout to render the PDF template, storing the response as ContentVersion linked to EOB via ContentDocumentLink. Delivery routing handled by a subsequent Flow Decision element branching on Delivery_Preference__c. Portal sharing uses a manual Share record insert in Apex. Retry logic uses a Scheduled Flow triggered by a Platform Event published on failure. Permission Set Group 'EOB_Viewer' controls policyholder community access to ContentVersion.

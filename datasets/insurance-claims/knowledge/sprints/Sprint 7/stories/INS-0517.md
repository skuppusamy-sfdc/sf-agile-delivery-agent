## Description

As a Claims Operations Manager, I need a FlexCard on the Policyholder 360 page that displays a summarized view of the member's EOB history, recent payment disbursements, and year-to-date benefit utilization, so that agents and claims staff can quickly assess a member's financial exposure and claims activity without navigating to multiple related lists.

The FlexCard should surface data from Explanation_of_Benefits__c, Payment__c, and Claim__c objects filtered by the active Policy__c record in context.

## Acceptance Criteria

Given a user navigates to a Contact (Policyholder) record with an active Policy__c
When the Policyholder 360 Lightning page loads
Then the EOB & Payment History FlexCard is visible and displays the 5 most recent Explanation_of_Benefits__c records sorted by EOB_Date__c descending

Given the FlexCard is rendered
When data is loaded via DataRaptor Extract
Then each EOB row displays: Claim__c.Claim_Number__c, EOB_Date__c, Total_Billed__c, Plan_Paid_Amount__c, Member_Responsibility__c, and Explanation_of_Benefits__c.Delivery_Status__c

Given the FlexCard is rendered
When year-to-date aggregation is computed
Then a summary band displays YTD_Deductible_Met__c, YTD_Out_of_Pocket__c, and Benefit_Utilization_Percentage__c pulled from Policy__c

Given a user clicks the 'View EOB' action on a FlexCard row
When the action fires
Then the associated EOB PDF (ContentVersion) opens in a new browser tab

-Given FlexCard uses inline Apex controller for data retrieval- (deprecated: replaced with DataRaptor Extract per OmniStudio best practices)

Given the member has no EOB records
When the FlexCard loads
Then an empty state message 'No EOB records found for this policy period' is displayed

## Solution

Build an OmniStudio FlexCard (FC_EOB_PaymentHistory) with a DataRaptor Extract (DR_EOB_ByPolicy) sourcing data from Explanation_of_Benefits__c and Payment__c with a SOQL filter on Policy__c.Id passed as input parameter. YTD aggregation is handled by a separate DataRaptor Turbo Extract querying Policy__c benefit accumulator fields. 'View EOB' row action invokes a Navigation Action to the ContentVersion record. FlexCard is embedded on the Policyholder 360 Lightning Record Page via the OmniStudio FlexCard component. Visibility is controlled by a Permission Set Group (PSG_Claims_Operations) to restrict access to authorized roles.

## Description

As a Claims Operations Manager, I need the system to apply Coordination of Benefits (COB) rules to Claim_Line__c adjudication when a member has dual coverage so that Acme pays only its appropriate share and the member's total benefit does not exceed 100% of the allowed amount. COB determination must follow the standard 'Birthday Rule' for dependents and 'Active Employment' rule for primary subscribers.

## Acceptance Criteria

Given a Claim__c where Claim__c.COB_Applicable__c = TRUE and Claim__c.Secondary_Payer__c is populated
When adjudication processing begins
Then the system must retrieve the primary payer's paid amount from Claim__c.Primary_Payer_Paid_Amount__c before calculating Acme's liability

Given Claim_Line__c adjudication with COB
When Acme is the secondary payer
Then Claim_Line__c.Acme_Liability__c = MIN(Claim_Line__c.Allowed_Amount__c - Claim__c.Primary_Payer_Paid_Amount__c, 0) and Claim_Line__c.COB_Savings__c is populated with the reduction amount

Given Acme is the primary payer under Birthday Rule
When the insured's Policy__c.Relationship_to_Subscriber__c = 'Dependent'
Then COB order is determined by comparing Policy__c.Primary_Subscriber_DOB__c against secondary policy holder DOB; earlier birthday month/day = primary

Given a Claim__c where COB results in zero member liability
When adjudication is finalized
Then Claim__c.Member_Responsibility__c = 0.00 and EOB__c.COB_Indicator__c = TRUE (per INS-0530 EOB generation)

-Given COB applies to dental LOB claims, a separate dental COB fee schedule lookup is required-

Given invalid or missing primary payer data
When COB calculation cannot be completed
Then Claim__c.Status__c is set to 'Pended' with Claim__c.Pend_Reason__c = 'COB_Data_Missing' and a Task is created for the Claims Operations queue

## Solution

Apex service class (COBAdjudicationService) invoked from the ClaimsAdjudicationEngine during the Claim_Line__c processing loop. COB rules stored in a Custom Metadata Type (COB_Rule__mdt) keyed by LOB and payer order. Birthday Rule logic implemented as a static utility method comparing subscriber DOB fields. Integration with external COB data via a REST callout to the CMS COB contractor API (BCRC) using Named Credential 'CMS_COB_API'. Results written back to Claim_Line__c fields via Apex DML within a transaction-safe savepoint pattern.

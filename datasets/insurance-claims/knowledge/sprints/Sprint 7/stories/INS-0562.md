## Description

As a Claims Operations Manager, I need an Explanation_of_Benefits__c (EOB) record and corresponding PDF document to be automatically generated when a Payment__c record transitions to 'Approved' status so that policyholders receive a compliant, itemized summary of their claim adjudication in accordance with state mandate timelines. Extends INS-0561.

## Acceptance Criteria

Given a Payment__c record where Payment__c.Status__c transitions to 'Approved'
When the Record-Triggered Flow fires
Then an Explanation_of_Benefits__c record is created with EOB__c.Claim__c = Payment__c.Claim__c, EOB__c.Policy__c = Claim__c.Policy__c, EOB__c.Payment__c = Payment__c.Id, and EOB__c.Status__c = 'Draft'

Given an EOB__c record in 'Draft' status
When the Conga CLM merge template 'EOB_Standard_v3' is invoked
Then a PDF is generated containing all Claim_Line__c records associated with the parent Claim__c including Claim_Line__c.CPT_Code__c, Claim_Line__c.Billed_Amount__c, Claim_Line__c.Allowed_Amount__c, Claim_Line__c.Member_Responsibility__c, and Claim_Line__c.Denial_Reason__c (if applicable)

Given the EOB PDF is successfully generated
When EOB__c.Delivery_Method__c = 'Portal'
Then the PDF is attached to the EOB__c record as a ContentDocument and EOB__c.Status__c is updated to 'Delivered'

Given EOB__c.Delivery_Method__c = 'Mail'
When the PDF is generated
Then a print fulfillment Platform Event (EOBMailFulfillment__e) is published with the ContentDocument.Id and Policy__c.Mailing_Address__c fields

Given any EOB generation failure
When a Conga merge exception is thrown
Then EOB__c.Status__c = 'Generation Failed', a Case is auto-created assigned to the Claims Operations queue, and an error is logged to the ClaimsErrorLog__c custom object

-EOB must include provider Tax ID (TIN) on the document face- (removed per legal review, TIN masked per data privacy policy INS-0548)

## Solution

Record-Triggered Flow on Payment__c (after-save) triggers an Apex invocable action (EOBGenerationService.cls) that creates the EOB__c record and invokes the Conga CLM REST API asynchronously via a Queueable Apex class (CongaEOBQueueable.cls). The Conga template 'EOB_Standard_v3' is mapped to Claim__c, Claim_Line__c (related list), and Provider__c data. Delivery routing logic is handled via a subsequent Flow subflow. Platform Event EOBMailFulfillment__e is published via EventBus.publish(). ContentDocumentLink is created programmatically to associate the PDF with EOB__c. Error handling uses a try/catch in the Queueable with DML to ClaimsErrorLog__c.

## Description

As a Provider Relations Specialist, I need a guided OmniScript to capture and update provider EFT enrollment preferences, bank account details, and ERA delivery settings on the Provider__c (Person Account) record so that payment disbursement and remittance configurations are accurately maintained without direct object editing. The OmniScript must support both initial enrollment and update scenarios, and must mask sensitive bank account data in the UI post-save. This story supports the payment method defaulting logic introduced in INS-0553.

## Acceptance Criteria

Given a Provider Relations Specialist launches the 'Provider_Payment_Preferences' OmniScript from the Provider__c record page action menu
When the OmniScript loads
Then it pre-populates existing values from Provider__c fields: EFT_Enrolled__c, Bank_Account_Last4__c, Bank_Routing_Number__c (masked), ERA_Delivery_Preference__c, SFTP_Endpoint__c via a DataRaptor Extract

Given the specialist selects EFT_Enrolled__c = true
When the EFT enrollment step is displayed
Then the following fields are required: Bank_Name__c, Bank_Routing_Number__c (9 digits, validated), Bank_Account_Number__c (masked input, 4-17 digits), Bank_Account_Type__c ('Checking' or 'Savings')
And a pre-note verification checkbox 'Pre-Note_Verification_Sent__c' is presented

Given the specialist selects ERA_Delivery_Preference__c = 'SFTP'
When the ERA delivery step renders
Then SFTP_Endpoint__c becomes a required field with URL format validation
And ERA_Contact_Email__c is required for transmission failure notifications

Given the specialist completes and submits the OmniScript
When the DataRaptor Turbo Action saves the data
Then Provider__c fields are updated: EFT_Enrolled__c, Bank_Account_Last4__c (last 4 digits of entered account number only), Bank_Routing_Number__c, ERA_Delivery_Preference__c, SFTP_Endpoint__c, Payment_Preferences_Last_Updated__c = TODAY()
And Bank_Account_Number__c full value is NOT stored on Provider__c (stored only in encrypted custom object Provider_Bank_Detail__c with field-level encryption)
And a Provider_Change_Notification__c (PCN) record is created with Change_Type__c = 'Payment Preference Update' and linked to Provider__c

Given the PCN record is created
When Change_Type__c = 'Payment Preference Update'
Then the PCN is auto-assigned to the Provider Relations Specialist queue for 30-day monitoring per network agreement terms

-Bank account validation to call real-time ABA routing number verification API- (descoped: ABA validation moved to Phase 2 pending vendor contract; Phase 1 uses regex validation only)

Given a Provider Relations Specialist attempts to view a previously saved Provider_Bank_Detail__c record
When the record is opened
Then Bank_Account_Number__c displays as masked (****1234) and is not accessible without the 'Sensitive_Banking_Data_Viewer' Permission Set

## Solution

Build OmniStudio OmniScript 'Provider_Payment_Preferences' (Type: Provider, SubType: PaymentSetup) with DataRaptor Extract for pre-population and DataRaptor Turbo Action for save. OmniScript steps: (1) EFT Enrollment, (2) Bank Details (conditional on EFT=true), (3) ERA Delivery Preferences, (4) Review & Submit. Bank account masking implemented via OmniScript input type='password' with custom LWC masked display component on review step. Provider_Bank_Detail__c stores encrypted account number using Salesforce Shield Platform Encryption (Deterministic encryption on Account_Number_Encrypted__c). PCN record creation handled via DataRaptor Turbo Action secondary object mapping. Permission Set 'Sensitive_Banking_Data_Viewer' restricts FLS on Provider_Bank_Detail__c.Bank_Account_Number__c. OmniScript launched via Quick Action on Provider__c (Person Account) page layout.

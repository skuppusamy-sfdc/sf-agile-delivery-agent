## Description

As a Claims Operations Manager, I need Payment__c records to be automatically generated when a Claim__c reaches 'Adjudicated' status so that disbursement processing can begin without manual intervention and payment cycle times are reduced. This story extends the adjudication workflow established in INS-0523 to cover downstream payment initiation.

## Acceptance Criteria

Given a Claim__c record where Status__c transitions to 'Adjudicated' and Adjudication_Outcome__c = 'Approved'
When the Record-Triggered Flow fires on the Claim__c update
Then a Payment__c record is created with the following field mappings:
  - Payment__c.Claim__c = Claim__c.Id
  - Payment__c.Payment_Amount__c = Claim__c.Approved_Amount__c
  - Payment__c.Payee_Type__c = derived from Claim__c.Bill_Type__c ('Provider' or 'Member')
  - Payment__c.Payment_Status__c = 'Pending'
  - Payment__c.LOB__c = Claim__c.Line_of_Business__c
  - Payment__c.Payment_Method__c = defaulted to 'EFT' if Provider__c.EFT_Enrolled__c = true, else 'Check'

Given a Claim__c with Status__c = 'Adjudicated' and Adjudication_Outcome__c = 'Denied'
When the flow evaluates the outcome
Then NO Payment__c record is created and Claim__c.Payment_Suppressed__c = true

Given a Claim__c associated with a COB scenario where COB_Flag__c = true
When the Payment__c record is auto-created
Then Payment__c.COB_Hold__c = true and Payment__c.Payment_Status__c = 'On Hold - COB Review'

Given a Payment__c record is successfully created
When the process completes
Then a Platform Event (Payment_Initiated__e) is published with Claim_Id__c, Payment_Id__c, and Payment_Amount__c populated

-Given duplicate payment check runs asynchronously via batch- (superseded: duplicate check now runs synchronously in Flow decision element per INS-0541)

Given any unhandled exception occurs during Payment__c creation
When the flow fault path is triggered
Then Claim__c.Payment_Error_Flag__c = true, an error log record is created in Integration_Error_Log__c, and the Claims Operations Manager queue is notified via Salesforce Notification.

## Solution

Implement a Record-Triggered Flow on Claim__c (After Save, on Update) with entry criteria Status__c changed to 'Adjudicated'. Use Flow decision elements to branch on Adjudication_Outcome__c and COB_Flag__c. Create Payment__c via Flow Create Records action with field mappings as defined in ACs. Publish Payment_Initiated__e Platform Event via Flow action. Add fault path to set Claim__c.Payment_Error_Flag__c via a separate Update Records element and create Integration_Error_Log__c record. Apex invocable method used for EFT vs. Check payment method derivation logic to keep Flow maintainable. Permission Set Group 'Claims_Payment_Ops' required for manual override of Payment_Status__c.

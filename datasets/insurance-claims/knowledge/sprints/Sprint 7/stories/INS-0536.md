## Description

As a Claims Operations Manager, I need the adjudication engine to validate each Claim_Line__c CPT code against the applicable Fee_Schedule__c linked to the rendering Provider__c's Provider_Network_Agreement__c (PNA) so that allowed amounts are calculated accurately based on contracted rates rather than billed charges, ensuring correct payment disbursement and preventing overpayment.

## Acceptance Criteria

Given a Claim_Line__c with Claim_Line__c.CPT_Code__c populated and a rendering Provider__c with an active Provider_Network_Agreement__c (PNA__c.Status__c = 'Active')
When adjudication processing evaluates the Claim_Line__c
Then the system queries Fee_Schedule__c WHERE Fee_Schedule__c.PNA__c = PNA.Id AND Fee_Schedule__c.CPT_Code__c = Claim_Line__c.CPT_Code__c AND Fee_Schedule__c.Effective_Date__c <= Claim__c.Service_Date__c AND Fee_Schedule__c.Expiration_Date__c >= Claim__c.Service_Date__c

Given a matching Fee_Schedule__c record is found
When the allowed amount is calculated
Then Claim_Line__c.Allowed_Amount__c = Fee_Schedule__c.Contracted_Rate__c and Claim_Line__c.Fee_Schedule__c = matched Fee_Schedule__c.Id

Given a Claim_Line__c CPT code not present in the Fee_Schedule__c for the PNA
When the lookup returns no results
Then Claim_Line__c.Allowed_Amount__c defaults to Claim_Line__c.Billed_Amount__c * GSA__c.Default_Discount_Percentage__c (from the Group_Service_Agreement__c linked to the Claim__c) and Claim_Line__c.Fee_Schedule_Match__c = FALSE

Given a Provider__c with no active PNA (out-of-network provider)
When adjudication runs
Then Claim__c.Network_Status__c = 'Out-of-Network', Claim_Line__c.Allowed_Amount__c is calculated using the OON fee schedule (Fee_Schedule__c.Schedule_Type__c = 'OON_UCR') and Claim_Line__c.OON_Indicator__c = TRUE

-Given Facility__c (Business Account) claims, fee schedule lookup uses Facility__c.NPI__c instead of Provider__c.NPI__c for PNA matching-

Given a Fee_Schedule__c record with Fee_Schedule__c.Rate_Type__c = 'Percentage_of_Medicare'
When the contracted rate is calculated
Then Claim_Line__c.Allowed_Amount__c = CMS_Medicare_Rate__c.Rate__c * Fee_Schedule__c.Medicare_Percentage__c / 100, where CMS_Medicare_Rate__c is retrieved via the CMS integration DataRaptor

Given adjudication completes for all Claim_Line__c records
When all lines are processed
Then Claim__c.Total_Approved_Amount__c = SUM(Claim_Line__c.Allowed_Amount__c) WHERE Claim_Line__c.Status__c != 'Denied' and Claim__c.Status__c advances to 'Adjudicated'

## Solution

Apex service class (FeeScheduleLookupService) called from the ClaimsAdjudicationEngine per Claim_Line__c. Uses a SOQL query with date-effective filtering on Fee_Schedule__c. Custom Metadata Type (Fee_Schedule_Config__mdt) stores fallback logic rules per LOB. GSA default discount retrieved via a separate query on Group_Service_Agreement__c keyed by Claim__c.Group_Policy__c. CMS Medicare rate integration uses a DataRaptor Transform invoking a Named Credential callout to the CMS MPFS API. OON logic branched via a Decision element in the parent Adjudication OmniScript. All allowed amount writes performed in a bulkified Apex loop with governor limit safeguards (chunked in sets of 200). Fee schedule audit trail written to a custom Fee_Schedule_Audit__c object for compliance reporting.

## Description

As a Claims Operations Manager, I need Payment__c records to be automatically created when a Claim__c reaches 'Adjudicated' status so that disbursement workflows are triggered without manual intervention, reducing processing lag and human error in high-volume claim cycles.

## Acceptance Criteria

Given a Claim__c record where Status__c transitions to 'Adjudicated'
When the Record-Triggered Flow fires on the Claim__c update
Then a Payment__c record is created with Payment__c.Claim__c populated as the parent lookup, Payment__c.Status__c = 'Pending Disbursement', Payment__c.Amount__c = Claim__c.Approved_Amount__c, and Payment__c.Payment_Method__c defaulting to the value stored on the related Policy__c.Preferred_Payment_Method__c

Given the Claim__c is associated with a TPA-managed policy (Policy__c.TPA_Managed__c = true)
When the Payment__c record is created
Then Payment__c.Payee_Type__c must be set to 'TPA' and Payment__c.TPA_Account__c must be populated from Policy__c.TPA_Account__c

Given the Claim__c has one or more Claim_Line__c child records
When the Payment__c is created
Then each Claim_Line__c where Claim_Line__c.Line_Status__c = 'Approved' must be linked to the Payment__c via a Payment_Line__c junction record with Payment_Line__c.Approved_Amount__c mirroring Claim_Line__c.Approved_Amount__c

Given a Payment__c record is successfully created
When the creation completes
Then a Platform Event (Payment_Created__e) must be published with fields: Payment_Id__c, Claim_Id__c, Amount__c, and Timestamp__c to notify downstream ERA generation services

Given a Claim__c with Status__c = 'Denied'
When the flow evaluates the trigger condition
Then no Payment__c record should be created and the flow must exit gracefully with a logged Claim_Processing_Event__c record where Event_Type__c = 'Payment Skipped - Denied Claim'

## Solution

Implement a Record-Triggered Flow on Claim__c object firing After Update, with entry condition Claim__c.Status__c = 'Adjudicated'. Flow uses a Decision element to branch on Policy__c.TPA_Managed__c. A Create Records element generates the Payment__c record. A Loop element iterates approved Claim_Line__c records to create Payment_Line__c junction records. An Apex action (PaymentEventPublisher.cls) is invoked at flow conclusion to publish the Payment_Created__e Platform Event. Bulk-safe design using collection variables throughout. Custom metadata (Payment_Config__mdt) stores default payment method mappings per LOB.

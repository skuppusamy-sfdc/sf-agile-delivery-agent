## Description

**Bug Report**

**Summary:** When a Payment__c transitions to 'Disbursed' and the parent Claim__c has zero associated Claim_Line__c records (edge case occurring in manually adjusted direct-pay claims), the Conga CLM callout in the EOB generation flow receives a null line-item collection, causing the PDF render to fail silently — no error is logged, EOB.Status__c remains 'Generated' (incorrectly), and no PDF is attached to the EOB record.

**Steps to Reproduce:**
1. Create a Claim__c record with Claim__c.Direct_Pay__c = TRUE and no Claim_Line__c child records
2. Manually set Claim__c.Approved_Amount__c = $500.00 and transition Status__c to 'Adjudicated'
3. Observe Payment__c auto-created via INS-0577 flow with Amount__c = $500.00
4. Transition Payment__c.Status__c to 'Disbursed'
5. Observe EOB record created — EOB.Status__c = 'Generated' but no ContentVersion PDF is attached
6. No ClaimsProcessing_Error__c record is created

**Expected Result:** EOB.Status__c = 'Generation Failed', a ClaimsProcessing_Error__c is logged with Error_Code__c = 'MISSING_CLAIM_LINES', and the retry scheduled flow is triggered per INS-0578 AC7

**Actual Result:** EOB.Status__c = 'Generated' with no PDF attached and no error logged

**Environment:** Acme Insurance Full Sandbox
**Introduced in:** INS-0578 implementation
**Severity:** Medium — affects direct-pay claim workflows; no overpayment risk but compliance risk due to missing EOB documentation

## Acceptance Criteria

As per Description. Additionally: Given a Claim__c.Direct_Pay__c = TRUE with zero Claim_Line__c records, when EOB PDF generation is attempted, then the Apex callout method must validate that the Claim_Line__c collection is non-empty before invoking Conga CLM; if empty, EOB.Status__c must be set to 'Generation Failed', Error_Code__c = 'MISSING_CLAIM_LINES' must be logged on ClaimsProcessing_Error__c, and the retry Scheduled Flow from INS-0578 must be invoked.

## Solution

Root cause is in the Apex invocable method responsible for EOB generation (introduced in INS-0578): the Conga CLM callout is executed before a null/empty check on the Claim_Line__c list. Fix: add a guard clause in the Apex method — if (claimLines == null || claimLines.isEmpty()) throw a custom ClaimsProcessingException with error code 'MISSING_CLAIM_LINES'. The existing try-catch block in the method will then correctly set EOB.Status__c = 'Generation Failed', insert the ClaimsProcessing_Error__c record, and publish the retry Platform Event. For direct-pay claims, a separate Conga template (EOB_DirectPay_Template) will be configured that renders a simplified EOB without line-item detail — this template is selected when Claim__c.Direct_Pay__c = TRUE, resolving both the error and the long-term documentation gap. Unit test added for zero-line-item scenario.

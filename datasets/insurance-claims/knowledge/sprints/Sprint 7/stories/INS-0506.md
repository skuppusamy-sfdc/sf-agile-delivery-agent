## Description

As a Claims Operations Manager, I need the system to automatically generate a Payment__c record when a Claim__c reaches 'Adjudicated' status so that disbursement to providers or policyholders is initiated without manual intervention, reducing processing time and human error across all LOBs.

## Acceptance Criteria

Given a Claim__c record where Claim__c.Status__c transitions to 'Adjudicated' and Claim__c.Approved_Amount__c > 0,
When the Record-Triggered Flow fires on the Claim__c update,
Then a Payment__c record is created with Payment__c.Claim__c populated, Payment__c.Amount__c = Claim__c.Approved_Amount__c, Payment__c.Status__c = 'Pending', and Payment__c.Payment_Method__c derived from Provider__c.Preferred_Payment_Method__c on the associated Provider__c (Person Account).

Given Payment__c.Payment_Method__c = 'EFT',
When the Payment__c record is created,
Then Payment__c.Bank_Account_Number__c and Payment__c.Routing_Number__c must be populated from the linked Provider__c; if either field is blank, Payment__c.Status__c must be set to 'On Hold' and a Task is created for the Provider Relations Specialist.

Given Payment__c.Payment_Method__c = 'Check',
When the Payment__c record is created,
Then Payment__c.Payee_Mailing_Address__c must be validated against Provider__c.BillingAddress; if address is missing, Payment__c.Status__c = 'On Hold'.

Given a Claim__c with Claim__c.COB_Applied__c = true,
When the Payment__c record is generated,
Then Payment__c.Amount__c must reflect the post-COB approved amount stored in Claim__c.COB_Adjusted_Amount__c, not Claim__c.Approved_Amount__c.

Given any Payment__c record creation failure,
When the Apex trigger exception is caught,
Then an error log entry is written to the Integration_Error_Log__c object with fields: Error_Source__c = 'PaymentEngine', Related_Claim__c, Timestamp__c, and Error_Message__c, and a Platform Event (Payment_Failure_Event__e) is published for downstream alerting.

## Solution

Implement a Record-Triggered Flow on Claim__c (fires After Update, optimized for 'Status__c changes to Adjudicated'). Flow calls an invocable Apex method (PaymentCreationService.createPayment()) that handles Payment__c insertion, COB adjustment logic, and payment method validation. An Apex Trigger on Payment__c publishes Payment_Failure_Event__e via EventBus.publish() on any DML exception. Integration_Error_Log__c records written via a separate bulkified Apex utility class. Permission Set Group 'Claims_Payment_Ops' grants FLS on Payment__c sensitive fields (Bank_Account_Number__c, Routing_Number__c) restricted to Claims Operations Manager role.

## Description

As a Claims Operations Manager, I need a Payment Disbursement FlexCard embedded on the Claim__c record page that displays all associated Payment__c records with key financial metrics, so that I can quickly assess the disbursement status of a claim without navigating away or running reports.

The FlexCard must show payment totals, individual payment line details, check/EFT reference numbers, and a quick-action button to trigger a payment void or reissue OmniScript where applicable.

## Acceptance Criteria

Given a Claims Operations Manager views a Claim__c record page,
When the Payment Disbursement FlexCard loads,
Then all Payment__c records where Payment__c.Claim__c = current Claim__c.Id are displayed in a card list, showing Payment__c.Payment_Amount__c, Payment__c.Payment_Date__c, Payment__c.Payment_Method__c (EFT/Check), Payment__c.Reference_Number__c, and Payment__c.Status__c.

Given multiple payments exist on the claim,
When the FlexCard renders,
Then a summary row displays: Total Billed (sum of Claim_Line__c.Billed_Amount__c), Total Allowed (sum of Claim_Line__c.Allowed_Amount__c), Total Plan Paid (sum of Payment__c.Payment_Amount__c), and Outstanding Balance.

Given a Payment__c record with Payment__c.Status__c = 'Issued',
When the Claims Operations Manager clicks 'Void & Reissue',
Then the VoidReissuePayment OmniScript launches in a modal, pre-populated with Payment__c.Id and Claim__c.Id.

Given a Payment__c record with Payment__c.Status__c = 'Voided' or 'Cancelled',
When the FlexCard renders that record,
Then the row is visually distinguished with a strikethrough style and the 'Void & Reissue' action is hidden.

-Given the FlexCard loads, When the user is a read-only Claims Auditor, Then all action buttons are hidden based on Profile.- (Deprecated: replaced by Permission Set Group check on FlexCard conditional actions.)

## Solution

Build an OmniStudio FlexCard (PaymentDisbursementCard) with a DataRaptor Extract as the primary data source, querying Payment__c and aggregating Claim_Line__c financials via a SOQL DataRaptor. A child FlexCard state displays individual payment rows with conditional action visibility driven by the Payment__c.Status__c field value using FlexCard State conditions. The 'Void & Reissue' action launches the VoidReissuePayment OmniScript (built in INS-0596) via an OmniScript Launch action. The FlexCard is embedded on the Claim__c Lightning Record Page via the OmniStudio FlexCard component in App Builder. Visibility of action buttons is controlled by a FlexCard Conditional State referencing the running user's Permission Set Group membership via a DataRaptor Transform that queries PermissionSetAssignment.

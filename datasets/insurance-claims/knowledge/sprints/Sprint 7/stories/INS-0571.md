## Description

As a Provider Relations Specialist, I need the system to generate an ANSI X12 835 ERA file for each payment batch and deliver it electronically to participating providers, so that providers can auto-post payments in their practice management systems and reduce manual reconciliation effort. This story covers ERA file construction and secure SFTP delivery; provider portal display is handled in INS-0574.

## Acceptance Criteria

Given one or more Payment__c records with Payment_Status__c = 'Cleared' and Provider__c.ERA_Enrollment__c = true,
When the nightly ERA generation batch runs,
Then an X12 835 ERA file is produced per Provider__c.TIN__c grouping, stored as a ContentVersion attached to a new ERA_Batch__c record, with ERA_Batch__c.Status__c = 'Generated' and Record_Count__c populated.

Given ERA_Batch__c.Status__c = 'Generated',
When SFTP transmission is triggered via the DataIntegration layer using Named Credential 'Provider_SFTP_NC',
Then the ERA file is delivered to the provider's configured SFTP path (Provider__c.SFTP_Path__c), ERA_Batch__c.Status__c = 'Delivered', and Delivery_Timestamp__c is set.

Given Provider__c.ERA_Enrollment__c = false,
When the ERA batch processes that provider's payments,
Then no ERA file is generated and Payment__c records are flagged with ERA_Eligible__c = false and a paper EOB is queued instead.

-Given ERA delivery, the system should also fax a summary to the provider's billing office.- (Deprecated per provider communication policy update)

Given an SFTP delivery failure,
When the Named Credential callout throws an exception,
Then ERA_Batch__c.Status__c = 'Delivery_Failed', Failure_Reason__c is logged, and a retry is scheduled for 2 hours later up to 3 attempts before escalating to Integration_Error_Log__c.

## Solution

Implement ERAGenerationBatch.cls (Schedulable + Batchable) that queries cleared Payment__c records grouped by TIN. Use an X12835Builder Apex utility class to construct compliant 835 transaction sets. Store generated file as ContentVersion via ConnectApi or standard DML. SFTP delivery via HttpCallout using Named Credential. Retry logic implemented using Schedulable Apex chained from the batch finish() method. Custom Metadata (ERA_Config__mdt) stores delimiter settings, ISA/GS envelope identifiers per trading partner. ERA_Batch__c is a new custom object added to the Claims_Processor_PSG and Provider_Relations_PSG Permission Set Groups.

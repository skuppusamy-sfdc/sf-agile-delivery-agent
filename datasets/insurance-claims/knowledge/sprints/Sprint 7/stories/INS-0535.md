## Description

As a Claims Operations Manager, I need Agentforce to provide AI-generated natural language summaries of a claim's adjudication rationale and current payment status when a member or service agent queries a specific claim so that call handle time is reduced and members receive clear, plain-language explanations of their EOB without requiring specialist escalation.

## Acceptance Criteria

Given an authenticated member or service agent invokes the Agentforce 'Explain My Claim' action with a valid Claim__c.Claim_Number__c
When the Agentforce topic 'Claims_Inquiry' is triggered
Then the AI response includes: claim status (Claim__c.Status__c), total approved amount (Claim__c.Total_Approved_Amount__c), member responsibility (Claim__c.Member_Responsibility__c), and a plain-language explanation of any denied Claim_Line__c records referencing Claim_Line__c.Denial_Reason_Code__c

Given a Claim__c with Payment__c.Status__c = 'Disbursed'
When the member asks 'When was my payment sent?'
Then Agentforce returns Payment__c.Disbursement_Date__c and Payment__c.Payment_Method__c in a conversational response

Given a Claim__c with Claim__c.SIU_Flagged__c = TRUE
When Agentforce attempts to summarize the claim
Then the AI response is suppressed and a human handoff message is returned: 'This claim requires specialist review. Connecting you to a Claims Specialist.'

Given the Agentforce response is generated
When the interaction is complete
Then the conversation is logged as an AgentWork record linked to the Claim__c and a Claim__c internal note is appended with the AI summary text for audit trail purposes

Given a member query for a Claim__c not associated with their Policy__c
When Agentforce evaluates data access
Then the response is denied with 'I was unable to locate that claim on your account' and no Claim__c data is returned (data masking enforced)

## Solution

Configure Agentforce Agent with a custom Topic 'Claims_Inquiry' and Actions mapped to Apex invocable methods (ClaimsAISummaryAction) that retrieve Claim__c, Claim_Line__c, and Payment__c data. Use Einstein Generative AI prompt template ('ClaimAdjudicationSummary_v1') with grounded data injection from the Apex action response. SIU suppression implemented as a pre-action Flow check on Claim__c.SIU_Flagged__c. Data security enforced via Salesforce record sharing rules (Claim__c shared to Policy__c owner) — Apex runs in user mode. AgentWork logging via standard Omni-Channel AgentWork object. Prompt template stored in Prompt Builder with LOB-specific variants managed via Custom Metadata.

## Description

**Bug Description:**
The duplicate disbursement validation logic incorrectly flags legitimate COB secondary claim payments as duplicates, placing them in 'On Hold - Duplicate' status and blocking disbursement.

**Steps to Reproduce:**
1. Create a Claim__c record with COB_Flag__c = true and COB_Sequence__c = 'Secondary'.
2. Ensure a prior Payment__c exists for the same member and same date of service under a different Claim__c (the primary payer claim).
3. Adjudicate the secondary claim — set Claim__c.Status__c = 'Adjudicated', Adjudication_Outcome__c = 'Approved'.
4. Observe the auto-created Payment__c record.

**Expected Result:**
Payment__c.Payment_Status__c = 'Pending' and COB_Hold__c = true (COB review hold, NOT duplicate hold). The duplicate check should exclude records where COB_Flag__c = true.

**Actual Result:**
Payment__c.Payment_Status__c = 'On Hold - Duplicate' because the duplicate detection Apex trigger (PaymentDuplicateCheck_Trigger) matches on Member__c + Date_of_Service__c + Billed_Amount__c without excluding COB secondary claims, causing false positive duplicate flags.

**Impact:** Medium — affects all COB secondary claim payments. Currently 47 Payment__c records incorrectly suppressed in UAT environment.

**Root Cause (preliminary):** PaymentDuplicateCheck_Trigger handler class method `checkForDuplicates()` SOQL query does not include a filter for `COB_Flag__c = false` on the queried Payment__c records.

## Acceptance Criteria

As per Description. Fix verified when COB secondary claim payments are no longer flagged as duplicates. Regression test: non-COB duplicate payments must still be correctly identified and held.

## Solution

Update PaymentDuplicateCheck_TriggerHandler.cls method `checkForDuplicates()` to add `AND COB_Flag__c = false` to the SOQL query that retrieves existing Payment__c records for duplicate comparison. Additionally, add a secondary condition: if the incoming Payment__c.COB_Sequence__c = 'Secondary', bypass the duplicate check entirely and set COB_Hold__c = true per standard COB workflow. Add unit test coverage for COB secondary scenario in PaymentDuplicateCheck_Test.cls. Deploy fix via change set to UAT for regression validation before production push.

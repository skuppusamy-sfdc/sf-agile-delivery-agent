## Description

As a Claims Operations Manager, I need Agentforce to automatically surface recommended payment hold reasons when a Claim__c record has Claim__c.SIU_Referral__c = true so that claims analysts receive AI-assisted guidance on appropriate hold classifications, reducing SIU-related payment leakage and ensuring compliance with fraud investigation protocols.

## Acceptance Criteria

Given a Claim__c record where SIU_Referral__c is set to true
When the Claims Analyst opens the Claim__c record page
Then the Agentforce panel displays a contextual prompt: 'This claim has been referred to SIU. Recommended hold reasons based on claim profile:' followed by up to 3 AI-generated hold reason suggestions ranked by confidence score

Given the Agentforce panel displays hold reason suggestions
When the analyst selects a suggested hold reason
Then Claim__c.Payment_Hold_Reason__c is populated with the selected value and Claim__c.Payment_Hold__c = true is set automatically

Given Claim__c.Payment_Hold__c = true
When any associated Payment__c record has Status__c = 'Pending'
Then Payment__c.Status__c is updated to 'On Hold' and Payment__c.Hold_Reason__c mirrors Claim__c.Payment_Hold_Reason__c

Given the Agentforce suggestion is generated
When the AI model evaluates the claim
Then the model considers: Claim__c.LOB__c, Claim__c.Total_Billed_Amount__c, Provider__c.SIU_History__c, and historical SIU outcomes from the Einstein model training dataset

Given the analyst disagrees with all AI suggestions
When the analyst selects 'Override — Enter Manual Reason'
Then a free-text field is enabled for Claim__c.Payment_Hold_Reason__c and Agentforce_Override__c = true is logged for model feedback

Given Agentforce is unavailable (service degradation)
When the Claim__c record page loads with SIU_Referral__c = true
Then a fallback static picklist of standard hold reasons is displayed in place of AI suggestions and Agentforce_Fallback__c = true is logged

## Solution

Configure Agentforce Agent Action using Einstein Generative AI with a custom Prompt Template (SIU_Hold_Reason_Prompt) that injects Claim__c context fields via merge fields. The Agent Topic is scoped to 'Claims SIU Review'. A Record-Triggered Flow on Claim__c fires when SIU_Referral__c = true to invoke the Agentforce action via Apex. An LWC (siu_hold_recommendation) renders suggestions in the utility bar with accept/override controls. Override feedback is captured in an Agentforce_Feedback__c custom object for model retraining. Fallback handled via a Custom Setting (Agentforce_Config__c.Is_Available__c) checked before invoking the AI action.

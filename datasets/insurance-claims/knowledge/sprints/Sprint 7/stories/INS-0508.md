## Description

As a Claims Operations Manager, I need the system to generate an ANSI X12 835 (ERA) transaction file for each batch of EFT payments processed on a given business day and transmit it to the provider's designated SFTP endpoint, so that providers can automatically reconcile payments against their billing systems and reduce manual remittance inquiries to Provider Relations.

## Acceptance Criteria

Given one or more Payment__c records where Payment__c.Status__c = 'Processed' and Payment__c.Payment_Method__c = 'EFT' and Payment__c.ERA_Included__c = false,
When the nightly scheduled Apex batch job runs at 20:00 CT,
Then all qualifying Payment__c records are grouped by Provider__c.TIN__c and a single 835 ERA file is generated per TIN.

Given the ERA file is generated,
When the SFTP transmission is attempted using Provider__c.SFTP_Endpoint__c,
Then the file is transmitted and Payment__c.ERA_Included__c = true, Payment__c.ERA_File_Name__c is stamped with the generated filename (format: ERA_[TIN]_[YYYYMMDD].835), and Payment__c.ERA_Transmission_Date__c = TODAY().

Given SFTP transmission fails (timeout or authentication error),
When the exception is caught in the batch Apex finish() method,
Then an Integration_Error_Log__c record is created, Payment__c.ERA_Transmission_Status__c = 'Failed', and a Case is auto-created and assigned to the Provider Relations queue with Subject = 'ERA Transmission Failure - [Provider NPI]'.

Given the ERA file content,
When validated against ANSI X12 835 schema,
Then the file must include: ISA/GS envelope segments, ST-SE transaction set, CLP segments per Claim__c, SVC segments per Claim_Line__c with CPT_Code__c and allowed amounts, and CAS segments reflecting adjustment reason codes from Claim_Line__c.Adjustment_Reason_Code__c.

Given Provider__c.SFTP_Endpoint__c is null,
When the batch processes that provider's payments,
Then those Payment__c records are skipped, Payment__c.ERA_Transmission_Status__c = 'No SFTP Configured', and the provider is added to a daily exception report emailed to the Claims Operations Manager.

## Solution

Scheduled Apex Batch class (ERAGenerationBatch) implementing Database.Batchable and Database.Stateful. Start() queries eligible Payment__c records; execute() groups by TIN and constructs 835 file content using a dedicated ERAFileBuilder Apex utility class. Finish() calls SFTPTransmissionService (Apex HTTP callout to internal SFTP proxy microservice) per provider TIN. Named Credential 'SFTP_ERA_Proxy' stores endpoint and auth. File content stored temporarily in ContentVersion before transmission. Integration_Error_Log__c and Case creation handled in finish() via Platform Event (ERA_Failure_Event__e) to decouple error handling. Custom Metadata Type (ERA_Config__mdt) stores batch size, schedule time, and file naming convention.

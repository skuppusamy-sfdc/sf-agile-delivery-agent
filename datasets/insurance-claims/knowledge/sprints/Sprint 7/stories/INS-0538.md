## Description

As a Claims Operations Manager, I need Explanation_of_Benefits__c (EOB) records to be automatically generated and delivered to claimants via DocuSign e-delivery and the member portal upon claim payment issuance so that policyholders receive timely, compliant EOB documentation without manual PDF production. This story extends INS-0537 by consuming the Payment_Initiated__e event to trigger EOB creation.

## Acceptance Criteria

Given a Payment__c record where Payment_Status__c transitions to 'Issued'
When the Record-Triggered Flow on Payment__c fires
Then an Explanation_of_Benefits__c record is created with EOB_Status__c = 'Draft', linked to Claim__c, Policy__c, and Payment__c via lookup fields, and EOB_Generated_Date__c = TODAY()

Given the Explanation_of_Benefits__c record is in 'Draft' status
When Conga CLM template merge is invoked via Apex (CongaEOBMergeService.cls)
Then the EOB PDF is generated with all Claim_Line__c details including CPT_Code__c, ICD10_Code__c, Billed_Amount__c, Approved_Amount__c, Denial_Reason_Code__c, and Provider__c.Name

Given the EOB PDF is generated
When the claimant's Policy__c.Delivery_Preference__c = 'Electronic'
Then a DocuSign envelope is sent to the email on Policy__c.Policyholder_Email__c and EOB_Status__c updates to 'Sent'

Given the claimant's Policy__c.Delivery_Preference__c = 'Portal'
When the EOB PDF is finalized
Then the document is attached to the Explanation_of_Benefits__c record as a Salesforce File and made visible in the member portal via a ContentDocumentLink with Visibility = 'AllUsers'

Given DocuSign delivery confirmation is received
When the DocuSign status webhook fires
Then Explanation_of_Benefits__c.EOB_Status__c = 'Delivered' and EOB_Delivery_Timestamp__c is populated

Given an EOB delivery failure (DocuSign bounce or invalid email)
When the failure event is received
Then EOB_Status__c = 'Delivery_Failed', a Case is auto-created linked to Policy__c with Subject = 'EOB Delivery Failure - ' + Claim__c.Name, and the Claims Operations queue is notified via in-app notification

## Solution

Record-Triggered Flow on Payment__c (after-save, Status__c = 'Issued') creates Explanation_of_Benefits__c via Apex invocable (EOBRecordBuilder.cls). Conga CLM merge is triggered asynchronously via a Queueable Apex class (CongaEOBMergeService.cls) using Conga's REST API. DocuSign envelope creation handled by DocuSignEnvelopeService.cls using the DocuSign eSignature for Salesforce managed package. Delivery preference branching handled within the Queueable. Inbound DocuSign webhook processed by DocuSignCallbackHandler.cls (REST resource). Portal visibility managed via ContentDocumentLink insertion in Apex. LWC component 'eobDocumentViewer' surfaced on the member Experience Cloud portal page.

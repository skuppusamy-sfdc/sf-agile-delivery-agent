## Description

As a Claims Operations Manager, I need the system to automatically generate and transmit an 835 ERA file to the rendering provider or facility after a Payment__c record is posted, so that providers receive standardized electronic remittance data enabling automated reconciliation without manual intervention.

Extends INS-0514 — ERA generation must reference the same adjudication data used for EOB production. ERA files must conform to ANSI X12 835 transaction set standards and be transmitted via the provider's preferred clearinghouse endpoint stored on the Provider__c or Facility__c record.

## Acceptance Criteria

Given a Payment__c record is created with Payment_Status__c = 'Posted'
When the associated Claim__c.Claim_Type__c is 'Professional' or 'Institutional'
Then an ERA batch job is triggered and ERA_File__c record is created with ERA_Status__c = 'Pending Transmission'

Given the ERA_File__c record is in 'Pending Transmission' status
When the nightly scheduled Apex batch (ERA835BatchJob) executes
Then an 835 transaction file is constructed referencing Payment__c.Payment_Amount__c, Claim__c.Claim_Number__c, and each Claim_Line__c.Allowed_Amount__c and Claim_Line__c.Adjustment_Reason_Code__c

Given the provider is a Person Account (Provider__c)
When the ERA file is ready for transmission
Then the file is routed to Provider__c.Clearinghouse_Endpoint__c and ERA_File__c.Transmission_Method__c = 'EDI'

Given the provider is a Facility__c (Business Account)
When ERA transmission completes successfully
Then ERA_File__c.ERA_Status__c = 'Transmitted' and ERA_File__c.Transmission_Timestamp__c is populated

-Given ERA transmission uses SFTP to legacy clearinghouse- (deprecated: all new transmissions use API-based clearinghouse per architecture decision ADR-0023)

Given an ERA transmission failure occurs
When the clearinghouse returns an error code
Then ERA_File__c.ERA_Status__c = 'Transmission Failed', ERA_File__c.Failure_Reason__c is populated, and a high-priority Case is auto-created and assigned to the Claims Operations queue

## Solution

Build a Record-Triggered Flow on Payment__c (fires on creation when Payment_Status__c = 'Posted') to publish a Platform Event (ERA_Payment_Posted__e) with Payment Id payload. An Apex Platform Event trigger subscribes and enqueues an ERA835BuilderQueueable job. The Queueable class assembles the 835 file structure using a custom ERA835Builder Apex utility class, persists the output as a ContentVersion linked to the ERA_File__c record, then invokes a Named Credential-based REST callout to the clearinghouse API. A Scheduled Apex job retries failed transmissions every 4 hours up to 3 attempts. All transmission logs are written to ERA_Transmission_Log__c for audit.

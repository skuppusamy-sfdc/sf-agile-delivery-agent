## Description

As a Claims Operations Manager, I need Payment__c records to be automatically created and submitted to the EFT disbursement queue when an EOB is finalized, so that providers and members receive accurate, timely payments without manual intervention and we maintain a full audit trail within Salesforce. Extends INS-0569.

## Acceptance Criteria

Given an Explanation_of_Benefits__c record with EOB_Status__c = 'Generated' and Claim__c.Payment_Method__c = 'EFT',
When the EOB_Ready__e Platform Event is consumed by the PaymentOrchestration subscriber,
Then a Payment__c record is created with Payment_Status__c = 'Pending', Payment_Type__c = 'EFT', Payee_Type__c derived from Claim__c.Payee__c (either Provider__c or Policyholder__c lookup populated), Amount__c = EOB.Plan_Paid__c, and Claim__c lookup set.

Given Payment__c.Payment_Status__c = 'Pending' and Payee_Type__c = 'Provider',
When the nightly scheduled Apex batch (PaymentDisbursementBatch.cls) runs,
Then Payment__c records are grouped by Provider__c.TIN__c, an outbound EFT file is generated in NACHA format, transmitted to the payment clearinghouse via Named Credential 'EFT_Clearinghouse_NC', and Payment__c.Payment_Status__c updated to 'Submitted' with Submission_Timestamp__c populated.

Given a successful EFT transmission acknowledgment received via inbound REST callback,
When the EFT_Acknowledgment__e Platform Event is published by the integration layer,
Then Payment__c.Payment_Status__c = 'Cleared', Payment__c.Clearinghouse_Trace_Number__c is populated, and Claim__c.Payment_Cleared__c = true.

Given Payment__c.Payment_Type__c = 'Check' (fallback when EFT banking info is absent),
When the batch processes the record,
Then Payment__c is routed to the Print_Queue__c object with Check_Mailing_Address__c copied from Provider__c.BillingAddress and Payment__c.Payment_Status__c = 'Check_Queued'.

Given any EFT transmission failure,
When the clearinghouse returns a rejection code,
Then Payment__c.Payment_Status__c = 'Failed', Failure_Reason__c is populated with the clearinghouse error code, and a high-priority Case is auto-created assigned to the 'Claims_Payment_Ops' queue with Subject = 'EFT Payment Failure - [Payment__c.Name]'.

## Solution

Build a Platform Event subscriber (Apex trigger on EOB_Ready__e) to create Payment__c records. Implement PaymentDisbursementBatch.cls as a Schedulable + Batchable Apex class with batch size 50 to respect callout limits. Use a Named Credential for EFT clearinghouse authentication. NACHA file generation handled by NACHAFileBuilder utility class. Inbound acknowledgment processed via a REST Apex endpoint exposed through a Connected App. Store clearinghouse configuration in Custom Metadata (EFT_Config__mdt) per LOB. All DML operations wrapped in Savepoint/Rollback pattern to ensure atomicity. Add Payment__c to the Claims_Processor_PSG Permission Set Group.

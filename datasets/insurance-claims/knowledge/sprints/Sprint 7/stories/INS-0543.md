## Description

**Bug Report — INS-0543**

**Environment:** Acme Insurance Full Sandbox (SF-FULL-01)
**Discovered by:** Emily Watson, Compliance Officer, during EOB compliance review

**Steps to Reproduce:**
1. Create a Claim__c record with Status__c = 'Adjudicated' and an associated Payment__c with Payment_Status__c = 'Issued'.
2. Ensure at least one Claim_Line__c has ICD10_Code__c = 'M54.5' (includes a decimal point).
3. Observe the EOB generation process triggered by INS-0538 automation.
4. Check the Explanation_of_Benefits__c record and the Conga merge job logs.

**Expected Result:** EOB PDF is generated successfully with ICD-10 code 'M54.5' rendered correctly in the diagnosis section.

**Actual Result:** The Conga CLM merge job completes with status 'Success' in the Conga activity log, but no PDF is attached to the Explanation_of_Benefits__c record. EOB_Status__c remains 'Draft' indefinitely. No error is surfaced to the user or logged in Salesforce debug logs. Issue is reproducible with any ICD-10 code containing a decimal (e.g., 'E11.9', 'J18.9') and does not occur with codes lacking decimals.

**Hypothesis:** The Conga template XML likely has an unescaped merge field for ICD10_Code__c that treats the decimal as a field path separator, causing a silent merge failure. Alternatively, the DataRaptor feeding the Conga template may be stripping or misformatting the decimal during JSON serialization.

## Acceptance Criteria

As per Description. Fix verified when EOB PDFs are generated successfully for Claim_Line__c records with ICD-10 codes containing decimal points (tested with M54.5, E11.9, J18.9, Z00.00). EOB_Status__c must progress to 'Sent' or 'Delivered' as expected per INS-0538 flow.

## Solution

Investigate Conga CLM template (EOB_Template_v3) for unescaped ICD10_Code__c merge field references. Update the template to wrap the ICD10_Code__c merge field in a CDATA block or apply Conga's string-escape function. Additionally, review CongaEOBMergeService.cls to ensure ICD10_Code__c values are passed as String type (not parsed as numeric) in the parameter map. Update DataRaptor Extract 'DR_Extract_EOBLineDetails' to enforce String data type on ICD10_Code__c output node. Add an Apex test in CongaEOBMergeServiceTest.cls with decimal-containing ICD-10 codes. Validate fix in sandbox with full regression of INS-0538 acceptance criteria.

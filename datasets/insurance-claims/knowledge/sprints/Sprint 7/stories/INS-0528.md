## Description

**Bug Report**

**Summary:** When a Claim__c has Claim_Line__c records that match more than one active Fee_Schedule__c record (e.g., overlapping effective date ranges for the same CPT code and provider type), the payment calculation engine applies the fee schedule with the highest reimbursement rate rather than the most recently effective one, resulting in overpayment.

**Steps to Reproduce:**
1. Create a Fee_Schedule__c record: CPT_Code__c = '99213', Provider_Type__c = 'PCP', Effective_Date__c = 01/01/2024, End_Date__c = 12/31/2024, Allowed_Amount__c = $85.00.
2. Create a second Fee_Schedule__c record: CPT_Code__c = '99213', Provider_Type__c = 'PCP', Effective_Date__c = 07/01/2024, End_Date__c = 12/31/2024, Allowed_Amount__c = $92.00.
3. Create a Claim__c with Service_Date__c = 08/15/2024 and a Claim_Line__c with CPT_Code__c = '99213'.
4. Trigger adjudication by setting Claim__c.Status__c = 'Ready for Adjudication'.
5. Observe Claim_Line__c.Allowed_Amount__c = $92.00 (highest rate) instead of expected $85.00 (most recently effective as of service date, with earlier start date taking precedence per business rule).

**Expected Result:** Claim_Line__c.Allowed_Amount__c = $85.00. Fee schedule selection must use the record with the earliest Effective_Date__c that is still valid as of Claim__c.Service_Date__c, per Fee Schedule Priority Rule FSR-2024-03.

**Actual Result:** Claim_Line__c.Allowed_Amount__c = $92.00 (highest Allowed_Amount__c selected).

**Environment:** Acme Insurance UAT Sandbox
**Frequency:** Reproducible 100% when overlapping fee schedules exist for same CPT + Provider Type combination.
**Impact:** Financial — overpayment risk across all claims in adjudication queue with overlapping fee schedules (~340 Claim__c records identified in UAT).

## Acceptance Criteria

As per Description. Fix must ensure Fee_Schedule__c selection SOQL query in FeeScheduleSelector Apex class orders by Fee_Schedule__c.Effective_Date__c ASC and limits to 1 record, not by Fee_Schedule__c.Allowed_Amount__c DESC.

## Solution

Root cause identified in FeeScheduleSelector.getApplicableSchedule() Apex class: ORDER BY clause incorrectly uses 'Allowed_Amount__c DESC LIMIT 1' instead of 'Effective_Date__c ASC LIMIT 1'. Fix: Update SOQL in FeeScheduleSelector to: SELECT Id, Allowed_Amount__c FROM Fee_Schedule__c WHERE CPT_Code__c = :cptCode AND Provider_Type__c = :providerType AND Effective_Date__c <= :serviceDate AND (End_Date__c = null OR End_Date__c >= :serviceDate) ORDER BY Effective_Date__c ASC LIMIT 1. Add unit test FeeScheduleSelectorTest.testOverlappingScheduleSelection() with 95%+ coverage. Deploy fix to UAT for regression testing against the 340 identified Claim__c records. Post-fix, trigger re-adjudication batch for affected claims.

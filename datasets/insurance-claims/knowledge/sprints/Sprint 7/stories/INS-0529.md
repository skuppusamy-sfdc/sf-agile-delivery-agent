## Description

As a Claims Operations Manager, I need Payment__c records to be automatically created when a Claim__c is moved to 'Adjudicated' status so that disbursement processing can begin without manual intervention, reducing cycle time and human error in the payment initiation workflow.

## Acceptance Criteria

Given a Claim__c record where Status__c transitions to 'Adjudicated' and Claim__c.Total_Approved_Amount__c > 0
When the Record-Triggered Flow fires on the Claim__c update
Then a Payment__c record is created with Payment__c.Status__c = 'Pending', Payment__c.Payment_Amount__c = Claim__c.Total_Approved_Amount__c, Payment__c.Claim__c = triggering Claim__c Id, and Payment__c.Payment_Method__c defaulting to 'EFT'

Given a Claim__c with LOB__c = 'Workers Comp'
When Payment__c is auto-created
Then Payment__c.Payment_Type__c must be set to 'Indemnity' and Payment__c.TPA_Routed__c = TRUE if Claim__c.TPA__c is populated

Given a Claim__c where Status__c = 'Adjudicated' but Claim__c.Total_Approved_Amount__c = 0 or NULL
When the flow evaluates the trigger condition
Then NO Payment__c record is created and a Claim__c internal note (Task) is logged: 'Zero-dollar adjudication — payment suppressed'

Given duplicate prevention logic
When a Claim__c is re-saved without a status change
Then no duplicate Payment__c records are created (idempotency check on Payment__c.Claim__c + Payment__c.Status__c = 'Pending')

Given a successful Payment__c creation
When the record is committed
Then a Platform Event (Payment_Initiated__e) is published with payload: claimId, paymentId, paymentAmount, paymentMethod for downstream ERA integration consumption

## Solution

Implement a Record-Triggered Flow on Claim__c firing AFTER UPDATE, with entry condition Claim__c.Status__c CHANGED TO 'Adjudicated'. Use a Decision element to branch on Total_Approved_Amount__c > 0. Create Payment__c via a Create Records element. Publish Payment_Initiated__e Platform Event via a separate flow action. Add an Apex-based duplicate guard (before insert trigger on Payment__c) to enforce one active pending payment per claim. Permission Set Group 'Claims_Payment_Ops' must include access to Payment__c create/edit FLS.

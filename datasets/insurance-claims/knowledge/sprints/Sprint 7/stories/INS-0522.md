## Description

As a Claims Operations Manager, I need Payment__c records to be automatically created when a Claim__c reaches 'Adjudicated' status so that disbursement processing can begin without manual intervention, reducing cycle time and human error in the payment workflow. This story extends the adjudication logic established in INS-0492.

## Acceptance Criteria

Given a Claim__c record where Status__c transitions to 'Adjudicated' and Claim__c.Total_Approved_Amount__c > 0,
When the Record-Triggered Flow fires on the Claim__c object post-save,
Then a Payment__c record is created with Payment__c.Claim__c = triggering Claim__c.Id, Payment__c.Status__c = 'Pending', Payment__c.Amount__c = Claim__c.Total_Approved_Amount__c, Payment__c.Payment_Method__c derived from Policy__c.Preferred_Payment_Method__c, and Payment__c.Due_Date__c = TODAY() + 5 business days.

Given a Claim__c with Claim__c.LOB__c = 'Workers Compensation',
When the Payment__c record is auto-created,
Then Payment__c.Payment_Type__c must be set to 'EFT' regardless of Policy__c.Preferred_Payment_Method__c and a Task is created for the assigned Claims Operations Manager to verify banking details.

Given a Claim__c.COB_Indicator__c = true,
When adjudication is approved,
Then the system must NOT auto-create a Payment__c record and instead set Claim__c.Status__c = 'Pending COB Review' and assign to the COB queue.

Given a duplicate Payment__c already exists for the same Claim__c.Id with Payment__c.Status__c IN ('Pending', 'Issued'),
When the flow attempts to create a new Payment__c,
Then creation is blocked, a Platform Event (Payment_Duplicate_Alert__e) is published, and Claim__c.Payment_Hold_Reason__c = 'Duplicate Payment Detected'.

-Given Payment__c.Payment_Method__c = 'Check', auto-print job triggers within 1 hour- (Deprecated: batch print job handled by INS-0527)

Given any failure in Payment__c creation,
When the flow encounters a fault path,
Then Claim__c.Status__c reverts to 'Adjudicated - Payment Error', an email alert is sent to the Claims Operations Manager, and the error is logged in a custom Payment_Error_Log__c record.

## Solution

Implement a Record-Triggered Flow on Claim__c object (After Save, on Status__c change to 'Adjudicated'). Use Decision elements to branch on LOB__c, COB_Indicator__c, and duplicate check via SOQL (Get Records on Payment__c filtered by Claim__c = recordId AND Status__c IN ('Pending','Issued')). Auto-create Payment__c via Create Records element. Publish Platform Event Payment_Duplicate_Alert__e via platform event element for duplicate scenario. Fault path uses Update Records to set Claim__c.Status__c and invokes a Send Email action. Business day calculation handled by an invocable Apex method BusinessDayCalculator.addBusinessDays(). Deploy with associated Permission Set Group 'Claims_Payment_Automation_PSG'.

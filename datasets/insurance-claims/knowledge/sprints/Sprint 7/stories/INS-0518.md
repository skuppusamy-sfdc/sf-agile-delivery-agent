## Description

**Bug Report — Payment Disbursement Status Incorrect on Workday HTTP 202 Response**

**Steps to Reproduce:**
1. Create a Claim__c record and advance Status__c to 'Adjudicated' with Total_Approved_Amount__c = $500.00
2. Confirm Payment__c record is created with Payment_Status__c = 'Approved'
3. Observe the Workday disbursement callout fires via the PaymentDisbursementQueueable class
4. Workday sandbox returns HTTP 202 (Accepted — async processing) instead of HTTP 200
5. Observe Payment__c.Payment_Status__c is immediately set to 'Disbursed' and Payment__c.Disbursement_Date__c is populated

**Expected Behavior:**
When Workday returns HTTP 202, the payment should remain in Payment_Status__c = 'Pending Confirmation' until the async Workday callback confirms final posting. Disbursement_Date__c should only be set upon receipt of the confirmation webhook.

**Actual Behavior:**
HTTP 202 is treated identically to HTTP 200, causing premature 'Disbursed' status. This results in ERA files (INS-0515) being generated before the payment is actually posted in Workday, leading to provider reconciliation errors.

**Environment:** Acme Insurance Full Sandbox (SF-SANDBOX-02)
**Frequency:** Reproducible 100% of time when Workday async mode is active
**Impact:** High — ERA files transmitted with unconfirmed payment data; potential regulatory reporting errors

## Acceptance Criteria

As per Description.

Given Workday callout returns HTTP 202
When the Apex callout response is processed
Then Payment__c.Payment_Status__c = 'Pending Confirmation' and Disbursement_Date__c remains null

Given Payment__c.Payment_Status__c = 'Pending Confirmation'
When the Workday confirmation webhook is received
Then Payment__c.Payment_Status__c = 'Disbursed' and Disbursement_Date__c = confirmation timestamp

## Solution

Update PaymentDisbursementQueueable.handleCalloutResponse() to differentiate HTTP 200 vs HTTP 202 response codes. For 202, set Payment_Status__c = 'Pending Confirmation' and store Workday's async correlation ID in Payment__c.External_Payment_Reference__c. Expose a new REST endpoint (PaymentConfirmationWebhookController) to receive Workday's async callback, match on External_Payment_Reference__c, and update Payment_Status__c to 'Disbursed'. Add a Scheduled Apex job to flag Payment__c records stuck in 'Pending Confirmation' for more than 24 hours with Payment_Status__c = 'Confirmation Timeout' and alert the Claims Operations queue.

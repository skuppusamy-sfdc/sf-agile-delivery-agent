## Description

As a Claims Operations Manager, I need a FlexCard on the Policyholder 360 page that displays all Explanation_of_Benefits__c records associated with a member's Policy__c, with expandable line-level Claim_Line__c detail, so that I can quickly review a member's full payment history without navigating away from the member record.

## Acceptance Criteria

Given a user with the 'Claims_Operations' Permission Set Group navigates to a Person Account (policyholder) record
When the Policyholder 360 Lightning page loads
Then the 'EOB History' FlexCard is visible and displays all EOB__c records where EOB__c.Policy__c = current Policy__c.Id, ordered by EOB__c.Service_Date__c descending

Given the EOB History FlexCard is rendered
When EOB records are present
Then each card row displays EOB__c.Claim__c (hyperlinked), EOB__c.Service_Date__c, EOB__c.Total_Billed__c, EOB__c.Total_Allowed__c, EOB__c.Member_Responsibility__c, and EOB__c.Status__c as a colored badge (green='Delivered', yellow='Draft', red='Generation Failed')

Given a user clicks the expand chevron on an EOB card row
When the child DataRaptor Extract fires
Then all related Claim_Line__c records are displayed inline showing Claim_Line__c.CPT_Code__c, Claim_Line__c.Service_Description__c, Claim_Line__c.Billed_Amount__c, Claim_Line__c.Allowed_Amount__c, and Claim_Line__c.Denial_Reason__c

Given the EOB FlexCard is loaded
When there are no EOB__c records for the policy
Then an empty state message 'No EOB records found for this member' is displayed

-FlexCard must support PDF inline preview via ContentDocument viewer- (deferred to Phase 2 per UX team decision)

## Solution

OmniStudio FlexCard 'EOBHistoryCard' built with a DataRaptor Extract as the primary data source, querying EOB__c and related Policy__c. Child DataRaptor Extract for Claim_Line__c is triggered on expand action via a conditional block. Badge coloring implemented using FlexCard state CSS classes mapped to EOB__c.Status__c values. FlexCard is embedded in the Policyholder 360 Lightning App Builder page via the OmniStudio FlexCard component. DataRaptor inputs receive the Policy__c.Id from the page context via merge field. Deployed via OmniStudio Deployment package.

## Description

As a Broker Relations Coordinator, I need a real-time commission dashboard within the Agent Portal so that agents and brokers can view earned, pending, and paid commissions broken down by LOB, policy effective date, and payment cycle without requiring manual report requests from the back office.

This story extends INS-0989 (Agent Portal base layout) to add a dedicated Commission Tracking tab surfacing data from Commission_Transaction__c and Payment__c records linked to the agent's Producer_Code__c.

## Acceptance Criteria

Given an authenticated agent user with the 'Agent_Portal_User' Permission Set Group assigned,
When they navigate to the Commission Dashboard tab,
Then a FlexCard displays a summary tile showing: YTD Earned (Commission_Transaction__c.Earned_Amount__c sum), Pending Approval (Commission_Transaction__c.Status__c = 'Pending'), and Last Payment Date (Payment__c.Payment_Date__c) filtered by Agent__c = current user's linked Contact.

Given the agent selects a date range filter,
When the filter is applied,
Then the underlying DataRaptor Transform re-queries Commission_Transaction__c records where Effective_Date__c falls within the selected range and refreshes all summary tiles without a full page reload.

Given a commission record has Status__c = 'Disputed',
When it appears in the dashboard,
Then it is highlighted in amber and a tooltip displays Commission_Transaction__c.Dispute_Reason__c text.

Given the agent clicks 'Export to CSV',
When the export is triggered,
Then a Conga CLM-generated report is emailed to the agent's User.Email containing all visible Commission_Transaction__c rows with fields: Policy__c.Policy_Number__c, LOB__c, Earned_Amount__c, Status__c, Payment__c.Check_Number__c.

-Given the dashboard loads, a banner should display the agent's current tier level from Producer_Tier__c- (deprecated: tier display moved to Agent Profile page per UX review INS-1011).

Given the agent has zero commission records in the current filter period,
When the dashboard renders,
Then an empty-state illustration and message 'No commission activity found for the selected period' is displayed instead of a blank FlexCard.

## Solution

Build an OmniStudio FlexCard ('Agent_Commission_Dashboard_FC') with child FlexCards for summary tiles and a detail datatable. DataRaptor Extract queries Commission_Transaction__c and Payment__c with SOQL filter tokens bound to the date range OmniScript inputs. Use a Platform Event (Commission_Updated__e) published by an Apex Trigger on Commission_Transaction__c after_update to push real-time tile refreshes via a Lightning Message Channel subscription in the FlexCard JavaScript controller. Export functionality invokes a Conga CLM template via REST callout from an Apex action. Permission Set Group 'Agent_Portal_User' gates access via a Custom Tab visibility setting.

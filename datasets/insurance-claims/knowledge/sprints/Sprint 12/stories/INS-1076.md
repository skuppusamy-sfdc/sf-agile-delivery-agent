## Description

As a Broker Relations Coordinator, I need the system to automatically trigger a commission clawback calculation and update the associated Commission__c record when a Policy__c is cancelled or lapses within the chargeback window so that agents are accurately debited without manual spreadsheet reconciliation. The chargeback window is defined per LOB on the Commission_Schedule__c object.

## Acceptance Criteria

Given a Policy__c record transitions to Status__c = 'Cancelled' or Status__c = 'Lapsed',
When the Record-Triggered Flow (PolicyStatusChange_CommissionClawback) fires on the after-update context,
Then the flow checks whether Policy__c.Cancellation_Date__c falls within the chargeback window defined by Commission_Schedule__c.Chargeback_Days__c for the matching LOB__c.

Given the policy cancellation date is within the chargeback window,
When the flow identifies the related Commission__c record via Commission__c.Policy__c,
Then Commission__c.Status__c is updated to 'Clawed_Back', Commission__c.Clawback_Amount__c is set to the pro-rated value calculated by an invocable Apex method (CommissionClawbackCalculator.calculateProRata()), and Commission__c.Clawback_Reason__c is populated with 'Policy Lapse/Cancellation within chargeback window'.

Given the clawback record is created,
When Commission__c.Status__c = 'Clawed_Back',
Then a Payment__c record with Payment__c.Type__c = 'Clawback_Debit' and Payment__c.Amount__c = Commission__c.Clawback_Amount__c is auto-created and linked to the agent's Agent__c record.

Given the policy cancellation date is outside the chargeback window,
When the flow evaluates the date condition,
Then no clawback action is taken and Commission__c.Status__c remains 'Disbursed'.

Given a clawback Payment__c is created,
When the record is saved,
Then an automated email notification is sent to the assigned agent using an Email Alert referencing the 'Agent_Clawback_Notification' email template, including Policy__c.Policy_Number__c and Commission__c.Clawback_Amount__c.

## Solution

Implement a Record-Triggered Flow (PolicyStatusChange_CommissionClawback) on Policy__c, triggered after update when Status__c changes to 'Cancelled' or 'Lapsed'. The flow uses a Get Records element to fetch the related Commission_Schedule__c by LOB__c, then calls an Invocable Apex method (CommissionClawbackCalculator) that performs pro-rata calculation based on earned days vs. policy term. Flow then updates Commission__c via Update Records element and creates Payment__c via Create Records element. Email notification is dispatched via a Flow Email Alert action. Apex class is covered by CommissionClawbackCalculatorTest.cls at 92% coverage.

## Description

As a Claims Operations Manager, I need an Agentforce AI action to analyze newly submitted FNOL (First Notice of Loss) data on Claim__c records and recommend a triage classification — Standard, Complex, or SIU Referral — so that claims are routed to the appropriate handling team within minutes of submission rather than hours, improving cycle time and fraud detection rates.

The AI recommendation should be advisory; final classification remains with the assigned Claims Adjuster. The model should consider claim type, loss description, policy history, prior claim frequency, and ICD-10/CPT codes where applicable.

## Acceptance Criteria

Given a Claim__c record is created with Claim__c.Stage__c = 'FNOL Submitted',
When the Record-Triggered Flow fires the Agentforce triage action,
Then within 60 seconds Claim__c.AI_Triage_Recommendation__c is populated with one of: 'Standard', 'Complex', or 'SIU Referral'.

Given the AI recommendation is returned,
When Claim__c.AI_Triage_Recommendation__c = 'SIU Referral',
Then a Task is auto-created with Subject = 'AI-Flagged SIU Referral — Review Required', assigned to the SIU queue, with a due date of 1 business day, and Claim__c.SIU_Flag__c = true.

Given the AI recommendation is returned,
When Claim__c.AI_Triage_Recommendation__c = 'Complex',
Then Claim__c.Assigned_Team__c = 'Complex Claims Unit' and an email notification is sent to the Complex Claims queue using an Org-Wide Email Address.

Given the Agentforce action completes,
When any recommendation is stored,
Then Claim__c.AI_Confidence_Score__c (Percent field) is populated with the model's confidence value and Claim__c.AI_Recommendation_Timestamp__c = DateTime.now().

Given Claim__c.AI_Confidence_Score__c < 60,
When the recommendation is saved,
Then a warning banner is displayed on the Claim__c record page: 'Low confidence AI recommendation — manual review strongly advised' and Claim__c.AI_Low_Confidence_Flag__c = true.

Given the Claims Adjuster overrides the AI recommendation,
When the adjuster manually changes Claim__c.Triage_Classification__c to a value different from Claim__c.AI_Triage_Recommendation__c,
Then Claim__c.AI_Override__c = true and Claim__c.AI_Override_Reason__c is required (validation rule enforced) for audit and model retraining purposes.

-Given the AI model is unavailable, When the callout times out, Then Claim__c.Triage_Classification__c defaults to 'Complex' automatically-
(Deprecated: auto-default on timeout removed per SIU compliance guidance — claims must remain in 'Pending Triage' if AI is unavailable)

## Solution

Configure Agentforce Agent Action 'Claim_Triage_Recommendation' using Einstein Generative AI with a custom prompt template referencing Claim__c fields: Claim_Type__c, Loss_Description__c, Policy__c.Prior_Claim_Count__c, Claim_Line__c ICD10/CPT codes (via related list context). Record-Triggered Flow on Claim__c (after insert, Stage__c = 'FNOL Submitted') invokes the Agentforce Action asynchronously via Flow's 'Run Asynchronously' path. Apex invocable action 'AgentforceTriageInvocable' wraps the Einstein API call with error handling — on timeout, sets Claim__c.AI_Triage_Recommendation__c = null and Claim__c.Triage_Status__c = 'Pending Triage' (per revised AC). Validation Rule on Claim__c enforces AI_Override_Reason__c when AI_Override__c = true. LWC 'AI Recommendation Banner' component added to Claim__c Lightning Record Page via Dynamic Forms, conditionally rendering low-confidence warning.

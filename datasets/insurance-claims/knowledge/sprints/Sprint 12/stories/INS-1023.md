## Description

**Bug Report**

**Summary:** When a Policy__c has Split_Commission__c = true and two agents are listed in Commission_Split__c child records with a 50/50 split, the Earned_Amount__c on each Commission_Transaction__c is being populated with the full policy commission amount rather than the split percentage amount, resulting in double payment exposure.

**Environment:** Production sandbox (Full Copy) — Acme_INS_Sandbox_FC

**Steps to Reproduce:**
1. Create a Policy__c record with Annual_Premium__c = 10000.00, Commission_Rate__c = 0.10, Split_Commission__c = true.
2. Add two Commission_Split__c child records each with Split_Percentage__c = 50 linked to different Agent Contact records.
3. Run the 'Commission Calculation' batch job (CommissionCalcBatch) manually via Developer Console.
4. Navigate to Commission_Transaction__c records for both agents.
5. Observe Earned_Amount__c on both records.

**Expected Result:** Each Commission_Transaction__c.Earned_Amount__c = $500.00 (10,000 × 0.10 × 0.50).

**Actual Result:** Each Commission_Transaction__c.Earned_Amount__c = $1,000.00 (full commission, split percentage not applied).

**Impact:** Financial — if undetected through payment run, results in 2× commission disbursement. Affects approximately 340 active split-commission policies in production.

**Root Cause (suspected):** CommissionCalcBatch line 87 applies Commission_Rate__c × Annual_Premium__c but does not multiply by Split_Percentage__c / 100 when Split_Commission__c = true. The conditional branch exists but references the wrong field alias from the SOQL join.

## Acceptance Criteria

As per Description. Fix verified when Commission_Transaction__c.Earned_Amount__c = (Policy__c.Annual_Premium__c × Policy__c.Commission_Rate__c × Commission_Split__c.Split_Percentage__c / 100) for each split agent, and the sum of all split Earned_Amount__c values equals the total policy commission amount.

## Solution

Fix the CommissionCalcBatch Apex class at the identified line to correctly reference 'split.Split_Percentage__c' from the Commission_Split__c loop variable rather than the outer policy aggregate alias. Add a formula field Commission_Transaction__c.Split_Verified__c that cross-checks the sum of sibling transactions against Policy__c total commission and flags discrepancies. Add unit test method 'testSplitCommissionCalculation_FiftyFifty()' and 'testSplitCommissionCalculation_UnevenSplit()' to CommissionCalcBatchTest class. Deploy fix to sandbox, run full regression on CommissionCalcBatch, then promote to production via change set after UAT sign-off from Broker Relations Coordinator.

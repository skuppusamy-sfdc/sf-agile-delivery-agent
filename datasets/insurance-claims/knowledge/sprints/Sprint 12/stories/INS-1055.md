## Description

As a Claims Operations Manager, I need the system to automatically generate and deliver an Explanation_of_Benefits__c (EOB) document when a Claim__c reaches 'Adjudicated' status so that policyholders and providers receive timely, accurate payment explanations compliant with state mandate turnaround requirements.

For providers enrolled in Electronic Remittance Advice (ERA), the EOB data should also be transmitted as an 835 ERA file via the CMS integration. For policyholders, the EOB PDF should be delivered via DocuSign to the email on file.

## Acceptance Criteria

Given a Claim__c.Status__c transitions to 'Adjudicated' with Claim__c.Payment__c populated (Payment__c.Status__c = 'Approved'),
When the after-save Record-Triggered Flow fires,
Then an Explanation_of_Benefits__c record should be auto-created with: EOB.Claim__c = triggering Claim__c.Id, EOB.Generated_Date__c = today(), EOB.Total_Billed__c = sum of Claim_Line__c.Billed_Amount__c, EOB.Total_Allowed__c = sum of Claim_Line__c.Allowed_Amount__c, EOB.Total_Paid__c = Payment__c.Amount__c, EOB.Patient_Responsibility__c = EOB.Total_Billed__c - EOB.Total_Paid__c, EOB.Status__c = 'Generated'.

Given an EOB record is created with Status__c = 'Generated' and the policyholder's Policy__c.EOB_Delivery_Method__c = 'Email/DocuSign',
When the DocuSign integration is triggered via Apex callout,
Then the EOB PDF (generated by Conga CLM template 'EOB_Standard_v3') should be sent to Policy__c.Account__r.PersonEmail with DocuSign envelope status tracked on EOB.DocuSign_Envelope_Status__c.

Given a Provider__c has Provider__c.ERA_Enrolled__c = true and the claim's Facility__c or rendering provider is ERA-eligible,
When the EOB is generated,
Then a Platform Event (ERATransmission__e) should be published containing the EOB Id and Claim Id, triggering an Apex handler to format and transmit the 835 ERA file to the CMS integration endpoint, with transmission status written back to EOB.ERA_Transmission_Status__c.

Given EOB.ERA_Transmission_Status__c = 'Failed' after a CMS callout timeout,
When the retry logic executes (up to 3 attempts with exponential backoff),
Then if all retries fail, a Case should be auto-created assigned to the Claims Operations queue with Subject = 'ERA Transmission Failure – Manual Review Required' and linked to the EOB and Claim records.

Given a Claim__c with Claim__c.COB_Applicable__c = true (Coordination of Benefits),
When the EOB is generated,
Then EOB.COB_Indicator__c should be set to true and a secondary payer section should populate on the Conga EOB template from COB_Detail__c child records.

## Solution

Record-Triggered Flow on Claim__c (after save, Status__c changes to 'Adjudicated') with an async path to create the EOB record via Create Records. EOB creation triggers a second Record-Triggered Flow on Explanation_of_Benefits__c (after insert) that branches by delivery method. DocuSign branch: Apex invocable action ('GenerateAndSendEOBDocuSign') calls Conga CLM REST API to render the PDF, then DocuSign eSignature API to send (view-only envelope, no signature required). ERA branch: Platform Event ERATransmission__e published; Apex trigger handler on the event formats 835 EDI payload and calls CMS integration via Named Credential. Retry logic implemented in Queueable Apex with exponential backoff (3 attempts). COB section on Conga template driven by a SOQL-based template variable querying COB_Detail__c. All callouts governed by a Custom Metadata Type 'EOB_Integration_Settings__mdt' for endpoint and timeout configuration.

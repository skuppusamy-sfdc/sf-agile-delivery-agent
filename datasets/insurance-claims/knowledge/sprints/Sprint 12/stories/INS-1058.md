## Description

As a Broker Relations Coordinator, I need a real-time commission dashboard within the Agent Portal so that agents can view their earned, pending, and paid commissions broken down by Line of Business (LOB) without contacting the back office.

This feature should surface commission data from Payment__c records linked to Policy__c and present aggregated totals, trend charts, and drill-down capability to individual policy transactions. Extends INS-1028 (Agent Portal base layout).

## Acceptance Criteria

Given an authenticated agent user accesses the Agent Portal Commission Dashboard,
When the page loads,
Then a FlexCard displays three KPI tiles: 'Earned MTD', 'Pending Approval', and 'YTD Paid', each sourced from Payment__c.Amount__c filtered by Payment__c.Type__c = 'Commission' and Payment__c.Agent__c = current user's Agent_Profile__c.

Given the agent selects a LOB filter (e.g., 'Commercial Auto', 'Workers Comp', 'Health'),
When the filter is applied,
Then the dashboard refreshes to display only Payment__c records where Policy__c.Line_of_Business__c matches the selected LOB, with results updating within 3 seconds.

Given the agent clicks a KPI tile,
When the drill-down panel opens,
Then a DataRaptor-powered list displays individual Policy__c.Policy_Number__c, Policy__c.Policyholder_Name__c, Payment__c.Amount__c, Payment__c.Status__c, and Payment__c.Payment_Date__c for each contributing record.

Given Payment__c.Status__c = 'Pending Approval',
When displayed in the dashboard,
Then the record is highlighted in amber and a tooltip shows the pending reason from Payment__c.Hold_Reason__c.

Given the agent navigates to the 'Trend' tab,
When the chart renders,
Then a 12-month bar chart displays monthly commission totals aggregated by LOB using a custom Apex @AuraEnabled controller querying Payment__c with SOQL grouping by CALENDAR_MONTH(Payment__c.Payment_Date__c).

-Given the agent can export commission data to CSV directly from the dashboard-
-This requirement moved to INS-1072 (Agent Portal Export Utility)-

## Solution

Build an OmniStudio FlexCard (CommissionDashboard_FC) consuming three DataRaptor Extract actions targeting Payment__c. Use a child FlexCard for the drill-down panel. Implement a custom Apex controller (CommissionDashboardController.cls) for the trend chart SOQL aggregation. Deploy via Permission Set Group 'Agent_Portal_Standard' to restrict access. Platform Event (Commission_Updated__e) triggers a real-time refresh via LWC wire adapter when back-office posts commission approvals. Cache KPI totals in a Custom Metadata record (Commission_Cache__mdt) refreshed every 15 minutes via Scheduled Apex to reduce SOQL limits.

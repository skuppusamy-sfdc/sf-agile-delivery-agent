## Description

As a Provider Network Specialist, I need an automated workflow that processes inbound Provider Change Notifications (PCN) to update provider address, specialty, and TIN fields on Provider__c and associated Provider_Network_Agreement__c records so that the provider directory remains accurate and compliant with CMS data accuracy mandates within the required 30-day update window.

## Acceptance Criteria

Given a Provider_Change_Notification__c record is created with PCN_Type__c = 'Address Change',
When the record is saved with PCN_Status__c = 'Received',
Then a Record-Triggered Flow launches an approval process routed to the Provider Network Specialist queue.

Given the PCN approval is approved,
When the approval step completes,
Then the flow updates Provider__c.BillingStreet, Provider__c.BillingCity, Provider__c.BillingState, Provider__c.BillingPostalCode from PCN field values and sets PCN_Status__c = 'Applied'.

Given PCN_Type__c = 'Specialty Change',
When the PCN is approved,
Then Provider__c.Primary_Specialty__c is updated and all active Provider_Network_Agreement__c records (PNA__c.Status__c = 'Active') linked to the provider are flagged with PNA__c.Specialty_Review_Required__c = true.

Given PCN_Type__c = 'TIN Change',
When the PCN is received,
Then an automatic hold is placed (PCN_Status__c = 'Compliance Review') and a Case is created assigned to the Compliance Officer queue with Subject = 'TIN Change PCN — Compliance Review Required'.

Given a PCN remains in 'Received' status for more than 10 business days,
When the nightly scheduled flow runs,
Then an escalation email is sent to the Provider Network Specialist's manager and PCN_Status__c = 'Escalated'.

Given any PCN is applied,
When the update is committed,
Then Provider__c.Directory_Last_Updated__c = today and a Provider_Change_Notification__c.Applied_Date__c = today is stamped for audit trail.

## Solution

Record-Triggered Flow on Provider_Change_Notification__c handles routing logic and field updates. Approval Process 'PCN_Approval' with two steps: auto-approve for address changes with no TIN impact; manual approval for specialty and TIN changes. Scheduled Flow 'PCN_Escalation_Check' runs nightly using a SOQL query for overdue PCNs. TIN change compliance case creation in a separate subflow to keep main flow modular. Custom permission 'PCN_Approve' added to Permission Set Group 'Provider_Network_Ops'. All PCN audit history preserved via standard Salesforce field history tracking on Provider__c.

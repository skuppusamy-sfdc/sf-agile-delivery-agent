## Description

As a Broker Relations Coordinator, I need a guided digital onboarding OmniScript for new agent contracting so that prospective agents can complete their profile, upload licensing documentation, select appointed LOBs, and execute their Agent Agreement via DocuSign — all within the Agent Portal — reducing the current 10-day paper-based onboarding cycle to under 48 hours. The OmniScript should create the underlying Salesforce records and trigger background verification steps automatically.

## Acceptance Criteria

Given a prospective agent accesses the 'Join Our Network' public Experience Cloud page,
When they launch the onboarding OmniScript ('Agent_Onboarding_OS'),
Then Step 1 collects: First_Name__c, Last_Name__c, NPI__c, TIN__c, License_Number__c, License_State__c, License_Expiry_Date__c, and validates NPI format (10-digit numeric) client-side before allowing progression.

Given the agent completes Step 1 with a valid NPI,
When they advance to Step 2,
Then the OmniScript calls an Integration Procedure ('IP_NPPES_NPI_Lookup') to validate the NPI against the NPPES registry and pre-populates Agent_Legal_Name__c and Primary_Taxonomy_Code__c from the NPPES response.

Given NPPES validation returns a mismatch between submitted name and registry name,
When the agent is on Step 2,
Then an inline error message reads 'NPI registry mismatch detected. Please verify your NPI or contact support.' and progression is blocked.

Given the agent completes Steps 1–3 (LOB selection and E&O insurance upload),
When they reach Step 4 (Agreement Execution),
Then the OmniScript invokes the DocuSign Integration Procedure ('IP_DocuSign_Send_Agent_Agreement') to generate and send the Agent_Agreement__c envelope with the agent's email as the primary signer, and DocuSign_Envelope_Id__c is stored on the new Agent__c record.

Given the DocuSign envelope is sent successfully,
When the OmniScript completes,
Then a new Agent__c record is created with Onboarding_Status__c = 'Pending DocuSign', a Credentialing_Case__c is auto-created with Status__c = 'New' and Type__c = 'Initial Agent Credentialing', and a welcome email is dispatched via Flow EmailAlert.

Given the DocuSign envelope is completed (webhook callback received),
When the Platform Event 'DocuSign_Complete__e' fires,
Then Agent__c.Onboarding_Status__c transitions to 'Agreement Executed' and Agent__c.Agreement_Executed_Date__c = TODAY().

Given the agent's License_Expiry_Date__c is within 90 days of TODAY() at time of submission,
When the Agent__c record is created,
Then a License_Expiry_Warning__c = true flag is set and a Task is created assigned to the Broker Relations Coordinator with Subject = 'License Expiring Soon — Review Agent [Name]'.

## Solution

Build OmniScript 'Agent_Onboarding_OS' with 5 steps using OmniStudio Step Maps. Step 2 NPI validation uses an Integration Procedure calling a named credential to the NPPES FHIR API (GET /practitioners?identifier={npi}). Step 4 DocuSign invocation uses an existing Integration Procedure template extended with agent agreement template ID stored in Custom Metadata (DocuSign_Config__mdt). DataRaptor Post creates Agent__c and Credentialing_Case__c records. DocuSign webhook is received by a custom REST endpoint (@RestResource) that publishes DocuSign_Complete__e Platform Event. A Record-Triggered Flow on Agent__c subscribes to status changes for downstream notifications. License expiry logic handled in an Apex Trigger (before insert) on Agent__c. OmniScript deployed to Experience Cloud guest/authenticated pages with appropriate Guest User Profile permissions scoped to the onboarding flow only.

## Description

As a Broker Relations Coordinator, I need a structured pipeline to track agent recruitment stages from initial contact through licensing verification and appointment so that we can reduce time-to-active for new agents and ensure compliance with state appointment requirements before any policy is bound.

Recruitment records live on a custom object Agent_Recruitment__c linked to the candidate's Contact record. Pipeline stages must mirror our defined onboarding SLA gates.

## Acceptance Criteria

Given a new Agent_Recruitment__c record is created with Stage__c = 'Prospect',
When the record is saved,
Then a Record-Triggered Flow automatically creates a Task with Subject = 'Initial Outreach Call', Due Date = TODAY() + 2, assigned to the owning Broker Relations Coordinator.

Given Agent_Recruitment__c.Stage__c is updated to 'Background Check Submitted',
When the stage transition is saved,
Then a DocuSign envelope is sent via DocuSign for Salesforce API to the candidate's Contact.Email containing the 'Agent_Appointment_Agreement_v3' template, and Agent_Recruitment__c.DocuSign_Sent_Date__c is stamped with NOW().

Given Agent_Recruitment__c.Stage__c = 'License Verified' and License_State__c is populated,
When the record is saved,
Then a validation rule prevents progression to 'Appointed' unless NPI__c and TIN__c fields on the linked Contact are non-null.

Given the recruitment record reaches Stage__c = 'Appointed',
When the stage is saved,
Then a Salesforce Flow creates a linked User record (Profile = 'Agent Portal User'), sets Contact.Agent_Status__c = 'Active', and fires a Welcome Email via an org-wide email address 'noreply@acmeinsurance.com'.

Given any stage transition occurs,
When saved,
Then Agent_Recruitment__c.Last_Stage_Change_Date__c is updated and a Stage_History__c child record is created capturing Previous_Stage__c, New_Stage__c, Changed_By__c, and Change_Timestamp__c.

## Solution

Implement a Record-Triggered Flow ('Agent_Recruitment_Stage_Automation') with decision nodes for each stage transition. DocuSign envelope send is invoked via an Apex action class calling the DocuSign APEX Toolkit 'DSFSDocuSignAPIHelper'. User provisioning on 'Appointed' uses an Apex-invocable method to handle mixed DML (Contact + User creation). Stage history captured by an Apex Trigger (after update) on Agent_Recruitment__c inserting Stage_History__c child records. Validation rule enforces NPI/TIN completeness. Custom Path component on the Agent_Recruitment__c Lightning record page provides coordinator-facing pipeline visualization.

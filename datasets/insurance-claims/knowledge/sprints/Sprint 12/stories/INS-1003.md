## Description

As a Claims Operations Manager, I need Agentforce to provide AI-assisted triage recommendations immediately upon FNOL submission so that high-complexity and potentially fraudulent claims are surfaced to the SIU queue automatically, low-complexity claims are fast-tracked for straight-through processing, and claims adjusters receive a prioritized workload based on AI confidence scores. The AI recommendation should be non-binding and fully auditable per compliance requirements.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Stage__c = 'FNOL Submitted',
When the Record-Triggered Flow fires the Agentforce action ('Claim_Triage_Agent'),
Then the agent receives the Claim__c.Description__c, Claim__c.LOB__c, Claim__c.Claimed_Amount__c, Claim__c.Incident_Type__c, and the claimant's Policy__c.Claim_History_Count__c as grounding context.

Given the Agentforce action completes,
When the AI response is received,
Then a Claim_AI_Recommendation__c child record is created on the Claim__c with fields: Triage_Category__c (picklist: 'Straight-Through', 'Standard Review', 'Complex Review', 'SIU Referral'), AI_Confidence_Score__c (decimal 0.00–1.00), AI_Rationale__c (text, max 1000 chars), and Recommendation_Timestamp__c = NOW().

Given Claim_AI_Recommendation__c.Triage_Category__c = 'SIU Referral' AND AI_Confidence_Score__c >= 0.85,
When the record is saved,
Then Claim__c.SIU_Referral_Flag__c = true, Claim__c.Status__c = 'Pending SIU Review', and a Case is created in the 'SIU Queue' with Subject = 'AI-Flagged SIU Referral: [Claim__c.Claim_Number__c]'.

Given Claim_AI_Recommendation__c.Triage_Category__c = 'Straight-Through' AND AI_Confidence_Score__c >= 0.90,
When the record is saved,
Then Claim__c.Processing_Track__c = 'Fast Track' and the claim is auto-assigned to the 'Straight-Through Processing Queue' via an Assignment Rule.

Given any Claim_AI_Recommendation__c is created,
When a claims adjuster views the Claim__c record page,
Then the AI recommendation FlexCard ('Claim_AI_Triage_FC') displays the Triage_Category__c, AI_Confidence_Score__c as a visual gauge, AI_Rationale__c, and a disclaimer: 'This recommendation is AI-generated and non-binding. Human review required before action.'

Given the adjuster overrides the AI recommendation,
When they select a different triage category and save,
Then Claim_AI_Recommendation__c.Override_Flag__c = true, Override_Reason__c (required text) is captured, Override_By__c = current user, and the override is logged to the Claim__c Audit Trail field for compliance reporting.

## Solution

Configure an Agentforce Agent ('Claim_Triage_Agent') with a custom Agent Action backed by an Apex invocable method 'ClaimTriageAgentAction' that constructs the prompt with structured Claim__c context fields as grounding data. Use Einstein Generative AI with a custom prompt template stored in the Prompt Builder, constrained to return a structured JSON response (triage_category, confidence_score, rationale). The invocable is called from a Record-Triggered Flow (after insert, Claim__c.Stage__c = 'FNOL Submitted') asynchronously via a Flow 'Run Asynchronously' path to avoid blocking FNOL save. Parse the JSON response in Apex and perform DML to create Claim_AI_Recommendation__c. SIU routing logic in a second Record-Triggered Flow on Claim_AI_Recommendation__c (after insert). Agentforce actions and prompt templates versioned in source control. Compliance audit logging via a custom Apex handler writing to Claim_Audit_Log__c. FlexCard built in OmniStudio consuming a DataRaptor Extract on Claim_AI_Recommendation__c.

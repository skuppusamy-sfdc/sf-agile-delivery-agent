## Description

**Bug Report — EOB Generation Failure on COB Claims**

**Environment:** Full Sandbox (Acme-FS1)
**Severity:** High — affects all COB secondary payer claims in adjudication queue

**Steps to Reproduce:**
1. Navigate to a Claim__c record where Claim__c.COB_Indicator__c = true and Claim__c.Payer_Sequence__c = 'Secondary'.
2. Ensure at least one Claim_Line__c record has Claim_Line__c.COB_Adjustment_Amount__c populated with a non-zero value.
3. Trigger EOB generation by clicking the 'Generate EOB' button on the Claim__c record page.
4. Observe the resulting Explanation_of_Benefits__c record or error.

**Expected Behavior:**
A new Explanation_of_Benefits__c record is created with EOB__c.Total_Patient_Responsibility__c correctly reflecting the primary payer's payment subtracted from the allowed amount, and EOB__c.COB_Adjustment__c populated with the aggregate Claim_Line__c.COB_Adjustment_Amount__c.

**Actual Behavior:**
The EOB generation process throws an unhandled NullPointerException in EOBGenerationService.cls at line 214. The Explanation_of_Benefits__c record is either not created or is created in an incomplete state with EOB__c.COB_Adjustment__c = null and EOB__c.Status__c = 'Error'. A debug log entry shows: 'System.NullPointerException: Attempt to de-reference a null object — EOBGenerationService.buildCOBLineItems()'

**Root Cause (Preliminary):**
The buildCOBLineItems() method does not null-check Claim_Line__c.Primary_Payer_Paid__c before performing arithmetic. When the primary payer ERA has not yet been posted (ERA_Status__c = 'Pending'), the field is null, causing the NPE.

**Frequency:** Reproducible 100% on claims where ERA_Status__c != 'Reconciled'.

## Acceptance Criteria

As per Description.

Additional verification:
Given a Claim__c with COB_Indicator__c = true and ERA_Status__c = 'Pending',
When EOB generation is triggered,
Then the process does not throw an exception; instead, EOB__c.Status__c = 'Pending ERA Reconciliation' and a warning message is surfaced to the user.

Given ERA_Status__c transitions to 'Reconciled',
When the EOB regeneration is triggered (manually or via automation),
Then EOB__c.COB_Adjustment__c is correctly populated and EOB__c.Status__c = 'Generated'.

## Solution

Fix EOBGenerationService.cls buildCOBLineItems() method to null-check Claim_Line__c.Primary_Payer_Paid__c before arithmetic operations. Introduce a guard clause that sets EOB__c.Status__c = 'Pending ERA Reconciliation' when ERA_Status__c != 'Reconciled'. Add a Record-Triggered Flow on Claim__c to re-invoke EOB generation when ERA_Status__c changes to 'Reconciled'. Deploy updated Apex class with corresponding unit test coverage (target ≥ 90%) including null-value test scenarios. Update the EOB_Generation_Config__mdt to include a flag Allow_Partial_COB_EOB__c for future configurability.

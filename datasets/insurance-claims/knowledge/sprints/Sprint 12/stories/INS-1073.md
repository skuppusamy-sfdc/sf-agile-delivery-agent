## Description

**Bug Report**

**Summary:** When an agent submits a FNOL via the Agent Portal OmniScript ('FNOL_Agent_Submission_OS', built in INS-1072) and the pre-populated Policy__c.LOB__c = 'Workers Compensation', the OmniScript throws a null pointer exception on Step 2 ('Incident Details') and the session terminates, preventing claim submission.

**Steps to Reproduce:**
1. Log in to the Agent Portal as a test agent user.
2. Navigate to 'Submit FNOL'.
3. Enter a Policy_Number__c for a policy where Policy__c.LOB__c = 'Workers Compensation'.
4. Click 'Lookup Policy' — policy details populate correctly on Step 1.
5. Click 'Next' to advance to Step 2 ('Incident Details').
6. Observe: OmniScript displays a generic error toast 'An unexpected error occurred' and the session cannot be resumed.

**Expected Behavior:** Step 2 should render LOB-specific incident detail fields for Workers Compensation (e.g., Injury_Type__c, Employer_Name__c, Date_of_Injury__c) as defined in the OmniScript conditional element logic.

**Actual Behavior:** Null pointer exception thrown. Browser console shows: 'Cannot read properties of null (reading LOB_Step2_Config)'. Suspected cause: the Calculation Procedure ('CP_FNOL_Step2_LOB_Router') does not have a routing entry for LOB value 'Workers Compensation' (note: stored value is 'Workers Compensation' with a space; the CP matrix uses 'WorkersComp' without space, causing a null return that is not null-checked before the step rendering logic executes).

**Environment:** Sandbox S2, OmniStudio version 238.4, reproducible 100% of attempts with Workers Comp policies.

## Acceptance Criteria

As per Description.

Additional verification: Given Policy__c.LOB__c = 'Workers Compensation', When the agent advances to Step 2 of the FNOL OmniScript, Then the Step 2 incident fields render without error and include Injury_Type__c, Employer_Name__c, and Date_of_Injury__c input elements.

## Solution

Fix the Calculation Procedure ('CP_FNOL_Step2_LOB_Router') matrix to add a row mapping input value 'Workers Compensation' (with space, matching the Policy__c.LOB__c picklist value) to the correct step configuration key 'WC_Step2_Config'. Add a default/fallback row to the matrix returning a generic step config to prevent null returns for any unmapped LOB values. Add a null-check conditional element in the OmniScript Step 2 element tree to gracefully handle null config returns with a user-friendly message. Update unit tests in the OmniStudio Test Suite to cover all active LOB__c picklist values. Regression test all LOB paths in S2 before promoting to UAT.

## Description

As a Compliance Officer, I need a performance audit and security hardening review of the Agent Portal Experience Cloud site before Phase 2 go-live so that the portal meets Acme Insurance's internal SLA of sub-3-second page load times and passes the quarterly security review requiring no Guest User profile access to sensitive objects (Commission_Line__c, Policy__c, Claim__c, Payment__c). This is a prerequisite for the Phase 2 production deployment of INS-0981, INS-0982, INS-0983, INS-0986, and INS-0987.

## Acceptance Criteria

Given the Agent Portal Community is deployed to the staging sandbox,
When a Salesforce Lightning Performance Analysis (LPA) scan is executed,
Then no individual page in the Agent Portal exceeds 3 seconds Time to Interactive (TTI) under simulated load of 50 concurrent users, and all LWC components pass the 'No Synchronous Apex on Page Load' check.

Given a review of the Guest User Profile assigned to the Agent Portal site,
When object permissions are audited,
Then the Guest User profile has zero Read, Create, Edit, or Delete permissions on Commission_Line__c, Policy__c, Claim__c, Payment__c, and Commission_Dispute__c — confirmed via Salesforce Setup > Profiles > Guest User > Object Settings.

Given the Agent Portal uses sharing rules to expose data to authenticated community users,
When a sharing rule audit is performed,
Then all sharing rules scoped to 'All Internal Users' or 'All Portal Users' on Commission_Line__c and Payment__c are identified and replaced with role-based or criteria-based sharing rules that restrict access to the owning agent's records only.

Given the audit findings are compiled,
When the report is delivered to the Compliance Officer and Platform Admin,
Then a documented remediation log is attached to this story as a Confluence page link, listing each finding, severity (Critical/High/Medium/Low), remediation action taken, and sign-off date — with zero Critical or High findings remaining open at go-live.

## Solution

Conduct a structured audit across three workstreams: (1) Performance – use Salesforce built-in Lightning Usage App and Chrome DevTools LCP/TTI profiling against the staging community; identify and refactor any synchronous Apex calls on page load (convert to @wire or async patterns in LWC); review DataRaptor queries for missing indexes on Commission_Line__c.Agent__c and Commission_Summary__c.Agent__c (add custom indexes via Salesforce Support ticket if needed). (2) Security – export Guest User profile permissions via Salesforce CLI (sf org list metadata) and cross-reference against sensitive object list; remove all Guest User CRUD via Profile edit in Setup; replace broad sharing rules with criteria-based Sharing Rules (Commission_Line__c: shared when Owner.Profile.Name = 'Broker Community User'). (3) Reporting – compile findings in a Confluence audit template, assign severity using OWASP risk matrix, obtain sign-off from Compliance Officer (Sofia Hernandez) and Platform Architect (Michael Brown) before go-live gate review.

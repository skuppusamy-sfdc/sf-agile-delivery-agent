## Description

As a Broker Relations Coordinator, I need a real-time commission earnings dashboard on the Agent Portal so that licensed agents and brokers can view their current-period commissions, pending payments, and historical trends without contacting internal support teams. The dashboard should surface data from Payment__c and Policy__c records associated with the agent's Producer Code, aggregated by LOB (Life, P&C, Health) and commission tier.

## Acceptance Criteria

Given an authenticated agent accesses the Agent Portal Commission Dashboard,
When the dashboard loads,
Then a FlexCard displays total YTD commission earned sourced from Payment__c.Commission_Amount__c filtered by Payment__c.Producer_Code__c matching the logged-in agent's Contact.Producer_Code__c.

Given the agent selects a date range filter (current month, QTD, YTD, custom),
When the filter is applied,
Then the FlexCard refreshes and displays Payment__c records within the selected range grouped by Policy__c.Line_of_Business__c.

Given a Payment__c record has Status__c = 'Pending',
When displayed in the dashboard,
Then it appears in a 'Pending Commissions' section distinct from 'Paid' commissions, showing Payment__c.Expected_Payment_Date__c and Payment__c.Commission_Amount__c.

Given the agent views the LOB breakdown panel,
When commissions exist across multiple LOBs,
Then each LOB row displays: Policy count, total premium written (Policy__c.Annual_Premium__c sum), commission rate (Payment__c.Commission_Rate__c), and net commission earned.

Given the agent clicks a commission row,
When the detail panel opens,
Then it shows the linked Policy__c.Policy_Number__c, Policyholder name, effective date, and associated Claim__c records if any chargebacks are applied.

-Given the dashboard includes a CSV export button that exports all records regardless of permission set- (deprecated per compliance review INS-1031)
Given the agent exports commission data,
When the export action is triggered,
Then only Payment__c records owned by or shared to the agent's user via Sharing Rule are included in the export file.

Given an agent has no Payment__c records in the selected period,
When the dashboard renders,
Then an empty state message 'No commission activity found for the selected period' is displayed with a link to contact their Broker Relations Coordinator.

## Solution

Build an OmniStudio FlexCard ('AgentCommissionDashboard') consuming a DataRaptor Extract that queries Payment__c, Policy__c joined on Policy__c.Id = Payment__c.Policy__c, filtered by logged-in user's Producer_Code__c via a merge field. Aggregate totals computed in a DataRaptor Transform step. Date range filter implemented as a FlexCard State Action updating DataRaptor input parameters. LOB breakdown rendered as a child FlexCard. Detail drill-down uses a Navigation Action to an OmniScript ('CommissionDetailOS'). Sharing enforced via existing Broker Sharing Rule on Payment__c (criteria: Payment__c.Producer_Code__c = Contact.Producer_Code__c of related user). Export action calls an Apex-invocable method respecting CRUD/FLS. Lightning Web Component wrapper embedded in the Agent Portal Experience Cloud page via a custom App Builder component slot.

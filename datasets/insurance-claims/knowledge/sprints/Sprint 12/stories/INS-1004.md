## Description

As a Broker Relations Coordinator, I need a real-time commission dashboard on the Agent Portal so that agents can track their earned, pending, and paid commissions by LOB without submitting manual inquiries to the back office.

This story extends INS-0974 (Agent Portal Base Layout) and adds a dedicated Commission tab surfacing commission data from Payment__c and Policy__c records associated with the logged-in agent's broker account.

## Acceptance Criteria

Given an authenticated agent user logs into the Agent Portal,
When they navigate to the 'My Commissions' tab,
Then they see a FlexCard displaying total Earned, Pending, and Paid commission amounts aggregated from Payment__c records where Payment__c.Payee_Agent__c = current user's broker Account Id and Payment__c.Payment_Type__c = 'Commission'.

Given the Commission Dashboard is loaded,
When the agent selects a LOB filter (e.g., 'Auto', 'Property', 'Health'),
Then the dashboard filters Payment__c records by Policy__c.Line_of_Business__c matching the selected value and recalculates totals without a full page refresh.

Given a commission payment status changes to 'Paid' on Payment__c.Status__c,
When the Platform Event Commission_Status_Changed__e is fired,
Then the FlexCard refreshes within 30 seconds and moves the associated amount from 'Pending' to 'Paid' bucket.

Given the agent views a commission line item,
When they click the policy number hyperlink,
Then they are navigated to the Policy__c record detail page with read-only field-level access enforced by the Agent_Portal_Read_Only Permission Set Group.

-Given the dashboard loads historical commissions for all time by default- (deprecated — default view is current calendar year per compliance requirement CR-2024-118)
Given the dashboard loads,
When no filter is applied,
Then Payment__c records are filtered to Payment__c.Payment_Date__c within the current calendar year.

Given the agent wants to export commission data,
When they click 'Export to CSV',
Then a CSV file is generated via an Apex invocable action containing columns: Policy Number, LOB, Commission Amount, Status, Payment Date, and downloaded to the browser.

## Solution

Build a LWC-based FlexCard (OmniStudio) named 'AgentCommissionDashboard' consuming a DataRaptor Extract that queries Payment__c and related Policy__c fields. Use a Platform Event (Commission_Status_Changed__e) published by a Record-Triggered Flow on Payment__c to push real-time updates. LOB filter implemented as a DataRaptor filter parameter bound to an OmniScript dropdown. CSV export handled by an Apex @InvocableMethod called from an OmniScript action. Permission Set Group 'Agent_Portal_Read_Only' controls field-level security on Policy__c and Payment__c for external agent community users. Lightning Community page updated to include new tab via Experience Builder.

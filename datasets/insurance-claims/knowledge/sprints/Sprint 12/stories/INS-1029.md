## Description

As a Broker Relations Coordinator, I need the system to automatically calculate and post a commission chargeback transaction when a policy lapses within 90 days of the bind date so that agent commission ledgers are accurate and manual chargeback processing is eliminated. The chargeback amount should equal 100% of the original commission paid if lapse occurs within 30 days, and 50% if lapse occurs between 31 and 90 days.

## Acceptance Criteria

Given Policy__c.Status__c transitions to 'Lapsed',
When the Record-Triggered Flow evaluates Policy__c.Lapse_Date__c - Policy__c.Bind_Date__c,
Then:
  - If the difference is <= 30 days: a Commission_Transaction__c record is created with Type__c = 'Chargeback', Amount__c = original commission * -1.0
  - If the difference is between 31 and 90 days: Amount__c = original commission * -0.5
  - If the difference is > 90 days: no chargeback record is created.

Given a chargeback Commission_Transaction__c is created,
When it is saved,
Then Commission_Transaction__c.Status__c = 'Pending Review',
AND Commission_Transaction__c.Policy__c is populated,
AND Commission_Transaction__c.Agent__c is populated from Policy__c.Writing_Agent__c.

Given a Broker Relations Coordinator reviews the chargeback in the queue,
When they approve it (Status__c = 'Posted'),
Then the Platform Event 'CommissionPosted__e' fires (reusing the event defined in INS-1027),
AND the agent's YTD dashboard total reflects the negative adjustment.

Given a policy lapse is reversed (Policy__c.Status__c returns to 'Active') within 5 business days,
When the flow re-evaluates,
Then any associated 'Pending Review' chargeback Commission_Transaction__c records are automatically cancelled (Status__c = 'Cancelled'),
AND a Chatter post is added to the Policy__c record noting the reversal.

-Given chargeback notifications are sent via SMS to the agent's mobile number- (deprecated — email notification only per compliance review)

## Solution

Record-Triggered Flow on Policy__c (After Update, Status__c changed to 'Lapsed') invokes an Apex invocable method 'ChargebackCalculator.calculate(policyId)' which queries the original Commission_Transaction__c (Type__c = 'Commission', Status__c = 'Posted') and creates the chargeback record with correct amount based on day-difference logic. A separate flow path handles lapse reversal by querying for open chargeback records and updating Status__c to 'Cancelled'. Chatter post added via ConnectAPI in Apex. Reuses 'CommissionPosted__e' Platform Event from INS-1027.

## Description

As a Broker Relations Coordinator, I need a real-time commission dashboard within the Agent Portal so that licensed agents and brokers can view their year-to-date (YTD) earnings, pending commissions, and payment history across all Lines of Business (LOBs) without requiring manual reporting requests.

This dashboard should aggregate commission data from Payment__c records linked to the agent's Policy__c portfolio, broken down by LOB, policy effective date range, and commission tier. Extends INS-1005 (Agent Portal base layout).

## Acceptance Criteria

Given a logged-in agent with an active broker record (Broker__c.Status__c = 'Active'),
When they navigate to the Commission Dashboard tab in the Agent Portal,
Then a FlexCard displays YTD earned commissions aggregated from Payment__c records where Payment__c.Payee_Type__c = 'Agent' and Payment__c.Status__c = 'Disbursed'.

Given the dashboard is loaded,
When the agent selects a LOB filter (e.g., 'Commercial', 'Individual', 'Medicare'),
Then the commission totals refresh to reflect only Payment__c records linked to Policy__c.Line_of_Business__c matching the selected LOB.

Given a commission payment is in 'Pending' status,
When it appears in the dashboard,
Then it is displayed in a separate 'Pending Commissions' section where Payment__c.Status__c = 'Pending' and the expected disbursement date is shown from Payment__c.Expected_Disbursement_Date__c.

Given the agent has payments across more than one TPA,
When the dashboard renders,
Then commissions are grouped by TPA__c.Name with subtotals per TPA entity.

-Given the dashboard supports PDF export of commission statements- (superseded by INS-1041 which handles document generation centrally)

Given the agent views payment history,
When they click a specific Payment__c record row,
Then a detail panel expands showing Payment__c.Amount__c, Payment__c.Commission_Rate__c, Payment__c.Policy__c, and Payment__c.Payment_Date__c.

Given the page is accessed on a mobile device,
When the viewport is less than 768px,
Then the FlexCard layout collapses to a single-column responsive view without data truncation.

## Solution

Build a multi-step OmniScript-backed FlexCard (Commission_Dashboard_FC) using DataRaptors to query Payment__c with filters on Payee_Type__c, Status__c, and related Policy__c.Line_of_Business__c. Implement a Lightning Web Component (commissionSummaryChart) using Chart.js for the YTD bar chart visualization. Use a Platform Event (Commission_Updated__e) published from a Record-Triggered Flow on Payment__c to trigger real-time dashboard refresh without full page reload. Apply a Permission Set Group (PSG_Agent_Portal_Standard) to control field-level visibility on Payment__c.Commission_Rate__c. DataRaptor Extract DR_Agent_Commission_Summary will join Payment__c → Policy__c → Broker__c using relationship traversal. Cache results in a Custom Metadata Type (Commission_Dashboard_Config__mdt) for refresh interval settings.

## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces the provider's active Provider_Network_Agreement__c, associated Fee_Schedule__c rates, and NPI/TIN verification status so that I can quickly assess network participation without navigating multiple related lists. The card must reflect real-time NPPES NPI status retrieved via the DataIntegration layer.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then it displays: Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Credentialing_Status__c, and the count of active Provider_Network_Agreement__c records where PNA.Status__c = 'Active' and PNA.Expiration_Date__c >= TODAY().

Given at least one active PNA exists,
When the specialist expands the PNA section,
Then each PNA row shows: Provider_Network_Agreement__c.Agreement_Number__c, Provider_Network_Agreement__c.Network_Name__c, Provider_Network_Agreement__c.Effective_Date__c, Provider_Network_Agreement__c.Expiration_Date__c, and a link to the associated Fee_Schedule__c record.

Given a Fee_Schedule__c is linked to an active PNA,
When the specialist clicks 'View Rates',
Then a child FlexCard displays Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Allowed_Amount__c, Fee_Schedule__c.Effective_Date__c for up to 20 most recently updated CPT codes.

Given the NPPES integration is available,
When the FlexCard loads,
Then an NPI Verification badge shows 'Verified' (green) if the last NPPES check returned a match within 30 days (Provider__c.NPPES_Last_Verified__c >= TODAY() - 30), or 'Needs Verification' (amber) otherwise.

Given Provider__c.Credentialing_Status__c = 'Expired' or 'Suspended',
When the FlexCard renders,
Then a prominent warning banner displays 'Credentialing Alert: Provider not eligible for claims payment' and the associated Credentialing_Case__c.Id is linked.

## Solution

OmniStudio FlexCard ('Provider360Card') with a parent DataRaptor Extract querying Account (Provider__c) fields including NPI__c, TIN__c, Credentialing_Status__c. Child DataRaptor Extract retrieves Provider_Network_Agreement__c records filtered by Provider__c lookup and Status__c = 'Active'. Second child DataRaptor retrieves Fee_Schedule__c via PNA.Fee_Schedule__c lookup. NPPES verification badge logic implemented as a FlexCard Conditional State using formula on NPPES_Last_Verified__c. Warning banner uses a Conditional Visibility block on Credentialing_Status__c values. FlexCard deployed to Provider__c Lightning Record Page via App Builder. DataRaptor caching set to 5 minutes to balance performance vs. freshness. Permission Set 'Provider_Relations_Read' grants read access to PNA, Fee_Schedule__c, and Credentialing_Case__c objects.

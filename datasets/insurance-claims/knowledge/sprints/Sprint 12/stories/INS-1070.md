## Description

As a Provider Relations Specialist, I need the Provider 360 FlexCard to include a panel showing all active Provider_Network_Agreement__c (PNA) records linked to a provider, including associated Fee_Schedule__c and Group_Service_Agreement__c (GSA) summaries, so that I can quickly assess a provider's contracted network status without navigating multiple related lists.

## Acceptance Criteria

Given a Provider__c (Person Account) record is open in the Provider 360 FlexCard,
When the 'Network Agreements' panel renders,
Then all PNA records where Provider_Network_Agreement__c.Provider__c = Provider__c.Id AND Provider_Network_Agreement__c.Status__c = 'Active' are displayed.

Given an active PNA record is displayed,
When the specialist expands a PNA row,
Then Fee_Schedule__c.Name, Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.Expiration_Date__c, and the linked GSA (Group_Service_Agreement__c.Agreement_Number__c) are shown.

Given a PNA record has Provider_Network_Agreement__c.Expiration_Date__c within 90 days of today,
When the panel renders,
Then the PNA row displays an amber warning badge labeled 'Expiring Soon'.

Given a Provider__c has no active PNA records,
When the panel loads,
Then an empty state message 'No active network agreements found' is displayed and a 'Initiate PNA' quick action button is visible.

## Solution

Extend the existing 'Provider_360_FlexCard' OmniStudio FlexCard by adding a new 'Network Agreements' accordion section. Create a new DataRaptor Turbo Extract ('DR_Get_Provider_PNA_Summary') querying Provider_Network_Agreement__c with a child sub-query on Fee_Schedule__c and a lookup to Group_Service_Agreement__c. An OmniStudio Calculation Procedure evaluates the 90-day expiry threshold and returns a badge indicator value. The 'Initiate PNA' quick action invokes an existing OmniScript ('PNA_Initiation_OmniScript'). Ensure FLS checks are embedded in the DataRaptor for Provider Relations Specialist profile access.

## Description

As a Claims Operations Manager, I need an Agentforce AI agent to automatically triage incoming FNOL submissions by analyzing claim description text, loss type, and historical claim patterns so that claims are routed to the correct adjuster queue based on a computed complexity score, reducing manual triage time by an estimated 60%.

The AI agent should leverage Einstein Generative AI to parse free-text FNOL descriptions and classify claims into complexity tiers: 'Simple', 'Moderate', 'Complex', and 'SIU Referral'. Complex claims and SIU referrals require human review before assignment.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Stage__c = 'FNOL' and Claim__c.FNOL_Description__c is populated,
When the Agentforce FNOL Triage Agent is triggered via a Record-Triggered Flow,
Then the agent invokes an Einstein Prompt Template passing Claim__c.FNOL_Description__c, Claim__c.Loss_Type__c, and Claim__c.Estimated_Loss_Amount__c as inputs.

Given the Einstein Prompt Template returns a complexity classification,
When the AI response is parsed,
Then Claim__c.AI_Complexity_Score__c is populated with the numeric score (0–100) and Claim__c.AI_Complexity_Tier__c is set to one of: 'Simple', 'Moderate', 'Complex', 'SIU Referral'.

Given Claim__c.AI_Complexity_Tier__c = 'Simple' or 'Moderate',
When the auto-assignment logic executes,
Then Claim__c.OwnerId is set to the appropriate adjuster queue (Simple_Claims_Queue or Moderate_Claims_Queue) based on round-robin assignment using Omni-Channel routing configuration.

Given Claim__c.AI_Complexity_Tier__c = 'Complex',
When the classification is returned,
Then Claim__c.OwnerId is set to the Senior_Adjuster_Queue and Claim__c.Requires_Supervisor_Review__c = true, and a Task is created for the Claims Operations Manager with Subject = 'Complex FNOL Review Required'.

Given Claim__c.AI_Complexity_Tier__c = 'SIU Referral',
When the classification is returned,
Then a new Case is created with Case.RecordType = 'SIU_Investigation', Case.ParentId = Claim__c.Id, and the case is assigned to the SIU queue. Claim__c.Status__c is set to 'Pending SIU Review' and Claim__c.SIU_Referral_Date__c = TODAY().

Given the AI agent fails to return a classification within 10 seconds (timeout),
When the timeout condition is met,
Then Claim__c.AI_Complexity_Tier__c = 'Manual Review Required', the claim is routed to the General_Claims_Queue, and a Platform Event AI_Triage_Failure__e is published for monitoring.

## Solution

Configure an Agentforce Agent ('FNOL_Triage_Agent') with a custom Agent Action backed by an Einstein Prompt Template ('FNOL_Complexity_Classifier'). Prompt template uses a structured prompt with few-shot examples for each complexity tier. Agent Action invoked from a Record-Triggered Flow on Claim__c (fires on create when Stage__c = 'FNOL'). Response parsing handled by an Apex invocable method 'FNOLTriageResponseParser.parse()' that maps AI output to Claim__c fields. Omni-Channel routing rules configured for Simple/Moderate queues. SIU Case creation handled by a separate Apex method called from the same flow path. Platform Event AI_Triage_Failure__e monitored via a Flow subscription that logs to a custom AI_Error_Log__c object. Einstein Trust Layer enabled to ensure no PII is retained in prompt logs.

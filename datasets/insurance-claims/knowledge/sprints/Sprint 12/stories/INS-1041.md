## Description

As a licensed agent, I need the ability to generate and download a formatted commission statement PDF from the Agent Portal for any selected date range so that I can provide documentation to my agency, tax advisors, or carrier partners on demand.

This story centralizes document generation that was previously scoped within INS-1035 (Commission Dashboard). The statement should be generated using the existing Conga CLM template and include a breakdown of earned, pending, and chargeback Payment__c records for the selected period.

## Acceptance Criteria

Given an agent is viewing the Commission Dashboard in the Agent Portal,
When they click 'Download Statement' and select a date range,
Then a Conga CLM merge is triggered using template 'Commission_Statement_v2' and the output PDF is returned inline for download.

Given the PDF is generated,
When the agent downloads it,
Then it includes: agent name (Broker__c.Name), NPN (Broker__c.NPN__c), date range, itemized Payment__c records (earned, pending, chargeback), and a net commission total.

Given the statement generation is complete,
When the PDF is created,
Then a ContentDocument is attached to the Broker__c record and a Commission_Statement_Log__c entry is created with the generation timestamp and date range for audit purposes.

Given the selected date range contains no Payment__c records,
When the agent attempts to generate the statement,
Then a user-friendly message is displayed: 'No commission records found for the selected period' and no PDF is generated.

## Solution

Implement a Lightning Web Component (commissionStatementDownload) on the Agent Portal that accepts date range inputs and invokes a Conga CLM API callout via an Apex Controller (CommissionStatementController.cls). The Apex class queries Payment__c records filtered by date range and Payee_Type__c = 'Agent', constructs the Conga merge data map, and triggers the Conga CLM REST API. On success, the base64-encoded PDF response is returned to the LWC for browser download. ContentDocument insertion handled via ContentVersion insert in Apex. Commission_Statement_Log__c records written post-generation. Named Credential 'Conga_CLM_NC' used for API authentication.

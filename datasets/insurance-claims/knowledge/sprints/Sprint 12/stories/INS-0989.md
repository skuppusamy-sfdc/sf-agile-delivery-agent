## Description

As a Broker Relations Coordinator, I need a real-time commission dashboard within the Agent Portal so that agents can view their year-to-date (YTD) earnings, current tier status, and projected payouts across all active LOBs without contacting the back-office team. This dashboard should surface data from Commission_Transaction__c, Agent_Tier__c, and Policy__c records tied to the authenticated agent's Producer_Code__c.

## Acceptance Criteria

Given an authenticated agent logs into the Agent Portal,
When they navigate to the 'My Commission Dashboard' tab,
Then a FlexCard renders displaying: YTD gross commission (sum of Commission_Transaction__c.Amount__c where Status__c = 'Paid'), current tier (Agent_Tier__c.Tier_Level__c), and tier attainment percentage.

Given the dashboard is loaded,
When the agent selects a specific LOB from the LOB filter dropdown (values: Medical, Dental, Vision, Life),
Then all commission figures and policy counts filter to only records where Policy__c.Line_of_Business__c matches the selected value.

Given commission data exists for the trailing 12 months,
When the agent views the 'Trend' section,
Then a bar chart renders monthly commission totals using Commission_Transaction__c.Transaction_Date__c grouped by calendar month.

Given a Commission_Transaction__c record with Status__c = 'Pending',
When it appears in the dashboard,
Then it is displayed in a separate 'Pending Payouts' section and excluded from the YTD Paid total.

-Given the agent can export commission data to PDF via Conga CLM-
-When the export button is clicked, a Conga template generates a commission statement-
(Deferred to Sprint 14 per INS-1021)

Given the agent's tier changes (Agent_Tier__c.Tier_Level__c transitions from 'Silver' to 'Gold'),
When the change is saved in Salesforce,
Then the dashboard FlexCard reflects the updated tier within 60 seconds via a Platform Event (Commission_Tier_Updated__e) subscription.

Given any agent accesses the portal,
When they view the dashboard,
Then they can only see Commission_Transaction__c records where Agent__c = their own Contact ID, enforced via Sharing Rule and Permission Set Group: 'Agent_Portal_Standard'.

## Solution

Build the commission dashboard as an OmniStudio FlexCard ('Agent_Commission_Dashboard_FC') consuming a DataRaptor Turbo Extract that queries Commission_Transaction__c, Agent_Tier__c, and Policy__c. A Record-Triggered Flow on Commission_Transaction__c publishes a Commission_Tier_Updated__e Platform Event when Tier_Level__c changes, which the FlexCard subscribes to via an Emp API listener in the LWC wrapper. Sharing Rules scoped to Agent_Portal_Standard Permission Set Group restrict record visibility. Bar chart rendered via Chart.js embedded in a child LWC component. LOB filter state managed in OmniScript session variables.

## Description

As a Claims Operations Manager, I need the system to automatically generate an Electronic Remittance Advice (ERA / EDI 835) file when a batch of Payment__c records is approved for a TPA partner so that downstream TPA systems can auto-post remittance data without manual file preparation.

The ERA file must conform to the ANSI X12 835 standard and include Claim_Line__c adjudication detail for each included Claim__c.

## Acceptance Criteria

Given a Payment_Batch__c record transitions to Payment_Batch__c.Status__c = 'Approved' and Payment_Batch__c.Partner_Type__c = 'TPA',
When the Record-Triggered Flow fires,
Then an Apex batch job ('ERA_GenerationBatch') is enqueued to process all Payment__c records in the batch where Payment__c.Payment_Batch__c = Payment_Batch__c.Id.

Given the ERA generation batch runs,
When it processes each Payment__c record,
Then the output 835 file includes: ISA/GS envelope segments populated from TPA__c partner configuration, one CLP segment per Claim__c, and one SVC segment per Claim_Line__c with Claim_Line__c.CPT_Code__c, Claim_Line__c.Billed_Amount__c, Claim_Line__c.Allowed_Amount__c, and Claim_Line__c.Adjustment_Reason_Code__c.

Given the 835 file is generated successfully,
When the file is complete,
Then it is stored as a Salesforce File (ContentDocument) attached to the Payment_Batch__c record AND a Platform Event (ERA_Generated__e) is published with the ContentDocumentId for downstream SFTP delivery integration.

Given a Claim_Line__c has Claim_Line__c.COB_Applied__c = true,
When the SVC segment is built,
Then a CAS segment with COB adjustment reason code is included per X12 835 COB rules.

Given the batch encounters any Payment__c record with Payment__c.Status__c != 'Approved',
When that record is processed,
Then it is skipped and logged to a custom ERA_Exception_Log__c record with ERA_Exception_Log__c.Reason__c = 'Invalid Payment Status'.

-Given ERA files are also generated for non-TPA direct-pay batches- (Deprecated: ERA generation scope limited to TPA partners only per INS-1054 decision log.)

## Solution

Record-Triggered Flow on Payment_Batch__c (after-update, Status__c changes to 'Approved' and Partner_Type__c = 'TPA') enqueues the 'ERA_GenerationBatch' Apex class (implements Database.Batchable, Database.Stateful). The batch queries Payment__c and related Claim__c/Claim_Line__c records via a single composite SOQL query. An X12_835_Builder Apex utility class constructs the EDI string using StringBuilder patterns. The completed file is inserted via ContentVersion API and linked to Payment_Batch__c via ContentDocumentLink. A Platform Event (ERA_Generated__e) triggers a MuleSoft flow (via Change Data Capture listener) for SFTP delivery to TPA endpoints. Custom Metadata Type 'TPA_EDI_Config__mdt' stores ISA/GS envelope values per TPA.

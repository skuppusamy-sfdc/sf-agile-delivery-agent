## Description

As a licensed agent, I need a guided policy search and bind-quote workflow within the Agent Portal so that I can look up existing policyholders, generate a new Individual LOB quote, and bind the policy without leaving the portal experience.

The workflow must validate the applicant's eligibility, check for existing Policy__c records to avoid duplicate issuance, and trigger the underwriting referral path when the applicant's risk score exceeds the auto-bind threshold defined in the Group_Service_Agreement__c for the agent's appointed carrier.

## Acceptance Criteria

Given an agent initiates a new quote from the Agent Portal,
When they enter the applicant's SSN or Member ID,
Then the system searches existing Person Account (Provider__c) and Contact records and displays any active Policy__c records where Policy__c.Status__c IN ('Active', 'Pending Renewal').

Given no duplicate policy is found,
When the agent completes the applicant demographic form,
Then a new Policy__c record is created in Status__c = 'Quote' with Policy__c.Agent_of_Record__c populated from the logged-in user's Broker__c record.

Given the applicant's calculated risk score exceeds GSA__c.Auto_Bind_Threshold__c,
When the agent attempts to bind,
Then the workflow routes the quote to an Underwriting Analyst via a Case record (RecordType = 'Underwriting_Review') and sets Policy__c.Status__c = 'Pending Underwriting'.

Given the quote passes auto-bind rules,
When the agent clicks 'Bind Policy',
Then Policy__c.Status__c transitions to 'Active', Policy__c.Effective_Date__c is set to the next valid effective date, and a Welcome Kit task is created on the Policy__c record.

Given the bind action is successful,
When the policy is activated,
Then the agent's commission schedule is evaluated against the applicable Fee_Schedule__c and a Payment__c record is staged with Status__c = 'Pending' and Payee_Type__c = 'Agent'.

## Solution

Implement a multi-step OmniScript (OS_Agent_BindQuote_Individual) with DataRaptor Turbo Extract for duplicate policy lookup. Use a Screen Flow (SF_Underwriting_Referral) invoked conditionally via Apex Action when risk score threshold is breached. Policy__c status transitions managed by a Record-Triggered Flow with scheduled path for effective date assignment. Commission staging logic handled in Apex class (CommissionStagingService.cls) invoked as a Flow action post-bind. Integrate with the Underwriting Analyst queue via Omni-Channel routing configuration. Apply Validation Rules on Policy__c to enforce mandatory fields before Status__c = 'Active' is allowed.

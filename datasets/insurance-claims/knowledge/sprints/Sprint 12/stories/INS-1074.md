## Description

As a Broker Relations Coordinator, I need a real-time commission dashboard within the Agent Portal so that licensed agents can view their year-to-date (YTD) earnings, per-policy commission breakdowns, and pending disbursements without requiring manual reporting requests from the back office. The dashboard should support filtering by LOB (Auto, Health, Property) and display both earned and estimated future commissions based on active Policy__c records.

## Acceptance Criteria

Given an authenticated agent user with the 'Agent_Portal_User' Permission Set Group assigned,
When they navigate to the Commission Dashboard tab in the Agent Portal,
Then a FlexCard displays YTD earned commissions aggregated from Commission__c.Earned_Amount__c grouped by LOB__c field.

Given the agent has associated Policy__c records where Policy__c.Agent_Code__c matches the logged-in user's Agent_Code__c,
When the dashboard loads,
Then each policy row displays: Policy__c.Policy_Number__c, Policy__c.Effective_Date__c, Policy__c.LOB__c, Commission__c.Rate__c, Commission__c.Earned_Amount__c, and Commission__c.Status__c ('Pending'|'Disbursed'|'Clawed_Back').

Given the agent applies a LOB filter (e.g., 'Health'),
When the filter is submitted,
Then the DataRaptor Transform re-queries and returns only Commission__c records where Commission__c.LOB__c = 'Health' and the YTD total widget updates accordingly within 2 seconds.

Given a commission record has Commission__c.Status__c = 'Clawed_Back',
When it appears in the dashboard,
Then the row is highlighted in amber and a tooltip displays Commission__c.Clawback_Reason__c.

Given the agent clicks 'Export to CSV',
When the export is triggered,
Then a Conga CLM-generated report downloads containing all visible commission rows with no PII fields beyond the agent's own data.

-Given the dashboard auto-refreshes every 30 seconds via polling- (deprecated: replaced by Platform Event subscription on Commission__c updates)

Given a Platform Event Commission_Updated__e is published upon Commission__c record save,
When the agent's dashboard is open,
Then the FlexCard refreshes the affected row within 5 seconds without a full page reload.

## Solution

Build the Commission Dashboard as an OmniStudio FlexCard (CommissionDashboardCard) consuming a DataRaptor Extract that queries Commission__c joined to Policy__c via Commission__c.Policy__c lookup. A child FlexCard renders the per-policy breakdown table. LOB filter state is managed via OmniScript session variables passed into the DataRaptor as input parameters. Real-time refresh is achieved by subscribing to the Platform Event Commission_Updated__e using an LWC lightning/empApi connector embedded within the FlexCard action. CSV export invokes a Conga CLM template via REST callout from an Apex @AuraEnabled method. Permission isolation enforced via a custom Permission Set Group 'Agent_Portal_User' combined with a Sharing Rule on Commission__c scoped to Agent_Code__c = $User.Agent_Code__c.

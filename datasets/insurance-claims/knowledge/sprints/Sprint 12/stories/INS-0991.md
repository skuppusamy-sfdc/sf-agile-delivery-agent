## Description

As a Broker Relations Coordinator, I need the system to automatically calculate and create a Commission_Transaction__c record when a Policy__c record reaches Status__c = 'Active' so that commission payouts are triggered without manual intervention and are based on the agent's current tier rate defined in Agent_Tier__c. The calculation must account for LOB-specific rate tables and handle both new business and renewal scenarios.

## Acceptance Criteria

Given a Policy__c record transitions to Status__c = 'Active',
When the record is saved and Agent__c is populated,
Then a Commission_Transaction__c record is created with: Agent__c = Policy__c.Agent__c, Policy__c = the triggering Policy record, Transaction_Date__c = Policy__c.Effective_Date__c, Type__c = 'New Business' or 'Renewal' based on Policy__c.Is_Renewal__c, and Status__c = 'Pending'.

Given the Commission_Transaction__c record is being created,
When Agent_Tier__c.Tier_Level__c = 'Gold' and Policy__c.Line_of_Business__c = 'Medical',
Then Commission_Transaction__c.Rate__c = 0.045 and Amount__c = Policy__c.Annual_Premium__c * 0.045.

Given the Commission_Transaction__c record is being created,
When Policy__c.Is_Renewal__c = true,
Then Commission_Transaction__c.Type__c = 'Renewal' and the renewal rate from Agent_Tier__c.Renewal_Rate__c is applied instead of the new business rate.

Given a Policy__c record has no Agent__c populated,
When it transitions to Status__c = 'Active',
Then no Commission_Transaction__c is created and a Warning log entry is written to Commission_Calc_Error_Log__c with Error_Reason__c = 'No agent associated with policy'.

Given commission calculation completes successfully,
When Commission_Transaction__c.Status__c = 'Pending',
Then a notification email is sent to the assigned agent using the 'Commission_Pending_Notification' email template.

## Solution

Implement an Apex Trigger (PolicyCommissionTrigger) on Policy__c firing on after-update when Status__c changes to 'Active'. Trigger handler class (PolicyCommissionHandler) queries Agent_Tier__c for the current tier and LOB-specific rate, then bulkifies Commission_Transaction__c insertion. Rate lookup uses a custom metadata type (Commission_Rate_Config__mdt) keyed by Tier_Level__c and Line_of_Business__c to avoid hardcoded values. Error logging writes to Commission_Calc_Error_Log__c via a separate async Apex method. Email alert fired via a Record-Triggered Flow on Commission_Transaction__c insert.

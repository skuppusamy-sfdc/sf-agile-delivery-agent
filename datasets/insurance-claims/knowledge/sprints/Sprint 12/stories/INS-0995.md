## Description

As a licensed agent, I need a commission dispute submission form within the Agent Portal so that I can formally contest a commission calculation discrepancy, attach supporting documentation, and track the resolution status — all without calling the back-office team. Each dispute must auto-create a Salesforce Case record linked to the disputed Commission_Transaction__c and be subject to a defined SLA of 5 business days for first response.

## Acceptance Criteria

Given an agent is viewing a Commission_Transaction__c record in the portal,
When they click 'Dispute This Commission',
Then an OmniScript modal ('Commission_Dispute_OS') launches pre-populated with: Commission_Transaction__c.Id, Policy__c.Name, Transaction_Date__c, and Amount__c.

Given the dispute form is open,
When the agent selects a Dispute_Reason__c from the picklist (values: 'Incorrect Rate', 'Missing Policy', 'Tier Miscalculation', 'Duplicate Transaction', 'Other'),
Then if 'Other' is selected, a free-text Dispute_Notes__c field becomes required (minimum 50 characters).

Given the form is submitted,
When the OmniScript DataRaptor Transform executes,
Then a Case record is created with: RecordType = 'Commission_Dispute', Status = 'New', Priority = 'Medium', Commission_Transaction__c = disputed record Id, Subject = 'Commission Dispute — [Policy Name] — [Transaction Date]', and Origin = 'Agent Portal'.

Given the Case is created,
When Case.RecordType = 'Commission_Dispute',
Then the Entitlement Process 'Commission_Dispute_SLA_5BD' is applied, setting first-response milestone to 5 business days and Case.SlaStartDate = CreatedDate.

Given the Case SLA milestone is breached,
When the first-response milestone is not met within 5 business days,
Then an escalation email is sent to the Broker Relations Manager and Case.Priority escalates to 'High'.

Given the Case Status changes to 'Resolved' or 'Closed',
When the update is saved,
Then the agent receives an email notification using template 'Commission_Dispute_Resolved' and the portal's 'My Disputes' list view refreshes to reflect the new status.

-Given the agent can initiate a dispute directly from the Commission Dashboard FlexCard (INS-0989)-
-When the dispute button is clicked on a pending transaction in the FlexCard-
(Integration with INS-0989 FlexCard deferred to Sprint 13 — will require FlexCard action wiring update)

## Solution

Build Commission_Dispute_OS as an OmniScript with conditional field visibility driven by Dispute_Reason__c selection (OmniScript conditional visibility rules). DataRaptor Transform creates the Case record and links Commission_Transaction__c via a custom lookup field. Case RecordType assignment and Entitlement Process application handled via a Record-Triggered Flow on Case insert filtered by RecordType.DeveloperName = 'Commission_Dispute'. SLA milestone breach escalation configured via Entitlement Process milestone actions (email alert + field update). Agent email notification on Case closure via a second Record-Triggered Flow. 'My Disputes' list view on the portal uses a filtered DataRaptor Turbo Extract returning Cases where ContactId = running user's linked Contact and RecordType = 'Commission_Dispute'.

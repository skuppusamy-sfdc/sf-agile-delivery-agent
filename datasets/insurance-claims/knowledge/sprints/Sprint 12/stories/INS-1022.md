## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can immediately see the provider's active network agreements, current credentialing status, NPI/TIN verification state, and open claims count without navigating to multiple related lists.

This directly supports the Phase 2 goal of reducing average handle time for provider inquiries by 30% (per Sprint 11 retrospective target).

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then it displays the following sections without additional clicks:
  - Active PNAs: list of Provider_Network_Agreement__c records where Status__c = 'Active' showing PNA Name, Network_Name__c, Tier__c, and End_Date__c
  - Credentialing Status: Credentialing_Case__c.Status__c and Expiration_Date__c for the most recent open or approved case
  - NPI Verification: Provider__c.NPI_Verified__c checkbox state and NPI_Last_Verified_Date__c
  - Open Claims: COUNT of Claim__c records where Rendering_Provider__c = current provider and Claim__c.Status__c NOT IN ('Closed','Denied','Paid')

Given the provider has no active PNA records,
When the PNA section renders,
Then a warning badge displays 'No Active Network Agreement' in red with a quick-action button 'Initiate PNA' that opens the New PNA OmniScript.

Given Credentialing_Case__c.Expiration_Date__c is within 90 days,
When the credentialing section renders,
Then the expiration date field is highlighted amber and a 'Renew Credentialing' action button is surfaced.

Given the specialist clicks the Open Claims count,
When the action fires,
Then a modal datatable opens showing Claim__c.Claim_Number__c, Claim__c.Service_Date__c, Claim__c.Total_Billed_Amount__c, and Claim__c.Status__c for all matching open claims.

Given the FlexCard is viewed by a user without the 'Provider_Network_Read' Permission Set,
When the card attempts to load PNA data,
Then the PNA section is hidden and replaced with 'Insufficient permissions to view network data'.

## Solution

Build OmniStudio FlexCard 'Provider_360_FC' with four child card sections driven by separate DataRaptor Extract actions. The Open Claims count uses an Apex REST action to avoid SOQL row limits on high-volume providers. The 'Initiate PNA' button invokes the existing 'New_PNA_OmniScript' passing the Account ID as an input parameter. Credentialing 90-day warning uses a FlexCard conditional state with a formula comparing TODAY() to Expiration_Date__c. Permission-based section visibility is controlled via FlexCard Conditional Visibility rules checking the running user's Permission Set assignments via a custom UserPermissionService Apex class. FlexCard is added to the Provider__c Lightning record page via App Builder.

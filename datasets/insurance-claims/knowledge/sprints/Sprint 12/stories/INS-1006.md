## Description

As a Broker Relations Coordinator, I need the system to automatically calculate and post commission chargebacks when a policy lapses within the contractual clawback window so that agent commission balances are accurately maintained and manual reconciliation effort is eliminated.

A chargeback applies when Policy__c.Status__c transitions to 'Lapsed' and Policy__c.Days_In_Force__c is less than the chargeback threshold defined on the agent's Group_Service_Agreement__c (GSA__c.Chargeback_Days__c).

## Acceptance Criteria

Given a Policy__c record's Status__c changes to 'Lapsed',
When a Record-Triggered Flow evaluates the lapse event,
Then it queries the associated GSA__c record via Policy__c.GSA__c and compares Policy__c.Days_In_Force__c against GSA__c.Chargeback_Days__c.

Given Policy__c.Days_In_Force__c is less than GSA__c.Chargeback_Days__c,
When the chargeback condition is met,
Then a new Payment__c record is created with Payment__c.Payment_Type__c = 'Chargeback', Payment__c.Amount__c = (original commission amount * GSA__c.Chargeback_Percentage__c / 100), and Payment__c.Status__c = 'Posted'.

Given a chargeback Payment__c is created,
When the record is saved,
Then the originating agent's Broker_Account__c.Outstanding_Chargeback_Balance__c field is incremented by the chargeback amount via an after-insert Apex Trigger.

Given a chargeback is posted,
When the broker logs into the Agent Portal,
Then the Commission Dashboard (INS-1004) reflects the chargeback as a negative line item in the 'Adjustments' section.

Given the chargeback amount exceeds $5,000,
When the Payment__c record is created,
Then an automated email alert is sent to the assigned Broker Relations Coordinator using a Salesforce Email Alert action referencing the 'High_Value_Chargeback' email template.

## Solution

Record-Triggered Flow on Policy__c (fires on Status__c change to 'Lapsed') performs the GSA chargeback threshold evaluation and calls an Apex invocable method 'ChargebackCalculationService.calculateAndPost()' to handle Payment__c creation with proper currency handling. After-insert Apex Trigger on Payment__c updates Broker_Account__c balance via a queueable Apex job to avoid governor limit issues. Email Alert configured on Payment__c object with amount filter condition. Sharing Rules ensure broker community users can view their own chargeback Payment__c records.

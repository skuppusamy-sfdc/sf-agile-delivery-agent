## Description

As an agent user accessing the Agent Portal, I need a searchable policy list view and a Policy 360 detail page so that I can quickly locate bound policies I own, review coverage details, and identify open claims without navigating Salesforce Classic or calling the back office.

## Acceptance Criteria

Given an agent is logged into the Experience Cloud Agent Portal,
When they access the 'My Policies' page,
Then only Policy__c records where Policy__c.Agent__c = current user's Contact.Id are displayed, respecting Sharing Rules.

Given the agent enters a search string in the policy search bar,
When the search executes,
Then results filter Policy__c records by Policy__c.Policy_Number__c, Policy__c.Policyholder_Name__c, or Policy__c.TIN__c containing the search string (case-insensitive, minimum 3 characters).

Given the agent clicks into a specific Policy__c record,
When the Policy 360 page loads,
Then the following sections render: Policy Summary (Policy__c.Status__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, Policy__c.LOB__c), Open Claims (Claim__c records where Claim__c.Policy__c = Policy__c.Id AND Claim__c.Status__c NOT IN ('Closed','Denied')), and Commission Summary.

Given a Policy__c has zero open Claim__c records,
When the Open Claims section renders,
Then a 'No open claims' empty state message is displayed rather than an empty table.

-Given the agent can edit Policy__c fields inline from the 360 view- (Deprecated: edit access removed per compliance review, read-only portal enforced.)

## Solution

LWC component ('agentPolicy360') built as an Experience Cloud page using a combination of wire adapters (getRecord, getRelatedListRecords) for Policy__c and Claim__c data retrieval. A custom Apex controller ('AgentPolicySearchController') with @AuraEnabled(cacheable=true) methods handles the dynamic SOSL-based policy search. Sharing Rules on Policy__c enforce agent-level data visibility using 'Agent__c' lookup field to Contact. The Policy 360 layout is composed of sub-components: 'policyHeader', 'openClaimsList', 'commissionSummaryMini'. Deployed as a named Experience Cloud page with Guest profile access blocked.

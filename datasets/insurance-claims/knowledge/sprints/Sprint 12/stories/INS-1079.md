## Description

As a Claims Operations Manager, I need the Agentforce AI assistant embedded in the Claim__c record page to analyze the ICD-10 diagnosis codes and CPT procedure codes on associated Claim_Line__c records and auto-suggest appropriate denial reason codes so that adjusters can reduce manual lookup time and improve consistency in denial rationale across the claims team. The suggestion should reference the applicable coverage exclusion from the Group_Service_Agreement__c (GSA) where applicable.

## Acceptance Criteria

Given a Claim__c record with Status__c = 'Under_Review' and at least one Claim_Line__c child record with Claim_Line__c.ICD10_Code__c and Claim_Line__c.CPT_Code__c populated,
When the adjuster clicks 'Analyze Claim' in the Agentforce panel,
Then the Einstein AI action invokes the prompt template 'ClaimDenialReasonSuggestion_v2' and returns up to 3 ranked denial reason code suggestions with confidence scores.

Given the AI returns denial reason code suggestions,
When the adjuster reviews the panel,
Then each suggestion displays: Denial_Reason_Code__c.Code__c, Denial_Reason_Code__c.Description__c, AI confidence percentage, and — where matched — the relevant GSA exclusion clause from Group_Service_Agreement__c.Exclusion_Clause__c.

Given the adjuster accepts a suggested denial reason code,
When they click 'Apply',
Then Claim__c.Denial_Reason_Code__c is populated with the selected code, Claim__c.AI_Suggestion_Accepted__c is set to TRUE, and an audit entry is created on the Claim__c chatter feed noting the AI-assisted decision.

Given the adjuster rejects all AI suggestions,
When they click 'Dismiss',
Then Claim__c.AI_Suggestion_Accepted__c is set to FALSE and Claim__c.AI_Suggestion_Dismissed_Reason__c is required before dismissal is saved.

Given a Claim__c has Claim__c.SIU_Flag__c = TRUE,
When the Agentforce analysis runs,
Then the AI response includes an additional SIU referral recommendation and a prompt to notify the SIU team via a Quick Action.

Given the AI model cannot determine a suggestion with confidence > 40%,
When the analysis completes,
Then the panel displays: 'Insufficient data to generate a confident suggestion. Manual review required.' and no codes are auto-applied.

## Solution

Implement Agentforce Agent Action using Einstein Generative AI with a custom Prompt Template (ClaimDenialReasonSuggestion_v2) that merges Claim_Line__c ICD-10 and CPT fields via a Prompt Template merge field block. The action is exposed as an Apex Invocable Action (AgentforceClaimDenialAnalyzer.cls) called from the Agentforce Agent configuration. GSA exclusion clause matching uses a SOQL subquery on Group_Service_Agreement__c filtered by Claim__c.Group_Policy__c. Results are rendered in an LWC (agentforceClaimPanel) embedded in the Claim__c Lightning Record Page via a custom tab. Audit chatter post is created via ConnectApi.ChatterFeeds in the Apex action. SIU flag check is a conditional branch in the Apex action prior to returning the response payload.

## Description

As a policyholder (insured member), I need to log into the Policyholder Portal and download my Explanation_of_Benefits__c (EOB) documents for any processed claim so that I can reconcile my medical bills, submit to secondary insurers for COB purposes, and access my records without calling the member services line.

EOB downloads must be logged for audit purposes. DocuSign certificate-of-completion style access logs are required by our Compliance Officer for HIPAA access tracking.

This story builds upon the Policyholder 360 experience established in Phase 1 and extends the Experience Cloud portal site 'AcmeMember_Portal'.

## Acceptance Criteria

Given an authenticated policyholder logs into the AcmeMember_Portal Experience Cloud site,
When they navigate to 'My Claims' and select a Claim__c with Status__c = 'Paid' or 'Adjudicated',
Then a 'View EOB' button is visible and enabled only when Explanation_of_Benefits__c.Status__c = 'Final' and EOB is linked to the claim via Explanation_of_Benefits__c.Claim__c.

Given the policyholder clicks 'View EOB',
When the action fires,
Then the EOB PDF (stored as ContentDocument linked to Explanation_of_Benefits__c) is rendered in an in-portal preview modal AND a download link is provided — no redirect to an external URL.

Given the EOB document is accessed,
When the download or preview event occurs,
Then a Document_Access_Log__c record is created with: Member__c = current user's Contact, EOB__c = Explanation_of_Benefits__c.Id, Access_Timestamp__c = NOW(), Access_Type__c = 'Download' or 'Preview', IP_Address__c = request remote address, and Session_Id__c = current session identifier.

Given a policyholder attempts to access an EOB for a Claim__c not associated with their Policy__c,
When the request is evaluated,
Then a Sharing Rule and Apex-enforced CRUD/FLS check blocks access and logs an Access_Denied_Log__c record with Denial_Reason__c = 'Unauthorized Policy Access'.

Given Explanation_of_Benefits__c.Status__c = 'Draft' or 'Voided',
When the claim is viewed,
Then the 'View EOB' button is hidden and a message reads 'Your EOB is being prepared and will be available within 3–5 business days.'

-Given the policyholder requests ERA (Electronic Remittance Advice) export, an 835 file should be downloadable from the portal- (deprecated: ERA/835 export descoped to Phase 3 per product roadmap decision 2024-11-01).

Given 500 concurrent portal users each request EOB downloads simultaneously,
When load testing is performed in the staging environment,
Then average response time for the EOB modal render is ≤ 3 seconds at p95 with no Apex CPU limit exceptions.

## Solution

LWC component 'eobDownloadViewer' deployed to the AcmeMember_Portal Experience Cloud site. Component uses a wire adapter to query Explanation_of_Benefits__c (with Sharing Rules ensuring member-only visibility) and retrieves ContentDocument via ConnectApi.ContentVersion. Document preview uses the Salesforce Files Preview component embedded in an LWC modal. Document_Access_Log__c insertion handled by an Apex @AuraEnabled method with 'with sharing' enforcement and explicit CRUD checks. IP address captured via a custom Apex method reading ApexPages.currentPage().getHeaders().get('X-Forwarded-For'). Sharing Rule on Explanation_of_Benefits__c: share to portal user where EOB.Claim__c.Policy__c.Policyholder__c = $User.ContactId. Load test executed via JMeter against a Full Sandbox prior to production deployment. HIPAA access log retention policy set to 7 years via a Data Retention Policy on Document_Access_Log__c.

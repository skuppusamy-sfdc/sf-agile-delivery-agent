## Description

As a Claims Operations Manager, I need Agentforce to analyze pended Claim_Line__c records and suggest the most appropriate denial reason code (from our standard Denial_Reason__mdt custom metadata) so that claims examiners can resolve pended lines faster and with greater consistency, reducing examiner-to-examiner variance in denial coding.

The AI suggestion must be explainable — the examiner must see why Agentforce recommended a given code before accepting or overriding it. This feature is subject to Compliance Officer review prior to go-live.

## Acceptance Criteria

Given a Claim_Line__c record enters Status__c = 'Pended',
When the record is opened by a claims examiner,
Then an Agentforce prompt panel appears on the record page displaying up to 3 ranked denial reason code suggestions sourced from Denial_Reason__mdt, each with a confidence percentage and a natural-language explanation of why the code was suggested (e.g., 'CPT code 99213 billed exceeds frequency limit of 4 per year per policy LOB rules').

Given Agentforce returns suggestions,
When the examiner clicks 'Accept' on a suggestion,
Then Claim_Line__c.Denial_Reason_Code__c is populated with the selected Denial_Reason__mdt.DeveloperName, Claim_Line__c.AI_Suggestion_Accepted__c = true, and Claim_Line__c.AI_Confidence_Score__c is stamped with the accepted suggestion's confidence value.

Given the examiner selects a different denial code than any suggestion,
When they save the record,
Then Claim_Line__c.AI_Suggestion_Accepted__c = false and Claim_Line__c.AI_Override_Reason__c is required (validated by a Flow before save) to capture the examiner's rationale for audit trail.

Given Agentforce cannot generate a suggestion (e.g., insufficient context, ICD-10 code missing),
When the prompt panel renders,
Then a message displays 'AI suggestion unavailable — ICD-10 code required' and the panel offers a link to populate Claim_Line__c.ICD10_Code__c.

Given 30 days of production usage,
When the Compliance Officer runs the AI Audit report,
Then a ComplianceReporting dashboard shows: total AI suggestions made, acceptance rate %, override rate %, and top 5 overridden codes — all filterable by examiner, LOB, and date range.

-Given Agentforce confidence score < 40%, the suggestion should be automatically suppressed- (deprecated: threshold lowered to 25% after pilot testing showed high false-negative suppression rate — see Slack thread #claims-ai-pilot).

## Solution

Configure an Agentforce Agent Action using a custom Apex invocable class 'ClaimLineDenialSuggestionAction' that assembles a structured prompt context from Claim_Line__c fields (CPT_Code__c, ICD10_Code__c, Service_Date__c, Billed_Amount__c, Provider_Network_Agreement__c.Network_Tier__c, Policy__c.LOB__c) and submits to the Einstein LLM via the Salesforce AI Platform API. The prompt template is stored in a Prompt Template record ('Denial_Code_Suggestion_v1') managed via Prompt Builder. Denial_Reason__mdt records are injected into the prompt as a grounding list. Suggestions are returned as a JSON array parsed by the action and rendered in an LWC sidebar component ('claimLineDenialSuggestions') on the Claim_Line__c record page. Audit fields (AI_Suggestion_Accepted__c, AI_Confidence_Score__c, AI_Override_Reason__c) added to Claim_Line__c. Compliance dashboard built as a CRM Analytics (Tableau CRM) dashboard on the AI_Audit_Dataset.

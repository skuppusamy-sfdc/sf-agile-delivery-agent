## Description

As an Assignment Coordinator, I need the system to automatically generate a Group_Service_Agreement__c (GSA) document and route it for electronic signature via DocuSign when an employer Account record is approved for group coverage so that onboarding cycle time is reduced and manual document preparation is eliminated. The GSA document should be generated using the Conga CLM template library and pre-populated with employer, broker, and coverage details from Salesforce.

## Acceptance Criteria

Given an employer Business Account's Onboarding_Status__c transitions to 'Approved' (updated by an Underwriting Analyst),
When the Record-Triggered Flow 'GSA_GenerationFlow' fires (After Update),
Then a new Group_Service_Agreement__c record is created:
  - Group_Service_Agreement__c.Account__c = employer Account.Id
  - Group_Service_Agreement__c.Effective_Date__c = Account.Coverage_Effective_Date__c
  - Group_Service_Agreement__c.Status__c = 'Pending Signature'
  - Group_Service_Agreement__c.LOB__c = Account.Primary_LOB__c
  - Group_Service_Agreement__c.Broker__c = Account.Assigned_Broker__c

Given the Group_Service_Agreement__c record is created,
When the Conga CLM Apex integration is invoked,
Then a GSA document is generated using the 'GSA_Standard_Template_v3' Conga template,
AND the document is attached to the Group_Service_Agreement__c record as a Salesforce File (ContentDocument),
AND Group_Service_Agreement__c.Document_Generated_Date__c is stamped with TODAY().

Given the GSA document is generated successfully,
When the DocuSign routing step executes,
Then a DocuSign envelope is sent to the following signatories in order:
  1. Employer primary contact (Account.Primary_Contact__c → Contact.Email)
  2. Assigned Broker (Account.Assigned_Broker__c → Contact.Email)
  3. Acme Insurance countersigner (defined in Custom Metadata 'GSA_Countersigner__mdt')
AND Group_Service_Agreement__c.DocuSign_Envelope_Id__c is populated.

Given all signatories complete the DocuSign envelope,
When the DocuSign status webhook fires,
Then Group_Service_Agreement__c.Status__c transitions to 'Fully Executed',
AND Account.Onboarding_Status__c transitions to 'GSA Executed',
AND a welcome email is dispatched to the employer's primary contact using the 'Employer_Welcome_GSA_Executed' email template.

Given Account.Primary_LOB__c = 'Self-Funded',
When the GSA generation flow evaluates the LOB,
Then the Conga template used is 'GSA_SelfFunded_Template_v2' instead of the standard template,
AND a Compliance_Review_Task__c is created and assigned to the Compliance Officer role queue for self-funded plan review.

-Given the GSA routing includes a wet-signature fallback option for employers without email on file- (deprecated — all onboarding requires valid email per updated compliance policy effective Sprint 11)

Given the Conga CLM callout fails (HTTP timeout or error response),
When the flow catches the exception,
Then Group_Service_Agreement__c.Status__c is set to 'Document Generation Failed',
AND a Platform Event 'GSAGenerationError__e' is published,
AND an email alert is sent to the Assignment Coordinator queue.

## Solution

Record-Triggered Flow 'GSA_GenerationFlow' on Account object (After Update, Onboarding_Status__c changed to 'Approved') calls an Apex invocable method 'GSAOrchestrator.initiate(accountId)'. The Apex class: (1) creates the Group_Service_Agreement__c record, (2) invokes the Conga CLM REST API via a named credential to generate the document from the appropriate template (LOB-driven template selection via Custom Metadata 'GSA_Template_Map__mdt'), (3) attaches the returned document as ContentVersion/ContentDocumentLink, (4) triggers the DocuSign envelope via DocuSign for Salesforce Apex Toolkit with ordered recipient routing. DocuSign webhook handled by existing REST endpoint extended to support GSA object updates. Error handling via try/catch publishes 'GSAGenerationError__e' Platform Event. Compliance task creation for self-funded plans handled in a separate sub-flow triggered by Platform Event to keep the main flow governor-limit-safe.

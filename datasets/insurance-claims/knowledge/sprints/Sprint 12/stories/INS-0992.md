## Description

As a Provider Relations Specialist, I need the Provider 360 FlexCard to display active Provider_Network_Agreement__c (PNA) details and associated Fee_Schedule__c line items so that I can quickly assess a provider's contracted rates and network participation status without navigating multiple related lists. The FlexCard must support both Person Account (individual provider) and Business Account (Facility__c) record types.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider (Person Account) record,
When the Provider_360_FC FlexCard loads,
Then it displays all PNA records where Provider_Network_Agreement__c.Provider__c = current Account Id AND Provider_Network_Agreement__c.Status__c = 'Active', showing: Contract_Start_Date__c, Contract_End_Date__c, Network__c, and Specialty__c.

Given an active PNA is displayed,
When the specialist clicks 'View Fee Schedule',
Then a child FlexCard panel expands showing Fee_Schedule__c records linked to that PNA, displaying: CPT_Code__c, Description__c, Allowed_Amount__c, and Effective_Date__c.

Given a Facility (Business Account) record is opened,
When the Provider_360_FC FlexCard loads,
Then it correctly queries Provider_Network_Agreement__c.Facility__c instead of Provider__c and renders facility-specific network agreements.

Given a PNA has Provider_Network_Agreement__c.Expiration_Date__c within 90 days of today,
When it is rendered in the FlexCard,
Then it is highlighted with a yellow warning badge labeled 'Expiring Soon'.

Given a PNA has Provider_Network_Agreement__c.Status__c = 'Terminated',
When the FlexCard loads,
Then terminated PNAs are hidden by default but accessible via a 'Show Terminated' toggle.

-Given the FlexCard includes an inline edit button for Allowed_Amount__c on Fee_Schedule__c-
-When the specialist modifies the rate inline, it saves directly to Fee_Schedule__c-
(Removed — inline rate editing requires Underwriting Analyst approval workflow, deferred to INS-1034)

## Solution

Build Provider_360_FC as an OmniStudio FlexCard with two DataRaptor Turbo Extract data sources: one querying Provider_Network_Agreement__c filtered by Provider__c or Facility__c (conditional based on Account.RecordType.DeveloperName), and a second querying Fee_Schedule__c filtered by PNA Id passed as a flyout parameter. The 'Expiring Soon' badge uses a Conditional Action on the FlexCard state comparing Contract_End_Date__c to TODAY() + 90. The 'Show Terminated' toggle updates a FlexCard state variable that controls the DataRaptor filter. FlexCard is embedded on both Person Account and Business Account lightning pages via App Builder with record type visibility filters.

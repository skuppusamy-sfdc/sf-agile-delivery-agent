## Description

As an Assignment Coordinator, I need a unified policy search experience within the Agent Portal so that agents can quickly locate policyholder records, review coverage details, and identify open claims without toggling between multiple Salesforce apps or submitting IT requests.

The search should support lookup by Policy Number, Policyholder Name, TIN, or NPI (for group health policies), and surface a summarized 360 view of the matched Policy__c record.

## Acceptance Criteria

Given an agent enters a valid Policy__c.Policy_Number__c in the search bar,
When they submit the search,
Then the portal should return the matching Policy__c record with fields: Policy__c.Status__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, Policy__c.Line_of_Business__c, and Policyholder name (from Policy__c.Account__r.Name).

Given an agent searches by partial policyholder name (minimum 3 characters),
When results are returned,
Then a list of up to 10 matching Policy__c records should display, sorted by Policy__c.Effective_Date__c descending.

Given a Policy__c record is returned in search results,
When the agent clicks 'View Details',
Then an OmniScript side panel should open showing open Claim__c records (Claim__c.Status__c NOT IN ('Closed','Withdrawn')) linked via Claim__c.Policy__c, with Claim__c.Claim_Number__c, Claim__c.FNOL_Date__c, and Claim__c.Total_Incurred__c visible.

Given an agent searches using an NPI that matches a Provider__c (Person Account) linked to a group health Policy__c via Group_Service_Agreement__c,
When results render,
Then the policy card should display a 'Group Policy' badge and the GSA__c.Agreement_Name__c.

Given a search returns no results,
When the no-results state renders,
Then the portal should display 'No policies found. Verify the Policy Number or contact your Broker Relations Coordinator.' with a pre-populated email template link.

## Solution

LWC component ('AgentPolicySearch') with a debounced search input invoking an Apex @AuraEnabled method that queries Policy__c with SOQL LIKE filters across Policy_Number__c, Account__r.Name, and a custom indexed field Policy__c.TIN_NPI_Search__c (formula concatenating TIN and NPI). Results rendered in a lightning-datatable. 'View Details' triggers an OmniScript ('PolicyDetailSidePanel') loaded in a modal, with a DataRaptor Extract fetching related Claim__c records. Custom object-level Sharing Rules ensure agents only see policies where Policy__c.Assigned_Agent__c = running user. Indexed fields on Policy_Number__c and TIN_NPI_Search__c for query performance.

## Description

As an agent user on the Agent Portal, I need a guided quote-to-bind OmniScript workflow for Property & Casualty (P&C) lines so that I can search existing policies, create new quotes, and bind coverage without leaving the portal or contacting an underwriter for standard risk tiers.

The workflow must validate risk eligibility against the Group_Service_Agreement__c (GSA) terms applicable to the agent's book of business and auto-populate policyholder data from existing Provider__c (Person Account) records where available.

## Acceptance Criteria

Given an agent initiates a new quote from the Agent Portal,
When they enter a policyholder TIN or SSN in the search field,
Then the OmniScript DataRaptor performs a lookup against Provider__c.Tax_ID__c and pre-populates applicant fields if a matching record is found.

Given the policyholder data is loaded,
When the agent selects a LOB of 'Property' or 'Auto',
Then the OmniScript presents only the coverage options permitted under the agent's active Group_Service_Agreement__c where GSA__c.Status__c = 'Active' and GSA__c.LOB__c includes the selected line.

Given the agent completes all required quote fields,
When they click 'Calculate Premium',
Then an Apex-invoked rating engine call returns a premium estimate within 5 seconds and populates Policy__c.Estimated_Premium__c on a draft record with Policy__c.Status__c = 'Quote'.

Given the risk tier is evaluated as 'Standard' by the rating engine,
When the agent clicks 'Bind Policy',
Then Policy__c.Status__c transitions to 'Active', Policy__c.Effective_Date__c is set to today, and a DocuSign envelope is auto-sent to the policyholder email via the DocuSign Apex toolkit.

Given the risk tier is evaluated as 'Non-Standard' or 'High Risk',
When the bind action is attempted,
Then the OmniScript displays an inline message: 'This policy requires Underwriter review. A case has been submitted.' and creates a new Case record with Case.RecordType = 'Underwriting_Review' assigned to the Underwriting Analyst queue.

Given a policy is successfully bound,
When the transaction completes,
Then the agent's broker Account record is updated via a Record-Triggered Flow incrementing Broker_Account__c.Policies_Bound_YTD__c by 1 and a commission Payment__c record is created with Payment__c.Status__c = 'Pending' and Payment__c.Payment_Type__c = 'Commission'.

## Solution

Implement a multi-step OmniScript ('P&C_Quote_Bind_OS') with DataRaptor Turbo Extract for policyholder lookup against Provider__c. Rating engine integration via an Apex REST callout to the internal pricing microservice. DocuSign envelope creation using DocuSign Apex Toolkit's 'sendEnvelope' method triggered from OmniScript action. Risk tier routing logic implemented in an Apex-invoked Decision element. Record-Triggered Flow on Policy__c fires after Status__c changes to 'Active' to create Payment__c and update broker counters. OmniScript embedded in the Agent Portal Lightning Community page via an Aura wrapper component.

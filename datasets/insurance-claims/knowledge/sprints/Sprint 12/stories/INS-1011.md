## Description

As a Broker Relations Coordinator, I need a real-time commission dashboard within the Agent Portal so that agents can view their earned, pending, and paid commissions broken down by Line of Business (LOB) without requiring manual report exports or support tickets to finance.

This dashboard should surface commission data from Payment__c records linked to Policy__c via the producing agent relationship, segmented by LOB (Commercial, Personal, Group Health, Workers Comp). Extends INS-0981 (Agent Portal base frame).

## Acceptance Criteria

Given an authenticated agent user on the Agent Portal home page,
When the Commission Dashboard tab is selected,
Then a FlexCard displays three KPI tiles: 'Earned This Month', 'Pending Approval', and 'YTD Paid', each sourced from Payment__c.Amount__c aggregated by Payment__c.Commission_Status__c.

Given commission records exist for multiple LOBs,
When the agent views the dashboard,
Then a bar chart renders LOB-segmented totals using Policy__c.Line_of_Business__c as the grouping dimension, with values in USD formatted to two decimal places.

Given a Payment__c record transitions to Status = 'Paid',
When the Platform Event Commission_Status_Changed__e fires,
Then the FlexCard KPI tiles refresh within 10 seconds without a full page reload.

Given an agent has sub-producers under their hierarchy,
When the 'Include Downline' toggle is enabled,
Then the dashboard aggregates commission totals for all sub-producer Agent__c records linked via Agent_Hierarchy__c.Parent_Agent__c.

-Given a CSV export button is present on the dashboard, When clicked, Then a Conga CLM template generates a commission statement PDF-
(Deprecated: PDF export moved to INS-1017)

Given the agent has zero commission records for the current month,
When the dashboard loads,
Then an empty-state illustration and message 'No commissions recorded for this period' is displayed rather than null values or errors.

## Solution

Build a FlexCard (OmniStudio) consuming a DataRaptor Turbo Extract sourced from Payment__c with filter on Commission_Agent__c = current user's Agent__c record. Use a DataRaptor Transform to pivot LOB breakdowns. Platform Event (Commission_Status_Changed__e) published via Record-Triggered Flow on Payment__c Status__c change to 'Paid'; LWC event listener on the FlexCard refreshes DataRaptor action. Agent hierarchy aggregation handled by an Apex @AuraEnabled method using a recursive SOQL query against Agent_Hierarchy__c. Permission Set Group 'Agent_Portal_Commission_View' gates access. Deploy behind a custom permission 'View_Commission_Dashboard'.

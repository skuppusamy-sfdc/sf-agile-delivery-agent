## Description

As a licensed agent using the Agent Portal, I need to search for existing policyholder records and submit a bind request for an approved quote so that I can complete the policy issuance process end-to-end without switching to the internal Salesforce UI. The bind submission must trigger the standard FNOL-eligibility check and underwriting hold validation before Policy__c.Status__c is set to 'Active'.

## Acceptance Criteria

Given an agent enters a policyholder's last name, date of birth, and ZIP code in the search panel,
When the search executes,
Then results return matching Person Account records (Provider__c type excluded) within 2 seconds,
AND each result displays: Full Name, Policy__c.Policy_Number__c, Policy__c.Effective_Date__c, and Policy__c.Status__c.

Given an agent selects a quote record where Policy__c.Status__c = 'Approved Quote',
When the agent clicks 'Bind Policy',
Then an OmniScript 'BindQuoteOS' launches with pre-populated fields from the selected Policy__c record.

Given the OmniScript reaches the signature step,
When the agent submits,
Then a DocuSign envelope is sent to the policyholder's email (Contact.Email) via the DocuSign for Salesforce integration,
AND Policy__c.DocuSign_Envelope_Status__c is set to 'Sent'.

Given the DocuSign envelope status updates to 'Completed',
When the Record-Triggered Flow on Policy__c fires,
Then Policy__c.Status__c transitions to 'Active',
AND Policy__c.Bind_Date__c is stamped with TODAY(),
AND the bound policy appears in the agent's 'My Active Policies' list on the portal.

Given Policy__c has an open Underwriting_Hold__c record (Underwriting_Hold__c.Policy__c = Policy__c.Id AND Status__c = 'Open'),
When the agent attempts to bind,
Then the OmniScript displays a blocking error: 'This policy has an active underwriting hold. Contact your Underwriting Analyst.',
AND the bind action is prevented.

## Solution

LWC 'agentPolicySearch' component embedded in the Agent Portal Experience Cloud page, using @wire(getRecords) with a SOSL search Apex controller. OmniScript 'BindQuoteOS' built in OmniStudio with DataRaptor Load to update Policy__c on completion. DocuSign envelope triggered via Apex callout in the OmniScript Integration Procedure step. Record-Triggered Flow on Policy__c monitors DocuSign_Envelope_Status__c field changes (Before Save, After Save) to transition Status__c. Underwriting hold check implemented as an Apex before-trigger validation on Policy__c to block status transition when open holds exist.

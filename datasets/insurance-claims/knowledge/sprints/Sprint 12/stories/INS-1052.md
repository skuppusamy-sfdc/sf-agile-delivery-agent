## Description

As a Broker Relations Coordinator, I need the system to automatically calculate and create Commission_Transaction__c records when a new Policy__c is issued or renewed so that commission accruals are accurate, timely, and no longer require manual spreadsheet entry by the back office team.

Commission rates are stored on the Agent_Commission_Schedule__c object and vary by LOB, policy type, and whether the policy is new business or a renewal.

## Acceptance Criteria

Given a Policy__c record transitions to Policy__c.Status__c = 'Active' with Policy__c.Is_New_Business__c = true,
When the Record-Triggered Flow fires on the Policy__c after-save event,
Then a Commission_Transaction__c record should be created with: Commission_Transaction__c.Policy__c = the issuing policy, Commission_Transaction__c.Agent__c = Policy__c.Assigned_Agent__c, Commission_Transaction__c.Type__c = 'New Business', Commission_Transaction__c.Status__c = 'Pending', and Commission_Transaction__c.Amount__c calculated as Policy__c.Annual_Premium__c multiplied by the matching Agent_Commission_Schedule__c.New_Business_Rate__c.

Given a Policy__c transitions to Status__c = 'Active' with Policy__c.Is_Renewal__c = true,
When the flow fires,
Then Commission_Transaction__c.Type__c = 'Renewal' and the rate applied should be Agent_Commission_Schedule__c.Renewal_Rate__c for the matching LOB.

Given no matching Agent_Commission_Schedule__c record exists for the policy's LOB and agent tier,
When the flow attempts commission calculation,
Then no Commission_Transaction__c should be created and a Task should be created assigned to the Broker Relations Coordinator queue with Subject = 'Missing Commission Schedule – Review Required' referencing the Policy__c.Id.

-Given commission transactions are created in bulk during open enrollment, a batch process will handle creation- (superseded by Record-Triggered Flow with async path to handle bulk volumes)

Given a Commission_Transaction__c record is created successfully,
When its Status__c = 'Pending',
Then it should appear in the agent's Commission Dashboard (INS-1050) Pending Commission tile within one platform event cycle.

## Solution

Record-Triggered Flow on Policy__c (trigger: after save, entry criteria: Status__c changes to 'Active' AND (Is_New_Business__c = true OR Is_Renewal__c = true)). Flow uses a Get Records element to find the matching Agent_Commission_Schedule__c by LOB and Agent_Tier__c. A Decision element branches on whether a schedule was found. On match: Create Records element creates Commission_Transaction__c with formula-calculated Amount__c. On no-match: Create Records element creates a Task routed to the BrokerRelationsQueue. Async path used for bulk-safe execution. Platform Event CommissionCreated__e published post-creation to notify the Agent Portal dashboard (INS-1050).

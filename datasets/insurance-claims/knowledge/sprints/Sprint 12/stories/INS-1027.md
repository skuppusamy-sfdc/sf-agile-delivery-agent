## Description

As a Broker Relations Coordinator, I need agents to have access to a real-time commission dashboard within the Agent Portal so that they can track year-to-date earnings, current tier status, and projected bonuses without submitting manual inquiries to the back office. This story extends INS-0997 (Base Agent Portal Login and SSO) and adds the commission tracking layer on top of the authenticated session.

## Acceptance Criteria

Given an authenticated agent user with the 'Agent_Portal_User' Permission Set Group assigned,
When they navigate to the Commission Dashboard tab in the Agent Portal,
Then a FlexCard renders showing YTD gross commission (summed from Commission_Transaction__c.Amount__c where Commission_Transaction__c.Agent__c = current user's linked Contact),
AND current tier status is derived from Commission_Tier__c.Tier_Label__c based on the agent's rolling 12-month written premium on Policy__c records.

Given a commission transaction is posted (Commission_Transaction__c.Status__c = 'Posted'),
When the Platform Event 'CommissionPosted__e' fires,
Then the FlexCard data refreshes within 60 seconds without a full page reload,
AND the YTD total reflects the new transaction amount.

Given an agent is within 10% of the next commission tier threshold,
When the dashboard loads,
Then a contextual alert banner displays: 'You are $[X] away from reaching [Tier Name] status',
AND the threshold value is read from Commission_Tier__c.Min_Premium__c.

Given an agent selects a specific LOB filter (e.g., 'Commercial Auto', 'Workers Comp'),
When the filter is applied,
Then the YTD earnings chart re-renders showing only Commission_Transaction__c records where Policy__c.Line_of_Business__c matches the selected LOB.

-Given the dashboard includes a downloadable PDF of commission statements via Conga CLM integration- (deprecated — moved to INS-1041 in Sprint 13)

Given an agent has zero commission transactions in the current calendar year,
When the dashboard loads,
Then the FlexCard displays '$0.00 YTD' with an empty-state illustration and a CTA link to the 'New Business Submission' page.

## Solution

Build a FlexCard ('AgentCommissionDashboard') using OmniStudio that calls a DataRaptor Extract to aggregate Commission_Transaction__c records filtered by agent Contact lookup. A Record-Triggered Flow on Commission_Transaction__c (After Insert/Update, Status__c = 'Posted') publishes a 'CommissionPosted__e' Platform Event. A second Flow subscribed to that event calls the DataRaptor refresh endpoint to invalidate the cache. Tier logic encapsulated in an Apex invocable method 'CommissionTierEvaluator.evaluate()' called from within the OmniScript context. Permission Set Group 'Agent_Portal_User' grants read access to Commission_Transaction__c, Commission_Tier__c, and Policy__c (filtered via Sharing Rules scoped to Agent__c = current user).

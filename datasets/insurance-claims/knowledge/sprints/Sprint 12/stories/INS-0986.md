## Description

As a licensed broker, I need the ability to submit a commission dispute directly from the Agent Portal and track its resolution status so that I can flag discrepancies in my commission payments without relying on email or phone communication with the Broker Relations team. Each dispute should be tied to a specific Commission_Line__c record and routed automatically to the appropriate Broker Relations Coordinator for review.

## Acceptance Criteria

Given a broker views a Commission_Line__c record in their Agent Portal Commission History,
When they click 'Dispute This Commission',
Then a modal OmniScript form opens pre-populated with Commission_Line__c.Id, Commission_Line__c.Policy__c, and Commission_Line__c.Calculated_Amount__c.

Given the broker completes the dispute form with Dispute_Reason__c (picklist: Incorrect Rate, Missing Override, Wrong Policy, Other) and Dispute_Notes__c (text area, max 1000 chars),
When the broker submits the form,
Then a Commission_Dispute__c record is created with Commission_Dispute__c.Status__c = 'Submitted', Commission_Dispute__c.Commission_Line__c = related record, and Commission_Dispute__c.Submitted_By__c = logged-in user.

Given a Commission_Dispute__c record is created,
When the Record-Triggered Flow fires,
Then the dispute is assigned to the Broker_Relations_Queue and an email notification is sent to the queue using an Email Alert with template 'Commission_Dispute_New_Submission'.

Given a Broker Relations Coordinator updates Commission_Dispute__c.Status__c to 'Under Review', 'Resolved – Approved', or 'Resolved – Denied',
When the status changes,
Then the broker receives an automated email notification with the new status and Commission_Dispute__c.Resolution_Notes__c if populated.

Given the broker views the 'My Disputes' tab in the Agent Portal,
When the page loads,
Then a list view displays all Commission_Dispute__c records where Commission_Dispute__c.Submitted_By__c = current user, showing Status__c, Dispute_Reason__c, Commission_Line__c.Calculated_Amount__c, and Commission_Dispute__c.CreatedDate.

Given Commission_Dispute__c.Status__c = 'Resolved – Approved',
When the Coordinator saves the record,
Then a corrected Commission_Line__c record is automatically cloned from the original with the adjusted amount from Commission_Dispute__c.Approved_Amount__c and Status__c = 'Pending Approval'.

## Solution

Create Commission_Dispute__c custom object with fields: Status__c (picklist), Dispute_Reason__c (picklist), Dispute_Notes__c (long text), Resolution_Notes__c (long text), Approved_Amount__c (currency), Commission_Line__c (lookup), Submitted_By__c (lookup to User). Build dispute submission as an OmniScript ('OS_SubmitCommissionDispute') embedded in the Agent Portal via a FlexCard action. Record-Triggered Flow on Commission_Dispute__c handles queue assignment and email alerts (Email Alert component). A second Record-Triggered Flow on Commission_Dispute__c (after update, Status__c changed to 'Resolved – Approved') clones the Commission_Line__c using an Apex invocable method 'CommissionLineCloneService.cloneWithAdjustment()'. Community sharing: brokers see only their own Commission_Dispute__c records via Owner-based sharing.

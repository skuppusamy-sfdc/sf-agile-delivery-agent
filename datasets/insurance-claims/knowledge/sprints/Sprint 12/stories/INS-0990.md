## Description

As a Broker Relations Coordinator, I need a structured agent recruitment application OmniScript embedded in the Agent Portal so that prospective agents can complete their onboarding application, upload required licensing documents, and execute the broker agreement via DocuSign e-signature — all within a single guided experience. This extends the recruitment intake work completed in INS-0959. The completed application must create or update a Contact record with RecordType = 'Prospective_Agent' and an associated Agent_Application__c record.

## Acceptance Criteria

Given a prospective agent accesses the public-facing recruitment portal URL,
When they begin the OmniScript ('Agent_Recruitment_App_OS'),
Then they are prompted to enter: Full Name, NPI, TIN, State License Number(s), Resident State, and desired LOBs (multi-select picklist).

Given the applicant enters an NPI,
When they advance past Step 2,
Then an Apex callout validates the NPI against the NPPES API and populates Contact.NPI__c; if invalid, an inline error message reads 'NPI not found in NPPES registry — please verify and re-enter'.

Given all required fields are completed,
When the applicant reaches the document upload step,
Then they can attach up to 5 files (PDF or image, max 10MB each) which are stored as ContentDocument records linked to Agent_Application__c via ContentDocumentLink.

Given the application form is complete and documents are uploaded,
When the applicant clicks 'Proceed to Signature',
Then a DocuSign envelope is generated using template 'Broker_Agreement_v3' and sent to the applicant's provided email; Agent_Application__c.DocuSign_Status__c is set to 'Sent'.

Given DocuSign sends a completion webhook,
When Agent_Application__c.DocuSign_Status__c is updated to 'Completed',
Then Agent_Application__c.Status__c transitions to 'Pending Review' and a Task is auto-created and assigned to the Broker Relations Coordinator queue.

-Given the applicant completes CAQH attestation inline-
-When the CAQH section is reached, a CAQH ProView API call pre-fills credentialing data-
(Removed from scope — CAQH integration handled in CredentialingService sprint per INS-0978)

Given the Contact record is created,
When RecordType = 'Prospective_Agent',
Then the Permission Set Group 'Agent_Portal_Applicant' is automatically assigned via a Record-Triggered Flow.

## Solution

Extend the existing Agent_Recruitment_App_OS OmniScript (INS-0959 baseline) with additional steps for document upload and DocuSign integration. Use a DataRaptor Transform to upsert Contact and Agent_Application__c. NPPES validation implemented as an Apex REST callout class (NPPESValidationService) invoked via an OmniScript Integration Procedure. DocuSign envelope creation handled via DocuSign for Salesforce managed package (GenDocs action). A Platform Event (DocuSign_Webhook__e) received by an Apex trigger updates Agent_Application__c.DocuSign_Status__c and fires the downstream Record-Triggered Flow for Task creation and Permission Set Group assignment.

## Description

**Bug Report — ERA Generation Failure on Multi-COB Claims**

**Environment:** Acme Insurance FULL Sandbox
**Detected By:** Lisa Park (Claims Operations Manager) during UAT regression on Sprint 12 payment batch run.

**Steps to Reproduce:**
1. Create a Claim__c record with Claim__c.COB_Flag__c = TRUE and at least two Claim_COB_Adjustment__c child records (primary and secondary payer adjustments).
2. Adjudicate all Claim_Line__c records (Claim_Line__c.Status__c = 'Adjudicated').
3. Run the 'Generate ERA Batch' scheduled Apex job (ERAGenerationBatch) from the Developer Console.
4. Observe the batch job logs and the generated ERA flat file in the Files tab of the associated Payment__c record.

**Expected Result:**
The ERA file is generated successfully and includes all COB adjustment segments (CAS loop 2100) for each Claim_Line__c, with primary and secondary payer amounts correctly reflected.

**Actual Result:**
The batch job throws an unhandled NullPointerException in ERAGenerationBatch.execute() at line ~147 ('eraLine.cobAdjustments.get(0).adjustedAmount') when a Claim__c has more than one Claim_COB_Adjustment__c record. The ERA file is not generated, Payment__c.ERA_Status__c remains 'Pending', and the batch logs show: 'System.NullPointerException: Attempt to de-reference a null object.'

**Impact:** HIGH — All multi-COB claims in the payment batch are blocked. Approximately 340 claims in the current payment cycle are affected.

**Root Cause Hypothesis:** The ERAGenerationBatch Apex class appears to only query the first Claim_COB_Adjustment__c child record per claim (SOQL LIMIT 1) and does not handle the collection iteration when multiple COB adjustment records exist.

## Acceptance Criteria

As per Description. Fix verified when:
- ERAGenerationBatch processes a Claim__c with 2+ Claim_COB_Adjustment__c records without throwing NullPointerException.
- Generated ERA file contains correct CAS loop entries for each COB adjustment.
- Payment__c.ERA_Status__c transitions to 'Generated' on successful batch completion.
- Batch error log shows 0 failures for multi-COB claims in the regression test data set (min 50 claims).

## Solution

Fix ERAGenerationBatch.execute() to replace the single-record SOQL (SELECT ... FROM Claim_COB_Adjustment__c WHERE Claim__c = :claimId LIMIT 1) with a bulk Map<Id, List<Claim_COB_Adjustment__c>> pattern queried outside the loop. Update the ERA line builder to iterate over the full list of COB adjustments per claim and append a CAS segment for each. Add null-check guard before accessing cobAdjustments list. Add unit test coverage (currently at 61%) targeting multi-COB scenarios to reach >85% coverage threshold. Deploy fix to Full Sandbox for UAT re-run before production push.

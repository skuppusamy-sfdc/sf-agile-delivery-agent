## Description

As a Broker Relations Coordinator, I need a license and appointment status widget within the Agent Portal so that brokers can view their state appointment status, license expiration dates, and any outstanding compliance requirements without calling the back office.

This widget must surface data from the Agent_License__c and Agent_Appointment__c custom objects and flag licenses expiring within 60 days to prompt proactive renewal.

## Acceptance Criteria

Given an authenticated broker user navigates to the 'My Licenses & Appointments' tab in the Agent Portal,
When the widget loads,
Then a list of all Agent_License__c records linked to the broker's Contact record should display with: Agent_License__c.State__c, Agent_License__c.License_Number__c, Agent_License__c.License_Type__c, Agent_License__c.Expiration_Date__c, and Agent_License__c.Status__c.

Given an Agent_License__c.Expiration_Date__c is within 60 days of today,
When the widget renders that license row,
Then a yellow warning badge labeled 'Expiring Soon' should appear alongside the expiration date, and the row should sort to the top of the list.

Given an Agent_License__c.Expiration_Date__c has passed (status = 'Expired'),
When the widget renders,
Then the row should display a red 'Expired' badge and the broker should see a call-to-action button 'Initiate Renewal' that opens a pre-populated OmniScript for license renewal submission.

Given a broker has Agent_Appointment__c records linked to their Contact,
When the Appointments sub-tab is selected,
Then a table should display Agent_Appointment__c.State__c, Agent_Appointment__c.LOB__c, Agent_Appointment__c.Appointment_Status__c, and Agent_Appointment__c.Appointment_Date__c, filtered to show only active and pending appointments.

Given Agent_Appointment__c.Appointment_Status__c = 'Pending State Approval',
When the broker views the appointment row,
Then a tooltip should display 'Awaiting state confirmation. Processing typically takes 5–10 business days.' and the row should be non-editable.

-Given the widget should allow brokers to upload license renewal documents directly via the portal- (moved to Phase 3 per compliance team review)

## Solution

LWC component ('BrokerLicenseAppointmentWidget') with two child components: 'LicenseList' and 'AppointmentList'. LicenseList queries Agent_License__c via @AuraEnabled Apex controller with SOQL ORDER BY Expiration_Date__c ASC. Expiry warning logic computed client-side in JS using Date comparison against today. 'Initiate Renewal' button triggers navigation to an OmniScript ('BrokerLicenseRenewal') pre-seeded with the selected Agent_License__c fields via URL parameters. AppointmentList queries Agent_Appointment__c filtered by Contact__c = running user's ContactId and Appointment_Status__c IN ('Active','Pending State Approval'). Tooltip implemented as a lightning-helptext component. Custom Permission 'View_License_Appointments' gates widget visibility within the Agent Portal app. Sharing Rules on Agent_License__c and Agent_Appointment__c scoped to Contact__c ownership.

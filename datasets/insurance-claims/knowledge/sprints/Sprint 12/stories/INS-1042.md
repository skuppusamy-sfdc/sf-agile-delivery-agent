## Description

**Bug Report — Agentforce Claims AI Triage Misclassification**

**Environment:** Full Sandbox (Acme-FS1)
**Severity:** High — affects SIU referral accuracy and claims routing for multi-line property claims

**Steps to Reproduce:**
1. Create a new Claim__c record via the FNOL intake OmniScript with Claim__c.Claim_Type__c = 'Property' and Claim__c.LOB__c = 'Commercial'.
2. Add two or more Claim_Line__c records to the claim, each with a different Claim_Line__c.Damage_Category__c (e.g., 'Structural' and 'Contents').
3. Save the Claim__c record and allow the Agentforce Claims AI triage action to execute (triggered via Record-Triggered Flow on Claim__c insert).
4. Review Claim__c.AI_Triage_Recommendation__c and Claim__c.SIU_Referral_Flag__c.

**Expected Behavior:**
Agentforce evaluates all associated Claim_Line__c records in aggregate, applies the multi-line property triage prompt template, and returns a recommendation that accounts for total claimed amount across all lines. Claim__c.AI_Triage_Recommendation__c should reflect a composite risk score.

**Actual Behavior:**
Agentforce only evaluates the first Claim_Line__c record (ordered by CreatedDate ASC) and ignores subsequent lines. As a result, Claim__c.AI_Triage_Recommendation__c reflects only single-line analysis, the composite claimed amount is understated, and Claim__c.SIU_Referral_Flag__c = false on claims that should be flagged for SIU review based on total exposure.

**Root Cause (Preliminary):**
The Agentforce prompt template (Claims_Triage_Prompt_v3) passes a SOQL query result limited to LIMIT 1 on Claim_Line__c. The prompt action input variable 'claimLineItems' is incorrectly scoped to a single record variable instead of a collection variable in the invoking Flow.

**Impact:** Potential under-referral to SIU (Special Investigations Unit), compliance risk, and financial exposure on large commercial property claims.

## Acceptance Criteria

As per Description.

Additional regression criteria:
Given a Claim__c with three Claim_Line__c records of varying Damage_Category__c values,
When the Agentforce triage action executes,
Then Claim__c.AI_Triage_Recommendation__c reflects analysis of all three lines and Claim__c.AI_Total_Evaluated_Amount__c equals the sum of all Claim_Line__c.Claimed_Amount__c values.

Given the aggregate claimed amount across all Claim_Line__c records exceeds the SIU threshold defined in Claims_AI_Config__mdt.SIU_Referral_Threshold__c,
When triage completes,
Then Claim__c.SIU_Referral_Flag__c = true and a Case is created with RecordType = 'SIU_Investigation'.

## Solution

Fix the Record-Triggered Flow (RTF_Claims_AI_Triage) by changing the 'claimLineItems' input variable from a single Claim_Line__c record variable to a Salesforce Collection Variable. Update the Get Records element to remove the LIMIT 1 constraint and retrieve all Claim_Line__c records where Claim_Line__c.Claim__c = {!Claim__c.Id}. Update the Agentforce Prompt Template (Claims_Triage_Prompt_v3) input binding to accept the collection and iterate over line items in the prompt using the template's merge field loop syntax. Update Claims_AI_Config__mdt to expose SIU_Referral_Threshold__c as a configurable value. Add Flow unit tests via the Flow Test framework covering single-line, multi-line, and threshold-breach scenarios. Coordinate with Agentforce platform team to validate prompt token limits with large claim line collections (>10 lines).

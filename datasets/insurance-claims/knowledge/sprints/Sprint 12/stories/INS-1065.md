## Description

As a Claims Operations Manager, I need an Agentforce AI-assisted triage recommendation to appear on the Claim__c record immediately after FNOL intake is completed so that claims adjusters can quickly identify high-complexity claims, potential SIU referral candidates, and COB coordination requirements without relying solely on manual review.

The Agentforce panel should display a confidence-scored recommendation (e.g., 'Standard Processing', 'Complex Review', 'SIU Referral', 'COB Required') with the key signals that drove the recommendation, sourced from Claim__c, Policy__c, and Claim_Line__c data.

## Acceptance Criteria

Given a Claim__c record transitions to Claim__c.Status__c = 'FNOL Complete',
When the Record-Triggered Flow fires,
Then an Agentforce Agent Action is invoked passing Claim__c.Id, Claim__c.Loss_Type__c, Claim__c.Total_Billed_Amount__c, Policy__c.LOB__c, Policy__c.Policyholder_Risk_Score__c, and a count of prior Claim__c records for the same Policy__c within 24 months.

Given the Agentforce model processes the inputs,
When the recommendation is returned,
Then Claim__c.AI_Triage_Recommendation__c is populated with one of ['Standard Processing', 'Complex Review', 'SIU Referral', 'COB Required'], Claim__c.AI_Confidence_Score__c is populated as a percentage, and Claim__c.AI_Triage_Signals__c (Long Text Area) is populated with a human-readable explanation of the top 3 contributing factors.

Given Claim__c.AI_Triage_Recommendation__c = 'SIU Referral' AND Claim__c.AI_Confidence_Score__c >= 75,
When the recommendation is written to the record,
Then a Case (Type = 'SIU Referral', RecordType = 'SIU_Investigation') is auto-created and linked to the Claim__c, assigned to the 'SIU_Queue', and Claim__c.Status__c transitions to 'Pending SIU Review'.

Given Claim__c.AI_Triage_Recommendation__c = 'COB Required',
When the recommendation is written,
Then Claim__c.COB_Flag__c = TRUE and a Task is created for the assigned adjuster with Subject = 'Initiate COB Verification — [Claim Number]' and Due Date = TODAY() + 2.

Given an adjuster disagrees with the AI recommendation,
When the adjuster selects 'Override Recommendation' on the Claim record page,
Then Claim__c.AI_Triage_Override__c = TRUE, Claim__c.Override_Reason__c (Required picklist) must be populated, and the override event is logged to a Claim_AI_Audit__c record for compliance reporting including the original recommendation, confidence score, and override reason.

Given Claim__c.AI_Confidence_Score__c < 50,
When the recommendation is displayed,
Then the Agentforce panel renders a yellow 'Low Confidence' badge and the recommendation is advisory only — no automated downstream actions (SIU Case, COB flag) are triggered.

-Given the Agentforce panel displays a 'Chat with AI' button for free-form claim questions-
-Descoped from Sprint 12; targeted for Sprint 14 Agentforce Phase 2 rollout-

## Solution

Configure an Agentforce Agent with a custom Agent Action (ClaimTriageAction) backed by an Apex invocable method (ClaimTriageAgentAction.cls) that assembles the claim context payload and calls the Einstein generative AI model via the Agentforce Agent API. Use a Record-Triggered Flow on Claim__c (After Update, Status__c Changed to 'FNOL Complete') to invoke the Agent Action asynchronously via a Platform Event (Claim_Triage_Requested__e) consumed by a trigger-based Queueable. Write AI outputs back to Claim__c fields via a second Queueable (ClaimTriageResponseWriter.cls). Claim_AI_Audit__c custom object captures all recommendations and overrides for Compliance Officer review. Deploy a custom LWC (claimTriagePanel) on the Claim__c record page to render the Agentforce recommendation with confidence score badge and override modal. Permission Set 'Claims_Adjuster_AI' controls access to the override functionality.

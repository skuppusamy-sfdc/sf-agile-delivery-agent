## Description

**Bug Report — Stale Commission Tier Badge on Agent Portal Dashboard**

**Environment:** Acme Insurance Partial Sandbox
**Detected By:** Ryan Mitchell (Broker Relations Coordinator) during Sprint 12 commission dashboard QA (related to INS-1027).

**Steps to Reproduce:**
1. Log in as an agent user whose Commission_Tier__c.Tier_Label__c was 'Gold' at the start of the current session.
2. Simulate a mid-year tier downgrade by updating the agent's rolling premium total to fall below the Commission_Tier__c.Min_Premium__c threshold for 'Gold' (e.g., via a policy lapse processed by the INS-1029 chargeback flow).
3. Without refreshing the browser, observe the tier badge displayed on the Agent Portal Commission Dashboard FlexCard.
4. Manually refresh the browser page and observe again.

**Expected Result:**
After the tier downgrade event, the FlexCard tier badge should reflect the new tier ('Silver' or applicable lower tier) either in real-time (via Platform Event) or within the next scheduled cache refresh cycle (max 60 seconds per INS-1027 AC).

**Actual Result:**
The tier badge continues to display 'Gold' even after a manual browser refresh. The badge only updates correctly after the agent logs out and logs back in. The YTD dollar amount updates correctly (the Platform Event cache invalidation from INS-1027 is working), but the tier label is sourced from a separate cached DataRaptor Transform that is NOT invalidated by the 'CommissionPosted__e' Platform Event handler.

**Root Cause Hypothesis:** The 'CommissionPosted__e' subscriber flow only invalidates the YTD aggregate DataRaptor cache key ('DR_AgentYTD_{agentId}') but does not invalidate the tier evaluation DataRaptor Transform cache key ('DR_AgentTier_{agentId}'). The tier badge FlexCard state reads from the stale transform output.

## Acceptance Criteria

As per Description. Fix verified when:
- After a tier downgrade event, the correct tier badge renders on the Commission Dashboard FlexCard within 60 seconds without requiring logout/login.
- Both 'DR_AgentYTD_{agentId}' and 'DR_AgentTier_{agentId}' DataRaptor cache keys are invalidated by the 'CommissionPosted__e' Platform Event subscriber flow.
- Regression test confirms Gold → Silver and Silver → Bronze downgrade scenarios both display correctly.
- No regression on tier upgrade scenarios (Silver → Gold).

## Solution

Update the 'CommissionPosted__e' subscriber Record-Triggered Flow to invoke the DataRaptor cache invalidation for both cache keys: 'DR_AgentYTD_{agentId}' and 'DR_AgentTier_{agentId}'. This requires adding a second OmniStudio Integration Procedure call step in the existing flow action that was built for INS-1027. Verify the FlexCard 'AgentCommissionDashboard' tier badge state is bound to the DataRaptor Transform output (not a static field) so it re-renders on cache refresh. Add a test scenario in the existing FlexCard preview configuration covering the downgrade path.

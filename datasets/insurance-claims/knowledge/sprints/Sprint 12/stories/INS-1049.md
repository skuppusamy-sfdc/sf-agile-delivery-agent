## Description

As a Claims Operations Manager, I need an Agentforce AI-assisted triage panel embedded in the FNOL intake OmniScript so that claims intake agents receive a real-time recommendation on claim complexity (Simple, Complex, SIU Referral) based on ICD-10 codes, claimant history, policy type, and loss description sentiment analysis. This will reduce average FNOL handling time and improve SIU referral accuracy. The AI recommendation is advisory only — final triage decision remains with the human agent.

## Acceptance Criteria

Given a claims intake agent is completing the FNOL OmniScript and has entered Claim__c.Loss_Description__c, Claim__c.ICD10_Primary_Code__c, and Policy__c.LOB__c,
When the agent reaches the 'Triage Assessment' step,
Then the Agentforce panel displays an AI-generated triage recommendation: 'Simple', 'Complex', or 'SIU Referral', with a confidence score (0–100) and a plain-language rationale (max 200 characters).

Given the Agentforce recommendation is 'SIU Referral' with confidence >= 75,
When the panel renders,
Then a prominent amber banner states 'AI recommends SIU review — please confirm or override' and the Claim__c.SIU_Referral_Flag__c field is pre-populated to TRUE (editable by agent).

Given the agent accepts the AI recommendation,
When they click 'Accept Recommendation',
Then Claim__c.Triage_Category__c is set to the recommended value, Claim__c.AI_Triage_Confidence__c is populated with the confidence score, and Claim__c.Triage_Source__c = 'Agentforce'.

Given the agent overrides the AI recommendation,
When they select a different Triage_Category__c value and click 'Override',
Then Claim__c.Triage_Source__c = 'Agent Override', Claim__c.AI_Triage_Override_Reason__c (required text field) must be populated before the OmniScript can advance, and the override event is logged to a custom Agentforce_Audit_Log__c record for compliance review.

Given the Agentforce service is unavailable (timeout > 5 seconds),
When the triage step loads,
Then a graceful fallback message is displayed: 'AI triage unavailable — please complete manual triage' and the OmniScript allows progression with Claim__c.Triage_Source__c = 'Manual'.

Given any triage outcome,
When the FNOL OmniScript is submitted,
Then Agentforce_Audit_Log__c is created with fields: Claim__c lookup, AI_Recommendation__c, AI_Confidence_Score__c, Agent_Decision__c, Override_Reason__c, and Timestamp__c for all claims regardless of acceptance or override.

## Solution

Agentforce Agent Action ('ClaimTriageAction') configured with a custom Prompt Template referencing Claim__c.Loss_Description__c (sentiment input), ICD10_Primary_Code__c, Policy__c.LOB__c, and a DataRaptor-fed claimant history summary (prior claims count, prior SIU flags from Claim__c). Prompt Template uses Einstein generative AI with a structured JSON output schema enforcing 'recommendation', 'confidence', and 'rationale' fields. Agentforce action invoked from OmniScript via an Integration Procedure step with a 5-second timeout and fault path. OmniScript Decision element routes to SIU banner based on recommendation + confidence threshold. Accept/Override buttons call DataRaptor Loads to update Claim__c fields. Override path triggers a required input step for AI_Triage_Override_Reason__c. Agentforce_Audit_Log__c created via a final DataRaptor Load step in both Accept and Override paths. Permission Set 'Agentforce_Claims_User' grants access to the Agent Action and audit log object.

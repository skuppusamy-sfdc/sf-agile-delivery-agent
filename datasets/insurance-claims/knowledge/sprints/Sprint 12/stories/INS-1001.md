## Description

**Bug Report — Commission YTD Rollup Overcount on Renewed Policies**

**Steps to Reproduce:**
1. Navigate to a Policy__c record that has undergone 2 or more renewal cycles (Policy__c.Renewal_Count__c >= 2).
2. Verify that Commission_Transaction__c records exist for both the original term and each renewal, all with Status__c = 'Paid'.
3. Open the Agent Portal Commission Dashboard (INS-0997) for the agent linked to this policy.
4. Observe the YTD_Earned_Commission__c rollup figure displayed on the FlexCard.

**Expected Result:**
The YTD_Earned_Commission__c rollup should sum only Commission_Transaction__c records where Transaction_Date__c falls within the current calendar year (Jan 1 – Dec 31) and Status__c = 'Paid', regardless of how many policy renewal cycles exist.

**Actual Result:**
The rollup sums ALL Commission_Transaction__c records linked to the Policy__c across ALL years when Policy__c.Renewal_Count__c >= 2, inflating the YTD figure. For a policy with 3 renewal cycles, the YTD total is approximately 3x the correct value.

**Root Cause (Preliminary):**
The DataRaptor Extract 'DR_Commission_YTD_Extract' used in the Commission Dashboard FlexCard (INS-0997) is missing the Transaction_Date__c >= :startOfYear AND Transaction_Date__c <= :endOfYear filter conditions. The SOQL behind the DataRaptor is joining Commission_Transaction__c to Policy__c without the date range predicate, causing all historical transactions to be included.

**Environment:** Acme Insurance - Staging Org
**Regression:** Introduced in Sprint 11 deployment (INS-0981 DataRaptor refactor)
**Severity:** High — agents are seeing inflated commission figures and raising disputes unnecessarily.

## Acceptance Criteria

As per Description. Verify fix by confirming YTD_Earned_Commission__c on the dashboard matches the sum of Commission_Transaction__c.Commission_Amount__c where Status__c = 'Paid' AND YEAR(Transaction_Date__c) = YEAR(TODAY()) for all test policies with Renewal_Count__c >= 2.

## Solution

Update DataRaptor Extract 'DR_Commission_YTD_Extract': add filter conditions Transaction_Date__c >= {startOfYear} and Transaction_Date__c <= {endOfYear} using OmniStudio formula variables bound to the current calendar year. Inject startOfYear and endOfYear as input parameters from the FlexCard context using a Set Values action before the DataRaptor fires. Add a unit test in the QA sandbox using a Policy__c with Renewal_Count__c = 3 and Commission_Transaction__c records spanning 3 years to assert the rollup equals only the current-year sum. Update the DataRaptor version and re-activate. Regression test the full Commission Dashboard FlexCard (INS-0997) post-fix.

## Description

**Bug Report — Agent Portal Commission Dashboard Duplicate Records**

**Environment:** Acme Insurance UAT Sandbox (Sprint 12 deploy, 2024-11-04)

**Steps to Reproduce:**
1. Log in to the Agent Portal as a test agent user (e.g., agent.testuser01@acme-uat.com) who is assigned to a split-commission policy.
2. Navigate to the Commission Dashboard (INS-1058).
3. Observe the 'Earned MTD' KPI tile and drill-down list.
4. Note policies where Policy__c.Commission_Split__c = TRUE and two Agent_Commission__c junction records exist.

**Expected Behavior:**
Each Payment__c record should appear exactly once in the drill-down list. The 'Earned MTD' total should reflect only the current agent's share (Payment__c.Agent_Split_Percentage__c applied).

**Actual Behavior:**
Payment__c records for split-commission policies appear twice in the drill-down list, causing the 'Earned MTD' KPI to be inflated by the duplicate amount. Example: Policy ACM-00341 shows $1,200 twice, inflating the MTD total by $1,200.

**Root Cause (Suspected):**
The DataRaptor Extract action (CommissionDrilldown_DRX) joins Payment__c to Agent_Commission__c without a DISTINCT filter, returning one row per Agent_Commission__c junction record. When two agents share a policy, both junction records are returned for the querying agent.

**Affected Users:** Any agent assigned to a split-commission policy (Policy__c.Commission_Split__c = TRUE).

**Severity:** High — financial figures displayed to agents are incorrect.

## Acceptance Criteria

As per Description. Verify fix by confirming Payment__c records appear exactly once in the drill-down list for split-commission policies, and 'Earned MTD' KPI reflects only the current agent's proportional share calculated as Payment__c.Amount__c * Agent_Commission__c.Split_Percentage__c where Agent_Commission__c.Agent__c = current user's Agent_Profile__c.

## Solution

Update the CommissionDrilldown_DRX DataRaptor Extract to filter Agent_Commission__c by Agent__c = :currentUserId before joining to Payment__c, preventing cross-agent junction rows from being returned. Add a SOQL WHERE clause: Agent_Commission__c.Agent__c = '{UserContext.AgentProfileId}'. Additionally, update the KPI aggregate DataRaptor to SUM(Payment__c.Amount__c * Agent_Commission__c.Split_Percentage__c) rather than SUM(Payment__c.Amount__c) to correctly reflect split amounts. Regression test against 10 split-commission policies in UAT before promoting to production.

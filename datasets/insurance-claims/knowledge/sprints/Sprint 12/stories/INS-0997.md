## Description

As a Broker Relations Coordinator, I need a real-time commission dashboard on the Agent Portal so that agents can view their year-to-date earnings, pending commissions, and payment history broken down by LOB without contacting the back office. The dashboard should aggregate data from all active Policy__c records tied to the agent's Producer_Code__c and surface commission milestones and clawback alerts inline.

## Acceptance Criteria

Given an authenticated agent user with the 'Agent Portal User' Permission Set Group assigned,
When they navigate to the Commission Dashboard tab in the Agent Portal,
Then a FlexCard renders displaying YTD_Earned_Commission__c, Pending_Commission__c, and Clawback_Amount__c aggregated across all Policy__c records where Agent_Producer_Code__c matches the logged-in user's Producer_Code__c.

Given the dashboard is loaded,
When the agent selects a LOB filter (Auto, Home, Life, Health),
Then the YTD totals recalculate client-side and the underlying DataRaptor Extract re-queries Commission_Transaction__c records filtered by LOB__c = selected value.

Given a Commission_Transaction__c record has Status__c = 'Pending',
When it appears in the dashboard,
Then it is displayed in an amber badge with the Expected_Payment_Date__c shown as 'Est. Payment: MM/DD/YYYY'.

Given a clawback event exists where Clawback_Amount__c > 0 on a Commission_Transaction__c,
When the agent views the dashboard,
Then a red inline alert reads 'Clawback of $[amount] applied on [Clawback_Date__c] — Policy [Policy__c.Policy_Number__c]'.

Given the agent clicks 'View Payment History',
When the modal opens,
Then a paginated list of Payment__c records (Type__c = 'Commission') is displayed sorted by Payment_Date__c descending, showing Payment_Amount__c, Payment__c.Status__c, and linked Policy__c.Policy_Number__c.

-Given the agent can export commission data to PDF via Conga CLM integration- (deprecated — export feature moved to INS-1012)

Given the dashboard data is older than 5 minutes,
When the agent is on the page,
Then a Platform Event (Commission_Refresh__e) is fired and the FlexCard auto-refreshes without a full page reload.

## Solution

Build an OmniStudio FlexCard ('Agent_Commission_Dashboard_FC') consuming two DataRaptor Extracts: one for aggregated Commission_Transaction__c totals grouped by LOB__c, and one for Payment__c history filtered by Type__c = 'Commission'. Use a DataRaptor Transform to compute YTD totals and clawback deltas. A Record-Triggered Flow on Commission_Transaction__c (after insert/update) publishes a Commission_Refresh__e Platform Event. The FlexCard subscribes to this event via the built-in Platform Event action to trigger a live refresh. LOB filter state is managed via FlexCard State Actions. Deploy behind 'Agent_Portal_Commission' Permission Set Group. LWC wrapper component handles responsive layout within the Experience Cloud Agent Portal site.

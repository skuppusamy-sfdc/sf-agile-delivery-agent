## Description

As a Broker Relations Coordinator, I need an automated appointment verification workflow that queries the National Insurance Producer Registry (NIPR) during broker onboarding so that we can validate a prospective agent's license status, lines of authority, and state appointments in real time before activating their Broker__c record in Salesforce.

Currently, coordinators manually look up NIPR and enter data, leading to errors and compliance risk. The integration should also schedule periodic re-verification for active brokers to detect license lapses or revocations.

## Acceptance Criteria

Given a new Broker__c record is created with Broker__c.Onboarding_Status__c = 'License Verification Pending',
When the record is saved,
Then an outbound callout is made to the NIPR API using the broker's NPN (Broker__c.NPN__c) as the query key.

Given the NIPR API returns a successful response,
When the license data is parsed,
Then Broker__c.License_Status__c, Broker__c.Lines_of_Authority__c (multi-select), Broker__c.License_Expiration_Date__c, and Broker__c.Appointment_States__c are populated from the API response payload.

Given the NIPR response indicates an active license with valid state appointments,
When the verification completes,
Then Broker__c.Onboarding_Status__c transitions to 'Appointment Verified' and the record is routed to the Broker Relations Coordinator queue for final review.

Given the NIPR response indicates a suspended or revoked license,
When the verification completes,
Then Broker__c.Onboarding_Status__c = 'Verification Failed', a Compliance Case is created (RecordType = 'Broker_License_Issue'), and the Compliance Officer is notified via in-app notification.

Given an active Broker__c record exists,
When a scheduled job runs on the first of each month,
Then all brokers with Broker__c.License_Expiration_Date__c within 60 days are re-verified against NIPR and discrepancies are flagged in a Broker_Verification_Log__c record.

-Given NIPR verification results are stored as a PDF attachment on Broker__c- (deferred to Phase 3 document management initiative)

Given the NIPR API is unavailable (timeout or HTTP 5xx),
When the callout fails,
Then the error is logged to Integration_Error_Log__c, Broker__c.Onboarding_Status__c = 'Verification Retry Pending', and a retry is attempted via a Schedulable Apex job within 2 hours.

## Solution

Implement an Apex Callout class (NIPRIntegrationService.cls) using Named Credentials for NIPR endpoint authentication. Invoke the callout from a Record-Triggered Flow (RTF_Broker_NIPR_Verification) via Apex Action on Broker__c insert when Onboarding_Status__c = 'License Verification Pending'. Use a Queueable Apex pattern to handle async callout execution and avoid governor limit conflicts. Scheduled re-verification implemented via Schedulable Apex (NIPRMonthlyReverificationBatch.cls) using Batch Apex for large broker volumes. Integration_Error_Log__c records written in the catch block. Store API response JSON in Broker__c.Last_NIPR_Response_JSON__c (Long Text Area) for audit. Apply Field-Level Security on NPN__c and License_Status__c to restrict to PSG_Broker_Admin.

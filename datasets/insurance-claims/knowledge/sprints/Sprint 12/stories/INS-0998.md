## Description

As an agent using the Agent Portal, I need an automated commission dispute submission workflow so that I can flag discrepancies in my Commission_Transaction__c records without emailing the Broker Relations team, ensuring disputes are tracked, routed, and resolved within SLA. This story extends INS-0997 by adding an action from the Commission Dashboard directly into the dispute OmniScript.

## Acceptance Criteria

Given an agent views a Commission_Transaction__c record on the dashboard with Status__c != 'Disputed',
When they click 'Dispute This Commission',
Then an OmniScript ('Commission_Dispute_OS') launches in a modal overlay pre-populating Commission_Transaction__c.Id, Policy__c.Policy_Number__c, and Claimed_Commission_Amount__c.

Given the OmniScript is open,
When the agent selects a Dispute_Reason__c picklist value (options: 'Incorrect Rate', 'Missing Policy', 'Clawback Error', 'Duplicate Payment', 'Other') and enters a free-text description (minimum 20 characters),
Then the 'Submit' button becomes enabled.

Given the agent submits the dispute,
When the OmniScript DataRaptor Post executes,
Then a new Commission_Dispute__c record is created with Status__c = 'Open', Submitted_Date__c = TODAY(), Assigned_To__c auto-populated via Assignment Rules targeting the 'Broker Relations Queue', and a confirmation email is sent to the agent's User.Email via a Flow-invoked EmailAlert.

Given a Commission_Dispute__c record is created,
When Status__c transitions to 'Resolved',
Then the linked Commission_Transaction__c.Status__c is updated to 'Adjusted' or 'Confirmed' per the Resolution_Outcome__c value, via a Record-Triggered Flow.

Given the dispute SLA is 5 business days,
When a Commission_Dispute__c record has been Open for more than 5 business days,
Then an Escalation_Flag__c = true is set and a Task is created assigned to the Broker Relations Coordinator role.

## Solution

Create OmniScript 'Commission_Dispute_OS' with DataRaptor Pre-fetch to load Commission_Transaction__c context data. Use a DataRaptor Post action to create the Commission_Dispute__c record. Configure an Assignment Rule on Commission_Dispute__c to route to 'Broker Relations Queue' based on LOB__c. Build a Record-Triggered Flow (after update, Status__c changed to 'Resolved') to update the parent Commission_Transaction__c. For SLA enforcement, implement a Scheduled Flow running nightly querying Commission_Dispute__c records where Status__c = 'Open' AND CreatedDate <= N-5 business days, setting Escalation_Flag__c = true and creating Tasks. Expose the OmniScript as a Quick Action on the Agent Portal Commission Dashboard FlexCard (INS-0997).

## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard that surfaces the provider's active Provider_Network_Agreement__c (PNA) details, linked Fee_Schedule__c, and credentialing status so that I can answer provider inquiries in a single interaction without navigating across five separate record pages.

This story applies to both individual providers (Provider__c as Person Account) and facility providers (Facility__c as Business Account).

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c (Person Account) record,
When the Provider 360 FlexCard loads,
Then the card should display the active PNA details: Provider_Network_Agreement__c.Agreement_Name__c, Provider_Network_Agreement__c.Effective_Date__c, Provider_Network_Agreement__c.Expiration_Date__c, Provider_Network_Agreement__c.Network_Tier__c, and Provider_Network_Agreement__c.Status__c = 'Active'.

Given the active PNA has a linked Fee_Schedule__c,
When the FlexCard renders the PNA section,
Then a nested child card should display Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.Base_Rate_Multiplier__c.

Given a Provider__c has a Credentialing_Case__c with Credentialing_Case__c.Status__c IN ('In Review', 'Pending Documents', 'Approved'),
When the FlexCard loads,
Then a credentialing status badge should display with color-coded indicators: green for 'Approved', yellow for 'In Review' or 'Pending Documents', red for 'Expired' or 'Denied'.

Given a Provider__c has no active PNA (all PNAs have Status__c = 'Terminated' or 'Expired'),
When the FlexCard loads,
Then a warning banner should display: 'No Active Network Agreement – Provider may be Out of Network' with a button to initiate a new PNA via the NetworkContractingOmniScript.

-Given the FlexCard should also display the provider's open Claim__c count as a KPI tile- (descoped to INS-1061 per sprint planning)

Given the FlexCard is accessed by a user without the 'Provider_Network_Specialist' or 'Provider_Relations' Permission Set,
When they navigate to a Provider__c record,
Then the PNA and Fee Schedule sections should be hidden and replaced with 'Contact your Provider Network team for contract details.'

## Solution

OmniStudio FlexCard ('Provider360Card') built with two DataRaptor Turbo Extract actions: one querying Provider_Network_Agreement__c filtered by Provider__c.Id and Status__c = 'Active', a second querying Fee_Schedule__c via PNA's Fee_Schedule__c lookup. Child FlexCard for Fee Schedule details nested within the PNA state. Credentialing badge implemented as a conditional state on Credentialing_Case__c.Status__c using FlexCard state conditions. Custom Permission 'View_PNA_Details' checked via a Conditional Visibility rule on the PNA/Fee Schedule flyout states. FlexCard deployed to the Provider__c Lightning Record Page via App Builder with an audience filter restricting to internal users with relevant Permission Sets.

## Description

As a Broker Relations Coordinator, I need the system to automatically calculate and create renewal commission records when a Policy__c is renewed so that commission payments are initiated without manual data entry, ensuring accuracy and timeliness of broker compensation. The calculation must respect the agent's current commission schedule defined in Commission_Schedule__c and apply any applicable override tiers based on book-of-business volume thresholds.

## Acceptance Criteria

Given a Policy__c record transitions to Status__c = 'Renewed' and Policy__c.Renewal_Effective_Date__c is populated,
When the Record-Triggered Flow fires on Policy__c (after update),
Then a new Commission_Line__c record is created with Commission_Line__c.Commission_Type__c = 'Renewal', Commission_Line__c.Policy__c = the renewed policy, and Commission_Line__c.Status__c = 'Pending Approval'.

Given the agent has an active Commission_Schedule__c where Commission_Schedule__c.LOB__c matches Policy__c.LOB__c and Commission_Schedule__c.Effective_Date__c <= Policy__c.Renewal_Effective_Date__c,
When the commission record is created,
Then Commission_Line__c.Rate__c is populated from Commission_Schedule__c.Renewal_Rate__c and Commission_Line__c.Gross_Premium__c = Policy__c.Annual_Premium__c.

Given Commission_Line__c.Gross_Premium__c and Commission_Line__c.Rate__c are both populated,
When the record is saved,
Then Commission_Line__c.Calculated_Amount__c = Commission_Line__c.Gross_Premium__c * Commission_Line__c.Rate__c (formula field, currency, 2 decimal places).

Given the agent's YTD new business premium exceeds the Override_Tier__c.Volume_Threshold__c defined in their assigned Override_Tier__c record,
When the commission is calculated,
Then Commission_Line__c.Override_Amount__c is populated as (Override_Tier__c.Override_Rate__c - Commission_Schedule__c.Renewal_Rate__c) * Commission_Line__c.Gross_Premium__c.

-Given the commission requires a two-level approval for amounts exceeding $10,000-
-When Commission_Line__c.Calculated_Amount__c > 10000-
-Then an Approval Process is submitted automatically-
(Replaced by INS-0987 — threshold raised to $25,000 per Compliance Officer review)

Given the Policy__c has no active Commission_Schedule__c matching the LOB and effective date,
When the flow attempts to create the Commission_Line__c,
Then the record is created with Commission_Line__c.Status__c = 'Error – No Schedule Found' and a Task is created for the Broker Relations Coordinator to resolve.

## Solution

Record-Triggered Flow on Policy__c (after update, entry condition: Status__c changed to 'Renewed'). Flow uses a Get Records element to find the matching Commission_Schedule__c (LOB and date range). An Apex invocable method 'CommissionCalculationService.calculateRenewal()' handles override tier lookup via SOQL on Override_Tier__c, applying volume-based tiering. Commission_Line__c is created via Flow Create Records element. Error path creates a Task assigned to a queue ('Broker_Commission_Review'). Commission_Line__c.Calculated_Amount__c implemented as a formula field. Unit tests cover 90%+ code coverage on the Apex invocable class.

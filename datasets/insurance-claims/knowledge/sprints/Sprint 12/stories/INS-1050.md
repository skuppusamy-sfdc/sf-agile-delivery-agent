## Description

As a Broker Relations Coordinator, I need a real-time commission dashboard within the Agent Portal so that brokers and agents can view their earned, pending, and paid commissions by LOB without requiring manual report requests from the back office.

This dashboard should surface commission data tied to active Policy__c records and associated Payment__c transactions, enabling agents to self-serve their earnings visibility. Extends INS-1020 (Agent Portal Base Layout).

## Acceptance Criteria

Given an authenticated agent user with the 'Agent_Portal_User' Permission Set Group applied,
When they navigate to the Commission Dashboard tab in the Agent Portal,
Then they should see a FlexCard displaying total Earned Commission, Pending Commission, and YTD Paid Commission aggregated from Commission_Transaction__c.Amount__c grouped by Commission_Transaction__c.Status__c.

Given a broker has policies across multiple LOBs (e.g., Auto, Health, Property),
When the dashboard loads,
Then a LOB filter picklist should allow filtering Commission_Transaction__c records by Policy__c.Line_of_Business__c, refreshing all dashboard KPI tiles accordingly.

Given a Payment__c record with Payment__c.Type__c = 'Commission' transitions to Payment__c.Status__c = 'Disbursed',
When the Platform Event CommissionDisbursed__e is published,
Then the agent's dashboard YTD Paid tile should reflect the updated total within 30 seconds without a full page reload.

Given an agent clicks on the 'Pending Commission' KPI tile,
When the drill-down panel opens,
Then a DataRaptor-powered list should display each pending Commission_Transaction__c with columns: Policy__c.Name, Policy__c.Effective_Date__c, Commission_Transaction__c.Amount__c, Commission_Transaction__c.Expected_Pay_Date__c, and LOB.

-Given the dashboard includes a 'Request Adjustment' button on each commission line item so agents can self-submit disputes- (requirement moved to INS-1063)

Given an agent has zero commission records,
When the dashboard loads,
Then an empty-state illustration and message 'No commission activity found for the selected period' should display with a link to contact their Broker Relations Coordinator.

## Solution

Implement an OmniStudio FlexCard ('AgentCommissionDashboard') with four KPI child cards consuming DataRaptor Turbo Extract actions querying Commission_Transaction__c with SOQL aggregates. A Platform Event (CommissionDisbursed__e) published from a Record-Triggered Flow on Payment__c (entry criteria: Status__c changes to 'Disbursed' AND Type__c = 'Commission') will trigger an Apex-invocable method to push a refresh signal to the LWC event bus. LOB filtering implemented via OmniScript integration procedure passing filter context into the DataRaptor. Permission Set Group 'Agent_Portal_User' gates access via a Custom Permission 'View_Commission_Dashboard'. Sharing Rules on Commission_Transaction__c scoped to Agent__c lookup matching the running user's associated Contact record.

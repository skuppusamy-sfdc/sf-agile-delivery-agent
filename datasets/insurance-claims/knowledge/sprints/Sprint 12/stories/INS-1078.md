## Description

As a Claims Operations Manager, I need a Provider 360 FlexCard on the Provider__c (Person Account) record that surfaces the active Provider_Network_Agreement__c (PNA) details and linked Fee_Schedule__c rates so that claims adjusters can quickly verify network participation status and applicable reimbursement rates during claim adjudication without switching to the contracting system. This supports accurate Claim_Line__c pricing and reduces adjudication errors.

## Acceptance Criteria

Given a claims adjuster opens a Provider__c record,
When the Provider 360 FlexCard (Provider360Card) loads,
Then it displays the active PNA record where Provider_Network_Agreement__c.Status__c = 'Active' and Provider_Network_Agreement__c.Provider__c = current Provider__c.Id, showing: PNA__c.Contract_Start_Date__c, PNA__c.Contract_End_Date__c, PNA__c.Network_Tier__c, and PNA__c.TPA__c.

Given the active PNA has a linked Fee_Schedule__c,
When the FlexCard child section expands,
Then it lists Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Allowed_Amount__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.LOB__c for up to 20 most recently used CPT codes (ordered by Fee_Schedule__c.Last_Used_Date__c DESC).

Given the Provider__c has no active PNA (PNA Status__c != 'Active' or no PNA exists),
When the FlexCard loads,
Then a warning banner displays: 'No active network agreement found. Claims may be subject to out-of-network pricing.' and the Fee Schedule section is hidden.

Given a Fee_Schedule__c.Allowed_Amount__c is $0.00 or null,
When the rate row renders,
Then the row is flagged with a red indicator and tooltip: 'Rate not configured — contact Provider Network Specialist.'

-Given the FlexCard displays credentialing status from Credentialing_Case__c- (deprecated: credentialing status moved to separate Credentialing 360 tab per INS-1065 design revision)

Given the adjuster clicks 'View Full Contract',
When the action fires,
Then the user is navigated to the Provider_Network_Agreement__c record detail page in a new subtab within the console app.

## Solution

Build OmniStudio FlexCard (Provider360Card) deployed on the Provider__c Lightning Record Page via App Builder. Primary DataRaptor Extract (DR_Provider360_PNA_Extract) queries Provider_Network_Agreement__c filtered by Provider__c Id and Status__c = 'Active'. A child DataRaptor fetches top-20 Fee_Schedule__c records ordered by Last_Used_Date__c. Conditional FlexCard states handle the no-PNA scenario using a state condition on the PNA record count. The 'View Full Contract' navigation action uses a FlexCard Navigation Action (Navigate to Record). FlexCard is visible only to users with the 'Claims_Adjuster' or 'Claims_Manager' Permission Set.

## Description

As a licensed agent using the Agent Portal, I need a unified policy search interface so that I can quickly locate in-force Policy__c records by policyholder name, policy number, or TIN and view a summary card with coverage details, open claims, and renewal date — enabling faster client servicing without navigating multiple Salesforce tabs. Extends INS-1044 (Agent Portal Base Navigation Shell).

## Acceptance Criteria

Given an agent enters at least 3 characters in the policy search bar,
When the search executes,
Then an LWC lightning-datatable returns matching Policy__c records where Policy__c.Status__c = 'In-Force' and the agent's Agent_Code__c matches Policy__c.Agent_Code__c, displaying Policy_Number__c, Policyholder_Name__c, Effective_Date__c, Expiration_Date__c, and LOB__c.

Given a policy is selected from search results,
When the detail panel loads,
Then a FlexCard (PolicySummaryCard) renders: Policy__c.Coverage_Limit__c, Policy__c.Deductible__c, count of related Claim__c records where Claim__c.Status__c NOT IN ('Closed','Withdrawn'), and Policy__c.Renewal_Date__c with a color-coded indicator (red if renewal within 30 days).

Given the agent searches by TIN,
When the TIN matches a Provider__c (Person Account) linked as the policyholder,
Then results include policies linked via Policy__c.Policyholder__c relationship and display the NPI alongside the policy number.

Given no matching records are found,
When the search returns zero results,
Then a contextual message displays: 'No in-force policies found for your agency code. Contact your Broker Relations Coordinator for assistance.'

-Given the search queries all Policy__c records regardless of agent assignment- (deprecated: scope restricted to agent's own book of business per compliance review INS-1061)

## Solution

Implement an LWC component (agentPolicySearch) using the @wire adapter with a custom Apex controller (AgentPolicySearchController.cls) that performs a SOQL query on Policy__c with WITH SECURITY_ENFORCED and a dynamic WHERE clause supporting name, policy number, and TIN inputs. The detail panel is a child OmniStudio FlexCard (PolicySummaryCard) invoked via a FlexCard action passing the Policy__c Id. Renewal proximity indicator uses a formula field Policy__c.Days_To_Renewal__c driving a conditional FlexCard state. Search is debounced at 400ms using a JavaScript timer in the LWC to prevent excessive SOQL calls.

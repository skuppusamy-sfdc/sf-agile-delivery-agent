## Description

**Bug Report**

**Summary:** When the nightly CAQH data sync job updates Provider__c.CAQH_Status__c to 'Re-Attestation Required', the associated Credentialing_Case__c record is not automatically reopened, leaving the case in 'Closed' status incorrectly. This causes credentialing specialists to miss re-attestation deadlines and may result in compliance violations.

**Steps to Reproduce:**
1. Identify a Provider__c (Person Account) record with an associated Credentialing_Case__c where Credentialing_Case__c.Status__c = 'Closed' and Credentialing_Case__c.Credential_Type__c = 'Initial'.
2. Simulate the CAQH nightly batch by updating Provider__c.CAQH_Status__c from 'Active' to 'Re-Attestation Required' via the DataIntegration batch job or manually.
3. Navigate to the associated Credentialing_Case__c record.
4. Observe that Credentialing_Case__c.Status__c remains 'Closed'.

**Expected Behavior:** When Provider__c.CAQH_Status__c changes to 'Re-Attestation Required', the most recent associated Credentialing_Case__c with Status__c = 'Closed' should automatically be reopened (Status__c = 'In Review') and reassigned to the Credentialing Specialist queue, and a task should be created for follow-up within 15 days.

**Actual Behavior:** Credentialing_Case__c.Status__c remains 'Closed'. No task is created. No queue reassignment occurs.

**Environment:** Full Sandbox (FS1)
**Frequency:** 100% reproducible
**Severity:** High — affects ~340 provider records in FS1 data set
**Root Cause (suspected):** The Record-Triggered Flow on Provider__c that handles CAQH status changes has a filter condition checking for Credentialing_Case__c.Credential_Type__c = 'Renewal' only, missing the 'Initial' credential type records.

## Acceptance Criteria

As per Description.

Additional verification:
Given the CAQH batch updates Provider__c.CAQH_Status__c to 'Re-Attestation Required',
When the Record-Triggered Flow executes,
Then Credentialing_Case__c.Status__c is set to 'In Review' for ALL associated closed cases regardless of Credentialing_Case__c.Credential_Type__c value ('Initial' or 'Renewal').

Given the case is reopened,
When the status update completes,
Then a Task is created with Task.Subject = 'CAQH Re-Attestation Follow-Up', Task.ActivityDate = TODAY() + 15, and Task.OwnerId = Credentialing Specialist queue Id.

## Solution

Fix the entry condition on the 'CAQH_Status_Change_Handler' Record-Triggered Flow: remove the filter restricting to Credential_Type__c = 'Renewal' so the flow processes all Credentialing_Case__c records in 'Closed' status. Add a Get Records element to retrieve related Credentialing_Case__c records filtering on Status__c = 'Closed' (removing Credential_Type__c filter). Update the loop to set Status__c = 'In Review' and OwnerId to the Credentialing Specialist queue. Add a Create Records element for the follow-up Task. Deploy fix to FS1 for regression testing before production push. Run data fix script to reopen the ~340 affected records in production via anonymous Apex after deployment.

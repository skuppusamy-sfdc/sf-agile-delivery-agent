## Description

As an agent user in the Agent Portal, I need to submit a First Notice of Loss (FNOL) on behalf of a policyholder directly from the portal, with the system automatically looking up and pre-populating policy details based on the entered policy number, so that claim intake is faster, less error-prone, and does not require a separate call to the claims center.

## Acceptance Criteria

Given an agent is authenticated in the Agent Portal and navigates to 'Submit FNOL',
When the FNOL OmniScript loads,
Then the first step presents a Policy Number input field and a 'Lookup Policy' button.

Given the agent enters a valid Policy__c.Policy_Number__c and clicks 'Lookup Policy',
When the DataRaptor executes the lookup,
Then Policy__c.Policyholder_Name__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, and Policy__c.Status__c are pre-populated in read-only fields on the OmniScript.

Given Policy__c.Status__c = 'Lapsed' or 'Cancelled' at time of FNOL submission,
When the lookup returns the policy,
Then a warning banner displays 'Policy is not currently active — claim may require SIU review' and the Claim__c.SIU_Referral_Flag__c is set to true upon record creation.

Given the agent completes all required FNOL fields and submits,
When the OmniScript Integration Procedure executes,
Then a Claim__c record is created with Claim__c.Status__c = 'FNOL Received', Claim__c.FNOL_Submitted_By__c = agent's Contact.Id, Claim__c.Policy__c = matched Policy__c.Id, and Claim__c.Date_of_Loss__c = agent-entered date.

Given the FNOL is successfully submitted,
When the confirmation step renders,
Then the new Claim__c.Claim_Number__c is displayed to the agent and a confirmation email is sent to Policy__c.Policyholder_Email__c via an Email Alert.

Given the agent enters a Policy_Number__c that does not match any Policy__c record,
When the lookup is triggered,
Then an inline error message 'Policy not found — please verify the policy number' is displayed and form submission is blocked.

## Solution

Build a multi-step OmniScript ('FNOL_Agent_Submission_OS') with a DataRaptor Turbo Extract for policy lookup on step 1. An Integration Procedure ('IP_Create_FNOL_Claim') orchestrates Claim__c record creation via a DataRaptor Load, evaluates SIU flag logic via a Calculation Procedure, and triggers the confirmation email via an Apex Action calling Messaging.sendEmail(). The OmniScript is embedded as an LWC in the Agent Portal Experience Cloud page. A Screen Flow fallback is provided for accessibility. Claim__c record creation respects OWD (Private) with a Sharing Rule granting the submitting agent read access to their created claim.

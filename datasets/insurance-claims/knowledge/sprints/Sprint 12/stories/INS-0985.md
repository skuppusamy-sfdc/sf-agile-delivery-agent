## Description

**Bug Report – ERA Generation Failure on COB Claims**

**Environment:** Acme Insurance Full Sandbox (ACME_FS1)
**Severity:** High – blocking payment disbursement for COB-adjusted claims

**Steps to Reproduce:**
1. Navigate to a Claim__c record where Claim__c.COB_Applied__c = true and at least one Claim_Line__c has Claim_Line__c.COB_Adjustment_Amount__c > 0.
2. Set Claim__c.Status__c = 'Adjudicated' and Claim__c.Payment_Method__c = 'ERA'.
3. Trigger the 'Generate ERA' batch job (BatchERAFileGenerator) from the Payment Operations custom tab.
4. Observe the batch job execution log in Apex Jobs.

**Expected Behavior:** The ERA file (835 transaction set) is generated successfully, including CAS segments reflecting COB adjustment amounts for each Claim_Line__c with Claim_Line__c.COB_Adjustment_Amount__c > 0.

**Actual Behavior:** The BatchERAFileGenerator throws a System.NullPointerException at ERALineBuilder.buildCASSegment() line 87. The batch job completes with 0 successes and logs the error: 'Attempt to de-reference a null object'. All COB-adjusted claims remain in Claim__c.Payment_Status__c = 'Pending ERA' indefinitely.

**Root Cause Hypothesis:** ERALineBuilder.buildCASSegment() does not null-check Claim_Line__c.COB_Payer_Name__c before concatenating it into the CAS02 segment string. When COB_Payer_Name__c is null (populated only for secondary payer scenarios), the string builder throws NPE.

**Impact:** 47 claims totaling approximately $183,000 in pending disbursements are blocked as of sprint start.

## Acceptance Criteria

As per Description. Additionally:

Given a Claim__c with COB_Applied__c = true and one or more Claim_Line__c records where COB_Adjustment_Amount__c > 0 and COB_Payer_Name__c is null,
When the BatchERAFileGenerator processes the claim,
Then the ERA file is generated successfully with CAS segments using 'UNKNOWN' as the fallback value for CAS02 when COB_Payer_Name__c is null, and Claim__c.Payment_Status__c transitions to 'ERA Generated'.

Given the batch job processes a mix of COB and non-COB claims in the same batch execution,
When the job completes,
Then non-COB claims are not affected and their ERA records are generated correctly regardless of COB claim errors.

## Solution

Fix ERALineBuilder.buildCASSegment() (line 87) to null-check Claim_Line__c.COB_Payer_Name__c: use String.isBlank() guard and substitute 'UNKNOWN' as the default CAS02 value. Add a try-catch at the individual Claim_Line__c iteration level within the batch execute() method so a single line failure does not abort the entire batch scope. Update BatchERAFileGenerator to collect failed Claim__c IDs and set Payment_Status__c = 'ERA Error' with an error message logged to a custom ERA_Error_Log__c object for ops visibility. Deploy hotfix to sandbox, execute regression test on 47 blocked claims, then promote to production via change set.

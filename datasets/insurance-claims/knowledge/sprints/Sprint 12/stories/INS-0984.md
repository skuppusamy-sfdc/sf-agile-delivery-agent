## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard that surfaces active Provider_Network_Agreement__c (PNA) details and associated Fee_Schedule__c information so that I can quickly assess a provider's contracted status, network participation, and reimbursement rates during provider inquiries and claims adjudication support — without navigating across multiple Salesforce objects.

## Acceptance Criteria

Given a Provider (Person Account) record is open in the internal Salesforce console,
When the Provider 360 FlexCard loads,
Then it displays all active Provider_Network_Agreement__c records where PNA.Status__c = 'Active' and PNA.Provider__c = current Account.Id, showing PNA.Network_Name__c, PNA.Effective_Date__c, PNA.Termination_Date__c, and PNA.Contract_Type__c.

Given an active PNA has an associated Fee_Schedule__c (Fee_Schedule__c.PNA__c = PNA.Id),
When the specialist expands a PNA row on the FlexCard,
Then a child DataTable displays Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Allowed_Amount__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.LOB__c for up to 50 fee schedule lines.

Given a provider has a Provider_Network_Agreement__c where PNA.Termination_Date__c is within 90 days of today,
When the FlexCard renders,
Then a yellow warning badge displays 'Termination Pending' with the exact PNA.Termination_Date__c value.

Given the provider has no active PNA records,
When the FlexCard loads,
Then the network section displays 'No Active Network Agreements Found' and a 'Initiate PNA' quick action button is visible.

Given the logged-in user does not have the 'Provider_Network_Read' Permission Set,
When the FlexCard attempts to load PNA data,
Then the PNA section is hidden and replaced with 'Insufficient Permissions — contact your administrator'.

## Solution

Build the Provider 360 FlexCard using OmniStudio FlexCards with child card nesting. A DataRaptor Extract ('DR_GetProviderPNADetails') retrieves Provider_Network_Agreement__c and related Fee_Schedule__c records in a single SOQL with a subquery (limit 50 on Fee Schedule lines, ordered by Effective_Date__c DESC). The 90-day termination warning is implemented as a conditional state on the FlexCard using a formula: TODAY() + 90 >= PNA.Termination_Date__c. The 'Initiate PNA' quick action invokes an OmniScript ('OS_InitiatePNA') via FlexCard action. Permission enforcement handled via a custom Apex method exposed as a Remote Action that checks FeatureManagement or CRUD on Provider_Network_Agreement__c before returning data.

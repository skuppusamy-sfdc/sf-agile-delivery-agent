## Description

As a Claims Operations Manager, I need the system to ingest inbound ANSI X12 835 Electronic Remittance Advice (ERA) files from payer partners and automatically reconcile the payment details against open Payment__c records so that manual ERA posting is eliminated and claim adjudication accuracy is improved.

ERA files are delivered to an SFTP endpoint every business day at 6:00 AM CT. Each 835 file contains one or more Claim Payment segments (CLP) that map to Claim__c and Claim_Line__c records in Salesforce.

## Acceptance Criteria

Given an ERA 835 file is deposited on the SFTP endpoint,
When the scheduled Apex batch job 'ERA835IngestBatch' executes at 6:30 AM CT,
Then the file is retrieved, parsed, and each CLP segment is matched to a Payment__c record using Payment__c.Check_EFT_Number__c = CLP03 (payer claim control number).

Given a CLP segment is successfully matched to a Payment__c record,
When the reconciliation logic runs,
Then Payment__c.Status__c is updated to 'Reconciled', Payment__c.ERA_Received_Date__c = file processing date, and Payment__c.Paid_Amount__c is set to the CLP04 value from the 835.

Given a CLP segment contains a claim adjustment reason code (CARC) in the CAS segment,
When the adjustment is processed,
Then a Claim_Line__c.Adjustment_Reason_Code__c field is populated with the CARC value and Claim_Line__c.Adjusted_Amount__c is recalculated accordingly.

Given a CLP segment cannot be matched to any Payment__c record,
When the unmatched condition is detected,
Then an ERA_Exception__c record is created with ERA_Exception__c.Exception_Type__c = 'Unmatched CLP', ERA_Exception__c.Raw_Segment__c = the unparsed CLP string, and the record is assigned to the Claims Operations Manager queue for manual review.

Given the full ERA file is processed,
When the batch job completes,
Then a summary Platform Event ERA_Processing_Complete__e is published containing: file name, total CLPs processed, matched count, unmatched count, and total dollar amount reconciled.

-Given ERA files arrive in HL7 FHIR format, process using FHIR parser- (deprecated — confirmed all payer partners deliver ANSI X12 835 only per integration spec v2.3)

Given the ERA Processing Complete Platform Event is received,
When a downstream subscriber Flow processes the event,
Then an automated report summary email is sent to the Claims Operations Manager distribution list within 5 minutes of batch completion.

## Solution

Scheduled Apex Batch class 'ERA835IngestBatch' using Salesforce Files or an external SFTP adapter (MuleSoft) to retrieve the 835 file. Custom Apex parser class 'X12_835_Parser' tokenizes ISA/GS/ST/CLP/CAS/SVC segments. Reconciliation logic in 'ERA835ReconciliationService' Apex class performs SOQL lookup on Payment__c by Check_EFT_Number__c. Unmatched records create ERA_Exception__c via a separate Apex method. Platform Event ERA_Processing_Complete__e published at batch finish; subscriber Record-Triggered Flow sends email via Email Alert. Custom metadata type 'ERA_Field_Mapping__mdt' stores CLP segment-to-Salesforce field mappings for maintainability without code changes. Bulk-safe design using collections and Database.update with partial success handling.

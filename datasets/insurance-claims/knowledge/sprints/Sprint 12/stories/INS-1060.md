## Description

As a Broker Relations Coordinator, I need the system to automatically calculate and post commission chargebacks when a policy lapses within 90 days of the bind date so that agent commission accounts are adjusted without manual intervention and the TPA reconciliation process remains accurate.

A chargeback should create a negative Payment__c record linked to the original commission Payment__c and notify the agent via the portal.

## Acceptance Criteria

Given a Policy__c record transitions to Status__c = 'Lapsed',
When DATEVALUE(TODAY()) - Policy__c.Bind_Date__c <= 90,
Then a Record-Triggered Flow fires After Update and calls an Apex action (ChargebackCalculator.cls) to compute the chargeback amount as Payment__c.Amount__c * Policy__c.Chargeback_Rate__c.

Given the chargeback amount is calculated,
When the Apex action completes,
Then a new Payment__c record is inserted with Payment__c.Type__c = 'Chargeback', Payment__c.Amount__c = negative calculated value, Payment__c.Related_Commission__c = original Payment__c.Id, and Payment__c.Status__c = 'Posted'.

Given the chargeback Payment__c is created,
When the record is saved,
Then a Platform Event (Commission_Updated__e) is published to trigger real-time refresh of the Agent Portal commission dashboard (INS-1058).

Given the chargeback is posted,
When the agent next logs into the portal,
Then an in-app notification displays: 'Chargeback of $[amount] applied to Policy [Policy_Number__c] due to lapse within 90-day window.'

Given Policy__c.Lapse_Reason__c = 'Non-Payment' AND Policy__c.Reinstatement_Requested__c = TRUE,
When the chargeback is in 'Posted' status,
Then the chargeback Payment__c.Status__c is set to 'Held' and a Task is created for the Broker Relations Coordinator to review reinstatement eligibility before finalizing.

## Solution

Implement a Record-Triggered Flow on Policy__c (After Update, entry criteria Status__c Changed AND Status__c = 'Lapsed') calling an Apex invocable action (ChargebackCalculator.cls). The Apex class queries the originating Payment__c via Policy__c.Id, computes chargeback, inserts negative Payment__c, and publishes Commission_Updated__e Platform Event. Use Custom Metadata (Chargeback_Rules__mdt) to store LOB-specific chargeback rate overrides. Bell notification delivered via custom LWC notification bell component in the Agent Portal Experience Cloud site.

## Description

As a Provider Relations Specialist, I need to see a consolidated view of a provider's active Provider_Network_Agreement__c (PNA) records and associated Fee_Schedule__c details directly on the Provider 360 FlexCard so that I can answer provider inquiries about contracted rates and network participation status without navigating to multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c (Person Account) record in Salesforce,
When the 'Provider 360' FlexCard loads,
Then all PNA records where Provider_Network_Agreement__c.Provider__c = current Provider__c.Id AND Provider_Network_Agreement__c.Status__c = 'Active' are displayed in a child datatable.

Given the PNA datatable renders,
When the specialist clicks on a PNA row,
Then a flyout panel displays:
  - Provider_Network_Agreement__c.Network_Name__c
  - Provider_Network_Agreement__c.Effective_Date__c
  - Provider_Network_Agreement__c.Termination_Date__c
  - Provider_Network_Agreement__c.Contract_Type__c
  - Linked Fee_Schedule__c.Schedule_Name__c
  - Fee_Schedule__c.Effective_Date__c
  - Fee_Schedule__c.Base_Rate_Multiplier__c

Given a PNA has no linked Fee_Schedule__c,
When the flyout renders,
Then the fee schedule section displays 'No Fee Schedule on file — contact Network Contracting.'

Given a PNA's Termination_Date__c is within 60 days of TODAY(),
When the FlexCard renders,
Then the PNA row is highlighted in amber and a tooltip reads 'Contract expiring soon — renewal required.'

Given the specialist has the 'Provider_Relations_ReadOnly' Permission Set assigned (not full Network Contracting access),
When the FlexCard loads,
Then Fee_Schedule__c.Base_Rate_Multiplier__c is masked as '****' and a tooltip reads 'Rate details restricted — contact your Network Contracting team.'

## Solution

OmniStudio FlexCard 'Provider360Card' with a child DataRaptor Extract pulling Provider_Network_Agreement__c and related Fee_Schedule__c via a SOQL relationship query. Flyout panel implemented as a nested FlexCard state. Amber highlight logic driven by a FlexCard conditional formatting rule comparing Termination_Date__c to a formula field TODAY() + 60. Rate masking handled via a custom Lightning Data Service Apex controller that checks the running user's Permission Set assignments before returning Fee_Schedule__c.Base_Rate_Multiplier__c. Deployed to the Provider Record Page via Lightning App Builder.

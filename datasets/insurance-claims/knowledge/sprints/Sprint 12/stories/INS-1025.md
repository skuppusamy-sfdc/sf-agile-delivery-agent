## Description

As a Provider Network Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record and initiate the 90-day notice workflow when a Network_Termination_Case__c is approved so that we meet CMS and state regulatory requirements for provider termination notice without manual intervention.

Currently PCNs are created manually after termination case approval, causing an average 4-day delay that has resulted in two regulatory compliance findings in Q3.

## Acceptance Criteria

Given a Network_Termination_Case__c.Status__c transitions to 'Approved',
When the record is saved,
Then a Record-Triggered Flow immediately creates a Provider_Change_Notification__c record with:
  - PCN_Type__c = 'Termination'
  - Provider__c = Network_Termination_Case__c.Provider__c
  - PNA__c = Network_Termination_Case__c.Provider_Network_Agreement__c
  - Notice_Date__c = TODAY()
  - Effective_Termination_Date__c = TODAY() + 90
  - Status__c = 'Draft'
  - Generated_By__c = 'Automated'

Given the PCN record is created,
When Status__c = 'Draft',
Then a Conga CLM document generation action fires to produce the 'Provider_Termination_Notice_v2' PDF template populated with provider NPI, TIN, network name, and effective termination date, and the generated document is attached to the PCN as a ContentDocument.

Given the PCN document is generated,
When the document is attached,
Then PCN.Status__c auto-advances to 'Pending Review' and a Task is assigned to the owning Provider Network Specialist with Subject = 'Review and Send PCN — [Provider Name]' due within 2 business days.

Given the specialist sets PCN.Status__c = 'Sent',
When the record is saved,
Then Provider_Network_Agreement__c.Termination_Notice_Sent__c = true, Provider_Network_Agreement__c.Termination_Notice_Date__c = TODAY(), and an automated email is sent to the provider's Provider__c.Email using the 'PCN_Termination_Email' template.

Given Effective_Termination_Date__c arrives and PCN.Status__c = 'Sent',
When a scheduled Flow runs nightly,
Then Provider_Network_Agreement__c.Status__c is updated to 'Terminated' and Claim__c records in status 'Received' or 'In Adjudication' with this provider are flagged with Claim__c.Provider_Termination_Flag__c = true for SIU review.

Given the termination workflow completes,
When the Compliance Officer views the PCN Compliance Report,
Then the report shows Network_Termination_Case__c.Approved_Date__c, PCN.Notice_Date__c, and PCN.Sent_Date__c with a calculated Notice_Delay_Days__c field to evidence regulatory compliance.

## Solution

Record-Triggered Flow ('Network_Termination_PCN_Automation') on Network_Termination_Case__c fires after update when Status__c changes to 'Approved'. Flow creates Provider_Change_Notification__c, then calls an Apex action 'CongaPCNGenerationAction' invoking Conga CLM REST API with template ID and merge field map. A separate Scheduled Flow ('Nightly_PNA_Termination_Enforcer') runs at 2 AM daily querying PCN records where Effective_Termination_Date__c = TODAY() and Status__c = 'Sent'. Notice_Delay_Days__c is a formula field on PCN: Notice_Date__c - Network_Termination_Case__c.Approved_Date__c. Compliance report built as a Lightning Report with summary grouping by Termination_Reason__c and conditional highlighting for Notice_Delay_Days__c > 1.

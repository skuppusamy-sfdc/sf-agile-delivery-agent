## Description

As a Claims Operations Manager, I need the adjudication engine to automatically look up the contracted rate for each Claim_Line__c based on the provider's active Fee_Schedule__c, CPT code, and place of service so that allowed amounts are calculated consistently and without manual rate card lookups.

The lookup must respect the effective date of the Fee_Schedule__c and the provider's Provider_Network_Agreement__c (PNA) to ensure only in-network rates are applied.

## Acceptance Criteria

Given a Claim_Line__c record is created with Claim_Line__c.CPT_Code__c populated,
When the adjudication flow triggers the fee schedule lookup,
Then the system queries Fee_Schedule__c records where Fee_Schedule__c.Provider__c = Claim__c.Billing_Provider__c AND Fee_Schedule__c.Effective_Date__c <= Claim__c.Service_Date__c AND Fee_Schedule__c.Expiration_Date__c >= Claim__c.Service_Date__c.

Given a matching Fee_Schedule__c record is found,
When the CPT code lookup executes,
Then Fee_Schedule_Line__c records are queried where Fee_Schedule_Line__c.CPT_Code__c = Claim_Line__c.CPT_Code__c AND Fee_Schedule_Line__c.Place_of_Service__c = Claim_Line__c.Place_of_Service__c, and the matched rate is written to Claim_Line__c.Allowed_Amount__c.

Given the provider has an active Provider_Network_Agreement__c (PNA__c.Status__c = 'Active'),
When the rate is applied,
Then Claim_Line__c.Network_Status__c = 'In-Network' and Claim_Line__c.Reimbursement_Basis__c = 'Contracted Rate'.

Given no matching Fee_Schedule_Line__c is found for the CPT code,
When the lookup returns zero results,
Then Claim_Line__c.Allowed_Amount__c defaults to Claim_Line__c.Billed_Amount__c * (CMS_UCR_Rate__mdt.UCR_Percentage__c / 100) and Claim_Line__c.Network_Status__c = 'Out-of-Network'.

Given a provider's PNA__c.Status__c = 'Terminated',
When the fee schedule lookup runs,
Then regardless of any active Fee_Schedule__c, Claim_Line__c.Network_Status__c is forced to 'Out-of-Network' and a warning flag Claim_Line__c.PNA_Termination_Warning__c = true is set.

## Solution

Apex invocable method 'FeeScheduleLookupService.lookupAllowedAmount()' called from the Claim Adjudication Record-Triggered Flow. Method performs SOQL with date-effective filtering on Fee_Schedule__c and child Fee_Schedule_Line__c objects. PNA status check performed in the same Apex call before rate assignment. UCR fallback rate stored in Custom Metadata Type 'CMS_UCR_Rate__mdt' keyed by LOB and CPT code range. Bulk-safe implementation processes Claim_Line__c collections in a single transaction. Unit tests cover: matched rate, unmatched CPT fallback, terminated PNA override, and expired fee schedule edge cases. Deployed and regression tested in Sprint 11; marked Done after UAT sign-off by Claims Operations Manager.

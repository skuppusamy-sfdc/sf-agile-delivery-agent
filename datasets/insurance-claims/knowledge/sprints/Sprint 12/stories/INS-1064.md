## Description

As a Credentialing Specialist, I need the system to automatically fetch provider attestation data from CAQH ProView and NPI data from NPPES when a new Credentialing_Case__c is created so that I don't need to manually look up and re-enter provider demographics, license information, and board certifications during the initial credentialing review.

The fetched data should pre-populate the Credentialing_Case__c and the linked Provider__c (Person Account) record, with discrepancies flagged for specialist review.

## Acceptance Criteria

Given a new Credentialing_Case__c record is created with Credentialing_Case__c.Provider__c populated and Credentialing_Case__c.CAQH_Provider_ID__c populated,
When the record is saved,
Then a Record-Triggered Flow (After Insert) enqueues an Apex Queueable job (CAQHFetchJob.cls) to call the CAQH ProView REST API (named credential: CAQH_ProView_API).

Given the CAQH API call returns a successful response,
When the job processes the response payload,
Then the following fields are populated on Credentialing_Case__c: Credentialing_Case__c.CAQH_Attestation_Date__c, Credentialing_Case__c.Board_Certification__c, Credentialing_Case__c.Malpractice_Coverage_Verified__c, Credentialing_Case__c.DEA_Number__c, and Credentialing_Case__c.License_State__c.

Given the NPPES NPI Registry is also queried using Provider__c.NPI__c,
When the NPPES response is received,
Then Provider__c.First_Name__c, Provider__c.Last_Name__c, Provider__c.Primary_Specialty__c, Provider__c.Practice_Address__c, and Provider__c.TIN__c are compared against existing record values; any mismatch sets Credentialing_Case__c.Data_Discrepancy_Flag__c = TRUE and populates Credentialing_Case__c.Discrepancy_Details__c with a JSON diff summary.

Given Credentialing_Case__c.Data_Discrepancy_Flag__c = TRUE,
When the record is updated,
Then the Credentialing_Case__c is assigned to the 'Credentialing_Review_Queue' and a Task is created for the Credentialing Specialist with Subject = 'Review CAQH/NPPES Data Discrepancy — [Provider Name]'.

Given the CAQH API returns a 404 (provider not found) or timeout,
When the error is caught in the Queueable,
Then Credentialing_Case__c.CAQH_Fetch_Status__c = 'Failed', an Integration_Error_Log__c record is created, and the case is flagged for manual CAQH lookup.

## Solution

Record-Triggered Flow on Credentialing_Case__c (After Insert) calls Apex Queueable (CAQHFetchJob.cls) which performs sequential REST callouts: first to CAQH ProView API (named credential CAQH_ProView_API) and second to NPPES NPI Registry public API (named credential NPPES_API). JSON response parsing via a dedicated CAQHResponseParser utility class. Field comparison logic implemented in Apex with JSON serialization of delta stored in Credentialing_Case__c.Discrepancy_Details__c (Long Text Area). Custom Metadata (CAQH_Field_Mapping__mdt) drives field mapping to allow configuration without code changes. Integration_Error_Log__c captures API failures.

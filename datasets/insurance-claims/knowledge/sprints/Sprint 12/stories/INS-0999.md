## Description

As a Provider Network Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces all active Provider_Network_Agreement__c (PNA) contract terms — including reimbursement model, effective dates, and linked Fee_Schedule__c — so that I can quickly assess a provider's contractual standing without navigating multiple related lists during claim adjudication or re-credentialing reviews.

## Acceptance Criteria

Given a Provider__c record is open in the Service Console,
When the Provider 360 FlexCard loads,
Then it displays all Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active' AND PNA__c.Provider__c = current record Id, showing: PNA__c.Contract_Number__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, PNA__c.Reimbursement_Model__c, and the linked Fee_Schedule__c.Schedule_Name__c.

Given a PNA has PNA__c.Termination_Date__c within 30 days of TODAY(),
When the FlexCard renders that PNA row,
Then an amber warning icon and label 'Expiring Soon' are shown inline next to the Termination_Date__c value.

Given a Provider__c has no active PNA records,
When the FlexCard loads,
Then an empty-state message reads 'No Active Network Agreements on file. Initiate contracting via the New PNA action.'

Given the specialist clicks 'View Fee Schedule' on a PNA row,
When the action fires,
Then a child FlexCard panel expands inline showing Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.CPT_Code_Range__c, and Fee_Schedule__c.Allowed_Amount__c for up to 10 line items, paginated.

-Given the FlexCard also shows credentialing expiry from Credentialing_Case__c- (deprecated — credentialing panel moved to standalone Credentialing 360 FlexCard in INS-0984)

Given the FlexCard is rendered,
When the user's Profile does not include the 'Provider_Network_Read' Permission Set,
Then Fee_Schedule__c.Allowed_Amount__c fields are masked as '***' and a tooltip reads 'Restricted: Contact your Network Admin'.

## Solution

Build OmniStudio FlexCard 'Provider_360_PNA_FC' with a DataRaptor Extract querying Provider_Network_Agreement__c filtered by Provider__c = {recordId} AND Status__c = 'Active', with a left join to Fee_Schedule__c. Add a conditional state action for Termination_Date__c proximity using a formula: IF(DATEDIFF(Termination_Date__c, TODAY()) <= 30, 'expiring', 'active'). Child FlexCard 'Fee_Schedule_Detail_FC' is toggled via a Show/Hide card action. Field-level masking for Allowed_Amount__c implemented via a custom LWC wrapper that checks FeatureManagement.checkPermission('View_Fee_Schedule_Amounts') before rendering. Embed the FlexCard on the Provider__c Lightning Record Page via a custom tab in the Service Console app using Dynamic Forms.

## Description

As a Claims Operations Manager, I need the Fee Schedule Engine to apply tiered CPT code reimbursement rates based on the provider's network tier (Tier 1, Tier 2, Out-of-Network) defined in their Provider_Network_Agreement__c so that Claim_Line__c adjudication calculates the correct allowed amount without manual override.

Currently all providers are reimbursed at a flat Fee_Schedule__c rate regardless of PNA tier, causing systematic overpayment on Tier 2 providers (estimated $240K/month exposure identified by Underwriting Analyst team).

## Acceptance Criteria

Given a Claim_Line__c record is submitted for adjudication with a valid CPT_Code__c,
When the Fee Schedule Engine resolves the reimbursement rate,
Then it first looks up the rendering provider's Provider_Network_Agreement__c.Network_Tier__c (values: 'Tier1', 'Tier2', 'OON') before selecting the matching Fee_Schedule__c row.

Given Fee_Schedule__c records exist for the same CPT_Code__c with Tier__c = 'Tier1' (rate $150), 'Tier2' (rate $110), and 'OON' (rate $85),
When a Tier 2 provider's Claim_Line__c is adjudicated,
Then Claim_Line__c.Allowed_Amount__c = 110.00 and Claim_Line__c.Fee_Schedule_Applied__c is populated with the matching Fee_Schedule__c record ID.

Given no Fee_Schedule__c row exists for the CPT_Code__c + Tier combination,
When adjudication runs,
Then Claim_Line__c.Status__c = 'Pended' and Claim_Line__c.Pend_Reason__c = 'Missing Fee Schedule: CPT {code} Tier {tier}' and a Task is created for the Provider Network Specialist.

Given a provider's PNA has expired (Provider_Network_Agreement__c.End_Date__c < Claim_Line__c.Service_Date__c),
When adjudication runs,
Then the provider is treated as 'OON' for rate resolution and Claim__c.COB_Flag__c is evaluated.

-Given multiple active PNAs exist for a provider, the oldest PNA is used for tier resolution- (deprecated: logic updated to use PNA with most recent Effective_Date__c per INS-1008 resolution).

## Solution

Extend the existing Apex class 'FeeScheduleResolverService' with a new method 'resolveByTier(ClaimLine, Provider)' that queries Provider_Network_Agreement__c filtered by Provider__c, Status__c = 'Active', and Effective_Date__c <= Service_Date__c, ordered by Effective_Date__c DESC LIMIT 1 to get the current tier. Then queries Fee_Schedule__c with CPT_Code__c and Tier__c match. Result is cached in a Platform Cache partition ('FeeScheduleCache') keyed by CPT+Tier to reduce SOQL per bulk adjudication batch. Batch Apex class 'ClaimLineAdjudicationBatch' calls the updated service. Unit tests cover all tier permutations and the expired-PNA OON fallback scenario with ≥85% coverage.

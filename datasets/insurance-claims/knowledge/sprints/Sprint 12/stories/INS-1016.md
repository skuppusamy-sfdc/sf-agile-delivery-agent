## Description

**Bug Report — INS-1016**

**Summary:** The Commission Dashboard (delivered in INS-1011) displays duplicate Payment__c entries in the earnings breakdown when an agent has more than one active Group_Service_Agreement__c record. Each Payment__c appears once per active GSA rather than once per payment record, causing inflated YTD totals.

**Environment:** Acme Insurance Sprint 12 Sandbox (CS112)

**Steps to Reproduce:**
1. Log in as an agent user (e.g., test agent 'broker.test.multi@acmeins.com') who has 2 active GSA records linked to their Agent__c.
2. Navigate to Agent Portal > Commission Dashboard tab.
3. Observe the 'YTD Paid' KPI tile and the LOB bar chart.
4. Compare displayed total against a direct SOQL query: SELECT SUM(Amount__c) FROM Payment__c WHERE Commission_Agent__c = [agent_id] AND Commission_Status__c = 'Paid'.

**Expected Behavior:** Each Payment__c record appears exactly once in the dashboard totals regardless of how many GSAs are linked to the agent.

**Actual Behavior:** Payment__c records are duplicated in the DataRaptor Turbo Extract result set — one row per GSA — causing YTD total to be multiplied by the number of active GSAs (e.g., 2 GSAs = 2x inflated total).

**Root Cause Hypothesis:** The DataRaptor Turbo Extract on the Commission Dashboard FlexCard joins Payment__c to Group_Service_Agreement__c without a DISTINCT or deduplication step, producing a Cartesian-style result when multiple GSAs exist.

**Severity:** High — financial data inaccuracy visible to agents; potential for trust and compliance issues.

## Solution

Investigate the DataRaptor Turbo Extract 'DR_Commission_Dashboard_Extract' for the join condition between Payment__c and Group_Service_Agreement__c. Replace the direct GSA join with a subquery or restructure to query Payment__c independently (using Commission_Agent__c lookup) without joining GSA table. If GSA tier label is needed for display, fetch it via a separate DataRaptor lookup step keyed on Agent__c.Current_GSA__c (a formula/rollup field to be added). Add a unit test in the QA sandbox with a multi-GSA agent profile to validate deduplication. Regression test INS-1011 ACs post-fix.

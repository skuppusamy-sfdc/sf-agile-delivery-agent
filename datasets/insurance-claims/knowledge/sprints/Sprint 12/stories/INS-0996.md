## Description

**Bug Report: Stale Tier Data Displayed on Agent Commission Dashboard After Mid-Year Upgrade**

**Environment:** Acme Insurance Full Sandbox (Sprint 12) — reproducible in UAT
**Severity:** High — agents are seeing incorrect tier status (Silver instead of Gold) on the commission dashboard (INS-0989), leading to incorrect projected payout calculations being shown.

**Steps to Reproduce:**
1. Log in as an agent whose Agent_Tier__c.Tier_Level__c is currently 'Silver'.
2. Open the Agent Portal Commission Dashboard — note tier displayed as 'Silver'.
3. As an admin, update Agent_Tier__c.Tier_Level__c to 'Gold' and save.
4. Without clearing browser cache or refreshing, return to the Commission Dashboard.
5. Observe the tier still displays as 'Silver' and the projected payout still uses the Silver rate (0.03 vs Gold rate 0.045).
6. The correct tier only appears after a full browser hard-refresh (Ctrl+Shift+R).

**Expected Behavior:**
Within 60 seconds of Agent_Tier__c.Tier_Level__c being updated, the Commission Dashboard FlexCard should reflect the new tier and recalculate projected payout using the updated rate, as specified in INS-0989 AC 6 (Platform Event Commission_Tier_Updated__e subscription).

**Actual Behavior:**
The Platform Event (Commission_Tier_Updated__e) is confirmed publishing (verified in Event Monitoring logs) but the FlexCard Emp API listener is not re-executing the DataRaptor Turbo Extract on receipt of the event. The FlexCard is rendering cached DataRaptor output from the initial page load. Stale data persists until hard browser refresh.

**Root Cause Hypothesis:**
The DataRaptor Turbo Extract result is being cached at the OmniStudio session cache layer. The Platform Event listener triggers a FlexCard state update but does not invalidate or re-execute the DataRaptor Extract, only re-rendering the existing cached dataset.

**Impact:** Agents may make business decisions based on incorrect tier and projected commission figures. Affects all agents who receive a mid-year tier upgrade while an active portal session is open.

## Acceptance Criteria

As per Description.

Additional verification:
Given Agent_Tier__c.Tier_Level__c is updated from 'Silver' to 'Gold',
When the Commission_Tier_Updated__e Platform Event is received by the FlexCard Emp API listener,
Then the FlexCard forces a full DataRaptor Turbo Extract re-execution (cache-busted) and the displayed Tier_Level__c and Rate__c update to reflect 'Gold' within 60 seconds without requiring a browser refresh.

Given the fix is deployed,
When a regression test is run for all tier transitions (Bronze→Silver, Silver→Gold, Gold→Platinum),
Then all tier changes reflect on the dashboard within the 60-second SLA for active portal sessions.

## Solution

Root cause confirmed: the OmniStudio DataRaptor Turbo Extract has 'Enable Cache' set to true with a default session TTL of 300 seconds, causing stale results to be served on Platform Event receipt. Fix: disable DataRaptor-level caching for the Commission_Dashboard_DR extract (set cacheMaxAge = 0) and instead implement targeted cache invalidation in the LWC wrapper's Platform Event handler — on receipt of Commission_Tier_Updated__e, call the OmniStudio DataRaptor JavaScript API to force a fresh extract execution by appending a cache-busting timestamp parameter to the DataRaptor invocation. Validate fix in sandbox against all tier transition scenarios. Update INS-0989 solution documentation to note cache configuration requirement.

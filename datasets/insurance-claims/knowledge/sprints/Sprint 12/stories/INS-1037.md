## Description

As a Broker Relations Coordinator, I need an automated chargeback processing workflow so that when a Policy__c lapses within the clawback window defined in the agent's commission agreement, the system automatically calculates the chargeback amount, creates a debit Payment__c record against the agent's account, and notifies the agent via the Agent Portal.

The clawback window and chargeback percentage are configured per LOB in a custom metadata type. The process must handle partial-month proration and update the agent's net commission balance in real time.

## Acceptance Criteria

Given a Policy__c record transitions to Status__c = 'Lapsed',
When the lapse date falls within the chargeback window (defined in Commission_Chargeback_Config__mdt.Clawback_Window_Days__c),
Then a new Payment__c record is created with Payment__c.Type__c = 'Chargeback', Payment__c.Status__c = 'Pending', and Payment__c.Amount__c calculated as the prorated chargeback amount.

Given the chargeback Payment__c is created,
When the agent logs into the Agent Portal,
Then the chargeback appears in a 'Chargebacks & Adjustments' section of the Commission Dashboard with Policy__c reference, lapse date, and chargeback amount.

Given the chargeback is processed,
When the agent's YTD commission balance is recalculated,
Then the net figure reflects the deduction and Payment__c.Net_Commission_Balance__c is updated on the Broker__c record.

Given a policy lapses after the clawback window has expired,
When the system evaluates the chargeback eligibility,
Then no chargeback Payment__c is created and a log entry is written to the Commission_Audit_Log__c object.

-Given chargebacks trigger an automatic email to the agent- (replaced by in-portal notification per UX review)

## Solution

Implement a Record-Triggered Flow on Policy__c (RTF_Policy_Lapse_Chargeback) triggered on Status__c change to 'Lapsed'. Flow queries Commission_Chargeback_Config__mdt for LOB-specific rules and invokes an Apex Action (ChargebackCalculationService.cls) for proration math. Chargeback Payment__c records are inserted via the Apex service. A Platform Event (Commission_Chargeback__e) is published post-insert to update the Agent Portal FlexCard in real time. Commission_Audit_Log__c records are written via a separate Apex method for non-chargeback scenarios. Custom Permission 'View_Chargeback_Details' added to PSG_Agent_Portal_Standard.

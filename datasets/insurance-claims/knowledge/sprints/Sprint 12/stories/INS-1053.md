## Description

**Bug Report – Commission Amount Calculates as $0.00 for Renewal Policies**

**Environment:** Acme Insurance – Full Sandbox (Sprint 12 deploy, build 12.3)

**Steps to Reproduce:**
1. Navigate to an existing Policy__c record where Policy__c.Is_Renewal__c = true and Policy__c.Policy_Year__c = 2 or greater.
2. Update Policy__c.Status__c from 'Pending Renewal' to 'Active' to trigger the commission calculation flow (INS-1052).
3. Navigate to the related Commission_Transaction__c record created by the flow.
4. Observe Commission_Transaction__c.Amount__c.

**Expected Result:** Commission_Transaction__c.Amount__c should equal Policy__c.Annual_Premium__c × Agent_Commission_Schedule__c.Renewal_Rate__c for the agent's LOB and tier.

**Actual Result:** Commission_Transaction__c.Amount__c = $0.00 for all renewal policies where Agent_Commission_Schedule__c.Policy_Year_Min__c > 1. New business commissions (Policy_Year__c = 1) calculate correctly.

**Root Cause Hypothesis:** The Get Records element in the Record-Triggered Flow filtering Agent_Commission_Schedule__c does not include a filter condition on Agent_Commission_Schedule__c.Policy_Year_Min__c, causing the query to return no record for year-2+ schedules and defaulting the amount formula to null/$0.00 instead of routing to the 'Missing Schedule' task path.

**Workaround:** Manually update Commission_Transaction__c.Amount__c after creation. Affects approximately 14 renewal policies processed in Sprint 12 UAT.

## Acceptance Criteria

As per Description. Post-fix: Commission_Transaction__c.Amount__c must be non-zero and match the expected renewal rate for policies with Policy__c.Policy_Year__c >= 2 where a matching Agent_Commission_Schedule__c record with Policy_Year_Min__c <= Policy_Year__c <= Policy_Year_Max__c exists.

## Solution

Update the Get Records element in the 'PolicyIssuanceCommissionCalc' Record-Triggered Flow to add filter conditions: Agent_Commission_Schedule__c.Policy_Year_Min__c <= {!Policy.Policy_Year__c} AND Agent_Commission_Schedule__c.Policy_Year_Max__c >= {!Policy.Policy_Year__c}. Re-test with Policy_Year__c values of 1, 2, 3, and 5 to confirm correct schedule retrieval across all tiers. Additionally, add a null-safe check on the retrieved Renewal_Rate__c before the Amount formula assignment to ensure the 'Missing Schedule' task path fires correctly when no schedule is found for multi-year policies. Backfill the 14 impacted UAT Commission_Transaction__c records via Data Loader after fix deployment.

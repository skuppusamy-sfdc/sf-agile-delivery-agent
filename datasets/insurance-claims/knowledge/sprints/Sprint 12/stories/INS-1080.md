## Description

As a Broker Relations Coordinator, I need a guided onboarding checklist within the Agent Portal for newly recruited agents so that they can complete required steps — including profile completion, E&O insurance upload, state license verification, and DocuSign e-signature of the Broker Agreement — in a single workflow, reducing the average onboarding cycle time from 12 days to under 5 days. The checklist status should be tracked on the Agent__c record and visible to the Broker Relations team in Salesforce.

## Acceptance Criteria

Given a new Agent__c record is created with Agent__c.Status__c = 'Pending_Onboarding',
When the Record-Triggered Flow (AgentOnboarding_ChecklistInit) fires,
Then an Onboarding_Checklist__c record is created and linked to Agent__c.Id with five child Checklist_Item__c records: 'Profile Complete', 'E&O Insurance Upload', 'State License Verification', 'Background Check Consent', 'Broker Agreement Signed' — each with Checklist_Item__c.Status__c = 'Incomplete'.

Given the agent logs into the Agent Portal and navigates to the 'My Onboarding' tab,
When the OmniScript (AgentOnboardingWizard) loads,
Then the current checklist progress is displayed as a step indicator showing completed vs. incomplete items sourced from Checklist_Item__c.Status__c.

Given the agent reaches the 'Broker Agreement Signed' step,
When they click 'Send for Signature',
Then a DocuSign envelope is generated via the DocuSign for Salesforce managed package using the 'Broker_Agreement_Template_v3' template, pre-populated with Agent__c.First_Name__c, Agent__c.Last_Name__c, Agent__c.NPN__c, and Agent__c.TIN__c, and sent to Agent__c.Email__c.

Given DocuSign returns a completed envelope status webhook,
When the Platform Event DocuSign_Status__e is received with Status = 'Completed',
Then Checklist_Item__c.Status__c for 'Broker Agreement Signed' is set to 'Complete', Agent__c.Broker_Agreement_Signed_Date__c is stamped with today's date, and the signed PDF is attached to the Agent__c record as a Salesforce File (ContentDocument).

Given all five Checklist_Item__c records have Status__c = 'Complete',
When the final item is updated,
Then Agent__c.Status__c transitions to 'Active' via a Record-Triggered Flow, a welcome email is sent using the 'Agent_Welcome_Activated' email template, and the Broker Relations Coordinator is notified via a Salesforce Task with Subject = 'New Agent Activated: [Agent Name]'.

-Given state license verification is performed via a manual upload only- (deprecated: automated NIPR license lookup integration added to scope per INS-1071)

Given the agent's state license number is entered in the OmniScript,
When the 'Verify License' action is triggered,
Then an Apex callout to the NIPR API validates the license and populates Agent__c.License_Status__c with the returned verification result ('Active'|'Expired'|'Not Found').

## Solution

Build the onboarding workflow as an OmniScript (AgentOnboardingWizard) with six steps corresponding to checklist items. Checklist initialization handled by a Record-Triggered Flow on Agent__c after-insert. DocuSign envelope generation uses the DocuSign for Salesforce Apex Toolkit (DSFSDocuSignAPICallout) invoked from an Apex Action within the OmniScript. Platform Event (DocuSign_Status__e) is published by the DocuSign webhook handler (DocuSignWebhookHandler.cls) and consumed by a Platform Event-Triggered Flow that updates Checklist_Item__c and Agent__c fields. NIPR license verification is an Apex HTTP callout (NIRPLicenseVerificationService.cls) wrapped as an OmniScript Integration Procedure. All file attachments use ContentVersion insert linked to Agent__c via ContentDocumentLink. Checklist visibility for the Broker Relations team is surfaced via a List View on Onboarding_Checklist__c filtered by Status__c != 'Complete'.

## Description

**Bug Report**

**Summary:** Commission__c.Amount__c is being set to 0.00 when the parent Policy__c.Annual_Premium__c is stored in a non-USD currency (e.g., CAD) in orgs with multi-currency enabled.

**Steps to Reproduce:**
1. Enable multi-currency in a sandbox org (CAD as an active currency).
2. Create a Policy__c record with Policy__c.Annual_Premium__c = 5000 CAD and Policy__c.CurrencyIsoCode = 'CAD'.
3. Set Policy__c.Status__c = 'Active' to trigger the commission calculation Flow (RTF_Policy_Commission_Calculation, built in INS-1067).
4. Navigate to the resulting Commission__c record.
5. Observe Commission__c.Amount__c = 0.00.

**Expected Behavior:** Commission__c.Amount__c should equal Policy__c.Annual_Premium__c (converted to the org's corporate currency, USD) multiplied by Commission__c.Rate__c.

**Actual Behavior:** Commission__c.Amount__c = 0.00 regardless of premium value when CurrencyIsoCode != 'USD'.

**Root Cause (suspected):** The Apex invocable method 'CommissionTierCalculator.calculateRate' uses a formula that references Annual_Premium__c via a raw Decimal field assignment without invoking convertCurrency(), causing the currency conversion to fail silently and return null, which is then cast to 0.00.

**Environment:** Sandbox S3 (multi-currency enabled), API v59.0.

## Acceptance Criteria

As per Description. 

Additional verification: Given Policy__c.CurrencyIsoCode = 'CAD' and Annual_Premium__c = 5000 CAD, When the commission Flow fires, Then Commission__c.Amount__c = (5000 * USD_conversion_rate) * Commission__c.Rate__c and Commission__c.CurrencyIsoCode = 'USD' (corporate currency).

## Solution

Fix 'CommissionTierCalculator.calculateRate' Apex method to use the CurrencyConversionUtil helper class (or SOQL with convertCurrency() function) when retrieving Annual_Premium__c values in multi-currency orgs. Add a null-check guard before the multiplication assignment. Update the corresponding Apex test class to include multi-currency test scenarios using Test.setCurrencyIsoCode() equivalent patterns. Deploy fix to S3 sandbox for QA regression before promoting to UAT.

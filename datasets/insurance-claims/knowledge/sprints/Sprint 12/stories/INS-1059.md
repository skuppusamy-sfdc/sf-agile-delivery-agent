## Description

As an agent using the Agent Portal, I need a guided policy search and bind-quote workflow so that I can submit new business applications on behalf of policyholders without switching to the back-end Salesforce interface.

The workflow must support searching existing policyholders by TIN or SSN, pre-filling policy application fields, and submitting for underwriting review. The OmniScript should enforce state-specific underwriting rules and route to an Underwriting Analyst queue when manual review is required.

## Acceptance Criteria

Given an agent initiates a new business submission from the Agent Portal,
When the OmniScript 'NewBusinessSubmission_OS' launches,
Then Step 1 prompts for policyholder lookup by TIN, SSN, or Policy__c.Policy_Number__c with a SOQL-backed typeahead search against the Provider__c (Person Account) and Account objects.

Given a matching policyholder record is found,
When the agent selects the record,
Then Policy__c fields (Policy__c.Policyholder__c, Policy__c.Effective_Date__c, Policy__c.LOB__c, Policy__c.State__c) are pre-populated via DataRaptor Load.

Given the agent completes the coverage selection step,
When Policy__c.State__c is in ['CA', 'NY', 'FL'],
Then an Apex-invocable method (UnderwritingRulesEngine.cls) evaluates state-specific eligibility rules and returns a PASS or REFER decision stored in Policy__c.UW_Decision__c.

Given Policy__c.UW_Decision__c = 'REFER',
When the OmniScript reaches the submission step,
Then a Case record (Type = 'Underwriting Review') is auto-created and assigned to the 'Underwriting_Queue' via Record-Triggered Flow, and the agent sees an inline notification with the Case number.

Given Policy__c.UW_Decision__c = 'PASS',
When the agent clicks 'Bind Quote',
Then Policy__c.Status__c transitions to 'Active', Policy__c.Bind_Date__c is stamped with TODAY(), and a DocuSign envelope is sent to the policyholder's email via the DocuSign for Salesforce managed package.

Given the agent abandons the OmniScript mid-flow,
When the session is resumed within 24 hours,
Then OmniScript draft state is restored from OmniProcess save point and Policy__c.Status__c remains 'Draft'.

## Solution

Develop a multi-step OmniScript (NewBusinessSubmission_OS) with DataRaptor Turbo Extract for policyholder lookup and DataRaptor Load for field pre-population. Integrate Apex invocable action for underwriting rules engine. Use a Record-Triggered Flow on Policy__c (After Insert, when UW_Decision__c = 'REFER') to create the Case and assign to queue. DocuSign integration via named credential and REST callout in Apex (DocuSignEnvelopeService.cls). Deploy OmniScript to Agent Portal via LWC wrapper component embedded in the Experience Cloud page.

## Description

As a Credentialing Specialist, I need the system to automatically create a Credentialing_Case__c re-credentialing case 90 days before a provider's credentialing expiration date so that no provider falls out of network compliance due to an overlooked renewal. The automation must also trigger a CAQH re-attestation request via the existing DataIntegration layer and notify the assigned Credentialing Specialist.

## Acceptance Criteria

Given a scheduled Apex job runs daily at 6:00 AM,
When it queries Provider__c records where Provider__c.Credentialing_Expiration_Date__c = TODAY() + 90 and Provider__c.Credentialing_Status__c = 'Active',
Then a Credentialing_Case__c is created for each matching provider with: RecordType = 'Re_Credentialing', Status__c = 'Initiated', Priority__c = 'High', Due_Date__c = Provider__c.Credentialing_Expiration_Date__c - 10 days, OwnerId = Provider__c.Assigned_Credentialing_Specialist__c.

Given the Credentialing_Case__c is created,
When the Record-Triggered Flow fires on insert,
Then a Platform Event 'CAQH_Reattestation_Request__e' is published containing Provider__c.CAQH_Provider_Id__c and Credentialing_Case__c.Id for the DataIntegration middleware to process.

Given the Platform Event is published,
When the CAQH integration acknowledges receipt,
Then Credentialing_Case__c.CAQH_Request_Sent__c is set to TRUE and Credentialing_Case__c.CAQH_Request_Date__c = TODAY().

Given the Credentialing_Case__c is created,
When the assigned specialist's email is populated,
Then an automated email alert (via Flow email action) is sent with subject 'Re-Credentialing Required: [Provider Name] — Expires [Date]' and a deep link to the Credentialing_Case__c record.

-Given the job creates duplicate cases if run twice in one day, a deduplication check was not originally included- (fixed: job now checks for existing open Credentialing_Case__c with RecordType = 'Re_Credentialing' and Provider__c match before creating)

Given a provider's Credentialing_Expiration_Date__c is NULL,
When the scheduled job evaluates that provider,
Then no case is created and the provider's Id is logged to a custom object Automation_Error_Log__c with Error_Message__c = 'Credentialing expiration date missing — manual review required'.

## Solution

Scheduled Apex class 'RecredentialingCaseScheduler' implementing Schedulable and Batchable interfaces, batch size 200. SOQL: SELECT Id, Name, Credentialing_Expiration_Date__c, CAQH_Provider_Id__c, Assigned_Credentialing_Specialist__c FROM Account WHERE RecordType.DeveloperName = 'Provider' AND Credentialing_Expiration_Date__c = :Date.today().addDays(90) AND Credentialing_Status__c = 'Active'. Deduplication via sub-query checking existing Credentialing_Case__c. Case insert triggers Record-Triggered Flow that publishes Platform Event 'CAQH_Reattestation_Request__e'. MuleSoft integration (existing CAQH connector) subscribes to the Platform Event and calls CAQH ProView API. Callback updates Credentialing_Case__c via REST API callout. Email alert configured as Flow Send Email action using custom email template 'Recredentialing_Alert'. Automation_Error_Log__c records created via Flow fault path.

## Description

**Bug Report — INS-1077**

The Commission Dashboard (delivered in INS-1074) displays an incorrect YTD total when an agent has commission records spanning more than one LOB. The YTD earnings widget double-counts Commission__c records that have been reclassified from one LOB to another (e.g., 'Auto' → 'Health') due to a mid-term policy amendment.

**Steps to Reproduce:**
1. Log in as an agent user (test user: agent_test_multicob@acmeins.com) with the 'Agent_Portal_User' Permission Set Group.
2. Navigate to the Commission Dashboard tab.
3. Observe the YTD Total widget at the top of the FlexCard.
4. Note the YTD total displayed (e.g., $18,400).
5. Manually sum Commission__c.Earned_Amount__c for all records via SOQL: SELECT SUM(Earned_Amount__c) FROM Commission__c WHERE Agent_Code__c = 'AGT-00234'.
6. SOQL returns $14,200 — dashboard shows $18,400.

**Expected Behavior:** YTD total reflects the accurate sum of Commission__c.Earned_Amount__c for the agent's current LOB assignments without duplication.

**Actual Behavior:** The DataRaptor Extract used by the FlexCard performs a JOIN across both Commission__c and a historical Commission_Audit__c table, causing reclassified records to appear in both LOB buckets and be summed twice in the aggregation formula.

**Environment:** Full Sandbox (AcmeIns-FullSBX2)
**Regression:** Introduced in Sprint 12 deployment on 2024-11-04

## Acceptance Criteria

As per Description. Verify fix by confirming DataRaptor Extract returns deduplicated Commission__c records (by Commission__c.Id) and YTD total matches SOQL aggregate query result for agent AGT-00234 within $0.01 tolerance.

## Solution

Update the DataRaptor Extract (DR_CommissionDashboard_Extract) to remove the JOIN to Commission_Audit__c from the aggregation query. Add a DISTINCT filter on Commission__c.Id within the DataRaptor's JSON path to prevent duplicate row inclusion. Add a unit test in the DataRaptor test harness using agent AGT-00234 fixture data to assert SUM(Earned_Amount__c) = 14200.00. Regression test against INS-1074 acceptance criteria before promoting to QA.

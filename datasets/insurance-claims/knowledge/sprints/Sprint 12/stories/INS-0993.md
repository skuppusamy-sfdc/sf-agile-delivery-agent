## Description

**Bug Report: EOB Generation NullPointerException on Zero-Dollar Secondary Payer**

**Environment:** Acme Insurance Full Sandbox (Sprint 12)
**Severity:** High — blocking EOB generation for all COB claims where secondary payer allowed amount = $0.00

**Steps to Reproduce:**
1. Navigate to an adjudicated Claim__c record where COB_Indicator__c = 'Yes' and at least two Claim_Line__c records exist.
2. Set the secondary Claim_Line__c.Secondary_Payer_Allowed__c = 0.00 on one or more lines.
3. Click the 'Generate EOB' button on the Claim__c record.
4. Observe the error displayed in the toast notification.

**Expected Behavior:**
An Explanation_of_Benefits__c record is created with COB breakdown accurately reflecting $0.00 secondary payer responsibility. The EOB PDF is generated and attached to the Claim__c record.

**Actual Behavior:**
Apex class `EOBGenerationService.generateCOBSummary()` throws a `System.NullPointerException` at line 247. No Explanation_of_Benefits__c record is created. Error logged in Salesforce debug logs: `Attempt to de-reference a null object` — traced to division operation `primaryAllowed / secondaryAllowed` where secondaryAllowed = 0 causes divide-by-zero, not a true null, but the catch block incorrectly surfaces it as NullPointerException due to a missing null-check on the ratio variable.

**Impact:** Approximately 340 COB claims in the adjudicated queue cannot generate EOBs, blocking ERA file transmission to providers.

**Workaround:** Manually set Secondary_Payer_Allowed__c = 0.01 temporarily — NOT acceptable for production.

## Acceptance Criteria

As per Description.

Additional verification: Given a Claim__c with COB_Indicator__c = 'Yes' and at least one Claim_Line__c.Secondary_Payer_Allowed__c = 0.00, When 'Generate EOB' is triggered, Then Explanation_of_Benefits__c is created successfully with COB_Secondary_Amount__c = 0.00 and no exception is thrown.

Given the fix is deployed, When a regression test runs against all 340 impacted claims in sandbox, Then EOB generation succeeds for 100% of records and ERA transmission queue clears.

## Solution

Fix EOBGenerationService.generateCOBSummary() at line 247: add a null/zero guard before the ratio calculation — replace `Decimal ratio = primaryAllowed / secondaryAllowed` with a conditional: `Decimal ratio = (secondaryAllowed != null && secondaryAllowed != 0) ? primaryAllowed / secondaryAllowed : 0;`. Update the catch block to distinguish NullPointerException from ArithmeticException. Add a unit test (EOBGenerationServiceTest) covering zero-dollar secondary payer scenario with minimum 90% code coverage assertion. Deploy via change set to Full Sandbox for QA regression before production push.

## Description

As a Credentialing Specialist, I need the system to automatically send CAQH re-attestation reminder notifications to providers at 90, 60, and 30 days before their CAQH attestation expiry date so that providers complete re-attestation on time, preventing credentialing lapses and potential claim payment holds. The workflow must also create a Credentialing_Case__c record if no re-attestation is detected by the 30-day mark.

## Acceptance Criteria

Given a Provider__c record has Credentialing_Case__c.CAQH_Attestation_Expiry__c populated,
When a scheduled Flow ('CAQHReattestation_Scheduler') runs daily,
Then it identifies all providers whose CAQH_Attestation_Expiry__c falls exactly 90, 60, or 30 days from TODAY().

Given a provider is identified at the 90-day mark,
When the flow processes the record,
Then an email notification is sent to the provider's primary email (Provider__c.PersonEmail) using the 'CAQH_Reatt_90Day' email template,
AND a Task is created on the Provider__c record: Subject = 'CAQH Re-Attestation Due in 90 Days', Status = 'Open', OwnerId = assigned Credentialing Specialist.

Given a provider is identified at the 60-day mark,
When the flow processes the record,
Then a second email is sent using the 'CAQH_Reatt_60Day' template,
AND the existing 90-day Task is updated to Status = 'In Progress' if still open.

Given a provider is identified at the 30-day mark AND Credentialing_Case__c.CAQH_Reatt_Confirmed__c = FALSE,
When the flow processes,
Then a new Credentialing_Case__c is created:
  - Credentialing_Case__c.Case_Type__c = 'CAQH Re-Attestation'
  - Credentialing_Case__c.Status__c = 'Urgent'
  - Credentialing_Case__c.Provider__c = Provider__c.Id
  - Credentialing_Case__c.Due_Date__c = CAQH_Attestation_Expiry__c
AND a Chatter mention is posted to the assigned Credentialing Specialist.

Given the CAQH integration (via DataIntegration component, CAQH REST API) confirms re-attestation completion,
When the inbound webhook updates Credentialing_Case__c.CAQH_Reatt_Confirmed__c = TRUE,
Then all open reminder Tasks on the Provider__c are closed (Status = 'Completed'),
AND the Credentialing_Case__c.Status__c is set to 'Closed — Complete'.

Given a provider's NPI is flagged as inactive in the NPPES integration response,
When the scheduler evaluates the record,
Then the re-attestation workflow is skipped and a separate Compliance_Alert__c record is created with Alert_Type__c = 'Inactive NPI'.

## Solution

Scheduled Flow 'CAQHReattestation_Scheduler' runs daily at 6 AM, using a Get Records element to retrieve Credentialing_Case__c records filtered by CAQH_Attestation_Expiry__c using date formula conditions (TODAY() + 90, +60, +30). Decision elements branch by day-bucket and call sub-flows for each notification tier. Email alerts sent via Send Email action using org-wide address. Credentialing_Case__c creation handled in a separate sub-flow with a Create Records element. CAQH webhook handled by a REST API Apex endpoint (CAQHWebhookHandler) that updates Credentialing_Case__c and fires a Record-Triggered Flow to close tasks. NPPES check reuses the existing DataIntegration.NPPESService Apex class from Sprint 10.

## Description

As a Broker Relations Coordinator, I need a visual onboarding status tracker within the Agent Portal so that newly recruited brokers can monitor the progress of their appointment and contracting steps in real time, reducing manual status-check calls to the Broker Relations team. The tracker should reflect all stages from initial application through NPN verification, E&O submission, DocuSign contract execution, and final appointment activation. Extends INS-0951 (Broker Recruitment Application Form).

## Acceptance Criteria

Given a broker has submitted their recruitment application and a Broker_Application__c record exists with Status__c = 'Submitted',
When the broker logs into the Agent Portal and navigates to 'My Onboarding',
Then a vertical stepper component displays all onboarding stages: Application Submitted → NPN Verified → E&O Received → Contract Sent → Contract Executed → Appointment Active.

Given the Broker_Application__c.NPN_Verification_Status__c = 'Verified' (sourced from NPPES integration),
When the tracker refreshes,
Then the 'NPN Verified' step displays a green checkmark and Broker_Application__c.NPN_Verified_Date__c.

Given a DocuSign envelope has been sent for the broker contract and DocuSign_Envelope__c.Status__c = 'Sent',
When the broker views the 'Contract Sent' step,
Then a 'Sign Now' deep-link button is rendered that opens the DocuSign signing ceremony in a new tab.

Given DocuSign_Envelope__c.Status__c transitions to 'Completed',
When the Record-Triggered Flow on DocuSign_Envelope__c fires,
Then Broker_Application__c.Status__c is updated to 'Contract Executed' and the stepper reflects the completed state within 60 seconds.

Given any onboarding step is in 'Action Required' state (e.g., E&O document missing),
When the broker views that step,
Then a red indicator and inline help text display the specific action required and a file upload widget is available to submit the missing document directly to Broker_Application__c (Files attachment).

## Solution

Implement the onboarding tracker as an LWC progress indicator embedded in the Agent Portal Experience Cloud site. A Record-Triggered Flow on Broker_Application__c (after update) evaluates field changes and updates a JSON-serialized Stage_State__c (Long Text) field that the LWC reads to render stepper state. DocuSign integration uses the existing DocuSign for Salesforce managed package; envelope status callbacks update DocuSign_Envelope__c via webhook and trigger the flow. File uploads use Salesforce Files (ContentDocumentLink) associated to Broker_Application__c. An Apex invocable method handles NPPES callout for NPN verification on demand. Permission Set Group 'Broker_Portal_User' gates access.

## Description

As a Broker Relations Coordinator, I need the system to automatically calculate and create a Commission__c record using tiered rate logic when a Policy__c is moved to 'Active' status so that manual commission entry errors are eliminated and agents are paid accurately based on their tier agreement.

Tiered rates are stored on the Broker_Agreement__c object and vary by LOB and annual premium band.

## Acceptance Criteria

Given a Policy__c record transitions to Policy__c.Status__c = 'Active' via any channel (UI, API, or Flow),
When the Record-Triggered Flow fires on after-update,
Then exactly one Commission__c record is created with Commission__c.Policy__c populated with the triggering Policy__c.Id.

Given the agent has an active Broker_Agreement__c with Broker_Agreement__c.Status__c = 'Active' and Broker_Agreement__c.LOB__c matching Policy__c.LOB__c,
When the commission is calculated,
Then Commission__c.Rate__c is set based on the applicable tier from Broker_Agreement__c.Tier_Schedule__c where Policy__c.Annual_Premium__c falls within the tier's min/max band.

Given no active Broker_Agreement__c exists for the agent at the time of policy bind,
When the Flow attempts commission creation,
Then no Commission__c record is created and a Task is created on the Policy__c record with Subject = 'Manual Commission Review Required'.

Given Commission__c is successfully created,
When the record is saved,
Then Commission__c.Status__c = 'Pending', Commission__c.Amount__c = Policy__c.Annual_Premium__c * Commission__c.Rate__c, and Commission__c.LOB__c mirrors Policy__c.LOB__c.

Given a Policy__c is cancelled within 30 days of bind (Policy__c.Status__c = 'Cancelled' and cancellation date within 30 days),
When the cancellation is processed,
Then Commission__c.Status__c is updated to 'Clawed_Back' and Commission__c.Clawback_Reason__c = 'Early Cancellation'.

## Solution

Implement a Record-Triggered Flow ('RTF_Policy_Commission_Calculation') on Policy__c, triggering on after-update when Status__c changes to 'Active'. The Flow uses a Get Records element to retrieve the matching Broker_Agreement__c and a Decision element to route to tier calculation logic. Tier lookup is handled via an Apex-invocable method ('CommissionTierCalculator.calculateRate') that accepts premium amount and LOB and returns the applicable rate from a Custom Metadata Type ('Commission_Tier__mdt') for maintainability. A second scheduled Flow runs nightly to evaluate clawback eligibility on recently activated policies.

## Description

As a Broker Relations Coordinator, I need the system to automatically calculate and record commission chargebacks when a Policy__c lapses within the contractual clawback window so that broker commission balances are accurately maintained in Payment__c without manual intervention. The chargeback window is defined per GSA (Group_Service_Agreement__c.Chargeback_Window_Days__c) or defaults to 180 days if no GSA is linked.

## Acceptance Criteria

Given a Policy__c.Status__c transitions to 'Lapsed' via any update mechanism,
When the Record-Triggered Flow evaluates the record,
Then the flow checks whether TODAY() - Policy__c.Effective_Date__c <= the applicable chargeback window (from linked Group_Service_Agreement__c.Chargeback_Window_Days__c or default 180).

Given the policy is within the chargeback window,
When the flow proceeds,
Then a new Payment__c record is created with: Payment__c.Type__c = 'Chargeback', Payment__c.Commission_Amount__c = negative of original Payment__c.Commission_Amount__c on the originating payment, Payment__c.Policy__c = lapsed Policy__c.Id, Payment__c.Producer_Code__c copied from original payment, Payment__c.Status__c = 'Pending'.

Given the chargeback Payment__c is created,
When the broker's YTD commission is recalculated on the INS-1043 dashboard,
Then the chargeback amount is subtracted and the updated net total is reflected accurately.

Given the policy lapses outside the chargeback window,
When the flow evaluates,
Then no chargeback Payment__c is created and a Policy__c.Chargeback_Exempt__c checkbox is set to TRUE with a timestamp in Policy__c.Chargeback_Exempt_Reason__c = 'Outside clawback window'.

-Given a manual override field Policy__c.Force_Chargeback__c = TRUE bypasses the window check- (removed — compliance requires window enforcement without override per Compliance Officer review)

Given a chargeback Payment__c is created,
When the Broker Relations Coordinator reviews the record,
Then the Payment__c.Related_Original_Payment__c lookup field references the originating commission payment for full traceability.

## Solution

Record-Triggered Flow on Policy__c (after update, Status__c changed to 'Lapsed'). Flow uses Get Records to retrieve the most recent Payment__c where Type__c = 'Commission' and Policy__c = triggering record Id. Decision element checks chargeback window using formula: {!TODAY} - {!Policy.Effective_Date__c} <= {!GSA.Chargeback_Window_Days__c} (with null-safe default of 180). Create Records element generates chargeback Payment__c. A separate path sets Chargeback_Exempt__c and Chargeback_Exempt_Reason__c for out-of-window cases. Apex invocable method handles null-safe GSA lookup to avoid flow governor limit issues on bulk lapse scenarios (e.g., batch policy terminations). Bulkification handled via Platform Event 'Policy_Lapse_Event__e' published from the flow and consumed by an Apex trigger that processes chargebacks in bulk.

## Description

As a Provider Relations Specialist, I need the Provider 360 FlexCard to display a consolidated view of a provider's active Provider_Network_Agreement__c (PNA) records and associated Fee_Schedule__c details so that I can quickly assess a provider's contracted rates, network participation status, and upcoming renewal dates without navigating to multiple related lists.

This enhancement extends the existing Provider 360 layout built in INS-998 and should surface key PNA fields alongside the credentialing status from Credentialing_Case__c.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider record (Person Account),
When the Provider 360 FlexCard loads,
Then all active PNA records where Provider_Network_Agreement__c.Status__c = 'Active' and Provider_Network_Agreement__c.Provider__c = current record ID are displayed in a child card section.

Given active PNAs are displayed,
When the specialist reviews the PNA section,
Then each row shows Provider_Network_Agreement__c.Network_Name__c, Provider_Network_Agreement__c.Effective_Date__c, Provider_Network_Agreement__c.Termination_Date__c, and Provider_Network_Agreement__c.Fee_Schedule__c (linked).

Given a PNA has an associated Fee_Schedule__c,
When the specialist clicks the fee schedule link,
Then a modal panel opens displaying Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, and a summary count of CPT code line items from Fee_Schedule_Line__c.

Given a PNA is within 90 days of its Termination_Date__c,
When it appears in the FlexCard,
Then it is highlighted with a warning badge labeled 'Renewal Due' and Provider_Network_Agreement__c.Renewal_Status__c is shown.

Given the provider has an open Credentialing_Case__c,
When the FlexCard loads,
Then a credentialing status banner displays Credentialing_Case__c.Status__c and Credentialing_Case__c.Expiration_Date__c at the top of the card.

## Solution

Extend existing Provider_360_FC FlexCard by adding a child DataRaptor Extract (DR_Provider_PNA_Summary) querying Provider_Network_Agreement__c with filter on Status__c = 'Active' and join to Fee_Schedule__c. Add a conditional card state for the 90-day renewal warning using a calculated field in the DataRaptor Transform. Fee schedule modal implemented as a nested FlexCard (Fee_Schedule_Detail_FC) launched via OmniScript action. Credentialing banner driven by a separate DataRaptor querying Credentialing_Case__c ordered by CreatedDate DESC LIMIT 1. No Apex required; all data access via OmniStudio DataRaptors with SOQL relationship traversal.

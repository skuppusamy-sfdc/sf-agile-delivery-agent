- **Labels**: Phase2, KnowledgeTransfer, Hypercare, Runbook

## Description

As a Credentialing Specialist transitioning to the steady-state support model, I need a comprehensive operational runbook documented in Salesforce Knowledge covering the end-to-end Credentialing_Case__c automation workflow — including CAQH data sync triggers, ICD-10/NPI validation steps, automated expiry notifications, and escalation paths — so that the Level 1 and Level 2 support teams can independently troubleshoot and resolve credentialing automation failures without requiring platform team intervention after hypercare ends.

Runbook must be authored as Salesforce Knowledge Articles and linked from the Credentialing_Case__c record page via a Knowledge sidebar component.

## Acceptance Criteria

Given the runbook Knowledge Articles are published,
When a support team member searches Salesforce Knowledge with keyword 'Credentialing Automation',
Then at least 5 articles are returned covering:
  1. CAQH Sync: How the nightly Integration Procedure triggers and updates Credentialing_Case__c.CAQH_Status__c
  2. NPI Validation Failure: Steps to manually re-trigger NPPES callout and interpret DataIntegration_Error__c logs
  3. Credentialing Expiry Workflow: How the scheduled Flow evaluates Credentialing_Case__c.License_Expiry_Date__c and generates Provider_Change_Notification__c records
  4. Escalation Matrix: Decision tree for P1/P2/P3 credentialing automation failures with SLA targets
  5. Rollback Procedure: Steps to revert a Credentialing_Case__c.Status__c from 'Approved' to 'Under Review' without triggering downstream PNA amendments

Given a Credentialing_Case__c record is open,
When the record page is viewed by a user with the 'Credentialing_Support' Permission Set,
Then the Knowledge Sidebar component displays contextually filtered articles based on Credentialing_Case__c.Case_Type__c value.

Given all 5 articles are published,
When a Credentialing Specialist reviews them,
Then each article has been peer-reviewed (Knowledge Article.Validation_Status = 'Validated') and includes a 'Last Reviewed Date' field set to within the current sprint.

## Solution

Author 5 Salesforce Knowledge Articles using the standard Knowledge object (Knowledge__kav) with Article Type = 'Credentialing_Runbook'. Configure the Knowledge Sidebar Lightning component on the Credentialing_Case__c Lightning Record Page with a filter mapping Article__c.Topic__c to Credentialing_Case__c.Case_Type__c. Use a Record-Triggered Flow on Knowledge__kav to auto-assign peer reviewer (Anika Gupta) when Draft status is set. Publish articles via the standard Knowledge publication workflow. Link articles to related Credentialing_Case__c records using the standard KnowledgeArticleVersion association. Ensure articles are indexed for Einstein Search relevance.

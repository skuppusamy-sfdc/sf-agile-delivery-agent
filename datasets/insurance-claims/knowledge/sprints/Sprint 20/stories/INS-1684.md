## Description

As a Credentialing Specialist, I need Salesforce to automatically create a Credentialing_Case__c record and notify me when a Provider__c's CAQH attestation is within 60 days of expiry, so that I can proactively initiate re-credentialing workflows and prevent provider network disruption due to lapsed credentials — a key stabilization requirement identified during Sprint 19 hypercare.

## Acceptance Criteria

Given a Provider__c record where Provider__c.CAQH_Attestation_Expiry_Date__c is within 60 calendar days of today,
When the nightly Scheduled Flow 'CAQH_Expiry_Check_Scheduled_Flow' executes,
Then a Credentialing_Case__c is created with: Case__c.Subject = 'CAQH Re-Attestation Required — [Provider Name]', Case__c.Status__c = 'Initiated', Case__c.Priority__c = 'High', Case__c.RecordType = 'Credentialing_Renewal', and Case__c.Provider__c lookup populated.

Given the Credentialing_Case__c is created,
When the record is saved,
Then a Salesforce in-app notification and email (using the 'Credentialing_Expiry_Alert' email template) is sent to the Provider__c.Assigned_Credentialing_Specialist__c user.

Given a Credentialing_Case__c already exists for the provider with Status__c IN ('Initiated', 'In Review', 'Pending Provider Response'),
When the scheduled flow runs,
Then no duplicate Credentialing_Case__c is created (idempotency check via SOQL on Provider__c lookup + Status__c filter).

-Given CAQH expiry is within 30 days, escalate Case Priority to 'Critical'- (superseded — escalation handled by separate SLA entitlement process on Credentialing_Case__c)

Given Provider__c.CAQH_Attestation_Expiry_Date__c is null or Provider__c.Credentialing_Status__c = 'Exempt',
When the scheduled flow evaluates the record,
Then no Credentialing_Case__c is created and the provider record is logged in a Custom Object 'Flow_Audit_Log__c' with Reason__c = 'CAQH Expiry Check Skipped'.

Given the Credentialing_Case__c is created,
When a Credentialing Specialist opens it,
Then a related Quick Action 'Initiate CAQH Re-Attestation' is visible, which launches the 'CAQH_ReAttestation_OmniScript' pre-populated with Provider__c.CAQH_Provider_ID__c.

## Solution

Implement a Scheduled Flow ('CAQH_Expiry_Check_Scheduled_Flow') running nightly at 02:00 UTC. Flow uses a Get Records element on Provider__c filtered by CAQH_Attestation_Expiry_Date__c <= TODAY+60 AND Credentialing_Status__c != 'Exempt' AND CAQH_Attestation_Expiry_Date__c != null. Loop element checks for existing open Credentialing_Case__c via a nested Get Records (idempotency). Create Records element creates Credentialing_Case__c with RecordType 'Credentialing_Renewal'. Custom Notification sent via Salesforce Notification Builder action. Email alert uses existing Workflow Email Alert 'Credentialing_Expiry_Alert' repurposed as a Flow Send Email action. Flow_Audit_Log__c records created via a separate Create Records element in the 'skip' branch. Quick Action configured on Credentialing_Case__c page layout invoking OmniScript via Lightning Web Component wrapper.

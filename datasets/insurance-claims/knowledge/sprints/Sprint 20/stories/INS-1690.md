## Description

As a Provider Network Specialist on the L2 support team, I need comprehensive runbook documentation for the Provider Network Contracting OmniScript suite so that I can independently troubleshoot, configure, and re-deploy contracting workflows post-hypercare without dependency on the implementation SI team.

## Acceptance Criteria

Given the knowledge transfer documentation is complete,
When an L2 support analyst references the runbook,
Then it covers the following OmniScript components with step-by-step configuration notes:
  - 'OS_PNA_Initiation' (Provider_Network_Agreement__c creation flow)
  - 'OS_GSA_Amendment' (Group_Service_Agreement__c amendment flow)
  - 'OS_FeeSchedule_Assignment' (Fee_Schedule__c assignment to PNA)

Given the runbook is published to the internal Confluence space 'AcmeIns-Salesforce-L2',
When an L2 analyst follows the 'OmniScript Re-Activation' procedure,
Then they can successfully deactivate, clone, modify, and re-activate an OmniScript version in the Full Sandbox without SI assistance.

Given the runbook includes integration touchpoints,
When documenting the Conga CLM integration on OS_PNA_Initiation,
Then the doc specifies the Named Credential 'CongaCLM_NC', the Integration Procedure 'IP_CongaCLM_DocGen', and the Provider_Network_Agreement__c.Contract_Status__c field state transitions ('Draft' → 'Sent for Signature' → 'Executed').

Given the runbook is reviewed,
When submitted for sign-off,
Then it must be approved by the Claims Operations Manager (Sarah Chen) and one Provider Network Specialist before the sprint retrospective.

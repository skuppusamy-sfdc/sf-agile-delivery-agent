## Description

As a Credentialing Specialist, I need the system to automatically detect when a provider's CAQH ProView re-attestation is approaching expiry (within 60, 30, and 7 days) and generate a Provider_Change_Notification__c (PCN) outreach task so that I can ensure continuous credentialing compliance and prevent providers from inadvertently falling out of network due to lapsed attestation, which would trigger unnecessary Network_Termination_Case__c creation.

## Acceptance Criteria

Given a scheduled nightly Apex batch job runs at 02:00 UTC,
When the job evaluates all Credentialing_Case__c records where Status__c = 'Active' and CAQH_Attestation_Expiry_Date__c is not null,
Then for each record where the expiry date is exactly 60, 30, or 7 calendar days from today:
  - A Provider_Change_Notification__c is created with PCN.Type__c = 'CAQH Re-Attestation Reminder', PCN.Provider__c = Credentialing_Case__c.Provider__c, PCN.Due_Date__c = CAQH_Attestation_Expiry_Date__c, and PCN.Days_Until_Expiry__c populated accordingly.
  - A Task is created assigned to the Credentialing_Case__c.Owner with Subject = 'CAQH Re-Attestation Expiring in [X] Days — [Provider Name]' and ActivityDate = today.

Given the PCN is created,
When the Credentialing Specialist opens the PCN record,
Then a 'Send Outreach Email' quick action triggers an email using the 'CAQH_Reattestaton_Reminder' email template, addressed to Provider__c.NPI_Contact_Email__c, with the CAQH ProView re-attestation portal deep link included.

Given a provider completes CAQH re-attestation and the nightly CAQH API sync (via DataIntegration/MuleSoft) updates Credentialing_Case__c.CAQH_Attestation_Expiry_Date__c to a future date beyond 60 days,
When the batch job next runs,
Then any open PCN records of Type__c = 'CAQH Re-Attestation Reminder' for that provider are automatically closed (PCN.Status__c = 'Resolved') and the associated Tasks are marked Complete.

Given the batch job encounters an error processing a specific Credentialing_Case__c record,
When the error is caught,
Then the job continues processing remaining records (stateful batch), the error is logged to a custom object 'Integration_Error_Log__c' with fields: Error_Message__c, Related_Record_Id__c, Job_Name__c = 'CAQH_Expiry_Alert_Batch', and a Platform Event is published to notify the integration monitoring dashboard.

-Given a provider has opted out of email outreach via Provider__c.Email_Opt_Out__c = true, suppress email but still create PCN and Task- (deprecated — opt-out logic now handled globally by the Communication Preference Engine, INS-1661; PCN and Task still created regardless)

## Solution

Implement 'CAQHAttestationExpiryBatch' as a stateful Apex Batch class (Database.Batchable, Database.Stateful) scheduled via Apex Scheduler. Batch queries Credentialing_Case__c with SOQL date literals for 60/30/7-day windows. PCN and Task creation handled in the execute() method with bulkified DML. CAQH API sync is an existing MuleSoft integration that updates Credentialing_Case__c via a REST callout — the resolution logic queries PCN records in the finish() method. Email outreach uses a Salesforce Email Template with merge fields and is sent via Messaging.sendEmail() in the quick action Apex controller. Platform Event 'Integration_Error__e' published on exception. Batch scheduled for 02:00 UTC via System.schedule() in an anonymous Apex deployment script.

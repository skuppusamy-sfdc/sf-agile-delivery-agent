## Description

**Bug Report — EOB PDF Download Blank for Zero-Pay Claim Lines**

**Steps to Reproduce:**
1. Log in to the Policyholder Experience Community Portal as a test policyholder whose Policy__c has at least one Claim__c with a mix of paid and zero-pay Claim_Line__c records (Claim_Line__c.Paid_Amount__c = 0.00 AND Claim_Line__c.Status__c = 'Denied').
2. Navigate to 'My Claims' → select the relevant claim → click 'Download EOB'.
3. Observe the downloaded PDF.

**Expected Behavior:**
The EOB PDF renders all Claim_Line__c records including zero-pay/denied lines, displaying the denial reason code from Claim_Line__c.Denial_Reason_Code__c and the patient responsibility amount from Claim_Line__c.Patient_Responsibility__c.

**Actual Behavior:**
The downloaded PDF is blank (0 KB file) when any Claim_Line__c on the claim has Paid_Amount__c = 0.00. Claims where all lines have Paid_Amount__c > 0 generate correctly. Reproducible in Full Sandbox and Production Community.

**Console Error (Chrome DevTools):**
`LWC [eobDownloadButton]: TypeError: Cannot read properties of null (reading 'paidAmount') at eobPdfRenderer.js:112`

**Frequency:** Affects ~18% of EOB downloads based on Production debug log sampling over 48 hours. Reported by 12 policyholders via Support Case queue.

**Environment:** Production Community (API v58), Full Sandbox confirmed.

## Acceptance Criteria

As per Description.

Given the fix is deployed,
When a policyholder downloads an EOB for a claim containing one or more Claim_Line__c records with Paid_Amount__c = 0.00,
Then the PDF renders all lines including zero-pay lines with Denial_Reason_Code__c and Patient_Responsibility__c values displayed, and the file size is > 0 KB.

## Solution

Root cause: LWC 'eobDownloadButton' passes Claim_Line__c data to eobPdfRenderer.js without null-coalescing Paid_Amount__c before arithmetic. Fix: update eobPdfRenderer.js line 112 to use `const paidAmt = line.paidAmount ?? 0.00;` null-coalescing operator. Also update the Apex controller 'EOBPdfController.cls' getClaimLines() method to explicitly include WHERE clause without filtering out zero-pay lines (current SOQL inadvertently excludes Claim_Line__c where Paid_Amount__c = null via implicit null filter). Add LWC Jest unit test for zero-pay scenario. Deploy via unlocked package version bump. Regression test full EOB download suite in Full Sandbox before Production push.

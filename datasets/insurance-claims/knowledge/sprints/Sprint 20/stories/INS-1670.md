## Description

As a Claims Operations Manager, I need an Agentforce Copilot Action configured in Salesforce that can interpret inbound FNOL intake data (submitted via the policyholder portal or phone transcript), extract key claim attributes using Einstein AI, and auto-draft a Claim__c record with pre-populated fields so that claims intake time is reduced by at least 40% and human adjudicators can focus on validation rather than data entry.

## Acceptance Criteria

Given a FNOL submission payload is received (via OmniScript 'FNOL_Intake_OS' or parsed phone transcript),
When the Agentforce Copilot Action 'DraftClaimFromFNOL' is invoked,
Then a Claim__c record is created in Status__c = 'Draft' with the following fields populated from the AI extraction: Claim__c.Policy__c (matched via Policy__c.Policy_Number__c), Claim__c.Date_of_Loss__c, Claim__c.LOB__c, Claim__c.Incident_Description__c (truncated to 32,000 chars), and Claim__c.FNOL_Channel__c = 'Agentforce'.

Given the AI extraction confidence score for Policy__c match is below 85%,
When the Copilot Action completes,
Then Claim__c.AI_Confidence_Score__c is stamped with the numeric value, Claim__c.Requires_Manual_Review__c = TRUE, and a Task is created and assigned to the Claims intake queue with Subject = 'Low Confidence FNOL — Manual Policy Match Required'.

Given the Claim__c draft is created,
When an adjudicator opens the record,
Then the Agentforce Copilot panel displays a natural-language summary of the FNOL details and suggests next-best actions: 'Assign SIU Review', 'Request Medical Records', or 'Proceed to Adjudication'.

Given the FNOL data contains ICD-10 diagnosis codes in the transcript,
When the Copilot Action processes the payload,
Then Claim_Line__c child records are auto-created with Claim_Line__c.ICD10_Code__c populated and Claim_Line__c.Status__c = 'Pending Review'.

Given the Agentforce action is triggered,
When processing completes in any outcome,
Then an audit record is written to Claim_AI_Audit__c with fields: Claim_AI_Audit__c.Action_Name__c = 'DraftClaimFromFNOL', Claim_AI_Audit__c.Timestamp__c, Claim_AI_Audit__c.Input_Token_Count__c, Claim_AI_Audit__c.Output_Token_Count__c, and Claim_AI_Audit__c.Model_Version__c for compliance traceability.

## Solution

Configure an Agentforce Copilot Action of type 'Apex' invoking the class 'FNOLAgentforceAction' decorated with @InvocableMethod. The Apex class calls the Einstein Generative AI API (via Connect API / aiplatform endpoint) with a structured prompt template stored in a Custom Metadata Type 'AI_Prompt_Template__mdt' (record: FNOL_Extraction_v2). Parse the JSON response to map extracted entities to Claim__c and Claim_Line__c fields using a DataRaptor-style field mapping class. Implement a Policy__c lookup by Policy_Number__c with fuzzy match scoring logic in Apex. Write audit records to Claim_AI_Audit__c via a separate after-insert Apex Trigger to avoid governor limit conflicts. Configure the Copilot Action in the Agentforce Agent Builder, assign to the 'Claims Intake Agent' agent definition, and expose next-best-action suggestions via Einstein Next Best Action strategy referencing a Recommendation object. Ensure all LLM calls pass through the org's AI Audit Policy and Data Mask rules for PII fields (SSN, DOB).

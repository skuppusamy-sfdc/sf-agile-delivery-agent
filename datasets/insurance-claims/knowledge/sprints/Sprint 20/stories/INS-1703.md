## Description

As a Claims Operations Manager, I need the Agentforce Claims AI agent to automatically evaluate incoming Claim__c records at the point of adjudication and surface a prioritised list of fraud indicators — drawn from claim history, provider billing patterns, and ICD-10/CPT code anomalies — so that high-risk claims are routed to the SIU queue without requiring a manual triage step, reducing SIU backlog and improving fraud detection rates.

This story covers the Agentforce topic configuration, the Einstein prompt template, and the Salesforce Flow routing logic. Integration with the external predictive scoring API is handled in INS-1698.

## Acceptance Criteria

Given a Claim__c record where Claim__c.Status__c transitions to 'Pending Adjudication' via a Record-Triggered Flow, When the Agentforce Claims AI topic action is invoked, Then an Einstein Generative AI prompt template evaluates the claim and returns a structured JSON payload containing: fraud_risk_score (0–100), top_three_indicators (array of strings), and recommended_action ('Route to SIU' | 'Flag for Review' | 'Clear').

Given the AI response payload contains fraud_risk_score >= 75 AND recommended_action = 'Route to SIU', When the downstream Flow processes the response, Then Claim__c.SIU_Referral_Flag__c is set to TRUE, Claim__c.SIU_Referral_Reason__c is populated with the concatenated top_three_indicators, Claim__c.Status__c is updated to 'SIU Review', and the record is assigned to the 'SIU_Triage' Omni-Channel queue with a Case priority of 'High'.

Given the AI response payload contains fraud_risk_score BETWEEN 40 AND 74, When the Flow processes the response, Then Claim__c.AI_Review_Flag__c is set to TRUE and a Task is created on the Claim__c record assigned to the owning Claims Examiner with Subject = 'AI Fraud Flag — Manual Review Required' and Due Date = TODAY() + 2 business days.

Given the AI response payload contains fraud_risk_score < 40, When the Flow processes the response, Then no SIU routing occurs, Claim__c.AI_Cleared_Date__c is stamped with the current datetime, and processing continues to standard adjudication.

Given any Claim__c where Claim__c.LOB__c = 'Workers Compensation', When the Agentforce topic is invoked, Then the prompt template variant 'WC_Fraud_Detection_v2' is selected over the default 'General_Fraud_Detection_v1' to apply Workers Comp-specific ICD-10 billing pattern rules.

Given the Agentforce API call fails or times out after 10 seconds, When the invocable action catches the exception, Then Claim__c.AI_Evaluation_Status__c is set to 'Error — Manual Review', a Platform Event (AI_Evaluation_Error__e) is published for monitoring, and the claim is NOT blocked from continuing to adjudication.

Given a Compliance Officer running the 'SIU AI Referral Audit' report, When the report is filtered by date range, Then all Claim__c records with SIU_Referral_Flag__c = TRUE display: Claim__c.Name, fraud_risk_score (stored in Claim__c.AI_Fraud_Score__c), SIU_Referral_Reason__c, and the datetime the referral was created (Claim__c.SIU_Referral_Date__c).

## Solution

Agentforce Agent configured with a custom Topic ('Claims Fraud Triage') and an Einstein Prompt Template ('General_Fraud_Detection_v1') referencing merge fields from Claim__c, related Claim_Line__c CPT codes, and a DataRaptor-aggregated provider billing history summary injected as a context string. A second prompt template variant 'WC_Fraud_Detection_v2' is created for Workers Comp LOB. The Agentforce topic action is exposed as an Apex Invocable Action (AgentforceClaimsFraudEvaluator) called from a Record-Triggered Flow on Claim__c (trigger: Status__c changes to 'Pending Adjudication', runs asynchronously via Flow's 'Run Asynchronously' path to avoid governor limit conflicts). The Invocable Action deserialises the JSON response and returns fraud_risk_score, top_three_indicators, and recommended_action as Flow output variables. Downstream Flow decision elements branch on score thresholds and perform DML updates via Flow Record Update elements. Omni-Channel routing uses a Flow-invoked PendingServiceRouting record insert targeting the SIU_Triage queue. Platform Event AI_Evaluation_Error__e is published via Apex within the catch block. All score and referral fields added to Claim__c as custom fields; field-level security restricted to Claims_Operations and SIU_Analyst Permission Set Groups. Report built in the 'SIU Operations' report folder with row-level sharing enforced via Sharing Rule on Claim__c.

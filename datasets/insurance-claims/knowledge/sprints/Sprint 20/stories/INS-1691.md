## Description

As a Credentialing Specialist, I need an automated process that detects when a provider's CAQH ProView attestation is approaching expiry (within 60 days) and generates a Provider_Change_Notification__c (PCN) record and sends a templated email reminder to the provider so that credentialing lapses that could trigger network termination are proactively prevented.

## Acceptance Criteria

Given a Credentialing_Case__c record exists with Credentialing_Case__c.CAQH_Attestation_Date__c populated,
When a scheduled flow 'SF_CAQH_Expiry_Check' runs daily at 06:00 UTC,
Then all Credentialing_Case__c records where CAQH_Attestation_Date__c <= TODAY() + 60 AND CAQH_Attestation_Date__c >= TODAY() AND Credentialing_Case__c.Status__c = 'Active' are identified.

Given an expiring attestation is detected,
When the scheduled flow processes the record,
Then a Provider_Change_Notification__c is created with:
  - PCN__c.Type__c = 'CAQH Re-Attestation Required'
  - PCN__c.Provider__c = linked Provider__c record Id
  - PCN__c.Effective_Date__c = Credentialing_Case__c.CAQH_Attestation_Date__c
  - PCN__c.Status__c = 'Pending Provider Action'
  - PCN__c.Days_Until_Expiry__c = calculated field value

Given the PCN is created,
When the record-triggered flow 'RTF_PCN_Notification_Email' fires on PCN__c insert where Type__c = 'CAQH Re-Attestation Required',
Then an email is dispatched to Provider__c.Email using template 'CAQH_Reatt_Reminder_60Day_ET' with merge fields for provider name, NPI, and attestation expiry date.

Given a PCN already exists for the provider with Type__c = 'CAQH Re-Attestation Required' and Status__c != 'Closed',
When the scheduled flow runs,
Then no duplicate PCN is created (idempotency check on Provider__c + Type__c + Effective_Date__c).

-Given the 30-day reminder also triggers a second PCN of Type__c = 'CAQH Final Warning', When attestation is still pending at T-30, Then escalate to Credentialing Specialist queue- (deferred to Phase 3 per INS-1712)

Given the process runs in Production,
When the daily batch completes,
Then a summary Chatter post is made on the 'Credentialing Operations' Chatter Group record with counts of PCNs created and providers affected.

## Solution

Build Scheduled Flow 'SF_CAQH_Expiry_Check' with a Get Records element querying Credentialing_Case__c filtered by CAQH_Attestation_Date__c range and Status__c = 'Active'. Loop collection and use Decision element for idempotency check via a Get Records on existing open PCN__c. Create PCN via Create Records element. Platform Event 'PCN_Created__e' published post-insert for downstream audit trail. Record-Triggered Flow 'RTF_PCN_Notification_Email' fires on PCN__c after insert, calls Send Email action with EmailTemplate API name 'CAQH_Reatt_Reminder_60Day_ET'. Schedule flow via Flow Builder schedule trigger at 06:00 UTC. Chatter post via Post to Chatter flow action on the Credentialing_Operations_Group__c record Id stored in Custom Metadata 'Flow_Config__mdt'.

## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces the provider's active Provider_Network_Agreement__c (PNA), associated Fee_Schedule__c rates, NPI verification status from NPPES, and CAQH credentialing profile link, so that I can answer provider inquiries during hypercare without switching between multiple systems.

## Acceptance Criteria

Given a Provider__c record is open and at least one Provider_Network_Agreement__c exists with PNA__c.Status__c = 'Active',
When the Provider 360 FlexCard loads,
Then it displays PNA__c.Contract_Effective_Date__c, PNA__c.Contract_Expiration_Date__c, PNA__c.Network_Tier__c, and PNA__c.Reimbursement_Model__c.

Given the Provider__c record has an associated Fee_Schedule__c via Fee_Schedule__c.Provider__c lookup,
When the FlexCard's 'Fee Schedule' tab is selected,
Then it lists up to 20 CPT codes with Fee_Schedule__c.Allowed_Amount__c and Fee_Schedule__c.Effective_Date__c, sorted by most recently updated.

Given Provider__c.NPI__c is populated,
When the FlexCard loads,
Then a badge displays NPPES verification status retrieved via the DataRaptor Integration action (HTTP GET to NPPES API) with values 'Verified', 'Mismatch', or 'Not Found'.

Given Provider__c.CAQH_Provider_ID__c is not null,
When the FlexCard loads,
Then a hyperlink 'View CAQH Profile' renders, pointing to the CAQH ProView deep-link URL constructed from Provider__c.CAQH_Provider_ID__c.

Given no active PNA exists for the provider,
When the FlexCard loads,
Then the PNA section displays the message 'No Active Network Agreement — contact Provider Network Contracting' in amber highlight.

## Solution

Extend the existing Provider 360 OmniStudio FlexCard ('Provider_360_FC') built in INS-1612. Add two new child card slots: (1) PNA Summary bound to a DataRaptor Turbo Extract on Provider_Network_Agreement__c filtered by Status__c = 'Active' and Provider__c lookup; (2) Fee Schedule tab using a DataRaptor Extract with SOQL on Fee_Schedule__c ordered by LastModifiedDate DESC LIMIT 20. NPPES verification calls the existing Named Credential 'NPPES_REST_API' via a DataRaptor Integration action with response cached in a Salesforce Custom Setting (NPPES_Cache__c) for 24 hours to avoid API throttling. CAQH URL pattern stored in Custom Metadata. FlexCard deployed to Provider Record Page via Lightning App Builder with component visibility rule: Profile IN ('Provider_Relations', 'Network_Contracting', 'Credentialing').

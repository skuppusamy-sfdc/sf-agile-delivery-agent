## Description

**Bug Report — EOB PDF Generation Failure (High Volume Claim Lines)**

**Environment:** Production (Hypercare)

**Steps to Reproduce:**
1. Navigate to any Claim__c record where the count of related Claim_Line__c records exceeds 25.
2. Click the 'Generate EOB' button on the Claim record page.
3. The OmniScript launches the Conga Composer integration call.
4. Observe the result.

**Expected Behavior:** A fully populated Explanation_of_Benefits__c PDF is generated and attached to the Claim__c record as a ContentDocument, with all Claim_Line__c rows rendered in the EOB line-item table, and EOB.Status__c set to 'Generated'.

**Actual Behavior:** Conga Composer returns a 'Template Rendering Error — Row Limit Exceeded' error. The Explanation_of_Benefits__c record is created with Status__c = 'Error' and no PDF is attached. The Claims Adjuster receives a generic toast error: 'EOB generation failed. Please contact support.' No error details are surfaced to the user.

**Frequency:** Reproducible 100% of the time for Claim__c records with Claim_Line__c count > 25. Affects approximately 340 claims in production as of sprint start.

**Business Impact:** ERA files cannot be transmitted to providers for affected claims, blocking payment disbursement and violating the 30-day clean claim payment SLA.

**Root Cause (Preliminary):** The Conga EOB template ('EOB_Standard_Template_v3') uses a static table with a hardcoded 25-row repeat region. Claims with LOB__c = 'Workers Compensation' and 'Facility' claim types regularly exceed this limit.

## Acceptance Criteria

As per Description. Additionally:

Given a Claim__c record with more than 25 related Claim_Line__c records,
When the 'Generate EOB' action is triggered,
Then the Conga template renders all Claim_Line__c rows without truncation and the resulting PDF is attached to the Claim__c record as a ContentDocument.

Given the EOB generation succeeds,
When the process completes,
Then Explanation_of_Benefits__c.Status__c = 'Generated' and Explanation_of_Benefits__c.Line_Item_Count__c equals the exact count of related Claim_Line__c records.

Given EOB generation fails for any reason,
When the error is caught,
Then a descriptive error message is logged to Claim__c.EOB_Error_Detail__c (new field) and surfaced to the adjuster in the toast notification (not a generic message).

## Solution

Update the Conga Composer EOB template ('EOB_Standard_Template_v3') to use a dynamic repeat region with no hardcoded row cap — Conga supports dynamic table rows via the CongaTableStart/CongaTableEnd merge field syntax. Validate fix in full sandbox against Claim__c records with 25, 50, 100, and 200 Claim_Line__c records. Add an Apex try-catch wrapper in the 'EOBGenerationService' class to capture Conga API error responses and write them to the new Claim__c.EOB_Error_Detail__c field (Text 255). Update the OmniScript error handler to display the EOB_Error_Detail__c value in the toast. Deploy a one-time Apex batch job to re-trigger EOB generation for the 340 affected production claims. Regression test ERA transmission end-to-end.

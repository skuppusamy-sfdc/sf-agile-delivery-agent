## Description

As a Provider Network Specialist, I need the system to automatically generate a Provider_Change_Notification__c record and dispatch a DocuSign envelope containing the formal termination notice to the provider's email on file when a Network_Termination_Case__c is approved through the standard approval process so that regulatory 90-day notice requirements are met without manual document preparation and the termination audit trail is maintained in Salesforce.

## Acceptance Criteria

Given a Network_Termination_Case__c record reaches Approval_Status__c = 'Approved' via the 'Provider Termination Approval Process',
When the post-approval automation fires,
Then a Provider_Change_Notification__c record is created with: PCN.Type__c = 'Termination Notice', PCN.Provider__c = Network_Termination_Case__c.Provider__c, PCN.Effective_Termination_Date__c = Network_Termination_Case__c.Requested_Termination_Date__c, PCN.Status__c = 'Pending Delivery', and PCN.Regulatory_Notice_Days__c = 90.

Given the Provider_Change_Notification__c is created,
When the DocuSign envelope is dispatched,
Then the envelope is sent to Provider__c.Email (from the linked Person Account) with the Termination Notice template ('Acme_Provider_Termination_90Day_v3') merged with Provider__c.Name, Provider__c.NPI__c, Provider__c.TIN__c, Network_Termination_Case__c.Termination_Reason__c, and PCN.Effective_Termination_Date__c.

Given the provider signs the DocuSign envelope,
When the DocuSign completed webhook fires,
Then PCN.Status__c updates to 'Delivered and Acknowledged', PCN.DocuSign_Envelope_Id__c is stamped, and Network_Termination_Case__c.Status__c advances to 'Notice Delivered'.

Given the provider does NOT sign within 14 days,
When the scheduled follow-up Flow path executes,
Then a Task is created on the Network_Termination_Case__c assigned to the owning Provider Network Specialist with Subject = 'Termination Notice Unsigned — Follow Up Required' and a second DocuSign reminder is sent automatically.

Given the Network_Termination_Case__c is approved,
When the linked Provider_Network_Agreement__c is evaluated,
Then PNA.Status__c is updated to 'Pending Termination' and PNA.Scheduled_Termination_Date__c = PCN.Effective_Termination_Date__c.

-Given the termination notice is sent via standard email template without DocuSign- (deprecated; DocuSign mandatory per compliance review CR-2024-089).

## Solution

Implement a Record-Triggered Flow on Network_Termination_Case__c (trigger: after update, condition: Approval_Status__c changed to 'Approved') with two immediate actions: (1) Create Provider_Change_Notification__c child record via Create Records element, (2) Call Apex InvocableMethod 'DocuSignEnvelopeDispatcher.sendTerminationNotice()' passing PCN Id and template name from Custom Metadata 'DocuSign_Template_Mapping__mdt'. The Apex class uses the DocuSign eSignature REST API via Named Credential 'DocuSign_Prod_NC' to create and send the envelope with field merge data. Configure a DocuSign Connect webhook pointing to a Salesforce Site endpoint that fires a Platform Event 'DocuSignStatusUpdate__e' on envelope completion; subscribe via a second Record-Triggered Flow on the Platform Event to update PCN and Network_Termination_Case__c. Scheduled Flow path on Provider_Change_Notification__c checks for unsigned envelopes at 14-day mark. Update PNA via a sub-flow invoked from the main termination flow.

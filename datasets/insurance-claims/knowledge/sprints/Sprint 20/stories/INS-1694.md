## Description

As a Broker Relations Coordinator, I need an automated Group Service Agreement (GSA) renewal workflow that triggers 90 days before a GSA expiry date, generates a pre-populated renewal document via DocuSign, routes it to the employer group's signatory and the assigned broker for countersignature, and updates the Group_Service_Agreement__c record status throughout the signing lifecycle so that GSA lapses are eliminated and brokers are kept informed without manual coordinator intervention.

## Acceptance Criteria

Given a Group_Service_Agreement__c record with GSA__c.Expiry_Date__c = TODAY() + 90 and GSA__c.Status__c = 'Active',
When the scheduled flow 'SF_GSA_Renewal_Trigger' runs daily,
Then a renewal sub-process is initiated and GSA__c.Renewal_Status__c transitions from NULL to 'Renewal Initiated'.

Given the renewal is initiated,
When the Integration Procedure 'IP_DocuSign_GSA_Renewal' is invoked,
Then a DocuSign envelope is created using template 'GSA_Renewal_DS_Template' with the following pre-populated merge fields:
  - GSA__c.Account__r.Name (Employer Group Name)
  - GSA__c.Effective_Date__c (new effective date = GSA__c.Expiry_Date__c + 1 day)
  - GSA__c.Annual_Premium__c
  - GSA__c.LOB__c
  - Assigned Broker's Contact Name and Email from GSA__c.Broker__r.Email

Given the DocuSign envelope is sent,
When GSA__c.Renewal_Status__c = 'Renewal Initiated',
Then the envelope is routed first to the employer group signatory (GSA__c.Group_Signatory__c contact) and upon their completion, to the broker (GSA__c.Broker__c contact) for countersignature.

Given the DocuSign envelope status changes,
When a DocuSign Connect webhook payload is received by the Salesforce REST endpoint 'DocuSignWebhookHandler',
Then GSA__c.Renewal_Status__c is updated according to the following state machine:
  - Envelope 'sent' → 'Pending Group Signature'
  - Envelope 'delivered' to broker → 'Pending Broker Countersignature'
  - Envelope 'completed' → 'Renewed'; GSA__c.Status__c = 'Active'; GSA__c.Expiry_Date__c = new expiry date
  - Envelope 'declined' → 'Renewal Declined'; Task created for Broker Relations Coordinator
  - Envelope 'voided' → 'Renewal Voided'; Chatter post on GSA record

Given the envelope is completed,
When GSA__c.Renewal_Status__c transitions to 'Renewed',
Then an automated email notification is sent to the assigned Broker Relations Coordinator and the broker using email template 'GSA_Renewal_Complete_ET' with the executed document attached as a ContentVersion linked to the GSA__c record.

-Given the GSA renewal also triggers a Fee_Schedule__c review task, When Renewal_Status__c = 'Renewed', Then create a child Fee Schedule Review Task__c record- (descoped to Phase 3)

Given a GSA is already in renewal,
When the daily scheduled flow runs,
Then no duplicate renewal envelope is sent if GSA__c.Renewal_Status__c IN ('Renewal Initiated', 'Pending Group Signature', 'Pending Broker Countersignature').

Given the workflow is deployed,
When tested end-to-end in Full Sandbox using DocuSign Developer account,
Then all five envelope state transitions update GSA__c correctly within 60 seconds of webhook receipt.

## Solution

Build Scheduled Flow 'SF_GSA_Renewal_Trigger' querying Group_Service_Agreement__c with Expiry_Date__c filter and idempotency check on Renewal_Status__c. Invoke Integration Procedure 'IP_DocuSign_GSA_Renewal' via Apex action from flow; IP calls DocuSign eSignature REST API v2.1 via Named Credential 'DocuSign_NC' to create envelope from template. Implement Apex REST resource '@RestResource(urlMapping='/docusign/webhook/*')' class 'DocuSignWebhookHandler' to parse Connect JSON payload and update GSA__c via Record-Triggered Flow on Renewal_Status__c changes for downstream notifications. Store DocuSign Template ID in Custom Metadata 'DocuSign_Config__mdt' for environment portability. Attach completed envelope PDF via DocuSign API getDocument() call, store as ContentVersion with ContentDocumentLink to GSA__c. Full test coverage in DocuSignWebhookHandlerTest.cls with mock HTTP callout using StaticResource JSON fixtures for each envelope state.

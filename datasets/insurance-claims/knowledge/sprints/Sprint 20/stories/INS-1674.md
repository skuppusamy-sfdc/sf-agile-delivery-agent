## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard embedded on the Provider__c (Person Account) record page that surfaces the provider's active Provider_Network_Agreement__c (PNA) status, associated Fee_Schedule__c effective dates, NPI, TIN, and current credentialing status from Credentialing_Case__c so that I can answer adjudicator inquiries during hypercare without navigating across multiple objects.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record,
When the Provider 360 FlexCard loads,
Then the following data is displayed in a single card:
  - Provider__c.NPI__c
  - Provider__c.TIN__c
  - Provider__c.Specialty__c
  - Most recent Provider_Network_Agreement__c where PNA.Status__c = 'Active' and PNA.Provider__c = current record Id, showing: PNA.Contract_Effective_Date__c, PNA.Contract_End_Date__c, PNA.Network__c
  - Associated Fee_Schedule__c.Effective_Date__c and Fee_Schedule__c.Schedule_Type__c
  - Most recent Credentialing_Case__c.Status__c linked to the provider

Given a provider has NO active PNA record,
When the FlexCard loads,
Then the PNA section displays a yellow warning badge reading 'No Active Network Agreement' and the Fee Schedule section is suppressed.

Given a provider's Credentialing_Case__c.Status__c = 'Suspended' or 'Revoked',
When the FlexCard loads,
Then a red alert banner reads 'Credentialing Issue — Verify Before Payment' and a link to the Credentialing_Case__c record is shown.

Given the FlexCard is rendered,
When the specialist clicks 'View Full PNA',
Then the user is navigated to the Provider_Network_Agreement__c record in a new subtab.

-Given a 'Send PCN' button is available on the FlexCard- (deprecated — PCN initiation moved to dedicated Provider Termination flow, see INS-1658)

## Solution

Implement as an OmniStudio FlexCard on the Provider__c Account record page using a DataRaptor Turbo Extract to query Provider_Network_Agreement__c, Fee_Schedule__c, and Credentialing_Case__c via related fields. Conditional state badges (green/yellow/red) configured via FlexCard Conditional Visibility rules based on field values. The 'View Full PNA' action uses a Navigation Action element. FlexCard is added to the Provider Record Page via Lightning App Builder with visibility filter limiting to users with 'Provider_Relations' Permission Set. DataRaptor uses a single multi-object extract to minimize SOQL calls.

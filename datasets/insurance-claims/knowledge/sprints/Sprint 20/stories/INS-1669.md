## Description

As a Credentialing Specialist, I need comprehensive internal runbook documentation covering the automated credentialing workflows, CAQH data sync integration, and Credentialing_Case__c lifecycle management so that the Acme Insurance operations team can independently support, troubleshoot, and extend the credentialing platform after the implementation partner's hypercare period ends.

## Acceptance Criteria

Given the knowledge transfer documentation is complete,
When a Credentialing Specialist references the runbook,
Then it contains step-by-step instructions for manually triggering the CAQH re-attestation sync via the 'Sync CAQH Profile' quick action on Credentialing_Case__c, including expected Credentialing_Case__c.CAQH_Last_Sync_Datetime__c update behavior.

Given a Credentialing_Case__c enters Status__c = 'Pending Primary Source Verification',
When the runbook section on automated PSV is reviewed,
Then it documents the scheduled Apex job 'CredentialingPSVBatchJob' (CronExpression, batch size, error handling), the Platform Events fired on completion (CredentialingPSVComplete__e), and the downstream Record-Triggered Flow that advances Status__c to 'PSV Complete'.

Given the CAQH integration endpoint is unreachable,
When the runbook troubleshooting section is consulted,
Then it provides the Named Credential name ('CAQH_Prod_NC'), steps to verify Remote Site Settings, how to inspect Apex debug logs for the 'CredentialingCAQHCallout' class, and escalation contacts at CAQH.

Given the documentation is published,
When it is stored in Salesforce,
Then it exists as a Knowledge__c article with Article_Type__c = 'Runbook', visible to the 'Credentialing_Operations' Public Group, and tagged with Data_Category__c = 'Provider_Credentialing'.

Given a new Credentialing Specialist joins post-hypercare,
When they complete the onboarding checklist,
Then the runbook includes a sandbox walkthrough exercise covering end-to-end Credentialing_Case__c creation through Credentialing_Case__c.Status__c = 'Credentialed' with sample NPI and TIN values.

## Solution

Author documentation collaboratively in Confluence during Sprint 20, then migrate final content into Salesforce Knowledge as a Knowledge__c article via the Salesforce Knowledge import wizard (CSV + HTML body). Create a Knowledge Category hierarchy under 'Provider_Credentialing' data category. Assign the article to the 'Credentialing_Operations' Public Group via Sharing Settings. Include annotated Flow screenshots exported from the Flow Builder for 'CredentialingCaseLifecycleFlow' and 'CAQHSyncOrchestrationFlow'. Record a Loom walkthrough video linked from the Knowledge article body. Validate article visibility using a test user with 'Credentialing_Specialist' Permission Set Group only.

## Description

As a Claims Operations Manager, I need a real-time hypercare dashboard surfacing critical claims processing KPIs — including adjudication throughput, pending FNOL queue depth, average cycle time, and payment error rates — so that I can identify and escalate stabilization issues during the Sprint 20 go-live window without relying on ad-hoc SOQL reports.

## Acceptance Criteria

Given the Claims Operations Manager navigates to the 'Hypercare Command Center' Lightning App Page,
When the page loads,
Then a FlexCard component displays live counts of Claim__c records grouped by Claim__c.Status__c values: 'FNOL Received', 'Under Review', 'Adjudicated', 'Pending Payment', and 'Closed'.

Given Claim__c.Status__c = 'Pending Payment' count exceeds the configurable threshold (default 150),
When the FlexCard refreshes (every 90 seconds via Platform Event Claims_KPI_Refresh__e),
Then the KPI tile border turns red and a bell icon badge increments on the Lightning App nav bar.

Given the Operations Manager clicks the 'Adjudication Cycle Time' tile,
When the drill-down panel opens,
Then it displays average days between Claim__c.FNOL_Received_Date__c and Claim__c.Adjudication_Date__c, segmented by Claim__c.Line_of_Business__c (Medical, Dental, Vision).

Given any Payment__c record enters Status__c = 'Error' within the last 24 hours,
When the dashboard refreshes,
Then the 'Payment Error Rate' tile shows count and percentage of total disbursements, with a link to a filtered Payment__c list view.

-Given a CSV export button is present on each KPI tile- (deprecated — export handled via INS-1650 Reporting Suite)

Given the dashboard is accessed by a user without the 'Claims_Hypercare_Monitor' Permission Set Group,
When they navigate to the page,
Then they see a 'Access Restricted' message and the FlexCards render no data.

## Solution

Build an OmniStudio FlexCard ('Claims_Hypercare_KPI_FC') with five child cards, each bound to a DataRaptor Turbo Extract querying aggregated Claim__c and Payment__c data. A Platform Event (Claims_KPI_Refresh__e) is published every 90 seconds by a Scheduled Apex job (ClaimsKPIPublisherBatch) to trigger FlexCard refresh via LWC event listener. Threshold configuration stored in Custom Metadata Type Claims_KPI_Threshold__mdt. Row-level visibility enforced via Permission Set Group 'Claims_Hypercare_Monitor'. Drill-down panel implemented as an OmniScript embedded in a Lightning Modal. Deploy via unlocked package to Production scratch org post-hypercare validation.

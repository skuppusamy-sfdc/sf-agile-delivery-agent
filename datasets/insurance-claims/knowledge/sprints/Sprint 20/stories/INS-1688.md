## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page that consolidates NPI verification status from NPPES, TIN validation, CAQH attestation date, and active PNA summary so that I can assess provider standing in a single view without switching between external portals and multiple Salesforce tabs. Extends INS-1657.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record,
When the record page loads,
Then the 'Provider 360 Summary' FlexCard is displayed in the right-hand panel showing:
  - Provider__c.NPI__c value with NPPES verification badge (Verified / Unverified) sourced from Provider__c.NPPES_Verification_Status__c
  - Provider__c.TIN__c masked as 'XX-XXXNNNN' with validation status from Provider__c.TIN_Validation_Status__c
  - Most recent Credentialing_Case__c.CAQH_Attestation_Date__c linked to the provider
  - Count of active Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active' and PNA__c.Provider__c = current record Id

Given the CAQH attestation date is older than 365 days,
When the FlexCard renders,
Then the CAQH tile displays a yellow warning icon and the label 'Re-attestation Required'.

Given Provider__c.NPPES_Verification_Status__c = 'Unverified',
When the FlexCard renders,
Then a red badge 'NPI Unverified' is shown and a 'Trigger NPPES Check' quick action button is visible that invokes the NPPES_Integration_Flow.

-Given the FlexCard shows Fee_Schedule__c assignments inline, When clicked, Then a modal lists all fee schedule lines- (moved to INS-1695)

Given the provider has an open Network_Termination_Case__c,
When the FlexCard loads,
Then a banner 'Active Termination Case: [Case Number]' is displayed with a link to the Network_Termination_Case__c record.

## Solution

Extend existing OmniStudio FlexCard 'Provider360_FC' (created in INS-1657) by adding three new card states: NPI_Verified, NPI_Unverified, CAQH_Warning. Use DataRaptor Extract 'DR_Provider360_Summary' with child relationship queries on Credentialing_Case__c and Provider_Network_Agreement__c. Mask TIN via DataRaptor Transform formula. Wire 'Trigger NPPES Check' button to Integration Procedure 'IP_NPPES_Verification' which calls the NPPES REST API via Named Credential 'NPPES_NC'. Deploy updated FlexCard version using OmniStudio version control. Validate with Credentialing Specialist UAT sign-off.

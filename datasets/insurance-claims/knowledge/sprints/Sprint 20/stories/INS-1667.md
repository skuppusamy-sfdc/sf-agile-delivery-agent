## Description

As a Provider Relations Specialist, I need the Provider 360 page (Person Account for Provider__c) to display a consolidated FlexCard showing all active Provider_Network_Agreement__c records, linked Fee_Schedule__c effective dates, and current credentialing status from Credentialing_Case__c so that during hypercare triage calls I can answer provider inquiries without navigating across five separate related lists. Extends INS-1631.

## Acceptance Criteria

Given a Provider__c (Person Account) record is open in the Provider Relations Specialist's console,
When the 'Provider 360 Summary' FlexCard loads,
Then it displays all Provider_Network_Agreement__c records where PNA.Status__c = 'Active' AND PNA.Provider__c = current record Id, showing PNA.Contract_Effective_Date__c, PNA.Contract_Termination_Date__c, PNA.Network_Tier__c, and PNA.TPA__c.

Given an active PNA exists,
When the FlexCard fee schedule section renders,
Then Fee_Schedule__c records linked via Fee_Schedule__c.PNA__c are displayed with Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Allowed_Amount__c, and Fee_Schedule__c.Effective_Date__c, filtered to show only records where Fee_Schedule__c.Effective_Date__c <= TODAY() AND (Fee_Schedule__c.Expiration_Date__c >= TODAY() OR Fee_Schedule__c.Expiration_Date__c = NULL).

Given the Provider__c has an open Credentialing_Case__c,
When the FlexCard credentialing badge renders,
Then it shows Credentialing_Case__c.Status__c, Credentialing_Case__c.CAQH_Profile_Status__c, and Credentialing_Case__c.Primary_Specialty__c with a color-coded indicator (green = 'Credentialed', yellow = 'In Review', red = 'Expired' or 'Suspended').

Given Provider__c.NPI__c is populated,
When the FlexCard header loads,
Then a 'Verify NPI' action button calls the NPPES integration endpoint and displays the returned Provider__c.NPI_Verification_Status__c inline.

-Given the FlexCard shows historical terminated PNAs in the same list- (removed per UX feedback; terminated PNAs moved to collapsible sub-section only).

## Solution

Build OmniStudio FlexCard 'Provider360Summary' with three child card states: PNA List, Fee Schedule Detail (triggered by PNA row click), and Credentialing Badge. DataRaptor Extract 'DR_GetActivePNAsByProvider' uses a SOQL join across Provider_Network_Agreement__c and Fee_Schedule__c with dynamic filter tokens for effective date logic. Credentialing badge uses a separate DataRaptor Extract 'DR_GetCredentialingStatus' querying Credentialing_Case__c. NPPES verification action calls an Apex REST callout wrapped in an OmniStudio Integration Procedure 'IP_VerifyNPI_NPPES'. Deploy FlexCard to the Provider__c Lightning Record Page via a targeted App-Profile-RecordType assignment. Add fields to existing 'ProviderRelations_FieldAccess' Permission Set Group.

## Description

As a Tier-1 Call Center Agent, I need an Agentforce Copilot action embedded in the Service Console that allows me to ask natural-language questions about a policyholder's claim status — such as 'What is the current status of John Smith's most recent claim?' or 'Why is claim CLM-00452 pended?' — so that I can reduce average handle time during hypercare when claim inquiry volume is elevated and adjuster availability is limited. This story builds on the Policyholder 360 foundation established in INS-1631.

## Acceptance Criteria

Given a Call Center Agent has the 'Agentforce_Claims_Copilot' Permission Set assigned and is working in the Service Console,
When they open the Agentforce Copilot panel and type a natural-language claim status query referencing a policyholder name or Claim__c.Name,
Then the Copilot action 'Get Claim Status' resolves the correct Claim__c record and returns a structured response including:
  - Claim__c.Name
  - Claim__c.Status__c
  - Claim__c.LOB__c
  - Claim__c.Date_of_Loss__c
  - Claim__c.Pend_Reason__c (if Status__c = 'Pended')
  - Claim__c.Assigned_Adjuster__c
  - Most recent Payment__c.Payment_Date__c and Payment__c.Amount__c if Status__c = 'Paid'

Given the agent asks about a pended claim,
When the Copilot returns the pend reason,
Then the response also includes a plain-English explanation of the pend reason code sourced from a Custom Metadata Type 'Pend_Reason_Description__mdt' mapped to Claim__c.Pend_Reason__c values.

Given the policyholder has multiple open Claim__c records,
When the Copilot action is invoked with only the policyholder name,
Then the response lists all open claims (Status__c != 'Closed') with a disambiguation prompt asking the agent to specify a claim number.

Given the agent's query references a Claim__c record belonging to a Policy__c not associated with the current Service Console context (i.e., different policyholder),
When the Copilot action executes,
Then it enforces Salesforce sharing rules and returns a 'No accessible claim found' message rather than exposing data outside the agent's permission scope.

Given the Copilot action is invoked,
When the response is generated,
Then the interaction is logged to a 'Copilot_Interaction_Log__c' custom object with fields: Agent__c, Query_Text__c, Resolved_Claim__c, Response_Timestamp__c, and Confidence_Score__c for hypercare monitoring.

## Solution

Configure an Agentforce Copilot Custom Action ('Get_Claim_Status') backed by an Apex invocable method 'ClaimStatusCopilotAction.getClaimStatus()' that accepts a natural-language parsed input (policyholder name or claim number) from the Einstein LLM intent extraction layer. The Apex method queries Claim__c with WITH USER_MODE to enforce sharing. Pend reason plain-English mapping sourced from Custom Metadata Type 'Pend_Reason_Description__mdt' (admin-maintainable). The Copilot Action is registered in the Agentforce Agent Builder with a topic definition scoped to 'Claims Status Inquiry'. Disambiguation logic implemented as a conditional branch in the Copilot Action response template. Copilot_Interaction_Log__c records written asynchronously via a Platform Event to avoid blocking the response. Permission Set 'Agentforce_Claims_Copilot' grants access to the action and the log object. Hypercare monitoring dashboard (INS-1673) includes a tile for daily Copilot interaction volume and average confidence score.

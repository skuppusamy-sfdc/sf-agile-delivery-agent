## Description

**Bug Report — Hypercare P1**

**Summary:** Explanation_of_Benefits__c (EOB) PDF documents fail to generate via the Conga CLM integration when the parent Claim__c record has more than 25 associated Claim_Line__c records. The generation job times out and the EOB__c.Generation_Status__c field is set to 'Error' with a generic message, leaving policyholders without required payment documentation and blocking ERA file transmission.

**Steps to Reproduce:**
1. Identify or create a Claim__c record (e.g., CLM-008841 in Acme Sandbox) with 26 or more Claim_Line__c child records, all in Status__c = 'Adjudicated'.
2. Trigger EOB generation via the 'Generate EOB' quick action on the Claim__c record page.
3. Observe the Conga CLM generation job in the background (monitor via Setup > Apex Jobs).
4. After approximately 45 seconds, the job completes with Status = 'Failed'.
5. Verify Explanation_of_Benefits__c.Generation_Status__c = 'Error' and Explanation_of_Benefits__c.Error_Message__c = 'Conga: Template rendering timeout exceeded'.

**Expected Result:** EOB PDF is generated successfully for claims with any number of Claim_Line__c records and Explanation_of_Benefits__c.Generation_Status__c = 'Generated'.

**Actual Result:** Generation fails with timeout error for claims with > 25 Claim_Line__c records. Affects approximately 47 high-volume institutional claims (facility billing) since go-live.

**Root Cause (Preliminary):** The Conga CLM template ('EOB_Standard_v2') uses a nested repeating section that performs an inline SOQL query per Claim_Line__c row during rendering, causing O(n) query execution and exceeding the 30-second Conga rendering timeout for large claim sets.

**Workaround:** Claims Operations team is manually generating EOBs using a legacy PDF tool for affected claims. This is not sustainable beyond 72 hours.

## Acceptance Criteria

As per Description. Fix verified when Explanation_of_Benefits__c.Generation_Status__c = 'Generated' for Claim__c records with up to 100 Claim_Line__c records, and EOB PDF attachment is present on the EOB__c record within 60 seconds of generation trigger.

## Solution

Optimize the Conga CLM template ('EOB_Standard_v2') by replacing the inline per-row SOQL with a pre-fetched DataRaptor Extract that retrieves all Claim_Line__c records for the parent Claim__c in a single query before template rendering begins. The DataRaptor output is passed as a JSON payload to the Conga template via an Integration Procedure, eliminating per-row queries. Update the Conga template repeating section to iterate over the pre-fetched JSON array. For claims with > 50 Claim_Line__c records, implement asynchronous EOB generation using a Queueable Apex class (EOBGenerationQueueable.cls) invoked by the 'Generate EOB' quick action Flow, with status polling handled by a client-side LWC component that subscribes to a Platform Event (EOB_Generation_Status__e). Update Explanation_of_Benefits__c.Generation_Status__c to 'Processing' during async generation.

## Description

As a Claims Operations Manager, I need a real-time hypercare dashboard surfacing critical claims processing KPIs — including pend rate, auto-adjudication rate, average days-to-pay, and SIU referral volume — so that I can proactively identify post-go-live instability and escalate to the stabilization team before SLA breaches occur. This story extends INS-1643 (Claims Operations Home Page) by adding a dedicated hypercare tab with live alert banners.

## Acceptance Criteria

Given the Claims Operations Manager navigates to the Hypercare Dashboard tab on the Claims Operations Home Page,
When the page loads,
Then a FlexCard renders showing the following KPIs refreshed every 5 minutes:
  - Claim__c records where Status__c = 'Pended' as a percentage of all claims submitted in the last 24 hours
  - Claim__c records where Auto_Adjudicated__c = true as a percentage of total adjudicated claims
  - Average of Claim__c.Days_To_Pay__c for claims closed in the current rolling 7-day window
  - Count of Claim__c records where SIU_Referral__c = true opened in the current business day

Given any KPI breaches a configured threshold (e.g., pend rate > 15%, auto-adjudication rate < 70%),
When the FlexCard refreshes,
Then an alert banner displays in red with the threshold label, current value, and a 'Create Escalation Case' quick action that pre-populates Case.Subject with 'Hypercare KPI Breach — [KPI Name]' and Case.Priority = 'High'.

Given the Operations Manager clicks 'Drill Down' on any KPI tile,
When the modal opens,
Then a DataRaptor-backed list view displays the underlying Claim__c records filtered by the relevant condition, showing fields: Claim__c.Name, Claim__c.Status__c, Claim__c.LOB__c, Claim__c.Assigned_Adjuster__c, Claim__c.Days_To_Pay__c.

-Given a CSV export button is available on each drill-down modal- (deprecated — export handled via standard Salesforce report, see INS-1651)

Given the dashboard is accessed by a user without the 'Claims_Operations_Manager' Permission Set Group,
When they attempt to navigate to the Hypercare tab,
Then they receive a 'You do not have access to this page' message and the tab is hidden via a custom Tab Visibility rule.

## Solution

Build the hypercare dashboard as an OmniStudio FlexCard with child FlexCards for each KPI tile. Each child card uses a DataRaptor Turbo Extract querying Claim__c with SOQL aggregate functions (COUNT, AVG). A Platform Event (KPI_Threshold_Breach__e) is published by a scheduled Apex job (every 5 minutes) when thresholds are crossed; the FlexCard subscribes via Lightning Empathy API to trigger the alert banner without full page reload. The 'Create Escalation Case' quick action is an OmniScript embedded in the FlexCard action. Tab visibility is controlled via a Permission Set Group 'Claims_Operations_Manager_PSG' and a custom Lightning App Page with audience targeting. Thresholds stored in Custom Metadata Type (KPI_Threshold__mdt) for admin configurability.

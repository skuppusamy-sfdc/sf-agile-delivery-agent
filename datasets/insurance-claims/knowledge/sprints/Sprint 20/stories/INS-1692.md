## Description

As a Compliance Officer, I need a comprehensive field-level security (FLS) audit report and remediation plan for all PHI-designated fields across Claim__c, Policy__c, Explanation_of_Benefits__c, and Payment__c objects so that we meet HIPAA minimum-necessary access standards before the hypercare window closes and the platform transitions to steady-state operations.

## Acceptance Criteria

Given the FLS audit is executed,
When the Apex script 'PHI_FLS_Audit.cls' is run in Production,
Then it produces a PHI_Audit_Log__c record for each Profile and Permission Set that has Read or Edit access to any field tagged with PHI_Flag__c = TRUE on the following objects:
  - Claim__c: Diagnosis_Code_ICD10__c, SSN_Last4__c, Member_DOB__c, Treatment_Notes__c
  - Policy__c: Policyholder_SSN__c, Medical_History_Flag__c
  - Explanation_of_Benefits__c: EOB__c.Service_Description__c, EOB__c.Allowed_Amount__c
  - Payment__c: Payment__c.Bank_Account_Last4__c

Given the audit log is generated,
When the Compliance Officer reviews PHI_Audit_Log__c records,
Then any Profile or Permission Set with access beyond the approved matrix (stored in Custom Metadata 'PHI_Access_Matrix__mdt') is flagged with PHI_Audit_Log__c.Violation_Flag__c = TRUE.

Given violations are identified,
When the remediation Change Set 'CS_PHI_FLS_Remediation_v1' is deployed,
Then all flagged FLS violations are resolved and a re-run of the audit script produces zero records with Violation_Flag__c = TRUE.

Given the audit is complete,
When the Compliance Officer generates the compliance report,
Then a PDF report is produced via Conga Composer template 'PHI_FLS_Compliance_Report_CT' and attached to a Compliance_Review__c record with Status__c = 'Passed' or 'Failed'.

Given ongoing monitoring is required,
When any Permission Set or Profile FLS change is made to a PHI-tagged field,
Then a Setup Audit Trail entry is captured and a daily scheduled report 'PHI_FLS_Daily_Monitor_Report' emails the Compliance Officer distribution list.

## Solution

Develop Apex class 'PHI_FLS_Audit.cls' using Schema.DescribeFieldResult and Schema.DescribeSObjectResult to enumerate FLS per Profile/PermissionSet. Cross-reference against Custom Metadata Type 'PHI_Access_Matrix__mdt' rows. Insert PHI_Audit_Log__c records in bulk (Database.insert with allOrNone=false). Build remediation Change Set adjusting FLS on violating profiles via metadata API (FieldPermissions). Configure Conga Composer template for PDF output. Schedule daily report via Salesforce Report subscription on 'PHI_FLS_Violations_Report'. Document remediation steps in the compliance runbook linked to INS-1690.

## Description

**Bug Report — ERA 835 Generation Failure on COB Multi-Line Claims**

**Steps to Reproduce:**
1. Navigate to a Claim__c record where Claim__c.COB_Flag__c = TRUE and the claim has 3 or more Claim_Line__c child records.
2. Ensure at least one Claim_Line__c has Claim_Line__c.COB_Adjustment_Amount__c > 0 and Claim_Line__c.Status__c = 'Adjudicated'.
3. Trigger the 'Generate ERA 835' quick action from the Claim__c record.
4. Monitor the ERA_Generation_Log__c record created by the process.

**Expected Behavior:**
An ERA_835_File__c record is created with Status__c = 'Generated' and all Claim_Line__c records are reflected in the 835 transaction set with accurate COB adjustment loop (CAS segment) values.

**Actual Behavior:**
The ERA_Generation_Log__c record shows Status__c = 'Failed' with Error_Message__c = 'NullPointerException at ERA835Generator.cls line 247: COB_Adjustment_Amount__c null on loop iteration'. Only single-line COB claims and non-COB multi-line claims generate successfully. Affects approximately 340 claims in the Production org as of Sprint 20 day 3.

**Impact:** Payment__c disbursements for affected claims are blocked. Provider__c remittance is delayed. Potential CMS compliance exposure if ERA filing SLA > 30 days is breached.

**Environment:** Production (v47.3.2), also reproducible in Full Sandbox.

## Acceptance Criteria

As per Description.

Additionally:
Given a Claim__c with COB_Flag__c = TRUE and >= 3 Claim_Line__c records where at least one has COB_Adjustment_Amount__c populated,
When the 'Generate ERA 835' action is triggered,
Then ERA_Generation_Log__c.Status__c = 'Generated' and no NullPointerException is thrown.

Given the fix is deployed,
When the batch reprocess job 'ERA835_Reprocess_Batch' is executed against the 340 backlogged claims,
Then all eligible claims produce ERA_835_File__c records and Payment__c.Status__c transitions from 'Blocked' to 'Disbursed' where payment conditions are met.

## Solution

Root cause: ERA835Generator.cls line 247 iterates Claim_Line__c list without null-checking COB_Adjustment_Amount__c before performing decimal arithmetic. Fix: add null-safe guard `cobaAmt = line.COB_Adjustment_Amount__c != null ? line.COB_Adjustment_Amount__c : 0.00;` before CAS segment construction. Add corresponding unit test in ERA835GeneratorTest.cls covering multi-line COB scenarios (target 95% class coverage). Deploy hotfix via Change Set 'CS_ERA835_COB_Fix_v1'. Post-deploy: execute 'ERA835_Reprocess_Batch' in Production with scope 50, targeting Payment__c.Status__c = 'Blocked' records created after Phase 2 go-live. Notify Provider Relations team via Chatter post on affected Provider__c records.

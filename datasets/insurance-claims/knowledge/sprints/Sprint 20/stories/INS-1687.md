## Description

As a Claims Operations Manager, I need a real-time KPI dashboard FlexCard surfaced on the Claims home page so that I can monitor adjudication throughput, pending FNOL volumes, and SLA breach risk during the hypercare stabilization window without navigating to separate report tabs.

## Acceptance Criteria

Given the Claims Operations Manager is logged in with the 'Claims_Ops_Manager' Permission Set Group applied,
When they navigate to the Claims Home page,
Then a FlexCard titled 'Hypercare Claims KPI' is visible as the first component in the page layout.

Given the FlexCard is rendered,
When data is fetched via DataRaptor Extract,
Then the following metrics are displayed with auto-refresh every 5 minutes:
  - Total open Claim__c records where Claim__c.Status__c IN ('FNOL Received', 'Under Review', 'Pending Information')
  - Count of Claim__c records where Claim__c.SLA_Breach_Flag__c = TRUE
  - Average Claim__c.Days_In_Adjudication__c for claims closed in the last 7 days
  - Count of Payment__c records where Payment__c.Status__c = 'Failed' and Payment__c.CreatedDate = LAST_N_DAYS:1

Given a metric value exceeds a threshold,
When Claim__c.SLA_Breach_Flag__c count > 50 OR Payment__c failed count > 10,
Then the relevant KPI tile renders with a red '#FF3B30' background and a toast notification is sent to the assigned Claims Operations Manager via platform event 'KPI_Threshold_Alert__e'.

Given the FlexCard is in view,
When the manager clicks any KPI tile,
Then the FlexCard navigates to a filtered List View of the relevant Claim__c or Payment__c records matching the displayed criteria.

-Given the FlexCard supports drill-down to Claim_Line__c level metrics, When a line-level tile is clicked, Then a child FlexCard renders inline- (superseded by INS-1701 scope)

Given the user does not have the 'Claims_Ops_Manager' PSG,
When they access the Claims Home page,
Then the FlexCard is not rendered and no KPI data is exposed.

## Solution

Build OmniStudio FlexCard 'HypercareClaimsKPI_FC' with six DataRaptor Extract actions sourcing Claim__c, Payment__c aggregates via SOQL. Configure auto-refresh using FlexCard Refresh Action with a 300-second interval. Publish Platform Event KPI_Threshold_Alert__e via an Apex-invocable action triggered from the FlexCard conditional action block. Apply conditional styling using FlexCard State Actions mapped to threshold fields. Restrict visibility via Permission Set Group 'Claims_Ops_Manager' on the Lightning App Page component visibility filter. Deploy via OmniStudio Migration Tool to Production during hypercare go-live window.

## Description

As a Claims Operations Manager, I need a real-time hypercare monitoring dashboard surfacing critical KPIs for Claim__c volume, adjudication cycle times, Payment__c disbursement failures, and EOB generation errors so that I can proactively identify and escalate processing bottlenecks during the Sprint 20 stabilization window without relying on ad-hoc SOQL queries or manual report refreshes.

## Acceptance Criteria

Given the Claims Operations Manager navigates to the 'Hypercare Command Center' Lightning App Page,
When the dashboard loads,
Then a FlexCard component displays the rolling 24-hour count of Claim__c records where Claim__c.Status__c IN ('Submitted','Pending Adjudication') grouped by Claim__c.LOB__c.

Given a Claim__c record has remained in Status__c = 'Pending Adjudication' for more than 48 hours,
When the KPI tile refreshes (every 5 minutes via Platform Event),
Then the record is highlighted in red with a tooltip showing Claim__c.Adjudication_SLA_Breach__c = TRUE and the assigned queue name.

Given Payment__c.Status__c transitions to 'Failed',
When the Platform Event ClaimPaymentFailed__e is fired,
Then the dashboard counter for 'Payment Disbursement Failures (Today)' increments within 60 seconds and displays Payment__c.Failure_Reason__c in a modal drill-down.

Given an EOB generation error is logged (Explanation_of_Benefits__c.Generation_Status__c = 'Error'),
When the dashboard KPI panel renders,
Then the EOB Error tile shows a count broken down by Explanation_of_Benefits__c.Error_Code__c and links to the parent Claim__c record.

Given the Claims Operations Manager clicks 'Export Hypercare Report',
When the export action is triggered,
Then a Conga CLM-generated PDF is emailed to the manager's Salesforce User email within 2 minutes, containing all KPI snapshots timestamped to the second.

-Given the dashboard auto-refreshes every 60 seconds via scheduled Apex- (superseded by Platform Event approach per INS-1648 design review).

## Solution

Build a Lightning App Page ('Hypercare Command Center') using OmniStudio FlexCards consuming DataRaptor Extract definitions that query Claim__c, Payment__c, and Explanation_of_Benefits__c aggregates. Publish a Platform Event (ClaimPaymentFailed__e) from an Apex Trigger on Payment__c after_update where Status__c changes to 'Failed'. Subscribe to the Platform Event via an LWC empApi adapter on the dashboard to push real-time counter updates without page reload. Deploy a Record-Triggered Flow on Claim__c to stamp Claim__c.Adjudication_SLA_Breach__c = TRUE when LastModifiedDate delta exceeds 48 hours using a scheduled path. Conga CLM integration called via Apex InvocableMethod on the export button action. Permission Set Group 'Hypercare_Dashboard_Access' grants FLS to all referenced fields.

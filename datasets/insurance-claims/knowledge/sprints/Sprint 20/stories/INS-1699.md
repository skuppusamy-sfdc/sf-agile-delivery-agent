## Description

As a Claims Operations Manager, I need an Agentforce Copilot Action that allows claims intake staff to submit a natural-language FNOL description (e.g., 'Policyholder John Smith reports a rear-end collision on 10/14/2024 involving vehicle VIN 1HGBH41JXMN109186') and have the system automatically extract key entities, validate the associated Policy__c, and create a draft Claim__c record pre-populated with extracted data, so that FNOL processing time is reduced by at least 30% and manual data entry errors are eliminated during the hypercare-monitored go-live period.

This is a Phase 2 priority tied to the Agentforce pilot cohort (15 claims intake agents). Extends INS-1678 (Agentforce Copilot Base Configuration).

## Acceptance Criteria

Given an intake agent invokes the 'Create FNOL Claim' Copilot Action in the Claims Lightning App,
When the agent types or pastes a natural-language FNOL description,
Then Agentforce extracts the following entities and maps them to Claim__c fields:
  - Policyholder name → matched to Policy__c.Policyholder_Name__c via fuzzy lookup
  - Date of loss → Claim__c.Date_of_Loss__c
  - Incident description → Claim__c.Incident_Description__c (max 32,000 characters)
  - LOB inference (Auto/Property/Health) → Claim__c.LOB__c
  - Vehicle VIN (if Auto LOB) → Claim__c.Vehicle_VIN__c

Given the extracted Policy__c match has a confidence score >= 0.85,
When the draft Claim__c is created,
Then Claim__c.Status__c = 'Draft', Claim__c.FNOL_Source__c = 'Agentforce_Copilot', and a confirmation card is presented to the agent showing all extracted fields for review before final save.

Given the Policy__c match confidence score is < 0.85,
When the extraction completes,
Then the Copilot Action returns a disambiguation prompt listing up to 3 candidate Policy__c records with Policy__c.Policy_Number__c, Policy__c.Effective_Date__c, and Policy__c.LOB__c for agent selection.

Given the FNOL description contains indicators of potential fraud (e.g., keywords matching SIU_Keyword_List__mdt entries),
When the draft Claim__c is saved,
Then Claim__c.SIU_Flag__c = true and a Task is auto-created assigned to the SIU queue with Subject = 'AI-Flagged FNOL — Review Required'.

Given the Copilot Action is executed,
When the interaction completes (success or failure),
Then an Agentforce_Interaction_Log__c record is created capturing: Action_Name__c = 'Create_FNOL_Claim', Confidence_Score__c, Extracted_Entity_Count__c, Outcome__c ('Draft_Created' | 'Disambiguation_Required' | 'Error'), and Processing_Time_ms__c.

-Given the action also auto-assigns an adjudicator via round-robin (deprecated — adjudicator assignment handled by INS-1682 Assignment Coordinator Flow)-

## Solution

Configure an Agentforce Copilot Action ('Create_FNOL_Claim') using the Agentforce Action Builder, backed by an Apex class (FNOLExtractionAction.cls) that calls the Einstein NLP API for named entity recognition. Entity-to-field mapping logic resides in a Custom Metadata Type (FNOL_Entity_Mapping__mdt). Policy__c fuzzy lookup uses a SOSL query with FIND syntax and a custom Apex scoring algorithm for confidence calculation. SIU keyword matching references SIU_Keyword_List__mdt. Draft Claim__c creation uses a before-insert Record-Triggered Flow to enforce field defaulting. Interaction logging uses a Platform Event (Agentforce_Event__e) consumed by a Record-Triggered Flow that creates Agentforce_Interaction_Log__c. Copilot response cards are configured in the Agentforce Builder UI with dynamic field slots.

## Description

As a Claims Operations Manager, I need a real-time hypercare dashboard surfacing critical claims processing KPIs — including adjudication cycle time, pended claim volume, ERA reconciliation failures, and EOB generation errors — so that I can proactively identify and escalate system instability issues during the Phase 2 go-live stabilization window.

This story extends INS-1665 (Claims Operations Lightning App) by adding a dedicated Hypercare tab with live metric tiles and drill-down capability to underlying Claim__c and Payment__c records.

## Acceptance Criteria

Given the Claims Operations Manager navigates to the Hypercare tab in the Claims Operations Lightning App,
When the page loads,
Then a FlexCard-based KPI panel displays the following real-time metrics refreshed every 60 seconds:
  - Total open Claim__c records where Claim__c.Status__c IN ('Pended', 'Under Review')
  - Average Claim__c.Adjudication_Cycle_Days__c for claims closed in the last 24 hours
  - Count of Payment__c records where Payment__c.ERA_Reconciliation_Status__c = 'Failed'
  - Count of Explanation_of_Benefits__c records where EOB__c.Generation_Status__c = 'Error'

Given a KPI tile shows a value exceeding a configured threshold,
When the threshold is breached (e.g., pended claims > 500),
Then the tile renders with a red RAG indicator and a toast notification is dispatched via Platform Event (Hypercare_Alert__e) to subscribed ops team members.

Given the Claims Operations Manager clicks a KPI tile,
When the drill-down panel opens,
Then a related list of up to 200 underlying records is displayed with columns: Record ID, Claim__c.LOB__c, Claim__c.Assigned_Adjudicator__c, Claim__c.Last_Status_Change__c, and a quick-action button to reassign.

-Given the dashboard also displays SIU referral queue depth (deprecated — moved to INS-1701)-

Given the user has the 'Claims_Ops_Manager' Permission Set Group assigned,
When they access the Hypercare tab,
Then they can view all KPIs across all LOBs without record-level sharing restrictions (enforced via Apex without sharing class).

## Solution

Build a LWC-based Hypercare Dashboard composed of multiple FlexCard child components (OmniStudio) for each KPI tile. Use a Platform Event (Hypercare_Alert__e) published by a scheduled Apex job (every 60 seconds via Schedulable interface) that queries aggregated Claim__c, Payment__c, and EOB__c metrics. The scheduled job stores results in a custom metadata type (Hypercare_Metric_Snapshot__mdt) for trend comparison. Drill-down panels use OmniScript with DataRaptor Extract actions to fetch record lists. A Permission Set Group 'Claims_Ops_Manager' grants FLS and object-level access. Threshold configuration stored in Custom Metadata Type (Hypercare_Threshold__mdt) to allow admin-tunable limits without code deployment.

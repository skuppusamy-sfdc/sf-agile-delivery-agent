## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record that surfaces the provider's active Provider_Network_Agreement__c records, associated Fee_Schedule__c line items, and current credentialing status from Credentialing_Case__c so that I can resolve provider inquiries during hypercare without navigating across five separate objects.

Extends INS-1672 (Provider 360 base FlexCard scaffold).

## Acceptance Criteria

Given a Provider__c record where at least one Provider_Network_Agreement__c.Status__c = 'Active', When a Provider Relations Specialist opens the Provider 360 FlexCard, Then the FlexCard displays a child datatable listing all active PNA records with fields: PNA.Contract_Effective_Date__c, PNA.Contract_End_Date__c, PNA.LOB__c, PNA.Network_Tier__c, and PNA.TIN__c.

Given an active PNA record, When the specialist expands the PNA row, Then a nested DataRaptor-driven table renders all linked Fee_Schedule__c records filtered by Fee_Schedule__c.Effective_Date__c <= TODAY() and Fee_Schedule__c.Expiration_Date__c >= TODAY(), displaying CPT code, allowed amount, and modifier.

Given a Provider__c record with an open Credentialing_Case__c where Credentialing_Case__c.Status__c IN ('In Review', 'Pending Primary Source Verification'), When the FlexCard loads, Then a prominent amber badge reading 'Credentialing In Progress' is displayed in the header card band with a deep-link to the Credentialing_Case__c record.

Given a Provider__c with NPI stored in Provider__c.NPI__c, When the FlexCard header renders, Then the NPI value is displayed alongside the provider name and is copyable via a clipboard icon action.

-Given a Provider__c with CAQH profile data, When the FlexCard loads, Then a CAQH attestation date badge is shown — deprecated in favour of dedicated CAQH integration story INS-1689-

Given any Provider__c record regardless of PNA status, When the FlexCard is viewed by a user without the 'Provider_Network_Specialist' Permission Set Group, Then the Fee_Schedule__c nested table is hidden and replaced with 'Access Restricted — contact your administrator'.

## Solution

OmniStudio FlexCard built on the Provider__c (Person Account) object. Primary DataRaptor Extract retrieves Provider__c fields plus a child SOQL for Provider_Network_Agreement__c filtered by Status__c = 'Active'. A second nested DataRaptor Extract is invoked on row-expand action to query Fee_Schedule__c via a junction to PNA, applying date-range filters server-side to avoid large data payloads. Credentialing badge driven by a conditional visibility state action querying Credentialing_Case__c with a SOQL WHERE Provider__c = :recordId AND Status__c IN ('In Review','Pending Primary Source Verification') LIMIT 1. Permission-based hiding implemented via a Custom Permission ('View_Fee_Schedule_Details') checked in FlexCard state using a Set Values action on load; fee schedule card child is conditionally rendered against that state boolean. Clipboard copy implemented as a client-side OmniStudio custom action. Deployed via unlocked package; Permission Set Group 'Provider_Network_Specialist_PSG' grants the custom permission.

## Description

As a Provider Network Specialist, I need the system to automatically validate and link the correct Fee_Schedule__c record to a Provider_Network_Agreement__c when DocuSign returns a completed envelope status, so that claims adjudication always references a fee schedule whose effective date aligns with the PNA contract start date — preventing underpayment or overpayment errors at adjudication time.

Currently, fee schedule association is a manual step performed post-contract execution, leading to a known gap window where claims are adjudicated without a valid fee schedule reference.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record has PNA.DocuSign_Envelope_Status__c = 'Sent' and a DocuSign completed webhook is received,
When the Platform Event 'DocuSign_Envelope_Completed__e' fires with the matching envelope ID,
Then PNA.Status__c is updated to 'Executed' and PNA.Execution_Date__c is set to the envelope completion timestamp.

Given PNA.Status__c transitions to 'Executed',
When the fee schedule validation logic runs,
Then the system queries Fee_Schedule__c records where Fee_Schedule__c.LOB__c = PNA.LOB__c AND Fee_Schedule__c.Network__c = PNA.Network_Name__c AND Fee_Schedule__c.Effective_Date__c <= PNA.Contract_Start_Date__c AND (Fee_Schedule__c.Expiration_Date__c >= PNA.Contract_Start_Date__c OR Fee_Schedule__c.Expiration_Date__c = NULL).

Given exactly one matching Fee_Schedule__c is found,
When the validation passes,
Then PNA.Fee_Schedule__c (lookup field) is populated with the matching record ID and a Chatter post is added to the PNA record: 'Fee Schedule [Fee_Schedule__c.Name] automatically linked upon contract execution.'

Given zero matching Fee_Schedule__c records are found,
When the validation fails,
Then PNA.Fee_Schedule_Validation_Status__c = 'No Match Found', a Task is created assigned to the owning Provider Network Specialist with Subject = 'Manual Fee Schedule Assignment Required — PNA [PNA.Name]' and Due Date = TODAY + 2 business days, and PNA.Status__c is NOT changed to 'Active'.

Given more than one matching Fee_Schedule__c record is found,
When an ambiguous match condition is detected,
Then PNA.Fee_Schedule_Validation_Status__c = 'Multiple Matches — Review Required' and the same Task escalation logic from the zero-match scenario applies.

-Given the PNA is for LOB = 'Dental', skip fee schedule validation and auto-link the default dental fee schedule.- (Dental LOB out of scope for Phase 2; deferred to Phase 3)

## Solution

Implement a Platform Event subscriber using a Record-Triggered Flow on DocuSign_Envelope_Completed__e filtered to object type = 'Provider_Network_Agreement__c'. The flow invokes an Apex invocable method (FeeScheduleValidationService.validateAndLink) that performs the SOQL match logic with bulkification support. The Apex class uses a custom metadata type (Fee_Schedule_Match_Config__mdt) to externalize the matching field mappings, allowing network ops to adjust matching rules without code changes. Task creation for exceptions uses a Flow subflow (Create_Specialist_Task_Subflow) shared with other network contracting automation. All outcomes are logged to a custom object (PNA_Automation_Log__c) for audit trail. Unit test coverage target: 90%.

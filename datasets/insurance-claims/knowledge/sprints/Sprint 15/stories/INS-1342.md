## Description

**Bug Report — UAT Cycle 3, Sprint 15**

**Summary:** When a Fee_Schedule__c record's Allowed_Amount__c is updated for a specific CPT code while a related Claim__c is in 'In Adjudication' status, the child Claim_Line__c.Allowed_Amount__c values are not recalculated to reflect the updated fee schedule. The stale allowed amount is carried through to the EOB and Payment__c, resulting in incorrect payment disbursements.

**Steps to Reproduce:**
1. Identify a Claim__c record in Status__c = 'In Adjudication' with at least one Claim_Line__c record where Claim_Line__c.CPT_Code__c = '99213'.
2. Navigate to the linked Fee_Schedule__c record for CPT code '99213' under the provider's active PNA.
3. Update Fee_Schedule__c.Allowed_Amount__c from $85.00 to $92.50 and save.
4. Return to the Claim__c record and manually trigger adjudication completion (set Status__c = 'Adjudicated').
5. Inspect the child Claim_Line__c.Allowed_Amount__c value.

**Expected Result:** Claim_Line__c.Allowed_Amount__c = $92.50 (reflecting the updated Fee_Schedule__c value at time of adjudication).

**Actual Result:** Claim_Line__c.Allowed_Amount__c = $85.00 (retaining the stale value from when the claim line was first created).

**Impact:** Affects any claim in 'In Adjudication' status when a fee schedule update occurs concurrently. UAT identified 3 test claims with incorrect allowed amounts. In production, fee schedule updates are batched monthly, so risk is highest at month-end processing cycles.

**Root Cause Hypothesis:** The FeeScheduleLookupService Apex class caches the Fee_Schedule__c allowed amount at Claim_Line__c creation time and does not re-query the fee schedule at adjudication time.

## Acceptance Criteria

As per Description. Verify that at the point Claim__c.Status__c transitions to 'Adjudicated', each Claim_Line__c.Allowed_Amount__c is re-evaluated against the current Fee_Schedule__c.Allowed_Amount__c for the matching CPT_Code__c and PNA. Verify that if no matching Fee_Schedule__c record exists at adjudication time, Claim_Line__c.Allowed_Amount__c is set to $0.00 and Claim_Line__c.Adjudication_Note__c = 'No active fee schedule found for CPT [code] at adjudication date'.

## Solution

Fix FeeScheduleLookupService.applyAllowedAmounts() to perform a fresh SOQL query against Fee_Schedule__c at adjudication time rather than relying on the value stored at claim line creation. Modify the Record-Triggered Flow on Claim__c (adjudication transition) to call the updated Apex @InvocableMethod post-status-change. Add a SOQL query filter: Fee_Schedule__c.CPT_Code__c = Claim_Line__c.CPT_Code__c AND Fee_Schedule__c.PNA__c = Claim__c.PNA__c AND Fee_Schedule__c.Effective_Date__c <= adjudication date. Add unit test for concurrent fee schedule update scenario and bulk adjudication (200 claims, 10 lines each).

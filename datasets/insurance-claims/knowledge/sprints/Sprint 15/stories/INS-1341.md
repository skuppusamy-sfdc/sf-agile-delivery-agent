## Description

As a Provider Network Specialist, I need a guided OmniScript to initiate a Network_Termination_Case__c for a provider, which enforces state-specific regulatory notice period requirements (e.g., 60-day or 90-day notice depending on state and LOB), validates that all open Claim__c records for the provider are resolved or reassigned before termination is finalized, and generates a Provider_Change_Notification__c (PCN) for member notification, so that Acme Insurance remains compliant with CMS and state DOI termination regulations.

This story also delivers the 'Network History' tab on the Provider 360 FlexCard referenced in INS-1337.

## Acceptance Criteria

Given a Provider Network Specialist initiates a termination via the OmniScript on a Provider__c record,
When the OmniScript Step 1 loads,
Then the specialist must select a Termination_Reason__c from the picklist ('Voluntary', 'Credentialing Failure', 'Contract Breach', 'Retirement', 'Fraud/SIU Referral') and input a requested Termination_Date__c.

Given the Termination_Date__c is entered and the provider's primary state is determined from Provider__c.Primary_State__c,
When the OmniScript validates the date,
Then the system checks the Notice_Period_Rule__c custom metadata record for the matching State + LOB combination and blocks submission if (Termination_Date__c - TODAY()) < the required notice period in days, displaying an error message: 'Termination date must be at least [X] days from today per [State] regulatory requirements.'

Given Termination_Reason__c = 'Fraud/SIU Referral',
When the specialist submits the OmniScript,
Then the required notice period validation is bypassed, Network_Termination_Case__c.Expedited_Termination__c = true, and a high-priority Task is auto-created for the SIU queue.

Given the OmniScript validation passes,
When the Network_Termination_Case__c is created,
Then NTC.Status__c = 'Pending Approval', NTC.Provider__c = the initiating Provider__c Id, NTC.Effective_Termination_Date__c = the validated Termination_Date__c, NTC.Termination_Reason__c is populated, and all active Provider_Network_Agreement__c records for the provider have PNA.Pending_Termination__c = true.

Given open Claim__c records exist where Claim__c.Rendering_Provider__c = the terminating provider and Claim__c.Status__c NOT IN ('Adjudicated', 'Closed', 'Denied'),
When the OmniScript Step 3 (Claims Review) loads,
Then a DataTable displays all open claims with Claim__c.Claim_Number__c, Claim__c.Status__c, and Claim__c.Total_Billed_Amount__c, and the specialist must mark each as 'Acknowledged' before proceeding.

Given the Network_Termination_Case__c is approved (NTC.Status__c = 'Approved'),
When the approval process completes,
Then a Provider_Change_Notification__c record is created with PCN.Notification_Type__c = 'Provider Termination', PCN.Affected_Members_Count__c populated from a SOQL aggregate of active Policy__c records linked to the provider's panel, and PCN.Regulatory_Deadline__c = NTC.Effective_Termination_Date__c - 30 days.

Given the 'Network History' tab requirement from INS-1337,
When a Provider Relations Specialist views the Provider 360 FlexCard Network History tab,
Then all Network_Termination_Case__c records for the provider are displayed with NTC.Effective_Termination_Date__c, NTC.Termination_Reason__c, and NTC.Status__c.

## Solution

OmniScript 'Provider_Network_Termination_OS' built in OmniStudio with 4 steps: (1) Termination Details, (2) Regulatory Validation, (3) Open Claims Review, (4) Confirmation & Submit. Notice period rules stored in custom metadata type 'Notice_Period_Rule__mdt' keyed by State__c + LOB__c. DataRaptor Turbo Extract used for open claims lookup in Step 3. Integration Procedure 'CreateTerminationCase_IP' handles Network_Termination_Case__c creation, PNA bulk update, and PCN generation in a single transaction. Approval process on Network_Termination_Case__c uses a 2-step approval (Provider Network Manager → Compliance Officer). Network History FlexCard child card added to the Provider 360 FlexCard (INS-1337) as a new tab action. Custom metadata deployed via unlocked package.

## Description

As a Policyholder (insured member), I need a consolidated dashboard on the member-facing Experience Cloud portal that shows my active claims, recent payment history, and allows me to download my EOB documents so that I can self-serve common inquiries without calling the claims center, reducing inbound call volume by an estimated 15%.

This story covers the Experience Cloud LWC components and the underlying Apex data services with appropriate record-level security to ensure members only see their own data.

## Acceptance Criteria

Given an authenticated Experience Cloud user with a linked Policy__c record (via Policy__c.Policyholder_Contact__c = User.ContactId),
When the 'My Claims' dashboard component loads,
Then only Claim__c records where Claim__c.Policy__c is in the member's active policy set AND Claim__c.Status__c != 'Draft' are displayed, showing: Claim__c.Claim_Number__c, Claim__c.Date_of_Service__c, Claim__c.Status__c, and Claim__c.Total_Billed_Amount__c.

Given the member has Claim__c records in multiple statuses,
When the claims list renders,
Then claims are sorted by Claim__c.Date_of_Service__c descending, with a status filter pill allowing the member to filter by 'In Process', 'Adjudicated', 'Denied'.

Given the member navigates to the 'Payment History' tab,
When the component loads,
Then Payment__c records where Payment__c.Policy__c is in the member's policy set AND Payment__c.Status__c = 'Disbursed' are shown for the last 24 months, displaying Payment__c.Payment_Date__c, Payment__c.Amount__c, Payment__c.Payment_Method__c, and Payment__c.ERA_Reference__c.

Given the member has associated Explanation_of_Benefits__c records,
When the 'My EOBs' section renders,
Then each EOB is listed with EOB__c.EOB_Date__c, EOB__c.Claim__c (claim number hyperlink), and a 'Download PDF' button that retrieves the ContentDocument linked to the EOB record via ContentDocumentLink.

Given a member attempts to access a Claim__c or EOB record belonging to a different policyholder,
When the data service Apex method is invoked,
Then the method returns an empty list and logs a Security_Audit__c record with fields: User__c, Attempted_Record_Id__c, Access_Type__c = 'Unauthorized Portal Access', Timestamp__c = NOW().

Given the portal page loads on a mobile device (viewport < 768px),
When the dashboard renders,
Then all three sections (Claims, Payments, EOBs) are displayed in a single-column stacked layout using responsive SLDS grid classes.

## Solution

Three LWC components: 'memberClaimsDashboard', 'memberPaymentHistory', 'memberEOBDownload' composed within an Experience Cloud Flexipage. Each component calls a corresponding Apex @AuraEnabled(cacheable=true) method in 'MemberPortalDataService' class. All SOQL queries enforce record-level security using WITH USER_MODE to leverage the Experience Cloud guest/authenticated user's sharing rules. A dedicated Guest User Profile is configured with read-only access to Claim__c, Payment__c, EOB__c, and ContentDocumentLink via a custom Permission Set 'Member_Portal_Read'. Security audit logging implemented in a separate @future method to avoid DML-in-loop issues. EOB PDF download uses a Lightning File Download URL constructed from ContentDocument.Id. Responsive layout uses SLDS utility classes (slds-grid slds-wrap slds-col) with custom CSS breakpoints in the LWC component CSS file.

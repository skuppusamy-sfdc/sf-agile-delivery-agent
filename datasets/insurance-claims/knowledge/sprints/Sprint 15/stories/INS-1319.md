## Description

As a Compliance Officer, I need an automated Network_Termination_Case__c workflow that enforces state-mandated regulatory notice periods before a provider's PNA termination becomes effective so that Acme Insurance remains compliant with state notice requirements (ranging from 30 to 90 days depending on LOB and state) and avoids regulatory penalties.

The workflow must gate the PNA status transition, generate the required termination notice document via Conga CLM, and notify all downstream stakeholders including the provider, the Assignment Coordinator, and any active Group_Service_Agreement__c holders.

## Acceptance Criteria

Given a Network_Termination_Case__c record is created with Termination_Case__c.Termination_Reason__c populated and linked to a Provider_Network_Agreement__c,
When the case is submitted for approval,
Then a Record-Triggered Flow calculates Termination_Case__c.Required_Notice_Days__c by looking up a custom metadata type 'State_Notice_Requirement__mdt' keyed on PNA__c.State__c and PNA__c.LOB__c, and sets Termination_Case__c.Earliest_Effective_Termination_Date__c = TODAY() + Required_Notice_Days__c.

Given Termination_Case__c.Earliest_Effective_Termination_Date__c is calculated,
When a user attempts to set PNA__c.Termination_Date__c to a date earlier than Termination_Case__c.Earliest_Effective_Termination_Date__c,
Then a Validation Rule on Provider_Network_Agreement__c blocks the save with the error: 'Termination date cannot be earlier than the required regulatory notice period end date of [Earliest_Effective_Termination_Date__c]. Please update the termination date or contact Compliance.'

Given the Network_Termination_Case__c is approved by the Compliance Officer in the Approval Process 'Network_Termination_Approval',
When the final approval step is completed,
Then a Conga CLM merge is triggered via Apex to generate the 'Provider Termination Notice' document using template 'TMPL_PROV_TERM_NOTICE', merged with Provider__c, PNA__c, and Termination_Case__c data, and attached to the Termination_Case__c as a ContentDocument.

Given the termination notice document is generated,
When the document is attached,
Then an automated email is sent to Provider__c.Email (primary contact) and Provider__c.Secondary_Email__c (if populated) using Email Template 'Provider_Termination_Notice_Email', including the attached PDF.

Given active Group_Service_Agreement__c records exist where GSA__c.Primary_Provider__c = the terminating provider,
When the termination case is approved,
Then a Task is created for each linked GSA assigned to the Broker Relations Coordinator with Subject = 'GSA Impact Review — Provider Termination Pending' and Due Date = Termination_Case__c.Earliest_Effective_Termination_Date__c - 10 days.

-Given the provider has open Claim__c records in 'In Adjudication' status, auto-reassign to alternate in-network provider-
(Deprecated — auto-reassignment of open claims out of scope per legal review LGL-2024-047; handled by Assignment Coordinator manually)

Given Termination_Case__c.Termination_Date__c arrives,
When a scheduled Apex job 'NetworkTerminationBatchJob' runs nightly,
Then all PNA__c records linked to approved termination cases where PNA__c.Termination_Date__c = TODAY() are updated: PNA__c.Status__c = 'Terminated', PNA__c.Active__c = FALSE.

## Solution

Record-Triggered Flow 'Calculate_Notice_Period' fires on Network_Termination_Case__c insert/update when Termination_Reason__c is populated. Flow queries 'State_Notice_Requirement__mdt' via Get Records element and sets Required_Notice_Days__c and Earliest_Effective_Termination_Date__c. Approval Process 'Network_Termination_Approval' configured with two steps: (1) Provider Network Specialist, (2) Compliance Officer. Final approval step invokes Apex invocable method 'CongaCLMMergeService.triggerTerminationNotice(caseId)' which calls the Conga CLM REST API via Named Credential. Email alerts configured in Approval Process using a custom Email Template with Visualforce merge fields. GSA Task creation handled in a separate After-Save Flow triggered on Termination_Case__c approval status change. Scheduled Apex 'NetworkTerminationBatchJob' implemented as Database.Batchable with batch size 50, scheduled via System.scheduleBatch() at 11:00 PM nightly. Validation Rule uses BLANKVALUE and DATE comparison formula.

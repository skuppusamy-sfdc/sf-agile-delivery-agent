## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces the provider's active Provider_Network_Agreement__c, associated Fee_Schedule__c effective dates, and contracted LOB summary, so that I can resolve provider inquiries without navigating across multiple related lists.

## Acceptance Criteria

Given a Provider__c record with at least one Provider_Network_Agreement__c where PNA__c.Status__c = 'Active',
When the Provider 360 FlexCard loads,
Then the card must display PNA__c.Contract_Effective_Date__c, PNA__c.Contract_Termination_Date__c, PNA__c.Network_Tier__c, and the related Fee_Schedule__c.Schedule_Name__c and Fee_Schedule__c.Effective_Date__c.

Given a provider with multiple active PNA records across different LOBs,
When the FlexCard renders,
Then each PNA must appear as a distinct card row grouped by PNA__c.Line_of_Business__c, sorted ascending by PNA__c.Contract_Effective_Date__c.

Given a Provider__c record with no active PNA,
When the FlexCard loads,
Then an empty-state message 'No active network agreements on file' must display and a 'Initiate Contracting' quick action button must be visible.

Given a Provider Relations Specialist with the 'Provider_Network_Read' Permission Set Group,
When they view the FlexCard,
Then Fee_Schedule__c.Rate_Amount__c fields must be masked as '***' unless the user also holds the 'Fee_Schedule_Viewer' Permission Set.

-FlexCard must also display Credentialing_Case__c.Expiry_Date__c inline — moved to INS-1355 per scope review-

## Solution

Build an OmniStudio FlexCard (ProviderNetworkSummaryCard) backed by a DataRaptor Extract that joins Provider__c, Provider_Network_Agreement__c, and Fee_Schedule__c using SOQL with a WHERE PNA__c.Status__c = 'Active' filter. A conditional state on the FlexCard handles the empty-state rendering. Field-level masking for Fee_Schedule__c.Rate_Amount__c is enforced via a DataRaptor transform that checks the running user's Permission Set membership using a custom Apex REST action. Deploy FlexCard to the Provider__c Lightning Record Page via App Builder with audience targeting for the Provider Relations role.

## Description

As a Provider Relations Specialist, I need a consolidated FlexCard on the Provider 360 page that displays a rolling 12-month adjudication summary for all Claim_Line__c records associated with a given Provider__c (Person Account), so that I can quickly assess claim volume, denial rates, and payment trends without navigating to multiple related lists.

## Acceptance Criteria

Given a Provider__c record is open in the Provider 360 OmniStudio layout,
When the 'Claim Adjudication Summary' FlexCard loads,
Then it must display aggregated Claim_Line__c data including: total billed amount (sum of Claim_Line__c.Billed_Amount__c), total allowed amount (sum of Claim_Line__c.Allowed_Amount__c), total paid amount (sum of Claim_Line__c.Paid_Amount__c), denial count (count where Claim_Line__c.Status__c = 'Denied'), and overall denial rate as a percentage.

Given the FlexCard is rendered,
When the rolling window is calculated,
Then only Claim_Line__c records where Claim_Line__c.Service_Date__c >= TODAY() - 365 days are included in the aggregation.

Given a Provider__c record has zero associated claims in the last 12 months,
When the FlexCard loads,
Then a 'No claims found in the last 12 months' empty-state message is displayed and all numeric fields show '--'.

Given the FlexCard is displayed,
When a Provider Relations Specialist clicks the 'View All Claim Lines' action button,
Then a filtered list view of Claim_Line__c records scoped to that Provider__c opens in a modal.

-Given the FlexCard includes a sparkline trend chart for monthly paid amounts (deprecated — deferred to INS-1401)-

Given the page is accessed by a user without the 'Provider_360_Full_Access' Permission Set Group,
When the FlexCard attempts to load,
Then the adjudication summary section is hidden and no Claim_Line__c data is exposed.

## Solution

Build an OmniStudio FlexCard ('Provider_Adjudication_Summary_FC') driven by a DataRaptor Turbo Extract that aggregates Claim_Line__c fields via a SOQL GROUP BY on Provider__c lookup. The DataRaptor uses a date filter formula for the rolling 12-month window. Empty-state handling configured via FlexCard conditional visibility conditions. The 'View All Claim Lines' action invokes an OmniScript modal passing Provider__c.Id as an input parameter. Visibility governed by a Permission Set Group check embedded in the FlexCard's data source conditions. Deploy via OmniStudio package to the Provider 360 Lightning App page.

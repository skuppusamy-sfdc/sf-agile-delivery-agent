## Description

As a Claims Operations Manager, I need Agentforce to automatically triage incoming FNOL submissions and generate a fraud risk score on each new Claim__c record at the point of creation, so that high-risk claims are immediately flagged for SIU review, routine low-risk claims are auto-routed to standard adjudication queues, and the overall claims intake cycle time is reduced.

The fraud risk model has been pre-trained and published as an Einstein Prediction Builder model 'Claim_Fraud_Risk_v2'. This story covers the Agentforce agent action configuration, topic definition, and Salesforce-side orchestration.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.FNOL_Submitted__c = true,
When the Agentforce Claims Triage Agent is triggered via a Record-Triggered Flow,
Then the agent invokes the 'Assess Claim Fraud Risk' agent action, which calls the Einstein Prediction Builder model 'Claim_Fraud_Risk_v2' with input features: Claim__c.Claim_Type__c, Claim__c.Total_Billed_Amount__c, Claim__c.Rendering_Provider__c, Claim__c.Date_of_Service__c, and Policy__c.Policy_Tenure_Years__c.

Given the fraud risk score is returned from the model,
When the score is written back to the Claim__c record,
Then Claim__c.AI_Fraud_Risk_Score__c (Number, 0–100) is populated, Claim__c.AI_Risk_Category__c is set to 'Low' (0–39), 'Medium' (40–69), or 'High' (70–100) based on score ranges, and Claim__c.AI_Triage_Timestamp__c = the datetime of scoring.

Given Claim__c.AI_Risk_Category__c = 'High',
When the Agentforce agent completes triage,
Then Claim__c.Status__c = 'Pending SIU Review', Claim__c.Assigned_Queue__c = 'SIU Review Queue', and a high-priority Case is created linked to the Claim__c with Case.Subject = 'AI High-Risk Flag: [Claim Number]' and Case.Priority = 'High'.

Given Claim__c.AI_Risk_Category__c = 'Low' or 'Medium',
When the Agentforce agent completes triage,
Then Claim__c.Status__c = 'In Adjudication' and Claim__c is assigned to the standard adjudication queue based on Claim__c.LOB__c using the existing queue routing rules.

Given the Agentforce agent action fails (model unavailable or API timeout),
When the failure is caught,
Then Claim__c.AI_Triage_Status__c = 'Scoring Failed', Claim__c is routed to the 'Manual Triage' queue, and an error Platform Event 'AI_Triage_Error__e' is published for monitoring.

-Given high-risk claims, Agentforce should auto-deny the claim- (Removed: legal and compliance review confirmed AI cannot auto-deny; human review mandatory for all SIU-flagged claims per regulatory requirement)

Given a Claims Operations Manager reviews the triage dashboard,
When they view the Agentforce Activity summary,
Then a custom report shows daily counts of claims by AI_Risk_Category__c, average AI_Fraud_Risk_Score__c by Claim_Type__c, and the percentage of 'Scoring Failed' outcomes over the trailing 30 days.

## Solution

Configure Agentforce Agent 'Claims Triage Agent' with a Topic 'FNOL Fraud Assessment' and a custom Agent Action 'Assess Claim Fraud Risk' backed by an Apex class 'AgentforceClaimTriageAction' implementing the @InvocableMethod interface. The Apex class calls the Einstein Prediction Builder API for model 'Claim_Fraud_Risk_v2' using ConnectApi.EinsteinAI. Record-Triggered Flow on Claim__c (after insert, when FNOL_Submitted__c = true) invokes the agent via Agentforce API callout. SIU Case creation handled within the Apex action class. Failure handling publishes AI_Triage_Error__e Platform Event consumed by a monitoring Flow that creates an internal alert Task. Custom report built on Claim__c with AI scoring fields; dashboard component added to Claims Operations Manager home page. Permission Set 'Agentforce_Claims_PSG' grants access to agent actions.

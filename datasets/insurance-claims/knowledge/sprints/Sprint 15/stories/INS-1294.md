## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can instantly assess a provider's credentialing status, active network agreements, open PCNs, and recent claims activity without navigating across multiple objects.

Extends INS-1263 (Provider record page layout redesign).

The FlexCard should serve as the primary at-a-glance view for daily provider management tasks including renewals, dispute resolution, and network participation inquiries.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the card displays Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Specialty__c, and Provider__c.Board_Certification_Status__c in the header section.

Given the provider has one or more Credentialing_Case__c records,
When the FlexCard credentialing panel renders,
Then it shows the most recent Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and a color-coded badge: green for 'Active', yellow for 'Expiring Within 90 Days', red for 'Expired' or 'Suspended'.

Given the provider has active Provider_Network_Agreement__c (PNA) records,
When the network agreements panel loads,
Then all PNA records where PNA.Status__c = 'Active' AND PNA.Effective_Date__c <= TODAY are listed with PNA.Network_Name__c, PNA.LOB__c, and PNA.Contract_End_Date__c.

Given the provider has Provider_Change_Notification__c (PCN) records,
When the PCN panel renders,
Then open PCNs where PCN.Status__c IN ('Pending Review', 'Escalated') are displayed with PCN.Change_Type__c and PCN.Submitted_Date__c, limited to the 5 most recent.

Given the provider is linked to Claim__c records via Claim__c.Rendering_Provider__c,
When the claims activity panel loads,
Then a summary shows: total claims in last 90 days, total billed amount (sum of Claim__c.Total_Billed_Amount__c), and count of claims by Claim__c.Status__c grouped as 'Open', 'Adjudicated', 'Denied'.

-Given the provider has a Fee_Schedule__c record, display the full fee schedule line items in the FlexCard.- (Moved to separate story INS-1301 due to complexity)

Given the user's profile does not include the 'Provider_Financial_Data_Viewer' Permission Set Group,
When the FlexCard renders,
Then the claims billed amount summary panel is hidden and replaced with a 'Contact your administrator for financial data access' message.

## Solution

Build using OmniStudio FlexCard with multiple child cards (nested FlexCard pattern). Use DataRaptors for each data panel: DR_Provider360_CredentialingStatus (Turbo Extract on Credentialing_Case__c), DR_Provider360_PNAList (Turbo Extract on Provider_Network_Agreement__c), DR_Provider360_PCNOpen (Turbo Extract on Provider_Change_Notification__c), DR_Provider360_ClaimsSummary (aggregate SOQL on Claim__c). Implement conditional visibility on the financial panel using FlexCard state actions checking the running user's Permission Set Group membership via a custom @AuraEnabled Apex method (ProviderFlexCardPermissionService.cls). Deploy FlexCard to Provider__c Lightning Record Page via App Builder. Cache DataRaptor results for claims summary with a 15-minute TTL using Platform Cache to reduce SOQL load during high-volume usage periods.

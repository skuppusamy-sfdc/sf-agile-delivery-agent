## Description

**Bug Report — UAT Cycle 3**

During ERA posting for institutional claims, the Claim_Line__c adjudication status fields are not refreshing on the parent Explanation_of_Benefits__c record after a successful 835 transaction is processed. The EOB header shows 'Posted' but individual line statuses remain stale.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record with Status__c = 'In Adjudication' and at least 3 associated Claim_Line__c records.
2. Trigger ERA posting via the 'Post ERA' button on the Payment__c record.
3. Confirm the Payment__c.Status__c transitions to 'Posted'.
4. Open the related Explanation_of_Benefits__c record.
5. Observe the child Claim_Line__c related list — Status__c fields still show 'Pending' instead of the adjudicated values.

**Expected Behavior:** All related Claim_Line__c.Status__c values should update to 'Adjudicated' (or 'Denied' / 'Partial' as applicable) within the same transaction batch that posts the ERA.

**Actual Behavior:** Claim_Line__c.Status__c remains 'Pending'. A manual page refresh does not resolve this; the values only update after a full re-query triggered by navigating away and back.

**Environment:** Acme INS Partial Sandbox (PS2). Reproducible 100% of attempts on institutional claim type. Professional claim type unaffected.

## Solution

Root cause identified in the Record-Triggered Flow 'ERA_Post_Update_ClaimLines' — the flow iterates over Claim_Line__c records using a Get Records element but the DML update step is inside a loop, causing governor limit pressure and a silent rollback on larger claim sets (>10 lines). Fix: refactor the flow to collect all Claim_Line__c records into a collection variable outside the loop and perform a single bulk Update Records element after the loop exits. Additionally, add a Platform Event (ERA_Posted__e) publish step so the EOB Lightning Web Component (lwc/eobLineDetail) subscribes via empApi and forces a client-side refresh without requiring navigation. Unit test coverage to be updated in ClaimLineAdjudicationServiceTest.cls.

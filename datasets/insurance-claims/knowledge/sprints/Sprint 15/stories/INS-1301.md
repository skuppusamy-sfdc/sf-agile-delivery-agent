## Description

As a Claims Operations Manager, I need the system to automatically generate a compliant ANSI X12 835 Electronic Remittance Advice (ERA) file and attach it to the associated Explanation_of_Benefits__c (EOB) record whenever a Payment__c record transitions to Status__c = 'Disbursed', so that downstream provider billing systems receive timely and accurate remittance data without manual intervention. Extends INS-1270 (Payment__c lifecycle automation).

## Acceptance Criteria

Given a Payment__c record transitions to Status__c = 'Disbursed' via any update path (UI, API, or Flow), when the Record-Triggered Flow fires, then a Platform Event 'ERA_Generation_Requested__e' must be published within 2 seconds containing Payment__c.Id, Payment__c.Total_Paid_Amount__c, and Payment__c.Provider__c.

Given the ERA_Generation_Requested__e Platform Event is received by the Apex subscriber, when the ERA file is constructed, then it must include all child Claim_Line__c records linked via Payment__c.Claim__c with fields: Claim_Line__c.CPT_Code__c, Claim_Line__c.Billed_Amount__c, Claim_Line__c.Allowed_Amount__c, Claim_Line__c.Adjustment_Reason_Code__c, and Claim_Line__c.Paid_Amount__c.

Given the ERA file is successfully generated, when the process completes, then a ContentVersion record must be created and linked to the associated Explanation_of_Benefits__c record via ContentDocumentLink, with ContentVersion.Title set to 'ERA_[Payment__c.Name]_[YYYYMMDD]'.

Given the ERA generation fails due to missing mandatory fields (e.g., null NPI on Provider__c), when the error occurs, then Payment__c.ERA_Generation_Status__c must be set to 'Failed', a Task must be created assigned to the Claims Operations queue, and an error log entry must be written to ERA_Error_Log__c with the fault reason.

Given a Payment__c associated with a TPA-managed Claim__c (Claim__c.TPA_Managed__c = true), when the ERA is generated, then EOB__c.Delivery_Method__c must be evaluated: if 'SFTP', the file must be queued to the outbound SFTP Integration Procedure; if 'Portal', the file must be surfaced on the Provider Portal only.

## Solution

Implement a Record-Triggered Flow on Payment__c (after update, Status__c changed to 'Disbursed') that publishes ERA_Generation_Requested__e Platform Event. Build an Apex Platform Event trigger subscriber (ERA_GenerationSubscriber) that invokes ERA835BuilderService Apex class to construct the X12 835 payload from Claim_Line__c aggregation. Use ContentVersion + ContentDocumentLink insertion for file attachment. Route delivery via OmniStudio Integration Procedure 'ERA_DeliveryRouter' which branches on EOB__c.Delivery_Method__c. All Apex classes must have 90%+ test coverage. Add custom field ERA_Generation_Status__c (Picklist) to Payment__c.

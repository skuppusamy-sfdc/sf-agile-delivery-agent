## Description

As a Claims Operations Manager, I need an Agentforce-powered AI triage step embedded in the FNOL OmniScript that analyzes incoming Claim__c data against historical claim patterns and flags high-risk claims for SIU referral with an explainable recommendation so that the SIU team can prioritize investigations and reduce fraudulent payment disbursements. Extends INS-1314 (FNOL OmniScript base implementation).

## Acceptance Criteria

Given a new Claim__c record is being created via the FNOL OmniScript,
When the adjuster reaches the 'Claim Review' step and clicks 'Run AI Triage',
Then the OmniScript invokes the Agentforce 'Claims_Triage_Agent' topic action which returns a risk score (0–100) stored in Claim__c.AI_Risk_Score__c and a recommendation string stored in Claim__c.AI_Triage_Recommendation__c.

Given the AI risk score is returned,
When Claim__c.AI_Risk_Score__c >= 75,
Then the OmniScript displays a prominent 'High Fraud Risk' banner with the AI explanation text and a 'Refer to SIU' action button; Claim__c.SIU_Flag__c is set to TRUE automatically.

Given the AI risk score is between 40 and 74,
When the triage result renders,
Then a 'Moderate Risk – Manual Review Recommended' advisory is shown and Claim__c.SIU_Flag__c remains FALSE but Claim__c.Review_Required__c = TRUE.

Given the adjuster clicks 'Refer to SIU',
When the action is confirmed,
Then a new Case is created with RecordType = 'SIU_Investigation', linked to the Claim__c via Case.Claim__c, assigned to the SIU queue, and Claim__c.Status__c is updated to 'SIU Referral Pending'.

Given the Agentforce API call fails or times out,
When the failure occurs during OmniScript execution,
Then the OmniScript displays a non-blocking warning 'AI Triage Unavailable – Proceeding Manually', allows the adjuster to continue FNOL completion, and logs the failure to Error_Log__c.

-Given the AI model uses PII data fields for scoring, Then Provider__c.SSN__c and Policy__c.Member_DOB__c are included in the prompt payload- (removed per compliance review — PII fields excluded from AI prompt per Compliance Officer directive INS-COMP-44)

Given a Compliance Officer reviews AI-assisted referrals,
When they access the Claim__c record,
Then Claim__c.AI_Risk_Score__c, Claim__c.AI_Triage_Recommendation__c, and the timestamp of the triage run (Claim__c.AI_Triage_Run_DateTime__c) are visible in the 'AI Insights' Lightning component.

## Solution

Configure Agentforce Agent 'Claims_Triage_Agent' with a custom Topic 'Claim Risk Assessment' and Action that calls an Apex class 'AgentforceClaimsTriageAction' (InvocableMethod). The Apex class constructs a prompt using non-PII Claim__c fields (LOB, Claim_Type__c, Billed_Amount__c, Provider specialty, historical claim count for NPI) and invokes the Einstein LLM API via ConnectApi.EinsteinLLM. Response JSON is parsed to extract risk_score and recommendation. OmniScript integration via a REST DataRaptor calling the Agentforce topic action endpoint. SIU Case creation handled by a sub-OmniScript step invoking a Flow. AI Insights LWC ('aiClaimsInsightsPanel') reads Claim__c AI fields and renders with appropriate visual risk indicators.

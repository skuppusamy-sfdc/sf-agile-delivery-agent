## Description

As a Provider Network Specialist, I need an automated termination workflow that creates a Network_Termination_Case__c, generates a termination notice via DocuSign, updates the associated Provider_Network_Agreement__c status, and notifies downstream claims adjudication so that provider terminations are executed consistently, auditably, and within regulatory notice period requirements (typically 60–90 days depending on LOB).

## Acceptance Criteria

Given a Provider Network Specialist initiates termination by clicking 'Initiate Termination' on a PNA record where PNA.Status__c = 'Active',
When the OmniScript 'OS_InitiateProviderTermination' is launched,
Then a new Network_Termination_Case__c is created with NTC.PNA__c = current PNA Id, NTC.Status__c = 'Pending Approval', NTC.Requested_By__c = running user, and NTC.Notice_Period_Days__c auto-populated based on PNA.LOB__c lookup to a custom metadata type 'Termination_Notice_Config__mdt'.

Given the Network_Termination_Case__c is created,
When NTC.Status__c = 'Pending Approval',
Then a Salesforce Approval Process routes the record to the assigned Claims Operations Manager queue; the approver receives an email notification with a summary of the provider's active PNAs and open Claim__c records where Claim__c.Provider__c = terminating provider and Claim__c.Status__c NOT IN ('Closed','Denied').

Given the NTC is approved,
When NTC.Status__c transitions to 'Approved',
Then a Record-Triggered Flow invokes the DocuSign eSignature API via a named credential to send the termination notice template ('Provider_Termination_Notice_v2') to the provider's email on file (Provider__c.Email).

Given DocuSign returns a 'Completed' envelope status callback,
When the DocuSign status Platform Event is received,
Then PNA.Status__c is updated to 'Pending Termination', PNA.Termination_Date__c is set to TODAY() + NTC.Notice_Period_Days__c, and a PCN record is auto-created with PCN.Change_Type__c = 'Termination' and PCN.Status__c = 'Sent'.

Given the termination effective date is reached (PNA.Termination_Date__c = TODAY()),
When a scheduled Apex batch job 'ProviderTerminationBatch' runs nightly,
Then PNA.Status__c is updated to 'Terminated', Provider__c.Network_Status__c is set to 'Out of Network', and any open Claim__c records linked to the provider are flagged with Claim__c.Network_Status_Flag__c = 'Provider Terminated — Review Required'.

Given the entire termination workflow completes,
When a Compliance Officer queries the NTC record,
Then all audit fields (NTC.Approval_Date__c, NTC.DocuSign_Envelope_Id__c, NTC.Termination_Effective_Date__c, NTC.Completed_By__c) are populated and a complete Chatter feed history is present on the NTC record.

## Solution

Implement a multi-step OmniScript 'OS_InitiateProviderTermination' with DataRaptor Transform steps for NTC creation and LOB-based notice period lookup via custom metadata 'Termination_Notice_Config__mdt'. Approval Process configured with two-step routing: (1) auto-approval if NTC.Termination_Reason__c = 'Voluntary' and no open claims; (2) manager queue escalation otherwise. DocuSign integration via existing DocuSign for Salesforce managed package — configure envelope template mapping in DocuSign Setup. Platform Event 'DocuSign_Status__e' subscriber flow handles PNA update and PCN creation. Scheduled Apex 'ProviderTerminationBatch' implements Database.Batchable<SObject> with scope 200, querying PNA records where Termination_Date__c = TODAY() and Status__c = 'Pending Termination'. All DML wrapped in try/catch with custom exception logging to Integration_Error_Log__c.

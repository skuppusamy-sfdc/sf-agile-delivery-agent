## Description

As a Member Services Representative, I need the Policy__c record page to display a real-time summary panel showing the count of active claims, the date of the last FNOL submission, and a COB flag indicator, so that I can quickly orient myself to a policyholder's current claims activity during a service call without switching between objects or running reports.

## Acceptance Criteria

Given a Member Services Rep opens a Policy__c record page,
When the Policyholder Claims Summary LWC loads,
Then the panel displays:
- Active_Claims_Count: COUNT of Claim__c records WHERE Claim__c.Policy__c = Policy__c.Id AND Claim__c.Status__c NOT IN ('Closed', 'Denied', 'Fully Denied')
- Last_FNOL_Date: MAX(Claim__c.FNOL_Date__c) across all related Claim__c records
- COB_Active: TRUE if any related Claim__c has COB_Indicator__c = TRUE and Status__c NOT IN ('Closed', 'Denied')

Given the policy has zero claims on record,
When the panel renders,
Then Active_Claims_Count displays '0', Last_FNOL_Date displays 'No claims on file', and COB_Active displays 'N/A'.

Given the COB_Active flag is TRUE,
When the panel renders,
Then the COB indicator is displayed as a yellow warning badge with tooltip text: 'Coordination of Benefits active — verify primary/secondary payer sequence before processing payment'.

-Given the panel includes a 'File New Claim' quick action button- (deprecated — quick action relocated to Policy__c page header per UX review INS-UX-44)

Given the panel data is loaded,
When the underlying Claim__c records are updated by another user,
Then the panel refreshes automatically within 30 seconds using Lightning Data Service wire adapter without a full page reload.

## Solution

Build a custom LWC (policyholderClaimsSummary) using @wire(getRecord) with Lightning Data Service for Policy__c base fields and a separate @wire Apex-backed method (PolicyClaimsSummaryController.getSummaryData) returning a wrapper object with aggregated claim counts via SOQL aggregate functions (COUNT, MAX). COB flag derived server-side. Component uses lightning-card with lightning-badge for COB indicator and conditional CSS classes (slds-theme--warning). Auto-refresh implemented via setInterval calling refreshApex every 30 seconds. LWC added to Policy__c Lightning Record Page via App Builder with Audience targeting 'Member Services Profile' and 'Claims Supervisor Profile'. Component tested with Jest unit tests covering zero-claim, single-claim, and multi-claim COB scenarios.

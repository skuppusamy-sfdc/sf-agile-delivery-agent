## Description

As a Claims Operations Manager, I need an Agentforce AI-powered triage recommendation to be generated automatically upon FNOL submission of a new Claim__c record, so that high-complexity, high-risk, or potentially fraudulent claims are flagged for SIU review or fast-tracked for straight-through processing — reducing manual triage time and improving adjudication accuracy. The AI recommendation should analyze claim characteristics including LOB, ICD-10 codes, CPT codes, claimant claim history, and provider risk score.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Status__c = 'Submitted' via the FNOL OmniScript,
When the Agentforce triage action is invoked,
Then an AI-generated triage recommendation is written to Claim__c.AI_Triage_Recommendation__c (picklist: 'Straight-Through Processing', 'Standard Review', 'Complex Review', 'SIU Referral') within 30 seconds of Claim__c creation.

Given the Agentforce model evaluates the claim,
When Claim__c.AI_Triage_Score__c >= 85 (out of 100),
Then Claim__c.AI_Triage_Recommendation__c = 'SIU Referral' and a high-priority Task is auto-created and assigned to the 'SIU_Intake' queue with Subject 'AI-Flagged: SIU Review Required — ' + Claim__c.Name.

Given the Agentforce model evaluates the claim,
When Claim__c.AI_Triage_Score__c <= 25 AND Claim__c.LOB__c IN ('Auto', 'Property') AND no prior open Claim__c records exist for the same Policy__c,
Then Claim__c.AI_Triage_Recommendation__c = 'Straight-Through Processing' and Claim__c.Status__c is advanced to 'Auto-Adjudication Queue'.

Given the AI recommendation is generated,
When a Claims Operations Manager reviews the Claim__c record,
Then Claim__c.AI_Triage_Rationale__c (long text) is populated with a human-readable explanation of the top 3 factors influencing the recommendation (e.g., 'High-risk CPT code 99215 detected; Provider risk score 78/100; 3 prior claims on same Policy__c in 12 months').

Given the Agentforce service is unavailable or times out,
When the triage action fails,
Then Claim__c.AI_Triage_Status__c = 'Error — Manual Review Required', no recommendation is written, and the claim is routed to the 'Claims_Manual_Triage' queue without blocking claim creation.

Given a Claims Operations Manager disagrees with the AI recommendation,
When they update Claim__c.AI_Triage_Override__c = TRUE and provide Claim__c.AI_Triage_Override_Reason__c,
Then the override is logged and Claim__c.AI_Triage_Recommendation__c can be manually updated, preserving the original AI recommendation in Claim__c.AI_Original_Recommendation__c for audit.

Given the feature is live,
When the Compliance Officer runs the monthly AI model accuracy report,
Then override rate, SIU conversion rate, and straight-through processing accuracy metrics are available in the 'Agentforce Claims Triage' CRM Analytics dashboard.

## Solution

Configure an Agentforce Agent Action ('Claims_Triage_Action') backed by an Apex invocable method ('AgentforceClaimsTriage.evaluate') that calls the Agentforce Einstein model endpoint via a Named Credential. The Apex method constructs a JSON payload from Claim__c fields (LOB__c, ICD10_Codes__c, CPT_Codes__c) plus aggregated Policy__c claim history via a SOQL subquery. The Agentforce model (fine-tuned on historical Claim__c adjudication outcomes) returns a score and recommendation. A Platform Event ('ClaimsTriageResult__e') is published by the Apex callout callback and consumed by a Platform Event-Triggered Flow ('Claim_Triage_Result_Flow') that writes results back to Claim__c fields and creates SIU Tasks where required. Timeout/error handling implemented via try-catch in Apex with fallback DML setting AI_Triage_Status__c = 'Error'. Override audit trail maintained via a custom Claim_AI_Audit__c child object. CRM Analytics dashboard built on Claim__c with Agentforce triage fields as dimensions/measures.

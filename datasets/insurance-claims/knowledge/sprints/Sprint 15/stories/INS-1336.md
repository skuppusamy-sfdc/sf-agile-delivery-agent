## Description

As a Broker Relations Coordinator, I need brokers to be able to log into the Agent Portal Experience Cloud site and download their commission statements filtered by date range and LOB so that brokers have self-service access to their payment history, reducing inbound calls to the Broker Relations team by an estimated 30% and improving broker satisfaction scores.

This story covers the portal-facing LWC and the underlying Apex data service; PDF generation via Conga CLM is handled in a follow-on story.

## Acceptance Criteria

Given a broker logs into the Agent Portal Experience Cloud site,
When they navigate to the 'My Commissions' page,
Then they see only Payment__c records where Payment__c.Payee__c = their linked Contact Id and Payment__c.Payment_Type__c = 'Commission' — enforced via Sharing Rules (Criteria-Based Sharing on Payment__c).

Given the broker is on the 'My Commissions' page,
When they select a date range (Start Date, End Date) and optionally filter by LOB (Payment__c.LOB__c picklist),
Then the commission statement table refreshes to show only matching Payment__c records with columns: Payment__c.Name, Payment__c.Amount__c, Payment__c.Payment_Date__c, Payment__c.LOB__c, Payment__c.Status__c, and Payment__c.Policy__c (policy number).

Given the filtered results are displayed,
When the broker clicks 'Download CSV',
Then a CSV file is generated client-side from the current filtered dataset and downloaded to the browser — the file name follows the pattern 'CommissionStatement_[BrokerName]_[StartDate]_[EndDate].csv'.

Given a broker has no Payment__c records matching the selected filter criteria,
When the table renders,
Then an empty state illustration and message 'No commission records found for the selected period' are displayed — no error is thrown.

Given the broker portal page is accessed on a mobile device (viewport < 768px),
When the 'My Commissions' LWC renders,
Then the table collapses to a card-based layout showing Payment__c.Amount__c and Payment__c.Payment_Date__c prominently, with an expandable row for additional fields.

## Solution

Build LWC 'brokerCommissionStatement' deployed on the Agent Portal Experience Cloud site (LWC target: lightningCommunity__Page). Apex controller 'BrokerCommissionController.cls' with @AuraEnabled method 'getCommissions(String startDate, String endDate, String lob)' — uses WITH SECURITY_ENFORCED SOQL on Payment__c to enforce FLS and record-level sharing. Criteria-Based Sharing Rule on Payment__c: share records where Payment_Type__c = 'Commission' with the 'Broker_Portal_User' Public Group based on Payee__c = current user's Contact. CSV generation handled entirely client-side in the LWC JavaScript controller using a utility function that converts the wire-fetched data array to CSV blob and triggers download via an anchor element. Mobile responsive layout implemented using SLDS grid breakpoint classes and conditional rendering via @track isMobile property set from window.innerWidth. Guest user profile hardened — Payment__c object not accessible without authenticated broker session.

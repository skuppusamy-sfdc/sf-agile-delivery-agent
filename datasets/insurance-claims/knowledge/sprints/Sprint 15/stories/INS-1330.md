## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can instantly view a provider's active credentialing status, network agreement summary, fee schedule assignments, and recent PCN history without navigating across multiple related lists or tabs.

Extends INS-1299 (Provider 360 base layout approved in Phase 1).

## Acceptance Criteria

Given a Provider__c record is open in the console,
When the Provider 360 FlexCard loads,
Then the card displays the provider's NPI (Provider__c.NPI__c), TIN (Provider__c.TIN__c), and primary specialty (Provider__c.Primary_Specialty__c) in the header band.

Given the provider has one or more active Credentialing_Case__c records,
When the FlexCard renders,
Then the credentialing panel shows the most recent Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and a color-coded badge (Green = Active, Yellow = Expiring within 90 days, Red = Expired).

Given the provider has active Provider_Network_Agreement__c (PNA) records,
When the network agreements section loads,
Then all PNA records where PNA.Status__c = 'Active' and PNA.Effective_Date__c <= TODAY() and PNA.Termination_Date__c >= TODAY() are listed with LOB, effective date, and contracting tier.

-Given the provider has a linked Fee_Schedule__c, the card shall display all CPT code ranges inline.- (Deprecated — replaced by drill-down modal per UX review)

Given the provider has Provider_Change_Notification__c (PCN) records in the last 180 days,
When the PCN history panel renders,
Then up to 5 most recent PCNs are shown with PCN.Change_Type__c, PCN.Effective_Date__c, and PCN.Status__c.

Given the FlexCard is viewed by a user without the 'Provider_Network_Specialist' Permission Set Group,
When the card loads,
Then the TIN field is masked (showing last 4 digits only) and the 'Edit PNA' action button is hidden.

## Solution

Build OmniStudio FlexCard 'Provider360Card' with multiple DataRaptor Extract sources: (1) DR_Extract_ProviderHeader pulling from Account (Person Account) with NPI__c and TIN__c; (2) DR_Extract_CredentialingStatus querying Credentialing_Case__c ordered by CreatedDate DESC LIMIT 1; (3) DR_Extract_ActivePNAs querying Provider_Network_Agreement__c with date-range filter; (4) DR_Extract_RecentPCNs querying Provider_Change_Notification__c with 180-day filter. Use FlexCard conditional visibility states for the color-coded credentialing badge (State conditions on Expiration_Date__c formula). TIN masking implemented via a FlexCard custom action calling an Apex @AuraEnabled method that checks the running user's Permission Set Group membership (PermissionSetGroupAssignment SOQL). Deploy via OmniStudio deployment package to PS2 sandbox for UAT validation.

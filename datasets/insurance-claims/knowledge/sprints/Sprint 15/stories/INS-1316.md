## Description

As a Provider Network Specialist, I need the system to automatically validate that an active Fee_Schedule__c record exists and covers the PNA effective date range when a Provider_Network_Agreement__c is countersigned via DocuSign so that we prevent network agreements from going live without a corresponding reimbursement schedule, reducing downstream claim adjudication failures.

This story covers the DocuSign envelope completed callback integration into Salesforce and the subsequent Fee Schedule gap check logic.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record has DocuSign envelope status updated to 'Completed',
When the DocuSign Connect callback fires and updates PNA__c.DocuSign_Status__c = 'Completed',
Then a Platform Event 'PNA_Countersigned__e' is published containing PNA__c.Id, PNA__c.Effective_Start_Date__c, PNA__c.Effective_End_Date__c, and PNA__c.LOB__c.

Given the 'PNA_Countersigned__e' Platform Event is received,
When the triggered Apex subscriber 'FeeScheduleValidationService' executes,
Then it queries Fee_Schedule__c records where Fee_Schedule__c.LOB__c = PNA.LOB__c AND Fee_Schedule__c.Effective_Start_Date__c <= PNA.Effective_Start_Date__c AND Fee_Schedule__c.Effective_End_Date__c >= PNA.Effective_End_Date__c AND Fee_Schedule__c.Status__c = 'Active'.

Given no matching Fee_Schedule__c record is found,
When the validation fails,
Then PNA__c.Status__c is set to 'Pending Fee Schedule' (not 'Active'), a Task is created assigned to the owning Provider Network Specialist with Subject = 'Fee Schedule Gap — Action Required' and Due Date = TODAY() + 3, and an in-app notification is sent via Custom Notification Type 'PNA_Fee_Gap_Alert'.

Given a matching active Fee_Schedule__c record is found,
When the validation passes,
Then PNA__c.Status__c is set to 'Active', PNA__c.Countersigned_Date__c = TODAY(), and PNA__c.Linked_Fee_Schedule__c is populated with the matching Fee_Schedule__c.Id.

-Given multiple fee schedules match, select the one with the highest Fee_Schedule__c.Version__c number-
(Deprecated — replaced by selecting the schedule with the most recent Fee_Schedule__c.Approved_Date__c per compliance requirement CR-2024-088)

Given PNA__c.Status__c transitions to 'Active',
When the record is saved,
Then a Chatter post is added to the PNA record feed: '@[Provider Network Specialist] PNA activated with Fee Schedule [Fee_Schedule__c.Name] effective [PNA.Effective_Start_Date__c].'

## Solution

DocuSign Connect configured to POST to a named Salesforce Connected App endpoint. An Apex REST resource '@RestResource(urlMapping="/docusign/pna-callback")' receives the payload, updates PNA__c.DocuSign_Status__c, and publishes 'PNA_Countersigned__e' via EventBus.publish(). An Apex Trigger on PNA_Countersigned__e invokes 'FeeScheduleValidationService' as a queueable to avoid governor limits. Status transitions enforced via a Validation Rule on PNA__c preventing manual override of 'Pending Fee Schedule' status without the 'PNA_Admin_Override' Permission Set. Custom Notification deployed via Notification Builder. All logic covered by Apex test class 'FeeScheduleValidationServiceTest' with 94% coverage.

## Description

As an Underwriting Analyst, I need the Group/Employer onboarding OmniScript to support the creation of multiple Group_Service_Agreement__c records across different Lines of Business (Medical, Dental, Vision) in a single guided workflow, with tiered rate configuration per LOB, so that employer group onboarding can be completed in one session rather than requiring separate manual GSA creation for each LOB.

This is a Phase 2 enhancement to the base onboarding OmniScript delivered in INS-1278. The new multi-LOB step must be inserted after the employer verification step without breaking the existing single-LOB flow for groups with only one LOB.

## Acceptance Criteria

Given an Underwriting Analyst begins the Group Employer Onboarding OmniScript,
When they reach the 'Lines of Business Selection' step,
Then they can select one or more LOBs from a multi-select checkbox group (Medical, Dental, Vision, Pharmacy) and the OmniScript dynamically renders a separate rate configuration sub-step for each selected LOB.

Given the analyst configures rates for a selected LOB,
When they enter tier information,
Then they must specify at minimum: GSA_LOB__c, Effective_Date__c, Expiration_Date__c, Tier_1_Employee_Rate__c, Tier_2_Employee_Spouse_Rate__c, Tier_3_Family_Rate__c, and the linked Fee_Schedule__c (validated as active and matching the LOB).

Given all LOB configurations are completed and the analyst clicks 'Submit',
When the OmniScript Integration Procedure executes,
Then one Group_Service_Agreement__c record is created per selected LOB with all configured fields populated, all linked to the same parent Account (employer group), and GSA.Status__c = 'Pending Approval'.

Given a GSA is created with Status__c = 'Pending Approval',
When the record is inserted,
Then an Approval Process is automatically submitted for the 'GSA_Underwriting_Approval' process, routing to the Underwriting Manager queue for groups with Total_Enrolled_Employees__c > 500, and auto-approving for groups <= 500.

-Given the OmniScript should auto-populate broker commission rates from the linked broker's BrokerManagement profile- (deprecated — commission rate logic is being redesigned in INS-1342 and will be handled via a separate DataRaptor post-submission)

Given an analyst selects only one LOB,
When the OmniScript renders,
Then the multi-LOB configuration panel collapses to a single-LOB layout identical to the INS-1278 baseline flow, ensuring backward compatibility.

Given any GSA creation fails during the Integration Procedure,
When the error occurs,
Then the OmniScript displays a step-level error message identifying which LOB failed, rolls back any successfully created GSAs for that submission via a compensating DataRaptor Load (delete), and allows the analyst to correct and resubmit.

## Solution

Extend the existing 'GroupEmployerOnboarding' OmniScript (INS-1278) by inserting a new 'Multi-LOB Configuration' step using a Repeat block to dynamically render LOB sub-forms based on the multi-select checkbox output. Each LOB sub-form is a nested OmniScript Step with conditional visibility. The Integration Procedure 'IP_CreateMultiLOBGSAs' uses a Loop Action over the selected LOBs array, calling a DataRaptor Load (DR_Load_GSA) for each iteration. Approval Process submission handled via an Apex Action (GSAApprovalSubmitAction.cls) invoked from the Integration Procedure post-loop. Rollback on error implemented with a secondary DataRaptor Load configured for delete, keyed on the GSA IDs captured in the loop. Custom Metadata Type 'LOB_FeeSchedule_Validation__mdt' used to validate Fee Schedule LOB alignment at the OmniScript step level before submission.

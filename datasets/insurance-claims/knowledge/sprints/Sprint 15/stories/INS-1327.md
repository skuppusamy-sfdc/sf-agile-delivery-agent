## Description

As a Credentialing Specialist, I need Salesforce to perform a nightly scheduled sync with CAQH ProView so that provider credentialing data (license status, board certifications, malpractice history, DEA registration) is automatically refreshed on the Provider__c and Credentialing_Case__c records, reducing manual data entry and ensuring credentialing decisions are based on current verified information.

The sync must handle both new provider onboarding (create mode) and existing provider re-credentialing (update mode) based on whether a matching Provider__c record exists for the CAQH Provider ID.

## Acceptance Criteria

Given the nightly CAQH sync batch job runs at 02:00 AM local time,
When the job executes,
Then it calls the CAQH ProView REST API for all Provider__c records where CAQH_Provider_ID__c IS NOT NULL and Credentialing_Case__c.Next_Review_Date__c <= TODAY + 90 days.

Given the CAQH API returns updated license data,
When the sync processes a Provider__c record,
Then Provider__c.License_Status__c, Provider__c.License_Expiration_Date__c, Provider__c.DEA_Registration_Number__c, and Provider__c.Board_Certification__c are updated with CAQH-sourced values, and Provider__c.CAQH_Last_Sync_Date__c is set to the current timestamp.

Given a discrepancy is detected between the current Salesforce value and the CAQH-returned value for a critical field (License_Status__c, Malpractice_Coverage_Active__c),
When the sync runs,
Then a Credentialing_Discrepancy__c record is created with Discrepancy_Type__c, Old_Value__c, New_Value__c, and a lookup to the Provider__c, and the related Credentialing_Case__c.Status__c is set to 'Discrepancy Review'.

Given a provider's license has expired per CAQH data (License_Status__c = 'Expired'),
When the sync updates the Provider__c record,
Then all active Provider_Network_Agreement__c records for that provider are flagged with PNA.Credentialing_Hold__c = TRUE and an alert email is sent to the assigned Credentialing Specialist via an Org-Wide Email Alert.

Given the CAQH API returns a 4xx or 5xx error for a specific provider,
When the batch processes that record,
Then the error is logged in an Integration_Error_Log__c record with Error_Code__c, Error_Message__c, and Provider__c lookup, and the batch continues processing remaining records without failing the entire job.

Given the batch job completes,
When all records are processed,
Then a Batch_Job_Summary__c record is created with Total_Processed__c, Total_Updated__c, Total_Errors__c, and Job_Completion_Time__c, and a summary email is sent to the Credentialing Operations distribution list.

## Solution

Implement a Batchable and Schedulable Apex class (CAQHCredentialingSyncBatch.cls) using Database.Batchable<SObject> with a scope of 50 records per batch to respect CAQH API rate limits. HTTP callouts to CAQH ProView REST API via Named Credential 'CAQH_ProView'. A separate Queueable class (CAQHCalloutQueueable.cls) handles the actual callout from within the batch to comply with the no-callout-in-batch restriction pattern (using Database.AllowsCallouts interface). Discrepancy detection uses a comparison utility class (CAQHFieldComparisonUtil.cls). Schedule registered via Apex Scheduler at 02:00 AM. Custom Metadata Type 'CAQH_Sync_Config__mdt' stores critical field list and API endpoint configuration. Integration_Error_Log__c and Batch_Job_Summary__c written in the finish() method.

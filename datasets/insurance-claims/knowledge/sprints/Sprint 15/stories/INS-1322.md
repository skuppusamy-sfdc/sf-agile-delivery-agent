## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can instantly view a provider's current network participation status, active credentialing case, PNA effective dates, and recent PCNs without navigating across multiple objects.

This story extends INS-1291 (Provider 360 data model alignment) and consumes the DataRaptor extracts established in that work.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record in Salesforce,
When the Provider 360 FlexCard loads,
Then the card displays Provider_Network_Agreement__c.Status__c, PNA Effective_Date__c, and Termination_Date__c for all active PNAs (Status__c = 'Active') in a child list section.

Given a provider has an open Credentialing_Case__c,
When the FlexCard renders,
Then the Credentialing Summary section displays Credentialing_Case__c.Stage__c, Credentialing_Case__c.Expiration_Date__c, and a color-coded status indicator (Green = 'Credentialed', Yellow = 'Pending Renewal' within 90 days, Red = 'Expired').

Given a provider has one or more Provider_Change_Notification__c records,
When the FlexCard loads,
Then the PCN section shows the three most recent PCNs sorted by PCN_Submitted_Date__c descending, displaying PCN_Type__c and PCN_Status__c.

Given the provider's NPI is stored in Provider__c.NPI__c,
When the FlexCard loads,
Then a 'Verify NPI' action button calls the NPPES integration and displays last verification timestamp from Provider__c.NPI_Last_Verified__c.

-Given the FlexCard is embedded on the Facility__c page, it should show affiliated providers list- (deprecated — Facility 360 is a separate story INS-1335)

Given a provider has no active PNA records,
When the FlexCard loads,
Then the PNA section displays the message 'No Active Network Agreements' and surfaces a 'Initiate Contracting' quick action.

Given any data section fails to load due to a DataRaptor error,
When the FlexCard renders,
Then an inline error banner displays 'Data unavailable — contact system administrator' without crashing the entire card.

## Solution

Build using OmniStudio FlexCard with three child cards (PNA Summary, Credentialing Summary, PCN History). Create three DataRaptor Extract definitions: DR_Extract_ActivePNAs, DR_Extract_CredentialingCase, DR_Extract_RecentPCNs. Use Integration Procedures to orchestrate parallel DataRaptor calls and merge results. NPPES 'Verify NPI' action invokes an existing named credential callout wrapped in an Apex REST action (NppesVerificationAction.cls). Embed FlexCard on Provider__c Lightning Record Page via App Builder. Apply Permission Set Group 'Provider_Relations_Full' to control visibility of the 'Initiate Contracting' quick action.

## Description

As a Broker Relations Coordinator, I need an automated workflow that generates a Group_Service_Agreement__c (GSA) contract document via Conga CLM, routes it for e-signature through DocuSign, and upon completion automatically activates the GSA record and triggers downstream Policy__c group enrollment setup so that employer group onboarding is completed faster and with a full audit trail of contract execution.

## Acceptance Criteria

Given a Group_Service_Agreement__c record where GSA__c.Status__c = 'Pending Signature' and GSA__c.Broker__c is populated,
When the Broker Relations Coordinator clicks the 'Send for Signature' quick action,
Then a Conga CLM document generation is triggered using template 'GSA_Standard_Group_Contract_v2', producing a PDF attached to the GSA__c record as a ContentDocument.

Given the Conga document is generated successfully,
When the PDF is available,
Then a DocuSign envelope is automatically created and sent to the signatories defined in GSA__c.Primary_Signatory_Email__c and GSA__c.Secondary_Signatory_Email__c, and GSA__c.DocuSign_Envelope_Id__c is populated.

Given the DocuSign envelope is sent,
When GSA__c.DocuSign_Envelope_Id__c is populated,
Then GSA__c.Status__c updates to 'Awaiting Signature' and a 7-day auto-reminder is scheduled via a Time-Triggered Flow to resend the DocuSign envelope if not signed.

Given all required signatories have completed e-signature in DocuSign,
When the DocuSign status webhook fires the 'Envelope Completed' event,
Then GSA__c.Status__c is updated to 'Active', GSA__c.Execution_Date__c = the DocuSign completion timestamp, and the signed PDF is attached to the GSA__c record as a new ContentVersion with Title = 'Executed GSA – [GSA Number]'.

Given GSA__c.Status__c transitions to 'Active',
When the activation occurs,
Then a Platform Event 'GSA_Activated__e' is published with GSA_Id__c and Effective_Date__c, triggering a subscriber Flow that creates the initial group Policy__c shell records for each enrolled member group defined in GSA__c.Member_Group_Count__c.

Given a signatory declines to sign in DocuSign,
When the 'Envelope Declined' webhook event is received,
Then GSA__c.Status__c reverts to 'Declined', a Task is created for the Broker Relations Coordinator with Subject = 'GSA Signature Declined – Action Required', and a Chatter notification is posted to the GSA__c record.

-Given the GSA requires a wet signature fallback option, Then a 'Print and Mail' button is available on the GSA record- (deferred to Phase 3 — wet signature fallback not required for MVP per business sign-off)

## Solution

Implement a Screen Flow 'GSA_Signature_Initiation_Flow' launched from a Quick Action on GSA__c. Flow calls Apex action 'CongaGSADocumentGenerator' to produce the contract PDF, then calls Apex action 'DocuSignEnvelopeCreator' using the DocuSign eSignature API (via Named Credential 'DocuSign_Prod'). A Remote Site Setting and Connected App are configured for DocuSign OAuth. DocuSign webhook events received via a public Site Visualforce endpoint that deserializes the JSON payload and invokes Apex 'DocuSignWebhookHandler' to update GSA__c fields. GSA activation Platform Event published via Apex EventBus.publish(). Subscriber Flow creates Policy__c records using a Fast Create element. Time-based reminder managed by a Scheduled-Triggered Flow querying GSA__c records in 'Awaiting Signature' status past 7 days.

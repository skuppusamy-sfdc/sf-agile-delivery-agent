## Description

**Bug Report — UAT Phase 2 Defect**

During UAT testing of the Group/Employer onboarding OmniScript 'OS_GroupOnboarding_v4', testers discovered that when the Group_Service_Agreement__c.Record_Type is set to 'Self-Funded', the OmniScript skips Step 3 ('TIN Verification & IRS Validation') entirely and proceeds directly to Step 4 ('Benefit Plan Configuration'). This results in GSA records being created without a validated TIN__c value, which violates compliance requirements and causes downstream payment routing failures in the PaymentEngine.

**Steps to Reproduce:**
1. Launch OmniScript 'OS_GroupOnboarding_v4' from the Group/Employer Account record page.
2. On Step 1, select GSA Record Type = 'Self-Funded'.
3. Complete Step 2 (Employer Demographics).
4. Observe that Step 3 (TIN Verification) is skipped — the step indicator jumps from 2 to 4.
5. Complete the OmniScript and submit.
6. Navigate to the created Group_Service_Agreement__c record and observe TIN__c = null.

**Expected:** Step 3 TIN Verification must execute for ALL GSA record types including 'Self-Funded'.

**Actual:** Step 3 is bypassed for 'Self-Funded' record type; Group_Service_Agreement__c.TIN__c remains null on the created record.

**Environment:** Sandbox ACME-UAT2
**Regression:** This regression was introduced in the Sprint 14 deployment (INS-1274 — GSA record type expansion). The conditional visibility formula on Step 3 was inadvertently set to exclude 'Self-Funded'.

## Solution

Open OmniScript 'OS_GroupOnboarding_v4' in OmniStudio Designer. Navigate to Step 3 element 'TIN_Verification_Step'. Inspect the Conditional View expression — current value is: '{RecordType} != "Self-Funded"' which incorrectly hides the step for Self-Funded. Remove this conditional expression entirely so Step 3 renders for all record types. Re-validate that the DataRaptor Transform 'DRT_ValidateTIN_IRS' is invoked on Step 3 completion for all record types. Update the DataRaptor Post 'DRP_CreateGSA' to include a mandatory non-null assertion on TIN__c before GSA record creation. Add a validation message 'TIN is required for all Group Service Agreement types' if TIN__c is blank at submission. Deploy corrected OmniScript version to ACME-UAT2 and execute regression test suite RT-GSA-001 through RT-GSA-012.

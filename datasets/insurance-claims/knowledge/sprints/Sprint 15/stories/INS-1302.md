## Description

**Bug Report – UAT Sprint 15**

**Summary:** When a new Credentialing_Case__c is created for a Provider__c whose NPI (Provider__c.NPI__c) begins with one or more leading zeros (e.g., '0123456789'), the auto-assignment Flow fails silently and the case is left with Owner = Integration User rather than being routed to the correct Credentialing Specialist queue.

**Steps to Reproduce:**
1. Navigate to a Provider__c record where NPI__c = '0198765432' (leading zero).
2. Manually trigger 'Initiate Credentialing' quick action or allow CAQH inbound integration to create a Credentialing_Case__c record.
3. Observe Credentialing_Case__c.OwnerId after record creation.
4. Expected: Case is assigned to 'Credentialing_Specialists' queue based on Provider__c.State__c routing logic.
5. Actual: Credentialing_Case__c.OwnerId = Integration User (005xx000001XXXXX); no assignment rule fires.

**Root Cause Hypothesis:** The assignment Flow uses a Text formula to cast NPI__c to a Number for NPPES lookup callout construction. NPIs with leading zeros are truncated (e.g., '0198765432' becomes '198765432'), causing the NPPES validation callout to return HTTP 404, which triggers an uncaught fault path that bypasses the assignment logic entirely.

**Environment:** UAT Sandbox (CS92)
**Frequency:** Reproducible 100% with leading-zero NPIs; approximately 340 affected Provider records identified in UAT data.

## Acceptance Criteria

As per Description.

Additionally: Given a fix is deployed, when a Credentialing_Case__c is created for any Provider__c.NPI__c value (including leading zeros), then Credentialing_Case__c.OwnerId must reflect the correct queue within the same transaction and Credentialing_Case__c.Assignment_Status__c must be set to 'Assigned'.

## Solution

Fix the NPI-to-Number cast in the Record-Triggered Flow 'Credentialing_AutoAssign_Flow' by replacing the numeric formula variable with a Text variable throughout the NPPES callout URL construction step. Update the HTTP callout action to pass NPI as a zero-padded 10-character string. Add a fault path handler on the NPPES callout element that logs to Integration_Error_Log__c but does NOT halt assignment logic — assignment must proceed using Provider__c.State__c even if NPPES validation is unavailable. Add Flow unit tests covering NPI values: '0123456789', '0000000001', and a standard '1234567890'. Regression test all 340 affected UAT records via Data Loader re-trigger script.

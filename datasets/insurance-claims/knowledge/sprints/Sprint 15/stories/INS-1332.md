## Description

As a Claims Operations Manager, I need an Agentforce-powered AI triage assistant embedded in the FNOL intake OmniScript so that claims are automatically categorized by complexity and severity, preliminary fraud indicators are surfaced to adjusters in real time, and low-complexity claims are routed for straight-through processing — reducing average handle time and improving SIU referral accuracy.

This story covers the Agentforce agent configuration, topic/action definitions, and integration with the existing FNOL OmniScript (OS_FNOL_Intake_v3) built in INS-1287.

## Acceptance Criteria

Given a Claims Adjuster is completing the FNOL OmniScript for a new Claim__c,
When the adjuster reaches the 'Triage' step,
Then the Agentforce Claims Triage Agent is invoked with the in-progress claim data (Claim__c.Loss_Type__c, Claim__c.Reported_Amount__c, Claim__c.Loss_Date__c, Policy__c.LOB__c) and returns a triage recommendation within 5 seconds.

Given the Agentforce agent processes the FNOL data,
When the AI model evaluates the claim,
Then a Claim__c.Triage_Category__c picklist value ('Straight-Through', 'Standard Review', 'Complex Review', 'SIU Referral') is populated on the Claim__c record based on the agent's response.

Given the agent identifies two or more fraud indicators (e.g., loss date within 30 days of policy inception, reported amount > 150% of policy limit, provider flagged in Fee_Schedule__c.Fraud_Watch__c = true),
When the triage response is returned,
Then Claim__c.SIU_Flag__c is set to true, a Chatter post is created on the Claim__c record tagging the SIU queue (@SIU_Review_Queue) with the AI-generated fraud indicator summary, and Claim__c.Triage_Category__c is forced to 'SIU Referral' regardless of other scoring.

Given Claim__c.Triage_Category__c = 'Straight-Through',
When the FNOL OmniScript completes,
Then the Claim__c is auto-assigned to the 'Auto-Adjudication' queue and Claim__c.Status__c is set to 'In Adjudication' without requiring manual assignment.

Given the Agentforce agent is unavailable or returns an error,
When the triage step is reached,
Then the OmniScript gracefully degrades — displaying a warning banner to the adjuster and defaulting Claim__c.Triage_Category__c = 'Standard Review' — without blocking FNOL submission.

-Given the agent returns a confidence score below 60%, the claim shall be auto-escalated to a senior adjuster.- (Deprecated — confidence threshold logic deferred to Phase 3 per product decision INS-1301)

## Solution

Configure Agentforce Agent 'Claims_Triage_Assistant' in Agent Builder with a dedicated Topic 'FNOL_Triage' and custom Actions: (1) 'EvaluateClaimTriage' — Apex @InvocableMethod in ClaimsTriageAgentAction.cls that assembles claim context and calls the Einstein generative AI API via named credential; (2) 'FlagSIUIndicators' — queries fraud indicator rules stored in custom metadata 'SIU_Fraud_Rule__mdt' and cross-references Provider__c fraud watch flag. Integrate agent invocation into OS_FNOL_Intake_v3 via an OmniScript Integration Procedure step (IP_InvokeClaimsTriage) at the Triage step. Graceful degradation handled by a try/catch in the Integration Procedure with a fallback Set Values element. Chatter post created via ConnectApi.ChatterFeeds in a separate Apex future method to avoid mixed DML. Agent configuration gated behind 'Agentforce_Claims_User' Permission Set.

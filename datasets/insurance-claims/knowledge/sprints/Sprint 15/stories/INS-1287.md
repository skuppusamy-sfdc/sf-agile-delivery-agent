## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces the provider's active PNA contract status, assigned Fee Schedule tier, and upcoming credentialing expiration date, so that I can make informed network participation and outreach decisions without navigating across multiple related lists and objects.

Extends INS-1256 (Provider 360 base FlexCard scaffold).

## Acceptance Criteria

Given a Provider__c record with at least one active Provider_Network_Agreement__c,
When a Provider Relations Specialist opens the record page,
Then the FlexCard header displays Provider__c.NPI__c, Provider__c.TIN__c, and Provider__c.Network_Status__c with a color-coded badge (Green = Active, Yellow = Pending, Red = Terminated).

Given the provider has an associated Fee_Schedule__c record,
When the FlexCard loads,
Then the 'Contract & Fees' panel shows Fee_Schedule__c.Tier__c, Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.Expiration_Date__c, and a hyperlink to the parent PNA record.

Given a Credentialing_Case__c record exists with Credentialing_Case__c.Status__c = 'Active' and Credentialing_Case__c.Expiration_Date__c is within 90 days of today,
When the FlexCard renders,
Then a warning banner displays 'Credentialing expires in [N] days — Action Required' and the Credentialing_Case__c.Id is linked.

Given the provider has NO active Provider_Network_Agreement__c,
When the FlexCard loads,
Then the 'Contract & Fees' panel displays 'No Active Contract' and a 'Initiate PNA' quick-action button is visible.

-Given the provider is flagged as out-of-network, suppress all fee schedule data from the FlexCard view- (superseded — out-of-network providers now display fee schedule data in read-only mode per compliance review INS-1271).

Given a user without the 'Provider_Network_Specialist' Permission Set Group,
When the FlexCard loads,
Then the 'Initiate PNA' and 'Edit Fee Schedule' action buttons are hidden and the card renders in read-only mode.

## Solution

Build OmniStudio FlexCard 'Provider360_Summary' using DataRaptor Extract 'DR_Get_ProviderNetworkSummary' to fetch related PNA, Fee_Schedule__c, and Credentialing_Case__c data in a single SOQL-optimized query. Use FlexCard conditional states to drive badge color via a State configuration mapping Network_Status__c values to CSS tokens. Surface the credentialing expiry warning via a Flyout child card bound to the Credentialing_Case__c dataset. Quick-action buttons ('Initiate PNA', 'Edit Fee Schedule') are wired to OmniScript 'OS_InitiatePNA_v2' and a standard Edit action respectively, and visibility is controlled by a FlexCard Action Condition referencing the running user's Permission Set Group via a DataRaptor that queries PermissionSetAssignment. Deploy to the Provider__c Lightning Record Page via App Builder with audience targeting for Provider Relations profile.

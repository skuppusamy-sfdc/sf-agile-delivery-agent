## Description

As a Claims Operations Manager, I need the system to automatically apply Coordination of Benefits (COB) priority rules across Claim_Line__c records when a secondary payer is identified on the associated Policy__c, so that payment calculations reflect correct primary/secondary liability splits and prevent overpayment.

This story extends INS-1321 (Primary Payer adjudication engine) by layering COB logic on top of the existing adjudication pipeline.

## Acceptance Criteria

Given a Claim__c record with COB_Indicator__c = true and a linked secondary Policy__c record,
When the claim moves to Status__c = 'Pending Adjudication',
Then the system must identify the primary payer via Policy__c.COB_Order__c = 1 and calculate Claim_Line__c.Primary_Allowed_Amount__c before passing residual liability to the secondary payer.

Given Claim_Line__c records where Primary_Paid_Amount__c has been populated by the primary payer ERA,
When secondary adjudication runs,
Then Claim_Line__c.Secondary_Liability__c = Claim_Line__c.Patient_Responsibility__c - Claim_Line__c.Primary_Paid_Amount__c, and Secondary_Liability__c must never be negative.

Given a COB scenario where both payers share the same TIN on Provider__c,
When adjudication completes,
Then a validation error must surface: 'COB configuration invalid: duplicate TIN detected across payer records' and Claim__c.Status__c must revert to 'Pended'.

Given a completed COB adjudication,
When the EOB is generated,
Then Explanation_of_Benefits__c.COB_Savings__c must equal the sum of all Claim_Line__c.Primary_Paid_Amount__c values, and EOB__c.Secondary_Payer_Name__c must be populated from the secondary Policy__c.Carrier_Name__c.

-Given a Medicare Advantage plan as secondary payer, apply MCO-specific COB override flag on Policy__c.MCO_COB_Override__c — deprecated in favor of standard COB_Order__c logic per compliance review 2024-11-01-

## Solution

Implement a Record-Triggered Flow on Claim__c firing on Status__c change to 'Pending Adjudication'. Flow invokes an Apex invocable method (COBPriorityCalculator.calculateCOB()) that queries related Policy__c records ordered by COB_Order__c, iterates Claim_Line__c children, and writes Primary_Allowed_Amount__c and Secondary_Liability__c. A Platform Event (COB_Adjudication_Complete__e) is published upon successful calculation to trigger downstream EOB generation via a second subscriber Flow. Custom metadata type COB_Rules__mdt stores payer-priority override configurations to avoid hardcoding. Unit tests cover negative liability guard, duplicate TIN validation, and standard two-payer scenario.

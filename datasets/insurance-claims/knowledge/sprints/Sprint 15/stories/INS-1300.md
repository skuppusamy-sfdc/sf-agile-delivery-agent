## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record that surfaces active Provider_Network_Agreement__c (PNA) records, associated Fee_Schedule__c line items, and current credentialing status from Credentialing_Case__c so that I can assess a provider's full network participation without navigating multiple related lists or running separate reports.

## Acceptance Criteria

Given a Provider__c record with at least one active PNA, when the Provider 360 FlexCard loads, then the card must display PNA.Contract_Effective_Date__c, PNA.Contract_Expiration_Date__c, PNA.Network_Tier__c, and PNA.Status__c = 'Active' in a sortable child card section.

Given a PNA record linked to a Fee_Schedule__c, when the specialist expands the PNA child card, then Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Allowed_Amount__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.LOB__c must be displayed in a tabular DataRaptor-driven grid limited to 50 rows by default with a 'Load More' action.

Given a Provider__c record with an open Credentialing_Case__c, when the FlexCard renders, then a status badge must reflect Credentialing_Case__c.Status__c using conditional rendering: 'In Review' = yellow, 'Approved' = green, 'Suspended' = red.

Given a Provider__c whose most recent PNA has PNA.Contract_Expiration_Date__c within 90 days of today, when the FlexCard loads, then a warning banner reading 'Contract expiring soon – review required' must appear at the top of the card.

-Given the FlexCard must also display Group_Service_Agreement__c (GSA) summary – descoped to INS-1315 per Sprint 14 review.-

Given a user without the 'Provider_Network_Read' Permission Set Group, when the FlexCard loads, then PNA financial fields (Fee_Schedule__c.Allowed_Amount__c) must be masked and display '***' in place of values.

## Solution

Build OmniStudio FlexCard 'Provider360_Card' with a primary DataRaptor Extract sourcing Provider__c, child DataRaptor Turbo Extract for PNA records filtered by Status__c = 'Active', and a nested DataRaptor Extract for Fee_Schedule__c rows. Use FlexCard conditional styling states for credentialing badge colors. Implement a 'Load More' action via an Integration Procedure call with offset pagination. Apply field-level masking using a custom Apex REST action checked against the running user's Permission Set Group membership. Deploy via OmniStudio version-controlled package.

## Description

As a Credentialing Specialist, I need an automated reminder workflow that monitors Credentialing_Case__c records for upcoming credential expiration dates and sends tiered reminder notifications to the provider and internal credentialing team at 90, 60, and 30 days prior to expiration so that re-attestation with CAQH is completed on time and provider network participation is not interrupted.

## Acceptance Criteria

Given a Credentialing_Case__c record where Credentialing_Case__c.Credential_Expiry_Date__c is not null and Credentialing_Case__c.Status__c = 'Active',
When the scheduled daily batch job runs and the expiry date is exactly 90 days from today,
Then an email notification using template 'Credential_Expiry_90Day' is sent to the linked Provider__c.Email and a Task is created on the Credentialing_Case__c with Subject = '90-Day Credential Expiry Reminder', Status = 'Open', ActivityDate = TODAY(), OwnerId = the assigned Credentialing Specialist.

Given the same Credentialing_Case__c has already received a 90-day reminder,
When the scheduled job runs and the expiry date is exactly 60 days from today,
Then a second email using template 'Credential_Expiry_60Day' is sent and Credentialing_Case__c.Reminder_Stage__c is updated to '60-Day'.

Given the expiry date is exactly 30 days from today,
When the scheduled job runs,
Then the email template 'Credential_Expiry_30Day' is sent, Credentialing_Case__c.Reminder_Stage__c = '30-Day', and the Credentialing_Case__c.Priority__c is escalated to 'High'.

Given the provider completes CAQH re-attestation and the CAQH integration updates Credentialing_Case__c.CAQH_Attestation_Date__c,
When the new attestation date is populated and is after the previous expiry date,
Then all open reminder Tasks linked to the case are auto-closed (Status = 'Completed') and Credentialing_Case__c.Reminder_Stage__c is reset to null.

Given a Credentialing_Case__c has passed its Credential_Expiry_Date__c with no re-attestation,
When the scheduled job detects the overdue condition,
Then Credentialing_Case__c.Status__c is updated to 'Expired', a Platform Event 'Credential_Expired__e' is published, and the linked PNA__c.Status__c is flagged as 'Pending Review'.

## Solution

Implement a Schedulable Apex class 'CredentialExpiryReminderBatch' implementing Database.Batchable and Schedulable interfaces, scheduled to run daily at 6:00 AM. Batch queries Credentialing_Case__c with Status__c = 'Active' and Credential_Expiry_Date__c within the next 90 days. Logic evaluates the day-delta and Reminder_Stage__c to determine which reminder tier to fire. Email alerts sent via Messaging.sendEmail with OrgWideEmailAddress. Task creation and field updates performed in bulkified DML. CAQH attestation reset handled by a separate Record-Triggered Flow on Credentialing_Case__c triggering on CAQH_Attestation_Date__c change. Platform Event published via Apex EventBus.publish() on expiry detection.

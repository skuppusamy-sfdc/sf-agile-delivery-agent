## Description

**Bug Report — UAT Phase 2**

**Summary:** The Explanation_of_Benefits__c (EOB) PDF generated via Conga CLM template fails with a generic error when the parent Claim__c record has more than 20 associated Claim_Line__c records. Claims with ≤20 lines generate successfully.

**Steps to Reproduce:**
1. Navigate to any Claim__c record where the count of related Claim_Line__c records > 20 (e.g., Claim__c ID: CLM-0094821 in UAT sandbox).
2. Click the 'Generate EOB' quick action button on the Claim__c record page.
3. Observe the Conga CLM merge job initiated via the 'Generate_EOB_OmniScript' OmniScript.
4. Wait for the async Conga callback.

**Expected Result:** EOB PDF is generated successfully and attached to the Explanation_of_Benefits__c record as a ContentVersion file. EOB__c.Status__c is set to 'Generated'. A confirmation toast appears.

**Actual Result:** An unhandled exception is thrown: 'Conga CLM: Template merge failed — row limit exceeded on sub-table CLAIM_LINES.' EOB__c.Status__c remains 'Pending'. No PDF is attached. No user-facing error message is displayed; the OmniScript silently completes.

**Environment:** UAT Sandbox (CS112)
**Frequency:** 100% reproducible with >20 claim lines
**Regression:** Was working in Sprint 13; likely broken by INS-1341 which added additional Claim_Line__c repeating table rows to the Conga template.

**Workaround:** None currently available for claims with high line counts.

## Acceptance Criteria

As per Description. Additionally: EOB generation must succeed for Claim__c records with up to 150 Claim_Line__c records without error. If generation fails for any reason, EOB__c.Status__c must be set to 'Error' and EOB__c.Error_Message__c must be populated with a human-readable description.

## Solution

Root cause: Conga CLM template sub-table for CLAIM_LINES has a hardcoded row limit of 20 inherited from the base template. Fix requires: (1) Update Conga template 'EOB_Standard_v3' to remove the sub-table row cap — configure repeating section with no upper bound. (2) Update the DataRaptor Extract feeding the OmniScript to paginate Claim_Line__c retrieval in batches of 50 if count > 50, merging results before Conga submission. (3) Add explicit error handling in the 'Generate_EOB_OmniScript' error path to set EOB__c.Status__c = 'Error' and populate EOB__c.Error_Message__c via a DataRaptor Transform. (4) Add a unit test asserting successful merge for a Claim__c with 25 Claim_Line__c records.

## Description

As a Claims Operations Manager, I need an Agentforce-powered next-best-action panel embedded on the Claim__c record page during FNOL triage so that claims handlers receive AI-driven recommendations (e.g., SIU referral, COB investigation, fast-track approval) based on claim attributes, policy history, and provider risk profile, reducing average FNOL handle time by 20%.

The panel should display ranked recommendations with confidence scores and allow the handler to accept, dismiss, or escalate each suggestion with a single click.

## Acceptance Criteria

Given a Claim__c record with Claim__c.Status__c = 'FNOL' is opened by a user with the 'Claims_Handler' Permission Set Group,
When the Agentforce NBA panel LWC ('agentforceClaimsNBA') loads,
Then the panel invokes the Agentforce Action 'ClaimTriageRecommendation' passing Claim__c.Id, Claim__c.LOB__c, Claim__c.Total_Billed_Amount__c, Claim__c.Provider__c, and Policy__c.Claim_History_Count__c.

Given the Agentforce model returns recommendations,
When the panel renders,
Then up to 5 ranked recommendations are displayed, each showing: recommendation type (e.g., 'SIU Referral', 'COB Investigation', 'Fast Track'), confidence score as a percentage, and a one-sentence rationale string.

Given the handler clicks 'Accept' on a recommendation of type 'SIU Referral',
When the action is confirmed,
Then Claim__c.SIU_Referral_Flag__c is set to TRUE, a new Case is created with RecordType = 'SIU_Investigation', linked to the Claim__c, and Claim__c.Status__c transitions to 'SIU Pending'.

Given the handler clicks 'Accept' on a recommendation of type 'Fast Track',
When the action is confirmed,
Then Claim__c.Fast_Track_Flag__c = TRUE and Claim__c.Priority__c = 'High' are set, and the claim is assigned to the 'Fast Track Claims Queue' via an Omni-Channel routing update.

Given the handler clicks 'Dismiss' on any recommendation,
When the dismissal is logged,
Then a Claim_NBA_Feedback__c record is created with fields: Claim__c (lookup), Recommendation_Type__c, Confidence_Score__c, Handler_Action__c = 'Dismissed', and Dismissed_At__c = NOW() for model retraining telemetry.

Given the Agentforce service is unavailable (timeout > 5s),
When the panel fails to load recommendations,
Then a graceful fallback message 'AI recommendations temporarily unavailable — please use manual triage checklist' is displayed and no error is thrown to the console.

## Solution

LWC 'agentforceClaimsNBA' embedded in the Claim__c Lightning Record Page via App Builder in a collapsible sidebar region. Component calls an Apex controller 'AgentforceClaimsNBAController.getRecommendations(claimId)' which invokes the Agentforce Agent Action API using a Named Credential. Recommendations returned as a typed Apex inner class list and serialized to JSON for the LWC. Accept/Dismiss actions handled via separate Apex methods that perform DML and publish a 'ClaimNBAAction__e' Platform Event for downstream telemetry. Fallback implemented via try/catch with a 5-second HTTP callout timeout on the Named Credential. Claim_NBA_Feedback__c is a new custom object with a Master-Detail to Claim__c, included in the Phase 2 metadata package. Model training pipeline for feedback data coordinated with the Data Science team outside Salesforce scope.

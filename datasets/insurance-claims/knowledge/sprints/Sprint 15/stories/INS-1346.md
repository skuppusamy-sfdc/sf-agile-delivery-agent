## Description

**Bug Report — UAT Sprint 15**

**Summary:** When a Claims Operations Manager uses the 'Close Claims' bulk list view action to close multiple Claim__c records simultaneously, the child Claim_Line__c records retain their previous Status__c value and do not update to 'Adjudicated', breaking downstream EOB generation and ERA reconciliation.

**Environment:** Acme Insurance Full Sandbox (FS1)

**Steps to Reproduce:**
1. Log in as a user with 'Claims_Ops_Manager' Permission Set Group.
2. Navigate to the Claims list view 'Open Claims – Bulk Processing'.
3. Select 10 or more Claim__c records where Claim__c.Status__c = 'Pending Adjudication'.
4. Click the 'Close Claims' list view action button.
5. Confirm the bulk close dialog.
6. Observe Claim__c.Status__c updates to 'Closed' on all selected records.
7. Navigate to any of the closed Claim__c records and inspect related Claim_Line__c records.

**Expected Result:** All child Claim_Line__c records where Claim_Line__c.Status__c = 'Pending' should update to 'Adjudicated' and Claim_Line__c.Adjudication_Date__c should be set to today's date.

**Actual Result:** Claim_Line__c.Status__c remains 'Pending' and Claim_Line__c.Adjudication_Date__c is null. No error message is displayed to the user.

**Frequency:** 100% reproducible with 10+ records; single-record close via record detail page works correctly.

**Root Cause Hypothesis:** The Record-Triggered Flow on Claim__c that updates child Claim_Line__c records likely hits the 200-record DML limit per transaction when bulk processing exceeds governor limits, causing silent failure.

## Acceptance Criteria

As per Description. Fix must be verified in FS1 with a bulk selection of 50 Claim__c records. All child Claim_Line__c records must reflect Status__c = 'Adjudicated' and Adjudication_Date__c = TODAY() after bulk close. No Platform Errors should appear in the debug log.

## Solution

Refactor the existing Record-Triggered Flow on Claim__c 'Update_ClaimLines_On_Close' to use an Apex invocable action instead of a Flow loop with individual DML updates. The Apex action 'BulkClaimLineAdjudicationHandler' will receive a List<Id> of Claim__c IDs, query all related Claim_Line__c records in a single SOQL query, perform a bulk update in one DML statement, and handle partial failure with Database.update(records, false) logging failures to a custom Error_Log__c object. Update the list view action's underlying Quick Action to invoke the corrected Flow. Add a Process Exception handler to surface errors to the running user via a Toast notification.

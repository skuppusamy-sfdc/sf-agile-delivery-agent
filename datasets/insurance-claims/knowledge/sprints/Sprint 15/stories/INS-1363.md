## Description

As a Claims Operations Manager, I need the FNOL/Claim intake OmniScript to capture Coordination of Benefits (COB) information — specifically whether a secondary payer exists, the secondary payer name, and payer priority — so that adjudication logic can correctly sequence primary and secondary payment calculations and reduce manual rework on COB claims.

## Acceptance Criteria

Given a claims intake OmniScript ('FNOL_Claim_Intake_OS') is being completed,
When the claimant indicates a secondary insurance exists (COB toggle = TRUE),
Then the following fields become visible and required: Claim__c.COB_Secondary_Payer_Name__c (text), Claim__c.COB_Secondary_Policy_Number__c (text), Claim__c.COB_Payer_Priority__c (picklist: 'Primary', 'Secondary', 'Tertiary'), and Claim__c.COB_Secondary_TPA__c (lookup to Account).

Given the COB toggle = FALSE,
When the intake form is submitted,
Then Claim__c.COB_Indicator__c = FALSE, all COB secondary fields remain null, and no validation errors are thrown.

Given the intake form is submitted with COB_Indicator__c = TRUE,
When Claim__c is created,
Then Claim__c.COB_Indicator__c = TRUE and Claim__c.Status__c is set to 'COB Review Pending' rather than the standard 'Submitted'.

Given a Claim__c with COB_Indicator__c = TRUE enters adjudication,
When the adjudication engine evaluates payment,
Then the system reads Claim__c.COB_Payer_Priority__c to determine payment sequencing and logs this in the Claim__c.Adjudication_Notes__c field.

Given a user with the 'Claims_Read_Only' Permission Set Group,
When they view a Claim__c record with COB data,
Then all COB fields are visible but not editable.

## Solution

Extend the existing 'FNOL_Claim_Intake_OS' OmniScript with a new Step ('COB Information') containing a toggle element bound to Claim__c.COB_Indicator__c. Conditional visibility on COB sub-fields controlled by OmniScript conditional view rules. DataRaptor Upsert updated to map new COB fields on Claim__c save. Status defaulting logic handled by a Record-Triggered Flow ('Claim_COB_Status_Default_Flow') firing before-save on create, checking COB_Indicator__c and overriding Status__c accordingly. Field-level security for 'Claims_Read_Only' PSG configured via Profile/Permission Set field permissions. New fields added to the Claim__c page layout and the COB-specific Lightning record page section.

## Description

As a Claims Operations Manager, I need an Agentforce-powered suggested-action panel embedded on the Claim__c record page for newly submitted FNOL claims, so that claims adjusters receive AI-generated triage recommendations (e.g., SIU referral likelihood, fast-track eligibility, missing documentation prompts) in real time, reducing average handle time and improving adjudication accuracy.

This story covers the Agentforce Agent configuration, the LWC panel integration, and the Prompt Template authoring for FNOL triage. Data grounding will leverage Claim__c, Policy__c, and historical Payment__c records.

## Acceptance Criteria

Given a Claim__c record with Claim__c.Status__c = 'FNOL Submitted' is opened by a user with the 'Claims_Adjuster' Permission Set Group,
When the record page loads,
Then the 'AI Triage Assistant' LWC panel is visible in the right utility bar and displays a loading spinner while the Agentforce inference call completes.

Given the Agentforce inference completes successfully,
When the panel renders,
Then it displays up to 3 ranked suggested actions from the following set: ['Refer to SIU', 'Fast-Track Eligible', 'Request Missing Documentation', 'Assign to Complex Claims Team', 'Auto-Adjudicate'] each with a confidence score percentage and a one-sentence rationale grounded in Claim__c field values.

Given the adjuster clicks 'Accept' on a suggested action of 'Refer to SIU',
When the action is confirmed,
Then Claim__c.SIU_Referral_Flag__c is set to true, Claim__c.Status__c updates to 'SIU Review', and a new Case is created linked to the Claim__c with Subject = 'SIU Referral — [Claim__c.Name]'.

Given the adjuster clicks 'Accept' on 'Fast-Track Eligible',
When the action is confirmed,
Then Claim__c.Fast_Track__c = true and Claim__c.Target_Resolution_Date__c = TODAY() + 5.

Given the Agentforce inference call fails or times out (>10 seconds),
When the panel detects the failure,
Then a graceful fallback message is displayed: 'AI recommendations temporarily unavailable. Please proceed with manual triage.' and no Claim__c fields are modified.

Given a Claim__c record with Status__c NOT equal to 'FNOL Submitted',
When the record page loads,
Then the AI Triage Assistant panel is hidden and does not invoke the Agentforce API.

## Solution

Configure an Agentforce Service Agent 'Claims_Triage_Agent' with a custom Agent Action 'FNOLTriageAction' backed by an Apex class 'AgentforceTriageActionHandler' that queries Claim__c, related Policy__c, and last 12 months of Payment__c history as grounding context. Author a Prompt Template 'FNOL_Triage_Prompt_v1' using the Prompt Builder with Flex Template type, injecting grounding data via merge fields. Build LWC 'agentforceTriagePanel' that calls the Agentforce API via a wire adapter on record load, conditionally renders based on Claim__c.Status__c using @wire(getRecord), and dispatches accepted actions via an Apex controller 'TriageActionDispatcher.executeAction()'. The LWC is placed in the Lightning Record Page right sidebar using App Builder with a component visibility filter: Claim__c.Status__c Equals 'FNOL Submitted'. Permission to invoke Agentforce is gated by a custom Permission 'Agentforce_Claims_Triage' included in the Claims_Adjuster Permission Set Group.

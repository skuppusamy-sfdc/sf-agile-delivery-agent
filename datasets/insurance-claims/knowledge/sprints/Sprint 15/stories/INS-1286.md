## Description

**Bug Report — UAT Phase 2 Defect**

When a Claim__c record has Coordination_of_Benefits__c = true and a secondary payer submits an ERA payment, the associated Claim_Line__c records are not updating their Adjudication_Status__c field from 'Pending Secondary' to 'Adjudicated'. The EOB record is created but reflects stale line-level data.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c where COB_Flag__c = true and Primary_Payment__c has been posted.
2. Simulate an inbound ERA 835 file for the secondary payer via the DataIntegration sandbox endpoint.
3. Observe that Payment__c record is created successfully with correct Amount_Paid__c.
4. Navigate to the related Explanation_of_Benefits__c record.
5. Expand the Claim Line Items related list.

**Expected:** Each Claim_Line__c.Adjudication_Status__c = 'Adjudicated'; EOB_Line_Amount__c reflects the net patient responsibility after both payer contributions.

**Actual:** Claim_Line__c.Adjudication_Status__c remains 'Pending Secondary'; EOB_Line_Amount__c shows the full billed charge rather than the post-COB patient responsibility amount.

**Environment:** Sandbox ACME-UAT2
**Frequency:** Reproducible 100% with multi-payer ERA files; single-payer ERA unaffected.

## Solution

Root cause identified in the Record-Triggered Flow 'Update_ClaimLine_On_Payment_Posted' — the entry condition checks Payment__c.Payer_Sequence__c = 'Primary' only, excluding secondary payer payment records from triggering the downstream Claim_Line__c update logic. Fix: add a second entry condition branch for Payer_Sequence__c = 'Secondary' with a separate Update Records element that sets Adjudication_Status__c = 'Adjudicated' and recalculates Patient_Responsibility__c using the COB_Calculation_Formula__c formula field. Additionally, update the Apex invocable method 'EOBLineItemSyncService.syncLinesFromPayment()' to pass the correct payer sequence context so EOB line amounts are derived from the net post-COB figure. Deploy updated Flow version and Apex class to ACME-UAT2 for regression.

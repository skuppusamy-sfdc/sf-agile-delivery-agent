## Description

**Bug Report — UAT Round 3**

When a Claims Operations Manager applies a COB (Coordination of Benefits) override on a Claim_Line__c record, the parent Explanation_of_Benefits__c record does not reflect the updated adjudicated amount and the EOB Status__c remains stuck at 'Pending' instead of transitioning to 'Adjudicated'.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record with Status__c = 'In Adjudication' and at least two Claim_Line__c child records.
2. On one Claim_Line__c, set COB_Override__c = TRUE and manually enter COB_Adjusted_Amount__c = 150.00.
3. Save the Claim_Line__c record.
4. Navigate to the related Explanation_of_Benefits__c record.
5. Observe EOB_Status__c and Total_Adjudicated_Amount__c fields.

**Expected Result:** EOB_Status__c transitions to 'Adjudicated' and Total_Adjudicated_Amount__c rolls up the corrected COB-adjusted line amounts.

**Actual Result:** EOB_Status__c remains 'Pending' and Total_Adjudicated_Amount__c still reflects the pre-override value. Roll-up summary field appears to be bypassed when COB_Override__c is set via the UI.

**Environment:** Acme UAT Sandbox (Sprint 15 refresh)
**Severity:** High — blocks payment disbursement for COB claims in regression suite.

## Solution

Root cause identified as a Record-Triggered Flow on Claim_Line__c that fires on 'Update' but has an entry condition filtering out records where COB_Override__c = TRUE — likely a legacy filter added in INS-1198 that was never removed. Fix: remove the erroneous entry condition from the 'Update EOB on Line Adjudication' flow so COB override records are included. Additionally, confirm the Roll-Up Summary field Total_Adjudicated_Amount__c on Explanation_of_Benefits__c correctly references COB_Adjusted_Amount__c when COB_Override__c is TRUE; if not, replace with an Apex-managed aggregation via a trigger helper class (ClaimLineRollupService.cls) to handle conditional field sourcing. Deploy fix to UAT and re-execute regression script COB-TC-07.

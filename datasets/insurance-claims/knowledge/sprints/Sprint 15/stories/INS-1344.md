## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record that surfaces the provider's active Provider_Network_Agreement__c (PNA) details, associated Fee_Schedule__c line summaries, and NPI/TIN verification status so that I can assess network participation and reimbursement configuration without navigating multiple related lists.

## Acceptance Criteria

Given a Provider (Person Account) record is open in the Provider Relations console app,
When the Provider 360 FlexCard renders,
Then the card displays the provider's NPI (Provider__c.NPI__c), TIN (Provider__c.TIN__c), and CAQH profile status (Provider__c.CAQH_Status__c) in the header band.

Given the provider has one or more Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active',
When the FlexCard PNA child card renders,
Then it lists each active PNA showing PNA__c.Network_Name__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, and PNA__c.LOB__c.

Given an active PNA has an associated Fee_Schedule__c record,
When the user expands the PNA row,
Then the nested Fee Schedule section displays Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, and a count of active CPT code lines (Claim_Line__c count where Status__c = 'Active').

Given the provider has NO active PNA records,
When the FlexCard renders the PNA section,
Then a 'No Active Network Agreements' message is displayed with a quick-action button to initiate a new PNA via OmniScript.

-Given the FlexCard displays Group_Service_Agreement__c (GSA) data inline, Then GSA fields are shown in the PNA band- (deprecated — GSA moved to separate tab per INS-1331)

Given the FlexCard is viewed by a user without the 'Provider_Network_Read' Permission Set Group,
When the card attempts to load PNA data,
Then PNA and Fee Schedule sections are hidden and a 'Insufficient Access' notice is shown.

## Solution

Build OmniStudio FlexCard 'Provider360_PNA_Card' with a primary DataRaptor Extract pulling from Provider (Account), with child DataRaptor Turbo Extracts for PNA__c (filtered Status__c = 'Active') and Fee_Schedule__c. Use a Flyout card action to trigger the 'PNA_Initiation' OmniScript for new agreements. Apply Conditional View visibility rules tied to the running user's Permission Set Group membership checked via a DataRaptor Transform. Deploy card via the Provider Relations Lightning App Page using the OmniStudio FlexCard component. Ensure SOQL in DataRaptors uses WITH SECURITY_ENFORCED.

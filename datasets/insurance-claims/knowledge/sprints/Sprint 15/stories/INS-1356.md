## Description

**Bug Report — UAT Cycle 3 Finding**

**Steps to Reproduce:**
1. Navigate to a Policyholder Account record where the associated Policy__c has 50+ Claim__c records with Claim__c.Status__c = 'Active' or 'Pending Adjudication'.
2. Click the 'Policyholder 360' tab on the Account Lightning Record Page.
3. Observe the dashboard component behavior.

**Expected Result:**
The Policyholder 360 LWC dashboard loads within 3 seconds, displaying a paginated claims summary, total billed YTD, and deductible accumulator.

**Actual Result:**
The LWC component renders a blank white panel with a browser console error: 'Maximum stack size exceeded' in policyholderDashboard.js line 247. The component never recovers without a full page refresh. Issue is consistent above 50 claims; below 50 the component loads correctly.

**Environment:** Full Sandbox (ACME-FS2) and Partial Sandbox (ACME-PS1)
**Frequency:** 100% reproducible above 50 active claims.
**Impact:** Affects approximately 340 policyholder records in UAT data set with high claim volumes.

## Acceptance Criteria

As per Description.

Additionally: After fix, the Policyholder 360 dashboard must load successfully for Policy__c records with up to 500 active Claim__c records, with initial render time under 3 seconds as measured by Lightning Component performance tooling. Claims list must implement server-side pagination returning a maximum of 20 records per page.

## Solution

Root cause is an infinite recursion in the LWC wire adapter — the @wire(getClaimsForPolicy) call is re-triggering on each reactive property update within the connected callback, causing a stack overflow when the dataset is large. Fix: refactor policyholderDashboard.js to use an imperative Apex call (PolicyholderDashboardController.getPagedClaims()) with explicit pagination parameters (pageSize=20, pageNumber) instead of a reactive wire. Implement server-side SOQL with LIMIT/OFFSET on Claim__c filtered by Policy__c and Status__c. Add a Lightning spinner during load. Update Apex controller with @AuraEnabled(cacheable=true) and ensure SOQL uses selective indexed fields (Policy__c, Status__c) to maintain query performance.

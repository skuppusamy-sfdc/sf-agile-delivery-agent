## Description

As a Provider Network Specialist, I need the system to automatically validate that any Fee_Schedule__c record linked to a Provider_Network_Agreement__c (PNA) has an Effective_Date__c that falls within the PNA's Contract_Start_Date__c and Contract_End_Date__c range at the time of save, so that we prevent downstream claim adjudication errors caused by misaligned fee schedule windows. Extends INS-1329.

## Acceptance Criteria

Given a user attempts to save a Provider_Network_Agreement__c record with a related Fee_Schedule__c,
When Fee_Schedule__c.Effective_Date__c is earlier than PNA.Contract_Start_Date__c,
Then the save is blocked and an error message reads: 'Fee Schedule effective date cannot precede the PNA contract start date.'

Given a user attempts to save a PNA with a related Fee_Schedule__c,
When Fee_Schedule__c.Expiration_Date__c is later than PNA.Contract_End_Date__c,
Then the save is blocked and an error message reads: 'Fee Schedule expiration date cannot extend beyond the PNA contract end date.'

Given a valid date range alignment,
When the PNA is saved,
Then PNA.Fee_Schedule_Validated__c (checkbox) is set to TRUE and a timestamp is written to PNA.Fee_Schedule_Validated_Date__c.

Given a PNA record where PNA.Contract_End_Date__c is null (open-ended contract),
When the save validation runs,
Then only the start date lower-bound check is enforced and the upper-bound check is skipped.

-Given a bulk import of PNA records via Data Loader, the validation fires synchronously (deprecated — async Platform Event pattern adopted per architecture review)-

## Solution

Implement a Record-Triggered Flow on Provider_Network_Agreement__c firing before-save on create and update. Flow retrieves related Fee_Schedule__c records via a Get Records element filtered by PNA lookup. Date comparison logic uses formula resources. Error messages surfaced via the Flow's 'Custom Error Message' element. PNA.Fee_Schedule_Validated__c and PNA.Fee_Schedule_Validated_Date__c updated via an Assignment element on the success path. Bulk import scenario handled by a separate asynchronous Apex Batch class that publishes a 'FeeScheduleValidation__e' Platform Event post-insert, consumed by a Platform Event-Triggered Flow.

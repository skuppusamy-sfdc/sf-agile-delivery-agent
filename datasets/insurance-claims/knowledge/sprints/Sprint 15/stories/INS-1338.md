## Description

As a Claims Operations Manager, I need the system to automatically generate an Electronic Remittance Advice (ERA) file in ANSI X12 835 format and create a linked Explanation_of_Benefits__c (EOB) record whenever a Claim__c transitions to 'Adjudicated' status, so that downstream Payment__c processing and provider remittance are triggered without manual intervention.

This story covers the Salesforce-side orchestration; the actual 835 file transmission to the clearinghouse is handled by the existing MuleSoft middleware layer.

## Acceptance Criteria

Given a Claim__c record where Claim__c.Status__c transitions from 'In Review' to 'Adjudicated',
When the Record-Triggered Flow fires on the Claim__c update,
Then a new Explanation_of_Benefits__c record is created with EOB.Claim__c = the adjudicated Claim__c Id, EOB.Member_ID__c populated from Claim__c.Policy__c.Member_ID__c, EOB.Total_Billed_Amount__c = sum of all child Claim_Line__c.Billed_Amount__c, EOB.Total_Allowed_Amount__c = sum of all child Claim_Line__c.Allowed_Amount__c, and EOB.Status__c = 'Pending Issuance'.

Given the EOB record is created successfully,
When the Payment__c record is generated,
Then Payment__c.EOB__c is populated with the new EOB Id, Payment__c.Payee_NPI__c = Claim__c.Rendering_Provider__c.NPI__c, Payment__c.Payment_Amount__c = EOB.Total_Allowed_Amount__c minus EOB.Member_Responsibility__c, and Payment__c.Status__c = 'Pending Approval'.

Given Payment__c.Payment_Amount__c > $10,000,
When the Payment__c record is created,
Then Payment__c.Requires_Manager_Approval__c = true and a Task is created assigned to the Claims Operations Manager queue.

Given the ERA generation Platform Event is published,
When MuleSoft consumes the ERA_Generation__e Platform Event,
Then the event payload includes Claim__c.Id, EOB.Id, Payment__c.Id, rendering provider TIN, and all Claim_Line__c CPT codes and adjudicated amounts.

-Given adjudication, the system should also print a paper EOB and mail to the member- (Deprecated: paper EOB replaced by digital EOB portal delivery per compliance decision CR-2024-089)

Given a Claim__c adjudication where COB applies (Claim__c.COB_Applicable__c = true),
When the EOB is generated,
Then EOB.Primary_Payer_Amount__c and EOB.Secondary_Payer_Amount__c are populated from Claim__c.COB_Primary_Paid__c and the net Payment__c.Payment_Amount__c reflects the coordination offset.

## Solution

Record-Triggered Flow on Claim__c (after save, optimized for performance) handles EOB and Payment__c creation via Apex action to allow bulkification. Apex class 'ClaimAdjudicationService' invokes a @InvocableMethod to create child records in a single DML transaction. Upon successful Payment__c insert, a Platform Event ERA_Generation__e is published with the required payload for MuleSoft consumption. COB logic handled in a separate Apex method 'COBCalculationService.calculateNetPayable()'. All records stamped with a Process_Source__c = 'Auto-Adjudication' audit field. Unit tests cover bulk scenario (200 simultaneous adjudications), COB scenario, and high-value approval routing.

## Description

As a Credentialing Specialist, I need Agentforce to proactively summarize expiring Credentialing_Case__c records and surface recommended next actions within the case feed, so that I can prioritize re-credentialing outreach without manually reviewing each record across a high-volume queue.

The solution should leverage the Agentforce Einstein Copilot action framework to generate natural-language summaries referencing CAQH verification status, NPI standing from NPPES, and outstanding document checklist items on each Credentialing_Case__c.

## Acceptance Criteria

Given a Credentialing_Case__c where Credentialing_Case__c.Expiry_Date__c is within 90 days of today and Status__c != 'Closed',
When the nightly scheduled Agentforce scan runs,
Then a Chatter post must be added to the case feed containing an AI-generated summary including: provider NPI, CAQH profile last verified date, days until expiry, and a bulleted list of incomplete checklist items from Credentialing_Checklist__c.

Given the AI summary is generated,
When the Credentialing Specialist reviews the case,
Then a 'Recommended Action' section must appear in the summary with one of three values: 'Initiate Re-credentialing', 'Awaiting Provider Documents', or 'Escalate to Medical Director', based on rules defined in Agentforce_Action_Rules__mdt.

Given a Credentialing_Case__c where CAQH verification returned a discrepancy (CAQH_Status__c = 'Discrepancy'),
When the AI summary is generated,
Then the summary must include a highlighted warning: 'CAQH discrepancy detected — manual review required' and the case must be assigned to the 'Credentialing_Escalation' queue.

Given the Agentforce action runs,
When NPPES API returns a non-200 response for the provider NPI lookup,
Then the summary must note 'NPPES verification unavailable — retry scheduled' and a Platform Event (Credentialing_Alert__e) must be published for monitoring.

Given a Credentialing Specialist with read-only access,
When they view the Agentforce-generated summary,
Then they must not see underlying prompt instructions or raw API response payloads — only the formatted summary output.

## Solution

Configure an Agentforce Einstein Copilot custom action (CredentialingExpirySummaryAction) using an Apex class that queries Credentialing_Case__c and related Credentialing_Checklist__c records. The Apex class calls an external NPPES REST callout and reads CAQH_Status__c from the provider record. Results are passed as a structured prompt context to the Einstein LLM via the Agent Action framework. A Scheduled Flow triggers the Agentforce scan nightly at 2 AM. Recommended action classification uses Custom Metadata Type (Agentforce_Action_Rules__mdt) to keep logic configurable. Chatter posts are created via ConnectApi.ChatterFeeds. Platform Event published on NPPES failure for ops monitoring dashboard.

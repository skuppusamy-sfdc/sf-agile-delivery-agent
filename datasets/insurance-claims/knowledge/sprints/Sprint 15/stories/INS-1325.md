## Description

As a Provider Network Specialist, I need the system to automatically generate a Provider_Change_Notification__c record and trigger a DocuSign envelope for the termination letter when a Network_Termination_Case__c is moved to 'Closed — Terminated' status so that the provider receives timely written notice and the audit trail is maintained in Salesforce without manual document creation.

## Acceptance Criteria

Given a Network_Termination_Case__c record's Status__c is updated to 'Closed — Terminated',
When the record is saved,
Then a new Provider_Change_Notification__c is created with PCN_Type__c = 'Termination', PCN_Status__c = 'Pending Delivery', PCN_Effective_Date__c = Network_Termination_Case__c.Termination_Effective_Date__c, and a lookup to the related Provider__c.

Given the Provider_Change_Notification__c is created,
When the record is inserted,
Then a DocuSign envelope is automatically sent to the email address on Provider__c.PersonEmail containing the pre-approved termination letter template (DocuSign Template ID stored in Custom Metadata Type 'DocuSign_Template_Mapping__mdt' with Template_Key__c = 'PROVIDER_TERMINATION').

Given the DocuSign envelope is sent,
When the provider completes or declines the envelope,
Then PCN_Status__c updates to 'Delivered' or 'Declined' respectively via the DocuSign status callback, and Network_Termination_Case__c.DocuSign_Envelope_Status__c is updated in sync.

Given the termination case is linked to an active Provider_Network_Agreement__c,
When PCN_Status__c = 'Delivered',
Then PNA.Status__c is automatically updated to 'Terminated' and PNA.Termination_Date__c is set to Network_Termination_Case__c.Termination_Effective_Date__c.

Given the termination affects a provider associated with open Claim__c records (Claim__c.Status__c NOT IN ('Closed','Denied')),
When the PNA is terminated,
Then a Warning_Flag__c = TRUE is set on each affected open claim and a notification is sent to the assigned adjuster's Chatter feed.

## Solution

Record-Triggered Flow on Network_Termination_Case__c (after update, when Status__c changes to 'Closed — Terminated') creates the PCN record and invokes an Apex action (DocuSignEnvelopeService.cls) to call the DocuSign eSignature REST API using a Named Credential. DocuSign callback handled by a public REST endpoint (DocuSignCallbackController.cls) that updates PCN and PNA records. PNA termination and open claim flagging handled in a separate after-update Apex Trigger on Provider_Change_Notification__c to keep flow complexity manageable. Custom Metadata Type 'DocuSign_Template_Mapping__mdt' used for template ID configuration to avoid hardcoding.

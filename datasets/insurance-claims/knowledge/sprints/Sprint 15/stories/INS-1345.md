## Description

As a Claims Operations Manager, I need the system to automatically generate a compliant Explanation_of_Benefits__c (EOB) PDF using Conga CLM and attach it to the parent Claim__c record when a Payment__c record transitions to Status__c = 'Disbursed' so that policyholders receive timely, accurate EOB documentation and the claims audit trail is complete for compliance reporting.

## Acceptance Criteria

Given a Payment__c record exists linked to a Claim__c record,
When Payment__c.Status__c is updated to 'Disbursed',
Then a Record-Triggered Flow fires and invokes the Conga Composer API to generate the EOB PDF using template 'EOB_Standard_v3'.

Given the Conga API call succeeds,
When the PDF is returned,
Then a new ContentVersion is created and linked to the Claim__c record via ContentDocumentLink, and a new Explanation_of_Benefits__c record is created with EOB__c.Claim__c = [parent Claim Id], EOB__c.Generated_Date__c = TODAY(), EOB__c.Status__c = 'Sent', and EOB__c.Document_Id__c = [ContentDocument Id].

Given the EOB__c record is created,
When the policyholder's email is present on Policy__c.Policyholder_Email__c,
Then an automated email with the EOB PDF attachment is sent via Salesforce Email-to-Case outbound template 'EOB_Policyholder_Notification'.

Given the Conga API call fails (timeout or error),
When the failure is detected in the Flow fault path,
Then a Platform Event 'EOB_Generation_Failure__e' is published with fields: Claim_Id__c, Payment_Id__c, Error_Message__c, and a Case is auto-created for the Claims Operations queue with Subject = 'EOB Generation Failed – [Claim Number]'.

Given a Payment__c is voided (Status__c updated to 'Voided') after an EOB was already generated,
When the void transition occurs,
Then the linked EOB__c.Status__c is updated to 'Cancelled' and a chatter post is added to the Claim__c record noting the void and EOB cancellation.

-Given COB is applicable on the Claim, Then a secondary EOB is generated for the secondary payer- (out of scope for Sprint 15 — tracked in INS-1358)

## Solution

Implement a Record-Triggered Flow on Payment__c, triggering on update when Status__c changes to 'Disbursed'. Flow calls an Apex invocable action 'CongaEOBGeneratorAction' which constructs the Conga API request with merge fields sourced from Claim__c, Claim_Line__c, Policy__c, and Provider__c. On success, Apex creates ContentVersion and EOB__c records. On failure, Apex publishes EOB_Generation_Failure__e Platform Event; a separate Platform Event-Triggered Flow creates the follow-up Case. Void logic handled by a second Record-Triggered Flow on Payment__c for the 'Voided' transition. All Apex written bulkified and covered at 90%+ test coverage.

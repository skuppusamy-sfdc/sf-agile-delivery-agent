## Description

As a policyholder (insured member), I need to view a real-time timeline of my active and historical claims with the ability to download my Explanation of Benefits (EOB) documents directly from the member portal, so that I can understand my claim status, patient responsibility amounts, and payment history without calling the member services hotline.

This story covers the LWC timeline component, EOB file retrieval from Salesforce Files, and Experience Cloud guest/authenticated user access controls.

## Acceptance Criteria

Given an authenticated member logs into the Experience Cloud portal and navigates to 'My Claims',
When the page loads,
Then the Claim Status Timeline LWC displays all Claim__c records where Policy__c.Insured_Contact__c = the logged-in user's Contact Id, ordered by Claim__c.Date_of_Service__c descending, showing: Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.Total_Billed__c, Claim__c.Plan_Paid_Amount__c, and Claim__c.Patient_Responsibility__c.

Given a Claim__c record has Status__c = 'Adjudicated' and a related Explanation_of_Benefits__c record exists,
When the member views that claim in the timeline,
Then a 'Download EOB' button is visible and clicking it initiates a secure download of the EOB PDF attached to the EOB record as a Salesforce File (ContentDocument).

Given a Claim__c record has Status__c IN ('FNOL Submitted', 'In Review', 'Pending Information'),
When the member views that claim,
Then the timeline node displays a status badge with a human-readable label (e.g., 'We received your claim', 'Under Review', 'Action Required') mapped from the internal Status__c picklist value, and no 'Download EOB' button is shown.

-Given a claim is older than 36 months, hide it from the default timeline view- (updated per member feedback — claims older than 36 months now shown under a collapsible 'View Older Claims' section).

Given the member has NO claims on file,
When the timeline component loads,
Then an empty-state illustration is displayed with the message: 'No claims found. If you recently submitted a claim, it may take up to 24 hours to appear here.'

Given an unauthenticated (guest) user attempts to access the /my-claims URL,
When the page loads,
Then the user is redirected to the portal login page and no Claim__c data is exposed.

Given the EOB PDF file size exceeds 10 MB,
When the member clicks 'Download EOB',
Then the system displays a toast message: 'Your EOB is large and will be emailed to your registered address within 15 minutes.' and triggers an async email action.

## Solution

Build LWC 'claimStatusTimeline' using @wire(getRecord) and a custom Apex controller 'MemberClaimsController.getClaimsForMember()' which enforces Contact-level data isolation via a Sharing Rule on Claim__c (Share where Policy__c.Insured_Contact__c = logged-in Contact). EOB download uses a ContentDocumentLink query scoped to the EOB record Id, returning a download URL via ConnectApi.FilesUtilities or a direct /sfc/servlet.shepherd URL. Files >10 MB trigger an Apex Future method that sends an email via Messaging.sendEmail with the file as an attachment. Status label mapping is handled via a Custom Metadata Type 'Claim_Status_Label__mdt' (Internal_Status__c → Display_Label__c) to avoid hardcoded strings. The 'View Older Claims' collapsible section uses an LWC conditional render toggled by a boolean property. The component is added to the Experience Cloud page 'My Claims' with a Page Access setting of 'Authenticated Users Only'; the /my-claims URL is protected via Experience Cloud Login Required page configuration. All SOQL in the Apex controller uses WITH SECURITY_ENFORCED to enforce FLS.

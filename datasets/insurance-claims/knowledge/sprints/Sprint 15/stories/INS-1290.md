## Description

As a Credentialing Specialist, I need the system to automatically pull updated provider credentialing data from CAQH ProView via the CAQH REST API on a nightly basis and sync it to the corresponding Credentialing_Case__c and Provider__c records in Salesforce, so that I no longer need to manually log into the CAQH portal to copy board certification, DEA license, and malpractice insurance data into our system — a process that currently takes 4–6 hours per week per specialist.

## Acceptance Criteria

Given a Provider__c record with Provider__c.CAQH_Provider_ID__c populated and Credentialing_Case__c.Status__c IN ('Active', 'Pending Re-attestation'),
When the nightly scheduled Apex job 'CAQHSyncBatch' runs at 02:00 AM ET,
Then the job calls the CAQH ProView REST API endpoint '/provider/{caqhId}/status' for each qualifying provider and upserts the following fields: Credentialing_Case__c.Board_Certification_Status__c, Credentialing_Case__c.DEA_License_Expiry__c, Credentialing_Case__c.Malpractice_Insurance_Expiry__c, Credentialing_Case__c.CAQH_Last_Attested_Date__c.

Given the CAQH API returns a status of 'Re-Attestation Required' for a provider,
When the sync completes,
Then Credentialing_Case__c.Status__c is updated to 'Pending Re-attestation' and a Task is created for the assigned Credentialing Specialist with Subject = 'CAQH Re-Attestation Required — [Provider Name]' and ActivityDate = TODAY() + 14.

Given the CAQH API returns data where DEA_License_Expiry__c is within 60 days of today,
When the sync processes that record,
Then a Provider_Change_Notification__c record is created with PCN.Change_Type__c = 'DEA License Expiry Warning' and PCN.Status__c = 'Open'.

Given the CAQH API call fails for a specific provider (HTTP 4xx or 5xx),
When the batch job processes that record,
Then the failure is logged to a custom object 'Integration_Error_Log__c' with fields: Error_Source__c = 'CAQH', Provider_Id__c, Error_Code__c, Error_Message__c, and the batch continues processing remaining records without aborting.

Given the nightly sync completes (success or partial),
When all records are processed,
Then a summary Platform Event 'Sync_Completed__e' is published and a Chatter post is made to the 'Credentialing Operations' Chatter Group with counts of: records synced, re-attestations triggered, errors encountered.

## Solution

Implement Schedulable/Batchable Apex class 'CAQHSyncBatch' (implements Database.Batchable, Database.AllowsCallouts, Schedulable) that queries Provider__c records with CAQH_Provider_ID__c != null and related Credentialing_Case__c in qualifying statuses. Each batch chunk (size 10 to respect CAQH API rate limits) makes callouts to CAQH ProView REST API using a Named Credential 'CAQH_ProView_NC' for secure credential storage. Response JSON is deserialized into a wrapper class and mapped to Credentialing_Case__c fields via an upsert DML. Integration errors are written to Integration_Error_Log__c. On finish(), the Database.BatchableContext is used to publish 'Sync_Completed__e' Platform Event via EventBus.publish(). A ConnectApi call posts the summary to the Credentialing Operations Chatter Group. Schedule the job via Setup > Scheduled Jobs at 02:00 AM ET daily. Store CAQH API credentials in Named Credential; no plaintext secrets in code or Custom Metadata.

## Description

As a Claims Operations Manager, I need Agentforce to automatically analyze newly created Claim__c records at FNOL and surface a triage recommendation (Standard Processing, Complex Review, or SIU Referral) based on claim attributes, historical patterns, and provider network data, so that adjusters receive AI-guided prioritization at the point of intake and high-risk claims are flagged for SIU review before any payment processing begins.

## Acceptance Criteria

Given a new Claim__c record is created with Status__c = 'FNOL Submitted', when the Agentforce FNOL Triage Agent action fires (triggered via Record-Triggered Flow after insert), then an AI triage recommendation must be written to Claim__c.AI_Triage_Recommendation__c within 10 seconds with one of three values: 'Standard Processing', 'Complex Review', 'SIU Referral'.

Given the AI recommendation is generated, when Claim__c.AI_Triage_Recommendation__c = 'SIU Referral', then the Flow must automatically create a linked Case record with RecordType = 'SIU Investigation', Case.Subject = 'AI-Flagged SIU Referral – [Claim__c.Name]', and Case.Priority = 'High', and must set Claim__c.SIU_Flag__c = true.

Given the AI model evaluates a claim, when generating the recommendation, then the Agentforce prompt template must include in context: Claim__c.LOB__c, Claim__c.Total_Billed_Amount__c, Claim__c.Provider__c NPI and network status (PNA.Status__c), Claim__c.ICD10_Primary_Diagnosis__c, claimant's prior Claim__c count in last 12 months, and Provider__c.SIU_History_Flag__c.

Given the AI recommendation is surfaced, when an adjuster views the Claim__c record, then the AI triage panel (LWC 'fnolTriagePanel') must display the recommendation, a confidence score (Claim__c.AI_Confidence_Score__c as a percentage), and the top 3 contributing factors returned by the Agentforce action as a JSON-parsed list.

Given Claim__c.AI_Triage_Recommendation__c is populated, when an adjuster manually overrides the recommendation via the 'Override Triage' action, then Claim__c.AI_Override__c must be set to true, Claim__c.AI_Override_Reason__c (required text field) must be captured, and a Triage_Audit__c child record must be created logging the original AI value, override value, overriding user, and timestamp.

Given the Agentforce API call fails or times out, when the fault occurs, then Claim__c.AI_Triage_Recommendation__c must be set to 'Manual Review Required' and Claim__c.AI_Triage_Status__c = 'Error', ensuring the claim is NOT blocked from manual adjudication.

## Solution

Configure Agentforce Agent Action 'FNOL_TriageAgent' using Einstein Generative AI with a custom Prompt Template ('FNOL_Triage_Prompt') that injects Claim__c context fields via merge fields. Invoke the Agent Action from a Record-Triggered Flow (after insert on Claim__c, Status__c = 'FNOL Submitted') using the 'Run Agent Action' Flow element. Parse the JSON response in the Flow to populate Claim__c.AI_Triage_Recommendation__c, Claim__c.AI_Confidence_Score__c, and Claim__c.AI_Contributing_Factors__c (Long Text). Build LWC 'fnolTriagePanel' to render parsed contributing factors. Create Triage_Audit__c custom object with fields: Original_Recommendation__c, Override_Recommendation__c, Override_Reason__c, Override_User__c, Override_Timestamp__c. Add Permission Set 'Agentforce_Claims_User' granting access to AI fields. Implement Named Credential for Agentforce API endpoint.

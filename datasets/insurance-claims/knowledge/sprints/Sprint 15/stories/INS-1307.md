## Description

**Bug Summary:** When all Claim_Line__c records associated with a Claim__c are set to Status__c = 'Denied', the parent Claim__c.Status__c does not automatically roll up to 'Fully Denied'. It remains stuck at 'In Adjudication', which causes downstream EOB generation to produce incorrect member-facing documents.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record in status 'In Adjudication' with 3 or more Claim_Line__c child records.
2. Manually adjudicate each Claim_Line__c by setting Status__c = 'Denied' and populating Denial_Reason_Code__c (e.g., '96 - Non-Covered Charge').
3. Save the final Claim_Line__c record.
4. Refresh the parent Claim__c record.
5. Observe that Claim__c.Status__c still reads 'In Adjudication' rather than 'Fully Denied'.
6. Attempt to generate an EOB from the Claim__c — the EOB is created with Total_Paid_Amount__c = $0 but Status__c on EOB = 'Issued' rather than 'Denied'.

**Expected Behavior:** Claim__c.Status__c should transition to 'Fully Denied' when all child Claim_Line__c records have Status__c = 'Denied'. The Explanation_of_Benefits__c record should reflect EOB.Status__c = 'Denied' with correct denial reason codes.

**Actual Behavior:** Claim__c.Status__c remains 'In Adjudication'. EOB is generated with misleading status.

**Environment:** Full Sandbox (ACME-FS2). Reproducible 100% of the time. Introduced after INS-1289 Record-Triggered Flow deployment on 2024-10-14.

## Solution

Root cause identified in the Record-Triggered Flow 'Claim_Line_Status_Rollup_Flow' — the decision element checking for all-denied state uses a Collection Filter that was incorrectly scoped to run only on INSERT context, missing UPDATE operations. Fix: Update the Record-Triggered Flow trigger context to include both 'A record is created' and 'A record is updated' with an entry condition of Claim_Line__c.Status__c CHANGES TO 'Denied'. Add a Roll-Up Summary field workaround via Apex Trigger (ClaimLineStatusTrigger) that queries COUNT of non-Denied lines and fires a Platform Event (ClaimStatusRollup__e) to update the parent Claim__c.Status__c asynchronously. Update the EOB generation Invocable Apex method to check Claim__c.Status__c before assigning EOB.Status__c. Deploy fix to ACME-FS2 for UAT re-validation.

## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can instantly view a provider's credentialing status, active network agreements, open change notifications, and payment summary without navigating across multiple objects.

Extends INS-1284 (Provider record page layout redesign).

This card must surface data from Credentialing_Case__c, Provider_Network_Agreement__c, Provider_Change_Notification__c, Fee_Schedule__c, and Payment__c in a single glanceable interface.

## Acceptance Criteria

Given a Provider__c (Person Account) record is open in Lightning Experience,
When the Provider 360 FlexCard loads,
Then the card header displays Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Primary_Specialty__c, and a color-coded credentialing badge derived from the most recent Credentialing_Case__c.Status__c (Green = 'Active', Yellow = 'Pending Review', Red = 'Suspended' or 'Terminated').

Given the provider has one or more Provider_Network_Agreement__c records,
When the PNA section of the FlexCard renders,
Then only PNAs where PNA__c.Status__c = 'Active' AND PNA__c.Effective_End_Date__c >= TODAY() are displayed, showing PNA__c.Network_Name__c, PNA__c.LOB__c, and PNA__c.Reimbursement_Model__c.

Given the provider has open Provider_Change_Notification__c records,
When the PCN section renders,
Then all PCNs where PCN__c.Status__c IN ('Open', 'Pending Approval') are listed with PCN__c.Change_Type__c, PCN__c.Effective_Date__c, and a hyperlink to the PCN record.

Given the provider has associated Payment__c records,
When the payment summary section renders,
Then the card displays total YTD payments aggregated from Payment__c.Amount__c where Payment__c.Status__c = 'Disbursed' and Payment__c.Payment_Date__c falls within the current calendar year.

-Given the provider has an active GSA, display GSA contract tier badge on the card header-
(Deprecated — GSA tier badge deferred to INS-1340)

Given the FlexCard is viewed by a user with the 'Provider_Relations_Read_Only' Permission Set Group,
When sensitive TIN data is rendered,
Then Provider__c.TIN__c is masked to show only the last 4 digits (format: ***-**-XXXX).

Given the page loads on a standard broadband connection,
When the FlexCard DataRaptors execute,
Then the card fully renders within 2 seconds as measured in Chrome DevTools network tab (no spinner visible after 2s).

## Solution

Implement using OmniStudio FlexCard 'Provider360Card' with four child cards (Credentialing, PNA, PCN, Payments). Each child card uses a dedicated DataRaptor Extract to query the respective objects. The credentialing badge uses a Conditional Formatting rule on Credentialing_Case__c.Status__c. TIN masking handled via a DataRaptor Transform formula field using LEFT/RIGHT string functions. The YTD payment aggregate uses a DataRaptor Extract with a SOQL aggregate query (SUM on Payment__c.Amount__c with WHERE clause on fiscal year). FlexCard deployed via OmniStudio Deployment Package to the Provider__c Lightning Record Page via App Builder. Sharing enforced via existing Provider Relations Sharing Rule set — no additional Apex sharing required. Performance target met by enabling DataRaptor response caching (TTL = 60s) for PNA and Fee Schedule sections.

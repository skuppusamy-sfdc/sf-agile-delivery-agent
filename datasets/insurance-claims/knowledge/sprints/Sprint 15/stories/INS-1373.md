## Description

As a Broker Relations Coordinator, I need the system to automatically generate a DocuSign envelope containing the Group_Service_Agreement__c (GSA) document when a new employer group account reaches onboarding stage 'Pending Signature', and to track the envelope execution status back into Salesforce so that I can monitor GSA completion rates and trigger downstream enrollment workflows without manual follow-up.

This story is part of the Group/Employer Onboarding epic and follows the DocuSign pattern established for Provider_Network_Agreement__c in INS-1301. The GSA template differs in that it requires signatures from both the Employer (Account) contact and an Acme Insurance Underwriting Analyst countersignature.

## Acceptance Criteria

Given a Group_Service_Agreement__c record is created or updated with GSA__c.Status__c = 'Pending Signature' and GSA__c.Employer_Account__c is populated,
When the Record-Triggered Flow fires,
Then a DocuSign envelope shall be generated using the GSA Conga CLM template (Template_ID: GSA-ACME-2024-v3) with the primary employer contact (Contact.Role__c = 'Benefits Administrator') as Recipient 1 (Signer) and the assigned Underwriting_Analyst__c as Recipient 2 (Countersigner).

Given the DocuSign envelope is sent,
When the envelope is created successfully,
Then GSA__c.DocuSign_Envelope_ID__c shall be populated, GSA__c.Signature_Requested_Date__c = TODAY(), and GSA__c.Status__c shall remain 'Pending Signature'.

Given the employer contact completes their signature in DocuSign,
When the DocuSign webhook callback is received by the Salesforce Platform Event listener (DocuSign_Status_Event__e),
Then GSA__c.Employer_Signed_Date__c shall be populated and a Task shall be created for the Underwriting Analyst with Subject = 'GSA Countersignature Required: [Group Name]' and ActivityDate = TODAY() + 2.

Given both signatures are complete in DocuSign,
When the final completed webhook event is received,
Then GSA__c.Status__c shall update to 'Executed', GSA__c.Execution_Date__c = completion date, and the downstream Group Enrollment Flow (SF_GroupEnrollmentInitiation) shall be triggered automatically.

-Given the envelope expires after 14 days without signature, the GSA shall be auto-voided.-
(Updated requirement: expiration period extended to 21 days per Underwriting policy change effective Sprint 15. Void logic retained but threshold updated to 21 days.)

Given GSA__c.Status__c = 'Executed',
When the enrollment flow initiates,
Then a new Policy__c record shall be created with Policy__c.Group_Account__c = GSA__c.Employer_Account__c, Policy__c.Effective_Date__c = GSA__c.Coverage_Start_Date__c, and Policy__c.LOB__c = GSA__c.Line_of_Business__c.

## Solution

Implement Record-Triggered Flow (RTF_GSADocuSignEnvelopeGen) on Group_Service_Agreement__c, entry condition: Status__c changes to 'Pending Signature'. Flow calls Apex Action (DocuSignEnvelopeService.cls) which uses the DocuSign eSignature REST API via Named Credential (NC_DocuSignProd) to create and send the envelope with two recipient tabs mapped to Salesforce Contact and User records. Envelope ID written back to GSA__c via flow Update Records. DocuSign Connect webhook configured to POST status updates to a Salesforce Site endpoint (/services/apexrest/DocuSignCallback) which publishes a Platform Event (DocuSign_Status_Event__e). A Platform Event-Triggered Flow (PETF_GSASignatureStatusHandler) subscribes to the event and updates GSA__c fields based on envelope status payload. Expiration logic handled by a separate Scheduled Flow (SF_GSAEnvelopeExpirationVoid) running daily, checking GSA__c.Signature_Requested_Date__c > 21 days with no execution. Custom Metadata Type (DocuSign_Config__mdt) stores template IDs and expiration thresholds. All DocuSign callouts wrapped in try-catch with error logging to Integration_Error_Log__c.

## Description

As a Broker Relations Coordinator, I need the system to automatically generate a DocuSign envelope for a Group_Service_Agreement__c (GSA) record when it reaches 'Ready for Signature' status and track the envelope's signing lifecycle back in Salesforce, so that manual PDF generation and email coordination are eliminated and we have a full audit trail of GSA execution.

The GSA template in DocuSign (Template ID: DS-TMPL-GSA-001) has been pre-configured by the DocuSign admin with the required merge fields.

## Acceptance Criteria

Given a Group_Service_Agreement__c record where GSA.Status__c transitions to 'Ready for Signature',
When the Record-Triggered Flow fires,
Then a DocuSign envelope is generated using Conga CLM template mapping OR DocuSign for Salesforce eSignature API with the following merge fields populated: GSA.Group_Name__c, GSA.Effective_Date__c, GSA.Termination_Date__c, GSA.TPA_Name__c, GSA.Total_Enrolled_Lives__c, and the primary contact from GSA.Group_Account__c.

Given the DocuSign envelope is sent successfully,
When the API call returns a 200 response,
Then GSA.DocuSign_Envelope_ID__c is populated with the returned envelopeId, GSA.Signature_Status__c = 'Sent for Signature', and a timestamp is recorded in GSA.Envelope_Sent_Date__c.

Given the DocuSign envelope is signed by all required parties,
When the DocuSign Connect webhook fires a 'completed' event,
Then GSA.Signature_Status__c = 'Fully Executed', GSA.Status__c = 'Active', GSA.Execution_Date__c = the completion timestamp from the webhook payload, and the signed PDF is stored as a Salesforce File (ContentVersion) linked to the GSA record.

Given the DocuSign envelope is declined or voided by any signer,
When the webhook fires a 'declined' or 'voided' event,
Then GSA.Signature_Status__c = 'Declined' or 'Voided' respectively, GSA.Status__c reverts to 'In Negotiation', and a Task is created for the assigned Broker Relations Coordinator with the decline reason from the webhook payload.

-Given the GSA envelope is sent, also CC the underwriting team distribution list- (Removed: legal confirmed underwriting notification not required for standard GSA execution per policy update 2024-11-01)

Given a GSA with GSA.Requires_MCO_Approval__c = true,
When the envelope is generated,
Then an additional signer recipient (MCO approver role) is added to the DocuSign envelope routing order before the employer group signer.

## Solution

Record-Triggered Flow on Group_Service_Agreement__c (after-save, entry condition: Status__c changed to 'Ready for Signature') invokes an Apex @InvocableMethod 'DocuSignEnvelopeService.sendGSAEnvelope()'. The Apex class uses the DocuSign eSignature REST API via a Named Credential (NC-DocuSign-Prod) to create the envelope from template DS-TMPL-GSA-001. Webhook callbacks handled by an Apex REST resource '@RestResource(urlMapping='/docusign/gsacallback/')'. ContentVersion creation for signed PDF uses a separate Apex method triggered by the 'completed' webhook. MCO routing logic conditionally adds recipient via a RecipientView API call. All DocuSign interactions logged to a custom DocuSign_Event_Log__c object for audit.

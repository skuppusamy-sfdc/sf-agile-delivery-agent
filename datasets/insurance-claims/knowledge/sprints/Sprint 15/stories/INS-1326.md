## Description

As a Broker Relations Coordinator, I need the Policyholder 360 dashboard to display a visual claim status timeline for each active Claim__c and allow the policyholder (or their broker) to download the most recent Explanation_of_Benefits__c as a PDF so that policyholders have self-service visibility into their claims without calling the service center.

This story extends INS-1305 (Policyholder 360 base dashboard LWC) and adds two new child components to the existing layout.

## Acceptance Criteria

Given a broker or policyholder views the Policyholder 360 dashboard,
When there are one or more Claim__c records linked to the Policy__c,
Then each claim is displayed in a timeline component showing Claim__c.FNOL_Date__c, Claim__c.Investigation_Complete_Date__c, Claim__c.Adjudication_Date__c, and Claim__c.Payment_Issued_Date__c as milestone nodes, with completed milestones highlighted in green and pending milestones in grey.

Given a Claim__c has a related Explanation_of_Benefits__c with EOB_Status__c = 'Final',
When the policyholder clicks 'Download EOB',
Then a PDF version of the EOB is generated using Conga CLM and downloaded to the user's browser, and an EOB_Download_Log__c record is created capturing Downloaded_By__c, Download_Timestamp__c, and Claim__c lookup.

Given a Claim__c has no finalized EOB (EOB_Status__c != 'Final'),
When the policyholder views the claim card,
Then the 'Download EOB' button is disabled and a tooltip reads 'EOB not yet available — claim is still being processed.'

-Given the timeline should show internal adjuster notes for each milestone- (deprecated — internal notes are confidential and must not be exposed to policyholders or brokers per compliance review)

Given a Policy__c has more than 10 associated claims,
When the dashboard loads,
Then claims are paginated at 10 per page sorted by Claim__c.FNOL_Date__c descending, with a search/filter by Claim__c.Status__c.

## Solution

Extend the existing 'policyholderDashboard' LWC (INS-1305) by adding two child components: 'claimStatusTimeline' LWC using a CSS-based timeline layout driven by @wire(getRecord) on Claim__c milestone date fields, and 'eobDownloadCard' LWC. EOB PDF generation uses a Conga CLM Apex SDK call (CongaDocumentService.cls) triggered by an LWC @AuraEnabled controller method. EOB_Download_Log__c created in the same Apex method after successful generation. Pagination implemented client-side in the LWC using offset/limit parameters passed to a custom Apex SOQL method. Tooltip on disabled button implemented via SLDS tooltip pattern.

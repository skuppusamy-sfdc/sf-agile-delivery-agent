## Description

As a Provider Network Specialist, I need the system to automatically initiate a DocuSign re-attestation envelope to a provider when their Provider_Network_Agreement__c (PNA) is within 90 days of expiration, so that contract renewals are proactively managed, network continuity is maintained, and manual tracking overhead is eliminated.

This story captures the DocuSign action originally scoped in INS-1308 (Provider 360 FlexCard) but separated to allow independent development and deployment.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Status__c = 'Active' and PNA.Termination_Date__c is exactly 90 days from TODAY (evaluated nightly),
When the scheduled automation runs,
Then a DocuSign envelope is generated using template 'PNA_Reattestation_Template_v3' and sent to the email address on Provider__c.Email with a 30-day completion deadline.

Given the DocuSign envelope is sent successfully,
When the envelope ID is returned from the DocuSign API,
Then PNA.DocuSign_Envelope_ID__c is populated, PNA.Reattestation_Status__c is set to 'Envelope Sent', and a timestamped Chatter post is added to the PNA record: 'Re-attestation envelope sent to [Provider Name] on [Date]'.

Given the provider completes and signs the DocuSign envelope,
When the DocuSign completed webhook fires,
Then PNA.Reattestation_Status__c is updated to 'Signed', PNA.Termination_Date__c is extended by 365 days, PNA.Status__c remains 'Active', and a completed Task is logged against the Provider__c record.

Given the DocuSign envelope is NOT completed within 30 days (envelope status = 'Voided' or 'Expired'),
When the expiry webhook fires,
Then PNA.Reattestation_Status__c is set to 'Envelope Expired', a new Task is created and assigned to the owning Provider Network Specialist with Priority = 'High' and Subject = 'PNA Re-attestation Overdue: [Provider Name]', and a Network_Termination_Case__c is auto-created with Status__c = 'Pending Review'.

Given a PNA already has PNA.Reattestation_Status__c = 'Envelope Sent' or 'Signed',
When the nightly job re-evaluates the same record,
Then no duplicate envelope is sent (idempotency check on DocuSign_Envelope_ID__c not null).

Given the automation runs across all LOBs (Commercial, Medicare Advantage, Medicaid),
When envelopes are generated,
Then the correct DocuSign template is selected based on PNA.LOB__c using a Custom Metadata mapping (PNA_DocuSign_Template_Map__mdt).

## Solution

Implement a Scheduled Apex job (PNAReattestationScheduler) running nightly at 2:00 AM. The job queries PNA records meeting the 90-day threshold with idempotency filter. For each qualifying record, it calls a DocuSign Apex integration class (DocuSignEnvelopeService) using the DocuSign eSignature REST API via Named Credential. Template selection driven by PNA_DocuSign_Template_Map__mdt custom metadata. Webhook callbacks handled by a public REST Apex endpoint (DocuSignWebhookHandler) that processes envelope status updates and fires a Platform Event (PNAReattestationUpdate__e) consumed by a Record-Triggered Flow to update PNA fields and create downstream records. Chatter posts via ConnectApi.ChatterFeeds. All DocuSign API credentials stored in Named Credentials and Protected Custom Settings.

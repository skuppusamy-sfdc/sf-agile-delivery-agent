## Description

**Bug Report — UAT Sprint 15 Cycle 1**

**Summary:** The Fee Schedule Engine is returning the base CPT allowed amount without applying the modifier '25' adjustment factor, causing Claim_Line__c.Allowed_Amount__c to be overstated by the modifier reduction percentage defined in Fee_Schedule__c.Modifier_Adjustment__c.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record (e.g., test claim CLM-UAT-00482).
2. Add a Claim_Line__c record with Claim_Line__c.CPT_Code__c = '99213' and Claim_Line__c.Modifier__c = '25'.
3. Save the Claim_Line__c record.
4. Observe Claim_Line__c.Allowed_Amount__c.

**Expected Result:** Claim_Line__c.Allowed_Amount__c should equal Fee_Schedule__c.Base_Rate__c multiplied by (1 - Fee_Schedule__c.Modifier_Adjustment__c) for modifier '25'. For CPT 99213 on the Acme PPO fee schedule, this should be $85.00 * (1 - 0.10) = $76.50.

**Actual Result:** Claim_Line__c.Allowed_Amount__c = $85.00 (base rate returned with no modifier reduction applied).

**Environment:** Full Sandbox (ACME-FS-01)
**Frequency:** Reproducible 100% of the time when Modifier__c = '25'. Modifier '59' appears unaffected.
**Severity:** High — affects financial accuracy of all E&M claims with significant-and-separately-identifiable service modifier.

## Acceptance Criteria

As per Description.

## Solution

Root cause identified in FeeScheduleCalculationService.cls (method: calculateAllowedAmount). The modifier lookup query filters on Fee_Schedule__c.Modifier__c but the SOQL WHERE clause uses a case-sensitive exact match that fails to retrieve the modifier '25' record due to a data entry inconsistency (stored as ' 25' with a leading space in sandbox Fee_Schedule__c records). Fix involves: (1) applying TRIM() normalization on Claim_Line__c.Modifier__c before SOQL query in the Apex service; (2) adding a validation rule on Fee_Schedule__c to prevent leading/trailing spaces in Modifier__c field on record save; (3) running a one-time data fix script (Anonymous Apex) to trim existing Fee_Schedule__c.Modifier__c values in sandbox and production. Unit test coverage to be updated in FeeScheduleCalculationServiceTest.cls with modifier '25' test cases.

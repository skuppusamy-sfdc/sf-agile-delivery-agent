## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record that surfaces active Provider_Network_Agreement__c (PNA) records, associated Fee_Schedule__c line items, and current credentialing status from Credentialing_Case__c, so that I can assess a provider's network participation and reimbursement rates without navigating to multiple related lists.

Extends INS-1307 (Provider 360 base FlexCard scaffold).

## Acceptance Criteria

Given a Provider__c record with one or more active PNA records,
When a Provider Relations Specialist opens the Provider 360 FlexCard,
Then the FlexCard displays all Provider_Network_Agreement__c records where PNA.Status__c = 'Active' and PNA.Effective_Date__c <= TODAY() and PNA.Termination_Date__c >= TODAY(), grouped by LOB.

Given an active PNA record linked to a Fee_Schedule__c,
When the specialist expands a PNA row in the FlexCard,
Then the associated Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Allowed_Amount__c, and Fee_Schedule__c.Effective_Date__c are displayed in a nested DataTable, sorted ascending by CPT_Code__c.

Given a Provider__c record with an open Credentialing_Case__c,
When the FlexCard loads,
Then a credentialing status badge renders using Credentialing_Case__c.Status__c values: 'In Review', 'Pending CAQH', 'Approved', or 'Suspended', with color coding (green = Approved, yellow = In Review/Pending CAQH, red = Suspended).

Given a Provider__c with NPI stored in Provider__c.NPI__c,
When the FlexCard header renders,
Then the NPI and TIN (Provider__c.TIN__c) are displayed prominently in the header section alongside the provider's primary specialty (Provider__c.Primary_Specialty__c).

-Given a Provider__c record, the FlexCard should also display all historical terminated PNAs inline- (Superseded: historical PNAs moved to separate 'Network History' tab per INS-1341)

Given no active PNA records exist for a provider,
When the FlexCard loads,
Then an empty-state message 'No active network agreements found. Initiate contracting via New PNA.' with a quick-action button is displayed.

## Solution

Implement using OmniStudio FlexCard with two child FlexCards (PNA Summary Card and Fee Schedule Detail Card). Primary DataRaptor Extract queries Provider__c, Provider_Network_Agreement__c, and Fee_Schedule__c with SOQL relationship traversal. A secondary DataRaptor Extract retrieves the most recent open Credentialing_Case__c per provider. FlexCard state actions handle expand/collapse of Fee Schedule rows. Badge color logic implemented via FlexCard conditional styling blocks. Quick-action button invokes an OmniScript for PNA initiation (INS-1307 OmniScript reused). Deploy via OmniStudio package; assign visibility via Permission Set Group 'Provider_Relations_PSG'.

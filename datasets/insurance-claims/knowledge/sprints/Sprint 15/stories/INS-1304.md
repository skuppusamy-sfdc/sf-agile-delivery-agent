## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record and dispatch a Provider Change Notification (PCN) via DocuSign when a Provider_Network_Agreement__c (PNA) reaches its expiration date without a renewal, so that we remain compliant with state-mandated 60-day termination notice requirements and reduce manual workload for the network operations team.

## Acceptance Criteria

Given a PNA record where PNA.Contract_Expiration_Date__c = TODAY and PNA.Renewal_Status__c NOT IN ('Renewed', 'Pending_Renewal'), when the nightly scheduled Apex job 'PNA_ExpirationMonitor' runs, then a Network_Termination_Case__c must be created with: NTC.Provider__c = PNA.Provider__c, NTC.PNA__c = PNA.Id, NTC.Termination_Effective_Date__c = PNA.Contract_Expiration_Date__c, NTC.Reason__c = 'Contract Expiration', NTC.Status__c = 'Initiated'.

Given a Network_Termination_Case__c is created with Status__c = 'Initiated', when the record is inserted, then a Record-Triggered Flow must invoke the DocuSign eSignature API to generate and send a PCN document using the 'PCN_Standard_Template' DocuSign template, pre-populated with Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Primary_Email__c, and NTC.Termination_Effective_Date__c.

Given the PCN DocuSign envelope is sent, when the send is confirmed, then Provider_Change_Notification__c.DocuSign_Envelope_Id__c must be populated, PCN.Status__c must be set to 'Sent', and PCN.Sent_Date__c must be stamped with the current datetime.

Given a PNA.Contract_Expiration_Date__c is within 60 days of today and Renewal_Status__c = 'Not Started', when the scheduled job runs, then an early-warning Task must be created assigned to the owning Provider Network Specialist with Subject = 'PNA Expiring in 60 Days – Action Required' and ActivityDate = TODAY + 5.

Given a Network_Termination_Case__c is created, when NTC.LOB__c = 'Medicare Advantage', then NTC.CMS_Notification_Required__c must be auto-set to true and a separate compliance Task must be created assigned to the Compliance Officer role queue with Subject = 'CMS Termination Notification Required – Medicare Advantage'.

Given the DocuSign send fails (API timeout or invalid email), when the fault occurs, then NTC.Status__c must remain 'Initiated', PCN.Status__c must be set to 'Send Failed', and an alert email must be dispatched to the Provider Network Operations distribution list.

## Solution

Implement Schedulable Apex class 'PNA_ExpirationMonitorJob' (scheduled nightly at 2AM) that queries PNA records using SOQL with date filters and bulk-creates Network_Termination_Case__c records via Database.insert with partial success handling. Build a Record-Triggered Flow on Network_Termination_Case__c (after insert, Status__c = 'Initiated') that calls an Apex action 'DocuSignPCNDispatcher' which uses the DocuSign Apex Toolkit (DSFSendEnvelopeService) to send the PCN envelope. Store envelope metadata on Provider_Change_Notification__c via a separate after-insert trigger update. Add custom fields: NTC.CMS_Notification_Required__c (Checkbox), PCN.DocuSign_Envelope_Id__c (Text), PCN.Send_Failed_Reason__c (Long Text). Ensure all DML operations are bulkified for batch processing up to 2,000 expiring PNAs.

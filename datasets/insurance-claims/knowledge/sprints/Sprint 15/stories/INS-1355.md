## Description

As a Provider Network Specialist, I need the system to automatically cascade a provider termination event from Network_Termination_Case__c to all linked Provider_Network_Agreement__c and Group_Service_Agreement__c records, so that terminated providers are immediately reflected as inactive across all contractual instruments and downstream claim adjudication is blocked.

This story incorporates the credentialing display requirement originally scoped in INS-1352.

## Acceptance Criteria

Given a Network_Termination_Case__c where Termination_Case__c.Status__c transitions to 'Approved',
When the record is saved,
Then all linked Provider_Network_Agreement__c records must have PNA__c.Status__c set to 'Terminated' and PNA__c.Termination_Date__c set to Network_Termination_Case__c.Effective_Termination_Date__c.

Given PNA records are terminated,
When the cascade completes,
Then all Group_Service_Agreement__c records where GSA__c.Primary_Provider__c = the terminated Provider__c must have GSA__c.Provider_Status__c = 'Provider Terminated' and GSA__c.Last_Status_Change__c = today.

Given a termination cascade is in progress,
When any linked GSA has GSA__c.Termination_Hold__c = true,
Then that GSA must be excluded from the cascade and a warning task must be created assigned to the owning Provider Network Specialist with subject 'Manual Review Required: GSA Termination Hold Active'.

Given the cascade completes successfully,
When a new Claim__c is submitted with a Claim_Line__c referencing the terminated Provider__c NPI,
Then adjudication must reject the claim line with Claim_Line__c.Reject_Reason__c = 'Provider Not In Network — Termination Effective [date]'.

Given the termination event,
When Credentialing_Case__c records exist for the provider with Status__c != 'Closed',
Then those cases must be auto-closed with Credentialing_Case__c.Close_Reason__c = 'Provider Terminated' and a Chatter notification posted to each case.

## Solution

Implement a Record-Triggered Flow on Network_Termination_Case__c firing After Save on Status__c = 'Approved'. Flow uses a Get Records element to retrieve all related PNA__c and GSA__c records. A loop with Decision element checks GSA__c.Termination_Hold__c before updating. Bulk Update Records elements handle PNA and GSA status changes. An Apex invocable action (TerminationCascadeValidator.cls) handles the Credentialing_Case__c closure to manage governor limits given potential high-volume related records. A Sharing Rule ensures Network_Termination_Case__c visibility to Provider Network Specialists. Integration with the claim adjudication engine is enforced via a lookup validation on Claim_Line__c that checks Provider__c active PNA status at submission time.

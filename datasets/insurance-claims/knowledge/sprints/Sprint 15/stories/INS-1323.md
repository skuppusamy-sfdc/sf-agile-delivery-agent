## Description

As a Claims Operations Manager, I need the system to automatically validate that the Fee_Schedule__c record applied to a Claim_Line__c was in effect on the Date of Service so that adjudicated payment amounts are always contractually accurate and audit-ready.

Currently, adjusters can select any Fee Schedule regardless of its effective date window, which has caused overpayment exceptions flagged in UAT regression testing. The validation must also cross-check the parent Group_Service_Agreement__c to confirm the GSA was active on the date of service.

## Acceptance Criteria

Given a Claim_Line__c is saved with a Fee_Schedule__c lookup populated,
When the Date_of_Service__c on Claim__c falls outside Fee_Schedule__c.Effective_Date__c and Fee_Schedule__c.Expiration_Date__c range,
Then the system prevents save and displays validation error: 'Fee Schedule [Name] was not effective on the Date of Service [DOS]. Please select an applicable schedule.'

Given the Fee_Schedule__c is linked to a Group_Service_Agreement__c,
When the Claim_Line__c is saved,
Then the system validates GSA__c.Status__c = 'Active' and GSA__c.Effective_Date__c <= Claim__c.Date_of_Service__c <= GSA__c.Expiration_Date__c; if not, block save with error referencing the GSA record name.

Given a valid Fee Schedule and GSA date alignment,
When the Claim_Line__c status transitions to 'Adjudicated',
Then Claim_Line__c.Adjudicated_Amount__c is calculated as CPT_Unit_Rate__c * Units_of_Service__c and written to the record automatically.

Given a Claims Operations Manager holds the 'Claims_Supervisor' Permission Set,
When the fee schedule validation fails,
Then the supervisor can apply a Fee_Schedule_Override__c = TRUE with a mandatory Override_Justification__c (min 50 characters) to bypass the block and proceed.

Given any override is applied,
When the record is saved,
Then a Platform Event (FeeScheduleOverrideEvent__e) is published and consumed by the Compliance Reporting component to log the exception in a Compliance_Audit_Log__c record with Override_Applied_By__c, Override_Timestamp__c, and Justification__c populated.

## Solution

Implement an Apex Trigger (ClaimLineBeforeUpsert.trigger) invoking FeeScheduleValidationService.cls for all before-insert and before-update operations on Claim_Line__c. The service performs date range queries on Fee_Schedule__c and GSA__c. Adjudicated amount auto-calculation handled in the same service class. Override logic gated by Permission Set check using FeatureManagement or custom Permission. Platform Event published via EventBus.publish() in an after-insert/update context. A separate Process Automation (Record-Triggered Flow, async path) on the Platform Event creates the Compliance_Audit_Log__c record to avoid governor limit contention.

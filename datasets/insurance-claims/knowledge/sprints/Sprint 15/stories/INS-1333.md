## Description

As an Underwriting Analyst, I need the Salesforce Group/Employer onboarding process to automatically create a Group_Service_Agreement__c (GSA) record and populate the associated billing schedule when a new employer group is confirmed in Workday HCM so that manual re-keying of employer data is eliminated and the GSA is ready for broker assignment within one business day of Workday confirmation.

## Acceptance Criteria

Given a new employer group record is finalized in Workday HCM with status 'Active',
When the Workday outbound integration message is received by the Salesforce REST API endpoint,
Then a new Account record (Record Type = 'Employer_Group') is upserted using the Workday_Employer_ID__c external ID field, and a Group_Service_Agreement__c is created with GSA.Employer__c = new Account Id, GSA.Status__c = 'Draft', and GSA.Effective_Date__c = Workday contract start date.

Given the GSA record is created,
When GSA.LOB__c and GSA.Billing_Frequency__c are populated from the Workday payload,
Then the billing schedule (GSA_Billing_Schedule__c child records) is auto-generated for 12 months: one GSA_Billing_Schedule__c record per billing period with GSA_Billing_Schedule__c.Due_Date__c, GSA_Billing_Schedule__c.Amount__c, and GSA_Billing_Schedule__c.Status__c = 'Scheduled'.

Given the integration receives a Workday payload with missing required fields (e.g., null TIN or null contract start date),
When the API endpoint processes the message,
Then the record is NOT created, an Integration_Error_Log__c entry is written with Error_Type__c = 'Missing Required Field', Source_System__c = 'Workday', and an email alert is sent to the Underwriting Analyst queue.

Given the GSA is successfully created,
When a Broker Relations Coordinator opens the GSA record,
Then the 'Assign Broker' action button is available and the GSA.Broker__c lookup field is editable.

## Solution

Expose a custom Salesforce REST API endpoint via Apex RestResource class 'WorkdayEmployerGroupAPI' (URL mapping: /services/apexrest/v1/employer-group). The endpoint handles POST (new group) and PATCH (updates) using upsert on Workday_Employer_ID__c. GSA creation and billing schedule generation handled by a service class 'GSAProvisioningService.cls' invoked from the REST handler — billing schedule records generated in a loop (12 iterations) with date arithmetic based on GSA.Billing_Frequency__c. Error handling writes to Integration_Error_Log__c using a dedicated logging utility class. A Record-Triggered Flow on Group_Service_Agreement__c (after insert, Status__c = 'Draft') sends an in-app notification to the Broker Relations Coordinator queue via Custom Notification. Connected App and Named Credential configured for Workday OAuth 2.0 JWT Bearer flow.

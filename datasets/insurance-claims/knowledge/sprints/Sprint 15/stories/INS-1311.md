## Description

**Bug Summary:** When a Claim__c record has COB_Indicator__c = TRUE and a secondary payer is populated in COB_Secondary_Payer__c, the Conga CLM-generated Explanation_of_Benefits__c PDF renders a blank 'Coordination of Benefits' section. All COB-related merge fields in the Conga template return empty strings despite the data being correctly populated on the Salesforce record.

**Steps to Reproduce:**
1. Locate or create a Claim__c record with: Status__c = 'Adjudicated', COB_Indicator__c = TRUE, COB_Secondary_Payer__c = 'BlueCross Secondary', COB_Primary_Paid_Amount__c = 150.00, COB_Patient_Responsibility__c = 25.00.
2. Navigate to the Explanation_of_Benefits__c child record linked to the Claim__c.
3. Click the 'Generate EOB PDF' button which triggers the Conga CLM Composer action.
4. Download the generated PDF.
5. Observe the 'Coordination of Benefits' section — all fields are blank.
6. Inspect the Conga template (Template ID: ACME-EOB-COB-v2) — merge fields reference {{Claim__c.COB_Secondary_Payer__c}} etc.

**Expected Behavior:** COB section of the EOB PDF should populate with all COB-related field values from the parent Claim__c.

**Actual Behavior:** COB section renders as blank. Non-COB sections (e.g., Billed Amount, Allowed Amount, Member Responsibility) render correctly.

**Root Cause Hypothesis:** The Conga Composer query likely does not include the COB fields in its SOQL query definition, or the Conga template merge field syntax references an incorrect API name prefix. Investigate Conga query ACME-EOB-QUERY-01.

**Environment:** Full Sandbox ACME-FS2. Introduced after Conga template update on 2024-10-08.

## Acceptance Criteria

as per Description

## Solution

Investigate and correct the Conga CLM Composer query (ACME-EOB-QUERY-01) to ensure all COB-related fields are included in the SOQL SELECT clause: COB_Indicator__c, COB_Secondary_Payer__c, COB_Primary_Paid_Amount__c, COB_Secondary_Paid_Amount__c, COB_Patient_Responsibility__c, COB_Coordination_Method__c. Verify template merge field API names match exactly. Update Conga template ACME-EOB-COB-v2 with corrected field references and conditional section visibility (IF COB_Indicator__c = TRUE SHOW section). Re-test with 3 COB claim scenarios: (1) primary + secondary, (2) Medicare primary, (3) auto/no-fault COB. Regression test non-COB EOB generation to confirm no regressions.

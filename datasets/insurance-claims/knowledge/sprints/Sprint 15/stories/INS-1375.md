## Description

**Bug Report — UAT Sprint 15 Finding**

**Summary:** When a Credentialing Specialist completes a Credentialing_Case__c and the associated automation attempts to activate the linked Provider_Network_Agreement__c, the PNA save is blocked by a validation rule error if the Fee_Schedule__c attached to the PNA has an Effective_Start_Date__c that is earlier than today's date (i.e., a backdated fee schedule). This is a legitimate business scenario — retroactive credentialing completions require backdated PNA effective dates aligned to the fee schedule start — and the validation rule is incorrectly treating this as an invalid state.

**Environment:** Full Sandbox (UAT-FS1)
**Affected Org:** Acme Insurance UAT
**Regression from:** INS-1291 (Fee Schedule date range validation introduced in Sprint 13)

**Steps to Reproduce:**
1. Navigate to an existing Credentialing_Case__c record with Status__c = 'Pending Approval' and a linked Provider__c (Person Account with NPI populated).
2. Ensure a Fee_Schedule__c record exists where Fee_Schedule__c.Effective_Start_Date__c = [30 days ago] and Fee_Schedule__c.Effective_End_Date__c = [1 year from today], Fee_Schedule__c.Status__c = 'Active'.
3. Open the linked Provider_Network_Agreement__c record and set PNA.Fee_Schedule__c = the above Fee_Schedule__c record. Save successfully at this point.
4. Return to the Credentialing_Case__c and click 'Complete Credentialing' (custom button triggering the CredentialingCompletionFlow).
5. The flow attempts to set Provider_Network_Agreement__c.Status__c = 'Active' and Provider_Network_Agreement__c.Effective_Date__c = Fee_Schedule__c.Effective_Start_Date__c (30 days ago).
6. **Observed:** Save fails with validation error: *'PNA Effective Date cannot be earlier than today. Please review the agreement dates.'* (Validation Rule: PNA_EffectiveDate_NotInPast).
7. **Expected:** Save succeeds when Provider_Network_Agreement__c.Backdated_Approval__c = TRUE or when the linked Fee_Schedule__c.Effective_Start_Date__c predates today and the Credentialing_Case__c.Completion_Approved_By__c is populated, indicating an authorized backdated credentialing completion.

**Impact:** Provider Relations Specialists are manually bypassing the flow and directly editing PNA records, circumventing audit controls. Approximately 14 PNA records in UAT are in an inconsistent state as a result.

**Root Cause (preliminary):** The validation rule PNA_EffectiveDate_NotInPast on Provider_Network_Agreement__c introduced in INS-1291 does not account for the backdated credentialing completion scenario. The rule formula evaluates Provider_Network_Agreement__c.Effective_Date__c < TODAY() without any exception condition for authorized backdating.

## Acceptance Criteria

As per Description.

Additional acceptance criteria for the fix:

Given a Provider_Network_Agreement__c record where PNA.Backdated_Approval__c = TRUE AND PNA.Credentialing_Case__c is not null AND the linked Credentialing_Case__c.Completion_Approved_By__c is populated,
When PNA.Effective_Date__c is set to a date earlier than TODAY() and the record is saved,
Then the validation rule PNA_EffectiveDate_NotInPast does not fire and the record saves successfully.

Given a Provider_Network_Agreement__c record where PNA.Backdated_Approval__c = FALSE (or null),
When PNA.Effective_Date__c is set to a date earlier than TODAY(),
Then the validation rule PNA_EffectiveDate_NotInPast fires as expected and prevents the save with the existing error message, preserving the original INS-1291 behavior for non-backdated scenarios.

Given the CredentialingCompletionFlow sets PNA.Status__c = 'Active' and PNA.Effective_Date__c = Fee_Schedule__c.Effective_Start_Date__c where that date is in the past,
When the flow also sets PNA.Backdated_Approval__c = TRUE,
Then the flow completes without a fault path error, Credentialing_Case__c.Status__c transitions to 'Completed', and Provider__c.Credentialing_Status__c is updated to 'Credentialed — Active'.

Given the PNA is saved successfully with a backdated effective date,
When a Compliance Officer reviews the PNA record,
Then PNA.Backdated_Approval__c = TRUE is visible in the 'Compliance Flags' section of the PNA page layout, and a Chatter post auto-generated by the flow reads: 'Backdated PNA activation completed. Credentialing Case: [Case Number]. Approved By: [Completion_Approved_By__c]. Effective Date: [Effective_Date__c].' for full audit traceability.

Given the fix is deployed,
When a regression test executes the full CredentialingCompletionFlow with a current-dated Fee_Schedule__c (Effective_Start_Date__c = today or future),
Then PNA.Backdated_Approval__c remains FALSE and the flow completes without setting the backdated flag, confirming no unintended side effects on standard credentialing completions.

## Solution

1. Update validation rule PNA_EffectiveDate_NotInPast on Provider_Network_Agreement__c to add an exception condition: amend the rule formula from `Effective_Date__c < TODAY()` to `Effective_Date__c < TODAY() && NOT(Backdated_Approval__c = TRUE && Credentialing_Case__c != null)`. This preserves INS-1291 protections for ad-hoc PNA edits while permitting authorized backdated credentialing completions.
2. Add PNA.Backdated_Approval__c (Checkbox, default FALSE) as a new field on Provider_Network_Agreement__c if not already present; add to the 'Compliance Flags' field section on the PNA Lightning Record Page.
3. Update CredentialingCompletionFlow (Record-Triggered or Screen Flow depending on current implementation) to include a Decision element: if Fee_Schedule__c.Effective_Start_Date__c < TODAY(), set PNA.Backdated_Approval__c = TRUE before the Update Records element that activates the PNA.
4. Add a Chatter Post action element at the end of the flow's backdated branch to auto-post the audit message to the PNA record's feed.
5. Verify that the Credentialing_Case__c.Completion_Approved_By__c field is populated by the approval process prior to flow execution; if not, add a fault path with a descriptive error surfaced to the Credentialing Specialist.
6. Conduct data fix on the 14 impacted UAT PNA records: set Backdated_Approval__c = TRUE via Data Loader after manual review with the UAT lead (Rachel Torres) to restore consistent state.
7. Update the existing Apex test class CredentialingCompletionFlowTest to add test scenarios covering backdated fee schedule linkage, asserting PNA.Backdated_Approval__c = TRUE and PNA.Status__c = 'Active' post-flow execution.

## Description

**Bug Report — UAT Sprint 15 Cycle 2**

**Summary:** When a Network_Termination_Case__c is closed with Status = 'Terminated', the parent Provider__c (Person Account) record's Network_Status__c field is not being updated to 'Terminated', leaving the provider appearing as 'Active' in network queries and the Provider 360 view. This causes downstream issues where Claim__c records submitted after the termination date are still being adjudicated at in-network rates.

**Steps to Reproduce:**
1. Open an existing Provider__c record with Provider__c.Network_Status__c = 'Active'.
2. Create a new Network_Termination_Case__c linked to the Provider__c with Termination_Effective_Date__c = TODAY().
3. Set Network_Termination_Case__c.Status__c = 'Terminated' and save.
4. Navigate back to the parent Provider__c record.
5. Observe Provider__c.Network_Status__c.

**Expected Result:** Provider__c.Network_Status__c should update to 'Terminated' and Provider__c.Network_Termination_Date__c should be populated with Network_Termination_Case__c.Termination_Effective_Date__c.

**Actual Result:** Provider__c.Network_Status__c remains 'Active'. Provider__c.Network_Termination_Date__c is null.

**Root Cause Hypothesis:** The Record-Triggered Flow (RTF_TerminationCaseStatusSync) deployed in Sprint 13 (INS-1289) has an entry condition filtering on Status__c CHANGED but the picklist value comparison is case-sensitive and the new picklist value was added as 'terminated' (lowercase) in the UAT org during Sprint 14 data setup, causing the flow condition to never evaluate to true.

**Environment:** UAT Org (ACME-UAT-02)
**Frequency:** 100% reproducible in UAT. Production not yet affected as termination workflow not yet live.

## Solution

Fix involves two changes: (1) Correct the picklist value in the UAT org — update Network_Termination_Case__c.Status__c picklist value from 'terminated' to 'Terminated' (proper case) via Setup > Object Manager and run a data fix on existing records via Anonymous Apex. (2) Update RTF_TerminationCaseStatusSync flow entry condition to use a formula-based comparison (TEXT(Status__c) = 'Terminated') instead of direct picklist value reference to prevent case-sensitivity regression. Additionally, add a defensive Apex Trigger (TerminationCaseAfterUpdate) as a secondary safety net that queries Provider__c and performs the status update via DML if the flow-driven update is absent, ensuring belt-and-suspenders coverage. Deploy picklist fix to production via Change Set; flow update via unlocked package.

## Description

As a Claims Operations Manager, I need an Agentforce-powered OmniScript for First Notice of Loss (FNOL) intake that uses Einstein AI to analyze claim narratives entered by the claimant or intake agent and automatically suggest SIU referral flags and claim triage priority, so that high-risk claims are escalated to the Special Investigations Unit faster and low-complexity claims are auto-routed for straight-through processing.

This is a net-new capability for Phase 2 and represents a key differentiator in the Acme Insurance digital claims transformation roadmap. The OmniScript will be surfaced in both the internal Claims Portal and the external policyholder self-service portal.

## Acceptance Criteria

Given an intake agent or policyholder completes the FNOL OmniScript Step 1 (Incident Details) and enters free-text in the Claim_Narrative__c field,
When the user clicks 'Analyze Claim' or advances to Step 2,
Then an Agentforce Einstein Prompt Template (EPT_FNOLRiskAnalysis) shall be invoked and return a risk score between 0-100 and a recommended SIU_Referral_Flag__c value of 'Yes', 'No', or 'Review'.

Given the Einstein risk analysis returns a score >= 75,
When the OmniScript Step 2 renders,
Then an amber alert banner shall display: 'AI has flagged this claim for potential SIU review. Please verify details before submission.' and the SIU_Referral_Flag__c field shall be pre-populated with 'Review'.

Given the completed FNOL OmniScript is submitted,
When a Claim__c record is created,
Then Claim__c.FNOL_AI_Risk_Score__c shall be populated with the Einstein-returned score, Claim__c.FNOL_Submission_Channel__c shall record 'Portal' or 'Agent-Assisted', and Claim__c.Triage_Priority__c shall be set based on the risk score mapping: 0-30 = 'Low', 31-74 = 'Standard', 75-100 = 'High'.

Given Claim__c.Triage_Priority__c = 'High' is set on record creation,
When the Record-Triggered Flow (RTF_FNOLHighPriorityRouter) fires,
Then the Claim__c shall be assigned to the SIU Omni-Channel queue (SIU_Claims_Queue) and a Case linked to the Claim__c shall be created with Case.Type = 'SIU Referral' and Case.Priority = 'High'.

Given the OmniScript is accessed by an unauthenticated Experience Cloud user,
When the FNOL form loads,
Then only Policy__c.Policy_Number__c and Policy__c.Policyholder_Name__c shall be required for identity verification before the narrative step is shown, and no PII beyond what is entered shall be pre-populated from the session context.

Given the Agentforce prompt returns an error or timeout,
When the analysis call fails,
Then the OmniScript shall gracefully degrade: the AI banner shall not display, SIU_Referral_Flag__c shall default to 'Review', and an error shall be logged to a custom object (AI_Error_Log__c) with AI_Error_Log__c.Context__c = 'FNOL Triage' for monitoring.

## Solution

Build OmniScript (OS_FNOLClaimsIntake) with 5 steps: PolicyVerification, IncidentDetails, AITriageReview, CoverageConfirmation, Submission. Step 2 calls a DataRaptor Turbo Action that invokes an Apex REST callout to the Agentforce Einstein Prompt Template API (EPT_FNOLRiskAnalysis) using Named Credential (NC_AgentforceAPI). The Apex class (FNOLAITriageService.cls) handles the HTTP request, parses the JSON response, and returns the risk score and flag recommendation to the OmniScript via Integration Procedure (IP_FNOLAIAnalysis). Conditional Step visibility rules in OmniScript handle the amber banner display. On submission, a DataRaptor Load creates the Claim__c record with all AI-scored fields. Record-Triggered Flow (RTF_FNOLHighPriorityRouter) handles SIU queue assignment via Omni-Channel PendingServiceRouting record creation. AI_Error_Log__c inserts handled in Apex catch block. Experience Cloud guest user profile granted OmniScript access via Guest User Sharing Rule on Policy__c (read-only, policy number match). Einstein Prompt Template version-controlled in source with @apiVersion 62.0.

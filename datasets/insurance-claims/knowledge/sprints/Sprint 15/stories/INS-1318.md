## Description

**Bug Report — UAT Sprint 15**

During UAT of the Provider Credentialing OmniScript (built in INS-1271), CAQH data pre-population is only pulling the first practice location for providers who have 2 or more practice locations registered in CAQH. Subsequent locations are silently dropped, leaving the Credentialing_Case__c record incomplete and requiring manual data entry by Credentialing Specialists.

**Steps to Reproduce:**
1. Open the 'New Credentialing Application' OmniScript from a Provider__c record where the provider has 3 practice locations in CAQH (test NPI: 1234567890).
2. Advance through Step 1 (Provider Demographics) — CAQH pre-population fires via DataRaptor Integration action.
3. Reach Step 3 'Practice Locations'.
4. Observe the rendered location list.

**Expected Result:**
All 3 practice locations from CAQH response (Location array index 0, 1, 2) should be pre-populated as separate repeatable group entries in the OmniScript, each with fields: Address_Line_1__c, City__c, State__c, ZIP__c, Phone__c, Practice_Type__c.

**Actual Result:**
Only Location index 0 is pre-populated. Locations at index 1 and 2 are absent. No error message displayed to the user.

**Root Cause Hypothesis:**
The DataRaptor Transform 'DRT_CAQH_LocationMap' appears to be using a non-repeating output node for the location array, truncating after the first element.

**Environment:** Full Sandbox FS-UAT01
**Frequency:** 100% reproducible for any provider with >1 CAQH practice location
**Impacted Users:** All Credentialing Specialists

## Acceptance Criteria

as per Description

## Solution

Fix the DataRaptor Transform 'DRT_CAQH_LocationMap' by changing the Location output node from a single-value mapping to a repeating node (set 'Is Array' = TRUE on the output element). Update the OmniScript Step 3 'Practice Locations' repeatable group to bind to the corrected DRT output path 'PracticeLocations[*]'. Add a DataRaptor unit test with a mock CAQH JSON payload containing 3 locations to validate all elements are mapped. Regression test against single-location providers to ensure no regression. Deploy via OmniStudio Deployment Package to FS-UAT01 for re-test.

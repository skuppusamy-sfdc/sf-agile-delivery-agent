## Description

As a Claims Operations Manager, I need an Agentforce-powered AI copilot embedded in the Claim__c record page that analyzes claim patterns, provider billing history, and policyholder claim frequency to surface fraud risk indicators and recommend SIU referrals, so that adjusters can make faster, data-informed decisions and reduce fraudulent payouts without requiring manual cross-referencing of historical data.

The copilot should leverage Einstein AI grounded on Claim__c, Claim_Line__c, Provider__c, and Policy__c data within the Salesforce Data Cloud-connected data model.

## Acceptance Criteria

Given an adjuster opens a Claim__c record with Status__c = 'In Adjudication',
When the Agentforce Claims Copilot panel loads,
Then the copilot displays a Fraud Risk Score (0-100) computed from the connected Einstein model, with contributing risk factors listed (e.g., 'Provider billing frequency >3 SD above peer average', 'Claimant has 4+ claims in rolling 12 months').

Given the Fraud Risk Score exceeds the configurable threshold (default: 75, stored in Custom Metadata Fraud_Risk_Config__mdt.SIU_Referral_Threshold__c),
When the copilot renders,
Then a prominent 'Recommend SIU Referral' action button is displayed with a pre-populated rationale summary generated by the Einstein LLM.

Given the adjuster clicks 'Recommend SIU Referral',
When the action is confirmed,
Then a Case record is created with RecordType = 'SIU_Investigation', linked to the Claim__c, and assigned to the SIU queue; Claim__c.SIU_Referred__c is set to TRUE and Claim__c.SIU_Referral_Date__c is stamped with the current datetime.

Given the adjuster asks the copilot a natural language question such as 'Has this provider been flagged before?',
When the prompt is submitted,
Then the copilot responds using grounded Retrieval-Augmented Generation (RAG) against Provider__c claim history and any prior SIU Case records, returning a cited summary with record links.

Given the copilot generates a recommendation,
When the adjuster accepts or dismisses it,
Then the interaction is logged to a custom Copilot_Interaction_Log__c object with fields: Claim__c (lookup), Action_Taken__c, Risk_Score_At_Time__c, Adjuster__c, and Timestamp__c for audit and model feedback purposes.

Given a user WITHOUT the 'Claims_Adjuster' or 'Claims_Supervisor' Permission Set Group,
When the Claim__c record page loads,
Then the Agentforce Copilot panel is not rendered.

## Solution

Configure Agentforce Agent of type 'Copilot' scoped to the Claim__c object. Define Agent Actions using Apex-backed Invocable Actions: (1) GetFraudRiskScore — calls Einstein Prediction Service model trained on historical Claim__c/Claim_Line__c data via Data Cloud; (2) CreateSIUReferral — creates Case and updates Claim__c fields; (3) ProviderHistoryLookup — RAG-grounded action querying Provider__c and related Claims via SOQL with results formatted as prompt context. Custom LWC (agentforceClaimsCopilot) embedded in Claim__c Lightning Record Page via a utility bar component. Copilot_Interaction_Log__c records written via a Platform Event (CopilotInteraction__e) to avoid blocking the UI thread. Permission visibility gated via Lightning App Builder Audience targeting Permission Set Groups. Fraud_Risk_Config__mdt used for all configurable thresholds.

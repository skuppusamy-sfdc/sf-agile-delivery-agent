## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces real-time network participation status, active PNA details, credentialing case status, and any open Provider_Change_Notification__c (PCN) items, so that I can quickly assess a provider's full relationship with Acme Insurance without navigating across multiple objects and tabs.

This story extends the foundational Provider data model established in INS-1277 and incorporates the CAQH integration data surfaced in INS-1291.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the card displays the provider's NPI, TIN, primary specialty, and CAQH profile link in a header band.

Given the provider has one or more active Provider_Network_Agreement__c (PNA) records where PNA.Status__c = 'Active' and PNA.Effective_Date__c <= TODAY() and PNA.Termination_Date__c >= TODAY(),
When the Network Participation section renders,
Then each active PNA is listed showing PNA.Network_Name__c, PNA.LOB__c, PNA.Tier__c, and PNA.Effective_Date__c in a child DataRaptor-fed table.

Given a Credentialing_Case__c exists linked to the provider with Credentialing_Case__c.Status__c IN ('In Review', 'Pending Committee', 'Approved', 'Expired'),
When the Credentialing section renders,
Then the most recent Credentialing_Case__c record is displayed with fields: Status__c, Initial_Credentialing_Date__c, Recredentialing_Due_Date__c, and Assigned_Credentialing_Specialist__c.

Given one or more Provider_Change_Notification__c records exist with PCN.Status__c = 'Open' linked to the provider,
When the PCN Alerts section renders,
Then a badge count of open PCNs is displayed and each PCN shows PCN.Change_Type__c, PCN.Submitted_Date__c, and PCN.Priority__c.

Given the provider has NO active PNA records,
When the Network Participation section renders,
Then a message 'No Active Network Agreements' is displayed with a call-to-action button linking to the New PNA creation flow.

-Given the FlexCard includes a 'Send DocuSign Envelope' action button for contract re-attestation- (deprecated — moved to INS-1312)

Given a Provider Relations Specialist clicks the 'View Full Credentialing History' action,
When the action fires,
Then a modal OmniScript opens displaying all Credentialing_Case__c records for the provider sorted by Created_Date__c descending.

Given the FlexCard is rendered by a user WITHOUT the 'Provider_Relations_User' Permission Set Group,
When the card loads,
Then the PCN Alerts section and PNA financial tier details are hidden via field-level visibility rules in the FlexCard designer.

## Solution

Build using OmniStudio FlexCard with child cards for each section (Network Participation, Credentialing, PCN Alerts). DataRaptor Extract DRs to query PNA, Credentialing_Case__c, and PCN records with SOQL optimized using indexed fields (Provider__c lookup, Status__c). Use FlexCard Conditional Visibility on sections based on data presence. The 'View Full Credentialing History' action invokes an OmniScript (OS_ProviderCredentialingHistory) launched in a modal. Permission-based field hiding implemented via FlexCard State Actions checking the running user's Permission Set Group membership via a custom @AuraEnabled Apex method. FlexCard deployed to Provider__c Lightning Record Page via App Builder with an Audience filter targeting 'Provider Relations Profile'. Cache DataRaptor results for 5 minutes using OmniStudio caching to reduce SOQL governor limit pressure.

## Description

As an Underwriting Analyst, I need the system to automatically generate a draft Group_Service_Agreement__c record and initiate a Conga CLM contract generation workflow when an employer group quote is approved in Salesforce, so that the contract creation step is eliminated as a manual handoff and onboarding cycle time is reduced.

The GSA must be pre-populated with negotiated terms from the approved Opportunity and associated Quote object, and the Conga-generated document must be routed for internal review before being sent to the employer group contact via DocuSign.

## Acceptance Criteria

Given an Opportunity where Opportunity.RecordType = 'Group Employer' and Opportunity.StageName transitions to 'Closed Won',
When the record is saved,
Then a Group_Service_Agreement__c record must be auto-created with GSA__c.Status__c = 'Draft', GSA__c.Employer_Account__c = Opportunity.AccountId, GSA__c.Effective_Date__c = Opportunity.Contract_Start_Date__c, GSA__c.LOB__c = Opportunity.Line_of_Business__c, and GSA__c.Negotiated_Rate_Basis__c = Opportunity.Rate_Basis__c.

Given the GSA draft is created,
When the Conga CLM template merge runs,
Then the generated document must include all fields from GSA__c and the associated Quote line items, and must be attached to the GSA__c record as a ContentDocument with Document_Type__c = 'Group Service Agreement Draft'.

Given the Conga document is generated,
When the document is attached to GSA__c,
Then a Salesforce Approval Process must be triggered routing the GSA to the Underwriting Analyst's manager (as defined in the Salesforce role hierarchy) for internal review before external distribution.

Given the internal reviewer approves the GSA,
When Approval Process step 1 is approved,
Then a DocuSign envelope must be automatically sent to the employer group's primary contact (Account.Primary_Contact__c) for e-signature, and GSA__c.Status__c must update to 'Pending Signature'.

Given a GSA where the employer group's SIC code falls under a regulated industry (GSA__c.Regulated_Industry__c = true),
When the GSA is created,
Then GSA__c.Compliance_Review_Required__c must be set to true and the record must be assigned to the Compliance Officer queue before the Conga merge step runs.

-GSA must also auto-create individual Provider_Network_Agreement__c records for all in-network providers at the time of GSA approval — descoped to Phase 3 per volume complexity assessment-

## Solution

Implement a Record-Triggered Flow on Opportunity firing After Save on StageName = 'Closed Won' with RecordType filter for 'Group Employer'. Flow creates GSA__c via Create Records element mapping Opportunity fields. A Conga CLM API callout is made via an Apex invocable action (CongaGSAMergeService.cls) using the GSA template ID stored in Custom Metadata (Conga_Template_Config__mdt). The Apex class uses a named credential (CongaCLM_Endpoint) for authentication. Upon document attachment via ContentDocumentLink, a Platform Event (GSA_Document_Ready__e) triggers the Salesforce Approval Process submission via Process.SubmitRequest in a subscriber Apex handler. DocuSign envelope creation uses the DocuSign for Salesforce managed package action configured in the Approval Process final-approval action. Regulated industry routing handled by a Decision element in the originating Flow before the Conga callout step.

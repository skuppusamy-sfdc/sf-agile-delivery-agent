## Description

**Bug Report — UAT Sprint 15**

During UAT cycle for COB secondary payer scenarios, the Explanation_of_Benefits__c record is not reflecting the correct adjudicated amounts on individual Claim_Line__c records when a secondary payer payment is applied.

**Steps to Reproduce:**
1. Create a Policy__c record with LOB = 'Medical' and COB indicator = TRUE.
2. Create a Claim__c with Status__c = 'In Adjudication' linked to the above policy.
3. Add three Claim_Line__c records with CPT codes 99213, 99214, 99215 and billed amounts $150, $200, $250.
4. Apply primary payer adjudication — set Claim_Line__c.Primary_Paid_Amount__c for each line.
5. Trigger secondary COB payer logic via the 'Apply COB' quick action.
6. Navigate to the associated Explanation_of_Benefits__c record.

**Expected Result:**
Each Claim_Line__c.EOB_Adjudicated_Amount__c should reflect the net amount after COB secondary payer offset. EOB header Explanation_of_Benefits__c.Total_Member_Responsibility__c should sum correctly.

**Actual Result:**
Claim_Line__c.EOB_Adjudicated_Amount__c remains NULL for all lines. Explanation_of_Benefits__c.Total_Member_Responsibility__c displays the pre-COB billed total with no secondary offset applied.

**Environment:** Full Sandbox FS-UAT01
**Frequency:** 100% reproducible
**Severity:** High — blocks EOB generation for all COB claims in Phase 2 scope.

## Solution

Root cause identified: the Record-Triggered Flow 'Update_EOB_On_COB_Application' has a faulty entry condition — it fires on Claim__c update but the field reference for secondary payer payload uses an incorrect merge field path (Claim__c.COB_Secondary_Payload__r.Amount__c instead of Claim__c.Secondary_Payer_Detail__r.Adjusted_Amount__c). Fix involves: (1) correcting the field reference in the Flow's Update Records element for Claim_Line__c loop, (2) adding a null-safe formula check before summing into Explanation_of_Benefits__c.Total_Member_Responsibility__c via an Apex-invocable method 'EOBRollupService.recalculateMemberResponsibility()', (3) deploying updated Flow version and retiring v3 via Change Set to FS-UAT01.

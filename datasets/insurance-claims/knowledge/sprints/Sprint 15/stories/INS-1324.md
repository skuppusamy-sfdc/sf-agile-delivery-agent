## Description

As a Claims Operations Manager, I need an Agentforce AI triage assistant embedded in the FNOL intake screen so that first-notice-of-loss submissions are automatically categorized by LOB, initial coverage verified against the active Policy__c, and low-complexity claims are routed for straight-through processing without manual adjuster intervention.

The assistant should reduce average FNOL handling time by surfacing relevant policy details, flagging potential SIU referral indicators, and pre-populating Claim__c fields from policyholder-provided information.

## Acceptance Criteria

Given a Claims Operations Manager initiates a new FNOL from the Claim__c new record screen,
When the Agentforce assistant activates,
Then it queries the related Policy__c record and displays Policy__c.Coverage_Type__c, Policy__c.Policy_Status__c, Policy__c.Effective_Date__c, and Policy__c.Expiration_Date__c in the AI summary panel before any fields are manually entered.

Given the policyholder describes the loss event in the FNOL narrative text field,
When the Agentforce assistant processes the input,
Then it auto-suggests Claim__c.Loss_Type__c and Claim__c.LOB__c values with a confidence score >= 0.80 displayed to the adjuster, who can accept or override.

Given the AI triage assigns a Complexity_Score__c <= 3 (out of 10),
When the FNOL is submitted,
Then Claim__c.Routing_Flag__c is set to 'Straight-Through' and the claim is automatically assigned to the 'Auto-Adjudication' queue without adjuster assignment.

Given the Agentforce assistant detects two or more SIU indicator patterns (e.g., duplicate provider NPI, loss date within 30 days of policy inception, prior claim within 90 days),
When the FNOL is submitted,
Then Claim__c.SIU_Referral_Flag__c = TRUE and a Task is auto-created and assigned to the SIU queue with a summary of flagged indicators.

-Given the assistant should auto-approve claims under $500 without adjuster review- (deprecated — legal review determined auto-approval requires additional compliance sign-off; deferred to INS-1340)

Given the Agentforce session is completed,
When the FNOL record is saved,
Then an AI_Interaction_Log__c record is created capturing AI_Model_Version__c, Confidence_Scores__c (JSON), and Adjuster_Overrides__c for auditability.

## Solution

Configure an Agentforce Agent of type 'Claims Triage Assistant' using the Agentforce Agent Builder. Define custom Agent Actions: (1) GetPolicyCoverageAction — Apex-backed action querying Policy__c by policyholder TIN/NPI; (2) TriageClaimComplexityAction — invokes an Einstein prompt template trained on historical Claim__c data; (3) SIUIndicatorScanAction — Apex service (SIUIndicatorService.cls) running rule-based checks. Embed the agent in the FNOL Lightning Record Page via the Agentforce component. AI_Interaction_Log__c creation handled by a Record-Triggered Flow on Claim__c after insert. Routing to queues managed by an Omni-Channel routing configuration referencing Complexity_Score__c.

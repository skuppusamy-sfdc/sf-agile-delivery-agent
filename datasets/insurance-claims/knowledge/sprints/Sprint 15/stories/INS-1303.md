## Description

As a Claims Operations Manager, I need a Coordination of Benefits (COB) summary panel embedded on the Policyholder's Account page layout that displays all active Policy__c records for the member, their coverage priority order, and any open COB-related Claim__c records flagged for dual coverage review, so that claims adjusters can make accurate primary/secondary payer determinations without leaving the member record.

## Acceptance Criteria

Given a member Account record with multiple active Policy__c records (Policy__c.Status__c = 'Active' and Policy__c.COB_Applicable__c = true), when the COB panel LWC loads, then it must display each policy with: Policy__c.Policy_Number__c, Policy__c.Carrier_Name__c, Policy__c.Coverage_Priority__c (Primary/Secondary/Tertiary), and Policy__c.Effective_Date__c.

Given the COB panel is loaded, when a Claim__c exists with Claim__c.COB_Flag__c = true linked to the member, then those claims must appear in a 'Pending COB Review' sub-section showing Claim__c.Claim_Number__c, Claim__c.Total_Billed_Amount__c, Claim__c.Status__c, and Claim__c.Primary_Payer__c.

Given a member has no COB-applicable policies, when the panel renders, then it must display an empty-state message: 'No active COB arrangements on file for this member.'

Given a COB claim is resolved (Claim__c.COB_Flag__c = true AND Claim__c.Status__c = 'Adjudicated'), when the panel refreshes, then the claim must move from 'Pending COB Review' to a collapsible 'Resolved COB Claims' section.

-Given the panel must show Medicare/Medicaid CMS eligibility verification status inline – descoped per Compliance review, targeted for Phase 3.-

## Solution

Build LWC component 'cobSummaryPanel' using @wire(getRecord) for Policy__c retrieval filtered by AccountId and COB_Applicable__c = true. Use a secondary @wire or imperative Apex call (COBClaimsController.getCOBClaims) to fetch related Claim__c records with COB_Flag__c = true. Implement collapsible sections using SLDS accordion markup. Add the component to the Account Lightning Page via App Builder with visibility filter: Account.RecordType = 'Policyholder'. Ensure CRUD/FLS checks in all Apex methods. Add component to the Policyholder permission set profile.

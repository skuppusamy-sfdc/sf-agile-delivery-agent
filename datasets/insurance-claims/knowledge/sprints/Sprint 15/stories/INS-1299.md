## Description

As a policyholder (authenticated Experience Cloud user), I need to view real-time ERA payment status for my processed claims and download my Explanation of Benefits (EOB) documents directly from my Policyholder 360 portal page so that I can reconcile my medical expenses, understand my cost-sharing responsibilities, and reduce inbound calls to the member services team.

This story enhances the existing Policyholder 360 LWC dashboard introduced in Phase 1 (INS-1198). ERA data is sourced from the Payment__c object which is updated via the CMS/clearinghouse ERA inbound integration.

## Acceptance Criteria

Given an authenticated policyholder views their Policyholder 360 Experience Cloud page,
When the Claims & Payments section loads,
Then all Claim__c records where Claim__c.Policy__c = the policyholder's Policy__c.Id are displayed in a paginated list (10 per page) sorted by Claim__c.Service_Date__c descending, showing Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, and Claim__c.Total_Paid_Amount__c.

Given a Claim__c record has one or more related Payment__c records,
When the policyholder expands a claim row,
Then each Payment__c is listed with Payment__c.ERA_Transaction_ID__c, Payment__c.Payment_Date__c, Payment__c.Payment_Amount__c, and Payment__c.Payment_Status__c (displayed as a colored pill: 'Paid' = green, 'Pending' = yellow, 'Reversed' = red).

Given a Claim__c record has a related Explanation_of_Benefits__c record where EOB.Document_Generated__c = true,
When the policyholder views the claim row,
Then a 'Download EOB' button is visible that initiates a secure download of the ContentDocument linked to the EOB record via a Salesforce Guest User sharing-safe Apex controller.

Given the policyholder clicks 'Download EOB',
When the download is initiated,
Then an audit record is created on EOB_Download_Log__c with fields: Policyholder__c, EOB__c, Download_DateTime__c, and IP_Address__c (captured from request headers) for compliance logging.

-Given the policyholder has claims older than 3 years, display them in a separate 'Archived Claims' accordion section.- (Deferred to Phase 3 — archive data model not yet finalized)

Given the policyholder's Policy__c.Status__c = 'Lapsed' or 'Cancelled',
When the Claims & Payments section loads,
Then a banner displays: 'Your policy is no longer active. Historical claims and EOBs are available below for reference only.' and all action buttons (e.g., file new claim) are hidden.

## Solution

Extend the existing 'policyholderClaimsDashboard' LWC (Phase 1 component from INS-1198) with a new child component 'claimsPaymentDetail' for ERA expansion rows. Wire methods use @wire(getRecord) for Claim__c and a custom Apex controller (PolicyholderERAController.cls) with @AuraEnabled(cacheable=true) for Payment__c retrieval — controller enforces WITH SECURITY_ENFORCED to respect field-level security. EOB download uses a non-cacheable Apex method (EOBDownloadController.getSecureDownloadURL) returning a time-limited ContentDistribution URL. Audit logging to EOB_Download_Log__c via a Platform Event (EOB_Download_Event__e) to avoid DML in the download controller context. Add the new LWC to the Policyholder Experience Cloud page via Experience Builder. Permission Set (Policyholder_Claims_Viewer_PS) controls access to Payment__c and EOB fields for community users.

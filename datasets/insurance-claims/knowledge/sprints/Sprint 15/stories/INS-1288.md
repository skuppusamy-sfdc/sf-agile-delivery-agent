## Description

As a Provider Network Specialist, I need the system to automatically generate a Provider_Change_Notification__c record and dispatch a DocuSign envelope to the affected provider when key terms on a Provider_Network_Agreement__c are amended (e.g., reimbursement rate changes, LOB additions/removals, or network tier reclassification), so that we maintain audit compliance and eliminate the current manual PDF-and-email process that causes 2–3 day notification delays.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record is saved with changes to any of the following fields: Reimbursement_Rate__c, LOB__c, Network_Tier__c, or Effective_Date__c,
When the record-triggered automation fires (after save, update context),
Then a new Provider_Change_Notification__c record is created with PCN.Status__c = 'Pending Signature', PCN.PNA_Reference__c = the amended PNA Id, PCN.Change_Type__c populated based on which field changed, and PCN.Notification_Date__c = TODAY().

Given the Provider_Change_Notification__c record is created with Status__c = 'Pending Signature',
When the Platform Event 'PCN_Created__e' is published,
Then a DocuSign envelope is dispatched to Provider__c.Email__c within 5 minutes containing the system-generated PCN PDF (rendered via Conga CLM template 'PCN_Standard_v3').

Given the DocuSign envelope is sent,
When the provider completes the signature,
Then PCN.Status__c updates to 'Executed', PCN.Signed_Date__c is populated, and the signed document is attached to the PCN record as a Salesforce File.

Given the DocuSign envelope is NOT signed within 10 business days,
When the scheduled Apex job 'PCNEscalationBatch' runs nightly,
Then PCN.Status__c updates to 'Escalated', a Task is created assigned to the owning Provider Network Specialist, and a Chatter post is added to the PNA record tagging the assigned specialist.

Given a PNA amendment is made by a user with the 'Read_Only_Network' permission set,
When the record is saved,
Then no PCN record is created and a validation rule error is displayed: 'You do not have permission to amend PNA terms. Contact your Network Contracting Manager.'

-Given LOB changes, generate a separate GSA amendment notification in addition to the PCN- (descoped to Phase 3 per INS-1280 backlog refinement).

## Solution

Implement a Record-Triggered Flow 'RTF_PNA_Amendment_PCN_Generator' (After Save, fires on Update when any of Reimbursement_Rate__c, LOB__c, Network_Tier__c, Effective_Date__c are changed) to create the Provider_Change_Notification__c record and publish a Platform Event 'PCN_Created__e' with the PCN Id and provider email as payload fields. A separate Event-Triggered Flow subscribes to 'PCN_Created__e' and calls an Apex invocable action 'DocuSignEnvelopeService.dispatchPCNEnvelope()' which uses the DocuSign eSignature API (JWT auth) and Conga CLM REST API to merge the PCN template. DocuSign callback webhook updates PCN status via an unauthenticated Site endpoint. Scheduled Apex 'PCNEscalationBatch' queries PCN records where Status__c = 'Pending Signature' AND Notification_Date__c <= TODAY()-10 and performs bulk escalation. Validation rule added directly on Provider_Network_Agreement__c using $Permission.Read_Only_Network custom permission check.

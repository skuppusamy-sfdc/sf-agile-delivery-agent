## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces the provider's current network status, all active Provider_Network_Agreement__c (PNA) records, open Credentialing_Case__c items, and any pending Provider_Change_Notification__c (PCN) records so that I have a single consolidated view without navigating across multiple related lists.

The FlexCard must be performant — loading within 2 seconds — and must respect field-level security for sensitive credentialing data.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then it displays Provider__c.Network_Status__c, Provider__c.NPI__c, Provider__c.TIN__c, and Provider__c.Primary_Specialty__c in the header panel.

Given the FlexCard header is rendered,
When the Active Agreements section loads,
Then all Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active' AND PNA__c.Provider__c = current record Id are listed with PNA__c.Contract_Effective_Date__c and PNA__c.LOB__c visible.

Given the Credentialing section renders,
When Credentialing_Case__c records are fetched,
Then only cases where Credentialing_Case__c.Status__c IN ('Open','In Review','Pending CAQH') are shown, sorted by Credentialing_Case__c.Due_Date__c ascending.

Given a pending PCN exists,
When Provider_Change_Notification__c.Status__c = 'Pending Approval' for the provider,
Then a yellow warning badge displays on the FlexCard header reading 'Pending Change Notification'.

Given the user does not have the 'View_Credentialing_Sensitive' Permission Set,
When the FlexCard renders,
Then Credentialing_Case__c.CAQH_Password__c and Credentialing_Case__c.SSN_Last4__c fields are masked and display as '****'.

## Solution

Build an OmniStudio FlexCard (Name: Provider360_Card) backed by an Integration Procedure (IP: Provider360_DataFetch) that uses DataRaptor Extract steps to query Provider__c, PNA__c, Credentialing_Case__c, and PCN__c in parallel using the 'Parallel' node to meet the 2-second SLA. Apply FLS checks within the Integration Procedure using a conditional branching step that reads the running user's Permission Set assignments via a SOQL DataRaptor. Use FlexCard conditional visibility states to show/hide the PCN warning badge. Deploy the FlexCard as a Lightning Web Component wrapper on the Provider__c record page via App Builder. Cache Integration Procedure responses using OmniStudio cache settings with a 60-second TTL to reduce SOQL calls for high-frequency page loads.

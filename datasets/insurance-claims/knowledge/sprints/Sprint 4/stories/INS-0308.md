## Description

As a Claims Operations Manager, I need the system to automatically adjudicate clean medical claims where the total billed amount is under $500 and all Claim_Line__c items reference in-network providers, so that low-complexity claims are resolved without manual review, improving throughput and reducing average claim cycle time.

A 'clean claim' is defined as: (1) no duplicate claim flag, (2) all CPT codes on Claim_Line__c records are valid and present on the active Fee_Schedule__c for the provider's Provider_Network_Agreement__c (PNA), (3) ICD-10 diagnosis code on Claim__c.Primary_Diagnosis_Code__c is present and passes CMS format validation, (4) Claim__c.COB_Required__c = FALSE.

## Acceptance Criteria

Given a Claim__c record in Status__c = 'FNOL Received' is saved or updated,
When the Record-Triggered Flow evaluates the auto-adjudication entry criteria,
Then the flow must only proceed if ALL of the following are true:
  - Claim__c.Total_Billed_Amount__c < 500
  - Claim__c.Provider_Network_Status__c = 'In-Network'
  - Claim__c.COB_Required__c = FALSE
  - Claim__c.Duplicate_Claim_Flag__c = FALSE
  - Claim__c.SIU_Referral__c = FALSE

Given the entry criteria are met,
When the adjudication Apex service (ClaimAdjudicationService.adjudicateClaimLines()) executes,
Then for each Claim_Line__c child record:
  - Claim_Line__c.Allowed_Amount__c must be populated from the matching Fee_Schedule__c line where Fee_Schedule__c.CPT_Code__c = Claim_Line__c.CPT_Code__c and Fee_Schedule__c.Effective_Date__c <= Claim__c.Date_of_Service__c and Fee_Schedule__c.Expiration_Date__c >= Claim__c.Date_of_Service__c
  - Claim_Line__c.Adjudication_Status__c = 'Approved'
  - Claim_Line__c.Denial_Reason_Code__c must be NULL

Given all Claim_Line__c records are adjudicated as 'Approved',
When the parent Claim__c is updated by the service,
Then:
  - Claim__c.Status__c = 'Adjudicated'
  - Claim__c.Adjudication_Method__c = 'Auto'
  - Claim__c.Adjudicated_Date__c = TODAY()
  - Claim__c.Total_Allowed_Amount__c = SUM of all Claim_Line__c.Allowed_Amount__c
  - A Payment__c record is created with Payment__c.Status__c = 'Pending' and Payment__c.Payment_Amount__c = Claim__c.Total_Allowed_Amount__c

Given any single Claim_Line__c CPT code is NOT found on the active Fee_Schedule__c,
When the adjudication service encounters the missing fee schedule entry,
Then:
  - Claim_Line__c.Adjudication_Status__c = 'Pended'
  - Claim_Line__c.Denial_Reason_Code__c = 'FS-001: CPT Not on Fee Schedule'
  - Claim__c.Status__c = 'Pended — Manual Review'
  - Claim__c.Assigned_Queue__c = 'Claims Adjudication Review'
  - No Payment__c record is created

Given Claim__c.Total_Billed_Amount__c >= 500 or any other entry criterion fails,
When the Record-Triggered Flow evaluates the criteria,
Then the claim is routed to manual review:
  - Claim__c.Status__c = 'Pending Manual Review'
  - Claim__c.Adjudication_Method__c = 'Manual'
  - Claim__c.Assigned_Queue__c = 'Claims Adjudication Review'

-Given Claim__c.LOB__c = 'Auto', the auto-adjudication engine applies the same rules-
Note: Auto-adjudication engine is scoped to LOB__c = 'Medical' only in Phase 1. Auto LOB adjudication deferred to Phase 2.

## Solution

Implement a Record-Triggered Flow (Object: Claim__c, Trigger: After Save, entry criteria as specified) that calls an Apex invocable action (ClaimAdjudicationService.adjudicateClaimLines) via an Action element. The Apex class queries all child Claim_Line__c records, performs a bulk SOQL lookup against Fee_Schedule__c filtered by CPT_Code__c, Effective_Date__c, and Expiration_Date__c, and calculates Allowed_Amount__c per line. A separate Apex method handles the parent Claim__c rollup and Payment__c record creation in a single DML transaction. Use a Custom Metadata Type (Adjudication_Rule__mdt) to store the $500 threshold and other configurable rule parameters to avoid hardcoding. Implement a duplicate check via a before-insert Apex Trigger on Claim__c that queries existing claims by Policy__c, Date_of_Service__c, and Total_Billed_Amount__c and sets Duplicate_Claim_Flag__c accordingly. All adjudication actions are logged to a custom Adjudication_Audit__c object for compliance reporting.

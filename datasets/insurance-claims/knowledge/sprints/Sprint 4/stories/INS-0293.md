## Description

As a Claims Operations Manager, I need a Policyholder 360 FlexCard embedded on the Contact/Person Account record page so that I can see all active policies, open claims, and recent payment disbursements in a single view without navigating to multiple objects.

The FlexCard must display: active Policy__c records, open Claim__c records (Status__c not in 'Closed','Denied'), and the last 5 Payment__c records linked to the policyholder. Extends INS-0278 (Policyholder record page layout redesign).

## Acceptance Criteria

Given a Claims Operations user views a Person Account record page,
When the Policyholder 360 FlexCard loads,
Then it displays all Policy__c records where Policy__c.Policyholder__c = current Account Id and Policy__c.Status__c = 'Active', showing Policy Number, LOB, and Effective/Expiration dates.

Given the FlexCard is loaded,
When the Claims panel is rendered,
Then all Claim__c records where Claim__c.Status__c NOT IN ('Closed', 'Denied') are listed with Claim__c.Claim_Number__c, Claim__c.Date_of_Loss__c, and Claim__c.Status__c.

Given the FlexCard is loaded,
When the Payments panel renders,
Then the 5 most recent Payment__c records (ordered by Payment__c.Payment_Date__c DESC) are shown with Payment__c.Amount__c and Payment__c.Payment_Method__c.

Given there are no active policies for the policyholder,
When the Policies panel renders,
Then an empty-state message 'No active policies found' is displayed.

Given a user clicks a Claim Number hyperlink in the FlexCard,
When the link is activated,
Then the user is navigated to the corresponding Claim__c record detail page.

## Solution

Create an OmniStudio FlexCard (Name: Policyholder360_FC) with three card blocks: Policies, Claims, Payments. Use three DataRaptor Extract data sources querying Policy__c, Claim__c, and Payment__c respectively, each filtered by Account Id passed as input parameter. Embed the FlexCard on the Person Account Lightning Record Page via the OmniStudio FlexCard component. Use flyout child cards for claim line details. Apply Sharing Rules to respect Claims Operations visibility. Cache DataRaptor results for 60 seconds to reduce SOQL load.

## Description

As a Claims Operations Manager, I need a guided FNOL intake OmniScript so that adjusters and self-service policyholders can submit first notice of loss details accurately and consistently across Auto and Medical LOBs, reducing manual data entry errors and accelerating Claim__c record creation.

## Acceptance Criteria

Given a policyholder or adjuster initiates a new FNOL,
When they launch the FNOL OmniScript from the Policy__c record,
Then the script pre-populates Policy__c.PolicyNumber__c, Policy__c.LOB__c, and insured name from the active policy context.

Given the LOB is 'Medical',
When the user reaches the incident details step,
Then ICD-10 diagnosis code lookup (minimum 3-character typeahead) and Date_of_Service__c fields are displayed and required.

Given the LOB is 'Auto',
When the user reaches the incident details step,
Then fields for Accident_Date__c, Accident_State__c, Vehicle_VIN__c, and At_Fault_Indicator__c are displayed.

Given all required fields are completed,
When the user submits the OmniScript,
Then a Claim__c record is created with Claim__c.Status__c = 'FNOL Received', Claim__c.Source__c = 'Self-Service' or 'Adjuster' based on running user profile, and a unique Claim_Number__c is auto-generated via custom autonumber.

Given the Claim__c record is created,
When the OmniScript completes,
Then a confirmation screen displays the Claim_Number__c and a DocuSign envelope is triggered for the claimant's signed statement of loss.

-Given the LOB is 'Workers Comp', a separate WC intake step is shown- (deprecated — WC FNOL deferred to Phase 2; see INS-0251)

Given a required field is left blank on submission,
When the user clicks 'Submit',
Then inline validation messages appear per field and the OmniScript does not advance.

## Solution

Build a multi-step OmniScript (Type: FNOL, Sub-Type: Intake) using OmniStudio. Use DataRaptor Extract to pull Policy__c context on launch. Conditional step rendering driven by LOB__c picklist value via SetValues and Conditional Views. On final step, invoke a DataRaptor Post to create Claim__c and related Claim_Line__c stub. Trigger DocuSign envelope via Integration Procedure calling DocuSign REST API. Auto-number configured on Claim__c object (format: CLM-{YYYY}-{00000}). Deploy OmniScript as an embedded LWC on the Policy__c Lightning Record Page via a custom tab.

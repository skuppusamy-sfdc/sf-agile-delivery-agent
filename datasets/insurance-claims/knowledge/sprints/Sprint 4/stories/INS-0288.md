## Description

As a Provider Relations Specialist, I need automated alerts when a Provider_Network_Agreement__c (PNA) is approaching its termination date so that I can proactively initiate renewal negotiations and avoid unintended provider network gaps that could disrupt claims processing.

Alerts must fire at 60 days and 30 days prior to PNA_Termination_Date__c.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record with PNA_Status__c = 'Active' exists in the system,
When the scheduled job runs daily at 06:00 UTC,
Then any PNA where DATEDIFF(PNA_Termination_Date__c, TODAY()) = 60 must trigger a Task created and assigned to Provider_Network_Agreement__c.Owner with Subject = 'PNA Expiry Alert: 60 Days — [Provider Name]' and Due Date = TODAY().

Given the same daily job runs,
When DATEDIFF(PNA_Termination_Date__c, TODAY()) = 30,
Then a high-priority Task (Priority = 'High') must be created AND an email notification must be sent to the owning Provider Relations Specialist and their manager via an Email Alert.

Given a PNA has already had a 60-day alert Task created (identified by Task.Subject containing '60 Days'),
When the 30-day threshold is reached,
Then a new separate Task must be created — the system must not duplicate or update the existing 60-day Task.

Given a PNA is renewed (PNA_Status__c updated to 'Renewed' or PNA_Termination_Date__c extended),
When the scheduled job runs,
Then no further expiry alerts must be generated for that PNA record.

-Given PNA_Status__c = 'Pending Signature', the same 60/30-day alerts should fire- (Descoped; pending-signature PNAs excluded from alert logic per compliance review)

## Solution

1. Schedulable Apex class (PNAExpiryAlertScheduler.cls) implementing Schedulable interface, scheduled via Setup > Scheduled Jobs at 06:00 UTC daily.
2. SOQL query within scheduler bulkified to retrieve all Active PNAs where Termination_Date__c = TODAY()+60 OR Termination_Date__c = TODAY()+30.
3. Apex helper (PNAAlertHelper.cls) creates Task records in bulk using Database.insert with allOrNone=false.
4. Email Alert (PNA_30Day_Expiry_Alert) configured on Provider_Network_Agreement__c object, triggered by Apex via Messaging.sendEmail for 30-day threshold.
5. Custom field PNA_Alert_60_Sent__c (Checkbox) and PNA_Alert_30_Sent__c (Checkbox) on Provider_Network_Agreement__c to prevent duplicate alert generation.
6. Apex test class with mock date injection to simulate 60-day and 30-day scenarios.

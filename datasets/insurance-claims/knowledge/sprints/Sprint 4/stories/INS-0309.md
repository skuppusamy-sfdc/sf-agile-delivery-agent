## Description

**Bug Report — Claim Status Regression on Child Record Update**

**Environment:** Acme Insurance Full Sandbox (INS-FULL-SBX)
**Discovered During:** INS-0308 adjudication rule engine UAT — Sprint 4

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record where Claim__c.Status__c = 'Adjudicated' and Claim__c.Adjudication_Method__c = 'Auto'.
2. Open any related Claim_Line__c child record.
3. Edit any field on the Claim_Line__c record (e.g., update Claim_Line__c.Units__c from 1 to 2 for testing purposes) and save.
4. Navigate back to the parent Claim__c record.
5. Observe Claim__c.Status__c field value.

**Expected Result:**
Claim__c.Status__c remains 'Adjudicated'. Child record edits to Claim_Line__c should not alter the parent claim status unless triggered by a deliberate re-adjudication action.

**Actual Result:**
Claim__c.Status__c reverts to 'FNOL Received' immediately after the Claim_Line__c save. The Claim__c.Adjudicated_Date__c is also cleared (set to NULL). The Payment__c record previously created remains in 'Pending' status but is now orphaned from the parent claim's status context, creating a data integrity risk.

**Root Cause Hypothesis:**
The Record-Triggered Flow on Claim__c (deployed as part of INS-0308) appears to be firing on an indirect record update triggered by the Claim_Line__c roll-up summary field recalculation on Claim__c.Total_Billed_Amount__c. This is causing the flow's 'New Claim Intake' path (which sets Status__c = 'FNOL Received') to execute erroneously because the flow entry condition does not exclude records already in a terminal adjudication status.

**Frequency:** 100% reproducible in sandbox.
**Severity:** High — blocks UAT sign-off for INS-0308.

## Solution

Investigate the Record-Triggered Flow entry criteria on Claim__c. Add an explicit exclusion condition to the flow's entry filter: Claim__c.Status__c NOT IN ('Adjudicated', 'Pended — Manual Review', 'Denied', 'Closed', 'Pending Manual Review') to prevent re-entry for terminal-state records. Additionally, review whether the roll-up summary field on Claim__c.Total_Billed_Amount__c is causing a spurious record update that triggers the flow; if so, consider converting the roll-up to an Apex-calculated field updated only during explicit adjudication service calls. Add a flow fault path and a debug log review step. Regression test all Claim__c status transition scenarios in the INS-FULL-SBX sandbox prior to promoting the fix to UAT.

## Description

As a Claims Operations Manager, I need a Policyholder 360 FlexCard displayed on the Contact (Person Account) record page so that I can instantly see all active policies, open claims, and recent payment history for a given policyholder without navigating to multiple objects.

The FlexCard should surface:
- Active Policy__c records (Status__c = 'Active') with policy number, LOB, and expiration date
- Open Claim__c records (Status__c not in 'Closed','Denied') with claim number, date of loss, and current priority
- Last 3 Payment__c records with amount, date, and status
- COB indicator if the policyholder has more than one active Policy__c

## Acceptance Criteria

Given a Person Account record page loads for a policyholder with 2 active policies,
When the Policyholder 360 FlexCard renders,
Then both Policy__c records with Policy__c.Status__c = 'Active' are displayed with Policy__c.PolicyNumber__c, Policy__c.LOB__c, and Policy__c.ExpirationDate__c visible.

Given the policyholder has 3 open claims,
When the FlexCard renders,
Then all 3 Claim__c records where Claim__c.Status__c NOT IN ('Closed','Denied') appear with Claim__c.Name, Claim__c.DateOfLoss__c, and Claim__c.Priority__c.

Given the policyholder has more than one active Policy__c,
When the FlexCard renders,
Then a COB warning badge ('COB Review Required') is displayed prominently in the card header.

Given the policyholder has no open claims,
When the FlexCard renders,
Then the Claims section displays 'No open claims on file' — no blank panel or error state.

Given a Payment__c record has Status__c = 'Failed',
When it appears in the Last 3 Payments section,
Then it is highlighted in red with a 'Failed' badge.

Given the FlexCard is viewed by a user without the 'Claims_Read' Permission Set,
When the page loads,
Then the Claims section is hidden and a message 'Claims data not available for your role' is shown.

## Solution

Build OmniStudio FlexCard ('FC_Policyholder360') using three child DataRaptor Extract actions:
1. 'DR_GetActivePolicies' — queries Policy__c WHERE PolicyHolder__c = :recordId AND Status__c = 'Active'
2. 'DR_GetOpenClaims' — queries Claim__c WHERE Policyholder__c = :recordId AND Status__c NOT IN ('Closed','Denied')
3. 'DR_GetRecentPayments' — queries Payment__c WHERE Policyholder__c = :recordId ORDER BY PaymentDate__c DESC LIMIT 3

COB badge logic: conditional state on FlexCard evaluating count of active policies > 1.
Payment failure highlight: FlexCard state styling using conditional CSS class on Payment__c.Status__c = 'Failed'.
Permission gating: use FlexCard conditional visibility tied to Custom Permission 'Claims_Read_CP'.
Deploy FlexCard to Person Account Lightning Record Page via App Builder.

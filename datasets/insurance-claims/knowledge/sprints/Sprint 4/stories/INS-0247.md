## Description

As a Claims Operations Manager, I need the Salesforce platform to automatically validate coverage eligibility when a new Claim__c record transitions to 'Under Review' status so that ineligible claims are flagged immediately for manual review rather than progressing through the adjudication pipeline, reducing downstream payment errors and compliance risk. Extends INS-0245.

## Acceptance Criteria

Given a Claim__c record transitions from Status__c = 'FNOL Received' to Status__c = 'Under Review',
When the adjudication eligibility rule fires,
Then the system checks that Claim__c.Date_of_Loss__c is between Policy__c.Effective_Date__c and Policy__c.Expiration_Date__c on the associated Policy__c record.

Given the Date_of_Loss__c falls outside the policy effective period,
When the eligibility check fails,
Then Claim__c.Status__c is set to 'Pended — Eligibility' and Claim__c.Adjudication_Denial_Reason__c is populated with 'Policy not active on date of loss' and a Case is created with Subject = 'Eligibility Review Required' linked to the Claim__c.

Given the Date_of_Loss__c is within the policy effective period,
When the system validates the LOB coverage,
Then it confirms that the Claim__c.LOB__c value matches at least one Policy__c.Coverage_Type__c entry on the related Policy__c; if no match, set Claim__c.Status__c = 'Pended — Coverage Mismatch' and populate Claim__c.Adjudication_Denial_Reason__c accordingly.

Given the claim passes both date and LOB coverage checks,
When all eligibility rules pass,
Then Claim__c.Eligibility_Validated__c (checkbox) is set to true and Claim__c.Status__c remains 'Under Review' to allow adjudication to continue.

Given any eligibility rule fails,
When the pend action completes,
Then a Platform Event (Claim_Status_Change__e) is published with fields: Claim_Id__c, Previous_Status__c, New_Status__c, and Reason__c so that downstream subscribers (e.g., notification service) can react.

Given the Claim__c.LOB__c = 'Health',
When eligibility validation runs,
Then the system additionally checks that Claim__c.Treating_Provider_NPI__c resolves to an active Provider__c record where Provider__c.Network_Status__c = 'In-Network' on the related Provider_Network_Agreement__c; if out-of-network, set Claim__c.Network_Flag__c = 'Out-of-Network' without blocking adjudication.

## Solution

Implement an Apex Trigger (ClaimAdjudicationTrigger) on Claim__c for the 'after update' event, delegating logic to ClaimEligibilityService.cls. The service queries Policy__c via a SOQL join on Claim__c.Policy__c and evaluates date and LOB rules in a stateless, bulkified pattern. On failure, use a separate CaseCreationService.cls to insert the review Case. Publish Claim_Status_Change__e Platform Event via EventBus.publish(). All rule thresholds stored in Custom Metadata Type (Adjudication_Rule__mdt) for admin configurability without code deployments. Unit test coverage target ≥ 85%.

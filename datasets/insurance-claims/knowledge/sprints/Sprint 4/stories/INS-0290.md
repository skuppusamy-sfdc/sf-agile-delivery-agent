## Description

As a Claims Operations Manager, I need Agentforce to surface AI-suggested pend reasons on Claim__c records that are routed to 'Pending Manual Review' so that claims adjusters can quickly understand why a claim was flagged and take action faster, reducing average handle time.

The AI suggestion should analyze Claim_Line__c anomalies such as duplicate CPT codes, out-of-range billed amounts relative to Fee_Schedule__c, and mismatched ICD-10 diagnosis-to-procedure pairings.

## Acceptance Criteria

Given a Claim__c record transitions to Status__c = 'Pending Manual Review',
When the Record-Triggered Flow fires,
Then an Agentforce Agent Action must be invoked to analyze the claim's Claim_Line__c records and return a suggested pend reason string.

Given the Agentforce action returns a suggestion,
When the response is received,
Then Claim__c.AI_Suggested_Pend_Reason__c (Long Text Area) must be populated with the suggestion and Claim__c.AI_Suggestion_Confidence__c (Percent field) must be set to the returned confidence score.

Given the AI suggestion is populated,
When the adjuster reviews the claim,
Then a 'Accept Suggestion' quick action must be available on the Claim__c record page that copies AI_Suggested_Pend_Reason__c into Claim__c.Pend_Reason__c with a single click.

Given the Agentforce action times out or returns an error,
When the invocation fails,
Then Claim__c.AI_Suggestion_Status__c must be set to 'Unavailable' and the adjuster must be able to manually enter Pend_Reason__c without obstruction.

Given Claim__c.AI_Suggestion_Confidence__c < 60,
When the suggestion is displayed,
Then a visual indicator (amber badge) must appear on the FlexCard noting 'Low Confidence Suggestion — Please Verify'.

## Solution

1. Record-Triggered Flow (RTF_InvokeAgentforceOnPend) on Claim__c, After Update, condition: Status__c CHANGED TO 'Pending Manual Review'.
2. Flow calls Apex invocable (AgentforceClaimsAnalysisAction.cls) which constructs a prompt using Einstein Prompt Template (ClaimPendReasonAnalysis_v1) referencing Claim__c and related Claim_Line__c data.
3. Prompt Template uses merge fields: {!Claim.Claim_Number__c}, {!Claim.LOB__c}, and a JSON serialization of Claim_Line__c CPT/ICD-10 pairs.
4. Agentforce Agent configured with a custom Topic (Claims Anomaly Detection) and Action bound to the Apex class.
5. New custom fields on Claim__c: AI_Suggested_Pend_Reason__c (Long Text Area 2000), AI_Suggestion_Confidence__c (Percent, 2 decimal), AI_Suggestion_Status__c (Picklist: Pending|Available|Unavailable).
6. LWC (claimAISuggestionBadge) renders the amber low-confidence indicator, embedded in the Claim__c Lightning Record Page via App Builder.
7. Quick Action (Accept_AI_Suggestion) implemented as a Screen Flow that copies field values with user confirmation step.

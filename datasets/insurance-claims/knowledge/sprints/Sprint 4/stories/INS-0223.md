## Description

As a Claims Operations Manager, I need a Policyholder 360 FlexCard embedded on the Policy__c record page that displays a real-time summary of all associated claims — including open, closed, and denied counts, most recent claim date, and total paid amount — so that I can assess a policyholder's claims history at a glance without running separate reports.

## Acceptance Criteria

Given a Claims user opens any Policy__c record page,
When the Policyholder 360 FlexCard renders,
Then it displays: total count of Claim__c records by Status__c group ('Open', 'Closed', 'Denied'), Policy__c.Total_Paid_Amount__c (rollup), and the most recent Claim__c.CreatedDate.

Given the FlexCard is rendered,
When a user clicks the 'View All Claims' action button,
Then a child FlexCard datatable expands inline showing Claim__c.Claim_Number__c, Claim__c.Date_of_Loss__c, Claim__c.Status__c, and Claim__c.Total_Claim_Amount__c for all related claims, sorted by Date_of_Loss__c descending.

Given a claim in the datatable has Status__c = 'Denied',
When the row renders,
Then the status badge displays in red with the value of Claim__c.Denial_Reason__c shown as a tooltip.

Given the FlexCard data source (DataRaptor Extract),
When the policy has zero associated claims,
Then the FlexCard displays an empty state message: 'No claims on record for this policy.' and the 'View All Claims' button is hidden.

-Given the FlexCard panel, include a 'File New Claim' quick action that launches the FNOL OmniScript inline.- (superseded — FNOL launch moved to a separate utility bar component per UX review)

## Solution

Build an OmniStudio FlexCard (Name: Policyholder_Claims_Summary) with a DataRaptor Extract as the data source, querying Claim__c records related to the Policy__c via recordId context. Use FlexCard states to toggle between summary view and expanded datatable (child FlexCard: Policyholder_Claims_Detail). Conditional styling on Status__c field using FlexCard conditional CSS classes. Embed the FlexCard on the Policy__c Lightning Record Page via the OmniStudio FlexCard component in App Builder. DataRaptor uses an aggregate SOQL query for counts and SUM of Payment__c.Amount__c. Assign visibility via Permission Set Group: Claims_Ops_PSG.

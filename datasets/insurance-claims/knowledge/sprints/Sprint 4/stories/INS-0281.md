## Description

As a Claims Operations Manager, I need an Explanation_of_Benefits__c (EOB) record automatically generated when a Claim__c reaches 'Adjudicated' status so that the policyholder and provider receive a complete, accurate summary of claim adjudication decisions, allowed amounts, member responsibility, and any denial reasons, fulfilling regulatory disclosure requirements.

The EOB must aggregate all Claim_Line__c child records and produce a structured summary document available for policyholder self-service and ERA file generation.

## Acceptance Criteria

Given Claim__c.Status__c transitions to 'Adjudicated',
When the EOB generation process triggers,
Then one Explanation_of_Benefits__c record is created linked to Claim__c via EOB.Claim__c lookup with EOB.Status__c = 'Draft'.

Given the EOB record is created,
When line-level data is aggregated,
Then EOB.Total_Billed_Amount__c = SUM(Claim_Line__c.Billed_Amount__c), EOB.Total_Allowed_Amount__c = SUM(Claim_Line__c.Allowed_Amount__c), and EOB.Member_Responsibility__c = EOB.Total_Billed_Amount__c - EOB.Total_Allowed_Amount__c.

Given any Claim_Line__c.Adjudication_Flag__c = 'Denied',
When the EOB is generated,
Then EOB.Contains_Denials__c = TRUE and each denied line's Claim_Line__c.Denial_Reason_Code__c is captured in the related EOB_Line__c.Denial_Reason__c child record.

Given the EOB record Status__c = 'Draft',
When a Compliance Officer reviews and approves it,
Then EOB.Status__c transitions to 'Final' and EOB.Finalized_Date__c = TODAY().

-Given EOB.Status__c = 'Final', the system shall automatically email the EOB PDF to the policyholder- (deprecated: email delivery moved to Phase 2 per stakeholder decision)

Given EOB.Status__c = 'Final',
When the ERA file generation batch runs,
Then the EOB data is included in the 835 ERA transaction file output.

## Solution

Implement a Record-Triggered Flow on Claim__c (trigger: after update, condition: Status__c changed to 'Adjudicated'). The flow calls an Apex invocable method (EOBGenerator.generate(claimId)) that creates the Explanation_of_Benefits__c header record and iterates over Claim_Line__c children to create EOB_Line__c detail records. Rollup summary fields on EOB__c aggregate totals from EOB_Line__c. A separate Apex Batch (ERAFileBatch) queries EOB__c records with Status__c = 'Final' and generates 835-format flat files stored as ContentDocument linked to the EOB record. A custom Lightning Web Component (eobViewer) embedded on the Claim__c record page renders EOB details using a @wire adapter. Permission Set Group 'Compliance_Officer_PSG' grants EOB approval access via a custom approval process.

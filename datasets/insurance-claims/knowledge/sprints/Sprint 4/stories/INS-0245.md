## Description

As a Claims Operations Manager, I need a guided FNOL intake OmniScript that supports both Auto and Health LOB claims so that adjusters can capture all required loss details at First Notice of Loss without switching between multiple screens or systems. The OmniScript must dynamically branch based on LOB selection and pre-populate policyholder data from the associated Policy__c record.

## Acceptance Criteria

Given a logged-in adjuster navigates to the FNOL OmniScript,
When they select LOB = 'Auto',
Then the script presents auto-specific fields: Claim__c.Date_of_Loss__c, Claim__c.Loss_Description__c, Claim__c.Vehicle_VIN__c, Claim__c.At_Fault_Party__c, and Claim__c.Accident_State__c.

Given a logged-in adjuster navigates to the FNOL OmniScript,
When they select LOB = 'Health',
Then the script presents health-specific fields: Claim__c.Date_of_Service__c, Claim__c.Primary_Diagnosis_ICD10__c, Claim__c.Treating_Provider_NPI__c, and Claim__c.Facility_Name__c.

Given the adjuster enters a valid Policy Number,
When the OmniScript DataRaptor lookup executes,
Then Policy__c.PolicyHolder__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, and Policy__c.Coverage_Type__c are pre-populated and read-only on the intake form.

Given all required fields are completed,
When the adjuster submits the FNOL form,
Then a new Claim__c record is created with Claim__c.Status__c = 'FNOL Received', Claim__c.Source__c = 'Adjuster Portal', and a system-generated Claim__c.Claim_Number__c using the format 'CLM-{YYYY}-{0000000}'.

Given a Claim__c record is successfully created,
When the OmniScript completes,
Then a confirmation screen displays the new Claim__c.Claim_Number__c and a Task is auto-created and assigned to the submitting adjuster with Subject = 'Initial Claim Review' and Due Date = TODAY() + 2 business days.

-Given the adjuster selects LOB = 'Workers Comp', the WC-specific branching step is presented- (deprecated — WC LOB deferred to Phase 2, see INS-0198).

## Solution

Build a multi-step OmniScript (Type: FNOL_Intake, Sub-Type: Auto_Health) with conditional branching steps driven by a LOB picklist. Use a DataRaptor Extract to look up Policy__c by Policy_Number__c and map fields into OmniScript elements. On submit, use a DataRaptor Transform + Integration Procedure to upsert Claim__c and trigger a Record-Triggered Flow on Claim__c insert to auto-create the follow-up Task. Deploy a FlexCard on the Claim__c record page to surface FNOL summary details post-creation. Assign access via Permission Set Group 'Claims_Adjuster_PSG'.

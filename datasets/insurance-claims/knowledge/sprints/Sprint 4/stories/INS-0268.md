## Description

As a Claims Operations Manager, I need a guided FNOL intake OmniScript that captures all initial claim details for Auto and Medical lines of business so that a Claim__c record is created with accurate, structured data from the first point of contact and downstream adjudication can begin without manual rework.

The OmniScript must support both inbound call-center entry and self-service policyholder submission, branching on LOB to collect relevant fields (e.g., accident details for Auto, diagnosis codes for Medical). Integration with the Policy__c lookup must validate policy status before allowing FNOL submission.

## Acceptance Criteria

Given a Claims Operations Manager or authenticated Policyholder initiates the FNOL OmniScript,
When they select LOB = 'Auto',
Then the script presents accident date, loss location, vehicle VIN, and at-fault party fields; ICD-10 diagnosis fields are hidden.

Given a user selects LOB = 'Medical',
When the LOB step is completed,
Then ICD-10 primary and secondary diagnosis code fields are displayed and CPT code capture is enabled; vehicle fields are suppressed.

Given the user enters a Policy Number,
When the DataRaptor lookup executes against Policy__c,
Then if Policy__c.Status__c != 'Active' an inline error reads 'Policy is not active — FNOL cannot be submitted' and the Next button is disabled.

Given all required fields are completed,
When the OmniScript final step is submitted,
Then a Claim__c record is created with Claim__c.Status__c = 'FNOL Received', Claim__c.LOB__c populated, Claim__c.Date_of_Loss__c set, and Claim__c.Policy__c lookup resolved.

Given successful Claim__c creation,
When the record is saved,
Then a Platform Event (FNOL_Submitted__e) is published containing Claim__c.Id and Claim__c.LOB__c so downstream adjudication flows can subscribe.

Given the submission completes,
When the confirmation screen renders,
Then the Claim Reference Number (Claim__c.Name) is displayed and a confirmation email is sent to the policyholder via a triggered Email Alert.

-Given the user uploads supporting documents during FNOL, When files are attached, Then they are stored as ContentDocument linked to Claim__c (deprecated — moved to INS-0271)-

## Solution

Build a multi-step OmniScript (Type: FNOL_Intake, Sub-Type: ClaimsEntry) with conditional branching on LOB picklist. Use DataRaptor Transform to query Policy__c by policy number and validate Status__c. On final step, use a DataRaptor Post to upsert Claim__c. Publish FNOL_Submitted__e Platform Event via an Integration Procedure called from the OmniScript's final action. Deploy a Record-Triggered Flow on Claim__c (After Insert, Status__c = 'FNOL Received') to send the policyholder confirmation email. Wrap the OmniScript in an LWC for embedding in the Experience Cloud self-service portal.

## Description

As a Claims Operations Manager, I need the system to automatically generate an Explanation_of_Benefits__c (EOB) record when a Claim__c transitions to Status__c = 'Adjudicated' so that policyholders receive a standardized, accurate breakdown of what was billed, what was allowed, what the plan paid, and what the member owes, fulfilling regulatory EOB delivery requirements.

The EOB must consolidate all Claim_Line__c data and be available for both policyholder portal display and ERA file generation for provider remittance. Phase 1 covers EOB record creation and portal display; ERA file transmission is covered in a subsequent story.

## Acceptance Criteria

Given a Claim__c record transitions to Status__c = 'Adjudicated',
When the EOB generation flow triggers,
Then exactly one Explanation_of_Benefits__c record is created with EOB__c.Claim__c = the adjudicated Claim__c.Id.

Given the EOB__c record is created,
When all child Claim_Line__c records are processed,
Then EOB__c.Total_Billed__c = SUM(Claim_Line__c.Billed_Amount__c), EOB__c.Total_Allowed__c = SUM(Claim_Line__c.Allowed_Amount__c), EOB__c.Total_Plan_Paid__c = SUM(Claim_Line__c.Plan_Paid_Amount__c), and EOB__c.Total_Member_Liability__c = SUM(Claim_Line__c.Member_Liability__c).

Given the EOB__c record is created,
When EOB__c.Status__c is set,
Then it defaults to 'Generated' and EOB__c.Generated_Date__c = TODAY().

Given the EOB__c record has Status__c = 'Generated',
When the policyholder logs into Experience Cloud and navigates to their claim,
Then the EOB FlexCard displays all line-level detail including Claim_Line__c.CPT_Code__c, Claim_Line__c.Service_Date__c, Claim_Line__c.Denial_Reason__c (if applicable), and the four monetary summary fields.

Given an EOB__c already exists for a Claim__c,
When the Claim__c is re-adjudicated (Status__c set back to 'In Adjudication' then 'Adjudicated'),
Then the existing EOB__c is versioned (EOB__c.Version__c incremented by 1) and a new EOB__c is created as the current version rather than overwriting the original.

Given EOB__c.Status__c = 'Generated',
When the system clock reaches EOB__c.Generated_Date__c + 3 business days,
Then a scheduled Flow sends an email notification to the policyholder (Policy__c.Policyholder_Email__c) with subject 'Your Explanation of Benefits is Ready — Claim [Claim__c.Name]'.

Given a Claim_Line__c has Claim_Line__c.Denial_Reason__c populated,
When the EOB is displayed,
Then the denial reason code and a human-readable description (from Denial_Reason_Code__mdt custom metadata) are shown inline on the EOB line detail.

## Solution

Create a Record-Triggered Flow on Claim__c (After Update, entry condition: Status__c changes to 'Adjudicated', runs once per record). The flow calls an Apex invocable method 'EOBGenerationService.createEOB(claimId)' that queries all Claim_Line__c children, computes aggregate totals, checks for existing EOB__c records (for versioning), and performs the EOB__c DML insert. Use a Custom Metadata Type (Denial_Reason_Code__mdt) keyed on denial code to supply human-readable descriptions without hardcoding. Build an OmniStudio FlexCard (EOB_Summary_Card) backed by a DataRaptor Extract for Experience Cloud portal display. Schedule a separate Scheduled Flow ('EOB_Notification_Scheduler') running nightly to identify EOB__c records where Generated_Date__c = TODAY() - 3 and send email alerts. Add a Sharing Rule to ensure Policyholder community users can read their own EOB__c records via Policy__c ownership.

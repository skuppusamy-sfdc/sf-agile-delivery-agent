## Description

As a Claims Operations Manager, I need real-time provider network status validation on Claim_Line__c records during claim entry so that adjusters are immediately alerted when a rendering provider's NPI is not covered under an active Provider_Network_Agreement__c for the claim's LOB and service date, preventing incorrect in-network payment approvals.

This story supports the adjudication rules engine built in INS-0302 by surfacing network status at the line level before batch adjudication runs.

## Acceptance Criteria

Given an adjuster enters or updates Claim_Line__c.Rendering_Provider_NPI__c on a Claim_Line__c record,
When the field is saved,
Then a Record-Triggered Flow queries Provider__c WHERE Provider__c.NPI__c = Claim_Line__c.Rendering_Provider_NPI__c and retrieves the linked Provider_Network_Agreement__c records.

Given a matching Provider_Network_Agreement__c is found,
When Provider_Network_Agreement__c.Status__c = 'Active' AND Provider_Network_Agreement__c.Effective_Date__c <= Claim__c.Date_of_Service__c AND Provider_Network_Agreement__c.Expiration_Date__c >= Claim__c.Date_of_Service__c AND Provider_Network_Agreement__c.LOB__c = Claim__c.LOB__c,
Then Claim_Line__c.Network_Status__c = 'In-Network' and Claim_Line__c.Network_Validation_Date__c = TODAY().

Given no active PNA is found for the NPI and LOB combination,
When the validation logic completes,
Then Claim_Line__c.Network_Status__c = 'Out-of-Network', a warning message is displayed inline on the record page: 'Provider NPI {NPI} has no active network agreement for {LOB} as of {Date_of_Service}', and Claim_Line__c.OON_Flag__c = TRUE.

Given Claim_Line__c.OON_Flag__c = TRUE,
When the parent Claim__c adjudication status is evaluated,
Then Claim__c.Status__c cannot be set to 'Adjudicated' via the auto-adjudication engine and Claim__c.Pend_Reason__c = 'Out of Network Provider'.

Given the NPI entered does not match any Provider__c record in Salesforce,
When the validation flow executes,
Then Claim_Line__c.Network_Status__c = 'Provider Not Found', Claim_Line__c.OON_Flag__c = TRUE, and an outbound callout is queued to NPPES API to attempt provider record creation via DataIntegration component.

-Given the provider NPI matches a Facility__c (Business Account) record instead of a Provider__c (Person Account), When validation runs, Then Facility__c.Network_Status__c is used for the check- (deprecated — facility billing NPI validation deferred to Sprint 6, see INS-0341)

## Solution

Implement a Record-Triggered Flow (Object: Claim_Line__c, Trigger: After Save, Optimized for Actions and Related Records) that invokes an Apex invocable method (ProviderNetworkValidationService.validateNPI()) to perform SOQL lookups against Provider__c and Provider_Network_Agreement__c. The Apex method accepts NPI, LOB, and Date of Service and returns a NetworkValidationResult wrapper class. Flow updates Claim_Line__c.Network_Status__c and Claim_Line__c.OON_Flag__c based on the result. For NPPES callout, publish a Platform Event (Provider_NPI_Lookup_Request__e) to decouple the synchronous flow from the async HTTP callout. Add a Custom Label for the OON warning message to support future localization.

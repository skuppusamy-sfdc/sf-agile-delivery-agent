## Description

**Bug Summary:** The FNOL OmniScript (INS-0229) fails to resolve the Policy__c lookup when the entered policy number contains hyphens (e.g., 'POL-2024-008812'). The DataRaptor Extract performs an exact-match SOQL query on Policy__c.Policy_Number__c, which does not account for hyphenated formats, causing the lookup to return no results and leaving Claim__c.Policy__c blank on record creation.

**Steps to Reproduce:**
1. Open the FNOL Intake OmniScript from the Claims App Home page.
2. Select LOB = 'Health'.
3. In the Policy Number field, enter a policy number in the format 'POL-2024-008812'.
4. Observe the policy lookup step — no policy record is returned.
5. Proceed to submit the FNOL.
6. Open the newly created Claim__c record and observe Claim__c.Policy__c = null.

**Expected:** The DataRaptor lookup should match Policy__c.Policy_Number__c = 'POL-2024-008812' and populate Claim__c.Policy__c with the correct Policy__c record ID.

**Actual:** No policy record is returned; Claim__c.Policy__c remains null after FNOL submission. No validation error is surfaced to the user.

**Environment:** Partial Sandbox (PS2). Reproducible with 100% of hyphenated policy numbers tested (8/8 test cases failed).

## Acceptance Criteria

As per Description. Additionally:
Given a policy number with hyphens is entered in the FNOL OmniScript,
When the DataRaptor Extract executes the lookup,
Then the query normalizes the input by trimming whitespace and performing a case-insensitive match, and the correct Policy__c record is returned.

Given the policy lookup returns a valid Policy__c record,
When the FNOL is submitted,
Then Claim__c.Policy__c is populated and not null on the created Claim__c record.

## Solution

Update the DataRaptor Extract (DR_Extract_PolicyLookup) to use a SOQL LIKE or normalized formula that strips/ignores hyphens during the WHERE clause evaluation. Alternatively, add a DataRaptor formula field to normalize the input string (replace '-' with '') before the SOQL filter executes. Also add an OmniScript validation step to alert the user if no policy is found before allowing FNOL submission, preventing silent data gaps. Regression test all policy number formats in PS2 before promoting to UAT.

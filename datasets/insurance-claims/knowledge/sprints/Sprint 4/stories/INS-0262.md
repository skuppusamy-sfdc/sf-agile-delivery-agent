## Description

As a Claims Operations Manager, I need a FlexCard embedded on the Policy__c record page that displays a real-time summary of all active and recently closed claims associated with that policy so that I can assess claim history and exposure without navigating away from the policy record.

## Acceptance Criteria

Given a Policy__c record page is open,
When the Policyholder 360 FlexCard loads,
Then it displays all Claim__c records where Claim__c.Policy__c = current record Id and Claim__c.Status__c IN ('FNOL Received', 'Under Review', 'Adjudicated', 'Closed — Paid') within the last 24 months.

Given the FlexCard data loads,
When claims are displayed,
Then each row shows: Claim__c.Name, Claim__c.Date_of_Loss__c, Claim__c.LOB__c, Claim__c.Status__c, Claim__c.Total_Claim_Amount__c, and Claim__c.Assigned_Adjuster__c.

Given more than 10 claims exist,
When the FlexCard renders,
Then pagination is applied showing 10 records per page with a visible record count (e.g., 'Showing 1–10 of 23').

Given a claim row is clicked,
When the user selects the claim,
Then a navigation action opens the Claim__c detail record page in the same tab.

Given the policy has zero claims in the lookback window,
When the FlexCard loads,
Then an empty-state message reads: 'No active or recent claims found for this policy.'

## Solution

Build an OmniStudio FlexCard (Name: PolicyholderClaimsSummary) with a DataRaptor Extract data source querying Claim__c filtered by Policy__c lookup and Status__c values. Configure card states for data-present and empty-state. Add a List element with column definitions mapped to Claim__c fields. Implement pagination via FlexCard built-in pagination controls (pageSize=10). Navigation action uses the NavigateTo action type targeting the Claim__c record. Embed the FlexCard on the Policy__c Lightning Record Page via the OmniStudio FlexCard component in App Builder. Assign visibility to 'Claims Operations' Permission Set Group.

## Description

As a Claims Operations Manager, I need a Policyholder 360 FlexCard embedded on the Account record page so that I can immediately see a policyholder's active policies, open claims, and recent payment history without navigating away from the record.

## Acceptance Criteria

Given a Claims rep opens a Person Account (Provider__c or Policyholder) record,
When the FlexCard loads,
Then it must display all Policy__c records where Policy__c.Policyholder__c = current Account Id and Policy__c.Status__c = 'Active', showing Policy_Number__c, LOB__c, and Policy_Effective_Date__c.

Given the FlexCard is rendered,
When claims data is fetched,
Then all Claim__c records where Claim__c.Policyholder__c = current Account Id and Claim__c.Status__c NOT IN ('Closed', 'Void') must be displayed with Claim_Number__c, Status__c, and Total_Billed_Amount__c.

Given the FlexCard is rendered,
When payment data is fetched,
Then the 5 most recent Payment__c records (ordered by Payment__c.Payment_Date__c DESC) linked to the policyholder's claims must be shown with Amount__c and Status__c.

-Given the FlexCard loads, it must also display the policyholder's SIU flag if Claim__c.SIU_Referral__c = true- (Moved to SIU module in Sprint 7)

Given no active policies exist for the Account,
When the FlexCard renders,
Then an empty-state message 'No Active Policies Found' must be displayed.

## Solution

1. OmniStudio FlexCard (FC_Policyholder360) with three DataRaptor Extract actions:
   - DR_GetActivePolicies: fetches Policy__c filtered by Policyholder__c and Status__c.
   - DR_GetOpenClaims: fetches Claim__c with Status__c filter.
   - DR_GetRecentPayments: fetches Payment__c with ORDER BY Payment_Date__c DESC LIMIT 5.
2. FlexCard uses tabbed layout (Policies | Claims | Payments) built with card child elements.
3. Flyout actions on each Claim row to open the Claim__c record in a subtab.
4. FlexCard deployed to the Account Lightning Record Page via App Builder, visible only to users with Permission Set 'PSG_Claims_Intake' or 'PSG_Claims_Supervisor'.

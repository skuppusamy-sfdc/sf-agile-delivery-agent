## Description

As a Claims Operations Manager, I need Agentforce to surface AI-generated denial reason code recommendations on Claim_Line__c records during adjudication so that claims adjusters can make faster, more consistent denial decisions based on historical adjudication patterns and CPT/ICD-10 combinations, reducing average handle time per claim.

## Acceptance Criteria

Given a Claim__c.Status__c = 'Pending Adjudication' and one or more Claim_Line__c records exist,
When the adjudicator opens the Claim__c record page,
Then the Agentforce Claims AI panel displays a ranked list of up to 3 suggested denial reason codes per Claim_Line__c, each with a confidence score (0–100%).

Given the AI panel displays a suggested denial reason code,
When the adjudicator clicks 'Apply Suggestion',
Then Claim_Line__c.Denial_Reason_Code__c is populated with the selected code and Claim_Line__c.AI_Suggestion_Applied__c = true.

Given the adjudicator overrides the AI suggestion with a different denial code,
When the manual code is saved,
Then Claim_Line__c.AI_Suggestion_Applied__c = false and Claim_Line__c.AI_Override_Reason__c is required (free text, max 255 chars) to capture the override rationale for model feedback.

Given the AI model returns no suggestions (confidence < 40% threshold),
When the panel renders,
Then a message is displayed: 'Insufficient data for AI recommendation — please apply denial code manually.'

-Given the AI panel is visible to all Salesforce users, When any user opens the Claim record, Then the panel is rendered- (deprecated — panel restricted to Claims Adjudicator and Claims Operations Manager profiles only via Permission Set Group).

Given the Agentforce recommendation is generated,
When the result is returned,
Then the response latency must be <= 3 seconds as measured from Claim record page load to panel population, validated via Salesforce Event Monitoring logs.

## Solution

Configure an Agentforce Agent (Type: Claims AI Assistant) with a custom Agent Action that invokes an Apex class (ClaimsAIDenialAdvisor.cls) connecting to the Einstein AI platform via the Salesforce AI API. The Apex class passes Claim_Line__c.CPT_Code__c, Claim_Line__c.ICD10_Code__c, Claim__c.LOB__c, and historical denial patterns from a custom Denial_History__c aggregation object as prompt context. Surface recommendations in a custom LWC (claimsAIAdvisorPanel) embedded on the Claim__c record page via a Dynamic Form. Restrict visibility using Permission Set Group (PSG_Claims_Adjudicators). Store AI interaction logs in a custom AI_Interaction_Log__c object for auditability and model retraining feedback loops. Implement a Named Credential for Einstein API authentication.

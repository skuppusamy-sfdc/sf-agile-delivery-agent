## Description

As a Claims Operations Manager, I need the system to automatically validate coverage eligibility for each Claim_Line__c against the associated Policy__c during adjudication so that ineligible claim lines are flagged before payment processing, reducing erroneous disbursements. Extends INS-0199 (Policy Coverage Object Model).

## Acceptance Criteria

Given a Claim__c record transitions to Status__c = 'Pending Adjudication',
When the adjudication rule engine executes,
Then each child Claim_Line__c is evaluated against Policy__c.Effective_Date__c and Policy__c.Expiration_Date__c relative to Claim__c.Date_of_Service__c.

Given a Claim_Line__c.Date_of_Service__c falls outside the Policy effective period,
When the eligibility check runs,
Then Claim_Line__c.Adjudication_Status__c is set to 'Denied' and Claim_Line__c.Denial_Reason_Code__c = 'E001 - Coverage Not Active'.

Given a Claim_Line__c includes a CPT code,
When the rule engine checks covered services,
Then the CPT code is validated against the active Fee_Schedule__c linked to the Policy__c.Group_Service_Agreement__c, and Claim_Line__c.Allowed_Amount__c is populated from Fee_Schedule__c.Allowed_Amount__c.

Given all Claim_Line__c records pass eligibility validation,
When the adjudication engine completes,
Then Claim__c.Status__c is updated to 'Adjudicated' and Claim__c.Adjudication_Date__c is stamped with the current datetime.

Given one or more Claim_Line__c records are denied,
When adjudication completes,
Then Claim__c.Status__c = 'Partially Adjudicated' and a Task is created and assigned to the Claims Operations queue for manual review.

-Given a Claim_Line__c has ICD-10 code populated, When the rule engine runs, Then ICD-10 is cross-validated against covered diagnosis codes- (deferred to Phase 2 — ICD-10 validation ruleset not yet loaded).

Given the adjudication engine encounters an unhandled exception,
When the error occurs,
Then the exception is logged to a custom Adjudication_Error_Log__c object with fields: Error_Message__c, Claim__c lookup, and Timestamp__c, and Claim__c.Status__c remains unchanged.

## Solution

Implement an Apex invocable method (AdjudicationRuleEngine.cls) called from a Record-Triggered Flow on Claim__c when Status__c changes to 'Pending Adjudication'. The Apex class queries all child Claim_Line__c records in a single SOQL call, applies eligibility date checks, and performs a Fee_Schedule__c lookup by CPT code and GSA. Use a custom metadata type (Adjudication_Rule__mdt) to store denial reason codes for maintainability. Bulkify all DML operations. Write unit tests targeting >85% code coverage. Log errors to Adjudication_Error_Log__c via a separate try/catch block with a future method to avoid mixed DML.

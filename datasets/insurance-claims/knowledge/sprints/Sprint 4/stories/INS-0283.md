## Description

As a Compliance Officer, I need the system to automatically evaluate claims against a configurable set of fraud indicator rules and escalate qualifying claims to the SIU team so that potential fraudulent claims are identified early in the adjudication cycle, reducing financial leakage and meeting state regulatory requirements for fraud reporting.

This story implements the first phase of the Agentforce-assisted SIU referral workflow, including rule-based flag scoring and automatic case creation for claims exceeding the referral threshold. Agentforce will surface the top fraud indicators as a natural-language summary on the SIU case.

## Acceptance Criteria

Given a Claim__c record with Status__c = 'In Adjudication',
When the SIU scoring engine evaluates the claim,
Then each applicable fraud indicator rule defined in SIU_Rule__mdt (custom metadata) is evaluated and a score is assigned to Claim__c.SIU_Risk_Score__c (integer 0–100).

Given Claim__c.SIU_Risk_Score__c >= the threshold defined in SIU_Config__mdt.Referral_Threshold__c (default: 65),
When scoring completes,
Then a Case record is created with Case.RecordTypeId = 'SIU_Investigation', Case.Subject = 'SIU Referral — ' + Claim__c.Claim_Number__c, Case.Status = 'New', and Case.Priority = 'High', linked to Claim__c via a lookup.

Given the SIU Case is created,
When Agentforce processes the claim data,
Then an AI-generated summary of the top 3 fraud indicators is populated in Case.Agentforce_Fraud_Summary__c as plain text, citing specific Claim__c and Claim_Line__c field values that triggered each indicator.

Given Claim__c.SIU_Risk_Score__c >= 65,
When the SIU Case is created,
Then Claim__c.Status__c is updated to 'SIU Review' and normal adjudication processing is paused (Claim__c.Adjudication_Hold__c = TRUE).

Given Claim__c.SIU_Risk_Score__c < 65,
When scoring completes,
Then no SIU Case is created and Claim__c.SIU_Risk_Score__c is stored for audit purposes without altering adjudication flow.

Given the SIU Case is resolved with Case.Resolution__c = 'No Fraud Found',
When the Case is closed,
Then a Record-Triggered Flow resets Claim__c.Adjudication_Hold__c = FALSE and Claim__c.Status__c reverts to 'In Adjudication' to resume normal processing.

Given any SIU referral event occurs,
When the case is created,
Then a ComplianceReporting__c audit record is created capturing Claim__c.Id, SIU_Risk_Score__c, referral timestamp, and the Compliance Officer user who owns the case for state regulatory reporting.

## Solution

Implement an Apex class (SIURuleEngine) invoked by a Record-Triggered Flow on Claim__c (after update, when Status__c = 'In Adjudication'). SIURuleEngine reads fraud indicator definitions from SIU_Rule__mdt (fields: Rule_Name__c, Weight__c, SOQL_Condition__c, Active__c) and dynamically evaluates each rule against the Claim__c and its Claim_Line__c children using dynamic SOQL. Scores are summed and written to Claim__c.SIU_Risk_Score__c. If threshold is met, an Apex method creates the Case record and calls the Agentforce Agent Action API (Einstein Generative AI) via a Named Credential, passing claim context as a structured prompt. The AI response is stored in Case.Agentforce_Fraud_Summary__c. A second Record-Triggered Flow on Case (after update, Status = 'Closed') handles the hold release logic. All SIU rule weights and thresholds are manageable by Compliance Officers via a custom Lightning Web Component (siuRuleManager) without code deployment. Custom Permission Set 'SIU_Investigator_PS' restricts Case SIU record type access.

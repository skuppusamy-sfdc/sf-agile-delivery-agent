## Description

As a Claims Operations Manager, I need the system to automatically evaluate basic adjudication rules — including coverage eligibility verification and duplicate claim detection — on newly submitted Claim__c records so that ineligible or duplicate claims are flagged before manual review begins, reducing adjudicator workload and preventing erroneous payments.

## Acceptance Criteria

Given a Claim__c record transitions to Status__c = 'FNOL Received',
When the adjudication rules Apex trigger fires (after update),
Then the system checks Policy__c.Effective_Date__c <= Claim__c.Date_of_Loss__c <= Policy__c.Expiration_Date__c; if the check fails, Claim__c.Adjudication_Status__c is set to 'Denied – Ineligible Coverage' and Claim__c.Denial_Reason__c is populated with 'Policy not active on date of loss'.

Given coverage eligibility passes,
When the duplicate detection logic runs,
Then the system queries existing Claim__c records matching the same Policy__c, Claim__c.Date_of_Loss__c (±1 day), and Claim__c.LOB__c; if a duplicate is found, Claim__c.Duplicate_Flag__c is set to TRUE and a Task is created and assigned to the owning Claims queue for manual review.

Given a Health LOB claim,
When adjudication rules execute,
Then each associated Claim_Line__c is checked: Claim_Line__c.CPT_Code__c must exist in the active Fee_Schedule__c records linked to the claim's Provider_Network_Agreement__c; lines with unmatched CPT codes set Claim_Line__c.Status__c = 'Pended – CPT Not Found'.

Given all Claim_Line__c records pass CPT validation,
When the adjudication run completes,
Then Claim__c.Status__c is updated to 'Under Review' and Claim__c.Adjudication_Run_Date__c is stamped with the current datetime.

Given any adjudication rule results in a denial or pend,
When the status is set,
Then a Platform Event (Adjudication_Result__e) is published containing Claim__c.Id and result code so that downstream notification flows are triggered.

## Solution

Implement an Apex Trigger (ClaimAdjudicationTrigger) on Claim__c [After Update] with a handler class (ClaimAdjudicationHandler). Eligibility check queries Policy__c via SOQL. Duplicate detection uses a SOQL aggregate query with date range filter. CPT validation iterates Claim_Line__c child records and cross-references Fee_Schedule__c via a SOQL join on Provider_Network_Agreement__c. All rule results are collected and written in a single DML update (bulkified). Platform Event published via EventBus.publish(). A separate Record-Triggered Flow handles Task creation for duplicate flags to keep Apex lean. Custom metadata type (Adjudication_Rule__mdt) stores rule toggle flags for each LOB to allow admin-configurable rule activation without code deployment.

## Description

As a Claims Operations Manager, I need the FNOL intake process to validate whether the rendering provider is in-network at the time of service so that the claim is correctly flagged for in-network or out-of-network adjudication routing from the outset.

## Acceptance Criteria

Given a Claim__c is created via FNOL with a Provider__c lookup populated,
When the record is saved,
Then a Record-Triggered Flow checks for an active Provider_Network_Agreement__c (PNA) where PNA.Provider__c = Claim__c.Provider__c AND PNA.Effective_Date__c <= Claim__c.Date_of_Service__c AND PNA.Termination_Date__c >= Claim__c.Date_of_Service__c.

Given an active PNA exists for the provider at the date of service,
When the network check completes,
Then Claim__c.Network_Status__c = 'In-Network' and Claim__c.PNA__c is populated with the matching PNA record ID.

Given no active PNA exists for the provider at the date of service,
When the network check completes,
Then Claim__c.Network_Status__c = 'Out-of-Network' and Claim__c.Network_Status__c field is highlighted in red on the record page via a custom LWC status indicator.

Given Claim__c.Network_Status__c = 'Out-of-Network',
When the claim proceeds to adjudication,
Then Claim__c.Routing_Queue__c = 'OON Adjudication Team' and the standard in-network fee schedule is bypassed.

Given the Provider__c record has Provider__c.NPI__c = null,
When the network validation runs,
Then a validation error is thrown: 'Provider NPI is required for network status determination.'

## Solution

Build a Record-Triggered Flow (Object: Claim__c, Trigger: After Insert) that calls an Apex invocable action (ProviderNetworkValidator.cls) to query Provider_Network_Agreement__c with date-range logic. Apex is preferred over Flow formula due to the date overlap query complexity. Return network status and PNA Id to the Flow for field updates via a before-save flow update step. Create a custom LWC (networkStatusBadge) using conditional CSS classes to render the red/green indicator on the Claim__c Lightning Record Page. Add a Permission Set (PS_Claims_Network_Read) for PNA object access.

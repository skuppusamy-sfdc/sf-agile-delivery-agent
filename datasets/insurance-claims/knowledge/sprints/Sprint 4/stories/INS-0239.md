## Description

As a Claims Operations Manager, I need a Claim 360 FlexCard on the Claim__c Lightning Record Page so that I can see a consolidated view of claim status, associated policy details, rendering provider information, and outstanding payment balances without navigating across multiple objects.

## Acceptance Criteria

Given a user with the 'Claims Adjuster' Permission Set Group opens a Claim__c record,
When the Claim 360 FlexCard loads,
Then it displays Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, Claim__c.Total_Allowed_Amount__c, and Claim__c.Total_Paid_Amount__c in the header band.

Given the Claim__c has a related Policy__c,
When the FlexCard renders the Policy section,
Then Policy__c.PolicyNumber__c, Policy__c.LOB__c, Policy__c.Group_Name__c, and Policy__c.Policy_Status__c are shown with a status badge (green=Active, red=Lapsed, amber=Pending).

Given the Claim__c has a related Provider__c (Person Account),
When the Provider section renders,
Then Provider__c.NPI__c, Provider__c.Specialty__c, Provider__c.Network_Status__c, and the most recent Provider_Network_Agreement__c.Contract_End_Date__c are displayed.

Given one or more Payment__c records are related to the Claim__c,
When the Payments section renders,
Then a datatable shows Payment__c.Payment_Date__c, Payment__c.Amount__c, Payment__c.Payment_Method__c, and Payment__c.ERA_Status__c for up to 10 most recent records.

-Given the FlexCard is also placed on the Policy__c page layout, it shows all claims for that policy- (deprecated — standalone Claims List FlexCard to be built separately in INS-0248)

Given a user without the 'Claims Adjuster' Permission Set Group,
When they navigate to the Claim__c record page,
Then the FlexCard is not visible and no data is exposed.

## Solution

Build an OmniStudio FlexCard (Name: Claim360Card) backed by an Integration Procedure (IP: Claim_GetClaimContext) that chains DataRaptor Extracts across Claim__c, Policy__c, Provider__c (Person Account), and Payment__c using relationship traversal. Status badge coloring implemented via FlexCard Conditional Styles. Payment datatable uses a child FlexCard block with sort by Payment__c.Payment_Date__c DESC, LIMIT 10. Visibility controlled via FlexCard Conditions referencing the running user's Permission Set Group membership, validated in the IP using a custom Apex action. Deploy FlexCard to Claim__c Lightning Record Page via App Builder.

## Description

As a Claims Operations Manager, I need the FNOL intake process to capture Coordination of Benefits (COB) indicators and automatically flag Claim__c records that require COB review so that claims with multiple payers are routed correctly and overpayment risk is minimized.

During FNOL intake (OmniScript — INS-0292), the adjuster should be prompted to indicate whether the claimant has other active insurance coverage. If COB is indicated, the Claim__c record must be flagged and routed to a dedicated COB review queue rather than proceeding through standard auto-adjudication.

## Acceptance Criteria

Given the FNOL OmniScript is in progress and the LOB = 'Medical',
When the claimant information step renders,
Then a COB question is displayed: 'Does the claimant have other active health insurance coverage?' with Yes/No radio buttons.

Given the adjuster selects 'Yes' to COB,
When the OmniScript completes and the Claim__c is created,
Then Claim__c.COB_Indicator__c = TRUE and Claim__c.Status__c = 'Pended — COB Review' and the record is assigned to the 'COB Review Queue'.

Given Claim__c.COB_Indicator__c = TRUE,
When the adjudication engine (INS-0294) evaluates the claim,
Then the auto-adjudication path is bypassed entirely and Claim__c.Adjudication_Method__c remains NULL until COB review is complete.

Given the adjuster selects 'No' to COB or the LOB is not 'Medical',
When the Claim__c is created,
Then Claim__c.COB_Indicator__c = FALSE and normal adjudication routing applies.

Given Claim__c.COB_Indicator__c = TRUE,
When a COB review is completed and Claim__c.COB_Review_Status__c = 'Resolved',
Then the claim is automatically re-queued for adjudication via a Record-Triggered Flow and Claim__c.Status__c transitions to 'FNOL Received'.

## Solution

Add COB_Indicator__c (Checkbox), COB_Review_Status__c (Picklist: Pending Review, In Review, Resolved), and COB_Notes__c (Long Text) fields to Claim__c. Update the FNOL OmniScript (INS-0292) to include the COB question step, conditionally shown for LOB = 'Medical' using OmniScript Conditional Views. Pass COB_Indicator__c value to the DataRaptor Turbo Action that creates the Claim__c record. Add a condition to the AdjudicationEngine.cls (INS-0294) to exit early when COB_Indicator__c = TRUE. Build a Record-Triggered Flow on Claim__c (After Update, COB_Review_Status__c changed to 'Resolved') to reset Status__c = 'FNOL Received' and re-trigger adjudication. Create a COB_Review_Queue list view for Claims Operations.

## Description

As a Claims Operations Manager, I need an Agentforce AI assistant embedded on the Claim__c record page to suggest probable pend reasons and next-best actions for claims flagged for manual review (Claim__c.Manual_Review_Flag__c = TRUE) so that adjusters can resolve pended claims faster and with greater consistency, reducing average handle time for complex claims.

The AI suggestions should be grounded in historical pend reason data from closed Claim__c records and relevant policy coverage rules stored in Salesforce, presented as ranked recommendations with confidence scores.

## Acceptance Criteria

Given Claim__c.Manual_Review_Flag__c = TRUE and Claim__c.Status__c = 'Pended',
When an adjuster opens the Claim__c record page,
Then the Agentforce Claims Assistant panel activates automatically and displays a prompt: 'I found {N} similar pended claims. Here are suggested pend reasons based on claim history.'

Given the Agentforce assistant is active,
When it analyzes the current Claim__c (LOB, CPT codes, ICD-10 codes, billed amount, provider NPI, pend history),
Then it returns up to 3 ranked pend reason suggestions, each with: suggested Claim__c.Pend_Reason__c value, confidence percentage (e.g., '87% match'), and a brief rationale citing the top contributing factor (e.g., 'Provider NPI 1234567890 has 14 prior OON pends').

Given the adjuster selects a suggested pend reason,
When they click 'Apply Recommendation',
Then Claim__c.Pend_Reason__c is updated to the selected value, Claim__c.AI_Recommendation_Applied__c = TRUE, and Claim__c.AI_Recommendation_Confidence__c is stamped with the confidence score.

Given the adjuster rejects all suggestions,
When they click 'Dismiss',
Then Claim__c.AI_Recommendation_Dismissed__c = TRUE and Claim__c.AI_Dismissal_Reason__c is captured via a required picklist prompt (values: 'Incorrect Suggestion', 'Already Resolved', 'Insufficient Data', 'Other').

Given Agentforce recommendation telemetry is enabled,
When any recommendation interaction occurs (applied or dismissed),
Then an AI_Interaction_Log__c record is created capturing: Claim__c.Id, recommendation text, confidence score, adjuster action, and timestamp — for model feedback loop reporting.

## Solution

Configure an Agentforce Agent (Type: Claims Assistant) using Salesforce Einstein Copilot framework. Define a custom Agent Action (Topic: Claim Pend Analysis) backed by an Apex class (ClaimPendRecommendationAction) that implements the Agentforce invocable interface. The Apex class performs a SOQL query on historical Claim__c records (Status = 'Closed', Pend_Reason__c != null) filtered by LOB and CPT code overlap, then calls the Einstein Generative AI API with a structured prompt including claim context and historical pend pattern summary. Response is parsed into a ranked recommendation list returned to the Agentforce panel. Embed the Agentforce panel as an LWC component (agentforceCopilotPanel) on the Claim__c Lightning Record Page, visible only when Manual_Review_Flag__c = TRUE via dynamic component visibility. Log interactions via a Record-Triggered Flow on Claim__c field changes to AI_Recommendation_Applied__c and AI_Recommendation_Dismissed__c.

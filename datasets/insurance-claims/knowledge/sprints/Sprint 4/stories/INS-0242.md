## Description

As a Claims Operations Manager, I need Agentforce to surface likely fraud indicators on newly created Claim__c records based on configurable SIU rules so that high-risk claims are proactively flagged for Special Investigations Unit review before adjudication completes, reducing fraudulent payouts and improving compliance posture.

## Acceptance Criteria

Given a Claim__c record is created with Status__c = 'FNOL Received',
When the Agentforce Claims Fraud Indicator agent action is triggered via Platform Event (ClaimCreated__e),
Then the agent evaluates the claim against a minimum of five configurable SIU rule criteria stored in a custom metadata type (SIU_Rule__mdt).

Given the SIU rules include 'Duplicate Claim - Same Provider + DOS + Diagnosis within 30 days',
When the agent detects a matching prior Claim__c record,
Then Claim__c.SIU_Flag__c = true, Claim__c.SIU_Risk_Score__c is populated (range 0–100), and a SIU referral task is created and assigned to the SIU queue.

Given the SIU risk score is >= 75,
When the score is written back to the Claim__c,
Then Claim__c.Status__c is automatically set to 'SIU Hold' and adjudication processing is paused until manually released by a Compliance Officer.

Given the SIU risk score is between 40 and 74,
When the score is written back,
Then Claim__c.SIU_Flag__c = true but Status__c remains 'Under Review'; a low-priority task is created for adjuster awareness.

Given the SIU risk score is < 40,
When the evaluation completes,
Then no SIU flag is set and normal adjudication proceeds without interruption.

Given a Compliance Officer reviews and clears an SIU Hold,
When they update Claim__c.SIU_Cleared_By__c and Claim__c.SIU_Cleared_Date__c,
Then Claim__c.Status__c reverts to 'Under Review' and adjudication resumes automatically via a Record-Triggered Flow.

## Solution

Configure an Agentforce Agent (Type: Service) with a custom Agent Action mapped to an Apex invocable method (SIUFraudEvaluatorAction.cls). The action is triggered by a Platform Event (ClaimCreated__e) published by the FNOL OmniScript on Claim__c creation. SIU rules are maintained in SIU_Rule__mdt custom metadata (fields: Rule_Name__c, Rule_Logic__c, Weight__c, Active__c) allowing admin-level updates without code deployment. Risk scoring uses a weighted sum of triggered rules. Score and flag fields written to Claim__c via Apex DML. SIU Hold status transition managed by a Record-Triggered Flow with entry criteria SIU_Risk_Score__c >= 75. Agentforce agent grounded with Claim__c, Policy__c, and Provider__c data using standard Data Cloud or direct SOQL grounding depending on org configuration.

## Description

As a Claims Operations Manager, I need Claim_Line__c records to be validated against the current CMS ICD-10-CM code set upon save so that claims with invalid or inactive diagnosis codes are flagged before adjudication begins, preventing downstream payment errors and CMS audit findings.

## Acceptance Criteria

Given a user saves a Claim_Line__c record with a value in Claim_Line__c.ICD10_Code__c,
When the record is saved,
Then the system validates the code against the active CMS ICD-10-CM reference table (ICD10_Reference__c) where ICD10_Reference__c.Effective_Date__c <= Date_of_Service__c and ICD10_Reference__c.Expiration_Date__c >= Date_of_Service__c.

Given the ICD-10 code is valid and active,
When validation passes,
Then Claim_Line__c.ICD10_Validation_Status__c = 'Valid' and the record saves without error.

Given the ICD-10 code does not exist in ICD10_Reference__c,
When the record is saved,
Then a save error is thrown: 'ICD-10 code {value} is not recognized. Please verify and resubmit.' and Claim_Line__c.ICD10_Validation_Status__c = 'Invalid'.

Given the ICD-10 code exists but is inactive for the Date_of_Service__c,
When the record is saved,
Then a save error is thrown: 'ICD-10 code {value} was not active on the date of service {Date_of_Service__c}.' and Claim_Line__c.Adjudication_Status__c = 'Pended — ICD10 Review'.

Given the Claim_Line__c.ICD10_Code__c field is blank,
When the record is saved,
Then a required field validation error is displayed: 'Diagnosis code is required for all claim lines.'

Given a bulk import of Claim_Line__c records via Data Loader,
When the batch executes,
Then ICD-10 validation runs for each record and failed rows are captured in a Batch_Error_Log__c with the row index, Claim_Line__c Id, and error message.

## Solution

Implement validation logic in an Apex Trigger (ClaimLineTrigger, before insert/before update) that performs a SOQL query against the custom ICD10_Reference__c object (loaded via CMS annual data refresh integration). For bulk scenarios, use a trigger handler pattern (ClaimLineHandler) with a Map-based lookup to minimize SOQL queries within governor limits. A scheduled Apex job (ScheduledICD10RefreshBatch) refreshes ICD10_Reference__c records quarterly via CMS flat-file DataIntegration. A custom Validation Rule also enforces the blank-field check as a first-pass gate before the trigger fires.

## Description

As a Claims Operations Manager, I need a Policyholder 360 FlexCard embedded on the Claim__c record page so that adjusters can instantly view the claimant's active coverage details, policy status, and prior claim history without navigating away from the current claim, improving adjudication speed and reducing errors.

Extends INS-0290 (Policy 360 FlexCard on Policy__c page) by surfacing a claim-context variant.

## Acceptance Criteria

Given an adjuster opens a Claim__c record,
When the Policyholder 360 FlexCard loads,
Then it displays Policy__c.Policy_Number__c, Policy__c.Status__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, and Policy__c.Coverage_Tier__c in the card header section.

Given the FlexCard is loaded,
When the 'Prior Claims' section renders,
Then it lists all Claim__c records where Claim__c.Policy__c = current policy, ordered by Claim__c.Date_of_Loss__c DESC, showing Claim__c.Claim_Number__c, Claim__c.Status__c, and Claim__c.Total_Paid_Amount__c for up to the last 5 claims.

Given Policy__c.Status__c = 'Lapsed' or 'Cancelled',
When the FlexCard header renders,
Then a red banner displays 'POLICY INACTIVE — Verify Coverage Before Adjudication' and Claim__c.Status__c cannot be advanced past 'Pending Review'.

Given the adjuster clicks a prior claim row in the FlexCard,
When the navigation action fires,
Then the user is navigated to that Claim__c record detail page in a new subtab (workspace navigation).

-Then a modal preview of the prior claim opens inline within the FlexCard- (deprecated — inline modal removed due to performance concerns in UAT)

## Solution

Build an OmniStudio FlexCard (Name: Policyholder360_ClaimContext) with a DataRaptor Extract as the data source querying Policy__c and related Claim__c records via SOQL with a 5-record limit and ORDER BY Date_of_Loss__c DESC. Use FlexCard conditional styling (CSS class binding) for the red banner when Policy__c.Status__c IN ('Lapsed','Cancelled'). Embed the FlexCard on the Claim__c Lightning Record Page via the OmniStudio FlexCard LWC component. Add a Navigate action on prior claim rows using the NavigationMixin. Assign via Claims_Adjuster_PSG Permission Set Group.

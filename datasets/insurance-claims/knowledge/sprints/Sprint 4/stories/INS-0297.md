## Description

As a Claims Operations Manager, I need Agentforce to analyze pended Claim__c records and suggest the most likely pend reason and a recommended next best action so that claims adjusters can resolve pended claims faster and reduce average handle time.

When a Claim__c record transitions to Status__c = 'Pended', Agentforce should evaluate available claim data (LOB, CPT codes, ICD-10, provider network status, billed amount) against historical pend patterns and surface a natural-language suggestion in a sidebar panel on the Claim__c record page. The suggested pend reason and next action should be logged to a new Claim__c field for audit purposes.

## Acceptance Criteria

Given a Claim__c record has Status__c = 'Pended',
When the Claims adjuster opens the record page,
Then the Agentforce sidebar displays a suggested pend reason (mapped to Claim__c.AI_Suggested_Pend_Reason__c) and a recommended next action in plain English within 5 seconds of page load.

Given Agentforce returns a suggestion,
When the adjuster accepts the suggestion by clicking 'Apply Suggestion',
Then Claim__c.Pend_Reason__c is updated to the AI-suggested value and Claim__c.AI_Suggestion_Accepted__c = TRUE is recorded for reporting.

Given Agentforce returns a suggestion,
When the adjuster dismisses the suggestion,
Then Claim__c.AI_Suggestion_Accepted__c = FALSE and the dismissal is logged with a timestamp in Claim__c.AI_Suggestion_Dismissed_Date__c.

Given Agentforce cannot determine a suggestion (low confidence score < 0.6),
When the sidebar renders,
Then the message 'Insufficient data for AI recommendation — manual review required' is displayed and no fields are auto-populated.

-Given the adjuster has the SIU Analyst role, Agentforce also flags potential fraud indicators inline- (deferred — SIU integration scoped to Phase 2 per compliance review).

## Solution

Configure an Agentforce Agent (Type: Claims Assistant) with a custom Topic: 'Pend Reason Analysis'. Create an Agent Action backed by an Apex class (AgentforceClaimsPendAnalysis.cls) that queries Claim__c, Claim_Line__c, and historical pended claims via SOQL and passes structured context to the Einstein LLM prompt template. Use a Prompt Template (Type: Record) on Claim__c to generate the natural-language suggestion. Expose the Agentforce sidebar via the Einstein Copilot component on the Claim__c Lightning Record Page. Store AI outputs in custom fields (AI_Suggested_Pend_Reason__c, AI_Suggestion_Accepted__c, AI_Suggestion_Dismissed_Date__c) on Claim__c. Log all AI interactions to a custom object AI_Interaction_Log__c for compliance audit trail.

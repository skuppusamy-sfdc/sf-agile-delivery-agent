## Description

As a Claims Operations Manager, I need a Policyholder 360 FlexCard embedded on the Claim__c record page so that I can instantly view the claimant's active policies, open claims, and coverage details without navigating away from the claim record.

## Acceptance Criteria

Given a Claims rep opens any Claim__c record,
When the Policyholder 360 FlexCard loads,
Then it displays the related Policy__c.Policy_Number__c, Policy__c.Coverage_Type__c, Policy__c.Effective_Date__c, and Policy__c.Expiration_Date__c.

Given the policyholder has more than one active Policy__c record,
When the FlexCard renders,
Then all active policies are listed in a scrollable child card section, filtered by Policy__c.Status__c = 'Active'.

Given the policyholder has open Claim__c records other than the current one,
When the FlexCard loads,
Then a badge count displays the number of open claims and links to a filtered list view of Claim__c records where Claim__c.Status__c NOT IN ('Closed', 'Denied').

Given the Claim__c record is for LOB = 'Health',
When the FlexCard loads,
Then the COB indicator field Claim__c.COB_Applicable__c is prominently displayed with a warning icon if value = true.

## Solution

Create an OmniStudio FlexCard (Name: PolicyHolder360_ClaimSummary) with a DataRaptor Extract as the data source, joining Policy__c and Claim__c via the shared Contact/Account lookup. Use child FlexCards for the open claims badge. Embed the FlexCard on the Claim__c Lightning Record Page via the OmniStudio FlexCard component. Apply a Permission Set (PS_Claims_Ops_Read) to control visibility. Cache DataRaptor results for 30 seconds to reduce SOQL overhead.

## Description

**Bug Report — FNOL OmniScript allows submission against a Lapsed policy**

**Environment:** Acme Insurance Full Sandbox (ACME_FS1)
**Build:** Sprint 4 Day 3 deployment (2024-11-13)
**Related Story:** INS-0253

**Steps to Reproduce:**
1. Navigate to the Claims Console App and launch the FNOL Intake OmniScript.
2. In the Policy Number field, enter a policy number whose corresponding Policy__c.Status__c = 'Lapsed' (test data: POL-009821).
3. The DataRaptor Extract populates policyholder details successfully (no error shown).
4. Complete all remaining FNOL fields (date of loss, claimant name, estimated loss amount).
5. Click 'Submit' on the final review step.

**Expected Behavior:**
The OmniScript should validate Policy__c.Status__c upon policy lookup. If Status__c = 'Lapsed' or 'Cancelled', an inline error message should display: 'This policy is not active. A claim cannot be filed against a lapsed or cancelled policy.' Submission should be blocked.

**Actual Behavior:**
No validation error is shown. The OmniScript completes submission and creates a Claim__c record with Claim__c.Status__c = 'New' linked to the lapsed policy. The Claim__c.Policy__c lookup is successfully set to the lapsed Policy__c record. Downstream adjudication rules then fail silently because the policy is inactive.

**Impact:** High — invalid claims are entering the adjudication queue and require manual identification and voiding by Claims Operations staff. Estimated 4–6 bad records created in sandbox during integration testing.

**Workaround:** None currently. Manual review of Claim__c records after submission.

## Acceptance Criteria

As per Description.

Additionally:
Given Policy__c.Status__c = 'Lapsed' or 'Cancelled' is returned by the DataRaptor Extract,
When the policy lookup step completes,
Then an OmniScript inline validation error fires immediately and the 'Next' button is disabled.

Given a valid policy (Status__c = 'Active') is entered,
When the policy lookup completes,
Then no validation error is shown and the OmniScript proceeds normally.

Given the fix is deployed,
When regression testing is run against test policy POL-009821 (Lapsed),
Then no Claim__c record is created and the error message 'This policy is not active...' is displayed.

## Solution

Root cause: The DataRaptor Extract ('DR_PolicyLookup') returns Policy__c data including Status__c, but the OmniScript Step 1 has no conditional validation action checking Status__c value before enabling the Next button.

Fix approach:
1. Add an OmniScript Validation action on Step 1 after the DataRaptor Extract response is received.
2. Use a Formula element to evaluate: IF({policyStatus} != 'Active', true, false).
3. If validation returns true (policy not active), display inline error message and set button state to disabled.
4. Add a Set Values action to clear all pre-populated policyholder fields if an invalid policy status is detected, preventing stale data carry-forward.
5. Add an Apex test to the existing FNOL test class covering the lapsed-policy negative scenario.
6. Regression test in ACME_FS1 sandbox against POL-009821 before promoting to UAT.

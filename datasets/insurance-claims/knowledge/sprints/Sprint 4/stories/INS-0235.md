# INS-0235: Credentialing Case Auto-Creation from Provider FNOL Claim Submission for Unverified Providers

## Description

As a Credentialing Specialist, I need the system to automatically create a Credentialing_Case__c record when a claim is submitted referencing a Provider__c whose credentialing status is unverified or expired so that the credentialing team is proactively alerted and can resolve the provider's status before claim payment is released.

## Acceptance Criteria

Given a Claim__c is created via FNOL with Provider__c populated,
When the system evaluates Provider__c.Credentialing_Status__c,
Then if the value is 'Unverified' or 'Expired', a Credentialing_Case__c record is auto-created with Credentialing_Case__c.Provider__c = Claim__c.Provider__c and Credentialing_Case__c.Trigger_Source__c = 'FNOL Claim Submission'.

Given a Credentialing_Case__c is auto-created,
When the record is inserted,
Then Credentialing_Case__c.Status__c = 'Open', Credentialing_Case__c.Priority__c = 'High', and the case is assigned to the 'Credentialing Team' queue.

Given the Credentialing_Case__c is created,
When the related Claim__c is evaluated for payment eligibility,
Then Claim__c.Payment_Hold__c = true and Claim__c.Hold_Reason__c = 'Provider Credentialing Pending' until the Credentialing_Case__c.Status__c = 'Closed - Approved'.

Given Provider__c.Credentialing_Status__c = 'Active',
When the claim is submitted,
Then no Credentialing_Case__c is created and Claim__c.Payment_Hold__c remains false.

## Solution

Implement a Record-Triggered Flow (Object: Claim__c, Trigger: After Insert) that queries the related Provider__c.Credentialing_Status__c. If status is 'Unverified' or 'Expired', the Flow creates a Credentialing_Case__c record and updates Claim__c.Payment_Hold__c = true via a before-save flow path. Use a Flow Decision element for the credentialing status check. Add a separate Record-Triggered Flow on Credentialing_Case__c (After Update, Status__c = 'Closed - Approved') to release the payment hold by setting Claim__c.Payment_Hold__c = false on the related claim. Store credentialing status thresholds in a custom metadata type (Credentialing_Config__mdt) for admin configurability.

## Description

**Bug Report — FNOL OmniScript Null Pointer on Policy Lookup**

**Environment:** Acme Insurance Full Sandbox (CS47)
**Build:** Sprint 4 deployment — 2024-11-12

**Steps to Reproduce:**
1. Log in as a user with the Claims_Intake_Users Permission Set Group.
2. Open the FNOL Intake OmniScript (INS-0292).
3. In the Policy Number field, enter a valid Policy Number for a policy that has NO prior Claim__c records (i.e., a brand-new policyholder).
4. Tab out of the Policy Number field to trigger the DataRaptor Extract lookup.
5. Observe the OmniScript step.

**Expected Behavior:** The DataRaptor Extract should populate policyholder name, LOB, and coverage dates from Policy__c. The OmniScript should proceed normally even if no prior Claim__c records exist for the policy.

**Actual Behavior:** The OmniScript throws an unhandled error overlay: 'There was a problem loading your data. Please try again.' Browser console shows: 'TypeError: Cannot read properties of null (reading length)' originating from the DataRaptor Extract action that attempts to retrieve prior claim count for display in the summary panel.

**Root Cause (Suspected):** The DataRaptor Extract for prior claim count uses a formula to access the result array length without a null-check. When the SOQL returns zero rows, the array is null rather than empty, causing the null pointer.

**Affected Users:** Any Claims Operations user submitting FNOL for a first-time claimant.
**Severity:** High — blocks FNOL submission for new policyholders.

## Acceptance Criteria

As per Description.

Given a valid Policy__c with no associated Claim__c records,
When the policy number is entered and the DataRaptor Extract executes,
Then the OmniScript loads successfully without error and displays '0 prior claims' in the summary panel.

Given the fix is deployed,
When any policy number (with or without prior claims) is entered,
Then the DataRaptor Extract result array is null-safe and the OmniScript proceeds to the next step without an error overlay.

## Solution

Update the DataRaptor Extract (Name: DR_Extract_PolicyFNOLSummary) to add a null-check condition on the Claim__c result set: use an IF(ISNULL(claimArray), 0, claimArray.length) expression in the DataRaptor formula field. Additionally, add an OmniScript Set Values element before the summary step to default the prior_claim_count variable to '0' if the DataRaptor output is empty. Add a negative test case to the OmniScript integration test suite covering zero-claim policies.

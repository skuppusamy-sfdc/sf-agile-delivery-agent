## Description

As a Claims Operations Manager, I need the system to automatically assign a priority tier to newly created Claim__c records based on the estimated loss amount and LOB so that high-value or high-risk claims are immediately surfaced to senior adjusters without manual triage.

Priority rules:
- Estimated loss >= $50,000 → Priority__c = 'Critical'
- Estimated loss $10,000–$49,999 → Priority__c = 'High'
- Estimated loss < $10,000 → Priority__c = 'Standard'
- Any claim with LOB = 'Workers Comp' → minimum Priority__c = 'High' regardless of amount
- Any claim flagged with SIU_Referral__c = true → Priority__c = 'Critical' (overrides all)

## Acceptance Criteria

Given a new Claim__c record is created with EstimatedLossAmount__c = 75000 and LOB__c = 'Auto',
When the record-triggered Flow evaluates,
Then Claim__c.Priority__c = 'Critical'.

Given a new Claim__c record is created with EstimatedLossAmount__c = 8000 and LOB__c = 'Workers Comp',
When the Flow evaluates,
Then Claim__c.Priority__c = 'High' (Workers Comp floor applied).

Given a new Claim__c record is created with EstimatedLossAmount__c = 5000, LOB__c = 'Medical', and SIU_Referral__c = true,
When the Flow evaluates,
Then Claim__c.Priority__c = 'Critical' (SIU override).

Given an existing Claim__c record is updated (not created),
When the Flow trigger condition checks {!$Record.IsNew},
Then the priority re-evaluation logic does NOT fire to prevent unintended overwrites.

-Given Priority__c is manually set by an adjuster, the Flow should not overwrite it- (deprecated — manual override removed from scope per sprint planning)

## Solution

Implement a Record-Triggered Flow ('RTF_Claim_Priority_Assignment') on Claim__c, trigger: After Save, condition: record is new (IsNew = true).

Flow decision tree:
1. Check SIU_Referral__c = true → set Priority__c = 'Critical', exit
2. Check LOB__c = 'Workers Comp' AND EstimatedLossAmount__c < 10000 → set Priority__c = 'High', exit
3. Check EstimatedLossAmount__c >= 50000 → 'Critical'
4. Check EstimatedLossAmount__c >= 10000 → 'High'
5. Default → 'Standard'

Use Fast Field Update (Before Save trigger on re-evaluation path) for efficiency. Add Flow to 'ClaimsProcessing_Flows' change set. Unit test via @isTest Apex class covering all 5 priority branches.

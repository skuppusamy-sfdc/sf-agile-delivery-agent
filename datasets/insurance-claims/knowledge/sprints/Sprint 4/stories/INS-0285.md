## Description

As a Claims Operations Manager, I need the system to automatically adjudicate clean claims where the total billed amount is under $500 and all Claim_Line__c CPT codes are matched to an active Fee_Schedule__c entry so that low-complexity claims are processed without manual intervention, reducing queue volume.

Extends INS-0284 (FNOL Intake OmniScript) — this rule fires after FNOL_Submitted__e is received.

## Acceptance Criteria

Given a Claim__c record with Status__c = 'FNOL Received' is created,
When the FNOL_Submitted__e Platform Event is consumed by the adjudication subscriber,
Then the system must evaluate all child Claim_Line__c records for CPT_Code__c matches against Fee_Schedule__c where Fee_Schedule__c.Effective_Date__c <= Service_Date__c and Fee_Schedule__c.Termination_Date__c >= Service_Date__c.

Given all Claim_Line__c records have matched Fee_Schedule__c entries AND SUM(Claim_Line__c.Billed_Amount__c) < 500,
When auto-adjudication criteria are met,
Then Claim__c.Status__c must be updated to 'Auto-Adjudicated', Claim__c.Adjudicated_Amount__c populated with the fee-schedule allowed amount, and Claim__c.Adjudication_Method__c = 'Automatic'.

Given any Claim_Line__c CPT_Code__c has no matching active Fee_Schedule__c entry,
When the evaluation runs,
Then Claim__c.Status__c must be set to 'Pending Manual Review' and Claim__c.Pend_Reason__c = 'Unmatched CPT Code'.

Given SUM(Claim_Line__c.Billed_Amount__c) >= 500,
When the threshold check fails,
Then Claim__c.Status__c must be set to 'Pending Manual Review' and Claim__c.Pend_Reason__c = 'Exceeds Auto-Adjudication Threshold'.

Given Claim__c.Status__c transitions to 'Auto-Adjudicated',
When the update is committed,
Then a Payment__c record must be created with Payment__c.Claim__c = the adjudicated claim, Payment__c.Amount__c = Claim__c.Adjudicated_Amount__c, and Payment__c.Status__c = 'Pending Disbursement'.

Given any runtime exception during adjudication,
When the error occurs,
Then the claim must be routed to Status__c = 'Error - Adjudication' and a Case must be created and assigned to the Claims Operations queue.

## Solution

1. Platform Event Trigger subscriber (Apex): AdjudicationEventSubscriber.cls listens on FNOL_Submitted__e.
2. Apex Service class (ClaimAdjudicationService.cls) encapsulates adjudication logic with bulkified SOQL for Fee_Schedule__c matching.
3. Custom Metadata Type (Adjudication_Rule__mdt) stores the $500 threshold so it is configurable without code deployment.
4. Record-Triggered Flow (RTF_CreatePaymentOnAdjudication) fires After Update on Claim__c when Status__c changes to 'Auto-Adjudicated' to create Payment__c child record.
5. Apex unit tests must achieve ≥85% coverage with positive, negative, and bulk (200-record) test scenarios.
6. Custom index on Fee_Schedule__c.CPT_Code__c and Fee_Schedule__c.Effective_Date__c to support performant SOQL lookups.

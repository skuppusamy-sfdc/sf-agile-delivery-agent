## Description

As a Claims Operations Manager, I need an Agentforce-powered claim assignment recommendation engine so that newly created FNOL claims (Claim__c.Status__c = 'New') are intelligently routed to the most appropriate adjuster based on current workload, LOB specialization, and claim complexity — reducing average time-to-assignment from 4 hours to under 30 minutes.

The AI recommendation should consider:
- Adjuster current open claim count (workload)
- Adjuster LOB specialization tags (stored on User object: User.LOB_Specialization__c)
- Claim priority (Claim__c.Priority__c) — Critical claims should only be routed to senior adjusters (User.AdjusterTier__c = 'Senior')
- Geographic region match (Claim__c.LossState__c vs. User.LicensedStates__c)

This story supersedes the round-robin assignment approach deprecated in INS-0253.

## Acceptance Criteria

Given a new Claim__c record is created with Status__c = 'New' and LOB__c = 'Medical',
When the Agentforce Assignment Agent evaluates available adjusters,
Then it returns a ranked list of up to 3 recommended adjusters whose User.LOB_Specialization__c includes 'Medical', ordered by ascending open claim count.

Given Claim__c.Priority__c = 'Critical',
When the Agentforce agent evaluates,
Then only adjusters with User.AdjusterTier__c = 'Senior' are included in the recommendation set.

Given Claim__c.LossState__c = 'CA',
When the agent evaluates,
Then only adjusters whose User.LicensedStates__c contains 'CA' are eligible for recommendation.

Given the Claims Operations Manager accepts the top recommendation,
When they click 'Assign' on the Agentforce recommendation panel,
Then Claim__c.OwnerId is updated to the selected adjuster's User.Id and Claim__c.AssignmentMethod__c = 'AI-Recommended'.

Given no eligible adjuster meets all criteria,
When the agent completes evaluation,
Then the claim is placed in the 'Unassigned_Claims' Queue and a Slack notification is sent to the Claims Operations Manager.

## Solution

Configure an Agentforce Agent ('Claims Assignment Agent') using the Agentforce Agent Builder:
- Agent Topic: 'Claim Assignment Recommendation'
- Agent Actions:
  1. Apex Action 'GetEligibleAdjusters' — queries User records with active profiles, filters by LOB specialization, licensed states, adjuster tier; returns JSON list
  2. Apex Action 'GetAdjusterWorkload' — counts open Claim__c records per adjuster (Status__c NOT IN ('Closed','Denied'))
  3. Flow Action 'AssignClaimToAdjuster' — updates Claim__c.OwnerId and AssignmentMethod__c
- Integrate agent panel as an LWC component ('c/agentforceAssignmentPanel') embedded on the Claim__c Lightning Record Page
- Slack notification via Platform Event ('Unassigned_Claim__e') consumed by a Flow that calls Slack API
- Einstein Prompt Template used for natural-language recommendation summary displayed to the manager
- Permission Set 'Agentforce_Claims_User' grants access to agent features

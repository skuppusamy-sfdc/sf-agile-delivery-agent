## Description

As a Claims Operations Manager, I need the adjudication engine to automatically determine whether the rendering provider on a claim is in-network or out-of-network by checking the Provider_Network_Agreement__c (PNA) at the time of service so that the correct reimbursement rates and member cost-sharing are applied and EOB documents reflect accurate benefit tier information.

## Acceptance Criteria

Given a Claim__c record enters adjudication with a populated Claim__c.Rendering_Provider__c (linked to a Provider__c Person Account),
When the network determination logic runs,
Then the system queries Provider_Network_Agreement__c where PNA__c.Provider__c = Claim__c.Rendering_Provider__c AND PNA__c.Effective_Date__c <= Claim__c.Date_of_Loss__c AND PNA__c.Termination_Date__c >= Claim__c.Date_of_Loss__c AND PNA__c.Status__c = 'Active'.

Given an active PNA exists for the rendering provider at the date of service,
When the network check completes,
Then Claim__c.Network_Status__c = 'In-Network' and the associated Fee_Schedule__c.In_Network_Rate__c is used to calculate Claim_Line__c.Allowed_Amount__c.

Given no active PNA exists for the rendering provider at the date of service,
When the network check completes,
Then Claim__c.Network_Status__c = 'Out-of-Network' and Claim_Line__c.Allowed_Amount__c is calculated using the Fee_Schedule__c.Out_of_Network_Rate__c (UCR rate).

Given the provider has a PNA with PNA__c.Status__c = 'Terminated',
When the network check runs and the Claim__c.Date_of_Loss__c is after PNA__c.Termination_Date__c,
Then Claim__c.Network_Status__c = 'Out-of-Network' and Claim__c.Adjudication_Notes__c appends 'Provider network agreement terminated prior to date of service. Applied OON rates.'

-Given the provider NPI is not on file, pend the claim for manual review- (replaced by INS-0266 NPI verification story).

Given Claim__c.Network_Status__c is determined,
When the Explanation_of_Benefits__c (EOB) record is generated,
Then EOB__c.Network_Tier__c reflects the determined network status and EOB__c.Provider_Name__c is populated from Provider__c.Name.

Given a Facility__c (Business Account) is listed as the rendering facility instead of a Provider__c,
When the network check runs,
Then the system queries PNA__c where PNA__c.Facility__c = Claim__c.Rendering_Facility__c and applies the same date-of-service logic.

## Solution

Extend the BatchAdjudicationEligibilityCheck Apex class (INS-0261) with a new method (determineNetworkStatus) that executes PNA lookups using a Map<Id, Provider_Network_Agreement__c> keyed by Provider__c Id. Implement separate handling for Provider__c and Facility__c rendering entities using a polymorphic lookup pattern. Network status result is written to Claim__c.Network_Status__c via a before-update context in the trigger handler. Fee schedule selection logic branches on Network_Status__c value. EOB generation handled downstream by a Record-Triggered Flow on Claim__c when Status__c = 'Adjudicated'. A custom Permission Set grants read access to PNA__c for the Claims Integration User profile.

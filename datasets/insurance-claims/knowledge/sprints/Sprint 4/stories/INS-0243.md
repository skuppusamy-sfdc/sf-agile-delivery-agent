## Description

**Bug Report — NPPES NPI Validation Not Enforced During Claim Intake**

**Environment:** Acme Insurance Full Sandbox (SF-FULL-01)
**Related Stories:** INS-0241

**Steps to Reproduce:**
1. Navigate to a Claim__c record in 'FNOL Received' status.
2. In the Rendering_Provider_NPI__c field, manually enter an invalid or fictitious 10-digit NPI (e.g., '1234567890').
3. Save the Claim__c record.
4. Observe whether any validation error or warning is raised.

**Expected Behavior:** The system should call the NPPES NPI Registry API to validate the entered NPI. If the NPI is not found or is deactivated in NPPES, a validation error should prevent saving, or at minimum set Claim__c.NPI_Validation_Status__c = 'Invalid' and warn the user.

**Actual Behavior:** The Claim__c record saves successfully with any 10-digit numeric string in Rendering_Provider_NPI__c. No API call to NPPES is made. NPI_Validation_Status__c remains blank. This allows claims to proceed through adjudication with fictitious provider NPIs, creating downstream fraud risk and provider matching failures (as noted in INS-0241 AC4).

**Impact:** MEDIUM — Fictitious NPIs bypass the in-network determination logic in INS-0241, potentially causing incorrect fee schedule application and undetected fraudulent provider submissions.

**Frequency:** Reproducible 100% of the time when a non-existent NPI is entered manually.

## Acceptance Criteria

As per Description. Fix must validate NPI format (10 digits, Luhn-adjacent NPI check digit algorithm) client-side and call NPPES API server-side on Claim__c save. NPI_Validation_Status__c must reflect 'Valid', 'Invalid', or 'NPPES Unavailable' outcomes. Invalid NPIs must block claim progression past 'FNOL Received' status.

## Solution

Implement two-layer validation: (1) LWC input component on Claim__c record page with client-side NPI format check (regex: ^[0-9]{10}$) and NPI check-digit validation per CMS specification. (2) Apex trigger (ClaimNPIValidationTrigger.cls) on Claim__c before-insert/before-update that calls a future method or Queueable Apex to invoke the NPPES NPI Registry REST API (endpoint: https://npiregistry.cms.hhs.gov/api/). Response parsed to set NPI_Validation_Status__c picklist (Valid/Invalid/NPPES Unavailable). If status = 'Invalid', add to trigger's addError() to block DML. Named Credential configured for NPPES endpoint. Custom Setting for enabling/disabling live API call in sandbox environments.

## Description

**Bug Report**

**Summary:** The FNOL intake OmniScript (INS-0221) throws an unhandled null pointer error and crashes to a blank screen when the looked-up Policy__c record exists but has zero active Coverage_Line__c child records.

**Steps to Reproduce:**
1. Navigate to the FNOL intake OmniScript tab in the Claims Console.
2. Enter a valid Policy Number for a policy that has Policy__c.Status__c = 'Active' but no associated Coverage_Line__c records (e.g., test policy ACM-POL-00042 in Staging).
3. Click 'Next' to advance past the Policy Lookup step.
4. Observe the OmniScript step for 'Coverage Selection'.

**Expected Behavior:** The OmniScript should display a user-friendly inline validation message: 'No active coverage lines found for this policy. Please contact the policy administrator.' and prevent advancement to the next step.

**Actual Behavior:** The DataRaptor Transform on the Coverage Selection step returns a null collection. The OmniScript attempts to iterate the null array to populate a select dropdown and throws: 'Cannot read properties of null (reading length)' — the entire OmniScript crashes and the screen goes blank. No error message is shown to the user.

**Environment:** Staging org (API v59.0). Reproducible 100% of the time with policies lacking Coverage_Line__c records.

**Severity:** High — blocks claims intake for any policyholder whose coverage lines have not yet been migrated or have expired.

## Solution

Add a null/empty check in the DataRaptor Transform output mapping for the Coverage_Line__c collection before it is bound to the OmniScript select element. In the OmniScript Coverage Selection step, add a Conditional View that checks if the coverage lines array length === 0 and renders an error message block instead of the dropdown. Additionally, add a default empty array initializer in the Integration Procedure that fetches coverage lines so downstream steps never receive a null reference. Update OmniScript step validation to block 'Next' navigation when the coverage lines array is empty. Regression test with policies having 0, 1, and N coverage lines.

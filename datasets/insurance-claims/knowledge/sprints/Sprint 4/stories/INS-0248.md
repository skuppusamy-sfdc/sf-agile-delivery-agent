## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that displays the provider's NPI verification status from NPPES, active Provider_Network_Agreement__c records, and any open Credentialing_Case__c items so that I can assess a provider's full contracting and credentialing posture in a single view without toggling between systems.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record,
When the Provider 360 FlexCard loads,
Then it displays Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Specialty__c, and an NPI Verification badge reflecting the most recent DataIntegration sync with NPPES (Provider__c.NPI_Verified__c = true/false and Provider__c.NPI_Last_Verified_Date__c).

Given the FlexCard renders the Network Agreements section,
When there are related Provider_Network_Agreement__c records with PNA__c.Status__c = 'Active',
Then each active PNA is listed showing PNA__c.Network_Name__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, and PNA__c.Fee_Schedule__c.

Given there are open Credentialing_Case__c records (Credentialing_Case__c.Status__c NOT IN ('Closed', 'Withdrawn')),
When the FlexCard renders,
Then the Credentialing section shows Credentialing_Case__c.Case_Number__c, Credentialing_Case__c.Credentialing_Type__c, Credentialing_Case__c.Status__c, and Credentialing_Case__c.Assigned_Specialist__c.

Given a provider has no active PNA records,
When the Network Agreements section renders,
Then an amber warning banner displays: 'No active network agreements. Provider may be out-of-network.'

## Solution

Build an OmniStudio FlexCard (Name: Provider_360_FC) with three card states: NPI & Demographics, Network Agreements, and Credentialing Cases. DataRaptor Extract actions retrieve PNA and Credentialing_Case__c data using AccountId as the input key. NPI verification badge uses a conditional icon state driven by Provider__c.NPI_Verified__c boolean. NPPES sync is handled by a scheduled Integration Procedure (IP: NPPES_NPI_Sync) that calls the NPPES REST API and updates Provider__c via DataRaptor Load. FlexCard embedded on the Account Lightning Record Page filtered by Record Type = 'Provider'.

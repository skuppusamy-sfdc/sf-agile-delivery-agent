## Description

As a Claims Operations Manager, I need the system to automatically evaluate coverage eligibility for each Claim_Line__c against the associated Policy__c and Fee_Schedule__c so that adjudication decisions are applied consistently and without manual lookup, reducing processing time and payment errors. Extends INS-0255 (Fee Schedule data load).

## Acceptance Criteria

Given a Claim__c record with Status__c = 'FNOL Received' transitions to Status__c = 'Under Review',
When the adjudication rule engine is triggered,
Then each child Claim_Line__c record is evaluated for coverage eligibility within 10 seconds.

Given a Claim_Line__c has a valid CPT code in Claim_Line__c.CPT_Code__c that exists in the active Fee_Schedule__c,
When the eligibility check runs,
Then Claim_Line__c.Adjudication_Status__c is set to 'Eligible' and Claim_Line__c.Allowed_Amount__c is populated from Fee_Schedule__c.Allowed_Amount__c.

Given a Claim_Line__c.CPT_Code__c is not found in any active Fee_Schedule__c record linked to the policy's LOB,
When the eligibility check runs,
Then Claim_Line__c.Adjudication_Status__c = 'Denied — CPT Not Covered' and Claim_Line__c.Denial_Reason_Code__c = 'CO-4'.

Given the Date_of_Service__c on Claim_Line__c falls outside Policy__c.Effective_Date__c and Policy__c.Expiration_Date__c,
When the eligibility check runs,
Then Claim_Line__c.Adjudication_Status__c = 'Denied — Coverage Lapsed' and Claim_Line__c.Denial_Reason_Code__c = 'CO-15'.

Given all Claim_Line__c records on a Claim__c have been evaluated,
When all lines are set to either 'Eligible' or 'Denied',
Then Claim__c.Status__c transitions to 'Adjudicated' and Claim__c.Adjudication_Completed_Date__c is stamped.

-Given Claim_Line__c.ICD10_Code__c is invalid, set Status to 'Pended'- (moved to INS-0263 ICD-10 validation story).

Given a processing error occurs during rule evaluation,
When the Apex batch catches an exception,
Then Claim__c.Status__c is set to 'Adjudication Error', an Error_Log__c record is created with the exception message, and a Case is opened and assigned to the 'Claims Ops Escalation' queue.

## Solution

Implement an Apex Batch class (BatchAdjudicationEligibilityCheck) invoked by a Record-Triggered Flow on Claim__c when Status__c changes to 'Under Review'. The batch iterates over related Claim_Line__c records, performs a SOQL lookup against Fee_Schedule__c filtered by LOB__c and Effective_Date__c, and applies denial reason codes per CMS standard remark codes. Results written back via DML update on Claim_Line__c. A post-execute method checks aggregate line statuses and updates the parent Claim__c. Error handling writes to a custom Error_Log__c object and fires a Case creation via Process Invocable Action. Unit test coverage target: ≥85%.

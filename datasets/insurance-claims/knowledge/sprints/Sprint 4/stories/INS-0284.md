## Description

As a Claims Operations Manager, I need a guided FNOL intake OmniScript so that claims representatives can capture all required first-notice details for Auto and Medical LOBs in a single, validated session without switching between screens.

The OmniScript must create a Claim__c record and one or more Claim_Line__c records upon submission, pre-populating data from the linked Policy__c.

## Acceptance Criteria

Given a logged-in claims rep navigates to 'New FNOL' from the Claims console,
When the OmniScript launches,
Then Step 1 must prompt for Policy__c lookup by Policy_Number__c or Policyholder TIN and validate that Policy__c.Status__c = 'Active'.

Given a valid Policy__c is selected,
When the rep proceeds to Step 2,
Then Claim__c fields Policyholder__c, LOB__c, Policy_Effective_Date__c, and Group_Service_Agreement__c must be auto-populated via DataRaptor Extract from the matched Policy__c record.

Given the LOB__c = 'Medical',
When the rep reaches the Claim Lines step,
Then at least one Claim_Line__c must be created with CPT_Code__c, ICD10_Diagnosis_Code__c, Service_Date__c, and Billed_Amount__c as required fields.

Given the LOB__c = 'Auto',
When the rep reaches the Incident step,
Then Claim__c.Incident_Date__c, Claim__c.Incident_Description__c, and Claim__c.Reported_By__c must be required.

Given all required fields are populated,
When the rep clicks Submit,
Then Claim__c.Status__c must be set to 'FNOL Received', Claim__c.FNOL_Submitted_Date__c must be stamped with the current timestamp, and a confirmation number (Claim__c.Claim_Number__c) must be displayed.

Given submission is successful,
When the record is created,
Then a Platform Event (FNOL_Submitted__e) must be published to trigger downstream adjudication queue assignment.

-Given LOB__c = 'Workers Comp', the OmniScript should branch to a WC-specific step set- (Descoped to Sprint 6; see INS-0311)

## Solution

Build a multi-step OmniScript (FNOL_Intake_v1) in OmniStudio with the following structure:
1. DataRaptor Extract (DR_GetPolicyByNumber) to fetch Policy__c and linked GSA__c on policy lookup.
2. Conditional branching elements for LOB__c = 'Medical' vs 'Auto'.
3. DataRaptor Transform to map OmniScript state to Claim__c and Claim_Line__c field API names.
4. DataRaptor Post to upsert Claim__c (External ID: Claim_Number__c) and insert related Claim_Line__c records.
5. Apex Action (PublishFNOLEvent) to fire FNOL_Submitted__e Platform Event post-save.
6. Custom validation rules on Claim__c to enforce ICD-10 format (regex: ^[A-Z][0-9]{2}\.?[0-9A-Z]{0,4}$).
7. Assign relevant Permission Set Group (PSG_Claims_Intake) to ensure reps have Create access on Claim__c and Claim_Line__c.

## Description

As a Compliance Officer, I need properly scoped Salesforce Permission Set Groups defined for all Phase 1 claims processing roles so that each user persona has least-privilege access to Claims, Policy, Provider, and Payment objects, ensuring audit readiness and SOC 2 compliance alignment before the FNOL and adjudication features go live.

## Acceptance Criteria

Given the Permission Set Groups are deployed to the Full Sandbox,
When a user assigned to 'Claims_Adjuster_PSG' accesses Salesforce,
Then they have Read/Create/Edit access to Claim__c and Claim_Line__c, Read-Only access to Policy__c and Provider__c, and NO access to Payment__c or Fee_Schedule__c records.

Given a user assigned to 'Claims_Manager_PSG',
When they access Salesforce,
Then they have Read/Create/Edit/Delete access to Claim__c, Claim_Line__c, and Payment__c, and Read access to all other Phase 1 objects including EOB__c and Fee_Schedule__c.

Given a user assigned to 'Provider_Relations_PSG',
When they access Salesforce,
Then they have Read/Create/Edit access to Provider__c (Person Account), Provider_Network_Agreement__c, and Credentialing_Case__c, and Read-Only access to Claim__c with no access to Payment__c.

Given a user assigned to 'SIU_Analyst_PSG',
When they access Salesforce,
Then they have Read-Only access to Claim__c, Claim_Line__c, and Payment__c with an additional custom permission 'SIU_Flag_Claim__c' that enables the 'Refer to SIU' action on Claim__c.

Given all PSGs are deployed,
When a Compliance Officer runs the Permission Set Group Audit Report (custom report type: PSG_Object_Access__rt),
Then the report confirms no PSG grants access beyond its defined scope and the report is exportable as a PDF for audit evidence.

## Solution

Create four Permission Set Groups in Salesforce Setup: Claims_Adjuster_PSG, Claims_Manager_PSG, Provider_Relations_PSG, and SIU_Analyst_PSG. Each PSG aggregates purpose-built Permission Sets (e.g., PS_Claim_ReadWrite, PS_Payment_ReadWrite) using the muting permission set pattern to suppress over-permissioned base PS entries. Deploy via Salesforce DX (sfdx force:source:deploy) with metadata in permissionsets/ and permissionsetgroups/ directories. Custom Permission 'SIU_Flag_Claim' created as a Custom Permission metadata record and referenced in SIU_Analyst_PSG. Audit report built as a custom report type on PermissionSetGroup + PermissionSetGroupComponent objects.

## Description

As a Claims Operations Manager, I need the system to automatically evaluate each Claim_Line__c record against the applicable Fee_Schedule__c so that allowed amounts are calculated without manual adjuster intervention for clean claims, reducing adjudication cycle time.

The engine must resolve the correct Fee_Schedule__c based on Provider_Network_Agreement__c (PNA) effective dates, the Claim__c.Date_of_Service__c, and the Claim_Line__c.CPT_Code__c. Billed amounts exceeding the fee schedule maximum should be reduced to the allowed amount and flagged.

## Acceptance Criteria

Given a Claim__c record with Status__c = 'FNOL Received' and one or more Claim_Line__c children,
When the adjudication batch job runs,
Then each Claim_Line__c.Allowed_Amount__c is populated by matching Claim_Line__c.CPT_Code__c to Fee_Schedule__c.CPT_Code__c where Fee_Schedule__c.Effective_Date__c <= Claim__c.Date_of_Service__c <= Fee_Schedule__c.Expiration_Date__c.

Given a matching Fee_Schedule__c record is found,
When Claim_Line__c.Billed_Amount__c > Fee_Schedule__c.Max_Allowed_Amount__c,
Then Claim_Line__c.Allowed_Amount__c = Fee_Schedule__c.Max_Allowed_Amount__c and Claim_Line__c.Adjudication_Flag__c = 'Reduced — Exceeds Fee Schedule'.

Given a matching Fee_Schedule__c record is found,
When Claim_Line__c.Billed_Amount__c <= Fee_Schedule__c.Max_Allowed_Amount__c,
Then Claim_Line__c.Allowed_Amount__c = Claim_Line__c.Billed_Amount__c and Claim_Line__c.Adjudication_Flag__c = 'Clean'.

Given no matching Fee_Schedule__c record exists for the CPT code and date range,
When adjudication is attempted,
Then Claim_Line__c.Adjudication_Flag__c = 'Pending Manual Review' and Claim__c.Status__c remains 'In Adjudication' without advancing.

Given all Claim_Line__c records on a Claim__c are flagged 'Clean',
When the batch completes,
Then Claim__c.Status__c is updated to 'Adjudicated' and Claim__c.Adjudication_Completed_Date__c = TODAY().

Given the batch processes more than 200 Claim_Line__c records,
When executing,
Then the Apex Batch class processes records in chunks of 200 without hitting governor limits and logs batch results to a custom Adjudication_Batch_Log__c object.

## Solution

Implement an Apex Batch class (AdjudicationBatch) implementing Database.Batchable and Database.Stateful. Query Claim_Line__c records where Claim__c.Status__c = 'FNOL Received'. For each record, query Fee_Schedule__c using a SOQL join on CPT_Code__c and date range. Apply allowed amount logic and set Adjudication_Flag__c. Use a helper AdjudicationRuleEngine Apex class to encapsulate rule logic for testability (>85% code coverage required). After all lines are processed, use a Record-Triggered Flow on Claim__c (scheduled path, 0 hours after Adjudication_Completed_Date__c) to update Status__c to 'Adjudicated'. Schedule the batch via a Schedulable wrapper to run every 15 minutes using a Named Credential-protected cron. Custom metadata type (Adjudication_Config__mdt) stores configurable thresholds.

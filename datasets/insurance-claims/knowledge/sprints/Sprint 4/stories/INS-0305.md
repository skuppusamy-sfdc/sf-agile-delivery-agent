## Description

As a Claims Operations Manager, I need the system to automatically generate an Explanation_of_Benefits__c (EOB) record when a Claim__c reaches 'Adjudicated' status so that policyholders and providers receive accurate, timely payment explanation data in compliance with state-mandated EOB delivery timelines (typically 30 days from date of service).

The EOB must summarize billed amounts, allowed amounts, member responsibility (deductible/copay/coinsurance), and plan payment per Claim_Line__c, and must be linked to both the Claim__c and the Payment__c record.

## Acceptance Criteria

Given Claim__c.Status__c transitions to 'Adjudicated',
When the Record-Triggered Flow fires on the Claim__c object,
Then one Explanation_of_Benefits__c record is created with EOB__c.Claim__c = Claim__c.Id, EOB__c.Policy__c = Claim__c.Policy__c, EOB__c.EOB_Date__c = TODAY(), and EOB__c.Status__c = 'Draft'.

Given the EOB__c record is created,
When the EOB line detail population job runs,
Then one EOB_Line__c child record is created per Claim_Line__c, populated with: EOB_Line__c.CPT_Code__c, EOB_Line__c.Billed_Amount__c = Claim_Line__c.Billed_Amount__c, EOB_Line__c.Allowed_Amount__c = Claim_Line__c.Allowed_Amount__c, EOB_Line__c.Member_Responsibility__c = Claim_Line__c.Copay_Amount__c + Claim_Line__c.Deductible_Applied__c + Claim_Line__c.Coinsurance_Amount__c, and EOB_Line__c.Plan_Payment__c = Claim_Line__c.Allowed_Amount__c - EOB_Line__c.Member_Responsibility__c.

Given the Payment__c record exists for the adjudicated claim,
When EOB__c.Status__c = 'Draft',
Then EOB__c.Payment__c is populated with the linked Payment__c.Id.

Given EOB__c.Status__c = 'Draft',
When all EOB_Line__c records are created and EOB__c.Total_Plan_Payment__c = SUM(EOB_Line__c.Plan_Payment__c) is calculated,
Then EOB__c.Status__c is updated to 'Final' and EOB__c.Finalized_Date__c = TODAY().

Given EOB__c.Status__c = 'Final',
When the EOB delivery schedule runs,
Then a Salesforce-generated PDF (via Conga CLM template: EOB_Standard_v2) is attached to the EOB__c record as a ContentDocument and EOB__c.Delivery_Method__c is set based on Policy__c.Communication_Preference__c ('Mail', 'Email', or 'Portal').

## Solution

Use a Record-Triggered Flow (Object: Claim__c, Trigger: After Update, Entry Criteria: Status__c CHANGED TO 'Adjudicated') to create the parent Explanation_of_Benefits__c record. Invoke an Apex invocable method (EOBGenerationService.createEOBLines()) to bulk-create EOB_Line__c records from related Claim_Line__c records — this avoids flow loop DML issues. Use a separate Record-Triggered Flow on EOB__c to roll up Total_Plan_Payment__c and flip Status__c to 'Final' once all lines are present (validated via EOB__c.Expected_Line_Count__c = COUNT(EOB_Line__c)). Conga CLM merge job triggered via a scheduled Flow action on EOB__c records in 'Final' status. Store merged PDF via ContentVersion insert linked to EOB__c via ContentDocumentLink.

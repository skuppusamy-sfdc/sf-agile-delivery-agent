## Description

As a Claims Operations Manager, I need the claims intake process to automatically validate that the rendering provider on a submitted claim has an active Provider_Network_Agreement__c (PNA) for the relevant LOB and effective date range so that out-of-network claims are correctly identified and flagged for appropriate reimbursement handling rather than being processed as in-network. Extends INS-0221.

## Acceptance Criteria

Given a Claim__c record is created via FNOL intake with a Provider__c lookup populated,
When the PNA validation step in the adjudication Integration Procedure runs,
Then the system queries Provider_Network_Agreement__c where PNA__c.Provider__c = Claim__c.Provider__c AND PNA__c.LOB__c = Claim__c.LOB__c AND PNA__c.Status__c = 'Active' AND PNA__c.Effective_Date__c <= Claim__c.Date_of_Loss__c AND PNA__c.Termination_Date__c >= Claim__c.Date_of_Loss__c.

Given no active PNA is found for the provider and LOB combination,
When the validation completes,
Then Claim__c.Network_Status__c is set to 'Out-of-Network' and Claim__c.Status__c remains 'Under Review' with a flag Claim__c.OON_Flag__c = TRUE.

Given an active PNA is found,
When the validation completes,
Then Claim__c.Network_Status__c is set to 'In-Network' and the related Provider_Network_Agreement__c.Id is stamped on Claim__c.PNA__c lookup field.

Given Claim__c.OON_Flag__c = TRUE,
When the record is saved,
Then a Record-Triggered Flow creates a follow-up Task assigned to the Provider Network Specialist queue with Subject = 'OON Claim Review Required – [Claim Number]' and Due Date = TODAY() + 2 business days.

Given the PNA lookup finds multiple active agreements,
When selecting which PNA to associate,
Then the system selects the PNA with the most recent PNA__c.Effective_Date__c and logs a warning in Claim__c.Adjudication_Notes__c.

## Solution

Extend the ClaimAdjudicationHandler Apex class (from INS-0222) with a PNAValidationService inner class. SOQL query on Provider_Network_Agreement__c with compound WHERE clause. Results update Claim__c fields via a single bulkified DML. OON Task creation handled by a Record-Triggered Flow on Claim__c [After Update] filtered on OON_Flag__c changing to TRUE, using the Get Records element to fetch the Provider Network Specialist queue Id. Custom field Claim__c.PNA__c (Lookup to Provider_Network_Agreement__c) added via field set. Multiple PNA resolution logic uses SOQL ORDER BY Effective_Date__c DESC LIMIT 1.

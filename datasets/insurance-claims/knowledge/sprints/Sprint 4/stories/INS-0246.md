## Description

As a Claims Operations Manager, I need a Policyholder 360 FlexCard embedded on the Contact record page so that I can see all active Policy__c records, open Claim__c records, and recent Payment__c disbursements for a given policyholder in a single consolidated view without navigating away from the Contact record.

## Acceptance Criteria

Given a Claims Operations Manager opens a Contact record flagged as a Policyholder (Contact.Is_Policyholder__c = true),
When the Policyholder 360 FlexCard loads,
Then it displays all related Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.PolicyHolder__c = Contact.Id, showing Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, and Policy__c.Coverage_Limit__c.

Given the FlexCard is rendered,
When there are related Claim__c records with Claim__c.Status__c IN ('FNOL Received', 'Under Review', 'Pending Payment'),
Then each open claim is listed with Claim__c.Claim_Number__c, Claim__c.Date_of_Loss__c, Claim__c.LOB__c, and a color-coded status badge (yellow = 'Under Review', red = 'Pending Payment', blue = 'FNOL Received').

Given the FlexCard is rendered,
When the policyholder has Payment__c records in the last 90 days (Payment__c.Payment_Date__c >= LAST_90_DAYS),
Then the three most recent payments are shown with Payment__c.Amount__c, Payment__c.Payment_Date__c, and Payment__c.Payment_Method__c.

Given a Contact with no active policies,
When the FlexCard loads,
Then an empty-state message reads 'No active policies found for this policyholder.' and no claim or payment sections are rendered.

-Given the FlexCard loads, a 'Request Policy Change' quick action button is displayed- (deprecated — policy change action moved to Agent Portal epic, see INS-0231).

## Solution

Build an OmniStudio FlexCard (Name: Policyholder_360_FC) with three child card states: Active Policies, Open Claims, and Recent Payments. Use three separate DataRaptor Extract actions as card data sources, each filtered by ContactId passed as an input parameter. Configure conditional visibility on the Payments section using a FlexCard state condition on record count > 0. Embed the FlexCard on the Contact Lightning Record Page via the OmniStudio FlexCard component, restricted to the 'Claims_Adjuster_PSG' and 'Claims_Manager_PSG' Permission Set Groups.

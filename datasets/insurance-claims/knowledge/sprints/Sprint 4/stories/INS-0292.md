## Description

As a Claims Operations Manager, I need a guided FNOL intake OmniScript that captures all initial claim details for Auto and Medical lines of business so that a Claim__c record is created accurately and routed for adjudication without manual re-keying.

The OmniScript must collect: policyholder identity (Policy__c lookup), date/time of loss, LOB-specific loss details, injured party information, and an optional Provider__c lookup for Medical LOB. On completion, a Claim__c record is created with Status__c = 'FNOL Received' and a Platform Event fires to trigger downstream assignment.

## Acceptance Criteria

Given a logged-in Claims Operations user opens the FNOL OmniScript,
When they enter a valid Policy Number that matches an active Policy__c record,
Then the policyholder name, coverage effective dates, and LOB are auto-populated via DataRaptor Extract.

Given the user selects LOB = 'Medical',
When the LOB step renders,
Then a Provider__c lookup field (filtered to in-network providers via Provider_Network_Agreement__c.Status__c = 'Active') is displayed and the ICD-10 diagnosis code field is required.

Given the user selects LOB = 'Auto',
When the LOB step renders,
Then the Provider__c lookup is hidden and vehicle VIN and accident location fields are required.

Given all required fields are completed,
When the user clicks 'Submit FNOL',
Then a Claim__c record is created with Claim__c.Status__c = 'FNOL Received', Claim__c.Date_of_Loss__c populated, and Claim__c.LOB__c set correctly.

Given the Claim__c record is created successfully,
When the OmniScript completes,
Then a ClaimFNOLReceived__e Platform Event is published containing Claim__c.Id and Claim__c.LOB__c so that the Assignment Coordinator queue is notified.

Given a Policy__c record has Status__c = 'Lapsed' or 'Cancelled',
When the policy number is entered in the OmniScript,
Then an inline error message reads 'Policy is not currently active — FNOL cannot be submitted' and submission is blocked.

-Given the user abandons the OmniScript mid-way, a draft Claim__c with Status__c = 'Draft' is saved automatically- (deprecated — auto-draft removed per compliance review INS-0281).

## Solution

Build a multi-step OmniScript (Type: FNOL, Sub-Type: ClaimsIntake) in OmniStudio. Use DataRaptor Extract to query Policy__c on policy number entry. Implement conditional step visibility using OmniScript Conditional Views for LOB branching. On final step, use a DataRaptor Turbo Action to perform the Claim__c insert. Publish ClaimFNOLReceived__e via an Apex Action called from the OmniScript. Deploy a Record-Triggered Flow on Claim__c (After Insert, Status__c = 'FNOL Received') to create a Task for the Assignment Coordinator. Grant access via Permission Set Group: Claims_Intake_Users.

## Description

As a Claims Operations Manager, I need the adjudication process to automatically determine whether the rendering provider on a claim is in-network or out-of-network at the time of service, so that correct fee schedules and member cost-sharing rules are applied, ensuring accurate EOB generation and reducing provider disputes.

## Acceptance Criteria

Given a Claim__c record has a rendering Provider__c (Person Account) populated,
When adjudication rules execute (triggered by Status__c = 'Under Review'),
Then the system queries Provider_Network_Agreement__c records where PNA.Provider__c = Claim__c.Rendering_Provider__c and PNA.Effective_Date__c <= Claim__c.Date_of_Service__c and PNA.Termination_Date__c >= Claim__c.Date_of_Service__c and PNA.Status__c = 'Active'.

Given an active PNA is found,
When the network check completes,
Then Claim__c.Network_Status__c = 'In-Network' and the in-network Fee_Schedule__c is applied to all Claim_Line__c records.

Given no active PNA is found for the date of service,
When the network check completes,
Then Claim__c.Network_Status__c = 'Out-of-Network', Claim_Line__c.Cost_Share_Tier__c = 'OON', and Claim__c.Adjudication_Flag__c includes 'Out-of-Network Provider'.

Given the provider's NPI on the claim does not match any Provider__c record in Salesforce,
When the NPI lookup runs,
Then Claim__c.Adjudication_Flag__c = 'Unknown Provider - NPI Not Found' and a Case is created and routed to the Provider Relations Specialist queue for manual resolution.

Given Claim__c.Network_Status__c is set,
When the Explanation_of_Benefits__c (EOB) record is auto-generated,
Then EOB__c.Network_Status__c mirrors Claim__c.Network_Status__c and EOB__c.Applied_Fee_Schedule__c is populated with the Fee_Schedule__c record ID used.

## Solution

Extend the ClaimAdjudicationService.cls Apex class (built in INS-0238) with a new method: evaluateProviderNetworkStatus(List<Claim__c> claims). Method performs a SOQL query on Provider_Network_Agreement__c with date-range and status filters. NPI lookup cross-references Provider__c.NPI__c field. Network status and fee schedule assignment written back via DML update on Claim__c and Claim_Line__c. EOB__c record creation handled by a separate Record-Triggered Flow on Claim__c (trigger: Network_Status__c populated). Permission Set Group 'Claims Adjudicator' grants read access to PNA and Fee_Schedule__c objects.

## Description

As a Policyholder, I need to upload supporting claim documents (e.g., medical bills, police reports, photos) through the Experience Cloud self-service portal and have them automatically associated with my Claim__c record so that the adjuster has all necessary evidence without requiring me to mail or fax documents.

This story was split from INS-0268 (FNOL OmniScript) to isolate document handling complexity. Supported file types: PDF, JPG, PNG, DOCX. Maximum file size: 10 MB per file, 50 MB total per claim.

## Acceptance Criteria

Given an authenticated Policyholder navigates to the Claims detail page in Experience Cloud,
When they click 'Upload Documents',
Then an LWC file uploader component is displayed showing accepted file types (PDF, JPG, PNG, DOCX) and size limits.

Given the Policyholder selects a valid file,
When the file is uploaded,
Then a ContentDocument record is created and a ContentDocumentLink links it to the corresponding Claim__c.Id with ShareType = 'V'.

Given the uploaded file exceeds 10 MB,
When the client-side validation runs,
Then an error message 'File exceeds maximum size of 10 MB. Please compress and retry.' is displayed and the file is not uploaded.

Given a file is successfully attached,
When the ContentDocumentLink is created,
Then Claim__c.Document_Count__c (roll-up summary field) increments and the adjuster's Claim record page refreshes to show the new document in the Files related list.

Given the total uploaded file size for a Claim__c exceeds 50 MB,
When the next upload is attempted,
Then the upload is blocked and a message reads 'Maximum total document size of 50 MB reached for this claim.'

Given a document is uploaded by a Policyholder,
When the ContentDocumentLink is saved,
Then a Record-Triggered Flow sends an in-app notification to the assigned adjuster (Claim__c.OwnerId) with message 'New document uploaded for Claim [Claim__c.Name]'.

## Solution

Build a custom LWC 'claimDocumentUploader' using the lightning-file-upload base component with accept and max-file-size attributes set client-side. Implement an Apex controller method 'ClaimDocumentController.linkDocument(contentDocumentId, claimId)' to create the ContentDocumentLink with appropriate sharing. Add a before-insert Apex Trigger on ContentDocumentLink to enforce 50 MB aggregate limit by querying existing ContentVersion sizes for the Claim. Configure a Record-Triggered Flow on ContentDocumentLink (After Insert) filtered by LinkedEntityId = Claim__c to fire the adjuster notification via Custom Notification. Expose the LWC on the Experience Cloud Claims detail page via a community page layout. Apply Guest User Profile restrictions to prevent unauthenticated uploads.

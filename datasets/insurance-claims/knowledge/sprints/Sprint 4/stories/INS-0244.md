## Description

As a Claims Operations Manager, I need the FNOL intake process to capture and flag Coordination of Benefits (COB) information when a claimant indicates they have secondary insurance coverage, so that primary and secondary payer sequencing is correctly established before adjudication begins, preventing overpayment and ensuring compliance with CMS COB regulations.

## Acceptance Criteria

Given a claimant indicates 'Yes' to the 'Other Insurance Coverage?' question during FNOL OmniScript intake (Step 3),
When the COB step is revealed,
Then additional fields are displayed: COB_Payer_Name__c, COB_Policy_Number__c, COB_Group_Number__c, COB_Payer_Priority__c (picklist: Primary / Secondary / Tertiary), and COB_Effective_Date__c — all required if COB = 'Yes'.

Given COB information is submitted,
When the Claim__c record is created,
Then Claim__c.COB_Flag__c = true and a related COB_Detail__c child record is created with all captured payer details.

Given Claim__c.COB_Flag__c = true,
When the adjudication rule engine runs (per INS-0238),
Then Claim__c.Adjudication_Flag__c includes 'COB - Requires Primary Payer EOB' and Status__c is set to 'Pending COB Resolution' until a primary payer EOB document is attached.

Given a COB-flagged claim,
When a Compliance Officer or adjuster attaches a document with Document_Type__c = 'Primary Payer EOB' to the Claim__c ContentDocumentLink,
Then a Record-Triggered Flow detects the attachment and sets Claim__c.COB_Primary_EOB_Received__c = true and transitions Status__c back to 'Under Review' automatically.

-Given COB is flagged, the system auto-requests EOB from the primary payer via EDI 276/277 transaction- (deprecated — EDI integration deferred to Phase 2 pending TPA agreement; manual attachment process used for Phase 1)

Given Claim__c.COB_Flag__c = false (no secondary coverage indicated),
When adjudication runs,
Then no COB-related flags are set and normal adjudication proceeds without interruption.

Given a Compliance Officer runs the 'COB Pending Resolution' report,
When the report executes,
Then all Claim__c records with COB_Flag__c = true and COB_Primary_EOB_Received__c = false and Status__c = 'Pending COB Resolution' aged > 10 business days are listed with Claim_Number__c, Claimant_Name__c, and Days_Pending__c.

## Solution

Extend the FNOL OmniScript (INS-0237) with a conditional Step 3b (COB Details) revealed via SetValues when OtherCoverage__c = 'Yes'. DataRaptor Post creates both Claim__c and COB_Detail__c child records in a single transaction using a parent-child DataRaptor Post map. COB adjudication flag logic added to ClaimAdjudicationService.cls (INS-0238). EOB document detection implemented via Record-Triggered Flow on ContentDocumentLink (trigger: after insert) with filter on LinkedEntityId = Claim__c ID and Document_Type__c = 'Primary Payer EOB' using a related ContentVersion lookup. Compliance report built as a Salesforce CRM Analytics (CRMA) dashboard tile and also as a standard List View with formula field Days_Pending__c = TODAY() - Status_Changed_Date__c. Permission Set Group 'Compliance Reviewer' granted access to COB_Detail__c object.

## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces current credentialing status, active network agreements, and open claims so that I can assess a provider's full relationship with Acme Insurance in one place without switching between multiple tabs.

The FlexCard should show: Credentialing_Case__c summary (most recent, with status), all active Provider_Network_Agreement__c records, and a count + list of Claim__c records where Claim__c.Rendering_Provider__c = Provider and Status__c not in ('Closed','Denied').

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the most recent Credentialing_Case__c (ordered by CreatedDate DESC) is displayed showing Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and Credentialing_Case__c.CAQH_Profile_ID__c.

Given the FlexCard is loaded,
When the Network Agreements panel renders,
Then all Provider_Network_Agreement__c records where PNA.Provider__c = current Account Id and PNA.Status__c = 'Active' are listed with PNA.Network_Name__c, PNA.Effective_Date__c, and PNA.Termination_Date__c.

Given the FlexCard is loaded,
When the Open Claims panel renders,
Then Claim__c records where Rendering_Provider__c = current Account Id and Status__c NOT IN ('Closed','Denied') are shown, with a badge displaying the total count.

Given the provider has a Credentialing_Case__c with Status__c = 'Expired',
When the FlexCard renders the credentialing panel,
Then the status badge is displayed in red and a warning icon reads 'Credentialing Expired — Claims may be impacted'.

## Solution

Build an OmniStudio FlexCard (Name: Provider360_FC) using three DataRaptor Extract data sources targeting Credentialing_Case__c, Provider_Network_Agreement__c, and Claim__c. Use conditional styling states on the credentialing block to render the red badge when Status__c = 'Expired'. Embed on the Person Account Lightning Record Page (filtered to Record Type = 'Provider') using dynamic page assignments. Apply a Permission Set (Provider_Relations_Read) to control visibility. Use flyout cards for PNA detail drill-down.

## Description

As a Claims Operations Manager, I need a Provider 360 FlexCard embedded on the Provider__c (Person Account) record page that surfaces the provider's current credentialing status, active PNA agreements, and recent claim activity so that adjudicators and Provider Relations Specialists can assess provider standing in a single view without navigating multiple related lists.

## Acceptance Criteria

Given a user with the Claims_Ops_PSG or Provider_Relations_PSG Permission Set Group opens a Provider__c record page,
When the Provider 360 FlexCard renders,
Then it displays: Provider__c.Credentialing_Status__c, the count of active Provider_Network_Agreement__c records (PNA__c.Status__c = 'Active'), Provider__c.NPI__c, Provider__c.TIN__c, and the most recent Credentialing_Case__c.Status__c.

Given the FlexCard is rendered,
When Provider__c.Credentialing_Status__c = 'Expired' or 'Suspended',
Then the credentialing status badge renders in red and a warning banner displays: 'Caution: Claims associated with this provider may be subject to denial pending re-credentialing.'

Given the user clicks 'View Active Agreements',
When the child FlexCard datatable expands,
Then it lists all Provider_Network_Agreement__c records with PNA__c.LOB__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, and PNA__c.Network_Tier__c.

Given the FlexCard renders,
When there are Claim__c records in the last 90 days with this provider as rendering provider,
Then a 'Recent Claims' section shows count of claims by Status__c and a sparkline-style count badge.

Given the user is in the Provider_Relations_PSG,
When the FlexCard renders,
Then an additional 'Initiate PCN' action button is visible that launches a Provider_Change_Notification__c quick action Screen Flow; this button is hidden for Claims_Ops_PSG users.

## Solution

Build OmniStudio FlexCard (Name: Provider_360_Summary) with multiple DataRaptor Extract data sources: one for Provider__c base fields, one for PNA count aggregate, one for recent Claim__c summary. Use FlexCard states for collapsed/expanded PNA datatable. Conditional styling on Credentialing_Status__c using FlexCard CSS state conditions. Role-based button visibility controlled via FlexCard conditional visibility expression referencing $User.PermissionSetGroupAssignments. 'Initiate PCN' action configured as a FlexCard Action of type 'Screen Flow' pointing to the PCN_Initiation_Flow. Embed on Provider__c (Person Account) Lightning Record Page via App Builder with component visibility filter: Record Type = 'Provider'.

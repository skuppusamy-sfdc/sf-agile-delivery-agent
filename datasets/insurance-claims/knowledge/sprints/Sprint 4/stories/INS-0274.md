## Description

As a Claims Operations Manager, I need an Agentforce-powered AI assistant embedded on the Claim__c record page that can summarize the full claim history, highlight anomalies (e.g., duplicate CPT codes, out-of-network providers, SIU flags), and suggest the next best action (e.g., 'Request itemized bill', 'Escalate to SIU', 'Approve for payment') so that adjusters can process claims faster and with greater consistency.

This is a Phase 1 implementation covering read-only summarization and NBA suggestions. Agentforce autonomous action execution (e.g., auto-approving claims) is deferred to Phase 2.

## Acceptance Criteria

Given an adjuster opens a Claim__c record with Status__c IN ('In Adjudication', 'Pending Review — OON', 'Pending Review — Fee Schedule'),
When the Agentforce Claims Assistant panel loads,
Then a natural-language summary is generated covering Claim__c.Date_of_Loss__c, total billed amount, number of Claim_Line__c records, provider name, and current status.

Given the AI summary is generated,
When Claim__c.SIU_Flag__c = true,
Then the summary prominently displays a red 'SIU Review Required' badge and the suggested next best action is 'Escalate to Special Investigations Unit'.

Given the AI analyzes Claim_Line__c records,
When duplicate CPT_Code__c values appear across lines with the same Service_Date__c,
Then the assistant highlights the duplicate lines and suggests 'Request itemized bill from provider before approving'.

Given the next best action suggestion is displayed,
When the adjuster clicks 'Accept Suggestion',
Then a Task is created on the Claim__c record with Subject = the suggested action text, Status = 'Not Started', and OwnerId = the adjuster's user Id.

Given the Agentforce panel is active,
When the adjuster types a free-text question such as 'Has this provider had claims denied in the last 6 months?',
Then Agentforce queries Claim__c records linked to the same Provider__c with Claim__c.Status__c = 'Denied' within the last 180 days and returns a plain-English response with a count and list of Claim__c.Name values.

-Given adjuster accepts an NBA of 'Approve for Payment', When confirmed, Then Claim__c.Status__c auto-transitions to 'Approved' (deprecated — autonomous status change deferred to Phase 2 per compliance review)-

## Solution

Configure an Agentforce Agent (Type: Claims_Assistant) in Salesforce AI Studio with access to the Claim__c, Claim_Line__c, and Provider__c objects via Agent Topics and Actions. Define a custom Agent Topic 'Claim_Summary' with a prompt template referencing merged fields from Claim__c and child Claim_Line__c. Build a custom Apex Agent Action 'ClaimAnomalyDetector.detectAnomalies(claimId)' returning a structured list of anomaly objects consumed by the prompt. Configure Next Best Action Strategy using Einstein NBA with a Decision Table (Claim_NBA_Strategy__dt) mapping Status__c + SIU_Flag__c + duplicate CPT flag to recommended actions. Embed the Agentforce panel as a Dynamic Form component on the Claim__c Lightning record page, visible only to users with the 'Claims_Adjuster' Permission Set Group. Log all AI interactions to a custom object AI_Interaction_Log__c for auditability.

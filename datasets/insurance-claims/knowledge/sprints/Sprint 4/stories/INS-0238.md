## Description

As a Claims Operations Manager, I need the system to automatically validate eligibility, coverage limits, and policy status when a Claim__c record transitions from 'FNOL Received' to 'Under Review', so that ineligible or out-of-coverage claims are flagged immediately, reducing manual triage time and ensuring compliant adjudication. Extends INS-0237.

## Acceptance Criteria

Given a Claim__c.Status__c transitions to 'Under Review',
When the Record-Triggered Flow fires,
Then the system checks Policy__c.Policy_Status__c = 'Active'; if not active, Claim__c.Adjudication_Flag__c is set to 'Ineligible - Policy Lapsed' and Status__c is set to 'Pending Review'.

Given the policy is active,
When adjudication rules execute,
Then the system validates that Claim__c.Date_of_Service__c falls within Policy__c.Effective_Date__c and Policy__c.Expiration_Date__c; if outside range, Claim__c.Adjudication_Flag__c = 'Ineligible - Out of Coverage Period'.

Given the claim passes date validation,
When coverage limit checks run,
Then each Claim_Line__c.Billed_Amount__c is compared against the applicable Fee_Schedule__c.Allowed_Amount__c for the matching CPT code; Claim_Line__c.Allowed_Amount__c is populated accordingly.

Given a Claim_Line__c CPT code has no matching Fee_Schedule__c record,
When the rule engine processes that line,
Then Claim_Line__c.Adjudication_Status__c = 'Pending - No Fee Schedule Match' and a task is created and assigned to the Claims Operations Manager queue.

Given all Claim_Line__c records pass validation,
When the adjudication batch completes,
Then Claim__c.Status__c = 'Adjudicated - Pending Payment' and Claim__c.Adjudicated_Date__c is stamped with the current datetime.

Given any adjudication flag is set,
When the claim record is saved,
Then an email notification is sent to the assigned adjuster using the 'Claim Flagged for Review' email template.

## Solution

Implement a Record-Triggered Flow on Claim__c (trigger: Status__c changes to 'Under Review') that invokes an Apex action class (ClaimAdjudicationService.cls). The Apex class performs bulkified SOQL against Policy__c, Claim_Line__c, and Fee_Schedule__c. Fee schedule matching logic uses CPT_Code__c + LOB__c + Effective_Date__c as composite key. Adjudication flags stored in a custom multi-select picklist Adjudication_Flag__c on Claim__c. Task creation handled via Apex insert. Email alert via Salesforce Email Alert action within the Flow. Unit test coverage target: 90%+.

## Description

As a Claims Operations Manager, I need an automated adjudication rule engine that evaluates incoming Claim__c records and auto-adjudicates 'clean' claims (no missing fields, in-network provider, billed amount ≤ $500) so that straight-through processing is achieved for low-complexity claims and manual review queues are reduced.

A clean claim is defined as: all required Claim_Line__c fields populated (CPT code, ICD-10, NPI), the servicing Provider__c has an Active Provider_Network_Agreement__c, and the total billed amount does not exceed $500. Fee schedule reimbursement is looked up from Fee_Schedule__c by CPT code and LOB.

## Acceptance Criteria

Given a Claim__c record is created or updated with Status__c = 'FNOL Received',
When the adjudication engine evaluates the record,
Then it checks that all Claim_Line__c records have CPT_Code__c, ICD10_Code__c, and Rendering_NPI__c populated; if any are missing, Claim__c.Status__c is set to 'Pended' and Claim__c.Pend_Reason__c = 'Incomplete Claim Data'.

Given all Claim_Line__c fields are populated,
When the engine validates provider network status,
Then it queries Provider_Network_Agreement__c where PNA.Provider__c = Claim__c.Rendering_Provider__c and PNA.Status__c = 'Active' and PNA.Effective_Date__c <= Date_of_Loss__c and PNA.Termination_Date__c >= Date_of_Loss__c; if no active PNA is found, Claim__c.Status__c = 'Pended' and Pend_Reason__c = 'Out-of-Network Provider'.

Given the provider is confirmed in-network,
When the engine evaluates the billed amount,
Then it sums all Claim_Line__c.Billed_Amount__c; if total > $500, Claim__c.Status__c = 'Assigned for Review' and the claim is routed to the Claims Operations Manager queue.

Given total billed amount ≤ $500 and provider is in-network,
When the engine performs fee schedule lookup,
Then for each Claim_Line__c it queries Fee_Schedule__c where Fee_Schedule__c.CPT_Code__c = Claim_Line__c.CPT_Code__c and Fee_Schedule__c.LOB__c = Claim__c.LOB__c and retrieves Fee_Schedule__c.Allowed_Amount__c, setting Claim_Line__c.Allowed_Amount__c accordingly.

Given fee schedule amounts are applied to all Claim_Line__c records,
When adjudication completes successfully,
Then Claim__c.Status__c = 'Adjudicated', Claim__c.Adjudication_Date__c = TODAY(), and Claim__c.Adjudication_Method__c = 'Auto'.

Given Claim__c.Status__c transitions to 'Adjudicated',
When the record is saved,
Then a Record-Triggered Flow creates a Payment__c record in 'Pending Approval' status with Payment__c.Amount__c = sum of all Claim_Line__c.Allowed_Amount__c.

-Given the claim LOB = 'Dental', auto-adjudication applies the same rules- (deprecated — Dental LOB deferred to Phase 2 per INS-0289).

## Solution

Implement an Apex class AdjudicationEngine.cls invoked by a Record-Triggered Flow (After Insert/Update on Claim__c where Status__c = 'FNOL Received'). The Apex class performs bulkified SOQL queries for Claim_Line__c, Provider_Network_Agreement__c, and Fee_Schedule__c. Use Custom Metadata Type (Adjudication_Rule__mdt) to store configurable thresholds (e.g., auto-adjudication ceiling amount) to avoid hardcoding. On completion, update Claim__c and Claim_Line__c records via a single DML operation. A separate Record-Triggered Flow on Claim__c (After Update, Status__c changed to 'Adjudicated') creates the Payment__c record. Unit test coverage target: ≥ 85%.

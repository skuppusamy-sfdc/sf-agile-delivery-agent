## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) Lightning Record Page that surfaces the provider's active Provider_Network_Agreement__c (PNA), current Fee_Schedule__c details, credentialing status from Credentialing_Case__c, and a summary of open Claim__c records, so that I can quickly assess a provider's network standing and claim activity without navigating across multiple related lists.

This story supports the broader Provider Data Maintenance theme and complements the Provider Network Contracting epic work completed in Sprint 2.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the FlexCard must display the following sections populated via DataRaptor Turbo Extract:

**Section 1 — Network Status:**
  - Provider_Network_Agreement__c.Status__c (badge: green = 'Active', yellow = 'Pending Renewal', red = 'Terminated')
  - Provider_Network_Agreement__c.Effective_Date__c
  - Provider_Network_Agreement__c.Expiration_Date__c
  - Provider_Network_Agreement__c.Network_Tier__c
  - If no active PNA exists, display: 'No Active Network Agreement — Contact Network Contracting'

**Section 2 — Fee Schedule Summary:**
  - Fee_Schedule__c.Schedule_Name__c
  - Fee_Schedule__c.Effective_Date__c
  - Count of active CPT codes on the fee schedule (aggregated)
  - -Fee_Schedule__c.Base_Rate__c displayed as currency- (removed per compliance review — rates not to be surfaced in UI without audit trail)

**Section 3 — Credentialing Status:**
  - Most recent Credentialing_Case__c.Status__c for the provider
  - Credentialing_Case__c.Expiration_Date__c with a visual alert if expiring within 90 days (Credentialing_Case__c.Expiration_Date__c <= TODAY() + 90)
  - Credentialing_Case__c.Primary_Specialty__c

Given the Provider Relations Specialist clicks the 'Open Claims' section of the FlexCard,
When the expandable child card renders,
Then it must display a list of up to 10 most recent Claim__c records where Claim__c.Billing_Provider__c = Provider__c.Id AND Claim__c.Status__c NOT IN ('Closed', 'Denied') showing:
  - Claim__c.Claim_Number__c (hyperlinked to Claim__c record)
  - Claim__c.Date_of_Loss__c
  - Claim__c.Status__c
  - Claim__c.Total_Billed_Amount__c
  - Claim__c.Total_Allowed_Amount__c

Given the FlexCard is viewed by a user without the 'Provider Network Specialist' or 'Claims Operations Manager' Permission Set Group,
When the FlexCard loads,
Then the Fee Schedule section and Open Claims financial columns (Total_Billed_Amount__c, Total_Allowed_Amount__c) must be hidden, showing only provider network status and credentialing information.

Given the FlexCard data is loaded,
When the underlying DataRaptor query executes,
Then the total page load time for the FlexCard must not exceed 3 seconds in sandbox under standard data volumes (tested with up to 500 Claim__c records per provider).

## Solution

Build a multi-section OmniStudio FlexCard (FlexCard Name: Provider360_Summary) with child FlexCards for each section. Use three DataRaptor Turbo Extract actions: (1) DR_Get_Active_PNA — queries Provider_Network_Agreement__c filtered by Provider__c lookup and Status__c = 'Active', (2) DR_Get_Fee_Schedule_Summary — queries Fee_Schedule__c joined to PNA with aggregate count of active CPT lines, (3) DR_Get_Open_Claims — queries Claim__c with LIMIT 10 ORDER BY Date_of_Loss__c DESC. Implement field-level visibility on the Fee Schedule and financial columns using FlexCard Conditional Visibility rules bound to a custom User Permission (canViewProviderFinancials) checked via an Integration Procedure that queries the running user's Permission Set Group assignments. The credentialing expiry alert uses a FlexCard State with a conditional CSS class applied when Credentialing_Case__c.Expiration_Date__c is within 90 days, calculated in the DataRaptor using a formula field. Deploy the FlexCard as an LWC on the Provider__c (Person Account) Lightning Record Page via App Builder with audience filtering to show only for Provider Relations Specialist and Claims Operations profiles. Cache DataRaptor results using OmniStudio session cache with a 5-minute TTL to meet the 3-second load SLA.

## Description

**Bug Report**

**Summary:** The duplicate claim detection logic introduced in INS-0222 is incorrectly flagging legitimate claims as duplicates when a policyholder has multiple active LOB coverages (e.g., both Auto and Health) on the same Policy__c, and two claims are filed on the same date for different LOBs.

**Steps to Reproduce:**
1. Identify or create a Policy__c record with two active Coverage_Line__c records — one for LOB = 'Auto' and one for LOB = 'Health' (e.g., test policy ACM-POL-00078 in Staging).
2. Submit a Claim__c via FNOL OmniScript with LOB = 'Auto' and Date_of_Loss__c = TODAY.
3. Submit a second Claim__c via FNOL OmniScript with LOB = 'Health' and Date_of_Loss__c = TODAY for the same policy.
4. Observe Claim__c.Duplicate_Flag__c on the second submitted claim.

**Expected Behavior:** The second claim (LOB = 'Health') should NOT be flagged as a duplicate because it is a distinct LOB from the first claim (LOB = 'Auto') on the same date.

**Actual Behavior:** The duplicate detection SOQL query in ClaimAdjudicationHandler does not include Claim__c.LOB__c in its WHERE clause filter. As a result, any two claims on the same policy within ±1 day are matched regardless of LOB, causing Claim__c.Duplicate_Flag__c = TRUE on the second legitimate claim and generating a spurious Task for manual review.

**Root Cause (suspected):** Missing AND LOB__c = :currentClaim.LOB__c predicate in the duplicate detection SOQL query in ClaimAdjudicationHandler.duplicateCheck() method (line ~87).

**Environment:** Staging org. Reproducible 100% with multi-LOB policies.

## Acceptance Criteria

as per Description

## Solution

Update the SOQL query in ClaimAdjudicationHandler.duplicateCheck() to include AND LOB__c = :currentClaim.LOB__c in the WHERE clause so that duplicate matching is scoped to the same LOB. Also add AND Id != :currentClaim.Id to prevent the current record from matching itself during update triggers. Update the corresponding Apex test class (ClaimAdjudicationHandlerTest) to add a test method covering multi-LOB same-date scenario asserting Duplicate_Flag__c = FALSE on the second claim. Deploy fix through standard sandbox-to-production pipeline. Regression test INS-0222 acceptance criteria to confirm existing duplicate detection for same-LOB same-date claims still functions correctly.

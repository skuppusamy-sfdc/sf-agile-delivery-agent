## Description

As a Claims Operations Manager, I need a Claim 360 FlexCard embedded on the Claim__c record page so that adjusters can see all relevant claim, policy, provider, and payment information in one place without navigating to multiple related lists, improving adjudication speed and accuracy.

## Acceptance Criteria

Given an adjuster opens a Claim__c record page,
When the Claim 360 FlexCard loads,
Then it displays Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.Date_of_Loss__c, Claim__c.Total_Billed_Amount__c, and Claim__c.Total_Allowed_Amount__c in the header section.

Given the Claim__c is linked to a Policy__c,
When the FlexCard renders the Policy panel,
Then it shows Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Group_Name__c, and a color-coded badge for Policy__c.Status__c (green = Active, red = Lapsed, yellow = Pending Renewal).

Given the Claim__c has an associated Provider__c (Person Account),
When the Provider panel renders,
Then it displays Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Specialty__c, and a network status badge derived from the active Provider_Network_Agreement__c.Network_Status__c.

Given the Claim__c has one or more Payment__c child records,
When the Payments panel renders,
Then each Payment__c.Payment_Amount__c, Payment__c.Payment_Date__c, and Payment__c.Payment_Status__c are listed in a datatable, sorted descending by Payment__c.Payment_Date__c.

-Given the FlexCard includes a 'Reopen Claim' action button visible to all profiles- (deprecated: action visibility now controlled by Permission Set Group per INS-0271)

## Solution

Build an OmniStudio FlexCard (Name: Claim360Card) with four child cards (ClaimHeader, PolicyPanel, ProviderPanel, PaymentsPanel). Each child card uses a dedicated DataRaptor Extract as its data source. The ProviderPanel DataRaptor performs a multi-object query joining Provider__c and Provider_Network_Agreement__c filtered by PNA.Status__c = 'Active'. Embed the FlexCard on the Claim__c Lightning Record Page via an App Builder drag-and-drop. Use conditional rendering states on the network status badge (Active = green, Terminated = red, Pending = amber). Assign FlexCard visibility via Permission Set Group 'Claims_Adjuster_PSG'. DataRaptors are version-controlled in the OmniStudio deployment package.

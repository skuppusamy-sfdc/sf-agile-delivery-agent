## Description

As a Claims Operations Manager, I need Claim_Line__c records to be automatically validated against the active Fee_Schedule__c when they are saved so that billed CPT codes are confirmed as covered services with accurate allowed amounts populated before adjudication proceeds, reducing manual re-work and overpayment risk. This story extends INS-0247 by adding line-level adjudication inputs to the eligibility-validated claim.

## Acceptance Criteria

Given an adjuster creates or updates a Claim_Line__c record on a Claim__c with Claim__c.Eligibility_Validated__c = true,
When the record is saved,
Then the system identifies the applicable Fee_Schedule__c by matching Claim__c.LOB__c and Claim__c.Network_Flag__c to Fee_Schedule__c.LOB__c and Fee_Schedule__c.Network_Type__c where Fee_Schedule__c.Effective_Date__c <= Claim__c.Date_of_Service__c and Fee_Schedule__c.Expiration_Date__c >= Claim__c.Date_of_Service__c.

Given the applicable Fee_Schedule__c is identified,
When the Claim_Line__c.CPT_Code__c is evaluated,
Then the system checks for a matching Fee_Schedule_Line__c record where Fee_Schedule_Line__c.CPT_Code__c = Claim_Line__c.CPT_Code__c and Fee_Schedule_Line__c.Fee_Schedule__c = the identified Fee_Schedule__c.Id.

Given a matching Fee_Schedule_Line__c is found,
When the CPT code is valid,
Then Claim_Line__c.Allowed_Amount__c is populated with Fee_Schedule_Line__c.Allowed_Amount__c, Claim_Line__c.CPT_Valid__c (checkbox) is set to true, and Claim_Line__c.Status__c = 'Validated'.

Given no matching Fee_Schedule_Line__c is found for the entered CPT code,
When the validation fails,
Then Claim_Line__c.CPT_Valid__c is set to false, Claim_Line__c.Status__c = 'Pended — Invalid CPT', and an error message is displayed inline: 'CPT Code {CPT_Code__c} is not covered under the applicable fee schedule. Please verify the code or escalate to SIU if fraud is suspected.'

Given Claim_Line__c.Billed_Amount__c is greater than Fee_Schedule_Line__c.Allowed_Amount__c by more than 20%,
When the line is saved,
Then Claim_Line__c.Variance_Flag__c is set to true and a warning notification is sent to the assigned adjuster via Salesforce In-App Notification.

-Given Claim_Line__c.Modifier__c is populated, modifier-specific fee schedule lookups are performed- (deprecated — modifier adjudication deferred to Phase 2).

Given all Claim_Line__c records on a Claim__c have Claim_Line__c.Status__c = 'Validated',
When the last line is validated,
Then Claim__c.All_Lines_Validated__c (checkbox) is automatically set to true via a Roll-Up Summary or Record-Triggered Flow.

## Solution

Implement an Apex Trigger (ClaimLineTrigger) on Claim_Line__c for 'before insert, before update' events, delegating to FeeScheduleValidationService.cls. The service performs a two-step SOQL: first to identify Fee_Schedule__c by LOB, network type, and date range; second to find the matching Fee_Schedule_Line__c by CPT code. All allowed amounts are written back in the before-trigger context. A Record-Triggered Flow on Claim_Line__c (after update) handles the roll-up to Claim__c.All_Lines_Validated__c. Variance threshold (20%) stored in Custom Metadata Type (Adjudication_Rule__mdt.CPT_Variance_Threshold__c). In-app notification uses Salesforce Custom Notification via Apex NotificationBuilder API.

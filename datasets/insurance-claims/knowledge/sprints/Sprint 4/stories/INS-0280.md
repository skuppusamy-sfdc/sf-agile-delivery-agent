# INS-0280: Provider Network Status Validation at FNOL — Block Claims for Out-of-Network Providers Without Prior Auth

## Description

As a Claims Operations Manager, I need the FNOL intake process to validate the rendering provider's network status against their active Provider_Network_Agreement__c so that out-of-network claims without a prior authorization are flagged or blocked at intake, reducing downstream payment errors and compliance risk.

The validation should check PNA effective dates, network tier, and whether a valid prior authorization number exists on the claim before allowing it to advance past FNOL status.

## Acceptance Criteria

Given a Claim__c is submitted via FNOL with a rendering Provider__c selected,
When the system checks Provider_Network_Agreement__c records for that provider,
Then it queries PNA records where PNA.Provider__c = Claim__c.Rendering_Provider__c and PNA.Effective_Date__c <= Claim__c.Date_of_Service__c <= PNA.Termination_Date__c.

Given a matching active PNA is found with PNA.Network_Tier__c = 'In-Network',
When the validation runs,
Then Claim__c.Provider_Network_Status__c = 'In-Network' and processing continues normally.

Given no active PNA is found or PNA.Network_Tier__c = 'Out-of-Network',
When the validation runs and Claim__c.Prior_Auth_Number__c is blank,
Then Claim__c.Status__c is set to 'Pending Authorization' and a Task is created assigned to the Claims Operations Manager with Subject 'Prior Auth Required — Out-of-Network Provider'.

Given no active PNA is found and Claim__c.Prior_Auth_Number__c is populated,
When the validation runs,
Then Claim__c.Provider_Network_Status__c = 'Out-of-Network — Auth on File' and a warning Chatter post is made on the Claim__c record tagging the assigned adjuster.

Given PNA.Network_Tier__c = 'Pending Credentialing',
When the validation runs,
Then Claim__c.Status__c = 'Pending Credentialing Review' and an automatic notification is sent to the Credentialing Specialist via a custom notification type.

Given the provider NPI does not exist in the system as a Provider__c record,
When the FNOL is submitted,
Then a blocking error 'Provider NPI not found — please verify NPI or create a new provider record' prevents Claim__c creation.

## Solution

Implement a Record-Triggered Flow on Claim__c (trigger: after insert, entry condition: Status__c = 'FNOL Received'). The flow calls an Apex invocable method (ProviderNetworkValidator.validate()) that queries Provider_Network_Agreement__c with the date range and network tier filter. The method returns a validation result enum (IN_NETWORK, OUT_OF_NETWORK_NO_AUTH, OUT_OF_NETWORK_AUTH_ON_FILE, PENDING_CREDENTIALING, NPI_NOT_FOUND). The flow branches on the result and updates Claim__c.Provider_Network_Status__c, creates Tasks, and fires Custom Notifications using the Notification Builder API. Chatter posts use ConnectApi in an @future method to avoid mixed DML. All logic is covered by Apex unit tests with mock PNA data via a Test Data Factory.

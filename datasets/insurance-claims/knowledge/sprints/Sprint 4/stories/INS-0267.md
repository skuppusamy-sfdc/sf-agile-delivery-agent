## Description

As a Claims Operations Manager, I need an Agentforce-powered Next Best Action (NBA) panel embedded on the Claim__c record page that surfaces AI-generated recommendations for resolving pended claims so that adjusters can act faster on stalled claims, reduce average handle time, and improve SLA compliance. This story covers the initial NBA configuration for the top 3 pend reason codes: 'Pended — ICD10 Review', 'Pended — NPI Review', and 'Pended — COB Required'.

## Acceptance Criteria

Given a Claim__c record with Status__c = 'Pended — ICD10 Review' is open by an adjuster,
When the Agentforce NBA panel loads on the record page,
Then the panel displays a recommendation card: 'Verify ICD-10 code {Claim_Line__c.ICD10_Code__c} against CMS reference. Suggested action: Update ICD-10 or escalate to clinical review.' with a one-click 'Open ICD-10 Lookup' action button.

Given a Claim__c record with Status__c = 'Pended — NPI Review',
When the NBA panel loads,
Then the panel displays: 'NPI {Claim__c.Rendering_Provider_NPI__c} could not be verified. Suggested action: Contact provider to confirm NPI or update Provider__c record.' with a 'Create Provider Outreach Task' action button that pre-populates a Task with Subject = 'NPI Discrepancy — Provider Outreach' and assigns to the current user.

Given a Claim__c record with Status__c = 'Pended — COB Required',
When the NBA panel loads,
Then the panel displays a COB resolution checklist recommendation and a 'Launch COB OmniScript' action that opens the COB data-gathering flow in a modal.

Given the adjuster dismisses a recommendation,
When the dismiss action is clicked,
Then the recommendation is logged in a NBA_Feedback__c record with fields: Claim__c lookup, Recommendation_Type__c, Action_Taken__c = 'Dismissed', and Dismissed_By__c = current user, for model feedback purposes.

Given the NBA panel is visible,
When no pend-specific recommendation applies (claim is not in a pended status),
Then the panel displays a default message: 'No AI recommendations at this time. Claim is in {Claim__c.Status__c} status.'

## Solution

Configure an Einstein Next Best Action strategy using Salesforce Agentforce (formerly Einstein for Service). Define a Recommendation object strategy (ClaimsPendResolution) with three Recommendation records corresponding to the three pend codes. Use a Flow-based recommendation strategy that queries Claim__c.Status__c to filter applicable recommendations. Embed the 'Next Best Action' standard Lightning component on the Claim__c Lightning Record Page, scoped to the 'Claims Adjuster' Permission Set Group. Action buttons are implemented as Quick Actions on Claim__c: one invokes a Screen Flow for ICD-10 lookup, one creates a Task via an Autolaunched Flow, and one launches the COB OmniScript via a Lightning Navigation action. NBA_Feedback__c is written via a Record-Triggered Flow on recommendation dismissal event. Model refinement pipeline to be defined in Phase 2.

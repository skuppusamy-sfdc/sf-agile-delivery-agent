## Description

As a Claims Operations Manager, I need Agentforce to analyze pended Claim__c records and surface a suggested denial reason with supporting rationale so that adjudicators can make faster, more consistent decisions on pended claims without having to manually cross-reference policy terms, fee schedules, and ICD-10/CPT code mappings.

## Acceptance Criteria

Given a Claim__c record has Status__c = 'Pended' and at least one Claim_Line__c with Status__c = 'Pended – CPT Not Found' or 'Pended – Coverage Mismatch',
When an adjudicator opens the Claim__c record page,
Then the Agentforce sidebar panel displays an AI-generated suggestion card with: recommended Denial_Reason__c value, confidence score (High/Medium/Low), and a plain-language rationale referencing specific Claim_Line__c.CPT_Code__c and Policy__c coverage terms.

Given the Agentforce suggestion is displayed,
When the adjudicator clicks 'Accept Suggestion',
Then Claim__c.Denial_Reason__c is populated with the suggested value, Claim__c.Status__c is updated to 'Denied', and an audit entry is written to Claim__c.Adjudication_Notes__c noting 'Denial reason accepted via Agentforce recommendation'.

Given the adjudicator clicks 'Reject Suggestion',
When the rejection is logged,
Then a Feedback__c record is created linked to the Claim__c capturing the rejected suggestion text and adjudicator's free-text override reason for model retraining purposes.

Given the confidence score is 'Low',
When the suggestion card renders,
Then a warning banner is displayed: 'Low confidence — manual review strongly recommended' and the 'Accept Suggestion' button is disabled.

-Given the Agentforce panel, include automated claim closure upon acceptance without adjudicator confirmation.- (removed — compliance requires human-in-the-loop confirmation step per Compliance Officer review)

## Solution

Configure an Agentforce Agent (Einstein Copilot) with a custom Agent Action: 'Suggest Claim Denial Reason'. The action invokes an Apex invocable method (ClaimDenialRecommendationAction) that assembles a prompt context string from Claim__c, Claim_Line__c, Policy__c, and Fee_Schedule__c data and calls the Einstein LLM via the Connect API (generative AI endpoint). Response is parsed for denial reason and confidence. The Agentforce panel is surfaced as a custom LWC (claimAgentforcePanel) embedded on the Claim__c Lightning Record Page via Dynamic Forms. Accept/Reject actions trigger a Screen Flow for confirmation before DML. Feedback__c records are created via an Apex after-action handler. Model prompt template stored as a Flex Template in Prompt Builder.

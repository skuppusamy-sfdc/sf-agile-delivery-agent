## Description

As a Claims Operations Manager, I need a guided FNOL intake OmniScript so that front-line staff can capture all required initial claim details for Auto and Medical LOBs in a single, validated workflow without switching between multiple screens.

This story covers the creation of the OmniScript wizard that creates a Claim__c record, associates it to the relevant Policy__c, and captures the incident date, loss description, and initial contact information for the claimant. Extends INS-0246 (Policy lookup DataRaptor).

## Acceptance Criteria

Given a logged-in Claims Operations Manager opens the FNOL OmniScript,
When they search by Policy__c.Policy_Number__c or insured name,
Then the OmniScript populates Policy__c.Status__c, Policy__c.LOB__c, and Policy__c.Effective_Date__c in read-only display fields before proceeding to Step 2.

Given the policy is retrieved successfully,
When the user completes Step 2 (Incident Details),
Then Claim__c.Date_of_Loss__c must be on or after Policy__c.Effective_Date__c and on or before Policy__c.Expiration_Date__c, otherwise a blocking validation message 'Loss date falls outside active policy period' is displayed.

Given all required fields are completed,
When the user clicks 'Submit FNOL',
Then a Claim__c record is created with Claim__c.Status__c = 'FNOL Received', Claim__c.FNOL_Submitted_Date__c = TODAY(), and Claim__c.Claim_Source__c = 'Phone' or 'Web' based on channel selection.

Given the Claim__c record is created,
When the OmniScript completes,
Then a Platform Event (FNOL_Submitted__e) is fired with Claim__c.Id and Policy__c.Id payload fields so that downstream adjudication rules are triggered.

-Given the LOB is Auto, the OmniScript shall collect vehicle VIN on Step 1- (deprecated: VIN capture moved to INS-0279)

Given Claim__c.LOB__c = 'Medical',
When the FNOL is submitted,
Then Claim__c.Assigned_Adjuster__c is auto-populated via the Assignment Round Robin rule set defined in the Assignment Coordinator configuration.

## Solution

Build a 4-step OmniScript (Type: FNOL, Sub-Type: Intake) using OmniStudio. Step 1 uses a DataRaptor Extract to query Policy__c by Policy_Number__c or Account name. Step 2 captures incident details with a date range validation formula element. Step 3 presents a summary FlexCard preview. Step 4 uses a DataRaptor Transform + Integration Procedure to upsert Claim__c and fire a Platform Event (FNOL_Submitted__e) via Apex Action. A Record-Triggered Flow on Claim__c (trigger: after insert, Status__c = 'FNOL Received') handles adjuster auto-assignment using a custom Round Robin Apex invocable method. Permission Set Group 'Claims_Intake_PSG' grants OmniScript runtime access.

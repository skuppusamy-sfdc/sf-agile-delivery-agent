## Description

As a Claims Operations Manager, I need the system to automatically apply benefit limits, annual deductibles, and co-insurance percentages to each Claim_Line__c record during adjudication so that the calculated allowed amount and member liability are accurate and consistent with the active Fee_Schedule__c and the insured's Group_Service_Agreement__c (GSA).

This story covers Phase 1 basic rules only: out-of-network exclusions, annual deductible application, and co-insurance split. COB (Coordination of Benefits) logic is deferred to Phase 2.

## Acceptance Criteria

Given a Claim__c record transitions to Status__c = 'In Adjudication',
When the adjudication engine processes each child Claim_Line__c,
Then the engine looks up the active Fee_Schedule__c matching Claim_Line__c.CPT_Code__c and Claim_Line__c.Service_Date__c to populate Claim_Line__c.Allowed_Amount__c.

Given Claim_Line__c.Allowed_Amount__c is resolved,
When the insured's remaining annual deductible (Policy__c.Remaining_Deductible__c) is greater than zero,
Then Claim_Line__c.Deductible_Applied__c is set to MIN(Claim_Line__c.Allowed_Amount__c, Policy__c.Remaining_Deductible__c) and Policy__c.Remaining_Deductible__c is decremented accordingly.

Given deductible is satisfied,
When co-insurance percentage is retrieved from GSA__c.Coinsurance_Pct__c,
Then Claim_Line__c.Plan_Paid_Amount__c = (Claim_Line__c.Allowed_Amount__c - Claim_Line__c.Deductible_Applied__c) * GSA__c.Coinsurance_Pct__c and Claim_Line__c.Member_Liability__c = the remainder.

Given Claim_Line__c.Provider__c is not listed in any active Provider_Network_Agreement__c,
When the out-of-network check runs,
Then Claim_Line__c.Network_Status__c = 'Out of Network', Claim_Line__c.Plan_Paid_Amount__c = 0.00, and Claim__c.Status__c is set to 'Pending Review — OON'.

Given all Claim_Line__c records on a Claim__c are adjudicated without flags,
When the final line is processed,
Then Claim__c.Status__c transitions to 'Adjudicated' and Claim__c.Total_Plan_Paid__c is the SUM of all child Claim_Line__c.Plan_Paid_Amount__c values.

-Given CPT code is not found on Fee_Schedule__c, When lookup fails, Then Claim_Line__c.Allowed_Amount__c defaults to Claim_Line__c.Billed_Amount__c (deprecated — zero-pay with manual review flag required instead)-

Given CPT code is not found on Fee_Schedule__c,
When lookup fails,
Then Claim_Line__c.Plan_Paid_Amount__c = 0.00, Claim_Line__c.Denial_Reason__c = 'CPT Not on Fee Schedule', and Claim__c.Status__c = 'Pending Review — Fee Schedule'.

## Solution

Develop an Apex invocable method 'AdjudicationEngine.processClaimLines(claimId)' called from a Record-Triggered Flow on Claim__c (Before Update, Status__c changes to 'In Adjudication'). The Apex class queries all Claim_Line__c children, performs SOQL lookup on Fee_Schedule__c with CPT_Code__c and effective date range, evaluates PNA linkage for network status, and applies deductible/co-insurance arithmetic. Use a Custom Metadata Type (Adjudication_Rule__mdt) to store co-insurance percentages and deductible logic per LOB, keeping business rules configurable without code deployment. Update Policy__c.Remaining_Deductible__c via a separate DML in a Unit of Work pattern to respect governor limits. Write Apex test class with 90%+ coverage covering OON, deductible exhaustion, and CPT-not-found scenarios.

## Description

As a Claims Operations Manager, I need the system to automatically adjudicate low-complexity medical claims where the total billed amount is under $500 and all Claim_Line__c CPT codes are on the active Fee_Schedule__c so that routine claims are processed without manual intervention, reducing adjuster workload by an estimated 30%.

Adjudication rules for Sprint 4 are scoped to Medical LOB only. Auto-adjudication must validate provider network status, fee schedule match, and policy active status before approving payment.

## Acceptance Criteria

Given a Claim__c record with Claim__c.LOB__c = 'Medical', Claim__c.Status__c = 'FNOL Received', and Claim__c.Total_Billed_Amount__c < 500,
When the adjudication rules engine batch job executes,
Then the engine evaluates all associated Claim_Line__c records for CPT code validity against Fee_Schedule__c.CPT_Code__c where Fee_Schedule__c.Effective_Date__c <= Claim__c.Date_of_Service__c and Fee_Schedule__c.Expiration_Date__c >= Claim__c.Date_of_Service__c.

Given all Claim_Line__c CPT codes match active Fee_Schedule__c entries,
When the policy status check passes (Policy__c.Status__c = 'Active'),
Then each Claim_Line__c.Allowed_Amount__c is set to Fee_Schedule__c.Allowed_Amount__c, Claim_Line__c.Adjudication_Status__c = 'Approved', and Claim__c.Status__c is updated to 'Adjudicated'.

Given the rendering Provider__c (NPI) is checked against Provider_Network_Agreement__c,
When Provider_Network_Agreement__c.Status__c != 'Active' or no PNA record exists for the provider,
Then Claim__c.Status__c = 'Pended' and Claim__c.Pend_Reason__c = 'Out of Network Provider' and the claim is excluded from auto-adjudication.

Given any Claim_Line__c CPT code is NOT found on an active Fee_Schedule__c,
When the engine processes that claim,
Then Claim__c.Status__c = 'Pended', Claim__c.Pend_Reason__c = 'Fee Schedule Mismatch', and a task is created and assigned to the Claims Operations queue.

Given Claim__c.Total_Billed_Amount__c >= 500,
When the engine evaluates the claim,
Then the claim is bypassed for auto-adjudication and Claim__c.Status__c remains 'FNOL Received' with Claim__c.Manual_Review_Flag__c = TRUE.

Given auto-adjudication completes successfully,
When Claim__c.Status__c = 'Adjudicated',
Then a Payment__c record is created with Payment__c.Amount__c = SUM(Claim_Line__c.Allowed_Amount__c), Payment__c.Status__c = 'Pending Disbursement', and Payment__c.Claim__c = Claim__c.Id.

Given the adjudication batch runs,
When 100 or more claims are processed in a single execution,
Then all DML operations are bulkified and no governor limit exceptions are thrown (validated via Apex test class with 200-record dataset).

## Solution

Implement an Apex Batch class (AdjudicationRulesEngineBatch) implementing Database.Batchable<SObject> and Database.Stateful. Query scope: Claim__c WHERE Status__c = 'FNOL Received' AND LOB__c = 'Medical' AND Total_Billed_Amount__c < 500. Within execute(), call an AdjudicationService Apex class that encapsulates: (1) Fee schedule lookup via SOQL on Fee_Schedule__c, (2) PNA status check on Provider_Network_Agreement__c, (3) Policy status validation. Use a single DML update list for Claim__c and Claim_Line__c records per batch chunk. Create Payment__c records in finish() method via a separate DML operation. Schedule via Apex Scheduler to run nightly at 2:00 AM. Minimum 85% Apex code coverage required. Add Custom Metadata Type (Adjudication_Rule__mdt) for threshold configuration ($500 limit) to avoid hardcoding.

## Description

As a Claims Operations Manager, I need a guided FNOL intake OmniScript so that front-line staff and self-service policyholders can capture all initial claim details accurately and consistently across Auto and Medical LOBs, reducing manual data entry errors and accelerating claim creation in Salesforce.

The OmniScript must create a Claim__c record linked to the relevant Policy__c and pre-populate claimant data from the matched Person Account (Provider__c or policyholder contact).

## Acceptance Criteria

Given a logged-in Claims Operations user on the FNOL intake screen,
When they search by Policy__c.Policy_Number__c,
Then the OmniScript auto-populates Policy__c.Policyholder__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, and Policy__c.Expiration_Date__c into the intake form read-only fields.

Given the user selects LOB = 'Medical',
When the LOB step loads,
Then ICD-10 diagnosis code fields (Claim__c.Primary_Diagnosis_Code__c, Claim__c.Secondary_Diagnosis_Code__c) are displayed and CPT code entry is enabled on Claim_Line__c records.

Given the user selects LOB = 'Auto',
When the LOB step loads,
Then vehicle identification fields (Claim__c.Vehicle_VIN__c, Claim__c.Accident_Date__c, Claim__c.At_Fault_Party__c) are shown and ICD-10 fields are hidden.

Given all required FNOL fields are completed,
When the user clicks 'Submit FNOL',
Then a Claim__c record is created with Claim__c.Status__c = 'FNOL Received', Claim__c.FNOL_Date__c = TODAY(), and Claim__c.Source__c = 'Agent Assisted' or 'Self Service' based on session context.

Given the FNOL is submitted successfully,
When the Claim__c record is created,
Then a Platform Event (FNOL_Submitted__e) is fired containing Claim__c.Id, Policy__c.Id, and LOB__c to trigger downstream adjudication queue assignment.

Given a duplicate claim check is run on submission,
When Claim__c.Policy__c + Claim__c.Date_of_Loss__c + Claim__c.Primary_Diagnosis_Code__c match an existing open Claim__c,
Then the OmniScript displays a duplicate warning modal and requires the user to confirm intent before proceeding.

-Given the user is a TPA representative, When they submit FNOL, Then Claim__c.TPA_Flag__c is set to TRUE automatically- (deprecated — TPA flag now set via Record-Triggered Flow on Policy__c.Account_Type__c, see INS-0285)

## Solution

Build a multi-step OmniScript (Type: FNOL_Intake, Sub-Type: ClaimCreation) with conditional LOB branching using OmniScript conditional views. Use DataRaptor Extract to query Policy__c and related Contact on policy number lookup. Use DataRaptor Transform to map OmniScript JSON to Claim__c and Claim_Line__c field API names. Fire FNOL_Submitted__e Platform Event via an Integration Procedure on final step. Duplicate detection via a DataRaptor Turbo Extract querying existing Claim__c records. Deploy OmniScript as an embedded LWC on the Policy__c record page via FlexCard action button. Assign access via Permission Set Group: Claims_Intake_PSG.

## Description

As a Compliance Officer, I need a complete audit trail of all SIU (Special Investigations Unit) referrals generated from Claim__c records — including both manual referrals and those initiated by the Agentforce Claims AI fraud scoring model — to be captured in a structured SIU_Referral_Audit__c object with case linkage, so that Acme Insurance can produce NICB (National Insurance Crime Bureau) mandated fraud reporting and demonstrate AI model governance to state regulators.

## Acceptance Criteria

Given a Claims Operations Manager manually creates an SIU referral from a Claim__c record by clicking 'Refer to SIU', When the action is confirmed, Then an SIU_Referral_Audit__c record is created with: SIU_Referral_Audit__c.Claim__c (lookup), SIU_Referral_Audit__c.Referral_Source__c = 'Manual', SIU_Referral_Audit__c.Referred_By__c (User lookup), SIU_Referral_Audit__c.Referral_Date__c (system datetime), SIU_Referral_Audit__c.Referral_Reason__c (required picklist: 'Billing Anomaly'|'Identity Discrepancy'|'Provider Pattern'|'Claimant Pattern'|'Other'), and SIU_Referral_Audit__c.Status__c = 'Pending SIU Review'.

Given the Agentforce Claims AI fraud scoring model assigns a Claim__c.Fraud_Risk_Score__c ≥ 85 (on a 0–100 scale), When the score is written to the record, Then an SIU_Referral_Audit__c record is automatically created with SIU_Referral_Audit__c.Referral_Source__c = 'Agentforce AI', SIU_Referral_Audit__c.AI_Model_Version__c (populated from Custom Metadata Type Agentforce_Model_Config__mdt.Current_Fraud_Model_Version__c), SIU_Referral_Audit__c.AI_Confidence_Score__c = Claim__c.Fraud_Risk_Score__c, and SIU_Referral_Audit__c.AI_Triggering_Factors__c (JSON string of top 3 model feature contributions).

Given an SIU_Referral_Audit__c record is created (manual or AI), When the record is saved, Then a corresponding Case record (Record Type: 'SIU Investigation') is auto-created and linked via SIU_Referral_Audit__c.SIU_Case__c (lookup to Case), with Case.Subject = 'SIU Investigation — [Claim Number]', Case.Priority = 'High', and Case.OwnerId = the SIU Queue.

Given a Compliance Officer accesses the 'SIU Compliance Report' dashboard, When they filter by date range and Referral_Source__c, Then the dashboard displays: total referrals by source (Manual vs. AI), AI model referral acceptance rate (SIU cases where Case.Status = 'Closed — Confirmed Fraud' / total AI referrals), average days from referral to case closure, and a geographic heat map by Claim__c.Loss_State__c.

Given an SIU investigation Case is closed with Case.Resolution__c = 'Confirmed Fraud', When the closure is saved, Then Claim__c.Status__c is updated to 'Fraud Confirmed', Claim__c.SIU_Outcome__c = 'Confirmed Fraud', and a regulatory reporting flag SIU_Referral_Audit__c.NICB_Reportable__c is set to TRUE, queuing the record for the next NICB batch submission.

## Solution

Quick Action 'Refer to SIU' on Claim__c invokes a Screen Flow collecting Referral_Reason__c before creating SIU_Referral_Audit__c and the linked Case. Record-Triggered Flow on Claim__c monitors Fraud_Risk_Score__c crossing the 85 threshold (fires on update when new value ≥ 85 and old value < 85) and calls an Apex invocable action 'CreateAIReferralAudit' that reads Agentforce_Model_Config__mdt and constructs the SIU_Referral_Audit__c with AI metadata. Agentforce Einstein model feature contributions are passed via a named JSON field populated by the Agentforce Action output. Case auto-creation is handled within the same Apex invocable. CRM Analytics dashboard uses datasets on SIU_Referral_Audit__c, Case, and Claim__c with a custom SAQL geo-aggregation for the heat map. NICB batch reporting uses a Scheduled Apex class querying SIU_Referral_Audit__c where NICB_Reportable__c = TRUE and submission status is pending.

## Description

As a Provider Network Specialist, I need automated compliance alerts when a Provider_Network_Agreement__c (PNA) is within 90, 60, and 30 days of expiration, and I need a Provider_Change_Notification__c (PCN) record auto-generated at the 30-day threshold, so that we avoid inadvertent out-of-network claims processing and meet CMS network adequacy notification requirements.

## Acceptance Criteria

Given a PNA__c.Contract_End_Date__c is 90 days from today,
When the nightly scheduled compliance check runs,
Then a Task must be created assigned to the PNA__c.Owner with Subject = 'PNA Expiring in 90 Days — Action Required', Priority = 'High', and a related Compliance_Alert__c record with Alert_Type__c = '90-Day PNA Expiry' must be inserted.

Given a PNA__c.Contract_End_Date__c is 30 days from today,
When the nightly check runs,
Then a Provider_Change_Notification__c record must be auto-created with PCN_Type__c = 'Contract Expiry', Related_PNA__c = PNA__c.Id, Effective_Date__c = PNA__c.Contract_End_Date__c, Status__c = 'Pending Review', and assigned to the Provider_Relations_Specialist queue.

Given a PCN is auto-generated,
When the Provider Network Specialist reviews it,
Then they must be able to set PCN.Resolution__c to 'Renewed', 'Terminated', or 'Extended' which must write back to PNA__c.Status__c accordingly via a Record-Triggered Flow.

Given PNA__c.Status__c is set to 'Terminated',
When Claims Processing evaluates network eligibility for a Claim_Line__c,
Then any Claim_Line__c.Provider_NPI__c associated with the terminated PNA must trigger Claim__c.Network_Status__c = 'Out of Network' and a compliance flag Claim__c.Network_Compliance_Flag__c = true.

Given the alert emails must be sent,
When a Compliance_Alert__c is created,
Then an Email Alert action in the Flow must send a formatted notification to the Provider Relations Specialist and CC the Compliance Officer distribution list configured in Org_Email_Config__mdt.

## Solution

Scheduled Flow (runs nightly at 2:00 AM) queries PNA__c records filtered by Contract_End_Date__c BETWEEN TODAY+29 AND TODAY+91. Flow decision elements branch on day threshold (90, 60, 30). At 30-day branch, a Create Records element inserts Provider_Change_Notification__c. Task creation handled by Flow Create Records element. PCN-to-PNA writeback implemented as a separate Record-Triggered Flow on Provider_Change_Notification__c (after update, fires when Resolution__c changes). Network eligibility check uses an Apex-invocable method called from the Claims Adjudication OmniScript. Custom Metadata Type Org_Email_Config__mdt stores distribution list addresses. All flows include fault paths logging to Integration_Error_Log__c.

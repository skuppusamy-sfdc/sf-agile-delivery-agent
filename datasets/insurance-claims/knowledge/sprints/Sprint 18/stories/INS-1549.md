## Description

As an Underwriting Analyst, I need an automated monthly extract of adjudicated Medicare Advantage claims data formatted to CMS EDGE Server specifications (837I/837P flat-file layout) so that we meet the Risk Adjustment Data Submission (RADS) deadline without manual data pulls from multiple systems.

Extends INS-1518 (Claims Data Warehouse sync).

## Acceptance Criteria

Given the first business day of each month at 02:00 AM CST,
When the scheduled Apex batch job 'EDGEServerExtractBatch' executes,
Then it must query all Claim__c records where Claim__c.LOB__c = 'Medicare Advantage' AND Claim__c.Status__c = 'Adjudicated' AND Claim__c.Service_Date__c falls within the prior calendar month.

Given the batch job selects eligible claims,
When the extract file is generated,
Then each row must include: Member_ID (Policy__c.CMS_HIC_Number__c), NPI (Provider__c.NPI__c), TIN (Provider__c.TIN__c), ICD-10 diagnosis codes from Claim_Line__c.ICD10_Primary__c and Claim_Line__c.ICD10_Secondary__c (up to 12), CPT procedure codes from Claim_Line__c.CPT_Code__c, Paid_Amount (Payment__c.Amount__c), and DOS (Claim__c.Date_of_Service__c).

Given the extract file is generated,
When the file size exceeds 50 MB,
Then the file must be split into sequential numbered segments (e.g., EDGE_202504_001.txt, EDGE_202504_002.txt) and all segments must be uploaded to the configured SFTP endpoint stored in Custom Metadata Type 'CMS_Integration_Config__mdt'.

Given the SFTP upload completes successfully,
When all files are confirmed transferred,
Then a Regulatory_Submission__c record must be created with fields: Submission_Type__c = 'EDGE', Submission_Date__c, Record_Count__c, File_Names__c, Status__c = 'Submitted', and a notification email sent to the Compliance Officer distribution list stored in 'CMS_Integration_Config__mdt.Notification_Email__c'.

Given the batch job encounters any claim with a missing Policy__c.CMS_HIC_Number__c or Provider__c.NPI__c,
When processing that record,
Then the claim must be written to a Regulatory_Exception__c record with Exception_Type__c = 'Missing Required CMS Field', excluded from the extract file, and included in a daily exception report emailed to the Underwriting Analyst group.

## Solution

Implement a Schedulable Apex class 'EDGEServerExtractScheduler' that invokes 'EDGEServerExtractBatch' (Database.Batchable, Database.Stateful) with a batch size of 200. The batch collects Claim__c, related Claim_Line__c, Payment__c, Policy__c, and Provider__c data in the execute() method, building a StringBuilder for the flat-file content. File segmentation logic splits the StringBuilder at 50 MB boundaries. Salesforce Files (ContentVersion) stores each segment temporarily before an Apex callout (HttpRequest) pushes files to the SFTP endpoint via a Named Credential. A Custom Metadata Type 'CMS_Integration_Config__mdt' stores endpoint URL, credentials reference, and notification emails. Regulatory_Submission__c and Regulatory_Exception__c DML operations occur in the finish() method. A Record-Triggered Flow on Regulatory_Submission__c sends the confirmation email via an Email Alert.

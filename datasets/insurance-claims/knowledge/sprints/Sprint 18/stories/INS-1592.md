## Description

As a Compliance Officer, I need an immutable, timestamped audit trail capturing every status transition and field-level change on Claim__c and Claim_Line__c records throughout the adjudication lifecycle so that Acme Insurance can satisfy state DOI examination requests, CMS audit requirements, and internal SIU investigations without manual reconstruction of claim history.

## Acceptance Criteria

Given a Claim__c record transitions from any status to any other status (e.g., 'Received' → 'In Adjudication' → 'Adjudicated' → 'Paid'), When the status change is saved, Then a Claim_Audit_Event__e Platform Event is published and a corresponding Claim_Audit_Log__c record is created with fields: Claim_Audit_Log__c.Claim__c (lookup), Claim_Audit_Log__c.Changed_Field__c, Claim_Audit_Log__c.Old_Value__c, Claim_Audit_Log__c.New_Value__c, Claim_Audit_Log__c.Changed_By__c (User lookup), Claim_Audit_Log__c.Change_Timestamp__c (system datetime, non-editable), and Claim_Audit_Log__c.Change_Source__c ('User'|'Automation'|'Integration').

Given a Claim_Line__c field is modified (e.g., Claim_Line__c.Allowed_Amount__c, Claim_Line__c.CPT_Code__c, Claim_Line__c.ICD10_Primary__c), When the record is saved by any actor (user, Apex, Flow, API), Then a corresponding Claim_Audit_Log__c child record is created linking to both the Claim_Line__c and its parent Claim__c.

Given an audit log record has been created, When any user (including System Administrator) attempts to edit or delete Claim_Audit_Log__c, Then the operation is blocked by an Apex-enforced restriction and an error message 'Audit records are immutable and cannot be modified or deleted per compliance policy' is surfaced.

Given a Compliance Officer views a Claim__c record, When they open the 'Audit Trail' tab on the Claim Lightning Record Page, Then a paginated LWC component displays all Claim_Audit_Log__c entries sorted by Change_Timestamp__c descending, with column filters for Changed_Field__c, Change_Source__c, and Changed_By__c.

-Given audit logs older than 3 years are archived to Salesforce Big Objects- (superseded: retention period updated to 7 years per updated DOI regulation; see INS-1597 for Big Object archival implementation)

Given a CMS or DOI audit export is requested, When the Compliance Officer clicks 'Export Audit Trail' on the Claim__c record, Then a Conga CLM-generated PDF report is produced containing all Claim_Audit_Log__c entries for that claim, formatted per the standard regulatory template, and attached as a ContentVersion to the Claim__c record.

## Solution

Implement a Record-Triggered Flow on Claim__c (after-save, all changes) that publishes a Claim_Audit_Event__e Platform Event. A separate Apex trigger handler (ClaimAuditEventTrigger) consumes the Platform Event asynchronously and inserts Claim_Audit_Log__c records. Field-level change detection uses a before-trigger snapshot pattern stored in a transient Apex map. Claim_Line__c changes are handled by a parallel Record-Triggered Flow. Immutability is enforced via an Apex before-delete and before-update trigger on Claim_Audit_Log__c that throws a System.AuraHandledException. The Audit Trail LWC uses @wire(getRelatedListRecords) with server-side pagination via a custom Apex controller. Conga CLM template is configured with a SOQL dataset pulling Claim_Audit_Log__c. A Permission Set Group 'Compliance_Audit_Viewer' grants read-only access to Claim_Audit_Log__c for Compliance Officers.

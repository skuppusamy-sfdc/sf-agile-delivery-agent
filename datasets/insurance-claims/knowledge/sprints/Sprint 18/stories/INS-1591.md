## Description

As a Provider Network Specialist, I need a mandatory compliance checklist enforced on the Network_Termination_Case__c object that must be fully completed before the case status can be moved to 'Offboarding Complete' so that we comply with state regulatory requirements for provider termination notice periods, member notification obligations, and continuity-of-care provisions before a provider is removed from the network directory.

## Acceptance Criteria

Given a Network_Termination_Case__c record exists with Status__c = 'In Progress',
When a user attempts to change Status__c to 'Offboarding Complete',
Then a Validation Rule must check that all of the following checklist fields are true: Termination_Notice_Sent__c = true, Member_Notification_Sent__c = true, Continuity_of_Care_Review_Complete__c = true, Claims_Run_Out_Period_Confirmed__c = true, and PNA_Termination_Signed__c = true. If any are false, the save must be blocked with field-level error messages identifying each incomplete item.

Given Termination_Notice_Sent__c is set to true on a Network_Termination_Case__c,
When the record is saved,
Then Termination_Notice_Date__c must be auto-stamped with today's date and a Compliance_Alert_Log__c record must be inserted with Alert_Type__c = 'Termination Notice Issued' and Related_Provider__c populated.

Given a Network_Termination_Case__c reaches Status__c = 'Offboarding Complete',
When the transition is saved,
Then a Record-Triggered Flow must: (a) set Provider__c.Network_Status__c = 'Terminated', (b) update all active Provider_Network_Agreement__c records for that provider to PNA_Status__c = 'Terminated' with PNA_Termination_Date__c = today, (c) create a final Network_Termination_Audit__c record capturing all checklist field values and the completing user.

Given the termination impacts members with active care relationships,
When Continuity_of_Care_Review_Complete__c is set to true,
Then the system must query all active Policy__c records where the provider appears as PCP__c or Specialist__c and generate a Task for the Assignment Coordinator queue for each affected member.

-Given termination letters were to be sent via postal mail only-
Given termination letters must be sent via both DocuSign (electronic) and postal mail,
When Termination_Notice_Sent__c is set to true,
Then a DocuSign envelope must be automatically triggered using the 'Provider Termination Notice' template with the provider's Primary_Email__c as recipient, and Docusign_Envelope_Id__c must be populated on the Network_Termination_Case__c record upon successful send.

## Solution

1. Validation Rule on Network_Termination_Case__c enforces all five checklist boolean fields before allowing Status__c = 'Offboarding Complete'. 2. Record-Triggered Flow (After Save) on Network_Termination_Case__c: branch on Termination_Notice_Sent__c changing to true — stamps Termination_Notice_Date__c, inserts Compliance_Alert_Log__c, and calls Apex action to trigger DocuSign envelope via DocuSign Apex Toolkit (DSFLib). Apex action populates Docusign_Envelope_Id__c from API response. 3. Separate Flow branch on Status__c = 'Offboarding Complete': updates Provider__c.Network_Status__c, loops through related PNA records to set termination fields, creates Network_Termination_Audit__c. 4. Apex invocable method queries Policy__c records by PCP__c or Specialist__c lookup matching provider; bulk-creates Tasks assigned to Assignment Coordinator queue. 5. Network_Termination_Audit__c OWD = Private; accessible via Compliance Officer and Provider Network Specialist Permission Set Groups.

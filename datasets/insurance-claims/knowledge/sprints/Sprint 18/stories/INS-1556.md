## Description

As a Compliance Officer, I need a state-specific regulatory reporting dashboard that aggregates Claim__c, Payment__c, and EOB data by LOB and jurisdiction so that we can generate accurate monthly and quarterly DOI submission packages without manual data extraction from multiple systems.

## Acceptance Criteria

Given the Compliance Officer selects a State__c and Reporting_Period__c on the Regulatory Reporting OmniScript, When they click 'Generate Report', Then the system must query all Claim__c records where Policy__c.State__c matches the selected state and Claim__c.Closed_Date__c falls within the reporting period, returning aggregate counts of: Total_Claims_Submitted__c, Total_Claims_Adjudicated__c, Total_Claims_Denied__c, Average_Processing_Days__c.

Given the report is generated, When Total_Claims_Denied__c / Total_Claims_Submitted__c > 0.15 (15% denial threshold), Then the dashboard must highlight the metric in red and auto-create a Compliance_Alert__c record with Alert_Type__c = 'High Denial Rate' linked to the reporting run.

Given the Compliance Officer clicks 'Export to CSV', When the export is triggered for datasets under 50,000 records, Then a CSV file must be generated via Conga CLM template and emailed to the logged-in user's email within 5 minutes.

Given a Payment__c record has Payment_Status__c = 'Reversed', When it is included in the reporting period, Then it must be counted separately in a Reversals__c aggregate column and excluded from Net_Paid_Amount__c calculations.

Given the dashboard loads, When the user has the Permission Set 'Compliance_Reporting_Access', Then all widgets must render within 3 seconds; users without this permission must see an 'Access Restricted' message and no data.

## Solution

Build an OmniScript 'RegulatoryReportingWizard' with DataRaptor Extract steps querying Claim__c, Payment__c, and EOB__c with aggregate SOQL. A custom Apex action 'RegulatoryReportCalculator' handles denial rate thresholds and auto-creates Compliance_Alert__c records via a separate Apex invocable method to respect mixed DML boundaries. FlexCard 'RegulatoryDashboardCard' surfaces KPIs with conditional color formatting using expression sets. Conga CLM is called via REST API for CSV export, with results delivered via Salesforce Email Action. Report filters leverage custom indexes on Policy__c.State__c and Claim__c.Closed_Date__c. Permission Set Group 'Compliance_Reporting_PSG' controls access.

## Description

As a Compliance Officer, I need a comprehensive, tamper-evident audit trail for every status transition on Claim__c and Claim_Line__c records so that we can satisfy state DOI examination requests, SOX controls, and internal SIU investigations without manual reconstruction of adjudication history.

## Acceptance Criteria

Given a Claim__c record undergoes any Status__c transition (e.g., 'Submitted' → 'In Review' → 'Adjudicated' → 'Paid'), When the transition is saved by any user or automated process, Then a Claim_Audit_Event__c child record must be created capturing: Old_Status__c, New_Status__c, Changed_By__c (User lookup), Changed_Date__c (system timestamp), Change_Source__c ('User' | 'Flow' | 'Apex' | 'Integration'), and IP_Address__c.

Given a Claim_Line__c record has its Adjudication_Status__c changed, When the record is saved, Then a corresponding Claim_Line_Audit__c record is created linked to both Claim_Line__c and its parent Claim__c, capturing CPT_Code__c, Billed_Amount__c, Allowed_Amount__c, and Denial_Reason_Code__c at the time of change.

Given a Compliance Officer or SIU Analyst views a Claim__c record, When they open the 'Audit Trail' tab on the Claim layout, Then they see a chronological timeline LWC component displaying all Claim_Audit_Event__c and Claim_Line_Audit__c entries, sortable by Changed_Date__c, filterable by Change_Source__c.

Given an audit trail record (Claim_Audit_Event__c) has been created, When any user (including System Administrator) attempts to edit or delete it, Then the operation is blocked by an Apex-enforced restriction and a platform event Compliance_Violation_Alert__e is fired to notify the Compliance Officer queue.

-Given audit records older than 3 years are purged via scheduled job- (deprecated per legal hold policy INS-1498; all records retained indefinitely)

Given a regulatory export is requested, When the Compliance Officer clicks 'Export Audit Report' on the Claim__c record, Then a Conga CLM-generated PDF is produced containing the full audit trail, digitally timestamped, and stored as a ContentVersion linked to the Claim__c.

## Solution

Implement an Apex Trigger (ClaimAuditTrigger) on Claim__c and Claim_Line__c using 'before update' and 'after update' contexts to capture field-level deltas and insert Claim_Audit_Event__c / Claim_Line_Audit__c records in a separate transaction via a Queueable Apex class to avoid recursive trigger loops. Use a custom Platform Event (Compliance_Violation_Alert__e) fired via Apex when DML on audit records is attempted; subscribe via a Record-Triggered Flow on the Compliance Officer queue case. LWC 'claimAuditTimeline' uses @wire(getRecord) and a custom Apex @AuraEnabled controller to render chronological audit entries. Conga CLM template configured for PDF generation with digital timestamp metadata. Field History Tracking enabled as a secondary fallback on Claim__c.Status__c. Permission Set Group 'Compliance_ReadOnly_PSG' grants read access to Claim_Audit_Event__c without edit/delete. OWD for Claim_Audit_Event__c set to Private with no sharing rules to prevent lateral access.

## Description

As a Compliance Officer, I need a real-time regulatory reporting dashboard aligned to state Department of Insurance (DOI) requirements so that I can monitor claims payment timeliness, denial rates by LOB, and pending acknowledgment SLAs without exporting data to external spreadsheets. This story extends INS-1549 which established the base Compliance Permission Set Group.

## Acceptance Criteria

Given the Compliance Officer opens the 'DOI Regulatory Metrics' Lightning App,
When the dashboard loads,
Then it must display the following KPI tiles refreshed every 15 minutes: (1) % Claims Paid within 30 days (Claim__c.Payment_Date__c - Claim__c.Received_Date__c <= 30), (2) % Claims Acknowledged within 10 business days, (3) Denial Rate by LOB__c, (4) Pending SIU Referrals count where Claim__c.SIU_Referral_Status__c = 'Open'.

Given the dashboard is displayed,
When a KPI tile value breaches a configurable threshold (stored in Compliance_Threshold__mdt),
Then the tile must render with a red RAG indicator and a tooltip showing the threshold value and current delta.

Given a Compliance Officer selects a state from the 'Reporting State' picklist filter (mapped to Policy__c.State__c),
When the filter is applied,
Then all dashboard components must re-query scoped to that state's claims only, including Claim__c records where related Policy__c.State__c matches.

Given the end of a reporting period,
When the Compliance Officer clicks 'Generate DOI Report',
Then a structured PDF report (Conga template DOI_RPT_001) must be produced summarizing all metrics, stamped with Report_Generated_By__c, Report_Period_Start__c, and Report_Period_End__c, and saved as a ContentDocument linked to a new Regulatory_Submission__c record.

-Given metrics are sourced from a nightly batch Apex job- (deprecated — replaced by real-time SOQL-backed CRM Analytics dataset per architecture review)

Given a Regulatory_Submission__c record is created,
When its Submission_Status__c is set to 'Submitted',
Then an automated email notification must be dispatched to the Compliance Officer and their manager (via Org-Wide Email Address 'compliance-noreply@acmeinsurance.com') confirming submission details.

## Solution

Build a CRM Analytics (Tableau CQ) dashboard with SAQL-backed datasets sourced from Claim__c, Policy__c, and Payment__c objects. Configure a scheduled dataflow refresh every 15 minutes using incremental sync on Claim__c.LastModifiedDate. Store threshold values in a Custom Metadata Type (Compliance_Threshold__mdt) with fields: Metric_Name__c, Warning_Threshold__c, Critical_Threshold__c, LOB__c. Implement Conga Composer for PDF generation triggered via a Lightning Quick Action on Regulatory_Submission__c. Create a Record-Triggered Flow on Regulatory_Submission__c (after update, when Submission_Status__c changes to 'Submitted') to send email alert using an Org-Wide Email Address. Deploy a Permission Set Group 'DOI_Compliance_Reporting' to control dashboard and Regulatory_Submission__c access.

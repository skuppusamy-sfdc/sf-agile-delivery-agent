## Description

As a Compliance Officer, I need a real-time regulatory reporting dashboard that aggregates key claims processing KPIs required for monthly state Department of Insurance (DOI) submissions — including prompt payment compliance rates, denial ratios by LOB, average days-to-pay, and COB recovery totals — so that Acme Insurance can automate report generation and reduce manual data extraction effort by at least 80%.

## Acceptance Criteria

Given the Compliance Officer accesses the 'DOI Monthly Submission' Lightning App page, When the page loads, Then a suite of CRM Analytics (Einstein Analytics) dashboards displays the following widgets, all filterable by State__c, LOB__c, and reporting month: (1) Prompt Payment Compliance Rate (% of Payment__c records where Payment__c.Payment_Date__c minus Claim__c.Received_Date__c is ≤ state statutory threshold), (2) Denial Rate by LOB (count of Claim__c where Claim__c.Status__c = 'Denied' / total claims), (3) Average Days to Pay (mean of Payment__c.Payment_Date__c minus Claim__c.Adjudication_Date__c), (4) COB Recovery Total (sum of Payment__c.COB_Recovery_Amount__c).

Given the reporting period end date is reached (last calendar day of month), When a scheduled Apex job runs at 11:59 PM, Then a DOI_Report_Snapshot__c record is created capturing all metric values for that period, with DOI_Report_Snapshot__c.Reporting_Period__c, DOI_Report_Snapshot__c.State__c, DOI_Report_Snapshot__c.LOB__c, DOI_Report_Snapshot__c.Generated_By__c = 'Scheduled Job', and DOI_Report_Snapshot__c.Status__c = 'Pending Review'.

Given a DOI_Report_Snapshot__c record has Status__c = 'Pending Review', When the Compliance Officer reviews and clicks 'Approve for Submission', Then DOI_Report_Snapshot__c.Status__c transitions to 'Approved' and an email notification is sent to the Compliance Officer's manager via a Flow-triggered email alert.

Given an Approved DOI_Report_Snapshot__c record, When the Compliance Officer clicks 'Generate Submission Package', Then Conga CLM produces a formatted Excel-compatible .xlsx report attached as ContentVersion to the DOI_Report_Snapshot__c record, matching the target state's DOI template format.

Given any dashboard metric value falls outside a configured compliance threshold (e.g., Prompt Payment Rate < 95%), When the dashboard renders, Then the affected widget displays a red RAG indicator and a Salesforce In-App Notification is sent to all users in the 'Compliance_Officers' Public Group.

## Solution

CRM Analytics (Tableau CRM) dataset built on Claim__c, Payment__c, and Policy__c objects using a scheduled dataflow running nightly. Dashboard built with SAQL for custom prompt-pay compliance calculations incorporating state-specific threshold values stored in a Custom Metadata Type (DOI_State_Threshold__mdt). Scheduled Apex (DOISnapshotScheduler) implements Schedulable interface for month-end snapshot creation. Approval process on DOI_Report_Snapshot__c with email alerts configured via Flow. Conga CLM template uses a JSON dataset connector to pull snapshot field values. RAG threshold alerting uses a Record-Triggered Flow on DOI_Report_Snapshot__c that calls a custom notification action.

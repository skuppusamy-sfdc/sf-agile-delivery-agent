## Description

As a Compliance Officer, I need a tamper-evident audit trail capturing every status transition on Claim__c and Claim_Line__c records so that our organization meets state-mandated claims handling timelines and can produce evidence during regulatory audits without manual reconstruction of events.

## Acceptance Criteria

Given a Claim__c record transitions from any status to another (e.g., Status__c changes from 'Pending Review' to 'Adjudicated'),
When the transition is saved by any user or automated process,
Then a new Claim_Audit_Event__c record must be created capturing: Claim__c (lookup), Prior_Status__c, New_Status__c, Changed_By__c (User), Changed_DateTime__c (UTC), Trigger_Source__c ('User'|'Flow'|'Apex'|'Integration'), and IP_Address__c of the session.

Given a Claim_Line__c adjudication decision is recorded (Claim_Line__c.Adjudication_Status__c updated),
When the record is saved,
Then a corresponding Claim_Line_Audit_Event__c must be created with Parent_Claim__c, Line_Number__c, Allowed_Amount__c, Denied_Reason_Code__c, and Adjudicator_User__c populated.

Given an audit trail record (Claim_Audit_Event__c) is created,
When any user attempts to edit or delete the record,
Then the system must block the operation and surface an error: 'Audit records are immutable. Contact your Compliance Officer.'

Given a Compliance Officer navigates to the Claim 360 page,
When they view the Audit Trail tab,
Then all Claim_Audit_Event__c records must be displayed in reverse-chronological order in an LWC timeline component, filterable by Trigger_Source__c and date range.

Given a regulatory export is requested,
When the Compliance Officer clicks 'Export Audit Trail' on a Claim__c record,
Then a CSV file must be generated via Conga and emailed to the requesting user within 2 minutes, containing all Claim_Audit_Event__c and Claim_Line_Audit_Event__c records for that claim.

-Given audit events are stored on the Claim__c record itself using a Long Text Area field- (deprecated — separate object approach mandated by Compliance team INS-1541)

## Solution

Implement an Apex Trigger (after insert, after update) on both Claim__c and Claim_Line__c to create Claim_Audit_Event__c and Claim_Line_Audit_Event__c records respectively. Use a TriggerHandler pattern with a dedicated AuditTrailService Apex class. Enforce immutability via a second Apex Trigger (before update, before delete) on Claim_Audit_Event__c that throws an AuraHandledException. Deploy a Record-Triggered Flow on Claim__c as a supplementary capture mechanism for Flow-originated changes, publishing a Platform Event (Claim_Audit_PE__e) consumed by an Apex handler to stamp Trigger_Source__c = 'Flow'. Build the LWC timeline component (claimAuditTimeline) using wire adapter to query Claim_Audit_Event__c with SOQL ordering. Conga Composer template configured for CSV export, triggered via a Quick Action on Claim__c layout. Restrict Claim_Audit_Event__c create/read via Permission Set Group 'Compliance_ReadOnly' for audit consumers.

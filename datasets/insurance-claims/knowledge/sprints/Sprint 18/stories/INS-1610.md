# INS-1610: Implement PII masking for TIN and NPI fields in all compliance report exports

## Description

As a Compliance Officer, I need all data exports and regulatory report extracts that include Provider__c.TIN__c and Provider__c.NPI__c fields to apply format-preserving masking (showing only the last 4 digits) unless the requesting user holds the 'PII_Unmasked_Access' Permission Set so that the organization complies with data minimization requirements under applicable state privacy regulations and reduces exposure in the event of unauthorized report access.

## Acceptance Criteria

Given a user without the 'PII_Unmasked_Access' Permission Set generates any compliance report export (Regulatory_Report__c ContentVersion, ad-hoc report, or DataRaptor extract),
When the export file is produced,
Then Provider__c.TIN__c must appear as '***-**-XXXX' and Provider__c.NPI__c must appear as '******XXXX' where XXXX represents the last 4 digits.

Given a user WITH the 'PII_Unmasked_Access' Permission Set generates the same export,
When the export file is produced,
Then Provider__c.TIN__c and Provider__c.NPI__c must appear unmasked in their full formats.

Given the masking utility is invoked,
When TIN__c or NPI__c is null or fewer than 4 characters,
Then the masked output must be '****' and a warning must be logged to Compliance_Data_Quality_Log__c with Field_Name__c, Provider__c lookup, and Detected_DateTime__c.

Given the PII masking logic is implemented as a reusable Apex utility class,
When called from any report generation context (batch job, LWC download, OmniScript),
Then the same masking output format must be produced consistently across all channels.

-Given masking applies to Claim__c.Claimant_SSN__c as well- (deprecated: SSN masking is handled separately in INS-1588 and is out of scope here)

Given a masked export is generated,
When the export ContentVersion is saved,
Then a Compliance_Export_Log__c record must be created capturing: Exported_By__c, Export_DateTime__c, Report_Type__c, Masking_Applied__c (checkbox = true/false), and Record_Count__c.

## Solution

Develop a reusable Apex utility class 'PIIMaskingService' with static methods maskTIN(String tin) and maskNPI(String npi) that check FeatureManagement or Permission Set assignment for the running user via a cached UserInfo query. The service is called from: (1) the NAIC_Quarterly_Extract_Batch finish() method before ContentVersion creation, (2) a custom LWC 'complianceExportButton' that triggers on-demand exports, and (3) DataRaptor Transform formulas via an Apex-backed custom formula function registered in OmniStudio. A custom Permission Set 'PII_Unmasked_Access' is created and assigned only to the Compliance Officer and SIU roles via a Permission Set Group. Compliance_Export_Log__c inserts are handled by a Platform Event 'Compliance_Export_Event__e' to avoid blocking the export transaction.

## Description

Steps to Reproduce:
1. Log in as the CAQH Integration service user.
2. Use Bulk API v2 to update a batch of Credentialing_Case__c records, setting Credentialing_Status__c from 'In Review' to 'Approved' for 50+ records simultaneously.
3. Navigate to any updated Credentialing_Case__c record and inspect the Credentialing_Audit_Log__c child records.
4. Observe that Credentialing_Audit_Log__c.Status_Change_Timestamp__c is null for all records updated via Bulk API, while the same field is correctly populated when records are updated via the UI or standard REST API.

Expected Behavior: Credentialing_Audit_Log__c.Status_Change_Timestamp__c must be populated with the exact datetime of the status change regardless of the DML source (UI, REST API, Bulk API, or Flow).

Actual Behavior: Status_Change_Timestamp__c remains null when updates originate from Bulk API v2 calls, creating gaps in the compliance audit trail for CAQH-driven credentialing approvals. This is a compliance risk as state credentialing regulations require a complete audit trail with timestamps for all status transitions.

Environment: Production, also reproducible in Full Sandbox.
Regression introduced in: Sprint 16 deployment (INS-1534 Apex trigger refactor).

## Acceptance Criteria

As per Description. Verify fix in Full Sandbox using Bulk API v2 test harness with 200-record batch. Confirm Credentialing_Audit_Log__c.Status_Change_Timestamp__c is populated for all records. Confirm fix does not break existing UI and REST API audit log creation. Add regression test to CredentialingAuditTriggerTest Apex test class covering Bulk API context simulation.

## Solution

Investigate the CredentialingAuditTrigger Apex trigger introduced in INS-1534. The root cause is likely a conditional block checking System.isBatch() or System.isFuture() that inadvertently skips timestamp assignment in Bulk API context. Fix by replacing System.now() assignment with a trigger context-agnostic approach: always assign Status_Change_Timestamp__c = System.now() within the trigger handler's buildAuditLog() method regardless of execution context. Ensure the fix is applied in both before-insert and after-update trigger contexts on Credentialing_Case__c. Update CredentialingAuditTriggerTest to include a @isTest method simulating bulk DML via a 200-record list insert/update to verify timestamp population. Deploy to Full Sandbox for QA validation before production hotfix.

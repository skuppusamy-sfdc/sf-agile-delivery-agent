## Description

As a Provider Network Specialist, I need a structured compliance checklist embedded within the Network_Termination_Case__c workflow and a full audit log of all termination actions so that Acme Insurance meets the 60-day advance written notice requirement under state network adequacy regulations, maintains a defensible termination record, and ensures all downstream impacts (PNA suspension, claims holds, PCN issuance) are tracked and completed before a provider is marked as terminated.

## Acceptance Criteria

Given a Network_Termination_Case__c record is created with Termination_Type__c = 'Voluntary' | 'Involuntary' | 'Non-Renewal', When the record is saved, Then a set of Termination_Checklist_Item__c child records is auto-generated covering: (1) 60-Day Notice Letter Sent, (2) PNA Suspension Initiated, (3) Open Claims Review Completed, (4) PCN Issued to Affected Groups, (5) Credentialing Case Closed, (6) Final Payment Reconciliation Confirmed — each with Checklist_Status__c = 'Pending' and Due_Date__c calculated from Termination_Effective_Date__c.

Given a Termination_Checklist_Item__c with Item_Type__c = '60-Day Notice Letter Sent' is updated to Checklist_Status__c = 'Complete', When the update is saved, Then the system validates that Letter_Sent_Date__c is at least 60 calendar days before the parent Network_Termination_Case__c.Termination_Effective_Date__c; if not, a validation rule prevents the save and surfaces the error: 'Notice date does not satisfy 60-day regulatory requirement. Termination effective date must be adjusted.'

Given all Termination_Checklist_Item__c records for a Network_Termination_Case__c have Checklist_Status__c = 'Complete', When the final item is marked complete, Then a Record-Triggered Flow automatically sets Network_Termination_Case__c.Status__c = 'Ready for Final Approval' and creates an Approval Process submission task for the Provider Network Manager.

Given a Network_Termination_Case__c Status__c changes at any point, When the change is committed, Then a Termination_Audit_Log__c child record is created capturing: Previous_Status__c, New_Status__c, Changed_By__c, Timestamp__c, and a free-text Notes__c field pre-populated with the triggering action description.

-Given PCN is auto-sent via email to all GSA contacts upon termination approval- (deferred; PCN auto-distribution requires Group_Service_Agreement__c contact role model refactor per INS-1532, targeted Sprint 20)

Given a Provider Network Specialist views the Network_Termination_Case__c record, When they open the 'Termination Summary' OmniScript, Then it displays a read-only checklist progress bar (X of 6 items complete), the 60-day notice compliance indicator (Compliant / Non-Compliant), and a list of all open Claim__c records linked to the terminating provider where Claim__c.Status__c != 'Paid' and Claim__c.Status__c != 'Denied'.

## Solution

Record-Triggered Flow on Network_Termination_Case__c (after insert) creates Termination_Checklist_Item__c records using a loop with predefined checklist item metadata stored in a Custom Metadata Type (Termination_Checklist_Config__mdt) to allow admin configuration without code changes. Validation Rule on Termination_Checklist_Item__c enforces 60-day notice logic using a cross-object formula referencing parent Termination_Effective_Date__c. Termination_Audit_Log__c records created via Apex Trigger (after update) on Network_Termination_Case__c to capture all status transitions. OmniScript 'TerminationSummaryOS' uses DataRaptor Extract to pull checklist items and open claims, with a DataRaptor Transform computing compliance status. Approval Process configured on Network_Termination_Case__c with a designated Provider Network Manager approver step. Custom Metadata Type used for checklist item configuration ensures maintainability.

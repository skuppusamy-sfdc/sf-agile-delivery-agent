## Description

**Bug Report — EOB PDF Generation Failure on High-Volume Claims**

**Environment:** Production Sandbox (Full Copy)
**Reported By:** Emily Watson (Claims Operations Manager)
**Severity:** High — Blocking EOB delivery to members for facility claims

**Steps to Reproduce:**
1. Navigate to a Claim__c record where the count of related Claim_Line__c records exceeds 25 (e.g., Claim ID: CLM-2024-88341, a facility inpatient claim).
2. Click the 'Generate EOB' button on the Claim 360 Lightning page.
3. Observe the Conga Composer template 'EOB_Standard_Template_v3' is invoked.
4. Wait for the generation process to complete.

**Expected Result:** A fully populated Explanation_of_Benefits__c PDF is generated and attached to the Claim__c record as a ContentDocument, and EOB__c.Status__c is set to 'Generated'.

**Actual Result:** The Conga Composer job times out after approximately 120 seconds. A generic error toast 'An error occurred while generating the document' is displayed. No PDF is attached. EOB__c.Status__c remains 'Pending'. Integration_Error_Log__c shows: 'Conga Composer: Apex CPU time limit exceeded (15001ms)' with Claim__c.Id in the context field.

**Additional Context:**
- Claims with ≤ 25 Claim_Line__c records generate successfully.
- Issue first observed after Sprint 17 deployment of INS-1558 (Claim_Line__c bulk import feature) which introduced a new formula field on Claim_Line__c (Adjusted_Allowed_Amount__c) that triggers a chain of cross-object formula recalculations.
- Affects approximately 340 facility claims currently in 'Pending EOB' status.
- Workaround: None available; manual PDF generation not feasible at scale.

## Acceptance Criteria

As per Description. Fix must ensure EOB PDF generation succeeds for Claim__c records with up to 500 Claim_Line__c records within the Apex CPU governor limit. EOB__c.Status__c must be set to 'Generated' upon success and 'Failed' with populated Error_Details__c upon failure. Regression test must confirm claims with ≤ 25 lines are unaffected.

## Solution

Root cause: Conga Composer callback Apex class iterates Claim_Line__c records inline, triggering the Adjusted_Allowed_Amount__c formula recalculation chain per row. Fix: Refactor the Conga pre-processing Apex class to query Claim_Line__c records in a single SOQL call with all required fields explicitly listed (avoiding formula field re-evaluation in loop), and store pre-computed values in a transient wrapper list passed to the Conga template via a JSON parameter. Additionally, move EOB generation to an Apex Queueable job to shift execution out of the synchronous context, bypassing the 10-second synchronous CPU limit. Update Integration_Error_Log__c fault handler to capture Queueable job ID for traceability. Deploy updated Conga template 'EOB_Standard_Template_v4' with data source switched from live SOQL to the pre-computed JSON payload.

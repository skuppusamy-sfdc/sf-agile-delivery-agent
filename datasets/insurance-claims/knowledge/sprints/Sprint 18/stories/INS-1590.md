# INS-1590: Agentforce Claims AI – audit logging for AI-assisted adjudication recommendations

- **Sprint**: Acme Insurance Sprint 18 (Phase 2)

## Description

As a Compliance Officer, I need every AI-generated adjudication recommendation produced by the Agentforce Claims AI agent to be logged in a dedicated audit object with the full prompt context, model response, confidence score, and the human decision outcome so that we can demonstrate human-in-the-loop oversight to state regulators and provide an explainability trail for any denied claim that is appealed. This story establishes the compliance foundation for the Agentforce Claims AI feature introduced in Sprint 16.

## Acceptance Criteria

Given the Agentforce Claims AI agent generates an adjudication recommendation for a Claim__c record,
When the recommendation is returned to the Claims Operations Manager UI,
Then an AI_Adjudication_Audit__c record must be created with: Claim__c (lookup), AI_Recommendation__c (picklist: 'Approve'/'Deny'/'Pend'/'Refer to SIU'), AI_Confidence_Score__c (decimal 0.00–1.00), Prompt_Context_Hash__c (SHA-256 hash of prompt payload for traceability without storing PII in plaintext), Model_Version__c (populated from Agentforce API response metadata), and Recommendation_Timestamp__c.

Given an AI_Adjudication_Audit__c record exists for a Claim__c,
When the Claims Operations Manager accepts, modifies, or overrides the AI recommendation,
Then AI_Adjudication_Audit__c.Human_Decision__c must be updated with the final adjudication value and AI_Adjudication_Audit__c.Override_Reason__c must be required if Human_Decision__c differs from AI_Recommendation__c.

Given an AI recommendation resulted in a claim denial and the member files an appeal,
When the Compliance Officer opens the related AI_Adjudication_Audit__c record,
Then a 'Generate Explainability Report' button must be visible that produces a PDF via Conga containing the recommendation rationale fields, model version, confidence score, and human override details.

Given AI_Adjudication_Audit__c records contain sensitive model metadata,
When any user other than Compliance Officer or Claims Operations Manager attempts to access these records,
Then OWD (Private) and Permission Set Group controls must prevent read access, enforced at the object level.

Given regulatory requirements mandate retention,
When an AI_Adjudication_Audit__c record reaches 7 years of age,
Then it must be archived to Salesforce Big Objects (AI_Adjudication_Archive__b) rather than deleted, preserving all fields.

## Solution

1. Create AI_Adjudication_Audit__c custom object; set OWD to Private; grant access via 'Claims_AI_Compliance_PSG' Permission Set Group. 2. Agentforce agent action (Apex-backed) invokes Agentforce Einstein API; on response, constructs and inserts AI_Adjudication_Audit__c record; SHA-256 hash of prompt computed via Crypto.generateDigest() in Apex. 3. LWC 'aiRecommendationPanel' displays recommendation to Claims Operations Manager; on decision capture, calls Apex to update AI_Adjudication_Audit__c.Human_Decision__c; Validation Rule enforces Override_Reason__c when values differ. 4. 'Generate Explainability Report' LWC quick action triggers Conga Composer REST callout with AI_Adjudication_Audit__c Id. 5. Scheduled Apex batch (nightly) queries AI_Adjudication_Audit__c records where CreatedDate <= 7 years ago; inserts to AI_Adjudication_Archive__b Big Object and deletes source records. 6. Platform Event (AI_Audit_Alert__e) fires if AI_Confidence_Score__c < 0.60 to flag low-confidence recommendations for manual review queue.

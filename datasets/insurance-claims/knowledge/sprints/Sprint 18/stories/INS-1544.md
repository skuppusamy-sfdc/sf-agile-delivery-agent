## Description

Steps to Reproduce:
1. Navigate to any Claim__c record that has 26 or more related Claim_Line__c records.
2. Click the 'Generate EOB' button on the Claim__c record page.
3. Observe the Conga CLM merge job initiated via the 'GenerateEOB' Apex invocable action.
4. Wait approximately 45–90 seconds for generation to complete.
5. Observe the result: a generic Salesforce error toast appears reading 'An error occurred while trying to generate the document.'
6. Navigate to the related Explanation_of_Benefits__c record — it is created but EOB_Document_URL__c is null and EOB_Status__c = 'Generation Failed'.

Expected Behavior: The EOB PDF is successfully generated including all Claim_Line__c records regardless of line count, stored as a ContentVersion, and linked to both the Explanation_of_Benefits__c and Claim__c records. EOB_Status__c should be set to 'Generated'.

Actual Behavior: Generation fails silently for claims with >25 lines. Investigation indicates the Conga CLM template query is hitting a SOQL row limit within the template's repeating section — the template uses an inline SOQL query on Claim_Line__c without a LIMIT clause but the Conga session governor appears to cap at 25 rows in the current template version (v3.1). Affects approximately 12% of inpatient facility claims based on production data analysis. First reported by Claims Operations Manager during UAT cycle for Sprint 17.

## Acceptance Criteria

As per Description. Verify fix against Claim__c records with exactly 25, 26, 50, and 100+ Claim_Line__c records in the QA sandbox.

## Solution

Root cause: Conga CLM template 'EOB_Standard_v3.1' uses an inline SOQL query in the repeating table section that is subject to a 25-row fetch limit in the current Conga session configuration. Fix approach: (1) Refactor the Apex invocable action 'GenerateEOBAction' to pre-fetch all Claim_Line__c records via a bulk SOQL query and pass them as a JSON parameter to the Conga merge call using the Conga Composer API 'DS7' parameter override, bypassing the template-level query entirely. (2) Update Conga template to v3.2 using a parameter-fed repeating section instead of inline SOQL. (3) Add try-catch error handling in the invocable action to set EOB_Status__c = 'Generation Failed' and create a Compliance_Exception__c record with Exception_Type__c = 'EOB Generation Error' on failure. (4) Add a unit test asserting EOB generation for a Claim__c with 100 Claim_Line__c records completes without exception and ContentVersion is linked.

## Description

As a Credentialing Specialist, I need an automated nightly sync with the CAQH ProView API to pull updated credentialing attestation data for active providers so that re-verification cycles are triggered on time and Acme Insurance remains compliant with NCQA credentialing standards requiring re-credentialing every 36 months. The sync must update Credentialing_Case__c records and flag providers whose CAQH attestation is expired or expiring within 90 days. Provider__c records (Person Accounts) should reflect current CAQH attestation status without manual intervention.

## Acceptance Criteria

Given the nightly CAQH sync job runs at 02:00 AM EST,
When the CAQH ProView API returns updated attestation data for a Provider__c identified by NPI__c,
Then the associated Credentialing_Case__c record is updated with: CAQH_Attestation_Date__c, CAQH_Profile_Status__c ('Complete'|'Incomplete'|'Expired'), and Last_CAQH_Sync__c = current DateTime.

Given a provider's CAQH attestation expiry date is within 90 days,
When the sync job processes that provider,
Then a Task is created assigned to the owning Credentialing Specialist with Subject = 'CAQH Re-Attestation Required - [Provider Name]', Due_Date__c = CAQH_Attestation_Expiry__c - 30 days, and Priority = 'High'.

Given a provider's CAQH_Profile_Status__c = 'Expired',
When the sync updates the Credentialing_Case__c,
Then Provider__c.Credentialing_Status__c is set to 'Re-Credentialing Required' and Provider__c.Active_Network_Flag__c remains unchanged until a Credentialing Specialist manually resolves the case.

Given the CAQH API returns an HTTP 4xx or 5xx error for a specific NPI,
When the sync job encounters the error,
Then the error is logged to Integration_Error_Log__c with fields: NPI__c, Error_Code__c, Error_Message__c, Sync_Date__c, and the job continues processing remaining providers without aborting.

Given the sync job completes its full run,
When all records are processed,
Then a Sync_Summary_Report__c record is created with: Total_Providers_Processed__c, Success_Count__c, Error_Count__c, Expiring_Within_90_Days__c, and a summary email is sent to the Credentialing Operations distribution list.

-Given CAQH sync also updates malpractice insurance data on Provider__c, When new policy limits are returned, Then Malpractice_Coverage_Amount__c is updated.- (Deferred — malpractice field mapping pending data governance review)

## Solution

Implement a Schedulable Apex class (CAQHNightlySyncScheduler) invoking a Queueable Apex chain (CAQHSyncQueueable) to process providers in batches of 50 to respect CAQH API rate limits. Named Credentials store CAQH API endpoint and OAuth 2.0 credentials. HTTP callouts made via a CAQHApiService Apex class. Results mapped to Credentialing_Case__c and Provider__c via a CAQHDataMapper utility class. Integration errors written to Integration_Error_Log__c via a dedicated logging service. Task creation handled by a Record-Triggered Flow on Credentialing_Case__c firing when CAQH_Attestation_Expiry__c changes and falls within 90-day threshold. Sync_Summary_Report__c creation and email alert triggered via an Apex-invoked OrgWideEmailAddress send at job completion.

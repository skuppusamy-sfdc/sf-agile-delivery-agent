## Description

As a Credentialing Specialist, I need a complete audit log of every CAQH data synchronization event for Provider (Person Account) and Credentialing_Case__c records so that I can demonstrate to auditors that provider credential verifications are current, traceable to a specific CAQH query timestamp, and have not been manually overridden without justification.

## Acceptance Criteria

Given the nightly CAQH integration job runs and retrieves credential data for a provider,
When the DataRaptor or Apex callout to the CAQH ProView API completes (success or failure),
Then a CAQH_Sync_Log__c record must be created with: Provider__c (lookup to Person Account), NPI__c, TIN__c, Sync_Timestamp__c (UTC), Sync_Status__c ('Success'|'Failed'|'Partial'), CAQH_Provider_ID__c, Fields_Updated__c (multi-select picklist of updated fields), Raw_Response_Payload__c (Long Text Area, encrypted at rest), and Error_Message__c (if failed).

Given a CAQH sync updates Credentialing_Case__c.License_Expiry_Date__c or Credentialing_Case__c.Board_Certification_Status__c,
When the update is applied,
Then the corresponding CAQH_Sync_Log__c.Fields_Updated__c must include the specific field name and the prior value must be captured in Prior_Value_Snapshot__c (JSON blob).

Given a Credentialing Specialist manually overrides a CAQH-synced field on a Credentialing_Case__c,
When they save the record,
Then they must be presented with a mandatory Override_Justification__c text prompt (minimum 50 characters), and a Manual_Override_Log__c record must be created capturing Override_By__c, Override_DateTime__c, Field_Overridden__c, Prior_CAQH_Value__c, and New_Manual_Value__c.

Given a Credentialing_Case__c record has not had a successful CAQH sync in more than 180 days,
When the daily compliance check job runs,
Then Credentialing_Case__c.Credential_Verification_Status__c must be set to 'Stale — Re-verification Required' and a Task must be assigned to the case owner.

## Solution

Build a dedicated CAQH integration Apex class (CAQHProViewIntegrationService) using Named Credentials for secure endpoint management. On each API response, instantiate and insert CAQH_Sync_Log__c via a dedicated SyncLogBuilder inner class. Encrypt Raw_Response_Payload__c using Salesforce Shield Platform Encryption (deterministic encryption on NPI__c, probabilistic on payload). Implement a before-update Apex Trigger on Credentialing_Case__c to detect changes to CAQH-managed fields, prompt for justification via a Flow Screen launched from the trigger context via a Platform Event (Override_Prompt_PE__e) consumed by an OmniScript overlay. Daily stale-check via Scheduled Apex (CAQHStalenessCheckBatch). Store the 180-day threshold in Custom Metadata (Credentialing_Config__mdt.CAQH_Staleness_Days__c) for configurability.

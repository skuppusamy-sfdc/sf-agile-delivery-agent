## Description

As a Compliance Officer, I need a regulatory reporting dashboard that visualizes SIU referral rates, AI-generated fraud indicator scores, and claim investigation outcomes so that Acme Insurance can produce the quarterly fraud reporting required by state insurance regulations and identify systemic fraud patterns across LOBs and provider networks.

## Acceptance Criteria

Given the Compliance Officer accesses the 'SIU Regulatory Dashboard' Lightning App Page, When the page loads, Then it must display KPI tiles showing: Total Claims Reviewed (Claim__c count), SIU Referral Rate (Claim__c records where SIU_Referred__c = true / total), Average Fraud_Score__c (from Agentforce scoring), and Open SIU Investigations count — all filterable by Claim__c.LOB__c and a date range picker.

Given the dashboard is filtered, When a Compliance Officer selects LOB = 'Commercial' and a 90-day date range, Then all underlying SOQL queries must respect the filter and return results within 5 seconds for datasets up to 500,000 Claim__c records.

Given an Agentforce AI action has scored a claim, When Claim__c.Fraud_Score__c >= 75, Then Claim__c.SIU_Referred__c must be automatically set to true, an SIU_Case__c record created with SIU_Case__c.Referral_Source__c = 'Agentforce AI', and the claim assigned to the SIU Queue.

Given the quarterly reporting cycle, When a Compliance Officer clicks 'Generate Quarterly Fraud Report', Then the system must produce a Conga-merged PDF report containing: SIU referral rate by LOB, top 10 flagged providers by NPI (Provider__c.NPI__c), investigation outcome distribution (Substantiated / Unsubstantiated / Pending), and total recovered amount from Payment__c.Recovery_Amount__c.

Given the quarterly report is generated, When the PDF is created, Then a Regulatory_Filing__c record must be created with fields: Report_Type__c = 'Quarterly Fraud Report', Filing_Period__c, Report_Document__c (ContentVersion lookup), Filed_By__c, and Filing_Status__c = 'Draft'.

Given a provider appears in the top 10 flagged list, When a Compliance Officer clicks the provider's NPI hyperlink, Then the system must navigate to the Provider 360 FlexCard showing the Provider__c record with all associated Claim__c SIU referrals, Credentialing_Case__c status, and active PNA records visible on a single screen.

## Solution

Build the SIU Regulatory Dashboard as a CRM Analytics (Tableau CRM) dashboard with custom SAQL queries against synced Claim__c, Payment__c, and Provider__c datasets. Use an Agentforce Agent Action (FraudScoringAction) that invokes an Einstein Prediction Builder model trained on historical SIU referral data, writing scores back to Claim__c.Fraud_Score__c via a Record-Triggered Flow. Build a Record-Triggered Flow on Claim__c for auto-referral logic when Fraud_Score__c >= 75. Implement the 'Generate Quarterly Fraud Report' button as an LWC invoking a Conga CLM API callout via an Apex controller (SIUReportController). Create the Provider 360 FlexCard using OmniStudio FlexCards with DataRaptor Turbo Extract to aggregate Claim__c, Credentialing_Case__c, and PNA data. Apply Permission Set Group 'SIU_Compliance_Access' to restrict dashboard and SIU_Case__c access to authorized Compliance Officers and SIU staff.

## Description

Steps to Reproduce:
1. Create a new Provider_Change_Notification__c record with Change_Type__c = 'Address Change' and LOB__c = 'Medicare Advantage'.
2. Set Notification_Date__c to any valid date (e.g., 2025-03-15).
3. Save the record.
4. Observe the Regulatory_Deadline__c field — it is blank/null.
5. Repeat with LOB__c = 'Commercial' — Regulatory_Deadline__c correctly populates as Notification_Date__c + 30 days.
6. Repeat with LOB__c = 'Medicaid' — Regulatory_Deadline__c correctly populates as Notification_Date__c + 15 days.

Expected Behavior: For LOB__c = 'Medicare Advantage', Regulatory_Deadline__c should be auto-populated as Notification_Date__c + 10 calendar days per CMS Medicare Advantage network adequacy notification requirements (42 CFR §422.222).

Actual Behavior: Regulatory_Deadline__c remains null for all Medicare Advantage PCN records. The Record-Triggered Flow 'PCN_Regulatory_Deadline_Setter' contains a decision branch for 'Medicare Advantage' but the LOB__c picklist value used in the flow condition is 'Medicare Advantage' while the actual picklist API value stored in the field is 'Medicare_Advantage' (underscore variant introduced in Sprint 15 data migration INS-1487). The mismatch causes the branch to never evaluate to TRUE, falling through to the null default path.

Impact: Approximately 847 PCN records created since Sprint 15 go-live have null Regulatory_Deadline__c, creating compliance exposure for CMS reporting. Compliance Officer escalated as P1 during Sprint 17 retrospective.

## Acceptance Criteria

As per Description. Additionally verify that the fix includes a data fix script backfilling Regulatory_Deadline__c on all existing Provider_Change_Notification__c records where LOB__c = 'Medicare_Advantage' and Regulatory_Deadline__c = null, setting value to Notification_Date__c + 10. Confirm count of backfilled records matches the 847 identified in production analysis.

## Solution

Root cause confirmed: Flow decision element in 'PCN_Regulatory_Deadline_Setter' uses display label 'Medicare Advantage' instead of API value 'Medicare_Advantage' for the picklist comparison. Fix: (1) Update the Flow decision condition to use the correct API value 'Medicare_Advantage' in the picklist equals operator. (2) Activate updated Flow version 4. (3) Execute a one-time anonymous Apex data fix script in production (reviewed and approved via change management) to backfill Regulatory_Deadline__c = Notification_Date__c + 10 on all affected records, run in batch of 200 with a dry-run count validation step. (4) Add a unit test to the existing PCNFlowTest class asserting Regulatory_Deadline__c is populated correctly for all three LOB values including 'Medicare_Advantage'. (5) Post-fix, schedule a daily report on Provider_Change_Notification__c where Regulatory_Deadline__c = null as an ongoing data quality monitor.

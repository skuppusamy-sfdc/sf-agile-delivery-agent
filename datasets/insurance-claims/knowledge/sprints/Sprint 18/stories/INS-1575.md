## Description

As a Compliance Officer, I need a mandatory compliance checklist enforced on the Network_Termination_Case__c object that validates all state-mandated notice periods have been observed and required notifications have been sent before a termination case can be closed, so that Acme Insurance avoids regulatory penalties for improper provider termination procedures.

## Acceptance Criteria

Given a Network_Termination_Case__c.Status__c is set to 'Closed — Terminated',
When the record save is attempted,
Then a validation rule must confirm that: (1) Termination_Notice_Sent_Date__c is not null, (2) DATEDIFF(Termination_Effective_Date__c, Termination_Notice_Sent_Date__c) >= the minimum notice days from State_Compliance_Config__mdt for the provider's state, and (3) Member_Notification_Sent__c = true. If any condition fails, the save must be blocked with a field-specific error message.

Given the compliance checklist section on the Network_Termination_Case__c Lightning record page,
When a Provider Relations Specialist opens a termination case,
Then a Compliance_Checklist__c FlexCard must render showing each checklist item (Notice Sent, Member Notification, CMS Notification if Medicare Advantage, Continuity of Care Letter Issued) with status indicators (Complete/Incomplete/Waived) sourced from child Compliance_Checklist_Item__c records.

Given a termination involves a Medicare Advantage LOB provider (Policy__c.LOB__c = 'Medicare Advantage'),
When the termination case is created,
Then an additional Compliance_Checklist_Item__c record must be auto-created with Item_Type__c = 'CMS Notification' and Due_Date__c = Termination_Effective_Date__c minus 60 days per CMS requirements.

Given all Compliance_Checklist_Item__c records for a Network_Termination_Case__c have Status__c = 'Complete' or 'Waived',
When the last item is updated,
Then Network_Termination_Case__c.Compliance_Clearance__c must be automatically set to true via a Record-Triggered Flow, enabling the 'Close Case' button on the page layout.

Given a checklist item is set to 'Waived',
When the save is attempted,
Then Waiver_Justification__c (long text) must be populated and Waiver_Approved_By__c must reference a user with the 'Compliance_Manager' custom permission; otherwise the save must be blocked.

## Solution

Validation rules on Network_Termination_Case__c enforce notice period and notification requirements using State_Compliance_Config__mdt for dynamic thresholds. OmniStudio FlexCard 'Termination_Compliance_Checklist_FC' built with a DataRaptor Extract sourcing Compliance_Checklist_Item__c records. Record-Triggered Flow on Network_Termination_Case__c (after insert) evaluates LOB to auto-create Medicare Advantage CMS Notification checklist item. Separate Record-Triggered Flow on Compliance_Checklist_Item__c (after update) uses a Get Records element to check all sibling items and updates parent Compliance_Clearance__c. Custom permission 'Compliance_Manager' assigned via Permission Set Group. 'Close Case' button visibility controlled by a Dynamic Action condition on Compliance_Clearance__c = true.

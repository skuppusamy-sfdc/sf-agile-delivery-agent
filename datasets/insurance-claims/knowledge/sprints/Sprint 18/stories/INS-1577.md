## Description

As a Compliance Officer, I need a regulatory reporting dashboard that aggregates Claim__c and Payment__c data by Line of Business (LOB) and state jurisdiction so that I can produce accurate, timely submissions to state Departments of Insurance (DOI) without manual data extraction from multiple reports. This dashboard must surface key metrics including claim adjudication cycle times, denial rates by ICD-10 code category, and total disbursements by TIN, ensuring Acme Insurance meets all statutory reporting deadlines across all 12 active states.

## Acceptance Criteria

Given a Compliance Officer accesses the Regulatory Reporting Dashboard,
When the dashboard loads for a selected reporting period and state jurisdiction,
Then Claim__c records filtered by Claim__c.Jurisdiction__c and Claim__c.LOB__c are displayed with correct aggregation of total claim count, paid amount (Payment__c.Disbursement_Amount__c), and average adjudication cycle time (Claim__c.Adjudication_Cycle_Days__c).

Given claims with Claim__c.Status__c = 'Denied',
When grouped by ICD-10 diagnosis code (Claim__c.Primary_Diagnosis_Code__c),
Then denial rate percentage is calculated as (denied claims / total claims) * 100 and displayed in a sortable data table with drill-down to individual Claim__c records.

Given the user selects 'Export for DOI Submission',
When the export is triggered,
Then a CSV file is generated conforming to NAIC uniform reporting standards, including all required fields: Claim__c.Claim_Number__c, Provider__c.NPI__c, Provider__c.TIN__c, Payment__c.Disbursement_Date__c, and Claim__c.Denial_Reason_Code__c.

Given a reporting period is finalized and exported,
When the export completes successfully,
Then a Compliance_Report_Log__c record is created with fields: Report_Type__c = 'DOI Submission', Submitted_By__c (running user), Submission_Date__c, Record_Count__c, and Status__c = 'Exported'.

Given the dashboard is accessed by a user without the 'Compliance_Reporting' Permission Set Group,
When they attempt to navigate to the dashboard,
Then access is denied and a standard Salesforce 'Insufficient Privileges' page is displayed.

-Given the dashboard auto-submits to DOI portals via API integration, When submission completes, Then Compliance_Report_Log__c.Status__c = 'Auto-Submitted'.- (Descoped to Phase 3 per compliance team agreement)

## Solution

Build a custom LWC dashboard component (regulatoryReportingDashboard) leveraging Apex controllers with SOQL aggregation queries on Claim__c, Payment__c, and Provider__c objects. Implement a Report_Period__c custom object to store finalized reporting windows. Use a Record-Triggered Flow on Compliance_Report_Log__c to notify the Compliance Officer via custom notification when export completes. CSV generation handled via an Apex ContentVersion approach using Blob encoding. Permission access controlled via a dedicated 'Compliance_Reporting_PSG' Permission Set Group assigned to Compliance Officer profiles. Dashboard filters backed by custom metadata (Reporting_Jurisdiction_Config__mdt) to manage state-specific field mappings.

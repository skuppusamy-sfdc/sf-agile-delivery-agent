## Description

As a Credentialing Specialist, I need a real-time compliance dashboard and automated alert system for expiring provider credentials on Credentialing_Case__c records so that no provider with lapsed licensure, DEA certification, or malpractice insurance remains active in the network, thereby protecting Acme Insurance from regulatory exposure and CMS audit findings. Extends INS-1511.

## Acceptance Criteria

Given a Credentialing_Case__c record exists with Credential_Type__c = 'Medical License' | 'DEA Certificate' | 'Malpractice Insurance' | 'Board Certification', When the Expiration_Date__c is 90, 60, or 30 days from today, Then a Task is auto-created and assigned to the owning Credentialing Specialist with Subject = 'Credential Expiry Alert – [Credential_Type__c] – [Provider__r.Name]' and Due_Date__c set to 15 days before expiration.

Given a credential has expired (Expiration_Date__c < TODAY()), When the daily scheduled Flow runs at 6:00 AM EST, Then Credentialing_Case__c.Status__c is set to 'Expired', the linked Provider__c (Person Account) record has Active_In_Network__c set to FALSE, and all active Provider_Network_Agreement__c records for that provider have PNA_Status__c set to 'Suspended – Credential Lapse'.

Given a Credentialing Specialist accesses the 'Credential Compliance' FlexCard on the Provider 360 page, When the card loads, Then it displays a color-coded status ring: Green (>90 days), Yellow (31–90 days), Red (≤30 days or Expired), sourced from a DataRaptor Transform aggregating all Credentialing_Case__c records for the provider.

Given a CAQH re-attestation is completed and the new Expiration_Date__c is updated on Credentialing_Case__c, When the record is saved, Then if the new date is in the future and all other credentials are current, Active_In_Network__c is automatically reset to TRUE and PNA_Status__c reverts to 'Active' via a Record-Triggered Flow.

Given the Compliance Officer opens the 'Credentialing Compliance Dashboard' report, When the dashboard loads, Then it shows counts of credentials by status tier (Current / Expiring Soon / Expired), filterable by Provider_Specialty__c, State__c, and LOB__c, with drill-down to individual Credentialing_Case__c records.

## Solution

Scheduled Flow ('Daily_Credential_Expiry_Check') runs at 6:00 AM EST using a Get Records element on Credentialing_Case__c filtered by Expiration_Date__c <= TODAY(), then loops to update Status__c and fires a subflow to update Provider__c.Active_In_Network__c and bulk-update PNA records via Apex invocable action to handle SOQL relationship queries within governor limits. Proactive Task creation handled by a separate Record-Triggered Flow on Credentialing_Case__c triggering when Expiration_Date__c changes, using a formula to compute days-until-expiry. FlexCard 'CredentialComplianceCard' built in OmniStudio using a DataRaptor Extract for credential data and a DataRaptor Transform for color-code logic. CMS Reports dashboard built with Einstein Analytics (CRM Analytics) using a custom dataset sourced from Credentialing_Case__c and Provider__c. CAQH integration webhook updates Credentialing_Case__c via a named credential REST callout in Apex.

## Description

As a Compliance Officer, I need every status transition on Claim__c to be recorded in an immutable audit log object so that we can satisfy state DOI examination requests and internal SIU investigations without relying on Salesforce field history (which is limited to 18 months and subject to purge).

## Acceptance Criteria

Given a Claim__c record undergoes any Status__c change (e.g., 'Submitted' → 'In Review' → 'Adjudicated' → 'Paid'),
When the record is saved,
Then a new Claim_Audit_Event__c record must be created with: Claim__c (lookup), Old_Status__c, New_Status__c, Changed_By__c (User lookup), Change_Timestamp__c (datetime, system-set), Change_Reason__c (required picklist), and IP_Address__c populated via Apex.

Given a Claim_Audit_Event__c record is created,
When any user (including System Administrator) attempts to edit or delete it,
Then the operation must be blocked by an Apex trigger with error message: 'Audit records are immutable and cannot be modified or deleted.'

Given the audit trail is queried,
When a Compliance Officer opens the Claim 360 page,
Then a timeline LWC component must display all Claim_Audit_Event__c records in descending Change_Timestamp__c order, showing user full name, old/new status, reason, and timestamp.

-Given bulk claim adjudication via batch, audit records were to be written synchronously-
Given bulk claim adjudication runs via Batch Apex,
When more than 200 Claim__c records transition in a single execution,
Then audit events must be published via Platform Event (Claim_Status_Changed__e) and persisted asynchronously to avoid governor limit violations.

Given a DOI export is requested,
When a Compliance Officer runs the 'Claim Audit Export' report,
Then the export must include all Claim_Audit_Event__c fields and be available in CSV format with a date-range filter on Change_Timestamp__c.

## Solution

1. Create Claim_Audit_Event__c custom object with all required fields; set OWD to Private, grant read-only access via Compliance Officer Permission Set Group. 2. Build Record-Triggered Flow on Claim__c (After Save, on Status__c change) to publish Claim_Status_Changed__e Platform Event carrying old/new values, user ID, and timestamp. 3. Platform Event Triggered Flow (or Apex EventBus subscriber) inserts Claim_Audit_Event__c records. 4. Apex Trigger (before delete, before update) on Claim_Audit_Event__c throws AuraHandledException to enforce immutability. 5. LWC 'claimAuditTimeline' queries Claim_Audit_Event__c via @wire getRelatedListRecords and renders a lightning-timeline. 6. CRM Analytics (CRMA) dataset scheduled to refresh nightly for DOI export dashboard.

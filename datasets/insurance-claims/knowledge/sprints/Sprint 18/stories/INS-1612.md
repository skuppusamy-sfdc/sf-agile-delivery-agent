## Description

As a Compliance Officer, I need an automated data retention enforcement mechanism that flags Claim__c and Payment__c records that have exceeded their state-mandated retention period (configurable per LOB and state) and initiates a structured archival workflow so that the organization can demonstrate compliance with records retention regulations during audits while managing Salesforce data storage costs.

## Acceptance Criteria

Given a custom metadata type 'Retention_Policy__mdt' is configured with LOB__c, State__c, Object_API_Name__c, Retention_Years__c, and Action__c ('Archive'|'Delete'|'Legal_Hold'),
When the scheduled batch job 'RetentionPolicyEnforcement_Batch' runs weekly on Sunday at 01:00 UTC,
Then all Claim__c records where Claim_Closed_Date__c is older than (today minus Retention_Years__c * 365) AND Claim__c.LOB__c and Policy__c.State__c match the policy must be identified.

Given a Claim__c record is identified as past retention period,
When Action__c = 'Archive',
Then Claim__c.Retention_Status__c must be set to 'Pending Archive', a Retention_Review_Case__c must be created with all identifying metadata, and the record must NOT be deleted until the review case is closed.

Given a Claim__c record has Retention_Status__c = 'Pending Archive',
When a Compliance Officer sets the Retention_Review_Case__c.Status__c = 'Approved for Archive',
Then the Claim__c and all child Claim_Line__c, Payment__c, and Explanation_of_Benefits__c records must be exported to a ContentVersion archive package and Claim__c.Retention_Status__c updated to 'Archived'.

Given a Claim__c record is under active SIU investigation (Claim__c.SIU_Flag__c = true),
When the retention batch identifies it as past retention period,
Then the record must be placed in Retention_Status__c = 'Legal Hold' and must be excluded from all archival or deletion actions regardless of retention policy.

Given the retention batch completes a run,
When processing is finished,
Then a Batch_Execution_Log__c record must be created with: Records_Evaluated__c, Records_Flagged__c, Records_Archived__c, Records_On_Legal_Hold__c, Errors__c (JSON), and Execution_DateTime__c.

Given Retention_Policy__mdt has no matching record for a given LOB/State combination,
When a Claim__c record would otherwise be evaluated,
Then the record must be skipped and logged in Batch_Execution_Log__c.Unmatched_Policy_Records__c as a JSON array of Claim IDs.

## Solution

Build a Schedulable Batch Apex class 'RetentionPolicyEnforcement_Batch' implementing Database.Batchable<SObject>, Database.Stateful, and Database.AllowsCallouts. The start() method dynamically constructs SOQL based on all distinct LOB/State combinations from Retention_Policy__mdt records. The execute() method evaluates each Claim__c against its matched policy and performs DML updates to Retention_Status__c and inserts to Retention_Review_Case__c. The ContentVersion archive package in the finish() method is assembled using a JSON serialization of the Claim__c and all child records, stored as a single ContentVersion with NetworkId and LinkedEntityId pointing to the Retention_Review_Case__c. A Record-Triggered Flow on Retention_Review_Case__c handles the 'Approved for Archive' transition and invokes an Apex Action for the archive export. Custom Metadata Type 'Retention_Policy__mdt' is managed via a custom Setup Lightning App page for Compliance Officer self-service configuration.

## Description

**Bug Report**

**Environment:** Production (also reproduced in UAT)
**Reported by:** Claims Operations Manager (Emily Watson)

**Steps to Reproduce:**
1. Navigate to a Claim__c record where one or more Claim_Line__c child records have been manually deleted after adjudication (Status__c = 'Adjudicated').
2. Click the 'Generate EOB' button on the Claim Lightning Record Page.
3. The Conga CLM merge job is triggered via the 'GenerateEOBController' Apex class.
4. Observe the resulting error.

**Expected Behavior:**
The EOB document is generated successfully. If Claim_Line__c records are missing, the EOB should either (a) include only the lines present or (b) display a user-friendly validation error stating 'One or more claim lines are missing. Please contact Claims Operations before generating the EOB.'

**Actual Behavior:**
Apex throws an unhandled NullPointerException: 'Attempt to de-reference a null object' at GenerateEOBController.buildLineItemMap() line 47. The error is surfaced as a generic 'An internal server error has occurred' toast to the end user with no actionable guidance. No Explanation_of_Benefits__c record is created, but the EOB generation Claim__c.EOB_Status__c field is incorrectly set to 'Generated' despite the failure.

**Impact:** High — Claims Operations team cannot generate EOBs for ~34 affected claims in production. Regulatory prompt-pay clock continues to run.

**Root Cause (Preliminary):** GenerateEOBController.buildLineItemMap() iterates over a List<Claim_Line__c> that is populated from a SOQL query result and then accessed by index without null/empty-list guard. Additionally, the Claim__c.EOB_Status__c field update is performed before the Conga callout result is confirmed.

## Acceptance Criteria

As per Description.

Additional fix validation:
- Claim__c.EOB_Status__c must only be set to 'Generated' AFTER a successful HTTP 200 response is received from the Conga CLM API and an Explanation_of_Benefits__c record is successfully inserted.
- A null/empty guard must be added to GenerateEOBController.buildLineItemMap() so that if the Claim_Line__c list is null or empty, the method throws a custom ClaimsProcessingException with message code 'EOB_MISSING_LINES' rather than a NullPointerException.
- The user-facing toast on failure must display the message: 'EOB generation failed: One or more claim lines are missing. Reference Claim ID: [Claim__c.Name]. Contact Claims Operations.'
- A unit test must achieve ≥ 85% code coverage on the updated GenerateEOBController class, including a negative test case with an empty Claim_Line__c list.

## Solution

Fix GenerateEOBController.buildLineItemMap() to add an explicit null and isEmpty() check on the claimLines list before iteration. Extract the Claim__c.EOB_Status__c DML update into a separate private method 'updateEOBStatus()' called only within the Conga API success branch (HTTP 200 + Explanation_of_Benefits__c insert confirmed). Add a custom exception class 'ClaimsProcessingException' if not already present. Update the LWC 'generateEobButton' to handle the new exception message code and render the user-friendly toast. Deploy a hotfix branch through the standard CI/CD pipeline (Salesforce DX scratch org validation → UAT → Production). Backfill the 34 affected claims by resetting EOB_Status__c to 'Pending' via an anonymous Apex script executed by a System Admin.

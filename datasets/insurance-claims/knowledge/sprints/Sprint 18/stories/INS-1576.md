## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to generate concise, compliance-aware claim summaries on demand from the Claim__c record page — including flagging potential SIU referral indicators, COB coordination gaps, and ICD-10/CPT coding anomalies — so that claims examiners can make faster, better-informed adjudication decisions while maintaining a full audit trail of every AI-generated output for regulatory review.

## Acceptance Criteria

Given a Claims Examiner activates the Agentforce panel on a Claim__c record page,
When they click 'Generate Claim Summary',
Then the AI must produce a structured summary including: Claim overview (Claim__c.Claim_Number__c, Member name, Provider NPI, Total_Billed_Amount__c), top 3 Claim_Line__c items by billed amount with CPT codes and ICD-10 diagnosis codes, current Claim__c.Status__c, and any open SIU_Referral__c records linked to the claim.

Given the AI summary is generated,
When the output is rendered,
Then if any Claim_Line__c.CPT_Code__c appears on the Fraud_Indicator_CPT__mdt list, the summary must include a highlighted SIU flag with the specific CPT code and a suggested action of 'Consider SIU Referral'.

Given the AI summary is displayed,
When the Claims Examiner views COB information,
Then the summary must check Policy__c.COB_Flag__c = true and display all related COB_Coordination__c records; if COB_Coordination__c.Secondary_Payer_Response__c is null and Policy__c.COB_Flag__c = true, the summary must flag 'COB Response Pending — Hold Payment'.

Given any AI summary is generated (regardless of user action),
When the generation completes successfully or fails,
Then an AI_Interaction_Log__c record must be created capturing: Claim__c (lookup), User__c, Prompt_Hash__c (SHA-256 of input prompt, not raw prompt for PII compliance), Response_Summary__c (first 500 chars), Model_Version__c, Generation_Timestamp__c, and Compliance_Reviewed__c = false (default).

Given an AI_Interaction_Log__c is created,
When a Compliance Officer performs a periodic AI audit,
Then they must be able to set Compliance_Reviewed__c = true and add Reviewer_Notes__c; AI_Interaction_Log__c records must be read-only for all other profiles.

Given the AI model returns a response with confidence below threshold,
When Agentforce_Confidence_Score__c < 0.75 (configurable via Agentforce_Config__mdt),
Then the summary must display a prominent disclaimer 'AI confidence below threshold — manual review required' and Claim__c.AI_Review_Required__c must be set to true.

## Solution

Agentforce Agent configured with a custom 'Claims Summary' Agent Action backed by an Apex invocable class that assembles claim context (Claim__c, Claim_Line__c, Policy__c, SIU_Referral__c, COB_Coordination__c) into a structured prompt. Einstein Generative AI (GPT-4o via Salesforce Model Garden) processes the prompt. Response parsed in Apex and rendered in a custom LWC 'claimAISummaryPanel' embedded in the Claim 360 Lightning page. AI_Interaction_Log__c inserted via a separate Apex method in a future context to avoid blocking the UI response. Prompt_Hash__c generated using Crypto.generateDigest('SHA-256', ...) Apex method. Fraud_Indicator_CPT__mdt evaluated in the Apex invocable before prompt submission to inject pre-computed flags. Agentforce_Config__mdt stores model version, confidence threshold, and max token limit. Permission Set 'Agentforce_Claims_User' controls access to the AI panel component.

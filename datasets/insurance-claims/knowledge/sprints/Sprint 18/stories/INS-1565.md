## Description

As a Provider Network Specialist, I need automated notifications triggered at 90, 60, and 30 days before a Provider_Network_Agreement__c (PNA) expiration date so that my team can proactively initiate renewal workflows and avoid lapses in network coverage that could result in regulatory non-compliance or claims payment errors.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record has PNA_Expiry_Date__c populated and PNA_Status__c = 'Active',
When today's date is exactly 90, 60, or 30 days before PNA_Expiry_Date__c,
Then a Task must be auto-created assigned to PNA_Owner__c (lookup to User) with Subject = 'PNA Renewal Required — [Provider Name] — [X] Days to Expiry', Priority = 'High', and ActivityDate = today.

Given the 90-day notification Task is created,
When it is created,
Then an in-app notification (Custom Notification Type 'PNA_Expiry_Alert') must also be sent to the assigned Provider Network Specialist and their queue 'Provider_Network_Renewals'.

Given a PNA_Expiry_Date__c passes with PNA_Status__c still = 'Active',
When the scheduled job runs the following morning,
Then PNA_Status__c must be automatically set to 'Expired', a Compliance_Event__c record must be created with Event_Type__c = 'PNA Lapsed', and all active Claim_Line__c records referencing this PNA via Claim_Line__c.PNA__c must have a Warning_Flag__c = true stamped.

Given a PNA is renewed (PNA_Status__c transitions from 'Expired' or 'Pending Renewal' to 'Active'),
When the record is saved,
Then all Warning_Flag__c = true Claim_Line__c records associated with the previously expired PNA must have Warning_Flag__c reset to false via an asynchronous Apex Queueable job.

## Solution

Implement a Scheduled Apex job (PNAExpiryNotificationBatch) running nightly at 06:00 UTC, querying Provider_Network_Agreement__c records where PNA_Expiry_Date__c IN (TODAY+90, TODAY+60, TODAY+30) and PNA_Status__c = 'Active'. Use Messaging.CustomNotification API for in-app alerts. A separate batch scope handles lapsed PNAs and bulk-updates Claim_Line__c via a Queueable chain (PNALapseClaimLineFlagQueueable) to avoid governor limits. Record-Triggered Flow on Provider_Network_Agreement__c (after update, PNA_Status__c changed to 'Active') enqueues the reset Queueable. Custom Notification Type configured in Setup. All batch jobs logged to a Batch_Execution_Log__c custom object for audit purposes.

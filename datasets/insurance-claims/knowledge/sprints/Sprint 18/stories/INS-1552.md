## Description

As a Compliance Officer, I need Salesforce Shield Field Audit Trail (FAT) enabled and configured for all Protected Health Information (PHI) fields on Claim__c, Policy__c, and Explanation_of_Benefits__c objects so that we maintain a 10-year immutable history of PHI field changes as required by HIPAA and our state DOI consent decree.

This story supersedes the S3 export requirement deprecated in INS-1548.

## Acceptance Criteria

Given Shield Field Audit Trail is provisioned in the production org,
When the configuration is complete,
Then the following fields must be tracked (retained for 10 years via custom retention policy): Claim__c.Diagnosis_Code_Primary__c, Claim__c.Claimant_SSN_Encrypted__c, Claim__c.Date_of_Service__c, Policy__c.Member_DOB__c, Policy__c.Member_SSN_Encrypted__c, Policy__c.Coverage_Effective_Date__c, Explanation_of_Benefits__c.Paid_Amount__c, Explanation_of_Benefits__c.Patient_Responsibility__c.

Given a user with the 'Claims_Adjudicator' profile edits Claim__c.Diagnosis_Code_Primary__c,
When the change is saved,
Then the Field History entry must capture: field name, old value, new value, user ID, and timestamp — queryable via the FieldHistoryArchive object in Salesforce's BigObjects infrastructure.

Given a Compliance Officer needs to produce a PHI access report for a DOI audit,
When they run the 'PHI Field Change Report' in the Compliance Reports folder,
Then the report must query FieldHistoryArchive filtered by object type and date range, returning results within 30 seconds for date ranges up to 90 days.

-Given FAT data is mirrored to an external SIEM via nightly batch- (deprecated in favor of Salesforce Event Monitoring streaming to Splunk, tracked in platform team backlog).

Given Shield Platform Encryption is active on Claim__c.Claimant_SSN_Encrypted__c and Policy__c.Member_SSN_Encrypted__c,
When FAT records these fields,
Then the archived old/new values must remain encrypted at rest and only decryptable by users with the 'View Encrypted Data' permission in the Compliance_Officer Permission Set Group.

## Solution

Work with the Salesforce Account Executive / Shield provisioning team to enable Field Audit Trail with a 10-year custom retention policy in the production org (Setup → Field Audit Trail). Configure field tracking via the Salesforce Metadata API (HistoryRetentionPolicy on each target object). Validate FieldHistoryArchive queryability via SOQL in Developer Console. Build a custom report type 'PHI Field Change History' using a DataRaptor Extract (OmniStudio) that queries FieldHistoryArchive with indexed filters on ParentId and CreatedDate. Confirm Shield Platform Encryption key management policies with the security team to ensure encrypted field values are stored and retrieved correctly within FAT. Document the configuration in the Compliance Runbook stored in Salesforce CMS.

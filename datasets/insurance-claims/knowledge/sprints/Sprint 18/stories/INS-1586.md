## Description

As a Credentialing Specialist, I need automated alerts when a Provider's (Provider__c / Person Account) credentialing documents are approaching expiration and for each alert event to be written to the compliance audit log so that we can demonstrate proactive oversight to auditors and prevent lapsed providers from appearing in-network.

## Acceptance Criteria

Given a Credentialing_Case__c record has a License_Expiration_Date__c field populated,
When the system date is 90, 60, or 30 days before License_Expiration_Date__c,
Then a Task must be created assigned to the Credentialing_Case__c.Owner with Subject = 'License Expiring in [X] days – [Provider Name]' and a Compliance_Alert_Log__c record must be inserted with Alert_Type__c = 'License Expiration', Triggered_Date__c = today, and Related_Provider__c lookup populated.

Given a CAQH re-attestation is due (CAQH_Reattestion_Due_Date__c on Provider__c),
When the date is within 45 days of expiration,
Then an automated outreach email must be sent to the Provider's Primary_Email__c using the 'CAQH Re-attestation Reminder' email template and logged to Compliance_Alert_Log__c.

Given a provider's Credentialing_Case__c.Status__c transitions to 'Lapsed',
When the transition is saved,
Then Provider__c.Network_Status__c must be automatically set to 'Suspended – Credentialing Lapsed' and all active Provider_Network_Agreement__c records for that provider must have PNA_Status__c set to 'Suspended'.

-Given CAQH API polling was scheduled hourly-
Given CAQH API polling is scheduled every 6 hours,
When a provider's CAQH profile attestation status returns 'Expired',
Then Credentialing_Case__c.CAQH_Status__c must be updated to 'Expired' and the alert pipeline above must be triggered.

## Solution

1. Scheduled Flow runs nightly querying Credentialing_Case__c records where License_Expiration_Date__c is within 90/60/30 days; creates Tasks and Compliance_Alert_Log__c via Flow Create Records elements. 2. Record-Triggered Flow on Credentialing_Case__c (After Save, Status__c = 'Lapsed') updates Provider__c.Network_Status__c and fires sub-flow to bulk-update related PNA records. 3. Apex Schedulable class calls CAQH REST API every 6 hours using Named Credential; parses response and updates Credentialing_Case__c fields; invokes alert Flow via Flow.Interview. 4. Email alerts use org-wide email address and a custom HtmlEmailTemplate. 5. Compliance_Alert_Log__c OWD set to Private; Credentialing Specialist Profile granted Read access via Sharing Rule.

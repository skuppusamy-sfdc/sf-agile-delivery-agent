## Description

As a Compliance Officer, I need Claim_Audit_Log__c records older than 7 years to be automatically archived to a Salesforce Big Object (Claim_Audit_Archive__b) while remaining queryable for regulatory audit purposes, so that Acme Insurance maintains DOI-mandated 7-year audit retention without degrading org storage performance or standard object query performance. This story addresses the retention period correction noted in INS-1592 acceptance criteria.

## Acceptance Criteria

Given a scheduled archival job runs on the first Sunday of each month at 2:00 AM, When the job executes, Then all Claim_Audit_Log__c records where Claim_Audit_Log__c.Change_Timestamp__c < TODAY() - 2555 days (7 years) are copied to the Claim_Audit_Archive__b Big Object with all corresponding fields mapped, and the source Claim_Audit_Log__c records are deleted only after successful Big Object insertion is confirmed.

Given the archival job encounters an insertion failure for any Claim_Audit_Archive__b record, When the failure occurs, Then the corresponding Claim_Audit_Log__c record is NOT deleted, an Archival_Error_Log__c record is created with the error message and affected record ID, and the Compliance Officer is notified via Salesforce In-App Notification.

Given a Compliance Officer needs to retrieve archived audit records during a DOI examination, When they use the 'Archived Audit Search' OmniScript (accessible from the Compliance App), Then they can search Claim_Audit_Archive__b by Claim__c ID, date range, and Changed_By__c, with results displayed in a paginated FlexCard (Big Object SOQL requires index-based querying — search must include Claim__c ID as a leading index field).

Given the archival job completes a run, When the job finishes, Then an Archival_Run_Summary__c record is created capturing: total records evaluated, total records archived, total records failed, job duration in seconds, and Archival_Run_Summary__c.Status__c = 'Completed' or 'Completed with Errors'.

Given a Claim_Audit_Archive__b record exists, When any user or process attempts to delete it, Then the operation fails with error 'Archived compliance records cannot be deleted — contact Compliance Officer' enforced by org-wide Big Object immutability (Big Objects natively do not support delete; document this constraint in solution notes).

## Solution

Define Claim_Audit_Archive__b Big Object with indexed fields: Claim__c__c (text, index position 1), Change_Timestamp__c (datetime, index position 2), Changed_By__c (text, index position 3). Remaining fields are non-indexed. Scheduled Apex (ClaimAuditArchivalScheduler) implements Schedulable and calls a Batchable class (ClaimAuditArchivalBatch) with batch size of 200. Batch execute() method performs Database.insertImmediate() for Big Object records (required API for Big Objects), checks for success, then deletes source records. Error handling writes to Archival_Error_Log__c via a separate DML in a new transaction. OmniScript 'ArchivedAuditSearch' uses a DataRaptor Integration Procedure calling a custom Apex REST endpoint that executes SOQL against Claim_Audit_Archive__b using mandatory index fields. FlexCard renders paginated results with export to CSV. Note: Big Objects do not support triggers, workflow rules, or standard reports — all access must be via Apex or SOQL with indexed fields.

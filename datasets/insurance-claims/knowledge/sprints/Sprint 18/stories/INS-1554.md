## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to generate compliant, audit-logged summaries of complex multi-line claims (including COB, SIU flags, and ICD-10/CPT details) so that adjusters can rapidly understand claim context without manually reviewing 20+ child records, while ensuring every AI-generated summary is traceable for regulatory audit purposes.

## Acceptance Criteria

Given an adjuster opens a Claim__c record with 5 or more Claim_Line__c child records,
When they invoke the Agentforce 'Summarize Claim' action from the utility bar,
Then the AI-generated summary must include: total billed vs. allowed vs. paid amounts (aggregated from Claim_Line__c and Payment__c), primary and secondary ICD-10 codes with plain-language descriptions, COB indicator (Claim__c.COB_Flag__c = TRUE/FALSE), SIU referral status (Claim__c.SIU_Referral__c), and the rendering provider's NPI and network status.

Given the Agentforce action generates a summary,
When the response is returned to the adjuster,
Then an AI_Interaction_Log__c record must be created with fields: Claim__c (lookup), User__c, Prompt_Hash__c (SHA-256 of the prompt for PII avoidance), Response_Summary__c (first 255 chars), Full_Response_URL__c (link to Salesforce File storing full text), Timestamp__c, and Model_Version__c.

Given the Agentforce prompt is constructed,
When Claim__c.Claimant_SSN_Encrypted__c or Policy__c.Member_DOB__c fields are in scope,
Then these fields must be explicitly excluded from the prompt payload; the Apex prompt-builder class must mask or omit PHI fields before submitting to the Einstein LLM endpoint.

Given an AI_Interaction_Log__c record is created,
When a Compliance Officer queries the 'Agentforce Audit Log' report,
Then they must be able to filter by date range, user, and Claim__c.LOB__c and export results for DOI submission.

Given the Agentforce model returns a response with confidence score < 0.75,
When the summary is displayed,
Then a disclaimer banner must appear: 'AI confidence is below threshold. Please verify claim details manually before taking action.' and Claim__c.AI_Review_Required__c must be set to TRUE.

## Solution

Configure an Agentforce Agent in Setup with a custom Agent Action 'SummarizeClaim' backed by an Apex class 'ClaimSummaryPromptBuilder' that queries Claim__c, Claim_Line__c, Payment__c, Policy__c, and Provider__c (excluding PHI fields via an explicit field exclusion list). The prompt template is stored in a Custom Metadata Type 'Agentforce_Prompt_Config__mdt' for maintainability. The Apex class calls Einstein Generative AI APIs via ConnectApi or HTTP callout to the configured LLM endpoint (Named Credential). Post-response, an Apex method persists the AI_Interaction_Log__c record and conditionally sets Claim__c.AI_Review_Required__c via a DML update. A LWC 'claimAISummaryPanel' renders the response in the utility bar with the confidence-score disclaimer logic. PHI field exclusion is unit-tested with a dedicated test class asserting that SSN and DOB fields are never present in the constructed prompt string.

## Description

As a Provider Relations Specialist, I need the system to automatically track and enforce the 90-day advance notice requirement for provider network terminations so that Acme Insurance complies with state-mandated termination notice regulations and avoids regulatory penalties. When a Network_Termination_Case__c is initiated, the system must calculate the earliest permissible termination effective date, generate a DocuSign-enabled termination notice to the provider, and prevent any Provider_Network_Agreement__c from being set to 'Terminated' status before the 90-day notice period has elapsed. The Provider Change Notification (PCN) workflow must also be triggered to notify downstream systems of the pending termination.

## Acceptance Criteria

Given a Network_Termination_Case__c record is created with Termination_Type__c = 'Voluntary' or 'Involuntary' and Termination_Initiated_Date__c is set,
When the record is saved,
Then Earliest_Effective_Termination_Date__c is automatically calculated as Termination_Initiated_Date__c + 90 days, and a validation rule prevents any value less than this date from being saved to Requested_Termination_Date__c.

Given the Network_Termination_Case__c is saved and Earliest_Effective_Termination_Date__c is populated,
When a DocuSign envelope is generated via the 'Send Termination Notice' action,
Then the envelope is sent to the email address on Provider__c.PersonEmail using the 'Provider_Termination_Notice_v2' DocuSign template, pre-populated with Provider__c.Name, Provider__c.NPI__c, PNA__c.Agreement_Number__c, and Earliest_Effective_Termination_Date__c.

Given the DocuSign envelope status returns 'Completed' (provider has signed/acknowledged),
When the DocuSign status webhook updates the Network_Termination_Case__c,
Then Network_Termination_Case__c.Notice_Acknowledged__c = TRUE, Notice_Acknowledged_Date__c = current Date, and Case Status__c advances to 'Notice Period Active'.

Given Network_Termination_Case__c.Status__c = 'Notice Period Active' and today's date is before Earliest_Effective_Termination_Date__c,
When a user attempts to set Provider_Network_Agreement__c.Status__c = 'Terminated' on the related PNA,
Then a validation rule blocks the save with message: 'Provider Network Agreement cannot be terminated before the 90-day notice period expires on [Earliest_Effective_Termination_Date__c].'

Given the Earliest_Effective_Termination_Date__c is reached,
When the daily scheduled job runs,
Then a Provider_Change_Notification__c (PCN) record is created with PCN_Type__c = 'Termination', Effective_Date__c = Earliest_Effective_Termination_Date__c, and Status__c = 'Pending Distribution', and the related Provider_Network_Agreement__c.Status__c is updated to 'Terminated'.

-Given Termination_Type__c = 'For Cause', When the case is created, Then the 90-day notice period is waived and immediate termination is permitted.- (Revised: For Cause terminations now require Legal team approval via a separate Case escalation — see INS-1590 for For Cause workflow)

## Solution

Record-Triggered Flow on Network_Termination_Case__c (fires on Create) computes Earliest_Effective_Termination_Date__c using a formula element (Termination_Initiated_Date__c + 90). DocuSign envelope generation triggered via a Screen Flow action invoking the DocuSign for Salesforce managed package 'Send Envelope' invocable action, with template mapping defined in DocuSign Template Mapping custom metadata (DocuSign_Template_Mapping__mdt). DocuSign webhook handled by a public REST endpoint (DocuSignWebhookController) updating the Network_Termination_Case__c via a without-sharing service class. Validation rule on Provider_Network_Agreement__c references the parent Network_Termination_Case__c via a cross-object formula. Scheduled Apex (TerminationEffectiveDateJob) runs nightly to create PCN records and update PNA statuses for cases reaching their effective date.

## Description

As a Provider Relations Specialist, I need an automated compliance workflow on Network_Termination_Case__c that enforces the mandatory 60-day advance written notice requirement and triggers regulatory filing tasks so that Acme Insurance meets state-mandated provider termination notice obligations and avoids DOI penalties.

Related to INS-1550 (PNA expiry compliance).

## Acceptance Criteria

Given a Network_Termination_Case__c record is created with Termination_Type__c = 'Involuntary',
When the record is saved,
Then the system must automatically set Regulatory_Notice_Due_Date__c = CreatedDate + 60 days AND create a Task assigned to the case owner with Subject = 'Send Regulatory Termination Notice – [Provider Name]', Due_Date = CreatedDate + 5 days, Priority = 'High'.

Given the Regulatory_Notice_Due_Date__c is within 10 days and Network_Termination_Case__c.Notice_Sent__c = FALSE,
When the daily scheduled flow runs,
Then an escalation email must be sent to the Provider Relations Specialist and their manager (User.ManagerId lookup), and Network_Termination_Case__c.Escalation_Flag__c must be set to TRUE.

Given a Provider Relations Specialist clicks 'Send Notice' on the Network_Termination_Case__c record,
When DocuSign envelope is dispatched to Provider__c.Email__c,
Then Network_Termination_Case__c.Notice_Sent__c must be set to TRUE, Notice_Sent_Date__c = TODAY(), and a ContentVersion (PDF) of the notice letter must be attached to the case record.

Given Network_Termination_Case__c.Notice_Sent__c = TRUE AND Termination_Effective_Date__c = TODAY(),
When the termination becomes effective,
Then Provider_Network_Agreement__c.Status__c must be updated to 'Terminated', Provider__c.Network_Status__c must be updated to 'Out of Network', and all open Claim__c records referencing that provider with Service_Date__c > Termination_Effective_Date__c must have Claim__c.Compliance_Hold__c = TRUE.

Given Termination_Type__c = 'Involuntary' AND Termination_Reason__c IN ('Quality of Care', 'Fraud', 'License Revocation'),
When the termination is finalized,
Then a Regulatory_Submission__c record must be created with Submission_Type__c = 'Provider Termination Report' and a Task assigned to the Compliance Officer to file with the state DOI within 15 days per regulatory requirement.

Given a Network_Termination_Case__c is created for a provider credentialed across multiple LOBs,
When the termination is processed,
Then ALL associated Provider_Network_Agreement__c records (regardless of LOB__c) linked to that Provider__c must be evaluated and individually updated, not just the PNA referenced on the termination case.

## Solution

Implement a Record-Triggered Flow on Network_Termination_Case__c (after insert, after update) with decision branches for Termination_Type__c. The flow handles Task creation, date calculations, and field updates. A Scheduled Flow runs daily for escalation checks using a Get Records element filtered on Regulatory_Notice_Due_Date__c and Notice_Sent__c. DocuSign integration uses the existing DocuSign for Salesforce managed package; a custom Apex invocable method 'SendTerminationNoticeInvocable' is called from the flow to trigger envelope dispatch and attach the returned PDF as ContentVersion. A separate Record-Triggered Flow on Network_Termination_Case__c (after update, fires when Status__c changes to 'Terminated') uses a Loop element to iterate over and update all related PNA records and a subflow to bulk-update Claim__c records via an Apex invocable to avoid flow DML governor limits. Permission Set Group 'Provider_Termination_Processor' controls access to the Send Notice button (LWC quick action).

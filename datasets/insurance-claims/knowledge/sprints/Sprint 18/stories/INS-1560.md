## Description

As a Compliance Officer, I need the system to generate an IRS 1099-MISC/1099-NEC regulatory report for all providers (Provider__c Person Accounts and Facility__c Business Accounts) who received aggregate Payment__c disbursements exceeding $600 in a calendar year, with validated TIN data, so that Acme Insurance meets IRS reporting obligations and avoids B-Notice penalties.

## Acceptance Criteria

Given the Compliance Officer initiates the 1099 Report generation OmniScript for a selected Tax_Year__c, When the report runs, Then the system must aggregate Payment__c.Net_Paid_Amount__c by Provider__c or Facility__c where Payment__c.Payment_Status__c = 'Paid' and Payment__c.Payment_Date__c falls within the selected calendar year, returning only payees with aggregate total >= $600.00.

Given the aggregate query returns payee records, When TIN data is evaluated, Then any Provider__c or Facility__c with TIN__c = null or TIN_Verification_Status__c != 'Verified' must be flagged with a 1099_Exception__c record with Exception_Type__c = 'Missing or Unverified TIN' and excluded from the final submission file.

Given the 1099 exception list is generated, When the Compliance Officer reviews it, Then they must be able to trigger a bulk TIN verification request to the IRS TIN Matching API (via Named Credential 'IRS_TIN_Match') directly from the OmniScript, with results updating Provider__c.TIN_Verification_Status__c asynchronously.

Given all TIN validations are resolved, When the Compliance Officer clicks 'Generate Submission File', Then the system must produce an IRS-compliant 1099 flat file (Publication 1220 format) via a Conga CLM template and store it as a ContentVersion record linked to the Compliance_Report_Run__c object with File_Type__c = '1099_Submission'.

-Given 1099 data is exported directly to the legacy FTP server 'acme-irs-ftp.internal'- (deprecated: IRS now requires FIRE system direct upload; FTP export removed per compliance directive CD-2024-07)

Given the submission file is generated, When the Compliance_Report_Run__c record is created, Then it must capture: Report_Type__c = '1099', Tax_Year__c, Total_Payees__c, Total_Amount_Reported__c, Generated_By__c, Generated_Date__c, and Submission_Status__c = 'Pending Upload'.

Given a provider appears on the 1099 report, When their Provider 360 FlexCard is viewed by a Compliance Officer, Then a '1099 Issued' badge must display for the applicable tax year with a link to the Compliance_Report_Run__c.

## Solution

OmniScript '1099ReportWizard' with DataRaptor Turbo Extract for aggregate Payment__c SOQL using GROUP BY Provider__c, Facility__c with HAVING clause for $600 threshold. Apex class '1099TINValidationService' handles bulk IRS TIN Matching API callouts via Named Credential with Queueable chaining for large provider sets. Conga CLM REST API generates the Publication 1220 flat file from a pre-built template. ContentVersion inserted and linked to Compliance_Report_Run__c (custom object). Custom Metadata Type '1099_Config__mdt' stores tax year thresholds and IRS endpoint configuration for environment portability. FlexCard '1099BadgeComponent' conditionally renders based on a DataRaptor lookup for Compliance_Report_Run__c records matching the provider and year.

## Description

**Bug Report — INS-1566**

**Summary:** Claim_Audit_Event__c records created when Claim__c records are bulk-updated via Salesforce Data Loader are missing the Trigger_Source__c field value, leaving it null instead of stamping 'Integration'.

**Steps to Reproduce:**
1. Prepare a CSV with 10 Claim__c records with Status__c = 'Pending Review'.
2. Use Salesforce Data Loader (v57) to update Status__c to 'Adjudicated' for all 10 records.
3. Navigate to any of the updated Claim__c records and open the Audit Trail tab.
4. Observe the newly created Claim_Audit_Event__c records.

**Expected Behavior:** Trigger_Source__c should be populated with 'Integration' for all Data Loader-originated updates, consistent with the logic defined in INS-1563.

**Actual Behavior:** Trigger_Source__c is null on all 10 Claim_Audit_Event__c records created by the Data Loader operation. This creates a compliance gap as regulators cannot distinguish automated bulk updates from user-initiated changes.

**Environment:** Full Sandbox (ACME-FS2)
**Salesforce Version:** Winter '25
**Frequency:** 100% reproducible
**Impact:** Compliance Officer cannot certify audit trail completeness for DOI submission (blocks INS-1564).

## Acceptance Criteria

As per Description. Additionally: after fix, re-run Data Loader update of 10 Claim__c records and confirm all resulting Claim_Audit_Event__c.Trigger_Source__c = 'Integration'. Verify user-initiated single-record saves still produce Trigger_Source__c = 'User'. Verify Flow-triggered updates produce Trigger_Source__c = 'Flow'.

## Solution

Root cause: the AuditTrailService Apex class uses UserInfo.getUserType() to detect trigger source but Data Loader sessions return 'Standard' user type identical to interactive users. Fix: implement detection logic using Request.getCurrent().getQuiddity() method (available in API v49+). Map Quiddity values: SYNCHRONOUS (interactive user) → 'User', BATCH_APEX / FUTURE / QUEUEABLE → 'Apex', REMOTE_ACTION / AURA_ENABLED / VF → 'User', BULK_API / BULK_API_V2 → 'Integration'. Update AuditTrailService.determineTriggerSource() method accordingly and add unit test coverage for each Quiddity branch (minimum 90% coverage). Deploy to ACME-FS2 sandbox for regression testing before production push.

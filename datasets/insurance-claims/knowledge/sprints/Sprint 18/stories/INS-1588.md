## Description

As a Provider Relations Specialist, I need the system to automatically validate a provider's NPI and TIN against the NPPES registry when a Provider_Network_Agreement__c is created or when Provider__c.NPI__c or Provider__c.TIN__c fields are updated so that we maintain a compliant, accurate provider directory and avoid claims payment to unverified providers.

## Acceptance Criteria

Given a Provider__c record is created or has NPI__c updated,
When the record is saved,
Then an asynchronous Apex queueable job must call the NPPES NPI Registry API (api.nppes.cms.hhs.gov) using the stored Named Credential 'NPPES_API',
And Provider__c.NPPES_Validation_Status__c must be set to 'Pending' immediately on save,
And updated to 'Validated', 'Mismatch', or 'Not Found' based on API response within 5 minutes.

Given the NPPES API returns a 'Mismatch' for Provider__c.TIN__c versus the registry value,
When NPPES_Validation_Status__c is set to 'Mismatch',
Then a Provider_Change_Notification__c (PCN) record must be auto-created with PCN_Type__c = 'Data Discrepancy', PCN_Status__c = 'Open', and assigned to the Provider Relations Specialist queue.

Given a Provider_Network_Agreement__c is being activated (PNA_Status__c changing to 'Active'),
When Provider__c.NPPES_Validation_Status__c is not 'Validated',
Then the PNA activation must be blocked with validation error: 'Provider NPI/TIN must be validated against NPPES before activating a Network Agreement.'

Given the NPPES API is unavailable (HTTP 5xx or timeout),
When the callout fails,
Then Provider__c.NPPES_Validation_Status__c must be set to 'Validation Error', a retry must be attempted after 15 minutes (up to 3 retries), and a Platform Event must be published to alert the integration monitoring dashboard.

Given validated provider data,
When NPPES_Validation_Status__c = 'Validated',
Then Provider__c.NPPES_Last_Validated_Date__c must be stamped with the current datetime and Provider__c.NPPES_Validated_Name__c must be populated with the legal name returned by the API.

## Solution

1. Record-Triggered Flow on Provider__c (After Save, NPI__c or TIN__c changed) sets NPPES_Validation_Status__c = 'Pending' and enqueues NPPESValidationQueueable Apex class. 2. NPPESValidationQueueable performs HTTP callout to NPPES REST API via Named Credential; parses JSON response; updates Provider__c fields via DML. 3. On mismatch, Apex inserts Provider_Change_Notification__c and assigns to queue via Group lookup. 4. Validation Rule on Provider_Network_Agreement__c prevents activation unless related Provider__c.NPPES_Validation_Status__c = 'Validated'. 5. Retry logic implemented using Schedulable Apex with exponential backoff stored in NPPES_Retry_Log__c custom object. 6. Platform Event (Integration_Alert__e) published on failure; monitored via existing integration health FlexCard on ops dashboard.

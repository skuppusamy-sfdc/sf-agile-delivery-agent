## Description

As a Claims Operations Manager, I need Agentforce to analyze in-flight Claim__c records during adjudication and surface real-time compliance flags and SIU referral recommendations directly within the claims workspace so that adjusters can make informed decisions faster, reduce false-negative fraud escapes, and maintain a documented AI-assisted review trail that satisfies compliance requirements for AI-assisted claims decisions.

## Acceptance Criteria

Given a Claim__c record has Status__c = 'In Review' and is opened by a claims adjuster, When the Agentforce Claims Compliance Copilot panel loads in the claims workspace, Then it displays AI-generated compliance flags categorized as: 'Billing Anomaly' (e.g., unbundling CPT codes), 'Duplicate Claim Risk' (matching Claim_Number__c or provider/date/CPT combination within 90 days), 'COB Coordination Required' (Policy__c.COB_Flag__c = TRUE with no COB_Resolution__c populated), and 'SIU Referral Recommended' — each with a confidence score (0–100) and supporting evidence summary.

Given Agentforce surfaces a 'SIU Referral Recommended' flag with confidence score >= 75, When the adjuster reviews the recommendation, Then a one-click 'Create SIU Referral' action is available that pre-populates a new Case record with RecordType = 'SIU Investigation', linked to the Claim__c, with AI_Referral_Reason__c populated from the Agentforce evidence summary and Referral_Source__c = 'Agentforce AI'.

Given any Agentforce recommendation is displayed, When the adjuster accepts or dismisses a flag, Then an AI_Review_Log__c record is created capturing: Claim__c lookup, Flag_Type__c, Confidence_Score__c, Adjuster_Action__c ('Accepted' | 'Dismissed' | 'Escalated'), Action_Timestamp__c, and Adjuster_Rationale__c (free text, required on dismissal of flags with confidence >= 80).

Given the AI_Review_Log__c is created, When a Compliance Officer runs the 'Agentforce Decision Audit' report, Then all AI-assisted adjudication decisions are visible with full flag details, adjuster actions, and rationale, filterable by Flag_Type__c, Confidence_Score__c range, and Adjuster_Action__c.

Given Agentforce generates a recommendation, When the recommendation is displayed, Then a disclaimer is shown: 'This recommendation is AI-generated and does not constitute a final claims determination. Adjuster review and approval required per Acme Insurance Claims Policy CP-2024-07.'

## Solution

Agentforce Copilot configured with a custom Agent Topic 'Claims Compliance Review' containing Actions mapped to Apex invocable methods: (1) 'GetDuplicateClaimRisk' — queries Claim__c for matching provider/date/CPT within 90 days; (2) 'GetCPTBundlingFlags' — calls a custom Einstein ML model trained on CPT unbundling patterns; (3) 'GetCOBStatus' — checks Policy__c.COB_Flag__c and COB_Resolution__c. Agentforce Prompt Template 'ClaimsCompliancePrompt' uses a Flex template with dynamic slot injection of claim data. AI_Review_Log__c records created via a Platform Event 'AI_Flag_Action__e' fired from the LWC 'agentforceClaimsPanel' when adjuster takes action, consumed by a Record-Triggered Flow. Custom LWC renders the Agentforce panel within the Claim__c Lightning Record Page using the agentforce-conversation-kit base component. Permission Set 'Agentforce_Claims_User_PS' controls access. CRM Analytics dashboard for Compliance Officer report built on AI_Review_Log__c dataset.

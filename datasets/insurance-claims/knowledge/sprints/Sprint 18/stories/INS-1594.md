## Description

As a Provider Relations Specialist, I need all Provider Change Notifications (PCNs) to automatically generate a compliance audit log entry and trigger a configurable hold on associated active claims when the change affects credentialing-sensitive fields (NPI, TIN, license status) so that Acme Insurance maintains data integrity and avoids erroneous payments during provider data transitions. Extends INS-1562.

## Acceptance Criteria

Given a Provider_Change_Notification__c record is created with PCN_Type__c = 'NPI Change' or 'TIN Change' or 'License Suspension', When the record is saved, Then a PCN_Audit_Log__c record is created with PCN_Audit_Log__c.PCN__c (lookup), PCN_Audit_Log__c.Provider__c (lookup to Person Account), PCN_Audit_Log__c.Effective_Date__c, PCN_Audit_Log__c.Compliance_Hold_Applied__c = TRUE, and PCN_Audit_Log__c.Created_By__c.

Given a compliance hold is triggered by a credentialing-sensitive PCN, When the hold is applied, Then all Claim__c records where Claim__c.Rendering_Provider__c matches the Provider Account and Claim__c.Status__c IN ('Received', 'In Adjudication') are updated to Claim__c.Compliance_Hold__c = TRUE and Claim__c.Hold_Reason__c = 'PCN Compliance Hold — Pending Verification'.

Given claims are placed on compliance hold, When the associated Credentialing_Case__c linked to the PCN reaches Credentialing_Case__c.Status__c = 'Approved', Then an Apex-invocable method automatically lifts the hold by setting Claim__c.Compliance_Hold__c = FALSE and Claim__c.Hold_Reason__c = NULL on all previously held claims, and a chatter post is added to each claim: 'Compliance hold lifted — PCN [PCN Number] resolved'.

Given a Provider Relations Specialist views a Provider (Person Account) record, When they open the 'Compliance Activity' tab, Then a FlexCard displays all PCN_Audit_Log__c entries for that provider sorted by Created_Date descending, showing PCN type, effective date, hold status, and resolution date.

-Given PCN records are submitted via email, a manual log entry process is used- (deprecated: replaced by automated PCN ingestion via DataRaptor integration with CAQH feed)

## Solution

Record-Triggered Flow on Provider_Change_Notification__c (after-insert, after-update) checks PCN_Type__c against a Custom Metadata Type (Compliance_Sensitive_PCN_Types__mdt) to determine if a hold is warranted. Apex invocable action 'ApplyPCNComplianceHold' performs a bulk SOQL query on Claim__c filtered by Rendering_Provider__c and eligible statuses, then bulk-updates Claim__c.Compliance_Hold__c. A second Record-Triggered Flow on Credentialing_Case__c monitors Status__c = 'Approved' and calls the 'LiftPCNComplianceHold' invocable action. Chatter posts use ConnectApi.ChatterFeeds. FlexCard built in OmniStudio consuming a DataRaptor Extract on PCN_Audit_Log__c.

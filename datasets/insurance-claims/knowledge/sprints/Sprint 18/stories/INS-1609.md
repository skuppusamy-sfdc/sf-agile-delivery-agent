## Description

As a Provider Relations Specialist, I need the Provider 360 FlexCard to surface a filterable EOB audit history panel showing all Explanation_of_Benefits__c records linked to a Provider Account (Person Account) across all associated claims so that I can quickly validate billing pattern compliance during network audits and respond to provider disputes without navigating multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist opens the Provider 360 FlexCard for a Provider__c (Person Account),
When the 'EOB History' tab is selected,
Then all Explanation_of_Benefits__c records where EOB.Claim__r.Rendering_Provider__c = Provider__c.Id must be displayed, showing: EOB_Number__c, Service_Date__c, CPT_Code__c, Billed_Amount__c, Allowed_Amount__c, Paid_Amount__c, Denial_Reason__c, and EOB_Status__c.

Given the EOB History panel is loaded,
When the specialist filters by EOB_Status__c = 'Denied',
Then only denied EOBs must be shown and a summary count 'X of Y EOBs Denied' must appear at the top of the panel.

Given the specialist filters by date range,
When Service_Date__c is outside the selected range,
Then those EOB records must be excluded from the panel and from aggregate totals.

Given an EOB record has Denial_Reason__c populated,
When the specialist hovers over the denial reason icon,
Then a tooltip must display the full Denial_Reason__c text and the associated ICD-10 or CPT code from the parent Claim_Line__c.

Given more than 50 EOB records exist for a provider,
When the panel loads,
Then pagination must be implemented showing 20 records per page with Previous/Next controls.

## Solution

Build an OmniStudio FlexCard 'Provider360_EOB_History' with a DataRaptor Extract source querying Explanation_of_Benefits__c with a cross-object filter on Claim__c.Rendering_Provider__c. Filter controls (Status picklist, Date Range) will be implemented as FlexCard input parameters passed to the DataRaptor at runtime. Pagination is handled via DataRaptor offset/limit parameters wired to FlexCard state variables. The tooltip on Denial_Reason__c uses a FlexCard conditional block with a child card overlay. The FlexCard is embedded in the existing Provider 360 OmniStudio App via the Lightning App Builder tab slot. A DataRaptor Transform aggregates denial counts for the summary banner.

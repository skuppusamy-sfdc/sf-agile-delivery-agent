## Description

As a Compliance Officer, I need every Explanation_of_Benefits__c (EOB) document generation and delivery event to be logged in a tamper-evident audit record so that Acme Insurance can respond to state Department of Insurance (DOI) inquiries with evidence that EOBs were generated and delivered within the mandated 30-day post-adjudication window.

## Acceptance Criteria

Given a Claim__c record transitions to Status__c = 'Adjudicated', When the EOB generation process is triggered via the Conga CLM template merge, Then an EOB_Audit_Log__c record must be created with fields: Claim__c (lookup), Policy__c (lookup), EOB_Document_ID__c (ContentVersion Id), Generation_Timestamp__c, Generation_Method__c = 'Automated', Days_Since_Adjudication__c (formula field), and Compliance_Window_Met__c (checkbox, true if Days_Since_Adjudication__c <= 30).

Given an EOB_Audit_Log__c record is created, When Compliance_Window_Met__c = false, Then a High priority Case must be auto-created with Subject = 'EOB Compliance Window Breach - [Claim Number]', Owner = Compliance Queue, and linked to both the Claim__c and EOB_Audit_Log__c records.

Given an EOB is delivered to the policyholder (via email, portal, or mail), When the delivery confirmation is received, Then EOB_Audit_Log__c.Delivery_Method__c, Delivery_Timestamp__c, and Delivery_Status__c = 'Delivered' must be updated on the corresponding log record.

Given a Compliance Officer runs the 'EOB Compliance Dashboard', When filtering by date range and LOB, Then the dashboard must display: total EOBs generated, % delivered within 30-day window, breach count by LOB, and a drill-down list of breached Claim__c records with Claim__c.Claim_Number__c and Policy__c.Policy_Number__c visible.

-Given manual EOB regeneration, When a Claims Ops Manager triggers regeneration, Then the original EOB_Audit_Log__c is overwritten — superseded: original log must be preserved and a new log created with Generation_Method__c = 'Manual Regeneration' and Supersedes_Log__c lookup to the original-.

## Solution

Implement a Record-Triggered Flow on Claim__c (after save, Status__c changes to 'Adjudicated') to invoke an Apex action (EOBAuditLogService.createLog()) that creates the EOB_Audit_Log__c record and triggers the Conga CLM merge API callout via a Queueable Apex job (EOBGenerationQueueable). Use a Platform Event (EOB_Delivery_Event__e) published by the policyholder portal to update delivery status on EOB_Audit_Log__c via a Trigger-Based Flow subscribing to the Platform Event. Build the compliance dashboard as a CRM Analytics (Tableau CRM) dashboard with SAQL queries against EOB_Audit_Log__c and Claim__c datasets. Apply Field-Level Security on EOB_Audit_Log__c to prevent non-admin users from editing Generation_Timestamp__c and Compliance_Window_Met__c. Use a Custom Permission 'EOB_Manual_Regen' to gate the manual regeneration button on the Claim__c page layout.

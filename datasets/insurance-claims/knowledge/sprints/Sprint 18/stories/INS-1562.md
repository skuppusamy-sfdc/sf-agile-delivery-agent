# INS-1562: Compliance Reporting Permission Set Group Misconfigured — Compliance Officers Can View PII in SIU Investigation Records

## Description

**Bug Report — Security/Compliance Priority**

**Steps to Reproduce:**
1. Log in as a user assigned only the 'Compliance_Reporting_PSG' Permission Set Group (no SIU-specific permissions).
2. Navigate to the Claim__c record for any claim where SIU_Referral_Status__c = 'Under Investigation'.
3. Open the related SIU_Investigation__c record from the related list.
4. Observe accessible fields.

**Expected Behavior:** Users in the 'Compliance_Reporting_PSG' Permission Set Group should have NO access to SIU_Investigation__c records. The related list should be hidden and direct URL navigation to SIU records should return an 'Insufficient Privileges' error. SIU investigation data contains highly sensitive PII including witness statements, claimant SSN references, and fraud allegation details.

**Actual Behavior:** Users in 'Compliance_Reporting_PSG' can view all fields on SIU_Investigation__c records including SIU_Narrative__c (free-text investigation notes), Claimant_SSN_Last4__c, Witness_Contact__c, and Allegation_Type__c. This is a data privacy violation and potential HIPAA/CCPA compliance risk.

**Root Cause Hypothesis:** The 'Compliance_Reporting_PSG' Permission Set Group was incorrectly assembled during Sprint 16 setup. It appears to include the 'SIU_Read_Access_PS' Permission Set, which was added erroneously. Additionally, the SIU_Investigation__c object OWD may be set to 'Public Read Only' instead of 'Private', allowing role-hierarchy visibility to bypass intended restrictions.

**Severity:** Critical — Data privacy violation. Escalated to CISO team. Immediate remediation required before Sprint 18 release to production.

## Acceptance Criteria

As per Description. Verified fix must confirm: (1) 'SIU_Read_Access_PS' is removed from 'Compliance_Reporting_PSG' Permission Set Group. (2) SIU_Investigation__c OWD is set to 'Private'. (3) A Compliance Officer profile user cannot access SIU_Investigation__c via related list, direct URL, SOQL in Developer Console, or API. (4) SIU Investigators and SIU Managers retain full access to SIU_Investigation__c records via dedicated 'SIU_Full_Access_PS' Permission Set. (5) A security scan using Salesforce Health Check and a manual penetration test checklist item confirms remediation.

## Solution

Immediate: Remove 'SIU_Read_Access_PS' from 'Compliance_Reporting_PSG' Permission Set Group in Setup. Change SIU_Investigation__c OWD from 'Public Read Only' to 'Private' in Security Settings (coordinate with admin for sharing rule impact assessment). Verify SIU Investigator and SIU Manager roles have explicit 'SIU_Full_Access_PS' Permission Set assigned — add Sharing Rules scoped to Role 'SIU_Investigator' and 'SIU_Manager' to restore appropriate record access after OWD tightening. Remove SIU_Investigation__c related list from Claim__c Lightning page layouts assigned to Compliance Officer profiles. Add a SOQL-based Apex test in the security test suite asserting that a System.runAs() block with Compliance_Reporting_PSG user cannot query SIU_Investigation__c records. Document remediation steps in Compliance_Incident_Log__c with Incident_Type__c = 'Unauthorized Data Exposure — Resolved'.

## Description

As a Provider Relations Specialist, I need automated notifications and a regulatory disclosure log when Provider_Network_Agreement__c (PNA) contracts are within 90, 60, and 30 days of expiry so that the network team can proactively renegotiate or terminate agreements before lapse and maintain compliant network adequacy reporting to state regulators. Extends INS-1569 (PNA Lifecycle Management).

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA__c.Contract_End_Date__c is not null and PNA__c.Status__c = 'Active', When the daily scheduled Flow runs, Then any PNA expiring within 90 days must have PNA__c.Expiry_Alert_Stage__c set to '90-Day Warning' and a Task created for the owning Provider Relations Specialist with Subject = 'PNA Expiry: 90-Day Action Required' and ActivityDate = today + 5 business days.

Given PNA__c.Expiry_Alert_Stage__c = '90-Day Warning' and expiry is now within 60 days, When the daily scheduled Flow runs, Then Expiry_Alert_Stage__c must advance to '60-Day Warning', a new Task created, and a Provider_Change_Notification__c (PCN) record must be auto-created with PCN__c.Notification_Type__c = 'Contract Expiry Warning' and PCN__c.Regulatory_Filing_Required__c = true.

Given PNA__c.Expiry_Alert_Stage__c = '60-Day Warning' and expiry is now within 30 days, When the daily scheduled Flow runs, Then Expiry_Alert_Stage__c must advance to '30-Day Warning', the PNA owner's manager must receive an escalation email, and PCN__c.Escalation_Level__c must be set to 'Senior Leadership'.

Given a PNA reaches its Contract_End_Date__c without renewal (PNA__c.Renewal_PNA__c lookup is null), When the date passes, Then PNA__c.Status__c must automatically transition to 'Lapsed' and all associated Claim__c records in 'Pending' status referencing the provider must have a Warning_Flag__c = 'Provider PNA Lapsed' applied.

Given any expiry notification is generated, When the PCN record is created or updated, Then a Regulatory_Disclosure_Log__c record must be created capturing: PCN__c (lookup), Disclosure_Date__c, Recipient_Type__c, State_Filed__c (from PNA__c.State__c), and Filing_Deadline__c = Disclosure_Date__c + 30 days.

## Solution

Build a Scheduled Flow ('PNA Expiry Monitor') running daily at 06:00 local time using a Get Records element filtered on Contract_End_Date__c and Status__c = 'Active'. Use Decision elements for 90/60/30-day threshold branching and Update Records elements for Expiry_Alert_Stage__c transitions. Create Tasks and PCN records via Create Records flow elements. Use a Record-Triggered Flow on Provider_Network_Agreement__c for the lapse transition when Contract_End_Date__c equals TODAY() and Renewal_PNA__c is null. Implement a before-save flow on Claim__c to apply Warning_Flag__c based on a lookup to the provider's active PNA status using a custom formula field cross-object reference. Use Custom Notification Types for in-app alerts to Provider Relations Specialists. Store escalation email templates in OmniStudio Email Templates for consistency with other PCN communications.

## Description

As a Compliance Officer, I need a permission-gated FlexCard on the Fee_Schedule__c record page that displays the complete change history of fee schedule rates (Fee_Schedule_Line__c records) including who changed each CPT code rate, when, and from what value to what value so that I can satisfy auditor requests for fee schedule amendment documentation without needing edit access to the underlying records.

## Acceptance Criteria

Given a Compliance Officer opens a Fee_Schedule__c record,
When the 'Fee Schedule Audit History' FlexCard loads,
Then it must display all Fee_Schedule_Line__c change events from Fee_Schedule_Audit__c child records, showing: CPT_Code__c, Old_Rate__c, New_Rate__c, Effective_Date__c, Modified_By__c (user name), and Modification_Timestamp__c.

Given the FlexCard is rendered,
When a user without the 'Fee_Schedule_Audit_Viewer' Permission Set is logged in,
Then the FlexCard must render a 'You do not have permission to view fee schedule audit history' message and must not expose any Fee_Schedule_Audit__c data.

Given more than 50 Fee_Schedule_Audit__c records exist for a single Fee_Schedule__c,
When the FlexCard loads,
Then records must be paginated at 25 per page with next/previous navigation controls.

-Given export to Excel was a requirement-
Given export to PDF is required for audit submissions,
When the Compliance Officer clicks 'Export Audit History',
Then a Conga Composer template must generate a PDF of all Fee_Schedule_Audit__c records for the current Fee_Schedule__c, stamped with export datetime and the exporting user's name.

## Solution

1. Create Fee_Schedule_Audit__c object as child of Fee_Schedule__c; populate via Record-Triggered Flow on Fee_Schedule_Line__c (After Update) capturing old/new Rate__c values using $Record__Prior. 2. Build OmniStudio FlexCard 'FeeScheduleAuditHistory' using DataRaptor Extract pulling Fee_Schedule_Audit__c records filtered by parent Fee_Schedule__c Id; configure card state for unauthorized users. 3. Add conditional visibility on FlexCard using Custom Permission check ('Fee_Schedule_Audit_Viewer') via DataRaptor formula. 4. Implement pagination using FlexCard built-in datatable with pageSize=25. 5. 'Export Audit History' action button triggers Apex Controller invoking Conga Composer via REST API with Fee_Schedule__c Id parameter. 6. Assign 'Fee_Schedule_Audit_Viewer' custom permission to Compliance Officer Permission Set Group.

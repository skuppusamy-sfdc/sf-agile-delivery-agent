## Description

As a Credentialing Specialist, I need a detailed audit log of every CAQH data synchronization event for Provider (Person Account) and Credentialing_Case__c records so that I can identify and remediate data discrepancies between Salesforce and CAQH ProView in under 24 hours, satisfying NCQA credentialing standards for data integrity documentation. Extends INS-1540.

## Acceptance Criteria

Given the CAQH integration sync job executes,
When provider data is retrieved from the CAQH ProView API,
Then a CAQH_Sync_Log__c record must be created for each Provider (Person Account) processed, capturing: Provider__c (lookup), Sync_Timestamp__c, CAQH_Provider_ID__c, Sync_Status__c ('Success'|'Partial'|'Failed'), Fields_Updated__c (long text area listing field API names), Discrepancy_Count__c (integer), and Raw_Response_Payload__c (encrypted long text).

Given a field value in CAQH differs from the corresponding Salesforce field (e.g., CAQH NPI differs from Provider__c.NPI__c),
When the sync log is written,
Then a child CAQH_Discrepancy__c record must be created with Field_Name__c, CAQH_Value__c, Salesforce_Value__c, Discrepancy_Severity__c ('Critical' for NPI/TIN/License mismatches, 'Warning' for address/phone), and Resolved__c = false.

Given a CAQH_Discrepancy__c with Discrepancy_Severity__c = 'Critical' is created,
When the record is inserted,
Then the related Credentialing_Case__c.Status__c must be set to 'On Hold — Data Discrepancy' and a high-priority Task must be assigned to the Credentialing Specialist queue.

Given a Credentialing Specialist resolves a discrepancy,
When CAQH_Discrepancy__c.Resolved__c is set to true and Resolution_Notes__c is populated,
Then if all sibling CAQH_Discrepancy__c records for the same Provider__c are resolved, Credentialing_Case__c.Status__c must automatically revert to 'In Review'.

-Given sync logs older than 30 days are purged- (deprecated: retention extended to 3 years per NCQA audit requirement; archival strategy deferred to Platform Readiness epic)

Given a sync job fails entirely (HTTP 5xx from CAQH API),
When the failure is detected,
Then CAQH_Sync_Log__c.Sync_Status__c = 'Failed', an Integration_Error_Log__c record must be created with Error_Code__c, Error_Message__c, and Retry_Count__c, and an automatic retry must be attempted up to 3 times with exponential backoff before alerting the integration team via a custom notification.

## Solution

MuleSoft integration layer calls CAQH ProView REST API and posts results to a Salesforce Platform Event (CAQH_Sync_Result__e). A Platform Event-Triggered Flow processes the event and creates CAQH_Sync_Log__c and CAQH_Discrepancy__c records. Critical discrepancy handling implemented via a Record-Triggered Flow on CAQH_Discrepancy__c (after insert). Credentialing_Case__c status revert logic uses an Apex Trigger with aggregate SOQL to check all sibling discrepancies. Raw_Response_Payload__c encrypted using Salesforce Shield Platform Encryption with a tenant secret. Retry logic implemented in MuleSoft with an Until-Successful scope. Integration_Error_Log__c inserted via Apex in the Platform Event subscriber's fault handler.

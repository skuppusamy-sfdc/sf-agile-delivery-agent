## Description

As a Provider Network Specialist, I need a structured workflow on the Network_Termination_Case__c object to track mandatory regulatory notice periods, member notification requirements, and state filing deadlines when terminating a provider from the network so that Acme Insurance avoids regulatory penalties for inadequate termination notice and maintains continuity of care compliance.

## Acceptance Criteria

Given a Network_Termination_Case__c (NTC) record is created with NTC.Termination_Type__c = 'Involuntary', When the record is saved, Then the system must auto-populate NTC.Member_Notification_Deadline__c = NTC.Effective_Termination_Date__c - 30 days, NTC.State_Filing_Deadline__c = NTC.Effective_Termination_Date__c - 60 days, and NTC.Regulatory_Notice_Status__c = 'Pending'.

Given NTC.Termination_Type__c = 'Voluntary' (provider-initiated), When the record is saved, Then NTC.Member_Notification_Deadline__c = NTC.Effective_Termination_Date__c - 15 days and NTC.State_Filing_Deadline__c = NTC.Effective_Termination_Date__c - 30 days per voluntary termination regulatory standards.

Given NTC.State_Filing_Deadline__c is within 10 business days, When the daily scheduled Flow runs, Then a High priority Task must be created for the owning Provider Network Specialist with Subject = 'State Filing Due: [Provider Name] Termination' and a PCN record created with PCN.Notification_Type__c = 'Network Termination' and PCN.Regulatory_Filing_Required__c = true.

Given the state filing is completed, When a Provider Network Specialist updates NTC.Regulatory_Notice_Status__c = 'Filed', Then NTC.Filing_Confirmation_Number__c must be a required field (enforced via validation rule) and NTC.Filed_Date__c must be auto-stamped with today's date.

Given an NTC reaches NTC.Effective_Termination_Date__c with NTC.Regulatory_Notice_Status__c != 'Filed', When the date arrives, Then NTC.Termination_Status__c must be set to 'Blocked - Regulatory Hold', the termination must not proceed, and an escalation email must be sent to the VP of Provider Network and Compliance Officer.

Given a provider is terminated, When NTC.Termination_Status__c = 'Complete', Then all associated Provider_Network_Agreement__c records for the provider must have PNA.Status__c updated to 'Terminated' and PNA.Termination_Date__c populated from NTC.Effective_Termination_Date__c via a child-record update flow.

## Solution

Build a Before-Save Record-Triggered Flow on Network_Termination_Case__c to auto-populate deadline date fields based on Termination_Type__c using a Decision element and date formula assignments. Create a Validation Rule on NTC enforcing Filing_Confirmation_Number__c when Regulatory_Notice_Status__c = 'Filed'. Implement a Scheduled Flow ('NTC Deadline Monitor') for daily Task and PCN creation within the 10-business-day window, using a custom business day calculation Apex invocable action (BusinessDayCalculator.getBusinessDaysRemaining()). Build an After-Save Record-Triggered Flow for the regulatory hold escalation when Effective_Termination_Date__c = TODAY() and status is not 'Filed'. Use a child-record update sub-flow to bulk-update PNA records upon termination completion. Implement the escalation email using a Custom Notification and Email Alert combination for redundancy. Store state-specific notice period rules in Custom Metadata Type (State_Termination_Rules__mdt) keyed by state abbreviation.

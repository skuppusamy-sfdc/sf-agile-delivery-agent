## Description

As a Compliance Officer, I need an automated monthly CMS regulatory reporting extract that compiles adjudicated Medicare Advantage claims data from Claim__c, Claim_Line__c, and Payment__c objects into the EDPS-compliant flat file format so that Acme Insurance meets the CMS submission deadline without manual data extraction and formatting effort.

## Acceptance Criteria

Given the last calendar day of each month, When the scheduled Apex batch job (CMSReportingBatch) executes at 02:00 UTC, Then it must query all Claim__c records where Claim__c.LOB__c = 'Medicare Advantage' AND Claim__c.Status__c = 'Adjudicated' AND Claim__c.Adjudication_Date__c falls within the reporting month.

Given qualifying claims are identified, When the extract is generated, Then each output row must include: Member_ID (from Policy__c.Member_ID__c), NPI (from Provider__c.NPI__c), TIN (from Provider__c.TIN__c), ICD-10 diagnosis codes (from Claim__c.Primary_Diagnosis_Code__c and Claim__c.Secondary_Diagnosis_Codes__c), CPT codes (from Claim_Line__c.CPT_Code__c), Allowed_Amount (from Claim_Line__c.Allowed_Amount__c), Paid_Amount (from Payment__c.Net_Payment_Amount__c), and COB indicator (from Claim__c.COB_Flag__c).

Given the extract file is generated, When the file is complete, Then it must be saved as a Salesforce File (ContentVersion) attached to a CMS_Report_Submission__c record with fields: Report_Month__c, Record_Count__c, Total_Paid_Amount__c, Submission_Status__c = 'Pending Review', and Generated_By__c.

Given the CMS_Report_Submission__c record is created, When Submission_Status__c = 'Approved', Then the file must be transmitted to the CMS SFTP endpoint via a named credential callout and Submission_Status__c updated to 'Submitted' with Submission_Timestamp__c populated.

Given a submission failure occurs, When the SFTP callout returns a non-200 response or times out, Then an automated email alert must be sent to the Compliance Officer distribution list and CMS_Report_Submission__c.Submission_Status__c must be set to 'Failed' with Error_Detail__c capturing the HTTP response body.

-Given manual override is needed, When a Compliance Officer clicks 'Force Submit', Then the system bypasses approval — removed per SOX control requirements, manual submissions require VP Compliance approval via Salesforce Approval Process-.

## Solution

Build CMSReportingBatch as a Database.Batchable and Database.Schedulable Apex class with scope size of 200. Use DataRaptor Extract (OmniStudio) to assemble the multi-object data structure for Claim__c, Claim_Line__c, and Payment__c joins. Generate the flat file using an Apex StringBuilder pattern within the batch finish() method and insert as ContentVersion via Salesforce Files API. Create an Integration Procedure (CMS_SFTP_Submission) to handle the named credential-based SFTP callout using a custom HttpCalloutMock for test coverage. Build a CMS_Report_Submission__c Approval Process with two-step approval (Compliance Officer → VP Compliance) triggered when Submission_Status__c = 'Pending Review'. Schedule the batch via a Scheduled Flow as a backup mechanism. Store SFTP credentials in Named Credentials with the 'Per-User' principal type disabled for service account use.

## Description

As a Compliance Officer, I need a comprehensive, tamper-evident audit trail automatically captured for all adjudication lifecycle events on Claim__c and Claim_Line__c records so that the organization meets state and federal regulatory requirements (including CMS audit readiness), supports SIU investigations, and provides defensible documentation during external audits or litigation.

This story extends INS-1584 (Payment audit logging) to cover the full upstream adjudication workflow, including status transitions, COB determinations, fee schedule applications, and manual override events.

Key events requiring audit capture:
- Claim__c.Status__c transitions (e.g., 'Received' → 'In Review' → 'Adjudicated' → 'Paid' / 'Denied')
- Claim_Line__c.Adjudication_Status__c changes
- Manual overrides to Claim_Line__c.Allowed_Amount__c or Claim_Line__c.Paid_Amount__c
- Fee_Schedule__c version applied at time of adjudication
- COB rule applied (Claim__c.COB_Rule__c)
- Assignment to SIU queue (Claim__c.SIU_Referral__c = true)
- EOB generation events linked to Explanation_of_Benefits__c

## Acceptance Criteria

Given a Claim__c record undergoes any Status__c transition,
When the transition is saved (insert or update),
Then a new Claim_Audit_Event__c record must be created within the same transaction containing: Claim_Audit_Event__c.Claim__c (lookup), Claim_Audit_Event__c.Event_Type__c = 'Status Change', Claim_Audit_Event__c.Old_Value__c, Claim_Audit_Event__c.New_Value__c, Claim_Audit_Event__c.Changed_By__c (User lookup), Claim_Audit_Event__c.Change_Timestamp__c (system datetime, UTC), and Claim_Audit_Event__c.LOB__c mirrored from Claim__c.Line_of_Business__c.

Given a Claims Operations Manager or adjudicator manually overrides Claim_Line__c.Allowed_Amount__c or Claim_Line__c.Paid_Amount__c,
When the record is saved,
Then a Claim_Audit_Event__c record must be created with Event_Type__c = 'Manual Override', capturing the delta amount (New_Value__c - Old_Value__c), Override_Justification__c (required rich-text field, minimum 20 characters), and the Fee_Schedule__c.Version__c that was active at adjudication time; if Override_Justification__c is blank the save must be blocked with a validation error: 'A justification is required for manual amount overrides per compliance policy.'

Given a Claim__c.SIU_Referral__c field is set to true,
When the record is saved,
Then a Claim_Audit_Event__c with Event_Type__c = 'SIU Referral' must be created and a Platform Event (Compliance_Alert__e) must be published with payload fields: Claim_Id__c, Referral_Reason__c, Referring_User__c, and Timestamp__c, so that downstream SIU case management systems can subscribe and react in near-real-time.

Given an Explanation_of_Benefits__c record is generated and linked to a Claim__c,
When EOB_Status__c transitions to 'Generated' or 'Sent',
Then a Claim_Audit_Event__c with Event_Type__c = 'EOB Event' must be created referencing both Claim__c and Explanation_of_Benefits__c lookup fields, and EOB_Delivery_Channel__c (Mail / Portal / ERA) must be recorded.

-Given a TPA user modifies Claim__c.Adjudication_Notes__c, a separate audit record must be created — DEPRECATED: TPA note changes are now handled by the Field History Tracking fallback per architecture review 2024-11-02-

Given a Compliance Officer accesses the Claim 360 page,
When they view the 'Audit Trail' tab,
Then all Claim_Audit_Event__c records for that Claim__c must be displayed in a LWC timeline component sorted descending by Change_Timestamp__c, with filters for Event_Type__c and date range, and the timeline must load within 3 seconds for claims with up to 500 audit events.

Given the audit trail records are written,
When any user (including System Administrator) attempts to delete or edit a Claim_Audit_Event__c record,
Then the operation must be blocked by an Apex-enforced restriction (no delete/update DML permitted on the object) and a Permission Set Group 'Compliance_ReadOnly_PSG' must govern read access for non-privileged roles.

Given regulatory reporting is required,
When the Compliance Officer runs the 'Monthly Adjudication Audit Summary' report,
Then the report must aggregate Claim_Audit_Event__c records by Event_Type__c, LOB__c, and calendar month, and must be exportable to CSV with all PII fields masked according to Claim__c.Data_Sensitivity__c = 'High' rules.

## Solution

1. **Apex Trigger (ClaimAuditTrailTrigger)** on Claim__c (after insert, after update) and Claim_Line__c (after update): captures old/new field values using Trigger.oldMap / Trigger.newMap and bulk-inserts Claim_Audit_Event__c records via a dedicated AuditTrailService Apex class to keep trigger logic thin.

2. **Record-Triggered Flow** considered but rejected for this use case due to governor limit risk at scale; pure Apex chosen for transactional integrity and bulkification.

3. **Platform Event (Compliance_Alert__e)**: published from AuditTrailService using EventBus.publish() when SIU_Referral__c = true; a separate Platform Event-Triggered Flow routes the event to the SIU Case queue and creates a Task on the Claim__c.

4. **Validation Rule** on Claim_Line__c: enforces Override_Justification__c is populated when Allowed_Amount__c or Paid_Amount__c is changed by a non-system user (using $Profile.Name exclusion list for integration profiles).

5. **LWC (claimAuditTimeline)**: retrieves Claim_Audit_Event__c records via a @wire Apex controller with cursor-based pagination (SOQL OFFSET), renders a vertical timeline using SLDS, and includes client-side filtering by Event_Type__c picklist and lightning-datepicker range.

6. **Permission Set Group (Compliance_ReadOnly_PSG)**: bundles read-only OLS on Claim_Audit_Event__c; Apex trigger on Claim_Audit_Event__c (before update, before delete) throws AuraHandledException to prevent modification regardless of profile.

7. **Custom Report Type**: Claim Audit Events with Claims as primary object; report includes formula fields for masked NPI/TIN display based on running user's Permission Set membership.

8. **Test Classes**: minimum 85% coverage, including bulk test (200 records), negative tests for blocked delete, and mock for EventBus.publish().

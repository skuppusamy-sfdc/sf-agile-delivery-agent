## Description

As a Compliance Officer, I need an automated regulatory reporting dashboard that aggregates Claim__c and Claim_Line__c data into state Department of Insurance (DOI) required metrics — including prompt-pay compliance rates, denial rates by LOB, and average days-to-adjudication — so that monthly DOI submissions can be generated directly from Salesforce without manual Excel reconciliation.

## Acceptance Criteria

Given the reporting period filter is set to a calendar month,
When the Compliance Officer runs the DOI Metrics Dashboard,
Then the dashboard must display: (1) Total Claims Received (count of Claim__c records with Received_Date__c in period), (2) Prompt-Pay Compliance Rate (% of Claim__c records where Paid_Date__c minus Received_Date__c ≤ state_prompt_pay_threshold_days per State_Config__mdt), (3) Denial Rate by LOB__c, (4) Average_Days_To_Adjudication__c computed as AVG(Adjudicated_Date__c - Received_Date__c).

Given the dashboard is rendered,
When a Compliance Officer clicks 'Export for DOI Submission',
Then a CSV file must be generated via a Conga Composer template matching the state-mandated column layout and downloaded within 30 seconds for datasets up to 100,000 Claim__c records.

Given a Claim__c.LOB__c value is not in the approved picklist ('Medical', 'Dental', 'Vision', 'Behavioral Health', 'Pharmacy'),
When the aggregation runs,
Then those records must be flagged in a Data_Quality_Exception__c object with Exception_Type__c = 'Invalid LOB' and excluded from DOI totals.

Given multi-state operations,
When the dashboard loads,
Then a State__c filter picklist must restrict data to policies where Policy__c.State__c matches the selected state, and prompt-pay thresholds must be sourced from State_Compliance_Config__mdt keyed by State__c.

-Given manual data entry for prior-period corrections is required- (deprecated: corrections must now flow through the formal Claim_Adjustment__c process per INS-1562 requirements)

## Solution

Build a CRM Analytics (Tableau CRM) dashboard consuming a dataflow that joins Claim__c, Claim_Line__c, and Policy__c datasets. A scheduled Apex batch job (Batchable, Schedulable) pre-aggregates nightly into a Regulatory_Report_Snapshot__c custom object to ensure sub-second dashboard load. State thresholds stored in State_Compliance_Config__mdt with fields State__c, Prompt_Pay_Days__c, Report_Template_ID__c. Conga Composer template mapped to Regulatory_Report_Snapshot__c fields for CSV export. Data_Quality_Exception__c records inserted via a Record-Triggered Flow on Claim__c evaluating LOB__c on create/update. Permission Set 'DOI_Reporting_Access' scoped to Compliance Officer profile.

## Description

As a Provider Network Specialist, I need a real-time compliance dashboard surfacing all Provider_Network_Agreement__c (PNA) records approaching expiry or already expired so that I can proactively remediate network gaps before a state DOI audit identifies out-of-network compliance deficiencies.

## Acceptance Criteria

Given a Provider Relations Specialist accesses the 'PNA Compliance Dashboard' Lightning App Page,
When the page loads,
Then a FlexCard must display aggregate KPI tiles showing: (1) Count of PNA__c records with Expiry_Date__c < TODAY(), (2) Count with Expiry_Date__c BETWEEN TODAY() AND TODAY()+30, (3) Count with Expiry_Date__c BETWEEN TODAY()+31 AND TODAY()+90, segmented by LOB__c (Commercial, Medicare Advantage, Medicaid).

Given a PNA record has Expiry_Date__c within 60 days,
When the nightly scheduled flow runs at midnight,
Then a Task must be created assigned to Provider_Network_Agreement__c.Owner with Subject = 'PNA Expiry Action Required – [Provider Name]', Due_Date = Expiry_Date__c - 30 days, and Priority = 'High', unless an open Task of the same subject already exists (duplicate prevention via SOQL check).

Given a PNA record has Expiry_Date__c < TODAY() AND Provider_Network_Agreement__c.Status__c != 'Terminated',
When any Claim__c is submitted referencing that provider,
Then Claim__c.Compliance_Hold__c must be set to TRUE and Claim__c.Hold_Reason__c = 'Expired PNA – Pending Renewal' via a Record-Triggered Flow, preventing payment processing.

-Given expired PNAs are auto-terminated after 90 days- (deprecated; legal confirmed manual termination workflow required per INS-1553).

Given a Compliance Officer exports the dashboard data,
When they click 'Export to CSV',
Then the export must include PNA ID, Provider NPI, Provider TIN, Contract_Type__c, LOB__c, Expiry_Date__c, Days_Until_Expiry (formula), and Assigned_Network_Specialist__c.

## Solution

Build an OmniStudio FlexCard 'PNAComplianceDashboard' with a DataRaptor Extract sourcing data from Provider_Network_Agreement__c with SOQL date filters. KPI tiles use conditional styling (red/amber/green) driven by custom DataRaptor transforms. A Scheduled Flow runs nightly querying PNA records within the expiry windows and creates Tasks via the Create Records element with a Get Records element pre-check for duplicate tasks. A Record-Triggered Flow on Claim__c (before insert/update) checks a related PNA's Expiry_Date__c via a Get Records element and sets Compliance_Hold__c. The CSV export uses a custom LWC with a CSV serializer utility calling a @wire Apex method.

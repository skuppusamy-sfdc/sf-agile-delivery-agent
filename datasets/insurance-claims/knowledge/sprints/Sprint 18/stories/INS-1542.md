## Description

As a Compliance Officer, I need an automated monthly data extract that compiles adjudicated Claim__c records into a structured format aligned with CMS-1500 reporting requirements so that we can submit timely and accurate reports to state Departments of Insurance without manual data assembly, reducing submission errors and avoiding regulatory penalties.

## Acceptance Criteria

Given the last calendar day of each month at 11:00 PM EST, When the scheduled Apex job 'DOI_Monthly_Extract_Job' executes, Then it queries all Claim__c records where Status__c = 'Adjudicated' OR Status__c = 'Paid' AND Adjudication_Date__c falls within the prior calendar month, grouped by LOB__c and State__c.

Given the extract query completes, When records are aggregated, Then each output row must include: Claim__c.Claim_Number__c, Policy__c.Policy_Number__c, Provider__c.NPI__c, Provider__c.TIN__c, Facility__c.NPI__c, Claim__c.Total_Billed_Amount__c, Claim__c.Total_Allowed_Amount__c, Claim__c.Total_Paid_Amount__c, Claim__c.ICD10_Primary_Code__c, Claim_Line__c.CPT_Code__c (all lines), Payment__c.Payment_Date__c, and EOB__c.EOB_Number__c.

Given the extract data is assembled, When the file is generated, Then it must be output as a pipe-delimited .txt file conforming to the state DOI submission schema v2.4, stored as a ContentVersion on the associated Regulatory_Submission__c record with Document_Type__c = 'DOI Monthly Extract'.

Given the extract file is stored, When generation is complete, Then an email notification is sent to the Compliance Officer queue with file metadata (record count, total billed, total paid, LOB breakdown) and a direct link to the Regulatory_Submission__c record.

Given the extract job encounters any Claim__c record with missing Provider__c.NPI__c or Policy__c.Policy_Number__c, When the record is processed, Then it is written to a separate Compliance_Exception__c record with Exception_Type__c = 'Missing Required Field', linked to the Claim__c, and excluded from the primary extract file.

-Given extract files are emailed directly to the DOI portal address- (superseded; direct SFTP integration deferred to Sprint 20 per INS-1560 dependency)

Given the Regulatory_Submission__c record is created, When a Compliance Officer reviews it, Then the record displays a count of included claims, exception count, submission status (Draft | Submitted | Acknowledged), and a rollup of Total_Paid_Amount__c across all included Claim__c records.

## Solution

Schedulable Apex class 'DOI_Monthly_Extract_Job' implementing Schedulable and Batchable interfaces (batch size 200) to handle governor limits on large claim volumes. DataRaptor Extract configured as a secondary on-demand option for ad hoc pulls. File generation via Apex ContentVersion insert with VersionData populated from a StringBuilder assembling pipe-delimited rows. Regulatory_Submission__c is a custom object with a Master-Detail to a new Reporting_Period__c object. Compliance_Exception__c records inserted in a separate Apex Queueable. Email alerts sent via Messaging.SingleEmailMessage to a queue email address. Permission Set 'Compliance_Reporting_PS' grants access to Regulatory_Submission__c and Compliance_Exception__c. A Record-Triggered Flow on Regulatory_Submission__c updates Status__c to 'Ready for Review' when ContentVersion is attached.

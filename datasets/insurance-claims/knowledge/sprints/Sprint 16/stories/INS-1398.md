## Description

As a Broker Relations Coordinator, I need a Broker Production Scorecard report accessible from both the internal Salesforce org and the Agent Portal so that I can evaluate broker performance across key production metrics — including gross premium written, active policy count, and loss ratio — enabling data-driven decisions for broker tiering, incentive programs, and renewal negotiations.

## Acceptance Criteria

Given the Broker Relations Coordinator runs the 'Broker Production Scorecard' report,
When results are returned,
Then the report must display the following columns grouped by Broker (Account.RecordType = 'Broker'):
  - Broker Name (Account.Name)
  - NPN (Account.NPN__c)
  - Gross Premium Written YTD: SUM(Policy__c.Annual_Premium__c) where Policy__c.Broker__c = Broker Account Id and Policy__c.Effective_Date__c within current year
  - Active Policy Count: COUNT(Policy__c) where Policy__c.Status__c = 'Active'
  - Total Incurred Claims YTD: SUM(Claim__c.Total_Incurred__c) where Claim__c.Policy__r.Broker__c = Broker Account Id
  - Loss Ratio %: (Total Incurred Claims YTD / Gross Premium Written YTD) * 100

Given the Loss Ratio % column,
When the value exceeds 85%,
Then the cell must display a red conditional highlight; values between 70-85% display amber; below 70% display green

Given the report is accessed via the Agent Portal (Experience Cloud),
When a Broker user logs in,
Then they must only see their own scorecard row (scoped by Account.Id = running user's AccountId), enforced via Sharing Rules and the 'Broker_Portal_User' Permission Set

Given the Broker Relations Coordinator exports the scorecard,
When exported to Excel,
Then all calculated columns (Loss Ratio %, Gross Premium Written YTD) must retain their numeric formatting and not export as text

-Given the scorecard includes a 'Broker Tier' column auto-calculated from loss ratio bands- (deprecated: Broker Tier auto-assignment is being redesigned as an Agentforce recommendation feature in a future sprint)

Given the report is scheduled,
When configured for monthly delivery,
Then the report must be emailed to the Broker Relations Coordinator distribution group on the 1st of each month at 07:00 AM local time

## Solution

Build a Salesforce Summary Report 'Broker Production Scorecard' on the Account object with cross-object fields from Policy__c and Claim__c via lookup relationships. Loss Ratio % implemented as a custom Summary Formula: (SUM_Total_Incurred / SUM_Annual_Premium) * 100 with divide-by-zero guard. Conditional highlighting rules configured for Loss Ratio % thresholds. Report shared to Agent Portal (Experience Cloud) via Report tab in the Broker Portal app; row-level scoping enforced via Account Sharing Rule (Owner = Running User) and 'Broker_Portal_User' Permission Set restricting report access. Scheduled report delivery configured via Report Subscription with monthly recurrence. Excel export formatting validated via Salesforce standard report export (numbers stored as numeric fields, not formula text fields).

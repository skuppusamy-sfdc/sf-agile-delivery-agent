## Description

As a Provider Network Specialist, I need an automated reporting mechanism that identifies Provider_Network_Agreement__c records approaching expiration at 90, 60, and 30-day intervals and dispatches alert notifications to the responsible Contract Owner so that contract renewals are initiated proactively and network continuity is preserved. Extends INS-1362 (PNA Lifecycle Management).

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA_Expiration_Date__c is within 90 calendar days of today and PNA_Status__c = 'Active',
When the nightly scheduled job executes,
Then a Task must be created on the PNA record with Subject = 'PNA Expiration Alert – 90 Days', Status = 'Not Started', OwnerId = Provider_Network_Agreement__c.Contract_Owner__c, and ActivityDate = today

Given the same conditions at 60-day and 30-day thresholds,
When the scheduled job runs,
Then separate Tasks are created with Subject values 'PNA Expiration Alert – 60 Days' and 'PNA Expiration Alert – 30 Days' respectively; duplicate Tasks for the same threshold window must not be created if one already exists

Given the alert Task is created,
When the Contract Owner logs in,
Then they must see the Task surfaced in the 'PNA Renewal Alerts' list view filtered by Task.Subject LIKE 'PNA Expiration Alert%'

Given a PNA_Status__c is updated to 'Renewed' or 'Terminated',
When the record is saved,
Then any open expiration alert Tasks on that PNA must be automatically closed (Status = 'Completed') via a Record-Triggered Flow

Given the 'PNA Expiration Summary' report,
When a Compliance Officer runs it,
Then it must display columns: Provider_Network_Agreement__c.Name, Provider__r.Name, PNA_Expiration_Date__c, PNA_Status__c, Days_Until_Expiration__c (formula field), Contract_Owner__r.Name, grouped by expiration window (30/60/90 days)

## Solution

Implement a Scheduled Apex job (PNAExpirationAlertScheduler) that queries Provider_Network_Agreement__c records with PNA_Expiration_Date__c within rolling 90-day window and PNA_Status__c = 'Active'. Apex logic checks for existing open Tasks per threshold window to prevent duplicates using a Set of (PNA Id + threshold label) pairs. Task creation in bulk-safe DML. Record-Triggered Flow on Provider_Network_Agreement__c fires on PNA_Status__c change to 'Renewed'/'Terminated' and invokes an Apex Action to bulk-close related Tasks. Formula field Days_Until_Expiration__c = PNA_Expiration_Date__c - TODAY() deployed on Provider_Network_Agreement__c. Summary report built in Salesforce Reports with custom grouping.

## Description

As a Compliance Officer, I need an automated monthly report that validates EOB issuance compliance — specifically that every adjudicated Claim__c has a corresponding Explanation_of_Benefits__c record issued within the state-mandated 30-day window — so that I can submit accurate compliance attestations to state regulators and proactively identify violations before they result in fines.

The report must cover all LOBs (Medical, Dental, Vision) and flag any Claim__c records where EOB issuance is late, missing, or issued to an incorrect mailing address.

## Acceptance Criteria

Given the scheduled Apex job 'EOB_Compliance_Report_Job' runs on the 1st of each month at 2 AM,
When it executes,
Then it queries all Claim__c records WHERE Status__c = 'Adjudicated' AND Adjudication_Date__c = LAST_MONTH and cross-references EOB__c.Claim__c lookup.

Given the query runs,
When a Claim__c has no linked EOB__c record,
Then it is flagged as 'EOB Missing' and included in the compliance exception list with fields: Claim__c.Name, Claim__c.Policy__c, Claim__c.Adjudication_Date__c, Claim__c.Claimant__c, Claim__c.Line_of_Business__c.

Given a Claim__c has a linked EOB__c,
When EOB__c.Issue_Date__c > Claim__c.Adjudication_Date__c + 30,
Then the record is flagged as 'EOB Late Issuance' with EOB__c.Issue_Date__c and calculated days-late value included.

Given a Claim__c has a linked EOB__c,
When EOB__c.Mailing_Address__c != Policy__c.Policyholder_Mailing_Address__c,
Then the record is flagged as 'Address Mismatch' with both address values included for manual review.

Given the report generation is complete,
When all exceptions are compiled,
Then a Compliance_Report__c record is created with: Report_Month__c, Total_Claims_Evaluated__c, EOB_Missing_Count__c, EOB_Late_Count__c, Address_Mismatch_Count__c, Compliance_Rate__c (formula: (Total - Exceptions) / Total * 100), Status__c = 'Pending Review'.

Given the Compliance_Report__c record is created,
When Status__c = 'Pending Review',
Then a Task is auto-created and assigned to the 'Compliance Officer' role queue with Subject = 'Review Monthly EOB Compliance Report — [Month Year]' and Due_Date__c = CreatedDate + 5.

Given the Compliance Officer reviews and sets Status__c = 'Approved for Submission',
When the record is saved,
Then a Conga CLM merge job is triggered to generate a formatted PDF compliance attestation document attached to the Compliance_Report__c record.

## Solution

Implement a Schedulable Apex class 'EOB_Compliance_Report_Job' with batch processing (Database.Batchable, scope 200) iterating over prior-month adjudicated Claim__c records. Inner class performs EOB lookup via SOQL aggregate on EOB__c grouped by Claim__c. Exception categorization logic in Apex with results written to a custom Compliance_Exception__c junction object linked to Compliance_Report__c. Compliance_Report__c record creation via Apex DML. Task creation handled by a Record-Triggered Flow on Compliance_Report__c INSERT WHERE Status__c = 'Pending Review'. Conga CLM integration triggered by a second Record-Triggered Flow on Status__c change to 'Approved for Submission' invoking Conga REST API via Named Credential. Custom Permission Set 'Compliance_Report_Admin' grants CRUD on Compliance_Report__c and Compliance_Exception__c.

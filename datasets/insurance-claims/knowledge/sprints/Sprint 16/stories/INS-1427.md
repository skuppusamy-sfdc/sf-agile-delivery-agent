## Description

**Bug Report — Claims Payment Summary Report Double-Counting COB Lines**

**Environment:** Production Sandbox (Full Copy) | Salesforce API v59.0

**Steps to Reproduce:**
1. Navigate to Reports → 'Claims Payment Summary by Provider' report.
2. Select a date range that includes claims with Coordination of Benefits (COB) secondary payer payments.
3. Run the report.
4. Observe the 'Total Paid Amount' summary column.

**Expected Behavior:**
The 'Total Paid Amount' column should reflect the sum of Payment__c.Amount__c for Payment__c records where Payment__c.Payment_Type__c = 'Primary' only, or the net COB-adjusted amount as stored in Claim__c.Net_Paid_Amount__c.

**Actual Behavior:**
The report sums ALL Payment__c.Amount__c records linked to a Claim__c regardless of Payment__c.Payment_Type__c value. When a claim has both a Payment__c (Type = 'Primary') and a Payment__c (Type = 'COB Secondary'), both amounts are included in the total, inflating the reported payment figure. In a sample of 340 COB claims, total paid was overstated by $218,450.

**Root Cause Hypothesis:**
The custom report type 'Claims with Payments' joins Claim__c to Payment__c as a one-to-many without a filter on Payment__c.Payment_Type__c. The summary formula does not exclude COB Secondary records.

**Impact:** High — finance reconciliation reports are being pulled from this dashboard; overstated figures are being submitted in weekly TPA settlement reports.

## Acceptance Criteria

As per Description.

Additional verification:
Given a Claim__c with two Payment__c child records (one Payment_Type__c = 'Primary', one Payment_Type__c = 'COB Secondary'),
When the 'Claims Payment Summary by Provider' report runs,
Then the 'Total Paid Amount' column reflects only the sum of Payment__c.Amount__c where Payment__c.Payment_Type__c = 'Primary', matching Claim__c.Net_Paid_Amount__c exactly.

Given the fix is deployed,
When QA re-runs the report against the 340 COB claim sample set,
Then the total paid variance is $0.00 compared to the Claim__c.Net_Paid_Amount__c aggregate.

## Solution

Fix the custom Report Type 'Claims with Payments' by adding a cross-filter or modifying the underlying report to include a field filter on Payment__c.Payment_Type__c = 'Primary'. Additionally, update the summary formula column 'Total Paid Amount' to use a conditional formula: IF(Payment__c.Payment_Type__c = 'Primary', Payment__c.Amount__c, 0) summed at the report level. As a secondary safeguard, add a validation in the PaymentEngine Apex trigger (PaymentTriggerHandler.cls) to stamp Claim__c.COB_Secondary_Paid__c separately from Claim__c.Net_Paid_Amount__c to prevent future aggregation ambiguity. Deploy report change via Report Builder; no metadata deployment required. Apex trigger change requires sandbox regression and change set deployment.

## Description

**Bug Description:**
The Provider Network Analytics dashboard (delivered in INS-1438) incorrectly includes PNA records in the 'Active Contracted Providers' count when PNA__c.Termination_Date__c equals TODAY(). These providers should be counted as terminated, not active, as of their termination date.

**Steps to Reproduce:**
1. Create a Provider_Network_Agreement__c record with PNA__c.Status__c = 'Active' and PNA__c.Termination_Date__c = TODAY().
2. Navigate to the Provider Network Summary Lightning page.
3. Observe the 'Network Coverage Analytics' FlexCard — the provider is counted in the 'Active Contracted' metric.
4. Expected: Provider should NOT appear in the Active count when Termination_Date__c = TODAY().
5. Actual: Provider IS counted as Active, inflating the active provider count by the number of same-day terminations.

**Impact:** On the first of each month (typical termination batch date), active provider counts are overstated by an average of 12–18 providers based on current PNA data volume, causing network adequacy reports submitted to state regulators to reflect incorrect counts.

**Root Cause (suspected):** DataRaptor Extract filter condition uses `PNA__c.Termination_Date__c >= TODAY()` instead of `PNA__c.Termination_Date__c > TODAY()`, inclusive of today's date.

## Acceptance Criteria

As per Description.

Given a PNA__c record where PNA__c.Termination_Date__c = TODAY(),
When the Network Coverage Analytics FlexCard loads,
Then that provider is NOT included in the Active Contracted count.

Given a PNA__c record where PNA__c.Termination_Date__c = YESTERDAY(),
When the FlexCard loads,
Then that provider is NOT included in the Active count and appears in the 'Recently Terminated' section.

Given a PNA__c record where PNA__c.Termination_Date__c = TOMORROW(),
When the FlexCard loads,
Then that provider IS included in the Active Contracted count.

## Solution

Update the DataRaptor Extract 'DR_Extract_ActivePNA' filter condition from `Termination_Date__c >= :TODAY` to `Termination_Date__c > :TODAY` (strict greater-than). Also update the SAQL filter in the CRM Analytics dataset recipe for the 'Network_PNA_Active' dataset to use `termination_date > toDate("yyyy-MM-dd", now())`. Deploy updated DataRaptor via OmniStudio deployment tool to Production. Regression test with boundary date scenarios: Termination_Date = yesterday, today, tomorrow, and null. Update the DataRaptor Extract unit test class 'DR_ActivePNA_Test' with new boundary assertions. No Apex changes required.

## Description

As a Provider Relations Specialist, I need a dashboard component visualizing the active Network_Termination_Case__c pipeline grouped by termination reason and effective date so that I can track voluntary vs. involuntary terminations, ensure Provider_Change_Notification__c (PCN) letters are issued within the contractually required 60-day notice window, and prevent claims adjudication errors caused by adjudicating claims against terminated providers.

## Acceptance Criteria

Given Network_Termination_Case__c records where Network_Termination_Case__c.Status__c NOT IN ('Completed', 'Cancelled'),
When the 'Provider Termination Pipeline' dashboard component renders,
Then a stacked bar chart displays count of cases grouped by:
  - X-axis: Network_Termination_Case__c.Effective_Termination_Date__c (grouped by month)
  - Stack segments: Network_Termination_Case__c.Termination_Reason__c (Voluntary | Involuntary | Credentialing Failure | Fraud | Contract Non-Renewal)

Given a Network_Termination_Case__c record where Effective_Termination_Date__c is within 60 days of TODAY AND Network_Termination_Case__c.PCN_Issued__c = FALSE,
When the component renders,
Then the case is highlighted in red in an accompanying list view tile and Provider_Relations_Specialist__c is notified via an automated Salesforce Task with Subject = 'PCN Required: [Provider Name] - Termination in X days'.

Given the dashboard component is rendered,
When the Provider Relations Specialist clicks a bar segment,
Then a drill-down list displays the underlying Network_Termination_Case__c records with fields: Case_Number__c, Provider__c.Name, Provider__c.NPI__c, Effective_Termination_Date__c, Termination_Reason__c, PCN_Issued__c, PCN_Issue_Date__c.

Given a Network_Termination_Case__c.Status__c transitions to 'Completed',
When the dashboard refreshes,
Then the completed case is removed from the pipeline chart and the count decreases accordingly.

Given the dashboard is accessed by a Compliance Officer role,
When they view the component,
Then all termination cases across all Provider Relations Specialists are visible (not filtered to owner).

## Solution

Build dashboard using a custom Report Type on Network_Termination_Case__c with lookup to Provider__c. Stacked bar chart rendered as a standard Salesforce Dashboard chart component. PCN alert logic implemented via a Record-Triggered Flow (TerminationPCNAlertFlow) on Network_Termination_Case__c, triggered on create/update, with scheduled path evaluating daily: IF Effective_Termination_Date__c <= TODAY + 60 AND PCN_Issued__c = FALSE, create a Task record assigned to Provider_Relations_Specialist__c. Drill-down functionality uses a linked report with URL filter parameters. Compliance Officer visibility achieved via a Criteria-Based Sharing Rule sharing all Network_Termination_Case__c records to the 'Compliance Officer' Public Group. Permission Set Group 'Provider_Relations_PSG' scoped to owner-filtered view.

## Description

As an Underwriting Analyst, I need a Fee Schedule Utilization report that shows the top 25 CPT codes by claims volume, their contracted rate from the applicable Fee_Schedule__c, the average billed amount from Claim_Line__c, and the variance between billed and contracted amounts so that I can identify high-cost CPT codes where contracted rates may need renegotiation and inform actuarial pricing models.

The report must join Claim_Line__c CPT data with the active Fee_Schedule__c line items and must be filterable by Provider__c, Facility__c, and LOB.

## Acceptance Criteria

Given an Underwriting Analyst accesses the 'Fee Schedule Utilization' report,
When the report loads,
Then it displays the top 25 CPT codes (Claim_Line__c.CPT_Code__c) ranked by COUNT(Claim_Line__c.Id) DESC with the following columns:
  - Claim_Line__c.CPT_Code__c
  - Claim_Line__c.CPT_Description__c
  - COUNT(Claim_Line__c.Id) as 'Claim Line Volume'
  - AVG(Claim_Line__c.Billed_Amount__c) as 'Avg Billed Amount'
  - Fee_Schedule__c.Contracted_Rate__c (matched on Fee_Schedule__c.CPT_Code__c = Claim_Line__c.CPT_Code__c)
  - Variance = AVG(Claim_Line__c.Billed_Amount__c) - Fee_Schedule__c.Contracted_Rate__c

Given the report is displayed,
When the Variance column value is positive (billed > contracted),
Then the Variance value is displayed in red text.

Given the report is displayed,
When the Variance column value is negative or zero (billed <= contracted),
Then the Variance value is displayed in green text.

Given the Underwriting Analyst applies the Provider filter,
When a specific Provider__c is selected,
Then the report recalculates using only Claim_Line__c records WHERE Claim_Line__c.Claim__r.Provider__c = selected Provider__c.Id.

Given the report is filtered by LOB,
When LOB = 'Workers Comp' is selected,
Then only Claim_Line__c records linked to Claim__c.LOB__c = 'Workers Comp' are included AND the Fee_Schedule__c join applies only to Fee_Schedule__c records WHERE Fee_Schedule__c.LOB__c = 'Workers Comp'.

Given the report is run,
When there is no matching Fee_Schedule__c.Contracted_Rate__c for a given CPT code,
Then the Contracted Rate column displays 'No Rate on File' and the Variance column displays 'N/A' for that row.

## Solution

Build a custom Salesforce Report Type joining Claim_Line__c (primary) with Claim__c (parent via Claim_Line__c.Claim__c) and Fee_Schedule__c (custom lookup Fee_Schedule__c.CPT_Code__c matched via a junction or formula-based relationship). Due to limitations of standard Report Types for multi-object CPT matching, implement a CRM Analytics Dataflow that joins the Claim_Line__c dataset with the Fee_Schedule__c dataset on CPT_Code__c using a Flatten transformation. Build the top-25 CPT ranking as a SAQL query with ORDER BY volume DESC LIMIT 25. Variance field computed as a SAQL calculated column. Conditional color formatting for Variance implemented via CRM Analytics conditional formatting rules. Provider and LOB filters implemented as dashboard-level global filters. Report embedded as a CRM Analytics lens within the Underwriting Analytics Lightning App page.

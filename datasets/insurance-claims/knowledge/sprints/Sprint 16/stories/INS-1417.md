## Description

As a Credentialing Specialist, I need a FlexCard-based pipeline view of all active Credentialing_Case__c records assigned to me, grouped by stage and showing days-in-stage, so that I can prioritize cases at risk of breaching credentialing SLA thresholds and meet NCQA turnaround requirements.

The FlexCard must surface CAQH verification status and flag cases where the provider's CAQH profile has not been attested within the last 120 days.

## Acceptance Criteria

Given a Credentialing Specialist is logged in,
When they open the Credentialing Workqueue page,
Then a FlexCard displays all Credentialing_Case__c records WHERE Credentialing_Case__c.OwnerId = current user AND Credentialing_Case__c.Status__c NOT IN ('Approved','Denied','Withdrawn').

Given the FlexCard is rendered,
When records are displayed,
Then each card shows: Credentialing_Case__c.Name, Provider__c.Name (via lookup), Credentialing_Case__c.Stage__c, Credentialing_Case__c.Days_In_Stage__c (formula field), Credentialing_Case__c.CAQH_Profile_ID__c, Credentialing_Case__c.CAQH_Last_Attested_Date__c.

Given a Credentialing_Case__c.CAQH_Last_Attested_Date__c < TODAY()-120,
When the FlexCard renders that card,
Then a yellow warning badge 'CAQH Re-attestation Required' is displayed on the card.

Given a Credentialing_Case__c.Days_In_Stage__c > 14,
When the FlexCard renders that card,
Then the card border is highlighted in red and a tooltip reads 'SLA Breach Risk'.

Given the FlexCard is displayed,
When the Credentialing Specialist clicks 'Open Case',
Then the user is navigated to the Credentialing_Case__c record page.

-Given a bulk reassignment button is present on the FlexCard, When clicked, Then selected cases are reassigned via a modal.- (Deprecated: bulk reassignment descoped to Phase 3)

Given the FlexCard loads,
When there are no active cases assigned to the current user,
Then an empty state message 'No active credentialing cases assigned to you' is displayed.

## Solution

Build an OmniStudio FlexCard using a DataRaptor Extract as the data source, querying Credentialing_Case__c with a SOQL filter on OwnerId and Status__c. Days_In_Stage__c implemented as a formula field (Date difference between Stage_Entry_Date__c and TODAY()). CAQH re-attestation badge implemented using FlexCard conditional visibility state with condition CAQH_Last_Attested_Date__c < TODAY()-120. SLA breach red border implemented as a FlexCard state with condition Days_In_Stage__c > 14. FlexCard deployed to a new 'Credentialing Workqueue' Lightning App Page via Lightning App Builder. DataRaptor caches results for 5 minutes to reduce SOQL query volume.

## Description

As a Compliance Officer, I need an audit report that tracks Explanation_of_Benefits__c (EOB) issuance timeliness against state-mandated prompt payment windows and reconciles Electronic Remittance Advice (ERA) transactions against corresponding Payment__c records so that I can identify regulatory compliance gaps, prepare for state DOI audits, and file accurate prompt payment attestations.

This story delivers the bulk export capability referenced in the deprecation note in INS-1423.

## Acceptance Criteria

Given the Compliance Officer navigates to the 'EOB & ERA Compliance Audit' report,
When the report runs for a selected date range,
Then it returns all Explanation_of_Benefits__c records with the following columns:
  - Explanation_of_Benefits__c.Name
  - Explanation_of_Benefits__c.EOB_Issue_Date__c
  - Explanation_of_Benefits__c.Claim__c (linked Claim__c.Name)
  - Claim__c.Date_of_Service__c
  - Explanation_of_Benefits__c.Days_to_Issue__c (calculated field: EOB_Issue_Date__c - Claim__c.Received_Date__c)
  - Explanation_of_Benefits__c.Prompt_Payment_Compliant__c (checkbox)
  - Explanation_of_Benefits__c.State_Jurisdiction__c

Given the report,
When Explanation_of_Benefits__c.Days_to_Issue__c > the state-mandated threshold stored in Explanation_of_Benefits__c.State_Prompt_Payment_Days__c,
Then Explanation_of_Benefits__c.Prompt_Payment_Compliant__c = false and the row is flagged 'Non-Compliant' with red conditional formatting.

Given the ERA reconciliation section,
When a Payment__c record has Payment__c.ERA_Transaction_ID__c populated,
Then the report confirms Payment__c.Amount__c matches the corresponding ERA transaction amount stored in Payment__c.ERA_Matched_Amount__c; any variance > $0.01 is flagged as 'ERA Mismatch'.

Given the report summary footer,
When the report runs,
Then it displays: total EOB count, % Prompt_Payment_Compliant__c = true, count of ERA mismatches, and total dollar variance from ERA mismatches.

Given the Compliance Officer clicks 'Export to CSV',
When the export runs,
Then a CSV file is generated containing all report columns including Claim__c.ICD10_Primary_Code__c and Claim__c.CPT_Code__c for audit evidence purposes, and the export is logged in a custom Audit_Export_Log__c record with fields: Exported_By__c, Export_Timestamp__c, Report_Name__c, Record_Count__c.

Given a user without the 'Compliance_Officer' Permission Set Group,
When they attempt to run this report,
Then they receive an insufficient privileges error and the export button is not rendered.

-Given the report auto-emails to the state DOI on a monthly schedule- (deprecated — regulatory submission workflow descoped to Phase 3 per compliance team direction)

-Given ICD-9 codes are included in the export columns- (deprecated — ICD-9 fully retired; ICD-10 only per CMS mandate)

## Solution

Build a custom Report Type 'EOB with Claims and Payments' joining Explanation_of_Benefits__c (primary) → Claim__c (via Claim__c lookup) → Payment__c (via Claim__c child relationship). Add custom formula fields: Days_to_Issue__c (DATE formula) and Prompt_Payment_Compliant__c (formula checkbox comparing Days_to_Issue__c to State_Prompt_Payment_Days__c). ERA Mismatch flag implemented as a custom report summary formula. Conditional formatting applied via Report Builder highlight rules. CSV export button implemented as a custom LWC ('ComplianceExportBtn') embedded in the report header via a custom Report Header Component, which invokes an Apex method (ComplianceReportExportService.cls) to generate the CSV via ContentVersion and logs the Audit_Export_Log__c record via a Record-Triggered Flow on ContentVersion creation. Report access restricted via folder sharing limited to the 'Compliance_Ops' Public Group.

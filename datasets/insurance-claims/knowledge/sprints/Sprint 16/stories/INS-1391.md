## Description

As a Broker Relations Coordinator, I need an OmniStudio FlexCard on the Broker (Partner Account) record page that displays real-time production analytics — including total written premium volume, active policy count, and open claims count by LOB — so that I can assess broker performance at a glance during account review calls, identify underperforming books of business, and prioritize relationship management activities without navigating to separate report tabs.

## Acceptance Criteria

Given a Broker Relations Coordinator opens a Partner Account record where Account.RecordType.DeveloperName = 'Broker_Account',
When the 'Broker Production Analytics' FlexCard renders,
Then the following metrics are displayed in card tiles:
  - Total Written Premium: SUM(Policy__c.Annual_Premium__c) WHERE Policy__c.Broker__c = current Account.Id AND Policy__c.Status__c = 'Active'
  - Active Policy Count: COUNT(Policy__c) WHERE Policy__c.Broker__c = current Account.Id AND Policy__c.Status__c = 'Active'
  - Open Claims Count: COUNT(Claim__c) WHERE Claim__c.Policy__r.Broker__c = current Account.Id AND Claim__c.Status__c NOT IN ('Closed','Denied','Withdrawn')
  - Open Claims by LOB: breakdown table showing count per Claim__c.LOB__c value

Given the FlexCard is rendered,
When the Broker Relations Coordinator clicks the 'Active Policy Count' tile,
Then a flyout panel displays a list of the underlying Policy__c records with columns: Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Annual_Premium__c, Policy__c.Group_Service_Agreement__c (GSA link).

Given the FlexCard is rendered,
When Total Written Premium is calculated,
Then the value is formatted as USD currency with comma separators and displayed as '$X,XXX,XXX'.

Given a Broker Account has no associated Policy__c records,
When the FlexCard renders,
Then all metric tiles display '0' or '$0.00' with a helper text 'No active policies on record' — no null pointer or blank tile is shown.

Given the FlexCard is accessed by a user without the 'Broker_Relations_Coordinator' or 'Broker_Relations_Manager' Permission Set,
When they open a Broker Account record page,
Then the FlexCard component is not visible on the Lightning Record Page (controlled via Lightning App Builder visibility rule on Permission Set).

-Given a sub-tile showing Group_Service_Agreement__c (GSA) renewal dates was included in the original wireframe-
-Then GSA renewal date nearest to expiry would surface in the FlexCard header-

## Solution

Build an OmniStudio FlexCard (BrokerProductionAnalyticsFC) with a DataRaptor Extract (DR_BrokerProductionMetrics) as the primary data source. The DataRaptor executes aggregate SOQL against Policy__c and Claim__c filtered by the current record's Account.Id passed as an input parameter. The flyout panel for Active Policy Count implemented as a child FlexCard (BrokerPolicyListFC) triggered by a FlexCard Action of type 'Show/Hide'. Currency formatting applied via FlexCard field formatting options (type: Currency, locale: en-US). Empty state handled by a FlexCard conditional visibility rule on the metric tile blocks (show if count > 0; show helper text block if count = 0). Lightning Record Page visibility rule set to 'Permission Set = Broker_Relations_Coordinator OR Broker_Relations_Manager'. DataRaptor and FlexCard deployed via OmniStudio export/import and tracked in the INS project Salesforce DX repository under force-app/main/default/omniStudio.

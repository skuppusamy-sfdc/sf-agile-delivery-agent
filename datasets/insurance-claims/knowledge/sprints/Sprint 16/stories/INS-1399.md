## Description

**Bug Report**

**Summary:** The 'Active Terminations by Region' bar chart on the Network Termination Analytics dashboard fails to render and displays a blank widget with the error 'Query returned no results' for Salesforce users whose profile has access to more than one Network_Region__c value on Network_Termination_Case__c. Single-region users are unaffected.

**Steps to Reproduce:**
1. Log in as a user with the 'Multi_Region_Network_Manager' Permission Set assigned (grants access to Network_Region__c values: 'Northeast', 'Southeast', 'Midwest').
2. Navigate to the 'Network Termination Analytics' CRM Analytics dashboard.
3. Observe the 'Active Terminations by Region' bar chart widget.
4. Note the widget displays 'Query returned no results' despite Network_Termination_Case__c records existing across all three regions with Status__c = 'Active'.
5. Log out and log in as a single-region user (access to 'Northeast' only).
6. Observe the same widget — chart renders correctly showing Northeast termination counts.

**Expected Result:** The 'Active Terminations by Region' chart must render all regions accessible to the running user, aggregating Network_Termination_Case__c records by Network_Region__c where Status__c IN ('Active', 'Pending Termination').

**Actual Result:** Chart renders no data for multi-region users. Suspected root cause: the Analytics Row-Level Security predicate is applying an equality filter (Network_Region__c = single value) rather than an IN filter for users with multiple region entitlements, resulting in a null intersection and empty query results.

**Environment:** Acme Insurance FULL sandbox and Production
**Affected Users:** 4 confirmed Network Operations Managers with multi-region access
**Severity:** High — blocking Sprint 16 dashboard UAT sign-off
**Frequency:** 100% reproducible for multi-region users

## Acceptance Criteria

As per Description. Additionally:
- After fix, multi-region users must see aggregated termination counts for all regions they are entitled to, with each region displayed as a separate bar
- Single-region user behavior must be unaffected (regression)
- Verify the Analytics RLS predicate correctly resolves to an IN clause when a user has multiple Network_Region__c entitlements stored in Custom Permission or related object
- Network_Termination_Case__c.Status__c filter must include both 'Active' and 'Pending Termination' values in the chart SAQL query

## Solution

Investigate and update the CRM Analytics Row-Level Security predicate class (NetworkTerminationRLSPredicate) to handle multi-value region entitlements. Replace the current equality predicate ('Network_Region__c' == userAttribute('network_region')) with an IN-based predicate that resolves the user's region entitlements from a related junction object (User_Region_Entitlement__c) or Custom Permission Set Group membership. Update the Analytics User Attribute mapping to pass a pipe-delimited list of region values and update the SAQL predicate to use the in() function: Network_Region__c in [split(userAttribute('network_regions'), '|')]. Update the dataset recipe to re-register the corrected RLS predicate. Run dataflow refresh and validate in FULL sandbox with multi-region test user before deploying to production. Update SAQL status filter to: Status__c in ['Active', 'Pending Termination'].

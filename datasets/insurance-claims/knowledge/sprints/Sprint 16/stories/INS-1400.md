## Description

As a Claims Operations Manager, I need a real-time Salesforce dashboard displaying adjudication throughput KPIs so that I can monitor daily claim processing velocity, identify bottlenecks, and ensure SLA compliance across all Lines of Business (LOB).

The dashboard must surface metrics including: total Claim__c records adjudicated per day, average adjudication cycle time (FNOL to payment), denial rate by LOB, and pending Claim_Line__c items awaiting CPT code validation. Data must refresh every 5 minutes via streaming and cover all active TPA-managed claims.

## Acceptance Criteria

Given a Claims Operations Manager with the 'Claims_Ops_Manager' Permission Set Group assigned,
When they navigate to the 'Claims Operations Dashboard' Lightning App page,
Then they see a KPI tile showing count of Claim__c records where Claim__c.Status__c = 'Adjudicated' for the current business day, updated within 5 minutes of record change.

Given the dashboard is loaded,
When the 'Adjudication Cycle Time' tile renders,
Then it displays the average number of calendar days between Claim__c.FNOL_Date__c and Claim__c.Adjudication_Date__c for claims closed in the trailing 30 days, segmented by Claim__c.Line_of_Business__c.

Given claims exist with Claim__c.Denial_Reason__c populated,
When the 'Denial Rate by LOB' chart renders,
Then it shows a bar chart grouping denied claims (Claim__c.Status__c = 'Denied') by Claim__c.Line_of_Business__c as a percentage of total submitted claims per LOB.

Given Claim_Line__c records with Claim_Line__c.CPT_Validation_Status__c = 'Pending',
When the 'Pending CPT Validation' tile renders,
Then it displays the count and a drill-down list view filtered to those records with columns: Claim_Line__c.Name, Claim_Line__c.CPT_Code__c, Claim_Line__c.Submitted_Amount__c, and related Claim__c.Policy__c.

-Given the manager clicks 'Export', Then a CSV export of all visible tiles fires via a Conga template-

Given the dashboard is accessed on a mobile device,
When the screen width is below 768px,
Then all KPI tiles stack vertically and remain readable without horizontal scrolling.

## Solution

Build a Lightning App Page using CRM Analytics (Tableau CRM) with a dedicated 'Claims Ops' dataset sourced via a DataSync job on Claim__c and Claim_Line__c objects. Use SAQL queries for cycle time calculations. KPI tiles implemented as CRM Analytics dashboard widgets with 5-minute auto-refresh enabled via the dashboard's schedule property. Drill-downs link back to standard Salesforce list views filtered by relevant Claim__c criteria. A Permission Set Group 'Claims_Ops_Manager' gates access. Mobile responsiveness handled via CRM Analytics responsive layout templates. Deprecate the legacy Visualforce-based 'ClaimsSummary' page (VF_ClaimsSummary) once UAT sign-off is obtained.

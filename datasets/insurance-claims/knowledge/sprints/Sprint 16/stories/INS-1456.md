## Description

As a Provider Relations Specialist, I need a consolidated 'Network Participation Summary' FlexCard on the Provider 360 page that displays all active Provider_Network_Agreement__c, Group_Service_Agreement__c, and Fee_Schedule__c associations for a given provider, so that I can answer provider inquiries about their network participation and reimbursement schedules during a single interaction without toggling between multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the 'Network Participation Summary' FlexCard loads,
Then it displays three tabbed sections: 'Active PNAs', 'Active GSAs', 'Fee Schedules'.

Given the 'Active PNAs' tab is selected,
When data renders,
Then each row shows: PNA__c.Name, PNA__c.Network_Region__c, PNA__c.Contract_Start_Date__c, PNA__c.Contract_End_Date__c, PNA__c.Contract_Tier__c, PNA__c.Status__c — filtered to Status__c = 'Active'.

Given the 'Active GSAs' tab is selected,
When data renders,
Then each row shows: GSA__c.Name, GSA__c.Employer_Group__c (lookup name), GSA__c.Effective_Date__c, GSA__c.Termination_Date__c — filtered to GSA__c.Status__c = 'Active' AND GSA__c.Provider__c = current Provider__c Id.

Given the 'Fee Schedules' tab is selected,
When data renders,
Then each row shows: Fee_Schedule__c.Name, Fee_Schedule__c.LOB__c, Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.Rate_Basis__c — linked via PNA__c.Associated_Fee_Schedule__c.

Given a Provider__c has zero active PNAs,
When the 'Active PNAs' tab renders,
Then an empty state message reads: 'No active network agreements on file.'

Given any row in 'Active PNAs' is clicked,
When the action fires,
Then the user navigates to the corresponding PNA__c record detail page.

## Solution

Build OmniStudio FlexCard 'ProviderNetworkSummaryCard' with three child DataRaptor Extract data sources: (1) DR_GetActivePNAs querying PNA__c WHERE Provider__c = :recordId AND Status__c = 'Active', (2) DR_GetActiveGSAs querying GSA__c WHERE Provider__c = :recordId AND Status__c = 'Active', (3) DR_GetFeeSchedules querying Fee_Schedule__c via PNA__c.Associated_Fee_Schedule__c join. Tabbed layout implemented using FlexCard Tab component. Row-click navigation action uses NavigationMixin pattern via FlexCard Action of type 'Navigation'. FlexCard deployed to Provider__c Lightning Record Page via App Builder in the 'Network' accordion section. Cacheable Apex method @AuraEnabled(cacheable=true) used for DR fallback on large datasets exceeding DataRaptor 2000-row limit.

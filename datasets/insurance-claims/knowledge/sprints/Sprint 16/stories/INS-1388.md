## Description

As an Underwriting Analyst, I need a Fee Schedule utilization report that cross-references Claim_Line__c CPT codes against Fee_Schedule__c allowed amounts and displays reimbursement variance (billed amount vs. allowed amount vs. paid amount) aggregated by CPT code and provider so that I can identify pricing anomalies, validate Fee Schedule accuracy, and support annual rate renegotiation with high-volume providers.

## Acceptance Criteria

Given Claim_Line__c records where Claim_Line__c.Status__c = 'Adjudicated' and Claim__c.Date_of_Service__c falls within the selected date range,
When the 'Fee Schedule Utilization' report runs,
Then the following columns are returned:
  - Claim_Line__c.CPT_Code__c
  - Fee_Schedule__c.Allowed_Amount__c (matched via Fee_Schedule__c.CPT_Code__c = Claim_Line__c.CPT_Code__c AND Fee_Schedule__c.Effective_Date__c <= Claim__c.Date_of_Service__c)
  - Claim_Line__c.Billed_Amount__c
  - Claim_Line__c.Allowed_Amount__c
  - Claim_Line__c.Paid_Amount__c
  - Variance_Amount__c = Fee_Schedule__c.Allowed_Amount__c - Claim_Line__c.Paid_Amount__c
  - Variance_Pct__c = (Variance_Amount__c / Fee_Schedule__c.Allowed_Amount__c) * 100
  - Provider__c.Name
  - Provider__c.NPI__c
  - Claim_Line__c.ICD10_Primary_Diagnosis__c

Given the report is grouped by Claim_Line__c.CPT_Code__c,
When a summary row is rendered,
Then it displays: total claim line count, SUM(Billed_Amount__c), SUM(Allowed_Amount__c), SUM(Paid_Amount__c), and AVG(Variance_Pct__c) per CPT code.

Given Variance_Pct__c > 10% for a CPT code group,
When the report is rendered,
Then the summary row for that CPT code is highlighted in amber.

Given Variance_Pct__c > 25% for a CPT code group,
When the report is rendered,
Then the summary row is highlighted in red and a flag icon is displayed.

Given the Underwriting Analyst exports the report,
When they select 'Export to CSV',
Then all columns including variance calculations are included in the export with no truncation of CPT_Code__c values.

Given a Claim_Line__c record has no matching Fee_Schedule__c record for its CPT_Code__c on the date of service,
When the report runs,
Then the record is included with Fee_Schedule__c.Allowed_Amount__c = NULL and flagged in a 'No Fee Schedule Match' column = TRUE.

Given the report is accessed,
When the user does not have the 'Underwriting_Analyst' or 'Claims_Ops_Manager' Permission Set,
Then the report is not visible in the report folder and direct URL access returns an insufficient privileges error.

## Solution

Create a custom Salesforce Report Type with Claim_Line__c as the primary object, joined to Claim__c (parent), Provider__c (via Claim__c.Provider__c), and Fee_Schedule__c as a cross-filter lookup resolved via a custom junction or formula field (Claim_Line__c.Matched_Fee_Schedule__c lookup populated by an Apex Trigger on Claim_Line__c insert/update — FeeScheduleMatchTrigger.cls). Variance formula fields (Variance_Amount__c, Variance_Pct__c) deployed on Claim_Line__c. Conditional highlighting configured via report formatting rules. 'No Fee Schedule Match' flag implemented as a formula field (Claim_Line__c.No_FS_Match__c = ISBLANK(Matched_Fee_Schedule__c)). Report folder access restricted via Permission Set Group 'Underwriting_Analytics_PSG'. Export includes all fields via standard Salesforce report export; CPT truncation prevented by ensuring CPT_Code__c field type is Text(10).

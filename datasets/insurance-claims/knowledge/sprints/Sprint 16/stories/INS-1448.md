## Description

**Bug Report – INS-1448**

**Summary:** The Provider Network Agreement expiry report (introduced in Sprint 14) is undercounting expiring PNAs because PNA records where Provider_Network_Agreement__c.Effective_End_Date__c IS NULL are being excluded from the 'Expiring in 90 Days' bucket, even though NULL end dates should be treated as evergreen/no-expiry and reported separately — not silently dropped.

**Steps to Reproduce:**
1. Navigate to the 'PNA Expiry Dashboard' in the Provider Network app.
2. Set the date filter to 'Next 90 Days'.
3. Observe the 'Expiring Soon' tile count.
4. Run a manual SOQL query: SELECT Id, Name, Effective_End_Date__c FROM Provider_Network_Agreement__c WHERE Status__c = 'Active'.
5. Note that records with Effective_End_Date__c = NULL are present in the org but absent from the dashboard tile.

**Expected Behavior:** NULL Effective_End_Date__c records should appear in a separate 'Evergreen / No Expiry Date' category tile on the dashboard, not be excluded entirely. The 'Expiring Soon' tile should only reflect records with a populated Effective_End_Date__c within the next 90 days.

**Actual Behavior:** NULL Effective_End_Date__c records are silently dropped by the SOQL WHERE clause filter (WHERE Effective_End_Date__c <= NEXT_N_DAYS:90), causing the 'Expiring Soon' count to be understated and the 'Evergreen' tile to show 0 instead of the actual count (currently 47 records in production).

**Environment:** Production & Full Sandbox
**Severity:** High — affects compliance reporting accuracy for MCO audits
**Affected Component:** PnaExpiryDashboardController.cls, line 38

## Acceptance Criteria

As per Description.

Given the fix is deployed,
When the PNA Expiry Dashboard loads,
Then Provider_Network_Agreement__c records where Effective_End_Date__c IS NULL AND Status__c = 'Active' must appear in the 'Evergreen / No Expiry Date' tile with accurate count.

Given the fix is deployed,
When the 'Expiring in 90 Days' tile is queried,
Then only records where Effective_End_Date__c != NULL AND Effective_End_Date__c <= NEXT_N_DAYS:90 AND Status__c = 'Active' must be included.

## Solution

Fix PnaExpiryDashboardController.cls line 38: split the single SOQL query into two queries — (1) expiring records with populated Effective_End_Date__c using the existing date filter, (2) evergreen records using WHERE Effective_End_Date__c = NULL AND Status__c = 'Active'. Update the LWC 'pnaExpiryDashboard' to render both result sets in separate KPI tiles. Add a unit test in PnaExpiryDashboardControllerTest.cls covering the NULL end date scenario with a minimum of 95% code coverage. Deploy via change set to Full Sandbox for QA validation before production push.

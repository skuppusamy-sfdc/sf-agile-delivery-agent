## Description

As a Broker Relations Coordinator, I need a Group/Employer onboarding analytics dashboard that visualizes Group_Service_Agreement__c execution rates by stage and a funnel view of employer accounts progressing through onboarding milestones so that I can identify drop-off points, accelerate stalled onboarding cases, and report pipeline health to leadership.

The dashboard must integrate with DocuSign envelope status data synced to Group_Service_Agreement__c.DocuSign_Envelope_Status__c.

## Acceptance Criteria

Given the Broker Relations Coordinator navigates to the 'Group Onboarding Analytics' dashboard,
When the page loads,
Then a GSA Execution Rate KPI tile displays: (COUNT of Group_Service_Agreement__c where DocuSign_Envelope_Status__c = 'Completed' AND CreatedDate = THIS_QUARTER) / (COUNT of all GSA records created THIS_QUARTER) as a percentage.

Given the Onboarding Stage Funnel chart renders,
When displayed,
Then it shows a horizontal funnel visualization with stages mapped from Group_Service_Agreement__c.Onboarding_Stage__c picklist values: Prospect → Application Received → Underwriting Review → GSA Sent → GSA Executed → Active, with record counts at each stage.

Given the funnel is displayed,
When the drop-off rate between 'GSA Sent' and 'GSA Executed' stages exceeds 25%,
Then the funnel segment is highlighted in amber and a tooltip reads 'High drop-off — review DocuSign envelope aging'.

Given the dashboard includes a DocuSign Envelope Aging table,
When rendered,
Then it lists Group_Service_Agreement__c records where DocuSign_Envelope_Status__c = 'Sent' and DATEDIFF(TODAY(), GSA_Sent_Date__c) > 7, showing columns: Account.Name, Group_Service_Agreement__c.Contract_Number__c, GSA_Sent_Date__c, Broker__c, and Days_Pending__c.

Given a Broker Relations Coordinator clicks an Account row in the aging table,
When navigation occurs,
Then the user is taken to the Employer Account (Business Account) record page.

-Given the dashboard, When a coordinator clicks 'Resend DocuSign', Then a new envelope is sent directly from the dashboard.- (deprecated — DocuSign resend must be triggered from the GSA record OmniScript per security policy; dashboard is read-only)

Given the dashboard filter for Broker (lookup to Broker__c object) is applied,
When a specific broker is selected,
Then all dashboard components filter to Group_Service_Agreement__c records where Broker__c = selected value.

## Solution

Build CRM Analytics dashboard 'Group_Onboarding_Analytics_Dash' with three widgets: (1) KPI tile using SAQL COUNT with filter on DocuSign_Envelope_Status__c = 'Completed'; (2) Funnel chart using SAQL GROUP BY Onboarding_Stage__c with custom sort order matching picklist sequence — implemented via a SAQL CASE WHEN sort index; (3) Aging table widget querying GSA dataset filtered by DocuSign_Envelope_Status__c = 'Sent' with computed Days_Pending__c measure (DATEDIFF on GSA_Sent_Date__c). DocuSign envelope status synced to Group_Service_Agreement__c.DocuSign_Envelope_Status__c via existing Platform Event (DocuSign_Status_Update__e) handled by a Record-Triggered Flow. Drop-off amber threshold implemented as a CRM Analytics conditional formatting rule (threshold: > 0.25 ratio between GSA Sent and GSA Executed counts). Dashboard secured via CRM Analytics app sharing with 'Broker_Relations_PSG' Permission Set Group. Broker filter implemented as a Global Filter widget bound to all three dashboard components.

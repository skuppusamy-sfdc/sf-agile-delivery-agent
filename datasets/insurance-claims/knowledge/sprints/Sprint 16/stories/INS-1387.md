## Description

As a Claims Operations Manager, I need a trend line chart on the Claims Operations Dashboard showing weekly EOB issuance volumes segmented by LOB (Medical, Dental, Vision, Workers Comp) and payment method (EFT, Check, ERA) over a rolling 13-week period so that I can monitor disbursement patterns, detect anomalies in ERA file generation, and support financial reconciliation reporting.

## Acceptance Criteria

Given Explanation_of_Benefits__c records where EOB__c.Issue_Date__c falls within the last 91 days (13 weeks),
When the 'EOB Issuance Trend' chart renders on the Claims Ops Dashboard,
Then the X-axis displays ISO week labels (e.g., 'W01 2025') and the Y-axis displays count of EOB records issued per week.

Given the chart is rendered,
When segmented by LOB,
Then separate series lines are drawn for each distinct value of EOB__c.LOB__c.

Given the chart is rendered,
When the user hovers over a data point,
Then a tooltip displays: Week label, LOB, Payment_Method__c (EFT | Check | ERA), and EOB count.

Given a Payment__c record is created with Payment__c.Payment_Method__c = 'ERA',
When the linked EOB__c record is included in the chart,
Then it is counted under the 'ERA' series for its corresponding week.

Given the dashboard is viewed,
When no EOB records exist for a given week-LOB combination,
Then the chart renders a zero-value data point (no gap in the trend line).

## Solution

Build a custom Salesforce Report on Explanation_of_Benefits__c joined to Payment__c via EOB__c.Payment__c lookup. Use report date grouping by Calendar Week on EOB__c.Issue_Date__c with a custom date range filter of 'Last 91 Days'. Add bucket field on Payment__c.Payment_Method__c grouping EFT, Check, ERA. Render as a Line Chart on the Claims Ops Dashboard Lightning page. Zero-fill gaps handled by a custom LWC wrapper (lwc/eobTrendChart) that post-processes the ConnectApi.ReportResults data to insert zero-count entries for missing week-LOB-method combinations before rendering via Chart.js embedded in the LWC.

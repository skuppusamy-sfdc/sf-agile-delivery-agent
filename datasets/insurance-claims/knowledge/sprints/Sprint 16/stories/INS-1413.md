## Description

As a Compliance Officer, I need an automated aging report and escalation workflow for Network_Termination_Case__c records that have exceeded their state-mandated notice period SLA — specifically surfacing cases where the Provider_Change_Notification__c (PCN) has not been issued within 30 days of Network_Termination_Case__c.Termination_Effective_Date__c — so that Acme Insurance avoids regulatory penalties for late PCN issuance under CMS network adequacy requirements.

## Acceptance Criteria

Given a Network_Termination_Case__c record where Network_Termination_Case__c.Status__c = 'PCN Pending' AND (TODAY() - Network_Termination_Case__c.Termination_Effective_Date__c) > 30,
When the 'Termination Case Aging' scheduled report runs at 07:00 CST Monday–Friday,
Then the record must appear in the 'Overdue PCN Issuance' section with columns: Network_Termination_Case__c.Case_Number__c, related Provider__c.Name, Provider__c.NPI__c, Network_Termination_Case__c.Termination_Reason__c, Network_Termination_Case__c.Termination_Effective_Date__c, Network_Termination_Case__c.State_Jurisdiction__c, and calculated 'Days_Overdue' formula.

Given a Network_Termination_Case__c record enters the 'Overdue PCN Issuance' bucket,
When the scheduled automation runs,
Then a Task must be created with Subject = 'URGENT: PCN Overdue — Regulatory Risk', assigned to the Network_Termination_Case__c.Case_Owner, with ActivityDate = TODAY() + 1, and a Chatter post must be made on the case record tagging the Compliance Officer user group '@ComplianceOfficers'.

Given the escalation Task has been open for 5 additional business days without Network_Termination_Case__c.Status__c changing to 'PCN Issued',
When the automation re-evaluates,
Then the Network_Termination_Case__c.Escalation_Level__c field must be set to 'VP Review' and an email alert must be sent to the VP_Network_Operations email group using the 'PCN_Escalation_VP' email template.

Given a Provider_Change_Notification__c record is created and PCN.Network_Termination_Case__c lookup is populated AND PCN.Status__c = 'Sent',
When the PCN is saved,
Then Network_Termination_Case__c.Status__c must automatically update to 'PCN Issued' and the overdue Task must be closed (Status = 'Completed') via a Record-Triggered Flow.

-Given the report includes a CMS Reporting Export button that submits directly to CMS HPMS portal via API- (deprecated — CMS direct submission deferred pending security review, manual export only for this sprint)

## Solution

Build a Salesforce Summary Report 'Termination Case Aging' on Network_Termination_Case__c with a cross-object formula field 'Days_Since_Termination_Effective' = TODAY() - Network_Termination_Case__c.Termination_Effective_Date__c. Schedule report via Report Subscription. For escalation automation: create a Scheduled-Triggered Flow 'TerminationCaseEscalationFlow' running daily at 07:00 CST, querying Network_Termination_Case__c records matching overdue criteria, creating Tasks and Chatter posts via Flow actions. A second path in the same Flow handles 5-day re-escalation by checking open Task age using a Get Records element on Task object filtered by WhatId and CreatedDate. Record-Triggered Flow on Provider_Change_Notification__c (fires on PCN.Status__c change to 'Sent') updates parent Network_Termination_Case__c via a field update and closes related Tasks. All flows deployed via Change Set with corresponding test coverage in TerminationEscalationFlowTest.cls.

## Description

As a Compliance Officer, I need a dedicated SIU (Special Investigations Unit) Referral Trend dashboard showing monthly referral volume by LOB, referral outcome distribution (Confirmed Fraud, Unsubstantiated, Pending), average investigation cycle time, and total dollar amount under investigation so that I can report on fraud prevention program effectiveness to senior leadership and regulatory bodies.

The dashboard must draw from Claim__c records where Claim__c.SIU_Referral_Flag__c = true and associated Case records of Record Type 'SIU Investigation'.

## Acceptance Criteria

Given a user with the 'Compliance Officer' Permission Set Group,
When they open the 'SIU Fraud Analytics' dashboard,
Then the following components are visible:
  1. Bar chart: Monthly SIU referral volume (last 12 months) grouped by Claim__c.LOB__c
  2. Donut chart: Referral outcome distribution (Case.SIU_Outcome__c values: 'Confirmed Fraud', 'Unsubstantiated', 'Referred to Law Enforcement', 'Pending')
  3. KPI tile: Average investigation cycle time in days (Case.ClosedDate - Case.CreatedDate WHERE RecordType.Name = 'SIU Investigation')
  4. KPI tile: Total dollar amount under active investigation (SUM Claim__c.Total_Billed_Amount__c WHERE Claim__c.SIU_Referral_Flag__c = true AND Case.Status != 'Closed')

Given the dashboard is loaded,
When the Compliance Officer selects a specific month bar in the trend chart,
Then a drill-down table appears showing individual Claim__c records: Claim__c.Name, Claim__c.Policy__c, Claim__c.LOB__c, Claim__c.Total_Billed_Amount__c, Case.SIU_Outcome__c.

Given the dashboard is open,
When the LOB filter is applied,
Then all four dashboard components refresh simultaneously to reflect only the selected LOB.

Given the dashboard is accessed,
When the user's profile does not include the 'Compliance Officer' Permission Set Group,
Then the dashboard is not visible in the app navigation and returns an insufficient privileges error if accessed via direct URL.

-Given a geographic heat map component is present showing SIU referrals by provider ZIP code, When loaded, Then it renders using Provider__c.BillingPostalCode.- (Deprecated: geographic map component deferred to Phase 3 due to data privacy review)

Given the dashboard data,
When data is exported by the Compliance Officer,
Then the export is logged in a custom Audit_Log__c record with fields: Audit_Log__c.User__c, Audit_Log__c.Action__c = 'Dashboard Export', Audit_Log__c.Timestamp__c, Audit_Log__c.Dashboard_Name__c = 'SIU Fraud Analytics'.

## Solution

Build a CRM Analytics (Tableau CRM) dashboard using a dedicated SIU Dataset sourced from a Dataflow joining Claim__c (filtered by SIU_Referral_Flag__c = true) with the Case object (RecordType = 'SIU Investigation') via Claim__c.Id → Case.Claim__c lookup. Bar chart and donut chart built as SAQL-driven chart widgets. KPI tiles use SAQL aggregation queries. Dashboard-level LOB filter implemented as a global filter binding to all widgets. Permission restriction enforced via CRM Analytics Asset Sharing set to 'Compliance Officer' Permission Set Group only. Dashboard export audit logging implemented via a Platform Event (DashboardExportEvent__e) published by a CRM Analytics notification webhook, consumed by an Apex trigger that inserts Audit_Log__c records.

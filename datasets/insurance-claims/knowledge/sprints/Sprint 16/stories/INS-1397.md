## Description

As a Claims Operations Manager, I need a Payment Disbursement Analytics dashboard that consolidates Payment__c and ERA reconciliation data so that I can track payment aging buckets, identify unreconciled ERA transactions, monitor outstanding balances by payer, and ensure timely payment processing in compliance with prompt payment regulations. This dashboard is referenced by INS-1394 for the deprecated ERA column migration.

## Acceptance Criteria

Given the Claims Operations Manager opens the 'Payment Disbursement Analytics' dashboard,
When the dashboard loads,
Then the following panels must be present:
  - 'Payment Aging Summary' bar chart with buckets: 0-30 days, 31-60 days, 61-90 days, 90+ days based on Payment__c.Payment_Due_Date__c
  - 'ERA Reconciliation Status' donut chart showing distribution of Payment__c.ERA_Status__c values ('Reconciled', 'Pending', 'Exception', 'Not Applicable')
  - 'Outstanding Balance by Payer' table showing Payer_Name__c and SUM(Payment__c.Outstanding_Balance__c)
  - 'Total Disbursed MTD' KPI tile showing SUM(Payment__c.Amount__c) where Payment_Date__c = current month and Payment__c.Status__c = 'Disbursed'

Given the 'Payment Aging Summary' chart,
When a payment bucket bar is clicked,
Then the dashboard must cross-filter all other panels to show only Payment__c records in that aging bucket

Given Payment__c.ERA_Status__c = 'Exception',
When the ERA Reconciliation Status donut segment is clicked,
Then a detail table must expand showing: Payment__c.Name, Claim__r.Claim_Number__c, Payment__c.Amount__c, Payment__c.ERA_Exception_Reason__c, Provider__r.Name

Given the dashboard date range filter,
When the user selects a custom date range via the date picker widget,
Then Payment_Date__c on Payment__c must be used as the filter dimension across all panels

Given a Payment__c record has been in ERA_Status__c = 'Pending' for more than 5 business days,
When the Outstanding Balance table renders,
Then those rows must be highlighted in amber to indicate stale ERA pending status

Given the dashboard is accessed by a user with 'TPA_Analyst' Permission Set,
When the dashboard loads,
Then the 'Outstanding Balance by Payer' panel must be scoped to only payers associated with that TPA user's TPA_Account__c relationship, enforced via Analytics Row-Level Security predicate on Payment__c.TPA_Account__c

## Solution

Create CRM Analytics dashboard 'Payment Disbursement Analytics' with datasets built from Payment__c, Claim__c, and Provider__c dataflow nodes. Payment aging bucket implemented via SAQL case expression on datediff(today, Payment_Due_Date__c). ERA Exception detail table implemented as a faceted lens triggered by donut segment selection. Amber highlight for stale ERA Pending records implemented via SAQL conditional color binding: if(ERA_Status__c == 'Pending' && datediff(today, ERA_Submitted_Date__c) > 5, '#FF8C00', '#FFFFFF'). Row-Level Security implemented via Analytics Security Predicate on TPA_Account__c filtered by running user's related TPA_Account__c, resolved via a custom Apex RLS class. Dashboard deployed to 'Claims Finance' Analytics App with sharing rules scoped to Claims Operations Manager role and TPA Analyst permission set.

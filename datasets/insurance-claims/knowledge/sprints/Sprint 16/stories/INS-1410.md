## Description

As a Credentialing Specialist, I need a FlexCard embedded on the Provider__c (Person Account) record page that displays all active Credentialing_Case__c records associated with the provider — including CAQH verification status, primary source verification (PSV) checklist completion percentage, and days-in-stage — so that I can manage the credentialing pipeline without navigating away from the Provider 360 page.

## Acceptance Criteria

Given a Provider__c record page is open and the provider has one or more Credentialing_Case__c records where Credentialing_Case__c.Status__c IN ('Initiated', 'PSV In Progress', 'Committee Review', 'Pending Approval'),
When the 'Credentialing Pipeline' FlexCard renders in the Provider 360 Lightning page,
Then each active case must display: Credentialing_Case__c.Case_Number__c, Credentialing_Case__c.Status__c, Credentialing_Case__c.CAQH_Profile_Status__c, Credentialing_Case__c.PSV_Completion_Pct__c as a progress bar (0–100%), Credentialing_Case__c.Days_In_Current_Stage__c, and Credentialing_Case__c.Assigned_Credentialing_Specialist__c.

Given the FlexCard is rendered,
When Credentialing_Case__c.Days_In_Current_Stage__c > 14,
Then the Days_In_Stage field must render with a red badge indicator and tooltip text 'Exceeds SLA threshold'.

Given the provider has no active Credentialing_Case__c records,
When the FlexCard renders,
Then the card must display an empty state message: 'No active credentialing cases. Click + to initiate a new credentialing request.' with a button that launches the 'Initiate_Credentialing' OmniScript.

Given the Credentialing Specialist clicks the 'Sync CAQH' action button on a FlexCard row,
When the action fires,
Then an Apex callout to the CAQH ProView API must be triggered, updating Credentialing_Case__c.CAQH_Profile_Status__c and Credentialing_Case__c.CAQH_Last_Sync_Date__c on the related case record within 10 seconds, and the FlexCard must refresh automatically.

-Given the FlexCard includes a tab for displaying NPPES NPI registry data inline- (deprecated — NPPES inline display moved to separate INS-1388 Provider Data Maintenance story)

## Solution

Build an OmniStudio FlexCard 'CredentialingPipelineFC' using a DataRaptor Extract sourcing Credentialing_Case__c filtered by Provider__c lookup and Status__c IN active values. Configure a conditional state on Days_In_Current_Stage__c > 14 to apply the red badge CSS class. Progress bar for PSV_Completion_Pct__c implemented as a custom FlexCard child element using the built-in percentage bar component. The 'Sync CAQH' action triggers an Integration Procedure 'CAQH_Sync_IP' which calls an Apex @InvocableMethod (CAQHIntegrationService.syncProviderProfile) making a named credential REST callout to CAQH ProView. FlexCard embedded on Provider__c Lightning Record Page via App Builder. Visibility controlled via a Permission Set Group 'Credentialing_Ops_PSG'.

## Description

As a Claims Operations Manager, I need an EOB summary analytics dashboard that displays payment accuracy rates, ERA reconciliation match rates, and COB adjustment totals across all finalized claims for the current and prior month so that I can identify systematic underpayment or overpayment patterns and ensure ERA files are reconciling correctly with Payment__c records in Salesforce.

Key objects: Explanation_of_Benefits__c (EOB), Payment__c, Claim__c, Claim_Line__c.

## Acceptance Criteria

Given EOB records exist with EOB__c.Status__c = 'Finalized',
When the 'EOB Payment Accuracy' dashboard loads,
Then a KPI tile displays the Payment Accuracy Rate = (count of EOB__c where EOB__c.Paid_Amount__c = EOB__c.Expected_Allowed_Amount__c) / (total finalized EOB count) × 100, for the current calendar month.

Given ERA files have been processed and Payment__c records created,
When the ERA Reconciliation tile renders,
Then it shows ERA Match Rate = (count of Payment__c where Payment__c.ERA_Reconciled__c = TRUE) / (total Payment__c count with Payment__c.Payment_Date__c in current month) × 100.

Given COB adjustments exist on Claim__c records (Claim__c.COB_Adjustment_Amount__c > 0),
When the COB summary panel renders,
Then total COB adjustment amount is displayed segmented by Claim__c.COB_Type__c (Primary, Secondary, Tertiary).

Given a Payment__c record has Payment__c.ERA_Reconciled__c = FALSE and Payment__c.Payment_Date__c < TODAY() - 3,
When it appears in the unreconciled payments list,
Then it is highlighted and includes Payment__c.Check_Number__c, Payment__c.Amount__c, Payment__c.Provider__c name, and a 'Reconcile Now' quick action button.

Given the dashboard includes a month-over-month trend line,
When rendered,
Then it plots EOB__c payment accuracy rate for the trailing 6 months using EOB__c.Finalized_Date__c as the date dimension.

Given the user has the 'Claims_Ops_Manager' Permission Set Group,
When they access the dashboard,
Then all tiles load within 5 seconds and no data from other tenant orgs is visible (org-level data isolation confirmed).

## Solution

Built using CRM Analytics (Tableau CRM) with a scheduled DataFlow combining EOB__c, Payment__c, Claim__c, and Claim_Line__c datasets. Payment Accuracy KPI uses a SAQL computed measure comparing Paid_Amount__c to Expected_Allowed_Amount__c. ERA Reconciliation rate uses a filtered count measure on Payment__c.ERA_Reconciled__c. COB segmentation uses a groupBy on COB_Type__c. Unreconciled payments list widget uses a Values Table with conditional formatting (row color = red when ERA_Reconciled__c = FALSE and Payment_Date__c < TODAY()-3). The 'Reconcile Now' button is a Lightning Quick Action on Payment__c that triggers an Apex-invocable method to initiate ERA re-match. Month-over-month trend uses a Time Series widget with a 6-month lookback SAQL window. Dashboard secured via Permission Set Group 'Claims_Ops_Manager'.

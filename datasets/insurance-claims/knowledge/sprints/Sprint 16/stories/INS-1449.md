## Description

As a Broker Relations Coordinator, I need a broker performance scorecard that surfaces submission volume, policy bind rate, and loss ratio per broker so that I can identify top-performing broker partners, flag underperformers for remediation conversations, and support commission tier reviews with Underwriting Analysts.

## Acceptance Criteria

Given the Broker Relations Coordinator opens the 'Broker Performance Scorecard' dashboard,
When the current quarter filter is active,
Then the scorecard grid must display one row per active broker (Contact.RecordType = 'Broker') with columns: Broker Name, NPN, Submissions (count of Policy__c records where Policy__c.Broker__c = broker AND Policy__c.CreatedDate = THIS_QUARTER), Bound Policies (Policy__c.Status__c = 'Active'), Bind Rate % (Bound / Submissions), Total Premium (SUM Policy__c.Annual_Premium__c), Total Claims (count of Claim__c linked to broker policies), Loss Ratio % (Total Incurred Claims / Total Premium).

Given the grid is rendered,
When a broker's Loss Ratio % exceeds 85%,
Then that row must be highlighted red and a 'Remediation Flag' icon must appear, with a hover tooltip showing the top 3 CPT codes by claim cost for that broker's book.

Given the coordinator clicks a broker row,
When the detail panel slides in,
Then it must show the broker's Group_Service_Agreement__c records, commission tier (Contact.Commission_Tier__c), last review date (Contact.Last_Review_Date__c), and a sparkline of monthly submission volume for the trailing 12 months.

-Given the scorecard includes a 'Projected Next Quarter Volume' AI forecast column powered by Einstein Prediction Builder (deprecated — Einstein Prediction Builder license not provisioned for this org; deferred to Phase 3)-

Given the coordinator exports the scorecard,
When the export is triggered,
Then a CSV must be generated with all visible columns plus Contact.Email and Contact.Phone for offline broker outreach use.

## Solution

Build the scorecard as a CRM Analytics dashboard with a Recipe joining Policy__c, Claim__c, and Contact (Broker RecordType) datasets. Loss Ratio is a computed metric in the Recipe using SUM(Claim__c.Total_Incurred_Amount__c) / SUM(Policy__c.Annual_Premium__c). The detail panel is an LWC ('brokerScorecardDetail') launched via a CRMA dashboard action URL that passes the broker Contact Id as a URL parameter; the LWC then queries GSA and sparkline data via Apex. Row-level color coding is implemented via a CRMA conditional formatting rule on the Loss_Ratio computed column. CSV export uses a custom Apex REST endpoint (BrokerScorecardExportController.cls) invoked from a dashboard button. Permission Set 'Broker_Scorecard_Viewer' restricts access to Broker Relations Coordinator and Underwriting Analyst profiles.

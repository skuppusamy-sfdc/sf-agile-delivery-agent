## Description

As a Claims Operations Manager, I need an executive-level CRM Analytics dashboard surfacing real-time KPIs for claim volume, average cycle time, denial rates, and total disbursements so that I can monitor operational health and identify bottlenecks without exporting data to external BI tools.

This dashboard should aggregate data across Claim__c, Claim_Line__c, Payment__c, and EOB records, segmented by LOB (Medical, Dental, Vision) and TPA channel.

## Acceptance Criteria

Given the Claims Operations Manager navigates to the 'Claims Ops Executive Dashboard' CRM Analytics app,
When the dashboard loads,
Then the following KPI tiles must render within 3 seconds using live SAQL queries:
  - Total Open Claims (Claim__c.Status__c IN ('Submitted','In Review','Pending Info'))
  - Average Claim Cycle Time in days (Claim__c.Closed_Date__c - Claim__c.FNOL_Date__c)
  - First-Pass Adjudication Rate (Claim__c records closed without reopening / total closed)
  - Total Payments Disbursed MTD (SUM of Payment__c.Amount__c WHERE Payment__c.Payment_Date__c = THIS_MONTH)
  - Denial Rate % by LOB (Claim__c.Denial_Reason__c IS NOT NULL / total Claim__c)

Given the dashboard is loaded,
When the Claims Operations Manager clicks a KPI tile,
Then a drill-down panel opens listing individual Claim__c records filtered by the tile's criteria, displaying fields: Claim__c.Name, Claim__c.Policy__c (lookup name), Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, Claim__c.Assigned_Adjudicator__c.

Given a LOB filter picklist exists on the dashboard,
When the user selects 'Medical' from the LOB filter,
Then all KPI tiles and charts refresh to show data where Claim__c.Line_of_Business__c = 'Medical'.

Given the dashboard is accessed by a user without the 'Claims_Executive_Viewer' Permission Set Group,
When they attempt to navigate to the app,
Then they receive an 'Insufficient Privileges' message and are redirected to the default home page.

-Given a date range selector allows custom fiscal quarter input, Then KPIs recalculate accordingly- (deferred to INS-1471)

Given the dashboard is viewed on a tablet device (min 1024px width),
When the layout renders,
Then all KPI tiles stack in a 2-column responsive grid without horizontal scrolling.

## Solution

Build a CRM Analytics (Tableau CRM) dashboard app 'Claims Ops Executive Dashboard' with SAQL-based datasets sourced from Claim__c, Payment__c, Claim_Line__c, and EOB__c via scheduled dataflow running every 4 hours. KPI tiles use Compact Chart widget type. Drill-down panels leverage Connected Dataset with a global filter binding. LOB filter widget uses a picklist dataset column. Row-level security enforced via a Security Predicate on the Claim__c dataset tied to User.Profile. Permission Set Group 'Claims_Executive_Viewer' gates app visibility. Responsive layout configured via the Analytics Studio flex-grid template.

## Description

As a Provider Relations Specialist, I need a Network Participation Summary panel embedded within the Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can instantly view a provider's active PNA agreements, associated Fee Schedules, effective dates, and network tier without navigating away from the provider record. This improves call handling efficiency and reduces average handle time during provider inquiries.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the 'Network Participation Summary' panel must be visible and display a list of all related Provider_Network_Agreement__c records where PNA_Status__c IN ('Active', 'Pending Renewal')

Given the Network Participation Summary panel is rendered,
When PNA records are listed,
Then each row must display: Provider_Network_Agreement__c.Name, PNA_Effective_Date__c, PNA_Expiration_Date__c, Network_Tier__c, and the related Fee_Schedule__c.Fee_Schedule_Name__c linked via Provider_Network_Agreement__c.Fee_Schedule__c lookup

Given a PNA record in the panel has PNA_Expiration_Date__c within 30 days of today,
When the panel renders,
Then that row must display an amber warning badge with text 'Expiring Soon'

Given a Provider__c record has zero active PNA records,
When the panel loads,
Then an empty state message 'No active network agreements found' must be displayed with a 'Initiate PNA' quick action button

Given the Provider Relations Specialist clicks a PNA Name hyperlink in the panel,
When clicked,
Then the user must be navigated to the Provider_Network_Agreement__c record detail page in a new browser tab

-Given the panel includes a 'GSA Linked Agreements' sub-section sourced from Group_Service_Agreement__c- (deprecated: GSA panel scoped to separate FlexCard component in INS-1403)

Given the FlexCard is embedded on the Provider__c record page,
When a user without the 'Provider_Network_Viewer' Permission Set attempts to view the page,
Then the Network Participation Summary panel must be hidden and replaced with 'Access Restricted' placeholder text

Given the panel displays more than 5 PNA records,
When rendered,
Then a 'Show All (N)' pagination control must appear, with the panel defaulting to the 5 most recently effective records sorted by PNA_Effective_Date__c DESC

## Solution

Build an OmniStudio FlexCard 'ProviderNetworkSummaryCard' as a child card embedded in the existing Provider 360 parent FlexCard. DataRaptor Extract action queries Provider_Network_Agreement__c filtered by Provider__c Id (via Account lookup) and PNA_Status__c IN ('Active','Pending Renewal'), joining Fee_Schedule__c via relationship. Expiry warning badge implemented via FlexCard conditional state with formula: PNA_Expiration_Date__c - TODAY() <= 30. Empty state handled via FlexCard State configuration. Pagination implemented using FlexCard built-in repeat element with pageSize=5. Permission visibility enforced via FlexCard Conditional Visibility expression checking running user's Permission Set using a DataRaptor that queries PermissionSetAssignment. 'Initiate PNA' quick action invokes an OmniScript (NewPNAOmniScript) via NavigationMixin.

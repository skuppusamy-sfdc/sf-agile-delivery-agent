## Description

**Bug Report — EOB PDF Generation Silent Failure on High Line-Count Claims**

**Steps to Reproduce:**
1. Navigate to any Claim__c record where the count of related Claim_Line__c records exceeds 50 (e.g., Claim #CLM-2024-008847 in the Staging org).
2. Set Claim__c.Status__c = 'Adjudicated'.
3. Click the 'Generate EOB' quick action button on the Claim record page.
4. Observe the Conga Composer merge job status.

**Expected Behavior:** A fully populated Explanation_of_Benefits__c PDF is generated, attached to the EOB__c record as a ContentDocument, and the EOB__c.Generation_Status__c field is set to 'Completed'.

**Actual Behavior:** The Conga Composer job initiates but returns no error to the UI. The EOB__c record is created with Generation_Status__c = 'Pending' and never transitions. No ContentDocument is attached. Checking debug logs reveals a 'Maximum View State Exceeded' error in the Conga callback Apex class (EOBGenerationCallbackController.cls) at line 214, caused by attempting to bind all 50+ Claim_Line__c records into the Visualforce merge template simultaneously.

**Environment:** Staging (API v59.0). Reproducible 100% with > 50 Claim_Line__c records. Claims with ≤ 49 lines generate successfully.

**Impact:** High — approximately 12% of inbound institutional claims (UB-04 claim type) regularly carry 50–200 claim lines. Affected EOBs are not delivered to policyholders or submitted as ERA to payers, causing payment delay and compliance risk.

## Acceptance Criteria

As per Description. Additional validation: EOB__c.Generation_Status__c must transition to 'Completed' (not remain 'Pending') for Claim__c records with 51, 100, and 200 Claim_Line__c children in regression testing. No 'Maximum View State Exceeded' errors in debug logs post-fix.

## Solution

Refactor EOBGenerationCallbackController.cls to paginate Claim_Line__c retrieval using SOQL offset pagination (batches of 25 lines) rather than binding the full collection to the Visualforce page state. Alternatively, migrate EOB generation off Visualforce-backed Conga template to a server-side Conga API (REST) invocation via an Apex Future method (EOBGenerationService.generateAsync) that passes a JSON payload of Claim_Line__c data, bypassing View State constraints entirely. Update the Record-Triggered Flow on Claim__c (Status__c = 'Adjudicated') to invoke the Apex Future method instead of the existing quick action. Add a Platform Event (EOB_Generation_Complete__e) published on callback to update EOB__c.Generation_Status__c and trigger ContentDocument attachment. Include unit tests covering 1, 50, 51, and 200 line scenarios with @isTest(SeeAllData=false) mock data.

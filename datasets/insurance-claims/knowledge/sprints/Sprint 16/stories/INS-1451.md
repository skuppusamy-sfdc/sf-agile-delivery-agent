## Description

**Bug Report – INS-1451**

**Summary:** The 'Claims by Line of Business' CRM Analytics widget on the Claims Operations Dashboard (INS-1444) is double-counting MTD claim totals for Coordination of Benefits (COB) claims. COB claims have two Claim__c records — a primary and a secondary — both stamped with the same Claim__c.Line_of_Business__c value. The Recipe does not deduplicate on Claim__c.COB_Primary_Claim_Id__c, causing secondary COB claims to inflate the LOB totals.

**Steps to Reproduce:**
1. Open the Claims Operations Dashboard in the Claims Ops Lightning App.
2. Note the 'Commercial Health' LOB tile showing MTD claim count (e.g., 1,240).
3. Run SOQL: SELECT COUNT(Id), Line_of_Business__c FROM Claim__c WHERE COB_Flag__c = TRUE AND CreatedDate = THIS_MONTH GROUP BY Line_of_Business__c.
4. Note the COB claim count (e.g., 86 secondary COB claims).
5. Expected MTD count: 1,240 - 86 = 1,154. Actual dashboard shows 1,240 (secondary COB claims not excluded).

**Expected Behavior:** The 'Claims by LOB' widget must count only primary claims (Claim__c.COB_Flag__c = FALSE OR Claim__c.COB_Role__c = 'Primary') to avoid double-counting COB encounters. Secondary COB claims (Claim__c.COB_Role__c = 'Secondary') should be reported in a separate 'COB Secondary Claims' breakdown widget.

**Actual Behavior:** Both primary and secondary COB Claim__c records are included in the LOB count aggregation, inflating MTD totals by the number of secondary COB claims.

**Environment:** Production CRM Analytics dataset (last refreshed 2024-11-14 02:00 UTC)
**Severity:** Medium — affects reporting accuracy but does not impact payment processing
**Affected Artifact:** CRMA Recipe 'Claims_MTD_by_LOB_Recipe', Filter node step 3

## Acceptance Criteria

As per Description.

Given the Recipe fix is deployed and the dataset is refreshed,
When the 'Claims by LOB' widget renders for Commercial Health MTD,
Then the count must equal only Claim__c records where COB_Role__c = 'Primary' OR COB_Flag__c = FALSE, matching a manual SOQL validation query within a ±1 record tolerance (accounting for in-flight records during refresh).

Given the fix is applied,
When a separate 'COB Secondary Claims' widget is added to the dashboard,
Then it must display a count of Claim__c records where COB_Role__c = 'Secondary' for the same MTD window, clearly labeled to avoid confusion with primary LOB totals.

## Solution

Edit the CRMA Recipe 'Claims_MTD_by_LOB_Recipe': in the Filter node (step 3), add an additional filter condition: COB_Role__c IN ('Primary', '') OR COB_Flag__c = FALSE to exclude secondary COB records from the main LOB aggregation. Add a parallel Recipe branch that filters for COB_Role__c = 'Secondary' to power the new 'COB Secondary Claims' widget. Re-run the dataset refresh after Recipe deployment and validate output against a manual SOQL count. Update the CRMA dashboard JSON definition to add the new COB Secondary widget in the bottom row of the Claims Ops Dashboard. Document the COB deduplication logic in the internal Confluence data dictionary for the Claims_MTD dataset.

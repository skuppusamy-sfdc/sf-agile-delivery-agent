## Description

As a Broker Relations Coordinator, I need a Salesforce dashboard scorecard that tracks individual broker and broker agency performance metrics — including policy submission volume, quote-to-bind conversion rates, and average premium by LOB — so that I can identify top-performing brokers for incentive programs and flag underperforming relationships for remediation outreach.

The scorecard must pull data from Policy__c and Group_Service_Agreement__c (GSA) objects and be accessible to brokers themselves via the Agent Portal (Experience Cloud site) in a read-only, self-service view.

## Acceptance Criteria

Given a Broker Relations Coordinator with the 'Broker_Analytics' Permission Set,
When they open the 'Broker Performance Scorecard' dashboard,
Then they see a table listing each Broker__c (Contact or Person Account) with columns: total Policy__c submissions (Policy__c.Broker__c = broker), quote-to-bind conversion rate, average Policy__c.Annual_Premium__c, and active GSA count.

Given the scorecard table is rendered,
When a broker's quote-to-bind conversion rate falls below 20%,
Then their row is highlighted in amber and a 'Remediation Flag' icon appears.

Given a broker logs into the Agent Portal (Experience Cloud),
When they navigate to 'My Performance',
Then they see their own scorecard data only — filtered to Policy__c.Broker__c = running user's linked Broker__c record — with no access to other brokers' data.

Given Group_Service_Agreement__c records linked to a broker,
When the 'Active GSA' column renders,
Then it counts only GSA records where Group_Service_Agreement__c.Status__c = 'Active' AND Group_Service_Agreement__c.Broker__c = broker.

Given the Broker Relations Coordinator clicks 'Download Scorecard',
When the export is triggered,
Then a CSV file is generated containing all visible scorecard rows with the columns as displayed, timestamped with the export date.

Given the dashboard filter 'LOB' is applied,
When a specific Claim__c.Line_of_Business__c value is selected,
Then all metrics recalculate to reflect only Policy__c records with Policy__c.Line_of_Business__c matching the selected value.

## Solution

Build a CRM Analytics dashboard ('Broker_Performance_Scorecard') on a dataset joining Policy__c, Group_Service_Agreement__c, and Broker__c (Contact). Conversion rate calculated as SAQL ratio of Policy__c.Stage__c = 'Bound' to total submissions per broker. The Agent Portal view uses a separate Lightning Web Component ('brokerSelfScorecard') embedded in the Experience Cloud page, which calls an Apex controller method executing SOQL WITH USER_MODE to enforce data isolation. Experience Cloud Sharing Set on Policy__c restricts broker users to records where Policy__c.Broker__c = Contact.Id. CSV export uses a LWC-based client-side CSV generation utility. Permission Set 'Broker_Analytics' grants internal user access to the CRM Analytics app.

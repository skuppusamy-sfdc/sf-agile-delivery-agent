## Description

As an Underwriting Analyst, I need a Salesforce summary report that aggregates total Payment__c disbursements grouped by TPA__c (lookup to Account), LOB__c, and Payment__c.Payment_Method__c (EFT, Check, Virtual Card) for any user-selected date range — so that the Finance team can perform monthly reconciliation against ERA files and identify discrepancies between Salesforce payment records and bank settlement data.

## Acceptance Criteria

Given the Underwriting Analyst runs the 'Payment Disbursement by TPA & LOB' summary report,
When a custom date range is applied to Payment__c.Disbursement_Date__c,
Then the report must display grouped rows by TPA__c.Name > LOB__c > Payment_Method__c with subtotals of Payment__c.Amount__c and record count per group, and a grand total row at the bottom.

Given the report is exported,
When the analyst selects 'Export — Formatted .xlsx',
Then the exported file must preserve grouping hierarchy, subtotals, and include a header row with column names matching the Salesforce field labels.

Given a Payment__c record where Payment__c.Status__c = 'Voided',
When the report runs,
Then voided payments must be excluded from all subtotal and grand total calculations but must appear as separate line items in a 'Voided Payments' summary section at the bottom of the report.

Given the report is added to the 'Finance Reconciliation' dashboard,
When the dashboard refreshes,
Then the report widget must reflect data as of the last full hour (not real-time) to comply with Finance's data freeze policy during month-end close (Payment__c.Disbursement_Date__c <= LAST_MONTH for month-end runs).

## Solution

Create a Salesforce Summary Report using a custom Report Type 'Payments with TPA and Claim' (Payment__c → Claim__c → TPA__c). Configure grouping levels: Group Row 1 = TPA__c.Name, Group Row 2 = LOB__c, Group Row 3 = Payment_Method__c. Add summary formula for SUM(Payment__c.Amount__c) and COUNT(Id). Apply a cross-filter to exclude Voided payments from main groupings; add a separate summary section via a second report block (joined report) for voided records. Add report to 'Finance Reconciliation' Lightning Dashboard with a 1-hour refresh schedule. Export formatting handled natively by Salesforce report export. Row-level sharing enforced via Sharing Rule: Finance_Role__c users can view all Payment__c records regardless of OwnerId.

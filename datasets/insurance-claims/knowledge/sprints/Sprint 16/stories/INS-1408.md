## Description

As a Claims Operations Manager, I need a real-time Salesforce CRM Analytics dashboard displaying key claims processing KPIs — including average adjudication cycle time, total paid amount by LOB, denial rate by CPT code range, and pending FNOL volume — so that I can proactively identify bottlenecks and optimize team throughput during daily stand-ups and executive reporting cycles.

## Acceptance Criteria

Given the Claims Operations Manager navigates to the 'Claims Ops Command Center' CRM Analytics dashboard,
When the dashboard loads,
Then KPI tiles must display: (1) Claim__c records where Status__c = 'Pending Adjudication' grouped by LOB__c, (2) average Claim__c.Adjudication_Cycle_Days__c for the rolling 30-day window, (3) total Payment__c.Amount__c where Payment__c.Status__c = 'Disbursed' for current fiscal month, and (4) count of Claim__c records where FNOL_Received_Date__c is within the last 7 days.

Given the dashboard is loaded,
When the Claims Ops Manager selects a LOB__c filter value (e.g., 'Workers Comp', 'Group Health', 'Auto'),
Then all tiles and charts must dynamically filter to reflect only Claim__c records matching the selected LOB__c value without page reload.

Given the trend chart widget is rendered,
When the manager views the 'Denial Rate by CPT Range' bar chart,
Then Claim_Line__c.Denial_Reason_Code__c must be grouped by CPT__c range buckets (00100-01999, 10004-69990, 70010-79999, 80047-89398, 90281-99607) and displayed as percentage of total Claim_Line__c submissions per bucket.

Given the dashboard is accessed by a user without the 'Claims_Ops_Dashboard_Access' Permission Set,
When they attempt to navigate to the dashboard URL,
Then they must receive a 'Insufficient Privileges' error and be redirected to the standard home page.

-Given the dashboard includes a real-time SIU referral counter widget sourced from a separate SIU Salesforce org via cross-org connect- (deprecated — SIU integration deferred to Sprint 18, see INS-1441)

## Solution

Create a CRM Analytics (Tableau CRM) dashboard 'Claims Ops Command Center' using SAQL queries against the Claim__c, Claim_Line__c, and Payment__c datasets synced via standard Data Sync connectors. Build five XMD-configured KPI widgets and two bar/line combo charts. Implement a global LOB__c dashboard filter linked to all widgets via filter faceting. Row-level security enforced via a CRM Analytics Security Predicate referencing the User.Profile and a custom Permission Set Group 'Claims_Ops_Dashboard_PSG'. Deploy via unlocked package. Trend data for cycle time sourced from a scheduled Apex batch (ClaimsKPIAggregatorBatch.cls) that writes daily snapshots to Claims_KPI_Snapshot__c custom object to support historical trending beyond the 18-month CRM Analytics data retention window.

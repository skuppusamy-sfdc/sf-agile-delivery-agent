## Description

As a Provider Network Specialist, I need a scheduled Salesforce report and dashboard component that surfaces all Provider_Network_Agreement__c (PNA) and Group_Service_Agreement__c (GSA) records expiring within 30, 60, and 90 days — segmented by Provider__c specialty, geographic region, and LOB — so that I can prioritize outreach and renewal workflows before network adequacy gaps occur. Extends INS-1378 (Provider Network Adequacy baseline report).

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Expiration_Date__c falls within today + 90 days AND PNA.Status__c != 'Terminated',
When the 'PNA/GSA Expiry Watch' report runs (scheduled daily at 06:00 CST),
Then the record must appear in the appropriate expiry bucket column: '0-30 Days', '31-60 Days', or '61-90 Days', with columns displaying PNA.Contract_Number__c, related Provider__c.Name, Provider__c.Primary_Specialty__c, Provider__c.NPI__c, PNA.LOB__c, PNA.Expiration_Date__c, and PNA.Assigned_Network_Specialist__c.

Given the report is rendered on the 'Provider Network Contracting' dashboard,
When a Network Specialist clicks a row in the expiry report widget,
Then the user must be navigated directly to the Provider_Network_Agreement__c record detail page in a new browser tab.

Given a GSA record where GSA.End_Date__c falls within the 90-day window AND GSA.Auto_Renewal__c = false,
When the report executes,
Then the GSA record must appear in the expiry list with GSA.Group_Name__c, GSA.TIN__c, GSA.Covered_Lives__c, and GSA.Renewal_Owner__c displayed.

Given the report is run by a user assigned the 'Provider_Network_ReadOnly' Permission Set,
When the report renders,
Then Provider__c.TIN__c and Provider__c.Bank_Account__c fields must be masked (showing only last 4 characters) per field-level security configuration.

Given no PNA or GSA records fall within the 90-day expiry window,
When the report runs,
Then the report must display a 'No contracts expiring in the next 90 days' informational message rather than an empty table.

## Solution

Build a Salesforce Report using a custom Report Type joining Provider_Network_Agreement__c with Provider__c (Person Account) and Group_Service_Agreement__c. Add a formula column for 'Days_Until_Expiry' = PNA.Expiration_Date__c - TODAY(). Use bucketing on the formula column for 30/60/90-day ranges. Schedule the report via Report Subscription (daily, 06:00 CST) with email delivery to the Provider_Network_Specialist public group. Dashboard component uses a summary table widget. FLS for TIN__c and Bank_Account__c masked via Permission Set overrides. A Record-Triggered Flow on Provider_Network_Agreement__c fires when Expiration_Date__c changes and updates a PNA_Expiry_Bucket__c formula field used as a list view filter for quick access outside the dashboard.

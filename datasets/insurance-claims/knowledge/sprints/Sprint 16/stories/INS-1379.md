## Description

**Bug Report — INS-1379**

**Environment:** Production Sandbox (Full Copy) — Sprint 16 UAT

**Summary:** The 'Claims Disbursement Summary' report is double-counting total Payment__c amounts for claims where Claim__c.COB_Flag__c = TRUE. Each Payment__c record tied to a COB claim appears twice in the report aggregate, inflating disbursement totals by roughly 2x for those claim lines.

**Steps to Reproduce:**
1. Navigate to Reports > Claims Disbursement Summary.
2. Apply filter: Report Date Range = current month.
3. Observe the 'Total Disbursed' summary row.
4. Identify a COB claim (Claim__c.COB_Flag__c = TRUE) and manually sum its Payment__c.Amount__c values.
5. Compare manual sum to report aggregate — report shows double the expected value.

**Expected Behavior:** Each Payment__c record is counted exactly once; Total Disbursed reflects the true sum of Payment__c.Amount__c across all unique Payment__c Ids.

**Actual Behavior:** Payment__c records where the parent Claim__c.COB_Flag__c = TRUE are joined twice due to a cross-object formula field (Payment__c.Primary_Payer_Amount__c) creating a duplicate row in the underlying joined report definition.

**Impact:** Claims Operations Manager is reporting inflated disbursement figures to finance. Potential compliance risk if figures are submitted to CMS.

**Workaround:** Manually filter report by Claim__c.COB_Flag__c = FALSE for accurate non-COB totals.

## Solution

Root cause: the 'Claims Disbursement Summary' joined report has two report blocks both sourcing Payment__c — Block A joins via Claim__c and Block B joins via Explanation_of_Benefits__c (EOB__c), causing COB claims to match both join paths. Fix: remove Block B from the joined report and replace with a Summary Formula column that calculates Primary_Payer_Amount__c inline using PARENTGROUPVAL. Additionally, add a report filter on Payment__c.Is_Duplicate_COB__c (new checkbox formula field: IF(Claim__c.COB_Flag__c = TRUE AND Payment__c.Payment_Sequence__c > 1, TRUE, FALSE)) to exclude secondary sequence records. Validate fix by comparing report total against aggregate SOQL: SELECT SUM(Amount__c) FROM Payment__c WHERE Claim__r.COB_Flag__c = TRUE in Developer Console.

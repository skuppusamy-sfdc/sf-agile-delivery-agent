## Description

As a Credentialing Specialist, I need a FlexCard embedded on the Provider 360 page that displays the current CAQH verification status, last attestation date, and the age (in days) of any open Credentialing_Case__c records, so that I can quickly assess a provider's credentialing posture without navigating to multiple related lists or external CAQH ProView screens.

## Acceptance Criteria

Given a Credentialing Specialist opens a Provider__c (Person Account) record page,
When the 'Credentialing Pipeline' FlexCard renders,
Then it displays the following fields sourced from Credentialing_Case__c (most recent open record):
  - Credentialing_Case__c.Status__c
  - Credentialing_Case__c.Case_Age_Days__c (formula: TODAY() - Credentialing_Case__c.CreatedDate)
  - Credentialing_Case__c.CAQH_Profile_ID__c
  - Credentialing_Case__c.Last_CAQH_Attestation_Date__c
  - Credentialing_Case__c.Assigned_Credentialing_Specialist__c

Given the CAQH integration DataRaptor has retrieved attestation data,
When the attestation date is more than 365 days ago,
Then the FlexCard displays a red warning badge labeled 'CAQH Attestation Overdue'.

Given the Provider__c record has no open Credentialing_Case__c,
When the FlexCard renders,
Then it displays an empty state message: 'No active credentialing case. Click to initiate.' with a button that launches the 'Initiate Credentialing' OmniScript.

-Given the FlexCard also shows NPI validation status from NPPES, Then a green checkmark appears if NPI is active- (removed — NPPES widget moved to separate INS-1448 FlexCard)

Given the FlexCard action button 'View Full Case' is clicked,
When the navigation fires,
Then the user is redirected to the related Credentialing_Case__c record detail page.

## Solution

Build OmniStudio FlexCard 'CredentialingPipelineCard' using a DataRaptor Extract as the data source querying Credentialing_Case__c WHERE Provider__c = :recordId AND Status__c != 'Closed' ORDER BY CreatedDate DESC LIMIT 1. Conditional state logic drives badge color using FlexCard State configuration (red state when Last_CAQH_Attestation_Date__c < LAST_N_DAYS:365). Empty state configured as a separate FlexCard State with OmniScript navigation action pointing to 'InitiateCredentialing' OmniScript. FlexCard embedded on Provider__c Lightning Record Page via App Builder. CAQH data refresh triggered by a Platform Event (CAQH_Sync_Event__e) published by the nightly DataRaptor Turbo Extract integration job.

## Description

**Bug Report: Fee Schedule Dashboard Double-Counting Allowed Amounts**

**Steps to Reproduce:**
1. Ensure at least two Fee_Schedule__c records exist for the same Provider__c and CPT code, one with Fee_Schedule__c.Effective_Date__c = 2024-01-01 (Status: 'Superseded') and one with Fee_Schedule__c.Effective_Date__c = 2024-07-01 (Status: 'Active').
2. Navigate to Reports & Dashboards > FeeScheduleEngine > 'Allowed Amount Summary by Provider' dashboard.
3. Observe the 'Total Allowed Amount' figure for the affected provider.
4. Compare to the sum of Claim_Line__c.Allowed_Amount__c for that provider in a separate list view.

**Expected Behavior:**
The dashboard should sum Claim_Line__c.Allowed_Amount__c only for Claim_Line__c records linked to Fee_Schedule__c records where Fee_Schedule__c.Status__c = 'Active'.

**Actual Behavior:**
The dashboard aggregates Claim_Line__c.Allowed_Amount__c across ALL Fee_Schedule__c versions (both 'Active' and 'Superseded'), resulting in inflated totals — in one case 187% of the correct value. This is causing incorrect payment reconciliation reports to be surfaced to the Underwriting Analyst team.

**Environment:** PROD (confirmed), UAT (reproduced)
**Severity:** High — financial reporting integrity issue affecting Payment__c reconciliation.
**Workaround:** None currently available. Underwriting Analysts have been advised not to use this dashboard pending fix.

## Acceptance Criteria

As per Description.

Given the fix is deployed,
When the 'Allowed Amount Summary by Provider' dashboard loads,
Then total allowed amounts are calculated using only Claim_Line__c records joined to Fee_Schedule__c where Fee_Schedule__c.Status__c = 'Active', verified by reconciling dashboard totals against a manual SOQL query: SELECT SUM(Allowed_Amount__c) FROM Claim_Line__c WHERE Fee_Schedule__r.Status__c = 'Active'.

Given a Fee_Schedule__c record is updated from Status__c = 'Active' to 'Superseded',
When the dashboard next refreshes,
Then the affected provider's allowed amount total decreases to exclude the superseded schedule's contributions.

## Solution

Root cause: the CRM Analytics dataset recipe joining Claim_Line__c to Fee_Schedule__c uses a LEFT JOIN without a filter on Fee_Schedule__c.Status__c, causing all fee schedule versions to be included in aggregations. Fix: add an input node filter in the dataset recipe to restrict Fee_Schedule__c records to Status__c = 'Active' before the join step. Additionally, add a SAQL-level WHERE clause in the affected dashboard widgets as a secondary safeguard. Update the DataSync schedule to rebuild the dataset after the recipe change. Regression test by running the corrected SOQL query against PROD data and comparing to dashboard output. Deploy recipe change via CRM Analytics changeset. Notify Underwriting Analyst team lead (Lisa Park) when fix is promoted to PROD and dashboard totals are verified.

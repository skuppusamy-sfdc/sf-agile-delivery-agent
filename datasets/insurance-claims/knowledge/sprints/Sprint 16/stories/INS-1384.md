## Description

As a Claims Operations Manager, I need a real-time analytics dashboard displaying adjudication throughput, denial rates, average days-to-pay, and pending claim volumes by LOB so that I can proactively manage team workload and identify processing bottlenecks before they impact SLA compliance.

## Acceptance Criteria

Given the Claims Operations Manager navigates to the 'Claims Ops Dashboard' Lightning App Page,
When the page loads,
Then KPI tiles display the following aggregated metrics refreshed within the last 15 minutes:
  - Total open Claim__c records where Claim__c.Status__c IN ('Submitted','In Review','Pending Info')
  - Adjudication throughput count: Claim__c records where Claim__c.Status__c = 'Adjudicated' AND Claim__c.Adjudication_Date__c = TODAY
  - Denial rate %: (COUNT of Claim__c.Status__c = 'Denied' / COUNT of all closed claims) * 100 for rolling 30-day window
  - Average days-to-pay: AVG(Payment__c.Payment_Date__c - Claim__c.Date_of_Service__c) grouped by Claim__c.LOB__c

Given the dashboard is rendered,
When the Claims Operations Manager applies the LOB filter (values: Medical, Dental, Vision, Workers Comp),
Then all KPI tiles and embedded report charts filter accordingly without full page reload.

Given a Claim__c record transitions to Status__c = 'Denied',
When the denial is saved,
Then the denial rate KPI tile reflects the updated value within the next scheduled 15-minute refresh cycle.

Given the dashboard is accessed by a user without the 'Claims_Ops_Manager' Permission Set Group,
When they attempt to navigate to the Lightning App Page,
Then they are redirected to an access-denied screen and the underlying CRT_Claims_Ops_Dashboard report folder is not visible.

-Given a secondary tile for SIU referral rate was included in v1 scope-
-Then SIU referral tile displays COUNT of Claim__c records where Claim__c.SIU_Referred__c = true-
(Descoped to INS-1401 per Sprint 15 grooming)

## Solution

Build a Lightning App Page using App Builder with a combination of standard Report Chart components and custom LWC KPI tile components (lwc/claimsKpiTile). KPI tiles call @wire-connected Apex controllers (ClaimsOpsMetricsController.cls) that execute SOQL aggregate queries against Claim__c and Payment__c with FLS-safe field access via Security.stripInaccessible(). Schedule a Platform Cache refresh via a Scheduled Apex job (ClaimsMetricsCacheRefresher.cls) running every 15 minutes to populate an Org Cache partition ('claimsOpsMetrics'). LOB filter implemented as a Lightning message channel (ClaimsDashboardFilter__c LMC) broadcasting filter state to all child components. Access controlled via Permission Set Group 'Claims_Ops_Manager_PSG' assigned to the Lightning App visibility rules.

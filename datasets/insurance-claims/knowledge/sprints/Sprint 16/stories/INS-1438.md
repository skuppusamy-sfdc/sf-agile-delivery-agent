## Description

As a Provider Network Specialist, I need a FlexCard-based analytics panel embedded on the Provider Network summary page that shows contracted vs. actively credentialed provider counts segmented by specialty and geographic region so that I can identify network gaps and prioritize recruitment outreach without running manual SOQL reports.

This story extends INS-1418 (Provider 360 FlexCard foundation) by adding the analytics panel as a child card within the existing OmniStudio FlexCard layout.

Data sources: Provider__c (Person Account), Provider_Network_Agreement__c (PNA), Credentialing_Case__c.

## Acceptance Criteria

Given a Provider Network Specialist opens the Provider Network Summary Lightning page,
When the 'Network Coverage Analytics' FlexCard loads,
Then it displays a donut chart of Provider__c records segmented by Provider__c.Specialty__c where PNA__c.Status__c = 'Active' and PNA__c.Effective_Date__c <= TODAY() and PNA__c.Termination_Date__c >= TODAY() OR PNA__c.Termination_Date__c = NULL.

Given provider geographic data is populated,
When the region filter is set to a specific state (Provider__c.BillingState),
Then the FlexCard refreshes to show only providers in that state, with a count badge showing 'X Contracted / Y Credentialed'.

Given a specialty row is selected in the breakdown table,
When the user clicks 'View Providers',
Then a child FlexCard panel expands inline listing Provider__c.Name, Provider__c.NPI__c, PNA__c.Contract_Effective_Date__c, and Credentialing_Case__c.Status__c for matching records.

Given a provider has an active Credentialing_Case__c.Status__c = 'Pending CAQH Verification',
When they appear in the panel,
Then their row is highlighted in amber with a tooltip 'Credentialing In Progress'.

-Given network gap thresholds are configurable via Custom Metadata- (superseded — thresholds now managed via Network_Gap_Config__mdt per INS-1431)

Given fewer than 3 active providers exist for a given specialty in a region,
When the FlexCard renders,
Then a 'Network Gap' warning badge (red) is displayed next to that specialty/region combination.

## Solution

Extend the existing OmniStudio FlexCard 'ProviderNetwork360Card' (INS-1418) by adding a child FlexCard 'NetworkAnalyticsPanel'. Use a DataRaptor Extract to query Provider__c joined to PNA__c and Credentialing_Case__c via relationship fields. Aggregate counts by Specialty__c and BillingState using a DataRaptor Transform with groupBy operations. Render the donut chart using the FlexCard Chart block (type: donut). Network gap logic implemented as a DataRaptor Transform conditional that checks count < 3 and sets a 'gapFlag' field consumed by the FlexCard's conditional styling block. Network_Gap_Config__mdt custom metadata type provides specialty-level threshold overrides. FlexCard deployed via OmniStudio deployment tool; Permission Set 'Provider_Network_Analyst_PS' grants DataRaptor access.

## Description

**Bug Report — EOB Disbursement Reconciliation Report Double-Counting COB Payments**

**Steps to Reproduce:**
1. Navigate to the 'EOB Disbursement Reconciliation' Salesforce report in the 'Payment Analytics' folder.
2. Set the report date filter to any period containing Claim__c records where Claim__c.COB_Indicator__c = true and at least one associated Payment__c record has Payment__c.Payer_Type__c = 'Secondary'.
3. Run the report.
4. Observe the 'Total Disbursed' summary column value.
5. Manually sum Payment__c.Amount__c from SOQL query: SELECT SUM(Amount__c) FROM Payment__c WHERE Status__c = 'Paid' AND Claim__r.COB_Indicator__c = true for the same period.

**Expected Result:** The report 'Total Disbursed' summary must equal the SOQL aggregate sum, with secondary COB Payment__c records counted once.

**Actual Result:** The report double-counts Payment__c.Amount__c for claims with COB secondary payer records, inflating the 'Total Disbursed' figure by the total of all Payment__c records where Payment__c.Payer_Type__c = 'Secondary'. Root cause suspected in the report's cross-filter joining Explanation_of_Benefits__c to Payment__c creating a fan-out on the EOB-to-Payment junction.

**Environment:** Production sandbox (Partial Copy). Reproducible on 14 of 14 test Claim__c records with COB_Indicator__c = true. First observed after Sprint 15 deployment of INS-1418 EOB enhancements.

## Acceptance Criteria

As per Description. Additionally: after fix deployment, execute the reconciliation report against the 14 identified COB test Claim__c records and confirm 'Total Disbursed' matches the SOQL aggregate sum within $0.00 variance. Regression test non-COB claims to confirm no change to their aggregated totals.

## Solution

Investigation confirmed the report type 'EOB with Payments' introduced in INS-1418 creates a many-to-many fan-out when Explanation_of_Benefits__c has multiple child EOB_Line__c records each joined to the same Payment__c. Fix: modify the custom report type to use a primary object of Payment__c with a related Explanation_of_Benefits__c lookup (not the reverse), ensuring Payment__c is the grain of the report. Add a bucket field to segment Payment__c.Payer_Type__c into 'Primary' and 'Secondary' buckets for separate sub-total columns. Update the 'Total Disbursed' summary formula to use PARENTGROUPVAL to avoid cross-object double-counting. Deploy report type change to sandbox, validate with QA against 14 COB test records, then promote to production.

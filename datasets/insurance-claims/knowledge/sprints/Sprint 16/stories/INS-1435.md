# INS-1435: Scheduled report distribution: automate weekly claims analytics digest email to regional Claims Operations Managers

## Description

As a Claims Operations Manager, I need an automated weekly email digest delivered every Monday at 7 AM (recipient's local timezone) that summarizes the prior week's claims activity from the 'Claims Ops Executive Dashboard' CRM Analytics app so that I can review performance metrics before the weekly ops standup without manually pulling reports. The digest must include summary KPIs, a link to the live dashboard, and flag any Claim__c records where Claim__c.Cycle_Time_Days__c exceeded the 30-day SLA threshold during the prior week.

## Acceptance Criteria

Given the automated digest job is configured and active,
When Monday arrives at 7 AM in the recipient's Salesforce User.TimeZoneSidKey timezone,
Then each user with the 'Claims_Executive_Viewer' Permission Set Group must receive an HTML email containing: total claims closed in the prior week (Claim__c.Status__c changed to 'Adjudicated' within Mon–Sun), total Payment__c.Amount__c disbursed in the prior week, count of Claim__c records where Cycle_Time_Days__c > 30 (SLA breached), and a hyperlink deep-linking to the 'Claims Ops Executive Dashboard' CRM Analytics app.

Given the digest email is generated,
When there are Claim__c records with Cycle_Time_Days__c > 30 adjudicated in the prior week,
Then the email body must include a highlighted section listing up to 10 such claims with: Claim__c.Claim_Number__c, Claim__c.Cycle_Time_Days__c, and Claim__c.Adjudicator__c (lookup to User).

Given a Claims Operations Manager is assigned to a specific Region__c value on their User record,
When the digest is generated,
Then the KPI figures must be scoped to Claim__c records where Claim__c.Region__c matches the recipient's User.Region__c custom field, ensuring regional relevance.

Given the digest job fails to execute for any recipient,
When the failure occurs,
Then a Platform Event 'Report_Digest_Failure__e' must be published with fields: Recipient_User_Id__c, Failure_Reason__c, and Scheduled_Run_Time__c, and the Salesforce admin email alias must receive an error notification within 15 minutes.

## Solution

Implement a Scheduled Apex class 'WeeklyClaimsDigestScheduler' that runs every Monday at 7 AM UTC (timezone offset handled per User.TimeZoneSidKey). The class queries User records with 'Claims_Executive_Viewer' PSG, then for each user executes aggregate SOQL against Claim__c and Payment__c filtered by prior week date range and User.Region__c. Email body built using a Visualforce Email Template 'Weekly_Claims_Digest_VF' with merge fields for KPI values and a conditional section for SLA-breached claims (capped at 10 records via LIMIT). Emails dispatched via Messaging.sendEmail in batch mode respecting governor limits. On failure, publish Report_Digest_Failure__e Platform Event via EventBus.publish(); a Process Automation subscribes to the event and sends admin alert via org-wide email alias.

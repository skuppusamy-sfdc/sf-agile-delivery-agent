## Description

**Bug Report — Network Termination Dashboard Active Count Inflation**

**Environment:** Full Sandbox | Salesforce API v59.0

**Steps to Reproduce:**
1. Navigate to the 'Provider Network Termination Overview' dashboard.
2. Observe the 'Active Terminations in Progress' KPI tile.
3. Cross-reference by running the underlying 'Active Network Termination Cases' report directly.
4. Compare the KPI tile count to a manual SOQL query: SELECT COUNT() FROM Network_Termination_Case__c WHERE Status__c NOT IN ('Withdrawn','Closed','Completed').

**Expected Behavior:**
The 'Active Terminations in Progress' tile should only count Network_Termination_Case__c records where Status__c IN ('Initiated', 'Pending Provider Response', 'Appeal Review', 'Approved — Pending Effective Date').

**Actual Behavior:**
The dashboard KPI tile count includes Network_Termination_Case__c records where Status__c = 'Withdrawn', inflating the active count. In current data, 23 withdrawn cases are being counted, overstating active terminations by ~18%.

**Root Cause:**
The dashboard component's source report filter on Network_Termination_Case__c.Status__c uses 'does not equal Closed' but fails to also exclude 'Withdrawn' and 'Completed' statuses, which were added to the Status__c picklist in Sprint 14 without updating the report filter.

**Impact:** Medium — Provider Network Specialists are making incorrect staffing decisions based on inflated termination workload counts.

## Acceptance Criteria

As per Description.

Given the fix is applied,
When the 'Provider Network Termination Overview' dashboard KPI tile renders,
Then the count matches the result of: SELECT COUNT() FROM Network_Termination_Case__c WHERE Status__c IN ('Initiated','Pending Provider Response','Appeal Review','Approved — Pending Effective Date').

Given Status__c = 'Withdrawn' records exist,
When the dashboard loads,
Then zero 'Withdrawn' Network_Termination_Case__c records appear in the active count tile.

## Solution

Update the source report 'Active Network Termination Cases' filter on Network_Termination_Case__c.Status__c from 'does not equal Closed' to 'equals Initiated OR Pending Provider Response OR Appeal Review OR Approved — Pending Effective Date' using an explicit inclusion filter to be resilient to future picklist additions. Refresh the dashboard to pick up the corrected report. Add a code comment in the report description noting that any new Status__c picklist values must be evaluated for inclusion/exclusion in this filter. No Apex or Flow changes required — report metadata only, deployed via change set.

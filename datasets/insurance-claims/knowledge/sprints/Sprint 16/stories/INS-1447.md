## Description

As a Claims Operations Manager, I need an EOB payment accuracy report reconciling Claim_Line__c billed amounts against final paid amounts on linked Explanation_of_Benefits__c records so that I can identify systemic over/underpayment patterns by CPT code, provider, and LOB, and support ERA submission accuracy to CMS.

## Acceptance Criteria

Given the Claims Operations Manager opens the 'EOB Payment Accuracy' CRM Analytics dashboard,
When the date range filter is set,
Then the report must join Claim_Line__c.Billed_Amount__c, Claim_Line__c.Allowed_Amount__c, and Explanation_of_Benefits__c.Paid_Amount__c for all EOB records where EOB.Status__c = 'Finalized' within the selected period.

Given the joined dataset is rendered,
When variance is calculated,
Then Variance__c = Claim_Line__c.Billed_Amount__c - Explanation_of_Benefits__c.Paid_Amount__c must be displayed per line, and lines where ABS(Variance__c) > 500 must be flagged with a 'Review Required' label.

Given the report is grouped by CPT code,
When data is displayed,
Then the top 10 CPT codes by total absolute variance must be surfaced in a ranked bar chart with drill-down to underlying Claim_Line__c records.

Given the specialist drills into a flagged line,
When the detail panel opens,
Then it must show: Claim__c.Name, Provider__c.NPI__c, Claim_Line__c.CPT_Code__c, Claim_Line__c.ICD10_Primary__c, Fee_Schedule__c.Base_Rate__c, EOB.Paid_Amount__c, Variance__c, and Claim__c.Line_of_Business__c.

Given the report is exported,
When the ERA reconciliation export is triggered,
Then output must conform to the X12 835 field mapping spec: PR (Patient Responsibility), CO (Contractual Obligation), OA (Other Adjustment) reason codes derived from Claim_Line__c.Adjustment_Reason_Code__c.

## Solution

Implement as a CRM Analytics (CRMA) dashboard with a Recipe that joins Claim_Line__c, Explanation_of_Benefits__c, and Fee_Schedule__c datasets via Claim__c as the bridge object. A calculated column in the Recipe computes Variance and flags rows exceeding the $500 threshold using a CASE expression. The CPT bar chart is a CRMA Bar widget linked to a facet filter for drill-down. The ERA export is handled by a custom Apex REST endpoint (EraExportController.cls) that formats output as a downloadable flat file mapped to X12 835 segment structure. A Platform Event (Payment_Variance_Alert__e) is published for each flagged line so downstream monitoring flows can create follow-up Tasks on the parent Claim__c.

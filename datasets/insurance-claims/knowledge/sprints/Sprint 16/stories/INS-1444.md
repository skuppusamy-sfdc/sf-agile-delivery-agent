## Description

As a Claims Operations Manager, I need a real-time analytics dashboard surfacing claims adjudication KPIs, aging buckets, and payment throughput so that I can monitor operational health and intervene on bottlenecks before SLA breaches occur. The dashboard should aggregate data across all active LOBs and support drill-down from summary tiles into underlying Claim__c records.

## Acceptance Criteria

Given the Claims Operations Manager navigates to the 'Claims Ops Dashboard' Lightning App Page,
When the page loads,
Then KPI tiles must display: (1) Total Open Claims count where Claim__c.Status__c IN ('Submitted','In Review','Pending Info'), (2) Average Days to Adjudication calculated from Claim__c.FNOL_Date__c to Claim__c.Adjudication_Date__c, (3) Total Payments Disbursed (MTD) summing Payment__c.Amount__c where Payment__c.Status__c = 'Paid' and Payment__c.Payment_Date__c = THIS_MONTH, (4) Denial Rate % = denied Claim__c records / total closed Claim__c records for the rolling 30-day window.

Given the dashboard is loaded,
When the Claims Manager views the Adjudication Funnel chart,
Then a horizontal bar or funnel chart must show counts per Claim__c.Status__c stage in the order: FNOL Received → In Review → Pending Info → Adjudicated → Payment Issued → Closed, with stage-over-stage drop-off percentages rendered as secondary labels.

Given the Claims Manager selects an LOB filter (e.g., 'Commercial Health', 'Workers Comp', 'Auto'),
When the filter is applied,
Then all KPI tiles, the funnel chart, and the aging grid must re-render scoped to Claim__c.Line_of_Business__c matching the selected value without a full page reload.

Given the aging grid is visible,
When data is rendered,
Then Claim__c records must be bucketed into columns: 0–15 days, 16–30 days, 31–60 days, 61–90 days, 90+ days (calculated from Claim__c.FNOL_Date__c to TODAY), with row-level drill-down opening a filtered list view of underlying claims.

Given a KPI tile is clicked,
When the drill-down modal opens,
Then a datatable must display Claim__c.Name, Claim__c.Claimant_Name__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, Claim__c.Assigned_Adjuster__c, and Claim__c.FNOL_Date__c columns, sortable and exportable to CSV.

-Given the dashboard includes a SIU referral widget showing open SIU flags per adjuster (deprecated — moved to INS-1389)-

## Solution

Implement the dashboard as a Lightning App Page composed of multiple LWC components: (1) 'claimsKpiTile' LWC using wire-connected Apex controller (ClaimsDashboardController.cls) with SOQL aggregates and @AuraEnabled(cacheable=true) methods; (2) 'adjudicationFunnel' LWC using Chart.js wrapped in a Salesforce-safe canvas component for the funnel visualization; (3) 'claimsAgingGrid' LWC with dynamic SOQL date bucketing via Apex. A parent 'claimsOpsDashboard' LWC manages LOB filter state and broadcasts selection via LMS (Lightning Message Service) using a ClaimsDashboardFilterChannel message channel so all child components react independently. CRM Analytics (Einstein Analytics) dataset refresh scheduled every 15 minutes via a Scheduled Flow invoking a REST callout to the Analytics REST API. Permission Set Group 'Claims_Ops_Dashboard_Access' gates visibility.

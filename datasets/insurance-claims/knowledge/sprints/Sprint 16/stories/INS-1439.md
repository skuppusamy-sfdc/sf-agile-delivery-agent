## Description

As a Credentialing Specialist, I need a Salesforce report and dashboard component that tracks Credentialing_Case__c pipeline throughput by stage, assigned specialist, and CAQH verification lag time so that the credentialing team can identify bottlenecks, meet NCQA turnaround standards, and ensure providers are credentialed before their PNA effective dates.

The report must surface cases where CAQH verification has been pending for more than 5 business days, as this is a key SLA breach indicator.

## Acceptance Criteria

Given Credentialing_Case__c records exist in the system,
When a Credentialing Specialist runs the 'Credentialing Pipeline by Stage' report,
Then the report displays a matrix of Credentialing_Case__c records grouped by Credentialing_Case__c.Stage__c (rows) and Credentialing_Case__c.OwnerId (columns), with count and average Credentialing_Case__c.Days_In_Stage__c as measures.

Given a Credentialing_Case__c.Stage__c = 'CAQH Verification' and Credentialing_Case__c.CAQH_Submitted_Date__c is populated,
When the report runs,
Then records where (TODAY() - Credentialing_Case__c.CAQH_Submitted_Date__c) > 5 are flagged with a 'CAQH Lag Breach' bucket column.

Given the dashboard component 'Credentialing Throughput' is added to the Credentialing Manager dashboard,
When it renders,
Then it shows a stacked bar chart of cases opened vs. cases closed (Credentialing_Case__c.Status__c = 'Credentialed') per week for the trailing 8 weeks.

Given a case has Credentialing_Case__c.Provider_PNA_Effective_Date__c populated,
When that date is within 10 calendar days and Credentialing_Case__c.Status__c != 'Credentialed',
Then the record appears in a 'At Risk — PNA Start Approaching' report section highlighted in red.

Given the report is scheduled,
When it runs every Monday at 7:00 AM CT,
Then a report subscription email is sent to all users in the 'Credentialing_Team' public group.

## Solution

Build a native Salesforce Report using the Credentialing_Case__c object with a custom report type that includes Provider__c (lookup) and PNA__c (lookup) related fields. Add a formula field 'CAQH_Lag_Days__c' (formula: TODAY() - CAQH_Submitted_Date__c) to Credentialing_Case__c to support bucketing. Create bucket columns in the report for lag breach thresholds. The 'At Risk' section uses a report filter: PNA_Effective_Date__c <= NEXT_N_DAYS:10 AND Status__c != 'Credentialed'. Dashboard component uses a stacked bar chart report. Report subscription configured via Salesforce native scheduling to the 'Credentialing_Team' public group. A Record-Triggered Flow on Credentialing_Case__c updates Days_In_Stage__c when Stage__c changes, using a before-save flow that calculates the delta from a Stage_Entry_Timestamp__c field.

## Description

As an Underwriting Analyst, I need a Salesforce analytics report and dashboard component that shows broker-level production metrics including written premium volume, active policy count, and loss ratio segmented by LOB and appointment status so that I can identify high-performing brokers for preferred network tiers and flag underperforming books for remediation review.

Key objects: Policy__c, Claim__c, Account (Broker/Agency), Group_Service_Agreement__c (GSA).

Loss ratio calculation: (Total Incurred Claim__c.Total_Incurred_Amount__c) / (Total Policy__c.Written_Premium__c) × 100 for the rolling 12-month period.

## Acceptance Criteria

Given Policy__c records are linked to a Broker Account via Policy__c.Broker__c,
When the 'Broker Production Summary' report runs,
Then it displays one row per Broker Account with columns: Broker Name, NPN (Account.NPN__c), Appointment Status (Account.Appointment_Status__c), LOB (Policy__c.Line_of_Business__c), Policy Count, Total Written Premium (SUM of Policy__c.Written_Premium__c), Total Incurred Claims (SUM of Claim__c.Total_Incurred_Amount__c), and Loss Ratio.

Given the Loss Ratio formula field 'Broker_Loss_Ratio__c' is computed on a summary report,
When Loss Ratio > 85%,
Then the row is highlighted red in the dashboard component; when 70%–85%, amber; when < 70%, green.

Given a Broker Account has Account.Appointment_Status__c = 'Terminated',
When the report runs with the default filter,
Then terminated brokers are excluded unless the 'Include Terminated' checkbox filter is explicitly checked.

Given the report is embedded as a dashboard component,
When an Underwriting Analyst clicks a broker row,
Then they are navigated to the Broker Account record page showing all related Policy__c and GSA__c records.

-Given the report includes a 'Renewal Rate' column calculated from Policy__c.Renewal_Status__c- (removed from scope — deferred to INS-1461 in Sprint 17)

Given the dashboard is accessed,
When the Underwriting Analyst applies the LOB filter,
Then all report components filter to the selected Policy__c.Line_of_Business__c value and the loss ratio recalculates accordingly.

## Solution

Create a custom Salesforce Report Type joining Account (Broker) → Policy__c → Claim__c. Add a cross-object formula field on Policy__c 'Broker_Written_Premium_Rollup__c' if not already present, or use a Roll-Up Summary on a junction object. For Loss Ratio, create a custom summary formula in the report: (SUM(Claim__c.Total_Incurred_Amount__c) / SUM(Policy__c.Written_Premium__c)) * 100. Apply conditional highlighting via Report Conditional Formatting rules (red > 85, amber 70–85, green < 70). Build a Salesforce Dashboard with a summary table component linked to this report. The 'Include Terminated' filter uses a report filter toggle tied to Account.Appointment_Status__c. Row-level drill navigation uses the standard Salesforce report drill-to-record feature. Permission Set 'Underwriting_Analyst_PS' grants access to the report folder 'Underwriting Analytics'.

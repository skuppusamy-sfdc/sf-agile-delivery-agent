## Description

As a Provider Network Specialist, I need an OmniStudio FlexCard-based analytics panel on the Provider 360 page that displays contracted provider counts by specialty and a heatmap of Provider_Network_Agreement__c (PNA) expiration risk so that I can proactively manage contract renewals and avoid network adequacy gaps.

Extends INS-1370 (Provider 360 base page layout). The panel should highlight PNAs expiring within 30, 60, and 90 days and flag agreements where Provider__c.Credentialing_Status__c is not 'Active'.

## Acceptance Criteria

Given a Provider Network Specialist views the Provider 360 Lightning page,
When the 'Network Analytics' FlexCard loads,
Then it displays a count of Provider__c (Person Account) records with active PNAs grouped by Provider__c.Specialty__c field.

Given Provider_Network_Agreement__c records exist,
When the expiration heatmap section renders,
Then PNAs with Provider_Network_Agreement__c.Expiration_Date__c within 0–30 days are shown in RED, 31–60 days in AMBER, and 61–90 days in YELLOW, with counts per bucket.

Given a PNA record appears in the heatmap,
When the related Provider__c.Credentialing_Status__c != 'Active',
Then an inline warning icon is displayed next to the provider name with tooltip text 'Credentialing not current — review required'.

Given the user clicks a heatmap bucket,
When the drill-down panel opens,
Then it lists Provider_Network_Agreement__c.Name, Provider_Network_Agreement__c.Expiration_Date__c, Provider__c.NPI__c, and Provider__c.TIN__c for all matching records.

Given the FlexCard is rendered,
When no PNAs fall within the 90-day expiration window,
Then the heatmap displays a 'No contracts at risk' message with a green status indicator.

## Solution

Build an OmniStudio FlexCard ('Provider_Network_Analytics_FC') consuming a DataRaptor Extract that queries Provider_Network_Agreement__c joined to Provider__c (Person Account). Use conditional styling blocks in the FlexCard for color-coded expiration buckets based on date difference formula fields. Drill-down implemented via a child FlexCard triggered by a card action. The FlexCard is embedded on the Provider 360 Lightning Record Page via an App Builder slot defined in INS-1370. DataRaptor caches results for 10 minutes to limit SOQL governor impact. A Custom Permission 'View_Network_Analytics' is required on the user's profile or Permission Set.

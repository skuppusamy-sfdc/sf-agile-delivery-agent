## Description

As a Provider Network Specialist, I need an automated expiration report and proactive email alerts for Provider_Network_Agreement__c records approaching their end date so that contract renewals are initiated before agreements lapse and provider continuity is maintained.

Alerts must fire at 90, 60, and 30 days prior to Provider_Network_Agreement__c.Effective_End_Date__c and route to the assigned Provider Relations Specialist.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where Status__c = 'Active',
When today's date is exactly 90, 60, or 30 days before Effective_End_Date__c,
Then a Task is auto-created on the PNA record with Subject = 'Contract Renewal Required – [X] Days Remaining', Priority = 'High', and OwnerId set to the PNA's Provider_Relations_Specialist__c lookup field value.

Given the Task is created,
When the assigned Provider Relations Specialist receives the notification,
Then an email alert is dispatched using the 'PNA_Renewal_Alert' Email Template, referencing PNA fields: Provider_Network_Agreement__c.Contract_Number__c, Provider__c.Name, and Effective_End_Date__c.

Given the 'PNA Expiration Report' list view is accessed under the Provider Network tab,
When rendered,
Then it displays all PNA records where Effective_End_Date__c is within the next 90 days and Status__c != 'Terminated', sorted ascending by Effective_End_Date__c.

Given a PNA record's Status__c is updated to 'Renewed' or 'Terminated',
When saved,
Then any open renewal Tasks associated with that PNA are automatically closed (Status = 'Completed') and no further alerts fire.

-Given the alert fires, When no Provider_Relations_Specialist__c is assigned, Then the alert routes to the generic 'Provider Network Ops' queue.- (deprecated — now routes to Provider_Network_Manager__c field on PNA; see updated routing matrix)

Given the report is exported,
When a Compliance Officer runs the 'PNA Expiration Audit' report,
Then the report includes columns: Contract_Number__c, NPI__c, TIN__c, Effective_End_Date__c, Days_Until_Expiration__c (formula field), and Assigned_Specialist__c.

## Solution

Implement a Scheduled Flow ('PNA_Expiration_Alert_Flow') running daily at 06:00 UTC querying Provider_Network_Agreement__c records via Get Records where Status__c = 'Active' and Effective_End_Date__c is within rolling 90-day window. Decision elements branch on days-remaining formula to create Tasks and trigger Email Alerts via the Flow Send Email action referencing the 'PNA_Renewal_Alert' Email Template. Days_Until_Expiration__c implemented as a Formula field: TODAY() - Effective_End_Date__c. A supplemental Record-Triggered Flow on PNA fires on Status__c update to 'Renewed'/'Terminated' to bulk-close open Tasks via Update Records. Report built in CRM Analytics pinned to the Provider Network Dashboard.

## Description

As a Compliance Officer, I need a dedicated SIU (Special Investigations Unit) referral trend dashboard in Salesforce that tracks the volume of fraud referrals, current investigation statuses, average investigation cycle time, and total recovery amounts by claim type and LOB so that I can report on anti-fraud program effectiveness to state regulators and internal leadership on a monthly basis.

Key objects: Claim__c (with Claim__c.SIU_Referral_Status__c and Claim__c.SIU_Referral_Date__c fields), Case (SIU Investigation Case linked to Claim__c), Payment__c (recovery payments with Payment__c.Payment_Type__c = 'Recovery').

## Acceptance Criteria

Given Claim__c records with Claim__c.SIU_Referral_Status__c != NULL exist,
When the 'SIU Referral Trends' dashboard loads,
Then a line chart displays monthly referral volume for the trailing 12 months, using Claim__c.SIU_Referral_Date__c as the date axis, segmented by Claim__c.Line_of_Business__c.

Given SIU Cases are linked to Claim__c via Case.Claim__c lookup,
When the investigation status breakdown tile renders,
Then it shows a pie chart of open SIU Cases grouped by Case.Status with counts for: 'Under Investigation', 'Pending Prosecution', 'Closed — Fraud Confirmed', 'Closed — No Fraud Found'.

Given Payment__c records with Payment__c.Payment_Type__c = 'Recovery' are linked to Claim__c,
When the recovery metrics panel renders,
Then total recovery amount (SUM of Payment__c.Amount__c) is displayed for the current year-to-date, segmented by Claim__c.Claim_Type__c.

Given an SIU Case has Case.Status = 'Under Investigation' and Case.CreatedDate < TODAY() - 90,
When it appears in the aged investigations list,
Then it is flagged as 'Aged > 90 Days' with the assigned investigator (Case.OwnerId) and Claim__c.Total_Claim_Amount__c visible.

Given the Compliance Officer exports the dashboard data,
When they click 'Export to CSV',
Then the export includes all visible rows with no PII fields beyond Claim__c.Id and Claim__c.Claim_Number__c (PHI/PII masking enforced per data governance policy).

Given the dashboard is scheduled for monthly distribution,
When it runs on the first business day of each month,
Then a PDF snapshot is emailed to the 'Compliance_Leadership' Chatter group and archived as a ContentDocument linked to the 'SIU Monthly Reports' library.

## Solution

Build a CRM Analytics (Tableau CRM) dashboard 'SIU_Referral_Trends' with a DataFlow joining Claim__c, Case, and Payment__c datasets. Line chart uses a SAQL time-series groupBy on SIU_Referral_Date__c with a month() function. Pie chart uses a groupBy on Case.Status with a filter dataset for SIU case record type. Recovery amount uses a filtered SUM measure on Payment__c.Amount__c where Payment_Type__c = 'Recovery'. Aged investigations list uses a Values Table with a SAQL filter: case_created_date < dateAdd(now(), -90, "day"). PII masking enforced by excluding sensitive fields from the DataFlow output schema; a Field-Level Security review is required before deployment. Monthly PDF export uses CRM Analytics Schedule and Subscribe feature, delivering to a Chatter group via Platform Event trigger that calls a ContentVersion insert Apex class. Permission Set 'SIU_Compliance_Dashboard_PS' gates access.

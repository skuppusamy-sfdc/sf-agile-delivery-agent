## Description

As a Credentialing Specialist, I need a Salesforce native report and dashboard that displays the current pipeline of Credentialing_Case__c records by stage and assigned specialist so that I can identify bottlenecks, balance workload across the credentialing team, and ensure NCQA turnaround time standards are met. The dashboard must include aging indicators for cases exceeding stage-level SLA thresholds sourced from a custom metadata configuration.

## Acceptance Criteria

Given the Credentialing Specialist navigates to the 'Credentialing Operations' dashboard in Salesforce,
When the dashboard loads,
Then a stacked bar chart must display count of Credentialing_Case__c records grouped by Credentialing_Case__c.Stage__c picklist values ('Application Received', 'Primary Source Verification', 'Committee Review', 'Approved', 'Denied', 'Pending Additional Info') for the current quarter.

Given the workload distribution component on the dashboard,
When rendered,
Then a horizontal bar chart must show count of open Credentialing_Case__c records (Status__c != 'Closed') grouped by Credentialing_Case__c.OwnerId resolved to User full name, sorted descending by count.

Given a Credentialing_Case__c record has been in the current Stage__c longer than the SLA threshold defined in the 'Credentialing_SLA_Config__mdt' custom metadata record for that stage,
When it appears in the dashboard data table,
Then the row must display a red 'SLA Breached' icon in the 'SLA_Status__c' formula field column, and the case must be counted in the 'Breached SLA' KPI tile.

Given the report underlying the pipeline chart,
When exported by a user with the 'Credentialing_Manager' permission set,
Then the export must include: Credentialing_Case__c.Name, Credentialing_Case__c.Provider__c (NPI), Credentialing_Case__c.Stage__c, Credentialing_Case__c.Days_In_Stage__c, Credentialing_Case__c.CAQH_Profile_Status__c, and Credentialing_Case__c.NPPES_Verified__c.

-Given the dashboard includes a CAQH integration status widget, When a provider's CAQH attestation is expired, Then the widget must display a red alert banner.- (Removed from this story; CAQH real-time status widget moved to INS-1436.)

Given the dashboard refresh schedule,
When configured,
Then the underlying Salesforce report must be set to auto-refresh every 60 minutes during business hours (8 AM–6 PM local time) via a scheduled report subscription.

## Solution

Create a native Salesforce Report Folder 'Credentialing Analytics' with report type 'Credentialing Cases with Providers'. Build a Salesforce Dashboard using Lightning Dashboard components: stacked bar chart (grouped by Stage__c), horizontal bar (grouped by OwnerId), and a summary KPI tile for SLA breaches. SLA threshold logic implemented as a formula field Credentialing_Case__c.Days_In_Stage__c compared against values retrieved from Credentialing_SLA_Config__mdt via an Apex-invocable method called from a Record-Triggered Flow on Stage__c change, which stamps SLA_Status__c. Dashboard scheduled subscription set to hourly refresh 8 AM–6 PM via Salesforce Report Subscription. Export permissions gated by 'Credentialing_Manager' Permission Set.

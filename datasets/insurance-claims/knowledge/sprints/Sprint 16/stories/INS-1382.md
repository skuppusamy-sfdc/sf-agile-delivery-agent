## Description

**Bug Report — INS-1382**

**Environment:** Full Sandbox — Sprint 16 Regression

**Summary:** The 'Provider Termination Activity' report applying a relative date filter of 'Last 30 Days' on Network_Termination_Case__c.Termination_Effective_Date__c omits records from the prior calendar month when the report is run on the 1st–3rd of the current month. Records with Termination_Effective_Date__c values in the final days of the previous month are excluded despite falling within the 30-day window.

**Steps to Reproduce:**
1. Set system date context to the 2nd of any month (e.g., August 2, 2025).
2. Navigate to Reports > Provider Termination Activity.
3. Confirm filter: Termination_Effective_Date__c = 'Last 30 Days'.
4. Run the report.
5. Manually query: SELECT Id, Termination_Effective_Date__c FROM Network_Termination_Case__c WHERE Termination_Effective_Date__c = LAST_N_DAYS:30 in Developer Console.
6. Compare report row count to SOQL row count — report returns fewer records.

**Expected Behavior:** Report row count equals SOQL LAST_N_DAYS:30 row count.

**Actual Behavior:** Report is applying a 'Current Month' filter override due to a misconfigured 'Filter Logic' row in the report definition that ANDs Termination_Effective_Date__c with a hidden MONTH(Termination_Effective_Date__c) = MONTH(TODAY()) condition introduced in a prior report edit.

**Impact:** Provider Relations Specialists and Compliance Officers are missing termination records in monthly review cycles, creating potential regulatory reporting gaps for PCN (Provider Change Notification) compliance.

**Workaround:** Manually set the date filter to a fixed range (e.g., July 3 – August 2) instead of using the relative 'Last 30 Days' option.

## Solution

Open the 'Provider Termination Activity' report definition in Report Builder. Inspect Filter Logic — identify and remove the erroneous filter row applying MONTH() = MONTH(TODAY()) condition on Termination_Effective_Date__c (introduced during a prior ad-hoc edit). Confirm the sole date filter reads: Network_Termination_Case__c.Termination_Effective_Date__c equals 'Last 30 Days' with no additional AND/OR month-level condition. Save and re-run report. Validate row count matches SOQL: SELECT COUNT() FROM Network_Termination_Case__c WHERE Termination_Effective_Date__c = LAST_N_DAYS:30. Deploy corrected report definition to Production via change set. Add a report description note: 'Do not add month-boundary filter conditions — use relative date filters only per reporting standards.'

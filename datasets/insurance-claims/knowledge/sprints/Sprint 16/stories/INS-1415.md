## Description

As a Claims Operations Manager, I need a real-time KPI dashboard surfacing claim volume, adjudication cycle times, denial rates, and pending payment totals so that I can monitor operational health and identify bottlenecks across all LOBs without running manual reports.

This dashboard will serve as the primary landing page for Claims Operations Manager users and must aggregate data from Claim__c, Claim_Line__c, and Payment__c objects. Extends INS-1385 (Claims Operations homepage redesign).

## Acceptance Criteria

Given a user with the 'Claims Operations Manager' Permission Set Group assigned,
When they navigate to the Claims Operations home page,
Then a CRM Analytics (Tableau CRM) dashboard renders within 5 seconds displaying the following KPI tiles:
  - Total Open Claims (Claim__c.Status__c IN ('Submitted','In Review','Pending Info'))
  - Average Adjudication Cycle Time in days (Claim__c.Adjudication_Date__c - Claim__c.FNOL_Date__c)
  - Denial Rate % (Claim__c.Status__c = 'Denied' / total Claim__c count for period)
  - Total Pending Disbursement (SUM of Payment__c.Amount__c WHERE Payment__c.Status__c = 'Pending')

Given the dashboard is loaded,
When the Claims Operations Manager clicks any KPI tile,
Then a drill-down report opens filtered to the relevant Claim__c records with columns: Claim__c.Name, Claim__c.Policy__c, Claim__c.LOB__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c.

Given the dashboard is open,
When the manager applies the LOB filter (options: Medical, Dental, Vision, Workers Comp),
Then all KPI tiles and charts refresh to reflect only Claim__c records matching Claim__c.LOB__c = selected value.

Given the dashboard date range filter,
When a custom date range is selected,
Then all metrics recalculate using Claim__c.FNOL_Date__c as the anchor date field.

-Given a TPA sub-filter is applied, Then claims are further segmented by Claim__c.TPA_Account__c lookup field.- (Deprecated: TPA segmentation moved to INS-1431)

Given the dashboard is viewed on a 1920x1080 resolution monitor,
When the page loads,
Then no horizontal scrolling is required and all 4 KPI tiles are visible above the fold.

## Solution

Build a CRM Analytics (Tableau CRM) dashboard using SAQL queries against the ClaimsDataset (sourced from Claim__c, Claim_Line__c, Payment__c via a scheduled Dataflow running nightly at 2:00 AM). KPI tiles implemented as number widgets with conditional formatting (red threshold: denial rate > 15%, cycle time > 30 days). Drill-down configured via faceting linked to a connected Claim__c list report. LOB and date range filters implemented as dashboard-level global filters. Permission-based visibility enforced via CRM Analytics Sharing Inheritance tied to the 'Claims Operations Manager' Permission Set Group. Dashboard embedded on the Claims Operations App home page via Lightning App Builder.

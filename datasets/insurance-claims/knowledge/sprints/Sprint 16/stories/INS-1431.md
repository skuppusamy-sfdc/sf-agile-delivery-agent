## Description

As a Provider Network Specialist, I need a CRM Analytics heatmap report that visualizes Provider_Network_Agreement__c (PNA) expiration dates segmented by geographic region and provider specialty so that I can proactively prioritize re-contracting outreach before network adequacy gaps occur. The report must highlight PNAs expiring within 30, 60, and 90 days with distinct color bands and surface the associated Provider__c NPI and TIN for immediate action.

## Acceptance Criteria

Given the Provider Network Specialist opens the 'Network Contracting Analytics' CRM Analytics app,
When the 'PNA Expiration Heatmap' dashboard page loads,
Then a geo-map widget must render U.S. states color-coded by count of Provider_Network_Agreement__c records where Provider_Network_Agreement__c.Expiration_Date__c is within 90 days of today and Provider_Network_Agreement__c.Status__c = 'Active'.

Given the heatmap is displayed,
When the specialist hovers over a state tile,
Then a tooltip must show: state name, count of expiring PNAs, and breakdown by expiration band (0-30 days in red, 31-60 days in amber, 61-90 days in yellow).

Given the specialty filter widget on the dashboard,
When the specialist selects Provider__c.Primary_Specialty__c = 'Orthopedic Surgery',
Then the heatmap and the supporting data table must filter to only PNA records linked to Provider__c records with that specialty value.

Given the data table below the heatmap,
When any expiration band filter is active,
Then each row must display: Provider_Network_Agreement__c.Name, Provider__c.NPI__c, Provider__c.TIN__c, Provider_Network_Agreement__c.Expiration_Date__c, Provider_Network_Agreement__c.Contract_Type__c, and a Salesforce record action button to launch the 'Initiate Re-Contracting' OmniScript.

Given a PNA record where Provider_Network_Agreement__c.Auto_Renew__c = true,
When it appears in the expiration heatmap dataset,
Then it must be visually distinguished with a green 'Auto-Renew' badge in the data table and excluded from the urgent outreach count KPI tile.

## Solution

Build CRM Analytics dashboard page 'PNA Expiration Heatmap' within the existing Network Contracting Analytics app. Dataset sourced via scheduled DataSync from Provider_Network_Agreement__c joined to Provider__c (Person Account). Geo-map widget uses the 'State_Region__c' formula field on PNA. Expiration band classification handled in SAQL using date difference computations against today's date. Specialty filter is a List widget bound to Provider__c.Primary_Specialty__c. Data table rendered as a CRM Analytics List widget with Salesforce Actions enabled to invoke the 'Initiate_Recontracting' OmniScript via a Lightning action URL. Auto-Renew badge implemented via a conditional formatting rule on the table column.

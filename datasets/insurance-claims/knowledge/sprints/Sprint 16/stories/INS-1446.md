## Description

As a Credentialing Specialist, I need a pipeline report showing all Credentialing_Case__c records grouped by stage, assigned specialist, and aging so that I can manage workload distribution, identify stalled applications, and ensure NCQA-mandated 180-day credentialing windows are not breached.

## Acceptance Criteria

Given the Credentialing Specialist navigates to the 'Credentialing Pipeline' report,
When the report renders,
Then it must group Credentialing_Case__c records by Credentialing_Case__c.Stage__c (values: Application Received, Primary Source Verification, Committee Review, Approved, Denied, Expired) and display count and average age (days since Credentialing_Case__c.Application_Date__c) per stage.

Given the report is displayed,
When a record has been in the current stage for more than 30 days,
Then the row must display a red 'Stalled' indicator and Credentialing_Case__c.Stalled_Flag__c must be set to TRUE via a scheduled update.

Given the specialist applies the 'My Cases' filter,
When results render,
Then only Credentialing_Case__c records where Credentialing_Case__c.Assigned_Specialist__c = CURRENT_USER are shown, with a summary footer showing: Total Cases, Cases Due Within 30 Days, Cases Breaching 180-Day Window.

Given a case is within 14 days of the 180-day NCQA deadline (calculated from Credentialing_Case__c.Application_Date__c),
When the report loads,
Then those rows must be highlighted amber and a Bell notification must be dispatched to the assigned specialist via Salesforce In-App Notification.

## Solution

Build a native Salesforce Report (Tabular with Row-Level Formula) on the Credentialing_Case__c object grouped by Stage__c and Assigned_Specialist__c. A Record-Triggered Flow on Credentialing_Case__c fires on update (entry criteria: Stage__c changed) to calculate stage age and stamp Credentialing_Case__c.Days_In_Current_Stage__c. A Scheduled Flow runs nightly to set Credentialing_Case__c.Stalled_Flag__c = TRUE where Days_In_Current_Stage__c > 30. The 180-day breach warning is handled by a Scheduled Apex class (CredentialingDeadlineAlertBatch.cls) that queries approaching deadlines and calls ConnectApi.Notifications to push in-app alerts. The report is embedded in a Lightning App Page via the standard Report Chart component with a dashboard filter for 'My Cases'.

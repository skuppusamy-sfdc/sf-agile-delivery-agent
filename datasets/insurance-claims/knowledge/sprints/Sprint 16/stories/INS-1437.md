## Description

As a Claims Operations Manager, I need an executive-level Lightning dashboard that surfaces real-time KPIs for claims adjudication cycle time, denial rates by LOB, and pending FNOL volumes so that leadership can make data-driven staffing and workflow decisions without exporting to Excel.

This dashboard should aggregate data across Claim__c, Claim_Line__c, and Payment__c objects and present segmented views by LOB (Medical, Dental, Vision, Workers Comp).

## Acceptance Criteria

Given the Claims Operations Manager has the 'Claims_Exec_Dashboard' Permission Set Group assigned,
When they navigate to the 'Claims Operations Executive Dashboard' Lightning page,
Then they see a summary tile showing total open Claim__c records where Claim__c.Status__c IN ('Submitted', 'In Review', 'Pending Info') updated within the last 24 hours.

Given claims data exists across multiple LOBs,
When the dashboard loads,
Then a bar chart displays average Claim__c.Adjudication_Cycle_Days__c grouped by Claim__c.Line_of_Business__c for the current rolling 30-day period, with a benchmark reference line at 14 days.

Given Claim__c records with Claim__c.Status__c = 'Denied',
When the denial rate tile renders,
Then it shows denial rate as (count of Denied Claims / count of total finalized Claims) × 100, segmented by Claim__c.Denial_Reason_Code__c, for the current calendar month.

Given a Claims Operations Manager selects a specific LOB filter on the dashboard,
When the filter is applied,
Then ALL dashboard components refresh to reflect only Claim__c records where Claim__c.Line_of_Business__c matches the selected value, within ≤ 3 seconds.

Given Payment__c records exist with Payment__c.Status__c = 'Pending',
When the pending disbursements tile renders,
Then it displays total Payment__c.Amount__c sum and count, grouped by Payment__c.Payment_Method__c (EFT, Check, Virtual Card).

-Given a drill-down click on any KPI tile navigates the user to a pre-filtered list view of Claim__c records- (deprecated — drill-through replaced by related Quick Action modal per INS-1429)

Given the dashboard is viewed on a 1080p monitor,
When rendered,
Then all components are visible without horizontal scrolling and comply with WCAG 2.1 AA contrast standards.

## Solution

Build a native Salesforce Lightning Dashboard using CRM Analytics (Tableau CRM) with SAQL-based datasets joining Claim__c, Claim_Line__c, and Payment__c. Create a DataFlow scheduled to run every 4 hours to refresh the 'Claims_Ops_KPI' dataset. Use an Extended Metadata (XMD) file to define LOB color coding. A Permission Set Group 'Claims_Exec_Dashboard' will gate access. Dashboard filters will use Global Filters bound to LOB__c dimension. Denial rate KPI uses a Compare Table widget with a computed measure. Cycle time benchmark line implemented via a Reference Line on the bar chart widget. All SAQL queries parameterized to support rolling date windows.

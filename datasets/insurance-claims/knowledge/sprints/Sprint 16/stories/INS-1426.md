## Description

As a Provider Relations Specialist, I need a FlexCard embedded on the Provider__c (Person Account) record page that consolidates the provider's active PNA details, current credentialing case status, NPI/TIN verification state, and a rolling 90-day claims summary so that I can conduct informed provider outreach calls without navigating across multiple Salesforce objects.

Extends INS-1424 (PNA reporting) by surfacing contract expiration alerts directly on the provider record.

## Acceptance Criteria

Given a Provider__c record page is loaded by a Provider Relations Specialist,
When the 'Provider 360' FlexCard component renders,
Then the following data blocks are displayed:
  - Provider__c.NPI__c and Provider__c.TIN__c with NPPES verification badge (Verified / Unverified) sourced from Provider__c.NPPES_Verification_Status__c
  - Active Provider_Network_Agreement__c records (Status__c = 'Active') with PNA name, effective date, and Contract_End_Date__c
  - Most recent Credentialing_Case__c record with Status__c and Days_Open__c
  - Count of Claim__c records linked via Provider__c lookup where Claim__c.Service_Date__c >= LAST 90 DAYS, grouped by Claim__c.Status__c

Given the FlexCard PNA section,
When Provider_Network_Agreement__c.Contract_End_Date__c <= TODAY() + 90,
Then a yellow warning badge labeled 'Expiring Soon' is displayed next to the PNA name.

Given the FlexCard PNA section,
When Provider_Network_Agreement__c.Contract_End_Date__c <= TODAY() + 30,
Then the badge escalates to red and is labeled 'Critical — Renewal Required'.

Given the credentialing block,
When Credentialing_Case__c.Status__c = 'Approved' AND Credentialing_Case__c.Expiration_Date__c > TODAY(),
Then a green 'Credentialed' badge is shown; otherwise a contextual status badge reflects the current state.

Given the Provider Relations Specialist clicks 'View All Claims' in the claims summary block,
When the action fires,
Then a modal opens displaying a filterable list of Claim__c records with columns: Claim__c.Name, Claim__c.Service_Date__c, Claim__c.Total_Billed_Amount__c, Claim__c.Status__c.

-Given a 'Send DocuSign Envelope' quick action button on the FlexCard- (deprecated — DocuSign quick action relocated to PNA record page per UX review)

## Solution

Build using OmniStudio FlexCards. Primary FlexCard 'Provider360_FC' uses a DataRaptor Extract source to query Provider__c fields, active Provider_Network_Agreement__c child records (SOQL with Status__c filter), most recent Credentialing_Case__c (ORDER BY CreatedDate DESC LIMIT 1), and Claim__c aggregate counts via a DataRaptor Transform with SOQL aggregate. Conditional badge styling implemented via FlexCard State Actions with CSS class bindings on Contract_End_Date__c and Credentialing_Case__c.Status__c. 'View All Claims' modal implemented as a child FlexCard ('Provider360_Claims_Modal_FC') invoked via a FlexCard Action of type 'Show Card'. Embed the FlexCard on the Provider__c Lightning Record Page via App Builder, restricted to the 'Provider_Relations' Profile and 'Provider_Ops_PSG' Permission Set Group.

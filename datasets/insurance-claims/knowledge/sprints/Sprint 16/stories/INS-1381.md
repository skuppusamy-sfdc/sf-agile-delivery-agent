## Description

As a Claims Operations Manager, I need Agentforce to automatically generate a plain-language claim summary narrative when a new Claim__c record reaches Status__c = 'Under Review' so that adjusters have an AI-drafted synopsis of the claim — including diagnosis codes, billed amounts, provider details, and COB flags — reducing manual review preparation time by an estimated 30%.

The generated narrative must be stored on Claim__c.AI_Summary__c (Long Text Area) and surfaced in the adjuster workspace.

## Acceptance Criteria

Given a Claim__c record's Status__c transitions to 'Under Review',
When the Record-Triggered Flow fires,
Then an Agentforce prompt action is invoked within 30 seconds, passing the following context fields: Claim__c.Claim_Number__c, Claim__c.Total_Billed_Amount__c, Claim__c.COB_Flag__c, Claim__c.Primary_Diagnosis_Code__c (ICD-10), related Claim_Line__c CPT codes (up to 10 lines), and Provider__c.Name + NPI__c.

Given the Agentforce prompt executes successfully,
When the AI response is received,
Then Claim__c.AI_Summary__c is populated with a narrative not exceeding 500 words, written in plain English, summarizing the claim event, services rendered, billed vs. allowed amounts, and any COB or SIU flags.

Given Claim__c.AI_Summary__c is populated,
When the adjuster opens the Claim__c record,
Then the AI Summary is displayed in a dedicated 'AI Claim Narrative' section on the Lightning record page with a visible 'AI-Generated — Review Required' disclaimer label.

Given the Agentforce API call fails or times out,
When the error is caught,
Then Claim__c.AI_Summary_Status__c is set to 'Generation Failed', a Platform Event (Agentforce_Error__e) is published for monitoring, and the adjuster sees an inline error message prompting manual summary entry.

Given a user with the 'SIU_Analyst' Permission Set views the record,
When Claim__c.SIU_Flag__c = TRUE,
Then an additional AI-generated fraud risk synopsis paragraph is appended to AI_Summary__c, prefixed with '[SIU NOTICE]'.

## Solution

Implement a Record-Triggered Flow ('Claim_Agentforce_Summary_Flow') on Claim__c triggering on Status__c change to 'Under Review' (entry condition: ISCHANGED(Status__c) AND Status__c = 'Under Review'). Flow calls an Apex invocable action ('AgentforceClaimSummaryAction') that constructs a prompt payload by querying related Claim_Line__c (up to 10 records) and Provider__c fields, then calls the Agentforce Einstein Generative AI API via a named credential ('Agentforce_NC'). Response text is written back to Claim__c.AI_Summary__c via a Flow Update Records element. Error handling: Try-Catch in Apex sets AI_Summary_Status__c = 'Generation Failed' and publishes Agentforce_Error__e Platform Event consumed by a monitoring flow alerting the integration ops team. SIU paragraph appended via conditional logic in Apex checking Claim__c.SIU_Flag__c. AI Summary section on Lightning page built as a custom LWC ('claimAiNarrativePanel') with the disclaimer badge.

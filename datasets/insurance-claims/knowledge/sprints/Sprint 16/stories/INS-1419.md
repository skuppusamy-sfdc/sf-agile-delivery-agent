## Description

As a Provider Relations Specialist, I need the Provider 360 page to display an inline panel showing any active Network_Termination_Case__c records and the last 5 Provider_Change_Notification__c (PCN) entries for a provider so that I can quickly assess a provider's network standing and compliance history without navigating away from the provider record.

This story extends INS-1402 (Provider 360 base FlexCard layout).

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c (Person Account) record page,
When the Provider 360 component loads,
Then a 'Network Status' tab is visible within the FlexCard displaying:
  - Active Network_Termination_Case__c records WHERE Network_Termination_Case__c.Provider__c = current Provider__c.Id AND Network_Termination_Case__c.Status__c NOT IN ('Closed','Withdrawn')
  - Columns: Network_Termination_Case__c.Name, Network_Termination_Case__c.Termination_Reason__c, Network_Termination_Case__c.Effective_Termination_Date__c, Network_Termination_Case__c.Status__c

Given the 'Network Status' tab is active,
When no active Network_Termination_Case__c records exist for the provider,
Then the panel displays 'No active termination cases' with a green status indicator.

Given the 'Network Status' tab is active,
When active Network_Termination_Case__c records exist,
Then a red banner 'Active Termination In Progress' is displayed at the top of the tab.

Given the Provider 360 page loads,
When the 'PCN History' sub-section renders,
Then the 5 most recent Provider_Change_Notification__c records are displayed (ordered by PCN__c.CreatedDate DESC) with columns: PCN__c.Name, PCN__c.Change_Type__c, PCN__c.Effective_Date__c, PCN__c.Status__c.

Given a PCN__c.Status__c = 'Rejected',
When that PCN record is rendered in the history list,
Then the row is displayed with orange text styling.

Given the Provider__c record has no PCN history,
When the PCN History sub-section loads,
Then 'No PCN records found' is displayed.

## Solution

Extend the existing Provider 360 OmniStudio FlexCard (INS-1402) by adding a new 'Network Status' tab state. Use two DataRaptor Extract actions: (1) DR_GetActiveTerminationCases — queries Network_Termination_Case__c filtered by Provider__c and Status__c; (2) DR_GetPCNHistory — queries Provider_Change_Notification__c filtered by Provider__c, ordered by CreatedDate DESC, LIMIT 5. Conditional state logic for red banner implemented via FlexCard state conditions on termination case count > 0. PCN row orange styling implemented via FlexCard conditional field-level CSS class binding on PCN__c.Status__c = 'Rejected'. Both DataRaptors registered and activated in OmniStudio. FlexCard redeployed to the Provider__c record page Lightning App Builder layout.

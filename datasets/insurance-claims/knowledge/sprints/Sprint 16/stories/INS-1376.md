## Description

As a Claims Operations Manager, I need a real-time analytics dashboard surfacing adjudication throughput, average cycle time, and denial rates across LOBs so that I can identify bottlenecks and meet SLA targets without exporting data to external spreadsheets.

This dashboard will serve as the primary operational view for sprint 16 reporting deliverables and extends foundational work established in INS-1346.

## Acceptance Criteria

Given the Claims Operations Manager navigates to the 'Claims Ops Dashboard' Lightning App Page,
When the page loads,
Then a CRT (Claims Resolution Time) KPI tile displays the average number of days between Claim__c.FNOL_Date__c and Claim__c.Adjudicated_Date__c, filtered to the current calendar month.

Given the dashboard is rendered,
When the LOB filter picklist (values: Medical, Dental, Vision, Behavioral Health) is applied,
Then all report components refresh to reflect only Claim__c records where Claim__c.Line_of_Business__c matches the selected value.

Given claims data is present,
When the Adjudication Throughput chart renders,
Then it displays a bar chart grouped by week showing count of Claim__c records where Claim__c.Status__c = 'Adjudicated' for the trailing 13 weeks.

Given the Denial Rate tile is visible,
When computed,
Then it shows (COUNT of Claim__c where Status__c = 'Denied') / (COUNT of all closed Claim__c) expressed as a percentage, with a red threshold indicator when value exceeds 15%.

Given a claim record is clicked from any dashboard table component,
When navigation occurs,
Then the user is taken directly to the Claim__c record detail page without losing dashboard filter context.

-Given an export button is present, When clicked, Then a CSV of all dashboard data downloads to the user's machine.- (deprecated — export removed per compliance review, use standard Salesforce report export instead)

Given the dashboard is accessed by a user without the 'Claims_Ops_Manager' Permission Set Group,
When the page loads,
Then a 'Access Restricted' message is displayed and no PHI claim data is visible.

## Solution

Create a Lightning App Page using App Builder housing five Report Chart components and two custom LWC KPI tiles (ClaimsKpiTile). KPI tiles query Claim__c via @wire(getRecord) with dynamic SOQL scoped to running user's assigned region via a Sharing Rule on Claim__c (criteria: Claim__c.Region__c = User.Region__c). LOB filter implemented as a LWC parent component emitting a CustomEvent('lobFilterChange') consumed by sibling KPI tiles. Denial rate threshold alert uses conditional CSS class binding. Permission enforcement via Permission Set Group 'Claims_Ops_Manager_PSG'. All report definitions stored as CRM Analytics (Tableau CRM) datasets refreshed via nightly scheduled dataflow on Claim__c and Claim_Line__c objects.

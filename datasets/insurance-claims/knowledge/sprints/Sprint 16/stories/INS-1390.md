## Description

**Bug Report — Claims Operations Dashboard Denial Rate KPI Tile**

**Environment:** Acme Insurance Full Sandbox (FS1)
**Build:** Sprint 16 deployment — INS-1384 implementation

**Steps to Reproduce:**
1. Log in as a user with the 'Claims_Ops_Manager' Permission Set Group.
2. Navigate to the 'Claims Ops Dashboard' Lightning App Page.
3. Observe the 'Denial Rate %' KPI tile — note the value displayed with no LOB filter applied (e.g., 12.4%).
4. Apply the LOB filter and select 'Workers Comp'.
5. Observe the 'Denial Rate %' KPI tile value after filter is applied.

**Expected Behavior:**
The denial rate KPI tile should recalculate and display the denial rate percentage scoped only to Claim__c records where Claim__c.LOB__c = 'Workers Comp' for the rolling 30-day window.

**Actual Behavior:**
The denial rate KPI tile continues to display the unfiltered (all-LOB) denial rate percentage (e.g., 12.4%) regardless of the 'Workers Comp' LOB filter selection. All other KPI tiles (open claim count, adjudication throughput, average days-to-pay) respond correctly to the LOB filter. The denial rate tile appears to be ignoring the Lightning Message Channel broadcast.

**Root Cause Hypothesis:**
The lwc/claimsKpiTile component instance rendering the denial rate metric may not be subscribed to the ClaimsDashboardFilter__c Lightning Message Channel, or the messageHandler is not correctly extracting the LOB filter payload and passing it to the ClaimsOpsMetricsController.getDenialRate() Apex method.

**Impact:** High — Claims Operations Managers are making LOB-specific staffing decisions based on an inaccurate Workers Comp denial rate figure. Workers Comp LOB has a distinct regulatory reporting requirement under state TPA statutes.

## Acceptance Criteria

As per Description. Verify fix by confirming the denial rate KPI tile value changes when LOB filter = 'Workers Comp' is applied and matches the value returned by directly executing: SELECT COUNT() FROM Claim__c WHERE LOB__c = 'Workers Comp' AND Status__c = 'Denied' AND Adjudication_Date__c = LAST_N_DAYS:30 divided by total closed Workers Comp claims in the same window.

## Solution

Investigate lwc/claimsKpiTile component's @wire(MessageContext) subscription registration. Confirm that the denial rate tile instance includes a subscribeToMessageChannel() call in connectedCallback() referencing ClaimsDashboardFilter__c LMC and that the handleFilterChange() messageHandler correctly passes the lobFilter parameter to the imperative Apex call ClaimsOpsMetricsController.getDenialRate(lobFilter). Likely fix: the denial rate tile was cloned from the throughput tile component but the LMC subscription was not added during the clone. Add subscription, re-deploy LWC, and validate in FS1 sandbox with test data covering Workers Comp denied claims.

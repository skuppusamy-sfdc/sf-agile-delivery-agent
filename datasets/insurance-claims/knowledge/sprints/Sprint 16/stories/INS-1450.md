## Description

As a Compliance Officer, I need a Network Termination Case aging dashboard that shows all open Network_Termination_Case__c records grouped by termination reason, days open, and required notice period compliance status so that I can ensure the organization meets state-mandated provider termination notice requirements (typically 60–90 days) and avoid regulatory penalties.

## Acceptance Criteria

Given the Compliance Officer opens the 'Network Termination Aging' Lightning App Page,
When the page loads,
Then all Network_Termination_Case__c records where Network_Termination_Case__c.Status__c NOT IN ('Completed','Withdrawn') must be displayed grouped by Network_Termination_Case__c.Termination_Reason__c (values: Quality of Care, Fraud, Contract Non-Renewal, Voluntary, Credentialing Failure).

Given the grouped list is rendered,
When a termination case has a Network_Termination_Case__c.Required_Notice_Date__c within 10 days of TODAY,
Then the row must display a red 'Notice Due Soon' badge and trigger an automatic Task on the case owner with Subject = 'Urgent: Termination Notice Required – [Provider Name]'.

Given the dashboard is viewed,
When the compliance summary panel renders,
Then it must show: (1) Total Open Termination Cases, (2) Cases Within Notice Window (Required_Notice_Date__c <= TODAY + 10), (3) Cases Overdue (Required_Notice_Date__c < TODAY AND Status__c != 'Completed'), (4) Average Days Open (TODAY - Network_Termination_Case__c.CreatedDate).

Given a case row is selected,
When the detail view opens,
Then it must display the linked Provider__c.NPI__c, Provider__c.Primary_Specialty__c, the associated PCN record (Provider_Change_Notification__c.Status__c), and any active Claim__c records in flight for that provider (Claim__c.Provider__c = provider AND Claim__c.Status__c NOT IN ('Closed','Denied')).

## Solution

Implement as a Lightning App Page with a custom LWC ('terminationCaseAgingDashboard') using Apex controller (TerminationCaseAgingController.cls) with a SOQL query joining Network_Termination_Case__c, Provider__c, and Provider_Change_Notification__c. Notice window breach detection is handled by a Record-Triggered Flow on Network_Termination_Case__c that fires on create/update, evaluates Required_Notice_Date__c proximity using a formula, and creates a high-priority Task via the standard Create Records element. The compliance summary panel KPIs use @AuraEnabled(cacheable=false) Apex methods with real-time SOQL aggregates to avoid stale data. A Scheduled Flow runs at 7 AM daily to re-evaluate all open cases and publish a Compliance_Alert__e Platform Event consumed by the Compliance Officer's notification bell via a trigger-based subscriber.

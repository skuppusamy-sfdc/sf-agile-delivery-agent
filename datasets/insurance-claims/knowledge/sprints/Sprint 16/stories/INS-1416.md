## Description

As a Provider Network Specialist, I need a report and dashboard component showing all Provider_Network_Agreement__c records expiring within 30, 60, and 90 days so that I can proactively initiate renewal workflows and prevent gaps in network coverage that would disrupt claims adjudication.

The report must also flag PNAs where the associated Fee_Schedule__c has a Fee_Schedule__c.Effective_End_Date__c that precedes the PNA__c.Contract_End_Date__c, indicating a fee schedule gap risk.

## Acceptance Criteria

Given a Provider Network Specialist accesses the 'Network Contract Expiration' report,
When the report loads,
Then it displays Provider_Network_Agreement__c records grouped into three bands:
  - Expiring in 0-30 days (PNA__c.Contract_End_Date__c <= TODAY()+30)
  - Expiring in 31-60 days
  - Expiring in 61-90 days
With columns: PNA__c.Name, Provider__c (lookup to Person Account), PNA__c.Contract_Status__c, PNA__c.LOB__c, PNA__c.Contract_End_Date__c, Fee_Schedule__c.Effective_End_Date__c.

Given a PNA record is displayed in the report,
When Fee_Schedule__c.Effective_End_Date__c < PNA__c.Contract_End_Date__c,
Then a 'Fee Schedule Gap Risk' flag column displays 'Yes' in red text for that row.

Given the report is embedded in the Provider Network dashboard,
When a user clicks a PNA record row,
Then the user is navigated to the Provider_Network_Agreement__c record detail page.

Given a PNA__c.Contract_Status__c = 'Terminated' or 'Expired',
When the report runs,
Then those records are excluded from all three expiration bands.

Given the report runs,
When results are exported,
Then the export includes all visible columns in .csv format and respects the user's field-level security.

## Solution

Create a custom Salesforce Report Type joining Provider_Network_Agreement__c as the primary object with Fee_Schedule__c as a related object (lookup relationship). Build a standard Salesforce tabular report with grouping rows by expiration band using bucket fields on PNA__c.Contract_End_Date__c. Fee Schedule Gap Risk flag implemented as a Cross-Filter or formula field PNA__c.Fee_Gap_Risk__c (Formula, Checkbox) deployed via a before-save Record-Triggered Flow that evaluates Fee_Schedule__c.Effective_End_Date__c < PNA__c.Contract_End_Date__c. Report added as a dashboard component (table widget) on the Provider Network Analytics dashboard. Schedule report to run and email to Provider Network Specialist profile weekly on Mondays at 7:00 AM.

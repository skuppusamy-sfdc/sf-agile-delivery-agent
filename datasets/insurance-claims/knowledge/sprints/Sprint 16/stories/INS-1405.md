## Description

As a Claims Operations Manager, I need an Agentforce-powered natural language query interface embedded in the Claims Operations dashboard so that I can ask plain-English questions about claim status, payment trends, and SIU flags without needing to manually build or navigate reports.

Example queries the interface must support:
- 'How many claims were denied for COB issues last month?'
- 'Show me all open Claim__c records assigned to providers with expired PNAs'
- 'What is the average adjudication time for ICD-10 code Z79.4 claims this quarter?'
- 'List claims flagged by SIU in the last 7 days'

The interface must respect the user's Salesforce data sharing rules and Permission Set Groups — users should never see Claim__c records outside their sharing access.

## Acceptance Criteria

Given the Agentforce 'Claims Query Agent' is configured and the user has the 'Claims_AI_User' Permission Set,
When the user types a natural language query into the Agentforce chat panel on the Claims Operations dashboard,
Then the agent parses the query, maps it to relevant Salesforce objects (Claim__c, Claim_Line__c, Payment__c, Provider_Network_Agreement__c), and returns a structured result set within 10 seconds.

Given the user asks 'How many claims were denied for COB issues last month?',
When the agent processes the query,
Then it returns a count of Claim__c records where Claim__c.Status__c = 'Denied' AND Claim__c.Denial_Reason__c = 'COB' AND Claim__c.Closed_Date__c falls within the prior calendar month.

Given the user asks about SIU-flagged claims,
When the agent responds,
Then it returns only Claim__c records where Claim__c.SIU_Flag__c = true AND Claim__c.SIU_Flag_Date__c >= LAST_N_DAYS:7, respecting the user's org-wide default sharing rules.

Given a user without SIU data access submits an SIU query,
When the agent processes the request,
Then it returns a permission-aware message: 'You do not have access to SIU-flagged claim data. Contact your administrator.' and logs the query attempt in Agentforce__Audit_Log__c.

Given the agent cannot confidently map a query to a Salesforce object,
When the confidence score falls below 0.75,
Then the agent responds with a clarifying question rather than returning potentially incorrect data.

-Given the agent supports voice input via browser microphone API, When the user speaks a query, Then it transcribes and processes it-

## Solution

Configure an Agentforce Agent ('Claims_Query_Agent') using the Agentforce Agent Builder with a custom Agent Topic scoped to Claims reporting. Define Agent Actions backed by Apex invocable methods that dynamically construct SOQL queries based on entity and filter parameters extracted by the Einstein LLM. Use Retrieval Augmented Generation (RAG) with a Data Cloud data lake object sourcing Claim__c, Claim_Line__c, and Payment__c. Enforce row-level security by executing all SOQL in user context (WITH USER_MODE) within the Apex action classes. Audit logging uses a Platform Event (AgentQueryAudit__e) consumed by a Record-Triggered Flow that writes to Agentforce__Audit_Log__c. The chat panel is an LWC ('claimsAgentforcePanel') embedded on the Lightning App Page via App Builder. Permission Set 'Claims_AI_User' gates access to the agent topic and LWC.

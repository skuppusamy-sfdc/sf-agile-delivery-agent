## Description

As a Credentialing Specialist, I need a Salesforce report and dashboard component that shows the average time each Credentialing_Case__c record spends in each pipeline stage so that I can identify process delays, reduce credentialing cycle times, and meet NCQA turnaround benchmarks (target: ≤45 days from application to approval).

The report must cover all cases opened in the current calendar year and break down time-in-stage for: 'Application Received', 'Primary Source Verification', 'Committee Review', 'Approved', and 'Denied'.

## Acceptance Criteria

Given a Credentialing Specialist with 'Credentialing_Reporting' Permission Set,
When they open the 'Credentialing Pipeline Analysis' report,
Then the report shows one row per Credentialing_Case__c.Stage__c value with columns for average days in stage, median days in stage, and count of cases.

Given Credentialing_Case__c.Open_Date__c falls within the current calendar year,
When the report filters are applied,
Then only cases where Credentialing_Case__c.Open_Date__c >= YYYY-01-01 are included.

Given the dashboard component for this report is added to the Credentialing Ops dashboard,
When any Credentialing_Case__c.Stage__c average exceeds 10 days,
Then that row is highlighted in amber on the dashboard chart.

-Given the report includes CAQH verification timestamps pulled via DataIntegration, When a case has CAQH data, Then CAQH_Verification_Date__c is shown as a separate column-

Given a case reaches Credentialing_Case__c.Stage__c = 'Approved' or 'Denied',
When total elapsed days from Credentialing_Case__c.Open_Date__c exceeds 45,
Then the case is flagged with Credentialing_Case__c.SLA_Breach__c = true and appears in a separate 'SLA Exceptions' report section.

## Solution

Create a custom Salesforce Report Type based on Credentialing_Case__c with a related Stage_History__c child object that logs stage entry/exit timestamps via a Record-Triggered Flow on Credentialing_Case__c.Stage__c field changes. The flow writes to a custom Stage_History__c object with fields: Stage_Name__c, Entry_Date__c, Exit_Date__c, and Duration_Days__c (formula). The report uses summary format grouped by Stage_Name__c. Dashboard component is a horizontal bar chart. SLA breach detection uses a separate scheduled Apex batch job (CredentialingSLABatch) running nightly to stamp Credentialing_Case__c.SLA_Breach__c. Permission Set 'Credentialing_Reporting' grants read access to Stage_History__c.

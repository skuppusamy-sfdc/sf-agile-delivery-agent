## Description

As a Credentialing Specialist, I need an aging report on Credentialing_Case__c records segmented by CAQH verification status and days-open buckets so that I can identify stalled credentialing workflows, ensure timely CAQH attestation follow-up, and meet the 60-day credentialing turnaround SLA required by MCO contracts.

## Acceptance Criteria

Given any Credentialing_Case__c where Credentialing_Case__c.Status__c NOT IN ('Approved', 'Denied', 'Withdrawn'),
When the 'Credentialing Aging Report' is viewed,
Then records are displayed with columns:
  - Credentialing_Case__c.Case_Number__c
  - Provider__c.Name
  - Provider__c.NPI__c
  - Credentialing_Case__c.CAQH_Profile_Status__c (values: 'Verified', 'Pending Attestation', 'Expired', 'Not Found')
  - Credentialing_Case__c.Days_Open__c (formula field)
  - Credentialing_Case__c.Assigned_Credentialing_Specialist__c
  - Credentialing_Case__c.LOB__c

Given the report is rendered,
When grouped by CAQH_Profile_Status__c,
Then a summary row count and average Days_Open__c is shown per group.

Given a Credentialing_Case__c where Days_Open__c >= 45,
When displayed in the report,
Then the row is conditionally highlighted in amber to indicate approaching SLA breach.

Given a Credentialing_Case__c where Days_Open__c >= 60,
When displayed in the report,
Then the row is conditionally highlighted in red and Credentialing_Case__c.SLA_Breach_Flag__c is set to TRUE via a scheduled Record-Triggered Flow running daily.

Given the report is filtered by 'My Cases',
When a Credentialing Specialist views it,
Then only records where Credentialing_Case__c.Assigned_Credentialing_Specialist__c = current user are returned.

-Given integration with NPPES was in scope for real-time NPI validation within this report-
-Then NPI validation status column would display NPPES lookup result inline-
(Deferred to DataIntegration epic per architecture review; NPPES integration tracked in INS-1377)

## Solution

Create a custom Report Type on Credentialing_Case__c with lookup to Provider__c (Person Account). Add formula field Days_Open__c = TODAY() - Credentialing_Case__c.CreatedDate on the object. Deploy SLA_Breach_Flag__c (Checkbox) on Credentialing_Case__c. Build a scheduled Record-Triggered Flow (CredentialingAgingSLAFlow) on Credentialing_Case__c firing on a scheduled path (daily at 07:00 AM) to evaluate Days_Open__c >= 60 and set SLA_Breach_Flag__c = TRUE. Report uses standard conditional highlighting via Salesforce report formatting rules. Deploy report to 'Credentialing Operations' dashboard folder with OWD-restricted sharing to the Credentialing_Specialist Permission Set.

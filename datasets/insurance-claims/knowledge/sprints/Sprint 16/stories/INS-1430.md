# INS-1430: Build Claims Operations Executive Dashboard with KPI tiles and drill-down capability

## Description

As a Claims Operations Manager, I need an executive-level CRM Analytics dashboard that surfaces real-time KPIs across all active Claim__c records so that I can monitor adjudication throughput, average cycle time, and pending payment volumes without exporting data to spreadsheets. The dashboard must support LOB-level filtering (Medical, Dental, Vision, Workers Comp) and allow drill-down from summary tiles into underlying Claim__c and Claim_Line__c records.

## Acceptance Criteria

Given the Claims Operations Manager navigates to the 'Claims Ops Executive Dashboard' CRM Analytics app,
When the dashboard loads,
Then KPI tiles must display: total open Claim__c records where Claim__c.Status__c IN ('Submitted','In Review','Pending Info'), average Claim__c.Cycle_Time_Days__c for claims closed in the current month, total Payment__c.Amount__c sum for disbursements in the current calendar month, and SIU referral rate as a percentage of total claims.

Given the LOB filter widget is present on the dashboard,
When the Claims Operations Manager selects LOB = 'Workers Comp',
Then all KPI tiles, bar charts, and trend lines must filter to Claim__c records where Claim__c.Line_of_Business__c = 'Workers Comp' within 2 seconds of filter selection.

Given a KPI tile showing pending payment volume is displayed,
When the manager clicks the tile,
Then a drill-down panel must open listing individual Payment__c records where Payment__c.Status__c = 'Pending' with columns: Payment__c.Name, Payment__c.Amount__c, Payment__c.Provider__c (lookup to Provider__c), Payment__c.Due_Date__c, and Claim__c.Claim_Number__c.

Given the dashboard trend line widget for adjudication cycle time,
When the date range is set to 'Last 90 Days',
Then the line chart must plot average Claim__c.Cycle_Time_Days__c grouped by ISO week, with a reference line at the SLA threshold of 30 days defined in a dashboard variable.

-Given a 'Download to PDF' button is present on the dashboard header, When clicked, Then a formatted PDF report must be generated via Conga CLM and emailed to the manager's Salesforce User email address.- (Requirement superseded by INS-1435 which handles scheduled report distribution.)

Given the dashboard is accessed by a user without the 'Claims_Executive_Viewer' Permission Set Group,
When navigation to the dashboard app is attempted,
Then the user must receive a 'Access Restricted' error page and the dashboard dataset must not be queryable via SOQL by that user profile.

## Solution

Create a CRM Analytics (Tableau CRM) dashboard app 'Claims Ops Executive Dashboard' with SAQL-backed datasets sourced from Claim__c, Claim_Line__c, and Payment__c via a daily scheduled dataflow plus a live Direct Data connection for near-real-time tile refresh. KPI tiles built as Compare Table widgets. LOB filter implemented as a Global Filter widget bound to Claim__c.Line_of_Business__c picklist values. Drill-down panels use Faceting between widgets. Reference line on cycle time chart set via a dashboard variable 'SLA_Threshold_Days' defaulting to 30. Row-level security enforced via the CRM Analytics dataset security predicate tied to the 'Claims_Executive_Viewer' Permission Set Group. Deprecate PDF button requirement per INS-1435.

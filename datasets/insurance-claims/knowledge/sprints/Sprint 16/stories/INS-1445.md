## Description

As a Provider Network Specialist, I need a geographic heat map and gap analysis report showing in-network provider density by specialty and ZIP code so that I can identify underserved regions and prioritize PNA outreach campaigns to close access gaps before the next MCO audit cycle. Extends INS-1414 (Provider 360 FlexCard implementation) by consuming the same Provider__c geolocation fields.

## Acceptance Criteria

Given the Provider Network Specialist opens the 'Network Coverage Analytics' tab,
When the heat map renders,
Then it must plot active Provider__c records (Provider__c.Network_Status__c = 'Active') as density layers on a U.S. map, color-coded by Provider__c.Primary_Specialty__c, using Provider__c.Practice_ZIP__c for geolocation lookup.

Given the specialist selects a specialty filter (e.g., 'Cardiology', 'Primary Care', 'Mental Health'),
When the filter is applied,
Then the heat map must refresh to show only providers matching Provider__c.Primary_Specialty__c = [selected value] and the summary panel must update: Total In-Network Providers, Total Active PNAs (Provider_Network_Agreement__c.Status__c = 'Active'), and Average PNA Reimbursement Rate from Fee_Schedule__c.Base_Rate__c.

Given the gap analysis grid is displayed,
When data loads,
Then ZIP codes with fewer than the configurable threshold (default: 2) active providers of a given specialty must be flagged with a 'Coverage Gap' badge, and the grid must expose columns: ZIP Code, State, Specialty, Active Provider Count, Nearest In-Network Provider (miles), Gap Severity (Low/Medium/High).

Given a Coverage Gap row is selected,
When the specialist clicks 'Initiate Outreach',
Then a new Provider_Network_Agreement__c record must be created in Draft status with PNA.Coverage_Region__c pre-populated from the selected ZIP, and a Task must be created assigned to the current user with Subject = 'PNA Outreach – [ZIP] [Specialty]'.

Given the report is generated,
When the specialist exports,
Then a CSV export must include all gap rows with fields: ZIP, State, Specialty, Gap_Severity, Nearest_NPI, and Estimated_Population_Impacted__c sourced from a custom ZIP_Demographics__c object.

-Given the map renders MCO-specific network adequacy thresholds as overlay polygons (deprecated — MCO overlay deferred to Phase 3)-

## Solution

Build an OmniStudio FlexCard ('NetworkCoverageMapCard') embedding a Leaflet.js map via a custom LWC child component. A DataRaptor Extract retrieves active Provider__c records with geolocation data; a custom Apex service (ProviderGeoService.cls) performs ZIP-to-lat/long translation using a pre-loaded ZIP_Geocode__c custom metadata table. Gap analysis logic implemented in a CRM Analytics dashboard recipe joining Provider__c and ZIP_Demographics__c datasets. The 'Initiate Outreach' action triggers a Screen Flow (Provider_Outreach_Initiation_Flow) that creates PNA and Task records via Apex-invocable method. Scheduled Apex (ProviderNetworkGapBatchJob.cls) re-calculates gap flags nightly and stamps Provider__c.Coverage_Gap_Flag__c for fast filtering.

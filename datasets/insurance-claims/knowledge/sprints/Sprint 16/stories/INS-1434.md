## Description

As a Provider Relations Specialist, I need the Provider 360 FlexCard on the Provider__c (Person Account) record page to include an embedded analytics tile that surfaces the provider's current credentialing status from Credentialing_Case__c, count of active Provider_Network_Agreement__c records, and a mini trend chart of Claim__c volumes processed for that provider in the last 6 months so that I can have a comprehensive view without navigating away from the provider record. Extends INS-1400 (Provider 360 FlexCard foundation).

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then an 'Analytics Snapshot' section must render within the FlexCard displaying: most recent Credentialing_Case__c.Stage__c and Credentialing_Case__c.Expiration_Date__c for the provider, count of Provider_Network_Agreement__c records where PNA.Status__c = 'Active' and PNA.Provider__c = current Provider__c.Id, and a 6-month bar sparkline of Claim__c records where Claim__c.Billing_Provider__c = current Provider__c.Id grouped by calendar month.

Given the credentialing status displayed in the Analytics Snapshot section,
When Credentialing_Case__c.Stage__c = 'Pending Additional Info' or Credentialing_Case__c.Expiration_Date__c is within 60 days of today,
Then the credentialing status badge must render in amber with the label 'Action Required'.

Given the active PNA count tile,
When the specialist clicks the count hyperlink,
Then a modal must open displaying a list of active Provider_Network_Agreement__c records with columns: PNA.Name, PNA.Contract_Type__c, PNA.Effective_Date__c, PNA.Expiration_Date__c, and PNA.Fee_Schedule__c.

Given the 6-month claim volume sparkline,
When rendered,
Then each bar must represent the count of Claim__c.Id grouped by month, and hovering over a bar must display a tooltip with the month label and exact claim count.

-Given the Analytics Snapshot section includes a 'CAQH Score' metric sourced from a real-time CAQH API callout, When the FlexCard loads, Then the CAQH score must display within 3 seconds.- (Descoped from this story; CAQH real-time integration tracked in INS-1436.)

Given a user without the 'Provider_Relations_Full_Access' Permission Set Group,
When they view the Provider 360 FlexCard,
Then the Analytics Snapshot section must be hidden and replaced with a 'Contact your administrator for full access' message.

## Solution

Extend the existing 'Provider_360_FlexCard' OmniStudio FlexCard (built in INS-1400) by adding a new 'Analytics Snapshot' child card element. DataRaptor Extract actions fetch: (1) most recent Credentialing_Case__c via SOQL ordered by CreatedDate DESC limit 1 filtered by Provider__c lookup; (2) COUNT of active PNA records; (3) 6-month Claim__c monthly aggregation via a DataRaptor Transform with date bucketing. Sparkline rendered using a FlexCard Chart element (Bar type). Amber badge logic implemented via FlexCard conditional styling on a calculated field comparing Stage__c and date diff. PNA modal uses a FlexCard Action of type 'Show Child Card' with a DataRaptor-backed list. Visibility of Analytics Snapshot section controlled by a FlexCard Condition referencing the 'Provider_Relations_Full_Access' PSG membership checked via a custom permission.

## Description

As a Claims Operations Manager, I need a dashboard that visualizes Explanation_of_Benefits__c (EOB) disbursement trends broken down by Payment__c.Payment_Method__c (check, EFT, virtual card) and payer so that I can track ERA reconciliation rates, monitor outstanding balances, and report on payment cycle performance to finance leadership.

The dashboard must cover rolling 12-month data and allow filtering by Claim__c.Line_of_Business__c and Payment__c.Payer__c.

## Acceptance Criteria

Given a Claims Operations Manager views the 'Payment Disbursement' dashboard,
When the page loads,
Then a line chart displays monthly Payment__c.Total_Paid_Amount__c aggregated by Payment__c.Payment_Method__c for the trailing 12 months.

Given the LOB filter is applied,
When the user selects a value from the Claim__c.Line_of_Business__c picklist filter,
Then all dashboard components re-render to show only data for Claim__c records matching that LOB value.

Given Payment__c records with Payment__c.ERA_Reconciled__c = false,
When the 'Unreconciled ERA' KPI tile renders,
Then it shows the count and total Payment__c.Total_Paid_Amount__c of unreconciled payments, with a red indicator if count > 50.

Given Explanation_of_Benefits__c records linked to Payment__c,
When the 'EOB Issued vs Payment Sent' chart renders,
Then it shows a dual-axis chart comparing count of EOB__c.Issue_Date__c events vs Payment__c.Payment_Date__c events per week.

Given the user clicks 'Export to Finance',
When the export action fires,
Then a Conga Composer template generates a formatted Excel report of Payment__c records filtered to the current dashboard state and emails it to the user's registered address within 2 minutes.

## Solution

CRM Analytics dashboard ('EOB_Payment_Disbursement') built on a joined dataset of Payment__c, Explanation_of_Benefits__c, and Claim__c synced via hourly DataSync. SAQL used for rolling 12-month window calculations. LOB and Payer filters implemented as dashboard facets that bind to all widgets. Unreconciled ERA tile uses a conditional formatting rule on count threshold. Dual-axis chart built with CRM Analytics XMD configuration. Conga Composer integration triggered via a Dashboard Action button calling a named Conga template ('Finance_Payment_Report_v2') through a REST API callout from an Apex action. The dataset refresh schedule is set to every 60 minutes during business hours.

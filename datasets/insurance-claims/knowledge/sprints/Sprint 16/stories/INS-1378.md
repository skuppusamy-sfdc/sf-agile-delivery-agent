## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard embedded on the Provider__c (Person Account) record page that surfaces credentialing status, all active Provider_Network_Agreements, and a count of open Claim__c records associated with the provider so that I can assess provider standing without navigating across multiple objects.

This story extends INS-1359 which delivered the base Provider Account layout.

## Acceptance Criteria

Given the Provider 360 FlexCard is embedded on the Provider__c record page,
When a Provider Relations Specialist opens any Provider__c record,
Then the FlexCard renders within 2 seconds and displays: Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, NPI__c, TIN__c, and Board_Certification__c.

Given the PNA section of the FlexCard,
When rendered,
Then it lists all Provider_Network_Agreement__c records linked to the provider where Status__c IN ('Active', 'Pending Renewal'), showing Contract_Number__c, Network__c, and Effective_End_Date__c.

Given the Open Claims section,
When rendered,
Then it displays a count badge showing COUNT of Claim__c records where Provider__c = current record Id AND Claim__c.Status__c NOT IN ('Adjudicated', 'Denied', 'Closed'), with a 'View All' navigation action.

Given the credentialing status is 'Expired' or 'Suspended',
When the FlexCard renders,
Then the Credentialing Status field is highlighted with a red warning badge and an inline message reads 'Provider credentialing requires immediate review'.

Given a user without the 'Provider_Network_ReadOnly' Permission Set,
When the record page loads,
Then the FlexCard displays a masked placeholder and logs a Platform Event (Provider_Access_Violation__e) for compliance audit.

## Solution

Build OmniStudio FlexCard 'Provider360_FC' with three child DataRaptor Extract actions: DR_GetProviderCredentials (querying Credentialing_Case__c), DR_GetActivePNAs (querying Provider_Network_Agreement__c with SOQL filter on Status__c), and DR_GetOpenClaimsCount (aggregate SOQL COUNT on Claim__c). Conditional styling on credentialing badge using FlexCard State configuration with condition: Credentialing_Case__c.Status__c IN ['Expired','Suspended'] triggers 'warning' state CSS. Platform Event Provider_Access_Violation__e published via Apex invocable called from FlexCard action on permission failure. FlexCard embedded via Lightning App Builder as a Tab in the Provider__c record page layout, visible only to Permission Set Group 'Provider_Network_Specialists_PSG'.

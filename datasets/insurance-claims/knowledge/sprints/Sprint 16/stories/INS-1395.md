## Description

**Bug Report**

**Summary:** The 'Days in Current Status' KPI tile on the Credentialing Operations Dashboard displays an incorrect (inflated) value for Credentialing_Case__c records that have been re-opened after a prior 'Closed' status, causing Credentialing Specialists to misread case aging.

**Steps to Reproduce:**
1. Create a new Credentialing_Case__c record with Status__c = 'In Review'.
2. Advance the case to Status__c = 'Closed – Approved' and save. Note the closure date.
3. Re-open the case by setting Status__c = 'Re-Opened' and save.
4. Navigate to the Credentialing Operations Dashboard.
5. Observe the 'Days in Current Status' tile value for this case.

**Expected Result:** 'Days in Current Status' should reflect the number of days since the most recent Status__c change (i.e., since the 'Re-Opened' transition), sourced from Credentialing_Case__c.Status_Last_Changed_Date__c.

**Actual Result:** The tile displays the total number of days since the original Credentialing_Case__c.CreatedDate, ignoring the re-open event. For a case created 45 days ago and re-opened today, the tile shows '45 days' instead of '0 days'.

**Root Cause (Suspected):** The SAQL query in the CRM Analytics dashboard widget uses CreatedDate instead of Status_Last_Changed_Date__c as the basis for the days calculation.

**Environment:** Acme Insurance FULL sandbox
**Affected Users:** All Credentialing Specialists and Credentialing Supervisors
**Frequency:** 100% reproducible

## Acceptance Criteria

As per Description. Additionally:
- After fix, 'Days in Current Status' for a re-opened case must equal TODAY() - Credentialing_Case__c.Status_Last_Changed_Date__c
- Verify Status_Last_Changed_Date__c is updated by the Record-Triggered Flow on every Status__c transition (including Re-Opened)
- Regression test: newly created cases (never closed) must still show correct values post-fix

## Solution

Update the SAQL expression in the CRM Analytics 'Days in Current Status' widget to reference Status_Last_Changed_Date__c instead of CreatedDate: `datediff(toDate("today"), date('Status_Last_Changed_Date__c', "yyyy-MM-dd"))`. Verify the Record-Triggered Flow (CredentialingCase_StatusChangeFlow) correctly stamps Status_Last_Changed_Date__c = NOW() on every Status__c field change, including the 'Re-Opened' transition. Update the Analytics dataflow to include Status_Last_Changed_Date__c in the Credentialing_Case__c dataset extract. Deploy and validate in FULL sandbox before promoting to production.

## Description

**Bug Report: PCN Status Miscounting in Network Termination Report**

**Steps to Reproduce:**
1. Navigate to Reports > Provider Network > 'Network Termination Status Summary' report.
2. Note the count displayed in the 'PCN Sent' column.
3. Navigate to Provider_Change_Notification__c list view filtered by PCN_Status__c = 'Draft'.
4. Note that several Draft records are present (confirmed 14 records in UAT org as of 2024-11-18).
5. Compare: the report 'PCN Sent' count matches the total of BOTH 'Sent' AND 'Draft' PCN records combined.

**Expected Behavior:**
The 'PCN Sent' column in the Network Termination Status Summary report should count only Provider_Change_Notification__c records where PCN_Status__c = 'Sent'.

**Actual Behavior:**
The report counts Provider_Change_Notification__c records where PCN_Status__c IN ('Sent', 'Draft'), inflating the 'PCN Sent' metric and misrepresenting regulatory compliance posture to the Compliance Officer.

**Environment:** UAT Sandbox (CS92)
**Severity:** High — affects compliance reporting submitted to state regulators.
**Related:** Network_Termination_Case__c records linked to the miscounted PCNs are also showing incorrect 'Notifications Sent' rollup values on the parent case.

## Acceptance Criteria

As per Description.

Additionally:
Given the corrected report filter is deployed,
When the 'Network Termination Status Summary' report is run,
Then the 'PCN Sent' count matches exactly the count of Provider_Change_Notification__c records with PCN_Status__c = 'Sent' only, verified by spot-check against list view.

Given Network_Termination_Case__c has a rollup summary field 'Notifications_Sent_Count__c',
When the fix is applied,
Then Notifications_Sent_Count__c reflects only child PCN records with PCN_Status__c = 'Sent'.

## Solution

Root cause: the custom report type filter on Provider_Change_Notification__c was using a filter logic condition 'PCN_Status__c != NULL' instead of 'PCN_Status__c = Sent'. Fix: update the saved report filter to PCN_Status__c = 'Sent'. Additionally, audit and correct the Rollup Summary Field 'Notifications_Sent_Count__c' on Network_Termination_Case__c — the filter criteria on the rollup must be updated from no filter to PCN_Status__c = 'Sent'. Deploy report and rollup field change via changeset to UAT for regression testing before promoting to PROD. Add a report validation test to the regression suite.

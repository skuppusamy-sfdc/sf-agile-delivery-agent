## Description

As a Claims Operations Manager, I need the system to automatically look up the applicable contracted rate from the Fee_Schedule__c object when a Claim_Line__c is submitted with a valid CPT code, so that adjudication applies the correct in-network reimbursement amount without manual intervention.

This story extends the foundational Fee_Schedule__c object build from INS-0341 and wires it into the claims adjudication pipeline.

## Acceptance Criteria

Given a Claim_Line__c record is created or updated with a CPT_Code__c value and the parent Claim__c.Network_Status__c = 'In-Network',
When the claim line is saved,
Then the system queries Fee_Schedule__c records filtered by LOB__c, Effective_Date__c <= Claim__c.Date_of_Service__c, and Expiration_Date__c >= Claim__c.Date_of_Service__c and populates Claim_Line__c.Contracted_Rate__c with the matching Fee_Schedule__c.Allowed_Amount__c.

Given a matching Fee_Schedule__c record exists for the CPT code and the Provider_Network_Agreement__c.Status__c = 'Active',
When the lookup executes,
Then Claim_Line__c.Fee_Schedule_Matched__c is set to TRUE and Claim_Line__c.Fee_Schedule_Lookup_Source__c is populated with the Fee_Schedule__c record ID.

Given no matching Fee_Schedule__c record is found for the submitted CPT code,
When the lookup executes,
Then Claim_Line__c.Fee_Schedule_Matched__c is set to FALSE, Claim_Line__c.Contracted_Rate__c remains NULL, and a Task is created for the assigned Provider Network Specialist with Subject = 'Fee Schedule Gap – CPT Not Found'.

Given a Fee_Schedule__c record has Fee_Schedule__c.Rate_Type__c = 'Percentage of Medicare',
When the rate is applied,
Then Claim_Line__c.Contracted_Rate__c = CMS_Medicare_Rate__c * (Fee_Schedule__c.Medicare_Percentage__c / 100) rounded to 2 decimal places.

-Given multiple Fee_Schedule__c records match the same CPT, the first created record is used- (superseded: priority ranking field Fee_Schedule__c.Priority_Rank__c now governs tie-breaking as of Sprint 4 grooming).

Given the lookup completes successfully,
When Claim__c.Status__c transitions to 'Pending Adjudication',
Then all child Claim_Line__c records must have Fee_Schedule_Matched__c = TRUE or a supervisor override flag Claim_Line__c.Manual_Rate_Override__c = TRUE before the transition is permitted.

## Solution

Implement an Apex invocable method `FeeScheduleLookupService.resolveContractedRate()` called from a Record-Triggered Flow on Claim_Line__c (trigger: after insert/update on CPT_Code__c or Network_Status__c change). The service performs a SOQL query on Fee_Schedule__c with composite key matching on CPT_Code__c, LOB__c, and date range. A custom metadata type `Fee_Schedule_Config__mdt` stores percentage-of-Medicare multiplier rules per LOB to avoid hardcoding. For the 'Percentage of Medicare' rate type, a separate callout to the CMS integration (via named credential `CMS_MPFS`) retrieves the current Medicare Physician Fee Schedule rate. Results are written back via a DML update within the same transaction. A Platform Event `Fee_Schedule_Gap__e` is fired when no match is found, consumed by a downstream Flow that creates the gap Task and notifies the Provider Network Specialist via in-app notification.

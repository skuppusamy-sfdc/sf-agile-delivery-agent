## Description

As a Credentialing Specialist, I need the system to automatically retrieve and populate provider credentialing data from the CAQH ProView API when a Credentialing_Case__c is opened so that I can reduce manual data entry time, eliminate transcription errors, and accelerate the provider onboarding timeline from an average of 14 days to under 7 days.

## Acceptance Criteria

Given a Credentialing_Case__c is created with Case_Type__c = 'Initial Credentialing' or 'Re-Credentialing' and the linked Provider__c record has Provider__c.CAQH_Provider_ID__c populated,
When the record is saved,
Then an asynchronous Apex Queueable job is enqueued to call the CAQH ProView REST API endpoint '/api/providers/{caqhProviderId}/credentialingStatus'.

Given the CAQH API returns a successful response (HTTP 200),
When the response payload is parsed,
Then the following Credentialing_Case__c fields are auto-populated: Board_Certification__c, Medical_License_Number__c, License_State__c, License_Expiration_Date__c, DEA_Number__c, DEA_Expiration_Date__c, Malpractice_Coverage_Amount__c, Last_CAQH_Attestation_Date__c.

Given the CAQH API returns HTTP 404 (provider not found),
When the error is handled,
Then Credentialing_Case__c.CAQH_Sync_Status__c is set to 'Provider Not Found in CAQH', a Task is created for the Credentialing Specialist with Subject = 'Manual CAQH Data Entry Required – Provider Not Found', and no fields are overwritten.

Given the CAQH API returns any non-200/404 response or times out,
When the error is handled,
Then Credentialing_Case__c.CAQH_Sync_Status__c is set to 'Sync Failed – Retry Pending', the error details are logged to a custom CAQH_Integration_Log__c record, and the Queueable job retries up to 3 times with exponential backoff.

Given the CAQH data is successfully populated,
When the Credentialing Specialist reviews the case,
Then Credentialing_Case__c.CAQH_Sync_Status__c = 'Synced', Credentialing_Case__c.CAQH_Last_Sync_Datetime__c is stamped with the current timestamp, and all auto-populated fields are marked read-only via field-level security until the specialist explicitly clicks 'Override CAQH Data'.

Given a provider's CAQH attestation is older than 365 days (Last_CAQH_Attestation_Date__c < TODAY() - 365),
When the sync completes,
Then a warning is displayed on the Credentialing_Case__c record: 'CAQH attestation is outdated. Provider re-attestation may be required before credentialing approval.'

-Given CAQH sync runs on every save of Credentialing_Case__c regardless of case type.- (Deprecated: sync only triggers on Case_Type__c = 'Initial Credentialing' or 'Re-Credentialing' to avoid unnecessary API calls per architecture review 2025-01-10)

## Solution

Named Credential ('CAQH_ProView_NC') configured with CAQH API base URL and OAuth 2.0 client credentials. Apex Queueable class ('CAQHProviderSyncJob') performs HTTP callout using HttpRequest/HttpResponse, parses JSON response with a dedicated CAQHResponseWrapper inner class, and performs DML update on Credentialing_Case__c. Record-Triggered Flow (After Insert on Credentialing_Case__c, filter: Case_Type__c IN ('Initial Credentialing', 'Re-Credentialing') AND Provider__c.CAQH_Provider_ID__c != null) enqueues the Queueable job via an Apex Action. Retry logic implemented with a CAQH_Retry_Count__c field on Credentialing_Case__c, incremented on each failed attempt. Custom Metadata Type (CAQH_Config__mdt) stores retry limits and timeout thresholds. Read-only field enforcement post-sync handled via a Before Save Flow checking CAQH_Sync_Status__c. Permission Set 'CAQH_Integration_Admin' grants access to integration logs.

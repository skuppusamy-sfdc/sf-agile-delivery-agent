## Description

As a Provider Network Specialist, I need to initiate and manage Provider_Network_Agreement__c records directly from a Provider__c (Person Account) or Facility__c (Business Account) record, with automated contract generation via Conga CLM, so that contracting turnaround time is reduced and all executed agreements are stored and linked in Salesforce without manual document handling.

## Acceptance Criteria

Given a Provider__c or Facility__c record with Status__c = 'Credentialed' and no active PNA exists,
When the Provider Network Specialist clicks 'Initiate Contract' on the FlexCard action menu,
Then a new Provider_Network_Agreement__c record is created with PNA.Status__c = 'Draft', PNA.Effective_Date__c = NULL, and PNA.Provider__c or PNA.Facility__c populated accordingly.

Given a Provider_Network_Agreement__c record in 'Draft' status,
When the user clicks 'Generate Contract Document',
Then an OmniScript triggers the Conga CLM API, merges provider NPI, TIN, network tier, and LOB fields into the contract template, and attaches the resulting PDF to the PNA record's Files section.

Given the Conga CLM document has been sent for e-signature via DocuSign,
When all parties have completed signing,
Then a Platform Event (Contract_Signed__e) is published, a Record-Triggered Flow updates PNA.Status__c = 'Active', PNA.Executed_Date__c = TODAY(), and a notification is sent to the assigned Provider Relations Specialist.

Given PNA.Status__c transitions to 'Active',
When the record is saved,
Then PNA.Network_Tier__c must be a valid picklist value ('Tier 1', 'Tier 2', 'Tier 3') and PNA.Termination_Date__c must be >= PNA.Effective_Date__c + 365 days, otherwise a validation error is thrown.

Given a PNA record is set to 'Active',
When viewed from the Provider 360 FlexCard,
Then the active PNA summary (network tier, LOB, effective date, termination date) is displayed in a dedicated 'Contracting' tab with a link to the executed document.

-Given a PNA is in 'Pending Signature' status, the system shall auto-void after 30 calendar days- (superseded: void logic moved to INS-0385 automated expiry story)

## Solution

OmniScript 'Initiate_PNA_Contract' drives the user through provider selection, LOB assignment, and network tier selection. A DataRaptor Transform maps provider/facility fields to the Conga CLM merge template via a REST callout in an Apex invocable action (CongaCLMService.generateContract()). DocuSign envelope status webhook fires a Platform Event (Contract_Signed__e) consumed by a Record-Triggered Flow on Provider_Network_Agreement__c to set status to 'Active'. Validation Rules on Provider_Network_Agreement__c enforce date and tier constraints. FlexCard 'Provider_360_Contracting' tab surfaces PNA data via a DataRaptor Extract. Permission Set Group 'Provider_Network_Contracting_PSG' controls access.

## Description

**Bug Report**

**Summary:** When a Provider Network Specialist clones an existing Provider_Network_Agreement__c record using the standard Salesforce 'Clone' button, the cloned PNA record is created but all associated PNA_Fee_Schedule__c junction records are NOT copied to the new PNA. The provider ends up with an active PNA that has no fee schedule associations, causing downstream Claim_Line__c adjudication failures (Fee_Schedule_Match_Status__c = 'Unmatched').

**Steps to Reproduce:**
1. Navigate to any Provider_Network_Agreement__c record with at least one PNA_Fee_Schedule__c child record.
2. Click the standard 'Clone' button on the record detail page.
3. Update Contract_Start_Date__c and Contract_End_Date__c on the clone modal.
4. Save the cloned PNA record.
5. Navigate to the cloned PNA's related list 'Fee Schedule Associations'.

**Expected Result:** The cloned PNA_Fee_Schedule__c junction records are created referencing the new PNA Id, copying Fee_Schedule__c lookups from the source PNA.

**Actual Result:** The 'Fee Schedule Associations' related list on the cloned PNA is empty. No PNA_Fee_Schedule__c records are created.

**Environment:** Acme Insurance UAT sandbox (CS112)
**Frequency:** 100% reproducible
**Impact:** HIGH — affects contract renewal workflow; two providers in UAT have been onboarded without fee schedules linked.

as per Description

## Solution

Implement a Record-Triggered Flow (PNA_Clone_Fee_Schedule_Flow) or Apex Trigger on Provider_Network_Agreement__c after insert that detects when a new PNA is created via clone (by checking if ClonedFromId is populated). If ClonedFromId is not null, query all PNA_Fee_Schedule__c records where Provider_Network_Agreement__c = ClonedFromId and create corresponding PNA_Fee_Schedule__c records for the new PNA Id. Add a custom 'Clone PNA' quick action (Screen Flow) to replace the standard Clone button on the PNA record page, which explicitly handles junction record duplication and warns the user if any cloned Fee_Schedule__c records are expired.

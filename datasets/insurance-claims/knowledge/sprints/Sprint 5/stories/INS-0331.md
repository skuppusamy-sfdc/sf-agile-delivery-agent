## Description

As a Provider Network Specialist, I need the Provider_Network_Agreement__c record to automatically transition to 'Active' status when the DocuSign envelope is countersigned by both parties, so that manual status updates are eliminated and the contract activation timestamp is audit-accurate. This story supersedes the nightly batch approach deprecated in INS-0327.

## Acceptance Criteria

Given a DocuSign envelope associated with a Provider_Network_Agreement__c reaches 'Completed' status (all signers have signed),
When the DocuSign Connect webhook fires a POST to the Salesforce Apex REST endpoint '/services/apexrest/docusign/pna-callback',
Then the endpoint parses the EnvelopeId from the payload and queries Provider_Network_Agreement__c where DocuSign_Envelope_ID__c = [EnvelopeId].

Given the matching PNA record is found and Provider_Network_Agreement__c.Status__c = 'Pending Signature',
When the webhook handler processes the event,
Then Provider_Network_Agreement__c.Status__c is updated to 'Active', Provider_Network_Agreement__c.Countersign_Date__c = [timestamp from DocuSign payload], and Provider_Network_Agreement__c.Signed_Document_URL__c = [document download URL from payload].

Given the webhook fires but no matching PNA is found,
When the handler cannot resolve the EnvelopeId,
Then a Platform Event (DocuSign_Error__e) is published with fields: Envelope_ID__c, Error_Message__c = 'PNA record not found', Timestamp__c, and an alert email is sent to the Provider Network Ops DL.

Given the PNA transitions to 'Active',
When the Record-Triggered Flow (PNA_Activation_Flow) executes,
Then a Provider_Change_Notification__c (PCN) record is created with PCN.Type__c = 'Network Addition', PCN.Provider__c = PNA.Provider__c, PCN.Effective_Date__c = PNA.Contract_Start_Date__c.

## Solution

Apex REST class (DocuSignPNACallbackController) with @RestResource annotation handles inbound POST from DocuSign Connect. Payload parsed via JSON.deserializeUntyped. Record-Triggered Flow (PNA_Activation_Flow) on Provider_Network_Agreement__c fires on Status__c change to 'Active' to create the PCN. Platform Event (DocuSign_Error__e) published via EventBus.publish() for error scenarios. Connected App credentials for DocuSign Connect configured in Named Credential 'DocuSign_Connect_NC'. Endpoint secured with IP allowlist for DocuSign IP ranges.

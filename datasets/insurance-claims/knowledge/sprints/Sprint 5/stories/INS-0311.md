# INS-0311: Build Fee Schedule Engine to calculate allowed amounts on Claim Lines based on contracted rates

## Description

As a Claims Operations Manager, I need the system to automatically calculate allowed amounts on each Claim_Line__c record by matching the submitted CPT code against the applicable Fee_Schedule__c tied to the provider's active Provider_Network_Agreement__c, so that adjudicators do not manually look up contracted rates and payment accuracy is improved across all LOBs.

## Acceptance Criteria

Given a Claim_Line__c is created with a valid CPT__c code and the parent Claim__c has a linked Provider__c,
When the Claim__c.Status__c transitions to 'In Adjudication',
Then the system must query the Fee_Schedule__c records associated with the Provider_Network_Agreement__c where PNA.Effective_Date__c <= today <= PNA.Expiration_Date__c and PNA.Network_Status__c = 'Active',
And set Claim_Line__c.Allowed_Amount__c = Fee_Schedule__c.Contracted_Rate__c for the matching CPT__c + ICD10_Code__c combination.

Given a matching Fee_Schedule__c record is found for the CPT code,
When the Fee_Schedule__c.Rate_Type__c = 'Percentage of Billed',
Then Claim_Line__c.Allowed_Amount__c = Claim_Line__c.Billed_Amount__c * (Fee_Schedule__c.Contracted_Rate__c / 100),
And Claim_Line__c.Rate_Basis__c must be stamped with 'Percentage of Billed'.

Given no active Fee_Schedule__c record matches the submitted CPT code,
When the lookup returns zero results,
Then Claim_Line__c.Pricing_Exception__c must be set to TRUE,
And a Task must be created and assigned to the Claims Operations queue for manual rate review,
And Claim__c.Status__c must remain 'Pending - Rate Review' and must NOT advance to 'Adjudicated'.

Given a provider is associated with a Facility__c (Business Account) rather than a Person Account,
When the fee schedule lookup runs,
Then the system must resolve the Fee_Schedule__c via Facility__c.Primary_PNA__c rather than Provider__c.Active_PNA__c.

-Given Fee_Schedule__c.Legacy_Rate_Flag__c = TRUE the system should apply a 5% uplift — deprecated per compliance review CR-2024-118-

Given the fee schedule calculation completes successfully,
When all Claim_Line__c records on a Claim__c have Allowed_Amount__c populated,
Then Claim__c.Total_Allowed_Amount__c must equal the SUM of all child Claim_Line__c.Allowed_Amount__c values,
And an audit log entry must be written to Claim_Audit__c with Action__c = 'Fee Schedule Applied', Timestamp__c, and Running_User__c.

## Solution

Implement an Apex invocable method 'FeeScheduleCalculationService' called from a Record-Triggered Flow on Claim__c when Status__c changes to 'In Adjudication'. The Apex class queries Fee_Schedule__c with a composite index on CPT__c + LOB__c + Effective_Date__c for performance. Use a custom metadata type (Fee_Schedule_Config__mdt) to store rate-type logic so business rules can be updated without code deployment. A separate Apex batch job (FeeScheduleRecalcBatch) will handle bulk re-pricing when Fee_Schedule__c records are updated. Platform Events (Fee_Schedule_Applied__e) will be fired post-calculation for downstream ERA generation. Unit tests must cover ≥85% coverage including no-match and facility-based scenarios.

## Description

As a Claims Operations Manager, I need the system to automatically look up the applicable reimbursement rate from a Fee_Schedule__c record and populate Claim_Line__c.Allowed_Amount__c during adjudication, so that payment calculations are accurate, consistent, and eliminate manual rate lookups by claims examiners. The engine must resolve the correct fee schedule by traversing: Claim → Provider → PNA → Fee_Schedule__c, then matching on CPT code and place of service.

## Acceptance Criteria

Given a Claim_Line__c record is created or updated with Claim_Line__c.CPT_Code__c populated and the parent Claim__c.Status__c = 'In Adjudication'
When the Fee Schedule Engine executes
Then the engine must identify the active PNA for the rendering provider (Claim__c.Rendering_Provider__c → Provider_Network_Agreement__c where PNA.Status__c = 'Active' and PNA.LOB__c matches Claim__c.Line_of_Business__c).

Given a valid active PNA is resolved
When the engine queries the linked Fee_Schedule__c
Then it must match Fee_Schedule_Line__c records where Fee_Schedule_Line__c.CPT_Code__c = Claim_Line__c.CPT_Code__c AND Fee_Schedule_Line__c.Place_of_Service__c = Claim_Line__c.Place_of_Service__c.

Given a matching Fee_Schedule_Line__c record is found
When the allowed amount is calculated
Then Claim_Line__c.Allowed_Amount__c must be set to Fee_Schedule_Line__c.Reimbursement_Rate__c and Claim_Line__c.Fee_Schedule_Source__c must be stamped with the Fee_Schedule__c.Name for audit purposes.

Given no matching CPT/place-of-service combination is found in the fee schedule
When the engine completes its lookup
Then Claim_Line__c.Allowed_Amount__c must remain null, Claim_Line__c.Adjudication_Flag__c must be set to 'Manual Review Required', and a Task must be created assigned to the Claims Operations queue.

Given a provider is out-of-network (no active PNA exists)
When the engine executes
Then it must apply the default out-of-network rate from a custom metadata type (Fee_Schedule_Config__mdt) where Config_Key__c = 'OON_Default_Rate_Multiplier' and stamp Claim_Line__c.Network_Status__c = 'Out-of-Network'.

Given the fee schedule lookup completes for all Claim_Line__c children
When all lines have Allowed_Amount__c populated
Then Claim__c.Total_Allowed_Amount__c must be updated via a roll-up to the sum of all Claim_Line__c.Allowed_Amount__c values.

## Solution

Implement an Apex invocable class (FeeScheduleResolutionService) called from a Record-Triggered Flow on Claim_Line__c (after insert/update, async). The service performs a 2-step SOQL chain: (1) query PNA via Claim rendering provider and LOB, (2) query Fee_Schedule_Line__c with CPT and place-of-service index. Use Custom Metadata Type (Fee_Schedule_Config__mdt) for OON fallback configuration. Bulk-safe implementation handles up to 200 claim lines per transaction. Roll-up of Total_Allowed_Amount__c on Claim__c implemented via standard Roll-Up Summary field. Unit test coverage target: 90%+.

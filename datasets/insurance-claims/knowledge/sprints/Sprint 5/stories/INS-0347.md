## Description

As a Provider Network Specialist, I need a Group Service Agreement (GSA) creation and management flow for Facility__c (Business Account) records so that multi-specialty hospital systems and ambulatory surgery centers can be contracted with tiered fee schedules across multiple specialties and LOBs in a single agreement, replacing the current practice of creating individual PNAs per physician.

This story builds the Facility contracting pathway referenced as superseded scope in INS-0343 and INS-0345.

## Acceptance Criteria

Given a Provider Network Specialist opens a Facility__c record where Account_Type__c = 'Hospital' or 'Ambulatory Surgery Center',
When they initiate the 'New Group Service Agreement' action,
Then a Group_Service_Agreement__c record is created with Facility__c lookup pre-populated, Status__c = 'Draft', and the OmniScript wizard launches at Step 1: Agreement Scope.

Given Step 2 of the GSA wizard (Rate Tier Configuration),
When the user adds a specialty tier (e.g., Specialty__c = 'Cardiology', Tier__c = 'Tier 1'),
Then a child GSA_Rate_Tier__c record is created linked to the GSA, and the user can associate a Fee_Schedule__c filtered by LOB__c and Specialty__c.

Given a GSA contains 3 or more specialty tiers,
When the user proceeds to Step 3 (Covered Providers),
Then a multi-select lookup allows adding Provider__c records (Person Accounts) affiliated with the Facility__c via the Facility_Provider_Affiliation__c junction object, and each added provider inherits the GSA's network tier unless individually overridden.

Given GSA_Status__c = 'Active',
When a Claim__c is submitted with Rendering_Provider__c matching a provider covered under the GSA and Facility__c matching the GSA's facility,
Then adjudication uses the GSA's applicable GSA_Rate_Tier__c rate rather than any individual PNA rate, and Claim__c.Contract_Type__c is stamped as 'GSA'.

Given a GSA is approaching its Expiration_Date__c within 60 days,
When a nightly scheduled flow runs,
Then a Provider_Change_Notification__c of Type__c = 'Contract Renewal Alert' is created and assigned to the Provider Network Specialist queue, and GSA__c.Renewal_Alert_Sent__c is set to true to prevent duplicate alerts.

Given a Compliance Officer needs to audit GSA rate tiers,
When they run the 'GSA Rate Audit' report,
Then the report displays GSA__c.Name, Facility__c.Name, GSA_Rate_Tier__c.Specialty__c, Fee_Schedule__c.Rate_Amount__c, and Effective_Date__c for all active GSAs, filterable by LOB__c.

## Solution

1. OmniScript 'GSA_Creation_Wizard' (5 steps) with DataRaptor Transforms for pre-population from Facility__c.
2. Custom objects: GSA_Rate_Tier__c (child of Group_Service_Agreement__c) with fields Specialty__c (picklist), Tier__c (picklist), Fee_Schedule__c (lookup).
3. Junction object Facility_Provider_Affiliation__c linking Facility__c and Provider__c; multi-select step uses a custom LWC datatable within the OmniScript.
4. Record-Triggered Flow on Claim__c (before insert) — Apex-invoked decision logic in ClaimContractResolutionService: checks GSA coverage before falling back to PNA lookup.
5. Scheduled Flow 'GSA_Renewal_Alert_Flow' running nightly, querying GSA records where Expiration_Date__c <= TODAY() + 60 AND Renewal_Alert_Sent__c = false.
6. Custom Report Type 'GSA Rate Tiers with Fee Schedules' registered in Setup for Compliance Officer reporting.

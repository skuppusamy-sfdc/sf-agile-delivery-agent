## Description

**Bug Report — INS-0354**

**Environment:** Acme Insurance Full Sandbox (FS1)
**Severity:** High — Blocks PNA activation workflow (INS-0350)

**Steps to Reproduce:**
1. Navigate to an existing Provider__c record with a valid NPI and TIN.
2. Launch the PNA creation OmniScript wizard (INS-0350) and complete all steps.
3. Click 'Generate & Send for Signature' — confirm DocuSign envelope is dispatched (Status__c = 'Pending Signature').
4. In the DocuSign test harness, simulate a completed signature event for the envelope.
5. Wait 5 minutes and refresh the Provider_Network_Agreement__c record.

**Expected Result:** PNA.Status__c should update to 'Active' and PNA.Effective_Date__c should be populated with the signature completion timestamp date.

**Actual Result:** PNA.Status__c remains 'Pending Signature'. PNA.Effective_Date__c is null. No error appears in the UI. Checking Debug Logs reveals the Platform Event (DocuSign_Completed__e) is firing successfully, but the subscriber Record-Triggered Flow (PNA_DocuSign_Completion_Handler) is failing silently with a FIELD_INTEGRITY_EXCEPTION on PNA.Effective_Date__c — the flow is attempting to write a DateTime value directly into a Date field.

**Root Cause (Identified):** In the Record-Triggered Flow, the 'Set Variable' element maps {!DocuSign_Completed__e.CompletedDateTime__c} (DateTime type) directly to PNA.Effective_Date__c (Date type) without a type conversion formula.

**Regression Risk:** All PNAs created via the wizard since Sprint 5 Day 1 deployment are stuck in 'Pending Signature'. Estimated impact: 14 PNA records in FS1.

## Acceptance Criteria

As per Description. Fix must ensure PNA.Status__c = 'Active' and PNA.Effective_Date__c is populated as a Date value (not DateTime) upon DocuSign completion event processing. Regression test must confirm no FIELD_INTEGRITY_EXCEPTION appears in debug logs post-fix.

## Solution

In the PNA_DocuSign_Completion_Handler Record-Triggered Flow (Platform Event trigger on DocuSign_Completed__e), update the 'Set Variable' element for Effective_Date__c to use the formula: DATEVALUE({!DocuSign_Completed__e.CompletedDateTime__c}) to convert DateTime to Date before assignment. Re-activate the flow. Execute a one-time data fix script (Anonymous Apex) to query all PNA records where Status__c = 'Pending Signature' and CreatedDate >= Sprint 5 start date, then re-query the associated DocuSign envelope status via DocuSign Apex Toolkit and apply the correction. Validate fix in FS1 before promoting to UAT.

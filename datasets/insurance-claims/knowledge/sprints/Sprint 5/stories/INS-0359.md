## Description

As a Provider Relations Specialist, I need the Provider_Network_Agreement__c (PNA) record to automatically progress through its lifecycle stages based on contract document events in Conga CLM and DocuSign, so that network contracting status is always accurate in Salesforce without manual status updates.

The PNA lifecycle states are: Draft → Under Legal Review → Sent for Signature → Executed → Active → Expired/Terminated.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record is created with Status__c = 'Draft',
When a Conga CLM contract document is generated and sent for legal review via the 'Send to Legal' quick action,
Then PNA.Status__c updates to 'Under Legal Review' and PNA.Legal_Review_Start_Date__c is stamped with the current datetime.

Given PNA.Status__c = 'Under Legal Review',
When the Conga CLM webhook fires a `contract.approved` event,
Then PNA.Status__c transitions to 'Sent for Signature' and a DocuSign envelope is automatically generated using the PNA contract template and sent to Provider__c.Email (NPI-verified contact).

Given the DocuSign envelope is sent,
When all parties complete signature (DocuSign status = 'Completed'),
Then PNA.Status__c = 'Executed', PNA.Execution_Date__c is stamped, and PNA.Contract_Document_URL__c is populated with the signed document link from DocuSign.

Given PNA.Status__c = 'Executed' and PNA.Effective_Date__c <= Today,
When a scheduled Flow runs nightly,
Then PNA.Status__c transitions to 'Active' and all associated Fee_Schedule__c records with matching PNA lookup are set to Fee_Schedule__c.Active__c = TRUE.

Given PNA.Expiration_Date__c = Today + 90 days,
When the nightly scheduled Flow evaluates the record,
Then a Provider_Change_Notification__c (PCN) record is auto-created with PCN.Type__c = 'Contract Renewal Reminder' and PCN.Assigned_To__c = PNA.Owner.

Given PNA.Expiration_Date__c < Today,
When the nightly scheduled Flow runs,
Then PNA.Status__c = 'Expired', all child Fee_Schedule__c records are deactivated (Active__c = FALSE), and any in-flight Claim__c records with Claim__c.PNA_Lookup__c referencing this PNA are flagged with Claim__c.Network_Verification_Required__c = TRUE.

## Solution

Build a Salesforce Platform Event listener (`CongaCLM_Event__e`) to receive inbound webhook payloads from Conga CLM via a REST endpoint exposed through Salesforce Experience Cloud or a MuleSoft proxy. An Apex trigger on the Platform Event fires a `PNALifecycleService` class that evaluates the event type and performs the appropriate DML on Provider_Network_Agreement__c. DocuSign envelope generation is handled via the DocuSign for Salesforce managed package's Apex API (`dfsle.EnvelopeService.sendEnvelope()`). The nightly expiration/activation logic is implemented as a Scheduled Flow (runs at 02:00 UTC) with batch sub-flows for large volumes. A custom Permission Set Group `PNA_Contract_Manager` gates the 'Send to Legal' quick action visibility.

## Description

As a Compliance Officer, I need all changes to Fee_Schedule__c rate lines (Fee_Schedule_Line__c) to be version-controlled with effective dating so that I can audit historical contracted rates for any given service date and produce accurate ERA/EOB reconciliation reports when providers dispute adjudicated amounts.

This is particularly critical for CMS audit responses where we must demonstrate the exact fee schedule rate in effect on the date of service for any given CPT code. Extends INS-0388 (fee schedule lookup engine) by adding the historical rate lookup dimension.

## Acceptance Criteria

Given a Fee_Schedule_Line__c record has its Rate__c field updated,
When the record is saved,
Then the previous rate value is archived to a Fee_Schedule_Line_History__c child record with fields: Historical_Rate__c (prior value), Effective_To__c (date of change - 1 day), Changed_By__c (running user), and Change_Reason__c (required text field).

Given a Fee_Schedule_Line_History__c archive record is created,
When a Claims Operations Manager queries rate history for a specific CPT code,
Then they can view a chronological timeline of all rate changes via a related list on Fee_Schedule__c sorted by Effective_To__c DESC.

Given a Claim_Line__c is adjudicated,
When Claim_Line__c.Allowed_Amount__c is set,
Then Claim_Line__c.Fee_Schedule_Line_Snapshot__c stores the Fee_Schedule_Line__c.Id and Fee_Schedule_Line__c.Rate__c value at time of adjudication as an immutable audit record.

Given a user attempts to delete a Fee_Schedule_Line__c record that has associated Fee_Schedule_Line_History__c children or linked Claim_Line__c records,
When the delete is attempted,
Then a before-delete Apex Trigger prevents deletion and displays: 'This rate line cannot be deleted as it has adjudication history. Set Status__c to Inactive instead.'

## Solution

Create Fee_Schedule_Line_History__c custom object as a child of Fee_Schedule_Line__c with fields: Historical_Rate__c (Currency), Effective_From__c (Date), Effective_To__c (Date), Changed_By__c (Lookup to User), Change_Reason__c (Text 255, required). Implement before-update Apex Trigger (FeeScheduleLineTrigger.cls) to capture old rate value and insert the history record in the same transaction. Add Claim_Line__c.Fee_Schedule_Line_Snapshot__c as a Long Text Area storing a JSON snapshot (CPT code, rate, fee schedule name, effective date) at adjudication time — populated by the FeeScheduleLookupService.cls updated as part of INS-0388 fix. Before-delete trigger logic added to FeeScheduleLineTrigger.cls. Field History Tracking also enabled on Fee_Schedule_Line__c.Rate__c as a secondary audit mechanism. Add a custom report type (Fee Schedule Rate History) for Compliance Officer reporting.

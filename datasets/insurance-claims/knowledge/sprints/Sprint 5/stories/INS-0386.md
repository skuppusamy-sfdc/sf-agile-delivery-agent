## Description

**Bug Report**

**Summary:** When a Fee_Schedule_Line__c record has Rate_Type__c = 'Percent of Medicare', the FeeScheduleLookupService (built in INS-0381) fails to correctly calculate and persist the resolved dollar amount to Claim_Line__c.Contracted_Rate__c. The field is left NULL, causing downstream payment calculation to produce a $0.00 payment on the EOB.

**Steps to Reproduce:**
1. Navigate to an active Provider_Network_Agreement__c with a linked Fee_Schedule__c.
2. Open the Fee_Schedule__c and add a Fee_Schedule_Line__c with CPT_Code__c = '99213', Rate_Type__c = 'Percent of Medicare', Allowed_Amount__c = 110 (representing 110%).
3. Submit a new Claim__c with a Claim_Line__c referencing CPT code 99213 and the above provider.
4. Allow the adjudication flow to run (or trigger manually via 'Run Adjudication' button in sandbox).
5. Inspect Claim_Line__c.Contracted_Rate__c — observe the field is NULL.
6. Inspect the generated Explanation_of_Benefits__c — observe EOB.Total_Allowed_Amount__c = $0.00.

**Expected Behavior:** FeeScheduleLookupService should retrieve the current Medicare Allowable rate for CPT 99213 from the CMS_Medicare_Rate__mdt custom metadata, multiply by (Allowed_Amount__c / 100), and persist the resulting dollar value to Claim_Line__c.Contracted_Rate__c.

**Actual Behavior:** Claim_Line__c.Contracted_Rate__c = NULL. EOB.Total_Allowed_Amount__c = $0.00. No error is thrown or logged.

**Environment:** Full Sandbox (ACME-FS1)
**Severity:** High — blocks all Medicare-percentage-based claims from paying correctly.
**Regression:** Introduced in Sprint 5 deployment on 2024-11-04.

## Acceptance Criteria

As per Description.

Additional verification:
Given the fix is deployed,
When a Claim_Line__c with a 'Percent of Medicare' fee schedule line is adjudicated,
Then Claim_Line__c.Contracted_Rate__c = (CMS_Medicare_Rate__mdt.Rate__c for matching CPT code) * (Fee_Schedule_Line__c.Allowed_Amount__c / 100), rounded to 2 decimal places.

Given the fix is deployed,
When Rate_Type__c = 'Percent of Medicare' but no matching CMS_Medicare_Rate__mdt record exists for the CPT code,
Then Claim_Line__c.Contracted_Rate__c remains NULL and Claim__c.Adjudication_Notes__c is appended with 'No Medicare reference rate found for CPT [code]; manual review required'.

## Solution

Root cause: In FeeScheduleLookupService.resolveContractedRate(), the switch-case block for Rate_Type__c = 'Percent of Medicare' contains a null-check guard on CMS_Medicare_Rate__mdt query result that returns null (silently) instead of falling through to the warning path. Fix: (1) Correct the null guard to fall through to the warning/logging branch. (2) Ensure the Medicare rate lookup queries CMS_Medicare_Rate__mdt by CPT_Code__c with a case-insensitive SOQL filter. (3) Add a unit test in FeeScheduleLookupServiceTest covering the 'Percent of Medicare' path with and without a matching metadata record. Deploy fix to ACME-FS1 and regression test INS-0381 acceptance criteria.

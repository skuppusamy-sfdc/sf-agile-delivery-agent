## Description

As an Underwriting Analyst, I need the system to support Group Service Agreement (GSA)-level fee schedule overrides that supersede the standard PNA-level rates for specific employer groups, so that custom-negotiated rates for large group clients are accurately applied during claims adjudication without overwriting the base fee schedule.

## Acceptance Criteria

Given a Group_Service_Agreement__c record exists with GSA.Fee_Schedule_Override__c = TRUE and a child Fee_Schedule__c record with GSA_Override__c = TRUE,
When a Claim__c is submitted where Claim__c.Policy__c.Group_Account__c matches GSA.Group_Account__c,
Then the fee schedule lookup in INS-0358 must first check for a GSA-level Fee_Schedule__c override before falling back to the PNA-level rate.

Given a GSA-level Fee_Schedule__c override is found,
When the Claim_Line__c.Contracted_Rate__c is populated,
Then Claim_Line__c.Rate_Source__c = 'GSA Override' and Claim_Line__c.GSA_Lookup__c is populated with the Group_Service_Agreement__c record ID.

Given no GSA-level override exists for the CPT code,
When the lookup falls back to the PNA-level Fee_Schedule__c,
Then Claim_Line__c.Rate_Source__c = 'PNA Standard' and processing continues as defined in INS-0358.

Given a GSA.Effective_Date__c > Claim__c.Date_of_Service__c,
When the GSA override lookup runs,
Then the GSA override is not applied and the system falls back to the PNA rate, logging a warning in Claim__c.Adjudication_Notes__c.

Given an Underwriting Analyst creates a new GSA fee schedule override,
When the Fee_Schedule__c record is saved with GSA_Override__c = TRUE,
Then a validation rule prevents saving if GSA.Status__c != 'Active' or if a duplicate CPT code override already exists for the same GSA and date range (error: 'Duplicate GSA rate override detected for CPT {CPT_Code__c}').

## Solution

Extend the `FeeScheduleLookupService` Apex class from INS-0358 to implement a two-pass lookup strategy: Pass 1 queries Fee_Schedule__c WHERE GSA_Override__c = TRUE AND GSA_Lookup__c IN (active GSAs for the policy group). Pass 2 falls back to the standard PNA query if Pass 1 returns no results. A new custom field `Claim_Line__c.Rate_Source__c` (picklist: 'GSA Override', 'PNA Standard', 'Manual Override') captures the resolution path for audit purposes. A Validation Rule on Fee_Schedule__c enforces the duplicate override check using a SOQL-based formula. Custom Metadata Type `Fee_Schedule_Priority__mdt` stores the lookup priority order (GSA > PNA > Default) per LOB to allow configuration without code changes.

# INS-0321: Implement Group Service Agreement (GSA) creation flow for employer group onboarding

## Description

As a Broker Relations Coordinator, I need a guided OmniScript to create a Group_Service_Agreement__c (GSA) record when onboarding a new employer group so that all contractual terms, effective dates, LOB coverage, and TPA designations are captured accurately and consistently. The flow must support multi-step data entry including group demographics, coverage tiers, TPA assignment, and broker of record linkage.

## Acceptance Criteria

Given a Broker Relations Coordinator navigates to the Employer Group Account,
When they launch the 'New GSA' OmniScript,
Then a multi-step wizard is presented with steps: Group Info, Coverage Configuration, TPA Assignment, Broker of Record, Review & Submit.

Given the user completes the Group Info step,
When they enter Group_Service_Agreement__c.Group_Effective_Date__c,
Then the field must be a future date, otherwise an inline validation error is displayed: 'GSA effective date must be a future date.'

Given the TPA Assignment step is reached,
When the user searches for a TPA,
Then only Account records where Account.Type = 'TPA' and Account.Active__c = true are returned.

Given all steps are completed,
When the user clicks 'Submit',
Then a Group_Service_Agreement__c record is created with Status__c = 'Pending Approval' and a Salesforce Approval Process is triggered for underwriting review.

Given the GSA approval is completed,
When Group_Service_Agreement__c.Status__c transitions to 'Executed',
Then a Conga CLM document generation job is invoked to produce the executed GSA PDF, stored as a ContentDocument linked to the GSA record.

## Solution

Build a 5-step OmniScript (Type: GSA_Onboarding, SubType: New) using OmniStudio. DataRaptor Transforms handle pre-population of group account data. Integration Procedure calls Conga CLM API on GSA execution. A Record-Triggered Flow on Group_Service_Agreement__c handles status transition logic and notifies the assigned Broker Relations Coordinator via custom notification. Approval Process configured with two-tier approval: Underwriting Analyst (first) and Compliance Officer (final). Permission Set 'GSA_Coordinator_PS' grants OmniScript access.

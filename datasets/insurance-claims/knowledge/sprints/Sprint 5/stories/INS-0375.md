## Description

As a Provider Network Specialist, I need automated email and in-app notifications sent to the assigned Provider Relations Specialist when a Group_Service_Agreement__c is within 60, 30, and 15 days of its GSA.Expiration_Date__c so that renewals are initiated proactively and no provider group lapses out of network unexpectedly.

## Acceptance Criteria

Given a Group_Service_Agreement__c record where GSA.Status__c = 'Active',
When the system date is exactly 60 days before GSA.Expiration_Date__c,
Then a custom notification (Bell notification) and an email must be sent to the user in GSA.Provider_Relations_Owner__c with subject 'GSA Expiring in 60 Days: {GSA.Name}'.

Given the same record,
When the system date is 30 days before expiry,
Then a second notification must fire and GSA.Renewal_Alert_Stage__c must be set to '30-Day Warning'.

Given the same record,
When the system date is 15 days before expiry,
Then GSA.Renewal_Alert_Stage__c must be set to '15-Day Warning' and a Task must be auto-created on the GSA record with Subject = 'Urgent: GSA Renewal Required', Due Date = GSA.Expiration_Date__c, and Owner = GSA.Provider_Relations_Owner__c.

Given a GSA where a renewal has been initiated (GSA.Renewal_In_Progress__c = true),
When any of the above thresholds are crossed,
Then the notification must be suppressed.

-Given expiry within 7 days, escalate to Provider Network Manager via Chatter post- (deprecated — escalation handled by separate SLA process in INS-0388)

## Solution

1. Scheduled Flow (runs daily) queries Group_Service_Agreement__c WHERE Status__c = 'Active' AND Renewal_In_Progress__c = false AND Expiration_Date__c IN (TODAY+60, TODAY+30, TODAY+15).
2. Flow branches on days-remaining value and sends Custom Notification via 'Send Custom Notification' core action using a custom notification type 'GSA_Expiry_Alert'.
3. Email Alert action uses an HTML email template 'GSA_Expiry_Notification' referencing merge fields from GSA.
4. Task creation on 15-day branch uses 'Create Records' element with auto-populated fields.
5. Renewal_Alert_Stage__c picklist field added to GSA object (None / 60-Day Warning / 30-Day Warning / 15-Day Warning).
6. Flow invocable tested with mock dates via a custom label 'GSA_Test_Date_Override' for sandbox validation.

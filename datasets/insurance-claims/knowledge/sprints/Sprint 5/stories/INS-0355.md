## Description

As a Broker Relations Coordinator, I need a screen flow to create and configure a Group_Service_Agreement__c (GSA) record for a new employer group, so that all required contracting data — including group size, LOB selections, TPA designation, and associated fee schedules — is captured in a structured and validated manner before the agreement is submitted for underwriting review. The flow must support both fully-insured and self-funded group types.

## Acceptance Criteria

Given a Broker Relations Coordinator launches the 'New GSA' screen flow from a Facility__c (Business Account) record representing an employer group
When Screen 1 (Group Profile) renders
Then the flow must pre-populate Group_Service_Agreement__c.Employer_Name__c from Account.Name, display a required picklist for GSA.Group_Type__c (values: Fully Insured, Self-Funded, Level Funded), and a required number field for GSA.Enrolled_Employees__c with a minimum validation of 2.

Given the coordinator selects 'Self-Funded' or 'Level Funded' as Group_Type__c
When Screen 2 (TPA Designation) renders
Then a required lookup to a TPA_Account__c (filtered Account records where RecordType = 'Third Party Administrator') must appear; this screen must be skipped entirely if Group_Type__c = 'Fully Insured'.

Given Screen 3 (LOB & Fee Schedule Selection) renders
When the coordinator selects one or more LOBs via a multi-select checkbox component
Then for each selected LOB, a dependent lookup must display available Fee_Schedule__c records filtered by Fee_Schedule__c.LOB__c and Fee_Schedule__c.Status__c = 'Active', and at least one fee schedule per selected LOB must be required before advancing.

Given all screens are completed
When the coordinator clicks 'Submit for Underwriting Review'
Then a GSA record must be created with GSA.Status__c = 'Pending Underwriting', GSA.Submitted_Date__c = TODAY(), and a Chatter post must be made on the GSA record notifying the Underwriting Analyst group (@Underwriting_Team).

Given the GSA is submitted
When an Underwriting Analyst opens the record
Then GSA.Underwriter__c (lookup to User) must be auto-assigned using round-robin logic from the Underwriting queue (Queue: Acme_Underwriting_Queue).

## Solution

Build a Screen Flow (GSA_Onboarding_Flow) with conditional screen visibility using Flow Decision elements to branch on Group_Type__c value. Multi-select LOB component built as a custom LWC (lWC: lob-multiselect-picker) that emits selected values to the flow via @api. Fee Schedule dependent lookup uses a Get Records element filtered dynamically. TPA lookup uses a filtered record collection. On submit, a Create Records element creates the GSA, and a Post to Chatter action uses the 'Post to Chatter' core action with a mention token for the Underwriting group. Round-robin auto-assignment implemented via an Apex invocable (QueueRoundRobinAssigner) called as a Flow Action after GSA creation. Flow launched via a Quick Action on Facility__c record page.

## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that consolidates active Provider_Network_Agreement__c records, current Credentialing_Case__c status, network tier assignments, and key identifiers (NPI, TIN) so that I can assess a provider's full network participation profile without navigating multiple related lists.

This story delivers the primary provider-facing UI component that supports both the PNA contracting workflow (INS-0343) and day-to-day provider data maintenance operations.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then it displays the following data sections: (1) Provider Identifiers — NPI__c, TIN__c, CAQH_ID__c, License_State__c; (2) Active Agreements — a child card list of Provider_Network_Agreement__c records where Status__c = 'Active', showing LOB__c, Network_Tier__c, Effective_Date__c; (3) Credentialing Summary — most recent Credentialing_Case__c.Status__c and Expiration_Date__c; (4) Quick Actions — 'New Network Agreement', 'Initiate PCN', 'View EOBs'.

Given a Provider__c has no active PNA records,
When the Agreements section of the FlexCard renders,
Then a contextual empty-state message 'No active network agreements. Click New Network Agreement to begin contracting.' is displayed with the action button inline.

Given a Credentialing_Case__c.Expiration_Date__c is within 90 days of today,
When the FlexCard credentialing section renders,
Then the expiration date is highlighted in amber and a tooltip reads 'Credentialing expires soon — initiate re-credentialing'.

-Given the FlexCard is embedded on the Facility__c record page, it shows GSA records instead of PNAs.- (Superseded — Facility 360 card is a separate story in Sprint 6)

Given a Provider Relations Specialist clicks 'Initiate PCN' from the FlexCard,
When the action fires,
Then a new Provider_Change_Notification__c record is created in Draft status pre-populated with Provider__c lookup and a modal confirmation is shown.

## Solution

1. OmniStudio FlexCard 'Provider_360_FC' with three child cards (Identifiers, Agreements, Credentialing) using DataRaptor Extract datasources.
2. DataRaptor Extract 'DR_Provider_Identifiers' queries Provider__c fields; 'DR_Active_PNAs' queries Provider_Network_Agreement__c with SOQL filter Status__c = 'Active'; 'DR_Latest_Credentialing' queries Credentialing_Case__c ORDER BY CreatedDate DESC LIMIT 1.
3. Conditional rendering on Credentialing expiry: FlexCard state expression TODAY() + 90 >= Expiration_Date__c triggers amber CSS class.
4. FlexCard Action 'Initiate PCN' calls Integration Procedure 'IP_Create_PCN_Draft' which invokes a Flow to create Provider_Change_Notification__c.
5. FlexCard deployed to Provider__c Lightning Record Page via App Builder, visible to Permission Set Group 'Provider_Relations_PSG'.

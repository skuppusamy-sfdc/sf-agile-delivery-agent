## Description

As a Credentialing Specialist, I need Salesforce to poll the CAQH ProView API on a scheduled basis and update the corresponding Credentialing_Case__c and Provider__c records with the latest attestation data — including board certifications, malpractice history, and license expiry dates — so that manual data entry is eliminated and re-credentialing cycles are triggered automatically when attestation lapses.

## Acceptance Criteria

Given the CAQH sync job runs every 24 hours,
When a provider's CAQH attestation date has changed since the last sync,
Then Credentialing_Case__c.CAQH_Last_Attestation_Date__c, License_Expiry_Date__c, and Board_Certification_Status__c are updated to reflect the latest CAQH ProView response.

Given the CAQH API returns a provider record with Authorization_Status__c = 'Authorized',
When the sync processes the record,
Then Provider__c.CAQH_Profile_Status__c is set to 'Authorized' and Credentialing_Case__c.Status__c remains unchanged unless a lapse condition is detected.

Given a provider's License_Expiry_Date__c returned by CAQH is within 90 days of TODAY,
When the sync processes the record,
Then Credentialing_Case__c.Status__c is updated to 'Re-Credentialing Required' and a Task is created for the assigned Credentialing Specialist with Subject = 'License Expiry Alert — Action Required'.

Given the CAQH API returns an HTTP 4xx or 5xx error for a specific NPI,
When the sync encounters the error,
Then the error is logged to a custom object 'Integration_Error_Log__c' with fields Error_Code__c, NPI__c, Timestamp__c, and Source_System__c = 'CAQH', and the Credentialing_Case__c record is NOT updated.

Given a Provider__c record does not have a CAQH_Provider_ID__c populated,
When the sync job iterates over credentialing cases,
Then that provider is skipped and logged to Integration_Error_Log__c with Error_Code__c = 'MISSING_CAQH_ID'.

Given a successful sync run completes,
When all records are processed,
Then a summary Platform Event 'CAQH_Sync_Complete__e' is published with Total_Processed__c, Total_Updated__c, and Total_Errors__c counts for monitoring dashboards.

## Solution

Build a Schedulable + Queueable Apex chain: 'CAQHSyncScheduler' triggers 'CAQHSyncQueueable' which calls the CAQH ProView REST API using a Named Credential 'CAQH_ProView_NC'. Responses are deserialized via a custom Apex wrapper class 'CAQHProViewResponse'. Field updates are batched using Database.update with allOrNone=false. Error logging uses a dedicated Apex utility 'IntegrationErrorLogger'. Platform Event publication is handled post-sync. A Custom Metadata Type 'CAQH_Sync_Config__mdt' stores endpoint URL, batch size, and re-credential threshold days for admin configurability without code changes.

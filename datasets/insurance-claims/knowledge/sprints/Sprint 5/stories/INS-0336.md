## Description

As a Provider Relations Specialist, I need Provider_Network_Agreement__c records to enforce valid status transitions and effective date rules so that expired or pending contracts cannot be used as the basis for claim reimbursement calculations, ensuring compliance with state network adequacy regulations.

The PNA lifecycle follows: Draft → Pending Signature → Active → Expired | Terminated. Retroactive activation must be blocked unless approved by a Compliance Officer.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record is in Status__c = 'Draft',
When a Provider Relations Specialist attempts to set Status__c = 'Active' directly (skipping 'Pending Signature'),
Then a validation error is thrown: 'PNA must progress through Pending Signature before activation. Please route for DocuSign execution.'

Given a PNA record has PNA__c.Effective_Start_Date__c in the past (more than 30 days prior to today),
When any user attempts to set Status__c = 'Active',
Then the save is blocked unless the record has PNA__c.Retroactive_Approval__c = TRUE and PNA__c.Compliance_Approver__c is populated with a Compliance Officer user.

Given PNA__c.Effective_End_Date__c is reached (today >= Effective_End_Date__c),
When a nightly scheduled Apex job runs,
Then PNA__c.Status__c is automatically set to 'Expired' and a Provider_Change_Notification__c (PCN) record is created with PCN__c.Notification_Type__c = 'Contract Expiration' linked to the Provider.

Given a PNA transitions to Status__c = 'Active',
When the transition is saved,
Then all linked Fee_Schedule__c records with Fee_Schedule__c.Status__c = 'Pending' are automatically set to 'Active'.

Given a PNA is Status__c = 'Terminated',
When any downstream process attempts to reference PNA__c for claims adjudication,
Then the rate lookup returns null and Claim_Line__c.Rate_Source__c = 'No Active Contract'.

## Solution

Implement a Record-Triggered Flow on Provider_Network_Agreement__c for before-save transitions to enforce status path validation via decision elements. Retroactive activation check uses a Custom Permission 'Retroactive_PNA_Activation' assigned only to Compliance Officer Profile. Nightly expiration job implemented as a Schedulable Apex class `PNAExpirationBatchJob` using Database.Batchable with batch size 200. PCN creation on expiration handled via after-update Record-Triggered Flow invoking a related record create action. Field History Tracking enabled on Status__c, Effective_Start_Date__c, Effective_End_Date__c for audit trail.

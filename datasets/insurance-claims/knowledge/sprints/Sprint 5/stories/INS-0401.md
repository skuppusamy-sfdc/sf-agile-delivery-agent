## Description

As a Claims Operations Manager, I need a dedicated Provider Network Contracting dashboard in Salesforce displaying real-time KPIs for active PNAs, contracts expiring within 90 days, fee schedule coverage by LOB, and uncontracted provider claims volume so that I can make data-driven decisions about network adequacy and prioritize contracting team workloads during quarterly business reviews.

## Acceptance Criteria

Given a user with the 'Claims_Operations_Manager' Profile or Permission Set accesses the 'Provider Network Contracting' Lightning App,
When the dashboard page loads,
Then the following KPI tiles are displayed: (1) Total Active PNAs, (2) PNAs Expiring in 90 Days, (3) PNAs Expiring in 30 Days, (4) Fee Schedule Coverage % by LOB, (5) Out-of-Network Claim Lines (Last 30 Days), (6) Average Days to Contract Execution.

Given the 'PNAs Expiring in 90 Days' KPI tile is displayed,
When the manager clicks the tile,
Then a drilldown list view opens showing PNA records filtered by Contract_End_Date__c <= TODAY() + 90 AND Status__c = 'Active', sorted by Contract_End_Date__c ascending, with columns: Provider Name, NPI, LOB, Contract End Date, Assigned Specialist, Renewal Status.

Given the 'Fee Schedule Coverage % by LOB' tile is rendered,
When the data is calculated,
Then coverage is defined as: (Count of Active PNAs with a linked Active Fee_Schedule__c) / (Total Active PNAs) * 100, grouped by PNA.LOB__c, and displayed as a donut chart.

Given the 'Out-of-Network Claim Lines' tile is rendered,
When the data is sourced,
Then it queries Claim_Line__c records where Network_Status__c = 'Out-of-Network' AND Date_of_Service__c >= LAST_N_DAYS:30, and the tile displays both count and total Billed_Amount__c sum.

Given the dashboard is viewed on any date,
When data is refreshed,
Then all KPIs reflect data as of the current date and a 'Last Refreshed' timestamp is displayed in the dashboard footer.

Given a Provider Network Specialist (non-manager role) accesses the same app,
When the dashboard loads,
Then KPI data is filtered by PNA.Assigned_Specialist__c = current user, showing only their assigned contracts.

## Solution

Lightning Dashboard built using standard Salesforce Report + Dashboard framework. Six custom Report Types created: (1) PNA with Contract Dates, (2) PNA with Fee Schedule Coverage, (3) Claim Lines by Network Status. Dashboard components use standard Metric, Donut, and Table chart types. Drilldown list views use filtered List View configurations linked from dashboard component URLs. Role-based data scoping implemented via Sharing Rules on PNA (criteria-based: Assigned_Specialist__c = current user for Specialist role) combined with Report filter logic. 'Last Refreshed' timestamp displayed via a custom LWC dashboard header component that reads the dashboard's last run date via the Analytics REST API. Dashboard added to the 'Provider Network Contracting' Lightning App home page tab.

## Description

**Bug Report — GSA OmniScript Step 3 TPA Assignment NullPointerException**

**Related Story:** INS-0321

**Steps to Reproduce:**
1. Log in as a Broker Relations Coordinator in Full Sandbox (FS1).
2. Navigate to any Employer Group Account record.
3. Launch the 'New GSA' OmniScript.
4. Complete Step 1 (Group Info) and Step 2 (Coverage Configuration) successfully.
5. On Step 3 (TPA Assignment), search for and select a TPA Account that has Account.Type = 'TPA' but has zero active Contact records.
6. Click 'Next' to proceed to Step 4.

**Expected Behavior:**
The OmniScript advances to Step 4 (Broker of Record) with the TPA Account saved to the session data.

**Actual Behavior:**
The OmniScript throws a generic error: 'An unexpected error occurred. Please try again.' The browser console shows: 'TypeError: Cannot read properties of undefined (reading ContactId)' originating from the DataRaptor Transform 'DR_GetTPAPrimaryContact'.

**Root Cause (Preliminary):**
The DataRaptor Transform 'DR_GetTPAPrimaryContact' assumes at least one Contact exists on the TPA Account and attempts to read the first Contact's Id without null-checking. When the TPA has no contacts, the transform returns an empty array and the subsequent field mapping throws a null reference error.

**Environment:** Acme Insurance - Full Sandbox (FS1)
**Frequency:** Reproducible 100% when TPA Account has 0 active contacts.
**Workaround:** Manually add a placeholder Contact to the TPA Account before running the OmniScript.

## Acceptance Criteria

As per Description. Verify fix by confirming the OmniScript advances to Step 4 when a TPA Account with zero contacts is selected. Confirm that when a TPA Account has one or more contacts, the primary contact is still pre-populated correctly in the TPA Contact field. No regression on Steps 1, 2, 4, and 5.

## Solution

Update the DataRaptor Transform 'DR_GetTPAPrimaryContact' to include a null/empty array check on the Contacts output node before mapping ContactId. Add a conditional branch: if the contacts array is empty, set a default value of null for the TPA_Primary_Contact__c field in the OmniScript session data rather than attempting property access. Update the OmniScript Step 3 element to display an informational message 'No primary contact found for selected TPA — you may add one after GSA submission' when TPA_Primary_Contact__c is null, without blocking navigation to Step 4. Regression test all five OmniScript steps in FS1 before promoting fix to UAT.

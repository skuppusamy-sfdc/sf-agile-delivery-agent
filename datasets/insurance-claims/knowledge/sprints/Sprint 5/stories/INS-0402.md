## Description

**Bug Report – INS-0402**

**Environment:** Acme Insurance UAT Org (v47.3)
**Reported By:** Emily Watson (Broker Relations Coordinator)
**Date Observed:** 2025-01-15

**Steps to Reproduce:**
1. Log in as a user with the 'Broker Relations Coordinator' Profile.
2. Navigate to the Provider Network Agreements tab.
3. Attempt to view a PNA record where PNA.Region__c = 'Northeast' while the logged-in user's Region__c = 'Midwest'.
4. Observe the record is either not visible in list views or returns an 'Insufficient Privileges' error on direct URL access.

**Expected Result:** Broker Relations Coordinators should have Read access to all active PNA records across all regions to support cross-regional broker inquiries and GSA negotiations, per business requirement BRD-NET-045.

**Actual Result:** The current Criteria-Based Sharing Rule ('PNA_BrokerCoordinator_Share') only shares PNA records where PNA.Region__c matches the user's own Region__c field, inadvertently blocking cross-region visibility.

**Root Cause Hypothesis:** The sharing rule criterion was configured as 'PNA.Region__c EQUALS User.Region__c' instead of the intended 'PNA.Status__c EQUALS Active' (share all active PNAs with the Broker Relations Coordinator Role). This appears to have been a misconfiguration during Sprint 3 setup (reference INS-0361).

**Impact:** Broker Relations Coordinators are unable to respond to cross-regional provider inquiries, causing delays in GSA negotiations and escalations to Provider Network Specialists for routine data lookups. Affects approximately 8 active Broker Relations Coordinator users.

## Acceptance Criteria

As per Description. Verify fix by confirming a 'Broker Relations Coordinator' profile user can view PNA records from all regions where Status__c = 'Active'. Confirm users cannot edit PNA records (Read-only access only). Regression test: confirm Provider Network Specialists retain their existing OWD-based edit access and that PNA records with Status__c = 'Terminated' or 'Draft' remain inaccessible to Broker Relations Coordinators.

## Solution

Update the existing Criteria-Based Sharing Rule ('PNA_BrokerCoordinator_Share') in Setup > Sharing Settings for the Provider_Network_Agreement__c object. Change the sharing criterion from 'PNA.Region__c EQUALS User.Region__c' to 'PNA.Status__c EQUALS Active'. Set the shared-to target as the 'Broker Relations Coordinator' Role with 'Read Only' access level. Recalculate sharing after the rule update to propagate access changes to existing records. Document the change in the Security Configuration Runbook and update the related test script in the UAT test case suite referencing INS-0361.

## Description

As a Claims Operations Manager, I need the fee schedule rate resolution engine to recognize when a Claim is associated with a Group_Service_Agreement__c (GSA) that carries employer-specific rate overrides, so that group employer clients with negotiated carve-outs receive correct reimbursement rates rather than defaulting to the standard PNA fee schedule.

Rate hierarchy (highest to lowest priority): GSA Override Rate → PNA Fee Schedule Rate → UCR (Usual, Customary, and Reasonable) Default.

## Acceptance Criteria

Given a Claim__c is linked to a Policy__c that has Policy__c.GSA__c populated,
When fee schedule rate resolution runs on Claim_Line__c,
Then the system first checks for a matching Fee_Schedule__c record where Fee_Schedule__c.Parent_GSA__c equals the Policy's GSA.

Given a GSA-level Fee_Schedule__c record exists for the CPT code on Claim_Line__c,
When the GSA rate is found and GSA__c.Status__c = 'Active',
Then Claim_Line__c.Contracted_Rate__c = GSA fee schedule rate, Claim_Line__c.Rate_Source__c = 'GSA Override', and Claim_Line__c.GSA__c is populated with the GSA record ID.

Given no GSA-level fee schedule entry exists for the CPT code,
When the GSA rate lookup returns no results,
Then the system falls through to the standard PNA Fee_Schedule__c lookup and Claim_Line__c.Rate_Source__c = 'Fee Schedule'.

Given neither GSA nor PNA fee schedule entries exist for the CPT code,
When both lookups return null,
Then Claim_Line__c.Contracted_Rate__c = Fee_Schedule__c.UCR_Rate__c (from UCR table), Claim_Line__c.Rate_Source__c = 'UCR Default', and Claim_Line__c.Requires_Manual_Review__c = TRUE.

Given a GSA__c.Override_Effective_End_Date__c has passed,
When rate resolution runs,
Then the GSA override is ignored and the system falls through to PNA rate, logging a warning in Integration_Log__c with message 'GSA Override Expired — Falling Back to PNA Rate'.

## Solution

Extend the `FeeScheduleRateLookupService` Apex class (built in INS-0335) to implement a rate resolution chain pattern. Add a `resolveGSARate(Id claimLineId, Id gsaId, String cptCode)` method that queries Fee_Schedule__c filtered by Parent_GSA__c and CPT_Code__c. Chain resolution implemented via Strategy design pattern with three resolver classes: `GSARateResolver`, `PNARateResolver`, `UCRRateResolver` — each implementing a common `IRateResolver` interface. The invocable method passes context to the chain and returns the first non-null result. Rate_Source__c population handled within each resolver. Integration_Log__c entries written via a separate `AuditLogService` utility class. Custom Metadata Type `Rate_Resolution_Config__mdt` controls fallback behavior toggles per LOB.

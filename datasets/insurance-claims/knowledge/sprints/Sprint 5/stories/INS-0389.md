## Description

As a Provider Network Specialist, I need a real-time contract pipeline dashboard embedded on my home page that shows PNA records grouped by status (Draft, Pending Countersignature, Active, Expiring Soon, Terminated) so that I can prioritize outreach and avoid lapses in provider network coverage.

The dashboard must show key metrics including total active contracts by LOB, contracts expiring in 30/60/90 days, and contracts awaiting provider countersignature in DocuSign.

## Acceptance Criteria

Given the Provider Network Specialist loads their Salesforce Home Page,
When the FlexCard renders,
Then it displays PNA counts grouped by PNA.Status__c with drill-down navigation to filtered list views.

Given the FlexCard is rendered,
When PNA.Expiration_Date__c is within 30 days,
Then the record row is highlighted in red; within 31–60 days in amber; within 61–90 days in yellow.

Given the specialist clicks a status group tile,
When the drill-down action fires,
Then a filtered list of Provider_Network_Agreement__c records matching that status opens in a side panel.

Given a PNA record has PNA.DocuSign_Envelope_Status__c = 'Sent' for more than 14 days,
When the dashboard refreshes,
Then that record appears in the 'Awaiting Signature – Overdue' section with the provider's name and NPI displayed.

## Solution

Build an OmniStudio FlexCard (PNAPipelineCard) backed by a DataRaptor Extract that aggregates PNA counts using SOQL GROUP BY on Status__c. Use a DataRaptor Turbo Extract for performance on large data volumes (>10k PNA records). Embed FlexCard on the Lightning Home Page via App Builder. Implement conditional styling states in the FlexCard for color-coded expiry bands. Refresh interval set via a custom label (FlexCard_Refresh_Interval_Seconds, default 300). Permission Set Group (Provider_Network_Specialist_PSG) gates access to the dashboard component.

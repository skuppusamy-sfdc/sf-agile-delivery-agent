## Description

As a Claims Operations Manager, I need an Agentforce AI assistant embedded on the Claim__c record page that can answer adjudicators' natural language questions about the provider's active PNA terms, applicable fee schedule rates, and COB rules for the claim in context, so that adjudicators can resolve complex claims faster without leaving the record or consulting separate reference documents. This story extends the fee schedule foundation established in INS-0311.

## Acceptance Criteria

Given a Claims Adjudicator opens a Claim__c record with Status__c IN ('In Adjudication', 'Pending - Rate Review'),
When the Agentforce panel is visible in the utility bar,
Then the agent must be pre-seeded with the Claim__c.Id context and must be able to answer questions scoped to that specific claim without the adjudicator specifying the claim number.

Given the adjudicator asks 'What is the contracted rate for CPT 99214 for this provider?',
When Agentforce processes the prompt,
Then the agent must invoke a custom Apex Action ('GetFeeScheduleForClaim') that queries Fee_Schedule__c using the claim's Provider__c and LOB__c context,
And return the response in a structured format: CPT Code, Contracted Rate, Rate Type, Fee Schedule Name, and Expiration Date.

Given the adjudicator asks 'Is this provider in-network?',
When the agent evaluates the question,
Then it must check Provider_Network_Agreement__c.Contract_Status__c and PNA.Expiration_Date__c,
And respond with network status, tier, and a warning if the PNA expires within 30 days.

Given the claim has COB indicators (Claim__c.COB_Flag__c = TRUE),
When the adjudicator asks about coordination of benefits rules,
Then the agent must retrieve and summarize the COB rules from Policy__c.COB_Type__c linked to the claim,
And cite the specific policy field values in its response.

Given the agent generates a response,
When the adjudicator clicks 'Add to Claim Notes',
Then the agent's response text must be appended to Claim__c.Adjudication_Notes__c with a prefix 'AI Assist [timestamp]:',
And an audit record must be created in Claim_Audit__c with Action__c = 'Agentforce Response Logged'.

Given a question falls outside the agent's defined topic scope (e.g., unrelated HR or IT questions),
When the agent evaluates the prompt intent,
Then it must respond with a deflection message: 'I can only assist with claims adjudication, provider network, and fee schedule topics for the current claim.' and must NOT attempt to answer out-of-scope queries.

## Solution

Configure an Agentforce Agent ('Claims Adjudication Assistant') in Salesforce Agent Builder with a custom System Prompt scoped to claims adjudication domain. Define three custom Topics: (1) Fee Schedule Lookup, (2) Provider Network Status, (3) COB Rules Summary. Each topic references a custom Apex Action class (invocable method) that performs the relevant SOQL queries against Fee_Schedule__c, PNA, and Policy__c. Einstein Trust Layer will be enabled to prevent PII leakage outside the org. The 'Add to Claim Notes' action uses a custom LWC button in the agent conversation panel that calls an Apex method to update Claim__c.Adjudication_Notes__c. A custom Permission Set 'Agentforce_Claims_User_PS' will control access. Agent actions will be logged to Agentforce Audit Trail and mirrored to Claim_Audit__c via a Platform Event listener.

## Description

As a Credentialing Specialist, I need the system to automatically pull provider demographic and licensure data from CAQH ProView and NPPES (NPI Registry) when a new Credentialing_Case__c is opened, so that I do not need to manually re-enter NPI, TIN, license numbers, and board certifications that are already maintained in these external registries, reducing credentialing turnaround time and data entry errors.

## Acceptance Criteria

Given a Credentialing_Case__c is created with Credentialing_Case__c.RecordType = 'Initial Credentialing' and a linked Provider__c record,
When the record is saved and Provider__c.NPI__c is populated,
Then a Platform Event (Credentialing_Sync_Request__e) must be published with the NPI value and Credentialing_Case__c Id.

Given the Platform Event is consumed by the CredentialingIntegrationService,
When the NPPES API call to https://npiregistry.cms.hhs.gov/api returns a 200 response,
Then the following fields must be updated on Provider__c: Provider__c.First_Name__c, Provider__c.Last_Name__c, Provider__c.Primary_Taxonomy__c, Provider__c.NPPES_Last_Verified__c = TODAY.

Given the CAQH ProView API returns provider credentialing data,
When Credentialing_Case__c.CAQH_Provider_ID__c is populated,
Then the following fields must be populated on Credentialing_Case__c: CAQH_Status__c, License_Number__c, License_State__c, License_Expiration_Date__c, DEA_Number__c, Board_Certification__c, Malpractice_Insurance_Carrier__c, Malpractice_Policy_Expiration__c.

Given the NPPES API returns a 404 or the NPI is not found,
When the integration response is processed,
Then Credentialing_Case__c.Integration_Error__c must be set to 'NPI Not Found in NPPES Registry',
And Credentialing_Case__c.Status__c must be set to 'Pending Manual Review',
And a Task must be created for the Credentialing Specialist queue.

Given the CAQH authorization token has expired,
When the callout fails with HTTP 401,
Then the system must re-authenticate using stored Named Credential 'CAQH_ProView_NC' and retry the callout once,
And if the retry also fails, log an error to Integration_Error_Log__c with Error_Source__c = 'CAQH', Error_Code__c = '401', and alert the DataIntegration support queue.

-Given CAQH data conflicts with existing Provider__c fields the system should auto-overwrite without notification — removed per data governance policy DG-2024-07-

Given all credentialing data is successfully synced,
When Credentialing_Case__c.CAQH_Status__c = 'Complete' and all required license fields are populated,
Then Credentialing_Case__c.Status__c must automatically advance to 'Ready for Committee Review'.

## Solution

Publish a Platform Event (Credentialing_Sync_Request__e) from a Record-Triggered Flow on Credentialing_Case__c after-insert. An Apex Platform Event trigger (CredentialingEventTrigger) invokes CredentialingIntegrationService.cls which makes sequential callouts: first to NPPES REST API (Named Credential: NPPES_NC), then to CAQH ProView REST API (Named Credential: CAQH_ProView_NC). Both callouts are wrapped in a Queueable Apex class to avoid mixed DML and callout governor limit issues. JSON response parsing uses custom wrapper classes. A custom object Integration_Error_Log__c captures all failures with full request/response payloads (truncated to 131,072 chars). Named Credentials store all endpoint URLs and OAuth tokens. A custom Permission Set 'Credentialing_Integration_Admin_PS' controls who can view the raw sync log.

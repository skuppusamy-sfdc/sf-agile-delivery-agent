## Description

As a Credentialing Specialist, I need a Credentialing_Case__c record to be automatically created and assigned to the credentialing queue when a Provider_Network_Agreement__c is moved to 'Pending Signature' status so that credentialing review runs in parallel with the contract signature process and onboarding timelines are not delayed.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Status__c transitions from 'Draft' to 'Pending Signature',
When the record is saved,
Then a Credentialing_Case__c record must be created with:
  - Credentialing_Case__c.Provider__c = PNA.Provider__c
  - Credentialing_Case__c.PNA__c = PNA.Id
  - Credentialing_Case__c.Status__c = 'New'
  - Credentialing_Case__c.Source__c = 'Network Contracting'
  - Credentialing_Case__c.Due_Date__c = PNA.Effective_Date__c - 5 days
  - OwnerId = Queue 'Credentialing_Review_Queue'

Given the Credentialing_Case__c is created,
When the record is visible to the Credentialing Specialist,
Then the case must display a link to the provider's CAQH profile URL stored in Provider__c.CAQH_Profile_URL__c.

Given a PNA where PNA.Provider__c already has an open Credentialing_Case__c (Status__c NOT IN ('Closed','Withdrawn')),
When the new PNA transitions to 'Pending Signature',
Then a duplicate Credentialing_Case__c must NOT be created; instead, the existing case must be linked to the new PNA via a junction object PNA_Credentialing_Link__c.

Given the Credentialing_Case__c is created,
When the case is assigned to the queue,
Then an email notification must be sent to all members of 'Credentialing_Review_Queue' using email template 'Credentialing_New_Case_Alert'.

-Given the case is created, auto-verify NPI via NPPES API call- (deprecated — NPPES verification moved to nightly batch per INS-0355)

## Solution

1. Record-Triggered Flow on Provider_Network_Agreement__c (after save, update context) triggers when Status__c changes to 'Pending Signature'.
2. Flow calls Apex Invocable Action 'CredentialingCaseManager.createOrLinkCase(pnaId)' to handle duplicate detection logic (SOQL for open cases on same provider) and case creation.
3. Apex creates Credentialing_Case__c with all required fields; Due_Date__c calculated as DATE(PNA.Effective_Date__c) - 5.
4. If duplicate found, Apex creates PNA_Credentialing_Link__c junction record.
5. Email notification sent via Messaging.sendEmail() using OrgWideEmailAddress 'noreply@acmeinsurance.com'.
6. Unit tests: CredentialingCaseManagerTest covers new case creation, duplicate detection, and queue assignment with System.runAs() for queue context.
7. Queue 'Credentialing_Review_Queue' configured in Setup with email notification enabled.

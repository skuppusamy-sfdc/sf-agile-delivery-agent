## Description

As a Claims Operations Manager, I need the system to automatically look up the contracted allowed amount from the applicable Fee_Schedule__c when a Claim_Line__c is saved, so that adjudication is consistent, auditable, and eliminates manual rate entry errors across all LOBs. This story extends the PNA association work delivered in INS-0327.

## Acceptance Criteria

Given a Claim_Line__c record is created or updated with Claim_Line__c.CPT_Code__c populated and Claim_Line__c.Service_Date__c set,
When the record is saved,
Then an Apex trigger queries Fee_Schedule_Line__c records where Fee_Schedule_Line__c.CPT_Code__c = Claim_Line__c.CPT_Code__c AND Fee_Schedule_Line__c.Fee_Schedule__c = [PNA-associated Fee Schedule] AND Fee_Schedule_Line__c.Effective_Date__c <= Claim_Line__c.Service_Date__c AND Fee_Schedule_Line__c.Expiration_Date__c >= Claim_Line__c.Service_Date__c.

Given exactly one matching Fee_Schedule_Line__c is found,
When the lookup succeeds,
Then Claim_Line__c.Allowed_Amount__c is set to Fee_Schedule_Line__c.Allowed_Amount__c and Claim_Line__c.Fee_Schedule_Match_Status__c = 'Matched'.

Given no matching Fee_Schedule_Line__c is found,
When the lookup returns zero results,
Then Claim_Line__c.Fee_Schedule_Match_Status__c = 'Unmatched' and Claim_Line__c.Adjudication_Hold__c = TRUE and a Task is created assigned to the Claims Operations Manager queue with Subject = 'Fee Schedule Gap: CPT [code] not found'.

Given multiple overlapping Fee_Schedule_Line__c records are found (data quality issue),
When the lookup returns more than one result,
Then Claim_Line__c.Fee_Schedule_Match_Status__c = 'Conflict' and Claim_Line__c.Adjudication_Hold__c = TRUE and a Case is created on the parent Claim__c with Case.Type = 'Fee Schedule Conflict'.

-Given Claim_Line__c.Override_Allowed_Amount__c is populated by a supervisor, Then Claim_Line__c.Allowed_Amount__c should be overwritten by the trigger- (deprecated: supervisor override is now handled by a separate manual adjustment flow, not the trigger)

Given Claim_Line__c.Fee_Schedule_Match_Status__c = 'Matched',
When Claim__c.Status__c advances to 'Adjudicated',
Then Claim_Line__c.Adjudication_Hold__c must equal FALSE.

## Solution

Apex Trigger (ClaimLineTrigger) on Claim_Line__c, after insert/update, invokes a ClaimLineFeeScheduleService Apex class that performs a SOQL query on Fee_Schedule_Line__c with compound index on CPT_Code__c + Fee_Schedule__c + Effective_Date__c. Results are cached in a Platform Cache partition (FeeSchedulePartition) keyed by Fee_Schedule__c Id + CPT_Code__c to reduce SOQL consumption in bulk adjudication runs. Unit test coverage target ≥ 90%. Custom Metadata Type (Fee_Schedule_Config__mdt) stores configurable hold-reason codes.

## Description

As a Broker Relations Coordinator, I need a guided Screen Flow to create and configure a Group_Service_Agreement__c record for a new employer group, capturing group demographics, selected network tiers, LOB elections, and TPA designation, so that downstream policy issuance and provider network access can be correctly configured without manual data entry errors. Extends INS-0351 (Account hierarchy setup for employer groups).

## Acceptance Criteria

Given the Broker Relations Coordinator initiates the 'New GSA' flow from an employer Account record,
When the flow launches,
Then Step 1 collects: GSA.Group_Name__c (auto-populated from Account.Name), GSA.Effective_Date__c, GSA.Renewal_Date__c (must be >= Effective_Date__c + 365), GSA.TIN__c (validated 9-digit format), and GSA.SIC_Code__c.

Given Step 1 data is valid,
When the user proceeds to Step 2,
Then they are presented with a multi-select checklist of available Network_Tier__c options ('Tier 1', 'Tier 2', 'Tier 3') and LOB elections ('Medical', 'Dental', 'Vision', 'Behavioral Health'), with at least one LOB required.

Given Step 2 is complete,
When the user proceeds to Step 3,
Then they can optionally designate a TPA by searching for an existing TPA Account record; if designated, GSA.TPA_Account__c is populated and GSA.TPA_Administered__c = TRUE.

Given all steps are complete,
When the user clicks 'Submit',
Then GSA.Status__c = 'Pending Approval', a Salesforce Approval Process is triggered for manager review, and a confirmation email is sent to the Broker Relations Coordinator using an Email Alert.

-Given GSA approval, auto-generate Policy__c shell records for each enrolled member- (descoped: moved to Phase 2 member enrollment epic)

## Solution

Screen Flow 'GSA_Onboarding_Flow' with 4 screens (Group Info, Network/LOB Selection, TPA Designation, Review & Submit). Flow uses custom Apex invocable action 'ValidateTIN' for TIN format/uniqueness check against existing GSA records. Multi-select LOB checkboxes implemented as a Flow screen component with custom LWC 'lobSelector'. On submit, Flow creates GSA record, triggers standard Salesforce Approval Process 'GSA_Manager_Approval', and invokes an Email Alert via Flow Send Email action. Flow is surfaced as a Quick Action on the Account (Business Account) page layout, restricted via Permission Set Group 'Broker_Relations_PSG'.

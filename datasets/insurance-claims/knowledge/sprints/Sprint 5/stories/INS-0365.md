## Description

As a Provider Network Specialist, I need to initiate and manage a Provider_Network_Agreement__c record end-to-end — from draft through countersigned execution — so that contracted providers can be activated in-network without manual paperwork bottlenecks. The flow must support both individual Provider__c (Person Account) and Facility__c (Business Account) contract parties, and route the envelope to DocuSign automatically upon submission.

## Acceptance Criteria

Given a Provider Network Specialist navigates to a Provider__c or Facility__c record and clicks 'Initiate PNA',
When the OmniScript launches,
Then a new Provider_Network_Agreement__c record is created in Status__c = 'Draft' with Contract_Party_Type__c auto-populated as 'Individual' or 'Facility' based on the source record type.

Given the PNA record is in 'Draft' status and all required fields (Effective_Date__c, LOB__c, TIN__c, NPI__c) are populated,
When the specialist clicks 'Send for Signature',
Then a DocuSign envelope is generated via the DocuSign for Salesforce package and Provider_Network_Agreement__c.DocuSign_Envelope_Status__c is set to 'Sent'.

Given the provider countersigns the DocuSign envelope,
When the DocuSign callback webhook fires,
Then Provider_Network_Agreement__c.Status__c transitions to 'Executed', Executed_Date__c is stamped with today's date, and a Chatter post notifies the owning Provider Network Specialist.

Given the PNA Status__c transitions to 'Executed',
When the record is saved,
Then the associated Provider__c or Facility__c record's Network_Status__c field is updated to 'Active - Contracted' via a Record-Triggered Flow.

-Given a TPA-managed provider, route the envelope to the TPA contact instead of the provider directly.- (Superseded by INS-0341 TPA routing story)

Given a PNA is voided (Status__c = 'Void'),
When the record is saved,
Then the linked Provider__c or Facility__c Network_Status__c reverts to 'Out of Network' and a Provider_Change_Notification__c (PCN) record is auto-created.

## Solution

Implement an OmniScript ('PNA_Initiation_OS') with DataRaptor Transform steps to pre-populate contract party data from the source Account record. Use the DocuSign for Salesforce managed package 'dfsle__EnvelopeConfiguration__c' to define the template mapping. A Record-Triggered Flow on Provider_Network_Agreement__c handles status-change side effects (Network_Status__c updates, PCN creation). A named credential 'DocuSign_Prod' handles the webhook callback authentication. Permission Set Group 'Provider_Network_Specialist_PSG' grants access to the OmniScript and PNA object.

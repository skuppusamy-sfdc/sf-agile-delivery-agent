## Description

**Bug Report – INS-0399**

**Environment:** Acme Insurance UAT Org (v47.3)
**Reported By:** Rachel Torres (Provider Network Specialist)
**Date Observed:** 2025-01-14

**Steps to Reproduce:**
1. Navigate to any existing Fee_Schedule__c record with Status__c = 'Active' and a valid Effective_Date__c in the past (e.g., 2024-01-01).
2. Click the 'Clone' standard button from the record detail page.
3. On the cloned record form, update Fee_Schedule__c.Name to a new value and change Effective_Date__c to a future date (e.g., 2026-01-01).
4. Leave all other fields unchanged and click 'Save'.

**Expected Result:** The cloned Fee_Schedule__c record saves successfully with the new Effective_Date__c and Status__c defaults to 'Draft'.

**Actual Result:** A validation error fires: 'Effective Date cannot be earlier than the Contract Start Date.' even though the newly entered Effective_Date__c (2026-01-01) is clearly after the parent PNA.Contract_Start_Date__c. The record cannot be saved.

**Root Cause Hypothesis:** The Apex Trigger (FeeScheduleValidationTrigger, Before Insert) appears to be reading the original record's Effective_Date__c from a stale Trigger.old context rather than Trigger.new when the Clone action is used, causing the comparison to use the source record's date instead of the newly entered value.

**Impact:** Provider Network Specialists are unable to clone fee schedules for annual renewals, forcing full manual re-entry of all CPT code lines. Estimated 4–6 hours of rework per affected PNA per renewal cycle.

## Acceptance Criteria

As per Description. Verify fix by cloning a Fee_Schedule__c record with a future Effective_Date__c and confirming successful save without validation error. Regression test: confirm validation still fires correctly when a genuinely invalid date (before PNA Contract_Start_Date__c) is entered on both new and cloned records.

## Solution

Fix FeeScheduleValidationTrigger (Before Insert/Before Update) to reference Trigger.new[0].Effective_Date__c exclusively for date comparison logic, removing any inadvertent reference to Trigger.old in the Insert context. Add null-safe guard for Trigger.old on Insert operations. Update associated Apex test class (FeeScheduleValidationTriggerTest) to add test method covering the Clone scenario with future-dated Effective_Date__c. Deploy via change set to UAT for regression validation before production push.

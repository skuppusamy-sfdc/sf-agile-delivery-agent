## Description

As a Provider Network Specialist, I need automated notifications sent at 90, 60, and 30 days before a Provider_Network_Agreement__c.Contract_End_Date__c so that my team can proactively initiate contract renewals and avoid unintended network gaps that could disrupt claims adjudication for in-network providers.

## Acceptance Criteria

Given a PNA record where Status__c = 'Active' and Contract_End_Date__c is not null,
When the scheduled job runs daily at 6:00 AM CST,
Then the system identifies all PNA records where Contract_End_Date__c equals TODAY() + 90, TODAY() + 60, or TODAY() + 30.

Given a PNA is identified at the 90-day threshold,
When the notification is generated,
Then a Task is created on the PNA record with Subject = 'PNA Expiring in 90 Days – Renewal Review Required', Status = 'Open', Priority = 'High', and OwnerId set to the PNA.Assigned_Specialist__c lookup field.

Given a PNA is identified at the 60-day threshold,
When the notification is generated,
Then in addition to the Task, an email alert is sent to PNA.Assigned_Specialist__c and their manager (via Role Hierarchy) using the 'PNA_60Day_Expiration_Alert' email template.

Given a PNA is identified at the 30-day threshold,
When the notification fires,
Then PNA.Renewal_Status__c is automatically set to 'Escalated', a high-priority Task is created, and a Chatter post is added to the PNA record tagging the Provider Network Contracting group.

Given a PNA.Status__c is updated to 'Renewed' or 'Terminated' before the 30-day threshold,
When the scheduled job runs,
Then no further notifications are generated for that PNA record.

Given the notification job completes,
When more than 0 PNA records were processed,
Then a summary Platform Event (PNA_Notification_Summary__e) is published with fields: Records_Processed__c, Notifications_Sent__c, Errors__c, and Job_Run_Timestamp__c for monitoring.

## Solution

Build a Schedulable Apex class ('PNAExpirationNotificationJob') that queries PNA records meeting the threshold criteria daily. Use Apex to create Tasks and publish Platform Events (PNA_Notification_Summary__e). Email alerts at 60-day threshold leverage existing Email Alert actions invoked from Apex. A Record-Triggered Flow (After Update on PNA, Status__c changed to 'Renewed'/'Terminated') sets a PNA.Suppress_Notifications__c checkbox to prevent redundant alerts. Chatter posts at 30-day threshold via ConnectApi.ChatterFeeds.postFeedElement in Apex. Schedule the job via Anonymous Apex in deployment runbook. Custom Permission Set 'PNA_Notification_Admin' grants access to monitor Platform Event logs via a custom Report Type.

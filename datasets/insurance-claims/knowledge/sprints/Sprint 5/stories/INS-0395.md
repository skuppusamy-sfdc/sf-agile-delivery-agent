## Description

As a Provider Network Specialist, I need a guided wizard to create and configure Fee_Schedule__c records linked to a Provider_Network_Agreement__c so that I can accurately define CPT-based reimbursement rates without manual data entry errors and ensure fee schedules are effective-dated correctly for each LOB.

## Acceptance Criteria

Given a Provider Network Specialist navigates to a PNA record with Status__c = 'Active' or 'Pending Execution',
When they click 'New Fee Schedule' action button,
Then an OmniScript wizard launches with steps: (1) Schedule Header, (2) CPT Code Assignment, (3) Rate Configuration, (4) Effective Dating, (5) Review & Submit.

Given the specialist completes Step 1 (Schedule Header),
When they enter Fee_Schedule__c.Name, LOB__c, Schedule_Type__c (e.g., 'Per Diem', 'UCR', 'Case Rate', 'Percent of Billed'),
Then Fee_Schedule__c.Provider_Network_Agreement__c is auto-populated from the parent PNA and Fee_Schedule__c.Status__c defaults to 'Draft'.

Given the specialist is on Step 2 (CPT Code Assignment),
When they search for CPT codes via the integrated NPPES/CMS lookup,
Then matching CPT codes with standard descriptions are returned and selectable for bulk-add to the Fee_Schedule_Line__c junction object.

Given the specialist enters rates on Step 3,
When Fee_Schedule_Line__c.Reimbursement_Rate__c is entered and Schedule_Type__c = 'Percent of Billed',
Then the system validates the value is between 0.01 and 1.00 and displays an inline error if out of range.

Given the specialist sets effective dates on Step 4,
When Fee_Schedule__c.Effective_Date__c is set to a date before the parent PNA.Contract_Start_Date__c,
Then a warning message is displayed: 'Effective date precedes contract start date. Please confirm.' and the specialist must acknowledge before proceeding.

-Given duplicate CPT code entries are submitted, the system silently deduplicates without notification.- (Superseded: system now raises a validation error on duplicate CPT entries per INS-0381 feedback)

Given Step 5 (Review & Submit) is reached,
When the specialist clicks 'Submit for Approval',
Then Fee_Schedule__c.Status__c transitions to 'Pending Approval', an Approval Process is triggered, and the assigned Provider Relations Specialist receives a Task with Subject = 'Fee Schedule Approval Required'.

## Solution

Implement a multi-step OmniScript ('FS_Creation_Wizard') with DataRaptors for read/write operations against Fee_Schedule__c and Fee_Schedule_Line__c. Use a DataRaptor Integration Action for CMS CPT code lookup via REST callout. Effective date validation handled via a Formula Resource within OmniScript. Duplicate CPT detection via a DataRaptor Extract querying existing Fee_Schedule_Line__c records before insert. On final submission, invoke an Apex Action to trigger the standard Salesforce Approval Process on Fee_Schedule__c. A Record-Triggered Flow on Fee_Schedule__c (After Save, Status__c changes to 'Pending Approval') creates the follow-up Task for the Provider Relations Specialist. FlexCard embedded on the PNA record page surfaces the 'New Fee Schedule' quick action.

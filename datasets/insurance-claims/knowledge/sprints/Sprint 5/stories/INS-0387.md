## Description

As a Provider Network Specialist, I need to create a Provider_Network_Agreement__c (PNA) record directly from the Provider 360 page and associate it with the appropriate Fee_Schedule__c so that contracted rates are immediately available for claims adjudication without manual data re-entry across systems.

Background: Currently, network contracting staff must navigate to three separate screens to establish a new PNA and link a fee schedule. This causes data entry errors and delays in making new providers available for in-network claims routing. Extends INS-0357 (Provider 360 FlexCard foundation).

## Acceptance Criteria

Given a Provider Relations Specialist is on the Provider__c (Person Account) record page,
When they click 'New Network Agreement' from the PNA quick action button,
Then an OmniScript modal launches pre-populated with Provider__c.NPI__c and Provider__c.TIN__c fields.

Given the OmniScript is open,
When the specialist selects a Line of Business (LOB) from the LOB__c picklist (values: Medical, Dental, Vision, Behavioral Health),
Then the Fee_Schedule__c lookup filters to show only Fee_Schedule__c records where Fee_Schedule__c.LOB__c matches the selected LOB and Fee_Schedule__c.Status__c = 'Active'.

Given valid PNA data has been entered including Effective_Date__c, Expiration_Date__c, and a linked Fee_Schedule__c,
When the specialist submits the OmniScript,
Then a Provider_Network_Agreement__c record is created with PNA.Status__c = 'Pending Countersignature' and a DocuSign envelope is automatically sent to the provider's email on file (Provider__c.Email).

Given the PNA record is created,
When PNA.Status__c transitions to 'Active',
Then a Platform Event (PNA_Activated__e) is fired so downstream fee schedule engine can cache the new contracted rate.

-Given the specialist selects a legacy TPA fee schedule, the system should auto-map CPT code crosswalks from the TPA mapping table.- (Deprecated: TPA crosswalk handled in INS-0401)

Given PNA.Expiration_Date__c is within 90 days of today,
When the record is saved or updated,
Then a Task is auto-created and assigned to the owning Provider Relations Specialist with Subject = 'PNA Renewal Required – [Provider Name]' and Due_Date__c = Expiration_Date__c minus 90 days.

## Solution

Build a multi-step OmniScript (NetworkContracting/NewPNAWizard) with DataRaptor Load actions to create the Provider_Network_Agreement__c record. Use a DataRaptor Extract to pre-populate NPI__c and TIN__c from the parent Provider__c. Implement a Record-Triggered Flow on Provider_Network_Agreement__c for the 90-day renewal Task creation. Fire PNA_Activated__e Platform Event via Apex Trigger (PNATrigger.cls) on Status__c change to 'Active'. DocuSign envelope initiation handled via DocuSign for Salesforce REST callout within the OmniScript Integration Procedure. Deploy a FlexCard on the Provider 360 App Page to surface active PNA summary and expiration countdown.

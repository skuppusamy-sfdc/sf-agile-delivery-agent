## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that consolidates the provider's active PNA details, current Fee_Schedule__c name and effective dates, credentialing status from Credentialing_Case__c, and NPI/TIN so that I can quickly assess a provider's network standing without navigating across multiple related lists. This supports faster resolution of provider inquiries and claim dispute calls.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record,
When the Provider 360 FlexCard loads,
Then the following data is displayed in distinct card sections:
- Active PNA: Provider_Network_Agreement__c.Agreement_Number__c, Status__c, Effective_Date__c, Expiration_Date__c
- Fee Schedule: Fee_Schedule__c.Name, Rate_Effective_Date__c, LOB__c
- Credentialing: Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c
- Provider Identifiers: Provider__c.NPI__c, Provider__c.TIN__c

Given no active PNA exists for the provider,
When the FlexCard loads,
Then the PNA section displays 'No Active Agreement on File' in an amber warning badge.

Given the Credentialing_Case__c.Expiration_Date__c is within 90 days of today,
When the FlexCard renders,
Then the credentialing section displays a red 'Expiring Soon' indicator.

Given the FlexCard is loaded,
When the user clicks 'Initiate New PNA',
Then the PNA creation OmniScript (INS-0319) launches in a modal overlay.

-Given the FlexCard displays claims history summary with last 12 months of Claim__c counts-
Note: Claims history panel descoped to Sprint 6 per INS-0331.

## Solution

Build an OmniStudio FlexCard (FlexCard Name: Provider360_FC) with child cards for PNA, Fee Schedule, Credentialing, and Provider Identifiers sections. DataRaptor Extracts (DR_GetActivePNA, DR_GetFeeSchedule, DR_GetCredentialingStatus) retrieve related data. Conditional rendering uses State properties for the amber/red badge logic based on date comparisons. The 'Initiate New PNA' action invokes the OmniScript via a Navigate action. FlexCard deployed to the Provider__c Lightning Record Page via App Builder. Integration Procedure handles multi-object data aggregation to minimize SOQL calls. Visibility controlled via Permission Set Group 'Provider_Relations_PSG'.

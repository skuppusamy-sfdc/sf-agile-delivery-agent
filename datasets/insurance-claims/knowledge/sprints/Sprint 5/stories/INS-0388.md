## Description

As a Claims Operations Manager, I need the system to automatically look up the contracted allowed amount from the active Fee_Schedule__c when a Claim_Line__c is submitted for adjudication so that in-network reimbursements are calculated accurately and consistently without manual rate lookups.

The fee schedule lookup must resolve the correct rate using the hierarchy: (1) Provider-specific PNA rate, (2) Facility-level GSA rate, (3) Default network fee schedule rate. CPT codes and ICD-10 diagnosis codes must both be validated during lookup.

## Acceptance Criteria

Given a Claim_Line__c record is created with Claim_Line__c.CPT_Code__c populated and the parent Claim__c.Provider__c has an active PNA,
When the adjudication engine runs,
Then Claim_Line__c.Allowed_Amount__c is populated by matching CPT_Code__c against Fee_Schedule__c.Fee_Schedule_Line__c records where Fee_Schedule_Line__c.CPT_Code__c = Claim_Line__c.CPT_Code__c and Fee_Schedule__c.Effective_Date__c <= Claim__c.Service_Date__c <= Fee_Schedule__c.Expiration_Date__c.

Given a CPT code on the Claim_Line__c does not exist in any active Fee_Schedule__c,
When the adjudication engine runs,
Then Claim_Line__c.Adjudication_Status__c is set to 'Pended' and Claim_Line__c.Pend_Reason__c = 'CPT Not on Fee Schedule' and a Task is created for the Claims Operations Manager.

Given a provider has both a PNA-level rate and a default network rate for the same CPT code,
When the rate lookup executes,
Then the PNA-specific rate takes precedence (rate hierarchy level 1) and Claim_Line__c.Rate_Source__c = 'PNA'.

Given a Facility__c (Business Account) is billed on the claim and a Group_Service_Agreement__c (GSA) exists with GSA.Status__c = 'Active',
When no PNA-level rate is found,
Then the GSA rate is applied and Claim_Line__c.Rate_Source__c = 'GSA'.

Given the adjudication completes successfully,
When Claim_Line__c.Allowed_Amount__c is set,
Then Claim__c.Total_Allowed_Amount__c (roll-up summary) reflects the updated sum within the same transaction.

## Solution

Implement FeeScheduleLookupService.cls (Apex invocable method) called from a Record-Triggered Flow on Claim_Line__c (after insert/update where Adjudication_Status__c changes to 'Ready for Adjudication'). The Apex service executes a SOQL query with date-range filtering on Fee_Schedule_Line__c using a custom index on CPT_Code__c. Rate hierarchy logic implemented as a strategy pattern (PNARateResolver, GSARateResolver, DefaultRateResolver classes). Bulk-safe design supports up to 200 Claim_Line__c records per transaction. Add a custom metadata type (Fee_Schedule_Config__mdt) for configurable hierarchy order. Unit test coverage target: 90%.

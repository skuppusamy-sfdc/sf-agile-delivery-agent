## Description

As a Provider Network Specialist, I need automated alerts and a renewal task workflow triggered when a Fee_Schedule__c or Provider_Network_Agreement__c is approaching expiration, so that I can proactively renew contracts before they lapse and prevent claims from being adjudicated against expired fee schedules. Alerts should fire at 90-day, 60-day, and 30-day thresholds before the expiration date.

## Acceptance Criteria

Given a scheduled batch job runs nightly at 2:00 AM CST
When Fee_Schedule__c.Expiration_Date__c is within 90 calendar days of the current date and Fee_Schedule__c.Status__c = 'Active'
Then a Task must be created with Subject = 'Fee Schedule Expiring in 90 Days', assigned to the Provider Network Specialist identified in Fee_Schedule__c.Owner, with ActivityDate = TODAY() + 5 and a link to the Fee_Schedule__c record.

Given the same nightly batch
When Provider_Network_Agreement__c.Expiration_Date__c is within 60 days and PNA.Status__c = 'Active'
Then an in-app notification (Custom Notification Type: PNA_Expiration_Alert) must be sent to the PNA Owner and a Task created with Priority = 'High'.

Given a PNA reaches 30 days to expiration
When the batch processes the record
Then PNA.Renewal_Status__c must be automatically set to 'Renewal In Progress' if it is currently null or 'Not Started', and an email alert using the template 'PNA_30Day_Expiration_Notice' must be dispatched to the provider's email (Provider__c.PersonEmail).

Given a Fee_Schedule__c.Expiration_Date__c has passed and Status__c is still 'Active'
When the nightly batch runs
Then Fee_Schedule__c.Status__c must be updated to 'Expired' and all linked PNA records (via PNA.Fee_Schedule__c) must have PNA.Fee_Schedule_Status_Flag__c stamped as 'Fee Schedule Expired - Review Required'.

Given the renewal workflow is triggered
When a new Fee_Schedule__c record is created as a renewal (Fee_Schedule__c.Predecessor_Fee_Schedule__c populated with the expiring record's Id)
Then the batch must suppress further expiration alerts for the predecessor record and update predecessor Fee_Schedule__c.Renewal_Record__c with the new record's Id.

## Solution

Implement a Schedulable Apex class (FeeScheduleExpirationBatch implements Database.Batchable, Schedulable) querying Fee_Schedule__c and PNA records with date-range filters. Batch size: 200. Custom Notification sent via Apex ConnectApi.Notifications. Email alerts use an Org-Wide Email Address and Classic Email Template (PNA_30Day_Expiration_Notice). Task creation uses SObject DML within the batch execute method. Nightly schedule registered via Setup > Scheduled Jobs at 02:00 CST. A Custom Field (Fee_Schedule__c.Renewal_Record__c, Lookup to Fee_Schedule__c) tracks the renewal chain. Suppression logic uses a Set of predecessor IDs filtered before alert DML.

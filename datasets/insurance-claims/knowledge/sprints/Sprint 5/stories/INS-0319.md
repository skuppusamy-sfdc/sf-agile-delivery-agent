## Description

As a Provider Network Specialist, I need the ability to create a Provider_Network_Agreement__c (PNA) record linked to a specific Fee_Schedule__c so that contracted providers are accurately reimbursed according to negotiated rates when claims are adjudicated. The PNA must capture effective dates, LOB applicability, TIN, NPI, and the associated fee schedule version to ensure downstream Claim_Line__c adjudication pulls the correct allowed amounts.

## Acceptance Criteria

Given a Provider Relations Specialist is on the Provider__c (Person Account) record,
When they initiate a new PNA from the related list,
Then a Provider_Network_Agreement__c record is created with Provider_Network_Agreement__c.Status__c defaulting to 'Pending Execution'.

Given the PNA record is open,
When the user selects a Fee_Schedule__c lookup,
Then only active Fee_Schedule__c records where Fee_Schedule__c.LOB__c matches the PNA's Provider_Network_Agreement__c.Line_of_Business__c are returned in the lookup filter.

Given a Fee_Schedule__c is selected,
When the PNA effective date (Provider_Network_Agreement__c.Effective_Date__c) is entered,
Then a validation rule prevents saving if Provider_Network_Agreement__c.Effective_Date__c is earlier than Fee_Schedule__c.Rate_Effective_Date__c.

Given the PNA is saved with Status__c = 'Active',
When a Claim_Line__c is adjudicated for the linked Provider__c,
Then Claim_Line__c.Allowed_Amount__c is populated by referencing the linked Fee_Schedule__c CPT rate.

Given a duplicate active PNA exists for the same Provider__c, TIN, and LOB,
When a new PNA is saved with overlapping effective dates,
Then an error is thrown: 'An active PNA already exists for this provider and LOB within the specified date range.'

-Given PNA approval requires a wet signature, DocuSign envelope is triggered on save-
Given PNA approval requires e-signature, a DocuSign envelope is triggered via the 'Send for Signature' button using the standard PNA template.

## Solution

Build a Record-Triggered Flow on Provider_Network_Agreement__c (after save, created/updated) to validate fee schedule LOB alignment and trigger duplicate detection via an invocable Apex method. Implement a custom lookup filter on the Fee_Schedule__c field using filter criteria on Fee_Schedule__c.Status__c = 'Active' and LOB match. Validation rules enforce date integrity. DocuSign integration leverages the existing DocuSign for Salesforce managed package with a custom mapping set for PNA fields. Apex trigger (PNATrigger.cls) handles duplicate PNA detection with a SOQL query on overlapping date ranges. Permission Set Group 'Provider_Network_Contracting_PSG' grants access to PNA and Fee Schedule objects.

## Description

**Bug Report — INS-0315**

**Summary:** When a provider has multiple active Provider_Network_Agreement__c records covering different LOBs (e.g., one for 'Commercial' and one for 'Medicare Advantage'), the fee schedule lookup in the FeeScheduleCalculationService introduced in INS-0311 returns the first PNA found by SOQL insertion order rather than the PNA that matches the Claim__c.LOB__c value. This causes incorrect Allowed_Amount__c to be stamped on Claim_Line__c records.

**Steps to Reproduce:**
1. Ensure Provider__c record 'Dr. James Holloway' (NPI: 1234567890) has two active PNA records: PNA-001 (LOB__c = 'Commercial', Fee Schedule Rate = $150.00) and PNA-002 (LOB__c = 'Medicare Advantage', Fee Schedule Rate = $112.50).
2. Create a Claim__c with Claim__c.LOB__c = 'Medicare Advantage' linked to Dr. Holloway.
3. Add a Claim_Line__c with CPT__c = '99213'.
4. Transition Claim__c.Status__c to 'In Adjudication'.
5. Observe Claim_Line__c.Allowed_Amount__c.

**Expected Result:** Claim_Line__c.Allowed_Amount__c = $112.50 (Medicare Advantage rate from PNA-002).

**Actual Result:** Claim_Line__c.Allowed_Amount__c = $150.00 (Commercial rate from PNA-001, returned first by SOQL).

**Impact:** HIGH — Incorrect payment amounts will be issued on all Medicare Advantage claims for multi-LOB providers. Potential CMS compliance violation if Medicare rates are overpaid.

## Acceptance Criteria

As per Description. Additional: the SOQL query in FeeScheduleCalculationService must include a WHERE clause filter on PNA.LOB__c = Claim__c.LOB__c to ensure LOB-specific rate resolution. Regression test must cover single-LOB provider, multi-LOB provider (correct LOB match), and multi-LOB provider (no LOB match — should set Pricing_Exception__c = TRUE).

## Solution

Fix the SOQL query in FeeScheduleCalculationService.cls to add an explicit filter: WHERE Provider__c = :claim.Provider__c AND LOB__c = :claim.LOB__c AND Network_Status__c = 'Active' AND Effective_Date__c <= TODAY AND Expiration_Date__c >= TODAY ORDER BY Effective_Date__c DESC LIMIT 1. Add a new @isTest class 'FeeScheduleCalculationServiceTest_MultiLOB' with data factory setup creating dual-PNA scenarios. Deploy fix to sandbox for QA regression before promoting to UAT. Update the custom metadata type (Fee_Schedule_Config__mdt) to document the LOB-resolution priority logic for future maintainers.

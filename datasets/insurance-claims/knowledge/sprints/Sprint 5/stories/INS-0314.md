# INS-0314: Build Provider 360 FlexCard displaying active PNAs, fee schedules, credentialing status, and open claims

## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that consolidates active PNA details, linked Fee_Schedule__c summaries, current Credentialing_Case__c status, and a count of open Claim__c records, so that I have a single-pane view when handling provider inquiries without navigating across multiple related lists.

## Acceptance Criteria

Given a Provider__c record is opened by a user with the 'Provider_Relations_PSG' Permission Set Group,
When the Provider 360 FlexCard loads,
Then it must display all Provider_Network_Agreement__c records where PNA.Contract_Status__c IN ('Active', 'Pending Signature') in a child card list, showing PNA.Network_Tier__c, PNA.Effective_Date__c, PNA.Expiration_Date__c, and PNA.LOB__c.

Given the FlexCard is rendering the credentialing section,
When a Credentialing_Case__c exists for the provider with Credentialing_Case__c.Status__c != 'Closed',
Then the card must display a colored status badge: green for 'Approved', yellow for 'In Review', red for 'Suspended' or 'Denied'.

Given the FlexCard loads the claims summary section,
When a DataRaptor queries Claim__c records linked to the provider,
Then the card must display counts for: Open Claims, Claims in Adjudication, and Claims Pending Payment, grouped by LOB__c.

Given the provider has no active PNA,
When the FlexCard PNA section loads,
Then a prominent banner must display 'No Active Network Agreement — Provider is Out of Network' in red,
And an action button 'Initiate New Agreement' must be visible that launches the INS-0312 OmniScript.

Given the FlexCard fee schedule section,
When Fee_Schedule__c.Expiration_Date__c is within 60 days,
Then the fee schedule row must display a warning icon and tooltip text 'Expiring Soon — Renewal Required'.

## Solution

Build an OmniStudio FlexCard ('Provider360_FC') with a parent DataRaptor Extract sourcing Provider__c base fields and four child card states for PNA, Fee Schedule, Credentialing, and Claims sections. Each section uses a dedicated DataRaptor with SOQL child queries. Conditional rendering via FlexCard state conditions will drive badge colors and banner visibility. The 'Initiate New Agreement' action will invoke the OmniScript built in INS-0312 via a FlexCard Action of type OmniScript. The FlexCard is surfaced on the Provider__c Lightning Record Page via a custom Lightning App Builder region, visibility filtered by Permission Set Group using component visibility rules.

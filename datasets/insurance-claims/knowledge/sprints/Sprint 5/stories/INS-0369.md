# INS-0369: Automated Network Termination Case creation when PNA Termination_Date__c is within 60 days

## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record and notify the assigned Provider Relations Specialist when a Provider_Network_Agreement__c Termination_Date__c falls within the next 60 calendar days so that the team can proactively manage network continuity, issue required member notices, and avoid compliance violations related to inadequate network access.

## Acceptance Criteria

Given a scheduled Apex job runs nightly at 02:00 UTC,
When a Provider_Network_Agreement__c record has Status__c = 'Executed' and Termination_Date__c <= (TODAY + 60) and Termination_Date__c >= TODAY,
Then a Network_Termination_Case__c is created with Termination_Trigger__c = 'Approaching Expiry', linked PNA__c populated, and Status__c = 'Open'.

Given a Network_Termination_Case__c is created,
When the record is inserted,
Then an email notification is sent to the Provider_Network_Agreement__c.OwnerId using the 'PNA_Termination_Alert' email template, including the provider NPI, LOB, and termination date.

Given a Network_Termination_Case__c already exists for a PNA in the current termination cycle,
When the nightly job re-evaluates the same PNA,
Then no duplicate Network_Termination_Case__c is created (idempotency check on PNA__c + Status__c != 'Closed').

Given a PNA Termination_Date__c is extended (updated to a future date beyond 60 days),
When the PNA record is saved,
Then any open Network_Termination_Case__c linked to that PNA is automatically closed with Closure_Reason__c = 'PNA Extended' via a Record-Triggered Flow.

## Solution

Implement a Schedulable Apex class 'PNATerminationMonitorJob' querying PNA records meeting the 60-day window using a SOQL date literal (Termination_Date__c <= NEXT_N_DAYS:60). Idempotency is enforced by a sub-query checking for existing open Network_Termination_Case__c records. Email alerts use Messaging.SingleEmailMessage with an org-wide email address. A Record-Triggered Flow on Provider_Network_Agreement__c (field: Termination_Date__c changed) handles auto-closure of stale termination cases. Job is scheduled via Setup > Apex Classes > Schedule.

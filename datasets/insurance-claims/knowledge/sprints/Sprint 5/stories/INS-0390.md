## Description

As a Provider Network Specialist, I need the system to automatically initiate a GSA renewal workflow 120 days before expiration, generate a pre-populated renewal contract document via Conga CLM, and route it for internal approval before sending to the facility so that we reduce contract lapse risk and manual document preparation effort.

GSAs govern facility-level (Facility__c / Business Account) reimbursement terms and fee schedules for institutional billing. A lapsed GSA results in claims defaulting to out-of-network rates, causing member and provider abrasion.

## Acceptance Criteria

Given a Group_Service_Agreement__c record exists with GSA.Status__c = 'Active' and GSA.Expiration_Date__c is 120 days from today,
When the nightly scheduled batch job runs,
Then GSA.Renewal_Status__c is set to 'Renewal Initiated' and a renewal Opportunity-equivalent record (GSA_Renewal_Case__c) is created linked to the parent Facility__c.

Given GSA.Renewal_Status__c = 'Renewal Initiated',
When the Conga CLM document generation step runs,
Then a pre-populated GSA renewal document is generated using the GSA_Renewal_Template__c template, merging fields: Facility__c.Name, GSA.Effective_Date__c (set to GSA.Expiration_Date__c + 1 day), GSA.Fee_Schedule__c (linked active fee schedule name), and GSA.Contracted_Rates_Table__c.

Given the Conga document is generated,
When it is attached to the GSA_Renewal_Case__c,
Then a Salesforce Approval Process (GSA_Renewal_Approval) is automatically submitted requiring sign-off from the Claims Operations Manager and Compliance Officer.

Given the Approval Process is approved by all required approvers,
When the final approval step completes,
Then a DocuSign envelope is sent to Facility__c.Primary_Contract_Email__c with the signed renewal document.

-Given the facility has an active TPA relationship, route renewal approval to the TPA Coordinator role instead.- (Deprecated: TPA routing moved to separate workflow INS-0408)

Given the GSA is not renewed and GSA.Expiration_Date__c has passed,
When the nightly batch runs,
Then GSA.Status__c is automatically set to 'Expired' and all associated Claim__c records in 'Pending' status with Claim__c.Facility__c matching the GSA facility are flagged with Claim__c.Network_Status__c = 'Out of Network – GSA Expired'.

## Solution

Implement a Schedulable Apex class (GSARenewalBatchJob.cls) running nightly at 02:00 UTC that queries GSAs expiring in exactly 120 days. Use Conga CLM REST API callout (CongaIntegrationService.cls) to trigger document generation with template ID stored in Custom Metadata (Conga_Template_Config__mdt). Salesforce Approval Process configured with multi-step parallel approvers (Claims Operations Manager + Compliance Officer). Record-Triggered Flow on GSA.Status__c change to 'Expired' updates downstream Claim__c records via a related list update loop. Store DocuSign envelope ID on GSA.DocuSign_Envelope_ID__c for audit traceability.

## Description

As a Claims Operations Manager, I need the claim adjudication screen to automatically determine and display whether the rendering provider on a Claim_Line__c is in-network or out-of-network at the time of service so that the correct Fee_Schedule__c is applied and the appropriate member cost-sharing rules are triggered, reducing manual override rates and payment errors. Extends INS-0378.

## Acceptance Criteria

Given a Claim_Line__c record is created or updated with Rendering_Provider__c (lookup to Provider__c) and Date_of_Service__c populated,
When the adjudication engine evaluates network status,
Then the system queries Provider_Network_Agreement__c records where PNA.Provider__c = Rendering_Provider__c AND PNA.Effective_Date__c <= Date_of_Service__c AND PNA.Contract_End_Date__c >= Date_of_Service__c AND PNA.Status__c = 'Active'.

Given a matching active PNA is found,
When network status is determined,
Then Claim_Line__c.Network_Status__c is set to 'In-Network' and Claim_Line__c.Applied_Fee_Schedule__c is populated with the PNA's linked Fee_Schedule__c record ID.

Given no matching active PNA is found,
When network status is determined,
Then Claim_Line__c.Network_Status__c is set to 'Out-of-Network', Claim_Line__c.Applied_Fee_Schedule__c is left null, and Claim_Line__c.OON_Override_Required__c is set to TRUE.

Given Claim_Line__c.OON_Override_Required__c = TRUE,
When a Claims Adjudicator views the claim line,
Then a prominent warning banner is displayed on the LWC adjudication component reading: 'Out-of-Network Provider – Manual rate entry or OON fee schedule required before adjudication can be completed.'

Given the provider has multiple active PNAs for different LOBs,
When the system resolves network status,
Then it filters by matching Claim__c.Line_of_Business__c = PNA.LOB__c to select the correct agreement.

-Given the provider NPI is not found in NPPES, network status defaults to 'Out-of-Network' silently.- (Deprecated: system must now log a Provider_Data_Exception__c record and alert the Provider Data Maintenance team per compliance requirement CR-2024-117)

Given Claim_Line__c.Network_Status__c is set,
When the parent Claim__c record is viewed,
Then a FlexCard summary panel displays a count of In-Network vs. Out-of-Network lines with total allowed amounts for each category.

## Solution

Apex Trigger (ClaimLineNetworkStatusTrigger, After Insert/After Update) invokes a NetworkStatusService Apex class that performs SOQL against PNA records with composite index on Provider__c, Effective_Date__c, Contract_End_Date__c, Status__c, and LOB__c. Service class sets Network_Status__c, Applied_Fee_Schedule__c, and OON_Override_Required__c fields. LWC component ('claimLineAdjudicationPanel') displays the warning banner conditionally based on OON_Override_Required__c via wire adapter. FlexCard ('ClaimNetworkSummaryCard') uses a DataRaptor Extract aggregating Claim_Line__c grouped by Network_Status__c. Custom Metadata Type (Network_Status_Config__mdt) stores LOB-to-fee-schedule fallback mappings for configuration without code changes. Permission Set Group 'Claims_Adjudicator_PSG' grants FLS on new fields.

## Description

As a Provider Network Specialist, I need a guided OmniScript that collects all required information to create a Provider_Network_Agreement__c record and routes the generated contract PDF to the provider's primary contact via DocuSign for eSignature so that the contracting workflow is fully digital, auditable, and reduces manual data entry errors.

Extends INS-0368 (DocuSign integration baseline configuration).

## Acceptance Criteria

Given the OmniScript is launched from a Provider__c record,
When Step 1 loads,
Then the script must pre-populate Provider__c.Name, Provider__c.NPI__c, Provider__c.TIN__c, and Provider__c.Primary_Specialty__c from the source record using a DataRaptor Load.

Given the specialist reaches the 'Agreement Terms' step,
When they select LOB__c and Network_Tier__c,
Then the script must dynamically display the matching active Fee_Schedule__c records (filtered by LOB and tier) in a selection table using a DataRaptor Turbo Extract.

Given all required fields are completed,
When the specialist clicks 'Generate & Send for Signature',
Then a Conga CLM merge action must generate the contract PDF using template 'PNA_Standard_Template_v3' and the DocuSign integration must route the envelope to the email address in Provider__c.SignatoryEmail__c.

Given the DocuSign envelope is sent,
When the PNA record is created,
Then Provider_Network_Agreement__c.DocuSign_Envelope_Id__c must be populated and PNA.Status__c must be set to 'Pending Signature'.

Given the provider completes eSignature,
When the DocuSign webhook callback fires,
Then PNA.Status__c must auto-update to 'Active', PNA.Execution_Date__c must be stamped, and a Chatter post must notify the assigned Provider Network Specialist.

Given the provider declines to sign,
When the DocuSign decline callback fires,
Then PNA.Status__c must be set to 'Declined' and a Task must be created for the specialist with Subject = 'Provider Declined PNA — Follow Up Required'.

Given a user without 'PNA_Create' permission set group,
When they attempt to launch the OmniScript,
Then the script must display an access denied message and not render Step 1.

## Solution

1. Build OmniScript 'PNA_Intake_OmniScript' (v1.0) with steps: Provider Info (DataRaptor Load), Agreement Terms (DataRaptor Turbo Extract for Fee Schedules), Review & Submit.
2. Integration Procedure 'PNA_CongaDocuSign_IP' orchestrates: (a) DataRaptor Transform to build Conga merge field JSON, (b) HTTP Action to Conga CLM REST API to generate PDF, (c) HTTP Action to DocuSign REST API to create and send envelope.
3. DocuSign webhook (Platform Event DocuSign_Status__e) consumed by a Record-Triggered Flow on PNA to handle status transitions and Chatter posts.
4. Permission gate implemented via an Apex Invocable Action 'CheckOmniScriptAccess' called on script load.
5. All Integration Procedure calls are logged to a custom object OmniScript_Execution_Log__c for audit.
6. Conga template 'PNA_Standard_Template_v3' managed in Conga CLM template library with dynamic fee schedule table section.

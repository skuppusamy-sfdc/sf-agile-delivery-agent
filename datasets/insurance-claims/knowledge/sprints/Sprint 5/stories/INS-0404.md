## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard embedded on the Provider__c (Person Account) record page that surfaces active Provider_Network_Agreement__c status, the count of associated Fee_Schedule__c records, network tier classification, and credentialing status, so that I can assess a provider's full contracting and participation picture without navigating to multiple related lists or running reports.

This story covers the OmniStudio FlexCard and supporting DataRaptors only. The OmniScript for PNA amendment initiation is tracked separately in INS-0408.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page,
When the Provider 360 FlexCard loads,
Then it displays the following data elements within 3 seconds of page load:
  - Provider__c.Name, NPI__c, TIN__c, Primary_Specialty__c
  - Count of Provider_Network_Agreement__c records where PNA.Status__c IN ('Active','Pending Amendment') linked to this Provider
  - Most recent Provider_Network_Agreement__c.Contract_Effective_Date__c and PNA.Expiration_Date__c
  - Provider_Network_Agreement__c.Network_Tier__c (e.g., 'Tier 1 — Preferred', 'Tier 2 — Standard')
  - Count of active Fee_Schedule__c records (Fee_Schedule__c.Expiration_Date__c >= TODAY) linked via PNA
  - Credentialing_Case__c most recent status: Credentialing_Case__c.Status__c and Credentialing_Case__c.Expiration_Date__c

Given the Provider has NO active Provider_Network_Agreement__c records,
When the FlexCard renders the contract section,
Then display a styled warning badge labeled 'No Active PNA — Out of Network' in amber and the Fee Schedule count displays '0'.

Given the Provider_Network_Agreement__c.Expiration_Date__c is within 90 days of today,
When the FlexCard renders,
Then the expiration date field is highlighted in orange with the label 'Expiring Soon' and a hyperlink 'Initiate Renewal' that navigates to the PNA record.

-Given the FlexCard displays a 'Suspend Provider' action button, when clicked it sets Provider__c.Network_Status__c = 'Suspended'- (DEPRECATED — suspension action moved to Network_Termination_Case__c workflow per compliance review; remove button from FlexCard design)

Given a Provider Relations Specialist clicks the 'View All Fee Schedules' link on the FlexCard,
When the action fires,
Then a child FlexCard panel expands inline listing each Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Rate_Amount__c, and Fee_Schedule__c.Effective_Date__c for the active PNA (maximum 50 rows; paginated).

Given the FlexCard is viewed by a user without the ProviderNetwork_ReadOnly Permission Set,
When the page loads,
Then TIN__c is masked (last 4 digits visible: ***-**-1234) and Fee_Schedule__c.Rate_Amount__c is hidden entirely.

## Solution

1. **DataRaptor Extract — DR_ProviderNetwork360**: Single turbo extract joining Provider__c, Provider_Network_Agreement__c (most recent active), Fee_Schedule__c (count via aggregate), and Credentialing_Case__c (most recent by CreatedDate). Returns flattened JSON node for FlexCard consumption.

2. **DataRaptor Transform — DR_Provider360_Transform**: Applies business logic: calculates days to PNA expiration (DATEDIFF), sets flag field `pnaExpiringSoon` (boolean) if <= 90 days, masks TIN based on running user Permission Set membership checked via Integration Procedure context variable.

3. **Integration Procedure — IP_Provider360_Load**: Orchestrates DR_ProviderNetwork360 → DR_Provider360_Transform → returns merged JSON. Caches result for 60 seconds using IP caching to reduce SOQL on high-traffic pages.

4. **FlexCard — FC_Provider360**: Built in OmniStudio FlexCard Designer. Card States: 'Active PNA' (green header), 'No Active PNA' (amber warning state), 'Expiring Soon' (orange highlight state). Child FlexCard FC_FeeScheduleList handles the expandable fee schedule panel with a separate DataRaptor Extract scoped to Fee_Schedule__c by PNA Id. Uses FlexCard Action of type 'Navigation' for 'Initiate Renewal' link.

5. **TIN Masking**: Conditional visibility on FlexCard field using `{hasProviderNetworkReadOnly}` boolean from Integration Procedure context (resolved via Apex Remote Action checking PermissionSetAssignment at IP runtime).

6. **Lightning App Builder**: FlexCard deployed as a custom Lightning component on the Provider__c (Person Account) record page, positioned in the right-hand panel. Visibility rule: RecordType = 'Healthcare Provider'.

7. **QA**: Test in sandbox with providers having 0, 1, and 3+ PNA records; validate masking with two user profiles.

## Description

As a Provider Network Specialist, I need to associate one or more Fee_Schedule__c records to an active Provider_Network_Agreement__c and map specific CPT codes with contracted reimbursement rates, so that the Claims Payment Engine can accurately adjudicate Claim_Line__c items against the correct contracted rate during payment processing.

## Acceptance Criteria

Given an active Provider_Network_Agreement__c record,
When the user navigates to the 'Fee Schedules' related list,
Then they can add one or more Fee_Schedule__c records, each with Fee_Schedule__c.LOB__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.Expiration_Date__c populated.

Given a Fee_Schedule__c is linked to a PNA,
When the user opens the Fee Schedule detail,
Then they can add Fee_Schedule_Line__c child records with fields: CPT_Code__c (validated against a custom CPT Master metadata table), Modifier__c, Allowed_Amount__c (currency), and Rate_Type__c ('Percent of Billed', 'Flat Rate', 'Percent of Medicare').

Given Fee_Schedule_Line__c.Rate_Type__c = 'Percent of Medicare',
When Allowed_Amount__c is entered,
Then the system validates that Allowed_Amount__c is between 0.01 and 999.99 (percent) and displays a helper text 'Enter as a percentage, e.g., 110 for 110% of Medicare rate'.

Given a Claim_Line__c is being adjudicated with Claim_Line__c.CPT_Code__c and the rendering provider has an active PNA with a matching Fee_Schedule_Line__c,
When the Payment Engine invokes the fee lookup,
Then Claim_Line__c.Contracted_Rate__c is populated with the matched Fee_Schedule_Line__c.Allowed_Amount__c and Claim_Line__c.Fee_Schedule__c is stamped with the source Fee_Schedule__c record ID.

Given two overlapping Fee_Schedule__c records exist for the same PNA and LOB,
When the effective date ranges overlap,
Then a validation error is thrown: 'Overlapping fee schedule date ranges are not permitted for the same LOB on a single PNA.'

Given Fee_Schedule__c.Expiration_Date__c < TODAY(),
When a Claim_Line__c adjudication attempts to reference the expired schedule,
Then the Payment Engine falls back to the default network fee schedule and logs a warning on the Claim__c record: Claim__c.Fee_Schedule_Warning__c = 'Expired fee schedule; default applied'.

## Solution

Fee_Schedule__c and Fee_Schedule_Line__c custom objects with a master-detail relationship to Provider_Network_Agreement__c. CPT code validation uses a Custom Metadata Type (CPT_Master__mdt) loaded from CMS CPT reference data via a scheduled DataRaptor integration. Apex class 'FeeScheduleLookupService' is invoked by the Payment Engine via an Apex invocable action to resolve contracted rates during Claim_Line__c adjudication, with fallback logic and warning stamping. Overlapping date validation is enforced by an Apex Trigger (before insert/update) on Fee_Schedule__c. LWC 'feeScheduleLineEditor' provides an inline editable grid for CPT code entry on the Fee Schedule detail page.

## Description

As a Provider Network Specialist, I need the system to automatically identify Provider_Network_Agreement__c records approaching their termination date (within 90, 60, and 30 days) and trigger a structured renewal workflow including notifications to the assigned specialist and the provider, so that contract lapses are prevented and network continuity is maintained. This story also implements the PNA auto-void logic for unsigned contracts descoped from INS-0380.

## Acceptance Criteria

Given a scheduled Apex job runs nightly,
When PNA.Status__c = 'Active' and PNA.Termination_Date__c is within 90 calendar days of TODAY(),
Then PNA.Renewal_Alert_Stage__c is set to '90-Day Warning' and a Task is created assigned to PNA.Owner with Subject = 'PNA Renewal Required — 90 Days Remaining: [Provider Name]'.

Given PNA.Renewal_Alert_Stage__c = '90-Day Warning' and no renewal action has been taken,
When PNA.Termination_Date__c is within 60 days,
Then PNA.Renewal_Alert_Stage__c updates to '60-Day Warning' and an escalation email is sent to the Provider Network Specialist's manager using an Email Alert.

Given PNA.Renewal_Alert_Stage__c = '60-Day Warning' and no renewal action has been taken,
When PNA.Termination_Date__c is within 30 days,
Then PNA.Renewal_Alert_Stage__c = '30-Day Warning', Claim__c records referencing this provider are flagged with Claim__c.PNA_Expiry_Risk__c = TRUE for claims with Date_of_Service__c beyond the PNA.Termination_Date__c.

Given a PNA.Status__c = 'Pending Signature' (contract sent but not yet executed),
When PNA.Contract_Sent_Date__c + 30 days < TODAY(),
Then the nightly job sets PNA.Status__c = 'Void — Unsigned' and logs a Chatter post on the PNA record: '@[Owner] Contract auto-voided due to no signature received within 30 days.'

Given the Provider Network Specialist clicks 'Initiate Renewal' on a PNA in any warning stage,
When the action is triggered,
Then a new draft PNA is cloned from the expiring record with PNA.Status__c = 'Draft', PNA.Effective_Date__c = expiring PNA.Termination_Date__c + 1, and PNA.Parent_PNA__c = expiring PNA.Id.

## Solution

Scheduled Apex class 'PNARenewalMonitorJob' implements Schedulable and Batchable interfaces, processing PNA records in batches of 200. Job is scheduled via custom Settings (Scheduled_Job_Config__mdt) to run at 02:00 AM daily. Renewal_Alert_Stage__c is a picklist field added to Provider_Network_Agreement__c. Escalation email uses Workflow Email Alert targeting Manager field via User relationship. Claim flagging uses a SOQL query joining Claim__c to Provider__c to PNA within the batch. 'Initiate Renewal' Quick Action triggers a Record-Triggered Flow that clones the PNA using Apex 'PNACloneService.cloneForRenewal()'. Chatter post on auto-void uses ConnectAPI in Apex. Custom Metadata Type 'PNA_Renewal_Config__mdt' stores threshold days (90/60/30) to allow configuration without code changes.

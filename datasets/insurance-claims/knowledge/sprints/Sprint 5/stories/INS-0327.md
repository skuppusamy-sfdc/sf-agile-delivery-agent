## Description

As a Provider Network Specialist, I need a guided OmniScript to create a Provider_Network_Agreement__c record and associate one or more Fee_Schedule__c records to it, so that contracted rates are captured accurately at the time of network onboarding and downstream claim adjudication can reference the correct fee schedule without manual lookup.

## Acceptance Criteria

Given a Provider Network Specialist navigates to the PNA creation OmniScript on a Provider__c (Person Account) record,
When they complete Step 1 (Contract Details), Then Provider_Network_Agreement__c.Contract_Start_Date__c, Provider_Network_Agreement__c.Contract_End_Date__c, Provider_Network_Agreement__c.LOB__c (picklist: Commercial, Medicare Advantage, Medicaid), and Provider_Network_Agreement__c.TIN__c must all be required fields before advancing.

Given the specialist selects LOB__c = 'Medicare Advantage',
When Step 2 (Fee Schedule) loads, Then only Fee_Schedule__c records where Fee_Schedule__c.LOB__c = 'Medicare Advantage' and Fee_Schedule__c.Effective_Date__c <= TODAY() and Fee_Schedule__c.Expiration_Date__c >= TODAY() are displayed in the lookup datatable.

Given a Fee_Schedule__c record is selected,
When the specialist clicks 'Associate',
Then a junction record PNA_Fee_Schedule__c is created with PNA_Fee_Schedule__c.Provider_Network_Agreement__c = [current PNA] and PNA_Fee_Schedule__c.Fee_Schedule__c = [selected record] and PNA_Fee_Schedule__c.Association_Date__c = TODAY().

Given the OmniScript is submitted,
When Provider_Network_Agreement__c.Status__c transitions from 'Draft' to 'Pending Signature',
Then a DocuSign envelope is automatically sent to the provider's email captured in Provider__c.Email using the 'PNA Standard Template v2' envelope template.

-Given the contract is countersigned, Then Provider_Network_Agreement__c.Status__c should auto-advance to 'Active' via a scheduled batch job running nightly- (deprecated: replaced by DocuSign Connect webhook Platform Event in INS-0331)

Given Provider_Network_Agreement__c.Status__c = 'Active',
When a Claim_Line__c is adjudicated referencing the same Provider__c and LOB__c,
Then Claim_Line__c.Fee_Schedule_ID__c must be populated with the associated Fee_Schedule__c.Id.

## Solution

Implement a multi-step OmniScript (NetworkContracting/PNA_Creation_OS) with DataRaptor Extract steps to query active Fee_Schedule__c records filtered by LOB. Use a DataRaptor Transform to map OmniScript output to Provider_Network_Agreement__c fields. On final Submit action, invoke an Apex REST callout action to the DocuSign eSignature API to generate the envelope. A Platform Event (PNA_Status_Change__e) published by the DocuSign Connect webhook triggers a Record-Triggered Flow on Provider_Network_Agreement__c to set Status__c = 'Active' and stamp Active_Date__c = TODAY(). Permission Set Group 'PNS_Contracting_PSG' gates access to the OmniScript and related objects.

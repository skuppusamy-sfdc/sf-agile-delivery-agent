## Description

As a Provider Network Specialist, I need a guided OmniScript wizard to create and configure a Group_Service_Agreement__c record for a new employer group so that all required contracting, billing, and LOB attributes are captured consistently before the group is activated on the platform. The wizard must validate TIN uniqueness and link to an existing Broker Relations Coordinator's contact record where applicable.

## Acceptance Criteria

Given a Provider Network Specialist launches the 'New GSA Wizard' OmniScript from the Group_Service_Agreement__c list view,
When the wizard completes all steps,
Then a Group_Service_Agreement__c record is created with Status__c = 'Pending Approval', Group_TIN__c populated, and Effective_Date__c set.

Given a TIN entered in the wizard already exists on another Group_Service_Agreement__c with Status__c != 'Terminated',
When the specialist proceeds past Step 1,
Then a warning banner displays 'Active GSA exists for TIN [value] — GSA-XXXX' and blocks progression until acknowledged by a Compliance Officer role.

Given a Broker Relations Coordinator is associated with the group,
When the specialist searches and selects a Contact in the wizard,
Then Group_Service_Agreement__c.Broker_Contact__c is populated and a Task is auto-created for the broker to review the agreement within 5 business days.

Given the GSA wizard is submitted,
When Status__c transitions from 'Pending Approval' to 'Active',
Then the associated Policy__c records linked via Group_Policy_Junction__c are updated to reflect the new GSA and a Platform Event 'GSA_Activated__e' is published for downstream MCO integrations.

## Solution

OmniScript 'GSA_Onboarding_OS' with four steps: Group Details, LOB Selection, Broker Assignment, and Review & Submit. DataRaptor Extract steps handle TIN duplicate detection. A Record-Triggered Flow on Group_Service_Agreement__c publishes the 'GSA_Activated__e' Platform Event on status transition to 'Active'. An Apex invocable action handles Task creation for broker review. Conga CLM integration generates the GSA document for wet or e-signature via a connected app callout.

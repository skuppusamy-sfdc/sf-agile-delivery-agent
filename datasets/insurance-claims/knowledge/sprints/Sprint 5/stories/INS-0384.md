## Description

As a Provider Relations Specialist, I need a comprehensive Provider 360 FlexCard on both Provider__c (Person Account) and Facility__c (Business Account) record pages that consolidates active PNAs, associated fee schedules, current credentialing status from the Credentialing_Case__c, open and recent claims, and any active Provider_Change_Notification__c records, so that I can manage provider relationships from a single view without navigating across multiple related lists.

## Acceptance Criteria

Given a Provider__c record page is loaded,
When the Provider 360 FlexCard renders,
Then it displays the following tabs: 'Contracting' (active PNAs), 'Credentialing' (latest Credentialing_Case__c status), 'Claims' (last 10 Claim__c records by submitted date), 'Notifications' (open PCN records).

Given the 'Contracting' tab is active,
When PNA data is loaded via DataRaptor Extract,
Then each active PNA row shows: PNA.Network_Tier__c, PNA.LOB__c, PNA.Effective_Date__c, PNA.Termination_Date__c, and a badge indicator (green = 'Active', yellow = 'Expiring within 90 days', red = 'Expired').

Given the 'Credentialing' tab is active,
When the DataRaptor loads the most recent Credentialing_Case__c linked to the Provider__c,
Then it displays: Credentialing_Case__c.Status__c, Credentialing_Case__c.CAQH_Profile_ID__c, Credentialing_Case__c.Expiration_Date__c, and a 'Re-credential' action button visible only if Status__c = 'Active' and Expiration_Date__c <= TODAY() + 90.

Given the 'Claims' tab is active,
When the claims DataRaptor runs,
Then Claim__c records are displayed with: Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, Claim__c.Total_Paid_Amount__c, and Claim__c.Date_of_Service__c, sorted descending by Date_of_Service__c.

Given the 'Notifications' tab is active,
When PCN data loads,
Then open Provider_Change_Notification__c records show: PCN.Type__c, PCN.Submitted_Date__c, PCN.Status__c, and a hyperlink to the PCN record detail.

-Tab 'Billing' showing ERA history to be included in this card- (descoped to INS-0391 ERA FlexCard story)

## Solution

OmniStudio FlexCard 'Provider_360_Card' with 4 child FlexCards (tabbed layout): 'Provider_Contracting_Card', 'Provider_Credentialing_Card', 'Provider_Claims_Card', 'Provider_PCN_Card'. Each child card uses a dedicated DataRaptor Extract for its data source. Conditional badge coloring on PNA expiry implemented via FlexCard State Actions with date arithmetic. 'Re-credential' action launches OmniScript 'Initiate_Recredentialing_OS'. FlexCard is activated on both Person Account and Business Account record pages via App Builder. DataRaptor Turbo Extract used for Claims tab to optimize SOQL performance on large claim volumes. Visibility of action buttons controlled by FlexCard conditional rendering tied to user Permission Set Group membership.

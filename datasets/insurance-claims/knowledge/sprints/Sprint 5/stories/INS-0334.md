## Description

As a Claims Operations Manager, I need the Agentforce AI assistant embedded in the FNOL intake OmniScript to suggest active in-network Provider__c records based on the claimant's location and the claim's LOB, so that claims are routed to contracted providers from the start, reducing out-of-network leakage and manual provider assignment effort during high-volume FNOL periods.

## Acceptance Criteria

Given a claims agent is on Step 3 (Provider Assignment) of the FNOL OmniScript and has entered the claimant's ZIP code and Claim__c.LOB__c,
When the Agentforce 'Find In-Network Provider' action is triggered,
Then the AI agent queries Provider__c records where the associated Provider_Network_Agreement__c.Status__c = 'Active' AND Provider_Network_Agreement__c.LOB__c = Claim__c.LOB__c AND Provider__c.Service_ZIP__c is within 25 miles of the claimant ZIP code.

Given the Agentforce query returns results,
When the suggestions panel renders,
Then up to 5 Provider__c records are displayed showing Provider__c.Name, Provider__c.Specialty__c, Provider__c.NPI__c, Provider__c.Phone, and a calculated distance field (miles from claimant ZIP).

Given the agent selects a suggested provider,
When the selection is confirmed,
Then Claim__c.Assigned_Provider__c is populated with the selected Provider__c.Id and Claim__c.Provider_Assignment_Source__c = 'Agentforce AI' for tracking and reporting.

-Given no in-network providers are found within 25 miles, Then the Agentforce panel should display an error and halt FNOL progression- (deprecated: replaced by expanded radius fallback to 50 miles before showing 'No In-Network Providers' message)

Given no in-network providers are found within 50 miles,
When the fallback search also returns zero results,
Then the Agentforce panel displays: 'No in-network providers found within 50 miles. Claim flagged for manual provider assignment.' and Claim__c.Manual_Assignment_Required__c = TRUE.

Given Claim__c.Provider_Assignment_Source__c = 'Agentforce AI',
When the claim is closed,
Then the outcome is logged to an Agentforce_Feedback__c custom object with fields: Feedback_Type__c, Claim__c, Provider__c, Outcome__c for model retraining purposes.

## Solution

Agentforce Agent Action configured with a custom Apex action class (AgentforceProviderSearchAction) that accepts ZIP code and LOB as inputs, performs a SOQL geo-distance query on Provider__c using Salesforce Geolocation fields (Provider__c.Location__Latitude__s, Provider__c.Location__Longitude__s) via DISTANCE() and GEOLOCATION() functions, filtered against active PNA records. Results returned as a JSON list to the Agentforce reasoning layer. OmniScript Step 3 embeds an Agentforce component (agentforce-conversation-kit) in a custom LWC panel. Agentforce_Feedback__c logging handled by a Record-Triggered Flow on Claim__c firing on Status__c = 'Closed'. Einstein Generative AI model grounding uses the Fee_Schedule__c and Provider_Network_Agreement__c Data Cloud data streams configured in the Acme Insurance Data Cloud org.

## Description

As a Provider Network Specialist, I need the Fee_Schedule__c object to support CPT code-level rate lookups tied to a Provider_Network_Agreement__c so that Claim_Line__c adjudication can automatically resolve the contracted reimbursement rate without manual intervention.

Background: Currently, fee schedule rates are maintained in spreadsheets and manually keyed into claims. This sprint establishes the Salesforce-native Fee Schedule Engine that will serve as the authoritative rate source for all in-network adjudication. Extends INS-0305 (PNA data model foundation).

## Acceptance Criteria

Given a Fee_Schedule__c record is linked to an active Provider_Network_Agreement__c (PNA__c.Status__c = 'Active'),
When a Claim_Line__c is created with a valid CPT__c code and Service_Date__c falls within Fee_Schedule__c.Effective_Start_Date__c and Fee_Schedule__c.Effective_End_Date__c,
Then Fee_Schedule__c.Contracted_Rate__c is populated on Claim_Line__c.Contracted_Rate__c and Claim_Line__c.Rate_Source__c = 'Fee Schedule'.

Given a CPT code on Claim_Line__c does not match any Fee_Schedule__c entry for the associated PNA,
When adjudication logic attempts rate resolution,
Then Claim_Line__c.Rate_Source__c = 'Unlisted' and Claim_Line__c.Requires_Manual_Review__c = TRUE and a Task is created assigned to the Claims Operations Manager queue.

Given a Fee_Schedule__c record has Fee_Schedule__c.LOB__c populated,
When the Claim__c.Line_of_Business__c does not match Fee_Schedule__c.LOB__c,
Then the rate lookup is skipped for that fee schedule entry and the next applicable entry is evaluated.

Given multiple Fee_Schedule__c records exist for the same CPT code under a single PNA,
When rate resolution runs,
Then the record with the most recent Fee_Schedule__c.Effective_Start_Date__c is applied and a warning log entry is written to Integration_Log__c.

-Given Fee_Schedule__c.Modifier__c is null, the base CPT rate applies to all modifier variants- (deprecated — modifier matching now handled by INS-0341).

Given a Provider is a Facility (Account.RecordType = 'Facility'),
When rate lookup executes,
Then Fee_Schedule__c.Facility_Rate__c is used instead of Fee_Schedule__c.Provider_Rate__c.

## Solution

Implement an Apex invocable method `FeeScheduleRateLookupService` called from a Record-Triggered Flow on Claim_Line__c (after-insert, after-update on CPT__c or Service_Date__c fields). The service queries Fee_Schedule__c with indexed fields (PNA__c, CPT_Code__c, Effective_Start_Date__c, Effective_End_Date__c) using a SOQL query with date range filtering. LOB and facility/provider type branching handled within the Apex class. Unlisted CPT results publish a `ClaimLineReviewRequired__e` Platform Event consumed by a separate Flow that creates the review Task. Fee schedule data loaded via DataRaptor Turbo Extract for performance on high-volume adjudication batches. Permission Set Group 'Fee_Schedule_Admin_PSG' controls write access to Fee_Schedule__c.

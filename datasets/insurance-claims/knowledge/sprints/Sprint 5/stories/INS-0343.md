## Description

As a Provider Network Specialist, I need a guided OmniScript wizard to create and send Provider_Network_Agreement__c records for e-signature via DocuSign so that contract execution turnaround time is reduced from 14 days to under 48 hours and all agreement metadata is captured in Salesforce at the point of initiation.

Background: Currently, PNA creation is a manual, email-driven process with no audit trail in Salesforce. This story establishes the foundational contracting workflow for Phase 1 LOBs (Commercial HMO, PPO).

## Acceptance Criteria

Given a Provider Relations Specialist navigates to a Provider__c (Person Account) record with NPI verified and Credentialing_Case__c.Status__c = 'Approved',
When they launch the 'New Network Agreement' OmniScript from the Provider 360 FlexCard action menu,
Then a new Provider_Network_Agreement__c record is created in Status__c = 'Draft' with Provider__c, TIN__c, NPI__c, Effective_Date__c, and LOB__c pre-populated from the Provider__c record.

Given the PNA wizard step 2 is displayed,
When the user selects Fee_Schedule__c from a filtered lookup (active schedules matching LOB__c and Network_Tier__c),
Then Fee_Schedule__c is stamped on PNA__c.Fee_Schedule_ID__c and a read-only rate summary panel renders below the lookup showing CPT code ranges and reimbursement percentages.

Given all required wizard steps are complete and PNA__c.Status__c = 'Pending Signature',
When the user clicks 'Send for Signature',
Then a DocuSign envelope is created via the DocuSign for Salesforce managed package, the envelope ID is stored in PNA__c.DocuSign_Envelope_ID__c, and the provider's email on Provider__c.Email receives the DocuSign request within 5 minutes.

Given the provider completes DocuSign e-signature,
When the DocuSign webhook callback fires,
Then PNA__c.Status__c transitions to 'Active', PNA__c.Executed_Date__c is stamped with the completion timestamp, and a Salesforce Platform Event (PNA_Executed__e) is published to notify downstream Fee Schedule activation.

-Given the wizard is accessed from a Facility__c (Business Account) record, the same PNA flow applies with Group_Service_Agreement__c auto-linked.- (Superseded — GSA wizard handled in INS-0347)

Given PNA__c.Status__c = 'Active',
When a Claims Processor attempts to submit a Claim__c referencing this Provider__c,
Then Claim__c.Network_Status__c auto-populates as 'In-Network' via a Record-Triggered Flow on Claim__c insert.

## Solution

1. OmniScript (Type: PNA_Creation, Sub-Type: Wizard) built in OmniStudio with 5 steps: Provider Lookup, LOB & Tier Selection, Fee Schedule Assignment, Contract Terms Review, Signature Dispatch.
2. DataRaptor Transform to pre-populate wizard fields from Provider__c and active Credentialing_Case__c.
3. Integration Procedure calling DocuSign REST API via Named Credential (DocuSign_Prod) to create envelope and attach Conga CLM-generated PDF template.
4. Record-Triggered Flow on Provider_Network_Agreement__c (after update, Status__c changed to 'Active') publishes PNA_Executed__e Platform Event.
5. Separate Record-Triggered Flow on Claim__c (before insert) queries active PNA via SOQL and sets Claim__c.Network_Status__c.
6. Permission Set Group 'Provider_Network_Contracting_PSG' grants OmniScript runtime + PNA object CRUD to Provider Relations Specialist profile.

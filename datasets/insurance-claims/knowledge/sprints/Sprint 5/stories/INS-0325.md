## Description

As a Credentialing Specialist, I need the Salesforce Credentialing_Case__c record to automatically pull and display credential verification data from CAQH ProView via API so that I can review board certifications, malpractice history, DEA license, and education credentials without manually logging into the CAQH portal. This reduces credentialing cycle time and ensures data accuracy by eliminating manual re-entry. The integration must use the provider's CAQH_Provider_ID__c stored on Provider__c.

## Acceptance Criteria

Given a Credentialing_Case__c record is in Status__c = 'Initial Review',
When the Credentialing Specialist clicks 'Fetch CAQH Data',
Then a callout is made to the CAQH ProView REST API using the linked Provider__c.CAQH_Provider_ID__c.

Given the CAQH API returns a successful response (HTTP 200),
When data is received,
Then the following Credentialing_Case__c fields are populated:
- Credentialing_Case__c.Board_Certification__c
- Credentialing_Case__c.DEA_License_Number__c
- Credentialing_Case__c.DEA_Expiration_Date__c
- Credentialing_Case__c.Malpractice_Coverage_Amount__c
- Credentialing_Case__c.CAQH_Last_Attested_Date__c
- Credentialing_Case__c.CAQH_Data_Retrieved_Datetime__c = NOW()

Given the CAQH API returns an error (HTTP 401, 404, or 500),
When the callout fails,
Then Credentialing_Case__c.CAQH_Integration_Status__c = 'Failed' and an error log entry is created in Integration_Log__c with the HTTP status code and response body.

Given CAQH data is retrieved,
When Credentialing_Case__c.DEA_Expiration_Date__c is in the past,
Then a validation warning is displayed: 'DEA license is expired. Credentialing cannot be approved until renewed.'

Given the provider does not have a CAQH_Provider_ID__c on their Provider__c record,
When the 'Fetch CAQH Data' button is clicked,
Then an error message is displayed: 'CAQH Provider ID is required. Please update the Provider record before proceeding.'

-Given NPPES NPI Registry data is also fetched in the same callout sequence-
CAQH and NPPES integrations are separate callouts; NPPES integration covered in INS-0312.

## Solution

Build an Apex class (CAQHIntegrationService.cls) using Named Credentials ('CAQH_ProView_NC') for secure API authentication. The class is invoked via an LWC quick action button ('Fetch CAQH Data') on the Credentialing_Case__c record page. Async callout executed via a Queueable Apex job to avoid Visualforce/LWC callout restrictions. Response JSON is deserialized using a custom wrapper class (CAQHResponseWrapper.cls). Integration_Log__c records created via a utility logging class (IntegrationLogger.cls). Remote Site Settings and CSP Trusted Sites configured for CAQH ProView endpoint. Connected App credentials stored in Custom Metadata Type (CAQH_Integration_Settings__mdt). Retry logic (max 3 attempts) implemented in the Queueable job.

## Description

As a Provider Network Specialist, I need the creation of a Network_Termination_Case__c record to trigger an automated workflow that flags the associated provider as out-of-network, terminates linked PNA records, and generates a Claims Impact Report identifying all open Claim__c records that reference the terminated provider, so that Claims Operations can take timely action to avoid incorrect in-network payment after termination.

Compliance requirement: Per state regulation, providers must receive a 30-day advance termination notice (PCN) before the effective termination date. This workflow must enforce that notice period.

## Acceptance Criteria

Given a Network_Termination_Case__c record is created with Termination_Case__c.Status__c = 'Initiated' and Termination_Case__c.Termination_Effective_Date__c populated,
When the record is saved,
Then a Provider_Change_Notification__c (PCN) is automatically created with PCN__c.Notification_Type__c = 'Termination Notice', PCN__c.Notice_Date__c = TODAY(), and PCN__c.Required_Response_By__c = Termination_Effective_Date__c - 30 days.

Given Termination_Case__c.Termination_Effective_Date__c is less than 30 days from today,
When the record is saved,
Then a validation error is thrown: 'Termination Effective Date must be at least 30 days from today to comply with provider notice requirements. Please update Termination_Effective_Date__c.'

Given a Network_Termination_Case__c reaches Status__c = 'Approved' and today >= Termination_Effective_Date__c,
When the nightly termination batch job runs,
Then all linked Provider_Network_Agreement__c records have Status__c set to 'Terminated', Provider__c.Network_Status__c = 'Out of Network', and Provider__c.Out_of_Network_Date__c = Termination_Effective_Date__c.

Given the provider is flagged Out of Network,
When the Claims Impact Report is generated,
Then a Report__c (or CSV attachment) is created listing all Claim__c records where Claim__c.Rendering_Provider__c = terminated provider AND Claim__c.Status__c IN ('Submitted', 'In Adjudication', 'Pending Information') with columns: Claim__c.Name, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, Policy__c.Policy_Number__c, Claim__c.Service_Date__c.

Given the Claims Impact Report is generated,
When it contains 1 or more affected claims,
Then a Task is created for each open Claim assigned to the Claims Operations Manager queue with Subject = 'Provider Terminated — Review Claim for Out-of-Network Reprocessing' and the Termination Case linked in Task.WhatId.

Given Provider__c.Network_Status__c = 'Out of Network',
When a new Claim__c is submitted with Claim__c.Rendering_Provider__c = this provider,
Then Claim__c.In_Network__c = FALSE and Claim__c.Out_of_Network_Warning__c = TRUE and the claim routes to the out-of-network adjudication queue.

-Given a termination is initiated, all future appointments for the provider in QGenda are automatically cancelled- (deprecated — QGenda integration deferred to Phase 2 per architecture decision ADR-014).

## Solution

Record-Triggered Flow on Network_Termination_Case__c (after-insert) creates the PCN record. Before-save Flow enforces the 30-day validation using a formula comparing Termination_Effective_Date__c to TODAY(). Nightly Schedulable Apex batch `ProviderTerminationBatchJob` queries Network_Termination_Case__c WHERE Status__c = 'Approved' AND Termination_Effective_Date__c <= TODAY() and processes PNA termination and Provider flag updates in bulk (Database.Batchable, batch size 50 to stay within DML limits given child record updates). Claims Impact Report generated via a separate Apex class `TerminationClaimsImpactReportService` invoked as a post-processing step in the batch's finish() method — queries open claims and creates ContentVersion CSV attachment on the Termination Case. Task creation for impacted claims uses a Platform Event `ProviderTerminationImpact__e` published per affected claim and consumed by a Flow-triggered process to avoid hitting DML limits in the batch context. New Claim__c validation handled by a Record-Triggered Flow (before-insert) that queries Provider__c.Network_Status__c. Sharing Rules ensure Network_Termination_Case__c records are visible to Claims Operations Manager role and above.

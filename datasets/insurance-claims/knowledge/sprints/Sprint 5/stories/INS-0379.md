## Description

As a Claims Operations Manager, I need the claims adjudication engine to automatically look up and apply the correct Fee_Schedule__c rate to each Claim_Line__c based on the rendering provider's active PNA and the Claim__c.Date_of_Service__c so that Claim_Line__c.Allowed_Amount__c is accurately calculated and EOB line-level detail reflects the correct contracted rate.

## Acceptance Criteria

Given a Claim_Line__c record where Claim_Line__c.Status__c = 'Pending Adjudication',
When the adjudication engine processes the line,
Then the engine must identify the Provider_Network_Agreement__c linked to Claim__c.Rendering_Provider__c where PNA.Status__c = 'Active' AND PNA.Effective_Date__c <= Claim__c.Date_of_Service__c AND PNA.Expiration_Date__c >= Claim__c.Date_of_Service__c.

Given a valid PNA is found,
When the engine retrieves the Fee Schedule,
Then it must select the Fee_Schedule__c record matching Fee_Schedule__c.CPT_Code__c = Claim_Line__c.CPT_Code__c AND Fee_Schedule__c.LOB__c = Claim__c.LOB__c with the most recent Fee_Schedule__c.Effective_Date__c that is <= Claim__c.Date_of_Service__c.

Given the matching Fee_Schedule__c is found,
When the allowed amount is calculated,
Then Claim_Line__c.Allowed_Amount__c = Fee_Schedule__c.Rate_Amount__c, Claim_Line__c.Fee_Schedule__c (lookup) must be stamped with the matched record Id, and Claim_Line__c.Adjudication_Method__c = 'Fee Schedule'.

Given no matching Fee_Schedule__c is found for the CPT code,
When the engine cannot resolve a rate,
Then Claim_Line__c.Status__c must be set to 'Pended — Fee Schedule Missing', Claim_Line__c.Pend_Reason__c = 'No contracted rate found for CPT: {CPT_Code__c}', and a Task must be created for the Claims Operations queue.

Given the rendering provider has no active PNA on the date of service,
When the engine evaluates network status,
Then Claim_Line__c.Network_Status__c must be set to 'Out of Network' and Claim_Line__c.Adjudication_Method__c = 'UCR' (Usual, Customary and Reasonable).

Given Claim_Line__c.Allowed_Amount__c is set,
When the claim line adjudication completes,
Then Claim_Line__c.Patient_Responsibility__c = Claim_Line__c.Billed_Amount__c - Claim_Line__c.Allowed_Amount__c (minimum zero) and Claim_Line__c.Status__c = 'Adjudicated'.

Given all Claim_Line__c records on a Claim__c are 'Adjudicated',
When the parent Claim__c is evaluated,
Then Claim__c.Status__c must auto-update to 'Ready for Payment' via a Roll-Up Summary or Record-Triggered Flow.

## Solution

1. Apex class 'FeeScheduleAdjudicationService' with method 'adjudicateClaimLine(Id claimLineId)' encapsulates all lookup logic.
2. SOQL with date-range filtering on PNA and Fee_Schedule__c uses indexed fields (Rendering_Provider__c, LOB__c, CPT_Code__c) with selective queries to avoid governor limits.
3. Apex Trigger 'ClaimLineTrigger' (after update, on Status__c = 'Pending Adjudication') invokes FeeScheduleAdjudicationService via Queueable Apex to handle bulk claim processing asynchronously.
4. UCR fallback rate retrieved from a custom metadata type 'UCR_Rate__mdt' keyed by CPT code and geographic region (Claim__c.Service_State__c).
5. Pend Task creation uses a separate Queueable to avoid mixed DML errors.
6. Record-Triggered Flow on Claim__c monitors all child Claim_Line__c statuses via a roll-up summary field Claim__c.Open_Lines_Count__c; when = 0, transitions Claim__c.Status__c.
7. FeeScheduleAdjudicationServiceTest achieves >85% coverage with bulk test (200 claim lines), no-PNA scenario, no-fee-schedule scenario, and UCR fallback.

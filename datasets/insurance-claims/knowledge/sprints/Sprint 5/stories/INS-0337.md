## Description

As a Provider Relations Specialist, I need Provider_Network_Agreement__c records in 'Pending Signature' status to automatically trigger a DocuSign envelope routed to the provider's designated signatory so that PNA execution is fully digital, auditable, and does not require manual PDF handling or email coordination outside of Salesforce.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record transitions to Status__c = 'Pending Signature',
When the Record-Triggered Flow fires,
Then a DocuSign envelope is created using the mapped PNA contract template and sent to the email address on Provider__c.Signatory_Email__c within 5 minutes.

Given a DocuSign envelope is sent,
When the provider completes signing,
Then PNA__c.Status__c automatically updates to 'Active', PNA__c.Execution_Date__c is populated with the signature timestamp, and the executed PDF is stored as a Salesforce File (ContentDocument) linked to the PNA record.

Given a DocuSign envelope is sent,
When the envelope expires (after 14 days without signature),
Then PNA__c.Status__c reverts to 'Draft', a Task is created for the assigned Provider Relations Specialist with Subject = 'PNA DocuSign Envelope Expired — Action Required', and a notification is sent via Salesforce Bell notification.

Given a Provider Relations Specialist needs to void a sent envelope,
When they click the 'Void Envelope' quick action on the PNA record,
Then the DocuSign envelope is voided via API call and PNA__c.DocuSign_Envelope_Status__c = 'Voided' with a void reason captured in PNA__c.Void_Reason__c.

-Given the provider has no Signatory_Email__c, the envelope falls back to Provider__c.Email- (deprecated — validation now blocks PNA from reaching Pending Signature without Signatory_Email__c populated per INS-0336).

## Solution

Leverage DocuSign for Salesforce managed package (DocuSign eSignature). Configure a DocuSign Template mapped to Provider_Network_Agreement__c fields using DocuSign Template Mapping. Record-Triggered Flow (after-update, Status__c changed to 'Pending Signature') invokes an Apex action `DocuSignEnvelopeLaunchService` that calls the DocuSign REST API via Named Credential. DocuSign Connect webhook callback updates PNA__c status and attaches the completed document via ContentVersion insert in an Apex REST endpoint. Custom quick action 'Void Envelope' triggers an Apex invocable method. All API callouts handled asynchronously via Queueable Apex to avoid governor limit conflicts.

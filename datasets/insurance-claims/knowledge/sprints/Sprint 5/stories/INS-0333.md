## Description

As a Provider Network Specialist, I need an automated workflow on the Network_Termination_Case__c object that generates a Provider_Change_Notification__c (PCN) and enforces the 90-day advance notice requirement, so that Acme Insurance remains compliant with state regulatory mandates and providers receive timely written notification of network termination.

## Acceptance Criteria

Given a Network_Termination_Case__c is created with Network_Termination_Case__c.Termination_Reason__c populated and Network_Termination_Case__c.Requested_Termination_Date__c set,
When the record is saved,
Then the system validates that Requested_Termination_Date__c >= TODAY() + 90; if not, a validation error fires: 'Termination date must be at least 90 days from today to comply with notice requirements.'

Given Network_Termination_Case__c.Status__c is set to 'Approved',
When the approval is recorded,
Then a Provider_Change_Notification__c is auto-created with PCN.Type__c = 'Network Termination', PCN.Provider__c = Network_Termination_Case__c.Provider__c, PCN.Effective_Date__c = Network_Termination_Case__c.Requested_Termination_Date__c, PCN.Notice_Sent_Date__c = TODAY(), and PCN.Status__c = 'Pending Delivery'.

Given the PCN is created,
When PCN.Status__c = 'Pending Delivery',
Then a DocuSign envelope is sent to Provider__c.Email using the 'Network Termination Notice v1' template and PCN.DocuSign_Envelope_ID__c is stamped with the returned envelope ID.

Given the Requested_Termination_Date__c arrives,
When a scheduled Apex job (TerminationDateProcessor) runs nightly,
Then Provider_Network_Agreement__c.Status__c is updated to 'Terminated', Provider_Network_Agreement__c.Termination_Date__c = Requested_Termination_Date__c, and all active Credentialing_Case__c records for the provider are set to Credentialing_Case__c.Status__c = 'Inactive'.

Given Network_Termination_Case__c.Termination_Reason__c = 'For Cause',
When the case is approved,
Then the 90-day notice validation is bypassed (immediate termination path) and a Compliance_Flag__c = TRUE is set on the PCN for SIU review.

Given a terminated Provider__c is referenced on a new Claim__c submitted after the termination date,
When Claim__c is saved,
Then Claim__c.Status__c = 'Pended' and Claim__c.Pend_Reason__c = 'Provider Out of Network' and an alert Task is assigned to the Claims Operations Manager.

## Solution

Validation Rule on Network_Termination_Case__c enforces 90-day minimum (bypassed via custom permission 'Bypass_Termination_Notice_VP' for For Cause scenarios). Record-Triggered Flow (Termination_Approval_Flow) fires on Status__c = 'Approved' to create PCN and invoke DocuSign Apex callout. Scheduled Apex class (TerminationDateProcessor) queries Network_Termination_Case__c where Requested_Termination_Date__c = TODAY() and Status__c = 'Approved', bulk-updates PNA and Credentialing_Case__c. Claim validation handled by a Record-Triggered Flow on Claim__c checking Provider__c's PNA status at time of service. Custom Report Type 'Termination Pipeline' built for Compliance Officer dashboard.

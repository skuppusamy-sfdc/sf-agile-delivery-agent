## Description

As a Compliance Officer, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record when a Provider_Network_Agreement__c is terminated so that affected policyholders and downstream systems are notified within regulatory timeframes. The PCN must capture termination reason, effective termination date, impacted LOB, and the list of active Policy__c records associated with the terminated provider, enabling timely member communication and claims routing updates.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record has Status__c = 'Active',
When Status__c is updated to 'Terminated',
Then a Provider_Change_Notification__c record is automatically created with:
- PCN.Provider__c = PNA.Provider__c
- PCN.Termination_Date__c = PNA.Termination_Effective_Date__c
- PCN.LOB__c = PNA.Line_of_Business__c
- PCN.Status__c = 'Open'
- PCN.Notification_Deadline__c = Termination_Date__c + 30 days (per state regulatory requirement).

Given the PCN is created,
When the process runs,
Then all active Policy__c records where Policy__c.Primary_Provider__c = terminated Provider__c are identified and linked to the PCN via a PCN_Policy_Junction__c record.

Given the PCN.Notification_Deadline__c is within 5 business days,
When the scheduled job runs daily,
Then a high-priority task is created and assigned to the Provider Network Specialist responsible for the PCN.

Given PCN.Status__c = 'Open',
When all member notifications are confirmed sent,
Then PCN.Status__c can be manually updated to 'Closed' by a user with the 'PCN_Manager' Permission Set.

Given the PNA termination is for a Facility__c (Business Account) rather than a Provider__c (Person Account),
When the PCN is generated,
Then PCN.Facility__c is populated instead of PCN.Provider__c, and PCN.Record_Type is set to 'Facility_Termination'.

## Solution

Implement a Record-Triggered Flow on Provider_Network_Agreement__c (after update, when Status__c changes to 'Terminated') that calls an Apex invocable method (PCNGenerationService.generatePCN()) to create the PCN and query related Policy__c records. Bulk policy linkage handled via a Queueable Apex job (PCNPolicyLinkQueueable.cls) to avoid governor limits when providers have large panel sizes. A Scheduled Flow runs nightly to check PCN.Notification_Deadline__c proximity and create tasks. Custom notification type 'PCN_Deadline_Alert' used for in-app alerts. Record Type logic handled in the Flow decision element.

## Description

**Bug Summary:** When a provider declines to sign a Provider_Network_Agreement__c DocuSign envelope, the PNA record's Status__c remains stuck at 'Pending Signature' indefinitely instead of transitioning to 'Declined'. Provider Relations Specialists have no visibility into the declination and continue to follow up on agreements that the provider has already rejected.

**Environment:** Acme Insurance Full Sandbox (AcmeFSSB1) and confirmed in Partial Sandbox (AcmePSSB2).

**Steps to Reproduce:**
1. Create a PNA record via the INS-0343 wizard and send for DocuSign signature (PNA__c.Status__c = 'Pending Signature').
2. Open the DocuSign email as the provider recipient and click 'Decline to Sign'.
3. Enter a declination reason in the DocuSign dialog and confirm.
4. Return to Salesforce and refresh the Provider_Network_Agreement__c record.
5. Observe PNA__c.Status__c.

**Expected Result:** PNA__c.Status__c = 'Declined', PNA__c.Decline_Reason__c populated with provider's stated reason, and a Task created for the Provider Relations Specialist.

**Actual Result:** PNA__c.Status__c remains 'Pending Signature'. No Task is created. DocuSign Connect webhook log shows the 'envelope-declined' event is firing but the Salesforce endpoint is returning HTTP 200 without processing the status update.

**Root Cause Hypothesis:** The DocuSign Connect listener (Apex REST endpoint 'DocuSignWebhookHandler') has a conditional block handling 'envelope-completed' and 'envelope-voided' events but is missing a case for 'envelope-declined'. The declined event is received and silently ignored.

## Acceptance Criteria

As per Description.

## Solution

1. Update Apex REST Class 'DocuSignWebhookHandler' to add a case branch for envelopeStatus = 'declined': set PNA__c.Status__c = 'Declined', parse declineReason from DocuSign event payload into PNA__c.Decline_Reason__c (new Text Area field, 500 chars).
2. Add Record-Triggered Flow on PNA__c (after update, Status__c changed to 'Declined') to create a Task assigned to the owning Provider Relations Specialist with Subject = 'PNA Declined by Provider — Action Required' and due date = TODAY() + 3.
3. Add test coverage in DocuSignWebhookHandlerTest for the declined event payload scenario, targeting 90%+ class coverage.
4. Deploy PNA__c.Decline_Reason__c field and add to Provider 360 FlexCard (INS-0345) agreement child card.

## Description

**Bug Report**

**Environment:** Acme Insurance Full Sandbox (FS1)
**Discovered During:** Sprint 5 UAT — Fee Schedule Expiry Testing

**Steps to Reproduce:**
1. Create a Fee_Schedule__c record with Status__c = 'Active' and Expiration_Date__c = TODAY (system date at time of test: 2025-01-14).
2. Wait for or manually trigger the scheduled Apex job 'BatchFeeScheduleExpiry' (introduced in INS-0373).
3. Query Fee_Schedule__c WHERE Id = [test record Id].

**Expected Result:** Fee_Schedule__c.Status__c = 'Expired' and Fee_Schedule_Expired__e Platform Event is published.

**Actual Result:** Fee_Schedule__c.Status__c remains 'Active'. No Platform Event is observed in Event Log. Batch job shows 'Completed' in Apex Jobs with 0 records processed.

**Root Cause Hypothesis:** The SOQL query in BatchFeeScheduleExpiry uses `Expiration_Date__c = TODAY` but the Date formula evaluates to a DateTime boundary mismatch in UTC vs. org timezone (America/Chicago, UTC-6). Records created with a date of 2025-01-14 are not matched when the batch runs at 00:05 UTC on 2025-01-14 because the org local date is still 2025-01-13.

**Impact:** Claims adjudication may reference expired fee schedules, resulting in incorrect Claim_Line__c.Allowed_Amount__c calculations and potential overpayment.

as per Description

## Solution

1. Update BatchFeeScheduleExpiry SOQL to use `Expiration_Date__c <= :Date.today()` instead of equality to catch any records that may have been missed in prior runs.
2. Adjust scheduled job execution time from 00:05 UTC to 08:00 UTC (02:00 AM America/Chicago) to ensure org local date is unambiguously the target date when the batch fires.
3. Add a secondary guard: Record-Triggered Flow on Fee_Schedule__c (before save, update context) that checks IF Expiration_Date__c < TODAY AND Status__c = 'Active' THEN set Status__c = 'Expired' as a real-time fallback.
4. Add unit test BatchFeeScheduleExpiryTest with Test.setFixedDate() to simulate UTC/local timezone boundary conditions — assert 0 active records remain post-batch.
5. Deploy Platform Event publish inside a try/catch with error logging to Fee_Schedule_Audit__c to surface any future publish failures.

## Description

As a Provider Network Specialist, I need a comprehensive Group_Service_Agreement__c record page that embeds the associated fee schedules and a filterable participating provider roster so that I can manage all contract-related data from a single view without switching between multiple objects and tabs during employer onboarding negotiations.

## Acceptance Criteria

Given a user with the 'Provider_Network_Specialist' Permission Set opens a GSA record,
When the record page loads,
Then the page layout includes the following sections: (1) GSA Header Details, (2) Associated Fee Schedules (related list), (3) Participating Provider Roster (FlexCard), (4) Contract Documents (DocuSign integration panel), (5) Activity Timeline.

Given the Participating Provider Roster FlexCard loads,
When the specialist views the roster,
Then all Provider__c records linked via PNA records where PNA.GSA__c = current GSA record ID are displayed with columns: Provider Name, NPI, Specialty, Network Status, PNA Effective Date, PNA End Date.

Given the specialist uses the roster filter panel,
When they filter by Specialty__c or Network_Status__c,
Then the FlexCard dynamically refreshes displaying only matching providers without a full page reload.

Given a Fee_Schedule__c record is linked to the GSA,
When the specialist clicks 'View Rates' on the related list,
Then a modal displays the Fee_Schedule_Line__c records in a paginated table with CPT Code, Description, Reimbursement Rate, and Effective Date columns.

Given GSA.Status__c = 'Expired',
When any user views the record page,
Then a banner warning is displayed at the top of the page: 'This Group Service Agreement has expired. Fee schedules and provider rosters are read-only.' and all edit actions are disabled.

Given the specialist clicks 'Export Roster' on the FlexCard,
When the export action is triggered,
Then a CSV file is generated containing all roster data and downloaded to the user's browser.

## Solution

Lightning App Builder page layout configured with custom LWC sections. FlexCard ('GSAProviderRosterCard') built in OmniStudio with a DataRaptor Extract joining Provider__c, PNA, and GSA records. Filter panel implemented as OmniScript child component embedded in the FlexCard. 'View Rates' modal uses a second FlexCard ('FeeScheduleRatesCard') launched via FlexCard action. GSA expiration banner driven by a Record-Triggered Flow (Before Save on GSA) that sets a GSA.Is_Expired__c formula field; LWC conditionally renders the banner based on this field. CSV export uses a custom Apex REST endpoint invoked via FlexCard action, returning base64-encoded CSV content. DocuSign panel leverages the standard DocuSign for Salesforce managed package component.

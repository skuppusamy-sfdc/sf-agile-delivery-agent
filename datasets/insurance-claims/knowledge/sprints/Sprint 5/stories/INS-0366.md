## Description

As a Provider Network Specialist, I need to upload and version CPT-based fee schedules onto Fee_Schedule__c records linked to a Provider_Network_Agreement__c so that Claim_Line__c adjudication can reference the correct contracted rate at the time of service. Fee schedules must support effective dating so that rate changes do not retroactively alter previously adjudicated claims. Extends INS-0358 (Claim Line adjudication engine).

## Acceptance Criteria

Given a Provider Network Specialist uploads a CSV of CPT codes and allowed amounts,
When the DataRaptor Load executes,
Then individual Fee_Schedule_Line__c child records are created under the parent Fee_Schedule__c with CPT_Code__c, Allowed_Amount__c, Effective_Date__c, and Termination_Date__c populated correctly.

Given a Fee_Schedule__c record with Effective_Date__c = '2025-01-01' and Termination_Date__c = '2025-12-31',
When a Claim_Line__c with Date_of_Service__c = '2025-06-15' is adjudicated,
Then the adjudication engine selects this fee schedule version and populates Claim_Line__c.Contracted_Rate__c accordingly.

Given a new fee schedule version is uploaded for the same PNA and overlapping effective period,
When the system detects a date overlap,
Then an error message 'Fee schedule date range overlaps with existing version [Fee_Schedule__c.Name]' is displayed and the upload is blocked.

Given a Fee_Schedule__c record is marked Active__c = true,
When more than one Active fee schedule exists for the same Provider_Network_Agreement__c and LOB__c combination,
Then a validation rule fires and prevents saving with the message 'Only one active fee schedule per PNA/LOB is permitted.'

Given a Fee_Schedule__c is superseded by a new version,
When the new version is activated,
Then the prior version's Active__c is automatically set to false and Termination_Date__c is back-filled to (new version Effective_Date__c - 1 day) via a Record-Triggered Flow.

## Solution

Build a custom LWC 'feeScheduleUploader' that accepts CSV files, parses rows client-side, and calls an Apex @AuraEnabled method 'FeeScheduleImportController.importLines()' to perform bulk upsert of Fee_Schedule_Line__c records. Versioning logic is enforced via an Apex Trigger (FeeScheduleTrigger) on before-insert/update. A Record-Triggered Flow handles auto-deactivation of prior versions. The adjudication rate lookup is a separate Apex utility class 'FeeScheduleRateLookup' invoked by the Claim Line adjudication engine. CRUD/FLS enforced via Permission Set Group 'Fee_Schedule_Manager_PSG'.

## Description

As a Provider Relations Specialist, I need a single FlexCard on the Provider__c (Person Account) record page that surfaces active Provider_Network_Agreements__c, linked Fee_Schedule__c versions, current Credentialing_Case__c status, and any open Provider_Change_Notification__c items so that I can assess a provider's full contracting and credentialing posture without navigating multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record,
When the Provider 360 FlexCard loads,
Then it displays all Provider_Network_Agreement__c records where Status__c = 'Executed' and Termination_Date__c >= TODAY() in a child card list, showing PNA Name, LOB__c, Effective_Date__c, and Contracted_Rate_Tier__c.

Given the FlexCard loads PNA data,
When a linked Fee_Schedule__c with Active__c = true exists,
Then the fee schedule name and Effective_Date__c are displayed inline beneath each PNA row.

Given a Credentialing_Case__c record exists for the provider,
When the FlexCard loads,
Then the most recent Credentialing_Case__c.Status__c and CAQH_Profile_Status__c values are displayed in a 'Credentialing' badge section.

Given one or more Provider_Change_Notification__c records exist with PCN_Status__c = 'Open',
When the FlexCard loads,
Then an alert banner 'X Open PCN(s)' is shown in amber, linking to the PCN list filtered by provider.

-Display QGenda scheduling availability on the FlexCard.- (Descoped to Phase 2 per INS-0362)

Given the provider has no active PNAs,
When the FlexCard loads,
Then an empty-state message 'No active network agreements found' is displayed with a quick action button to initiate a new PNA.

## Solution

Build an OmniStudio FlexCard 'Provider360_FC' with a primary DataRaptor Extract fetching Provider__c base data and child DataRaptor Turbo Extracts for PNA, Fee_Schedule__c, Credentialing_Case__c, and PCN related lists. Conditional rendering blocks handle empty states and alert banners. The FlexCard is embedded on the Provider__c Lightning Record Page via an App Builder drag-and-drop. A custom CSS swatch aligns with Acme's Lightning Design System theme. Visibility controlled by Permission Set 'Provider_Relations_View_PSG'.

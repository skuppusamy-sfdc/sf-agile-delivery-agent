## Description

As a Claims Operations Manager, I need the Agentforce AI assistant embedded in the FNOL intake screen to automatically suggest in-network Provider__c and Facility__c records based on the claimant's ZIP code, the Policy__c.LOB__c, and the reported injury type so that claims representatives can guide members toward contracted providers during first contact, reducing out-of-network leakage and downstream payment disputes. Extends INS-0350 (FNOL OmniScript baseline).

## Acceptance Criteria

Given a claims representative is on Step 3 ('Provider Assignment') of the FNOL OmniScript and has entered the claimant's ZIP code,
When Agentforce evaluates the context,
Then a ranked list of up to 5 Provider__c or Facility__c records is surfaced in the Agentforce side panel, filtered by Network_Status__c = 'Active - Contracted', LOB__c matching Policy__c.LOB__c, and proximity to the claimant ZIP code (within 25 miles radius).

Given Agentforce returns provider suggestions,
When the claims representative selects a suggested provider,
Then Claim__c.Treating_Provider__c is auto-populated with the selected Provider__c Id and Claim__c.Provider_Assignment_Source__c is set to 'Agentforce AI'.

Given the LOB__c is 'Workers Compensation' and the reported injury ICD-10 code is in the musculoskeletal range (M00–M99),
When Agentforce generates suggestions,
Then providers with Specialty__c = 'Orthopedics' or 'Physical Therapy' are ranked higher in the suggestion list.

Given no in-network providers exist within 25 miles of the claimant ZIP code,
When Agentforce evaluates the context,
Then the panel displays the message 'No in-network providers found within 25 miles. Expanding search to 50 miles.' and re-queries automatically.

Given a provider suggestion is presented,
When the claims representative hovers over a suggestion card,
Then a tooltip displays Provider__c.NPI__c, Facility__c.TIN__c (if applicable), Credentialing_Case__c.Status__c, and the most recent PNA Effective_Date__c for transparency.

-Surface QGenda real-time appointment availability within the suggestion card.- (Descoped to Phase 2 pending QGenda API contract approval)

## Solution

Configure an Agentforce Agent Action 'Suggest_InNetwork_Providers' backed by an Apex invocable class 'InNetworkProviderSuggestionAction' that queries Provider__c and Facility__c using geolocation fields (Geolocation__c) and a SOQL DISTANCE function filtered by Network_Status__c and LOB__c. ICD-10 specialty ranking is handled by a Custom Metadata Type 'ICD10_Specialty_Map__mdt'. The Agentforce Topic configuration references the FNOL OmniScript context variables (ZIP code, LOB, ICD-10) via Einstein Copilot Context Grounding. Provider suggestion cards are rendered as a custom LWC 'providerSuggestionCard' embedded in the Agentforce side panel slot. Audit trail of AI-driven assignments is captured via Claim__c.Provider_Assignment_Source__c field.

## Description

As a Credentialing Specialist, I need the system to automatically validate a provider's NPI against the NPPES (National Plan and Provider Enumeration System) registry when a new Provider__c record is saved so that we onboard only legitimately enumerated providers and reduce downstream claims denials due to invalid NPI submissions to CMS.

Validation must check NPI status, provider name match, and taxonomy code alignment with the LOB being contracted. Results must be stored on the Provider__c record for audit purposes.

## Acceptance Criteria

Given a new Provider__c record is being created with Provider__c.NPI__c populated,
When the record is saved,
Then a callout is made to the NPPES NPI Registry API (https://npiregistry.cms.hhs.gov/api) using the NPI value.

Given the NPPES API returns a valid, active NPI,
When the response is processed,
Then Provider__c.NPI_Validation_Status__c = 'Validated', Provider__c.NPI_Validated_Date__c = today, and Provider__c.NPPES_Provider_Name__c is populated with the name returned by NPPES for cross-reference.

Given the NPPES API returns an inactive or deactivated NPI,
When the response is processed,
Then Provider__c.NPI_Validation_Status__c = 'Invalid – Deactivated' and the record save is blocked with a validation error message: 'NPI [value] is deactivated in NPPES. Please verify provider enumeration before proceeding.'

Given the NPPES API is unavailable (HTTP 503 or timeout),
When the callout fails,
Then Provider__c.NPI_Validation_Status__c = 'Pending – API Unavailable' and a Platform Event (NPI_Validation_Retry__e) is published for async retry processing; the record save is not blocked.

Given Provider__c.NPI_Validation_Status__c = 'Validated',
When a Credentialing_Case__c is created for the provider,
Then Credentialing_Case__c.NPI_Verified__c is auto-checked (true) via a Record-Triggered Flow.

## Solution

Implement NPPESValidationService.cls as an Apex @future(callout=true) method invoked from a before-insert/before-update Apex Trigger (ProviderTrigger.cls) when NPI__c is populated or changed. Use Named Credential (NPPES_API) for endpoint management. Parse JSON response using a lightweight wrapper class (NPPESResponseWrapper.cls). Platform Event (NPI_Validation_Retry__e) published for retry logic handled by a separate Trigger on the Platform Event with exponential backoff (max 3 retries, stored in Custom Metadata NPI_Retry_Config__mdt). Store full NPPES JSON response in a Long Text Area field (Provider__c.NPPES_Raw_Response__c) for audit. Exclude sandbox orgs from live callout using a Custom Setting (Integration_Config__c.NPPES_Callout_Enabled__c).

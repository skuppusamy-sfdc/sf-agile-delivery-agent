## Description

As a Provider Network Specialist, I need a guided OmniScript wizard to create and send Provider_Network_Agreement__c records for execution via DocuSign, so that the contracting workflow is standardized, auditable, and reduces manual back-and-forth with providers. The wizard must capture all required contract terms including effective dates, LOB coverage, TIN/NPI validation, and fee schedule linkage before generating the contract package.

## Acceptance Criteria

Given a Provider Network Specialist navigates to a Provider__c (Person Account) record and clicks 'Initiate Contract'
When the OmniScript wizard launches
Then Step 1 must display provider NPI (Provider__c.NPI__c) and TIN (Provider__c.TIN__c) in read-only fields pre-populated from the record, and block progression if either field is null.

Given the specialist completes Step 1 (Provider Verification)
When they advance to Step 2 (Contract Terms)
Then the wizard must present picklist fields for LOB (Provider_Network_Agreement__c.Line_of_Business__c) with values: Medical, Dental, Vision, Behavioral Health; and Contract_Type__c with values: Participating, Preferred, Exclusive.

Given the specialist selects a LOB and Contract Type
When they advance to Step 3 (Fee Schedule Assignment)
Then a filterable lookup must display only active Fee_Schedule__c records where Fee_Schedule__c.LOB__c matches the selected LOB and Fee_Schedule__c.Status__c = 'Active'.

Given all wizard steps are completed
When the specialist clicks 'Generate & Send for Signature'
Then a Provider_Network_Agreement__c record must be created with Status__c = 'Pending Signature', the linked Fee_Schedule__c populated in PNA.Fee_Schedule__c, and a DocuSign envelope dispatched to the email address on Provider__c.PersonEmail within 2 minutes.

Given the DocuSign envelope is sent
When the provider executes the document
Then a Platform Event (DocuSign_Completed__e) must fire, updating PNA.Status__c to 'Active' and PNA.Effective_Date__c to the signature timestamp date.

-Given a PNA already exists in 'Active' status for the same Provider and LOB, the wizard must present a warning modal and prevent duplicate submission- (superseded: duplicate check now handled by INS-0312 validation rule on PNA object).

## Solution

Build a 4-step OmniScript (NetworkContracting_PNA_Wizard) with DataRaptor Extracts pre-populating provider data from Provider__c. Use a DataRaptor Transform to validate NPI against NPPES integration callout (existing Named Credential: NPPES_API). Fee Schedule lookup implemented via a filtered SOQL DataRaptor Extract. On final step, invoke an Apex action (PNAContractingService.generateEnvelope()) that calls the DocuSign Apex Toolkit to create the envelope. A Platform Event subscriber (Record-Triggered Flow on DocuSign_Completed__e) updates PNA.Status__c and Effective_Date__c. Deploy a Permission Set Group (PSG_Provider_Network_Specialist) to gate wizard access.

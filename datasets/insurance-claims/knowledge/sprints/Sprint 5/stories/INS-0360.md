## Description

As a Provider Relations Specialist, I need a FlexCard embedded on the Provider__c (Person Account) record page that surfaces a consolidated summary of the provider's active network agreements, credentialing status, and recent claims activity, so that I can quickly assess a provider's standing without navigating across multiple related lists.

This story extends INS-0347 (Provider Account page layout redesign).

## Acceptance Criteria

Given a Provider__c record is open in the Salesforce Lightning UI,
When the Provider 360 FlexCard loads,
Then it displays the following data panels:
  - Panel 1: Active PNA count (Provider_Network_Agreement__c where Status__c = 'Active'), list of Network_Name__c values, and earliest Expiration_Date__c.
  - Panel 2: Credentialing_Case__c.Status__c for the most recent open credentialing case, CAQH_Profile_Status__c, and License_Expiration_Date__c.
  - Panel 3: Count of Claim__c records in the last 90 days grouped by Claim__c.Status__c (Submitted, In Review, Adjudicated, Denied).
  - Panel 4: NPI, TIN, and primary specialty from Provider__c standard and custom fields.

Given the provider has no active PNA records,
When the FlexCard loads,
Then Panel 1 displays a warning badge 'No Active Network Agreement' in amber.

Given Credentialing_Case__c.License_Expiration_Date__c <= Today + 30 days,
When the FlexCard loads,
Then Panel 2 displays a red alert icon with label 'License Expiring Soon'.

Given the user clicks 'View All Claims' in Panel 3,
When the action fires,
Then the user is navigated to a filtered Claim__c list view scoped to the provider's NPI.

-Given the FlexCard includes a 'Send DocuSign Envelope' quick action button- (removed from scope; moved to INS-0362).

## Solution

Build the Provider 360 FlexCard using OmniStudio FlexCards with child DataRaptors for each panel. DataRaptor Turbo Extract `DR_Provider360_PNA` queries Provider_Network_Agreement__c with a SOQL filter on Status__c and Provider__c lookup. `DR_Provider360_Credentialing` fetches the most recent Credentialing_Case__c ordered by CreatedDate DESC LIMIT 1. `DR_Provider360_ClaimSummary` uses an aggregate SOQL to group Claim__c by Status__c for the last 90 days. Conditional styling in the FlexCard JSON template uses `%vlocity_namespace%` state variables to drive the amber/red badge logic. The FlexCard is embedded on the Provider__c Lightning Record Page via a custom LWC wrapper that passes the recordId. A Permission Set `Provider_360_Viewer` controls FlexCard visibility for Provider Relations Specialist and Claims Operations Manager profiles.

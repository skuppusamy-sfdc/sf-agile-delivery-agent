## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard embedded on the Provider__c (Person Account) record page that displays a real-time summary of the provider's active PNA contracts, current fee schedule coverage, and credentialing status so that I can quickly assess a provider's network standing without navigating multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the card displays: (1) count of Active PNA records, (2) earliest PNA__c.Effective_End_Date__c among active contracts labeled 'Next Expiring Contract', (3) Credentialing_Case__c.Status__c for the most recent open credentialing case, (4) count of Fee_Schedule__c records with Status__c = 'Active'.

Given a provider has no active PNA records,
When the FlexCard renders,
Then a banner displays: 'No Active Network Agreements — Provider is Out of Network' in amber styling.

Given a PNA__c.Effective_End_Date__c is within 60 days of today,
When the FlexCard renders the contract summary,
Then the expiring contract row is highlighted in amber and a tooltip reads 'Contract expiring soon — initiate renewal'.

Given a Provider Relations Specialist clicks the 'View All Contracts' action on the FlexCard,
When the action fires,
Then an OmniScript launches displaying the full list of PNA records with Status__c, Effective_Start_Date__c, Effective_End_Date__c, and LOB__c columns.

Given the provider's NPI on Provider__c.NPI__c is flagged in the CMS exclusion list (via Integration_Log__c.CMS_Exclusion_Flag__c = TRUE),
When the FlexCard loads,
Then a red alert banner displays: 'CMS Exclusion Flag Active — Claims Payment Blocked'.

## Solution

Build OmniStudio FlexCard 'Provider360ContractSummary' with a DataRaptor Turbo Extract as the primary data source querying Provider_Network_Agreement__c, Fee_Schedule__c, and Credentialing_Case__c child records. Use conditional styling states for amber/red banners based on date calculations and flag fields. 'View All Contracts' action launches a lightweight OmniScript 'ProviderContractListOS' with a DataRaptor List Extract. FlexCard embedded on Provider__c Lightning Record Page via App Builder. DataRaptor input maps Provider__c Id from the record context. Flyout card for PNA row detail uses a nested DataRaptor Extract. Deployed to Provider Relations Specialist and Claims Operations Manager Permission Set Groups.

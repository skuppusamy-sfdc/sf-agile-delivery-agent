## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard embedded on the Provider__c (Person Account) record page that consolidates active PNAs, linked fee schedules, credentialing case status, and recent claims activity in a single view, so that I can quickly assess a provider's full relationship with Acme Insurance without navigating across multiple objects. This extends the foundational provider record layout established in INS-0318.

## Acceptance Criteria

Given a Provider Relations Specialist opens any Provider__c record
When the Provider 360 FlexCard renders
Then it must display the following panels: (1) Contract Summary, (2) Credentialing Status, (3) Recent Claims Activity, (4) Network Participation.

Given the Contract Summary panel renders
When Provider_Network_Agreement__c records exist for the provider
Then each active PNA must display: PNA.Name, PNA.LOB__c, PNA.Effective_Date__c, PNA.Expiration_Date__c, and the linked Fee_Schedule__c.Name as a clickable navigation link.

Given the Credentialing Status panel renders
When a Credentialing_Case__c record exists with Credentialing_Case__c.Provider__c = current Provider__c.Id
Then the panel must show the most recent Credentialing_Case__c.Status__c and Credentialing_Case__c.Expiration_Date__c with a color-coded badge: Green = 'Credentialed', Yellow = 'Pending Renewal' (expiring within 90 days), Red = 'Expired' or 'Suspended'.

Given the Recent Claims Activity panel renders
When Claim__c records exist where Claim__c.Rendering_Provider__c = Provider__c.Id
Then the panel must show the 5 most recent claims sorted by Claim__c.Date_of_Service__c descending, displaying Claim__c.Claim_Number__c, Claim__c.Status__c, and Claim__c.Total_Billed_Amount__c.

-Given the Network Participation panel renders, it must show QGenda schedule availability via iframe embed- (superseded: QGenda iframe blocked by CSP policy; replaced with deep-link button to QGenda provider profile).

Given a provider has no active PNA records
When the Contract Summary panel renders
Then a contextual action button 'Initiate Contract' must appear, launching the PNA wizard from INS-0350.

## Solution

Build OmniStudio FlexCard (Provider360_FlexCard) with 4 child cards using DataRaptor List actions. DataRaptor DR_GetProviderPNAs queries PNA with SOQL filter on Status__c = 'Active'. DR_GetCredentialingStatus retrieves latest Credentialing_Case__c ordered by CreatedDate DESC LIMIT 1. DR_GetRecentClaims queries Claim__c with LIMIT 5 ORDER BY Date_of_Service__c DESC. Color-coded badge logic implemented via FlexCard Conditional Visibility states. 'Initiate Contract' action uses a Navigate action type pointing to the INS-0350 OmniScript. FlexCard assigned to Provider__c Lightning Record Page via App Builder.

## Description

As a Provider Relations Specialist, I need a FlexCard on the Provider__c (Person Account) record page that displays the provider's active Provider_Network_Agreement__c records alongside the linked active Fee_Schedule__c details so that I can quickly assess network participation status and reimbursement tiers without navigating to multiple related lists.

## Acceptance Criteria

Given a Provider__c record with one or more associated Provider_Network_Agreement__c records,
When the Provider 360 FlexCard loads,
Then the card must display PNA.Name, PNA.Status__c, PNA.Effective_Date__c, PNA.Expiration_Date__c, and the linked Fee_Schedule__c.Rate_Amount__c for each active PNA.

Given a Provider__c with no active PNA,
When the FlexCard loads,
Then the card must display the message 'No active network agreements on file' in a warning state.

Given a PNA where Fee_Schedule__c.Status__c = 'Expired',
When the FlexCard renders,
Then the Fee Schedule row must display a red badge labeled 'Expired' next to the rate amount.

Given the FlexCard is displayed,
When the specialist clicks 'View Agreement',
Then the user is navigated to the Provider_Network_Agreement__c record detail page in the same browser tab.

Given a Provider__c record accessed by a user without the 'Provider_Network_Read' permission set,
When the FlexCard loads,
Then PNA rate data must be masked and display '—' in place of Fee_Schedule__c.Rate_Amount__c.

## Solution

1. Build OmniStudio FlexCard 'Provider360_PNA_Card' using a DataRaptor Extract that queries Provider_Network_Agreement__c and child Fee_Schedule__c (WHERE Status__c = 'Active') filtered by Provider__c Id.
2. DataRaptor Transform normalizes the nested JSON for card rendering.
3. Conditional state badges implemented via FlexCard State configuration (green=Active, red=Expired, yellow=Pending).
4. 'View Agreement' action uses FlexCard Navigation Action (recordId from PNA).
5. Field masking handled via a custom Apex invocable action called from the DataRaptor that checks PERM('Provider_Network_Read') and returns masked values when false.
6. FlexCard embedded on Provider__c Lightning Record Page via App Builder.

## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard embedded on the Provider__c (Person Account) record page that surfaces active PNA details, associated fee schedules, and current credentialing status in a single view, so that I can answer provider inquiries without navigating across multiple related lists and tabs.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record,
When the Provider 360 FlexCard loads,
Then it displays Provider_Network_Agreement__c.Contract_Start_Date__c, Provider_Network_Agreement__c.Contract_End_Date__c, Provider_Network_Agreement__c.LOB__c, and Provider_Network_Agreement__c.Status__c for all PNAs where Status__c IN ('Active', 'Pending Renewal').

Given the provider has an active Credentialing_Case__c,
When the FlexCard renders the Credentialing section,
Then it shows Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and a color-coded badge: green if Expiration_Date__c > TODAY() + 90, yellow if within 90 days, red if expired.

Given the provider has associated Fee_Schedule__c records via PNA_Fee_Schedule__c junction,
When the Fee Schedule child card panel is expanded,
Then it lists Fee_Schedule__c.Name, Fee_Schedule__c.LOB__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.Expiration_Date__c for all active schedules.

Given Provider_Network_Agreement__c.Status__c = 'Terminated',
When the FlexCard loads,
Then terminated PNAs are collapsed by default under a 'Show Terminated Agreements' toggle and displayed with a red 'Terminated' badge.

Given the specialist clicks 'Initiate Renewal' on a PNA row,
When the action fires,
Then the PNA Creation OmniScript (INS-0327) launches in a modal pre-populated with the existing PNA's LOB__c and TIN__c.

## Solution

OmniStudio FlexCard (Provider360_FC) with three child cards: PNA_Summary_Card, FeeSchedule_Card, Credentialing_Card. DataRaptor Extract DRs query Provider_Network_Agreement__c, PNA_Fee_Schedule__c, Fee_Schedule__c, and Credentialing_Case__c with relationship traversal. Badge color logic implemented via FlexCard conditional CSS state classes. 'Initiate Renewal' action invokes the PNA_Creation_OS OmniScript via an OmniScript Launch action with input JSON mapping. FlexCard deployed to the Provider__c Lightning Record Page via App Builder with visibility rule: Profile IN ('Provider Relations Specialist', 'Provider Network Specialist').

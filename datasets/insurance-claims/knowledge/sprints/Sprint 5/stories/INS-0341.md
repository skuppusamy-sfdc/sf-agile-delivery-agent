## Description

As a Claims Operations Manager, I need the Fee_Schedule__c and Claim_Line__c objects to support CPT modifier codes (e.g., -25, -50, -59, -TC) so that modifier-specific reimbursement adjustments are applied automatically during adjudication rather than requiring manual rate correction post-payment.

This story was referenced as a dependency in INS-0335 (where modifier matching was deferred). Common modifiers and their typical rate impact: Modifier 50 (bilateral procedure) = 150% of base rate; Modifier 26 (professional component) = percentage of base; Modifier TC (technical component) = percentage of base.

## Acceptance Criteria

Given a Fee_Schedule__c record has Fee_Schedule__c.CPT_Modifier__c populated (e.g., '50'),
When Claim_Line__c.CPT_Modifier__c matches Fee_Schedule__c.CPT_Modifier__c and the CPT code matches,
Then Claim_Line__c.Contracted_Rate__c = Fee_Schedule__c.Modifier_Adjusted_Rate__c and Claim_Line__c.Modifier_Applied__c = TRUE.

Given a Claim_Line__c has CPT_Modifier__c = '50' (bilateral),
When no modifier-specific Fee_Schedule__c entry exists but a base CPT entry exists,
Then Claim_Line__c.Contracted_Rate__c = Fee_Schedule__c.Contracted_Rate__c * 1.50 and Claim_Line__c.Rate_Calculation_Note__c = 'Bilateral modifier applied — 150% of base rate'.

Given a Claim_Line__c has CPT_Modifier__c = '26' or 'TC',
When the base Fee_Schedule__c record has Fee_Schedule__c.Professional_Component_Pct__c and Fee_Schedule__c.Technical_Component_Pct__c populated,
Then the respective percentage is applied to Fee_Schedule__c.Contracted_Rate__c to derive Claim_Line__c.Contracted_Rate__c.

Given a CPT modifier is not recognized in the Custom Metadata Type `CPT_Modifier_Config__mdt`,
When rate resolution encounters the modifier,
Then the base rate is applied, Claim_Line__c.Modifier_Applied__c = FALSE, and Claim_Line__c.Requires_Manual_Review__c = TRUE with Rate_Calculation_Note__c = 'Unrecognized modifier — manual review required'.

Given a Provider Relations Specialist needs to add a new modifier configuration,
When they update CPT_Modifier_Config__mdt via Setup,
Then the new modifier rule is immediately active without a code deployment.

## Solution

Extend the `FeeScheduleRateLookupService` Apex class (INS-0335) with a `resolveModifierRate(String cptCode, String modifier, Id pnaId)` method. Modifier behavior rules stored in Custom Metadata Type `CPT_Modifier_Config__mdt` with fields: Modifier_Code__c, Calculation_Type__c (Flat Rate / Percentage / Multiplier), Rate_Multiplier__c, Is_Active__c. Modifier lookup runs after base CPT rate is resolved. Add new fields to Fee_Schedule__c: CPT_Modifier__c (Text 5), Modifier_Adjusted_Rate__c (Currency), Professional_Component_Pct__c (Percent), Technical_Component_Pct__c (Percent). Add to Claim_Line__c: CPT_Modifier__c (Text 5), Modifier_Applied__c (Checkbox), Rate_Calculation_Note__c (Text 255). Update FeeScheduleOverlapValidator Apex trigger (INS-0340) to include CPT_Modifier__c in the uniqueness check.

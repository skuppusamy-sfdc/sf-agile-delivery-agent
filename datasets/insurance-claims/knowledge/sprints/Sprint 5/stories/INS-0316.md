## Description

As a Broker Relations Coordinator, I need a guided Screen Flow to create a Group_Service_Agreement__c record linked to an employer group Account, so that all GSA terms — including effective dates, enrolled LOBs, TPA designation, and broker commission schedule — are captured in a standardized way during the group onboarding process, reducing data entry errors and ensuring downstream policy issuance can proceed without manual rework.

## Acceptance Criteria

Given a Broker Relations Coordinator opens an employer group Account record and clicks 'Create Group Service Agreement',
When the Screen Flow launches,
Then Screen 1 must auto-populate GSA.Account__c with the originating Account Id and display the Account's TIN__c as a read-only field.

Given the coordinator reaches Screen 2 (LOB Selection),
When GSA.LOB__c includes 'Workers Comp',
Then an additional required field GSA.TPA_Account__c must become visible and mandatory,
And a lookup filter must restrict TPA_Account__c to Accounts where Account.RecordType = 'TPA Partner' and Account.TPA_Active__c = TRUE.

Given the coordinator enters effective date information,
When GSA.Effective_Date__c is not the first day of a calendar month,
Then the flow must display a warning message: 'GSA effective dates must align to the first of the month per underwriting guidelines' and require the coordinator to confirm or correct before proceeding.

Given Screen 3 (Broker Commission),
When GSA.Broker_Commission_Rate__c > 5.0%,
Then the flow must require GSA.Underwriting_Approval_Required__c = TRUE and create a Task assigned to the Underwriting Analyst queue with Subject = 'GSA Commission Approval Required — Exceeds 5% Threshold'.

Given the coordinator completes all screens and clicks 'Submit',
When the GSA record is created successfully,
Then GSA.Status__c must be set to 'Pending Execution',
And a Conga CLM document generation job must be triggered to produce the GSA PDF,
And the PDF must be attached to the GSA record as a Salesforce File with a sharing setting of 'All Users with Link'.

Given a duplicate GSA already exists for the same Account with overlapping effective dates,
When the flow attempts to save,
Then a duplicate warning screen must appear listing the conflicting GSA record name and its date range,
And the coordinator must not be able to proceed without either cancelling or explicitly acknowledging the conflict via a checkbox.

## Solution

Build a Lightning Screen Flow ('GSA_Onboarding_Flow') with 4 screens: (1) Account Confirmation, (2) LOB & TPA Selection with dynamic field visibility via flow variables, (3) Commission & Underwriting, (4) Review & Submit. Apex action 'GSADuplicateCheckAction' will perform the overlap date query before the final save screen. Conga CLM will be invoked via an Apex callout in a separate after-insert Flow triggered on GSA creation. A custom Permission Set 'GSA_Create_PS' will be required for flow access. The flow will be embedded as a Quick Action on the Account record page, restricted to RecordType = 'Employer Group'.

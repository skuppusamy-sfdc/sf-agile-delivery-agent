## Description

As a Claims Operations Manager, I need the system to automatically detect when a Claim_Line__c references a rendering provider or facility with no active Provider_Network_Agreement__c (out-of-network), apply the appropriate out-of-network (OON) benefit tier from the Policy__c, and flag the claim for manual review when OON costs exceed the policy's OON threshold, so that adjudication accuracy is maintained and OON overpayments are prevented.

## Acceptance Criteria

Given a Claim__c is submitted with Claim__c.Rendering_Provider__c referencing a Provider__c with no active PNA for the relevant LOB,
When the claim enters the adjudication queue,
Then Claim__c.Network_Status__c is automatically set to 'Out-of-Network' and Claim__c.OON_Flag__c = TRUE.

Given Claim__c.Network_Status__c = 'Out-of-Network',
When the Payment Engine calculates the allowed amount for each Claim_Line__c,
Then Claim_Line__c.Contracted_Rate__c is populated using the OON fee schedule derived from Policy__c.OON_Benefit_Tier__c, and Claim_Line__c.Network_Status__c = 'OON'.

Given the sum of all Claim_Line__c.Billed_Amount__c on the Claim__c exceeds Policy__c.OON_Annual_Threshold__c,
When adjudication completes,
Then Claim__c.Status__c is set to 'Pending Manual Review' and a Task is created and assigned to the Claims Operations Manager queue with Subject = 'OON Threshold Exceeded — Manual Review Required'.

Given Claim__c.OON_Flag__c = TRUE,
When the EOB is generated,
Then Explanation_of_Benefits__c.OON_Indicator__c = TRUE and the EOB includes the standard OON disclosure language from a Custom Label (OON_Disclosure_Text).

Given a provider has an active PNA but PNA.Network_Tier__c does not match the Policy__c.Required_Network_Tier__c,
When adjudication runs,
Then Claim__c.Network_Status__c = 'Tier Mismatch' and Claim__c.Adjudication_Notes__c is appended with 'Provider tier [X] does not meet policy required tier [Y]'.

## Solution

Record-Triggered Flow on Claim__c (after insert, after update where Status__c changes to 'Submitted') invokes Apex class 'NetworkEligibilityValidator'. The validator queries active PNA records for the rendering provider/facility filtered by LOB and validates network tier against Policy__c.Required_Network_Tier__c. OON fee schedule lookup uses FeeScheduleLookupService (built in INS-0381) with an OON fallback path. Threshold breach logic creates a Task via Flow action. EOB generation (existing service) is updated to consume Claim__c.OON_Flag__c and inject Custom Label content. All network status transitions are logged to a Claim_Audit__c child record for compliance reporting.

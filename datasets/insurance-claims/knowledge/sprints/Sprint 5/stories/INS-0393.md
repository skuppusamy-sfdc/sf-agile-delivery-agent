## Description

As a Provider Relations Specialist, I need a structured termination workflow triggered by the creation of a Network_Termination_Case__c so that all associated Provider_Network_Agreement__c records are systematically inactivated, a Provider_Change_Notification__c (PCN) is generated for regulatory compliance, and impacted in-flight claims are flagged for review.

Provider terminations must comply with state-mandated notice periods (typically 60–90 days) and CMS network adequacy reporting requirements. This workflow must prevent premature PNA inactivation before the effective termination date.

## Acceptance Criteria

Given a Network_Termination_Case__c is created with Termination_Case__c.Status__c = 'Initiated' and Termination_Case__c.Termination_Effective_Date__c is populated,
When the record is saved,
Then a validation rule enforces that Termination_Effective_Date__c >= today + 60 days (configurable via Custom Metadata Termination_Config__mdt.Min_Notice_Days__c).

Given Termination_Case__c.Status__c = 'Approved',
When the approval is granted,
Then all Provider_Network_Agreement__c records linked to the terminated Provider__c where PNA.Status__c = 'Active' are updated to PNA.Pending_Termination__c = true and PNA.Termination_Date__c = Termination_Case__c.Termination_Effective_Date__c.

Given PNA.Pending_Termination__c = true and today >= PNA.Termination_Date__c,
When the nightly scheduled job runs,
Then PNA.Status__c is set to 'Terminated' and a Provider_Change_Notification__c record is auto-created with PCN.Type__c = 'Termination', PCN.Provider__c = the terminated provider, and PCN.Effective_Date__c = PNA.Termination_Date__c.

Given the PCN record is created,
When PCN.Status__c = 'Generated',
Then a Conga CLM document is generated using the PCN_Notice_Template and attached to the PCN record for mailing to the provider.

Given the provider termination is effective,
When open Claim__c records exist with Claim__c.Provider__c = terminated provider and Claim__c.Status__c IN ('Submitted', 'In Review', 'Pended'),
Then each Claim__c.Network_Review_Flag__c is set to true and a Task is created for the Claims Operations Manager to review network status impact.

-Given the provider is re-credentialed within 30 days of termination, auto-reinstate the PNA.- (Deprecated: Reinstatement handled via separate re-credentialing flow INS-0411)

## Solution

Use a multi-step Salesforce Approval Process (Provider_Termination_Approval) with approvers: Provider Relations Manager and Compliance Officer. Record-Triggered Flow on Network_Termination_Case__c (Status__c = 'Approved') to set PNA.Pending_Termination__c = true via a related record update loop. Schedulable Apex (ProviderTerminationBatchJob.cls) runs nightly to flip PNA.Status__c = 'Terminated' and create PCN records. Conga CLM callout in Integration Procedure (TerminationNoticeDocGen). Claim flagging via a separate Record-Triggered Flow on PNA (after update, Status__c changed to 'Terminated') that queries and updates related open Claim__c records. All termination events logged to a custom Audit_Log__c object for compliance reporting.

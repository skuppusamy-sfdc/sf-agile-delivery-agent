## Description

As a Credentialing Specialist, I need Salesforce to automatically retrieve and sync provider credentialing data from CAQH ProView into the Credentialing_Case__c object when a new credentialing case is opened, so that specialists do not need to manually re-enter data already attested by the provider in CAQH, reducing data entry errors and accelerating the credentialing cycle time.

CAQH data to be synced includes: license details, DEA certificate, board certifications, malpractice insurance, education/training history, and work history.

## Acceptance Criteria

Given a Credentialing_Case__c record is created with Case_Type__c = 'Initial Credentialing' or 'Re-Credentialing' and Provider__c.CAQH_Provider_ID__c is populated,
When the record is saved,
Then an asynchronous Apex callout is triggered to the CAQH ProView REST API (`/api/providers/{caqhProviderId}/profile`) using named credential `CAQH_ProView_NC`.

Given the CAQH API returns a 200 response,
When the sync completes,
Then the following fields on Credentialing_Case__c are populated:
  - CAQH_Profile_Status__c (from CAQH `profileStatus`)
  - License_Number__c, License_State__c, License_Expiration_Date__c (from primary state license)
  - DEA_Certificate_Number__c, DEA_Expiration_Date__c
  - Malpractice_Carrier__c, Malpractice_Policy_Number__c, Malpractice_Expiration_Date__c
  - CAQH_Last_Sync_DateTime__c = current datetime.

Given the CAQH API returns a 404 (provider not found) or 401 (unauthorized),
When the callout fails,
Then Credentialing_Case__c.CAQH_Sync_Status__c = 'Failed', an error log is written to a custom object `Integration_Error_Log__c`, and a Task is created for the Credentialing Specialist with Subject = 'CAQH Sync Failed – Manual Entry Required'.

Given CAQH returns a re-attestation flag (`reAttestationRequired = true`),
When the sync processes the response,
Then Credentialing_Case__c.CAQH_ReAttestation_Required__c = TRUE and a warning banner is displayed on the case record page via a custom LWC alert component.

Given a Credentialing Specialist clicks the 'Refresh from CAQH' quick action on an existing Credentialing_Case__c,
When the action executes,
Then the same sync process is triggered on demand, CAQH_Last_Sync_DateTime__c is updated, and any field-level changes are tracked in the record's field history.

## Solution

Create an Apex class `CAQHIntegrationService` with a `@future(callout=true)` method invoked from a Record-Triggered Flow on Credentialing_Case__c (after insert, condition: CAQH_Provider_ID__c != null). The service makes an HTTP GET callout to CAQH ProView REST API using Named Credential `CAQH_ProView_NC` with OAuth 2.0 client credentials flow. A custom JSON deserializer class `CAQHProfileResponse` maps the API response to Credentialing_Case__c fields. Error handling writes to `Integration_Error_Log__c` using a separate DML operation in a catch block. The 'Refresh from CAQH' quick action is implemented as a Screen Flow with a single confirmation screen that invokes the same Apex invocable action `CAQHIntegrationService.syncProviderProfile()`. Field History Tracking is enabled on CAQH-sourced fields on Credentialing_Case__c. A Remote Site Setting and CSP Trusted Site entry are configured for the CAQH ProView API endpoint.

## Description

As a Broker Relations Coordinator, I need a structured intake screen to capture Group_Service_Agreement__c contract terms when onboarding a new employer group, so that the GSA is linked to the correct Facility__c (Business Account), broker of record, and associated LOB policies before the group's effective date.

## Acceptance Criteria

Given a Broker Relations Coordinator opens the 'New GSA' quick action on a Facility__c (Business Account) record,
When the intake form loads,
Then the following fields are required: Group_Service_Agreement__c.Effective_Date__c, Group_Service_Agreement__c.Termination_Date__c, Group_Service_Agreement__c.LOB__c, Group_Service_Agreement__c.TPA__c (lookup to Account where Type = 'TPA'), Group_Service_Agreement__c.Broker_of_Record__c (lookup to Contact where Role = 'Broker').

Given Group_Service_Agreement__c.Effective_Date__c is set to a date in the past,
When the coordinator attempts to save,
Then a validation error fires: 'GSA Effective Date cannot be in the past for new agreements.'

Given the GSA is saved with Status__c = 'Draft',
When Group_Service_Agreement__c.Status__c is updated to 'Pending Approval',
Then a Salesforce Approval Process ('GSA_Approval_Process') routes the record to the Underwriting Analyst queue for review.

Given the Underwriting Analyst approves the GSA,
When the approval step completes,
Then Group_Service_Agreement__c.Status__c = 'Active' and Group_Service_Agreement__c.Approved_By__c = [approving user] and Group_Service_Agreement__c.Approval_Date__c = TODAY().

Given an active GSA exists for the Facility__c,
When a second GSA with the same LOB__c and overlapping effective dates is submitted,
Then a duplicate rule fires blocking save with message: 'An active GSA for this LOB already exists for this group. Please review existing agreement [GSA Name].'

## Solution

Screen Flow (GSA_Intake_Flow) launched via Quick Action on Facility__c. Flow uses a Decision element to validate Effective_Date__c >= TODAY(). On save, a Record-Triggered Flow transitions Status__c and submits to the GSA_Approval_Process Approval Process. Duplicate detection via a custom Apex-based DuplicateRule that checks Group_Service_Agreement__c with SOQL filtering on Facility__c + LOB__c + date range overlap. Conga CLM integration queued for Phase 2 document generation.

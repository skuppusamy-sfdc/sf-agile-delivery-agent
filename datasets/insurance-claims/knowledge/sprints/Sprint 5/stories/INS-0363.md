## Description

As a Provider Network Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) and perform a claims impact assessment when a Network_Termination_Case__c is initiated for a provider, so that affected policyholders are notified within regulatory timelines and open claims are flagged for review before the termination effective date.

## Acceptance Criteria

Given a Network_Termination_Case__c record is created with Termination_Case__c.Status__c = 'Initiated' and Termination_Case__c.Termination_Effective_Date__c is populated,
When the record is saved,
Then a Provider_Change_Notification__c record is auto-created with PCN.Type__c = 'Provider Termination', PCN.Provider__c = Termination_Case__c.Provider__c, PCN.Effective_Date__c = Termination_Case__c.Termination_Effective_Date__c, and PCN.Status__c = 'Pending Review'.

Given the PCN is created,
When the system runs the claims impact assessment,
Then all Claim__c records where Claim__c.Rendering_Provider__c = terminated Provider__c AND Claim__c.Status__c IN ('Submitted', 'In Review', 'Pending Adjudication') AND Claim__c.Date_of_Service__c >= Termination_Effective_Date__c are identified and Claim__c.Network_Verification_Required__c = TRUE is set on each.

Given the claims impact count > 0,
When the assessment completes,
Then Termination_Case__c.Impacted_Claims_Count__c is populated with the total count and a chatter post is made on the Termination_Case__c record tagging the assigned Provider Network Specialist.

Given the Termination_Effective_Date__c is within 30 days of today,
When the PCN is created,
Then PCN.Priority__c = 'Urgent' and an email alert is sent to the Compliance Officer role queue using the 'Provider_Termination_Urgent' email template.

Given Termination_Case__c.Termination_Type__c = 'For Cause',
When the PCN is generated,
Then PCN.Requires_Member_Notification__c = TRUE and PCN.Regulatory_Notice_Deadline__c = Termination_Effective_Date__c - 30 days per state compliance rules stored in Custom Metadata `State_Compliance_Rule__mdt`.

-Given the provider has an active Credentialing_Case__c, auto-close it upon termination- (deferred to Phase 2 credentialing workflow; see INS-0389).

Given Termination_Case__c.Status__c transitions to 'Approved',
When the approval is recorded,
Then all associated Provider_Network_Agreement__c records for the provider are updated to PNA.Status__c = 'Terminated' and PNA.Termination_Date__c = Termination_Effective_Date__c.

## Solution

Implement a Record-Triggered Flow on Network_Termination_Case__c (trigger: after insert, entry condition Status__c = 'Initiated') that calls an Apex invocable action `TerminationImpactAssessmentService.execute()`. This service performs a batch SOQL query on Claim__c using the provider NPI and date range, updates affected claims via `Database.update()` with partial success handling, and fires a `Termination_PCN_Created__e` Platform Event to trigger downstream PCN creation and email alerts. The PCN record creation is handled in a separate autolaunched Flow subscribed to the Platform Event, enabling decoupled processing. Custom Metadata `State_Compliance_Rule__mdt` stores per-state notice period requirements (e.g., California = 45 days, Texas = 30 days) used to calculate PCN.Regulatory_Notice_Deadline__c. An Approval Process on Network_Termination_Case__c governs the 'Initiated → Approved' transition with a two-step approval chain (Provider Network Manager → Compliance Officer).

## Description

As a Provider Network Specialist, I need a guided multi-step OmniScript wizard to create a new Provider_Network_Agreement__c record so that all required contracting data — including fee schedule linkage, effective dates, network tier, and LOB assignments — is captured consistently and validated before the agreement is sent for DocuSign execution. Extends INS-0281 (Provider 360 FlexCard foundation).

## Acceptance Criteria

Given a Provider Network Specialist navigates to a Provider__c or Facility__c record and clicks 'Initiate New Agreement',
When the OmniScript launches,
Then Step 1 must pre-populate Provider_Network_Agreement__c.Provider__c or PNA.Facility__c from the originating record context,
And the specialist must not be able to manually edit the NPI__c or TIN__c fields — these must be read-only and sourced from the Account record.

Given the specialist is on Step 2 (Contract Terms),
When PNA.Network_Tier__c = 'Tier 1 — Preferred',
Then the Fee_Schedule__c lookup must be filtered to only show records where Fee_Schedule__c.Tier__c = 'Preferred' and Fee_Schedule__c.Status__c = 'Approved'.

Given the specialist submits Step 3 (Effective Dates),
When PNA.Effective_Date__c is less than TODAY or PNA.Expiration_Date__c is less than PNA.Effective_Date__c,
Then the OmniScript must display an inline validation error and prevent progression to Step 4.

Given all steps are completed and the specialist clicks 'Submit for Execution',
When the OmniScript Integration Procedure calls the DocuSign API,
Then PNA.Contract_Status__c must be set to 'Pending Signature',
And a DocuSign envelope ID must be stored in PNA.DocuSign_Envelope_ID__c,
And a Chatter post must be added to the PNA record tagging the assigned Provider Relations Specialist.

Given PNA.LOB__c is multi-select and includes 'Workers Comp',
When the agreement is submitted,
Then a Compliance Officer must be added as a record follower automatically via a Record-Triggered Flow,
And PNA.Compliance_Review_Required__c must be set to TRUE.

## Solution

Build a 5-step OmniScript (Type: ProviderNetworkAgreement, SubType: Create) using OmniStudio. DataRaptors will handle pre-population from the Provider__c / Facility__c context record. An Integration Procedure will orchestrate: (1) duplicate PNA check via SOQL, (2) DocuSign envelope creation via named credential callout, (3) Chatter post via ConnectApi. A custom LWC wrapper will embed the OmniScript on both the Provider__c and Facility__c Lightning record pages via dynamic actions. Permission Set Group 'PNA_Contracting_PSG' will gate access. Conga CLM template merge will generate the contract PDF prior to DocuSign send.

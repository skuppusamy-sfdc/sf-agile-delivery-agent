## Description

As a Provider Relations Specialist, I need automated alerts when a Provider_Network_Agreement__c is approaching expiration and when its linked Fee_Schedule__c records are expiring, so that I can initiate renewal negotiations in time to avoid gaps in network coverage that could cause incorrect claim payment or provider disruption.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Expiration_Date__c is within 90 days of TODAY and PNA.Contract_Status__c = 'Active',
When the nightly scheduled Flow runs,
Then a Task must be created with Subject = 'PNA Renewal Required — 90 Day Notice', assigned to PNA.Owner, with ActivityDate = TODAY + 5,
And PNA.Renewal_Alert_Stage__c must be set to '90 Day'.

Given PNA.Expiration_Date__c is within 30 days and PNA.Renewal_Alert_Stage__c != '30 Day',
When the nightly scheduled Flow runs,
Then an additional Task must be created with Priority = 'High',
And an email alert must be sent to PNA.Owner and the Provider Relations Specialist queue,
And PNA.Renewal_Alert_Stage__c must be updated to '30 Day'.

Given a Fee_Schedule__c record where Fee_Schedule__c.Expiration_Date__c is within 60 days and Fee_Schedule__c.Status__c = 'Approved',
When the nightly scheduled Flow runs,
Then a Case must be created with RecordType = 'Fee Schedule Renewal', linked to the parent PNA, and assigned to the Provider Network Specialist queue.

Given PNA.Expiration_Date__c has passed and PNA.Contract_Status__c is still 'Active',
When the nightly Flow evaluates the record,
Then PNA.Contract_Status__c must automatically transition to 'Expired',
And all linked Fee_Schedule__c records must have Fee_Schedule__c.Status__c set to 'Inactive',
And any open Claim_Line__c records referencing this PNA must have Claim_Line__c.Pricing_Exception__c = TRUE.

-Given PNA.Auto_Renew__c = TRUE the system should extend Expiration_Date__c by 12 months without specialist action — removed pending legal review-

## Solution

Build a Scheduled Flow ('PNA_Expiration_Alert_Flow') running nightly at 2:00 AM EST. The flow uses Get Records to batch-query PNAs and Fee Schedules meeting threshold criteria, then loops to create Tasks and Cases. A separate Apex Schedulable class (FeeScheduleExpirationBatch) handles the bulk Fee_Schedule__c status update to 'Inactive' and cascades Claim_Line__c Pricing_Exception__c flags via a SOQL for-loop with Database.update(). Custom notification types will be configured for in-app alerts. A custom report type 'PNA Expiration Dashboard' will be built for Provider Relations leadership visibility.

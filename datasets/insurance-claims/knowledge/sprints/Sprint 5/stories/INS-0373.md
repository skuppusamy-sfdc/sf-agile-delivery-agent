## Description

As a Provider Network Specialist, I need to create and version Fee Schedule records linked to a Provider_Network_Agreement__c so that reimbursement rates are tracked historically and claims adjudication always references the correct effective rate for a given date of service.

Extends INS-0351 (PNA record structure baseline).

## Acceptance Criteria

Given a Provider_Network_Agreement__c record in 'Active' status,
When a Provider Network Specialist creates a new Fee_Schedule__c record,
Then Fee_Schedule__c.Provider_Network_Agreement__c must be populated, Fee_Schedule__c.Effective_Date__c must be a past or present date, and Fee_Schedule__c.Status__c defaults to 'Draft'.

Given a Fee_Schedule__c record in 'Draft' status,
When the user clicks 'Activate',
Then Fee_Schedule__c.Status__c transitions to 'Active' and Fee_Schedule__c.Activated_By__c is stamped with the running user's Id via a Record-Triggered Flow.

Given an 'Active' Fee_Schedule__c record,
When a new Fee_Schedule__c with the same Fee_Schedule__c.LOB__c and overlapping effective date range is activated on the same PNA,
Then the system must auto-set the prior record's Fee_Schedule__c.Status__c to 'Superseded' and populate Fee_Schedule__c.Superseded_Date__c with today's date.

Given any Fee_Schedule__c record,
When Fee_Schedule__c.Expiration_Date__c is reached,
Then a scheduled Apex job must transition Fee_Schedule__c.Status__c to 'Expired' and fire a Platform Event (Fee_Schedule_Expired__e) consumed by the claims adjudication engine.

-Given a Fee_Schedule__c, the CPT code list must be stored as a child object CPT_Rate__c- (deprecated — CPT rates now stored on Claim_Line__c.Allowed_Amount__c lookup; see INS-0362)

Given a user without the 'Fee_Schedule_Admin' Permission Set Group,
When they attempt to edit Fee_Schedule__c.Rate_Amount__c on an 'Active' record,
Then a field-level security restriction must block the edit and display a toast error.

## Solution

1. Create Fee_Schedule__c object with fields: Effective_Date__c (Date), Expiration_Date__c (Date), Status__c (Picklist: Draft/Active/Superseded/Expired), LOB__c (Picklist), Rate_Amount__c (Currency), Activated_By__c (Lookup User), Superseded_Date__c (Date), Provider_Network_Agreement__c (Master-Detail).
2. Record-Triggered Flow on Fee_Schedule__c (after save) handles status stamping on activation.
3. Apex Trigger (before insert/update) enforces no overlapping active fee schedules per LOB per PNA — throws AuraHandledException with user-friendly message.
4. Scheduled Apex (BatchFeeScheduleExpiry) runs nightly, queries Fee_Schedule__c WHERE Expiration_Date__c = TODAY AND Status__c = 'Active', updates status and publishes Fee_Schedule_Expired__e Platform Event.
5. Permission Set Group 'Fee_Schedule_Admin' restricts FLS on Rate_Amount__c for read-only profiles.
6. All status transitions logged via a custom History tracking object Fee_Schedule_Audit__c.

## Description

As a Claims Operations Manager, I need the Agentforce AI assistant to analyze stalled FNOL claims and recommend a prioritized next-best action (e.g., request additional documentation, escalate to SIU, schedule a provider call) so that claims examiners can reduce average cycle time on first notice of loss claims and meet our 30-day FNOL resolution SLA.

## Acceptance Criteria

Given a Claim__c record where Claim__c.FNOL_Received_Date__c is populated, Claim__c.Status__c = 'FNOL Received' or 'Pending Documentation', and no Claim__c activity (Tasks, Emails, or status change) has occurred in the last 5 business days,
When the Agentforce Claims AI panel is opened on the Claim__c record page,
Then the AI must surface at least one next-best action recommendation with an action label, rationale text, and a confidence score displayed as a percentage.

Given the AI recommendation is 'Request Additional Documentation',
When the examiner clicks 'Accept & Execute',
Then an OmniScript 'RequestDocumentation' must launch pre-populated with Claim__c.Claimant_Name__c, Claim__c.Claim_Number__c, and Claim__c.Missing_Documents__c, and a Task must be created with Task.Subject = 'Documentation Request Sent' linked to the Claim__c.

Given the AI recommendation is 'Escalate to SIU',
When the examiner clicks 'Accept & Execute',
Then Claim__c.SIU_Referral_Flag__c must be set to true, Claim__c.Status__c must change to 'SIU Review', and a Case must be created and linked to the Claim__c with Case.RecordType = 'SIU Investigation'.

Given an examiner dismisses a recommendation,
When the dismiss action is taken,
Then the dismissal must be logged in a custom object 'AI_Recommendation_Log__c' with fields: Claim__c, Recommendation_Type__c, Confidence_Score__c, Action_Taken__c = 'Dismissed', Dismissed_By__c, and Dismissed_At__c.

Given the AI panel loads on a Claim__c where Claim__c.Status__c = 'Adjudicated' or 'Closed',
When the panel renders,
Then no recommendations must be displayed and a message 'No actions required — claim is resolved' must appear.

Given any AI recommendation is accepted or dismissed,
When 30 days of data has accumulated,
Then a Recommendation Accuracy report on AI_Recommendation_Log__c must be available in the 'Claims AI Performance' dashboard showing acceptance rate, dismissal rate, and average confidence score by Recommendation_Type__c.

## Solution

Configure an Agentforce Agent 'ClaimsExaminerAssistant' with a custom Topic 'FNOL Stalled Claim Analysis'. Define an Agent Action backed by an Apex @InvocableMethod 'FNOLStalledClaimsAdvisor.getRecommendations(claimId)' that queries Claim__c fields, related Task history, and computes staleness using business-day logic. Return a List<RecommendationWrapper> with fields: actionLabel, rationale, confidenceScore. Surface recommendations in a custom LWC 'agentforceClaimsPanel' embedded on the Claim__c Lightning Record Page, consuming the Agentforce Einstein Copilot API response via @wire. Implement 'Accept & Execute' actions by invoking relevant OmniScripts or Record-Triggered Flows via LWC imperative Apex calls. Log all interactions to AI_Recommendation_Log__c via a dedicated Apex service class 'AIRecommendationLogger'. Build the 'Claims AI Performance' CRM Analytics dashboard on top of AI_Recommendation_Log__c using SAQL queries. Restrict the Agentforce panel to Claims_Operations_PSG Permission Set Group.

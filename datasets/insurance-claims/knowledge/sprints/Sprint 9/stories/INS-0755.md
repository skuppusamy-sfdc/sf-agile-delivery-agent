## Description

As a Claims Operations Manager, I need a consolidated Policyholder 360 FlexCard that surfaces active Policy__c records, open Claim__c records, and a coverage history summary so that I can assess a policyholder's full profile without navigating between multiple objects during adjudication.

## Acceptance Criteria

Given a policyholder record (Person Account) is opened in Salesforce,
When the Policyholder 360 FlexCard loads,
Then it must display all active Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.PolicyHolder__c = current Account Id.

Given the FlexCard is rendered,
When the Claims panel is expanded,
Then all Claim__c records where Claim__c.Status__c IN ('Open','Pending Review','In Adjudication') and Claim__c.Insured__c = current Account Id must be listed with fields: Claim__c.Name, Claim__c.Date_of_Loss__c, Claim__c.LOB__c, Claim__c.Total_Billed_Amount__c, Claim__c.Assigned_Adjuster__c.

Given a policy has an associated Fee_Schedule__c,
When the Coverage Summary tab is selected,
Then the FlexCard must display Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.Expiration_Date__c, and Fee_Schedule__c.Network_Tier__c grouped by Policy__c.LOB__c.

Given the policyholder has COB (Coordination of Benefits) flags,
When COB indicator is present (Policy__c.COB_Flag__c = true),
Then a prominent COB banner must appear at the top of the FlexCard with primary and secondary payer details.

-Given a policyholder has a TPA assignment, a TPA badge must render in the header (superseded by INS-0731 TPA panel).-

Given the FlexCard is viewed by a user without the 'Claims_Adjuster' Permission Set Group,
When the card loads,
Then financial fields (Claim__c.Total_Paid_Amount__c, Payment__c records) must be hidden via field-level security.

## Solution

Build an OmniStudio FlexCard 'Policyholder360Card' with child DataRaptors: DR_GetActivePolicies (SOQL on Policy__c), DR_GetOpenClaims (SOQL on Claim__c with status filter), DR_GetCoverageSummary (join Policy__c to Fee_Schedule__c). Use FlexCard states to toggle between Claims, Coverage, and COB tabs. COB banner implemented as a conditional block driven by Policy__c.COB_Flag__c. Field visibility enforced via Permission Set Group 'Claims_Adjuster_PSG' and OmniStudio data mask configuration. Deploy via OmniStudio Migration Tool to production.

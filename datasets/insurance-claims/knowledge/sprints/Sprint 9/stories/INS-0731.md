## Description

As a Claims Operations Manager, I need a unified Policyholder 360 FlexCard embedded on the Provider__c (Person Account) record page so that I can instantly view a policyholder's active coverage, recent claims, and payment history without navigating across multiple objects. This story extends INS-0701 which established the base OmniStudio layout framework.

## Acceptance Criteria

Given a Person Account record where Account.RecordType.DeveloperName = 'Policyholder', when a Claims Operations Manager opens the record, then the Policyholder 360 FlexCard must render within 2 seconds and display all active Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.PolicyHolder__c = current Account Id.

Given the FlexCard is loaded, when the 'Coverage Summary' section is expanded, then it must display Policy__c.LOB__c, Policy__c.EffectiveDate__c, Policy__c.TerminationDate__c, Policy__c.GroupNumber__c, and Policy__c.TPA__c lookup fields in a structured card layout.

Given the FlexCard is loaded, when the 'Claims Timeline' tab is selected, then all Claim__c records where Claim__c.Insured__c = current Account Id must be rendered in descending order by Claim__c.DateOfService__c, showing Claim__c.ClaimNumber__c, Claim__c.Status__c, Claim__c.TotalBilledAmount__c, and Claim__c.TotalPaidAmount__c.

Given a Claim__c record with Claim__c.Status__c = 'Adjudicated', when it appears on the timeline, then a green status badge must be displayed; when Claim__c.Status__c = 'Denied', a red badge must display; when Claim__c.Status__c = 'Pending' or 'In Review', a yellow badge must display.

Given the FlexCard is displayed, when the policyholder has no active Policy__c records, then a contextual empty-state message 'No active coverage found. Contact Broker Relations for assistance.' must render in place of the coverage summary panel.

-Given the FlexCard loads, it must also display Provider__c network tier from PNA__c — deprecated, moved to Provider 360 epic-

Given a Claims Operations Manager clicks a Claim__c.ClaimNumber__c hyperlink within the timeline, then the user must be navigated to the corresponding Claim__c detail record page in the same browser tab.

## Solution

Build an OmniStudio FlexCard named 'Policyholder360Card' with two child cards: 'CoverageSummaryCard' and 'ClaimsTimelineCard'. Use DataRaptor Extract actions to query Policy__c and Claim__c with appropriate SOQL filter conditions. Embed the FlexCard on the Person Account Lightning Record Page using the OmniStudio FlexCard component. Implement conditional rendering using FlexCard state actions to drive badge color logic based on Claim__c.Status__c values. Use a DataRaptor Transform to flatten nested Policy__c and Claim__c fields into a single response payload for rendering performance. Apply Permission Set Group 'Claims_Ops_Standard' to control visibility. Cache DataRaptor responses using OmniStudio DataRaptor cache keys with a 60-second TTL to meet the 2-second render SLA.

# INS-0732: Implement coverage history tab on Policyholder record showing all historical Policy__c versions with LOB breakdown

## Description

As an Underwriting Analyst, I need a dedicated Coverage History tab on the Policyholder (Person Account) record page that displays all historical and current Policy__c records grouped by LOB so that I can perform accurate COB analysis and identify coverage gaps during the underwriting review process.

## Acceptance Criteria

Given a Person Account record with RecordType 'Policyholder', when the 'Coverage History' tab is selected, then a Lightning Web Component (LWC) must display all Policy__c records associated via Policy__c.PolicyHolder__c regardless of Policy__c.Status__c value, sorted by Policy__c.EffectiveDate__c descending.

Given the coverage history list is rendered, when policies are displayed, then records must be visually grouped by Policy__c.LOB__c values (e.g., 'Medical', 'Dental', 'Vision', 'Workers Comp') using collapsible accordion sections per LOB group.

Given the coverage history tab is loaded, when a Policy__c record has Policy__c.Status__c = 'Terminated', then Policy__c.TerminationDate__c and Policy__c.TerminationReason__c must be displayed in a 'Terminated' sub-row beneath the policy header.

Given more than 50 Policy__c records exist for a single policyholder, when the tab loads, then pagination must be implemented with a default page size of 20 records and the user must be able to navigate forward and backward through pages without a full page reload.

Given the coverage history is displayed, when the Underwriting Analyst selects the 'Export' button, then a CSV file must be generated containing all Policy__c fields: Policy__c.PolicyNumber__c, Policy__c.LOB__c, Policy__c.EffectiveDate__c, Policy__c.TerminationDate__c, Policy__c.TPA__c, Policy__c.GroupNumber__c, Policy__c.Status__c, and Policy__c.AnnualPremium__c.

## Solution

Develop an LWC named 'policyholderCoverageHistory' using @wire adapter with a custom Apex controller method 'getCoverageHistoryByPolicyholder(Id accountId)' that queries Policy__c with SOQL including all relevant fields and no Status__c filter. Use JavaScript groupBy logic client-side to bucket policies by LOB__c. Implement lightning-accordion for LOB groupings. Add server-side pagination using OFFSET/LIMIT in the Apex query. For CSV export, use a client-side JavaScript Blob generation approach to avoid governor limits. Deploy on the Person Account Lightning Record Page as a tab via App Builder. Apply field-level security via the 'Underwriting_Standard' Permission Set to restrict Policy__c.AnnualPremium__c visibility to authorized roles only.

# INS-0748: Display full coverage history timeline on Policyholder record page

## Description

As a Claims Operations Manager, I need a chronological coverage history timeline on the policyholder record page that shows all historical Policy__c enrollments, LOB changes, and termination events so that I can quickly assess continuity of coverage when evaluating a claim for eligibility and COB purposes.

## Acceptance Criteria

Given a policyholder has more than one Policy__c record (active or inactive),
When the Coverage History Timeline LWC loads on the Person Account page,
Then all Policy__c records associated via Policy__c.Insured_Contact__c must be displayed in descending order by Policy__c.Effective_Date__c.

Given each Policy__c record in the timeline,
When rendered,
Then the timeline node must show Policy__c.LOB__c, Policy__c.Group_Plan_Name__c, Policy__c.Effective_Date__c, Policy__c.Termination_Date__c, and Policy__c.Termination_Reason__c.

Given a gap of more than 30 days exists between consecutive Policy__c.Termination_Date__c and the next Policy__c.Effective_Date__c,
When the timeline renders,
Then a visual gap indicator labeled 'Coverage Gap: X days' must appear between the two timeline nodes.

Given a Policy__c record has Policy__c.COB_Indicator__c = true,
When that node renders,
Then a 'COB' badge must be displayed on the timeline node.

Given the timeline contains more than 10 Policy__c records,
When the component loads,
Then only the 10 most recent records are shown by default with a 'Show All' toggle to load the full history.

## Solution

Develop a custom LWC 'coverageHistoryTimeline' using a Wire Adapter calling an Apex @AuraEnabled method 'getCoverageHistory(recordId)' that queries Policy__c WHERE Insured_Contact__c = :recordId ORDER BY Effective_Date__c DESC. Compute coverage gaps in Apex by iterating sorted policy list and comparing Termination_Date__c to next Effective_Date__c. Return a wrapper list with a computed 'gapDays' integer. Render the timeline using SLDS Timeline component markup. Use a @track showAll boolean to toggle between sliced (10) and full list. Place LWC on Person Account Lightning Record Page via App Builder, restricted to Claims_Operations_PSG and Provider_Relations_PSG Permission Set Groups.

## Description

**Bug Report — Fee Schedule Null Rate on Modified CPT Codes**

**Steps to Reproduce:**
1. Create a Claim_Line__c record with Claim_Line__c.CPT_Code__c = '99213' and Claim_Line__c.CPT_Modifier__c = '25'.
2. Associate the Claim_Line__c to a Claim__c with a valid Fee_Schedule__c linked via the Group_Service_Agreement__c.
3. Trigger adjudication by setting Claim__c.Status__c = 'Ready for Adjudication' and saving.
4. Observe Claim_Line__c.Allowed_Amount__c after the adjudication flow completes.

**Expected Behavior:** The FeeScheduleEngine should match on the composite key of CPT_Code__c + CPT_Modifier__c (e.g., '99213-25') and return the Fee_Schedule__c.Allowed_Rate__c for that modifier-specific line. If no modifier-specific rate exists, it should fall back to the base CPT code rate.

**Actual Behavior:** Claim_Line__c.Allowed_Amount__c = null. The SOQL query in 'FeeScheduleLookupService.cls' (line 84) constructs the lookup key using only CPT_Code__c, ignoring CPT_Modifier__c entirely, resulting in no match when the Fee_Schedule__c record is keyed on the composite '99213-25' value. This causes the adjudication flow to skip rate assignment and leave Allowed_Amount__c null, which subsequently causes Payment__c creation to fail with a required field validation error.

**Affected Records:** Estimated 340 Claim_Line__c records in the current sprint cycle with non-null CPT_Modifier__c values.

**Environment:** Full sandbox (FS01). Introduced in the FeeScheduleEngine refactor deployed in Sprint 8.

## Acceptance Criteria

as per Description

## Solution

Update 'FeeScheduleLookupService.cls' line 84 to construct the composite lookup key as: String lookupKey = String.isNotBlank(claimLine.CPT_Modifier__c) ? claimLine.CPT_Code__c + '-' + claimLine.CPT_Modifier__c : claimLine.CPT_Code__c; Modify the SOQL query to first attempt a match on Fee_Schedule__c.Rate_Key__c = compositeKey, then fall back to base CPT code if no result. Add Apex unit test 'FeeScheduleLookupServiceTest.cls' covering: (a) modifier-specific match, (b) fallback to base CPT, (c) no match returns zero rate with error flag. Run a Data Loader bulk update on the 340 affected Claim_Line__c records in FS01 post-fix to re-trigger adjudication. Coordinate with Claims Operations Manager to identify any Payment__c records blocked by the null Allowed_Amount__c validation and reprocess.

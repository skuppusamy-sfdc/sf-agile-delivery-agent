## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can instantly see the provider's active network agreements, associated fee schedules, NPI/TIN verification status, and any open credentialing cases without switching between multiple tabs. Extends INS-0709 Provider base FlexCard foundation.

## Acceptance Criteria

Given a Provider__c record where the associated Provider_Network_Agreement__c.Status__c = 'Active', when the Provider 360 FlexCard loads, then it must display PNA__c.Contract_Start_Date__c, PNA__c.Contract_End_Date__c, PNA__c.Network_Tier__c, and PNA__c.Specialty__c.

Given a Fee_Schedule__c record linked to the active PNA via Fee_Schedule__c.PNA__c, when the fee schedule section renders, then Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.Fee_Schedule_Type__c must be visible.

Given the Provider__c record has one or more Credentialing_Case__c records with Credentialing_Case__c.Status__c IN ('Open','Pending Review','Awaiting CAQH'), when the credentialing section renders, then each open case must display Credentialing_Case__c.Case_Number__c, Credentialing_Case__c.Credentialing_Type__c, and Credentialing_Case__c.Due_Date__c with a color-coded urgency indicator (red if due within 7 days).

Given the Provider__c.NPI__c is not validated against the NPPES integration, when the card loads, then a 'NPI Unverified' warning badge must appear on the provider header.

-Given the provider has an open Network_Termination_Case__c, display a termination alert banner on the card- (moved to INS-0744 termination workflow story)

Given a Provider Relations Specialist with the 'Provider_Network_ReadWrite' Permission Set Group, when they click 'Initiate Credentialing' on the FlexCard action bar, then a new Credentialing_Case__c must be created with Credentialing_Case__c.Provider__c populated and Status__c = 'Open'.

## Solution

Extend the existing OmniStudio FlexCard 'ProviderBase_Card' (built in INS-0709) by adding three new child DataRaptor Turbo Extract sources: one for Provider_Network_Agreement__c, one for Fee_Schedule__c (joined via PNA), and one for Credentialing_Case__c. Use FlexCard conditional states to drive the 'NPI Unverified' badge based on a Boolean field Provider__c.NPI_Verified__c. Implement the urgency color logic using a FlexCard custom style with a conditional expression on Credentialing_Case__c.Due_Date__c. The 'Initiate Credentialing' action should invoke an OmniScript 'InitiateCredentialing_OS' via a FlexCard action of type 'OmniScript'. Wrap NPPES validation status in a nightly scheduled Apex batch that updates Provider__c.NPI_Verified__c.

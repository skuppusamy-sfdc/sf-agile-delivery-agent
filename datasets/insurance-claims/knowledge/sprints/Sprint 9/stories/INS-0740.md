# INS-0740: Display full coverage history timeline on Policyholder record with LOB and effective date ranges

## Description

As a Claims Operations Manager, I need a visual coverage history timeline on the policyholder's record so that I can review all historical and current Policy__c enrollments, LOB changes, and gap periods without submitting a data request to the Underwriting team.

## Acceptance Criteria

Given a Person Account representing a policyholder has two or more associated Policy__c records, when the Coverage History Timeline LWC loads on the Account record page, then each policy period must be rendered as a horizontal timeline bar showing Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Termination_Date__c, and Policy__c.Status__c.

Given a gap exists between Policy__c.Termination_Date__c of one record and Policy__c.Effective_Date__c of the next, when the timeline renders, then the gap period must be visually indicated with a distinct 'Coverage Gap' styling and the gap duration in days must be displayed.

Given the policyholder has a Policy__c with LOB__c = 'Medicare Advantage', when that bar is rendered, then a CMS indicator icon must appear alongside the LOB label.

Given the user clicks on any timeline bar, when the detail panel opens, then it must display Policy__c.Group_Service_Agreement__c (lookup to GSA), Policy__c.TPA__c, Policy__c.Premium_Amount__c, and a hyperlink to the full Policy__c record.

Given the timeline contains more than 10 policy records, when the component loads, then only the 10 most recent records must display by default with a 'Show All History' toggle.

## Solution

Build a custom LWC named 'coverageHistoryTimeline' using a CSS-driven horizontal bar layout. Wire an Apex Controller method 'getCoverageHistory(accountId)' that queries Policy__c WHERE Policy__c.Policyholder__c = :accountId ORDER BY Effective_Date__c DESC. Compute gap periods in Apex by sorting records and comparing adjacent Termination_Date__c and Effective_Date__c values; return a wrapper class including a 'GapDays' integer field. Use a @wire adapter to call the Apex method reactively. Deploy LWC to the Person Account Lightning Record Page via App Builder with a 'Policyholder' audience filter on the page assignment.

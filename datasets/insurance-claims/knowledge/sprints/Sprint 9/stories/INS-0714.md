# INS-0714: Expose Group Service Agreement (GSA) coverage tier details on employer group portal

- **Sprint**: Acme Insurance Sprint 9 (Phase 2)

- **Labels**: Phase2, GSA, EmployerPortal

## Description

As a Broker Relations Coordinator, I need employer group administrators to be able to view their active Group_Service_Agreement__c (GSA) details, enrolled member counts by coverage tier, and upcoming renewal dates through the employer group portal so that brokers spend less time fielding routine GSA status inquiries and employer groups can self-manage basic coverage information.

## Acceptance Criteria

Given an employer group administrator logs into the Experience Cloud employer portal,
When they navigate to the 'Plan & Agreement' section,
Then all Group_Service_Agreement__c records where GSA.Group_Account__c = the logged-in user's Account AND GSA.Status__c IN ('Active', 'Pending Renewal') are displayed.

Given a GSA record is displayed,
When the administrator expands the coverage tier panel,
Then the following GSA fields are shown: GSA.Agreement_Number__c, GSA.Effective_Date__c, GSA.Renewal_Date__c, GSA.Plan_Type__c, GSA.Total_Enrolled_Members__c, GSA.Employee_Only_Count__c, GSA.Employee_Spouse_Count__c, GSA.Family_Count__c.

Given GSA.Renewal_Date__c is within 90 days,
When the GSA card renders,
Then a banner displays: 'Your agreement renews on [Renewal_Date__c]. Contact your broker to review renewal options.' with a 'Contact Broker' button that pre-populates a Case__c of RecordType 'GSA_Renewal_Inquiry' with GSA.Agreement_Number__c in the subject.

Given the employer portal user's profile,
When the GSA component loads,
Then GSA.Contracted_Rate__c and GSA.Administrative_Fee__c fields are NOT rendered (financial terms visible to brokers only via internal portal), enforced via field-level security on the 'Employer_Portal_User' Profile.

Given an employer administrator clicks 'Download Agreement Summary',
When the action fires,
Then a Conga CLM PDF is generated from the GSA record using template 'GSA_Employer_Summary_v1' and downloaded to the user's browser within 10 seconds.

-Given GSA.Status__c = 'Expired', display the full historical agreement details- (descoped to Phase 3 per Broker Relations Coordinator prioritization review)

## Solution

Build an LWC 'gsaEmployerPortalView' deployed to the Experience Cloud Employer Portal site. Use @wire with Apex controller 'GSAPortalController' method 'getEmployerGSAs(accountId)' with WITH SECURITY_ENFORCED and explicit FLS checks to exclude financial fields. Renewal banner logic implemented client-side in JavaScript using date comparison against today's date. 'Contact Broker' button fires a createRecord LDS call to instantiate Case__c with RecordTypeId resolved via a Custom Label 'GSA_Renewal_Case_RecordType_Id'. Conga CLM PDF generation uses the same Apex callout pattern established in INS-0712 ('generateGSAPdf' invocable action). Portal sharing enforced via a Sharing Rule on Group_Service_Agreement__c: 'Share GSA with Group Account portal users' using Account-based criteria. Permission Set 'Employer_Portal_GSA_Viewer' controls component access and is assigned during Group/Employer Onboarding via a Record-Triggered Flow on Account when Account.Type__c = 'Employer Group'.

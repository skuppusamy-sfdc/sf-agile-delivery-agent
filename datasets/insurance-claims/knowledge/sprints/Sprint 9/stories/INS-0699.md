## Description

As a Claims Operations Manager, I need a unified Policyholder 360 FlexCard on the Policy__c record page so that I can view the policyholder's active coverage details, open claims, and recent EOB history in a single consolidated view without navigating across multiple objects.

This story extends the foundational layout work completed in INS-0669 and adds the coverage history panel as a new tab within the existing FlexCard framework.

## Acceptance Criteria

Given a Claims Operations Manager is viewing a Policy__c record,
When the Policyholder 360 FlexCard loads,
Then the 'Coverage Summary' tab must display Policy__c.Coverage_Start_Date__c, Policy__c.Coverage_End_Date__c, Policy__c.LOB__c, Policy__c.Group_Number__c, and Policy__c.Plan_Type__c.

Given the policyholder has one or more Claim__c records linked via Claim__c.Policy__c,
When the 'Open Claims' tab is selected,
Then only Claim__c records where Claim__c.Status__c IN ('Submitted', 'In Review', 'Pending Info') are displayed, sorted by Claim__c.Date_of_Service__c descending.

Given the policyholder has Explanation_of_Benefits__c records,
When the 'EOB History' tab is selected,
Then up to 10 most recent EOB__c records are shown, displaying EOB__c.EOB_Date__c, EOB__c.Total_Billed__c, EOB__c.Total_Allowed__c, EOB__c.Member_Responsibility__c, and a download link for EOB__c.Document_URL__c.

Given a Policy__c record where Policy__c.COB_Indicator__c = true,
When the FlexCard loads,
Then a 'COB Active' badge is displayed prominently in the header section of the FlexCard.

-Given the FlexCard loads, a 'Lifetime Benefit Used' progress bar shall be rendered using Policy__c.Lifetime_Max__c and Policy__c.Lifetime_Used__c — deprecated in favor of separate Benefit Utilization LWC (see INS-0703)-

Given the user does not have the 'Claims_Ops_Full_Access' Permission Set Group,
When the FlexCard renders,
Then the EOB download link is hidden and replaced with a 'Request Access' tooltip.

## Solution

Build OmniStudio FlexCard 'Policyholder360Card' with three child cards (CoverageTab, OpenClaimsTab, EOBHistoryTab). Use DataRaptor Extract actions to query Policy__c, Claim__c, and Explanation_of_Benefits__c with SOQL filters on Status__c and relationship fields. Implement conditional rendering using FlexCard State Actions for the COB badge. Permission visibility controlled via OmniStudio DataRaptor input parameter mapped to running user's Permission Set Group membership checked via Apex @AuraEnabled method. Deploy to the Policy__c Lightning Record Page via App Builder with audience targeting for Claims Operations Manager profile.

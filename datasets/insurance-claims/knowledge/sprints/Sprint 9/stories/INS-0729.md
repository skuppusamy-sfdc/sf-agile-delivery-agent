## Description

As a Claims Operations Manager, I need a guided OmniScript that walks payment staff through the ERA reconciliation process — matching incoming ERA 835 transaction data to Payment__c and Claim_Line__c records — so that disbursement discrepancies are identified, documented, and escalated to the SIU when thresholds are exceeded.

## Acceptance Criteria

Given an ERA file has been ingested and Payment__c records created with Payment__c.ERA_Transaction_ID__c populated and Payment__c.Reconciliation_Status__c = 'Pending', when the OmniScript is launched from the Payment__c record, then Step 1 displays a summary DataRaptor-sourced table of all Claim_Line__c records associated with the Payment__c, showing Claim_Line__c.CPT_Code__c, Claim_Line__c.Billed_Amount__c, Claim_Line__c.Allowed_Amount__c, and Claim_Line__c.Paid_Amount__c.

Given the OmniScript Step 1 data is displayed, when the user proceeds to Step 2, then the ERA expected payment amount (Payment__c.ERA_Expected_Amount__c) is compared to Payment__c.Actual_Disbursement_Amount__c and the variance is surfaced as a calculated field.

Given a variance of less than $50.00 between ERA_Expected_Amount__c and Actual_Disbursement_Amount__c, when the user submits the OmniScript, then Payment__c.Reconciliation_Status__c is set to 'Reconciled' and Payment__c.Reconciliation_Date__c is stamped.

Given a variance of $50.00 or greater, when the user proceeds to Step 3, then a mandatory 'Discrepancy Reason' picklist (mapped to Payment__c.Discrepancy_Reason__c) and free-text notes field are required before submission.

Given a variance exceeding $500.00, when the OmniScript is submitted, then an SIU referral case (Case record with RecordType = 'SIU_Referral') is automatically created linked to the Payment__c and Claim__c records, and the SIU queue is notified via a Platform Event 'SIU_Referral__e'.

Given the OmniScript is completed successfully for any variance tier, when submission is confirmed, then a finalized Explanation_of_Benefits__c record is created or updated with EOB_Status__c = 'Finalized' and linked to the reconciled Payment__c.

## Solution

OmniScript 'ERA_Reconciliation_OS' built with three steps: (1) DataRaptor Extract for Claim_Line__c summary, (2) formula element for variance calculation using Payment__c fields, (3) conditional branching based on variance thresholds using OmniScript Conditional Views. DataRaptor Transform writes reconciliation outcomes back to Payment__c and Explanation_of_Benefits__c. SIU referral logic implemented via an Integration Procedure that creates the SIU Case record and publishes 'SIU_Referral__e' Platform Event. Platform Event subscriber (Apex trigger on SIU_Referral__e) routes to the SIU queue using Assignment Rules. OmniScript launched via a custom Quick Action on the Payment__c record page.

## Description

As a Provider Network Specialist, I need automated in-app notifications and task creation when a Provider_Network_Agreement__c (PNA) is approaching expiration so that I can proactively initiate contract renewals and avoid unintended network gaps that could delay claims adjudication. Extends INS-0717.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA__c.End_Date__c is within 90 days of today and PNA__c.Status__c = 'Active',
When the nightly scheduled Apex job runs,
Then a Platform Event 'PNA_Expiration_Alert__e' must be published with fields: PNA_Id__c, Provider_Id__c, End_Date__c, and Days_Until_Expiration__c.

Given the Platform Event 'PNA_Expiration_Alert__e' is published,
When the subscriber Process/Flow triggers,
Then a Task must be created with Task.Subject = 'PNA Renewal Required', Task.ActivityDate = PNA__c.End_Date__c minus 30 days, Task.WhatId = PNA__c.Id, and Task.OwnerId = PNA__c.Network_Specialist__c.

Given the Task is created,
When the assigned Provider Network Specialist logs into Salesforce,
Then a Bell Notification (Custom Notification Type 'PNA_Expiration_Notice') must appear with the provider name and days remaining.

Given PNA__c.Status__c is changed to 'Renewed' or 'Terminated' before the expiration date,
When the Record-Triggered Flow evaluates,
Then any open Tasks related to that PNA must be closed with Task.Status = 'Completed' and Task.Description updated to 'Auto-closed: PNA status updated'.

-Given PNA expiration is within 30 days, escalate to the Provider Relations Director via email (deprecated — escalation now handled by Case escalation rule in INS-0717)-

## Solution

Create a Schedulable Apex class 'PNAExpirationAlertBatch' implementing Database.Batchable and Schedulable, querying PNA__c WHERE End_Date__c <= NEXT_N_DAYS:90 AND Status__c = 'Active'. For each record, publish a PNA_Expiration_Alert__e Platform Event via EventBus.publish(). Build a Record-Triggered Flow on PNA_Expiration_Alert__e (after-save) to create the Task and fire a Custom Notification using the 'Send Custom Notification' action targeting the PNA__c.Network_Specialist__c user. Build a second Record-Triggered Flow on PNA__c (after-update) to auto-close Tasks when Status__c changes to 'Renewed' or 'Terminated'. Schedule the batch nightly at 02:00 AM via Anonymous Apex in deployment script.

# INS-0697: Surface provider network termination risk alerts on Claim_Line__c adjudication screen

## Description

As a Claims Operations Manager, I need real-time network termination risk alerts surfaced on the Claim_Line__c adjudication screen when the billing provider has an open Network_Termination_Case__c or a PNA approaching expiration so that I can apply correct reimbursement logic and flag claims for manual review before incorrect in-network rates are applied.

## Acceptance Criteria

Given a Claim_Line__c record is open in the adjudication screen,
When the adjudication screen LWC loads,
Then the component must check for any Network_Termination_Case__c records where NTC.Provider__c = Claim_Line__c.BillingProvider__c and NTC.Status__c IN ('Open','In Review','Pending Effective Date').

Given an open Network_Termination_Case__c exists for the billing provider,
When the alert evaluates,
Then a red inline alert banner must display: 'WARNING: Provider [NPI] has an active network termination case (Case: [NTC.CaseNumber__c]). Verify reimbursement rate before adjudicating.' and Claim_Line__c.AdjudicationHold__c must be set to true.

Given no termination case exists but the provider's PNA.TerminationDate__c is within 60 days of today,
When the screen loads,
Then a yellow advisory banner must display: 'NOTICE: Provider network agreement expires [PNA.TerminationDate__c]. Confirm in-network status with Provider Network team.'

Given the Agentforce Claims AI assistant is enabled,
When a termination alert fires,
Then the Agentforce panel must auto-populate a suggested action: 'Review NTC Case [CaseNumber], confirm out-of-network rate applicability, and update Claim_Line__c.NetworkStatus__c accordingly.'

Given the adjudicator acknowledges and overrides the hold,
When Claim_Line__c.AdjudicationHold__c is manually set to false,
Then an override reason must be required via a modal (Claim_Line__c.HoldOverrideReason__c, mandatory picklist) and an Audit_Log__c entry must be created recording the user, timestamp, and reason.

## Solution

Extend the existing Claim_Line__c adjudication LWC 'claimLineAdjudicationPanel' with a new child component 'providerTerminationAlert'. Use @wire to call Apex controller 'ProviderNetworkAlertCtrl.cls' (with sharing) querying Network_Termination_Case__c and Provider_Network_Agreement__c. Conditional rendering drives banner color and message. Agentforce integration via Einstein Copilot Action 'SuggestTerminationReviewAction' triggered by a custom topic. Hold override modal implemented as a child LWC with a required picklist. Audit_Log__c written via a Record-Triggered Flow on Claim_Line__c.AdjudicationHold__c field change.

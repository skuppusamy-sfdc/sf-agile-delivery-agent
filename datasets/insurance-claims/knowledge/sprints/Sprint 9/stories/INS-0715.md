## Description

As a Claims Operations Manager, I need a unified Policyholder 360 FlexCard on the Policy__c record page that surfaces active coverage details, open and closed claims, and a chronological interaction timeline so that my team can resolve inquiries without navigating across multiple objects and tabs.

## Acceptance Criteria

Given a Policy__c record with Status__c = 'Active', when a Claims Operations Manager opens the record page, then the Policyholder 360 FlexCard must display:
  - Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, and Policy__c.Expiration_Date__c in the header band
  - A coverage summary section pulling from Coverage__c child records where Coverage__c.Status__c = 'Active'
  - A claims timeline listing all related Claim__c records ordered by Claim__c.FNOL_Date__c descending, showing Claim__c.Claim_Number__c, Claim__c.Status__c, and Claim__c.Total_Billed_Amount__c

Given a Policy__c record with two or more active Coverage__c lines, when the FlexCard renders, then each coverage row must show Coverage__c.Coverage_Type__c, Coverage__c.Deductible_Amount__c, Coverage__c.Deductible_Met__c, and Coverage__c.Out_of_Pocket_Max__c

Given a Claim__c with Status__c = 'Adjudicated', when it appears in the timeline, then a green 'Adjudicated' badge must render and the Claim__c.EOB_Issued_Date__c must be visible inline

Given a Policy__c with no related Claim__c records, when the claims timeline section loads, then an empty-state message 'No claims on record for this policy' must display without throwing a console error

-Given a toggle to switch between Member view and TPA view on the FlexCard header (deprecated — TPA view moved to INS-0719)-

Given the FlexCard is viewed by a user without the 'Claims_Ops_Read' Permission Set Group, when the record page loads, then the FlexCard must render a restricted-access placeholder and log a Platform Event to the Compliance_Audit__e stream

## Solution

Build an OmniStudio FlexCard 'Policyholder360Card' with a primary DataRaptor Extract sourcing Policy__c, child Coverage__c via SOQL relationship query, and Claim__c via a secondary DataRaptor. Use Flyout child cards for claim detail drill-down. Conditional rendering states handle empty claims list. A separate DataRaptor Transform normalises currency fields. Permission check implemented via a Custom Permission 'View_Policyholder_360' evaluated in the FlexCard Conditional Visibility expression. Platform Event Compliance_Audit__e published via an Apex invocable called from the FlexCard action on access denial. Deploy via OmniStudio Migration Tool as part of the Phase 2 package.

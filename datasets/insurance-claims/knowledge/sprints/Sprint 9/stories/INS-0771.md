## Description

As a Claims Operations Manager, I need a consolidated Policyholder 360 FlexCard on the Policy__c record page so that I can immediately view a member's active coverage, recent claims activity, COB details, and payment history without navigating across multiple objects.

This story extends INS-0741 (base policyholder layout) by adding the coverage history panel and real-time claim status tiles.

Background: Currently, adjusters must open Policy__c, then navigate to related Claim__c list views, then cross-reference Payment__c records manually. This causes handle time delays of 4–7 minutes per inquiry. The FlexCard must aggregate data from Policy__c, Claim__c, Explanation_of_Benefits__c, and Payment__c in a single OmniStudio-driven component.

## Acceptance Criteria

Given a Claims Operations Manager opens a Policy__c record with Status__c = 'Active',
When the Policyholder 360 FlexCard loads,
Then the card must display Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Termination_Date__c, and Policy__c.Group_Plan_Name__c in the header band.

Given the policyholder has one or more Claim__c records linked via Policy__c lookup,
When the FlexCard renders the Claims Summary tile,
Then it must show the 5 most recent Claim__c records sorted by Claim__c.Date_of_Service__c descending, displaying Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, and Claim__c.Total_Paid_Amount__c.

Given a Claim__c.Status__c = 'Adjudicated' exists on the policy,
When the EOB tile is rendered,
Then a hyperlink to the related Explanation_of_Benefits__c.EOB_Document_URL__c must be present and open in a new browser tab.

Given COB is applicable (Policy__c.COB_Indicator__c = true),
When the coverage panel renders,
Then a COB badge must appear and display Policy__c.Primary_Carrier__c and Policy__c.Secondary_Carrier__c fields.

-Given a Payment__c.Status__c = 'Pending' exists, display a red alert banner on the FlexCard header.- (Deprecated: replaced by the Payments sub-tab in INS-0774)

Given the user does not have the 'Claims_Adjuster' or 'Claims_Manager' Permission Set Group assigned,
When they navigate to the Policy__c record page,
Then the Policyholder 360 FlexCard must not render and the page must display a 'Access Restricted' message.

Given the FlexCard is loaded on a record with more than 200 related Claim__c records,
When the Claims Summary tile loads,
Then response time must not exceed 2 seconds as measured by DataRaptor Extract execution logs.

## Solution

1. Build a DataRaptor Extract (DR_Extract_Policyholder360) that queries Policy__c, Claim__c (5 most recent via SOQL ORDER BY Date_of_Service__c DESC LIMIT 5), Explanation_of_Benefits__c, and Payment__c in a single multi-object extract using junction mappings.
2. Create an Integration Procedure (IP_Policyholder360_Aggregator) that orchestrates the DataRaptor Extract, applies COB_Indicator__c conditional logic, and returns a normalized JSON response.
3. Build the FlexCard (FC_Policyholder360) in OmniStudio FlexCard Designer with four card states: Active, Inactive, COB-Active, and Error. Use conditional visibility on the COB badge using flyout actions.
4. Embed the FlexCard on the Policy__c Lightning Record Page via App Builder, constrained to a Permission Set Group (PSG_Claims_Staff) using a component visibility filter.
5. Cache the Integration Procedure response using Platform Cache (Session partition, TTL 300s) to meet the 2-second SLA.
6. Unit test coverage: minimum 85% on any Apex helper class used for fallback SOQL if DataRaptor limits are hit.

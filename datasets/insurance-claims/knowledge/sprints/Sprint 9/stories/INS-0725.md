## Description

As a Claims Operations Manager, I need EOB documents linked directly on the Policyholder 360 view by resolving the associated DocuSign envelope IDs stored on Explanation_of_Benefits__c records so that policyholders and internal users can access finalized EOB PDFs without leaving Salesforce. Extends INS-0723.

## Acceptance Criteria

Given an Explanation_of_Benefits__c record with EOB_DocuSign_Envelope_ID__c populated and EOB_Status__c = 'Completed', when the EOB Documents section of the Policyholder 360 view loads, then a hyperlink to the DocuSign envelope is rendered using the stored envelope ID.

Given an EOB_Status__c = 'Voided' or 'In Progress', when the EOB Documents section loads, then those records are excluded from the display.

Given a successful DocuSign envelope lookup, when the user clicks the EOB link, then the document opens in a new browser tab via the DocuSign Retrieve Document REST API endpoint.

Given a DocuSign API call that returns a non-200 HTTP response, when the lookup fails, then the UI displays 'Document temporarily unavailable' and a Platform Event 'DocuSign_Error__e' is published with the Explanation_of_Benefits__c.Id and HTTP status code.

-Given a policyholder with more than 10 EOBs, when the section loads, then all EOBs are listed in a flat table.- (Deprecated — replaced with paginated list showing 5 per page per UX review)

## Solution

Create Apex class 'EOBDocuSignResolverService.cls' with a @AuraEnabled method that queries Explanation_of_Benefits__c filtered by Policy__c lookup and EOB_Status__c = 'Completed', then constructs DocuSign document retrieval URLs using the stored EOB_DocuSign_Envelope_ID__c. HTTP callout handled via Named Credential 'DocuSign_Org'. Error handling publishes Platform Event 'DocuSign_Error__e' via EventBus.publish(). LWC 'eobDocumentList' consumes the Apex method and renders paginated results. Platform Event subscriber triggers a Record-Triggered Flow to log errors to a custom Integration_Error_Log__c object.

## Description

As a Broker Relations Coordinator, I need brokers to be able to search for policyholder coverage records in the Agent Portal using either a Group Number or a TIN (Tax Identification Number) so that brokers can self-serve eligibility lookups for their employer group clients without contacting the Acme Insurance service center, reducing inbound call volume by an estimated 15%.

## Acceptance Criteria

Given a broker is authenticated in the Agent Portal (Experience Cloud site), when they navigate to the 'Coverage Lookup' page, then a search form must be presented with two mutually exclusive input options: 'Search by Group Number' and 'Search by TIN', controlled by a radio button toggle.

Given the broker selects 'Search by Group Number' and enters a value, when they submit the form, then the system must query Policy__c records where Policy__c.GroupNumber__c matches the entered value AND Policy__c.BrokerOfRecord__c = the authenticated broker's linked Account Id, returning only policies the broker is authorized to view.

Given the broker selects 'Search by TIN' and enters a 9-digit numeric value, when they submit the form, then the system must query Policy__c records where Policy__c.EmployerTIN__c matches the entered value AND Policy__c.BrokerOfRecord__c = the authenticated broker's linked Account Id.

Given a search returns results, when the results table renders, then it must display Policy__c.PolicyNumber__c, Policy__c.PolicyHolderName__c, Policy__c.LOB__c, Policy__c.EffectiveDate__c, Policy__c.Status__c, and Policy__c.GroupNumber__c columns. Policy__c.EmployerTIN__c must NOT be displayed in the results table for security compliance.

Given a search returns zero results, when the results area renders, then the message 'No coverage records found matching your search criteria. Please verify the Group Number or TIN and try again.' must be displayed.

Given a broker enters a TIN with fewer than 9 digits or non-numeric characters, when they attempt to submit the form, then client-side validation must block submission and display 'TIN must be exactly 9 numeric digits.' inline beneath the TIN input field.

-Given search results are displayed, brokers may download full Policy__c PDF summaries — deprecated pending legal review of broker data access policy-

## Solution

Build an LWC 'agentCoverageLookup' deployed on the Agent Portal (Experience Cloud) page. Implement a Guest/Authenticated User Apex controller 'AgentCoverageLookupController' with a @AuraEnabled method that accepts searchType (String) and searchValue (String) parameters. Apply Sharing Rules and an Apex WITH SECURITY_ENFORCED clause to enforce broker-of-record data isolation — Policy__c.BrokerOfRecord__c must equal the running user's Contact.AccountId. Implement client-side regex validation for TIN format in the LWC JavaScript before invoking the Apex method. Use lightning-datatable to render results. Mask Policy__c.EmployerTIN__c at the Apex layer by excluding it from the SELECT clause in the results query. Apply the 'Broker_Portal_Standard' Permission Set to the Experience Cloud profile. Log all search activity to a custom AuditLog__c object for compliance tracking.

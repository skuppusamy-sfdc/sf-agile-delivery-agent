## Description

**Bug Report — Policyholder 360 FlexCard: Duplicate EOB Records on COB Policies**

**Environment:** Full Sandbox (Sprint 9 UAT)
**Severity:** High — affects all policyholders with COB_Indicator__c = true (~18% of member population)

**Steps to Reproduce:**
1. Navigate to any Policy__c record where Policy__c.COB_Indicator__c = true.
2. Open the Policyholder 360 FlexCard (deployed as part of INS-0699).
3. Click the 'EOB History' tab.
4. Observe the list of Explanation_of_Benefits__c records displayed.

**Expected Behavior:**
Each Explanation_of_Benefits__c record should appear exactly once, displaying EOB__c.EOB_Date__c, EOB__c.Total_Billed__c, EOB__c.Total_Allowed__c, and EOB__c.Member_Responsibility__c.

**Actual Behavior:**
Each EOB record is rendered twice in the list. For policyholders with a secondary payer (EOB__c.Payer_Sequence__c = 'Secondary'), the duplication results in four instances of the same record appearing. This causes the displayed EOB__c.Member_Responsibility__c totals to appear doubled, misleading claims staff.

**Root Cause Hypothesis:**
The DataRaptor Extract 'DR_Get_EOB_History' used by the FlexCard appears to perform a JOIN across both EOB__c.Policy__c and EOB__c.Claim__c.Policy__c relationships simultaneously, returning duplicate rows when both relationships resolve to the same Policy__c record. This is a known OmniStudio DataRaptor behavior when multiple lookup paths exist to the same parent.

## Acceptance Criteria

As per Description. Verify fix by confirming zero duplicate EOB__c records on Policy__c records where COB_Indicator__c = true in the UAT sandbox. Regression test on Policy__c records where COB_Indicator__c = false to confirm no records are lost.

## Solution

Update DataRaptor Extract 'DR_Get_EOB_History' to query EOB__c exclusively via the EOB__c.Policy__c lookup, removing the secondary JOIN path through EOB__c.Claim__c.Policy__c. Add a DISTINCT equivalent filter using DataRaptor's 'Group By' on EOB__c.Id to prevent duplicates as a safety net. Re-deploy the updated DataRaptor and regression test the Policyholder 360 FlexCard across COB and non-COB policy samples in Full Sandbox before promoting to Production.

## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that consolidates credentialing status from Credentialing_Case__c, active Provider_Network_Agreement__c records, network tier, and any open Provider_Change_Notification__c items so that I can manage provider relationships from a single view without toggling between related lists.

## Acceptance Criteria

Given a Provider__c record page is opened,
When the Provider 360 FlexCard loads,
Then the header section must display: Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Specialty__c, Provider__c.Board_Certification__c, and a CAQH status badge sourced from Provider__c.CAQH_Status__c.

Given the provider has one or more Credentialing_Case__c records,
When the Credentialing panel is expanded,
Then all Credentialing_Case__c records where Credentialing_Case__c.Provider__c = current record Id must be shown with fields: Credentialing_Case__c.Status__c, Credentialing_Case__c.Credentialing_Type__c, Credentialing_Case__c.Expiration_Date__c, Credentialing_Case__c.Assigned_Specialist__c.

Given a Credentialing_Case__c.Expiration_Date__c is within 90 days of today,
When the FlexCard renders,
Then an amber 'Expiring Soon' badge must appear next to that credentialing record.

Given the provider has active Provider_Network_Agreement__c records,
When the PNA panel is rendered,
Then PNA__c.Network_Tier__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, and PNA__c.Fee_Schedule__c must be displayed per agreement.

Given an open Provider_Change_Notification__c (PCN) record exists where PCN.Status__c != 'Closed',
When the FlexCard header area renders,
Then a red PCN alert banner must appear listing PCN.Change_Type__c and PCN.Submitted_Date__c.

-Given the provider is associated with a GSA, GSA details must appear in the header (moved to INS-0760).-

## Solution

Build OmniStudio FlexCard 'Provider360Card' with DataRaptors: DR_GetProviderHeader (Provider__c fields + CAQH_Status__c), DR_GetCredentialingCases (Credentialing_Case__c filtered by Provider__c), DR_GetActivePNAs (Provider_Network_Agreement__c filtered by status and provider), DR_GetOpenPCNs (Provider_Change_Notification__c where Status__c != 'Closed'). Expiry logic handled in DataRaptor Transform using date difference formula. PCN banner uses FlexCard conditional state. Permission Set 'Provider_Relations_PSG' controls visibility of TIN__c field. Integration with CAQH API surfaced via a named credential and Apex callout cached in Provider__c.CAQH_Status__c via nightly scheduled job.

## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to automatically suggest a coverage eligibility response during FNOL (First Notice of Loss) intake by analyzing the incoming Claim__c data against the associated Policy__c coverage terms and ICD-10/CPT codes submitted so that adjusters can make faster, more consistent eligibility determinations and reduce manual policy lookups by at least 40%.

## Acceptance Criteria

Given an adjuster opens a new Claim__c record in FNOL status (Claim__c.Status__c = 'FNOL'),
When the Agentforce Claims AI panel loads on the claim record page,
Then the AI must retrieve the linked Policy__c record via Claim__c.Policy__c and evaluate Policy__c.Coverage_Terms__c, Policy__c.Effective_Date__c, Policy__c.LOB__c, and Policy__c.Deductible_Amount__c.

Given the AI has retrieved policy coverage terms,
When the Claim__c has one or more Claim_Line__c records with Claim_Line__c.CPT_Code__c and Claim_Line__c.ICD10_Code__c populated,
Then the AI must return a suggested eligibility determination of 'Covered', 'Partially Covered', or 'Not Covered' for each Claim_Line__c with a confidence score (0-100) and a plain-language rationale.

Given the AI suggests an eligibility determination,
When the adjuster accepts the suggestion,
Then Claim_Line__c.AI_Eligibility_Suggestion__c must be stamped with the suggested value, Claim_Line__c.AI_Confidence_Score__c must be populated, and Claim_Line__c.AI_Reviewed_By__c must record the adjuster's User Id.

Given the AI confidence score is below 60,
When the suggestion is displayed,
Then a 'Low Confidence — Manual Review Required' warning must appear and the Claim__c must be flagged with Claim__c.SIU_Review_Flag__c = true for SIU team awareness.

Given a COB scenario (Policy__c.COB_Flag__c = true),
When the AI evaluates eligibility,
Then it must factor in the secondary payer coordination and adjust the 'Covered' amount suggestion accordingly, noting primary vs. secondary payer responsibility in the rationale.

## Solution

Configure Agentforce Agent with a custom 'Claims Eligibility Advisor' topic and action set. Build Apex invocable action 'ClaimsEligibilityAdvisorAction.cls' to retrieve Policy__c and Claim_Line__c data and construct the prompt context payload for the Einstein LLM. Use Einstein Prompt Builder template 'FNOL_Eligibility_Prompt' referencing Policy__c coverage fields and CPT/ICD-10 mappings maintained in a custom metadata type 'Coverage_Code_Mapping__mdt'. AI response parsed in Apex and written back to Claim_Line__c fields via DML. Low-confidence routing triggers a Record-Triggered Flow to set SIU_Review_Flag__c. COB logic handled in a separate Apex method called conditionally. Add Agentforce panel LWC to Claim__c Lightning Record Page.

# INS-0713: Add Fee Schedule lookup validation to Claim_Line__c adjudication to prevent incorrect payment rates

- **Sprint**: Acme Insurance Sprint 9 (Phase 2)

## Description

As a Claims Operations Manager, I need the Claim_Line__c adjudication process to validate CPT codes against the applicable Fee_Schedule__c at the time of service before calculating allowed amounts so that payments are disbursed at the correct contracted rate and not defaulted to billed charges when a matching fee schedule line is not found.

## Acceptance Criteria

Given a Claim_Line__c record enters adjudication (Claim__c.Status__c transitions to 'In Adjudication'),
When the Fee Schedule lookup fires,
Then the system identifies the applicable Fee_Schedule__c by matching: Fee_Schedule__c.LOB__c = Claim__c.LOB__c AND Fee_Schedule__c.Effective_Date__c <= Claim_Line__c.Service_Date__c AND Fee_Schedule__c.End_Date__c >= Claim_Line__c.Service_Date__c AND Fee_Schedule__c.Provider_Network_Agreement__c = Claim__c.PNA__c.

Given a matching Fee_Schedule__c is found,
When the system locates the CPT code in Fee_Schedule_Line__c (Fee_Schedule_Line__c.CPT_Code__c = Claim_Line__c.CPT_Code__c),
Then Claim_Line__c.Allowed_Amount__c is set to Fee_Schedule_Line__c.Allowed_Amount__c and Claim_Line__c.Fee_Schedule_Applied__c is stamped with the Fee_Schedule__c.Name.

Given no matching Fee_Schedule_Line__c is found for the CPT code,
When the validation runs,
Then Claim_Line__c.Status__c is set to 'Pended – Fee Schedule Miss', Claim__c.Status__c is updated to 'Pended', and a Task is auto-created on the Claim__c record assigned to the Claims Operations queue with Subject = 'Fee Schedule Miss: CPT [code] – Manual Review Required'.

-Given no matching fee schedule, default Claim_Line__c.Allowed_Amount__c to Claim_Line__c.Billed_Amount__c- (removed per Underwriting Analyst review — defaulting to billed charges is non-compliant with CMS guidelines)

Given the ICD-10 diagnosis code on Claim__c.Primary_Diagnosis_Code__c is required for certain CPT codes (as defined in Fee_Schedule_Line__c.Requires_Diagnosis__c = true),
When adjudication runs,
Then if Claim__c.Primary_Diagnosis_Code__c is blank, Claim_Line__c.Status__c is set to 'Pended – Missing Diagnosis' and a validation error message is logged in Claim__c.Adjudication_Notes__c.

Given a Claim_Line__c is successfully adjudicated with a fee schedule match,
When processing completes,
Then Payment__c child record is created with Payment__c.Amount__c = Claim_Line__c.Allowed_Amount__c, Payment__c.Status__c = 'Pending Disbursement', and Payment__c.Claim_Line__c = the processed record Id.

## Solution

Implement an Apex Trigger 'ClaimLineAdjudicationTrigger' on Claim_Line__c (before update, after update) delegating to a handler class 'ClaimLineAdjudicationHandler'. Fee Schedule lookup logic encapsulated in a reusable service class 'FeeScheduleLookupService' with method 'resolveAllowedAmount(ClaimLine, PNAId, ServiceDate, LOB)' to allow future invocation from OmniScript flows. Use a Map-based bulk pattern to resolve all CPT codes in a single SOQL query per trigger batch. Payment__c record creation handled in an after-update context using a Queueable Apex job 'PaymentCreationQueueable' to avoid mixed DML errors. All pend logic updates Claim__c via a separate after-update handler to prevent recursive trigger firing (controlled via a static Boolean flag in TriggerContextUtil). Comprehensive test class 'FeeScheduleLookupServiceTest' with 90%+ coverage including bulk (200 record) scenarios.

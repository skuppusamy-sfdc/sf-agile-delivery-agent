## Description

As a Claims Operations Manager, I need a dedicated Coordination of Benefits (COB) panel on the Policyholder 360 view that displays secondary payer information and COB priority order so that claims adjudicators can correctly sequence payments across primary and secondary insurers and reduce overpayment risk. This story implements the COB banner deferred from INS-0747.

## Acceptance Criteria

Given a Policy__c record where Policy__c.COB_Indicator__c = true,
When the Policyholder 360 page loads,
Then a 'Coordination of Benefits' panel must be visible displaying Policy__c.Primary_Payer__c, Policy__c.Secondary_Payer__c, Policy__c.COB_Priority__c, and Policy__c.Other_Insurance_Effective_Date__c.

Given the COB panel is visible,
When a user clicks 'Edit COB Details',
Then an inline edit modal must allow updates to Policy__c.Secondary_Payer__c and Policy__c.COB_Priority__c fields, with changes saved via a Record-Triggered Flow that stamps Policy__c.COB_Last_Updated__c = NOW() and Policy__c.COB_Updated_By__c = current user.

Given Policy__c.COB_Indicator__c = false or null,
When the Policyholder 360 page loads,
Then the COB panel must be hidden and must not render any empty field containers.

Given a Claim__c is being adjudicated where the parent Policy__c.COB_Indicator__c = true,
When the claim is opened by a Claims Operations Manager,
Then a warning banner 'COB Policy — Verify Secondary Payer Before Adjudication' must appear at the top of the Claim__c record page.

Given Policy__c.COB_Priority__c = 'Secondary',
When the EOB is generated,
Then EOB__c.COB_Sequence__c must be set to '2' and EOB__c.Primary_Payer_Paid__c must be a required field before the EOB can be finalized.

## Solution

Extend the OmniStudio FlexCard 'Policyholder360Card' (built in INS-0747) with a new child card state 'COBPanel' conditionally rendered when Policy__c.COB_Indicator__c = true. Use a DataRaptor Extract to pull COB-related fields from Policy__c. Build an OmniScript 'EditCOBDetails' launched from a FlexCard action button, using a DataRaptor Load to update Policy__c fields and stamp audit fields via formula or Flow. For the Claim__c warning banner, implement a Record-Triggered Flow on Claim__c (before-save) that checks the related Policy__c.COB_Indicator__c via a Get Records element and sets a Claim__c.COB_Warning_Active__c checkbox, which drives a dynamic Lightning page component visibility rule. Add a validation rule on EOB__c to enforce EOB__c.Primary_Payer_Paid__c when COB_Sequence__c = '2'.

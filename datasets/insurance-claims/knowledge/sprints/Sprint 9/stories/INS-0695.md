## Description

As a Broker Relations Coordinator, I need an Experience Cloud agent portal dashboard that displays a book-of-business claims summary widget showing claim volume, total billed amounts, and average adjudication cycle time by LOB so that brokers can self-serve reporting on their clients' claims activity without requiring ad-hoc requests to the Claims Operations team.

## Acceptance Criteria

Given a broker logs into the Experience Cloud Agent Portal,
When the dashboard page loads,
Then the claims summary widget must display only Claim__c records where Claim__c.BrokerAccount__c = the logged-in user's associated broker Account Id (enforced via Sharing Rules).

Given the widget loads claim data,
When it renders the summary metrics,
Then it must display: total open claim count, total Claim__c.TotalBilledAmount__c sum, total Claim__c.TotalPaidAmount__c sum, and average adjudication cycle time in days (Claim__c.AdjudicatedDate__c minus Claim__c.ReceivedDate__c) grouped by Claim__c.LOB__c.

Given the broker selects a date range filter,
When the filter is applied,
Then the widget must re-query using Claim__c.DateOfLoss__c between the selected start and end dates.

Given a broker has no associated claims in the selected period,
When the widget renders,
Then it must display 'No claims activity for the selected period' with a prompt to contact their assigned Broker Relations Coordinator.

Given the portal page loads,
When the session context is evaluated,
Then brokers must not see any Claim__c, Policy__c, or Policyholder data outside their assigned book of business — validated via Salesforce Sharing Rules and a custom Apex with-sharing class.

## Solution

Build the dashboard as a Lightning Experience Cloud page using a custom LWC 'brokerClaimsSummaryWidget'. Use an Apex controller 'BrokerClaimsSummaryCtrl.cls' declared 'with sharing' to enforce record-level security. Aggregate metrics computed via SOQL aggregate functions (SUM, AVG, COUNT). Date range filter implemented as LWC input date properties passed to Apex. Deploy on Experience Cloud site 'AcmeBrokerPortal' Lightning page. Configure Sharing Rules on Claim__c based on Claim__c.BrokerAccount__c lookup to enforce book-of-business isolation. Apply Guest/Authenticated User Permission Set Group 'BrokerPortal_Standard'.

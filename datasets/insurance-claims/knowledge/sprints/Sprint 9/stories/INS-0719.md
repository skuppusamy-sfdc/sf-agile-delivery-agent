## Description

As a TPA (Third Party Administrator) user accessing the Agent Portal Experience Cloud site, I need a dedicated claims dashboard that shows real-time claim status for all policies under my administered groups and allows me to download Explanation_of_Benefits__c (EOB) documents so that I can service employer group clients without requiring direct access to the core Salesforce org.

## Acceptance Criteria

Given a TPA user logs into the Agent Portal Experience Cloud site with the 'TPA_Portal_User' profile, when the Claims Dashboard page loads, then the user must see only Claim__c records where Claim__c.Administered_By_TPA__c = the logged-in user's associated Account, enforced via Sharing Rules and an Apex-managed sharing recalculation

Given the TPA claims dashboard is loaded, when displayed, then each row must show Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.Date_of_Service__c, Claim__c.Total_Billed_Amount__c, Claim__c.Total_Paid_Amount__c, and a 'Download EOB' action column

Given a Claim__c with Status__c = 'Adjudicated' and a related Explanation_of_Benefits__c record, when the TPA user clicks 'Download EOB', then the system must retrieve the Explanation_of_Benefits__c.EOB_Document_URL__c and initiate a secure file download, logging a ContentDocument access event to the Compliance_Audit__e Platform Event stream

Given a Claim__c with Status__c NOT IN ('Adjudicated', 'Closed'), when the TPA user views the row, then the 'Download EOB' button must be disabled and display tooltip text 'EOB not yet available'

Given the TPA user applies a date range filter on the dashboard, when the filter is submitted, then the dashboard must re-render showing only Claim__c records where Claim__c.Date_of_Service__c falls within the selected range, with results returning in under 3 seconds for datasets up to 500 records

Given a TPA user attempts to access a Claim__c URL directly that is not within their administered group, when the page loads, then Salesforce Guest/Authenticated sharing must block access and redirect to a '403 — Access Denied' custom error page

## Solution

Build Experience Cloud site page 'TPA Claims Dashboard' using an LWC 'tpaClaimsDashboard' with server-side Apex controller 'TPAClaimsDashboardController' enforcing WITH SECURITY_ENFORCED SOQL. Implement Apex Managed Sharing on Claim__c via 'ClaimShareService' class recalculated by a Record-Triggered Flow on Claim__c.Administered_By_TPA__c changes. EOB download action calls an Apex method retrieving ContentDocumentLink and generating a short-lived download URL. Platform Event Compliance_Audit__e published via Apex. Date filter implemented as @wire reactive property. Custom 403 page built as an Experience Cloud custom error page component. Permission Set Group 'TPA_Portal_Access' controls field-level security.

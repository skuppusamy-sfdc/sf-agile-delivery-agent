# INS-0745: Add ERA reconciliation status indicator to Payment__c record page for claims payment visibility

- **Sprint**: Acme Insurance Sprint 9 (Phase 2)
- **Status**: Open
- **Type**: Story
- **Priority**: Medium
- **Resolution**: Unresolved
- **Assignee**: Amanda Foster
- **Reporter**: David Kim
- **Components**: PaymentEngine, ClaimsProcessing, DataIntegration
- **Epic**: Claims Payment & Disbursement
- **Story Points**: 3
- **Labels**: Phase2, ERA, PaymentReconciliation

## Description

As a Claims Operations Manager, I need an Electronic Remittance Advice (ERA) reconciliation status indicator on the Payment__c record page so that I can immediately identify whether an outbound payment has been matched to an inbound ERA file from the payer/TPA without navigating to external reconciliation reports.

## Acceptance Criteria

Given a Payment__c record exists with Payment__c.Payment_Method__c = 'EFT' or 'Check', when the ERA Reconciliation Status component loads on the Payment__c record page, then it must display Payment__c.ERA_Status__c with one of the following values: 'Awaiting ERA', 'ERA Matched', 'ERA Mismatch', or 'Manual Review Required'.

Given an inbound ERA file is processed by the DataIntegration layer and a matching Payment__c record is found via Payment__c.Check_Number__c or Payment__c.EFT_Trace_Number__c, when the match is confirmed, then Payment__c.ERA_Status__c must be automatically updated to 'ERA Matched' and Payment__c.ERA_Received_Date__c must be stamped with the current DateTime.

Given Payment__c.ERA_Status__c = 'ERA Mismatch', when the record page loads, then a highlighted alert banner must display the mismatch detail stored in Payment__c.ERA_Mismatch_Reason__c and surface a 'Escalate to SIU' quick action button.

-Given the ERA file arrives via SFTP batch, display raw ERA 835 segment data in a related list- (descoped to Phase 3 ERA raw data story)

Given a Claims Operations Manager clicks 'Escalate to SIU', when the action completes, then a Case record must be created with Case.RecordType = 'SIU Investigation', Case.Related_Payment__c populated, and Case.Status = 'New', and the Payment__c.ERA_Status__c must update to 'Manual Review Required'.

## Solution

Add a custom field ERA_Status__c (Picklist) and ERA_Received_Date__c (DateTime) to the Payment__c object. Build a Record-Triggered Flow 'ERAStatusUpdate_RTF' on Payment__c to handle status transitions. Create an Apex REST endpoint '/services/apexrest/ERAReconciliation' to receive ERA match results from the middleware integration layer (MuleSoft), which will POST match/mismatch payloads referencing Payment__c.Check_Number__c or EFT_Trace_Number__c. The endpoint updates Payment__c via DML. Build the 'Escalate to SIU' functionality as a Screen Flow invoked via a Quick Action on Payment__c that creates the Case record with appropriate RecordType. Add the ERA Status indicator as an LWC 'eraStatusBadge' placed in the Payment__c Lightning Record Page highlights panel region.

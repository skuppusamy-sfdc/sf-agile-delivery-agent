## Description

As a Provider Network Specialist, I need automated notifications when a Provider_Network_Agreement__c (PNA) is approaching its expiration date so that I can initiate renewal workflows proactively and avoid coverage gaps that could impact claims adjudication for in-network providers.

Currently, PNA expiry tracking is manual and relies on a weekly spreadsheet review by the network team. This has resulted in 3 lapsed agreements in Q3 causing claims payment delays. The solution must trigger notifications at 90-day, 60-day, and 30-day intervals before PNA__c.Contract_End_Date__c and escalate to the Provider Relations Specialist's manager at 15 days.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record has PNA__c.Status__c = 'Active' and PNA__c.Contract_End_Date__c is exactly 90 days from today,
When the scheduled Apex job runs at 06:00 UTC daily,
Then a Platform Event (PNA_Expiry_Alert__e) must be published with fields: PNA_Id__c, Provider_Name__c, Contract_End_Date__c, Days_Until_Expiry__c = 90, and Alert_Tier__c = '90-Day'.

Given a PNA_Expiry_Alert__e Platform Event is published,
When the triggered Flow (Flow_PNA_Expiry_Notification) processes the event,
Then a Task must be created on the Provider_Network_Agreement__c record with Subject = 'PNA Renewal Required – 90 Day Notice', Status = 'Open', Priority = 'Normal', and assigned to PNA__c.Owner.

Given PNA__c.Contract_End_Date__c is 30 days from today,
When the Platform Event is published with Alert_Tier__c = '30-Day',
Then the Task Priority must be set to 'High' and a Chatter post must be added to the PNA record tagging the assigned Provider Relations Specialist.

Given PNA__c.Contract_End_Date__c is 15 days from today,
When the escalation logic executes,
Then an email notification must be sent to the Provider Relations Specialist's manager (derived from User.ManagerId) with PNA__c.Provider_Name__c, PNA__c.Contract_End_Date__c, and PNA__c.Estimated_Annual_Value__c included in the email body.

Given a PNA__c.Status__c is updated to 'Renewed' or 'Terminated' before the expiry date,
When the daily scheduled job evaluates the record,
Then no further PNA_Expiry_Alert__e events must be published for that record.

Given the system processes more than 500 PNA records approaching expiry on the same day,
When the scheduled Apex job executes,
Then it must use batch Apex (BatchPNAExpiryCheck) with batch size = 50 to avoid governor limit violations.

## Solution

1. Create a Scheduled Apex class (BatchPNAExpiryCheck implements Database.Batchable, Schedulable) that queries PNA__c records where Status__c = 'Active' AND Contract_End_Date__c IN (TODAY+90, TODAY+60, TODAY+30, TODAY+15) using dynamic date literals.
2. In the execute() method, publish PNA_Expiry_Alert__e Platform Events using EventBus.publish() in bulk-safe batches.
3. Define the PNA_Expiry_Alert__e Platform Event object with fields: PNA_Id__c (Text), Provider_Name__c (Text), Contract_End_Date__c (Date), Days_Until_Expiry__c (Number), Alert_Tier__c (Text).
4. Build a Record-Triggered Flow (Flow_PNA_Expiry_Notification) subscribed to PNA_Expiry_Alert__e that creates Tasks and Chatter posts using conditional branches on Alert_Tier__c values.
5. For 15-day escalation, use an Apex-invocable method (PNAEscalationEmailHandler) called from the Flow to resolve User.ManagerId and send email via Messaging.SingleEmailMessage.
6. Schedule the batch job via Anonymous Apex: System.schedule('PNA Expiry Daily Check', '0 0 6 * * ?', new BatchPNAExpiryCheck());
7. Add a custom field PNA__c.Last_Expiry_Alert_Sent__c (Date) to prevent duplicate event publishing on the same day.
8. Deploy with 90%+ Apex test coverage including governor limit stress tests at 500+ record volume.

## Description

As a Provider Relations Specialist, I need automated alerts when Provider_Network_Agreement__c records are approaching expiration (90, 60, and 30 days out) so that I can proactively initiate renewal negotiations and avoid provider network gaps that could disrupt member access to care. Extends INS-0733 (PNA Lifecycle Management).

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA__c.Status__c = 'Active' and PNA__c.Expiration_Date__c is within 90 calendar days of today,
When the nightly scheduled flow runs,
Then a Task must be created with Subject = 'PNA Expiring in 90 Days - Review Required', assigned to the PNA__c.Owner, with ActivityDate = today + 5 business days, and linked to the PNA record.

Given the same PNA__c record reaches the 60-day threshold,
When the scheduled flow runs,
Then a second Task must be created with Subject = 'PNA Expiring in 60 Days - Renewal Negotiation Due' and Priority = 'High', and an in-app Bell Notification must be sent to the PNA__c.Owner.

Given the PNA__c record reaches the 30-day threshold,
When the scheduled flow runs,
Then the PNA__c.Alert_Level__c picklist field must be updated to 'Critical', a Task with Priority = 'High' and Subject = 'PNA Expiring in 30 Days - URGENT: Execute Renewal or Termination' must be created, and an email alert must be sent to both PNA__c.Owner and PNA__c.Network_Director__c.

Given a PNA__c record has PNA__c.Status__c = 'Pending Renewal' or 'Terminated',
When the scheduled flow evaluates it,
Then no new Tasks or alerts must be generated for that record.

Given a Task has already been created for a specific threshold (tracked via PNA__c.Alert_90_Sent__c, PNA__c.Alert_60_Sent__c, PNA__c.Alert_30_Sent__c boolean fields),
When the nightly flow runs again,
Then duplicate Tasks must not be created.

## Solution

Implement a Scheduled Flow ('PNA_Expiration_Alert_Flow') running nightly at 06:00 UTC. Flow uses Get Records on Provider_Network_Agreement__c filtered by Status__c = 'Active' and Expiration_Date__c <= TODAY+90. Decision elements branch by threshold tier. Task creation via Create Records flow element. Email alerts via Send Email flow element using a custom Email Template ('PNA_Expiration_Alert'). Bell notifications via Custom Notification Type deployed via Notification Builder. Checkbox fields (Alert_90_Sent__c etc.) updated via Update Records to prevent duplication. Flow tested with 500+ PNA records in sandbox to validate governor limit compliance.

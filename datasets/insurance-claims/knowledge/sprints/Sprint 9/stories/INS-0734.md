## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record that consolidates credentialing status from Credentialing_Case__c, active Provider_Network_Agreement__c (PNA) details, and Fee_Schedule__c tier assignment so that I can conduct provider reviews and respond to provider inquiries without switching between multiple record pages. This story extends INS-0718 which delivered the base Provider Account layout.

## Acceptance Criteria

Given a Person Account with RecordType.DeveloperName = 'Provider', when a Provider Relations Specialist opens the record, then the Provider 360 FlexCard must render and display the most recent Credentialing_Case__c record where Credentialing_Case__c.Provider__c = current Account Id, showing Credentialing_Case__c.Status__c, Credentialing_Case__c.ExpirationDate__c, and Credentialing_Case__c.CaqhProfileId__c.

Given the FlexCard is rendered, when the provider has an active Credentialing_Case__c with Status__c = 'Expired' or ExpirationDate__c < TODAY(), then a red 'Credentials Expired' badge must appear prominently in the card header.

Given the FlexCard is rendered, when the 'Network Agreements' section is expanded, then all Provider_Network_Agreement__c records where PNA.Provider__c = current Account Id and PNA.Status__c = 'Active' must be listed showing PNA.NetworkName__c, PNA.EffectiveDate__c, PNA.ExpirationDate__c, and PNA.ContractType__c.

Given the FlexCard is rendered, when the 'Fee Schedule' section is expanded, then the linked Fee_Schedule__c record must display Fee_Schedule__c.ScheduleName__c, Fee_Schedule__c.Tier__c, Fee_Schedule__c.EffectiveDate__c, and Fee_Schedule__c.LOB__c.

Given a provider has no active PNA records, when the Network Agreements section renders, then the message 'No active network agreements on file. Initiate contracting via Provider Network Contracting workflow.' must be displayed with a button that navigates to the PNA new record creation OmniScript.

Given the FlexCard is displayed, when the provider's NPI on Provider__c.NPI__c is clicked, then an external link must open the NPPES NPI Registry lookup URL in a new browser tab using the NPI value as the query parameter.

## Solution

Build OmniStudio FlexCard 'Provider360Card' with three child card panels: CredentialingStatusCard, NetworkAgreementsCard, and FeeScheduleCard. Use DataRaptor Extract actions for each panel with targeted SOQL queries on Credentialing_Case__c, Provider_Network_Agreement__c, and Fee_Schedule__c. Implement OmniStudio Conditional Views to drive the 'Credentials Expired' badge using a date comparison formula against TODAY(). For the NPPES external link, use a FlexCard Action of type 'Navigate' with a URL template referencing the NPI__c field token. For the PNA new record button, invoke an OmniScript 'PNA_Initiation_OmniScript' via a FlexCard Action of type 'OmniScript'. Apply the 'Provider_Relations_Standard' Permission Set Group to restrict access. Deploy the FlexCard on the Provider Person Account Lightning Record Page via the OmniStudio FlexCard App Builder component.

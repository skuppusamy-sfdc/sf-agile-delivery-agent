## Description

As a Broker Relations Coordinator, I need an Agent Portal dashboard in Salesforce Experience Cloud that shows agents their full book-of-business with active policy counts, open claim counts by status, and pending renewal summaries so that agents can self-serve on client status without calling the Claims Operations team, reducing inbound support volume by an estimated 30%.

## Acceptance Criteria

Given a licensed agent logs into the Salesforce Experience Cloud Agent Portal,
When the dashboard page loads,
Then the 'My Book of Business' widget must display the count of Policy__c records where Policy__c.Agent__c = current user's Contact Id and Policy__c.Status__c = 'Active'.

Given the Claims Summary widget renders,
When the agent views the dashboard,
Then it must display counts of Claim__c records grouped by Claim__c.Status__c (Open, Pending Review, Adjudicated, Closed) for all policies in the agent's book of business, rendered as a donut chart using a Lightning Web Component.

Given the Pending Renewals widget renders,
When the agent views the dashboard,
Then it must list all Policy__c records where Policy__c.Agent__c = current user's Contact Id, Policy__c.Status__c = 'Active', and Policy__c.Renewal_Date__c <= TODAY + 60, sorted by Policy__c.Renewal_Date__c ascending.

Given an agent clicks on a policy in the Pending Renewals widget,
When the navigation event fires,
Then the agent must be routed to the Policy__c detail page within the Experience Cloud site (not the internal Salesforce org).

Given an agent has zero policies in their book of business,
When the dashboard loads,
Then each widget must display an empty-state illustration with the message 'No records found. Contact your Broker Relations Coordinator for assistance.' instead of an empty table or blank space.

Given the dashboard is accessed by an Experience Cloud Guest User profile,
When authentication is attempted,
Then access must be denied and the user redirected to the login page — dashboard data must never be exposed without authentication.

## Solution

Build the Agent Portal dashboard as a Lightning Experience Cloud page in a Customer Community Plus license site. Develop three LWC widgets: 'agentBookOfBusiness', 'agentClaimsSummary', 'agentPendingRenewals'. Each widget uses @wire with a custom Apex controller (AgentPortalDashboardController.cls) annotated with @AuraEnabled(cacheable=true). Enforce data isolation using Sharing Rules on Policy__c and Claim__c (Agent__c = $User.ContactId) and a Permission Set Group ('Agent_Portal_User') with object-level read access. Donut chart implemented using Chart.js loaded as a Static Resource. Navigation handled via NavigationMixin pointing to the Experience Cloud named page. All Apex methods enforce WITH SECURITY_ENFORCED in SOQL.

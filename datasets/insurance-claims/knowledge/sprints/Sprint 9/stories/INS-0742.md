## Description

As a Compliance Officer, I need an automated audit trail that captures every endorsement and mid-term change to a Policy__c record so that we can produce a complete coverage change history for regulatory audits and dispute resolution without relying on manual field history tracking.

## Acceptance Criteria

Given a Policy__c record is updated and any of the following fields change — Policy__c.Coverage_Limit__c, Policy__c.Deductible__c, Policy__c.Premium_Amount__c, Policy__c.LOB__c, or Policy__c.Status__c — when the Record-Triggered Flow fires on update, then a new Policy_Change_Log__c record must be created capturing: Policy_Change_Log__c.Policy__c (lookup), Policy_Change_Log__c.Changed_Field__c (text), Policy_Change_Log__c.Old_Value__c (text), Policy_Change_Log__c.New_Value__c (text), Policy_Change_Log__c.Changed_By__c (User lookup), and Policy_Change_Log__c.Change_Timestamp__c (DateTime = NOW()).

Given a Policy_Change_Log__c record is created, when a Compliance Officer navigates to the Policy__c related list 'Coverage Change History', then all log records must appear sorted by Policy_Change_Log__c.Change_Timestamp__c descending.

Given the audit trail records are older than 7 years, when the nightly archival batch runs, then those Policy_Change_Log__c records must be soft-deleted by setting Policy_Change_Log__c.Archived__c = true rather than hard-deleted, in compliance with the 7-year retention policy.

Given a Compliance Officer exports the coverage history, when they use the 'Export Audit Trail' button, then a Conga CLM-generated PDF must be produced containing all Policy_Change_Log__c entries for the selected Policy__c, formatted with the Acme Insurance audit report template.

## Solution

Create a Record-Triggered Flow 'PolicyChangeAudit_RTF' on the Policy__c object, triggering on Update, running in 'Fast Field Updates' context for detection and then invoking a subflow 'CreatePolicyChangeLog_SubFlow' in an After-Save context to insert Policy_Change_Log__c records. Use $Record and $Record__Prior global variables to capture old and new values for each monitored field. Loop through a hard-coded list of field API names within the flow using a collection loop and Decision element. Build the 7-year archival as a Scheduled Apex Batch class 'PolicyChangeLogArchivalBatch' running nightly via a Scheduled Job. Integrate Conga CLM template via a custom Quick Action on Policy__c that passes Policy__c.Id as a merge parameter to the Conga template 'AuditTrailReport_Template'.

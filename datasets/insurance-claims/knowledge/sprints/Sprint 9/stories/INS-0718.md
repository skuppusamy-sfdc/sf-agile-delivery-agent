# INS-0718: Create automated COB (Coordination of Benefits) detection flag on Claim__c at FNOL submission

## Description

As a Claims Operations Manager, I need the system to automatically set a COB_Flag__c on a new Claim__c record when the related policyholder has more than one active Policy__c with overlapping coverage effective dates at the time of FNOL submission, so that claims examiners are immediately alerted to potential coordination of benefits scenarios without manual review.

## Acceptance Criteria

Given a new Claim__c is created with Claim__c.Status__c = 'FNOL Received', when the record is saved, then a Record-Triggered Flow must fire and evaluate whether the related Policy__c.Policyholder__c has two or more Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.Effective_Date__c <= Claim__c.Date_of_Service__c and Policy__c.Expiration_Date__c >= Claim__c.Date_of_Service__c

Given the COB condition is true, when the flow evaluation completes, then Claim__c.COB_Flag__c must be set to TRUE and Claim__c.COB_Review_Required__c must be set to TRUE within the same transaction

Given Claim__c.COB_Flag__c is set to TRUE, when the record is saved, then a Task must be auto-created and assigned to the Claims Operations queue with Subject = 'COB Review Required — ' + Claim__c.Claim_Number__c and Due_Date = TODAY() + 3

Given the COB condition is false, when the flow fires, then Claim__c.COB_Flag__c must remain FALSE and no Task must be created

## Solution

Implement a Record-Triggered Flow 'RTF_Claim_COB_Detection' on Claim__c, triggering on Create when Status__c = 'FNOL Received'. Flow uses a Get Records element to retrieve sibling Policy__c records via Policyholder__c lookup with date overlap criteria. Decision element evaluates record count >= 2. Update Records element sets COB_Flag__c and COB_Review_Required__c. Create Records element generates the Task. Bulkification handled by invoking an Apex action 'COBDetectionService.evaluateCOB()' for large batch scenarios to avoid SOQL limits in Flow. Unit tests cover positive, negative, and bulk scenarios with >= 200 records.

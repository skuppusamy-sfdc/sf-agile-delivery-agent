## Description

As a Provider Network Specialist, I need automated alerts when Provider_Network_Agreement__c (PNA) records are approaching expiration so that I can initiate renewal workflows before providers fall out of network and claims are incorrectly adjudicated at out-of-network rates. Extends INS-0677.

## Acceptance Criteria

Given a scheduled Apex job runs nightly at 02:00 UTC,
When any Provider_Network_Agreement__c.End_Date__c is within 60, 30, or 15 days of the current date AND Provider_Network_Agreement__c.Status__c = 'Active',
Then a PNA_Expiration_Alert__e Platform Event is published for each qualifying record containing fields: PNA_Id__c, Provider_Name__c, Days_Until_Expiration__c, and Network_Tier__c.

Given a PNA_Expiration_Alert__e event is published,
When the subscribing Record-Triggered Flow 'PNA_Expiration_Notification_Flow' processes the event,
Then a Task is created on the Provider_Network_Agreement__c record with Subject = 'PNA Expiring in [X] days – Renewal Required', assigned to Provider_Network_Agreement__c.Owner and due date = Provider_Network_Agreement__c.End_Date__c minus 10 days.

Given a PNA reaches 15-day expiration threshold,
When the alert fires,
Then Provider_Network_Agreement__c.Renewal_Status__c is updated to 'Urgent – Escalated' and the record owner's manager (via Salesforce User hierarchy) receives an additional email notification.

Given Provider_Network_Agreement__c.Status__c = 'Terminated' or 'Expired',
When the nightly job evaluates records,
Then those records are excluded from alert processing.

Given the alert job runs,
When execution completes,
Then a Custom Metadata log entry (PNA_Job_Log__mdt) is written with run timestamp, record count evaluated, and alert count published.

## Solution

Implement a Schedulable Apex class 'PNAExpirationAlertScheduler' that queries Provider_Network_Agreement__c with date-range filters and publishes PNA_Expiration_Alert__e Platform Events in batches of 150 (respecting governor limits). Subscribe to the Platform Event using a Record-Triggered Flow on PNA_Expiration_Alert__e object. Escalation logic for 15-day threshold handled in a separate Apex invocable action 'EscalatePNARenewal' called from the Flow. Custom Metadata Type 'PNA_Alert_Config__mdt' stores threshold day values (60/30/15) to allow admin configuration without code changes. Unit tests achieve >85% coverage with mock Platform Event publishing.

# INS-0753: Agent Portal: Add coverage history lookup for broker-serviced policyholders

## Description

As a Broker Relations Coordinator, I need brokers to be able to look up the full coverage history of policyholders they service through the Agent Portal so that brokers can proactively address renewal opportunities, identify coverage gaps, and answer policyholder inquiries without calling the internal service desk.

## Acceptance Criteria

Given a broker user logs into the Agent Portal Experience Cloud site,
When they search for a policyholder by Policy__c.Policy_Number__c or by the policyholder's last name and date of birth,
Then results must only return Policy__c records where Policy__c.Servicing_Broker__c = the logged-in broker's associated Contact or Account, enforcing data isolation via Sharing Rules.

Given the broker selects a policyholder from the search results,
When the Coverage History page loads,
Then it must display all Policy__c records for that policyholder showing Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Termination_Date__c, Policy__c.Plan_Code__c, and Policy__c.Premium_Amount__c.

Given a Policy__c.Status__c = 'Active',
When displayed in the broker portal coverage list,
Then an 'Active' green badge must appear and a 'Request Renewal Quote' button must be available that launches a new OmniScript 'BrokerRenewalRequest'.

Given a Policy__c.Status__c = 'Lapsed' and Policy__c.Termination_Date__c is within the last 60 days,
When displayed in the broker portal,
Then a 'Reinstatement Eligible' amber badge must appear with a 'Submit Reinstatement' action button.

-Given broker can view Claim__c records for their policyholders from this page (deprecated — claims visibility for brokers is out of scope for Phase 2 per compliance review)-

Given a broker attempts to access a policyholder not linked to their book of business,
When the search or direct URL access is attempted,
Then a 'Not Authorized' error message must be displayed and the access attempt must be logged in a custom object 'Portal_Access_Audit__c' with fields: User__c, Attempted_Policy_Number__c, Timestamp__c, and Access_Result__c = 'Denied'.

## Solution

Build an OmniScript 'BrokerPolicyholderSearch' on the Experience Cloud Agent Portal site. Use a DataRaptor Extract with a SOQL filter enforcing Policy__c.Servicing_Broker__c = :currentUserId to respect broker data isolation. Supplement with a Sharing Rule on Policy__c: 'Share Policy__c records with the broker Contact where Policy__c.Servicing_Broker__c = Contact.Id'. Build a secondary LWC 'brokerCoverageHistoryList' to render the policy list with conditional badge rendering via CSS classes. Launch 'BrokerRenewalRequest' OmniScript from a FlexCard action. For unauthorized access logging, implement an Apex @AuraEnabled method 'logPortalAccessAttempt()' that inserts Portal_Access_Audit__c and is called from the LWC error handler. Apply Guest Profile and Partner Community Profile restrictions via Permission Set.

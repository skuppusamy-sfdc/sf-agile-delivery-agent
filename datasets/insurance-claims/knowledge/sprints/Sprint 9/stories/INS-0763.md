## Description

As a Claims Operations Manager, I need a unified Policyholder 360 FlexCard that surfaces active coverage details, open claims, and recent EOB history in a single view so that I can make faster adjudication decisions without navigating across multiple Salesforce objects.

## Acceptance Criteria

Given a Claims Operations Manager opens the Policyholder 360 FlexCard on a Provider__c (Person Account) record,
When the FlexCard loads,
Then it must display the linked Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.Policyholder__c = current record Id.

Given the Coverage panel is rendered,
When more than one active Policy__c exists,
Then each policy tile must show Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, and Policy__c.Termination_Date__c.

Given the Claims Summary panel is rendered,
When the FlexCard loads,
Then it must display all Claim__c records where Claim__c.Policyholder__c = current record Id and Claim__c.Status__c IN ('Open', 'Pending Review', 'Adjudicated') sorted by Claim__c.Date_of_Loss__c descending, limited to 10 records.

Given an open Claim__c is listed in the Claims Summary panel,
When the user clicks the claim number hyperlink,
Then navigation must route to the Claim__c detail page without opening a new browser tab.

Given the EOB History panel is rendered,
When Explanation_of_Benefits__c records exist linked to the policyholder,
Then the panel must display EOB__c.EOB_Number__c, EOB__c.Payment_Date__c, EOB__c.Total_Billed__c, EOB__c.Total_Allowed__c, and EOB__c.Member_Responsibility__c for the 5 most recent records.

-Given a COB flag is present on the Policy__c record, the FlexCard must render a COB warning banner in red.- (Superseded by INS-0771 COB Alert component)

Given the FlexCard is viewed by a user without the 'Claims_Ops_Full_Access' Permission Set Group,
When the card loads,
Then financial fields (EOB__c.Total_Allowed__c, Payment__c.Amount__c) must be masked and display '***'.

## Solution

Implement using OmniStudio FlexCard with three child cards (Coverage, Claims Summary, EOB History) driven by DataRaptor Extract actions querying Policy__c, Claim__c, and Explanation_of_Benefits__c via SOQL with relationship filters. Use FlexCard Flyout for EOB detail drill-down. Field-level masking enforced via a custom Apex @AuraEnabled method that checks FeatureManagement.checkPermission('View_Financial_Fields'). Deploy via OmniStudio Migration Tool to Production. Assign FlexCard to the ClaimsOps App page via Lightning App Builder.

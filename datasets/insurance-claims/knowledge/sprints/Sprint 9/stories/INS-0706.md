## Description

As a Claims Operations Manager, I need a Payment Disbursement Summary panel on the Claim__c record page that displays all associated Payment__c records, ERA reconciliation status, and total amounts paid versus billed so that I can quickly identify underpayments, overpayments, and unreconciled ERA transactions without running a separate report.

Payment__c records are linked to Claim__c via Payment__c.Claim__c. ERA reconciliation status is tracked on Payment__c.ERA_Status__c. This panel also surfaces any Claim_Line__c level payment variances.

## Acceptance Criteria

Given a Claim__c record with one or more Payment__c child records,
When the Payment Disbursement Summary LWC loads,
Then it displays a summary header showing: Claim__c.Total_Billed__c, Claim__c.Total_Allowed__c, Claim__c.Total_Paid__c, and a computed variance field (Total_Billed__c minus Total_Paid__c).

Given one or more Payment__c records linked to the Claim__c,
When the payment detail table renders,
Then each row displays Payment__c.Payment_Date__c, Payment__c.Payment_Amount__c, Payment__c.Payment_Method__c, Payment__c.Check_EFT_Number__c, Payment__c.ERA_Status__c, and Payment__c.Payee_Name__c.

Given a Payment__c record where Payment__c.ERA_Status__c = 'Unreconciled',
When the table renders that row,
Then the row is highlighted in amber and an 'ERA Pending' badge is displayed in the ERA Status column.

Given a Payment__c record where Payment__c.ERA_Status__c = 'Reconciled',
When the table renders that row,
Then a green checkmark icon is displayed and Payment__c.ERA_Reconciliation_Date__c is shown.

Given a Claim__c where the sum of Payment__c.Payment_Amount__c exceeds Claim__c.Total_Allowed__c,
When the summary header renders,
Then the variance field is displayed in red with the label 'Overpayment Detected' and a 'Flag for Review' button creates a Case__c with RecordType = 'Overpayment_Review' and links it to the Claim__c.

Given a Claim__c where Claim__c.Status__c = 'Adjudicated' and no Payment__c records exist,
When the panel loads,
Then a warning message 'No payments recorded for adjudicated claim — verify disbursement' is displayed.

-Given Claim_Line__c level payment breakdowns are required in this panel — deferred to Phase 3 per scope reduction decision on 2024-11-14-

## Solution

Build LWC 'paymentDisbursementSummary' with Apex controller 'PaymentDisbursementController.getPaymentSummary(claimId)' that queries Payment__c WHERE Claim__c = :claimId and aggregates totals. Variance computation performed server-side in Apex wrapper class and returned as a computed field. Row highlighting driven by conditional CSS classes in LWC template based on ERA_Status__c value. 'Flag for Review' button triggers a Record-Triggered Flow (invoked via LWC dispatchEvent to parent) that creates a Case__c record with the Overpayment_Review RecordType and sets Case__c.Related_Claim__c. Overpayment detection threshold comparison performed in Apex using Decimal comparison. Component placed in a new 'Payment & Disbursement' tab on the Claim__c Lightning Record Page.

## Description

As an Underwriting Analyst, I need a visual coverage history timeline on the Policy__c record page that shows all historical coverage periods, LOB changes, and rider additions/removals so that I can quickly assess coverage continuity during underwriting reviews and claim eligibility checks.

## Acceptance Criteria

Given an Underwriting Analyst navigates to a Policy__c record page,
When the Coverage History Timeline LWC loads,
Then it must retrieve all Coverage_Period__c child records linked via Coverage_Period__c.Policy__c and render them in chronological order by Coverage_Period__c.Effective_Date__c ascending.

Given a coverage gap exists (i.e., two consecutive Coverage_Period__c records where Coverage_Period__c.End_Date__c and the next Coverage_Period__c.Effective_Date__c differ by more than 1 day),
When the timeline renders,
Then a red gap indicator with tooltip text 'Coverage Gap: [N] days' must appear between the affected timeline nodes.

Given a rider was added or removed during a coverage period,
When the timeline node for that period is expanded,
Then the Rider__c.Rider_Name__c, Rider__c.Effective_Date__c, and Rider__c.Status__c fields must be displayed in a nested sub-list.

Given the policy has more than 20 Coverage_Period__c records,
When the timeline first loads,
Then only the 10 most recent records are displayed, with a 'Load More' button to paginate in increments of 10.

-Given the timeline is viewed on mobile, it must collapse to a list view automatically.- (Deferred to Phase 3 mobile initiative)

## Solution

Build as a custom LWC ('coverageHistoryTimeline') using a CSS-based vertical timeline layout. Data fetched via an @AuraEnabled Apex controller (CoverageHistoryController.cls) using a SOQL query on Coverage_Period__c with a subquery on Rider__c. Gap detection logic implemented in Apex and returned as a boolean flag per record. Paginate using server-side OFFSET/LIMIT in SOQL. Deploy component to Policy__c Lightning Record Page via Lightning App Builder. Add component to the 'Underwriting_View' page layout only, restricted by Permission Set.

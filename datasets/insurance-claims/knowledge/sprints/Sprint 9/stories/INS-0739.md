## Description

As a Claims Operations Manager, I need a unified Policyholder 360 FlexCard on the Policy__c record page so that I can quickly assess a member's active coverage, open claims, and recent EOB history without navigating across multiple objects.

## Acceptance Criteria

Given a Policy__c record in 'Active' status, when the Policyholder 360 FlexCard loads, then the card must display Policy__c.Policy_Number__c, Policy__c.Effective_Date__c, Policy__c.Termination_Date__c, and Policy__c.LOB__c in the header band.

Given the policyholder has one or more Claim__c records linked via Claim__c.Policy__c, when the FlexCard renders the Claims section, then all claims where Claim__c.Status__c IN ('Open','Pending Adjudication','Adjudicated') must appear as child cards sorted by Claim__c.Date_of_Service__c descending, showing Claim__c.Claim_Number__c, Claim__c.Total_Billed_Amount__c, and Claim__c.Total_Paid_Amount__c.

Given an Explanation_of_Benefits__c (EOB) record exists with EOB__c.Policy__c populated, when the EOB section renders, then the three most recent EOBs must be listed showing EOB__c.EOB_Date__c, EOB__c.Member_Responsibility__c, and EOB__c.Plan_Paid_Amount__c.

Given the Policy__c.Status__c = 'Lapsed' or 'Terminated', when the FlexCard loads, then a warning banner displaying 'Policy Inactive – Coverage Ended [Termination_Date__c]' must appear at the top of the card.

-Given a COB flag on the policy, when the card loads, display a secondary payer badge- (superseded by INS-0751 COB widget story)

Given any user without the 'Claims_Viewer' Permission Set Group assignment, when they navigate to the Policy__c record page, then the Policyholder 360 FlexCard must not render and must display a 'Restricted Access' placeholder.

## Solution

Build an OmniStudio FlexCard named 'Policyholder360_Card' with a primary DataRaptor Extract sourcing Policy__c fields and two child DataRaptor Turbo Extracts for Claim__c and Explanation_of_Benefits__c child records. Use FlexCard conditional visibility states to toggle the 'Inactive Policy' warning banner based on Policy__c.Status__c value. Apply a custom CSS state for the warning banner using the FlexCard Styles tab. Enforce visibility via a Permission Set Group check using a custom Apex Action called from the FlexCard's 'Before Card Loads' action. Deploy FlexCard to the Policy__c Lightning Record Page via App Builder. SOQL in DataRaptor must include LIMIT 3 ORDER BY EOB__c.EOB_Date__c DESC for the EOB child source.

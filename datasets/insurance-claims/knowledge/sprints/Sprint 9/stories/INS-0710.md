## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record that consolidates credentialing status from Credentialing_Case__c, all active Provider_Network_Agreement__c records, open Provider_Change_Notification__c (PCN) items, and CAQH verification status so that I have a single-pane-of-glass view when handling provider inquiries.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record,
When the Provider 360 FlexCard loads,
Then the Credentialing panel displays the most recent Credentialing_Case__c record with fields: Credentialing_Case__c.Status__c, Credentialing_Case__c.Credentialing_Type__c, Credentialing_Case__c.Expiration_Date__c, and Credentialing_Case__c.CAQH_Profile_Status__c.

Given the provider has multiple active PNAs,
When the Network Agreements panel renders,
Then all Provider_Network_Agreement__c records where Status__c = 'Active' are listed showing: PNA.Contract_Number__c, PNA.Network_Tier__c, PNA.Effective_Date__c, PNA.End_Date__c, and PNA.Fee_Schedule__r.Name.

Given open PCN records exist (Provider_Change_Notification__c.Status__c IN ('Submitted','Under Review')),
When the FlexCard PCN panel loads,
Then each PCN is shown with Provider_Change_Notification__c.Change_Type__c, Provider_Change_Notification__c.Submitted_Date__c, and a direct navigation action to the PCN record.

Given the provider's NPI is stored in Provider__c.NPI__c,
When the FlexCard loads,
Then a callout is made to the NPPES integration endpoint and the returned NPI status is displayed as 'Active' or 'Deactivated' with last verification timestamp.

-Given CAQH_Profile_Status__c = 'Expired', display a red banner with re-attestation link- (replaced by dedicated PCN workflow in INS-0701)

Given a Provider Relations Specialist clicks 'Initiate Renewal' on a PNA card,
When the action fires,
Then an OmniScript 'PNA_Renewal_OS' is launched in a modal pre-populated with the selected PNA's Contract_Number__c and provider NPI.

## Solution

Develop an OmniStudio FlexCard 'Provider_360_FC' with four card states: Credentialing, Network Agreements, PCNs, and NPI Verification. Use DataRaptor Extract actions for Credentialing_Case__c and PNA data. NPPES lookup handled via a custom Integration Procedure 'NPPES_NPI_Lookup_IP' that calls the NPPES REST API (https://npiregistry.cms.hhs.gov/api) and caches results in a custom object NPI_Verification_Cache__c with a 24-hour TTL to minimize external callouts. 'Initiate Renewal' action uses OmniStudio Navigation action to launch PNA_Renewal_OS OmniScript. FlexCard deployed to Provider__c Lightning Record Page via App Builder with visibility rule limiting to 'Provider_Relations' Permission Set Group.

## Description

As an Underwriting Analyst, I need a visual coverage history timeline on the Policy__c record page that displays all historical coverage periods, amendments, and LOB changes so that I can accurately assess risk continuity and identify coverage gaps during policy review. Extends INS-0681 (Policy record page redesign).

## Acceptance Criteria

Given a Policy__c record is open,
When the Coverage History Timeline LWC loads,
Then it must query all related Coverage_Period__c records where Coverage_Period__c.Policy__c = current Policy Id, ordered by Coverage_Period__c.StartDate__c descending.

Given coverage period records exist,
When the timeline renders,
Then each node must display Coverage_Period__c.StartDate__c, Coverage_Period__c.EndDate__c, Coverage_Period__c.CoverageType__c, Coverage_Period__c.LOB__c, and Coverage_Period__c.AmendmentReason__c.

Given a coverage gap exists (EndDate of one period does not equal StartDate of the next),
When the timeline renders,
Then a red gap indicator with the gap duration in days must appear between the two nodes.

Given the timeline has more than 10 coverage period records,
When the component loads,
Then it must paginate with 10 records per page and display a 'Load More' button.

-Given the component loads, it must also display Claim__c count per coverage period- (removed — moved to Policyholder 360 FlexCard INS-0691)

## Solution

Develop a custom LWC 'coverageHistoryTimeline' using a vertical CSS timeline pattern. Use @wire adapter with a custom Apex controller 'CoverageHistoryController.cls' to retrieve Coverage_Period__c records. Gap detection logic implemented in Apex using date arithmetic. Pagination handled client-side via JavaScript tracked properties. Deploy on Policy__c Lightning Record Page via Lightning App Builder. Apex class annotated with @AuraEnabled(cacheable=true) for wire compatibility.

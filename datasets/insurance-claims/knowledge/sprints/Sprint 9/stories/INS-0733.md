## Description

As a Claims Operations Manager, I need the system to automatically detect and flag potential Coordination of Benefits (COB) scenarios when a new Claim__c is submitted for a policyholder who has more than one active Policy__c record with overlapping coverage dates, so that COB review workflows are triggered before adjudication proceeds and overpayment risk is mitigated.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Status__c = 'Submitted', when a Record-Triggered Flow evaluates the policyholder's associated Policy__c records, then if two or more Policy__c records exist where Policy__c.Status__c = 'Active' and Policy__c.LOB__c matches the Claim__c.LOB__c field, the flow must set Claim__c.COBFlag__c = true and Claim__c.Status__c = 'COB Review'.

Given Claim__c.COBFlag__c is set to true, when the record is saved, then a Task must be created and assigned to the Claims Operations queue with Subject = 'COB Review Required – [Claim__c.ClaimNumber__c]', Priority = 'High', and ActivityDate = TODAY() + 2 business days.

Given the COB flag is triggered, when the Claim__c record page is opened, then a lightning-alert banner with severity 'warning' must display the message: 'COB Review Required: Multiple active policies detected. Adjudication is paused pending review.'

Given Claim__c.COBFlag__c = true and Claim__c.Status__c = 'COB Review', when a Claims Operations Manager manually sets Claim__c.COBReviewOutcome__c to either 'COB Applicable' or 'COB Not Applicable' and saves, then Claim__c.Status__c must automatically transition to 'In Review' and the COB warning banner must be dismissed.

-Given COB is detected, send an automated email to the policyholder — deprecated per Compliance Officer review, replaced by internal Task workflow only-

## Solution

Implement a Record-Triggered Flow named 'Claim_COB_Detection_Flow' that fires After Save on Claim__c insert when Status__c = 'Submitted'. Use a Get Records element to retrieve related Policy__c records filtered by PolicyHolder__c and LOB__c. Add a Decision element to evaluate count of active overlapping policies. Use an Update Records element to set COBFlag__c = true and Status__c = 'COB Review'. Use a Create Records element to generate the Task assigned to the Claims Ops queue. Build an LWC 'cobAlertBanner' using @api-decorated properties to read Claim__c.COBFlag__c and conditionally render a lightning-alert. Add a separate Record-Triggered Flow 'Claim_COB_Resolution_Flow' that fires on COBReviewOutcome__c change to handle status transition.

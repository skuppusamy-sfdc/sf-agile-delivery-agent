## Description

As a Claims Operations Manager, I need an Agentforce AI-assisted FNOL triage prompt template integrated into the Claim__c intake flow so that claims adjusters can receive instant AI-generated coverage verification summaries, recommended ICD-10/CPT code suggestions, and SIU referral flags during First Notice of Loss intake, reducing average FNOL handling time by at least 20%.

This story establishes the foundational Agentforce prompt template and agent action configuration. A follow-on story will handle the full OmniScript integration. The AI model must reference live Policy__c, Claim__c, and Provider__c data via grounding, not hallucinate coverage details.

The prompt must adhere to Acme Insurance's AI Governance Policy v2.1, requiring all AI-generated outputs to display a confidence score and a disclaimer when confidence < 80%.

## Acceptance Criteria

Given an adjuster initiates a new Claim__c record with Claim__c.Status__c = 'FNOL' and a linked Policy__c,
When the Agentforce Claims Triage Agent action is invoked from the Claim__c record page,
Then the AI must return a structured triage summary containing: Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Termination_Date__c, active coverage confirmation (Yes/No), and any Policy__c.Exclusion_Notes__c relevant to the claim type.

Given the adjuster enters a free-text incident description in the FNOL intake panel,
When the Agentforce prompt processes the input,
Then the AI must suggest a ranked list of up to 3 ICD-10 codes and up to 3 CPT codes with confidence scores (0–100%) for each suggestion, grounded against Claim_Line__c historical data for the same LOB__c.

Given an AI-generated ICD-10 or CPT suggestion has a confidence score below 80%,
When the suggestion is rendered in the adjuster UI,
Then a yellow warning banner must display: 'AI Confidence Below Threshold – Manual Verification Required' per AI Governance Policy v2.1.

Given Claim__c.Total_Billed_Amount__c > $50,000 OR the incident description contains SIU trigger keywords (configurable via Custom Metadata Type: SIU_Trigger_Keywords__mdt),
When the Agentforce triage completes,
Then an SIU referral flag (Claim__c.SIU_Flag__c = true) must be automatically set and a Task must be created assigned to the SIU queue with Subject = 'AI-Flagged SIU Review – FNOL Triage'.

Given the Agentforce agent action executes,
When grounding data is retrieved from Policy__c and Claim__c,
Then all data retrieval must occur via named Apex action (AgentforceClaimsTriageAction.cls) with enforced CRUD/FLS checks — no direct SOQL in the prompt template.

-Given the agent returns a triage summary, automatically pre-populate Claim__c.Diagnosis_Code__c with the top ICD-10 suggestion.- (Deprecated: legal review determined auto-population of diagnosis codes requires adjuster explicit confirmation; deferred to INS-0783)

Given an adjuster accepts an AI-suggested ICD-10 or CPT code by clicking 'Apply',
When the acceptance is recorded,
Then an AI_Interaction_Log__c record must be created with fields: Claim__c lookup, Suggested_Code__c, Confidence_Score__c, Adjuster_Action__c = 'Accepted', Interaction_Timestamp__c.

Given the Agentforce agent is invoked on a Policy__c with Status__c = 'Lapsed' or 'Cancelled',
When the triage summary is generated,
Then the coverage confirmation field must return 'No – Policy Not Active' and the adjuster must be prompted to initiate a coverage dispute workflow before proceeding.

## Solution

1. Configure an Agentforce Agent (Claims_Triage_Agent) in Setup > Agents with the 'Claims Adjuster' user profile persona and a custom system prompt defining the triage role, output format, and AI Governance Policy v2.1 disclaimer rules.
2. Build the Prompt Template (PT_FNOL_Triage) in Prompt Builder using the 'Record' template type grounded on Claim__c, with merge fields pulling Policy__c data via relationship traversal (Claim__c.Policy__r fields).
3. Create an Apex invocable action class (AgentforceClaimsTriageAction.cls) decorated with @InvocableMethod that performs SOQL with enforced CRUD/FLS on Policy__c, Claim__c, and Claim_Line__c, returning a structured TriageSummaryWrapper to the agent.
4. Implement ICD-10/CPT suggestion logic using a custom Apex scoring method that queries historical Claim_Line__c records grouped by LOB__c and ICD10_Code__c/CPT_Code__c, returning ranked suggestions with frequency-based confidence scores.
5. Create SIU_Trigger_Keywords__mdt Custom Metadata Type with records for initial keyword set (e.g., 'fraud', 'staged', 'exaggerated', 'prior injury'). The Apex action evaluates the incident description against active keyword records.
6. Build an LWC (agentforceTriagePanel) embedded on the Claim__c record page via a utility bar component, rendering the AI triage output with Accept/Reject actions per suggestion and the AI Governance disclaimer banner.
7. Create AI_Interaction_Log__c custom object with all required fields; write to it via a Record-Triggered Flow on the LWC's accept/reject custom event firing a Platform Event (AI_Interaction_Event__e).
8. Ensure the prompt template is version-controlled; deploy via unlocked package. Minimum 85% Apex coverage on AgentforceClaimsTriageAction.cls including SIU flag and lapsed policy branches.

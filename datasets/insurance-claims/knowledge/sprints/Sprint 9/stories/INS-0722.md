## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant embedded on the Claim__c record page to proactively surface an SIU (Special Investigations Unit) referral recommendation when a claim exhibits high-risk indicators — such as duplicate billing, provider anomalies, or ICD-10 diagnosis code combinations flagged in our fraud ruleset — so that examiners receive AI-assisted guidance without leaving the claims workspace.

## Acceptance Criteria

Given the Agentforce Claims AI assistant is active on a Claim__c record page, when the page loads for a Claim__c where Claim__c.SIU_Risk_Score__c >= 75, then the assistant must proactively display a recommendation card stating 'High fraud risk detected — consider SIU referral' without requiring the examiner to prompt the AI

Given the recommendation card is displayed, when the examiner clicks 'View Risk Factors', then the assistant must enumerate the specific triggered risk indicators from Claim__c.SIU_Risk_Flags__c (a multi-select picklist) in a readable bulleted format, including flag labels such as 'Duplicate CPT Billing', 'Unbundling Detected', 'ICD-10 Inconsistency', and 'Provider Anomaly'

Given the examiner clicks 'Initiate SIU Referral' in the assistant panel, when the action is confirmed, then an Agentforce Action must invoke an Apex action 'SIUReferralService.createReferral()' that creates a Case record with RecordType = 'SIU_Referral', sets Case.Related_Claim__c = current Claim__c.Id, and sets Claim__c.SIU_Referred__c = TRUE in a single transaction

Given the SIU referral Case is created successfully, when the Agentforce action completes, then the assistant must display a confirmation message with the new Case.CaseNumber and a clickable link to the Case record

Given a Claim__c where Claim__c.SIU_Risk_Score__c < 75, when the Agentforce assistant loads, then no SIU recommendation card must be displayed and the assistant must remain in its default claims inquiry mode

Given the Agentforce assistant surfaces an SIU recommendation, when the examiner dismisses it by clicking 'Dismiss', then Claim__c.SIU_Recommendation_Dismissed__c must be set to TRUE and the recommendation must not reappear unless Claim__c.SIU_Risk_Score__c is recalculated and increases above the threshold

## Solution

Configure an Agentforce Agent 'Claims AI Assistant' with a custom Topic 'SIU Risk Detection' and associated Actions. Define an Agent Action 'CheckSIURiskScore' as a Flow-type action that reads Claim__c.SIU_Risk_Score__c and Claim__c.SIU_Risk_Flags__c via a Get Records Flow element and returns structured data to the agent context. Define a second Agent Action 'InitiateSIUReferral' backed by Apex invocable 'SIUReferralService.createReferral()'. Configure the Topic's Instructions to trigger proactively when risk score threshold is met using an Einstein Generative AI prompt template 'SIU_Recommendation_Prompt_v1' stored in Prompt Builder. The dismiss action is handled by a Record Update Flow invocable. Agentforce embedded in Claim__c page via the Agent component in Lightning App Builder. Audit all AI recommendations to Compliance_Audit__e Platform Event for regulatory traceability.

## Description

As a Claims Operations Manager, I need a Policyholder 360 FlexCard that surfaces active Policy__c records, open Claim__c records, and current coverage details in a single consolidated view so that I can quickly assess a policyholder's full profile without navigating across multiple objects during FNOL intake.

## Acceptance Criteria

Given a Policyholder (Person Account) record is open in Salesforce,
When the Claims Operations Manager views the Policyholder 360 FlexCard,
Then the FlexCard must display all active Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.PolicyHolder__c = current Account Id.

Given active policies are displayed,
When the user expands a policy tile,
Then the card must show Policy__c.LOB__c, Policy__c.EffectiveDate__c, Policy__c.ExpirationDate__c, Policy__c.GroupNumber__c, and Policy__c.TPA__c lookup name.

Given open claims exist for the policyholder,
When the FlexCard loads,
Then all Claim__c records where Claim__c.Status__c IN ('Open','In Review','Pending Documentation') and Claim__c.Policyholder__c = current Account Id must render in a datatable showing Claim__c.ClaimNumber__c, Claim__c.DateOfLoss__c, Claim__c.TotalBilledAmount__c, and Claim__c.TotalPaidAmount__c.

Given a policyholder has no open claims,
When the FlexCard renders the claims section,
Then an empty-state message 'No open claims on file' must display.

-Given the card loads, a COB indicator badge must show if Policy__c.COBStatus__c = 'Secondary'- (superseded by INS-0688 COB component)

Given any data load error occurs from a DataRaptor Extract,
When the FlexCard fails to retrieve records,
Then a user-friendly error toast 'Unable to load policyholder data. Please refresh or contact support.' must appear and the error must be logged to a custom Exception_Log__c record.

## Solution

Build an OmniStudio FlexCard 'Policyholder360Card' with two child card states: ActivePolicies and OpenClaims. Use DataRaptor Extract actions to query Policy__c and Claim__c with SOQL filters. Embed the FlexCard on the Person Account Lightning Record Page via a Lightning App Builder tab. Apply a custom Permission Set Group 'ClaimsOps_Policyholder360' to restrict visibility. Use conditional rendering actions within the FlexCard for empty-state handling. Log DataRaptor errors via a Platform Event 'Integration_Error__e' consumed by a Record-Triggered Flow on Exception_Log__c.

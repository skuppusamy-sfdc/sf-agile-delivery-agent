## Description

As a Provider Network Specialist, I need an automated workflow that initiates a provider network termination process, generates the required Provider Change Notification (PCN), and surfaces an open claims impact assessment so that we meet state-mandated termination notice timelines and proactively manage in-flight claims before the termination effective date.

## Acceptance Criteria

Given a Network_Termination_Case__c record is created with Network_Termination_Case__c.Status__c = 'Initiated', when the Record-Triggered Flow fires, then the flow must automatically set Network_Termination_Case__c.Notice_Required_By__c = Network_Termination_Case__c.Termination_Effective_Date__c minus the value stored in Network_Termination_Case__c.State_Notice_Days__c.

Given the Network_Termination_Case__c.Status__c transitions to 'PCN Pending', when the flow advances to the PCN generation step, then a Provider_Change_Notification__c record must be created with PCN__c.Provider__c, PCN__c.Termination_Case__c, PCN__c.PCN_Type__c = 'Termination', PCN__c.Effective_Date__c = Network_Termination_Case__c.Termination_Effective_Date__c, and PCN__c.Status__c = 'Draft'.

Given the PCN__c.Status__c = 'Draft', when the Provider Network Specialist clicks 'Send for DocuSign Signature', then the DocuSign eSignature integration must be triggered using the PCN DocuSign template, routing to the signer defined in PCN__c.Signatory_Contact__c, and PCN__c.DocuSign_Envelope_Id__c must be populated upon envelope creation.

Given a provider has open Claim__c records where Claim__c.Rendering_Provider__c = the terminating Provider__c.Id AND Claim__c.Status__c IN ('Open','Pending Adjudication'), when the termination case is initiated, then a Claims Impact Assessment section on the Network_Termination_Case__c must display the count of impacted claims (Network_Termination_Case__c.Open_Claims_Count__c) and the total billed amount (Network_Termination_Case__c.Open_Claims_Billed_Total__c).

Given the Network_Termination_Case__c.Termination_Effective_Date__c is within 30 days and open claims exist, when the nightly Platform Event batch runs, then a Task must be created and assigned to the owning Provider Relations Specialist with Subject = 'Urgent: Open Claims Require Reassignment Prior to Termination' and ActivityDate = TODAY() + 1.

## Solution

Build a Record-Triggered Flow 'ProviderTermination_RTF' on Network_Termination_Case__c triggering on Create and Update. Use a Decision element to branch on Status__c value. Invoke an Apex Action 'calculateNoticeRequiredDate' for state-specific notice day logic sourced from a Custom Metadata Type 'State_Termination_Rules__mdt'. Create PCN__c records via a Flow Create Records element. Integrate DocuSign via the DocuSign for Salesforce managed package action 'Send with DocuSign' configured with the PCN template ID stored in a Custom Setting. Compute open claims impact using an Apex-invocable method 'computeClaimsImpact(terminationCaseId)' that aggregates Claim__c via SOQL aggregate query. Publish a Platform Event 'TerminationAlert__e' from a Scheduled-Triggered Flow that queries Network_Termination_Case__c records within 30-day window nightly, triggering Task creation via a Platform Event-Triggered Flow.

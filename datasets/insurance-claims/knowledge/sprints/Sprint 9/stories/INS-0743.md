## Description

Steps to Reproduce:
1. Navigate to any Person Account record that has been configured as a policyholder but has NO associated Policy__c records (e.g., a newly onboarded member pending first enrollment).
2. Open the Account Lightning Record Page where the 'coverageHistoryTimeline' LWC (delivered in INS-0740) is embedded.
3. Observe the component rendering behavior.

Expected Behavior: The LWC should render an empty state illustration with the message 'No coverage history found for this policyholder.' and a 'Enroll in New Policy' quick action button.

Actual Behavior: The component throws an uncaught JavaScript TypeError: 'Cannot read properties of undefined (reading length)' in the browser console. The entire component area goes blank and a generic Salesforce error spinner persists indefinitely. The Lightning Record Page partially breaks, collapsing adjacent components in the same region.

Environment: Observed in Acme Insurance FULL SANDBOX. Not yet verified in Production. Salesforce API version 59.0. Browser: Chrome 124.

## Acceptance Criteria

As per Description. Additionally: Given the Apex method 'getCoverageHistory(accountId)' returns an empty list, when the LWC's @wire handler receives the result, then the component must handle the empty array gracefully without throwing a JS exception, and the empty state message 'No coverage history found for this policyholder.' must render within the component boundary.

## Solution

Add a null/empty array guard in the LWC 'coverageHistoryTimeline' JavaScript controller within the @wire handler for getCoverageHistory. Replace the direct array.length reference with an optional chaining check (data?.length ?? 0). Add an 'isEmpty' tracked boolean property set to true when the returned list is empty or null, and use a conditional template directive in the HTML to render the empty state template block. Update the Apex Controller 'getCoverageHistoryController.getCoverageHistory()' to return an empty List<PolicyWrapper> instead of null when no records are found, to prevent the wire adapter from receiving an undefined payload.

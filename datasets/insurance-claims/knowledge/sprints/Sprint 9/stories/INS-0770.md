# INS-0770: Bug: Network_Termination_Case__c fails to update linked Provider_Network_Agreement__c status when termination reason is 'Voluntary Withdrawal'

## Description

**Bug Report — Network Termination Case Automation**

**Steps to Reproduce:**
1. Open an existing Provider_Network_Agreement__c record with PNA__c.Status__c = 'Active'.
2. Create a new Network_Termination_Case__c child record with Network_Termination_Case__c.Termination_Reason__c = 'Voluntary Withdrawal' and Network_Termination_Case__c.Effective_Termination_Date__c = a future date.
3. Set Network_Termination_Case__c.Status__c = 'Approved'.
4. Save the record.
5. Navigate back to the parent Provider_Network_Agreement__c record.

**Expected Behavior:**
Upon the Network_Termination_Case__c reaching Status__c = 'Approved', the parent PNA__c.Status__c should automatically update to 'Pending Termination' and PNA__c.Termination_Effective_Date__c should be populated with the value from Network_Termination_Case__c.Effective_Termination_Date__c.

**Actual Behavior:**
The PNA__c.Status__c remains 'Active' and PNA__c.Termination_Effective_Date__c is null. No error is thrown. The Record-Triggered Flow ('NTC_Approval_PNA_Update_Flow') appears to execute (flow interview logs show completion) but the PNA fields are not updated.

**Note:** This behavior is ONLY observed when Termination_Reason__c = 'Voluntary Withdrawal'. All other termination reasons ('Cause', 'Non-Renewal', 'Regulatory Action') correctly trigger the PNA update. Introduced in Sprint 8 when 'Voluntary Withdrawal' was added as a new picklist value — the flow Decision element was not updated to include this value in the routing criteria.

**Environment:** Acme Insurance Partial Sandbox (SF-PARTIAL-01) and confirmed in Full Sandbox (SF-FULL-02)
**Severity:** High — affects provider offboarding compliance SLA (PCN must be sent within 30 days of termination approval per CMS regulations)

## Acceptance Criteria

As per Description. Fix must ensure that when Network_Termination_Case__c.Status__c = 'Approved' and Termination_Reason__c = 'Voluntary Withdrawal', the parent PNA__c.Status__c is updated to 'Pending Termination' and PNA__c.Termination_Effective_Date__c is populated. Regression test must confirm all five termination reason values correctly trigger the PNA update. A Provider_Change_Notification__c record must also be auto-created as per existing flow logic for other termination reasons.

## Solution

Fix the 'NTC_Approval_PNA_Update_Flow' Record-Triggered Flow by updating the Decision element condition to include 'Voluntary Withdrawal' in the OR criteria alongside existing termination reason values. Alternatively, refactor the Decision element to use a formula-based condition checking that Status__c = 'Approved' regardless of Termination_Reason__c value, since all approved terminations should trigger the same PNA update path. Update the flow in Sandbox, execute regression test using Flow Debug with all five termination reason scenarios, document results in the test matrix, then promote to Production via Change Set. Update the Provider_Change_Notification__c creation logic in the same flow to ensure PCN records are generated for Voluntary Withdrawal cases to meet CMS 30-day notification SLA.

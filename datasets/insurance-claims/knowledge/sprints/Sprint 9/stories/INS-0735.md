## Description

**Bug Report – INS-0735**

The Coverage History LWC introduced in INS-0732 throws an unhandled JavaScript TypeError and renders a blank tab when any Policy__c record in the policyholder's coverage history has a null value for Policy__c.TPA__c (the TPA lookup field).

**Steps to Reproduce:**
1. Navigate to any Person Account record with RecordType = 'Policyholder'.
2. Ensure at least one related Policy__c record has Policy__c.TPA__c = null (no TPA assigned — common for direct-pay individual policies).
3. Click the 'Coverage History' tab on the record page.
4. Observe the tab content area.

**Expected Behavior:** The Coverage History tab renders all Policy__c records successfully. For rows where Policy__c.TPA__c is null, the TPA column should display a dash ('—') or the text 'N/A — Direct Pay'.

**Actual Behavior:** The LWC JavaScript controller throws 'TypeError: Cannot read properties of null (reading 'Name')' in the browser console. The entire Coverage History tab renders blank with no error message surfaced to the user. All Policy__c records, including those with valid TPA__c values, fail to render.

**Environment:** Acme Insurance – Partial Sandbox (PS2)
**Affected Apex Class:** PolicyholderCoverageHistoryController.getCoverageHistoryByPolicyholder()
**Affected LWC:** policyholderCoverageHistory.js (line ~87, TPA name accessor)
**Regression:** Introduced in Sprint 9 deployment on 2024-11-04.

## Solution

In policyholderCoverageHistory.js, add a null-safe accessor for the TPA name field: replace direct property access 'policy.TPA__r.Name' with an optional chaining expression 'policy.TPA__r?.Name ?? "N/A — Direct Pay"'. In the Apex controller PolicyholderCoverageHistoryController, ensure the SOQL query uses a LEFT OUTER JOIN pattern (standard SOQL behavior) and that the wrapper class TPA field is typed as String with a null-safe assignment. Add a @AuraEnabled(cacheable=true) method unit test in PolicyholderCoverageHistoryControllerTest covering the scenario where Policy__c.TPA__c is null. Deploy fix to PS2 sandbox, validate with QA on the affected policyholder records, then promote to UAT.

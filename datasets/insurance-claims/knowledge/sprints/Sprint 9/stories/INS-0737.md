# INS-0737: Agentforce Claims AI: enable intelligent claim triage suggestions based on ICD-10 and CPT code patterns on FNOL submission

## Description

As a Claims Operations Manager, I need the Agentforce AI assistant embedded on the Claim__c record page to analyze ICD-10 diagnosis codes and CPT procedure codes submitted at FNOL and suggest an appropriate claim triage category and routing queue so that high-complexity or SIU-referral-worthy claims are identified earlier in the process, reducing manual review time and improving adjudication accuracy.

## Acceptance Criteria

Given a Claim__c record with Claim__c.Status__c = 'FNOL Submitted' is opened in Salesforce, when a Claims Operations Manager opens the Agentforce panel, then the AI assistant must display a 'Triage Analysis' card populated with data derived from the associated Claim_Line__c records' ICD10Code__c and CPTCode__c fields.

Given the Agentforce triage analysis is invoked, when the AI evaluates the CPT and ICD-10 code combinations, then it must return one of the following suggested triage categories: 'Routine', 'Complex Medical', 'High-Cost Implant', 'Behavioral Health', 'Workers Comp – Lost Time', or 'SIU Referral Recommended', along with a confidence score expressed as a percentage.

Given the AI suggests 'SIU Referral Recommended' with a confidence score >= 75%, when the Claims Operations Manager reviews the suggestion, then a one-click action button 'Refer to SIU' must be presented that, when clicked, creates a new Case record with RecordType = 'SIU Investigation', links it to the Claim__c via Case.RelatedClaim__c, and sets Case.Priority = 'High'.

Given the triage suggestion is displayed, when the Claims Operations Manager accepts the suggestion by clicking 'Apply Triage', then Claim__c.TriageCategory__c must be updated to the suggested value and Claim__c.AssignedQueue__c must be set to the corresponding routing queue mapped to that triage category.

Given the AI triage analysis is run, when the interaction completes, then the AI prompt, response payload, confidence score, and accepted/rejected outcome must be persisted to a new AgentforceInteractionLog__c record linked to the Claim__c for auditability and model feedback purposes.

## Solution

Configure an Agentforce Agent of type 'Claims Triage Assistant' using Salesforce Agentforce Agent Builder. Define a custom Agent Action named 'AnalyzeClaimCodesForTriage' implemented as an Apex class that queries Claim_Line__c records by Claim__c Id, extracts ICD10Code__c and CPTCode__c values, and invokes an Einstein Prompt Template named 'ClaimTriagePromptTemplate' via the Connect API. The prompt template uses a structured system prompt with few-shot examples mapping code pattern families to triage categories. The Apex action returns a structured JSON response with triageCategory and confidenceScore fields. Implement a Lightning Record Page Action button 'Apply Triage' as an LWC invoking a Record-Triggered Flow 'Apply_Triage_Suggestion_Flow' to update Claim__c fields. Create the 'Refer to SIU' quick action as a Screen Flow. Persist all interactions to AgentforceInteractionLog__c via a Platform Event 'AgentforceInteraction__e' consumed by a Platform Event-Triggered Flow.

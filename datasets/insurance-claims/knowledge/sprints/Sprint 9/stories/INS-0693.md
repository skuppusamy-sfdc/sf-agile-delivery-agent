## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record that consolidates credentialing status from Credentialing_Case__c, active Provider_Network_Agreement__c records, and any open Provider_Change_Notification__c items so that I can manage provider relationships from a single pane of glass without switching between multiple object tabs.

## Acceptance Criteria

Given a Provider__c Person Account record is open,
When the Provider 360 FlexCard renders,
Then it must display the most recent Credentialing_Case__c record where Credentialing_Case__c.Provider__c = current Account Id, showing Credentialing_Case__c.Status__c, Credentialing_Case__c.ExpirationDate__c, and Credentialing_Case__c.CaqhProfileId__c.

Given the credentialing expiration date is within 90 days,
When the FlexCard renders the credentialing section,
Then a yellow warning badge 'Expiring Soon' must display adjacent to Credentialing_Case__c.ExpirationDate__c.

Given the credentialing expiration date is past today,
When the FlexCard renders,
Then a red badge 'Expired' must display and the PNA section must show a system warning 'Verify network eligibility before processing claims'.

Given active PNA records exist,
When the agreements section loads,
Then all Provider_Network_Agreement__c records where PNA.Status__c = 'Active' and PNA.Provider__c = current Account Id must list PNA.NetworkName__c, PNA.EffectiveDate__c, PNA.TerminationDate__c, and PNA.LOB__c.

Given open PCN records exist,
When the notifications panel renders,
Then all Provider_Change_Notification__c records where PCN.Provider__c = current Account Id and PCN.Status__c NOT IN ('Closed','Withdrawn') must display with PCN.ChangeType__c and PCN.SubmittedDate__c.

Given the provider's NPI is on file,
When the header section renders,
Then Provider__c.NPI__c and Provider__c.TIN__c must display in the card header alongside the provider's specialty.

## Solution

Build OmniStudio FlexCard 'Provider360Card' with three card states: CredentialingStatus, NetworkAgreements, and PendingNotifications. Use DataRaptor Extract actions for each state with appropriate SOQL. Implement conditional state actions for badge coloring using FlexCard conditional visibility. Embed on Provider Person Account Lightning Record Page. Integrate CAQH profile link using Provider__c.CaqhProfileId__c as a URL action. Apply Permission Set Group 'ProviderRelations_360' for access control.

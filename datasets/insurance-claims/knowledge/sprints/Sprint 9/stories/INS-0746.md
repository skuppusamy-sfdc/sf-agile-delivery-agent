## Description

Steps to Reproduce:
1. Open an existing Provider_Network_Agreement__c (PNA) record that has an active, linked Fee_Schedule__c record (Fee_Schedule__c.Status__c = 'Active').
2. Amend the linked Fee_Schedule__c record: update Fee_Schedule__c.Effective_Date__c to a new mid-term date and change Fee_Schedule__c.Fee_Schedule_Type__c from 'Medicare-Based' to 'Contracted Rate'.
3. Save the Fee_Schedule__c record.
4. Navigate back to the PNA record and open the Provider 360 FlexCard (INS-0741).
5. Observe the Fee Schedule section of the FlexCard.

Expected Behavior: The FlexCard Fee Schedule section should immediately reflect the updated Fee_Schedule__c.Effective_Date__c and Fee_Schedule__c.Fee_Schedule_Type__c = 'Contracted Rate'.

Actual Behavior: The FlexCard Fee Schedule section continues to display the old Fee_Schedule__c.Effective_Date__c and Fee_Schedule__c.Fee_Schedule_Type__c = 'Medicare-Based' until a hard browser refresh is performed. The stale data persists for approximately 15–20 minutes, which is the OmniStudio DataRaptor cache TTL. This is causing Provider Relations Specialists to apply incorrect fee schedule information when authorizing claim payments against PNA records.

Environment: Reproducible in FULL SANDBOX and PARTIAL SANDBOX. Salesforce API version 59.0. OmniStudio version 248.0.0.

## Acceptance Criteria

As per Description. Additionally: Given a Fee_Schedule__c record linked to a PNA is updated (any field change), when the Provider 360 FlexCard is next loaded on the associated PNA or Provider__c record, then the FlexCard must display the current Fee_Schedule__c field values with zero stale cache data — confirmed by verifying Fee_Schedule__c.LastModifiedDate matches the displayed data context.

## Solution

Investigate the DataRaptor Turbo Extract 'DR_Extract_FeeSchedule' cache configuration within the 'ProviderBase_Card' FlexCard (extended in INS-0741). Disable or reduce the DataRaptor cache TTL for this specific DataRaptor by setting the Cache TTL to 0 in the DataRaptor configuration for the Fee Schedule extract. Alternatively, implement a Record-Triggered Flow 'FeeScheduleChange_CacheInvalidation_RTF' on Fee_Schedule__c (After Update) that publishes a Platform Event 'CacheInvalidation__e' with the related PNA Id, consumed by a Platform Event-Triggered Flow that stamps a Fee_Schedule__c.Last_Cache_Bust__c DateTime field — this field should be included as a DataRaptor source field to force cache-key variation on each load. Evaluate whether the FlexCard action 'Refresh' can be auto-triggered on component mount via a FlexCard lifecycle action to force a fresh DataRaptor call on every page load as a short-term mitigation.

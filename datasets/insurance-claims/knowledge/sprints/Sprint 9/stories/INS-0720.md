## Description

As a Provider Network Specialist, I need an automated workflow that, upon a Network_Termination_Case__c being approved, generates a Provider_Change_Notification__c (PCN) record, populates the termination letter via Conga CLM, and dispatches it to the provider via DocuSign so that we meet the contractually mandated 90-day termination notice SLA without manual document preparation.

## Acceptance Criteria

Given a Network_Termination_Case__c with Status__c = 'Approved' is saved, when the Record-Triggered Flow fires, then a Provider_Change_Notification__c record must be created with PCN.Type__c = 'Termination', PCN.Provider__c = Network_Termination_Case__c.Provider__c, PCN.Effective_Termination_Date__c = Network_Termination_Case__c.Approved_Date__c + 90, and PCN.Status__c = 'Pending Delivery'

Given the PCN record is created, when the flow continues, then a Conga CLM document generation callout must be triggered using the 'ProviderTerminationLetter_v2' template, merging Provider__c.Name, Provider__c.NPI__c, Provider__c.TIN__c, PNA.Contract_Number__c, and PCN.Effective_Termination_Date__c into the letter body

Given the Conga CLM document is generated successfully, when the document is available as a ContentDocument, then a DocuSign envelope must be sent to Provider__c.Email (mapped from Person Account PersonEmail field) with the termination letter attached and a signature required from the provider

Given the DocuSign envelope status returns 'Completed', when the webhook callback is received, then PCN.Status__c must be updated to 'Delivered', PCN.Delivery_Timestamp__c must be stamped with the completion datetime, and Network_Termination_Case__c.Status__c must advance to 'Notice Sent'

Given the DocuSign envelope status returns 'Declined' or 'Voided', when the callback is received, then PCN.Status__c must be set to 'Delivery Failed' and a High-priority Case must be created and assigned to the Provider Network queue

-Given a manual override checkbox on PCN to bypass DocuSign and send via postal mail (deprecated — postal mail process handled offline per Compliance Officer decision on 2024-11-01)-

Given the entire workflow completes, when Network_Termination_Case__c.Status__c = 'Notice Sent', then all related Provider_Network_Agreement__c records for the provider must have PNA.Pending_Termination__c set to TRUE

## Solution

Implement Record-Triggered Flow 'RTF_NetworkTermination_PCNWorkflow' on Network_Termination_Case__c firing on Update when Status__c changes to 'Approved'. Flow creates PCN via Create Records element. Conga CLM invoked via Apex invocable action 'CongaCLMService.generateDocument()' using named credential 'Conga_CLM_Prod'. DocuSign envelope sent via DocuSign for Salesforce managed package Apex API 'dfsle.EnvelopeService.sendEnvelope()'. DocuSign status callback handled by a custom REST endpoint '/services/apexrest/docusign/callback' updating PCN and NTC records. Bulk PNA update performed via a second Apex invocable to avoid Flow DML limits. All callouts wrapped in Queueable Apex to respect mixed-DML restrictions.

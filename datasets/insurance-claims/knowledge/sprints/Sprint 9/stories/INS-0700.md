## Description

As an Underwriting Analyst, I need a chronological coverage history timeline displayed on the policyholder's record so that I can quickly identify plan changes, LOB transitions, lapses in coverage, and retroactive enrollment events without pulling manual reports.

Coverage history is sourced from the Coverage_History__c object (child of Policy__c) and must reflect all historical plan versions including COBRA periods, open enrollment changes, and mid-year special enrollment events.

## Acceptance Criteria

Given a Policy__c record with one or more Coverage_History__c child records,
When the Coverage History Timeline LWC component loads,
Then all Coverage_History__c records are displayed in descending chronological order by Coverage_History__c.Effective_Date__c.

Given a Coverage_History__c record where Coverage_History__c.Change_Reason__c = 'COBRA',
When the timeline renders that record,
Then the entry is highlighted with a yellow indicator and the label 'COBRA Period' is appended to the plan name.

Given a gap exists between Coverage_History__c.End_Date__c of one record and Coverage_History__c.Effective_Date__c of the next,
When the timeline renders,
Then a 'Coverage Lapse' marker is injected between the two entries showing the number of lapse days.

Given a Coverage_History__c record where Coverage_History__c.Retro_Adjustment__c = true,
When the timeline renders that record,
Then a 'Retro' badge is displayed and Coverage_History__c.Retro_Adjustment_Date__c is shown.

Given the timeline has more than 20 Coverage_History__c records,
When the component loads,
Then only the 10 most recent records are shown by default with a 'Show Full History' toggle to load all records.

## Solution

Develop a custom LWC 'coverageHistoryTimeline' using a vertical timeline CSS layout. Wire to an Apex controller method getCoverageHistory(policyId) that queries Coverage_History__c ordered by Effective_Date__c DESC with a default LIMIT 10. Lapse detection logic implemented in Apex by comparing sorted date ranges and returning synthetic 'lapse' marker objects in the response list. Retro badge and COBRA highlighting driven by conditional CSS classes in the LWC template. 'Show Full History' toggle calls a second Apex method with no LIMIT. Place component on Policy__c Lightning Record Page in a new 'Coverage History' tab.

## Description

As a Broker Relations Coordinator, I need a visual coverage history timeline on the Policy__c record page so that I can quickly review all historical coverage periods, LOB changes, and plan amendments for a policyholder during broker-assisted inquiries without pulling external reports.

Coverage history is currently tracked across multiple Policy__c child records and a legacy Coverage_History__c custom object. The timeline must consolidate both sources and present them in chronological order. This component must support at least 10 years of coverage history records without pagination performance degradation.

## Acceptance Criteria

Given a Policy__c record is open and Coverage_History__c records exist with Coverage_History__c.Policy__c matching the current record ID,
When the Coverage History LWC loads,
Then all Coverage_History__c records must be displayed in a vertical timeline sorted by Coverage_History__c.Effective_Date__c ascending.

Given a Coverage_History__c record has Coverage_History__c.Change_Reason__c = 'Plan Amendment',
When the timeline node is rendered,
Then the node must display an amber 'Amendment' badge alongside Coverage_History__c.Plan_Name__c, Coverage_History__c.Effective_Date__c, and Coverage_History__c.End_Date__c.

Given a Coverage_History__c record has Coverage_History__c.Change_Reason__c = 'Termination',
When the timeline node is rendered,
Then the node must display a red 'Terminated' badge and Coverage_History__c.Termination_Reason__c must be shown in a tooltip on hover.

Given the policy has LOB__c values spanning multiple lines of business across history records,
When the timeline is filtered using the LOB dropdown,
Then only Coverage_History__c records matching the selected LOB__c value must be displayed.

-Given Coverage_History__c records exceed 50, implement server-side pagination with page size = 10.- (Deprecated: replaced by virtual scrolling per UX review on 2024-11-14)

Given the LWC is loaded on a mobile form factor (Salesforce Mobile App),
When the timeline renders,
Then it must be fully responsive and timeline nodes must stack vertically without horizontal scroll.

Given no Coverage_History__c records exist for the Policy__c,
When the component loads,
Then an empty state illustration with the message 'No coverage history available for this policy.' must be displayed.

## Solution

1. Create a custom LWC (coverageHistoryTimeline) using a SLDS timeline pattern. Use @wire(getRelatedListRecords) to fetch Coverage_History__c records filtered by Policy__c lookup.
2. Implement virtual scrolling using a custom scroll observer (IntersectionObserver API) within the LWC to handle large history sets without pagination, replacing the deprecated server-side pagination approach.
3. Build an Apex controller (CoverageHistoryController.cls) with a @AuraEnabled(cacheable=true) method that queries Coverage_History__c with ORDER BY Effective_Date__c ASC, supporting LOB__c filtering via dynamic SOQL.
4. Use LWC getter functions for badge color logic based on Change_Reason__c values (Plan Amendment = amber, Termination = red, Renewal = green).
5. Add the LWC to the Policy__c Lightning Record Page in a new 'Coverage History' tab using App Builder.
6. Enforce field-level security on Coverage_History__c.Termination_Reason__c — visible only to PSG_Claims_Staff and PSG_Broker_Coordinators.
7. Write Jest unit tests for all badge rendering scenarios and the empty state condition.

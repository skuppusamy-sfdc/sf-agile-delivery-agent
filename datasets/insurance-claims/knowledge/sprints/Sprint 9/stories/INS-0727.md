## Description

As a Provider Network Specialist, I need a dedicated Lightning dashboard that surfaces all Provider_Network_Agreement__c records expiring within 30, 60, and 90 days so that I can proactively initiate contract renewals via Conga CLM before agreements lapse and providers fall out of network.

## Acceptance Criteria

Given Provider_Network_Agreement__c records with PNA_Expiration_Date__c within 30 days of today, when the dashboard loads, then a '30-Day Expiring PNAs' report chart displays the count grouped by Provider__c.Specialty__c.

Given Provider_Network_Agreement__c records with PNA_Expiration_Date__c between 31 and 60 days from today, when the dashboard loads, then a '60-Day Expiring PNAs' report table displays PNA_Number__c, Provider__c.Name, PNA_Expiration_Date__c, and PNA_Status__c.

Given Provider_Network_Agreement__c records with PNA_Expiration_Date__c between 61 and 90 days from today, when the dashboard loads, then a '90-Day Expiring PNAs' report table is rendered with the same columns.

Given a Provider_Network_Agreement__c.PNA_Status__c = 'Terminated' or 'Superseded', when the dashboard loads, then those records are excluded from all three expiration buckets.

Given the dashboard is accessed by a user without the 'Provider_Network_Specialist' Permission Set Group, when the page loads, then the dashboard components return zero records due to Sharing Rule restrictions scoped to Provider Network team ownership.

Given a Provider_Network_Agreement__c enters the 30-day expiration window, when the scheduled job runs nightly at 02:00 UTC, then a Task is created on the PNA record assigned to the owning Provider Network Specialist with Subject = 'PNA Renewal Required — Expiring in 30 Days'.

## Solution

Create three custom Report Types on Provider_Network_Agreement__c with date filter formulas referencing TODAY(). Dashboard built in Lightning with three report-driven components (donut chart for 30-day, tabular for 60 and 90-day). Sharing Rules configured for Provider Network team role hierarchy. Scheduled Apex class 'PNAExpirationAlertBatch.cls' (implements Database.Batchable, Schedulable) runs nightly to query expiring PNAs and create Task records via DML. Permission Set Group 'Provider_Network_Specialist' controls dashboard visibility. Conga CLM renewal initiation button added to PNA record page as a Quick Action calling the Conga CLM REST API via Named Credential.

## Description

**Bug Report – Production Defect**

When a Claim__c has Coordination of Benefits (COB) active (Policy__c.COB_Indicator__c = true) and multiple Payment__c records exist (one per carrier), the generated Explanation_of_Benefits__c PDF is displaying the sum of ALL Payment__c.Amount_Paid__c values rather than only the primary carrier's payment amount. This results in inflated disbursement totals on member-facing EOB documents, which is a compliance violation under CMS EOB accuracy guidelines.

**Steps to Reproduce:**
1. Open a Policy__c record with COB_Indicator__c = true and Primary_Carrier__c populated.
2. Navigate to a related Claim__c record with Status__c = 'Adjudicated' and two related Payment__c records: one with Payment__c.Carrier_Type__c = 'Primary' (Amount_Paid__c = $150.00) and one with Payment__c.Carrier_Type__c = 'Secondary' (Amount_Paid__c = $40.00).
3. Click 'Generate EOB' button on the Claim__c record.
4. Open the generated Explanation_of_Benefits__c PDF.

**Expected Result:**
EOB PDF displays Payment__c.Amount_Paid__c = $150.00 under 'Amount Paid by Plan' and $40.00 in a separate 'COB Secondary Payment' line.

**Actual Result:**
EOB PDF displays $190.00 under 'Amount Paid by Plan' with no separate COB line, misrepresenting total plan liability.

**Environment:** Production & Full Sandbox
**Frequency:** 100% reproducible when COB_Indicator__c = true
**Affected Records:** ~340 EOBs generated since Sprint 8 deployment (INS-0758 introduced the multi-payment logic)
**Severity:** P1 – Compliance/Financial Impact

## Acceptance Criteria

As per Description.

Additionally:
Given a Claim__c with COB_Indicator__c = true is adjudicated and EOB is generated,
When the Conga CLM EOB template merges Payment__c data,
Then only Payment__c records where Carrier_Type__c = 'Primary' must contribute to the 'Amount Paid by Plan' field (EOB_Primary_Payment__c) and Payment__c records where Carrier_Type__c = 'Secondary' must populate EOB_COB_Secondary_Amount__c as a separate line.

Given the fix is deployed,
When the batch reprocessing job runs against the ~340 affected Explanation_of_Benefits__c records,
Then all EOB_Primary_Payment__c values must reflect only the primary Payment__c.Amount_Paid__c and audit log entries must be created in EOB_Correction_Log__c for each corrected record.

## Solution

**Root Cause:** The Apex method EOBGenerationService.calculateTotalPaid() introduced in INS-0758 uses a SUM aggregate SOQL query on Payment__c without a WHERE Carrier_Type__c = 'Primary' filter when COB_Indicator__c = true.

**Fix Approach:**
1. Update EOBGenerationService.cls: add a conditional branch — if Claim__c.Policy__r.COB_Indicator__c == true, filter the Payment__c query with WHERE Carrier_Type__c = 'Primary' for primary amount and a separate query for 'Secondary'.
2. Update the Conga CLM EOB merge template to include the new EOB_COB_Secondary_Amount__c merge field as a distinct line item in the COB section (coordinate with Conga template admin).
3. Add Explanation_of_Benefits__c.EOB_COB_Secondary_Amount__c (Currency) and EOB_Primary_Payment__c (Currency) fields if not already present; deprecate the overloaded EOB_Total_Paid__c field for COB scenarios.
4. Write a one-time batch class (BatchEOBCOBCorrection) to reprocess the ~340 affected records, regenerate PDFs, and write correction entries to EOB_Correction_Log__c (new custom object with fields: EOB_Id__c, Original_Amount__c, Corrected_Amount__c, Correction_Date__c, Corrected_By__c).
5. Add regression test in EOBGenerationServiceTest.cls specifically covering COB = true with mixed Carrier_Type__c payment records.
6. Notify Compliance Officer (per HIPAA EOB accuracy policy) before deploying the batch reprocessing job.

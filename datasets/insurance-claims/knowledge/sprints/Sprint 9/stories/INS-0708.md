## Description

As a Claims Operations Manager, I need a chronological coverage history timeline on the Policy__c record page that shows all historical plan changes, LOB transitions, and TPA assignments so that I can verify eligibility windows accurately when adjudicating retroactive claims.

## Acceptance Criteria

Given a Policy__c record is opened,
When the Coverage History Timeline LWC component loads,
Then all Policy_Coverage_History__c child records related to Policy__c are displayed in descending order by Policy_Coverage_History__c.Effective_Date__c.

Given coverage history records exist,
When the timeline renders each entry,
Then each node displays: Policy_Coverage_History__c.Plan_Name__c, Policy_Coverage_History__c.LOB__c, Policy_Coverage_History__c.TPA_Name__c, Policy_Coverage_History__c.Effective_Date__c, Policy_Coverage_History__c.End_Date__c, and Policy_Coverage_History__c.Change_Reason__c.

Given a coverage gap exists (Policy_Coverage_History__c.End_Date__c of one record does not immediately precede the next record's Effective_Date__c),
When the timeline renders,
Then a red 'Coverage Gap' indicator is inserted between the two nodes showing the number of gap days.

Given more than 20 history records exist,
When the timeline first loads,
Then only the 20 most recent are shown with a 'Load More' button that retrieves the next 20 via @wire.

-Given a history record has Change_Reason__c = 'Administrative Correction', display an audit footnote inline- (removed per Compliance Officer review, audit trail handled separately in ComplianceReporting component)

## Solution

Develop a custom LWC 'coverageHistoryTimeline' placed in the Policy__c Lightning Record Page via App Builder. Use @wire with a custom Apex method 'getCoverageHistory(policyId, pageSize, offset)' returning a paginated list of Policy_Coverage_History__c records. Gap detection logic implemented in Apex by comparing sorted date ranges server-side. Timeline visual built with SLDS path/timeline markup. Component communicates with parent page via Lightning Message Service (LMS) channel 'CoverageContext_MC' to allow other components (e.g., INS-0707 FlexCard) to react to selected history nodes.

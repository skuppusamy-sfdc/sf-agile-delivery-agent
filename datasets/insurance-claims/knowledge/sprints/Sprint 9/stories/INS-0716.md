## Description

As a Claims Operations Manager, I need a dedicated Coverage History tab on the Policy__c Lightning record page that displays all historical and current Coverage__c snapshots, filterable by LOB__c, so that I can quickly identify coverage gaps or overlaps when adjudicating a claim.

## Acceptance Criteria

Given a Policy__c record, when the 'Coverage History' tab is selected, then an LWC datatable must load all Coverage__c records related to the policy, sorted by Coverage__c.Effective_Date__c descending

Given the coverage history datatable is loaded, when the user selects a LOB filter value (e.g., 'Medical', 'Dental', 'Vision'), then the datatable must re-query and display only Coverage__c records where Coverage__c.LOB__c matches the selected value without a full page reload

Given a Coverage__c record where Coverage__c.End_Date__c < TODAY(), when it appears in the datatable, then the row must be styled with a grey 'Expired' badge in the Coverage__c.Status__c column

Given a Coverage__c record where Coverage__c.Effective_Date__c <= TODAY() AND Coverage__c.End_Date__c >= TODAY(), when displayed, then the row must show a green 'Active' badge

Given more than 50 Coverage__c records exist on a single policy, when the tab loads, then server-side pagination must be applied with a default page size of 25 records and page navigation controls visible

Given the user clicks a Coverage__c row, when the row expands, then Coverage__c.Deductible_Amount__c, Coverage__c.Copay_Amount__c, Coverage__c.Coinsurance_Pct__c, and Coverage__c.Network_Tier__c must display in an inline detail panel

## Solution

Build LWC 'coverageHistoryTab' using @wire(getRelatedListRecords) with dynamic filtering passed as @track properties. Server-side pagination implemented via Apex controller 'CoverageHistoryController' with OFFSET/LIMIT SOQL. LOB filter rendered as lightning-combobox populated from Coverage__c.LOB__c picklist values via Apex describe call. Row expansion uses conditional rendering with CSS transition. Badge logic driven by formula fields Coverage__c.Coverage_Status_Display__c deployed as part of this story. Add the LWC to Policy__c Lightning App Builder page under a new 'Coverage History' tab component.

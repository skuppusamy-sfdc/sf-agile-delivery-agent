## Description

As a policyholder accessing the self-service portal, I need to view a clear Explanation of Benefits (EOB) summary for each processed Claim__c so that I can understand what was billed, what my insurance covered, and what my out-of-pocket responsibility is without calling the claims helpline.

## Acceptance Criteria

Given an authenticated policyholder accesses the self-service portal and navigates to their claims list,
When they select a Claim__c with Status__c = 'Adjudicated',
Then the EOB Summary component loads displaying the associated Explanation_of_Benefits__c record with fields: EOB.Date_Processed__c, EOB.Total_Billed_Amount__c, EOB.Plan_Paid_Amount__c, EOB.Member_Responsibility__c, EOB.Deductible_Applied__c, EOB.Copay_Amount__c, EOB.Coinsurance_Amount__c.

Given the EOB Summary is displayed,
When the policyholder expands the Claim Line Details section,
Then all child Claim_Line__c records are listed showing: Claim_Line__c.CPT_Code__c, Claim_Line__c.Service_Date__c, Claim_Line__c.Billed_Amount__c, Claim_Line__c.Allowed_Amount__c, Claim_Line__c.Paid_Amount__c, and Claim_Line__c.Denial_Reason__c (if applicable).

Given a Claim__c has Status__c = 'Pending' or 'In Review',
When the policyholder navigates to that claim,
Then the EOB Summary section displays a message: 'Your claim is still being processed. EOB will be available once adjudication is complete.' and the EOB fields are not rendered.

Given the EOB is rendered,
When the policyholder clicks 'Download EOB PDF',
Then a Conga CLM-generated PDF of the Explanation_of_Benefits__c record is downloaded, formatted per the standard EOB template 'EOB_Member_Template_v2'.

Given the policyholder's Experience Cloud profile,
When the component loads,
Then only Claim__c records where Claim__c.Policyholder__c = the logged-in User's Contact are accessible, enforced via Sharing Rules on Claim__c (Owner-based sharing + sharing rule: 'Share with Policyholder Contact').

## Solution

Build an LWC 'eobSummaryPortal' deployed to the Experience Cloud self-service site (Acme Member Portal). Use @wire with Apex method 'getEOBForClaim(claimId)' decorated with @AuraEnabled(cacheable=true), enforcing CRUD/FLS checks and WITH SECURITY_ENFORCED in SOQL. Claim_Line__c child records fetched in the same Apex call via nested query. PDF download triggers a Conga CLM REST API callout from an Apex action 'generateEOBPdf' passing the EOB record Id and template name. Guest user access is blocked via a custom Permission Set 'Member_Portal_EOB_Viewer' required for component visibility. Sharing enforced at the org level via Apex Managed Sharing on Claim__c with share reason 'Policyholder_Access'.

## Description

As a Claims Operations Manager, I need the system to automatically generate and deliver an Explanation_of_Benefits__c (EOB) document to the policyholder when a Claim__c record transitions to Status__c = 'Adjudicated' so that we meet regulatory delivery timelines and eliminate manual EOB preparation effort. This story extends INS-0672 (Payment__c disbursement automation).

## Acceptance Criteria

Given a Claim__c record Status__c is updated to 'Adjudicated',
When the Record-Triggered Flow fires on Claim__c,
Then a new Explanation_of_Benefits__c record must be created with EOB.Claim__c = Claim Id, EOB.PolicyHolder__c = Claim__c.Policyholder__c, EOB.AdjudicatedDate__c = today, EOB.TotalBilledAmount__c = Claim__c.TotalBilledAmount__c, EOB.TotalAllowedAmount__c = Claim__c.TotalAllowedAmount__c, EOB.TotalPaidAmount__c = Claim__c.TotalPaidAmount__c, and EOB.Status__c = 'Generated'.

Given the EOB record is created,
When the Conga CLM template merge runs,
Then the EOB PDF must include all related Claim_Line__c records showing Claim_Line__c.CPTCode__c, Claim_Line__c.ICD10Code__c, Claim_Line__c.BilledAmount__c, Claim_Line__c.AllowedAmount__c, Claim_Line__c.PatientResponsibility__c, and Claim_Line__c.DenialReason__c (if applicable).

Given the EOB PDF is generated,
When the delivery step executes,
Then the document must be sent via DocuSign to the email on file at Policyholder Account.PersonEmail with subject 'Your Explanation of Benefits — Claim [Claim__c.ClaimNumber__c]'.

Given DocuSign delivery succeeds,
When the envelope status returns 'Delivered',
Then EOB.Status__c must update to 'Delivered' and EOB.DeliveredDate__c must be stamped with the delivery timestamp.

Given DocuSign delivery fails after 3 retry attempts,
When the failure is confirmed,
Then EOB.Status__c must set to 'Delivery Failed', a Task must be created assigned to the owning Claims Operations Manager with Subject 'Manual EOB Delivery Required — [ClaimNumber__c]', and a Platform Event 'EOB_Delivery_Failure__e' must fire.

-Given COB applies, a secondary EOB must be generated for the secondary carrier- (deferred to Phase 3 COB enhancement)

Given the EOB is generated for any claim,
When the record is saved,
Then EOB.RegulatoryDeadline__c must be calculated as AdjudicatedDate__c + 30 days per state compliance rules.

## Solution

Implement a Record-Triggered Flow on Claim__c (after update, Status__c changed to 'Adjudicated') that calls an invocable Apex action 'EOBGenerationService.cls' to create the Explanation_of_Benefits__c record and trigger Conga CLM merge via REST API callout. DocuSign envelope creation handled via DocuSign for Salesforce managed package API. Retry logic implemented in a Queueable Apex class 'EOBDeliveryRetryQueueable.cls' with a max retry counter stored on EOB.RetryCount__c. Platform Event 'EOB_Delivery_Failure__e' published on final failure and consumed by a separate notification Flow. All callouts wrapped in try/catch with Exception_Log__c entries.

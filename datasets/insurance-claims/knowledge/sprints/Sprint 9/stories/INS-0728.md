## Description

As a Credentialing Specialist, I need a Credentialing_Case__c record to be automatically created when a new Provider__c (Person Account) is saved with a valid NPI verified against the NPPES registry so that the credentialing workflow is triggered without manual case creation, reducing onboarding delays and compliance risk.

## Acceptance Criteria

Given a new Provider__c record is saved with Provider__c.NPI__c populated, when the Record-Triggered Flow fires on the after-insert context, then an outbound NPPES API callout is initiated using the stored NPI value.

Given the NPPES API returns a 200 response with a matching NPI record, when the callout completes, then Provider__c.NPI_Verified__c is set to true and Provider__c.NPI_Verification_Date__c is stamped with the current date.

Given Provider__c.NPI_Verified__c transitions to true, when the verification is confirmed, then a Credentialing_Case__c record is created with Credentialing_Case__c.Provider__c = the new Provider__c Id, Credentialing_Case__c.Status__c = 'Initiated', Credentialing_Case__c.Case_Type__c = 'Initial Credentialing', and Credentialing_Case__c.CAQH_Profile_ID__c populated if available from the NPPES response.

Given the NPPES API returns a 404 or the NPI is deactivated, when the callout completes, then Provider__c.NPI_Verified__c = false, a Provider_Change_Notification__c record is created with PCN_Type__c = 'NPI Verification Failure', and the assigned Credentialing Specialist is notified via a Salesforce in-app notification.

Given a Provider__c record is updated (not inserted) with a changed NPI__c value, when the Record-Triggered Flow evaluates, then the NPI verification and case creation logic is re-executed only if the new NPI differs from the prior value.

-Given a Provider__c.Provider_Type__c = 'Facility', when NPI verification succeeds, then a Credentialing_Case__c is also created for the associated Facility__c record.- (Deprecated — Facility credentialing flow handled separately per INS-0711)

## Solution

Record-Triggered Flow on Provider__c (after save, insert and update contexts) calls an invocable Apex method 'NPPESVerificationService.cls' that performs an HTTP GET callout to the NPPES NPI Registry API (https://npiregistry.cms.hhs.gov/api/) via Named Credential 'NPPES_Registry'. Response parsing extracts NPI status and CAQH profile data. Credentialing_Case__c creation handled within the same Apex transaction using a dedicated CaseFactory pattern. PCN creation and in-app notification triggered via a separate after-insert Record-Triggered Flow on Provider__c.NPI_Verified__c field change. All callouts wrapped in try/catch with error logging to Integration_Error_Log__c. Governor limit guard: callout deferred to a Queueable if invoked from a batch context.

## Description

As an Underwriting Analyst, I need a visual coverage history timeline on the Policyholder (Person Account) record page that shows all historical and current Policy__c records with effective and termination dates so that I can identify coverage gaps, lapses, and overlapping periods when evaluating new policy applications or claim eligibility. Extends INS-0755 Policyholder 360 FlexCard by providing a drill-down timeline view.

## Acceptance Criteria

Given a Policyholder record page is loaded in Lightning Experience,
When the Coverage History Timeline LWC component renders,
Then it must display all Policy__c records linked to the account sorted by Policy__c.Effective_Date__c descending, showing: Policy__c.Name, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Termination_Date__c, Policy__c.Status__c, Policy__c.Carrier__c.

Given a gap exists between consecutive Policy__c records (Termination_Date__c of record N < Effective_Date__c of record N+1),
When the timeline renders,
Then a red 'Coverage Gap' indicator must be displayed between those two policy bars showing the gap duration in days.

Given a policyholder has more than 20 historical Policy__c records,
When the timeline first loads,
Then only the most recent 10 records are shown with a 'Load More' button; all records must be retrievable via pagination.

Given a Policy__c.Status__c = 'Lapsed',
When that policy bar is rendered on the timeline,
Then it must be styled with a grey striped pattern and a 'Lapsed' badge.

Given a user clicks on any policy bar in the timeline,
When the click event fires,
Then a side panel must open displaying Policy__c full detail including associated Claim__c count and EOB summary.

## Solution

Develop LWC 'coverageHistoryTimeline' using a custom SVG/CSS timeline layout. Wire to @wire getRecord and a custom Apex controller 'CoverageHistoryController.cls' with a method getCoverageHistory(accountId) performing SOQL on Policy__c with ORDER BY Effective_Date__c DESC. Gap detection logic implemented in Apex, returning a wrapper class with gap intervals. Pagination handled via server-side offset/limit. Side panel uses lightning-record-detail-form. Add component to Policyholder Lightning Record Page via App Builder. Include JEST unit tests for gap calculation logic.

# INS-0750: Credentialing Case auto-creation on CAQH profile verification failure

## Description

As a Credentialing Specialist, I need a Credentialing_Case__c to be automatically created when the inbound CAQH verification integration response indicates a failed or incomplete provider profile so that credentialing exceptions are tracked, assigned, and resolved within SLA without requiring manual monitoring of integration logs.

## Acceptance Criteria

Given the CAQH integration polling job receives a response payload with status = 'FAILED' or 'INCOMPLETE' for a given NPI,
When the DataRaptor Transform processes the inbound payload,
Then a Credentialing_Case__c record must be created with Credentialing_Case__c.Status__c = 'Pending Review', Credentialing_Case__c.Source__c = 'CAQH', Credentialing_Case__c.NPI__c = the NPI from the payload, and Credentialing_Case__c.Failure_Reason__c populated from the payload's 'failureDescription' field.

Given a Credentialing_Case__c is created from a CAQH failure,
When the record is saved,
Then it must be automatically assigned to the Credentialing Specialist queue ('Credentialing_Review_Queue') via Assignment Rule.

Given a Credentialing_Case__c with Source__c = 'CAQH' already exists in status 'Pending Review' or 'In Review' for the same NPI,
When a duplicate CAQH failure response is received,
Then no new Credentialing_Case__c must be created; instead, a note must be appended to the existing case's Credentialing_Case__c.Integration_Log__c field with a timestamp.

Given a Credentialing_Case__c is created,
When the SLA timer starts,
Then an Entitlement Process 'CAQH_Credentialing_SLA' must be applied with a 5-business-day resolution milestone.

## Solution

Build an OmniScript 'CAQHVerificationProcessor' triggered by a scheduled DataRaptor that calls the CAQH REST API endpoint. On failure status, invoke a DataRaptor Turbo Action to upsert Credentialing_Case__c using NPI__c as the external ID key for deduplication. Use a Apex @InvocableMethod 'appendIntegrationLog' called from within the OmniScript to handle the duplicate-log-append scenario. Apply Entitlement Process 'CAQH_Credentialing_SLA' via a Record-Triggered Flow on Credentialing_Case__c insert when Source__c = 'CAQH'. Configure Case Assignment Rule 'CAQH_Credentialing_Assignment' to route to Credentialing_Review_Queue.

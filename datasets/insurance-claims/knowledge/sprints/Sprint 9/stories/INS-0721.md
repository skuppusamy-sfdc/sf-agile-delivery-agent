## Description

**Bug Report — INS-0721**

The Coverage History tab introduced in INS-0716 throws an unhandled Apex exception when a Policy__c record has zero related Coverage__c records, crashing the entire Lightning record page for the user.

**Steps to Reproduce:**
1. Navigate to any Policy__c record where no Coverage__c child records exist (e.g., newly created test policy with Policy__c.Status__c = 'Pending').
2. Click the 'Coverage History' tab on the Lightning record page.
3. Observe the tab area — the LWC renders a red error banner and the browser console shows: 'System.QueryException: List has no rows for assignment to SObject' originating from CoverageHistoryController.getCoverageRecords().

**Expected Behavior:** The tab should render an empty-state message: 'No coverage records found for this policy.' with no exception thrown.

**Actual Behavior:** An unhandled Apex exception crashes the LWC and displays a generic Salesforce error to the user. The error is also logged in the debug logs but no user-friendly messaging is shown.

**Environment:** Acme Insurance Full Sandbox (Sprint 9 deployment). Reproducible 100% of the time on zero-coverage policies.

## Acceptance Criteria

As per Description. Additionally: CoverageHistoryController.getCoverageRecords() must use a List<Coverage__c> return type (not a single SObject assignment) so that an empty query result returns an empty list without throwing a QueryException. The LWC must handle an empty list response by rendering the empty-state message component.

## Solution

Fix CoverageHistoryController.getCoverageRecords() to assign SOQL result to List<Coverage__c> instead of a single Coverage__c SObject variable. Add null/empty list check before returning. Update LWC 'coverageHistoryTab' to conditionally render an empty-state <c-empty-state> component when the returned list length === 0. Add a new @isTest method 'testGetCoverageRecords_EmptyResult' to CoverageHistoryControllerTest covering the zero-record scenario. Deploy hotfix to Full Sandbox for QA verification before merging to Phase 2 package.

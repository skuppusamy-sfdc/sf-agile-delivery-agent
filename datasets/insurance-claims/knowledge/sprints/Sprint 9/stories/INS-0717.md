## Description

As a Provider Relations Specialist, I need the existing Provider 360 FlexCard on the Provider__c (Person Account) record page to be extended with a credentialing status panel and a summary of active Provider_Network_Agreement__c (PNA) contracts so that I have a single-pane view when fielding provider inquiries. Extends INS-0685.

## Acceptance Criteria

Given a Provider__c record, when the Provider 360 FlexCard loads, then a 'Credentialing Status' panel must display the most recent Credentialing_Case__c record related to the provider, showing Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and Credentialing_Case__c.CAQH_Profile_ID__c

Given a Credentialing_Case__c.Status__c = 'Expired' or Credentialing_Case__c.Expiration_Date__c < TODAY() + 30, when the panel renders, then a red warning banner 'Credentialing Expiring or Expired' must appear with a link to the Credentialing_Case__c record

Given one or more Provider_Network_Agreement__c records where PNA.Status__c = 'Active' and PNA.Provider__c = current record, when the FlexCard loads, then a PNA summary table must show PNA.Contract_Number__c, PNA.Network__c, PNA.Effective_Date__c, PNA.Termination_Date__c, and PNA.Fee_Schedule__r.Name

Given a Provider__c with no active PNA records, when the PNA section loads, then display 'No active network agreements on file'

-Given a 'Recontract' quick action button in the PNA section (superseded — quick action moved to Network_Termination_Case__c workflow in INS-0720)-

Given the Provider__c.NPI__c or Provider__c.TIN__c fields are null, when the FlexCard loads, then display an inline validation warning 'NPI or TIN missing — data integrity risk' in the header section

## Solution

Extend the existing OmniStudio FlexCard 'Provider360Card' by adding two new child card states: 'CredentialingPanel' and 'PNASummaryPanel'. Add a DataRaptor Extract 'DR_GetCredentialingStatus' querying Credentialing_Case__c ordered by CreatedDate DESC with LIMIT 1. Add a second DataRaptor Extract 'DR_GetActivePNAs' querying Provider_Network_Agreement__c with Status__c filter. Conditional rendering expressions handle warning banners. NPI/TIN null check implemented as a FlexCard conditional state. Deploy via OmniStudio Migration Tool; update existing FlexCard version to v3.2 in the Phase 2 package.

## Description

As a Credentialing Specialist, I need a credentialing status panel on the Provider__c (Person Account) record page that shows the current Credentialing_Case__c status, CAQH profile completeness score, last CAQH sync timestamp, and NPI verification status so that I can manage provider onboarding efficiently and ensure network participation eligibility before claims are processed against the provider.

CAQH data is sourced via the existing CAQH REST API integration (DataIntegration middleware) and stored on the Credentialing_Case__c object.

## Acceptance Criteria

Given a Provider__c record with an associated Credentialing_Case__c record,
When the Credentialing Status Panel LWC loads,
Then it displays Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, Credentialing_Case__c.CAQH_Profile_Score__c, Credentialing_Case__c.Last_CAQH_Sync__c, and Credentialing_Case__c.NPI_Verified__c.

Given Credentialing_Case__c.Status__c = 'Approved',
When the panel renders,
Then a green 'Credentialed' badge is displayed and Credentialing_Case__c.Expiration_Date__c is shown with a countdown in days.

Given Credentialing_Case__c.Status__c IN ('Pending', 'In Review', 'Suspended'),
When the panel renders,
Then a yellow 'Pending Credentialing' badge is shown and all associated Claim__c records for the provider display a 'Credentialing Hold' warning via a related list alert.

Given Credentialing_Case__c.Last_CAQH_Sync__c is older than 180 days,
When the panel renders,
Then an 'Outdated CAQH Data' warning is shown with a 'Trigger CAQH Sync' button that calls the CAQH API integration via an Apex invocable action.

Given Credentialing_Case__c.NPI_Verified__c = false,
When the panel renders,
Then the NPI field displays a red 'Unverified' indicator and a link to the NPPES NPI Registry lookup URL is provided.

Given a Provider__c record with no associated Credentialing_Case__c record,
When the panel loads,
Then a message 'No Credentialing Case Found — Initiate Credentialing' is displayed with a Quick Action button to create a new Credentialing_Case__c.

## Solution

Build LWC 'credentialingStatusPanel' wired to Apex controller 'CredentialingStatusController.getCredentialingData(providerId)'. Apex queries Credentialing_Case__c filtered by Credentialing_Case__c.Provider__c with ORDER BY CreatedDate DESC LIMIT 1. Badge and indicator logic via conditional rendering in LWC template. 'Trigger CAQH Sync' button invokes @InvocableMethod 'CAQHSyncInvocable' which calls the CAQH REST API via Named Credential and updates Credentialing_Case__c.Last_CAQH_Sync__c and CAQH_Profile_Score__c. NPPES URL constructed dynamically using Provider__c.NPI__c. Quick Action for new Credentialing_Case__c uses a standard Screen Flow pre-populated with Provider__c.Id. Component placed on Provider__c (Person Account) Lightning Record Page.

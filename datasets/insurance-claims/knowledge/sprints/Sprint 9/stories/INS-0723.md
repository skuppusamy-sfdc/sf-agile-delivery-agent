## Description

As a Claims Operations Manager, I need a Policyholder 360 FlexCard that surfaces active coverage details, recent claim activity, and COB information in a single consolidated view so that agents can quickly assess a member's full insurance picture without navigating multiple records.

## Acceptance Criteria

Given a Policy__c record in 'Active' status, when a Claims Operations Manager opens the Policyholder 360 FlexCard, then the card displays Policy__c.Policy_Number__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, and Policy__c.LOB__c.

Given an active policyholder with one or more Claim__c records, when the FlexCard loads, then the Recent Claims datatable displays the last 5 Claim__c records sorted by Claim__c.FNOL_Date__c descending, showing Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, and Claim__c.Total_Paid_Amount__c.

Given a policyholder with COB information, when the FlexCard renders, then a COB indicator badge is shown and the secondary payer name from Policy__c.COB_Secondary_Payer__c is displayed.

-Given a TPA-managed policy, when the card loads, then the TPA name field is shown in the header section.- (Deprecated — TPA details moved to separate TPA Summary FlexCard per INS-0698)

Given a policyholder with no Claim__c records, when the FlexCard loads, then the Recent Claims section displays the message 'No claims on file for this member.'

Given any user without the 'Policyholder_360_Viewer' Permission Set Group assigned, when they attempt to load the FlexCard, then the component renders a permission error state and logs an Application_Error__c record.

## Solution

Build OmniStudio FlexCard 'Policyholder360Card' with a primary DataRaptor Extract sourcing Policy__c and related Contact (Person Account) fields. A child DataRaptor Extract retrieves the top 5 Claim__c records via SOQL with ORDER BY FNOL_Date__c DESC LIMIT 5. COB badge rendered as a conditional FlexCard state using Policy__c.COB_Active__c boolean. Permission enforcement via Permission Set Group 'Policyholder_360_Viewer' applied at the FlexCard action level. FlexCard embedded on the Policy__c Lightning Record Page via App Builder.

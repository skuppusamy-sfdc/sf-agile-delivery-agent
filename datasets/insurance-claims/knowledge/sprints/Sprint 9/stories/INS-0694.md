## Description

**Bug Report**

**Summary:** Coverage_Period__c records are not correctly filtering by LOB__c when a user selects a Line of Business filter on the Policyholder summary screen, causing all coverage periods to display regardless of the selected LOB.

**Steps to Reproduce:**
1. Navigate to any Person Account record with multiple Policy__c records spanning different LOBs (e.g., 'Medical', 'Dental', 'Vision').
2. Open the Policyholder Summary screen (INS-0691 FlexCard).
3. Use the LOB filter dropdown and select 'Dental'.
4. Observe the Coverage_Period__c datatable results.

**Expected Behavior:** Only Coverage_Period__c records where Coverage_Period__c.LOB__c = 'Dental' should display.

**Actual Behavior:** All Coverage_Period__c records for all LOBs continue to display. The LOB__c filter value is being passed as null to the DataRaptor Extract action due to a missing input map binding on the FlexCard filter component.

**Environment:** Acme Insurance - Full Sandbox (FS1)
**Affected Release:** Sprint 9 deploy 11/14/2024
**Frequency:** 100% reproducible
**Impact:** Claims Operations Managers are reviewing incorrect coverage data during FNOL intake, creating risk of erroneous claim adjudication.

## Acceptance Criteria

As per Description. Fix must ensure DataRaptor Extract input parameter 'LOB_Filter' is correctly bound to the FlexCard filter component output value. Regression test must confirm all LOB filter values ('Medical','Dental','Vision','Behavioral Health') return correctly scoped Coverage_Period__c records.

## Solution

Inspect the FlexCard 'Policyholder360Card' filter component input map configuration. Correct the DataRaptor Extract input parameter binding for 'LOB_Filter' to reference the FlexCard filter component's output property '%LOBFilter%'. Update the DataRaptor Extract SOQL WHERE clause to include 'AND Coverage_Period__c.LOB__c = :LOB_Filter' with null-safe handling. Deploy fix to FS1 and validate with QA regression suite covering all four LOB values. Update DataRaptor version and re-activate FlexCard.

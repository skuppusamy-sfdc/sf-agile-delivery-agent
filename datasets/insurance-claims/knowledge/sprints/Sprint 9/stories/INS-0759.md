## Description

As a Claims Operations Manager, I need a payment disbursement summary panel embedded on the Policyholder 360 page that aggregates Payment__c records and associated Explanation_of_Benefits__c (EOB) documents by claim so that I can quickly verify total paid amounts, outstanding balances, and ERA reconciliation status without running separate reports.

## Acceptance Criteria

Given a Policyholder record page is open and the Payment Summary panel is loaded,
When the panel renders,
Then it must display an aggregate view of all Payment__c records linked to the policyholder's Claim__c records, grouped by Claim__c.Name, showing: SUM(Payment__c.Amount__c) as 'Total Paid', SUM(Claim__c.Total_Billed_Amount__c) - SUM(Payment__c.Amount__c) as 'Outstanding Balance', and Payment__c.ERA_Status__c.

Given a Payment__c.ERA_Status__c = 'Reconciled',
When the panel row renders,
Then a green checkmark icon must appear in the ERA Status column.

Given a Payment__c.ERA_Status__c = 'Pending' for more than 30 days (Payment__c.Payment_Date__c < TODAY - 30),
When the panel renders,
Then that row must be highlighted in amber and flagged with 'ERA Overdue'.

Given an EOB document exists (Explanation_of_Benefits__c.Claim__c = Claim__c.Id),
When a user clicks the EOB icon on a payment row,
Then the associated EOB PDF (stored as ContentDocument linked to Explanation_of_Benefits__c) must open in a modal preview without navigating away from the page.

Given the panel is loaded,
When there are no Payment__c records for the policyholder,
Then an empty state message 'No disbursements recorded for this policyholder' must be displayed.

-Given a TPA is managing the claim, TPA remittance details must appear in the panel (deferred to Phase 3).-

## Solution

Develop LWC 'paymentDisbursementPanel' with Apex controller 'PaymentDisbursementController.cls' using AggregateResult SOQL queries grouping Payment__c by Claim__c. ERA overdue logic computed in Apex using Date.today() comparison. EOB modal built with lightning-overlay-library and ContentDocumentLink SOQL to retrieve file. Wire service used for reactive data refresh. Add component to Policyholder Lightning Record Page via App Builder below the Coverage History Timeline. Ensure CRUD/FLS checks on Payment__c and Explanation_of_Benefits__c objects.

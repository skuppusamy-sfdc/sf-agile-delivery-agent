## Description

As a Credentialing Specialist, I need the Salesforce Credentialing_Case__c record to automatically pull and sync provider credentialing data from CAQH ProView via API so that I can eliminate manual data re-entry, reduce credentialing cycle time, and ensure our provider data reflects the most current attestation status from the authoritative source.

## Acceptance Criteria

Given a Credentialing_Case__c record is created with Credentialing_Case__c.Status__c = 'Initiated' and a valid Credentialing_Case__c.Provider_NPI__c is populated,
When the Record-Triggered Flow fires on record creation,
Then a Platform Event (CAQH_Sync_Request__e) must be published with payload fields: NPI, Organization_ID, Request_Type = 'INITIAL_PULL'.

Given the CAQH_Sync_Request__e Platform Event is consumed by the CAQHIntegrationService Apex class,
When the CAQH ProView REST API call returns HTTP 200,
Then the following fields on Credentialing_Case__c must be updated: Credentialing_Case__c.CAQH_Provider_ID__c, Credentialing_Case__c.License_State__c, Credentialing_Case__c.License_Expiration_Date__c, Credentialing_Case__c.Board_Certification__c, Credentialing_Case__c.Malpractice_Coverage_Amount__c, and Credentialing_Case__c.CAQH_Last_Sync_DateTime__c = current datetime.

Given the CAQH API returns HTTP 404 (provider not found),
When the Platform Event is processed,
Then Credentialing_Case__c.CAQH_Sync_Status__c must be set to 'Provider Not Found in CAQH' and a Task must be created for the Credentialing Specialist with Subject = 'Manual CAQH Verification Required - NPI Not Found'.

Given a Credentialing_Case__c record has CAQH_Last_Sync_DateTime__c populated,
When more than 30 days have elapsed since the last sync,
Then a nightly Scheduled Flow must trigger a re-sync by publishing a new CAQH_Sync_Request__e with Request_Type = 'REFRESH'.

Given the CAQH API returns data where the provider's license expiration date is within 60 days,
When the sync completes,
Then Credentialing_Case__c.License_Alert__c must be set to true and an email alert must be sent to the assigned Credentialing Specialist.

-Given the sync response includes DEA schedule data, it must be stored on the Provider__c (Person Account) directly.- (Descoped — DEA data storage addressed in INS-0779)

## Solution

Implement a Record-Triggered Flow on Credentialing_Case__c (after save, on create where Status__c = 'Initiated') that publishes a CAQH_Sync_Request__e Platform Event. Build CAQHIntegrationService.cls as an Apex class implementing Queueable interface to consume the event asynchronously and make a Named Credential-authenticated callout to CAQH ProView REST API (endpoint configured in External Credential 'CAQH_ProView'). Parse JSON response using a CAQHResponseWrapper inner class. Use Database.update with allOrNone=false for field updates. Store API credentials in Named Credentials / External Credentials (no hardcoding). Build nightly re-sync via Scheduled Apex (CAQHResyncScheduler.cls). All callout logic wrapped in try-catch with error logging to Integration_Error_Log__c custom object.

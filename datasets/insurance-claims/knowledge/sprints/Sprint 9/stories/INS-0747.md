## Description

As a Claims Operations Manager, I need a unified Policyholder 360 FlexCard that surfaces active Policy__c records, recent Claim__c activity, and current EOB__c summaries in a single view so that I can make faster adjudication decisions without navigating across multiple objects and tabs.

## Acceptance Criteria

Given a Claims Operations Manager opens a Provider__c (Person Account) record that is linked to one or more Policy__c records,
When the Policyholder 360 FlexCard loads,
Then it must display Policy__c.Policy_Number__c, Policy__c.Effective_Date__c, Policy__c.Termination_Date__c, and Policy__c.LOB__c for each active policy.

Given the policyholder has Claims in any status,
When the Claims Summary section renders,
Then the FlexCard must list the 5 most recent Claim__c records sorted by Claim__c.Date_of_Service__c descending, showing Claim__c.Claim_Number__c, Claim__c.Status__c, and Claim__c.Total_Billed_Amount__c.

Given a Claim__c record has a related Explanation_of_Benefits__c,
When the EOB section renders,
Then EOB__c.Plan_Paid_Amount__c, EOB__c.Member_Responsibility__c, and EOB__c.EOB_Date__c must be displayed inline beneath the parent claim row.

Given a Policy__c.Status__c = 'Lapsed' or 'Terminated',
When the FlexCard renders that policy,
Then a red badge labeled 'INACTIVE' must appear adjacent to Policy__c.Policy_Number__c.

-Given a COB flag is present on the policy, display a secondary payer banner (deprecated — moved to INS-0751)-

Given no Claim__c records exist for the policyholder,
When the Claims Summary section renders,
Then an empty-state message 'No claims on file for this policyholder' must be displayed and no error must be thrown.

## Solution

Build an OmniStudio FlexCard ('Policyholder360Card') with a primary DataRaptor Extract sourcing Policy__c filtered by Policy__c.Insured_Contact__c = recordId. Add a child DataRaptor Extract for Claim__c with a SOQL filter on Claim__c.Policyholder__c and ORDER BY Date_of_Service__c DESC LIMIT 5. Nest a second child DataRaptor for EOB__c joined on EOB__c.Claim__c. Apply conditional state styling (red badge) via FlexCard State configuration when Policy__c.Status__c IN ('Lapsed','Terminated'). Deploy the FlexCard to the Person Account Lightning page via App Builder. Assign visibility via Permission Set Group 'Claims_Operations_PSG'.

## Description

As a Claims Operations Manager, I need a consolidated Policyholder 360 FlexCard that surfaces active Policy__c records, recent Claim__c activity, and COB flags so that I can assess a member's full coverage picture without navigating multiple objects during adjudication.

## Acceptance Criteria

Given a Claims Operations Manager opens a Provider__c (Person Account) record,
When the Policyholder 360 FlexCard loads,
Then all Policy__c records where Policy__c.Policyholder__c = current Person Account AND Policy__c.Status__c = 'Active' are displayed with fields: Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Termination_Date__c.

Given the FlexCard is rendered,
When the Claims Summary section loads,
Then the 5 most recent Claim__c records are displayed ordered by Claim__c.FNOL_Date__c DESC, showing Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, and Claim__c.Total_Paid_Amount__c.

Given a Policy__c has COB coordination active (Policy__c.COB_Flag__c = true),
When the coverage panel renders,
Then a prominent 'COB Active' badge is displayed and the secondary payer name from Policy__c.Secondary_Payer__c is shown inline.

-Given the FlexCard loads, a static PDF benefit summary link is displayed- (superseded by INS-0698 dynamic document generation)

Given the user does not have the 'Claims_Adjudicator' Permission Set Group assigned,
When they access the FlexCard,
Then the Total_Paid_Amount__c field is masked and displays '***' in place of the value.

Given any Policy__c.Termination_Date__c is within 30 days of today,
When the FlexCard renders,
Then a yellow warning banner displays: 'Coverage expiring soon – verify renewal status.'

## Solution

Build an OmniStudio FlexCard named 'Policyholder_360_FC' with a primary DataRaptor Extract sourcing Policy__c and a child DataRaptor for Claim__c using a SOQL datasource filtered by Policyholder__c lookup. Use a FlexCard conditional state for COB badge rendering driven by Policy__c.COB_Flag__c. Implement a custom Apex @AuraEnabled method 'getPolicyholderCOBDetails' to handle multi-policy COB aggregation where OmniStudio native queries are insufficient. Field-level masking enforced via a custom Lightning Web Component wrapper that checks FeatureManagement.checkPermission('View_Payment_Amounts'). Deploy FlexCard via OmniStudio Migration Tool to production.

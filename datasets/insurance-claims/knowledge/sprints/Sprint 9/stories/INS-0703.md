## Description

As a Claims Operations Manager, I need a Benefit Utilization component on the Policy__c record page that displays real-time deductible met, out-of-pocket maximum progress, and individual accumulator balances so that I can accurately assess member cost-sharing status during claim adjudication without exiting Salesforce.

This component supersedes the deprecated Lifetime Benefit progress bar originally scoped in INS-0699 and provides a more comprehensive accumulator view sourced from the Benefit_Accumulator__c object.

## Acceptance Criteria

Given a Claims Operations Manager views a Policy__c record,
When the Benefit Utilization LWC loads,
Then it displays three progress bars: Individual Deductible (Benefit_Accumulator__c.Individual_Deductible_Met__c / Policy__c.Individual_Deductible__c), Family Deductible (Benefit_Accumulator__c.Family_Deductible_Met__c / Policy__c.Family_Deductible__c), and Out-of-Pocket Max (Benefit_Accumulator__c.OOP_Met__c / Policy__c.OOP_Max__c).

Given an accumulator where the met amount equals or exceeds the policy limit,
When the progress bar renders,
Then the bar fills to 100% in green and a 'Met' badge is displayed next to the label.

Given Policy__c.Benefit_Period__c = 'Calendar Year',
When the component loads,
Then a 'Benefit Year' label displays 'Jan 1 [YYYY] – Dec 31 [YYYY]' using the current year.

Given Policy__c.Plan_Type__c = 'HDHP',
When the component loads,
Then an additional 'HSA Eligible' indicator is shown with Policy__c.HSA_Contribution_Limit__c displayed.

Given Benefit_Accumulator__c.Last_Updated__c is more than 24 hours ago,
When the component loads,
Then a warning banner displays: 'Accumulator data may be stale. Last synced: [Benefit_Accumulator__c.Last_Updated__c]'.

-Given Policy__c.COB_Indicator__c = true, display a combined primary and secondary accumulator view — deferred to Phase 3 COB enhancement sprint-

## Solution

Develop LWC 'benefitUtilizationPanel' with reactive wire adapter calling Apex controller 'BenefitAccumulatorController.getAccumulators(policyId)'. Apex queries Benefit_Accumulator__c with Policy__c lookup and returns a wrapper class with computed percentage fields. Progress bars implemented as custom LWC sub-components using CSS custom properties for dynamic width. Stale data warning uses a JavaScript Date comparison in the connectedCallback lifecycle hook against Benefit_Accumulator__c.Last_Updated__c. HDHP and HSA logic driven by conditional rendering directives (lwc:if). Place component on Policy__c Lightning Record Page below the Policyholder 360 FlexCard in a dedicated 'Benefit Utilization' section.

## Description

As a Claims Operations Manager, I need an Agentforce AI copilot embedded in the Claim__c intake queue that can automatically analyze newly submitted FNOL records, suggest triage priority levels, flag potential SIU referral indicators, and recommend the appropriate adjuster assignment based on claim type and LOB so that we reduce manual triage time from an average of 45 minutes per claim to under 10 minutes.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Status__c = 'FNOL Received',
When the Agentforce Copilot action is triggered (automatically on creation or manually via the 'Analyze Claim' button),
Then the copilot must return a recommended Claim__c.Triage_Priority__c value of 'Low', 'Medium', 'High', or 'Critical' with a confidence score >= 0.75 stored in Claim__c.AI_Triage_Confidence__c.

Given the copilot analyzes a Claim__c record,
When SIU indicator patterns are detected (e.g., Claim__c.Date_of_Loss__c is within 30 days of Policy__c.Effective_Date__c, or Claim__c.Claimed_Amount__c > 3x the policy's historical average),
Then Claim__c.SIU_Referral_Flag__c must be set to true and a Task must be created with Subject = 'SIU Review Required - AI Flagged' assigned to the SIU queue.

Given the copilot recommends an adjuster assignment,
When the recommendation is generated,
Then it must consider Claim__c.LOB__c, Claim__c.Claim_Type__c, and the adjuster's current open claim load (from User.Open_Claim_Count__c) and return the top 3 adjuster recommendations ranked by suitability score in the Copilot response panel.

Given a Claims Operations Manager reviews the copilot's recommendations,
When the manager accepts a recommended Triage Priority,
Then Claim__c.Triage_Priority__c must be updated, Claim__c.Triage_Method__c must be set to 'AI-Assisted', and Claim__c.Triage_Accepted_By__c must be stamped with the current user Id.

Given a manager overrides the AI recommendation,
When the override is saved,
Then Claim__c.AI_Override_Reason__c (text field) must be required and the override event must be logged to an AI_Audit_Log__c record for compliance tracking.

## Solution

Configure an Agentforce Agent ('Claims Triage Copilot') using the Agentforce Agent Builder in Setup. Define custom Agent Actions backed by Apex Invocable Actions: 'AnalyzeClaimForTriage' (calls Einstein Prediction Service model trained on historical Claim__c data), 'CheckSIUIndicators' (rule-based Apex logic), and 'RecommendAdjuster' (queries User records with sharing). Embed the Agentforce Copilot component on the Claim__c Lightning Record Page via Lightning App Builder. Use a Record-Triggered Flow to auto-invoke the copilot on FNOL creation. Store all AI recommendations and outcomes in AI_Audit_Log__c for compliance. Einstein Prediction Service model trained on 24 months of anonymized Claim__c data from UAT org. Data masking applied to all PII fields before model training per Acme data governance policy.

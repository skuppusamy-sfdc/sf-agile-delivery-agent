## Description

As a Provider Relations Specialist, I need a Provider 360 page on the Provider__c (Person Account) record that surfaces credentialing status, active Provider Network Agreements (PNAs), associated Fee Schedules, and network affiliation history so that I can manage provider relationships from a single unified view. Extends INS-0031 (Provider Account data model setup).

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c Person Account record,
When the Provider 360 OmniScript-driven page loads,
Then the header FlexCard displays: Account.NPI__c, Account.TIN__c, Account.Primary_Specialty__c, Account.CAQH_ID__c, and Account.Provider_Type__c.

Given the Credentialing section of the Provider 360 page,
When a Credentialing_Case__c record exists for the provider,
Then the most recent Credentialing_Case__c is shown with fields: Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, Credentialing_Case__c.Credentialing_Body__c, and a color-coded status indicator (Green = 'Active', Yellow = 'Expiring Within 90 Days', Red = 'Expired' or 'Suspended').

Given the PNA section of the Provider 360 page,
When the provider has associated Provider_Network_Agreement__c records,
Then all PNAs where Provider_Network_Agreement__c.Status__c = 'Active' and Provider_Network_Agreement__c.Provider__c = current Account Id are listed with: Provider_Network_Agreement__c.Network_Name__c, Provider_Network_Agreement__c.Effective_Date__c, Provider_Network_Agreement__c.Termination_Date__c, and Provider_Network_Agreement__c.Contract_Type__c.

Given a PNA has an associated Fee_Schedule__c,
When the user expands a PNA row,
Then Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.LOB__c are displayed inline.

Given the provider has a Network_Termination_Case__c record,
When Network_Termination_Case__c.Status__c != 'Closed',
Then a prominent warning banner is displayed on the Provider 360 page: 'Active Termination Case in Progress — Case #[Network_Termination_Case__c.Case_Number__c]'.

-Then the provider's full claims payment history is shown in the Provider 360 view- (deprecated — claims payment history moved to separate Claims tab per stakeholder review 2024-11-15)

Given the user role is 'Provider_Network_Specialist',
When they attempt to edit PNA records directly from the 360 page,
Then inline editing is enabled for Provider_Network_Agreement__c.Status__c and Provider_Network_Agreement__c.Termination_Date__c fields only.

## Solution

Implement OmniStudio FlexCard 'Provider360_FC' as the primary container with nested child FlexCards for Credentialing, PNA, and Fee Schedule sections. DataRaptors: DR_GetProviderHeader (SOQL on Account), DR_GetCredentialingStatus (Credentialing_Case__c with ORDER BY CreatedDate DESC LIMIT 1), DR_GetActivePNAs (Provider_Network_Agreement__c with child Fee_Schedule__c), DR_GetTerminationWarning. Use FlexCard State conditions for color-coded credentialing badge. Inline edit on PNA fields via FlexCard action calling a Record-Triggered Flow update. Deploy to Account Lightning Record Page scoped to Provider__c record type. Add custom Permission Set 'Provider360_Edit_PSG' for inline edit access.

## Description

As a Provider Relations Specialist, I need a Provider 360 page layout on the Provider__c (Person Account) record that consolidates network agreement status, fee schedule tier, credentialing case history, and NPI/TIN verification indicators so that I can manage provider relationships from a single screen without switching between multiple related lists.

This story covers individual providers (Person Account). Facility/Group accounts are addressed in INS-0103.

## Acceptance Criteria

Given a Provider__c Person Account record is open,
When the Provider Relations Specialist views the Provider 360 page,
Then the following components are visible:
  - Network Agreement Summary: Provider_Network_Agreement__c.PNA_Number__c, Provider_Network_Agreement__c.Effective_Date__c, Provider_Network_Agreement__c.Termination_Date__c, Provider_Network_Agreement__c.Network_Status__c (values: 'Active', 'Pending', 'Terminated')
  - Fee Schedule Tier: Fee_Schedule__c.Tier__c and Fee_Schedule__c.Effective_Date__c pulled from the most recent active Fee_Schedule__c linked to the PNA
  - Credentialing Summary: count of open Credentialing_Case__c records where Credentialing_Case__c.Status__c != 'Closed', displayed as a badge
  - NPI Verification Indicator: green checkmark if Provider__c.NPI_Verified__c = TRUE and Provider__c.NPI_Verification_Date__c is within 365 days; red warning icon otherwise
  - TIN Status: displays Provider__c.TIN_Status__c field value with color coding (green = 'Verified', yellow = 'Pending', red = 'Failed')

Given a Provider__c record has multiple PNA records,
When the Network Agreement Summary panel loads,
Then only the PNA with the most recent Provider_Network_Agreement__c.Effective_Date__c and Network_Status__c = 'Active' is shown in the summary; all others are accessible via a 'View All Agreements' link.

Given a provider's NPI has not been verified via the NPPES integration,
When the Provider 360 page loads,
Then a banner message reads: 'NPI unverified — NPPES sync required. Last attempted: [Provider__c.NPI_Last_Sync_Attempt__c]'.

Given the Credentialing Summary badge shows a count > 0,
When the specialist clicks the badge,
Then a modal opens listing open Credentialing_Case__c records with fields: Case Number, Credentialing_Case__c.Type__c, Credentialing_Case__c.Assigned_To__c, Credentialing_Case__c.Due_Date__c.

-Given CAQH integration is complete, display CAQH profile status on the Provider 360 header- (deprecated — CAQH widget deferred to Phase 2 per architecture review 2024-11-14)

## Solution

Construct the Provider 360 experience using OmniStudio FlexCard 'Provider360_PersonAccount_FC' embedded in the Person Account Lightning Record Page. Use DataRaptors: DR_Get_Active_PNA (SOQL on Provider_Network_Agreement__c filtered by Account lookup and Network_Status__c = 'Active', ORDER BY Effective_Date__c DESC LIMIT 1), DR_Get_Fee_Schedule_Tier (join to Fee_Schedule__c via PNA Id), DR_Get_Credentialing_Summary (COUNT query on Credentialing_Case__c), DR_Get_NPI_TIN_Status (fields from Provider__c itself). NPI verification badge logic implemented as a DataRaptor Transform with date arithmetic. The 'View All Agreements' link invokes a child FlexCard flyout. Modal for credentialing cases implemented as an OmniScript embedded in a FlexCard action. Assign visibility via a custom Permission Set 'Provider_Relations_Read' included in the 'Provider_Network_PSG' Permission Set Group.

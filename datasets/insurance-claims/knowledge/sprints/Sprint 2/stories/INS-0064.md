## Description

**Bug Report — Provider 360 Credentialing Section Null Pointer Error**

**Environment:** Acme Insurance Full Sandbox (FULL1)
**Discovered by:** Marcus Johnson during INS-0062 UAT on 2024-12-03

**Steps to Reproduce:**
1. Log in as a user with 'Provider_Network_Specialist' profile.
2. Navigate to any Provider__c (Person Account) record that has NO associated Credentialing_Case__c records (e.g., Account Id: 001XX000004TEST1 — newly onboarded provider 'Dr. Alan Pierce, NPI: 1234567890').
3. The Provider 360 FlexCard loads and attempts to render the Credentialing Status child card.
4. Observe the page.

**Expected Result:** The Credentialing Status section renders a graceful empty state message: 'No credentialing records found. Initiate credentialing via the New Credentialing Case button.'

**Actual Result:** The page displays a JavaScript console error: 'TypeError: Cannot read properties of null (reading Status__c)' and the entire Provider 360 FlexCard fails to render — all sections (PNA, Fee Schedule, Termination Warning) are also blank.

**Root Cause (Suspected):** DataRaptor Extract 'DR_GetCredentialingStatus' returns an empty response when no Credentialing_Case__c exists. The FlexCard JSON mapping assumes a non-null object and attempts to access nested field 'Status__c' without a null guard, causing the entire card to crash rather than just the credentialing section.

**Impact:** High — affects all newly onboarded providers who have not yet completed credentialing. Provider Relations team cannot use the 360 page for approximately 34 providers currently in the onboarding pipeline.

## Acceptance Criteria

As per Description. Additionally: Given the fix is applied, When a Provider__c record with no Credentialing_Case__c records is opened, Then the Provider 360 FlexCard loads fully with all other sections (PNA, Fee Schedule, Termination Warning) rendering correctly, and the Credentialing section displays the empty state message 'No credentialing records found.' without any JavaScript errors in the browser console.

## Solution

Fix DataRaptor Extract 'DR_GetCredentialingStatus' to return a default empty object structure with null-safe field values when no Credentialing_Case__c records are found, using a DRTransform Set action to populate default values. Add a null guard conditional in the Provider360_FC FlexCard credentialing child card using a FlexCard State condition: IF credentialingData == null OR credentialingData.length == 0 THEN show 'emptyState' card state. Add unit test in Apex test class 'ProviderCredentialingDR_Test' covering the no-records scenario. Regression test all Provider 360 FlexCard sections post-fix.

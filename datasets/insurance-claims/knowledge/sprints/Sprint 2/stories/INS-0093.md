## Description

As a Claims Operations Manager, I need a unified Policyholder 360 FlexCard on the Contact/Person Account record page so that I can quickly assess a policyholder's active coverage, open claims, recent payments, and EOB history without navigating across multiple objects.

## Acceptance Criteria

Given a Contact record where Contact.RecordType = 'Policyholder', when the Policyholder 360 FlexCard loads, then it must display all related Policy__c records where Policy__c.Status__c IN ('Active', 'Pending Renewal') grouped by Policy__c.Line_of_Business__c (LOB).

Given an active policyholder, when the Claims panel is rendered, then all Claim__c records where Claim__c.Policyholder__c = Contact.Id and Claim__c.Status__c != 'Closed' must appear with fields: Claim__c.Claim_Number__c, Claim__c.FNOL_Date__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, Claim__c.Total_Paid_Amount__c.

Given a policyholder with payment history, when the Payments panel is rendered, then Payment__c records linked via Payment__c.Claim__r.Policyholder__c must show the 5 most recent payments ordered by Payment__c.Payment_Date__c DESC, displaying Payment__c.Payment_Amount__c, Payment__c.Payment_Method__c, and Payment__c.ERA_Reference__c.

Given a policyholder with COB (Coordination of Benefits) flags, when Policy__c.COB_Flag__c = true, then a prominent warning banner with text 'COB Active – Verify Primary Coverage' must render at the top of the FlexCard.

-Given a TPA-managed policy, display TPA contact details inline on the FlexCard.- (Superseded by INS-0087 TPA Portal story)

Given any user without the 'Claims_Ops_Full_Access' Permission Set Group, when the FlexCard loads, then Payment__c.ERA_Reference__c and Policy__c.Premium_Amount__c fields must be masked/hidden per field-level security.

## Solution

Build OmniStudio FlexCard 'Policyholder360Card' with 5 child cards (Policy Summary, Open Claims, Recent Payments, EOB History, Alerts). Use DataRaptor Turbo Extract DRs for each data panel querying Policy__c, Claim__c, Payment__c, and Explanation_of_Benefits__c with appropriate SOQL filters. Implement conditional state on FlexCard for COB banner using FlexCard State conditions on Policy__c.COB_Flag__c. Apply FLS enforcement via OmniStudio Data Tools respecting running user's Permission Set Group. Deploy FlexCard to the Policyholder Lightning Record Page via App Builder. Unit test DataRaptors with mock policyholder data covering edge cases (no claims, lapsed policy).

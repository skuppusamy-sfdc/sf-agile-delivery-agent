## Description

As a Claims Operations Manager, I need an EOB History tab within the Policyholder 360 view that lists all Explanation_of_Benefits__c records for the member and allows drill-down to individual Claim_Line__c adjudication details so that I can answer member billing inquiries and audit adjudication accuracy without pulling paper EOBs.

Extends INS-0101 (Policyholder 360 FlexCard) by adding a new tab to the existing FlexCard tabbed navigation.

## Acceptance Criteria

Given the Policyholder 360 FlexCard is loaded on a Contact record,
When the user clicks the 'EOB History' tab,
Then a list of Explanation_of_Benefits__c records is displayed ordered by EOB__c.EOB_Date__c DESC, showing: EOB__c.EOB_Number__c, EOB__c.Claim__c (hyperlink), EOB__c.Total_Billed__c, EOB__c.Total_Allowed__c, EOB__c.Member_Responsibility__c, EOB__c.EOB_Date__c.

Given an EOB record is listed,
When the user clicks the EOB__c.EOB_Number__c link,
Then a flyout panel opens showing all associated Claim_Line__c records with: Claim_Line__c.Line_Number__c, Claim_Line__c.CPT_Code__c, Claim_Line__c.ICD10_Primary__c, Claim_Line__c.Billed_Amount__c, Claim_Line__c.Allowed_Amount__c, Claim_Line__c.Denial_Reason__c, Claim_Line__c.Adjudication_Status__c.

Given a Claim_Line__c has Adjudication_Status__c = 'Denied',
When the claim line row renders in the flyout,
Then the row is displayed with a red left border and Claim_Line__c.Denial_Reason__c is shown in bold.

Given a member has more than 20 EOB records,
When the EOB History tab loads,
Then results are paginated at 10 records per page with Previous/Next navigation controls.

-Given ERA integration is active, display ERA transaction ID alongside each EOB- (deprecated — ERA linkage deferred; ERA integration not in scope for Phase 1)

## Solution

Extend the 'Policyholder360_FC' FlexCard (INS-0101) by adding a new Tab state for 'EOB History'. Add DataRaptor DR_Get_EOB_History (SOQL on Explanation_of_Benefits__c WHERE Contact__c = :recordId ORDER BY EOB_Date__c DESC, with offset/limit parameters for pagination). Implement pagination state variables (pageNumber, pageSize=10) within the FlexCard. Drill-down flyout implemented as a child FlexCard 'EOB_LineDetail_FC' invoked via a FlexCard action, with its own DataRaptor DR_Get_ClaimLines_By_EOB. Conditional row styling for denied lines uses FlexCard CSS class conditions on Claim_Line__c.Adjudication_Status__c. All new DataRaptors registered in the existing OmniStudio deployment package.

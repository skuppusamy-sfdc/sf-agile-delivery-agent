## Description

**Bug Report — Provider 360 CAQH Credentialing Status Badge**

**Environment:** Acme Insurance Full Sandbox (FS1)
**Found During:** UAT Sprint 2, Provider 360 review session

**Steps to Reproduce:**
1. Navigate to a Provider (Person Account) record where the CAQH profile status was recently updated from 'Pending' to 'Complete' in the CAQH ProView system.
2. Trigger the CAQH sync integration (scheduled job 'CAQH_Profile_Sync_Batch' or manual run via Developer Console).
3. Confirm the Credentialing_Case__c.CAQH_Profile_Status__c field has been updated to 'Complete' via SOQL in Workbench.
4. Navigate to the Provider 360 FlexCard and observe the 'Credentialing Status' panel.

**Expected Behavior:**
The credentialing status badge on the Provider 360 FlexCard should display 'Complete' (green badge) immediately after the Credentialing_Case__c.CAQH_Profile_Status__c field is updated by the integration batch job.

**Actual Behavior:**
The badge continues to display the stale 'Pending' value. The FlexCard DataRaptor cache is not invalidated on field update. A manual browser page refresh does NOT resolve the issue — the DataRaptor Extract appears to be serving a server-side cached response. The correct value only appears after approximately 4 hours, suggesting a platform cache TTL issue.

**Impact:** Provider Relations Specialists are making credentialing decisions based on stale data, causing delays in provider onboarding and network participation approvals.

**Workaround:** None currently available other than waiting for cache expiry.

## Acceptance Criteria

As per Description.

Additional verification: After CAQH_Profile_Sync_Batch completes and updates Credentialing_Case__c.CAQH_Profile_Status__c, the Provider 360 FlexCard 'Credentialing Status' panel must reflect the updated value within 60 seconds without requiring a manual page reload.

## Solution

Root cause: DataRaptor Extract 'DR_Extract_CredentialingStatus' has Platform Cache enabled with a 4-hour TTL set in the DataRaptor configuration. Fix: Disable Platform Cache on this specific DataRaptor Extract OR reduce TTL to 0 for real-time accuracy. Additionally, implement a Record-Triggered Flow on Credentialing_Case__c that fires After Update when CAQH_Profile_Status__c is changed, publishing a Platform Event 'Credentialing_Status_Changed__e' with the Provider Account Id. The Provider 360 FlexCard will subscribe to this Platform Event via an Emp API subscription to trigger a client-side refresh of the CredentialingStatus panel without full page reload. Update DataRaptor 'DR_Extract_CredentialingStatus' cache setting in OmniStudio configuration.

## Description

As a Claims Operations Manager, I need a COB (Coordination of Benefits) Alerts panel added to the existing Policyholder 360 FlexCard (INS-0085) so that I can immediately identify policyholders with dual coverage and apply correct primary/secondary payer sequencing during claims adjudication.

This story was carved out from INS-0085 during sprint planning. The COB panel must surface active Coordination_of_Benefits__c records, identify the primary and secondary payer, and flag any COB records where verification is overdue.

## Acceptance Criteria

Given a Claims Operations Manager views a Policyholder 360 FlexCard,
When the 'COB Alerts' panel loads,
Then it displays all Coordination_of_Benefits__c records where COB__c.Policyholder__c = current Contact Id AND COB__c.Status__c = 'Active', showing COB__c.Other_Payer_Name__c, COB__c.Payer_Sequence__c, COB__c.Verification_Date__c, COB__c.Coverage_Type__c.

Given a COB record has not been verified in the past 365 days (COB__c.Verification_Date__c < TODAY - 365),
When the panel renders,
Then that COB row is highlighted in amber with the label 'Verification Overdue'.

Given no active COB records exist for the policyholder,
When the panel loads,
Then the panel displays the message 'No active COB records on file.' with no error state.

Given the user clicks 'Initiate COB Verification' action on a COB row,
When the action fires,
Then an OmniScript 'COB_Verification_OS' launches in a modal, pre-populated with COB__c.Id and Policyholder Contact Id.

## Solution

Extend the existing 'Policyholder_360_Card' FlexCard (built in INS-0085) by adding a new child card panel 'COBAlerts'. DataRaptor Extract queries Coordination_of_Benefits__c with filter on Policyholder__c and Status__c. Conditional state logic applies amber highlight class when Verification_Date__c delta > 365 days using a FlexCard formula state. 'Initiate COB Verification' row action invokes OmniScript 'COB_Verification_OS' via FlexCard action of type OmniScript. OmniScript pre-populates via input map from FlexCard row context. No new DataRaptor required beyond the extract.

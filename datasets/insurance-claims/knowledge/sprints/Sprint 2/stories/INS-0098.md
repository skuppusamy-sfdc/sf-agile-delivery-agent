# INS-0098: Policyholder 360 – display COB (Coordination of Benefits) secondary payer details and priority order

## Description

As a Claims Operations Manager, I need the Policyholder 360 view to display full Coordination of Benefits (COB) information — including secondary payer details, payer priority order, and COB-impacted claim lines — so that adjusters can correctly apply COB rules during adjudication and avoid duplicate payment errors.

## Acceptance Criteria

Given a Policy__c record where Policy__c.COB_Flag__c = TRUE, when the COB panel on the Policyholder 360 FlexCard renders, then it must display all related COB_Coverage__c records with fields: COB_Coverage__c.Payer_Name__c, COB_Coverage__c.Payer_Priority__c (Primary/Secondary/Tertiary), COB_Coverage__c.Policy_Number__c, COB_Coverage__c.Effective_Date__c, COB_Coverage__c.Group_Number__c.

Given multiple COB coverages exist, when the COB panel renders, then records must be sorted ascending by COB_Coverage__c.Payer_Priority_Order__c (integer field, 1 = primary).

Given a Claim__c where Claim__c.COB_Applied__c = TRUE, when the Claims panel renders, then Claim__c records must show Claim__c.COB_Adjustment_Amount__c alongside Claim__c.Total_Paid_Amount__c with a tooltip explaining the COB reduction.

Given a user with the 'Claims_COB_Viewer' Permission Set, when they access the COB panel, then all COB fields must be fully visible. Given a user without this Permission Set, then COB_Coverage__c.Policy_Number__c and COB_Coverage__c.Group_Number__c must be masked.

Given Policy__c.COB_Flag__c = FALSE or no COB_Coverage__c records exist, when the FlexCard loads, then the COB panel must be hidden (not rendered) rather than showing an empty section.

## Solution

Extend the Policyholder360Card OmniStudio FlexCard (INS-0093) with a new COB child card. Create DataRaptor Turbo Extract DR_GetCOBCoverages querying COB_Coverage__c filtered by Policy__c.Id with ORDER BY Payer_Priority_Order__c ASC. Add FlexCard conditional visibility on COB panel using State condition: data.cobCoverages.length > 0 AND data.policy.COB_Flag__c = true. Implement field masking for Policy_Number__c and Group_Number__c using FlexCard conditional rendering tied to a custom permission check via a DataRaptor that queries PermissionSetAssignment for running user. Add tooltip LWC component for COB adjustment explanation on Claims panel. Update DR_GetOpenClaims DataRaptor to include Claim__c.COB_Applied__c and Claim__c.COB_Adjustment_Amount__c fields.

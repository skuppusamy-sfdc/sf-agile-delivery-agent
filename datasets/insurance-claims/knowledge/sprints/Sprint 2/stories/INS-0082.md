## Description

As a Credentialing Specialist, I need the Provider 360 page to automatically flag providers whose CAQH credentials are expiring within 90 days so that I can proactively initiate re-credentialing workflows and prevent network gaps. The flag should also trigger an automated task assignment to the responsible Credentialing Specialist. Extends INS-0078 credentialing status component.

## Acceptance Criteria

Given a Credentialing_Case__c record exists where Credentialing_Case__c.Expiration_Date__c <= TODAY() + 90 AND Credentialing_Case__c.Status__c = 'Active',
When a scheduled batch job runs nightly at 02:00 UTC,
Then Credentialing_Case__c.Credential_Alert_Flag__c is set to TRUE and Credentialing_Case__c.Alert_Triggered_Date__c is stamped with the current date.

Given Credentialing_Case__c.Credential_Alert_Flag__c = TRUE,
When the Provider 360 page loads (INS-0078),
Then the credentialing status indicator renders in amber (warning) for 31–90 days remaining and red (critical) for 0–30 days remaining.

Given the alert flag is set,
When the batch job completes,
Then a Task is created (Task.Subject = 'Re-credentialing Required: [Provider Name]', Task.ActivityDate = TODAY() + 30, Task.OwnerId = Credentialing_Case__c.Assigned_Credentialing_Specialist__c) and a Salesforce Notification is sent to the assigned specialist.

Given a provider's Credentialing_Case__c.Expiration_Date__c has passed (< TODAY()),
When the Provider 360 page loads,
Then the status indicator renders in red with label 'EXPIRED' and Account.Network_Status__c is automatically updated to 'Credentialing Hold'.

Given the CAQH integration is active,
When Credentialing_Case__c.CAQH_Profile_ID__c is populated,
Then a 'Refresh from CAQH' button on the credentialing component triggers a callout to the CAQH ProView API and updates Credentialing_Case__c.Expiration_Date__c with the returned attestation date.

## Solution

Implement a Schedulable Apex batch class 'CredentialExpiryBatch' running nightly via Scheduled Job, querying Credentialing_Case__c with date arithmetic and bulk-updating Credential_Alert_Flag__c and Alert_Triggered_Date__c. Task creation handled within the same batch using Database.insert with allOrNone=false. Custom Notification sent via Messaging.CustomNotification API. Provider 360 LWC 'providerCredentialingStatus' (INS-0078) updated to read Credential_Alert_Flag__c and days-remaining calculation for color-coded indicator. CAQH refresh button triggers a Named Credential-based callout via Apex @AuraEnabled method. Record-Triggered Flow on Credentialing_Case__c handles the Account.Network_Status__c update to 'Credentialing Hold' on expiry.

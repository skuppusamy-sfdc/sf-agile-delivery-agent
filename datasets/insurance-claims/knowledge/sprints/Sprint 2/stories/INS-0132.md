## Description

As a Claims Operations Manager, I need a unified Policyholder 360 FlexCard on the Contact/Person Account record page so that I can view a policyholder's active policies, open claims, recent EOBs, and payment history in a single consolidated view without navigating across multiple objects.

This card should surface data from Policy__c, Claim__c, Payment__c, and Explanation_of_Benefits__c and present it in collapsible sections. Extends INS-0102 (Policyholder data model setup).

## Acceptance Criteria

Given a Claims Operations Manager opens a Person Account record tagged as a policyholder,
When the Policyholder 360 FlexCard loads,
Then the 'Active Policies' section displays all Policy__c records where Policy__c.Status__c = 'Active' AND Policy__c.PolicyHolder__c = current Account Id, showing fields: Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c.

Given the policyholder has one or more Claim__c records,
When the 'Open Claims' section renders,
Then only claims where Claim__c.Status__c IN ('FNOL Received', 'Under Review', 'Pending Information', 'Adjudicated - Pending Payment') are shown, sorted by Claim__c.Date_of_Service__c descending, displaying Claim__c.Claim_Number__c, Claim__c.Total_Billed_Amount__c, and Claim__c.Assigned_Adjuster__c.

Given the policyholder has Payment__c records linked via Payment__c.Policyholder__c,
When the 'Payment History' section is expanded,
Then the last 12 months of payments are displayed showing Payment__c.Payment_Date__c, Payment__c.Amount__c, Payment__c.Payment_Method__c, and Payment__c.Status__c, with a running total labeled 'YTD Paid'.

Given an EOB has been generated for a claim,
When the 'Recent EOBs' section loads,
Then up to 5 Explanation_of_Benefits__c records are shown where EOB__c.Policyholder__c = current Account Id, displaying EOB__c.EOB_Number__c, EOB__c.Issue_Date__c, and a DocuSign envelope status badge where available.

-Given the policyholder has a TPA assignment, display TPA__c.Name in the policy header section-
(Deprecated: TPA display moved to Policy detail page per INS-0118)

Given the FlexCard is viewed by a user without the 'Claims_Ops_Full_Access' Permission Set Group,
When the card renders,
Then Payment__c.Amount__c and EOB financial fields are masked and a 'Request Access' link is shown.

Given no data exists for a section,
When the section is expanded,
Then an empty state message reads: 'No [section name] records found for this policyholder.' and a contextual Quick Action button is present (e.g., 'Create New Claim').

## Solution

Build using OmniStudio FlexCard with child FlexCards for each section (Policies, Claims, Payments, EOBs). Use DataRaptor Turbo Extract procedures to query each object with policyholder Account Id as input parameter. Implement field-level masking via FlexCard conditional visibility bound to a custom permission 'View_PH_Financials'. Deploy FlexCard to the Person Account Lightning Record Page via App Builder. Cache DataRaptor responses using OmniStudio cache keys with a 5-minute TTL to reduce SOQL load. Permission Set Group 'Claims_Ops_Full_Access' controls financial field visibility. Add a Platform Event listener (Claim_Status_Changed__e) to trigger FlexCard refresh when claim status updates.

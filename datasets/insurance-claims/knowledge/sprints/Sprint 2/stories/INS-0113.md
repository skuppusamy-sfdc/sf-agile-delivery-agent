## Description

As a Credentialing Specialist, I need a credentialing timeline component on the Provider 360 page that visually displays key milestones from the Credentialing_Case__c record — including CAQH attestation date, primary source verification completion, committee review date, and approval/denial decision — so that I can quickly assess where a provider is in the credentialing lifecycle without opening the full case record.

## Acceptance Criteria

Given a Provider__c record has one or more Credentialing_Case__c records,
When the Credentialing Timeline component loads,
Then the most recent Credentialing_Case__c (by CreatedDate DESC) must be selected and its milestone dates displayed: Credentialing_Case__c.CAQH_Attestation_Date__c, Credentialing_Case__c.PSV_Completion_Date__c, Credentialing_Case__c.Committee_Review_Date__c, Credentialing_Case__c.Decision_Date__c, and Credentialing_Case__c.Expiration_Date__c.

Given a milestone date field is populated,
When the timeline renders,
Then the milestone node must appear as a filled green circle; if the date is null and the previous milestone is complete, it must appear as an amber pending circle; if the case is in a terminal error state (Credentialing_Case__c.Status__c = 'Denied'), all subsequent nodes must appear as grey.

Given Credentialing_Case__c.Expiration_Date__c is within 90 days of TODAY,
When the timeline renders,
Then an amber 'Expiring Soon' badge must appear adjacent to the Expiration milestone node.

Given the user clicks a milestone node,
When the interaction fires,
Then a popover must display the milestone label, date value, and any associated Credentialing_Case__c.Notes__c text for that stage.

## Solution

Implement a custom LWC 'credentialingTimeline' using an SVG-based horizontal timeline rendered via @wire to an Apex controller 'CredentialingTimelineController.getTimelineData(accountId)'. Apex queries Credentialing_Case__c with all milestone date fields. Timeline node state (filled/pending/grey) computed in LWC JavaScript based on date null-checks and Status__c value. 'Expiring Soon' badge uses a computed property comparing Expiration_Date__c to today via JS Date arithmetic. Popover uses SLDS popover component triggered on node click. LWC deployed to the Provider Account Lightning Record Page alongside the Provider360Card FlexCard.

## Description

**Bug Report — Incorrect Credentialing Expiration Date on Provider 360**

**Steps to Reproduce:**
1. Identify a Provider__c record with two or more Credentialing_Case__c records (e.g., provider credentialed in 2022 and re-credentialed in 2024).
2. Navigate to the Provider 360 Lightning Record Page.
3. Observe the Credentialing Status panel — note the displayed Credentialing_Case__c.Expiration_Date__c.
4. Compare to the Credentialing_Case__c related list — confirm the displayed date does not match the most recent closed/active case.

**Expected Behavior:** The panel should display the Credentialing_Case__c record with the most recent Credentialing_Case__c.Closed_Date__c (i.e., the latest completed credentialing cycle), showing its Expiration_Date__c.

**Actual Behavior:** The DataRaptor Extract DR_GetCredentialingStatus retrieves the first Credentialing_Case__c record ordered by CreatedDate ASC (oldest first), displaying a stale expiration date from a prior credentialing cycle. This has caused a Provider Relations Specialist to incorrectly flag an in-network provider as expired.

**Environment:** Full Sandbox (ACME_FS1) and Production
**Affected Users:** Provider Relations Specialist, Credentialing Specialist
**Frequency:** 100% reproducible on providers with 2+ credentialing cases.
**Related Story:** INS-0125

## Acceptance Criteria

As per Description. Given a Provider__c with multiple Credentialing_Case__c records, when the Credentialing panel on Provider 360 loads, then the panel displays the Credentialing_Case__c record with the most recent Credentialing_Case__c.Closed_Date__c, and the displayed Credentialing_Case__c.Expiration_Date__c matches that record.

## Solution

Root cause: DataRaptor Extract DR_GetCredentialingStatus SOQL query is missing ORDER BY Closed_Date__c DESC LIMIT 1. Fix: Update the DR_GetCredentialingStatus DataRaptor Extract query to 'SELECT Id, Status__c, Expiration_Date__c, Credentialing_Body__c, Closed_Date__c FROM Credentialing_Case__c WHERE Provider__c = :providerId ORDER BY Closed_Date__c DESC NULLS LAST LIMIT 1'. Validate fix against providers with 1, 2, and 5+ credentialing cases in ACME_FS1. Hotfix to Production after UAT sign-off from Credentialing Specialist lead.

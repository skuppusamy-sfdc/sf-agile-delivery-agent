## Description

As a Provider Network Specialist, I need a Provider 360 page on the Provider__c (Person Account) record so that I can view a provider's current network participation, active Provider Network Agreements (PNAs), associated Fee Schedules, credentialing status from CAQH/NPPES, and any open Provider Change Notifications (PCNs) in a single unified view.

## Acceptance Criteria

Given a Provider__c record where Account.RecordType = 'Individual Provider', when the Provider 360 page loads, then the page header must display: Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Primary_Specialty__c, Provider__c.CAQH_Provider_ID__c, and Provider__c.Network_Status__c.

Given a provider with active contracting, when the PNA panel renders, then all Provider_Network_Agreement__c records where PNA__c.Provider__c = Provider__c.Id and PNA__c.Status__c = 'Active' must display with PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, PNA__c.Network_Tier__c, and a hyperlink to the associated Fee_Schedule__c record.

Given an active PNA, when a Fee Schedule is linked (PNA__c.Fee_Schedule__c != null), then Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.LOB__c must be visible in a nested expandable section.

Given a provider with credentialing history, when the Credentialing panel renders, then all Credentialing_Case__c records where Credentialing_Case__c.Provider__c = Provider__c.Id must display ordered by Credentialing_Case__c.CreatedDate DESC, showing Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and Credentialing_Case__c.CAQH_Attestation_Date__c.

Given a provider with open PCNs, when Provider_Change_Notification__c.Provider__c = Provider__c.Id and PCN__c.Status__c IN ('Open', 'Pending Review'), then an alert badge must appear on the Provider 360 page header with count of open PCNs.

Given a provider whose Credentialing_Case__c.Expiration_Date__c is within 90 days of today, then a yellow warning indicator 'Credential Expiring Soon' must display next to the credentialing record row.

## Solution

Build OmniStudio FlexCard 'Provider360Card' deployed on the Provider__c (Person Account) Lightning Record Page. Create 4 DataRaptor Turbo Extracts: DR_GetActivePNAs (joins Provider_Network_Agreement__c + Fee_Schedule__c), DR_GetCredentialingCases, DR_GetOpenPCNs, DR_GetProviderHeader. Use FlexCard States for alert badge logic (PCN count > 0) and credential expiry warning (date formula comparing today vs Credentialing_Case__c.Expiration_Date__c). Implement FlexCard flyout for Fee Schedule details. Ensure CAQH_Provider_ID__c and TIN__c fields are protected by FLS under 'Provider_Network_Read' Permission Set. Deploy via OmniStudio package to sandbox for UAT.

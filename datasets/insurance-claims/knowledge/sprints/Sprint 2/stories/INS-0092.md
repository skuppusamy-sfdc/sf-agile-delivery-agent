# INS-0092: Create Agentforce Claims Intake AI action to suggest claim type and LOB classification from FNOL free-text description

## Description

As a Claims Operations Manager, I need an Agentforce AI action embedded in the FNOL intake workflow so that when an adjuster enters a free-text loss description during First Notice of Loss, the system suggests the appropriate Claim__c.Claim_Type__c and Claim__c.LOB__c classification, reducing manual categorization errors and accelerating claim setup time.

The Agentforce action should analyze the FNOL description text, return the top 2 suggested Claim_Type__c values with confidence scores, and allow the adjuster to accept or override the suggestion before saving the Claim__c record. All AI suggestions and adjuster decisions (accept/override) must be logged for model feedback and compliance audit purposes.

## Acceptance Criteria

Given an adjuster is completing the FNOL OmniScript and has entered at least 20 characters in the Claim__c.Loss_Description__c field,
When they advance past the 'Incident Details' step,
Then the Agentforce AI action 'Classify_FNOL_Claim_Type' is invoked automatically and returns up to 2 suggested values for Claim__c.Claim_Type__c (e.g., 'Auto - Collision', 'Auto - Comprehensive') each with a confidence score between 0.00 and 1.00.

Given the Agentforce action returns suggestions,
When the suggestions panel renders in the OmniScript,
Then each suggestion displays: suggested Claim_Type__c label, confidence score as a percentage, and a brief rationale string (max 100 characters) generated by the model.

Given the adjuster accepts a suggestion,
When they click 'Accept',
Then Claim__c.Claim_Type__c and Claim__c.LOB__c are pre-populated with the accepted values and a FNOL_AI_Suggestion__c record is created with fields: Suggestion__c.Suggested_Value__c, Suggestion__c.Confidence_Score__c, Suggestion__c.Adjuster_Decision__c = 'Accepted', Suggestion__c.Claim__c = new Claim Id.

Given the adjuster overrides the suggestion and manually selects a different Claim_Type__c,
When they save the FNOL,
Then a FNOL_AI_Suggestion__c record is created with Suggestion__c.Adjuster_Decision__c = 'Overridden' and Suggestion__c.Override_Value__c = the manually selected value, for model retraining feedback.

Given the Agentforce service is unavailable or returns an error,
When the FNOL OmniScript reaches the classification step,
Then the AI suggestion panel is hidden and the adjuster proceeds with manual Claim_Type__c selection; an error is logged to Integration_Error_Log__c with Source__c = 'Agentforce_FNOL_Classifier'.

Given the confidence score of the top suggestion is below 0.60,
When the suggestion panel renders,
Then a disclaimer displays: 'Low confidence suggestion — please verify claim type manually before proceeding.'

## Solution

Configure an Agentforce Agent Action 'Classify_FNOL_Claim_Type' using a custom Apex class 'FNOLClaimTypeClassifierAction' that invokes the Einstein Generative AI API via a Connect API callout, passing Claim__c.Loss_Description__c as the prompt input along with a system prompt defining the valid Claim_Type__c and LOB__c picklist values. The Apex class implements the AgentAction interface and returns a structured response object with top 2 suggestions and confidence scores. The Agentforce action is registered in the Agent Builder and exposed as a callable step in the FNOL OmniScript 'FNOL_Intake_OS' via an OmniScript Agentforce Action element. FNOL_AI_Suggestion__c is a custom object created to log all suggestions; a Record-Triggered Flow on Claim__c (After Insert) creates the suggestion log record. Permission Set 'Agentforce_Claims_User' grants access to the action and custom object.

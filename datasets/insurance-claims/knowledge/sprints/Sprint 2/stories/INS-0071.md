## Description

As a Provider Relations Specialist, I need a Facility 360 page on the Facility__c (Business Account) record that displays Group Service Agreement details, affiliated individual providers, network participation status, and active fee schedules so that I can manage facility contracting and ensure accurate network directory information.

This story complements INS-0070 which handles Person Account (individual provider) views.

## Acceptance Criteria

Given a Facility__c Business Account record,
When a Provider Relations Specialist opens the account page,
Then the Facility 360 Lightning Page renders with sections: Facility Profile, Group Service Agreements, Affiliated Providers, and Network Participation Status.

Given the Facility Profile section,
When the page loads,
Then it displays Facility__c.NPI__c, Facility__c.TIN__c, Facility__c.Facility_Type__c, Facility__c.Accreditation_Body__c, Facility__c.Accreditation_Expiry__c, and Facility__c.CMS_Certification_Number__c.

Given the Group Service Agreements section,
When the page loads,
Then all Group_Service_Agreement__c records where GSA.Facility__c = Account.Id are listed with GSA.Agreement_Number__c, GSA.Status__c, GSA.Effective_Date__c, GSA.Expiration_Date__c, and GSA.LOB__c.
And GSAs with Status__c = 'Expired' are visually distinguished with a grey italic style.

Given the Affiliated Providers section,
When the page loads,
Then all Provider__c Person Account records with a lookup to this Facility__c are listed showing Provider__c.Name, Provider__c.Primary_Specialty__c, Provider__c.NPI__c, and their current Credentialing_Case__c.Status__c.

-Given the Network Participation Status section, When the page loads, Then it shows participation in all MCO networks using a hardcoded picklist.- (Deprecated — replaced with dynamic PNA-based network lookup per architecture review 2024-11-05)

Given the Network Participation Status section (revised),
When the page loads,
Then network participation is derived from active Provider_Network_Agreement__c records where PNA.Facility__c = Account.Id, displaying PNA.Network__c and PNA.Status__c.

## Solution

Create custom Lightning Record Page 'Facility_360_Page' assigned to Business Account record type 'Facility'. Build OmniStudio FlexCards: 'FacilityProfile_FC', 'GSASummary_FC', 'AffiliatedProviders_FC', 'FacilityNetworkStatus_FC'. Use DataRaptor Extracts with multi-object joins across Group_Service_Agreement__c, Provider__c, Credentialing_Case__c, and Provider_Network_Agreement__c. Apply conditional CSS classes via FlexCard State Actions for expired GSA styling. Use a child FlexCard with pagination for Affiliated Providers list (max 10 per page). Assign page via App Builder to 'Provider Relations Specialist' profile.

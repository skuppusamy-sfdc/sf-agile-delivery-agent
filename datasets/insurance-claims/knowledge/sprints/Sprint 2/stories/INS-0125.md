## Description

As a Provider Relations Specialist, I need a Provider 360 page on the Provider__c (Person Account) record so that I can instantly see a provider's network participation status, active PNAs, current credentialing case status, associated fee schedules, and any open Provider Change Notifications (PCNs) without switching between multiple tabs or apps.

## Acceptance Criteria

Given a Provider__c record, when the Provider 360 page loads, then a summary header FlexCard displays Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Specialty__c, Provider__c.Board_Certification__c, and Provider__c.CAQH_ID__c.

Given a provider with Provider_Network_Agreement__c records, when the Network Agreements panel renders, then all PNA__c records where PNA__c.Provider__c = current Provider Id and PNA__c.Status__c IN ('Active','Pending Signature') are displayed showing PNA__c.Network_Name__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, and PNA__c.Contract_Type__c.

Given a provider with a Credentialing_Case__c record, when the Credentialing panel loads, then the most recent Credentialing_Case__c where Credentialing_Case__c.Provider__c = current Provider Id is shown with Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and Credentialing_Case__c.Credentialing_Body__c.

Given a provider linked to Fee_Schedule__c records, when the Fee Schedule panel renders, then all Fee_Schedule__c records where Fee_Schedule__c.Provider__c = current Provider Id and Fee_Schedule__c.Active__c = true are listed with Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.LOB__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.Base_Rate__c.

Given a provider with open Provider_Change_Notification__c records, when the PCN panel loads, then all PCN__c records where PCN__c.Provider__c = current Provider Id and PCN__c.Status__c != 'Closed' are shown with PCN__c.Change_Type__c, PCN__c.Submitted_Date__c, and PCN__c.Assigned_To__c.

Given any panel fails to load due to a DataRaptor error, then a toast error message displays 'Unable to load [panel name] data. Please refresh or contact support.' and the error is logged to a custom object DataRaptor_Error_Log__c.

## Solution

Build OmniStudio FlexCard 'Provider360_FC' on the Account object (filtered to RecordType = 'Provider'). Implement five DataRaptor Extract actions: DR_GetProviderSummary, DR_GetActivePNAs, DR_GetCredentialingStatus, DR_GetActiveFeeSchedules, DR_GetOpenPCNs. Use FlexCard accordion panels for each section. Wire error handling via a DataRaptor Transform that writes to DataRaptor_Error_Log__c using a Platform Event. Assign the page layout via a Lightning App Builder Provider Record Page with conditional component visibility. Apply Field-Level Security on NPI__c and TIN__c fields to the 'Provider_Data_Viewer' Permission Set Group.

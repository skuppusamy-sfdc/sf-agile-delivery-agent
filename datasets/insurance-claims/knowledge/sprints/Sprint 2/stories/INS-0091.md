## Description

**Bug Report — Policyholder 360 Open Claims Panel Duplicate Records**

**Environment:** Acme Insurance Full Sandbox (FS1)
**Found During:** SIT Sprint 2, Policyholder 360 panel testing
**Related Story:** INS-0085

**Steps to Reproduce:**
1. Identify a Policyholder Person Account that has at least one Claim__c record with 3 or more associated Claim_Line__c child records.
2. Navigate to the Policyholder 360 FlexCard built in INS-0085.
3. Observe the 'Open Claims' panel.

**Expected Behavior:**
The Open Claims panel should display each Claim__c record exactly once, regardless of how many Claim_Line__c child records are associated with it.

**Actual Behavior:**
The Open Claims panel displays the same Claim__c record N times, where N equals the number of Claim_Line__c records linked to that claim. For example, a claim with 4 line items appears 4 times in the panel, causing significant confusion for Claims Operations staff.

**Root Cause (suspected):**
The DataRaptor Extract 'DR_Extract_OpenClaims' appears to be performing a JOIN-style query across Claim__c and Claim_Line__c without a DISTINCT or GROUP BY clause, returning one row per Claim_Line__c rather than one row per Claim__c.

**Impact:** High — Claims Operations Managers are reporting inflated open claim counts and are escalating to IT as a data integrity concern.

## Acceptance Criteria

As per Description.

Verification: For a Policyholder with 2 distinct Claim__c records (one with 4 Claim_Line__c records, one with 1 Claim_Line__c record), the Open Claims panel must display exactly 2 rows — one per Claim__c — with no duplicates.

## Solution

Fix the DataRaptor Turbo Extract 'DR_Extract_OpenClaims' in OmniStudio: remove the child Claim_Line__c sub-query from the primary extract (it was incorrectly added to support a CPT code tooltip that has since been removed). The extract should query Claim__c directly with filter Claim__c.Policyholder__c = :recordId AND Claim__c.Status__c IN ('FNOL Received','In Review','Pending Documentation') without any JOIN to Claim_Line__c. If Claim_Line__c data is needed in future panels, it should be fetched via a separate child DataRaptor Extract scoped to a selected Claim__c row. Deploy fix to sandbox, execute regression test on INS-0085 FlexCard, then promote to UAT.

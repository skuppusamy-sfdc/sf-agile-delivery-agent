## Description

As a Provider Relations Specialist, I need the Provider 360 page for facility-based providers (Facility__c Business Accounts) to display the provider's current scheduling availability and next 3 upcoming appointments pulled from QGenda, so that I can coordinate care coordination referrals and network adequacy assessments without switching to the QGenda portal.

## Acceptance Criteria

Given a Provider Relations Specialist views a Facility__c (Business Account with RecordType = 'Facility') record,
When the Provider 360 page loads and the facility has a valid Facility__c.QGenda_Resource_ID__c value,
Then a 'Scheduling Availability' section is displayed showing the provider's current availability status from QGenda: 'Available', 'Unavailable', or 'On Leave'.

Given the Scheduling Availability section is visible,
When the QGenda API call returns appointment data,
Then the next 3 upcoming appointments are displayed with: Appointment Date/Time, Appointment Type, and Facility__c.QGenda_Resource_ID__c as reference, sourced via QGenda REST API through Named Credential 'QGenda_NC'.

Given Facility__c.QGenda_Resource_ID__c is NULL or blank,
When the Provider 360 page loads,
Then the Scheduling section displays: 'QGenda integration not configured for this facility. Contact your system administrator.' and no API callout is made.

Given the QGenda API returns availability status = 'On Leave',
When the scheduling section renders,
Then an orange 'Provider On Leave' badge is shown and Facility__c.Accepting_New_Patients__c is displayed as 'No' regardless of its stored value, with a tooltip: 'Availability sourced from QGenda — may differ from stored record data.'

-Given the scheduling section, When the user clicks 'Book Appointment', Then an appointment booking OmniScript is launched pre-populated with provider details- (deprecated — appointment booking out of scope for Phase 1; deferred to Phase 2 Agent Portal per product roadmap decision 2024-11-28)

## Solution

Create LWC 'providerQGendaScheduling' embedded in the Facility__c Lightning Record Page via App Builder, visible only when RecordType = 'Facility'. LWC calls Apex controller 'QGendaSchedulingController.getProviderAvailability(facilityId)' which reads Facility__c.QGenda_Resource_ID__c and makes a @future(callout=true) or Continuation callout to QGenda REST API '/api/schedule/resource/{resourceId}/availability' via Named Credential 'QGenda_NC'. Parse response JSON into QGendaAvailabilityWrapper and return to LWC. Cache responses in Platform Cache (Org partition) with 5-minute TTL to reduce QGenda API call volume. Apply conditional rendering in LWC template for null QGenda_Resource_ID__c guard. Add custom metadata 'QGenda_Integration_Config__mdt' to store API endpoint, timeout values, and feature toggle 'Scheduling_Enabled__c' for environment-specific control.

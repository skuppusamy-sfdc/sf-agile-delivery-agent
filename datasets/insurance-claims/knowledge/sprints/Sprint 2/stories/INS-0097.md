## Description

As a Provider Network Specialist, I need Provider__c.Network_Status__c to automatically reflect the current state of a provider's Provider_Network_Agreement__c lifecycle so that the Provider 360 view always shows accurate network participation status without requiring manual updates, reducing compliance risk and adjudication errors.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA__c.Status__c is updated to 'Active' and PNA__c.Effective_Date__c <= TODAY, when the record is saved, then the parent Provider__c.Network_Status__c must be set to 'In-Network' within 5 seconds via automation.

Given a PNA__c.Status__c updated to 'Terminated' (via Network_Termination_Case__c workflow), when the record is saved and no other active PNA exists for that provider, then Provider__c.Network_Status__c must be set to 'Out-of-Network'.

Given a provider with multiple PNAs across different LOBs, when one PNA is terminated but at least one other PNA__c.Status__c = 'Active', then Provider__c.Network_Status__c must remain 'In-Network' and Provider__c.Partial_Network_Flag__c must be set to TRUE.

Given a PNA__c.Effective_Date__c is in the future (PNA__c.Status__c = 'Pending'), when PNA__c.Effective_Date__c = TODAY (evaluated at midnight batch), then PNA__c.Status__c must auto-transition to 'Active' and trigger the Network_Status__c update.

Given any Network_Status__c change on Provider__c, when the field value changes, then a Provider_Change_Notification__c record must be auto-created with PCN__c.Change_Type__c = 'Network Status Change', PCN__c.Previous_Value__c = old value, PCN__c.New_Value__c = new value, and PCN__c.Status__c = 'Open'.

-Given a provider flagged as TPA-delegated, bypass the Network_Status__c automation and defer to TPA feed.- (Removed per compliance review; TPA sync handled via DataIntegration pipeline)

## Solution

Implement Record-Triggered Flow on Provider_Network_Agreement__c object triggering on Create and Update when PNA__c.Status__c changes. Flow logic: (1) Query all sibling PNA records for same Provider__c, (2) evaluate aggregate active status, (3) use Update Records element to set Provider__c.Network_Status__c and Provider__c.Partial_Network_Flag__c. For midnight PNA activation, build a Scheduled Apex job (PNAActivationScheduler) running daily at 00:05 to query PNA__c where Status__c = 'Pending' and Effective_Date__c = TODAY, bulk-update Status__c to 'Active'. PCN auto-creation handled by a separate Record-Triggered Flow on Provider__c firing After Update on Network_Status__c field change, creating Provider_Change_Notification__c via Create Records element. Bulkify all Apex for governor limits (batch size 200).

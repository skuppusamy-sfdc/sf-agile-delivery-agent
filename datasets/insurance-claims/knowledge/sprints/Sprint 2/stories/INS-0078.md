## Description

As a Provider Network Specialist, I need a Provider 360 page on the Provider__c (Person Account) record so that I can quickly assess a provider's contracting status, active Provider Network Agreements (PNAs), applicable Fee Schedules, and current credentialing state from a single consolidated view, enabling faster network management decisions.

## Acceptance Criteria

Given an Account record with RecordType = 'Provider' (Person Account),
When the Provider Network Specialist navigates to the account,
Then the Provider 360 Lightning page loads and displays the provider's NPI (Account.NPI__c), TIN (Account.TIN__c), primary specialty (Account.Primary_Specialty__c), and network participation status (Account.Network_Status__c).

Given one or more Provider_Network_Agreement__c records exist,
When the PNA related list component renders,
Then only PNAs where PNA__c.Status__c = 'Active' AND PNA__c.Provider__c = Account.Id are shown, displaying PNA__c.Agreement_Number__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, and PNA__c.LOB__c.

Given a Fee_Schedule__c is linked to an active PNA,
When the provider clicks a PNA row,
Then a side panel opens showing Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, and a summary count of associated CPT codes.

Given a Credentialing_Case__c record exists for the provider,
When the Credentialing Status component renders,
Then Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and Credentialing_Case__c.CAQH_Profile_ID__c are displayed, with a red indicator if Credentialing_Case__c.Expiration_Date__c is within 90 days of today.

Given the provider has a Network_Termination_Case__c in status 'Pending' or 'In Review',
When the page loads,
Then a banner alert is displayed: 'Active Termination Case: [Network_Termination_Case__c.Case_Number__c]' in red.

Given the user has Profile = 'Provider Relations Specialist',
When the page renders,
Then the TIN field (Account.TIN__c) is masked to show only the last 4 digits.

## Solution

Build a custom Lightning Record Page for the Provider Person Account RecordType using App Builder. Develop two LWC components: 'providerPnaPanel' (queries PNA and Fee_Schedule__c via @wire getRelatedListRecords) and 'providerCredentialingStatus' (Apex-backed @wire method with field-level masking logic for TIN). Termination banner implemented as a Record-Triggered Flow on Account load using a Screen Flow embedded component. TIN masking enforced via Apex with Profile-based conditional rendering and supplemented by Field-Level Security on Account.TIN__c. Deploy page assignment via Permission Set Group 'Provider_Network_PSG'.

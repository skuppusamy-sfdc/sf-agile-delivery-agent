## Description

As a Claims Operations Manager, I need a visual claims history timeline on the Policyholder 360 page that shows key milestones for each claim — including FNOL date, assignment, adjudication, and payment — so that I can quickly assess claim lifecycle progress and identify bottlenecks or stalled claims without manually reviewing activity history logs.

The timeline should cover all Claim__c records for the policyholder across all LOBs.

## Acceptance Criteria

Given a Contact record with at least one related Claim__c record,
When the Claims History Timeline LWC loads on the Policyholder 360 page,
Then each claim is displayed as a horizontal timeline row with milestone nodes for: FNOL (Claim__c.FNOL_Date__c), Assignment (Claim__c.Assignment_Date__c), Adjudication (Claim__c.Adjudication_Date__c), Payment (Payment__c.Payment_Date__c of most recent payment), and Closure (Claim__c.Closed_Date__c).

Given a milestone date field is null (e.g., Claim__c.Adjudication_Date__c is blank),
When the timeline renders,
Then that milestone node is displayed as an unfilled grey circle with a tooltip: 'Pending'.

Given a milestone date is populated,
When the timeline renders,
Then that milestone node is a filled circle in the Acme brand color with the date displayed below the node in MM/DD/YYYY format.

Given a Claim__c.Status__c = 'SIU Referral',
When the timeline renders for that claim,
Then a distinct red 'SIU' flag icon appears on the claim row and the claim row background is highlighted in light red.

Given more than 10 Claim__c records exist for the policyholder,
When the timeline component loads,
Then a maximum of 10 claims are displayed by default (sorted by Claim__c.FNOL_Date__c descending) with a 'Load More' button to retrieve the next 10.

Given the user clicks on any claim row in the timeline,
When the click event fires,
Then the user is navigated to the Claim__c record detail page in the same browser tab.

Given the timeline is embedded in the Policyholder 360 FlexCard page,
When the page loads on a mobile form factor,
Then the timeline degrades gracefully to a vertical stacked milestone list layout.

## Solution

Build a custom LWC 'policyholderClaimsTimeline' using SVG-based horizontal timeline rendering. Use @wire(getRecord) or a custom Apex controller 'PolicyholderClaimsTimelineCtrl' with a SOQL query joining Claim__c and Payment__c (most recent per claim via aggregation). Implement pagination using offset-based SOQL with a 'Load More' button triggering a subsequent Apex callout. Use CSS custom properties for brand color theming. Add conditional CSS classes for SIU flag display based on Claim__c.Status__c value. Use NavigationMixin for claim row click navigation. Register the LWC as an exposed component in App Builder and add to the 'Policyholder_360_Page' Lightning Record Page. Apply SLDS responsive grid for mobile degradation to vertical layout using CSS media query breakpoints.

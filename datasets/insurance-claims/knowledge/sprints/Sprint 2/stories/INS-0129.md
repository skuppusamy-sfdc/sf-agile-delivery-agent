## Description

As a Claims Operations Manager, I need the Policyholder 360 view to surface any active SIU (Special Investigations Unit) referrals and fraud flags associated with a claimant so that adjusters are alerted before processing payments on claims linked to policyholders under active fraud investigation, reducing improper payment risk and ensuring compliance with state fraud reporting mandates.

## Acceptance Criteria

Given a Contact record where Contact.SIU_Flagged__c = true, when the Policyholder 360 FlexCard loads, then a prominent red 'SIU Alert' banner is displayed above all other panels reading 'This policyholder has an active SIU referral. Do not process payments without SIU clearance.'

Given the SIU Alert banner is visible, when a user with the 'SIU_Investigator' Permission Set Group clicks 'View SIU Case', then they are navigated to the related Case record where Case.RecordType.DeveloperName = 'SIU_Investigation' and Case.Status != 'Closed'.

Given a user without the 'SIU_Investigator' Permission Set Group clicks 'View SIU Case', then the button is hidden and a tooltip reads 'SIU case access restricted. Contact your SIU team lead.'

Given a Claim__c record where Claim__c.Claimant__c is linked to an active SIU Case, when a user attempts to set Claim__c.Status__c = 'Approved for Payment', then a validation rule prevents the status change and displays: 'Payment approval blocked: Active SIU investigation on file. Case: [Case Number].'

Given Contact.SIU_Flagged__c = false and no open SIU Case records exist, when the FlexCard loads, then no SIU banner or related UI elements are rendered.

## Solution

Extend Policyholder360_FC (INS-0124) with a conditional SIU Alert state driven by DataRaptor Extract 'DR_GetSIUStatus' querying Case records filtered by RecordType = 'SIU_Investigation', AccountId/ContactId = current Contact, and Status != 'Closed'. Implement FlexCard conditional visibility on the SIU banner using state variable 'siuActive = true'. For permission-based button visibility, use FlexCard action conditions referencing a custom permission 'View_SIU_Cases'. Implement a Salesforce Validation Rule on Claim__c: 'AND(ISPICKVAL(Status__c, "Approved for Payment"), Claimant__r.SIU_Flagged__c = TRUE)'. Apply Sharing Rules to restrict SIU Case visibility to the 'SIU_Investigator' Permission Set Group.

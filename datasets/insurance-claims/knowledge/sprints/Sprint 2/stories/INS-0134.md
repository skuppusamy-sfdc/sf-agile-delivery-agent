## Description

**Bug Report — Policyholder 360 COB Indicator Missing**

**Environment:** Acme Insurance - Full Sandbox (ACME_FS01)
**Build:** Sprint 2 FlexCard deployment dated 2024-11-14

**Steps to Reproduce:**
1. Navigate to any Person Account record where two or more Policy__c records exist with Policy__c.Coverage_Type__c = 'Primary' on one and 'Secondary' on the other, and both have Policy__c.Status__c = 'Active'.
2. Open the Policyholder 360 FlexCard (deployed as part of INS-0132).
3. Observe the 'Active Policies' section header area where the COB indicator badge should appear.

**Expected Behavior:**
When a policyholder has both a primary and secondary active policy, a yellow 'COB' badge should display in the Active Policies section header, and each Policy__c row should show the Policy__c.COB_Order__c field value ('Primary' or 'Secondary').

**Actual Behavior:**
The COB badge does not render. The Policy__c.COB_Order__c field is absent from all policy rows regardless of coverage type. Single-policy policyholders are unaffected.

**Root Cause Hypothesis:**
The DataRaptor Turbo Extract for the Active Policies section does not include Policy__c.COB_Order__c or Policy__c.Coverage_Type__c in its field mapping. Additionally, the FlexCard conditional state expression for the COB badge appears to reference an incorrect field alias ('cob_flag' instead of 'COB_Indicator__c').

**Impact:**
Claims Operations staff cannot identify COB situations at a glance, risking incorrect primary/secondary claim routing and potential overpayment on adjudicated claims.

## Acceptance Criteria

As per Description.

Given a Person Account has two active Policy__c records where one has Policy__c.Coverage_Type__c = 'Primary' and another has Policy__c.Coverage_Type__c = 'Secondary',
When the Policyholder 360 FlexCard Active Policies section renders,
Then a yellow 'COB' badge is visible in the section header and each policy row displays Policy__c.COB_Order__c correctly.

Given a Person Account has only one active Policy__c record,
When the FlexCard renders,
Then no COB badge is shown and Policy__c.COB_Order__c is still displayed (with value 'Primary' or blank as applicable).

Given the fix is deployed,
When a regression test is run on all existing Policyholder 360 FlexCard test scenarios from INS-0132,
Then all previously passing acceptance criteria continue to pass with no new failures.

## Solution

Fix DataRaptor Turbo Extract 'DR_Extract_PolicyholderActivePolicies' to include Policy__c.COB_Order__c and Policy__c.Coverage_Type__c in the Output JSON structure. Update the FlexCard conditional state expression for the COB badge: change field alias reference from 'cob_flag' to 'COB_Indicator__c' and set display condition to: {policies_list | where: 'Coverage_Type__c == Secondary' | size} > 0. Redeploy FlexCard version to Full Sandbox and execute regression suite. No Apex changes required — pure OmniStudio configuration fix.

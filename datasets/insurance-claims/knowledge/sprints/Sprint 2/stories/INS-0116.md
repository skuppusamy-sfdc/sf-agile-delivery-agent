## Description

As a Claims Operations Manager, I need a unified Policyholder 360 FlexCard on the Contact/Person Account record so that I can instantly view all active policies, open claims, and recent payment history without navigating to multiple objects.

This view should surface real-time data from Policy__c, Claim__c, and Payment__c and replace the legacy tabbed sidebar component used in the previous implementation.

## Acceptance Criteria

Given a Claims Operations Manager opens a Policyholder Person Account record,
When the Policyholder 360 FlexCard loads,
Then it must display all associated Policy__c records where Policy__c.Status__c = 'Active' grouped by Policy__c.Line_of_Business__c (LOB).

Given the FlexCard is rendered,
When the 'Open Claims' section is populated,
Then all Claim__c records where Claim__c.Status__c IN ('FNOL Received', 'Under Review', 'Pending Adjudication') must be listed with Claim__c.Claim_Number__c, Claim__c.Date_of_Loss__c, and Claim__c.Total_Billed_Amount__c visible.

Given a policyholder has Payment__c records,
When the Payment History panel is rendered,
Then the 5 most recent Payment__c records sorted by Payment__c.Payment_Date__c DESC must display Payment__c.Amount__c, Payment__c.Payment_Method__c, and Payment__c.Status__c.

Given the FlexCard is viewed by a user without the 'Claims_Ops_Full_Access' Permission Set Group,
When the record loads,
Then Payment__c.Amount__c fields must be masked and the 'Initiate Payment' action button must not render.

-Given the FlexCard includes a 'Policy Documents' tab showing ContentDocument links, When a document is clicked, Then it opens inline in a modal- (deprecated — moved to INS-0121)

Given any section of the FlexCard fails to load data,
When a DataRaptor extract returns an empty result or error,
Then a user-friendly inline message 'No records found' must display per section without breaking the overall card render.

## Solution

Build an OmniStudio FlexCard named 'Policyholder_360_FC' with three child card states: ActivePolicies, OpenClaims, and PaymentHistory. Each state uses a dedicated DataRaptor Extract (DR_Extract_ActivePolicies, DR_Extract_OpenClaims, DR_Extract_RecentPayments) querying respective objects with SOQL filters. Conditional rendering via FlexCard conditions handles permission-based field masking — integrate with Permission Set Group 'Claims_Ops_Full_Access' checked via a custom @wire getPermissionSetAssignment Apex method. Deploy FlexCard to the Person Account Lightning Record Page via App Builder. Cache DataRaptor results for 60 seconds using OmniStudio cache settings to reduce SOQL load.

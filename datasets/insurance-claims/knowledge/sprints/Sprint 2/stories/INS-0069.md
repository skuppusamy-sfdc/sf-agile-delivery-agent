## Description

As a Claims Operations Manager, I need a unified Policyholder 360 FlexCard on the Contact record so that I can view a policyholder's active policies, open claims, recent payments, and EOB history without navigating to multiple objects.

This card should serve as the primary entry point for all policyholder interactions during claims adjudication and follow-up. Extends INS-0039 (Policyholder data model setup).

## Acceptance Criteria

Given a Contact record with at least one related Policy__c record,
When a Claims Operations Manager opens the Contact page,
Then the Policyholder 360 FlexCard renders with the following sections: Active Policies, Open Claims, Recent Payments, and EOB Summary.

Given the Active Policies section,
When the FlexCard loads,
Then it displays Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, and Policy__c.Status__c filtered to Status__c = 'Active'.

Given the Open Claims section,
When the FlexCard loads,
Then it lists all Claim__c records where Claim__c.Policyholder__c = Contact.Id and Claim__c.Status__c IN ('Submitted', 'In Review', 'Pending Information') sorted by Claim__c.FNOL_Date__c descending.

Given the Recent Payments section,
When the FlexCard loads,
Then it displays the last 5 Payment__c records related to the policyholder's claims, showing Payment__c.Amount__c, Payment__c.Payment_Date__c, and Payment__c.Payment_Method__c.

Given the EOB Summary section,
When a user clicks on a claim row,
Then a nested FlexCard expands to show related Explanation_of_Benefits__c records with EOB.EOB_Number__c, EOB.Total_Billed__c, EOB.Total_Allowed__c, and EOB.Patient_Responsibility__c.

-Given the COB indicator field, When COB__c = TRUE on Policy__c, Then display a red 'COB Active' badge on the policy row.- (Superseded — COB badge moved to INS-0081)

Given a Contact with no related Policy__c records,
When the FlexCard loads,
Then an empty state message reads: 'No active policies found for this policyholder.'

## Solution

Build OmniStudio FlexCard 'Policyholder360_FC' with child FlexCards for each section. Use DataRaptors (DR_GetActivePolicies, DR_GetOpenClaims, DR_GetRecentPayments, DR_GetEOBSummary) to retrieve data from Policy__c, Claim__c, Payment__c, and Explanation_of_Benefits__c. Leverage SOQL in DataRaptor Extract with relationship traversal. Conditional visibility on sections using FlexCard State Actions. Deploy FlexCard on Contact Lightning Record Page via App Builder. Apply Permission Set Group 'Claims_Ops_Standard' to restrict visibility. Use flyout panel for EOB nested FlexCard to avoid page reload.

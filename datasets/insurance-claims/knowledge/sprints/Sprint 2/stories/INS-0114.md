## Description

**Bug Report — Provider 360 FlexCard null NPI crash**

**Steps to Reproduce:**
1. Navigate to any Provider__c (Person Account) record where Provider__c.NPI__c is null or blank.
2. Open the Provider 360 FlexCard page (deployed as part of INS-0109).
3. Observe the Identity Verification tile in the FlexCard.

**Expected Behavior:**
The FlexCard should render gracefully with a 'NPI Not Recorded' placeholder in the Identity Verification tile, and NPPES verification badge should display as 'Unknown' (grey).

**Actual Behavior:**
The entire FlexCard fails to render. The browser console shows: 'TypeError: Cannot read properties of null (reading NPPES_Verification_Status__c)' originating from DataRaptor Transform DR_MapNPPESVerificationBadge. The OmniStudio FlexCard displays a generic 'An error occurred loading this component' message, blocking access to all other Provider 360 panels.

**Environment:** Full Sandbox (CS47)
**Frequency:** Reproducible 100% on any provider record with null NPI.
**Affected Records:** Approximately 340 Provider__c records in sandbox with NPI__c = null (legacy migrated data).

## Acceptance Criteria

As per Description.

Given Provider__c.NPI__c = null,
When the Provider 360 FlexCard loads,
Then the FlexCard must render all other panels successfully and the Identity Verification tile must display 'NPI Not Recorded' with a grey 'Unknown' badge — no error must be thrown.

Given the NPI field is null,
When DR_MapNPPESVerificationBadge DataRaptor Transform executes,
Then the transform must handle null input gracefully and return default values: badge_color = 'grey', badge_label = 'Unknown', last_verified = 'N/A'.

## Solution

Fix DataRaptor Transform DR_MapNPPESVerificationBadge to add a null-check conditional at the input mapping layer: if NPI__c IS NULL, short-circuit to default output values (badge_color='grey', badge_label='Unknown', last_verified='N/A') without proceeding to the NPPES field mapping branch. Additionally, add a FlexCard conditional state on the Identity Verification block to display a static 'NPI Not Recorded' text element when Provider__c.NPI__c IS NULL. Regression test all 340 affected sandbox records post-fix. Update DataRaptor unit test suite to include null-NPI test scenario.

## Description

As a Provider Relations Specialist, I need a Provider 360 page on the Provider__c (Person Account) record that consolidates the provider's active Provider_Network_Agreements, credentialing status from Credentialing_Case__c, NPI/TIN verification status from NPPES/CAQH, and any open Provider_Change_Notifications so that I can manage provider relationships from a single screen without toggling between related lists.

## Acceptance Criteria

Given a Provider__c (Person Account) record is open,
When the Provider 360 OmniScript-driven page loads,
Then all Provider_Network_Agreement__c records where PNA__c.Provider__c = current Account Id and PNA__c.Status__c = 'Active' must be displayed with PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, PNA__c.Network_Name__c, and PNA__c.Contract_Type__c.

Given a Credentialing_Case__c exists for the provider,
When the Credentialing panel loads,
Then the most recent Credentialing_Case__c record (sorted by CreatedDate DESC) must show Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and Credentialing_Case__c.CAQH_Profile_Status__c.

Given the provider's NPI is stored in Provider__c.NPI__c,
When the Identity Verification tile renders,
Then the system must display the last NPPES verification timestamp (Provider__c.NPPES_Last_Verified__c) and a status badge: 'Verified' (green), 'Pending' (amber), or 'Failed' (red) based on Provider__c.NPPES_Verification_Status__c.

Given one or more Provider_Change_Notification__c records exist with PCN__c.Status__c = 'Open',
When the Alerts section renders,
Then each open PCN must be listed with PCN__c.Change_Type__c, PCN__c.Effective_Date__c, and a 'Review' action button that navigates to the PCN record.

Given a provider has an active Network_Termination_Case__c,
When the page header loads,
Then a red banner 'Termination In Progress' must display with Network_Termination_Case__c.Expected_Termination_Date__c.

Given the page is accessed by a user with the 'Provider_Relations_Read_Only' Permission Set,
When the Provider 360 page loads,
Then all edit/action buttons must be hidden and fields must be read-only.

## Solution

Build OmniStudio FlexCard 'Provider360Card' on the Account object filtered to RecordType = 'Provider'. Use multiple DataRaptor Extracts: DR_GetActivePNAs, DR_GetLatestCredentialingCase, DR_GetOpenPCNs. NPPES verification badge uses a DataRaptor Transform to map Provider__c.NPPES_Verification_Status__c to badge color tokens. Termination banner uses FlexCard conditional state bound to a child DataRaptor checking for active Network_Termination_Case__c. Permission-based button visibility enforced via FlexCard Conditional Visibility on user's Profile/PSG. Extend existing Account Lightning Record Page (Provider layout) via App Builder.

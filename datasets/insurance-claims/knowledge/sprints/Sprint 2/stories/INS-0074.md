## Description

As a Credentialing Specialist, I need automated alerts when a provider's credentialing is expiring within 90, 60, or 30 days so that I can initiate re-credentialing workflows proactively and prevent providers from delivering services under a lapsed credential, which creates compliance and claims payment risk.

Alerts should surface both on the Provider 360 page (INS-0070) and as Salesforce bell notifications to the assigned Credentialing Specialist.

## Acceptance Criteria

Given a Credentialing_Case__c record where Credentialing_Case__c.Expiration_Date__c is exactly 90 days from today,
When the nightly scheduled job 'Credentialing_Expiry_Alert_Batch' runs,
Then a Platform Event 'Credentialing_Alert__e' is published with fields: Provider_Id__c, Alert_Tier__c = '90-Day', Expiration_Date__c, Assigned_Specialist__c.

Given the Platform Event is published,
When the event is consumed by the subscriber flow 'PEFlow_CredentialingAlert',
Then a Salesforce Custom Notification of type 'Credentialing Expiry Alert' is sent to the user identified in Assigned_Specialist__c with the message: 'Provider [Name] credentialing expires on [Expiration_Date__c]. Initiate re-credentialing now.'

Given the Provider 360 page loads for a provider with Credentialing_Case__c.Expiration_Date__c within 90 days,
When the CredentialingStatus_FC FlexCard renders,
Then the expiration date field is highlighted in yellow for 90-day, orange for 60-day, and red for 30-day thresholds.

Given a Credentialing_Case__c.Status__c transitions to 'Lapsed' (Expiration_Date__c has passed),
When the Record-Triggered Flow 'RT_Flow_CredentialingLapsed' fires,
Then the related Provider__c.Credentialing_Status__c field is updated to 'Lapsed' and a Task is created assigned to the Credentialing Specialist with Subject = 'URGENT: Credentialing Lapsed — Immediate Action Required' and Priority = 'High'.

Given a provider whose credential has lapsed (Provider__c.Credentialing_Status__c = 'Lapsed'),
When a new Claim__c is submitted with Claim__c.Rendering_Provider__c = that Provider__c.Id,
Then the claim is automatically routed to Claim__c.Status__c = 'Pending — Credentialing Review' and a warning is shown to the submitting user.

-Given the 30-day alert, When fired, Then also suspend the provider's active PNAs automatically.- (Removed from scope — PNA suspension requires legal review, moved to backlog INS-0099)

## Solution

Build Apex Batch class 'Credentialing_Expiry_Alert_Batch' implementing Database.Batchable and Schedulable, querying Credentialing_Case__c WHERE Expiration_Date__c IN (TODAY+90, TODAY+60, TODAY+30) AND Status__c != 'Lapsed'. For each qualifying record, publish Platform Event 'Credentialing_Alert__e' using EventBus.publish(). Build Platform Event-triggered Flow 'PEFlow_CredentialingAlert' that calls the Custom Notification action with notification type 'Credentialing Expiry Alert'. Build Record-Triggered Flow 'RT_Flow_CredentialingLapsed' on Credentialing_Case__c (trigger: after update, condition: Status__c changed to 'Lapsed') to update Provider__c.Credentialing_Status__c and create Task. Update CredentialingStatus_FC FlexCard conditional State Actions to apply CSS color classes based on date delta formula. Add Apex Trigger 'ClaimCredentialingCheck' on Claim__c (before insert) to validate rendering provider credentialing status.

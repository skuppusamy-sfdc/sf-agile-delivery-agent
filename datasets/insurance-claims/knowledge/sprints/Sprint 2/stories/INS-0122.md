# INS-0122: Provider 360 — Network Termination Case Status Banner and Restricted Actions for Providers Under Termination Review

## Description

As a Provider Network Specialist, I need the Provider 360 page to surface an alert banner and restrict certain actions when a provider has an open Network_Termination_Case__c so that team members do not inadvertently renew contracts, issue new PNAs, or modify fee schedules for a provider who is under active termination review.

## Acceptance Criteria

Given a Provider__c record has an associated Network_Termination_Case__c where NTC__c.Status__c IN ('Initiated', 'Under Review', 'Pending Approval'),
When any user opens the Provider 360 page,
Then a red banner must display at the top of the page with the message: 'TERMINATION IN PROGRESS — Case [NTC__c.Case_Number__c] opened [NTC__c.CreatedDate]. Contact Provider Relations before taking action.'

Given the termination banner is active,
When a user attempts to click 'New PNA' or 'Renew GSA' quick action buttons on the Provider 360 page,
Then those buttons must be hidden or disabled for users without the 'Provider_Network_Admin' Permission Set Group.

Given Network_Termination_Case__c.Status__c = 'Withdrawn' or 'Closed — No Action',
When the Provider 360 page loads,
Then the termination banner must not display and all standard actions must be re-enabled.

Given a Credentialing Specialist opens the same provider record,
When the termination banner is active,
Then a secondary informational note must display: 'Credentialing activities may continue pending termination outcome.' to avoid blocking credentialing workflows.

Given the provider has multiple historical NTC records,
When the banner logic evaluates,
Then only the most recent NTC record (sorted by NTC__c.CreatedDate DESC) with an open status must drive the banner display.

## Solution

Extend LWC 'providerNetworkSummary' (INS-0117) with a new child component 'providerTerminationBanner'. On component load, an @wire Apex call (ProviderTerminationService.getOpenTerminationCase) queries Network_Termination_Case__c for the current provider Id, filtered by open statuses, ordered by CreatedDate DESC LIMIT 1. Banner visibility and action button suppression controlled via reactive JavaScript properties. Quick action button hiding implemented via Dynamic Actions in App Builder with a formula condition referencing a custom formula field Provider__c.Has_Open_Termination__c (checkbox, driven by a Roll-Up Summary or Apex-maintained field). Permission Set Group check for 'Provider_Network_Admin' performed in LWC via custom permission @wire. All state changes to NTC__c.Status__c trigger a Record-Triggered Flow that recalculates Provider__c.Has_Open_Termination__c.

## Description

As a Provider Network Specialist, I need a Fee Schedule lookup panel on the Provider 360 page that allows me to search for applicable Fee_Schedule__c rates by CPT code, LOB, and network name so that I can quickly validate reimbursement rates during provider calls without switching to a separate system.

## Acceptance Criteria

Given the Provider 360 page is open for an in-network provider,
When the Fee Schedule panel loads,
Then a search form must present input fields for CPT Code (text, max 5 chars), LOB__c (picklist: Commercial, Medicare Advantage, Medicaid, Self-Funded), and Network_Name__c (picklist populated from active PNA__c.Network_Name__c values for this provider).

Given the user enters a valid CPT code and selects LOB and Network,
When they click 'Look Up Rate',
Then the system must query Fee_Schedule__c where Fee_Schedule__c.CPT_Code__c = entered value AND Fee_Schedule__c.LOB__c = selected LOB AND Fee_Schedule__c.Network__c = selected network AND Fee_Schedule__c.Effective_Date__c <= TODAY AND Fee_Schedule__c.Expiration_Date__c >= TODAY, returning Fee_Schedule__c.Allowed_Amount__c and Fee_Schedule__c.Rate_Type__c.

Given the query returns no matching Fee_Schedule__c record,
When results are displayed,
Then the panel must show 'No active fee schedule found for the selected criteria' and offer a 'Request Rate Review' quick action that creates a Case with RecordType = 'Fee Schedule Inquiry'.

Given multiple Fee_Schedule__c records match (e.g., tiered rates),
When results render,
Then all matching rows must be displayed in a table sorted by Fee_Schedule__c.Effective_Date__c DESC with Fee_Schedule__c.Modifier__c visible as an additional column.

Given the user does not have the 'FeeSchedule_View' custom permission,
When the Fee Schedule panel attempts to load,
Then the panel must be hidden and replaced with 'You do not have permission to view fee schedule data.'

## Solution

Add a new 'Fee Schedule Lookup' block to the Provider360Card FlexCard. Implement an OmniScript 'OS_FeeScheduleLookup' embedded within the FlexCard action panel. Use a DataRaptor Turbo Extract DR_FeeScheduleSearch with dynamic input parameters (CPT_Code, LOB, Network) mapped from OmniScript inputs. 'Request Rate Review' action uses an Integration Procedure IP_CreateFeeScheduleInquiryCase that invokes a Record-Triggered-equivalent Apex method to insert a Case record. Custom permission 'FeeSchedule_View' checked via FlexCard Conditional Visibility expression on $User.customPermissions.

# INS-0099: Fee Schedule Engine – display applicable CPT-level reimbursement rates on Provider 360 for adjudication reference

## Description

As a Claims Operations Manager, I need to view a provider's applicable Fee Schedule with CPT code-level reimbursement rates directly on the Provider 360 page so that I can quickly validate whether a submitted Claim_Line__c amount is within contracted rates during adjudication, without exporting to external spreadsheets. Extends INS-0094.

## Acceptance Criteria

Given a Provider__c with an active PNA__c linked to a Fee_Schedule__c, when the Fee Schedule panel on the Provider 360 page renders, then Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.LOB__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.Reimbursement_Basis__c (e.g., '% of Medicare', 'Per Diem', 'Case Rate') must display in the panel header.

Given a Fee_Schedule__c with associated Fee_Schedule_Line__c records, when a user expands the CPT detail section, then Fee_Schedule_Line__c records must render in a datatable showing: Fee_Schedule_Line__c.CPT_Code__c, Fee_Schedule_Line__c.ICD10_Category__c, Fee_Schedule_Line__c.Allowed_Amount__c, Fee_Schedule_Line__c.Unit_Type__c, and Fee_Schedule_Line__c.Modifier__c.

Given a searchable CPT panel, when a user types a CPT code in the search input, then the datatable must filter Fee_Schedule_Line__c records client-side to matching Fee_Schedule_Line__c.CPT_Code__c values in real time.

Given a Fee_Schedule__c.Effective_Date__c in the future, when the panel renders, then a blue informational badge 'Pending – Effective [date]' must appear and the rate rows must be visually distinguished (italicized).

Given a provider with multiple active Fee Schedules across different LOBs (e.g., Commercial, Medicare Advantage), when the panel renders, then each Fee Schedule must appear as a separate collapsible accordion section labeled by Fee_Schedule__c.LOB__c.

## Solution

Add Fee Schedule accordion section to Provider360Card FlexCard (INS-0094). Create DataRaptor Turbo Extract DR_GetFeeScheduleLines querying Fee_Schedule_Line__c JOIN Fee_Schedule__c JOIN Provider_Network_Agreement__c filtered by Provider__c.Id and PNA__c.Status__c = 'Active'. Build LWC 'feeScheduleSearch' component embedded in FlexCard flyout for client-side CPT search using JavaScript Array.filter() on the DataRaptor response payload. Use FlexCard accordion/expandable card pattern for multi-LOB fee schedule display. Implement FlexCard State for pending fee schedule badge (Effective_Date__c > TODAY). Ensure Fee_Schedule_Line__c.Allowed_Amount__c is protected by FLS under 'Fee_Schedule_Viewer' Permission Set. Coordinate with INS-0095 on the LOB-level display requirement moved from Facility 360 scope.

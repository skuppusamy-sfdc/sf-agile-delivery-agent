## Description

As a Claims Operations Manager, I need a COB detail modal accessible from the Policyholder 360 FlexCard so that I can review primary and secondary payer information, COB order, and any active COB disputes without leaving the policyholder record. This is required to ensure accurate claims adjudication sequencing and compliance with CMS COB guidelines.

## Acceptance Criteria

Given a Contact with Contact.COB_Flag__c = TRUE,
When the Claims Operations Manager clicks the COB badge on the Policyholder 360 FlexCard (INS-0077),
Then a modal opens displaying all COB__c records where COB__c.Policyholder__c = Contact.Id, ordered by COB__c.Payer_Order__c ascending.

Given a COB__c record exists,
When the modal renders,
Then each row displays COB__c.Payer_Name__c, COB__c.Payer_Order__c ('Primary'/'Secondary'/'Tertiary'), COB__c.Group_Number__c, COB__c.Member_ID__c, and COB__c.Effective_Date__c.

Given a COB__c record has COB__c.Dispute_Status__c = 'Open',
When the modal row renders,
Then a yellow 'Dispute Pending' badge is shown inline and COB__c.Dispute_Opened_Date__c is displayed.

Given the user has the 'Claims_Operations_PSG' Permission Set Group,
When the modal is open,
Then an 'Edit COB Order' button is visible that launches an OmniScript for reordering payer priority.

-Given a TPA__c is linked to the policy, display TPA contact information in the COB modal- (superseded: TPA details scoped to Policy Detail modal per INS-0077 AC note).

Given no COB__c records exist despite COB_Flag__c = TRUE,
When the modal opens,
Then a warning message is displayed: 'COB flag is set but no COB records found. Please contact the Compliance Officer to review.'

## Solution

Extend the Policyholder 360 FlexCard (INS-0077) by adding a modal action on the COB badge state. Build an OmniScript 'COBDetailOS' that uses a DataRaptor Extract to query COB__c records ordered by Payer_Order__c. Reorder functionality implemented as a drag-and-drop step within the OmniScript, writing back via DataRaptor Turbo Action. Dispute badge rendered via FlexCard conditional state on COB__c.Dispute_Status__c. Empty-state warning uses a conditional block in the Integration Procedure. Permission check for 'Edit COB Order' button enforced via Custom Permission 'COB_Edit_Permission' referenced in FlexCard state condition.

## Description

As a Provider Network Specialist, I need a Provider 360 page on the Provider__c (Person Account) record that consolidates NPI, TIN, credentialing status, active Provider Network Agreements, and associated Fee Schedules so that I can make informed contracting and network decisions without switching between multiple tabs and related lists.

This story covers the Person Account (individual provider) view. Facility accounts are handled in INS-0071.

## Acceptance Criteria

Given a Provider__c Person Account record,
When a Provider Network Specialist opens the account page,
Then the Provider 360 Lightning Page displays the following components: Provider Identity Panel, Credentialing Status Card, Active PNA Summary, Fee Schedule Summary, and Open PCN Alerts.

Given the Provider Identity Panel,
When the page loads,
Then it displays Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Primary_Specialty__c, Provider__c.License_State__c, Provider__c.License_Number__c, and Provider__c.CAQH_Provider_ID__c.

Given the Credentialing Status Card,
When the page loads,
Then it shows the most recent Credentialing_Case__c record related to the provider, displaying Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and Credentialing_Case__c.Credentialing_Body__c.
And if Credentialing_Case__c.Expiration_Date__c is within 90 days of today, a yellow 'Expiring Soon' warning badge is displayed.
And if Credentialing_Case__c.Status__c = 'Lapsed', a red 'Credentialing Lapsed' alert banner is displayed at the top of the page.

Given the Active PNA Summary,
When the page loads,
Then all Provider_Network_Agreement__c records where PNA.Provider__c = Account.Id and PNA.Status__c = 'Active' are listed with PNA.Agreement_Number__c, PNA.Network__c, PNA.Effective_Date__c, and PNA.Termination_Date__c.

Given the Fee Schedule Summary,
When a PNA row is selected,
Then the associated Fee_Schedule__c records are displayed showing Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.LOB__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.Rate_Basis__c.

Given the Open PCN Alerts section,
When Provider_Change_Notification__c records exist with PCN.Status__c IN ('Pending Review', 'Awaiting Signature') and PCN.Provider__c = Account.Id,
Then each PCN is listed with PCN.Change_Type__c, PCN.Submitted_Date__c, and a direct link to the PCN record.

Given a provider with no active PNAs,
When the PNA Summary section loads,
Then the section displays: 'No active network agreements on file.'

## Solution

Build a custom Lightning Record Page 'Provider_360_Page' using App Builder, replacing the default Account layout for Person Account record type 'Provider'. Use OmniStudio FlexCards: 'ProviderIdentity_FC', 'CredentialingStatus_FC', 'ActivePNA_FC', 'FeeScheduleSummary_FC', 'PCNAlerts_FC'. DataRaptors extract data from Credentialing_Case__c, Provider_Network_Agreement__c, Fee_Schedule__c, Provider_Change_Notification__c. Implement conditional badge rendering using FlexCard Conditional Actions with date formula comparisons. Use a Record-Triggered Flow on Credentialing_Case__c to set a formula field Provider__c.Credentialing_Alert_Status__c to drive real-time banner display. Assign page to Profile 'Provider Network Specialist' and Permission Set 'ProviderOps_View'.

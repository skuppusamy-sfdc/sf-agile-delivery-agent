## Description

As a Claims Operations Manager, I need a Policyholder 360 FlexCard on the Contact/Person Account record that consolidates active policies, open claims, recent payments, and COB details so that I can quickly assess a policyholder's full insurance profile without navigating multiple objects.

## Acceptance Criteria

Given a Claims Operations Manager views a Person Account record with associated Policy__c records,
When the Policyholder 360 FlexCard loads,
Then all active Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.Policyholder__c = current Contact Id are displayed with fields: Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, and Policy__c.Premium_Amount__c.

Given the FlexCard is rendered,
When the 'Open Claims' section is evaluated,
Then all Claim__c records where Claim__c.Status__c IN ('FNOL Received', 'Under Review', 'Pending Adjudication') and Claim__c.Policyholder__c = current Contact Id are listed, showing Claim__c.Claim_Number__c, Claim__c.Date_of_Loss__c, Claim__c.Total_Billed_Amount__c, and Claim__c.Assigned_Adjuster__c.

Given a policyholder has Payment__c records,
When the Payment History section loads,
Then the 5 most recent Payment__c records are shown ordered by Payment__c.Payment_Date__c DESC, displaying Payment__c.Payment_Amount__c, Payment__c.Payment_Method__c, and Payment__c.EOB_Reference__c.

Given a policyholder has a COB arrangement,
When COB data is present (Policy__c.COB_Flag__c = true),
Then a COB indicator badge is displayed on the FlexCard header with the secondary carrier name from Policy__c.Secondary_Carrier__c.

-Given the FlexCard includes a 'File New Claim' quick action button launching a modal form- (deprecated — moved to standalone FNOL OmniScript per INS-0047)

Given the user does not have the 'Claims_Viewer' Permission Set Group assigned,
When they navigate to the Policyholder 360 tab,
Then the FlexCard renders a 'Restricted Access' message and no PHI data is exposed.

## Solution

Build OmniStudio FlexCard 'Policyholder360_FC' with child cards for Policies, Claims, and Payments. Use DataRaptor Extract DRs for each data section: DR_GetActivePolicies, DR_GetOpenClaims, DR_GetRecentPayments — all keyed on recordId. Implement conditional rendering states for COB badge using FlexCard State conditions on Policy__c.COB_Flag__c. Apply field-level security via Permission Set Group 'Claims_Viewer_PSG'. Deploy FlexCard to Contact Lightning Record Page via App Builder. Cache DataRaptor responses for 60 seconds to reduce SOQL load on high-traffic pages.

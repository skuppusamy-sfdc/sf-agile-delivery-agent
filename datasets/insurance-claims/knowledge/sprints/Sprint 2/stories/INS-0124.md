## Description

As a Claims Operations Manager, I need a consolidated Policyholder 360 FlexCard on the Contact/Person Account record so that I can view all active policies, open claims, recent EOBs, and payment history in a single screen without navigating across multiple objects.

## Acceptance Criteria

Given a Contact record with associated Policy__c records, when the Policyholder 360 FlexCard loads, then all Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.Policyholder__c = current Contact Id are displayed in a scrollable datatable showing Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, and Policy__c.Premium_Amount__c.

Given a policyholder with one or more Claim__c records, when the Claims panel is rendered, then all Claim__c records where Claim__c.Claimant__c = current Contact Id and Claim__c.Status__c IN ('Open','In Review','Pending Payment') are listed with Claim__c.Claim_Number__c, Claim__c.FNOL_Date__c, Claim__c.Total_Incurred__c, and Claim__c.Assigned_Adjuster__c visible.

Given a policyholder with Payment__c records, when the Payment History panel loads, then the five most recent Payment__c records ordered by Payment__c.Payment_Date__c DESC are shown, displaying Payment__c.Amount__c, Payment__c.Payment_Method__c, and Payment__c.Status__c.

Given a policyholder with Explanation_of_Benefits__c records, when the EOB panel renders, then EOB__c records where EOB__c.Policyholder__c = current Contact Id are shown with EOB__c.Service_Date__c, EOB__c.Provider_Name__c, EOB__c.Billed_Amount__c, EOB__c.Allowed_Amount__c, and EOB__c.Patient_Responsibility__c.

Given a user without the 'Claims_Viewer' Permission Set Group, when the FlexCard loads, then the Claims panel is hidden and a message 'Insufficient permissions to view claims data' is displayed.

-Given a COB indicator, display a secondary payer badge on the policy tile- (Superseded by INS-0131 which handles COB display separately.)

## Solution

Build an OmniStudio FlexCard ('Policyholder360_FC') on the Contact object. Use four child DataRaptor Extract actions: DR_GetActivePolicies, DR_GetOpenClaims, DR_GetRecentPayments, DR_GetRecentEOBs — each joined via Contact Id. Conditionally render panels using FlexCard conditional visibility rules tied to the user's Permission Set Group membership, queried via a DataRaptor Transform. Deploy the FlexCard on the Contact Lightning Record Page via a Lightning App Builder tab. Cache DataRaptor results for 30 seconds using OmniStudio caching config to reduce SOQL load.

## Description

As a Provider Relations Specialist, I need a Related Claims panel on the Provider 360 page that shows recent claims billed by the provider — including claim status, total billed amount, allowed amount, payment status, and ERA reference — so that I can quickly address provider inquiries about claim payment status and ERA reconciliation without requiring the provider to call the claims department.

This panel supplements the Provider 360 page built in INS-0070 and should be added as a new collapsible section.

## Acceptance Criteria

Given a Provider__c Person Account record,
When the Provider 360 page loads,
Then a 'Related Claims' collapsible panel is visible at the bottom of the page, collapsed by default.

Given the Related Claims panel is expanded,
When the panel loads,
Then it displays Claim__c records where Claim__c.Rendering_Provider__c = Account.Id OR Claim__c.Billing_Provider__c = Account.Id, showing: Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, Claim__c.Total_Allowed_Amount__c, Claim__c.Service_Date__c, and Payment__c.ERA_Reference_Number__c of the most recent associated payment.

Given the panel loads,
When results are returned,
Then claims are sorted by Claim__c.Service_Date__c descending and limited to the most recent 25 records.

Given a Claim__c with no associated Payment__c record,
When the ERA Reference column renders for that claim,
Then the cell displays 'Awaiting Payment' in italicized grey text.

Given a Claim__c.Status__c = 'Denied',
When the claim row renders in the panel,
Then the Status field displays in red bold text and a tooltip on hover reads: 'Claim denied. Review EOB for denial reason code.'

Given a Provider Relations Specialist clicks on a Claim__c.Claim_Number__c link in the panel,
When the link is activated,
Then the user navigates to the Claim__c record detail page.

-Given the panel, When loaded, Then also display Claim_Line__c level detail inline.- (Descoped — claim line detail adds excessive load time; deferred to INS-0084 with lazy loading approach)

## Solution

Add a new 'RelatedClaims_FC' OmniStudio FlexCard to the Provider_360_Page Lightning Record Page as a collapsible section using FlexCard Accordion layout. Build DataRaptor Extract 'DR_GetProviderRelatedClaims' with a SOQL query on Claim__c with OR filter on Rendering_Provider__c and Billing_Provider__c, joined to most recent Payment__c via a subquery using ORDER BY CreatedDate DESC LIMIT 1. Apply conditional text styling in FlexCard for denied claims using State Actions with CSS class 'slds-text-color_error slds-text-title_bold'. Use FlexCard action of type 'Navigation' on Claim_Number__c field for record navigation. Set panel default state to collapsed using FlexCard Accordion initial state property. Ensure DataRaptor handles null Payment__c gracefully by returning literal string 'Awaiting Payment' using a Transform step.

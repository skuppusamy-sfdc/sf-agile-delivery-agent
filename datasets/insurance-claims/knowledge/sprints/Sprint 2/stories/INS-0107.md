## Description

As a Claims Operations Manager, I need Agentforce-powered next-best-action (NBA) recommendations displayed on open Claim__c records so that I can prioritize adjudication tasks, identify potential SIU referral triggers, and reduce manual triage time by surfacing AI-driven insights based on claim attributes, claimant history, and provider patterns.

The Agentforce topic for Claims Operations will be configured to analyze Claim__c, Claim_Line__c, Policy__c, and Provider__c data and surface up to three ranked NBA cards per claim.

## Acceptance Criteria

Given an open Claim__c record with Claim__c.Status__c in ('Submitted', 'In Review', 'Pending Information') is viewed by a Claims Operations Manager,
When the Agentforce NBA component loads in the utility bar or record page sidebar,
Then up to 3 ranked recommendation cards are displayed, each containing: Action Title, Reasoning Summary (max 150 characters), Confidence Score (%), and an action button.

Given a claim has Claim__c.Total_Billed_Amount__c exceeding the Policy__c.Coverage_Limit__c by more than 10%,
When Agentforce evaluates the claim,
Then a recommendation card 'Review Coverage Limit Exceedance' is surfaced with Confidence Score >= 80% and an action button 'Flag for Supervisor Review' that sets Claim__c.Supervisor_Review_Flag__c = TRUE.

Given a Claim__c has 3 or more Claim_Line__c records with Claim_Line__c.Denial_Reason__c = 'Duplicate Submission' within the last 90 days for the same Provider__c,
When Agentforce evaluates the claim,
Then a recommendation card 'Potential Duplicate Billing Pattern — Consider SIU Referral' is surfaced.

Given the user clicks 'Accept' on any NBA recommendation card,
When the action is executed,
Then a Claim__c chatter post is auto-created logging: the recommendation title, confidence score, and the name of the user who accepted it.

Given a Claims Operations Manager does not have the 'Agentforce_Claims_User' Permission Set assigned,
When they view a Claim__c record,
Then the Agentforce NBA component is not visible on the page layout.

-Given Agentforce is connected to an external fraud scoring API, display the fraud score on the NBA card- (deprecated — external fraud API integration descoped from Phase 1; revisit in Phase 2 SIU workstream)

## Solution

Configure an Agentforce Agent Topic 'Claims_Operations_NBA' scoped to the Claim__c object. Define Agent Actions using Apex-invocable methods: (1) CheckCoverageLimitExceedance — queries Policy__c.Coverage_Limit__c and compares to Claim__c.Total_Billed_Amount__c; (2) DetectDuplicateBillingPattern — SOQL aggregate on Claim_Line__c with date filter and provider grouping; (3) GeneralClaimRiskScore — Einstein scoring model (Claims_Risk_Score_v1) trained on historical adjudication data. Agent instructions prompt configured in Agentforce Agent Builder to rank actions by confidence threshold (>= 70%) and return top 3. NBA cards rendered via the standard Agentforce Einstein Next Best Action LWC component embedded in the Claim__c Lightning Record Page right sidebar. 'Accept' action handled by a Record-Triggered Flow invoked from the NBA component's action callback, creating the chatter post via ConnectApi.ChatterFeeds. Permission gating via 'Agentforce_Claims_User' Permission Set included in the Claims_Ops_PSG Permission Set Group.

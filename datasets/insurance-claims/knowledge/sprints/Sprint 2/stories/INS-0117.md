## Description

As a Provider Network Specialist, I need a Provider 360 page on the Provider__c (Person Account) record that surfaces the provider's active Provider_Network_Agreement__c (PNA), associated Fee_Schedule__c lines, and current Credentialing_Case__c status so that I can assess network participation and credentialing posture in a single view.

Extends INS-0086 (Provider Account data model setup). This story builds the UI layer on top of the data model delivered in that story.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c Person Account record,
When the Provider 360 Lightning Page loads,
Then a 'Network Agreements' related list must display all Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active' and PNA__c.Provider__c = current record Id, showing PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, PNA__c.Network_Name__c, and PNA__c.LOB__c.

Given the provider has an active PNA,
When the 'Fee Schedules' panel renders,
Then all Fee_Schedule__c records linked via Fee_Schedule__c.PNA__c must display Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Allowed_Amount__c, and Fee_Schedule__c.Effective_Date__c.

Given the provider has one or more Credentialing_Case__c records,
When the Credentialing Status component loads,
Then the most recent Credentialing_Case__c sorted by Credentialing_Case__c.CreatedDate DESC must display with Credentialing_Case__c.Status__c, Credentialing_Case__c.CAQH_Profile_Status__c, and Credentialing_Case__c.Expiration_Date__c.

Given Credentialing_Case__c.Expiration_Date__c is within 90 days of today,
When the panel renders,
Then a yellow warning badge labeled 'Expiring Soon' must appear adjacent to the credentialing record.

Given Credentialing_Case__c.Status__c = 'Suspended' or 'Terminated',
When the panel renders,
Then a red alert banner 'Credentialing Action Required' must display at the top of the Provider 360 page.

Given the provider's NPI field (Provider__c.NPI__c) is populated,
When the page loads,
Then a 'Verify with NPPES' quick action button must be available that triggers an outbound callout to the NPPES API and updates Provider__c.NPPES_Last_Verified__c with the current timestamp.

## Solution

Create a custom Lightning Record Page 'Provider_360_Page' using App Builder, assigned to the Provider__c (Person Account) record type 'Rendering_Provider'. Build an LWC 'providerNetworkSummary' using @wire getRelatedListRecords to pull PNA and Fee Schedule data. Credentialing status component is a separate LWC 'providerCredentialingStatus' with conditional CSS classes for badge/banner rendering. NPPES verification uses an Apex @AuraEnabled callout method (NPPESIntegrationService.verifyProvider) invoked from the LWC, with Named Credential 'NPPES_API' for endpoint auth. A Record-Triggered Flow on Credentialing_Case__c fires on Status__c change to update a formula-driven alert field on Provider__c used by the banner LWC.

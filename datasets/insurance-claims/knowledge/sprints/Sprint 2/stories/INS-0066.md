## Description

As an Underwriting Analyst, I need the Policyholder 360 view to display real-time Medicare/Medicaid eligibility verification data fetched from the CMS (Centers for Medicare & Medicaid Services) integration so that claims adjudicators can confirm a policyholder's government program eligibility at the time of claim review, reducing COB errors and duplicate payment risk.

## Acceptance Criteria

Given a Claims Operations Manager or Underwriting Analyst views a Policyholder 360 page,
When the policy's Policy__c.LOB__c = 'Medicare' OR 'Medicaid',
Then an 'Eligibility Verification' section is visible on the Policyholder 360 FlexCard showing last verified date, eligibility status, and Part A/Part B coverage flags sourced from CMS.

Given the Eligibility section is visible,
When the user clicks 'Verify Now',
Then a callout is made to the CMS Beneficiary FHIR API via named credential 'CMS_FHIR_NC' using the policyholder's Policy__c.Medicare_Beneficiary_ID__c (MBI), and the response is parsed and stored in Policy__c.CMS_Eligibility_Status__c, Policy__c.CMS_Last_Verified_Date__c, Policy__c.CMS_Part_A_Active__c, and Policy__c.CMS_Part_B_Active__c.

Given the CMS API callout completes successfully,
When Policy__c.CMS_Eligibility_Status__c = 'Ineligible',
Then a red warning banner is displayed on the Policyholder 360 page: 'CMS Eligibility Inactive — Claims adjudication may be impacted. Last Verified: [Policy__c.CMS_Last_Verified_Date__c]'.

Given the CMS API returns a timeout or error response (HTTP 4xx/5xx),
When the callout fails,
Then the error is logged to a custom object 'Integration_Error_Log__c' with fields: Error_Log__c.Source_System__c = 'CMS', Error_Log__c.Record_Id__c = Policy__c.Id, Error_Log__c.Error_Code__c, Error_Log__c.Timestamp__c, and the UI displays: 'Eligibility verification unavailable — please retry or contact IT Support.'

Given Policy__c.LOB__c is NOT 'Medicare' or 'Medicaid',
When the Policyholder 360 page loads,
Then the Eligibility Verification section is hidden and no CMS callout is made.

Given eligibility data was last verified more than 30 days ago,
When the Eligibility section renders,
Then a yellow 'Stale Data' badge is shown next to Policy__c.CMS_Last_Verified_Date__c prompting re-verification.

## Solution

Create Apex class 'CMSEligibilityService' with @future(callout=true) method calling CMS Beneficiary FHIR R4 API via Named Credential 'CMS_FHIR_NC'. Parse JSON response using CMSEligibilityResponse wrapper class and update Policy__c fields via DML. Expose Apex method as invocable action 'VerifyCMSEligibility' called from FlexCard action button in Policyholder360_FC. Implement error logging via Apex to Integration_Error_Log__c. Add Record-Triggered Flow on Policy__c to auto-trigger CMS verification when LOB__c changes to 'Medicare' or 'Medicaid'. Store Named Credential secrets in Salesforce org-level Named Credential with per-user principal for audit trail. Add 30-day staleness check using Formula field Policy__c.Eligibility_Data_Stale__c = (TODAY() - CMS_Last_Verified_Date__c) > 30.

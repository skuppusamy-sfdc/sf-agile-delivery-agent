## Description

As a Claims Operations Manager, I need a unified Policyholder 360 FlexCard on the Contact/Person Account record page so that I can view all active policies, open claims, recent payments, and COB flags without navigating across multiple objects.

## Acceptance Criteria

Given a Policyholder record is open in Salesforce,
When the Policyholder 360 FlexCard loads,
Then it must display all related Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.PolicyHolder__c = current Contact Id.

Given an active Policy__c exists,
When the Claims panel is rendered,
Then all Claim__c records with Claim__c.Claim_Status__c IN ('Open','Pending Adjudication','In Review') must be listed with Claim__c.Claim_Number__c, Claim__c.Date_of_Service__c, Claim__c.Total_Billed_Amount__c, and Claim__c.LOB__c visible.

Given a Claim__c record has one or more Payment__c children,
When the Payment Summary tile renders,
Then Payment__c.Payment_Amount__c values must be summed and displayed as 'Total Paid to Date' alongside Payment__c.Payment_Status__c breakdown (Issued / Voided / Pending).

Given the policyholder has a COB flag (Policy__c.COB_Flag__c = true),
When the FlexCard header section loads,
Then a prominent amber banner reading 'COB Review Required' must appear with a link to the related COB case.

Given the FlexCard is viewed by a user without the 'Claims_Ops_Full_Access' Permission Set Group,
When sensitive fields such as Policy__c.SSN_Masked__c are requested,
Then those fields must be masked or hidden per field-level security.

-Given a FNOL is in draft status, display it in the Open Claims panel- (deprecated — FNOL panel moved to INS-0112)

Given no active policies exist for the policyholder,
When the FlexCard loads,
Then an empty-state message 'No Active Policies Found' must display with a 'Create New Policy' quick action button.

## Solution

Build OmniStudio FlexCard 'Policyholder360Card' targeting the Contact object. Use DataRaptor Extract (DR_GetPolicyholder360Data) to query Policy__c, Claim__c, and Payment__c via SOQL with aggregate functions for payment totals. Implement conditional card states for COB banner using FlexCard State conditions on Policy__c.COB_Flag__c. Apply FLS enforcement via DataRaptor field mapping respecting running user's Permission Set Group. Deploy FlexCard to the Contact Lightning Record Page via App Builder. Child Claim panel uses a DataRaptor Turbo Extract for performance on large claim volumes.

## Description

As a Claims Operations Manager, I need a unified Policyholder 360 FlexCard on the Contact record so that I can quickly assess a member's full insurance profile — including active policies, open claims, COB details, and recent payment history — without navigating across multiple objects.

This story extends the base OmniStudio framework established in INS-0071 and surfaces data from Policy__c, Claim__c, Payment__c, and EOB records associated with the policyholder.

## Acceptance Criteria

Given a Contact record with the 'Policyholder' record type is open in Salesforce,
When the Claims Operations Manager views the Policyholder 360 FlexCard,
Then the FlexCard displays the following sections without page reload:
  - Active Policies panel: Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, Policy__c.Status__c = 'Active'
  - Open Claims panel: Claim__c.Claim_Number__c, Claim__c.FNOL_Date__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, Claim__c.Adjudicated_Amount__c
  - COB Indicator: displays 'Primary' or 'Secondary' based on Policy__c.COB_Priority__c field value
  - Recent Payments panel: last 5 Payment__c records ordered by Payment__c.Payment_Date__c DESC, showing Payment__c.Amount__c and Payment__c.Payment_Method__c

Given a policyholder has no open claims,
When the Claims panel renders,
Then an empty-state message 'No open claims on file' is displayed and the panel does not throw a null pointer error.

Given the FlexCard is loaded,
When the user clicks the Claim_Number__c hyperlink in the Open Claims panel,
Then the browser navigates to the corresponding Claim__c record detail page.

-Given a policyholder has SIU flags, display a red banner on the FlexCard header- (deprecated — SIU banner moved to INS-0115)

Given a user does not have the 'Claims_Ops_Full_Access' Permission Set Group assigned,
When they view the Policyholder 360 FlexCard,
Then the Recent Payments panel is hidden and replaced with 'Access Restricted'.

## Solution

Build an OmniStudio FlexCard named 'Policyholder360_FC' anchored to the Contact object. Use four child DataRaptors (DR_Get_Active_Policies, DR_Get_Open_Claims, DR_Get_COB_Priority, DR_Get_Recent_Payments) to fetch related records via SOQL with appropriate WHERE clauses filtering on Contact lookup fields. Embed the FlexCard in the Contact Lightning Record Page via the OmniStudio FlexCard component. Apply field-level visibility conditions on the Payments panel using a Conditional View tied to the running user's Permission Set Group membership, checked via a DataRaptor Transform querying PermissionSetAssignment. Deploy via OmniStudio deployment package and include in the Phase 1 unlocked package.

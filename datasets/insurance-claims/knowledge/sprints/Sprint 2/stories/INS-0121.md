## Description

As a Provider Relations Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record whenever key demographic or credentialing fields are updated on a Provider__c (Person Account) so that downstream teams and integrated systems (CMS, CAQH) are notified of material changes within the required compliance window.

Triggered fields include: NPI, TIN, practice address, specialty, and board certification expiration. This supports regulatory obligations under CMS provider data accuracy standards.

## Acceptance Criteria

Given a user updates Provider__c.NPI__c on an existing provider record,
When the record is saved,
Then a Provider_Change_Notification__c record must be auto-created with PCN__c.Change_Type__c = 'NPI Update', PCN__c.Provider__c = triggering record Id, PCN__c.Old_Value__c = prior NPI, PCN__c.New_Value__c = updated NPI, PCN__c.Status__c = 'Pending Review', and PCN__c.Created_Date__c = today.

Given a user updates Provider__c.TIN__c,
When the record is saved,
Then a PCN must be created with PCN__c.Change_Type__c = 'TIN Update' and the same field population rules as above.

Given Provider__c.Board_Certification_Expiration__c is updated,
When the record is saved,
Then a PCN must be created with PCN__c.Change_Type__c = 'Credentialing Data Update'.

Given a PCN is created with Status__c = 'Pending Review',
When 72 hours elapse without a status change,
Then an escalation task must be auto-assigned to the provider's owning Provider Relations Specialist with Subject = 'PCN Requires Review — [Provider Name]' and Due Date = CreatedDate + 3 days.

Given a PCN__c.Status__c is updated to 'Approved',
When the approval occurs,
Then a Platform Event 'ProviderDataChanged__e' must be published containing PCN__c.Change_Type__c, PCN__c.Provider__c, and PCN__c.New_Value__c for downstream CAQH and CMS integration consumption.

-Given the PCN is approved, When the event fires, Then a Workday API callout must update the provider's HR record in real time- (deprecated — Workday sync deferred; providers are not Workday-managed entities)

Given a bulk data load updates 500+ provider records simultaneously,
When the trigger fires,
Then PCN creation must execute in bulk-safe Apex without hitting governor limits (no SOQL/DML inside loops).

## Solution

Implement Apex Trigger 'ProviderChangeTrigger' on Provider__c (Account) object, firing on AFTER UPDATE. Trigger handler class 'ProviderChangeTriggerHandler' uses a static map of monitored field API names. For each changed field in Trigger.new vs Trigger.old comparison, a PCN SObject is instantiated and added to a list for bulk insert. Platform Event 'ProviderDataChanged__e' published via EventBus.publish() on PCN Status__c = 'Approved' via a separate Record-Triggered Flow on Provider_Change_Notification__c. 72-hour escalation handled by a Scheduled Flow 'PCN_Escalation_Scheduler' running every 6 hours, querying PCN records where Status__c = 'Pending Review' AND CreatedDate <= N-3days. All Apex written with Trigger Framework pattern (handler + domain separation) and 95%+ test coverage target.

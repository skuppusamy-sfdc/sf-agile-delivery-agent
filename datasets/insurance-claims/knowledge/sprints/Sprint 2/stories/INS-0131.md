## Description

As a Claims Operations Manager, I need an Agentforce AI assistant embedded in the Policyholder 360 view so that adjusters can generate a natural-language summary of a claimant's claim history, open items, and payment status on demand — reducing the average handle time for complex multi-claim policyholders and surfacing actionable next steps without requiring manual data aggregation across Claim__c, Payment__c, and EOB records.

## Acceptance Criteria

Given an adjuster is viewing the Policyholder 360 page, when they click the 'Summarize with AI' button in the Agentforce panel, then an Agentforce prompt is invoked that retrieves all Claim__c records for the policyholder where Claim__c.Status__c != 'Closed' and generates a plain-English summary including total open claim count, Claim__c.Total_Incurred__c sum, and the oldest Claim__c.FNOL_Date__c.

Given the AI summary is generated, when it is displayed in the Agentforce panel, then the response includes a 'Recommended Next Steps' section with at least one suggested action (e.g., 'Schedule COB review', 'Request medical records for Claim [Claim Number]', 'Escalate to SIU') based on claim flags present on Claim__c (Claim__c.COB_Review_Required__c, Claim__c.SIU_Referral__c, Claim__c.Medical_Records_Requested__c).

Given the AI summary references a specific Claim__c record, when the adjuster clicks the claim number hyperlink in the summary, then they are navigated to the Claim__c detail record page.

Given the Agentforce response is generated, when it is displayed, then a disclaimer reads: 'AI-generated summary. Verify all data against source records before taking action.' and the prompt response is logged to a custom object AI_Interaction_Log__c with fields: AI_Interaction_Log__c.User__c, AI_Interaction_Log__c.Record_Id__c, AI_Interaction_Log__c.Prompt_Text__c, AI_Interaction_Log__c.Response_Text__c, AI_Interaction_Log__c.Timestamp__c.

Given a policyholder with zero open claims, when the 'Summarize with AI' button is clicked, then the AI response returns 'No open claims found for this policyholder. All claims are in Closed status.' and no Recommended Next Steps are generated.

-Given the AI summary, auto-create a follow-up Task from the recommended next steps- (Descoped pending Agentforce Actions governance approval — targeted for Sprint 4.)

## Solution

Configure an Agentforce Agent 'Claims_Adjuster_Assistant' in Salesforce AI Studio with a custom Prompt Template 'ClaimSummaryPrompt' using the Einstein Prompt Builder. The prompt template grounding uses a DataRaptor Extract to fetch Claim__c, Payment__c, and EOB__c data for the current Contact, injected as structured context into the LLM prompt via the Prompt Template merge fields. Embed the Agentforce component on the Policyholder 360 Lightning Record Page as an LWC wrapper 'c/agentforceClaimsSummaryPanel' that invokes the Agentforce Agent via the Einstein API. Implement AI_Interaction_Log__c writes via an Apex after-trigger on the LWC callout response. Apply a custom Permission Set 'Agentforce_Claims_User' to control access to the AI panel. Ensure all LLM calls route through Acme's approved Einstein Trust Layer with data masking on NPI__c and TIN__c fields.

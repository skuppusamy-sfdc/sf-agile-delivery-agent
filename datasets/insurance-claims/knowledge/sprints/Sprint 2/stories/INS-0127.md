## Description

**Bug Report — Provider 360 FlexCard Null PNA Crash**

**Steps to Reproduce:**
1. Navigate to any Provider__c (Person Account) record where no Provider_Network_Agreement__c records exist (e.g., a newly onboarded provider pending contracting).
2. Open the Provider 360 Lightning Record Page.
3. Observe the Network Agreements panel within the Provider360_FC FlexCard.

**Expected Behavior:** The Network Agreements panel should render an empty state message: 'No active network agreements on file. Initiate contracting via the New PNA button.'

**Actual Behavior:** The FlexCard throws an unhandled JavaScript error: 'Cannot read properties of undefined (reading 0)' and the entire FlexCard fails to render, including the Summary Header, Credentialing, Fee Schedule, and PCN panels which have valid data.

**Environment:** Full Sandbox (ACME_FS1)
**Affected Users:** Provider Relations Specialist, Credentialing Specialist
**Frequency:** 100% reproducible on providers with zero PNA records.
**Related Story:** INS-0125

## Acceptance Criteria

As per Description. Additionally: Given a Provider__c record with zero PNA records, when the Provider 360 FlexCard loads, then all other panels (Summary, Credentialing, Fee Schedule, PCN) render successfully and the Network Agreements panel displays the empty state message without a JavaScript error.

## Solution

Root cause: DataRaptor Extract DR_GetActivePNAs returns an empty array which the FlexCard datatable action attempts to index at position [0] without a null check. Fix: Add a null/empty array guard in the FlexCard action mapping using a conditional expression 'IF(ISEMPTY(PNAList), [], PNAList)'. Add an empty state card state in Provider360_FC with condition 'PNAList.length == 0' rendering the 'No active network agreements' message. Deploy fix to ACME_FS1 for regression testing before promoting to UAT.

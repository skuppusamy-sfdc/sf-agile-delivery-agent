## Description

As a Claims Operations Manager, I need a Coordination of Benefits (COB) indicator and secondary payer details panel within the Policyholder 360 view so that adjusters can immediately identify COB scenarios and access secondary payer information before adjudicating a claim, reducing overpayment risk and compliance exposure.

This story was partially scoped in INS-0069 but extracted as a standalone story per sprint planning on 2024-11-08.

## Acceptance Criteria

Given a Policy__c record where Policy__c.COB_Active__c = TRUE,
When the Policyholder 360 FlexCard loads,
Then a prominent 'COB Active' badge in amber is displayed adjacent to the policy row within the Active Policies section.

Given the COB indicator is clicked,
When a Claims Operations Manager interacts with the badge,
Then a flyout panel opens displaying: Policy__c.Secondary_Payer_Name__c, Policy__c.Secondary_Payer_Policy_Number__c, Policy__c.Secondary_Payer_Phone__c, Policy__c.COB_Order__c, and Policy__c.COB_Verified_Date__c.

Given a Claim__c record in context where the related Policy__c.COB_Active__c = TRUE,
When the claim is opened from the Policyholder 360 FlexCard,
Then Claim__c.COB_Flag__c is automatically set to TRUE via a Record-Triggered Flow if not already set.

Given a COB scenario where Policy__c.COB_Order__c = 'Primary',
When the COB panel is displayed,
Then a guidance note reads: 'Acme Insurance is the primary payer. Bill secondary payer after adjudication.'

Given a COB scenario where Policy__c.COB_Order__c = 'Secondary',
When the COB panel is displayed,
Then a guidance note reads: 'Acme Insurance is the secondary payer. Require primary EOB before processing.' and a prompt to attach the primary EOB is shown.

Given a Policy__c record where Policy__c.COB_Active__c = FALSE,
When the Policyholder 360 FlexCard loads,
Then no COB badge or panel is rendered for that policy row.

## Solution

Extend the 'Policyholder360_FC' FlexCard built in INS-0069 by adding a conditional COB State Action that evaluates Policy__c.COB_Active__c. Build a new 'COBDetails_FC' child FlexCard using a DataRaptor Extract targeting Policy__c COB fields. Use FlexCard flyout action to open the COB panel without page navigation. Implement a Record-Triggered Flow 'RT_Flow_SetClaimCOBFlag' on Claim__c (trigger: before save, entry condition: Related_Policy__r.COB_Active__c = TRUE) to auto-set Claim__c.COB_Flag__c = TRUE. Add conditional text rendering in the COB flyout using OmniScript Set Values actions driven by Policy__c.COB_Order__c picklist value.

## Description

**Bug Report — Policyholder 360 FlexCard NullPointerException on Empty Policy Set**

**Steps to Reproduce:**
1. Log in as a user with the 'Claims_Operations_PSG' Permission Set Group.
2. Navigate to any Contact record with RecordType = 'Policyholder' that has zero Policy__c records linked (e.g., newly onboarded policyholder Contact.Id = '0035g000004XyZAAA0').
3. Observe the Policyholder 360 FlexCard component on the Lightning page.

**Expected Behavior:**
The FlexCard renders an empty-state message: 'No active policies found for this policyholder.' and all other tabs (Open Claims, Payment History) remain accessible.

**Actual Behavior:**
The FlexCard throws a runtime error: 'Uncaught NullPointerException in DataRaptor Extract: PolicyholderPolicyDRX — result set is null' and the entire FlexCard fails to render, displaying a blank white panel. The error is logged in Salesforce debug logs under the Integration Procedure 'Policyholder360IP'.

**Environment:** Full Sandbox (AcmeIns-FS1)
**Affected Build:** Sprint 2 Day 3 deployment
**Linked Story:** INS-0077

## Acceptance Criteria

As per Description. Additionally: Given a Policyholder Contact with no linked Policy__c records, When the FlexCard loads, Then the DataRaptor Extract 'PolicyholderPolicyDRX' returns an empty array (not null) and the FlexCard renders the empty-state illustration without console errors.

## Solution

Fix the DataRaptor Extract 'PolicyholderPolicyDRX' to include a null-check output transform that returns an empty JSON array [] when the SOQL result set is null. Update the Integration Procedure 'Policyholder360IP' to add a Conditional Action block that routes to an empty-state response node when policy count = 0. Add a FlexCard conditional state 'No_Policies' that renders the empty-state message panel. Add unit test coverage in the OmniStudio test suite and a dedicated Apex test method for the null-result path.

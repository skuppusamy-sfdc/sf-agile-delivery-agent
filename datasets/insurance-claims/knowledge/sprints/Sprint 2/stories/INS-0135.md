## Description

As a Provider Network Specialist, I need the Salesforce platform to automatically detect changes to a provider's NPI or TIN sourced from nightly NPPES and CAQH data feeds and update the corresponding Provider__c record fields, so that provider demographic data stays current and any material change automatically initiates a Provider Change Notification (PCN) workflow for compliance review.

Currently, NPI and TIN discrepancies are discovered manually during claims adjudication, causing ERA matching failures and delayed payments. This story implements the inbound data sync and PCN auto-creation logic. The outbound PCN notification to payers is handled in a follow-on story.

## Acceptance Criteria

Given the nightly NPPES data feed file is received and processed by the DataIntegration middleware,
When a provider record in the feed has an NPI value that differs from Provider__c.NPI__c on the matched Salesforce record (matched via Provider__c.CAQH_Provider_ID__c),
Then Provider__c.NPI__c is updated to the new value, Provider__c.Last_NPPES_Sync_Date__c is stamped with the current datetime, and a Provider_Change_Notification__c record is created with PCN__c.Change_Type__c = 'NPI Change', PCN__c.Previous_Value__c = old NPI, PCN__c.New_Value__c = new NPI, PCN__c.Status__c = 'Pending Review', and PCN__c.Source__c = 'NPPES Auto-Sync'.

Given the nightly CAQH data feed is processed,
When a provider's TIN in the feed differs from Provider__c.TIN__c,
Then Provider__c.TIN__c is updated, Provider__c.Last_CAQH_Sync_Date__c is stamped, and a PCN__c record is created with PCN__c.Change_Type__c = 'TIN Change', PCN__c.Status__c = 'Pending Review', PCN__c.Source__c = 'CAQH Auto-Sync', and PCN__c.Assigned_To__c auto-populated with the Provider__c.Primary_Network_Specialist__c lookup value.

Given a PCN__c record is created by the auto-sync process,
When PCN__c.Status__c = 'Pending Review',
Then an email notification is sent to the assigned Provider Network Specialist using the 'PCN_Review_Required' email template, including Provider__c.Provider_Name__c, PCN__c.Change_Type__c, PCN__c.Previous_Value__c, and PCN__c.New_Value__c in the body.

Given no change is detected in NPI or TIN between the feed and the existing Provider__c record,
When the sync process runs,
Then no PCN__c is created, Provider__c.Last_NPPES_Sync_Date__c or Provider__c.Last_CAQH_Sync_Date__c is still updated, and a sync log entry is written to the custom object Integration_Sync_Log__c with Sync_Log__c.Result__c = 'No Change Detected'.

-Given a TIN change is detected, automatically suspend all active PNA__c records for the provider pending review-
(Deprecated: Auto-suspension of PNAs on TIN change removed — legal confirmed this requires manual approval step; see updated compliance guidance CR-2024-089)

Given the inbound feed contains a provider CAQH_Provider_ID__c that does not match any existing Provider__c record,
When the sync process encounters the unmatched record,
Then no update is made, an Integration_Sync_Log__c entry is created with Sync_Log__c.Result__c = 'Match Not Found' and Sync_Log__c.Raw_Payload__c populated, and a daily digest report is sent to the DataIntegration team distribution list summarizing all unmatched records.

Given the sync job processes more than 200 provider records in a single batch,
When the Apex batch job executes,
Then all records are processed without hitting governor limits, using a batch size of 50, and the total job runtime is logged in Integration_Sync_Log__c.

## Solution

Implement an Apex Batch class 'NPPES_CAQH_ProviderSyncBatch' scheduled nightly via Scheduled Apex. Inbound feed files (CSV/JSON) are delivered to a Salesforce-connected S3 bucket and retrieved via a named credential callout. Batch job queries Provider__c records by CAQH_Provider_ID__c for matching. Field comparisons performed in Apex; differing NPI/TIN values trigger DML updates to Provider__c and insert of Provider_Change_Notification__c via a dedicated Apex service class 'PCNAutoCreationService'. A Record-Triggered Flow on Provider_Change_Notification__c (trigger: After Insert, condition: Source__c = 'NPPES Auto-Sync' OR 'CAQH Auto-Sync') sends the email alert using the 'PCN_Review_Required' template via Send Email action. Integration_Sync_Log__c records written in the batch finish method. Platform Event 'Provider_Data_Sync_Complete__e' fired on batch completion for downstream consumers. Unit test coverage target: 85%+. All field updates governed by the 'Provider_Data_Integration' Permission Set assigned to the integration user.

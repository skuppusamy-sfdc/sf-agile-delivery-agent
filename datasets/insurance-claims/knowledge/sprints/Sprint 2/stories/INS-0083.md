## Description

**Bug Report — TIN Masking Bypass for System Administrator Profile**

**Steps to Reproduce:**
1. Log in as a user with Profile = 'System Administrator' (not assigned 'Provider_Network_PSG').
2. Navigate to any Provider__c (Person Account) record with RecordType = 'Provider'.
3. Observe the Provider 360 Lightning page — specifically the TIN field rendered by the 'providerCredentialingStatus' LWC component.

**Expected Behavior:**
All users regardless of profile — including System Administrators — should see Account.TIN__c masked to show only the last 4 digits (e.g., '***-**-6789') as per the data privacy requirement in INS-0078 AC #6.

**Actual Behavior:**
Users with Profile = 'System Administrator' see the full unmasked TIN (e.g., '12-3456789') in the LWC component. The masking logic in the Apex controller uses a Profile name check that explicitly excludes 'System Administrator', which was a development shortcut and does not meet compliance requirements.

**Compliance Impact:** Potential violation of IRS TIN handling guidelines and internal data privacy policy DPP-2024-07.
**Environment:** Full Sandbox (AcmeIns-FS1) and Production
**Affected Component:** 'providerCredentialingStatus' LWC, Apex class 'ProviderCredentialingController'
**Linked Story:** INS-0078

## Acceptance Criteria

As per Description. Additionally: Given any authenticated Salesforce user regardless of Profile or Permission Set, When the Provider 360 page loads and the TIN field is rendered, Then Account.TIN__c is always masked to show only the last 4 digits in the UI layer, with no exceptions for System Administrator or any other profile.

## Solution

Remove the Profile-based bypass in 'ProviderCredentialingController' Apex class. Implement TIN masking purely in the Apex @AuraEnabled method using String manipulation (right() + padding) with no profile exemption logic — masking applied unconditionally before the value is returned to the LWC. Remove Field-Level Security reliance for masking (FLS may still grant read access for backend processes, but UI always receives masked value). Add a dedicated Apex test method asserting that a System Administrator running user receives the masked value. Conduct a security review with the Compliance Officer (Sofia Hernandez) before deploying to Production. Log remediation in compliance tracker under DPP-2024-07.

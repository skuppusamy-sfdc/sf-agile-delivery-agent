## Description

As a Provider Relations Specialist, I need a comprehensive Provider 360 view on both Provider__c (Person Account) and Facility__c (Business Account) record pages so that I can quickly assess a provider's contracting status, active network agreements, fee schedule assignments, credentialing standing, and any open Provider Change Notifications (PCNs) without switching between multiple tabs or objects.

This story covers the OmniStudio FlexCard and supporting DataRaptors. Credentialing Case integration with CAQH is handled separately in INS-0136.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c (Person Account) record,
When the Provider 360 FlexCard loads,
Then the header section displays: Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Specialty__c, Provider__c.Board_Certification_Status__c, and a color-coded credentialing badge derived from the most recent Credentialing_Case__c.Status__c linked to this provider.

Given the provider has one or more Provider_Network_Agreement__c records,
When the 'Network Agreements' tab is selected,
Then all PNA__c records where PNA__c.Provider__c = current Account Id AND PNA__c.Status__c = 'Active' are displayed, showing PNA__c.Network_Name__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, PNA__c.LOB__c, and PNA__c.Contracting_Specialist__c.

Given a PNA has an associated Fee_Schedule__c,
When the user clicks 'View Fee Schedule' on a PNA row,
Then a modal opens displaying Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.Base_Rate__c, and a list of up to 10 CPT codes from the Fee_Schedule__c related list.

Given the provider has an active Group_Service_Agreement__c (GSA),
When the 'GSA' section renders,
Then GSA__c.Agreement_Name__c, GSA__c.Employer_Group__c, GSA__c.Effective_Date__c, and GSA__c.MCO_Name__c are displayed.

Given one or more Provider_Change_Notification__c records exist with PCN__c.Status__c IN ('Submitted', 'Pending Review'),
When the 'Open PCNs' section loads,
Then each PCN is listed with PCN__c.Change_Type__c, PCN__c.Submitted_Date__c, and PCN__c.Assigned_To__c, and a red badge counter appears on the tab label.

Given a Facility__c (Business Account) record is opened,
When the Provider 360 FlexCard loads,
Then the same sections render but source data is filtered by Facility__c relationships, and the header shows Facility__c.NPI__c, Facility__c.Facility_Type__c, and Facility__c.Accreditation_Status__c instead of individual provider fields.

Given no active PNA exists for the provider,
When the Network Agreements tab is selected,
Then the empty state displays: 'No active network agreements on file.' with a 'Initiate Contracting' Quick Action button that pre-populates a new PNA__c record with the current provider's Account Id.

## Solution

Implement using OmniStudio FlexCard with tabbed layout. Create separate DataRaptor Turbo Extract procedures for: (1) PNA + Fee Schedule data, (2) Credentialing_Case__c latest record, (3) PCN open records, (4) GSA data. Use a parent FlexCard with child FlexCards per tab. Conditional rendering logic applied at FlexCard state level to distinguish Person Account (Provider__c) vs Business Account (Facility__c) using Account.RecordType.DeveloperName. Deploy to both Provider and Facility Lightning Record Pages via App Builder. PCN badge counter driven by a DataRaptor aggregate query. 'Initiate Contracting' Quick Action invokes an OmniScript pre-populated via FlexCard action parameters. Assign to 'Provider_Relations_Standard' Permission Set Group for read access; 'Provider_Network_Admin' PSG required for write actions.

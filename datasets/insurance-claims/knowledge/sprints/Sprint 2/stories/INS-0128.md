## Description

As a Credentialing Specialist, I need the Provider 360 summary header (extends INS-0125) to display real-time data freshness indicators for NPPES and CAQH so that I can immediately identify whether a provider's NPI registration data and credentialing attestation are current, reducing the risk of contracting with providers whose credentials have lapsed or whose NPI status has changed with CMS.

## Acceptance Criteria

Given a Provider__c record, when the Provider 360 summary header loads, then two sync status badges are displayed: one for NPPES (sourced from Provider__c.NPPES_Last_Sync__c) and one for CAQH (sourced from Provider__c.CAQH_Last_Attestation_Date__c).

Given Provider__c.NPPES_Last_Sync__c is within the last 30 days, when the NPPES badge renders, then it displays a green 'Synced' indicator with the formatted date.

Given Provider__c.NPPES_Last_Sync__c is older than 30 days or NULL, when the NPPES badge renders, then it displays a red 'Stale — Resync Required' indicator and a 'Trigger NPPES Sync' action button is enabled.

Given Provider__c.CAQH_Last_Attestation_Date__c is within 90 days, when the CAQH badge renders, then it displays a green 'Attested' indicator.

Given Provider__c.CAQH_Last_Attestation_Date__c is older than 90 days or NULL, when the CAQH badge renders, then it displays an amber 'Re-attestation Due' indicator and a Task is auto-created on the Provider__c record assigned to the owning Credentialing Specialist with Subject = 'CAQH Re-attestation Required' and Due Date = TODAY() + 7.

Given a user clicks 'Trigger NPPES Sync', when the action fires, then a Platform Event NPPES_Sync_Request__e is published with Provider__c.NPI__c as the payload, and Provider__c.NPPES_Sync_Status__c is set to 'Sync In Progress'.

-Given CAQH sync, auto-update Provider__c.Specialty__c from CAQH response- (Descoped to Phase 2 per architecture review.)

## Solution

Extend Provider360_FC summary header panel (INS-0125) with two badge components driven by DataRaptor Transform 'DRT_EvaluateProviderDataFreshness' computing staleness from NPPES_Last_Sync__c and CAQH_Last_Attestation_Date__c relative to TODAY(). Implement 'Trigger NPPES Sync' as a FlexCard action that invokes an Apex action class 'NPPESSyncRequestAction' publishing the NPPES_Sync_Request__e Platform Event. Build a Platform Event-triggered Flow 'PEF_HandleNPPESSyncRequest' that calls an external NPPES REST API callout via Named Credential 'NPPES_API' and updates Provider__c fields on response. For CAQH task creation, implement a Record-Triggered Flow 'RTF_CreateCAQHReattestationTask' on Provider__c firing when CAQH_Last_Attestation_Date__c is evaluated as stale on record load (scheduled flow daily batch fallback).

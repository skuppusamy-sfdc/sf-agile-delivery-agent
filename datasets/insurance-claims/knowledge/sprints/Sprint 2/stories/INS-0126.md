## Description

As a Claims Operations Manager, I need a Coordination of Benefits (COB) indicator banner on the Policyholder 360 view (extends INS-0124) so that adjusters are immediately alerted when a claimant has multiple active policies across carriers, reducing duplicate payment errors and ensuring proper primary/secondary payer sequencing during adjudication.

## Acceptance Criteria

Given a policyholder with two or more Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.COB_Applicable__c = true, when the Policyholder 360 FlexCard loads, then a yellow warning banner reading 'COB Alert: Multiple active coverages detected. Verify payer sequencing before adjudication.' is rendered at the top of the FlexCard.

Given the COB banner is displayed, when a user clicks 'View COB Details', then an OmniScript modal opens displaying all active Policy__c records with Policy__c.Carrier_Name__c, Policy__c.Plan_Type__c, Policy__c.COB_Order__c (Primary/Secondary/Tertiary), and Policy__c.Group_Number__c.

Given a policyholder with only one active Policy__c or where Policy__c.COB_Applicable__c = false on all policies, when the FlexCard loads, then no COB banner is rendered.

Given a Claim__c record is opened for a COB-flagged policyholder, when Claim__c.Status__c transitions from 'FNOL Received' to 'In Review', then a Record-Triggered Flow sets Claim__c.COB_Review_Required__c = true and creates a Task assigned to the adjuster with Subject = 'COB Review Required Prior to Adjudication'.

## Solution

Extend the Policyholder360_FC FlexCard (INS-0124) with a conditional banner state driven by a DataRaptor Transform 'DRT_EvaluateCOBStatus' that counts active COB-applicable policies for the Contact. If count >= 2, set a FlexCard state variable 'cobAlert = true' to trigger banner visibility. Build an OmniScript 'OS_COBDetails' launched from the banner CTA button, populated by DataRaptor Extract DR_GetCOBPolicies. Implement a Record-Triggered Flow 'RTF_SetCOBReviewFlag' on Claim__c triggered when Status__c changes to 'In Review', with entry condition Claim__c.Claimant__c.COB_Flag__c = true.

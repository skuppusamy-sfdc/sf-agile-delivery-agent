## Description

As a Provider Relations Specialist, I need a Provider 360 page on the Provider Person Account record so that I can quickly assess a provider's contracting status, credentialing standing, and network participation without opening multiple related lists or tabs.

This extends the foundational Provider Account model established in INS-0055. The 360 view must surface Provider_Network_Agreement__c (PNA) records, Credentialing_Case__c status, active Fee_Schedule__c associations, and any open Provider_Change_Notification__c (PCN) items.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider (Person Account) record,
When the Provider 360 FlexCard loads,
Then the 'Network Agreements' panel displays all Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active' AND PNA__c.Provider__c = current Account Id, showing PNA__c.Network_Name__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, PNA__c.LOB__c.

Given the provider has a Credentialing_Case__c record,
When the 'Credentialing Status' panel renders,
Then it displays the most recent Credentialing_Case__c ordered by CreatedDate DESC, showing Credentialing_Case__c.Status__c, Credentialing_Case__c.CAQH_Profile_Status__c, Credentialing_Case__c.Expiration_Date__c, and Credentialing_Case__c.Assigned_Credentialing_Specialist__c.

Given the provider has active Fee Schedules,
When the 'Fee Schedule' panel renders,
Then it displays Fee_Schedule__c records where Fee_Schedule__c.Provider__c = current Account Id AND Fee_Schedule__c.Status__c = 'Active', showing Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c.

Given there is an open Provider_Change_Notification__c linked to the provider,
When the PCN Alerts panel renders,
Then it displays a warning badge with count of open PCN records where PCN__c.Status__c NOT IN ('Closed','Withdrawn') and PCN__c.Provider__c = current Account Id.

Given the provider's NPI is flagged in the NPPES integration as 'Deactivated',
When the page loads,
Then a red banner displays: 'Warning: NPI {Provider__c.NPI__c} is marked Deactivated in NPPES. Verify before processing claims.'

-Then the page also shows a Medicare Exclusion flag sourced from CMS- (deprecated — CMS exclusion check deferred to Phase 2)

Given the user holds the 'Provider_Network_User' Permission Set Group,
When they view the Provider 360 page,
Then all panels are visible and data is scoped to providers within the user's assigned network territory via Sharing Rules.

## Solution

Build OmniStudio FlexCard 'Provider_360_Card' with child panels: NetworkAgreements (DataRaptor Extract on PNA__c), CredentialingStatus (DataRaptor Turbo Extract on Credentialing_Case__c with LIMIT 1 ORDER BY CreatedDate DESC), FeeSchedules (DataRaptor Extract on Fee_Schedule__c), PCNAlerts (DataRaptor Extract with COUNT aggregation on PCN__c). NPPES deactivation banner driven by a conditional state on Provider__c.NPI_Status__c field populated via scheduled DataIntegration job from NPPES API. FlexCard placed on Provider Person Account Lightning page via App Builder, gated by Permission Set Group 'Provider_Network_User'. Sharing Rules on PNA__c scoped by Provider_Territory__c lookup.

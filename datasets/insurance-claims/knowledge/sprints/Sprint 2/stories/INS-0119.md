## Description

As a Provider Network Specialist, I need a Group Service Agreement (GSA) summary component on Facility__c (Business Account) record pages so that I can review contracted service terms, associated TIN, and covered LOBs for a facility without leaving the account record.

Facilities may have multiple GSAs across different LOBs (e.g., Commercial, Medicare Advantage, Medicaid MCO), and this component must clearly differentiate active vs. expired agreements.

## Acceptance Criteria

Given a Provider Network Specialist opens a Facility__c Business Account record,
When the GSA Summary component loads,
Then all Group_Service_Agreement__c records where GSA__c.Facility__c = current record Id must be listed, showing GSA__c.Agreement_Number__c, GSA__c.LOB__c, GSA__c.Status__c, GSA__c.Effective_Date__c, GSA__c.Expiration_Date__c, and GSA__c.TIN__c.

Given GSA__c.Status__c = 'Active',
When the component renders the agreement row,
Then a green 'Active' pill badge must display inline with the agreement.

Given GSA__c.Status__c = 'Expired' or GSA__c.Expiration_Date__c < TODAY(),
When the component renders,
Then the row must display a grey 'Expired' badge and the row background must be visually muted (light grey).

Given a GSA record is expiring within 60 days (GSA__c.Expiration_Date__c <= TODAY() + 60 AND > TODAY()),
When the component renders,
Then an amber 'Expiring Soon' badge must display and a toast notification must be shown once per session to the viewing user.

Given a Facility__c has no associated GSA records,
When the component loads,
Then a message 'No Group Service Agreements on file. Click + New GSA to begin contracting.' must display with a quick action shortcut.

Given the user has the 'Provider_Network_Edit' Permission Set,
When viewing an Active GSA row,
Then an inline 'Renew' action button must appear that launches an OmniScript 'GSA_Renewal_OS' pre-populated with the current GSA fields.

## Solution

Build LWC 'gsaSummaryCard' deployed to the Facility__c (Business Account) Lightning Record Page via App Builder. Use @wire getRelatedListRecords with dynamic filtering on GSA__c. CSS conditional classes applied via JavaScript getter functions based on GSA__c.Status__c and date comparisons. Toast notification for expiring GSAs uses NavigationMixin and a sessionStorage flag to prevent repeat display. 'Renew' button invokes OmniStudio OmniScript 'GSA_Renewal_OS' via lightning/navigation omniscript standard__webPage. Permission check via @wire getPermissionSetAssignment Apex method. Sharing Rules ensure Provider_Network_Specialist profile can read Facility__c and GSA__c records across all account ownership.

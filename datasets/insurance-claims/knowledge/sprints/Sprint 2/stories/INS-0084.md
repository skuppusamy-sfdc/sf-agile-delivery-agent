# INS-0084: Provider Change Notification (PCN) alert banner on Provider 360 for pending demographic or network changes

## Description

As a Provider Relations Specialist, I need a Provider Change Notification (PCN) alert banner displayed on the Provider 360 page whenever a pending PCN exists for a provider so that I am immediately aware of unresolved demographic updates (e.g., address changes, NPI corrections, specialty updates) or network status changes before taking any contracting or credentialing actions. Unresolved PCNs can cause claims routing errors and ERA mismatches if actioned without awareness.

## Acceptance Criteria

Given a Provider_Change_Notification__c record exists where PCN__c.Provider__c = Account.Id AND PCN__c.Status__c IN ('Pending', 'Under Review'),
When the Provider Relations Specialist loads the Provider 360 page (INS-0078),
Then a yellow alert banner is displayed at the top of the page with text: 'Pending Change Notification: [PCN__c.Change_Type__c] — Submitted [PCN__c.Submitted_Date__c]. Review before proceeding.'

Given multiple PCN__c records are in 'Pending' or 'Under Review' status,
When the banner renders,
Then the banner shows the count of pending PCNs (e.g., '3 Pending Change Notifications') and a 'View All' link that opens a filtered list view of PCN__c records for that provider.

Given a PCN__c.Change_Type__c = 'NPI Correction' or 'TIN Update',
When the banner renders,
Then the banner severity is escalated to red (critical) with additional text: 'NPI/TIN changes may impact claims routing and ERA matching. Contact the Compliance Officer before proceeding.'

Given a PCN__c.Status__c transitions to 'Resolved' or 'Rejected',
When a Record-Triggered Flow fires on PCN__c update,
Then the provider's Account.Pending_PCN_Flag__c is set to FALSE and the banner no longer appears on next page load.

Given the user has the Custom Permission 'PCN_Resolve_Permission',
When the alert banner is displayed,
Then a 'Resolve PCN' quick action button is visible in the banner that launches an OmniScript 'PCNResolutionOS' for inline resolution.

-Given PCN originates from NPPES integration, display NPPES source badge on the banner- (deferred to Phase 2 NPPES integration epic).

## Solution

Add a 'providerPcnAlertBanner' LWC to the Provider 360 Lightning Record Page (INS-0078) positioned at the top of the layout. LWC uses an @wire Apex method 'getPendingPcns' to query Provider_Change_Notification__c filtered by provider Account Id and status. Banner severity logic (yellow vs. red) implemented in the LWC JavaScript controller based on PCN__c.Change_Type__c values. 'View All' link uses NavigationMixin to open a filtered List View. Account.Pending_PCN_Flag__c maintained by a Record-Triggered Flow on Provider_Change_Notification__c (after update, on Status__c change). 'Resolve PCN' button launches OmniScript 'PCNResolutionOS' via an OmniStudio Navigation action. Custom Permission 'PCN_Resolve_Permission' added to 'Provider_Network_PSG' Permission Set Group for eligible roles.

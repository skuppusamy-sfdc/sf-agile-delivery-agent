# INS-0063: Policyholder 360 — surface Explanation of Benefits (EOB) documents with DocuSign status

- **Labels**: Phase1, EOB, DocuSign, 360View

## Description

As a Claims Operations Manager, I need the Policyholder 360 view to include a dedicated EOB section that lists all Explanation_of_Benefits__c records linked to the policyholder's claims, along with their DocuSign delivery status, so that I can confirm EOBs have been sent and acknowledged without leaving the policyholder record.

## Acceptance Criteria

Given a policyholder has at least one finalized claim,
When the EOB section of the Policyholder 360 FlexCard loads,
Then all Explanation_of_Benefits__c records where EOB.Policyholder__c = current Contact Id are displayed, sorted by Explanation_of_Benefits__c.EOB_Date__c DESC.

Given an EOB record is displayed,
When the DocuSign envelope status is available via the DocuSign__Status__c field on Explanation_of_Benefits__c,
Then the EOB row shows a status chip: 'Sent', 'Delivered', 'Completed', or 'Voided' mapped from Explanation_of_Benefits__c.DocuSign__Status__c values.

Given an EOB with DocuSign__Status__c = 'Voided' or NULL,
When the Claims Operations Manager views that EOB row,
Then a 'Resend EOB' action button is visible that triggers the DocuSign envelope resend via an Apex invocable action 'EOBDocuSignResendAction'.

Given the EOB section loads,
When more than 10 EOB records exist,
Then pagination is implemented showing 10 records per page with a 'Load More' control.

Given the EOB is linked to a Claim__c,
When the user clicks the Claim Number hyperlink in the EOB row,
Then navigation opens the related Claim__c record in a new subtab within the console.

## Solution

Extend the Policyholder360_FC FlexCard (built in INS-0061) with a new 'EOB' child card section. Create DataRaptor Extract 'DR_GetPolicyholderEOBs' querying Explanation_of_Benefits__c with DocuSign__Status__c field. Implement FlexCard action 'Resend EOB' calling Apex invocable method 'EOBDocuSignResendAction' which calls DocuSign REST API via named credential 'DocuSign_NC'. Implement client-side pagination using FlexCard state variable 'eobPageOffset'. Use OmniStudio Navigation action for Claim__c subtab navigation in Lightning Console app.

## Description

**Bug Summary:** The Active PNA Summary section on the Provider 360 page (INS-0070) returns zero results for providers whose TIN contains leading zeros (e.g., '012345678'), even when active Provider_Network_Agreement__c records exist for those providers.

**Steps to Reproduce:**
1. Navigate to a Provider__c Person Account record where Provider__c.TIN__c begins with '0' (e.g., TIN = '034567891').
2. Observe the Provider 360 Lightning Page.
3. Navigate to the 'Active PNA Summary' FlexCard section.
4. Note that 0 PNA records are displayed despite 3 active PNAs existing in the system for this provider.
5. Navigate directly to Provider_Network_Agreement__c list view, filter by PNA.Provider__c = [same provider], Status__c = 'Active' — records ARE present.

**Expected Behavior:** All active Provider_Network_Agreement__c records for the provider are displayed regardless of TIN format.

**Actual Behavior:** The DataRaptor Extract 'DR_GetActivePNAs' returns empty results when the provider TIN has leading zeros due to implicit numeric cast in the SOQL WHERE clause filtering on TIN__c.

**Environment:** Acme Insurance - Full Sandbox (FS1)
**Reported By:** Provider Network Specialist during UAT on 2024-11-12

## Acceptance Criteria

As per Description.

Given a Provider__c record with TIN__c = '034567891' and 3 related active PNA records,
When the Provider 360 page loads,
Then the Active PNA Summary section displays all 3 PNA records correctly.

Given any Provider__c record with a TIN containing leading zeros,
When the DataRaptor Extract DR_GetActivePNAs executes,
Then the SOQL filter treats TIN__c as a Text field and preserves leading zeros in the comparison.

## Solution

Root cause: DataRaptor Extract 'DR_GetActivePNAs' is performing a numeric comparison on TIN__c (Text field) due to a misconfigured Input Parameter type set to 'Number' instead of 'Text'. Fix: Open DR_GetActivePNAs in OmniStudio DataRaptor, navigate to the Input tab, change the TIN parameter data type from 'Number' to 'Text'. Validate SOQL filter reads: WHERE TIN__c = :inputTIN as a string literal. Add a unit test in Apex test class 'ProviderPNADataRaptorTest' with a provider TIN of '034567891' to assert non-empty result set. Regression test all DataRaptor Extracts that reference TIN__c or NPI__c fields to confirm correct data type mapping.

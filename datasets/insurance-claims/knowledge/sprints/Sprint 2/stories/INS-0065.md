## Description

As a Provider Network Specialist, I need a contextual quick-action panel on the Provider 360 page that allows me to initiate a Provider Change Notification (PCN), trigger a credentialing renewal workflow, or flag a provider for SIU review directly from the provider record, so that I can take immediate action without navigating away from the 360 view.

## Acceptance Criteria

Given a Provider Relations Specialist is on the Provider 360 page,
When the Quick Actions panel is visible,
Then three action buttons are present: 'Initiate PCN', 'Request Credentialing Renewal', and 'Flag for SIU Review'.

Given the user clicks 'Initiate PCN',
When the PCN OmniScript launches,
Then a new Provider_Change_Notification__c record is pre-populated with: PCN.Provider__c = current Account Id, PCN.NPI__c = Account.NPI__c, PCN.Initiated_By__c = running User Id, PCN.Status__c = 'Draft', and PCN.Effective_Date__c defaulting to today + 30 days.

Given the user clicks 'Request Credentialing Renewal',
When the provider's most recent Credentialing_Case__c.Expiration_Date__c is within 90 days or already past,
Then a new Credentialing_Case__c is created with Status__c = 'Renewal Requested', Parent_Provider__c = current Account Id, and a Chatter post is made on the record tagging the assigned Credentialing Specialist.

Given the user clicks 'Request Credentialing Renewal',
When the provider's Credentialing_Case__c.Expiration_Date__c is MORE than 90 days in the future,
Then a warning modal displays: 'Credentialing is not due for renewal. Current expiration: [date]. Proceed anyway?' with Confirm/Cancel options.

Given the user clicks 'Flag for SIU Review',
When the action is confirmed in the modal,
Then Account.SIU_Flag__c is set to TRUE, a Case record is created with RecordType = 'SIU_Investigation', Case.AccountId = current provider, and a Platform Event 'SIU_Alert__e' is published with provider NPI and TIN for downstream SIU system notification.

-Given the Quick Actions panel, When the user clicks 'Terminate Provider', Then a Network_Termination_Case__c is created- (deprecated — provider termination action moved to dedicated Termination workflow per compliance review; see INS-0058)

## Solution

Build LWC 'providerQuickActionPanel' embedded in the Provider 360 Lightning Record Page via App Builder. 'Initiate PCN' button launches OmniScript 'InitiatePCN_OS' with pre-population via input JSON from LWC. 'Request Credentialing Renewal' button calls Apex controller 'CredentialingRenewalController.requestRenewal()' which queries latest Credentialing_Case__c, evaluates expiration logic, creates new case record, and posts Chatter via ConnectApi.ChatterFeeds. 'Flag for SIU Review' button calls Apex 'SIUFlagController.flagProvider()' which updates Account.SIU_Flag__c, creates SIU Case, and publishes SIU_Alert__e Platform Event via EventBus.publish(). Add custom Permission Set 'Provider_QuickActions_PS' restricting SIU flag action to Compliance Officer and Senior Network Specialist roles only.

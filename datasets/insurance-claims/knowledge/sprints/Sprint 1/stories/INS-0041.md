## Description

As a Claims Operations Manager, I need a Policyholder 360 page on the Account (Policyholder) record that consolidates active policy coverage, open claims, recent EOBs, and COB indicators so that my team can assess a member's full insurance picture during a call or FNOL intake without switching applications.

## Acceptance Criteria

Given an Account record with Account_Type__c = 'Policyholder' is opened
When the Policyholder 360 Lightning page loads
Then a Coverage Summary LWC component displays all active Policy__c records (Policy_Status__c = 'Active') related to the account, showing: Policy_Number__c, LOB__c, Effective_Date__c, Termination_Date__c, and TPA__c name

Given the policyholder has open claims
When the Active Claims Snapshot component renders
Then it shows all Claim__c records where Claim_Status__c IN ('New', 'In Review') related via Policy__c.Policyholder__c, displaying: Claim_Number__c, FNOL_Date__c, Claim_Status__c, Total_Billed_Amount__c, and an inline link to the Claim__c record

Given the policyholder has COB eligibility
When COB_Flag__c = true on any related Claim__c
Then a prominent COB Alert banner is displayed at the top of the page with text: 'Coordination of Benefits required — review primary/secondary coverage before adjudication'

Given the policyholder has received EOBs
When the EOB History section renders
Then the 5 most recent Explanation_of_Benefits__c records are listed with: EOB_Number__c, EOB_Date__c, Claim__c reference, Total_Paid_Amount__c, and a 'Download PDF' button that retrieves the associated ContentVersion file

Given the page loads for an account with more than 50 related claims
When the Active Claims Snapshot component initializes
Then pagination is applied (10 records per page) and total claim count is displayed as 'Showing 1–10 of [N] open claims'

Given a user without PSG_Claims_Adjudicator or PSG_Claims_Manager permission set group
When they access the Policyholder 360 page
Then Payment__c amounts and EOB details are hidden and replaced with 'Access Restricted' placeholder text

## Solution

Custom LWC components: `policyholderCoverageSummary`, `activeClaimsSnapshot`, `eobHistoryPanel`, and `cobAlertBanner` composed onto a custom Lightning Record Page via App Builder, restricted to Account record type 'Policyholder'. Data fetched via `@wire(getRecord)` for account fields and Apex `@AuraEnabled(cacheable=true)` methods with SOQL queries using WITH SECURITY_ENFORCED for FLS compliance. Pagination implemented client-side in LWC using `currentPage` and `pageSize` tracked properties. COB banner visibility controlled via a getter evaluating the claims wire response. EOB PDF download triggers a server-side Apex call to retrieve ContentVersion BLOB and return as a base64 data URL. Custom Lightning page assigned to Policyholder profile and Claims Operations app.

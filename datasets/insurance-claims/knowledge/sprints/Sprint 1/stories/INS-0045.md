## Description

As a Compliance Officer, I need the Salesforce org to have My Domain configured, baseline password policies enforced, and session security settings aligned with HIPAA and state insurance regulatory requirements so that all subsequent configuration and integrations are built on a compliant, auditable foundation.

This story covers org-level setup tasks that must be completed before any object schema, permission model, or integration work begins in Sprint 1.

## Acceptance Criteria

Given the Salesforce org is provisioned for Acme Insurance,
When My Domain is deployed to all users,
Then the custom domain 'acme-insurance.my.salesforce.com' is active and Lightning Experience is enforced for all internal profiles.

Given baseline security settings are applied,
When an admin reviews Session Settings,
Then session timeout is set to 2 hours, 'Lock sessions to the IP address from which they originated' is enabled, and clickjack protection is enabled for all page types.

Given password policy requirements,
When the org-wide password policy is saved,
Then minimum password length is 12 characters, complexity requires mixed case + number + special character, and password expiration is set to 90 days.

Given HIPAA audit trail requirements,
When Field History Tracking is reviewed,
Then Field Audit Trail retention is set to 10 years on the org and at least the Setup Audit Trail is confirmed active.

-Given Single Sign-On (SSO) is configured with Okta in Sprint 1,
Then SAML metadata exchange is complete and test logins succeed.- (Deferred to Sprint 2 — INS-0071)

Given the deployment is complete,
When a Compliance Officer runs a Security Health Check,
Then the overall score is 85 or above with no Critical risks outstanding.

## Solution

1. Enable and deploy My Domain via Setup > My Domain; enforce Lightning Experience via profile settings.
2. Configure Session Settings and Password Policies via Setup using documented HIPAA baseline values.
3. Enable Field Audit Trail via Setup > Field Audit Trail and assign to all relevant objects once schema is created (tracked via INS-0046).
4. Create a Platform Readiness checklist record in a custom Setup_Checklist__c object (scratch org pattern) to track completion.
5. Export Security Health Check baseline report and attach to this story as evidence artifact.
6. All changes captured in Setup Audit Trail; no Apex required for this story.

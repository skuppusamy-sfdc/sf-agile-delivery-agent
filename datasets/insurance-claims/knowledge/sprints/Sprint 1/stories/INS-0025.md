# INS-0025: Build Provider 360 FlexCard with NPI, TIN, Credentialing Status, and Active Network Agreements

## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces key provider attributes — NPI, TIN, credentialing status, active PNA details, associated Facility__c, and open Provider_Change_Notification__c records — in a single unified view so that I can quickly assess a provider's network standing without navigating across multiple objects during a call or case review.

Extends INS-0023 (core object model must be deployed before FlexCard data sources can be configured).

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record, when the Provider 360 FlexCard loads, then:

1. The header tile must display: Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Specialty__c, Provider__c.Provider_Type__c (Picklist: MD|DO|NP|PA|Facility), and a color-coded status badge driven by Provider__c.Credentialing_Status__c (Active=Green, Pending=Yellow, Suspended=Red, Terminated=Grey).

2. Given the provider has one or more Provider_Network_Agreement__c records, when the FlexCard renders the 'Network Agreements' section, then it must display a datatable showing: PNA__c.Agreement_Number__c, PNA__c.Network_Name__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, PNA__c.Status__c filtered to Status__c IN ('Active','Pending_Renewal') only.

3. Given the provider is linked to a Facility__c (Business Account), when the FlexCard renders, then the Facility tile must show: Facility__c.Name, Facility__c.NPI__c, Facility__c.Facility_Type__c, Facility__c.Accreditation_Status__c.

4. Given open Provider_Change_Notification__c records exist for the provider, when the PCN section renders, then it must show PCN count badge and list PCN__c.Change_Type__c, PCN__c.Effective_Date__c, PCN__c.Status__c where Status__c != 'Closed'.

5. -Given the FlexCard includes a claims history mini-chart, when rendered, then it must show last 12 months of paid claims- (Descoped to Phase 2 — see INS-0047).

6. Given the FlexCard is embedded on the Provider__c Lightning record page, when a user without PSG 'Provider_Network_Specialist_PSG' views the page, then the PNA datatable section must be hidden and replaced with 'Access Restricted' placeholder text.

## Solution

Build using OmniStudio FlexCard. Create DataRaptor Extract 'DR_Provider360_Header' to fetch Provider__c fields via SOQL. Create DataRaptor Turbo Extract 'DR_Provider_PNA_List' with child relationship query on Provider_Network_Agreement__c filtered by Status__c. Create DataRaptor Extract 'DR_Provider_PCN_Open' for open PCN records. Compose FlexCards: parent FlexCard 'FC_Provider360' with three child FlexCards: 'FC_Provider_Header', 'FC_PNA_List', 'FC_PCN_Badge'. Use Conditional Visibility on PNA datatable block tied to $User.PermissionSetGroup assignment check via Custom Permission 'CAN_VIEW_PNA'. Activate FlexCard and embed via Lightning App Builder on Provider__c record page layout. Cache DataRaptor responses for 5 minutes using OmniStudio cache settings to reduce SOQL governor limit consumption.

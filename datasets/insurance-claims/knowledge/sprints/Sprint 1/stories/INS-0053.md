## Description

As a Compliance Officer, I need a well-structured Permission Set Group model for all Claims Processing roles so that each persona has access to exactly the objects and fields required for their function — no more, no less — satisfying the principle of least privilege and supporting audit requirements.

Roles in scope for Phase 1: Claims Operations Manager, Provider Network Specialist, Credentialing Specialist, Underwriting Analyst, SIU Investigator.

## Acceptance Criteria

Given the Permission Set Groups are deployed,
When a user assigned PSG 'PSG_Claims_Operations_Manager' accesses Claim__c,
Then they have Read/Edit access to all Claim__c fields except Payment__c.Bank_Account__c (no access) and Claim__c.SIU_Flag__c (Read-only).

Given the Permission Set Groups are deployed,
When a user assigned PSG 'PSG_Credentialing_Specialist' accesses Provider__c,
Then they have Read/Edit on Credentialing_Case__c, Read/Edit on NPI__c and TIN__c (unmasked), and Read-only on Provider_Network_Agreement__c.

Given the Permission Set Groups are deployed,
When a user assigned PSG 'PSG_SIU_Investigator' accesses Claim__c,
Then Claim__c.SIU_Flag__c is Read/Edit, and Payment__c records linked to flagged claims are Read-only.

Given a user with no PSG assignment attempts to access Claim__c,
When they navigate to the Claims tab,
Then they receive a 'Insufficient Privileges' error and no claim data is exposed.

Given all PSGs are deployed,
When a Compliance Officer runs the Permission Set Group Assignment report,
Then all 5 PSGs appear with their constituent Permission Sets listed and no overlapping Edit rights exist on Payment__c.Bank_Account__c across non-Payment roles.

## Solution

Create 5 Permission Sets (PS_Claims_Core, PS_Credentialing_Core, PS_SIU_Core, PS_Underwriting_Core, PS_Network_Specialist_Core) and group them into Permission Set Groups via Setup > Permission Set Groups. Field-Level Security configured per object via metadata API (fieldPermissions in .permissionset-meta.xml). Muting Permission Set 'MPS_Restrict_BankAccount' applied within PSG_Claims_Operations_Manager to explicitly deny Payment__c.Bank_Account__c. Sharing Model: Claim__c OWD set to Private; Sharing Rules created for role-based visibility. Deployed via Salesforce CLI scratch org validation before production push. Post-deploy checklist verified by Compliance Officer sign-off.

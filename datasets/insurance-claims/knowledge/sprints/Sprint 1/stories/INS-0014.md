## Description

As a Broker Relations Coordinator, I need a dedicated Account Record Type for Group/Employer accounts with a tailored page layout and an onboarding checklist path component so that I can efficiently manage Group Service Agreement (GSA) setup, track onboarding milestones, and ensure all required employer documentation is collected before the first policy effective date. This replaces ad-hoc tracking in external spreadsheets.

## Acceptance Criteria

Given the 'Group_Employer' Record Type is created on the Account object,
When a Broker Relations Coordinator creates a new Account with this Record Type,
Then the page layout includes fields: Group_ID__c (Text 15, Unique), SIC_Code__c (Text 4), Employer_Size__c (Picklist: 1-50|51-100|101-500|501-2500|2500+), Domicile_State__c (Picklist of US states), ERISA_Exempt__c (Checkbox), Broker_Of_Record__c (Lookup to Account where Account.Type = 'Broker'), Onboarding_Status__c (Picklist: Prospect|Application Received|Underwriting Review|GSA Executed|Active|Terminated).

Given the Account page layout for 'Group_Employer' Record Type is rendered,
When the Broker Relations Coordinator views the record,
Then a Lightning Path component is visible at the top of the page showing Onboarding_Status__c stages in sequence: Prospect → Application Received → Underwriting Review → GSA Executed → Active.

Given the Group_Service_Agreement__c (GSA) object is linked to the Group/Employer Account,
When a GSA record is created with GSA__c.Account__c pointing to a 'Group_Employer' Account,
Then GSA__c includes: GSA_Number__c (Auto Number), Effective_Date__c (Date, required), Renewal_Date__c (Date), LOB__c (Multi-Select Picklist: Medical|Dental|Vision|Pharmacy), Minimum_Enrollment__c (Number), Premium_Rate_Basis__c (Picklist: Per Employee|Per Employee Per Month|Composite), and Status__c (Picklist: Draft|Executed|Expired|Terminated).

Given a GSA record is saved with Status__c = 'Executed',
When the Record-Triggered Flow 'RTF_GSA_Executed_Update_Account' evaluates,
Then the parent Account's Onboarding_Status__c is automatically updated to 'GSA Executed' if it was previously 'Underwriting Review'.

Given the 'Group_Employer' Account record type is deployed,
When the Broker Relations Coordinator opens the Related Lists section,
Then they see related lists for: Group_Service_Agreements (GSA__c), Policies (Policy__c), Contacts (with role filter for Benefits Administrator and HR Contact), and Open Activities.

-Given DocuSign integration is available, When GSA Status__c moves to 'Executed', Then auto-trigger DocuSign envelope to Broker_Of_Record__c contact- (deprecated: DocuSign GSA trigger deferred to Phase 2, tracked in INS-0023)

## Solution

Create 'Group_Employer' Record Type on Account object and build page layout 'Group_Employer_Account_Layout' via Object Manager. Add custom fields to Account object for Group_ID__c, SIC_Code__c, Employer_Size__c, Domicile_State__c, ERISA_Exempt__c, Broker_Of_Record__c, Onboarding_Status__c. Configure Lightning Path on Onboarding_Status__c via Path Settings in Setup, including Key Fields and Guidance for Success text for each stage. Deploy Group_Service_Agreement__c custom object with all required fields. Build Record-Triggered Flow 'RTF_GSA_Executed_Update_Account' (After Save, fires when GSA__c.Status__c changes to 'Executed') using Get Records to fetch parent Account, Decision element to check current Onboarding_Status__c = 'Underwriting Review', then Update Records to set 'GSA Executed'. Assign 'Group_Employer' Record Type visibility to PSG 'Broker_Relations_Coordinator' (new PSG to be created extending INS-0008 scope). Validate with UAT script covering full onboarding status progression.

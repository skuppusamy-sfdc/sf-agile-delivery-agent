## Description

As a Claims Operations Manager, I need the core Salesforce custom objects (Policy__c, Claim__c, Claim_Line__c, Payment__c, and Explanation_of_Benefits__c) deployed with all required fields and relationships so that the platform can support end-to-end claims processing workflows from FNOL through adjudication and payment disbursement.

## Acceptance Criteria

Given the Salesforce org is provisioned for Phase 1, when the data model deployment is complete, then the following objects must exist with the specified fields:

1. Given Policy__c is created, when viewed in Schema Builder, then it must contain: Policy__c.Policy_Number__c (Text, Unique, External ID), Policy__c.LOB__c (Picklist: Medical|Dental|Vision|Workers_Comp), Policy__c.Effective_Date__c (Date), Policy__c.Termination_Date__c (Date), Policy__c.Status__c (Picklist: Active|Lapsed|Cancelled|Pending), Policy__c.TPA__c (Lookup: Account), Policy__c.Group_Service_Agreement__c (Lookup: Group_Service_Agreement__c).

2. Given Claim__c is created, when viewed in Schema Builder, then it must contain: Claim__c.Claim_Number__c (Auto Number, External ID), Claim__c.Policy__c (Master-Detail: Policy__c), Claim__c.Status__c (Picklist: New|FNOL_Received|In_Review|Pending_Info|Adjudicated|Denied|Closed), Claim__c.Date_of_Service__c (Date), Claim__c.Total_Billed_Amount__c (Currency), Claim__c.Total_Allowed_Amount__c (Currency), Claim__c.Total_Paid_Amount__c (Currency, Roll-Up from Payment__c), Claim__c.LOB__c (Picklist: Medical|Dental|Vision|Workers_Comp), Claim__c.SIU_Flag__c (Checkbox, default false).

3. Given Claim_Line__c is created, when related to a Claim__c record, then it must contain: Claim_Line__c.Claim__c (Master-Detail: Claim__c), Claim_Line__c.CPT_Code__c (Text, 10), Claim_Line__c.ICD10_Primary__c (Text, 10), Claim_Line__c.Billed_Amount__c (Currency), Claim_Line__c.Allowed_Amount__c (Currency), Claim_Line__c.Line_Status__c (Picklist: Pending|Approved|Denied|Adjusted).

4. Given Payment__c is created, when related to a Claim__c, then it must contain: Payment__c.Claim__c (Master-Detail: Claim__c), Payment__c.Payment_Amount__c (Currency), Payment__c.Payment_Date__c (Date), Payment__c.Payment_Method__c (Picklist: EFT|Check|Virtual_Card), Payment__c.ERA_Reference__c (Text, 50), Payment__c.Status__c (Picklist: Scheduled|Issued|Cleared|Voided).

5. -Given EOB__c uses a junction object between Claim__c and Provider__c- Given Explanation_of_Benefits__c is created, when related to Claim__c, then it must contain: EOB__c.Claim__c (Lookup: Claim__c), EOB__c.Generated_Date__c (DateTime), EOB__c.EOB_Document_URL__c (URL), EOB__c.Delivery_Method__c (Picklist: Mail|Email|Portal).

6. Given all objects are deployed, when a system admin reviews field-level security, then all currency fields must be read-only for the 'Claims_Viewer' profile and editable for 'Claims_Adjudicator' profile.

## Solution

Deploy metadata via Salesforce DX (SFDX) scratch org validated against full sandbox. Use Custom Metadata Types for LOB configuration values to avoid hardcoded picklist dependencies. Implement Record-Triggered Flow on Claim__c to auto-populate Claim__c.LOB__c from parent Policy__c.LOB__c on record creation. Roll-up summary field on Claim__c.Total_Paid_Amount__c aggregates Payment__c.Payment_Amount__c where Payment__c.Status__c = 'Issued' OR 'Cleared'. All object deployments tracked via change set INS-PHASE1-CORE-OBJECTS. Validation rules added post-deployment in INS-0025.

## Description

As a Broker Relations Coordinator, I need a structured Account hierarchy model for Group/Employer clients and a configured Group_Service_Agreement__c (GSA) object so that I can onboard employer groups, associate their subsidiary divisions, and link their coverage agreements — enabling accurate policy issuance, billing, and downstream claims adjudication for group members.

This story covers the Account hierarchy configuration (Business Account record types), GSA object schema, and the relationship between GSA and Policy__c.

## Acceptance Criteria

Given the Account object is configured for Phase 1,
When a new Account is created with Record Type 'Employer_Group',
Then the following fields are available: Group_ID__c (Text, Unique, External ID), FEIN__c (Text, 9 chars, encrypted), Group_Size__c (Number), SIC_Code__c (Text), Renewal_Month__c (Picklist: Jan|Feb|...|Dec), Broker_of_Record__c (Lookup to Account, Record Type = 'Broker_Agency'), Effective_Date__c (Date), Status__c (Picklist: Prospect|Active|Lapsed|Terminated).

Given an Employer_Group Account exists,
When a child Account with Record Type 'Employer_Division' is created,
Then Account.ParentId must reference an Employer_Group Account — a Validation Rule 'VR_Division_Requires_Group_Parent' enforces this.

Given the Group_Service_Agreement__c object is deployed,
When a GSA record is created,
Then the following fields exist: GSA_Number__c (Auto-Number, format GSA-{0000000}), Employer_Group__c (Lookup to Account, Record Type = Employer_Group, required), LOB__c (Picklist: Health|Dental|Vision|Life, Multi-select), Effective_Date__c (Date, required), Termination_Date__c (Date), Status__c (Picklist: Draft|Active|Expired|Terminated, default='Draft'), Minimum_Participation_Pct__c (Percent), Premium_Rate_Basis__c (Picklist: Per Employee|Per Member|Composite), Docusign_Envelope_ID__c (Text).

Given a GSA record has Status__c = 'Active',
When a new Policy__c is created and linked via Policy__c.GSA__c,
Then Policy__c.LOB__c must match one of the values in GSA.LOB__c — enforced by a Validation Rule 'VR_Policy_LOB_Matches_GSA'.

Given a GSA is in 'Draft' status,
When the Broker Relations Coordinator clicks 'Send for Signature' action button,
Then a DocuSign envelope is initiated via the DocuSign for Salesforce managed package, Docusign_Envelope_ID__c is populated, and GSA.Status__c updates to 'Pending Signature'.

## Solution

Account Record Types configured via Setup: 'Employer_Group' and 'Employer_Division' added to Business Account page layouts. FEIN__c encrypted using Salesforce Shield Platform Encryption (deterministic encryption to allow filtering). Validation Rule 'VR_Division_Requires_Group_Parent' uses VLOOKUP on Account.RecordType.DeveloperName. Group_Service_Agreement__c deployed via metadata package. DocuSign integration uses DocuSign for Salesforce managed package (v8.x) — 'Send for Signature' implemented as a Quick Action invoking an Apex-invocable method 'DocuSignEnvelopeService.sendGSAEnvelope()' that constructs the envelope via DocuSign REST API using Named Credential NC_DocuSign. Status update on envelope completion handled by DocuSign callback Apex class updating GSA.Status__c = 'Active'.

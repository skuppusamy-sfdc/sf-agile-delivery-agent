## Description

As a Provider Network Specialist, I need the Provider_Network_Agreement__c (PNA) and Fee_Schedule__c objects to be fully configured with all Phase 1 fields so that network contracting workflows, fee schedule lookups during claims adjudication, and Conga CLM contract generation can be built on a reliable data model.

Extends INS-0049 (Provider and Facility schema). PNA is the junction between a Provider/Facility Account and the network contract terms. Fee_Schedule__c stores CPT-level reimbursement rates referenced during Claim_Line__c adjudication.

## Acceptance Criteria

Given the Provider_Network_Agreement__c object is created,
When a developer reviews the object in Setup,
Then the following fields exist:
  - PNA__c.Provider__c (Lookup to Account, required)
  - PNA__c.Network__c (Picklist: Acme PPO, Acme HMO, Acme EPO, Acme Dental, Acme Vision)
  - PNA__c.Agreement_Status__c (Picklist: Draft, Pending Signature, Active, Expired, Terminated, Suspended)
  - PNA__c.Effective_Date__c (Date, required)
  - PNA__c.Termination_Date__c (Date)
  - PNA__c.Contract_Type__c (Picklist: Full Risk, Partial Risk, Fee for Service, Capitation, Value-Based)
  - PNA__c.Conga_Contract_ID__c (Text 50, external ID)
  - PNA__c.DocuSign_Envelope_ID__c (Text 100)
  - PNA__c.Signed_Date__c (Date)
  - PNA__c.Primary_Contact__c (Lookup to Contact)
  - PNA__c.Fee_Schedule__c (Lookup to Fee_Schedule__c)
  - PNA__c.Auto_Renewal__c (Checkbox)
  - PNA__c.Renewal_Notice_Days__c (Number, default 90)
  - PNA__c.LOB__c (Picklist: Medical, Dental, Vision, Pharmacy, Behavioral Health)
  - PNA__c.MCO_Reference__c (Text 50)

Given the Fee_Schedule__c object is created,
When a developer reviews the object,
Then the following fields exist:
  - Fee_Schedule__c.Schedule_Name__c (Text 100, required)
  - Fee_Schedule__c.Effective_Date__c (Date, required)
  - Fee_Schedule__c.Expiration_Date__c (Date)
  - Fee_Schedule__c.LOB__c (Picklist: Medical, Dental, Vision)
  - Fee_Schedule__c.Schedule_Type__c (Picklist: Medicare-Based, Negotiated, UCR, Capitation)
  - Fee_Schedule__c.Medicare_Percentage__c (Percent, 2 decimal places)
  - Fee_Schedule__c.CPT_Code__c (Text 10)
  - Fee_Schedule__c.Allowed_Amount__c (Currency)
  - Fee_Schedule__c.Modifier_Adjustment__c (Percent)
  - Fee_Schedule__c.Geographic_Region__c (Text 50)
  - Fee_Schedule__c.PNA__c (Lookup to Provider_Network_Agreement__c)

Given a PNA record transitions to 'Active' status,
When Agreement_Status__c is set to 'Active',
Then Effective_Date__c must be populated and not in the past (validation rule).

Given a PNA record is 'Active' and Termination_Date__c is reached,
When the daily scheduled job runs,
Then Agreement_Status__c is automatically updated to 'Expired'.

Given a Fee_Schedule__c record is linked to a PNA,
When Claim_Line__c.CPT_Code__c matches Fee_Schedule__c.CPT_Code__c and the PNA is Active,
Then Claim_Line__c.Allowed_Amount__c can be auto-populated (lookup logic covered in FeeScheduleEngine stories).

Given reporting requirements,
When a Provider Network Specialist runs the 'Expiring Agreements' report,
Then all PNA records with Termination_Date__c within 60 days and Agreement_Status__c = 'Active' are returned.

## Solution

1. Create PNA__c and Fee_Schedule__c objects in ins-network-contracting package via source-tracked development.
2. PNA__c Lookup to Account (not Master-Detail) to allow provider record independence from PNA lifecycle.
3. Validation rule on PNA__c: AND(ISPICKVAL(Agreement_Status__c,'Active'), ISBLANK(Effective_Date__c)) blocks save.
4. Scheduled Apex job (PNAExpirationBatch.cls) runs nightly; queries PNA records where Termination_Date__c <= TODAY() and Agreement_Status__c = 'Active'; updates status to 'Expired' in bulk (batch size 200).
5. Conga_Contract_ID__c and DocuSign_Envelope_ID__c set as External IDs to support future Conga CLM and DocuSign integration callbacks.
6. 'Expiring Agreements' report built in Reports tab using PNA__c report type filtered on Termination_Date__c and Agreement_Status__c.
7. Fee_Schedule__c designed as a child of PNA__c but also queryable independently for bulk CPT rate lookups.

## Description

As a Credentialing Specialist, I need the Credentialing_Case__c object configured and integrated with CAQH ProView and NPPES so that I can initiate provider credentialing workflows, pull real-time NPI registry data, and track credential verification status end-to-end within Salesforce, reducing manual data entry and credentialing cycle time.

## Acceptance Criteria

Given the Credentialing_Case__c object is deployed,
When a new Credentialing Case is created,
Then it must contain: Provider__c (Lookup to Account, RecordType = 'Provider'), Case_Type__c (Picklist: Initial Credentialing|Re-Credentialing|Delegated Review), Status__c (Picklist: Initiated|Pending CAQH|Under Review|Approved|Denied|Deferred), NPI__c (Text 10), CAQH_Provider_ID__c (Text 20), License_State__c (Picklist of US States), License_Expiry_Date__c (Date), Board_Certification__c (Text 100), Credentialing_Specialist__c (Lookup to User), Due_Date__c (Date, auto-populated as TODAY() + 30 on creation).

Given a Credentialing_Case__c record has NPI__c populated,
When the Credentialing Specialist clicks 'Verify NPI via NPPES',
Then a callout is made to the NPPES NPI Registry API (https://npiregistry.cms.hhs.gov/api/), the response populates Provider__c Account fields: NPI_Status__c, Provider_First_Name__c, Provider_Last_Name__c, Primary_Taxonomy_Code__c, and a timestamp field NPI_Last_Verified__c is stamped on the Credentialing_Case__c.

Given a Credentialing_Case__c has CAQH_Provider_ID__c populated,
When the Credentialing Specialist clicks 'Pull CAQH Data',
Then a callout to CAQH ProView API retrieves the provider attestation status, and Credentialing_Case__c.CAQH_Attestation_Status__c is updated (Complete|Incomplete|Expired) and Credentialing_Case__c.Status__c transitions from 'Initiated' to 'Pending CAQH' if attestation is 'Incomplete'.

Given Credentialing_Case__c.License_Expiry_Date__c is within 90 days of TODAY(),
When the record is saved or the nightly batch runs,
Then a high-priority Task is created for Credentialing_Specialist__c with Subject: 'License Expiry Alert — Action Required within 90 Days' and a Chatter mention is posted on the Provider Account.

Given a Credentialing_Case__c.Status__c is set to 'Approved',
When the record is saved,
Then the parent Provider Account field Credentialing_Status__c is updated to 'Credentialed' and Credentialed_Since__c is stamped with TODAY().

-Given a Credentialing Case is Approved, When a PNA exists for the Provider, Then PNA.Contract_Status__c auto-transitions to 'Active'- (deprecated: PNA activation requires manual review per compliance team; removed from this story scope)

## Solution

Credentialing_Case__c deployed via SFDX. NPPES integration uses Apex HttpCallout class (NPPESIntegrationService) invoked via Lightning Quick Action on the record page. CAQH ProView integration via separate Apex service class (CAQHIntegrationService) using Named Credential (NC_CAQH). Both callout classes implement Queueable interface to avoid governor limit conflicts. License expiry alert handled by Record-Triggered Flow (After Save) using a scheduled path 90 days before License_Expiry_Date__c. Status transition to 'Approved' triggers a second Record-Triggered Flow that performs a cross-object field update on the parent Account via Flow's Update Records element. All callout responses logged to an Integration_Log__c custom object for audit trail.

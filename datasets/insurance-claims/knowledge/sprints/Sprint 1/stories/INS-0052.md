## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can immediately view a provider's NPI, TIN, current credentialing status, and all active Provider Network Agreements (PNAs) without navigating to multiple related lists or external systems.

This accelerates provider inquiry resolution and reduces time-to-answer during provider calls. Extends foundational Person Account model established in INS-0051.

## Acceptance Criteria

Given a Provider__c (Person Account) record is open in Lightning,
When the Provider 360 FlexCard renders,
Then the header block displays: NPI__c (formatted as 10-digit string), TIN__c (masked as ***-**-XXXX for non-admin profiles), Provider_Type__c, and Specialty__c.

Given the FlexCard is rendered,
When the Credentialing section loads,
Then it shows the most recent Credentialing_Case__c.Status__c and Credentialing_Case__c.Expiration_Date__c, with a red indicator if Expiration_Date__c is within 90 days of today.

Given the FlexCard is rendered,
When the Network Agreements section loads,
Then it lists all Provider_Network_Agreement__c records where PNA.Provider__c = current record Id AND PNA.Status__c = 'Active', displaying: PNA.Agreement_Number__c, PNA.Network_Name__c, PNA.Effective_Date__c, PNA.Expiration_Date__c.

Given a provider has zero active PNAs,
When the Network Agreements section loads,
Then a styled empty-state message reads 'No Active Network Agreements — contact Provider Network Contracting.'

Given the FlexCard is viewed by a Credentialing Specialist profile,
When TIN__c is displayed,
Then the full unmasked TIN is shown per field-level security rules defined in INS-0053.

-Given the FlexCard is rendered, When the user clicks NPI, Then it deep-links to NPPES external lookup- (deprecated: external NPPES link deferred to Phase 2 per scope review)

## Solution

Build using OmniStudio FlexCard with two child DataRaptor Extract actions: (1) DR_Extract_ProviderHeader pulling from Account (Person Account) object for NPI__c, TIN__c, Provider_Type__c, Specialty__c; (2) DR_Extract_ActivePNAs querying Provider_Network_Agreement__c with SOQL filter on Provider__c = :recordId AND Status__c = 'Active'. TIN masking implemented via DataRaptor formula node using SUBSTITUTE() conditional on running user's Profile. Credentialing section uses a separate DataRaptor querying Credentialing_Case__c ordered by CreatedDate DESC LIMIT 1. FlexCard deployed to Provider__c Lightning Record Page via App Builder. Conditional styling for 90-day expiry uses FlexCard State with a date-diff formula node.

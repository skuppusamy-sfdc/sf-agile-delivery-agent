## Description

As a Claims Operations Manager, I need a Record-Triggered Flow on Claim__c that automatically identifies Coordination of Benefits (COB) scenarios and populates the relevant COB fields when a claimant has more than one active Policy__c record so that adjusters are immediately alerted to potential duplicate coverage and can apply correct payment sequencing without manual lookup.

COB logic must follow the birthday rule for dependent claims and the 'active employment' rule for primary/secondary determination.

## Acceptance Criteria

Given a new Claim__c record is created with a Claimant_Contact__c value,
When the Record-Triggered Flow fires on Create,
Then the flow queries Policy__c records where Policy__c.Insured_Contact__c = Claim__c.Claimant_Contact__c AND Policy__c.Status__c = 'Active' AND Policy__c.Expiration_Date__c >= TODAY.

Given the query returns more than one active Policy,
When the COB evaluation logic runs,
Then Claim__c.COB_Indicator__c is set to TRUE, Claim__c.Primary_Policy__c is populated with the policy determined primary, and Claim__c.Secondary_Policy__c is populated with the secondary policy.

Given Claim__c.COB_Indicator__c is set to TRUE,
When the record is saved,
Then a Task is created assigned to the owning adjuster with Subject = 'COB Review Required — [Claim_Number__c]' and Due Date = TODAY + 2 business days.

Given the query returns exactly one active Policy,
When the flow completes,
Then Claim__c.COB_Indicator__c remains FALSE and no Task is created.

Given a Claim__c record is updated (not created),
When the trigger evaluates entry conditions,
Then the flow only re-executes if Claimant_Contact__c has changed, preventing recursive execution.

## Solution

Record-Triggered Flow on Claim__c, trigger: After Save, entry condition: {!$Record.Claimant_Contact__c} IS NOT NULL. Flow uses Get Records element to query Policy__c with filter on Insured_Contact__c and Status__c. Decision element branches on collection size > 1. COB primary/secondary determination uses a Loop with Assignment elements applying birthday-rule date comparison (Policy__c.Primary_Insured_DOB__c). Update Records element sets COB fields in a separate 'before-save' scheduled path to avoid recursion. Task creation via Create Records element. Flow fault path sends Platform Event 'COB_Evaluation_Error__e' for monitoring. Recursion guard via Process Metadata custom setting 'Flow_Recursion_Guard__mdt'.

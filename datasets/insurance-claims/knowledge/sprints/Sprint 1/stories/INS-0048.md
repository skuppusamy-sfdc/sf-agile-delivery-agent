# INS-0048: Create Policy__c object schema with Policyholder relationship and coverage detail fields for Phase 1

## Description

As a Claims Operations Manager, I need the Policy__c object to be fully configured with policyholder relationships, coverage dates, LOB, group/employer linkage, and deductible/out-of-pocket tracking fields so that claims adjudicators can validate coverage at the time of claim intake and COB logic can reference accurate policy data.

This story covers Policy__c schema only. The Policyholder 360 FlexCard and related Lightning page are in a later sprint.

## Acceptance Criteria

Given the Policy__c object is created,
When a developer inspects the object metadata,
Then the following fields exist:
  - Policy__c.Policy_Number__c (Auto Number, format POL-{00000000})
  - Policy__c.Policyholder__c (Lookup to Contact)
  - Policy__c.Group_Account__c (Lookup to Account where RecordType = Employer_Group)
  - Policy__c.Effective_Date__c (Date, required)
  - Policy__c.Termination_Date__c (Date)
  - Policy__c.Status__c (Picklist: Active, Lapsed, Terminated, Pending, COBRA)
  - Policy__c.LOB__c (Picklist: Medical, Dental, Vision, Pharmacy, Behavioral Health)
  - Policy__c.Plan_Type__c (Picklist: HMO, PPO, EPO, HDHP, Indemnity)
  - Policy__c.Individual_Deductible__c (Currency)
  - Policy__c.Family_Deductible__c (Currency)
  - Policy__c.Individual_Deductible_Met__c (Currency, default 0)
  - Policy__c.Family_Deductible_Met__c (Currency, default 0)
  - Policy__c.Individual_OOP_Max__c (Currency)
  - Policy__c.Family_OOP_Max__c (Currency)
  - Policy__c.Individual_OOP_Met__c (Currency, default 0)
  - Policy__c.Broker__c (Lookup to Account where RecordType = Broker)
  - Policy__c.TPA__c (Lookup to Account where RecordType = TPA)
  - Policy__c.COB_Order__c (Picklist: Primary, Secondary, Tertiary)

Given a Claim__c record is linked to a Policy__c,
When the claim's Date_of_Service__c falls outside Policy__c.Effective_Date__c and Policy__c.Termination_Date__c,
Then a validation rule on Claim__c prevents saving and displays: 'Date of Service falls outside the active policy period. Please verify coverage dates.'

Given Policy__c.Status__c is set to 'Terminated',
When a user attempts to create a new Claim__c linked to that Policy__c,
Then the same validation rule blocks the save.

Given field history tracking requirements,
When Field History Tracking is enabled on Policy__c,
Then Status__c, Termination_Date__c, Individual_Deductible_Met__c, and Family_Deductible_Met__c are tracked.

Given search and reporting requirements,
When Policy__c is added to global search layouts,
Then Policy_Number__c and Policyholder__c.Name are displayed in search results.

## Solution

1. Create Policy__c object via source-tracked scratch org development in ins-policy-core package.
2. Lookups to Contact (Policyholder__c), Account (Group, Broker, TPA) use standard relationship fields with filter criteria enforced via lookup filters on RecordType.
3. Validation rule on Claim__c cross-object referencing Policy__c.Effective_Date__c and Policy__c.Termination_Date__c using DATEVALUE() formula.
4. Field History Tracking enabled in object metadata.
5. Search layout updated via searchLayouts metadata XML.
6. COB_Order__c picklist will feed COB adjudication logic in a future sprint (INS-0063 placeholder).
7. All metadata deployed via Salesforce CLI; delta deployment validated against INS-0046 schema to ensure no field name collisions.

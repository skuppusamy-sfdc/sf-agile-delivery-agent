## Description

As a Claims Operations Manager, I need the Salesforce platform to automatically calculate the Allowed Amount on each Claim_Line__c record by looking up the applicable CPT code rate from the associated Fee_Schedule__c when a claim line is created or updated, so that adjudicators do not need to manually reference fee schedule documents and allowed amount calculations are consistent and auditable.

Fee schedules are linked to providers via the Provider_Network_Agreement__c. The lookup chain is: Claim__c → Policy__c → Provider__c → PNA__c → Fee_Schedule__c → Fee_Schedule_Line__c (CPT code rates). Multiple fee schedules may exist per provider; the most recently effective one for the claim's LOB must be selected.

## Acceptance Criteria

Given a Claim_Line__c record is created with a valid CPT_Code__c value,
When the record is saved,
Then the system must traverse the lookup chain: Claim_Line__c.Claim__c → Claim__c.Policy__c → Policy__c.Provider__c → active PNA__c (Status__c = 'Executed') → Fee_Schedule__c (where Fee_Schedule__c.LOB__c matches Claim__c.LOB__c and Effective_Date__c <= Service_Date__c, selecting the most recent) → Fee_Schedule_Line__c where Fee_Schedule_Line__c.CPT_Code__c = Claim_Line__c.CPT_Code__c.

Given a matching Fee_Schedule_Line__c record is found,
When the allowed amount is calculated,
Then Claim_Line__c.Allowed_Amount__c = Fee_Schedule_Line__c.Rate__c * Claim_Line__c.Units__c, rounded to 2 decimal places.

Given a matching Fee_Schedule_Line__c record is NOT found for the given CPT code,
When the save completes,
Then:
  - Claim_Line__c.Allowed_Amount__c must be set to 0.00
  - Claim_Line__c.Line_Status__c must be set to 'Denied'
  - Claim_Line__c.Denial_Reason__c must be set to 'CPT Code Not on Fee Schedule'
  - A warning toast notification must appear to the adjudicator: 'CPT Code [value] not found on applicable Fee Schedule. Line denied.'

Given a Claim_Line__c allowed amount is calculated,
When the parent Claim__c record is viewed,
Then Claim__c.Total_Allowed_Amount__c (rollup summary) must reflect the sum of all child Claim_Line__c.Allowed_Amount__c values.

Given an adjudicator manually overrides Claim_Line__c.Allowed_Amount__c,
When the override is saved,
Then Claim_Line__c.Manual_Override_Flag__c must be set to true and Claim_Line__c.Override_Reason__c must be required (cannot be blank).

## Solution

Allowed amount calculation implemented in an Apex Trigger (ClaimLineTrigger) with a handler class (ClaimLineAllowedAmountHandler) executing after insert and after update on Claim_Line__c. Handler performs a SOQL query to traverse the lookup chain using a single aggregate query with ORDER BY Fee_Schedule__c.Effective_Date__c DESC LIMIT 1 to select the most recent applicable fee schedule. Results cached in a Map<String, Decimal> (CPT code → rate) to handle bulk claim line inserts (e.g., batch adjudication scenarios) within governor limits. Denial logic sets Line_Status__c and Denial_Reason__c fields in the same Apex DML operation. Warning toast for missing CPT codes surfaced via a Platform Event (ClaimLineAlert__e) published from the trigger and subscribed to by an LWC (claimLineAlertBanner) on the Claim__c record page. Manual override detection uses Trigger.isUpdate with field change check on Allowed_Amount__c.

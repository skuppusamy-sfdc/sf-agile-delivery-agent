## Description

As a Claims Operations Manager, I need the core Salesforce custom object schema (Policy__c, Claim__c, Claim_Line__c, Payment__c, EOB__c) to be deployed and configured so that the claims processing platform has a stable data foundation for all downstream development in Phase 1 and beyond.

## Acceptance Criteria

Given the org is a freshly provisioned Salesforce Health Cloud sandbox,
When the deployment package is executed,
Then Policy__c exists with fields: Policy_Number__c (Text 20, Unique, External ID), LOB__c (Picklist: Medical|Dental|Vision|Workers Comp), Effective_Date__c (Date), Termination_Date__c (Date), TPA__c (Lookup to Account), Status__c (Picklist: Active|Lapsed|Cancelled|Pending).

Given Policy__c is deployed,
When a Claim__c record is created,
Then Claim__c has required lookups: Policy__c (Master-Detail), Provider__c (Lookup to Account), Facility__c (Lookup to Account), and fields: FNOL_Date__c (DateTime), Claim_Status__c (Picklist: New|In Review|Adjudicated|Denied|Closed|Pended), Total_Billed_Amount__c (Currency), Total_Allowed_Amount__c (Currency), Total_Paid_Amount__c (Currency, roll-up from Payment__c).

Given Claim__c is deployed,
When Claim_Line__c records are created,
Then Claim_Line__c enforces Master-Detail to Claim__c, and includes: CPT_Code__c (Text 10), ICD10_Primary__c (Text 10), Line_Status__c (Picklist: Pending|Approved|Denied|Adjusted), Billed_Amount__c (Currency), Allowed_Amount__c (Currency), Copay__c (Currency), Coinsurance__c (Percent).

Given all objects are deployed,
When a validation rule check is run,
Then Termination_Date__c on Policy__c must be greater than or equal to Effective_Date__c (error message: 'Termination Date cannot precede Effective Date').

-Given EOB__c is standalone object, When linked to Claim__c, Then use a Lookup relationship- (deprecated: EOB__c now uses Master-Detail to Claim__c per revised data model INS-0003)

Given EOB__c is deployed with Master-Detail to Claim__c,
When the schema is validated,
Then EOB__c includes: EOB_Number__c (Auto Number), ERA_Reference__c (Text 30), Remittance_Date__c (Date), EOB_Status__c (Picklist: Draft|Sent|Acknowledged).

## Solution

Deploy metadata via Salesforce DX scratch org pipeline. Custom objects created as XML metadata components in the force-app/main/default/objects directory. Validation rules implemented declaratively. Roll-up summary field (Total_Paid_Amount__c on Claim__c) aggregates SUM of Payment__c.Amount__c where Payment__c.Status__c = 'Processed'. Field-level security configured at Profile level as interim measure; will be migrated to Permission Set Groups in INS-0010. Deployment validated via Apex unit tests asserting field existence using Schema.SObjectType introspection (minimum 75% coverage).

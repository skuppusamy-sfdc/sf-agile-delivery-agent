## Description

As a Provider Network Specialist, I need a Provider 360 FlexCard displayed on the Provider (Account) record page so that I can instantly view a provider's NPI, TIN, active PNA status, credentialing case status, network participation summary, and recent claims activity without navigating across multiple objects. This accelerates contract renewal and dispute resolution workflows.

## Acceptance Criteria

Given a Provider__c (Person Account) record is opened by a user with PSG 'Provider_Network_Specialist',
When the Provider 360 FlexCard loads,
Then the card displays: Provider_NPI__c, Provider_TIN__c (masked as XXX-XX-####), Primary_Specialty__c, and CAQH_Profile_ID__c.

Given the FlexCard is rendered,
When the PNA section is evaluated,
Then all active Provider_Network_Agreement__c records (PNA__c.Status__c = 'Active') linked to the Provider are listed with columns: Network_Name__c, Contract_Effective_Date__c, Contract_Termination_Date__c, Fee_Schedule__c (lookup name).

Given the FlexCard is rendered,
When the Credentialing section loads,
Then the most recent Credentialing_Case__c record is shown with fields: Case_Number__c, Credentialing_Status__c, Expiration_Date__c, and a color-coded badge (Green = Active, Yellow = Expiring within 90 days, Red = Expired).

Given the FlexCard is rendered,
When the Claims Activity section loads,
Then a count of Claim__c records linked to the provider in the last 12 months is shown, segmented by Claim_Status__c values, sourced via DataRaptor Extract.

Given a provider has no active PNA records,
When the PNA section renders,
Then a message displays: 'No active network agreements found. Initiate contracting via New PNA button.'

-Given FlexCard is on the Account page layout, When user is a Claims Ops Manager, Then FlexCard is also visible to that role- (deprecated: Claims Ops Manager FlexCard is a separate component per UX review, tracked in INS-0017)

## Solution

Build OmniStudio FlexCard 'Provider_360_FC' with child cards for PNA, Credentialing, and Claims sections. DataRaptor Extract 'DR_Get_Provider_360' queries Provider__c, Provider_Network_Agreement__c, Credentialing_Case__c, and Claim__c via SOQL with relationship traversal. TIN masking applied via DataRaptor formula transform. Credentialing badge color logic implemented using FlexCard State configuration with conditional CSS classes (omnistudio-theme). FlexCard activated on the Account Lightning Record Page using App Builder, restricted by PSG via Visibility Rules. Claims Activity aggregation uses a DataRaptor Turbo Extract with GROUP BY Claim_Status__c for performance on large datasets.

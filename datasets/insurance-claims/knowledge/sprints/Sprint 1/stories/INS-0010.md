## Description

As a Compliance Officer, I need Salesforce Sharing Rules and a Role Hierarchy configured to enforce multi-region data segregation on Claim__c and Policy__c records so that claims adjusters in the Northeast region cannot view or edit claims belonging to the Southeast or West regions, satisfying our internal data governance policy and supporting future state-level compliance requirements.

## Acceptance Criteria

Given the Role Hierarchy is configured,
When it is reviewed in Setup,
Then the hierarchy follows: CEO > VP_Claims > [Regional_Claims_Director_NE | Regional_Claims_Director_SE | Regional_Claims_Director_West] > [Claims_Adjuster_NE | Claims_Adjuster_SE | Claims_Adjuster_West].

Given Claim__c OWD is set to 'Private',
When a Claims_Adjuster_NE user attempts to view a Claim__c where Region__c = 'Southeast',
Then the record is not returned in list views, search results, or SOQL queries run in user context.

Given the Criteria-Based Sharing Rule 'Share_NE_Claims_To_NE_Adjusters' is active,
When Claim__c.Region__c = 'Northeast',
Then the record is shared with the Role 'Claims_Adjuster_NE' with Read/Write access.

Given Policy__c OWD is set to 'Private',
When a Regional Claims Director accesses their region's Policy__c records,
Then they see all policies where Policy__c.Region__c matches their role's region, and the role hierarchy grants upward visibility to VP_Claims and CEO.

Given sharing rules are deployed,
When a Salesforce Administrator runs the 'Sharing Rule Audit' custom report,
Then all three regional Criteria-Based Sharing Rules are listed as Active with correct shared-to Role and access level.

## Solution

Configure Organization-Wide Defaults (OWD) for Claim__c and Policy__c to 'Private' via Setup > Sharing Settings. Build Role Hierarchy with 8 roles as defined in ACs. Create three Criteria-Based Sharing Rules per object (6 total) using Setup > Sharing Rules, criteria on Region__c picklist field. Add Region__c (Picklist: Northeast|Southeast|West|National) to both Claim__c and Policy__c if not already present from INS-0007 schema deployment. For edge cases where a claim spans regions (MCO contracts), implement an Apex Managed Sharing class triggered by a Record-Triggered Flow on Claim__c after-save, inserting ClaimShare records programmatically for multi-region exposure. Apex sharing class must be covered by unit tests asserting Share row existence.

## Description

As a Provider Network Specialist, I need automated notifications when a Provider_Network_Agreement__c (PNA) is approaching expiration (90, 60, and 30 days prior) so that my team can proactively initiate contract renewal workflows and prevent providers from inadvertently becoming out-of-network, which would disrupt claims adjudication.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA_Status__c = 'Active'
When today's date is exactly 90 days before Termination_Date__c
Then a Salesforce In-App Notification and an Email Alert are sent to the Provider_Network_Agreement__c.Owner with subject: 'PNA Expiring in 90 Days — [Provider Name] — [PNA_Number__c]'

Given the same PNA record
When today's date is exactly 60 days before Termination_Date__c
Then a Task is auto-created on the PNA record with Subject = 'Initiate Renewal — PNA Expiring in 60 Days', Priority = 'High', Due_Date = today + 14 days, Assigned_To = PNA Owner

Given the same PNA record
When today's date is exactly 30 days before Termination_Date__c
Then an Escalation Email is sent to both the PNA Owner and their manager (resolved via Role Hierarchy), and PNA_Renewal_Status__c is auto-set to 'Escalated' if still = 'Not Started'

Given a PNA renewal has already been initiated (PNA_Renewal_Status__c = 'In Progress' or 'Completed')
When the scheduled job evaluates the 30-day threshold
Then NO escalation notification is sent and no status override occurs

Given a PNA record is terminated early (PNA_Status__c manually set to 'Terminated')
When the scheduled job runs
Then the PNA is excluded from all expiry notification logic

-Given notifications should also trigger for Group_Service_Agreement__c (GSA) expiry using the same thresholds
Then GSA expiry logic should be included in the same scheduled job- (superseded — GSA expiry automation scoped to INS-0058 in Sprint 2)

## Solution

Scheduled Apex class `PNAExpiryNotificationBatch` implementing `Database.Batchable<SObject>` and `Schedulable`, scheduled nightly at 6:00 AM UTC via Apex Scheduler. SOQL query filters PNA records where PNA_Status__c = 'Active' AND Termination_Date__c IN (TODAY+30, TODAY+60, TODAY+90). Batch processes each threshold in separate if-blocks. Email Alerts sent via `Messaging.sendEmail()` using custom EmailTemplate records. In-App notifications sent via `Messaging.CustomNotification` with a custom notification type 'PNA_Expiry_Alert'. Task creation via standard Task SObject insert. Manager lookup resolved via `UserRole.ParentRoleId` traversal in a helper method. All notification records logged to a custom Notification_Log__c object for audit. Idempotency enforced by checking for existing open Tasks with matching Subject + WhatId before inserting.

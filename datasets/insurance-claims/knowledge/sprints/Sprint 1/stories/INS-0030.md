## Description

As a Claims Operations Manager, I need the core Salesforce custom objects (Policy__c, Claim__c, Claim_Line__c, Payment__c, EOB__c) deployed with correct field schema, relationships, and validation rules so that the platform can support end-to-end claims processing workflows from FNOL through payment disbursement.

## Acceptance Criteria

Given the org is a fresh Salesforce Health Cloud sandbox,
When the deployment package is applied,
Then Policy__c exists with fields: Policy_Number__c (Unique, External ID), LOB__c (Picklist: Medical|Dental|Vision|Workers Comp), Effective_Date__c, Termination_Date__c, TPA_Indicator__c (Checkbox), Group_ID__c (Text 18).

Given Policy__c is deployed,
When Claim__c schema is validated,
Then Claim__c exists with fields: Claim_Number__c (Auto-Number: CLM-{0000000}), FNOL_Date__c (Date), Status__c (Picklist: New|In Review|Adjudicated|Denied|Paid|Void), Policy__c (Master-Detail to Policy__c), Claimant__c (Lookup to Contact), Total_Billed_Amount__c (Currency), Total_Allowed_Amount__c (Currency), COB_Applied__c (Checkbox), SIU_Referral__c (Checkbox).

Given Claim__c is deployed,
When Claim_Line__c schema is validated,
Then Claim_Line__c exists with: Claim__c (Master-Detail), CPT_Code__c (Text 10), ICD10_Primary__c (Text 10), ICD10_Secondary__c (Text 10), Billed_Amount__c (Currency), Allowed_Amount__c (Currency), Line_Status__c (Picklist: Pending|Approved|Denied|Bundled), Service_Date__c (Date), Rendering_Provider__c (Lookup to Account).

Given all objects are deployed,
When page layouts and compact layouts are configured,
Then each object has a primary page layout assigned to the System Administrator profile and compact layout shows at minimum: record name, Status__c, and primary currency field.

-Given Payment__c is deployed, When linked to Claim__c, Then Payment__c.Check_Number__c is a required field- (superseded: ERA integration will populate this field; manual entry deprecated per INS-0035 dependency)

Given the schema deployment is complete,
When a validation rule on Claim__c is triggered,
Then a Claim__c record cannot be saved with Status__c = 'Adjudicated' if Total_Allowed_Amount__c is null or zero.

## Solution

Deploy via Salesforce DX source-tracked sandbox. Custom objects defined in object metadata XML. Field-level security configured via Permission Set (PS_Claims_Base) rather than profile mutation. Validation rules implemented declaratively using ISBLANK() and ISPICKVAL() formula logic. Master-Detail relationships enforce rollup summary fields: Policy__c.Total_Claims_Billed__c (SUM of Claim__c.Total_Billed_Amount__c). Record-Triggered Flow on Claim__c (After Save) stamps Claim__c.Last_Status_Change_Date__c on every Status__c transition. Deployment validated via Apex test class ClaimSchemaTest covering all required fields and validation rule triggers.

## Description

As a Provider Network Specialist, I need the Provider_Network_Agreement__c (PNA) object to support a full contract lifecycle from Draft through Executed, with automated DocuSign envelope generation for provider signature collection, so that Acme Insurance can digitize and track all network contracting activity within Salesforce and eliminate manual PDF routing via email.

The PNA lifecycle stages are: Draft → Pending Internal Approval → Sent for Signature → Executed → Expired → Terminated. This story covers the Draft through Executed stages only. Terminated stage is covered in the Provider Termination epic (Phase 2).

## Acceptance Criteria

Given a Provider Network Specialist creates a new Provider_Network_Agreement__c record,
When the record is saved with Status__c = 'Draft',
Then the following fields must be required:
  - PNA__c.Provider__c (Lookup to Account where IsPersonAccount = true)
  - PNA__c.Effective_Date__c (Date)
  - PNA__c.LOB__c (Picklist)
  - PNA__c.Contract_Type__c (Picklist): values = ['Participating', 'Non-Participating', 'Preferred', 'Exclusive']
  - PNA__c.Fee_Schedule__c (Lookup to Fee_Schedule__c)

Given a PNA record is in Status__c = 'Draft',
When a Provider Network Specialist clicks the 'Submit for Internal Approval' button,
Then a Salesforce Approval Process must be triggered, routing to the assigned Claims Operations Manager, and PNA__c.Status__c must update to 'Pending Internal Approval'.

Given the internal approval is approved by the Claims Operations Manager,
When the approval step completes,
Then:
  - PNA__c.Status__c must update to 'Sent for Signature'
  - A DocuSign envelope must be automatically generated using the 'PNA_Standard_Template' DocuSign template ID
  - The envelope must be sent to the email address on the related Account (Provider__c.Email)
  - PNA__c.DocuSign_Envelope_ID__c must be populated with the returned envelope GUID
  - A Chatter post must be created on the PNA record: 'DocuSign envelope sent to [Provider Name] on [Date].'

Given a DocuSign envelope has been sent,
When the provider completes the signature and DocuSign sends the completion callback,
Then:
  - PNA__c.Status__c must update to 'Executed'
  - PNA__c.Executed_Date__c must be set to today's date
  - The signed PDF must be stored as a Salesforce File (ContentDocument) linked to the PNA record
  - The related Account.Provider_Status__c must update to 'Active' if it was previously 'Pending Credentialing'

Given a PNA is in 'Executed' status,
When PNA__c.Expiration_Date__c is within 90 days of today,
Then a Task must be created and assigned to the Provider Network Specialist with Subject = 'PNA Renewal Required — [Provider Name]' and Due Date = Expiration_Date__c minus 60 days.

Given a Provider Network Specialist attempts to edit PNA__c.Fee_Schedule__c on an Executed PNA,
When the save is attempted,
Then a validation rule must block the save with message: 'Fee Schedule cannot be modified on an Executed agreement. Please initiate a contract amendment.'

## Solution

PNA lifecycle managed via a Record-Triggered Flow (RTF_PNA_StatusLifecycle) on Provider_Network_Agreement__c, triggering on create and update. Approval Process (AP_PNA_InternalApproval) configured with single approver (manager of submitting user's role hierarchy). DocuSign integration uses DocuSign eSignature for Salesforce managed package; envelope send action invoked via Apex (DocuSignEnvelopeService.sendPNAEnvelope()) called from the RTF post-approval step. DocuSign completion webhook registered to a custom Salesforce Site endpoint (PNADocuSignCallbackController) which updates Status__c and attaches the signed document via ContentVersion insert. Chatter post via ConnectApi.ChatterFeeds in the callback controller. 90-day renewal Task created via a separate Scheduled-Triggered Flow running nightly. Validation rule (VR_PNA_LockFeeScheduleOnExecuted) uses ISPICKVAL(Status__c, 'Executed') && ISCHANGED(Fee_Schedule__c).

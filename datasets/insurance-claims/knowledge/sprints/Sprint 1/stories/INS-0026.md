## Description

As a Broker Relations Coordinator, I need a guided OmniScript that walks through the Group/Employer onboarding process — capturing employer details, selecting LOB coverage tiers, associating a Broker of Record, and generating a Group_Service_Agreement__c record — culminating in a DocuSign envelope sent to the employer's authorized signatory so that we can digitize and accelerate the GSA execution process and reduce paper-based onboarding cycle time from 10 days to under 3 days.

## Acceptance Criteria

Given a Broker Relations Coordinator launches the 'Group Employer Onboarding' OmniScript, when they complete all steps, then:

1. Step 1 (Employer Info): must capture Account.Name, Account.BillingAddress, Account.Industry, Account.NumberOfEmployees, and a lookup to existing Account or option to create new. Validation must prevent progression if NumberOfEmployees < 2.

2. Step 2 (Coverage Selection): must present a multi-select for LOB (Medical, Dental, Vision, Workers_Comp) and for each selected LOB, prompt for coverage tier (Picklist: Bronze|Silver|Gold|Platinum). At least one LOB must be selected to advance.

3. Step 3 (Broker of Record): must include a lookup to Contact where Contact.RecordType = 'Broker' and display Contact.NPN__c (National Producer Number), Contact.License_State__c, Contact.License_Expiry__c. If Contact.License_Expiry__c < TODAY(), display inline warning 'Broker license expired — coordinator approval required' and set GSA__c.Broker_Approval_Required__c = true.

4. Step 4 (GSA Summary & Review): must display all captured values in a read-only summary. On 'Submit', a DataRaptor Transform must create Group_Service_Agreement__c with: GSA__c.Account__c, GSA__c.Effective_Date__c = TODAY(), GSA__c.Status__c = 'Pending_Signature', GSA__c.LOB_Coverage__c (multi-select picklist), GSA__c.Broker__c.

5. Given GSA__c record is created with Status__c = 'Pending_Signature', when the OmniScript completes, then a DocuSign envelope must be triggered via Integration Procedure 'IP_Send_GSA_DocuSign' sending the GSA template to Account's primary contact email within 2 minutes.

6. Given DocuSign envelope is sent, when the employer completes signing, then a DocuSign Connect webhook must update GSA__c.Status__c = 'Active' and GSA__c.Executed_Date__c = signature timestamp.

7. Given the OmniScript is accessed by a user without PSG 'Broker_Relations_Coordinator_PSG', when they attempt to launch the script, then they must receive an error message: 'You do not have permission to onboard Group Employers. Contact your system administrator.'

## Solution

Build 4-step OmniScript 'OS_Group_Employer_Onboarding' using OmniStudio. Use DataRaptor Extract for broker lookup with filter on RecordType.Name = 'Broker'. Use DataRaptor Transform 'DRT_Create_GSA' for GSA__c record creation. Build Integration Procedure 'IP_Send_GSA_DocuSign' using DocuSign OmniStudio connector action to send envelope with template ID mapped in Custom Metadata Type 'DocuSign_Template_Config__mdt'. Configure DocuSign Connect endpoint to call a Platform Event 'GSA_Signed__e' which triggers a Record-Triggered Flow on GSA__c to update Status__c and Executed_Date__c. Add Custom Permission 'CAN_ONBOARD_GROUP_EMPLOYER' and gate OmniScript launch via LWC wrapper component that checks permission before rendering.

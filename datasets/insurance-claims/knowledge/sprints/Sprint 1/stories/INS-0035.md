## Description

**Bug Report — INS-0035**

**Environment:** Acme Insurance Sprint 1 Sandbox (CS117)
**Reported By:** Rachel Torres (Claims Operations Manager)
**Severity:** High — Blocking payment processing for all COB claims

**Summary:**
When attempting to save a new Payment__c record linked to a Claim__c where COB_Applied__c = true, the system throws an unhandled NullPointerException and the record fails to save. This blocks payment disbursement for all COB claims, which represent approximately 23% of daily claim volume.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record where COB_Applied__c = true and Status__c = 'Adjudicated'.
2. Click 'New Payment' from the Payments related list.
3. Populate Payment__c fields: Amount__c = 150.00, Payment_Method__c = 'EFT', Payment_Date__c = TODAY().
4. Click Save.
5. Observe error: 'System.NullPointerException: Attempt to de-reference a null object' in the page error banner.
6. Check debug logs: error originates at PaymentTriggerHandler.cls, line 47, method calculateCOBOffset().

**Expected Behavior:**
Payment__c record saves successfully. COB offset amount is calculated and populated in Payment__c.COB_Offset_Amount__c based on Claim__c.Secondary_Payer_Paid__c.

**Actual Behavior:**
NullPointerException thrown at PaymentTriggerHandler.cls line 47. Payment__c record is not saved. No partial commit occurs.

**Root Cause (Preliminary):**
PaymentTriggerHandler.calculateCOBOffset() reads Claim__c.Secondary_Payer_Paid__c but this field was added to Claim__c schema after the trigger was last deployed. The trigger handler does not null-check Secondary_Payer_Paid__c before arithmetic operation, causing NPE when the field is null on older claim records.

**Workaround:**
Manually populate Claim__c.Secondary_Payer_Paid__c = 0.00 before creating Payment. Extremely inefficient at scale.

## Solution

Fix PaymentTriggerHandler.cls line 47: add null-safe check before calculateCOBOffset() — replace direct field access with null coalescing pattern: Decimal secondaryPaid = claim.Secondary_Payer_Paid__c != null ? claim.Secondary_Payer_Paid__c : 0.00. Deploy fix to sandbox and validate with Apex test PaymentTriggerHandlerTest.cls covering three scenarios: (1) COB_Applied__c = true with Secondary_Payer_Paid__c populated, (2) COB_Applied__c = true with Secondary_Payer_Paid__c = null, (3) COB_Applied__c = false. Ensure 100% code coverage on the handler method. Add a backfill Data Loader job to set Secondary_Payer_Paid__c = 0.00 on all Claim__c records where COB_Applied__c = true and Secondary_Payer_Paid__c IS NULL.

## Description

As a Credentialing Specialist, I need Named Credentials configured in Salesforce for the CAQH ProView API, NPPES NPI Registry API, and CMS PECOS endpoint so that future Apex callouts for provider data verification can authenticate securely without hardcoding credentials, and so that integration readiness is established before the credentialing workflow is built in Sprint 2.

## Acceptance Criteria

Given the Named Credential 'NC_CAQH_ProView' is configured,
When an Apex callout is made using it,
Then authentication uses OAuth 2.0 Client Credentials flow with the client_id and client_secret stored in the associated External Credential, and the endpoint resolves to 'https://proview.caqh.org/api/v2'.

Given the Named Credential 'NC_NPPES_Registry' is configured,
When an Apex callout is made using it,
Then the endpoint resolves to 'https://npiregistry.cms.hhs.gov/api' and authentication is set to 'No Authentication' (public API), with the Named Credential used solely for endpoint management.

Given the Named Credential 'NC_CMS_PECOS' is configured,
When tested via a developer Apex anonymous window,
Then an HTTP GET to the configured endpoint returns a 200 or 401 (credential verification required) and does NOT return a network-level error, confirming firewall and CSP trusted site configuration is correct.

Given all three Named Credentials are deployed,
When a System Admin reviews Setup > Named Credentials,
Then no plaintext passwords or API keys are visible in any configuration field, and all secrets are stored in External Credentials with Per-User or Per-Org principal assignment as appropriate.

## Solution

Configure Named Credentials and External Credentials via Setup UI, then export as metadata for source control. CAQH uses OAuth 2.0 with an External Credential storing client_id and client_secret as Protected Custom Headers. NPPES is a public REST API requiring no auth — Named Credential configured with 'No Authentication' and 'Allow Merge Fields in HTTP Body' disabled. CMS PECOS uses a certificate-based mutual TLS configuration; the client certificate uploaded to Certificate and Key Management in Setup and referenced in the External Credential. Remote Site Settings and CSP Trusted Sites added for all three domains. Apex HttpCalloutMock classes created for each endpoint to support unit testing in Sprint 2 callout development.

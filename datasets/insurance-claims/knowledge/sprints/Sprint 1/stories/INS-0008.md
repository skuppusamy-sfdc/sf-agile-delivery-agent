## Description

As a Compliance Officer, I need role-based access controls implemented via Salesforce Permission Set Groups so that Claims Operations Managers, Provider Network Specialists, Credentialing Specialists, and SIU investigators each have least-privilege access to sensitive claims and provider data, ensuring regulatory compliance with HIPAA minimum-necessary standards.

## Acceptance Criteria

Given the Permission Set Groups are configured,
When a user assigned to PSG 'Claims_Ops_Manager' logs in,
Then they have Read/Create/Edit on Claim__c, Claim_Line__c, Payment__c, EOB__c and Read-only on Policy__c, Provider__c (Account).

Given the PSG 'Provider_Network_Specialist' is assigned,
When the user navigates to Provider Network objects,
Then they have Read/Create/Edit on Provider_Network_Agreement__c, Fee_Schedule__c, Group_Service_Agreement__c, and Read-only on Claim__c with no access to Payment__c.Bank_Account__c field.

Given the PSG 'Credentialing_Specialist' is assigned,
When the user accesses credentialing records,
Then they have full CRUD on Credentialing_Case__c and Read-only on Provider__c (Account), with no visibility to Claim__c financial fields (Total_Paid_Amount__c, Total_Allowed_Amount__c).

Given the PSG 'SIU_Investigator' is assigned,
When the user searches for claims,
Then they have Read-only access to Claim__c, Claim_Line__c, Policy__c, and full CRUD on a custom SIU_Case__c object; they cannot Edit or Delete Payment__c records.

Given any PSG is assigned to a test user in the sandbox,
When the System Administrator runs the Permission Set Group audit report,
Then all four PSGs are listed with correct muting permission sets applied to suppress inherited Profile permissions that exceed least-privilege requirements.

## Solution

Create four Permission Sets (PS_Claims_Ops_Manager_Base, PS_Provider_Network_Base, PS_Credentialing_Base, PS_SIU_Base) and corresponding Muting Permission Sets to suppress over-permissive Profile grants. Bundle each pair into a Permission Set Group via Setup > Permission Set Groups. Field-level security for sensitive fields (Payment__c.Bank_Account__c, Payment__c.Routing_Number__c) explicitly set to hidden in each non-Payment PSG. Automate assignment via a Record-Triggered Flow on User object when User.Department__c field is populated during Workday HR integration (stub flow in Phase 1, full Workday integration deferred to Phase 2). Validate with Salesforce Security Health Check post-deployment.

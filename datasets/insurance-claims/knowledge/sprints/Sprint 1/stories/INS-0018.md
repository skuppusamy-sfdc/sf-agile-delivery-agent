# INS-0018: Deploy Provider__c (Person Account) and Facility__c (Business Account) record types with NPI and TIN fields

- **Priority**: High
- **Resolution**: Unresolved

- **Reporter**: Marcus Johnson

- **Epic**: Provider 360

- **Labels**: Phase1, DataModel, ProviderData

## Description

As a Provider Relations Specialist, I need Provider__c modeled as a Person Account record type and Facility__c modeled as a Business Account record type within the Accounts object, each with healthcare-specific fields like NPI, TIN, and specialty codes, so that provider and facility data is centrally managed and available for credentialing, network contracting, and claims adjudication workflows.

## Acceptance Criteria

Given Person Accounts are enabled in the org,
When a new Account record is created with Record Type = 'Provider',
Then the following fields are visible and editable: NPI__c (Text 10, Unique), TIN__c (Text 9, Encrypted), Primary_Specialty__c (Picklist, mapped to NUCC taxonomy), Secondary_Specialty__c (Picklist), License_Number__c (Text 20), License_State__c (Picklist, US states), License_Expiration_Date__c (Date), CAQH_Provider_ID__c (Text 10), Credentialing_Status__c (Picklist: Not Started, In Progress, Active, Suspended, Terminated), Network_Status__c (Picklist: In-Network, Out-of-Network, Pending).

Given a Business Account record with Record Type = 'Facility',
When created by a Provider Network Specialist,
Then the following fields are present: Facility_NPI__c (Text 10, Unique), TIN__c (Text 9, Encrypted), Facility_Type__c (Picklist: Hospital, Urgent Care, ASC, Lab, Pharmacy, SNF), CMS_Certification_Number__c (Text 10), Accreditation_Body__c (Picklist: Joint Commission, DNV, ACHC, None), Contract_Status__c (Picklist: Active, Expired, Pending, Terminated).

Given a Provider__c record where License_Expiration_Date__c is within 60 days of today,
When the record is viewed,
Then a warning banner is displayed: 'Provider license expiring soon. Credentialing review required.'

Given a Provider__c or Facility__c record,
When NPI__c is saved,
Then a duplicate rule fires and blocks creation if another Account record of the same record type already has the same NPI__c value, displaying: 'An active provider record with this NPI already exists.'

-Given TIN__c is stored as plain text, When displayed on the record, Then only the last 4 digits are visible to non-admin users- (Superseded: TIN__c will use Salesforce Shield Platform Encryption with a deterministic encryption scheme to support SOQL filtering.)

## Solution

Enable Person Accounts in the sandbox org via Salesforce Support request (already submitted). Create 'Provider' Record Type on Account for Person Accounts and 'Facility' Record Type for Business Accounts. Deploy custom fields via metadata XML. TIN__c encrypted using Salesforce Shield Platform Encryption (deterministic policy) to allow SOQL WHERE clauses. Duplicate Rules and Matching Rules configured for NPI__c uniqueness check across record types. License expiry warning implemented as a Record-Triggered Flow (After Save, scheduled path) that creates a Task and sets a custom Warning_Banner_Message__c formula field. Page layouts assigned per record type. Provider 360 FlexCard (OmniStudio) scoped for Sprint 2 build.

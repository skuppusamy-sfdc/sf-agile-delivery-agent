## Description

As a Provider Network Specialist, I need the Provider_Network_Agreement__c (PNA) object configured with contract lifecycle status tracking and integrated with Conga CLM so that I can manage the full contracting workflow — from initial outreach through executed agreement — with document versioning, e-signature via DocuSign, and automated effective date population directly within Salesforce.

## Acceptance Criteria

Given the PNA object is deployed,
When a new Provider_Network_Agreement__c record is created,
Then it must capture: Provider__c (Lookup to Account where RecordType = 'Provider'), Facility__c (Lookup to Account where RecordType = 'Facility'), Contract_Status__c (Picklist: Draft|Sent for Signature|Executed|Terminated|Expired), Effective_Date__c (Date), Termination_Date__c (Date), LOB__c (Multi-select Picklist: Medical|Dental|Vision|Workers Comp), NPI__c (Text 10, validated by regex ^[0-9]{10}$), TIN__c (Text 9, encrypted), Fee_Schedule__c (Lookup to Fee_Schedule__c), Conga_Contract_ID__c (Text 50, External ID).

Given a PNA record has Contract_Status__c = 'Draft',
When the Provider Network Specialist clicks 'Send for Signature',
Then a Conga CLM contract generation request is initiated via Named Credential callout, a DocuSign envelope is created, and PNA.Contract_Status__c transitions to 'Sent for Signature'.

Given a DocuSign envelope for a PNA is fully signed,
When the DocuSign completion webhook fires,
Then PNA.Contract_Status__c is updated to 'Executed', PNA.Effective_Date__c is stamped with TODAY(), and a Chatter post is added to the PNA record: 'Contract executed on {date} — countersigned document attached.'

Given a PNA record has Contract_Status__c = 'Executed',
When a user attempts to edit PNA.TIN__c or PNA.NPI__c,
Then a validation rule blocks the save with message: 'TIN and NPI cannot be modified on an Executed agreement. Initiate an amendment.'

Given a PNA record has Termination_Date__c populated and Termination_Date__c <= TODAY(),
When a nightly scheduled job runs,
Then PNA.Contract_Status__c is automatically updated to 'Expired' and a task is created for the assigned Provider Network Specialist with Subject: 'PNA Expired — Review Required'.

## Solution

Provider_Network_Agreement__c deployed with encrypted TIN__c field using Salesforce Shield Platform Encryption. Conga CLM integration via REST callout using Named Credential (NC_CongaCLM). Outbound callout triggered by a Screen Flow launched from 'Send for Signature' quick action button on PNA layout. DocuSign completion webhook handled by a public Apex REST endpoint (@RestResource) that parses envelope status payload and fires a Record-Triggered Flow update on PNA. Nightly expiration job implemented as Schedulable Apex (PNAExpirationBatch) running at 02:00 UTC. Validation rule on PNA uses ISPICKVAL(Contract_Status__c, 'Executed') && (ISCHANGED(TIN__c) || ISCHANGED(NPI__c)).

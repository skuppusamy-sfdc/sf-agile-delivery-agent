## Description

As a Claims Operations Manager, I need the core Salesforce custom objects and field schema deployed in the Acme Insurance org so that all downstream claims processing configuration has a stable, validated data model to build upon.

This story covers the initial object creation, field definitions, and master-detail/lookup relationships for the Phase 1 domain objects including Policy__c, Claim__c, Claim_Line__c, and Payment__c.

## Acceptance Criteria

Given the Salesforce org is a fresh Developer/Sandbox environment,
When the deployment package is applied,
Then Policy__c exists with the following fields: Policy_Number__c (Text, Unique, External ID), LOB__c (Picklist: Auto|Health|Property|Liability), Effective_Date__c (Date), Expiration_Date__c (Date), Status__c (Picklist: Active|Lapsed|Cancelled|Pending), TPA_Account__c (Lookup to Account).

Given Policy__c is deployed,
When Claim__c is created,
Then Claim__c has a Master-Detail relationship to Policy__c via Policy__c field, and includes: Claim_Number__c (Auto-Number, format CLM-{0000000}), FNOL_Date__c (DateTime), Status__c (Picklist: New|Under Review|Adjudicated|Denied|Closed|Reopened), LOB__c (Formula, mirrors Policy__c.LOB__c), Total_Billed_Amount__c (Currency), Total_Allowed_Amount__c (Currency), Total_Paid_Amount__c (Roll-Up Summary from Payment__c).

Given Claim__c is deployed,
When Claim_Line__c is created,
Then Claim_Line__c has a Master-Detail to Claim__c and includes: CPT_Code__c (Text, 5 chars), ICD10_Primary__c (Text, 8 chars), Billed_Amount__c (Currency), Allowed_Amount__c (Currency), Copay__c (Currency), Deductible_Applied__c (Currency), Coinsurance__c (Percent), Line_Status__c (Picklist: Pending|Approved|Denied|Adjusted).

Given all objects are deployed,
When validation is run via Setup > Object Manager,
Then all objects appear under the Acme Insurance namespace, all required fields are marked required, and no deployment errors exist in the org.

-Given Payment__c is deployed, When a Payment record is created without a linked Claim__c, Then the system should allow orphaned payments for TPA reconciliation- (deprecated: orphaned payments not permitted per updated data model decision 2024-11-15)

## Solution

Deploy via Salesforce CLI (sf project deploy start) using a structured metadata package. Custom objects defined in /objects directory with field metadata XML. Roll-Up Summary field Total_Paid_Amount__c on Claim__c aggregates Payment__c.Amount__c where Payment__c.Status__c = 'Issued'. Validation Rules added post-deployment via separate story. Use Permission Set baseline from INS-0053 to verify field-level security post-deploy. All objects assigned to 'Claims Processing' App in App Manager.

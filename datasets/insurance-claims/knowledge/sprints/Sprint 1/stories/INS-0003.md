## Description

As a Claims Operations Manager, I need a Policyholder 360 FlexCard embedded on the Policy__c record page that displays a consolidated view of the policyholder's active policy details, open and recently closed claims, and payment history so that I can quickly assess a member's overall claims status without navigating to multiple objects.

The FlexCard must surface data from Policy__c, Claim__c, Payment__c, and Explanation_of_Benefits__c in a single, performant view. It should support drill-down navigation to individual Claim__c records. This story depends on INS-0001 for object availability.

## Acceptance Criteria

Given a Claims Operations Manager is viewing a Policy__c record page,
When the Policyholder 360 FlexCard loads,
Then the following Policy summary fields must be displayed in the card header:
  - Policy__c.Policy_Number__c
  - Policy__c.Effective_Date__c
  - Policy__c.Expiration_Date__c
  - Policy__c.LOB__c
  - Policy__c.Status__c (displayed as a color-coded badge: 'Active' = green, 'Lapsed' = red, 'Pending' = yellow)
  - Policy__c.Group_Name__c (visible only for Group_Policy record type)

Given the FlexCard is loaded for a policy with associated claims,
When the Claims panel is rendered,
Then a datatable must display all Claim__c records where Claim__c.Policy__c = current record ID and Claim__c.Status__c IN ('New', 'In Review', 'Pending Information', 'Adjudicated') with columns: Claim Number, FNOL Date, LOB, Total Billed Amount, Total Paid Amount, Status.

Given the FlexCard is loaded,
When the Payment History panel is rendered,
Then the last 10 Payment__c records linked to claims under this policy must display with: Payment__c.Payment_Date__c, Payment__c.Amount__c, Payment__c.Payment_Method__c, Payment__c.Status__c.

Given a Claims Operations Manager clicks a Claim Number in the claims datatable,
When the navigation action fires,
Then the user is navigated to the corresponding Claim__c record detail page.

Given a policy has COB_Indicator__c = true on any associated Claim__c,
When the FlexCard renders,
Then a prominent COB alert banner must appear at the top of the card reading 'Coordination of Benefits Active — Review Required'.

-Given the FlexCard loads, When the policyholder has an active SIU referral, Then a red SIU flag icon must display in the header — deprecated, SIU flag moved to dedicated SIU FlexCard in Phase 2-

Given the FlexCard is viewed by a user with PSG_Claims_Adjudicator,
When the card renders Payment__c data,
Then Payment__c.Payment_Method__c must be masked as '****' per FLS rules established in INS-0002.

## Solution

Built using OmniStudio FlexCard. Primary DataRaptor Extract (DRX_PolicyHolder360_Extract) retrieves Policy__c, related Claim__c (filtered by status), and last 10 Payment__c records via a single Apex REST call to avoid governor limits on large member panels. DataRaptor uses child relationship queries with SOQL ORDER BY and LIMIT clauses. FlexCard conditionally renders the COB alert banner using a State property bound to a custom formula checking for any claim with COB_Indicator__c = true. Navigation action uses NavigationMixin to route to Claim__c record page. Color-coded status badge implemented via FlexCard conditional CSS classes. FlexCard deployed to Policy__c Lightning Record Page via App Builder, positioned in the main column. FLS enforcement relies on running-user context in DataRaptor SOQL execution — no Apex without sharing required due to OWD configuration from INS-0002.

## Description

As a Compliance Officer, I need the Salesforce org to have My Domain configured, baseline password policies enforced, and session security settings hardened so that Acme Insurance meets HIPAA and SOC 2 platform-level security requirements prior to any PHI data being loaded into the environment.

## Acceptance Criteria

Given the Salesforce org is provisioned,
When My Domain is deployed to all users,
Then the custom domain 'acme-insurance.my.salesforce.com' is active and all logins redirect through it.

Given the session security settings page in Setup,
When reviewed by the platform admin,
Then session timeout is set to 2 hours, 'Lock sessions to the IP address from which they originated' is enabled, and TLS 1.2 or higher is enforced.

Given the password policy configuration,
When applied org-wide,
Then minimum password length is 12 characters, complexity requirements include uppercase, lowercase, number and special character, and maximum password age is 90 days.

Given a user attempts login from an unrecognized IP,
When MFA is enforced via the High Assurance session policy,
Then the user is challenged with a second factor before gaining access to any record.

-Given a Named Credential is configured for the CMS integration endpoint, When tested, Then OAuth 2.0 token exchange completes successfully- (Superseded by INS-0021 which owns all Named Credential setup.)

## Solution

Configure My Domain via Setup > My Domain wizard and deploy to all users. Apply Session Settings and Password Policies via Setup UI. Enable MFA enforcement via the 'Require MFA for all direct UI logins' org permission. Document all settings in the Platform Readiness runbook stored in Confluence. No custom Apex required; all changes are declarative Setup configurations. Verification performed via Salesforce Optimizer and the Security Health Check tool targeting a score of 85 or above.

## Description

**Bug Report**

**Summary:** Provider_Network_Agreement__c records are not transitioning Status__c to 'Expired' when Expiration_Date__c is in the past. PNA records remain in 'Active' status indefinitely, causing Provider 360 FlexCard (INS-0052) to display stale active agreements and potentially allowing claims to be adjudicated against expired contracts.

**Steps to Reproduce:**
1. Create a Provider_Network_Agreement__c record with Status__c = 'Active' and Expiration_Date__c = TODAY - 1 (yesterday).
2. Wait for or manually trigger the nightly Scheduled Flow 'SF_PNA_Status_Update' (or advance system date in sandbox).
3. Navigate to the PNA record.
4. Observe Status__c field value.

**Expected Result:** Status__c = 'Expired'

**Actual Result:** Status__c = 'Active' — no change observed.

**Environment:** Acme Insurance Full Sandbox (FS-01)
**Detected By:** Rachel Torres during Provider 360 UAT
**Severity:** High — financial risk if expired-contract claims are paid at contracted rates

## Acceptance Criteria

As per Description.

Additional verification: Given a PNA with Expiration_Date__c = TODAY - 1 and Status__c = 'Active', When the scheduled job executes, Then Status__c = 'Expired' within the same day the job runs.

Given a PNA with Expiration_Date__c = TODAY (same day), When the scheduled job executes, Then Status__c remains 'Active' (not expired until day after).

## Solution

Root cause identified: Scheduled Flow 'SF_PNA_Status_Update' has a filter condition using Expiration_Date__c < {!$Flow.CurrentDate} but the date formula was incorrectly referencing {!$Flow.CurrentDateTime} (DateTime) compared against a Date field, causing type mismatch and the filter returning zero records. Fix: Update Get Records filter to use a Date formula variable initialized to TODAY. Additionally, add a fault path email alert to notify Platform Admin if the scheduled flow encounters errors. Deploy fix to FS-01 sandbox, execute flow manually, verify 3 test PNA records transition correctly before promoting to production.

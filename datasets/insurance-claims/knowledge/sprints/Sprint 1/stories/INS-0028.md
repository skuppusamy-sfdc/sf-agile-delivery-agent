## Description

**Bug Report — Security / Access Control**

The 'Claims_Adjudicator_PSG' Permission Set Group, as deployed in INS-0024, is incorrectly granting Edit access to Payment__c records. Per the approved access matrix, Claims Adjudicators must have Read-Only access to Payment__c with no ability to create, edit, or delete payment records. This was identified during UAT regression testing on 2024-01-18.

**Steps to Reproduce:**
1. Log in as test user 'uat.adjudicator@acmeinsurance.test' (assigned Claims_Adjudicator_PSG only, no other PSGs or profiles beyond Standard User).
2. Navigate to any existing Payment__c record.
3. Click 'Edit' button — button is visible and functional.
4. Modify Payment__c.Payment_Amount__c to any value and click Save.
5. Observe: Save succeeds. Record is updated.

**Expected Behavior:** Edit button should not be visible. Attempting edit via URL manipulation should return 'Insufficient Privileges' error. Payment__c.Payment_Amount__c should be read-only.

**Actual Behavior:** Edit button is visible and functional. Claims Adjudicator can modify Payment__c.Payment_Amount__c, Payment__c.Payment_Date__c, and Payment__c.Status__c — all high-sensitivity financial fields.

**Impact:** High — HIPAA and SOX compliance risk. Any Claims Adjudicator could alter payment records without audit trail controls in place.

**Environment:** Full Sandbox (ACME-FS-01)
**Regression:** Not present in dev sandbox — likely introduced during PSG bundling in change set CS-INS-PHASE1-SEC-002.

## Acceptance Criteria

As per Description.

Additional verification steps post-fix:
1. Re-test with uat.adjudicator@acmeinsurance.test — Edit button must not appear on Payment__c records.
2. Confirm Payment__c CRUD in PS_Claims_Adjudicator permission set shows: Read = true, Create = false, Edit = false, Delete = false, View All = false, Modify All = false.
3. Confirm Muting Permission Set 'MPS_Block_Payment_Edit' is active within Claims_Adjudicator_PSG and suppresses any Edit permission inherited from profile or other bundled permission sets.
4. Run Salesforce Permission Analyzer report for user 'uat.adjudicator@acmeinsurance.test' and confirm effective permissions on Payment__c show Edit = false.
5. Verify fix does not regress Read access — adjudicator must still be able to view Payment__c records linked to their assigned Claim__c records.

## Solution

Root cause: PS_Financial_ReadOnly permission set included in Claims_Adjudicator_PSG bundle incorrectly had Payment__c Edit = true from a prior copy-paste of PS_Payment_Admin during initial setup. Fix: Open PS_Financial_ReadOnly in Setup > Permission Sets > Object Settings > Payment__c and set Edit = false, Create = false, Delete = false. Additionally create Muting Permission Set 'MPS_Block_Payment_Edit' with Payment__c Edit = true (muting sets suppress permissions) and add to Claims_Adjudicator_PSG. Re-deploy PSG via change set. Run SFDX permission audit command post-deployment to validate effective permissions before promoting to UAT.

## Description

As a Claims Operations Manager, I need the core Salesforce custom objects (Policy__c, Claim__c, Claim_Line__c, Provider__c, Facility__c, Payment__c, and Explanation_of_Benefits__c) to be created and configured with all required fields, record types, and page layouts so that the platform is ready to support end-to-end claims processing workflows in subsequent sprints.

This story covers the foundational object model setup for Phase 1. All objects must conform to the Acme Insurance data dictionary v2.3. Integration fields (NPI, TIN, CPT codes, ICD-10 codes) must be present and correctly typed to support downstream CMS and CAQH integrations.

## Acceptance Criteria

Given the Salesforce org is provisioned for the Acme Insurance implementation,
When a System Administrator navigates to Setup > Object Manager,
Then the following custom objects must exist with API names as specified: Policy__c, Claim__c, Claim_Line__c, Payment__c, Explanation_of_Benefits__c, Provider_Network_Agreement__c, Fee_Schedule__c.

Given the Claim__c object is created,
When reviewing its fields,
Then the following fields must be present with correct data types:
  - Claim__c.Status__c (Picklist): values = ['New', 'In Review', 'Pending Information', 'Adjudicated', 'Denied', 'Closed', 'Void']
  - Claim__c.LOB__c (Picklist): values = ['Commercial', 'Medicare Advantage', 'Medicaid', 'TPA']
  - Claim__c.FNOL_Date__c (Date/Time)
  - Claim__c.Total_Billed_Amount__c (Currency)
  - Claim__c.Total_Allowed_Amount__c (Currency)
  - Claim__c.Total_Paid_Amount__c (Currency, read-only via rollup)
  - Claim__c.ICD10_Primary_Diagnosis__c (Text, 10 chars)
  - Claim__c.Policy__c (Master-Detail → Policy__c)
  - Claim__c.SIU_Referral_Flag__c (Checkbox, default false)
  - Claim__c.COB_Indicator__c (Checkbox, default false)

Given the Claim_Line__c object is created,
When reviewing its fields,
Then the following fields must be present:
  - Claim_Line__c.CPT_Code__c (Text, 5 chars)
  - Claim_Line__c.Line_Status__c (Picklist): values = ['Pending', 'Approved', 'Denied', 'Adjusted']
  - Claim_Line__c.Billed_Amount__c (Currency)
  - Claim_Line__c.Allowed_Amount__c (Currency)
  - Claim_Line__c.Paid_Amount__c (Currency)
  - Claim_Line__c.Units__c (Number)
  - Claim_Line__c.Service_Date__c (Date)
  - Claim_Line__c.Claim__c (Master-Detail → Claim__c)

Given the Policy__c object is created,
When reviewing record types,
Then the following record types must exist: 'Individual_Policy', 'Group_Policy', 'Medicare_Supplement'.

Given the Provider__c (Person Account) configuration is complete,
When reviewing the Account object for Person Accounts,
Then the following custom fields must exist on Account:
  - Account.NPI__c (Text, 10 chars, unique)
  - Account.TIN__c (Text, 9 chars, encrypted)
  - Account.CAQH_Provider_ID__c (Text, 8 chars)
  - Account.Provider_Status__c (Picklist): values = ['Active', 'Inactive', 'Suspended', 'Terminated', 'Pending Credentialing']
  - Account.Primary_Specialty__c (Text)
  - -Account.Legacy_Provider_Code__c (Text, 20 chars) [deprecated — migrated to external ID field in data migration story]-

Given all objects are created,
When a System Administrator runs a validation check,
Then no required fields should be missing as per the Acme Insurance data dictionary v2.3, and all Master-Detail and Lookup relationships must resolve without errors.

## Solution

Deploy custom object metadata via Salesforce DX (SFDX) source format. Use object-meta.xml and field-meta.xml files managed in the project Git repository under force-app/main/default/objects/. Record types defined in recordType-meta.xml per object. Person Accounts enabled via Salesforce Support case prior to this story. TIN__c field encrypted using Salesforce Shield Platform Encryption (deterministic encryption to support querying). Rollup Summary field for Claim__c.Total_Paid_Amount__c aggregates Payment__c.Amount__c where Payment__c.Status__c = 'Processed'. All metadata deployed via CI/CD pipeline (GitHub Actions) to the Acme Insurance Dev sandbox. Page layouts assigned per record type using profileLayout assignments in metadata.

## Description

As a Claims Operations Manager, I need an Agentforce AI action embedded in the Claim__c record page that assists adjusters during First Notice of Loss (FNOL) triage by summarizing claimant history, flagging potential SIU indicators, and recommending an initial reserve amount based on LOB and claim type — so that adjusters can make faster, more consistent initial decisions and reduce average FNOL-to-assignment time.

Phase 1 scope: Agentforce topic and action configuration with grounded data from Claim__c, Policy__c, and Claimant history. Reserve recommendation uses a rules-based prompt template (ML model integration deferred to Phase 2).

## Acceptance Criteria

Given an adjuster opens a newly created Claim__c record with Status__c = 'New',
When the Agentforce panel is activated on the record page,
Then the AI agent presents a structured FNOL Summary including: Claimant name, Policy__c.LOB__c, Policy__c.Status__c, prior Claim count for the claimant in the last 24 months, and FNOL_Date__c.

Given the FNOL Summary is displayed,
When the agent evaluates SIU indicators,
Then if prior claim count >= 3 within 24 months OR Claim__c.Loss_Type__c = 'Total Loss' AND Policy__c.Policy_Age_Months__c < 6, the agent surfaces a warning: 'Potential SIU indicators detected — recommend SIU referral review' and sets a recommended Claim__c.SIU_Referral_Recommended__c = TRUE (pending adjuster confirmation).

Given the agent completes FNOL triage,
When the adjuster requests a reserve recommendation,
Then the agent returns an Initial_Reserve_Recommendation__c value calculated from the Prompt Template 'PT_FNOL_Reserve_Recommendation' which uses LOB__c, Loss_Type__c, and State__c as grounding variables, and displays the recommended amount with a confidence label (High|Medium|Low) based on data completeness.

Given the adjuster reviews the AI recommendation,
When they click 'Accept Recommendation',
Then Claim__c.Initial_Reserve__c is updated to the recommended value, Claim__c.Status__c transitions to 'Under Review', and an Audit_Log__c record is created capturing: Action = 'AI Reserve Accepted', Agent_Session_ID__c, Recommended_Amount__c, Accepted_By__c, Timestamp__c.

Given the adjuster clicks 'Override Recommendation',
When they enter a manual reserve amount,
Then Claim__c.Initial_Reserve__c is set to the manual value and Audit_Log__c records Action = 'AI Reserve Overridden' with both Recommended_Amount__c and Override_Amount__c captured for model feedback.

-Given SIU indicators are detected, When adjuster accepts, Then auto-create SIU Case and assign to SIU queue immediately- (deprecated: auto-SIU case creation requires SIU queue setup in Phase 2 — Phase 1 only sets flag)

## Solution

Configure Agentforce Agent in Setup > Agents: Agent Type = 'Custom', assigned to Claims Adjuster profile group. Create Agentforce Topic 'FNOL_Triage' with description scoping it to Claim__c context. Build two Agent Actions: (1) 'Summarize_FNOL_Context' — Apex-backed action class 'FNOLSummaryAgentAction' implementing Invocable method, queries Policy__c and prior Claim__c records via SOQL with Claimant_Contact__c filter; (2) 'Recommend_Initial_Reserve' — Prompt Template action using 'PT_FNOL_Reserve_Recommendation' (Flex Template type) with merge fields for LOB__c, Loss_Type__c, State__c, grounded against Einstein Trust Layer. Prompt Template stored in Setup > Prompt Builder. Audit_Log__c custom object created with fields listed in ACs. Accept/Override buttons implemented as LWC 'c-fnol-ai-action-panel' embedded in Claim__c record page, invoking Apex controller 'FNOLReserveAcceptanceController' for DML and audit logging. Einstein Trust Layer configured to mask TIN__c and Bank_Account__c from prompt context.

## Description

**Bug Report — Extends INS-0037**

**Summary:** When creating a new Policy__c record using the Quick Create action from the Policyholder Account page, selecting LOB__c = 'Vision' results in the record being saved with the default 'Medical_Policy' record type instead of 'Vision_Policy'. All other LOB values (Medical, Dental, Behavioral Health) assign correctly.

**Steps to Reproduce:**
1. Navigate to any Account record configured as a Policyholder.
2. Click the 'New Policy' Quick Action button in the Activity panel.
3. In the modal, set LOB__c = 'Vision'.
4. Fill in all other required fields (Policy_Number__c, Effective_Date__c, Policy_Status__c = 'Active').
5. Click Save.
6. Open the newly created Policy__c record.
7. Observe the Record Type displayed in the record detail header.

**Expected Result:** Policy__c record is saved with Record Type = 'Vision_Policy'.

**Actual Result:** Policy__c record is saved with Record Type = 'Medical_Policy' (the profile default).

**Environment:** Acme-INS-DEV1 sandbox, API v59.0, Chrome 124

**Frequency:** 100% reproducible with LOB__c = 'Vision'. Intermittent (approx. 30%) with LOB__c = 'Behavioral Health' — separate ticket may be warranted.

**Impact:** Claims Operations staff creating Vision policies are unknowingly applying incorrect page layouts and validation rules, causing downstream FNOL intake failures.

## Acceptance Criteria

As per Description. Verified fix must confirm Policy__c.RecordType.DeveloperName = 'Vision_Policy' when LOB__c = 'Vision' is selected via Quick Create action, Global New button, and API insert.

## Solution

Root cause identified: the Quick Action 'New Policy' is bound to a page layout that does not include a record type selection step, and the Record-Triggered Flow responsible for post-save record type assignment has a filter condition using exact-match on LOB__c picklist value 'Vision ' (trailing space in Flow text literal) causing the Vision branch to never evaluate to true. Fix: correct the trailing whitespace in the Flow text resource RT_Vision_Label. Additionally, update the Quick Action definition to explicitly pass RecordTypeId using the Vision_Policy developer name resolved via a before-save Record-Triggered Flow using `$RecordType.Id` merge field. Regression test all five LOB record type paths in the CI pipeline before deploying to QA.

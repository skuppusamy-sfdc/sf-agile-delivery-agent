## Description

As a Compliance Officer, I need Org-Wide Defaults (OWDs) and object-level sharing rules configured for the Claims and Provider data domains so that sensitive PHI contained in Claim__c, Claim_Line__c, and Provider__c records is accessible only to authorized roles, satisfying HIPAA minimum necessary access principles and Acme Insurance's internal data governance policy.

## Acceptance Criteria

Given the OWD configuration for Claim__c,
When set,
Then OWD is 'Private' and only the record owner and users above them in the role hierarchy can access Claim__c records by default.

Given the OWD configuration for Policy__c,
When set,
Then OWD is 'Private' with 'Grant Access Using Hierarchies' enabled.

Given the OWD for Provider__c (Account — Provider record type),
When set,
Then OWD is 'Public Read Only' so that all internal users can look up provider records during claims entry without requiring explicit sharing.

Given a Criteria-Based Sharing Rule named 'Share_Claims_To_SIU',
When Claim__c.SIU_Flag__c = true,
Then the record is shared with the 'SIU_Analysts' Public Group with Read Only access.

Given a Criteria-Based Sharing Rule named 'Share_Claims_By_LOB',
When Claim__c (inherited from Policy__c) LOB__c = 'Workers Comp',
Then the record is shared with the 'Workers_Comp_Claims_Team' Role with Read/Write access.

Given the OWD for Payment__c,
When set,
Then OWD is 'Private' and a sharing rule grants Read Only access to the 'Provider_Relations' Role so that Provider Relations Specialists can view payment history during provider calls.

Given all OWDs and sharing rules are deployed,
When the Salesforce Sharing Debugger is run for a test Claims Operations Manager user,
Then the user can access Claim__c records within their LOB assignment but cannot access Claim__c records belonging to a different LOB team.

## Solution

Configure OWDs via Setup > Sharing Settings. Criteria-Based Sharing Rules deployed via metadata (SharingRules XML). Public Groups 'SIU_Analysts' and 'Workers_Comp_Claims_Team' created via Setup and referenced in sharing rule metadata. Role hierarchy designed with three tiers: Executive > LOB Manager > Specialist. Sharing Debugger used for UAT validation. Note: With-sharing enforcement on all Apex classes and LWC controllers verified in code review checklist. Manual sharing option disabled on Claim__c and Payment__c to prevent ad-hoc access grants outside the defined model.

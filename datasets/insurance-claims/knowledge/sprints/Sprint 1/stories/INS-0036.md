## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider Account record page so that I can see a consolidated view of a provider's network participation status, active PNAs, credentialing case history, and associated facilities without switching between multiple related lists, enabling faster and more informed provider relationship management. Extends INS-0032 (Policyholder 360 FlexCard pattern).

## Acceptance Criteria

Given an Account record with RecordType = 'Provider' is open,
When the Provider 360 FlexCard loads,
Then the following data sections are displayed: Network Status Banner (In-Network|Out-of-Network|Pending|Terminated based on Account.Network_Status__c), Active PNAs (Provider_Network_Agreement__c where Provider__c = Account.Id and Contract_Status__c = 'Executed'), Credentialing Summary (most recent Credentialing_Case__c record: Status__c, Due_Date__c, Credentialing_Specialist__c), Associated Facilities (child Account records where RecordType = 'Facility' and Primary_Provider__c = Account.Id), NPI and TIN display (Account.NPI__c shown in full; Account.TIN__c masked as ***-**-XXXX).

Given the Provider 360 FlexCard is rendering the Network Status Banner,
When Account.Network_Status__c = 'Terminated',
Then the banner renders in red with the text 'TERMINATED — Effective: {Account.Network_Termination_Date__c}' and a link to the associated Network_Termination_Case__c record.

Given the FlexCard is loaded by a user with PSG_Provider_Network_Specialist,
When the user clicks 'Initiate Re-Credentialing' in the Credentialing Summary section,
Then a new Credentialing_Case__c record is created with Case_Type__c = 'Re-Credentialing', Provider__c = current Account.Id, and Status__c = 'Initiated' via a DataRaptor Transform and Action.

Given a Provider Account has no executed PNAs,
When the Active PNAs section renders,
Then the message 'No active network agreements on file.' is displayed with a 'Create PNA' button visible only to users with PSG_Provider_Network_Specialist.

-Given the FlexCard is loaded, When Account.CAQH_Provider_ID__c is populated, Then a 'Verify CAQH' button appears inline in the Credentialing Summary card- (deprecated: CAQH verification moved to Credentialing_Case__c record action per INS-0034 design decision)

## Solution

OmniStudio FlexCard (FC_Provider360) built with five child card components. DataRaptor Extracts: DR_Extract_ActivePNAs (filters PNA by Provider__c and Contract_Status__c = 'Executed'), DR_Extract_CredentialingHistory (orders by CreatedDate DESC, limit 1), DR_Extract_AssociatedFacilities (child Account query). TIN masking implemented in DataRaptor Transform using a formula field that returns SUBSTITUTE(TIN__c, LEFT(TIN__c,7), '*-**-') pattern. 'Initiate Re-Credentialing' action uses Integration Procedure (IP_CreateRecredentialingCase) combining a DataRaptor Transform to build the record payload and a DataRaptor Load to insert Credentialing_Case__c. FlexCard embedded on Provider Account Lightning Page using App Builder dynamic form visibility conditioned on RecordType = 'Provider'. Apex-managed sharing (INS-0031) ensures Provider Relations Specialists see only providers in their assigned territory via Territory Management 2.0.

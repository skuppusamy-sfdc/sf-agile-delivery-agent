## Description

As a Claims Operations Manager, I need the foundational custom objects Policy__c, Claim__c, and Claim_Line__c deployed to the Salesforce org with their core fields, validation rules, and master-detail/lookup relationships so that downstream sprint work on FNOL intake, adjudication, and EOB generation can be built on a stable, well-defined data model.

## Acceptance Criteria

Given the Policy__c object is deployed,
When a record is created,
Then the following fields are present and required where noted: Policy_Number__c (Text 20, Unique, External ID), Effective_Date__c (Date, Required), Expiration_Date__c (Date, Required), LOB__c (Picklist: Medical, Dental, Vision, Workers Comp), Status__c (Picklist: Active, Lapsed, Cancelled, Pending), TPA__c (Lookup to Account), Policyholder__c (Lookup to Contact).

Given the Claim__c object is deployed,
When a record is created,
Then the following fields are present: Claim_Number__c (Auto Number, format CLM-{0000000}), Policy__c (Master-Detail to Policy__c, Required), FNOL_Date__c (Date, Required), Date_of_Service__c (Date), Status__c (Picklist: New, In Review, Pending Info, Adjudicated, Denied, Paid, Closed), Total_Billed_Amount__c (Currency), Total_Allowed_Amount__c (Currency), Total_Paid_Amount__c (Currency, Read-Only formula), LOB__c (Formula, inherits from Policy__c.LOB__c), SIU_Flag__c (Checkbox, default false), COB_Indicator__c (Checkbox).

Given the Claim_Line__c object is deployed,
When a record is created,
Then the following fields are present: Claim__c (Master-Detail to Claim__c, Required), Line_Number__c (Number, Required), CPT_Code__c (Text 10), ICD10_Primary__c (Text 10), Billed_Amount__c (Currency), Allowed_Amount__c (Currency), Paid_Amount__c (Currency), Service_Date__c (Date), Rendering_Provider__c (Lookup to Provider__c), Facility__c (Lookup to Facility__c), Status__c (Picklist: Pending, Approved, Denied, Bundled, Adjusted).

Given a Claim__c record is saved where Expiration_Date__c on the related Policy__c is earlier than Claim__c.Date_of_Service__c,
When the validation rule fires,
Then the save is blocked with the error: 'Date of Service falls outside the active policy period. Please verify policy dates before submitting.'

Given the Claim__c object page layout,
When viewed by a Claims Operations Manager,
Then the Claim Lines related list is visible and includes columns: Line_Number__c, CPT_Code__c, Billed_Amount__c, Allowed_Amount__c, Status__c.

Given Total_Paid_Amount__c on Claim__c,
When any child Claim_Line__c.Paid_Amount__c is updated,
Then Total_Paid_Amount__c reflects the sum of all active (non-Denied) child line paid amounts via a roll-up summary field.

## Solution

Deploy all three objects via Salesforce CLI (sf deploy) from a structured source-format metadata package in the project Git repository. Fields, validation rules, and roll-up summaries defined in XML metadata. Master-Detail relationships enforce cascade delete from Claim__c to Claim_Line__c and from Policy__c to Claim__c. Roll-up summary for Total_Paid_Amount__c uses SUM with filter Claim_Line__c.Status__c != 'Denied'. Compact layouts and page layouts deployed via metadata. Object-level help text populated for all required fields. Changes validated against a full sandbox before deployment to Developer Pro scratch org used by the sprint team.

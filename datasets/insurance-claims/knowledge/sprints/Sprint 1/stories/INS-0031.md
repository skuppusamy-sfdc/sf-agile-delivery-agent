## Description

As a Compliance Officer, I need a granular, role-based permission model implemented via Salesforce Permission Set Groups so that Claims Operations staff, Provider Network Specialists, and Credentialing Specialists each access only the objects and fields relevant to their function, satisfying SOC 2 and HIPAA minimum-necessary access principles.

## Acceptance Criteria

Given the permission model design document is approved,
When Permission Sets are created,
Then the following atomic Permission Sets exist: PS_Claims_Read, PS_Claims_Write, PS_Claims_Adjudicate, PS_Provider_Network_Read, PS_Provider_Network_Write, PS_Credentialing_Read, PS_Credentialing_Write, PS_Payment_View, PS_SIU_Flag.

Given atomic Permission Sets exist,
When Permission Set Groups are assembled,
Then PSG_Claims_Operator includes PS_Claims_Read + PS_Claims_Write and grants CRUD on Claim__c, Claim_Line__c, and Read on Policy__c.
Then PSG_Claims_Adjudicator includes all PSG_Claims_Operator sets plus PS_Claims_Adjudicate granting Edit on Claim__c.Status__c and Claim__c.Total_Allowed_Amount__c.
Then PSG_Provider_Network_Specialist includes PS_Provider_Network_Read + PS_Provider_Network_Write granting CRUD on Provider_Network_Agreement__c, Fee_Schedule__c, and Read on Account (Provider__c, Facility__c).
Then PSG_Credentialing_Specialist includes PS_Credentialing_Read + PS_Credentialing_Write granting CRUD on Credentialing_Case__c and Read on Account.

Given PSGs are configured,
When a user assigned only PSG_Claims_Operator attempts to edit Claim__c.Status__c to 'Adjudicated',
Then the system prevents the save and displays the error: 'Insufficient privileges to adjudicate claims. Contact your administrator.'

Given the Compliance Officer role exists,
When PSG_Compliance_Auditor is assigned,
Then the user has Read-only access to Claim__c, Payment__c, EOB__c, and Credentialing_Case__c with no Create, Edit, or Delete permissions on any object.

Given all PSGs are deployed,
When a Muting Permission Set is applied to PSG_Claims_Operator,
Then access to Payment__c.Check_Number__c field is muted for that group, visible only to PSG_Claims_Adjudicator and above.

## Solution

Permission Sets and Permission Set Groups deployed via SFDX metadata. Muting Permission Sets used within PSGs to suppress sensitive field access (e.g., Payment__c.Check_Number__c, Claim__c.SIU_Referral__c). Object-level and FLS configured per PS, not on profiles — all users ride the Minimum Access profile. Sharing Model: Claim__c OWD set to Private; Sharing Rules grant access by Role Hierarchy. Criteria-Based Sharing Rule: Claim__c.SIU_Referral__c = true shares record to SIU Queue. Apex-managed sharing considered for future dynamic rules (INS-0036). Automated test in PermissionModelTest.cls validates that a Claims Operator user cannot update Claim__c.Status__c = 'Adjudicated' via System.runAs().

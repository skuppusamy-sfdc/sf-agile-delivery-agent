## Description

As a Compliance Officer, I need a robust permission model using Salesforce Permission Set Groups and a defined Role Hierarchy so that each internal role (Claims Operations Manager, Provider Network Specialist, Credentialing Specialist, SIU Analyst, Broker Relations Coordinator) has least-privilege access to sensitive objects like Claim__c, Payment__c, and Provider_Network_Agreement__c, ensuring regulatory compliance with HIPAA data segmentation requirements.

## Acceptance Criteria

Given the permission model is designed, when Permission Set Groups are deployed, then the following PSGs must exist and be validated:

1. Given a user is assigned PSG 'Claims_Adjudicator_PSG', when they access Claim__c, then they must have: Read/Create/Edit on Claim__c, Read/Create/Edit on Claim_Line__c, Read-Only on Payment__c, No Access to Provider_Network_Agreement__c, Read-Only on Policy__c.

2. Given a user is assigned PSG 'Provider_Network_Specialist_PSG', when they access provider records, then they must have: Read/Create/Edit on Provider__c (Person Account), Read/Create/Edit on Provider_Network_Agreement__c, Read/Create/Edit on Fee_Schedule__c, Read-Only on Claim__c (no financial fields), No Access to Payment__c.

3. Given a user is assigned PSG 'Credentialing_Specialist_PSG', when they access credentialing records, then they must have: Read/Create/Edit on Credentialing_Case__c, Read/Create/Edit on Provider__c, Read-Only on Provider_Network_Agreement__c, No Access to Claim__c, No Access to Payment__c.

4. Given the Role Hierarchy is configured, when records are shared upward, then the hierarchy must be: CEO > VP_Claims > Claims_Operations_Manager > Claims_Adjudicator, and separately CEO > VP_Provider_Relations > Provider_Network_Manager > Provider_Network_Specialist > Credentialing_Specialist.

5. Given an SIU Analyst user exists, when they access a Claim__c with Claim__c.SIU_Flag__c = true, then they must have Read/Edit access; when SIU_Flag__c = false, they must have Read-Only access enforced via a Sharing Rule.

6. Given a Broker Relations Coordinator is assigned their PSG, when they access Policy__c, then they must have Read-Only access and zero access to Claim__c financial fields (Total_Billed_Amount__c, Total_Paid_Amount__c).

## Solution

Create Permission Sets: PS_Claims_Adjudicator, PS_Claims_Read, PS_Provider_Network_Edit, PS_Credentialing_Edit, PS_SIU_Full, PS_Financial_ReadOnly. Bundle into Permission Set Groups using Salesforce PSG framework (Spring '20+). Configure Role Hierarchy via Setup > Roles with 'Grant Access Using Hierarchies' enabled only for Claims and Provider objects. Implement Criteria-Based Sharing Rule on Claim__c: share records where SIU_Flag__c = true to SIU_Analyst role with Read/Write. Use Muting Permission Set within PSG to suppress over-permissioned baseline profile rights. Document access matrix in Confluence and link to Compliance Officer sign-off.

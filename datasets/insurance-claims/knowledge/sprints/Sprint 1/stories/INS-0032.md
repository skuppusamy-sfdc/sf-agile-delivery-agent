## Description

As a Claims Operations Manager, I need a Policyholder 360 FlexCard embedded on the Contact record page so that my team can instantly view a policyholder's active policies, open and recent claims, COB details, and EOB history without navigating away from the Contact record, reducing average handle time during inbound member calls.

## Acceptance Criteria

Given a Contact record representing a policyholder is open,
When the Policyholder 360 FlexCard loads,
Then it displays the following data cards: Active Policies (Policy__c where Status__c = 'Active' and PolicyHolder__c = Contact.Id), Open Claims (Claim__c where Status__c IN ('New','In Review') and Claimant__c = Contact.Id), Recent EOBs (last 5 EOB__c records ordered by EOB_Date__c DESC), COB Indicator (Contact.COB_Active__c = true renders a red 'COB Active' badge).

Given the FlexCard is rendering Active Policies,
When Policy__c.TPA_Indicator__c = true,
Then a 'TPA' badge is shown alongside the policy name and the TPA_Name__c field value is displayed.

Given a Claim record is shown in the Open Claims card,
When the user clicks the Claim_Number__c link,
Then the user is navigated to the Claim__c record detail page in the same browser tab.

Given the Contact has no active policies,
When the FlexCard loads,
Then the Active Policies card displays the message: 'No active policies found for this member.'

-Given the FlexCard is loaded, When the user clicks 'File New Claim', Then an OmniScript for FNOL is launched in a modal- (deprecated: FNOL OmniScript launch moved to INS-0040 in Sprint 2)

Given the FlexCard is embedded on the Contact Lightning page,
When the page loads for a user with PSG_Claims_Operator,
Then the FlexCard renders without error and all DataRaptor Extract actions complete within 3 seconds for a member with up to 50 historical claims.

## Solution

Built using OmniStudio FlexCard with child cards for each data domain. DataRaptor Extract (DR_Extract_PolicyholderPolicies, DR_Extract_PolicyholderClaims, DR_Extract_RecentEOBs) query the respective objects filtered by Contact.Id passed as input parameter. COB badge implemented via FlexCard conditional state using Contact.COB_Active__c boolean. FlexCard embedded on Contact Lightning Record Page via App Builder, visibility filtered by PSG_Claims_Operator and above. Performance optimization: DataRaptor caching enabled with 60-second TTL for EOB list. Navigation action uses OmniStudio NavigationMixin to route to Claim__c record.

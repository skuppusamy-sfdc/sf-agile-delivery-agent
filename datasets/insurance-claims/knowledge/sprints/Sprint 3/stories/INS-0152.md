## Description

As a Credentialing Specialist, I need the system to call the CMS NPPES NPI Registry API when an NPI is entered on a Provider__c record so that provider demographic data (name, taxonomy, address, entity type) is validated and auto-populated, reducing manual data entry errors and accelerating the credentialing process.

## Acceptance Criteria

Given a user enters a 10-digit NPI in Provider__c.NPI__c and clicks the 'Validate NPI' button
When the NPPES API call is made
Then if a match is found, Provider__c.FirstName, Provider__c.LastName, Provider__c.Provider_Taxonomy_Code__c, Provider__c.Practice_Address__c, and Provider__c.NPI_Entity_Type__c are auto-populated from the API response

Given the NPPES API returns a valid match
When data is populated
Then Provider__c.NPI_Validation_Status__c is set to 'Validated' and Provider__c.NPI_Last_Validated_Date__c = TODAY()

Given the NPI entered does not exist in the NPPES registry
When the API call returns no results
Then Provider__c.NPI_Validation_Status__c is set to 'Invalid' and an error message is displayed: 'NPI not found in NPPES registry. Please verify the NPI and retry.'

Given the NPPES API is unreachable or returns a timeout
When the callout fails
Then Provider__c.NPI_Validation_Status__c is set to 'Validation Pending' and a retry Platform Event (NPI_Validation_Retry__e) is published for async re-attempt within 30 minutes

Given the NPI_Entity_Type__c returned by NPPES is 'Organization' (Type 2)
When the record is a Person Account (Provider__c)
Then a validation warning is surfaced: 'NPI Entity Type mismatch — Organization NPI entered on individual provider record'

Given a validated NPI already exists on another active Provider__c record
When a duplicate NPI is entered
Then a duplicate warning is shown with a link to the existing record and save is blocked until resolved

## Solution

Build an LWC 'npiValidationPanel' embedded on the Provider__c record page via a Lightning App Builder slot. On button click, the LWC calls an Apex controller method (NPPESIntegrationService.validateNPI(String npi)) which makes a named credential-secured HTTP GET callout to https://npiregistry.cms.hhs.gov/api/?number={npi}&version=2.1. Response JSON is parsed via a custom NPPESResponseWrapper class. Field mapping and record update performed via DML in the Apex service layer. Retry logic implemented using a Scheduled Apex job triggered by the NPI_Validation_Retry__e Platform Event subscriber. Duplicate NPI check uses a SOQL query with a custom unique index on Provider__c.NPI__c. Named Credential 'NPPES_Registry' configured in Setup for endpoint management.

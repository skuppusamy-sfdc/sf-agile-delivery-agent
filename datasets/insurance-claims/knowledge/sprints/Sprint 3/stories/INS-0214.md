## Description

As a Provider Network Specialist, I need a guided OmniScript intake form to create a new Credentialing_Case__c and associated Provider__c (Person Account) record in a single workflow, so that I can standardize data collection, enforce required credentialing fields, and reduce data entry errors during provider onboarding. Extends INS-0183.

## Acceptance Criteria

Given a Provider Network Specialist launches the 'New Provider Credentialing' OmniScript from the App Launcher or Provider Network console tab,
When Step 1 (Provider Demographics) is completed,
Then a Person Account (Provider__c) record is created or matched via TIN__c and NPI__c duplicate rules before advancing to Step 2.

Given the specialist enters an NPI that already exists on an active Provider__c record,
When the duplicate check DataRaptor executes,
Then a warning modal displays the existing provider's name, NPI, and current credentialing status, and the specialist must confirm intent to proceed or cancel.

Given all required steps are completed (Demographics, License Info, Malpractice History, Board Certifications, CAQH Profile Link),
When the specialist clicks 'Submit',
Then a Credentialing_Case__c is created with Status__c = 'Application Received', Credentialing_Type__c populated from Step 1 selection (values: 'Initial', 'Re-credentialing', 'Delegated'), and the Provider__c lookup is set.

Given the OmniScript is submitted successfully,
When the Credentialing_Case__c record is created,
Then an automated email is sent to the provider's Email__c address using the 'Provider Credentialing Acknowledgement' email template, and a DocuSign envelope is initiated for the Credentialing Application Agreement.

Given a user closes the browser or navigates away mid-OmniScript,
When the session is resumed within 24 hours,
Then OmniScript resume functionality restores all previously entered field values from the OmniScript cache.

Given the LOB selected in Step 1 is 'Behavioral Health',
When the OmniScript advances to Step 3 (License Info),
Then additional conditional fields for DEA Registration Number and Prescribing Authority are displayed and marked required.

## Solution

Build a multi-step OmniScript (Type: 'Provider Credentialing Intake', Sub-Type: 'New') using OmniStudio Designer. Each step uses DataRaptor Extract actions for duplicate checking against Provider__c using Matching Rules. Step completion triggers DataRaptor Transform + Load actions to upsert Provider__c. Final submission step triggers an Integration Procedure that (1) creates Credentialing_Case__c, (2) invokes DocuSign via named credential for envelope initiation, and (3) sends email via Messaging.sendEmail Apex. Conditional rendering controlled by OmniScript conditional views driven by LOB__c selection. Resume functionality enabled via OmniScript 'Save State' configuration. FlexCard embedded on Credentialing_Case__c record page displays real-time case status.

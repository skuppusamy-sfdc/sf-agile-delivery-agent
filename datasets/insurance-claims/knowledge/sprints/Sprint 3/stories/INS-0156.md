## Description

As a Provider Network Specialist, I need a guided OmniScript to be launched automatically when a Credentialing_Case__c is approved so that a Provider_Network_Agreement__c (PNA) record is created, pre-populated with credentialing and provider data, and a DocuSign envelope is sent to the provider for e-signature, ensuring a seamless transition from credentialing to contracting.

## Acceptance Criteria

Given a Credentialing_Case__c.Status__c is updated to 'Approved'
When the approval is saved
Then a Record-Triggered Flow launches the 'PNA_Initiation_OmniScript' and a Provider_Network_Agreement__c record is created with PNA__c.Status__c = 'Draft', PNA__c.Provider__c = Credentialing_Case__c.Provider__c, PNA__c.Effective_Date__c = TODAY() + 30, and PNA__c.LOB__c populated from Credentialing_Case__c.Line_of_Business__c

Given the PNA record is created in 'Draft' status
When the OmniScript completes the contract generation step
Then a Conga CLM document is generated using the applicable PNA template (selected based on PNA__c.LOB__c and PNA__c.Network_Type__c) and attached to the PNA__c record as a ContentDocument

Given the Conga document is successfully generated
When the DocuSign step executes
Then a DocuSign envelope is sent to Provider__c.PersonEmail with the PNA document, and PNA__c.DocuSign_Envelope_ID__c and PNA__c.DocuSign_Sent_Date__c are populated

Given the provider completes the DocuSign signature
When the DocuSign callback is received
Then PNA__c.Status__c is updated to 'Executed', PNA__c.Execution_Date__c = signature timestamp, and Provider__c.Network_Participation_Status__c is updated to 'Active'

Given the DocuSign envelope is not signed within 14 days
When the expiry threshold is reached
Then PNA__c.Status__c is set to 'Signature Overdue' and a Task is created for the assigned Provider Network Specialist to follow up

## Solution

Build OmniScript 'PNA_Initiation_OmniScript' in OmniStudio with steps: (1) DataRaptor Extract to pull Provider__c and Credentialing_Case__c data, (2) Review and confirm screen with editable PNA fields, (3) DataRaptor Transform to map fields to PNA__c layout, (4) DataRaptor Load to create PNA__c record, (5) Integration Procedure call to Conga CLM REST API for document generation using template ID resolved from a custom metadata type (PNA_Template_Map__mdt keyed by LOB__c + Network_Type__c), (6) Integration Procedure call to DocuSign eSignature API to create and send envelope. DocuSign callback handled via a custom Salesforce Site endpoint that triggers a Record-Triggered Flow to update PNA__c status. Overdue signature check implemented as a Scheduled Flow running nightly querying PNA__c where Status__c = 'Sent for Signature' AND DocuSign_Sent_Date__c <= TODAY()-14.

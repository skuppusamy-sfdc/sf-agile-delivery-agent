## Description

As a Credentialing Specialist, I need a Credentialing_Case__c record to be automatically created when a new Provider__c (Person Account) is submitted for network participation so that credentialing workflows are initiated without manual intervention and no provider falls through the cracks during onboarding.

## Acceptance Criteria

Given a new Provider__c record is created with Provider_Status__c = 'Pending Credentialing' and a valid NPI field populated,
When the record is saved,
Then a Credentialing_Case__c record is auto-created with:
  - Credentialing_Case__c.Status__c = 'Initiated'
  - Credentialing_Case__c.Provider__c = (linked Provider__c Id)
  - Credentialing_Case__c.Case_Type__c = 'Initial Credentialing'
  - Credentialing_Case__c.Due_Date__c = TODAY() + 45 days
  - Credentialing_Case__c.Assigned_To__c = queue 'Credentialing Team'

Given a Provider__c record is updated (not newly created),
When Provider_Status__c transitions to 'Re-Credentialing Due',
Then a new Credentialing_Case__c is created with Case_Type__c = 'Re-Credentialing' and Due_Date__c = TODAY() + 60 days.

Given a Provider__c record is created without a populated NPI field,
When the record is saved,
Then no Credentialing_Case__c is created and a validation error is displayed: 'NPI is required to initiate credentialing.'

-Given a Provider__c is created via Data Loader bulk import, the flow should fire synchronously- (deprecated — bulk imports now handled via async Platform Event, see INS-0194)

Given a Credentialing_Case__c already exists for the Provider__c with Status__c not in ('Closed', 'Withdrawn'),
When a duplicate trigger condition is met,
Then no duplicate Credentialing_Case__c is created and a warning toast is displayed to the user.

## Solution

Implement a Record-Triggered Flow on Provider__c object firing on Create and Update (entry condition: Provider_Status__c changes to 'Pending Credentialing' or 'Re-Credentialing Due'). Use a Get Records element to check for existing open Credentialing_Case__c records before creation to prevent duplicates. A Decision element branches on NPI__c null check. Credentialing_Case__c creation handled via Create Records element. For bulk import scenarios, a Platform Event (Credentialing_Initiation__e) is published and consumed by a separate Apex handler (CredentialingCaseService.cls) — see INS-0194. Assign to Credentialing Team queue via Assignment Rules. Deploy via unlocked package. Permission Set Group 'Credentialing_Ops_PSG' grants FLS on all relevant fields.

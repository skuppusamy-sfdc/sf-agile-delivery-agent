## Description

As a Provider Network Specialist, I need the system to prevent a Provider_Network_Agreement__c (PNA) record from being set to 'Active' status unless the associated provider has an approved Credentialing_Case__c, so that we do not admit uncredentialed providers into the network and remain compliant with state regulatory requirements and NCQA standards. Extends INS-0197.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record is being saved with PNA.Status__c = 'Active',
When the system evaluates the linked Provider__c record,
Then it checks for the existence of at least one Credentialing_Case__c where Status__c = 'Approved' AND Credentialing_Type__c IN ('Initial', 'Re-credentialing') AND Approval_Date__c is not null.

Given no qualifying approved Credentialing_Case__c exists for the provider,
When the PNA save is attempted with Status__c = 'Active',
Then a validation error is thrown: 'Provider Network Agreement cannot be activated: No approved credentialing case found for this provider. Please complete the credentialing process before activating the agreement.'

Given an approved Credentialing_Case__c exists but Approval_Date__c is older than 36 months (re-credentialing cycle threshold),
When the PNA activation is attempted,
Then a warning message is displayed (non-blocking): 'Warning: Provider credentialing approval is older than 36 months. Re-credentialing may be required. Confirm activation with your Credentialing Specialist.'

-Given PNA records with Status__c = 'Provisional' are also blocked by this rule, Then the same validation applies.- (Superseded: 'Provisional' status is excluded from this gate per compliance team clarification on 2024-09-12. Only 'Active' status is gated.)

Given the PNA is being created by a System Administrator profile via Data Loader for a bulk migration,
When a custom field PNA.Bypass_Credentialing_Gate__c = true is set,
Then the validation is skipped and an entry is written to Integration_Error_Log__c with a note that the gate was bypassed, the running user, and a timestamp.

## Solution

Implement validation logic using a Before Save Record-Triggered Flow on Provider_Network_Agreement__c with entry criteria Status__c = 'Active'. A Get Records element retrieves related Credentialing_Case__c records filtered by Status__c = 'Approved' and Credentialing_Type__c IN ('Initial','Re-credentialing'). Decision elements evaluate count and Approval_Date__c age. Custom Error element surfaces the blocking message for the hard gate. A Screen Flow path (launched from a Quick Action) handles the soft warning for the 36-month threshold, displaying a confirmation modal before proceeding. Bypass logic checks PNA.Bypass_Credentialing_Gate__c and running user's Profile via {!$Profile.Name} formula. Bypass events are logged via an Apex After-Save invocable called from the flow, inserting Integration_Error_Log__c records. Permission to set Bypass_Credentialing_Gate__c = true is restricted via Field-Level Security to the 'System Administrator' and 'Data Migration' Permission Set Group only.

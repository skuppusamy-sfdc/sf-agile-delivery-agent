## Description

As a Credentialing Specialist, I need bulk Provider__c imports via Data Loader to asynchronously trigger Credentialing_Case__c creation through a Platform Event architecture so that governor limits are not breached during large provider onboarding migrations and all imported providers receive a credentialing case without manual follow-up. Extends INS-0191 which handles real-time single-record creation.

## Acceptance Criteria

Given a batch of Provider__c records is imported via Data Loader with Provider_Status__c = 'Pending Credentialing' and valid NPI__c values,
When the import completes,
Then a Credentialing_Initiation__e Platform Event is published for each qualifying Provider__c record within the same transaction batch.

Given Credentialing_Initiation__e events are published,
When the subscribing Apex Trigger (CredentialingEventTrigger.cls) processes the events,
Then one Credentialing_Case__c is created per event with:
  - Status__c = 'Initiated'
  - Case_Type__c = 'Initial Credentialing'
  - Provider__c = event payload Provider_Id__c
  - Due_Date__c = TODAY() + 45
  - Assigned_To__c = 'Credentialing Team' queue

Given a batch of 500 Provider__c records is imported,
When Platform Events are processed,
Then all 500 Credentialing_Case__c records are created within 10 minutes and no Apex CPU or DML governor limit exceptions are logged in the debug log.

Given a Provider__c imported via bulk has a duplicate open Credentialing_Case__c,
When the event handler processes the event,
Then no duplicate case is created and a Custom Log__c record (Type__c = 'Duplicate Skipped') is written for audit.

Given the Platform Event replay mechanism,
When an event fails to process due to a transient error,
Then the event is retried up to 3 times before being routed to a Dead Letter Queue custom object (Event_Dead_Letter__c) for manual review.

## Solution

Publish Credentialing_Initiation__e Platform Events from an after-insert Apex Trigger on Provider__c (scoped to bulk context via System.isBulk() check). Subscribing Apex Trigger (CredentialingEventTrigger.cls) on the Platform Event object processes events in bulkified fashion using CredentialingCaseService.cls (shared service layer with INS-0191). Duplicate detection via SOQL on Credentialing_Case__c before DML. Dead letter handling via a catch block writing to Event_Dead_Letter__c. Retry counter stored in event payload field Retry_Count__c. Platform Event channel configured with High Volume delivery. Unit tests cover 200-record bulk scenarios. Monitoring via Salesforce Event Monitoring and a custom Dashboard tracking Event_Dead_Letter__c volume.

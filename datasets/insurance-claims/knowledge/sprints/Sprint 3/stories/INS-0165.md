## Description

As a Claims Operations Manager, I need a nightly batch job that identifies Credentialing_Case__c records approaching or breaching their SLA target completion date so that credentialing supervisors are alerted and cases are automatically escalated before regulatory deadlines are missed. Most LOBs require credentialing decisions within 30 days of application receipt per NCQA standards.

## Acceptance Criteria

Given the batch job runs nightly at 11:00 PM local time,
When it queries all Credentialing_Case__c records where Status__c NOT IN ('Closed','Withdrawn','Approved','Denied'),
Then it evaluates each record against the SLA threshold defined in Credentialing_SLA_Config__mdt for the record's LOB__c.

Given a Credentialing_Case__c is within 5 business days of its SLA target,
When the batch processes that record,
Then:
  - Credentialing_Case__c.SLA_Status__c = 'At Risk'
  - A Task is created for the assigned Credentialing Specialist with Subject = 'SLA At Risk — Action Required'

Given a Credentialing_Case__c has passed its SLA target date,
When the batch processes that record,
Then:
  - Credentialing_Case__c.SLA_Status__c = 'Breached'
  - Credentialing_Case__c.Escalated__c = true
  - Credentialing_Case__c.OwnerId is updated to the Credentialing Supervisor queue
  - A Platform Event (Credentialing_SLA_Breach__e) is published with payload fields: Case_Id__c, Provider_Id__c, LOB__c, Days_Overdue__c

Given the batch completes its run,
When all records are processed,
Then a Batch_Run_Summary__c record is created with: Total_Processed__c, At_Risk_Count__c, Breached_Count__c, Run_Date__c, Status__c = 'Completed'.

Given the batch encounters a DML exception on any record,
When the error occurs,
Then the record is skipped, the exception is logged to Integration_Error_Log__c, and processing continues for remaining records.

## Solution

Implement Batchable Apex class (CredentialingSLABatch) implementing Database.Batchable<SObject> and Database.Stateful. Query scope: all open Credentialing_Case__c records. Execute method evaluates SLA using Custom Metadata Type (Credentialing_SLA_Config__mdt) keyed on LOB__c. Business day calculation uses a utility class (BusinessDayCalculator) referencing a Holiday__c custom object. Platform Event publish uses EventBus.publish(). Batch_Run_Summary__c is written in the finish() method. Scheduled via a Schedulable Apex class (CredentialingSLABatchScheduler) configured in Setup. Batch size set to 200 to stay within DML governor limits.

## Description

As a Claims Operations Manager, I need a Salesforce dashboard dedicated to Provider Credentialing workload visibility that displays open Credentialing_Case__c volumes by status, assignee, and aging buckets, so that I can identify bottlenecks, balance team workload, and ensure credentialing SLAs are met across all LOBs and provider types.

## Acceptance Criteria

Given the 'Provider Credentialing Operations' dashboard is accessed by a user with the 'Credentialing Manager' Permission Set,
When the dashboard loads,
Then it displays a minimum of five dashboard components: (1) Open Cases by Status__c (donut chart), (2) Cases by Assignee (bar chart), (3) Cases Aging > 30 Days (metric tile), (4) Cases by Credentialing_Type__c (stacked bar), (5) Average Days to Approval by LOB__c (line chart – rolling 90 days).

Given the dashboard component 'Cases Aging > 30 Days' is rendered,
When the underlying report is evaluated,
Then it counts Credentialing_Case__c records where Status__c NOT IN ('Approved', 'Denied', 'Withdrawn') AND CreatedDate <= TODAY() - 30, grouped by Owner.Name and Status__c.

Given a Credentialing Specialist (non-manager) accesses the dashboard,
When the 'Cases by Assignee' component renders,
Then the data is filtered to display only the cases owned by the running user (row-level security via Sharing Rules and report filter on OwnerId = $User.Id).

Given the dashboard is refreshed manually or on its scheduled 8:00 AM daily refresh,
When data is updated,
Then all report components reflect data as of the last full refresh and a 'Last Refreshed' timestamp is visible on the dashboard header.

Given a manager clicks on any dashboard component bar or slice,
When the drill-down action executes,
Then the user is navigated to the underlying Salesforce report pre-filtered by the selected dimension (e.g., clicking 'Under Review' slice shows all open cases with Status__c = 'Under Review').

## Solution

Create a Salesforce Lightning Dashboard ('Provider Credentialing Operations') in the Credentialing Operations app. Build five underlying Custom Report Types: (1) Credentialing Cases with Provider (Credentialing_Case__c with Provider__c), (2) Credentialing Cases with Owner (standard with User lookup). Reports use summary format with row-level filtering. Dashboard components use standard Lightning chart types (Donut, Horizontal Bar, Metric, Stacked Bar, Line). Row-level security for Specialist view is enforced via a Sharing Rule on Credentialing_Case__c (Share with Role: Credentialing Specialist) combined with a report filter formula using $User.Id. Dashboard folder is shared via the 'Credentialing_Operations_Folder' with role-based access. Scheduled dashboard refresh is configured via the Dashboard Schedule feature set to 8:00 AM EST daily. A Dynamic Dashboard variant is created for Specialist self-service views using 'Run as logged-in user' setting.

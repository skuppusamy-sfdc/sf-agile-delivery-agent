## Description

As a Credentialing Specialist, I need Salesforce to automatically retrieve provider credentialing data from CAQH ProView using the provider's CAQH_Provider_ID__c so that the Credentialing_Case__c is populated with verified education, work history, malpractice, and licensure data without manual re-entry, reducing credentialing cycle time. Extends INS-0151.

## Acceptance Criteria

Given a Credentialing_Case__c record has Status__c = 'Application Received' and the related Provider__c has a non-null CAQH_Provider_ID__c
When the 'Retrieve CAQH Data' action is triggered (manually or automatically on case creation)
Then an Apex callout is made to the CAQH ProView REST API and the response is parsed

Given a successful CAQH API response
When data is received
Then Credentialing_Case__c.CAQH_Last_Sync_Date__c = TODAY(), Credentialing_Case__c.Licensure_State__c, Credentialing_Case__c.License_Expiration_Date__c, Credentialing_Case__c.Malpractice_Coverage_Amount__c, and Credentialing_Case__c.DEA_Expiration_Date__c are updated from the response payload

Given the CAQH API returns an attestation status of 'Not Attested'
When the sync completes
Then Credentialing_Case__c.Status__c is set to 'Awaiting Provider Attestation' and a Task is created for the assigned Credentialing Specialist to follow up with the provider

Given the CAQH API returns a 401 Unauthorized response
When the callout fails
Then an error log record (Integration_Error_Log__c) is created with Error_Type__c = 'CAQH Auth Failure', Source_Record_Id__c = Credentialing_Case__c.Id, and the Credentialing Specialist is notified via email alert

Given a provider's CAQH data has been synced previously
When a re-sync is triggered
Then only fields where the CAQH value differs from the current Salesforce value are updated, and a Credentialing_Change_Log__c record is created for each changed field capturing Old_Value__c and New_Value__c

-Given CAQH data sync is triggered, also update Provider__c.Practice_Address__c from CAQH response- (deprecated — address updates handled separately via Provider Data Maintenance workflow per INS-0162)

## Solution

Develop CAQHIntegrationService Apex class with a @future(callout=true) method for async retrieval and an Invocable method for Flow-triggered calls. Named Credential 'CAQH_ProView' configured with OAuth 2.0 client credentials flow. Response deserialization via CAQHResponseWrapper inner classes. Field diff logic implemented in a separate CAQHFieldComparisonUtil class that iterates over a custom metadata type (CAQH_Field_Mapping__mdt) defining source-to-target field mappings, enabling admin-configurable sync without code changes. Change log records created via a bulkified DML operation. Integration error logging uses a dedicated Integration_Error_Log__c object with a separate sharing model. Scheduled Apex job (CAQHNightlySyncScheduler) triggers re-sync for all active Credentialing_Case__c records nightly at 2:00 AM.

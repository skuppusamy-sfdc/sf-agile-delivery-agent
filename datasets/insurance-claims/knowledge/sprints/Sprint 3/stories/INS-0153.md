## Description

As a Provider Relations Specialist, I need a Credentialing 360 FlexCard on the Provider__c record page so that I can immediately see the provider's current credentialing status, upcoming document expirations, CAQH attestation date, and open Credentialing_Case__c records without navigating to multiple related lists.

## Acceptance Criteria

Given a Provider__c record page is loaded
When the Credentialing 360 FlexCard renders
Then it displays Provider__c.NPI__c, Provider__c.NPI_Validation_Status__c, Provider__c.CAQH_Provider_ID__c, Provider__c.CAQH_Attestation_Date__c, and Provider__c.Network_Participation_Status__c in a structured header block

Given the provider has Credentialing_Case__c records
When the FlexCard loads
Then all related Credentialing_Case__c records with Status__c != 'Closed' are listed with Credentialing_Case__c.Status__c, Credentialing_Case__c.Assigned_Specialist__c, and Credentialing_Case__c.Due_Date__c

Given a provider has credentialing documents with Expiration_Date__c within 90 days
When the FlexCard renders the document section
Then expiring documents are highlighted in amber and documents already expired are highlighted in red with a label 'EXPIRED'

Given the CAQH_Attestation_Date__c is more than 120 days in the past
When the FlexCard loads
Then a banner warning is displayed: 'CAQH attestation overdue — provider re-attestation required'

Given a user with the 'Provider_Relations_Read_Only' Permission Set Group
When they view the FlexCard
Then action buttons (e.g., 'Initiate Re-Credentialing', 'Request CAQH Re-Attestation') are hidden and the card is read-only

## Solution

Build an OmniStudio FlexCard 'ProviderCredentialing360' using multiple DataRaptor Extract datasources: one querying Provider__c fields, one querying related Credentialing_Case__c records (filtered by Status__c != 'Closed'), and one querying Credentialing_Document__c records with Expiration_Date__c <= TODAY()+90. Conditional styling states applied for document expiry color coding using FlexCard State configuration. Action buttons invoke OmniScripts for re-credentialing initiation and CAQH re-attestation request. FlexCard deployed to the Provider__c Lightning Record Page via the OmniStudio FlexCard runtime component. Permission-based button visibility controlled via the FlexCard's Conditional Visibility expressions referencing the running user's Permission Set Group.

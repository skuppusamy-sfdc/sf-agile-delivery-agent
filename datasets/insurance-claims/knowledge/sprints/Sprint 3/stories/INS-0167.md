## Description

As a Credentialing Specialist, I need a Credentialing_Case__c record to be automatically created whenever a Provider__c (Person Account) is submitted for network participation so that the credentialing workflow is initiated without manual intervention and no providers are onboarded without a completed credentialing review.

## Acceptance Criteria

Given a Provider__c record has Network_Participation_Status__c set to 'Pending Credentialing' and a valid NPI is present
When the record is saved
Then a Credentialing_Case__c record is created with Credentialing_Case__c.Status__c = 'Initiated', Credentialing_Case__c.Provider__c populated with the originating Provider__c Id, and Credentialing_Case__c.Submitted_Date__c = TODAY()

Given a Credentialing_Case__c is created
When the record is inserted
Then the assigned Credentialing Specialist is notified via Salesforce in-app notification and email using the 'New Credentialing Case Assigned' email alert template

Given a Provider__c record already has an open Credentialing_Case__c (Status__c NOT IN ('Approved','Denied','Withdrawn'))
When another network participation submission is attempted
Then the system prevents duplicate Credentialing_Case__c creation and displays the error: 'An active credentialing case already exists for this provider.'

Given a Provider__c record is missing NPI field (Provider__c.NPI__c is null)
When the user attempts to set Network_Participation_Status__c to 'Pending Credentialing'
Then a validation error is thrown: 'NPI is required before submitting a provider for credentialing.'

-Given a Provider__c of type 'Facility' is submitted, a Facility_Credentialing_Case__c record type should be used- (deprecated — Facility credentialing moved to INS-0171 scope)

Given a Credentialing_Case__c is created
When viewed by a Credentialing Specialist
Then the record displays the Provider 360 FlexCard embedded on the Credentialing_Case__c layout showing NPI, TIN, specialty, and active PNA records

## Solution

Implement a Record-Triggered Flow on Provider__c object triggering on Create and Update where Network_Participation_Status__c changes to 'Pending Credentialing'. Flow checks for existing open Credentialing_Case__c via Get Records element. If none found, creates Credentialing_Case__c via Create Records element. Duplicate check enforced via before-save flow path with custom error message. Email alert configured via Flow Send Email action using org-wide address. Provider 360 FlexCard (OmniStudio) embedded via App Builder on Credentialing_Case__c record page. Validation Rule added to Provider__c to enforce NPI presence on status transition.

## Description

As a Credentialing Specialist, I need the system to automatically retrieve provider credentialing data from CAQH ProView using the provider's NPI and CAQH ID so that manual data entry on the Credentialing_Case__c record is eliminated, attestation currency is enforced, and time-to-credential is reduced from an average of 22 days to under 10 days.

## Acceptance Criteria

Given a Credentialing_Case__c record has Status__c = 'Initiated' and the related Provider__c has both NPI__c and CAQH_ID__c populated,
When the 'Fetch CAQH Data' button is clicked or the record enters 'Initiated' status,
Then a callout is made to CAQH ProView API and the following fields are populated on Credentialing_Case__c:
  - CAQH_Last_Attestation_Date__c
  - Board_Certification__c
  - DEA_Number__c
  - Malpractice_Insurance_Expiry__c
  - Medical_License_State__c
  - Medical_License_Expiry__c
  - CAQH_Profile_Status__c ('Complete', 'Incomplete', 'Expired')

Given CAQH_Profile_Status__c = 'Expired' (attestation older than 120 days),
When data is fetched,
Then Credentialing_Case__c.Status__c is set to 'Pending Re-attestation' and an automated email is sent to the provider at Provider__c.Email using the 'CAQH Re-attestation Required' template.

Given CAQH_ID__c is NOT populated on Provider__c,
When the fetch is attempted,
Then the system displays: 'CAQH ID is required. Please update the Provider record before fetching credentialing data.' and no callout is made.

Given the CAQH API returns a successful response,
When data is written to Credentialing_Case__c,
Then a timestamped entry is appended to Credentialing_Case__c.Data_Fetch_Log__c (Long Text Area) recording the fetch datetime, fields updated, and API response code.

Given Malpractice_Insurance_Expiry__c < TODAY() + 60 days,
When credentialing data is populated,
Then a high-priority Task is created on the Credentialing_Case__c assigned to the Credentialing Specialist with Subject = 'Malpractice Insurance Expiring Soon — Action Required'.

## Solution

Create Apex class 'CAQHProViewService' with HttpCallout to CAQH ProView REST API using Named Credential 'CAQH_ProView'. Parse JSON response and map to Credentialing_Case__c fields via a dedicated field mapping Apex method. Invoke via a Screen Flow 'Fetch CAQH Data' launched from a custom Quick Action button on Credentialing_Case__c, and also via a Record-Triggered Flow on status change to 'Initiated' calling the Apex as an invocable action. Implement Queueable Apex for async execution to avoid blocking UI. Use Custom Metadata Type 'CAQH_Field_Mapping__mdt' to store field-to-JSON-path mappings for maintainability. Log all API interactions to a custom object 'Integration_Audit_Log__c'. Ensure all CAQH credentials are stored in Named Credentials; no plaintext secrets in code. Test with mock HttpCalloutMock for complete, incomplete, and expired profile scenarios.

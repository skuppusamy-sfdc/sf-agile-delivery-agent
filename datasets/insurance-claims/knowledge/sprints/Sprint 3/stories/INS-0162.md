## Description

As a Credentialing Specialist, I need the system to retrieve a provider's credentialing documents and attestation status from CAQH ProView automatically when a Credentialing_Case__c is opened so that I don't have to manually request and upload documents, reducing credentialing cycle time from an average of 14 days to under 7 days.

This story picks up TIN validation requirements originally scoped in INS-0159.

## Acceptance Criteria

Given a Credentialing_Case__c.Status__c transitions to 'Open' and the linked Provider__c.CAQH_Provider_ID__c is populated,
When the status transition is saved,
Then an outbound callout is made to the CAQH ProView REST API to retrieve the provider's attestation status and document list.

Given the CAQH API returns a successful response,
When documents are retrieved,
Then:
  - Credentialing_Case__c.CAQH_Attestation_Status__c is set to the returned status value
  - Credentialing_Case__c.CAQH_Last_Sync_Date__c = NOW()
  - Credentialing_Case__c.TIN__c is populated from CAQH tax_identification_number field
  - Retrieved document metadata is stored as ContentDocument records linked to the Credentialing_Case__c via ContentDocumentLink

Given the provider's CAQH attestation is expired (attestation_date > 365 days ago),
When the sync completes,
Then Credentialing_Case__c.CAQH_Attestation_Status__c = 'Expired' and the case is auto-flagged with Credentialing_Case__c.Requires_Re_Attestation__c = true.

Given Provider__c.CAQH_Provider_ID__c is blank when the case opens,
When the transition is saved,
Then Credentialing_Case__c.CAQH_Sync_Status__c = 'Skipped — No CAQH ID' and a task is created for the assigned specialist to collect the CAQH ID.

Given the CAQH API returns HTTP 401 (unauthorized),
When the error is received,
Then an Apex exception is logged to Integration_Error_Log__c with fields: Error_Code__c = '401', Source_System__c = 'CAQH', Related_Record_Id__c = Credentialing_Case__c.Id, and the Compliance Officer is notified via email alert.

Given a successful CAQH sync,
When Credentialing_Case__c.TIN__c is populated,
Then a duplicate check is run against existing Provider__c.TIN__c; if mismatch detected, Credentialing_Case__c.TIN_Mismatch_Flag__c = true.

## Solution

Record-Triggered Flow on Credentialing_Case__c fires on Status__c change to 'Open', invokes an Apex Action (CAQHSyncAction) that calls a Queueable Apex class (CAQHProViewQueueable) using a Named Credential ('CAQH_ProView'). Document metadata is handled via ContentVersion insert with a NetworkId scoped to the internal community. Integration_Error_Log__c inserts are handled in a catch block with a separate savepoint to avoid rollback. TIN mismatch logic is a separate Apex method called post-sync. A Custom Metadata Type (CAQH_Config__mdt) stores environment-specific API versions and attestation expiry thresholds for easy configuration without code changes.

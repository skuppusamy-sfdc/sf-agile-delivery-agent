## Description

As a Provider Network Specialist, I need a guided OmniScript that allows me to search the NPPES registry by NPI number or provider name, review the returned data, and create or update a Provider__c (Person Account) record pre-populated with NPPES data, so that manual data entry errors are minimized during provider onboarding and data quality standards are maintained from the point of record creation.

## Acceptance Criteria

Given a Provider Network Specialist launches the 'NPI Lookup & Provider Create' OmniScript from the Provider__c list view action button
When Step 1 (Search) is completed with either NPI__c or Last_Name + State combination
Then an Integration Procedure (IP_NPPES_Search) calls the NPPES API and returns up to 10 matching results displayed in a DataTable with columns: NPI, Provider Name, Taxonomy, City, State, Enumeration Type

Given search results are displayed
When the specialist selects a result row
Then Step 2 (Review & Map) pre-populates the following fields from NPPES data:
  - Provider__c.FirstName, Provider__c.LastName
  - Provider__c.NPI__c
  - Provider__c.NPI_Taxonomy_Code__c
  - Provider__c.Mailing_State__c, Provider__c.Mailing_City__c, Provider__c.Mailing_Zip__c
  - Provider__c.Enumeration_Type__c ('Individual' or 'Organizational')
  - Provider__c.NPI_Validation_Status__c = 'Validated'
  - Provider__c.NPI_Validated_Date__c = TODAY()

Given the specialist is on Step 2
When the NPI already exists on an existing Provider__c record
Then a duplicate warning panel displays: 'Provider with NPI [X] already exists: [Provider Name] ([Record Link])'
And the specialist is offered two actions: 'Update Existing Record' or 'Cancel'
And the 'Create New Record' path is blocked

Given the specialist confirms creation on Step 3 (Confirm & Submit)
When the OmniScript submits
Then a new Provider__c Person Account is created with RecordType = 'Provider' and Provider_Status__c = 'Pending Credentialing'
And the user is redirected to the newly created Provider__c record page
And the Provider 360 FlexCard (INS-0209) loads automatically

Given the NPPES API returns zero results
When Step 1 search completes
Then an informational message is shown: 'No providers found matching your search criteria. Please verify the NPI or broaden your search.'
And a 'Manual Entry' fallback button is displayed that advances to Step 2 with all fields blank and NPI_Validation_Status__c = 'Unvalidated'

-Step 4 (Fee Schedule Assignment) to be included in this OmniScript- (decoupled; Fee Schedule assignment handled post-credentialing via separate workflow)

## Solution

Build a 4-step OmniScript (OS_NPI_Lookup_Provider_Create) using OmniStudio. Step 1 uses a Remote Action to call Integration Procedure (IP_NPPES_Search), which uses an HTTP Action to query NPPES REST API via Named Credential. Step 2 uses a DataRaptor Extract (DR_CheckDuplicateNPI) to query existing Provider__c records by NPI__c before allowing progression. The duplicate check result drives a Conditional View to show/hide the warning panel. Step 3 uses a DataRaptor Upsert (DR_UpsertProvider) to create or update the Provider__c record. Navigation post-submit uses a Navigate Action pointed at the new record ID returned by the DataRaptor. The OmniScript is surfaced as a Quick Action on the Provider__c list view and as a Global Action in the App Launcher. Permission Set 'Provider_Onboarding_OS' controls access.

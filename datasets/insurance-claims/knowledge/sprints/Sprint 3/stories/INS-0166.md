## Description

As a Provider Network Specialist, I need a Provider_Network_Agreement__c (PNA) record to be automatically created and linked to the appropriate Fee_Schedule__c when a Credentialing_Case__c is approved so that newly credentialed providers are immediately active in the network with correct reimbursement rates, eliminating the current manual contract setup step that averages 3 days post-approval.

This story extends INS-0159 (Credentialing Case auto-creation) by completing the downstream contracting workflow.

## Acceptance Criteria

Given a Credentialing_Case__c.Status__c transitions to 'Approved',
When the status is saved,
Then a new Provider_Network_Agreement__c is created with:
  - PNA__c.Provider__c = Credentialing_Case__c.Provider__c
  - PNA__c.Status__c = 'Active'
  - PNA__c.Effective_Date__c = TODAY()
  - PNA__c.LOB__c = Credentialing_Case__c.LOB__c
  - PNA__c.Credentialing_Case__c = the approving case Id

Given the new PNA is created,
When the system looks up the applicable Fee_Schedule__c,
Then it queries Fee_Schedule__c where Fee_Schedule__c.LOB__c = PNA__c.LOB__c AND Fee_Schedule__c.Effective_Date__c <= TODAY() AND Fee_Schedule__c.Expiration_Date__c >= TODAY() AND Fee_Schedule__c.Is_Default__c = true,
And PNA__c.Fee_Schedule__c is set to the matching record's Id.

Given no active default Fee_Schedule__c is found for the provider's LOB,
When the PNA is created,
Then PNA__c.Fee_Schedule__c remains blank, PNA__c.Fee_Schedule_Status__c = 'Pending Assignment', and a Task is created for the Provider Network Specialist queue with Subject = 'Fee Schedule Assignment Required — New PNA'.

Given the Credentialing_Case__c.Type__c = 'Re-Credentialing',
When the case is approved,
Then instead of creating a new PNA, the existing active PNA (linked via Credentialing_Case__c.Related_PNA__c) is updated:
  - PNA__c.Last_Recredentialed_Date__c = TODAY()
  - PNA__c.Next_Recredential_Due_Date__c = TODAY() + 730 days (2 years)

-Given PNA creation should also trigger a DocuSign envelope to the provider for countersignature- (deprecated: DocuSign countersignature flow deferred to Phase 2 per stakeholder decision on 2024-11-14)

Given the PNA is successfully created or updated,
When the operation completes,
Then Provider__c.Network_Participation_Status__c is updated to 'Active — In Network' and a Provider_Change_Notification__c (PCN) record is created with PCN__c.Change_Type__c = 'New Network Participation' for downstream notifications.

## Solution

Record-Triggered Flow on Credentialing_Case__c fires on Status__c change to 'Approved'. A Decision element branches on Credentialing_Case__c.Type__c ('Initial' vs 'Re-Credentialing'). For initial credentialing: Create Records element provisions PNA; a Get Records element fetches the default Fee_Schedule__c using filter criteria. For re-credentialing: Get Records fetches the existing PNA via Related_PNA__c lookup, then Update Records sets recredentialing date fields. Fee schedule not-found path creates a Task via Create Records. Provider__c status update and PCN creation handled in the same Flow via additional Update/Create Records elements to keep all logic in one transaction. Flow uses Fault paths on each DML element to log errors to Integration_Error_Log__c via an Apex Action.

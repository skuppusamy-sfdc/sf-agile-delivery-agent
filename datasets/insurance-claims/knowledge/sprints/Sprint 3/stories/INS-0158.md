# INS-0158: Bug — Provider_Network_Agreement__c Effective_Date__c incorrectly defaults to current date instead of 30 days forward when created from Credentialing_Case__c approval

## Description

Steps to Reproduce:
1. Navigate to a Credentialing_Case__c record with Status__c = 'Committee Review'.
2. Use the 'Approve' quick action to advance Status__c to 'Approved'.
3. Observe the auto-created Provider_Network_Agreement__c record.
4. Check PNA__c.Effective_Date__c on the newly created record.

Expected Behavior: PNA__c.Effective_Date__c should be set to TODAY() + 30 days as specified in the acceptance criteria for INS-0156.

Actual Behavior: PNA__c.Effective_Date__c is being set to TODAY() (current date) instead of TODAY() + 30. This causes the PNA to appear immediately effective upon creation, which is incorrect per contracting policy and creates downstream issues in Fee_Schedule__c effective date lookups and claims adjudication rate resolution.

Additional Notes: Issue is reproducible 100% of the time in Full Sandbox. The Record-Triggered Flow 'Credentialing_Approval_PNA_Creation_Flow' was reviewed — the Create Records element for PNA__c uses a formula for Effective_Date__c that appears to reference TODAY() without the +30 offset. The formula field reads: {!$Flow.CurrentDate} with no addition. Likely a copy-paste error from an earlier draft of the flow where the date was set dynamically. Introduced in Sprint 2 deployment. No workaround currently available for the Credentialing team.

## Acceptance Criteria

As per Description. PNA__c.Effective_Date__c must equal TODAY() + 30 days when auto-created from a Credentialing_Case__c approval. Verified in Full Sandbox and UAT before Production deployment.

## Solution

Open the Record-Triggered Flow 'Credentialing_Approval_PNA_Creation_Flow' in Flow Builder. Locate the Create Records element for Provider_Network_Agreement__c. Update the Effective_Date__c formula value from {!$Flow.CurrentDate} to {!$Flow.CurrentDate} + 30 using a separate Formula resource (e.g., 'frmPNAEffectiveDate' of type Date with formula: TODAY() + 30). Save and activate the updated flow version. Deactivate the prior version. Regression test by approving a Credentialing_Case__c in Full Sandbox and asserting PNA__c.Effective_Date__c = TODAY() + 30. Also verify no impact to PNA__c.DocuSign_Sent_Date__c or Fee_Schedule__c lookups. Update flow version notes to document the fix and reference this bug ID INS-0158.

## Description

As a Provider Network Specialist, I need NPI numbers entered on Provider__c and Claim__c records to be validated in real time against the NPPES (National Plan and Provider Enumeration System) registry, so that invalid or inactive NPIs are caught at data entry rather than causing downstream claim adjudication failures or compliance violations.

## Acceptance Criteria

Given a user enters a value in Provider__c.NPI__c and tabs out of the field
When the LWC field-level validation fires
Then a callout is made to the NPPES NPI Registry API (https://npiregistry.cms.hhs.gov/api)
And if the NPI is valid and active, Provider__c.NPI_Validation_Status__c = 'Validated' and Provider__c.NPI_Validated_Date__c = TODAY()
And the provider's legal name returned by NPPES is displayed in a read-only field Provider__c.NPPES_Legal_Name__c for visual confirmation

Given the NPPES API returns no results for the entered NPI
When validation completes
Then an inline error message is displayed: 'NPI not found in NPPES registry. Please verify and re-enter.'
And Provider__c.NPI_Validation_Status__c = 'Invalid'
And the record cannot be saved until NPI is corrected or an override reason is logged in Provider__c.NPI_Override_Reason__c (requires Credentialing Manager permission)

Given a Claim__c record is being created or edited with Rendering_Provider_NPI__c populated
When the claim is saved
Then an Apex Trigger validates the NPI against a cached NPPES result in Provider__c (if provider exists in org) or performs a live callout if no cached record is found
And if NPI is invalid, Claim__c.Status__c is set to 'Pended - Invalid NPI' and Claim__c cannot progress to adjudication

Given NPI validation succeeds and Provider__c.NPI_Taxonomy_Code__c is returned by NPPES
When the record is saved
Then Provider__c.NPI_Taxonomy_Code__c is auto-populated from the NPPES response

Given the NPPES API is unavailable (timeout or 5xx error)
When validation is attempted
Then a non-blocking warning is shown: 'NPI validation service temporarily unavailable. Validation will be retried automatically.'
And Credentialing_Case__c.NPI_Validation_Status__c = 'Pending Retry'

## Solution

Build a custom LWC component (npiValidationField) that wraps the standard NPI input field and calls an Apex controller (NPIValidationController.cls) via @wire or imperative Apex on the blur event. NPIValidationController performs a callout to NPPES using a Named Credential ('NPPES_Registry_NC'). Results are cached in a Platform Cache partition (NPICache) keyed by NPI number with a 24-hour TTL to reduce redundant external callouts. For Claim__c, implement an Apex Before-Insert/Before-Update trigger (ClaimNPIValidationTrigger.cls) that checks Platform Cache first before making a live callout. A Custom Permission ('NPI_Override') controls access to the override reason field. Deploy a Scheduled Apex job (NPIRevalidationBatch.cls) running nightly to re-validate all Provider__c records with NPI_Validated_Date__c older than 90 days.

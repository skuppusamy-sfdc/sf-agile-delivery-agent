## Description

As a Compliance Officer, I need a scheduled batch job to periodically re-validate NPI numbers for all active Provider__c records against the NPPES registry so that deactivated, invalid, or mismatched NPIs are identified promptly and flagged for remediation, supporting regulatory compliance and clean provider data for claims adjudication.

## Acceptance Criteria

Given the batch job 'NPPESNightlyValidationBatch' is scheduled to run nightly
When it executes
Then all Provider__c records where Network_Participation_Status__c = 'Active' AND NPI__c != null are included in the batch scope

Given a provider's NPI is re-validated against NPPES
When the API response indicates the NPI is deactivated
Then Provider__c.NPI_Validation_Status__c is set to 'Deactivated', Provider__c.NPI_Deactivation_Date__c is populated, and a Provider_Change_Notification__c (PCN) record is created with PCN__c.Change_Type__c = 'NPI Deactivated' and PCN__c.Priority__c = 'High'

Given the NPPES response shows a name or taxonomy mismatch compared to the current Provider__c record
When the discrepancy is detected
Then a Provider_Change_Notification__c is created with PCN__c.Change_Type__c = 'Data Mismatch', PCN__c.Discrepancy_Details__c populated with a JSON summary of differing fields, and PCN__c.Status__c = 'Pending Review'

Given the batch job completes processing
When all records have been evaluated
Then a Batch_Job_Summary__c record is created capturing Total_Records_Processed__c, Records_With_Discrepancies__c, Records_Deactivated__c, and Job_Completion_Timestamp__c

Given a batch run produces more than 50 discrepancy records
When the job completes
Then an email report is sent to the Compliance Officer distribution list with a summary attachment generated via Conga

Given the NPPES API rate limit is reached during batch execution
When a 429 response is received
Then the batch pauses callouts for that chunk and logs an Integration_Error_Log__c record, resuming on the next scheduled execution without marking affected providers as invalid

## Solution

Implement Database.Batchable and Database.AllowsCallouts Apex class 'NPPESNightlyValidationBatch' with a batch size of 10 to respect NPPES API rate limits (10 callouts per execution context). Each execute() chunk calls NPPESIntegrationService.validateNPI() (reused from INS-0152). A custom metadata type (NPPES_Field_Comparison__mdt) defines which Provider__c fields to compare against NPPES response fields. PCN records created via a bulkified helper. Batch_Job_Summary__c record created in the finish() method. Rate limit handling uses a try-catch on CalloutException with HTTP status 429 check, logging to Integration_Error_Log__c. Job scheduled via Scheduled Apex (NPPESValidationScheduler) set to execute at 1:00 AM daily. Email report triggered by a Process-level Invocable Action calling Conga if Records_With_Discrepancies__c > 50.

## Description

As a Credentialing Specialist, I need the system to automatically initiate a re-credentialing Credentialing_Case__c when a provider's active Provider_Network_Agreement__c is within 90 days of expiration so that re-credentialing is proactively managed, network continuity is maintained, and Acme Insurance remains compliant with NCQA re-credentialing interval standards (every 36 months). This story extends the credentialing workflow established in INS-0167.

## Acceptance Criteria

Given a scheduled job runs nightly at 2:00 AM
When a Provider_Network_Agreement__c record has PNA.Status__c = 'Active' and PNA.Expiration_Date__c = TODAY() + 90
Then a new Credentialing_Case__c is created with: Credentialing_Case__c.Record_Type = 'Re-Credentialing', Credentialing_Case__c.Provider__c = PNA.Provider__c, Credentialing_Case__c.Related_PNA__c = PNA.Id, Credentialing_Case__c.Status__c = 'Initiated', Credentialing_Case__c.Due_Date__c = PNA.Expiration_Date__c - 30

Given a re-credentialing Credentialing_Case__c is created
When the record is inserted
Then the assigned Credentialing Specialist receives an in-app notification and email: 'Re-credentialing required for [Provider Name] — PNA expires on [Expiration Date].'

Given a PNA already has an open re-credentialing Credentialing_Case__c (Status__c NOT IN ('Approved','Denied','Withdrawn'))
When the nightly job runs
Then no duplicate Credentialing_Case__c is created for that PNA

Given a re-credentialing Credentialing_Case__c reaches Status__c = 'Approved'
When the record is saved
Then PNA.Expiration_Date__c is extended by 36 months and PNA.Last_Recredentialing_Date__c = TODAY()

Given a re-credentialing Credentialing_Case__c reaches Status__c = 'Denied' or remains open past PNA.Expiration_Date__c
When the condition is detected
Then PNA.Status__c is set to 'Expired', Provider__c.Network_Participation_Status__c is set to 'Out of Network', and a Provider_Change_Notification__c (PCN) record is created with PCN.Change_Type__c = 'Network Termination — Re-credentialing Failure'

Given the re-credentialing process is initiated
When Credentialing_Case__c.Record_Type = 'Re-Credentialing'
Then the CAQH data fetch (per INS-0169 integration) is automatically triggered without requiring manual Specialist action

-Given re-credentialing is triggered, also suspend active claims payment for the provider during the review window- (deprecated — claims payment suspension during re-credentialing removed per Compliance Officer review dated Sprint 2)

## Solution

Apex Schedulable class (RecredentialingInitiationScheduler) scheduled via Setup > Scheduled Jobs to run nightly at 02:00 AM. Scheduler executes a SOQL query on Provider_Network_Agreement__c filtering on Status__c = 'Active' and Expiration_Date__c = TODAY() + 90. Batch Apex class (RecredentialingBatch) processes results in chunks of 200. Duplicate check performed via subquery on Credentialing_Case__c for open cases linked to each PNA.Id. Credentialing_Case__c creation uses a dedicated Re-Credentialing Record Type. Email notifications sent via Messaging.sendEmail in Apex. PNA extension and PCN creation handled by a separate Record-Triggered Flow on Credentialing_Case__c (after-update) detecting Status__c transitions. CAQH auto-fetch invoked via Platform Event (CAQH_Fetch_Request__e) published on Credentialing_Case__c creation, consumed by CAQHIntegrationService. All batch operations wrapped in try-catch with errors logged to Integration_Error_Log__c.

# INS-0190: Configure Credentialing Case assignment rules to route cases by LOB and provider specialty

## Description

As a Claims Operations Manager, I need Credentialing_Case__c records to be automatically assigned to the correct Credentialing Specialist based on the case's LOB__c and the provider's primary specialty (Provider__c.Primary_Specialty__c), so that workload is distributed according to team expertise and cases are not left unassigned in a generic queue. Extends INS-0186.

## Acceptance Criteria

Given a new Credentialing_Case__c is created with LOB__c = 'Medicare Advantage' and Provider__c.Primary_Specialty__c = 'Cardiology',
When the record is saved,
Then the OwnerId is set to the Credentialing Specialist configured for the 'Medicare Advantage – Cardiology' routing rule in the Assignment_Routing_Rule__mdt Custom Metadata Type.

Given a Credentialing_Case__c is created with a LOB__c and specialty combination that does not match any specific routing rule,
When the assignment logic runs,
Then the OwnerId falls back to the default queue defined in Assignment_Routing_Rule__mdt where Rule_Type__c = 'Default' and the case is assigned to the 'Credentialing – General Queue'.

Given a Credentialing_Case__c.Entity_Type__c = 'Facility',
When the assignment logic runs,
Then routing is based solely on LOB__c (facility cases do not have a provider specialty), and the assigned specialist is drawn from the 'Facility Credentialing' specialist group.

Given the assigned Credentialing Specialist is marked as inactive (User.IsActive = false) in Salesforce,
When the assignment rule attempts to assign the case to that user,
Then the system falls back to the specialist's backup assignee defined in Credentialing_Specialist_Config__mdt.Backup_Assignee__c, and a Chatter post is added noting the fallback assignment.

Given a Credentialing_Case__c ownership is changed manually by a Claims Operations Manager,
When the manual reassignment is saved,
Then the automated assignment rule does not override the manual assignment, and Credentialing_Case__c.Manual_Assignment_Override__c is set to true to suppress future auto-routing.

## Solution

Implement assignment logic using a Record-Triggered Flow on Credentialing_Case__c (After Save, on create) that reads routing configuration from a Custom Metadata Type (Assignment_Routing_Rule__mdt) with fields: LOB__c, Specialty__c, Assigned_User_Id__c, Queue_Id__c, Rule_Type__c. A loop element iterates matching rules in priority order. An Apex invocable method (CredentialingAssignmentService.cls) handles the active-user check and backup assignee fallback to keep Flow complexity manageable. The Manual_Assignment_Override__c field is a checkbox defaulting to false; a separate Flow path checks this field before running assignment logic on subsequent updates. A Permission Set restricts the ability to set Manual_Assignment_Override__c to the Claims Operations Manager role. Custom Metadata records are pre-populated via a Metadata deployment script as part of this story's delivery.

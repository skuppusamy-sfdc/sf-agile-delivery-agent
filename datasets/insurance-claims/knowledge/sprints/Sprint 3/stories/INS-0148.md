## Description

As a Provider Network Specialist, I need the ability to generate a Provider_Network_Agreement__c record and automatically send the associated PNA contract document for e-signature via DocuSign when a Credentialing_Case__c is moved to 'Approved' status, so that network onboarding is accelerated and paper-based contract execution is eliminated.

## Acceptance Criteria

Given a Credentialing_Case__c.Status__c transitions to 'Approved' and Credentialing_Case__c.NPI_Validation_Status__c = 'Validated',
When the Record-Triggered Flow fires on the Credentialing_Case__c after-update,
Then a new Provider_Network_Agreement__c is automatically created with PNA__c.Provider__c = Credentialing_Case__c.Provider__c, PNA__c.Status__c = 'Pending Signature', PNA__c.Effective_Date__c = today + 1 business day, and PNA__c.LOB__c inherited from Credentialing_Case__c.LOB__c.

Given the Provider_Network_Agreement__c is created with Status__c = 'Pending Signature',
When the DocuSign envelope generation action fires,
Then a DocuSign envelope is created using the 'Standard_PNA_Template' DocuSign template, pre-populated with Provider__c.Name, Provider__c.NPI__c, Provider__c.TIN__c, PNA__c.Effective_Date__c, and PNA__c.LOB__c merge fields, and sent to Provider__c.Email.

Given the provider completes DocuSign e-signature,
When the DocuSign status callback updates the envelope to 'Completed',
Then PNA__c.Status__c is updated to 'Active', PNA__c.Signed_Date__c is stamped, the executed PDF is attached to the PNA__c record as a Salesforce File (ContentDocument), and Provider__c.Network_Status__c is set to 'In-Network'.

Given the provider declines to sign or the DocuSign envelope expires (after 14 days),
When the DocuSign callback fires with status 'Declined' or 'Voided',
Then PNA__c.Status__c is set to 'Signature Failed', Credentialing_Case__c.Status__c is reverted to 'Approved — Awaiting Contract', and a Task is created for the Provider Network Specialist to follow up.

Given a PNA already exists for the provider with Status__c = 'Active' for the same LOB__c,
When the Credentialing_Case__c approval flow fires,
Then no duplicate PNA is created; instead, the existing PNA__c.Renewal_Triggered__c checkbox is set to TRUE and a renewal task is created.

## Solution

Record-Triggered Flow on Credentialing_Case__c (After Update, when Status__c changes to 'Approved') calls an Apex @InvocableMethod 'PNACreationService.createPNA()' which handles duplicate PNA detection via SOQL and creates the PNA record. DocuSign integration uses the DocuSign eSignature for Salesforce managed package — envelope send invoked via DocuSign Apex Toolkit (DSFSEnvelopeService). DocuSign template 'Standard_PNA_Template' pre-configured in DocuSign admin with Salesforce merge field mappings. Callback handling via DocuSign Connect webhook updating PNA__c status via an Apex REST endpoint. File attachment uses ContentVersion/ContentDocumentLink insertion. Permission Set 'PNA_Network_Specialist_PS' grants access to PNA creation and DocuSign send actions.

## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces credentialing status, active Provider_Network_Agreement__c records, NPI validation state, and recent claims activity, so that I can assess a provider's full standing without navigating across multiple objects and tabs. Extends INS-0208 (NPI validation) and INS-0206 (Credentialing Case auto-creation).

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page
When the Provider 360 FlexCard loads
Then the following data blocks are displayed:
  Block 1 – Identity: Provider__c.NPI__c, Provider__c.NPI_Validation_Status__c (badge: green='Validated', red='Invalid', amber='Pending Retry'), Provider__c.TIN__c, Provider__c.Specialty__c
  Block 2 – Credentialing: Most recent Credentialing_Case__c.Status__c, Credentialing_Case__c.Due_Date__c, Credentialing_Case__c.Assigned_Specialist__c
  Block 3 – Network Agreements: List of active Provider_Network_Agreement__c records with PNA__c.Network_Name__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c
  Block 4 – Recent Claims: Last 5 Claim__c records with Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c

Given the provider has no active Credentialing_Case__c
When the FlexCard loads
Then Block 2 displays 'No Active Credentialing Case' with an action button 'Initiate Credentialing' that launches the credentialing OmniScript

Given the provider's Provider_Network_Agreement__c.Termination_Date__c is within 30 days
When the FlexCard renders Block 3
Then the affected PNA row is highlighted in amber with tooltip 'Agreement expiring soon'

Given the FlexCard is viewed by a user without the 'Provider_Financials_View' Permission Set
When Block 4 renders
Then Claim__c.Total_Billed_Amount__c is masked and displayed as '***'

-Block 5 – Fee Schedule summary to be included in this card- (moved to standalone Fee Schedule FlexCard in Phase 2)

## Solution

Build the Provider 360 FlexCard using OmniStudio FlexCards with child cards for each data block. DataRaptors (DR_GetProviderCredentialing, DR_GetActiveAgreements, DR_GetRecentClaims) retrieve data from respective objects using SOQL with appropriate field-level security enforcement. The NPI status badge uses a conditional state on the FlexCard driven by Provider__c.NPI_Validation_Status__c picklist value. The 'Initiate Credentialing' action button invokes the Credentialing OmniScript (built in INS-0206). Financial masking is implemented via a DataRaptor Transform that checks the running user's Permission Set assignments using a custom Apex Remote Action (UserPermissionCheckAction.cls). The FlexCard is deployed to the Provider__c Lightning Record Page via App Builder with audience targeting restricting visibility to the 'Provider Relations' and 'Credentialing' profiles.

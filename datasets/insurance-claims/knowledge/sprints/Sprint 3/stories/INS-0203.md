## Description

As a Credentialing Specialist, I need the Credentialing_Case__c to follow a governed status lifecycle with automated approval routing to a Credentialing Committee queue when all primary source verifications are complete so that credentialing decisions are auditable, compliant with NCQA standards, and do not rely on ad-hoc email communication.

## Acceptance Criteria

Given a Credentialing_Case__c record exists with Status__c = 'In Review'
When all of the following fields are populated and verified:
  - Credentialing_Case__c.NPI_Verified__c = TRUE
  - Credentialing_Case__c.License_Verified__c = TRUE
  - Credentialing_Case__c.Malpractice_Verified__c = TRUE
  - Credentialing_Case__c.CAQH_Profile_Status__c = 'Complete'
Then Status__c automatically transitions to 'Pending Committee Review' and the record is submitted to the 'Credentialing Committee Approval Process'

Given the record enters the Approval Process
When it is assigned to the Credentialing Committee queue
Then an email notification is sent to all queue members using an Email Alert with template 'Credentialing_Committee_Review_Request'
And Credentialing_Case__c.Submitted_to_Committee_Date__c = TODAY()

Given a Committee member approves the Credentialing_Case__c
When the approval step is completed
Then Credentialing_Case__c.Status__c = 'Closed - Approved'
And Provider__c.Provider_Status__c = 'Credentialed'
And Provider__c.Credentialing_Effective_Date__c = TODAY()
And a Provider_Change_Notification__c (PCN) record is created with PCN__c.Change_Type__c = 'Credentialing Approval'

Given a Committee member rejects the Credentialing_Case__c
When the rejection step is completed with a required Rejection_Reason__c comment
Then Credentialing_Case__c.Status__c = 'Closed - Denied'
And Provider__c.Provider_Status__c = 'Credentialing Denied'
And an automated letter task is created for the Provider Relations Specialist

Given Credentialing_Case__c.Due_Date__c = TODAY() and Status__c NOT IN ('Closed - Approved','Closed - Denied')
When the nightly scheduled job runs
Then an escalation email is sent to the Credentialing Specialist's manager and Credentialing_Case__c.Escalated__c = TRUE

## Solution

Configure a Salesforce Approval Process 'Credentialing_Committee_Review' on Credentialing_Case__c with entry criteria: NPI_Verified__c = TRUE AND License_Verified__c = TRUE AND Malpractice_Verified__c = TRUE AND CAQH_Profile_Status__c = 'Complete'. Initial submission triggered by a Record-Triggered Flow on Credentialing_Case__c detecting the transition to 'Pending Committee Review' using an Apex action 'SubmitForApproval' invoking Approval.ProcessSubmitRequest. Approval steps route to the 'Credentialing Committee' Queue. Final approval actions update Provider__c fields via a Flow launched from the Approval Process Final Approval Action (Apex-invocable method). PCN record creation handled in the same Apex-invocable method. Escalation handled by a Scheduled Flow running daily filtering on Due_Date__c <= TODAY() and Escalated__c = FALSE, sending email via a Send Email action and updating the flag. All approval history auditable via the standard Approval History related list.

## Description

As a Credentialing Specialist, I need the system to automatically retrieve provider credentialing documents and attestation data from CAQH ProView via API when a Credentialing_Case__c is opened so that manual document collection time is reduced, credentialing cycle time targets are met, and compliance with NCQA credentialing standards is maintained.

## Acceptance Criteria

Given a Credentialing_Case__c record is created with Status__c = 'Initiated' and the linked Provider__c has a populated CAQH_Provider_ID__c field,
When the CAQH ProView API is called (within 15 minutes of case creation),
Then the following fields on Credentialing_Case__c are auto-populated from the CAQH response:
  - CAQH_Last_Attestation_Date__c
  - CAQH_Authorization_Status__c ('Authorized' | 'Not Authorized' | 'Expired')
  - CAQH_Profile_Complete__c (Boolean)
  - License_Expiry_Date__c (from primary state license in CAQH response)

Given the CAQH API returns authorization status = 'Not Authorized',
When the response is processed,
Then Credentialing_Case__c.Status__c is updated to 'Awaiting Provider Authorization' and an automated email is sent to the provider's Provider__c.Email using email template 'CAQH_Authorization_Request_ET'.

Given CAQH returns a complete and authorized profile,
When documents are available,
Then CAQH document URLs are stored as ContentDocument records linked to the Credentialing_Case__c via ContentDocumentLink, with document category tag 'CAQH_Retrieved'.

Given the CAQH API returns an error or the provider has no CAQH_Provider_ID__c,
When the integration fails,
Then Credentialing_Case__c.CAQH_Integration_Status__c = 'Manual Collection Required' and a Task is created assigned to the case owner with Subject = 'Manual Credential Document Collection Required'.

-Given CAQH documents should be stored in a legacy SharePoint folder via MuleSoft- (deprecated — document storage standardized to Salesforce Files/ContentDocument per platform readiness decision)

Given a Credentialing Specialist clicks the 'Refresh CAQH Data' button on the Credentialing_Case__c record,
When clicked,
Then a fresh CAQH API call is made, all CAQH-populated fields are refreshed, and a field CAQH_Last_Sync_DateTime__c is updated to the current timestamp.

## Solution

Implement a Scheduled Apex job (CAQHSyncScheduler.cls) running every 15 minutes querying Credentialing_Case__c records with Status__c = 'Initiated' and CAQH_Integration_Status__c = 'Pending'. Callout logic in CAQHProViewService.cls using Named Credential 'CAQH_ProView_NC'. Response deserialized via CAQHResponseWrapper.cls. ContentDocument creation handled in DocumentStorageService.cls. 'Refresh CAQH Data' button implemented as a Quick Action invoking an Apex-backed LWC (caqhRefreshAction). Email sending via Messaging.sendEmail() with org-wide address. All CAQH field mappings documented in Custom Metadata Type (CAQH_Field_Mapping__mdt) for admin configurability without code changes.

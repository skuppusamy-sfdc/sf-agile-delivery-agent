## Description

As a Credentialing Specialist, I need the system to automatically validate a provider's NPI against the NPPES registry when a new Credentialing_Case__c record is created, so that I can immediately identify invalid or deactivated NPIs before investing credentialing effort and avoid downstream claim adjudication failures.

## Acceptance Criteria

Given a new Credentialing_Case__c is submitted with a Provider__c lookup populated,
When the record is saved and Provider__c.NPI__c is not null,
Then a callout is made to the NPPES NPI Registry API (https://npiregistry.cms.hhs.gov/api) using the NPI value.

Given the NPPES API returns a valid, active NPI record,
When the response enumeration_type matches 'NPI-1' (Individual) or 'NPI-2' (Organizational),
Then Credentialing_Case__c.NPI_Validation_Status__c is set to 'Validated', Credentialing_Case__c.NPI_Validated_Date__c is stamped with today's date, and Provider__c.NPI_Status__c is updated to 'Active'.

Given the NPPES API returns no results or a deactivated record,
When the response result_count equals 0 or deactivation_date is populated,
Then Credentialing_Case__c.NPI_Validation_Status__c is set to 'Failed', a Task is auto-created assigned to the Credentialing Specialist with Subject 'NPI Validation Failed — Manual Review Required', and the Credentialing_Case__c.Status__c is set to 'On Hold'.

Given the NPPES API is unreachable or returns an HTTP error (4xx/5xx),
When the callout times out or errors,
Then Credentialing_Case__c.NPI_Validation_Status__c is set to 'Pending — API Unavailable', a Platform Event (NPI_Validation_Event__e) is published for retry processing, and no Status__c change occurs.

Given a Credentialing_Case__c already has NPI_Validation_Status__c = 'Validated',
When the record is edited and saved without changing Provider__c.NPI__c,
Then no repeat NPPES callout is triggered.

-Given the provider is a Facility__c (Business Account), NPI-2 validation shall follow a separate flow path- (superseded: unified flow handles both NPI-1 and NPI-2 per INS-0147).

## Solution

Implement a Record-Triggered Flow on Credentialing_Case__c (After Save, onCreate) that invokes an Apex @InvocableMethod 'NPPESValidationService.validateNPI()'. The Apex class performs an HttpCallout to the NPPES REST API using a Named Credential ('NPPES_Registry'). Response parsing handled via a JSON deserialize into a custom Apex DTO. On success/failure, a second flow element updates Credentialing_Case__c fields and Provider__c via a cross-object field update. Retry logic uses a Platform Event (NPI_Validation_Event__e) consumed by a separate Trigger-based handler with exponential backoff (max 3 retries). Named Credential and Remote Site Setting configured for sandbox and production endpoints. Custom Label used for API endpoint URL to support environment switching.

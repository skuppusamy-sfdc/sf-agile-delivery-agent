## Description

As a Credentialing Specialist, I need Salesforce to automatically retrieve and populate provider credentialing data from CAQH ProView when a Credentialing_Case__c is initiated, so that manual data entry is eliminated and credentialing turnaround time is reduced from an average of 30 days to under 15 days.

## Acceptance Criteria

Given a Credentialing_Case__c record is created with Status__c = 'Initiated' and the linked Provider__c.NPI__c is populated
When the CAQH sync is triggered (automatically within 5 minutes via Platform Event)
Then the following fields on Provider__c are populated from the CAQH ProView response:
  - Provider__c.CAQH_Provider_ID__c
  - Provider__c.Board_Certification__c
  - Provider__c.Medical_School__c
  - Provider__c.Graduation_Year__c
  - Provider__c.Malpractice_Expiry_Date__c
  - Provider__c.DEA_Number__c
  - Provider__c.State_License_Number__c
  - Provider__c.State_License_Expiry__c

Given the CAQH API returns a successful 200 response
When data is written to Provider__c
Then Credentialing_Case__c.CAQH_Last_Sync_DateTime__c is stamped with the current timestamp and Credentialing_Case__c.Data_Source__c = 'CAQH ProView'

Given the CAQH API returns a 404 (provider not found in CAQH)
When the sync attempt is logged
Then Credentialing_Case__c.CAQH_Sync_Status__c = 'Provider Not Found' and a Task is created for the assigned specialist to manually collect credentials

Given the CAQH API returns a 401 or 503
When the error is caught
Then the Platform Event retry mechanism attempts re-sync up to 3 times at 10-minute intervals before setting Credentialing_Case__c.CAQH_Sync_Status__c = 'Sync Failed - Manual Review'

Given CAQH data is successfully retrieved
When Provider__c.Malpractice_Expiry_Date__c is within 60 days of TODAY()
Then a high-priority Task is auto-created on the Credentialing_Case__c with Subject = 'Malpractice Insurance Expiring Soon - Action Required'

-Given re-credentialing cases, perform a delta sync only- (deferred to Phase 2 optimization sprint)

## Solution

Create a Platform Event (CAQH_Sync_Request__e) published by the Credentialing_Case__c Record-Triggered Flow on creation. An Apex trigger (CAQHSyncEventTrigger.cls) subscribes to the Platform Event and calls CAQHIntegrationService.cls, which performs a callout to the CAQH ProView REST API (v3) using Named Credentials ('CAQH_ProView_NC'). The response JSON is deserialized using a CAQHResponseWrapper.cls inner class. Field mapping is handled by a Custom Metadata Type (CAQH_Field_Mapping__mdt) to allow admin-configurable mappings without code changes. Retry logic uses a Queueable Apex chain (CAQHRetryQueueable.cls) with a retry counter stored on Credentialing_Case__c.CAQH_Retry_Count__c. All API responses are logged to an Integration_Log__c object for audit purposes.

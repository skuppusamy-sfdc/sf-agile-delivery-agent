## Description

As a Credentialing Specialist, I need a guided OmniScript that allows me to initiate a re-credentialing workflow for a provider whose Provider_Network_Agreement__c is approaching expiration so that re-credentialing is completed before the agreement lapses and the provider is not inadvertently terminated from the network.

## Acceptance Criteria

Given a Credentialing Specialist launches the 'Initiate Re-Credentialing' OmniScript from a Provider__c record,
When the OmniScript loads (Step 1),
Then it displays a summary screen showing:
  - Provider__c.Name, Provider__c.NPI__c, Provider__c.NPI_Status__c
  - All active Provider_Network_Agreement__c records where PNA__c.Expiration_Date__c <= TODAY() + 90 days, sorted by soonest expiration

Given the Credentialing Specialist selects one or more PNA records to re-credential,
When they advance to Step 2,
Then the OmniScript prompts for:
  - Re_Credential_Reason__c (picklist: 'Scheduled Renewal', 'Triggered by Complaint', 'LOB Expansion', 'Regulatory Requirement')
  - Target_Completion_Date__c (date field, must be before PNA__c.Expiration_Date__c)
  - Notes__c (text area, optional)

Given the specialist completes all required fields and submits,
When the OmniScript final step fires,
Then:
  - A new Credentialing_Case__c is created with Credentialing_Case__c.Type__c = 'Re-Credentialing'
  - Credentialing_Case__c.Related_PNA__c is linked to the selected PNA record
  - Credentialing_Case__c.Target_Completion_Date__c is set from OmniScript input
  - Provider__c.Network_Participation_Status__c is updated to 'Re-Credentialing In Progress'
  - A CAQH sync is triggered per INS-0162 logic

Given the Provider__c.NPI_Status__c = 'Invalid' at time of OmniScript launch,
When Step 1 loads,
Then an error block is displayed: 'Re-credentialing cannot be initiated — NPI is invalid. Resolve NPI status before proceeding.' and the specialist cannot advance past Step 1.

-Given re-credentialing should also trigger a DocuSign envelope for provider attestation- (deprecated: DocuSign integration for re-credentialing deferred to Phase 2)

## Solution

Build OmniScript (Re_Credentialing_Initiation_OS) with 3 steps: (1) DataRaptor Extract to load Provider__c and filtered PNA records; (2) input form step with conditional validation; (3) DataRaptor Transform + Integration Procedure to create Credentialing_Case__c, update Provider__c status, and invoke the CAQH sync Apex Action from INS-0162. NPI status guard implemented as a Set Values element at Step 1 with a conditional block. OmniScript embedded on Provider__c Lightning Record Page as a Quick Action. Integration Procedure (IP_ReCredentialing_Submit) handles all DML to keep OmniScript stateless. Custom Labels used for all user-facing error messages to support future localization.

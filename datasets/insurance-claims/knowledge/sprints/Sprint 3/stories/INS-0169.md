## Description

As a Credentialing Specialist, I need the system to connect to CAQH ProView using the provider's CAQH ID and retrieve credentialing attestation data and documents so that I do not need to manually request and upload supporting documentation for each credentialing case, reducing processing time from an average of 5 days to under 24 hours.

## Acceptance Criteria

Given a Credentialing_Case__c record has Status__c = 'Initiated' and the related Provider__c.CAQH_ID__c is populated
When a Credentialing Specialist clicks the 'Fetch CAQH Data' button on the Credentialing_Case__c record page
Then the system calls the CAQH ProView API and returns the provider's attestation status, last attestation date, and document list

Given the CAQH API returns documents (e.g., DEA certificate, board certification, malpractice insurance)
When the sync completes
Then each document is stored as a Salesforce File (ContentDocument) linked to the Credentialing_Case__c record, with ContentDocument.Title formatted as '[DocumentType] — [ProviderLastName] — [AttestationDate]'

Given CAQH attestation data is retrieved
When the sync completes
Then the following fields are updated on Credentialing_Case__c: CAQH_Attestation_Status__c, CAQH_Last_Attestation_Date__c, CAQH_Sync_Timestamp__c = NOW()

Given the provider's CAQH attestation is expired (last attestation > 120 days ago)
When data is fetched
Then Credentialing_Case__c.Status__c is set to 'Pending Provider Action', a task is created assigned to the Provider Relations Specialist with Subject = 'CAQH Re-attestation Required', and Due_Date__c = TODAY() + 14

Given a Provider__c record does not have a CAQH_ID__c value
When the 'Fetch CAQH Data' button is clicked
Then an error modal displays: 'CAQH ID is required to retrieve credentialing data. Please update the provider record before proceeding.'

Given the CAQH API returns a 401 Unauthorized response
When the callout fails
Then the error is logged to Integration_Error_Log__c and the Credentialing Operations Manager is notified via a Salesforce Notification with message: 'CAQH authentication failure — credential rotation may be required.'

## Solution

LWC button component ('Fetch CAQH Data') placed on Credentialing_Case__c record page via App Builder. Button invokes an Apex controller method (CAQHIntegrationController.fetchProviderData) which performs a synchronous callout to CAQH ProView REST API. CAQH credentials stored in Named Credential (CAQH_ProView_NC). Document binary responses handled via ContentVersion insertion with NetworkId scoped to internal community. Field updates performed in the same transaction. Attestation expiry logic handled in Apex with Task creation via SObject insert. Error logging utility class shared with INS-0168 integration error pattern. Custom Permission 'CAQH_Fetch_Access' controls button visibility via LWC conditional rendering.

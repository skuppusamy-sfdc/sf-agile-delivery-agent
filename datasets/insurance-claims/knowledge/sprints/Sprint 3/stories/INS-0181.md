## Description

As a Compliance Officer, I need the system to automatically monitor credentialing expiration dates on Credentialing_Case__c records and enforce suspension of Provider__c network participation when credentials lapse so that Acme Insurance does not inadvertently process claims for providers whose credentials have expired, thereby reducing regulatory and financial risk.

This story covers the scheduled monitoring and auto-suspension logic. PCN (Provider Change Notification) generation for downstream notification is out of scope for this sprint.

## Acceptance Criteria

Given a scheduled Apex job runs nightly at 2:00 AM,
When any of the following expiration fields on Credentialing_Case__c are within 30 days of today or already past:
  - Medical_License_Expiry__c
  - Malpractice_Insurance_Expiry__c
  - DEA_Expiry__c
Then a warning Task is created on the related Provider__c assigned to the Credentialing Specialist with Subject = '[Credential Type] Expiring in [X] Days — Renewal Required'.

Given any credentialing expiration date on Credentialing_Case__c has passed (< TODAY()),
When the nightly job runs,
Then:
  - Provider__c.Provider_Status__c is set to 'Suspended — Credentials Lapsed'
  - All active Provider_Network_Agreement__c records for the provider have PNA.Status__c set to 'Suspended'
  - Credentialing_Case__c.Status__c = 'Lapsed'
  - A Chatter post is made on Provider__c tagging the Provider Relations Specialist and Compliance Officer

Given Provider__c.Provider_Status__c = 'Suspended — Credentials Lapsed',
When a new Claim__c is submitted with Billing_Provider_NPI__c matching the suspended provider,
Then Claim__c.Status__c is set to 'Pended — Provider Suspended' and Claim__c.Pend_Reason__c = 'Provider credentials lapsed. Manual review required.'

Given a provider's credential is renewed and Credentialing_Case__c.Status__c is updated to 'Re-Credentialed',
When the record is saved,
Then Provider__c.Provider_Status__c is automatically restored to 'Active' and suspended PNA records are reactivated if PNA.Effective_End_Date__c >= TODAY().

Given the nightly job completes,
When processing is finished,
Then a summary report record is written to 'Credentialing_Audit_Log__c' capturing: job run datetime, count of providers warned, count of providers suspended, count of PNAs suspended.

## Solution

Implement a Schedulable Apex class 'CredentialingExpirationMonitorBatch' extending Database.Batchable and Schedulable interfaces, scheduled via Setup > Scheduled Jobs at 02:00 AM daily. Batch query scopes Credentialing_Case__c records where Status__c NOT IN ('Lapsed','Closed') and any expiry field <= TODAY() + 30. Use separate batch scope for already-lapsed records. Suspension logic updates Provider__c and related PNA records via DML collections (bulkified). Chatter posts via ConnectApi.ChatterFeeds. Re-credentialing reactivation handled by a Record-Triggered Flow on Credentialing_Case__c firing on Update when Status__c changes to 'Re-Credentialed'. Claim pend logic implemented via a Before-Insert Apex Trigger on Claim__c checking provider status via a SOQL lookup. Audit log written via an After-Batch finish() method. Add Permission Set 'Credentialing_Monitor_PS' for the integration/scheduled user running the batch.

## Description

As a Provider Network Specialist, I need a guided OmniScript to create Provider_Network_Agreement__c (PNA) records that enforces credentialing pre-requisites, links the appropriate Fee_Schedule__c, validates provider TIN against IRS records, and routes the completed agreement to DocuSign for provider signature, so that network contracting is standardized, compliant, and audit-ready from initiation to execution.

## Acceptance Criteria

Given a Provider Network Specialist launches the 'Create Provider Network Agreement' OmniScript from a Provider__c record
When Step 1 (Eligibility Check) loads
Then the OmniScript queries Provider__c.Provider_Status__c and Credentialing_Case__c.Status__c
And if Provider_Status__c != 'Credentialed', the OmniScript displays a blocking error: 'Provider must be fully credentialed before a PNA can be created' and prevents progression to Step 2

Given the provider passes the eligibility check
When Step 2 (Agreement Details) is completed with PNA.LOB__c, PNA.Network_Type__c, and PNA.Effective_Date__c
Then Step 3 presents a filtered lookup of Fee_Schedule__c records where Fee_Schedule__c.LOB__c = PNA.LOB__c AND Fee_Schedule__c.Effective_Date__c <= PNA.Effective_Date__c AND Fee_Schedule__c.Status__c = 'Active'
And the Provider Network Specialist must select exactly one Fee_Schedule__c to proceed

Given Step 3 Fee Schedule selection is complete
When Step 4 (TIN Validation) loads
Then Provider__c.TIN__c is displayed for confirmation
And a DataRaptor callout validates TIN format (9-digit numeric, no dashes) before allowing progression

-Given Step 5 should include a Group_Service_Agreement__c linkage for group providers- (deprecated: GSA linkage moved to separate GSA Amendment OmniScript per INS-0129 scope)

Given all steps are completed
When the Provider Network Specialist clicks 'Submit'
Then a Provider_Network_Agreement__c record is created with Status__c = 'Pending Signature'
And a DocuSign envelope is automatically generated via DocuSign for Salesforce (DSF) using the 'PNA_Standard_Template' DocuSign template
And the envelope is sent to Provider__c.Email for e-signature
And PNA.DocuSign_Envelope_ID__c is populated with the returned envelope ID

Given the DocuSign envelope is completed by the provider
When the DocuSign Connect webhook fires
Then PNA.Status__c is updated to 'Active' and PNA.Executed_Date__c = signature completion date
And Provider__c.Provider_Status__c remains 'Credentialed' (no change)

## Solution

OmniStudio OmniScript (PNA_Creation_OS) with 5 steps. Step 1 uses DataRaptor Extract to query Provider__c and related Credentialing_Case__c. Step 3 fee schedule lookup uses a DataRaptor Extract with dynamic filter parameters passed from Step 2 selections. TIN format validation in Step 4 via OmniScript Formula element with regex. PNA record creation via DataRaptor Post (Integration Procedure: PNA_CreateRecord_IP). DocuSign envelope generation via DocuSign for Salesforce REST API called from Integration Procedure using Named Credential 'DocuSign_DSF'. Webhook handling via a custom Apex REST endpoint (DocuSignWebhookHandler.cls) that processes envelope status callbacks and updates PNA__c via DML. OmniScript embedded as Quick Action on Provider__c record page.

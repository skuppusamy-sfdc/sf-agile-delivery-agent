## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces credentialing status, all active Provider_Network_Agreement__c records, and a timeline of NPI validation events, so that I can assess a provider's network standing and credentialing health without navigating across multiple objects.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the card displays: Provider__c.Credentialing_Status__c, Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Primary_Specialty__c, and Provider__c.NPI_Status__c in a header band.

Given the provider has one or more Provider_Network_Agreement__c records with PNA__c.Status__c = 'Active',
When the FlexCard PNA child block renders,
Then each active PNA is listed with PNA__c.Network_Name__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, and PNA__c.LOB__c visible.

Given the provider has Credentialing_Case__c records,
When the NPI Validation History tab is selected on the FlexCard,
Then all Credentialing_Case__c records are shown sorted by CreatedDate DESC, displaying Credentialing_Case__c.NPI_Validation_Status__c and Credentialing_Case__c.NPI_Validated_Date__c.

Given a provider has no active PNAs,
When the PNA block renders,
Then an inline message 'No Active Network Agreements Found' is displayed with a quick-action button to initiate a new PNA.

-Given the FlexCard must display CAQH profile completion percentage pulled via CAQH API- (deferred to Phase 2 pending CAQH integration design).

## Solution

Build an OmniStudio FlexCard ('Provider360_Card') using a DataRaptor Extract as the primary data source, joining Provider__c, Provider_Network_Agreement__c (child), and Credentialing_Case__c (child) in a single DR. Use FlexCard child cards for PNA list and Credentialing history tabs. Conditional visibility states handle 'No Active PNA' scenario. Quick-action button triggers an OmniScript ('New_PNA_OmniScript') via a FlexCard action. FlexCard deployed to the Provider__c Lightning Record Page via App Builder. Permission Set Group 'Provider_Relations_PS_Group' grants access to the FlexCard and underlying DataRaptors.

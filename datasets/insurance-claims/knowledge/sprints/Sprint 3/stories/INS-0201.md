## Description

As a Credentialing Specialist, I need the Salesforce platform to retrieve a provider's CAQH ProView attestation profile and synchronize relevant credentialing data onto the Credentialing_Case__c and Provider__c records so that manual re-keying of education, training, work history, and malpractice data is eliminated and re-attestation currency is continuously monitored.

## Acceptance Criteria

Given a Provider__c record has a valid Provider__c.CAQH_Provider_ID__c populated
When a Credentialing Specialist clicks 'Sync CAQH Profile' on the Credentialing_Case__c record
Then the system calls the CAQH ProView REST API and updates:
  - Credentialing_Case__c.CAQH_Attestation_Date__c
  - Credentialing_Case__c.CAQH_Profile_Status__c ('Complete', 'Incomplete', 'Expired')
  - Provider__c.Medical_School__c
  - Provider__c.Residency_Program__c
  - Provider__c.Malpractice_Carrier__c
  - Provider__c.Malpractice_Expiry_Date__c

Given the CAQH profile status returns 'Expired' (re-attestation date > 120 days ago)
When the sync completes
Then Credentialing_Case__c.Status__c is set to 'Pending Provider Action'
And a task is created assigned to the Provider Relations Specialist with Subject: 'CAQH Re-attestation Required - [Provider Name]' and Due_Date__c = TODAY() + 14

Given a CAQH sync is triggered
When the sync completes successfully
Then Credentialing_Case__c.Last_CAQH_Sync__c = NOW() is stamped
And an entry is added to the record's Chatter feed: 'CAQH profile synced successfully. Attestation date: [date]. Profile status: [status].'

-Given CAQH_Provider_ID__c is blank, display inline error on the button-
Given CAQH_Provider_ID__c is blank
When sync is attempted
Then a toast error message displays: 'CAQH Provider ID is required to perform this sync. Please update the Provider record.'

## Solution

Implement an Apex class 'CAQHIntegrationService' using Named Credential 'CAQH_ProView_API' for OAuth 2.0 authentication. An @AuraEnabled method is exposed to an LWC 'caqhSyncButton' rendered as a Quick Action on Credentialing_Case__c. The Apex handler deserializes the CAQH JSON response and performs a DML update on both Credentialing_Case__c and the related Provider__c. Task creation on expired profiles is handled via a Record-Triggered Flow on Credentialing_Case__c firing when CAQH_Profile_Status__c changes to 'Expired'. Chatter post uses ConnectAPI.ChatterFeeds.postFeedElement in the Apex handler. Schedule a nightly Schedulable Apex job 'CAQHBatchSync' implementing Database.Batchable to proactively re-sync all providers with Credentialing_Case__c.Status__c = 'Active Credentialing' and Last_CAQH_Sync__c < LAST_N_DAYS:7.

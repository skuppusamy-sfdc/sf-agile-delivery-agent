## Description

As a Provider Relations Specialist, I need a consolidated Credentialing Case 360 FlexCard embedded on the Provider__c (Person Account) record page, so that I can instantly view all active and historical credentialing cases, NPI validation status, CAQH profile link, and outstanding action items without navigating away from the provider record. Extends INS-0183.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c Person Account record,
When the Credentialing 360 FlexCard loads,
Then it displays a list of all associated Credentialing_Case__c records sorted by CreatedDate descending, showing fields: Credentialing_Case__c.Name, Status__c, NPI_Validation_Status__c, Credentialing_Case__c.LOB__c, and Assigned_Credentialing_Specialist__c.

Given the Provider__c record has a populated CAQH_Provider_ID__c field,
When the FlexCard renders,
Then a 'View CAQH Profile' action button appears that opens the CAQH ProView URL in a new browser tab using the stored CAQH_Provider_ID__c value.

Given one or more Credentialing_Case__c records have Status__c = 'Pending – NPI Review' or 'Pending – Document Collection',
When the FlexCard loads,
Then an alert banner is displayed at the top of the card in amber color with the count of pending action items and a quick-action link to each case.

Given no Credentialing_Case__c records exist for the Provider__c,
When the FlexCard renders,
Then an empty state message reads 'No credentialing cases on file. Click New to initiate credentialing.' with a 'New Credentialing Case' quick action button.

-Then the FlexCard also displays Fee_Schedule__c assignments inline- (descoped to INS-0191 in Sprint 4)

## Solution

Build an OmniStudio FlexCard (CredentialingCase360FC) using a DataRaptor Extract source that queries Credentialing_Case__c filtered by Provider__c lookup. Conditional rendering states handle the empty-state and alert-banner scenarios using FlexCard conditional visibility expressions. The CAQH deep-link action uses a URL Action flyout. The FlexCard is surfaced via a Lightning App Builder component on the Provider__c (Person Account) page layout, restricted to the 'Provider Relations' Permission Set Group. A child card displays open Tasks linked to the Credentialing_Case__c for action-item visibility.

## Description

As a Credentialing Specialist, I need Salesforce to automatically retrieve provider attestation documents and credentialing data from CAQH ProView using the provider's CAQH ID, so that manual document collection is eliminated and Credentialing_Case__c records are populated with verified source-of-truth data directly from CAQH, reducing credentialing cycle time.

## Acceptance Criteria

Given a Credentialing_Case__c record has Credentialing_Case__c.CAQH_ID__c populated and Status__c = 'Initiated'
When a Credentialing Specialist clicks the 'Retrieve CAQH Data' action button
Then a callout is made to the CAQH ProView Universal Provider Datasource API
And the following fields are populated on Credentialing_Case__c:
  - Credentialing_Case__c.Board_Certification__c
  - Credentialing_Case__c.License_Number__c
  - Credentialing_Case__c.License_Expiry_Date__c
  - Credentialing_Case__c.Malpractice_Coverage_Expiry__c
  - Credentialing_Case__c.CAQH_Attestation_Date__c
  - Credentialing_Case__c.CAQH_Data_Retrieved_Date__c = NOW()

Given CAQH data is successfully retrieved
When Credentialing_Case__c.License_Expiry_Date__c is within 90 days of TODAY()
Then Credentialing_Case__c.Flags__c is updated to include 'License Expiring Soon' and the assigned Credentialing Specialist receives a Salesforce Task with Subject = 'License Expiry Follow-Up Required'

Given CAQH data is retrieved and Credentialing_Case__c.CAQH_Attestation_Date__c is older than 120 days
When the record is evaluated
Then Credentialing_Case__c.Status__c is set to 'Pending Re-Attestation' and Provider__c.Provider_Status__c is updated to 'Credentialing On Hold'

Given the CAQH API returns a 'Provider Not Found' error
When the error is received
Then Credentialing_Case__c.CAQH_Retrieval_Status__c = 'Provider Not Found' and a Chatter post is made on the Credentialing_Case__c record tagging the assigned specialist

Given retrieved documents include PDF attachments (malpractice certificate, DEA certificate)
When the API response includes base64-encoded file content
Then files are stored as Salesforce Files (ContentDocument) linked to both Credentialing_Case__c and Provider__c via ContentDocumentLink

## Solution

Apex class (CAQHIntegrationService.cls) handles OAuth 2.0 authentication and REST callout to CAQH ProView API using Named Credential 'CAQH_ProView'. Response deserialized via CAQHResponseWrapper.cls. 'Retrieve CAQH Data' button on Credentialing_Case__c record page invokes an LWC (caqhDataRetrieval) which calls an Apex @AuraEnabled method. Field population handled in Apex service layer. License expiry evaluation and Task creation handled by a separate Record-Triggered Flow (After Update) on Credentialing_Case__c. Base64 PDF decoding and ContentDocument insertion handled in Apex. All API credentials and endpoint URLs stored in Custom Metadata Type (CAQH_Integration_Config__mdt). Integration errors published as Platform Events (Integration_Error__e) consumed by ComplianceReporting component.

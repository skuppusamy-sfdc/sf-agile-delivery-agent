# INS-0175: Build Credentialing Case auto-creation flow triggered on new Provider record submission

## Description

As a Credentialing Specialist, I need a Credentialing_Case__c record to be automatically created whenever a new Provider__c (Person Account) is submitted for network participation so that the credentialing workflow begins immediately without manual intervention and no provider falls through the cracks during onboarding.

## Acceptance Criteria

Given a new Provider__c record is created with Provider_Status__c = 'Pending Credentialing' and a valid NPI field (NPI__c) is populated,
When the record is saved,
Then a Credentialing_Case__c record is auto-created with:
  - Credentialing_Case__c.Status__c = 'Initiated'
  - Credentialing_Case__c.Provider__c = [linked Provider__c record Id]
  - Credentialing_Case__c.LOB__c inherited from Provider__c.Line_of_Business__c
  - Credentialing_Case__c.Assigned_Specialist__c = queue 'Credentialing Queue'
  - Credentialing_Case__c.Due_Date__c = TODAY() + 30 days

Given a Provider__c record is created WITHOUT NPI__c populated,
When the record is saved,
Then NO Credentialing_Case__c is created and a validation error is displayed: 'NPI is required to initiate credentialing.'

Given a Credentialing_Case__c is successfully created,
When the case is initiated,
Then an email notification is sent to the assigned Credentialing Specialist using the 'Credentialing Initiated' email template and a Task is created on the Provider__c record with Subject = 'Complete CAQH Profile Verification'.

-Given Provider_Type__c = 'Facility', route to Facility Credentialing Queue instead- (superseded by INS-0178 facility routing story)

Given a duplicate Provider__c is detected (matching NPI__c on an existing active record),
When save is attempted,
Then the flow halts and displays: 'An active provider with this NPI already exists. Please review record [Provider Name].'

## Solution

Implement a Record-Triggered Flow on Provider__c object firing on 'Create' with entry condition Provider_Status__c = 'Pending Credentialing'. Use a Decision element to check NPI__c is not null and run a Get Records element to detect NPI duplicates against existing Provider__c records. On success path, use Create Records element to instantiate Credentialing_Case__c with field mappings as specified. Fire a Platform Event 'Credentialing_Initiated__e' for downstream notification handling. Add a separate Apex-invocable method for duplicate NPI detection to handle bulk insert scenarios safely. Deploy associated email alert via Flow Send Email action using org-wide address. Include a Custom Permission 'Bypass_Credentialing_Flow' for admin override scenarios.

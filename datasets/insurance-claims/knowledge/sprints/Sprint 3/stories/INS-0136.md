## Description

As a Credentialing Specialist, I need a Credentialing_Case__c record to be automatically created whenever a new Provider__c (Person Account) is submitted for network participation, so that the credentialing workflow is initiated without manual intervention and no provider falls through the cracks during onboarding.

## Acceptance Criteria

Given a Provider__c record is created or updated with Provider_Status__c = 'Pending Credentialing'
When the Record-Triggered Flow fires on the Provider__c object
Then a new Credentialing_Case__c record is created with:
  - Credentialing_Case__c.Status__c = 'Initiated'
  - Credentialing_Case__c.Provider__c = Provider__c.Id
  - Credentialing_Case__c.NPI__c = Provider__c.NPI__c
  - Credentialing_Case__c.TIN__c = Provider__c.TIN__c
  - Credentialing_Case__c.Assigned_Specialist__c = round-robin assigned Credentialing Specialist queue member
  - Credentialing_Case__c.Due_Date__c = TODAY() + 30 days

Given a Provider__c already has an open Credentialing_Case__c (Status__c NOT IN ('Approved','Denied','Withdrawn'))
When a second submission trigger fires
Then no duplicate Credentialing_Case__c is created and a warning toast is surfaced to the submitting user

Given Credentialing_Case__c is successfully created
When the record is saved
Then an automated email notification is dispatched to the assigned Credentialing Specialist using the 'New_Credentialing_Assignment' email template

-Given Provider__c.LOB__c = 'Dental', a separate Dental_Credentialing_Case__c object should be created instead- (deprecated: unified Credentialing_Case__c object now handles all LOBs via RecordType)

Given the Flow encounters a DML exception
When the error is thrown
Then a Platform Event (Credentialing_Error__e) is published and the error is logged to the Credentialing_Error_Log__c custom object with Provider__c.Id and timestamp

## Solution

Implement a Record-Triggered Flow on Provider__c (After Save, trigger on create and update where Provider_Status__c changes to 'Pending Credentialing'). Use a Get Records element to check for existing open Credentialing_Case__c records. If none found, use a Create Records element to instantiate Credentialing_Case__c. Round-robin assignment handled via a custom Apex invocable action (CredentialingQueueAssigner.cls) called from the Flow. Email notification via Flow Send Email action referencing org-wide email address. Platform Event published via Apex fault path. Deploy associated Permission Set Group 'Credentialing_Specialist_PSG' to ensure field-level security on Credentialing_Case__c.

## Description

As a Provider Network Specialist, I need a draft Provider_Network_Agreement__c (PNA) record to be automatically created when a Credentialing_Case__c is approved so that the contracting workflow is immediately initiated without requiring the Network Contracting team to manually identify newly credentialed providers and create agreements from scratch. This story extends INS-0203 (Credentialing approval workflow).

## Acceptance Criteria

Given a Credentialing_Case__c transitions to Status__c = 'Closed - Approved'
When the approval final action fires
Then a Provider_Network_Agreement__c record is created with:
  - PNA__c.Provider__c = Credentialing_Case__c.Provider__c
  - PNA__c.Status__c = 'Draft'
  - PNA__c.LOB__c = Credentialing_Case__c.LOB__c
  - PNA__c.Effective_Date__c = Credentialing_Case__c.Credentialing_Effective_Date__c
  - PNA__c.Agreement_Type__c = 'Standard Participating'
  - PNA__c.Origination_Source__c = 'Auto-Generated from Credentialing'

Given the draft PNA is created
When the Provider Network Specialist opens the PNA record
Then the Fee_Schedule__c lookup is pre-populated based on a matching rule: Fee_Schedule__c where LOB__c = PNA__c.LOB__c and Effective_Date__c <= TODAY() and Expiration_Date__c >= TODAY() and Is_Default__c = TRUE

Given a matching default Fee_Schedule__c is NOT found
When the PNA is created
Then PNA__c.Fee_Schedule__c remains blank
And a Task is created for the Provider Network Specialist: 'Manual Fee Schedule Assignment Required for [Provider Name] PNA'

-Given multi-LOB credentialing, one PNA per LOB should be created in the same transaction-
Given a Credentialing_Case__c has a single LOB__c value
When the auto-creation logic fires
Then exactly one PNA record is created (multi-LOB support deferred to Phase 2)

Given the PNA is created with Status__c = 'Draft'
When the Provider Network Specialist updates Status__c to 'Pending Signature'
Then a DocuSign envelope is sent to the provider's email (Provider__c.PersonEmail) via DocuSign for Salesforce integration with the standard PNA contract template

## Solution

Extend the Apex-invocable method 'CredentialingApprovalActions' (built in INS-0203) to include PNA creation logic. Use a SOQL query to find the default Fee_Schedule__c matching LOB and date criteria. If found, set the lookup; if not, create a follow-up Task via DML. Bulkify the invocable method to handle list inputs. For the DocuSign trigger, configure a Record-Triggered Flow on Provider_Network_Agreement__c firing After Update when Status__c changes from 'Draft' to 'Pending Signature'. The Flow calls the DocuSign for Salesforce invocable action 'Send Envelope' with the mapped PNA template Id stored in Custom Metadata Type 'DocuSign_Template_Config__mdt' keyed on Agreement_Type__c. Ensure all DML operations in the Apex method are wrapped in a Savepoint / rollback pattern to maintain atomicity between Credentialing_Case__c updates and PNA creation.

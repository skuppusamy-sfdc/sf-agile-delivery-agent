## Description

As a Credentialing Specialist, I need the system to automatically identify providers whose credentials are approaching expiration and trigger re-credentialing workflows with tiered reminder notifications, so that I can proactively manage credentialing renewals, avoid lapses in provider network eligibility, and maintain compliance with NCQA re-credentialing cycle requirements (every 36 months).

## Acceptance Criteria

Given a scheduled job runs nightly at 2:00 AM EST,
When the job evaluates all Provider__c records where License_Expiration_Date__c is within 90, 60, or 30 days from the current date AND NPI_Validation_Status__c = 'Valid',
Then a Provider_Change_Notification__c (PCN) record is created for each qualifying provider with PCN.Notification_Type__c = 'Re-credentialing Reminder' and PCN.Days_Until_Expiration__c populated.

Given a PCN record is created for a 90-day expiration threshold,
When the record is inserted,
Then an email notification is sent to the assigned Provider Relations Specialist using the 'Re-credentialing 90-Day Reminder' email template, and a follow-up Task is created with ActivityDate = TODAY() + 5 days.

Given a PCN record is created for a 30-day expiration threshold,
When the record is inserted,
Then the associated Provider__c.Credentialing_Risk_Flag__c is set to 'High', the provider's active Provider_Network_Agreement__c records are flagged with PNA.At_Risk_of_Termination__c = true, and the Credentialing Specialist's manager (from Salesforce User hierarchy) is CC'd on the email notification.

Given a new Credentialing_Case__c with Credentialing_Type__c = 'Re-credentialing' is opened for a provider,
When the case is created,
Then any existing open PCN records for that provider with Notification_Type__c = 'Re-credentialing Reminder' are automatically closed with PCN.Status__c = 'Resolved – Case Opened'.

Given a provider's License_Expiration_Date__c has already passed (i.e., credential is expired),
When the nightly job evaluates the record,
Then Provider__c.NPI_Validation_Status__c is set to 'Invalid', all active PNA records for the provider are updated to PNA.Status__c = 'Suspended – Credential Expired', and a P1 Task is created for the Credentialing Specialist with Subject = 'URGENT: Provider Credential Expired – Immediate Action Required'.

## Solution

Implement a Scheduled Flow ('Nightly_Credentialing_Expiration_Check') that runs daily at 2:00 AM EST. The flow uses a Get Records element to query Provider__c records with date-based filter criteria (License_Expiration_Date__c thresholds). A Loop element iterates over results, with Decision elements branching on 90/60/30-day and past-expiration scenarios. PCN records are created via Create Records elements. Email notifications use Send Email elements referencing org-wide email templates. A separate Record-Triggered Flow on Credentialing_Case__c (After Save, Credentialing_Type__c = 'Re-credentialing', IsNew = true) handles PCN auto-resolution via Update Records. Custom Metadata Type (Recredentialing_Config__mdt) stores threshold day values for admin configurability. Batch Apex (CredentialingExpirationBatch) serves as a fallback for bulk processing when provider volumes exceed Flow governor limits.

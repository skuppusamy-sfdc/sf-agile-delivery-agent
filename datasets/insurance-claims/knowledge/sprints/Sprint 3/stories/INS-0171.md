## Description

As a Provider Network Specialist, I need a Provider_Network_Agreement__c (PNA) record and a corresponding contract document to be automatically generated via Conga CLM when a Credentialing_Case__c is approved so that newly credentialed providers are onboarded into the network without manual contract drafting delays. The generated PNA should reflect the correct LOB, effective date, and applicable Fee_Schedule__c.

## Acceptance Criteria

Given a Credentialing_Case__c.Status__c transitions to 'Approved'
When the record is saved
Then a Provider_Network_Agreement__c record is created with: PNA.Provider__c = Credentialing_Case__c.Provider__c, PNA.Status__c = 'Draft', PNA.Effective_Date__c = FIRST DAY of next calendar month, PNA.LOB__c = Credentialing_Case__c.LOB__c

Given the PNA record is created
When Conga CLM is invoked
Then a contract document is generated using the 'Standard Provider Network Agreement v3' template and attached to the PNA record as a ContentDocument

Given the PNA contract document is generated
When the document is ready
Then a DocuSign envelope is automatically sent to the provider's email (Provider__c.PersonEmail) for e-signature with a 14-day expiration

Given a Fee_Schedule__c record exists with Fee_Schedule__c.LOB__c matching PNA.LOB__c and Fee_Schedule__c.Effective_Date__c <= PNA.Effective_Date__c
When the PNA is created
Then PNA.Fee_Schedule__c is auto-populated with the most recently effective matching Fee_Schedule__c record

Given no matching Fee_Schedule__c is found
When the PNA is created
Then PNA.Fee_Schedule__c remains null and a Task is created for the Provider Network Specialist: Subject = 'Assign Fee Schedule to PNA', Due_Date__c = TODAY() + 5

-Given PNA is generated, also auto-create a Group_Service_Agreement__c if the provider is affiliated with a group practice- (deprecated — GSA auto-creation moved to separate epic in Phase 2)

Given the DocuSign envelope is signed by the provider
When the signed document event is received via DocuSign webhook
Then PNA.Status__c is updated to 'Active' and Provider__c.Network_Participation_Status__c is updated to 'In Network'

## Solution

Record-Triggered Flow on Credentialing_Case__c (after-save) detects Status__c transition to 'Approved'. Flow creates Provider_Network_Agreement__c via Create Records element. Conga CLM invoked via Apex action (CongaCLMIntegrationService.generatePNAContract) passing PNA Id and template name stored in Custom Metadata. DocuSign envelope creation handled via DocuSign for Salesforce managed package Apex API (DSProAPI.createAndSendEnvelope). Fee Schedule lookup performed in Flow via Get Records with SOQL filter on LOB__c and Effective_Date__c, sorted descending, limit 1. DocuSign webhook processed via a public-facing Salesforce Site Apex REST endpoint that publishes a Platform Event (DocuSign_Event__e) consumed by a trigger handler to update PNA and Provider status fields.

## Description

**Bug Description:**
The Credentialing_Case__c Status__c field can be manually advanced to 'Approved' even when mandatory credentialing documents (DEA Certificate, Malpractice Insurance, Board Certification) are not attached to the record as ContentDocuments. This creates a compliance risk as providers can be approved for network participation without complete documentation.

**Steps to Reproduce:**
1. Navigate to any Credentialing_Case__c record with Status__c = 'Under Review'
2. Ensure zero ContentDocuments are linked to the record
3. Manually edit the Status__c picklist field and select 'Approved'
4. Click Save
5. Observe: Record saves successfully with Status__c = 'Approved' and no validation error is thrown

**Expected Behavior:**
System should block the transition to 'Approved' and display a validation error listing the missing required document types.

**Actual Behavior:**
Record saves to 'Approved' with no warning or error, bypassing document completeness checks.

**Environment:** Full Sandbox (CS47)
**Regression:** Introduced after deployment of INS-0155 (Credentialing Status Path update)

## Acceptance Criteria

as per Description

## Solution

Add a before-save Record-Triggered Flow or Apex Trigger (before update) on Credentialing_Case__c that queries ContentDocumentLink for related files when Status__c is transitioning to 'Approved'. Cross-reference retrieved ContentDocument.Title values against a Custom Metadata Type (Required_Credentialing_Docs__mdt) storing mandatory document type names per LOB. If any required document type is absent, add a field-level error on Status__c with a message enumerating missing documents. Validate fix does not break existing Approved records (query-only, not retroactive). Deploy validation to Full Sandbox for UAT sign-off by Compliance Officer before production push.

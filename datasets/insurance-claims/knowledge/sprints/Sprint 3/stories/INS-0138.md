## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces credentialing status, active Provider_Network_Agreement__c records, NPI validation badge, and recent Claim__c activity, so that I can assess a provider's full network standing without navigating across multiple objects and related lists.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page
When the Provider 360 FlexCard loads
Then the following data blocks are rendered:
  - NPI Validation Badge: color-coded pill sourced from Provider__c.NPI_Validation_Status__c (Green='Valid', Yellow='Validation Pending', Red='Invalid'/'Deactivated')
  - Credentialing Status block: Credentialing_Case__c.Status__c for the most recent open case
  - Active PNA count: count of Provider_Network_Agreement__c records where PNA.Status__c = 'Active' and PNA.Provider__c = current Provider__c.Id
  - Last 5 Claim__c records linked to Provider__c, showing Claim__c.Claim_Number__c, Claim__c.Status__c, and Claim__c.Total_Billed_Amount__c

Given the FlexCard loads on a Provider__c with no active Credentialing_Case__c
When the credentialing block renders
Then a 'Not Credentialed' state message is displayed with a 'Initiate Credentialing' action button that launches the Credentialing OmniScript

Given the Provider__c.NPI_Validation_Status__c = 'Deactivated'
When the FlexCard renders
Then a high-visibility red alert banner is displayed reading 'NPI Deactivated — Provider may not be eligible for reimbursement'

-Given the FlexCard should also display Fee_Schedule__c line items directly on the card- (deprecated: Fee Schedule detail moved to separate Fee Schedule FlexCard per UX review)

Given the FlexCard is viewed by a user without the 'Provider_Network_Specialist_PSG' Permission Set Group
When the card attempts to load PNA financial terms
Then PNA.Contracted_Rate__c and PNA.Fee_Schedule__c fields are masked and display '*** Restricted ***'

## Solution

Build OmniStudio FlexCard (ProviderNetwork_360_FC) with multiple DataRaptor Extract datasources: one for Provider__c base fields, one for related Credentialing_Case__c (ordered by CreatedDate DESC, limit 1), one for PNA aggregate count, and one for recent Claim__c records. NPI badge implemented as a conditional State on the FlexCard using NPI_Validation_Status__c picklist value. 'Initiate Credentialing' action button launches OmniScript (CredentialingInitiation_OS). Field masking for restricted financial fields handled via a DataRaptor Transform that checks the running user's Permission Set Group membership via a custom Apex REST resource. FlexCard embedded on Provider__c Lightning Record Page via App Builder.

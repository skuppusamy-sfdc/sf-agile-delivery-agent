## Description

As a Provider Network Specialist, I need a guided OmniScript to generate a Provider_Network_Agreement__c record and route the associated contract for e-signature via DocuSign when a provider's Credentialing_Case__c reaches 'Approved' status so that network contracting is initiated automatically and contract turnaround time is reduced from the current 15-day manual average.

## Acceptance Criteria

Given a Credentialing_Case__c.Status__c transitions to 'Approved',
When the Record-Triggered Flow fires,
Then the Provider Network Specialist is notified via a custom notification (CustomNotificationType: 'Credentialing_Approved_CNT') with a deep link to launch the PNA OmniScript (PNA_Generation_OS) from the Provider__c record.

Given the Provider Network Specialist launches PNA_Generation_OS,
When Step 1 (Provider Details Review) loads,
Then the OmniScript pre-populates from the linked Provider__c:
  - Provider Name, NPI__c, Primary_Taxonomy_Code__c, TIN__c, Primary_Practice_Address__c
  - Read-only display of Credentialing_Case__c.Approval_Date__c

Given Step 2 (Agreement Terms) in the OmniScript,
When the Specialist selects LOB__c (e.g., 'Commercial', 'Medicare Advantage', 'Medicaid'),
Then available Fee_Schedule__c records filtered by LOB__c and Effective_Date__c <= TODAY() are displayed in a picklist for selection and linked to the PNA via Fee_Schedule__c lookup.

Given the Specialist completes all required steps and clicks 'Generate Agreement',
When the OmniScript submits,
Then:
  - Provider_Network_Agreement__c is created with Status__c = 'Pending Signature', Effective_Date__c = first day of next month, and linked Provider__c and Fee_Schedule__c
  - A Conga CLM template merge is triggered to generate the PNA PDF
  - DocuSign envelope is sent to Provider__c.Email for e-signature
  - PNA.DocuSign_Envelope_ID__c is populated with the returned envelope ID

Given the DocuSign envelope is signed by the provider,
When the DocuSign status webhook fires,
Then Provider_Network_Agreement__c.Status__c updates to 'Active' and PNA.Executed_Date__c = signature date.

-Given PNA contracts should also require wet signature for Medicare Advantage LOB- (deprecated — all LOBs standardized to DocuSign e-signature per Compliance Officer approval, Sprint 2)

Given the DocuSign envelope is declined or voided,
When the webhook fires with status 'Declined' or 'Voided',
Then PNA.Status__c = 'Signature Declined' and a Task is created for the Provider Network Specialist with Subject = 'PNA Signature Declined — Follow Up Required'.

## Solution

Build OmniScript (PNA_Generation_OS) in OmniStudio with 4 steps: Provider Review, Agreement Terms, Fee Schedule Selection, Review & Submit. DataRaptors (DR_GetProviderForPNA, DR_GetFeeSchedules, DR_CreatePNA) handle data retrieval and DML. Conga CLM integration via REST callout in an Integration Procedure (IP_GeneratePNADocument). DocuSign integration via DocuSign for Salesforce managed package — envelope created programmatically using DocuSign API action in Integration Procedure (IP_SendPNADocuSign). DocuSign webhook handled by a custom REST endpoint (DocuSignWebhookController.cls) updating PNA status. Record-Triggered Flow on Credentialing_Case__c fires on Status__c = 'Approved' to send custom notification. Permission Set 'Provider_Network_Specialist_PS' grants access to OmniScript and PNA object CRUD.

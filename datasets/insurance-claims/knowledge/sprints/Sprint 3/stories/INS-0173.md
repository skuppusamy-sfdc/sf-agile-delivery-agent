## Description

**Bug Description:**
The NPI format validation (10 numeric digits, Luhn algorithm check) implemented as part of INS-0168 is being bypassed when Provider__c records are bulk-inserted using Salesforce's native Data Import Wizard or the Data Loader tool. Records with invalid NPI values (e.g., 9-digit NPIs, NPIs containing alpha characters) are being successfully created without triggering the NPPES validation flow or the NPI format Validation Rule.

**Steps to Reproduce:**
1. Prepare a CSV file with a Provider__c record where NPI__c = '123456789A' (invalid — contains alpha character)
2. Use Salesforce Data Import Wizard to import the record against the Person Account object
3. Map the NPI field to Provider__c.NPI__c
4. Complete the import
5. Observe: Record is created successfully with NPI__c = '123456789A', NPPES_Verified__c = null, no validation error logged

**Expected Behavior:**
Validation Rule should fire regardless of insert method. NPPES async validation should be enqueued post-insert.

**Actual Behavior:**
Validation Rule does not fire during bulk import. NPPES Queueable is not enqueued. Invalid NPI values persist in the org.

**Root Cause Hypothesis:**
Validation Rule formula may reference a field that evaluates differently in bulk context, or the Record-Triggered Flow has a filter condition that excludes API-originated inserts. Needs investigation.

**Environment:** Full Sandbox (CS47), Data Loader v58

## Solution

Investigate Validation Rule formula for Provider__c.NPI__c — confirm REGEX validation does not have an AND condition checking ISCHANGED() or $User.UITheme that would exclude API context. Refactor Validation Rule to use REGEX(NPI__c, '[0-9]{10}') without UI-context dependencies. Review Record-Triggered Flow entry criteria for any 'Only when record is changed by UI' filter. Add an Apex Trigger (before insert, before update) as a fallback validation layer for NPI format check, ensuring coverage across all DML entry points including API and Data Loader. Enqueue NPPESValidationQueueable from Apex Trigger after-insert context for all new Provider__c records where NPI__c is populated. Write unit tests with @isTest using Test.setMock to cover bulk insert scenario (200+ records) and assert Validation Rule exception is thrown for invalid NPI format.

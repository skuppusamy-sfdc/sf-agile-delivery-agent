## Description

**Bug Report**

**Summary:** A Credentialing Specialist can manually set Credentialing_Case__c.Status__c to 'Approved' via the record page picklist even when Provider__c.NPI_Validation_Status__c is 'Pending', 'Invalid', or 'Not Found'. This bypasses a critical compliance gate and could result in a provider being approved for network participation without a valid NPI.

**Steps to Reproduce:**
1. Create a new Provider__c record with NPI__c = '1234567890' and NPI_Validation_Status__c = 'Pending'.
2. Create a linked Credentialing_Case__c record with Status__c = 'Under Review'.
3. On the Credentialing_Case__c record page, manually edit the Status__c picklist field.
4. Select 'Approved' from the picklist and save.
5. Observe: Record saves successfully with Status__c = 'Approved' despite NPI_Validation_Status__c = 'Pending'.

**Expected Behavior:** The system should prevent Status__c from being set to 'Approved' if the linked Provider__c.NPI_Validation_Status__c is not 'Valid'. A validation error message should be displayed: 'Cannot approve credentialing case: Provider NPI validation must be completed and valid before approval.'

**Actual Behavior:** Status__c is saved as 'Approved' with no error or warning. No blocking validation exists.

**Environment:** Acme Insurance – Full Sandbox (Sprint 3)
**Severity:** High – Compliance risk
**Reported By:** Rachel Torres (Credentialing Specialist UAT tester)

as per Description

## Solution

Implement a Record-Triggered Flow on Credentialing_Case__c (Before Save, entry criteria: Status__c changes to 'Approved') that performs a Get Records action to retrieve the related Provider__c.NPI_Validation_Status__c. A Decision element checks if NPI_Validation_Status__c != 'Valid'; if true, a Custom Error element is invoked to surface the validation message inline on the record page. Additionally, add an Apex Trigger (CredentialingCaseTrigger, before update) as a secondary enforcement layer in case the Flow is bypassed via API or Data Loader. The Apex handler class (CredentialingCaseValidationHandler) checks the same condition and throws a System.AuraHandledException with a user-friendly message. Unit tests cover positive (NPI Valid) and negative (NPI Pending, Invalid, Not Found) scenarios with a minimum 90% code coverage target.

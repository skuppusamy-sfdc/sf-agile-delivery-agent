## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that consolidates credentialing status, NPI validation outcome, active Provider_Network_Agreement__c records, and Fee_Schedule__c assignments so that I have a single-pane view to service provider inquiries without navigating multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the following data blocks are visible:
  - NPI badge showing Provider__c.NPI__c and Provider__c.NPI_Status__c with color coding (green=Valid, yellow=Validation Pending, red=Invalid)
  - Active Credentialing_Case__c count and most recent Credentialing_Case__c.Status__c
  - Count of active Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active'
  - Linked Fee_Schedule__c.Name for the provider's primary LOB

Given the provider has no active Provider_Network_Agreement__c,
When the FlexCard loads,
Then the PNA section displays 'No Active Network Agreements' with a quick-action button to initiate contracting.

Given the Provider__c.NPI_Status__c = 'Invalid',
When the FlexCard loads,
Then a prominent warning banner is shown: 'NPI validation failed — credentialing cannot proceed.'

Given the FlexCard is accessed by a user without the 'Provider_Network_Specialist' Permission Set Group,
When the page loads,
Then sensitive fields (Provider__c.TIN__c, Fee_Schedule__c.Rate_Override__c) are masked.

## Solution

Build OmniStudio FlexCard (Provider_360_Card) with a primary DataRaptor Extract sourcing Provider__c fields, a child DataRaptor for related Credentialing_Case__c (filtered by Status__c NOT IN ('Closed','Withdrawn')), and a second child DataRaptor for Provider_Network_Agreement__c (Status__c = 'Active'). Conditional rendering states handle zero-PNA and invalid-NPI scenarios. TIN masking implemented via a custom Lightning Web Component (lwc:TINMaskDisplay) embedded as a FlexCard action. Permission checking uses a DataRaptor Transform that calls a custom Apex method to evaluate the current user's Permission Set Group membership. FlexCard deployed to Provider__c Lightning Record Page via App Builder.

## Description

As a Credentialing Specialist, I need an automated monitoring system that identifies providers whose credentialing approvals, licenses, or malpractice coverages are approaching expiry, and proactively initiates re-credentialing workflows, so that Acme Insurance remains compliant with NCQA and state regulatory requirements and avoids paying claims to providers with lapsed credentials.

## Acceptance Criteria

Given a scheduled Apex job runs nightly at 2:00 AM EST
When it evaluates all Provider__c records where Provider_Status__c = 'Credentialed'
Then it identifies records where any of the following conditions are met:
  - Credentialing_Case__c.License_Expiry_Date__c <= TODAY() + 90 days
  - Credentialing_Case__c.Malpractice_Coverage_Expiry__c <= TODAY() + 60 days
  - Credentialing_Case__c.Approval_Date__c <= TODAY() - 730 days (2-year re-credentialing cycle)
And for each identified Provider__c, a new Credentialing_Case__c is created with RecordType = 'Re-Credentialing' and Status__c = 'Initiated' (if no open re-credentialing case already exists)

Given a re-credentialing Credentialing_Case__c is created
When Status__c = 'Initiated'
Then the assigned Credentialing Specialist receives a Task with Subject = 'Re-Credentialing Required' and ActivityDate = Credentialing_Case__c.Due_Date__c
And Provider__c.Re_Credentialing_Due_Date__c is stamped

Given a provider's license has already expired (License_Expiry_Date__c < TODAY())
When the nightly job evaluates the record
Then Provider__c.Provider_Status__c is updated to 'Credentialing Suspended'
And all active Provider_Network_Agreement__c records for that provider are flagged: PNA.Status__c = 'Suspended - Credential Lapse'
And the Provider Relations Specialist assigned to the PNA is notified via Salesforce In-App Notification

Given a Credentialing Specialist accesses the 'Credentialing Expiry Monitor' Lightning Dashboard
When the dashboard loads
Then it displays the following report components:
  - 'Expiring in 30 Days' count (donut chart)
  - 'Expiring in 31-90 Days' count (donut chart)
  - 'Currently Suspended' provider list with Provider__c.Name, NPI__c, and License_Expiry_Date__c
  - Trend line of re-credentialing cases opened per month (last 12 months)

## Solution

Schedulable Apex class (CredentialingExpiryMonitor_Scheduler.cls) queries Credentialing_Case__c and Provider__c records using SOQL with date formula comparisons. Re-credentialing case creation and PNA suspension logic encapsulated in CredentialingExpiryService.cls service layer. Scheduler registered via Custom Metadata Type (Scheduled_Job_Config__mdt) for admin-configurable run times. Lightning Dashboard 'Credentialing Expiry Monitor' built with four CRM Analytics (formerly Tableau CRM) reports on Credentialing_Case__c and Provider__c. In-App Notifications sent via Apex ConnectApi.ChatterFeeds or Custom Notification Type. Permission Set Group 'Credentialing_Manager_PSG' grants dashboard access.

## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces credentialing status, all active Provider_Network_Agreement__c (PNA) records, NPI validation badge, and TIN on file so that I have a single-pane-of-glass view without navigating to multiple related lists during provider calls or dispute resolution.

## Acceptance Criteria

Given a Provider__c record page is loaded by a user with the 'Provider_Relations' Permission Set Group,
When the Provider 360 FlexCard renders,
Then the following data sections are visible:
  - NPI badge: green if Provider__c.NPI_Validated__c = TRUE, red if FALSE, grey if 'Validation Pending'
  - Credentialing status pill sourced from most recent Credentialing_Case__c.Status__c
  - TIN displayed as masked value (last 4 digits visible): Provider__c.TIN__c
  - Count of active PNA records where Provider_Network_Agreement__c.Status__c = 'Active' AND Effective_End_Date__c >= TODAY()

Given the provider has one or more active PNA records,
When the 'Network Agreements' section is expanded,
Then each PNA card shows: PNA Name, Network__c, Effective_Start_Date__c, Effective_End_Date__c, and Fee_Schedule__c name.

Given the provider has NO active PNA records,
When the FlexCard renders,
Then display the message: 'No active network agreements on file. Initiate contracting via Provider Network Contracting workflow.'

Given a Credentialing_Case__c.Status__c = 'Suspended',
When the FlexCard renders,
Then display a prominent red banner: 'CREDENTIALING SUSPENDED — Claims may be impacted' above all other sections.

-Display CAQH profile last attestation date inline on the main card- (moved to separate CAQH integration story INS-0181)

## Solution

Build OmniStudio FlexCard 'Provider360Card' using DataRaptors to fetch Provider__c base data, a child DataRaptor Extract for related Credentialing_Case__c (ordered by CreatedDate DESC, limit 1), and a second child DataRaptor for Provider_Network_Agreement__c with SOQL filter on Status__c = 'Active'. Use Flyout actions for PNA detail expansion. Implement conditional styling on NPI badge using FlexCard State Actions driven by NPI_Validated__c and NPI_Status__c field values. TIN masking handled via a DataRaptor formula expression. Deploy FlexCard to Provider__c Lightning Record Page via App Builder, visible only to users with 'Provider_Relations_PSG' Permission Set Group. Ensure FlexCard is mobile-aware using responsive layout settings.

## Description

As a Credentialing Specialist, I need a guided multi-step OmniScript intake form for initiating provider credentialing, so that all required data (provider demographics, NPI, TIN, CAQH ID, license information, LOB applicability, and supporting documents) is captured consistently and a Credentialing_Case__c record is created with the correct initial field values upon submission.

## Acceptance Criteria

Given a Credentialing Specialist launches the 'New Provider Credentialing' OmniScript from the Provider__c record or the Credentialing Cases list view,
When Step 1 (Provider Identification) loads,
Then the form pre-populates Provider__c.FirstName, Provider__c.LastName, Provider__c.NPI__c, and Provider__c.TIN__c if the OmniScript was launched from an existing Provider__c record.

Given the user is on Step 2 (License & Specialty),
When they enter a state license number in the License_Number__c field,
Then the License_State__c picklist is required, License_Expiration_Date__c must be a future date, and at least one Provider_Specialty__c value must be selected from the multi-select picklist.

Given the user is on Step 3 (LOB & Network Assignment),
When they select one or more LOB__c values (e.g., 'Commercial', 'Medicare Advantage', 'Medicaid'),
Then the Network__c lookup filters to show only networks associated with the selected LOB(s) via a DataRaptor Transform.

Given the user reaches Step 4 (Document Upload),
When they attempt to proceed without uploading at minimum a W-9 and a copy of the provider's active state license,
Then the OmniScript displays a validation error: 'Required documents missing: W-9, State License' and blocks progression.

Given all steps are completed and the user clicks 'Submit',
When the OmniScript finalizes,
Then a Credentialing_Case__c record is created with Status__c = 'Intake Submitted', Entity_Type__c correctly set to 'Individual' or 'Facility' based on the provider type, and all captured data mapped to the correct fields via a DataRaptor Post.

Given the OmniScript submission is successful,
When the confirmation screen displays,
Then the newly created Credentialing_Case__c.Name (auto-number) is shown, a DocuSign envelope is triggered for the provider attestation form, and the submitting specialist receives an email confirmation.

## Solution

Build a multi-step OmniScript (ProviderCredentialingIntakeOS) in OmniStudio with 5 steps: Provider Identification, License & Specialty, LOB & Network Assignment, Document Upload, and Review & Submit. DataRaptor Extract actions pre-populate provider fields on launch. DataRaptor Transform handles LOB-to-network filtering. Step 4 uses the OmniScript File Upload element with custom validation via a formula function checking uploaded file categories against a required document list stored in a Custom Metadata Type (Required_Credentialing_Documents__mdt). The Submit action uses a DataRaptor Post to create Credentialing_Case__c and related child records. DocuSign envelope trigger is handled via an Integration Procedure calling the DocuSign eSignature API through a Named Credential. The OmniScript is embedded as a quick action on the Provider__c page layout.

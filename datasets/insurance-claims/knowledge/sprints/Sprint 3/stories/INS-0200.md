## Description

As a Credentialing Specialist, I need the system to validate a provider's NPI against the NPPES registry and auto-populate verified demographic and taxonomy data on the Provider__c record so that manual data entry errors are eliminated and credentialing decisions are based on authoritative CMS source data. Extends INS-0169 (Provider__c data model buildout).

## Acceptance Criteria

Given a Credentialing Specialist enters a 10-digit value in Provider__c.NPI__c and clicks 'Validate NPI'
When the LWC component invokes the NPPES NPI Registry REST API (https://npiregistry.cms.hhs.gov/api/)
Then within 5 seconds the following fields are populated on Provider__c:
  - Provider__c.NPI_Status__c = 'Active' or 'Deactivated' per NPPES response
  - Provider__c.NPI_Enumeration_Type__c (NPI-1 Individual or NPI-2 Organizational)
  - Provider__c.Primary_Taxonomy_Code__c from taxonomy[0].code
  - Provider__c.Taxonomy_Description__c from taxonomy[0].desc
  - Provider__c.NPI_Last_Verified_Date__c = TODAY()

Given the NPPES API returns a deactivated or non-existent NPI
When the validation result is processed
Then Provider__c.NPI_Status__c = 'Invalid' and a warning banner displays: 'NPI not found or deactivated in NPPES. Credentialing cannot proceed until a valid NPI is confirmed.'
And Provider_Status__c is set to 'NPI Review Required'

Given a valid NPI is confirmed
When Provider__c.NPI_Status__c = 'Active' is saved
Then the associated Credentialing_Case__c.NPI_Verified__c checkbox is set to TRUE via a Platform Event (NPI_Validated__e) published by the Apex callout handler

Given the NPPES API is unavailable (timeout or HTTP 5xx)
When the callout fails
Then the error is logged to Integration_Error_Log__c with fields: Error_Source__c = 'NPPES', Provider__c = {record Id}, Error_Message__c = {HTTP status + body}, and the user sees: 'NPI validation service is temporarily unavailable. Please retry or contact your system administrator.'

Given an NPI of type NPI-2 (Organizational) is entered on a Provider__c (Person Account)
When validation completes
Then a validation warning is displayed: 'NPI-2 Organizational NPIs should be associated with a Facility__c (Business Account) record, not an individual provider.'

## Solution

Build an LWC 'npiValidationPanel' placed on the Provider__c Lightning Record Page via App Builder. The component calls an @AuraEnabled Apex controller method 'NppesIntegrationService.validateNPI(String npi)' which performs an HTTP GET callout to the NPPES REST API using a Named Credential 'NPPES_Registry'. Response is deserialized via a custom NppesResponseWrapper class. On success, the controller updates Provider__c fields via a DML update and publishes an NPI_Validated__e Platform Event. A Platform Event-Triggered Flow subscribes to NPI_Validated__e and updates the linked Credentialing_Case__c.NPI_Verified__c. Callout errors are written to Integration_Error_Log__c via a separate Apex handler implementing the Database.AllowsCallouts interface. Named Credential and Remote Site Setting for npiregistry.cms.hhs.gov configured in Setup. Unit tests mock HttpCalloutMock to achieve >85% coverage.

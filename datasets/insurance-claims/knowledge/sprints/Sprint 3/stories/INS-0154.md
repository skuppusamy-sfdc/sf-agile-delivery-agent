# INS-0154: Bug — Credentialing_Case__c Status does not advance to 'Committee Review' when all required credentialing documents are marked Received

## Description

Steps to Reproduce:
1. Navigate to an existing Credentialing_Case__c record with Status__c = 'Documents Pending'.
2. Open all related Credentialing_Document__c records and set each Document_Status__c to 'Received'.
3. Save the last Credentialing_Document__c record.
4. Return to the parent Credentialing_Case__c record.

Expected Behavior: Credentialing_Case__c.Status__c automatically advances to 'Committee Review' once all child Credentialing_Document__c records have Document_Status__c = 'Received'.

Actual Behavior: Credentialing_Case__c.Status__c remains at 'Documents Pending' and does not transition. The status must be manually updated by the Credentialing Specialist, introducing risk of process delays and SLA breaches.

Additional Notes: Issue appears to have been introduced in the Sprint 2 deployment on 2024-11-14. Affects all Credentialing_Case__c records regardless of LOB. Confirmed in both Partial Sandbox and UAT org. The Record-Triggered Flow 'Credentialing_Document_Status_Rollup_Flow' is suspected — the decision element condition checking ALL documents as 'Received' uses a COUNT formula that does not account for newly inserted records in the same transaction.

## Acceptance Criteria

As per Description. Additionally: Credentialing_Case__c.Status__c must equal 'Committee Review' within the same save transaction when the last Credentialing_Document__c.Document_Status__c is set to 'Received'. No manual intervention should be required.

## Solution

Root cause identified in the Record-Triggered Flow 'Credentialing_Document_Status_Rollup_Flow' on Credentialing_Document__c. The Get Records element fetching sibling documents uses a filter that inadvertently excludes the triggering record (using $Record.Id in a NOT EQUAL filter intended for a different purpose). Fix: remove the erroneous exclusion filter so all sibling documents including the current record are evaluated. Add a Decision element: IF COUNT(Credentialing_Document__c where Parent_Case__c = triggeringRecord.Parent_Case__c AND Document_Status__c != 'Received') = 0 THEN update parent Credentialing_Case__c.Status__c = 'Committee Review'. Deploy fix to Partial Sandbox for regression testing before promoting to UAT and Production. Add unit test in CredentialingDocumentTriggerTest covering the all-documents-received scenario with minimum 90% code coverage.

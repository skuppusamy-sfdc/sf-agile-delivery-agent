## Description

As a Provider Relations Specialist, I need a comprehensive Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can immediately see credentialing status, NPI validation state, active Provider Network Agreements, and recent claims activity in a single unified view without navigating across multiple objects.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the following data sections are displayed:
  1. Header: Provider__c.Name, Provider__c.NPI__c, NPI_Validation_Status__c badge (color-coded: green='Validated', yellow='Pending Validation', red='Invalid')
  2. Credentialing Panel: linked Credentialing_Case__c.Status__c, Credentialing_Case__c.Due_Date__c, Credentialing_Case__c.Case_Type__c for the most recent open case
  3. Network Agreements Panel: list of active Provider_Network_Agreement__c records where PNA.Status__c = 'Active' and PNA.Effective_Date__c <= TODAY() and PNA.Termination_Date__c >= TODAY()
  4. Claims Activity Panel: count of Claim__c records linked to provider in last 90 days, grouped by Claim__c.Status__c

Given no active Credentialing_Case__c exists for the provider,
When the FlexCard renders the Credentialing Panel,
Then a placeholder message is shown: 'No active credentialing case found.'

Given the Provider__c has more than 5 active PNAs,
When the Network Agreements Panel renders,
Then a paginated list shows 5 records per page with a 'View All' link navigating to a filtered PNA list view.

-Given the FlexCard should display Fee_Schedule__c line items inline- (deprecated — fee schedule detail moved to separate Fee Schedule tab per UX review)

Given a Provider Relations Specialist clicks the 'Initiate Re-Credentialing' action button on the FlexCard,
When clicked,
Then an OmniScript (ProviderReCredentialingOS) launches in a modal to capture re-credentialing details.

## Solution

Build using OmniStudio FlexCard (Provider360_FC) with multiple child cards for each panel section. DataRaptors (DR_GetProviderCredentialing, DR_GetActivePNAs, DR_GetProviderClaimsSummary) retrieve data via SOQL. NPI validation badge implemented as a custom FlexCard State with conditional CSS classes. The 'Initiate Re-Credentialing' action invokes OmniScript (ProviderReCredentialingOS) via FlexCard Action of type 'OmniScript'. FlexCard deployed to Provider__c (Person Account) Lightning Record Page via App Builder. Caching strategy: DataRaptor cache TTL set to 300 seconds for claims summary to reduce SOQL governor limit exposure.

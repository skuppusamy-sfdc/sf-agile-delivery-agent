## Description

**Bug Report — NPI Validation Date stale on Provider record update**

**Steps to Reproduce:**
1. Open an existing Provider__c record where NPI_Validated__c = TRUE and NPI_Validation_Date__c is set to a prior date.
2. Edit the NPI__c field to a new valid NPI number (e.g., change from 1234567890 to 0987654321).
3. Save the record.
4. Observe Provider__c.NPI_Validation_Date__c — it retains the OLD date from the previous validation.
5. Provider__c.NPI_Validated__c also incorrectly remains TRUE without re-running NPPES validation.

**Expected Behavior:**
When NPI__c is changed on an existing Provider__c record, NPI_Validated__c should be reset to FALSE, NPI_Status__c should be set to 'Validation Pending', and NPI_Validation_Date__c should be cleared until the NPPES re-validation completes.

**Actual Behavior:**
NPI_Validation_Date__c retains the previous validation date. NPI_Validated__c remains TRUE. No re-validation callout is triggered. This creates a data integrity risk where claims may adjudicate against an unvalidated NPI.

**Environment:** Full Sandbox (FS-01)
**Regression:** Introduced in INS-0176 deployment on 2024-11-04
**Severity:** High — potential compliance impact if invalid NPIs pass adjudication checks.

## Solution

Root cause: The Record-Triggered Flow on Provider__c built in INS-0176 uses entry condition 'Record is Created' only, missing the 'Record is Updated AND NPI__c is Changed' condition. Fix: Update the Record-Triggered Flow trigger to also fire on Update when ISCHANGED(NPI__c) = TRUE. Add a flow path that resets NPI_Validated__c = FALSE, NPI_Status__c = 'Validation Pending', and clears NPI_Validation_Date__c before publishing the 'NPI_Validation_Retry__e' Platform Event for async re-validation. Also update the NPPESValidationService Apex class (INS-0176) to handle re-validation writes correctly on update DML. Add regression test to existing NPPESValidationService_Test class covering the NPI-change-on-existing-record scenario. Deploy and validate in FS-01 before promoting to UAT.

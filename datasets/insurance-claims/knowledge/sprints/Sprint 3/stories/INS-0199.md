## Description

As a Credentialing Specialist, I need a Credentialing_Case__c record to be automatically created when a new Provider__c (Person Account) record is submitted for network participation so that the credentialing review process is initiated without manual intervention and no provider slips through without a formal review cycle.

## Acceptance Criteria

Given a new Provider__c record is saved with Provider_Status__c = 'Pending Credentialing'
When the record is submitted by a Provider Network Specialist
Then a Credentialing_Case__c record is auto-created with:
  - Credentialing_Case__c.Status__c = 'Initiated'
  - Credentialing_Case__c.Provider__c = Provider__c.Id
  - Credentialing_Case__c.LOB__c populated from Provider__c.Line_of_Business__c
  - Credentialing_Case__c.Assigned_Specialist__c defaulting to the queue 'Credentialing Queue'
  - Credentialing_Case__c.Due_Date__c = TODAY() + 30 calendar days

Given the Credentialing_Case__c is created
When the assigned Credentialing Specialist opens the record
Then a checklist of required credentialing documents is visible as a related list (Credentialing_Document__c records with Document_Type__c values: 'Medical License', 'DEA Certificate', 'Malpractice Insurance', 'CAQH Profile', 'Board Certification')

Given a Provider__c record already has an active Credentialing_Case__c (Status__c NOT IN 'Closed - Approved', 'Closed - Denied')
When a duplicate submission is attempted
Then the system displays a validation error: 'An active Credentialing Case already exists for this provider. Please review Case [Case Number] before resubmitting.'

-Given Provider__c.NPI__c is blank, the case should still be created with a warning flag-
Given Provider__c.NPI__c is blank at time of submission
When the record-triggered flow evaluates the record
Then the submission is blocked with error: 'NPI is required before a Credentialing Case can be initiated.' and Provider_Status__c remains 'Pending Credentialing'

## Solution

Implement a Record-Triggered Flow on Provider__c (Person Account) firing After Save on Create. Flow checks for existing active Credentialing_Case__c via a Get Records element filtered on Provider__c = {!$Record.Id} and Status__c NOT IN ('Closed - Approved','Closed - Denied'). If none found, a Create Records element generates the Credentialing_Case__c with mapped fields. A separate Before-Save flow enforces the NPI__c validation using a Decision element and a Custom Error message element. Assign created cases to the 'Credentialing Queue' using the AssignedToId field mapped to the Queue SObject Id retrieved via a Get Records on Group where Type = 'Queue' and Name = 'Credentialing Queue'. Deploy Permission Set Group 'Credentialing_Ops_PSG' to grant FLS on all Credentialing_Case__c fields to Credentialing Specialist profile.

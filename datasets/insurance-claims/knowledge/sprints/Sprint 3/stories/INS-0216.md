## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that displays the provider's current credentialing status, active Provider_Network_Agreement__c (PNA) records, and any open Credentialing_Case__c or Network_Termination_Case__c records, so that I can quickly assess a provider's full network standing without navigating multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the card displays: NPI__c, TIN__c, NPI_Validation_Status__c (with color-coded badge: green='Valid', yellow='Pending', red='Invalid'/'Not Found'), and CAQH_Attestation_Date__c.

Given the Provider__c has one or more active Provider_Network_Agreement__c records (PNA__c.Status__c = 'Active'),
When the PNA section of the FlexCard renders,
Then each active PNA is listed with PNA.Network_Name__c, PNA.Effective_Date__c, PNA.Termination_Date__c, and PNA.LOB__c.

Given the Provider__c has an open Credentialing_Case__c (Status__c NOT IN ('Approved', 'Denied', 'Withdrawn')),
When the Credentialing section renders,
Then the open case is displayed with a direct navigation link, Credentialing_Case__c.Status__c, Credentialing_Type__c, and Days_Open__c (formula field).

Given there are no active PNAs and no open credentialing cases,
When the relevant FlexCard sections render,
Then empty-state messages are displayed: 'No active network agreements found' and 'No open credentialing cases' respectively.

-Given the FlexCard also displays Claim__c history summary for the provider, When the card loads, Then total claims in the last 12 months are shown.- (Descoped to INS-0231 – Provider Claims Summary FlexCard.)

## Solution

Build an OmniStudio FlexCard (Name: 'Provider_360_Summary') with child FlexCards for PNA, Credentialing Cases, and Termination Cases sections. DataRaptor Extract actions query Provider__c, Provider_Network_Agreement__c, Credentialing_Case__c, and Network_Termination_Case__c via SOQL. Conditional visibility on card sections driven by DataRaptor response null checks. Color-coded badges implemented via FlexCard state configuration mapped to NPI_Validation_Status__c picklist values. The FlexCard is embedded on the Provider__c Lightning Record Page via App Builder. A custom Permission Set Group ('Provider_Relations_Console') controls visibility. DataRaptor caching enabled with a 5-minute TTL to reduce SOQL query volume on high-traffic record pages.

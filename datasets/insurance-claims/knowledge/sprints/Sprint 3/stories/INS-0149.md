## Description

As a Compliance Officer, I need the system to proactively monitor provider credentialing expiration dates and automatically generate re-credentialing tasks and notifications at defined intervals before expiration, so that Acme Insurance maintains continuous compliance with NCQA and state regulatory credentialing standards and avoids lapsed provider credentials in the network.

## Acceptance Criteria

Given a scheduled Apex job runs nightly at 2:00 AM,
When Provider__c records are evaluated where Provider__c.Credentialing_Expiry_Date__c is within 90, 60, or 30 calendar days from today and Provider__c.Network_Status__c = 'In-Network',
Then a Credentialing_Case__c of RecordType 'Re-Credentialing' is created for each qualifying provider not already having an open Re-Credentialing case, with Credentialing_Case__c.Status__c = 'New' and Credentialing_Case__c.Trigger_Reason__c = 'Scheduled Re-Credentialing'.

Given the re-credentialing Credentialing_Case__c is created at the 90-day threshold,
When the record is committed,
Then an email notification is sent to the provider's credentialing contact (Provider__c.Credentialing_Contact_Email__c) using the 'Re_Credentialing_90Day_Email_Template' and a Task is assigned to the owning Credentialing Specialist with Due Date = today + 30 days.

Given a provider's Credentialing_Expiry_Date__c passes without an active PNA renewal or re-credentialing case reaching 'Approved',
When the nightly job evaluates the provider and expiry date is in the past,
Then Provider__c.Network_Status__c is set to 'Suspended', all active PNA__c records for the provider have PNA__c.Status__c set to 'Suspended', and a Provider_Change_Notification__c (PCN) record is created with PCN__c.Change_Type__c = 'Credential Expiration Suspension'.

Given a Credentialing Specialist manually creates a re-credentialing Credentialing_Case__c before the automated job runs,
When the nightly job evaluates the provider,
Then the job detects the open case and does not create a duplicate (deduplication via SOQL check on RecordType + Status__c not in ('Closed','Withdrawn','Approved')).

Given the PCN is created for a suspended provider,
When PCN__c.Status__c = 'Issued',
Then any open Claim__c records referencing the provider with Claim__c.Status__c = 'Pending Adjudication' are flagged with Claim__c.Provider_Credential_Flag__c = TRUE for Claims Operations Manager review.

## Solution

Implement a Schedulable Apex class 'CredentialingExpirationMonitor' scheduled via a Scheduled Job to run nightly at 2:00 AM in the org's local timezone. Batch Apex (Database.Batchable) processes Provider__c records in chunks of 200 to avoid governor limits. Re-credentialing Credentialing_Case__c creation uses a custom RecordType 'Re_Credentialing'. Email notifications use Messaging.SingleEmailMessage with an org-wide email address. PCN creation and PNA suspension handled in the same batch execution context. Claim flagging uses a separate SOQL query on Claim__c filtered by Provider__c. Custom Metadata Type 'Credentialing_Threshold_Config__mdt' stores the 90/60/30-day threshold values for admin configurability without code changes. Monitoring dashboard built with a CRM Analytics (CRMA) dataset refreshed post-batch execution.

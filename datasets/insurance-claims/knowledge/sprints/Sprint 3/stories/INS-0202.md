## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) Lightning Record Page that surfaces credentialing status, NPI validation state, active Provider_Network_Agreement__c records, and a summary of open Claim__c records so that I can assess a provider's full engagement with Acme Insurance without navigating across multiple objects and related lists.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page
When the Provider 360 FlexCard renders
Then the following data blocks are visible:
  - NPI block: Provider__c.NPI__c, Provider__c.NPI_Status__c (color-coded: green = Active, red = Invalid, amber = Pending), Provider__c.NPI_Last_Verified_Date__c
  - Credentialing block: Latest Credentialing_Case__c.Status__c, Credentialing_Case__c.Due_Date__c, CAQH_Profile_Status__c
  - Network block: Count of active Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active' and PNA__c.Effective_Date__c <= TODAY() and PNA__c.Termination_Date__c >= TODAY()
  - Claims block: Count of Claim__c records where Claim__c.Billing_Provider__c = Provider__c.Id and Claim__c.Status__c IN ('Submitted','In Review','Pending Information')

Given the FlexCard is rendered
When the Credentialing_Case__c.Status__c = 'Suspended'
Then the credentialing block header renders with a red warning banner: 'Provider Credentialing Suspended — Claims may not be payable'

Given a Provider Relations Specialist clicks the 'View Active Agreements' action on the Network block
When the action is triggered
Then an OmniScript modal opens listing all active PNA records with fields: PNA__c.Agreement_Number__c, PNA__c.LOB__c, PNA__c.Effective_Date__c, PNA__c.Reimbursement_Model__c

Given no active PNA records exist for the provider
When the Network block renders
Then the block displays: 'No active network agreements. Provider is currently out-of-network.' with a 'Initiate Contracting' quick action button

## Solution

Build OmniStudio FlexCard 'Provider360Card' using DataRaptors to source data from Provider__c, Credentialing_Case__c (child, ordered by CreatedDate DESC, limit 1), Provider_Network_Agreement__c (aggregate count with filter), and Claim__c (aggregate count with filter). Use a DataRaptor Extract for each data source and wire them as DataSources on the FlexCard. Conditional state styling applied via FlexCard State configuration mapping NPI_Status__c and Credentialing_Case__c.Status__c values to color tokens. The 'View Active Agreements' action launches an OmniScript 'ViewProviderAgreements' in a modal using the FlexCard Action type 'OmniScript'. The FlexCard is deployed to the Provider__c Lightning Record Page via App Builder as a custom component. Assign FlexCard visibility via Permission Set 'Provider_Relations_PS'.

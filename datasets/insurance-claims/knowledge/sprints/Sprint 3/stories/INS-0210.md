## Description

**Bug Report**

**Summary:** Credentialing_Case__c.Due_Date__c is being set to TODAY() + 45 calendar days regardless of whether the creation date falls on a weekend or CMS-recognized public holiday. Compliance requires the 45-day window to be calculated in business days.

**Steps to Reproduce:**
1. Navigate to Salesforce and create a new Provider__c record with Provider_Status__c = 'Pending Credentialing' on a Saturday (or set system date to a Saturday in sandbox).
2. Save the record.
3. Observe the auto-created Credentialing_Case__c record.
4. Check Credentialing_Case__c.Due_Date__c.

**Expected Behavior:** Credentialing_Case__c.Due_Date__c should be set to the date that is 45 *business days* from the creation date, skipping weekends and public holidays defined in the Business_Holiday__c custom object.

**Actual Behavior:** Credentialing_Case__c.Due_Date__c = creation date + 45 *calendar* days. For a provider created on Saturday Dec 28, 2024, Due_Date__c = Feb 11, 2025 (calendar), but expected = Feb 27, 2025 (business days excluding holidays).

**Impact:** Credentialing cases are being flagged as overdue prematurely, causing false SLA breach alerts and incorrect compliance reporting to the Compliance Officer. Affects all providers onboarded since Sprint 2 go-live (estimated 47 records).

**Environment:** Production-mirror sandbox (Acme-Full-Copy-S3)
**Reported by:** Emily Watson (QA Lead) during Sprint 3 regression cycle

## Acceptance Criteria

As per Description. Additionally:
Given Credentialing_Case__c is created on any day of the week
When Due_Date__c is calculated
Then the 45-business-day calculation must skip Saturday, Sunday, and any date present in Business_Holiday__c.Holiday_Date__c where Business_Holiday__c.Active__c = true
And a data fix script must be provided to retroactively correct the 47 affected Credentialing_Case__c records in production

## Solution

Replace the current Flow formula (TODAY() + 45) in the Record-Triggered Flow (Credentialing_Case_Creation_Flow) with an invocable Apex method call (BusinessDayCalculatorAction.cls). The Apex method accepts a start date and number of business days, queries Business_Holiday__c for active holidays within the projected date range, and iterates day-by-day skipping weekends and holidays. A Custom Metadata Type (Business_Day_Config__mdt) will store the business day count per case type (Initial = 45, Re-Credentialing = 30) to avoid hardcoding. Provide an accompanying data fix Apex script (CredentialingDueDateDataFix.cls) executed as an anonymous Apex block in production after deployment, with a before/after CSV export for audit trail submitted to the Compliance Officer.

## Description

As a Claims Operations Manager, I need a Salesforce Lightning Dashboard dedicated to the credentialing pipeline so that I can monitor Credentialing_Case__c volume by status, identify aging cases at risk of breaching the 45-day credentialing turnaround SLA, and track credentialing throughput by LOB to support capacity planning and compliance reporting.

## Acceptance Criteria

Given the Claims Operations Manager navigates to the 'Credentialing Operations' Lightning Dashboard,
When the dashboard loads,
Then the following report-backed components are displayed:
  1. Donut Chart — Credentialing_Case__c count grouped by Status__c (Initiated, In Review, Awaiting Provider Authorization, Approved, Denied, Withdrawn)
  2. Bar Chart — Credentialing_Case__c count by LOB__c (sourced from linked Provider_Network_Agreement__c.LOB__c or Credentialing_Case__c.LOB__c)
  3. Lightning Table — Top 20 aging Credentialing_Case__c records where Due_Date__c <= TODAY() + 7 days and Status__c NOT IN ('Approved','Denied','Withdrawn'), showing Case Number, Provider Name, Assigned To, Due Date, Days Open
  4. Metric Tile — Average cycle time in days for cases closed in the current calendar month (Credentialing_Case__c.Closed_Date__c - Credentialing_Case__c.CreatedDate)
  5. Metric Tile — Count of cases with Status__c = 'Awaiting Provider Authorization' > 10 days

Given the dashboard is filtered by a global Date Range filter,
When the Operations Manager selects a custom date range,
Then all dashboard components reflect data within the selected CreatedDate range.

Given a Credentialing_Case__c Due_Date__c is within 3 days of TODAY(),
When the aging table renders,
Then the Due Date cell is highlighted in red using a conditional formatting rule on the underlying report.

Given the Operations Manager clicks a bar in the LOB chart,
When the drill-down is triggered,
Then the user is navigated to the source report filtered to the selected LOB__c value.

## Solution

Build 5 custom reports in the 'Credentialing Operations' report folder using Credentialing_Case__c as the primary object with cross-object fields from Provider__c (Name, NPI__c) and Provider_Network_Agreement__c (LOB__c). Reports: (1) CC_StatusSummary_R — summary report grouped by Status__c; (2) CC_LOBBreakdown_R — summary by LOB__c; (3) CC_AgingPipeline_R — tabular with filter Due_Date__c <= NEXT_N_DAYS:7; (4) CC_AvgCycleTime_R — summary with formula field for cycle time; (5) CC_PendingAuth_R — metric filter. Dashboard (Credentialing_Operations_DB) built in Lightning with dynamic filter widget for date range. Conditional highlighting configured on CC_AgingPipeline_R using Report Conditional Formatting (native Salesforce feature — no Apex required). Dashboard shared via Folder permissions to 'Claims Operations Manager' profile. Scheduled dashboard email subscription set for 8 AM daily.

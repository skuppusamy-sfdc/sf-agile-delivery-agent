## Description

As a Credentialing Specialist, I need a guided OmniScript checklist embedded on the Credentialing_Case__c record that walks me through all primary source verification (PSV) steps — including medical license, DEA, board certification, malpractice, and OIG exclusion check — so that verification is standardized, auditable, and compliant with NCQA credentialing standards.

## Acceptance Criteria

Given a Credentialing_Case__c record with Status__c = 'In Review' is open,
When the Credentialing Specialist clicks 'Start PSV Checklist',
Then the OmniScript 'PSV_Verification_Checklist' launches in a modal with the following steps presented sequentially:
  Step 1: Medical License Verification (State Board lookup, License Number, Expiry Date)
  Step 2: DEA Registration Verification
  Step 3: Board Certification Verification
  Step 4: Malpractice Insurance Verification
  Step 5: OIG Exclusion Check (link to OIG LEIE database result)
  Step 6: NPDB Query confirmation
  Step 7: Review & Submit

Given the Specialist completes all 7 steps and clicks Submit,
When the OmniScript finalizes,
Then Credentialing_Case__c.PSV_Completed__c = TRUE, Credentialing_Case__c.PSV_Completion_Date__c = TODAY(), Credentialing_Case__c.Status__c advances to 'PSV Complete', and a PDF summary of the checklist responses is generated and attached to the Credentialing_Case__c Files section.

Given any PSV step returns a discrepancy (e.g., license expired, OIG exclusion found),
When the Specialist marks the step as 'Failed',
Then Credentialing_Case__c.Status__c = 'PSV Discrepancy', a high-priority Case is created in the Compliance queue, and the Compliance Officer is notified via email.

Given the OmniScript is partially completed and the user closes the modal,
When the Specialist reopens the OmniScript on the same Credentialing_Case__c,
Then previously entered responses are pre-populated (resume capability) and incomplete steps are highlighted.

-Step 3 Board Certification to auto-populate from CAQH data fetch- (dependency on INS-0178; will be wired in Phase 2 integration sprint)

Given Credentialing_Case__c.LOB__c = 'Medicare Advantage',
When the OmniScript loads,
Then an additional Step 2b 'Medicare Enrollment Verification (PECOS)' is injected between Steps 2 and 3.

## Solution

Build OmniScript 'PSV_Verification_Checklist' in OmniStudio with 7 sequential Step elements and conditional branching using Set Values and Conditional View actions. Use DataRaptor Load to write checklist responses to Credentialing_Case__c and a custom object 'PSV_Verification_Step__c' (child of Credentialing_Case__c) for granular step-level audit trail. Implement resume functionality using a DataRaptor Extract on OmniScript load to pre-populate existing PSV_Verification_Step__c records. PDF generation via Conga Composer triggered by an Apex Action invocable from the OmniScript Submit step, attaching output to ContentVersion linked to Credentialing_Case__c. LOB-based step injection handled via a Conditional Block element checking Credentialing_Case__c.LOB__c. Discrepancy path triggers a Flow Sub-Flow to create the Compliance Case and send email alert. Deploy OmniScript as a Quick Action on Credentialing_Case__c Lightning Record Page.

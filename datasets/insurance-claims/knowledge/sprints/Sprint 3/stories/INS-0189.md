## Description

As a Credentialing Specialist, I need the system to retrieve and sync provider credentialing data from CAQH ProView into Salesforce Provider__c and Credentialing_Case__c records using the provider's CAQH_Provider_ID__c, so that manual data entry is minimized, data accuracy is improved, and re-attestation expiry dates are tracked for proactive outreach.

## Acceptance Criteria

Given a Provider__c record has a non-null CAQH_Provider_ID__c value,
When a Credentialing Specialist clicks the 'Sync from CAQH' action button on the Provider__c record page,
Then the system makes an authenticated callout to the CAQH ProView REST API and retrieves the provider's current profile data.

Given a successful CAQH API response is received,
When the sync completes,
Then the following fields are updated on Provider__c: Provider__c.Board_Certification__c, Provider__c.DEA_Number__c, Provider__c.DEA_Expiration_Date__c, Provider__c.Malpractice_Insurance_Carrier__c, Provider__c.Malpractice_Policy_Expiration__c, and Provider__c.CAQH_Attestation_Date__c.

Given Provider__c.CAQH_Attestation_Date__c is set or updated,
When the value indicates the re-attestation window is within 45 days of expiry (CAQH re-attestation is required every 120 days),
Then a Task is auto-created with Subject = 'CAQH Re-Attestation Due – [Provider Name]', Due Date = CAQH_Attestation_Date__c + 120 - 45, and assigned to the provider's owning Credentialing Specialist.

Given the CAQH API returns a provider with Authorization_Status__c = 'Revoked' or 'Expired',
When the sync processes the response,
Then Provider__c.CAQH_Authorization_Status__c is updated accordingly, an alert is posted to the Credentialing_Case__c Chatter feed, and the Credentialing_Case__c.Status__c is set to 'Pending – CAQH Authorization Review'.

Given the CAQH sync is triggered but the CAQH_Provider_ID__c is null or malformed,
When the action is invoked,
Then the system displays an error toast message: 'CAQH Provider ID is missing or invalid. Please update the provider record before syncing.' and no callout is made.

Given a nightly scheduled batch sync is configured,
When the batch runs at 2:00 AM EST,
Then all Provider__c records with CAQH_Provider_ID__c populated and CAQH_Last_Sync_Date__c older than 30 days are included in the sync batch, with a maximum of 200 records per batch execution to respect API rate limits.

## Solution

Build an Apex REST callout class (CAQHProViewService.cls) using a Named Credential for CAQH API authentication. A Lightning Web Component (caqhSyncAction) provides the 'Sync from CAQH' quick action button on the Provider__c page, invoking an Apex Controller method. Field mapping from CAQH response JSON to Provider__c fields is defined in a Custom Metadata Type (CAQH_Field_Mapping__mdt) for maintainability. Re-attestation Task creation is handled by a Record-Triggered Flow on Provider__c firing on CAQH_Attestation_Date__c change. The nightly batch is implemented as a Schedulable Apex class (CAQHNightlySyncBatch.cls) using Database.Batchable with scope 200. CAQH API credentials are stored in a Named Credential with a custom Auth Provider. All sync activity is logged to a custom CAQH_Sync_Log__c object for audit purposes.

## Description

Steps to Reproduce:
1. Create a new Provider__c (Person Account) record with a valid NPI__c value.
2. Create a Credentialing_Case__c linked to the Provider__c and set Status__c = 'Intake Submitted'.
3. Observe the NPPES validation callout fires (confirmed via debug logs — HTTP 200 returned with valid NPI data).
4. Navigate to the linked Provider__c record.
5. Observe that Provider__c.NPI_Status__c still shows 'Unverified' and Provider__c.NPI_Verified_Date__c remains null.

Expected Behavior: Upon successful NPPES validation (NPI_Validation_Status__c = 'Verified' on Credentialing_Case__c), the linked Provider__c.NPI_Status__c should be updated to 'Active' and Provider__c.NPI_Verified_Date__c should be stamped with today's date.

Actual Behavior: The Credentialing_Case__c fields update correctly, but the Provider__c fields are not updated. Debug logs show the Queueable Apex job (NPPESProviderUpdateQueueable) is enqueued but throws a System.QueryException: 'List has no rows for assignment to SObject' when attempting to re-query the Provider__c record by Id, likely due to a null Provider__c lookup on the Credentialing_Case__c being passed to the Queueable constructor.

Environment: Acme Insurance – Full Sandbox (FS1)
Regression: Not present in Sprint 2 dev sandbox. Introduced after INS-0183 deployment on 2024-11-14.

## Acceptance Criteria

As per Description. Verified fix must demonstrate Provider__c.NPI_Status__c = 'Active' and Provider__c.NPI_Verified_Date__c populated within the same transaction cycle as Credentialing_Case__c.NPI_Validation_Status__c = 'Verified'. Unit test coverage must cover the null Provider__c lookup scenario with an appropriate assertion.

## Solution

Root cause: NPPESProviderUpdateQueueable.cls receives the Credentialing_Case__c Id but re-queries without the Provider__c field in the SOQL SELECT clause, causing the SObject assignment to fail silently in some execution paths. Fix: Update the SOQL in the Queueable to include Provider__c in the field list: SELECT Id, Provider__c, NPI_Validation_Status__c FROM Credentialing_Case__c WHERE Id = :caseId. Add a null-check guard before the Provider__c DML update. Add a @IsTest method covering the null-lookup edge case. Deploy fix to FS1 and regression test INS-0183 acceptance criteria end-to-end.

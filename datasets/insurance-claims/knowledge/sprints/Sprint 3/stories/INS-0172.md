## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces credentialing status, active Provider_Network_Agreement__c records, NPI verification status, and a summary of open Claim__c records so that I can assess a provider's full engagement with Acme Insurance in a single view without navigating to multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist navigates to any Provider__c record page
When the page loads
Then the Provider 360 FlexCard is visible and displays: Provider__c.NPI__c, Provider__c.NPPES_Verified__c (shown as a green/red badge), Provider__c.Primary_Taxonomy_Code__c, and Provider__c.Network_Participation_Status__c

Given the FlexCard is loaded
When active PNA records exist (PNA.Status__c = 'Active')
Then each active PNA is listed in a card child element showing PNA.LOB__c, PNA.Effective_Date__c, PNA.Expiration_Date__c, and PNA.Fee_Schedule__c

Given the FlexCard is loaded
When open Credentialing_Case__c records exist (Status__c NOT IN ('Approved','Denied','Withdrawn'))
Then the FlexCard displays a 'Credentialing In Progress' badge with Credentialing_Case__c.Status__c and Credentialing_Case__c.Submitted_Date__c

Given the FlexCard is loaded
When Claim__c records related to the provider exist with Claim__c.Status__c IN ('Submitted','Under Review','Pending Information')
Then a count of open claims is displayed with a hyperlink navigating to a filtered Claims list view scoped to that provider

Given the Provider__c record has no active PNAs, no open credentialing cases, and no open claims
When the FlexCard loads
Then each section displays a contextual empty-state message (e.g., 'No active network agreements on file.')

## Solution

OmniStudio FlexCard built using FlexCard Designer with parent DataRaptor Extract sourcing Provider__c fields. Child DataRaptor Extracts configured for related PNA, Credentialing_Case__c, and Claim__c counts using relationship queries. NPPES verification badge implemented via FlexCard conditional styling (green = NPPES_Verified__c true, red = false). FlexCard deployed to Provider__c (Person Account) record page via App Builder. Claims count hyperlink uses NavigationMixin.Navigate in a child LWC action embedded within the FlexCard. FlexCard access controlled via Permission Set Group 'Provider_Relations_Standard' assigned to Provider Relations Specialist profile.

## Description

As a Credentialing Specialist, I need the Salesforce platform to retrieve and auto-populate credentialing data from CAQH ProView when a provider's CAQH_Provider_ID__c is entered on a Credentialing_Case__c, so that I can eliminate duplicate data entry, reduce credentialing cycle time, and leverage the provider's already-attested CAQH profile for primary source verification.

## Acceptance Criteria

Given a Credentialing_Case__c record has a non-null CAQH_Provider_ID__c and the associated Provider__c.NPI_Validation_Status__c = 'Valid',
When a Credentialing Specialist clicks the 'Fetch CAQH Data' action button on the record page,
Then a callout to the CAQH ProView API is initiated and returns a response within 10 seconds.

Given a successful CAQH API response (HTTP 200),
When the response payload is parsed,
Then the following Provider__c fields are updated: License_Number__c, License_State__c, License_Expiration_Date__c, Board_Certification__c, Malpractice_Carrier__c, Malpractice_Policy_Expiration__c, and CAQH_Attestation_Date__c.

Given the CAQH attestation date is more than 120 days old,
When data is fetched and CAQH_Attestation_Date__c is evaluated,
Then a warning is displayed on the Credentialing_Case__c record: 'CAQH profile attestation is outdated. Contact provider to re-attest before proceeding.' and Credentialing_Case__c.Status__c remains unchanged.

Given the CAQH API returns a 404 (provider not found),
When the response is processed,
Then Provider__c.CAQH_Status__c is set to 'Not Found' and a Task is auto-created for the assigned Credentialing Specialist to obtain the correct CAQH ID from the provider.

Given data is successfully fetched from CAQH,
When fields are written to Provider__c,
Then a Credentialing_Data_Audit__c child record is created capturing the previous field values, new values, timestamp, and running user for compliance audit trail purposes.

## Solution

Create a Named Credential ('CAQH_ProView') for the CAQH REST API endpoint with OAuth 2.0 authentication. An Apex class (CAQHIntegrationService) handles the callout and response mapping. A Quick Action on Credentialing_Case__c invokes an Apex Action via a Screen Flow, presenting a confirmation dialog before executing the callout. Response mapping is defined in a Custom Metadata Type (CAQH_Field_Mapping__mdt) to allow admin-configurable field mappings without code changes. Audit trail records are inserted via a dedicated Apex method called within the same transaction. Remote Site Settings and CSP Trusted Sites are configured for the CAQH endpoint. Error handling uses a custom exception class (CAQHIntegrationException) with structured logging to a custom object (Integration_Error_Log__c).

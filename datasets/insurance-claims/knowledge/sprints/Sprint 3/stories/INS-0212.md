## Description

As a Compliance Officer, I need Provider_Change_Notification__c (PCN) records to be automatically generated whenever a provider's credentialing status change has a material impact on their active Provider_Network_Agreement__c (PNA) records, so that downstream stakeholders (MCO partners, TPAs, and internal Claims Operations) are notified within the regulatory 30-day window and audit trails are maintained for CMS reporting.

## Acceptance Criteria

Given a Credentialing_Case__c.Status__c transitions to 'Denied' or 'Suspended'
When the status change is saved
Then the system queries all active Provider_Network_Agreement__c records where PNA__c.Provider__c = the affected provider AND PNA__c.Status__c = 'Active'
And for each active PNA, a Provider_Change_Notification__c is created with:
  - PCN__c.Provider__c = affected Provider__c Id
  - PCN__c.PNA__c = linked PNA Id
  - PCN__c.Change_Type__c = 'Credentialing Status Change'
  - PCN__c.Change_Reason__c = Credentialing_Case__c.Denial_Reason__c
  - PCN__c.Effective_Date__c = TODAY()
  - PCN__c.Notification_Deadline__c = TODAY() + 30
  - PCN__c.Status__c = 'Pending Distribution'
  - PCN__c.LOB__c = PNA__c.LOB__c

Given PCN records are created
When PCN__c.Status__c = 'Pending Distribution'
Then an automated email is sent to all contacts on the related PNA__c with Role__c = 'MCO Liaison' or 'TPA Contact' using the 'Provider_Change_Notification_Email' email template
And PCN__c.Last_Notification_Sent__c is stamped with the send timestamp

Given a PCN is created and Notification_Deadline__c is within 5 days
When a daily scheduled job runs
Then an escalation Task is created on the PCN assigned to the Provider Relations Specialist with Subject = 'PCN Deadline Approaching - Immediate Action Required'

Given Credentialing_Case__c.Status__c transitions back to 'Active' (reinstatement)
When the status change is saved
Then all open PCN__c records for the provider with Status__c = 'Pending Distribution' are updated to Status__c = 'Voided' and PCN__c.Void_Reason__c = 'Provider Reinstated'

Given a PCN is generated
When the PCN record is viewed
Then a read-only audit panel shows: created by (System), creation trigger (Credentialing Status Change), linked Credentialing_Case__c, and all email send events logged in PCN_Notification_Log__c child records

## Solution

Implement a Record-Triggered Flow (PCN_Generation_Flow) on Credentialing_Case__c firing on Update when Status__c changes to 'Denied', 'Suspended', or 'Active' (for reinstatement). A Loop element iterates over related PNA records retrieved by a Get Records element. Create Records element generates PCN__c records in bulk within the loop (bulkified via collection variable and single Create Records outside loop). Email distribution uses a Send Email action referencing the 'Provider_Change_Notification_Email' template stored in Classic Email Templates (accessible via Flow). The escalation job is a Scheduled Flow running daily at 7 AM CST querying PCN__c records with Notification_Deadline__c = TODAY() + 5 AND Status__c = 'Pending Distribution'. PCN_Notification_Log__c child records are created via a separate Platform Event (PCN_Email_Sent__e) published by the email send action and consumed by a trigger that writes the log record. A Permission Set Group ('PCN_Manager_PSG') grants Compliance Officers full CRUD on PCN__c.

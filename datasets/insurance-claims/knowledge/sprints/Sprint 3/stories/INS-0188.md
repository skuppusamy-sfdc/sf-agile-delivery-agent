## Description

As a Provider Network Specialist, I need a Provider_Network_Agreement__c (PNA) record to be automatically created and routed for DocuSign execution when a Credentialing_Case__c reaches Status__c = 'Credentialing Complete', so that the contracting step immediately follows credentialing approval without manual handoff delays and providers can begin seeing patients under the network faster.

## Acceptance Criteria

Given a Credentialing_Case__c record transitions to Status__c = 'Credentialing Complete',
When the status update is saved,
Then a new Provider_Network_Agreement__c (PNA) record is automatically created with PNA.Provider__c = Credentialing_Case__c.Provider__c, PNA.Network__c populated from Credentialing_Case__c.Network__c, PNA.LOB__c from Credentialing_Case__c.LOB__c, PNA.Status__c = 'Draft', and PNA.Effective_Date__c = first day of the following month.

Given the PNA record is created,
When the record creation completes,
Then a DocuSign envelope is triggered using the standard PNA agreement template (stored in DocuSign Template Id Custom Setting PNA_Template_Id__c), sent to both the provider's email (Provider__c.PersonEmail) and the Provider Network Specialist's email.

Given a PNA already exists for the same Provider__c and Network__c combination with PNA.Status__c not equal to 'Terminated',
When the Credentialing_Case__c reaches 'Credentialing Complete',
Then the system does NOT create a duplicate PNA, instead links the existing PNA to the Credentialing_Case__c via Credentialing_Case__c.Associated_PNA__c lookup and posts a Chatter message: 'Active PNA already exists – linked to this credentialing case.'

Given the PNA DocuSign envelope is completed by all signatories,
When the DocuSign callback updates the envelope status,
Then PNA.Status__c is updated to 'Executed', PNA.Executed_Date__c is stamped, and Provider__c.Network_Participation_Status__c is set to 'Active In-Network'.

-Given credentialing completion also triggers a Fee_Schedule__c assignment- (descoped — fee schedule auto-assignment moved to INS-0193)

## Solution

Record-Triggered Flow on Credentialing_Case__c fires After Save when Status__c changes to 'Credentialing Complete'. A Decision element checks for an existing active PNA via a Get Records node querying Provider_Network_Agreement__c by Provider__c and Network__c. If no active PNA exists, a Create Records element creates the PNA. An Apex Action (DocuSignEnvelopeService.cls) is called to trigger the DocuSign envelope using the template ID from Custom Settings. DocuSign callback handling uses an existing Platform Event listener (DocuSign_Status_Event__e) that routes completed-envelope events to a handler updating PNA.Status__c and Provider__c.Network_Participation_Status__c. Duplicate check logic is enforced both in the Flow and via a Duplicate Rule on Provider_Network_Agreement__c as a safety net.

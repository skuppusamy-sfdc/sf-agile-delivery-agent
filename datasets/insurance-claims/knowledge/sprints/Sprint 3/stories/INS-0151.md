## Description

As a Credentialing Specialist, I need a Credentialing_Case__c record to be automatically created whenever a Provider__c (Person Account) is submitted for network participation so that the credentialing workflow is initiated without manual intervention and no applications fall through the cracks.

## Acceptance Criteria

Given a Provider__c record has Network_Participation_Status__c set to 'Pending Credentialing' and a valid NPI is present
When the record is saved
Then a Credentialing_Case__c record is created with Credentialing_Case__c.Status__c = 'Application Received', Credentialing_Case__c.Provider__c = Provider__c.Id, and Credentialing_Case__c.Assigned_Specialist__c populated via round-robin assignment

Given a Credentialing_Case__c is auto-created
When the record is inserted
Then a Task is created with Subject = 'Initial Credentialing Review' and Due Date = TODAY() + 5 business days, assigned to the Credentialing_Case__c.Assigned_Specialist__c

Given a Provider__c record is submitted without a valid 10-digit NPI
When the save is attempted
Then an inline validation error is displayed: 'A valid 10-digit NPI is required before submitting for credentialing' and no Credentialing_Case__c is created

Given a Provider__c already has an open Credentialing_Case__c with Status__c != 'Closed' or 'Denied'
When Network_Participation_Status__c is set to 'Pending Credentialing' again
Then no duplicate Credentialing_Case__c is created and a warning toast is shown: 'An active credentialing case already exists for this provider'

-Given the provider is of type 'Facility', create a Credentialing_Case__c against Facility__c instead- (superseded by INS-0155 which handles Facility credentialing separately)

## Solution

Implement a Record-Triggered Flow on Provider__c (Person Account) firing After Save on Create and Update where Network_Participation_Status__c changes to 'Pending Credentialing'. Flow includes a Get Records element to check for existing open Credentialing_Case__c records. If none found, a Create Records element instantiates Credentialing_Case__c. Round-robin assignment handled via a custom Apex invocable method (CredentialingAssignmentService.assignSpecialist()). Validation for NPI format enforced via a separate Before Save Flow using a regex formula: NOT(REGEX(NPI__c, '[0-9]{10}')). Platform Event (Credentialing_Initiated__e) published post-creation to notify downstream CAQH integration listener.

## Description

As a Broker Relations Coordinator, I need a guided broker onboarding OmniScript that captures agency details, license information, and appointment agreements, and routes the completed application through DocuSign for e-signature so that broker recruitment cycles are shortened and paper-based appointment packets are eliminated.

## Acceptance Criteria

Given a Broker Relations Coordinator initiates a new broker onboarding application,
When the OmniScript Step 1 (Agency Information) loads,
Then the following fields must be required: Agency_Name__c, TIN__c (validated against 9-digit format), NPI__c (10-digit numeric validation), Primary_State_of_Operation__c, and Agency_Type__c (picklist: 'Independent', 'Captive', 'MGA', 'TPA').

Given TIN__c and NPI__c are entered,
When the user advances to Step 2,
Then a real-time duplicate check must query existing Account records where Account.TIN__c = entered TIN or Account.NPI__c = entered NPI; if a match is found, display 'Duplicate broker record detected — ID: [Account ID]' and block progression.

Given Step 2 (License Details) is active,
When the broker adds state license entries,
Then each entry must capture License_State__c, License_Number__c, License_Type__c, and License_Expiration__c; License_Expiration__c must be a future date or display error 'Expired license cannot be submitted'.

Given all steps are complete,
When the coordinator submits the application,
Then an Account record (RecordType = 'Broker_Agency') and a related Broker_Application__c record must be created with Broker_Application__c.Status__c = 'Pending Signature'.

Given the records are created,
When the DocuSign envelope is triggered,
Then the Broker Appointment Agreement template must be sent to the email captured in Step 1 with a 10-business-day expiration on the envelope.

Given DocuSign returns a 'Completed' webhook event,
When the Platform Event DocuSign_Status__e is received,
Then Broker_Application__c.Status__c must update to 'Executed' and Account.Broker_Status__c must update to 'Active — Pending Credentialing'.

-Given the application is submitted, auto-assign to the regional Broker Relations Coordinator queue based on Primary_State_of_Operation__c- (deprecated: manual assignment retained for MVP; routing logic deferred to Sprint 8)

## Solution

Build OmniScript 'Broker_Onboarding_OS' with 5 steps. Use DataRaptor Transform for TIN/NPI duplicate check via SOQL. Integration Procedure orchestrates: (1) Account + Broker_Application__c creation via DataRaptor Load, (2) DocuSign envelope trigger via DocuSign for Salesforce REST API using named credential. A Platform Event subscriber (Record-Triggered Flow on DocuSign_Status__e) updates Broker_Application__c.Status__c and Account.Broker_Status__c on completion callback. Permission Set Group 'Broker_Recruitment_Ops' gates access. Deploy OmniStudio artifacts via version activation.

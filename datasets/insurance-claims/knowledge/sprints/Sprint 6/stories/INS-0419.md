## Description

As a Broker Relations Coordinator, I need a multi-step OmniScript broker onboarding application that captures producer license details, validates the National Producer Number (NPN) format, collects Errors & Omissions (E&O) insurance information, and routes the completed application to a Credentialing Specialist for approval so that broker recruitment is standardized and auditable within Salesforce.

## Acceptance Criteria

Given a prospective broker accesses the public-facing onboarding portal page,
When the OmniScript loads,
Then step 1 collects: First Name, Last Name, Email, Phone, Business Entity Name, NPN, and State of Domicile — all fields marked required.

Given the broker enters their NPN on step 1,
When the broker advances to step 2,
Then the OmniScript performs a real-time NPN format validation: must be 1–10 numeric digits; if invalid, an inline error message 'NPN must be a numeric value between 1 and 10 digits' is displayed and the broker cannot advance.

Given NPN format validation passes,
When step 2 loads,
Then the broker provides E&O insurance details: Carrier Name, Policy Number, Coverage Amount (minimum $1,000,000 enforced), and Expiration Date (must be > TODAY()).

Given the broker completes all steps and submits,
When the OmniScript DataRaptor Load action fires,
Then a new Account record is created with RecordType = 'Broker_Agency', a related Contact with Contact.NPN__c populated, and a Credentialing_Case__c record with Credentialing_Case__c.Status__c = 'Application Received', Credentialing_Case__c.Type__c = 'Broker License Verification', linked to the new Contact.

Given the Credentialing_Case__c is created,
When the record is inserted,
Then a Record-Triggered Flow assigns the case to the 'Broker_Credentialing_Queue' and sends a confirmation email to the applicant's submitted email address using an Email Alert.

-Given the broker application is submitted, When the application is complete, Then a DocuSign envelope for the Broker Agreement is auto-sent immediately.- (Deferred: DocuSign envelope trigger moved to post-approval step per legal team requirement — tracked in INS-0423.)

Given a duplicate NPN already exists on an existing Contact record,
When the OmniScript attempts to create the Contact,
Then the DataRaptor Load is blocked, an error message 'An active broker record with this NPN already exists. Please contact Broker Relations.' is displayed, and no records are created.

## Solution

Build OmniScript 'Broker_Onboarding_Application_OS' with 5 steps: Personal Info, NPN Validation, E&O Details, Document Upload, Review & Submit. NPN format validation implemented as an OmniScript Formula element with regex pattern '^[0-9]{1,10}$'. Duplicate NPN check via DataRaptor Extract querying Contact.NPN__c before DataRaptor Load. DataRaptor Load 'DR_Create_Broker_Account_Contact_Case' creates Account, Contact, and Credentialing_Case__c in a single transaction using multi-object load. Record-Triggered Flow 'Broker_Case_Assignment_Flow' handles queue assignment and email alert on Credentialing_Case__c insert. OmniScript hosted on a public Experience Cloud page with Guest User Profile restricted to OmniScript execution only.

## Description

As a Broker Relations Coordinator, I need brokers to submit a First Notice of Loss (FNOL) directly from the Agent Portal via a guided OmniScript so that claims intake is accelerated, manual data entry errors are reduced, and the Claims Operations team receives a pre-validated Claim__c record without re-keying policy data.

## Acceptance Criteria

Given a broker selects 'Submit New Claim' from the portal navigation,
When the FNOL OmniScript launches,
Then Step 1 presents a policy search by Policy__c.Policy_Number__c or insured name, and auto-populates Claim__c.Policy__c, Claim__c.Insured_Name__c, Claim__c.Effective_Date__c, and Claim__c.Line_of_Business__c from the matched Policy__c record

Given the broker searches for a policy,
When no matching Policy__c record is found with Status__c = 'Active',
Then an inline error message reads 'Policy not found or is not currently active. Please verify the policy number or contact your underwriter.' and submission is blocked

Given a valid policy is selected,
When the broker reaches Step 2 (Loss Details),
Then the following fields are required: Claim__c.Date_of_Loss__c, Claim__c.Loss_Type__c (picklist), Claim__c.Loss_Description__c (min 25 characters), and Claim__c.Reported_By__c defaults to the logged-in broker's Contact record

Given the broker completes all steps,
When they click 'Submit FNOL',
Then a new Claim__c record is created with Claim__c.Status__c = 'FNOL Submitted', Claim__c.FNOL_Source__c = 'Broker Portal', and Claim__c.Submission_Timestamp__c = DateTime.now()

Given the FNOL is submitted successfully,
When the record is created,
Then a confirmation screen displays the generated Claim__c.Claim_Number__c and a DocuSign envelope is sent to the insured's email on file for the Loss Acknowledgement form

-Given the broker can attach supporting documents during FNOL submission up to 10 files- (deprecated — file attachment moved to post-FNOL document portal per compliance review)

Given the submitted Claim__c.Loss_Type__c = 'Fraud Indicator',
When the record is saved,
Then a Case is automatically created and assigned to the SIU queue with Case.Subject = 'SIU Review - ' + Claim__c.Claim_Number__c and Case.Priority = 'High'

## Solution

Develop a 6-step OmniScript 'BrokerFNOL_OS' in OmniStudio. Step 1 uses a DataRaptor Extract (DR_PolicyLookup) with SOQL on Policy__c filtered by Status__c = 'Active'. Step 2-4 capture loss details, claimant info, and coverage verification using conditional step visibility. On submit, a DataRaptor Transform maps OmniScript input to Claim__c field API names, then an Integration Procedure (IP_CreateFNOLClaim) calls a custom Apex invocable action 'FNOLClaimCreator' that performs DML and triggers a DocuSign envelope via the DocuSign for Salesforce API. SIU routing handled by a separate Record-Triggered Flow on Claim__c filtering on Loss_Type__c = 'Fraud Indicator' that creates a Case and assigns to SIU Queue via Assignment Rule. OmniScript deployed to Experience Cloud Agent Portal with Guest/Broker profile access gated by Permission Set 'Broker_FNOL_Submit'.

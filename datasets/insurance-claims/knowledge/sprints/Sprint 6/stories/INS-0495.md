## Description

As a Claims Operations Manager, I need the system to automatically generate an Electronic Remittance Advice (ERA) file in ANSI X12 835 format when a Claim__c is adjudicated and deliver it to the configured TPA via secure SFTP so that downstream payment reconciliation is automated and compliant with EDI standards. This story extends the payment disbursement framework established in INS-0477.

## Acceptance Criteria

Given a Claim__c.Status__c transitions to 'Adjudicated',
When the Record-Triggered Flow fires on Claim__c update,
Then a Platform Event (ERA_Generation_Event__e) is published with fields: Claim_Id__c, TPA_Id__c, Adjudication_Date__c, Total_Allowed_Amount__c.

Given the ERA_Generation_Event__e is consumed by the Apex subscriber,
When the ERA file is assembled,
Then all Claim_Line__c records (Claim_Line__c.Claim__c = Claim__c.Id) are included with Claim_Line__c.CPT_Code__c, Claim_Line__c.Billed_Amount__c, Claim_Line__c.Allowed_Amount__c, Claim_Line__c.Adjustment_Reason_Code__c, and Claim_Line__c.Payment_Amount__c mapped to the 835 loop/segment structure.

Given the ERA file is generated,
When the SFTP delivery is attempted,
Then the file is deposited to the TPA's configured SFTP endpoint stored in Provider__c.TPA_SFTP_Endpoint__c with filename format 'ERA_{Claim_Number}_{YYYYMMDD}.835'.

Given the SFTP delivery succeeds,
When the transfer is confirmed,
Then Payment__c.ERA_Sent__c = true, Payment__c.ERA_Sent_Date__c = today, and Payment__c.ERA_File_Name__c is populated.

Given the SFTP delivery fails after 3 retry attempts,
When all retries are exhausted,
Then a Case is auto-created with Subject = 'ERA Delivery Failure - [Claim Number]', Case.Priority = 'High', Case.Queue = 'Claims_EDI_Exceptions', and Payment__c.ERA_Status__c = 'Delivery Failed'.

Given an ERA is regenerated due to claim adjustment,
When a corrected ERA is sent,
Then the 835 file includes Transaction Set Purpose Code '18' (Reissue) and the prior ERA file name is stored in Payment__c.Prior_ERA_File_Name__c.

## Solution

Record-Triggered Flow on Claim__c (after update, Status__c changed to 'Adjudicated') publishes ERA_Generation_Event__e Platform Event. Apex Platform Event trigger (ERAGenerationEventHandler) subscribes and invokes ERAFileBuilderService class to assemble X12 835 string from Claim__c and Claim_Line__c data. SFTP delivery via Named Credential + Apex HttpRequest callout to MuleSoft middleware (SFTP Proxy API). Retry logic implemented with Queueable Apex chain (max 3 retries, exponential backoff). Case creation on failure via Apex CaseFactory utility class. All ERA file metadata written back to Payment__c via DML in the Queueable. Custom metadata type (ERA_Config__mdt) stores TPA-specific 835 segment mappings.

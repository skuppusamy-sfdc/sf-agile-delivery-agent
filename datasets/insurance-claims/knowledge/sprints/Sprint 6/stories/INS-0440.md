## Description

**Bug Report**

**Summary:** When a Payment__c record tied to a Claim__c is voided (Payment__c.Status__c set to 'Voided') while the parent Claim__c is still in 'In Adjudication' status, the Record-Triggered Flow 'RTF_UpdateClaimOnPaymentChange' incorrectly updates Claim__c.Status__c to 'Closed' instead of reverting to 'In Adjudication'.

**Steps to Reproduce:**
1. Open any Claim__c record with Claim__c.Status__c = 'In Adjudication'.
2. Navigate to the related Payment__c record and set Payment__c.Status__c = 'Issued'.
3. Observe Claim__c.Status__c updates to 'Payment Issued' (correct).
4. Return to the Payment__c record and set Payment__c.Status__c = 'Voided'.
5. Observe Claim__c.Status__c — it incorrectly reads 'Closed' instead of 'In Adjudication'.

**Expected Behavior:** When Payment__c.Status__c transitions to 'Voided', Claim__c.Status__c should revert to 'In Adjudication' and Claim__c.Void_Reason__c should be populated from Payment__c.Void_Reason__c.

**Actual Behavior:** Claim__c.Status__c is set to 'Closed' and Claim__c.Void_Reason__c remains null.

**Environment:** Acme Insurance Full Sandbox (ACME_FS1)
**Frequency:** 100% reproducible
**Impact:** High — Claims Operations team is manually reopening claims; affects downstream EOB generation.

## Acceptance Criteria

As per Description.

Given the fix is deployed,
When Payment__c.Status__c is updated to 'Voided' and the parent Claim__c.Status__c = 'Payment Issued',
Then Claim__c.Status__c must revert to 'In Adjudication' and Claim__c.Void_Reason__c must be populated from Payment__c.Void_Reason__c.

Given the fix is deployed,
When Payment__c.Status__c is updated to 'Voided' and the parent Claim__c has Claim__c.Status__c = 'Closed' (legitimately closed prior to this payment),
Then Claim__c.Status__c must NOT be changed.

## Solution

Update Record-Triggered Flow 'RTF_UpdateClaimOnPaymentChange' — add a decision node before the 'Set Claim Closed' assignment element to check if {!$Record.Status__c} = 'Voided'. Within that branch, add a secondary condition checking the prior Claim__c.Status__c value using a Get Records element. If prior status was 'Payment Issued', update Claim__c.Status__c = 'In Adjudication' and map Claim__c.Void_Reason__c = Payment__c.Void_Reason__c. Add a Flow fault path with error logging to a custom ClaimFlowError__c object. Deploy unit test updates to PaymentTriggerHandler_Test covering void scenario with 95%+ coverage.

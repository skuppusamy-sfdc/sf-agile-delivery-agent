## Description

As a Broker Relations Coordinator, I need a consolidated dashboard FlexCard on the Agent/Broker Portal that surfaces active Policy__c records, open Claim__c counts, and pending EOB statuses for all policyholders within my book of business, so that I can quickly triage client needs without navigating multiple Salesforce objects manually.

## Acceptance Criteria

Given a licensed broker user logs into the Agent/Broker Portal,
When the dashboard loads,
Then a FlexCard titled 'My Book of Business' must render within 3 seconds displaying all Policy__c records where Policy__c.Broker_Agent__c = current user's linked Contact ID and Policy__c.Status__c = 'Active'.

Given the broker dashboard FlexCard is rendered,
When the broker views a policy row,
Then each row must display: Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, and a real-time count of child Claim__c records where Claim__c.Status__c IN ('Open','In Review','Pending Documentation').

Given a Claim__c record linked to a policy in the broker's book of business has Status__c = 'Adjudicated',
When the broker views the claims summary panel,
Then the associated Explanation_of_Benefits__c.EOB_Status__c value must be displayed as a color-coded badge: 'Issued' = green, 'Pending' = yellow, 'Disputed' = red.

Given the broker clicks a Policy__c row on the FlexCard,
When the drill-down action is triggered,
Then an OmniScript modal opens displaying the Policyholder 360 view including Policy__c, active Claim__c lines, and most recent Payment__c records.

-Given the broker can export the dashboard to PDF via Conga CLM integration- (deprecated — moved to Phase 2, see INS-0451)

Given a broker has zero active policies,
When the FlexCard loads,
Then an empty-state illustration and message 'No active policies found for your book of business' must be displayed instead of a blank panel.

## Solution

Build OmniStudio FlexCard 'BrokerBookOfBusiness_FC' using DataRaptor Extract to query Policy__c with a SOQL filter on Broker_Agent__c = :userId and Status__c = 'Active'. Child Claim__c counts fetched via a secondary DataRaptor with aggregate SOQL (COUNT). EOB badge logic implemented via FlexCard State configuration with conditional CSS classes. Drill-down triggers OmniScript 'PolicyholderMiniView_OS' via FlexCard Action. Apply Permission Set Group 'Broker_Portal_PSG' to restrict data visibility. Cache DataRaptor results with a 60-second TTL to meet the 3-second render SLA.

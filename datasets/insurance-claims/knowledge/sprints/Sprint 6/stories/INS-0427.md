## Description

As a Provider Relations Specialist, I need an automated Provider Change Notification (PCN) workflow that creates a Provider_Change_Notification__c record and routes it for review whenever a provider's TIN, primary address, or NPI is modified on their Provider__c (Person Account) record, so that downstream PNA, Fee_Schedule__c, and Payment__c records can be audited for impact and regulatory change notification requirements are met within the required 30-day window.

## Acceptance Criteria

Given a Provider Relations Specialist or integrated data feed modifies Provider__c.TIN__c, Provider__c.BillingStreet, Provider__c.BillingCity, Provider__c.BillingState, Provider__c.BillingPostalCode, or Provider__c.NPI__c on any Provider__c record,
When the record is saved,
Then a Provider_Change_Notification__c record is automatically created with: PCN.Provider__c = modified Provider__c, PCN.Change_Type__c populated (e.g., 'TIN Change', 'Address Change', 'NPI Change'), PCN.Old_Value__c and PCN.New_Value__c capturing before/after field values, PCN.Status__c = 'Pending Review', and PCN.Notification_Due_Date__c = TODAY() + 30.

Given a PCN record is created with PCN.Change_Type__c = 'TIN Change',
When the record is saved,
Then the PCN is automatically assigned to the 'Provider_TIN_Review' queue and PCN.Priority__c = 'High' is set.

Given a PCN record is created with PCN.Change_Type__c = 'Address Change',
When the record is saved,
Then the PCN is assigned to the owning Provider Relations Specialist's queue and PCN.Priority__c = 'Medium'.

Given an active Provider_Network_Agreement__c exists for the modified provider,
When the PCN is created,
Then a PCN_PNA_Junction__c record is created linking the PCN to each active PNA for impact tracking, and a Task is created on each PNA record with Subject = 'Review PNA for PCN Impact — {PCN.Name}'.

Given a PCN record reaches PCN.Notification_Due_Date__c without PCN.Status__c = 'Completed',
When the daily scheduled flow runs,
Then PCN.Status__c is updated to 'Overdue', a Platform Event 'PCN_Overdue__e' is published, and an escalation email is sent to the Compliance Officer.

Given the Provider Relations Specialist resolves the PCN,
When PCN.Status__c is set to 'Completed',
Then PCN.Completed_Date__c is stamped and a CAQH re-attestation check is triggered via the CAQH integration if PCN.Change_Type__c = 'TIN Change'.

## Solution

Implement an Apex Trigger (ProviderChangeTrigger.cls) on Account object (before update) to capture old/new field values for TIN__c, Billing address fields, and NPI__c using Trigger.oldMap and Trigger.newMap. Trigger calls ProviderChangeNotificationService.cls to insert Provider_Change_Notification__c records and related PCN_PNA_Junction__c records. Assignment routing handled by a Record-Triggered Flow on Provider_Change_Notification__c (after insert). A Scheduled Flow ('PCN_Overdue_Escalation_Flow') runs daily at 06:00 UTC to identify overdue PCNs and publish PCN_Overdue__e Platform Events. CAQH integration via Named Credential callout in CAQHIntegrationService.cls. Custom Metadata Type 'PCN_Routing_Config__mdt' drives queue assignment logic.

## Description

**Bug Report — relates to INS-0431**

**Environment:** Acme Insurance Full Sandbox (FS1)
**Build:** Sprint 6 Deploy Tag v6.2.1

**Steps to Reproduce:**
1. Log in as a Provider Network Specialist user.
2. Navigate to any Provider__c (Person Account) record that has at least one Provider_Network_Agreement__c record with PNA__c.Status__c = 'Pending Renewal'.
3. Observe the Provider 360 FlexCard 'Provider360_FC' on the record page, specifically the PNA status badge in the Network Agreements panel.

**Expected Behavior:** PNA records with PNA__c.Status__c = 'Pending Renewal' should render with an **amber/yellow** status badge, consistent with the design specification in the FlexCard State configuration (State name: 'PNA_PendingRenewal', CSS class: 'slds-badge_warning').

**Actual Behavior:** PNA records with PNA__c.Status__c = 'Pending Renewal' are rendering with a **green** status badge — the same badge style used for PNA__c.Status__c = 'Active'. This is caused by the FlexCard State condition evaluating 'Pending Renewal' as truthy for the 'Active' state due to a missing exact-match operator (currently using 'contains' instead of 'equals' on the Status__c field comparison).

**Impact:** Provider Network Specialists may incorrectly interpret a 'Pending Renewal' PNA as fully Active, potentially approving claims against agreements that are due to lapse. Compliance risk flagged by Compliance Officer (Ryan Mitchell).

**Screenshot:** Attached to Jira ticket (sandbox_pna_badge_bug.png)

## Acceptance Criteria

As per Description. FlexCard State condition for 'PNA_Active' state must use operator 'equals' (not 'contains') on PNA__c.Status__c. 'Pending Renewal' must render amber badge (slds-badge_warning). Regression test must confirm 'Active' = green, 'Pending Renewal' = amber, 'Terminated' = red, 'Expired' = grey across all PNA status values defined in the PNA__c.Status__c picklist.

## Solution

Update FlexCard 'Provider360_FC' State configuration: change the 'PNA_Active' state condition from operator 'contains' to 'equals' for the PNA__c.Status__c field comparison value 'Active'. Add explicit FlexCard State 'PNA_PendingRenewal' with condition PNA__c.Status__c equals 'Pending Renewal' and assign CSS class 'slds-badge_warning'. Validate all five PNA status picklist values ('Active', 'Pending Renewal', 'Terminated', 'Expired', 'Draft') have corresponding FlexCard States. Deploy updated FlexCard to sandbox for QA regression, then promote to production via Change Set. No Apex changes required.

## Description

As a Broker Relations Coordinator, I need a payment status tracker widget on the Agent Portal that shows the current disbursement status of Payment__c records linked to my clients' Claim__c records, so that I can proactively update policyholders on reimbursement timelines without calling the Claims Operations team.

## Acceptance Criteria

Given a broker views the Agent Portal Claims detail panel for a specific Claim__c,
When one or more Payment__c records exist with Payment__c.Claim__c = Claim__c.Id,
Then the payment tracker must display each Payment__c record showing: Payment__c.Payment_Amount__c, Payment__c.Payment_Date__c, Payment__c.Payment_Method__c, Payment__c.Status__c, and Payment__c.Check_EFT_Number__c (masked to last 4 digits for security).

Given a Payment__c.Status__c transitions from 'Approved' to 'Disbursed',
When the broker's portal dashboard is active or refreshed,
Then the payment status badge must update in real-time (or within one polling cycle of ≤ 60 seconds) without requiring a full page reload.

Given a Payment__c.Status__c = 'Rejected' or 'Returned',
When the broker views the payment tracker,
Then a red alert banner must display with Payment__c.Rejection_Reason__c text and a 'Contact Claims Team' CTA button that pre-populates a Case record linked to the Claim__c.

Given a Claim__c has no associated Payment__c records,
When the payment tracker section renders,
Then display 'No disbursements processed yet. Claim is currently in [Claim__c.Status__c] status.' with the current status value interpolated.

-Given the broker can initiate a payment reversal from the portal- (deprecated — payment reversal is a Claims Operations Manager function only, removed from broker portal scope per compliance review)

## Solution

Extend existing OmniStudio FlexCard 'BrokerClaimsDetail_FC' with a new child card panel for Payment__c records using a DataRaptor Extract filtered by Claim__c.Id. Implement LWC polling component 'paymentStatusPoller' using setInterval (60s) to refresh Payment__c Status__c via a wire adapter to a custom Apex controller 'PaymentStatusController'. Mask Check_EFT_Number__c using an Apex formula helper. Rejection banner uses FlexCard conditional state. 'Contact Claims Team' CTA invokes a Record-Triggered-compatible Screen Flow 'CreateClaimsEnquiryCase_SF' pre-populating Case.RelatedClaim__c. Field visibility controlled by FLS on the Broker_Portal_PSG Permission Set Group.

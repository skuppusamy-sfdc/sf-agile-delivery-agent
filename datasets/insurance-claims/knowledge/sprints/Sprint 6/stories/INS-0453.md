## Description

As a Broker Relations Coordinator, I need a consolidated dashboard FlexCard on the Agent/Broker Portal that surfaces real-time counts of active policies and open claims segmented by Line of Business (LOB), so that brokers can quickly assess their book of business without navigating multiple Salesforce record pages.

## Acceptance Criteria

Given a logged-in broker user with the 'Broker Community' Permission Set Group applied,
When the broker lands on the Agent Portal home page,
Then a FlexCard titled 'My Book of Business' is displayed showing Policy__c counts where Policy__c.Status__c = 'Active' grouped by Policy__c.Line_of_Business__c.

Given the FlexCard has loaded,
When the broker views the Claims section,
Then Claim__c records where Claim__c.Assigned_Broker__c = current user's Contact Id and Claim__c.Status__c NOT IN ('Closed','Withdrawn') are counted and displayed per LOB.

Given a broker has zero open claims for a given LOB,
When the FlexCard renders that LOB row,
Then the claim count displays '0' rather than omitting the row entirely.

Given the FlexCard data,
When counts are rendered,
Then data is sourced via a DataRaptor Extract pulling from Policy__c and Claim__c with a refresh interval no greater than 5 minutes using a Platform Cache partition.

-Given broker clicks a LOB row, a modal showing individual policy numbers shall appear- (superseded by INS-0461 drill-down story)

Given any broker user on a mobile device,
When the FlexCard is viewed at viewport width < 768px,
Then the card stacks LOB rows vertically and remains fully readable without horizontal scroll.

## Solution

Build an OmniStudio FlexCard ('BrokerBookOfBusiness_FC') with two DataRaptor Extract actions: one for Policy__c aggregation by Line_of_Business__c filtered on Status__c = 'Active', and one for Claim__c open counts. Use a Flyout child card for future drill-down (stubbed). Apply Platform Cache (session partition) with 300-second TTL to reduce SOQL queries on high-traffic portal pages. FlexCard exposed via Experience Cloud page assigned to the 'Broker Community' profile. Permission Set Group 'Broker_Portal_Access' gates visibility via a Conditional View state.

## Description

As a Credentialing Specialist, I need the system to automatically sync provider demographic and licensure data from CAQH ProView into the corresponding Provider__c and Credentialing_Case__c records on a nightly basis so that credentialing data remains current without manual re-entry, reducing the risk of processing claims against providers with lapsed credentials.

## Acceptance Criteria

Given the nightly CAQH sync batch job 'CAQHProviderSyncBatch' executes,
When the job queries CAQH ProView REST API for all providers where Provider__c.CAQH_Provider_ID__c is populated,
Then the following Provider__c fields are updated from the CAQH response payload:
  - Provider__c.License_Number__c
  - Provider__c.License_State__c
  - Provider__c.License_Expiration_Date__c
  - Provider__c.Board_Certification__c
  - Provider__c.CAQH_Last_Sync_Date__c = DateTime.now()

Given the CAQH response indicates a provider's license has expired (License_Expiration_Date__c < today),
When the sync updates the Provider__c record,
Then the associated Credentialing_Case__c.Status__c is set to 'Requires Review' and a Task is created assigned to the Credentialing Specialist queue with Subject = 'Expired License Detected - ' + Provider__c.Name

Given the CAQH API returns a 404 for a given CAQH_Provider_ID__c,
When the sync processes that provider record,
Then Provider__c.CAQH_Sync_Status__c = 'ID Not Found', the record is skipped without error, and the provider's Id is logged in a custom object CAQHSyncError__c with fields: Provider__c (lookup), Error_Code__c = '404', Sync_Date__c = today

Given the batch job completes,
When all records have been processed,
Then a summary email is sent to the Credentialing Operations distribution list containing: total records processed, total updates applied, total errors logged, and a link to the CAQHSyncError__c list view

-Given sync runs twice daily at 6 AM and 6 PM CT- (deprecated — reduced to nightly at 2 AM CT per CAQH API rate limit agreement)

## Solution

Build Apex Batch class 'CAQHProviderSyncBatch' implementing Database.Batchable, Database.AllowsCallouts, and Schedulable. The start() method queries Provider__c WHERE CAQH_Provider_ID__c != null. Each execute() chunk (size 10 to respect CAQH API rate limits) calls the CAQH ProView REST API via a Named Credential 'CAQH_ProView_NC' using HttpRequest. Parse JSON response with a custom wrapper class 'CAQHProviderWrapper'. Update Provider__c fields in bulk DML. License expiration logic triggers a sub-flow 'HandleExpiredLicense_Flow' via Apex-invoked Flow. Errors written to CAQHSyncError__c via a separate DML list. Completion email sent using Messaging.sendEmail() in the finish() method with a dynamically built HTML body. Schedule via cron '0 0 2 * * ?' in a Schedulable wrapper. Store API credentials securely in Named Credential with OAuth 2.0 client credentials flow.

## Description

As a Claims Operations Manager, I need an Agentforce-powered triage recommendation panel on the Claim__c record page that analyzes incoming FNOL claims and surfaces an AI-generated complexity score, recommended routing (standard adjudication vs. SIU referral vs. complex review), and supporting rationale based on historical Claim__c data patterns, so that my team can prioritize their review queue more efficiently and reduce misrouted claims.

## Acceptance Criteria

Given a Claim__c record with Status__c = 'FNOL Received' is opened by a Claims Operations Manager,
When the Agentforce Triage Panel LWC loads on the record page,
Then the panel displays an AI-generated Triage_Complexity_Score__c (1–10 scale), a Recommended_Routing__c value IN ('Standard Adjudication', 'SIU Referral', 'Complex Review'), and a plain-language rationale summary (max 200 characters) sourced from the Agentforce response.

Given the Agentforce recommendation is displayed,
When the Claims Operations Manager clicks 'Accept Recommendation',
Then Claim__c.Routing_Decision__c = Recommended_Routing__c, Claim__c.Triage_Completed_By__c = current user, Claim__c.Triage_Timestamp__c = NOW(), and a Claim__c chatter post logs the accepted AI recommendation with the rationale.

Given the Claims Operations Manager disagrees with the AI recommendation,
When they click 'Override',
Then a modal prompts for Override_Reason__c (required picklist: 'Insufficient Context', 'Policy Exception', 'Regulatory Requirement', 'Other') and the Claim__c.Routing_Decision__c is set to the manager's manual selection, with Claim__c.AI_Override__c = TRUE for reporting.

Given a Claim__c where Claim__c.LOB__c = 'Workers Compensation' and Claim__c.Total_Billed_Amount__c > 50000,
When the Agentforce model evaluates the claim,
Then the Triage_Complexity_Score__c must be >= 7 and Recommended_Routing__c must be 'Complex Review' (validated against model output in UAT with synthetic test claims).

Given the Agentforce API call times out or returns an error,
When the triage panel loads,
Then the panel displays 'AI Triage Unavailable — Please Route Manually' and logs the failure to Integration_Log__c with Integration_Log__c.Source__c = 'Agentforce', Integration_Log__c.Error_Code__c, and Integration_Log__c.Claim__c = Claim__c.Id.

Given AI_Override__c = TRUE across claims in a reporting period,
When a Claims Operations Manager runs the 'AI Triage Override Report',
Then the report displays override rate by Override_Reason__c and LOB, sourced from Claim__c object with filters on Triage_Timestamp__c.

## Solution

Build an Agentforce Agent Action ('ClaimTriageRecommendation') backed by an Apex invocable method that constructs a prompt using Claim__c field values (LOB__c, Total_Billed_Amount__c, Loss_Type__c, FNOL_Date__c, Prior_Claims_Count__c) and calls the Salesforce Einstein LLM API via Connect API. Response parsed in Apex to populate Triage_Complexity_Score__c, Recommended_Routing__c, and rationale text. LWC ('agentforceTriagePanel') displays the response with Accept/Override actions wired to an Apex controller. Override modal built as a child LWC with a @wire adapter. Chatter post via ConnectApi.ChatterFeeds. Integration error logging via Apex try-catch writing to Integration_Log__c. Custom Report Type on Claim__c for override analytics dashboard.

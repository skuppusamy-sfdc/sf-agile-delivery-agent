## Description

As a Provider Relations Specialist, I need a single-pane Provider 360 FlexCard on the Provider__c (Person Account) record page that aggregates NPI status from NPPES, credentialing status from Credentialing_Case__c, active PNA agreements, and open PCNs so that I can assess a provider's full network standing without switching between multiple record types or external systems.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record,
When the Provider 360 FlexCard loads,
Then the following data panels are displayed:
  - Panel 1 'Identity': Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Specialty__c, NPPES verification badge (Active/Inactive) sourced from the last NPPES_Sync_Date__c and NPPES_Status__c fields
  - Panel 2 'Credentialing': Most recent Credentialing_Case__c.Status__c and Credentialing_Case__c.Expiration_Date__c linked to this provider
  - Panel 3 'Network Agreements': Count of active Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active' and PNA__c.Provider__c = current Provider__c Id
  - Panel 4 'Open Notifications': Count of Provider_Change_Notification__c records where PCN__c.Status__c != 'Closed'

Given the provider's most recent Credentialing_Case__c.Expiration_Date__c is within 90 days of today,
When the FlexCard renders,
Then Panel 2 displays a yellow warning icon and the label 'Credentialing Expiring Soon'

Given the provider has no active Provider_Network_Agreement__c records,
When the FlexCard renders,
Then Panel 3 displays 'No Active Agreements' with a quick-action button 'Initiate PNA' that launches the PNA creation Flow

Given a Provider Relations Specialist clicks 'View All Agreements' in Panel 3,
When the action fires,
Then a modal renders a related list of PNA records showing PNA__c.Agreement_Type__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, and PNA__c.Fee_Schedule__c

## Solution

Create OmniStudio FlexCard 'Provider360_FC' with four DataRaptor Extract child cards. DR_ProviderIdentity queries Provider__c and related NPPES_Sync__c custom object. DR_ProviderCredentialing queries Credentialing_Case__c ORDER BY CreatedDate DESC LIMIT 1. DR_ProviderPNA aggregates Provider_Network_Agreement__c. DR_ProviderPCN counts open PCN records. Expiration warning logic implemented via a Flyout Action conditional state on date difference formula. 'Initiate PNA' quick action launches a Screen Flow 'InitiatePNA_Flow' via a FlexCard action. Deploy FlexCard to the Provider__c Lightning Record Page via App Builder. Ensure FLS compliance for NPI__c and TIN__c fields using a dedicated Permission Set 'Provider_360_Viewer'.

## Description

As a Claims Operations Manager, I need an Agentforce-powered triage recommendation panel on the Claim__c record page that analyzes incoming FNOL data and suggests an assignment queue, priority level, and SIU referral flag, so that Assignment Coordinators can process higher claim volumes with consistent accuracy and reduced cognitive load.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Status__c = 'FNOL Received',
When the Agentforce triage agent evaluates the record,
Then within 30 seconds a Triage_Recommendation__c child record must be created containing: Triage_Recommendation__c.Suggested_Queue__c, Triage_Recommendation__c.Suggested_Priority__c (High/Medium/Low), Triage_Recommendation__c.SIU_Referral_Flag__c (Boolean), and Triage_Recommendation__c.Confidence_Score__c (0.00–1.00).

Given the Agentforce model evaluates Claim__c.Loss_Type__c = 'Bodily Injury' AND Claim__c.Estimated_Loss_Amount__c > 50000,
When the triage recommendation is generated,
Then Triage_Recommendation__c.Suggested_Priority__c must be 'High' and Triage_Recommendation__c.Suggested_Queue__c must be 'Complex Claims Unit'.

Given the Agentforce model flags Triage_Recommendation__c.SIU_Referral_Flag__c = true,
When the Assignment Coordinator views the Claim__c record page,
Then a prominent orange banner 'SIU Review Recommended — Confidence: [score]%' must appear at the top of the page with a one-click 'Create SIU Referral' action.

Given an Assignment Coordinator overrides the Agentforce recommendation,
When they manually update Claim__c.Assigned_Queue__c or Claim__c.Priority__c,
Then Triage_Recommendation__c.Override_Flag__c must be set to true and Triage_Recommendation__c.Override_Reason__c must be required before saving, for audit trail purposes.

Given the Agentforce triage engine is unavailable (API timeout > 15s),
When the FNOL is received,
Then the system must fall back to a deterministic rule-based Apex class 'ClaimsTriageFallbackService' and set Triage_Recommendation__c.Source__c = 'Fallback Rules Engine' to distinguish from AI-generated recommendations.

Given a Compliance Officer reviews triage decisions,
When they run the 'Triage Audit Report' in ComplianceReporting,
Then all Triage_Recommendation__c records must be filterable by Source__c, Override_Flag__c, SIU_Referral_Flag__c, and CreatedDate range.

## Solution

Configure Agentforce Agent with a custom 'ClaimsTriage' topic and Actions mapped to Apex invocable method 'AgentforceClaimsTriageAction.evaluate()'. The action accepts Claim__c.Id, queries relevant fields, and calls Einstein Generative AI API with a structured prompt template stored in a Custom Metadata Type 'AI_Prompt_Template__mdt'. On response, creates Triage_Recommendation__c via DataWeave transform. Record-Triggered Flow on Claim__c (after-insert, Status__c = 'FNOL Received') invokes the Agentforce action asynchronously via Platform Event 'TriageRequest__e'. Fallback logic in Apex catches CalloutException and routes to 'ClaimsTriageFallbackService'. SIU banner implemented as LWC 'siuReferralBanner' conditioned on Triage_Recommendation__c.SIU_Referral_Flag__c. Override enforcement via Before-Save Record-Triggered Flow validation.

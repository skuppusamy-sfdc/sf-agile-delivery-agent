## Description

As a Provider Network Specialist, I need to associate Fee_Schedule__c records and their child Claim_Line__c-level fee entries to a Provider_Network_Agreement__c (PNA) with CPT code validation so that adjudication uses the correct contracted rate when processing Claim_Line__c records and we avoid overpayment or underpayment errors.

## Acceptance Criteria

Given a Provider Network Specialist opens a PNA record and navigates to the Fee Schedules related list,
When they click 'Add Fee Schedule',
Then an OmniScript 'AddFeeSchedule_OS' launches allowing selection of an existing Fee_Schedule__c where Fee_Schedule__c.LOB__c = PNA.LOB__c and Fee_Schedule__c.Effective_Date__c <= PNA.Effective_Date__c.

Given a Fee_Schedule__c is linked to a PNA,
When the specialist adds a fee line,
Then Fee_Schedule__c.CPT_Code__c must be validated against the CMS CPT code reference list stored in a custom metadata type 'CPT_Code_Reference__mdt' and an error is thrown if the CPT code does not exist: 'CPT Code {value} is not recognized. Please verify against CMS schedule.'

Given a valid CPT code is entered,
When the fee line is saved,
Then Fee_Schedule__c.Contracted_Rate__c must be a positive decimal value and Fee_Schedule__c.Rate_Type__c must be populated with a value from the picklist: 'Per Diem', 'Case Rate', 'Fee for Service', 'Capitation'.

Given a PNA has an associated Fee_Schedule__c,
When a Claim_Line__c is adjudicated where Claim_Line__c.CPT_Code__c matches Fee_Schedule__c.CPT_Code__c and the PNA is active,
Then Claim_Line__c.Contracted_Rate__c is auto-populated from Fee_Schedule__c.Contracted_Rate__c and Claim_Line__c.Fee_Schedule__c lookup is set.

-Given fee schedules support retroactive rate adjustments with back-dated Effective_Date__c (deprecated — retroactive adjustments require Compliance Officer approval workflow, deferred to Phase 2)-

## Solution

Build OmniScript 'AddFeeSchedule_OS' with DataRaptor Extract DRX_EligibleFeeSchedules filtering by LOB and effective date. CPT code validation uses a custom Apex invocable action 'ValidateCPTCode' that queries CPT_Code_Reference__mdt. Fee line creation handled via DataRaptor Turbo Action. Adjudication rate lookup implemented as an Apex method in existing 'ClaimAdjudicationEngine' class — queries active PNA via Provider__c lookup on Claim__c, then matches Fee_Schedule__c by CPT code and effective date. Record-Triggered Flow 'RTF_ClaimLineRateLookup' on Claim_Line__c after insert triggers the Apex invocable. Custom metadata 'CPT_Code_Reference__mdt' populated via Data Loader from CMS annual CPT release file.

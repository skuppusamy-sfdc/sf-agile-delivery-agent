## Description

As a Claims Operations Manager, I need an automated ERA (Electronic Remittance Advice) auto-posting integration that ingests inbound 835 transaction files from our clearinghouse, matches ERA line items to Claim_Line__c records, and updates Payment__c records accordingly, so that manual ERA reconciliation effort is eliminated and payment posting turnaround is reduced from 48 hours to real-time.

## Acceptance Criteria

Given an 835 ERA file is received from the clearinghouse SFTP endpoint,
When the scheduled Apex batch job 'ERAIngestionBatch' runs (every 15 minutes),
Then each CLP segment is parsed and matched to Claim__c.Claim_Number__c; unmatched CLPs are written to ERA_Exception__c with ERA_Exception__c.Reason__c = 'Claim Not Found'.

Given a CLP segment is matched to a Claim__c record,
When SVC line items are parsed,
Then each SVC maps to a Claim_Line__c record matched on Claim_Line__c.CPT_Code__c and Claim_Line__c.Service_Date__c; Payment__c records are created with Payment__c.Amount__c, Payment__c.Adjustment_Code__c, and Payment__c.Payment_Date__c populated from the 835 data.

Given a Payment__c record is created via ERA auto-posting,
When all Claim_Line__c records on a Claim__c are in status 'Paid',
Then Claim__c.Status__c is automatically updated to 'Adjudicated' via Record-Triggered Flow.

Given an ERA contains a CO-45 contractual adjustment code,
When the Payment__c record is created,
Then Payment__c.Adjustment_Type__c = 'Contractual' and Payment__c.Adjustment_Amount__c is populated; Claim_Line__c.Allowed_Amount__c is updated to reflect the Fee_Schedule__c contracted rate.

-Given ERA auto-posting completes, When all payments are posted, Then a paper EOB is printed and mailed to the provider- (deprecated — paper EOB replaced by electronic EOB generation per INS-0458)

Given ERA auto-posting encounters a duplicate 835 file (same ISA control number),
When the batch job processes the file,
Then the file is rejected, no Payment__c records are created, and an ERA_Exception__c record is created with ERA_Exception__c.Reason__c = 'Duplicate File' and ERA_Exception__c.ISA_Control_Number__c populated.

## Solution

Develop Apex class 'ERAParser835' to parse X12 835 EDI format. Implement 'ERAIngestionBatch' as Database.Batchable and Database.Schedulable with 15-minute cron. Use Custom Metadata Type 'ERA_Adjustment_Code_Mapping__mdt' to map CARC codes to Payment__c.Adjustment_Type__c picklist values. Build Record-Triggered Flow on Payment__c (after insert) to evaluate Claim__c rollup and update Status__c. Store raw 835 files in Salesforce Files linked to ERA_Batch__c header record for audit trail.

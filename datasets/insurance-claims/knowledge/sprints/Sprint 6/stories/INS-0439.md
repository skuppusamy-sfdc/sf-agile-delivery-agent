## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page that consolidates credentialing status from Credentialing_Case__c, active Provider_Network_Agreement__c (PNA) records, and any open Provider_Change_Notification__c (PCN) items so that I can assess a provider's full network standing without navigating multiple objects.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record,
When the Provider 360 FlexCard loads,
Then the card displays the most recent Credentialing_Case__c.Status__c and Credentialing_Case__c.Expiration_Date__c where Credentialing_Case__c.Provider__c = current record ID, ordered by CreatedDate DESC LIMIT 1.

Given the provider has one or more PNA records,
When the Network Agreements panel renders,
Then all Provider_Network_Agreement__c records where PNA.Status__c IN ('Active', 'Pending Renewal') and PNA.Provider__c = current record ID are listed with PNA.Network_Name__c, PNA.Effective_Date__c, PNA.Termination_Date__c, and PNA.LOB__c visible.

Given the provider has an open PCN,
When the Notifications panel renders,
Then Provider_Change_Notification__c records where PCN.Status__c != 'Closed' and PCN.Provider__c = current record ID are displayed with PCN.Change_Type__c and PCN.Submitted_Date__c.

-Given the FlexCard shows Fee_Schedule__c line items inline (deprecated — Fee Schedule detail moved to separate INS-0443 story)-

Given the provider's Credentialing_Case__c.Expiration_Date__c is within 60 days,
When the FlexCard renders,
Then a yellow warning badge 'Credential Expiring Soon' is displayed on the credentialing panel.

Given the provider has no active PNA records,
When the network panel renders,
Then the message 'No active network agreements found.' is displayed with a 'Create PNA' quick action button.

## Solution

Build OmniStudio FlexCard 'Provider360_FC' with three child card sections using separate DataRaptor Extracts: DRX_ProviderCredentialStatus (Credentialing_Case__c), DRX_ProviderPNAList (Provider_Network_Agreement__c), DRX_OpenPCNList (Provider_Change_Notification__c). Conditional rendering logic in FlexCard JSON evaluates days-until-expiry for badge display. Quick action 'Create PNA' invokes a pre-configured Quick Action on Provider__c. FlexCard deployed via Lightning App Builder on the Provider__c record page layout assigned to Provider Relations profile.

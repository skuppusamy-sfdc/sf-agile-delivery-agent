## Description

As a Provider Network Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that consolidates NPI, TIN, active Provider_Network_Agreement__c status, Fee_Schedule__c assignments, and Credentialing_Case__c summary, so that I have a single-pane view to assess provider standing without switching between multiple related lists.

## Acceptance Criteria

Given a Provider Network Specialist opens a Provider__c record,
When the Provider 360 FlexCard loads,
Then the following fields must be displayed: Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Provider_Type__c, Provider__c.Network_Status__c, and Provider__c.Primary_Specialty__c.

Given the Provider__c record has one or more Provider_Network_Agreement__c records,
When the FlexCard renders the PNA section,
Then all PNA records where PNA__c.Status__c = 'Active' must be listed showing PNA__c.Contract_Effective_Date__c, PNA__c.Contract_End_Date__c, and PNA__c.Network_Tier__c.

Given a Credentialing_Case__c exists linked to the Provider__c,
When the credentialing summary panel renders,
Then the most recent Credentialing_Case__c record must be shown with fields: Credentialing_Case__c.Status__c, Credentialing_Case__c.CAQH_Profile_Status__c, and Credentialing_Case__c.Expiration_Date__c. If Credentialing_Case__c.Expiration_Date__c is within 90 days, display a yellow warning badge.

Given a Provider__c record has no active PNA,
When the PNA section of the FlexCard renders,
Then display the message 'No active network agreements on file' with a quick-action button to initiate a new PNA from the NetworkContracting flow.

-Given the FlexCard displays Fee_Schedule__c line item details inline- (deprecated — Fee Schedule detail moved to dedicated Fee Schedule tab per design review INS-0418)

## Solution

Build OmniStudio FlexCard 'Provider360_FC' with multiple DataRaptor Extract sources: one for Provider__c core fields, one for related PNA__c (filtered Active), one for most recent Credentialing_Case__c (ORDER BY CreatedDate DESC LIMIT 1). Use FlexCard States for credentialing expiry warning logic (conditional on date delta < 90). Quick-action button invokes a Screen Flow 'InitiatePNA_SF' via FlexCard Action. Deploy to Provider__c Lightning record page via App Builder. Permission controlled via 'ProviderNetwork_Specialist_PSG' Permission Set Group.

## Description

As a Broker Relations Coordinator, I need a real-time dashboard FlexCard on the Agent/Broker Portal that surfaces active Policy__c records, pending Claim__c counts, and recent FNOL submissions tied to the broker's book of business, so that brokers can self-serve key metrics without contacting the internal operations team.

## Acceptance Criteria

Given a licensed broker user logs into the Agent/Broker Portal,
When the Broker Dashboard page loads,
Then a FlexCard renders displaying all Policy__c records where Policy__c.Broker_Account__c = the broker's linked Account and Policy__c.Status__c IN ('Active', 'Pending Renewal').

Given the broker dashboard FlexCard is rendered,
When Claim__c records exist with Claim__c.Policy__c linked to the broker's book and Claim__c.Status__c = 'Pending',
Then the FlexCard displays a badge count of pending claims grouped by LOB (Claim__c.Line_of_Business__c).

Given a broker clicks the 'View Claims' action on the FlexCard,
When the action fires,
Then the broker is navigated to a filtered Claim__c list view scoped to their book of business with columns: Claim__c.Name, Claim__c.FNOL_Date__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c.

Given a broker's book of business includes TPA-administered policies,
When the dashboard loads,
Then Policy__c records where Policy__c.TPA_Administered__c = TRUE are visually flagged with a 'TPA' pill label on the FlexCard.

-Given the FlexCard includes a COB indicator column for all policies- (deprecated — COB display moved to Policyholder 360 epic INS-0418)

Given the broker portal is accessed on a mobile viewport (< 768px),
When the FlexCard renders,
Then the layout collapses to a single-column card stack with no horizontal scroll.

## Solution

Build an OmniStudio FlexCard ('BrokerDashboardCard') with two DataRaptor Extract sources: one querying Policy__c with a SOQL filter on Broker_Account__c and Status__c, and one querying an aggregate SOQL on Claim__c grouped by Line_of_Business__c. Use FlexCard Child Cards for the pending claims badge. Navigation action uses a NavigationMixin LWC wrapper. Mobile responsiveness handled via FlexCard Conditional Rendering states. A Permission Set Group ('Broker_Portal_User_PSG') gates access. Custom Label set for TPA pill text to support future i18n.

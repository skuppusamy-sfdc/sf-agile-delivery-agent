## Description

**Bug Report**

**Summary:** A CPT code format validation on Claim_Line__c is incorrectly triggering for claims in 'Draft' status, preventing claims intake staff from saving incomplete Claim_Line__c records during FNOL intake.

**Steps to Reproduce:**
1. Create a new Claim__c record with Claim__c.Status__c = 'Draft'.
2. Navigate to the Claim Lines related list and click 'New'.
3. Enter a partial record — populate Claim_Line__c.Service_Date__c and Claim_Line__c.Provider__c but leave Claim_Line__c.CPT_Code__c blank.
4. Click Save.
5. Observe: A validation error fires — 'CPT Code must be a valid 5-digit numeric code' — blocking the save.

**Expected Behavior:** CPT code validation should only fire when Claim__c.Status__c is 'Submitted' or beyond. Draft claim lines should be saveable without a CPT code.

**Actual Behavior:** Validation rule 'VR_ClaimLine_CPTFormat' fires regardless of parent Claim__c.Status__c, blocking save on Draft records.

**Environment:** Acme Insurance Full Sandbox (AIS-FS01)
**Reported By:** Claims Operations Manager (Rachel Torres)
**Frequency:** 100% reproducible
**Impact:** High — blocking FNOL intake workflow for all claims staff.

## Acceptance Criteria

As per Description.

Given the fix is deployed,
When a Claim_Line__c record is saved with a blank CPT_Code__c and the parent Claim__c.Status__c = 'Draft',
Then the save succeeds without any validation error.

Given the fix is deployed,
When a Claim_Line__c record is saved with an invalid CPT_Code__c (e.g., '12AB') and the parent Claim__c.Status__c = 'Submitted',
Then the validation error 'CPT Code must be a valid 5-digit numeric code' fires as expected.

## Solution

Update Validation Rule 'VR_ClaimLine_CPTFormat' on Claim_Line__c to add a parent status guard. Modify the rule formula to: AND(NOT(ISPICKVAL(Claim__r.Status__c, 'Draft')), NOT(ISPICKVAL(Claim__r.Status__c, 'FNOL Received')), NOT(REGEX(CPT_Code__c, '[0-9]{5}')), NOT(ISBLANK(CPT_Code__c)) ). Deploy via changeset to Full Sandbox for regression testing before production push. Update unit tests in 'ClaimLineValidationTest' Apex class to cover Draft status bypass scenario.

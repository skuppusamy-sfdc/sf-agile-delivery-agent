## Description

As a Broker Relations Coordinator, I need a self-service broker onboarding application accessible from the public-facing portal that captures broker demographic data, NPN (National Producer Number), appointed states, and LOB preferences, and automatically triggers a DocuSign broker agreement envelope upon submission so that broker recruitment is accelerated and paper-based contracting is eliminated.

## Acceptance Criteria

Given a prospective broker navigates to the public onboarding URL,
When the onboarding Screen Flow 'BrokerOnboarding_Flow' renders,
Then Screen 1 collects: First Name, Last Name, Email, Phone, NPN, Tax ID (TIN), Primary State of Residence, and Years of Experience (all required)

Given the broker submits their NPN on Screen 1,
When the Flow invokes the NIPR (National Insurance Producer Registry) validation callout,
Then if the NPN is valid and active, the broker advances to Screen 2; if invalid, an error message 'NPN not found or inactive in NIPR. Please verify your producer number.' is displayed and submission is blocked

Given a valid NPN is confirmed,
When the broker completes Screen 2 (Appointed States and LOB Preferences),
Then they may select 1 to 50 states via a multi-select picklist and at least one LOB from: Commercial Lines, Personal Lines, Group Health, Workers Comp

Given the broker completes all screens and clicks 'Submit Application',
When the Flow executes the final step,
Then:
  - A Contact record is created with Contact.RecordTypeId = 'Broker' and Contact.NPN__c populated
  - A Group_Service_Agreement__c (GSA) record is created in Draft status linked to the new Contact
  - A DocuSign envelope is sent to the broker's submitted email containing the Broker Appointment Agreement template, with merge fields populated from the Contact and GSA records
  - The broker receives a confirmation email with subject 'Acme Insurance — Broker Application Received'

Given the DocuSign envelope is completed by the broker,
When the DocuSign status webhook fires,
Then GSA__c.Status__c updates to 'Executed', GSA__c.Execution_Date__c = envelope completion date, and the Contact is assigned the Permission Set 'Broker_Portal_Access'

Given the DocuSign envelope expires without completion (after 30 days),
When the expiry webhook fires,
Then GSA__c.Status__c = 'Expired' and a Task is created for the Broker Relations Coordinator to follow up

## Solution

Build a Lightning Screen Flow 'BrokerOnboarding_Flow' with 3 screens deployed to an unauthenticated Experience Cloud page. NIPR validation performed via an Apex invocable action 'NIPRValidationAction' using a Named Credential 'NIPR_NC' REST callout. Contact and GSA creation handled in Flow Create Records elements with appropriate RecordType assignments. DocuSign envelope triggered via DocuSign for Salesforce 'Send via DocuSign' Flow action using a pre-configured envelope template 'Broker_Appointment_Agreement'. DocuSign Connect webhook handled by a custom REST endpoint Apex class 'DocuSignWebhookHandler' that updates GSA__c.Status__c and triggers a Permission Set assignment via Apex. Expiry handling implemented via a Scheduled Flow querying GSA__c WHERE Status__c = 'Pending Signature' AND DocuSign_Expiry_Date__c < TODAY. Public access secured via Guest User Profile with minimal object permissions scoped to the onboarding flow only.

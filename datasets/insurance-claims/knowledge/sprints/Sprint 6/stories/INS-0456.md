## Description

As a Provider Network Specialist, I need an automated workflow that detects NPI or TIN discrepancies between incoming NPPES data feed records and existing Provider__c data in Salesforce, creates a Provider_Change_Notification__c (PCN) record, and routes it for specialist review, so that provider data integrity is maintained and downstream claims adjudication errors caused by stale NPI/TIN values are prevented.

## Acceptance Criteria

Given a nightly NPPES data file is ingested via the DataIntegration batch job,
When a Provider__c.NPI__c or Provider__c.TIN__c value in Salesforce differs from the corresponding NPPES record,
Then a Provider_Change_Notification__c record is created with PCN__c.Change_Type__c = 'NPI Update' or 'TIN Update', PCN__c.Old_Value__c, PCN__c.New_Value__c, PCN__c.Source__c = 'NPPES', and PCN__c.Status__c = 'Pending Review'.

Given a PCN record is created,
When PCN__c.Change_Type__c = 'TIN Update',
Then PCN__c.Priority__c is automatically set to 'High' and assigned to the Provider Network Specialist queue, because TIN changes affect tax reporting and payment routing.

Given a PCN record is created,
When PCN__c.Change_Type__c = 'NPI Update',
Then PCN__c.Priority__c = 'Medium' and assigned to Provider_Network_Queue.

Given a PCN record in 'Pending Review' status,
When a Provider Network Specialist approves the change (PCN__c.Status__c = 'Approved'),
Then a Record-Triggered Flow automatically updates Provider__c.NPI__c or Provider__c.TIN__c with PCN__c.New_Value__c and stamps PCN__c.Approved_By__c and PCN__c.Approval_Date__c.

Given a PCN approval updates Provider__c.TIN__c,
When the update is committed,
Then all related active Payment__c records where Payment__c.Status__c = 'Scheduled' are flagged with Payment__c.TIN_Review_Required__c = true via a batch Apex job (PCNTINUpdateBatch) to prevent erroneous disbursement.

Given a duplicate NPPES change notification for the same Provider__c within 24 hours,
When the batch job runs,
Then no duplicate PCN__c record is created; instead, the existing PCN__c.Reprocessed_Count__c is incremented.

-Given PCN approval, a DocuSign envelope must be sent to the provider for change acknowledgment- (deferred to Phase 2 per compliance team decision)

## Solution

Scheduled Apex batch class (NPPESChangeDetectionBatch) runs nightly at 02:00 UTC, consuming NPPES JSON feed from a Named Credential endpoint. Batch compares NPI__c and TIN__c fields using a Map<String, Provider__c> keyed on Provider__c.NPPES_ID__c. On mismatch, inserts PCN__c records via a separate DML operation respecting governor limits (200-record chunks). Record-Triggered Flow on PCN__c (after update, Status__c changed to 'Approved') updates parent Provider__c and invokes PCNTINUpdateBatch via Queueable Apex. Duplicate suppression uses a before-insert Apex Trigger checking open PCN records within a 24-hour window. Custom metadata (PCN_Config__mdt) stores queue IDs and priority mapping.

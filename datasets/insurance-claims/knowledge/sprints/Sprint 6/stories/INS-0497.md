## Description

As a Credentialing Specialist, I need the system to automatically trigger a CAQH ProView profile verification API call when a new Credentialing_Case__c is created so that I do not have to manually initiate CAQH lookups, reducing credentialing cycle time and ensuring provider data accuracy at the start of every credentialing workflow.

## Acceptance Criteria

Given a new Credentialing_Case__c is created with Credentialing_Case__c.Stage__c = 'Initial Review' and Credentialing_Case__c.Provider__c populated,
When the Record-Triggered Flow fires (after insert),
Then an asynchronous Apex job is enqueued to call the CAQH ProView REST API using the provider's Provider__c.CAQH_Provider_ID__c value.

Given the CAQH API call returns a successful response (HTTP 200),
When the response is parsed,
Then the following fields on Credentialing_Case__c are updated: CAQH_Profile_Status__c (from response 'profileStatus'), CAQH_Last_Attested_Date__c, CAQH_DEA_Verified__c (boolean), CAQH_Malpractice_Verified__c (boolean), and CAQH_Verification_Timestamp__c = DateTime.now().

Given the CAQH API returns a 404 (provider not found),
When the error is handled,
Then Credentialing_Case__c.CAQH_Profile_Status__c = 'Not Found', a Chatter post is added to the case: '@[Credentialing Specialist] CAQH profile not found for NPI: [NPI value]. Manual verification required.', and Credentialing_Case__c.Stage__c remains 'Initial Review'.

Given Provider__c.CAQH_Provider_ID__c is blank at the time of case creation,
When the flow evaluates the trigger condition,
Then the CAQH API call is skipped and Credentialing_Case__c.CAQH_Profile_Status__c = 'CAQH ID Missing — Manual Entry Required' is set.

-Given the CAQH response includes board certification data, populate Credentialing_Case__c.Board_Certification_Status__c- (deprecated — board cert data to be sourced from NPPES integration per architecture decision ADR-031, tracked in INS-0503)

## Solution

Record-Triggered Flow on Credentialing_Case__c (after insert, Stage__c = 'Initial Review') calls an Apex Action (invocable method) that enqueues a QueueableApex class (CAQHVerificationQueueable). The Queueable performs an HTTP callout to CAQH ProView REST API via Named Credential (CAQH_ProView_NC). Response JSON is parsed using a custom CAQHResponseWrapper Apex class. Field updates written back to Credentialing_Case__c via DML. Chatter post via ConnectApi.ChatterFeeds. Error handling logs failures to Integration_Error_Log__c. Custom metadata (CAQH_Integration_Config__mdt) stores endpoint URL, timeout, and retry settings. Apex test class CAQHVerificationQueueableTest uses HttpCalloutMock for unit coverage.

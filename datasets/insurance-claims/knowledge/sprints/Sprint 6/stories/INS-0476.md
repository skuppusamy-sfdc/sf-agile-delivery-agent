## Description

As a Broker Relations Coordinator, I need a consolidated dashboard FlexCard on the Agent/Broker Portal that surfaces active policy counts, open claim summaries, and pending FNOL submissions grouped by LOB so that brokers can quickly assess their book of business without navigating multiple Salesforce screens.

## Acceptance Criteria

Given a licensed broker user logs into the Agent Portal,
When the dashboard loads,
Then a FlexCard renders displaying Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.Broker__c = current user's associated Contact ID, grouped by Policy__c.Line_of_Business__c.

Given the broker dashboard FlexCard is displayed,
When the broker views the Claims section,
Then Claim__c records with Claim__c.Status__c IN ('Open', 'Pending Review', 'In Adjudication') linked to the broker's policies are shown with counts per LOB.

Given a broker has pending FNOL submissions,
When the FNOL summary tile is rendered,
Then Claim__c records where Claim__c.FNOL_Submitted__c = true AND Claim__c.Status__c = 'Draft' are listed with Claim__c.Date_of_Loss__c and Claim__c.Claimant_Name__c visible.

Given the broker clicks a policy count tile,
When navigation occurs,
Then the broker is redirected to a filtered list view of Policy__c records scoped to their broker code.

-Given the dashboard includes a TPA summary panel, When a TPA-administered policy exists, Then TPA__c.Name and TPA__c.Contact_Phone__c are displayed inline.- (Deprecated — TPA panel deferred to Sprint 8)

Given the FlexCard data is loaded,
When the broker's book of business exceeds 500 active policies,
Then pagination or lazy-load is applied and initial render completes within 3 seconds.

## Solution

Build OmniStudio FlexCard 'BrokerDashboard_FC' with multiple DataRaptor Extract sources: one querying Policy__c filtered by Policy__c.Broker__c using the logged-in user's Contact lookup, a second querying Claim__c with a LEFT JOIN to Policy__c. Use FlexCard states to toggle between LOB tabs (Medical, Dental, Vision, P&C). Implement a Custom Permission 'Broker_Portal_Access' and assign via Permission Set Group 'Broker_PSG'. Cache DataRaptor results using Platform Cache (Session partition) with a 5-minute TTL to support large books of business. Navigation actions use OmniStudio Navigation Mixin to route to filtered list views.

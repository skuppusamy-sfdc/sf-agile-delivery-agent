## Description

As a Claims Operations Manager, I need an Agentforce Agent Action that evaluates newly submitted Claim__c records against a configurable set of SIU (Special Investigations Unit) referral criteria and surfaces a recommendation to the assigned claims handler so that potentially fraudulent claims are flagged earlier in the adjudication lifecycle, reducing leakage and improving SIU referral consistency. The action should be invokable from the Claims Handler's Lightning record page via the Agentforce sidebar.

## Acceptance Criteria

Given a claims handler opens a Claim__c record with Claim__c.Status__c = 'Under Review',
When the Agentforce Claims Triage Agent is invoked from the sidebar,
Then the agent retrieves Claim__c fields: Claim__c.Claim_Number__c, Claim__c.Loss_Type__c, Claim__c.Incurred_Amount__c, Claim__c.Date_of_Loss__c, Claim__c.Source__c, Claim__c.Claimant_Prior_Claims_Count__c, and all associated Claim_Line__c.CPT_Code__c values.

Given the agent has retrieved claim data,
When the Prompt Template is evaluated,
Then the agent applies SIU referral rules sourced from the SIU_Referral_Rule__mdt custom metadata type (fields: Rule_Name__c, Threshold_Value__c, LOB__c, Active__c) and generates a structured recommendation with: Recommendation (Refer to SIU / No Referral Needed / Monitor), Confidence_Score (0–100), and Primary_Trigger_Rule__c.

Given the agent recommendation is 'Refer to SIU' with Confidence_Score >= 75,
When the response is rendered in the Agentforce sidebar,
Then the UI displays the recommendation in a red alert banner with the Primary_Trigger_Rule__c explanation and a one-click 'Create SIU Referral' button that pre-populates a Case with Case.RecordType = 'SIU_Referral', Case.Related_Claim__c, and Case.Queue = 'SIU_Intake'.

Given the agent recommendation is 'Monitor',
When the response renders,
Then a yellow advisory banner is shown with the trigger rule and a 'Flag for Review' button that sets Claim__c.SIU_Monitor_Flag__c = true and logs a Claim__c Activity noting the AI recommendation and timestamp.

Given the Agentforce action is executed,
When the response is generated (regardless of recommendation),
Then the interaction is logged to a custom object AI_Interaction_Log__c with fields: Claim__c, Agent_Action__c = 'SIU Triage', Recommendation__c, Confidence_Score__c, Prompt_Version__c, Executed_By__c, Execution_Timestamp__c to support auditability and model performance monitoring.

## Solution

Configure an Agentforce Agent Action (SIU_Triage_ClaimAction) using Salesforce Einstein Copilot Studio. Define a Prompt Template (SIUTriagePrompt_v1) using the Flex Prompt Template type, injecting Claim__c and Claim_Line__c data via a FlexCard data merge. SIU rules fetched from SIU_Referral_Rule__mdt via an Apex invocable method (GetActiveSIURules) called within the Agent Action's pre-processing step. LLM response parsed by a custom Apex ResponseParserService that extracts structured JSON (recommendation, confidence, trigger). 'Create SIU Referral' button wired to a Record-Triggered Quick Action invoking a Flow that creates the SIU Case. AI_Interaction_Log__c writes handled by Apex post-action hook. Permission Set (Agentforce_ClaimsAI_PS) grants access to Agent Action for Claims Handler and Claims Operations Manager profiles.

## Description

**Bug Report — Fee Schedule Lookup Incorrect Allowed Amount**

**Environment:** Acme Insurance Full Sandbox (sprint6-full)
**Severity:** High — impacts payment accuracy for adjudicated claims

**Steps to Reproduce:**
1. Ensure two Fee_Schedule__c records exist for the same Provider__c and CPT code (e.g., CPT 99213) with overlapping effective date ranges but different LOB__c values (e.g., 'Commercial' and 'Medicare Advantage').
2. Create a Claim__c record with Claim__c.LOB__c = 'Medicare Advantage'.
3. Add a Claim_Line__c record with Claim_Line__c.CPT_Code__c = '99213'.
4. Trigger adjudication by setting Claim__c.Status__c = 'Ready for Adjudication'.
5. Observe Claim_Line__c.Allowed_Amount__c after the adjudication flow completes.

**Expected Behavior:** Claim_Line__c.Allowed_Amount__c is populated from the Fee_Schedule__c where Fee_Schedule__c.LOB__c = 'Medicare Advantage' for CPT 99213.

**Actual Behavior:** Claim_Line__c.Allowed_Amount__c is populated from the 'Commercial' Fee_Schedule__c record, resulting in an incorrect (lower) allowed amount. The lookup appears to return the first record ordered by CreatedDate ASC rather than filtering by LOB__c.

**Impact:** Payment__c disbursements will be underpaid for Medicare Advantage claims adjudicated against providers with dual LOB fee schedules. Estimated 12% of adjudicated claims in sprint6 regression data set are affected.

## Acceptance Criteria

As per Description. Fix must be verified with regression test covering: (1) single LOB fee schedule — existing behavior unchanged, (2) dual LOB fee schedules — correct LOB-matched allowed amount returned, (3) no active fee schedule found — Claim_Line__c.Allowed_Amount__c = null and Claim_Line__c.Fee_Schedule_Exception__c = true.

## Solution

Root cause: Apex class 'FeeScheduleLookupService.cls' SOQL query is missing WHERE Fee_Schedule__c.LOB__c = :claim.LOB__c filter. Fix by adding LOB__c filter and ordering by Fee_Schedule__c.Effective_Date__c DESC to resolve any remaining tie-breaking. Add a null-safety check to set Fee_Schedule_Exception__c = true when no matching record is found. Update unit tests in 'FeeScheduleLookupServiceTest.cls' to cover multi-LOB scenario with ≥ 95% code coverage. Deploy fix through Change Set 'CS_INS_0417_FeeScheduleFix'.

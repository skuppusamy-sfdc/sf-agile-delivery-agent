## Description

As a Broker Relations Coordinator, I need a consolidated broker dashboard FlexCard displaying active policies, open claims, and commission summaries so that brokers can self-serve key account metrics without contacting the internal operations team. This story supports the Agent/Broker Portal MVP for Sprint 6 and extends the broker profile work completed in INS-0462.

## Acceptance Criteria

Given a licensed broker user logs into the Agent Portal,
When the dashboard loads,
Then a FlexCard displays all Policy__c records where Policy__c.Broker__c = current user's linked Contact and Policy__c.Status__c = 'Active', showing Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, and Policy__c.Premium_Amount__c.

Given the broker dashboard is rendered,
When the Claim Summary section loads,
Then all Claim__c records linked via Policy__c.Broker__c are grouped by Claim__c.Status__c (Open, Adjudicated, Closed) with a count and total Claim__c.Incurred_Amount__c per status bucket.

Given a broker has more than 25 active policies,
When the policy list renders,
Then pagination is applied showing 25 records per page with sort capability on Policy__c.Effective_Date__c descending.

Given a broker clicks a Policy__c record row in the FlexCard,
When the navigation action fires,
Then the broker is routed to the Policyholder 360 detail page for that policy within the portal experience.

-Given the dashboard includes a commission YTD widget showing Payment__c.Commission_Amount__c totals- (deprecated — commission data deferred to INS-0511 per product decision 2024-11-14)

Given a broker user does NOT have an associated Contact with Contact.Broker_License_Active__c = true,
When the dashboard loads,
Then a banner message is displayed: 'Your broker license is inactive. Contact your Broker Relations Coordinator.' and all policy/claim data panels are hidden.

## Solution

Build an OmniStudio FlexCard (BrokerDashboard_FC) with two child DataRaptors: (1) DR_GetBrokerPolicies — SOQL on Policy__c filtered by Broker__c = :userId linked Contact, Status__c = 'Active'; (2) DR_GetBrokerClaimSummary — aggregate SOQL on Claim__c grouped by Status__c. Use FlexCard Datatable component for policy list with client-side pagination (pageSize=25). Navigation action uses OmniStudio Navigation Mixin to route to PolicyHolder360 OmniScript. Broker license validation handled via a Conditional View State on FlexCard using Contact.Broker_License_Active__c. Assign broker portal users to BrokerPortal_PSG (Permission Set Group) which grants Read on Policy__c, Claim__c via Sharing Rules scoped to Broker__c field.

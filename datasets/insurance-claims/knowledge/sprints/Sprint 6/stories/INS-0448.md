## Description

As a Provider Network Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record whenever a provider's NPI (Provider__c.NPI__c) or TIN (Provider__c.TIN__c) is updated on the Provider__c record so that downstream compliance, credentialing, and contracting teams are notified of critical data changes without manual intervention.

## Acceptance Criteria

Given a Provider__c record's NPI__c field is updated by any user,
When the record is saved,
Then a Provider_Change_Notification__c record is auto-created with PCN__c.Change_Type__c = 'NPI Change', PCN__c.Provider__c = Provider__c.Id, PCN__c.Old_Value__c = previous NPI, PCN__c.New_Value__c = updated NPI, PCN__c.Status__c = 'Pending Review', and PCN__c.Created_Date__c = today.

Given a Provider__c record's TIN__c field is updated,
When the record is saved,
Then a Provider_Change_Notification__c is created with PCN__c.Change_Type__c = 'TIN Change' and the same field population pattern as the NPI change rule.

Given both NPI__c and TIN__c are changed in the same save operation,
When the record is saved,
Then two separate PCN records are created, one for each change type.

Given a PCN record is created,
When PCN__c.Status__c = 'Pending Review',
Then a Salesforce Task is auto-assigned to the Provider__c.Owner (Provider Network Specialist) with Subject = 'Review PCN: [Change_Type__c] for [Provider__c.Name]' and Due Date = TODAY+2.

Given the PCN is created,
When PCN__c.Status__c = 'Pending Review',
Then an email notification is sent to the Credentialing_Case__c team queue using an Email Alert on the PCN__c object.

-Given PCN creation triggers an automatic CAQH re-verification API call- (deprecated — CAQH re-verification moved to Credentialing epic, tracked in INS-0431)

## Solution

Implement a Record-Triggered Flow on Provider__c (Before Save for old value capture, After Save for PCN creation) that checks if NPI__c or TIN__c ISCHANGED. Use two separate Create Records elements to handle individual PCN creation per changed field. A separate After Save flow path creates the Task record. Email Alert on Provider_Change_Notification__c object triggered by a Flow-invoked process. Old value capture uses $Record__Prior in the Before Save context stored in flow variables, passed via a Platform Event (PCN_Initiated__e) to the After Save context to avoid governor limit issues.

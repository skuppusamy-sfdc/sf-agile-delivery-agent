## Description

As a Provider Network Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces credentialing status, active network agreements, and upcoming contract milestones in a single view, so that I can quickly assess a provider's network standing without navigating across multiple related lists and tabs.

This story covers the internal Salesforce UI (not the portal-facing view). Portal-facing provider self-service is scoped to Phase 2.

## Acceptance Criteria

Given a Provider Network Specialist views a Provider__c (Person Account) record,
When the Provider 360 FlexCard loads in the Lightning Record Page,
Then it displays the following data points in clearly labeled tiles: Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Specialty__c, Provider__c.Network_Status__c, and the count of active Provider_Network_Agreement__c records (PNA__c.Status__c = 'Active') related to this provider.

Given the Provider 360 FlexCard is rendered,
When the related Credentialing_Case__c record has Credentialing_Case__c.Status__c = 'In Review' or 'Pending CAQH Verification',
Then a 'Credentialing In Progress' amber badge is displayed in the Credentialing tile with the Credentialing_Case__c.Due_Date__c value shown as 'Due: MM/DD/YYYY'.

Given the provider has one or more Provider_Network_Agreement__c records,
When PNA__c.End_Date__c is within 60 days of TODAY(),
Then the agreement row in the Agreements tile renders a red 'Expiring Soon' indicator alongside PNA__c.Network__c and PNA__c.LOB__c.

Given the Provider 360 FlexCard loads,
When Provider__c.Network_Status__c = 'Terminated',
Then the entire card header renders with a red background and a 'TERMINATED' watermark label, and all action buttons ('Initiate Credentialing', 'Add Agreement') are hidden.

-Given the FlexCard displays the provider's CAQH profile link as an external hyperlink- (removed — CAQH deep-link URL format not confirmed by integration team, tracked in INS-0498)

## Solution

Build OmniStudio FlexCard 'Provider360Card' activated on the Provider__c Lightning Record Page via a Custom Tab component override. Use three DataRaptor Extract actions: (1) DR_GetProviderCore — queries Provider__c fields; (2) DR_GetActiveAgreements — queries Provider_Network_Agreement__c with filter Status__c = 'Active' and related End_Date__c; (3) DR_GetCredentialingCase — queries most recent Credentialing_Case__c ordered by CreatedDate DESC LIMIT 1. Conditional rendering blocks in FlexCard JSON handle badge logic (amber/red) using 'Conditions' on field values. The 'Terminated' header style uses a FlexCard State with custom CSS class injected via the styleClass property. Deploy to the Provider__c Lightning App Builder page. Add FlexCard to the 'Provider_Network_Specialist' Permission Set for visibility control.

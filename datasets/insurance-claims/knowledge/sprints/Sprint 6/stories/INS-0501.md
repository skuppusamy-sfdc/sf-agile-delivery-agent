## Description

As a Broker Relations Coordinator, I need brokers to be able to submit a First Notice of Loss (FNOL) on behalf of their policyholders directly through the Agent/Broker Portal using a guided OmniScript wizard, so that FNOL intake is available 24/7 without requiring broker calls to the claims operations center and reduces manual data entry errors on Claim__c creation.

The OmniScript must support Property & Casualty and Health LOBs at MVP scope. Workers' Compensation LOB is deferred to Phase 2.

## Acceptance Criteria

Given a broker is authenticated in the Experience Cloud portal,
When they navigate to 'Submit a Claim' and select a policy,
Then an OmniScript step 'Policy Lookup' pre-populates Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Policyholder__c, and Policy__c.Coverage_Effective_Date__c as read-only fields sourced via DataRaptor Extract.

Given the broker reaches the 'Loss Details' step,
When Policy__c.LOB__c = 'Health',
Then conditional OmniScript steps render ICD-10 diagnosis code entry (Claim__c.Primary_Diagnosis_Code__c) and treating provider NPI lookup (Claim__c.Treating_Provider__c via Provider__c.NPI__c) and the 'Property Damage' step group is hidden.

Given the broker reaches the 'Loss Details' step,
When Policy__c.LOB__c = 'Property & Casualty',
Then conditional steps render Claim__c.Loss_Location__c (address fields), Claim__c.Damage_Description__c (rich text, max 2000 chars), and estimated loss amount Claim__c.Estimated_Loss_Amount__c.

Given the broker completes all required steps and clicks 'Submit',
When the OmniScript Integration Procedure executes,
Then a Claim__c record is created with Claim__c.Status__c = 'Submitted', Claim__c.FNOL_Source__c = 'Broker Portal', Claim__c.Submitted_Date__c = TODAY(), Claim__c.Broker__c = submitting broker's Contact.AccountId, and Claim__c.Policy__c = selected Policy__c.Id.

Given the Claim__c record is successfully created,
When the OmniScript final step renders,
Then the broker sees a confirmation screen displaying the generated Claim__c.Claim_Number__c and a system-generated email notification is sent to the policyholder's Account.PersonEmail via a Platform Event 'FNOL_Submitted__e'.

Given a broker attempts to submit FNOL for a policy where Policy__c.Status__c = 'Cancelled' or Policy__c.Status__c = 'Lapsed',
When the Policy Lookup step validates the selection,
Then an inline error message 'This policy is not active and cannot be used to file a claim. Please contact your Broker Relations Coordinator.' is displayed and the 'Next' button remains disabled.

-Given the broker can attach supporting documents during FNOL submission via the OmniScript file upload step- (deferred to INS-0512 — document attachment capability pending Content delivery network configuration)

## Solution

Build a multi-step OmniScript 'BrokerFNOLSubmission' (API name: BrokerFNOLSubmission_OS) with the following step structure: (1) Policy Search — DataRaptor Extract on Policy__c; (2) Claimant Info — pre-populated from Policy__c.Policyholder__c; (3) Loss Details — conditional step groups driven by LOB__c picklist value using OmniScript Conditional Views; (4) Review & Submit — read-only summary; (5) Confirmation. An Integration Procedure 'FNOL_Create_IP' orchestrates the Claim__c DML via a DataRaptor Post action and publishes the FNOL_Submitted__e Platform Event. An Apex trigger handler 'FNOLEventHandler' subscribes to the Platform Event via a Trigger on FNOL_Submitted__e to dispatch the policyholder email via Messaging.sendEmail. OmniScript deployed to Experience Cloud via Remote Site Settings and CSP Trusted Sites. Broker portal users granted access via Permission Set Group 'Broker_Portal_User_PSG'.

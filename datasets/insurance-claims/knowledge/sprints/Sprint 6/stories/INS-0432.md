## Description

**Bug Report — relates to INS-0430**

**Environment:** Acme Insurance Full Sandbox (FS1)
**Build:** Sprint 6 Deploy Tag v6.2.1

**Steps to Reproduce:**
1. Log in as a broker user whose User record has NO linked Contact (Contact.AccountId is null / broker onboarding incomplete).
2. Navigate to the Agent/Broker Portal home page.
3. Click 'File a New Claim (FNOL)' action button.
4. On Step 1 of the OmniScript 'AgentPortal_FNOL_OS', enter any valid Policy Number in the search field.
5. Click 'Search'.

**Expected Behavior:** The OmniScript should display a user-friendly validation message such as 'Your broker profile is incomplete. Please contact your administrator to link your portal account.' and prevent further progression.

**Actual Behavior:** The OmniScript throws an unhandled NullPointerException in the DataRaptor Extract 'DR_PolicyLookup_ByBroker' at the JSONPath mapping for `$.BrokerContactId`, causing a generic 'An error has occurred' modal with no actionable guidance. The browser console logs: `TypeError: Cannot read properties of null (reading 'Id')` from the LWC OmniScript runtime.

**Frequency:** 100% reproducible with affected user profile.
**Impact:** All newly onboarded broker users who have not yet completed the Contact-linking step in Workday/Salesforce sync are blocked from FNOL submission.

## Acceptance Criteria

As per Description. Fix must ensure DataRaptor 'DR_PolicyLookup_ByBroker' gracefully handles null BrokerContactId by returning an empty result set rather than throwing. OmniScript Step 1 must display a configured error message card when BrokerContactId is null, blocking further navigation without a system error modal.

## Solution

Add a null-check condition in DataRaptor Extract 'DR_PolicyLookup_ByBroker' input filter: if BrokerContactId is null or empty, return empty JSON array rather than executing SOQL. Add an OmniScript Validation Rule on Step 1 that checks the DataRaptor response for the BrokerContactId value before allowing Search execution; if null, display the configured error message using an OmniScript Message element. Update the Integration Procedure 'IP_FNOL_PolicySearch' to include a pre-check Apex action 'BrokerProfileValidationAction' that returns isBrokerProfileComplete boolean flag.

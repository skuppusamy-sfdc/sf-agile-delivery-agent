## Description

As a Credentialing Specialist, I need a Credentialing_Case__c record to be automatically created whenever a Provider_Network_Agreement__c (PNA) transitions to 'Executed' status for a provider who does not already have an active credentialing case so that no newly contracted provider begins seeing patients before credentialing is complete.

## Acceptance Criteria

Given a PNA record is updated,
When PNA.Status__c transitions from any prior value to 'Executed',
Then the system must query for existing Credentialing_Case__c records where Credentialing_Case__c.Provider__c = PNA.Provider__c and Credentialing_Case__c.Status__c IN ('Open', 'In Review', 'Pending CAQH').

Given no active Credentialing_Case__c is found,
When the flow creates the new record,
Then Credentialing_Case__c must be created with: Credentialing_Case__c.Provider__c = PNA.Provider__c, Credentialing_Case__c.PNA__c = PNA ID, Credentialing_Case__c.Status__c = 'Open', Credentialing_Case__c.Source__c = 'Auto — PNA Execution', Credentialing_Case__c.Priority__c = 'Standard', and Credentialing_Case__c.Due_Date__c = TODAY() + 30.

Given an active Credentialing_Case__c already exists for the provider,
When the flow evaluates the duplicate check,
Then no new Credentialing_Case__c must be created and the existing case ID must be stamped on PNA.Active_Credentialing_Case__c (lookup field).

Given the Credentialing_Case__c is created,
When the record is saved,
Then a Task must be created assigned to the Credentialing Queue with Subject = 'Initial Credentialing Review — [Provider Name]' and ActivityDate = Credentialing_Case__c.Due_Date__c.

Given the flow encounters a DML exception,
When the fault path is triggered,
Then an error record must be written to Error_Log__c and an email alert must be sent to the Credentialing Operations group email alias.

## Solution

Implement a Record-Triggered Flow 'PNA_Credentialing_Case_AutoCreate_RTF' on Provider_Network_Agreement__c, triggering on Update when PNA.Status__c changes to 'Executed' (entry condition: ISCHANGED(Status__c) AND Status__c = 'Executed'). Use a Get Records element to check for existing active Credentialing_Case__c. Decision element routes to Create Records (new case) or Update Records (stamp existing case ID on PNA). A separate Create Records element generates the Task assigned to Credentialing Queue. Fault path uses a custom invocable Apex method ErrorLogService.logFlowError() to write to Error_Log__c. Flow runs in system context with explicit sharing considerations documented.

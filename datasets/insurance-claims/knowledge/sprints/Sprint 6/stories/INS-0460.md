## Description

As a Broker Relations Coordinator, I need a guided multi-step OmniScript that captures all required employer group data and generates a Group_Service_Agreement__c (GSA) record, then automatically routes the GSA contract document via DocuSign for employer e-signature, so that group onboarding cycle time is reduced from the current 5-day manual process to under 48 hours.

Extends INS-0437 (GSA record model and validation rules).

## Acceptance Criteria

Given a Broker Relations Coordinator initiates 'New Group Onboarding' from the Account record page of a prospective employer (Account.RecordType = 'Employer Group'),
When the OmniScript launches,
Then Step 1 pre-populates Account.Name, Account.BillingAddress, and Account.Tax_ID__c if already present, and requires manual entry of Group_Size__c (numeric, min 2), Effective_Date__c (must be first of a calendar month), and selected LOB picklist (multi-select).

Given Step 2 (Plan Selection),
When the coordinator selects LOBs,
Then available Fee_Schedule__c records are dynamically filtered by Fee_Schedule__c.LOB__c and Fee_Schedule__c.Effective_Date__c <= proposed GSA Effective_Date__c and presented as a selectable table.

Given Step 3 (GSA Terms),
When the coordinator enters contract terms,
Then Group_Service_Agreement__c.Contract_Term_Months__c must be IN (12, 24, 36) and GSA__c.Auto_Renew__c defaults to true.

Given Step 4 (Review & Submit),
When the coordinator submits the OmniScript,
Then a Group_Service_Agreement__c record is created with GSA__c.Status__c = 'Pending Signature', GSA__c.Account__c linked to the employer Account, GSA__c.Assigned_Broker__c populated from the initiating user's associated Broker Contact, and all selected Fee_Schedule__c Ids stored in a related junction object (GSA_Fee_Schedule__c).

Given the GSA record is created,
When GSA__c.Status__c = 'Pending Signature',
Then a Conga CLM-generated contract document is merged using the GSA data and sent via DocuSign to the email address on Account.Primary_Contact__r.Email with a 7-day signing deadline.

Given the DocuSign envelope is completed (all signatories signed),
When the DocuSign status callback is received,
Then GSA__c.Status__c is updated to 'Active', GSA__c.Signed_Date__c is stamped, the signed PDF is attached to the GSA__c record as a ContentVersion, and a Welcome Email is sent to the employer contact via a Flow Email Action.

Given the DocuSign envelope expires without signature (after 7 days),
When the expiry webhook is received,
Then GSA__c.Status__c = 'Signature Expired', a Task is created for the Broker Relations Coordinator queue with Subject = 'GSA Signature Follow-Up Required', and GSA__c.Expiry_Follow_Up_Date__c = today + 2 business days.

-Given group size > 500, an Underwriting Analyst manual approval step is required before DocuSign routing- (superseded; approval threshold updated to 1000 per INS-0448 underwriting policy change)

## Solution

OmniScript ('GroupOnboarding_OS') with 5 steps using DataRaptor Extract for Account pre-population and Fee_Schedule__c dynamic filtering. Integration Procedure ('CreateGSAAndRoute_IP') orchestrates: (1) DataRaptor Load to create GSA__c and GSA_Fee_Schedule__c junction records, (2) Conga CLM API callout via Named Credential to generate the contract PDF, (3) DocuSign API callout to create and send the envelope with recipient mapped from Account.Primary_Contact__r.Email. DocuSign callback handled by a REST Apex endpoint (DocuSignCallbackController) that updates GSA__c fields and triggers a Record-Triggered Flow for post-signing actions. Approval Process on GSA__c for Group_Size__c > 1000 (updated threshold) routes to Underwriting_Analyst_Queue before Integration Procedure proceeds to DocuSign step. Custom Label stores the 7-day signing deadline for configurability.

## Description

As a Provider Network Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces active Provider_Network_Agreement__c (PNA) details and associated Fee_Schedule__c line summaries so that I can assess a provider's contract status and reimbursement rates without leaving the provider record.

## Acceptance Criteria

Given a Provider Network Specialist views a Provider__c record,
When the Provider 360 FlexCard loads,
Then all Provider_Network_Agreement__c records where PNA.Provider__c = current Provider__c ID and PNA.Status__c = 'Active' must be listed with PNA.Effective_Date__c, PNA.Expiration_Date__c, PNA.Network__c, and PNA.Contract_Type__c visible.

Given an active PNA is displayed,
When the specialist expands a PNA row,
Then associated Fee_Schedule__c records linked via Fee_Schedule__c.PNA__c must render with Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Allowed_Amount__c, and Fee_Schedule__c.Effective_Date__c in a child data table.

Given a PNA is expiring within 90 days,
When the FlexCard evaluates PNA.Expiration_Date__c,
Then a warning badge 'Expiring Soon' must appear on that PNA row and the row must be highlighted in amber.

Given the provider has no active PNA,
When the FlexCard loads,
Then an empty-state message 'No Active Network Agreements on file' must display with a quick-action button to create a new PNA.

Given the FlexCard is viewed by a user without the 'Provider_Network_Specialist' Permission Set,
When the component loads,
Then Fee_Schedule__c.Allowed_Amount__c must be masked and display as '***' to enforce rate confidentiality.

## Solution

Build OmniStudio FlexCard 'Provider_360_PNA_FC' with two DataRaptor Extract actions: one for PNA records and one for Fee_Schedule__c records triggered on row expand state action. Conditional styling rules in FlexCard handle the amber highlight and expiry badge logic using a calculated field (TODAY() - PNA.Expiration_Date__c < 90). A custom Lightning Web Component (LWC) wrapper handles the Permission Set check via @wire(getPermissionSetAssignments) and conditionally renders masked vs. unmasked Allowed_Amount__c. Deployed to the Provider__c Lightning Record Page via App Builder.

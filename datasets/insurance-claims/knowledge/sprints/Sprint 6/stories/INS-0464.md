# INS-0464: Bug: Fee Schedule Effective Date Validation Fires Incorrectly on PNA Renewal Records

## Description

Steps to Reproduce:
1. Navigate to an existing Provider_Network_Agreement__c record with Status__c = 'Active'.
2. Click 'Renew PNA' action button to create a renewal PNA record.
3. On the new PNA record, link an existing Fee_Schedule__c where Fee_Schedule__c.Effective_Date__c equals the existing PNA's Fee_Schedule__c.Expiration_Date__c (i.e., contiguous date range).
4. Save the Fee_Schedule__c record.

Expected Behavior: Contiguous effective dates (where new Effective_Date__c = prior Expiration_Date__c) should be allowed on renewal PNA records without triggering a validation error.

Actual Behavior: Validation Rule 'FS_No_Overlapping_Dates' fires and throws error: 'Fee Schedule effective date overlaps with an existing active schedule.' blocking the save. This prevents all PNA renewals from being completed via the standard renewal flow.

Environment: Acme Insurance - Full Sandbox (Sprint 6)
Regression introduced in INS-0451 Fee Schedule Overlap Validation deployment.
Severity: High — blocks Provider Network Specialist renewal workflow entirely.

## Acceptance Criteria

As per Description. Validation Rule 'FS_No_Overlapping_Dates' must distinguish between overlapping dates and contiguous dates. A contiguous date where Fee_Schedule__c.Effective_Date__c = prior Fee_Schedule__c.Expiration_Date__c on a renewal PNA must save without error.

## Solution

Update Validation Rule 'FS_No_Overlapping_Dates' formula to use strict less-than operator instead of less-than-or-equal when comparing Fee_Schedule__c.Effective_Date__c against existing records' Expiration_Date__c. Update associated Apex test class 'FeeScheduleValidationTest' to add @IsTest scenarios for contiguous date ranges. Deploy fix to Full Sandbox for QA regression before promoting to UAT.

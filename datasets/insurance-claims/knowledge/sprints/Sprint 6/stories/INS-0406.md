## Description

As a Broker Relations Coordinator, I need a consolidated dashboard FlexCard on the Agent Portal that displays active policy counts, open claim summaries, and pending FNOL submissions per broker so that brokers can quickly assess their book of business without navigating multiple Salesforce objects.

## Acceptance Criteria

Given a licensed broker user logs into the Agent Portal,
When the Broker Dashboard FlexCard loads,
Then it must display the count of active Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.Broker__c = current user's associated Contact ID.

Given the FlexCard is rendered,
When claim data is fetched,
Then Claim__c records with Claim__c.Assigned_Broker__c = broker ID and Claim__c.Status__c IN ('Open', 'Pending Review', 'In Adjudication') must be grouped by LOB and displayed as tile counts.

Given a broker has pending FNOL submissions,
When the FNOL section renders,
Then all Claim__c records where Claim__c.FNOL_Submitted__c = TRUE and Claim__c.Status__c = 'FNOL Received' must appear in a scrollable list with Claim__c.Date_of_Loss__c and Claim__c.Claimant_Name__c visible.

-Given broker clicks a policy tile, the full Policy__c detail page opens in a new tab- (deprecated: replaced by side-panel modal per UX decision INS-0398)

Given a broker clicks any claim count tile,
When the drill-down action fires,
Then a filtered list view of Claim__c records scoped to that LOB and status bucket must render within the portal without full page reload.

Given the dashboard is loading data,
When the DataRaptor fetch exceeds 3 seconds,
Then a skeleton loading state must display and an error toast must appear if fetch fails after 2 retries.

## Solution

Build an OmniStudio FlexCard named 'Broker_Dashboard_FC' with multiple DataRaptor Extract (DRE) actions sourcing from Policy__c and Claim__c. Use a child FlexCard for FNOL list rendering. Apply a Custom Permission 'Broker_Portal_Access' via Permission Set Group to gate visibility. State actions trigger a drill-down OmniScript for claim list navigation. Cache DataRaptor responses using Platform Cache (session partition) with a 5-minute TTL to reduce SOQL load. Sharing Rules on Claim__c and Policy__c scoped to Broker__c lookup field ensure data isolation.

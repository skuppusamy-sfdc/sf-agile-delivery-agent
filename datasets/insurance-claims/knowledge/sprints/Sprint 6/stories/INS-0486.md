## Description

As a Provider Network Specialist, I need the Provider 360 FlexCard to display the provider's current Provider_Network_Agreement__c status, effective dates, and associated Fee_Schedule__c tier so that I can quickly assess a provider's contractual standing without opening multiple related records during a call or claims review.

## Acceptance Criteria

Given a Provider__c (Person Account) record is opened by a Provider Network Specialist,
When the Provider 360 FlexCard loads,
Then the card displays the most recent Provider_Network_Agreement__c where PNA__c.Status__c = 'Active' and PNA__c.Provider__c = current Provider__c.Id, showing fields: PNA__c.Contract_Start_Date__c, PNA__c.Contract_End_Date__c, PNA__c.Network_Tier__c, and PNA__c.TIN__c.

Given the provider has an active PNA with an associated Fee_Schedule__c,
When the FlexCard renders the contracting section,
Then Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.Rate_Tier__c are displayed in a nested card state.

Given the provider has no active PNA (PNA__c.Status__c != 'Active' or no PNA record exists),
When the FlexCard loads,
Then the contracting section displays a warning badge: 'No Active Network Agreement' in amber.

-Given the FlexCard displays all historical PNA records in a scrollable list- (deprecated: historical PNA list moved to separate tab component per UX review)

Given the provider's PNA__c.Contract_End_Date__c is within 90 days of today,
When the FlexCard renders,
Then a 'Renewal Approaching' alert banner is shown with the exact expiration date.

## Solution

Extend existing 'Provider360Card' OmniStudio FlexCard by adding a new card state 'ContractingState'. Create DataRaptor Extract 'DR_GetActivePNA' querying Provider_Network_Agreement__c with filter PNA__c.Status__c='Active' and relationship to Fee_Schedule__c via lookup. Add conditional visibility rule on warning badge using FlexCard state condition: PNA_Count = 0. Renewal alert uses calculated field comparing PNA__c.Contract_End_Date__c to TODAY() + 90 via DataRaptor formula. Deploy via OmniStudio version activation. Update Permission Set Group 'ProviderNetworkSpecialist_PSG' to include read access to Fee_Schedule__c fields.

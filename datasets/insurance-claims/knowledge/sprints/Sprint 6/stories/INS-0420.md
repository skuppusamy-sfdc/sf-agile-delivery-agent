## Description

As a Claims Operations Manager, I need the system to automatically generate an Explanation of Benefits (EOB) document and deliver it to the policyholder via email when a Claim__c transitions to 'Adjudicated' status so that policyholders receive timely, accurate EOB communications in compliance with state regulatory timelines. The EOB must summarize all Claim_Line__c records, billed vs. allowed amounts, and the associated Payment__c details.

## Acceptance Criteria

Given a Claim__c.Status__c is changed to 'Adjudicated',
When the Record-Triggered Flow fires,
Then an Explanation_of_Benefits__c record is created with EOB__c.Claim__c = the adjudicated claim ID, EOB__c.Policyholder__c = Claim__c.Policyholder__c, EOB__c.EOB_Date__c = TODAY(), and EOB__c.Delivery_Status__c = 'Pending'.

Given the EOB record is created,
When the Conga CLM merge action is invoked,
Then a PDF is generated from the 'EOB_Standard_Template' Conga template, populated with: Claim__c.Claim_Number__c, all related Claim_Line__c records (Billed_Amount__c, Allowed_Amount__c, Patient_Responsibility__c, CPT_Code__c, ICD10_Code__c), and Payment__c.Payment_Amount__c, Payment__c.Payment_Date__c, Payment__c.Check_Number__c.

Given the EOB PDF is generated,
When the Conga action completes successfully,
Then the PDF is attached to the EOB__c record as a Salesforce File (ContentDocument), and an email is sent to Claim__c.Policyholder__c.Email using the 'EOB_Delivery_Email_Alert' template with the PDF as an attachment.

Given the EOB email is successfully sent,
When the email delivery callback is received,
Then EOB__c.Delivery_Status__c is updated to 'Delivered' and EOB__c.Delivery_Timestamp__c is set to the current datetime.

Given the email delivery fails (invalid email address or bounce),
When the failure is detected,
Then EOB__c.Delivery_Status__c = 'Failed', a Case is auto-created with Subject = 'EOB Delivery Failed - ' + Claim__c.Claim_Number__c, and the case is assigned to the 'Claims_Operations_Queue'.

Given the EOB is generated,
When the claim has zero Claim_Line__c child records,
Then the EOB generation is blocked, an error log entry is written to a custom 'Integration_Error_Log__c' object, and the Claim__c.Status__c is rolled back to 'Under Review' with a system note appended to the chatter feed.

## Solution

Record-Triggered Flow 'EOB_Generation_Flow' triggers on Claim__c update where Status__c changes to 'Adjudicated'. Flow creates EOB__c record, then calls an Invocable Apex method 'CongaEOBMergeAction.cls' that invokes Conga CLM REST API with the EOB template ID and Claim__c record ID. Conga callback handled via a named Apex REST endpoint '/services/apexrest/CongaCallback' that updates EOB__c.Delivery_Status__c. Email delivery uses Salesforce standard EmailMessage object linked to EOB__c. Rollback logic implemented in a separate Apex class 'EOBValidationService.cls' called before flow proceeds, checking COUNT(Claim_Line__c) > 0.

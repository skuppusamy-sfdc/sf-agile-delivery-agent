## Description

As a Broker Relations Coordinator, I need a consolidated FlexCard dashboard on the Agent Portal that displays an active policy count, open claims summary, and pending FNOL submissions grouped by LOB so that brokers can quickly assess their book of business without navigating multiple Salesforce tabs.

## Acceptance Criteria

Given a licensed broker logs into the Agent Portal,
When the dashboard loads,
Then a FlexCard titled 'My Book of Business' displays:
  - Total active Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.Broker__c = current user's associated Contact
  - Count of open Claim__c records where Claim__c.Status__c IN ('FNOL Submitted', 'Under Review', 'Pending Documentation')
  - Breakdown of claims by Claim__c.Line_of_Business__c (LOB) rendered as a horizontal bar chart
  - Last refreshed timestamp displayed in the card footer

Given the broker has zero active policies,
When the dashboard loads,
Then the FlexCard renders an empty-state illustration with the message 'No active policies found. Contact your Broker Relations Coordinator.'

Given the broker clicks a LOB segment in the bar chart,
When the drill-down action fires,
Then the portal navigates to a filtered Claim__c list view scoped to that LOB and the broker's TIN on file in the associated Provider__c record

-Given the dashboard auto-refreshes every 60 seconds via polling- (deprecated — replaced by Platform Event subscription per INS-0451)
Given a ClaimStatusChanged__e Platform Event is published,
When the event is received by the broker's active session,
Then the FlexCard claim counters update without a full page reload

Given the broker's Permission Set Group does not include 'Broker_Portal_Analytics',
When the dashboard loads,
Then the bar chart section is hidden and a lock icon with tooltip 'Upgrade your portal access to view analytics' is displayed

## Solution

Build OmniStudio FlexCard 'BrokerDashboard_FC' with two DataRaptor Extract actions: (1) DR_GetBrokerActivePolicies querying Policy__c filtered by broker Contact lookup and Status__c = 'Active'; (2) DR_GetBrokerOpenClaims aggregating Claim__c by Line_of_Business__c. Embed a child FlexCard for the LOB bar chart using Chart.js integration. Subscribe to ClaimStatusChanged__e Platform Event via an Aura-enabled LWC wrapper that calls refreshApex on event receipt. Gate analytics section via Permission Set Group 'Broker_Portal_Analytics' checked through a custom @wire getPermissions Apex method. Deploy via OmniStudio deployment package targeting the Experience Cloud Agent Portal site.

## Description

As a Broker Relations Coordinator, I need brokers to submit a First Notice of Loss (FNOL) directly through the Agent Portal OmniScript for Property & Casualty policies so that claim intake time is reduced and manual data entry errors are eliminated. The OmniScript must create a Claim__c record in 'FNOL Received' status and trigger downstream assignment workflows.

## Acceptance Criteria

Given a broker initiates a new FNOL from the portal,
When the OmniScript step 1 loads,
Then the broker must select a Policy__c record from a picklist filtered to Policy__c.Status__c = 'Active' AND Policy__c.LOB__c = 'Property & Casualty' AND Policy__c.Broker_of_Record__c = authenticated broker.

Given the policy is selected,
When the broker advances to step 2,
Then the OmniScript auto-populates Claim__c.Policyholder__c, Claim__c.Policy_Effective_Date__c, and Claim__c.Coverage_Type__c from the parent Policy__c via DataRaptor Transform.

Given the broker completes all required fields (Date_of_Loss__c, Loss_Description__c, Loss_Location__c, Estimated_Damage__c),
When the broker submits the form,
Then a Claim__c record is created with Claim__c.Status__c = 'FNOL Received', Claim__c.Source__c = 'Broker Portal', and a system-generated Claim__c.Claim_Number__c following pattern 'PNC-{YYYY}-{00000}'.

Given the Claim__c record is created,
When the record is saved,
Then a Record-Triggered Flow fires to create a Task for the Assignment Coordinator with Subject = 'New FNOL - Assign Adjuster' and ActivityDate = TODAY() + 1 business day.

Given the FNOL is submitted successfully,
When the OmniScript reaches the confirmation step,
Then a DocuSign envelope is NOT triggered at this stage — acknowledgment is sent via email alert only. (Note: DocuSign integration deferred to Phase 2 per INS-0401 decision log.)

Given a broker partially completes the FNOL and navigates away,
When the broker returns to the portal within 24 hours,
Then the OmniScript resumes from the saved checkpoint using OmniScript Resume functionality with all previously entered data intact.

## Solution

Build OmniScript 'FNOL_Submission_PC_OS' with 6 steps: Policy Lookup, Incident Details, Loss Location, Damage Estimation, Document Upload (optional), and Confirmation. Use DataRaptor Load action to create Claim__c on submit. Implement OmniScript Save & Resume using a custom Save Action DataRaptor writing to a staging Custom Object 'FNOL_Draft__c'. Record-Triggered Flow 'FNOL_Assignment_Task_Flow' fires on Claim__c insert where Source__c = 'Broker Portal'. Auto-number format configured on Claim__c.Claim_Number__c via custom Apex auto-number trigger to support LOB prefix logic.

## Description

As a Broker Relations Coordinator, I need a consolidated FlexCard dashboard on the Agent/Broker Portal that surfaces active policy counts, open FNOL submissions, and pending claims by LOB so that brokers can self-serve status updates without calling Claims Operations. This story extends INS-0384 (Broker Portal Authentication & SSO setup).

## Acceptance Criteria

Given a licensed broker logs into the Agent/Broker Portal via SSO,
When the dashboard loads,
Then a FlexCard displays the count of active Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.Broker_NPN__c matches the authenticated broker's NPN.

Given the broker dashboard is rendered,
When the 'Open Claims' tile is selected,
Then a child FlexCard lists all Claim__c records where Claim__c.Status__c IN ('FNOL Received', 'Under Review', 'Pending Documentation') and Claim__c.Broker_of_Record__c = broker's Contact ID, sorted by Claim__c.Date_of_Loss__c descending.

Given a claim row is displayed on the dashboard,
When the broker clicks 'View Details',
Then the broker is navigated to the Claim 360 OmniScript detail view showing Claim__c.Claim_Number__c, Claim__c.LOB__c, Claim__c.Total_Incurred__c, and Claim__c.Assigned_Adjuster__c.

Given the dashboard is loaded,
When claims data is retrieved,
Then the FlexCard DataRaptor uses a SOQL source filtered by Permission Set Group 'Broker_Portal_User' to enforce row-level security — brokers may NOT view claims outside their book of business.

-Given the broker dashboard loads, When no active policies exist, Then display a static 'No Active Policies' placeholder image.- (Superseded: replaced with dynamic empty-state component per UX revision 2.3)

Given the dashboard is viewed on a mobile viewport (< 768px),
When the FlexCard renders,
Then all tiles collapse into a single-column stacked layout without horizontal scrolling.

## Solution

Build OmniStudio FlexCard 'Broker_Dashboard_FC' with two child cards: 'Active_Policies_FC' and 'Open_Claims_FC'. Use DataRaptor Extract actions sourcing from Policy__c and Claim__c with dynamic filter tokens bound to the authenticated user's Contact.NPN__c. Apply Permission Set Group 'Broker_Portal_User' to restrict data visibility. Navigation action on claim row triggers OmniScript 'Claim_360_Detail_OS'. Responsive layout configured via FlexCard card layout settings with breakpoint overrides. Cache DataRaptor results with a 5-minute TTL using Platform Cache Partition 'BrokerPortalCache'.

## Description

As a Provider Network Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record that surfaces the active Provider_Network_Agreement__c, associated Fee_Schedule__c effective dates, and current credentialing status from Credentialing_Case__c, so that I have a single-pane-of-glass view without navigating multiple related lists.

## Acceptance Criteria

Given a Provider__c record is opened by a user with the 'Provider_Network_Specialist' Permission Set,
When the Provider 360 FlexCard renders,
Then it displays the most recent PNA where Provider_Network_Agreement__c.Status__c = 'Active' and Provider_Network_Agreement__c.Provider__c = current Provider__c.Id.

Given an active PNA exists,
When the Fee Schedule section of the FlexCard loads,
Then Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.Expiration_Date__c, and Fee_Schedule__c.LOB__c are displayed for all Fee_Schedule__c records linked to the PNA.

Given a Credentialing_Case__c exists for the provider,
When the credentialing tile renders,
Then Credentialing_Case__c.Status__c and Credentialing_Case__c.Expiration_Date__c are shown with a visual indicator: green if > 90 days to expiration, yellow if 31–90 days, red if ≤ 30 days.

Given no active PNA exists for a provider,
When the FlexCard loads,
Then a 'No Active Agreement' badge is displayed and a 'Initiate Contracting' action button is visible that launches the Provider Contracting OmniScript.

-Given the FlexCard includes a 'Send DocuSign Envelope' button for PNA renewal- (deprecated — DocuSign action moved to NetworkContracting dedicated component in INS-0447)

## Solution

Build OmniStudio FlexCard 'Provider360Card' using DataRaptor Extract queries across Provider_Network_Agreement__c, Fee_Schedule__c, and Credentialing_Case__c. Use conditional rendering states for credentialing expiry color coding via FlexCard State Actions. Add Custom Action invoking OmniScript 'InitiateProviderContracting'. Deploy on Provider__c Lightning record page via App Builder. Assign visibility via Permission Set Group 'Provider_Network_PSG'.

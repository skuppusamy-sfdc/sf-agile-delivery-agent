## Description

As a Claims Operations Manager, I need the system to automatically generate an Electronic Remittance Advice (ERA/835 EDI transaction set) and deliver it to the provider's designated clearinghouse endpoint when a Payment__c record is posted so that providers receive standardized remittance data that reconciles against their Explanation of Benefits and reduces manual payment posting effort on their end.

## Acceptance Criteria

Given a Payment__c record is created with Payment__c.Status__c = 'Posted' and Payment__c.Payment_Type__c = 'EFT',
When the Record-Triggered Flow 'Payment_Posted_ERA_Flow' fires on Payment__c insert/update,
Then an Apex queueable job 'ERA835GeneratorQueueable' is enqueued with the Payment__c.Id as input.

Given the ERA835GeneratorQueueable job executes,
When it builds the 835 transaction set,
Then the file includes: ISA/GS envelope segments populated from org-level Custom Metadata 'ERA_Config__mdt', CLP segments per Claim__c linked via Payment__c.Claim__c, SVC segments per Claim_Line__c with CPT code from Claim_Line__c.CPT_Code__c, CAS adjustment segments using Claim_Line__c.Adjustment_Reason_Code__c, and NM1 provider segment using Provider__c.NPI__c and Provider__c.TIN__c.

Given the 835 file is generated,
When the file is ready for delivery,
Then it is stored as a ContentVersion with ContentVersion.Title = 'ERA_835_' + Payment__c.Payment_Number__c + '_' + TODAY() and linked to the Payment__c record via ContentDocumentLink.

Given the Provider__c has a Clearinghouse_Endpoint__c field populated on their active PNA__c record,
When the ERA file is generated,
Then a callout is made to the configured clearinghouse SFTP endpoint using Named Credential 'ClearinghouseSFTP_NC' to deliver the file.

Given the SFTP delivery callout fails (timeout or 4xx/5xx response),
When the error is caught,
Then Payment__c.ERA_Delivery_Status__c is set to 'Delivery Failed', a Case is auto-created with Subject = 'ERA Delivery Failure - ' + Payment__c.Payment_Number__c and assigned to the Claims Operations queue.

Given the ERA file is successfully delivered,
When the callout returns HTTP 200,
Then Payment__c.ERA_Delivery_Status__c = 'Delivered', Payment__c.ERA_Delivered_Date__c = TODAY(), and an Explanation_of_Benefits__c record is created with EOB__c.Payment__c = Payment__c.Id and EOB__c.Status__c = 'Sent'.

-Given ERA files are batched and delivered nightly via scheduled job- (deprecated: moved to real-time delivery per Claims Ops requirement change in Sprint 5 planning)

## Solution

Record-Triggered Flow on Payment__c (after save, insert/update) checks Status__c='Posted' and Payment_Type__c='EFT', then calls Apex Action to enqueue ERA835GeneratorQueueable. Queueable class builds 835 EDI string using X12 EDI builder utility class, queries related Claim__c, Claim_Line__c, Provider__c, and PNA__c records. File stored via ContentVersion insert. SFTP delivery via HttpCallout using Named Credential. Error handling updates Payment__c fields via additional DML and creates Case via Case insert. EOB__c creation on success. Custom Metadata Type 'ERA_Config__mdt' stores ISA sender/receiver IDs and version identifiers. All logic covered by Apex unit tests with mock HttpCallout.

## Description

As a policyholder accessing the Acme Insurance self-service Experience Cloud portal, I need to view my Explanation of Benefits (EOB) documents for processed claims so that I can understand how my claims were adjudicated, what amounts were paid to providers, and what my remaining patient responsibility is — without needing to call the member services line. Extends INS-0401 (Policyholder Experience Cloud Site Setup).

## Acceptance Criteria

Given an authenticated policyholder user accesses the 'My Claims' section of the Experience Cloud portal,
When the EOB list view loads,
Then all Explanation_of_Benefits__c records where EOB.Policy__c = the user's linked Policy__c ID and EOB.Status__c = 'Final' must be displayed, sorted by EOB.EOB_Date__c descending.

Given the EOB list renders,
When each EOB row is displayed,
Then the following fields must be visible: EOB.Claim_Number__c (from parent Claim__c), EOB.EOB_Date__c, EOB.Total_Billed__c, EOB.Total_Allowed__c, EOB.Plan_Paid_Amount__c, EOB.Patient_Responsibility__c, and EOB.Provider_Name__c.

Given the policyholder clicks 'View EOB' on a row,
When the detail view opens,
Then a Conga-generated PDF of the EOB must be retrievable from the associated ContentVersion record linked via EOB.EOB_Document_ID__c; the PDF must open in a new browser tab.

Given an EOB is in 'Draft' or 'Voided' status,
When the EOB list query runs,
Then those records must be excluded from the portal view (EOB.Status__c filter strictly = 'Final').

-Given the policyholder downloads the EOB PDF, a download event must be logged to a custom Audit_Log__c object- (deprecated: audit logging deferred to Compliance sprint; replaced by standard Salesforce field history tracking on EOB.Last_Accessed__c)

Given the policyholder has no final EOB records,
When the list view renders,
Then an empty-state message must display: 'No Explanation of Benefits documents are available at this time. Processed claims may take 3–5 business days to appear.'

Given the portal is accessed by a guest or unauthenticated user,
When any EOB component attempts to load,
Then the component must redirect to the portal login page and no EOB data must be exposed; sharing rules on Explanation_of_Benefits__c must enforce owner-based access scoped to the policyholder's Experience Cloud user.

## Solution

Build an LWC 'eobListView' deployed to the Experience Cloud portal page via App Builder. Use @wire(getRecord) with a custom Apex @AuraEnabled(cacheable=true) method EobPortalController.getFinalisedEobs(policyId) that enforces WITH SECURITY_ENFORCED in SOQL and filters EOB.Status__c = 'Final'. A separate Apex method EobPortalController.getEobDocumentUrl(eobId) retrieves the ContentVersion download URL via ContentDocumentLink. Sharing Rules on Explanation_of_Benefits__c use a Guest User Sharing Rule scoped to portal users' Contact.AccountId matching Policy__c.Account__c. Conga Composer is triggered server-side during EOB finalization (via Record-Triggered Flow on Claim__c.Status__c = 'Adjudicated') to pre-generate and attach the PDF, not on-demand, to avoid portal callout limits.

## Description

As a Broker Relations Coordinator, I need an end-to-end broker onboarding OmniScript that collects broker NPI, TIN, and licensing information, creates the associated Account and Contact records, and triggers a DocuSign envelope for Broker Agreement execution, so that new broker onboarding time is reduced from 5 business days to same-day digital completion.

## Acceptance Criteria

Given a Broker Relations Coordinator initiates the 'Onboard New Broker' OmniScript,
When the NPI field is submitted,
Then an NPPES API callout validates the NPI and pre-populates Contact.FirstName, Contact.LastName, Contact.NPI__c, and Account.Name from the NPPES response.

Given NPI validation succeeds,
When the TIN is entered,
Then a duplicate check runs against existing Account.TIN__c values and if a match is found, the OmniScript halts and displays 'A broker record with this TIN already exists: [Account Name]' with a link to the existing record.

Given all required fields are collected (NPI, TIN, License_Number__c, License_State__c, License_Expiration_Date__c),
When the broker submits the final OmniScript step,
Then a new Account (RecordType = 'Broker_Agency') and Contact (RecordType = 'Broker') are created and a Broker_Agreement__c record is created with Broker_Agreement__c.Status__c = 'Pending Signature'.

Given the Broker_Agreement__c record is created,
When the DocuSign envelope trigger fires via Platform Event 'Broker_Onboarding_Complete__e',
Then a DocuSign envelope is sent to Contact.Email containing the standard Broker Participation Agreement template and Broker_Agreement__c.DocuSign_Envelope_ID__c is populated.

Given the broker completes DocuSign signing,
When the DocuSign status webhook callback is received,
Then Broker_Agreement__c.Status__c is updated to 'Active', Broker_Agreement__c.Execution_Date__c is stamped, and a Welcome Email is dispatched via Salesforce Email Alert.

Given the OmniScript is accessed by a user without the 'Broker_Recruitment_Access' Custom Permission,
When they navigate to the OmniScript URL directly,
Then access is denied and they are redirected to the standard home page.

## Solution

Build OmniScript 'BrokerOnboarding' with Integration Procedure calling NPPES REST API for NPI lookup. Use DataRaptor Transform for field mapping to Account and Contact creation. Publish Platform Event 'Broker_Onboarding_Complete__e' on final step; subscribe via Apex Trigger on Broker_Agreement__c to invoke DocuSign API callout class 'DocuSignBrokerEnvelopeService'. Build Record-Triggered Flow on Broker_Agreement__c to handle DocuSign webhook status updates. Assign via Permission Set Group 'Broker_Recruitment_PSG'.

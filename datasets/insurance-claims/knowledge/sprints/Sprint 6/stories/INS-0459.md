## Description

As a Claims Operations Manager, I need an Agentforce copilot action embedded on the Claim__c record page that analyzes claim attributes and historical patterns to generate an SIU (Special Investigations Unit) referral recommendation with a confidence score, so that claims adjusters can make faster, data-informed decisions on potentially fraudulent claims without manually cross-referencing multiple data points.

## Acceptance Criteria

Given a Claim__c record with Status__c IN ('FNOL Received','Under Review') is open in the internal app,
When the adjuster invokes the 'Analyze for SIU Referral' Agentforce copilot action,
Then the action retrieves Claim__c.Loss_Type__c, Claim__c.Total_Claimed_Amount__c, Claim__c.Date_of_Loss__c, related Claim_Line__c CPT codes, and the claimant's prior Claim__c history (last 24 months) to generate a fraud risk score.

Given the copilot action completes analysis,
When the risk score is computed,
Then the copilot surfaces a natural language summary in the Agentforce side panel stating: recommended action ('Refer to SIU' / 'No Referral Needed' / 'Monitor'), confidence percentage, and the top 3 contributing risk factors (e.g., 'Claim filed within 30 days of policy inception', 'CPT code frequency anomaly', 'Claimant has 3+ claims in 12 months').

Given the adjuster accepts the 'Refer to SIU' recommendation,
When the adjuster clicks 'Create SIU Referral' in the copilot panel,
Then Claim__c.SIU_Referral_Status__c is set to 'Referred', Claim__c.SIU_Referral_Date__c = today, and a new Case (RecordType = 'SIU Investigation') is created linked to the Claim__c with the AI-generated summary pre-populated in Case.Description.

Given the adjuster rejects the recommendation,
When the adjuster selects 'Dismiss',
Then Claim__c.AI_Triage_Override__c = true and Claim__c.AI_Override_Reason__c is required (picklist: 'Insufficient Evidence', 'Known Provider', 'Adjuster Judgment') for audit compliance.

Given any copilot action invocation,
When the analysis runs,
Then the prompt, model response, confidence score, and user action are logged to a custom AI_Audit_Log__c object with AI_Audit_Log__c.Claim__c, AI_Audit_Log__c.User__c, AI_Audit_Log__c.Timestamp__c, and AI_Audit_Log__c.Action_Taken__c for Compliance Officer review.

Given the Agentforce model,
When evaluated against a test dataset of 500 historical SIU referrals,
Then precision must be >= 78% and recall >= 72% before deployment to production.

## Solution

Agentforce Copilot Action ('SIU_Triage_Action') built using a custom Apex Copilot Action class (SIUTriageCopilotAction implements CopilotAction). Action invokes an Einstein Prediction Service model trained on historical Claim__c, Claim_Line__c, and SIU Case data exported via Data Cloud. Prompt template constructed using Agentforce Prompt Builder with dynamic merge fields for claim attributes. Copilot response rendered in the Einstein Copilot side panel on the Claim__c Lightning Record Page. 'Create SIU Referral' button triggers a Screen Flow (SIU_Referral_Flow) that creates the Case and updates Claim__c fields. AI_Audit_Log__c inserts handled by an after-action Apex invocable method called from the flow. Permission Set Group 'Claims_Adjuster_Agentforce' required to invoke the action.

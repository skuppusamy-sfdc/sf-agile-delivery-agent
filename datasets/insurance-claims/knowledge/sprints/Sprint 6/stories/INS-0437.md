## Description

As a Broker Relations Coordinator, I need a consolidated FlexCard dashboard on the Agent Portal that surfaces active Policy__c counts, open Claim__c statuses, and pending FNOL submissions tied to the broker's book of business so that brokers can self-serve key client data without contacting the operations team.

## Acceptance Criteria

Given a licensed broker logs into the Agent Portal Experience Cloud site,
When the broker's dashboard loads,
Then a FlexCard displays the total count of active Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.Broker_TIN__c matches the logged-in broker's TIN.

Given active policies are displayed,
When the broker views the Claims Summary panel,
Then Claim__c records are grouped by Claim__c.Status__c with counts for 'Open', 'In Review', 'Adjudicated', and 'Closed' statuses visible.

Given a broker has pending FNOL submissions,
When the FNOL tile renders,
Then Claim__c records where Claim__c.FNOL_Submitted__c = TRUE and Claim__c.Status__c = 'Pending Intake' are listed with Claim__c.Date_of_Loss__c and Claim__c.Claimant_Name__c visible.

Given the broker clicks a policy count tile,
When the drill-down action fires,
Then the broker is navigated to a filtered list view of Policy__c records scoped to their book of business via a custom Permission Set Group 'Broker_Portal_Access'.

-Given a broker can view Payment__c disbursement totals on the dashboard (deprecated — moved to INS-0441)-

Given the dashboard loads on a mobile viewport (< 768px),
When the FlexCard renders,
Then all tiles stack vertically and no horizontal scroll is introduced.

## Solution

Build OmniStudio FlexCard 'Broker_Dashboard_FC' with three child cards: ActivePolicies, ClaimsSummary, PendingFNOL. Use DataRaptor Extract DRX_BrokerPolicySummary to aggregate Policy__c and Claim__c counts filtered by logged-in user's Contact.TIN__c via a custom merge field. Deploy FlexCard on the Experience Cloud home page via a Lightning App Builder page assigned to the 'Broker' profile. Apply Sharing Rules on Policy__c to grant read access to broker Community users based on Policy__c.Broker_Account__c lookup. Permission Set Group 'Broker_Portal_Access' gates the drill-down navigation action.

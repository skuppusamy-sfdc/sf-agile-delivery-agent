## Description

As a Broker Relations Coordinator, I need a digital broker onboarding OmniScript that captures new broker application details, creates a linked Account and Contact in Salesforce, and automatically routes a broker agreement for e-signature via DocuSign so that we can reduce broker recruitment cycle time and eliminate paper-based onboarding processes.

## Acceptance Criteria

Given an internal Broker Relations Coordinator accesses the 'New Broker Application' OmniScript,
When Step 1 loads,
Then the form captures: Broker Legal Name, DBA Name, NPI (if applicable), TIN, State License Number, License Expiration Date, LOB Specializations (multi-select), and Primary Contact details.

Given the coordinator completes Step 1 and advances,
When Step 2 loads,
Then a duplicate check Integration Procedure queries existing Account records by TIN (Account.TIN__c) and State License Number (Account.License_Number__c); if a match is found, the OmniScript displays 'Potential Duplicate Broker Account Found: [Account.Name]' with options to link to existing or proceed as new.

Given no duplicate is found and the coordinator completes all steps,
When the OmniScript is submitted,
Then a new Account (RecordType = 'Broker') and Contact are created in Salesforce with Account.Status__c = 'Pending Agreement' and the Contact is linked as the primary broker contact.

Given the Account and Contact are created,
When Account.Status__c = 'Pending Agreement',
Then a DocuSign envelope containing the standard Broker Participation Agreement template is sent to the broker's email with a 7-day expiration.

Given the broker completes the DocuSign e-signature,
When the DocuSign completed webhook is received,
Then Account.Status__c is updated to 'Active — Pending Credentialing' and a Credentialing_Case__c is auto-created with Credentialing_Case__c.Type__c = 'Broker License Verification' and Credentialing_Case__c.Status__c = 'Open'.

-Given the OmniScript includes a Workday HR system lookup to verify coordinator identity- (deprecated — Workday SSO handles identity; removed from scope per Security Architect review)

Given the DocuSign envelope expires without completion (7 days),
When the expiration webhook is received,
Then Account.Status__c = 'Application Expired' and an automated email is sent to the Broker Relations Coordinator queue notifying of the lapse.

## Solution

Multi-step OmniScript ('BrokerOnboardingApp') with DataRaptor Transform for Account/Contact creation. Duplicate detection via Integration Procedure querying Account object. DocuSign envelope triggered by a Record-Triggered Flow on Account (After Insert, Status__c = 'Pending Agreement') using the DocuSign Apex Toolkit with a named Conga CLM template. DocuSign webhook updates handled via Platform Event (DocuSign_Status_Update__e) subscriber flow updating Account.Status__c and creating Credentialing_Case__c. Expiration handling via a Scheduled Flow running daily querying DocuSign envelope records past expiry. Permission Set Group ('Broker_Recruiter_PSG') controls access.

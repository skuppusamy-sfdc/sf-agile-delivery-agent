## Description

As a Broker Relations Coordinator, I need a self-service workflow on the Agent/Broker Portal that allows a Group/Employer administrator to review and eSign a Group_Service_Agreement__c (GSA) document via DocuSign, so that employer group onboarding cycle time is reduced and wet-signature paper processes are eliminated.

## Acceptance Criteria

Given a Group_Service_Agreement__c record has been created with GSA__c.Status__c = 'Pending Signature' and GSA__c.DocuSign_Envelope_Id__c is null,
When the Broker Relations Coordinator clicks 'Send for eSignature' on the portal,
Then a DocuSign envelope must be generated using the mapped GSA template, sent to the email address on GSA__c.Primary_Signatory_Email__c, and GSA__c.DocuSign_Envelope_Id__c must be populated with the returned envelope ID.

Given the DocuSign envelope is sent,
When the Group/Employer administrator completes the eSignature,
Then a DocuSign Connect webhook callback must update GSA__c.Status__c to 'Executed', GSA__c.Executed_Date__c to the signature timestamp, and attach the signed PDF to the GSA__c record as a Salesforce File (ContentVersion).

Given the DocuSign envelope is declined or voided by the signer,
When the webhook callback is received,
Then GSA__c.Status__c must be set to 'Signature Declined' and a Task must be auto-created and assigned to the owning Broker Relations Coordinator with Subject 'GSA eSignature Declined — Follow Up Required'.

Given a GSA__c record reaches GSA__c.Executed_Date__c,
When the status transitions to 'Executed',
Then a Record-Triggered Flow must automatically create associated Fee_Schedule__c and Policy__c enrollment records as defined in GSA__c.Coverage_Tiers__c configuration.

-Given the broker can manually upload a countersigned PDF to bypass DocuSign- (deprecated — all GSA executions must go through DocuSign per legal mandate, manual bypass removed)

Given the DocuSign API returns an error during envelope creation,
When the error is caught,
Then GSA__c.DocuSign_Error_Message__c must be populated, an error notification must be sent to the Broker Relations Coordinator via Salesforce Bell Notification, and the Send action must remain available for retry.

## Solution

Implement DocuSign for Salesforce managed package integration. Create custom Apex class 'GSADocuSignEnvelopeService' extending DocuSign API wrapper to generate envelope from Custom Metadata Type 'DocuSign_Template_Mapping__mdt' (keyed by GSA__c.LOB__c). Screen Flow 'SendGSAForSignature_SF' exposed on portal as a quick action. DocuSign Connect webhook handled by Apex REST resource 'DocuSignWebhookHandler' which processes envelope status callbacks and updates GSA__c via a Record-Triggered Flow 'GSAStatusSync_RTF'. Signed PDF attached using ContentVersion insert linked to GSA__c. Downstream Fee_Schedule__c and Policy__c creation handled by after-update Record-Triggered Flow 'GSAExecutedProvisioningFlow' conditioned on Status__c = 'Executed'. Bell notifications via Salesforce Notification Builder.

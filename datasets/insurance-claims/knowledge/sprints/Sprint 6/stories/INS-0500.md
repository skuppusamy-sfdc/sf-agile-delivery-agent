## Description

As a Broker Relations Coordinator, I need a consolidated dashboard FlexCard on the Agent/Broker Portal that surfaces real-time policy and claims activity for all policyholders under a broker's book of business, so that brokers can proactively manage renewals, open claims, and client service requests without requiring a full Salesforce license.

This story extends INS-0470 (Broker Portal Authentication & SSO) and builds upon the Experience Cloud site scaffolding established in that sprint.

## Acceptance Criteria

Given a licensed broker logs into the Experience Cloud portal,
When the Broker Dashboard page loads,
Then a FlexCard titled 'My Book of Business' renders displaying aggregated counts of: active Policy__c records (Policy__c.Status__c = 'Active'), open Claim__c records (Claim__c.Status__c IN ('Submitted','In Review','Pending Information')), and policies within 90 days of Policy__c.Expiration_Date__c.

Given the broker views the 'Active Policies' tile,
When they click 'View All',
Then a DataRaptor-backed list displays Policy__c.Policy_Number__c, Policy__c.Policyholder__c (lookup to Account), Policy__c.LOB__c, Policy__c.Premium_Amount__c, and Policy__c.Expiration_Date__c filtered by Policy__c.Broker__c = current user's Contact.Account.

Given the broker views the 'Open Claims' tile,
When a Claim__c record has Claim__c.Status__c = 'Pending Information',
Then the FlexCard row renders a highlighted amber indicator and a 'Needs Attention' badge alongside Claim__c.Claim_Number__c and Claim__c.Date_of_Loss__c.

Given the broker dashboard is loaded on a mobile viewport (< 768px),
When the FlexCard stack renders,
Then tiles reflow to single-column layout with no horizontal scroll and all action buttons remain tappable at minimum 44px height.

-Given the broker can edit Policy__c.Premium_Amount__c inline from the dashboard tile- (deprecated — write access removed per security review, read-only view only)

Given a broker has zero policies assigned (Policy__c.Broker__c is null or unmatched),
When the dashboard loads,
Then an empty-state illustration and message 'No policies found for your account. Contact your Broker Relations Coordinator.' is displayed instead of empty tiles.

## Solution

Implement OmniStudio FlexCard 'BrokerDashboardCard' with four child DataRaptor Extract actions querying Policy__c and Claim__c with SOQL filters scoped to the running user's broker Account via the Community User's Contact.AccountId. Use OmniStudio DataRaptor Transform to aggregate counts server-side. FlexCard conditional rendering (Show/Hide conditions) drives the amber badge logic. Responsive layout configured via FlexCard Layout breakpoints. Deploy via OmniStudio deployment tool to the 'Broker Portal' Experience Cloud site. Permission Set Group 'Broker_Portal_User_PSG' grants read-only FLS on all referenced fields. A Named Credential is not required as all data is internal Salesforce objects.

## Description

As a Broker Relations Coordinator, I need the broker onboarding workflow to automatically generate and send a Broker Service Agreement via DocuSign eSignature when a new broker Account record reaches onboarding stage, so that agreement execution is fully digital, auditable, and reduces manual PDF handling that currently adds 3–5 business days to broker activation timelines.

## Acceptance Criteria

Given a broker Account record has Account.RecordType = 'Broker' and Account.Onboarding_Stage__c is updated to 'Agreement Pending',
When the Record-Triggered Flow fires,
Then a DocuSign envelope is automatically generated using the 'Broker_Service_Agreement_v3' template and sent to the email address on Account.Broker_Primary_Email__c with a 7-day expiration.

Given the DocuSign envelope is sent,
When the broker completes signing,
Then Account.Onboarding_Stage__c is updated to 'Agreement Executed', Account.Agreement_Executed_Date__c is stamped with today's date, and the signed PDF is attached to the Account record as a Salesforce File with title 'Broker_Agreement_Signed_{Account.Name}_{Date}'.

Given the DocuSign envelope expires without signature (after 7 days),
When the expiration webhook fires,
Then Account.Onboarding_Stage__c is reset to 'Agreement Expired', a Task is created assigned to the owning Broker Relations Coordinator with Subject = 'Re-send Broker Agreement — ACTION REQUIRED', and an email alert is sent via Salesforce Email Alert template 'Broker_Agreement_Expired_Alert'.

Given a Broker Relations Coordinator is viewing the broker Account,
When they click the 'Resend Agreement' quick action,
Then a new DocuSign envelope void replaces the prior expired envelope and a new 7-day envelope is dispatched, logging a new DocuSign_Envelope__c child record with Envelope_Status__c = 'Sent'.

-Given the agreement also routes to the Compliance Officer for countersignature before sending to broker- (deprecated — countersignature removed from Phase 1 scope, deferred to Phase 2 per Compliance sign-off 2024-11-20)

## Solution

Implement a Record-Triggered Flow ('Broker_Agreement_DocuSign_Flow') on Account object, entry criteria Account.RecordType.Name = 'Broker' AND Account.Onboarding_Stage__c CHANGED TO 'Agreement Pending'. Flow calls the DocuSign for Salesforce managed package action 'Create Envelope from Template' with template ID mapped via Custom Metadata Type 'DocuSign_Template_Mapping__mdt'. DocuSign webhook callbacks handled by a REST API endpoint exposed via an Apex @RestResource class (DocuSignWebhookHandler.cls) that updates Account fields and creates Tasks. Custom object DocuSign_Envelope__c tracks envelope lifecycle. Permission Set 'Broker_Onboarding_Admin' grants access to quick action.

## Description

Steps to Reproduce:
1. Confirm a broker Experience Cloud user has the 'AgentPortal_PSG' Permission Set Group assigned and can successfully access the Agent Portal (INS-0461 FlexCard visible).
2. Allow the nightly Workday-to-Salesforce user sync integration (scheduled Apex batch 'WorkdayUserSyncBatch', runs at 02:00 UTC) to complete.
3. Log in as the same broker Experience Cloud user the following morning.
4. Observe the Agent Portal home page.

Expected Behavior: The broker's 'AgentPortal_PSG' Permission Set Group assignment is preserved after the Workday sync. Broker retains full Agent Portal access.

Actual Behavior: The 'AgentPortal_PSG' Permission Set Group is removed from the broker's User record after the sync completes. The broker sees a blank Agent Portal page with no FlexCard components rendered. Affected brokers must contact the Broker Relations Coordinator to have PSG manually reassigned.

Environment: Acme Insurance - Full Sandbox and Production
Frequency: 100% reproducible — affects all Experience Cloud broker users on every nightly sync cycle.
Root Cause Hypothesis: 'WorkdayUserSyncBatch' is executing a full User record upsert that overwrites the PermissionSetAssignment junction object, stripping non-Workday-managed PSG assignments.
Business Impact: All active brokers lose portal access each morning until manually remediated. Logged since Sprint 6 Day 3.

## Acceptance Criteria

As per Description. After the nightly WorkdayUserSyncBatch completes, all Experience Cloud broker users with a pre-existing 'AgentPortal_PSG' Permission Set Group assignment must retain that assignment. The fix must be verified across 3 consecutive nightly sync cycles in Full Sandbox before promotion to Production.

## Solution

Modify 'WorkdayUserSyncBatch' to query and cache existing PermissionSetAssignment records for affected User IDs before executing upsert operations. After upsert, re-apply any PSG assignments that were present pre-sync but not sourced from Workday. Alternatively, refactor sync to use field-level delta updates (only sync Workday-owned User fields) rather than full record upsert to prevent junction object collisions. Add post-sync Apex assertion in batch finish() method to validate 'AgentPortal_PSG' assignment count matches pre-sync baseline and publish a Platform Event 'Sync_Integrity_Alert__e' if discrepancy detected.

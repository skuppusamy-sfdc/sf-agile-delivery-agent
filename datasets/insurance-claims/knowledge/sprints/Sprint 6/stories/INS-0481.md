## Description

As a policyholder, I need an Explanation of Benefits (EOB) document to be automatically generated and made available for self-service download on the Agent Portal when my claim is adjudicated so that I can understand my benefit determination, provider payments, and any patient responsibility amounts without contacting the claims center.

## Acceptance Criteria

Given a Claim__c record transitions to Claim__c.Status__c = 'Adjudicated',
When the Record-Triggered Flow fires,
Then an Explanation_of_Benefits__c (EOB) record is created with EOB__c.Claim__c = Claim__c.Id, EOB__c.Generation_Date__c = today, EOB__c.Status__c = 'Generated', and EOB__c.Total_Billed_Amount__c, EOB__c.Total_Allowed_Amount__c, EOB__c.Patient_Responsibility__c populated from the adjudicated Claim_Line__c rollup fields.

Given the EOB__c record is created,
When the Conga document generation process runs,
Then a PDF is generated using the 'EOB_Standard_Template' and stored as a ContentVersion linked to both Claim__c and EOB__c via ContentDocumentLink.

Given the EOB PDF is stored,
When the policyholder logs into the Agent Portal and navigates to their claim,
Then the EOB is listed under 'Claim Documents' with EOB__c.Generation_Date__c and a 'Download EOB' button.

Given the policyholder clicks 'Download EOB',
When the download is initiated,
Then the PDF is served securely and an audit record is created on EOB__c.Last_Downloaded_Date__c = today and EOB__c.Download_Count__c is incremented by 1.

Given COB (Coordination of Benefits) applies to the claim,
When the EOB is generated,
Then EOB__c.COB_Applied__c = true and EOB__c.Secondary_Payer_Amount__c reflects the secondary insurer's payment amount from Claim__c.COB_Secondary_Payment__c.

Given the claim has multiple Claim_Line__c records,
When the EOB PDF is rendered,
Then each Claim_Line__c is listed with Claim_Line__c.CPT_Code__c, Claim_Line__c.Service_Date__c, Claim_Line__c.Billed_Amount__c, Claim_Line__c.Allowed_Amount__c, and Claim_Line__c.Denial_Reason__c (if applicable).

Given the EOB generation fails due to a Conga API error,
When the failure occurs,
Then EOB__c.Status__c = 'Generation Failed', a Platform Event 'EOB_Generation_Failed__e' is published, and an email alert is sent to the Claims Operations Manager queue.

## Solution

Implement a Record-Triggered Flow 'RTF_ClaimAdjudicated_EOBCreate' on Claim__c (After Update, condition: Status__c changed to 'Adjudicated') to create the EOB__c record and invoke an Apex action 'EOBDocumentGenerationAction' that calls the Conga CLM REST API asynchronously via a Queueable Apex class 'CongaEOBGenerationQueueable'. Store template IDs in Custom Metadata 'Conga_Template_Config__mdt'. On success, a second flow step updates EOB__c.Status__c = 'Generated' and fires an Email Alert to the policyholder. On failure, publish Platform Event 'EOB_Generation_Failed__e' consumed by a monitoring flow. Surface the EOB download LWC 'eobDownloadCard' on the Experience Cloud portal claim detail page with row-level sharing enforced via a Sharing Rule on EOB__c scoped to the policyholder's Contact.

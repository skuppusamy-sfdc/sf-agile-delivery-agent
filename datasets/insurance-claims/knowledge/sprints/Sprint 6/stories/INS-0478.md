## Description

As a Provider Network Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that consolidates active Provider_Network_Agreement__c (PNA) details, linked Fee_Schedule__c effective dates, and current Credentialing_Case__c status so that I can assess a provider's complete network standing from a single view without switching between multiple related lists.

## Acceptance Criteria

Given a Provider Network Specialist opens a Provider__c record,
When the Provider 360 FlexCard loads,
Then all Provider_Network_Agreement__c records where PNA__c.Provider__c = current Provider__c.Id AND PNA__c.Status__c = 'Active' are displayed with PNA__c.Contract_Start_Date__c, PNA__c.Contract_End_Date__c, and PNA__c.Network_Tier__c.

Given an active PNA exists for the provider,
When the Fee Schedule section renders,
Then the linked Fee_Schedule__c record is shown with Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.Expiration_Date__c, and Fee_Schedule__c.Schedule_Type__c (e.g., 'Medicare-Based', 'Negotiated').

Given the provider has a Credentialing_Case__c record,
When the Credentialing panel renders,
Then Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and Credentialing_Case__c.CAQH_Profile_ID__c are displayed.

Given a provider's Credentialing_Case__c.Expiration_Date__c is within 90 days of today,
When the FlexCard renders,
Then a visual warning badge (amber) displays: 'Credentialing expires in X days'.

-Given the FlexCard includes a NPPES verification status field, When the provider's NPI is validated, Then NPPES_Verified__c = true is shown as a green badge.- (Deprecated — NPPES integration moved to INS-0491)

Given no active PNA exists for the provider,
When the FlexCard loads,
Then the PNA section displays: 'No active network agreements found' with a quick-action button to create a new PNA.

## Solution

Build OmniStudio FlexCard 'Provider360_FC' with three card states: Active, Warning, No Agreement. Use three DataRaptor Extract actions: DR_GetActivePNA (queries Provider_Network_Agreement__c), DR_GetFeeSchedule (queries Fee_Schedule__c via PNA junction), DR_GetCredentialing (queries Credentialing_Case__c). Add a Formula element in the FlexCard for days-to-expiration calculation. Assign the FlexCard to the Provider__c Lightning Record Page via App Builder. Apply Field-Level Security via Permission Set 'Provider_Network_Specialist_PS' to restrict Credentialing_Case__c.CAQH_Profile_ID__c visibility.

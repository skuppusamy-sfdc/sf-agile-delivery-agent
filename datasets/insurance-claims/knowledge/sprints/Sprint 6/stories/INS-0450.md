## Description

**Bug Report — FNOL OmniScript Duplicate Submission**

**Environment:** Acme Insurance UAT Sandbox (CS112)
**Discovered During:** Sprint 6 UAT cycle for INS-0446

**Steps to Reproduce:**
1. Log in as a broker portal user.
2. Navigate to the Broker Dashboard and click 'Submit New Claim'.
3. Complete all steps of the FNOL OmniScript (Policy selection, loss details, confirmation).
4. On Step 3 (Review & Submit), click 'Submit'.
5. Observe the success confirmation screen.
6. Click the browser Back button.
7. Click 'Submit' again on the Review & Submit step.

**Expected Behavior:** The OmniScript should detect the previously submitted Claim__c (matching Policy__c.Id + FNOL_Date__c + Loss_Type__c) and display the duplicate warning message defined in INS-0446 AC5, blocking the second submission.

**Actual Behavior:** A second Claim__c record is created with Claim__c.Status__c = 'FNOL Received' and Claim__c.Submission_Source__c = 'Broker Portal', resulting in two identical Claim__c records. The duplicate detection Integration Procedure is NOT firing on the back-navigation resubmit path.

**Additional Notes:** The Platform Event (FNOL_Submitted__e) fires twice, triggering duplicate assignment routing tasks. Occurs in Chrome 124, Edge 122. Not reproducible in the OmniScript Preview tool (Studio), only in the deployed portal context.

## Acceptance Criteria

as per Description — duplicate Claim__c creation must be blocked on back-navigation resubmit; duplicate detection Integration Procedure must fire consistently regardless of navigation path; verified in UAT with broker portal user profile.

## Solution

Root cause investigation: OmniScript session state is not cleared after successful submission, allowing the Submit action to re-execute without re-running the pre-submit Integration Procedure step. Fix: (1) On the OmniScript 'Submit' action, set a hidden OmniScript element 'Submission_Token__c' (UUID generated at Step 1 load) and store it on the created Claim__c as Claim__c.Submission_Token__c (new custom field, Text 36). (2) Update the duplicate detection Integration Procedure to also query by Submission_Token__c. (3) Add a post-submit OmniScript action to clear/invalidate the session token. (4) Add an idempotency check in the Apex Action creating Claim__c to query existing records by token before insert. Deploy fix to UAT for re-test.

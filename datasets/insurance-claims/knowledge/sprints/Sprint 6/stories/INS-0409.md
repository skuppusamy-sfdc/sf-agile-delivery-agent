## Description

**Bug Report**

**Summary:** When a Claim__c record has Claim__c.COB_Active__c = TRUE and a secondary payer is identified, the adjudication engine incorrectly applies the full Fee_Schedule__c.Allowed_Amount__c to each Claim_Line__c instead of subtracting the secondary payer's paid amount (Claim__c.COB_Secondary_Paid__c), resulting in overpayment on Payment__c records.

**Steps to Reproduce:**
1. Create a Claim__c record with Claim__c.COB_Active__c = TRUE and Claim__c.COB_Secondary_Paid__c = 150.00.
2. Add two Claim_Line__c records each with Claim_Line__c.Billed_Amount__c = 300.00 and a linked Fee_Schedule__c with Allowed_Amount__c = 250.00.
3. Trigger adjudication via the 'Adjudicate Claim' flow action.
4. Observe Claim_Line__c.Adjudicated_Amount__c on both lines.

**Expected Result:** Claim_Line__c.Adjudicated_Amount__c should reflect Allowed_Amount__c minus the pro-rated COB_Secondary_Paid__c (i.e., 250.00 - 75.00 = 175.00 per line).

**Actual Result:** Claim_Line__c.Adjudicated_Amount__c = 250.00 on both lines; COB deduction is not applied. Payment__c.Total_Amount__c is overstated by the full COB_Secondary_Paid__c value.

**Environment:** Acme Insurance - Full Sandbox (CS87)
**Regression:** Introduced in Sprint 5 deployment (INS-0371 Adjudication Engine refactor)

## Acceptance Criteria

As per Description. Verify fix in Full Sandbox (CS87) with COB test cases covering single-line and multi-line claims. Confirm Payment__c.Total_Amount__c correctly reflects COB-adjusted totals post-fix.

## Solution

Root cause identified in AdjudicationService.cls — the COB deduction block was wrapped in a condition checking Claim__c.COB_Verified__c = TRUE rather than Claim__c.COB_Active__c = TRUE. The COB_Verified__c field is set asynchronously by a downstream integration and was not yet populated at adjudication time. Fix: update the condition to evaluate COB_Active__c and add a null-safe check for COB_Secondary_Paid__c before pro-ration calculation. Add a new @isTest class AdjudicationCOBTest with minimum 90% coverage. Deploy via change set to CS87 for QA validation before production.

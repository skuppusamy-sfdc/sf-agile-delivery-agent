## Description

As a broker, I need to view and download Explanation_of_Benefits__c (EOB) documents and reconcile Electronic Remittance Advice (ERA) data for claims within my book of business directly from the Agent Portal so that I can respond to client billing inquiries without requesting documents from the claims operations team. This story references the dashboard built in INS-0437.

## Acceptance Criteria

Given a broker navigates to the 'Payments & EOBs' section of the Agent Portal,
When the page loads,
Then all Explanation_of_Benefits__c records where EOB.Broker_TIN__c = logged-in broker's TIN and EOB.Status__c = 'Final' are displayed in a list sorted by EOB.Issue_Date__c DESC.

Given EOB records are listed,
When the broker clicks 'Download EOB',
Then the associated ContentDocument linked to the EOB record is served as a PDF download via a Visualforce PDF renderer or stored File, and the action is logged in EOB.Download_Log__c with timestamp and user ID.

Given a broker views the ERA Reconciliation panel,
When the panel loads,
Then Payment__c records where Payment__c.ERA_Status__c IN ('Pending Reconciliation', 'Reconciled') and Payment__c.Broker_Account__c = broker's Account ID are listed with Payment__c.Amount__c, Payment__c.Payment_Date__c, Payment__c.Check_Number__c, and Payment__c.ERA_Status__c.

-Given brokers can initiate ERA dispute directly from this screen (deprecated — ERA dispute workflow deferred to Phase 2)-

Given the broker applies a date filter,
When a custom date range is entered (from/to),
Then the EOB and Payment__c lists refresh to show only records within the selected EOB.Issue_Date__c and Payment__c.Payment_Date__c range respectively.

## Solution

Build LWC 'brokerEobEraPanel' deployed on the Experience Cloud portal page 'Payments & EOBs'. Use @wire adapter with a custom Apex controller 'BrokerEobEraController' that queries EOB__c and Payment__c with enforceObjectAndFieldSecurity. PDF download action calls ContentDocumentLink query scoped to EOB record and returns a URL via PageReference. ERA reconciliation list uses a second Apex method with dynamic SOQL filtered by broker Account ID. Download logging uses a Record-Triggered Flow on a custom object EOB_Download_Log__c. Date filter triggers imperative Apex call on change event. Permission Set Group 'Broker_Portal_Access' restricts field-level access to sensitive Payment__c fields.

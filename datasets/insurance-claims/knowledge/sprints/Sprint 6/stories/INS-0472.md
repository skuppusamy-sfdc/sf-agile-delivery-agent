## Description

As a Claims Operations Manager, I need the system to automatically generate an Electronic Remittance Advice (ERA) file in 835 EDI format and create a corresponding Payment__c record when a Claim__c is moved to 'Adjudicated' status so that providers receive timely, accurate payment notifications and our ERA reconciliation process is fully automated. Extends INS-0448.

## Acceptance Criteria

Given a Claim__c.Status__c transitions to 'Adjudicated',
When the Record-Triggered Flow fires,
Then a Payment__c record is created with:
  - Payment__c.Claim__c = triggering Claim__c.Id
  - Payment__c.Amount__c = Claim__c.Approved_Amount__c
  - Payment__c.Payment_Date__c = today + Claim__c.Policy__r.Payment_Terms_Days__c
  - Payment__c.Status__c = 'Scheduled'
  - Payment__c.ERA_Status__c = 'Pending Generation'

Given the Payment__c record is created with ERA_Status__c = 'Pending Generation',
When the PaymentEngine Apex batch job 'ERA835GeneratorBatch' runs (scheduled nightly at 11 PM CT),
Then an 835 EDI-formatted file is generated containing all Claim_Line__c records linked to the Claim__c, including CPT codes from Claim_Line__c.CPT_Code__c, ICD-10 diagnosis codes from Claim_Line__c.ICD10_Code__c, allowed amounts, and adjustment reason codes

Given the ERA file is successfully generated,
When the file is deposited to the provider's configured SFTP endpoint,
Then Payment__c.ERA_Status__c updates to 'Sent', Payment__c.ERA_File_Name__c is populated, and an Explanation_of_Benefits__c (EOB) record is created linked to the Payment__c and Claim__c

Given the SFTP transmission fails,
When the retry limit of 3 attempts is exceeded,
Then Payment__c.ERA_Status__c = 'Transmission Failed', a Case is created assigned to the Claims Operations queue with Case.Subject = 'ERA Transmission Failure - ' + Payment__c.Name, and a Platform Event PaymentError__e is published

-Given ERA files are batched weekly- (deprecated — moved to nightly batch per SLA requirement INS-0448)

Given a Claim__c involves COB (Coordination of Benefits),
When the ERA is generated,
Then the 835 file includes the COB segment with primary payer payment amount sourced from Claim__c.COB_Primary_Payer_Amount__c

## Solution

Implement a Record-Triggered Flow 'CreatePaymentOnAdjudication_Flow' on Claim__c, triggering on Status__c change to 'Adjudicated', creating Payment__c via a Create Records element. Build Apex class 'ERA835GeneratorBatch' implementing Database.Batchable and Schedulable interfaces; batch queries Payment__c WHERE ERA_Status__c = 'Pending Generation', aggregates related Claim_Line__c records, and constructs 835 EDI string using a utility class 'EDI835Builder'. File transmission to provider SFTP uses a named credential and HttpCallout from an async Queueable 'SFTPTransmitQueueable'. On success/failure, update Payment__c fields via a second DML operation. EOB record creation handled in the same Apex transaction. Platform Event PaymentError__e published via EventBus.publish() on failure. Schedule batch via Anonymous Apex cron expression '0 0 23 * * ?'.

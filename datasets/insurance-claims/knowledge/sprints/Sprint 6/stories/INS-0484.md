## Description

As a Broker Relations Coordinator, I need a consolidated FlexCard dashboard on the Agent/Broker Portal that surfaces active policies, open claims, and pending FNOL submissions linked to my book of business so that I can proactively manage client relationships and identify issues without navigating multiple Salesforce tabs.

## Acceptance Criteria

Given a licensed broker logs into the Agent/Broker Portal,
When the Broker Dashboard loads,
Then a FlexCard displays aggregated counts of: active Policy__c records (Policy__c.Status__c = 'Active'), open Claim__c records (Claim__c.Status__c IN ('Open','In Review','Pending Info')), and FNOL submissions within the last 30 days (Claim__c.FNOL_Date__c >= LAST_N_DAYS:30), all filtered by Policy__c.Broker_Account__c = current user's linked Broker Account.

Given the broker dashboard FlexCard is rendered,
When the broker clicks the 'Open Claims' count tile,
Then a drill-down DataRaptor-powered list view opens showing Claim__c.Claim_Number__c, Claim__c.Claimant_Name__c, Claim__c.LOB__c, Claim__c.Status__c, and Claim__c.Total_Billed_Amount__c columns.

Given a Claim__c record has Status__c = 'Adjudicated' and an associated Explanation_of_Benefits__c record exists,
When the broker views the claim row,
Then an EOB download link is visible and resolves to the correct ContentVersion file via the EOB__c.Document_Link__c field.

Given the broker dashboard is loaded on a mobile viewport (< 768px),
When the FlexCard renders,
Then all tile components stack vertically and no horizontal scroll is required.

-Given broker can see all policies across all LOBs on a single view- (deprecated: LOB-specific tabs will be handled in INS-0489)

## Solution

Build OmniStudio FlexCard 'BrokerDashboardCard' with four DataRaptor Extract actions: (1) ActivePolicyCount — SOQL on Policy__c filtered by Broker_Account__c and Status__c='Active'; (2) OpenClaimCount — SOQL on Claim__c with Status__c multi-picklist filter; (3) FNOLLast30Days — SOQL on Claim__c with FNOL_Date__c range; (4) EOBDocumentFetch — SOQL join on Explanation_of_Benefits__c to ContentVersion. Use OmniStudio Card State Actions for drill-down navigation. Apply Permission Set Group 'BrokerPortalUser_PSG' to restrict field-level visibility. Responsive layout via FlexCard card-body CSS grid configuration.

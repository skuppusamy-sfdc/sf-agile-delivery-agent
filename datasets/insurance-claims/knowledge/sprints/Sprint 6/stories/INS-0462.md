## Description

As a broker, I need a self-service Claim Status Tracker within the Agent Portal so that I can look up the current adjudication status, assigned Claims Operations Manager, and most recent EOB for any claim tied to my book of business without requiring internal staff intervention. Extends INS-0461 broker portal foundation.

## Acceptance Criteria

Given a broker enters a valid Claim__c.Claim_Number__c in the OmniScript search step,
When the DataRaptor lookup executes,
Then Claim__c.Status__c, Claim__c.Adjudicator__c, Claim__c.Date_of_Service__c, and the most recent Explanation_of_Benefits__c.EOB_Document_URL__c are returned and displayed.

Given a claim is returned,
When Claim__c.Status__c = 'Adjudicated',
Then the EOB download link is active and resolves to the associated Explanation_of_Benefits__c record's Salesforce Files attachment.

Given a broker searches for a Claim__c not linked to their Broker__c,
When the DataRaptor returns no results,
Then the OmniScript displays 'No claims found within your book of business' and logs a Search_Audit__c record with fields: Searched_By__c, Search_Term__c, Timestamp__c.

Given Claim__c.Status__c = 'SIU Review',
When the tracker loads the claim,
Then the status is masked and displays as 'Under Review' with no additional detail exposed to the broker.

## Solution

Develop OmniScript 'BrokerClaimStatusTracker' with a text input step feeding a DataRaptor Extract on Claim__c with broker-scoped SOQL (WHERE Policy__c.Broker__c = :currentUserId). Integrate Salesforce Files API for EOB document retrieval. Add Apex-invocable action to create Search_Audit__c records. Apply field-level security on Claim__c.SIU_Flag__c to restrict broker profile. Deploy within the same Experience Cloud site as INS-0461.

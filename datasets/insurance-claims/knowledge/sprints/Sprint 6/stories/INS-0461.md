## Description

As a Broker Relations Coordinator, I need a consolidated FlexCard dashboard on the Agent Portal that surfaces active Policy__c counts, open Claim__c statuses, and pending FNOL submissions per broker book of business, so that brokers can self-serve real-time portfolio insights without contacting the internal service desk.

## Acceptance Criteria

Given a licensed broker logs into the Agent Portal Experience Cloud site,
When the Broker Dashboard page loads,
Then a FlexCard displays aggregated counts of Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.Broker__c = current broker's Contact ID.

Given the FlexCard is rendered,
When claim data is fetched via DataRaptor Extract,
Then Claim__c records with Claim__c.Status__c IN ('Submitted', 'In Review', 'Pending Info') are displayed grouped by LOB (Claim__c.Line_of_Business__c).

Given a broker has open FNOL submissions,
When the FNOL tile is clicked,
Then the broker is navigated to a filtered list view showing Claim__c.FNOL_Status__c = 'Pending' records scoped to their book of business.

Given the dashboard loads,
When policy count exceeds 500 records,
Then a paginated DataRaptor Turbo Extract is used to prevent governor limit exceptions and page load time remains under 3 seconds.

-Given broker sees a 'Quick Quote' CTA button on the dashboard, When clicked, Then an OmniScript launches for new policy intake- (deprecated — Quick Quote descoped from Sprint 6, moved to Sprint 8)

Given a broker without an active Broker_Agreement__c record attempts login,
When the dashboard loads,
Then a permission-gated message displays: 'Your broker agreement is pending activation. Contact your Broker Relations Coordinator.' and no Policy__c or Claim__c data is exposed.

## Solution

Build OmniStudio FlexCard 'BrokerPortfolioDashboard' with child cards for Policies, Claims, and FNOL. Use DataRaptor Turbo Extract for Policy__c and Claim__c aggregation with SOQL filters scoped by Broker__c lookup. Implement Custom Permission 'Broker_Portal_Access' checked at FlexCard action level. Deploy via Permission Set Group 'Broker_Portal_PSG' assigned to Experience Cloud broker profile. Record-Triggered Flow on Broker_Agreement__c activation to assign PSG automatically.

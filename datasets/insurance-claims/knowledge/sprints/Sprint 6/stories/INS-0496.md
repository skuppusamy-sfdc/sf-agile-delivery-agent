## Description

**Bug Report — Security/Data Isolation Defect**

**Summary:** The broker portal policy search DataRaptor returns Policy__c records belonging to other brokers when the search input contains special characters such as hyphens (-), underscores (_), or percent signs (%), causing a SOQL LIKE clause injection that bypasses the Broker__c field filter.

**Steps to Reproduce:**
1. Log in to the Agent Portal as Broker User A (e.g., broker.usera@acmetest.com).
2. Navigate to the Broker Dashboard FlexCard (INS-0492).
3. Click 'Search Policy' and enter the value: '%-' (percent-hyphen) in the Policy Number search field.
4. Observe the returned policy list.

**Expected Result:** Only Policy__c records where Policy__c.Broker__c = Broker User A's linked Contact are returned, regardless of search input.

**Actual Result:** Policy__c records belonging to Broker User B and Broker User C are returned in the search results. The Broker__c field filter is effectively bypassed because the SOQL LIKE clause constructed in DR_GetBrokerPolicies does not escape special characters, resulting in a wildcard match across all brokers.

**Environment:** Acme Insurance - Full Sandbox (ACMEINS-FULL)
**Severity:** High — Data isolation breach between broker accounts
**Reproducible:** Yes, 100%
**Discovered By:** QA regression test INS-QA-0219

## Acceptance Criteria

As per Description. Verify fix by executing QA test INS-QA-0219 in Full Sandbox. Confirm Policy__c records for Broker User B and Broker User C are NOT returned when logged in as Broker User A regardless of search string input including '%', '_', '-', and '/*' characters.

## Solution

Fix DR_GetBrokerPolicies DataRaptor Extract: replace raw SOQL LIKE clause with a sanitized input approach — strip or escape SOQL special characters (%, _, \) from the search input string using a DataRaptor Formula function before binding to the WHERE clause. Add a mandatory secondary filter on Policy__c.Broker__c = :currentUserBrokerId that cannot be overridden by user input. Additionally, enforce a Sharing Rule on Policy__c (Criteria-Based: Broker__c = $User.Contact.Id) as a defense-in-depth measure. Unit test coverage added to DR test suite and Apex sharing test class PolicySharingTest updated.

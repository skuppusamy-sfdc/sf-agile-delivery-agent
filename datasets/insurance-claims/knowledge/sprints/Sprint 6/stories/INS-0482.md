## Description

As a Provider Relations Specialist, I need the system to automatically process Provider Change Notifications (PCNs) received from CMS and NPPES data feeds so that provider demographic updates — including NPI status changes, address updates, and TIN modifications — are reflected on Provider__c (Person Account) records within 24 hours of the source change, reducing manual data reconciliation effort and compliance risk.

## Acceptance Criteria

Given a PCN payload is received from the CMS data feed via the integration middleware,
When the inbound Platform Event 'Provider_Change_Notification__e' is published with PCN_Type__c = 'Address Update',
Then a Provider_Change_Notification__c record is created with PCN__c.Status__c = 'Received', PCN__c.Source__c = 'CMS', PCN__c.NPI__c populated, and PCN__c.Effective_Date__c set from the payload.

Given a PCN__c record is created,
When an Apex trigger processes the PCN,
Then the matching Provider__c record is located by Provider__c.NPI__c = PCN__c.NPI__c and the relevant fields (Provider__c.BillingStreet, Provider__c.BillingCity, Provider__c.BillingState, Provider__c.BillingPostalCode) are updated.

Given a PCN indicates a TIN change (PCN__c.PCN_Type__c = 'TIN Update'),
When the update is processed,
Then Provider__c.Tax_Identification_Number__c is updated and the associated Provider_Network_Agreement__c records are flagged with PNA__c.Pending_TIN_Review__c = true.

Given a PCN indicates NPI deactivation (PCN__c.PCN_Type__c = 'NPI Deactivation'),
When the event is processed,
Then Provider__c.NPI_Status__c is set to 'Inactive', a task is auto-created and assigned to the Provider Relations Specialist queue with Subject = 'NPI Deactivation — Review Network Status', and linked active PNA records are flagged for review.

Given a PCN cannot be matched to an existing Provider__c record by NPI,
When the matching logic fails,
Then PCN__c.Status__c = 'Unmatched', a Case is auto-created with Case.Subject = 'Unmatched PCN — Manual Review Required', and the PCN is added to the 'Unmatched PCN' queue.

Given all PCN processing is complete,
When the compliance officer runs the monthly PCN Audit Report,
Then the report reflects PCN__c records processed within the target SLA of 24 hours, with PCN__c.Processing_Duration_Hours__c calculated from PCN__c.Received_DateTime__c to PCN__c.Processed_DateTime__c.

## Solution

Build an integration using MuleSoft (external middleware) to consume CMS and NPPES data feeds and publish Platform Events 'Provider_Change_Notification__e' into Salesforce. Implement a Platform Event-triggered Flow 'PEF_PCN_Intake' to create PCN__c records. An Apex Trigger 'PCNTrigger' on Provider_Change_Notification__c (After Insert) dispatches processing to a Queueable Apex class 'PCNProcessingQueueable' that performs Provider__c lookup by NPI and field updates in bulkified DML. TIN change logic flags PNA records via a child SOQL loop with a batch size limit. Unmatched PCNs trigger Case creation via Flow invoked from the Queueable. A custom report type 'PCN Audit' on Provider_Change_Notification__c surfaces compliance metrics. Schedule a nightly Batch Apex 'PCNSLAMonitorBatch' to calculate Processing_Duration_Hours__c.

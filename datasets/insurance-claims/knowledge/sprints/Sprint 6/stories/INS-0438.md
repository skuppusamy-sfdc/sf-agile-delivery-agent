## Description

As a broker, I need a guided OmniScript on the Agent Portal to submit a First Notice of Loss (FNOL) on behalf of a policyholder so that I can initiate a claim without calling the claims operations team, reducing intake cycle time. Extends INS-0407 (base Claim__c intake flow).

## Acceptance Criteria

Given a broker initiates a new FNOL from the portal,
When Step 1 of the OmniScript renders,
Then the broker must search for and select a valid Policy__c where Policy__c.Status__c = 'Active' and Policy__c.Effective_Date__c <= TODAY() and Policy__c.Expiration_Date__c >= TODAY().

Given a valid policy is selected,
When the broker proceeds to Step 2,
Then the form pre-populates Claim__c.Insured_Name__c, Claim__c.LOB__c, and Claim__c.Coverage_Type__c from the parent Policy__c record via DataRaptor Transform.

Given the broker enters the Date of Loss,
When the value is submitted,
Then a validation rule prevents Claim__c.Date_of_Loss__c from being a future date and displays the error 'Date of Loss cannot be in the future.'

Given all required fields are completed,
When the broker clicks Submit,
Then a new Claim__c record is created with Claim__c.Status__c = 'Pending Intake', Claim__c.FNOL_Submitted__c = TRUE, Claim__c.Submission_Channel__c = 'Broker Portal', and Claim__c.Submitted_By__c = logged-in user's Contact ID.

Given the Claim__c record is created,
When the record save completes,
Then a Platform Event ClaimIntakeEvent__e is fired to notify the Claims Operations queue and a confirmation email is sent to the broker using the email template 'FNOL_Broker_Confirmation'.

Given the broker submits an FNOL for a policy with an active SIU flag (Policy__c.SIU_Flag__c = TRUE),
When the record is created,
Then Claim__c.Referred_to_SIU__c is automatically set to TRUE and the SIU queue is notified via a separate Platform Event SIUAlert__e.

## Solution

Build OmniScript 'Broker_FNOL_OS' with 4 steps: PolicyLookup, LossDetails, ClaimantInfo, ReviewAndSubmit. Use DataRaptor Extract DRX_PolicyLookupByBroker for policy search filtered by broker TIN. DataRaptor Transform DRT_PrePopulateClaimFields maps Policy__c fields to Claim__c. Integration Procedure IP_CreateFNOLClaim handles Claim__c creation, Platform Event publish (ClaimIntakeEvent__e, conditionally SIUAlert__e), and email dispatch via Messaging.sendEmail Apex call. OmniScript deployed to Experience Cloud portal page 'Submit a Claim' under broker navigation menu.

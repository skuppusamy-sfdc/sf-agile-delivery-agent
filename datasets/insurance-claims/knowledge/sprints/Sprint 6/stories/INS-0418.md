## Description

As a Provider Relations Specialist, I need the system to automatically create a Provider_Change_Notification__c (PCN) record whenever a provider's NPI or TIN is updated on the Provider__c record so that compliance tracking and downstream network agreement reviews are triggered without manual intervention. This supports CMS and NPPES data integrity requirements.

## Acceptance Criteria

Given a Provider Relations Specialist updates Provider__c.NPI__c on an existing provider record,
When the record is saved,
Then a Provider_Change_Notification__c record is automatically created with PCN__c.Change_Type__c = 'NPI Update', PCN__c.Previous_Value__c = old NPI, PCN__c.New_Value__c = new NPI, PCN__c.Provider__c = the provider's record ID, and PCN__c.Status__c = 'Pending Review'.

Given a Provider__c.TIN__c field is updated,
When the record is saved,
Then a separate PCN record is created with PCN__c.Change_Type__c = 'TIN Update' following the same field population rules as the NPI scenario.

Given a PCN record is created with Change_Type__c = 'TIN Update',
When the PCN is saved,
Then a Platform Event 'Provider_Data_Change__e' is published with payload fields: NPI, TIN (new), Provider_Record_ID, Change_Timestamp — to notify downstream contracting and fee schedule systems.

Given both NPI and TIN are updated in the same save operation,
When the trigger fires,
Then two separate PCN records are created (one per field change) — NOT a single combined record.

Given a PCN record is in 'Pending Review' status,
When 5 business days pass without the PCN being actioned,
Then an escalation email alert is sent to the Provider Relations Specialist queue.

## Solution

Implement Apex Trigger 'ProviderChangeTrigger' on Provider__c (before update context) to capture old/new values for NPI__c and TIN__c. Handler class 'ProviderChangeTriggerHandler.cls' uses a trigger framework pattern to create PCN records in after update context. Platform Event 'Provider_Data_Change__e' published via EventBus.publish() for TIN changes. Escalation handled by a Scheduled Flow 'PCN_Escalation_Flow' running daily, querying PCN records where Status__c = 'Pending Review' AND CreatedDate < LAST_N_DAYS:5. All logic bulkified to handle up to 200 records per transaction.

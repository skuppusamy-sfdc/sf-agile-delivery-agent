## Description

As a Broker Relations Coordinator, I need a consolidated dashboard FlexCard on the Agent/Broker Portal that surfaces active Policy__c records and open Claim__c records tied to a broker's book of business, so that brokers can self-serve status checks without calling the operations center and reduce inbound call volume by an estimated 30%.

## Acceptance Criteria

Given a licensed broker logs into the Agent/Broker Portal with the 'Broker_Portal_User' Permission Set Group assigned,
When the dashboard loads,
Then a FlexCard titled 'My Book of Business' renders displaying: total count of Policy__c records where Policy__c.Broker__c = current user's linked Contact and Policy__c.Status__c = 'Active'; and total count of Claim__c records where Claim__c.Broker__c = current user's linked Contact and Claim__c.Status__c IN ('Open','In Review','Pending Documentation').

Given the broker clicks the 'Active Policies' tile on the FlexCard,
When the drill-down action fires,
Then an OmniScript-driven list view renders showing Policy__c.Name, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Premium_Amount__c, and Policy__c.Policyholder__c (hyperlinked) for all matching records.

Given the broker clicks the 'Open Claims' tile,
When the drill-down renders,
Then Claim__c records are displayed with columns: Claim__c.Name, Claim__c.Date_of_Loss__c, Claim__c.Claimant__c, Claim__c.Status__c, Claim__c.Total_Incurred__c, and an action button 'View EOB' that is conditionally visible only when Claim__c.EOB_Generated__c = true.

Given the FlexCard data is loaded,
When more than 200 Policy__c or Claim__c records exist in the broker's book,
Then server-side pagination via DataRaptor Extract is applied with a page size of 25 records and a 'Load More' button is rendered.

-Given broker can see all policies across all brokerages in the same TPA group- (deprecated — scope reduced to own book of business only per INS-0398 decision)

Given the portal is accessed on a mobile viewport (< 768px),
When the FlexCard renders,
Then the layout collapses to a single-column stacked card view and all action buttons remain tappable with minimum 44px touch targets.

## Solution

Build OmniStudio FlexCard 'Broker_Dashboard_Card' with two DataRaptor Extract data sources: DR_Extract_BrokerActivePolicies and DR_Extract_BrokerOpenClaims, both filtered by the logged-in user's associated Contact ID resolved via a custom Apex action (BrokerContextResolver.cls). Drill-down actions invoke child OmniScripts. Apply Broker_Portal_User Permission Set Group to gate access. Deploy responsive CSS overrides via a static resource. Register FlexCard on the Experience Cloud portal page via Lightning App Builder.

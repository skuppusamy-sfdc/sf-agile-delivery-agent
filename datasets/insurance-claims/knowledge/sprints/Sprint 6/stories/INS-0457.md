## Description

As a Claims Operations Manager, I need the system to automatically generate an ANSI X12 835 Electronic Remittance Advice (ERA) file for all adjudicated Claim__c records associated with TPA partners at end-of-day, so that TPAs receive standardized payment remittance data that reconciles with the Explanation_of_Benefits__c (EOB) records in Salesforce without manual intervention.

## Acceptance Criteria

Given Claim__c records where Claim__c.Status__c = 'Adjudicated' and Claim__c.TPA_Partner__c IS NOT NULL and Payment__c.Status__c = 'Processed' with Payment__c.ERA_Generated__c = false,
When the nightly ERA generation job runs at 23:00 UTC,
Then an 835 file is generated per TPA partner containing all qualifying claims processed that business day.

Given an 835 file is being constructed,
When Claim_Line__c records are mapped,
Then each Claim_Line__c entry must include: CPT code (Claim_Line__c.CPT_Code__c), billed amount (Claim_Line__c.Billed_Amount__c), allowed amount (Claim_Line__c.Allowed_Amount__c), adjustment reason code (Claim_Line__c.Adjustment_Reason_Code__c), and NPI from the related Provider__c.NPI__c.

Given the 835 file is generated,
When the file passes internal format validation (loop counts, segment terminators, ISA/IEA envelope),
Then Payment__c.ERA_Generated__c is set to true, Payment__c.ERA_File_Name__c is stamped, and the file is transmitted to the TPA via the configured SFTP Named Credential.

Given an 835 generation failure for any claim line,
When the batch encounters a validation error (e.g., missing CPT_Code__c or null Allowed_Amount__c),
Then the affected Claim__c.ERA_Error__c field is populated with the error message, the claim is excluded from the current file, and a High-priority Case is created and assigned to the Claims_Ops_Queue.

Given a successful file transmission,
When the SFTP acknowledgment is received,
Then Explanation_of_Benefits__c.ERA_Transmission_Status__c is updated to 'Transmitted' and EOB__c.Transmission_Timestamp__c is stamped.

Given a TPA partner with COB (Coordination of Benefits) claims,
When those claims are included in the 835,
Then COB adjustment segments (CAS loop with CO-45 and OA-23 reason codes) are correctly appended per Claim__c.COB_Indicator__c = true.

## Solution

Apex batch class (ERA835GenerationBatch) scheduled via Scheduled Flow at 23:00 UTC. Batch aggregates eligible Payment__c and Claim_Line__c records grouped by TPA_Partner__c. A dedicated Apex utility class (ERA835Builder) constructs the ANSI X12 835 string using a segment-by-segment builder pattern with configurable delimiters stored in Custom Metadata (ERA_Config__mdt). File is written to Salesforce Files (ContentVersion) then pushed to TPA SFTP via an Apex HttpRequest using a Named Credential. Platform Event (ERA_Transmission_Complete__e) published on success, consumed by a Record-Triggered Flow updating EOB__c and Payment__c fields. Error records routed via a separate Queueable Apex job to avoid main batch rollback.

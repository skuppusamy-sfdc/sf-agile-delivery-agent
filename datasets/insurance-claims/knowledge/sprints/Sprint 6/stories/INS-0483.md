## Description

**Bug Report**

**Summary:** When a Broker Relations Coordinator clones an existing Group_Service_Agreement__c (GSA) record using the standard Salesforce Clone button, the GSA__c.Renewal_Date__c field is not populated on the cloned record even though the source record has a valid Renewal_Date__c value. This causes downstream automation (renewal reminder flows) to fail silently.

**Steps to Reproduce:**
1. Navigate to any existing Group_Service_Agreement__c record where GSA__c.Renewal_Date__c is populated (e.g., '2025-01-01') and GSA__c.Status__c = 'Active'.
2. Click the 'Clone' button from the record detail page.
3. On the clone pre-fill screen, observe that GSA__c.Renewal_Date__c is blank despite being populated on the source record.
4. Save the cloned record.
5. Navigate to the newly created GSA record and confirm GSA__c.Renewal_Date__c = null.
6. Observe: The Record-Triggered Flow 'RTF_GSA_RenewalReminder' does not schedule a reminder task because the trigger condition (Renewal_Date__c != null) evaluates to false.

**Expected Behavior:** GSA__c.Renewal_Date__c should be carried over to the cloned record, consistent with all other date fields on the object.

**Actual Behavior:** GSA__c.Renewal_Date__c is null on the cloned record. All other date fields (GSA__c.Effective_Date__c, GSA__c.Expiration_Date__c) clone correctly.

**Root Cause Hypothesis:** GSA__c.Renewal_Date__c field-level 'Do Not Clone' behavior may have been inadvertently set, or a before-insert Flow is blanking the field on clone.

**Environment:** Acme Insurance Full Sandbox (AIS-FS01) and Production
**Reported By:** Broker Relations Coordinator (Lisa Park)
**Frequency:** 100% reproducible
**Impact:** Medium — affects all GSA renewal workflows for group employer accounts.

## Acceptance Criteria

As per Description.

Given the fix is deployed,
When a GSA record with a populated GSA__c.Renewal_Date__c is cloned,
Then the cloned record's GSA__c.Renewal_Date__c matches the source record's value.

Given the cloned GSA record is saved with a valid Renewal_Date__c,
When the Record-Triggered Flow 'RTF_GSA_RenewalReminder' evaluates the new record,
Then a reminder Task is created with Task.Subject = 'GSA Renewal Approaching' and Task.ActivityDate = GSA__c.Renewal_Date__c minus 60 days.

## Solution

Investigate the before-save Record-Triggered Flow 'RTF_GSA_PreSave' for any logic that nullifies Renewal_Date__c on insert when IsClone() is not handled. Add a Flow decision element checking $Record__Prior context or use the ISCLONE() formula function to preserve the field value. Additionally, verify that the GSA__c.Renewal_Date__c field definition does not have any custom clone-exclusion metadata. If a before-insert Apex Trigger is involved, add a guard: if (Trigger.isInsert && record.IsClone()) skip the nullification logic. Deploy fix to sandbox, execute regression test covering clone scenario, then promote to production via Change Set. Update 'ClaimsContractingTest' Apex test class to include a GSA clone test method asserting Renewal_Date__c is not null post-clone.

## Description

As a Broker Relations Coordinator, I need brokers to be able to self-serve download their monthly commission statements as PDFs from the Agent Portal so that the Broker Relations team reduces manual fulfillment requests by at least 60% and brokers have on-demand access to their earnings documentation.

## Acceptance Criteria

Given a broker is authenticated on the Agent Portal,
When they navigate to the 'Commission Statements' section,
Then a list of available commission statement records is displayed for the trailing 24 months, each showing Commission_Statement__c.Statement_Period__c, Commission_Statement__c.Total_Commission_Amount__c, and Commission_Statement__c.Status__c = 'Finalized'.

Given the broker selects a statement and clicks 'Download PDF',
When the Conga CLM document generation action is triggered,
Then a PDF is generated using the 'Broker_Commission_Statement' Conga template and downloaded to the broker's device within 10 seconds.

Given a commission statement has Commission_Statement__c.Status__c = 'Draft' or 'Pending Review',
When the broker views the statements list,
Then those statements are visible but the 'Download PDF' button is disabled with tooltip: 'Statement not yet finalized'.

Given a broker downloads a statement,
When the download completes,
Then an Activity log (Task record) is auto-created on the broker's Contact record with Subject = 'Commission Statement Downloaded', Task.ActivityDate = today, and Task.Description referencing the statement period.

-Given the portal includes a YTD earnings chart widget, When the broker views the dashboard, Then a bar chart of monthly commissions for the current calendar year is rendered.- (Deprecated — YTD chart widget deferred to Phase 2)

Given the broker does not have any finalized commission statements,
When the section loads,
Then an empty state message displays: 'No finalized commission statements available. Contact your Broker Relations Coordinator.'

## Solution

Build an LWC 'commissionStatementList' querying Commission_Statement__c via @wire adapter with a SOQL filter on Broker__c = current user Contact ID and Status__c = 'Finalized'. Integrate Conga CLM via REST API callout from an Apex class 'CongaCommissionStatementController' invoked by an LWC action handler. Store Conga template ID in Custom Metadata Type 'Conga_Template_Config__mdt'. Auto-create Task records via a Record-Triggered Flow on ContentVersion insert (after Conga generates the document). Expose the LWC on the Agent Portal Experience Cloud page with visibility rules tied to the 'Broker_Portal_Access' Permission Set.

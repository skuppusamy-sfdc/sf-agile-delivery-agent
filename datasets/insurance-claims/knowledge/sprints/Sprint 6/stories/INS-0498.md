## Description

As a Broker Relations Coordinator, I need a guided OmniScript that walks through Group Service Agreement (GSA) data collection and initiates a DocuSign envelope for employer signature so that group employer onboarding is standardized, auditable, and completed without manual document handling. The GSA OmniScript should create a Group_Service_Agreement__c record and attach the signed document upon DocuSign completion.

## Acceptance Criteria

Given a Broker Relations Coordinator initiates the 'New Group Agreement' action from an Account (employer) record,
When the OmniScript launches,
Then Step 1 collects: Group_Service_Agreement__c.Employer_Account__c (auto-populated from context), GSA__c.Effective_Date__c, GSA__c.Plan_Type__c (picklist: Medical, Dental, Vision, Ancillary), GSA__c.Enrolled_Employee_Count__c, GSA__c.Broker__c.

Given Step 1 data is complete,
When the coordinator advances to Step 2,
Then the system validates that Account.Type = 'Employer Group' and Account.Active__c = true; if not, an error block displays: 'This account is not eligible for a GSA. Verify account type and active status before proceeding.'

Given all GSA data is collected and validated,
When the coordinator reaches the Review & Sign step,
Then a Conga CLM template merge preview of the GSA document is generated showing all populated fields before DocuSign is triggered.

Given the coordinator confirms and clicks 'Send for Signature',
When the DocuSign Integration Procedure step executes,
Then a DocuSign envelope is created using the 'GSA_Standard_Template' DocuSign template ID, with the employer's primary contact (Contact.Role__c = 'Authorized Signatory') as signer 1 and the Broker Relations Coordinator as CC recipient.

Given the DocuSign envelope status changes to 'Completed',
When the DocuSign status webhook fires,
Then Group_Service_Agreement__c.Status__c = 'Executed', GSA__c.Execution_Date__c = today, and the signed PDF is attached to the GSA record as a Salesforce File (ContentVersion) with ContentVersion.Title = 'Signed_GSA_{GSA_Number}_{Date}'.

Given the DocuSign envelope status changes to 'Declined' or 'Voided',
When the webhook fires,
Then GSA__c.Status__c = 'Signature Declined', a Task is created assigned to the Broker Relations Coordinator with Subject = 'GSA Signature Declined — Follow Up Required', and the coordinator receives an email notification.

## Solution

OmniScript (GSA_Execution_OS) with steps: EmployerDetails (DataRaptor Extract for Account validation), PlanConfiguration, ReviewMerge (Conga CLM API callout via Integration Procedure to generate preview PDF), SignatureDispatch (DocuSign eSignature API via Integration Procedure using DocuSign_NC Named Credential). GSA record created via DataRaptor Turbo Action on OmniScript submit. DocuSign completion webhook received by a Salesforce Site endpoint invoking a REST Apex class (DocuSignWebhookHandler) that updates GSA__c fields and attaches signed document via ContentVersion + ContentDocumentLink DML. Platform Event (DocuSign_Status_Event__e) used to decouple webhook processing from DML. Task creation via Apex TaskFactory utility. Custom metadata (DocuSign_Template_Config__mdt) stores template IDs per agreement type.

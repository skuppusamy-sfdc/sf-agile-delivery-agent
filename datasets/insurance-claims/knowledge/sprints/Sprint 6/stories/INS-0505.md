## Description

As an Underwriting Analyst, I need a guided Screen Flow to create a Group_Service_Agreement__c (GSA) record and automatically send the agreement document for eSignature via DocuSign, so that employer group onboarding is accelerated and the manual process of generating, emailing, and tracking signature status is eliminated.

This story covers GSA creation and DocuSign envelope dispatch only. Conga CLM template merge for GSA document generation is handled in a separate story (INS-0508, Phase 1 backlog).

## Acceptance Criteria

Given an Underwriting Analyst opens the 'New Group Service Agreement' Screen Flow from the Account (Employer Group) record page,
When Step 1 'Agreement Details' renders,
Then the flow pre-populates GSA__c.Employer_Account__c with the current Account.Id and displays input fields for GSA__c.Effective_Date__c, GSA__c.Expiration_Date__c, GSA__c.LOB__c (picklist), GSA__c.Enrolled_Member_Count__c, and GSA__c.TPA__c (lookup to TPA Account).

Given the analyst completes Step 1 and clicks 'Next',
When GSA__c.Effective_Date__c is not populated or GSA__c.Expiration_Date__c <= GSA__c.Effective_Date__c,
Then the flow displays a validation error 'Expiration Date must be after Effective Date' and prevents navigation to Step 2.

Given all required fields are valid and the analyst reaches the 'Review & Submit' step,
When they click 'Create Agreement & Send for Signature',
Then: (1) A GSA__c record is created with GSA__c.Status__c = 'Pending Signature'; (2) A DocuSign envelope is dispatched using the DocuSign for Salesforce managed package action, with the signing recipient set to the Employer Group's Account primary contact (Contact.Email where Contact.Primary__c = TRUE); (3) GSA__c.DocuSign_Envelope_Id__c is populated with the returned envelope ID.

Given the DocuSign envelope is sent successfully,
When the employer group contact completes eSignature,
Then a DocuSign Connect webhook callback updates GSA__c.Status__c = 'Active', GSA__c.Signed_Date__c = signature completion date, and attaches the signed PDF as a ContentVersion linked to the GSA__c record.

Given the DocuSign envelope status returns 'Declined' or 'Voided',
When the webhook callback is received,
Then GSA__c.Status__c = 'Signature Declined', a Task is created on the GSA__c record with Subject = 'GSA Signature Declined — Follow Up Required' assigned to the owning Underwriting Analyst, and a Chatter post is added to the GSA__c record feed.

-Given the flow supports bulk GSA creation for multi-LOB employer groups in a single submission- (descoped — multi-LOB GSA support deferred to Phase 2 per product decision on 2024-11-14)

## Solution

Build a Lightning Screen Flow 'GSA_Creation_Flow' launched from the Account record page via a Quick Action button 'New Group Service Agreement'. Flow steps: (1) Screen — Agreement Details inputs; (2) Decision — date validation; (3) Screen — Review Summary (read-only); (4) Record Create — GSA__c DML; (5) Action — DocuSign CreateEnvelope Apex invocable action (from DocuSign for Salesforce managed package, API: dfsle.EnvelopeService.sendEnvelope) passing GSA__c.Id and template ID stored in Custom Metadata 'GSA_DocuSign_Config__mdt'; (6) Record Update — populate GSA__c.DocuSign_Envelope_Id__c with returned envelope ID. DocuSign Connect is configured in the DocuSign Admin console to POST to a Salesforce Site endpoint (Apex REST resource 'GSADocuSignWebhookHandler') that handles status callbacks. The webhook handler uses an Apex REST class with @HttpPost to parse the envelope status XML and perform GSA__c updates plus Task/Chatter post via ConnectApi. Permission Set 'GSA_Creator_PS' grants FLS on all GSA__c fields and the Quick Action is added to the Employer_Group_Account page layout.

## Description

As a Broker Relations Coordinator, I need the Broker Dashboard to display policy summaries organized by Line of Business (LOB) in separate tabs so that I can quickly navigate to the specific product lines relevant to a client conversation without scrolling through a mixed list of all policy types.

This story implements the LOB-specific tab structure that was descoped from INS-0484 per UX review.

## Acceptance Criteria

Given a broker is viewing the Broker Dashboard,
When the LOB tab bar renders,
Then tabs are dynamically generated based on the distinct values of Policy__c.LOB__c for policies where Policy__c.Broker_Account__c = current broker's account, and only LOBs with at least one active policy are shown as tabs.

Given the broker selects the 'Group Health' LOB tab,
When the tab content loads,
Then a DataRaptor-powered list displays only Policy__c records where Policy__c.LOB__c = 'Group Health' and Policy__c.Status__c = 'Active', showing columns: Policy__c.Policy_Number__c, Policy__c.Group_Name__c, Policy__c.Effective_Date__c, Policy__c.Member_Count__c, and Policy__c.Annual_Premium__c.

Given the broker selects any LOB tab,
When the tab content loads,
Then a count badge on each tab label reflects the number of active Policy__c records for that LOB, updating dynamically if policies change status.

Given a broker's book of business contains policies across more than 5 distinct LOBs,
When the tab bar renders,
Then the first 4 LOBs by active policy count are shown as primary tabs and remaining LOBs are collapsed into a 'More' dropdown.

Given the broker clicks a Policy__c.Policy_Number__c link within any LOB tab,
When the navigation fires,
Then the broker is taken to the Policy Detail page within the portal experience (not the internal Salesforce record page).

## Solution

Extend 'BrokerDashboardCard' FlexCard from INS-0484 with a new OmniStudio Tab component. Create Integration Procedure 'GetBrokerLOBTabs_IP' that uses a DataRaptor Extract with GROUP BY Policy__c.LOB__c and COUNT to return distinct LOBs and counts. Tab component iterates over IP response to render dynamic tabs. Each tab body is a child FlexCard 'BrokerLOBPoliciesCard' with its own DataRaptor Extract filtered by LOB value passed as input parameter. 'More' dropdown implemented via conditional FlexCard state with array slice logic. Policy detail navigation uses OmniStudio Navigation Action pointing to Experience Cloud page 'PolicyDetail'.

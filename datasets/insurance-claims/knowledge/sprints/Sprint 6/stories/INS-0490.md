## Description

As a Credentialing Specialist, I need the system to automatically create a Credentialing_Case__c record and trigger the initial credentialing workflow when a new Provider__c record's NPI is successfully verified against the NPPES registry so that credentialing is initiated without manual intervention and the provider record is immediately flagged with verification status.

## Acceptance Criteria

Given a new Provider__c record is saved with Provider__c.NPI__c populated and Provider__c.Credentialing_Status__c = 'Pending Verification',
When the Record-Triggered Flow 'Provider_NPI_Verify_Flow' fires on insert,
Then an Apex Action 'NPPESVerificationCallout' is invoked asynchronously with Provider__c.NPI__c as input.

Given the NPPES API callout returns a successful match (HTTP 200 with enumeration_type populated),
When the response is processed,
Then Provider__c.NPI_Verified__c = true, Provider__c.NPI_Verification_Date__c = TODAY(), Provider__c.Credentialing_Status__c = 'NPI Verified', and Provider__c.NPPES_Enumeration_Type__c is set from the API response value ('NPI-1' or 'NPI-2').

Given Provider__c.NPI_Verified__c is set to true,
When the update is committed,
Then a Credentialing_Case__c record is auto-created with: Credentialing_Case__c.Provider__c = Provider__c.Id, Credentialing_Case__c.Status__c = 'Initiated', Credentialing_Case__c.Case_Type__c = 'Initial Credentialing', Credentialing_Case__c.Source__c = 'NPPES Auto-Verify', and Credentialing_Case__c.OwnerId = Credentialing Specialist queue Id from Custom Metadata.

Given the NPPES API returns no match or HTTP 404,
When the response is processed,
Then Provider__c.NPI_Verified__c = false, Provider__c.Credentialing_Status__c = 'NPI Not Found', and a Task is created on the Provider__c record with Subject = 'Manual NPI Verification Required' assigned to the Credentialing queue.

Given the NPPES callout times out or returns a 5xx error,
When the exception is caught,
Then Provider__c.Credentialing_Status__c = 'Verification Error', a Platform Event 'NPI_Verify_Error__e' is published, and no Credentialing_Case__c is created.

Given a Credentialing_Case__c is successfully created,
When the record is committed,
Then an email notification is sent to the assigned Credentialing Specialist using Email Alert 'CredentialingCaseCreated_EA' with the provider's name, NPI, and case number in the body.

## Solution

Record-Triggered Flow on Provider__c (before/after insert) checks NPI__c is not blank and Credentialing_Status__c='Pending Verification', calls Apex Future method 'NPPESVerificationCallout.verifyNPI()'. Apex class makes HTTP GET to NPPES NPI Registry API v2.1 endpoint using Named Credential 'NPPES_API_NC'. Response parsed via JSON.deserialize into wrapper class. Provider__c fields updated via second DML call. Credentialing_Case__c insert in same transaction. Queue Id retrieved from Custom Metadata 'Credentialing_Config__mdt'. Platform Event published via EventBus.publish() on error path. Email Alert triggered via Process (or additional Flow) on Credentialing_Case__c insert. Full Apex test coverage including mock HTTP responses for 200, 404, and 500 scenarios.

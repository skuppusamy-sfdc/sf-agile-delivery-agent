## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page that consolidates NPI, TIN, active Provider_Network_Agreement__c (PNA) status, and any open Credentialing_Case__c flags, so that I can assess a provider's full network and credentialing posture in a single view without switching between multiple related lists.

## Acceptance Criteria

Given a Provider__c record is open in the internal Salesforce app,
When the Provider 360 FlexCard loads,
Then it displays Provider__c.NPI__c, Provider__c.TIN__c (masked to last 4 digits for non-admin profiles), Provider__c.Primary_Specialty__c, and Provider__c.Provider_Type__c.

Given the FlexCard PNA section,
When rendered,
Then all related Provider_Network_Agreement__c records where PNA__c.Status__c IN ('Active','Pending Renewal') are listed showing PNA__c.Network_Name__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, and a color-coded status badge (green = Active, amber = Pending Renewal).

Given the FlexCard Credentialing section,
When an open Credentialing_Case__c exists (Credentialing_Case__c.Status__c NOT IN ('Closed','Withdrawn')),
Then a warning banner 'Open Credentialing Action Required' is displayed with a link to the Credentialing_Case__c record.

Given no open Credentialing_Case__c records exist,
When the credentialing section renders,
Then a green 'Credentialing Current' indicator is shown with the most recent Credentialing_Case__c.Closed_Date__c.

Given a user with the 'Provider_Network_ReadOnly' Permission Set,
When the FlexCard loads,
Then TIN__c is masked and the 'Edit Provider' action button is hidden via a Conditional View state.

## Solution

OmniStudio FlexCard ('Provider360_FC') with three card states: Summary, PNA List (child FlexCard via Flyout), and Credentialing Flags. DataRaptor Extract actions for Provider__c fields, related PNA__c list (SOQL with status filter), and Credentialing_Case__c open record check. TIN masking implemented in DataRaptor Transform using a formula node (RIGHT(TIN,4) with asterisk prefix). Conditional View states driven by user Permission Set membership queried via a custom Integration Procedure. FlexCard deployed on Provider__c Lightning Record Page via App Builder.

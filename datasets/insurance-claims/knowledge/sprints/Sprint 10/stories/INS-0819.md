## Description

**Bug Report**

**Summary:** When a Credentialing_Case__c associated with a provider's termination is reversed (i.e., the provider successfully re-credentials and the termination is rescinded), the linked Network_Termination_Case__c record is not automatically updated to Status__c = 'Closed'. This leaves stale open termination cases that incorrectly trigger downstream notifications and skew compliance reporting.

**Steps to Reproduce:**
1. Navigate to a Provider__c record with an active Network_Termination_Case__c (NTC.Status__c = 'Pending Termination').
2. Open the linked Credentialing_Case__c record.
3. Update Credentialing_Case__c.Status__c from 'Termination Review' to 'Reinstated'.
4. Save the Credentialing_Case__c record.
5. Navigate back to the Network_Termination_Case__c record.

**Expected Result:** NTC.Status__c should automatically update to 'Closed' and NTC.Closure_Reason__c should be set to 'Credentialing Reversal' when the linked Credentialing_Case__c.Status__c = 'Reinstated'.

**Actual Result:** NTC.Status__c remains 'Pending Termination'. No closure event fires. The scheduled termination automation (INS-0816) continues to process the NTC incorrectly on the next run.

**Environment:** Full Sandbox (FS-01)
**Frequency:** 100% reproducible
**Severity:** High — affects compliance reporting and triggers erroneous PCN letters

## Acceptance Criteria

As per Description. Fix must ensure NTC.Status__c = 'Closed' and NTC.Closure_Reason__c = 'Credentialing Reversal' are set within the same transaction as the Credentialing_Case__c.Status__c update to 'Reinstated'. Regression test must confirm the INS-0816 scheduled flow skips NTC records with Status__c = 'Closed'.

## Solution

Investigate the existing Record-Triggered Flow on Credentialing_Case__c. Add a new path in the flow triggered on Status__c change to 'Reinstated': query related Network_Termination_Case__c via NTC.Credentialing_Case__c lookup, update NTC.Status__c = 'Closed' and NTC.Closure_Reason__c = 'Credentialing Reversal' via Update Records element. Confirm the INS-0816 scheduled flow Decision element already filters on Status__c != 'Closed'; if not, add the filter condition. Deploy fix to FS-01 for regression testing before promoting to UAT.

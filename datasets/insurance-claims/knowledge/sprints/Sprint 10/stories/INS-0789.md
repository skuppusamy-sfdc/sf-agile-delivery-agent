## Description

As a Credentialing Specialist, I need the system to automatically query CAQH and NPPES APIs when a Network_Termination_Case__c is opened so that any outstanding credentialing gaps or license discrepancies are surfaced on the Credentialing_Case__c record before the termination is finalized, ensuring we do not inadvertently terminate a provider who has active credentialing disputes pending resolution.

## Acceptance Criteria

Given a Network_Termination_Case__c.Status__c = 'Initiated' is created,
When the Platform Event (Provider_Termination_Event__e) is consumed by the CredentialingIntegrationService,
Then an outbound callout is made to the CAQH ProView API using Provider__c.CAQH_Provider_ID__c and the response is stored in a new Credentialing_External_Check__c child record.

Given the CAQH API response is received,
When CAQH_Status__c in the response = 'Incomplete' or 'Re-attestation Required',
Then Credentialing_Case__c.Termination_Hold_Flag__c = TRUE and Network_Termination_Case__c.Status__c is updated to 'On Hold — Credentialing Review' with a chatter post tagging the assigned Credentialing Specialist.

Given an NPPES lookup using Provider__c.NPI__c,
When the NPPES response indicates NPI_Status = 'Deactivated',
Then Credentialing_Case__c.NPI_Deactivation_Alert__c = TRUE and a High-priority task is created for the Credentialing Specialist.

Given either API returns an HTTP error (4xx or 5xx),
When the callout fails,
Then the error is logged in Integration_Error_Log__c with fields: Error_Code__c, Endpoint__c, Provider_NPI__c, Timestamp__c and a retry is scheduled after 30 minutes up to 3 attempts.

Given all credentialing checks pass (CAQH complete, NPI active),
When both API responses are stored,
Then Network_Termination_Case__c.Credentialing_Clearance__c = TRUE and the termination workflow is unblocked to proceed to 'Provider Notified'.

## Solution

Implement a Platform Event subscriber Apex trigger (ProviderTerminationEventTrigger) on Provider_Termination_Event__e that invokes a Queueable Apex class (CredentialingExternalCheckQueueable) to perform async callouts to CAQH ProView REST API and NPPES NPI Registry API. Named Credentials ('CAQH_ProView' and 'NPPES_Registry') manage endpoint URLs and auth tokens securely. API responses are parsed using JSON.deserializeUntyped() and mapped to Credentialing_External_Check__c via a DataRaptor-style field mapping class. Retry logic implemented using a custom Retry_Queue__c object with a Scheduled Apex job polling every 30 minutes. All integration errors written to Integration_Error_Log__c. Remote Site Settings and CSP Trusted Sites updated for both external endpoints.

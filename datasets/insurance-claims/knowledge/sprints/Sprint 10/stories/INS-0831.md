## Description

As a Provider Network Specialist, I need the system to automatically suppress the auto-renewal process for a Provider_Network_Agreement__c and optionally trigger a renewal offer workflow when a Network_Termination_Case__c is initiated, so that we do not inadvertently renew contracts for providers who are in an active termination process and so that we can offer re-contracting to providers whose termination is due to contract expiry rather than cause.

This story picks up the auto-renew suppression requirement that was deprecated from INS-0824 during sprint planning.

## Acceptance Criteria

Given a Network_Termination_Case__c is created with Status__c = 'Initiated'
When the record is inserted
Then the linked Provider_Network_Agreement__c.Auto_Renew__c is set to false and PNA.Auto_Renew_Suppressed_By_Termination__c = true and PNA.Suppression_Date__c = today

Given PNA.Auto_Renew__c is suppressed
When the Network_Termination_Case__c is subsequently closed with Status__c = 'Withdrawn' (termination cancelled)
Then PNA.Auto_Renew__c is restored to true, PNA.Auto_Renew_Suppressed_By_Termination__c = false, and a Task is created for the assigned Provider Network Specialist to confirm the reinstatement with subject 'Confirm PNA Auto-Renewal Reinstated: [Provider Name]'

Given a Network_Termination_Case__c with Termination_Type__c = 'Contract Expiry'
When Status__c = 'Initiated'
Then the system checks Provider__c.Termination_Eligible_For_Renewal_Offer__c (custom field, boolean) and if true, creates a child Renewal_Offer_Case__c record linked to the PNA with Status__c = 'Pending Specialist Review' and Due_Date__c = today + 14 days

Given a Network_Termination_Case__c with Termination_Type__c IN ('Involuntary','Voluntary Exit','Sanctions')
When Status__c = 'Initiated'
Then no Renewal_Offer_Case__c is created and a note is appended to the Network_Termination_Case__c chatter feed: 'Renewal offer suppressed — termination type not eligible for re-contracting'

Given the auto-renewal suppression logic executes for more than 50 PNA records in a single transaction (bulk scenario)
When the platform event (Provider_Termination_Event__e) is consumed
Then PNA updates are processed via Apex Batch to avoid DML governor limit exceptions and all updates are logged in Integration_Log__c

-Auto-renewal suppression should also update the linked Conga CLM contract record- (deprecated: Conga CLM sync deferred to Phase 3 integration sprint per architecture decision log ADL-047)

## Solution

Record-Triggered Flow on Network_Termination_Case__c (AFTER INSERT) handles single-record suppression via field updates on related PNA using a Get Records + Update Records flow pattern. A second flow path handles the 'Withdrawn' status restoration (AFTER UPDATE on Status__c change to 'Withdrawn'). Renewal_Offer_Case__c creation handled in the same flow via decision element on Termination_Type__c and Provider__c.Termination_Eligible_For_Renewal_Offer__c. Bulk scenario handled by a Platform Event-triggered flow subscribing to Provider_Termination_Event__e which calls Apex invocable 'PNAAutoRenewalSuppressionBatch'. Chatter post via ConnectApi.ChatterFeeds in Apex for the ineligible renewal message. Custom metadata 'Renewal_Offer_Config__mdt' stores eligible termination types for future configurability.

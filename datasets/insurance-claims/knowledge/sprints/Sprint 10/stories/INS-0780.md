## Description

**Bug Report**

**Summary:** When a Network_Termination_Case__c is auto-created by the TerminationExpiryBatch job (INS-0776), the Termination_Case.Effective_Date__c field is populated with the batch execution date (TODAY) rather than the source PNA.Termination_Date__c value, causing incorrect effective dates in downstream PCN generation and CMS reporting.

**Steps to Reproduce:**
1. Create a Provider_Network_Agreement__c with PNA.Termination_Date__c = 2025-08-15 and PNA.Status__c = 'Active'.
2. Manually execute the TerminationExpiryBatch job in the sandbox (System.scheduleBatch).
3. Navigate to the auto-created Network_Termination_Case__c record.
4. Observe Termination_Case.Effective_Date__c = batch run date (e.g., 2025-07-10) instead of 2025-08-15.

**Expected Behavior:** Termination_Case.Effective_Date__c = PNA.Termination_Date__c (2025-08-15).

**Actual Behavior:** Termination_Case.Effective_Date__c = System.today() at time of batch execution (2025-07-10).

**Impact:** HIGH — PCN.Send_Date__c is calculated as Effective_Date__c + 90 days, so incorrect effective dates push notification sends 35+ days early, violating provider contracts. Affects all batch-initiated termination cases in Sprint 10 regression.

**Environment:** Acme-SIT sandbox. Reproducible 100% of runs.

## Acceptance Criteria

As per Description. Verify fix by confirming Termination_Case.Effective_Date__c = PNA.Termination_Date__c for all batch-created records. Re-run regression suite for INS-0776 ACs post-fix.

## Solution

Root cause: In TerminationExpiryBatch.execute(), the Network_Termination_Case__c sObject is initialized with Effective_Date__c = Date.today() as a hardcoded default before the PNA record fields are mapped. Fix: Replace the hardcoded assignment with Effective_Date__c = pna.Termination_Date__c in the sObject construction loop. Add a null-safety check: if pna.Termination_Date__c == null, set Effective_Date__c = Date.today() and flag Termination_Case.Data_Quality_Flag__c = 'Missing PNA Termination Date'. Update unit test TerminationExpiryBatchTest to assert Effective_Date__c equality with source PNA date across 3 test scenarios (past, today, future termination dates).

## Description

As a Claims Operations Manager, I need the claims adjudication engine to enforce a 90-day tail period for terminated providers so that Claim__c records with a Date_of_Service__c falling within the tail window are processed at the in-network Fee_Schedule__c rate, while claims with DOS after the final termination date are automatically pended for out-of-network review. Extends INS-0754 (Fee Schedule Engine core logic).

## Acceptance Criteria

Given a Claim__c is submitted where Claim__c.Provider__c links to a Provider__c with Network_Status__c = 'Terminated' and Network_Termination_Case__c.Final_Termination_Date__c is populated,
When Claim__c.Date_of_Service__c <= (Final_Termination_Date__c + 90 days),
Then Claim__c.Adjudication_Rule__c = 'In-Network Tail Period' and the applicable Fee_Schedule__c is the last active in-network schedule as of the termination date.

Given a Claim__c where Date_of_Service__c > (Final_Termination_Date__c + 90 days),
When the claim is submitted,
Then Claim__c.Status__c = 'Pended' and Claim__c.Pend_Reason__c = 'Provider Terminated — Out of Network Review Required' and the claim is routed to the OON review queue.

Given a Claim__c in the tail period,
When the Fee_Schedule__c lookup executes,
Then Fee_Schedule__c.Effective_End_Date__c is validated to ensure it covers the DOS; if no valid schedule exists, Claim__c.Status__c = 'Pended' with Pend_Reason__c = 'No Active Fee Schedule for Tail Period'.

-Given LOB = 'Medicaid MCO', tail period is 60 days — removed, all LOBs standardized to 90 days per Compliance Officer directive dated 2024-11-12-

Given a bulk claim import via ERA file,
When 500+ claims are processed in a single batch,
Then all tail period evaluations complete within Apex CPU time limits using a batchable interface (ClaimTailPeriodEvaluationBatch) with batch size = 50.

Given the tail period adjudication completes,
When Claim__c.Adjudication_Rule__c = 'In-Network Tail Period',
Then the EOB__c generated reflects the in-network member cost-share and includes a remark code 'TP90' in EOB__c.Remark_Codes__c.

## Solution

Extend the existing ClaimsAdjudicationService Apex class with a new method evaluateTailPeriod(Claim__c claim). Method queries Network_Termination_Case__c via provider lookup to retrieve Final_Termination_Date__c, then computes tail window. Fee_Schedule__c lookup uses a SOQL query filtering on LOB__c, Provider__c, Effective_Start_Date__c <= DOS, Effective_End_Date__c >= DOS ordered by version descending. Batch processing implemented via ClaimTailPeriodEvaluationBatch implementing Database.Batchable and Database.Stateful for error aggregation. Record-Triggered Flow on Claim__c (before-save) sets Adjudication_Rule__c and Pend_Reason__c for real-time single-record submissions. Custom metadata type Tail_Period_Config__mdt stores the 90-day window value for easy configuration without code deployment.

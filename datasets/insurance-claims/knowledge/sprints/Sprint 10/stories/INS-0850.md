## Description

As a Compliance Officer, I need the system to automatically generate and route 90-day, 60-day, and 30-day termination notice letters through Conga CLM when a Network_Termination_Case__c reaches each milestone date so that Acme Insurance meets its contractual and regulatory notice obligations for all LOBs without manual document preparation.

## Acceptance Criteria

Given a Network_Termination_Case__c where NTC.Status__c = 'Initiated' and NTC.Termination_Effective_Date__c is set
When the system calculates that today = NTC.Termination_Effective_Date__c - 90 days
Then a Conga CLM document generation job is triggered using template 'TMPL_Provider_Termination_90Day' and a Provider_Change_Notification__c is created with PCN.Notification_Type__c = 'Termination_90Day', PCN.Status__c = 'Sent', PCN.Sent_Date__c = TODAY()

Given the Conga CLM job completes successfully
When the generated PDF is returned
Then the document is attached to the Network_Termination_Case__c as a ContentDocument and NTC.Last_Notice_Sent__c is stamped with the current date/time

Given a termination notice letter is generated
When PCN.Notification_Type__c = 'Termination_30Day'
Then NTC.Status__c is automatically advanced to 'Pending Effective' and an email alert is sent to NTC.Assigned_To__c and the Provider Relations Specialist queue

Given NTC.Termination_Reason__c = 'For Cause'
When any notice letter is generated
Then the Conga template selected is 'TMPL_Provider_Termination_ForCause' regardless of the day milestone, and NTC.Requires_Legal_Review__c is set to TRUE, triggering a task assigned to the Compliance Officer role queue

Given a Conga CLM API callout fails
When the document generation job returns an error
Then NTC.Document_Generation_Status__c = 'Failed', a high-priority Case is created for the Provider Network Admin queue, and no PCN status is updated until successful retry

-30-day notice triggers automatic CAQH re-attestation request- (removed; CAQH re-attestation is handled in Credentialing offboarding flow INS-0852)

## Solution

Record-Triggered Flow on Network_Termination_Case__c fires on create/update when Termination_Effective_Date__c changes. Flow uses three Decision elements to evaluate 90/60/30-day windows using DATEVALUE and formula fields. Each branch calls an Apex invocable action 'CongaCLMDocGenInvocable' that constructs the Conga API request payload (template ID, merge field map from NTC + Provider__c fields, output format PDF). Callout is made via Named Credential 'Conga_CLM_NC'. Async Apex (Queueable) handles the response polling and ContentDocument attachment. Custom metadata 'Termination_Notice_Template__mdt' maps LOB + termination reason to the correct Conga template ID, enabling no-code template swaps. Platform Event 'TerminationNoticeGenerated__e' notifies the ComplianceReporting component for audit trail capture.

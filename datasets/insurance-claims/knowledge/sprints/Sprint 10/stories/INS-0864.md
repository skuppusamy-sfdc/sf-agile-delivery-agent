## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record when a Provider_Network_Agreement__c reaches its expiration date or when a provider submits a voluntary exit request, so that the termination workflow is initiated without manual intervention and no in-network claims are processed post-termination.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Status__c = 'Active' and PNA.End_Date__c equals today's date,
When a scheduled batch job runs at 00:01 AM EST,
Then a Network_Termination_Case__c is created with Termination_Type__c = 'Contract Expiration', Status__c = 'Initiated', and linked via NTC.Provider_Network_Agreement__c lookup.

Given a Provider_Network_Agreement__c where PNA.Voluntary_Exit_Requested__c = true and PNA.Voluntary_Exit_Date__c is populated,
When the record is saved,
Then a Network_Termination_Case__c is created with Termination_Type__c = 'Voluntary Exit' and NTC.Effective_Termination_Date__c = PNA.Voluntary_Exit_Date__c.

Given a Network_Termination_Case__c is created,
When the record is inserted,
Then a Platform Event (Provider_Termination_Event__e) is published with fields: NPI__c, TIN__c, Effective_Date__c, and LOB__c sourced from the parent PNA.

Given the Platform Event is consumed,
When downstream claims processing evaluates provider eligibility,
Then any Claim__c with Claim.Rendering_Provider__c matching the terminated NPI and Claim.Service_Date__c >= NTC.Effective_Termination_Date__c must set Claim__c.Provider_Network_Status__c = 'Out-of-Network'.

-Given termination is triggered, automatically close all open Credentialing_Case__c records for the provider- (deprecated: handled in INS-0867)

Given a Network_Termination_Case__c is created,
When the assigned Provider Relations Specialist opens the record,
Then a 90-day termination checklist Task list is auto-generated via Flow with subjects: 'Send PCN to Members', 'Notify TPA', 'Update Fee Schedule', 'CMS Reporting'.

## Solution

Implement a Scheduled Apex job (ProviderPNAExpirationBatch) running nightly to query PNA records approaching expiration. Use a Record-Triggered Flow on Provider_Network_Agreement__c (After Save, on Voluntary_Exit_Requested__c change) to handle voluntary exits. Both paths invoke an Apex-invocable method (TerminationCaseFactory.createCase()) to insert Network_Termination_Case__c. Upon insert, an Apex Trigger on Network_Termination_Case__c publishes a Provider_Termination_Event__e Platform Event. A separate Platform Event-Triggered Flow consumes the event and updates downstream Claim__c records via a related list update. Checklist Tasks are generated via a Record-Triggered Flow (After Insert) on Network_Termination_Case__c using a custom metadata-driven task template (Termination_Task_Template__mdt).

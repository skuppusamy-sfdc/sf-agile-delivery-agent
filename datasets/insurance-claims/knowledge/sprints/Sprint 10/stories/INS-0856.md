## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record when a Provider_Network_Agreement__c (PNA) reaches its Contract_End_Date__c, so that the termination workflow is initiated without manual intervention and no provider exits the network without proper offboarding documentation.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where Contract_End_Date__c equals T-60 days from today and PNA_Status__c = 'Active'
When the scheduled batch job runs at 02:00 AM EST daily
Then a Network_Termination_Case__c is created with Status__c = 'Pending Review', Termination_Reason__c = 'Contract Expiration', and linked via PNA_Lookup__c to the originating PNA record

Given the Network_Termination_Case__c is successfully created
When the record is inserted
Then a Provider_Change_Notification__c (PCN) is auto-generated with PCN_Type__c = 'Termination Notice', Effective_Date__c = PNA.Contract_End_Date__c, and Provider__c populated from PNA.Provider__c

Given a PNA where PNA_Status__c = 'Suspended' or 'Terminated'
When the batch job evaluates the record
Then no duplicate Network_Termination_Case__c is created and the job logs a skipped-record entry in the Batch_Execution_Log__c object

Given the Network_Termination_Case__c is created
When the assigned Provider Network Specialist opens the case
Then all linked active Claim__c records with Claim_Status__c NOT IN ('Closed','Denied','Paid') are visible in the Related Claims subpanel

-Given PNA.Auto_Renew__c = true, termination case should be suppressed- (superseded: auto-renew logic moved to INS-0861)

Given the batch job completes
When more than 0 Network_Termination_Case__c records are created
Then an email notification is dispatched to the Provider_Network_Specialist__c queue with a summary count and list of affected Provider NPIs

## Solution

Implement a Scheduled Apex class (ProviderTerminationBatchJob) implementing Database.Batchable and Schedulable interfaces. Query PNA records where Contract_End_Date__c = TODAY + 60 AND PNA_Status__c = 'Active'. For each qualifying PNA, use a UnitOfWork pattern (fflib) to insert Network_Termination_Case__c and Provider_Change_Notification__c in a single DML transaction. A Record-Triggered Flow on Network_Termination_Case__c (After Insert) fires a Platform Event (TerminationInitiated__e) consumed by a downstream notification Apex handler. Batch_Execution_Log__c records are written via a separate async queueable. Deploy with a Permission Set Group 'ProviderTermination_PSG' granting CRUD on all new objects.

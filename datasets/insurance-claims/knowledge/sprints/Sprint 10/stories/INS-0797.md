## Description

As a Provider Network Specialist, I need Fee_Schedule__c records linked to a Provider_Network_Agreement__c to be automatically deactivated when the PNA reaches its termination effective date and the associated Network_Termination_Case__c is closed, so that claims adjudication does not incorrectly apply contracted rates to a terminated provider's claims and the PaymentEngine uses out-of-network default rates instead.

## Acceptance Criteria

Given a Network_Termination_Case__c transitions to NTC.Status__c = 'Closed',
When the Record-Triggered Flow on NTC fires,
Then all Fee_Schedule__c records where Fee_Schedule__c.PNA__c = NTC.PNA_Reference__c are updated:
  - Fee_Schedule__c.Active__c = false
  - Fee_Schedule__c.Deactivation_Date__c = NTC.Termination_Effective_Date__c
  - Fee_Schedule__c.Deactivation_Reason__c = 'Provider Network Termination'

Given multiple Fee_Schedule__c records are linked to the same PNA (e.g., separate schedules per CPT code group),
When the deactivation flow runs,
Then ALL linked Fee_Schedule__c records are deactivated in a single bulk operation without hitting DML limits.

Given a Fee_Schedule__c record is deactivated,
When a new Claim_Line__c adjudication attempt references that Fee_Schedule__c,
Then the FeeScheduleEngine returns a null contracted rate and Claim_Line__c.Rate_Type__c is set to 'Out-of-Network Default'.

Given a Fee_Schedule__c is deactivated,
When a Provider Network Specialist attempts to manually reactivate it without an active PNA,
Then a Validation Rule blocks the save with message: 'Cannot activate Fee Schedule without an active Provider Network Agreement.'

-Given PNA has a grace period extension, defer fee schedule deactivation by PNA.Grace_Period_Days__c- (deprecated: grace period logic descoped from Phase 2, deferred to Phase 3)

## Solution

Record-Triggered Flow on Network_Termination_Case__c fires after NTC.Status__c changes to 'Closed'. Flow uses a Get Records element to retrieve all Fee_Schedule__c records where PNA__c = NTC.PNA_Reference__c, then a Loop with Update Records to bulk-deactivate. For providers with >200 fee schedule records, flow invokes an Apex Action (FeeScheduleDeactivationBatch) to handle bulk DML within governor limits. Validation Rule added to Fee_Schedule__c object: AND(Active__c = true, NOT(ISBLANK(PNA__c)), PNA__r.Status__c != 'Active'). FeeScheduleEngine Apex class updated to check Active__c flag before returning contracted rate.

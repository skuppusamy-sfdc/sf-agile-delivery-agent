## Description

As a Compliance Officer, I need the system to automatically dispatch a Provider_Change_Notification__c (PCN) letter via DocuSign to the provider's NPI-registered contact email when a Network_Termination_Case__c status is updated to 'Approved for Termination', so that we satisfy the contractual and regulatory requirement to provide written notice within the mandated notice period (typically 60–90 days depending on LOB).

## Acceptance Criteria

Given a Network_Termination_Case__c where NTC.Status__c is changed to 'Approved for Termination',
When the Record-Triggered Flow fires,
Then a DocuSign envelope is created and sent to the email address stored in Provider__c.NPI_Contact_Email__c within 15 minutes of the status change.

Given the DocuSign envelope is generated,
When the envelope is dispatched,
Then the linked Provider_Change_Notification__c record is updated:
  - PCN.Status__c = 'Dispatched'
  - PCN.Dispatch_Timestamp__c = DateTime.now()
  - PCN.DocuSign_Envelope_ID__c = [returned envelope ID]

Given the LOB on the NTC is 'Medicare Advantage',
When the PCN template is selected,
Then the DocuSign template ID mapped in the LOB_PCN_Template_Mapping__mdt custom metadata record for LOB = 'Medicare Advantage' is used.

Given the DocuSign callout fails or times out,
When the Platform Event handler catches the exception,
Then PCN.Status__c is set to 'Dispatch Failed', an error is logged in Application_Log__c, and the assigned Provider Relations Specialist is notified via in-app notification.

Given PCN.Status__c = 'Dispatched',
When the provider electronically acknowledges receipt via DocuSign,
Then PCN.Acknowledged__c = true and PCN.Acknowledgement_Date__c is populated.

Given the NTC.LOB__c = 'Workers Compensation',
When the notice period is calculated,
Then NTC.Notice_Period_Days__c is set to the value in LOB_PCN_Template_Mapping__mdt.Notice_Period_Days__c (expected: 90 days for Workers Compensation).

## Solution

Record-Triggered Flow on Network_Termination_Case__c triggers on Status__c change to 'Approved for Termination'. Flow invokes an Apex Action (DocuSignPCNDispatchService) that constructs the DocuSign REST API envelope payload, selects the correct template from LOB_PCN_Template_Mapping__mdt custom metadata, and fires the callout asynchronously via a Queueable Apex class to avoid governor limit conflicts. DocuSign Connect webhook callback updates PCN record via a REST endpoint exposed as a Salesforce Site guest-user accessible Apex REST resource. Error handling via Platform Event (TerminationDispatchError__e) consumed by a separate Apex trigger that logs and notifies.

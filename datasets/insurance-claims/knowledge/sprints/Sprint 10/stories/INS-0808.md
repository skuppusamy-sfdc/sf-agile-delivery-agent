## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record when a Provider_Network_Agreement__c reaches its end date or when a voluntary exit request is submitted, so that the termination workflow is initiated without manual intervention and SLA timelines are preserved.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Status__c = 'Active' and PNA.End_Date__c equals TODAY()
When a scheduled batch job runs at 06:00 AM daily
Then a Network_Termination_Case__c is created with Termination_Type__c = 'Contract Expiry', linked via NTC.Provider_Network_Agreement__c lookup, and NTC.Status__c = 'Initiated'

Given a Provider Relations Specialist sets PNA.Exit_Request_Flag__c = TRUE and populates PNA.Voluntary_Exit_Reason__c
When the record is saved
Then a Record-Triggered Flow creates a Network_Termination_Case__c with Termination_Type__c = 'Voluntary Exit' within 5 seconds and assigns it to the Provider Network queue

Given a Network_Termination_Case__c is created
When the record is inserted
Then NTC.Effective_Termination_Date__c is auto-populated as PNA.End_Date__c + 30 days (notice period) and a Provider_Change_Notification__c (PCN) is created in Draft status

Given a PNA is associated with multiple LOBs
When the termination case is created
Then one Network_Termination_Case__c child record is generated per LOB entry in PNA.Line_of_Business__c multi-select picklist

-Given PNA.Auto_Renew__c = TRUE, suppress termination case creation- (deprecated: auto-renew logic moved to INS-0791)

Given a duplicate termination case already exists for the same PNA within the current calendar year
When the batch job attempts to create another
Then creation is blocked and a Platform Event (Termination_Duplicate_Alert__e) is fired to notify the Provider Network Specialist via bell notification

## Solution

Implement a Scheduled Apex batch class (BatchProviderTerminationInitiator) running daily at 06:00 AM to query PNA records nearing expiry. Supplement with a Record-Triggered Flow on Provider_Network_Agreement__c for voluntary exit path (fires on Update when Exit_Request_Flag__c changes to TRUE). Use a Platform Event (Termination_Duplicate_Alert__e) for duplicate suppression notifications. Network_Termination_Case__c creation logic encapsulated in an invocable Apex method (TerminationCaseService.createFromPNA()) to allow reuse across both paths. Custom metadata (Termination_Config__mdt) drives notice period days by LOB.

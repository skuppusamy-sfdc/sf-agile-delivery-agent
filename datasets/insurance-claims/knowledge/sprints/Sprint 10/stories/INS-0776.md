## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record when a Provider_Network_Agreement__c reaches its termination date or when a provider submits a voluntary exit request, so that the network exit workflow is initiated without manual intervention and SLA timelines are preserved.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Status__c = 'Active' and PNA.Termination_Date__c equals today's date,
When the scheduled batch job runs at 00:30 system time,
Then a Network_Termination_Case__c is created with Termination_Case.Origin__c = 'System-Auto', Termination_Case.Reason__c = 'Contract Expiry', Termination_Case.PNA_Lookup__c populated, and Termination_Case.Status__c = 'Initiated'.

Given a Provider Relations Specialist sets PNA.Exit_Type__c = 'Voluntary' and PNA.Voluntary_Exit_Date__c to a future date,
When the record is saved,
Then a Network_Termination_Case__c is created with Termination_Case.Reason__c = 'Voluntary Exit' and Termination_Case.Effective_Date__c = PNA.Voluntary_Exit_Date__c.

Given a Network_Termination_Case__c is created,
When the case is saved,
Then a Provider_Change_Notification__c (PCN) is auto-generated with PCN.Notification_Type__c = 'Termination', PCN.Provider__c linked to the associated Provider__c, and PCN.Send_Date__c = today + 90 days per regulatory requirement.

Given a PNA where PNA.LOB__c = 'Medicare Advantage',
When a termination case is initiated,
Then Termination_Case.CMS_Notification_Required__c = TRUE and a compliance task is created assigned to the Compliance Officer role queue.

-Given duplicate PNA records exist for the same NPI, When termination fires, Then all linked PNAs are terminated simultaneously- (superseded by INS-0761 deduplication logic; only primary PNA triggers auto-termination).

## Solution

Implement a Record-Triggered Flow on Provider_Network_Agreement__c for voluntary exit path (Entry Criteria: PNA.Exit_Type__c = 'Voluntary', trigger on create/update). Build a Scheduled Apex batch job (TerminationExpiryBatch) running nightly to query PNAs with Termination_Date__c = TODAY and Status__c = 'Active', then invoke a @future Apex method to create Network_Termination_Case__c records in bulk. Use a Platform Event (Network_Exit_Event__e) to decouple PCN creation from the termination case insert. Deploy a separate Record-Triggered Flow on Network_Termination_Case__c to evaluate LOB__c and set CMS_Notification_Required__c. Assign tasks via a Custom Metadata-driven routing table mapping LOB to queue.

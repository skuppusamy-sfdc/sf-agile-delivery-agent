## Description

As a Provider Network Specialist, I need a Network_Termination_Case__c to be automatically created when a Provider_Network_Agreement__c enters its contractual termination notice period so that the network exit workflow is triggered without manual intervention and compliance timelines are met.

This story extends the foundational PNA lifecycle work established in INS-0818 and adds automated downstream case generation.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Status__c = 'Active' and PNA.Termination_Notice_Date__c is reached (today >= Termination_Notice_Date__c)
When the scheduled Record-Triggered Flow evaluates the record at midnight batch
Then a new Network_Termination_Case__c is created with:
  - NTC.Provider__c = PNA.Provider__c
  - NTC.PNA__c = PNA.Id
  - NTC.Status__c = 'Initiated'
  - NTC.Termination_Effective_Date__c = PNA.Termination_Effective_Date__c
  - NTC.LOB__c = PNA.Line_of_Business__c
  - NTC.Assigned_To__c = PNA.Provider_Relations_Owner__c

Given a Network_Termination_Case__c is successfully created
When the record is saved
Then a Provider_Change_Notification__c (PCN) draft record is auto-created linked to the NTC with PCN.Status__c = 'Draft' and PCN.Notification_Type__c = 'Termination'

Given PNA.Status__c is already 'Terminated' or 'Expired'
When the scheduled flow evaluates the record
Then no duplicate Network_Termination_Case__c is created and a debug log entry is written

-Given PNA.Auto_Renew__c = true, suppress termination case creation- (deprecated per compliance review; auto-renew no longer blocks termination flow per INS-0831)

Given a Network_Termination_Case__c is created
When the record is inserted
Then PNA.Status__c is updated to 'Termination In Progress' and a Chatter post is added to the PNA record notifying the assigned Provider Relations Specialist

## Solution

Implement a Scheduled-Trigger Flow on Provider_Network_Agreement__c that fires daily. Flow uses a Get Records element to collect all PNAs where Status__c = 'Active' AND Termination_Notice_Date__c <= TODAY(). Loop element iterates each PNA and calls a sub-flow (Create_NTC_SubFlow) via Flow Reference. Sub-flow uses Create Records elements for both Network_Termination_Case__c and Provider_Change_Notification__c. An Apex invocable method handles the Chatter post (ConnectApi.ChatterFeeds) to avoid Flow governor limit concerns at scale. Custom metadata type (NTC_Config__mdt) stores configurable defaults for NTC.Status__c initial values per LOB. Platform Event (ProviderTerminationInitiated__e) is fired post-creation to notify downstream ClaimsProcessing component to flag active Claim__c records linked to the provider.

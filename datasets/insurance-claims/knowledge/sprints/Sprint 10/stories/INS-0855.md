## Description

As a Claims Operations Manager and Compliance Officer, I need a dedicated Salesforce CRM Analytics (CRMA) dashboard that provides real-time visibility into the provider network exit pipeline — including termination case volumes by status, overdue notice milestones, affected claim counts, and LOB breakdown — so that leadership can monitor compliance risk and resource allocation during network exit cycles without pulling manual reports.

## Acceptance Criteria

Given the 'Provider Network Exit Compliance' CRMA dashboard is accessed by a user with Permission Set 'PS_Claims_Operations_Manager' or 'PS_Compliance_Officer'
When the dashboard loads
Then it displays the following KPI tiles sourced from Network_Termination_Case__c and Provider_Change_Notification__c datasets:
  - Total Active Terminations (NTC.Status__c IN ('Initiated', 'In Review', 'Pending Effective'))
  - Overdue 90-Day Notices (NTC where 90-day notice PCN not sent and TODAY() > NTC.Termination_Effective_Date__c - 85 days)
  - Terminations by LOB (bar chart, NTC.LOB__c dimension)
  - Avg Days to Termination Completion (NTC.Actual_Termination_Date__c - NTC.CreatedDate)

Given the dashboard is rendered
When a user selects a LOB filter from the global filter bar
Then all tiles and charts refresh to reflect only Network_Termination_Case__c records matching the selected NTC.LOB__c value

Given the 'Affected Claims' tile is displayed
When the tile value is clicked
Then a drill-through opens a filtered Claim__c list view showing all claims where Claim__c.Status__c = 'Suspended - Provider Termination' linked to the filtered termination cohort

Given the dashboard is embedded on the 'Provider Network Operations' Lightning App Home Page
When a user without PS_Claims_Operations_Manager or PS_Compliance_Officer permission sets accesses the page
Then the dashboard widget is hidden and replaced with an 'Access Restricted' message component

-Include a Fee_Schedule__c impact analysis tile showing affected CPT codes per termination- (deferred to Sprint 12 pending FeeScheduleEngine dataset availability)

## Solution

Create a CRMA (Einstein Analytics) dashboard using CRMA Designer. Build two CRMA datasets via Dataflow: 'DS_NetworkTerminationCases' (source: Network_Termination_Case__c + Provider__c + Provider_Change_Notification__c joined on NTC.Id = PCN.NTC__c) and 'DS_SuspendedClaims' (source: Claim__c filtered on Status__c = 'Suspended - Provider Termination'). KPI tiles use SAQL queries with conditional coloring (amber/red thresholds configured in dashboard JSON). Drill-through to Claim__c list view implemented via CRMA Link step with URL template pointing to filtered Salesforce list view. Global LOB filter widget uses a CRMA Filter step bound to NTC.LOB__c across all connected steps. Dashboard visibility controlled via CRMA Sharing: share with Permission Set Groups 'PSG_Claims_Operations' and 'PSG_Compliance'. LWC wrapper component (networkExitDashboardEmbed) handles the access restriction logic using @wire(getPermissionSetAssignments) and conditionally renders the CRMA embedded dashboard or the restricted-access message.

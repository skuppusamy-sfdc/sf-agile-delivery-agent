# INS-0836: Auto-suspend open Claim__c lines linked to terminated provider upon network exit confirmation

## Description

As a Claims Operations Manager, I need the system to automatically identify and suspend all open Claim__c records and associated Claim_Line__c items that are linked to a provider whose Network_Termination_Case__c has been confirmed, so that payments are not erroneously disbursed to out-of-network providers post-termination and adjudication queues are updated for manual review.

## Acceptance Criteria

Given a Network_Termination_Case__c transitions to NTC.Status__c = 'Confirmed' with a valid NTC.Effective_Date__c,
When the status change is saved,
Then all Claim__c records where Claim.Rendering_Provider__c = NTC.Provider__c AND Claim.Status__c IN ('Submitted', 'In Adjudication', 'Pending Payment') AND Claim.Service_Date__c >= NTC.Effective_Date__c are updated to Claim.Status__c = 'Suspended - Provider Termination'.

Given Claim__c records are suspended,
When the bulk update completes,
Then all child Claim_Line__c records for those claims are updated to Claim_Line.Status__c = 'Suspended' and Claim_Line.Suspension_Reason__c = 'Provider Network Exit'.

Given Claim__c records are suspended,
When the update completes,
Then any associated Payment__c records in Payment.Status__c = 'Scheduled' are set to Payment.Status__c = 'On Hold' and Payment.Hold_Reason__c = 'Provider Termination Pending Review'.

Given the suspension batch runs,
When more than 50 claims are suspended in a single execution,
Then a Platform Event (Claims_Suspension_Summary__e) is fired with payload fields: Provider_Id__c, Total_Claims_Suspended__c, Total_Payment_Value_Held__c, and Execution_Timestamp__c.

Given a Claim__c with Service_Date__c < NTC.Effective_Date__c (pre-termination service),
When the suspension logic evaluates the claim,
Then the claim is excluded from suspension and remains in its current status to allow normal adjudication.

Given a suspended Claim__c,
When a Claims Operations Manager manually reviews and overrides the suspension,
Then Claim.Status__c can be updated to 'Override - Manual Review' only by users with the 'Claims_Termination_Override' Permission Set, and an audit log entry is created on the claim.

## Solution

Implement an Apex Trigger (after update) on Network_Termination_Case__c that fires when Status__c changes to 'Confirmed', calling a queueable Apex class (ProviderTerminationClaimSuspensionQueueable) to handle bulk claim and claim line suspension in governor-limit-safe batches. Use Database.update with allOrNone = false and log failures to a custom error object. Fire Claims_Suspension_Summary__e Platform Event post-execution. Payment hold logic implemented via a separate invocable Apex method called from the queueable. Build a Permission Set ('Claims_Termination_Override') and restrict status override via Apex validation. Add field-level history tracking on Claim__c.Status__c and Claim_Line__c.Status__c.

## Description

As a Provider Network Specialist, I need the system to identify and flag all Group_Service_Agreement__c (GSA) records where a terminating provider is listed as a participating member, so that the Broker Relations Coordinator and Provider Relations Specialist can review whether the GSA's minimum provider panel requirements are still met post-termination and take corrective action before the exit becomes effective.

## Acceptance Criteria

Given a Network_Termination_Case__c with Status__c = 'Approved' and a linked Provider__c
When the record is saved in 'Approved' status
Then a Record-Triggered Flow queries all Group_Service_Agreement__c records where the provider appears in the GSA_Provider_Members__c junction object and GSA.Status__c = 'Active'

Given one or more GSA records are identified
When the flow completes
Then each GSA record's GSA_Review_Required__c checkbox is set to TRUE and a Task is created per GSA assigned to the Provider Relations Specialist queue with Subject = 'GSA Panel Review – Terminating Provider', Priority = 'High', and a reference to the Network_Termination_Case__c

Given a GSA where Active_Provider_Count__c minus 1 (post-termination) falls below GSA.Minimum_Panel_Size__c
When the review task is generated
Then GSA.Panel_Breach_Risk__c is set to TRUE and an additional Task is created for the Compliance Officer with Subject = 'Compliance Alert – GSA Minimum Panel Breach Risk'

Given a GSA with GSA.LOB__c = 'Medicare Advantage'
When Panel_Breach_Risk__c = TRUE
Then a ComplianceReporting Platform Event (CMS_Panel_Alert__e) is published with GSA.CMS_Contract_Number__c and Provider__c.NPI__c payload fields for downstream CMS reporting integration

-Auto-terminate GSA if panel breach is confirmed- (deprecated: legal review required; manual action only per compliance directive dated 2024-11-02)

## Solution

Record-Triggered Flow on Network_Termination_Case__c (after update, Status__c = 'Approved') calls an Apex invocable action 'GSACascadeReviewAction' that performs a bulkified SOQL query on the GSA_Provider_Members__c junction object. Panel size calculation done in Apex with formula: Active_Provider_Count__c - 1 compared against Minimum_Panel_Size__c. Task creation via Apex DML in bulk. Platform Event CMS_Panel_Alert__e published via EventBus.publish() for Medicare Advantage cases. Custom List View 'GSA Panel Review Required' added to GSA object filtered by GSA_Review_Required__c = TRUE for ops team visibility. Permission Set Group 'Provider_Network_Ops' updated to include read/write on GSA_Review_Required__c and Panel_Breach_Risk__c.

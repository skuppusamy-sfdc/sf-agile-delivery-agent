## Description

As a Provider Relations Specialist, I need a guided OmniScript that walks me through the 10-step provider network termination intake process — including reason capture, LOB impact assessment, regulatory notification scheduling, open claims review, and DocuSign termination letter dispatch — so that all exit activities are standardized, auditable, and compliant with state and CMS notification timelines.

## Acceptance Criteria

Given a Provider Relations Specialist launches the 'Network Exit Intake' OmniScript from the Provider__c record action menu,
When Step 1 (Provider Identification) loads,
Then the script pre-populates Provider.NPI__c, Provider.TIN__c, Provider.Name, and a read-only list of all Active PNA records for that provider.

Given Step 3 (Termination Reason),
When the specialist selects Termination_Reason__c = 'For Cause',
Then Step 4 conditionally renders a 'SIU Referral Required' checkbox and a mandatory SIU_Case_Number__c text field.

Given Step 6 (Open Claims Impact),
When the OmniScript DataRaptor queries Claim__c records where Claim.Rendering_Provider__c = current provider and Claim.Status__c IN ('Received','In Adjudication','Pended'),
Then a count and list of open claims is displayed with Claim.Claim_Number__c, Claim.Date_of_Service__c, and Claim.Total_Billed_Amount__c.

Given Step 8 (Regulatory Notification),
When PNA.LOB__c includes 'Medicare Advantage' or 'Medicaid',
Then the system auto-calculates PCN.Send_Date__c = today + 90 days and displays a non-editable compliance warning banner citing 42 CFR 422.202.

Given Step 10 (Termination Letter),
When the specialist clicks 'Send for Signature',
Then a DocuSign envelope is generated using the 'Provider_Termination_Letter' template, pre-filled with provider and PNA data, and sent to Provider.Email__c; Network_Termination_Case__c.DocuSign_Status__c is set to 'Sent'.

Given the OmniScript completes all 10 steps,
When the specialist clicks 'Submit',
Then Network_Termination_Case__c.Status__c = 'In Review', all captured data is persisted via a DataRaptor Turbo Action, and a confirmation screen displays the case number.

## Solution

Build a 10-step OmniScript 'NetworkExitIntake' in OmniStudio. Use DataRaptor Extract actions for pre-population of Provider__c, PNA, and open Claim__c data at Steps 1 and 6. Implement conditional step visibility using OmniScript Conditional Views driven by Termination_Reason__c picklist value. Integrate DocuSign via the OmniStudio DocuSign integration action at Step 10, referencing the 'Provider_Termination_Letter' template ID stored in Custom Metadata. Use a DataRaptor Turbo Action for final record creation/update across Network_Termination_Case__c and Provider_Change_Notification__c. Deploy the OmniScript as an Action on the Provider__c page layout via a Quick Action LWC wrapper.

## Description

As a Credentialing Specialist, I need the system to validate that the NPI on Provider__c.NPI__c matches the active NPI record in the NPPES registry at the time a Network_Termination_Case__c is created, so that termination notices are not sent to providers with stale or incorrect NPI data and Acme Insurance avoids compliance exposure from misdirected communications.

## Acceptance Criteria

Given a Network_Termination_Case__c is created and linked to a Provider__c record
When the case Status__c transitions from 'Draft' to 'Pending Review'
Then an asynchronous callout is made to the NPPES NPI Registry API (endpoint: /api/?number={NPI}) and the response is stored in a new NPI_Validation_Response__c child record

Given the NPPES API returns a valid active NPI matching Provider__c.NPI__c
When the validation completes
Then Network_Termination_Case__c.NPI_Validation_Status__c = 'Validated' and the case proceeds in workflow

Given the NPPES API returns a deactivated NPI or no match
When the validation completes
Then NPI_Validation_Status__c = 'Mismatch - Manual Review Required', the case is assigned to the 'Data Quality' queue, and a warning banner is displayed on the case record via a Lightning Web Component

Given the NPPES API is unavailable (HTTP 500 or timeout)
When the callout fails
Then NPI_Validation_Status__c = 'Validation Pending - API Unavailable', a retry is scheduled via a Schedulable Apex job within 4 hours, and the termination workflow is NOT blocked

## Solution

Implement a Queueable Apex class 'NPPESValidationQueueable' triggered by a Record-Triggered Flow on Network_Termination_Case__c (After Update, when Status__c changes to 'Pending Review'). The queueable performs an HTTP callout to NPPES via Named Credential 'NPPES_Registry'. Response JSON is parsed and written to NPI_Validation_Response__c. Retry logic uses a custom object 'API_Retry_Queue__c' processed by a Scheduled Apex job running every 4 hours. Warning banner LWC 'npiValidationAlert' uses @wire to read NPI_Validation_Status__c and conditionally renders a lightning-badge. Remote Site Setting for NPPES endpoint added to org configuration.

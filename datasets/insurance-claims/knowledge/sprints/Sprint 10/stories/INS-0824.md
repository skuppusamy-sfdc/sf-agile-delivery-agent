## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record when a Provider_Network_Agreement__c reaches its expiry date or when a provider submits a voluntary exit request, so that the termination workflow is initiated without manual intervention and compliance timelines are met.

This story extends INS-0794 (PNA lifecycle management) by adding the terminal state transitions.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Status__c = 'Active' and PNA.End_Date__c equals today's date
When the nightly scheduled flow executes
Then a Network_Termination_Case__c is created with Termination_Type__c = 'Contract Expiry', Status__c = 'Initiated', and linked via Network_Termination_Case__c.PNA__c to the originating PNA

Given a Provider_Network_Agreement__c where PNA.Voluntary_Exit_Requested__c = true and PNA.Exit_Request_Date__c is populated
When the record is saved
Then a Network_Termination_Case__c is created immediately via Record-Triggered Flow with Termination_Type__c = 'Voluntary Exit' and Requested_Effective_Date__c = PNA.Exit_Request_Date__c + 90 days (per contractual notice period)

Given a Network_Termination_Case__c is created
When the record is inserted
Then a Platform Event (Provider_Termination_Event__e) is published with Provider_NPI__c, TIN__c, Termination_Type__c, and Effective_Date__c populated for downstream consumption by ClaimsProcessing and FeeScheduleEngine components

Given a Network_Termination_Case__c is created
When Status__c = 'Initiated'
Then the assigned Provider Network Specialist receives a Task with Subject = 'Review Termination: [Provider Name]' and Due_Date__c = today + 5 business days

-Given PNA.Auto_Renew__c = true, suppress Network_Termination_Case__c creation- (deprecated: auto-renew logic moved to INS-0831)

Given a Network_Termination_Case__c already exists for a PNA in Status__c IN ('Initiated','In Review','Pending Approval')
When the scheduled flow attempts to create a duplicate
Then no new record is created and a warning log entry is written to Integration_Log__c with Log_Type__c = 'Duplicate Suppressed'

## Solution

Implement two entry points: (1) A Scheduled-Triggered Flow running nightly at 02:00 UTC querying PNA records where End_Date__c = TODAY and Status__c = 'Active' — iterates and calls an invocable Apex method 'TerminationCaseFactory.createFromPNA()' to bulkify DML. (2) A Record-Triggered Flow on Provider_Network_Agreement__c, AFTER SAVE, on create/update, fires when Voluntary_Exit_Requested__c changes to true. Both paths call the same Apex service class. Platform Event (Provider_Termination_Event__e) published via Apex EventBus.publish(). Duplicate guard implemented via SOQL before-insert check in Apex. Custom metadata type (Termination_Config__mdt) drives the 90-day notice period to avoid hardcoding.

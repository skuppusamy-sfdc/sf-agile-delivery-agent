## Description

**Bug Report**

**Environment:** Full Sandbox (Acme-FS2)
**Discovered:** Sprint 10 Day 3 regression testing

**Steps to Reproduce:**
1. Create a Provider_Network_Agreement__c with PNA.Status__c = 'Active' and PNA.Termination_Effective_Date__c = yesterday's date.
2. Confirm a Network_Termination_Case__c exists linked to the PNA with NTC.Status__c = 'Pending Effective'.
3. Allow the nightly scheduled flow 'SF_NTC_StatusPromotion' to execute (or manually run via Dev Console).
4. Observe NTC.Status__c after flow execution.

**Expected Result:** NTC.Status__c transitions to 'Terminated' and PNA.Status__c updates to 'Terminated' with PNA.Actual_Termination_Date__c stamped to the effective date.

**Actual Result:** NTC.Status__c remains 'Pending Effective'. PNA.Status__c is unchanged. No error is thrown; the flow exits silently.

**Root Cause Hypothesis:** The Get Records filter in 'SF_NTC_StatusPromotion' uses `NTC.Termination_Effective_Date__c = TODAY()` (exact match) rather than `<= TODAY()`, causing records whose effective date was in the past (e.g., weekend dates when flow did not execute) to be skipped permanently.

**Impact:** High — providers whose termination effective date fell on a weekend or system downtime window remain incorrectly active in the network, creating claims payment eligibility risk.

## Acceptance Criteria

As per Description. Fix verified when NTC.Status__c = 'Terminated' and PNA.Status__c = 'Terminated' are correctly set for all records where NTC.Termination_Effective_Date__c <= TODAY() after scheduled flow execution. Regression test must confirm records with past effective dates (up to 7 days prior) are also correctly promoted.

## Solution

Update the Get Records SOQL filter in the 'SF_NTC_StatusPromotion' scheduled flow from an equality operator (= TODAY()) to a less-than-or-equal operator (<= TODAY()) on NTC.Termination_Effective_Date__c. Add a secondary filter to exclude records already in NTC.Status__c IN ('Terminated', 'Cancelled') to prevent redundant processing. Add a fault path on the Update Records element that writes to a custom object 'Flow_Error_Log__c' with error message and record ID. Deploy fix to Full Sandbox, execute flow manually, validate via SOQL query in Dev Console. Promote to UAT after QA sign-off.

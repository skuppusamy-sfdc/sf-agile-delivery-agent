# INS-0860: Suppress claim payments to terminated providers: block Payment__c creation when PNA_Status__c = 'Terminated'

- **Sprint**: Acme Insurance Sprint 10 (Phase 2)
- **Status**: Done
- **Type**: Story
- **Priority**: High
- **Resolution**: Done
- **Assignee**: Anika Gupta
- **Reporter**: Marcus Johnson

- **Story Points**: 5
- **Labels**: Phase2, PaymentEngine, ProviderTermination, Compliance

## Description

As a Claims Operations Manager, I need the payment engine to automatically suppress Payment__c record creation for any Claim__c linked to a provider whose PNA_Status__c = 'Terminated' as of the Date_of_Service__c, so that Acme Insurance does not disburse funds to out-of-network providers post-termination and avoids regulatory penalties for improper payments.

## Acceptance Criteria

Given a Claim__c record where Rendering_Provider__c links to a Provider__c whose active PNA has PNA_Status__c = 'Terminated' AND PNA.Termination_Date__c <= Claim__c.Date_of_Service__c
When the payment adjudication engine attempts to create a Payment__c record
Then the Payment__c insert is blocked, Claim__c.Payment_Suppression_Reason__c is set to 'Provider Network Terminated', and Claim__c.Status__c transitions to 'Pended - Provider Termination'

Given Claim__c.Status__c = 'Pended - Provider Termination'
When the record is saved
Then a Task is auto-created and assigned to the Claims Operations Manager queue with Subject = 'Pended Payment: Terminated Provider' and a link to the originating Claim__c

Given a Claim__c with Date_of_Service__c BEFORE the PNA.Termination_Date__c (valid service window)
When the payment engine evaluates the claim
Then the payment suppression logic does NOT fire and Payment__c creation proceeds normally per standard adjudication rules

Given a Payment__c that was created prior to PNA termination and is in Payment__c.Status__c = 'Scheduled'
When the PNA.PNA_Status__c is updated to 'Terminated'
Then a Platform Event (PaymentReviewRequired__e) is published for all linked scheduled payments, triggering a manual review task rather than an automatic void

-Given TPA-administered claims, suppression should apply only if TPA_Flag__c = false- (removed: TPA suppression rules standardized globally per compliance directive CR-2024-118)

## Solution

Implement Apex Trigger 'PaymentInsertTrigger' (Before Insert) on Payment__c. Handler class 'PaymentTerminationGuard' queries Provider_Network_Agreement__c WHERE Provider__c = claim.Rendering_Provider__c AND PNA_Status__c = 'Terminated' AND Termination_Date__c <= claim.Date_of_Service__c. If match found, add Trigger.new record to addError() list with custom error message and invoke a separate method to update Claim__c fields via a queueable Apex job (to avoid mixed DML). Platform Event PaymentReviewRequired__e published via EventBus.publish() in an After Update trigger path for scheduled payments. Custom Metadata Type 'PaymentSuppressionConfig__mdt' stores suppression-enabled LOB codes for runtime toggle without code deployment.

## Description

**Bug Report**

**Summary:** Network_Termination_Case__c.Status__c is not automatically transitioning to 'Closed' after all linked Provider_Change_Notification__c records have PCN.Acknowledged__c = true, leaving NTCs perpetually in 'Approved for Termination' status and blocking downstream reporting.

**Steps to Reproduce:**
1. Open an existing Network_Termination_Case__c record where NTC.Status__c = 'Approved for Termination'.
2. Navigate to the related PCN list and confirm all PCN records show PCN.Acknowledged__c = true and PCN.Status__c = 'Dispatched'.
3. Wait for the scheduled Flow or trigger to evaluate the NTC closure condition.
4. Observe that NTC.Status__c remains 'Approved for Termination' and does not transition to 'Closed'.

**Expected Behavior:** When all related Provider_Change_Notification__c records linked to a Network_Termination_Case__c have PCN.Acknowledged__c = true, the NTC.Status__c should automatically update to 'Closed' and NTC.Closed_Date__c should be set to today's date.

**Actual Behavior:** NTC.Status__c remains 'Approved for Termination'. NTC.Closed_Date__c is null. No error is visible in debug logs at first inspection.

**Root Cause (Preliminary):** The Roll-Up Summary field NTC.All_PCNs_Acknowledged__c on Network_Termination_Case__c appears to be returning false even when all child PCNs are acknowledged. Suspected issue: the Roll-Up Summary filter criteria includes PCN.Status__c = 'Acknowledged' but the workflow sets PCN.Status__c = 'Dispatched' (not 'Acknowledged') even when PCN.Acknowledged__c = true. Filter mismatch in the Roll-Up Summary definition.

**Environment:** Acme Insurance Full Sandbox (ACME_FS1)
**Regression:** Introduced in Sprint 9 deployment on 2024-11-14.

## Acceptance Criteria

As per Description. Additionally: After fix, NTC.Status__c transitions to 'Closed' within 5 minutes of the last PCN.Acknowledged__c being set to true, and NTC.Closed_Date__c = TODAY.

## Solution

Fix the Roll-Up Summary field NTC.All_PCNs_Acknowledged__c filter criteria to evaluate PCN.Acknowledged__c = true (checkbox field) rather than PCN.Status__c = 'Acknowledged'. Alternatively, if Roll-Up Summary recalculation latency is the root cause, replace with an Apex-based roll-up in the PCN after-update trigger that checks all sibling PCN records and directly updates the parent NTC. Update the Record-Triggered Flow on Network_Termination_Case__c to evaluate NTC.All_PCNs_Acknowledged__c = true as the closure condition trigger. Deploy fix to sandbox, execute unit tests, and re-test end-to-end scenario before promoting to production.

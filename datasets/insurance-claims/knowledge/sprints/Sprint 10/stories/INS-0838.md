## Description

**Bug Report**

**Summary:** When a Provider_Network_Agreement__c is terminated and the linked Network_Termination_Case__c reaches Status__c = 'Confirmed', the associated Fee_Schedule__c records and their child Fee_Schedule_Line__c items are NOT being deactivated. This causes the FeeScheduleEngine to continue returning in-network reimbursement rates for CPT codes billed by the terminated provider, resulting in incorrect EOB calculations and potential overpayments.

**Steps to Reproduce:**
1. Identify a Provider__c with an Active PNA and at least one Fee_Schedule__c with Fee_Schedule.Status__c = 'Active' and multiple Fee_Schedule_Line__c records.
2. Create a Network_Termination_Case__c linked to the PNA and advance NTC.Status__c to 'Confirmed'.
3. Submit a new Claim__c for the same provider with a Service_Date__c = NTC.Effective_Date__c + 1.
4. Run adjudication on the claim.
5. Open the resulting Explanation_of_Benefits__c (EOB) record.
6. Observe that EOB.Allowed_Amount__c reflects in-network rates from the now-terminated Fee_Schedule__c.

**Expected Result:** Fee_Schedule__c.Status__c should be set to 'Inactive' and all child Fee_Schedule_Line__c.Active__c = false upon NTC confirmation, causing adjudication to apply out-of-network rates.

**Actual Result:** Fee_Schedule__c.Status__c remains 'Active'. Root cause identified: the Apex Trigger on Network_Termination_Case__c (NTCStatusTrigger) has a missing branch for fee schedule deactivation in the handleConfirmed() method — the method stub exists but contains only a TODO comment.

**Environment:** Acme UAT Sandbox (CS112) and reproduced in Acme Full Sandbox (CS089)
**Financial Impact:** High — 3 claims in UAT were adjudicated at incorrect in-network rates totaling $4,280 in simulated overpayment.

## Acceptance Criteria

As per Description.

Additional verification:
- Fee_Schedule__c.Status__c = 'Inactive' for all fee schedules linked to the terminated PNA after NTC.Status__c = 'Confirmed'.
- All child Fee_Schedule_Line__c.Active__c = false.
- New Claim__c adjudicated after termination effective date returns out-of-network rates on EOB.
- Existing claims with Service_Date__c < NTC.Effective_Date__c are not affected.

## Solution

Implement the missing handleConfirmed() logic in NTCStatusTrigger's Apex handler class (NTCStatusHandler). Query all Fee_Schedule__c records where Fee_Schedule.Provider_Network_Agreement__c = NTC.Provider_Network_Agreement__c and Status__c = 'Active'. Bulk update Fee_Schedule__c.Status__c = 'Inactive' and all related Fee_Schedule_Line__c.Active__c = false using a child subquery. Wrap in a Queueable to avoid governor limit issues with large fee schedules. Add Apex test methods covering: (a) fee schedule deactivation on NTC confirmation, (b) no deactivation on other status transitions, (c) claim adjudication rate validation post-deactivation using mock FeeScheduleEngine responses. Deploy to both CS112 and CS089, re-run impacted UAT claim scenarios.

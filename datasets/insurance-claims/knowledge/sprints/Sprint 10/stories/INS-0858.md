## Description

Steps to Reproduce:
1. Navigate to an existing Network_Termination_Case__c record with Status__c = 'Pending Review'.
2. Open the linked Provider_Network_Agreement__c and set PNA_Status__c = 'Active' (simulating a reinstatement approval).
3. Save the PNA record.
4. Return to the Network_Termination_Case__c record.

Expected Behavior: Network_Termination_Case__c.Status__c should automatically update to 'Closed - Rescinded' and a closure timestamp should be written to Case_Closed_Date__c.

Actual Behavior: Network_Termination_Case__c.Status__c remains 'Pending Review'. No update occurs. Case_Closed_Date__c is null.

Environment: Acme Insurance Full Sandbox (FS-01), API v59.0
First observed: Sprint 10 Day 3 regression run
Frequency: 100% reproducible
Impact: Provider Relations Specialists must manually close termination cases, creating audit trail gaps for compliance reporting.

## Solution

Root cause: The Record-Triggered Flow 'PNA_StatusChange_Flow' on Provider_Network_Agreement__c (After Update) has a broken field reference — the lookup to Network_Termination_Case__c uses PNA.Termination_Case__c (null for legacy records) instead of querying via PNA_Lookup__c on the case. Fix: Add a Get Records element to retrieve the related Network_Termination_Case__c where PNA_Lookup__c = PNA.Id AND Status__c NOT IN ('Closed - Rescinded','Closed - Completed'). Add Update Records element setting Status__c = 'Closed - Rescinded' and Case_Closed_Date__c = NOW(). Add null-safe check before Update Records to prevent flow fault if no case exists. Deploy fix to FS-01 for regression validation before production push.

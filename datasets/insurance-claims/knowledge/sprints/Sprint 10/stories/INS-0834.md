## Description

**Bug Report**

**Summary:** When a Network_Termination_Case__c is manually created by a Provider Network Specialist with Termination_Type__c = 'Involuntary', the Effective_Date__c field remains null even when the source Provider_Network_Agreement__c has a valid End_Date__c populated.

**Steps to Reproduce:**
1. Navigate to a Provider_Network_Agreement__c record with PNA.Status__c = 'Active' and PNA.End_Date__c = a future date.
2. Click 'Initiate Termination' quick action button.
3. In the OmniScript modal, select Termination_Type__c = 'Involuntary'.
4. Complete all required fields and click Submit.
5. Open the newly created Network_Termination_Case__c record.
6. Observe that NTC.Effective_Date__c is blank.

**Expected Result:** NTC.Effective_Date__c should be auto-populated from PNA.End_Date__c when Termination_Type__c = 'Involuntary'.

**Actual Result:** NTC.Effective_Date__c = null. The field mapping in the DataRaptor Transform ('DRT_CreateTerminationCase') is missing the Effective_Date__c output node for the 'Involuntary' branch.

**Environment:** Acme UAT Sandbox (CS112)
**Regression:** Was working in Sprint 9 UAT; broke after OmniScript patch deployed 2024-11-14.

## Acceptance Criteria

As per Description. Fix verified when NTC.Effective_Date__c is populated = PNA.End_Date__c for Termination_Type__c = 'Involuntary' upon OmniScript submission.

## Solution

Open DataRaptor Transform 'DRT_CreateTerminationCase' and add missing output mapping: source node PNA.End_Date__c → target field Network_Termination_Case__c.Effective_Date__c, scoped to the conditional branch where Termination_Type__c = 'Involuntary'. Re-deploy and regression test all three Termination_Type__c values ('Expiry', 'Voluntary', 'Involuntary') in UAT sandbox. Add a unit test in the Apex test class TestNTCCreationService to assert Effective_Date__c is not null for all termination type branches.

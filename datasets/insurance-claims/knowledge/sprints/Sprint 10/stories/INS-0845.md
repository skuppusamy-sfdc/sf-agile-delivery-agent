# INS-0845: Extend Fee_Schedule__c inactivation logic to automatically expire rates on provider network exit date

## Description

As a Provider Network Specialist, I need all Fee_Schedule__c records linked to a terminating provider's Provider_Network_Agreement__c to be automatically inactivated on the NTC.Effective_Termination_Date__c, so that claims submitted after network exit are not inadvertently adjudicated at contracted rates and are instead processed at out-of-network reimbursement levels. Extends INS-0840.

## Acceptance Criteria

Given a Network_Termination_Case__c where NTC.Status__c = 'Termination Confirmed' and NTC.Effective_Termination_Date__c is populated
When a scheduled Apex job (ScheduledFeeScheduleInactivation) runs daily at 12:00 AM UTC
Then all Fee_Schedule__c records where:
  - FS.Provider_Network_Agreement__c = NTC.PNA_Id__c
  - FS.Effective_End_Date__c >= NTC.Effective_Termination_Date__c OR FS.Effective_End_Date__c = NULL
Are updated with:
  - FS.Effective_End_Date__c = NTC.Effective_Termination_Date__c
  - FS.Status__c = 'Inactive'
  - FS.Inactivation_Reason__c = 'Provider Network Exit'

Given FS.Status__c is already 'Inactive' prior to the job run
When the scheduled job evaluates the fee schedule
Then the record is skipped and no DML is performed

Given a Fee_Schedule__c inactivation is completed
When a new Claim_Line__c is created referencing the inactivated FS
Then Claim_Line__c.Fee_Schedule_Applied__c = NULL and Claim_Line__c.Reimbursement_Type__c = 'Out-of-Network'

-Given FS.LOB__c = 'Medicare Advantage', apply CMS rate floor before inactivation- (deferred to Phase 3 — CMS integration not in scope for Sprint 10)

## Solution

Implement Schedulable Apex class 'ScheduledFeeScheduleInactivation' that queries Network_Termination_Case__c records where Status__c = 'Termination Confirmed' and Effective_Termination_Date__c = TODAY. For each NTC, a sub-query retrieves related Fee_Schedule__c records via PNA_Id__c lookup. Batch update performed using Database.update with allOrNone = false; partial failures logged to Error_Log__c. Schedule registered via Anonymous Apex in deployment script (System.schedule). Claim_Line__c fee schedule resolution logic updated in existing Apex trigger (ClaimLineTriggerHandler) to check FS.Status__c before applying rates. Custom Metadata (FeeSchedule_Config__mdt) stores job schedule cron expression for environment-specific configuration.

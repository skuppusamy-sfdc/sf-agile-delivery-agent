## Description

As a Provider Network Specialist, I need a guided OmniScript to process involuntary (for-cause) provider terminations — including fraud, credentialing lapse, and license revocation — so that the termination is documented with the correct Termination_Reason__c, a DocuSign termination notice is dispatched to the provider's email on file, and the case is routed to the Compliance Officer queue for final sign-off.

## Acceptance Criteria

Given the Provider Network Specialist launches the 'Involuntary Termination' OmniScript from a Network_Termination_Case__c record
When Step 1 loads
Then the script pre-populates Provider__c.Name, Provider__c.NPI__c, PNA.Network_Name__c, and PNA.Contract_End_Date__c as read-only display fields

Given Step 2 'Select Termination Cause'
When the specialist selects a Termination_Reason__c value from the picklist ['Fraud/Abuse','License Revocation','Credentialing Lapse','Quality of Care','Mutual Agreement']
Then the following step is conditionally displayed: if 'Fraud/Abuse' is selected, an SIU_Referral_Number__c field becomes required; if 'Credentialing Lapse' is selected, the linked Credentialing_Case__c lookup becomes required

Given all required fields are populated and the specialist clicks 'Send Termination Notice'
When the OmniScript Integration Procedure fires
Then a DocuSign envelope is created via the DocuSign for Salesforce API using the 'Provider_Termination_Notice_v3' template, addressed to Provider__c.Email__c, with Effective_Termination_Date__c merged into the document body

Given the DocuSign envelope is successfully dispatched
When the Integration Procedure response returns envelope_status = 'sent'
Then Network_Termination_Case__c.DocuSign_Envelope_Id__c is populated, Notice_Sent_Date__c = TODAY(), and Status__c transitions to 'Notice Sent'

Given Status__c = 'Notice Sent'
When the record is saved
Then the case is automatically assigned to the 'Compliance Review' queue via an Assignment Rule, and a Task is created for the Compliance Officer with Subject = 'Review Involuntary Termination - [Provider NPI]' and ActivityDate = TODAY() + 5

Given the Compliance Officer approves the termination in the Approval Process
When the final approval step is completed
Then PNA.PNA_Status__c is updated to 'Terminated', PNA.Termination_Date__c = Effective_Termination_Date__c, and all active Fee_Schedule__c records linked to the PNA are end-dated with Fee_Schedule__c.End_Date__c = Effective_Termination_Date__c

## Solution

Build OmniScript 'InvoluntaryTerminationOS' (Type: ProviderTermination, SubType: ForCause) with 4 steps: PreFill, CauseSelection, ReviewAndSend, Confirmation. Use DataRaptor Load to write back to Network_Termination_Case__c. Integration Procedure 'SendTerminationNotice_IP' calls DocuSign REST API via Named Credential 'DocuSign_Prod' and parses envelope ID from response. Conditional step visibility controlled by OmniScript Conditional Views on Termination_Reason__c picklist value. Approval Process 'ComplianceTerminationApproval' configured with 2-step approval (Provider Network Manager → Compliance Officer). Record-Triggered Flow on Approval final approval step updates PNA and Fee_Schedule__c records via a loop. OmniScript deployed as a Quick Action on Network_Termination_Case__c Lightning Record Page.

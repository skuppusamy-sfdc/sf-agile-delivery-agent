## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces active Network_Termination_Case__c records, recent Provider_Change_Notification__c history, and current PNA status, so that I have a single consolidated view when handling provider exit calls without navigating multiple related lists. Extends INS-0826 (Provider 360 base FlexCard implementation).

## Acceptance Criteria

Given a Provider__c record with one or more Network_Termination_Case__c records in Status__c IN ('Pending Review','In Progress')
When the Provider Relations Specialist loads the Provider Account page
Then the FlexCard 'Active Terminations' tile displays Case Number, Status__c, Termination_Reason__c, and Effective_Date__c for each active case

Given a Provider__c with no active termination cases
When the FlexCard loads
Then the 'Active Terminations' tile displays the message 'No active termination cases' with a green status indicator

Given a Provider__c record
When the FlexCard loads
Then the PCN History section lists the 5 most recent Provider_Change_Notification__c records sorted by CreatedDate DESC, showing PCN_Type__c, Effective_Date__c, and PCN_Status__c

Given a Provider__c with PNA records
When the FlexCard renders
Then each PNA is shown with Network_Name__c, PNA_Status__c, Contract_Start_Date__c, Contract_End_Date__c, and a color-coded badge (green=Active, yellow=Pending Termination, red=Terminated)

-Given Provider NPI mismatch between NPPES and Provider__c.NPI__c, display a warning banner- (deferred to INS-0862 data quality story)

## Solution

Build OmniStudio FlexCard 'ProviderTermination360FC' using a DataRaptor Extract source querying Provider__c, Network_Termination_Case__c, Provider_Change_Notification__c, and Provider_Network_Agreement__c via SOQL with relationship traversal. Use Conditional Rendering on FlexCard states (empty, single, multi) for the Active Terminations tile. Badge colors implemented via CSS custom properties in the FlexCard theme. Deploy FlexCard to the Provider Account Lightning Record Page via a dynamic component with visibility rule: Profile IN ('Provider Relations','Network Specialist'). DataRaptor caching set to 30-second TTL to reduce SOQL governor limit exposure.

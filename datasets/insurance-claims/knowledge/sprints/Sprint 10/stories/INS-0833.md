## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page to surface any active Network_Termination_Case__c records and pending Provider_Change_Notification__c items inline, so that I can immediately assess a provider's network exit status without navigating to related lists. Extends INS-0819 (Provider 360 base FlexCard).

## Acceptance Criteria

Given a Provider__c record with one or more related Network_Termination_Case__c records where NTC.Status__c IN ('Initiated','In Review','Pending Approval'),
When the Provider 360 FlexCard loads,
Then a 'Termination Cases' tile displays NTC.Case_Number__c, NTC.Termination_Type__c, NTC.Effective_Date__c, and NTC.Status__c for each active case.

Given a Provider__c with no active termination cases,
When the FlexCard loads,
Then the tile renders with the message 'No active termination cases' and a green status indicator.

Given one or more Provider_Change_Notification__c records linked to the provider with PCN.Notification_Status__c = 'Pending',
When the FlexCard loads,
Then a 'Pending Notifications' badge displays the count and lists PCN.Change_Type__c and PCN.Created_Date__c.

Given the Provider Relations Specialist clicks 'View Details' on a termination case tile,
When the action fires,
Then the user is navigated to the Network_Termination_Case__c record page in the same tab.

-The FlexCard should also surface Credentialing_Case__c status inline- (superseded: credentialing tile scoped to INS-0840 in Sprint 11)

## Solution

Build an OmniStudio FlexCard ('Provider360_TerminationCard') using two DataRaptor Extract sources: one querying Network_Termination_Case__c filtered by Provider__c = :recordId and Status__c IN ('Initiated','In Review','Pending Approval'), and one querying Provider_Change_Notification__c filtered by Provider__c = :recordId and Notification_Status__c = 'Pending'. Use conditional rendering states for the empty/no-cases scenario. Add the FlexCard as a Lightning Web Component wrapper on the Provider__c Lightning Record Page via App Builder. Ensure DataRaptor queries respect Sharing Rules so only the assigned specialist's network records are visible.

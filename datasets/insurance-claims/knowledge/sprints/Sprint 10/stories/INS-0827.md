## Description

**Bug Report**

**Summary:** When a Network_Termination_Case__c is created with Termination_Type__c = 'Involuntary' (e.g., credentialing failure, sanctions), the Effective_Date__c field remains null. This causes downstream failures in the PCN generation flow (INS-0826) and the fee schedule deactivation logic.

**Steps to Reproduce:**
1. Navigate to a Provider__c record for an active in-network provider.
2. Manually create a new Network_Termination_Case__c record.
3. Set Termination_Type__c = 'Involuntary'.
4. Set Involuntary_Reason__c = 'Credentialing Failure'.
5. Save the record.
6. Observe the Effective_Date__c field — it is blank.

**Expected Behavior:** Effective_Date__c should be auto-populated with today's date + 30 days (per policy for involuntary terminations) when Termination_Type__c = 'Involuntary' and Effective_Date__c is null on save.

**Actual Behavior:** Effective_Date__c remains null. No validation error is thrown. Downstream Flow (PCN generation) throws a null pointer fault on the date arithmetic step.

**Environment:** Full Sandbox (ACME-FS01)
**Regression:** Not present in Sprint 9 build — introduced by the Record-Triggered Flow changes in INS-0824 deployment.

**Severity:** High — blocks PCN compliance workflow for involuntary termination path.

## Acceptance Criteria

As per Description.

Additional verification: After fix, deploy to ACME-FS01 and re-execute steps to reproduce. Confirm Network_Termination_Case__c.Effective_Date__c = TODAY + 30 days when Termination_Type__c = 'Involuntary'. Confirm PCN generation flow (INS-0826) completes without fault for this termination type.

## Solution

Root cause: The Record-Triggered Flow introduced in INS-0824 contains a decision element that routes 'Involuntary' termination type to a separate path which lacks the Effective_Date__c assignment element present in the 'Voluntary Exit' path. Fix: Add a formula-based Assignment element in the 'Involuntary' flow path setting Effective_Date__c = {!$Flow.CurrentDate} + 30 (using Add Days formula). Also add a before-save validation rule as a safety net: IF(ISPICKVAL(Termination_Type__c,'Involuntary') && ISBLANK(Effective_Date__c), TRUE, FALSE) with error message. Unit test update required in TerminationCaseFactoryTest to cover the involuntary path.

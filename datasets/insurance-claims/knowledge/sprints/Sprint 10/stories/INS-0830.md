## Description

**Bug Report**

**Summary:** When a Claim__c contains multiple Claim_Line__c records where some lines have Service_Date__c before the provider's termination effective date and at least one line has Service_Date__c on or after the effective date, the claim-level flag Claim__c.Provider_Termination_Flag__c is not being set to true. Only the individual Claim_Line__c.Pricing_Status__c = 'Fee Schedule Not Found' is set on the post-termination lines, but the parent claim does not reflect the mixed-date issue.

**Steps to Reproduce:**
1. Identify a Provider__c with an approved Network_Termination_Case__c and Effective_Date__c = 2025-06-01.
2. Create a Claim__c with Billing_Provider__c pointing to this provider.
3. Add Claim_Line__c Line 1: Service_Date__c = 2025-05-28 (pre-termination), CPT code 99213.
4. Add Claim_Line__c Line 2: Service_Date__c = 2025-06-03 (post-termination), CPT code 99214.
5. Submit the claim through the adjudication engine.
6. Observe Claim__c.Provider_Termination_Flag__c — it remains false/unchecked.
7. Observe Claim__c.Status__c — it remains 'In Adjudication' rather than rolling up to 'Pended - Provider Eligibility'.

**Expected Behavior:** Claim__c.Provider_Termination_Flag__c = true and Claim__c.Status__c = 'Pended - Provider Eligibility' whenever ANY child Claim_Line__c has Pricing_Status__c = 'Fee Schedule Not Found' due to provider termination.

**Actual Behavior:** Claim__c.Provider_Termination_Flag__c = false. Claim__c.Status__c = 'In Adjudication'. Post-termination line is correctly flagged at line level but roll-up to claim header is missing.

**Environment:** Full Sandbox (ACME-FS01) and Partial Sandbox (ACME-PS02) — reproducible in both.
**Impact:** Claims Operations team is manually reviewing each claim; estimated 40-60 affected claims per week in production based on current termination volume.

## Acceptance Criteria

As per Description.

Additional verification:
- Confirm Claim__c.Provider_Termination_Flag__c = true when ANY child Claim_Line__c.Pricing_Status__c = 'Fee Schedule Not Found' with Termination_Flag__c = true.
- Confirm Claim__c.Status__c = 'Pended - Provider Eligibility' in the straddling date scenario.
- Confirm that a Claim__c where ALL lines are pre-termination does NOT have Provider_Termination_Flag__c set to true.
- Regression test: confirm all-post-termination claim (covered by INS-0828 AC#2) still behaves correctly after fix.

## Solution

Root cause: The roll-up summary field on Claim__c counting child Claim_Line__c records where Pricing_Status__c = 'Fee Schedule Not Found' was not implemented — only individual line-level logic exists. Fix options: (1) Add a Roll-Up Summary field Claim__c.Terminated_Line_Count__c (COUNT of Claim_Line__c where Termination_Flag__c = true); then a Formula field or Flow sets Provider_Termination_Flag__c = Terminated_Line_Count__c > 0. (2) Alternatively, add an Apex Trigger on Claim_Line__c (after insert/update) that queries sibling lines and updates parent Claim__c. Recommended: option 1 (declarative) for maintainability. Also add a Record-Triggered Flow on Claim__c to set Status__c = 'Pended - Provider Eligibility' when Provider_Termination_Flag__c becomes true and Status__c != 'Closed'.

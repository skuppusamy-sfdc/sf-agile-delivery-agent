## Description

As a Provider Relations Specialist, I need a real-time FlexCard on the Provider__c (Person Account) record page that surfaces any open Network_Termination_Case__c records, associated PNA exit timelines, and outstanding Provider_Change_Notification__c statuses, so that I have a single view of a provider's network exit posture without navigating to multiple related lists. Extends INS-0751 (Provider 360 base FlexCard).

## Acceptance Criteria

Given a Provider__c record with one or more Network_Termination_Case__c records where Termination_Case.Status__c != 'Closed',
When the Provider 360 page loads,
Then the 'Network Exit Status' FlexCard panel is visible and displays Termination_Case.Reason__c, Termination_Case.Effective_Date__c, and a color-coded status badge (Red = 'Initiated', Amber = 'In Review', Green = 'Closed').

Given a provider has no open Network_Termination_Case__c records,
When the FlexCard panel renders,
Then the panel displays the message 'No Active Termination Cases' and the badge is suppressed.

Given a PCN record linked to the provider where PCN.Status__c = 'Pending Send',
When the FlexCard renders,
Then a 'Notifications Pending' warning icon appears with a count of unsent PCNs and a quick-action button 'View PCNs'.

Given the Provider Relations Specialist clicks 'View PCNs',
When the action fires,
Then a child FlexCard expands inline showing PCN.Notification_Type__c, PCN.Send_Date__c, and PCN.Delivery_Channel__c for each pending record.

-FlexCard must also display Credentialing_Case__c expiry dates- (moved to INS-0779 scope).

## Solution

Build an OmniStudio FlexCard 'ProviderNetworkExitCard' with a DataRaptor Extract source querying Network_Termination_Case__c and Provider_Change_Notification__c via a parent-child SOQL. Use conditional visibility rules on card states (No Data, Active, Warning) driven by Termination_Case.Status__c values. Implement a child FlexCard 'PCNDetailCard' triggered by the 'View PCNs' flyout action. Deploy via OmniStudio version control and assign to the Provider 360 Lightning App page via a dynamic component visibility rule scoped to the 'Provider_Relations' Permission Set Group.

## Description

**Bug Report**

**Summary:** Network_Termination_Case__c.Effective_Termination_Date__c remains null when Termination_Type__c is set to 'Involuntary', causing downstream claims eligibility checks to fail with a null reference error.

**Steps to Reproduce:**
1. Navigate to any active Provider_Network_Agreement__c record.
2. Click 'Initiate Termination' action button.
3. In the OmniScript modal, select Termination_Type__c = 'Involuntary' and populate Termination_Reason__c = 'Fraud Investigation'.
4. Set Involuntary_Termination_Date__c to any future date (e.g., 30 days from today).
5. Click 'Submit'.
6. Open the newly created Network_Termination_Case__c record.

**Expected Result:** NTC.Effective_Termination_Date__c should be populated with the value entered in Involuntary_Termination_Date__c from the OmniScript (Step 4).

**Actual Result:** NTC.Effective_Termination_Date__c = null. The Involuntary_Termination_Date__c field value is captured in the OmniScript JSON but is not mapped to the NTC record on creation.

**Impact:** Claims eligibility validation Apex (ClaimEligibilityValidator.java, line 214) throws a NullPointerException when evaluating Claim.Service_Date__c >= NTC.Effective_Termination_Date__c, causing Claim__c records to remain stuck in 'Pending Eligibility' status.

**Environment:** Partial Sandbox (PS2)
**Regression:** Not present in Sprint 9 build; introduced after INS-0864 deployment.

## Acceptance Criteria

As per Description. Verify NTC.Effective_Termination_Date__c is correctly populated for all Termination_Type__c values: 'Contract Expiration', 'Voluntary Exit', and 'Involuntary' in regression test.

## Solution

Root cause identified: The DataRaptor Transform step in the 'Initiate Termination' OmniScript is missing the field mapping for Involuntary_Termination_Date__c → Network_Termination_Case__c.Effective_Termination_Date__c for the 'Involuntary' branch. Fix: Add explicit mapping in DataRaptor Transform (DR_NTC_Create_Transform) for the conditional path where Termination_Type__c = 'Involuntary'. Also add a validation rule on Network_Termination_Case__c: ISBLANK(Effective_Termination_Date__c) as a safety net to prevent null submissions regardless of entry path. Deploy fix to PS2 for regression testing before promoting to Full Sandbox.

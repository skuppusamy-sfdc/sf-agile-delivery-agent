## Description

As a Provider Network Specialist, I need a Network_Termination_Case__c record to be automatically created when a Provider_Network_Agreement__c reaches its termination date or when a voluntary exit is flagged, so that the termination workflow is triggered without manual intervention and compliance timelines are met.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Status__c transitions to 'Pending Termination' or PNA.Termination_Type__c is set to 'Voluntary Exit'
When the Record-Triggered Flow fires on save
Then a Network_Termination_Case__c is created with:
  - NTC.Provider__c = PNA.Provider__c
  - NTC.Effective_Termination_Date__c = PNA.End_Date__c
  - NTC.Termination_Type__c = PNA.Termination_Type__c
  - NTC.Status__c = 'Open'
  - NTC.LOB__c = PNA.Line_of_Business__c
  - NTC.Origin__c = 'Automated'

Given the Network_Termination_Case__c is created
When NTC.Status__c = 'Open'
Then a task is auto-assigned to the owning Provider Relations Specialist with a due date of NTC.Effective_Termination_Date__c minus 30 days

Given a PNA where PNA.Termination_Type__c = 'Involuntary'
When the flow fires
Then NTC.Priority__c = 'High' and NTC.Compliance_Review_Required__c = TRUE

-Given PNA.Auto_Renew__c = TRUE, suppress termination case creation- (deprecated per INS-0831 — auto-renew logic moved to separate flow)

Given a duplicate termination case already exists for the same Provider__c and Effective_Termination_Date__c
When the flow attempts to create a new NTC
Then creation is suppressed and a warning email is sent to the assigned Provider Network Specialist

## Solution

Implement a Record-Triggered Flow on Provider_Network_Agreement__c firing on Create and Update when Status__c changes to 'Pending Termination' or Termination_Type__c is populated. Flow uses a Get Records element to check for duplicate Network_Termination_Case__c records before creation. Decision element branches on Termination_Type__c to set Priority__c. An Apex invocable action handles duplicate-check email notification via Messaging.sendEmail. Platform Event (Provider_Termination_Event__e) published post-creation to notify downstream systems. Flow is bulkified and wrapped in a try/catch Apex handler for DML exception logging to a custom Error_Log__c object.

## Description

**Bug Report**

**Environment:** Acme Insurance Full Sandbox (Sprint 10)
**Severity:** Medium — impacts Credentialing Specialist audit workflow; no data loss

**Steps to Reproduce:**
1. Navigate to any Provider__c (Person Account) record that has both a Credentialing_Case__c in Status__c = 'Active' and a Provider_Network_Agreement__c in Status__c = 'Pending Termination'.
2. Open the Credentialing_Case__c related list on the Provider__c record page.
3. Attempt to filter the related list by the associated PNA status using the 'Network Agreement Status' filter column.
4. Observe available filter values in the dropdown.

**Expected Behavior:**
The filter dropdown for 'Network Agreement Status' on the Credentialing_Case__c related list should include 'Pending Termination' as a selectable value, allowing Credentialing Specialists to isolate credential records tied to providers in the termination pipeline.

**Actual Behavior:**
'Pending Termination' does not appear in the filter dropdown. Only 'Active', 'Expired', and 'Suspended' are available. This prevents Credentialing Specialists from identifying which open credentialing cases belong to providers currently undergoing network exit, causing them to manually cross-reference the Network_Termination_Case__c object.

**Root Cause Hypothesis:**
The 'Pending Termination' picklist value was added to PNA.Status__c in Sprint 9 (INS-0821) but the corresponding filter field set on the Credentialing_Case__c related list component and the List View filter criteria were not updated to include the new value.

**Impact:**
- Credentialing Specialists unable to efficiently audit ~40 open credentialing cases linked to Sprint 10 termination batch
- Compliance reporting dashboard (CR-009) showing incomplete termination-credentialing intersection data

## Acceptance Criteria

As per Description. Fix validated when 'Pending Termination' appears in the Network Agreement Status filter dropdown on the Credentialing_Case__c related list on Provider__c record page in UAT. Compliance reporting dashboard CR-009 must also reflect accurate count of Credentialing_Case__c records linked to PNAs with Status__c = 'Pending Termination' after fix is deployed.

## Solution

Add 'Pending Termination' to the Field Set 'Credentialing_Case_Related_List_Filters' on Credentialing_Case__c via Setup > Object Manager. Update the related List View filter criteria on the Credentialing_Case__c related list component in the Provider__c Lightning record page to include the new picklist value. Verify the PNA.Status__c picklist value 'Pending Termination' is included in all relevant List Views, Report Types, and the ComplianceReporting dashboard filter panels (CR-009 report uses a custom Report Type 'Credentialing Cases with PNA' — update filter criteria there as well). Deploy metadata via change set including: FieldSet, ListViewFilter, and ReportType components. No Apex changes required.

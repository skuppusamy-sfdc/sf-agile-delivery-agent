## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces the provider's active Provider_Network_Agreement__c details, current credentialing status from Credentialing_Case__c, and any open Network_Termination_Case__c records, so that I have a single-pane view when managing provider exit workflows without navigating across multiple objects.

## Acceptance Criteria

Given a Provider__c record is opened in Lightning Experience,
When the Provider 360 FlexCard loads,
Then it displays PNA.Network_Name__c, PNA.Status__c, PNA.End_Date__c, and PNA.LOB__c for all active agreements in a data table with row-level actions.

Given a provider has one or more Network_Termination_Case__c records,
When the FlexCard renders the Termination section,
Then it shows NTC.Status__c, NTC.Termination_Type__c, NTC.Effective_Termination_Date__c, and a badge indicator: red for 'Initiated', amber for 'In Progress', green for 'Closed'.

Given a provider's Credentialing_Case__c.Status__c = 'Suspended' or 'Revoked',
When the FlexCard header renders,
Then a prominent warning banner displays: 'Credentialing Alert – Action Required' with a link to the open Credentialing_Case__c.

Given the FlexCard is viewed by a user with the 'Provider_Network_ReadOnly' Permission Set Group,
When row-level actions are evaluated,
Then action buttons 'Initiate Termination' and 'Amend PNA' are hidden; only 'View Details' is accessible.

Given the provider has no active PNA records,
When the FlexCard PNA section loads,
Then an empty state message 'No Active Network Agreements Found' is displayed with a CTA button 'Create New PNA' visible only to users with 'Provider_Network_Admin' Permission Set Group.

## Solution

Build using OmniStudio FlexCards with child FlexCards nested for PNA, Termination, and Credentialing sections. DataRaptors (Extract type) fetch related Provider_Network_Agreement__c, Network_Termination_Case__c, and Credentialing_Case__c records filtered by Provider__c lookup. Badge color logic implemented via FlexCard conditional styling using State/Action mappings on NTC.Status__c. Permission-based action visibility controlled through FlexCard Action conditions referencing $User.PermissionSetGroupAssignments. Deployed to the Provider__c Lightning Record Page via a custom Tab component.

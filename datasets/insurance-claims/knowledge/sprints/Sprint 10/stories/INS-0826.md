## Description

As a Compliance Officer, I need the system to automatically generate Provider_Change_Notification__c (PCN) records for all affected policyholders when a provider's Network_Termination_Case__c is approved, so that we meet the state-mandated 90-day advance written notification requirement and avoid regulatory penalties.

Affected policyholders are those whose active Policy__c records reference the terminating provider as their PCP or a specialist under a GSA.

## Acceptance Criteria

Given a Network_Termination_Case__c where Status__c transitions to 'Approved'
When the Record-Triggered Flow fires
Then the system queries all Policy__c records where Policy__c.PCP_Provider__c = terminating Provider__c.Id AND Policy__c.Status__c = 'Active'
And a Provider_Change_Notification__c is created for each affected policy with PCN.Notification_Type__c = 'Provider Termination', PCN.Required_Send_Date__c = Network_Termination_Case__c.Effective_Date__c - 90 days, PCN.Status__c = 'Pending'

Given PCN records are created
When PCN.Required_Send_Date__c is within 5 days of today
Then a high-priority Task is created for the assigned Compliance Officer and the PCN.Status__c auto-updates to 'Send Imminent'

Given a PCN record with Status__c = 'Pending'
When the Conga CLM integration generates the notification letter
Then PCN.Document_Generated__c = true, PCN.Conga_Document_Id__c is populated, and PCN.Status__c = 'Document Ready'

Given the total affected policyholders exceeds 500
When PCN batch creation runs
Then processing is handled via Apex Batch (batch size = 200) and a summary Integration_Log__c record is written with total PCNs created, LOB breakdown, and any failed records

Given a Policy__c is covered under a Group_Service_Agreement__c (GSA)
When PCN generation runs
Then the Broker Relations Coordinator linked via GSA.Broker__c also receives a separate PCN with Notification_Type__c = 'Group Plan Provider Exit Alert'

## Solution

Record-Triggered Flow on Network_Termination_Case__c (AFTER UPDATE, fires on Status__c change to 'Approved') calls Apex invocable method 'PCNGenerationService.generateForTermination()'. Apex queries affected Policy__c records using SOQL with indexed fields. For counts > 500, enqueues Apex Batch job 'PCNBatchGenerator' (implements Database.Batchable). Conga CLM called via Named Credential REST callout in a separate Queueable Apex class to avoid governor limits in the batch context. Custom metadata 'PCN_Config__mdt' stores the 90-day offset and batch size thresholds.

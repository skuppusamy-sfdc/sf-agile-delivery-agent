## Description

Steps to Reproduce:
1. Identify a Claim__c record with Status__c = 'Pended - Provider Termination' created during Sprint 9 testing (Date_of_Service__c between 2024-09-01 and 2024-09-30).
2. Manually override Claim__c.Status__c to 'In Adjudication' to force re-adjudication.
3. Observe Claim_Line__c.Allowed_Amount__c recalculation via the Fee Schedule Engine.
4. Compare Claim_Line__c.Allowed_Amount__c against the expected value from the active Fee_Schedule__c for the provider's CPT code and LOB.

Expected Behavior: Claim_Line__c.Allowed_Amount__c should be calculated using the Fee_Schedule__c record active as of Date_of_Service__c, referencing CPT_Code__c and the provider's contracted rate.

Actual Behavior: Claim_Line__c.Allowed_Amount__c is populated with $0.00 for all affected claim lines. Fee_Schedule__c lookup returns null because the FeeScheduleEngine is incorrectly filtering on Fee_Schedule__c.End_Date__c < TODAY() instead of Fee_Schedule__c.End_Date__c < Date_of_Service__c.

Environment: Acme Insurance Full Sandbox (FS-01)
Regression introduced: Sprint 9 Fee Schedule end-dating change (INS-0831)
Impact: ~340 pended claim lines show $0.00 allowed amount, blocking payment processing for valid pre-termination dates of service. Financial exposure estimated at $127,000.

## Solution

Root cause: FeeScheduleEngineService.cls method 'getApplicableFeeSchedule()' uses a hardcoded Datetime.now() comparison instead of passing Claim__c.Date_of_Service__c as the effective date parameter. This was introduced in the INS-0831 refactor. Fix: Update SOQL filter in getApplicableFeeSchedule() from 'End_Date__c >= :Date.today()' to 'End_Date__c >= :dateOfService' where dateOfService is passed as a method parameter. Update all 3 callers of this method to pass Claim__c.Date_of_Service__c. Write a data fix script (Anonymous Apex) to re-trigger adjudication for the ~340 affected Claim_Line__c records by setting a Re_Adjudicate__c checkbox field to true, which fires an existing Record-Triggered Flow. Validate fix in FS-01 against 50 sample claims before production deployment. Coordinate with Claims Operations Manager on production run scheduling.

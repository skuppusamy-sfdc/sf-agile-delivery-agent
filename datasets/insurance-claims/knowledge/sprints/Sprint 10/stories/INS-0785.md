## Description

As a Provider Relations Specialist, I need a consolidated Provider Termination 360 FlexCard on the Network_Termination_Case__c record page so that I can view all in-flight Claim__c records, pending Payment__c disbursements, open Credentialing_Case__c items, and the network exit timeline in a single surface without navigating to multiple objects.

## Acceptance Criteria

Given a Network_Termination_Case__c record page is open,
When the FlexCard loads,
Then it displays: (1) provider NPI and TIN from the linked Provider__c, (2) count and list of Claim__c records where Claim__c.Provider__c = Provider__c.Id AND Claim__c.Status__c NOT IN ('Paid','Denied','Void'), (3) sum of Payment__c.Amount__c where Payment__c.Status__c = 'Pending' linked to the provider.

Given the FlexCard is rendered,
When there are Claim_Line__c records with CPT codes flagged for post-termination date of service,
Then those lines are highlighted in amber with a tooltip: 'DOS after termination effective date — review required'.

Given the network exit timeline section,
When the FlexCard loads,
Then it renders a visual stepper showing: Termination Initiated → Provider Notified → 90-Day Tail Period → Final Termination with dates populated from Network_Termination_Case__c milestone fields.

Given a user with Profile = 'Claims Operations Manager',
When they view the FlexCard,
Then the Payment__c disbursement panel is visible with an inline 'Place Hold' action button that sets Payment__c.Hold_Flag__c = TRUE.

Given a user without the 'View_Financials' Permission Set,
When the FlexCard loads,
Then the Payment__c panel is hidden and replaced with 'Insufficient privileges to view payment data'.

## Solution

Build an OmniStudio FlexCard 'ProviderTermination360' anchored to Network_Termination_Case__c. Use DataRaptor Extract to fetch related Claim__c, Payment__c, and Credentialing_Case__c data via SOQL with relationship traversal. Implement conditional visibility blocks using FlexCard state properties tied to User.PermissionSetAssignments. The timeline stepper is a custom child FlexCard component using milestone date fields: Initiation_Date__c, Notification_Sent_Date__c, Tail_Period_End_Date__c, Final_Termination_Date__c on Network_Termination_Case__c. The 'Place Hold' action invokes an Integration Procedure that calls a DataRaptor Transform and updates Payment__c via a DML action node. Deploy via OmniStudio package version-controlled in the feature branch.

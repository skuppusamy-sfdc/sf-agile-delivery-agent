## Description

Steps to Reproduce:
1. Navigate to any Network_Termination_Case__c record with Status__c = 'Provider Notified'.
2. Edit the record inline and change Status__c to 'Closed' without populating Termination_Effective_Date__c.
3. Click Save.

Expected Behavior: A validation error is thrown: 'Termination Effective Date is required before closing a termination case.' The record save is blocked.

Actual Behavior: The record saves successfully with Status__c = 'Closed' and Termination_Effective_Date__c = NULL. This causes downstream failures in the tail period calculation logic (INS-0786) which references this field, resulting in NullPointerException in ClaimsAdjudicationService.evaluateTailPeriod().

Environment: Acme Insurance FULL sandbox. Reproducible 100% of the time via inline edit; the Validation Rule 'VR_TerminationDateRequired' appears to only fire on the standard edit page, not inline edit or API updates. Bulk API loads from the provider data migration also bypass this rule.

## Acceptance Criteria

As per Description. Additionally: Given a Network_Termination_Case__c is updated to Status__c = 'Closed' via any channel (UI inline edit, API, Flow, Bulk API), When Termination_Effective_Date__c is NULL, Then a validation error is returned and the record is not saved.

## Solution

The existing Validation Rule 'VR_TerminationDateRequired' uses a formula that evaluates incorrectly when triggered via inline edit due to a known Salesforce behavior with prior value functions on picklist fields. Fix: rewrite the validation rule formula to use ISCHANGED(Status__c) && ISPICKVAL(Status__c, 'Closed') && ISBLANK(Termination_Effective_Date__c) removing any dependency on PRIORVALUE() for picklist fields. Additionally, add a before-save Record-Triggered Flow as a secondary enforcement layer that throws a custom error on the Termination_Effective_Date__c field for API and bulk contexts. Update the ClaimsAdjudicationService.evaluateTailPeriod() method with a null-check guard to prevent NPE as a defensive measure. Regression test across UI, REST API, and Bulk API v2 in the FULL sandbox before promoting to UAT.

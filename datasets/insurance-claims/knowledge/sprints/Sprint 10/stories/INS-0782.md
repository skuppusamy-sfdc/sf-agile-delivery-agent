## Description

As a Provider Network Specialist, I need an automated impact report that identifies all Group_Service_Agreement__c records whose covered provider panels include a terminating provider, so that Broker Relations Coordinators and employer group clients can be proactively notified of network adequacy changes before the termination effective date. This supports compliance with state network adequacy regulations requiring 30-day advance notice to group policyholders.

## Acceptance Criteria

Given a Network_Termination_Case__c is created and Termination_Case.Status__c = 'Initiated',
When the termination case is saved,
Then a Salesforce Report (type: Summary) named 'GSA_Termination_Impact_[CaseNumber]' is generated via Apex and attached to the Network_Termination_Case__c as a ContentDocumentLink.

Given the impact report is generated,
When the report runs,
Then it lists all GSA__c records where GSA.Provider_Panel__c includes the terminating Provider__c NPI, grouped by GSA.Employer_Group__c, showing GSA.Contract_Number__c, GSA.Effective_Date__c, GSA.Renewal_Date__c, and GSA.Broker_of_Record__c.

Given the impact report identifies 1 or more affected GSAs,
When the report is attached,
Then a Task is created for each unique GSA.Broker_of_Record__c with Subject = 'Provider Termination Network Impact – Action Required', Due_Date__c = Termination_Case.Effective_Date__c - 30 days, and Priority = 'High'.

Given a GSA where GSA.LOB__c = 'Fully Insured' is identified in the impact report,
When the task is created,
Then an additional field Task.Compliance_Flag__c = 'State Notice Required' is set and the task is also assigned to the Compliance Officer queue.

Given the terminating provider serves as the sole in-network specialist for a given specialty in a GSA's service area,
When the report evaluates network adequacy,
Then GSA.Network_Adequacy_Flag__c is set to 'At Risk' and flagged in the report with a 'Critical' severity indicator.

## Solution

Build an Apex invocable method 'GSAImpactReportGenerator' triggered from the Network_Termination_Case__c Record-Triggered Flow (after insert, async path). The method queries Group_Service_Agreement__c via a junction object GSA_Provider__c (linking GSA to Provider__c) filtered by the terminating provider's Id. Use Salesforce Reports & Dashboards API (Analytics REST API) to clone a master report template and scope it to the specific termination case via a report filter. Attach the report snapshot as a ContentVersion/ContentDocumentLink. Use a Custom Metadata type 'Network_Adequacy_Rule__mdt' to define specialty-level adequacy thresholds per service area. Task creation uses a bulk-safe loop with Database.insert(). Deploy sharing rules to ensure Broker Relations Coordinators have read access to Network_Termination_Case__c via a Criteria-Based Sharing Rule.

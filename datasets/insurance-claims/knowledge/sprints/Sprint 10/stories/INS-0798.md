## Description

As a Provider Relations Specialist, I need a guided OmniScript intake flow to manually initiate a provider network exit when the termination is voluntary (provider-initiated) or driven by a credentialing failure, so that all required data is captured consistently, the correct Network_Termination_Case__c and PCN records are created with proper LOB mapping, and the Compliance Officer is notified of high-risk terminations (e.g., for-cause exits) without relying on ad-hoc record creation that results in data quality issues.

## Acceptance Criteria

Given the Provider Relations Specialist launches the 'Provider Network Exit' OmniScript from the Provider Account action menu,
When Step 1 (Provider Identification) is completed,
Then the OmniScript pre-populates Provider__c.Name, Provider__c.NPI__c, and Provider__c.TIN__c via a DataRaptor Extract.

Given Step 2 (Termination Details) is presented,
When the specialist selects Termination_Type__c from the picklist ('Voluntary', 'For Cause', 'Credentialing Failure', 'Contract Non-Renewal'),
Then the subsequent steps are conditionally rendered based on the selected type (e.g., 'For Cause' reveals an additional mandatory 'Cause Description' text area).

Given Step 3 (LOB and PNA Selection) is presented,
When the specialist selects the applicable PNA from a DataRaptor-powered datatable showing only PNAs where PNA.Status__c = 'Active' for the selected provider,
Then PNA.Network_Name__c, PNA.Effective_Date__c, and PNA.LOB__c are auto-populated into the OmniScript context.

Given Step 4 (Notice Period Confirmation) is presented,
When the LOB is confirmed,
Then the OmniScript displays the calculated NTC.Termination_Effective_Date__c based on LOB_PCN_Template_Mapping__mdt.Notice_Period_Days__c added to today's date, and allows the specialist to accept or override the date.

Given the specialist completes all steps and clicks 'Submit',
When the OmniScript Integration Procedure fires,
Then the following records are created:
  - Network_Termination_Case__c with all captured fields
  - Provider_Change_Notification__c linked to the NTC
  - A Task assigned to the Compliance Officer if Termination_Type__c = 'For Cause'

Given the OmniScript submission is successful,
When the confirmation screen is shown,
Then the NTC record ID and a link to the newly created record are displayed to the specialist.

Given a specialist partially completes the OmniScript and navigates away,
When they return to the Provider Account and relaunch the OmniScript,
Then a draft state is restored if the session is within the same browser session (no persistent draft storage required).

## Solution

Build OmniScript 'ProviderNetworkExitOS' in OmniStudio with 5 steps using conditional rendering via OmniScript Conditional Views. DataRaptor Extracts used for provider pre-population and PNA datatable. Custom metadata LOB_PCN_Template_Mapping__mdt drives notice period calculation via a Formula element in the OmniScript. Integration Procedure 'CreateProviderExitRecords_IP' orchestrates record creation: (1) DataRaptor Transform to map OmniScript context to NTC fields, (2) DataRaptor Load to create NTC, (3) DataRaptor Load to create PCN, (4) Apex Action for conditional Task creation for For Cause scenarios. OmniScript deployed as a Quick Action on the Provider Account page layout.

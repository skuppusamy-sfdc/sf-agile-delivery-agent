## Description

As a Provider Network Specialist, I need the system to evaluate PNA auto-renewal eligibility before triggering any termination workflow, so that providers with Auto_Renew__c = true and no disqualifying credentialing flags do not receive erroneous termination notices. This story implements the auto-renew suppression logic deferred from INS-0856.

## Acceptance Criteria

Given a Provider_Network_Agreement__c where Auto_Renew__c = true AND Contract_End_Date__c = TODAY + 60
When the ProviderTerminationBatchJob (INS-0856) evaluates the record
Then no Network_Termination_Case__c is created and PNA.Renewal_Evaluation_Status__c is set to 'Auto-Renewal Eligible'

Given PNA.Auto_Renew__c = true but the linked Provider__c has an open Credentialing_Case__c with Credentialing_Status__c IN ('Suspended','Revoked','Lapsed')
When the batch job evaluates the PNA
Then auto-renewal is disqualified: a Network_Termination_Case__c IS created with Termination_Reason__c = 'Credentialing Disqualification', and PNA.Renewal_Evaluation_Status__c = 'Auto-Renewal Blocked - Credentialing'

Given PNA.Auto_Renew__c = true and renewal is eligible
When the system extends the contract
Then PNA.Contract_End_Date__c is updated to current Contract_End_Date__c + PNA.Renewal_Term_Months__c (in months), PNA.Last_Renewal_Date__c = TODAY(), and PNA.Renewal_Count__c is incremented by 1

Given a PNA that has been auto-renewed
When the renewal is applied
Then all linked Fee_Schedule__c records with End_Date__c = old Contract_End_Date__c are updated: End_Date__c = new Contract_End_Date__c and Fee_Schedule__c.Status__c remains 'Active'

Given PNA.Max_Renewal_Count__c is defined and PNA.Renewal_Count__c >= PNA.Max_Renewal_Count__c
When the batch job evaluates the PNA
Then auto-renewal is suppressed, Renewal_Evaluation_Status__c = 'Max Renewals Reached', and termination workflow proceeds as standard

## Solution

Extend the ProviderTerminationBatchJob (INS-0856) with a pre-evaluation method 'evaluateAutoRenewal()' called before termination case creation logic. Method queries Credentialing_Case__c for disqualifying statuses using a SOQL subquery on Provider__c. Renewal extension logic implemented in a separate Invocable Apex method 'ExtendPNAContract' callable from both the batch and a standalone Flow. Fee_Schedule__c bulk update handled via Database.update() with allOrNone=false and error logging to Batch_Execution_Log__c. Custom Metadata Type 'AutoRenewalDisqualifiers__mdt' stores the list of disqualifying Credentialing_Status__c values for admin-configurable maintenance. Unit tests target 95%+ coverage with mock Credentialing_Case__c data.

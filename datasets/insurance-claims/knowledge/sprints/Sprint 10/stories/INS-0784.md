## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record when a Provider_Network_Agreement__c reaches its expiration date or when a voluntary exit flag is set, so that the termination workflow is initiated without manual intervention and no providers are inadvertently left active in the network past their contract end date.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Status__c = 'Active' and PNA.End_Date__c equals today's date,
When the nightly scheduled Apex job runs,
Then a Network_Termination_Case__c is created with Termination_Reason__c = 'Contract Expiration', linked via PNA_Lookup__c, and Status__c = 'Initiated'.

Given a Provider_Network_Agreement__c where PNA.Voluntary_Exit_Requested__c is set to TRUE by a Provider Relations Specialist,
When the record is saved,
Then a Network_Termination_Case__c is immediately created via Record-Triggered Flow with Termination_Reason__c = 'Voluntary Exit' and Effective_Termination_Date__c = PNA.Requested_Exit_Date__c.

Given a Network_Termination_Case__c is successfully created,
When the record is inserted,
Then a Platform Event (Provider_Termination_Event__e) is published with fields: Provider_NPI__c, TIN__c, Effective_Date__c, and Termination_Reason__c for downstream consumption.

Given the termination case is created,
When Status__c = 'Initiated',
Then the associated Provider__c (Person Account) record field Network_Status__c is updated to 'Termination Pending' and a task is auto-assigned to the owning Provider Relations Specialist.

-Given PNA.LOB__c = 'Medicare Advantage', an additional CMS notification record should be auto-created — deprecated in favor of manual CMS outreach per Compliance Officer guidance-

Given a PNA already has a linked open Network_Termination_Case__c,
When a duplicate termination trigger fires,
Then no second Network_Termination_Case__c is created and a warning toast is displayed to the initiating user.

## Solution

Implement a Record-Triggered Flow on Provider_Network_Agreement__c firing on create/update when End_Date__c = TODAY() or Voluntary_Exit_Requested__c = TRUE. Flow performs a duplicate check via Get Records on Network_Termination_Case__c filtering by PNA_Lookup__c and Status__c NOT IN ('Closed','Withdrawn'). If no duplicate, flow creates Network_Termination_Case__c and publishes a Provider_Termination_Event__e Platform Event. A separate Scheduled Apex job (ProviderPNAExpirationBatch) runs nightly at 01:00 UTC to catch any records missed by real-time flow (e.g., bulk data loads). An Apex Trigger (NetworkTerminationCaseTrigger) on after-insert updates Provider__c.Network_Status__c via a queueable job to avoid DML governor limits. Permission Set Group 'Provider_Network_Ops' grants FLS to all new fields.

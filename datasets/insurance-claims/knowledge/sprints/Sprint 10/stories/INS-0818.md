## Description

As a Provider Network Specialist, I need a guided OmniScript checklist for processing a provider's network exit so that all required termination steps — including notifying downstream systems, suspending fee schedules, and generating the PCN letter — are completed in the correct sequence and documented on the Network_Termination_Case__c record.

## Acceptance Criteria

Given a Network_Termination_Case__c with NTC.Status__c = 'Initiated'
When the specialist launches the Network Exit Checklist OmniScript
Then Step 1 displays provider details (NPI, TIN, Provider__c.Name, PNA.Line_of_Business__c) pre-populated via DataRaptor Extract — read-only

Given Step 2 (Fee Schedule Suspension)
When the specialist confirms suspension
Then all Fee_Schedule__c records linked to the provider where FS.Status__c = 'Active' and FS.PNA__c = NTC.PNA__c are updated to FS.Status__c = 'Suspended' via DataRaptor Turbo Action

Given Step 3 (Open Claims Review)
When the OmniScript queries Claim__c records
Then it surfaces all claims where Claim__c.Provider__c = NTC.Provider__c AND Claim__c.Status__c IN ('Submitted','In Adjudication') with a count and list displayed to the specialist for manual review acknowledgment

Given Step 4 (PCN Letter Generation)
When the specialist clicks 'Generate PCN Letter'
Then a Provider_Change_Notification__c record is created with PCN.Type__c = 'Termination Notice' and a Conga CLM merge is triggered to generate the PDF, attached to the NTC record

Given Step 5 (Final Confirmation)
When the specialist submits the checklist
Then NTC.Status__c is updated to 'Pending Termination', NTC.Checklist_Completed_Date__c = TODAY, NTC.Completed_By__c = running user, and a Platform Event 'Provider_Termination_Event__e' is published with payload including Provider NPI and Effective Date

Given any step fails due to a DataRaptor or integration error
When the error occurs
Then the OmniScript displays an inline error message and logs to Integration_Error_Log__c without advancing the step

## Solution

Build a multi-step OmniScript 'Network_Exit_Checklist_OS' in OmniStudio. Step 1: DataRaptor Extract for provider/PNA data. Step 2: DataRaptor Turbo Action for Fee_Schedule__c bulk update. Step 3: DataRaptor Extract for open Claim__c records with acknowledgment checkbox. Step 4: Integration Procedure calling Conga CLM REST API for document merge and a DataRaptor Load to create PCN record. Step 5: DataRaptor Load to update NTC, followed by Apex Action to publish Provider_Termination_Event__e Platform Event. Implement error handling on each step with SetErrors. Expose OmniScript as a Lightning Web Component embedded in the FlexCard action (INS-0817).

## Description

As a Provider Relations Specialist, I need a guided OmniScript on the Provider Portal (Experience Cloud) that allows providers to submit a voluntary network exit request, select their requested exit date, provide a reason, and acknowledge the 90-day tail period terms, so that exit requests are captured in a structured, auditable manner directly into Salesforce without requiring manual intake via phone or email.

## Acceptance Criteria

Given an authenticated provider user on the Experience Cloud portal,
When they navigate to 'Request Network Exit',
Then the OmniScript 'VoluntaryProviderExit_OS' launches and displays their current active Provider_Network_Agreement__c records fetched via DataRaptor Extract.

Given the OmniScript Step 1 (Agreement Selection),
When the provider selects a PNA,
Then Step 2 displays the PNA.LOB__c, PNA.Network_Name__c, PNA.End_Date__c, and a date picker for Requested_Exit_Date__c constrained to minimum TODAY() + 30 days.

Given Step 3 (Exit Reason),
When the provider selects Reason__c from a picklist ('Retiring', 'Relocating', 'Business Closure', 'Contract Dispute', 'Other'),
Then if 'Other' is selected, a free-text field Exit_Reason_Detail__c (max 500 chars) is revealed.

Given Step 4 (Tail Period Acknowledgment),
When the provider checks the acknowledgment checkbox,
Then the 'Submit' button is enabled; if unchecked, submission is blocked with inline message 'You must acknowledge the 90-day tail period terms to proceed'.

Given the provider submits the OmniScript,
When the Integration Procedure executes,
Then PNA.Voluntary_Exit_Requested__c = TRUE, PNA.Requested_Exit_Date__c = selected date, PNA.Exit_Reason__c = selected reason, and a Network_Termination_Case__c is created per the logic in INS-0784 with Source__c = 'Provider Self-Service Portal'.

Given successful submission,
When the confirmation screen renders,
Then a confirmation number (Network_Termination_Case__c.CaseNumber) is displayed and a confirmation email is sent to the provider's verified email address.

## Solution

Build OmniScript 'VoluntaryProviderExit_OS' with 5 steps using OmniStudio Designer. DataRaptor Extract retrieves active PNA records for the logged-in community user via User.ContactId → Provider__c relationship. Step navigation controlled by conditional visibility on OmniScript elements. Integration Procedure 'VoluntaryExit_IP' orchestrates: (1) DataRaptor Transform to map OmniScript inputs to PNA fields, (2) DML Update on Provider_Network_Agreement__c, (3) invokes the Record-Triggered Flow indirectly by setting Voluntary_Exit_Requested__c = TRUE which triggers INS-0784 flow. Confirmation email sent via EmailTemplate 'Provider_Exit_Confirmation' using a Send Email action in the Integration Procedure. Experience Cloud sharing rules ensure providers can only read/edit their own PNA records via a Provider_Community_User Sharing Set. Guest user profile explicitly denied access to Network_Termination_Case__c.

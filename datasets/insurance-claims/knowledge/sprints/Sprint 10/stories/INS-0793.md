## Description

As a Provider Relations Specialist, I need a consolidated Provider Termination 360 FlexCard on the Provider__c (Person Account) record page that displays the active Network_Termination_Case__c, all associated Provider_Network_Agreement__c records in a terminal state, and any open Claim__c records linked to the provider that may be impacted by the network exit, so that I can assess financial exposure and coordinate care transitions before the termination effective date. Extends INS-0792.

## Acceptance Criteria

Given a Provider Account record with at least one open Network_Termination_Case__c,
When the Provider Relations Specialist views the record page,
Then the FlexCard renders a 'Termination Summary' section showing:
  - NTC.Status__c
  - NTC.Termination_Effective_Date__c
  - NTC.LOB__c
  - Count of linked open Claim__c records where Claim__c.Provider__c = Provider.Id AND Claim__c.Status__c IN ('Submitted','In Adjudication')

Given the FlexCard is rendered,
When the open claim count is greater than 0,
Then a warning badge displays in amber with text 'Open Claims at Risk: {count}'.

Given the FlexCard is rendered,
When the open claim count equals 0,
Then the badge displays in green with text 'No Open Claims Impacted'.

Given a Provider with multiple PNA records in 'Terminated' or 'Pending Termination' status,
When the FlexCard PNA datatable section is expanded,
Then all matching PNAs are listed with columns: PNA.Network_Name__c, PNA.Effective_Date__c, PNA.Termination_Date__c, PNA.Status__c.

Given the FlexCard is loaded,
When the DataRaptor fetch executes,
Then the response time is under 2 seconds for providers with fewer than 500 linked claims.

-Given a Provider with Credentialing_Case__c in 'Active' status, display credentialing badge on FlexCard- (deprecated: credentialing badge moved to separate Credentialing FlexCard per INS-0787)

## Solution

Build OmniStudio FlexCard 'ProviderTermination360FC' using two DataRaptor Extract sources: (1) DR_GetProviderNTCDetails fetching Network_Termination_Case__c and PNA records via SOQL join, (2) DR_GetOpenClaimsForProvider fetching Claim__c count. Conditional styling applied via FlexCard State Actions to toggle badge color based on claim count threshold. FlexCard deployed to Provider Account Lightning Record Page via App Builder. DataRaptor queries optimized with indexed fields (Provider__c, Status__c). LWC wrapper component used for custom badge rendering where OmniStudio native styling is insufficient.

## Description

As a Credentialing Specialist, I need any open Credentialing_Case__c records linked to a provider to be automatically suspended when a Network_Termination_Case__c is created for that provider, so that credentialing resources are not wasted on providers exiting the network and the credentialing pipeline accurately reflects actionable cases only.

## Acceptance Criteria

Given a Network_Termination_Case__c is created for a Provider__c (as triggered by INS-0808)
When the NTC insert event fires
Then all Credentialing_Case__c records where CC.Provider__c = NTC.Provider__c and CC.Status__c IN ('In Progress', 'Pending CAQH', 'Pending NPPES Verification', 'Awaiting Approval') are updated to CC.Status__c = 'Suspended - Provider Termination'

Given a Credentialing_Case__c is suspended
When the update completes
Then CC.Suspension_Reason__c = 'Network Termination Initiated' and CC.Suspension_Date__c = TODAY() are stamped, and a Task is created for the Credentialing Specialist with Subject = 'Review suspended credentialing case due to provider termination'

Given a Credentialing_Case__c with CC.Status__c = 'Approved' or 'Closed'
When the termination NTC is created for the same provider
Then those closed/approved cases are NOT updated (suspension applies to open/in-progress cases only)

Given a Network_Termination_Case__c is subsequently withdrawn (NTC.Status__c = 'Withdrawn')
When the withdrawal is saved
Then all previously suspended Credentialing_Case__c records for that provider are restored to CC.Status__c = 'In Progress' and CC.Suspension_Reason__c is cleared

Given more than 10 open Credentialing_Case__c records exist for a single provider
When the bulk suspension runs
Then all records are updated within the same transaction without hitting governor limits (verified by test with 200-record dataset)

## Solution

Implement an Apex Trigger (CredentialingCaseSuspensionTrigger.cls) on Network_Termination_Case__c (after insert, after update). Trigger handler class (CredentialingCaseSuspensionHandler.cls) performs a bulkified SOQL query for open Credentialing_Case__c records by Provider__c Id set. DML update uses a collected list to stay within governor limits. Withdrawal restoration logic is handled in the after-update path by checking NTC.Status__c = 'Withdrawn' and querying CC records with Suspension_Reason__c = 'Network Termination Initiated'. Task creation uses a helper method in TaskFactory.cls. Full unit test coverage in TestClass_CredentialingCaseSuspension.cls targeting 95%+ coverage.

## Description

As a Claims Operations Manager, I need the system to automatically sunset active Fee_Schedule__c records and block in-network repricing on Claim_Line__c records when a provider's Network_Termination_Case__c reaches 'Effective' status, so that post-termination claims are not inadvertently adjudicated at contracted in-network rates.

## Acceptance Criteria

Given a Network_Termination_Case__c where Status__c transitions to 'Effective'
When the record is saved
Then all Fee_Schedule__c records linked to the provider via Fee_Schedule__c.Provider__c where Fee_Schedule__c.Effective_End_Date__c is NULL or greater than Effective_Termination_Date__c are updated: Fee_Schedule__c.Effective_End_Date__c = Effective_Termination_Date__c and Fee_Schedule__c.Status__c = 'Inactive'

Given a Fee_Schedule__c is set to 'Inactive'
When a new Claim_Line__c is submitted with Claim_Line__c.Rendering_Provider__c matching the terminated provider
Then Claim_Line__c.Network_Status__c is set to 'Out-of-Network', Claim_Line__c.Repricing_Blocked__c = TRUE, and Claim_Line__c.Block_Reason__c = 'Provider Terminated – No Active Fee Schedule'

Given existing open Claim__c records (Status__c = 'Pending Adjudication') with Claim_Line__c records linked to the terminated provider
When the termination becomes effective
Then a Salesforce Task is created for each affected Claim__c assigned to the Claims Operations queue with Subject = 'Manual Review Required – Provider Termination Impact' and Priority = 'High'

Given the Fee_Schedule__c sunset process runs
When more than 50 Fee_Schedule__c records are affected for a single provider
Then a Batch Apex job is invoked instead of synchronous processing to avoid governor limit violations and a Custom_Log__c record is written with batch job ID and record counts

## Solution

Implement a Record-Triggered Flow on Network_Termination_Case__c (after update, Status__c changed to 'Effective') that invokes an Apex action. The Apex action queries and bulk-updates Fee_Schedule__c records. A threshold check (>50 records) triggers FeatureScheduleSunsetBatch Apex class. A separate Apex Trigger on Claim_Line__c (before insert/update) checks provider termination status via a cached Custom Metadata lookup (Provider_Termination_Cache__mdt refreshed hourly) and sets Repricing_Blocked__c accordingly. Task creation handled via Apex bulkified DML. All operations wrapped in try/catch with error logging to Custom_Log__c.

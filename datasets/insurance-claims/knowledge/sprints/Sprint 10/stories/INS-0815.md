## Description

**Bug Report**

**Summary:** The repricing logic introduced in INS-0811 contains an off-by-one error in the date comparison. Claims where Claim__c.Service_Date__c equals exactly NTC.Effective_Termination_Date__c are being incorrectly priced at in-network contracted rates instead of OON rates. The boundary condition (service date ON the effective termination date) should be treated as post-termination.

**Steps to Reproduce:**
1. Create a Provider__c with an associated Network_Termination_Case__c where NTC.Status__c = 'Closed - Terminated' and NTC.Effective_Termination_Date__c = 2025-01-20.
2. Submit a Claim__c with Claim__c.Service_Date__c = 2025-01-20 (same day as effective termination date).
3. Run the repricing engine against the claim.
4. Observe Claim__c.Repricing_Basis__c = 'In-Network' (incorrect).

**Expected Behavior:** Claim__c.Repricing_Basis__c = 'Out-of-Network' when Service_Date__c >= NTC.Effective_Termination_Date__c.

**Actual Behavior:** Claim__c.Repricing_Basis__c = 'In-Network' when Service_Date__c = NTC.Effective_Termination_Date__c. Only Service_Date__c strictly greater than (>) the effective date triggers OON repricing.

**Environment:** Full Sandbox (ACM-FULL-SBX02) and Production
**Regression:** Logic defect in FeeScheduleEvaluator.cls line 147 — condition uses '>' instead of '>='.
**Impact:** High — Financial exposure: in-network rates applied to services rendered on or after termination date. Payment__c records may have been issued at incorrect contracted rates. SIU review may be required for affected claims.

## Acceptance Criteria

As per Description. Fix verified by unit test asserting Claim__c.Repricing_Basis__c = 'Out-of-Network' for Service_Date__c equal to, and greater than, NTC.Effective_Termination_Date__c. Regression test must also confirm Service_Date__c one day BEFORE effective termination date still yields 'In-Network' repricing. Claims Operations Manager to confirm Payment__c remediation scope for any incorrectly disbursed payments in production.

## Solution

Fix is a single-line change in FeeScheduleEvaluator.cls: update the date comparison on line 147 from 'serviceDate > terminationDate' to 'serviceDate >= terminationDate'. Deploy hotfix branch (hotfix/INS-0815-fee-schedule-boundary) through standard CI/CD pipeline. Update TestClass_FeeScheduleEvaluator.cls to add explicit boundary condition test methods for equal-date scenario. Post-deployment, run a Salesforce report (RPT_TerminatedProviderClaimsReview) to identify Claim__c records with Service_Date__c = NTC.Effective_Termination_Date__c that were processed since INS-0811 deployment. Coordinate with Claims Operations Manager and Payment Engine team to assess Payment__c reversal and reprocessing needs for affected records.

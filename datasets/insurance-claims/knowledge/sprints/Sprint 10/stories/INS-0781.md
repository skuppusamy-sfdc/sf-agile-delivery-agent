# INS-0781: Automated re-credentialing hold: suspend Credentialing_Case__c renewal when provider initiates voluntary network exit

## Description

As a Credentialing Specialist, I need the system to automatically place any in-progress Credentialing_Case__c renewal on hold when the associated provider initiates a voluntary network exit (i.e., a Network_Termination_Case__c with Termination_Case.Reason__c = 'Voluntary Exit' is created), so that credentialing resources are not wasted on providers who are exiting the network and CAQH re-attestation workflows are not unnecessarily triggered.

## Acceptance Criteria

Given a Network_Termination_Case__c is created with Termination_Case.Reason__c = 'Voluntary Exit' and Termination_Case.Provider__c linked to a Provider__c,
When the record is inserted,
Then any Credentialing_Case__c where Credentialing_Case.Provider__c = same provider and Credentialing_Case.Status__c IN ('In Progress', 'Pending CAQH Attestation', 'Awaiting Committee Review') is updated to Credentialing_Case.Status__c = 'On Hold – Network Exit' and Credentialing_Case.Hold_Reason__c = 'Voluntary Network Termination Initiated'.

Given Credentialing_Case.Status__c = 'On Hold – Network Exit',
When the Network_Termination_Case__c is subsequently set to Status__c = 'Cancelled' (provider withdraws exit request),
Then all held Credentialing_Case__c records revert to their previous Status__c value (stored in Credentialing_Case.Pre_Hold_Status__c) and Hold_Reason__c is cleared.

Given a Credentialing_Case__c is placed on hold,
When the hold is applied,
Then an email notification is sent to the assigned Credentialing Specialist using the 'Credentialing_Hold_Notification' email template, referencing Credentialing_Case.Case_Number__c and Termination_Case.Effective_Date__c.

-Given provider is For Cause termination, When termination case created, Then credentialing case is permanently closed- (For Cause path handled separately in INS-0783; this story covers Voluntary Exit only).

## Solution

Implement a Record-Triggered Flow on Network_Termination_Case__c (trigger: after insert, entry criteria: Reason__c = 'Voluntary Exit'). Use a Get Records element to find related Credentialing_Case__c records by Provider__c with qualifying statuses. Store current Status__c in Pre_Hold_Status__c via an Update Records element before setting the hold status. For the cancellation reversal path, add a separate Record-Triggered Flow on Network_Termination_Case__c (after update, entry criteria: Status__c changed to 'Cancelled') that restores Pre_Hold_Status__c. Use a Flow-triggered email alert for specialist notification referencing the 'Credentialing_Hold_Notification' template. Ensure governor limit safety by processing credentialing case updates in a loop with a collection variable.

## Description

As a Compliance Officer, I need the system to automatically generate Provider_Change_Notification__c (PCN) records and dispatch member-facing communications when a provider exits the network, so that affected policyholders are notified within the regulatory 30-day window and Acme Insurance remains compliant with state DOI requirements.

## Acceptance Criteria

Given a Network_Termination_Case__c.Status__c transitions to 'Provider Notified',
When the record is saved,
Then a batch process identifies all Policy__c records where Policy__c.Primary_Care_Provider__c = the terminated Provider__c.Id OR any active Claim__c.Provider__c matches within the last 24 months.

Given the affected Policy__c records are identified,
When the PCN generation job runs,
Then one Provider_Change_Notification__c is created per Policy__c with fields: PCN.Policy__c, PCN.Terminated_Provider__c, PCN.Notification_Due_Date__c = TODAY() + 30, PCN.Status__c = 'Pending Send', PCN.Regulatory_State__c derived from Policy__c.State__c.

Given PCN.Status__c = 'Pending Send',
When the Conga CLM template merge runs,
Then a PDF letter is generated using template 'PCN_Member_Letter_v3' and attached to the PCN record as a ContentDocument, and an outbound email is dispatched to Policy__c.Policyholder__r.Email.

Given PCN.Notification_Due_Date__c is within 5 days and PCN.Status__c != 'Sent',
When the daily compliance check runs,
Then a High-priority Case is auto-created and assigned to the Compliance Officer queue with subject 'PCN Overdue — Regulatory Risk'.

Given PCN.Status__c = 'Sent',
When the record is updated,
Then PCN.Sent_Timestamp__c is populated, PCN.Delivery_Channel__c is recorded ('Email' or 'USPS'), and the parent Network_Termination_Case__c.PCN_Completion_Status__c field is updated via roll-up.

## Solution

Record-Triggered Flow on Network_Termination_Case__c fires when Status__c changes to 'Provider Notified', invoking an Apex-invocable method (PCNGenerationService.generatePCNs) that performs a SOQL query for affected Policy__c records. PCN records are bulk-inserted. A separate Scheduled Flow triggers nightly to merge Conga CLM templates via HTTP callout to the Conga API, attaching resulting PDFs via ContentVersion insert. Email dispatch uses Salesforce OrgWideEmailAddress with a custom EmailTemplate. A ComplianceOverdueCheckBatch runs daily to flag overdue PCNs. All PCN records are governed by a Sharing Rule ensuring Compliance Officers and Provider Network Specialists have full CRUD access via Permission Set Group 'PCN_Compliance_Ops'.

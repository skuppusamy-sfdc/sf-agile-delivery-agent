# INS-0795: Claim_Line__c adjudication block for providers with active Network_Termination_Case__c past effective date

## Description

As a Claims Operations Manager, I need Claim_Line__c records to be automatically blocked from payment adjudication when the servicing provider has an active Network_Termination_Case__c and the service date on the claim line falls on or after the NTC.Termination_Effective_Date__c, so that we do not inadvertently pay out-of-network rates or process claims for providers who are no longer contracted, preventing financial leakage and compliance violations.

## Acceptance Criteria

Given a Claim_Line__c record is created or updated with Claim_Line__c.Service_Date__c on or after the associated Provider's NTC.Termination_Effective_Date__c,
When the before-insert/before-update Apex Trigger fires,
Then Claim_Line__c.Adjudication_Status__c is set to 'Blocked - Provider Terminated' and Claim_Line__c.Block_Reason__c is populated with 'Provider NPI {NPI} terminated effective {date}'.

Given a Claim_Line__c is blocked,
When the parent Claim__c is evaluated,
Then Claim__c.Status__c is updated to 'Pending Review' and assigned to the Claims Operations Manager queue.

Given a Claim_Line__c is in 'Blocked - Provider Terminated' status,
When a Claims Operations Manager reviews and overrides the block by setting Claim_Line__c.Override_Flag__c = true with a mandatory Claim_Line__c.Override_Justification__c,
Then the adjudication block is lifted and Claim_Line__c.Adjudication_Status__c reverts to 'Ready for Adjudication'.

Given the override is applied,
When the record is saved,
Then an Audit_Log__c record is created capturing: user, timestamp, original status, new status, and Override_Justification__c text.

Given a provider with NTC.Status__c = 'Withdrawn' (termination cancelled),
When the nightly reconciliation batch runs,
Then all Claim_Line__c records blocked under that NTC are released automatically and Claim_Line__c.Adjudication_Status__c is reset to 'Ready for Adjudication'.

Given the adjudication block logic,
When Claim_Line__c.Service_Date__c is BEFORE NTC.Termination_Effective_Date__c,
Then no block is applied regardless of the NTC status (retroactive claims are permitted).

## Solution

Apex Trigger (ClaimLineTrigger) on Claim_Line__c before-insert and before-update contexts queries related Provider__c for any open Network_Termination_Case__c where NTC.Status__c IN ('Approved for Termination','Active Termination') and NTC.Termination_Effective_Date__c <= Claim_Line__c.Service_Date__c. Trigger handler class (ClaimLineTerminationBlockHandler) applies the block and updates parent Claim__c via a separate DML operation in an after-trigger context to avoid mixed DML. Override field validation enforced by a Validation Rule requiring Override_Justification__c to be non-null when Override_Flag__c = true. Nightly Apex Batch (ClaimLineBlockReconBatch) reconciles withdrawn NTCs. Audit logging via Apex insert to Audit_Log__c.

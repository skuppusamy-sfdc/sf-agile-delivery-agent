## Description

As a Provider Relations Specialist, I need a consolidated FlexCard panel on the Provider__c (Person Account) record page that surfaces active Network_Termination_Case__c records, associated PCNs, and current PNA network participation status so that I can assess a provider's full network exit posture from a single screen without navigating multiple related lists.

## Acceptance Criteria

Given a Provider__c record is open in the Salesforce console
When the Provider 360 FlexCard loads
Then the Termination Summary panel displays all Network_Termination_Case__c records where NTC.Provider__c = current Provider Id and NTC.Status__c IN ('Initiated', 'In Review', 'Pending Effective') sorted by NTC.Termination_Effective_Date__c ascending

Given the FlexCard Termination panel is rendered
When NTC.Status__c = 'Initiated'
Then the status badge renders in amber; when NTC.Status__c = 'Pending Effective' the badge renders in red with a countdown showing days until NTC.Termination_Effective_Date__c

Given a Provider has no active Network_Termination_Case__c records
When the FlexCard panel loads
Then an empty-state message 'No active termination cases' is displayed and the panel header shows Provider_Network_Agreement__c count with PNA.Status__c = 'Active'

Given the FlexCard is rendered
When a user clicks the 'View PCN' action on a termination row
Then a modal opens displaying Provider_Change_Notification__c fields: PCN.Notification_Type__c, PCN.Sent_Date__c, PCN.Recipient_Count__c, PCN.Status__c without navigating away from the Provider record

-Display Credentialing_Case__c expiry dates inline within the termination panel- (moved to separate Credentialing 360 card per INS-0844 scope decision)

## Solution

OmniStudio FlexCard built on a DataRaptor Turbo Extract that joins Provider__c, Network_Termination_Case__c, Provider_Change_Notification__c, and Provider_Network_Agreement__c via SOQL with relationship traversal. DataRaptor Extract named 'DR_Provider360_TerminationSummary' returns a flattened JSON payload. FlexCard uses conditional CSS state classes for badge coloring. A custom LWC child component (providerTerminationCountdown) calculates days-remaining client-side using JavaScript Date arithmetic and is embedded via FlexCard custom component slot. The modal for PCN detail reuses the existing pcnDetailModal LWC component from INS-0836. FlexCard is added to the Provider__c Lightning App Builder page layout in the right-hand utility column, visible only to Permission Set Group 'PSG_Provider_Relations'.

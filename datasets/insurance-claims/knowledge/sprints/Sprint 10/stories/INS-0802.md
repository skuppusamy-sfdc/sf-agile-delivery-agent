## Description

As a Provider Network Specialist, I need a guided OmniScript to manually initiate a provider network termination so that I can capture the termination reason, effective date, impacted LOBs, and regulatory notice requirements in a structured, auditable way — ensuring compliance with state-mandated 60/90-day notice periods. Extends INS-0800.

## Acceptance Criteria

Given the OmniScript is launched from a Provider__c or Network_Termination_Case__c record
When Step 1 (Provider Identification) loads
Then the script pre-populates Provider__c.Name, Provider__c.NPI__c, Provider__c.TIN__c, and a read-only list of active PNA records for the provider

Given Step 2 (Termination Details) is presented
When the user selects Termination_Reason__c from the picklist
Then available values include: 'Voluntary Resignation', 'Contract Expiration', 'Credentialing Failure', 'Fraud/Abuse – SIU Referral', 'License Revocation', 'Mutual Agreement'

Given Termination_Reason__c = 'Fraud/Abuse – SIU Referral'
When the user proceeds to Step 3
Then an additional mandatory field SIU_Case_Number__c appears and the OmniScript creates a Task assigned to the SIU queue with Subject = 'SIU Review Required – Provider Termination'

Given the user enters Effective_Termination_Date__c
When the date is fewer than 60 calendar days from today for LOB__c = 'Commercial' or fewer than 90 days for LOB__c = 'Medicare Advantage'
Then an inline warning displays: 'Selected date may not meet state notice requirements. Compliance review required.' and Compliance_Review_Required__c is set to TRUE on the Network_Termination_Case__c

Given all steps are completed
When the user clicks 'Submit'
Then Network_Termination_Case__c is created/updated with Status__c = 'Submitted', all captured data is persisted, and a Conga CLM document package is triggered to generate the formal termination letter

Given the OmniScript submission
When the Conga document is generated
Then the document is attached to the Network_Termination_Case__c as a ContentDocument and PCN.Status__c is updated to 'Letter Generated'

## Solution

Build a 6-step OmniScript 'ProviderTerminationInitiation' using OmniStudio. Use DataRaptor Transforms for pre-population of provider and PNA data. Implement conditional step visibility via OmniScript conditional views for the SIU branch. Date validation logic handled via a custom OmniScript formula function. On submit, invoke an Integration Procedure that calls a Record-Triggered Flow stub to create/update Network_Termination_Case__c and then calls the Conga CLM REST API for document generation. Store Conga output via ContentVersion insert in Apex. Platform Event (TerminationSubmitted__e) published for audit trail downstream consumers.

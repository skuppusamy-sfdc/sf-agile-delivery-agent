# INS-0811: Claim_Line__c repricing logic must suspend for claims linked to terminated provider effective date

## Description

As a Claims Operations Manager, I need the Fee_Schedule__c repricing engine to detect when a Claim__c is submitted for a provider whose Network_Termination_Case__c effective date has passed, so that the claim is automatically routed to out-of-network (OON) repricing and flagged for manual review rather than incorrectly applying in-network contracted rates.

## Acceptance Criteria

Given a Claim__c is submitted where Claim__c.Provider__c has an associated Network_Termination_Case__c with NTC.Effective_Termination_Date__c <= Claim__c.Service_Date__c and NTC.Status__c = 'Closed - Terminated'
When the repricing engine evaluates the claim
Then Claim__c.Repricing_Basis__c = 'Out-of-Network' and Claim__c.Status__c = 'Pending Manual Review'

Given Claim__c.Repricing_Basis__c is set to 'Out-of-Network'
When the Fee_Schedule__c lookup occurs
Then the engine applies the OON fee schedule (Fee_Schedule__c.Schedule_Type__c = 'OON') rather than any contracted PNA rate and Claim_Line__c.Applied_Fee_Schedule__c is updated accordingly

Given the claim is flagged for manual review
When the Claims Operations queue is loaded
Then the claim appears with a visual indicator 'Terminated Provider' in the Claim__c.Review_Flag__c field and is prioritized above standard pending claims

Given a Claim__c.Service_Date__c falls before NTC.Effective_Termination_Date__c (i.e., service rendered while provider was still in-network)
When the repricing engine evaluates the claim
Then in-network Fee_Schedule__c rates are applied and no termination flag is set

Given a batch of 500+ claims in a single repricing run includes terminated provider claims
When the batch completes
Then all affected claims are correctly bifurcated within the same transaction with zero incorrect in-network rate applications (verified by test class assertion)

## Solution

Modify the existing FeeScheduleEngine Apex class (FeeScheduleEvaluator.cls) to include a pre-check query against Network_Termination_Case__c before rate lookup. Add a custom field Claim__c.Terminated_Provider_Flag__c (Checkbox) and Claim__c.Repricing_Basis__c (Picklist). A before-insert/before-update Apex Trigger on Claim__c calls the FeeScheduleEvaluator. Bulk-safe SOQL uses a Map<Id, Network_Termination_Case__c> keyed by Provider__c Id to avoid per-record queries. Custom metadata (Repricing_Rules__mdt) controls the OON fallback schedule ID per LOB. Unit test coverage target: 95%.

## Description

As a Provider Relations Specialist, I need a dedicated FlexCard panel on the Provider__c (Person Account) record page that surfaces active Network_Termination_Case__c records, key termination milestones, and the effective termination date, so that I have a single-pane view of a provider's network exit status without navigating to multiple related lists or running separate reports.

This panel must reflect real-time data and support both voluntary and involuntary termination scenarios across all LOBs including Commercial, Medicare Advantage, and Medicaid.

## Acceptance Criteria

Given a Provider__c record with one or more associated Network_Termination_Case__c records where Status__c != 'Closed',
When the Provider Relations Specialist opens the Provider 360 Lightning Record Page,
Then the 'Network Exit Status' FlexCard panel renders within the page and displays:
  - Network_Termination_Case__c.CaseNumber
  - Network_Termination_Case__c.Termination_Type__c
  - Network_Termination_Case__c.Status__c (color-coded: 'Open' = amber, 'In Review' = blue, 'Approved' = red, 'Closed' = grey)
  - Network_Termination_Case__c.Effective_Termination_Date__c
  - Network_Termination_Case__c.LOB__c
  - Countdown badge: days remaining until Effective_Termination_Date__c (calculated field via DataRaptor)

Given a Provider__c record with NO active Network_Termination_Case__c records,
When the FlexCard panel loads,
Then the panel displays the message 'No active termination cases' with a green status indicator and the panel does not collapse or throw a null reference error.

Given a provider has multiple active Network_Termination_Case__c records (e.g., separate LOB terminations),
When the FlexCard renders,
Then all active cases are listed in a child card repeater sorted by Effective_Termination_Date__c ascending.

-Given the FlexCard must show Fee_Schedule__c expiry date linked to the termination — DEPRECATED: Fee Schedule sunset moved to dedicated FeeScheduleEngine story INS-0875-

Given the Provider Relations Specialist clicks the CaseNumber link within the FlexCard,
When the action fires,
Then the user is navigated to the Network_Termination_Case__c record detail page in the same browser tab.

## Solution

Build an OmniStudio FlexCard ('NetworkExitStatusCard') with a DataRaptor Extract source querying Network_Termination_Case__c filtered by Provider__c = :recordId and Status__c NOT IN ('Closed'). The DataRaptor output feeds a child card repeater element. Conditional styling on the Status__c field uses FlexCard State configuration (color tokens mapped to status values). Days-remaining countdown is computed in the DataRaptor Transform using a formula node (TODAY() subtraction against Effective_Termination_Date__c). The FlexCard is deployed as a custom Lightning Web Component wrapper and placed on the Provider 360 Lightning Record Page via App Builder in the right-hand sidebar panel. A named Sharing Rule ensures Provider Relations Specialists have read access to Network_Termination_Case__c records across all owners. Empty-state handling uses a FlexCard conditional block on record count = 0.

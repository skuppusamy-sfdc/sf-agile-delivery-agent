## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record when a Provider_Network_Agreement__c reaches its termination effective date so that the termination workflow is initiated without manual intervention and no provider exits the network silently.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Termination_Effective_Date__c equals today's date AND PNA.Status__c = 'Active'
When the scheduled flow runs at 00:01 AM system time
Then a Network_Termination_Case__c is created with:
  - NTC.Provider__c = PNA.Provider__c
  - NTC.PNA__c = PNA.Id
  - NTC.Status__c = 'Initiated'
  - NTC.Termination_Reason__c = PNA.Termination_Reason__c
  - NTC.Effective_Date__c = PNA.Termination_Effective_Date__c
  - NTC.LOB__c = PNA.Line_of_Business__c

Given a Network_Termination_Case__c is successfully created
When the record is saved
Then PNA.Status__c is updated to 'Pending Termination' and a Provider_Change_Notification__c (PCN) record is created with PCN.Notification_Type__c = 'Termination Initiated'

Given a PNA where PNA.Termination_Effective_Date__c is in the future
When the scheduled flow runs
Then no Network_Termination_Case__c is created for that PNA

Given a PNA where a Network_Termination_Case__c already exists with NTC.Status__c != 'Closed'
When the scheduled flow runs
Then no duplicate Network_Termination_Case__c is created (deduplication check on NTC.PNA__c)

Given the batch completes processing
When any PNA record fails to generate an NTC
Then an error log entry is written to Integration_Error_Log__c with Error_Source__c = 'TerminationCaseCreation' and the Operations team is notified via email alert

## Solution

Implement a Scheduled-Triggered Flow ('Daily PNA Termination Check') that queries all Provider_Network_Agreement__c records where Termination_Effective_Date__c = TODAY and Status__c = 'Active'. Use a Decision element to check for existing open Network_Termination_Case__c via a Get Records sub-flow. Create Network_Termination_Case__c and PCN records via Create Records elements. Update PNA.Status__c via Update Records element. Fault paths write to Integration_Error_Log__c via Apex action. Schedule via Process Automation > Scheduled Jobs at 00:01 daily. Deploy a Permission Set Group 'Provider_Termination_Ops' granting FLS on all relevant fields.

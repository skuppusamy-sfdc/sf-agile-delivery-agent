## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record when a Provider_Network_Agreement__c reaches its expiry date or when a provider submits a voluntary exit request, so that the termination workflow is initiated without manual intervention and no provider exits the network untracked.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Status__c = 'Active' and PNA.End_Date__c equals today's date,
When the nightly scheduled batch job runs,
Then a Network_Termination_Case__c is created with Termination_Type__c = 'Expiry', linked via NTC.Provider_Network_Agreement__c lookup, and NTC.Status__c = 'Initiated'.

Given a Provider Relations Specialist updates PNA.Termination_Reason__c = 'Voluntary Exit' and sets PNA.Voluntary_Exit_Date__c,
When the record is saved,
Then a Network_Termination_Case__c is created immediately via Record-Triggered Flow with Termination_Type__c = 'Voluntary' and NTC.Effective_Date__c = PNA.Voluntary_Exit_Date__c.

Given a Network_Termination_Case__c is created,
When the record is inserted,
Then a Provider_Change_Notification__c (PCN) is also auto-created with PCN.Change_Type__c = 'Network Exit', PCN.Provider__c = PNA.Provider__c, and PCN.Notification_Status__c = 'Pending'.

Given the Network_Termination_Case__c is created,
When NTC.Status__c = 'Initiated',
Then the assigned Provider Network Specialist receives a Salesforce Task with Subject = 'Review Provider Termination' and ActivityDate = TODAY() + 5.

-Given PNA.Auto_Renew__c = true, the system should suppress termination case creation- (superseded: auto-renew logic moved to INS-0835)

Given a duplicate Network_Termination_Case__c already exists for the same PNA in status 'Initiated' or 'In Review',
When the batch or flow attempts to create another,
Then creation is blocked and a warning Platform Event (Network_Termination_Warning__e) is fired with payload including PNA Id and duplicate flag.

## Solution

Implement a Record-Triggered Flow on Provider_Network_Agreement__c (after save, on create/update) to handle voluntary exit scenario. Build a Schedulable Apex class (PNAExpiryTerminationBatch) running nightly at 02:00 UTC to query PNAs where End_Date__c = TODAY and Status__c = 'Active', bulk-creating Network_Termination_Case__c records. Use a before-insert Apex Trigger on Network_Termination_Case__c to check for duplicates and fire Network_Termination_Warning__e Platform Event if found. Auto-creation of PCN handled via a second Record-Triggered Flow on Network_Termination_Case__c (after insert). Deploy associated Permission Set Group updates to grant Provider Network Specialist write access to NTC object.

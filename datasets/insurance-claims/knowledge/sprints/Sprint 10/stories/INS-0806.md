## Description

**Bug Report**

**Summary:** When a Provider_Change_Notification__c (PCN) DocuSign envelope is dispatched as part of the termination workflow (INS-0800), the system is routing the envelope to the first Contact record alphabetically associated with the Provider__c Person Account rather than the contact designated with Contact.PCN_Primary_Recipient__c = TRUE.

**Steps to Reproduce:**
1. Create a Provider__c (Person Account) with two or more associated Contact records. Set PCN_Primary_Recipient__c = TRUE on one specific contact.
2. Trigger a Network_Termination_Case__c creation for this provider (manually or via batch).
3. Observe the auto-created PCN record and the DocuSign envelope recipient email.
4. Compare recipient email to the contact where PCN_Primary_Recipient__c = TRUE.

**Expected Result:** DocuSign envelope is addressed to the Contact where PCN_Primary_Recipient__c = TRUE and Contact.Email is used as the recipient address.

**Actual Result:** Envelope is sent to the alphabetically first Contact.Email associated with the provider, regardless of PCN_Primary_Recipient__c flag.

**Root Cause Hypothesis:** The DocuSign invocable action in the Record-Triggered Flow is querying contacts with LIMIT 1 and ORDER BY Name ASC instead of filtering WHERE PCN_Primary_Recipient__c = TRUE.

**Frequency:** 100% reproducible when provider has multiple contact records. Affects approximately 340 providers in production based on data query.

**Severity:** High – regulatory notices being delivered to incorrect recipients, potential compliance violation.

## Acceptance Criteria

As per Description. Fix verified when DocuSign envelope recipient email matches Contact.Email where Contact.PCN_Primary_Recipient__c = TRUE for all PCN records generated post-fix. Regression test confirms single-contact providers are unaffected.

## Solution

Update the SOQL query in the DocuSign invocable Apex action (PCNDocuSignDispatchAction.cls) to filter: SELECT Id, Email FROM Contact WHERE AccountId = :providerId AND PCN_Primary_Recipient__c = TRUE LIMIT 1. Add a null-check fallback: if no contact with PCN_Primary_Recipient__c = TRUE exists, fall back to the contact with IsPrimary__c = TRUE and log a warning to Custom_Log__c. Add a validation rule on Provider__c to warn (not block) when zero contacts have PCN_Primary_Recipient__c = TRUE. Deploy fix with unit tests covering: single contact, multiple contacts with flag set, multiple contacts with no flag set. Backfill review of previously sent envelopes via a one-time data report for Compliance Officer review.

## Description

As a Provider Relations Specialist, I need a structured OmniScript-driven exit interview workflow to capture a departing provider's voluntary exit reason, satisfaction ratings, and free-text feedback when they initiate a voluntary network termination, so that Provider Network leadership can analyze exit trends and improve retention strategies.

## Acceptance Criteria

Given a Provider_Network_Agreement__c with PNA.Termination_Reason__c = 'Voluntary Exit',
When the Provider Relations Specialist launches the 'Provider Exit Interview' OmniScript from the PNA record action menu,
Then the OmniScript loads pre-populated with Provider NPI, TIN, PNA.Contract_Number__c, and PNA.Network__c.

Given the OmniScript is active,
When the specialist completes Step 1 (Exit Reason),
Then the picklist field NTC.Exit_Primary_Reason__c is required and must capture one of: 'Reimbursement Rate', 'Administrative Burden', 'Relocation', 'Retirement', 'Contract Dispute', 'Other'.

Given Step 2 (Satisfaction Survey) is presented,
When the specialist records ratings,
Then NTC.Reimbursement_Satisfaction_Score__c (1–5 integer), NTC.Admin_Support_Score__c (1–5 integer), and NTC.Overall_Satisfaction_Score__c (1–5 integer) are all captured and validated for range.

Given all steps are completed and the OmniScript is submitted,
When the DataRaptor Transform executes,
Then the Network_Termination_Case__c is updated with exit interview data and NTC.Exit_Interview_Completed__c = true and NTC.Exit_Interview_Date__c = TODAY().

Given NTC.Exit_Primary_Reason__c = 'Reimbursement Rate' and NTC.Reimbursement_Satisfaction_Score__c <= 2,
When the OmniScript is submitted,
Then an automatic Task is created for the Provider Network Specialist with Subject = 'Rate Review Requested - Exiting Provider' and Priority = 'High'.

## Solution

Build a 4-step OmniScript ('Provider_Exit_Interview_OS') in OmniStudio: Step 1 - Exit Reason (picklist + conditional 'Other' text input); Step 2 - Satisfaction Ratings (number input elements with client-side range validation); Step 3 - Free Text Feedback (textarea, max 2000 chars); Step 4 - Confirmation summary. Use a DataRaptor Load to upsert Network_Termination_Case__c fields on submit. Implement a conditional Integration Procedure call post-submit to create the High priority Task when reimbursement conditions are met. Add the OmniScript as a Quick Action on Provider_Network_Agreement__c Lightning Record Page. Restrict action visibility via Permission Set to Provider Relations Specialist and Provider Network Specialist profiles.

## Description

As a Provider Network Specialist, I need a guided OmniScript that walks me through the end-to-end voluntary provider termination process — from capturing the exit request details through to generating the termination letter via DocuSign — so that the process is standardized, audit-ready, and no mandatory steps are skipped regardless of which specialist handles the case.

## Acceptance Criteria

Given a Provider Network Specialist opens the OmniScript 'VoluntaryProviderTermination_OS' from a Provider__c record
When Step 1 (Exit Request Capture) is completed
Then the OmniScript validates that Provider_Network_Agreement__c.Status__c = 'Active' before proceeding; if no active PNA exists, an error message 'No active PNA found for this provider. Please verify provider network status.' is displayed and the script halts

Given Step 1 validation passes
When the specialist completes Step 2 (Termination Details)
Then fields captured include: Requested_Effective_Date__c (date picker, must be >= today + 30 days), Termination_Reason__c (picklist), Additional_Notes__c (text area, max 1000 chars), and Accepting_New_Patients_Until__c (date)

Given Step 2 is completed
When Step 3 (Impacted Members Review) renders
Then a DataRaptor-driven table displays all Policy__c records with Policy__c.PCP_Provider__c = Provider__c.Id, showing Member_Name__c, Policy_Number__c, LOB__c, and Plan_Type__c — specialist must check an acknowledgment checkbox before proceeding

Given Step 3 acknowledgment is confirmed
When Step 4 (DocuSign Termination Agreement) renders
Then the OmniScript triggers a DocuSign envelope via Integration Procedure 'IP_SendTerminationDocuSign' with the provider's email pre-populated from Provider__c.PersonEmail and the template mapped to Termination_Type__c = 'Voluntary'

Given DocuSign envelope is sent successfully
When Step 5 (Summary & Submit) is completed
Then Network_Termination_Case__c is created (or updated if one exists in 'Draft' status) with all captured fields mapped, PNA.Voluntary_Exit_Requested__c = true, and a confirmation toast message displays the new case number

Given the OmniScript is accessible
When a user without Permission Set 'Provider_Termination_Ops' attempts to launch it
Then the script renders an access denied message and logs the attempt to Integration_Log__c with Log_Type__c = 'Unauthorized Access Attempt'

## Solution

OmniStudio OmniScript (Type: VoluntaryProviderTermination, Sub Type: OS) with 5 steps. Step 1 uses a DataRaptor Extract to check PNA status. Step 3 uses a DataRaptor Turbo Extract for member list with pagination. Step 4 calls Integration Procedure 'IP_SendTerminationDocuSign' which makes a REST callout to DocuSign eSignature API via Named Credential 'DocuSign_NC'. Step 5 calls Integration Procedure 'IP_UpsertTerminationCase' which uses DataRaptor Transform + Load to upsert Network_Termination_Case__c. Permission check in Step 1 uses a DataRaptor Extract querying PermissionSetAssignment. OmniScript embedded in Provider__c Lightning Record Page via custom Lightning Web Component wrapper for launch button.

## Description

As a Provider Relations Specialist, I need the Provider 360 FlexCard on the Provider__c (Person Account) record page to display an active termination status badge, effective termination date, and the associated Network_Termination_Case__c case number, so that I can immediately identify providers undergoing network exit without navigating away from the account. Extends INS-0783 (Provider 360 base FlexCard build).

## Acceptance Criteria

Given a Provider__c record with one or more related Network_Termination_Case__c records where NTC.Status__c != 'Closed'
When the Provider 360 FlexCard loads
Then a red 'Termination In Progress' badge is rendered in the header section of the FlexCard

Given the termination badge is visible
When the specialist clicks the badge
Then an expanded panel displays NTC.Case_Number__c, NTC.Termination_Type__c, NTC.Effective_Termination_Date__c, and NTC.Status__c for each open termination case

Given a Provider__c has no open Network_Termination_Case__c records
When the FlexCard loads
Then no termination badge is shown and the 'Network Status' indicator displays Provider__c.Network_Status__c = 'Active' in green

Given the FlexCard is rendered on a mobile form factor
When the screen width is below 768px
Then the termination panel collapses to a single-line summary with case number and effective date only

Given a Provider Relations Specialist with read-only Permission Set Group (PSG_ProviderRelations_ReadOnly)
When the FlexCard loads
Then the termination data is visible but no action buttons (e.g., 'Rescind Termination') are rendered

## Solution

Extend the existing OmniStudio FlexCard (ProviderAccount360_FC) built in INS-0783 by adding a new card state for 'Termination Active'. Use a DataRaptor Extract (DR_GetOpenTerminationCases) to query Network_Termination_Case__c filtered by Provider__c lookup and NTC.Status__c NOT IN ('Closed', 'Withdrawn'). Conditional rendering logic in the FlexCard JSON drives badge color and panel visibility. Mobile responsiveness handled via FlexCard breakpoint configuration. Permission-aware button rendering uses FlexCard conditional actions tied to the logged-in user's Permission Set Group membership evaluated via a custom formula field on the User object.

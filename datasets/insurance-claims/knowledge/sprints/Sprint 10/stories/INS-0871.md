## Description

As a Compliance Officer, I need a PCN dispatch retry queue and a real-time compliance monitoring dashboard so that I can track all Provider_Change_Notification__c records that failed DocuSign delivery, trigger retries up to 3 attempts, and escalate unresolvable failures before the state-mandated 90-day compliance window closes. This story implements the retry logic deprecated from INS-0866.

## Acceptance Criteria

Given a Provider_Change_Notification__c where PCN.DocuSign_Status__c = 'Failed' and PCN.Retry_Count__c < 3,
When the PCN Retry Scheduled Flow runs every 6 hours,
Then a new DocuSign dispatch is attempted and PCN.Retry_Count__c is incremented by 1 and PCN.Last_Retry_Timestamp__c = NOW().

Given a PCN retry attempt succeeds (DocuSign status = 'Delivered'),
When the webhook callback updates the record,
Then PCN.DocuSign_Status__c = 'Delivered', PCN.Retry_Resolved__c = true, and no further retries are scheduled.

Given PCN.Retry_Count__c = 3 and PCN.DocuSign_Status__c still = 'Failed',
When the retry queue evaluates the record,
Then PCN.Escalation_Status__c = 'Escalated' is set and a Case is auto-created with Subject = 'PCN Delivery Failure – Escalation Required', Origin = 'System', Priority = 'High', and assigned to the 'Compliance_Escalation' Queue.

Given the compliance monitoring dashboard is accessed by a Compliance Officer,
When the dashboard loads,
Then it displays: (1) Total PCNs dispatched this month, (2) PCNs Delivered vs. Failed vs. Pending, (3) PCNs within 10 days of Compliance_Deadline__c with status != 'Completed', (4) Average delivery time in hours, (5) Escalated PCN count by Network_Termination_Case__c.

Given a PCN record where PCN.Compliance_Deadline__c - TODAY() <= 5 and PCN.DocuSign_Status__c != 'Completed',
When the daily compliance check Flow runs,
Then PCN.Compliance_Risk_Flag__c = 'Critical' is set and the assigned Compliance Officer receives a Salesforce In-App Notification via Custom Notification Type 'PCN_Critical_Alert'.

## Solution

Retry queue implemented as a Scheduled Flow (every 6 hours) querying PCN records with DocuSign_Status__c = 'Failed' and Retry_Count__c < 3, calling Apex-invocable DocuSignRetryService.retryDispatch() in a loop with bulkification. Escalation Case creation via Flow Create Records action. Compliance monitoring dashboard built in CRM Analytics (Tableau CRM) with SAQL-based datasets joining Provider_Change_Notification__c, Network_Termination_Case__c, and Policy__c. Alternatively, a Lightning Dashboard with custom Report Types can be used as an MVP. Compliance_Risk_Flag__c updates and Custom Notifications dispatched via a Scheduled Flow using the 'Send Custom Notification' core action. Retry_Count__c and Last_Retry_Timestamp__c fields added to Provider_Change_Notification__c as part of this story's field deployment.

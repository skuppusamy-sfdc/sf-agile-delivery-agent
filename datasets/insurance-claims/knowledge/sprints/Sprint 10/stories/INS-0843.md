## Description

**Bug Report**

**Environment:** Acme Insurance UAT Sandbox (Sprint 10)
**Severity:** High — blocking termination notice generation (INS-0842)

**Steps to Reproduce:**
1. Prepare a CSV of 50+ Provider_Network_Agreement__c records with Status__c = 'Pending Termination' and End_Date__c populated.
2. Execute bulk upsert via Data Loader using the PNA external ID field.
3. Navigate to any resulting Network_Termination_Case__c record created by the INS-0840 automation.
4. Observe the Effective_Termination_Date__c field on the NTC record.

**Expected Behavior:**
NTC.Effective_Termination_Date__c should be populated with PNA.End_Date__c for all bulk-loaded records, consistent with behavior observed during single-record UI saves.

**Actual Behavior:**
NTC.Effective_Termination_Date__c is NULL for all NTC records created via bulk load. The field populates correctly when PNA records are saved individually through the UI or via Developer Console single-record DML.

**Root Cause Hypothesis:**
The Record-Triggered Flow (INS-0840) uses a Get Records element to retrieve PNA.End_Date__c in a separate query rather than referencing the triggering record's fields directly. In bulk context, the Get Records element may be querying stale data before the transaction commits, returning NULL for End_Date__c.

**Impact:**
- 100% of bulk-terminated providers missing effective date on NTC
- DocuSign notice generation (INS-0842) halted — NTC enters compliance flag state
- Estimated 200+ providers affected in upcoming Q4 network exit batch

## Acceptance Criteria

As per Description. Fix must be validated by re-running the 50-record bulk load in UAT and confirming NTC.Effective_Termination_Date__c is populated on all resulting records. Regression test must confirm single-record UI behavior is unaffected.

## Solution

Investigate the Record-Triggered Flow from INS-0840 and replace the Get Records element fetching PNA.End_Date__c with a direct reference to the triggering record variable {!$Record.End_Date__c} to avoid the stale-read issue in bulk context. If the flow requires related record data beyond the trigger record, use a before-save flow entry to cache fields into flow variables prior to any Get Records calls. Deploy fix to UAT, execute 100-record bulk load test, and validate with SOQL count query confirming zero NULL Effective_Termination_Date__c values on resulting NTC records. Update flow description and add inline comments.

## Description

As a Provider Network Specialist, I need a guided OmniScript intake form to capture a provider's voluntary network exit request, including exit reason, last service date, forwarding contact, and patient transition plan acknowledgment, so that all required data is collected consistently before a Network_Termination_Case__c is created and downstream PCN and credentialing suspension workflows are triggered accurately.

## Acceptance Criteria

Given a Provider Network Specialist opens the 'Voluntary Exit Request' OmniScript from the Provider__c record page action menu
When the OmniScript launches
Then Step 1 pre-populates Provider__c.Name, Provider__c.NPI__c, Provider__c.TIN__c, and the active PNA record name via DataRaptor Extract (DR_GetActivePNA)

Given Step 2 of the OmniScript
When the specialist selects NTC.Termination_Reason__c from a dynamic picklist
Then the available options are driven by Custom Metadata (Exit_Reason_Config__mdt) filtered by the provider's LOB and the selection is required before proceeding

Given the specialist reaches Step 3 (Patient Transition Plan)
When they indicate the provider has active patients (toggle = TRUE)
Then a conditional sub-step appears requiring entry of Transition_Plan_Notes__c (long text, 2000 char max) and Patient_Transition_Deadline__c (date, must be >= NTC.Effective_Termination_Date__c)

Given all required fields are completed
When the specialist clicks 'Submit'
Then the OmniScript calls a DataRaptor Transform to upsert PNA.Exit_Request_Flag__c = TRUE, PNA.Voluntary_Exit_Reason__c, and PNA.Last_Service_Date__c, which in turn triggers the Record-Triggered Flow from INS-0808 to create the Network_Termination_Case__c

Given the OmniScript submission is successful
When the final confirmation step renders
Then the newly created NTC.Case_Number__c is displayed with a hyperlink, and a summary email is sent to the specialist via OmniScript Email Action using template (ET_VoluntaryExitConfirmation)

Given a network specialist without the 'Provider_Network_Edit' Permission Set Group
When they attempt to open the OmniScript
Then access is denied with message: 'You do not have permission to submit voluntary exit requests. Contact your system administrator.'

-Given Step 4 collects CAQH re-attestation waiver acknowledgment- (deprecated: CAQH waiver step deferred to Phase 3 per INS-0799 scope reduction)

## Solution

Build a multi-step OmniScript (OS_ProviderVoluntaryExitIntake) in OmniStudio Designer with 4 active steps (Step 4 deprecated). DataRaptor Extract (DR_GetActivePNA) pre-populates provider context. Custom Metadata (Exit_Reason_Config__mdt) powers the dynamic reason picklist via a DataRaptor Turbo Extract. Step 3 conditional rendering uses OmniScript conditional views tied to the patient toggle boolean. Submission uses a DataRaptor Transform (DRT_UpsertPNAExitFields) writing back to Provider_Network_Agreement__c. OmniScript is surfaced as a Quick Action on the Provider__c page layout, restricted via Permission Set Group (PSG_ProviderNetwork_Edit). Email action uses Marketing Cloud Transactional Messaging or, as fallback, a standard Salesforce Email Alert.

## Description

**Bug Report**

**Summary:** When a Network_Termination_Case__c is created with Termination_Type__c = 'For Cause', the NTC.Effective_Termination_Date__c field remains NULL despite the Record-Triggered Flow logic introduced in INS-0808 that should auto-populate it as PNA.End_Date__c + 30 days.

**Steps to Reproduce:**
1. Navigate to an Active Provider_Network_Agreement__c record.
2. Set PNA.Exit_Request_Flag__c = TRUE and PNA.Voluntary_Exit_Reason__c = 'For Cause'.
3. Save the record.
4. Open the newly created Network_Termination_Case__c.
5. Observe NTC.Effective_Termination_Date__c is blank.

**Expected Behavior:** NTC.Effective_Termination_Date__c should be auto-populated as PNA.End_Date__c + 30 days per the flow logic defined in INS-0808 AC3.

**Actual Behavior:** NTC.Effective_Termination_Date__c is NULL. The PCN created downstream (INS-0810 flow) then fails validation because the date field is required.

**Environment:** Full Sandbox (ACM-FULL-SBX02)
**Regression:** Introduced in Sprint 10 deployment on 2025-01-14
**Impact:** High — For Cause terminations cannot proceed; PCN dispatch is blocked for all For Cause cases.

## Acceptance Criteria

As per Description. Verify NTC.Effective_Termination_Date__c is populated for ALL Termination_Type__c values including 'For Cause', 'Voluntary Exit', and 'Contract Expiry' after fix is deployed to sandbox.

## Solution

Root cause identified: the Record-Triggered Flow (RTF_PNA_TerminationCaseCreate) has a decision element branch for 'For Cause' that routes to a separate path missing the date assignment formula element. Fix: add a formula resource (frm_EffectiveTermDate) computing {!PNA.End_Date__c} + 30 to the 'For Cause' branch path and wire it to the NTC Create element's Effective_Termination_Date__c field assignment. Deploy fix to ACM-FULL-SBX02, execute test scenario, then promote to UAT. Update flow version notes and regression test suite (TestClass_TerminationCaseFlow.cls).

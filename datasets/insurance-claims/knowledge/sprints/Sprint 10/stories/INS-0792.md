## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record when a Provider_Network_Agreement__c record reaches its termination trigger date or is manually set to 'Pending Termination' status, so that the network exit workflow is initiated without manual intervention and no provider exits the network without a formal case on record.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Status__c is updated to 'Pending Termination' by a Provider Relations Specialist,
When the Record-Triggered Flow fires on the PNA record save,
Then a new Network_Termination_Case__c is created with the following field values:
  - NTC.Provider__c = PNA.Provider__c
  - NTC.PNA_Reference__c = PNA.Id
  - NTC.Termination_Effective_Date__c = PNA.Termination_Date__c
  - NTC.Status__c = 'Open'
  - NTC.LOB__c = PNA.Line_of_Business__c
  - NTC.Assigned_To__c = PNA.OwnerId

Given a Network_Termination_Case__c is successfully created,
When the record is saved,
Then a Provider_Change_Notification__c (PCN) record is auto-created linked to the NTC with PCN.Notification_Type__c = 'Network Exit' and PCN.Status__c = 'Pending Dispatch'.

Given a PNA record where PNA.Termination_Date__c is within 30 days of today,
When the nightly scheduled Apex batch job runs,
Then any PNA without an existing open Network_Termination_Case__c is flagged and a case is created automatically.

Given duplicate prevention logic is active,
When a PNA already has an open Network_Termination_Case__c (NTC.Status__c != 'Closed'),
Then no duplicate NTC is created and an error log entry is written to the Application_Log__c object.

-Given PNA.Auto_Renew__c = true, suppress termination case creation- (deprecated: auto-renew logic moved to INS-0795)

Given a Network_Termination_Case__c is created,
When viewed by a Claims Operations Manager,
Then the record is visible in the Provider 360 FlexCard under the 'Active Termination Cases' section.

## Solution

Implement a Record-Triggered Flow on Provider_Network_Agreement__c firing on create/update when Status__c changes to 'Pending Termination'. Flow creates Network_Termination_Case__c and chained Provider_Change_Notification__c using Create Records elements. A separate Apex Schedulable batch class (PNATerminationScanBatch) runs nightly at 02:00 UTC querying PNAs where Termination_Date__c <= TODAY+30 and no open NTC exists. Duplicate check performed via SOQL in a before-insert Apex Trigger on Network_Termination_Case__c. Error logging via custom Application_Log__c platform event handler. FlexCard updated in OmniStudio to surface NTC records on Provider Account page.

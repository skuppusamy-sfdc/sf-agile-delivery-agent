## Description

As a Credentialing Specialist, I need the system to automatically suspend any open Credentialing_Case__c records associated with a provider when their Network_Termination_Case__c is approved, and place a re-verification hold flag in the CAQH integration, so that credentialing resources are not wasted on providers exiting the network and our CAQH data remains accurate.

## Acceptance Criteria

Given a Network_Termination_Case__c.Status__c = 'Approved for Termination' for a given Provider__c,
When the approval is saved,
Then all related Credentialing_Case__c records where CC.Status__c IN ('In Review', 'Pending Documents', 'Awaiting CAQH') are updated to CC.Status__c = 'Suspended' and CC.Suspension_Reason__c = 'Provider Network Termination'.

Given Credentialing_Case__c records are suspended,
When the CAQH integration sync runs (every 4 hours via DataIntegration component),
Then a CAQH API PUT request is made with the provider's NPI setting ReVerificationHold = true and HoldReason = 'Network Exit – Acme Insurance'.

Given the CAQH API call succeeds,
When the response is received,
Then Credentialing_Case__c.CAQH_Hold_Confirmed__c = true and CAQH_Hold_Date__c = TODAY() are stamped on all suspended records.

Given the CAQH API call fails with a non-200 response,
When the failure is logged,
Then a Platform Event (Integration_Error_Event__e) is published with Error_Source__c = 'CAQH', Error_Code__c, and Provider_NPI__c for monitoring dashboard consumption.

## Solution

Record-Triggered Flow on Network_Termination_Case__c (After Save, criteria: Status__c changes to 'Approved for Termination') queries related Credentialing_Case__c via SOQL in an Apex-invocable action (CredentialingSuspensionService.suspendCases()). Bulk update performed via Database.update(). CAQH integration uses a named credential (CAQH_API_NC) and a custom Apex callout class (CAQHIntegrationService.setReVerificationHold()). Integration errors publish Integration_Error_Event__e via EventBus.publish(). All callouts are asynchronous via @future(callout=true) to avoid trigger callout restrictions.

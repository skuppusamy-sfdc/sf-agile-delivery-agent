## Description

As a Compliance Officer, I need the system to automatically generate a Provider_Change_Notification__c (PCN) document and dispatch it via DocuSign to the provider's NPI-registered contact email when a Network_Termination_Case__c is approved by the Provider Network Manager, so that we meet the CMS-mandated 60-day advance termination notice requirement and maintain an auditable dispatch record.

## Acceptance Criteria

Given a Network_Termination_Case__c where NTC.Status__c is updated to 'Approved' by a user with the 'Provider_Network_Manager' role
When the record is saved
Then a Provider_Change_Notification__c is created with PCN.Type__c = 'Termination Notice', PCN.Dispatch_Status__c = 'Pending', and PCN.Related_NTC__c populated

Given the PCN record is created
When the Conga CLM template merge runs
Then the PCN document is populated with Provider__c.NPI__c, Provider__c.TIN__c, PNA.Network_Name__c, NTC.Effective_Termination_Date__c, and the termination reason from NTC.Termination_Reason__c

Given the Conga merge completes successfully
When the DocuSign envelope is dispatched
Then the envelope is sent to Provider__c.NPI_Contact_Email__c and PCN.Dispatch_Timestamp__c is stamped with the send datetime and PCN.Dispatch_Status__c = 'Sent'

Given NTC.Effective_Termination_Date__c is less than 60 days from TODAY()
When the approval is attempted
Then a validation rule blocks approval and displays the error: 'Termination effective date must be at least 60 days from today to meet CMS notice requirements.'

Given a DocuSign envelope dispatch fails (HTTP 4xx or 5xx response)
When the failure is detected
Then PCN.Dispatch_Status__c = 'Failed', a Case comment is appended to the NTC, and a high-priority Task is assigned to the Provider Relations Specialist owner of the NTC

-Given PCN must be co-signed by the provider's designated representative- (deprecated: co-signature requirement removed per legal review on 2024-11-02)

## Solution

Record-Triggered Flow on Network_Termination_Case__c fires on Status__c change to 'Approved'. Flow calls an Apex invocable action (PCNGenerationService.initiate()) which triggers a Conga CLM API callout for document merge using a predefined template (TMPL_ProviderTerminationNotice). On successful merge, a second Apex callout dispatches the DocuSign envelope via the DocuSign eSignature REST API. DocuSign callback webhook updates PCN.Dispatch_Status__c via a custom REST endpoint exposed as a Named Credential-authenticated Site. Validation rule on Network_Termination_Case__c enforces the 60-day CMS rule. Error handling uses a custom exception logger writing to Integration_Error_Log__c.

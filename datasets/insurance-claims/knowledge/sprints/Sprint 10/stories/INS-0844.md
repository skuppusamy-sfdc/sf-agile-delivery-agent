# INS-0844: Create automated claims re-routing logic to redirect in-flight claims away from terminated providers

## Description

As a Claims Operations Manager, I need in-flight Claim__c records associated with a provider whose Network_Termination_Case__c reaches Status__c = 'Termination Confirmed' to be automatically flagged and routed to an Assignment Coordinator queue for re-assignment to an active in-network provider, so that claims are not stranded or incorrectly adjudicated against a terminated provider's fee schedule.

## Acceptance Criteria

Given a Network_Termination_Case__c where NTC.Status__c transitions to 'Termination Confirmed'
When the Platform Event (Provider_Termination_Event__e) is consumed by the claims re-routing Flow
Then all Claim__c records where:
  - Claim.Rendering_Provider__c = NTC.Provider__c
  - Claim.Status__c IN ('Received', 'In Adjudication', 'Pending Information')
  - Claim.Date_of_Service__c >= NTC.Effective_Termination_Date__c
Are updated with:
  - Claim.Reroute_Required__c = TRUE
  - Claim.Reroute_Reason__c = 'Provider Network Exit'
  - Claim.Status__c = 'Pending Reassignment'
  - Claim.OwnerId = [Assignment Coordinator Queue Id from Custom Metadata]

Given Claim.Date_of_Service__c is before NTC.Effective_Termination_Date__c
When the re-routing logic evaluates the claim
Then the claim is NOT flagged for re-routing and continues standard adjudication

Given more than 100 Claim__c records are associated with the terminated provider
When the Platform Event is processed
Then processing is handled in batches via Apex Batch class (BatchClaimsRerouter) with batch size of 50 to avoid governor limits

Given a Claim__c is flagged with Claim.Reroute_Required__c = TRUE
When an Assignment Coordinator opens the claim
Then the Provider_Reassignment_OmniScript launches automatically via a Lightning component action, pre-populated with the original Claim.Rendering_Provider__c and Claim.Service_Location__c

Given the batch job completes
When BatchClaimsRerouter.finish() executes
Then a summary email is sent to the Claims Operations Manager listing total claims re-routed, total skipped, and any errors

## Solution

Publish Provider_Termination_Event__e Platform Event from the INS-0840 termination flow upon NTC status reaching 'Termination Confirmed'. A Platform Event-Triggered Flow subscribes and invokes Apex invocable 'ClaimsRerouteEvaluator' which performs a SOQL query on Claim__c with the provider and date filters. For bulk scenarios (>100 records), the invocable enqueues BatchClaimsRerouter (implements Database.Batchable, Database.Stateful). Queue Id and batch size stored in Custom Metadata (ClaimsReroute_Config__mdt). OmniScript 'Provider_Reassignment_OS' embedded on Claim__c record page via Lightning App Builder action, visible only when Claim.Reroute_Required__c = TRUE. Batch completion email uses Messaging.sendEmail with a custom HTML template stored in EmailTemplate object.

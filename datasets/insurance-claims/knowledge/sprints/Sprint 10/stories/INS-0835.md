## Description

As a Compliance Officer, I need the system to enforce a 90-day advance written notice requirement for all involuntary provider network terminations and generate a compliant termination notice document via DocuSign, so that Acme Insurance meets state regulatory obligations and avoids penalties for improper network exit processing. This story implements the notice dispatch and tracking component of the termination workflow initiated in INS-0832.

## Acceptance Criteria

Given a Network_Termination_Case__c with NTC.Termination_Type__c = 'Involuntary' and NTC.Status__c = 'Initiated',
When the Compliance Officer or Provider Network Specialist advances the case to NTC.Status__c = 'Notice Required',
Then the system calculates NTC.Notice_Dispatch_Deadline__c = NTC.Effective_Date__c - 90 days and sets NTC.Compliance_Notice_Status__c = 'Pending Dispatch'.

Given NTC.Compliance_Notice_Status__c = 'Pending Dispatch',
When the record transitions to 'Notice Required',
Then a DocuSign envelope is auto-generated using the 'Provider_Termination_Notice_v2' template, pre-populated with Provider NPI, TIN, PNA contract number, effective termination date, and Acme signatory name from NTC.Authorized_Signatory__c.

Given the DocuSign envelope is sent,
When the provider completes or declines the acknowledgment,
Then NTC.Compliance_Notice_Status__c updates to 'Acknowledged' or 'Declined' respectively via DocuSign callback and NTC.Notice_Sent_Date__c is stamped.

Given today's date > NTC.Notice_Dispatch_Deadline__c and NTC.Compliance_Notice_Status__c = 'Pending Dispatch',
When the daily compliance check batch runs,
Then a Compliance Breach record (Compliance_Event__c) is created with Event_Type__c = 'Notice Deadline Missed' and a High priority Case is opened and assigned to the Compliance Officer queue.

Given NTC.Termination_Type__c = 'Voluntary' or 'Expiry',
When the termination case is processed,
Then the 90-day notice workflow is bypassed and NTC.Compliance_Notice_Status__c = 'Not Required'.

-Notice template version 'Provider_Termination_Notice_v1' to remain active as fallback- (deprecated: v1 template retired per legal review 2024-10-30; v2 mandatory for all new cases)

Given a Network_Termination_Case__c is closed or withdrawn,
When NTC.Status__c = 'Withdrawn',
Then any open DocuSign envelope is voided via API call and NTC.Compliance_Notice_Status__c = 'Voided'.

## Solution

Build a Record-Triggered Flow on Network_Termination_Case__c (after update, when Status__c changes to 'Notice Required') to compute Notice_Dispatch_Deadline__c via formula and invoke the DocuSign for Salesforce managed package action ('Send Envelope') referencing template 'Provider_Termination_Notice_v2'. Create a Schedulable Apex class (ComplianceNoticeDeadlineBatch) to run daily, querying NTCs past deadline. Implement a DocuSign Connect webhook handler (Apex REST endpoint) to process envelope status callbacks and update NTC fields. Use a separate Record-Triggered Flow to void envelopes on status = 'Withdrawn' via DocuSign API callout. Add Compliance_Event__c object to ComplianceReporting Permission Set Group.

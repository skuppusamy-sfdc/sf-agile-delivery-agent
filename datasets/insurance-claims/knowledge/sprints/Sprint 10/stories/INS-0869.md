# INS-0869: Enforce Fee_Schedule__c deactivation and Claim_Line__c repricing block for terminated provider NPI post-exit date

## Description

As a Claims Operations Manager, I need the system to automatically deactivate all Fee_Schedule__c records linked to a terminated provider's PNA and block any new Claim_Line__c repricing at in-network rates for service dates after the effective termination date, so that Acme Insurance does not inadvertently pay in-network rates for out-of-network services rendered post-termination.

## Acceptance Criteria

Given a Network_Termination_Case__c.Status__c = 'Closed – Termination Complete' for a Provider__c,
When the NTC is closed,
Then all Fee_Schedule__c records where FS.Provider_Network_Agreement__c = NTC.Provider_Network_Agreement__c and FS.Status__c = 'Active' are updated to FS.Status__c = 'Inactive' and FS.Deactivation_Reason__c = 'Provider Network Termination'.

Given a Fee_Schedule__c is deactivated,
When a new Claim__c is submitted with Claim.Rendering_NPI__c matching the terminated provider and Claim.Service_Date__c >= NTC.Effective_Termination_Date__c,
Then the FeeScheduleEngine returns no in-network rate and Claim__c.Pricing_Basis__c is set to 'UCR' (Usual, Customary, and Reasonable).

Given a Claim_Line__c is being adjudicated with an in-network fee schedule rate,
When ClaimLineAdjudicator validates provider network status,
Then if Provider__c.Network_Status__c = 'Terminated' and Claim_Line__c.Service_Date__c >= Provider__c.Termination_Effective_Date__c, adjudication is halted and Claim_Line__c.Adjudication_Hold_Reason__c = 'Provider Terminated – Rate Review Required'.

Given a Claim_Line__c is placed on hold for rate review,
When a Claims Operations Manager reviews the record,
Then they can manually override with Claim_Line__c.Override_Pricing_Basis__c and Claim_Line__c.Override_Approved_By__c (required field) to release the hold.

-Given UCR pricing is applied, automatically generate an EOB with out-of-network language- (deprecated: EOB language update scoped to separate payment sprint)

Given Fee_Schedule__c deactivation is complete,
When the Payment__c disbursement engine runs,
Then no Payment__c record is created at in-network contracted rates for claim lines meeting the termination criteria; an audit log entry is written to Payment_Audit_Log__c with Audit_Type__c = 'Terminated Provider Rate Block'.

## Solution

Record-Triggered Flow on Network_Termination_Case__c (After Save, on Status__c = 'Closed – Termination Complete') invokes Apex batch (FeeScheduleDeactivationBatch) to bulk-update related Fee_Schedule__c records. FeeScheduleEngine (custom Apex service class) is updated with a pre-pricing validation step that queries Provider__c.Network_Status__c and Termination_Effective_Date__c before returning rates. Claim_Line__c adjudication hold implemented via Apex Trigger (Before Insert/Update) on Claim_Line__c calling ProviderTerminationValidator.validate(). Override fields governed by a custom Permission Set ('Claims_Override_Specialist') and enforced via Field-Level Security. Payment audit logging via Apex invocable called from the Payment Engine flow.

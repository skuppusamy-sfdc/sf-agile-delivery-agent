## Description

As a Credentialing Specialist, I need a guided OmniScript offboarding workflow that is launched from a Network_Termination_Case__c record when NTC.Status__c transitions to 'Terminated' so that I can systematically revoke provider credentials, update CAQH attestation flags, close linked Credentialing_Case__c records, and generate a termination summary for the compliance audit trail.

## Acceptance Criteria

Given a Network_Termination_Case__c with NTC.Status__c = 'Terminated'
When a Credentialing Specialist clicks the 'Begin Credentialing Offboarding' action button on the NTC record
Then the OmniScript 'OS_Provider_Credentialing_Offboard' launches in a modal with the provider's NPI, TIN, and full name pre-populated from the linked Provider__c record

Given the OmniScript is on Step 2 'Active Credentials Review'
When the DataRaptor lookup executes
Then all Credentialing_Case__c records where CC.Provider__c = NTC.Provider__c AND CC.Status__c NOT IN ('Closed', 'Revoked') are displayed in an editable table with fields: CC.Credential_Type__c, CC.Expiry_Date__c, CC.Issuing_Body__c, CC.Status__c

Given the Credentialing Specialist selects credentials to revoke and clicks 'Confirm Revocation'
When the OmniScript Step 3 integration action fires
Then each selected Credentialing_Case__c is updated: CC.Status__c = 'Revoked', CC.Revocation_Date__c = TODAY(), CC.Revocation_Reason__c = 'Network Termination', CC.Linked_NTC__c = NTC.Id

Given the OmniScript reaches Step 4 'CAQH Notification'
When the integration step calls the CAQH ProView API via Named Credential 'CAQH_API_NC'
Then the provider's CAQH profile is flagged with status 'Inactive' and the API response code and timestamp are stored on Provider__c.CAQH_Last_Update__c and Provider__c.CAQH_Status__c

Given the OmniScript completes all steps
When the user clicks 'Finish'
Then NTC.Credentialing_Offboard_Complete__c = TRUE, a PDF summary is generated via Conga and attached to the NTC, and the Credentialing Specialist receives an in-app notification confirming completion

Given the CAQH API callout returns a non-200 response
When Step 4 executes
Then the OmniScript displays an inline error message and allows the Specialist to skip CAQH update with a mandatory skip reason recorded in NTC.CAQH_Skip_Reason__c

## Solution

OmniScript 'OS_Provider_Credentialing_Offboard' built with 6 steps using OmniStudio Designer. Step 1 uses a DataRaptor Extract (DR_NTC_ProviderHeader) to pre-populate provider context. Step 2 uses a DataRaptor Turbo Extract for Credentialing_Case__c list with an editable repeat block. Step 3 uses a DataRaptor Transform + Integration Procedure (IP_RevokeCredentials) that calls a batch Apex invocable to bulk-update Credentialing_Case__c records. Step 4 uses an Integration Procedure (IP_CAQH_Deactivate) with an HTTP Action element pointing to the CAQH ProView REST endpoint via Named Credential. Step 5 triggers Conga document generation via the same CongaCLMDocGenInvocable Apex action used in INS-0850. Step 6 fires a final DataRaptor Load to stamp NTC completion fields. OmniScript is surfaced as a Quick Action on Network_Termination_Case__c, visible only to Permission Set 'PS_Credentialing_Specialist'.

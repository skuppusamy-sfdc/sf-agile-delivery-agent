## Description

As a Compliance Officer, I need Provider_Change_Notification__c (PCN) records to be automatically generated for all active policyholders who have an ongoing care relationship with a provider exiting the network, so that affected members receive timely written notice of the network change as required under state continuity-of-care regulations.

## Acceptance Criteria

Given a Network_Termination_Case__c where NTC.Status__c = 'Termination Confirmed'
When the PCN generation Apex Batch job runs
Then for each Policy__c record where:
  - Policy.Status__c = 'Active'
  - A Claim__c exists with Claim.Rendering_Provider__c = NTC.Provider__c AND Claim.Date_of_Service__c within the last 12 months
A Provider_Change_Notification__c is created with:
  - PCN.Policy__c = Policy__c.Id
  - PCN.Provider__c = NTC.Provider__c
  - PCN.Notification_Type__c = 'Network Exit'
  - PCN.Effective_Date__c = NTC.Effective_Termination_Date__c
  - PCN.Status__c = 'Pending Send'
  - PCN.Communication_Channel__c = Policy.Preferred_Communication_Method__c

Given a PCN record with PCN.Status__c = 'Pending Send'
When the nightly notification dispatch job runs
Then PCN records are sent via the appropriate channel (Email via OrgWideEmailAddress, or postal flag set for mail fulfillment) and PCN.Status__c = 'Sent', PCN.Sent_Date__c = TODAY

Given a policyholder has multiple active policies all linked to the same terminated provider
When the batch generates PCNs
Then only one PCN is created per Policy__c — duplicate suppression enforced via upsert on external key PCN.Policy_Provider_Key__c (concatenation of Policy__c.Id + Provider__c.Id)

Given PCN.Communication_Channel__c = 'Portal'
When the PCN is dispatched
Then a Bell Notification is created in the Policyholder Experience portal and PCN.Portal_Notification_Id__c is populated

## Solution

Apex Batch class 'BatchPCNGenerator' triggered by Platform Event consumption from Provider_Termination_Event__e. Batch queries Policy__c records via a correlated subquery on Claim__c with date range filter. PCN creation uses Database.upsert on Policy_Provider_Key__c external ID field to prevent duplicates. Nightly dispatch handled by separate Schedulable class 'ScheduledPCNDispatch' which invokes EmailDispatch_Service Apex class for email channel and sets PCN.Postal_Flag__c = TRUE for mail channel (integrated with print fulfillment vendor via outbound message). Portal notifications created via custom LWC notification API call. All PCN activity logged to Compliance_Audit_Log__c with PCN.Id reference. Permission Set 'Compliance_Officer_PS' grants CRUD on Provider_Change_Notification__c.

## Description

As a Provider Relations Specialist, I need a consolidated FlexCard on the Provider__c (Person Account) record page that surfaces the provider's termination status, any in-flight Network_Termination_Case__c records, active Claim__c lines pending adjudication, and outstanding Payment__c disbursements, so that I can make informed decisions during the network exit process without navigating multiple objects.

## Acceptance Criteria

Given a Provider__c record is open in the Lightning UI
When the Provider Termination 360 FlexCard loads
Then it displays the most recent Network_Termination_Case__c linked to the provider, showing fields: Status__c, Termination_Type__c, Requested_Effective_Date__c, and Assigned_Specialist__c

Given the FlexCard is rendered
When there are Claim__c records where Claim__c.Billing_Provider__c = Provider__c.Id AND Claim__c.Status__c IN ('Submitted','Pended','In Adjudication')
Then a child card section lists up to 20 claims showing Claim__c.Claim_Number__c, Claim__c.Total_Billed_Amount__c, Claim__c.LOB__c, and Claim__c.Status__c with a 'View All' deep link

Given the FlexCard is rendered
When Payment__c records exist where Payment__c.Payee_Provider__c = Provider__c.Id AND Payment__c.Status__c IN ('Pending','Scheduled')
Then the total outstanding disbursement amount is displayed as a formatted currency aggregate and each payment row shows Payment__c.Check_Number__c, Payment__c.Amount__c, and Payment__c.Scheduled_Date__c

Given the provider has no active Network_Termination_Case__c
When the FlexCard loads
Then a green status badge labeled 'Active in Network' is displayed and the termination section renders an empty-state message: 'No termination cases on record'

Given the FlexCard is rendered for a user with Profile 'Claims Operations'
When the Network_Termination_Case__c Status__c = 'Pending Approval'
Then an action button 'Approve Termination' is visible and triggers the Termination Approval OmniScript

-Display credentialing score from Credentialing_Case__c on this card- (deprecated: moved to standalone Credentialing FlexCard per UX review)

## Solution

OmniStudio FlexCard built on Provider__c (Account) context. Three child DataRaptors: (1) DR_GetActiveTerminationCase — Turbo Extract on Network_Termination_Case__c filtered by Provider__c lookup; (2) DR_GetPendingClaims — Turbo Extract on Claim__c with aggregate count and list; (3) DR_GetOutstandingPayments — Turbo Extract with SUM aggregate on Payment__c.Amount__c. Conditional visibility rules on FlexCard states drive badge color (green/amber/red) based on Network_Termination_Case__c.Status__c. Action button invokes OmniScript 'TerminationApproval_OS' via FlexCard action. Permission visibility controlled by Permission Set Group 'Provider_Termination_Ops'.

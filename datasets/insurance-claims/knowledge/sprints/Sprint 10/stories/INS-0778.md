# INS-0778: Claim_Line__c in-network repricing must block payment when rendering provider's PNA is in terminated status

## Description

As a Claims Operations Manager, I need the claims adjudication engine to automatically block in-network repricing and flag Claim_Line__c records for manual review when the rendering provider's associated Provider_Network_Agreement__c has a Status__c of 'Terminated' as of the claim's Date_of_Service__c, so that Acme Insurance does not incorrectly apply contracted rates to out-of-network services and avoids regulatory exposure.

## Acceptance Criteria

Given a Claim__c is submitted where Claim.Rendering_Provider__c references a Provider__c whose PNA.Status__c = 'Terminated' and PNA.Termination_Effective_Date__c <= Claim.Date_of_Service__c,
When the adjudication engine processes the claim,
Then all child Claim_Line__c records are set to Claim_Line.Repricing_Status__c = 'Blocked – Provider Terminated' and Claim_Line.Network_Status__c = 'Out-of-Network'.

Given Claim_Line.Repricing_Status__c = 'Blocked – Provider Terminated',
When the PaymentEngine evaluates the claim,
Then no Payment__c record is generated and Claim.Status__c is set to 'Pended – Provider Verification'.

Given a Claim__c is pended for provider verification,
When the record is saved,
Then a Case is auto-created of RecordType 'Provider Termination Review' assigned to the Provider Network Specialist queue with a due date of today + 5 business days.

Given a Provider Network Specialist overrides the block by setting Claim.Override_Reason__c and Claim.Override_Approved_By__c,
When the override is saved,
Then Claim_Line.Repricing_Status__c transitions to 'Override – Manual Approval' and an audit log entry is written to the Claim_Audit__c object with Override_Reason__c and timestamp.

Given a claim where the PNA was terminated AFTER the Date_of_Service__c (retroactive termination scenario),
When adjudication runs,
Then Claim_Line.Network_Status__c remains 'In-Network' and a Claim_Flag__c record is created with Flag_Type__c = 'Retroactive Termination – Review Recommended'.

## Solution

Implement an Apex invocable method 'ProviderTerminationCheck' called from the Claims Adjudication Record-Triggered Flow (after-save, async path). The method queries Provider_Network_Agreement__c filtered by Provider__c = Claim.Rendering_Provider__c and checks Termination_Effective_Date__c against Date_of_Service__c. Use a Custom Metadata type 'Termination_Block_Config__mdt' to externalize the blocking rules per LOB. Write bulk-safe DML for Claim_Line__c updates using a Unit of Work pattern. Integrate with the FeeScheduleEngine component via a Platform Event 'Repricing_Block_Event__e' to halt the fee schedule lookup asynchronously. Override audit trail written via an Apex Trigger on Claim__c (after update) to Claim_Audit__c.

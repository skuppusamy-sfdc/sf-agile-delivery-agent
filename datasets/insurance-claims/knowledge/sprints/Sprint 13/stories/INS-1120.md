# INS-1120: Bug: Network_Termination_Case__c does not trigger PCN generation when termination type is 'Voluntary'

## Description

**Bug Report**

**Summary:** When a Network_Termination_Case__c record is created or updated with Network_Termination_Case__c.Termination_Type__c = 'Voluntary', the Record-Triggered Flow 'Generate_PCN_On_Termination' does not fire and no Provider_Change_Notification__c (PCN) record is created. Involuntary terminations correctly trigger PCN generation. This is a compliance gap — voluntary terminations still require PCN issuance within 30 days per state regulatory requirements.

**Steps to Reproduce:**
1. Create a new Network_Termination_Case__c record.
2. Set Network_Termination_Case__c.Provider__c to any active provider.
3. Set Network_Termination_Case__c.Termination_Type__c = 'Voluntary'.
4. Set Network_Termination_Case__c.Termination_Effective_Date__c = 30 days from today.
5. Save the record.
6. Observe: No Provider_Change_Notification__c record is created. Flow history shows 'Generate_PCN_On_Termination' did not execute.

**Root Cause (suspected):** The Flow entry condition on 'Generate_PCN_On_Termination' includes a filter: Network_Termination_Case__c.Termination_Type__c = 'Involuntary'. 'Voluntary' was not included in the picklist condition, likely an oversight during the Sprint 11 flow build.

**Expected Behavior:** A Provider_Change_Notification__c record is created for both 'Voluntary' and 'Involuntary' termination types.

**Actual Behavior:** PCN is only created for 'Involuntary' terminations.

**Compliance Risk:** HIGH — state regulators require PCN issuance for all termination types within 30 days.

## Acceptance Criteria

As per Description.

Given Network_Termination_Case__c.Termination_Type__c = 'Voluntary',
When the record is saved,
Then a Provider_Change_Notification__c must be created with PCN.Network_Termination_Case__c lookup, PCN.Provider__c, PCN.Notification_Type__c = 'Termination', PCN.Required_By_Date__c = Network_Termination_Case__c.Created_Date + 30 days, and PCN.Status__c = 'Pending'.

Given the fix is deployed,
When regression tests run for Involuntary terminations,
Then existing PCN generation behavior must remain unchanged.

## Solution

Update the Record-Triggered Flow 'Generate_PCN_On_Termination' entry condition to use an OR filter: Termination_Type__c = 'Involuntary' OR Termination_Type__c = 'Voluntary' (alternatively change to IS NOT NULL to future-proof against additional picklist values). Add a Flow variable to conditionally set PCN.Notification_Priority__c = 'Standard' for Voluntary vs 'High' for Involuntary. Update Flow unit test scenarios in the sandbox to cover both picklist values. Perform a one-time data fix Apex script to backfill PCN records for any Voluntary terminations created since Sprint 11 deployment that are missing PCNs. Compliance Officer sign-off required before production deployment.

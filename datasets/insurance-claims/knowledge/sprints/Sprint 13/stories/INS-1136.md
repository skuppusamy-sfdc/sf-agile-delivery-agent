## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard that surfaces real-time credentialing status, active PNA agreements, facility affiliations, and open PCNs so that I can resolve provider inquiries without navigating across five separate record pages.

## Acceptance Criteria

Given a Provider__c (Person Account) record is opened by a user with the 'Provider_Relations' Permission Set Group,
When the Provider 360 FlexCard loads,
Then the following data panels must render within 3 seconds:
  - Provider NPI, TIN, and specialty pulled from Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Primary_Specialty__c
  - Active credentialing status from the most recent Credentialing_Case__c where Credentialing_Case__c.Status__c = 'Active' linked via Provider__c lookup
  - Count of in-force Provider_Network_Agreement__c records where PNA__c.Effective_Status__c = 'Active'
  - Count of open Provider_Change_Notification__c records where PCN__c.Resolution_Status__c IN ('New','In Review')

Given the provider has an associated Network_Termination_Case__c with Status__c = 'Pending',
When the FlexCard renders,
Then a red warning banner reading 'Pending Termination — Review Required' must appear at the top of the card with a deep-link to the Network_Termination_Case__c record.

Given the provider's CAQH profile was last attested more than 365 days ago (Provider__c.CAQH_Last_Attestation_Date__c < TODAY()-365),
When the credentialing panel loads,
Then display a yellow stale-data indicator and the label 'CAQH Re-attestation Overdue'.

-Given a 'Quick Re-credential' action button on the FlexCard, When clicked, Then launch the legacy Visualforce re-credentialing wizard — DEPRECATED: replaced by OmniScript in INS-1098-

Given the FlexCard is viewed on a mobile form factor,
When the viewport is less than 768px,
Then all panels must collapse into an accordion layout and no horizontal scroll must occur.

## Solution

Implement OmniStudio FlexCard 'Provider_360_Card' with five child DataRaptors: DR_GetProviderCore (reads Provider__c + Account fields), DR_GetActiveCredentialing (SOQL on Credentialing_Case__c filtered by Status__c), DR_GetPNACount (aggregate SOQL on Provider_Network_Agreement__c), DR_GetOpenPCNs (SOQL on Provider_Change_Notification__c), and DR_GetTerminationStatus (SOQL on Network_Termination_Case__c). Use a Conditional State on the FlexCard to toggle the red termination banner. CAQH date comparison handled via a DataRaptor Transform formula. Deploy via OmniStudio package version; assign to Provider_Relations Permission Set Group. Responsive layout configured using OmniStudio FlexCard responsive breakpoints.

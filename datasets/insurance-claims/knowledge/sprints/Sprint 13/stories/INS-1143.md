## Description

**Bug Report — INS-1143**

**Summary:** When a Network_Termination_Case__c record is closed with a resolution of 'Rescinded' (i.e., the provider's termination was reversed), the associated Provider_Network_Agreement__c records are not having their PNA__c.Effective_Status__c field recalculated. The PNA records remain in 'Terminating' status indefinitely, causing the Provider 360 FlexCard (INS-1136) to continue displaying the termination warning banner and preventing new Claim__c records from passing provider eligibility validation.

**Steps to Reproduce:**
1. Open a Provider__c record with at least one PNA where PNA__c.Effective_Status__c = 'Terminating'.
2. Navigate to the linked Network_Termination_Case__c record.
3. Set Network_Termination_Case__c.Status__c = 'Closed' and Network_Termination_Case__c.Resolution__c = 'Rescinded'.
4. Save the record.
5. Navigate back to the Provider__c record and inspect all related PNA__c.Effective_Status__c values.
6. Submit a new Claim__c for this provider and observe the eligibility check result.

**Expected:** All PNA__c records linked to the provider via PNA__c.Provider__c should have Effective_Status__c reset to 'Active' and Claim__c eligibility validation should pass.

**Actual:** PNA__c.Effective_Status__c remains 'Terminating'. New Claim__c records fail eligibility with error 'Provider Not Active in Network' and are incorrectly pended.

## Acceptance Criteria

As per Description. Additionally: when Network_Termination_Case__c.Resolution__c = 'Rescinded' and Status__c = 'Closed', a Record-Triggered Flow or Apex Trigger must query all PNA__c records where PNA__c.Provider__c = Network_Termination_Case__c.Provider__c AND PNA__c.Effective_Status__c = 'Terminating' and bulk-update them to PNA__c.Effective_Status__c = 'Active'. A Provider_Change_Notification__c record must be auto-created with PCN__c.Change_Type__c = 'Termination Rescinded' for compliance audit purposes.

## Solution

Add an Apex Trigger 'NetworkTerminationCaseTrigger' (after update) on Network_Termination_Case__c. In the trigger handler class 'NetworkTerminationCaseHandler', check for records transitioning to Status__c = 'Closed' with Resolution__c = 'Rescinded'. Perform a bulk SOQL query on Provider_Network_Agreement__c filtered by Provider__c lookup and Effective_Status__c = 'Terminating'. Bulk update PNA records to 'Active' using a List DML. Insert Provider_Change_Notification__c records in the same transaction. Add negative test case to existing 'NetworkTerminationCaseHandlerTest' class to cover rescission path. Verify fix resolves FlexCard termination banner behavior described in INS-1136.

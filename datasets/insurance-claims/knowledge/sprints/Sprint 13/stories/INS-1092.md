## Description

As a Provider Network Specialist, I need Salesforce to automatically create a Provider_Change_Notification__c (PCN) record and initiate a review workflow when NPPES data sync detects a change to a provider's NPI__c or TIN__c on the Provider__c record, so that our network agreements and credentialing records remain accurate and compliant with CMS data integrity requirements. Extends INS-1071 (NPPES nightly batch integration).

## Acceptance Criteria

Given the NPPES nightly batch job runs and returns updated provider data
When the inbound NPI or TIN value differs from Provider__c.NPI__c or Provider__c.TIN__c
Then a Provider_Change_Notification__c record is created with PCN_Type__c = 'NPI Change' or 'TIN Change', PCN_Status__c = 'Pending Review', and linked to the Provider__c via Provider__c lookup

Given a PCN record is created with PCN_Type__c = 'TIN Change'
When PCN_Status__c = 'Pending Review'
Then the record is assigned to the 'Provider Data Integrity' queue and a Task is created for the assigned Provider Network Specialist with Due_Date__c = TODAY() + 3

Given a PCN record is created with PCN_Type__c = 'NPI Change'
When the provider has one or more active Provider_Network_Agreement__c records
Then each active PNA record's PNA_Review_Required__c field is set to TRUE and a chatter post is added: 'NPI change detected — PNA review required before next claims cycle'

Given the PCN review is completed by the specialist
When PCN_Status__c is updated to 'Approved'
Then Provider__c.NPI__c or Provider__c.TIN__c is updated to the new value, PCN_Resolved_Date__c = TODAY(), and an audit entry is written to Provider_Audit_Log__c

Given the PCN review is completed and PCN_Status__c = 'Rejected'
When the rejection is saved
Then the NPPES inbound staging record is flagged as 'Disputed' and no changes are made to the Provider__c record

Given a TIN change is approved
When active Claim__c records exist with Claim_Status__c IN ('Pending Payment', 'In Adjudication') linked to the provider
Then those claims are placed in a 'TIN Review Hold' status and a notification is sent to the Claims Operations Manager

## Solution

Extend the NPPES batch integration class (NPPESDataSyncBatch.cls from INS-1071) to compare inbound NPI/TIN values against existing Provider__c fields. On detected change, invoke an Apex method 'PCNCreationService.createPCN()' to instantiate Provider_Change_Notification__c. A Record-Triggered Flow on PCN__c (after insert) handles queue assignment and Task creation based on PCN_Type__c. Approval Process 'PCN_TIN_NPI_Review' manages the Pending → Approved/Rejected lifecycle. On approval, a Flow invokes Apex to update Provider__c fields and write to Provider_Audit_Log__c. Claims hold logic implemented via a separate Apex trigger on Provider_Change_Notification__c after update.

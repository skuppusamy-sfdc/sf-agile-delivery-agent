## Description

As a Credentialing Specialist, I need all active and pending provider credentialing records to be migrated from the legacy CredStar system into Salesforce Credentialing_Case__c objects so that the credentialing team operates exclusively in Salesforce by Sprint 14 cutover, eliminating dual-system maintenance.

## Acceptance Criteria

Given the CredStar data extract CSV is loaded into the Salesforce Data Migration staging object 'Migration_Staging__c',
When the Apex batch class 'CredStarMigrationBatch' executes,
Then each row is transformed and inserted as a Credentialing_Case__c with the following field mappings: CredStar.ProviderNPI → Provider__c lookup resolved via NPI__c, CredStar.CaseStatus → Credentialing_Case__c.Status__c using value map in 'CredStar_Status_Map__mdt', CredStar.ExpiryDate → Credentialing_Case__c.Credential_Expiry_Date__c.

Given a CredStar record contains a ProviderNPI that does not match any existing Provider__c.NPI__c,
When the batch processes that row,
Then the row is skipped and logged to 'Migration_Error_Log__c' with Migration_Error_Log__c.Error_Type__c = 'Provider Not Found' and Migration_Error_Log__c.Source_Record_ID__c = CredStar record ID.

Given migration completes,
When a reconciliation report is generated,
Then the total count of Credentialing_Case__c records created must equal CredStar source record count minus logged exceptions, with a tolerance of 0 unaccounted records.

Given a migrated Credentialing_Case__c has Credential_Expiry_Date__c within 90 days of migration run date,
When the record is created,
Then Credentialing_Case__c.Status__c = 'Expiring Soon' and a Task is auto-created for the assigned Credentialing Specialist queue.

Given the migration batch runs in production,
When execution completes,
Then a Compliance Officer receives an email summary report listing total migrated, total exceptions, and any records with Status__c = 'Lapsed' for immediate review.

-Given CAQH profile data is co-migrated alongside CredStar records in the same batch run- (deprecated — CAQH sync deferred to INS-1161 due to CAQH API rate limits)

## Solution

Implement 'CredStarMigrationBatch' as an Apex Database.Batchable class with batch size 200. Use custom metadata type 'CredStar_Status_Map__mdt' for status value translation. NPI-to-Provider__c lookup resolved via SOQL map pre-built before batch execution to avoid per-row queries. Errors written to 'Migration_Error_Log__c' using a separate Database.SaveResult handler. Post-migration reconciliation report generated by a Scheduled Apex job 'MigrationReconciliationJob' that queries both Migration_Staging__c and Credentialing_Case__c counts. Deploy via Change Set in a maintenance window; full rollback script prepared using Migration_Staging__c as source of truth.

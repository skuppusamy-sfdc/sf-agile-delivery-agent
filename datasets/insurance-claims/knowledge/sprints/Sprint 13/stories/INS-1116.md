## Description

**Bug Report**

**Summary:** The Fee Schedule lookup Apex class returns a null allowed amount when a Claim_Line__c CPT code includes a modifier suffix (e.g., '99213-25', '27447-RT'), causing Claim_Line__c.Allowed_Amount__c to remain blank and blocking adjudication.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record in the Claims console.
2. Add a new Claim_Line__c with Claim_Line__c.CPT_Code__c = '99213-25' and Claim_Line__c.Service_Date__c = today.
3. Save the Claim_Line__c record.
4. Observe that the Record-Triggered Flow 'Populate_Allowed_Amount' fires but Claim_Line__c.Allowed_Amount__c remains null.
5. Check debug logs — FeeScheduleLookupService.getRate() returns null because the SOQL query matches on exact CPT string '99213-25' against Fee_Schedule_Line__c.CPT_Code__c which stores only base code '99213'.

**Expected Behavior:** The fee schedule lookup should strip the modifier suffix and match on the base 5-character CPT code '99213', returning the correct allowed amount from Fee_Schedule_Line__c.

**Actual Behavior:** Null is returned; Claim_Line__c.Allowed_Amount__c is not populated; adjudication flow throws a validation error 'Allowed Amount required for adjudication'.

**Environment:** Production (also reproducible in Full Sandbox)
**Frequency:** 100% reproducible with any modifier-appended CPT code
**Impact:** ~340 claims per day blocked in adjudication queue

## Acceptance Criteria

As per Description.

Additional verification:
Given Claim_Line__c.CPT_Code__c = '99213-25',
When FeeScheduleLookupService.getRate() is invoked,
Then it must parse the base code as LEFT(CPT_Code__c, 5) = '99213' and return Fee_Schedule_Line__c.Allowed_Amount__c for that base code.

Given Claim_Line__c.CPT_Code__c = '27447-RT',
When the rate lookup runs,
Then Claim_Line__c.Modifier__c must be populated with 'RT' and Claim_Line__c.Allowed_Amount__c must reflect the base '27447' rate.

## Solution

Fix FeeScheduleLookupService.getRate() Apex method to use String.left(5) to extract the base CPT code before executing the SOQL query against Fee_Schedule_Line__c. Populate a new Claim_Line__c.Modifier__c field with the suffix characters (positions 6+) when present. Update the Record-Triggered Flow 'Populate_Allowed_Amount' to pass the parsed base code. Add CPT modifier test cases to FeeScheduleLookupServiceTest covering -25, -RT, -LT, -59 modifiers. Deploy hotfix to sandbox for QA validation before production push.

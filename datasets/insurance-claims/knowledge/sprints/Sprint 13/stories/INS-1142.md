## Description

As a Broker Relations Coordinator, I need a guided OmniScript onboarding wizard for new group/employer accounts that collects plan selection, enrollment census data, and billing preferences, then auto-generates a Group_Service_Agreement__c record and routes the GSA for e-signature via DocuSign so that employer group onboarding time is reduced from 10 business days to under 3.

## Acceptance Criteria

Given a Broker Relations Coordinator initiates the 'New Group Onboarding' OmniScript from an Account record where Account.RecordType = 'Employer Group',
When the OmniScript Step 1 (Group Details) is completed,
Then the following fields must be captured and mapped: Account.Name, Account.TIN__c, Account.SIC_Code__c, Account.Enrollment_Effective_Date__c, and Account.Billing_Frequency__c.

Given Step 2 (Plan Selection) is completed with at least one LOB selected,
When the coordinator proceeds to Step 3,
Then a Group_Service_Agreement__c record must be created in 'Draft' status (GSA__c.Status__c = 'Draft') with GSA__c.Effective_Date__c = Account.Enrollment_Effective_Date__c and GSA__c.LOB__c populated from selections.

Given the GSA__c record is in 'Draft' status and all required fields are populated,
When the coordinator clicks 'Send for Signature' in Step 4,
Then the OmniScript must invoke the DocuSign Integration Procedure to generate a pre-populated GSA PDF and send an envelope to the employer's signatory contact (Contact.Role__c = 'Authorized Signer').

Given the DocuSign envelope is sent,
When the employer countersigns,
Then a DocuSign status callback must update GSA__c.Status__c = 'Executed', GSA__c.Executed_Date__c = signature timestamp, and GSA__c.DocuSign_Envelope_ID__c must be populated.

Given the OmniScript is abandoned mid-flow (browser closed or navigated away),
When the coordinator re-opens the OmniScript from the same Account,
Then a resume prompt must appear and previously entered data must be pre-populated from a saved OmniScript session state stored in OmniProcess__c.

## Solution

Develop a 5-step OmniScript 'Group_Employer_Onboarding_OS' with DataRaptor Loads for Account field updates and GSA__c creation. Step 4 triggers an Integration Procedure 'IP_DocuSign_GSA_Send' that calls the DocuSign REST API via a Named Credential, passing a Conga-generated PDF as the envelope document. DocuSign webhook callback handled by an Apex REST resource 'DocuSignCallbackController' that updates GSA__c fields. OmniScript session persistence uses OmniProcess__c with a lookup to the Account. Deploy OmniScript with Permission Set 'Broker_Relations_PS' granting access. Unit tested with mock DocuSign API responses in sandbox.

## Description

As a Credentialing Specialist, I need historical credentialing records from CAQH ProView to be bulk-loaded into Salesforce Credentialing_Case__c records and then kept in sync via a nightly delta pull so that the legacy credentialing system can be decommissioned at the end of Phase 2 without data loss. The initial load covers approximately 18,000 provider credentialing records.

## Acceptance Criteria

Given the CAQH bulk extract CSV (caqh_export_20241001.csv) is staged in the Salesforce Files or an S3 bucket
When the Data Migration batch job 'CAQH_BulkLoad_Batch' is executed
Then each row produces one Credentialing_Case__c record with Credentialing_Case__c.CAQH_Provider_ID__c, Credentialing_Case__c.NPI__c, Credentialing_Case__c.Credentialing_Status__c, Credentialing_Case__c.Initial_Credentialing_Date__c, and Credentialing_Case__c.Expiration_Date__c populated from mapped CSV columns

Given a CAQH record's NPI matches an existing Provider__c.NPI__c
When the batch processes that row
Then Credentialing_Case__c.Provider__c is set to the matching Provider__c.Id; if no match exists, the row is written to a Migration_Error_Log__c record with Error_Type__c = 'Unmatched NPI'

Given the initial bulk load is complete
When the nightly delta sync scheduled job runs at 02:00 UTC
Then only CAQH records with a CAQH last-modified timestamp after the value stored in Migration_Config__mdt.CAQH_Last_Sync_Timestamp__c are retrieved via the CAQH REST API and upserted into Credentialing_Case__c using Credentialing_Case__c.CAQH_Provider_ID__c as the external ID

Given the delta sync completes
When the job finishes
Then Migration_Config__mdt.CAQH_Last_Sync_Timestamp__c is updated to the job start time and a summary email is sent to the Credentialing ops DL with counts of inserted, updated, and errored records

Given a Credentialing_Case__c.Expiration_Date__c is within 90 days of TODAY()
When the post-load validation step runs
Then Credentialing_Case__c.Renewal_Alert_Sent__c = false records trigger a batch notification to the assigned Credentialing_Specialist__c user

## Solution

Develop 'CAQH_BulkLoad_Batch' as a Database.Batchable and Database.Stateful Apex class processing 200 records per chunk. Use a pre-processing step with DataLoader or Bulk API 2.0 for the initial 18K record load (to stay within governor limits), with the Apex batch handling transformation and NPI matching logic. For the delta sync, implement a Schedulable Apex class ('CAQH_DeltaSync_Scheduler') that calls the CAQH REST API via a Named Credential ('CAQH_ProView_NC') with OAuth 2.0. Upsert via Database.upsert() using CAQH_Provider_ID__c as the external ID field. Store sync metadata in a Custom Metadata Type (Migration_Config__mdt) so timestamps can be updated without a deployment. Log all errors to a custom Migration_Error_Log__c object queryable by the data migration team.

## Description

As a Broker Relations Coordinator, I need a guided OmniScript to onboard new group employer accounts, collect Group_Service_Agreement__c details, and trigger a DocuSign envelope for GSA e-signature so that group onboarding cycle time is reduced from 10 business days to under 3 business days.

## Acceptance Criteria

Given a Broker Relations Coordinator initiates the 'Group Onboarding' OmniScript from an Account record with Account.RecordType = 'Employer_Group',
When Step 1 (Group Details) is completed,
Then the OmniScript must validate that Account.TIN__c is a 9-digit numeric value and Account.Effective_Date__c is not in the past before allowing progression.

Given Step 2 (Plan Selection) is reached,
When the coordinator selects one or more LOBs,
Then a GSA record must be staged in memory with Group_Service_Agreement__c.LOB__c, GSA.Group_Size__c, GSA.Benefit_Start_Date__c, and GSA.Renewal_Date__c populated from OmniScript inputs.

Given all OmniScript steps are completed and the coordinator clicks 'Submit',
When the DataRaptor Transform action executes,
Then a Group_Service_Agreement__c record must be created with GSA.Status__c = 'Pending Signature' and linked to the employer Account.

Given the GSA record is created with Status__c = 'Pending Signature',
When the DocuSign integration action fires,
Then a DocuSign envelope must be sent to the contacts listed in GSA.Signatory_Contact__c with the pre-mapped GSA template ID configured in Custom Metadata 'DocuSign_Template_Mapping__mdt'.

Given the DocuSign envelope is signed by all required parties,
When the DocuSign callback webhook is received,
Then GSA.Status__c must update to 'Executed', GSA.Executed_Date__c must be set to the signature timestamp, and a welcome task must be created for the assigned Broker Relations Coordinator.

-Given the OmniScript Step 3 collects COBRA election details-
-When the coordinator enters COBRA participant counts-
-Then COBRA_Plan__c records must be created-

## Solution

Build a 5-step OmniScript 'Group_Employer_Onboarding_OS' using OmniStudio. Use DataRaptor Load actions to create Account and Group_Service_Agreement__c records. Configure DocuSign integration action using the DocuSign for Salesforce managed package with envelope template mapping stored in DocuSign_Template_Mapping__mdt custom metadata. A Record-Triggered Flow on Group_Service_Agreement__c listens for Status__c change to 'Executed' and creates the follow-up task. Deploy via OmniStudio Deployment Package and restrict access via Permission Set Group 'Broker_Relations_PSG'.

## Description

As a Broker Relations Coordinator, I need a guided OmniScript that collects group employer information, generates a Group_Service_Agreement__c record, and triggers a DocuSign envelope for GSA eSignature so that employer onboarding is completed digitally without manual contract preparation. Extends INS-1143 (Broker Portal foundation).

## Acceptance Criteria

Given a Broker Relations Coordinator launches the 'New Group Onboarding' OmniScript from the Account record,
When Step 1 (Employer Details) is submitted,
Then the following fields are validated as required before progression:
  - Account.Name, Account.BillingState, Account.TIN__c (format: XX-XXXXXXX)
  - Group_Service_Agreement__c.Effective_Date__c must be >= today
  - Group_Service_Agreement__c.LOB__c must be a valid picklist value

Given all steps are completed and the coordinator clicks 'Generate GSA',
When the OmniScript action fires,
Then a Group_Service_Agreement__c record is created with Status__c = 'Pending Signature' and a DocuSign envelope is sent to the primary contact email within 2 minutes.

Given the DocuSign envelope is signed by all required signatories,
When the DocuSign status callback is received,
Then Group_Service_Agreement__c.Status__c updates to 'Executed', GSA__c.Signed_Date__c is populated, and the signed PDF is stored as a Salesforce File (ContentDocument) linked to the GSA record.

-Given the coordinator can manually upload a pre-signed PDF to bypass DocuSign, Then GSA Status__c = 'Executed' upon upload- (deprecated — manual bypass removed per compliance review; all GSAs must use DocuSign)

Given the DocuSign envelope expires without signature (after 14 days),
When expiration webhook fires,
Then Group_Service_Agreement__c.Status__c = 'Signature Expired' and a Task is created assigned to the originating coordinator with ActivityDate = today + 1.

## Solution

OmniScript 'Group_Onboarding_OS' built with 6 steps: Employer Details, Plan Selection, Billing Configuration, Contact Roles, GSA Terms Review, and Confirmation. DataRaptor Post action creates Account and GSA records in a single transaction. DocuSign integration via the DocuSign for Salesforce managed package; envelope template ID stored in Custom Metadata (DocuSign_Template__mdt). A Record-Triggered Flow on Group_Service_Agreement__c monitors DocuSign_Status__c field changes (populated by DocuSign package callbacks) and executes the status transition logic. Signed PDF attachment handled by DocuSign package's native ContentDocument creation. Permission Set 'Broker_Relations_PS' grants OmniScript launch access.

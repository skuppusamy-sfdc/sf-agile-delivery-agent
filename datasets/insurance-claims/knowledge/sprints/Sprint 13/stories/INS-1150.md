## Description

As a Provider Network Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) letter and initiate a 90-day notice countdown when a Network_Termination_Case__c is submitted so that we meet state-mandated advance notice requirements and maintain an auditable record of all termination communications without manual document preparation.

## Acceptance Criteria

Given a Network_Termination_Case__c record is created with Termination_Case__c.Status__c = 'Submitted' and Termination_Case__c.Termination_Type__c = 'Involuntary'
When the record is saved
Then a Provider_Change_Notification__c record is automatically created with PCN__c.Notification_Type__c = 'Termination Notice', PCN__c.Provider__c = Termination_Case__c.Provider__c, PCN__c.Notice_Date__c = TODAY(), and PCN__c.Effective_Termination_Date__c = TODAY() + 90

Given the PCN record is created
When Conga CLM generates the PCN letter
Then the letter is merged using template 'PCN_Involuntary_Termination_v2' and attached as a ContentVersion to both the PCN__c and Network_Termination_Case__c records with ContentVersion.Title = 'PCN Letter - [Provider Name] - [Notice Date]'

Given the PCN letter is generated
When the letter is ready for delivery
Then an automated certified mail task (Task.Subject = 'Send PCN via Certified Mail') is created and assigned to the Provider Network Specialist queue with a due date of TODAY() + 2 business days

Given PCN__c.Effective_Termination_Date__c is 30 days away
When the nightly scheduled Flow runs
Then a reminder Task is created for the assigned Provider Network Specialist with Subject = '30-Day PCN Termination Reminder' and the linked Network_Termination_Case__c.Status__c is updated to 'Termination Pending'

Given the Network_Termination_Case__c.Status__c is updated to 'Terminated'
When the termination is finalized
Then all active PNA__c records for the provider where PNA__c.Status__c = 'Active' are updated to PNA__c.Status__c = 'Terminated' and PNA__c.Termination_Date__c = PCN__c.Effective_Termination_Date__c

-Given the PCN letter is sent via email directly from Salesforce- (deprecated — legal requires certified physical mail per state regulation SR-2024-NV-14; email delivery removed)

Given the termination affects a provider with open Claim__c records in 'In Adjudication' status
When the PNA records are terminated
Then a warning notification is sent to the Claims Operations Manager queue listing all affected Claim__c.Id values with Claim__c.Provider__c matching the terminated provider

## Solution

Implement a Record-Triggered Flow on Network_Termination_Case__c (after insert, Status__c = 'Submitted', Termination_Type__c = 'Involuntary') that creates the PCN__c record and invokes a Conga CLM Apex invocable action to generate the merged letter. Use a Scheduled Flow running nightly at 06:00 UTC to query PCN__c records where Effective_Termination_Date__c = TODAY() + 30 and create reminder Tasks. Build a second Record-Triggered Flow on Network_Termination_Case__c (after update, Status__c changes to 'Terminated') that bulk-updates related PNA__c records via an Apex invocable batch action to avoid hitting DML limits on large provider networks. Generate the open-claims warning notification via a custom Apex class querying Claim__c and sending a Chatter post to the Claims Operations Manager group.

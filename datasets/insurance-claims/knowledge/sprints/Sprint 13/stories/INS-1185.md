## Description

As a Claims Operations Manager, I need the claims adjudication engine to detect dual-coverage scenarios on Policy__c records and apply COB (Coordination of Benefits) rules — including Birthday Rule and Gender Rule for dependent coverage — so that Acme Insurance correctly calculates primary vs. secondary payer liability and avoids overpayment on Claim__c records.

## Acceptance Criteria

Given a Claim__c is submitted where Policy__c.COB_Indicator__c = true,
When the adjudication process initiates,
Then the system must query all active Policy__c records linked to the same Policyholder via Policy__c.Policyholder__c and identify primary vs. secondary payer using COB_Rule__mdt custom metadata rules.

Given a dependent covered under two policies where both policies have Policy__c.Coverage_Type__c = 'Dependent',
When the Birthday Rule is evaluated,
Then the policy whose subscriber's birth month/day is earlier in the calendar year must be assigned Claim__c.Primary_Payer__c and the other assigned Claim__c.Secondary_Payer__c.

Given COB determination is complete,
When Claim__c.Primary_Payer_Liability__c and Claim__c.Secondary_Payer_Liability__c are calculated,
Then Primary_Payer_Liability__c + Secondary_Payer_Liability__c must not exceed Claim__c.Total_Billed_Amount__c and the delta must populate Claim__c.Member_Responsibility__c.

Given a Claim__c with COB applied,
When the EOB is generated,
Then Explanation_of_Benefits__c.COB_Applied__c must be set to true, and Explanation_of_Benefits__c.Primary_Payer_Paid__c and Explanation_of_Benefits__c.Secondary_Payer_Paid__c must reflect the split.

-Given COB results in zero member liability, suppress member billing notification- (deprecated — finance confirmed member billing notification always required per INS-1161 compliance ruling)

Given a COB calculation that results in secondary payer liability > $0,
When the claim is finalized,
Then a COB_Coordination_Record__c must be created and linked to both Policy__c records for audit trail.

## Solution

Apex invocable method 'COBEvaluator.evaluate()' called from Claim Adjudication Flow. Birthday Rule and Gender Rule logic encapsulated in COBRuleEngine Apex class with rules driven by COB_Rule__mdt Custom Metadata. Primary/secondary liability calculated using Fee_Schedule__c allowed amounts. EOB generation handled by existing EobGeneratorService class (extended). COB_Coordination_Record__c created via Record-Triggered Flow on Claim__c when Claim__c.COB_Indicator__c = true and Status__c = 'Adjudicated'. Sharing rules ensure Compliance Officer role has read access to COB_Coordination_Record__c.

## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record so that I can view real-time credentialing status, active PNA agreements, network participation, and open Credentialing_Case__c records without navigating across multiple objects.

## Acceptance Criteria

Given a Provider__c record is open in Salesforce,
When the Provider 360 FlexCard loads,
Then the card displays the provider's NPI, TIN, specialty, and current Credentialing_Case__c.Status__c in a summary header band.

Given a provider has one or more active Provider_Network_Agreement__c records,
When the PNA section of the FlexCard renders,
Then all PNAs where PNA__c.Status__c = 'Active' are listed with effective date, termination date, and associated Fee_Schedule__c.Name.

Given a provider has a pending Network_Termination_Case__c,
When the FlexCard loads,
Then a high-visibility warning banner displays showing Network_Termination_Case__c.Termination_Effective_Date__c and reason code.

Given the user holds the 'Provider Relations' Permission Set Group,
When they click 'Initiate Credentialing' on the FlexCard,
Then an OmniScript launches pre-populated with Provider__c.Id and CAQH_Profile_ID__c.

-Given the FlexCard displays claim volume metrics sourced from Claim__c rollup fields- (deprecated — moved to separate Analytics story INS-1147)

Given the provider's CAQH attestation expiry date is within 30 days,
When the FlexCard header renders,
Then an amber indicator displays with the label 'CAQH Attestation Expiring: [date]'.

## Solution

Build an OmniStudio FlexCard ('Provider360Card') with child DataRaptors pulling from Provider__c, Credentialing_Case__c, Provider_Network_Agreement__c, and Network_Termination_Case__c via SOQL DataRaptor Extract actions. Use conditional visibility states for the termination warning banner driven by a formula checking Network_Termination_Case__c existence. Embed an OmniScript launch action ('InitiateCredentialing_OS') wired to the 'Initiate Credentialing' button. Deploy FlexCard via Permission Set Group 'Provider_Relations_PSG'. Cache DataRaptor responses with a 60-second TTL to reduce SOQL consumption on high-traffic queues.

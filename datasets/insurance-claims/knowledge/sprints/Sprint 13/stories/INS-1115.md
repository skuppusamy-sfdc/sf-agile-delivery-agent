## Description

As a Claims Operations Manager, I need COB logic enforced during Claim__c adjudication so that when a claimant holds coverage under multiple policies, primary and secondary payer responsibilities are correctly calculated and Claim_Line__c amounts are adjusted accordingly, preventing overpayment. Extends INS-1089.

## Acceptance Criteria

Given a Claim__c record has Claim__c.COB_Indicator__c = true,
When the adjudication flow executes,
Then the flow must retrieve all active Policy__c records linked to the claimant via the Policyholder__c junction and rank them by Policy__c.COB_Priority__c (1 = Primary, 2 = Secondary).

Given the primary Policy__c has been identified,
When Claim_Line__c adjudication runs,
Then Claim_Line__c.Primary_Payer_Paid__c must be populated before any secondary calculation proceeds, and Claim_Line__c.Patient_Responsibility__c must reflect the remainder.

Given a secondary payer policy exists,
When secondary adjudication runs,
Then Claim_Line__c.Secondary_Payer_Paid__c must not exceed Claim_Line__c.Billed_Amount__c minus Claim_Line__c.Primary_Payer_Paid__c.

Given COB adjudication is complete,
When the Claim__c record is saved,
Then Claim__c.COB_Savings__c must be calculated as the difference between total billed and total paid across all payers, and Claim__c.Status__c set to 'Adjudicated'.

-Given the claimant is Medicare primary, apply Medicare as secondary crossover logic per legacy rules- (deprecated: Medicare crossover handled by dedicated CMS integration in INS-1101)

## Solution

Extended the existing Claim Adjudication OmniScript with a new COB Decision step using a DataRaptor Transform to rank Policy__c records by COB_Priority__c. Apex invocable method 'COBAdjudicationService.calculateCOB()' performs line-level calculations and writes results back to Claim_Line__c fields. Record-Triggered Flow on Claim__c (after update) computes COB_Savings__c via formula and stamps the adjudication timestamp. Unit tests cover primary-only, primary+secondary, and COB tie-breaker scenarios at 92% coverage.

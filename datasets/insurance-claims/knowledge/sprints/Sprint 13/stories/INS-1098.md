## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that displays the provider's active Provider_Network_Agreement__c (PNA) details, associated Fee_Schedule__c line items, and current credentialing status so that I can quickly assess a provider's network standing without navigating across multiple objects.

## Acceptance Criteria

Given a Provider__c record is open in the Salesforce UI,
When the Provider 360 FlexCard loads,
Then it displays all PNA records where Provider_Network_Agreement__c.Provider__c = current record AND Provider_Network_Agreement__c.Status__c = 'Active', showing PNA Name, Effective_Date__c, Expiration_Date__c, and LOB__c.

Given an active PNA exists for the provider,
When the specialist expands the Fee Schedule section of the FlexCard,
Then all Fee_Schedule__c records linked to the PNA are displayed with Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Allowed_Amount__c, and Fee_Schedule__c.Effective_Date__c.

Given a Credentialing_Case__c exists for the provider,
When the FlexCard renders,
Then the credentialing status badge reflects Credentialing_Case__c.Status__c with color coding: 'Approved' = green, 'Pending' = yellow, 'Expired' = red.

Given a provider has no active PNA,
When the FlexCard loads,
Then a contextual message 'No Active Network Agreement on File' is displayed with a quick-action button to initiate a new PNA via OmniScript.

-Fee Schedule data should be displayed inline without pagination- (superseded: pagination required after UX review due to providers with 200+ CPT codes)

## Solution

Build using OmniStudio FlexCard with nested DataRaptors for fetching PNA and Fee_Schedule__c data. Primary DataRaptor (DR_Get_Provider_PNA) queries Provider_Network_Agreement__c with SOQL filter on Status__c = 'Active'. Secondary DataRaptor (DR_Get_Fee_Schedule_Lines) is triggered on PNA card expand action. Credentialing badge uses a conditional state on FlexCard tied to Credentialing_Case__c.Status__c. Quick-action button launches OmniScript (OS_Initiate_PNA) in a modal. FlexCard is deployed via Lightning App Builder to the Provider__c record page layout.

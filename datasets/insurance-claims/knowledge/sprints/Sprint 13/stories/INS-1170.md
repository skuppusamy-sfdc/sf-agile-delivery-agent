## Description

**Bug Report — Claim Status COB Bypass**

**Environment:** Acme Insurance Full Sandbox (Sprint 13)
**Severity:** High — impacts COB claims adjudication accuracy

**Steps to Reproduce:**
1. Open any Claim__c record where Claim__c.COB_Flag__c = TRUE and Claim__c.Primary_Payer_EOB_Received__c = FALSE.
2. Manually set Claim__c.Status__c = 'Under Review' and save.
3. Navigate to the 'Adjudicate Claim' quick action and click 'Submit for Adjudication'.
4. Observe Claim__c.Status__c after the Record-Triggered Flow fires.

**Expected Behavior:** When COB_Flag__c = TRUE and Primary_Payer_EOB_Received__c = FALSE, the status transition to 'Adjudicated' must be blocked and Status__c should remain 'Pending COB' with a validation error surfaced to the user.

**Actual Behavior:** Claim__c.Status__c transitions to 'Adjudicated' regardless of COB flag, bypassing the COB hold logic entirely. This causes downstream Payment__c records to be generated prematurely before the primary payer EOB is received.

**Root Cause (Preliminary):** The Entry Condition on the 'Claims_Status_Transition_Flow' Record-Triggered Flow does not evaluate COB_Flag__c. The COB hold was implemented as a Validation Rule (Claim_COB_Hold_VR) but the quick action invokes a Screen Flow that performs a direct record update via a Data element, bypassing standard page layout save and therefore bypassing the Validation Rule.

**Impact:** Approximately 340 COB claims in the sandbox show Status__c = 'Adjudicated' with no Primary_Payer_EOB_Received__c. Compliance Officer has flagged this as a potential CMS audit risk.

## Acceptance Criteria

As per Description. Additional verification: After fix deployment, execute SOQL query SELECT Id, Status__c, COB_Flag__c, Primary_Payer_EOB_Received__c FROM Claim__c WHERE COB_Flag__c = TRUE AND Primary_Payer_EOB_Received__c = FALSE AND Status__c = 'Adjudicated' — result set must return 0 rows in regression test org.

## Solution

Add an explicit Decision element at the start of the 'Claims_Status_Transition_Flow' Record-Triggered Flow evaluating {!$Record.COB_Flag__c} = TRUE AND {!$Record.Primary_Payer_EOB_Received__c} = FALSE. If true, route to a Custom Error element surfacing the message 'COB hold active: Primary Payer EOB must be received before adjudication.' and set Status__c = 'Pending COB' via an Update Records element. Additionally, add the same COB condition check inside the Screen Flow's Apex action (ClaimAdjudicationController.adjudicateClaim()) as a server-side guard to prevent bypass via API. Write Apex test class (ClaimCOBStatusTest) covering positive and negative scenarios with minimum 90% code coverage. Data fix script to revert impacted sandbox records will be run by David Kim prior to UAT.

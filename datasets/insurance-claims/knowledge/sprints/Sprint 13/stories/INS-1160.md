## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record that surfaces real-time credentialing status, active PNA agreements, facility affiliations, and open PCNs so that I can make informed decisions during provider calls without navigating across multiple objects.

## Acceptance Criteria

Given a Provider__c record is open in Salesforce,
When the Provider 360 FlexCard loads,
Then it must display the provider's NPI, TIN, Credentialing_Case__c.Status__c, and Provider_Network_Agreement__c.Effective_Date__c without requiring a page refresh.

Given the provider has one or more active Provider_Network_Agreement__c records,
When the PNA section of the FlexCard renders,
Then each PNA row must show PNA__c.Network_Tier__c, PNA__c.LOB__c, PNA__c.Termination_Date__c, and a color-coded status badge (Green = Active, Yellow = Pending, Red = Terminated).

Given a Provider_Change_Notification__c (PCN) record exists with PCN__c.Status__c = 'Open' linked to the provider,
When the PCN panel is expanded on the FlexCard,
Then all open PCNs must be listed with PCN__c.Change_Type__c, PCN__c.Effective_Date__c, and PCN__c.Assigned_To__c visible.

Given a Credentialing_Case__c is in status 'Under Review' or 'Pending CAQH Verification',
When the Credentialing tile is rendered,
Then a warning banner must appear stating 'Active Credentialing Case — Network Actions Restricted' and the Re-Contract button must be disabled.

-Given the FlexCard loads CAQH data via a direct API callout on open,-
-This approach is deprecated in favor of a cached DataRaptor Integration Procedure per INS-1143.-

Given the Provider Relations Specialist does not have the 'Provider_Network_Admin' Permission Set Group,
When they view the FlexCard,
Then PNA financial terms (Fee_Schedule__c.Rate__c) must be masked and display as '****'.

## Solution

Build an OmniStudio FlexCard named 'Provider360_Card' with child cards for Credentialing, PNA Agreements, Facility Affiliations, and PCNs. Use DataRaptor Extract actions to query Credentialing_Case__c, Provider_Network_Agreement__c, and Provider_Change_Notification__c with relationship filters on Provider__c Id. Implement a DataRaptor Integration Procedure (cached, 15-min TTL) to retrieve CAQH profile data rather than a live callout. Conditionally render the Re-Contract button using a FlexCard State Action bound to Credentialing_Case__c.Status__c. Apply field-level masking for Fee_Schedule__c.Rate__c via a Custom Permission check within the DataRaptor Transform. Deploy via OmniStudio version sets; include a Permission Set Group 'Provider_Network_Admin' for unmasked access.

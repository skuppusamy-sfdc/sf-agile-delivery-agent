## Description

As a Claims Operations Manager, I need Agentforce to analyze a newly submitted FNOL (First Notice of Loss) Claim__c record and provide an AI-generated triage recommendation — including suggested claim complexity tier, potential SIU referral flag, and recommended adjudicator assignment — so that claims are routed to the right resource within minutes of submission rather than relying on manual review queues.

## Acceptance Criteria

Given a Claim__c record is created with Claim_Status__c = 'FNOL Submitted'
When the Record-Triggered Flow fires and invokes the Agentforce Claim Triage action
Then an AI_Triage_Recommendation__c child record is created on the Claim__c with fields: Recommended_Complexity_Tier__c (values: 'Simple', 'Moderate', 'Complex'), Recommended_Adjudicator_Queue__c, SIU_Referral_Flag__c (Boolean), and AI_Confidence_Score__c (0.00–1.00)

Given the Agentforce recommendation is generated
When AI_Confidence_Score__c >= 0.85
Then Claim__c.Complexity_Tier__c is auto-populated with the recommended value and Claim__c.Assigned_Queue__c is set accordingly without requiring human confirmation

Given the Agentforce recommendation is generated
When AI_Confidence_Score__c < 0.85
Then the recommendation is surfaced as a suggested action on the Claims Operations FlexCard with Accept/Override buttons; auto-assignment does not occur

Given SIU_Referral_Flag__c = TRUE on the AI_Triage_Recommendation__c
When the recommendation confidence score is >= 0.75
Then a Task is created and assigned to the SIU queue with Subject = 'AI-Flagged SIU Review: [Claim__c.Name]' and Priority = 'High'

Given the adjudicator overrides the AI recommendation
When the override is saved
Then AI_Triage_Recommendation__c.Override_Reason__c is required, Override_By__c = current user, Override_Date__c = NOW(), and the override event is logged to the AI_Audit_Trail__c object for compliance reporting

Given the Agentforce service is unavailable
When the callout times out or returns an error
Then Claim__c is assigned to the 'Manual Triage' queue, an alert is surfaced to the Claims Operations Manager, and Integration_Log__c records the failure with Error_Code__c and Timestamp__c

## Solution

Configure an Agentforce Agent Action 'ClaimTriageAction' using Einstein AI Platform with a custom Prompt Template stored in Setup > Prompt Builder. The prompt template references Claim__c merge fields (LOB__c, Claim_Type__c, Incident_Description__c, Total_Claimed_Amount__c, Provider__c.Credentialing_Status__c). Record-Triggered Flow on Claim__c (after insert, Status = 'FNOL Submitted') calls the Agentforce action via Flow's 'Run Agent Action' element. Response is parsed in an Apex invocable method 'TriageResponseParser.cls' which creates AI_Triage_Recommendation__c. LWC 'TriageRecommendationCard' surfaces the low-confidence recommendations with Accept/Override UX. All AI decisions logged to AI_Audit_Trail__c via a separate after-insert trigger.

## Description

As a Claims Operations Manager, I need an Agentforce-powered AI summary panel embedded on the Claim__c record page that analyzes FNOL intake data, claim history, and provider billing patterns to surface a plain-language triage summary and flag claims that meet SIU referral criteria so that adjusters can make faster, more consistent triage decisions at FNOL.

## Acceptance Criteria

Given a Claim__c record where Claim__c.FNOL_Completed__c = TRUE and Claim__c.Stage__c = 'FNOL Review',
When the adjuster opens the Claim__c record page,
Then the Agentforce Claims AI panel must render a plain-language summary of no more than 250 words covering: claimant demographics, incident description, LOB, and initial reserve amount (Claim__c.Initial_Reserve__c).

Given the AI model evaluates SIU referral criteria,
When any of the following conditions are met:
  - Claim__c.Initial_Reserve__c > 50000
  - The claimant has 3 or more prior Claim__c records in the past 24 months with Claim__c.Status__c = 'Closed — Paid'
  - The billing Provider__c has a Provider__c.SIU_Watch_Flag__c = TRUE
Then the panel must display a red 'SIU Referral Recommended' badge and a pre-populated 'Refer to SIU' quick action button.

Given the adjuster clicks 'Refer to SIU',
When the action executes,
Then Claim__c.SIU_Referred__c must be set to TRUE, Claim__c.SIU_Referral_Date__c = TODAY(), and a child SIU_Referral__c record must be created with SIU_Referral__c.AI_Confidence_Score__c populated from the Agentforce response payload.

Given the AI panel generates a summary,
When the response is returned,
Then the full prompt and response must be logged to an AI_Interaction_Log__c record linked to the Claim__c for audit and compliance purposes.

Given a user without the 'SIU_Referral_Access' Permission Set,
When the Agentforce panel renders,
Then the 'Refer to SIU' button must be hidden and replaced with a read-only informational label 'SIU Review Pending Supervisor Action'.

## Solution

Configure an Agentforce Agent Action using the Einstein Generative AI platform with a custom Prompt Template 'FNOL_Triage_Summary_Prompt' that injects Claim__c fields (Claim__c.Incident_Description__c, Claim__c.LOB__c, Claim__c.Initial_Reserve__c) and related object data via a DataRaptor Extract. Build an LWC 'agentforceClaimsTriage' embedded on the Claim__c Lightning Record Page using a Lightning App Builder dynamic form. The LWC calls an Apex controller 'AgentforceClaimsController.cls' which invokes the Agentforce API and evaluates SIU trigger rules using a Custom Metadata type 'SIU_Rule__mdt'. AI_Interaction_Log__c insertion handled in an after-execution Apex callout handler. Permission visibility controlled via a Lightning Component Attribute bound to a custom permission check.

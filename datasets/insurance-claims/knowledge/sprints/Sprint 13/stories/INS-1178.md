## Description

**Bug Report — ERA Payment Posting Modifier Mapping Failure**

**Environment:** Acme Insurance Full Sandbox (FS1)
**Discovered by:** Lisa Park (Claims Operations Manager) during UAT cycle for INS-1162

**Steps to Reproduce:**
1. Open an existing Claim__c record with at least two Claim_Line__c child records where one line has CPT_Code__c = '99213' with no modifier and a second line has CPT_Code__c = '99213' with Modifier__c = '25'.
2. Trigger the ERA inbound processing by uploading a test 835 EDI file via the ERA_Inbound__c intake screen.
3. Observe the Payment__c record created and review the child Claim_Line__c Allowed_Amount__c and Plan_Paid_Amount__c fields after posting.

**Expected Behavior:**
Each Claim_Line__c should receive the allowed amount corresponding to its specific CPT + modifier combination as defined in the applicable Fee_Schedule__c. The '99213-25' line should map to Fee_Schedule__c records where Fee_Schedule__c.CPT_Code__c = '99213' AND Fee_Schedule__c.Modifier__c = '25'.

**Actual Behavior:**
Both Claim_Line__c records receive the allowed amount from the unmodified '99213' Fee_Schedule__c entry. The modifier-specific fee schedule row is ignored. This results in Claim_Line__c.Allowed_Amount__c being overstated on the modified line in cases where the modifier reduces reimbursement (e.g., modifier 52 = reduced services).

**Impact:** Potential overpayment on claims with reduction modifiers (52, 53, 73, 74). SIU flagged 3 production-equivalent test claims with $1,200+ variance.

## Acceptance Criteria

As per Description. Verified fix must demonstrate correct Claim_Line__c.Allowed_Amount__c mapping for CPT codes with modifiers 25, 51, 52, 59, and GT in regression test suite.

## Solution

Root cause identified in EraPaymentPostingService.cls — the SOQL query for Fee_Schedule__c lookup omits the Modifier__c field in the WHERE clause when Claim_Line__c.Modifier__c is non-null. Fix: update the dynamic SOQL builder in EraPaymentPostingService.buildFeeScheduleQuery() to include 'AND Modifier__c = :modifierValue' when modifier is present, and 'AND Modifier__c = null' when absent. Add a compound index on Fee_Schedule__c (CPT_Code__c, Modifier__c, Effective_Date__c) to support query performance. Deploy updated Apex class and add 8 new Apex test methods covering modifier permutations to EraPaymentPostingServiceTest.cls, targeting >95% coverage on the modified method.

## Description

As a Claims Operations Manager, I need an automated ERA (Electronic Remittance Advice) reconciliation engine that ingests 835 transaction files from our clearinghouse and auto-matches payment records to open Claim_Line__c items so that manual payment posting effort is reduced by at least 70% and Payment__c records are created with full audit traceability.

## Acceptance Criteria

Given a valid 835 ERA file is received via the DataIntegration middleware,
When the ERA Reconciliation Batch Job executes,
Then each payment segment is parsed and a Payment__c record is created with Payment__c.ERA_Transaction_ID__c, Payment__c.Amount__c, Payment__c.Payment_Date__c, and Payment__c.Status__c = 'Posted'.

Given a Payment__c record is created from an ERA file,
When the NPI and TIN in the 835 file match a Provider__c record,
Then Payment__c.Provider__c is populated via lookup and Payment__c.Match_Confidence__c = 'High'.

Given a Claim_Line__c exists with Claim_Line__c.Status__c = 'Approved' and a matching ERA segment is found by ICN (Internal Control Number),
When the reconciliation engine processes the match,
Then Claim_Line__c.Status__c transitions to 'Paid', Claim_Line__c.Payment__c is populated, and Claim__c.Status__c is re-evaluated for full closure.

Given an ERA segment cannot be matched to any open Claim_Line__c,
When the batch completes processing,
Then an unmatched Payment__c record is created with Payment__c.Status__c = 'Pending Review' and a Case is auto-generated and assigned to the Claims Operations queue.

Given the reconciliation batch has completed,
When all line items are processed,
Then a Reconciliation_Summary__c record is created capturing ERA_File_Name__c, Total_Segments__c, Matched_Count__c, Unmatched_Count__c, and Total_Amount_Posted__c.

-Given duplicate 835 files are submitted, the system should silently discard them- (superseded: duplicate detection now handled upstream by clearinghouse per INS-1067)

Given a duplicate 835 file transaction ID is detected,
When the batch attempts to create a Payment__c,
Then the record is rejected, Payment__c.Status__c = 'Duplicate Rejected', and a warning Platform Event is fired to the ops dashboard.

## Solution

Implement an Apex Batch class (ERA_ReconciliationBatch) triggered via Scheduled Apex nightly at 2:00 AM CST. 835 files are staged in an external S3 bucket and fetched via Named Credential callout. A custom ERA_Parser Apex utility class handles X12 835 segment parsing. Record-Triggered Flow on Payment__c handles downstream Claim_Line__c status transitions to avoid recursive Apex. Platform Events (ERA_Reconciliation_Event__e) broadcast batch completion metrics to an LWC dashboard component on the Claims Operations Home Page. A custom Permission Set Group 'ERA_Admin_PSG' controls access to Reconciliation_Summary__c records.

## Description

As a Claims Operations Manager, I need an Agentforce AI copilot embedded in the Claim__c record page that assists claims adjusters during FNOL intake by suggesting claim categorization, flagging potential duplicate claims, and surfacing SIU referral indicators so that adjudicators can process FNOL events faster and with greater accuracy while reducing fraudulent claim leakage.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Status__c = 'FNOL' and Claim__c.Incident_Description__c populated,
When the Agentforce copilot panel loads on the record page,
Then the AI must return a suggested Claim__c.Claim_Type__c value and Claim__c.LOB__c classification with a confidence score >= 0.75 displayed in the copilot panel.

Given the FNOL intake form is submitted,
When Agentforce evaluates the claim against the duplicate detection model,
Then if a potential duplicate Claim__c is identified (same Policy__c, same Incident_Date__c, overlapping CPT codes on Claim_Line__c), the copilot must display a warning card listing the suspected duplicate Claim__c.Name and Claim__c.Status__c.

Given the AI copilot analyzes FNOL data,
When SIU referral indicators are detected (e.g., Claim__c.Incident_Description__c contains keywords matching the SIU ruleset, or provider's Claim__c submission frequency exceeds 3 standard deviations from peer mean),
Then the copilot must display a 'Refer to SIU' recommendation and pre-populate an SIU_Referral__c draft record with SIU_Referral__c.Claim__c, SIU_Referral__c.Trigger_Reason__c, and SIU_Referral__c.AI_Confidence_Score__c.

Given a claims adjuster accepts the AI's suggested Claim__c.Claim_Type__c,
When they click 'Apply Suggestion',
Then Claim__c.Claim_Type__c and Claim__c.LOB__c must be updated and an AI_Interaction_Log__c record must be created capturing AI_Interaction_Log__c.Model_Version__c, AI_Interaction_Log__c.Accepted__c = TRUE, and AI_Interaction_Log__c.Adjuster__c.

Given the copilot is active,
When a user without the 'Claims_AI_User' Permission Set Group accesses the Claim__c record,
Then the Agentforce copilot panel must not render and no AI API calls must be made.

## Solution

Configure an Agentforce Agent of type 'Copilot' scoped to the Claim__c object page using Agent Builder. Define three Agent Actions: (1) 'ClassifyClaim' — invokes an Apex invocable method that calls Einstein Prediction Service model 'Claim_Type_Classifier_v2'; (2) 'DetectDuplicate' — executes a SOSL query across Claim__c filtered by Policy__c, Incident_Date__c, and Claim_Line__c CPT codes; (3) 'EvaluateSIURisk' — calls an Apex class 'SIURiskEvaluator' using a custom rules engine backed by SIU_Rule__c metadata records. Embed the copilot panel as a Lightning Web Component 'agentforceFNOLCopilot' on the Claim__c Lightning Page via dynamic component visibility tied to the 'Claims_AI_User' custom permission. Log all interactions to AI_Interaction_Log__c via a Record-Triggered Flow on copilot action completion. Use a Permission Set Group 'Claims_AI_User' to gate access.

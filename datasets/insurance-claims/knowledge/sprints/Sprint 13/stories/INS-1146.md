## Description

**Bug Report — ERA 835 Inbound Integration**

**Steps to Reproduce:**
1. Trigger the scheduled ERA inbound batch job (DataIntegration_ERA835_Batch) with a test 835 file containing a provider NPI that does not exist in the local NPPES cache (Provider__c records).
2. Monitor the batch job execution log in Salesforce Setup > Apex Jobs.
3. Observe that the batch job completes with status 'Completed' and 0 failures reported.
4. Navigate to the associated Payment__c records — none are created for the unmatched NPI segments.
5. No error log, no AsyncApexJob failure record, and no alert email is generated.

**Expected Behavior:** When an ERA 835 segment contains an NPI that cannot be matched to a Provider__c record, the system should set ERA_Processing_Log__c.Status__c = 'Failed - NPI Not Found', increment ERA_Processing_Log__c.Unmatched_NPI_Count__c, and send an alert email to the DataIntegration ops queue.

**Actual Behavior:** The unmatched NPI segment is silently skipped. ERA_Processing_Log__c.Status__c remains 'Completed'. No Payment__c record is created and no notification is dispatched. This causes reconciliation discrepancies between the payer ledger and Salesforce Payment__c totals.

**Environment:** Partial Sandbox (PS1). Reproducible 100% of attempts with test file ERA_TEST_BADNPI_20240915.835.

**Impact:** High — financial reconciliation gap; potential compliance exposure under state prompt-pay regulations.

## Acceptance Criteria

As per Description

## Solution

Patch the 'ERA835BatchProcessor' Apex class: in the 'processSegment()' method, add a null-check after the SOQL query for Provider__c by NPI. On null result, insert an ERA_Processing_Log__c child record with Status__c = 'Failed - NPI Not Found' and the raw NPI value, increment a batch-level unmatched counter, and after batch finish, invoke a post-execute email alert via Messaging.sendEmail() to the configured ops queue address stored in Custom Metadata (ERA_Config__mdt.Ops_Alert_Email__c). Add a new Apex test method 'testUnmatchedNPI_SilentFailureFix' with a mock HTTP callout for NPPES and assert ERA_Processing_Log__c record creation.

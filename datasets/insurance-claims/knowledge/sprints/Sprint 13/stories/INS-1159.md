## Description

As an Assignment Coordinator, I need a Policyholder 360 LWC dashboard embedded on the Contact/Account record for insured members so that I can instantly view all active Policy__c records, open Claim__c records, and the last 12 months of Explanation_of_Benefits__c history without switching between tabs or running reports.

## Acceptance Criteria

Given a Contact record with Contact.Is_Insured_Member__c = true is opened,
When the 'Policyholder360Dashboard' LWC loads,
Then all Policy__c records where Policy__c.Insured_Contact__c = Contact.Id and Policy__c.Status__c = 'Active' are displayed in a tabular section showing Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, and Policy__c.Renewal_Date__c.

Given the policyholder has open claims,
When the Claims section renders,
Then all Claim__c records where Claim__c.Claimant__c = Contact.Id and Claim__c.Status__c IN ('Open', 'In Review', 'Pending COB Secondary') are listed with Claim__c.ClaimNumber__c, Claim__c.Date_of_Service__c, Claim__c.Total_Claimed_Amount__c, and a status badge color-coded by status value.

Given the EOB History section loads,
When the component queries Explanation_of_Benefits__c records,
Then only EOBs where EOB__c.Created_Date__c >= LAST_N_MONTHS:12 and EOB__c.Insured_Contact__c = Contact.Id are returned, sorted descending by Created_Date__c, with a maximum of 50 records displayed and a 'Load More' pagination control.

-Given the dashboard includes a real-time deductible accumulator widget sourced from Policy__c.YTD_Deductible_Met__c- (deprecated — deductible widget descoped to Phase 3 per product backlog grooming on 2024-11-05)

Given the LWC is accessed by a user without the 'Policyholder_360_Viewer' Permission Set,
When the component attempts to load,
Then the dashboard renders a permission error message and no PHI data is exposed.

Given the policyholder has no open claims and no active policies,
When the LWC loads,
Then each section displays an empty-state illustration with contextual helper text (e.g., 'No active policies found for this member').

## Solution

Build 'Policyholder360Dashboard' LWC with three child components: 'policyList', 'openClaimsList', and 'eobHistory'. Use @wire with custom Apex controller 'Policyholder360Controller' exposing @AuraEnabled(cacheable=true) methods for each data section. Implement server-side SOQL with WITH SECURITY_ENFORCED to enforce FLS. Pagination for EOB history handled via OFFSET/LIMIT pattern with a 'loadMore' button wired to an imperative Apex call. Permission check performed via FeatureManagement.checkPermission('Policyholder_360_Viewer') in the connectedCallback. Deploy LWC to the Contact Lightning Record Page via a targeted App Builder assignment scoped to the 'Claims_Service_App'. Conduct LWC Jest unit tests covering empty-state and permission-denied scenarios.

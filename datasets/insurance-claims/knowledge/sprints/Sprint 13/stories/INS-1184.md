## Description

As a Claims Operations Manager, I need the platform to automatically ingest inbound ANSI X12 835 Electronic Remittance Advice (ERA) files from our clearinghouse, reconcile each payment loop against open Claim_Line__c records, and create or update Payment__c records so that manual ERA posting is eliminated and payment turnaround SLAs are met.

## Acceptance Criteria

Given an 835 ERA file is deposited in the designated MuleSoft SFTP endpoint,
When the DataIntegration middleware publishes a Platform Event ERA_File_Received__e with payload including file_id, total_amount, and provider_npi,
Then a Salesforce Platform Event trigger must fire within 60 seconds and create an ERA_Batch__c header record with ERA_Batch__c.Status__c = 'Processing'.

Given the ERA_Batch__c record is created,
When Apex batch job ERAReconciliationBatch executes,
Then each 835 CLP loop must be matched to Claim__c via Claim__c.Clearinghouse_Claim_Number__c and each SVC segment matched to Claim_Line__c.CPT_Code__c + Claim_Line__c.Service_Date__c.

Given a successful CLP-to-Claim__c match,
When payment amounts are reconciled,
Then a Payment__c record must be created with Payment__c.Amount__c = paid amount from 835, Payment__c.Adjustment_Reason_Code__c populated from CAS segments, Payment__c.Status__c = 'Posted', and Payment__c.ERA_Batch__c = parent ERA_Batch__c.Id.

Given a CLP loop where Claim__c cannot be matched,
When reconciliation fails for that loop,
Then the loop must be written to ERA_Exception__c with ERA_Exception__c.Reason__c = 'Claim Not Found' and ERA_Batch__c.Exception_Count__c incremented.

Given all CLP loops are processed,
When ERA_Batch__c.Exception_Count__c = 0,
Then ERA_Batch__c.Status__c must be set to 'Reconciled Complete'; if exceptions exist, set to 'Reconciled With Exceptions' and notify Claims Operations Manager via custom notification.

Given Payment__c.Status__c is set to 'Posted',
When the parent Claim__c has all Claim_Line__c records with Claim_Line__c.Payment_Status__c = 'Paid',
Then Claim__c.Status__c must auto-update to 'Adjudicated' via Record-Triggered Flow.

## Solution

MuleSoft integration layer parses 835 file and publishes ERA_File_Received__e Platform Event per batch. Apex Platform Event trigger invokes ERAReconciliationBatch (Database.Batchable, batch size 50). Matching logic in ERAMatcher Apex class uses indexed fields Claim__c.Clearinghouse_Claim_Number__c and Claim_Line__c.CPT_Code__c. Payment__c creation via PaymentFactory Apex class. Record-Triggered Flow 'Claim_Status_Auto_Update_Flow' fires on Claim_Line__c update, uses Get Records to check all sibling lines, and updates parent Claim__c. Custom notification via CustomNotificationType 'ERA_Exception_Alert'.

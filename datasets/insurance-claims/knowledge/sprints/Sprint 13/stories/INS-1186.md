## Description

**Bug Report — Extends INS-1174 (Fee Schedule Engine Phase 2 build)**

**Steps to Reproduce:**
1. Navigate to any active Claim__c record where Claim__c.LOB__c = 'Commercial' and Claim__c.Status__c = 'Pending Adjudication'.
2. Open a child Claim_Line__c record where Claim_Line__c.CPT_Code__c = '99213' and Claim_Line__c.CPT_Modifier__c = '25'.
3. Trigger adjudication via the 'Adjudicate Claim' quick action.
4. Observe Claim_Line__c.Allowed_Amount__c after adjudication completes.

**Expected Result:** Fee_Schedule__c lookup matches on both Fee_Schedule__c.CPT_Code__c = '99213' AND Fee_Schedule__c.CPT_Modifier__c = '25', returning the modifier-specific allowed amount (e.g., $95.00).

**Actual Result:** Claim_Line__c.Allowed_Amount__c = null. The FeeScheduleLookupService Apex class performs a SOQL query filtering only on Fee_Schedule__c.CPT_Code__c and ignores Fee_Schedule__c.CPT_Modifier__c when the modifier field is non-null, causing no record to match and returning null.

**Impact:** High — all claims with CPT modifiers (estimated 23% of daily volume) are failing adjudication and stacking in 'Pending Adjudication' status, blocking payment disbursement.

**Environment:** Production (also reproducible in Staging)
**Regression Introduced:** Sprint 12 deployment on 2024-11-04

## Acceptance Criteria

As per Description.

Additional verification:
Given Claim_Line__c.CPT_Modifier__c is non-null,
When FeeScheduleLookupService executes,
Then the SOQL query must include WHERE Fee_Schedule__c.CPT_Code__c = :cptCode AND Fee_Schedule__c.CPT_Modifier__c = :modifier and return the correct allowed amount.

Given Claim_Line__c.CPT_Modifier__c is null,
When FeeScheduleLookupService executes,
Then the lookup must fall back to Fee_Schedule__c records where CPT_Modifier__c = null (base rate) without error.

## Solution

Fix FeeScheduleLookupService.lookupAllowedAmount() Apex method to include CPT_Modifier__c in SOQL WHERE clause when Claim_Line__c.CPT_Modifier__c is non-null. Implement fallback logic: if modifier-specific record not found, fall back to base CPT rate and log a Fee_Schedule_Exception__c record with Exception_Type__c = 'Modifier Fallback'. Add @isTest coverage for modifier/no-modifier scenarios. Deploy hotfix to Staging for QA validation before Production push. Backfill impacted Claim_Line__c records via DataLoader for claims stuck in 'Pending Adjudication' since Sprint 12 go-live.

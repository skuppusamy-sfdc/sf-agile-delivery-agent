## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record so that I can view credentialing status, active PNA agreements, network participation, and recent claims activity in a single consolidated view without navigating across multiple objects.

## Acceptance Criteria

Given a Provider__c record is open in Salesforce,
When the Provider 360 FlexCard loads,
Then the card must display Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Credentialing_Status__c, and Provider__c.Primary_Specialty__c in the header band.

Given the FlexCard header band renders,
When Credentialing_Case__c records exist linked to the provider,
Then the most recent Credentialing_Case__c.Status__c and Credentialing_Case__c.Expiration_Date__c must be shown with a color-coded badge (Green = Active, Amber = Expiring within 90 days, Red = Expired or Suspended).

Given the provider has one or more Provider_Network_Agreement__c records,
When the PNA tab is selected on the FlexCard,
Then all PNA records where PNA.Effective_Date__c <= TODAY and PNA.Termination_Date__c >= TODAY must be listed, showing PNA.Network_Name__c, PNA.LOB__c, and PNA.Contract_Type__c.

Given a provider has associated Fee_Schedule__c records,
When the Fee Schedule tab is active,
Then Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.CPT_Code_Range__c must display; -Fee_Schedule__c.Legacy_Rate_Code__c should also display- (deprecated per INS-1145 rate code consolidation).

Given any Network_Termination_Case__c record exists for the provider with status 'In Progress' or 'Pending',
When the FlexCard loads,
Then a prominent warning banner must appear stating 'Active Termination Case: [Network_Termination_Case__c.Case_Number__c]' with a link to the record.

Given the Claims Activity section of the FlexCard,
When rendered,
Then the count of Claim__c records where Claim__c.Rendering_Provider__c = Provider__c.Id grouped by Claim__c.Status__c ('Open', 'Adjudicated', 'Denied', 'Paid') for the trailing 12 months must be displayed as a summary bar chart.

Given the FlexCard is accessed by a user without the 'Provider_Network_Full_Access' Permission Set Group,
When the PNA and Fee Schedule tabs render,
Then financial rate fields (Fee_Schedule__c.Allowed_Amount__c) must be masked and replaced with '---'.

## Solution

Build an OmniStudio FlexCard (Provider360_Card) with child DataRaptors to query Provider__c, Credentialing_Case__c (SOQL with ORDER BY CreatedDate DESC LIMIT 1), Provider_Network_Agreement__c, Fee_Schedule__c, Network_Termination_Case__c, and an aggregate SOQL on Claim__c. Use a DataRaptor Transform to compute credentialing badge color logic. Conditional visibility on FlexCard states controls the warning banner. A custom Permission Set Group check via a DataRaptor Extract on PermissionSetAssignment gates financial field display. Deploy via OmniStudio Migration Tool into the Phase2 scratch org.

## Description

As a Credentialing Specialist, I need all legacy provider credentialing records migrated from the source CSV extracts into Salesforce Credentialing_Case__c objects and enriched with current CAQH profile data via the CAQH ProView API so that the credentialing team operates from a single source of truth beginning in Sprint 14.

## Acceptance Criteria

Given the migration CSV contains 4,200 provider credentialing records exported from the legacy system,
When the Data Loader migration job executes against the Full Sandbox,
Then 100% of records must be loaded as Credentialing_Case__c with Credentialing_Case__c.Provider__c correctly resolved via NPI lookup against existing Provider__c records, and a migration reconciliation report must be produced.

Given a Credentialing_Case__c record is created via migration with a non-null Credentialing_Case__c.CAQH_Profile_ID__c,
When the post-migration CAQH enrichment batch job 'CAQH_EnrichmentBatch' runs,
Then the job must call the CAQH ProView REST API and update Credentialing_Case__c.Board_Certification__c, Credentialing_Case__c.License_State__c, Credentialing_Case__c.License_Expiration_Date__c, and Credentialing_Case__c.CAQH_Last_Updated__c.

Given a provider NPI in the migration file does not match any Provider__c record in Salesforce,
When the migration job processes the row,
Then the row must be written to a migration exception report with columns: Legacy_ID, NPI, Error_Reason, and the Credentialing_Case__c must NOT be created.

-Given legacy records with Status = 'Lapsed' should be migrated as Credentialing_Case__c.Status__c = 'Expired'-
(Revised: 'Lapsed' maps to 'Requires Re-credentialing' per compliance team directive dated 2024-11-01)

Given the migration and enrichment are complete,
When a Credentialing Specialist reviews a migrated Credentialing_Case__c,
Then the record must display a Migration_Source__c = 'Legacy_System_2019' field value to distinguish migrated records from natively created ones.

## Solution

Use Salesforce Data Loader CLI in upsert mode with NPI__c as the external ID on Provider__c to resolve provider lookups during migration. Pre-process CSV with a Python transformation script to normalize status mappings per updated business rules. Post-migration, execute Apex Batchable 'CAQH_EnrichmentBatch' using a Named Credential 'CAQH_ProView_NC' for API authentication. CAQH API responses parsed and mapped via Apex JSON deserialization. Exception rows written to a DataIntegration_Error__c log. Full sandbox migration to be validated before production cutover in Sprint 14.

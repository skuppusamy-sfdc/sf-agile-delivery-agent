## Description

As a Claims Operations Manager, I need a Policyholder 360 EOB history timeline component on the Policy__c record page so that service representatives can quickly review a chronological list of all Explanation_of_Benefits__c records for a policyholder, see key payment and adjudication details inline, and offer the policyholder a direct PDF download link during phone interactions — reducing average handle time for EOB inquiries. This story extends INS-1131 which built the base Policyholder 360 record page layout.

## Acceptance Criteria

Given a service representative opens a Policy__c record
When the EOB History Timeline LWC loads
Then all EOB__c records where EOB__c.Policy__c = Policy__c.Id are displayed in descending order by EOB__c.EOB_Date__c with a maximum of 24 months of history shown by default

Given the EOB list is rendered
When the representative views each EOB row
Then each row displays EOB__c.EOB_Date__c, EOB__c.Claim__c (hyperlinked), EOB__c.Total_Billed__c, EOB__c.Total_Allowed__c, EOB__c.Member_Responsibility__c, and EOB__c.Payment_Status__c

Given an EOB__c record has a linked ContentDocument (PDF)
When the representative clicks 'Download EOB PDF' on that row
Then the ContentDocument is retrieved via a secure ContentDistribution URL and opens in a new browser tab; the download event is logged to EOB__c.Last_Download_Timestamp__c and EOB__c.Download_Count__c is incremented

Given the policyholder has more than 50 EOB records in the 24-month window
When the component initially loads
Then only the 20 most recent EOBs are displayed with a 'Load More' button that retrieves the next 20 in subsequent calls

Given the representative applies the 'Unpaid' filter in the component toolbar
When the filter is active
Then only EOB__c records where EOB__c.Payment_Status__c IN ('Pending', 'Denied', 'Partial Payment') are displayed and the total Member_Responsibility__c for the filtered set is shown in a summary footer

Given the user does not have the 'PHI_ViewSensitive' custom permission
When the EOB component renders
Then EOB__c.Member_Responsibility__c values are hidden and replaced with '-- Restricted --' and the PDF download button is disabled with tooltip 'Insufficient permissions'

## Solution

Build a custom LWC ('eobHistoryTimeline') using the wire service with @wire(getEOBHistory, {policyId: '$recordId', pageSize: 20, offset: '$currentOffset'}) backed by an Apex controller method annotated @AuraEnabled(cacheable=true). Implement server-side pagination using OFFSET and LIMIT in the SOQL query on EOB__c. The PDF download uses a secondary Apex method to generate a ContentDistribution record on-demand and return the DistributionPublicUrl, logging the download event via a separate DML call. Apply Custom Permission checks in the Apex controller using FeatureManagement.checkPermission('PHI_ViewSensitive') and return masked data accordingly. Add the LWC to the Policy__c Lightning Record Page via App Builder with visibility filter: Profile = 'Claims Service Representative' OR Permission Set = 'Claims_Supervisor'. Ensure the component is accessible (ARIA labels, keyboard navigation) per WCAG 2.1 AA standards.

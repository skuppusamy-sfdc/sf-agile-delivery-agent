## Description

As a Provider Network Specialist, I need an automated workflow that, upon initiation of a Network_Termination_Case__c, generates a Provider_Change_Notification__c (PCN) with the correct 90-day advance notice date, updates the Provider_Network_Agreement__c termination fields, and routes the case for Compliance Officer approval so that all network terminations meet state-mandated notice requirements and avoid regulatory penalties.

## Acceptance Criteria

Given a Network_Termination_Case__c record is created with Network_Termination_Case__c.Termination_Reason__c populated and Network_Termination_Case__c.Status__c = 'Initiated',
When the Record-Triggered Flow fires on insert,
Then a Provider_Change_Notification__c must be auto-created with PCN__c.Change_Type__c = 'Termination', PCN__c.Provider__c = Network_Termination_Case__c.Provider__c, PCN__c.Notice_Date__c = TODAY(), and PCN__c.Effective_Date__c = TODAY() + 90.

Given the PCN is created,
When the associated Provider_Network_Agreement__c records are evaluated,
Then all active PNA__c records for the provider must have PNA__c.Termination_Date__c set to PCN__c.Effective_Date__c and PNA__c.Termination_Reason__c = Network_Termination_Case__c.Termination_Reason__c.

Given the Network_Termination_Case__c is in status 'Initiated',
When the automated flow completes PCN and PNA updates,
Then the case must route to the 'Compliance_Review' queue via an Approval Process with Network_Termination_Case__c.Status__c = 'Pending Compliance Approval' and the assigned Compliance Officer must receive an email notification.

Given the Compliance Officer approves the termination in the Approval Process,
When the approval step completes,
Then Network_Termination_Case__c.Status__c must transition to 'Approved — Notice Issued', PCN__c.Status__c must be set to 'Sent', and a DocuSign envelope must be triggered to deliver the termination notice to the provider's primary email on Provider__c.Email.

Given Network_Termination_Case__c.Termination_Reason__c = 'For Cause',
When the case is evaluated by the flow,
Then the 90-day notice period must be overridden to 0 days (PCN__c.Effective_Date__c = TODAY()) and a Compliance_Alert__c record must be created with Compliance_Alert__c.Alert_Type__c = 'Immediate Termination — For Cause' for audit purposes.

Given a provider has open Claim__c records with Claim__c.Status__c IN ('FNOL','In Adjudication') at time of termination,
When the termination workflow completes,
Then a Task must be created for each open claim assigned to the Claims Operations queue with Subject = 'Review open claim — Provider termination initiated'.

## Solution

Implement a Record-Triggered Flow 'Network_Termination_Orchestration_Flow' (After Insert/Update on Network_Termination_Case__c) with decision branches for 'For Cause' vs. standard termination. Use Flow's Create Records element to generate PCN and update PNA records via a Loop + Update Records pattern. Submit to Approval Process 'Network_Termination_Approval' via Flow's Submit for Approval action. Build a separate Record-Triggered Flow on Approval Process final approval to trigger DocuSign via a named Apex invocable method 'DocuSignIntegration.sendTerminationNotice()'. Query open Claim__c records using a Get Records element filtered by Provider__c and Status__c, then loop to create Tasks. Store all termination events in a Compliance_Audit_Trail__c object for regulatory reporting. Activate flow version after UAT sign-off from Compliance Officer role.

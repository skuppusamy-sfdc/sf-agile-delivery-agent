## Description

**Bug Report — INS-1095**

**Summary:** When the CAQH ProView integration returns a status of 'Re-attestation Required' for a provider, the Credentialing_Case__c.CAQH_Status__c field is not updated and remains at its previous value (e.g., 'Attested'). This causes Credentialing Specialists to be unaware of re-attestation requirements until manual CAQH portal review.

**Steps to Reproduce:**
1. Ensure a Provider__c record exists with CAQH_Provider_ID__c populated and an active Credentialing_Case__c with CAQH_Status__c = 'Attested'.
2. In the CAQH ProView sandbox, set the provider's attestation status to 'Re-attestation Required' (simulate attestation expiry).
3. Trigger the CAQHSyncBatch.cls manually via Developer Console.
4. After batch completion, inspect Credentialing_Case__c.CAQH_Status__c.

**Expected Result:** Credentialing_Case__c.CAQH_Status__c = 'Re-attestation Required' and a Task is created for the Credentialing Specialist

**Actual Result:** Credentialing_Case__c.CAQH_Status__c remains 'Attested'. No Task is created.

**Root Cause (preliminary):** CAQHSyncBatch.cls status mapping dictionary (statusMap) does not contain an entry for the string 'Re-attestation Required' returned by CAQH API v3.2. The unmapped status silently falls through to a no-op else branch.

**Affected Records:** 12 providers identified in QA sandbox with this condition.

## Acceptance Criteria

As per Description. Fix verified when CAQHSyncBatch correctly maps 'Re-attestation Required' CAQH API response to Credentialing_Case__c.CAQH_Status__c = 'Re-attestation Required' and creates a Task for the assigned Credentialing Specialist. All existing status mappings must continue to function correctly per CAQHSyncBatch_Test.cls regression suite.

## Solution

Update statusMap in CAQHSyncBatch.cls to include entry: statusMap.put('Re-attestation Required', 'Re-attestation Required'). Add Task creation logic in the batch execute() method for this status — create Task with Subject = 'CAQH Re-attestation Required: [Provider Name]', ActivityDate = TODAY() + 5, OwnerId = Credentialing_Case__c.OwnerId. Add corresponding test method in CAQHSyncBatch_Test.cls using a mock HTTP callout response containing 'Re-attestation Required'. Review full CAQH API v3.2 response schema to ensure all status values are mapped in statusMap and no other values are missing.

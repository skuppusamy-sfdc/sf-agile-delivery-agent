## Description

As a Claims Operations Manager, I need inbound 835 Electronic Remittance Advice (ERA) files to be automatically parsed and reconciled against open Claim_Line__c records so that Payment__c records are created without manual data entry, reducing adjudication cycle time and payment errors. Extends INS-1138 (inbound 837 ingestion pipeline).

## Acceptance Criteria

Given a valid 835 ERA file is deposited in the designated MuleSoft SFTP endpoint,
When the integration pipeline triggers,
Then a Platform Event 'ERA_Received__e' is fired within 2 minutes containing the raw payload and a generated batch correlation ID.

Given the Platform Event is consumed by the Apex handler 'ERAReconciliationHandler',
When claim identifiers in the 835 loop (CLM segment) are matched against Claim__c.ClaimNumber__c,
Then a Payment__c record is created with Payment__c.Amount__c, Payment__c.PaymentDate__c, Payment__c.EFT_Trace_Number__c, and Payment__c.ERA_Batch_ID__c populated.

Given a Claim_Line__c exists with Status__c = 'Approved' and a matching service line in the 835 SVC segment,
When the ERA handler processes the line,
Then Claim_Line__c.Paid_Amount__c is updated and Claim_Line__c.Status__c transitions to 'Paid'.

Given an 835 contains a CO-45 adjustment reason code on a service line,
When the adjustment is processed,
Then an Explanation_of_Benefits__c record is created linked to the parent Claim__c with EOB__c.Adjustment_Reason_Code__c = 'CO-45' and EOB__c.Adjustment_Amount__c populated accurately.

Given a CLM segment in the 835 cannot be matched to any Claim__c record,
When reconciliation fails,
Then the unmatched line is written to a custom object 'ERA_Exception__c' with ERA_Exception__c.Reason__c = 'Claim Not Found' and an alert task is created for the Claims Operations queue.

Given all lines in a batch are successfully reconciled,
When processing completes,
Then Claim__c.Status__c is updated to 'Adjudicated' and a summary notification is sent to the Claims Operations Manager via custom notification type 'ERA_Batch_Complete'.

## Solution

Extend the existing MuleSoft 837 ingestion flow (INS-1138) with a parallel 835 channel. Use a Platform Event 'ERA_Received__e' to decouple file receipt from processing. Implement 'ERAReconciliationHandler' Apex class with Queueable chaining to handle large batch files within governor limits. Use a custom metadata type 'ERA_Adjustment_Code_Map__mdt' to translate X12 adjustment reason codes to internal EOB categories. Record-Triggered Flow on Payment__c insert cascades status updates to parent Claim__c. Deploy ERA_Exception__c with an Assignment Rule routing to the 'Claims_ERA_Exceptions' queue. All DML operations wrapped in Savepoint/Rollback for atomicity per batch segment.

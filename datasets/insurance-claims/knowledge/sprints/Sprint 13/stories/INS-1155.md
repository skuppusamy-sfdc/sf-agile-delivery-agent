## Description

**Bug Report — COB Secondary Adjudication Race Condition**

**Steps to Reproduce:**
1. Create a Claim__c record with Claim__c.COB_Indicator__c = 'Secondary' and Claim__c.Primary_Payer_Status__c = 'Paid'.
2. Trigger the primary payer ERA reconciliation (see INS-1153) so that Payment__c records are created for the primary payer.
3. Observe that the Record-Triggered Flow 'ClaimStatusUpdate_RTF' fires immediately after Payment__c insert.
4. Claim__c.Status__c is set to 'Closed' before the secondary payer COB adjudication has been initiated or completed.

**Expected Behavior:** Claim__c.Status__c should transition to 'Pending COB Secondary' when COB_Indicator__c = 'Secondary' and secondary adjudication has not yet been completed. Status should only reach 'Closed' after Claim__c.COB_Secondary_Paid_Amount__c is populated and Claim__c.COB_Secondary_Status__c = 'Adjudicated'.

**Actual Behavior:** Claim__c.Status__c = 'Closed' is set immediately upon primary Payment__c creation, bypassing COB secondary adjudication gate logic entirely.

**Environment:** Acme Insurance Full Sandbox (FS1)
**Frequency:** 100% reproducible
**Impact:** High — affects all COB secondary claims; secondary payer invoicing is not triggered, resulting in revenue leakage.

## Acceptance Criteria

As per Description.

Given Claim__c.COB_Indicator__c = 'Secondary' and a Payment__c is inserted for the primary payer,
When 'ClaimStatusUpdate_RTF' evaluates the status transition logic,
Then Claim__c.Status__c must be set to 'Pending COB Secondary' and NOT 'Closed'.

Given Claim__c.Status__c = 'Pending COB Secondary',
When Claim__c.COB_Secondary_Status__c is updated to 'Adjudicated',
Then and only then may Claim__c.Status__c transition to 'Closed'.

## Solution

Add a decision element to 'ClaimStatusUpdate_RTF' that checks Claim__c.COB_Indicator__c before executing the 'Close Claim' path. Insert a new flow path: IF COB_Indicator__c = 'Secondary' AND COB_Secondary_Status__c != 'Adjudicated' THEN set Status__c = 'Pending COB Secondary'. Add a second entry criteria on the RTF to handle the COB_Secondary_Status__c = 'Adjudicated' transition separately. Add a unit test in 'ClaimStatusUpdateTest' class covering COB secondary scenario with >75% coverage requirement. Regression test all existing ClaimStatusUpdate_RTF test cases in FS1 before promoting to UAT.

## Description

As a Claims Operations Manager, I need an Agentforce copilot action embedded on the Claim__c record page that generates a natural language summary of the claim's full adjudication history — including FNOL details, coverage verification outcomes, line-level adjudication decisions, COB determinations, and payment disbursements — so that I can onboard new adjusters and resolve escalations faster without reading through dozens of activity log entries.

## Acceptance Criteria

Given an Agentforce copilot is active on a Claim__c record page,
When the Claims Operations Manager invokes the 'Summarize Claim History' copilot action,
Then the action must retrieve Claim__c.FNOL_Date__c, Claim__c.Loss_Type__c, Claim__c.Status__c, all child Claim_Line__c records with Claim_Line__c.CPT_Code__c and Claim_Line__c.Adjudication_Decision__c, related Payment__c.Payment_Date__c and Payment__c.Amount__c, and any Explanation_of_Benefits__c.Denial_Reason__c values.

Given the data is retrieved,
When the prompt template is populated and submitted to the LLM,
Then the generated summary must be returned within 10 seconds and presented in the Agentforce side panel without navigating away from the Claim__c record.

Given the summary is generated,
When it contains any EOB denial reason,
Then the denial reason code must be cited with its plain-language description from the custom metadata type (EOB_Denial_Code__mdt.Description__c).

Given the summary is displayed,
When the user clicks 'Save Summary to Record',
Then the summary text must be saved to Claim__c.AI_Adjudication_Summary__c (Long Text Area) with a prepended timestamp and the Salesforce User name.

Given a Claim__c record is flagged with Claim__c.SIU_Referral__c = true,
When the copilot action is invoked,
Then the summary must include a bold warning header 'SIU REFERRAL ACTIVE' and must NOT expose Claim__c.SIU_Investigation_Notes__c in the LLM prompt for data security compliance.

## Solution

Configure an Agentforce Agent Action of type 'Prompt Template Action' in Einstein Studio. Build a Flex Prompt Template (ClaimAdjudicationSummary_PT) referencing a custom Apex invocable (ClaimHistoryContextBuilder.cls) that assembles the structured context object from Claim__c, Claim_Line__c, Payment__c, EOB__c, and EOB_Denial_Code__mdt. SIU gate implemented as a conditional merge field exclusion in the prompt template. The 'Save Summary' action is a separate Agentforce Flow Action invoking a Record Update on Claim__c.AI_Adjudication_Summary__c. Ground the model with a grounding data source restricted to the Claim__c record context to prevent hallucination of unrelated claims data. Permission Set Group 'Claims_AI_Users' controls access to the copilot action.

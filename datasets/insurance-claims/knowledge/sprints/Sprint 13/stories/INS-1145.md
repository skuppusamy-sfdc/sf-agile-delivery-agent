## Description

As a Claims Operations Manager, I need the system to automatically validate that each Claim_Line__c is adjudicated against the Fee_Schedule__c record whose effective date bracket covers the Claim__c.Date_of_Service__c so that incorrect reimbursement rates from expired or future fee schedules are never applied during payment calculation.

## Acceptance Criteria

Given a Claim_Line__c record is saved with Status__c = 'Pending Adjudication'
When the adjudication engine evaluates the line
Then the system queries Fee_Schedule__c where Fee_Schedule__c.CPT_Code__c = Claim_Line__c.CPT_Code__c AND Fee_Schedule__c.Effective_Date__c <= Claim__c.Date_of_Service__c AND Fee_Schedule__c.Expiration_Date__c >= Claim__c.Date_of_Service__c AND Fee_Schedule__c.LOB__c = Policy__c.Line_of_Business__c

Given exactly one matching Fee_Schedule__c record is found
When the allowed amount is calculated
Then Claim_Line__c.Allowed_Amount__c is set to Fee_Schedule__c.Allowed_Amount__c and Claim_Line__c.Fee_Schedule_Applied__c is populated with the Fee_Schedule__c record ID

Given no matching Fee_Schedule__c record exists for the date of service
When adjudication attempts to proceed
Then Claim_Line__c.Status__c is set to 'Pended' and Claim_Line__c.Pend_Reason__c = 'No Active Fee Schedule' and a Task is created assigned to the Claims Operations Manager queue

Given multiple Fee_Schedule__c records match (overlapping effective dates)
When the engine detects the conflict
Then adjudication is halted, Claim__c.Status__c is set to 'Pended - Fee Schedule Conflict', and a Platform Event (Fee_Schedule_Conflict__e) is published with Claim__c.Id and conflicting Fee_Schedule__c IDs for SIU review

Given Claim_Line__c.CPT_Code__c represents an unlisted procedure (CPT starting with '99499')
When fee schedule lookup occurs
Then the system applies the Fee_Schedule__c.Unlisted_Procedure_Rate__c override and logs Claim_Line__c.Adjudication_Notes__c = 'Unlisted procedure rate applied'

## Solution

Implement an Apex invocable method 'FeeScheduleValidationService.validateLine()' called from a Record-Triggered Flow on Claim_Line__c (trigger: before update, entry criteria: Status__c changes to 'Pending Adjudication'). The Apex class performs a SOQL query with date-range filtering against Fee_Schedule__c, handles the single-match, no-match, and multi-match scenarios, and publishes Fee_Schedule_Conflict__e Platform Events via EventBus.publish(). A separate Platform Event-triggered Flow consumes Fee_Schedule_Conflict__e to create SIU review Tasks. Add a custom index on Fee_Schedule__c.CPT_Code__c and Fee_Schedule__c.Effective_Date__c to optimize query performance. Unit test coverage target: 90%.

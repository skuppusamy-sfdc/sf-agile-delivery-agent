## Description

As a Broker Relations Coordinator, I need a guided OmniScript onboarding wizard for new group/employer accounts so that I can capture all required Group_Service_Agreement__c data, generate the GSA contract document via Conga CLM, and route it for DocuSign e-signature — all within Salesforce — reducing onboarding cycle time from 12 days to under 3 days.

The OmniScript must support three LOBs: Commercial Health, Dental, and Vision. Each LOB has distinct GSA template mappings in Conga CLM.

## Acceptance Criteria

Given a Broker Relations Coordinator initiates the 'New Group Onboarding' OmniScript from an Account (Business Account) record,
When Step 1 (Employer Details) is completed,
Then the OmniScript must validate that Account.TIN__c is unique across existing Group accounts and display an inline error if a duplicate is found.

Given the coordinator selects one or more LOBs in Step 2,
When 'Commercial Health' is selected,
Then additional fields must appear: Effective_Date__c, Renewal_Date__c, Funding_Type__c (Fully Insured / Self-Funded / Level Funded), and TPA__c lookup.

Given all OmniScript steps are completed and validated,
When the coordinator clicks 'Generate Contract',
Then a DataRaptor Transform must map OmniScript data to a Conga CLM template (selected by LOB via Custom Metadata GSA_Template_Map__mdt) and generate a draft Group_Service_Agreement__c record with GSA.Status__c = 'Pending Signature'.

Given a GSA record in Status__c = 'Pending Signature' is created,
When the Conga CLM document generation completes,
Then a DocuSign envelope must be automatically sent to the employer's primary contact (Account.Primary_Contact__c email) and the Broker of Record (GSA.Broker__r.Email) for co-signature within 5 business days.

Given the DocuSign envelope is fully executed (all parties signed),
When the DocuSign callback webhook fires,
Then GSA.Status__c must update to 'Active', GSA.Executed_Date__c must be stamped with today's date, and the signed PDF must be stored as a ContentVersion linked to the GSA record.

Given the DocuSign envelope is declined or voided,
When the webhook fires,
Then GSA.Status__c must be set to 'Signature Declined' and a Task must be created and assigned to the Broker Relations Coordinator with Subject = 'GSA Signature Declined - Action Required'.

-Given a GSA is for a Self-Funded employer, auto-create a TPA__c junction record linking the TPA account- (deprecated: TPA linking moved to post-onboarding workflow per compliance review)

## Solution

Build OmniScript 'GroupEmployerOnboarding' (6 steps) with DataRaptor Extract actions for duplicate TIN check and LOB-specific field rendering using Conditional Views. Use a DataRaptor Transform to prepare the Conga CLM merge payload. Invoke Conga CLM document generation via Integration Procedure calling the Conga REST API. DocuSign envelope creation handled by a second Integration Procedure step using DocuSign eSignature REST API. A Record-Triggered Flow on Group_Service_Agreement__c monitors Status__c changes from the DocuSign callback (handled via a custom REST endpoint Apex class). Store template mappings in GSA_Template_Map__mdt Custom Metadata.

## Description

As a Claims Operations Manager, I need the Agentforce AI agent to automatically triage incoming FNOL submissions and generate a fraud risk score on the newly created Claim__c record so that the SIU team is immediately alerted to high-risk claims and routine claims are routed for straight-through processing without manual review.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.FNOL_Received__c = true,
When the Agentforce Claims Triage agent is invoked via a Record-Triggered Flow,
Then the agent must analyze Claim__c.Loss_Description__c, Claim__c.Loss_Date__c, Policy__c.Policy_Inception_Date__c, and claimant loss history (prior Claim__c records) and return a structured JSON response.

Given the Agentforce response is received,
When the triage result is written back to Salesforce,
Then Claim__c.AI_Fraud_Risk_Score__c (0-100 numeric), Claim__c.AI_Triage_Category__c ('Routine'|'Complex'|'SIU Referral'), and Claim__c.AI_Triage_Rationale__c (text summary) must be populated.

Given Claim__c.AI_Triage_Category__c = 'SIU Referral',
When the record is saved,
Then a Task must be auto-created assigned to the SIU queue with Subject = 'AI-Flagged: SIU Review Required - ' + Claim__c.Claim_Number__c and a related Case of RecordType 'SIU_Investigation' must be opened.

Given Claim__c.AI_Triage_Category__c = 'Routine',
When the triage completes,
Then Claim__c.Status__c must advance to 'In Adjudication' and the straight-through processing flow must be triggered automatically.

Given any Agentforce invocation,
When the agent call completes (success or failure),
Then an AI_Audit_Log__c record must be written with AI_Audit_Log__c.Claim__c, AI_Audit_Log__c.Model_Version__c, AI_Audit_Log__c.Invocation_Timestamp__c, AI_Audit_Log__c.Response_Latency_ms__c, and AI_Audit_Log__c.Raw_Response__c for compliance traceability.

## Solution

Configure an Agentforce Agent Action mapped to a custom Apex invocable 'ClaimsFNOLTriageAction' that builds a prompt context from Claim__c and related Policy__c/prior claims data and calls the Einstein LLM via the Agentforce Agent API. The response JSON is parsed in Apex and fields written back to Claim__c. A Record-Triggered Flow (after insert on Claim__c, condition: FNOL_Received__c = true) invokes the Apex action asynchronously via a Queueable chain to avoid governor limit issues. SIU task and Case creation handled in the same Queueable. AI_Audit_Log__c written in a finally block to ensure capture regardless of LLM outcome. Model version tracked via Custom Metadata Type 'Agentforce_Config__mdt'.

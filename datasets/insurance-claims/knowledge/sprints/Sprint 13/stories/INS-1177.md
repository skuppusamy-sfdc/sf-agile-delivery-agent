## Description

As a Credentialing Specialist, I need all historical provider credentialing records migrated from the legacy flat-file system (CSV exports from the CAQH integration batch) into Salesforce Credentialing_Case__c objects so that the full credentialing history is available in the platform for audits, re-credentialing workflows, and compliance reporting without requiring dual-system lookups.

## Acceptance Criteria

Given the source CSV contains columns [NPI, TIN, Provider_Name, Credentialing_Status, Initial_Credential_Date, Expiration_Date, CAQH_Profile_ID, Specialty_Code],
When the migration ETL job executes,
Then each row must map to a Credentialing_Case__c record with: Credentialing_Case__c.NPI__c, Credentialing_Case__c.TIN__c, Credentialing_Case__c.Status__c, Credentialing_Case__c.Initial_Credential_Date__c, Credentialing_Case__c.Expiration_Date__c, Credentialing_Case__c.CAQH_Profile_ID__c, and Credentialing_Case__c.Specialty__c populated.

Given the migration runs,
When a CSV row's NPI matches an existing Provider__c.NPI__c in Salesforce,
Then Credentialing_Case__c.Provider__c must be set to that Provider__c.Id via upsert (external ID: Provider__c.NPI__c).

Given a CSV row's NPI does NOT match any existing Provider__c record,
When the migration job processes that row,
Then the row must be written to a migration exception log (Migration_Exception__c) with Migration_Exception__c.Error_Reason__c = 'Provider NPI not found' and skipped from the main upsert.

Given the full migration completes,
When a post-migration validation report is generated,
Then the total Credentialing_Case__c records created must equal source CSV row count minus exception count, and a reconciliation report must be emailed to the Credentialing Specialist team DL.

Given migrated Credentialing_Case__c records,
When Credentialing_Case__c.Expiration_Date__c < TODAY,
Then Credentialing_Case__c.Status__c must be set to 'Expired' regardless of the source CSV value.

## Solution

Use Salesforce Data Loader CLI scripted via a Python wrapper for the bulk upsert operation against Credentialing_Case__c using NPI__c as the external ID. Pre-migration data cleansing step in Python normalizes NPI format (10-digit zero-padded) and validates TIN format (XX-XXXXXXX). Exception rows are inserted into Migration_Exception__c via a secondary Data Loader job. A post-load Apex batch class (CredentialingExpirationSyncBatch) corrects Status__c for expired records. Reconciliation email uses Salesforce Messaging.sendEmail() triggered at batch completion. Full execution logged in a custom Migration_Run__c object for audit trail.

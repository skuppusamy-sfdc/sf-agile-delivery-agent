## Description

As a Credentialing Specialist, I need historical provider credentialing records from the legacy CAQH flat file export to be migrated into Salesforce Credentialing_Case__c records so that the full credentialing history is available for audits, re-credentialing workflows, and compliance reporting without requiring access to the legacy system after cutover.

## Acceptance Criteria

Given the CAQH migration CSV contains a provider NPI,
When the migration job executes,
Then a lookup to Provider__c (Person Account) must be performed on Provider__c.NPI__c and the Credentialing_Case__c.Provider__c lookup must be populated; records with no NPI match must be written to a migration exception log.

Given a matched provider record,
When Credentialing_Case__c is created,
Then the following fields must be populated: Credentialing_Case__c.Credentialing_Type__c (Initial/Re-credentialing), Credentialing_Case__c.Effective_Date__c, Credentialing_Case__c.Expiration_Date__c, Credentialing_Case__c.CAQH_Provider_ID__c, Credentialing_Case__c.Board_Certification__c, Credentialing_Case__c.Sanction_Flag__c, and Credentialing_Case__c.Status__c = 'Migrated'.

Given the NPPES API is available during migration,
When a provider NPI is processed,
Then an NPPES callout must validate NPI status and set Credentialing_Case__c.NPI_Verified__c = True if active, False if deactivated.

Given Credentialing_Case__c.Expiration_Date__c is in the past at time of migration,
When the record is created,
Then Credentialing_Case__c.Status__c must be set to 'Expired - Migrated' and a follow-up Task must be created for the assigned Credentialing Specialist.

Given the full migration run completes,
When the post-migration report is generated,
Then it must show: total records processed, successful inserts, exception count, NPI match rate, and be saved as a ContentDocument linked to the Migration_Run__c tracking record.

-Given a provider has duplicate CAQH entries, merge into a single Credentialing_Case__c- (deprecated — deduplication logic deferred to data quality sprint; duplicates flagged with Credentialing_Case__c.Duplicate_Flag__c = True for manual review)

Given migration data volume of approximately 42,000 records,
When the batch job runs,
Then it must complete within the nightly batch window (10 PM–4 AM EST) without exceeding governor limits.

## Solution

Apex Batch class (CAQHCredentialingMigrationBatch) with batch size of 200. Pre-migration: Data loaded to a staging custom object (Migration_Staging__c) via Salesforce Data Loader. Batch reads staging records, performs NPI lookup against Provider__c, makes synchronous NPPES REST callout per record (with future method fallback for callout governor limit management), and performs insert on Credentialing_Case__c. Exception rows written to Migration_Exception__c object. Post-migration Apex generates ContentDocument report and links to Migration_Run__c. Scheduled via Apex Scheduler for nightly execution window.

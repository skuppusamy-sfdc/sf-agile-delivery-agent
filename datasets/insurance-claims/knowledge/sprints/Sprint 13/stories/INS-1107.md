## Description

As a Claims Operations Manager, I need Coordination of Benefits (COB) waterfall logic implemented on Claim__c so that when a policyholder carries dual coverage, the system correctly identifies the primary and secondary payer, calculates primary liability, and routes the residual balance to the secondary payer workflow — ensuring compliance with NAIC COB model regulations and preventing overpayment. Extends INS-1089 (Policy__c coverage hierarchy).

## Acceptance Criteria

Given a Claim__c record has Claim__c.COB_Indicator__c = TRUE,
When the claim enters adjudication,
Then the COB_Waterfall_Flow (Record-Triggered Flow, After Save, Claim__c.Status__c changes to 'In Adjudication') evaluates Policy__c.COB_Priority__c ('Primary', 'Secondary') across all related Policy__c records linked via the PolicyHolder__c junction.

Given the primary Policy__c is identified,
When adjudication completes on the primary payer,
Then Claim__c.Primary_Payer_Paid__c is stamped and Claim__c.COB_Residual_Balance__c = Claim__c.Billed_Amount__c - Claim__c.Primary_Payer_Paid__c - Claim__c.Primary_Contractual_Adjustment__c.

Given Claim__c.COB_Residual_Balance__c > 0,
When the primary adjudication record is saved,
Then a child Claim__c record (COB Secondary Claim) is auto-created with Claim__c.Parent_Claim__c populated, Claim__c.Payer_Type__c = 'Secondary', and Claim__c.Billed_Amount__c = parent Claim__c.COB_Residual_Balance__c.

Given a secondary Claim__c is created,
When the secondary payer adjudicates,
Then Claim__c.Member_Liability__c = COB_Residual_Balance__c - Secondary_Payer_Paid__c and an Explanation_of_Benefits__c record is generated for both primary and secondary with EOB__c.COB_Sequence__c populated ('Primary', 'Secondary').

-Given COB applies to Medicare Advantage plans, apply CMS crossover claim rules automatically- (deprecated — Medicare Advantage COB deferred to Phase 3 per compliance review)

Given a COB claim is fully adjudicated,
When the member's Policyholder 360 page is viewed,
Then the COB summary panel shows total billed, primary paid, secondary paid, and member liability with drill-down to each EOB.

## Solution

Record-Triggered Flow (COB_Waterfall_Flow) on Claim__c handles primary/secondary identification using related Policy__c.COB_Priority__c. Apex invocable method 'COBResidualCalculator' performs the balance calculation to avoid complex formula recursion. Secondary Claim__c auto-creation via Apex (bulkified, invoked from flow). Custom Metadata Type (COB_Rules__mdt) stores payer-specific COB logic overrides. EOB generation handled by existing EOB_Generator_Batch (INS-1089). LWC COB Summary Panel embedded on Policyholder 360 App Page via Dynamic Forms.

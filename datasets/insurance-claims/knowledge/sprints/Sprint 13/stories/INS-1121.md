## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record so that I can view a provider's credentialing status, active PNA agreements, open claims, and payment history in a single consolidated view without navigating across multiple objects.

This story extends INS-1091 (Provider base record layout) by adding the dynamic FlexCard layer on top of the existing record page.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c (Person Account) record,
When the Provider 360 FlexCard loads,
Then it must display the following data panels:
  - Credentialing_Case__c: most recent case with fields Case_Status__c, Expiration_Date__c, CAQH_Profile_ID__c
  - Provider_Network_Agreement__c: count of Active PNAs, effective date range (Effective_Date__c to Termination_Date__c), and associated Fee_Schedule__c name
  - Claim__c: rolling 90-day count of claims by Claim__c.Status__c (Received, In Adjudication, Adjudicated, Paid)
  - Payment__c: last 3 payment records showing Payment__c.Amount__c, Payment__c.Payment_Date__c, and Payment__c.ERA_Reference__c

Given the provider has an NPI on file (Provider__c.NPI__c),
When the FlexCard renders the header section,
Then the NPI, TIN (Provider__c.TIN__c), and primary specialty (Provider__c.Primary_Specialty__c) must be displayed prominently with copy-to-clipboard affordance.

Given a Credentialing_Case__c with Expiration_Date__c within 90 days exists,
When the FlexCard renders,
Then a yellow warning badge labeled 'Credential Expiring Soon' must appear on the credentialing panel.

Given a Credentialing_Case__c with Expiration_Date__c is past today,
When the FlexCard renders,
Then a red badge labeled 'Credential Expired' must appear and the PNA panel must display a 'Network Eligibility at Risk' inline alert.

-Given the provider is associated with a Group_Service_Agreement__c, display GSA tier and capitation rate on the FlexCard header- (deprecated: GSA panel moved to separate FlexCard per INS-1098)

Given a user without the 'Provider_Finance_Viewer' Permission Set Group,
When the FlexCard loads,
Then the Payment__c panel must be hidden entirely and replaced with a 'Contact Finance for payment details' message.

## Solution

Build OmniStudio FlexCard 'Provider360Card' targeting the Account object (Person Account record type). Use multiple DataRaptors (DR_GetProviderCredentialing, DR_GetActivePNAs, DR_GetRecentClaims, DR_GetRecentPayments) as child actions to pull related data. Implement conditional visibility rules on FlexCard states for expiry badge logic. Use a custom Permission Set Group 'Provider_Finance_Viewer' checked via a DataRaptor Integration action against UserRecordAccess. Deploy FlexCard via OmniStudio deployment tool and embed on the Provider__c Lightning Record Page via App Builder.

## Description

As a Claims Operations Manager, I need the system to automatically validate Coordination of Benefits (COB) sequencing when a Claim__c record is submitted so that primary and secondary payer responsibilities are correctly established before adjudication begins, reducing manual rework and overpayment risk.

Extends INS-1158 (Policy coverage verification flow).

## Acceptance Criteria

Given a Claim__c record is created or updated with Claim__c.Status__c = 'Submitted',
When the record-triggered automation fires,
Then the system must check for the existence of a related Policy__c record where Policy__c.COB_Indicator__c = true and Policy__c.Policy_Status__c = 'Active'.

Given COB_Indicator__c is true on the associated Policy__c,
When more than one Policy__c is linked to the Claimant (via Policyholder__c lookup),
Then Claim__c.COB_Payer_Sequence__c must be set to 'Primary' or 'Secondary' based on the birthday rule logic (Policy__c.Subscriber_DOB__c comparison) per NAIC COB guidelines.

Given COB sequencing is determined,
When Claim__c.COB_Payer_Sequence__c = 'Secondary',
Then Claim__c.Primary_Payer_EOB_Attached__c must equal true before Claim__c.Status__c can advance to 'In Adjudication'; otherwise a validation error 'Primary payer EOB required for secondary COB claims' must block the status transition.

Given Claim__c.COB_Payer_Sequence__c is set to 'Primary',
When adjudication completes and Claim__c.Status__c transitions to 'Adjudicated',
Then a Platform Event (COB_Notification__e) must be published with payload fields: Claim_Id__c, Claimant_Id__c, Allowed_Amount__c, and Plan_Paid_Amount__c to notify downstream secondary payer processing.

Given a COB validation error occurs,
When the Claims Operations Manager reviews the record,
Then Claim__c.COB_Validation_Notes__c must be populated with a timestamped error description and Claim__c.Status__c must be set to 'Pended - COB Review'.

-Given duplicate COB records exist, the system should auto-merge Policy__c COB entries- (deprecated; deferred to Phase 3 deduplication project).

## Solution

Implement a Record-Triggered Flow on Claim__c (entry criteria: Status__c = 'Submitted') that invokes an Apex invocable method (CobSequenceValidator.evaluate()) to perform birthday rule logic against related Policy__c records. The Apex class queries PolicyHolder__c DOB fields and sets COB_Payer_Sequence__c accordingly. A separate before-update Flow enforces the EOB attachment gate. A Platform Event (COB_Notification__e) is published via Apex after adjudication status transition. Custom metadata type (COB_Rules__mdt) stores configurable birthday rule and tie-breaker logic to avoid hardcoding.

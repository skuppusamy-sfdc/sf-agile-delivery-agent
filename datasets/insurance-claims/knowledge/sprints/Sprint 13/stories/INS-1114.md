## Description

As a Claims Operations Manager, I need the system to automatically ingest CMS 835 Electronic Remittance Advice (ERA) files from the payer clearinghouse SFTP and create corresponding Payment__c and Explanation_of_Benefits__c records in Salesforce so that manual ERA reconciliation is eliminated and claim payment status is updated within 15 minutes of file receipt.

## Acceptance Criteria

Given a valid 835 ERA file is deposited on the clearinghouse SFTP at the configured inbound path,
When the MuleSoft polling job fires (every 15 minutes),
Then a Platform Event (ERA_Received__e) must be published with fields: ERA_Received__e.File_Name__c, ERA_Received__e.Transaction_Count__c, ERA_Received__e.Total_Payment_Amount__c.

Given the Platform Event is consumed by the ERA Processing Apex trigger,
When a matching Claim__c is found via Claim__c.Claim_Number__c = 835 CLM segment value,
Then a Payment__c record must be created with Payment__c.Amount__c, Payment__c.Payment_Date__c, Payment__c.Payment_Method__c = 'EFT', Payment__c.ERA_File_Reference__c, and Payment__c.Claim__c lookup populated.

Given a Payment__c record is successfully created,
When all Claim_Line__c records for the parent Claim__c are reconciled,
Then Claim__c.Status__c must be updated to 'Paid' and Claim__c.Payment_Posted_Date__c stamped with the current datetime.

Given an EOB must accompany each payment,
When the Payment__c record is saved,
Then an Explanation_of_Benefits__c record must be auto-created with EOB.Claim__c, EOB.Payment__c, EOB.Allowed_Amount__c, EOB.Paid_Amount__c, EOB.Denial_Reason_Code__c (if applicable), and EOB.Generated_Date__c.

Given an 835 file contains a CLM segment with no matching Claim__c.Claim_Number__c,
When the ERA processor encounters the unmatched record,
Then a Case record of RecordType 'ERA_Exception' must be created and assigned to the Claims Operations queue, with Case.Description populated with the raw CLM segment data.

Given an ERA file fails schema validation,
When the MuleSoft integration logs the error,
Then an alert email must be sent to the Claims Operations Manager role and the error logged in Integration_Log__c with Integration_Log__c.Error_Code__c and Integration_Log__c.File_Name__c.

## Solution

MuleSoft Anypoint integration polls SFTP every 15 minutes, parses X12 835 using the Healthcare EDI connector, and publishes ERA_Received__e Platform Events to Salesforce. An Apex Platform Event trigger (ERAProcessingTrigger) performs bulkified upsert of Payment__c and Explanation_of_Benefits__c records using external ID Claim__c.Claim_Number__c. A Record-Triggered Flow on Payment__c handles downstream Claim__c.Status__c update. Exception cases routed via Apex to the Claims Operations queue. Integration_Log__c records written for audit. All processing governed by Custom Metadata Type 'ERA_Config__mdt' for SFTP path and polling interval configuration.

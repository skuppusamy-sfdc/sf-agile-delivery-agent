## Description

As a Credentialing Specialist, I need historical provider credentialing data from legacy CAQH flat file exports to be migrated into Salesforce Credentialing_Case__c records, linked to the correct Provider__c (Person Account) via NPI matching, so that credentialing history is available in Salesforce prior to go-live and re-credentialing timelines can be accurately tracked.

## Acceptance Criteria

Given a CAQH flat file row with a valid NPI value,
When the migration ETL process runs,
Then a Credentialing_Case__c record is created with:
  - Credentialing_Case__c.Provider__c = matched Provider__c.Id WHERE Provider__c.NPI__c = [file NPI]
  - Credentialing_Case__c.Status__c = mapped from legacy status code per Migration_Status_Map__mdt
  - Credentialing_Case__c.Initial_Credentialing_Date__c populated from file field CRED_INITIAL_DT
  - Credentialing_Case__c.Expiration_Date__c populated from CRED_EXPIRY_DT
  - Credentialing_Case__c.Data_Source__c = 'CAQH Legacy Migration'
  - Credentialing_Case__c.RecordType = 'Historical_Credentialing'

Given a flat file row where the NPI does not match any Provider__c.NPI__c in Salesforce,
When the migration runs,
Then the row is written to a Migration_Error_Log__c record with Error_Type__c = 'NPI Unmatched' and the raw row data captured in Error_Detail__c (long text area).

Given a provider has multiple credentialing rows in the legacy file,
When migration completes,
Then all rows produce individual Credentialing_Case__c records; no deduplication is applied at migration time (deduplication is a post-migration data quality task).

Given migration completes for a full file batch,
When the post-load validation report is generated,
Then the report must show: total rows processed, successful inserts, NPI unmatched errors, and field-level null counts for Expiration_Date__c and Initial_Credentialing_Date__c.

Given Credentialing_Case__c records are created from migration,
When a Credentialing Specialist opens a Provider__c record,
Then migrated historical cases are visible in the Credentialing_Case__c related list filtered by Data_Source__c = 'CAQH Legacy Migration' with a 'Historical' visual indicator.

## Solution

Use Salesforce Data Loader CLI scripted via a Python wrapper for bulk insert of Credentialing_Case__c records. Pre-processing step in Python: parse CAQH fixed-width flat file, normalize date formats to ISO-8601, perform NPI lookup against a pre-exported Provider NPI reference CSV, and apply status code mapping from a configuration JSON (sourced from Migration_Status_Map__mdt export). Unmatched rows written to a separate error CSV and bulk-inserted into Migration_Error_Log__c as a second pass. Migration runs in a dedicated Migration Sandbox first; results validated by Credentialing team before Production execution. A post-migration Apex batch (CredentialingMigrationValidation_Batch) runs reconciliation counts and emails summary report to Amanda Foster and Kevin Nguyen.

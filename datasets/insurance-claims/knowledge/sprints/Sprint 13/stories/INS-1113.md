## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record so that I can instantly view credentialing status, active PNA agreements, network participation, and recent claims activity without navigating across multiple objects.

## Acceptance Criteria

Given a Provider__c record is open in the Provider Relations console,
When the Provider 360 FlexCard loads,
Then it must display: Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Credentialing_Status__c, and the most recent Credentialing_Case__c.Status__c linked via lookup.

Given the provider has one or more active Provider_Network_Agreement__c records,
When the PNA panel renders,
Then each PNA row must show PNA.Agreement_Effective_Date__c, PNA.Expiration_Date__c, PNA.LOB__c, and PNA.Network_Tier__c, color-coded green if status = 'Active', yellow if 'Pending Renewal', red if 'Terminated'.

Given the provider has an associated Fee_Schedule__c,
When the Fee Schedule tab is selected,
Then the FlexCard must display Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, and a count of linked CPT codes, sourced via a DataRaptor Extract from Fee_Schedule_Line__c.

Given a Network_Termination_Case__c exists with Network_Termination_Case__c.Provider__c = current provider,
When the Termination Alerts panel is visible,
Then a prominent warning banner must appear showing Network_Termination_Case__c.Termination_Effective_Date__c and Network_Termination_Case__c.Reason__c.

-Given the CAQH integration returns a last-attestation date older than 365 days, the FlexCard must auto-flag the provider for re-attestation outreach- (deprecated: replaced by PCN automation in INS-1097)

Given the user does not hold the 'Provider_Relations_Full_Access' Permission Set Group,
When they open the FlexCard,
Then PNA financial terms and Fee_Schedule__c rate columns must be hidden via field-level security.

## Solution

Build an OmniStudio FlexCard (ProviderNetwork360Card) using a multi-step DataRaptor Turbo Extract to query Provider__c, Credentialing_Case__c, Provider_Network_Agreement__c, Fee_Schedule__c, and Network_Termination_Case__c in a single composite call. Use Flyout child cards for PNA detail rows. Conditional rendering states handle color-coded status badges. Field visibility enforced via Permission Set Group 'Provider_Relations_Full_Access' and FLS on sensitive fee fields. Deploy via OmniStudio package version-controlled in the Sprint 13 branch.

## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces the provider's active Provider_Network_Agreement__c (PNA) details, associated Fee_Schedule__c rates, and current credentialing status, so that I can answer provider inquiries without navigating across five separate objects.

## Acceptance Criteria

Given a Provider__c record is open in the Lightning App
When the Provider 360 FlexCard loads
Then the card displays: NPI__c, TIN__c, Provider_Specialty__c, Credentialing_Status__c from the Provider__c record

Given the provider has one or more PNA records
When PNA.Status__c = 'Active' and PNA.Effective_Date__c <= TODAY() and PNA.Termination_Date__c >= TODAY()
Then the FlexCard displays PNA.Contract_Number__c, PNA.Network__c, PNA.Effective_Date__c, and PNA.Reimbursement_Model__c in a datatable child card

Given an active PNA is linked to a Fee_Schedule__c
When the specialist expands the PNA row
Then a nested FlexCard displays Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Allowed_Amount__c, and Fee_Schedule__c.Effective_Date__c for up to 50 rows

Given the provider has a Credentialing_Case__c in progress
When Credentialing_Case__c.Status__c IN ('Pending Review', 'Awaiting CAQH', 'Committee Review')
Then a banner is displayed on the FlexCard: 'Active Credentialing Case: [Credentialing_Case__c.CaseNumber]'

-Given the provider is part of a Group/Facility, display parent Facility__c contract summary on the same card-
(Removed — Facility contract summary moved to INS-1093)

Given no active PNA exists
When the FlexCard loads
Then display message: 'No active network agreements found. Contact Network Contracting team.'

## Solution

Build OmniStudio FlexCard 'Provider360_Card' with a primary DataRaptor Extract sourcing from Provider__c (Account) with child relationships to Provider_Network_Agreement__c and Fee_Schedule__c. Use a conditional State on the FlexCard to toggle between 'Active PNA', 'No PNA', and 'Credentialing In Progress' states. Nested child FlexCard 'PNA_FeeSchedule_Card' is embedded for fee schedule drill-down. DataRaptor filters PNA records using SOQL WHERE clause on Status__c and date fields. FlexCard deployed to the Provider__c Lightning Record Page via App Builder. Permission to view Fee_Schedule__c restricted via Field-Level Security on the 'Provider Relations' Permission Set Group.

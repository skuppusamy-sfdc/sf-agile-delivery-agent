## Description

As a Broker Relations Coordinator, I need a guided OmniScript onboarding wizard for new Group/Employer accounts that collects eligibility parameters, assigns a Group_Service_Agreement__c (GSA), and triggers DocuSign execution via Conga CLM so that employer groups can be fully onboarded within 2 business days without manual document preparation. Extends INS-1130 (Broker Portal foundation).

## Acceptance Criteria

Given a Broker Relations Coordinator initiates the 'Group_Onboarding_OmniScript' from an Account record of type 'Employer Group',
When Step 1 (Group Profile) is completed,
Then Account.Group_Effective_Date__c, Account.SIC_Code__c, Account.Eligible_Employee_Count__c, and Account.LOB__c must be captured and validated (Eligible_Employee_Count__c must be >= 2).

Given Step 2 (Plan Selection) is reached,
When the coordinator selects a plan,
Then a DataRaptor Integration Procedure must query active Fee_Schedule__c records filtered by Fee_Schedule__c.LOB__c = Account.LOB__c and Fee_Schedule__c.Effective_Date__c <= TODAY() and present them as a selection list.

Given all OmniScript steps are completed,
When the coordinator clicks 'Submit',
Then a Group_Service_Agreement__c record must be created with GSA__c.Status__c = 'Pending Execution', GSA__c.Employer_Account__c, GSA__c.Effective_Date__c, and GSA__c.Assigned_Fee_Schedule__c populated.

Given the GSA__c record is created with Status__c = 'Pending Execution',
When the Record-Triggered Flow fires on GSA__c insert,
Then a Conga CLM API callout must be initiated to generate the GSA contract document using the 'GSA_Standard_Template' and route it to DocuSign with the employer's primary contact as the first signer.

Given DocuSign returns a completed envelope status,
When the Conga CLM webhook callback is received,
Then GSA__c.Status__c must update to 'Executed', GSA__c.Executed_Date__c = envelope completion date, and a ContentVersion record must be attached to the GSA__c record.

-Given the OmniScript includes a manual PDF upload step for legacy broker submissions,-
-This step is removed as of Sprint 13 per compliance directive CD-2024-09.-

## Solution

Build a multi-step OmniScript 'Group_Onboarding_OmniScript' with DataRaptor Load actions for Account and GSA__c creation. Use a DataRaptor Integration Procedure to fetch active Fee_Schedule__c options. Implement a Record-Triggered Flow (After Insert on Group_Service_Agreement__c) that calls an Apex invocable method 'CongaCLMIntegration.initiateGSAContract()' to POST to the Conga CLM REST API with template ID and signer metadata. Create a Named Credential 'Conga_CLM_Prod' for secure endpoint authentication. Build an inbound REST endpoint (Apex @RestResource) to handle the DocuSign/Conga webhook callback and update GSA__c fields. Store the executed document as ContentVersion linked via ContentDocumentLink.

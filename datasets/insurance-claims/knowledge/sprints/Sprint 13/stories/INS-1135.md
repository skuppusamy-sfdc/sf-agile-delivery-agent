## Description

As an Assignment Coordinator, I need a Policyholder 360 dashboard embedded on the Contact record page that aggregates active Policy__c records, open and recent Claim__c records, and Coordination of Benefits (COB) information so that I can assess a policyholder's full coverage picture during inbound service calls without switching applications.

Extends INS-1105 (Policyholder portal single sign-on).

## Acceptance Criteria

Given a Contact record with Contact.RecordType = 'Policyholder' is opened by a user with the 'Assignment_Coordinator_PSG' Permission Set Group,
When the 'Policyholder 360' LWC component loads on the record page,
Then the component must display all Policy__c records where Policy__c.PolicyHolder__c = Contact.Id and Policy__c.Status__c IN ('Active', 'Pending Renewal') with fields: Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c.

Given the Policyholder has one or more Claim__c records,
When the Claims panel of the dashboard loads,
Then all Claim__c records where Claim__c.Claimant__c = Contact.Id and Claim__c.Date_of_Loss__c >= LAST_N_DAYS:365 must be displayed with Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, and Claim__c.Total_Paid_Amount__c.

Given the policyholder has a COB record on file,
When the COB Summary panel renders,
Then the component must display COB.Primary_Payer__c, COB.Secondary_Payer__c, COB.COB_Verification_Date__c, and a visual indicator (green/yellow/red) based on whether COB.COB_Verification_Date__c is within the last 180 days.

Given the dashboard is loading data for a policyholder with more than 50 claims,
When the Claims panel renders,
Then only the 10 most recent claims (sorted by Claim__c.Date_of_Loss__c DESC) must be shown by default with a 'View All' link navigating to a filtered list view.

Given a user without 'Assignment_Coordinator_PSG' or 'Claims_Ops_Manager_PSG' attempts to access the component,
When the LWC initializes,
Then the component must render an access-denied message and log a Security_Access_Attempt__c record with the user ID and timestamp.

## Solution

Develop a custom LWC 'policyholderDashboard360' using @wire adapters and Apex @AuraEnabled(cacheable=true) controller methods for Policy__c, Claim__c, and COB queries. Implement server-side SOQL with LIMIT clauses and ORDER BY for performance. COB color indicator implemented as a computed property in the LWC JavaScript controller based on date delta logic. Permission check performed in the Apex controller using FeatureManagement.checkPermission() against a custom permission 'View_Policyholder_360'. Security logging via an Apex helper that inserts Security_Access_Attempt__c records in a future method to avoid blocking UI load. Deploy component via Lightning App Builder and restrict via Profile/PSG assignment.

## Description

**Bug Summary:** The CPT code validation rule on Claim_Line__c is throwing a blocking validation error for out-of-network (OON) providers even when the claim is correctly flagged as OON and should bypass in-network Fee_Schedule__c lookups.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record where Claim__c.Network_Status__c = 'Out of Network'.
2. Create a new Claim_Line__c child record and enter a valid CPT code (e.g., '99213') in Claim_Line__c.CPT_Code__c.
3. Set Claim_Line__c.Provider__c to a Provider__c record whose PNA.Status__c = 'Terminated'.
4. Click Save.

**Expected Result:** The record saves successfully. CPT validation against Fee_Schedule__c should be skipped when the parent Claim__c.Network_Status__c = 'Out of Network'.

**Actual Result:** Validation error fires: 'CPT code 99213 is not found in the active Fee Schedule for this provider.' The claim line cannot be saved.

**Environment:** Acme Insurance - Full Sandbox (FS1)
**Regression:** Introduced after deployment of INS-1112 (Fee Schedule Engine refactor) on Sprint 12 Day 6.
**Frequency:** 100% reproducible for all OON claims.

## Solution

Investigate the Apex Trigger 'ClaimLineTrigger' and its handler class 'ClaimLineValidationHandler'. The validation logic in method 'validateCPTAgainstFeeSchedule()' is missing a guard clause checking the parent Claim__c.Network_Status__c field before executing the Fee_Schedule__c SOQL lookup. Add a SOQL query to retrieve the parent Claim__c.Network_Status__c within a trigger context map and short-circuit the validation when value equals 'Out of Network'. Add corresponding unit test coverage in 'ClaimLineValidationHandler_Test' for OON scenarios. Validate fix in FS1 before promoting to UAT.

## Description

As a Claims Operations Manager, I need an Agentforce AI agent embedded in the FNOL intake process to automatically classify incoming claims by LOB (Auto, Medical, Property, Liability) and assign an SIU (Special Investigations Unit) risk score so that high-risk claims are flagged for investigator review before adjudication begins, reducing fraudulent payouts and accelerating straight-through processing for low-risk claims.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Status__c = 'FNOL Received',
When the Agentforce Claims Triage agent is invoked via a Record-Triggered Flow,
Then the agent analyzes the claim description, loss type, claimant history, and provider data to populate Claim__c.LOB__c and Claim__c.AI_Claim_Type__c within 10 seconds.

Given the Agentforce agent has processed the FNOL,
When the SIU risk scoring model runs,
Then Claim__c.SIU_Risk_Score__c is populated as a numeric value (0–100) and Claim__c.SIU_Risk_Band__c is set to 'Low' (0–39), 'Medium' (40–69), or 'High' (70–100).

Given a claim receives an SIU_Risk_Band__c of 'High',
When the triage agent completes,
Then Claim__c.Status__c is set to 'SIU Review Pending', a Case is created and assigned to the SIU queue, and a notification is sent to the SIU supervisor via custom notification type.

Given a claim receives an SIU_Risk_Band__c of 'Low' and LOB__c is populated,
When the triage agent completes,
Then Claim__c.Status__c advances to 'In Review' and the claim is auto-assigned via Omni-Channel to the appropriate claims examiner queue based on LOB.

Given the AI classification is complete,
When a Claims Operations Manager reviews the record,
Then the Agentforce reasoning summary is stored in Claim__c.AI_Triage_Rationale__c (Long Text Area) for full auditability.

Given an Agentforce agent invocation fails due to a service timeout,
When the error is caught,
Then Claim__c.AI_Triage_Status__c = 'Failed – Manual Review Required' and the claim is placed in the manual triage queue without blocking FNOL completion.

## Solution

Configure Agentforce Agent in Salesforce AI Platform with a custom Agent Action (ClaimTriageAction) backed by an Apex invocable method that calls the Einstein AI classification endpoint via Named Credential. The SIU risk model is a custom Einstein Prediction Builder model trained on 3 years of historical claims data with fraud outcome labels. Record-Triggered Flow on Claim__c (After Insert, FNOL status condition) invokes the Agentforce agent asynchronously via @InvocableMethod. Omni-Channel routing config maps LOB values to specific capacity-based queues. AI_Triage_Rationale__c populated via Apex from the agent response payload. Fallback error handling managed in the Apex service class with a try/catch writing to a custom AI_Error_Log__c object.

## Description

As a Compliance Officer, I need the Network_Termination_Case__c workflow to enforce a mandatory 90-day notice period before a Provider_Network_Agreement__c can be set to 'Terminated' status so that Acme Insurance meets contractual and CMS regulatory obligations for provider termination notice. The system must prevent premature termination and generate required written notice documentation automatically.

## Acceptance Criteria

Given a Network_Termination_Case__c is created with Termination_Case__c.Requested_Termination_Date__c populated,
When the record is saved,
Then a validation rule fires if (Requested_Termination_Date__c - TODAY) < 90 days, displaying the error: 'Termination date must be at least 90 days from today per CMS regulatory requirements.'

Given a valid Network_Termination_Case__c is saved with a compliant termination date,
When Termination_Case__c.Status__c = 'Approved',
Then a Record-Triggered Flow generates a Termination_Notice__c document record linked to the case and triggers a Conga template merge to produce the formal provider termination letter (PDF).

Given the termination notice PDF is generated,
When the document is created,
Then it is attached to both the Network_Termination_Case__c and the linked Provider_Network_Agreement__c as Salesforce Files, and Provider_Network_Agreement__c.Termination_Notice_Sent_Date__c = TODAY.

Given the Requested_Termination_Date__c is reached (i.e., TODAY = Requested_Termination_Date__c),
When the nightly scheduled process runs,
Then Provider_Network_Agreement__c.Status__c is updated to 'Terminated', Provider_Network_Agreement__c.Termination_Date__c is set, and Provider__c.Network_Status__c is re-evaluated across all remaining active PNAs.

Given the PNA is terminated,
When any open Claim__c records reference the terminated provider with a Date_of_Service__c after the Termination_Date__c,
Then those Claim__c records are flagged with Claim__c.Network_Status_Flag__c = 'Out of Network – Post Termination' and routed to the Claims Operations Manager queue for manual review.

Given a Network_Termination_Case__c is created,
When the case is at any status,
Then only users with the 'Network_Termination_Manager' Permission Set can edit Termination_Case__c.Requested_Termination_Date__c after initial save.

## Solution

Validation rule on Network_Termination_Case__c enforces 90-day minimum using formula: (Requested_Termination_Date__c - TODAY()) < 90. Record-Triggered Flow (After Update, Status = 'Approved') calls an Invocable Apex method (TerminationNoticeService.generateNotice()) which invokes Conga Composer via REST API to merge the termination letter template and attach the resulting PDF via ContentVersion insert. Nightly Scheduled Apex (PNATerminationBatch) processes records where Requested_Termination_Date__c = TODAY and updates PNA Status__c. A second Record-Triggered Flow on Provider_Network_Agreement__c (After Update, Status = 'Terminated') queries related open Claim__c records and sets the Network_Status_Flag__c via a related list update sub-flow. Field-level security on Requested_Termination_Date__c is restricted via Permission Set to 'Network_Termination_Manager' only.

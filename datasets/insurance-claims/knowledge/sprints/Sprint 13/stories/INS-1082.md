## Description

As a Claims Operations Manager, I need Electronic Remittance Advice (ERA) files received from CMS to be automatically parsed and mapped to Payment__c and Claim_Line__c records in Salesforce so that manual ERA reconciliation is eliminated and payment posting time is reduced from 48 hours to under 2 hours.

## Acceptance Criteria

Given a valid 835 ERA file is received from the CMS integration endpoint,
When the file is processed by the inbound integration service,
Then a Payment__c record must be created with Payment__c.Payment_Amount__c, Payment__c.Payment_Date__c, Payment__c.ERA_Trace_Number__c, and Payment__c.Payment_Method__c = 'EFT'.

Given the ERA contains line-level adjudication detail,
When Payment__c is created,
Then each service line must create or update a corresponding Claim_Line__c record with Claim_Line__c.Paid_Amount__c, Claim_Line__c.Adjustment_Reason_Code__c, and Claim_Line__c.Remit_Status__c = 'Posted'.

Given a Claim__c referenced in the ERA has Status__c = 'Pending Payment',
When all Claim_Line__c records for that claim are updated,
Then a Record-Triggered Flow must set Claim__c.Status__c = 'Paid' and stamp Claim__c.Payment_Posted_Date__c with today's date.

Given an ERA line references a Claim_Line__c CPT code not found in the active Fee_Schedule__c,
When the mapping fails,
Then a Platform Event (ERA_Exception__e) must be published with ERA_Exception__e.Error_Type__c = 'CPT_Not_Found' and the line must be placed in Claim_Line__c.Remit_Status__c = 'Exception'.

Given an ERA file fails schema validation (malformed 835),
When ingestion is attempted,
Then no Payment__c or Claim_Line__c records must be created, and a Case must be auto-created with Subject = 'ERA Ingestion Failure - [Trace Number]' assigned to the Claims Operations queue.

Given successful ERA processing,
When reconciliation is complete,
Then an Explanation_of_Benefits__c record must be generated and linked to the parent Claim__c with EOB.Total_Paid__c and EOB.Generated_Date__c populated.

## Solution

Develop an Apex Batch class (ERA835IngestBatch) triggered by a Platform Event published from the MuleSoft CMS integration layer upon file receipt. The batch parses the 835 payload stored in a ContentDocument, performs CPT-to-Fee_Schedule__c lookups, and performs upsert DML on Payment__c and Claim_Line__c using external ID ERA_Trace_Number__c. A Record-Triggered Flow (RTF_ClaimStatusOnPayment) handles the Claim__c status transition. ERA_Exception__e Platform Events are consumed by a separate Apex trigger that creates exception Cases. EOB generation handled by an invocable Apex method called from the flow.

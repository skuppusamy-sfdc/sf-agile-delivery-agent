## Description

As a Credentialing Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record and trigger a review workflow when a provider's license expiration date is within 60 days or has passed so that the credentialing team can act proactively and maintain compliance with CMS credentialing standards.

## Acceptance Criteria

Given a Provider__c record has Provider__c.License_Expiration_Date__c populated,
When a nightly scheduled process runs,
Then any provider whose license expires within 60 calendar days generates a Provider_Change_Notification__c with PCN.Type__c = 'License Expiration Warning', PCN.Status__c = 'Open', and PCN.Provider__c linked accordingly.

Given a PCN of Type 'License Expiration Warning' is created,
When the record is saved,
Then a Task is auto-created and assigned to the provider's assigned Credentialing Specialist with Subject = 'License Renewal Required – Action Needed' and Due_Date = License_Expiration_Date__c minus 30 days.

Given a provider's License_Expiration_Date__c has passed,
When the nightly job runs,
Then Provider__c.Credentialing_Status__c is updated to 'Lapsed', all active PNA records for that provider have Provider_Network_Agreement__c.Status__c set to 'Suspended', and a PCN with Type__c = 'License Lapsed' is created.

Given a PCN of Type 'License Lapsed' is created,
When the record is inserted,
Then a Platform Event (PCN_Alert_Event__e) is fired, consumed by the Compliance Officer's notification bell and logged in the Compliance_Audit_Log__c object.

Given the provider successfully renews their license and the Credentialing Specialist updates Provider__c.License_Expiration_Date__c to a future date,
When the record is saved,
Then the open PCN is auto-closed with PCN.Status__c = 'Resolved' and suspended PNAs are restored to 'Active' status.

## Solution

Scheduled Apex class (LicenseExpirationMonitorBatch) runs nightly via CronTrigger. Batch queries Provider__c records where License_Expiration_Date__c <= TODAY+60. PCN creation and PNA suspension handled in Apex service class (PCNService.cls) to ensure bulkification. Record-Triggered Flow on Provider_Change_Notification__c handles Task creation on insert. Platform Event (PCN_Alert_Event__e) published via Apex and subscribed to by a CometD-based LWC notification component. Compliance_Audit_Log__c records written via an After-Insert Apex Trigger on Provider_Change_Notification__c.

## Description

As a Claims Operations Manager, I need a Policyholder 360 LWC component embedded on the Account (Policyholder) record page that consolidates active Policy__c records, open and recent Claim__c records, and Payment__c disbursement history so that agents can answer policyholder inquiries from a single screen without switching between related lists.

## Acceptance Criteria

Given a user with the 'Claims_Operations_PSG' Permission Set Group views a Policyholder Account record,
When the Policyholder 360 LWC loads,
Then the following panels render within 3 seconds:
  - Active Policies panel: Policy__c records WHERE Policy__c.Status__c = 'Active' AND Policy__c.Policyholder__c = :recordId, displaying Policy Number, LOB, Effective Date, Expiration Date
  - Open Claims panel: Claim__c records WHERE Claim__c.Status__c NOT IN ('Closed','Withdrawn','Payment Posted') ordered by Claim__c.FNOL_Date__c DESC, limit 10
  - Payment History panel: Payment__c records WHERE Payment__c.Claim__r.Policyholder__c = :recordId AND Payment__c.Payment_Date__c = LAST_N_DAYS:365, displaying amount, date, check/EFT number

Given the policyholder has no active Policy__c records,
When the Active Policies panel renders,
Then an empty state illustration and message 'No active policies found' is displayed — no error or blank space.

Given a claim row in the Open Claims panel is clicked,
When the navigation event fires,
Then the user is navigated to the Claim__c record detail page using NavigationMixin.Navigate.

Given the Payment History panel contains more than 10 records,
When the panel renders,
Then a 'View All' link is visible that opens a filtered list view of Payment__c scoped to the policyholder.

Given a Policy__c record has Policy__c.Expiration_Date__c within the next 30 days,
When the Active Policies panel renders,
Then that policy row is highlighted in amber and displays a 'Renewing Soon' badge.

-Given the LWC displays a COB indicator column in the Open Claims panel- (deprecated — COB details moved to Claim__c record page per UX review)

## Solution

Build 'policyholderDashboard' LWC with three child components: 'activePoliciesPanel', 'openClaimsPanel', 'paymentHistoryPanel'. Each child uses @wire with a custom Apex controller (PolicyholderDashboardController.cls) exposing @AuraEnabled(cacheable=true) methods with SOQL queries enforcing WITH SECURITY_ENFORCED. Implement Lightning Data Service for real-time refresh on Claim__c status changes. Use lightning-card with lightning-badge for the 'Renewing Soon' indicator driven by a getter comparing Expiration_Date__c to today. Deploy via Flexipage using App Builder; add to the 'Policyholder_Account_Page' Lightning Record Page. Sharing is governed by existing OWD and Sharing Rules on Claim__c and Policy__c — no custom sharing logic required.

## Description

As a Broker Relations Coordinator, I need to send a Group_Service_Agreement__c (GSA) contract for eSignature via DocuSign directly from the Salesforce GSA record page, so that employer group onboarding contracts are executed digitally without leaving Salesforce or using manual email workflows.

## Acceptance Criteria

Given a GSA record with GSA_Status__c = 'Pending Signature' and all required fields populated (Group_Name__c, Effective_Date__c, LOB__c, TPA__c)
When the Broker Relations Coordinator clicks 'Send for eSignature' on the GSA record
Then a DocuSign envelope is created using the 'GSA_Master_Template_v2' DocuSign template and sent to the contact(s) listed in GSA_Signatory_Contact__c

Given the DocuSign envelope is sent
When the envelope status is 'Sent'
Then GSA_Status__c updates to 'Awaiting Signature', DocuSign_Envelope_ID__c is populated on the GSA record, and a timestamp is written to GSA_Sent_For_Signature_Date__c

Given the signatory completes signing in DocuSign
When the DocuSign webhook callback is received by Salesforce
Then GSA_Status__c = 'Executed', GSA_Executed_Date__c = signature completion date, and the signed PDF is stored as a ContentDocument linked to the GSA record

Given the envelope is declined or voided by the signatory
When the webhook callback indicates status 'Declined' or 'Voided'
Then GSA_Status__c = 'Signature Declined', a Task is created for the Broker Relations Coordinator, and a Chatter notification is posted to the GSA record

-Given the GSA requires dual signatures (employer + TPA), both signatories must sign before status updates to 'Executed'-
(Dual-signature workflow deferred — single signatory flow only for Sprint 13; dual-sig tracked in INS-1101)

## Solution

Implement DocuSign eSignature for Salesforce (managed package) with a custom LWC button 'SendGSAForSignature' on the GSA record page. Button triggers an Apex controller 'GSADocuSignController.cls' that constructs the DocuSign EnvelopeDefinition using the template GUID stored in Custom Metadata 'DocuSign_Template_Config__mdt'. DocuSign Connect webhook is configured to POST status updates to a Salesforce Site endpoint backed by 'DocuSignWebhookHandler.cls'. Handler uses Platform Events to asynchronously update GSA__c fields and store the completed document via ContentVersion insert. Permission Set 'DocuSign_GSA_User' grants access to the Send button.

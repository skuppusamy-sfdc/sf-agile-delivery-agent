## Description

As a Broker Relations Coordinator, I need a guided OmniScript for onboarding new group employer accounts that collects Group_Service_Agreement__c details, validates TIN against IRS format, and triggers an automated eligibility seeding call to the employer's Workday HR system so that member Policy__c records are created in bulk at group inception without manual data entry.

Extends INS-1170 (GSA contract template library).

## Acceptance Criteria

Given the Broker Relations Coordinator initiates the Group Onboarding OmniScript,
When Step 1 (Employer Details) is completed,
Then Group_Service_Agreement__c.Employer_Name__c, Group_Service_Agreement__c.TIN__c, Group_Service_Agreement__c.Effective_Date__c, Group_Service_Agreement__c.Group_Size__c, and Group_Service_Agreement__c.LOB__c must be captured and validated; TIN__c must match regex pattern '^\d{2}-\d{7}$' or the step must not advance.

Given valid employer details are entered,
When Step 2 (Plan Selection) renders,
Then available Plan__c records filtered by GSA.LOB__c and GSA.Effective_Date__c must be presented as a selectable list; at least one plan must be selected to proceed.

Given plan selection is confirmed,
When Step 3 (Workday Integration) executes,
Then the Integration Procedure must call the Workday REST API endpoint (GET /workers?employer_tin={TIN}) and return a list of eligible employees; the employee count must be displayed to the coordinator for confirmation before Policy__c creation.

Given the coordinator confirms the employee list,
When the OmniScript final step submits,
Then one Policy__c record per employee must be created via bulk DataRaptor Load with Policy__c.Group_Service_Agreement__c populated, Policy__c.Policy_Status__c = 'Pending Activation', Policy__c.LOB__c from GSA, and Policy__c.Effective_Date__c = GSA.Effective_Date__c.

Given Policy__c bulk creation completes,
When the job finishes,
Then GSA.Onboarding_Status__c must be set to 'Eligibility Seeded', GSA.Member_Count__c must equal the count of created Policy__c records, and a confirmation email must be sent to the Broker Relations Coordinator and the employer's HR contact (GSA.HR_Contact_Email__c).

Given the Workday API call fails or returns a non-200 response,
When the Integration Procedure error handler fires,
Then the OmniScript must display an inline error message, GSA.Onboarding_Status__c must be set to 'Integration Error', and a Case must be auto-created with Case.Subject = 'Workday Eligibility Seeding Failed - [GSA.Group_Number__c]' assigned to the Assignment Coordinator queue.

-Given Group_Size__c > 500, a manual underwriting review step should be inserted into the OmniScript- (deprecated; underwriting review now handled by separate Underwriting_Review__c workflow per INS-1171).

## Solution

Build a multi-step OmniStudio OmniScript (GroupEmployer_Onboarding_OS) with 5 steps. Step 3 uses an Integration Procedure (Workday_Eligibility_Fetch_IP) with an HTTP Action calling the Workday REST API (Named Credential: Workday_HR_NC). TIN validation implemented as a custom OmniScript validation formula. Policy__c bulk creation uses a DataRaptor Load configured for batch mode (500 records/chunk) to respect DML governor limits. Error handling in the Integration Procedure uses a Conditional Block to route to error state. Confirmation email uses an OmniScript Email Action referencing a Lightning Email Template. Named Credential stores OAuth 2.0 client credentials for Workday. A custom Permission Set Group 'Broker_Onboarding_Users' controls OmniScript access.

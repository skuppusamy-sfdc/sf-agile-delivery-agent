## Description

As a Broker Relations Coordinator, I need a guided OmniScript for onboarding new group/employer accounts that captures eligibility data, generates a Group_Service_Agreement__c draft, and routes the GSA for eSignature via DocuSign so that the onboarding process is standardized and the average contracting cycle time is reduced from 10 business days to 4. Extends INS-1051.

## Acceptance Criteria

Given a Broker Relations Coordinator initiates the 'Group Employer Onboarding' OmniScript from an Account record of type 'Employer Group',
When Step 1 (Employer Details) is completed,
Then Account.Group_Effective_Date__c, Account.SIC_Code__c, Account.Enrolled_Members__c, and Account.TPA_Name__c must be captured and saved.

Given Step 2 (Plan Selection) is reached,
When the coordinator selects one or more LOBs,
Then the OmniScript must dynamically display only Fee_Schedule__c records where Fee_Schedule__c.LOB__c matches the selected value and Fee_Schedule__c.Status__c = 'Active'.

Given all required steps are completed,
When the coordinator clicks 'Generate GSA',
Then a Group_Service_Agreement__c record must be created with GSA.Status__c = 'Draft', GSA.Effective_Date__c, GSA.Employer_Account__c, and GSA.LOB__c populated.

Given a GSA record is in Status__c = 'Draft',
When the 'Send for Signature' action is triggered,
Then the DocuSign eSignature envelope must be sent to the contact designated as GSA.Primary_Signatory__c with the Conga-merged GSA PDF attached.

Given DocuSign returns a 'Completed' webhook callback,
When the callback is processed,
Then GSA.Status__c must be updated to 'Executed', GSA.Executed_Date__c stamped, and a Task created for the Broker Relations Coordinator to confirm member eligibility upload.

-Given the OmniScript includes a step for legacy paper form upload- (deprecated — paper ingestion removed per compliance team decision, digital-only workflow enforced)

Given DocuSign returns a 'Declined' status,
When the callback is processed,
Then GSA.Status__c must be set to 'Declined' and a notification must be sent to the assigned Broker Relations Coordinator via Salesforce bell notification.

## Solution

OmniScript (GroupEmployerOnboarding_OS) built with 6 steps using DataRaptors for dynamic Fee Schedule lookup (DR_GetActiveFeeSchedules). Conga CLM template mapped to GSA fields for PDF generation. DocuSign for Salesforce managed package used for envelope send; a custom Apex REST endpoint (/services/apexrest/docusign/callback) handles webhook callbacks and invokes a Flow (Flow_GSAStatusUpdate) to update GSA__c and create follow-up Tasks. Permission Set Group 'Broker_Relations_Coordinator' grants OmniScript access.

## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record that surfaces real-time credentialing status, active PNA agreements, open Credentialing_Case__c records, and network participation summary so that I can resolve provider inquiries without navigating across multiple objects and tabs.

## Acceptance Criteria

Given a Provider__c record is opened by a user with the 'Provider Relations' Permission Set Group,
When the Provider 360 FlexCard renders,
Then the following data blocks must be visible:
  - Provider NPI (Provider__c.NPI__c), TIN (Provider__c.TIN__c), and primary specialty
  - Active PNA count sourced from Provider_Network_Agreement__c WHERE Status__c = 'Active' AND Provider__c = :recordId
  - Open Credentialing_Case__c records WHERE Status__c NOT IN ('Closed','Withdrawn')
  - Most recent CAQH attestation date (Provider__c.CAQH_Attestation_Date__c)

Given the provider has one or more active Network_Termination_Case__c records,
When the FlexCard renders,
Then a prominent warning banner displaying 'Pending Termination' must appear in amber with the earliest Termination_Effective_Date__c value.

Given a user clicks the 'View Fee Schedules' action on the FlexCard,
When the action fires,
Then an OmniScript modal opens listing all Fee_Schedule__c records linked to the provider's active PNA agreements, grouped by LOB.

-Given the FlexCard displays claim payment history inline, Then Payment__c records for the last 90 days are shown in a child datatable- (deprecated — moved to separate Claims Payment tab per INS-1171)

Given the FlexCard is viewed on a mobile form factor,
When the screen width is below 768px,
Then all data blocks stack vertically and the warning banner remains pinned at top.

## Solution

Implement using OmniStudio FlexCard with a parent DataRaptor Turbo Extract querying Account (Provider__c), child DataRaptor List actions for PNA, Credentialing_Case__c, and Network_Termination_Case__c. Conditional rendering states handle the termination banner via FlexCard State conditions on aggregated termination count. The 'View Fee Schedules' action triggers an OmniScript (Fee_Schedule_Viewer_OS) passed the PNA Id list as an input JSON array. Deploy via OmniStudio Migration Tool to Production. Assign FlexCard visibility via Permission Set Group: Provider_Relations_PSG.

## Description

As a Broker Relations Coordinator, I need the Group/Employer onboarding workflow to automatically generate a Group_Service_Agreement__c (GSA) document via Conga CLM, route it for electronic signature through DocuSign, and upon employer countersignature automatically create the associated Policy__c records for enrolled members so that group onboarding cycle time is reduced from 15 days to under 5 days.

## Acceptance Criteria

Given a Group_Service_Agreement__c record where GSA__c.Status__c = 'Approved' and GSA__c.Effective_Date__c is populated,
When the Broker Relations Coordinator clicks 'Send for Signature',
Then Conga CLM must merge the GSA template using GSA__c field data and the resulting document must be sent to the employer contact via DocuSign within 2 minutes.

Given the DocuSign envelope is sent,
When GSA__c.DocuSign_Status__c is updated to 'Sent' via DocuSign webhook callback,
Then GSA__c.Status__c must update to 'Pending Signature' and a follow-up Task must be created for the Broker Relations Coordinator due in 3 business days.

Given the employer completes their DocuSign signature,
When DocuSign webhook fires with envelope status = 'Completed',
Then GSA__c.Status__c must update to 'Executed', GSA__c.Executed_Date__c = signature timestamp, and the countersigned PDF must be stored as a Salesforce File attached to the GSA__c record.

Given GSA__c.Status__c = 'Executed',
When the Policy__c creation automation runs,
Then one Policy__c record must be created per enrolled member in GSA__c related list GSA_Member__c, with Policy__c.Policy_Effective_Date__c = GSA__c.Effective_Date__c, Policy__c.LOB__c = GSA__c.Line_of_Business__c, and Policy__c.Status__c = 'Active'.

Given Policy__c records are bulk-created from a GSA with 500+ members,
When the creation job completes,
Then all Policy__c records must be created within 10 minutes and GSA__c.Policy_Creation_Status__c must be set to 'Complete' with GSA__c.Policy_Count__c = count of created policies.

-Given GSA has TPA__c populated, route DocuSign envelope to TPA contact for co-signature before employer- (deprecated — TPA co-signature moved to separate TPA addendum flow per legal review)

## Solution

Conga CLM template 'GSA_Standard_Template_v3' configured with merge fields mapped to GSA__c fields. DocuSign for Salesforce managed package handles envelope creation and webhook callbacks updating GSA__c.DocuSign_Status__c via connected DocuSign Status object. Record-Triggered Flow 'GSA_Policy_Creation_Flow' fires on GSA__c update when Status__c changes to 'Executed', calls Apex invocable method PolicyBulkCreator.createPoliciesFromGSA() which queries GSA_Member__c records and performs Bulk API insert of Policy__c. Platform Event GSA_Executed__e published on completion to trigger downstream notifications. File attachment handled via ContentVersion insert in Apex with DocuSign completed document retrieved via DocuSign REST API callout. Async processing via Queueable Apex for groups > 200 members.

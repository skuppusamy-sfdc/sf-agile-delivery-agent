## Description

As a Claims Operations Manager, I need the system to automatically reconcile inbound Electronic Remittance Advice (ERA) files against open Payment__c records so that manual payment posting effort is reduced by at least 70% and Claim__c records are updated to 'Adjudicated' status within minutes of ERA receipt.

## Acceptance Criteria

Given an ERA 835 file is received via the MuleSoft integration layer,
When the file is parsed and a Platform Event 'ERA_Received__e' is published,
Then an Apex trigger subscriber must create or update Payment__c records with Payment__c.ERA_Trace_Number__c, Payment__c.Paid_Amount__c, Payment__c.Payment_Date__c, and Payment__c.Remittance_Status__c = 'Posted' within 60 seconds.

Given a Payment__c record is successfully reconciled,
When all associated Claim_Line__c records have Claim_Line__c.Adjudicated_Amount__c populated,
Then the parent Claim__c.Status__c must automatically transition to 'Adjudicated' via a Record-Triggered Flow.

Given an ERA contains a claim with adjustment reason code CO-45 (Charge exceeds fee schedule),
When the reconciliation process evaluates the Claim_Line__c,
Then Claim_Line__c.Adjustment_Reason_Code__c must be set to 'CO-45' and Claim_Line__c.Write_Off_Amount__c must be calculated as (Claim_Line__c.Billed_Amount__c - Claim_Line__c.Adjudicated_Amount__c).

Given an ERA trace number does not match any existing Payment__c record,
When the reconciliation Apex job processes the event,
Then an Exception_Log__c record must be created with Exception_Log__c.Type__c = 'ERA_Unmatched' and a task must be assigned to the Claims Operations queue.

Given the ERA reconciliation batch runs,
When processing exceeds 10,000 records in a single execution,
Then the Apex Batchable job must chunk records into batches of 200 and log Batch_Job_Log__c.Records_Processed__c upon completion.

Given a Payment__c.Remittance_Status__c transitions to 'Posted',
When the EOB generation process is triggered,
Then an Explanation_of_Benefits__c record must be auto-created with EOB__c.Claim__c, EOB__c.Payment__c, EOB__c.Issue_Date__c = TODAY(), and EOB__c.Delivery_Method__c defaulting to the Policyholder's preferred contact method.

## Solution

Implement an Apex Platform Event subscriber (trigger on ERA_Received__e) that invokes an Apex Batchable class 'ERA_ReconciliationBatch' to match ERA 835 segments against Payment__c via ERA_Trace_Number__c. Use a Record-Triggered Flow (After Update, optimized, runs once) on Claim_Line__c to roll up adjudicated amounts and transition Claim__c.Status__c. Create a separate invocable Apex method 'EOBGenerator.createFromPayment()' called from the Flow to instantiate Explanation_of_Benefits__c. Publish Dead Letter Queue events for unmatched ERAs to a monitoring Platform Event 'ERA_Exception__e'. Ensure all DML operations respect HIPAA audit fields (Last_Modified_By, Timestamp) via a custom Apex audit trigger.

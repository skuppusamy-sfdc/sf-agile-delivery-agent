## Description

As a Claims Operations Manager, I need the Agentforce Einstein AI panel to surface next-best-action (NBA) recommendations during FNOL intake on the Claim__c record so that claims handlers are guided toward the correct LOB routing, potential SIU referral flags, and COB verification prompts without relying solely on manual judgment. The recommendations should be contextual based on Claim__c, Policy__c, and historical Claim__c patterns.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Status__c = 'FNOL Received'
When the claims handler opens the Claim__c record in the Lightning console
Then the Agentforce NBA panel renders within 3 seconds and displays between 1 and 5 ranked recommendations based on Claim__c.Claim_Type__c, Policy__c.Line_of_Business__c, and Claim__c.Reported_Loss_Amount__c

Given Claim__c.Reported_Loss_Amount__c > 50000 AND Claim__c.Claim_Type__c = 'Liability'
When Agentforce evaluates the claim
Then the top NBA recommendation is 'Escalate to SIU Review' with a confidence score displayed and an action button that sets Claim__c.SIU_Referral_Flag__c = true on click

Given Policy__c.COB_Indicator__c = true on the linked policy
When FNOL intake begins
Then Agentforce surfaces a 'Initiate COB Verification' recommendation with a link to the COB_Verification__c creation screen

Given the claims handler dismisses a recommendation
When the dismiss action is clicked
Then the dismissal is logged to an Agentforce_Action_Log__c record with fields: Claim__c.Id, Recommendation_Type__c, Handler_User__c, Dismissed_Timestamp__c for future model feedback

Given the Agentforce model cannot return a recommendation within 5 seconds due to API timeout
When the timeout threshold is breached
Then the NBA panel displays a graceful fallback message: 'Recommendations temporarily unavailable — please follow standard FNOL checklist' and logs a Platform Event (Agentforce_Timeout__e)

## Solution

Configure an Agentforce Agent Action connected to Einstein Next Best Action with a custom Recommendation Strategy built in Strategy Builder. Create a Recommendation__c object to store NBA outcomes for A/B evaluation. Embed the NBA component on the Claim__c Lightning Record Page via a custom LWC ('claimsFnolNbaPanel') that calls the Einstein NBA API via a wire adapter. Use a Record-Triggered Flow on Claim__c (after insert, Status__c = 'FNOL Received') to publish a Platform Event (FNOL_Intake__e) that the LWC subscribes to via Lightning Message Service to trigger NBA refresh. Implement the timeout fallback in the LWC with a Promise.race() pattern against a 5-second threshold. Log dismissals via an Apex controller method invoked from the LWC.

## Description

As a Claims Operations Manager, I need an automated Explanation of Benefits (EOB) document to be generated and delivered to the policyholder whenever a Claim__c record reaches 'Adjudicated' status, so that members receive timely, accurate benefit summaries without manual intervention from the claims team.

## Acceptance Criteria

Given a Claim__c record where Status__c transitions to 'Adjudicated'
When the Record-Triggered Flow fires on the Claim__c update
Then an Explanation_of_Benefits__c record is created with EOB_Generated_Date__c = TODAY(), linked via Claim__c lookup, and EOB_Status__c = 'Pending Delivery'

Given the EOB record is created
When all associated Claim_Line__c records have Line_Status__c = 'Processed'
Then the EOB document is rendered via Conga CLM template 'EOB_Standard_v3' and stored as a ContentDocument linked to the Claim__c

Given the EOB document is generated
When the policyholder has a valid email on Contact.Email
Then a platform event EOB_Delivery_Event__e is published and the EOB is emailed to the policyholder within 15 minutes

Given the policyholder's preferred delivery method is 'Portal'
When EOB_Delivery_Event__e is consumed
Then EOB_Status__c is updated to 'Available in Portal' and the document is surfaced on the Policyholder 360 FlexCard

-Given the claim involves COB, a secondary EOB is generated for the secondary payer automatically-
(Deferred to INS-1104 — COB EOB logic out of scope for Sprint 13)

Given any failure in Conga CLM rendering
When the callout returns a non-200 response
Then EOB_Status__c = 'Generation Failed', a Case is auto-created with Subject = 'EOB Generation Failure - [Claim__c.Name]', and assigned to the Claims Operations queue

## Solution

Implement a Record-Triggered Flow on Claim__c (trigger: after update, condition: Status__c changed to 'Adjudicated'). Flow invokes an Apex action 'EOBGenerationService.generateEOB()' which calls Conga CLM REST API to render the EOB template. On success, the Apex class creates the Explanation_of_Benefits__c record and publishes EOB_Delivery_Event__e via EventBus.publish(). A separate Platform Event-triggered Flow subscribes to EOB_Delivery_Event__e and routes delivery based on Policyholder__c.Preferred_Contact_Method__c. Error handling via custom exception logging to Integration_Log__c. Conga template ID stored in Custom Metadata Type 'EOB_Template_Config__mdt'.

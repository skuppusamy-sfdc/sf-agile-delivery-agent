## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record that surfaces real-time credentialing status, active PNA agreements, network participation, and open PCNs so that I can make informed decisions without navigating across multiple objects.

## Acceptance Criteria

Given a Provider__c record is open in Salesforce,
When the Provider 360 FlexCard loads,
Then it must display Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Credentialing_Status__c, and Provider__c.Network_Participation_Status__c in the header band.

Given the provider has one or more active Provider_Network_Agreement__c records,
When the PNA section of the FlexCard renders,
Then all PNAs where PNA__c.Status__c = 'Active' and PNA__c.Effective_End_Date__c >= TODAY() must be listed with LOB, effective dates, and reimbursement model.

Given a Credentialing_Case__c exists for the provider,
When the Credentialing panel loads,
Then Credentialing_Case__c.Status__c, Credentialing_Case__c.CAQH_Profile_Status__c, and Credentialing_Case__c.Expiration_Date__c must be displayed with a RAG (Red/Amber/Green) indicator where expiry < 30 days = Red, 30-90 days = Amber, > 90 days = Green.

Given an open Provider_Change_Notification__c (PCN) record linked to the provider,
When the PCN section renders,
Then all PCNs where PCN__c.Status__c != 'Closed' must appear with PCN__c.Change_Type__c and PCN__c.Effective_Date__c.

-Given the provider has associated Facility__c records, display them in a child tab- (deprecated — Facility view moved to INS-1187)

Given the user's Permission Set Group does not include 'Provider_Financial_Data_PSG',
When the FlexCard loads,
Then Fee_Schedule__c rate columns must be hidden and replaced with 'Access Restricted' label.

## Solution

Build OmniStudio FlexCard 'Provider360_Card' with child DataRaptors for PNA, Credentialing_Case__c, and PCN data. Use DataRaptor Extract DRX_Provider_PNA_Active with SOQL filtering PNA__c.Status__c = 'Active'. Implement conditional visibility rules on FlexCard actions using Merge Field conditions for Permission Set Group check via a custom @AuraEnabled Apex method isUserInPSG(). RAG logic implemented via FlexCard State configuration with threshold values stored in Custom Metadata Type Provider_Card_Thresholds__mdt. Deploy via OmniStudio migration tool to Production.

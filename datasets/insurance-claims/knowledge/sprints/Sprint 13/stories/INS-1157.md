## Description

As a Claims Operations Manager, I need an Agentforce-powered suggested action panel embedded on the Claim__c record page that analyzes FNOL data and historical claim patterns to surface SIU referral risk scores and recommended next actions so that high-risk claims are escalated to the Special Investigations Unit faster and low-risk claims are auto-routed for straight-through processing.

## Acceptance Criteria

Given a Claim__c record with Claim__c.FNOL_Completed__c = true is opened by a claims adjuster,
When the Agentforce Claims Triage panel loads,
Then an AI-generated SIU Risk Score (0–100) is displayed in the panel sourced from the 'ClaimFraudRisk_Topic' Agentforce Topic, with score stored in Claim__c.SIU_Risk_Score__c.

Given Claim__c.SIU_Risk_Score__c >= 75,
When the panel renders the recommended actions,
Then the primary suggested action is 'Refer to SIU' with a pre-populated SIU referral OmniScript launch button, and Claim__c.SIU_Referral_Recommended__c is set to true.

Given Claim__c.SIU_Risk_Score__c < 30 and Claim__c.Total_Claimed_Amount__c < 5000,
When the panel evaluates routing logic,
Then the suggested action is 'Approve for Straight-Through Processing' and the adjuster can accept with a single click, transitioning Claim__c.Status__c to 'Auto-Approved'.

Given the adjuster accepts or overrides an Agentforce suggestion,
When the action is confirmed,
Then an 'Agent_Action_Log__c' record is created capturing Agent_Action_Log__c.Suggested_Action__c, Agent_Action_Log__c.Adjuster_Decision__c (Accepted/Overridden), and Agent_Action_Log__c.Override_Reason__c if applicable.

Given the Agentforce model confidence score is below 0.60,
When the panel renders,
Then a disclaimer banner displays: 'Low confidence prediction — manual review recommended' and no auto-routing action is surfaced.

## Solution

Configure Agentforce 'ClaimFraudRisk_Topic' using Einstein Prediction Builder trained on historical Claim__c and Claim_Line__c data with CPT code anomaly features and provider billing pattern inputs. Embed the Agentforce action panel as a custom LWC ('agentforceClaimsTriage') placed in the Claim__c Lightning Record Page via App Builder. LWC calls Agentforce API via Apex controller 'AgentforceClaimsController'. Use a Record-Triggered Flow to write SIU_Risk_Score__c back to Claim__c on panel confirmation. Agent_Action_Log__c inserts handled by an Apex invocable method called from the LWC. Gate the panel visibility with a Permission Set 'Claims_AI_User_PS' assigned to adjuster profiles.

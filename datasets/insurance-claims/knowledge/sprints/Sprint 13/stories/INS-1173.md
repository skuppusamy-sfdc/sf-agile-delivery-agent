## Description

As a Claims Operations Manager, I need an Agentforce AI action integrated into the claims intake flow that extracts structured data from a free-text FNOL conversation transcript and pre-populates Claim__c fields so that intake agents spend less time on manual data entry and FNOL-to-claim creation time is reduced by at least 40%.

## Acceptance Criteria

Given an intake agent completes an FNOL call and pastes or uploads the call transcript into the Agentforce Claims Intake panel,
When the 'Extract Claim Details' AI action is invoked,
Then the following Claim__c fields are pre-populated with extracted values and displayed for agent review before save:
  - Claim__c.Incident_Date__c
  - Claim__c.Incident_Description__c
  - Claim__c.Claimant_Name__c
  - Claim__c.Policy__c (resolved by Policy Number mentioned in transcript matched to Policy__c.Policy_Number__c)
  - Claim__c.LOB__c
  - Claim__c.Reported_By__c

Given the AI action extracts a date that is in the future,
When the pre-populated fields are reviewed,
Then Claim__c.Incident_Date__c is flagged with an inline warning: 'Extracted date is in the future — please verify.'

Given the AI action cannot confidently match a Policy Number from the transcript (confidence score below 0.80),
When the extraction completes,
Then Claim__c.Policy__c is left blank and a yellow advisory banner reads 'Policy not identified — manual lookup required.'

Given the agent reviews and confirms the pre-populated fields,
When the agent clicks 'Create Claim',
Then a Claim__c record is created with Claim__c.Status__c = 'FNOL Received', Claim__c.FNOL_Date__c = today, and Claim__c.AI_Assisted_Intake__c = TRUE.

Given the Agentforce action is used,
When the Claim__c record is saved,
Then an AI_Interaction_Log__c record is created capturing: transcript snippet hash, extracted field values, confidence scores per field, and agent confirmation timestamp.

## Solution

Configure an Agentforce Agent Action using a custom Apex class (FNOLExtractionAction.cls) decorated with @InvocableMethod, called from the Agentforce Agent Builder topic 'Claims Intake'. The Apex class invokes Einstein Generative AI via the ConnectApi.EinsteinLLM API, passing a structured prompt template stored in Prompt Builder (FNOL_Extraction_Prompt template) that instructs the model to return a JSON payload of extracted claim fields with per-field confidence scores. A Lightning Web Component ('fnolIntakeReview') renders the extracted fields as editable inputs for agent review before Claim__c creation. AI_Interaction_Log__c insert handled in the Apex class after-action callback. Feature flag controlled via Custom Permission 'Agentforce_Claims_Intake_Enabled' to allow phased rollout by user profile.

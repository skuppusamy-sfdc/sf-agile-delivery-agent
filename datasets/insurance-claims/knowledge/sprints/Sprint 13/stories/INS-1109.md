## Description

As a Provider Network Specialist, I need the Network_Termination_Case__c object to drive an automated 90-day termination notice workflow — including DocuSign-based PCN delivery, PNA deactivation scheduling, and downstream Facility__c and Fee_Schedule__c record updates — so that Acme Insurance meets contractual and regulatory notice obligations without manual tracking across spreadsheets.

## Acceptance Criteria

Given a Network_Termination_Case__c is created with Network_Termination_Case__c.Termination_Reason__c populated and Network_Termination_Case__c.Status__c = 'Initiated',
When the record is saved,
Then a Record-Triggered Flow stamps Network_Termination_Case__c.Notice_Sent_Date__c = TODAY() and Network_Termination_Case__c.Effective_Termination_Date__c = TODAY() + 90.

Given Network_Termination_Case__c.Notice_Sent_Date__c is populated,
When the flow executes,
Then a DocuSign envelope is automatically sent to the email address on the related Provider__c.Email containing the Provider_Change_Notification__c (PCN) template, and PCN__c.Status__c = 'Sent', PCN__c.Sent_Date__c = TODAY() are stamped on the newly created PCN record.

Given the DocuSign envelope is acknowledged (PCN__c.DocuSign_Status__c = 'Completed'),
When the Platform Event (DocuSign_Status_Update__e) is received,
Then Network_Termination_Case__c.Status__c transitions to 'Notice Acknowledged' and a task is created for the assigned Provider Network Specialist to confirm termination details.

Given Network_Termination_Case__c.Effective_Termination_Date__c = TODAY() (scheduled check),
When the daily Scheduled Flow (NetworkTermination_EffectiveDate_Check) runs,
Then all related Provider_Network_Agreement__c records where PNA__c.Provider__c = terminated provider are updated to PNA__c.Status__c = 'Terminated' and PNA__c.Termination_Date__c = TODAY().

Given PNA records are terminated,
When the PNA update is committed,
Then all active Fee_Schedule__c records linked to the provider's TIN and LOB are end-dated (Fee_Schedule__c.End_Date__c = Effective_Termination_Date__c) and Facility__c.Network_Status__c is re-evaluated if the provider was the sole contracted provider at that facility.

Given a Network_Termination_Case__c.Termination_Reason__c = 'For Cause',
When the case is initiated,
Then the 90-day notice period is overridden to 0 days, Network_Termination_Case__c.Immediate_Termination__c = TRUE, and an SIU referral task is auto-created and assigned to the SIU queue.

-Given a provider is part of a Group/Employer GSA, cascade termination to all group members automatically- (superseded — group cascade termination scoped to INS-1115 in Sprint 14)

## Solution

Record-Triggered Flow (NetworkTermination_Initiation_Flow) handles notice date calculation, PCN record creation, and DocuSign envelope send via DocuSign for Salesforce managed package API. Platform Event trigger (DocuSign_Status_Update__e) updates case status on acknowledgement. Scheduled Flow (daily at 06:00 UTC) queries Network_Termination_Case__c where Effective_Termination_Date__c = TODAY() and invokes Apex invocable 'TerminationEffectiveDateProcessor' for PNA/Fee Schedule bulk updates (bulkified, handles up to 2,000 records per execution). For-Cause logic handled via Decision element in initiation flow checking Termination_Reason__c. All state transitions governed by a Termination_Case Status Picklist with validation rules preventing backward transitions.

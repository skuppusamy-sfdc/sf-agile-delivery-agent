## Description

As a Broker Relations Coordinator, I need a guided OmniScript for onboarding new group employer accounts that collects plan selection, LOB, and billing details, then automatically generates a draft Group_Service_Agreement__c and triggers a Conga CLM contract generation workflow so that onboarding time is reduced from 5 days to under 24 hours.

## Acceptance Criteria

Given the Broker Relations Coordinator launches the 'GroupEmployerOnboarding_OS' OmniScript from an Account record of type 'Employer',
When Step 1 (Employer Details) is completed,
Then the OmniScript validates that Account.TIN__c is a 9-digit numeric value and Account.SIC_Code__c is present before advancing.

Given Step 2 (Plan & LOB Selection) is reached,
When the coordinator selects one or more LOBs (Medical, Dental, Vision),
Then available Group_Service_Agreement__c templates are filtered by GSA__c.LOB__c matching the selection, presented as a picklist.

Given all OmniScript steps are completed and the coordinator clicks 'Submit',
When the Integration Procedure 'CreateGSAAndTriggerConga_IP' executes,
Then a Group_Service_Agreement__c record is created with GSA__c.Status__c = 'Draft', GSA__c.Effective_Date__c = selected start date, and GSA__c.Employer_Account__c linked.

Given the GSA__c record is created,
When the Conga CLM API callout fires,
Then a Conga contract envelope is generated using the mapped template ID stored in 'Conga_Template_Map__mdt' and the envelope ID is written back to GSA__c.Conga_Envelope_ID__c.

Given the Conga envelope is sent for e-signature via DocuSign,
When all signatories complete signing,
Then GSA__c.Status__c transitions to 'Executed' via a Record-Triggered Flow on the DocuSign status webhook update.

-Given the OmniScript includes a TPA selection step for self-funded groups- (deprecated — TPA assignment moved to post-onboarding workflow per compliance review)

Given the onboarding submission fails due to a Conga API timeout,
When the error is caught in the Integration Procedure,
Then GSA__c.Status__c is set to 'Pending Contract Generation', an error log entry is written to 'Integration_Error_Log__c', and a retry task is created for the coordinator.

## Solution

Build 'GroupEmployerOnboarding_OS' OmniScript with 5 steps using DataRaptor Transform actions for field mapping. Create Integration Procedure 'CreateGSAAndTriggerConga_IP' that calls a named credential 'Conga_CLM_NC' via HTTP Action. Use custom metadata type 'Conga_Template_Map__mdt' keyed on LOB__c to resolve template IDs dynamically. DocuSign status callback handled by an Apex REST endpoint updating GSA__c via a Record-Triggered Flow. Implement exponential backoff retry logic in the Apex callout class 'CongaCLMService'. Permission Set Group 'Broker_Relations_PSG' gates access to the OmniScript.

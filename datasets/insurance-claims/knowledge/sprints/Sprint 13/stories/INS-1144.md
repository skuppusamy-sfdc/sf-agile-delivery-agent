## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record so that I can view credentialing status, active PNA agreements, network participation, and open Credentialing_Case__c records without navigating away from the provider record. This story extends INS-1114 which established the base Provider data model.

## Acceptance Criteria

Given a Provider__c record is open in Salesforce
When the Provider 360 FlexCard loads
Then the card displays Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Credentialing_Status__c, and Provider__c.Primary_Specialty__c in the header band

Given the provider has one or more active Provider_Network_Agreement__c records
When the PNA section of the FlexCard renders
Then all PNAs where PNA__c.Status__c = 'Active' and PNA__c.Effective_Date__c <= TODAY() and PNA__c.Termination_Date__c >= TODAY() are listed with LOB, network tier, and reimbursement model

Given the provider has an open Credentialing_Case__c
When the credentialing panel loads
Then Credentialing_Case__c.Status__c, Credentialing_Case__c.CAQH_Profile_Status__c, and Credentialing_Case__c.Expiration_Date__c are displayed with a red badge if expiration is within 90 days

Given the FlexCard is viewed by a user without the 'Provider_Network_Admin' Permission Set Group
When the TIN section attempts to render
Then Provider__c.TIN__c is masked as '***-**-XXXX' and no unmasked value is exposed in the DOM

-Given a link to QGenda scheduling is shown on the FlexCard- (deprecated — QGenda deep-link deferred to Sprint 15)

Given the provider's CAQH profile was last verified more than 180 days ago
When the FlexCard header renders
Then a warning banner displays: 'CAQH re-attestation required' with a timestamp of Provider__c.CAQH_Last_Verified__c

## Solution

Implement an OmniStudio FlexCard ('Provider360Card') with three child DataRaptors: (1) DR_Provider_Header pulling from Provider__c and related Credentialing_Case__c via SOQL, (2) DR_PNA_List pulling active PNA__c records filtered by status and date range, (3) DR_Open_Cases pulling any Credentialing_Case__c or Network_Termination_Case__c in non-terminal states. Use a Flyout action card for PNA detail drill-down. Conditional visibility states on the TIN field driven by a Custom Permission check within the FlexCard JSON. Deploy via OmniStudio package to Production using the Metadata API pipeline. Add a Platform Cache partition for CAQH_Last_Verified__c to reduce repeated callout latency.

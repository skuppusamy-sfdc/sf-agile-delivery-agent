# INS-1084: Bug: Claim__c.Status__c incorrectly set to 'Closed' when COB secondary payer adjudication is pending

## Description

**Bug Report — COB Secondary Adjudication Race Condition**

**Steps to Reproduce:**
1. Create a Claim__c record with Claim__c.COB_Indicator__c = 'True' and Claim__c.Primary_Payer__c populated.
2. Submit the primary payer ERA via the ERA ingestion batch (INS-1082 flow).
3. Observe that the Record-Triggered Flow RTF_ClaimStatusOnPayment fires immediately upon primary Payment__c creation.
4. Navigate to the Claim__c record.
5. Observe Claim__c.Status__c = 'Closed' even though Claim__c.Secondary_Payer_Status__c = 'Pending'.

**Expected Result:** When Claim__c.COB_Indicator__c = 'True', the claim status should transition to 'Pending COB' after primary adjudication, not 'Closed'. Status should only reach 'Closed' once Claim__c.Secondary_Payer_Status__c = 'Adjudicated' or 'Waived'.

**Actual Result:** RTF_ClaimStatusOnPayment unconditionally sets Claim__c.Status__c = 'Closed' upon any Payment__c creation, ignoring COB flag. This causes EOB generation to fire prematurely and secondary payer payment to be blocked by a validation rule requiring Status__c != 'Closed'.

**Environment:** Full Sandbox (Acme-FS1)
**Regression:** Introduced in Sprint 12 deployment (INS-1067 flow changes)
**Severity:** High — affects all COB claims (~18% of claim volume per ops team estimate)

## Acceptance Criteria

As per Description.

Additional verification:
Given Claim__c.COB_Indicator__c = 'True' and primary Payment__c is created,
When RTF_ClaimStatusOnPayment executes,
Then Claim__c.Status__c must be set to 'Pending COB' and NOT 'Closed'.

Given Claim__c.Status__c = 'Pending COB' and a secondary Payment__c is posted with Payment__c.Payer_Type__c = 'Secondary',
When the flow re-evaluates,
Then Claim__c.Status__c must transition to 'Closed' and EOB generation must trigger.

## Solution

Add a decision node to RTF_ClaimStatusOnPayment flow: check Claim__c.COB_Indicator__c before setting terminal status. If COB_Indicator__c = True AND Secondary_Payer_Status__c != 'Adjudicated' AND != 'Waived', route to 'Pending COB' status assignment element instead of 'Closed'. Add a second flow entry condition on Payment__c creation where Payment__c.Payer_Type__c = 'Secondary' to handle the COB completion path. Deploy fix to Acme-FS1 sandbox for QA regression suite execution before promoting to UAT.

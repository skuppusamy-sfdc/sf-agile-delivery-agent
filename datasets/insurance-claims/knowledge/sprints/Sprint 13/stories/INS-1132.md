## Description

As a Claims Operations Manager, I need an Agentforce AI action that can process inbound FNOL requests submitted via the policyholder portal, extract key claim details using a configured prompt template, and create a draft Claim__c record so that first notice of loss handling is accelerated and manual data entry errors are reduced by at least 40%.

## Acceptance Criteria

Given a policyholder submits an FNOL form via the portal with loss description, date of loss, and policy number,
When the Agentforce 'FNOL_Triage_Action' is invoked,
Then the AI action must extract Claim__c.Date_of_Loss__c, Claim__c.Loss_Description__c, Claim__c.LOB__c, and Claim__c.Policy__c (via policy number lookup) and populate a draft Claim__c with Claim__c.Status__c = 'Draft'.

Given the FNOL loss description contains keywords associated with potential fraud (e.g., 'fire', 'total loss', 'multiple claimants'),
When the Agentforce action evaluates the description against the configured SIU keyword list in Custom Metadata 'SIU_Keywords__mdt',
Then Claim__c.SIU_Referral_Flag__c must be set to TRUE and a task assigned to the SIU queue must be created automatically.

Given the extracted policy number does not match any active Policy__c record,
When the Agentforce action attempts the lookup,
Then the draft Claim__c must not be created; instead, a case must be created with Subject = 'FNOL - Unmatched Policy' and assigned to the Claims Operations queue.

Given the draft Claim__c is created successfully,
When the Agentforce action completes,
Then the action must return a structured JSON response to the portal including Claim__c.Id, Claim__c.Claim_Number__c (auto-generated), and an estimated acknowledgment message.

Given the Agentforce action is executed,
When the prompt template is invoked,
Then the AI model must operate within a configured token limit of 2048 and the prompt template 'FNOL_Extraction_Prompt_v2' must be referenced from the Agentforce Prompt Template library.

## Solution

Configure an Agentforce Agent Action of type 'Flow' backed by an Autolaunched Flow 'FNOL_Triage_Flow'. The Flow calls an Apex invocable method 'FNOLExtractionService.extractFNOLDetails()' which invokes the Agentforce Prompt Template API using Einstein Generative AI. Extracted fields are mapped to Claim__c via a Create Records element. SIU keyword matching uses a Custom Metadata lookup within the Apex service. Configure the Agentforce Agent Topic 'Claims Intake' with the FNOL action registered and appropriate guardrails. Secure with Permission Set 'Agentforce_Claims_User'.

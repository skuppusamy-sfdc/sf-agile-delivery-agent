## Description

As a Credentialing Specialist, I need all historical provider credentialing records migrated from the legacy CredentialPro system into Salesforce Credentialing_Case__c objects so that the credentialing team operates from a single system of record post-cutover, with no loss of attestation history, license expiry dates, or CAQH profile linkages.

## Acceptance Criteria

Given the migration ETL extracts 14,200 provider credentialing records from CredentialPro,
When the data is loaded into Salesforce via Data Loader,
Then each Credentialing_Case__c must have: Credentialing_Case__c.Provider__c (matched by NPI via Provider__c.NPI__c), Credentialing_Case__c.CAQH_Provider_ID__c, Credentialing_Case__c.Initial_Credentialing_Date__c, Credentialing_Case__c.Recredentialing_Due_Date__c, and Credentialing_Case__c.Status__c mapped per the status translation matrix in the migration runbook.

Given the migration includes license records,
When Credentialing_Case__c records are created,
Then at least one child Provider_License__c record must exist per case, with Provider_License__c.License_Number__c, Provider_License__c.State__c, Provider_License__c.Expiration_Date__c, and Provider_License__c.License_Type__c populated.

Given a Credentialing_Case__c.Recredentialing_Due_Date__c is within 90 days of migration date,
When the record is inserted,
Then Credentialing_Case__c.Priority__c must automatically be set to 'High' via a Record-Triggered Flow.

Given the full migration load completes,
When a post-load reconciliation script runs,
Then record count in Salesforce must equal source count ± 0 (zero tolerance), and a migration audit report must be generated in Migration_Audit__c with Migration_Audit__c.Source_Count__c, Migration_Audit__c.Target_Count__c, and Migration_Audit__c.Discrepancy_Count__c.

Given providers with CAQH profiles,
When the CAQH ProView API is called post-migration,
Then Credentialing_Case__c.CAQH_Last_Attestation_Date__c must be refreshed and any mismatch logged in Integration_Log__c.

-Given DEA certificate data exists in CredentialPro, migrate DEA records to a new DEA_Certificate__c object- (deprecated: DEA migration deferred to Sprint 15 per architecture decision ADR-042)

## Solution

Three-phase migration approach: (1) Talend ETL job extracts CredentialPro data to CSV, applies status translation via lookup table, and validates NPI matching against NPPES API. (2) Salesforce Data Loader upsert using NPI as external ID on Provider__c. Credentialing_Case__c loaded with external ID Credentialing_Case__c.Legacy_CredentialPro_ID__c. Provider_License__c loaded as child records. (3) Post-load: Record-Triggered Flow on Credentialing_Case__c sets Priority__c based on Recredentialing_Due_Date__c; Apex batch job calls CAQH ProView REST API to refresh attestation dates. Migration_Audit__c records written by a post-load Apex script. Full rollback script maintained in Git for cutover safety.

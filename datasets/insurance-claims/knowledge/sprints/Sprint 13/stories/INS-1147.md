## Description

As a Broker Relations Coordinator, I need a guided OmniScript onboarding workflow for new group employer accounts so that I can capture employer demographics, plan selections, and Group_Service_Agreement__c terms, then automatically route the GSA for DocuSign e-signature without manually generating documents in Conga CLM. This reduces onboarding cycle time from an average of 14 days to a target of 3 business days.

## Acceptance Criteria

Given a Broker Relations Coordinator initiates the 'New Group Onboarding' OmniScript from an Account record where Account.RecordType = 'Group Employer'
When Step 1 (Employer Demographics) is completed
Then Account.Name, Account.TIN__c, Account.SIC_Code__c, Account.Group_Size__c, and Account.Effective_Date__c are validated and saved to the Account record before advancing to Step 2

Given Step 2 (Plan Selection) is reached
When the coordinator selects one or more LOB plan options
Then a Group_Service_Agreement__c draft record is created with GSA__c.Status__c = 'Draft', GSA__c.Employer_Account__c populated, and GSA__c.Requested_Effective_Date__c = Account.Effective_Date__c

Given the GSA draft is complete and Step 4 (Review & Sign) is reached
When the coordinator clicks 'Send for Signature'
Then the Conga CLM template 'GSA_Standard_Template_v3' is merged with GSA__c field values and the resulting envelope is dispatched via DocuSign API to GSA__c.Primary_Signatory_Email__c with a 5-business-day expiration

Given the DocuSign envelope status callback is received
When the envelope status = 'Completed'
Then GSA__c.Status__c is updated to 'Executed', GSA__c.Executed_Date__c = TODAY(), and a ContentVersion record linking the signed PDF is attached to the GSA__c record

Given the DocuSign envelope expires without signature
When the expiration webhook fires
Then GSA__c.Status__c = 'Signature Expired', a Task is created for the Broker Relations Coordinator with Subject = 'GSA Signature Follow-Up Required', and an automated email is sent to GSA__c.Primary_Signatory_Email__c

-Given the OmniScript includes a manual PDF upload step for legacy brokers- (deprecated — all brokers must use DocuSign e-signature path per Compliance directive CO-2024-07)

Given a Group_Size__c value exceeds 500 employees
When the OmniScript validates the plan selection step
Then the system displays an inline warning: 'Large Group account — Underwriting Analyst review required' and sets GSA__c.Underwriting_Review_Required__c = true

## Solution

Build a 6-step OmniScript ('NewGroupOnboarding_OS') in OmniStudio with DataRaptor Transform steps for Account upsert and GSA__c creation. Integrate Conga CLM via a REST callout action in Step 4 to merge the GSA template. Use a DocuSign for Salesforce embedded sending flow triggered by an Apex invocable action called from the OmniScript. Implement a DocuSign Connect webhook endpoint (Salesforce Site or Experience Cloud endpoint) that publishes a Platform Event (DocuSign_Status__e) consumed by a Platform Event-Triggered Flow to update GSA__c.Status__c and attach the signed document via ContentDocumentLink. Configure Sharing Rules so Broker Relations Coordinators can only access GSA__c records linked to their assigned employer accounts via a criteria-based sharing rule on GSA__c.Broker_User__c.

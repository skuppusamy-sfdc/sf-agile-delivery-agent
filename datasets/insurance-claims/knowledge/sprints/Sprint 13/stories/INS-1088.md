## Description

As an Assignment Coordinator, I need a Policyholder 360 LWC dashboard embedded on the Policy__c record page that displays the policyholder's active policies, open and recent claims, FNOL submission history, and a quick-action button to initiate a new FNOL so that coordinators have full context during inbound calls without switching between multiple tabs or legacy systems.

## Acceptance Criteria

Given a Policy__c record is opened by an Assignment Coordinator,
When the Policyholder 360 LWC renders,
Then it must display Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, and Policy__c.Status__c in the header section.

Given the policyholder has existing Claim__c records,
When the Claims Summary tab is selected,
Then all Claim__c records where Claim__c.Policy__c = current Policy Id must be listed, showing Claim__c.Claim_Number__c, Claim__c.Date_of_Loss__c, Claim__c.Status__c, and Claim__c.Total_Paid_Amount__c, sorted by Date_of_Loss__c descending.

Given the Assignment Coordinator clicks 'Submit FNOL',
When the FNOL modal opens,
Then it must pre-populate Claim__c.Policy__c, Claim__c.Policyholder__c, and Claim__c.LOB__c from the parent Policy__c record, and require Claim__c.Date_of_Loss__c and Claim__c.Loss_Description__c before submission.

Given a new FNOL is submitted via the modal,
When the save action completes,
Then a Claim__c record must be created with Claim__c.Status__c = 'FNOL Received', Claim__c.FNOL_Submitted_Date__c = today, and the Claims Summary tab must refresh to include the new record without a full page reload.

-Given a chat transcript summary from the legacy IVR system is displayed in the FNOL modal- (deprecated — IVR integration descoped from Phase 2; removed per INS-1073 decision log)

Given a Policy__c with Status__c = 'Lapsed' or 'Cancelled',
When the 'Submit FNOL' button is rendered,
Then the button must be disabled and a tooltip must display 'FNOL cannot be submitted for an inactive policy'.

## Solution

LWC component (policyholderDashboard360) composed of child components: policyHeader, claimsSummaryTab, and fnolQuickAction. Wire adapters used for Policy__c and Claim__c data retrieval via @wire(getRecord) and a custom Apex controller (PolicyholderDashboardController.cls) for the claims list query. FNOL modal built as a separate LWC (fnolSubmissionModal) using lightning-record-edit-form for Claim__c creation. NavigationMixin used for post-save tab refresh. Button disabled state driven by a reactive getter checking Policy__c.Status__c. Component exposed on the Policy__c Lightning App Builder page restricted to the 'Assignment_Coordinator' Permission Set Group.

## Description

As a Claims Operations Manager, I need the system to automatically determine Coordination of Benefits (COB) priority order when a Claim__c record is linked to a policyholder with multiple active Policy__c records so that primary and secondary payer responsibilities are assigned without manual intervention and EOB generation reflects accurate patient liability.

## Acceptance Criteria

Given a Claim__c record is submitted where the related Contact has more than one Policy__c with Policy__c.Status__c = 'Active',
When the claim enters the 'Adjudication' stage (Claim__c.Stage__c = 'Adjudication'),
Then a Record-Triggered Flow must invoke the COB determination Apex class and populate Claim__c.COB_Priority__c with values 'Primary', 'Secondary', or 'Tertiary'.

Given the birthday rule applies (insured's birth month/day compared across policies),
When COB logic evaluates two policies with the same LOB,
Then Policy__c with the policyholder whose birthday falls earliest in the calendar year must be designated 'Primary' per NAIC COB guidelines.

Given Claim__c.COB_Priority__c = 'Secondary' is set,
When the EOB is generated,
Then Explanation_of_Benefits__c.Primary_Payer_Paid__c must be populated before Explanation_of_Benefits__c.Patient_Responsibility__c is calculated.

Given a Claim__c where COB cannot be auto-determined (e.g., missing Policy__c.Other_Insurance_Indicator__c),
When the adjudication flow reaches the COB step,
Then Claim__c.Stage__c must be set to 'Pended — COB Review' and a Task must be created and assigned to the Claims Operations queue with Subject = 'Manual COB Review Required'.

## Solution

Develop Apex class 'COBDeterminationService' implementing the NAIC birthday rule and employer group size rule. Invoked via a Record-Triggered Flow on Claim__c when Stage__c changes to 'Adjudication'. The service queries all Policy__c records linked to the claimant Contact, compares Policy__c.Group_Size__c and Policy__c.Policyholder_DOB__c fields, and returns a priority map. Results written back to Claim__c via Flow's 'Update Records' element. EOB generation via existing Conga CLM template is updated to conditionally render primary payer fields. Custom metadata type 'COB_Rule__mdt' stores configurable rule weights to avoid hardcoding.

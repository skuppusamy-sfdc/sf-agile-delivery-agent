## Description

As a Broker Relations Coordinator, I need a guided OmniScript that walks through the Group/Employer onboarding process — from capturing employer demographic data, binding a Group_Service_Agreement__c (GSA), sending the GSA for e-signature via DocuSign, and creating the associated Policy__c records — so that the end-to-end group onboarding process is completed in Salesforce without manual handoffs to external systems.

## Acceptance Criteria

Given a Broker Relations Coordinator initiates the Group Onboarding OmniScript from an Account record,
When Step 1 (Employer Demographics) is completed,
Then the Account record is updated with Account.Industry, Account.NumberOfEmployees, Account.BillingAddress, and a custom Group_SIC_Code__c field, and validation confirms TIN__c is a 9-digit numeric value.

Given Step 1 is validated,
When Step 2 (GSA Configuration) is completed,
Then a Group_Service_Agreement__c record is created with GSA.Employer__c = current Account, GSA.Effective_Date__c, GSA.LOB__c, GSA.Broker__c, and GSA.Status__c = 'Pending Signature'.

Given the GSA record is created,
When Step 3 (DocuSign Send) is executed,
Then the OmniScript triggers a DocuSign envelope via the DocuSign for Salesforce API, attaching the GSA_Template.docx Conga template, and populates DocuSign_Envelope_ID__c on the GSA record.

Given the DocuSign envelope is sent,
When the employer completes e-signature,
Then a DocuSign Connect webhook updates GSA.Status__c = 'Executed', GSA.Signed_Date__c = signature timestamp, and the signed PDF is attached to the GSA record as a Salesforce File.

Given the GSA Status__c = 'Executed',
When Step 4 (Policy Binding) is triggered,
Then one or more Policy__c records are created per enrolled LOB with Policy__c.Group_Account__c = employer Account, Policy__c.GSA__c = the executed GSA, Policy__c.Status__c = 'Active', and Policy__c.Effective_Date__c matching GSA.Effective_Date__c.

-Policy__c records should be created immediately upon OmniScript Step 4 completion regardless of DocuSign status- (superseded: Policy binding now gated on GSA.Status__c = 'Executed' per compliance review)

Given the full onboarding flow completes,
When the coordinator views the Account,
Then a summary FlexCard displays the GSA record, all linked Policy__c records, and a DocuSign envelope status badge.

## Solution

Build a 4-step OmniScript (OS_Group_Employer_Onboarding) using OmniStudio. DataRaptors handle Account update (DR_Update_Employer_Demographics) and GSA/Policy creation (DR_Create_GSA, DR_Create_Group_Policies). DocuSign envelope trigger uses an Integration Procedure (IP_Send_GSA_DocuSign) calling the DocuSign REST API via Named Credential. Conga Composer used to generate the GSA PDF template before envelope creation. DocuSign Connect webhook received by a custom Apex REST endpoint (DocuSignWebhookHandler.cls) which updates GSA fields and attaches the signed document. Summary FlexCard (FC_Group_Onboarding_Summary) deployed to Account record page via Lightning App Builder. Permission Set Group 'Broker_Coordinator_PSG' grants access to the OmniScript and related objects.

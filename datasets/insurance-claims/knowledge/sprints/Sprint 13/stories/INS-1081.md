## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record so that I can quickly assess a provider's credentialing status, active network agreements, open claims, and any pending Provider Change Notifications without navigating across multiple objects.

## Acceptance Criteria

Given a Provider__c (Person Account) record is open in Salesforce,
When the Provider 360 FlexCard loads,
Then it must display: Provider.NPI__c, Provider.TIN__c, Provider.Credentialing_Status__c, and the most recent Credentialing_Case__c.Status__c.

Given the provider has one or more active Provider_Network_Agreement__c records,
When the PNA section of the FlexCard renders,
Then each PNA row must show PNA.Network_Name__c, PNA.Effective_Date__c, PNA.Termination_Date__c, and PNA.Agreement_Status__c = 'Active'.

Given the provider has open Provider_Change_Notification__c records,
When the PCN panel loads,
Then all PCNs where PCN.Status__c != 'Closed' must be listed with PCN.Change_Type__c and PCN.Submitted_Date__c visible.

Given a provider's Credentialing_Case__c.Expiration_Date__c is within 90 days of today,
When the FlexCard header renders,
Then a yellow warning badge reading 'Credentialing Expiring Soon' must appear.

Given a provider has Fee_Schedule__c records linked via PNA,
When the Fee Schedule tab is selected,
Then Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.LOB__c, and Fee_Schedule__c.Schedule_Type__c must be displayed in a sortable datatable.

-Given a QGenda schedule sync status field is shown on the FlexCard header- (deprecated — QGenda integration deferred to Sprint 15, see INS-1102)

Given the FlexCard is accessed by a user without the 'Provider_Network_Specialist' Permission Set Group,
When they open a Provider__c record,
Then the PCN and PNA edit actions must be hidden and read-only mode enforced.

## Solution

Build an OmniStudio FlexCard (ProviderNetwork360_FC) with child FlexCards for PNA, PCN, Credentialing, and Fee Schedule sections. Use DataRaptors (DR_GetProviderPNAs, DR_GetOpenPCNs, DR_GetCredentialingCase) to fetch related data. Conditional rendering via FlexCard state actions for the expiring credentialing badge. Permission visibility controlled via Permission Set Group 'Provider_Network_Specialist' checked in FlexCard conditional visibility rules. QGenda sync field placeholder stubbed but hidden pending INS-1102.

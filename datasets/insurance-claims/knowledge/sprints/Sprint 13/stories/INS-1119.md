## Description

As a Broker Relations Coordinator, I need the system to automatically generate a Group Service Agreement (GSA) document from an approved Conga CLM template and send it for e-signature via DocuSign when a Group/Employer Account is approved for onboarding, so that contract turnaround time is reduced from 5 days to under 4 hours.

## Acceptance Criteria

Given a Group/Employer Account (Account.RecordType = 'Group_Employer') has Account.Onboarding_Status__c changed to 'Approved',
When the Record-Triggered Flow fires,
Then a Group_Service_Agreement__c record must be auto-created with GSA.Account__c, GSA.Effective_Date__c = Account.Requested_Effective_Date__c, GSA.LOB__c = Account.LOB__c, GSA.Status__c = 'Draft', and GSA.Contract_Template__c = the active Conga template ID from Custom Metadata.

Given the GSA record is created,
When the Conga CLM API is called,
Then a PDF document must be generated using merge fields from GSA, Account, and related Policy__c data, and stored as a Salesforce File (ContentVersion) linked to the GSA record.

Given the Conga document is generated successfully,
When the DocuSign envelope is created,
Then the envelope must be sent to Account.Primary_Contact__c email with a 5-business-day expiry, and GSA.DocuSign_Envelope_ID__c must be populated.

Given the DocuSign envelope is signed by all parties,
When the DocuSign status callback is received,
Then GSA.Status__c must update to 'Executed', GSA.Executed_Date__c must be stamped, and Account.Onboarding_Status__c must advance to 'Contract Executed'.

Given the DocuSign envelope expires unsigned,
When the expiry webhook fires,
Then a Task must be created for the assigned Broker Relations Coordinator with Subject = 'GSA Signature Overdue: ' + Account.Name and GSA.Status__c set to 'Pending Re-send'.

-Given the employer has more than 500 enrolled members, escalate GSA review to Underwriting Analyst before DocuSign send- (deprecated: large group escalation now handled upstream in the UW approval workflow per INS-1105)

## Solution

Record-Triggered Flow on Account (after update, condition: Onboarding_Status__c changed to 'Approved' and RecordType = 'Group_Employer') calls an Apex invocable action 'GSAGenerationService.generateGSA()'. The Apex class creates the Group_Service_Agreement__c record, invokes the Conga CLM REST API with a template ID from Custom Metadata Type 'Conga_Config__mdt', stores the returned PDF as ContentVersion, then calls the DocuSign eSignature REST API to create and send the envelope. A separate Apex REST endpoint '/services/apexrest/DocuSignCallback' handles incoming status webhooks and updates GSA.Status__c and Account.Onboarding_Status__c via a Record-Triggered Flow. Integration errors logged to Integration_Log__c.

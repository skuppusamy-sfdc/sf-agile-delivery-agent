## Description

As a Claims Operations Manager, I need inbound 835 Electronic Remittance Advice (ERA) files from CMS to be automatically parsed and mapped to Payment__c and Claim_Line__c records in Salesforce so that manual remittance posting is eliminated and payment reconciliation time is reduced by at least 60%.

## Acceptance Criteria

Given an 835 ERA file is deposited in the configured MuleSoft SFTP endpoint,
When the DataIntegration middleware publishes a Platform Event (ERA_File_Received__e) with the file reference ID,
Then an Apex trigger on ERA_File_Received__e fires within 2 minutes and initiates the ERA_Parse_Batch Apex class.

Given the ERA file is successfully parsed,
When individual remittance line items are processed,
Then one Payment__c record is created per payer check/EFT with Payment__c.Payment_Amount__c, Payment__c.Payment_Date__c, Payment__c.ERA_Trace_Number__c, and Payment__c.Payment_Method__c ('EFT' or 'Check') populated from the 835 BPR segment.

Given a Payment__c record is created,
When Claim_Line__c matching occurs using Claim_Line__c.Claim_Control_Number__c against the 835 CLP segment,
Then Payment__c is related to the parent Claim__c and each matched Claim_Line__c.Paid_Amount__c and Claim_Line__c.Adjustment_Reason_Code__c are updated.

Given a Claim_Line__c match cannot be found for one or more ERA line items,
When the batch job completes,
Then an ERA_Exception__c record is created with ERA_Exception__c.Reason__c = 'Unmatched CLM Segment' and assigned to the Claims Operations queue for manual review.

Given all Claim_Line__c records on a Claim__c have Claim_Line__c.Payment_Status__c = 'Paid',
When the final Payment__c is posted,
Then a Record-Triggered Flow updates Claim__c.Status__c to 'Closed - Paid' and stamps Claim__c.Final_Payment_Date__c = TODAY().

Given the ERA batch processes more than 10,000 line items,
When Apex batch execution occurs,
Then the batch runs in chunks of 200 records and completes without hitting DML or SOQL governor limits.

## Solution

Implement MuleSoft integration flow to poll SFTP, parse X12 835 segments, and publish ERA_File_Received__e Platform Event with base64 file payload reference stored in Salesforce Files. Apex Platform Event trigger invokes ERA_Parse_Batch (Database.Batchable, Database.AllowsCallouts). Matching logic uses a custom metadata type (ERA_Matching_Config__mdt) to configure primary/fallback matching keys (Claim Control Number → TIN + DOS). Record-Triggered Flow handles Claim__c status rollup after Payment__c insertion. All ERA exceptions route to a dedicated Claims Payment Exceptions list view.

## Description

**Bug Report — COB Status Premature Adjudication**

**Environment:** Acme Insurance Full Sandbox (ACME_FS1)
**Severity:** High — affects financial reporting and ERA reconciliation

**Steps to Reproduce:**
1. Create a Claim__c record with Claim__c.COB_Indicator__c = TRUE and Claim__c.Primary_Payer__c and Claim__c.Secondary_Payer__c both populated.
2. Process primary payer adjudication: update all associated Claim_Line__c records to Status__c = 'Adjudicated' for the primary payer.
3. Observe Claim__c.Status__c after the Record-Triggered Flow (RTF_ClaimStatusRollup) fires.

**Expected Behavior:**
When COB_Indicator__c = TRUE and Secondary_Payer_Adjudication_Status__c != 'Complete', Claim__c.Status__c should remain 'In Adjudication - COB Pending' and must NOT advance to 'Adjudicated'.

**Actual Behavior:**
Claim__c.Status__c is set to 'Adjudicated' immediately after primary payer Claim_Line__c records are all updated, ignoring the secondary payer adjudication state. This causes premature EOB generation and incorrect ERA reconciliation downstream.

**Impact:**
- 47 COB claims in Sprint 12 UAT data set incorrectly marked 'Adjudicated'
- EOB records generated without secondary payer adjustment amounts
- Payment__c records created with overpayment amounts due to missing COB offset

**Root Cause (Preliminary):**
The RTF_ClaimStatusRollup Flow decision node checks only Claim_Line__c.Status__c aggregate but does not evaluate Claim__c.COB_Indicator__c or Claim__c.Secondary_Payer_Adjudication_Status__c before setting parent status.

## Acceptance Criteria

As per Description.

Additional validation:
Given Claim__c.COB_Indicator__c = TRUE,
When all Claim_Line__c records have Status__c = 'Adjudicated' for the primary payer,
Then Claim__c.Status__c must be set to 'In Adjudication - COB Pending' (not 'Adjudicated') until Claim__c.Secondary_Payer_Adjudication_Status__c = 'Complete'.

Given Claim__c.Secondary_Payer_Adjudication_Status__c is updated to 'Complete',
When the RTF fires,
Then Claim__c.Status__c must advance to 'Adjudicated' and EOB generation may proceed.

## Solution

Add a decision node in RTF_ClaimStatusRollup (Record-Triggered Flow on Claim_Line__c, After Save) to check the parent Claim__c.COB_Indicator__c and Claim__c.Secondary_Payer_Adjudication_Status__c via a Get Records element before executing the status rollup assignment. Add a second entry point to the same Flow triggered on Claim__c update when Secondary_Payer_Adjudication_Status__c changes to 'Complete' to re-evaluate the rollup. Add a unit test (ClaimCOBStatusFlowTest) covering COB = true/false scenarios with 90%+ code path coverage. Deploy fix via change set to ACME_FS1 for regression testing before production push.

## Description

As a Claims Operations Manager, I need incoming Electronic Remittance Advice (ERA) files from our clearinghouse to automatically reconcile against open Payment__c records so that manual reconciliation effort is eliminated and Payment__c.Status__c is updated to 'Reconciled' within minutes of ERA receipt.

## Acceptance Criteria

Given an ERA 835 file is received via the DataIntegration middleware and parsed into a PaymentERA__e Platform Event,
When the Platform Event fires,
Then a Record-Triggered Flow on Payment__c must match on Payment__c.Check_Number__c = ERA.CheckNumber and Payment__c.TIN__c = ERA.ProviderTIN.

Given a successful match is found,
When the reconciliation logic executes,
Then Payment__c.Status__c must be set to 'Reconciled', Payment__c.ERA_Received_Date__c must be stamped with TODAY(), and Payment__c.ERA_Transaction_ID__c must be populated from the Platform Event payload.

Given the ERA contains a claim-level adjustment reason code (CARC),
When reconciliation completes,
Then each associated Claim_Line__c.Adjustment_Reason_Code__c must be updated with the corresponding CARC value and Claim_Line__c.Reconciled__c set to TRUE.

Given no matching Payment__c record is found for an ERA line item,
When the Platform Event is processed,
Then create an Exception_Log__c record with Exception_Log__c.Type__c = 'ERA_Unmatched', Exception_Log__c.ERA_Transaction_ID__c populated, and assign ownership to the Claims Operations queue.

Given more than 500 ERA line items arrive in a single file,
When the Platform Event chain processes them,
Then Apex batch processing must be invoked automatically (batch size = 200) to avoid governor limit violations, and all batches must complete within the 10-minute Platform Event replay window.

-Given Payment__c.Legacy_Reconciled__c = TRUE, When ERA arrives, Then skip reconciliation — DEPRECATED: legacy flag retired after data migration in INS-1121-

## Solution

Create a custom Platform Event 'PaymentERA__e' with fields for CheckNumber, ProviderTIN, TotalAmount, LineItems (JSON blob), and TransactionID. Build an Apex Trigger on PaymentERA__e that deserializes the JSON payload and invokes 'ERAReconciliationBatch' (Database.Batchable) for files exceeding 500 lines, or a synchronous Apex service class 'ERAReconciliationService' for smaller payloads. The service performs SOQL on Payment__c with indexed fields (Check_Number__c, TIN__c). Claim_Line__c updates use a child relationship query. Exception_Log__c records inserted in a separate DML transaction with a try/catch. Deploy with a Connected App configuration for the clearinghouse middleware webhook endpoint.

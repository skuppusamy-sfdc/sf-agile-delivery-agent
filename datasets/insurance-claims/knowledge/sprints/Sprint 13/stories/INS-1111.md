## Description

As a Claims Operations Manager, I need an Agentforce-powered FNOL intake OmniScript that guides claimants through first notice of loss capture, auto-populates Claim__c fields from Policy__c data, and applies an AI-driven SIU fraud scoring model so that high-risk claims are automatically flagged for Special Investigations Unit review before adjudication begins — reducing fraudulent payouts and manual triage effort.

## Acceptance Criteria

Given a policyholder initiates FNOL via the self-service portal,
When the Agentforce FNOL OmniScript launches,
Then the script pre-populates Policy__c.Policy_Number__c, Policy__c.Coverage_Type__c, Policy__c.Effective_Date__c, and PolicyHolder__c name from the authenticated user's linked Policy__c without requiring manual entry.

Given the FNOL OmniScript is completed,
When the claimant submits,
Then a Claim__c record is created with Claim__c.Status__c = 'FNOL Received', Claim__c.FNOL_Date__c = TODAY(), Claim__c.LOB__c derived from Policy__c.Line_of_Business__c, and Claim__c.Intake_Channel__c = 'Self-Service Portal'.

Given a Claim__c record is created via FNOL,
When the Agentforce Einstein Classification model (Claims_Fraud_Score_v2) evaluates the claim,
Then Claim__c.SIU_Risk_Score__c (0–100) is populated within 60 seconds of FNOL submission using features: claim amount, provider NPI history, claimant claim frequency, ICD-10 code pattern, and time-to-FNOL.

Given Claim__c.SIU_Risk_Score__c >= 75,
When the score is written,
Then Claim__c.SIU_Referral_Flag__c = TRUE, Claim__c.Status__c = 'Pending SIU Review', and a high-priority Case is auto-created and assigned to the SIU queue with a summary of the top 3 fraud indicators from the model output.

Given Claim__c.SIU_Risk_Score__c < 75,
When the score is written,
Then the claim proceeds to standard adjudication and Claim__c.Status__c = 'In Adjudication'.

Given the Agentforce model scores a claim,
When the result is stored,
Then an Agentforce_Model_Audit__c record is created capturing the model version, input feature snapshot, score, and timestamp for regulatory explainability.

## Solution

OmniScript (FNOL_Agentforce_Intake) built in OmniStudio with DataRaptor Pre-Load to fetch Policy__c data on launch. On submit, DataRaptor Transform creates Claim__c via upsert. Agentforce Einstein Classification model (Claims_Fraud_Score_v2) invoked via Apex invocable action called from a Record-Triggered Flow (After Insert on Claim__c). Model hosted on Salesforce Einstein Platform; feature vectors assembled in Apex from Claim__c, Policy__c, and historical Claim__c aggregate queries. SIU Case auto-creation via Flow subflow reusing existing SIU_Referral_SubFlow. Agentforce_Model_Audit__c written by Apex after model response. Model refresh cadence: monthly retrain using Workday-sourced claims data pipeline.

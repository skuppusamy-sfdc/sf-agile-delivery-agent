## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record so that I can instantly view credentialing status, active PNA agreements, network participation, open Credentialing_Case__c records, and recent claims activity without navigating across multiple objects.

## Acceptance Criteria

Given a Provider__c record is open in Salesforce,
When the Provider 360 FlexCard loads,
Then the card displays Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Credentialing_Status__c, and Provider__c.Network_Participation_Status__c in the header band.

Given the provider has one or more active Provider_Network_Agreement__c records,
When the PNA section of the FlexCard renders,
Then all PNA records where PNA__c.Status__c = 'Active' and PNA__c.Effective_Date__c <= TODAY() and PNA__c.Termination_Date__c >= TODAY() are listed with LOB, contracted rate tier, and GSA linkage.

Given the provider has an open Credentialing_Case__c,
When the Credentialing panel loads,
Then Credentialing_Case__c.Stage__c, Credentialing_Case__c.CAQH_Profile_Status__c, and Credentialing_Case__c.Expiration_Date__c are surfaced with a visual alert if expiration is within 90 days.

Given the provider has submitted claims in the last 12 months,
When the Claims Activity panel loads,
Then a summary count of Claim__c records grouped by Claim__c.Status__c ('Received', 'In Adjudication', 'Adjudicated', 'Denied', 'Paid') is displayed.

Given a Provider Relations Specialist with the 'Provider_Network_Read' Permission Set Group,
When they access the FlexCard,
Then all data is visible but edit actions are suppressed; users with 'Provider_Network_Full_Access' see inline edit actions for credentialing fields.

-Given a PCN record exists, display PCN details in the main header band- (superseded — PCN moved to dedicated tab per INS-1098 design review)

## Solution

Build an OmniStudio FlexCard ('Provider360Card') with five child cards (Identity, PNA Agreements, Credentialing, Claims Activity, PCN History). Use DataRaptors (Extract type) to query Provider__c, Provider_Network_Agreement__c, Credentialing_Case__c, and Claim__c with aggregate SOQL for claims summary. Conditional visibility rules on each panel driven by FlexCard state properties. Deploy via Permission Set Group assignment. Cache DataRaptor responses for 5 minutes using Platform Cache to reduce SOQL governor limit pressure on high-traffic provider records.

## Description

As a Claims Operations Manager, I need an Agentforce AI action embedded in the Claim__c record page that analyzes configurable anomaly signals and recommends whether a claim should be referred to the Special Investigations Unit (SIU) so that potential fraud is flagged earlier in the adjudication lifecycle and manual review burden on adjusters is reduced.

## Acceptance Criteria

Given a Claims adjuster opens a Claim__c record with Claim__c.Status__c = 'Under Review',
When the Agentforce 'Analyze Claim for SIU Referral' action is invoked,
Then the AI action must evaluate the following signals: Claim__c.Total_Billed_Amount__c vs. Fee_Schedule__c benchmarks, provider Sanction_Flag__c on linked Provider__c, number of Claim__c records for the same Policy__c in the last 90 days, and Claim_Line__c CPT code frequency outliers.

Given the AI analysis returns a recommendation,
When the result is displayed,
Then the Agentforce panel must show a confidence score (High/Medium/Low), the top 3 contributing anomaly signals with plain-language explanations, and a 'Refer to SIU' action button.

Given the adjuster clicks 'Refer to SIU',
When the action is confirmed,
Then Claim__c.SIU_Referral_Flag__c must be set to True, Claim__c.Status__c must transition to 'SIU Review', a Case of RecordType 'SIU Investigation' must be created and linked to the Claim__c, and the SIU queue must be notified.

Given the adjuster clicks 'Dismiss Recommendation',
When the dismissal is logged,
Then Claim__c.SIU_AI_Dismissed__c = True, Claim__c.SIU_Dismiss_Reason__c must be required and captured, and the dismissal event must be written to a custom AI_Audit_Log__c record for compliance tracking.

Given Agentforce is unable to complete analysis due to missing Fee_Schedule__c data,
When the action is invoked,
Then a graceful error message 'Insufficient fee schedule data for analysis' must be displayed and no Claim__c fields must be modified.

## Solution

Configure an Agentforce Agent Action using a custom Apex invocable class (SIUAnomalyAnalysisAction) that aggregates signal data via SOQL and calls the Einstein AI scoring endpoint. The action is registered as an Agent Action in the Agentforce Agent Builder and surfaced on the Claim__c Lightning record page via an Agentforce LWC component. Record-Triggered Flow handles downstream Case creation and queue notification on SIU_Referral_Flag__c = True. AI_Audit_Log__c records written via an Apex after-trigger on Claim__c. Permission Set Group 'Claims_Adjuster_AI' controls access to the Agentforce action.

## Description

**Bug Report — INS-1140**

**Summary:** When a Claim_Line__c record is adjudicated and the FeeScheduleEngine Apex service finds more than one Fee_Schedule__c record matching the CPT code and LOB (e.g., two active rows with overlapping effective dates due to migration data issue), the service returns a null rate and the Claim_Line__c.Adjudicated_Amount__c is set to $0.00 instead of throwing a pend exception.

**Steps to Reproduce:**
1. Ensure two Fee_Schedule__c records exist with the same CPT_Code__c = '99213', LOB__c = 'Commercial', and overlapping Effective_Start_Date__c / Effective_End_Date__c ranges (both Status__c = 'Active').
2. Submit a Claim__c with a Claim_Line__c referencing CPT code 99213 for LOB = 'Commercial'.
3. Trigger adjudication by setting Claim__c.Stage__c = 'Adjudication'.
4. Observe Claim_Line__c.Adjudicated_Amount__c after the flow completes.

**Expected:** Claim_Line__c.Adjudicated_Amount__c should remain null and Claim__c.Stage__c should be set to 'Pended — Fee Schedule Conflict' with a Task assigned to the Claims Operations queue.

**Actual:** Claim_Line__c.Adjudicated_Amount__c = 0.00 and Claim__c.Stage__c = 'Adjudicated' — claim is incorrectly marked as fully processed with zero payment.

## Acceptance Criteria

As per Description. Fix must ensure that when FeeScheduleEngine SOQL returns List<Fee_Schedule__c>.size() > 1, the service throws a custom FeeScheduleConflictException, Claim__c.Stage__c is set to 'Pended — Fee Schedule Conflict', and Claim_Line__c.Adjudicated_Amount__c is not written.

## Solution

Update 'FeeScheduleEngineService.cls': modify the SOQL query result handling to check list size before accessing index [0]. If list size > 1, throw custom exception 'FeeScheduleConflictException'. Add a catch block in the calling Record-Triggered Flow's Apex action to set Claim__c.Stage__c = 'Pended — Fee Schedule Conflict' and create a Task via Flow. Add a unit test 'FeeScheduleEngineServiceTest.testMultipleActiveSchedulesThrowsException()' with @isTest data setup creating two overlapping Fee_Schedule__c records. Deploy hotfix to sandbox for QA validation before production push.

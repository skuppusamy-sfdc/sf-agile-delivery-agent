## Description

As a Provider Network Specialist, I need an automated workflow that, upon approval of a Network_Termination_Case__c, generates a Provider Change Notification (PCN) document, inactivates all associated Provider_Network_Agreement__c records, and notifies downstream systems so that provider terminations are processed consistently, compliantly, and without manual intervention across 6 currently siloed steps.

Terminations must comply with state-mandated notice periods (30–90 days depending on Provider__c.State__c) stored in Custom Metadata (State_Notice_Period__mdt).

## Acceptance Criteria

Given a Network_Termination_Case__c is submitted for approval with Termination_Reason__c and Effective_Termination_Date__c populated,
When the Salesforce Approval Process (ProviderTerminationApproval) reaches final approval,
Then Network_Termination_Case__c.Status__c must be set to 'Approved - Pending Effective Date'.

Given Network_Termination_Case__c.Status__c = 'Approved - Pending Effective Date',
When the Effective_Termination_Date__c is reached (scheduled Flow check at 08:00 daily),
Then:
  - All Provider_Network_Agreement__c records linked to the provider (PNA.Provider__c = termination case provider) with PNA.Status__c = 'Active' must be updated to PNA.Status__c = 'Terminated' and PNA.Termination_Date__c = Effective_Termination_Date__c
  - Network_Termination_Case__c.Status__c must advance to 'Terminated'

Given the termination is approved,
When the approval final step fires,
Then a PCN document must be generated via Conga CLM using template 'PCN_Standard_v2' and stored as a ContentVersion linked to the Network_Termination_Case__c, with PCN.Notice_Date__c = TODAY() and PCN.Effective_Date__c = Effective_Termination_Date__c.

Given Provider__c.State__c maps to a State_Notice_Period__mdt record,
When the Network_Termination_Case__c is submitted for approval,
Then a validation rule must prevent submission if (Effective_Termination_Date__c - TODAY()) < State_Notice_Period__mdt.Minimum_Days__c, displaying error: 'Termination date does not satisfy [State] minimum notice period of [X] days.'

Given the termination is processed,
When PNA records are inactivated,
Then a Platform Event (Provider_Termination_Event__e) must be published with fields: Provider_NPI__c, Termination_Date__c, Affected_PNA_Count__c so that downstream systems (claims adjudication engine, provider directory) can consume the event and update accordingly.

Given any active Claim__c records exist where Claim__c.Rendering_Provider__c = terminated provider and Claim__c.Status__c NOT IN ('Closed - Paid', 'Closed - Denied'),
When the termination workflow fires,
Then each such open claim must have a Task created and assigned to the Claims Operations queue with Subject = 'Open Claim - Provider Terminated' and a reference to the Network_Termination_Case__c.

## Solution

Salesforce Approval Process (ProviderTerminationApproval) with 2-tier approval (Provider Network Specialist → Compliance Officer). Approval final step invokes an Apex invocable method (TerminationApprovalHandler) that calls Conga CLM REST API for PCN generation and publishes Provider_Termination_Event__e. Scheduled-Triggered Flow (daily at 08:00) queries Network_Termination_Case__c records in 'Approved - Pending Effective Date' with Effective_Termination_Date__c = TODAY() and performs bulk PNA inactivation via a loop with DML (batch size governed by collection variable). Validation rule on Network_Termination_Case__c references a formula field (Notice_Days_Remaining__c) derived from State_Notice_Period__mdt via a custom formula. Open claim Task creation handled in the same scheduled Flow via a Get Records + Create Records pattern. State_Notice_Period__mdt populated with all 50-state configurations.

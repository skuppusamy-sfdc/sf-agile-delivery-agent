## Description

As a Claims Operations Manager, I need the platform to automatically ingest Electronic Remittance Advice (ERA) 835 files from the CMS integration endpoint and create or update Payment__c and Explanation_of_Benefits__c records in Salesforce so that manual ERA reconciliation effort is eliminated and payment posting accuracy reaches 99%+.

Extends INS-1098 (CMS outbound eligibility integration).

## Acceptance Criteria

Given a valid ANSI X12 835 ERA file is delivered to the configured CMS SFTP endpoint,
When the scheduled Apex batch job 'ERA_IngestBatch' runs (every 15 minutes),
Then a Payment__c record must be created with Payment__c.ERA_Transaction_ID__c, Payment__c.Amount__c, Payment__c.Payment_Date__c, Payment__c.Payer_ID__c, and Payment__c.Status__c = 'Posted'.

Given the ERA file contains line-level adjudication details,
When Payment__c records are created,
Then a corresponding Explanation_of_Benefits__c record must be created for each Claim_Line__c with EOB.Allowed_Amount__c, EOB.Paid_Amount__c, EOB.Adjustment_Reason_Code__c, and EOB.Claim_Line__c populated.

Given a Payment__c with the same ERA_Transaction_ID__c already exists in Salesforce,
When the batch attempts to insert a duplicate,
Then the record must be skipped and a DataIntegration_Error__c log record created with Error_Type__c = 'Duplicate ERA' and the original Payment__c ID referenced.

Given the ERA file contains a payment for a Claim__c with Claim__c.Status__c = 'Denied',
When the ERA is processed,
Then the Payment__c record must be created but Payment__c.Status__c must be set to 'Pending Review' and a Case must be auto-created and assigned to the Claims Operations queue.

Given the ERA ingestion job completes,
When all records are processed,
Then a Platform Event 'ERA_Ingestion_Complete__e' must be fired with fields: Total_Records__c, Success_Count__c, Error_Count__c, and Batch_Run_DateTime__c.

## Solution

Develop an Apex Schedulable/Batchable class 'ERA_IngestBatch' that polls the CMS SFTP endpoint via a Named Credential and external callout service. Parse 835 file segments using a custom Apex parser utility class. Use Database.insert with allOrNone=false for bulk upsert of Payment__c and EOB records. Fire Platform Event 'ERA_Ingestion_Complete__e' on batch finish. A separate Record-Triggered Flow on Payment__c handles downstream Claim__c status updates. Error logging written to DataIntegration_Error__c via a dedicated Apex logging service to avoid trigger recursion.

## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces active Provider_Network_Agreement__c (PNA) details and associated Fee_Schedule__c summaries so that I can quickly assess a provider's contracted status and reimbursement tiers without navigating multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record,
When the Provider 360 FlexCard loads,
Then it displays all related Provider_Network_Agreement__c records where PNA.Status__c = 'Active' and PNA.Effective_Date__c <= TODAY() and PNA.Termination_Date__c >= TODAY() (or is null).

Given an active PNA is displayed on the FlexCard,
When the specialist expands the PNA row,
Then the associated Fee_Schedule__c records are shown, including Fee_Schedule__c.LOB__c, Fee_Schedule__c.Rate_Type__c, Fee_Schedule__c.Effective_Date__c, and Fee_Schedule__c.Base_Rate_Percentage__c.

Given a Provider__c has no active PNA records,
When the FlexCard loads,
Then a message 'No Active Network Agreements Found' is displayed with a button to initiate a new PNA from the Contracting OmniScript.

Given a PNA has a Termination_Date__c within the next 30 days,
When the FlexCard renders,
Then the PNA row is highlighted with a warning badge labeled 'Expiring Soon' and shows the exact termination date.

-Given the FlexCard loads, it should also display Credentialing_Case__c status inline- (deprecated: credentialing data moved to separate Credentialing tab per UX review)

Given the FlexCard is viewed by a user without the 'PSG_Provider_Relations' Permission Set Group,
When the page loads,
Then the Fee_Schedule__c Base_Rate_Percentage__c field is masked and displays as '***' to enforce FLS compliance.

## Solution

Build OmniStudio FlexCard 'Provider360_PNA_Summary' using a DataRaptor Turbo Extract as the data source, querying Provider_Network_Agreement__c and Fee_Schedule__c with a parent-child SOQL join filtered on PNA.Provider__c = recordId. Conditional styling actions handle the 'Expiring Soon' badge using a State/Action on Termination_Date__c delta < 30. Field-Level Security enforced natively via DataRaptor respecting running user's FLS on Fee_Schedule__c.Base_Rate_Percentage__c. FlexCard deployed to the Provider__c Lightning Record Page via App Builder. Extends pattern established in INS-1474.

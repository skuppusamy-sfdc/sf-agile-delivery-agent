## Description

As a Claims Operations Manager, I need Agentforce to surface AI-driven 'Next Best Action' recommendations on stalled Claim__c records (those with Status__c = 'Pending — Provider Info' for more than 5 business days) that leverage provider network data — including current PNA status, active Fee_Schedule__c coverage for the billed CPT codes, and open PCN flags — so that claims adjusters can quickly identify the root cause of the stall and take the most efficient resolution action, reducing average claims cycle time and improving provider relations outcomes.

## Acceptance Criteria

Given a Claim__c record where Claim__c.Status__c = 'Pending — Provider Info' and the status has not changed for >= 5 business days,
When a claims adjuster opens the Claim__c record page,
Then the Agentforce Next Best Action panel displays at least one AI-generated recommendation with a Recommendation_Type__c drawn from: ['Request Missing NPI', 'Verify PNA Coverage', 'Escalate to Provider Relations', 'Check Open PCN', 'Manual Fee Schedule Override'].

Given Agentforce evaluates the stalled claim,
When the billing provider's Provider__c record has no active PNA (PNA.Status__c != 'Active') for the Claim__c.LOB__c,
Then the top recommendation is 'Verify PNA Coverage' with a deep link to the provider's active PNA list.

Given Agentforce evaluates the stalled claim,
When there is an open Provider_Change_Notification__c linked to the billing provider with PCN.Status__c = 'Open',
Then the recommendation 'Check Open PCN' is surfaced with PCN record ID and summary displayed in the recommendation card.

Given the Agentforce recommendation is displayed,
When the adjuster clicks 'Accept' on a recommendation,
Then a Claim_Action_Log__c record is created with fields: Action_Type__c = recommendation type, Actioned_By__c = current user, Action_Date__c = today, Claim__c = current claim ID, and Claim__c.Last_AI_Recommendation_Date__c is updated.

-Given Agentforce auto-resolves the claim by updating Claim__c.Status__c directly- (deprecated — auto-resolution capability descoped from Phase 2; adjuster must confirm all status changes)

Given the Agentforce model is invoked,
When the claim has an SIU referral flag (Claim__c.SIU_Referred__c = TRUE),
Then no AI recommendations are surfaced and the panel displays: 'This claim is under SIU review. AI recommendations are suspended.'

## Solution

Configure an Agentforce (Einstein Copilot / Next Best Action) strategy using the Next Best Action framework. Define a Strategy Builder flow that queries Claim__c context (billing provider, LOB, status duration), related PNA records, open PCNs, and Fee_Schedule__c coverage gaps. Recommendation objects (standard Next Best Action Recommendation__c) are populated via a Recommendation Strategy that calls an Apex invocable method (ClaimsAIRecommendationService) for complex provider network logic. The LWC Next Best Action component is embedded on the Claim__c Lightning Record Page. A Record-Triggered Flow on Claim_Action_Log__c (After Insert) updates Claim__c.Last_AI_Recommendation_Date__c. SIU suppression is implemented as a filter node in the Strategy Builder flow. Custom metadata (AI_Recommendation_Config__mdt) stores recommendation type labels and deep link URL templates. The solution respects Field-Level Security and Profile access — only users with the 'Claims_Adjuster_PSG' Permission Set Group see the recommendation panel.

## Description

As a Provider Network Specialist, I need a bulk update interface that allows me to update NPI and TIN fields across multiple Provider__c (Person Account) records simultaneously so that I can efficiently process mass provider data corrections received from CAQH and NPPES reconciliation reports without updating each record individually.

Background: During monthly NPPES reconciliation, the Provider Relations team receives correction files containing updated NPI and TIN values for 50–500 providers. Currently this is a manual, record-by-record process taking 2–3 days. This story delivers a managed bulk update capability with audit trail and rollback support.

Extends INS-1446 (Provider 360 base screen).

## Acceptance Criteria

Given a Provider Network Specialist with the 'Provider_Data_Maintenance' Permission Set Group assigned,
When they navigate to the Provider Data Maintenance OmniScript (/omni/bulk-provider-update),
Then they should see an upload zone accepting CSV files up to 5 MB with columns: NPI, TIN, Provider_Salesforce_Id.

Given a valid CSV is uploaded,
When the user clicks 'Preview Changes',
Then a DataRaptor-powered preview table must display: Provider__c.Name, Provider__c.NPI__c (current → new), Provider__c.TIN__c (current → new), and a validation status column ('Valid' | 'NPI Format Error' | 'Duplicate NPI' | 'Record Not Found').

Given the preview table is displayed,
When any row has status 'NPI Format Error',
Then Provider__c.NPI__c must be validated against the 10-digit numeric regex and the row must be highlighted in red with inline error text; the 'Confirm Bulk Update' button must remain disabled until all errors are resolved or removed.

Given all rows show 'Valid' status,
When the user clicks 'Confirm Bulk Update',
Then an Apex Batch job (BatchProviderNPITINUpdate) must execute, updating Provider__c.NPI__c and Provider__c.TIN__c, and creating a Provider_Change_Notification__c (PCN) record per provider with PCN.Change_Type__c = 'NPI/TIN Correction', PCN.Source__c = 'Bulk Upload', PCN.Initiated_By__c = running user, and PCN.Status__c = 'Pending Review'.

Given the batch job completes,
When results are available,
Then a completion email must be sent to the initiating user containing: total records processed, success count, failure count, and a downloadable error log CSV for any failed rows.

-Given a batch job is in flight, When a second batch is submitted by any user, Then it must be queued and not executed concurrently- (deprecated — concurrency now handled by Platform Event queue; see INS-1483)

Given a PCN record is created with PCN.Status__c = 'Pending Review',
When the Provider Relations Specialist reviews and sets PCN.Status__c = 'Approved',
Then Provider__c.Data_Last_Verified_Date__c must be stamped with today's date and Provider__c.Verification_Source__c must be set to 'NPPES Reconciliation'.

## Solution

1. OmniScript 'BulkProviderNPITINUpdate' with a File Upload step feeding a DataRaptor Extract for preview rendering in an embedded LWC table component.
2. Apex Batch class 'BatchProviderNPITINUpdate' implementing Database.Batchable and Database.Stateful; batch size 200; uses Database.update(records, false) for partial success.
3. Record-Triggered Flow on Provider_Change_Notification__c (after insert, PCN.Status__c = 'Approved') to stamp Provider__c.Data_Last_Verified_Date__c via a related record update.
4. Custom Permission Set Group 'Provider_Data_Maintenance' controlling access to the OmniScript URL and the Apex class.
5. Platform Event 'Bulk_Update_Completed__e' fired at batch finish to trigger the notification email via a triggered Salesforce Flow subscribing to the event.

## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page that consolidates active PNA details, current Fee Schedule assignments, and Credentialing Case status so that I can resolve provider inquiries in a single view without navigating across multiple related lists and tabs.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then it must display the following from Provider_Network_Agreement__c: PNA_Status__c, Effective_Date__c, Termination_Date__c, Network_Name__c, and LOB__c for all PNAs where PNA_Status__c IN ('Active', 'Pending Termination').

Given the FlexCard loads,
When the provider has one or more Fee_Schedule__c records linked via Fee_Schedule__c.Provider__c,
Then the card must display: Fee_Schedule_Name__c, Effective_Date__c, Expiration_Date__c, and Schedule_Type__c grouped by LOB.

Given the FlexCard loads,
When a Credentialing_Case__c exists for the provider with Credentialing_Status__c NOT IN ('Closed', 'Withdrawn'),
Then the card must display a 'Credentialing In Progress' badge with the Case_Stage__c value and expected completion date (Expected_Completion_Date__c).

Given the provider has no active PNA,
When the FlexCard renders the PNA section,
Then it must display a warning banner: 'No Active Network Agreement — Provider may be Out of Network'.

-Given the FlexCard loads, display the provider's QGenda schedule block- (deprecated: QGenda integration deferred to Sprint 19)

Given a Provider Relations Specialist clicks 'Create PCN' action on the FlexCard,
When the action is triggered,
Then an OmniScript 'Create Provider Change Notification' must launch in a modal, pre-populated with Provider__c.Id and Provider__c.NPI__c.

## Solution

Build OmniStudio FlexCard 'Provider360Card' using DataRaptors to fetch related PNA, Fee Schedule, and Credentialing Case data. DataRaptor Extract 'DR_GetProviderPNA' queries Provider_Network_Agreement__c filtered by PNA_Status__c. DataRaptor Extract 'DR_GetProviderFeeSchedules' queries Fee_Schedule__c. Conditional rendering via FlexCard states for 'No Active PNA' warning. 'Create PCN' action button triggers OmniScript 'OS_CreatePCN' in modal overlay. FlexCard deployed to Provider__c Lightning Record Page via App Builder. Integration Procedure 'IP_Provider360Summary' orchestrates all DataRaptor calls. Card is gated by Permission Set 'Provider_Relations_Specialist_PS'.

## Description

As a Provider Network Specialist, I need a bulk recontract wizard that identifies Provider_Network_Agreement__c (PNA) records expiring within a configurable look-ahead window and allows me to initiate renewal workflows for multiple providers simultaneously so that I can manage high-volume contract renewal cycles efficiently and reduce the risk of providers falling out of network during peak claims periods.

## Acceptance Criteria

Given a Provider Network Specialist navigates to the 'Bulk Recontract Wizard' tab in the Provider Network app,
When the page loads,
Then an OmniScript wizard must present Step 1: 'Select Expiry Window' with a picklist offering 30, 60, 90, or 120 days, defaulting to 90 days.

Given the specialist selects an expiry window and clicks 'Find Expiring Agreements',
When the DataRaptor queries PNA records,
Then results must display all PNA records where PNA.Expiration_Date__c <= TODAY() + [selected days] AND PNA.Status__c = 'Active' AND PNA.Auto_Renew__c = FALSE, showing columns: Provider Name, NPI__c, PNA.Agreement_Number__c, PNA.LOB__c, PNA.Expiration_Date__c, PNA.Network_Tier__c.

Given the results list is displayed,
When the specialist selects one or more PNA records using row checkboxes and clicks 'Initiate Renewal',
Then for each selected PNA, the system must create a new PNA record with Status__c = 'Pending Renewal', PNA.Parent_Agreement__c = original PNA Id, PNA.Effective_Date__c = original PNA.Expiration_Date__c + 1 day, and PNA.Assigned_To__c = current user.

Given new 'Pending Renewal' PNA records are created,
When the wizard completes Step 3 'Send for Signature',
Then a Conga CLM document generation job must be triggered for each new PNA using the 'PNA_Renewal_Template' Conga template, and the generated document must be sent via DocuSign to the email address on Provider__c.Email.

Given the DocuSign envelope is sent,
When the PNA record is updated,
Then PNA.DocuSign_Envelope_Id__c must be stamped and PNA.Signature_Status__c = 'Sent for Signature'.

Given the specialist selects more than 50 PNA records at once,
When they click 'Initiate Renewal',
Then the system must display a warning: 'You have selected [count] agreements. Processing will be queued asynchronously. You will receive an email notification upon completion.' and process via Queueable Apex.

-Given the wizard is complete, When the user exits, Then return to the Provider Network home page- (deprecated — redirect to a 'Renewal Summary' confirmation screen per UX feedback)

## Solution

Build OmniScript 'BulkRecontractWizard' with 3 steps: Step 1 (picklist input), Step 2 (DataRaptor Extract 'DR_GetExpiringPNAs' results in a DataGrid), Step 3 (confirmation and async trigger). Use Integration Procedure 'IP_InitiatePNARenewal' to orchestrate: DataRaptor Transform for new PNA field mapping, Apex Action 'CreateRenewalPNAsAction' for DML (sync <=50, Queueable >50), and HTTP Action calling Conga CLM REST API for document generation. DocuSign envelope stamping handled via DocuSign for Salesforce managed package trigger on PNA record. Deploy OmniScript to a custom tab using a FlexCard launcher. Permission gated via 'Provider_Network_Contracting' Permission Set Group.

## Description

As a Provider Network Specialist, I need a bulk update tool that allows me to update NPI and TIN fields on multiple Provider__c (Person Account) records simultaneously so that I can efficiently correct provider demographic data received from NPPES and CAQH integration feeds without manually updating each record one at a time. This tool should support CSV upload, field-level validation, and a dry-run preview before committing changes.

## Acceptance Criteria

Given a Provider Network Specialist has the 'Provider_Data_Maintenance' Permission Set Group assigned,
When they navigate to the Provider Data Maintenance LWC tool on the Provider Network app page,
Then they should see an upload interface accepting CSV files up to 10MB with columns: Provider_Id__c, NPI__c, TIN__c, Effective_Date__c.

Given a valid CSV is uploaded,
When the user clicks 'Dry Run',
Then the system should display a preview table showing current vs. proposed values for each Provider__c record, flagging any NPI__c values that fail 10-digit numeric format validation with a row-level error message.

Given the dry-run preview shows zero validation errors,
When the user clicks 'Commit Changes',
Then a Queueable Apex job should process updates in batches of 200, updating Provider__c.NPI__c and Provider__c.TIN__c fields and stamping Provider__c.Last_Data_Maintenance_Date__c with the current datetime.

Given the batch job completes,
When all records are processed,
Then a Provider_Change_Notification__c (PCN) record must be auto-created for each updated provider with PCN.Change_Type__c = 'NPI/TIN Update', PCN.Status__c = 'Pending Review', and PCN.Source__c = 'Bulk Maintenance Tool'.

Given any record in the batch fails to update,
When the job finishes,
Then failed records must be logged to a Bulk_Update_Error_Log__c custom object with fields: Error_Message__c, Provider_Id__c, Attempted_NPI__c, Attempted_TIN__c, and the submitting user must receive an in-app notification via Platform Event.

-Given the tool is accessible to all internal users, When any user opens the page, Then the upload button should be visible- (deprecated — access restricted to Permission Set Group only per security review)

Given a TIN__c value in the CSV contains fewer than 9 digits,
When validation runs,
Then that row must be flagged with error 'Invalid TIN: must be 9-digit numeric EIN or SSN format' and excluded from both dry-run and commit.

## Solution

Build an LWC 'providerBulkMaintenanceTool' with a file upload component using lightning-file-upload. On upload, invoke an Apex Controller 'ProviderBulkMaintenanceController' to parse CSV via a utility class and perform field-level validation (NPI regex: ^[0-9]{10}$, TIN regex: ^[0-9]{9}$). Dry-run results returned as a wrapper list to the LWC datatable. On commit, enqueue 'ProviderBulkUpdateQueueable' which processes records in 200-record chunks using Database.update with allOrNone=false. PCN creation handled in a separate Record-Triggered Flow on Provider__c after-save event, filtered by Last_Data_Maintenance_Date__c change. Errors published via Platform Event 'Bulk_Update_Error__e' and consumed by a Flow to write Bulk_Update_Error_Log__c records and send Bell notifications. Deploy behind 'Provider_Data_Maintenance' Permission Set Group.

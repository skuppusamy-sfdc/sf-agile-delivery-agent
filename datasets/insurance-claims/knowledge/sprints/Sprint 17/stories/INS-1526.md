## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces active PNA details, associated Fee_Schedule__c line items, and linked Group_Service_Agreement__c (GSA) entitlements so that I can answer provider inquiries without navigating across five separate related lists. Extends INS-1495 (Provider 360 foundation FlexCard).

## Acceptance Criteria

Given a Provider Relations Specialist opens any Provider__c record,
When the Provider 360 FlexCard loads,
Then it displays all Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active' AND PNA__c.Effective_End_Date__c >= TODAY(), grouped by LOB.

Given an active PNA is displayed,
When the specialist expands the PNA panel,
Then the associated Fee_Schedule__c records are shown with Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Allowed_Amount__c, and Fee_Schedule__c.Effective_Date__c columns.

Given the provider is linked to one or more Group_Service_Agreement__c records,
When the GSA panel is rendered,
Then GSA__c.Agreement_Number__c, GSA__c.Employer_Group__c, GSA__c.Status__c, and GSA__c.Renewal_Date__c are displayed; records with GSA__c.Renewal_Date__c within 90 days are highlighted in amber.

Given the provider has an open Credentialing_Case__c with Credentialing_Case__c.Status__c IN ('In Review', 'Pending Documents'),
When the FlexCard header renders,
Then a credentialing status badge is shown with a deep-link to the Credentialing_Case__c record.

Given a provider with no active PNA,
When the FlexCard loads,
Then an inline message 'No Active Network Agreements — contact Network Contracting' is shown and the Fee Schedule panel is hidden.

-Given the provider is out-of-network, display a red OON banner at the top of the FlexCard- (removed per UX review, replaced by PNA status badge)

## Solution

Build an OmniStudio FlexCard ('Provider360Card') using DataRaptor Turbo Extract actions to query Provider_Network_Agreement__c, Fee_Schedule__c, and Group_Service_Agreement__c with relationship filters. Child FlexCards handle PNA expansion and fee schedule tabular display. A custom Apex DataRaptor post-hook computes the 90-day GSA renewal warning flag. The FlexCard is deployed via a Lightning App Builder page override on the Provider__c (Person Account) record page and gated by the 'Provider_Relations_User' Permission Set Group.

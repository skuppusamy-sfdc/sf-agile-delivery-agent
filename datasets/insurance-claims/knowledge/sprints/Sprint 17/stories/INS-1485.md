## Description

As a Credentialing Specialist, I need Salesforce to automatically pull updated credentialing attestation status from the CAQH ProView API on a nightly basis and reflect those changes on the corresponding Credentialing_Case__c record so that I do not have to manually log into CAQH to check provider attestation currency, reducing credentialing review cycle time by an estimated 30%.

## Acceptance Criteria

Given a Credentialing_Case__c record exists with Credentialing_Case__c.CAQH_Provider_ID__c populated,
When the nightly scheduled Apex job 'CAQHSyncSchedulable' runs at 02:00 AM EST,
Then the job calls the CAQH ProView REST API endpoint /credentialing/v1/providers/{caqhId}/status for each eligible record.

Given the CAQH API returns a status of 'Initial Profile Complete',
When the sync job processes the response,
Then Credentialing_Case__c.CAQH_Attestation_Status__c is set to 'Complete' and Credentialing_Case__c.Last_CAQH_Sync__c = NOW().

Given the CAQH API returns a status of 'Re-Attestation Required',
When the sync job processes the response,
Then Credentialing_Case__c.CAQH_Attestation_Status__c = 'Re-Attestation Required', Credentialing_Case__c.Status__c transitions to 'Pending Provider Action', and a Task is created assigned to the owning Credentialing Specialist with Subject = 'CAQH Re-Attestation Required — [Provider Name]' and ActivityDate = TODAY() + 7.

Given the CAQH API call fails with HTTP 4xx or 5xx for a given provider,
When the sync job encounters the error,
Then the Credentialing_Case__c record is NOT updated, an error entry is written to Integration_Error_Log__c with fields: Error_Source__c = 'CAQH', Record_Id__c, HTTP_Status_Code__c, Error_Message__c, and Retry_Count__c is incremented.

Given Retry_Count__c reaches 3 for a given record,
When the next sync run evaluates that record,
Then the record is skipped and Credentialing_Case__c.Sync_Status__c = 'Manual Review Required' is set, triggering an email alert to the Credentialing Operations queue.

Given the sync job completes its full run,
When all records are processed,
Then a Platform Event 'CAQHSyncComplete__e' is published with fields: Total_Processed__c, Success_Count__c, Error_Count__c, Run_Timestamp__c, which a subscriber Flow uses to create a Sync_Audit_Log__c record.

## Solution

1. Implement 'CAQHSyncSchedulable' as a Schedulable Apex class that invokes 'CAQHSyncBatchable' (Database.Batchable, Database.AllowsCallouts) with batch size 10 (CAQH API rate limit consideration).
2. Named Credential 'CAQH_ProView_API' stores endpoint and OAuth 2.0 client credentials; HTTP callouts use HttpRequest with timeout = 10000ms.
3. Response parsing via JSON.deserializeUntyped(); status mapping handled by a Custom Metadata Type 'CAQH_Status_Mapping__mdt' for maintainability.
4. Task creation and Credentialing_Case__c field updates performed in the batch execute() method; errors written to Integration_Error_Log__c.
5. Platform Event 'CAQHSyncComplete__e' published in the batch finish() method via EventBus.publish().
6. A Record-Triggered Flow on Integration_Error_Log__c (after insert, Retry_Count__c = 3) sets Credentialing_Case__c.Sync_Status__c = 'Manual Review Required' and sends email via Send Email action.
7. Scheduled via Setup > Scheduled Jobs at 02:00 AM EST daily.

## Description

**Bug Report — INS-1528**

**Summary:** Provider__c.TIN__c field does not enforce numeric-only input or 9-digit length at the UI layer, allowing malformed TINs to be saved. These invalid TINs propagate to ERA files, causing Electronic Remittance Advice reconciliation failures in the downstream payment engine.

**Steps to Reproduce:**
1. Navigate to any Provider__c (Person Account) record.
2. Click Edit on the TIN__c field.
3. Enter a value such as '12-345678A' or '123 456 789' and save.
4. Observe the record saves successfully with no validation error.
5. Trigger an ERA generation run for a Claim__c linked to this provider.
6. Observe ERA file contains malformed TIN, and Payment__c reconciliation job throws 'TIN_FORMAT_INVALID' error in the integration log.

**Expected Behavior:** The TIN__c field should only accept exactly 9 numeric digits (no hyphens, spaces, or alpha characters) and display an inline validation error on save if the format is violated.

**Actual Behavior:** Non-numeric characters and incorrect lengths are accepted and persisted to Provider__c.TIN__c without error.

**Impact:** High — affects all ERA batches for providers with malformed TINs. Three Payment__c batches failed in production last week (batch IDs: ERA-20240918-004, ERA-20240919-011, ERA-20240921-007).

**Environment:** Production and Full Sandbox.

## Acceptance Criteria

As per Description. Additional: After fix, regression test must confirm Provider__c.TIN__c = '123456789' saves successfully, and Provider__c.TIN__c = '12-345678A' returns validation error 'TIN must be exactly 9 numeric digits.'

## Solution

Add a Validation Rule on Provider__c: NOT(REGEX(TIN__c, '[0-9]{9}')) with error message 'TIN must be exactly 9 numeric digits (no hyphens or spaces).' Apply the rule to both UI and API saves. Additionally, update the LWC provider edit form (providerDataForm) to include a client-side regex mask. Run a Data Quality report to identify existing records with malformed TIN__c values and route to Provider Network Specialists for correction via the INS-1525 bulk update tool.

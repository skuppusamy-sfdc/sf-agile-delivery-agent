## Description

As a Provider Network Specialist, I need a bulk update tool that allows me to update NPI and TIN fields across multiple Provider__c (Person Account) and Facility__c (Business Account) records simultaneously so that I can efficiently maintain accurate provider data without processing individual records one at a time, reducing administrative overhead during quarterly data reconciliation cycles with NPPES and CAQH.

## Acceptance Criteria

Given a Provider Network Specialist accesses the 'Provider Bulk Update' tab in Salesforce,
When they upload a CSV file containing up to 500 rows with columns [NPI, TIN, Provider_External_ID__c],
Then the system validates each NPI against the NPPES integration endpoint and flags any mismatches before committing changes.

Given a valid CSV is uploaded and validation passes,
When the specialist clicks 'Apply Bulk Update',
Then Provider__c.NPI__c and Provider__c.TIN__c are updated on all matched records, and a Bulk_Update_Log__c record is created with fields: Records_Processed__c, Records_Failed__c, Initiated_By__c, and Timestamp__c.

Given a row in the CSV fails NPI validation,
When the bulk job completes,
Then the failed row is included in a downloadable error report and Provider__c.NPI__c is NOT updated for that record.

Given the bulk update job processes more than 100 records,
When the job is submitted,
Then it is executed asynchronously via a Queueable Apex job and the specialist receives an in-app notification (Custom Notification) upon completion.

-Given the tool supports updates to Claim__c.Billing_Provider_NPI__c in the same transaction- (deprecated — Claim NPI sync handled separately in INS-1447)

Given a Facility__c (Business Account) record is included in the CSV,
When the TIN value differs from the existing Facility__c.TIN__c,
Then the system creates a Provider_Change_Notification__c (PCN) record linked to the Facility__c with PCN.Change_Type__c = 'TIN Update' and PCN.Status__c = 'Pending Review'.

Given the bulk update completes successfully,
When any updated Provider__c record has an active Provider_Network_Agreement__c (PNA) with PNA.Status__c = 'Active',
Then a Platform Event (ProviderDataChanged__e) is published for downstream Fee_Schedule__c revalidation.

## Solution

Build an LWC-based 'Provider Bulk Update' tool hosted on a custom Salesforce tab. The LWC handles CSV parsing client-side using Papa Parse and sends records in batches of 50 to an Apex REST endpoint. The endpoint invokes a Queueable Apex chain (ProviderBulkUpdateQueueable) that performs NPPES callouts for NPI validation, applies DML updates to Provider__c and Facility__c, creates PCN records via a helper service class, and publishes ProviderDataChanged__e Platform Events. A Record-Triggered Flow on Bulk_Update_Log__c sends Custom Notifications to the initiating user on job completion. Permission Set Group 'Provider_Data_Steward_PSG' gates access to the tool. Error rows are compiled into a ContentVersion (CSV) linked to the Bulk_Update_Log__c record for download.

## Description

As a Broker Relations Coordinator, I need a bulk GSA renewal workflow that allows me to select multiple expiring Group_Service_Agreement__c records for employer groups and trigger a DocuSign envelope send in a single action so that annual renewals during open enrollment season (Oct–Dec) do not require one-by-one manual processing, which currently takes up to 4 hours per coordinator per day during peak periods.

The workflow must support up to 200 GSA records per bulk action and integrate with the existing Conga CLM template library for contract document generation prior to DocuSign routing.

## Acceptance Criteria

Given a Broker Relations Coordinator navigates to the 'GSA Bulk Renewal' custom list view on the Group_Service_Agreement__c object,
When they apply the filter GSA.Status__c = 'Active' AND GSA.Expiration_Date__c <= TODAY() + 90,
Then the list view displays all qualifying records with columns: GSA.Account__c (Group Name), GSA.Expiration_Date__c, GSA.LOB__c, GSA.Primary_Broker__c, GSA.Renewal_Status__c.

Given records are displayed in the bulk renewal list view,
When the coordinator selects between 1 and 200 records and clicks 'Initiate Bulk Renewal',
Then the system validates that each selected GSA.Renewal_Status__c != 'Renewal In Progress' before proceeding; records already in renewal are excluded with a warning toast.

Given validation passes for selected records,
When the coordinator confirms the bulk action,
Then a Conga CLM document generation job is triggered for each GSA using the template mapped via GSA.LOB__c on the custom metadata type 'GSA_Template_Mapping__mdt', and GSA.Renewal_Status__c is set to 'Document Generation In Progress'.

Given Conga CLM document generation completes successfully for a GSA,
When the generated document is returned,
Then a DocuSign envelope is automatically sent to the email address on GSA.Primary_Contact__r.Email with the generated document attached, GSA.Renewal_Status__c = 'DocuSign Sent', and GSA.DocuSign_Sent_Date__c = TODAY().

Given the DocuSign envelope is signed by the recipient,
When the DocuSign callback webhook fires,
Then GSA.Renewal_Status__c = 'Signed — Pending Internal Approval', GSA.DocuSign_Completed_Date__c = NOW(), and an Approval Process submission is triggered on the GSA record for internal sign-off.

Given the internal Approval Process is approved,
When the final approver clicks 'Approve',
Then a new GSA record is created (cloned from the current) with GSA.Effective_Date__c = current GSA.Expiration_Date__c + 1 day, GSA.Expiration_Date__c = new effective date + 1 year, GSA.Status__c = 'Active', and the original GSA.Status__c is set to 'Superseded'.

Given the bulk action processes more than 200 selected records,
When the coordinator attempts to submit,
Then the system displays an error: 'Maximum 200 records per bulk renewal batch. Please reduce your selection.' and does not proceed.

## Solution

1. Build a custom LWC 'gsaBulkRenewalManager' embedded in a GSA list view via a Lightning App Page, supporting multi-select via a datatable component with server-side pagination.
2. On 'Initiate Bulk Renewal' click, invoke Apex controller 'GSABulkRenewalController.initiateRenewal(List<Id> gsaIds)' which validates Renewal_Status__c and updates records synchronously if <= 200.
3. Conga CLM integration via REST API callout from Apex Queueable 'CongaDocGenQueueable'; template selection driven by GSA_Template_Mapping__mdt custom metadata.
4. DocuSign send triggered via DocuSign for Salesforce managed package API method 'dsfs__DocuSign_Status__c' after Conga callback; envelope recipient pulled from GSA.Primary_Contact__r.Email.
5. DocuSign completion webhook handled by a public Site Apex REST endpoint 'DocuSignCallbackHandler' which publishes Platform Event 'DocuSignEvent__e'; a subscriber Flow updates GSA fields and submits the Approval Process via Submit for Approval action.
6. Approval Process on Group_Service_Agreement__c with two approval steps: Broker Relations Manager + Compliance Officer. Final approval action triggers a Record-Triggered Flow to clone the GSA and supersede the original.
7. 200-record limit enforced in both LWC (client-side) and Apex controller (server-side guard).

## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces all active Provider_Network_Agreement__c records and their associated Fee_Schedule__c entries so that I can quickly assess a provider's contracted network status and reimbursement rates without navigating multiple related lists.

## Acceptance Criteria

Given a Provider__c record is open in Lightning Experience,
When the Provider 360 FlexCard loads,
Then it displays all Provider_Network_Agreement__c records where PNA_Status__c = 'Active' and PNA_Effective_Date__c <= Today and PNA_Termination_Date__c >= Today (or is null).

Given an active PNA record is displayed on the FlexCard,
When the specialist clicks 'View Fee Schedule',
Then a child DataRaptor-driven panel expands showing Fee_Schedule__c records linked via Fee_Schedule__c.PNA__c with columns: CPT_Code__c, Modifier__c, Allowed_Amount__c, Effective_Date__c, and LOB__c.

Given a Provider__c has no active PNA records,
When the FlexCard loads,
Then a contextual message 'No Active Network Agreements Found' is displayed with a quick-action button to initiate a new PNA via OmniScript.

Given the FlexCard is viewed by a user without the 'Provider_Network_Read' Permission Set,
When they navigate to the Provider__c record,
Then the FlexCard renders a restricted-access message and Fee_Schedule__c Allowed_Amount__c fields are masked.

-Given PNA records include GSA linkage, display Group_Service_Agreement__c name inline- (deprecated — GSA display moved to separate GSA FlexCard component per design review).

## Solution

Build an OmniStudio FlexCard 'Provider360_NetworkAgreements_FC' using two nested DataRaptor Extracts: one for PNA records filtered by status and date range, one for child Fee_Schedule__c records keyed on PNA Id. Use a Conditional View State to toggle the restricted-access message based on Custom Permission 'View_Provider_Fee_Schedules'. Deploy FlexCard to the Provider__c Lightning Record Page via App Builder. Fee Schedule amounts masked using a DataRaptor Transform formula that returns '****' when the running user lacks the custom permission. Quick-action button invokes OmniScript 'New_PNA_Initiation_OS' (INS-1466).

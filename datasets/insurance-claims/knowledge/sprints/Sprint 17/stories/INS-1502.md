## Description

**Bug Report — INS-1502**

Related to INS-1497 (Provider 360 FlexCard implementation).

Steps to Reproduce:
1. Identify a Provider__c record that has a Provider_Network_Agreement__c with PNA_Status__c = 'Terminated' but PNA_Termination_Date__c set to a future date (e.g., 30 days from today — common in advance-notice termination scenarios).
2. Open the Provider__c Lightning Record Page containing the Provider 360 FlexCard.
3. Observe the PNA section of the FlexCard.

Expected Behavior:
Only PNA records where PNA_Status__c = 'Active' AND PNA_Effective_Date__c <= TODAY AND (PNA_Termination_Date__c >= TODAY OR PNA_Termination_Date__c = null) should be displayed.

Actual Behavior:
PNA records with PNA_Status__c = 'Terminated' but a future PNA_Termination_Date__c are incorrectly displayed in the Active Agreements section of the FlexCard. This causes Provider Relations Specialists to incorrectly treat future-terminated providers as fully active, potentially leading to incorrect claim adjudication decisions.

Additional Context:
- Root cause suspected: the DataRaptor Extract 'DR_Extract_ActivePNAs' used by the FlexCard filters only on PNA_Termination_Date__c >= TODAY without also filtering on PNA_Status__c = 'Active'.
- Observed in production by two Provider Relations Specialists on 14 provider records.
- No data corruption — purely a display issue.

## Acceptance Criteria

As per Description.

Additional verification:
Given a PNA record has PNA_Status__c = 'Terminated' and PNA_Termination_Date__c = TODAY + 30,
When the Provider 360 FlexCard loads,
Then that PNA record must NOT appear in the Active Agreements section.

Given a PNA record has PNA_Status__c = 'Active' and PNA_Termination_Date__c = TODAY + 30,
When the FlexCard loads,
Then that PNA record MUST appear in the Active Agreements section (advance-notice period, still technically active).

## Solution

Update the DataRaptor Extract 'DR_Extract_ActivePNAs' to add an additional filter condition on PNA_Status__c = 'Active' using an AND operator alongside the existing date range filter. Regression test with four PNA scenarios: (1) Active/future termination date, (2) Terminated/future termination date, (3) Active/null termination date, (4) Active/past termination date. Redeploy the DataRaptor and FlexCard to production after UAT sign-off. No Apex or Flow changes required.

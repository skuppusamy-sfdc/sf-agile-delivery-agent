## Description

As a Provider Network Specialist, I need the ability to perform bulk updates to Provider__c (Person Account) NPI__c and TIN__c fields via a validated CSV import tool so that I can efficiently correct provider demographic data received from NPPES and CAQH reconciliation reports without manually editing hundreds of individual records.

## Acceptance Criteria

Given a Provider Network Specialist uploads a CSV file containing Provider__c NPI__c and TIN__c update records,
When the file is submitted through the Bulk Provider Data Update OmniScript,
Then the system validates each row against NPPES NPI format (10-digit numeric) and TIN format (9-digit numeric, no dashes) before processing.

Given a CSV row contains an NPI__c value that already exists on a different active Provider__c record,
When the validation step runs,
Then the row is flagged with error code 'DUPLICATE_NPI' and excluded from the bulk update batch, and the specialist is shown a validation error summary screen.

Given all rows pass validation,
When the bulk update job is submitted,
Then a Salesforce Batch Apex job processes up to 10,000 Provider__c records per execution, updating NPI__c, TIN__c, and stamping Bulk_Update_Date__c with the current timestamp and Bulk_Update_Source__c = 'NPPES_Reconciliation'.

Given the batch job completes,
When results are available,
Then a Provider_Change_Notification__c (PCN) record is auto-created for each updated Provider__c with PCN.Change_Type__c = 'Demographic Update', PCN.Status__c = 'Pending Review', and PCN.Triggered_By__c = 'Bulk Import'.

Given the batch job encounters a DML error on any record,
When the error is caught,
Then the failed record is logged to a Bulk_Update_Error_Log__c custom object with fields: Error_Message__c, Provider_Id__c, NPI_Attempted__c, TIN_Attempted__c, and the remaining records continue processing (no all-or-nothing rollback).

-Given the specialist uploads files larger than 5MB, the system rejects the upload- (deprecated: file size limit increased to 25MB per INS-1489 platform readiness story)

Given the bulk update job completes successfully,
When the specialist views the job summary FlexCard,
Then it displays: Total Records Submitted, Records Updated, Records Failed, and a downloadable error report link.

## Solution

Implement an OmniScript ('Bulk Provider NPI TIN Update') with a File Upload step feeding into a DataRaptor Extract to parse CSV rows. A custom Apex invocable method handles pre-validation against NPPES format regex and duplicate NPI checks via SOQL. Batch Apex class 'BulkProviderDemographicUpdateBatch' (implements Database.Batchable, Database.AllowsCallouts) processes updates in chunks of 200. A Record-Triggered Flow on Provider__c (after update, when NPI__c or TIN__c changes) fires a Platform Event 'Provider_Data_Changed__e' to trigger downstream PCN creation. Results surfaced via a FlexCard on the Provider Network Specialist home page. Permission Set Group 'PSG_Provider_Data_Admin' controls access to the OmniScript.

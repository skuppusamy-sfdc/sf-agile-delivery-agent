## Description

As a Provider Network Specialist, I need a mass reassignment tool that transfers all active Provider_Network_Agreement__c (PNA) and Group_Service_Agreement__c (GSA) records from a source Facility__c (Business Account) to a target Facility__c when a facility merger or acquisition occurs, so that contracted network relationships are preserved without requiring individual record edits or re-contracting cycles.

## Acceptance Criteria

Given a Provider Network Specialist initiates a Facility Merger from the source Facility__c record using the 'Initiate Merger' quick action,
When the OmniScript 'Facility_Merger_Reassignment' launches,
Then the specialist can search for and select a target Facility__c using an Account lookup filtered to Type = 'Facility' and Status__c = 'Active'.

Given the source and target Facility__c records are selected,
When the specialist proceeds to the confirmation step,
Then the OmniScript displays a summary table listing: count of active PNA records (PNA.Facility__c = source, PNA.Status__c = 'Active'), count of active GSA records (GSA.Facility__c = source, GSA.Status__c = 'Active'), and count of open Claim__c records (Claim__c.Facility__c = source, Claim__c.Status__c NOT IN ('Closed','Paid','Denied')).

Given the specialist confirms the merger,
When the reassignment job runs,
Then all qualifying PNA records have PNA.Facility__c updated to the target Facility__c.Id, and PNA.Merger_Source_Facility__c (new field) is stamped with the original source Facility__c.Id for audit purposes.

Given a GSA record exists on the source Facility__c,
When the reassignment runs,
Then GSA.Facility__c is updated to the target Facility__c, and GSA.Notes__c is appended with 'Reassigned via Facility Merger on [DATE] from [Source Facility Name]'.

Given open Claim__c records exist on the source Facility__c,
When the reassignment completes,
Then open claims are NOT automatically reassigned; instead a Claim_Review_Task__c record is created for each open claim assigned to the Claims Operations Manager for manual review.

Given the reassignment job completes,
When all records are processed,
Then the source Facility__c.Status__c is updated to 'Merged' and Facility__c.Merged_Into__c is populated with the target Facility__c.Id.

## Solution

Build OmniScript 'Facility_Merger_Reassignment' with a DataRaptor Extract to retrieve PNA/GSA/Claim counts for the confirmation screen. Reassignment logic implemented in invocable Apex class 'FacilityMergerReassignmentService' called from an OmniScript Integration Procedure. Apex performs SOQL bulk queries and DML updates in batches of 200 using Database.update(records, false) for partial success handling. New custom fields: PNA.Merger_Source_Facility__c (Lookup to Account), Facility__c.Merged_Into__c (Lookup to Account), Facility__c.Status__c picklist value 'Merged' added via metadata. Quick Action on Facility__c record page launches OmniScript via LWC wrapper. Access controlled via Permission Set 'PS_Facility_Merger_Admin'.

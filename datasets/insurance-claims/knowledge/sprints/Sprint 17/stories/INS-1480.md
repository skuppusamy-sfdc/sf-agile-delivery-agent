## Description

As a Credentialing Specialist, I need the Salesforce platform to perform a nightly scheduled sync with the CAQH ProView API so that Provider__c credentialing data (board certifications, DEA license, malpractice history) is automatically pulled into the associated Credentialing_Case__c record, reducing manual data entry and ensuring credentialing decisions are based on current attestation data.

CAQH ProView REST API v3.0 will be used. Each Provider__c has a Provider__c.CAQH_Provider_ID__c field populated during initial onboarding. The sync should update only records where Provider__c.CAQH_Sync_Enabled__c = true.

## Acceptance Criteria

Given a Provider__c record where Provider__c.CAQH_Sync_Enabled__c = true and Provider__c.CAQH_Provider_ID__c is not null,
When the nightly scheduled Apex job 'ScheduledCAQHSync' runs at 02:00 UTC,
Then a callout is made to the CAQH ProView API endpoint GET /providers/{caqhProviderId}/credentialingstatus and the HTTP response is parsed.

Given a successful API response (HTTP 200),
When the response is processed,
Then the associated Credentialing_Case__c record (where CredCase.Provider__c = Provider__c.Id and CredCase.Status__c IN ('In Review', 'Pending CAQH Data')) must be updated with: CredCase.CAQH_Attestation_Date__c, CredCase.Board_Certification_Status__c, CredCase.DEA_Expiration_Date__c, CredCase.Malpractice_Coverage_Verified__c, and CredCase.CAQH_Last_Sync_DateTime__c = NOW().

Given a successful sync where CredCase.DEA_Expiration_Date__c is within 60 days of today,
When the record is updated,
Then a high-priority Task must be created on the Credentialing_Case__c assigned to the Credentialing Specialist queue with Subject = 'DEA License Expiring Soon – Action Required' and ActivityDate = CredCase.DEA_Expiration_Date__c - 30.

Given an API response of HTTP 404 (provider not found in CAQH),
When the error is encountered,
Then Provider__c.CAQH_Sync_Status__c must be set to 'Not Found in CAQH' and no Credentialing_Case__c update is performed; a Platform Event 'CAQH_Sync_Error__e' must be fired with the provider Id and error code.

Given an API response of HTTP 429 (rate limit exceeded),
When the error is encountered,
Then the batch must pause that record, log the failure to a custom object 'Integration_Error_Log__c' with fields: IEL.Integration_Name__c = 'CAQH ProView', IEL.Error_Code__c = '429', IEL.Record_Id__c = Provider__c.Id, and IEL.Retry_Scheduled__c = true; the record must be re-queued for the next nightly run.

Given the full batch completes,
When all records are processed,
Then a summary Platform Event 'CAQH_Sync_Completed__e' must be fired, consumed by a triggered Flow that emails the Credentialing Operations distribution list with: total providers synced, success count, error count, and a link to the Integration_Error_Log__c list view filtered by today's date.

-Given Provider__c.CAQH_Sync_Enabled__c = false, When the sync runs, Then a warning email should be sent to the Credentialing Specialist- (deprecated — no notification for disabled records per updated business requirements v2.3)

## Solution

1. Named Credential 'CAQH_ProView_API' configured with OAuth 2.0 client credentials flow; endpoint stored in Custom Metadata 'Integration_Endpoint__mdt'.
2. Apex Schedulable + Queueable chain: 'ScheduledCAQHSync' enqueues 'QueueableCAQHProviderSync' with batches of 50 providers to respect CAQH API rate limits.
3. Apex HTTP callout class 'CAQHProViewService' with methods for credential status retrieval and JSON deserialization into a typed wrapper class.
4. Record-Triggered Flow on Credentialing_Case__c (after update, DEA_Expiration_Date__c changes) for Task creation logic.
5. Platform Events 'CAQH_Sync_Error__e' and 'CAQH_Sync_Completed__e' with triggered Flows for error handling and completion notification.
6. Custom object 'Integration_Error_Log__c' with a dedicated list view and a scheduled purge job retaining 90 days of history.

## Description

**Bug Report — Incorrect Fee Schedule Rate on Claim Line Adjudication**

**Environment:** Acme Insurance Full Sandbox (Sprint 17)
**Reported By:** Emily Watson (Claims Operations Manager)
**Date Observed:** 2024-11-06
**Severity:** High — Financial impact on claim payments

**Steps to Reproduce:**
1. Identify a Provider__c who has two active Provider_Network_Agreement__c records: one for LOB = 'Commercial' and one for LOB = 'Medicare Advantage'.
2. Create a new Claim__c with Claim__c.LOB__c = 'Medicare Advantage' and assign the provider as Claim__c.Rendering_Provider__c.
3. Add a Claim_Line__c with CPT code '99213' and run adjudication via the 'Adjudicate Claim' button.
4. Observe the Claim_Line__c.Allowed_Amount__c that is populated.

**Expected Behavior:**
The FeeScheduleEngine should resolve the Fee_Schedule__c record matching BOTH the provider's active PNA for LOB = 'Medicare Advantage' AND the CPT code '99213', returning the Medicare Advantage contracted rate.

**Actual Behavior:**
The FeeScheduleEngine returns the rate from the Commercial LOB Fee_Schedule__c because the SOQL query in 'FeeScheduleLookupService.cls' uses 'LIMIT 1' without an ORDER BY clause, causing non-deterministic record selection when multiple PNA records exist for the same provider.

**Root Cause:**
In 'FeeScheduleLookupService.cls' line 87, the SOQL query:
`SELECT Id, Base_Rate_Percentage__c FROM Fee_Schedule__c WHERE Provider__c = :providerId AND CPT_Code__c = :cptCode LIMIT 1`
is missing the filter `AND LOB__c = :claimLOB`, causing it to return whichever Fee_Schedule__c record Salesforce retrieves first regardless of LOB.

**Financial Impact:**
Estimated 23 Claim_Line__c records in Sprint 17 sandbox adjudicated at incorrect rates. In production this pattern could result in over- or under-payment on Medicare Advantage claims.

**Related Story:** Extends INS-1498 (Fee Schedule Engine Refactor)

## Acceptance Criteria

As per Description.

Post-fix validation: Given a Provider__c has active PNAs for both 'Commercial' and 'Medicare Advantage' LOBs, when a Claim__c with LOB__c = 'Medicare Advantage' is adjudicated, then Claim_Line__c.Allowed_Amount__c must reflect the Fee_Schedule__c record where Fee_Schedule__c.LOB__c = 'Medicare Advantage' and Fee_Schedule__c.CPT_Code__c matches the claim line CPT code.

Given no matching Fee_Schedule__c exists for the claim's LOB and CPT code combination, when adjudication runs, then Claim_Line__c.Adjudication_Status__c = 'Pended — No Fee Schedule' and Claim__c.Status__c = 'Pended' with a Claim_Note__c created explaining the missing fee schedule.

## Solution

Fix 'FeeScheduleLookupService.cls' to include LOB__c filter in the SOQL query: `SELECT Id, Base_Rate_Percentage__c FROM Fee_Schedule__c WHERE Provider__c = :providerId AND CPT_Code__c = :cptCode AND LOB__c = :claimLOB AND Status__c = 'Active' ORDER BY Effective_Date__c DESC LIMIT 1`. Add unit tests covering single-PNA, multi-PNA same LOB, multi-PNA different LOB, and no-match scenarios. Deploy via unlocked package delta. Run post-deployment SOQL audit to identify affected Claim_Line__c records in sandbox for manual rate correction. Add a custom index on Fee_Schedule__c (Provider__c, CPT_Code__c, LOB__c) via a Salesforce support case to optimize query performance.

## Description

As a Provider Relations Specialist, I need Provider_Change_Notification__c (PCN) records to be automatically routed to the correct Salesforce queue based on the PCN.Change_Type__c value so that the right team receives and actions provider change requests without a manual triage step, reducing average PCN response time from 48 hours to under 8 hours.

## Acceptance Criteria

Given a new Provider_Change_Notification__c record is created (via any source: UI, Apex batch, API),
When PCN.Change_Type__c = 'NPI Change' or 'TIN Change',
Then PCN.OwnerId must be set to the 'Provider_Data_Integrity_Queue'.

Given a new PCN is created,
When PCN.Change_Type__c = 'Network Termination' or 'Network Addition',
Then PCN.OwnerId must be set to the 'Provider_Network_Contracts_Queue'.

Given a new PCN is created,
When PCN.Change_Type__c = 'Credentialing Status Change',
Then PCN.OwnerId must be set to the 'Credentialing_Review_Queue'.

Given a new PCN is created,
When PCN.Change_Type__c = 'Address Change' or 'Specialty Change',
Then PCN.OwnerId must be set to the 'Provider_Relations_Queue'.

Given a PCN is routed to any queue,
When routing completes,
Then PCN.Routing_Timestamp__c must be stamped and PCN.Routing_Status__c must be set to 'Routed'.

Given a PCN.Change_Type__c value is populated that does not match any configured routing rule,
When the routing flow evaluates the record,
Then PCN.OwnerId must default to the 'Provider_Relations_Queue' and PCN.Routing_Status__c must be set to 'Routed – Default' and a warning email sent to the Provider Network Operations distribution list.

Given a PCN has been in 'Routed' status for more than 8 business hours without PCN.Status__c being updated to 'In Progress' or 'Resolved',
When the SLA monitoring flow runs (every 30 minutes),
Then PCN.SLA_Breach__c must be set to TRUE and an escalation Task created for the queue manager.

## Solution

Record-Triggered Flow 'PCN_AutoRoute_RTF' on Provider_Change_Notification__c (After Insert, After Update on Change_Type__c). Flow uses a Decision element with outcome branches for each Change_Type__c value, assigning OwnerId to the appropriate queue via Get Records on Group object (Type = 'Queue', DeveloperName = queue API name). Custom metadata type 'PCN_Routing_Rule__mdt' stores Change_Type__c to Queue DeveloperName mappings for maintainability — flow reads from metadata rather than hard-coded Decision outcomes. Routing_Timestamp__c and Routing_Status__c stamped in the same flow via Update Records element. SLA breach monitoring implemented as a Scheduled Flow running every 30 minutes querying PCN records with SLA logic using business hours via BusinessHours.diff() Apex Invocable Method. Permission Set 'PCN_Manager_PS' grants queue managers visibility to SLA_Breach__c field on PCN list views.

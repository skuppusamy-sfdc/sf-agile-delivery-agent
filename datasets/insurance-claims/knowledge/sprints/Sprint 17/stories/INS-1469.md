## Description

As a Provider Network Specialist, I need the ability to perform bulk updates to Provider__c NPI and TIN fields via a controlled import process with pre-validation so that I can efficiently correct provider demographic data after the annual NPPES reconciliation cycle without creating duplicate or invalid records. This story extends INS-1439 (Provider NPPES Integration baseline).

## Acceptance Criteria

Given a Provider Network Specialist uploads a CSV file via the Bulk Provider Update Tool,
When the file contains NPI values,
Then each NPI must be validated against a 10-digit numeric format regex before any record is updated on Provider__c.NPI__c.

Given a valid import file is submitted,
When a TIN value on the import does not match the existing Provider__c.TIN__c and the Provider__c.PNA_Status__c = 'Active',
Then the system must create a Provider_Change_Notification__c record with PCN_Type__c = 'TIN Change' and PCN_Status__c = 'Pending Review' before committing the update.

Given the bulk import job completes,
When any row fails validation,
Then a Bulk_Import_Error_Log__c record must be created with fields: Error_Row__c, Error_Reason__c, Provider_Record_Id__c, and the failed row must NOT be updated.

Given all validations pass for a provider record,
When NPI__c is updated on Provider__c (Person Account),
Then the system must stamp Last_NPI_Verified_Date__c = TODAY() and NPI_Source__c = 'Bulk Import'.

Given a bulk job containing more than 500 records,
When the job is submitted,
Then the system must process records asynchronously via a Queueable Apex job and notify the submitting user via email upon completion with a summary of success and failure counts.

-Given a TIN change, automatically terminate the associated Fee_Schedule__c record- (deprecated: handled by INS-1451 termination flow)

Given the bulk import completes with any failures,
When the Provider Network Specialist views the import summary page,
Then they must see a count of: Total Submitted, Successfully Updated, Failed, PCNs Created.

## Solution

Build a custom LWC 'Bulk Provider Update Tool' surfaced on the Provider Network Admin App page. LWC handles CSV parsing client-side and calls an Apex controller (BulkProviderUpdateController.cls) which invokes a Queueable Apex class (BulkProviderUpdateQueueable.cls) for async processing of batches >200 records. Pre-validation logic checks NPI format via regex and queries existing Provider__c records for TIN mismatch. PCN creation handled via a Record-Triggered Flow on Provider__c firing on NPI__c or TIN__c field change (after-save context), calling a sub-flow 'Create PCN on Provider Change'. Bulk_Import_Error_Log__c is a custom object. Email notification uses Apex EmailService with a custom email template. Permission Set Group 'Provider_Network_Bulk_Admin' gates access to the tool.

## Description

As a Credentialing Specialist, I need a dedicated Salesforce dashboard and alert panel that surfaces all Credentialing_Case__c records with expiring credentials, pending CAQH verification, or overdue re-credentialing deadlines so that I can proactively manage credentialing queues, meet NCQA re-credentialing cycle requirements (every 36 months), and prevent providers from being inadvertently active in the network with lapsed credentials.

## Acceptance Criteria

Given a Credentialing Specialist opens the 'Credentialing Operations Dashboard',
When the dashboard loads,
Then it displays four report-based dashboard components: (1) Credentialing_Case__c records where Credential_Expiry_Date__c <= TODAY+60 days and Case_Status__c != 'Closed'; (2) Cases where CAQH_Verification_Status__c = 'Pending' for > 5 business days; (3) Cases where Re_Credentialing_Due_Date__c <= TODAY+90 days; (4) Cases where Case_Status__c = 'On Hold — Provider Unresponsive'.

Given a Credentialing_Case__c has Credential_Expiry_Date__c within 30 days,
When the dashboard panel refreshes,
Then the case row is displayed with a red urgency indicator and the field Days_Until_Expiry__c (formula field) is shown.

Given a CAQH verification has been pending for more than 5 business days,
When the alert panel detects this condition,
Then an automated task is created on the Credentialing_Case__c: Task.Subject = 'CAQH Follow-Up Required', Task.ActivityDate = TODAY, Task.OwnerId = Credentialing_Case__c.OwnerId.

Given the dashboard is accessed by a user without the 'Credentialing_Specialist_PSG' Permission Set Group,
When they navigate to the Credentialing Operations Dashboard tab,
Then they receive an access denied message and the dashboard components return no data.

Given a Credentialing_Case__c.CAQH_Provider_ID__c is populated,
When the specialist clicks 'Refresh CAQH Status' on the case record,
Then a callout is made to the CAQH ProView API and Credentialing_Case__c.CAQH_Verification_Status__c is updated with the latest status returned.

## Solution

Build the Credentialing Operations Dashboard using standard Salesforce CRM Analytics (formerly Einstein Analytics) or Lightning Dashboard with four report components backed by custom report types on Credentialing_Case__c. Create a formula field Days_Until_Expiry__c on Credentialing_Case__c. A Scheduled Flow (runs daily at 6 AM) evaluates CAQH pending duration and creates follow-up Tasks via the 'Create Records' element. The 'Refresh CAQH Status' button is an LWC quick action that calls an Apex controller (CAQHIntegrationController) which makes a Named Credential-based HTTP callout to the CAQH ProView REST API. Dashboard tab visibility is controlled by the 'Credentialing_Specialist_PSG' Permission Set Group with record-level access governed by Sharing Rules scoped to the Credentialing team's role hierarchy.

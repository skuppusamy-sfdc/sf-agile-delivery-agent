## Description

As a Credentialing Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record and a Credentialing_Case__c when a provider's license expiration date falls within 60 days of today so that re-credentialing workflows are initiated proactively and network compliance is maintained.

## Acceptance Criteria

Given a Provider__c record has License_Expiration_Date__c set,
When a scheduled daily Flow runs at 06:00 AM EST,
Then all Provider__c records where License_Expiration_Date__c <= TODAY+60 AND License_Expiration_Date__c >= TODAY and License_Alert_Sent__c = false are identified.

Given an eligible Provider__c record is identified,
When the Flow processes the record,
Then a Provider_Change_Notification__c is created with PCN_Type__c = 'License_Expiration_Warning', PCN_Status__c = 'Open', PCN_Due_Date__c = License_Expiration_Date__c - 30, and linked via PCN.Provider__c.

Given the PCN is created,
When the Flow continues,
Then a Credentialing_Case__c is created with Case_Type__c = 'License_Renewal', Case_Priority__c = 'High', linked to the same Provider__c, and assigned to the Credentialing queue.

Given the PCN and Credentialing_Case__c are created successfully,
When both records are committed,
Then Provider__c.License_Alert_Sent__c is set to true to prevent duplicate notifications.

Given a provider's license has already expired (License_Expiration_Date__c < TODAY),
When the scheduled Flow runs,
Then those records are excluded from this flow (handled by separate Network_Termination_Case__c workflow) and logged in Flow_Execution_Log__c with a note 'Expired — escalated to termination workflow'.

## Solution

Implement a Scheduled Flow 'License_Expiration_Alert_SF' with a Get Records element querying Provider__c with the date window filter. Loop element processes each record, creating PCN and Credentialing_Case__c via Create Records elements in a single transaction. A fault path writes errors to a custom Flow_Execution_Log__c object. License_Alert_Sent__c reset logic handled by a separate monthly maintenance Flow that resets the flag after license renewal is confirmed (Credentialing_Case__c.Status__c = 'Closed — Renewed'). Custom Metadata record stores the 60-day threshold to allow ops configuration without code changes.

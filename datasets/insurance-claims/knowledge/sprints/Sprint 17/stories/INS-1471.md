## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record when a Provider_Network_Agreement__c is approaching expiration within 60 days and no renewal has been initiated, so that the network team can proactively manage contract continuity and avoid inadvertent out-of-network gaps for active claimants.

## Acceptance Criteria

Given a scheduled daily batch process runs at 6:00 AM EST,
When a Provider_Network_Agreement__c record has PNA_Status__c = 'Active' AND Termination_Date__c <= TODAY() + 60 AND Termination_Date__c >= TODAY(),
And no Network_Termination_Case__c exists with Termination_Case_Status__c NOT IN ('Closed', 'Withdrawn') linked to that PNA,
Then a new Network_Termination_Case__c must be created with: Termination_Reason__c = 'Contract Expiry', Priority__c = 'High', Assigned_To__c = PNA's owning Provider Network Specialist, and Source_PNA__c = PNA record Id.

Given a Network_Termination_Case__c is created,
When the record is saved,
Then an in-app notification and email must be sent to the assigned Provider Network Specialist using the 'PNA Expiry Warning' email template.

Given a PNA has Termination_Date__c <= TODAY() + 30,
When the termination case is created,
Then Priority__c must be set to 'Critical' and an escalation task must be created assigned to the Provider Network Manager (sourced from Custom Metadata 'Network_Manager_Config__mdt').

Given a renewal PNA is created with PNA_Status__c = 'Pending' linked to the same Provider__c and same Network_Name__c,
When the daily batch evaluates the expiring PNA,
Then no duplicate Network_Termination_Case__c should be created and any existing open case should have Termination_Case_Status__c updated to 'Renewal In Progress'.

Given the batch job completes,
When the job log is reviewed,
Then a Batch_Job_Log__c record must be created with: Total_PNAs_Evaluated__c, Cases_Created__c, Cases_Skipped__c, and Job_Run_Date__c.

## Solution

Implement a Salesforce Scheduled Apex class (PNAExpiryBatchScheduler.cls) invoking a Batchable Apex class (PNAExpiryBatchProcessor.cls) with batch size 200. Query scope filters Provider_Network_Agreement__c by PNA_Status__c and Termination_Date__c range. Duplicate prevention logic queries existing Network_Termination_Case__c records. Priority logic implemented inline in batch execute method. Email notifications via Messaging.SingleEmailMessage using custom email template. Escalation task creation via Task object insert. Custom Metadata Type 'Network_Manager_Config__mdt' stores manager User Id per LOB. Batch_Job_Log__c inserted in batch finish method. Scheduled via Apex Scheduler to run daily at 6 AM EST.

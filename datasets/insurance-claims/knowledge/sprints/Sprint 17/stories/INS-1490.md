## Description

As a Credentialing Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record whenever a Credentialing_Case__c status transitions to 'Approved' or 'Denied' so that Provider Relations Specialists and downstream claims adjudication are immediately informed of credentialing outcomes without manual PCN creation.

## Acceptance Criteria

Given a Credentialing_Case__c record exists with Status__c = 'Under Review',
When a Credentialing Specialist updates Credentialing_Case__c.Status__c to 'Approved',
Then a new Provider_Change_Notification__c is auto-created with: PCN__c.Notification_Type__c = 'Credentialing Approval', PCN__c.Provider__c = Credentialing_Case__c.Provider__c, PCN__c.Effective_Date__c = today, PCN__c.Triggered_By__c = running user, PCN__c.Source_Case__c = Credentialing_Case__c.Id.

Given a Credentialing_Case__c.Status__c transitions to 'Denied',
When the PCN is auto-created,
Then PCN__c.Notification_Type__c = 'Credentialing Denial' and PCN__c.Denial_Reason__c is populated from Credentialing_Case__c.Denial_Reason__c.

Given the PCN record is created,
When PCN__c.Notification_Type__c = 'Credentialing Approval',
Then the linked Provider__c.Credentialing_Status__c is updated to 'Credentialed' and Provider__c.Credentialing_Expiry_Date__c is set to today + 730 days (2-year cycle).

Given the PCN is created for a Denial,
When any open Claim__c records exist with Claim__c.Rendering_Provider__c = the denied provider and Claim__c.Status__c IN ('Pending', 'In Adjudication'),
Then those claims are flagged: Claim__c.Provider_Credentialing_Hold__c = true and a Task is created for the Claims Operations Manager.

-Given PCN is generated, send fax notification to provider office — DEPRECATED, replaced by DocuSign email workflow-

Given the PCN record is created,
When the PCN__c.Status__c = 'New',
Then a DocuSign envelope is automatically sent to the provider's email on file (Provider__c.Email) using the 'PCN_Acknowledgement' DocuSign template.

## Solution

Implement a Record-Triggered Flow (after save) on Credentialing_Case__c firing on Status__c changes to 'Approved' or 'Denied'. Flow creates Provider_Change_Notification__c via a Create Records element and updates Provider__c fields via a related record update. A separate invocable Apex action (ClaimCredentialingHoldAction) is called from the flow to bulk-update open Claim__c records to avoid governor limits on large claim volumes. DocuSign envelope triggered via DocuSign for Salesforce native flow action 'Send with DocuSign'.

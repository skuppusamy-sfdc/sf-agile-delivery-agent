# INS-1478: Automated Provider Termination Case Creation on PNA Expiry

## Description

As a Compliance Officer, I need the system to automatically create a Network_Termination_Case__c record when a Provider_Network_Agreement__c reaches its termination date without a renewal so that the provider termination workflow is initiated promptly and regulatory notification timelines (typically 90-day written notice per state mandate) are not missed.

Background: Currently, PNA expirations are caught manually during weekly audits, leading to providers remaining in-network post-termination and resulting in incorrect Claim_Line__c adjudication at the contracted rate.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where PNA.Status__c = 'Active' and PNA.Termination_Date__c = TODAY() and no renewal PNA exists (no active PNA with same PNA.Provider__c and PNA.Effective_Date__c > PNA.Termination_Date__c),
When a scheduled Apex job (ScheduledPNAExpiryCheck) runs at 00:30 UTC daily,
Then a Network_Termination_Case__c must be created with: NTC.Provider__c = PNA.Provider__c, NTC.PNA__c = PNA.Id, NTC.Termination_Reason__c = 'Contract Expiry – No Renewal', NTC.Status__c = 'Open', NTC.Regulatory_Notice_Due_Date__c = TODAY() + 90, NTC.LOB__c = PNA.LOB__c.

Given the Network_Termination_Case__c is created,
When the record is inserted,
Then a Record-Triggered Flow must update PNA.Status__c from 'Active' to 'Pending Termination' and stamp PNA.Termination_Case__c = NTC.Id.

Given PNA.Status__c = 'Pending Termination',
When a new Claim_Line__c is submitted with Claim_Line__c.Rendering_Provider__c = PNA.Provider__c,
Then the Claims Processing engine must apply an out-of-network fee schedule and set Claim_Line__c.Network_Status_At_DOS__c = 'Out-of-Network – Pending Termination' and flag the parent Claim__c with Claim__c.Requires_Manual_Review__c = true.

Given a Network_Termination_Case__c is open,
When the assigned Provider Relations Specialist sets NTC.Status__c = 'Reinstatement Approved',
Then a Flow must revert PNA.Status__c to 'Active', clear PNA.Termination_Case__c, and create a new PCN record with PCN.Change_Type__c = 'Reinstatement' and PCN.Status__c = 'Pending Review'.

Given NTC.Regulatory_Notice_Due_Date__c is 10 days away and NTC.Status__c != 'Notice Sent',
When the daily scheduled job runs,
Then a high-priority Task must be created on the NTC record assigned to the Compliance Officer queue with Subject = 'ACTION REQUIRED: Regulatory Termination Notice Due in 10 Days'.

## Solution

1. Apex Schedulable class 'ScheduledPNAExpiryCheck' executing a SOQL query for qualifying PNA records and performing DML insert of Network_Termination_Case__c in bulk.
2. Record-Triggered Flow on Network_Termination_Case__c (after insert) to update parent PNA fields.
3. Apex Trigger on Claim_Line__c (before insert) calling a PNA status lookup via a custom metadata-driven network eligibility service class.
4. Scheduled Flow running daily at 00:30 UTC for the 10-day notice Task creation, querying NTC records by due date delta.
5. Custom Metadata Type 'Network_Termination_Config__mdt' storing the notice lead time (10 days) and regulatory notice period (90 days) to avoid hardcoding.

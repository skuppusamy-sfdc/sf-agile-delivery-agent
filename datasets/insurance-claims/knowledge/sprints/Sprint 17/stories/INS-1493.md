## Description

As a Provider Network Specialist, I need the system to automatically inactivate all active Provider_Network_Agreement__c (PNA) records and trigger downstream claims notifications when a Network_Termination_Case__c is moved to 'Approved' status so that terminated providers are removed from the network in a single coordinated action and any in-flight claims are flagged for review per CMS termination notification requirements.

## Acceptance Criteria

Given a Network_Termination_Case__c record with Status__c = 'Pending Approval' exists,
When an authorized user (with 'Network_Termination_Manager' Permission Set) updates Status__c to 'Approved',
Then the system retrieves all Provider_Network_Agreement__c records where PNA__c.Provider__c = Network_Termination_Case__c.Provider__c AND PNA__c.Status__c = 'Active'.

Given the active PNAs are retrieved,
When the termination is approved,
Then each PNA__c.Status__c is set to 'Terminated', PNA__c.Termination_Date__c = Network_Termination_Case__c.Effective_Termination_Date__c, and PNA__c.Termination_Reason__c = Network_Termination_Case__c.Termination_Reason__c.

Given PNAs are terminated,
When Provider__c.Network_Status__c is updated,
Then Provider__c.Network_Status__c = 'Terminated' and Provider__c.Termination_Effective_Date__c = Network_Termination_Case__c.Effective_Termination_Date__c.

Given the termination is processed,
When Claim__c records exist with Claim__c.Rendering_Provider__c = terminated provider AND Claim__c.Status__c IN ('FNOL', 'Pending', 'In Adjudication'),
Then each such claim is updated: Claim__c.Provider_Termination_Hold__c = true, Claim__c.Hold_Reason__c = 'Provider Network Termination — Review Required', and a Case is created linked to each flagged Claim__c assigned to the Claims Operations Manager queue.

Given the termination notification compliance requirement,
When the termination is approved and Effective_Termination_Date__c is >= 30 days in the future,
Then a Provider_Change_Notification__c is created with PCN__c.Notification_Type__c = 'Network Termination', PCN__c.Advance_Notice_Days__c = DATEDIFF(Effective_Termination_Date__c, today), and PCN__c.CMS_Reporting_Required__c = true.

Given the entire termination process completes,
When all PNAs are inactivated and claims flagged,
Then Network_Termination_Case__c.Termination_Processing_Status__c = 'Completed' and a summary chatter post is added to the case listing counts: PNAs terminated, Claims held.

## Solution

Implement a Record-Triggered Flow on Network_Termination_Case__c (after save, on Status__c change to 'Approved'). Flow calls two invocable Apex actions to handle bulk DML safely: (1) TerminatePNAsBulkAction — queries and bulk-updates PNA records and Provider__c; (2) FlagClaimsForTerminationAction — queries and bulk-updates in-flight Claim__c records and creates Cases. PCN creation handled within the Flow via Create Records element. Chatter post via Flow Post to Chatter action. All Apex actions implement Database.Stateful patterns for error collection. Permission gated via 'Network_Termination_Manager' Permission Set Group with a Custom Permission 'Approve_Network_Termination'.

## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) and Facility__c (Business Account) record pages that surfaces all active Provider_Network_Agreement__c (PNA), Fee_Schedule__c, and Group_Service_Agreement__c (GSA) entitlements in a single consolidated view so that I can quickly assess a provider's network standing and contractual entitlements without navigating multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then it displays all related PNA records where PNA.Status__c = 'Active' and PNA.Provider__c = Provider__c.Id, showing fields: PNA.Network_Name__c, PNA.Effective_Date__c, PNA.Termination_Date__c, PNA.LOB__c.

Given the FlexCard is loaded,
When the specialist clicks the 'Fee Schedules' tab within the card,
Then all Fee_Schedule__c records linked via PNA are displayed with columns: Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.CPT_Code_Range__c, Fee_Schedule__c.Allowed_Amount__c, Fee_Schedule__c.Effective_Date__c.

Given a provider has one or more GSA records,
When the 'GSA' tab is selected on the FlexCard,
Then Group_Service_Agreement__c records where GSA.Provider__c = Provider__c.Id and GSA.Status__c IN ('Active', 'Pending Renewal') are listed.

Given a PNA record has PNA.Termination_Date__c within 30 days of today,
When the FlexCard renders,
Then that PNA row is highlighted with a warning badge labeled 'Expiring Soon'.

-Given the FlexCard includes a 'Claims History' mini-list showing last 5 Claim__c records- (deprecated — Claims History moved to separate Claims 360 component per design review)

Given the Provider__c record has no active PNAs,
When the FlexCard loads,
Then a message 'No Active Network Agreements found' is displayed and a quick-action button 'Initiate PNA' is surfaced.

## Solution

Build the Provider 360 FlexCard using OmniStudio FlexCards with child FlexCards for PNA, Fee Schedule, and GSA tabs. DataRaptors (Extract type) retrieve related records using Provider__c.Id as the input key. A DataRaptor Transform formats the expiration warning logic (date diff < 30 days sets a flag field consumed by the FlexCard state). The FlexCard is deployed to both Provider__c (Person Account) and Facility__c (Business Account) Lightning record pages via App Builder. A conditional state on the FlexCard drives the 'Expiring Soon' badge CSS class. The 'Initiate PNA' quick action launches an OmniScript for PNA creation. Permission access controlled via the existing 'Provider_Relations_PSG' Permission Set Group.

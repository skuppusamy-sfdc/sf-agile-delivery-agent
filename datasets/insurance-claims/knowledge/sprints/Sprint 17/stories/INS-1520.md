## Description

**Bug Report — Provider Termination Date Not Propagating to Claim Lines**

**Environment:** Acme Insurance Production Sandbox (Sprint 17 refresh)

**Steps to Reproduce:**
1. Open an active Provider_Network_Agreement__c record where PNA.Status__c = 'Active' and PNA.Agreement_End_Date__c is NULL.
2. Update PNA.Status__c to 'Terminated' and set PNA.Termination_Date__c = TODAY().
3. Save the PNA record.
4. Navigate to any open Claim__c records linked to this provider where Claim__c.Status__c = 'In Adjudication'.
5. Inspect associated Claim_Line__c records.

**Expected Behavior:**
All Claim_Line__c records where Claim_Line__c.Service_Date__c >= PNA.Termination_Date__c should have Claim_Line__c.Provider_Network_Status__c updated to 'Out of Network' and Claim_Line__c.Termination_Flag__c set to TRUE.

**Actual Behavior:**
Claim_Line__c.Provider_Network_Status__c remains 'In Network' and Claim_Line__c.Termination_Flag__c remains FALSE after PNA termination. This causes incorrect in-network reimbursement rates to be applied during adjudication, resulting in overpayments.

**Impact:** High — Financial overpayment risk on active claims for terminated providers. Compliance Officer (Amanda Foster) has flagged this for urgent remediation.

**Root Cause (Initial Analysis):** The Record-Triggered Flow ('PNA_Termination_Flow') that fires on PNA.Status__c change does not include a subflow or related record update step targeting Claim_Line__c. The flow only updates the parent Provider__c.Network_Status__c field.

**Frequency:** Reproducible 100% of the time across all LOBs.

## Acceptance Criteria

As per Description.

Additional verification: After fix deployment, confirm Claim_Line__c.Provider_Network_Status__c = 'Out of Network' and Claim_Line__c.Termination_Flag__c = TRUE for all affected lines within 5 minutes of PNA termination save.

## Solution

Extend the existing Record-Triggered Flow ('PNA_Termination_Flow') to include a 'Get Records' element querying Claim_Line__c where Claim_Line__c.Provider_NPA_Id__c = PNA.Id AND Claim_Line__c.Service_Date__c >= {!PNA.Termination_Date__c} AND Claim_Line__c.Claim__r.Status__c = 'In Adjudication'. Add an 'Update Records' element to set Provider_Network_Status__c = 'Out of Network' and Termination_Flag__c = TRUE. For PNA records with large claim volumes (>1000 lines), route update to an invocable Apex method (ClaimLineTerminationUpdater) called asynchronously via a Queueable job to avoid governor limits. Add a Platform Event (PNA_Termination__e) to notify downstream payment engine to re-adjudicate affected claim lines. Write unit tests covering bulkified scenarios with 200+ Claim_Line__c records per PNA.

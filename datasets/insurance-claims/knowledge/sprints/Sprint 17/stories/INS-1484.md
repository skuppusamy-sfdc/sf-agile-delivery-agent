## Description

As a Provider Relations Specialist, I need the Provider 360 FlexCard on the Provider__c record page to display the provider's currently active Fee_Schedule__c entries and flag any Provider_Network_Agreement__c records expiring within 60 days so that I can proactively initiate contract renewals and avoid claim payment rate discrepancies.

Extends INS-1453 (Provider 360 base FlexCard implementation).

## Acceptance Criteria

Given a Provider Relations Specialist opens any Provider__c (Person Account) record page,
When the Provider 360 FlexCard renders,
Then it displays a 'Fee Schedules' tile listing all Fee_Schedule__c records where Fee_Schedule__c.Status__c = 'Active' and Fee_Schedule__c.Provider__c = current record Id, showing columns: Schedule_Name__c, LOB__c, Effective_Date__c, Expiration_Date__c.

Given a provider has one or more Provider_Network_Agreement__c records where PNA.Expiration_Date__c <= TODAY() + 60 AND PNA.Status__c = 'Active',
When the FlexCard renders,
Then an amber 'Expiring Soon' badge appears on the PNA tile for each qualifying agreement, displaying the PNA.Network_Name__c and exact PNA.Expiration_Date__c.

Given a PNA has already expired (PNA.Expiration_Date__c < TODAY()),
When the FlexCard renders,
Then the PNA tile displays a red 'Expired' badge and PNA.Status__c is shown as 'Expired'.

Given the FlexCard is displayed,
When the Provider Relations Specialist clicks 'Initiate Renewal' on an expiring PNA tile,
Then a new Network_Termination_Case__c is NOT created; instead a new PNA record is cloned with PNA.Status__c = 'Pending Renewal' and the user is navigated to the cloned record's edit page.

-Given no active fee schedules exist, When the FlexCard renders, Then the Fee Schedules tile is hidden entirely.- (Superseded: tile must always render; show 'No Active Fee Schedules Found' empty state message instead)

Given no active Fee_Schedule__c records exist for the provider,
When the FlexCard renders,
Then the Fee Schedules tile displays the empty state message 'No Active Fee Schedules on File' with a 'Create Fee Schedule' quick action button.

## Solution

1. Extend the existing Provider 360 FlexCard (INS-1453) by adding two new card blocks: 'FeeSchedules' and 'PNAAlerts'.
2. Create two DataRaptor Turbo Extract definitions: 'DR_GetActiveFeeSchedules' (SOQL on Fee_Schedule__c filtered by Status__c = 'Active') and 'DR_GetExpiringPNAs' (SOQL on PNA with date formula filter TODAY()+60).
3. Bind FlexCard conditional styles to PNA.Expiration_Date__c: amber badge when days-until-expiry between 1–60, red badge when <= 0.
4. 'Initiate Renewal' action button invokes an Integration Procedure 'IP_ClonePNAForRenewal' which performs a DataRaptor Load to clone the PNA record.
5. Empty state handled via FlexCard conditional visibility on the card block using {feeSchedules.length} == 0 expression.
6. FlexCard deployed to Provider__c Lightning Record Page via App Builder, visible only to users with 'Provider_Relations' Permission Set.

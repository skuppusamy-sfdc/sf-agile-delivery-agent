## Description

As an Assignment Coordinator, I need a workflow to mass reassign provider-to-facility affiliations (Provider__c to Facility__c Account relationships tracked via a junction object Provider_Facility_Affiliation__c) when a Facility__c undergoes a merger, acquisition, or closure so that all affected Provider__c records reflect the correct Facility__c affiliation without requiring individual record edits.

This need arises from a recent hospital network merger impacting 340 affiliated providers. The current manual process took 4 business days and introduced 23 data errors.

## Acceptance Criteria

Given an Assignment Coordinator with the 'Facility_Affiliation_Manager' Permission Set,
When they open the 'Mass Facility Reassignment' OmniScript,
Then they must be able to search and select a source Facility__c (Account.RecordType = 'Healthcare Facility') and a target Facility__c, and see a count of active Provider_Facility_Affiliation__c records (PFA.Facility__c = source, PFA.Status__c = 'Active') before proceeding.

Given source and target facilities are selected,
When the user clicks 'Preview Affected Providers',
Then a DataRaptor Extract must populate a table showing: Provider__c.Name, Provider__c.NPI__c, PFA.Affiliation_Type__c, PFA.Effective_Date__c for all qualifying records, with a checkbox to exclude individual providers from the mass update.

Given the preview is confirmed and the user clicks 'Execute Reassignment',
When the Apex Batch job 'BatchFacilityAffiliationReassign' runs,
Then for each selected PFA record: PFA.Facility__c must be updated to target Facility__c.Id, PFA.Reassignment_Date__c must be stamped with TODAY(), PFA.Previous_Facility__c must retain the source Facility__c.Id for audit, and a PCN record must be created with PCN.Change_Type__c = 'Facility Reassignment', PCN.Provider__c = PFA.Provider__c.

Given the batch completes,
When any PFA update fails (e.g., validation rule on target facility credentialing requirement),
Then the failed record must be logged to 'Integration_Error_Log__c' with IEL.Integration_Name__c = 'Facility Reassignment' and the Assignment Coordinator must receive an error summary email with a downloadable list of failed provider NPIs and failure reasons.

Given the reassignment is complete,
When a Claims Processor adjudicates a Claim_Line__c for a reassigned provider,
Then Claim_Line__c.Facility__c must resolve to the new target Facility__c.Id based on the updated PFA record, ensuring correct facility-level fee schedule application.

## Solution

1. OmniScript 'MassFacilityReassignment' with a Lookup step for source/target Facility__c, a DataRaptor Extract step for preview population, and a confirmation step with an exclusion checkbox list.
2. Apex Batch class 'BatchFacilityAffiliationReassign' (batch size 200) implementing Database.Batchable and Database.Stateful for error collection.
3. Record-Triggered Flow on Provider_Facility_Affiliation__c (after update, Facility__c changes) to fire PCN creation via invocable Apex.
4. Custom field PFA.Previous_Facility__c (Lookup to Account) added to Provider_Facility_Affiliation__c for audit trail.
5. Permission Set 'Facility_Affiliation_Manager' created with access to the OmniScript, Apex class, and relevant object CRUD permissions.

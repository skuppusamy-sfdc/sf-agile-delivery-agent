## Description

As a Provider Network Specialist, I need a bulk Fee Schedule clone utility that allows me to copy all active Fee_Schedule__c records for a given Network and LOB into a new contract year (e.g., 2024 → 2025) with updated effective and expiration dates, so that I can efficiently prepare fee schedules for annual contract rollovers without manually recreating hundreds of CPT code rate entries.

## Acceptance Criteria

Given a Provider Network Specialist accesses the 'Fee Schedule Rollover Tool' from the Provider Network Admin App,
When they select a Source_Network__c, Source_LOB__c, and New_Contract_Year__c (e.g., 2025),
Then the system must display a preview list of all Fee_Schedule__c records matching the criteria with Fee_Schedule_Status__c = 'Active'.

Given the specialist confirms the rollover action,
When the clone job executes,
Then for each source Fee_Schedule__c, a new Fee_Schedule__c must be created with: Effective_Date__c = January 1 of New_Contract_Year__c, Expiration_Date__c = December 31 of New_Contract_Year__c, Fee_Schedule_Status__c = 'Draft', Source_Fee_Schedule__c = Id of the cloned record, and all CPT code rate child records (Fee_Schedule_Line__c) must be deep-cloned.

Given the clone job completes,
When any Fee_Schedule__c or Fee_Schedule_Line__c record fails to clone,
Then the entire batch for that parent Fee_Schedule__c must be rolled back (all-or-nothing per parent) and logged in a Rollover_Error_Log__c record.

Given the cloned Fee_Schedule__c records are in 'Draft' status,
When a specialist reviews and approves a cloned record,
Then Fee_Schedule_Status__c must transition to 'Pending Approval' and trigger the existing Fee Schedule Approval Process.

-Given the rollover, automatically activate cloned fee schedules on January 1 of the new year- (deprecated: auto-activation removed per Compliance Officer review — manual activation required)

Given the rollover job processes more than 300 parent fee schedules,
When the job is submitted,
Then it must execute asynchronously and the user must receive a platform notification upon completion with counts: Cloned Successfully, Failed, Total CPT Lines Copied.

Given a rollover job is already in progress for the same Network + LOB + Year combination,
When a second rollover request is submitted,
Then the system must reject it with the message: 'A rollover job for [Network] [LOB] [Year] is already in progress or completed. Please review existing Draft records.'

## Solution

Build LWC 'Fee Schedule Rollover Tool' with a 3-step wizard: (1) Selection — SOQL-driven filtered list of eligible Fee_Schedule__c records; (2) Preview — DataTable showing records to be cloned; (3) Confirmation and async execution. Apex controller (FeeScheduleRolloverController.cls) validates no duplicate rollover exists and enqueues a Queueable Apex chain (FeeScheduleRolloverQueueable.cls) for deep clone of parent + child Fee_Schedule_Line__c records. Savepoint/rollback pattern used per parent record for atomicity. Rollover_Error_Log__c custom object stores failures. Platform Event (Rollover_Complete__evt) fired on job finish, consumed by LWC for real-time status update. Custom Metadata 'Rollover_Config__mdt' stores max batch size. Permission Set Group 'Fee_Schedule_Admin_PSG' required.

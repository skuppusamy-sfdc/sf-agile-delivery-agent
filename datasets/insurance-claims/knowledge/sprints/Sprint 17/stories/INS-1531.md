## Description

**Bug Report — INS-1531**

**Summary:** Network_Termination_Case__c records do not automatically transition to Status__c = 'Closed' when all associated Termination_Checklist_Item__c child records have Checklist_Item__c.Completed__c = TRUE. The status remains 'In Progress' indefinitely, requiring manual intervention by Provider Relations Specialists.

**Steps to Reproduce:**
1. Open any Network_Termination_Case__c record with Status__c = 'In Progress'.
2. Navigate to the Termination Checklist related list.
3. Mark all Termination_Checklist_Item__c child records as Completed__c = TRUE (one by one or via the 'Mark All Complete' button).
4. Refresh the Network_Termination_Case__c parent record.
5. Observe that NTC__c.Status__c remains 'In Progress' and does not update to 'Closed'.

**Expected Behavior:** When all child Termination_Checklist_Item__c records for a Network_Termination_Case__c are marked Completed__c = TRUE, the parent NTC__c.Status__c should automatically update to 'Closed' and NTC__c.Closed_Date__c should be set to TODAY().

**Actual Behavior:** NTC__c.Status__c stays 'In Progress'. NTC__c.Closed_Date__c is never populated.

**Root Cause (suspected):** The Record-Triggered Flow on Termination_Checklist_Item__c that was deployed in Sprint 15 (INS-1487) has a filter condition checking NTC__c.All_Items_Complete__c (a roll-up summary field), but the roll-up summary recalculation appears to run after the Flow trigger evaluation, causing a timing issue.

**Environment:** Full Sandbox and Production.

## Acceptance Criteria

As per Description. Regression: After fix, marking all checklist items complete on a test NTC must result in NTC__c.Status__c = 'Closed' and NTC__c.Closed_Date__c = TODAY() within one record save cycle without manual refresh.

## Solution

Investigate and resolve the roll-up summary timing issue by replacing the dependency on NTC__c.All_Items_Complete__c roll-up in the Flow trigger. Instead, update the Record-Triggered Flow on Termination_Checklist_Item__c (after update, when Completed__c changes to TRUE) to execute an Apex action (ChecklistCompletionChecker) that queries COUNT of incomplete sibling records in-transaction and performs a direct DML update on the parent Network_Termination_Case__c if count = 0. Remove the stale roll-up summary field dependency from the Flow entry criteria.

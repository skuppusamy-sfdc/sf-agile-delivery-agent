## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard that surfaces all active Provider_Network_Agreement__c (PNA) and Group_Service_Agreement__c (GSA) records associated with a provider, including expiry date alerts, so that I can proactively manage contract renewals and avoid lapses that could impact claims adjudication. Extends INS-1481 (Provider 360 base FlexCard foundation).

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c (Person Account) record page,
When the Provider 360 FlexCard loads,
Then it must display a tabbed section 'Agreements' listing all related PNA records where PNA.Status__c = 'Active' and PNA.Provider__c = current record Id, showing fields: PNA.Agreement_Number__c, PNA.Effective_Date__c, PNA.Expiration_Date__c, PNA.LOB__c, PNA.Network_Tier__c.

Given the FlexCard loads agreement data,
When a PNA.Expiration_Date__c is within 90 days of today,
Then the expiry date cell must render with an amber warning badge and tooltip text 'Expiring Soon'.

Given the FlexCard loads agreement data,
When a PNA.Expiration_Date__c is within 30 days of today or already past,
Then the expiry date cell must render with a red critical badge and tooltip text 'Action Required'.

Given the Agreements tab is active,
When the specialist clicks 'View GSAs',
Then a secondary DataRaptor-powered list must display related GSA records where GSA.Provider__c = current record Id and GSA.Status__c IN ('Active', 'Pending Renewal'), showing GSA.Agreement_Name__c, GSA.TPA__c, GSA.Renewal_Date__c.

Given no active PNA or GSA records exist for the provider,
When the Agreements tab renders,
Then display the empty-state message: 'No active agreements found. Contact Network Contracting to initiate a new PNA.'

## Solution

Extend the existing Provider 360 OmniStudio FlexCard (INS-1481 base card) by adding a child FlexCard 'ProviderAgreementsCard'. Use two DataRaptor Extract actions: 'DR_GetActivePNAsByProvider' (SOQL on Provider_Network_Agreement__c filtered by Provider__c and Status__c = 'Active') and 'DR_GetGSAsByProvider' (SOQL on Group_Service_Agreement__c). Implement conditional styling state blocks on the FlexCard for expiry thresholds using formula fields TODAY() comparisons. Deploy FlexCard to the Provider__c Lightning Record Page via App Builder. Ensure DataRaptors respect org-wide sharing by running in user context.

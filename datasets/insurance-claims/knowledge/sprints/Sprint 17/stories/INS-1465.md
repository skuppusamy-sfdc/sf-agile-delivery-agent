## Description

As a Provider Network Specialist, I need an automated workflow that, upon creation of a Network_Termination_Case__c record, orchestrates the end-to-end provider termination process — including PNA status updates, fee schedule deactivation, DocuSign termination notice delivery, and directory update triggers — so that provider terminations are executed consistently, within CMS-mandated notice periods, and with a complete audit trail.

## Acceptance Criteria

Given a Network_Termination_Case__c (NTC) record is created with NTC.Termination_Reason__c populated and NTC.Effective_Termination_Date__c set,
When NTC.Status__c is set to 'Approved',
Then all related Provider_Network_Agreement__c records where PNA.Provider__c = NTC.Provider__c and PNA.Status__c = 'Active' are updated to PNA.Status__c = 'Pending Termination' and PNA.Termination_Date__c = NTC.Effective_Termination_Date__c.

Given NTC.Status__c = 'Approved',
When the workflow executes,
Then a DocuSign envelope is sent to the provider's email (Provider__c.Email) containing the termination notice template (DocuSign Template ID stored in Custom Metadata: TerminationNotice_Template__mdt) within 2 hours of approval, and NTC.DocuSign_Envelope_ID__c is populated.

Given the DocuSign envelope is sent,
When the provider acknowledges receipt (DocuSign status = 'Completed'),
Then NTC.Notice_Acknowledged__c = TRUE and NTC.Notice_Acknowledged_Date__c = acknowledgment timestamp.

Given NTC.Effective_Termination_Date__c is reached,
When a scheduled Apex job runs nightly,
Then PNA.Status__c is updated from 'Pending Termination' to 'Terminated', all linked Fee_Schedule__c records are set to Fee_Schedule__c.Active__c = FALSE, and a Platform Event (ProviderTerminated__e) is published.

Given a ProviderTerminated__e event is published,
When the event is consumed,
Then the provider's record in the CMS network directory integration queue is flagged for removal (DataIntegration staging object: Directory_Sync_Queue__c with Sync_Action__c = 'Remove').

Given NTC.Termination_Reason__c = 'For Cause',
When NTC is created,
Then NTC is automatically escalated to the Compliance Officer role queue and NTC.Priority__c = 'Critical'.

-Given the NTC workflow sends a paper mail notice via the legacy mailing vendor API- (deprecated — paper mail notice requirement removed per Compliance sign-off 2024-Q3)

Given the termination workflow completes,
When all PNA records are in 'Terminated' status,
Then NTC.Status__c is automatically set to 'Closed' and a summary Chatter post is added to the NTC record listing all terminated PNA IDs.

## Solution

Implement a Record-Triggered Flow on Network_Termination_Case__c (After Save, fires on Status__c change to 'Approved'). The flow calls an Apex invocable action (TerminationOrchestrationService) which: (1) queries and bulk-updates related PNAs, (2) invokes the DocuSign for Salesforce managed package API to send the envelope using template ID from TerminationNotice_Template__mdt Custom Metadata, and (3) publishes the ProviderTerminated__e Platform Event. A separate DocuSign Connect webhook callback updates NTC.Notice_Acknowledged__c via an Apex REST endpoint. A nightly Scheduled Apex job (ProviderTerminationDateJob) processes records where PNA.Termination_Date__c = today. An EventBus Trigger subscriber on ProviderTerminated__e creates Directory_Sync_Queue__c records. 'For Cause' escalation is handled by a separate flow path with assignment to the Compliance queue. All operations are wrapped in try-catch with error logging to Integration_Error_Log__c.

## Description

As a Provider Network Specialist, I need a bulk update screen that allows me to update NPI and TIN fields across multiple Provider__c (Person Account) and Facility__c (Business Account) records simultaneously so that I can efficiently correct provider demographic data received from NPPES and CAQH reconciliation feeds without processing each record individually.

## Acceptance Criteria

Given a Provider Network Specialist has the 'Provider_Data_Maintenance' Permission Set Group assigned,
When they navigate to the Provider Data Maintenance OmniScript,
Then they can search and filter Provider__c records by NPI, TIN, Network_Status__c, or Provider_Specialty__c.

Given a list of Provider__c records is returned in the bulk update DataTable,
When the user selects up to 200 records and enters a corrected NPI or TIN value in the inline edit column,
Then the system validates the NPI against a 10-digit numeric format and the TIN against a 9-digit numeric format before allowing save.

Given NPI validation passes,
When the bulk save action is triggered,
Then a Platform Event (Provider_Data_Update__e) is published for each modified record and Provider__c.NPI__c and Provider__c.TIN__c are updated within the same transaction.

Given the Platform Event is consumed by the subscriber Apex trigger,
When Provider__c.NPI__c is changed,
Then all active Provider_Network_Agreement__c records linked via Provider__c lookup are stamped with PNA.Last_NPI_Change_Date__c = TODAY() and PNA.NPI_Change_Reason__c = 'Bulk NPPES Reconciliation'.

Given any record in the batch fails validation,
When the bulk save completes,
Then a Bulk_Update_Error_Log__c record is created with fields: Error_Record_Id__c, Error_Field__c, Error_Message__c, and Batch_Run_Id__c, and the failed records are highlighted in red in the DataTable.

-Given the user selects more than 500 records, the system should process all in a single DML operation-
Note: Superseded — batch size capped at 200 per INS-1502 governor limit analysis.

Given the bulk update completes successfully,
When the Provider Network Specialist reviews the audit trail,
Then a Provider_Change_Notification__c (PCN) record is auto-created per updated provider with PCN.Change_Type__c = 'NPI/TIN Update', PCN.Initiated_By__c = running user, and PCN.Status__c = 'Pending Review'.

## Solution

Build an OmniScript ('ProviderBulkUpdateOS') with an embedded DataRaptor Transform to query Provider__c and Facility__c records. Use a DataRaptor Load for the bulk DML operation. Publish Provider_Data_Update__e Platform Events post-save, consumed by an Apex Trigger (ProviderDataUpdateTrigger) that handles PNA stamping. Implement a custom Apex validation class (NPITINValidator) invoked as a callable action within the OmniScript. Create Bulk_Update_Error_Log__c object with appropriate FLS. Assign access via 'Provider_Data_Maintenance' Permission Set Group. Batch size enforcement handled via OmniScript Set Values element checking list count before proceeding to DML step.

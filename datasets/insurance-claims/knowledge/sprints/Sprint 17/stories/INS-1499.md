## Description

**Bug Report — INS-1499**

Steps to Reproduce:
1. Navigate to the Provider Operations App and open the 'Bulk TIN Update' utility (deployed in Sprint 16).
2. Upload a CSV file containing 50+ Provider__c records with updated TIN__c values.
3. Execute the bulk update job.
4. After job completion, open any Provider__c record that had an associated active Provider_Network_Agreement__c.
5. Observe the PNA_Effective_Date__c field on the related PNA record.

Expected Behavior:
The PNA_Effective_Date__c field on Provider_Network_Agreement__c should remain unchanged after a bulk TIN update on the parent Provider__c record.

Actual Behavior:
PNA_Effective_Date__c is set to null on all Provider_Network_Agreement__c records linked to Provider__c records included in the bulk update. This appears to affect 100% of records in the update batch.

Additional Context:
- First observed in Sprint 16 UAT by Provider Network Specialist team.
- Impacts approximately 340 PNA records in the UAT sandbox.
- Suspected root cause: the Apex Trigger 'ProviderTINChangeTrigger' on Provider__c incorrectly invokes a PNA update DML that passes an incomplete SObject with only PNA_Review_Required__c = true, inadvertently clearing other fields due to a full-object update pattern instead of field-specific update.
- Related to INS-1496 bulk sync work.

## Acceptance Criteria

As per Description.

Additional verification:
Given a bulk TIN update is executed for 50+ Provider__c records,
When the job completes,
Then PNA_Effective_Date__c, PNA_Termination_Date__c, and PNA_Status__c on all related Provider_Network_Agreement__c records must retain their pre-update values.

## Solution

Fix the 'ProviderTINChangeTrigger' Apex trigger handler class (ProviderTINChangeTriggerHandler.cls) to use a field-specific SObject update pattern. Replace the current full-object instantiation with a queried PNA record updated only on PNA_Review_Required__c field. Implement a unit test asserting PNA_Effective_Date__c is unchanged post-trigger execution with 200-record bulk test data. Deploy fix to UAT for regression testing before Sprint 17 release. Add a data repair script to restore null PNA_Effective_Date__c values in UAT from the last known good snapshot.

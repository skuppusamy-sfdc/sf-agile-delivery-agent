## Description

As a Claims Operations Manager, I need the system to automatically suspend new claims assignment to a provider when their Network_Termination_Case__c is approved so that terminated providers do not receive new claim referrals during the wind-down period, reducing re-work for Assignment Coordinators and ensuring compliance with termination notice requirements.

## Acceptance Criteria

Given a Network_Termination_Case__c record exists with Status__c = 'Pending Approval',
When an authorized approver (Claims Operations Manager or Compliance Officer role) approves the record via the Approval Process,
Then Network_Termination_Case__c.Status__c transitions to 'Approved' and Provider__c.Claims_Assignment_Eligible__c is set to FALSE within the same transaction.

Given Provider__c.Claims_Assignment_Eligible__c = FALSE,
When the Assignment Coordinator's claim routing logic (Omni-Channel or Flow-based) attempts to assign a new Claim__c to this provider,
Then the assignment is blocked and the Claim__c.Assignment_Block_Reason__c field is stamped with 'Provider Network Termination Pending'.

Given the termination case is approved,
When the system evaluates open Claim__c records currently assigned to the provider where Claim__c.Status__c IN ('New', 'In Review', 'Pending Info'),
Then a task is auto-created for the Assignment Coordinator with Subject = 'Reassign Claims — Provider Termination [Provider Name]', Due_Date = TODAY() + 5 business days, and related to the Network_Termination_Case__c.

Given the Network_Termination_Case__c.Termination_Type__c = 'Immediate',
When the approval is recorded,
Then Provider__c.Network_Status__c is immediately updated to 'Terminated' and all active PNA records (PNA.Provider__c = Provider__c.Id AND PNA.Status__c = 'Active') are updated to PNA.Status__c = 'Terminated' and PNA.Termination_Date__c = TODAY().

Given the Network_Termination_Case__c.Termination_Type__c = 'With Notice Period',
When the approval is recorded,
Then Provider__c.Network_Status__c = 'Termination Pending' and PNA records are scheduled for termination on PNA.Agreement_End_Date__c = Network_Termination_Case__c.Notice_Period_End_Date__c via a scheduled Flow.

Given all termination steps complete,
When the Compliance Officer audits the case,
Then Network_Termination_Case__c.Termination_Completed_Date__c is stamped and a PCN record is created with PCN.Change_Type__c = 'Network Termination', PCN.Status__c = 'Closed — Completed'.

## Solution

Configure a multi-step Salesforce Approval Process ('Provider_Termination_Approval') on Network_Termination_Case__c with two approval steps: (1) Claims Operations Manager initial approval, (2) Compliance Officer final approval. On final approval, a Record-Triggered Flow ('NTC_Approval_Handler_Flow') fires to update Provider__c.Claims_Assignment_Eligible__c = FALSE and branch on Termination_Type__c. For 'Immediate' terminations, invoke an Apex-invocable action (PNATerminationService.terminateAllActivePNAs()) to handle bulk PNA updates. For 'With Notice Period', use a Scheduled Flow triggered on NTC approval to execute PNA updates on the Notice_Period_End_Date__c. Auto-task creation handled via Flow Create Records element. PCN creation via a separate Flow after-save step. Omni-Channel routing configuration updated to check Provider__c.Claims_Assignment_Eligible__c as a routing eligibility condition via Apex-based routing logic.

## Description

As a Provider Relations Specialist, I need a FlexCard on the Provider__c (Person Account) record page that summarizes all active Provider_Network_Agreement__c (PNA) records and their associated Fee_Schedule__c entries so that I can quickly assess a provider's contracted network status and reimbursement tiers without navigating to multiple related lists.

This story is part of the Provider 360 epic and complements the credentialing status FlexCard delivered in Sprint 15.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page,
When the 'Provider 360 – Network & Fees' FlexCard loads,
Then it must display a card for each Provider_Network_Agreement__c where PNA.Status__c = 'Active' AND PNA.Provider__c = current record Id, showing: PNA.Network_Name__c, PNA.Effective_Date__c, PNA.Termination_Date__c, PNA.LOB__c, and a badge indicating PNA.Contract_Type__c ('Full Risk' | 'Fee-for-Service' | 'Capitation').

Given an active PNA card is displayed,
When the user expands the PNA card,
Then a nested DataRaptor-driven table must list all Fee_Schedule__c records where Fee_Schedule__c.PNA__c = PNA.Id and Fee_Schedule__c.Effective_Date__c <= TODAY() <= Fee_Schedule__c.Expiration_Date__c, showing: Fee_Schedule__c.CPT_Code__c, Fee_Schedule__c.Allowed_Amount__c, Fee_Schedule__c.Modifier__c.

Given no active PNA records exist for the provider,
When the FlexCard loads,
Then an empty-state message 'No Active Network Agreements – Contact Provider Network Contracting' must be displayed with a quick action button launching the 'New PNA' OmniScript.

Given a PNA with PNA.Termination_Date__c within 30 days of today,
When the FlexCard renders,
Then the PNA card must display a yellow warning banner: 'Agreement expiring on [PNA.Termination_Date__c] – renewal required'.

-Given more than 10 active PNAs exist, When the FlexCard loads, Then paginate at 5 per page- (deprecated — replaced with scrollable card list capped at 20 records per design review feedback)

## Solution

1. OmniStudio FlexCard 'Provider360_NetworkFees' with a DataRaptor Extract datasource querying Provider_Network_Agreement__c and child Fee_Schedule__c via a SOQL relationship query.
2. Conditional rendering states for empty-state and expiry warning using FlexCard State conditions on PNA.Termination_Date__c formula field.
3. Flyout action wired to the existing 'NewPNA_OmniScript' for the empty-state CTA.
4. FlexCard deployed to the Provider__c (Person Account) Lightning Record Page via App Builder, visible only to users with 'Provider_Relations' Permission Set.
5. DataRaptor Transform used to flatten nested Fee_Schedule__c child records into a repeatable card sub-section.

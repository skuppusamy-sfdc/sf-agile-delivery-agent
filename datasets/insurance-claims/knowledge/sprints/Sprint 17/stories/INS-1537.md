## Description

As a Credentialing Specialist, I need the system to automatically create a Credentialing_Case__c record when a new Provider__c is submitted through the provider self-service portal so that credentialing workflows are initiated without manual intervention and no new provider can be linked to an active PNA before credentialing is complete.

## Acceptance Criteria

Given a new Provider__c record is created via the Provider Portal OmniScript with Provider__c.RecordType = 'Provider' and Provider__c.Portal_Submission__c = TRUE,
When the record is saved,
Then a Credentialing_Case__c record must be auto-created with:
  - Credentialing_Case__c.Provider__c = new Provider__c.Id
  - Credentialing_Case__c.Status__c = 'Initiated'
  - Credentialing_Case__c.LOB__c = Provider__c.Line_of_Business__c
  - Credentialing_Case__c.Source__c = 'Provider Portal'
  - Credentialing_Case__c.OwnerId = Credentialing Specialist queue

Given the Credentialing_Case__c is created,
When the record is saved,
Then a DocuSign envelope must be sent to Provider__c.Email with the standard CAQH Authorization and Credentialing Attestation document template.

Given the provider has already submitted a CAQH profile (Provider__c.CAQH_ID__c is populated),
When the Credentialing_Case__c is created,
Then an automatic CAQH data pull must be initiated via the CAQH API integration and Credentialing_Case__c.CAQH_Verified__c must be set to TRUE upon successful response.

Given a user attempts to create a Provider_Network_Agreement__c with PNA.Provider__c pointing to a provider whose Credentialing_Case__c.Status__c != 'Approved',
When the PNA record is saved,
Then a validation rule must block the save and display the message: 'Provider credentialing must be in Approved status before a PNA can be created.'

-Send credentialing initiation SMS to provider mobile number.- (Deferred — SMS integration not in scope for Phase 2)

Given the Credentialing_Case__c remains in 'Initiated' status for more than 30 days,
When the daily scheduled flow runs,
Then Credentialing_Case__c.Status__c must be updated to 'Stalled' and a Task created for the assigned Credentialing Specialist with Subject = 'Follow-up: Credentialing Stalled > 30 Days'.

## Solution

Record-Triggered Flow on Provider__c (After Insert) checks Portal_Submission__c = TRUE and RecordType = 'Provider', then calls Apex Invocable Method 'CredentialingCaseFactory.create()' to insert Credentialing_Case__c. DocuSign envelope triggered via DocuSign Apex Toolkit (DSFLib) callout within the same Invocable Method using a Named Credential for auth. CAQH API pull handled by a separate Apex Future method 'CAQHIntegration.pullProfile()' to avoid mixed DML. Validation rule on Provider_Network_Agreement__c uses VLOOKUP against Credentialing_Case__c. Stalled case detection handled by Scheduled Flow running nightly at 06:00 EST querying Credentialing_Case__c via SOQL with filter on Status__c = 'Initiated' AND CreatedDate <= LAST_N_DAYS:30.

## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that consolidates active Provider_Network_Agreement__c records, associated Fee_Schedule__c entries, and current Credentialing_Case__c status so that I can assess a provider's complete network and credentialing posture without navigating across multiple objects.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then it displays the following sections: (1) Active PNAs — showing PNA.Agreement_Start_Date__c, PNA.Agreement_End_Date__c, PNA.Network_Name__c, PNA.LOB__c; (2) Fee Schedules — showing Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.CPT_Code_Range__c; (3) Credentialing — showing Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, Credentialing_Case__c.CAQH_Profile_Status__c.

Given a provider has multiple active PNA records across different LOBs,
When the FlexCard PNA section renders,
Then records are grouped by PNA.LOB__c and sorted by PNA.Agreement_End_Date__c ascending so expiring agreements appear first.

Given a Credentialing_Case__c.Status__c = 'Expired' or 'Suspended',
When the Credentialing section renders,
Then the status badge displays in red with a tooltip: 'Provider may not be eligible for new claim assignments. Contact Credentialing Specialist.'

Given the FlexCard is viewed by a user without the 'Provider_Network_Read' Permission Set,
When the page loads,
Then the PNA and Fee Schedule sections are hidden and replaced with a message: 'You do not have access to view network agreement details.'

Given a provider has no active Fee_Schedule__c records,
When the Fee Schedule section renders,
Then a message displays: 'No active fee schedules found. Contact Provider Network team to assign a schedule.'

## Solution

Build a FlexCard ('Provider360FC') using OmniStudio FlexCard Designer. Use three child DataRaptor Extract actions to query Provider_Network_Agreement__c (filtered by PNA.Status__c = 'Active'), Fee_Schedule__c (joined via PNA), and Credentialing_Case__c (filtered by most recent open/active record per provider). Implement conditional visibility on FlexCard states for the credentialing badge color (State: 'Expired'/'Suspended' = red, 'Active' = green). Use a Custom Permission ('View_Provider_Network_Details') checked via FlexCard conditional rendering to enforce field-level visibility. Deploy FlexCard to Provider__c Lightning Record Page via App Builder.

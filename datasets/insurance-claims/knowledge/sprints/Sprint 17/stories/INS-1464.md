## Description

**Bug Report — Fee Schedule Engine returning $0.00 Allowed Amount for modified CPT codes**

**Environment:** Acme Insurance Production Sandbox (Sprint 17 refresh)

**Steps to Reproduce:**
1. Navigate to any active Claim__c record with at least one Claim_Line__c.
2. On a Claim_Line__c record, set Claim_Line__c.CPT_Code__c = '99213-25' (CPT code with modifier '-25' appended).
3. Trigger adjudication by updating Claim__c.Status__c = 'In Adjudication'.
4. Observe Claim_Line__c.Allowed_Amount__c after the Fee Schedule lookup executes.

**Expected Result:** The Fee Schedule Engine should strip the modifier suffix ('-25', '-59', '-GT', etc.) from the CPT code before querying Fee_Schedule__c records, returning the base CPT allowed amount (e.g., Fee_Schedule__c.Allowed_Amount__c = $125.00 for CPT 99213).

**Actual Result:** Claim_Line__c.Allowed_Amount__c is set to $0.00 because the lookup query attempts an exact match on '99213-25' against Fee_Schedule__c.CPT_Code__c, which stores only the base code '99213', resulting in no match found.

**Impact:** High — affects all Claim_Line__c records submitted with modified CPT codes. Downstream Payment__c and EOB records are generated with $0.00 amounts, causing incorrect ERA files and provider payment disputes.

**Frequency:** Reproducible 100% with any CPT modifier. Estimated 340 Claim_Line__c records impacted in current sprint cycle.

## Acceptance Criteria

As per Description.

Additionally: After fix is deployed, a data remediation script must be run to recalculate Claim_Line__c.Allowed_Amount__c for all affected records where Claim_Line__c.Allowed_Amount__c = 0 AND Claim_Line__c.CPT_Code__c contains '-' AND Claim__c.Status__c != 'Closed'.

## Solution

Fix the FeeScheduleLookupService Apex class: before executing the SOQL query against Fee_Schedule__c, apply a String.split('-')[0] operation to extract the base CPT code from Claim_Line__c.CPT_Code__c. Store the stripped base code in a local variable (baseCptCode) used exclusively for the Fee_Schedule__c lookup, while preserving the original value in Claim_Line__c.CPT_Code__c. Add a new field Claim_Line__c.CPT_Modifier__c (Text, 10) to capture the extracted modifier for audit purposes. Update unit tests in FeeScheduleLookupServiceTest to include test cases for CPT codes with modifiers -25, -59, -GT, and -76. A one-time Batch Apex job (RemediateZeroAllowedAmountBatch) will reprocess affected Claim_Line__c records post-deployment.

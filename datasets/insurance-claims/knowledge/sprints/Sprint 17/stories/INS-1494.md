## Description

**Bug Report — Silent Failure on CAQH Sync when CAQH_ID__c is Null**

**Environment:** Acme Insurance Full Sandbox (ACME_FS1) and confirmed in Partial Sandbox (ACME_PS2)
**Discovered by:** Lisa Park (Credentialing Specialist) during Sprint 16 regression testing

**Steps to Reproduce:**
1. Navigate to a Provider__c (Person Account) record where Provider__c.CAQH_ID__c is blank/null.
2. Click the 'Sync with CAQH' quick action button on the Provider record page.
3. The OmniScript OS_CAQHProviderSync launches.
4. Proceed through the OmniScript steps and click 'Sync Now' on the final step.
5. Observe the result.

**Expected Result:** The OmniScript should display a validation error message: 'CAQH ID is required to initiate sync. Please enter a valid CAQH ID before proceeding.' The sync should not be submitted.

**Actual Result:** The OmniScript completes all steps and shows a success spinner, then closes with no error message. No CAQH API callout is made (confirmed via debug logs — CAQHIntegrationService.syncProvider() returns early on null CAQH ID check), but the user sees a success state. Additionally, Credentialing_Case__c.Last_CAQH_Sync_Date__c is incorrectly stamped with today's date despite no actual sync occurring.

**Impact:** Medium — Credentialing Specialists believe the sync was successful. Credentialing_Case__c records show false 'synced' timestamps, potentially masking providers with stale credentialing data.

**Root Cause:** OS_CAQHProviderSync OmniScript Step 3 'Submit' action does not validate Provider__c.CAQH_ID__c before invoking the Integration Procedure IP_CAQHSync. The Apex service returns null without throwing an exception, and the OmniScript interprets a null response as success.

## Acceptance Criteria

As per Description. Fix verified when: (1) clicking 'Sync Now' with null CAQH_ID__c displays inline OmniScript error and prevents submission; (2) Credentialing_Case__c.Last_CAQH_Sync_Date__c is NOT updated unless a successful CAQH API HTTP 200 response is received; (3) a successful sync with a valid CAQH_ID__c continues to function correctly (regression).

## Solution

Add a Validation step in OS_CAQHProviderSync OmniScript before the Submit action: use a Validation Rule element checking that the DataRaptor-fetched Provider__c.CAQH_ID__c is not null/empty, displaying the prescribed error message. In CAQHIntegrationService Apex, throw a custom CAQHSyncException when CAQH_ID__c is null rather than returning null silently. Update IP_CAQHSync Integration Procedure to only stamp Credentialing_Case__c.Last_CAQH_Sync_Date__c within a conditional block checking the HTTP response status code = 200.

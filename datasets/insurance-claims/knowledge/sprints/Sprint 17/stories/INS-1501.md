## Description

As a Credentialing Specialist, I need the Salesforce platform to automatically update a provider's Credentialing_Case__c status and Provider__c credentialing fields when CAQH returns an attestation completion event so that re-attestation workflows are closed out without manual data entry and provider network readiness is reflected in real time.

## Acceptance Criteria

Given CAQH sends a webhook payload to the Salesforce named credential endpoint indicating attestation completion for a provider CAQH_Provider_ID,
When the inbound REST API handler processes the payload,
Then the matching Provider__c record (matched via Provider__c.CAQH_Provider_ID__c) has Credentialing_Status__c updated to 'Attested', Last_Attestation_Date__c set to the attestation date from the payload, and Next_Attestation_Due_Date__c calculated as Last_Attestation_Date__c + 365 days.

Given the Provider__c record is updated,
When a matching open Credentialing_Case__c exists (Case_Type__c = 'Re-Attestation' and Status__c = 'Pending CAQH Response'),
Then the Credentialing_Case__c.Status__c is updated to 'Closed — Attested' and Closure_Date__c is stamped with today's date.

Given the CAQH payload includes updated board certification data,
When the sync processes,
Then Provider__c.Board_Certification__c and Board_Certification_Expiry__c are updated accordingly.

Given the CAQH_Provider_ID__c in the payload does not match any Provider__c record,
When the handler processes the event,
Then an Unmatched_CAQH_Event__c record is created with Raw_Payload__c, Received_DateTime__c, and Status__c = 'Unmatched' for manual resolution.

Given the attestation completion triggers the update,
When the Provider__c.Credentialing_Status__c changes to 'Attested',
Then a Record-Triggered Flow evaluates whether all active PNA records (PNA_Status__c = 'Suspended — Pending Credentialing') should be reactivated to PNA_Status__c = 'Active'.

## Solution

Implement a REST API Apex class 'CAQHAttestationWebhookHandler' exposed via Connected App and Named Credential. Inbound payload parsed using JSON.deserializeUntyped(). Provider__c matching uses a SOQL query on CAQH_Provider_ID__c with FOR UPDATE to prevent race conditions. Field updates executed in a single DML. Credentialing_Case__c closure handled in the same transaction. Unmatched events written to Unmatched_CAQH_Event__c object. PNA reactivation logic in a Record-Triggered Flow on Provider__c (fires on Credentialing_Status__c change) using a Get Records element to find suspended PNAs and an Update Records loop. Full integration test coverage using @HttpCalloutMock. Platform Event 'CAQH_Sync_Complete__e' published for downstream monitoring.

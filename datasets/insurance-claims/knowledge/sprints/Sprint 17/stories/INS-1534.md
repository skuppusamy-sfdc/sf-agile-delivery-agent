## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard embedded on the Provider__c (Person Account) record page so that I can see a consolidated view of the provider's active PNA, associated Fee Schedule, credentialing status, and open claims count without navigating to multiple related lists.

## Acceptance Criteria

Given a Provider__c record is open in Salesforce,
When the Provider 360 FlexCard loads,
Then it must display the following data blocks:
  - Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Specialty__c
  - Most recent Provider_Network_Agreement__c where PNA.Status__c = 'Active' (PNA.Effective_Date__c, PNA.Expiration_Date__c, PNA.Network__c)
  - Associated Fee_Schedule__c.Name and Fee_Schedule__c.Effective_Date__c
  - Count of Claim__c records where Claim__c.Provider__c = current record AND Claim__c.Status__c IN ('Open','In Review','Pending Info')
  - Most recent Credentialing_Case__c.Status__c

Given the provider has no active PNA,
When the FlexCard loads the PNA block,
Then a warning badge 'No Active Contract' must appear in amber with a quick-action button 'Initiate PNA' that launches the New PNA OmniScript.

Given the Credentialing_Case__c.Status__c = 'Suspended' or 'Terminated',
When the FlexCard renders,
Then the provider name header must display a red 'Credentialing Alert' indicator and the open claims count block must be highlighted in red.

-Display all historical PNA records in the FlexCard.- (Removed — historical PNA view deferred to Provider Network Contracting epic per INS-1489)

Given the FlexCard is viewed by a user without the 'Provider_Data_Admin_PSG' Permission Set Group,
When the FlexCard loads,
Then the TIN__c field must be masked (last 4 digits visible only) and the 'Initiate PNA' action button must be hidden.

## Solution

Build OmniStudio FlexCard 'Provider360_FC' with child DataRaptor Turbo Extract actions for each data block (PNA, Fee Schedule, Claims count, Credentialing). Use a single DataRaptor Extract with multi-object joins where possible to minimize SOQL queries. Conditional visibility on FlexCard states driven by JSON path expressions. Embed FlexCard on Provider__c Lightning Record Page via App Builder. TIN masking handled via a DataRaptor Transform formula. Quick-action 'Initiate PNA' launches OmniScript 'NewPNA_OS' in a modal. Agentforce AI summary panel pinned at top of FlexCard to surface natural-language provider risk summary using Einstein Generative AI prompt template 'Provider_Risk_Summary_PT'.

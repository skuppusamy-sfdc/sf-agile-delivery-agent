## Description

As a Credentialing Specialist, I need the system to automatically call the CAQH ProView API when a new Credentialing_Case__c is created for a Provider__c so that provider education, licensure, malpractice history, and DEA certification data is pre-populated on the case, reducing manual data entry time by an estimated 60% per case.

## Acceptance Criteria

Given a Credentialing Specialist or automated process creates a new Credentialing_Case__c with Credentialing_Case__c.Provider__c populated and Credentialing_Case__c.NPI__c populated,
When the record is saved,
Then an asynchronous callout to the CAQH ProView REST API is triggered within 30 seconds using the provider's NPI as the query key.

Given the CAQH API returns a successful response (HTTP 200),
When the response payload is parsed,
Then the following fields are populated on Credentialing_Case__c: CC__c.Medical_School__c, CC__c.Graduation_Year__c, CC__c.State_License_Number__c, CC__c.License_Expiry_Date__c, CC__c.DEA_Certificate_Number__c, CC__c.Malpractice_Carrier__c, CC__c.CAQH_Last_Attested_Date__c.

Given the CAQH API returns HTTP 404 (provider not found in CAQH),
When the callout resolves,
Then Credentialing_Case__c.CAQH_Status__c = 'Not Found' and a task is created for the Credentialing Specialist to manually collect documents.

Given the CAQH API returns any 5xx error,
When the callout fails,
Then the error is logged to a custom Integration_Error_Log__c record with IEL__c.Source__c = 'CAQH', IEL__c.Related_Record_Id__c = Credentialing_Case__c.Id, and a retry is attempted after 15 minutes (max 3 retries).

Given CAQH data is successfully populated,
When Credentialing_Case__c.CAQH_Last_Attested_Date__c is more than 120 days ago,
Then Credentialing_Case__c.CAQH_Attestation_Warning__c = TRUE and a warning banner is shown on the record page.

Given the provider already has a Credentialing_Case__c with Credentialing_Case__c.CAQH_Status__c = 'Verified' closed within the last 12 months,
When a new Credentialing_Case__c is created for the same provider,
Then the system presents a warning 'Recent credentialing on file — verify if re-credentialing is required' before triggering the CAQH callout.

-Given CAQH data conflicts with existing Provider__c fields, auto-overwrite Provider__c record- (removed — CAQH data to populate Credentialing_Case__c only, not auto-update Provider__c)

## Solution

Use a Record-Triggered Flow (after insert) on Credentialing_Case__c to publish a Platform Event (CAQH_Lookup_Request__e) containing the NPI and Case Id. A subscriber Apex class (CAQHProViewCalloutHandler) processes the event asynchronously, performs a named credential-backed REST callout to CAQH ProView API, parses the JSON response, and performs a DML update on the Credentialing_Case__c record. Retry logic is implemented via a custom Retry_Queue__c object processed by a Schedulable Apex class running every 15 minutes. Error logging uses a centralized Integration_Error_Log__c Apex utility class.

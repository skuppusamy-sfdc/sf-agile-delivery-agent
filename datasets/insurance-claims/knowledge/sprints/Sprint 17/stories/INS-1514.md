## Description

As a Provider Network Specialist, I need all active Fee_Schedule__c records linked to a provider to be automatically inactivated when their Network_Termination_Case__c is closed with a 'Terminated' outcome so that claims submitted after the termination effective date are not incorrectly adjudicated at contracted rates, preventing payment leakage and compliance violations.

## Acceptance Criteria

Given a Network_Termination_Case__c record exists with Status__c = 'Open' and Termination_Outcome__c = null,
When the Credentialing Specialist updates Status__c = 'Closed' and Termination_Outcome__c = 'Terminated' and saves,
Then a Record-Triggered Flow must fire and query all Fee_Schedule__c records where Fee_Schedule__c.Provider__c = Network_Termination_Case__c.Provider__c AND Fee_Schedule__c.Status__c = 'Active'.

Given the query returns one or more active Fee_Schedule__c records,
When the flow processes each record,
Then Fee_Schedule__c.Status__c must be set to 'Inactive' and Fee_Schedule__c.Inactivation_Reason__c = 'Provider Terminated' and Fee_Schedule__c.Inactivation_Date__c = Network_Termination_Case__c.Termination_Effective_Date__c.

Given the fee schedules are inactivated,
When the process completes,
Then the Network_Termination_Case__c.Fee_Schedules_Inactivated__c (checkbox) must be stamped TRUE and a chatter post must be added to the case: 'Fee schedules inactivated as of [Termination_Effective_Date__c]. Total inactivated: [count]'.

Given a Network_Termination_Case__c is closed with Termination_Outcome__c = 'Reinstated' or 'Withdrawn',
When the flow entry criteria evaluate,
Then the fee schedule inactivation logic must NOT execute and Fee_Schedule__c records must remain unchanged.

Given no active Fee_Schedule__c records exist for the provider,
When the termination case is closed as 'Terminated',
Then the flow should complete without error and set Network_Termination_Case__c.Fee_Schedules_Inactivated__c = TRUE with chatter post: 'No active fee schedules found for this provider at time of termination.'

## Solution

Implement a Record-Triggered Flow 'RTF_InactivateFeeSchedulesOnTermination' on Network_Termination_Case__c, triggering on update, entry condition: Status__c changed to 'Closed' AND Termination_Outcome__c = 'Terminated'. Use a Get Records element to retrieve related Fee_Schedule__c records. Use Loop + Update Records elements to set Status__c, Inactivation_Reason__c, and Inactivation_Date__c. Use a Count variable to track updated records. Post to Chatter using the 'Post to Chatter' core action. Add a Fault Path logging to Flow_Error_Log__c. Ensure the flow runs in system context to bypass sharing on Fee_Schedule__c. Test with providers having 0, 1, and 50+ fee schedules.

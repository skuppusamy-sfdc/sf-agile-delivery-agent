## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record when a Provider_Network_Agreement__c reaches its Effective_End_Date__c without a renewal so that the termination workflow is initiated consistently and no provider falls through the cracks during the off-boarding process.

## Acceptance Criteria

Given a scheduled job runs nightly at 02:00 UTC,
When a Provider_Network_Agreement__c record has PNA__c.Effective_End_Date__c = TODAY() AND PNA__c.Status__c = 'Active' AND PNA__c.Renewal_Submitted__c = FALSE,
Then a Network_Termination_Case__c is created with NTC__c.Origin__c = 'System - PNA Expiry', NTC__c.Status__c = 'Open', NTC__c.Provider__c = PNA__c.Provider__c, and NTC__c.Termination_Reason__c = 'Contract Expiration'.

Given the Network_Termination_Case__c is created,
When the record is saved,
Then PNA__c.Status__c is updated to 'Pending Termination' and an email alert is sent to the Provider Relations Specialist assigned on the PNA.

Given a Network_Termination_Case__c already exists for the same provider and PNA combination,
When the nightly job evaluates the PNA,
Then no duplicate Network_Termination_Case__c is created (idempotency check on NTC__c.PNA__c lookup).

Given the Network_Termination_Case__c is created,
When any open Claim__c records exist with Claim__c.Rendering_Provider__c = the terminated provider AND Claim__c.Status__c IN ('Submitted', 'In Adjudication'),
Then those claims are flagged with Claim__c.Provider_Termination_Flag__c = TRUE and assigned to the Claims Operations Manager queue for manual review.

## Solution

Implement a Schedulable Apex class (PNAExpiryTerminationScheduler) that runs nightly via a Scheduled Job. The class queries PNA records meeting expiry criteria, performs an upsert on Network_Termination_Case__c using a composite external ID (PNA Id + Effective_End_Date__c) for idempotency. A Record-Triggered Flow on Network_Termination_Case__c (after insert) updates PNA status and sends email alerts via an Email Alert action. A secondary SOQL query identifies impacted open claims and bulk-updates the Provider_Termination_Flag__c via a Batchable Apex class.

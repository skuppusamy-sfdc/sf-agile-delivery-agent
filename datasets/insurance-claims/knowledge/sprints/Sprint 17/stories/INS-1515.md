## Description

As a Credentialing Specialist, I need Salesforce to automatically sync provider credentialing status from CAQH ProView on a nightly scheduled basis so that Credentialing_Case__c records reflect the most current attestation and sanction data without requiring manual exports and imports. This integration should update key credentialing fields and flag discrepancies for human review.

## Acceptance Criteria

Given a scheduled Apex job 'CAQHCredentialingSyncBatch' is configured to run nightly at 2:00 AM EST,
When the job executes,
Then it must call the CAQH ProView REST API endpoint '/provider/credentialingstatus' for all Provider__c records where Provider__c.CAQH_Provider_Id__c is not null and Provider__c.Active__c = TRUE.

Given the CAQH API returns a successful response (HTTP 200) for a provider,
When the sync processes the response payload,
Then the following fields on the associated Credentialing_Case__c (where Credentialing_Case__c.Status__c != 'Closed') must be updated: Credentialing_Case__c.CAQH_Attestation_Date__c, Credentialing_Case__c.CAQH_Status__c, Credentialing_Case__c.Sanction_Flag__c, Credentialing_Case__c.Last_CAQH_Sync_DateTime__c = NOW().

Given the CAQH API returns Sanction_Flag = true for a provider,
When the sync updates Credentialing_Case__c.Sanction_Flag__c = TRUE,
Then a high-priority Task must be created and assigned to the Credentialing_Case__c.Owner with Subject = 'URGENT: Sanction flagged via CAQH sync — review required' and Due Date = TODAY().

Given the CAQH API returns HTTP 401 or 503,
When the batch encounters the error for a specific provider,
Then that provider's record must be skipped without failing the entire batch, and a CAQH_Sync_Error_Log__c record must be written with fields: Provider__c, Error_Code__c, Error_Message__c, Sync_Timestamp__c.

-Given the sync runs, When a provider's NPI__c in CAQH differs from Provider__c.NPI__c in Salesforce, Then auto-update Provider__c.NPI__c- (deprecated — NPI discrepancies must go to manual review queue per Compliance Officer directive, see INS-1511)

Given the CAQH API returns a NPI mismatch (CAQH NPI != Provider__c.NPI__c),
When the discrepancy is detected,
Then a Provider_Change_Notification__c must be created with PCN.Change_Type__c = 'NPI Discrepancy — CAQH Mismatch', PCN.Status__c = 'Pending Review', and the sync must NOT overwrite Provider__c.NPI__c.

Given the nightly batch completes,
When all providers are processed,
Then a summary email must be sent to the Credentialing Operations email group (credentialing-ops@acmeinsurance.com) with counts: Total Processed, Successfully Updated, Errors, Sanctions Flagged.

## Solution

Implement 'CAQHCredentialingSyncBatch' as a Batchable and Schedulable Apex class with a batch size of 50 (to respect CAQH API rate limits). Use Named Credential 'CAQH_ProView_NC' for secure callout authentication (OAuth 2.0 client credentials). Create Apex callout class 'CAQHProViewService' with method 'getCredentialingStatus(String caqhProviderId)' returning a typed wrapper. Use Database.update with allOrNone=false for Credentialing_Case__c updates. Task creation and PCN creation handled via helper service classes. Error logging to CAQH_Sync_Error_Log__c via a separate Database.insert call outside main try/catch. Summary email sent via Messaging.sendEmail in the batch finish() method. Schedule via Setup > Apex Classes > Schedule Apex at 2:00 AM EST daily.

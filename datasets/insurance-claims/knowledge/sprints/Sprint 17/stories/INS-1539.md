## Description

**Bug Report – INS-1539**

**Summary:** The Group_Service_Agreement__c (GSA) object allows users to set GSA.Effective_Date__c to a future date while simultaneously setting GSA.Status__c = 'Active'. This means a GSA can appear Active in the system before the contract has been executed, causing providers associated with that GSA to be treated as in-network for claims adjudication prematurely.

**Steps to Reproduce:**
1. Navigate to any Group_Service_Agreement__c record or create a new one.
2. Set GSA.Effective_Date__c to any date in the future (e.g., 30 days from today).
3. Set GSA.Status__c = 'Active'.
4. Save the record — save succeeds without any validation error.
5. Navigate to a Claim__c record and set Claim__c.GSA__c to the newly created GSA.
6. Run the adjudication check — claim is treated as covered under an active GSA.

**Expected:** A validation rule should prevent GSA.Status__c from being set to 'Active' when GSA.Effective_Date__c is in the future. Status should remain 'Pending Execution' until the effective date is reached.

**Actual:** No validation prevents a future-dated GSA from being marked Active. Downstream claims adjudication incorrectly treats these agreements as in-force.

## Acceptance Criteria

As per Description.

Given the fix is deployed,
When a user attempts to save a GSA with Status__c = 'Active' and Effective_Date__c > TODAY,
Then the save must be blocked with error message: 'GSA cannot be set to Active before the Effective Date. Current status must be Pending Execution.'

Given the fix is deployed,
When a GSA with Status__c = 'Pending Execution' reaches its Effective_Date__c,
Then a Scheduled Flow running nightly must auto-transition GSA.Status__c to 'Active' and log a GSA_Status_Change_Log__c entry.

## Solution

Add a new Validation Rule on Group_Service_Agreement__c: 'GSA_ActiveDate_Check' with formula: AND(ISPICKVAL(Status__c, 'Active'), Effective_Date__c > TODAY()). Error message as specified in AC. Additionally, create a Scheduled Flow 'GSA_AutoActivate_SF' running nightly at 01:00 EST that queries GSA records WHERE Status__c = 'Pending Execution' AND Effective_Date__c <= TODAY and updates Status__c to 'Active', creating a GSA_Status_Change_Log__c child record for each transition. Deploy validation rule to all sandboxes for regression testing before production push.

## Description

As a Provider Network Specialist, I need the system to automatically create a Network_Termination_Case__c record when a Provider_Network_Agreement__c (PNA) reaches its Termination_Date__c without a renewal or extension in place, so that the termination workflow is initiated without manual intervention and compliance timelines are met per state regulatory requirements.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record has Status__c = 'Active' and Termination_Date__c = TODAY() and no related PNA with Status__c IN ('Pending Renewal', 'Active') exists for the same Provider__c,
When the scheduled Apex job 'PNAExpiryCheckSchedulable' runs at 02:00 AM daily,
Then a new Network_Termination_Case__c is created with: NTC.Provider__c = PNA.Provider__c, NTC.PNA__c = PNA.Id, NTC.Termination_Reason__c = 'Contract Expiry', NTC.Status__c = 'Initiated', NTC.Effective_Termination_Date__c = PNA.Termination_Date__c.

Given the Network_Termination_Case__c is created,
When the record is saved,
Then the originating PNA.Status__c is updated to 'Terminated' and PNA.Termination_Case__c is populated with the new NTC record Id.

Given the NTC is created,
When the record insert completes,
Then an email notification is sent to the Provider Relations Specialist assigned to the PNA (PNA.Owner) and the Provider Network Manager distribution list, using the 'NTC_Initiation_Alert' email template.

Given a PNA has Termination_Date__c within 60 days and no renewal exists,
When the daily job runs,
Then a Task is created on the PNA record with Subject = 'PNA Expiring in 60 Days — Renewal Required', Due_Date = TODAY() + 7, assigned to PNA.OwnerId, with Priority = 'High'.

Given a Network_Termination_Case__c already exists for a PNA (NTC.Status__c != 'Closed'),
When the daily job evaluates the same PNA,
Then no duplicate NTC is created and the job logs a skip entry to the Termination_Job_Log__c object.

Given the NTC creation triggers downstream Claim__c impact assessment,
When NTC.Status__c transitions to 'Initiated',
Then a Platform Event 'Provider_Terminated__e' is published with payload fields: Provider_Id__c, Effective_Date__c, LOB__c, so that the Claims Processing engine can flag open Claim__c records where Claim__c.Rendering_Provider__c = terminated provider and Claim__c.Status__c NOT IN ('Closed', 'Denied', 'Paid').

## Solution

Implement Schedulable Apex class 'PNAExpiryCheckSchedulable' invoking Batchable 'PNATerminationBatch' for governor limit safety. Batch queries PNA records using SOQL with date filters and subquery for renewal check. Network_Termination_Case__c creation handled in batch execute method with allOrNothing=false. Record-Triggered Flow on Network_Termination_Case__c (after insert, entry condition: Status__c = 'Initiated') handles PNA status update and email notification via custom email action. Platform Event 'Provider_Terminated__e' published from Apex after NTC insert. Termination_Job_Log__c records written via a separate async queueable to avoid mixed DML. Custom Permission 'Can_Override_Termination' added to PSG_Provider_Network_Admin for manual NTC bypass.

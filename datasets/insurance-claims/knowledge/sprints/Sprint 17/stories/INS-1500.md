## Description

As a Provider Network Specialist, I need a Lightning Web Component (LWC) utility that allows me to mass reassign Provider__c records from one Network_Territory__c to another so that territory realignment activities — such as regional expansions or contractions — can be executed efficiently without requiring individual record edits or Salesforce Admin involvement.

## Acceptance Criteria

Given the specialist navigates to the Provider Operations App home page,
When they open the 'Territory Reassignment Utility' LWC,
Then they can select a Source_Territory__c and a Target_Territory__c from picklist fields sourced from the Network_Territory__c object.

Given source and target territories are selected,
When the specialist clicks 'Preview Affected Records',
Then a data table displays all Provider__c records with Provider__c.Network_Territory__c = Source_Territory__c, showing columns: Provider_Name__c, NPI_Number__c, Specialty__c, Current_Territory__c, and count of active PNA records.

Given the specialist reviews the preview and selects a subset of records using checkboxes,
When they click 'Execute Reassignment',
Then only the selected Provider__c records have their Network_Territory__c updated to the Target_Territory__c value.

Given the reassignment is executed,
When the batch DML completes,
Then a summary toast notification displays 'X records successfully reassigned to [Target Territory]' and a Reassignment_Audit_Log__c record is created capturing Reassigned_By__c, Reassignment_Date__c, Source_Territory__c, Target_Territory__c, and Record_Count__c.

Given a reassignment affects a Provider__c with an active Credentialing_Case__c (Status__c != 'Closed'),
When the reassignment executes,
Then the Credentialing_Case__c.Assigned_Territory__c is also updated to the Target_Territory__c and a chatter post is added to the case: 'Territory updated as part of bulk reassignment [Reassignment_Audit_Log__c ID]'.

Given the user does not have the 'Provider_Territory_Reassign' custom permission,
When they access the utility,
Then the 'Execute Reassignment' button is disabled and a message reads 'You do not have permission to execute territory reassignments. Contact your administrator.'

-Given reassignment should also update Fee_Schedule__c territory references- (deprecated — fee schedule territory linkage is managed at PNA level, not provider level).

## Solution

Build LWC 'providerTerritoryReassignmentUtility' with a two-panel layout: filter/selection panel and preview data table using lightning-datatable with row selection. Wire an Apex controller method 'getProvidersByTerritory' for the preview query. Execute reassignment via a separate Apex method 'executeProviderReassignment' that processes records in chunked DML (up to 200 per transaction) and publishes a Platform Event on completion. The LWC subscribes to the Platform Event via empApi to refresh the UI on completion. Custom Permission 'Provider_Territory_Reassign' enforced in both Apex (@AuraEnabled with permission check) and LWC (hasPermission wire adapter). Reassignment_Audit_Log__c creation and Credentialing_Case__c updates handled in the Apex method using a single savepoint/rollback pattern for atomicity.

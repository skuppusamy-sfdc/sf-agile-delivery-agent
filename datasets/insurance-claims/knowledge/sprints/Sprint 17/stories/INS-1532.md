## Description

As a Provider Relations Specialist, I need the system to automatically generate and send an amended Provider_Network_Agreement__c document via DocuSign when a Provider_Change_Notification__c (PCN) of type 'NPI/TIN Update' or 'Address Change' is approved so that the updated agreement reflects accurate provider data and is countersigned without requiring manual document preparation. This story supports the PCN records created by the INS-1525 bulk update workflow.

## Acceptance Criteria

Given a Provider_Change_Notification__c record has PCN__c.Change_Type__c IN ('NPI/TIN Update', 'Address Change') AND PCN__c.Status__c is updated to 'Approved',
When the record save completes,
Then a DocuSign envelope is automatically generated using the 'PNA_Amendment_Template' Conga CLM template, pre-populated with the updated Provider__c and PNA fields.

Given the DocuSign envelope is generated,
When the envelope is sent,
Then the primary signer is Provider__c.Email (provider contact) and the countersigner is the Provider Relations Specialist assigned on the PCN; PCN__c.DocuSign_Envelope_Id__c is populated with the returned envelope ID.

Given the provider countersigns the DocuSign envelope,
When DocuSign posts the completion webhook callback,
Then PCN__c.Status__c = 'Completed', PNA__c.Amendment_Executed_Date__c = completion date, and the signed PDF is attached to both the PCN__c and PNA__c records as a Salesforce File (ContentDocument).

Given the DocuSign envelope is sent but not signed within 7 days,
When the 7-day threshold passes,
Then PCN__c.DocuSign_Status__c = 'Reminder Sent' and a DocuSign reminder is automatically triggered via the DocuSign API; the assigned Provider Relations Specialist receives an in-app task.

Given PCN__c.Change_Type__c = 'Specialty Update' (outside the scope of this story),
When the PCN is approved,
Then no DocuSign envelope is triggered and a note 'DocuSign not required for Specialty Updates' is added to PCN__c.Notes__c.

-Given the provider has no email on file, send DocuSign to the provider's fax number via eFax integration- (removed from scope — fax integration deferred to Phase 3)

## Solution

Implement a Record-Triggered Flow on Provider_Change_Notification__c (after update, when Status__c changes to 'Approved') with a decision node filtering on Change_Type__c. For qualifying types, an Apex action (DocuSignPNAAmendmentSender) is invoked, which uses the DocuSign eSignature API (via named credential) to create and send an envelope from the pre-configured 'PNA_Amendment_Template' template ID stored in Custom Metadata (DocuSign_Config__mdt). A DocuSign Connect webhook endpoint (Salesforce Site-hosted Apex REST resource) handles completion callbacks and updates PCN and PNA records. The 7-day reminder is managed by a Scheduled Flow querying for PCN records with DocuSign_Status__c = 'Sent' and envelope creation date older than 7 days.

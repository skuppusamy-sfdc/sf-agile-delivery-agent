## Description

**Bug Report — Fee Schedule Engine CPT Modifier Mismatch (Workers Comp LOB)**

**Environment:** Acme Insurance Full Sandbox (ACME_FS1)
**Discovered by:** Rachel Torres (Claims Operations Manager) during UAT cycle

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record where Claim__c.Line_of_Business__c = 'Workers Comp' and Claim__c.Facility__c is a Facility Business Account.
2. Create a Claim_Line__c record with Claim_Line__c.CPT_Code__c = '99213' and Claim_Line__c.Modifier__c = '25'.
3. Associate the Claim__c to a Fee_Schedule__c record where Fee_Schedule__c.LOB__c = 'Workers Comp' and Fee_Schedule__c.State__c = 'CA'.
4. Click 'Calculate Reimbursement' on the Claim__c record.
5. Observe Claim_Line__c.Allowed_Amount__c populated by the FeeScheduleEngine.

**Expected Result:** Claim_Line__c.Allowed_Amount__c should reflect the Workers Comp CA fee schedule rate for CPT 99213 with modifier 25 applied (modifier 25 adds 0% adjustment per WC CA rules — base rate only).

**Actual Result:** The engine is incorrectly applying the Commercial fee schedule modifier logic (modifier 25 = +15% uplift) resulting in Claim_Line__c.Allowed_Amount__c being overstated by approximately 15%. This causes Payment__c records to be generated with inflated amounts.

**Impact:** High — approx. 340 Claim_Line__c records in production-like data are affected. Potential overpayment exposure of ~$18,400 in test dataset.

**Root Cause Hypothesis:** FeeScheduleEngineService Apex class method 'applyModifierAdjustment()' does not branch on Fee_Schedule__c.LOB__c before applying modifier uplift maps. The Commercial modifier map is used as default fallback.

## Acceptance Criteria

As per Description. Fix verified when Claim_Line__c.Allowed_Amount__c for CPT 99213 + modifier 25 under Workers Comp CA fee schedule returns base rate with 0% modifier adjustment. Regression test must confirm Commercial LOB modifier logic (modifier 25 = +15%) is unaffected.

## Solution

Patch FeeScheduleEngineService.applyModifierAdjustment() Apex method to accept Fee_Schedule__c.LOB__c as a parameter and select the correct modifier lookup map (WC_MODIFIER_MAP vs COMMERCIAL_MODIFIER_MAP) based on LOB value. Add LOB__c to the SOQL query fetching Fee_Schedule__c in FeeScheduleCalculationBatch. Deploy updated Apex with corresponding Apex test class updates achieving >85% coverage on modifier branching logic. Add a Custom Metadata Type (Fee_Schedule_Modifier_Rule__mdt) to externalize modifier adjustment percentages per LOB to avoid future hardcoding.

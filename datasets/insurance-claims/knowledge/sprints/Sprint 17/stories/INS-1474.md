## Description

As a Compliance Officer, I need Provider_Change_Notification__c records with PCN_Type__c = 'TIN Change' to be automatically routed to the Compliance Officer queue for review and approval before the underlying Provider__c.TIN__c field is updated, so that I can ensure TIN changes are validated against IRS and CMS records to prevent fraudulent provider data manipulation and improper payment redirection.

## Acceptance Criteria

Given a Provider_Change_Notification__c record is created with PCN_Type__c = 'TIN Change' and PCN_Status__c = 'Pending Review',
When the record is saved,
Then it must be automatically assigned to the Salesforce Queue named 'Compliance_Officer_Review_Queue' via an Assignment Rule.

Given the PCN is in 'Pending Review' status in the Compliance queue,
When a Compliance Officer opens the PCN record,
Then they must see a custom action button 'Approve TIN Change' and 'Reject TIN Change' rendered via LWC on the record page.

Given a Compliance Officer clicks 'Approve TIN Change',
When the approval action is confirmed,
Then: PCN_Status__c must update to 'Approved', Provider__c.TIN__c must be updated to PCN.New_TIN_Value__c, Provider__c.TIN_Last_Verified_Date__c must be stamped with TODAY(), and a task must be created for the Provider Network Specialist to notify the provider of the confirmed change.

Given a Compliance Officer clicks 'Reject TIN Change',
When a rejection reason is entered (minimum 20 characters required),
Then: PCN_Status__c must update to 'Rejected', PCN_Rejection_Reason__c must be populated, Provider__c.TIN__c must NOT be modified, and an email notification must be sent to the originating Provider Network Specialist.

Given a TIN Change PCN has been in 'Pending Review' status for more than 5 business days,
When the daily escalation batch runs,
Then the PCN record must have PCN_Escalation_Flag__c set to TRUE and an email must be sent to the Compliance Officer Manager (sourced from Custom Metadata 'Compliance_Config__mdt').

Given a PCN is approved or rejected,
When the PCN_Status__c is updated,
Then a PCN_Audit_Log__c child record must be created capturing: Action_Taken__c, Action_By__c, Action_Date__c, and Previous_TIN__c.

## Solution

Record-Triggered Flow on Provider_Change_Notification__c (after insert, entry condition: PCN_Type__c = 'TIN Change') assigns record to 'Compliance_Officer_Review_Queue' via flow Update Records element. Custom LWC 'PCNComplianceActions' with 'Approve' and 'Reject' buttons added to PCN Lightning Record Page via App Builder, visible only to Compliance Officer Permission Set. Apex controller (PCNComplianceController.cls) handles TIN update on Provider__c and PCN_Audit_Log__c creation within a single transaction using Database.SavePoint for rollback safety. Escalation handled by a Scheduled Flow running daily querying overdue PCNs and calling Send Email action. Custom Metadata 'Compliance_Config__mdt' stores escalation manager User Id. Field-Level Security on Provider__c.TIN__c restricts direct edit to System Admin and Compliance Officer profiles only.

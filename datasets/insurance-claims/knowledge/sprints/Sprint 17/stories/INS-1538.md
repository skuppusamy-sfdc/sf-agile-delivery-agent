## Description

As a Provider Network Specialist, I need a bulk CPT code rate update tool within Salesforce so that when CMS releases annual Medicare fee schedule updates, I can upload a CSV of CPT codes and new rates and apply them across multiple Fee_Schedule__c records and associated networks simultaneously, reducing a process that currently takes 3 days of manual entry to under 2 hours.

## Acceptance Criteria

Given the Provider Network Specialist uploads a valid CSV (columns: CPT_Code, New_Rate, Effective_Date, Network_Name) via the 'Fee Schedule Mass Update' LWC,
When the file is parsed,
Then the LWC must display a preview grid showing: CPT_Code, Current_Rate (from Fee_Schedule__c.Fee_Schedule_Line__c.Rate__c), New_Rate, Delta_Amount, Delta_Percent, Network_Name before any DML is executed.

Given the preview grid is displayed,
When the user reviews and clicks 'Confirm Update',
Then a batch Apex job (FeeSchedule_BulkUpdate_Batch) must be enqueued and a toast notification must display 'Batch job submitted – Job ID: {jobId}'.

Given the batch job runs,
When a matching Fee_Schedule_Line__c is found (match on CPT_Code__c AND Fee_Schedule__c.Network__c = Network_Name from CSV),
Then Fee_Schedule_Line__c.Rate__c must be updated to New_Rate and Fee_Schedule_Line__c.Effective_Date__c set to the CSV Effective_Date.

Given a CPT code in the CSV does not match any existing Fee_Schedule_Line__c for the specified network,
When the batch processes that row,
Then a new Fee_Schedule_Line__c must be created under the matching Fee_Schedule__c with Status__c = 'Pending Review'.

Given the batch completes,
When all rows are processed,
Then Fee_Schedule__c.Last_Mass_Update_Date__c must be stamped and a Platform Event (Fee_Schedule_Updated__e) published for each affected Fee_Schedule__c record.

Given a Fee_Schedule_Updated__e event is published,
When the Payment Engine subscriber receives the event,
Then any Claim_Line__c records in status 'Pending Adjudication' linked to affected fee schedules must be flagged for re-adjudication via Claim_Line__c.Requires_Readjudication__c = TRUE.

Given the uploaded CSV contains more than 5,000 rows,
When parsing completes,
Then the LWC must warn: 'Large file detected – processing will occur asynchronously. You will be notified by email upon completion.'

-Rate changes must be approved by Underwriting Analyst before applying.- (Superseded — approval workflow for fee schedule changes moved to a separate story INS-1541 in Sprint 18)

## Solution

LWC 'FeeScheduleMassUpdate_LWC' handles CSV file upload using lightning-file-upload component and Papa Parse JS library for client-side parsing. Preview grid implemented as lightning-datatable with inline delta calculations. On confirm, LWC calls Apex Controller method 'FeeScheduleUpdateController.submitBatch()' which enqueues FeeSchedule_BulkUpdate_Batch (implements Database.Batchable, Database.Stateful). Batch uses chunk size 200. Platform Event Fee_Schedule_Updated__e published in batch finish() method. PaymentEngine subscriber is an Apex Trigger on Fee_Schedule_Updated__e__e that queries and updates Claim_Line__c records. LWC deployed to Provider Network app page via App Builder. Custom Permission 'Fee_Schedule_Mass_Update' gates access to the LWC component.

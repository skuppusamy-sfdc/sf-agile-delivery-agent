## Description

**Bug Report – INS-1479**

**Summary:** When a Group_Service_Agreement__c (GSA) record is cloned using the standard Salesforce 'Clone' button, the GSA.LOB__c (Line of Business) multi-select picklist field is not carried over to the cloned record. All other fields clone correctly. This causes downstream issues where cloned GSAs are missing LOB context, leading to incorrect fee schedule resolution during Claim_Line__c adjudication.

**Environment:** Production, Sandbox (Full Copy – SF17-QA)

**Steps to Reproduce:**
1. Navigate to any existing Group_Service_Agreement__c record where GSA.LOB__c has one or more values selected (e.g., 'Medical', 'Dental').
2. Click the standard 'Clone' button from the record detail page.
3. Observe the cloned GSA record detail page — GSA.LOB__c field is blank.
4. Save the cloned record.
5. Navigate to a Claim_Line__c adjudication test and reference the cloned GSA — fee schedule lookup fails due to missing LOB.

**Expected Behavior:** GSA.LOB__c should be populated on the cloned record with the same values as the source GSA record.

**Actual Behavior:** GSA.LOB__c is blank on all cloned GSA records regardless of source record values.

**Frequency:** 100% reproducible.

**Impact:** High — Provider Network Specialists are cloning GSAs as part of the annual renewal workflow (approx. 200 GSAs per renewal cycle). All cloned records require manual LOB re-entry.

**Root Cause Hypothesis:** The GSA.LOB__c field has 'Do Not Clone' incorrectly set, or a before-insert Apex Trigger is clearing the field on clone operations (IsClone() check missing).

## Acceptance Criteria

As per Description.

Additional verification: After fix is deployed, clone 5 GSA records with varying LOB__c combinations ('Medical' only; 'Dental' only; 'Medical, Dental'; 'Medical, Vision, Dental'; empty) and confirm GSA.LOB__c matches source in all cases. Confirm no regression on GSA.Effective_Date__c, GSA.Facility__c, and GSA.Contract_Value__c clone behavior.

## Solution

1. Investigate the Apex Trigger 'GSATrigger' (before insert handler) for any logic that resets GSA.LOB__c when Trigger.new[i].IsClone() == true — remove or guard the clearing logic.
2. Verify GSA.LOB__c field-level metadata does not have any clone-exclusion configuration.
3. If the trigger is the root cause, add an IsClone() guard: if (!record.IsClone()) { record.LOB__c = null; } pattern correction.
4. Deploy fix to SF17-QA sandbox, execute reproduction steps for regression, then promote to Production via change set.
5. Add a unit test asserting LOB__c is preserved on cloned records to prevent regression.

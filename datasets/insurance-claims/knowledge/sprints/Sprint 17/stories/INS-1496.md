## Description

As a Provider Network Specialist, I need the system to perform nightly bulk updates of Provider__c NPI and TIN fields by consuming the NPPES integration feed so that provider demographic data remains accurate and compliant with CMS requirements without requiring manual intervention.

## Acceptance Criteria

Given a nightly NPPES feed file is delivered to the designated SFTP endpoint,
When the scheduled DataRaptor batch job triggers at 02:00 AM EST,
Then all Provider__c records with a matching NPI_Number__c are updated with the latest TIN__c, Provider_Legal_Name__c, and License_State__c values from the feed.

Given a Provider__c record NPI_Number__c does not match any entry in the NPPES feed,
When the batch completes processing,
Then the record is flagged with Data_Sync_Status__c = 'NPPES_Not_Found' and a Provider_Change_Notification__c (PCN) record is created with PCN_Type__c = 'NPI_Mismatch'.

Given a TIN__c value on Provider__c differs from the NPPES feed value,
When the record is updated,
Then the previous TIN__c value is stored in TIN_Previous__c and TIN_Change_Date__c is stamped with today's date.

Given the bulk update batch processes more than 5,000 records,
When execution completes,
Then a Platform Event (NPPES_Sync_Complete__e) is published containing Total_Processed__c, Total_Updated__c, Total_Errors__c, and Batch_Run_Date__c fields.

Given a Provider__c record has an active Provider_Network_Agreement__c (PNA) with PNA_Status__c = 'Active',
When the TIN__c is changed during bulk update,
Then the associated PNA record is flagged with PNA_Review_Required__c = true and assigned to the Provider Relations queue via Case creation.

-Given the NPPES feed includes taxonomy codes, update Provider__c.Taxonomy_Code__c automatically- (deprecated — taxonomy sync moved to CAQH integration per INS-1488).

## Solution

Implement a scheduled Apex Batch class (NppesProviderSyncBatch) invoked nightly via Scheduled Apex. The batch queries all Provider__c records in chunks of 200. A custom DataRaptor Extract retrieves the NPPES feed payload stored in a Custom Metadata endpoint config. Matching logic uses NPI_Number__c as the primary key. Field-level delta detection drives conditional updates. On TIN change, a Record-Triggered Flow fires post-save to create the PCN record and flag active PNAs. Platform Event publication handled in the batch finish() method. Error rows written to a custom object Batch_Error_Log__c for ops review. Permission Set Group 'Provider_Data_Admin_PSG' controls access to trigger manual reruns via a custom LWC admin panel on the Provider Operations App page.

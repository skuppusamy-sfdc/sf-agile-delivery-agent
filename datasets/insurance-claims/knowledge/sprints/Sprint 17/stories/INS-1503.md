## Description

As a Provider Network Specialist, I need a multi-step OmniScript that allows bulk import of provider-to-facility affiliation records via CSV upload so that large-scale hospital system onboarding events — where hundreds of individual providers are affiliated with a Facility__c (Business Account) — can be processed efficiently with built-in data validation and error reporting, rather than relying on manual data loads or Admin-executed Data Loader jobs.

## Acceptance Criteria

Given the specialist opens the 'Bulk Provider Facility Affiliation Import' OmniScript from the Provider Operations App,
When Step 1 loads,
Then they can upload a CSV file with required columns: Provider_NPI__c, Facility_NPI__c, Affiliation_Type__c, Affiliation_Start_Date__c, and optional Primary_Affiliation__c (boolean).

Given the CSV is uploaded,
When Step 2 (Validation Preview) loads,
Then the OmniScript invokes a DataRaptor Transform to validate each row: (a) Provider_NPI__c matches an existing Provider__c.NPI_Number__c, (b) Facility_NPI__c matches an existing Facility__c.Facility_NPI__c, (c) Affiliation_Type__c is a valid picklist value, (d) Affiliation_Start_Date__c is not a past date more than 2 years ago.

Given validation runs on the uploaded CSV,
When the preview table renders,
Then valid rows are highlighted green, invalid rows are highlighted red with an inline error message per row describing the specific validation failure, and a summary shows 'X of Y rows ready to import'.

Given the specialist reviews the validation results and clicks 'Import Valid Rows',
When the import executes,
Then only green (valid) rows are processed, creating Provider_Facility_Affiliation__c junction records with fields: Provider__c (lookup), Facility__c (lookup), Affiliation_Type__c, Start_Date__c, Primary_Affiliation__c, and Created_Via__c = 'Bulk Import OmniScript'.

Given the import completes,
When Step 3 (Confirmation) loads,
Then a downloadable error report CSV is available for all red rows, and a Bulk_Import_Audit__c record is created capturing Imported_By__c, Import_Date__c, Total_Rows__c, Success_Count__c, Error_Count__c, and a link to the uploaded file via ContentVersion.

Given any imported Provider_Facility_Affiliation__c record sets Primary_Affiliation__c = true,
When the record is created,
Then a Record-Triggered Flow ensures no other Provider_Facility_Affiliation__c record for the same Provider__c has Primary_Affiliation__c = true (enforcing single-primary constraint).

## Solution

Build a 3-step OmniScript 'BulkProviderFacilityAffiliationImport_OS' using a File Upload element on Step 1. Step 2 invokes a DataRaptor Transform 'DR_Transform_ValidateAffiliationCSV' that parses CSV rows and performs lookup validation against Provider__c and Facility__c using DataRaptor Extract sub-calls. Validation results stored in OmniScript JSON state. Step 2 table rendered via a custom LWC child component embedded in the OmniScript for color-coded row display. Import execution on Step 2 confirmation calls an Integration Procedure 'IP_CreateAffiliationRecords' that loops through valid rows and performs chunked Apex callout for DML (ApexRemoteAction). Error CSV generation uses a DataRaptor Transform to rebuild CSV from error row state and triggers a ContentVersion creation. Bulk_Import_Audit__c created in the IP finish step. Primary affiliation constraint enforced via Record-Triggered Flow on Provider_Facility_Affiliation__c object firing on create/update of Primary_Affiliation__c.

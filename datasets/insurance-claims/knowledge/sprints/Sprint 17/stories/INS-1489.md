## Description

As a Provider Network Specialist, I need the ability to perform bulk updates to Provider__c NPI and TIN fields from a dedicated data maintenance screen so that I can efficiently reconcile provider demographic data received from NPPES and CAQH without manually editing individual Person Account records one at a time. This story extends INS-1459 (Provider 360 base screen) and adds bulk-edit capability for NPI__c and TIN__c fields on the Provider__c object.

## Acceptance Criteria

Given a Provider Network Specialist has the 'Provider_Data_Maintenance' Permission Set Group assigned,
When they navigate to the Provider Data Maintenance LWC screen,
Then they see a filterable, paginated datatable listing Provider__c records with columns: Provider__c.Name, Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Network_Status__c, Provider__c.Last_NPPES_Sync_Date__c.

Given the user has selected one or more Provider__c rows in the datatable,
When they click 'Bulk Edit NPI/TIN',
Then an inline edit mode activates for Provider__c.NPI__c and Provider__c.TIN__c columns only, leaving all other fields read-only.

Given the user has entered updated NPI values in bulk edit mode,
When the NPI value does not match the 10-digit numeric format validated against NPPES,
Then the field highlights red with inline error message 'NPI must be a 10-digit numeric value per NPPES standards' and the Save button remains disabled.

Given all edited NPI and TIN values pass front-end validation,
When the user clicks 'Save Changes',
Then an Apex batch job (ProviderBulkUpdateBatch) is invoked, updating each Provider__c record and stamping Provider__c.Last_Data_Maintenance_Date__c = today and Provider__c.Data_Maintenance_Source__c = 'Manual Bulk Update'.

Given the batch job completes,
When any record fails due to a duplicate NPI validation rule,
Then those records are written to a Provider_Data_Maintenance_Error__c object with fields: Error_Type__c = 'Duplicate NPI', Provider__c (lookup), Original_NPI__c, Attempted_NPI__c, and the user receives a toast notification listing failed record counts.

-Given the user exports results to CSV after save, the export includes Provider__c.CAQH_ID__c — REMOVED per data privacy review-

Given a successful bulk save of 50+ records,
When the Platform Event Provider_Data_Updated__e is fired,
Then downstream PNA revalidation Flow (PNA_Revalidation_Flow) is triggered for all affected Provider_Network_Agreement__c records linked to updated providers.

## Solution

Build an LWC 'providerBulkMaintenanceScreen' embedded on a custom App Page. Use a server-side Apex controller (ProviderBulkMaintenanceController) with SOQL cursor pagination for large datasets. Inline edit leverages lightning-datatable draftValues pattern. On save, invoke ProviderBulkUpdateBatch (Database.Batchable, batch size 200). Publish Provider_Data_Updated__e Platform Event post-batch. A Record-Triggered Flow on Provider_Network_Agreement__c listens to the platform event via a triggered flow subscription and sets PNA__c.Revalidation_Required__c = true. Permission access gated via 'Provider_Data_Maintenance' Permission Set Group.

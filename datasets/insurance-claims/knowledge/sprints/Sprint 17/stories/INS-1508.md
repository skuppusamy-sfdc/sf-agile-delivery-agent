## Description

As a Credentialing Specialist, I need an automated integration between Salesforce and the CAQH ProView API so that Provider__c credentialing data (license status, board certifications, malpractice history) is synchronized into Credentialing_Case__c records on a nightly basis, eliminating manual data entry and reducing credentialing cycle time from 45 days to under 20 days.

## Acceptance Criteria

Given the nightly CAQH sync job is scheduled,
When the integration runs at 11:00 PM,
Then it queries all Provider__c records where CAQH_Provider_Id__c != NULL and Credentialing_Status__c IN ('Initial Credentialing', 'Re-Credentialing', 'Pending Verification') and calls the CAQH ProView REST API endpoint '/api/caqh/v2/provider/{caqhId}/profile'.

Given the CAQH API returns a successful 200 response,
When the response payload is parsed,
Then the following fields are updated on the associated Credentialing_Case__c: CC.License_State__c, CC.License_Number__c, CC.License_Expiry_Date__c, CC.Board_Certification__c, CC.Malpractice_Carrier__c, CC.Malpractice_Policy_Expiry__c, CC.CAQH_Last_Sync_Date__c = NOW().

Given the CAQH API returns a license with License_Expiry_Date__c < TODAY(),
When the Credentialing_Case__c is updated,
Then CC.Status__c is set to 'On Hold — License Expired' and a Task is created for the assigned Credentialing Specialist with Subject = 'Expired License Detected — CAQH Sync' and Priority = 'High'.

Given the CAQH API returns a 401 Unauthorized response,
When the error is caught,
Then the integration logs the failure to CAQH_Sync_Error_Log__c with fields: Provider_Id__c, Error_Code__c = '401', Error_Message__c, Sync_Timestamp__c, and sends an alert email to the Integration Admin distribution group.

Given the CAQH API returns data that differs from the current Credentialing_Case__c values,
When the sync updates the record,
Then a Credentialing_Change_History__c child record is created capturing: Field_Name__c, Old_Value__c, New_Value__c, Change_Source__c = 'CAQH_Sync', Change_Date__c for each modified field (audit trail requirement).

Given a Provider__c record has CAQH_Provider_Id__c = NULL,
When the sync job evaluates this record,
Then the record is skipped and logged to CAQH_Sync_Error_Log__c with Error_Code__c = 'MISSING_CAQH_ID' for Credentialing Specialist follow-up.

-Given the sync runs, it should also update Provider__c.Credentialing_Status__c directly- (deprecated: direct Provider__c status updates removed to preserve single source of truth on Credentialing_Case__c per data model review)

Given the full nightly sync completes,
When all records are processed,
Then a summary Platform Event 'CAQH_Sync_Complete__e' is published with payload: Total_Processed__c, Total_Updated__c, Total_Errors__c, Sync_Date__c, consumed by a downstream notification Flow.

## Solution

Implement Apex Queueable chain 'CAQHSyncQueueable' called from Schedulable 'CAQHNightlySyncSchedulable'. Named Credential 'CAQH_ProView_API' stores endpoint and OAuth 2.0 client credentials. HTTP callouts made per provider with 10-second timeout; chained Queueables handle >50 providers to avoid callout limits. DataRaptor-style field mapping handled in Apex via a custom metadata type 'CAQH_Field_Mapping__mdt' for maintainability. Credentialing_Change_History__c inserts batched in a separate Queueable to avoid mixed DML with callouts. Platform Event published post-completion. All credentials stored in Salesforce Named Credentials and Custom Protected Settings — no hardcoded tokens. Unit test coverage target: 90%+ with HttpCalloutMock implementation.

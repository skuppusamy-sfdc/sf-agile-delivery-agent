## Description

As a Credentialing Specialist, I need the system to automatically create a Provider_Change_Notification__c (PCN) record whenever an inbound CAQH profile status change is received via the DataIntegration layer so that the credentialing team is immediately alerted to review provider attestation changes that may affect network eligibility and active Credentialing_Case__c records. Extends INS-1488 (CAQH inbound feed integration setup).

## Acceptance Criteria

Given the CAQH inbound integration job runs on its scheduled 24-hour cycle,
When a Provider__c record's CAQH_Profile_Status__c field value changes (e.g., from 'Attested' to 'Expired' or 'Not Authorized'),
Then a Provider_Change_Notification__c record is created with: PCN.Change_Type__c = 'CAQH Status Change', PCN.Previous_Value__c = prior CAQH status, PCN.New_Value__c = new CAQH status, PCN.Provider__c = related Provider__c.Id, PCN.Status__c = 'Open', PCN.Priority__c = 'High' (if new status = 'Expired' or 'Suspended'), PCN.Priority__c = 'Medium' (all other status changes).

Given a PCN is created with PCN.Priority__c = 'High',
When the record is saved,
Then a Salesforce in-app notification and email alert is sent to the assigned Credentialing Specialist using the 'PCN_High_Priority_Alert' email template.

Given an open Credentialing_Case__c exists for the same Provider__c,
When the PCN is created,
Then the PCN is automatically associated via PCN.Credentialing_Case__c lookup and Credentialing_Case__c.Status__c is updated to 'Pending CAQH Review'.

Given the CAQH feed returns a status of 'Attested' for a provider whose current CAQH_Profile_Status__c is already 'Attested',
When the integration job processes this record,
Then no PCN is created and no DML is performed on the Provider__c record (idempotency check).

-Given a PCN is created, automatically trigger a DocuSign re-attestation envelope to the provider-
Note: Superseded — DocuSign re-attestation to be handled in a separate story per compliance review.

Given the integration batch completes,
When the Credentialing Specialist reviews the daily PCN queue,
Then all PCNs generated by the batch carry PCN.Source__c = 'CAQH Integration' for audit and reporting purposes.

## Solution

Extend the existing CAQH Scheduled Apex batch class (CAQHProfileSyncBatch) to detect field-level changes on Provider__c.CAQH_Profile_Status__c using a before-and-after comparison pattern within the execute() method. Invoke a PCNCreationService Apex class to handle PCN record creation and priority assignment logic. Use a Record-Triggered Flow on Provider_Change_Notification__c (after insert, PCN.Source__c = 'CAQH Integration' AND PCN.Priority__c = 'High') to send email alerts via the 'PCN_High_Priority_Alert' custom notification type. Implement idempotency by querying existing open PCNs before insert. Add Custom Metadata (CAQH_Status_Priority_Map__mdt) to drive priority assignment rules without hardcoding.

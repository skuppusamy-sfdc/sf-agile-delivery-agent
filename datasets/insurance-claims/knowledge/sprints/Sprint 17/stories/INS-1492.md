## Description

As a Provider Relations Specialist, I need the Provider 360 FlexCard on the Provider__c (Person Account) record page to display a summary of the provider's active Provider_Network_Agreement__c (PNA) and Group_Service_Agreement__c (GSA) records — including counts, effective dates, and network tier — so that I can quickly assess a provider's contracting status without navigating away from the provider record. This story extends INS-1471 (Provider 360 base FlexCard implementation).

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page,
When the Provider 360 FlexCard loads,
Then it displays two summary tiles: 'Active PNAs' showing count of Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active' AND PNA__c.Provider__c = current provider, and 'Active GSAs' showing count of Group_Service_Agreement__c records where GSA__c.Status__c = 'Active' AND GSA__c.Provider__c = current provider.

Given the Active PNAs tile shows count > 0,
When the specialist clicks the tile,
Then a drill-down panel expands inline listing each PNA with columns: PNA__c.Agreement_Number__c, PNA__c.Network_Name__c, PNA__c.Network_Tier__c, PNA__c.Effective_Date__c, PNA__c.Expiration_Date__c.

Given a PNA__c.Expiration_Date__c is within 90 days of today,
When the drill-down panel renders,
Then that PNA row is highlighted amber with label 'Expiring Soon'.

Given a PNA__c.Expiration_Date__c is in the past and PNA__c.Status__c = 'Expired',
When the drill-down panel renders,
Then expired PNAs are excluded from the active count tile but accessible via a collapsible 'Show Expired (N)' toggle.

-Given the FlexCard displays GSA tile, show Fee_Schedule__c.Base_Rate__c inline — REMOVED per legal review, rates not to be surfaced on provider-facing screens-

Given the Provider__c has zero active PNAs and zero active GSAs,
When the FlexCard loads,
Then both tiles display count = 0 and a contextual message 'No active agreements on file. Initiate contracting via New PNA button.' with a quick action button linking to the New PNA OmniScript.

## Solution

Extend the existing Provider 360 FlexCard (OmniStudio) built in INS-1471. Add two DataRaptor Extract actions: DR_GetActivePNAsByProvider and DR_GetActiveGSAsByProvider, each using SOQL with provider Id as input. FlexCard conditional rendering uses State properties for expand/collapse drill-down panels. Amber highlight implemented via FlexCard conditional CSS class binding on Expiration_Date__c delta calculation. 'New PNA' quick action button invokes OmniScript OS_NewProviderNetworkAgreement. Deploy via OmniStudio version set to Production after sandbox UAT sign-off.

## Description

**Bug Report — UAT Sprint 8**

During UAT validation of the Provider 360 FlexCard, Provider Relations Specialists reported that NPI__c and TIN__c fields are blank on Person Account (Provider__c) records that were recently synced via the NPPES integration batch job. Business Account (Facility__c) providers are unaffected.

**Steps to Reproduce:**
1. Run the NPPES nightly batch job (NppesProviderSyncBatch) in Full Sandbox or trigger manually via Anonymous Apex.
2. Navigate to any Provider__c (Person Account) record updated by the batch.
3. Open the 'Provider 360' FlexCard on the record page.
4. Observe NPI__c and TIN__c tiles — both display blank/null.
5. Navigate directly to the Provider__c record detail page — NPI__c and TIN__c fields are correctly populated at the record level.

**Expected Behavior:** Provider 360 FlexCard should display NPI__c and TIN__c values correctly for both Person Account and Business Account provider types.

**Actual Behavior:** FlexCard DataRaptor Extract is returning null for NPI__c and TIN__c on Person Account records only.

**Root Cause (Initial Assessment):** The DataRaptor Extract 'DR_Get_Provider360_Summary' queries against Account object but uses a hardcoded Object API Name filter of 'Account' without distinguishing the IsPersonAccount flag, causing the field mapping to resolve against Business Account field paths and missing the PersonAccount-specific field mappings for NPI__c and TIN__c.

**Environment:** Full Sandbox. Introduced in Sprint 7 NPPES integration story INS-0641.

## Acceptance Criteria

as per Description

## Solution

1. Open DataRaptor Extract 'DR_Get_Provider360_Summary' in OmniStudio.
2. Add a conditional branch on IsPersonAccount = true to route to a separate output field mapping block that correctly maps Provider__c.NPI__c and Provider__c.TIN__c for Person Account record types.
3. Update the FlexCard 'Provider360_FC' data source binding to pass IsPersonAccount as a context variable to the DataRaptor.
4. Add unit test coverage in NppesProviderSyncBatch_Test to assert NPI__c and TIN__c are non-null post-sync for Person Account records.
5. Validate FlexCard rendering in Full Sandbox for both Person Account and Business Account providers before UAT re-test.

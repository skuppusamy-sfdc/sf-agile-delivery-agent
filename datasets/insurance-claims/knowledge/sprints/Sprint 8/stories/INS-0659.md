## Description

**Bug Report — UAT Sprint 8**

On the Provider 360 FlexCard ('ProviderSummary_FlexCard'), the Provider_Network_Agreement__c.Effective_End_Date__c is displaying the earliest PNA end date across all LOBs rather than the end date specific to each LOB row. This causes Provider Relations Specialists to see misleading contract expiry information when a provider holds separate PNAs for Medical, Dental, and Vision LOBs.

**Steps to Reproduce:**
1. Open a Provider__c (Person Account) record that has 3 active PNA records: one Medical (ends 12/31/2025), one Dental (ends 06/30/2025), one Vision (ends 12/31/2026).
2. Navigate to the Provider 360 FlexCard embedded on the Provider__c layout.
3. Expand the 'Active Agreements' section.
4. Observe all three rows display '06/30/2025' as the Effective_End_Date__c.

**Expected:** Each PNA row displays its own Effective_End_Date__c.
**Actual:** All rows display the minimum date across the PNA set.

**Root Cause:** The DataRaptor Extract 'DR_GetProviderPNAs' has a MIN() aggregate on Effective_End_Date__c that was inadvertently left from a prior prototype query.

## Acceptance Criteria

as per Description

## Solution

1. Open DataRaptor Extract 'DR_GetProviderPNAs' in OmniStudio.
2. Remove the MIN() aggregate function from the Effective_End_Date__c field mapping; revert to a direct field extract.
3. Confirm the DataRaptor returns one row per Provider_Network_Agreement__c record keyed on PNA Id.
4. Republish the FlexCard 'ProviderSummary_FlexCard' after DataRaptor update.
5. Regression test across 5 provider profiles covering single-LOB, multi-LOB, and terminated PNA scenarios in UAT.

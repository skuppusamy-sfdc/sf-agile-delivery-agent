## Description

As a Broker Relations Coordinator, I need a guided OmniScript to onboard new group/employer accounts, automatically generate a Group_Service_Agreement__c (GSA) record populated with negotiated plan details, and route the agreement for e-signature via DocuSign, so that the onboarding process is standardized, auditable, and reduces manual contract preparation time by at least 60%.

This story covers the core onboarding OmniScript build and DocuSign routing. Workday billing sync for new groups will be addressed in a follow-on story.

## Acceptance Criteria

Given a Broker Relations Coordinator initiates the 'Group Employer Onboarding' OmniScript from an Account record of type 'Employer Group',
When Step 1 'Group Details' is completed (Group_Name__c, Employer_TIN__c, SIC_Code__c, Effective_Date__c, Renewal_Date__c, Broker_of_Record__c),
Then the data is validated against existing Account records to prevent duplicate TIN entries, displaying 'A group with this TIN already exists: [Account Name]' if a match is found.

Given Step 2 'Plan Selection' is completed,
When the coordinator selects one or more Plan__c records and enters enrolled member counts per plan,
Then a Group_Service_Agreement__c draft record is created with: Status__c = 'Draft', GSA_Effective_Date__c = Effective_Date__c, GSA_Renewal_Date__c = Renewal_Date__c, Total_Enrolled_Members__c = sum of all plan enrollment inputs, and a junction object record (GSA_Plan__c) created per selected plan.

Given the GSA draft record is created,
When Step 3 'Review & Sign' is reached,
Then a Conga CLM-generated GSA PDF is previewed inline within the OmniScript using a rendered document component.

Given the coordinator confirms the document and clicks 'Send for Signature',
When the DocuSign envelope is dispatched,
Then: DocuSign Envelope Status on the GSA = 'Sent', the primary signatory is populated from Group_Contact__c.Email__c, and Group_Service_Agreement__c.Status__c = 'Pending Signature'.

Given the DocuSign envelope is completed by all signatories,
When the DocuSign callback webhook fires,
Then Group_Service_Agreement__c.Status__c = 'Active', Signed_Date__c = envelope completion timestamp, and a 'GSA Executed' milestone task is created on the parent Account.

Given any step in the OmniScript is abandoned before submission,
When the user navigates away,
Then the draft GSA record is deleted via a cleanup Integration Procedure to prevent orphaned drafts.

-Given the group has more than 500 enrolled members, auto-route the GSA to the Underwriting Analyst queue for review before DocuSign dispatch.-
(Deferred to Phase 2 — underwriting review thresholds not yet finalized.)

## Solution

1. Build OmniScript 'GroupEmployer_OnboardingScript' with 3 steps: Group Details, Plan Selection, Review & Sign.
2. Use DataRaptor Extract 'DR_DuplicateTINCheck' to query existing Account records by Employer_TIN__c in Step 1.
3. Use Integration Procedure 'IP_CreateGSADraft' (calling DataRaptor Transform + Upsert) to create the GSA and GSA_Plan__c junction records from Step 2 outputs.
4. Integrate Conga CLM via REST callout within the Integration Procedure to generate the GSA PDF using template 'GSA_Standard_Template_v3'.
5. Use the DocuSign for Salesforce managed package action 'Send with DocuSign' triggered from an Apex invocable action within the OmniScript Submit action sequence.
6. Configure a DocuSign Connect webhook to call an Apex REST endpoint ('DocuSignCallbackController') that updates GSA Status__c and fires a Platform Event 'GSAExecuted__e'.
7. A Platform Event–triggered Flow listens for 'GSAExecuted__e' to create the milestone Task and update the Account.
8. Implement the orphan-cleanup Integration Procedure 'IP_CleanupDraftGSA' invoked on OmniScript cancel/abandon event.

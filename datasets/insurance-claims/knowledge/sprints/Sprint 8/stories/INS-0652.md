## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can instantly view a provider's active credentialing status, current network agreements, fee schedule assignments, and any open PCNs or termination cases without navigating across multiple related lists.

This story extends the foundational Provider data model established in INS-0603 and incorporates the CAQH integration data surfaced in INS-0618.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the card displays Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Specialty__c, and Provider__c.Board_Certification_Status__c in a summary header section.

Given a provider has one or more active Provider_Network_Agreement__c records,
When the PNA section of the FlexCard renders,
Then each PNA is listed with PNA__c.Network_Name__c, PNA__c.Effective_Date__c, PNA__c.Termination_Date__c, and PNA__c.Contract_Status__c, color-coded green for 'Active', yellow for 'Pending Renewal', red for 'Terminated'.

Given a provider has an associated Credentialing_Case__c record,
When the Credentialing section renders,
Then Credentialing_Case__c.Status__c, Credentialing_Case__c.CAQH_Profile_Status__c, and Credentialing_Case__c.Next_Recredentialing_Date__c are displayed, with a warning badge if Next_Recredentialing_Date__c is within 90 days of today.

Given a provider has open Provider_Change_Notification__c records with PCN__c.Status__c = 'Open' or 'Pending Review',
When the PCN section renders,
Then all open PCNs are listed with PCN__c.Change_Type__c and PCN__c.Submitted_Date__c, and a count badge shows the total number of open PCNs.

Given a provider has an active Network_Termination_Case__c,
When the FlexCard loads,
Then a prominent red banner displays 'TERMINATION IN PROGRESS' with Network_Termination_Case__c.Effective_Termination_Date__c.

-Given the FlexCard is viewed by a Claims Operations Manager, the Fee_Schedule__c assignment section is hidden for that role.- (Superseded — all roles now see fee schedule summary per updated business requirements from UAT feedback session 2024-11-14.)

Given any user views the FlexCard,
When the Fee Schedule section renders,
Then the most recently effective Fee_Schedule__c record is shown with Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.LOB__c, and Fee_Schedule__c.Effective_Date__c.

## Solution

Build using OmniStudio FlexCard with multiple DataRaptor Extract (DRE) data sources: (1) DRE_Provider_Summary pulling Provider__c base fields, (2) DRE_Active_PNAs querying Provider_Network_Agreement__c filtered by Contract_Status__c IN ('Active','Pending Renewal'), (3) DRE_Credentialing pulling latest Credentialing_Case__c via SOQL ordered by CreatedDate DESC LIMIT 1, (4) DRE_Open_PCNs aggregating Provider_Change_Notification__c counts, (5) DRE_Active_Termination querying Network_Termination_Case__c where Status__c != 'Closed'. Use FlexCard conditional visibility states for the termination banner. Apply custom CSS for color-coded PNA status badges. Embed FlexCard on Provider__c Lightning Record Page via App Builder. Assign visibility via Permission Set Group 'Provider_Relations_Full_Access'. Cache DataRaptor responses with a 5-minute TTL to reduce SOQL query load on high-traffic record pages.

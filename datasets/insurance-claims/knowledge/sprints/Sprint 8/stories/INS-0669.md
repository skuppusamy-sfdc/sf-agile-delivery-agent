## Description

**Bug Report — UAT Sprint 8**

The Provider Credentialing OmniScript (OS_ProviderCredentialingInitiation) throws an unhandled null pointer error and blocks submission when the CAQH API returns a null value for the provider's last attestation date. This is occurring for newly registered CAQH profiles where attestation has not yet been completed.

**Steps to Reproduce:**
1. Log in as a Credentialing Specialist.
2. Navigate to a Provider__c record for a provider with a newly registered CAQH profile (CAQH_Profile_Status__c = 'Registered — Pending Attestation').
3. Launch the OmniScript 'OS_ProviderCredentialingInitiation' from the quick action menu.
4. Complete Steps 1–3 of the OmniScript (Provider Identification, Specialty Info, License Entry).
5. On Step 4 (CAQH Verification), the OmniScript calls the CAQH Integration Procedure to retrieve profile data.
6. Observe: OmniScript displays generic error 'An unexpected error occurred. Please contact your administrator.' and the Next button is disabled.
7. Browser console shows: 'TypeError: Cannot read property 'attestation_date' of null' in the Integration Procedure response mapping.

**Expected Behavior:** When CAQH returns a null attestation date, the OmniScript should display a warning message 'CAQH attestation not yet completed — credentialing case will be created in Pending Attestation status' and allow the Credentialing Specialist to proceed.

**Actual Behavior:** OmniScript crashes on Step 4 and cannot be submitted.

**Environment:** Acme UAT Org
**Frequency:** 100% reproducible for CAQH_Profile_Status__c = 'Registered — Pending Attestation'
**Impact:** Credentialing Specialists cannot initiate credentialing cases for new providers, blocking onboarding pipeline.

as per Description

## Solution

Fix the Integration Procedure 'IP_CAQH_ProfileRetrieval' to add a null-check Conditional Action on the attestation_date node before mapping to the OmniScript context variable. If null is returned: (1) set context variable caqhAttestationDate = 'Pending'; (2) set context variable caqhAttestationWarning = true. In OmniScript Step 4, add a Conditional Display block showing the warning message when caqhAttestationWarning = true. Update the Credentialing_Case__c creation DataRaptor (DR_Create_Credentialing_Case) to set Status__c = 'Pending Attestation' when attestation date is null, rather than attempting to stamp CAQH_Last_Attestation_Date__c with a null value. Add unit test to IP for null response scenario. Regression test all CAQH integration paths.

## Description

**Bug Report — UAT Sprint 8 / Regression**

A regression was introduced in Sprint 7 (suspected INS-0611 deployment) that allows a Network_Termination_Case__c to be saved against a Facility__c (Business Account) record without a valid 10-digit NPI on the Provider__c lookup field. The NPPES validation that was in place for Person Account (individual provider) terminations does not extend to Business Account (facility) terminations, and the Sprint 7 changes appear to have broken the shared validation rule.

**Steps to Reproduce:**
1. Navigate to Network Termination Cases → New.
2. Set Termination_Case__c.Termination_Entity_Type__c = 'Facility'.
3. In the Facility__c lookup, select a Facility__c (Business Account) record where Account.NPI__c is blank or malformed (e.g., '12345').
4. Complete remaining required fields and click Save.
5. Observe: record saves successfully without NPI validation error.

**Expected:** Validation should fire: 'Facility NPI is required and must be a valid 10-digit NPI before a termination case can be submitted. Verify via NPPES integration.'

**Actual:** Record saves with no error. Network_Termination_Case__c.Status__c = 'Submitted' even with an invalid or missing NPI on the Facility account.

**Environment:** UAT Sandbox (Acme-UAT1)
**Regression from:** Sprint 7 — INS-0611 (NPI lookup refactor)
**Impact:** Compliance Officer flagged this as a regulatory risk — CMS requires valid NPIs on all termination notifications. SIU may also be impacted if terminated facilities bypass NPI checks.

## Solution

Root cause: INS-0611 refactored the NPI validation from a Validation Rule on Network_Termination_Case__c to an Apex Trigger ('NetworkTerminationTrigger') but the trigger's NPI check only evaluates the Person Account path (IF Account.IsPersonAccount = true). The Business Account (Facility) path was not included. Fix: (1) Update 'NetworkTerminationTrigger' → 'NetworkTerminationTriggerHandler.validateNPI()' to handle both branches: Person Account (Provider NPI) and Business Account (Facility NPI via Account.NPI__c). (2) Add a SOQL lookup to the related Facility__c Account record to retrieve NPI__c when Termination_Entity_Type__c = 'Facility'. (3) Throw an AuraHandledException with the prescribed message if NPI is null or does not match regex '^[0-9]{10}$'. (4) Re-enable the original Validation Rule as a fallback safety net (was deactivated in INS-0611 — reactivate for both entity types). (5) Deploy fix to UAT and run regression suite 'TerminationCase_Regression_Suite' in Acme-UAT1.

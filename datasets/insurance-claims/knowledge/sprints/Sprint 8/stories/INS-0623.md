## Description

**Bug Report — UAT Sprint 8**

The Provider 360 FlexCard rendered on the Provider__c (Person Account) record page displays the credentialing status badge using a cached DataRaptor value. When a Credentialing_Case__c is closed with Status__c = 'Approved', the FlexCard badge continues to show 'In Review' until the page is manually hard-refreshed (Ctrl+F5). This is a browser-cache/Platform Event wiring issue.

**Steps to Reproduce:**
1. Open a Provider__c record page where an active Credentialing_Case__c exists with Status__c = 'In Review'.
2. In a separate tab, navigate to the Credentialing_Case__c and set Status__c = 'Approved', then save.
3. Return to the Provider 360 tab without refreshing.
4. Observe the credentialing badge on the FlexCard.

**Expected:** FlexCard badge auto-refreshes to display 'Approved' within 5 seconds via Platform Event subscription.

**Actual:** Badge remains 'In Review'. No automatic refresh occurs.

## Solution

The Provider_360_CredentialingStatus DataRaptor Extract was not wired to the CredentialingStatusChanged__e Platform Event published by the CredentialingCaseStatusTrigger Apex trigger. Fix: (1) Verify Apex trigger publishes CredentialingStatusChanged__e on Credentialing_Case__c after-update when Status__c changes to 'Approved' or 'Denied'. (2) Update the Provider_360_FlexCard OmniScript action to subscribe to CredentialingStatusChanged__e via empApi in the FlexCard's LWC wrapper component and invoke a DataRaptor refresh action on event receipt. (3) Set cache TTL on the DataRaptor Extract to 0 for this specific card action. Validated in scratch org; deployed to full sandbox INT-01.

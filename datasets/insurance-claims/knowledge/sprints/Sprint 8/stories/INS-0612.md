## Description

As a Provider Network Specialist, I need an automated workflow triggered when a Network_Termination_Case__c is created so that the provider off-boarding process is consistently executed — including PNA termination date stamping, Provider_Change_Notification__c (PCN) generation, open claims flagging, and DocuSign termination letter dispatch — reducing manual effort and ensuring regulatory compliance with 90-day notice requirements.

This story extends INS-0589 (Network Termination Case Object Build) by adding the automation layer on top of the data model.

## Acceptance Criteria

Given a Network_Termination_Case__c record is created with Status__c = 'Approved' and Termination_Reason__c populated,
When the automation workflow triggers,
Then the following actions must execute in sequence:

**Step 1 – PNA Termination Date Stamping**
Given Network_Termination_Case__c.Provider__c is linked to one or more active Provider_Network_Agreement__c records,
When the workflow runs,
Then all PNA records where PNA.Status__c = 'Active' AND PNA.Provider__c = termination case provider are updated: PNA.Termination_Date__c = Network_Termination_Case__c.Effective_Termination_Date__c and PNA.Status__c = 'Pending Termination'.
PNA.Status__c must NOT be set to 'Terminated' until Effective_Termination_Date__c is reached (handled by a separate scheduled job).

**Step 2 – PCN Generation**
Given PNA records are updated to 'Pending Termination',
When the workflow proceeds,
Then a Provider_Change_Notification__c record is created with: PCN.Type__c = 'Termination', PCN.Provider__c = termination case provider, PCN.Effective_Date__c = Network_Termination_Case__c.Effective_Termination_Date__c, PCN.Network_Termination_Case__c = the originating case Id, PCN.Status__c = 'Draft'.

**Step 3 – Open Claims Flagging**
Given the provider has open Claim__c records (Status__c NOT IN ('Closed','Denied','Void','Adjudicated')),
When the workflow runs,
Then each open Claim__c where Claim__c.Rendering_Provider__c = termination provider is updated: Claim__c.Provider_Termination_Flag__c = true and Claim__c.Termination_Review_Required__c = true.
Additionally, a Case (standard object) is created linked to each flagged Claim__c and assigned to the Claims Operations Manager queue with Subject = 'Provider Termination — Open Claim Review Required'.

**Step 4 – DocuSign Termination Letter**
Given PCN.Status__c = 'Draft' and Network_Termination_Case__c.Send_Termination_Letter__c = true,
When the workflow reaches Step 4,
Then a DocuSign envelope is sent to Provider__c.Email using the 'Provider_Termination_Letter_v2' DocuSign template, with merge fields: provider name, NPI, effective termination date, and list of affected network names.
Network_Termination_Case__c.DocuSign_Envelope_Id__c must be populated with the returned envelope ID.

**Step 5 – Notification**
Given all prior steps complete successfully,
When the workflow finishes,
Then Network_Termination_Case__c.Status__c = 'In Progress' and a Chatter post is made on the case record tagging the assigned Provider Network Specialist: '@[Specialist] Termination workflow initiated for [Provider Name] — effective [date]. PCN #[PCN Id] created. [X] open claims flagged.'

Given any step fails,
When an exception is caught,
Then Network_Termination_Case__c.Workflow_Error__c is populated with the error message and Status__c = 'Workflow Error', and an email alert is sent to the Provider Network Operations email group.

## Solution

1. Build an Apex class 'NetworkTerminationWorkflowService' with a public static method executeTerminationWorkflow(Id terminationCaseId) containing all five steps in a single transaction with savepoints for rollback on failure.
2. Trigger the service via a Record-Triggered Flow 'RT_NetworkTerminationCase_WorkflowLaunch' on Network_Termination_Case__c, firing on Create and Update when Status__c changes to 'Approved', using an Apex Action element to invoke the service asynchronously via Queueable Apex (InsTerminationWorkflowQueueable) to avoid governor limit issues with bulk PNA and Claim__c updates.
3. DocuSign envelope creation handled via DocuSign for Salesforce managed package API (DSFSDocuSignAPI) called from within the Queueable class.
4. Chatter post created using ConnectApi.ChatterFeeds.postFeedElement within the Apex service.
5. Error handling uses a try/catch block updating Network_Termination_Case__c.Workflow_Error__c via a separate DML operation outside the primary savepoint.
6. Build a scheduled Apex job 'InsProviderTerminationActivationBatch' running nightly to flip PNA.Status__c from 'Pending Termination' to 'Terminated' when Effective_Termination_Date__c = TODAY().
7. Create Apex test class 'NetworkTerminationWorkflowServiceTest' with mocked DocuSign callouts and minimum 85% coverage.

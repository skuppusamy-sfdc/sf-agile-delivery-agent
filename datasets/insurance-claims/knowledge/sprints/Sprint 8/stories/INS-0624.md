## Description

As a Provider Relations Specialist, I need an automated notification workflow that alerts me when a Provider_Network_Agreement__c (PNA) is approaching its expiration date so that I can proactively initiate contract renewal discussions and avoid gaps in network coverage that could disrupt claims processing for in-network providers.

This story extends INS-0592 (PNA Lifecycle Management) by adding the outbound notification layer on top of the existing PNA status automation.

## Acceptance Criteria

Given a Provider_Network_Agreement__c record where Contract_End_Date__c is within 90 days of today AND PNA_Status__c = 'Active',
When the scheduled daily batch job 'PNA_ExpiryNotificationBatch' executes at 06:00 AM local time,
Then a Task__c record is created with Subject = 'PNA Expiry Alert — 90 Days', ActivityDate = today, OwnerId = PNA.Provider_Relations_Owner__c, and Status = 'Open'.

Given the same PNA is within 30 days of expiry,
When the batch executes,
Then an additional high-priority Task__c is created with Priority = 'High' AND an in-app Bell Notification is sent to the assigned Provider_Relations_Owner__c via Custom Notification Type 'PNA_Expiry_Critical'.

Given PNA_Status__c transitions to 'Renewal In Progress' (set manually by the specialist),
When the record is saved,
Then no further expiry Tasks are created for that PNA until Status__c returns to 'Active' or becomes 'Expired'.

Given a PNA where Contract_End_Date__c has passed AND PNA_Status__c is still 'Active',
When the batch executes,
Then PNA_Status__c is auto-updated to 'Expired' and a Case is created with RecordType = 'Network_Gap_Review' linked to the associated Provider__c.

-Given PNA expiry, send external email to provider NPI contact on file- (deprecated — provider outbound comms moved to PCN process per INS-0601).

## Solution

Implement Apex Batchable class 'PNA_ExpiryNotificationBatch' implementing Database.Batchable<SObject> and Schedulable. Query: SELECT Id, Contract_End_Date__c, PNA_Status__c, Provider_Relations_Owner__c, Provider__c FROM Provider_Network_Agreement__c WHERE PNA_Status__c = 'Active'. In execute(), calculate days-to-expiry using Date.today(). Use TaskService utility class to insert Task records. Leverage CustomNotification API for 30-day bell alerts. Schedule via System.schedule() in anonymous Apex post-deploy script. Record-Triggered Flow on Provider_Network_Agreement__c suppresses batch task creation when PNA_Status__c = 'Renewal In Progress'. Permission Set Group 'Provider_Relations_PSG' grants read access to new Custom Notification Type.

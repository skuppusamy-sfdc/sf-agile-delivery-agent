## Description

**Bug Summary:** After batch EOB generation runs overnight, the parent Claim__c.Status__c field is not rolling up correctly from adjudicated Claim_Line__c records, leaving Claims in 'Pending Adjudication' status despite all lines being fully processed.

**Steps to Reproduce:**
1. Create a Claim__c record with 3 or more Claim_Line__c child records in status 'Submitted'.
2. Trigger the nightly batch EOB generation job (InsBatchEOBGenerator) via Developer Console or scheduled job.
3. Confirm all Claim_Line__c records update to Status__c = 'Adjudicated' and Allowed_Amount__c is populated.
4. Navigate back to the parent Claim__c record.
5. Observe that Claim__c.Status__c remains 'Pending Adjudication' and Total_Allowed_Amount__c rollup field shows $0.00.

**Expected Behavior:** Once all child Claim_Line__c records reach Status__c = 'Adjudicated', the parent Claim__c.Status__c should automatically update to 'Adjudicated' and Total_Allowed_Amount__c should reflect the sum of all Claim_Line__c.Allowed_Amount__c values.

**Actual Behavior:** Parent Claim__c.Status__c remains 'Pending Adjudication'. Total_Allowed_Amount__c = $0.00. EOB__c record is generated but references incorrect claim totals.

**Root Cause (Suspected):** The Record-Triggered Flow 'Update_Claim_Status_on_Line_Adjudication' has a filter condition checking Claim_Line__c.Last_Modified_By = current user, which excludes batch context updates running as the integration service account.

## Acceptance Criteria

as per Description

## Solution

1. Modify the Record-Triggered Flow 'Update_Claim_Status_on_Line_Adjudication' to remove the Last_Modified_By filter condition that excludes batch/system context updates.
2. Replace the user-context filter with a condition on Claim_Line__c.Status__c EQUALS 'Adjudicated' (prior value not equal to 'Adjudicated') to ensure idempotency.
3. Update the Roll-Up Summary field Total_Allowed_Amount__c on Claim__c to use a FILTER condition of Claim_Line__c.Status__c = 'Adjudicated' only.
4. Add a scheduled Apex fallback job (InsClaimStatusReconciliationBatch) that runs post-EOB batch to catch any records missed by the flow, querying Claim__c records where Status__c = 'Pending Adjudication' and all child Claim_Line__c.Status__c = 'Adjudicated'.
5. Deploy fix to UAT sandbox and re-execute regression test suite INS-RT-044.

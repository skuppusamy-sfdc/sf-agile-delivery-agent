## Description

**Bug Report — UAT Sprint 8**

During UAT regression testing of COB workflows, the Explanation_of_Benefits__c record is not reflecting updated Claim_Line__c adjudication status after a secondary payer processes a COB payment.

**Steps to Reproduce:**
1. Create a Policy__c record with COB flag enabled (COB_Active__c = true).
2. Submit a Claim__c with two Claim_Line__c records against the policy.
3. Process a primary payer Payment__c record — EOB generates correctly.
4. Navigate to the secondary payer flow and submit a COB Payment__c record referencing the same Claim__c.
5. Observe the Explanation_of_Benefits__c.Line_Status__c field on each associated Claim_Line__c.

**Expected Behavior:**
All Claim_Line__c records linked to the EOB should reflect Line_Status__c = 'Adjudicated - COB Applied' after secondary payment is posted.

**Actual Behavior:**
Claim_Line__c.Line_Status__c remains 'Pending Secondary' indefinitely. The EOB PDF generated via Conga CLM does not include the secondary payer adjustment amounts.

**Environment:** Full Sandbox (ACME-FS01)
**Regression introduced by:** INS-0651 (COB Payment Engine refactor)

## Solution

Root cause identified: the Record-Triggered Flow on Payment__c (Flow: 'Update_EOB_On_Payment_Post') was filtering on Payment__c.Payer_Sequence__c = 'Primary' only, excluding secondary COB payments. Fix: updated Flow entry criteria to include Payer_Sequence__c IN ('Primary', 'Secondary'). Added a second Update Records element to set Claim_Line__c.Line_Status__c = 'Adjudicated - COB Applied' when Payment__c.COB_Payment__c = true. Deployed a fix to the Conga CLM template merge field mapping to include Secondary_Payer_Adjustment__c on the EOB data model. Regression test suite updated in Copado.

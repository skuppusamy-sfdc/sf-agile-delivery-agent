## Description

As a Claims Operations Manager, I need the Agentforce AI assistant to analyze incoming FNOL submissions and recommend a triage category (Straight-Through Processing, Complex Review, or SIU Referral) based on claim attributes, historical patterns, and policy standing so that claims are routed to the correct handling team faster and manual triage effort is reduced by an estimated 40%.

This is the Phase 1 implementation covering Medical LOB claims only. Auto/Property LOB triage is scoped to Phase 2.

## Acceptance Criteria

Given a new Claim__c is created with Claim__c.FNOL_Submitted__c = TRUE and Claim__c.LOB__c = 'Medical',
When the Agentforce triage agent processes the submission,
Then Claim__c.AI_Triage_Recommendation__c is populated with one of: 'Straight-Through Processing', 'Complex Review', or 'SIU Referral' within 60 seconds of FNOL submission.

Given Claim__c.Total_Billed_Amount__c > $50,000 OR Claim__c.Diagnosis_Code__c (ICD-10) is on the high-complexity watchlist (stored in Custom Metadata 'ICD10_Complexity__mdt'),
When the AI triage runs,
Then AI_Triage_Recommendation__c = 'Complex Review' and Claim__c.AI_Triage_Confidence_Score__c is populated as a percentage.

Given the rendering provider's NPI appears in the SIU watchlist (SIU_Provider_Watch__c object),
When the Agentforce agent evaluates the claim,
Then AI_Triage_Recommendation__c = 'SIU Referral' regardless of billed amount, and a SIU case is auto-created linked to the Claim__c.

Given AI_Triage_Confidence_Score__c < 70%,
When the recommendation is written to the Claim__c,
Then Claim__c.AI_Triage_Requires_Human_Review__c = TRUE and the claim is added to the 'Low Confidence Triage' queue for Claims Operations review.

Given a Claims Operations Manager overrides the AI recommendation,
When Claim__c.Triage_Category__c is manually updated,
Then Claim__c.AI_Triage_Override__c = TRUE, Claim__c.AI_Override_Reason__c is required (picklist), and the override is logged to a Claim_Audit__c record for model feedback purposes.

## Solution

Agentforce Agent configured with a custom 'Claims Triage' topic and three agent actions: (1) 'EvaluateClaimComplexity' — Apex invocable querying ICD10_Complexity__mdt and comparing Claim__c.Total_Billed_Amount__c thresholds; (2) 'CheckSIUWatchlist' — queries SIU_Provider_Watch__c by NPI; (3) 'WriteTriageRecommendation' — updates Claim__c fields via Apex. Agent triggered via Platform Event 'FNOL_Submitted__e' published by the FNOL submission Record-Triggered Flow. Confidence score derived from Einstein prompt template response metadata. Override tracking implemented via a separate Record-Triggered Flow on Claim__c.Triage_Category__c change, creating Claim_Audit__c records. Custom Permission Set 'PSG_AgentforceClaimsTriage' controls agent action visibility.

## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record that surfaces active Provider_Network_Agreement__c (PNA) contract details, assigned network tier, and current credentialing status from Credentialing_Case__c so that I can resolve provider inquiries in a single view without navigating across multiple objects.

Extends INS-0584 (Provider 360 base FlexCard scaffold).

## Acceptance Criteria

Given a Provider__c record exists with at least one active PNA,
When a Provider Relations Specialist opens the Provider 360 FlexCard,
Then the FlexCard displays Provider_Network_Agreement__c.Contract_Status__c = 'Active', PNA.Effective_Date__c, PNA.Termination_Date__c, and PNA.Network_Tier__c in a dedicated 'Contracting' section.

Given a Provider__c has an open Credentialing_Case__c,
When the FlexCard loads,
Then a 'Credentialing' section renders Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and a color-coded badge (Green = Active, Amber = Expiring within 90 days, Red = Expired or Lapsed).

Given a Provider__c has multiple PNA records across different LOBs,
When the FlexCard renders the Contracting section,
Then only PNA records where PNA.Contract_Status__c = 'Active' AND PNA.LOB__c is displayed, grouped by LOB with a count indicator for inactive contracts.

-Given no PNA exists, display a static 'No Contract on File' placeholder card with a 'Initiate Contract' action button linking to the PNA creation OmniScript.-

Given no active PNA exists,
When the FlexCard loads,
Then a 'No Active Contract' state is shown with an 'Initiate Contracting' quick action that launches the New_PNA_OmniScript.

Given the FlexCard is viewed by a user without the 'PNA_ReadOnly' Permission Set,
When the Contracting section attempts to render,
Then PNA financial fields (PNA.Base_Rate__c, PNA.Fee_Schedule__c lookup) are masked and a 'Restricted' label is shown.

## Solution

Built using OmniStudio FlexCard with two child DataRaptor Extract (DRE) actions: DRE_GetActivePNA queries Provider_Network_Agreement__c filtered by Provider__c = :recordId AND Contract_Status__c = 'Active', ordered by LOB__c. DRE_GetCredentialingStatus queries the most recent Credentialing_Case__c by Provider__c with a SOQL MAX(LastModifiedDate). FlexCard conditional states configured for empty PNA and expired credentialing scenarios. Badge color logic implemented via FlexCard 'Conditional Styling' rules on Credentialing_Case__c.Expiration_Date__c using TODAY() comparison formulas. Field-level masking enforced via Permission Set Group 'PSG_ProviderRelations_ReadOnly' excluding PNA financial fields. Deployed as a Lightning App Builder component on the Provider__c record page.

## Description

**Bug Report — UAT Sprint 8**

During UAT testing of the CAQH nightly sync integration, Credentialing_Case__c records are being automatically closed with Status__c = 'Closed — Complete' when the inbound CAQH profile sync returns a CAQH_Profile_Status__c value of 'Expired'. This is incorrect behavior — an expired CAQH profile should trigger a status of 'Pending Provider Action' and generate a task for the assigned Credentialing Specialist, not close the case.

**Steps to Reproduce:**
1. Identify a Credentialing_Case__c record with Status__c = 'In Review' and an associated Provider__c whose CAQH profile is set to 'Expired' in the CAQH sandbox.
2. Run the scheduled Apex job 'CAQH_ProfileSync_ScheduledJob' via Setup > Scheduled Jobs or Developer Console.
3. After the job completes, navigate to the Credentialing_Case__c record.
4. Observe that Status__c = 'Closed — Complete' and Credentialing_Case__c.Closure_Reason__c = 'CAQH Verified'.

**Expected Behavior:**
Credentialing_Case__c.Status__c should transition to 'Pending Provider Action'. A new Task should be created assigned to the Credentialing_Case__c.Owner with Subject = 'CAQH Profile Expired — Provider Action Required'. Credentialing_Case__c.CAQH_Profile_Status__c should reflect 'Expired'. No closure should occur.

**Actual Behavior:**
Credentialing_Case__c.Status__c = 'Closed — Complete', Closure_Reason__c = 'CAQH Verified'. Case is closed erroneously. Downstream network enrollment is triggered incorrectly for providers with expired CAQH profiles.

**Root Cause (suspected):**
The Apex class 'CAQHProfileSyncHandler' contains a conditional block that maps CAQH API response status codes to Salesforce statuses. The mapping for status code '04' (Expired) appears to be incorrectly mapped to the same branch as '02' (Active/Complete) due to a copy-paste error in a recent refactor.

as per Description

## Solution

Fix the status code mapping in 'CAQHProfileSyncHandler.cls' — update the switch/case block so that CAQH status code '04' maps to Credentialing_Case__c.Status__c = 'Pending Provider Action' and CAQH status code '02' maps to 'Approved'. Add a separate handler method 'createExpiredCAQHTask()' that inserts a Task record with Subject = 'CAQH Profile Expired — Provider Action Required', WhatId = Credentialing_Case__c.Id, OwnerId = Credentialing_Case__c.OwnerId, ActivityDate = Today + 5. Update Apex test class 'CAQHProfileSyncHandlerTest' to add test methods covering the 'Expired' and 'Inactive' CAQH status code paths with assert statements on Status__c and Task creation. Ensure code coverage remains above 85% after changes.

## Description

As a Credentialing Specialist, I need the system to automatically create a Credentialing_Case__c record when a provider's CAQH re-attestation is within 60 days of expiry or has lapsed, so that my team can proactively manage re-credentialing without relying on manual roster reviews.

This story extends the CAQH polling integration established in INS-0628 and adds the downstream case creation logic.

## Acceptance Criteria

Given a nightly scheduled Apex job (CAQHSyncBatch) polls the CAQH ProView API for all active Provider__c records,
When a provider's CAQH_Reattestiation_Expiry__c date is ≤ TODAY() + 60 days AND no open Credentialing_Case__c with RecordType 'Re-Credentialing' exists for that provider,
Then a new Credentialing_Case__c is created with: Status__c = 'Open', RecordType = 'Re-Credentialing', Provider__c = [linked provider], Due_Date__c = CAQH_Reattestiation_Expiry__c, Priority__c = 'High' if expiry ≤ 30 days else 'Medium'.

Given a Credentialing_Case__c is auto-created,
When the record is inserted,
Then an email notification is sent to the assigned Credentialing Specialist using the 'Credentialing_ReAttestation_Alert' email template, including the provider NPI, TIN, and expiry date in the body.

Given a provider's CAQH re-attestation has already lapsed (CAQH_Reattestiation_Expiry__c < TODAY()),
When the batch job runs,
Then Credentialing_Case__c.Priority__c = 'Critical' and Provider__c.Credentialing_Status__c is set to 'Suspended — Pending Re-Credentialing'.

Given the batch job encounters an API timeout or error for a specific NPI,
When the error occurs,
Then the provider record is skipped, an error row is logged to Integration_Error_Log__c with Source__c = 'CAQH', and the batch continues processing remaining records without full job failure.

-Given a provider opts out of CAQH ProView, the batch should flag them for manual review via a Task.-

## Solution

1. Extend CAQHSyncBatch (Apex, Database.Batchable + Database.AllowsCallouts) to evaluate CAQH_Reattestiation_Expiry__c after each API response is parsed.
2. Implement a duplicate-check SOQL within the batch to query open Credentialing_Case__c records per provider before insertion.
3. Use Apex DML with partial success handling (Database.insert with allOrNone=false) to isolate per-record failures.
4. Fire a 'CredentialingCaseCreated__e' Platform Event post-insert to trigger the email notification Flow ('CredentialingAlert_NotificationFlow').
5. Log API errors to Integration_Error_Log__c via a dedicated error handler utility class.
6. Schedule the batch via a Scheduled Apex class to run daily at 02:00 AM org time.
7. Assign the 'Credentialing_Specialist_PS' Permission Set to ensure case visibility is scoped correctly.

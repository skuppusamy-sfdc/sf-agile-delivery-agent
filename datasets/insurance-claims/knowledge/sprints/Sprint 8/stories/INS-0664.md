## Description

As a Claims Operations Manager, I need an Agentforce-powered prompt template to automatically draft the narrative section of an Explanation_of_Benefits__c (EOB) record based on Claim__c adjudication data, so that claims examiners spend less time authoring EOB language and can focus on exception handling and member communications.

The generated narrative should summarize claim disposition, any applied Fee_Schedule__c adjustments, COB sequencing results, and denial reason codes in plain language appropriate for member-facing communications.

## Acceptance Criteria

Given a Claim__c record with Status__c = 'Adjudicated' and at least one associated Claim_Line__c record,
When a claims examiner clicks the 'Generate EOB Narrative' action on the Claim__c record page,
Then the Agentforce prompt template 'EOB_NarrativeGen_Prompt' is invoked and a draft narrative is populated in Explanation_of_Benefits__c.Narrative_Draft__c within 10 seconds.

Given the prompt template is invoked,
When constructing the prompt context,
Then the following fields are injected as grounding data: Claim__c.Total_Billed_Amount__c, Claim__c.Total_Allowed_Amount__c, Claim__c.Total_Paid_Amount__c, Claim__c.Denial_Reason_Code__c, Claim__c.COB_Indicator__c, and a list of Claim_Line__c CPT codes and their adjudicated statuses.

Given the narrative draft is generated,
When it is displayed to the examiner,
Then the examiner can edit the draft inline in a Rich Text field before clicking 'Finalize EOB', which sets Explanation_of_Benefits__c.Status__c = 'Final' and locks the Narrative field.

Given Claim__c.Denial_Reason_Code__c is populated,
When the narrative is generated,
Then the denial reason is expressed in plain-language member-appropriate text mapped from the internal code via a custom metadata type 'Denial_Code_Plain_Language__mdt'.

Given the Agentforce API call fails or times out,
When the error occurs,
Then a user-facing toast message displays 'AI narrative generation is temporarily unavailable. Please draft manually.' and the error is logged to Integration_Error_Log__c with Source__c = 'Agentforce'.

## Solution

1. Create an Agentforce Prompt Template 'EOB_NarrativeGen_Prompt' of type 'Record' scoped to the Claim__c object, referencing merge fields for all required grounding data fields.
2. Build a custom LWC 'eobNarrativeGenerator' embedded on the Claim__c Lightning Record Page that invokes the prompt template via the Connect API (einstein/prompt-templates endpoint) using a wire adapter.
3. The LWC renders the returned text in a lightning-input-rich-text component for inline editing.
4. On 'Finalize EOB', an Apex controller method upserts the Explanation_of_Benefits__c record, sets Status__c = 'Final', and applies a field-level security lock via a Record-Triggered Flow that sets a Locked__c checkbox, triggering a Validation Rule to block further edits.
5. Build a custom metadata type 'Denial_Code_Plain_Language__mdt' with fields: Denial_Code__c (text), Plain_Language_Description__c (textarea); pre-populate with standard CARC/RARC codes.
6. Inject denial code plain-language mapping into the prompt via an Apex string builder prior to the API call.
7. Wrap the API callout in try/catch; on exception, publish an error Platform Event and surface the toast via LWC event handling.

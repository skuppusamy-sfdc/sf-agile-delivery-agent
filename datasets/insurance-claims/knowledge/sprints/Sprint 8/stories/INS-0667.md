## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can immediately assess a provider's network participation status, credentialing standing, and any outstanding change notifications without navigating across multiple objects.

This story extends the foundational Provider data model established in INS-0636 and incorporates the PNA status logic from the NetworkContracting epic.

## Acceptance Criteria

Given a Provider Relations Specialist navigates to a Provider__c (Person Account) record page,
When the Provider 360 FlexCard loads,
Then it must display the following sections without additional user interaction:

**Section 1 — Provider Summary**
Given the FlexCard is rendered,
When provider data is fetched,
Then Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Primary_Specialty__c, and Provider__c.Board_Certification_Status__c must be visible in the header band.

**Section 2 — Active Network Agreements**
Given the provider has one or more Provider_Network_Agreement__c records,
When PNA_Status__c = 'Active' and PNA_Effective_Date__c <= TODAY() and PNA_Termination_Date__c >= TODAY(),
Then each active PNA must display: PNA Name, LOB__c, Network_Tier__c, and Fee_Schedule__c lookup name.
-Given PNA_Status__c = 'Pending', display in amber badge- (deprecated — pending PNAs now shown in separate Contracting tab per INS-0651)

**Section 3 — Credentialing Status**
Given a Credentialing_Case__c exists linked to the provider,
When Credentialing_Case__c.Status__c is any value,
Then display the most recent Credentialing_Case__c with fields: Status__c, Expiration_Date__c, CAQH_Profile_Status__c, and Primary_Source_Verification_Date__c.
When no Credentialing_Case__c exists,
Then display 'No Credentialing Record Found' with a 'Initiate Credentialing' quick action button.

**Section 4 — Open Provider Change Notifications (PCNs)**
Given one or more Provider_Change_Notification__c records exist with PCN_Status__c != 'Closed',
When the FlexCard loads,
Then display a count badge and a scrollable list showing PCN_Type__c, Submitted_Date__c, and PCN_Status__c.
When PCN count = 0,
Then display 'No Open PCNs' in the section.

**Section 5 — Network Termination Cases**
Given a Network_Termination_Case__c exists with Termination_Status__c = 'In Progress' or 'Pending Member Notification',
When the FlexCard loads,
Then display a high-visibility warning banner with Termination_Effective_Date__c and Termination_Reason__c.

**Permissions:**
Given a user does NOT have the 'Provider_Relations_Specialist' Permission Set Group,
When they navigate to the Provider__c record page,
Then the Provider 360 FlexCard must not render and must display no data (field-level security enforced via DataRaptor).

## Solution

Build using OmniStudio FlexCard with 5 child cards (datasource per section). DataRaptors: (1) DR_Provider_Header — fetches Provider__c fields; (2) DR_Active_PNA_List — SOQL on Provider_Network_Agreement__c filtered by status and date; (3) DR_Credentialing_Latest — fetches most recent Credentialing_Case__c ordered by CreatedDate DESC LIMIT 1; (4) DR_Open_PCN_List — filters Provider_Change_Notification__c where PCN_Status__c NOT IN ('Closed'); (5) DR_Termination_Warning — checks Network_Termination_Case__c for active terminations. FlexCard deployed to Provider__c Lightning Record Page via App Builder. Visibility rule on FlexCard component tied to Permission Set Group 'Provider_Relations_Specialist'. Quick action 'Initiate Credentialing' invokes OmniScript OS_InitiateCredentialingCase. All DataRaptors respect FLS via 'Respect FLS' checkbox enabled.

## Description

As a Provider Network Specialist, I need NPI and TIN updates sourced from the NPPES and CAQH nightly data feeds to automatically sync to the corresponding Provider__c (Person Account) records and any active Provider_Network_Agreement__c records so that our provider directory remains accurate, claims are not rejected due to NPI/TIN mismatches, and we meet CMS data accuracy requirements for network adequacy reporting.

The integration receives a nightly JSON payload from CAQH ProView and a flat-file extract from NPPES. Both must be reconciled against Provider__c.NPI__c and Provider__c.TIN__c.

## Acceptance Criteria

Given the nightly NPPES flat-file is deposited in the designated MuleSoft SFTP endpoint,
When the DataIntegration batch job 'NPPES_Sync_Batch' executes at 02:00 AM,
Then each Provider__c record where NPI__c matches the NPPES record is updated with any changed fields: License_State__c, License_Status__c, Provider_Enumeration_Date__c.

Given a CAQH ProView payload includes a TIN change for a provider,
When the sync job processes the record,
Then Provider__c.TIN__c is updated AND a Provider_Change_Notification__c (PCN) record is created with Change_Type__c = 'TIN Update', Previous_Value__c = old TIN, New_Value__c = new TIN, and PCN_Status__c = 'Pending Review'.

Given a PCN is created for a TIN change,
When the PCN is created,
Then all active Provider_Network_Agreement__c records linked to the provider are updated to PNA_Status__c = 'Pending TIN Verification' and a Task is assigned to the Provider_Network_Specialist__c owner.

Given an NPI in the NPPES feed that does not match any Provider__c.NPI__c in Salesforce,
When the sync job encounters the record,
Then the record is written to the Integration_Error_Log__c object with Error_Type__c = 'NPI Not Found', Source_System__c = 'NPPES', and Raw_Payload__c = the JSON/flat-file row for manual review.

Given the sync job completes,
When all records are processed,
Then a summary Platform Event 'IntegrationSyncComplete__e' is published with fields: Total_Processed__c, Total_Updated__c, Total_Errors__c, Source_System__c = 'NPPES/CAQH', and a notification is sent to the DataIntegration ops queue.

## Solution

MuleSoft integration flow deposits parsed JSON to Salesforce via REST Composite API batch upsert on Provider__c using NPI__c as external ID. Apex Batchable 'NPPES_Sync_Batch' processes Integration_Staging__c records created by MuleSoft. PCN creation handled by Record-Triggered Flow on Provider__c (after update, when TIN__c is changed). PNA status update triggered by second Record-Triggered Flow on Provider_Change_Notification__c (after insert). Integration_Error_Log__c populated via Apex exception handler. Platform Event published at batch finish() method. Custom Permission Set 'Integration_Service_Account_PS' grants FLS on all touched fields to the named integration user.

## Description

**Bug Report — UAT Sprint 8**

During UAT regression testing, the Claims Operations team identified that after batch EOB generation runs overnight, the parent Claim__c.Status__c field is not rolling up correctly from child Claim_Line__c records.

**Steps to Reproduce:**
1. Create a Claim__c record with 3 or more Claim_Line__c child records in status 'Pending Adjudication'.
2. Trigger the nightly batch EOB generation job (BatchEOBGenerator class) manually via Anonymous Apex.
3. Confirm that Explanation_of_Benefits__c records are created for each Claim_Line__c.
4. Navigate back to the parent Claim__c record.
5. Observe that Claim__c.Status__c remains 'In Review' instead of updating to 'Adjudicated'.

**Expected Behavior:** Once all child Claim_Line__c records have an associated Explanation_of_Benefits__c with EOB.Status__c = 'Finalized', the parent Claim__c.Status__c should automatically update to 'Adjudicated'.

**Actual Behavior:** Claim__c.Status__c stays at 'In Review' indefinitely. Manual refresh and cache clearing do not resolve the issue. Affects all LOBs tested (Commercial, Medicare Advantage).

**Environment:** Full Sandbox (FS1). First observed in build INS-BUILD-088.

## Solution

Root cause identified: the Record-Triggered Flow 'Update_Claim_Status_on_EOB_Finalize' was configured to fire on Explanation_of_Benefits__c create/update but the entry condition checked EOB.Claim_Line__r.Claim__r.Id using a cross-object reference that exceeded the 10-level limit in the roll-up logic. Fix: replaced the cross-object Flow reference with an Apex after-insert/after-update trigger on Explanation_of_Benefits__c (EOBStatusRollupTrigger). The trigger calls EOBStatusRollupHandler.evaluateParentClaimStatus(), which queries all sibling Claim_Line__c records for the parent Claim__c, checks that every line has a Finalized EOB, and performs a DML update on Claim__c.Status__c = 'Adjudicated' in a bulkified pattern. Added a custom metadata type (Claim_Status_Rollup_Config__mdt) to control the target status value per LOB without code changes.

## Description

As a Provider Network Specialist, I need the system to automatically initiate member notification letters and create Provider_Change_Notification__c records when a Network_Termination_Case__c is approved so that we meet CMS-mandated 30-day advance member notification requirements and maintain a complete audit trail of all termination-related communications.

Termination cases may be voluntary (provider-initiated) or involuntary (plan-initiated for cause), and the notification workflow must branch accordingly. This story implements the core automation; DocuSign letter generation is handled via the existing Conga CLM integration established in INS-0644.

## Acceptance Criteria

**AC1 — Termination Case Approval Trigger**
Given a Network_Termination_Case__c record has Termination_Status__c changed to 'Approved',
When the Record-Triggered Flow fires,
Then the flow must evaluate Termination_Type__c ('Voluntary' or 'Involuntary') to determine the downstream notification path.

**AC2 — PCN Record Creation**
Given Termination_Status__c = 'Approved',
When the automation runs,
Then a Provider_Change_Notification__c must be created with:
- PCN_Type__c = 'Network Termination'
- PCN_Status__c = 'Open'
- Provider__c = Network_Termination_Case__c.Provider__c
- Effective_Date__c = Network_Termination_Case__c.Termination_Effective_Date__c
- Notification_Required_By__c = TODAY() + 30 (calculated field)
- Source_Case__c = Network_Termination_Case__c.Id

**AC3 — Member Impact Query**
Given the PCN is created,
When the flow queries active Policy__c records linked to the terminating provider via Claim__c history within the last 18 months,
Then a count of impacted members must be stamped on Network_Termination_Case__c.Impacted_Member_Count__c.

**AC4 — Involuntary Termination: Compliance Task**
Given Termination_Type__c = 'Involuntary',
When the automation runs,
Then a Task must be created for the Compliance Officer queue with:
- Subject = 'Compliance Review Required: Involuntary Provider Termination'
- ActivityDate = TODAY() + 5
- Related to Network_Termination_Case__c

**AC5 — Member Letter Generation**
Given Impacted_Member_Count__c > 0,
When the PCN is created,
Then a Platform Event (Provider_Termination_Event__e) must be published with Provider__c Id, Termination_Effective_Date__c, and Termination_Type__c to trigger the downstream Conga CLM letter generation integration.

**AC6 — Voluntary Termination: Provider Acknowledgement**
Given Termination_Type__c = 'Voluntary',
When the automation runs,
Then Network_Termination_Case__c.Provider_Acknowledgement_Status__c must be set to 'Pending DocuSign',
And the DocuSign envelope for provider acknowledgement must be triggered via the existing DocuSign Apex toolkit callout.

**AC7 — Audit Trail**
Given any of the above automation steps complete,
When records are created or updated,
Then a Termination_Audit_Log__c record must be created capturing: Action_Taken__c, Performed_By__c (Automated), Timestamp__c, and Related_Case__c.

-**AC8 — SIU Referral** Given Termination_Reason__c contains 'Fraud' or 'Abuse', auto-create SIU case- (deprecated — SIU referral workflow moved to dedicated SIU automation story in Phase 2)

## Solution

Primary automation via Record-Triggered Flow 'Network_Termination_Approval_Flow' on Network_Termination_Case__c, triggering on Update when Termination_Status__c changes to 'Approved'. Flow branches on Termination_Type__c using Decision element. PCN creation and Task creation handled within flow using Create Records elements. Impacted member count calculated via Apex-invocable action 'CalculateImpactedMembersAction' (called from flow) to handle complex SOQL aggregation without hitting flow SOQL limits. Platform Event Provider_Termination_Event__e published via flow Publish Platform Event element; downstream subscriber flow triggers Conga CLM API callout via External Service registered in Setup. DocuSign envelope trigger for voluntary termination uses existing DocuSign Apex toolkit class 'DocuSignEnvelopeService.sendTerminationAcknowledgement()' called via Invocable Apex. Termination_Audit_Log__c written by after-insert trigger on Provider_Change_Notification__c to ensure log is created regardless of flow path. All automation tested with 200-record bulk scenarios to validate governor limit compliance.

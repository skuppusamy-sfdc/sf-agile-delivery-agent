## Description

As a Claims Operations Manager, I need an Agentforce AI-powered claims triage agent embedded in the Claim__c record page so that the system can automatically analyze incoming claims against fraud indicator rules and historical claim patterns, recommend SIU referral when risk thresholds are met, and suggest an adjudication priority tier — reducing manual triage time and improving SIU referral accuracy.

The Agentforce agent should surface its recommendations as an actionable panel on the Claim__c record, allowing the claims examiner to accept, modify, or dismiss AI recommendations with a documented rationale. All AI recommendations and examiner decisions must be audit-logged for compliance reporting.

## Acceptance Criteria

Given a new Claim__c record is created with Status__c = 'Received',
When the Agentforce Triage Agent is triggered (via Record-Triggered Flow on Claim__c creation),
Then the agent analyzes Claim__c.Claim_Type__c, Claim__c.Total_Billed_Amount__c, Claim__c.Provider__c, Claim__c.Date_of_Service__c, and the associated claimant's prior Claim__c history (last 24 months) and returns a structured triage recommendation within 30 seconds.

Given the Agentforce agent completes analysis,
When the triage recommendation is generated,
Then the recommendation includes: (a) Priority_Tier__c = 'High' | 'Medium' | 'Low', (b) SIU_Referral_Recommended__c = true | false, (c) SIU_Risk_Score__c (0–100), (d) Triage_Rationale__c (natural language explanation, max 500 chars), and these values are written to a new AI_Triage_Recommendation__c child record linked to the Claim__c.

Given AI_Triage_Recommendation__c.SIU_Referral_Recommended__c = true and SIU_Risk_Score__c >= 75,
When the claims examiner views the Claim__c record page,
Then a prominent Agentforce recommendation panel displays the SIU referral suggestion with the Triage_Rationale__c and a 'Refer to SIU' action button that, when clicked, creates a new Case record with RecordType = 'SIU_Investigation', links it to the Claim__c, and sets Claim__c.SIU_Referral_Status__c = 'Referred'.

Given the examiner dismisses the SIU referral recommendation,
When the examiner selects 'Dismiss' and enters a dismissal reason,
Then AI_Triage_Recommendation__c.Examiner_Decision__c = 'Dismissed', AI_Triage_Recommendation__c.Dismissal_Reason__c is populated, and the action is logged to the Claim__c audit trail via a custom Audit_Log__c record with Audit_Log__c.Action__c = 'SIU Referral Dismissed'.

Given the Agentforce agent is unavailable or returns an error,
When the triage trigger fires,
Then Claim__c.AI_Triage_Status__c = 'Manual Review Required' and a Task is assigned to the Claim__c owner with Subject = 'Manual triage required — AI service unavailable'.

## Solution

Configure an Agentforce Agent in Salesforce AI Platform with a custom Agent Topic 'Claims Triage' and Agent Actions mapped to: (1) a custom Apex invocable action 'GetClaimantHistoryAction' that queries prior 24-month Claim__c records for the claimant, (2) a custom Apex invocable action 'CalculateSIURiskScore' implementing a weighted rules engine against configurable Custom Metadata Type 'SIU_Risk_Rule__mdt' records. Define the Agent Prompt Template using the Prompt Builder with Claim__c field grounding. Create AI_Triage_Recommendation__c custom object with all required fields. Build a Record-Triggered Flow 'Claim_Agentforce_Triage_Flow' (After Insert on Claim__c) that calls the Agentforce Agent via the Einstein API invocable action. Build a custom LWC 'agentforceTriagePanel' for the recommendation UI on the Claim__c record page. Implement Audit_Log__c writes via an Apex helper class 'ClaimAuditLogger'. Restrict SIU action button visibility to Permission Set 'Claims_Examiner_SIU_Access'.

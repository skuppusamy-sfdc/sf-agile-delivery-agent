## Description

As an Underwriting Analyst, I need the Fee Schedule Engine to perform effective-date-aware CPT code lookups that respect LOB (Line of Business) and state-based rate tiering on Fee_Schedule__c, so that Claim_Line__c adjudication applies the correct contracted rate for the date of service rather than defaulting to the most recently created fee schedule entry.

Currently the adjudication engine retrieves the first active Fee_Schedule__c record ordered by CreatedDate DESC, which causes incorrect reimbursement amounts when multiple fee schedules overlap for the same CPT code across different effective date windows — a critical finding from UAT cycle 2 regression testing.

Extends the base Fee Schedule configuration work completed in INS-0644.

## Acceptance Criteria

Given a Claim_Line__c record with a populated CPT_Code__c, Date_of_Service__c, LOB__c, and State__c,
When the adjudication engine invokes the FeeScheduleLookupService,
Then the service queries Fee_Schedule__c WHERE CPT_Code__c = [value] AND LOB__c = [value] AND State__c = [value] AND Effective_Start_Date__c <= Date_of_Service__c AND (Effective_End_Date__c >= Date_of_Service__c OR Effective_End_Date__c = null) and returns the single matching record.

Given multiple Fee_Schedule__c records match the CPT/LOB/State criteria for overlapping date ranges,
When the lookup executes,
Then the record with the most recent Effective_Start_Date__c is selected and the others are flagged in a Fee_Schedule_Conflict_Log__c record for Underwriting Analyst review.

Given no matching Fee_Schedule__c record is found for the CPT/LOB/State/date combination,
When the lookup returns null,
Then Claim_Line__c.Adjudication_Status__c = 'Pended — No Fee Schedule Match', Claim_Line__c.Pend_Reason__c = 'Missing CPT Rate', and the parent Claim__c.Status__c is set to 'Pended'.

Given a matching Fee_Schedule__c is found,
When the allowed amount is calculated,
Then Claim_Line__c.Allowed_Amount__c = Fee_Schedule__c.Rate__c * Claim_Line__c.Units__c, and Fee_Schedule__c.Id is stamped on Claim_Line__c.Applied_Fee_Schedule__c for audit traceability.

Given the adjudication engine is processing a batch of Claim_Line__c records,
When bulkified SOQL is used for fee schedule lookups,
Then the implementation must not exceed SOQL governor limits (max 1 SOQL query per unique CPT/LOB/State combination using a Map-based cache pattern).

Given a Fee_Schedule__c record is updated or a new one is inserted,
When the change affects a CPT/LOB/State combination with existing 'Pended — No Fee Schedule Match' Claim_Line__c records,
Then a nightly batch job (FeeScheduleRematchBatch) re-evaluates all pended lines and resubmits matches for adjudication.

-Given CMS Medicare rate tables are imported, the fee schedule lookup should fall back to CMS rates if no contracted rate exists.-
(Deferred to Phase 2 — CMS rate import integration not in Phase 1 scope.)

## Solution

1. Build Apex service class 'FeeScheduleLookupService' with a bulkified lookup method accepting a List<Claim_Line__c> and returning a Map<Id, Fee_Schedule__c> keyed by Claim_Line__c Id.
2. Implement an in-memory Map cache within the service keyed on a composite string (CPT_Code__c + LOB__c + State__c) to deduplicate SOQL calls within a single transaction.
3. SOQL query uses indexed fields (CPT_Code__c, LOB__c, State__c) with date range filter; add a custom compound index on Fee_Schedule__c for (CPT_Code__c, LOB__c, State__c, Effective_Start_Date__c) via a custom index request to Salesforce Support.
4. Conflict detection inserts Fee_Schedule_Conflict_Log__c records via a separate DML operation after the primary lookup.
5. Pend logic sets Claim_Line__c fields via the calling adjudication Apex trigger ('ClaimLine_AdjudicationTrigger') using the service's returned map.
6. Build FeeScheduleRematchBatch as a Database.Batchable class scheduled nightly at 03:00 AM; queries Claim_Line__c WHERE Pend_Reason__c = 'Missing CPT Rate' and re-invokes FeeScheduleLookupService.
7. Add a Record-Triggered Flow on Fee_Schedule__c (after insert/update) to publish a 'FeeScheduleUpdated__e' Platform Event that triggers the rematch batch via a Scheduled Action if pended lines exist for the affected CPT/LOB/State.
8. Ensure all Apex classes have ≥ 90% test coverage with test data covering: exact match, overlap conflict, no match, and bulk (200+ records) scenarios.

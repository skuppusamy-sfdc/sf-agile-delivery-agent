## Description

As a Broker Relations Coordinator, I need a guided OmniScript to onboard new group employer accounts so that I can capture group demographic data, LOB selections, and benefit plan configurations, automatically generate a Group Service Agreement (GSA) via Conga CLM, and route it for e-signature via DocuSign — all within Salesforce without switching between external systems.

This story covers the end-to-end onboarding flow from Account creation through executed GSA. Integration with the Workday HR system for employer census data import is out of scope for Phase 1 and will be addressed in Phase 2.

## Acceptance Criteria

Given a Broker Relations Coordinator launches the 'Group_Employer_Onboarding_OmniScript' from an Account record page,
When Step 1 'Employer Demographics' is completed,
Then a new Account record is created or updated with Account.Type = 'Group Employer', Account.TIN__c, Account.Group_Number__c (auto-generated via custom autonumber), Account.SIC_Code__c, Account.Effective_Date__c, and Account.Assigned_Broker__c populated.

Given the coordinator completes Step 2 'Plan & LOB Selection',
When one or more LOBs are selected (Medical, Dental, Vision, Pharmacy),
Then a Group_Service_Agreement__c record is created in Draft status with GSA__c.Employer_Account__c, GSA__c.LOB_Selections__c (multi-select picklist), GSA__c.Contract_Effective_Date__c, and GSA__c.Renewal_Date__c = Contract_Effective_Date__c + 1 year.

Given the GSA__c record is in Draft status,
When the coordinator clicks 'Generate Contract' in Step 3,
Then the OmniScript calls the Conga CLM Integration Procedure 'IP_Generate_GSA_Document' which merges GSA__c field data into the pre-approved GSA template (Template_ID = 'GSA_STANDARD_V3') and attaches the generated PDF to the GSA__c record as a Salesforce File (ContentDocument).

Given the GSA document is successfully generated,
When the coordinator clicks 'Send for Signature' in Step 4,
Then the DocuSign Integration Procedure 'IP_DocuSign_GSA_Envelope' creates and sends a DocuSign envelope to the employer's primary contact (Account.Primary_Signatory_Email__c) and the internal Broker Relations Coordinator; GSA__c.DocuSign_Envelope_ID__c is populated and GSA__c.Status__c transitions to 'Pending Signature'.

Given the DocuSign envelope is completed by all parties,
When the DocuSign callback webhook fires,
Then GSA__c.Status__c transitions to 'Executed', GSA__c.Executed_Date__c is stamped, and a Platform Event 'GSAExecuted__e' is fired to trigger downstream group enrollment processes.

-Given the coordinator is a TPA user, the Conga template used should be 'GSA_TPA_V1' instead of 'GSA_STANDARD_V3'.- (Requirement removed — TPA onboarding moved to Phase 2 per steering committee decision 2024-11-10.)

Given any step in the OmniScript results in a DML or callout error,
When the error is caught,
Then the error is logged to a custom object 'Integration_Error_Log__c' with fields Error_Source__c = 'Group_Onboarding_OmniScript', Error_Message__c, Record_ID__c, and Timestamp__c, and the coordinator is shown a user-friendly error message with a case reference number.

## Solution

Build 'Group_Employer_Onboarding_OmniScript' in OmniStudio with 4 step groups. Use DataRaptor Post actions for Account and GSA__c record creation. Create Integration Procedure 'IP_Generate_GSA_Document' using an HTTP Action callout to Conga CLM REST API with template ID and merge field payload constructed from DataRaptor Transform. Create Integration Procedure 'IP_DocuSign_GSA_Envelope' using DocuSign for Salesforce managed package eSignature API. Implement a Platform Event 'GSAExecuted__e' published via an Apex trigger on GSA__c when Status__c changes to 'Executed'. Build a Record-Triggered Flow 'GSA_Executed_Enrollment_Trigger_Flow' subscribing to GSAExecuted__e to initiate group enrollment. Create 'Integration_Error_Log__c' custom object with appropriate fields and a generic Apex error logging utility class. Assign OmniScript access via Permission Set Group 'Broker_Relations_Full_Access'.

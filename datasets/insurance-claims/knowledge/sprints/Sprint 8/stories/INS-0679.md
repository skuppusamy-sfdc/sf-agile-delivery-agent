## Description

As a Claims Operations Manager, I need the system to automatically generate an ANSI X12 835 (ERA — Electronic Remittance Advice) file when a Payment__c record is posted against a Claim__c so that providers can receive standardized remittance data for reconciliation, and the file is attached to the Payment__c record and made available for secure provider download.

This story depends on the Fee Schedule Engine logic delivered in INS-0677, as allowed amounts and adjustment reason codes on Claim_Line__c records are inputs to the ERA generation.

## Acceptance Criteria

**AC1 — ERA Generation Trigger**
Given a Payment__c record transitions to Payment_Status__c = 'Posted',
When the Record-Triggered Flow fires,
Then an ERA generation job is enqueued (via Queueable Apex) and the ERA file is generated within 60 seconds for claims with up to 50 Claim_Line__c records.

**AC2 — ERA File Content — Claim Level**
Given ERA generation runs for a posted Payment__c,
When the 835 file is assembled,
Then the CLP segment includes: Claim__c.Claim_Number__c (CLP01), Claim__c.Total_Billed_Amount__c (CLP03), Payment__c.Payment_Amount__c (CLP04), and the patient control number from Claim__c.Patient_Control_Number__c (CLP01 reference).

**AC3 — ERA File Content — Service Line Level**
Given each Claim_Line__c linked to the Claim__c,
When the SVC segment is generated,
Then it includes CPT_Code__c (SVC01-2), Billed_Amount__c (SVC02), Allowed_Amount__c (SVC03), and adjustment reason codes mapped from Claim_Line__c.Adjustment_Reason_Code__c to CARC (Claim Adjustment Reason Code) standard values.

**AC4 — File Attachment to Payment Record**
Given the ERA file is successfully generated,
When generation completes,
Then the file is saved as a ContentVersion record with ContentVersion.Title = 'ERA_[Payment__c.Payment_Number__c]_[YYYYMMDD].835', linked to the Payment__c record via ContentDocumentLink, and Payment__c.ERA_Generated__c is set to true.

**AC5 — Provider Notification**
Given Payment__c.ERA_Generated__c transitions to true,
When the field is updated,
Then an email notification is sent to the provider's billing contact email (Provider__c.Billing_Email__c) with a secure download link via Salesforce Experience Cloud portal.

**AC6 — Error Handling**
Given the ERA generation Queueable job encounters a missing required field (e.g., null CPT_Code__c on any Claim_Line__c),
When the job executes,
Then the error is logged to Error_Log__c, Payment__c.ERA_Generation_Status__c is set to 'Failed', and a Case is created and assigned to the Claims Operations queue for manual remediation.

## Solution

1. Create Apex Queueable class 'ERAGenerationQueueable' implementing the ANSI X12 835 file structure using a String builder pattern organized around ISA/GS/ST/BPR/CLP/SVC segment classes.
2. Record-Triggered Flow 'Payment_Posted_ERA_Enqueue_Flow' on Payment__c (After Save, Status__c changed to 'Posted') enqueues ERAGenerationQueueable via System.enqueueJob().
3. Segment builder classes: ISASegmentBuilder, CLPSegmentBuilder, SVCSegmentBuilder — each encapsulating the relevant Salesforce field mappings to EDI positions.
4. CARC mapping implemented as a Custom Metadata Type 'CARC_Mapping__mdt' with fields Adjustment_Reason_Code__c (Salesforce value) and CARC_Code__c (EDI standard value).
5. ContentVersion creation and ContentDocumentLink insertion after successful file string assembly.
6. Experience Cloud file sharing configured via ContentDistribution for provider portal download link generation.
7. Apex test class 'ERAGenerationQueueable_Test' with mock Payment__c + Claim__c + Claim_Line__c data, asserting ContentVersion creation and ERA_Generated__c = true.

## Description

As a Broker Relations Coordinator, I need a guided OmniScript wizard that walks me through creating a Group_Service_Agreement__c (GSA) and binding associated Policy__c records for a new employer group so that the onboarding process is standardized, reduces data entry errors, and ensures all required compliance fields are captured before the group's effective date.

The wizard must support multi-step data capture: (1) Employer/Group details, (2) LOB and plan selection, (3) GSA terms and effective dates, (4) DocuSign e-signature routing, (5) Policy__c binding confirmation.

The Broker Relations Coordinator initiates the wizard from the Account record of the employer group.

## Acceptance Criteria

Given a Broker Relations Coordinator opens the 'New Group Onboarding' OmniScript from an employer Account record,
When Step 1 loads,
Then the wizard pre-populates Group_Name__c, Group_TIN__c, Primary_Contact__c, and SIC_Code__c from the Account record using a DataRaptor Extract.

Given the coordinator reaches Step 2 (LOB/Plan Selection),
When they select LOB__c (e.g., 'Commercial — PPO'),
Then the available Plan_Code__c picklist dynamically filters to plans where Plan__c.LOB__c = selected LOB AND Plan__c.Status__c = 'Active' AND Plan__c.Effective_Date__c <= GSA proposed start date.

Given Step 3 (GSA Terms) is completed with all required fields,
When the coordinator clicks 'Proceed to Signature',
Then a Group_Service_Agreement__c record is created in Draft status with GSA_Status__c = 'Pending Signature', and a DocuSign envelope is sent to the employer's Primary_Contact__c email via the DocuSign integration.

Given the DocuSign envelope is completed by all signatories,
When the DocuSign status callback updates GSA_Status__c = 'Executed',
Then the OmniScript Step 5 confirmation screen displays a summary and triggers a DataRaptor Turbo Action to create one Policy__c record per selected plan with Policy_Status__c = 'Active', Effective_Date__c = GSA.Effective_Date__c, and Group_Account__c = employer Account Id.

Given a required field (e.g., Group_TIN__c) is missing in Step 1,
When the coordinator attempts to advance to Step 2,
Then the OmniScript displays an inline validation message: 'Group TIN is required before proceeding' and navigation is blocked.

-Given coordinator can skip DocuSign and manually mark GSA as Executed- (deprecated — manual execution bypass removed per compliance requirement; all GSAs must route through DocuSign).

Given the entire wizard is completed successfully,
When the final step is submitted,
Then a confirmation email is sent to the Broker Relations Coordinator and the employer Primary_Contact__c using an Org-Wide Email Address 'noreply@acmeinsurance.com', and a Task is created with Subject = 'GSA Onboarding Complete — Welcome Call' assigned to the coordinator.

## Solution

Build multi-step OmniScript 'Group_Employer_Onboarding_OS' with 5 steps using OmniStudio Designer. Step 1 uses DataRaptor Extract 'DR_Extract_AccountForGSA' to pre-populate fields. Step 2 plan filter uses DataRaptor Turbo Action with dynamic WHERE clause. GSA record creation in Step 3 via Integration Procedure 'IP_CreateGSA_SendDocuSign' which calls DataRaptor Transform + DocuSign OmniStudio connector action. DocuSign callback handled by Platform Event 'DocuSign_Status_Update__e' consumed by Record-Triggered Flow on Group_Service_Agreement__c to update GSA_Status__c and invoke Policy creation sub-flow. Policy__c bulk insert via Apex invocable 'PolicyBindingService.bindGroupPolicies()' called from Integration Procedure. OmniScript embedded as LWC on Account Lightning Record Page via dynamic action visibility rule. Permission Set Group 'Broker_Relations_PSG' grants access to OmniScript and all GSA/Policy fields.

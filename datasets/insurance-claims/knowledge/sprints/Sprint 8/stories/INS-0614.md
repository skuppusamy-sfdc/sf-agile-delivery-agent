## Description

**Bug Report — UAT Round 2**

During UAT, Claims Operations discovered that when a secondary payer processes a COB payment against an existing Claim__c, the associated Claim_Line__c records are not reflecting the updated adjudication status on the generated Explanation_of_Benefits__c (EOB).

**Steps to Reproduce:**
1. Create a Claim__c with LOB = 'Medical' and at least 3 Claim_Line__c records.
2. Set Claim__c.COB_Indicator__c = TRUE and populate Primary_Payer__c with an external payer TIN.
3. Post a primary Payment__c record linked to the Claim__c with Payment__c.Payment_Type__c = 'Primary'.
4. Verify Claim_Line__c.Adjudication_Status__c updates to 'Primary Adjudicated'.
5. Post a secondary Payment__c with Payment__c.Payment_Type__c = 'Secondary COB'.
6. Navigate to the auto-generated Explanation_of_Benefits__c record.

**Expected:** EOB__c.Line_Status__c on each child EOB line reflects 'Secondary Adjudicated' and EOB__c.Patient_Responsibility__c is recalculated.

**Actual:** EOB__c.Line_Status__c remains 'Primary Adjudicated'; EOB__c.Patient_Responsibility__c is not updated. The Claim_Line__c records themselves update correctly — the issue is isolated to the EOB generation step.

**Environment:** Acme-UAT sandbox. Reproducible 100% of the time with 2+ Claim_Line__c records. Single-line claims appear unaffected.

## Solution

Root cause identified in the Record-Triggered Flow 'RTF_EOB_LineSync' — the entry condition was filtering on Payment__c.Payment_Type__c = 'Primary' only, excluding Secondary COB payments from triggering the EOB line update logic. Fix: updated flow entry criteria to include Payment__c.Payment_Type__c IN ('Primary', 'Secondary COB', 'Tertiary COB'). Added a secondary decision element to branch COB-specific recalculation of EOB__c.Patient_Responsibility__c using a formula resource referencing Claim_Line__c.Allowed_Amount__c minus Payment__c.Amount_Paid__c. Deployed fix to UAT; regression suite passed across 47 existing COB test cases. No Apex changes required.

## Description

As a Claims Operations Manager, I need the Fee Schedule Engine to support multi-tier CPT code rate lookups when a Claim__c is associated with a Group_Service_Agreement__c (GSA) client so that employer group clients with negotiated tiered rates receive accurate adjudication amounts at the Claim_Line__c level, preventing overpayment and underpayment exceptions that currently require manual correction.

Background: GSA clients can have up to 3 fee tiers (Tier 1 = Preferred, Tier 2 = Standard, Tier 3 = Out-of-Network Fallback) each mapped to a separate Fee_Schedule__c record. The lookup priority must follow: GSA Tier 1 → GSA Tier 2 → GSA Tier 3 → Default Network Fee Schedule.

## Acceptance Criteria

Given a Claim__c.Group_Agreement__c is populated with a GSA Id,
When the Fee Schedule Engine processes a Claim_Line__c with a valid CPT_Code__c,
Then the engine must first query Fee_Schedule__c records WHERE Fee_Schedule__c.GSA__c = Claim__c.Group_Agreement__c AND Fee_Schedule__c.Tier__c = 'Tier 1' AND the CPT code falls within the schedule's CPT_Range_Start__c to CPT_Range_End__c.

Given a Tier 1 Fee_Schedule__c match is found for the CPT code,
When the allowed amount is calculated,
Then Claim_Line__c.Allowed_Amount__c must be set to Fee_Schedule__c.Allowed_Amount__c and Claim_Line__c.Rate_Source__c = 'GSA-Tier1'.

Given no Tier 1 match exists for the CPT code,
When the engine cascades to Tier 2,
Then Claim_Line__c.Allowed_Amount__c is set from the Tier 2 Fee_Schedule__c match and Claim_Line__c.Rate_Source__c = 'GSA-Tier2'.

Given no Tier 1 or Tier 2 match exists,
When the engine cascades to Tier 3,
Then Claim_Line__c.Allowed_Amount__c is set from Tier 3 and Claim_Line__c.Rate_Source__c = 'GSA-Tier3'.

Given no GSA tier match exists at any level,
When the fallback executes,
Then the engine must apply the default network Fee_Schedule__c (Fee_Schedule__c.Is_Default_Network__c = true) and set Claim_Line__c.Rate_Source__c = 'DefaultNetwork'.

Given a CPT code that is explicitly excluded in the GSA (GSA.Excluded_CPT_Codes__c contains the CPT value),
When the lookup runs,
Then Claim_Line__c.Adjudication_Status__c must be set to 'Excluded - GSA' and no Allowed_Amount__c should be calculated.

Given a bulk adjudication batch of 500+ Claim_Line__c records,
When the Fee Schedule Engine runs,
Then all lookups must complete within governor limits with no SOQL query exceptions, validated by a batch test class.

-Given the GSA has a carve-out for behavioral health CPT codes (CPT 90000–90999), apply a separate BH Fee Schedule.- (Deferred — behavioral health carve-out logic moved to INS-0698.)

## Solution

Implement Apex class 'FeeScheduleResolutionService' with method 'resolveGSATieredRate(List<Claim_Line__c> lines)'. The method: (1) Bulkifies all unique GSA Ids and CPT codes from the input list. (2) Executes a single SOQL query fetching all relevant Fee_Schedule__c records with a WHERE clause on GSA__c IN :gsaIds AND Tier__c IN ('Tier 1','Tier 2','Tier 3','Default'). (3) Builds an in-memory map keyed by GSA Id + Tier + CPT range for O(1) lookups per line. (4) Applies tier waterfall logic in a for-loop, populating Allowed_Amount__c and Rate_Source__c. (5) Performs a single DML update on Claim_Line__c list. Invoke this service from the existing Apex Trigger 'ClaimLineTrigger' on after insert/update via the TriggerHandler pattern. Add an Invocable Method wrapper for Flow compatibility. Target 95%+ test coverage with @isTest data factories covering all tier cascade scenarios and the excluded-CPT path.

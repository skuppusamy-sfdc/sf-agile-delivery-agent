## Description

As a Claims Operations Manager, I need the system to automatically resolve the applicable Fee_Schedule__c line item when a Claim_Line__c enters adjudication so that allowed amounts are calculated consistently without manual intervention, reducing adjudication cycle time and eliminating pricing errors.

The fee schedule lookup must consider the provider's active Provider_Network_Agreement__c, the submitted CPT code on the Claim_Line__c, and the date of service to resolve the correct fee schedule version.

## Acceptance Criteria

**AC1 — Successful Fee Schedule Resolution**
Given a Claim_Line__c record has Adjudication_Status__c changed to 'In Adjudication',
When the provider linked via Claim__c.Provider__c has an active PNA (PNA_Status__c = 'Active') with an associated Fee_Schedule__c,
And the Claim_Line__c.CPT_Code__c matches a Fee_Schedule_Line__c.CPT_Code__c within the fee schedule,
And the Claim_Line__c.Date_of_Service__c falls within Fee_Schedule__c.Effective_Date__c and Fee_Schedule__c.Expiration_Date__c,
Then Claim_Line__c.Allowed_Amount__c must be populated with Fee_Schedule_Line__c.Allowed_Amount__c,
And Claim_Line__c.Fee_Schedule__c (lookup) must be stamped with the resolved Fee_Schedule__c record ID.

**AC2 — No Matching CPT Code**
Given the CPT code on Claim_Line__c does not match any active Fee_Schedule_Line__c,
When adjudication lookup runs,
Then Claim_Line__c.Adjudication_Status__c must be set to 'Pended — Fee Schedule Exception',
And a Task must be created assigned to the Claims Operations queue with Subject = 'Fee Schedule Exception: CPT Not Found' referencing the Claim_Line__c.

**AC3 — Expired Fee Schedule**
Given the provider's PNA has an associated Fee_Schedule__c but the Date_of_Service__c falls outside the schedule's effective date range,
When lookup runs,
Then Claim_Line__c.Adjudication_Status__c = 'Pended — Expired Fee Schedule',
And Claim__c.Pend_Reason__c must be updated to 'Fee Schedule Expired'.

**AC4 — Out-of-Network Provider**
Given the provider does NOT have an active PNA linked to the claim's LOB__c,
When adjudication lookup runs,
Then Claim_Line__c.Network_Status__c must be set to 'Out-of-Network',
And Claim_Line__c.Allowed_Amount__c must be calculated using the default OON Fee_Schedule__c identified by Fee_Schedule__c.Is_Default_OON__c = true.

**AC5 — COB Scenario**
Given Claim__c.COB_Flag__c = true,
When fee schedule resolution completes,
Then Claim_Line__c.COB_Adjusted_Allowed_Amount__c must be calculated as Allowed_Amount__c minus Claim_Line__c.Primary_Payer_Paid_Amount__c,
And COB_Adjusted_Allowed_Amount__c must not be negative (floor at 0.00).

-**AC6 — GSA Override** Given a Group_Service_Agreement__c exists for the policyholder's employer group with GSA_Override_Fee_Schedule__c populated, the GSA fee schedule takes precedence over PNA fee schedule- (deprecated — GSA override logic moved to Phase 2 per INS-0659)

## Solution

Implement via Apex Trigger (ClaimLineAdjudicationTrigger) on Claim_Line__c, firing on Update when Adjudication_Status__c changes to 'In Adjudication'. Trigger invokes FeeScheduleResolutionService Apex class which: (1) queries Provider_Network_Agreement__c for active PNA matching provider + LOB; (2) queries Fee_Schedule_Line__c for CPT code match within effective date range using indexed fields (CPT_Code__c, Effective_Date__c); (3) applies resolution logic per ACs above; (4) uses Database.update with allOrNone=false to allow partial success logging. OON default fee schedule retrieved via Custom Metadata Type 'Fee_Schedule_Config__mdt' to avoid hardcoded IDs. Task creation for exceptions handled via separate invocable Apex called from Record-Triggered Flow to respect governor limits. Unit test coverage target: 90%+. Custom exception class FeeScheduleResolutionException for structured error logging to Integration_Error_Log__c.

## Description

As a Claims Operations Manager, I need an Agentforce AI assistant embedded on the Claim__c record page that can triage incoming FNOL submissions, recommend claim priority, and flag potential duplicate claims so that my team can process high volumes of new claims faster and reduce duplicate payment exposure.

This is a Phase 1 capability targeting the Commercial LOB only. Medicare Advantage and TPA-managed claims are out of scope for this sprint.

## Acceptance Criteria

Given a Claims Operations Manager opens a newly created Claim__c record with Claim__c.Status__c = 'FNOL Received',
When the Agentforce Claims AI panel loads,
Then the assistant must display an AI-generated triage summary including:
  - Recommended Claim__c.Priority__c value (Critical / High / Medium / Low) with a confidence score
  - Extracted ICD-10 codes from Claim__c.Diagnosis_Description__c with mapped Claim_Line__c.ICD10_Code__c suggestions
  - Estimated claim complexity (Simple / Moderate / Complex) based on number of Claim_Line__c records and total billed amount (Claim__c.Total_Billed_Amount__c)

Given the AI assistant has generated a triage summary,
When the Claims Analyst clicks 'Accept AI Recommendation',
Then Claim__c.Priority__c must update to the recommended value and Claim__c.AI_Triage_Accepted__c (checkbox) must be set to TRUE with Claim__c.AI_Triage_Timestamp__c = NOW().

Given the AI assistant scans for duplicate claims,
When a potential duplicate is identified (same Claim__c.Policy__c, same Claim__c.Date_of_Service__c, same primary CPT code on Claim_Line__c, submitted within 30 days),
Then the assistant must display a 'Potential Duplicate' alert listing the matching Claim__c.Name values with a direct link and a similarity confidence percentage.

Given a duplicate alert is displayed,
When the analyst clicks 'Mark as Duplicate',
Then Claim__c.Status__c must update to 'Duplicate — Pending Review' and a Task must be created for the SIU queue with the duplicate claim IDs referenced in the Task description.

Given the Agentforce assistant is active,
When the running user does not have the 'Agentforce_Claims_User' Permission Set assigned,
Then the AI panel must not render and must display a graceful 'Feature not available — contact your administrator' message.

-Given Medicare Advantage claims, the AI assistant should also check CMS coverage rules via API- *(deferred to Phase 2)*

## Solution

Implemented using Salesforce Agentforce platform with a custom Agent Topic ('Claims FNOL Triage') and two Agent Actions: (1) ClaimTriageAction — an Apex-backed Agent Action that calls a custom LLM prompt template (stored in Prompt Builder) using the Claim__c record context, returning structured JSON with priority recommendation and ICD-10 suggestions; (2) DuplicateClaimScanAction — an Apex-backed Agent Action executing a SOQL query with composite matching logic across Policy__c, Date_of_Service__c, and Claim_Line__c CPT codes, returning candidate duplicate records with a Jaro-Winkler similarity score computed in Apex. The Agentforce panel is embedded via an LWC (agentforceClaimsPanel) on the Claim__c Lightning Record Page. Permission gating uses a custom Permission Set 'Agentforce_Claims_User' checked in the LWC connectedCallback. All AI interaction logs are written to a custom object AI_Interaction_Log__c for auditability.

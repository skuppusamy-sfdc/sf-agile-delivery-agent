## Description

As a Claims Operations Manager, I need a Policyholder 360 summary panel on the insured's Account record page that displays their active policy coverage details, a count of open claims, and the three most recent Explanation_of_Benefits__c (EOB) records so that I can quickly assess a policyholder's overall claims and coverage status during inbound service interactions without navigating to multiple objects.

## Acceptance Criteria

Given an Account record with Account.RecordType = 'Policyholder' is open in Lightning Experience,
When the Policyholder 360 LWC component loads,
Then the component displays the active Policy__c record where Policy__c.Account__c = Account.Id AND Policy__c.Status__c = 'Active', showing: Policy__c.Policy_Number__c, Policy__c.Product_Type__c, Policy__c.Coverage_Effective_Date__c, Policy__c.Coverage_Termination_Date__c, and Policy__c.Deductible_Remaining__c.

Given the policyholder has open Claim__c records,
When the 'Active Claims' tile renders,
Then the tile shows a count of Claim__c records where Claim__c.Insured_Account__c = Account.Id AND Claim__c.Status__c NOT IN ('Closed', 'Denied', 'Withdrawn'), with a hyperlink navigating to a filtered Claim__c list view.

Given the policyholder has Explanation_of_Benefits__c records,
When the 'Recent EOBs' section renders,
Then the three most recent EOB records (ordered by EOB__c.EOB_Date__c DESC) are displayed showing: EOB__c.EOB_Number__c, EOB__c.Total_Billed__c, EOB__c.Total_Paid__c, EOB__c.Patient_Responsibility__c, and a download link for the EOB PDF stored as a Salesforce File.

Given the policyholder has no active Policy__c record,
When the coverage section renders,
Then the message 'No active policy found. Please verify enrollment status.' is displayed in an amber alert box.

Given the component is accessed by a user without the 'View_PHI_Data' custom permission,
When the component attempts to load Policy__c and EOB data,
Then sensitive fields (Policy__c.Deductible_Remaining__c, EOB__c.Patient_Responsibility__c) are masked as '***' and a tooltip reads 'You do not have permission to view this information.'

## Solution

Build as a custom LWC component 'c-policyholder-360-panel' using @wire(getRecord) for the Account record type check and two @wire(apex) calls to Apex controller methods: 'getPolicyholderActivePolicies()' and 'getRecentEOBs()' both with proper FLS-respecting SOQL using Security.stripInaccessible(). Custom permission 'View_PHI_Data' checked in Apex via FeatureManagement.checkPermission() to conditionally mask fields in the returned wrapper class. Active claims count retrieved via an Apex aggregate SOQL query returning a COUNT result. EOB PDF download link constructed using the ContentDocumentLink junction to retrieve the latest File version. Component added to the Policyholder Account Lightning Record Page via App Builder with visibility filter: Account.RecordType.DeveloperName = 'Policyholder'. Included in the Policyholder_Service_Agent Permission Set Group.

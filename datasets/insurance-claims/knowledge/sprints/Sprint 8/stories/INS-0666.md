## Description

**Bug Report — UAT Sprint 8**

During UAT cycle for Phase 1 EOB batch processing, Claims Operations identified that Claim__c.Status__c is not rolling up to 'Adjudicated' after the nightly EOB batch job completes successfully for all associated Claim_Line__c records.

**Steps to Reproduce:**
1. Create a Claim__c record with Status__c = 'In Review' and at least 3 Claim_Line__c child records.
2. Manually trigger the EOB batch class (EOBGenerationBatch) via anonymous Apex or wait for nightly schedule.
3. Verify that all Claim_Line__c records have Adjudication_Status__c = 'Approved'.
4. Navigate back to the parent Claim__c record.
5. Observe that Claim__c.Status__c remains 'In Review' instead of updating to 'Adjudicated'.

**Expected Behavior:** When all child Claim_Line__c records have Adjudication_Status__c = 'Approved', the parent Claim__c.Status__c should automatically update to 'Adjudicated'.

**Actual Behavior:** Claim__c.Status__c remains 'In Review'. The Explanation_of_Benefits__c record is created correctly, but the parent status rollup does not fire.

**Environment:** Acme UAT Org (Partial Sandbox)
**Frequency:** 100% reproducible
**Impact:** Claims Operations Manager cannot identify adjudicated claims in worklist views; Payment__c records cannot be initiated downstream.

## Solution

Root cause identified: The Record-Triggered Flow 'Claim_Status_Rollup_Flow' on Claim_Line__c has a faulty entry condition — it only fires on record Create, not on Update. The flow must be amended to trigger on both Create and Update when Adjudication_Status__c changes. Additionally, a rollup summary or Apex-based count check must confirm all sibling Claim_Line__c records are 'Approved' before setting Claim__c.Status__c = 'Adjudicated'. Fix involves: (1) Update Flow trigger to run on Create and Update with condition Adjudication_Status__c IS CHANGED; (2) Add a Get Records element to count non-approved sibling lines; (3) Add decision node — if count = 0, update parent via related record update element. Deploy via Change Set to UAT for re-validation.

## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can view a provider's credentialing status, active Provider Network Agreements, open claims, and any pending Provider Change Notifications in a single interface without navigating to multiple related lists.

Extends INS-0614 (Provider Data Maintenance base layout).

## Acceptance Criteria

Given a Provider__c record is open in Lightning Experience,
When the Provider 360 FlexCard loads,
Then the card displays the provider's NPI, TIN, and Credentialing_Case__c.Status__c from the most recent active Credentialing_Case__c child record.

Given the provider has one or more Provider_Network_Agreement__c records,
When the FlexCard renders the 'Active Agreements' section,
Then only PNA records where PNA.Status__c = 'Active' AND PNA.Effective_End_Date__c >= TODAY() are displayed, showing PNA.Network_Name__c, PNA.Fee_Schedule__c lookup name, and PNA.Reimbursement_Model__c.

Given the provider has associated Claim__c records,
When the 'Open Claims' tile renders,
Then a count badge shows the number of Claim__c records where Claim__c.Billing_Provider__c = Provider__c.Id AND Claim__c.Status__c IN ('Submitted', 'In Review', 'Pending COB').

Given a Provider_Change_Notification__c (PCN) record exists with PCN.Status__c = 'Pending Review',
When the FlexCard header banner evaluates PCN data,
Then a yellow warning banner displays: 'Pending Provider Change Notification — Action Required' with a deep link to the PCN record.

Given the provider's CAQH profile was last verified more than 90 days ago (Provider__c.CAQH_Last_Verified_Date__c < TODAY() - 90),
When the credentialing section renders,
Then a red indicator icon and text 'CAQH Re-attestation Overdue' is displayed.

-Given the provider is affiliated with a Facility__c, display the Facility's GSA contract tier inline on the FlexCard.- (Deprecated — moved to Facility 360 card, see INS-0638)

Given any DataRaptor fetch fails due to a timeout,
When the FlexCard error state activates,
Then a user-friendly message 'Unable to load provider data. Please refresh or contact IT support.' is shown without exposing technical error details.

## Solution

Build using OmniStudio FlexCard with three child cards (Credentialing, Agreements, Claims). Each child card backed by a dedicated DataRaptor Extract pulling from Provider__c, Credentialing_Case__c, Provider_Network_Agreement__c, and Claim__c. Use a DataRaptor Turbo Extract for the claims count aggregate to avoid SOQL limits. PCN warning banner implemented as a conditional FlexCard state driven by a DataRaptor that queries PCN records with Status__c = 'Pending Review'. CAQH date comparison handled via a formula field Provider__c.CAQH_Attestation_Overdue__c (Boolean) referenced in FlexCard conditional visibility. Assign the FlexCard to the Provider Relations Specialist Permission Set Group. Deploy via OmniStudio migration tool to UAT.

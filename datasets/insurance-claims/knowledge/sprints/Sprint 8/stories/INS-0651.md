## Description

**Bug Report — UAT Sprint 8**

During UAT regression testing of the ERA inbound processing pipeline, the parent Claim__c record's Status__c field is not rolling up correctly after all child Claim_Line__c records reach 'Adjudicated' status via the Electronic Remittance Advice batch job.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record in 'Submitted' status with 3 or more Claim_Line__c child records.
2. Trigger the ERA inbound batch job (InboundERABatchProcessor) via Developer Console or scheduled run.
3. Confirm all Claim_Line__c records have Status__c = 'Adjudicated' and Adjudicated_Amount__c is populated.
4. Navigate back to the parent Claim__c record.
5. Observe that Claim__c.Status__c remains 'Submitted' instead of transitioning to 'Adjudicated'.

**Expected Behavior:**
When all Claim_Line__c records associated with a Claim__c have Status__c = 'Adjudicated', the parent Claim__c.Status__c should automatically update to 'Adjudicated' and Claim__c.Adjudication_Completed_Date__c should be stamped with the current datetime.

**Actual Behavior:**
Claim__c.Status__c remains 'Submitted'. Claim__c.Adjudication_Completed_Date__c is null. No Platform Event is fired to trigger downstream EOB generation.

**Root Cause (suspected):**
The Record-Triggered Flow 'Claim_Line_Status_Rollup_Flow' has a faulty entry condition — it checks for ISCHANGED(Status__c) but the batch job updates are performed via Database.update() in a context where the prior value comparison is not resolving correctly, likely due to a bulk update governor limit bypass introduced in INS-0621.

as per Description

## Solution

Investigate the 'Claim_Line_Status_Rollup_Flow' Record-Triggered Flow entry conditions. Replace the ISCHANGED() check with an explicit prior value formula field (Claim_Line_Prior_Status__c) populated via an Apex before-update trigger to ensure reliable change detection in bulk DML contexts. Add a compensating Apex Queueable job (ClaimStatusRollupQueueable) that re-evaluates parent Claim__c status post-ERA batch completion by querying all child Claim_Line__c.Status__c values. Fire a ClaimAdjudicated__e Platform Event upon confirmed rollup to trigger the EOB generation flow. Add unit tests covering bulk scenarios (200+ Claim_Line__c records per Claim__c). Deploy fix to UAT sandbox for regression sign-off.

## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can instantly view a provider's active network agreements, current credentialing status, and open claims activity without navigating across multiple objects.

This story extends INS-0578 (Provider Record Page Redesign) by adding the dynamic data panels for network and claims context.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the card must display the following sections without requiring additional navigation:

**Section 1 – Provider Identity**
Given the FlexCard loads,
When Provider__c.NPI__c and Provider__c.TIN__c are populated,
Then both values display in the header band with a CMS NPPES verification badge (green checkmark if Provider__c.NPPES_Verified__c = true, red warning if false).

**Section 2 – Active Network Agreements**
Given the provider has one or more Provider_Network_Agreement__c records,
When PNA.Status__c = 'Active' AND PNA.Effective_Date__c <= TODAY() AND PNA.Termination_Date__c >= TODAY(),
Then each active PNA displays: PNA.Network_Name__c, PNA.LOB__c, PNA.Effective_Date__c, and PNA.Fee_Schedule__r.Name.
-When PNA count exceeds 5, display all agreements in a scrollable inline table-
When PNA count exceeds 5, display the first 3 with a 'View All (#)' expand action.
Given no active PNAs exist,
Then display a yellow alert banner: 'No Active Network Agreements — Provider may be out-of-network.'

**Section 3 – Credentialing Status**
Given a Credentialing_Case__c exists linked to the provider,
When Credentialing_Case__c.Status__c is any value,
Then display the most recent Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and a color-coded badge (Green = 'Credentialed', Yellow = 'Pending', Red = 'Expired' or 'Revoked').
Given Credentialing_Case__c.Expiration_Date__c is within 90 days of TODAY(),
Then display an inline warning: 'Credentialing expires in X days — recredentialing required.'

**Section 4 – Open Claims Summary**
Given one or more Claim__c records exist where Claim__c.Rendering_Provider__c = Provider__c.Id AND Claim__c.Status__c NOT IN ('Closed', 'Denied', 'Void'),
When the FlexCard renders,
Then display aggregate counts: Total Open Claims, Total Pending Payment Amount (sum of Claim__c.Total_Allowed_Amount__c), and oldest open Claim__c.Date_of_Service__c.

**Section 5 – Quick Actions**
Given the user has the 'Provider_Relations_User' Permission Set Group,
When viewing the FlexCard,
Then the following quick actions are available: 'Initiate Recredentialing', 'Create PCN', 'View Full Claims History'.

## Solution

1. Build OmniStudio FlexCard 'Provider360_FC' on the Provider__c object using Card Type = Standard.
2. Create four DataRaptor Extract DRs to feed each section: DR_GetActivePNAs (joins PNA + Fee_Schedule__c), DR_GetCredentialingStatus (latest Credentialing_Case__c by CreatedDate DESC), DR_GetOpenClaimsSummary (aggregate SOQL on Claim__c), DR_GetProviderIdentity (Provider__c + NPPES_Verified__c).
3. Use FlexCard Child Cards for Sections 2 and 4 to handle list rendering with conditional visibility states.
4. Implement conditional styling via FlexCard State Actions: map Credentialing_Case__c.Status__c values to CSS badge classes (ins-badge-green, ins-badge-yellow, ins-badge-red).
5. Wire 'View All PNAs' action to a navigation action opening a filtered Related List.
6. Implement Quick Action buttons using OmniScript launch actions for 'Initiate Recredentialing' (launches OS_RecredentialingInitiation) and 'Create PCN' (launches OS_CreatePCN).
7. Gate FlexCard visibility using Permission Set Group 'Provider_Relations_User' via FlexCard conditional rendering on UserInfo.
8. Add to Provider__c Lightning Record Page via Dynamic Form, positioned in the primary column.

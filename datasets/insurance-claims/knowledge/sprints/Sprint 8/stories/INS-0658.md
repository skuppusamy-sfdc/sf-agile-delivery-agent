## Description

**Bug Report — UAT Sprint 8**

During ERA ingestion testing, the parent Claim__c.Status__c field is not rolling up to 'Adjudicated' after all child Claim_Line__c records are marked as 'Paid'. The Claim remains stuck in 'Pending Payment' indefinitely.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record in status 'Pending Payment' with 3 associated Claim_Line__c records.
2. Trigger ERA file ingestion via the DataIntegration batch job (ERAIngestBatch).
3. Confirm all 3 Claim_Line__c.Status__c fields update to 'Paid' via the ERA payload.
4. Observe Claim__c.Status__c — it remains 'Pending Payment'.
5. Manually refresh the record; status does not change.

**Expected:** Claim__c.Status__c transitions to 'Adjudicated' once all Claim_Line__c children have Status__c = 'Paid'.
**Actual:** Claim__c.Status__c remains 'Pending Payment'. No Platform Event is fired to trigger the rollup Flow.

**Root Cause (suspected):** The Record-Triggered Flow on Claim_Line__c that publishes the ClaimLineStatusUpdated__e Platform Event has a filter condition checking `Status__c = 'Approved'` instead of `Status__c = 'Paid'`, introduced in the INS-0631 hotfix.

## Acceptance Criteria

as per Description

## Solution

1. Update the Record-Triggered Flow 'ClaimLine_StatusChange_Flow' entry condition from `Status__c = 'Approved'` to `Status__c = 'Paid'` to correctly fire the ClaimLineStatusUpdated__e Platform Event.
2. Update the Platform Event–triggered Flow 'Claim_AdjudicationRollup_Flow' to query all sibling Claim_Line__c records and verify all have Status__c = 'Paid' before setting Claim__c.Status__c = 'Adjudicated'.
3. Add a null-safe check for Claim_Line__c.Claim__c lookup to prevent orphaned lines from breaking the rollup.
4. Deploy fix to UAT sandbox and execute ERAIngestBatch regression test suite.
5. Add unit test coverage for the Platform Event publish path (target ≥ 90% coverage on updated classes).

## Description

**Bug Report — UAT Sprint 8**

When a Claim__c record has Coordination_of_Benefits__c = true and a secondary payer payment is applied via the COB workflow, the associated Explanation_of_Benefits__c (EOB) record fails to reflect the updated Claim_Line__c.Adjudication_Status__c values. The EOB continues to show 'Pending' at the line level even after the claim has been fully adjudicated.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record with COB_Indicator__c = true.
2. Apply a primary payer payment via the Payment__c object (Payment_Type__c = 'Primary').
3. Trigger the COB secondary payer calculation via the 'Calculate COB' quick action.
4. Confirm secondary Payment__c record is created (Payment_Type__c = 'Secondary').
5. Open the related Explanation_of_Benefits__c record.
6. Observe Claim_Line__c.Adjudication_Status__c on EOB line items.

**Expected:** All Claim_Line__c records related to the Claim__c should display Adjudication_Status__c = 'Adjudicated' on the EOB after secondary payer processing completes.

**Actual:** Claim_Line__c.Adjudication_Status__c remains 'Pending' on the EOB record. The parent Claim__c.Status__c correctly updates to 'Adjudicated' but the EOB line-level sync does not fire.

**Environment:** UAT Sandbox (Acme-UAT1)
**Frequency:** 100% reproducible
**Impact:** Claims Operations Manager cannot issue ERA files; EOB PDF generation via Conga CLM is producing incorrect member-facing documents.

## Solution

Root cause identified: the Record-Triggered Flow 'Update_EOB_Lines_on_COB_Completion' has a fault path that exits silently when Payment__c.Payment_Type__c = 'Secondary' and the parent Claim__c has more than 10 Claim_Line__c child records (bulk governor limit on Get Records element). Fix approach: (1) Replace the Get Records element inside the flow with an Apex-invocable method 'EOBLineStatusSyncService.syncLinesFromClaim(List<Id> claimIds)' that handles bulk retrieval with proper SOQL pagination. (2) The Apex method will query all Claim_Line__c WHERE Claim__c IN :claimIds and update Adjudication_Status__c = 'Adjudicated' WHERE all related Payment__c records are in status 'Posted'. (3) Update the Platform Event 'EOB_Refresh__e' subscriber to re-render the FlexCard on the EOB record page after sync. (4) Add a unit test covering claims with 15+ line items and COB = true, targeting 95% code coverage.

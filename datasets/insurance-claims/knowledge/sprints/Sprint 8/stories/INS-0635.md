## Description

As a Claims Operations Manager, I need an Agentforce-powered triage recommendation to be generated automatically during FNOL intake on a new Claim__c record so that claims are routed to the correct handling queue (Standard, Complex, SIU Referral) based on AI analysis of claim details, reducing manual triage time and improving assignment accuracy.

The Einstein Prompt Template will use structured data from Claim__c, Policy__c, and Claim_Line__c as grounding context and return a structured JSON triage recommendation that populates Claim__c.AI_Triage_Recommendation__c and Claim__c.Recommended_Queue__c.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Status__c = 'FNOL Received' and all required intake fields populated,
When the Agentforce triage action is triggered (via Record-Triggered Flow on after insert),
Then the Einstein Prompt Template 'ClaimTriageRecommendation_v1' must be invoked with the following grounding fields: Claim__c.Total_Billed_Amount__c, Claim__c.LOB__c, Claim__c.Injury_Description__c, Policy__c.Policy_Type__c, Policy__c.Coverage_Limit__c, and a count of related Claim_Line__c records.

Given the prompt template executes successfully,
When the AI response is parsed,
Then Claim__c.AI_Triage_Recommendation__c must be populated with the AI-generated summary (max 1000 characters) and Claim__c.Recommended_Queue__c must be set to one of: 'Standard Claims Queue', 'Complex Claims Queue', or 'SIU Referral Queue'.

Given the AI recommendation suggests 'SIU Referral Queue',
When Claim__c.Recommended_Queue__c is set,
Then a Case must be auto-created linked to the Claim__c with Case.Type = 'SIU Referral' and Case.OwnerId = the SIU queue Id, and a Chatter post must be added to the Claim__c record tagging the SIU team.

Given the Agentforce prompt invocation fails or times out,
When the error is caught,
Then Claim__c.AI_Triage_Status__c must be set to 'Failed - Manual Review Required' and no queue assignment should be made automatically; a Bell Notification must alert the Claims Operations Manager.

Given the triage recommendation is generated,
When a Claims Operations Manager reviews the FNOL record,
Then the AI recommendation and confidence score (Claim__c.AI_Confidence_Score__c) must be visible in a dedicated 'AI Triage' section on the Claim__c Lightning Record Page, with an 'Override' button to manually change the queue assignment.

-Given Claim__c.LOB__c = 'Workers Compensation', apply a specialized WC triage prompt template.- (Deferred to Phase 2 — WC-specific prompt template not yet approved by Compliance Officer.)

## Solution

Leverage Salesforce Agentforce / Einstein Generative AI platform: (1) Build Prompt Template 'ClaimTriageRecommendation_v1' in Setup → Prompt Builder, type 'Record' anchored to Claim__c, with merge fields pulling grounding data from Claim__c, related Policy__c (via lookup), and a COUNT of Claim_Line__c children via a template formula. (2) Define a structured output schema in the prompt instructing the LLM to return JSON: { 'recommendation': string, 'queue': enum, 'confidence': float }. (3) Create an Invocable Apex class 'AgentforceTriageInvoker.invokeTriage(List<Id> claimIds)' that calls the ConnectApi.EinsteinLLM.generateMessagesForPromptTemplate() method and parses the JSON response. (4) Wire the invocable into a Record-Triggered Flow 'RTF_FNOL_Intake_Triage' on Claim__c after insert WHERE Status__c = 'FNOL Received', using an async path to avoid blocking the transaction. (5) Build a child LWC 'claimAITriagePanel' for the AI Triage page section with an Override quick action backed by an OmniScript. (6) Store prompt template version in a Custom Metadata Type 'AI_Prompt_Config__mdt' for environment-specific overrides.

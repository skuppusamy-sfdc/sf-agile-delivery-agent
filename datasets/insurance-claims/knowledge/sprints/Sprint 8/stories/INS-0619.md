## Description

**Bug Report — UAT Round 2 — Group/Employer Onboarding**

When a Broker Relations Coordinator clones an existing Group_Service_Agreement__c (GSA) that has an expired template (GSA_Template__c.Status__c = 'Expired'), the effective date validation rule is not firing, allowing the new GSA to be saved with GSA.Effective_Date__c in the past.

**Steps to Reproduce:**
1. Identify a Group_Service_Agreement__c record with GSA.Template__c pointing to a GSA_Template__c where Status__c = 'Expired'.
2. Use the standard Salesforce 'Clone' button on the GSA record.
3. On the cloned record, leave GSA.Effective_Date__c as-is (copied from original, set to a past date e.g. 01/01/2023).
4. Click Save.

**Expected:** Validation rule fires: 'Effective Date must be today or a future date for new GSA records.' Save is blocked.

**Actual:** Record saves successfully with a past Effective_Date__c. GSA.Status__c incorrectly auto-sets to 'Active' via the status automation flow, creating a live GSA backed by an expired template.

**Impact:** 3 incorrect GSA records created in UAT environment. Broker Relations team flagged this as a compliance risk — an active GSA with an expired template could allow group enrollments under unapproved terms.

**Environment:** Acme-UAT sandbox. Also confirmed in Acme-QA. Not reproducible when creating a new GSA from scratch (validation rule fires correctly).

## Solution

Root cause: the existing validation rule 'VR_GSA_EffectiveDateFuture' uses the formula condition ISNEW() to restrict the check to new records only. When cloning, Salesforce treats the cloned record as a new record in the UI but the ISNEW() function evaluates to FALSE in certain clone contexts due to the record ID being null pre-save. Fix: replace ISNEW() with NOT(ISCLONE()) OR (ISCLONE() AND Effective_Date__c < TODAY()) as the combined trigger condition, ensuring validation fires on both new and cloned records with past dates. Additionally, updated the Record-Triggered Flow 'RTF_GSA_StatusAutomation' to add a decision node checking GSA_Template__c.Status__c != 'Expired' before setting GSA.Status__c = 'Active'. A cleanup script (Anonymous Apex) to be run by system admin to correct the 3 affected UAT records.

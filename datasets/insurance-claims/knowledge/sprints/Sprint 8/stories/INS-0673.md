## Description

**Bug Report — UAT Sprint 8**

After a Payment__c record is posted against a Claim__c, the associated Explanation_of_Benefits__c (EOB) record is not reflecting the updated adjudication status on individual Claim_Line__c records. The EOB continues to show 'Pending' on all line items even when payment has been fully disbursed.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record with Status__c = 'Approved'.
2. Create a Payment__c record linked to the Claim__c with Payment_Status__c = 'Posted' and Payment_Amount__c matching the approved amount.
3. Verify Claim_Line__c records — each should update to Adjudication_Status__c = 'Adjudicated'.
4. Navigate to the linked Explanation_of_Benefits__c record.
5. Observe that EOB_Line_Status__c on each EOB line still reads 'Pending' instead of 'Adjudicated'.

**Expected Behavior:** When Payment__c.Payment_Status__c transitions to 'Posted', all related Claim_Line__c records should update Adjudication_Status__c to 'Adjudicated', and the parent EOB__c record's EOB_Line_Status__c fields should mirror this change within the same transaction or via a near-real-time platform event.

**Actual Behavior:** Claim_Line__c.Adjudication_Status__c updates correctly, but EOB__c lines remain stale at 'Pending'.

**Environment:** Full Sandbox (UAT). Reproducible 100% of the time.

**Root Cause (Initial Assessment):** The Record-Triggered Flow on Payment__c updates Claim_Line__c via a related list update, but the subsequent EOB sync logic is gated behind a Process Builder that was deprecated in Sprint 6 and not fully replaced.

## Acceptance Criteria

as per Description

## Solution

1. Retire the legacy Process Builder 'EOB_Line_Sync_PB' (currently inactive but causing confusion).
2. Extend the existing Record-Triggered Flow 'Payment_Posted_Update_Flow' (on Payment__c, After Save, fires on Status change to 'Posted') to include a second update path:
   - Get Records: Claim_Line__c WHERE Claim__c = {triggering Payment__c.Claim__c}
   - Update Records: set Claim_Line__c.Adjudication_Status__c = 'Adjudicated'
3. Fire a Platform Event 'EOB_Sync_Event__e' carrying Claim__c Id after Claim_Line__c bulk update.
4. A separate Apex trigger handler (EOBSyncEventTrigger) subscribes to EOB_Sync_Event__e and performs a bulkified DML update on all EOB_Line__c records linked via EOB__c.Claim__c relationship, setting EOB_Line_Status__c = 'Adjudicated'.
5. Add a Flow fault path with error logging to Error_Log__c for any failed EOB sync events.
6. Deploy and validate in Full Sandbox before promoting to UAT.

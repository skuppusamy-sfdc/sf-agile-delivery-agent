## Description

**Bug Report — UAT Sprint 8**

When a Claim__c record has Coordination_of_Benefits__c = true and a secondary payer payment is applied, the associated Claim_Line__c records remain in Status__c = 'Pending' rather than transitioning to 'Adjudicated'. This causes the Explanation_of_Benefits__c (EOB) document generated via Conga CLM to reflect incorrect line-level status and incorrect patient responsibility amounts.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record where COB_Indicator__c = 'Secondary'.
2. Apply a primary payer Payment__c record with Payment_Type__c = 'Primary EOB' and a valid Payment_Amount__c.
3. Create a secondary Payment__c record linked to the same Claim__c with Payment_Type__c = 'Secondary COB'.
4. Save and trigger EOB generation via the 'Generate EOB' quick action.
5. Observe Claim_Line__c.Status__c values and the rendered EOB PDF.

**Expected:** All Claim_Line__c records transition to Status__c = 'Adjudicated' and EOB reflects correct Allowed_Amount__c, Patient_Responsibility__c, and Plan_Paid_Amount__c after COB waterfall is applied.

**Actual:** Claim_Line__c.Status__c remains 'Pending'. EOB shows Patient_Responsibility__c = full billed amount, ignoring secondary payer offset. Plan_Paid_Amount__c on EOB is also incorrect.

## Solution

Root cause identified in the Record-Triggered Flow 'Update_Claim_Line_Status_on_Payment' — the entry condition checks Payment__c.Payment_Type__c = 'Primary' only, missing the 'Secondary COB' type. Fix: update Flow entry criteria to include OR condition for Payment_Type__c = 'Secondary COB'. Additionally, update the Apex class ClaimCOBWaterfallService.applySecondaryPayer() to call recalculateLineStatus() after secondary payment is persisted. Conga CLM template merge fields for EOB__c.Patient_Responsibility__c must reference the recalculated field Claim__c.Calculated_Patient_Responsibility__c rather than the raw Billed_Amount__c. Deploy updated Flow (v14 → v15) and Apex patch to Sandbox for re-validation.

# INS-0613: Policyholder 360 — Add COB (Coordination of Benefits) indicator and secondary payer details to member summary panel

## Description

As a Claims Operations Manager, I need the Policyholder 360 member summary panel to clearly display Coordination of Benefits (COB) information — including whether a secondary payer exists, the secondary payer name, and the COB order — so that claims adjudicators can quickly identify members with dual coverage before adjudicating claims, reducing COB-related payment errors and rework.

## Acceptance Criteria

Given a Claims Operations Manager views a Policy__c record page with the Policyholder 360 LWC component,
When the member summary panel loads,
Then a 'Coordination of Benefits' section is visible in the panel.

Given Policy__c.COB_Active__c = true,
When the COB section renders,
Then the following fields must display:
- COB indicator badge: 'COB Active' in amber
- Policy__c.Secondary_Payer_Name__c
- Policy__c.Secondary_Payer_ID__c
- Policy__c.COB_Order__c (values: 'Primary', 'Secondary', 'Tertiary')
- Policy__c.COB_Effective_Date__c

Given Policy__c.COB_Active__c = false or null,
When the COB section renders,
Then display 'No COB on File' in grey text and suppress all secondary payer fields.

Given Policy__c.COB_Order__c = 'Secondary',
When a Claims Operations Manager is viewing the panel during claim adjudication,
Then a contextual warning banner displays: 'This member is secondary — verify primary EOB before adjudicating.'

-Given COB_Active__c = true, automatically trigger a COB verification task assigned to the Underwriting Analyst queue on page load.-
(Removed per UAT feedback — task creation caused performance issues on high-volume queues; deferred to INS-0634)

Given the LWC component loads,
When Policy__c.COB_Active__c = true AND Policy__c.COB_Effective_Date__c < TODAY() - 365,
Then display a stale data warning: 'COB information may be outdated — last verified over 12 months ago. Consider re-verification.'

Given the user has read-only access (no 'Claims_Adjudicator' or 'Claims_Operations_Manager' Permission Set Group),
When the panel loads,
Then COB details are masked: Secondary_Payer_ID__c displays as '***-**-[last 4]' and Secondary_Payer_Name__c is hidden.

## Solution

1. Extend the existing LWC component 'policyholderSummaryPanel' (introduced in INS-0571) by adding a new child component 'cobDetailsSection' that is conditionally rendered based on Policy__c.COB_Active__c.
2. Wire Policy__c fields (COB_Active__c, Secondary_Payer_Name__c, Secondary_Payer_ID__c, COB_Order__c, COB_Effective_Date__c) via @wire(getRecord) with field-level security enforcement.
3. Implement field masking logic in the LWC JavaScript controller by checking the current user's Permission Set Group membership via a custom Apex @AuraEnabled(cacheable=true) method 'getUserCOBAccessLevel()'.
4. COB stale data warning computed in the LWC JavaScript using a getter comparing COB_Effective_Date__c to today's date client-side.
5. Amber/grey badge styling implemented via SLDS utility classes with dynamic class binding in the LWC template.
6. Update the Policy__c Lightning Record Page layout in UAT to include the updated component and validate with Claims Operations Manager persona test user.

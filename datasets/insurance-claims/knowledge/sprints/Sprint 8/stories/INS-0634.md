## Description

As a Claims Operations Manager, I need a summary tile on the Policyholder 360 page that displays all active claims associated with a member's Policy__c records, including FNOL date and current claim status, so that I can quickly assess a member's claims history during inbound service calls without running separate SOQL reports or navigating away from the account record.

## Acceptance Criteria

Given a Policy__c record is associated with a member Account (Policy__c.Policyholder__c = Account.Id),
When the Policyholder 360 page loads,
Then the Active Claims Summary tile must display a list of all Claim__c records WHERE Claim__c.Policy__c IN (member's active policies) AND Claim__c.Status__c NOT IN ('Closed', 'Denied', 'Withdrawn'), showing: Claim__c.Name, Claim__c.FNOL_Date__c, Claim__c.Status__c, Claim__c.LOB__c, and Claim__c.Total_Billed_Amount__c.

Given a member has more than 5 active claims,
When the tile renders,
Then display the 5 most recent by Claim__c.FNOL_Date__c descending, with a 'View All' link navigating to a filtered list view.

Given a member has zero active claims,
When the tile loads,
Then display the message: 'No active claims on file for this member.'

Given a Claim__c.Status__c = 'Under SIU Review',
When the tile renders that claim row,
Then the status badge must display in red with a lock icon, visible only to users with the 'SIU_Reviewer' Permission Set.

Given the tile is viewed by a user without the 'SIU_Reviewer' Permission Set,
When a claim under SIU review is in the list,
Then that claim row must be hidden entirely from the tile (not just the badge).

Given the tile is loaded,
When a Claims Operations Manager clicks a Claim__c.Name link,
Then the claim record must open in a new subtab within the same console app without full page navigation.

## Solution

Build a new LWC 'policyholderActiveClaimsTile' deployed to the Account Lightning Record Page via App Builder. (1) Wire a custom Apex controller 'PolicyholderClaimsController.getActiveClaims(Id accountId)' that queries Claim__c with the appropriate WHERE clause, respecting Sharing Rules (WITH SHARING). (2) Implement SIU badge visibility using @wire(getPermissionSetAssignments) or a custom Apex method returning a Boolean 'isSIUReviewer' to conditionally render the badge and row via lwc:if directives. (3) Use the lightning/navigation module with 'standard__recordPage' for subtab navigation within the console. (4) Cache the Apex result with @AuraEnabled(cacheable=true) and a 5-minute TTL. (5) Add a 'View All' navigation action linking to a pre-filtered Claim__c list view scoped to the member's policy Ids. Validate rendering in both Service Console and standalone record page layouts.

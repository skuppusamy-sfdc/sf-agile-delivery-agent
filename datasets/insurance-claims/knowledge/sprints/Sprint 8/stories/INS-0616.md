## Description

As a Claims Operations Manager, I need Claim_Line__c adjudication to automatically resolve the correct Fee_Schedule__c allowed amount based on the rendering provider's active PNA network tier and the CPT code on the claim line so that manual fee lookups are eliminated and adjudication accuracy improves across all LOBs.

This story covers the core lookup logic. ERA posting integration is handled separately in INS-0622 (Phase 2 backlog).

## Acceptance Criteria

Given a Claim_Line__c is saved with Claim_Line__c.CPT_Code__c populated and the parent Claim__c.Rendering_Provider__c has an active PNA,
When the adjudication process triggers,
Then the system resolves Fee_Schedule__c by matching PNA.Network_Tier__c + Claim_Line__c.CPT_Code__c + Claim_Line__c.Service_Date__c within Fee_Schedule__c.Effective_Date__c and Fee_Schedule__c.Termination_Date__c, and writes Fee_Schedule__c.Allowed_Amount__c to Claim_Line__c.Allowed_Amount__c.

Given a matching Fee_Schedule__c record is found,
When Claim_Line__c.Allowed_Amount__c is set,
Then Claim_Line__c.Fee_Schedule_Lookup_Status__c = 'Resolved' and Claim_Line__c.Fee_Schedule__c (lookup) is populated with the matched record ID.

Given no active Fee_Schedule__c matches the CPT + tier + date combination,
When the lookup executes,
Then Claim_Line__c.Fee_Schedule_Lookup_Status__c = 'Unresolved — Manual Review' and a Task is created assigned to the Claims Operations queue with Subject = 'Fee Schedule Gap: ' + CPT_Code__c.

Given the rendering provider has multiple active PNA records across LOBs,
When the fee lookup runs,
Then the system matches the PNA whose PNA.LOB__c = parent Claim__c.LOB__c to ensure correct tier resolution.

Given Claim_Line__c.CPT_Code__c is blank or invalid (not in the CPT reference list),
When adjudication triggers,
Then Claim_Line__c.Fee_Schedule_Lookup_Status__c = 'Invalid CPT' and the claim line is flagged for SIU review via Claim__c.SIU_Flag__c = TRUE.

Given a Fee_Schedule__c record is found but its Fee_Schedule__c.Rate_Type__c = 'Percentage of Billed',
When Claim_Line__c.Allowed_Amount__c is calculated,
Then the system applies Fee_Schedule__c.Rate_Percentage__c * Claim_Line__c.Billed_Amount__c and stores the result.

## Solution

Implemented as an Apex invocable method 'FeeScheduleLookupService.resolveFeeSchedule()' called from a Record-Triggered Flow on Claim_Line__c (trigger: after insert/update, when CPT_Code__c is not null AND Adjudication_Status__c = 'Pending'). The Apex class performs a SOQL query on Fee_Schedule__c with indexed fields (Network_Tier__c, CPT_Code__c, Effective_Date__c) to ensure governor limit compliance in bulk scenarios. PNA resolution uses a separate helper method that queries Provider_Network_Agreement__c with a compound filter on Provider__c + LOB__c + Contract_Status__c = 'Active'. Rate_Type__c branching logic handled in Apex switch statement. Unresolved and SIU cases create Tasks via Database.insert in a separate transaction using a Platform Event to avoid mixed DML. Custom metadata type 'CPT_Reference__mdt' used for CPT code validation.

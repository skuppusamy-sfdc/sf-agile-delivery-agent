## Description

**Bug Summary:** EOB PDF documents generated through the Conga CLM integration are truncating Claim_Line__c detail rows when a parent Claim__c has more than 20 associated Claim_Line__c records. Claims with 20 or fewer lines generate correctly.

**Steps to Reproduce:**
1. Locate or create a Claim__c record with exactly 21 or more Claim_Line__c child records, all with Status__c = 'Adjudicated' and Allowed_Amount__c populated.
2. From the Claim__c record, click the 'Generate EOB' quick action button.
3. Observe the Conga CLM merge job execute (InsCongaEOBMergeJob).
4. Open the generated EOB__c record and download the attached PDF.
5. Count the Claim_Line__c detail rows in the PDF.

**Expected Behavior:** All Claim_Line__c records associated with the Claim__c should appear as individual line-item rows in the EOB PDF, regardless of count.

**Actual Behavior:** The EOB PDF contains exactly 20 claim line rows. Lines 21 and beyond are silently dropped. The EOB__c.Total_Lines_Generated__c field reflects only 20 even when 25+ lines exist. No error is thrown.

**Root Cause (Suspected):** The Conga template query (CongaQueryString) used in the EOB merge action has a hardcoded LIMIT 20 clause on the Claim_Line__c SOQL query parameter. This was a temporary dev constraint that was never removed before UAT deployment.

**Environment:** UAT Sandbox — InsuranceUAT1
**Frequency:** 100% reproducible with 21+ claim lines.

## Acceptance Criteria

as per Description

## Solution

1. Locate the Conga Composer action 'Generate_EOB_Action' on the Claim__c object and inspect the CongaQueryString parameter.
2. Remove the hardcoded 'LIMIT 20' from the Claim_Line__c sub-query within the CongaQueryString, replacing with 'LIMIT 1000' as a safe operational ceiling (per business confirmation that no single claim will exceed 1000 lines).
3. Update the Conga template 'EOB_Template_v3.docx' in the Conga template library to ensure the repeating table section for claim lines uses a dynamic repeat tag rather than a fixed-count loop.
4. Update EOB__c.Total_Lines_Generated__c population logic in the post-merge Flow 'RT_EOB_PostGeneration_Update' to use COUNT() of related Claim_Line__c rather than the Conga output metadata.
5. Re-test in InsuranceUAT1 with a 25-line claim and a 50-line claim to confirm resolution.
6. Update regression test case INS-RT-067 to include a 25-line scenario as a permanent regression guard.

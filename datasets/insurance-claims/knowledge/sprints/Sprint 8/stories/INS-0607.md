## Description

As a Claims Operations Manager, I need a unified Policyholder 360 dashboard LWC embedded on the Policy__c record page that displays: (1) a summary of active and recently closed Claim__c records with status badges, (2) the last 5 Payment__c disbursements with amounts and dates, and (3) key Policy__c coverage details including LOB, deductible met status, and out-of-pocket maximum — so that I can assess a policyholder's complete claims and financial picture in a single view during FNOL intake and ongoing claim management without context-switching across multiple related lists.

## Acceptance Criteria

Given a Policy__c record is opened by a Claims Operations Manager,
When the Policyholder 360 LWC loads,
Then the Claims Summary tile displays all Claim__c records where Claim__c.Policy__c = Policy__c.Id and Claim__c.Status__c IN ('Open', 'Under Review', 'Adjudicated') with color-coded status badges: Open = blue, Under Review = yellow, Adjudicated = green.

Given a Policy__c with related Payment__c records,
When the Policyholder 360 LWC loads,
Then the Recent Payments tile displays the 5 most recent Payment__c records ordered by Payment__c.Payment_Date__c DESC, showing Payment__c.Amount__c, Payment__c.Payment_Date__c, Payment__c.Payment_Method__c, and the linked Claim__c.Claim_Number__c.

Given a Policy__c record,
When the Coverage Details tile renders,
Then it displays: Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, Policy__c.Annual_Deductible__c, Policy__c.Deductible_Met__c (boolean badge), Policy__c.Out_of_Pocket_Maximum__c, and Policy__c.Out_of_Pocket_Met__c.

Given a Policy__c with no active Claim__c records,
When the Claims Summary tile renders,
Then a message 'No Active Claims' is displayed with a 'File New Claim (FNOL)' quick-action button.

Given a Claims Operations Manager without the 'Policy_Financial_Details_View' custom permission,
When the Coverage Details tile renders,
Then Policy__c.Annual_Deductible__c and Policy__c.Out_of_Pocket_Maximum__c fields display as '---'.

-Given any Policy__c record, When the LWC loads, Then display a predictive claim likelihood score powered by Einstein Analytics- (Deprecated — Einstein predictive scoring deferred to Phase 2 Agentforce integration)

Given the LWC is loaded on a Policy__c with more than 20 related Claim__c records,
When the Claims Summary tile renders,
Then only the 10 most recently modified Claim__c records are displayed with a 'View All Claims' link navigating to a filtered list view.

## Solution

Build a single LWC (lwc-policyholder360-dashboard) with three child components: lwc-ph360-claims-summary, lwc-ph360-recent-payments, and lwc-ph360-coverage-details. Use @wire(getRecord) with field-level security for Policy__c coverage fields. Implement Apex controller (Policyholder360Controller.cls) with @AuraEnabled(cacheable=true) methods: getActiveClaims(policyId) returning up to 10 Claim__c records, getRecentPayments(policyId) returning top 5 Payment__c records, and getCoverageDetails(policyId). Apply SOQL ORDER BY and LIMIT clauses server-side. Use a Custom Permission ('Policy_Financial_Details_View') checked in Apex via FeatureManagement.checkPermission() to conditionally null-out financial fields before returning to client. Implement conditional rendering in LWC template using {#if} directives for empty states and permission-masked fields. Add the LWC to the Policy__c Lightning Record Page via App Builder with a 'Claims Ops' audience filter. Update PSG-ClaimsOps-Manager to include the custom permission.

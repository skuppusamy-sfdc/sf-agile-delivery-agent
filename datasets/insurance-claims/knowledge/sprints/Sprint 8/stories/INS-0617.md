## Description

As a Provider Network Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record and dispatch a DocuSign envelope containing the termination letter when a Network_Termination_Case__c is approved so that provider termination communications are standardized, auditable, and dispatched within the SLA-required 30-day notice window.

Compliance requirement: per CMS contract, PCN must be issued no later than 30 calendar days before PNA.Termination_Date__c.

## Acceptance Criteria

Given a Network_Termination_Case__c.Status__c transitions to 'Approved' by a user with the 'Network_Director' role,
When the approval is saved,
Then a Provider_Change_Notification__c is created with PCN.Type__c = 'Termination', PCN.Provider__c = Network_Termination_Case__c.Provider__c, PCN.Effective_Date__c = PNA.Termination_Date__c, and PCN.Case__c = the originating Network_Termination_Case__c ID.

Given the PCN record is created,
When the DocuSign integration triggers,
Then a DocuSign envelope is sent to Provider__c.Email using the 'Provider_Termination_Letter_Template' Conga/DocuSign template, and PCN.DocuSign_Envelope_Status__c = 'Sent' with PCN.Envelope_ID__c populated.

Given PNA.Termination_Date__c is less than 30 days from today when the case is approved,
When the PCN is generated,
Then Network_Termination_Case__c.Compliance_Flag__c = TRUE and a high-priority Case comment is added: 'WARNING: Termination date is within 30-day CMS notice window. Compliance review required.'

-Given the provider has no active PNA, suppress PCN generation and log a warning on the Case.-

Given the provider has no active PNA at time of approval,
When the automation runs,
Then Network_Termination_Case__c.Status__c rolls back to 'Pending Review', an error message is displayed to the approver: 'No active PNA found. Cannot generate termination notice.', and no PCN or DocuSign envelope is created.

Given a DocuSign envelope dispatch fails (e.g., invalid provider email),
When the error is returned from the DocuSign API,
Then PCN.DocuSign_Envelope_Status__c = 'Failed', a Task is created for the assigned Provider Network Specialist, and Network_Termination_Case__c.Status__c = 'Pending — Communication Error'.

## Solution

Record-Triggered Flow on Network_Termination_Case__c fires on Status__c change to 'Approved'. Flow calls Apex invocable action 'PCNGenerationService.createPCNAndDispatch()' which: (1) queries active PNA for the provider, (2) validates 30-day window using Date arithmetic, (3) creates Provider_Change_Notification__c via DML, (4) calls DocuSign Apex Toolkit 'dfsle.PackageService.sendEnvelope()' with the mapped template ID stored in Custom Metadata 'DocuSign_Template_Config__mdt'. Compliance flag logic implemented in Apex with a dedicated method. Rollback on missing PNA uses Flow fault path with a Screen element displaying the error message to the approver. DocuSign failure handling via try/catch updating PCN status and creating Task via a separate Platform Event 'PCN_Comm_Failure__e' to avoid mixed DML errors.

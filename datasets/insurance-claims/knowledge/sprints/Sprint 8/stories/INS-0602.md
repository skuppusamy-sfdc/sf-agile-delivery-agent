## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces active Provider_Network_Agreement__c (PNA) contract details, network tier, effective/termination dates, and associated Fee_Schedule__c name inline — without navigating to related lists — so that I can quickly assess a provider's network standing during inbound provider calls and claims inquiries.

Extends INS-0571 (Provider 360 base FlexCard scaffold).

## Acceptance Criteria

Given a Provider__c record with one or more related Provider_Network_Agreement__c records,
When the Provider Relations Specialist opens the Provider 360 FlexCard on the Provider__c record page,
Then the FlexCard displays a tabbed child card for each PNA where PNA.Status__c = 'Active', showing: PNA.Contract_Number__c, PNA.Network_Tier__c, PNA.Effective_Date__c, PNA.Termination_Date__c, and the related Fee_Schedule__c.Name.

Given a Provider__c with no active PNA records,
When the FlexCard loads,
Then a contextual message 'No Active Network Agreements on File' is displayed in the PNA panel with a quick-action button to initiate a new PNA via the NetworkContracting OmniScript.

Given a PNA where PNA.Termination_Date__c is within 90 days of today,
When the FlexCard renders the PNA child card,
Then a yellow warning badge labeled 'Expiring Soon' is displayed adjacent to the PNA.Termination_Date__c field.

Given a Provider__c record,
When the FlexCard is rendered,
Then the DataRaptor Extract (DR-Provider360-PNADetails) response time must be under 2 seconds for providers with up to 10 active PNA records.

-Given a Provider__c record, When the FlexCard loads, Then display all historical PNA records regardless of status- (Deprecated — historical PNA view deferred to Phase 2 Provider Contract History story)

Given a Provider Relations Specialist without the 'PNA_Financials_View' Permission Set,
When the FlexCard renders,
Then Fee_Schedule__c.Base_Rate__c and Fee_Schedule__c.Multiplier__c fields are masked and display '---' in place of values.

## Solution

Build a child FlexCard (FC-PNA-ContractDetails) using OmniStudio FlexCards, driven by a DataRaptor Extract (DR-Provider360-PNADetails) querying Provider_Network_Agreement__c and related Fee_Schedule__c filtered by PNA.Status__c = 'Active' and PNA.Provider__c = {recordId}. Embed FC-PNA-ContractDetails as a child card within the existing Provider 360 parent FlexCard (FC-Provider360-Base) introduced in INS-0571. Implement the 90-day expiry warning using a FlexCard conditional state with a formula comparing TODAY() to PNA.Termination_Date__c. Use a Custom Permission ('PNA_Financials_View') checked via FlexCard conditional visibility to mask fee schedule financial fields. Deploy FlexCard to the Provider__c Lightning Record Page via App Builder. Update the Permission Set Group (PSG-ProviderRelations-Standard) to include the new custom permission for eligible roles.

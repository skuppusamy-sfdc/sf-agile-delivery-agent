## Description

**Bug Report — UAT Sprint 8**

When a Claim__c record undergoes Coordination of Benefits (COB) processing where a secondary payer adjudicates after the primary, the associated Claim_Line__c records are not updating their Status__c field to 'Adjudicated' on the generated Explanation_of_Benefits__c (EOB). The EOB is created but reflects stale line-level data.

**Steps to Reproduce:**
1. Create a Claim__c record with LOB = 'Medical' and COB_Indicator__c = true.
2. Associate two or more Claim_Line__c records with CPT codes 99213 and 99214.
3. Process primary payer adjudication — observe Claim_Line__c.Status__c updates correctly to 'Primary Adjudicated'.
4. Trigger secondary payer COB processing via the 'Apply COB' action button.
5. Navigate to the auto-generated Explanation_of_Benefits__c record.
6. Observe that Claim_Line__c.EOB_Line_Status__c still reads 'Pending' instead of 'Adjudicated'.

**Expected Behavior:** After COB secondary processing completes, all related Claim_Line__c records should reflect EOB_Line_Status__c = 'Adjudicated' and the EOB header Claim__c.Status__c should update to 'Adjudicated'.

**Actual Behavior:** EOB_Line_Status__c remains 'Pending'. Claim__c.Status__c does update correctly, but the line-level rollup is broken.

**Environment:** Full Sandbox (UAT). Reproducible 100% of the time when COB_Indicator__c = true and secondary payer is a non-MCO TPA.

## Solution

Root cause identified in the Record-Triggered Flow 'Update_EOB_Lines_On_COB_Completion' — the entry condition was filtering on Claim__c.Status__c = 'Adjudicated' but the flow fires before the COB Payment__c record is fully committed, creating a race condition. Fix: Convert the Record-Triggered Flow to run in 'After Save' context with a related-record update via Apex invocable to batch-update all child Claim_Line__c.EOB_Line_Status__c records. Add a Platform Event (COB_Processing_Complete__e) published from the COB Apex service class to decouple the trigger timing. Add SOQL-based assertion in the Apex test class covering TPA secondary payer path.

## Description

As a Claims Operations Manager, I need an Agentforce-powered OmniScript embedded in the FNOL intake process so that incoming first notice of loss submissions are automatically triaged by AI to assign an initial severity score, recommended LOB routing, and SIU referral flag, reducing manual triage time by at least 40%.

The OmniScript will be surfaced in both the internal Claims Portal and the external policyholder self-service portal. Agentforce topic and action configurations are being established in parallel by the AI Platform team.

## Acceptance Criteria

Given a Claims Operations user initiates FNOL intake via the OmniScript 'OS_FNOL_Intake_Agentforce',
When the policyholder's Policy__c.Policy_Number__c is entered and validated,
Then the OmniScript auto-populates Claim__c fields: Policyholder__c, Policy__c, LOB__c, and Coverage_Type__c from the matched policy record.

Given the FNOL intake form is completed with incident description, date of loss, and ICD-10 codes,
When the user clicks 'Analyze with Agentforce',
Then the Agentforce Einstein prompt template 'FNOL_Triage_Prompt' is invoked and returns: Severity_Score__c (1–5 integer), Recommended_LOB__c, and SIU_Referral_Recommended__c (boolean) within 5 seconds.

Given Agentforce returns SIU_Referral_Recommended__c = true,
When the OmniScript step 'Review AI Triage' renders,
Then a prominent warning banner displays: 'AI has flagged this claim for potential SIU review. Please confirm before submitting.' and the user must check an acknowledgment checkbox before proceeding.

Given the AI triage completes successfully,
When the FNOL is submitted,
Then Claim__c.AI_Triage_Score__c, Claim__c.AI_Triage_Timestamp__c, and Claim__c.AI_Triage_Model_Version__c are persisted on the created Claim__c record for auditability.

Given the Agentforce API call fails or times out,
When the OmniScript handles the error,
Then a fallback manual triage step is presented and Claim__c.AI_Triage_Status__c = 'Unavailable — Manual Review Required' is set.

-Given the policyholder is on a Government LOB (Medicare/Medicaid), the AI triage score must be suppressed and manual routing enforced.- (Requirement deferred to Phase 2 per compliance review)

Given the submitted Claim__c has Severity_Score__c >= 4,
When the record is created,
Then a Platform Event 'High_Severity_Claim__e' is published to trigger downstream assignment routing to the senior adjuster queue.

## Solution

OmniScript 'OS_FNOL_Intake_Agentforce' built with 6 steps: Policy Lookup, Incident Details, ICD-10 Code Entry, Agentforce Triage (Integration Procedure call), Triage Review, and Submission Confirmation. Integration Procedure 'IP_Agentforce_FNOL_Triage' invokes the Agentforce Einstein Generative action via HTTP Action element pointing to the Agentforce API endpoint with the 'FNOL_Triage_Prompt' template. DataRaptor Upsert persists AI results to Claim__c. Platform Event publish on high severity handled via Apex invocable called from a post-submit Record-Triggered Flow. Fallback error handling implemented via Try/Catch in the Integration Procedure with a conditional branch to manual triage step. OmniScript embedded on both internal App Page and Experience Cloud page using Lightning Web Component wrapper.

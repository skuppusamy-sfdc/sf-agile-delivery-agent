## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can instantly view a provider's active credentialing status, network participation, and PNA summary without navigating across multiple related lists.

This story extends the data foundation established in INS-0606 (Provider Data Maintenance DataRaptor layer) and surfaces that data through an OmniStudio FlexCard.

## Acceptance Criteria

Given a Provider Relations Specialist opens any Provider__c (Person Account) record,
When the Provider 360 FlexCard loads,
Then the card must display the following data blocks within 3 seconds:
  - Provider NPI (Provider__c.NPI__c), TIN (Provider__c.TIN__c), and primary specialty (Provider__c.Primary_Specialty__c)
  - Active Credentialing_Case__c count with a status badge (Active / Pending / Expired) derived from Credentialing_Case__c.Status__c
  - Count of active Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active' and PNA__c.Effective_End_Date__c >= TODAY
  - -Display all historical PNA records regardless of status- *(deprecated: scope reduced to active-only for Phase 1)*
  - A 'CAQH Verified' indicator sourced from Provider__c.CAQH_Verified__c (checkbox) with last verification date Provider__c.CAQH_Last_Verified_Date__c

Given the provider has a Network_Termination_Case__c with Status__c = 'In Progress',
When the FlexCard renders,
Then a prominent amber warning banner reading 'Termination In Progress' must appear at the top of the card with a link to the open Network_Termination_Case__c record.

Given the FlexCard is viewed by a user without the 'Provider_Network_Full_Access' Permission Set Group,
When the card attempts to load TIN data,
Then TIN must be masked as ***-**-XXXX and a tooltip 'Restricted — contact your administrator' must display.

Given the provider's Credentialing_Case__c.Expiration_Date__c is within 60 days of today,
When the FlexCard loads,
Then a red 'Credentialing Expiring Soon' badge must appear on the credentialing data block.

Given a Provider Relations Specialist clicks the 'View All Agreements' action on the FlexCard,
When the action fires,
Then an OmniScript modal (OS_ProviderPNASummary) must open displaying a paginated list of PNA records with columns: PNA__c.Name, PNA__c.Network__c, PNA__c.Effective_Start_Date__c, PNA__c.Effective_End_Date__c, PNA__c.Fee_Schedule__r.Name.

## Solution

Built using OmniStudio FlexCard 'FC_Provider360' with two child DataRaptors: DR_Provider360_Base (Extract) pulling Provider__c fields and DR_Provider360_Agreements (Extract) pulling related PNA and Credentialing_Case__c aggregates via SOQL with cross-object references. TIN masking implemented via a DataRaptor Transform formula using LEFT() and RPAD() functions conditioned on the running user's Permission Set Group membership checked via a custom Apex Remote Action (ProviderDataAccessController.getTINAccessLevel()). The amber termination banner uses a FlexCard conditional state driven by a boolean field populated in DR_Provider360_Base. The 'View All Agreements' action launches OmniScript OS_ProviderPNASummary embedded in an LWC modal wrapper. FlexCard deployed to the Provider__c Lightning Record Page via App Builder with audience targeting set to the 'Provider Relations' profile group.

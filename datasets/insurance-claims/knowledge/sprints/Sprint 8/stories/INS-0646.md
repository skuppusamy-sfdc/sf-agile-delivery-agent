## Description

As a Provider Network Specialist, I need the system to automatically validate that an associated Fee_Schedule__c record is active and within its effective date range before a Provider_Network_Agreement__c (PNA) can be set to 'Active' status, so that claims adjudication does not reference expired or future-dated fee schedules and result in incorrect reimbursement calculations.

## Acceptance Criteria

Given a user attempts to set PNA.Status__c = 'Active' on a Provider_Network_Agreement__c record,
When PNA.Fee_Schedule__c is null,
Then the save is blocked with a validation error: 'A Fee Schedule must be associated before activating this Provider Network Agreement.'

Given a PNA has an associated Fee_Schedule__c,
When Fee_Schedule__c.Effective_Start_Date__c > TODAY() at the time of PNA activation,
Then the save is blocked with the error: 'The linked Fee Schedule [Fee_Schedule__c.Name] is not yet effective. Activation date must be on or after [Fee_Schedule__c.Effective_Start_Date__c].'

Given a PNA has an associated Fee_Schedule__c,
When Fee_Schedule__c.Effective_End_Date__c < TODAY() at the time of PNA activation,
Then the save is blocked with the error: 'The linked Fee Schedule [Fee_Schedule__c.Name] has expired. Please associate a current Fee Schedule.'

Given all Fee Schedule date validations pass,
When PNA.Status__c is updated to 'Active',
Then PNA.Activation_Date__c is auto-populated with TODAY() and a Chatter post is created on the PNA record: 'PNA activated by [User Name] on [Date]. Fee Schedule: [Fee_Schedule__c.Name].'

Given a Provider Network Specialist with the 'PNA_Override' custom permission,
When they activate a PNA with a future-dated Fee Schedule,
Then a warning toast is shown but the save is allowed, and PNA.Override_Reason__c is required before completion.

## Solution

Implement validation logic via an Apex Trigger (before update) on Provider_Network_Agreement__c. The trigger handler class 'PNAActivationValidator' performs SOQL on the related Fee_Schedule__c to check Effective_Start_Date__c and Effective_End_Date__c. Custom permission 'PNA_Override' checked via FeatureManagement.checkPermission() to allow bypass path. Chatter post on activation handled by a separate Record-Triggered Flow (After Save, fires when Status__c changes to 'Active'). Auto-population of PNA.Activation_Date__c done within the same After Save flow using a formula resource TODAY(). Apex test coverage > 85% including override path.

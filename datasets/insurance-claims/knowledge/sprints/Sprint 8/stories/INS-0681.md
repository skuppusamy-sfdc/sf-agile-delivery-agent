## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page so that I can instantly view credentialing status, active PNA agreements, open PCNs, and recent claims activity without navigating across multiple objects.

This story extends INS-0642 (Provider Record Page Layout redesign) and incorporates the DataRaptor sources established in INS-0655.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record,
When the Provider 360 FlexCard loads,
Then the 'Credentialing Status' panel displays the most recent Credentialing_Case__c.Status__c and Credentialing_Case__c.Expiration_Date__c for the provider's NPI.

Given the provider has one or more Provider_Network_Agreement__c records,
When the FlexCard renders the 'Active Agreements' panel,
Then all PNA records where PNA__c.Status__c = 'Active' AND PNA__c.Effective_End_Date__c >= TODAY() are listed with columns: Agreement_Number__c, LOB__c, Effective_Start_Date__c, Effective_End_Date__c.

Given a Provider_Change_Notification__c (PCN) exists with PCN__c.Status__c IN ('Open', 'Pending Review'),
When the FlexCard loads,
Then an amber alert badge displays the count of open PCNs with a drill-down action linking to the PCN list view filtered by provider.

Given a provider has Claim__c records in the last 90 days,
When the 'Recent Claims Activity' panel renders,
Then it displays up to 10 claims sorted by Claim__c.Date_of_Service__c DESC, showing Claim__c.Claim_Number__c, Claim__c.Status__c, and Claim__c.Total_Billed_Amount__c.

-Given the provider is flagged for SIU review (Provider__c.SIU_Flag__c = true), a red banner must display across the full FlexCard header.- (Superseded by INS-0679 — SIU banner moved to separate compliance FlexCard)

Given the user does not hold the 'Provider_Network_Full_Access' Permission Set,
When the FlexCard loads,
Then the 'Fee Schedule' panel is hidden and no Fee_Schedule__c data is exposed.

## Solution

Built using OmniStudio FlexCard with five child cards (Credentialing, Agreements, PCN Alerts, Claims Activity, Fee Schedule). DataRaptor Extract actions query Credentialing_Case__c, Provider_Network_Agreement__c, Provider_Change_Notification__c, and Claim__c via SOQL with provider NPI as the correlation key. Conditional visibility on the Fee Schedule child card driven by a Custom Permission check using a DataRaptor Transform. PCN alert badge uses a Set Values action to compute open PCN count. FlexCard deployed to Provider__c Lightning Record Page via App Builder, restricted to 'Provider Relations' profile app. SLDS styling applied for amber/red badge states.

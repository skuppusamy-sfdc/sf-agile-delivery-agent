## Description

As a Claims Operations Manager, I need the FNOL intake OmniScript to capture Coordination of Benefits (COB) information when a claimant indicates they have other active coverage, so that the adjudication engine can correctly sequence primary and secondary payer responsibilities and avoid overpayment.

Currently the FNOL OmniScript ('FNOL_IntakeScript_v2') has no COB data capture step, forcing claims staff to manually update Claim__c after intake — a process gap flagged in UAT cycle 2.

## Acceptance Criteria

Given a claimant is completing the FNOL OmniScript step 'Coverage Details',
When the claimant selects 'Yes' to the question 'Do you have coverage under any other health plan?',
Then a conditional sub-step 'Other Coverage Details' is displayed, capturing: Other_Carrier_Name__c, Other_Policy_Number__c, Other_Group_Number__c, COB_Sequence__c (picklist: Primary / Secondary / Tertiary), and Other_Coverage_Effective_Date__c.

Given the COB sub-step is completed,
When the OmniScript is submitted,
Then Claim__c.COB_Indicator__c is set to TRUE and all captured other-coverage fields are saved to the Claim__c record.

Given the claimant selects 'No' to other coverage,
When the OmniScript is submitted,
Then Claim__c.COB_Indicator__c = FALSE and the COB sub-step fields remain null.

Given Claim__c.COB_Indicator__c = TRUE,
When the claim moves to status 'In Adjudication',
Then a validation rule prevents adjudication if Other_Carrier_Name__c is blank, displaying the message: 'COB carrier name is required before adjudication can proceed.'

Given the FNOL is submitted by a TPA user via the Agent Portal,
When COB = TRUE,
Then the TPA's TPA_Organization__c lookup is stamped on Claim__c.TPA__c automatically via the OmniScript Integration Procedure.

-Given COB Sequence = 'Tertiary', a SIU referral Task should be auto-created.-
(Requirement removed — SIU referral logic is handled downstream in the adjudication engine per compliance team review.)

## Solution

1. Add a conditional OmniScript step 'COB_OtherCoverageStep' to 'FNOL_IntakeScript_v2' using a Step Condition on the prior toggle element value.
2. Map all COB fields to Claim__c using a DataRaptor Transform ('DR_Transform_FNOL_COB') that writes to the Claim__c upsert DataRaptor.
3. Update the Integration Procedure 'IP_FNOL_Submit' to include the COB transform in the action sequence and stamp TPA__c based on the running user's associated TPA_Organization__c custom field.
4. Add a Validation Rule on Claim__c: IF(COB_Indicator__c = TRUE && ISBLANK(Other_Carrier_Name__c) && ISPICKVAL(Status__c, 'In Adjudication'), TRUE, FALSE).
5. Update FNOL OmniScript version to v2.3 and publish to both the internal Claims Console app and the Agent Portal Experience Cloud site.

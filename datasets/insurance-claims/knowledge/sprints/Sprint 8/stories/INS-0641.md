## Description

As a Claims Operations Manager overseeing member services, I need the Policyholder 360 dashboard on the Policy__c record to display a real-time summary of the policyholder's active and recently adjudicated claims alongside their Coordination of Benefits (COB) status so that member service representatives can answer inbound calls without toggling between multiple screens.

Builds upon the Policy__c page layout work completed in INS-0589.

## Acceptance Criteria

Given a member service representative opens a Policy__c record,
When the Policyholder 360 dashboard component loads,
Then it must display the following claim summary metrics in a tile layout:
  - Count of Claim__c records where Claim__c.Policy__c = this policy AND Claim__c.Status__c IN ('FNOL Received', 'In Review', 'Pending Adjudication') labeled 'Open Claims'
  - Count of Claim__c records where Claim__c.Status__c = 'Adjudicated' AND Claim__c.Adjudication_Date__c >= LAST_90_DAYS labeled 'Recently Adjudicated'
  - Total outstanding balance: SUM(Claim__c.Member_Responsibility__c) where Status__c != 'Paid' labeled 'Member Balance'

Given the policy has COB configured (Policy__c.COB_Active__c = TRUE),
When the dashboard renders,
Then a COB indicator badge must appear showing the primary and secondary payer names (Policy__c.Primary_Payer__c, Policy__c.Secondary_Payer__c) and the COB coordination method (Policy__c.COB_Method__c: 'Standard' / 'Non-Duplication' / 'Carve-Out').

Given a member service rep clicks the 'Open Claims' tile,
When the drill-down action fires,
Then a filtered list view of Claim__c records matching the open claims criteria must open in a modal, displaying columns: Claim__c.Name, Claim__c.Date_of_Service__c, Claim__c.Provider__r.Name, Claim__c.Total_Billed_Amount__c, Claim__c.Status__c.

Given a Policy__c record with no associated Claim__c records,
When the dashboard renders,
Then all metric tiles must display '0' with a 'No claims on file' message — no error or blank tiles.

Given the dashboard is accessed by a user with the 'Member_Services_Read_Only' Permission Set Group,
When the Member Balance tile renders,
Then the dollar amount must be visible but the 'Initiate Payment' quick action must be hidden.

## Solution

Built the Policyholder 360 Claims Summary as an LWC component (policyholderClaimsSummary) using @wire with a custom Apex controller (PolicyholderDashboardController) that performs aggregate SOQL queries (COUNT, SUM) on Claim__c filtered by Policy__c Id. COB badge implemented as a child LWC (cobStatusBadge) reading Policy__c COB fields via @wire getRecord. Drill-down modal uses lightning-datatable inside a lightning-modal component. Permission-based action visibility controlled via a @wire getPermissionSetAssignments call in the parent LWC rendering the 'Initiate Payment' action conditionally. Component added to the Policy__c Lightning Record Page via App Builder in a collapsible sidebar region. All SOQL queries use WITH SECURITY_ENFORCED to enforce FLS.

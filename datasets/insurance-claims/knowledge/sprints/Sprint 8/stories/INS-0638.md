## Description

As a Provider Network Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record and trigger a DocuSign envelope for counter-signature when a Provider_Network_Agreement__c is within 90 days of its Effective_End_Date__c, so that contract renewals are initiated proactively and compliance SLAs are met without manual intervention.

This story fulfills Compliance Officer requirement CR-2024-047 mandating 90-day advance notice for all PNA renewals across Commercial and Medicare Advantage LOBs.

## Acceptance Criteria

Given a scheduled Apex job (PNAExpirationCheckScheduler) runs daily at 06:00 UTC,
When a Provider_Network_Agreement__c record has PNA__c.Status__c = 'Active' AND PNA__c.Effective_End_Date__c = TODAY + 90 days AND no open PCN exists (PCN__c.PNA__c = this record AND PCN__c.Status__c != 'Closed'),
Then a new Provider_Change_Notification__c must be created with:
  - PCN__c.Type__c = 'Renewal'
  - PCN__c.Status__c = 'Pending Provider Signature'
  - PCN__c.PNA__c = the triggering PNA Id
  - PCN__c.Provider__c = PNA__c.Provider__c
  - PCN__c.Notification_Date__c = TODAY
  - PCN__c.Due_Date__c = PNA__c.Effective_End_Date__c - 30 days

Given the PCN record is created,
When the PCN__c.Status__c transitions to 'Pending Provider Signature',
Then a DocuSign envelope must be sent to the email address on Provider__c.PersonEmail using the 'PNA_Renewal_Template' DocuSign template, with merge fields populated from the parent PNA__c and associated Fee_Schedule__c.

Given the DocuSign envelope is signed by the provider,
When the DocuSign callback webhook fires,
Then PCN__c.Status__c must update to 'Executed' and PNA__c.Renewal_Initiated__c (checkbox) must be set to TRUE.

Given the PCN__c.Due_Date__c passes without PCN__c.Status__c = 'Executed',
When the daily scheduler runs,
Then an escalation Task must be created and assigned to the PNA__c.Owner with Subject = 'URGENT: PNA Renewal Overdue — [Provider Name]' and the PCN__c.Status__c must update to 'Escalated'.

Given a PNA__c record with LOB__c = 'Medicare Advantage',
When the PCN is generated,
Then PCN__c.CMS_Notification_Required__c must be set to TRUE and a separate compliance Task must be created for the Compliance Officer role queue.

-Given PNA records with Status__c = 'Suspended', the scheduler should also generate PCNs- *(deprecated: suspended PNAs excluded per legal review, active-only in scope)*

## Solution

Implemented a Schedulable Apex class (PNAExpirationCheckScheduler) that calls PNAExpirationBatchProcessor (Batchable, scope 200) daily. The batch queries PNA records meeting the 90-day window using a dynamic SOQL date literal and performs a subquery to exclude PNAs with existing open PCNs. PCN creation and field population handled in a bulkified service class (PCNGenerationService). DocuSign envelope dispatch uses the DocuSign for Salesforce managed package API via a Platform Event (PCN_DocuSign_Send__e) consumed by an Apex trigger on Provider_Change_Notification__c to keep the batch transaction lean and avoid DocuSign callout limits. DocuSign callback handled by a custom REST endpoint (DocuSignCallbackController). Escalation logic implemented as a separate Record-Triggered Flow on PCN__c firing on schedule (Due_Date__c passed) using Flow's scheduled path. Custom Metadata Type (PNA_Expiration_Config__mdt) stores the 90-day and 30-day thresholds per LOB to avoid hardcoding.

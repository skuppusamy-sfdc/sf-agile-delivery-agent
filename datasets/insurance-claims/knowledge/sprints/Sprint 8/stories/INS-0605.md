## Description

As a Claims Operations Manager, I need an Agentforce-powered Next Best Action (NBA) panel embedded on the Claim__c Lightning Record Page that surfaces AI-recommended actions — such as 'Request Additional Documentation', 'Refer to SIU', 'Initiate COB Review', or 'Approve for Payment' — based on the claim's current attributes, LOB, coverage type, and historical adjudication patterns, so that I can make faster, more consistent adjudication decisions and reduce manual triage time during high-volume claim periods.

## Acceptance Criteria

Given a Claim__c record with Status__c = 'Under Review' is opened by a Claims Operations Manager,
When the Agentforce NBA panel loads on the record page,
Then the panel displays between 1 and 3 ranked recommended actions with: action label, confidence score (displayed as a percentage), and a one-sentence rationale derived from the Einstein prompt template.

Given a Claim__c where Claim__c.Total_Billed_Amount__c > $50,000 and Claim__c.SIU_Flag__c = TRUE,
When the NBA panel evaluates the claim,
Then 'Refer to SIU' is surfaced as the top-ranked recommendation with confidence score >= 85%.

Given a Claim__c where the insured has an active COB indicator (Policy__c.COB_Active__c = TRUE),
When the NBA panel renders,
Then 'Initiate COB Review' appears as a recommended action with a direct quick-action button that pre-populates a new Task record linked to the Claim__c.

Given a Claims Operations Manager clicks an NBA recommended action button,
When the action is executed,
Then an NBA_Interaction__c record is created logging: Claim__c.Id, recommended action label, confidence score, user decision (Accepted/Dismissed), and timestamp — for model feedback and compliance audit purposes.

-Given a Claim__c with Status__c = 'Closed', When the NBA panel loads, Then display top 3 historical actions taken on similar claims for reference- (Deprecated — historical comparison view descoped from Phase 1)

Given a user without the 'Agentforce_Claims_NBA' Permission Set,
When the Claim__c record page loads,
Then the NBA panel component is not visible and no Einstein API calls are made.

## Solution

Configure an Agentforce Agent Action using Einstein Generative AI with a custom Prompt Template (PT-ClaimsNBA-ActionRecommendation) referencing Claim__c fields: Status__c, LOB__c, Total_Billed_Amount__c, SIU_Flag__c, Diagnosis_Code__c (ICD-10), and Policy__c.COB_Active__c as grounding data inputs. Build an LWC (lwc-agentforce-claims-nba) that invokes the Agentforce Agent via the Einstein Platform API and renders ranked recommendations with confidence scores. Wire the LWC to the Claim__c record page via a custom Lightning App Builder component with a visibility filter tied to the 'Agentforce_Claims_NBA' custom permission. Create an Apex invocable action (AgentforceNBALogger) called on user interaction to persist NBA_Interaction__c records. Add NBA_Interaction__c to the Compliance Reporting dashboard. Configure the Permission Set (PS-Agentforce-ClaimsNBA) and add to the PSG-ClaimsOps-Manager Permission Set Group.

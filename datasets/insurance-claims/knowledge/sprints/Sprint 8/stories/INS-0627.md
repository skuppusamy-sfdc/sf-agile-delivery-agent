## Description

As a Claims Operations Manager, I need the Agentforce Claims AI assistant to automatically analyze incoming Claim__c records for anomaly patterns indicative of potential fraud and surface a recommended SIU (Special Investigations Unit) referral flag directly on the claim record page so that adjusters can make faster, data-informed decisions on whether to escalate claims to SIU without manually cross-referencing historical billing data.

Anomaly patterns to detect include: duplicate CPT billing within a rolling 30-day window for the same Provider__c and Member, unusually high Claim_Line__c counts (>15 lines per claim), billed amounts exceeding 300% of Fee_Schedule__c Allowed_Amount__c, and provider NPI appearing on the CMS Excluded Provider list.

This story depends on INS-0625 (Fee Schedule Engine) for the allowed-amount comparison.

## Acceptance Criteria

Given a new Claim__c is submitted with Claim_Status__c = 'Received',
When the Agentforce Claims AI action 'Evaluate_SIU_Risk' is invoked via a Record-Triggered Flow (async, after insert),
Then the AI evaluates the claim against all configured anomaly rules and returns a SIU_Risk_Score__c (0–100) and SIU_Risk_Rationale__c (text summary) within 10 seconds.

Given SIU_Risk_Score__c >= 75,
When the score is written back to the Claim__c record,
Then SIU_Referral_Recommended__c (checkbox) is set to true AND Claim__c.Status__c is updated to 'Pended — SIU Review' AND a Case is auto-created with RecordType = 'SIU_Investigation', linked to the Claim__c, and assigned to the SIU Queue.

Given SIU_Risk_Score__c is between 40 and 74,
When the score is written to the Claim__c,
Then SIU_Referral_Recommended__c = false but a warning banner is displayed on the Claim__c record page via an LWC component reading the SIU_Risk_Score__c field.

Given a Provider__c whose NPI__c appears in the CMS_Excluded_Provider__c custom object (synced from CMS feed),
When the AI action evaluates the claim,
Then the rule 'CMS Exclusion Match' is included in SIU_Risk_Rationale__c and SIU_Risk_Score__c receives a minimum floor of 80 regardless of other rule scores.

Given an adjuster reviews the AI recommendation and disagrees,
When the adjuster sets SIU_Referral_Override__c = true and enters SIU_Override_Reason__c,
Then the SIU Case is not created (or is cancelled if already created) and an audit record is written to Claim_Audit_Log__c with Action__c = 'SIU Override', Performed_By__c = current user, and Timestamp__c = now.

## Solution

Configure Agentforce Agent Action 'Evaluate_SIU_Risk' using Einstein AI with a custom Apex invocable action 'SIUAnomalyEvaluator.evaluate()' as the underlying tool. The invocable action performs SOQL aggregation queries for duplicate CPT detection, line count checks, and fee schedule variance using FeeScheduleValidationService (INS-0625). CMS exclusion lookup queries CMS_Excluded_Provider__c via indexed NPI__c field. Risk score aggregation uses a weighted scoring Map<String, Integer> defined in Custom Metadata 'SIU_Rule_Weight__mdt'. Results written back via Platform Event 'SIURiskEvaluated__e' consumed by a Record-Triggered Flow that updates Claim__c fields and triggers Case creation sub-flow. LWC 'siuRiskBanner' reads SIU_Risk_Score__c via wire adapter and conditionally renders amber/red alert using SLDS utility classes. Permission Set Group 'SIU_Adjuster_PSG' controls field-level access to SIU fields.

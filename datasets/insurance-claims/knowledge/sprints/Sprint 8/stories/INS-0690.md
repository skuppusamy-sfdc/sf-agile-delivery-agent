# INS-0690: Automate Group/Employer onboarding record creation from GSA execution via DocuSign envelope completed event

- **Labels**: Phase1, DocuSign, GroupOnboarding, Automation

## Description

As a Broker Relations Coordinator, I need Salesforce to automatically create and populate a Group_Service_Agreement__c (GSA) record — along with associated Policy__c shell records for each enrolled line of business — when a DocuSign envelope for a new employer group contract reaches 'Completed' status, so that the manual data entry step currently performed by the Broker Relations team is eliminated, onboarding cycle time is reduced, and downstream underwriting and credentialing workflows can be triggered without coordinator intervention.

**Business Context:** The current process requires a Broker Relations Coordinator to monitor a DocuSign shared inbox, download the signed GSA PDF, manually create the Group_Service_Agreement__c record in Salesforce, and then individually create Policy__c shell records for each LOB (Medical, Dental, Vision) enrolled by the employer group. This process takes an average of 47 minutes per new group and introduces a 1–2 business day lag before Underwriting Analysts receive their task assignments. Acme Insurance onboards approximately 60–80 new employer groups per month.

**Scope:** This story covers the automated record creation and field population only. Underwriting workflow task assignment is handled in INS-0694 (not yet in sprint). DocuSign envelope configuration (template mapping, custom fields) is handled as a sub-task of this story.

## Acceptance Criteria

**AC1 — DocuSign Envelope Trigger**
Given a DocuSign envelope with Envelope_Type__c = 'Group Service Agreement' reaches status 'Completed',
When the DocuSign for Salesforce managed package fires the dsfs__DocuSign_Status__c completed event,
Then a Platform Event (GSA_Envelope_Completed__e) must be published containing: EnvelopeId, GroupAccountId (mapped from DocuSign custom field 'SF_Account_Id'), SignedDate, and a serialized JSON blob of LOB enrollment flags (Medical__c, Dental__c, Vision__c) extracted from DocuSign envelope custom fields.

**AC2 — GSA Record Auto-Creation**
Given the GSA_Envelope_Completed__e Platform Event is received by the subscriber Apex handler,
When the handler processes the event,
Then a new Group_Service_Agreement__c record must be created with:
- GSA.Account__c = GroupAccountId from event payload
- GSA.Status__c = 'Active'
- GSA.Effective_Date__c = SignedDate from event payload
- GSA.DocuSign_Envelope_Id__c = EnvelopeId from event payload
- GSA.Renewal_Date__c = SignedDate + 365 days (anniversary renewal default)
- GSA.Created_By_Automation__c = TRUE (audit flag)
- GSA.Broker_of_Record__c = populated from Account.Broker_of_Record__c lookup on the employer Account record.
If a GSA record already exists for the Account with GSA.Status__c = 'Active', the automation must NOT create a duplicate; instead it must log a GSA_Duplicate_Alert__c record and notify the assigned Broker Relations Coordinator via Salesforce In-App Notification.

**AC3 — Policy Shell Record Creation per LOB**
Given the GSA record is successfully created per AC2,
When the LOB enrollment flags are parsed from the event payload,
Then one Policy__c shell record must be created per enrolled LOB where the flag = TRUE:
- Policy__c.Group_Service_Agreement__c = new GSA record Id
- Policy__c.Account__c = GroupAccountId
- Policy__c.Line_of_Business__c = 'Medical' | 'Dental' | 'Vision' (per flag)
- Policy__c.Status__c = 'Pending Underwriting'
- Policy__c.Effective_Date__c = GSA.Effective_Date__c
- -Policy__c.Assigned_Underwriter__c = auto-assigned via round-robin — deprecated, moved to INS-0694-
- Policy__c.Policy_Number__c must be auto-generated via the existing PolicyNumberSequence custom metadata sequence (format: 'POL-{YYYY}-{LOB_CODE}-{SEQ6}').
A minimum of 1 and maximum of 3 Policy__c records may be created per GSA.

**AC4 — Attached Document Handling**
Given the DocuSign envelope is completed,
When the automation runs,
Then the signed GSA PDF document must be retrieved from DocuSign via API callout and stored as a Salesforce File (ContentDocument) linked to the Group_Service_Agreement__c record with ContentDocument.Title = 'Signed GSA — {Account.Name} — {SignedDate}'.
The ContentDocument must also be shared (ContentDocumentLink) to the related employer Account record.

**AC5 — Error Handling and Dead Letter Logging**
Given any step in the automation (Platform Event processing, GSA creation, Policy creation, document retrieval) throws an exception,
When the error occurs,
Then the Apex handler must catch the exception and create an Integration_Error_Log__c record with fields: Error_Source__c = 'GSA Onboarding Automation', Error_Message__c = exception message, Related_Envelope_Id__c, and Retry_Eligible__c = TRUE.
And an email alert must be sent to the Broker_Relations_Queue (Group Email) with error details and envelope ID for manual fallback.
The failed event must not be retried automatically more than 3 times (retry counter tracked on Integration_Error_Log__c.Retry_Count__c).

**AC6 — Compliance Audit Trail**
Given a GSA and associated Policy__c records are created via automation,
When a Compliance Officer queries the GSA record,
Then the GSA.Created_By_Automation__c = TRUE flag must be visible on the GSA record detail page,
And the Field History Tracking on Group_Service_Agreement__c must capture the initial creation values for Status__c, Effective_Date__c, and Broker_of_Record__c.
And a ComplianceReporting dashboard tile 'Automated GSA Onboardings — MTD' must reflect the new record within 24 hours via scheduled report refresh.

## Solution

**Integration Architecture:**
1. **DocuSign Platform Event Bridge:** Configure the DocuSign for Salesforce managed package (dsfs) to publish a custom Platform Event (GSA_Envelope_Completed__e) on envelope completion using the DocuSign Connect webhook. Map DocuSign envelope custom fields ('SF_Account_Id', 'LOB_Medical', 'LOB_Dental', 'LOB_Vision') to Platform Event fields via the DocuSign Connect mapping configuration in Setup.
2. **Apex Platform Event Subscriber (GSAOnboardingEventHandler):** Implement an @future-safe Apex class that subscribes to GSA_Envelope_Completed__e via a Trigger on the Platform Event object. The handler will: (a) query Account record for Broker_of_Record__c, (b) check for duplicate active GSA via SOQL (SELECT Id FROM Group_Service_Agreement__c WHERE Account__c = :accountId AND Status__c = 'Active'), (c) invoke GSARecordFactory.createGSA() and PolicyShellFactory.createPolicies() service classes, (d) enqueue a Queueable Apex job (GSADocumentRetrievalQueueable) for the DocuSign PDF callout to avoid hitting callout-in-trigger restrictions.
3. **Policy Number Generation:** Leverage the existing PolicyNumberSequence__mdt Custom Metadata Type and PolicyNumberGenerator Apex utility (built in INS-0641) to generate Policy__c.Policy_Number__c values per LOB using the 'POL-{YYYY}-{LOB_CODE}-{SEQ6}' format.
4. **Document Retrieval Queueable (GSADocumentRetrievalQueueable):** Make a Named Credential-authenticated callout to the DocuSign eSignature REST API (GET /v2.1/accounts/{accountId}/envelopes/{envelopeId}/documents/combined) to retrieve the signed PDF, convert to ContentVersion/ContentDocument, and create ContentDocumentLink records for both the GSA and Account.
5. **Error Logging:** Implement try-catch blocks throughout the handler chain writing to Integration_Error_Log__c. Use a Custom Notification Type for Broker Relations Coordinator in-app alerts. Configure an Email Alert Action on Integration_Error_Log__c Record-Triggered Flow for queue email delivery.
6. **Field History & Compliance:** Enable Field History Tracking on Group_Service_Agreement__c for Status__c, Effective_Date__c, Broker_of_Record__c via Setup (Object Manager). Build a scheduled CRM Analytics (Tableau CRM) dataset refresh for the 'Automated GSA Onboardings — MTD' dashboard tile targeting the ComplianceReporting dashboard.

## Description

As a Provider Relations Specialist, I need to view a provider's active Provider_Network_Agreement__c (PNA) details and associated Fee_Schedule__c tier directly on the Provider 360 FlexCard so that I can quickly answer provider inquiries about contracted rates and network participation status without navigating to multiple related lists.

Extends INS-0599 (Provider 360 base FlexCard implementation).

## Acceptance Criteria

Given a Provider__c (Person Account) record is open in the Provider 360 app page,
When the PNA Summary card section loads,
Then the FlexCard must display: PNA.Contract_Status__c, PNA.Effective_Date__c, PNA.Termination_Date__c, PNA.Network_Tier__c, and the linked Fee_Schedule__c.Schedule_Name__c and Fee_Schedule__c.Rate_Methodology__c.

Given a Provider__c has multiple PNA records,
When the FlexCard renders the PNA section,
Then only PNA records WHERE PNA.Contract_Status__c = 'Active' AND PNA.Effective_Date__c <= TODAY AND (PNA.Termination_Date__c = null OR PNA.Termination_Date__c >= TODAY) should be displayed, sorted by PNA.Effective_Date__c descending.

Given a Provider__c has no active PNA,
When the PNA section loads,
Then display an inline message: 'No active network agreement on file. Contact Network Contracting.' with a quick action button linking to create a new PNA record.

Given a Provider Relations Specialist clicks the Fee Schedule tier badge on the FlexCard,
When the popover opens,
Then it should display Fee_Schedule__c.CPT_Code_Count__c, Fee_Schedule__c.LOB__c, and Fee_Schedule__c.Approval_Date__c without navigating away from the Provider 360 page.

-Given the FlexCard is viewed by a Credentialing Specialist, the Fee Schedule rate amounts (Fee_Schedule__c.Allowed_Amount__c) should be visible.- (Deprecated per security review — rate amounts restricted to Provider Network Specialist and above via Permission Set Group 'PSG_ProviderContracting'.)

Given any user without the PSG_ProviderContracting permission set group,
When the FlexCard loads the PNA section,
Then Fee_Schedule__c.Allowed_Amount__c and Fee_Schedule__c.Rate_Methodology__c fields must be masked and display '---' per FLS enforcement.

## Solution

Build on the existing OmniStudio FlexCard 'Provider360_PNA_Card' (created in INS-0599). (1) Add a new DataRaptor Extract 'DR_Extract_ActivePNA_ByProvider' with SOQL targeting Provider_Network_Agreement__c joined to Fee_Schedule__c, filtered by Contract_Status__c = 'Active' and date range logic. (2) Create a child FlexCard state 'PNA_Active' and 'PNA_None' to handle conditional rendering. (3) Implement a Flyout action using OmniScript 'OS_FeeScheduleDetail_Popover' to display fee schedule metadata on badge click. (4) Enforce FLS on Fee_Schedule__c.Allowed_Amount__c via the existing Permission Set Group PSG_ProviderContracting — no custom Apex needed as OmniStudio respects running-user FLS in DataRaptor. (5) Deploy to UAT sandbox and validate with Provider Relations Specialist persona test user.

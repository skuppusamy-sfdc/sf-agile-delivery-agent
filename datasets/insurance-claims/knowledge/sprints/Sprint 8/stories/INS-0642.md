## Description

**Bug Report — UAT Sprint 8**

During UAT testing of the Group/Employer Onboarding OmniScript (OS_GroupEmployerOnboarding), testers discovered that the Group_Service_Agreement__c record fails to be created when the employer selects more than 5 plan options in the multi-select plan selection step.

**Steps to Reproduce:**
1. Launch OmniScript OS_GroupEmployerOnboarding from the Account (Business Account / Employer) record page.
2. Complete Steps 1–3 (Employer Details, Contact Information, Billing Setup).
3. On Step 4 'Plan Selection', select 6 or more plans from the available plan multi-select picklist.
4. Click 'Next' to proceed to Step 5 'Agreement Terms'.
5. Complete Step 5 and click 'Submit'.
6. Observe: a generic error toast appears — 'An error occurred while saving your data. Please try again.'
7. No Group_Service_Agreement__c record is created. No error is logged in the standard Salesforce error logs.

**Expected Behavior:** The GSA record should be created successfully regardless of the number of plan selections (up to the maximum of 20 plans supported per business requirements). Each selected plan should generate a child GSA_Plan_Line__c record linked to the parent GSA.

**Actual Behavior:** GSA creation silently fails when 6+ plans are selected. Selecting 5 or fewer plans works correctly.

**Additional Notes:** The issue appears to be a DataRaptor Save element hitting an undocumented field mapping limit. Discovered in FS1. Affects Commercial and Medicare Advantage group onboarding flows.

## Solution

Root cause identified: the DataRaptor Save element (DR_Save_GSA_WithPlans) was using a single DR Save with an inline repeating node for GSA_Plan_Line__c child records. The OmniStudio DataRaptor has an implicit limit on inline JSON node depth when the repeating collection exceeds 5 items due to the way the element serializes the input JSON bundle. Fix: refactored the OmniScript submission logic to use an Integration Procedure (IP_CreateGSAWithPlans) instead of a direct DataRaptor Save. The Integration Procedure calls DR_Save_GSA_Header (creates the parent Group_Service_Agreement__c) followed by a looping DataRaptor Save (DR_Save_GSA_PlanLine) using the IP's loop element to iterate over each selected plan and create individual GSA_Plan_Line__c records. Error handling added via IP's error path to surface meaningful messages back to the OmniScript UI. Tested successfully with 20 plan selections in FS1.

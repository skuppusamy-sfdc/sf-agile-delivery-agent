## Description

As a Provider Network Specialist, I need automated notifications when a Provider_Network_Agreement__c (PNA) is approaching its expiration date so that I can proactively initiate renewal workflows and avoid network gaps that could impact claims adjudication for in-network providers.

Currently, specialists rely on manual calendar reminders or ad-hoc reports to track PNA expirations, resulting in missed renewals and retroactive termination scenarios that generate downstream claims payment errors.

## Acceptance Criteria

**AC1 — 90-Day Advance Notification**
Given a Provider_Network_Agreement__c record has PNA__c.Contract_End_Date__c populated and PNA__c.Status__c = 'Active',
When the scheduled notification job runs daily and the Contract_End_Date__c is exactly 90 days from today,
Then a Salesforce in-app notification and an email alert are sent to the Provider_Network_Agreement__c.Owner (Provider Relations Specialist) with a link to the PNA record and the provider's name from PNA__c.Provider__r.Name.

**AC2 — 30-Day Advance Notification**
Given the same conditions as AC1,
When Contract_End_Date__c is exactly 30 days from today and no renewal PNA with PNA__c.Status__c = 'Pending Approval' or 'Active' exists for the same Provider__c,
Then a high-priority in-app notification and escalation email are sent to both the record Owner and their manager (via User.ManagerId lookup).

**AC3 — Renewal Task Creation**
Given a 30-day notification fires,
When the notification is generated,
Then a Task is automatically created on the PNA__c record with Subject = 'PNA Renewal Required — [Provider Name]', Priority = 'High', ActivityDate = today + 7 days, and OwnerId = PNA__c.OwnerId.

**AC4 — Suppress Duplicate Notifications**
Given a notification has already been sent for a given PNA__c record at a given threshold (90-day or 30-day),
When the daily job runs again,
Then no duplicate notification is sent (tracked via PNA__c.Last_Notification_Sent__c date field and PNA__c.Notification_Threshold_Sent__c picklist).

**AC5 — Exclusion of Terminated Agreements**
Given a PNA__c record has Status__c = 'Terminated' or 'Expired',
When the daily job runs,
Then no notification is generated for that record regardless of Contract_End_Date__c value.

## Solution

1. Create a Scheduled Flow 'PNA_Expiry_Notification_Flow' running daily at 6:00 AM org time.
2. Flow uses Get Records to retrieve Provider_Network_Agreement__c WHERE Status__c = 'Active' AND Contract_End_Date__c = TODAY+90 OR TODAY+30 AND Last_Notification_Sent__c != TODAY.
3. For each matching record, use a Decision element to branch on 90-day vs 30-day threshold.
4. 90-day path: Create a Custom Notification via 'Send Custom Notification' Flow action using Notification Type 'PNA_Expiry_Alert'.
5. 30-day path: Create Custom Notification + Task record creation via Flow Create Records element + escalation email via Email Alert referencing the 'PNA_30Day_Escalation' Email Template.
6. After notification, update PNA__c.Last_Notification_Sent__c = TODAY and PNA__c.Notification_Threshold_Sent__c to the relevant picklist value to prevent duplicates.
7. Add fault paths logging to Error_Log__c custom object.

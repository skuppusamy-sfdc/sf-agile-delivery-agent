# INS-0672: Configure Agentforce Claims AI assistant to surface claim pend reasons and suggest resolution actions for Claims Operations queue

## Description

As a Claims Operations Manager, I need the Agentforce AI assistant embedded in the Claims Operations console to automatically surface the reason a Claim__c is pended and suggest a prioritized list of resolution actions so that my team can reduce average pend resolution time and handle higher claim volumes without additional headcount.

The assistant should leverage Claim__c and Claim_Line__c data, interpret Pend_Reason__c codes, and recommend next-best-actions grounded in our internal claims handling guidelines stored in the Salesforce Knowledge base.

## Acceptance Criteria

**AC1 — Pend Reason Interpretation**
Given a Claims Operations Manager opens a Claim__c record with Status__c = 'Pended',
When the Agentforce assistant panel loads,
Then the assistant must display a plain-language interpretation of Claim__c.Pend_Reason__c (e.g., 'Fee Schedule Exception' → 'This claim is pended because no matching CPT code was found in the provider's active fee schedule for the date of service.').

**AC2 — Resolution Action Suggestions**
Given the pend reason has been interpreted,
When the assistant generates recommendations,
Then it must surface 2–4 suggested actions ranked by resolution likelihood, sourced from Salesforce Knowledge articles tagged with the matching Pend_Reason_Category__c.
Each suggestion must include an actionable deep link where applicable (e.g., 'Update Fee Schedule' → navigates to Fee_Schedule__c record).

**AC3 — EOB and Payment History Context**
Given the claim has prior Explanation_of_Benefits__c or Payment__c records,
When the assistant is asked 'What is the payment history for this claim?',
Then the assistant must return a summarized response citing EOB__c.EOB_Number__c, Payment__c.Payment_Amount__c, and Payment__c.Payment_Date__c without exposing raw SOQL or internal field names in the response.

**AC4 — SIU Flag Awareness**
Given Claim__c.SIU_Referral_Flag__c = true,
When the assistant loads,
Then it must display a prominent disclaimer: 'This claim has been referred to the Special Investigations Unit. Resolution actions are restricted pending SIU review.' and suppress action suggestions.

**AC5 — Grounding and Hallucination Guard**
Given the assistant cannot find a matching Knowledge article for a pend reason,
When generating a response,
Then the assistant must respond: 'I could not find a documented resolution path for this pend reason. Please escalate to your Claims Operations Manager.' and must not fabricate resolution steps.

## Solution

Configure Agentforce Agent in Salesforce using Agent Builder. Define Agent Topic 'Claims Pend Resolution' with scope limited to Claim__c context. Create Agent Actions: (1) 'GetPendReasonInterpretation' — custom Apex Action that maps Pend_Reason__c picklist values to plain-language strings via Custom Metadata Type 'Pend_Reason_Interpretation__mdt'; (2) 'SuggestResolutionActions' — Einstein Search Grounding action querying Salesforce Knowledge filtered by Pend_Reason_Category__c tag; (3) 'GetClaimPaymentHistory' — Apex Action returning structured payment/EOB summary. SIU flag check implemented as pre-condition in Agent Topic instructions using Merge Field from record context. Hallucination guard configured via Agent Topic system prompt with explicit instruction: 'Only provide resolution guidance grounded in retrieved Knowledge articles. If no article found, use the standard escalation response.' Agent embedded in Claims Console via Agentforce Agent component on Claim__c record page, visible only to users with 'Claims_Operations' Permission Set Group. Prompt templates stored in Prompt Builder for version control.

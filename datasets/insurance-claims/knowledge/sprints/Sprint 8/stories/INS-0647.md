## Description

**Bug Report — UAT Sprint 8**

When generating an Electronic Remittance Advice (ERA) file for a Payment__c record linked to a Claim__c that contains more than 50 Claim_Line__c child records, the ERA generation job completes with a 'Success' status in the UI but produces an empty or truncated output file. No error is surfaced to the user or logged to the Application event log.

**Steps to Reproduce:**
1. Create a Claim__c record with LOB = 'Facility' and associate exactly 55 Claim_Line__c records with mixed CPT codes.
2. Adjudicate the claim — set Claim__c.Status__c = 'Adjudicated'.
3. Create a Payment__c record linked to the Claim__c with Payment__c.Payment_Method__c = 'EFT'.
4. Click the 'Generate ERA' quick action on the Payment__c record.
5. Observe the success toast: 'ERA file generated successfully.'
6. Download the generated ERA file from the Files related list.
7. Open the file — observe it contains data for only the first 49 Claim_Line__c records; lines 50–55 are absent.

**Expected Behavior:** ERA file contains all 55 Claim_Line__c records in the ANSI X12 835 format.

**Actual Behavior:** ERA file silently truncates at 49 lines. Payment__c.ERA_Generation_Status__c = 'Complete' despite incomplete output.

**Environment:** Full Sandbox (UAT). Reproducible when Claim_Line__c count > 49. Claims with ≤ 49 lines generate correctly.

## Solution

Root cause is a hardcoded SOQL query limit of 50 records in the 'ERAGenerationService' Apex class method 'fetchClaimLines()' — a legacy stub limit never removed. Fix: Replace the SOQL query with a QueryLocator-based pattern using Database.getQueryLocator() to handle large claim line sets. Implement chunked processing in batches of 200 using the Batchable interface. Update Payment__c.ERA_Generation_Status__c to 'Processing' during batch execution and 'Complete' only upon successful Finish() callback. Add Platform Event (ERA_Generation_Event__e) to push real-time status to the UI via LWC subscription. Implement error logging to a custom ERA_Error_Log__c object with fields: Error_Message__c, Claim_Id__c, Payment_Id__c, Timestamp__c. Add unit tests covering 0, 1, 50, 51, and 200 line scenarios.

## Description

As a Claims Operations Manager, I need a Coordination of Benefits (COB) summary panel on the Policyholder 360 page (Policy__c record page) that displays the member's primary and secondary coverage details, active payer order, and any open COB disputes so that adjusters can quickly verify coverage order before processing claims without switching to external systems.

## Acceptance Criteria

Given a Policy__c record has COB_Active__c = TRUE,
When a Claims Operations Manager views the Policyholder 360 page,
Then the COB Summary LWC panel renders with: Primary Payer name (Policy__c.Primary_Payer__c), Secondary Payer name (Policy__c.Secondary_Payer__c), Policy__c.COB_Verification_Date__c, and Policy__c.COB_Order__c (picklist: Primary/Secondary/Tertiary).

Given the COB panel is visible,
When there are open Claim__c records with Claim__c.COB_Dispute_Flag__c = TRUE linked to the Policy__c,
Then a 'COB Disputes' sub-section renders listing each disputed Claim__c.Name, Claim__c.Status__c, and Claim__c.Total_Billed_Amount__c with a hyperlink to the Claim record.

Given Policy__c.COB_Active__c = FALSE,
When the Policyholder 360 page loads,
Then the COB Summary panel displays a 'No Active COB on File' empty state and a 'Add COB' quick action button.

Given Policy__c.COB_Verification_Date__c is more than 365 days in the past,
When the COB panel renders,
Then a yellow warning banner displays: 'COB verification is overdue. Last verified: [date]. Please re-verify with member.'

-Given the adjuster role does not have COB edit permissions, all fields in the panel should be read-only with no action buttons.-

Given any user viewing the panel does not have the 'COB_Edit' custom permission,
When the panel loads,
Then all fields are rendered in read-only mode and the 'Add COB' / 'Edit COB' action buttons are hidden.

## Solution

Built as a custom LWC 'cobSummaryPanel' deployed on the Policy__c Lightning Record Page via App Builder. Component uses @wire(getRecord) to fetch Policy__c COB fields and a separate @wire Apex method 'COBSummaryController.getOpenCOBDisputes(policyId)' returning a list of disputed Claim__c records. Warning banner logic implemented using a computed getter comparing COB_Verification_Date__c against today's date using JavaScript Date arithmetic. Custom permission 'COB_Edit' checked via @wire(getPermissions) pattern using Lightning Data Service. Empty state implemented using the SLDS illustration component. LWC added to the 'Policyholder 360' App Builder page template in a dedicated 'Coverage Details' accordion section.

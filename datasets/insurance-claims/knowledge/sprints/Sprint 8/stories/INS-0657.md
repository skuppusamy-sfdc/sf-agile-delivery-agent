## Description

As a Provider Network Specialist, I need a dedicated Lightning dashboard that surfaces all Provider_Network_Agreement__c records approaching expiry and tracks the PNA renewal pipeline so that I can proactively manage contract renewals, prevent unintended network gaps, and ensure continuity of provider participation across all LOBs.

The dashboard should provide actionable visibility into contracts expiring in the next 30, 60, and 90 days, segmented by LOB and Network_Region__c, and include drill-down to individual PNA records.

## Acceptance Criteria

Given a Provider Network Specialist accesses the 'Provider Network Operations' Lightning App,
When the 'Contract Renewal Pipeline' dashboard loads,
Then the dashboard displays a minimum of 5 report components: (1) PNAs expiring in 0–30 days, (2) PNAs expiring in 31–60 days, (3) PNAs expiring in 61–90 days, (4) PNAs by renewal stage (donut chart), (5) PNA count by LOB and Network_Region__c (matrix report).

Given the dashboard is loaded,
When a user clicks on any count in the expiry components,
Then the underlying report opens filtered to show Provider_Network_Agreement__c records with fields: PNA__c.Provider__c, PNA__c.Network_Name__c, PNA__c.Termination_Date__c, PNA__c.LOB__c, PNA__c.Network_Region__c, PNA__c.Contract_Status__c, PNA__c.Assigned_Specialist__c.

Given a PNA__c.Termination_Date__c is within 30 days of today and PNA__c.Contract_Status__c != 'Renewal In Progress',
When the 30-day expiry report renders,
Then those records are highlighted (via conditional formatting) and PNA__c.Renewal_Alert_Sent__c is checked to indicate whether an automated renewal alert has been dispatched.

Given the dashboard is configured,
When it is shared with the Provider Network Specialist role,
Then access is controlled via a Dashboard Folder shared to the 'Provider_Network_Specialists' Public Group; individual PNA records remain subject to standard Sharing Rules based on OWD = 'Private' with role-based sharing.

-Given the dashboard includes a 'Fee Schedule Variance' component showing CPT code reimbursement rate changes, that component is visible to all users.- (Removed — Fee Schedule Variance reporting deferred to Phase 2 per INS-0644 scope reduction.)

## Solution

Create 5 custom report types based on Provider_Network_Agreement__c as the primary object. Build individual Salesforce Reports for each dashboard component using relative date filters (e.g., Termination_Date__c = 'Next 30 Days'). Add conditional highlighting rules in the 30-day expiry report using report formatting on Termination_Date__c. Assemble a Lightning Dashboard 'Contract_Renewal_Pipeline_Dashboard' with report chart components. Create a Public Group 'Provider_Network_Specialists' and share the dashboard folder to this group. Configure a scheduled dashboard refresh (daily at 7:00 AM). Build a companion Record-Triggered Flow 'PNA_Renewal_Alert_Flow' that fires when PNA__c.Termination_Date__c enters the 30-day window and PNA__c.Contract_Status__c is not 'Renewal In Progress', sending an email alert and setting PNA__c.Renewal_Alert_Sent__c = true.

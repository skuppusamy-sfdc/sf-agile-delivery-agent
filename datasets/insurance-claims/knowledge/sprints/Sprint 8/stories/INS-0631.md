## Description

As a Credentialing Specialist, I need the system to automatically generate a Provider_Change_Notification__c (PCN) record when a Credentialing_Case__c transitions to 'Approved' status so that downstream network teams and payers are notified of the newly credentialed provider without manual data entry, reducing turnaround time and CAQH data discrepancies.

This story covers Phase 1 scope: PCN auto-creation and internal notification only. EDI/external payer notification is deferred to Phase 2.

## Acceptance Criteria

Given a Credentialing_Case__c record's Status__c field is updated to 'Approved',
When the record-triggered automation fires,
Then a new Provider_Change_Notification__c record must be created with: PCN.Provider__c = Credentialing_Case__c.Provider__c, PCN.Notification_Type__c = 'Initial Credentialing', PCN.Effective_Date__c = Credentialing_Case__c.Approval_Date__c, PCN.Status__c = 'Pending Distribution', PCN.Source_Case__c = Credentialing_Case__c.Id.

Given the PCN record is created,
When creation succeeds,
Then an in-app notification must be sent to all users with the role 'Provider Network Specialist' via Salesforce Bell Notification referencing the Provider__c.Name and PCN.Id.

Given a Credentialing_Case__c is updated to 'Approved' but a PCN with PCN.Source_Case__c = this case Id already exists (duplicate prevention),
When the automation evaluates,
Then no new PCN record should be created and a debug log entry 'PCN_DUPLICATE_SKIPPED' should be written.

Given a Credentialing_Case__c transitions from 'Approved' back to 'Under Review' (re-review scenario),
When the status change is saved,
Then the existing PCN.Status__c should be updated to 'On Hold' automatically.

-Given the Credentialing_Case__c.LOB__c = 'Medicare Advantage', trigger a CMS notification API call on PCN creation.- (Descoped to Phase 2 — INS-0714.)

## Solution

Implement a Record-Triggered Flow 'RTF_CredCase_Approved_CreatePCN' on Credentialing_Case__c, triggering on Update WHERE Status__c changes to 'Approved' (entry condition: ISCHANGED(Status__c) AND Status__c = 'Approved'). Flow steps: (1) Get Records — query existing PCN WHERE Source_Case__c = {!$Record.Id} to check for duplicates. (2) Decision element: if duplicate found, log via custom Log__c object and exit. (3) Create Records — instantiate Provider_Change_Notification__c with mapped fields. (4) Custom Notification action using the 'Bell Notification' core action targeting the 'Provider Network Specialist' public group. (5) Separate flow path handles the 'Approved' → 'Under Review' rollback: RTF updates PCN.Status__c = 'On Hold' via a second Get + Update Records path. Ensure flow is set to run in System Context with Sharing to respect OWD on PCN object.

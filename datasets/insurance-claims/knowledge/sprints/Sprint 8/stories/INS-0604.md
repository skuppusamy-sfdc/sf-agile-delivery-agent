## Description

**Bug Report — UAT Sprint 8**

The Explanation_of_Benefits__c (EOB) generation process throws an unhandled NullPointerException in Apex class `EOBGeneratorService` when any related Claim_Line__c record has a null or blank CPT_Code__c field. This causes the entire EOB batch job to fail, preventing EOB generation for all claims in the batch — including those with fully populated claim lines.

**Steps to Reproduce:**
1. Create a Claim__c record with Status__c = 'Adjudicated'.
2. Create two Claim_Line__c records linked to the Claim__c: one with a valid CPT_Code__c (e.g., '99213') and one with CPT_Code__c = null.
3. Trigger the EOB generation batch job (BatchEOBGenerator) via the Claims Operations Manager dashboard.
4. Observe the Apex job in Setup > Apex Jobs — status shows 'Failed'.
5. Review debug logs — `EOBGeneratorService.buildLineItemDescriptions()` throws `System.NullPointerException: Attempt to de-reference a null object` at line referencing `claimLine.CPT_Code__c.toUpperCase()`.

**Expected Behavior:** EOB generation should either (a) skip Claim_Line__c records with null CPT_Code__c and log a warning on the EOB record, or (b) substitute a configurable default description (e.g., 'Service Code Pending') and continue processing the batch without failure.

**Actual Behavior:** Entire batch job fails. No EOB records are created. Affected claims remain in 'Adjudicated' status indefinitely, blocking Payment__c disbursement.

**Frequency:** Reproducible 100% of the time when any claim line in the batch has a null CPT_Code__c. Affects approximately 4% of UAT test claims per QA team data.

**Environment:** Acme Insurance UAT Sandbox (API v59.0)

as per Description

## Solution

Fix `EOBGeneratorService.buildLineItemDescriptions()` method to add a null check before calling `.toUpperCase()` on `claimLine.CPT_Code__c`. Implement a null-safe helper method `sanitizeCPTCode(String cptCode)` that returns a configurable default label retrieved from Custom Metadata Type (EOB_Config__mdt) field `Default_Missing_CPT_Label__c` when CPT_Code__c is null or blank. Add a Claim_Line__c warning flag field (`EOB_Warning__c` checkbox) that is set to TRUE for lines where CPT substitution occurred. Log a warning entry to a custom EOB_Processing_Log__c object with fields: Claim__c lookup, Claim_Line__c lookup, Warning_Message__c, and Processed_Date__c. Ensure the batch class (BatchEOBGenerator) uses a try-catch at the individual claim scope (not batch scope) so a single problematic claim does not abort the entire batch. Add unit tests covering null CPT, blank CPT, and valid CPT scenarios with minimum 90% code coverage.

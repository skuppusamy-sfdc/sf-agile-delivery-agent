## Description

**Bug Report — UAT Cycle 3 Regression**

During UAT Sprint 8 regression testing, Claims Operations discovered that after an Electronic Remittance Advice (ERA) file is ingested and processed, individual Claim_Line__c records are correctly updated to Status__c = 'Adjudicated', but the parent Claim__c.Status__c field remains stuck at 'Pending Adjudication' and does not roll up to 'Adjudicated' as expected. This regression was introduced in the INS-0671 Payment Engine refactor.

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record in status 'Pending Adjudication' with 3 or more associated Claim_Line__c child records, all in status 'Pending'.
2. Trigger ERA file ingestion via the DataIntegration batch job (CMS ERA 835 inbound).
3. Confirm all Claim_Line__c records update to Status__c = 'Adjudicated' and Allowed_Amount__c is populated.
4. Navigate back to the parent Claim__c record.
5. Observe that Claim__c.Status__c still reads 'Pending Adjudication' and Claim__c.Total_Allowed_Amount__c is not recalculated.

**Expected Result:** Once all child Claim_Line__c records reach Status__c = 'Adjudicated', a Record-Triggered Flow or Apex rollup should fire and set Claim__c.Status__c = 'Adjudicated' and recalculate Claim__c.Total_Allowed_Amount__c as the sum of all Claim_Line__c.Allowed_Amount__c values.

**Actual Result:** Claim__c.Status__c = 'Pending Adjudication' persists. Claim__c.Total_Allowed_Amount__c = $0.00 or retains prior stale value. EOB generation is blocked downstream because the EOB trigger checks Claim__c.Status__c = 'Adjudicated' before firing.

**Impact:** High — EOB records (Explanation_of_Benefits__c) are not being auto-generated for adjudicated claims, blocking provider Payment__c disbursement and ERA reconciliation for approximately 340 claims in the UAT org.

## Acceptance Criteria

as per Description

Additional regression gate:
Given a Claim__c with all child Claim_Line__c.Status__c = 'Adjudicated',
When the ERA batch job completes processing,
Then Claim__c.Status__c must equal 'Adjudicated' within 60 seconds (async tolerance),
And Claim__c.Total_Allowed_Amount__c must equal the SUM of all related Claim_Line__c.Allowed_Amount__c values,
And an Explanation_of_Benefits__c record must be auto-created with EOB.Claim__c lookup populated and EOB.Status__c = 'Draft'.

## Solution

Root cause identified: the INS-0671 refactor moved Claim_Line__c status updates into a Platform Event (Claim_Line_Adjudicated__e) handler, which fires asynchronously. The existing Record-Triggered Flow on Claim_Line__c (after-save, 'All Records' entry criteria) that previously triggered the parent rollup was deactivated as part of that story but the Platform Event subscriber Apex class (ClaimLineAdjudicationEventHandler) was never updated to invoke the parent rollup logic.

**Fix Approach:**
1. Update ClaimLineAdjudicationEventHandler Apex class to call ClaimStatusRollupService.recalculate(claimIds) after processing each batch of Claim_Line_Adjudicated__e events.
2. ClaimStatusRollupService will query all Claim_Line__c children; if COUNT(Status__c = 'Adjudicated') = COUNT(ALL), set Claim__c.Status__c = 'Adjudicated' and recalculate Total_Allowed_Amount__c via a SUM aggregate SOQL.
3. Re-activate the downstream Record-Triggered Flow on Claim__c (after-update, when Status__c changes to 'Adjudicated') that creates the Explanation_of_Benefits__c draft record.
4. Add a Apex test class covering: partial adjudication (should NOT flip status), full adjudication (should flip), and mixed LOB scenarios (Medical vs. Dental Claim__c.Line_of_Business__c).
5. Deploy fix to UAT sandbox and re-execute regression suite INS-REG-047.

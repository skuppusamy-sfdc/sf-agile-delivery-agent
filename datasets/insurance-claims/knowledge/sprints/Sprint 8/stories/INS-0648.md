## Description

As a Claims Operations Manager, I need an Agentforce AI triage assistant embedded on the Claim__c record page during FNOL intake so that the system can automatically classify the claim by LOB and severity, suggest assignment to the appropriate claims examiner queue, and flag potential fraud indicators warranting SIU referral — reducing manual triage time and improving early fraud detection rates.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Status__c = 'FNOL Submitted',
When the Agentforce triage assistant activates on the record page,
Then within 5 seconds the assistant displays a recommended Claim__c.LOB__c classification based on the submitted loss description text and associated Policy__c.Product_Type__c.

Given the AI classification is displayed,
When the Claims Operations Manager accepts the recommendation,
Then Claim__c.LOB__c is updated to the AI-recommended value and Claim__c.AI_Triage_Accepted__c = true is recorded with the accepting user's ID and timestamp in Claim__c.AI_Triage_Accepted_By__c and Claim__c.AI_Triage_Timestamp__c.

Given the Agentforce assistant evaluates claim severity,
When the loss description or reserve amount (Claim__c.Initial_Reserve__c) exceeds the high-severity threshold defined in a custom metadata type (AI_Triage_Config__mdt.High_Severity_Reserve_Threshold__c),
Then the assistant recommends Claim__c.Severity__c = 'High' and pre-populates the assignment queue to 'Senior Examiner Queue'.

Given the AI model identifies two or more SIU fraud indicator patterns (e.g., provider flagged, late FNOL, duplicate CPT codes),
When the triage summary is rendered,
Then a prominent 'Potential Fraud — SIU Referral Recommended' banner is displayed with a one-click action to create a linked SIU referral task assigned to the SIU queue.

Given the Claims Operations Manager rejects the AI recommendation,
When they select 'Override' and manually set Claim__c.LOB__c,
Then Claim__c.AI_Triage_Override__c = true is recorded along with the override reason captured in Claim__c.AI_Override_Reason__c (required field on override).

Given any Agentforce API call fails or times out,
When the assistant cannot return a recommendation,
Then the UI displays 'AI triage unavailable — please classify manually' and the claim proceeds through standard manual triage without blocking FNOL submission.

## Solution

Implement Agentforce Agent configured with a 'Claims Triage' topic and custom Actions: (1) ClassifyClaimLOB — invokes an Apex action that calls the Einstein generative model via Connect API, passing Claim__c loss description and Policy__c.Product_Type__c as context. (2) EvaluateFraudIndicators — Apex action querying SIU fraud indicator rules stored in AI_Fraud_Rule__mdt custom metadata. Agent embedded in the Claim__c record page via an LWC wrapper component 'c-claims-triage-assistant' that subscribes to Agentforce response events. Acceptance/override logic handled by a Record-Triggered Flow (After Save) on Claim__c. AI_Triage_Config__mdt custom metadata type manages thresholds to avoid hardcoding. All AI interactions logged to a custom AI_Interaction_Log__c object for audit and model feedback purposes.

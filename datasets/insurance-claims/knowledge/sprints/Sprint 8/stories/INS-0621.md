## Description

As a Credentialing Specialist, I need a scheduled nightly sync with the CAQH ProView API to pull updated provider credentialing data (license status, DEA status, board certifications, malpractice history) and automatically update the corresponding Credentialing_Case__c and Provider__c profile fields so that credentialing decisions are based on current CAQH data without manual re-attestation requests.

This story covers the inbound sync only. Outbound provider enrollment submission to CAQH is scoped to INS-0629 (Phase 2).

## Acceptance Criteria

Given the nightly CAQH sync job runs at 02:00 AM EST,
When the Schedulable Apex job 'CAQHSyncScheduler' executes,
Then all Provider__c records where Provider__c.CAQH_Provider_ID__c is not null are included in the sync batch, processed in batches of 50 to respect CAQH API rate limits.

Given a CAQH API response is received for a provider,
When the sync processes the response payload,
Then the following Provider__c fields are updated: Provider__c.License_Status__c, Provider__c.License_Expiration_Date__c, Provider__c.DEA_Status__c, Provider__c.Board_Certification__c, Provider__c.Malpractice_Flag__c, and Provider__c.CAQH_Last_Sync_Date__c = NOW().

Given Provider__c.License_Status__c changes from 'Active' to 'Expired' or 'Suspended' during sync,
When the field update is committed,
Then an open Credentialing_Case__c linked to the provider has its Credentialing_Case__c.Status__c set to 'Suspended — License Issue' and a high-priority Task is created for the assigned Credentialing Specialist with Subject = 'CAQH Alert: License Status Change — ' + Provider__c.Name.

Given Provider__c.Malpractice_Flag__c changes to TRUE during sync,
When the update is processed,
Then Credentialing_Case__c.Requires_Medical_Director_Review__c = TRUE and the case is reassigned to the 'Medical Director Review' queue.

Given the CAQH API returns a 429 (rate limit) or 5xx error for a provider,
When the batch encounters the error,
Then the provider is logged to a CAQH_Sync_Error_Log__c record with fields: Provider__c (lookup), Error_Code__c, Error_Message__c, Sync_Timestamp__c, and the batch continues processing remaining records without failing.

Given the sync job completes,
When all records are processed,
Then a CAQH_Sync_Summary__c record is created with: Total_Processed__c, Total_Updated__c, Total_Errors__c, Sync_Date__c, and an email notification is sent to the Credentialing Operations distribution list using an Org-Wide Email Address.

Given Provider__c.CAQH_Provider_ID__c is present but CAQH returns 'Provider Not Found' (404),
When the sync processes this response,
Then Provider__c.CAQH_Sync_Status__c = 'Not Found — Manual Verification Required' and a Task is created for the Provider Network Specialist.

## Solution

Architecture: Schedulable Apex class 'CAQHSyncScheduler' invokes Batchable Apex 'CAQHSyncBatch' (implements Database.Batchable, Database.AllowsCallouts) with batch size 50. Each batch calls the CAQH ProView REST API endpoint '/api/provider/{caqhProviderId}/profilestatus' using a Named Credential 'CAQH_ProView_NC' for auth. JSON response deserialized via a custom Apex wrapper class 'CAQHProfileResponse'. Field mapping logic in a separate 'CAQHFieldMapper' utility class for maintainability. Error handling uses Database.SaveResult with partial success (allOrNothing=false). CAQH_Sync_Error_Log__c and CAQH_Sync_Summary__c records created in the batch finish() method. License status change detection compares incoming CAQH value against current Provider__c.License_Status__c before DML. Downstream Credentialing_Case__c updates triggered via a Record-Triggered Flow on Provider__c.License_Status__c change to avoid Apex-within-Apex callout restrictions. Named Credential and Remote Site Setting configured for CAQH ProView UAT endpoint.

## Description

As a Claims Operations Manager, I need a Policyholder 360 summary panel on the Policy__c record page so that I can quickly review a policyholder's active coverage details, open claims, and recent payment disbursements in a single view to support faster FNOL triage and claim ownership decisions.

This panel must be accessible to Claims Operations and read-only for Broker Relations Coordinators.

## Acceptance Criteria

**AC1 — Policy Summary Header**
Given a Claims Operations Manager opens a Policy__c record,
When the Policyholder 360 LWC component loads,
Then it must display Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, and Policy__c.Status__c in the header.
When Policy__c.Status__c = 'Lapsed' or 'Cancelled',
Then the header must render with a red status badge.

**AC2 — Open Claims Summary**
Given the policy has associated Claim__c records,
When Claim__c.Policy__c = current Policy__c.Id AND Claim__c.Status__c NOT IN ('Closed', 'Denied', 'Withdrawn'),
Then the panel must display a count of open claims and a list showing Claim__c.Claim_Number__c, Claim__c.Date_of_Loss__c, Claim__c.Claim_Type__c, and Claim__c.Status__c.
When no open claims exist,
Then display 'No Open Claims' with a 'File New Claim (FNOL)' action button.

**AC3 — Recent Payments**
Given Payment__c records exist linked to claims under this policy,
When Payment__c.Claim__r.Policy__c = current Policy__c.Id AND Payment__c.Payment_Status__c = 'Disbursed',
Then display the 5 most recent payments ordered by Payment__c.Payment_Date__c DESC, showing Payment_Amount__c, Payment_Date__c, and Payee_Name__c.

**AC4 — COB Indicator**
Given Policy__c.COB_Flag__c = true,
When the panel loads,
Then display a COB indicator badge with Policy__c.Primary_Payer_Name__c visible.

**AC5 — Broker Read-Only Access**
Given the logged-in user has the 'Broker_Relations_Coordinator' Permission Set Group,
When they view the Policyholder 360 panel,
Then all action buttons (FNOL, etc.) must be hidden,
And all financial fields (Payment_Amount__c, Allowed_Amount__c) must be masked per FLS configuration.

## Solution

Built as a custom LWC component 'policyholderSummaryPanel' using wire adapters and imperative Apex calls. ApexController 'PolicyholderSummaryController' exposes: (1) getPolicySummary(policyId) — returns Policy__c fields; (2) getOpenClaims(policyId) — SOQL with Status__c filter; (3) getRecentPayments(policyId) — cross-object SOQL via Claim__c relationship, LIMIT 5 ORDER BY Payment_Date__c DESC. Component uses @wire for initial load and refreshApex on FNOL action completion. Action button visibility controlled via @api userPermissions property populated by Apex using FeatureManagement.checkPermission(). FLS respected in all SOQL queries via WITH SECURITY_ENFORCED. Component added to Policy__c Lightning Record Page via App Builder in right sidebar, 1/3 width column. Deployed and validated in UAT — all 5 ACs passing.

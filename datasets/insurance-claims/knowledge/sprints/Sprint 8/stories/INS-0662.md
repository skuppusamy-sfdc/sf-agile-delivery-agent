## Description

**Bug Report — UAT Sprint 8**

When a Provider Relations Specialist attempts to submit a Provider_Change_Notification__c (PCN) as part of a Network_Termination_Case__c workflow for a facility-only provider (i.e., a provider whose record is a Facility__c / Business Account with no linked Provider__c Person Account), the submission fails with a validation error: 'Provider NPI is required.'

Facility-only providers use Facility_NPI__c on the Facility__c object, not the NPI__c field on Provider__c. The PCN validation rule does not account for this record type distinction.

**Steps to Reproduce:**
1. Open a Network_Termination_Case__c with Termination_Type__c = 'Facility' and a valid Facility__c lookup populated.
2. Click 'Submit PCN' action button.
3. The PCN creation screen appears; complete all required fields.
4. Click Save/Submit.
5. Error fires: 'Provider NPI is required.' even though Facility_NPI__c is populated on the linked Facility__c.

**Expected:** PCN submission succeeds for facility-only termination cases using Facility__c.Facility_NPI__c.
**Actual:** Validation error blocks submission; NPI sourced from Provider__c.NPI__c which is null for facility-only cases.

## Acceptance Criteria

as per Description

## Solution

1. Update the Validation Rule 'PCN_NPI_Required' on Provider_Change_Notification__c to: ISBLANK(Provider_NPI__c) && ISBLANK(Facility_NPI__c) — i.e., require at least one NPI field rather than mandating Provider_NPI__c unconditionally.
2. Update the 'SubmitPCN_Flow' Record-Triggered Flow to populate Provider_Change_Notification__c.Facility_NPI__c from the related Facility__c.Facility_NPI__c when Network_Termination_Case__c.Termination_Type__c = 'Facility'.
3. Update the PCN PDF generation Conga template to conditionally display either Provider NPI or Facility NPI based on termination type.
4. Regression test both provider-only and facility-only PCN submission paths in UAT.

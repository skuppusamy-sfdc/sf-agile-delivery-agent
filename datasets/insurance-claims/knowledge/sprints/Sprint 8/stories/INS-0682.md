## Description

As a Provider Network Specialist, I need the system to automatically validate that an active Fee_Schedule__c record exists and covers the PNA effective date range when a Provider_Network_Agreement__c envelope is completed in DocuSign, so that no PNA goes live without a corresponding fee schedule, preventing underpayment or claims processing failures.

The DocuSign envelope status webhook is already configured per INS-0631.

## Acceptance Criteria

Given a DocuSign envelope with envelope type 'PNA' reaches status 'Completed',
When the inbound Platform Event (DocuSign_Envelope_Event__e) is processed,
Then the system queries Fee_Schedule__c where Fee_Schedule__c.Provider__c = PNA.Provider__c AND Fee_Schedule__c.LOB__c = PNA.LOB__c AND Fee_Schedule__c.Effective_Start_Date__c <= PNA.Effective_Start_Date__c AND Fee_Schedule__c.Effective_End_Date__c >= PNA.Effective_End_Date__c.

Given a matching active Fee_Schedule__c record is found,
When validation passes,
Then Provider_Network_Agreement__c.Status__c is set to 'Active' and Provider_Network_Agreement__c.Fee_Schedule__c is populated with the matched record's Id.

Given no matching Fee_Schedule__c record is found,
When validation fails,
Then Provider_Network_Agreement__c.Status__c is set to 'Pending Fee Schedule' and a Task is created assigned to the owning Provider Network Specialist with Subject = 'Fee Schedule Missing — PNA [Agreement_Number__c]' and Due_Date = TODAY() + 3.

Given Provider_Network_Agreement__c.Status__c is set to 'Pending Fee Schedule',
When a new Fee_Schedule__c is created or updated that satisfies the date range criteria,
Then a Record-Triggered Flow re-evaluates the PNA and promotes Status__c to 'Active' automatically.

Given the PNA LOB__c = 'Medicare Advantage',
When fee schedule validation runs,
Then an additional check confirms Fee_Schedule__c.CMS_Approved__c = true before allowing Status__c = 'Active'.

## Solution

Platform Event subscriber (Apex trigger on DocuSign_Envelope_Event__e) invokes an invocable Apex method 'FeeScheduleValidationService.validatePNA()'. The method executes SOQL with date range overlap logic against Fee_Schedule__c. On failure, a Flow is called via Process Automation to create the follow-up Task. A secondary Record-Triggered Flow on Fee_Schedule__c (after insert/update) queries open 'Pending Fee Schedule' PNAs and re-invokes the validation service via @InvocableMethod. Medicare Advantage branch handled via a Flow Decision element checking LOB__c and CMS_Approved__c. Unit tests cover 4 scenarios with 95%+ coverage. Deployed via Copado.

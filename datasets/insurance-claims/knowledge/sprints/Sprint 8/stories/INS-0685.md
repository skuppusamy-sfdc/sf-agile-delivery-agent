## Description

As a Broker Relations Coordinator, I need a guided OmniScript for onboarding new group/employer accounts that automatically generates a Group_Service_Agreement__c, populates standard contract terms based on the selected LOB and group size tier, and routes the GSA to the appropriate signatories via DocuSign, so that onboarding cycle time is reduced from 10 days to under 3 days.

This story incorporates the Conga CLM template library configured in INS-0658 and the DocuSign envelope framework from INS-0631.

## Acceptance Criteria

Given a Broker Relations Coordinator launches the OmniScript 'OS_Group_Employer_Onboarding',
When the employer Account is selected,
Then the OmniScript validates that Account.Type = 'Employer/Group' and Account.TIN__c is populated before allowing progression to Step 2.

Given the coordinator selects LOB and group size (Group_Size_Tier__c: 'Small' 2–50, 'Mid-Market' 51–500, 'Large' 500+),
When the OmniScript reaches the 'Agreement Terms' step,
Then Group_Service_Agreement__c is pre-populated with standard fields: Contract_Term_Years__c, Renewal_Type__c, Admin_Fee_Percent__c, and TPA_Required__c based on a DataRaptor lookup against the GSA_Template_Config__mdt custom metadata type.

Given the GSA record is created,
When the coordinator clicks 'Send for Signature',
Then a DocuSign envelope is created with the Conga-generated GSA PDF attached, routed to: (1) Employer signatory (Contact role = 'Authorized Signer'), (2) Broker of Record (Contact role = 'Broker'), (3) Acme VP of Network Contracting (internal user).

Given the DocuSign envelope status reaches 'Completed',
When the Platform Event 'DocuSign_Envelope_Event__e' is received,
Then Group_Service_Agreement__c.Status__c is updated to 'Executed' and Group_Service_Agreement__c.Executed_Date__c = envelope completion date.

Given Group_Service_Agreement__c.Status__c = 'Executed',
When the record is updated,
Then a Record-Triggered Flow creates a follow-up Task for the Assignment Coordinator: Subject = 'Assign Benefits Administrator — [Account Name]', Due_Date = TODAY() + 1.

Given TPA_Required__c = true on the GSA,
When the GSA reaches 'Executed' status,
Then an additional Task is created for the Broker Relations Coordinator: Subject = 'Initiate TPA Agreement — [Account Name]' with a link to the TPA onboarding OmniScript.

## Solution

OmniScript 'OS_Group_Employer_Onboarding' with 7 steps built in OmniStudio. DataRaptor Extract retrieves GSA template configuration from GSA_Template_Config__mdt using LOB and Group_Size_Tier__c as keys. DataRaptor Upsert creates Group_Service_Agreement__c. Integration Procedure 'IP_Send_GSA_DocuSign' calls DocuSign eSignature REST API via Named Credential, attaching the Conga-generated PDF blob. Platform Event subscriber (Apex trigger on DocuSign_Envelope_Event__e) updates GSA status and fires a Record-Triggered Flow for Task creation. TPA conditional branch handled via Flow Decision on TPA_Required__c. Permission Set 'Broker_Relations_Full' required for OmniScript access, enforced via OmniScript visibility configuration.

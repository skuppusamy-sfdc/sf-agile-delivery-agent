## Description

As a Provider Network Specialist, I need an automated workflow on the Network_Termination_Case__c object that orchestrates the end-to-end provider termination process — including claims impact assessment, regulatory notification period enforcement, PCN generation, and PNA deactivation — so that terminations are executed compliantly and no in-flight claims are disrupted without proper handling.

Compliance requirement: CMS mandates a minimum 60-day written notice period for Medicare Advantage network terminations. State regulations vary; a configurable notice period by state is required.

## Acceptance Criteria

Given a Network_Termination_Case__c is created with Termination_Type__c = 'Voluntary' or 'Involuntary',
When the record is saved,
Then a Record-Triggered Flow calculates Network_Termination_Case__c.Notice_Period_End_Date__c as: TODAY() + Notice_Period_Days__c where Notice_Period_Days__c is sourced from Termination_Notice_Config__mdt filtered by State__c and LOB__c.

Given the termination case is created for a Medicare Advantage LOB provider,
When Notice_Period_Days__c is evaluated,
Then the system enforces a minimum of 60 days regardless of state configuration (CMS compliance rule).

Given Network_Termination_Case__c.Status__c = 'Approved',
When the approval is recorded,
Then the system queries all open Claim__c records where Claim__c.Rendering_Provider__c = Provider__c.Id AND Claim__c.Status__c NOT IN ('Adjudicated', 'Denied', 'Void') and populates Network_Termination_Case__c.Impacted_Open_Claims_Count__c.

Given Impacted_Open_Claims_Count__c > 0,
When the termination case moves to 'Approved' status,
Then a high-priority Task is created for the Claims Operations Manager: Subject = 'Review [N] In-Flight Claims — Provider Termination [Case Number]'.

Given the Notice_Period_End_Date__c is reached,
When a scheduled Apex job runs nightly,
Then all Provider_Network_Agreement__c records for the provider are updated: PNA__c.Status__c = 'Terminated', PNA__c.Actual_Termination_Date__c = TODAY(), and a Provider_Change_Notification__c (PCN) is auto-generated with PCN__c.Change_Type__c = 'Network Termination' and PCN__c.Effective_Date__c = TODAY().

Given the PCN is generated,
When PCN__c.Status__c = 'Generated',
Then an automated email notification is sent to the provider's primary contact using the 'Provider_Termination_Notice' email template, logged as an EmailMessage on the Network_Termination_Case__c.

## Solution

Record-Triggered Flow on Network_Termination_Case__c handles notice period calculation via Custom Metadata lookup (Termination_Notice_Config__mdt). CMS 60-day minimum enforced via a Decision element with formula: IF(LOB__c = 'Medicare Advantage' && Notice_Period_Days__c < 60, 60, Notice_Period_Days__c). Apex Invocable method 'ClaimsImpactAssessmentService.assessTermination()' performs SOQL aggregate query for open claims. Scheduled Apex class 'NetworkTerminationNightlyJob' implements Schedulable, queries termination cases where Notice_Period_End_Date__c <= TODAY() AND Status__c = 'Approved', bulk-updates PNAs, and calls a Flow to generate PCNs. Email notification sent via Apex SingleEmailMessage using OrgWideEmailAddress. All DML operations bulkified to handle large provider networks.

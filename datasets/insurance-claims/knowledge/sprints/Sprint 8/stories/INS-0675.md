## Description

As a Claims Operations Manager, I need a consolidated Policyholder 360 summary panel on the policyholder's Account record page so that I can quickly assess a member's active coverage, open claims, and COB (Coordination of Benefits) flags without navigating across multiple objects.

This panel should surface the most critical operational data points for a member in a single glance, reducing average handle time during FNOL intake and claims adjudication workflows. Extends INS-0643 (Policyholder base record layout).

## Acceptance Criteria

**AC1 — Active Policy Display**
Given a Claims Operations Manager views a policyholder Account record page,
When the Policyholder 360 panel loads,
Then it displays all related Policy__c records where Policy__c.Status__c = 'Active' and Policy__c.PolicyHolder__c = current Account Id, showing Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, and Policy__c.Termination_Date__c.

**AC2 — Open Claims Count and Quick Links**
Given the Policyholder 360 panel is rendered,
When there are Claim__c records linked to the policyholder with Claim__c.Status__c IN ('Submitted', 'In Review', 'Pending Additional Info'),
Then the panel displays a count badge and a scrollable list of up to 5 most recent open claims showing Claim__c.Claim_Number__c, Claim__c.Date_of_Service__c, Claim__c.Total_Billed_Amount__c, and a hyperlink to each Claim__c record.

**AC3 — COB Indicator**
Given a policyholder has COB_Flag__c = true on their Account record,
When the Policyholder 360 panel loads,
Then a prominent COB warning banner is displayed with the secondary payer name from Policy__c.Secondary_Payer__c and the COB_Order__c value.

**AC4 — No Active Policy State**
Given a policyholder has no Policy__c records with Status__c = 'Active',
When the panel loads,
Then an informational message 'No active policies on file' is displayed and the claims section is suppressed.

**AC5 — Performance**
Given the panel loads on an Account record page,
When data is fetched,
Then the panel renders within 2 seconds for accounts with up to 50 related Claim__c records (validated via Salesforce LWC wire adapter with @wire caching).

-**AC6 — Display SIU Flag on Panel** (deprecated — SIU indicators moved to separate restricted component per Compliance Officer request, see INS-0668)-

## Solution

1. Build a new LWC component 'policyholderSummaryPanel' using @wire(getRecord) and @wire(getRelatedListRecords) to fetch Policy__c and Claim__c data without imperative Apex where possible.
2. Use Apex @AuraEnabled(cacheable=true) method 'PolicyholderSummaryController.getSummaryData(accountId)' for the COB aggregation logic, returning a wrapper class with ActivePolicies, OpenClaims, and CobDetails.
3. Apply SLDS utility classes and lightning-badge for COB warning and open claim count indicators.
4. Add the component to the 'Claims Operations' Lightning App Builder page layout for Account, restricted via Permission Set Group 'Claims_Operations_PSG'.
5. Write LWC Jest tests for null/empty state rendering and Apex test class with 85%+ coverage targeting PolicyholderSummaryController.

## Description

As a Claims Operations Manager, I need an Agentforce AI triage agent to capture FNOL intake details and automatically score initial claim severity so that high-risk claims are immediately routed to senior adjusters while low-complexity claims proceed through straight-through processing without manual intervention. The agent should interpret unstructured claimant-submitted text, extract key loss details, and populate the Claim__c record accordingly.

## Acceptance Criteria

Given a claimant submits FNOL via the policyholder portal or inbound email channel,
When the Agentforce agent processes the submission,
Then a new Claim__c record is created with Claim__c.Status__c = 'FNOL Received', Claim__c.LOB__c populated from the linked Policy__c.Line_of_Business__c, and Claim__c.FNOL_Channel__c set to the originating channel.

Given the FNOL text contains injury descriptors or structural damage keywords,
When the AI severity scoring model evaluates the submission,
Then Claim__c.AI_Severity_Score__c is populated with a numeric value between 1–100 and Claim__c.AI_Severity_Tier__c is set to 'Low', 'Moderate', or 'High' based on configured thresholds (Low: 1–33, Moderate: 34–66, High: 67–100).

Given Claim__c.AI_Severity_Tier__c = 'High',
When the triage agent completes scoring,
Then the claim is automatically assigned to a senior adjuster queue via Omni-Channel routing and a Task is created with Subject = 'High Severity FNOL — Immediate Review Required' with ActivityDate = TODAY().

Given Claim__c.AI_Severity_Tier__c = 'Low' and Policy__c.Status__c = 'Active',
When no SIU flag is present (Claim__c.SIU_Referral__c = false),
Then the claim advances automatically to Claim__c.Status__c = 'Under Review' via straight-through processing without adjuster assignment.

Given the Agentforce agent cannot extract a required field (e.g., date of loss),
When confidence score for that field falls below 0.75,
Then the agent initiates a follow-up prompt to the claimant requesting clarification and sets Claim__c.FNOL_Incomplete__c = true.

-Given the agent populates ICD-10 codes directly from free-text injury descriptions, Then Claim__c.Primary_ICD10__c is auto-populated- (superseded by INS-1212 which handles clinical code extraction on Claim_Line__c).

Given a completed FNOL intake session,
When the Agentforce agent closes the conversation,
Then a full conversation transcript is stored as a ContentDocument linked to the Claim__c record and Claim__c.AI_Transcript_Available__c = true.

## Solution

Deploy an Agentforce Agent configured with a custom Agent Topic 'FNOL Intake & Triage' and associated Agent Actions backed by Apex invocable methods. A Record-Triggered Flow on Claim__c (trigger: after insert, Status__c = 'FNOL Received') fires a Platform Event (FNOL_Triage_Event__e) consumed by the severity scoring Apex service. The scoring service calls an Einstein ML model endpoint and writes back AI_Severity_Score__c and AI_Severity_Tier__c via a future method. Omni-Channel routing rules reference AI_Severity_Tier__c for queue assignment. Conversation transcripts are stored via ContentVersion insert in the Apex post-processing handler. A custom Permission Set Group 'Agentforce_FNOL_Agent' controls agent object access.

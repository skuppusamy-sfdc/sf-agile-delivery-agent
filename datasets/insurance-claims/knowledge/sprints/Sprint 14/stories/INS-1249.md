## Description

As a Credentialing Specialist, I need an automated workflow that monitors provider CAQH re-attestation due dates on Credentialing_Case__c records and triggers a DocuSign envelope 90, 60, and 30 days before expiration so that providers complete re-attestation on time and Credentialing_Case__c.Status__c remains current, avoiding network participation lapses.

## Acceptance Criteria

Given Credentialing_Case__c.CAQH_Reattestion_Due_Date__c is 90 days from today,
When the nightly scheduled Flow runs,
Then a DocuSign envelope is sent to Provider__c.Email (via DocuSign for Salesforce API) with the CAQH Re-attestation template and Credentialing_Case__c.DocuSign_Status__c = 'Sent – 90 Day'.

Given Credentialing_Case__c.DocuSign_Status__c = 'Sent – 90 Day' and the envelope has not been signed within 10 days,
When the scheduled Flow runs,
Then an escalation Task is created for the assigned Credentialing Specialist with Priority = 'High'.

Given the DocuSign envelope is completed by the provider,
When the DocuSign completion webhook fires,
Then Credentialing_Case__c.CAQH_Reattestion_Completed_Date__c = today, Credentialing_Case__c.Status__c = 'Re-attested', and Provider__c.Credentialing_Current__c = true.

Given Credentialing_Case__c.CAQH_Reattestion_Due_Date__c has passed and Status__c != 'Re-attested',
When the scheduled Flow runs,
Then Provider_Network_Agreement__c.Status__c is set to 'Suspended' and a Provider_Change_Notification__c (PCN) record is auto-created with PCN.Reason__c = 'Credentialing Lapsed'.

-Given provider opts out of DocuSign, send paper form via postal mail integration- (deprecated: postal mail integration descoped from Phase 2)

## Solution

Scheduled Flow ('CAQH_Reattestion_Reminder_Flow') running nightly, querying Credentialing_Case__c records with CAQH_Reattestion_Due_Date__c in 90, 60, and 30 days using formula-based date filters. DocuSign envelope triggered via DocuSign for Salesforce Apex toolkit (DSFSEnvelopeService). Webhook handler (DocuSignWebhookController.cls) processes completion callbacks and updates Credentialing_Case__c via a Record-Triggered Flow. PCN auto-creation handled by a separate Apex trigger on Credentialing_Case__c (after update, status lapse condition). Permission Set 'Credentialing_Automation_PS' grants FLS for all modified fields.

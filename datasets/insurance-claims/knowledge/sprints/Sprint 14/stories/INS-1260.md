## Description

As a Claims Operations Manager, I need the system to automatically generate an 835 ERA file upon batch payment processing and reconcile resulting Payment__c records against adjudicated Claim_Line__c entries so that manual ERA posting is eliminated and payment accuracy is verifiable end-to-end. Extends INS-1229 (Payment batch scheduling framework).

## Acceptance Criteria

Given a Payment__c record with Payment__c.Status__c = 'Approved' and Payment__c.Payment_Method__c = 'EFT'
When the nightly payment batch job executes
Then an 835 ERA file is generated and stored as a ContentDocument linked to the Payment__c record
And Payment__c.ERA_File_Generated__c = true and Payment__c.ERA_Generation_Date__c = today

Given the ERA file is generated
When auto-reconciliation runs
Then each Claim_Line__c.Paid_Amount__c is updated to match the ERA remittance detail
And Claim_Line__c.Reconciliation_Status__c = 'Matched'

Given a Claim_Line__c where the ERA paid amount differs from Claim_Line__c.Adjudicated_Amount__c by more than $0.01
When reconciliation runs
Then Claim_Line__c.Reconciliation_Status__c = 'Variance Detected'
And a Case is auto-created with RecordType = 'Payment Dispute' linked to the parent Claim__c

Given Payment__c.Payment_Method__c = 'Check'
When the batch job executes
Then ERA generation is skipped and Payment__c.ERA_File_Generated__c = false
And Payment__c.Check_Number__c must be populated before Status__c can transition to 'Disbursed'

Given all Claim_Line__c records on a Claim__c have Reconciliation_Status__c = 'Matched'
When reconciliation completes
Then Claim__c.Status__c is updated to 'Closed – Paid' and EOB generation is triggered

## Solution

Implement an Apex Batchable class (ERAGenerationBatch.cls) scheduled via Scheduled Apex to run nightly at 02:00 UTC. The batch queries Payment__c records in 'Approved' status with EFT method, constructs 835 EDI content using a custom EDI835Builder utility class, and saves output as ContentVersion linked to Payment__c via ContentDocumentLink. A second Apex Batchable (PaymentReconciliationBatch.cls) chained via Database.executeBatch in the finish() method performs line-level matching. Variances trigger a Record-Triggered Flow on Claim_Line__c to create dispute Cases. The Claim__c status rollup uses a Roll-Up Summary equivalent via Apex trigger to check all child Claim_Line__c reconciliation statuses.

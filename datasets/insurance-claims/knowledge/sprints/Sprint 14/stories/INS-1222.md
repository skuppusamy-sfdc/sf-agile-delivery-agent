## Description

As a Claims Operations Manager, I need the Agentforce AI Copilot to suggest appropriate EOB denial reason codes on Claim_Line__c records during adjudication review so that adjusters can apply consistent, compliant denial language faster and reduce rework from incorrect codes.

The copilot must leverage historical adjudication data from closed Claim__c records and align suggestions with the Fee_Schedule__c and Provider_Network_Agreement__c (PNA) in effect at the date of service.

## Acceptance Criteria

Given an adjuster opens a Claim__c record with Status__c = 'In Adjudication' and one or more Claim_Line__c records with Adjudication_Status__c = 'Pending Review',
When the Agentforce Copilot panel loads on the Claim detail page,
Then the copilot must display up to 3 ranked EOB denial reason code suggestions per Claim_Line__c, each with a confidence percentage and a plain-language rationale.

Given the Claim_Line__c.CPT_Code__c matches a service listed on the active Fee_Schedule__c linked to the associated PNA,
When the copilot evaluates coverage,
Then suggestions must include Fee_Schedule__c.Allowed_Amount__c vs. Claim_Line__c.Billed_Amount__c variance as a rationale signal.

Given the adjuster accepts a suggested denial code,
When they click 'Apply' in the copilot panel,
Then Claim_Line__c.EOB_Denial_Code__c must be populated, Claim_Line__c.Denial_Rationale__c must be stamped with the AI-generated text, and Claim_Line__c.Adjudication_Status__c must transition to 'Denied – Pending EOB'.

Given the adjuster overrides the copilot suggestion with a manual code,
When they save the record,
Then Claim_Line__c.AI_Suggestion_Overridden__c must be set to TRUE and Claim_Line__c.Override_Reason__c must be required (validation rule enforced).

Given any copilot suggestion interaction,
When the session ends,
Then an AI_Adjudication_Feedback__c record must be created capturing: Claim_Line_Id__c, Suggested_Code__c, Accepted__c (boolean), Override_Code__c, Adjuster_Id__c for model retraining pipeline.

## Solution

Build an Agentforce Copilot Action 'Suggest_EOB_Denial_Codes' backed by a prompt template that includes: Claim_Line__c context fields, related Fee_Schedule__c allowed amounts (retrieved via SOQL in an Apex invocable), and a RAG (Retrieval Augmented Generation) grounding dataset of the top 500 most-used denial code combinations from historical Claim_Line__c records. The copilot panel is surfaced as an LWC embedded in the Claim__c Lightning Record Page using the einstein_copilot component slot. Field updates on accept/override handled by a Record-Triggered Flow. AI_Adjudication_Feedback__c creation handled by an Apex Trigger on Claim_Line__c (after update) to ensure bulkification.

## Description

As a Claims Operations Manager, I need the Agentforce AI agent to automatically triage incoming FNOL submissions and assign a severity score to each new Claim__c record so that claims are routed to the correct adjudication queue without manual intervention, reducing initial triage time by at least 40%.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Status__c = 'FNOL Received' and Claim__c.LOB__c is populated,
When the Agentforce AI triage action is triggered via Platform Event (FNOL_Submitted__e),
Then Claim__c.AI_Severity_Score__c (numeric 1–100) must be populated within 10 seconds of record creation.

Given Claim__c.AI_Severity_Score__c >= 75,
When the severity threshold is met,
Then Claim__c.Priority__c must be set to 'Critical' and Claim__c.OwnerId must be reassigned to the 'High Severity Claims' queue automatically.

Given Claim__c.AI_Severity_Score__c between 40 and 74,
When the severity threshold is evaluated,
Then Claim__c.Priority__c must be set to 'Standard' and routed to the 'General Adjudication' queue.

Given Claim__c.AI_Severity_Score__c < 40,
When the severity threshold is evaluated,
Then Claim__c.Priority__c must be set to 'Low' and Claim__c.Status__c must transition to 'Auto-Adjudication Eligible'.

Given any FNOL submission where Claim__c.Suspected_Fraud__c = true,
When the AI triage evaluates the record,
Then the claim must be flagged with Claim__c.SIU_Referral_Required__c = true and a Task must be created and assigned to the SIU queue regardless of severity score.

-Given Claim__c.LOB__c = 'Workers Comp', a separate severity matrix must be applied (deprecated in favor of unified model per INS-1183)-

Given the AI model is unavailable or returns an error,
When the Platform Event processing fails,
Then Claim__c.AI_Triage_Status__c must be set to 'Manual Review Required' and an alert email must be sent to the Claims Operations Manager.

## Solution

Deploy an Agentforce Agent Action backed by an Apex invocable method (ClaimTriageAIService.cls) that consumes the FNOL_Submitted__e Platform Event. The Apex class calls the Einstein AI Model API endpoint configured in Named Credentials. A Record-Triggered Flow on Claim__c (After Insert, Status__c = 'FNOL Received') publishes the Platform Event. Severity score thresholds and queue routing logic are managed via Custom Metadata Type (AI_Triage_Config__mdt) to allow ops team adjustments without code deploys. SIU flagging uses a separate Apex trigger (ClaimSIUFlagTrigger.cls) with its own bypass flag in Custom Settings. Error handling writes to a custom AI_Processing_Log__c object for audit.

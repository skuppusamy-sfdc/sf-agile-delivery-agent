## Description

As a Credentialing Specialist, I need the system to automatically trigger a CAQH re-attestation reminder workflow when a provider's Credentialing_Case__c record is within 90, 60, and 30 days of its expiration date so that providers are proactively notified and credentialing lapses that could disrupt claims payment are prevented.

## Acceptance Criteria

Given a Credentialing_Case__c record where Credentialing_Case__c.Status__c = 'Active' and Credentialing_Case__c.Credential_Expiry_Date__c is populated,
When a scheduled daily Flow job evaluates expiry dates,
Then automated reminder Tasks must be created at the 90-day, 60-day, and 30-day marks, each assigned to the Credentialing_Case__c.OwnerId.

Given the 30-day reminder threshold is reached,
When the Task is created,
Then Task.Priority must be set to 'High' and Task.Subject must follow the pattern 'CAQH Re-Attestation Required - {Provider NPI} - Expires {Date}'.

Given a reminder Task is created,
When the workflow runs,
Then an automated outbound email must be sent to Provider__c.NPI_Contact_Email__c using the 'CAQH_ReAttestation_Reminder' email template, with the provider's Credentialing_Case__c.CAQH_Provider_ID__c included in the body.

Given Credentialing_Case__c.Credential_Expiry_Date__c has passed and Credentialing_Case__c.Status__c is still 'Active',
When the daily job evaluates the record,
Then Credentialing_Case__c.Status__c must be updated to 'Expired' and all associated Provider_Network_Agreement__c records where PNA__c.Provider__c = that provider must have PNA__c.Credentialing_Hold__c set to true.

Given PNA__c.Credentialing_Hold__c = true,
When a new Claim__c is submitted with Claim__c.Billing_Provider__c referencing that provider,
Then Claim__c.Status__c must be set to 'Pended - Credentialing Hold' and Claim__c.Pend_Reason__c = 'Provider Credentialing Expired'.

## Solution

Scheduled Flow (Credentialing_Expiry_Reminder_Flow) runs daily at 6 AM UTC, querying Credentialing_Case__c records via Get Records where Status__c = 'Active' and Credential_Expiry_Date__c <= TODAY() + 90. Decision elements branch on days-remaining buckets (90/60/30/0). Task creation and email sends are handled within the Flow using Send Email action with the org-wide address. The CAQH_Provider_ID__c is passed into the email template via merge fields. PNA hold logic is a separate sub-flow (PNA_CredentialingHold_SubFlow) called from the main flow. Claim pend logic is enforced via an Apex Trigger (ClaimSubmissionValidationTrigger.cls) that checks PNA__c.Credentialing_Hold__c at claim creation time. CAQH integration for status sync uses a DataRaptor Load mapped to the CAQH REST API via External Services.

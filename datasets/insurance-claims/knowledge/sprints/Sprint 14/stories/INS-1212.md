## Description

As a Claims Operations Manager, I need the Agentforce AI agent to extract CPT procedure codes and ICD-10 diagnosis codes from attached medical records and clinical notes and automatically populate them on the corresponding Claim_Line__c records so that manual coding effort is reduced, coding accuracy improves, and downstream adjudication against the Fee_Schedule__c can proceed without adjuster intervention. This story fulfills the clinical code extraction requirement superseded on INS-1207.

## Acceptance Criteria

Given a Claim__c record in Status__c = 'Under Review' has one or more ContentDocument attachments classified as ContentDocument.Type__c = 'Medical Record' or 'Clinical Note',
When the Agentforce clinical extraction agent action is invoked,
Then the agent processes each document and returns a structured JSON payload containing extracted CPT codes, ICD-10 codes, date of service, and rendering provider NPI.

Given the extraction payload is returned with confidence >= 0.85 per code,
When the Apex code-population handler processes the result,
Then each extracted CPT code is matched against Fee_Schedule__c.CPT_Code__c and a Claim_Line__c record is upserted with Claim_Line__c.CPT_Code__c, Claim_Line__c.ICD10_Primary__c, Claim_Line__c.ICD10_Secondary__c (if present), Claim_Line__c.Date_of_Service__c, and Claim_Line__c.AI_Extracted__c = true.

Given a CPT code extracted by the agent does not exist in any active Fee_Schedule__c for the billing provider's network,
When the code-population handler performs the Fee Schedule lookup,
Then Claim_Line__c.Fee_Schedule_Match__c = false and Claim_Line__c.Status__c = 'Pending Manual Review' with a warning notation logged in Claim_Line__c.Adjuster_Notes__c.

Given extraction confidence for any code falls below 0.85,
When the low-confidence result is returned,
Then the affected Claim_Line__c is created with Claim_Line__c.AI_Confidence_Score__c populated and Claim_Line__c.Status__c = 'Low Confidence — Coder Review' and the claim is added to the medical coding review queue.

Given all Claim_Line__c records on a Claim__c are in Status__c = 'Ready for Adjudication',
When the last line is updated,
Then a Record-Triggered Flow rolls up the status and sets Claim__c.Status__c = 'Ready for Adjudication' automatically.

Given Claim_Line__c.AI_Extracted__c = true,
When a human coder modifies Claim_Line__c.CPT_Code__c or Claim_Line__c.ICD10_Primary__c,
Then Claim_Line__c.AI_Override__c = true and Claim_Line__c.Override_Reason__c is required (validation rule enforced) for audit trail purposes.

## Solution

Agentforce Agent Action 'Extract_Clinical_Codes' calls an external Einstein NLP model endpoint via a Named Credential using a queueable Apex class (ClinicalCodeExtractionQueueable). The model is fine-tuned on CPT/ICD-10 medical coding corpora. Extracted JSON is deserialized in Apex and matched against Fee_Schedule__c via SOQL. Claim_Line__c records are upserted using an external ID field Claim_Line__c.Source_Line_Reference__c. A Record-Triggered Flow on Claim_Line__c handles the rollup to Claim__c.Status__c using a custom roll-up helper. A Validation Rule on Claim_Line__c enforces Override_Reason__c when AI_Override__c is set to true. The medical coding review queue is an Omni-Channel queue with a custom LWC work item card.

## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces credentialing status, active Provider_Network_Agreement__c records, open Claim__c counts, and any pending Provider_Change_Notification__c items so that I can assess a provider's full relationship with the network in a single view without navigating across multiple objects.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page
When the Provider 360 FlexCard ('Provider_360_FC') loads
Then the card must display:
  - Credentialing_Case__c.Status__c (most recent active case)
  - Credentialing_Case__c.CAQH_Profile_Status__c
  - Count of active Provider_Network_Agreement__c records where PNA.Status__c = 'Active' and PNA.Effective_End_Date__c >= TODAY
  - Count of open Claim__c records where Claim__c.Billing_Provider__c = Provider__c.Id AND Claim__c.Status__c NOT IN ('Paid', 'Denied', 'Void')
  - Count of pending Provider_Change_Notification__c where PCN.Status__c = 'Pending Review'

Given the provider has a Credentialing_Case__c.Status__c = 'Expired' or 'Suspended'
When the FlexCard renders
Then a red banner 'Credentialing Alert' must be displayed prominently at the top of the card
And a direct link to the active Credentialing_Case__c must be shown

Given the provider has active PNA records
When the specialist clicks 'View Network Agreements'
Then a child FlexCard panel must expand showing each PNA: Provider_Network_Agreement__c.Network_Name__c, PNA.Contract_Type__c, PNA.Effective_Start_Date__c, PNA.Effective_End_Date__c, and PNA.Fee_Schedule__r.Name

Given a Provider_Change_Notification__c.PCN_Type__c = 'Address Change' or 'TIN Change' is pending
When the FlexCard loads
Then a yellow 'Data Change Pending' badge must appear
And the specialist must be able to launch the PCN review OmniScript directly from the badge action

Given no open claims or pending PCNs exist
When the FlexCard loads
Then the respective sections must display 'None' with a green checkmark indicator

## Solution

Build OmniStudio FlexCard ('Provider_360_FC') using multiple DataRaptor Extract operations (DR_GetProviderCredentialing, DR_GetActivePNAs, DR_GetOpenClaims, DR_GetPendingPCNs) aggregated via a DataRaptor Turbo Extract for performance. Child panel for PNA details uses a conditional FlexCard state. PCN review action launches OmniScript ('PCN_Review_OS'). FlexCard deployed via Lightning App Builder onto the Provider__c (Person Account) record page. Access controlled via Permission Set Group 'Provider_Relations_PSG'. Caching applied (15-minute TTL) on DataRaptor Extracts to reduce SOQL load.

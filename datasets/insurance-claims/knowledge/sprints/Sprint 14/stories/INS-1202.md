## Description

**Bug Report – INS-1202**

**Summary:** When a Fee_Schedule__c record is updated (e.g., allowed amount change due to a contract amendment) while a related Claim__c is in 'Under Review' status, the associated Claim_Line__c.Allowed_Amount__c fields are NOT recalculated. Adjusters are unknowingly adjudicating claims against stale fee schedule data, resulting in incorrect payment amounts.

**Steps to Reproduce:**
1. Create a Claim__c with Status__c = 'Under Review' and two Claim_Line__c child records, each referencing a Fee_Schedule__c via Claim_Line__c.Fee_Schedule__c lookup.
2. Note the current Claim_Line__c.Allowed_Amount__c values (e.g., $150.00 and $200.00).
3. Navigate to the referenced Fee_Schedule__c record and update Fee_Schedule__c.Allowed_Amount__c from $150.00 to $175.00.
4. Return to the Claim_Line__c records.
5. Observe that Claim_Line__c.Allowed_Amount__c still shows $150.00.

**Expected Behavior:** When Fee_Schedule__c.Allowed_Amount__c is updated, all related Claim_Line__c records where the parent Claim__c.Status__c is 'Under Review' or 'FNOL Received' should have their Allowed_Amount__c recalculated and a Claim_Line__c.Fee_Schedule_Last_Refreshed__c timestamp updated.

**Actual Behavior:** Claim_Line__c.Allowed_Amount__c retains the value from the time the claim line was originally created. No recalculation occurs.

**Environment:** Production, also reproducible in Full Sandbox.
**Frequency:** 100% reproducible.
**Impact:** High – financial accuracy of claim payments affected. Reported by Claims Operations Manager during Sprint 13 UAT.

## Acceptance Criteria

As per Description. Verify fix in Full Sandbox before deploying to Production. Regression test must confirm no recalculation occurs when Claim__c.Status__c = 'Adjudicated' or 'Closed' (frozen state protection).

## Solution

Add an Apex Trigger (FeeScheduleTrigger, After Update) on Fee_Schedule__c that queries all Claim_Line__c records where Claim_Line__c.Fee_Schedule__c = updated record Id AND parent Claim__c.Status__c IN ('FNOL Received', 'Under Review'). For each qualifying line, recalculate Claim_Line__c.Allowed_Amount__c using the updated Fee_Schedule__c.Allowed_Amount__c and set Claim_Line__c.Fee_Schedule_Last_Refreshed__c = System.now(). Process in bulkified batches (200-record chunks) using a Queueable Apex class to avoid governor limit violations. Add a Custom Metadata flag (FeeSchedule_Config__mdt.Enable_Mid_Adjudication_Refresh__c) to allow toggling the behavior per environment.

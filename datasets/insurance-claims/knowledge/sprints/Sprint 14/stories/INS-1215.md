## Description

As a Claims Operations Manager, I need Agentforce to automatically detect potential Coordination of Benefits (COB) scenarios during claims adjudication so that duplicate payments are prevented and the correct primary/secondary payer order is enforced before any Payment__c records are generated. Extends INS-1184 (COB policy data model). The agent should evaluate active Policy__c records linked to the claimant, cross-reference payer order rules, and flag or auto-resolve COB conflicts.

## Acceptance Criteria

Given a Claim__c record reaches Claim__c.Stage__c = 'Pending Adjudication',
When the Agentforce COB Detection agent action is triggered,
Then the agent queries all Policy__c records where Policy__c.Insured_Person__c = Claim__c.Claimant__c and Policy__c.Status__c = 'Active' and evaluates whether more than one policy covers the same date of service.

Given two or more active policies are found for the same claimant and service date,
When COB evaluation completes,
Then Claim__c.COB_Flag__c = true, Claim__c.Primary_Payer_Policy__c and Claim__c.Secondary_Payer_Policy__c are populated per birthday rule logic, and Claim__c.Stage__c is updated to 'COB Review'.

Given Claim__c.COB_Flag__c = true and COB cannot be auto-resolved (e.g., conflicting payer order data),
When the agent confidence score for resolution is below the threshold defined in COB_Config__mdt.Auto_Resolve_Threshold__c,
Then a Task is created and assigned to the Claims COB Specialist queue with Subject = 'Manual COB Review Required — [Claim Number]'.

Given COB is resolved (auto or manual) and Claim__c.Primary_Payer_Policy__c is confirmed,
When Claim__c.Stage__c transitions to 'Approved',
Then Payment__c records are only generated against the confirmed primary policy and Payment__c.COB_Verified__c = true.

Given Claim__c.COB_Flag__c = true,
When an EOB (Explanation_of_Benefits__c) is generated,
Then EOB__c.COB_Disclosure_Text__c is populated from the active COB disclosure template stored in Custom Metadata (EOB_Template__mdt.COB_Disclosure__c).

## Solution

Implement an Agentforce Agent Action 'DetectCOBConflicts' backed by an Apex invocable method (COBDetectionService.cls). The action is triggered by a Record-Triggered Flow on Claim__c when Stage__c changes to 'Pending Adjudication'. The Apex service performs a SOQL query across Policy__c and evaluates birthday rule logic encapsulated in a COBPriorityRulesEngine inner class. COB configuration thresholds are read from COB_Config__mdt. Auto-resolution updates Claim__c fields via a partial DML update; unresolved cases create Tasks via Flow subflow. EOB template population handled by a separate Scheduled Flow that fires when Claim__c.COB_Flag__c = true and Stage__c = 'Approved'. Payment__c generation gated by a Before-Insert Apex Trigger (PaymentValidationTrigger.cls) that checks COB_Verified__c.

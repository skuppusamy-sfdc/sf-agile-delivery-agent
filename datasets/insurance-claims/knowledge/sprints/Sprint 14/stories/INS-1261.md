## Description

As a Claims Operations Manager, I need a configurable LOB-weighted multiplier table stored in a custom metadata type so that the Agentforce triage scoring model (INS-1259) can apply LOB-specific risk adjustments without requiring code deployments when multiplier values change. LOBs include: Auto, Workers Comp, General Liability, Property, and Medical Malpractice.

## Acceptance Criteria

Given the custom metadata type LOB_Triage_Multiplier__mdt exists
When the Agentforce triage Apex invocable method executes
Then it reads the active multiplier record matching Claim__c.LOB__c and applies the Multiplier_Value__c to the base AI score

Given a Claims Operations Manager updates a LOB_Triage_Multiplier__mdt record via the Setup UI or a custom Admin LWC
When the next FNOL triage runs
Then the updated multiplier is applied without any Apex redeployment

Given LOB__c = 'Medical Malpractice'
When triage scoring runs
Then the base score is multiplied by the configured value (default 1.25) before Priority__c assignment

Given an unrecognized or null LOB__c value on Claim__c
When triage scoring runs
Then the system applies a default multiplier of 1.0 and logs a warning to AI_Processing_Log__c with Warning_Code__c = 'UNKNOWN_LOB'

## Solution

Create custom metadata type LOB_Triage_Multiplier__mdt with fields: LOB_API_Name__c (text), Multiplier_Value__c (number, 4 decimal places), Is_Active__c (checkbox), Effective_Date__c (date). Modify ClaimsTriageAction.cls (from INS-1259) to query LOB_Triage_Multiplier__mdt using a static map cached in a @AuraEnabled static initializer block for governor limit efficiency. Build a lightweight Admin LWC (lobMultiplierAdmin) backed by a custom Apex controller for non-Setup-admin updates. Deploy via unlocked package component so metadata records are environment-promotable.

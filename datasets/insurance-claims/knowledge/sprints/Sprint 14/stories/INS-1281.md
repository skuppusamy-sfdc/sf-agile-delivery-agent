## Description

**Bug Report – INS-1281**

**Environment:** Acme Insurance Full Sandbox (Sprint 14 deploy, build 14.3.1) and Production (confirmed by Provider Relations team on 2024-11-08)

**Steps to Reproduce:**
1. Log in as a user with the 'Provider_Relations_Specialist' Profile.
2. Navigate to the Provider_Network_Agreement__c tab.
3. Attempt to view a PNA record where OwnerId = 'Network Contracting Queue' (a Salesforce Queue, not a user).
4. Observe the resulting page.

**Expected Result:** Provider Relations Specialist should have Read access to all active PNA records regardless of owning queue, as defined by the Sharing Rule 'PNA_Sharing_ContractingToProviderRelations' deployed in Sprint 12 (INS-1198).

**Actual Result:** User receives 'Insufficient Privileges' error. The PNA record is completely inaccessible. PNA records owned by individual Contracting team users (not the queue) ARE visible correctly.

**Root Cause Hypothesis:** The Criteria-Based Sharing Rule 'PNA_Sharing_ContractingToProviderRelations' may be configured with 'Owned by members of' = a Public Group rather than including the Queue. Queues must be explicitly added to the sharing rule's source group, or the rule must use criteria-based (record field value) sharing instead of ownership-based sharing.

**Impact:** Medium – Provider Relations Specialists cannot view ~34% of PNA records (those assigned to the queue during contract routing workflow), impacting the Provider 360 FlexCard (INS-1277) and daily provider inquiry resolution.

**Workaround:** None currently. Contracting team manually reassigns queue-owned PNAs to individual users on request, which is unsustainable.

## Solution

Navigate to Setup > Sharing Settings > Provider_Network_Agreement__c Sharing Rules. Edit 'PNA_Sharing_ContractingToProviderRelations' to change from ownership-based to criteria-based sharing using PNA.Status__c IN ('Active', 'Pending Signature', 'Under Review') as the criteria, sharing with the 'Provider_Relations_Specialists' Public Group with Read access. Alternatively, add the 'Network Contracting Queue' explicitly to the 'Network_Contracting_Team' Public Group used in the existing ownership-based rule. Validate fix by running the Sharing Rule recalculation and testing with a Provider Relations Specialist sandbox user. Document the sharing model in the data security runbook. Regression test to confirm Contracting team users retain Edit access to their own records.

## Description

As a Credentialing Specialist, I need the system to automatically initiate a CAQH ProView profile verification call when a new Credentialing_Case__c is created for a provider and sync the returned attestation data back to the Credentialing_Case__c and Provider__c records so that manual CAQH lookups are eliminated and credentialing cycle time is reduced. The integration must handle both initial verifications and re-attestation scenarios.

## Acceptance Criteria

Given a Credentialing_Case__c is created with Credentialing_Case__c.Case_Type__c = 'Initial Credentialing' or 'Re-Credentialing'
When the record is saved
Then a CAQH ProView API call is initiated asynchronously using the provider's NPI stored in Provider__c.NPI__c
And Credentialing_Case__c.CAQH_Verification_Status__c = 'In Progress'

Given the CAQH API returns a successful response (HTTP 200)
When the callout completes
Then Credentialing_Case__c.CAQH_Provider_ID__c is populated from the response
And Credentialing_Case__c.CAQH_Attestation_Date__c is set to the attestation date returned
And Credentialing_Case__c.CAQH_Verification_Status__c = 'Verified'
And Provider__c.CAQH_Status__c = 'Active'

Given the CAQH API returns a profile status of 'Incomplete' or 'Not Authorized'
When the callout completes
Then Credentialing_Case__c.CAQH_Verification_Status__c = 'Action Required'
And a Task is created for the assigned Credentialing Specialist with Subject = 'CAQH Profile Incomplete – Provider Action Needed'
And Credentialing_Case__c.Status__c remains 'Pending External Data'

Given the CAQH API callout fails (timeout or HTTP 4xx/5xx)
When the error is caught
Then Credentialing_Case__c.CAQH_Verification_Status__c = 'Verification Failed'
And the error details are logged to Integration_Error_Log__c with Error_Source__c = 'CAQH'
And a retry is scheduled after 4 hours (max 3 retries)

Given Credentialing_Case__c.Case_Type__c = 'Re-Credentialing' and CAQH attestation is older than 365 days
When verification completes
Then a flag Credentialing_Case__c.CAQH_Attestation_Stale__c = true is set
And the case is escalated to the Credentialing Supervisor queue

## Solution

Implement a Record-Triggered Flow (after-insert, async) on Credentialing_Case__c to invoke an Apex Queueable class (CAQHVerificationQueueable.cls). The Queueable performs an HTTP callout to the CAQH ProView REST API via a Named Credential (CAQH_ProView_NC). Response parsing is handled by a CAQHResponseParser inner class. Retry logic is implemented via a custom object (Integration_Retry_Queue__c) processed by a Scheduled Apex job running every 2 hours. All callout results update Credentialing_Case__c and Provider__c via DML in the Queueable. Integration_Error_Log__c records are created via a separate Platform Event (Integration_Error__e) to decouple error logging from the main transaction. Named Credential uses OAuth 2.0 client credentials flow with CAQH's sandbox and production endpoints managed via Custom Settings.

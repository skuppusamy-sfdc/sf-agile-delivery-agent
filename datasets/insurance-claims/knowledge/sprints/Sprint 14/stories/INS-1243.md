## Description

**Bug Report**

**Summary:** Provider_Change_Notification__c records where PCN.Notification_Channel__c = 'EDI 834' are not displayed in the Provider 360 FlexCard ('Provider_360_FC') built in INS-1239. Only PCNs with Notification_Channel__c = 'Manual' or 'Portal' appear. This causes Provider Relations Specialists to miss EDI-sourced address and TIN change notifications, creating compliance and payment routing risks.

**Steps to Reproduce:**
1. Open any Provider__c record that has one or more Provider_Change_Notification__c records with PCN.Notification_Channel__c = 'EDI 834' and PCN.Status__c = 'Pending Review'.
2. Observe the 'Pending Data Changes' section of the Provider 360 FlexCard.
3. Note that EDI 834-sourced PCNs are absent from the count and list.

**Expected Result:** All Provider_Change_Notification__c records with PCN.Status__c = 'Pending Review' regardless of Notification_Channel__c value should appear in the FlexCard.

**Actual Result:** Only PCNs with Notification_Channel__c IN ('Manual', 'Portal') are shown. EDI 834 records are silently excluded.

**Root Cause (Preliminary):** DataRaptor Extract 'DR_GetPendingPCNs' (built in INS-1239) contains a hardcoded filter condition: Notification_Channel__c = 'Manual' OR Notification_Channel__c = 'Portal'. The EDI 834 channel value was added post-build and was not included in the DR filter.

**Impact:** Medium — EDI 834 is the primary inbound channel for group/employer-submitted provider changes via TPA feeds. Approximately 40% of PCN volume is affected.

## Acceptance Criteria

As per Description.

Given a Provider_Change_Notification__c exists with PCN.Notification_Channel__c = 'EDI 834' and PCN.Status__c = 'Pending Review'
When a Provider Relations Specialist opens the Provider__c record page
Then the PCN must appear in the 'Pending Data Changes' section of the Provider 360 FlexCard
And the count badge must accurately reflect ALL pending PCNs regardless of Notification_Channel__c value
And regression must confirm 'Manual' and 'Portal' channel PCNs continue to display correctly

## Solution

Update DataRaptor Extract 'DR_GetPendingPCNs' to remove the hardcoded Notification_Channel__c filter and instead filter solely on PCN.Status__c = 'Pending Review' and PCN.Provider__c = :recordId. Re-activate the DataRaptor and republish the FlexCard ('Provider_360_FC'). Add a Notification_Channel__c picklist value validation note to the DR documentation. Update Apex test class (Provider360FlexCardTest.cls) to include EDI 834 test data scenarios. QA to verify in sandbox with all three channel types before production deployment.

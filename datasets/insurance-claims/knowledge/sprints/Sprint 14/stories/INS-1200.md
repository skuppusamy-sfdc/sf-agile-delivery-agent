# INS-1200: Provider 360 FlexCard displaying active PNA, Fee Schedule, and credentialing status for Provider (Person Account)

- **Sprint**: Acme Insurance Sprint 14 (Phase 2)
- **Status**: In Test
- **Type**: Story
- **Priority**: High
- **Resolution**: Unresolved
- **Assignee**: Marcus Johnson
- **Reporter**: Anika Gupta
- **Components**: ProviderNetwork, CredentialingService
- **Epic**: Provider 360
- **Story Points**: 5
- **Labels**: Phase2, OmniStudio, ProviderData

## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard embedded on the Provider__c (Person Account) record page that surfaces active Provider_Network_Agreement__c records, associated Fee_Schedule__c line items, and real-time credentialing status from Credentialing_Case__c so that I have a single unified view without navigating multiple related lists.

## Acceptance Criteria

Given a Provider__c record is open in Lightning Experience,
When the Provider 360 FlexCard loads,
Then it must display all Provider_Network_Agreement__c records where PNA.Status__c = 'Active' and PNA.Provider__c = current record Id, showing PNA.Effective_Date__c, PNA.Expiration_Date__c, and PNA.Network_Tier__c.

Given an active PNA exists,
When the specialist expands the Fee Schedule section,
Then all Fee_Schedule__c records linked via Fee_Schedule__c.PNA__c must display CPT code, allowed amount, and effective date in a DataRaptor-driven data table.

Given a Credentialing_Case__c record exists for the provider where Credentialing_Case__c.Provider__c = current record Id,
When the FlexCard loads,
Then the most recent Credentialing_Case__c.Status__c and Credentialing_Case__c.Expiration_Date__c must appear in the 'Credentialing Status' badge tile, color-coded: Green = 'Active', Yellow = 'Expiring Within 90 Days', Red = 'Expired' or 'Suspended'.

Given the provider has no active PNA,
When the FlexCard loads,
Then an inline message 'No Active Network Agreement on File' must display in the PNA section.

-Given the provider NPI is present, display a CMS NPI lookup button that opens a new tab.- (Deferred to Phase 3 – external CMS integration not in scope for Sprint 14)

## Solution

Build OmniStudio FlexCard 'Provider360Card' with three child card states: PNA Summary, Fee Schedule Detail, and Credentialing Status. Use DataRaptor Extract DRs to query Provider_Network_Agreement__c, Fee_Schedule__c (via PNA junction), and Credentialing_Case__c with appropriate SOQL filters. Implement conditional card state visibility using FlexCard conditional rendering expressions. Embed the FlexCard on the Provider__c Lightning Record Page via App Builder. A separate DataRaptor Transform normalizes credentialing status values to badge color tokens. Assign access via Permission Set 'Provider_Relations_OmniStudio'.

## Description

As a Claims Operations Manager, I need a Lightning Web Component (LWC) claims history timeline displayed on the Policy__c record page that shows all associated Claim__c records in chronological order with key adjudication details and allows authorized users to download the Explanation_of_Benefits__c (EOB) PDF directly from the timeline so that claims staff have a single-pane view of policyholder claim activity without switching to separate list views.

## Acceptance Criteria

Given a Policy__c record page is loaded in the Lightning console,
When the Claims History Timeline LWC renders,
Then all associated Claim__c records are displayed in descending chronological order by Claim__c.Date_of_Loss__c, showing: Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.LOB__c, Claim__c.Total_Billed_Amount__c, and Claim__c.Total_Paid_Amount__c.

Given Claim__c.Status__c = 'Adjudicated' and an Explanation_of_Benefits__c record exists with EOB.Claim__c = Claim__c.Id,
When the user clicks the 'Download EOB' button on the timeline entry,
Then the EOB PDF is downloaded from Salesforce Files (ContentVersion) to the user's browser without navigating away from the Policy__c page.

Given Claim__c.Status__c = 'Under SIU Review',
When the timeline renders,
Then the claim entry displays a lock icon and the 'Download EOB' button is hidden for users without the 'SIU_Claims_Viewer' Permission Set.

Given Policy__c has more than 20 associated Claim__c records,
When the timeline loads,
Then only the 10 most recent claims are displayed by default with a 'Load More' button to paginate in increments of 10.

Given Claim__c.Status__c = 'Denied',
When the timeline renders,
Then the claim entry row is highlighted in red and Claim__c.Denial_Reason__c is displayed as a tooltip on hover.

-Given Policy__c.Policy_Type__c = 'Group', show group member sub-claims in nested accordion- (deprecated: group sub-claim view moved to Group/Employer Onboarding epic)

## Solution

Build LWC 'claimsHistoryTimeline' with an Apex controller (ClaimsHistoryController.cls) using SOQL with LIMIT/OFFSET for pagination, querying Claim__c WHERE Policy__c = :recordId ORDER BY Date_of_Loss__c DESC. EOB download handled via a separate Apex method retrieving ContentVersion Id linked to Explanation_of_Benefits__c and returning a download URL using PageReference. Permission Set check for SIU visibility implemented via Custom Permission 'View_SIU_Claims' evaluated in Apex and surfaced as a wire property in LWC. Component registered on Policy__c Lightning record page via App Builder. CSS styling uses SLDS timeline component pattern for visual chronology.

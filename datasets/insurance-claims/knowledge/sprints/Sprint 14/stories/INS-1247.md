## Description

**Bug Report**

**Summary:** When a Claim_Line__c record is adjudicated for an out-of-network provider that has an active Group_Service_Agreement__c (GSA) override, the FeeScheduleEngine returns null for Fee_Schedule__c.Allowed_Amount__c instead of applying the GSA negotiated rate.

**Steps to Reproduce:**
1. Create a Provider__c record with Provider_Network_Agreement__c.Network_Tier__c = 'Out-of-Network'.
2. Create an active Group_Service_Agreement__c linked to the same Provider__c with GSA_Override_Rate__c = 0.85.
3. Create a Claim__c with a Claim_Line__c referencing CPT code 99213.
4. Trigger adjudication via the 'Adjudicate Claim' quick action.
5. Observe Claim_Line__c.Allowed_Amount__c = null.

**Expected:** Claim_Line__c.Allowed_Amount__c should be populated using Fee_Schedule__c rate multiplied by GSA_Override_Rate__c (e.g., $150 * 0.85 = $127.50).

**Actual:** Claim_Line__c.Allowed_Amount__c = null; Claim_Line__c.Fee_Schedule_Applied__c = false.

**Root Cause (Preliminary):** FeeScheduleLookupService.cls appears to exit the lookup chain early when Provider_Network_Agreement__c.Network_Tier__c = 'Out-of-Network' before checking for GSA overrides.

**Environment:** Acme Insurance – Full Sandbox (FS1)
**Frequency:** 100% reproducible
**Impact:** HIGH – Payments are being held for all OON providers with GSA agreements.

## Acceptance Criteria

As per Description. Fix must ensure Claim_Line__c.Allowed_Amount__c is correctly calculated for all OON providers with active GSA overrides and regression tests pass for in-network fee schedule lookups.

## Solution

Review and fix FeeScheduleLookupService.cls to add a GSA override check prior to the early-exit condition for OON providers. Logic order: (1) Check for active Group_Service_Agreement__c on Provider__c, (2) if GSA exists and GSA.Status__c = 'Active', apply GSA_Override_Rate__c to base Fee_Schedule__c rate, (3) only fall through to standard OON logic if no active GSA. Add unit tests in FeeScheduleLookupServiceTest.cls covering GSA override scenario. Deploy to FS1 for QA validation before production push.

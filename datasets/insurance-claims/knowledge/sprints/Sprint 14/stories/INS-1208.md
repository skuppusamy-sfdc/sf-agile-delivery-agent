## Description

As a Claims Operations Manager, I need the Agentforce AI reasoning layer to automatically detect Coordination of Benefits (COB) scenarios and sequence primary/secondary payer responsibilities on incoming claims so that overpayment is prevented and claims are adjudicated against the correct payer order without requiring manual analyst intervention. This story extends INS-1207 by applying AI reasoning post-FNOL triage.

## Acceptance Criteria

Given a Claim__c record where the linked Policy__c.COB_Indicator__c = true,
When the Agentforce COB reasoning action is triggered,
Then Claim__c.COB_Status__c is set to 'COB Detected' and the agent queries all active Policy__c records associated with the claimant's Contact to identify overlapping coverage.

Given two or more active Policy__c records are found for the same claimant,
When the agent applies standard COB birthday-rule and Medicare Secondary Payer logic,
Then Claim__c.Primary_Payer__c and Claim__c.Secondary_Payer__c lookup fields are populated and Claim__c.COB_Sequence_Method__c is set to 'Birthday Rule', 'MSP', or 'Group Plan Order' accordingly.

Given Claim__c.Primary_Payer__c is an external TPA,
When payer sequencing is confirmed,
Then an outbound COB verification request Platform Event (COB_Verification_Request__e) is published with the TPA's NPI and TIN, and Claim__c.Status__c transitions to 'Pending COB Response'.

Given the agent cannot resolve payer sequence with confidence > 0.80,
When COB ambiguity is detected,
Then Claim__c.COB_Status__c = 'Manual Review Required' and a Case is auto-created linked to the Claim__c with Case.Subject = 'COB Sequencing — Analyst Review' and Case.Priority = 'High'.

Given COB sequencing is resolved and Claim__c.COB_Status__c = 'Resolved',
When each Claim_Line__c is processed,
Then Claim_Line__c.Payer_Responsibility__c is set to 'Primary' or 'Secondary' and Claim_Line__c.Allowed_Amount__c is calculated net of primary payer's adjudicated amount from the linked EOB.

## Solution

Agentforce Agent Action 'Resolve_COB_Sequence' implemented as an Apex invocable class querying Policy__c via SOQL with a filter on PolicyHolder__c = claimant Contact Id and Policy__c.Status__c = 'Active'. COB logic encapsulated in a CobSequencingService Apex class using a Strategy pattern for Birthday Rule, MSP, and Group Plan Order algorithms. Platform Event COB_Verification_Request__e published via EventBus.publish(). A Screen Flow surfaces COB ambiguity cases to analysts via a custom LWC component embedded on the Claim__c record page. Claim_Line__c updates handled by a scheduled Apex batch job triggered after COB_Status__c = 'Resolved'.

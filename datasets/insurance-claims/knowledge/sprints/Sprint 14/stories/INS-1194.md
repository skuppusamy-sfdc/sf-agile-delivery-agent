## Description

As a Claims Operations Manager, I need the system to automatically generate an 835 ERA transaction file and attach it to the Payment__c record when a payment is posted to a provider so that downstream provider billing systems receive standardized remittance data, reducing manual ERA preparation effort and provider inquiry calls.

## Acceptance Criteria

Given a Payment__c record is created with Payment__c.Status__c = 'Posted' and Payment__c.Payment_Method__c IN ('EFT','Check'),
When the Record-Triggered Flow on Payment__c fires,
Then an ERA generation job must be enqueued via Queueable Apex within 2 minutes of the status change.

Given the ERA Queueable job runs,
When it processes the Payment__c and related Claim_Line__c records,
Then an 835 EDI-formatted flat file must be generated and stored as a ContentDocument linked to the Payment__c record with ContentDocument.Title following the pattern 'ERA_{Payment__c.Name}_{YYYYMMDD}'.

Given the ERA file is generated,
When Payment__c.ERA_Status__c is evaluated,
Then Payment__c.ERA_Status__c must be set to 'Generated' and Payment__c.ERA_Generated_Date__c must be stamped with the current datetime.

Given the ERA file generation fails,
When the Queueable Apex encounters an exception,
Then Payment__c.ERA_Status__c must be set to 'Generation Failed', a Case must be auto-created with Case.Subject = 'ERA Generation Failure - {Payment Name}', and the case must be assigned to the Claims Operations queue.

Given Payment__c.Provider__c is linked to a Provider__c record with Provider__c.ERA_Delivery_Method__c = 'SFTP',
When ERA_Status__c transitions to 'Generated',
Then the file must be pushed to the provider's configured SFTP endpoint via a named credential within 15 minutes.

Given the ERA includes line-level detail,
When the 835 file is constructed,
Then each Claim_Line__c record must contribute one CLP/SVC segment pair including Claim_Line__c.CPT_Code__c, Claim_Line__c.Billed_Amount__c, Claim_Line__c.Paid_Amount__c, and Claim_Line__c.Adjustment_Reason_Code__c.

## Solution

Record-Triggered Flow on Payment__c (After Insert/Update, Status__c = 'Posted') enqueues the ERAGenerationQueueable Apex class. The Queueable class retrieves all related Claim_Line__c records and constructs the 835 EDI string using a custom ERABuilder utility class (ERABuilder.cls). The generated string is saved as a ContentVersion/ContentDocument. SFTP delivery is handled via a callout in a separate Queueable chained from the first, using External Credential and Named Credential configured per provider. Custom Metadata Type (ERA_Config__mdt) stores segment delimiters and ISA header values per trading partner. Platform Event (ERA_Delivery_Event__e) is published on SFTP success/failure for downstream monitoring.

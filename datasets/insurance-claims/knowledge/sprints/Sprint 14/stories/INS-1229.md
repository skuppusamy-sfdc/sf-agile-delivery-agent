## Description

As a Claims Operations Manager, I need Agentforce to automatically classify incoming FNOL submissions by Line of Business (LOB) and severity tier so that claims are routed to the correct adjudication queue without manual triage, reducing average handle time and improving SLA compliance. The AI agent should analyze structured intake fields as well as free-text loss descriptions to produce a classification confidence score alongside the routing recommendation.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Status__c = 'FNOL Received' and Claim__c.Loss_Description__c is populated,
When the Agentforce Classification Agent is invoked via Platform Event (FNOL_Submitted__e),
Then Claim__c.LOB__c must be set to one of ['Auto', 'Property', 'Workers_Comp', 'General_Liability', 'Health'] with Claim__c.AI_Classification_Confidence__c >= 0.75.

Given the AI confidence score is < 0.75,
When classification completes,
Then Claim__c.Status__c must be set to 'Pending Manual Triage' and a Task must be created and assigned to the Claims Triage queue with Subject = 'Low-Confidence AI Classification — Review Required'.

Given Claim__c.LOB__c = 'Workers_Comp' and Claim__c.Injury_Severity__c is populated,
When severity scoring runs,
Then Claim__c.Severity_Tier__c must be set to one of ['1-Critical', '2-Major', '3-Moderate', '4-Minor'] and Claim__c.AI_Severity_Score__c must be stored as a numeric value between 0.00 and 1.00.

Given Claim__c.Severity_Tier__c = '1-Critical',
When triage routing executes,
Then the claim must be assigned to the 'Critical Claims Unit' queue and Claim__c.Priority__c must be set to 'High' within 60 seconds of FNOL receipt.

Given any AI classification action completes (success or failure),
When the agent run concludes,
Then an AI_Audit_Log__c record must be created with fields: AI_Audit_Log__c.Claim__c (lookup), AI_Audit_Log__c.Model_Version__c, AI_Audit_Log__c.Input_Tokens__c, AI_Audit_Log__c.Output_Summary__c, and AI_Audit_Log__c.Timestamp__c.

-Given Claim__c.Channel__c = 'Phone', the AI agent should transcribe the call and extract FNOL fields automatically- (descoped to INS-1244 — voice integration deferred to Phase 3).

## Solution

Deploy an Agentforce Agent Action bound to a custom Apex invocable class (FNOLClassificationAction.cls) that calls the Einstein generative AI API with a structured prompt template referencing Claim__c fields. Publish a Platform Event (FNOL_Submitted__e) from a Record-Triggered Flow on Claim__c insert where Status__c = 'FNOL Received'. The Apex action parses the LLM JSON response, maps LOB and severity outputs back to Claim__c via a DML update, and writes the AI_Audit_Log__c record. A separate Record-Triggered Flow handles downstream queue assignment based on Severity_Tier__c and LOB__c field values. Permission Set Group 'Agentforce_Claims_AI' grants the integration user access to the new custom objects and fields.

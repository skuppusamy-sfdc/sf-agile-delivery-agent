## Description

As a Provider Network Specialist, I need the system to automatically execute all downstream contract, claims, and notification actions when a Network_Termination_Case__c record is approved so that provider terminations are processed consistently, compliantly (meeting state-mandated notice periods), and without manual coordination across teams. Downstream impacts include deactivating PNA records, updating open claims, triggering member notifications, and generating the required PCN to affected Group Service Agreement holders.

## Acceptance Criteria

Given a Network_Termination_Case__c.Status__c transitions to 'Approved' with Network_Termination_Case__c.Effective_Termination_Date__c populated,
When the approval process completes,
Then all related Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active' must be updated to PNA__c.Status__c = 'Pending Termination' and PNA__c.Termination_Date__c = Network_Termination_Case__c.Effective_Termination_Date__c.

Given PNA__c.Status__c is set to 'Pending Termination',
When the effective termination date is reached (evaluated by a scheduled flow running nightly),
Then PNA__c.Status__c must transition to 'Terminated' and Provider__c.Network_Status__c must be updated to 'Out of Network'.

Given Provider__c.Network_Status__c = 'Out of Network',
When any new Claim__c is submitted with Claim__c.Rendering_Provider__c referencing the terminated provider and Claim__c.Date_of_Service__c > Effective_Termination_Date__c,
Then Claim__c.Status__c must be set to 'Pended — Out of Network Provider' and Claim__c.Pend_Reason__c = 'Provider Network Termination'.

Given open Claim__c records with Claim__c.Status__c IN ('FNOL Received', 'Under Review') and Claim__c.Rendering_Provider__c = terminated provider,
When the termination automation runs,
Then each open claim must receive a Task assigned to the Claims Operations queue with Subject = 'Provider Terminated — Claim Review Required' and Claim__c.Termination_Alert_Sent__c = true.

Given one or more Group_Service_Agreement__c records are linked to the terminated provider's PNA,
When the termination is approved,
Then a Provider_Change_Notification__c must be auto-created for each affected GSA with PCN__c.Change_Type__c = 'Provider Termination', PCN__c.Effective_Date__c = Network_Termination_Case__c.Effective_Termination_Date__c, and PCN__c.Status__c = 'Pending Review'.

Given the state in Provider__c.State_License__c requires a minimum 30-day advance notice per Compliance_Rule__mdt,
When the termination approval occurs with fewer than 30 days until the effective date,
Then the approval must be blocked, a validation error must display ('State compliance: minimum 30-day notice period not met'), and a Compliance_Alert__c record must be created and assigned to the Compliance Officer role.

## Solution

Implement a multi-step Record-Triggered Flow on Network_Termination_Case__c (AfterUpdate, Status__c = 'Approved') that orchestrates: (1) a Loop element updating related PNA records via Apex invocable (BulkPNATerminationAction.cls) to handle bulk DML safely, (2) a sub-flow for open claims task creation, (3) a sub-flow for PCN generation per GSA. Compliance date validation is enforced by a Before-Save Record-Triggered Flow that queries Compliance_Rule__mdt keyed on State__c. The nightly PNA status transition (Pending Termination → Terminated) uses a Scheduled Flow with a collection filter on PNA__c.Termination_Date__c = TODAY. Provider__c.Network_Status__c update is handled by a separate Record-Triggered Flow on PNA__c. All bulk operations use Apex to respect governor limits. A custom Permission Set ('Network_Termination_Admin') is required to approve Network_Termination_Case__c records.

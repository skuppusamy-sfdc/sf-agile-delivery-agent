## Description

As a Claims Operations Manager, I need Claim_Line__c records to be automatically validated against the applicable Fee_Schedule__c at adjudication time so that allowed amounts are calculated accurately, overpayments are prevented, and adjudicators receive AI-assisted explanations for any variance between billed and allowed amounts. The solution must resolve the correct fee schedule by matching on Provider__c NPI, Facility__c TIN, CPT code (Claim_Line__c.CPT_Code__c), and the effective date of service against Fee_Schedule__c.Effective_Date__c and Fee_Schedule__c.Termination_Date__c. An Agentforce Adjudication Assist agent must surface variance rationale inline on the Claim_Line__c record page.

## Acceptance Criteria

Given a Claim_Line__c record is saved with Claim_Line__c.Status__c = 'Pending Adjudication' and Claim_Line__c.CPT_Code__c is populated,
When the FeeScheduleEngine validation Apex trigger fires,
Then Fee_Schedule__c must be resolved by matching: (a) Claim_Line__c.Provider__c NPI to PNA.Provider__c, (b) Claim_Line__c.CPT_Code__c to Fee_Schedule__c.CPT_Code__c, and (c) Claim_Line__c.Date_of_Service__c falling within Fee_Schedule__c.Effective_Date__c and Fee_Schedule__c.Termination_Date__c.

Given a matching Fee_Schedule__c record is found,
When allowed amount is computed,
Then Claim_Line__c.Allowed_Amount__c must be set to Fee_Schedule__c.Allowed_Amount__c and Claim_Line__c.Fee_Schedule__c must be populated with the resolved Fee_Schedule__c record ID.

Given Claim_Line__c.Billed_Amount__c > Claim_Line__c.Allowed_Amount__c,
When the variance exceeds 10%,
Then Claim_Line__c.Variance_Flag__c must be set to true, Claim_Line__c.Variance_Pct__c must be calculated as ((Billed - Allowed) / Billed * 100), and Claim_Line__c.Status__c must be updated to 'Variance Review'.

Given Claim_Line__c.Variance_Flag__c = true,
When the Agentforce Adjudication Assist agent is invoked,
Then Claim_Line__c.AI_Variance_Rationale__c must be populated with a natural-language explanation referencing the CPT code description, applicable fee schedule name, and the contract rate source (PNA or GSA).

-Given no matching Fee_Schedule__c is found, Claim_Line__c.Allowed_Amount__c must default to Claim_Line__c.Billed_Amount__c and processing must continue- (deprecated: default-to-billed behavior disabled per compliance directive CO-2024-11; missing fee schedule now halts adjudication).

Given no matching Fee_Schedule__c is found,
When validation completes,
Then Claim_Line__c.Status__c must be set to 'Fee Schedule Missing' and a Task must be created and assigned to the Provider Network Specialist queue with Subject = 'Missing Fee Schedule: [CPT Code] - [Provider NPI]'.

Given Claim_Line__c.Status__c = 'Adjudicated' and Payment__c is generated,
When the EOB is produced,
Then Explanation_of_Benefits__c.Allowed_Amount__c must match the sum of all related Claim_Line__c.Allowed_Amount__c values and Explanation_of_Benefits__c.Fee_Schedule_Reference__c must list all resolved fee schedule names.

## Solution

Develop an Apex Trigger (ClaimLineAdjudicationTrigger) on Claim_Line__c (before update, condition: Status__c changes to 'Pending Adjudication') that calls a FeeScheduleResolutionService Apex class. The service performs a SOQL query joining Fee_Schedule__c, Provider_Network_Agreement__c, and Provider__c using NPI and CPT code with date-range filtering. Resolution priority order: (1) GSA-linked fee schedule, (2) PNA-linked fee schedule, (3) default network rate. If no match, the trigger sets Status__c to 'Fee Schedule Missing' and invokes a Flow to create the Task. For variance flagging, an after-update Record-Triggered Flow evaluates Variance_Pct__c and invokes the Agentforce Adjudication Assist AgentAction 'ExplainVariance' via an Apex invocable, which constructs a prompt from CPT code metadata, PNA contract terms, and billed/allowed delta. The AI rationale is written back via a separate DML operation to avoid trigger recursion. The EOB generation is handled by an existing OmniScript (extended in this story) that uses a DataRaptor Transform to aggregate Claim_Line__c allowed amounts. A new Permission Set 'FeeSchedule_Adjudicator_PS' grants read access to Fee_Schedule__c and Claim_Line__c.AI_Variance_Rationale__c to adjudicator profiles.

## Description

As a Claims Operations Manager, I need the system to automatically look up and apply the correct contracted rate from Fee_Schedule__c to each Claim_Line__c during adjudication so that payment calculations reflect PNA-negotiated rates without requiring manual rate entry by claims examiners.

## Acceptance Criteria

Given a Claim__c.Status__c transitions to 'In Adjudication',
When the FeeScheduleEngine evaluates each child Claim_Line__c,
Then for each line the system queries Fee_Schedule__c matching on Fee_Schedule__c.CPT_Code__c = Claim_Line__c.CPT_Code__c, Fee_Schedule__c.Provider__c = Claim__c.Billing_Provider__c, and Fee_Schedule__c.Effective_Date__c <= Claim__c.Date_of_Service__c <= Fee_Schedule__c.Expiration_Date__c.

Given a matching Fee_Schedule__c record is found,
When the rate is applied,
Then Claim_Line__c.Contracted_Rate__c is populated, Claim_Line__c.Allowed_Amount__c is calculated as MIN(Claim_Line__c.Billed_Amount__c, Claim_Line__c.Contracted_Rate__c), and Claim_Line__c.Rate_Source__c = 'Fee Schedule'.

Given no matching Fee_Schedule__c record exists for a Claim_Line__c,
When the engine cannot find a contracted rate,
Then Claim_Line__c.Rate_Source__c = 'UCR' and Claim_Line__c.Allowed_Amount__c is calculated using the Usual, Customary & Reasonable rate from the UCR_Rate__mdt custom metadata type.

-Given the provider has both a PNA and a GSA, the GSA rate always takes precedence-

Given the provider has both a PNA-linked Fee_Schedule__c and a GSA-linked Fee_Schedule__c,
When both are valid for the date of service,
Then the GSA rate is applied and Claim_Line__c.Rate_Override_Reason__c = 'GSA Supersedes PNA'.

Given all Claim_Line__c records on a Claim__c are adjudicated,
When the last line is updated,
Then Claim__c.Total_Allowed_Amount__c is rolled up via a Roll-Up Summary field and Claim__c.Status__c transitions to 'Adjudicated'.

## Solution

Apex invocable method FeeScheduleResolutionService.cls called from a Record-Triggered Flow on Claim__c (after update, Status__c = 'In Adjudication'). The service performs a priority-ordered SOQL query: GSA-linked Fee_Schedule__c first, then PNA-linked, then UCR_Rate__mdt fallback. Claim_Line__c updates batched via Database.update() in a single DML call. Custom Metadata Type UCR_Rate__mdt stores fallback rates by CPT code range. Roll-Up Summary field on Claim__c aggregates Claim_Line__c.Allowed_Amount__c. Unit tests in FeeScheduleResolutionServiceTest.cls achieve 92% coverage.

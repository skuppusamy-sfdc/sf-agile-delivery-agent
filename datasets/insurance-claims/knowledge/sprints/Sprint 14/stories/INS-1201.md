## Description

As a Claims Operations Manager, I need the system to automatically generate an Explanation_of_Benefits__c (EOB) record and deliver a formatted EOB PDF to the policyholder via DocuSign when a Claim__c transitions to 'Adjudicated' status so that we meet the state-mandated 5-business-day EOB delivery requirement and eliminate manual document preparation.

## Acceptance Criteria

Given Claim__c.Status__c transitions from 'Under Review' to 'Adjudicated',
When the Record-Triggered Flow fires on Claim__c update,
Then a new Explanation_of_Benefits__c record must be created with EOB.Claim__c = triggering Claim Id, EOB.Generated_Date__c = today, EOB.Delivery_Status__c = 'Pending', and EOB.Total_Billed__c, EOB.Total_Allowed__c, EOB.Member_Responsibility__c populated from corresponding Claim__c fields.

Given the EOB record is created,
When the DocuSign envelope is dispatched via Apex DocuSign API callout,
Then EOB.DocuSign_Envelope_Id__c must be populated, EOB.Delivery_Status__c = 'Sent', and EOB.Sent_Timestamp__c = current datetime.

Given the policyholder email (Policy__c.Policyholder_Email__c) is null or invalid,
When the DocuSign dispatch is attempted,
Then EOB.Delivery_Status__c = 'Failed – Invalid Email', a Case must be auto-created with Subject = 'EOB Delivery Failed – Missing Contact Info', and the Claims Operations Manager queue must be notified via custom notification.

Given the EOB is delivered and the policyholder opens the document,
When DocuSign fires the 'Viewed' webhook callback,
Then EOB.Delivery_Status__c = 'Viewed' and EOB.Viewed_Timestamp__c must be updated.

Given Claim__c.LOB__c = 'Dental' or 'Vision',
When the EOB is generated,
Then the Conga CLM template ID used must correspond to the LOB-specific template mapping defined in EOB_Template_Config__mdt.

## Solution

Implement a Record-Triggered Flow on Claim__c (After Update, condition: Status__c changed to 'Adjudicated') that creates the EOB record and calls an Apex invocable action (EOBGenerationService.generateAndDispatch()). The Apex class uses Conga CLM API to render the PDF from a metadata-driven template (EOB_Template_Config__mdt keyed by LOB__c), then passes the rendered document bytes to the DocuSign eSignature REST API to create and send the envelope. A DocuSign Connect webhook endpoint (Apex REST resource) handles status callbacks and updates EOB delivery fields. A Scheduled Flow runs nightly to retry any EOB records where Delivery_Status__c = 'Failed – Invalid Email' after a Case resolution update.

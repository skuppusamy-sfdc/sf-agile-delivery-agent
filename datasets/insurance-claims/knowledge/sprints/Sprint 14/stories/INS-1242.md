## Description

As a Provider Network Specialist, I need the Provider_Network_Agreement__c record to automatically transition to 'Active' status and populate key contract metadata when the associated DocuSign envelope reaches 'Completed' status so that network activation is immediate and error-free, eliminating the current 24–48 hour manual update lag that delays provider access to the network.

## Acceptance Criteria

Given a DocuSign envelope linked to a Provider_Network_Agreement__c record is marked 'Completed' by DocuSign
When the DocuSign Status Change webhook fires and is received by the Salesforce DocuSign Connect listener
Then Provider_Network_Agreement__c.Status__c must be updated from 'Pending Signature' to 'Active'
And Provider_Network_Agreement__c.Effective_Start_Date__c must be set to the envelope completion date
And Provider_Network_Agreement__c.DocuSign_Envelope_ID__c must be stamped with the envelope ID from the webhook payload
And Provider_Network_Agreement__c.Signed_Document_URL__c must be populated with the completed document link

Given Provider_Network_Agreement__c.Status__c transitions to 'Active'
When the Record-Triggered Flow (PNA_Activation_Flow) fires
Then a welcome notification Task must be created for the assigned Provider Relations Specialist
And if Provider_Network_Agreement__c.Contract_Type__c = 'GSA', the linked Group_Service_Agreement__c.Active_Provider_Count__c must be incremented by 1

Given the DocuSign envelope is 'Declined' or 'Voided'
When the webhook is received
Then Provider_Network_Agreement__c.Status__c must be set to 'Signature Declined'
And Provider_Network_Agreement__c.Decline_Reason__c must be populated from the webhook payload's 'declineReason' field
And a Task must be auto-created for the Provider Network Specialist to initiate re-contracting

## Solution

Configure DocuSign for Salesforce (managed package) Connect listener to handle envelope status webhooks. Create a Record-Triggered Flow (PNA_Activation_Flow) on Provider_Network_Agreement__c triggered by Status__c change to 'Active' for downstream actions. Webhook payload parsing handled by a custom REST endpoint (DocuSignWebhookHandler.cls, @RestResource) that maps envelope metadata to PNA fields. Group_Service_Agreement__c counter update performed via a before-update Apex Trigger (PNAActivationTrigger.cls) to avoid flow recursion. Integration user permissions scoped via Permission Set 'DocuSign_Integration_PS'.

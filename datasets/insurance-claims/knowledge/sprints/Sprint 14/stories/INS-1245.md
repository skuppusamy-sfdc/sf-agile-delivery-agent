## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces the provider's current credentialing status, active Provider_Network_Agreement__c (PNA) details, NPI verification status from NPPES, and CAQH profile completeness percentage so that I can assess provider standing without navigating to multiple objects.

## Acceptance Criteria

Given a Provider__c record is open in the Lightning console,
When the Provider 360 FlexCard loads,
Then the following data points are visible: Credentialing_Case__c.Status__c, Provider_Network_Agreement__c.Effective_Date__c, Provider_Network_Agreement__c.Expiration_Date__c, Provider_Network_Agreement__c.Network_Tier__c, and NPPES NPI verification badge.

Given Provider_Network_Agreement__c.Expiration_Date__c is within 60 days of today,
When the FlexCard renders,
Then the PNA expiration field displays in amber with a warning icon and tooltip text 'PNA Expiring Soon'.

Given Provider_Network_Agreement__c.Status__c = 'Terminated',
When the FlexCard renders,
Then the PNA section displays a red 'Terminated' badge and the Reinstate button is visible to users with the 'Provider_Network_Manager' Permission Set.

Given CAQH profile completeness < 80%,
When the FlexCard renders,
Then a CAQH alert tile displays: 'CAQH Profile Incomplete – [X]%' with a direct link to the CAQH portal.

-Given Provider__c.TIN__c is null, display TIN Missing warning on card- (deprecated: TIN validation moved to onboarding OmniScript INS-1231)

## Solution

Build an OmniStudio FlexCard ('Provider_360_Card') with multiple DataRaptor Extract actions: one for Credentialing_Case__c (filtered by Provider__c lookup, most recent), one for Provider_Network_Agreement__c (active records), and one for a named credential callout to NPPES REST API for NPI status. CAQH completeness sourced via a scheduled Apex job (CAQHSyncBatch.cls) writing to Provider__c.CAQH_Completeness__c. Conditional rendering via FlexCard state actions. FlexCard embedded on Provider__c Lightning record page via App Builder.

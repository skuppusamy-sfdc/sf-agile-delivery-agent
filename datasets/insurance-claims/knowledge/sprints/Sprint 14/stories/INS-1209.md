## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that displays real-time credentialing status, active Provider Network Agreements (PNAs), current Fee Schedule assignments, and open Claim__c records so that I can service provider inquiries without navigating across multiple objects and tabs.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record,
When the Provider 360 FlexCard renders,
Then the card displays Credentialing_Case__c.Status__c for the most recent open credentialing case, including CAQH profile last-verified date sourced from Credentialing_Case__c.CAQH_Last_Verified__c.

Given the provider has one or more active Provider_Network_Agreement__c records,
When the PNA section of the FlexCard loads,
Then all PNAs where PNA__c.Status__c = 'Active' and PNA__c.Effective_Date__c <= TODAY() and PNA__c.Termination_Date__c >= TODAY() are listed, showing PNA__c.Network_Name__c, PNA__c.Contract_Type__c, and PNA__c.Fee_Schedule__r.Name.

Given the provider has associated Fee_Schedule__c records,
When the Fee Schedule panel renders,
Then Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.LOB__c, and Fee_Schedule__c.Reimbursement_Model__c are displayed, with expired schedules visually distinguished by a warning badge.

Given there are open Claim__c records where Claim__c.Billing_Provider__c = Provider__c.Id and Claim__c.Status__c IN ('Under Review', 'Pending Information', 'Suspended'),
When the open claims panel loads,
Then a count badge and scrollable list of up to 10 most recent claims are shown with Claim__c.Claim_Number__c, Claim__c.Date_of_Service__c, and Claim__c.Total_Billed_Amount__c.

-Given the FlexCard displays a 'Send PCN' quick action button, When clicked, Then a Provider_Change_Notification__c record is created inline- (moved to dedicated PCN story INS-1213).

## Solution

Built using OmniStudio FlexCard with four child cards (Credentialing, PNA, Fee Schedule, Open Claims) each backed by a dedicated DataRaptor Extract pulling from Credentialing_Case__c, Provider_Network_Agreement__c, Fee_Schedule__c, and Claim__c. The parent FlexCard uses a DataRaptor Turbo Extract on Account (Provider__c) to seed the context. Conditional rendering via FlexCard state expressions controls warning badges on expired Fee_Schedule__c records. The FlexCard is deployed via a Lightning App Builder page assigned to the Provider Record Page using a Permission Set Group 'Provider_Relations_User'.

## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard dashboard on the Provider__c (Person Account) record page that displays real-time contract status, credentialing status, active claims activity, and network participation details so that I can manage provider relationships from a single view without navigating multiple objects.

This story extends INS-1241 (Provider 360 base layout) by adding the claims activity panel and Agentforce AI summary widget.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record,
When the Provider 360 FlexCard loads,
Then the dashboard displays the following panels without additional page navigation:
  - Contract Panel: Provider_Network_Agreement__c.Status__c, PNA.Effective_Date__c, PNA.Termination_Date__c, linked Fee_Schedule__c name
  - Credentialing Panel: Credentialing_Case__c.Status__c (most recent), Credentialing_Case__c.Expiration_Date__c, CAQH_ID__c
  - Claims Activity Panel: count of Claim__c records by Status__c (last 90 days), total Claim__c.Paid_Amount__c (last 90 days)
  - Network Panel: list of active Provider_Network_Agreement__c records with Network_Name__c and LOB__c.

Given the FlexCard claims activity panel loads,
When Provider__c has more than 500 Claim__c records in the last 90 days,
Then the panel renders within 3 seconds using a DataRaptor Turbo Extract with server-side aggregation, not client-side iteration.

Given Provider__c.Credentialing_Case__c.Expiration_Date__c is within 60 days of today,
When the FlexCard renders,
Then the Credentialing Panel header displays a yellow warning badge with text 'Expiring Soon' and a 'Renew' action button that launches the Credentialing Renewal OmniScript.

-Given the Agentforce AI Summary widget is enabled, display predictive churn risk score- (deprecated – AI widget moved to INS-1261)

Given a Provider Relations Specialist clicks 'View All Claims' in the Claims Activity Panel,
When the action fires,
Then a modal opens displaying a DataTable of Claim__c records filtered to that Provider__c with columns: Claim__c.Name, Claim__c.Date_of_Service__c, Claim__c.Status__c, Claim__c.Paid_Amount__c.

## Solution

Build OmniStudio FlexCard (Provider_360_Dashboard) with four child FlexCards embedded as sub-components. DataRaptor Turbo Extract procedures (DR_Get_Provider_Contract_Summary, DR_Get_Credentialing_Status, DR_Get_Claims_Activity_90Day, DR_Get_Network_Agreements) feed each panel. The claims aggregation DataRaptor uses a SOQL aggregate query (COUNT, SUM) to avoid governor limits. Conditional rendering on the Credentialing Panel uses FlexCard State Actions with a date comparison formula. The 'Renew' button triggers an OmniScript (Credentialing_Renewal_OS) via a FlexCard Action of type 'OmniScript'. The 'View All Claims' modal is implemented as a child FlexCard with a DataTable element. FlexCard is deployed to the Provider__c Lightning Record Page via a custom Lightning App Builder page layout restricted by Permission Set 'Provider_Relations_User'.

## Description

As a Claims Operations Manager, I need a configurable LOB-specific scoring multiplier framework within the Agentforce claim triage engine (extends INS-1199) so that severity scores can be adjusted per line of business (Auto, Workers Comp, Medical, Dental, Vision) without requiring code deployments, enabling the claims team to tune AI routing thresholds as loss patterns evolve.

## Acceptance Criteria

Given a Custom Metadata Type AI_LOB_Scoring_Config__mdt is configured with records for each LOB (Auto, Workers Comp, Medical, Dental, Vision) containing fields: Severity_Multiplier__c, Auto_Adjudicate_Threshold__c, SIU_Referral_Threshold__c, and Senior_Adjuster_Threshold__c,
When the Agentforce triage engine processes a Claim__c,
Then the raw AI score must be multiplied by the matching AI_LOB_Scoring_Config__mdt.Severity_Multiplier__c for the claim's LOB__c before being written to Claim__c.AI_Severity_Score__c.

Given Claim__c.LOB__c = 'Workers Comp' and AI_LOB_Scoring_Config__mdt Severity_Multiplier__c = 1.2,
When triage scoring completes,
Then Claim__c.AI_Severity_Score__c = (raw_score * 1.2), capped at 100, and Claim__c.AI_Score_Multiplier_Applied__c = '1.2 (Workers Comp)'.

Given no AI_LOB_Scoring_Config__mdt record exists for a given LOB,
When the triage engine attempts to retrieve the multiplier,
Then a default multiplier of 1.0 must be applied and Claim__c.AI_Score_Multiplier_Applied__c = '1.0 (Default – No LOB Config Found)'.

Given a Claims Operations Manager updates AI_LOB_Scoring_Config__mdt.SIU_Referral_Threshold__c for 'Auto' from 75 to 80,
When the next Auto claim is triaged,
Then the new threshold of 80 must be used without any code deployment.

Given Claim__c.AI_Severity_Score__c (post-multiplier) exceeds AI_LOB_Scoring_Config__mdt.SIU_Referral_Threshold__c for the claim's LOB,
When routing logic executes,
Then Claim__c.AI_Triage_Recommendation__c = 'SIU Referral' and a Platform Event SIUReferralTriggered__e must be published with Claim Id and LOB payload.

## Solution

Extend the ClaimTriageEngineService Apex class introduced in INS-1199 to query AI_LOB_Scoring_Config__mdt at runtime using a static map cache (refreshed per transaction). Implement a ScoringMultiplierResolver inner class that encapsulates LOB lookup and default fallback logic. Publish SIUReferralTriggered__e Platform Event from within the Apex scoring method when SIU threshold is breached. Create a Platform Event-Triggered Flow subscribing to SIUReferralTriggered__e that creates a Task in the SIU queue. Expose AI_LOB_Scoring_Config__mdt management to Claims Operations Manager via a custom Setup menu entry using a Lightning Web Component admin UI (AILOBConfigManager LWC). Deploy all new fields and CMDT via unlocked package.

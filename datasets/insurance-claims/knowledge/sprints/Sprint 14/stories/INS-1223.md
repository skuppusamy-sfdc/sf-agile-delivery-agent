## Description

As a Claims Operations Manager, I need the system to automatically generate an Electronic Remittance Advice (ERA) file in ANSI X12 835 format and trigger an EFT payment batch for all Claim__c records that reach Status__c = 'Adjudicated – Approved' so that providers receive accurate, timely payment without manual intervention from the payment team.

## Acceptance Criteria

Given one or more Claim__c records with Status__c = 'Adjudicated – Approved' and Payment__c records in state Payment_Status__c = 'Ready for Disbursement',
When the nightly scheduled Apex batch job 'ERA_Generation_Batch' runs at 02:00 UTC,
Then a valid ANSI X12 835 ERA file must be generated per TIN grouping, attached to the corresponding Provider__c (Person Account) record, and Payment__c.ERA_File_Generated__c must be set to TRUE.

Given the ERA file is successfully generated,
When the batch completes without errors,
Then Payment__c.Status__c must transition to 'EFT Submitted', Payment__c.Disbursement_Date__c must be stamped with T+1 business date, and a Platform Event 'Payment_Disbursement_Initiated__e' must be published.

Given a Provider__c record has Provider__c.EFT_Enrolled__c = FALSE,
When the batch attempts to include that provider's claims,
Then those Payment__c records must be excluded from the EFT batch, Payment__c.Status__c must be set to 'Check Payment Pending', and a Task must be auto-created assigned to the Provider Relations Specialist queue.

Given the ERA batch encounters a Claim__c with COB (Coordination of Benefits) flag Claim__c.COB_Active__c = TRUE,
When processing that claim,
Then the ERA must reflect primary vs. secondary payer amounts on each Claim_Line__c and Explanation_of_Benefits__c.COB_Adjustment_Amount__c must be populated before ERA generation.

-ERA files must be deposited to the legacy SFTP server at ftp.acme-legacy.internal- (deprecated: replaced by AWS S3 secure bucket integration per INS-1198)

Given ERA file generation is complete,
When the file is ready for transmission,
Then the file must be uploaded to the configured AWS S3 bucket path '/era/outbound/{YYYY-MM-DD}/' via the DataIntegration named credential 'AWS_S3_ERA_Outbound'.

## Solution

Implement a Schedulable Apex class 'ERA_Generation_Batch' implementing Database.Batchable and Database.Stateful. The batch queries Payment__c records in 'Ready for Disbursement' state, groups by Provider TIN, and constructs the 835 EDI string using a custom Apex EDI835Builder utility class. Files are attached using ContentVersion and linked to Provider__c via ContentDocumentLink. S3 upload performed via an Apex HTTP callout using the Named Credential 'AWS_S3_ERA_Outbound'. Platform Event published post-batch via EventBus.publish(). A Record-Triggered Flow handles the Check Payment Pending path and Task creation for non-EFT providers.

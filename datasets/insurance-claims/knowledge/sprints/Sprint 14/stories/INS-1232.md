## Description

As a Provider Relations Specialist, I need a unified Provider 360 FlexCard on the Provider__c (Person Account) Lightning record page that consolidates active Provider_Network_Agreement__c details, current credentialing status from Credentialing_Case__c, open Claim__c counts, and any pending Provider_Change_Notification__c records so that I can assess a provider's full relationship status without navigating multiple objects.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record,
When the Provider 360 FlexCard loads,
Then the card must display: Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Network_Status__c, and the count of active Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active' and PNA__c.Effective_End_Date__c >= TODAY.

Given the Provider__c has one or more Credentialing_Case__c records,
When the FlexCard credentialing section renders,
Then the most recent Credentialing_Case__c.Status__c and Credentialing_Case__c.Expiration_Date__c must be displayed with a color-coded badge: Green (> 90 days), Yellow (31–90 days), Red (≤ 30 days or Expired).

Given the Provider__c has related Claim__c records,
When the claims summary section renders,
Then the FlexCard must display counts grouped by Claim__c.Status__c ('Open', 'Under Review', 'Adjudicated', 'Denied') using a DataRaptor Extract feeding a bar chart child card.

Given one or more Provider_Change_Notification__c records exist with PCN__c.Status__c = 'Pending Review' linked to the Provider__c,
When the PCN alert section loads,
Then each pending PCN must be listed with PCN__c.Change_Type__c, PCN__c.Effective_Date__c, and a quick-action button to open the PCN record.

Given the FlexCard data takes longer than 5 seconds to load,
When a timeout occurs,
Then a user-friendly error state must display with a 'Retry' button and the error must be logged to an App_Performance_Log__c record.

## Solution

Build the Provider 360 experience using OmniStudio FlexCards with child card composition. A parent FlexCard (Provider360_Parent_FC) uses DataRaptor Extract (DR_Provider360_Summary) to fetch Provider__c base fields and PNA counts via a single SOQL query with aggregate functions. Child FlexCards handle credentialing badge logic (conditional rendering via FlexCard state), claims bar chart (DataRaptor Extract with GROUP BY on Status__c), and PCN list (DataRaptor Extract filtered by Status__c = 'Pending Review'). All DataRaptors are cached with a 5-minute TTL using OmniStudio remote actions. The FlexCard is deployed to the Provider__c Lightning Record Page via a custom Lightning App Builder tab. Performance logging is handled by an Apex invocable action called on timeout events.

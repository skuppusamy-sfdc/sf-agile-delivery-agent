## Description

As a Claims Operations Manager, I need the system to automatically generate an ANSI X12 835 Electronic Remittance Advice (ERA) file and deliver it to the billing provider upon successful claim adjudication so that providers receive timely, accurate payment explanations and manual remittance processing is eliminated.

The ERA must reflect all Claim_Line__c adjudication details, applied Fee_Schedule__c rates, COB adjustments, and any denial reason codes per CMS guidelines.

## Acceptance Criteria

Given a Claim__c.Status__c transitions to 'Adjudicated' and Payment__c.Status__c = 'Approved',
When the ERA generation process triggers,
Then an ERA_Transaction__c record is created with ERA_Transaction__c.File_Format__c = '835', ERA_Transaction__c.Claim__c linked, and ERA_Transaction__c.Generation_Status__c = 'Pending'.

Given ERA_Transaction__c.Generation_Status__c = 'Pending',
When the Apex batch job (ERAGenerationBatch) executes,
Then a well-formed X12 835 file is produced containing: ISA/GS envelope headers, one ST-SE transaction set per Claim__c, CLP segments for each Claim__c, and SVC segments for each Claim_Line__c with adjudicated CPT code, billed amount (Claim_Line__c.Billed_Amount__c), allowed amount (Claim_Line__c.Allowed_Amount__c), and CARC/RARC denial codes where Claim_Line__c.Status__c = 'Denied'.

Given the ERA file is generated successfully,
When delivery is initiated,
Then the file is transmitted to the Provider__c (Person Account) preferred delivery endpoint stored in Provider__c.ERA_Delivery_Method__c (values: 'Clearinghouse', 'SFTP', 'Portal') and ERA_Transaction__c.Delivery_Status__c = 'Sent' with ERA_Transaction__c.Sent_Datetime__c populated.

Given ERA_Transaction__c.Delivery_Status__c = 'Sent',
When the provider's clearinghouse returns a 999 acknowledgment,
Then ERA_Transaction__c.Acknowledgment_Status__c = 'Accepted' and Payment__c.ERA_Reconciled__c = true.

Given a Claim__c involves COB with a secondary payer,
When the ERA is generated,
Then the 835 file includes OI segments reflecting COB_Paid_Amount__c from Claim__c.COB_Secondary_Payment__c and the patient responsibility is calculated as Claim__c.Patient_Responsibility__c = Billed - Primary_Allowed - COB_Paid_Amount__c.

Given ERA generation fails due to missing Provider__c.NPI__c or Provider__c.TIN__c,
When the batch encounters the error,
Then ERA_Transaction__c.Generation_Status__c = 'Error', ERA_Transaction__c.Error_Detail__c is populated with the specific missing field name, and a Task is created assigned to the Provider Network Specialist queue.

## Solution

Implement an Apex Batch class (ERAGenerationBatch, batchable + schedulable) that queries Claim__c records where Status__c = 'Adjudicated' and no ERA_Transaction__c exists. For each claim, a separate ERABuilderService Apex class constructs the X12 835 string using a Custom Metadata Type (ERA_Segment_Config__mdt) for segment ordering and required fields. The generated file is stored as a Salesforce File (ContentVersion) linked to the ERA_Transaction__c. Delivery is handled by a named credential + external service callout to the clearinghouse API (configured per Provider__c.ERA_Delivery_Method__c). A Platform Event (ERA_Delivery_Event__e) handles async acknowledgment processing. Record-Triggered Flow on ERA_Transaction__c updates Payment__c.ERA_Reconciled__c upon acknowledgment. All field-level access controlled via Permission Set 'Claims_Payment_Processor'.

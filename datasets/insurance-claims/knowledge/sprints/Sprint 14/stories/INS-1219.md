## Description

As a Credentialing Specialist, I need an OmniScript-driven workflow that automatically pulls provider attestation data from CAQH ProView when a Credentialing_Case__c is created or enters re-credentialing so that manual data entry is eliminated, credentialing cycle times are reduced, and data accuracy is improved. The integration must map CAQH attestation fields to the corresponding Credentialing_Case__c and Provider__c (Person Account) fields in Salesforce.

## Acceptance Criteria

Given a Credentialing_Case__c is created with Credentialing_Case__c.Type__c = 'Initial Credentialing' or 'Re-Credentialing' and the linked Provider__c has a valid Account.NPI__c,
When the OmniScript 'CAQH_Attestation_Pull' is launched (manually or via Flow automation),
Then the OmniScript calls the CAQH ProView REST API via a named credential (CAQH_ProView_NC) using the provider's NPI and returns the attestation profile.

Given a successful CAQH API response,
When the DataRaptor Transform maps the response,
Then the following fields are populated on Credentialing_Case__c: CAQH_Provider_ID__c, License_Number__c, License_State__c, License_Expiration_Date__c, DEA_Number__c, DEA_Expiration_Date__c, Board_Certification__c, Malpractice_Insurance_Carrier__c, and Malpractice_Policy_Expiration_Date__c.

Given the CAQH attestation data is pulled successfully,
When the DataRaptor Load writes to Salesforce,
Then Credentialing_Case__c.CAQH_Last_Pull_Date__c is set to the current datetime and Credentialing_Case__c.Data_Source__c = 'CAQH ProView'.

Given the CAQH API returns a 404 or the provider is not found in CAQH,
When the OmniScript error handler fires,
Then a user-facing error message displays 'Provider NPI not found in CAQH ProView. Please verify NPI or complete manual attestation.' and Credentialing_Case__c.CAQH_Pull_Status__c = 'Not Found'.

Given CAQH attestation data indicates any license or DEA certificate is expired (expiration date < today),
When the DataRaptor Load completes,
Then Credentialing_Case__c.Status__c is set to 'Pending Additional Documentation' and a Task is created for the Credentialing Specialist with Subject = 'Expired Credential Detected — Action Required'.

## Solution

Build an OmniStudio OmniScript ('CAQH_Attestation_Pull') with the following steps: (1) DataRaptor Extract to retrieve Provider__c NPI and existing credentialing data; (2) HTTP Action step calling CAQH ProView REST API via Named Credential (CAQH_ProView_NC) with OAuth 2.0 authentication; (3) DataRaptor Transform to map JSON response fields to Salesforce field API names; (4) DataRaptor Load to upsert Credentialing_Case__c and Provider__c fields; (5) Conditional step for error handling setting CAQH_Pull_Status__c. A Record-Triggered Flow on Credentialing_Case__c (trigger on insert where Type__c IN ('Initial Credentialing', 'Re-Credentialing')) invokes the OmniScript via an Apex action (OmniScriptLaunchAction.cls). Expired credential detection logic runs in a post-load DataRaptor formula. Integration logging written to Integration_Log__c via a Fire-and-Forget Platform Event (Integration_Log__e).

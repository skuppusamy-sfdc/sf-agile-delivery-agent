## Description

As a Claims Operations Manager, I need the system to automatically generate an ANSI X12 835 Electronic Remittance Advice (ERA) file when a Payment__c record is posted and reconcile it against the associated Explanation_of_Benefits__c (EOB) and Claim_Line__c records so that providers receive accurate remittance data and Payment__c.Reconciliation_Status__c is updated accordingly. Extends INS-1214.

## Acceptance Criteria

Given Claim__c.Status__c transitions to 'Adjudicated' and Payment__c.Status__c = 'Approved',
When the payment posting flow executes,
Then an ERA 835 file is generated containing all associated Claim_Line__c records with CPT codes, allowed amounts, and adjustment reason codes.

Given the ERA file is generated successfully,
When the file is transmitted to the provider's designated clearinghouse endpoint,
Then Payment__c.ERA_Transmission_Status__c = 'Sent' and Payment__c.ERA_Sent_Date__c = today.

Given the clearinghouse returns a 999 acknowledgement,
When the acknowledgement is received via Platform Event (ERA_Ack_Event__e),
Then Payment__c.ERA_Transmission_Status__c = 'Acknowledged' and Explanation_of_Benefits__c.Status__c = 'Delivered'.

Given Payment__c.Amount_Paid__c does not match the sum of Claim_Line__c.Approved_Amount__c,
When reconciliation runs,
Then Payment__c.Reconciliation_Status__c = 'Variance Detected' and a Case is auto-created with Record Type 'Payment Dispute' assigned to the Claims Operations queue.

Given Payment__c.Payment_Method__c = 'EFT',
When ERA is generated,
Then ERA file includes Provider__c.Bank_Routing_Number__c and Provider__c.Bank_Account_Number__c (masked in UI, unmasked in file via Named Credential).

## Solution

Apex batch class (ERA835GeneratorBatch.cls) triggered by a Record-Triggered Flow on Payment__c (Status__c changed to 'Approved'). ERA file constructed using a custom Apex template engine mapping Claim_Line__c fields to X12 835 loop segments (CLP, SVC, CAS, PLB). File uploaded to Salesforce Files and transmitted via MuleSoft integration (named credential: Clearinghouse_API). Platform Event (ERA_Ack_Event__e) consumed by a trigger-based handler updating Payment__c. Reconciliation logic in a separate Apex class (PaymentReconciliationService.cls) called post-transmission.

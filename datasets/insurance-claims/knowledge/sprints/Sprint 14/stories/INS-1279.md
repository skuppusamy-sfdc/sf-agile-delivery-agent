## Description

As a Credentialing Specialist, I need the system to automatically detect when a provider's Credentialing_Case__c is approaching expiration or has lapsed and to trigger a multi-step notification and PNA suspension workflow so that Acme Insurance remains compliant with NCQA credentialing standards and does not pay claims to uncredentialed providers.

## Acceptance Criteria

Given a Credentialing_Case__c.Expiration_Date__c is 60 days from today,
When the nightly scheduled flow runs,
Then a Task is created for the assigned Credentialing Specialist with Subject = 'Credentialing Expiring in 60 Days – [Provider Name]' and Due Date = Credentialing_Case__c.Expiration_Date__c - 30 days.

Given a Credentialing_Case__c.Expiration_Date__c is 30 days from today and Status__c != 'Recredentialing In Progress',
When the nightly flow runs,
Then an automated email is sent to the provider's Provider__c.Email address using the 'Credentialing_Expiry_30Day' email template and Credentialing_Case__c.Recredentialing_Notified__c = true.

Given a Credentialing_Case__c.Expiration_Date__c has passed (today > Expiration_Date__c) and Status__c = 'Active',
When the nightly flow runs,
Then Credentialing_Case__c.Status__c transitions to 'Lapsed' and all related Provider_Network_Agreement__c records where PNA.Provider__c = provider are updated to PNA.Status__c = 'Suspended – Credentialing Lapsed'.

Given PNA.Status__c = 'Suspended – Credentialing Lapsed',
When a new Claim__c is submitted with Claim__c.Billing_Provider__c = the suspended provider,
Then the claim is auto-denied with Claim__c.Denial_Reason__c = 'Provider Not Credentialed' and a Provider_Change_Notification__c (PCN) is auto-generated and sent to the provider.

Given Credentialing_Case__c.Status__c = 'Lapsed' and a new Credentialing_Case__c is subsequently approved,
When Credentialing_Case__c.Status__c transitions to 'Active',
Then all PNA records previously suspended for this provider are evaluated and PNA.Status__c is restored to 'Active' if PNA.Original_Status__c = 'Active', and an audit entry is written to Credentialing_Audit_Log__c.

## Solution

Scheduled Flow 'Credentialing_Expiry_Monitor_Flow' runs nightly at 2 AM, queries Credentialing_Case__c records with Expiration_Date__c within 60 days or past today, and routes to sub-flows for 60-day notification, 30-day email, and lapse handling branches. PNA suspension handled via a separate Record-Triggered Flow on Credentialing_Case__c (after update, Status__c changes to 'Lapsed') which performs a related list update on Provider_Network_Agreement__c via Apex invocable CredentialingLapseHandler.cls. PCN auto-generation uses a Conga CLM template triggered via REST API callout. Custom Object Credentialing_Audit_Log__c captures all status transitions with Before/After values. Permission Set Group 'Credentialing_Automation_PSG' grants the running user access to all modified objects.

## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces real-time network status, active credentialing case status, recent claims activity, and PNA details so that I can have a complete view of a provider's engagement without navigating across multiple objects.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the card displays: Provider NPI (Account.NPI__c), TIN (Account.TIN__c), current network status (Provider_Network_Agreement__c.Status__c), PNA effective and termination dates, and the count of Claim__c records where Claim__c.Rendering_Provider__c = Provider__c.Id submitted in the last 90 days.

Given the provider has an active Credentialing_Case__c with Credentialing_Case__c.Status__c not in ('Approved', 'Closed'),
When the FlexCard renders the credentialing panel,
Then a highlighted badge displays the current Credentialing_Case__c.Status__c value and a direct navigation link to the Credentialing_Case__c record.

Given Provider_Network_Agreement__c.Status__c = 'Terminated' or a Network_Termination_Case__c is linked to the provider with Status__c = 'In Progress',
When the FlexCard loads,
Then a red warning banner displays 'Provider Network Status: Termination Pending or Active' and the FlexCard action buttons for 'Add to Panel' and 'Assign Claims' are disabled.

Given the FlexCard is viewed by a user without the 'Provider_Financials_View' Permission Set,
When the payment summary section attempts to render,
Then that section is hidden and replaced with 'Access Restricted — Contact your administrator'.

-Given provider specialty is displayed as a free-text field- (deprecated — specialty now sourced from CAQH integration via Account.CAQH_Specialty__c).

## Solution

Build an OmniStudio FlexCard (Provider_360_Card) with four child cards: Network Status, Credentialing, Claims Activity, and Payment Summary. Each child card uses a DataRaptor Extract as its data source. The Claims Activity DataRaptor executes a SOQL aggregate query on Claim__c filtered by Rendering_Provider__c and CreatedDate (last 90 days). Network status badge color is driven by a Conditional Formatting rule on Provider_Network_Agreement__c.Status__c. Credentialing badge uses a State Action to navigate to the linked Credentialing_Case__c. Conditional visibility on the Payment Summary card uses a FlexCard visibility condition referencing the running user's Permission Set via a Custom Permission (View_Provider_Financials). Action button disablement for terminated providers uses a FlexCard State with a disabledCondition expression. FlexCard deployed to the Provider__c Lightning Record Page via App Builder.

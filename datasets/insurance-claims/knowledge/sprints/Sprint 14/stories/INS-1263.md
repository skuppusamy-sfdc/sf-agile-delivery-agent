## Description

**Bug Report**

**Summary:** When a Claim__c record undergoes adjudication processing, the system incorrectly advances Claim__c.Status__c to 'Adjudicated' even when one or more child Claim_Line__c records have invalid or mismatched ICD-10 diagnosis pointer codes. This causes downstream ERA generation (INS-1260) to produce malformed 835 files with unmapped diagnosis codes, resulting in payer rejections.

**Steps to Reproduce:**
1. Create a Claim__c record with Claim__c.LOB__c = 'Medical' and at least 3 Claim_Line__c child records.
2. On Claim_Line__c record 2, set Diagnosis_Pointer__c = 'E' where only diagnosis codes A–D are declared on the parent Claim__c.
3. Set Claim_Line__c.CPT_Code__c = '99213' and Claim_Line__c.Billed_Amount__c = 150.00.
4. Trigger adjudication by updating Claim__c.Status__c = 'Pending Adjudication'.
5. Observe Claim__c.Status__c transitions to 'Adjudicated' without any validation error.

**Expected Behavior:** Adjudication should halt and Claim__c.Status__c should remain 'Pending Adjudication' with a validation error surfaced: 'One or more claim lines contain invalid ICD-10 diagnosis pointers. Please correct before adjudicating.'

**Actual Behavior:** Claim__c.Status__c = 'Adjudicated' is set regardless of invalid diagnosis pointers. Claim_Line__c.Adjudication_Status__c is set to 'Approved' erroneously.

**Environment:** Acme-UAT org, API v59.0
**Severity:** High – impacts ERA file integrity and downstream provider payments

## Acceptance Criteria

As per Description.

Additional regression check: Given all Claim_Line__c diagnosis pointers are valid (A–D mapping confirmed against parent Claim__c.Diagnosis_Codes__c)
When adjudication is triggered
Then Claim__c.Status__c transitions to 'Adjudicated' as expected with no false-positive errors.

## Solution

Root cause: The adjudication Record-Triggered Flow (AdjudicationStatusFlow) checks Claim_Line__c validation in a separate before-save flow that fires independently and does not block the parent Claim__c status update. Fix: Add an Apex Validation method in ClaimAdjudicationService.cls that queries all child Claim_Line__c.Diagnosis_Pointer__c values and cross-references against the parent Claim__c diagnosis code array (stored as Claim__c.Diagnosis_Codes_JSON__c) before allowing the status transition. Introduce a Custom Metadata-driven diagnosis pointer validation ruleset (DiagnosisPointerRule__mdt) to support configurable pointer ranges by LOB. Add a before-update Apex Trigger entry point (ClaimTrigger.cls) that calls the validation service and throws an AuraHandledException with a user-friendly message if invalid pointers are found.

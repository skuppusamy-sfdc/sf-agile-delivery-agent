## Description

As a Claims Operations Manager, I need a Policyholder 360 Lightning Web Component (LWC) embedded on the Policy__c record page that displays the policyholder's active policy details, all associated open and recently closed Claim__c records, and a Payment__c history summary so that adjusters have a complete member context view during claim handling without switching between multiple tabs or objects.

## Acceptance Criteria

Given a Policy__c record page is opened in Lightning Experience,
When the Policyholder360Dashboard LWC loads,
Then the component must display Policy__c.Policy_Number__c, Policy__c.LOB__c, Policy__c.Effective_Date__c, Policy__c.Expiration_Date__c, and Policy__c.Status__c in a summary header tile.

Given the policy has related Claim__c records,
When the LWC renders the Claims section,
Then all Claim__c records where Claim__c.Policy__c = current Policy Id and Claim__c.Status__c NOT IN ('Closed – No Payment', 'Withdrawn') must be displayed in a sortable data table with columns: Claim Number, Date of Loss, LOB, Status, Total Claimed Amount, AI Severity Score.

Given the policy has related Payment__c records (via Claim__c parent),
When the LWC renders the Payment History section,
Then Payment__c records from the last 24 months must display with Payment__c.Payment_Date__c, Payment__c.Amount__c, Payment__c.Payment_Method__c, and Payment__c.ERA_Reference__c, with a running total of Payment__c.Amount__c shown as 'Total Paid (24 Months)'.

Given the LWC is loading data for a policyholder with more than 50 Claim__c records,
When the claims data table renders,
Then pagination must be implemented (10 records per page) and a 'Load All' option must be available with a confirmation warning for performance.

Given Policy__c.Status__c = 'Lapsed' or 'Cancelled',
When the LWC header renders,
Then a prominent red banner must display 'Policy Inactive – Verify COB and Coverage Dates Before Processing Claims'.

-Given the policyholder has a COB record, display the COB coordination priority inline in the header.- (Deferred – COB object not yet deployed in Phase 2; targeted for Sprint 16)

Given a user without the 'Claims_Adjuster' or 'Claims_Manager' Permission Set,
When they access the Policy__c record page,
Then the Policyholder360Dashboard LWC must render a 'Insufficient Access' placeholder and must NOT expose Payment__c or Claim__c financial data.

## Solution

Build the Policyholder360Dashboard LWC with three child components: PolicySummaryHeader, ClaimsDataTable, and PaymentHistoryTable. Use @wire with custom Apex controllers (PolicyholderDashboardController.getOpenClaims(), PolicyholderDashboardController.getPaymentHistory()) to fetch data with SOQL enforcing WITH SECURITY_ENFORCED for field-level security. Implement client-side pagination in ClaimsDataTable using tracked currentPage and pageSize properties. Use lightning-datatable for sortable claims rendering. Conditional rendering via JavaScript getter for inactive policy banner. Payment history total calculated client-side using Array.reduce() on the wire response. Add the LWC to Policy__c Lightning Record Page via App Builder with visibility filter restricting display to Permission Set Group 'Claims_Processing_Users'. Deploy via unlocked package with appropriate field permissions in the package's Permission Set.

## Description

As a Claims Operations Manager, I need a consolidated claim history timeline on the Policy__c Lightning Record Page that displays all related Claim__c records with key milestones (FNOL date, adjudication date, payment date, EOB issued date) in chronological order so that I can quickly assess a policyholder's full claims history during coverage reviews and dispute resolution calls.

## Acceptance Criteria

Given a Claims Operations Manager opens a Policy__c record page,
When the Claim History Timeline LWC loads,
Then it must display all related Claim__c records ordered by Claim__c.FNOL_Date__c DESC, showing: Claim__c.Claim_Number__c, Claim__c.LOB__c, Claim__c.Status__c, Claim__c.Total_Billed_Amount__c, Claim__c.Total_Paid_Amount__c.

Given a Claim__c record has an associated Explanation_of_Benefits__c,
When the timeline renders that claim entry,
Then an expandable milestone row must show: EOB Issued Date (Explanation_of_Benefits__c.Issued_Date__c), EOB Total Allowed (Explanation_of_Benefits__c.Total_Allowed_Amount__c), and a link to the EOB PDF.

Given a Policy__c has more than 20 related Claim__c records,
When the timeline first loads,
Then only the 10 most recent claims must be displayed with a 'Load More' pagination control.

Given a Claim__c record with Status__c = 'Referred to SIU',
When that claim appears in the timeline,
Then it must be visually flagged with a red SIU indicator badge, visible only to users with the 'SIU_Investigator_PSG' or 'Claims_Manager_PSG' Permission Set Groups.

Given the timeline is accessed by a user without claims visibility permissions,
When the component renders,
Then Claim__c.Total_Paid_Amount__c and Claim__c.Total_Billed_Amount__c must be hidden and replaced with '––'.

## Solution

Build a custom LWC 'claimHistoryTimeline' using a Wire adapter calling a custom Apex controller 'ClaimHistoryController.getClaimTimeline(policyId, pageSize, offset)' that returns a paginated list of Claim__c with related Explanation_of_Benefits__c child records in a single SOQL query using a LEFT OUTER JOIN pattern. SIU badge visibility controlled via @wire(getRecord) on the running user's Permission Set Group assignments checked in the Apex controller using PermissionSetAssignment SOQL. Financial field masking handled in the Apex controller based on custom permission 'View_Claim_Financials'. LWC embedded on Policy__c Lightning Record Page via App Builder. Lazy loading implemented via a 'Load More' button triggering incremental Apex calls with offset parameter.

## Description

As a Claims Operations Manager, I need an Agentforce AI triage engine to automatically score inbound FNOL claims by severity at the point of creation on Claim__c so that high-complexity and potentially fraudulent claims are immediately routed to senior adjusters, reducing manual triage time by an estimated 40%.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Status__c = 'FNOL Received' and Claim__c.LOB__c is populated,
When the Agentforce Claim Triage Agent is invoked via Platform Event (ClaimFNOLReceived__e),
Then Claim__c.AI_Severity_Score__c (0–100) must be populated within 10 seconds and Claim__c.AI_Triage_Recommendation__c must be set to one of: 'Auto-Adjudicate', 'Standard Review', 'Senior Adjuster', 'SIU Referral'.

Given Claim__c.AI_Severity_Score__c >= 75,
When the triage engine completes scoring,
Then Claim__c.Status__c must transition to 'SIU Referral Pending' and a Task must be created and assigned to the SIU queue with Subject = 'High Severity FNOL – AI Referral'.

Given Claim__c.AI_Severity_Score__c < 30 and Claim__c.LOB__c = 'Auto' and Claim__c.Total_Claimed_Amount__c < 2500,
When triage completes,
Then Claim__c.AI_Triage_Recommendation__c = 'Auto-Adjudicate' and a downstream Flow must enqueue the claim for straight-through processing.

Given the Agentforce model returns a confidence score below 0.60,
When triage completes,
Then Claim__c.AI_Confidence_Flag__c = 'Low Confidence – Manual Review Required' and the claim must NOT be auto-adjudicated regardless of score.

Given any Agentforce API timeout or error,
When the Platform Event handler catches the exception,
Then Claim__c.AI_Triage_Status__c = 'Error – Pending Retry', an error log must be written to AI_Processing_Log__c, and the claim must fall back to standard manual queue assignment.

-Given Claim__c.LOB__c = 'Workers Comp', apply a flat severity multiplier of 1.2 to the raw AI score.- (Superseded – LOB-specific multipliers moved to INS-1203)

## Solution

Deploy an Agentforce Agent Action ('Claim Triage Scorer') backed by an Einstein Generative AI prompt template referencing structured claim metadata fields. A Record-Triggered Flow on Claim__c (After Insert, Async path) publishes ClaimFNOLReceived__e Platform Event containing Claim__c Id, LOB__c, Total_Claimed_Amount__c, and Incident_Description__c. A separate Platform Event-Triggered Flow invokes the Agentforce Agent Action via an Apex invocable method (ClaimTriageEngineService.scoreClaimSeverity()). The Apex class calls the Einstein API endpoint, maps the response to Claim__c fields, and performs a DML update. Retry logic (max 3 attempts, exponential backoff) is handled in a Queueable Apex chain. All AI responses are archived to AI_Processing_Log__c via an Apex helper. Permission Set Group 'Agentforce_Claims_AI' grants access to all new custom fields and objects.

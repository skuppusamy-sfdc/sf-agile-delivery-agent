## Description

As a Claims Operations Manager, I need the Agentforce AI agent to automatically triage incoming FNOL submissions and assign a priority score to each Claim__c record so that high-severity and high-value claims are routed to senior adjusters immediately, reducing manual triage time by at least 60%. The AI agent should evaluate injury severity, estimated loss amount, LOB, and litigation flag to compute a composite triage score.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Stage__c = 'FNOL Received'
When the Agentforce AI triage agent is invoked via Platform Event (FNOL_Submitted__e)
Then Claim__c.AI_Triage_Score__c is populated with a numeric value between 1–100 within 30 seconds

Given Claim__c.AI_Triage_Score__c >= 75
When triage completes
Then Claim__c.Priority__c is set to 'Critical' and Claim__c.Assigned_Adjuster_Tier__c = 'Senior'
And a Task is auto-created with Subject = 'Urgent FNOL Review' assigned to the Claims Operations queue

Given Claim__c.AI_Triage_Score__c between 40–74
When triage completes
Then Claim__c.Priority__c = 'High' and Claim__c.Assigned_Adjuster_Tier__c = 'Standard'

Given Claim__c.Litigation_Flag__c = true (regardless of score)
When triage completes
Then Claim__c.Priority__c is overridden to 'Critical' and SIU_Referral_Pending__c = true

Given the Agentforce agent fails to return a score within 60 seconds
When a timeout occurs
Then Claim__c.AI_Triage_Score__c = null, Claim__c.AI_Triage_Status__c = 'Failed – Manual Review Required'
And a Platform Event error log entry is written to AI_Processing_Log__c

-Given LOB = 'Workers Comp', apply a flat +10 bonus to triage score- (superseded by weighted LOB multiplier table in INS-1261)

## Solution

Deploy an Agentforce Agent Action backed by an Apex invocable method (ClaimsTriageAction.cls) that consumes the FNOL_Submitted__e Platform Event. The Apex class calls the Agentforce Einstein AI model endpoint via Named Credential, passing Claim__c fields (Estimated_Loss_Amount__c, Injury_Severity_Code__c, LOB__c, Litigation_Flag__c) as a JSON payload. Response is parsed and mapped back to Claim__c via a Record-Triggered Flow (after-save, async path) to avoid recursion. A separate Apex Queueable handles timeout monitoring with a 60-second threshold. AI_Processing_Log__c records are written via a Platform Event subscriber trigger. Permission Set Group 'Agentforce_Claims_AI' controls access to AI action fields.

## Description

As a Claims Operations Manager, I need Agentforce to automatically triage incoming FNOL submissions and route new Claim__c records to the appropriate claims queue based on LOB, injury severity score, and coverage type, so that high-complexity claims are escalated immediately and low-complexity claims are auto-assigned without manual intervention, reducing average handle time by at least 30%.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.FNOL_Received__c = true and Claim__c.LOB__c populated,
When Agentforce evaluates the submission within 60 seconds,
Then Claim__c.Triage_Score__c is populated with a value between 1–100 and Claim__c.Routing_Queue__c is set accordingly.

Given Claim__c.Triage_Score__c >= 75 (high severity),
When routing logic executes,
Then Claim__c.Status__c = 'Escalated – SIU Review' and a Task is created assigned to the SIU queue with Priority = 'High' and Due Date = TODAY().

Given Claim__c.LOB__c = 'Workers Comp' and Claim__c.Triage_Score__c < 40,
When Agentforce completes triage,
Then Claim__c.Status__c = 'Auto-Assigned' and Claim__c.Assigned_Adjuster__c is populated via round-robin logic from the Workers Comp adjuster pool.

Given any FNOL submission missing Claim__c.Policy__c lookup or Claim__c.Claimant_Name__c,
When Agentforce attempts triage,
Then triage is suspended, Claim__c.Status__c = 'Pending – Incomplete FNOL', and an automated email is sent to the submitting party using the 'FNOL_Incomplete_Notification' email template.

-Given Claim__c.LOB__c = 'Auto' the triage score cap is 60 regardless of injury flags- (superseded by unified scoring model in INS-1241)

Given a triage decision is made by Agentforce,
When the record is saved,
Then Claim__c.AI_Triage_Confidence__c (percent field) is logged and Claim__c.AI_Model_Version__c is stamped for audit traceability.

## Solution

Deploy an Agentforce Agent Action connected to a custom Apex invocable method (TriageFNOLAction.cls) that accepts Claim__c Id, calls an Einstein Prediction Service model trained on 24 months of historical claims data, and returns a triage score and recommended queue. A Record-Triggered Flow on Claim__c (after-save, created) fires the invocable action asynchronously via a Platform Event (FNOL_Triage_Requested__e) to avoid governor limit contention. Queue assignment uses a custom metadata type (Triage_Routing_Config__mdt) keyed by LOB and score band. OmniScript 'FNOL_Intake_OS' feeds structured data into Claim__c on creation. All AI decisions are written to a child AI_Decision_Log__c object for compliance reporting.

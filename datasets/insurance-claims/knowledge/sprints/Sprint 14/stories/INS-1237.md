## Description

As a Claims Operations Manager, I need the Agentforce Claims Copilot to analyze Claim_Line__c records during adjudication and surface recommended CPT code corrections (unbundling, upcoding, invalid ICD-10 linkage) directly within the claim record page so that adjusters can accept or reject AI recommendations with a single click, improving coding accuracy and reducing rework. Extends INS-1236 triage framework.

## Acceptance Criteria

Given a Claim__c.Status__c = 'In Adjudication' and one or more Claim_Line__c records are present
When an adjuster opens the Claim record page
Then the 'AI CPT Recommendations' FlexCard must render within the claim layout showing all flagged Claim_Line__c records
And each recommendation must display: Claim_Line__c.CPT_Code__c (current), AI_Recommended_CPT__c (suggested), AI_Confidence_Score__c (%), and AI_Flag_Reason__c (e.g., 'Potential Upcoding', 'Invalid ICD-10 Linkage', 'Unbundling Detected')

Given the adjuster clicks 'Accept' on a CPT recommendation
When the OmniScript 'CPT_Correction_OS' action is confirmed
Then Claim_Line__c.CPT_Code__c must be updated to AI_Recommended_CPT__c
And Claim_Line__c.CPT_Override_Reason__c = 'AI Copilot Accepted'
And Claim_Line__c.Allowed_Amount__c must be recalculated using the active Fee_Schedule__c linked to the provider's PNA
And an audit entry must be created in Claim_Audit_Trail__c with actor, timestamp, old value, and new value

Given the adjuster clicks 'Reject' on a recommendation
When a rejection reason is entered (mandatory)
Then Claim_Line__c.AI_Recommendation_Status__c = 'Rejected'
And Claim_Line__c.AI_Rejection_Reason__c is populated
And the original CPT code is preserved unchanged

Given Claim_Line__c has no AI flags
When the FlexCard loads
Then a 'No AI recommendations — coding validated' message must display
And AI_Copilot_Reviewed__c = TRUE on the parent Claim__c

## Solution

Build an OmniStudio FlexCard ('AI_CPT_Recommendations_FC') using a DataRaptor Extract to query Claim_Line__c records with AI flags. The Agentforce Copilot action calls an Apex invocable (CPTRecommendationService.cls) which invokes the Einstein NLP model trained on CPT/ICD-10 edits. Accept/Reject interactions are handled via an OmniScript ('CPT_Correction_OS') with a DataRaptor Transform to update Claim_Line__c and trigger fee recalculation via a Record-Triggered Flow on Claim_Line__c.CPT_Code__c update. Audit trail written via Apex trigger (ClaimLineAuditTrigger.cls).

## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard embedded on the Provider__c (Person Account) record page that consolidates credentialing status, active network agreements, and the last 90 days of claims activity so that I can answer provider inquiries without navigating across five separate objects.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the card displays Credentialing_Case__c.Status__c, Credentialing_Case__c.Expiration_Date__c, and CAQH profile last-verified date from Provider__c.CAQH_Last_Verified__c within 3 seconds.

Given the FlexCard is rendered,
When network agreement data is fetched,
Then all active Provider_Network_Agreement__c records where PNA.Status__c = 'Active' and PNA.Provider__c = current provider are listed with columns: PNA.Network_Name__c, PNA.Effective_Date__c, PNA.Termination_Date__c, PNA.LOB__c.

Given the FlexCard claims activity section loads,
When the DataRaptor retrieves claim data,
Then all Claim__c records where Claim__c.Billing_Provider__c = current provider and Claim__c.Date_of_Service__c >= TODAY-90 are displayed with Claim__c.Claim_Number__c, Claim__c.Status__c, Claim__c.Total_Allowed_Amount__c, and Claim__c.Payment_Status__c.

Given a provider has a Network_Termination_Case__c with Status__c = 'In Progress',
When the FlexCard renders,
Then a prominent warning banner displays 'Termination In Progress – Effective: [Network_Termination_Case__c.Effective_Termination_Date__c]'.

Given the specialist clicks the 'Reverify CAQH' action button on the FlexCard,
When the action fires,
Then an OmniScript 'CAQH_Reverify_OS' is launched in a modal, pre-populated with Provider__c.NPI__c and Provider__c.TIN__c.

## Solution

OmniStudio FlexCard 'Provider360_FC' with three child card states: Credentialing, Network Agreements, Claims Activity. DataRaptor Extract 'DR_Provider360_Extract' performs a single multi-object query joining Provider__c, Credentialing_Case__c, Provider_Network_Agreement__c, and Claim__c with a 90-day date filter. CAQH reverification action triggers OmniScript 'CAQH_Reverify_OS' which calls the CAQH ProView REST API via Named Credential. FlexCard embedded on Provider__c Lightning Record Page via App Builder. Conditional rendering logic in FlexCard JSON handles the termination warning banner. Permission Set 'Provider_Relations_PS' grants read access to all referenced objects.

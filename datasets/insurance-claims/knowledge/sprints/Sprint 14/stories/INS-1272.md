## Description

As a Claims Operations Manager, I need Agentforce to automatically identify potential duplicate Claim__c submissions by matching incoming claims against existing claims using a configurable combination of provider NPI, TIN, CPT codes on Claim_Line__c, service dates, and member Policy__c, so that duplicate payments are prevented, SIU is alerted to potential fraud patterns, and the manual duplicate review queue is reduced by at least 50%.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Status__c = 'Submitted',
When Agentforce duplicate detection runs within 90 seconds of record creation,
Then the system queries existing Claim__c records matching on: Provider__c.NPI__c, Provider__c.TIN__c, Policy__c (same member), Claim_Line__c.CPT_Code__c, and Claim_Line__c.Service_Date__c within a ±1 day window.

Given one or more matching Claim__c records are found with a confidence score >= 85%,
When duplicate detection completes,
Then Claim__c.Duplicate_Flag__c = true, Claim__c.Duplicate_Confidence_Score__c is populated, Claim__c.Status__c = 'Pending – Duplicate Review', and a lookup Claim__c.Potential_Duplicate_Of__c is set to the earliest matching claim Id.

Given Claim__c.Duplicate_Confidence_Score__c >= 95%,
When the high-confidence duplicate threshold is crossed,
Then a Case is auto-created and assigned to the SIU queue with Case.Subject = 'High-Confidence Duplicate Claim – Auto-Flagged', Case.Priority = 'High', and linked to both the new and original Claim__c records.

Given Claim__c.Duplicate_Flag__c = true and an adjudicator manually reviews and determines the claim is NOT a duplicate,
When the adjudicator updates Claim__c.Duplicate_Override__c = true and saves,
Then Claim__c.Status__c reverts to 'In Adjudication', Claim__c.Duplicate_Flag__c = false, and Claim__c.Duplicate_Override_Reason__c is required (validation rule enforced).

Given the duplicate detection model runs,
When any decision is made (duplicate or not),
Then an AI_Decision_Log__c child record is created capturing: model version, confidence score, matched fields, and decision timestamp for SIU and compliance audit purposes.

## Solution

Agentforce Agent Action triggers an Apex invocable method (DuplicateClaimDetectionAction.cls) via async Platform Event (Claim_Submitted__e) published by a Record-Triggered Flow on Claim__c (after-save, created). The detection logic uses a multi-field SOSL/SOQL query across Claim__c and Claim_Line__c, with a weighted scoring algorithm implemented in Apex (configurable weights stored in Duplicate_Detection_Config__mdt custom metadata). Confidence score calculation weighs NPI+TIN match (30%), CPT code match (30%), service date proximity (20%), and member/policy match (20%). SIU Case creation is handled by a separate invocable action to maintain separation of concerns. All decisions are logged to AI_Decision_Log__c. A CRM Analytics dashboard tracks duplicate detection rates, override rates, and SIU escalations weekly. Sharing rules ensure SIU queue members can view flagged Claim__c records across all ownership.

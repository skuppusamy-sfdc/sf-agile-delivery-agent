## Description

As a Claims Operations Manager, I need an Agentforce AI triage agent to automatically receive FNOL submissions, extract structured claim data, and assign an initial fraud-risk score so that my team can prioritize high-risk claims for SIU review without manual pre-screening effort. The agent should handle inbound FNOL data from web, phone transcript, and ERA channels, populate the Claim__c record, and route accordingly.

## Acceptance Criteria

Given an inbound FNOL payload is received via the Claims Intake API endpoint,
When the Agentforce triage agent processes the payload,
Then a new Claim__c record is created with Claim__c.Status__c = 'FNOL Received', Claim__c.LOB__c populated from policy lookup, and Claim__c.FNOL_Channel__c set to the originating channel (Web | Phone | ERA).

Given the Claim__c record is created,
When the AI scoring model evaluates claimant history, provider NPI validity via NPPES, and ICD-10 code plausibility,
Then Claim__c.AI_Risk_Score__c is populated with a numeric value 0–100 and Claim__c.AI_Risk_Tier__c is set to 'Low' (0–39), 'Medium' (40–69), or 'High' (70–100).

Given Claim__c.AI_Risk_Tier__c = 'High',
When the triage agent completes scoring,
Then a Case is automatically created and associated to the Claim__c record with RecordType = 'SIU Review' and assigned to the SIU queue within 60 seconds of FNOL receipt.

Given Claim__c.AI_Risk_Tier__c = 'Low' or 'Medium',
When scoring is complete,
Then the Claim__c.Status__c transitions to 'Pending Assignment' and an Assignment Coordinator task is created via Platform Event (Claim_Assignment_Event__e).

-Given duplicate FNOL submissions are detected by matching Claim__c.Policy__c and Claim__c.Date_of_Loss__c within a 24-hour window, the second submission is auto-rejected with status 'Duplicate'-

Given a duplicate FNOL is detected by matching Claim__c.Policy__c, Claim__c.Date_of_Loss__c, and Claim__c.Claimant_TIN__c,
When the agent identifies the duplicate,
Then Claim__c.Status__c is set to 'Duplicate – Pending Review' and both records are linked via Claim__c.Duplicate_Of__c lookup, preserving both for audit.

Given any FNOL processing failure (API timeout, missing required fields),
When the agent encounters the error,
Then a Platform Event (FNOL_Error_Event__e) is fired, the error is logged to Claim_Processing_Log__c, and an alert email is sent to the Claims Operations Manager queue.

## Solution

Implement an Agentforce Agent with a custom Agent Action backed by an Apex-invocable method (FNOLTriageAction.cls) that calls the NPPES NPI Registry API and internal fraud scoring microservice. A Record-Triggered Flow on Claim__c (after insert) fires the AI scoring Platform Event (FNOL_Scoring_Request__e). An Apex Platform Event trigger (FNOLScoringHandler.cls) consumes the event, calls Einstein scoring via Connect API, and writes back to Claim__c. OmniScript 'FNOL_Intake_OS' provides the web channel intake UI. Duplicate detection handled via a before-insert Apex Trigger with SOQL on Claim__c indexed fields. Permission Set Group 'Claims_AI_Agent_PSG' grants agent runtime permissions.

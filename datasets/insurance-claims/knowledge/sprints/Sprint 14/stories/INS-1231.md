## Description

As a Claims Operations Manager, I need the system to automatically generate an ANSI X12 835 Electronic Remittance Advice (ERA) file for each adjudicated claim batch and reconcile the resulting Payment__c records in Salesforce so that providers receive accurate remittance data and the payment ledger reflects real-time disbursement status. The ERA generation must respect the Fee_Schedule__c contractual rates from the active Provider_Network_Agreement__c.

## Acceptance Criteria

Given a Claim__c record transitions to Claim__c.Status__c = 'Adjudicated' and all related Claim_Line__c records have Claim_Line__c.Adjudicated_Amount__c populated,
When the nightly ERA batch job executes (scheduled Apex: ERAGenerationBatch.cls),
Then an ERA 835 file must be generated with one CLP segment per Claim__c and one SVC segment per Claim_Line__c, and the file must be deposited to the configured SFTP endpoint within the 2-hour batch window.

Given the ERA file is successfully transmitted,
When the transmission acknowledgment is received,
Then Payment__c.Status__c must be updated to 'Remittance Sent', Payment__c.ERA_File_Reference__c must store the generated file name, and Payment__c.Remittance_Date__c must be set to the transmission timestamp.

Given Payment__c.Status__c = 'Remittance Sent' and the provider's clearinghouse returns a 999 acknowledgment,
When the inbound acknowledgment is processed,
Then Payment__c.Status__c must transition to 'Acknowledged' and Explanation_of_Benefits__c.EOB_Status__c must be set to 'Delivered'.

Given a Claim_Line__c has Claim_Line__c.Denial_Reason_Code__c populated,
When the ERA is generated,
Then the corresponding CAS segment must include the correct CARC/RARC codes mapped from Claim_Line__c.Denial_Reason_Code__c per the internal CARC mapping table (ERA_Code_Mapping__mdt).

Given any ERA generation failure,
When the batch job catches an exception,
Then a Platform Event (ERA_Generation_Error__e) must fire, an Error_Log__c record must be created, and the Claims Operations Manager must receive an email alert via an Automated Process user flow.

-Given Payment__c.Payment_Method__c = 'Check', a physical check print job should be triggered automatically- (deferred — check printing integration moved to Phase 3).

## Solution

Build a Schedulable/Batchable Apex class (ERAGenerationBatch.cls) that queries adjudicated Claim__c and Claim_Line__c records, maps fields to ANSI X12 835 segment structures using a custom ERA_Segment_Builder utility class, and writes the flat file to a Named Credential-backed SFTP endpoint via an HttpCallout. Custom Metadata Type (ERA_Code_Mapping__mdt) stores CARC/RARC mappings. Inbound 999 acknowledgments are processed by a second Schedulable Apex class (ERA999AckProcessor.cls) triggered by a File-Based Integration Platform Event. A Record-Triggered Flow on Payment__c updates EOB status. All exceptions publish ERA_Generation_Error__e Platform Events consumed by an Automated Process flow for alerting.

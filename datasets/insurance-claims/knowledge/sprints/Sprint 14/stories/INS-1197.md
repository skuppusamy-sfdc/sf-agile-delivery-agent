## Description

As a Claims Operations Manager, I need an Agentforce AI agent to analyze incoming Claim__c records and detect potential duplicate submissions and Coordination of Benefits (COB) scenarios by comparing member, date of service, and CPT code patterns so that overpayments are prevented and COB recovery workflows are initiated automatically. This story builds on the FNOL triage capability delivered in INS-1191.

## Acceptance Criteria

Given a new Claim__c is created with Claim__c.Status__c = 'Received',
When the Agentforce COB Detection Agent Action is invoked,
Then the agent must query all existing Claim__c records for the same Policy__c.Member_ID__c, overlapping Claim__c.Date_of_Service__c, and matching Claim_Line__c.CPT_Code__c within a 180-day lookback window.

Given a potential duplicate is detected (match score >= 85% per AI model),
When the agent evaluates the record,
Then Claim__c.COB_Flag__c must be set to true, Claim__c.Status__c must transition to 'Pended - COB Review', and Claim__c.Duplicate_Claim_Reference__c must be populated with the matching Claim__c.Id.

Given Claim__c.COB_Flag__c = true,
When the pend is applied,
Then a Case must be auto-created with Case.RecordTypeId = 'COB Investigation', Case.Subject = 'COB Review - {Claim Name}', and assigned to the COB Specialist queue.

Given a COB scenario where Policy__c.Other_Insurance__c = true,
When Claim__c.COB_Flag__c is set,
Then Claim__c.COB_Primary_Payer__c and Claim__c.COB_Secondary_Payer__c fields must be auto-populated from the related Policy__c.Primary_Payer__c and Policy__c.Secondary_Payer__c fields.

Given the AI model confidence score is between 50% and 84%,
When a partial match is detected,
Then Claim__c.COB_Flag__c must remain false but Claim__c.AI_COB_Advisory__c must be set to 'Low Confidence Match - Manual Review Suggested' and a soft-alert must appear on the Claim record page via an LWC banner component.

Given the COB Case is resolved by a specialist,
When Case.Status = 'Closed' and Case.Resolution__c = 'Not Duplicate',
Then Claim__c.COB_Flag__c must be reset to false, Claim__c.Status__c must return to 'Received', and Claim__c.AI_COB_Override_Reason__c must be stamped with Case.Resolution_Notes__c.

## Solution

Agentforce Agent Topic configured for 'COB and Duplicate Detection' with a custom Agent Action (COBDetectionAction) backed by an Apex invocable class (COBDetectionService.cls). The service performs a SOQL query with aggregate matching logic and calls the Einstein AI Model API for probabilistic scoring. A Platform Event (COB_Detection_Event__e) is published after scoring to decouple detection from downstream pend/case creation logic. A subscribing Record-Triggered Automation handles Claim__c field updates. The LWC advisory banner (claimCOBAdvisoryBanner) uses a wire adapter to check Claim__c.AI_COB_Advisory__c and renders conditionally. Custom Metadata Type (COB_Detection_Config__mdt) stores lookback window days, match score thresholds, and CPT code exclusion lists to allow tuning without deployments. All AI decisions are logged to AI_Processing_Log__c for audit and model feedback.

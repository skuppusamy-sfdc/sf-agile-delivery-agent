# INS-1218: Bug — Claim__c.Stage__c incorrectly advances to 'Approved' when COB_Flag__c is true and COB_Verified__c is false

- **Sprint**: Acme Insurance Sprint 14 (Phase 2)
- **Status**: In Test
- **Type**: Bug
- **Priority**: High
- **Resolution**: Unresolved
- **Assignee**: Carlos Rodriguez
- **Reporter**: Emily Watson
- **Components**: ClaimsProcessing, PaymentEngine
- **Epic**: Agentforce (Claims AI)
- **Story Points**: 3
- **Labels**: Phase2, Bug, COB

## Description

Steps to Reproduce:
1. Create a Claim__c record with Claim__c.Stage__c = 'Pending Adjudication', Claim__c.COB_Flag__c = true, and Claim__c.COB_Verified__c = false.
2. Manually update Claim__c.Total_Allowed_Amount__c to any value greater than 0 (simulating fee schedule completion).
3. Observe that the Record-Triggered Flow 'Advance_Claim_Stage_Flow' fires and sets Claim__c.Stage__c = 'Approved'.
4. A Payment__c child record is then auto-created by the PaymentGenerationTrigger.cls despite COB not being verified.

Expected Behavior: Claim__c.Stage__c should NOT advance to 'Approved' when Claim__c.COB_Flag__c = true and Claim__c.COB_Verified__c = false. The stage should remain at 'Pending Adjudication' or transition to 'COB Review'.

Actual Behavior: Claim__c.Stage__c advances to 'Approved' and a Payment__c record is created without COB verification, creating a risk of duplicate payment.

Impact: High — financial risk of duplicate payments on COB claims. Discovered during UAT on INS-1215.

## Acceptance Criteria

As per Description. The fix must add a validation condition to the 'Advance_Claim_Stage_Flow' entry criteria ensuring Claim__c.COB_Flag__c = false OR Claim__c.COB_Verified__c = true before the stage advancement decision node is reached. Additionally, the PaymentValidationTrigger.cls Before-Insert handler must throw a System.AuraHandledException if Payment__c.Claim__r.COB_Flag__c = true and Payment__c.Claim__r.COB_Verified__c = false.

## Solution

1. Edit the Record-Triggered Flow 'Advance_Claim_Stage_Flow': add an entry condition — run only when (COB_Flag__c = false) OR (COB_Flag__c = true AND COB_Verified__c = true). 2. Update PaymentValidationTrigger.cls Before-Insert handler to query parent Claim__c for COB_Flag__c and COB_Verified__c and add a trigger.addError() call if the condition is violated. 3. Add a Flow-level fault path that sets Claim__c.Stage__c = 'COB Review' when COB_Flag__c = true and COB_Verified__c = false. 4. Deploy fix to sandbox, execute existing Apex test class (PaymentValidationTriggerTest.cls) and add two new test methods covering the COB gate scenario. 5. Regression test against INS-1215 acceptance criteria before promoting to UAT.

## Description

As a Claims Operations Manager, I need an Agentforce AI agent to analyze denied Claim_Line__c records on an adjudicated Claim__c and recommend specific EOB adjustments — including recode suggestions for ICD-10/CPT mismatches and medical necessity justifications — so that claims examiners can resolve denials faster and reduce the average re-adjudication cycle from 5 days to under 24 hours.

## Acceptance Criteria

Given a Claim__c.Status__c = 'Adjudicated' with one or more Claim_Line__c records where Claim_Line__c.Adjudication_Status__c = 'Denied',
When the Agentforce Payment Recommendation agent is invoked (manually by examiner or automatically on adjudication complete),
Then the agent analyzes each denied line and populates Claim_Line__c.AI_Recommendation__c with a structured recommendation text within 90 seconds.

Given a denial reason of 'ICD-10/CPT Mismatch' on a Claim_Line__c,
When the agent evaluates the code combination against CMS LCD/NCD policies,
Then Claim_Line__c.AI_Suggested_CPT__c is populated with the recommended alternative CPT code and Claim_Line__c.AI_Recommendation_Confidence__c is set to a percentage value reflecting model confidence.

Given a denial reason of 'Medical Necessity Not Established',
When the agent reviews the associated clinical notes from Claim__c.Clinical_Notes__c (rich text),
Then the agent generates a structured medical necessity justification summary in Claim_Line__c.AI_Recommendation__c and flags whether supporting documentation is sufficient (Claim_Line__c.AI_Docs_Sufficient__c = true/false).

Given the examiner reviews the AI recommendation,
When the examiner clicks 'Accept Recommendation' on the Claim Line LWC component,
Then Claim_Line__c.Adjudication_Status__c updates to 'Pending Re-adjudication', Claim_Line__c.CPT_Code__c is updated to AI_Suggested_CPT__c (if applicable), and an audit record is written to Claim_Adjudication_Audit__c with Action__c = 'AI Recommendation Accepted'.

Given the examiner rejects the recommendation,
When the examiner clicks 'Reject Recommendation' and enters a rejection reason,
Then Claim_Line__c.AI_Recommendation_Status__c = 'Rejected', the rejection reason is stored in Claim_Line__c.AI_Rejection_Reason__c, and the feedback is logged to AI_Model_Feedback__c for model retraining purposes.

-Given Claim__c.Total_Billed_Amount__c exceeds $50,000, the agent escalates all recommendations to a Senior Examiner queue regardless of denial reason-

Given Claim__c.Total_Billed_Amount__c exceeds $50,000 OR Claim_Line__c.AI_Recommendation_Confidence__c < 60%,
When the agent generates recommendations,
Then Claim__c.Escalated_For_Senior_Review__c = true and the claim is routed to the 'Senior Claims Examiner' queue with all AI recommendations visible but locked from auto-acceptance.

## Solution

Agentforce Agent 'Claims_Payment_Advisor' configured with Agent Topic 'Denial Analysis and EOB Recommendation'. Custom Agent Actions backed by Apex invocable class DenialAnalysisAction.cls which calls Einstein Generative AI (via Connect API /einstein/ai/generations) with a structured prompt including ICD-10/CPT context and CMS policy rules. CMS LCD/NCD data cached in Custom Metadata Type CMS_Policy_Rule__mdt refreshed weekly via scheduled DataRaptor. LWC component 'claimLineDenialReview' renders AI recommendations inline on the Claim_Line__c related list with Accept/Reject action buttons. Platform Event AI_Feedback_Event__e fires on rejection to feed the retraining pipeline. Audit logging via Apex after-update trigger on Claim_Line__c writing to Claim_Adjudication_Audit__c.

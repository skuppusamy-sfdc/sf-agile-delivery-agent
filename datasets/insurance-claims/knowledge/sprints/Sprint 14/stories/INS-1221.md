## Description

As a Claims Operations Manager, I need the Agentforce AI agent to automatically classify incoming FNOL submissions by Line of Business (LOB) and injury severity score so that claims are routed to the correct adjudication queue without manual triage, reducing average handle time by an estimated 40%.

This story extends INS-1191 (Agentforce base agent configuration) by adding classification topics and actions specific to FNOL intake.

## Acceptance Criteria

Given a new Claim__c record is created with Status__c = 'FNOL Received' and an associated FNOL document attached,
When the Agentforce Claims Triage agent processes the submission,
Then Claim__c.LOB__c must be populated with one of: 'Auto', 'Workers Comp', 'General Liability', 'Property' within 60 seconds of record creation.

Given the FNOL document contains injury description text,
When the AI agent evaluates the content using the configured Injury Severity topic,
Then Claim__c.Severity_Score__c must be set to a numeric value between 1–5 and Claim__c.Severity_Label__c must reflect 'Minor' (1–2), 'Moderate' (3), or 'Severe' (4–5).

Given Claim__c.Severity_Label__c = 'Severe',
When triage classification is complete,
Then Claim__c.Status__c must transition to 'Pending SIU Review' and a Task must be created and assigned to the SIU queue with Subject = 'Severe FNOL – AI Escalation Required'.

Given Claim__c.LOB__c = 'Workers Comp',
When the AI classification action completes,
Then Claim__c.Assigned_Queue__c must equal 'WC_Adjudication_Queue' and OmniScript 'WC_FNOL_Intake' must be triggered automatically.

-Given the AI agent confidence score is below 0.70, the claim must be auto-rejected and returned to the submitter- (deprecated: replaced by low-confidence routing rule in INS-1224)

Given the Agentforce agent confidence score on LOB classification is below 0.75,
When classification completes,
Then Claim__c.AI_Confidence_Score__c must be stamped and Claim__c.Status__c must be set to 'Manual Triage Required' with an alert surfaced on the Claims Operations Manager's home FlexCard.

Given any classification run completes (success or fallback),
When the agent action finishes,
Then a Platform Event 'Claim_Triage_Completed__e' must be published with fields: Claim_Id__c, LOB__c, Severity_Score__c, Confidence_Score__c, Triage_Timestamp__c.

## Solution

Configure Agentforce Agent with a custom 'FNOL Triage' Topic containing two Actions: (1) LOB_Classifier_Action invoking an Apex-backed external callout to the Acme NLP classification microservice, and (2) Severity_Scorer_Action using a prompt template grounded on the attached FNOL document via Einstein Document Extraction. A Record-Triggered Flow on Claim__c (trigger: Created, Status__c = 'FNOL Received') invokes the Agentforce agent via Agent Action API. Post-classification, a second Flow node updates Claim__c fields and publishes the Claim_Triage_Completed__e Platform Event. OmniScript invocation handled via a DataRaptor Transform feeding the WC_FNOL_Intake OmniScript. FlexCard on Claims Ops dashboard subscribes to the Platform Event via Lightning Streaming API to surface low-confidence alerts in real time.

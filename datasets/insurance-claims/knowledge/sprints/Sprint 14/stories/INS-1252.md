## Description

As a Claims Operations Manager, I need the Agentforce AI agent to automatically triage incoming FNOL submissions and route new Claim__c records to the appropriate claims queue based on LOB, injury severity score, and coverage type so that manual routing overhead is reduced by at least 60% and high-severity claims are prioritized within SLA windows.

This story extends INS-1238 (Agentforce base agent configuration) by adding domain-specific routing logic for medical, auto, and workers' compensation LOBs.

## Acceptance Criteria

Given a new Claim__c record is created with FNOL_Status__c = 'Received' and LOB__c populated,
When the Agentforce AI triage action fires within 2 minutes of record creation,
Then Claim__c.Triage_Score__c is populated with a value between 1–100 and Claim__c.Routing_Queue__c is set to the appropriate queue name matching LOB__c and Triage_Score__c thresholds.

Given a Claim__c has Triage_Score__c >= 80 (high severity),
When routing is completed,
Then Claim__c.Priority__c = 'Critical', Claim__c.SLA_Due_Date__c is set to NOW() + 4 hours, and a Case is auto-created linked to the Claim__c with Subject = 'High Severity FNOL – Immediate Review'.

Given a Claim__c LOB__c = 'Workers Compensation',
When the AI triage completes,
Then Claim__c.Assigned_Examiner__c is populated by round-robin assignment from the WC_Examiners queue and a Platform Event Claim_Triage_Complete__e is published with payload including Claim__c.Id and Triage_Score__c.

Given the Agentforce agent cannot determine LOB due to missing Policy__c relationship,
When triage is attempted,
Then Claim__c.Triage_Status__c = 'Pending Manual Review', an error log entry is created in Agentforce_Error_Log__c, and the claim is routed to the Default_Unassigned queue.

-Given LOB__c = 'Dental', apply specialty dental triage scoring model v1- (deprecated – dental LOB deferred to Sprint 16)

Given triage completes successfully for any LOB,
When the process finishes,
Then Claim__c.FNOL_Status__c transitions to 'Triaged' and an audit entry is written to Claim_Audit_Trail__c with fields: Action__c = 'AI Triage', Performed_By__c = 'Agentforce', Timestamp__c = system time.

## Solution

Configure an Agentforce Agent Action using a custom Apex invocable method (ClaimTriageService.triageClaim()) that accepts Claim__c.Id, queries related Policy__c for LOB__c and coverage details, and calls an Einstein ML scoring endpoint to derive Triage_Score__c. A Record-Triggered Flow on Claim__c (after insert, FNOL_Status__c = 'Received') invokes the Agentforce Action asynchronously via Platform Event (Claim_FNOL_Received__e). Routing logic is implemented in a separate Apex class (ClaimRoutingEngine) using Custom Metadata Type (Claim_Routing_Rule__mdt) keyed on LOB__c and score ranges. Round-robin assignment for WC uses OmniStudio DataRaptor to query queue membership. Agentforce_Error_Log__c records are written via a fault path in the Flow. Permission Set Group 'Agentforce_Claims_Agent' grants FLS on all modified fields.

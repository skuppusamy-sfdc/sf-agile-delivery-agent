## Description

As a Claims Operations Manager, I need an Agentforce Copilot panel embedded in the Claim__c Lightning record page so that claims examiners receive real-time, context-aware adjudication guidance — including applicable policy coverage checks, ICD-10/CPT code validity, COB rules, and SIU referral flags — without leaving the claim workspace. This extends the foundational AI infrastructure established in INS-1229.

## Acceptance Criteria

Given a claims examiner opens a Claim__c record with Claim__c.Status__c = 'Under Review',
When the Agentforce Copilot panel loads,
Then the panel must display a Coverage Summary derived from the related Policy__c.Coverage_Details__c and Policy__c.Effective_Date__c vs. Claim__c.Date_of_Loss__c comparison within 3 seconds.

Given one or more Claim_Line__c records exist with Claim_Line__c.CPT_Code__c populated,
When the Copilot performs a code validity check,
Then each CPT code must be validated against the active Fee_Schedule__c records linked via the Provider_Network_Agreement__c and any invalid codes must be flagged with Claim_Line__c.Code_Validation_Status__c = 'Invalid — Review'.

Given Claim__c.COB_Applicable__c = true,
When the Copilot evaluates coordination of benefits,
Then the panel must surface the primary vs. secondary payer determination and display the calculated Claim__c.COB_Adjusted_Amount__c.

Given the AI model detects anomaly patterns (e.g., duplicate billing, upcoding indicators, or provider billing frequency outliers),
When the Copilot analysis completes,
Then a SIU referral recommendation card must appear with Claim__c.SIU_Flag__c set to 'Recommended' and a draft Case record pre-populated for SIU queue assignment.

Given the examiner accepts or dismisses a Copilot recommendation,
When the interaction is recorded,
Then an AI_Copilot_Interaction__c record must be created capturing: examiner User ID, recommendation type, action taken ('Accepted'/'Dismissed'), and timestamp.

## Solution

Implement an Agentforce Copilot custom action set using Lightning Web Component (AgentforceCopilotPanel.lwc) embedded as a utility bar and record page component on the Claim__c Lightning App Page. The LWC communicates with an Apex controller (ClaimCopilotController.cls) that orchestrates calls to: (1) a SOQL-based coverage check, (2) an Apex callout to the internal Fee Schedule validation service, (3) the Einstein AI model for anomaly scoring. OmniStudio FlexCards surface the COB and SIU recommendation cards. A Record-Triggered Flow on AI_Copilot_Interaction__c creation rolls up acceptance rate metrics to a custom Claim__c field for reporting.

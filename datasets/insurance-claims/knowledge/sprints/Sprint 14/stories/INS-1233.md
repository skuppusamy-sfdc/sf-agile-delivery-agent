## Description

**Bug Report — Incorrect Adjudicated Amount with Tiered Fee Schedule Modifiers**

**Environment:** Acme Insurance Production Sandbox (Sprint 14 Refresh)
**Severity:** High — Financial Impact

**Steps to Reproduce:**
1. Navigate to an existing Claim__c record with at least 3 Claim_Line__c records.
2. Ensure the related Provider_Network_Agreement__c references a Fee_Schedule__c with Fee_Schedule__c.Rate_Type__c = 'Tiered' and at least two Fee_Schedule_Tier__c child records (e.g., Tier 1: units 1–5 at $120.00, Tier 2: units 6+ at $95.00).
3. Set Claim_Line__c.Units_Billed__c = 8 on one of the claim lines with a CPT code covered by the tiered schedule.
4. Trigger adjudication by updating Claim__c.Status__c to 'Ready for Adjudication'.
5. Observe Claim_Line__c.Adjudicated_Amount__c after the adjudication flow completes.

**Expected Result:** Claim_Line__c.Adjudicated_Amount__c = (5 × $120.00) + (3 × $95.00) = $885.00

**Actual Result:** Claim_Line__c.Adjudicated_Amount__c = 8 × $120.00 = $960.00 (flat Tier 1 rate incorrectly applied to all units)

**Root Cause Hypothesis:** The FeeScheduleCalculationService Apex class (FeeScheduleCalculationService.cls) appears to retrieve only the first Fee_Schedule_Tier__c record ordered by Tier_Sequence__c and applies it uniformly across all billed units, ignoring subsequent tier breakpoints.

**Impact:** Overpayment to providers on all claims with tiered fee schedules. Preliminary analysis suggests ~340 Claim_Line__c records in the current adjudication queue may be affected.

## Acceptance Criteria

As per Description. Additionally:

Given a Claim_Line__c with Units_Billed__c = 8 on a CPT code with a tiered Fee_Schedule__c (Tier 1: 1–5 units @ $120.00, Tier 2: 6+ units @ $95.00),
When adjudication runs,
Then Claim_Line__c.Adjudicated_Amount__c = $885.00 and Claim_Line__c.Fee_Schedule_Tier_Applied__c must list all tiers used (e.g., 'Tier 1: 5 units, Tier 2: 3 units').

Given a regression test suite is executed against all Fee_Schedule__c.Rate_Type__c = 'Tiered' scenarios,
When all tests pass,
Then zero Claim_Line__c records in the test dataset should reflect flat-rate calculation for multi-tier CPT codes.

## Solution

Fix FeeScheduleCalculationService.cls to iterate over all Fee_Schedule_Tier__c child records ordered by Tier_Sequence__c ASC, applying a loop that calculates units consumed per tier before moving to the next. Add a helper method (calculateTieredAmount) with unit test coverage ≥ 85%. Deploy a one-time data fix script (DataFixBatch_TieredFeeRecalc.cls) as a Schedulable Apex job to recalculate and correct the ~340 impacted Claim_Line__c records in the adjudication queue. Update Claim__c.Adjudication_Notes__c on affected records to document the correction. Regression test class (FeeScheduleCalculationServiceTest.cls) must include tiered, flat, and percentage-based rate type scenarios.

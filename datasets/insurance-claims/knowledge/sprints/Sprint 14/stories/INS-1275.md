## Description

As a Claims Operations Manager, I need the Agentforce AI agent to automatically determine Coordination of Benefits (COB) order when a claimant has multiple active policies so that payment calculations accurately reflect primary and secondary payer responsibilities, reducing manual COB research time by at least 60%. Extends INS-1244 (COB data model enhancements).

## Acceptance Criteria

Given a Claim__c record is created and the claimant has more than one active Policy__c record,
When the Agentforce COB agent runs,
Then Claim__c.COB_Status__c is set to 'COB Required' and all related Policy__c records are linked via Claim_Policy_COB__c junction object.

Given Claim__c.COB_Status__c = 'COB Required',
When the agent applies birthday rule and NAIC COB guidelines to rank policies,
Then Claim__c.Primary_Policy__c and Claim__c.Secondary_Policy__c lookups are populated and Claim__c.COB_Determination_Method__c records the applied rule ('Birthday Rule' | 'Active Employment' | 'Length of Coverage').

Given COB order is determined,
When the PaymentEngine calculates liability,
Then Payment__c.Primary_Payer_Amount__c and Payment__c.Secondary_Payer_Amount__c are calculated such that their sum does not exceed Claim__c.Total_Billed_Amount__c.

Given the AI agent cannot resolve COB order due to missing subscriber data,
When the agent exhausts its decision tree,
Then Claim__c.COB_Status__c is set to 'Manual COB Review' and the claim is routed to the COB specialist queue with a pre-populated FlexCard showing all relevant policy data.

Given COB determination is complete,
When the EOB is generated,
Then Explanation_of_Benefits__c.COB_Applied__c = true and both payer amounts are itemized per Claim_Line__c.

## Solution

Agentforce Agent Topic configured for 'COB Determination' with a reasoning prompt chain referencing Policy__c subscriber fields. Agent Actions invoke Apex class COBDeterminationService.cls which implements NAIC birthday rule logic. A Record-Triggered Flow on Claim__c (Status__c changes to 'FNOL Received') enqueues the COB evaluation via a Queueable Apex job. FlexCard 'COB_Review_FC' built in OmniStudio surfaces policy comparison data for manual review cases. Integration with external COB database via Named Credential and REST callout in COBExternalLookup.cls.

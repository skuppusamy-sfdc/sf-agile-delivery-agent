## Description

**Bug Report – INS-1278**

**Environment:** Acme Insurance Full Sandbox (Sprint 14 deploy, build 14.3.2)

**Steps to Reproduce:**
1. Create a Claim__c record with Status__c = 'In Adjudication' and a Billing_Provider__c that has NO active Fee_Schedule__c records.
2. Add 3 Claim_Line__c child records with CPT codes not present in any Fee_Schedule__c.
3. Allow the FeeScheduleResolutionService to run (observe via debug log).
4. Navigate back to the Claim__c record and observe Claim__c.Total_Allowed_Amount__c.

**Expected Result:** Claim__c.Total_Allowed_Amount__c should equal the sum of all Claim_Line__c.Allowed_Amount__c values, which are calculated from UCR_Rate__mdt fallback rates.

**Actual Result:** Claim__c.Total_Allowed_Amount__c displays NULL / blank. The Roll-Up Summary field is not aggregating UCR-sourced line amounts.

**Root Cause Hypothesis:** The Roll-Up Summary field filter on Claim__c may be excluding Claim_Line__c records where Rate_Source__c = 'UCR', only summing records where Rate_Source__c = 'Fee Schedule'. This was likely introduced when the GSA/PNA precedence filter was added in Sprint 13.

**Impact:** High – any claim adjudicated under UCR fallback (out-of-network providers, new CPT codes) will have a NULL total, blocking the Status__c transition to 'Adjudicated' and halting payment processing.

**Attachments:** Debug log snippet showing Claim_Line__c.Allowed_Amount__c populated correctly (e.g., $142.50, $88.00, $210.00) but parent roll-up = NULL.

## Solution

Investigate the Roll-Up Summary field 'Total_Allowed_Amount__c' filter criteria on Claim__c in Setup. Remove or update the filter condition that restricts aggregation to Rate_Source__c = 'Fee Schedule' only — the roll-up should aggregate ALL Claim_Line__c.Allowed_Amount__c records regardless of Rate_Source__c value. If the Roll-Up Summary filter cannot be removed without breaking other logic, migrate the aggregation to an Apex after-trigger on Claim_Line__c (ClaimLineTotalRollupTrigger.cls) that explicitly sums all child Allowed_Amount__c values and writes to Claim__c. Add regression test in FeeScheduleResolutionServiceTest.cls covering the all-UCR scenario.

## Description

**Bug Report**

**Summary:** When a Provider_Change_Notification__c (PCN) record is created with PCN.Reason__c = 'Voluntary Termination' or 'Involuntary Termination', the system fails to auto-associate the PCN to an existing open Network_Termination_Case__c for the same provider. The lookup field PCN.Network_Termination_Case__c remains null.

**Steps to Reproduce:**
1. Open an existing Network_Termination_Case__c record with Status = 'In Progress' for Provider__c = 'Dr. Jane Doe (NPI: 1234567890)'.
2. Navigate to related list and create a new Provider_Change_Notification__c with PCN.Reason__c = 'Voluntary Termination' and PCN.Provider__c = same provider.
3. Save the PCN record.
4. Observe PCN.Network_Termination_Case__c = null.

**Expected:** PCN.Network_Termination_Case__c should be auto-populated with the open Network_Termination_Case__c Id for the matching provider.

**Actual:** PCN.Network_Termination_Case__c = null; no association is made.

**Root Cause (Preliminary):** The Record-Triggered Flow 'PCN_Auto_Associate_Flow' appears to have an incorrect filter condition — it is checking Network_Termination_Case__c.Provider__c = PCN.Provider__c using a text comparison instead of an Id lookup, causing all matches to fail.

**Environment:** Acme Insurance – Full Sandbox (FS1) and Production
**Frequency:** 100% reproducible
**Impact:** MEDIUM – Provider Relations Specialists are manually linking PCNs, causing reporting inaccuracies on termination timelines.

## Acceptance Criteria

As per Description. Fix must ensure PCN.Network_Termination_Case__c is correctly auto-populated when an open Network_Termination_Case__c exists for the same Provider__c. Regression must confirm no impact to PCNs with Reason__c values other than termination types.

## Solution

Fix the Record-Triggered Flow 'PCN_Auto_Associate_Flow' (runs after insert on Provider_Change_Notification__c) by correcting the Get Records element filter to compare Network_Termination_Case__c.Provider__c (Id field) against PCN.Provider__c (Id field) using the 'Equals' operator with record Id references — not text formula. Add a null-check condition before the Update Records element. Deploy fix to FS1 first, execute test scenarios, then promote to Production via change set. Update Flow description with fix notes and INS-1250 reference.

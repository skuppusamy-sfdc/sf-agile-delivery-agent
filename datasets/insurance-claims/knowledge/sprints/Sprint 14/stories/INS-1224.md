## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) Lightning Record Page that surfaces the provider's active Provider_Network_Agreement__c (PNA), current Fee_Schedule__c details, NPI validation status, and open claims count so that I have a single-pane view during provider calls without navigating multiple related lists.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record page,
When the Provider 360 FlexCard loads,
Then the card must display: Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Credentialing_Status__c, the count of related Claim__c records with Status__c NOT IN ('Closed','Void'), and the name and effective date of the most recent active Provider_Network_Agreement__c (PNA__c.Status__c = 'Active').

Given the provider has an active PNA,
When the FlexCard renders the Fee Schedule section,
Then it must display Fee_Schedule__c.Schedule_Name__c, Fee_Schedule__c.Effective_Date__c, Fee_Schedule__c.Expiration_Date__c, and a color-coded badge: green if expiration is >90 days, yellow if 30–90 days, red if <30 days.

Given the provider's NPI validation via NPPES last occurred more than 180 days ago (Provider__c.NPI_Last_Validated__c < TODAY-180),
When the FlexCard renders,
Then a warning banner must appear: 'NPI validation overdue – last checked {NPI_Last_Validated__c}' with a quick action button to trigger re-validation.

Given a Provider Relations Specialist clicks the 'Re-validate NPI' quick action,
When the action fires,
Then an OmniScript 'NPI_Revalidation_OS' must launch in a modal, call the NPPES integration DataRaptor, and update Provider__c.NPI_Validation_Status__c and Provider__c.NPI_Last_Validated__c upon completion.

Given the FlexCard is viewed by a user without the 'Provider_Network_Specialist_PSG' Permission Set Group,
When the page loads,
Then the TIN field must be masked as 'XX-XXXNNNN' and the Fee Schedule expiration date must be hidden.

## Solution

Build OmniStudio FlexCard 'Provider_360_FC' with a primary DataRaptor Extract source querying Provider__c fields, a child DataRaptor for PNA (filtered by Status__c = 'Active', ordered by Effective_Date__c DESC, limit 1), a child DataRaptor for Fee_Schedule__c via PNA junction, and an aggregate SOQL DataRaptor for open Claim__c count. Color-coded badge logic implemented via FlexCard Conditional Formatting states. TIN masking handled via a FlexCard Action using a custom formula expression conditioned on the running user's Permission Set Group membership (checked via a DataRaptor Transform calling a custom Apex method). FlexCard embedded on Provider__c Lightning Record Page via App Builder.

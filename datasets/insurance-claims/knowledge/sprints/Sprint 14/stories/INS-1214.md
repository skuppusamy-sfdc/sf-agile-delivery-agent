## Description

As a Claims Operations Manager, I need an Agentforce AI triage agent to process incoming FNOL submissions and automatically score claim severity so that my team can prioritize high-value and high-risk claims without manual review queues delaying initial response SLAs. The agent should ingest structured FNOL data, evaluate ICD-10 diagnosis codes, total claimed amount, and LOB type, then assign a severity tier (Critical / High / Standard / Low) on the resulting Claim__c record.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Stage__c = 'FNOL Received' and Claim__c.LOB__c is populated,
When the Agentforce triage agent is invoked via Platform Event (FNOL_Submitted__e),
Then Claim__c.AI_Severity_Score__c is populated with a numeric value between 0–100 and Claim__c.Severity_Tier__c is set to one of: 'Critical', 'High', 'Standard', or 'Low' within 30 seconds of FNOL submission.

Given Claim__c.Total_Claimed_Amount__c >= 50000 OR any associated Claim_Line__c.ICD10_Code__c falls within the configured high-acuity code set,
When the AI severity scoring model evaluates the claim,
Then Claim__c.Severity_Tier__c must be set to 'Critical' and a Case is auto-created and assigned to the SIU queue.

Given Claim__c.Severity_Tier__c = 'Critical' or 'High',
When severity scoring completes,
Then a real-time notification (Custom Notification Type: 'High_Severity_FNOL') is pushed to all users in the Claims Operations Manager permission set group.

Given the Agentforce agent encounters missing or malformed ICD-10 codes on Claim_Line__c records,
When scoring is attempted,
Then Claim__c.AI_Triage_Status__c is set to 'Pending Manual Review' and Claim__c.AI_Triage_Notes__c is populated with a human-readable explanation of the data gap.

Given any LOB__c = 'Workers Compensation',
When triage scoring runs,
Then the agent applies the WC-specific severity weighting model (configured in Custom Metadata Type: AI_Severity_Model__mdt) rather than the default commercial model.

-Given Claim__c.Source_Channel__c = 'Phone', the agent should attempt voice transcript parsing before ICD-10 lookup- (deprecated — voice transcript integration deferred to Sprint 17).

## Solution

Deploy an Agentforce Agent with a custom Agent Action (Apex-backed) named 'TriageFNOLClaim'. The action subscribes to the FNOL_Submitted__e Platform Event via a Record-Triggered Flow that publishes the event on Claim__c insert where Stage__c = 'FNOL Received'. The Apex action class (TriageFNOLClaimAction.cls) calls an external Einstein AI scoring endpoint via Named Credential, maps the response to Claim__c.AI_Severity_Score__c and Claim__c.Severity_Tier__c, and performs a DML update. SIU Case creation is handled by a downstream Record-Triggered Flow on Claim__c firing on Severity_Tier__c change to 'Critical'. Severity model configuration stored in AI_Severity_Model__mdt Custom Metadata. Custom Notification delivered via Apex (Messaging.CustomNotification). All agent invocation logs written to Agentforce_Audit_Log__c for compliance traceability.

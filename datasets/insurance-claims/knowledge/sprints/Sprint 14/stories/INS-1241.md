## Description

As a Credentialing Specialist, I need the system to automatically identify Provider__c records whose CAQH re-attestation is due within 30, 15, and 7 days and send templated reminder communications via email and in-app notification so that credential lapses are proactively prevented and Credentialing_Case__c records are kept current, reducing manual tracking spreadsheet dependency.

## Acceptance Criteria

Given a Credentialing_Case__c.CAQH_Reattestion_Due_Date__c is within 30 calendar days of today
When the daily scheduled Flow (CAQH_Reattestion_Reminder_Flow) executes at 06:00 UTC
Then an email must be sent to the provider's Provider__c.NPI_Contact_Email__c using the 'CAQH_30Day_Reminder' email template
And a Task must be created on the Credentialing_Case__c: Subject = 'CAQH Re-attestation Due in 30 Days', Due_Date = TODAY+30, Assigned to the provider's primary Credentialing Specialist
And Credentialing_Case__c.Last_Reminder_Sent__c = TODAY

Given a Credentialing_Case__c.CAQH_Reattestion_Due_Date__c is within 15 days
When the flow executes
Then an escalation email must be sent to both the provider AND the Credentialing Specialist's manager (via User.ManagerId)
And Credentialing_Case__c.Reminder_Escalation_Level__c = '15-Day'

Given a Credentialing_Case__c.CAQH_Reattestion_Due_Date__c is within 7 days
When the flow executes
Then Credentialing_Case__c.Status__c must be updated to 'At Risk — Expiring'
And a High-priority Task must be assigned to the queue 'Credentialing Escalations'
And a Salesforce in-app notification (Custom Notification Type: 'Credential_Expiry_Alert') must be sent to the assigned Credentialing Specialist

Given Credentialing_Case__c.CAQH_Profile_Status__c is updated to 'Re-attested' (synced from CAQH integration)
When the CAQH DataSync batch job completes
Then all open reminder Tasks must be closed (Status = 'Completed')
And Credentialing_Case__c.Status__c must revert to 'Active'
And Credentialing_Case__c.CAQH_Reattestion_Due_Date__c must be updated to TODAY + 365

-Send SMS reminder via Twilio integration at 7-day mark-

## Solution

Implement a Scheduled Flow (CAQH_Reattestion_Reminder_Flow) using a Get Records element to query Credentialing_Case__c records by due date proximity. Decision elements branch into 30/15/7-day logic paths. Email Alerts use existing Org-Wide Email Address 'credentialing@acmeinsurance.com'. Custom Notification published via Apex action called from Flow for 7-day in-app alert. CAQH profile sync handled by existing DataSync batch (CAQHProfileSyncBatch.cls) — extend to update CAQH_Reattestion_Due_Date__c on successful re-attestation. Custom Metadata Type (Reminder_Config__mdt) stores day thresholds for admin configurability. Unit tests in CAQHReminderFlowTest.cls cover all three reminder tiers.

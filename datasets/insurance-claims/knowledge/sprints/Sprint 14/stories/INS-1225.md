## Description

**Bug Report**

**Environment:** Production Sandbox (Sprint 14 UAT)
**Reported by:** Amanda Foster (Claims Operations Manager)

**Summary:** The CPT code validation rule on Claim_Line__c is blocking adjudication for all Workers Comp (WC) claims, even when valid WC-specific CPT codes are entered. This is causing a complete adjudication stoppage for the WC LOB since Sprint 13 deployment.

**Steps to Reproduce:**
1. Create a Claim__c record with Claim__c.LOB__c = 'Workers Comp'.
2. Add a Claim_Line__c child record with Claim_Line__c.CPT_Code__c = '99213' (a valid WC CPT code).
3. Set Claim_Line__c.Adjudication_Status__c = 'Pending Review'.
4. Attempt to save the Claim_Line__c record.
5. Observe validation error: 'CPT code 99213 is not listed on the active Fee Schedule for this provider.'

**Expected Behavior:** The validation should cross-reference the Fee_Schedule__c records filtered by Fee_Schedule__c.LOB__c = 'Workers Comp' AND the associated PNA. CPT code 99213 is present on the WC fee schedule.

**Actual Behavior:** The validation rule is querying Fee_Schedule__c without filtering by LOB__c, causing it to match against the General Liability fee schedule where 99213 is not listed, and incorrectly blocking the save.

**Impact:** HIGH – All WC Claim_Line__c records are blocked from adjudication. Approximately 340 claims in backlog as of 2024-01-15.

**Root Cause (suspected):** Validation rule 'VR_CPT_FeeSchedule_Check' formula does not include a LOB__c filter in its SOQL-equivalent cross-object reference. Introduced in INS-1209 deployment.

## Acceptance Criteria

As per Description.

Given a Claim_Line__c record with a valid CPT code present on the Fee_Schedule__c for the matching LOB__c,
When the record is saved,
Then no validation error must fire and Claim_Line__c must save successfully.

Given a Claim_Line__c record with a CPT code NOT present on any Fee_Schedule__c for the matching LOB__c,
When the record is saved,
Then the validation error 'CPT code {CPT_Code__c} is not valid for LOB {LOB__c} fee schedule' must fire.

## Solution

Fix the validation rule 'VR_CPT_FeeSchedule_Check' on Claim_Line__c to include LOB__c context in the cross-object formula. Since standard validation rules cannot perform multi-filter SOQL, replace the validation rule with an Apex Trigger (before insert, before update) on Claim_Line__c that queries Fee_Schedule__c WHERE LOB__c = :claim.LOB__c AND Effective_Date__c <= TODAY AND (Expiration_Date__c = NULL OR Expiration_Date__c >= TODAY) and checks for CPT code membership. Deactivate the existing validation rule 'VR_CPT_FeeSchedule_Check' after Apex trigger deployment. Add unit test coverage targeting WC, Auto, and GL LOB scenarios with minimum 90% coverage.

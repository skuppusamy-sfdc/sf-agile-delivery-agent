## Description

**Bug Report**

**Summary:** When a Provider_Change_Notification__c record is saved with a PCN__c.Distribution_Date__c value set to any date prior to TODAY(), the DocuSign envelope callout is silently skipped and no error is surfaced to the user. The provider never receives the PCN communication, creating a compliance gap.

**Steps to Reproduce:**
1. Navigate to an approved Network_Termination_Case__c record.
2. Trigger the PCN distribution step via the OmniScript 'Provider Termination Workflow'.
3. On the PCN creation step, manually set Distribution_Date__c to any date prior to today (e.g., 30 days ago) — this can occur when back-office staff are processing retroactive terminations.
4. Save the Provider_Change_Notification__c record.
5. Observe: PCN__c.DocuSign_Status__c remains NULL and no envelope appears in the DocuSign admin console. No error toast or validation message is displayed.

**Expected Behavior:** The DocuSign envelope should be sent regardless of whether Distribution_Date__c is backdated, OR a validation error should prevent saving with a past date and prompt the user to confirm retroactive distribution intent before proceeding.

**Actual Behavior:** The Integration Procedure step 'IP_Send_DocuSign_PCN' contains a date comparison condition that evaluates Distribution_Date__c >= TODAY() before invoking the DocuSign REST callout. When this condition is false, the step exits silently with no fault path handling, leaving PCN__c.DocuSign_Status__c = NULL and the provider unnotified.

**Environment:** Full Sandbox (CS47)
**Regression:** Introduced in Sprint 12 deployment (INS-1177 Integration Procedure refactor)
**Severity:** High — regulatory PCN delivery timelines at risk

## Acceptance Criteria

As per Description.

Given a Provider_Change_Notification__c is saved with PCN__c.Distribution_Date__c < TODAY(),
When the OmniScript PCN distribution step executes,
Then either (a) the DocuSign callout proceeds and PCN__c.DocuSign_Status__c is set to 'Sent' with a backdated distribution audit note, OR (b) a blocking validation error is shown: 'Distribution Date cannot be in the past. Please confirm retroactive distribution or update the date.'

Given the DocuSign callout fails for any reason (backdated or not),
When the Integration Procedure fault path is reached,
Then PCN__c.DocuSign_Status__c = 'Failed', an error log record is created in Integration_Error_Log__c, and a high-priority Task is assigned to the Provider Network Specialist queue.

## Solution

Fix the Integration Procedure 'IP_Send_DocuSign_PCN' by removing the silent exit condition on Distribution_Date__c >= TODAY(). Replace with an explicit decision element: if backdated, set a confirmation flag and proceed with callout, logging an audit note on PCN__c.Backdated_Distribution_Note__c. Add a fault path to the DocuSign HTTP Action element that writes to Integration_Error_Log__c and creates a Task via a Set Values element. Add a Validation Rule on Provider_Change_Notification__c to warn (not block) on past Distribution_Date__c with a custom label message. Deploy fix to CS47 sandbox for regression testing before production push.

## Description

As a Credentialing Specialist, I need the system to automatically query the CAQH ProView API when a new Credentialing_Case__c is opened or reaches 'Primary Source Verification' status so that provider demographic, licensure, and attestation data is pulled directly into Salesforce without manual data entry, reducing credentialing cycle time and data entry errors.

The integration must handle both initial credentialing and re-credentialing workflows and respect CAQH authorization status before querying.

## Acceptance Criteria

Given a Credentialing_Case__c.Status__c transitions to 'Primary Source Verification' and Provider__c.CAQH_ID__c is populated,
When the CAQH integration process triggers,
Then a callout is made to the CAQH ProView REST API using Named Credential 'CAQH_ProView_NC' and the response is stored in Credentialing_Case__c.CAQH_Raw_Response__c (Long Text Area).

Given the CAQH API returns a successful response (HTTP 200),
When the response is parsed,
Then the following fields on Credentialing_Case__c are updated:
  - CAQH_Attestation_Date__c = response.attestationDate
  - CAQH_Profile_Status__c = response.profileStatus
  - License_Number__c = response.stateLicense[0].licenseNumber
  - License_State__c = response.stateLicense[0].state
  - License_Expiration_Date__c = response.stateLicense[0].expirationDate
  - DEA_Number__c = response.deaNumber
  - Board_Certification__c = response.boardCertification[0].certificationName

Given Provider__c.CAQH_Authorization_Status__c = 'Not Authorized',
When the integration attempts to query CAQH,
Then the callout is blocked, Credentialing_Case__c.CAQH_Sync_Status__c = 'Authorization Required', and a Task is created assigned to the Credentialing Specialist with Subject = 'Obtain CAQH Authorization – [Provider Name]'.

Given the CAQH API returns HTTP 404 (provider not found),
When the error is handled,
Then Credentialing_Case__c.CAQH_Sync_Status__c = 'Provider Not Found in CAQH', Credentialing_Case__c.Status__c is NOT changed, and an error notification is sent to the assigned Credentialing Specialist via Salesforce in-app notification.

Given a successful CAQH sync completes,
When all required credentialing fields are populated,
Then Credentialing_Case__c.CAQH_Sync_Status__c = 'Verified', Credentialing_Case__c.CAQH_Last_Sync_Date__c = today, and a Credentialing_Audit__c record is created with Action__c = 'CAQH Sync', Source__c = 'Automated', and all changed field values logged.

-Given re-credentialing interval = 2 years, auto-trigger CAQH sync 90 days before expiration- (deprecated – proactive re-credentialing scheduler moved to INS-1263)

## Solution

Implement an Apex class (CAQHIntegrationService) with an @InvocableMethod to be called from a Record-Triggered Flow on Credentialing_Case__c (after update, Status__c = 'Primary Source Verification'). The flow invokes the Apex action asynchronously via a Queueable dispatch to avoid mixed DML and callout restrictions. CAQHIntegrationService uses HttpRequest with Named Credential 'CAQH_ProView_NC' and a JSON deserializer (CAQHResponseWrapper) to map API fields to Salesforce fields. Authorization check queries Provider__c.CAQH_Authorization_Status__c before callout. Error handling uses a try/catch with status code branching. Credentialing_Audit__c records are written in an after-callout DML operation. A Custom Setting (CAQH_Integration_Config__c) stores environment-specific API version and timeout values. Remote Site Setting and Named Credential configured for CAQH ProView sandbox and production endpoints.

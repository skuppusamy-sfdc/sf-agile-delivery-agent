## Description

As a Claims Operations Manager, I need the system to automatically generate an ANSI X12 835 ERA file and update the corresponding Payment__c record when an Explanation_of_Benefits__c (EOB) is finalized so that providers receive accurate electronic remittance advice and payment records are reconciled in real time without manual export steps.

## Acceptance Criteria

Given an Explanation_of_Benefits__c.Status__c transitions to 'Finalized'
When the Record-Triggered Flow (EOB_Finalization_Flow) fires
Then Payment__c.Status__c must be updated to 'Remittance Ready'
And Payment__c.EOB__c must be populated with the finalizing EOB record ID
And Payment__c.Payment_Amount__c must equal the sum of all Claim_Line__c.Paid_Amount__c on the parent Claim__c

Given Payment__c.Status__c = 'Remittance Ready'
When the ERA generation batch job (ERA835GenerationBatch.cls) runs (scheduled every 2 hours)
Then an ERA 835 flat file must be generated conforming to ANSI X12 v005010X221A1
And the file must include: NPI, TIN, Claim__c.External_Claim_Number__c, CPT codes, allowed amounts, adjustment reason codes (CAS segments), and paid amounts per Claim_Line__c
And Payment__c.ERA_File_Generated__c = TRUE
And Payment__c.ERA_Generation_Timestamp__c must be populated

Given the ERA file is successfully transmitted to the provider clearinghouse endpoint
When the outbound callout (ERATransmissionService.cls) receives HTTP 200
Then Payment__c.Status__c must be updated to 'ERA Transmitted'
And Payment__c.Clearinghouse_Confirmation_ID__c must be populated from the response payload

Given the clearinghouse returns HTTP 4xx or 5xx
When transmission fails
Then Payment__c.Status__c must be set to 'ERA Transmission Failed'
And a High-priority Case must be auto-created and assigned to the 'Claims Payment Ops' queue
And a Platform Event (ERATransmissionFailed__e) must be published for downstream monitoring

-Payment__c.Check_Number__c must be populated for paper check fallback-
(Deprecated: paper check fallback moved to Phase 3 scope)

## Solution

Implement ERA835GenerationBatch.cls (Batchable + Schedulable Apex) to query Payment__c records with Status__c = 'Remittance Ready'. ERA 835 file construction handled by ERA835Builder.cls utility class using custom metadata (ERA_Segment_Config__mdt) for segment mappings. Outbound transmission via Named Credential to clearinghouse REST endpoint in ERATransmissionService.cls. Record-Triggered Flow on EOB__c handles status propagation to Payment__c. Platform Event published on failure for integration with monitoring dashboard. All EDI file content stored in ContentVersion linked to Payment__c.

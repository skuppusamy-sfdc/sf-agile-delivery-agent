## Description

As a Provider Network Specialist, I need the system to automatically detect when a Fee_Schedule__c record's effective period rolls over to a new contract year and trigger recalculation of any Claim_Line__c records that were adjudicated against the expiring fee schedule but not yet paid, so that providers are reimbursed at the correct contracted rate for the applicable service period without requiring manual fee schedule reassignment on individual claims.

## Acceptance Criteria

Given a Fee_Schedule__c record where Fee_Schedule__c.Effective_End_Date__c = TODAY() and a successor Fee_Schedule__c exists with Fee_Schedule__c.Effective_Start_Date__c = TODAY() + 1 linked to the same Provider_Network_Agreement__c,
When the nightly fee schedule rollover job executes,
Then Fee_Schedule__c.Status__c transitions from 'Active' to 'Expired' and the successor record transitions to 'Active'.

Given Claim_Line__c records with Claim_Line__c.Fee_Schedule__c linked to the now-expired Fee_Schedule__c and Claim_Line__c.Payment_Status__c = 'Pending',
When the rollover job identifies affected lines,
Then Claim_Line__c.Fee_Schedule__c is updated to the successor Fee_Schedule__c Id and Claim_Line__c.Allowed_Amount__c is recalculated using the new fee schedule rate for the matching CPT code.

Given Claim_Line__c.Allowed_Amount__c changes as a result of fee schedule rollover,
When the recalculation is saved,
Then Claim_Line__c.Rate_Change_Delta__c is populated with the difference (new rate minus old rate) and Claim_Line__c.Rate_Change_Reason__c = 'Fee Schedule Rollover – Automated'.

-Given the delta exceeds $500 per Claim_Line__c an automatic approval workflow is triggered- (deprecated — threshold moved to $250 per finance sign-off, tracked in INS-1269)

Given the delta between old and new Allowed_Amount__c exceeds $250 on any single Claim_Line__c,
When the recalculation completes,
Then an Approval Process is initiated on the parent Claim__c record requiring sign-off from a Claims Operations Manager before Payment__c is issued.

Given no successor Fee_Schedule__c exists when the current schedule expires,
When the rollover job runs,
Then Fee_Schedule__c.Status__c = 'Expired – No Successor', a Case is created assigned to the Provider Network Specialist queue with Subject = 'Fee Schedule Gap – Immediate Action Required', and all linked pending Claim_Line__c records are placed in Claim_Line__c.Payment_Status__c = 'On Hold – Fee Schedule Gap'.

## Solution

Implement a scheduled Apex batch class (FeeScheduleRolloverBatch.cls, executes nightly at 1:00 AM ET) that queries Fee_Schedule__c records expiring on the current date, identifies successor schedules via a parent PNA lookup, updates Status__c fields, and processes affected Claim_Line__c records in batches of 200. Rate recalculation uses a FeeScheduleRateCalculator utility class that performs CPT code lookups against the new Fee_Schedule__c detail records (Fee_Schedule_Line__c child object). Delta calculation and Rate_Change_Delta__c population are handled in the same batch scope. The $250 threshold check triggers Salesforce Approval Process 'Claim_Rate_Change_Approval' via the Process.SubmitRequest Apex API. A custom metadata type (Fee_Schedule_Rollover_Config__mdt) stores the delta threshold and other configurable parameters to avoid hardcoding. Rollover activity is logged to a Fee_Schedule_Rollover_Log__c object for audit. Alerts for missing successor schedules are sent via a Flow-triggered email to the Network Contracting team distribution list.

## Description

As a Claims Operations Manager, I need the system to automatically generate an Explanation of Benefits (EOB) document and produce an Electronic Remittance Advice (ERA) 835 file when a claim is adjudicated so that providers and policyholders receive timely, accurate payment reconciliation information and the payment engine can initiate disbursement without manual document preparation.

## Acceptance Criteria

Given Claim__c.Status__c transitions to 'Adjudicated',
When the post-adjudication Record-Triggered Flow fires,
Then a new Explanation_of_Benefits__c record is created with EOB__c.Claim__c = Claim__c.Id, EOB__c.Generated_Date__c = TODAY(), EOB__c.Total_Allowed_Amount__c = SUM(Claim_Line__c.Allowed_Amount__c), EOB__c.Total_Patient_Responsibility__c = SUM(Claim_Line__c.Patient_Responsibility__c), and EOB__c.Status__c = 'Draft'.

Given an EOB__c record is in 'Draft' status,
When the Conga CLM template merge job runs,
Then a PDF EOB document is generated using the 'Standard_EOB_Template_v3' Conga template, attached as a ContentDocument to both the Claim__c and the policyholder's Policy__c, and EOB__c.Status__c transitions to 'Issued'.

Given EOB__c.Status__c = 'Issued',
When the ERA generation Apex batch processes the adjudicated claim,
Then an 835 ERA file is produced conforming to HIPAA X12 5010 standards, stored as a ContentDocument linked to the Claim__c record with ContentDocument.Title = 'ERA_835_[Claim_Number]_[Date]', and Payment__c.ERA_File_Reference__c is populated.

Given the ERA file is produced and Payment__c.Disbursement_Method__c = 'EFT',
When the PaymentEngine processes the disbursement,
Then Payment__c.Status__c transitions from 'Pending' to 'Disbursed' and Payment__c.Disbursement_Date__c = TODAY() within the same transaction batch.

Given a COB scenario where Claim__c.COB_Status__c = 'Resolved',
When the EOB is generated,
Then EOB__c.Primary_Payer_Paid__c and EOB__c.Secondary_Payer_Paid__c are populated separately and each Claim_Line__c.Adjustment_Reason_Code__c is set per CARC (Claim Adjustment Reason Code) standards.

Given EOB__c.Status__c = 'Issued',
When the policyholder portal sync job runs,
Then the EOB PDF is surfaced in the Policyholder Experience portal under the claimant's claim history within 15 minutes of issuance.

## Solution

Record-Triggered Flow on Claim__c (after update, Status__c changed to 'Adjudicated') calls an Apex invocable action 'CreateEOBRecord' that performs aggregate SOQL on Claim_Line__c to compute totals and inserts the Explanation_of_Benefits__c. A separate Platform Event (EOB_Issued__e) triggers the Conga CLM API callout via a queueable Apex class to merge the PDF template. The ERA 835 batch is an Apex Batchable class (EraGenerationBatch) scheduled nightly and on-demand, writing X12 segments using a custom EDISegmentBuilder utility class. Payment__c status updates handled via a Record-Triggered Flow with field update on Disbursement_Date__c. A Named Credential stores the Conga API endpoint. Permission Set 'EOB_Generator' grants FLS on all EOB fields.

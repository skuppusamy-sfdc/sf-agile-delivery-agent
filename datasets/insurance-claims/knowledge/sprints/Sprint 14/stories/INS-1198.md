## Description

**Bug Summary:** When a Network_Termination_Case__c is created with Network_Termination_Case__c.Termination_Reason__c = 'Voluntary', the associated Provider_Change_Notification__c (PCN) record is not being automatically linked via the PCN__c.Termination_Case__c lookup field. This only affects 'Voluntary' termination reason; 'Involuntary' and 'Contract Expiry' reasons link correctly.

**Steps to Reproduce:**
1. Open a Provider__c record for an active in-network provider.
2. Create a new Network_Termination_Case__c with:
   - Network_Termination_Case__c.Provider__c = the provider's Id
   - Network_Termination_Case__c.Termination_Reason__c = 'Voluntary'
   - Network_Termination_Case__c.Effective_Termination_Date__c = any future date
3. Save the Network_Termination_Case__c.
4. Observe that a Provider_Change_Notification__c record IS created (confirmed via related list).
5. Open the PCN record — PCN__c.Termination_Case__c is NULL.
6. Repeat steps 1–5 with Termination_Reason__c = 'Involuntary' — PCN__c.Termination_Case__c is correctly populated.

**Expected Result:** PCN__c.Termination_Case__c must be populated with the Network_Termination_Case__c.Id for all termination reasons including 'Voluntary'.

**Actual Result:** PCN__c.Termination_Case__c = NULL when Termination_Reason__c = 'Voluntary'. This causes PCN records to be orphaned and prevents the compliance reporting dashboard from tracking voluntary termination notifications.

**Impact:** Medium — compliance reporting for state-mandated PCN filings is incomplete for voluntary terminations. No financial impact identified yet.

**Environment:** Production and Full Sandbox. First reported by Provider Network Specialist team on 2024-11-18.

## Acceptance Criteria

As per Description. After fix: PCN__c.Termination_Case__c must be populated for ALL Network_Termination_Case__c.Termination_Reason__c values including 'Voluntary', 'Involuntary', and 'Contract Expiry'. Regression test must confirm existing 'Involuntary' and 'Contract Expiry' linking is unaffected.

## Solution

Root cause investigation points to the PCN creation Flow (TerminationCase_PCN_AutoCreate_Flow) having a Decision element condition that evaluates Termination_Reason__c IN ('Involuntary', 'Contract Expiry') before executing the PCN link assignment element — 'Voluntary' was omitted from the picklist condition. Fix: update the Decision element condition to evaluate Termination_Reason__c != NULL (i.e., link for all reasons) or explicitly add 'Voluntary' to the IN list. Update the corresponding Flow unit test scenarios. Deploy to Full Sandbox for regression testing before production push. Add a validation rule or Flow fault path to alert if PCN__c.Termination_Case__c is null post-creation as a safety net.

## Description

As a Provider Relations Specialist, I need a comprehensive Provider 360 FlexCard on the Provider__c (Person Account) and Facility__c (Business Account) record pages that surfaces active Provider_Network_Agreement__c status, linked Fee_Schedule__c details, open and recently adjudicated Claim__c summaries, and Credentialing_Case__c status so that when handling inbound provider inquiries I have a single-pane view without navigating across multiple objects. Additionally, an Agentforce Provider Assist agent must be accessible directly from the FlexCard to answer natural-language questions about the provider's contract terms, network participation status, and claim payment history, reducing average handle time for provider inquiry calls. This story builds on the Provider Data Maintenance work completed in INS-1247.

## Acceptance Criteria

Given a Provider Relations Specialist opens a Provider__c record with at least one active Provider_Network_Agreement__c,
When the Provider 360 FlexCard renders,
Then the FlexCard must display: PNA.Status__c, PNA.Effective_Date__c, PNA.Termination_Date__c, PNA.Network_Tier__c, and a badge indicating 'In Network' (green) or 'Out of Network' (red) derived from PNA.Status__c = 'Active' and current date within effective range.

Given the FlexCard is rendered on a Facility__c (Business Account) record,
When the DataRaptor Extract executes,
Then it must retrieve the Facility__c.TIN__c, all linked Fee_Schedule__c records where Fee_Schedule__c.Effective_Date__c <= TODAY and Fee_Schedule__c.Termination_Date__c >= TODAY, and display Fee_Schedule__c.Name, Fee_Schedule__c.LOB__c, and Fee_Schedule__c.Rate_Basis__c in a child card list.

Given a Provider__c record has associated Claim__c records,
When the FlexCard Claims Summary section renders,
Then it must show: count of Claim__c records where Claim__c.Stage__c = 'Open' (last 90 days), count where Claim__c.Stage__c = 'Adjudicated' (last 90 days), total Claim__c.Paid_Amount__c sum for adjudicated claims (last 90 days), and a link to a filtered Claim list view scoped to the provider.

Given a Provider__c record has an active Credentialing_Case__c,
When the FlexCard renders,
Then Credentialing_Case__c.Status__c, Credentialing_Case__c.CAQH_Profile_Status__c, and Credentialing_Case__c.Expiration_Date__c must be displayed; if Credentialing_Case__c.Expiration_Date__c is within 60 days of TODAY a yellow warning badge must appear reading 'Credential Expiring Soon'.

-Given the FlexCard is accessed by a Credentialing Specialist profile, the PNA financial terms section (Fee_Schedule__c.Allowed_Amount__c) must be masked- (requirement moved to INS-1291 field-level security story; masking not in scope for this card).

Given a Provider Relations Specialist clicks the 'Ask Provider Assist' button on the FlexCard,
When the Agentforce Provider Assist agent panel opens,
Then the agent must have context pre-loaded with Provider__c.Id, active PNA record IDs, and the last 5 Claim__c record IDs so that questions such as 'What is this provider's current contracted rate for CPT 99213?' or 'Are there any open claims older than 30 days?' can be answered without the specialist re-entering context.

Given the Agentforce Provider Assist agent cannot resolve a query from available Salesforce data,
When the agent confidence score falls below 70%,
Then the agent must display a fallback message: 'I was unable to find a confident answer. Please review the Provider Network Agreement directly or escalate to your Provider Network Specialist.' and log the unresolved query to a new Provider_AI_Inquiry_Log__c record with fields: Query_Text__c, Provider__c (lookup), Timestamp__c, and Confidence_Score__c.

## Solution

Build the Provider 360 FlexCard in OmniStudio FlexCard Designer with a parent DataRaptor Extract ('DR_Provider360_Extract') that performs a multi-object query across Provider__c / Account, Provider_Network_Agreement__c, Fee_Schedule__c (via PNA junction), Credentialing_Case__c, and a SOQL aggregate for Claim__c metrics (COUNT and SUM grouped by Stage__c with a 90-day date filter). The FlexCard uses conditional visibility states for the 'In Network' badge (State 1: PNA.Status__c = 'Active' + date in range → green) and 'Credential Expiring Soon' badge (State 2: DAYS_BETWEEN(TODAY, Expiration_Date__c) <= 60). The 'Ask Provider Assist' button triggers an Agentforce embedded agent panel via an LWC wrapper component ('c-provider-assist-panel') that passes Provider__c.Id and related record IDs as agent context variables using the AgentContext API. The Agentforce Provider Assist AgentTopic 'Provider_Contract_Inquiry' is configured with AgentActions: (1) LookupPNATerms — DataRaptor Extract on PNA and Fee_Schedule__c; (2) SummarizeClaimHistory — Apex invocable returning aggregated claim metrics; (3) CheckCredentialStatus — DataRaptor Extract on Credentialing_Case__c with CAQH integration metadata. The fallback logging uses a Record-Triggered Flow on a custom confidence-score Platform Event to insert Provider_AI_Inquiry_Log__c records. A Permission Set Group 'Provider360_FlexCard_PSG' is assigned to Provider Relations Specialist and Claims Operations Manager profiles. The FlexCard is deployed to both the Provider__c Lightning Record Page and the Facility__c Lightning Record Page via Dynamic Forms-compatible App Builder configuration.

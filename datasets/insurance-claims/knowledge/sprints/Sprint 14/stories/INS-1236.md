## Description

As a Claims Operations Manager, I need Agentforce to automatically triage and classify newly submitted FNOL claims by severity tier (Low/Medium/High/Critical) and assign a fraud risk score so that claims are routed to the correct adjudication queue without manual intervention, reducing average triage time from 4 hours to under 15 minutes.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Stage__c = 'FNOL Received'
When the Agentforce Claims Triage Agent is invoked via Platform Event (ClaimFNOLReceived__e)
Then Claim__c.Severity_Tier__c must be populated with one of: 'Low', 'Medium', 'High', 'Critical' within 60 seconds
And Claim__c.AI_Fraud_Risk_Score__c (0–100) must be populated based on ICD-10 code pattern analysis, claimant history, and provider TIN anomaly checks
And Claim__c.AI_Triage_Rationale__c must contain a plain-language explanation (max 500 chars) of the scoring decision

Given Claim__c.AI_Fraud_Risk_Score__c >= 75
When triage completes
Then Claim__c.SIU_Referral_Flag__c must be set to TRUE
And a Task must be created and assigned to the SIU queue with Subject = 'AI High-Risk FNOL Review' and Priority = 'High'
And Claim__c.Status__c must be set to 'Pending SIU Review' (NOT 'Auto-Adjudication Queue')

Given Claim__c.AI_Fraud_Risk_Score__c < 75 AND Claim__c.Severity_Tier__c = 'Low'
When triage completes
Then Claim__c.Status__c must be set to 'Auto-Adjudication Queue'
And Claim__c.Assigned_Queue__c must reflect the correct LOB-based queue (Medical, Dental, Vision, Pharmacy)

Given any triage failure or Agentforce timeout (> 90 seconds)
When the Platform Event is not acknowledged
Then a fallback Record-Triggered Flow must set Claim__c.Status__c = 'Manual Triage Required'
And an error log entry must be written to AI_Processing_Log__c with AI_Processing_Log__c.Error_Code__c and timestamp

-Given Claim__c.LOB__c = 'Workers Comp', apply WC-specific severity matrix v1.2-

## Solution

Deploy an Agentforce Agent Action backed by an Apex invocable method (ClaimTriageAIService.cls) that calls the Einstein AI scoring endpoint. Publish ClaimFNOLReceived__e Platform Event from a Record-Triggered Flow on Claim__c insert. The Platform Event trigger invokes the Agentforce Agent asynchronously. Implement a scheduled fallback Apex job (ClaimTriageFallbackBatch.cls) to catch unprocessed FNOLs older than 90 seconds. Store triage configuration (thresholds, queue mappings) in Custom Metadata Type (Claim_Triage_Config__mdt) for admin-adjustable parameters without code deployment. Permission Set Group 'Agentforce_Claims_Triage_PSG' controls agent action visibility.

## Description

As a Provider Network Specialist, I need the system to automate the Network Termination Case workflow — including Provider Change Notification (PCN) distribution via DocuSign, claims impact analysis, and downstream PNA deactivation — so that provider terminations are processed compliantly, affected members are notified within regulatory timeframes, and in-flight claims against the terminating provider are flagged for review.

## Acceptance Criteria

Given a Network_Termination_Case__c is created with Status__c = 'Initiated',
When the termination workflow Screen Flow launches,
Then the specialist must confirm Provider__c.NPI__c, Provider__c.TIN__c, and the proposed PNA__c.Termination_Date__c before the case can advance to 'In Review'.

Given Network_Termination_Case__c.Status__c = 'Approved',
When the PCN distribution step executes,
Then a Provider_Change_Notification__c record is created linked to the case, a DocuSign envelope is sent to the provider's Provider__c.Primary_Email__c using the 'PCN_Termination_v2' template, and PCN__c.Distribution_Date__c = TODAY().

Given PCN distribution is triggered,
When the claims impact analysis Apex job runs,
Then all Claim__c records where Claim__c.Billing_Provider__c = terminating Provider__c.Id and Claim__c.Status__c NOT IN ('Adjudicated', 'Denied', 'Closed') are tagged with Claim__c.Termination_Impact_Flag__c = true and a Task is created for the assigned adjuster.

Given Network_Termination_Case__c.Status__c = 'PCN Acknowledged' (DocuSign callback received),
When the effective termination date is reached (PNA__c.Termination_Date__c = TODAY()),
Then all linked PNA__c records are updated to PNA__c.Status__c = 'Terminated' via a scheduled Flow and Provider__c.Network_Status__c = 'Out of Network'.

Given Provider__c.Network_Status__c = 'Out of Network',
When a new Claim__c is submitted with Claim__c.Billing_Provider__c = terminated provider,
Then Claim__c.Status__c is set to 'Suspended' and Claim__c.Suspension_Reason__c = 'Provider Out of Network — Termination Effective [date]'.

## Solution

A multi-step OmniScript 'Provider Termination Workflow' orchestrates the termination case lifecycle, calling Integration Procedures for DocuSign envelope creation and NPPES NPI validation. A Record-Triggered Flow on Network_Termination_Case__c (after update, Status__c = 'Approved') publishes a Platform Event (Termination_Initiated__e) consumed by an Apex trigger that spawns the ClaimsImpactAnalysisBatch Apex class. DocuSign callback is handled via a REST inbound endpoint updating PCN__c.DocuSign_Status__c. A Scheduled Flow runs nightly to check PNA__c.Termination_Date__c = TODAY() and bulk-updates PNA__c.Status__c. Sharing Rules ensure Network Termination Cases are visible only to Provider Network Specialist and Compliance Officer profiles.

## Description

As a Claims Operations Manager, I need the Agentforce AI agent to automatically detect Coordination of Benefits (COB) scenarios by analyzing the policyholder's Policy__c records and cross-referencing CMS data so that secondary payer fields on Claim__c are auto-populated, reducing manual COB research time and preventing duplicate payment errors.

## Acceptance Criteria

Given a Claim__c is submitted and Policy__c.COB_Indicator__c = true,
When the Agentforce COB Detection topic fires,
Then Claim__c.COB_Status__c = 'COB Detected' and Claim__c.Secondary_Payer__c is populated with the identified secondary payer name.

Given CMS MSP (Medicare Secondary Payer) data is available for the policyholder,
When Agentforce queries the CMS integration via named credential,
Then Claim__c.Medicare_Secondary_Payer__c = true and Claim__c.MSP_Type__c is populated with the appropriate MSP type code.

Given Claim__c.COB_Status__c = 'COB Detected',
When the claim proceeds to adjudication,
Then Claim__c.Primary_Paid_Amount__c must be populated before Claim__c.Status__c can transition to 'Adjudicated' (validation rule enforced).

Given the AI agent cannot determine COB status due to insufficient Policy__c data,
When detection runs,
Then Claim__c.COB_Status__c = 'Manual COB Review' and a Task is created for the assigned adjuster with Subject = 'COB Verification Required'.

Given Claim__c.COB_Status__c = 'COB Detected' and secondary payer responds with ERA 835,
When secondary payment is posted via Payment__c,
Then Claim__c.COB_Reconciliation_Status__c = 'Complete' and total paid does not exceed Claim__c.Billed_Amount__c (over-payment guard).

## Solution

Configure Agentforce Agent Topic 'COB Detection' with two custom actions: (1) PolicyCOBLookupAction invoking Apex to query all active Policy__c records for the Policyholder__c, and (2) CMSMSPCheckAction calling the CMS MBI lookup API via Named Credential. A Record-Triggered Flow on Claim__c (before save, Status__c = 'Submitted') publishes a Platform Event (COB_Check_Event__e). Einstein Prompt Template used for AI reasoning on ambiguous COB scenarios. Validation rule on Claim__c enforces Primary_Paid_Amount__c requirement. Over-payment guard implemented as an Apex before-insert trigger on Payment__c.

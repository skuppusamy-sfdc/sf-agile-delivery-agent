## Description

As an Underwriting Analyst, I need a COB summary panel on the Policy__c record page that clearly displays the primary and secondary coverage hierarchy, payer order, and any active COB investigations so that claims adjudicators can correctly apply COB rules during Claim__c processing and reduce overpayment risk. The panel must support both individual and group policy contexts.

## Acceptance Criteria

Given a Policy__c record page is loaded
When the COB Summary LWC renders
Then it displays all related COB_Coverage__c records sorted by COB_Coverage__c.Payer_Order__c ascending
And each row shows: Payer Name, Policy Number, Coverage_Type__c, Effective_Date__c, Termination_Date__c, and COB_Coverage__c.Is_Primary__c indicator

Given Policy__c.COB_Status__c = 'Under Investigation'
When the panel renders
Then a red banner is displayed: 'COB Investigation Active – Payments May Be Held'
And a link to the associated Case (RecordType = 'COB Investigation') is shown

Given a COB_Coverage__c record has Termination_Date__c in the past
When the panel renders
Then the record is displayed with a 'Terminated' badge and visually de-emphasized (grey row styling)

Given Policy__c.Policy_Type__c = 'Group'
When the panel renders
Then the Group_Service_Agreement__c.Group_Name__c and GSA__c.TPA_Name__c are displayed in a header summary row

-Given COB data is sourced from legacy mainframe via MuleSoft- (replaced by direct Salesforce COB_Coverage__c object as of Sprint 12)

Given a user with 'Claims_Adjudicator' Permission Set views the panel
When a Claim__c is being processed for this policy
Then a 'Apply COB Rules' action button is visible that launches the COB adjudication OmniScript

## Solution

Build a custom LWC (cobSummaryPanel) using @wire(getRelatedListRecords) to fetch COB_Coverage__c child records sorted by Payer_Order__c. Conditional CSS classes applied via JavaScript getter functions for terminated/active/investigation states. The 'Apply COB Rules' button conditionally rendered using a @wire(getPermissionSetAssignment) utility. The COB investigation banner checks Policy__c.COB_Status__c via a parent record wire. The OmniScript (COBAdjudicationOS) launched via NavigationMixin in the LWC action handler. LWC added to Policy__c Lightning Record Page via App Builder with audience targeting for Claims Adjudicator profile. Custom Permission 'View_COB_Panel' added to relevant Permission Set Groups.

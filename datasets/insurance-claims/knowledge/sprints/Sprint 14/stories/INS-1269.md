## Description

As a Provider Relations Specialist, I need a consolidated Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces active Provider_Network_Agreement__c records, current Credentialing_Case__c status, open Claim__c counts by status, and any pending Provider_Change_Notification__c items, so that I can quickly assess a provider's full relationship with Acme Insurance without navigating across multiple objects and tabs.

## Acceptance Criteria

Given a Provider Relations Specialist views a Provider__c record page with the Provider 360 FlexCard deployed,
When the FlexCard loads,
Then the card displays: Provider__c.NPI__c, Provider__c.TIN__c, Provider__c.Credentialing_Status__c, and Provider__c.Network_Participation__c within 2 seconds.

Given the provider has one or more Provider_Network_Agreement__c records with PNA__c.Status__c = 'Active',
When the PNA section of the FlexCard renders,
Then each active PNA is listed showing PNA__c.Contract_Effective_Date__c, PNA__c.Reimbursement_Model__c, PNA__c.Network_Name__c, and a link to the PNA record.

Given the provider has an open Credentialing_Case__c record,
When the credentialing section renders,
Then Credentialing_Case__c.Status__c, Credentialing_Case__c.CAQH_Profile_Status__c, and Credentialing_Case__c.Expiration_Date__c are displayed with a color-coded status indicator (Green = Active, Yellow = Expiring within 90 days, Red = Expired/Suspended).

-Given open Claims are displayed the count should include all Claim__c.Status__c values including 'Closed'- (deprecated — display only open/in-progress statuses per INS-1251 UX spec)

Given the provider has one or more Provider_Change_Notification__c records with PCN__c.Status__c IN ('Submitted', 'Under Review'),
When the PCN section renders,
Then each PCN is listed with PCN__c.Change_Type__c, PCN__c.Submitted_Date__c, and PCN__c.Assigned_Reviewer__c visible.

Given a Provider__c record has no related records in any section,
When that section of the FlexCard renders,
Then a 'No records found' placeholder message is displayed rather than a blank panel.

## Solution

Build using OmniStudio FlexCard with child FlexCards for each section (PNA, Credentialing, Claims, PCN). Each child card uses a DataRaptor Extract to query related records. The parent FlexCard uses an Integration Procedure (IP_Provider360_Summary) to orchestrate all DataRaptor calls in parallel and merge results. Color-coded credentialing status uses a Conditional Format action on the FlexCard state. The FlexCard is added to the Provider__c Lightning Record Page via App Builder. A custom Permission Set ('Provider_360_Viewer') controls FlexCard visibility. Caching is set to 5-minute TTL on the Integration Procedure to reduce SOQL load during high-volume periods.

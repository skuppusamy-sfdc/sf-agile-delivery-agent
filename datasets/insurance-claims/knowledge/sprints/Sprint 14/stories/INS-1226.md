## Description

As a Credentialing Specialist, I need the system to automatically initiate a Credentialing_Case__c record and populate it with data retrieved from the CAQH ProView integration whenever a new Provider__c record is created with a valid NPI so that credentialing workflows begin without manual data entry and turnaround time is reduced from an average of 12 days to under 7 days.

## Acceptance Criteria

Given a new Provider__c (Person Account) record is created with Provider__c.NPI__c populated and Provider__c.Record_Source__c = 'Network Onboarding',
When the record is saved,
Then a Platform Event 'Provider_Onboarding_Initiated__e' must be published with NPI__c, Provider_Id__c, and Onboarding_Timestamp__c.

Given the Platform Event is consumed by the CAQH integration subscriber flow,
When the CAQH ProView API returns a valid provider profile (HTTP 200),
Then a Credentialing_Case__c must be created with: Credentialing_Case__c.Provider__c = the originating Provider__c Id, Credentialing_Case__c.Status__c = 'Document Collection', Credentialing_Case__c.CAQH_Provider_Id__c populated, Credentialing_Case__c.License_Expiration_Date__c populated, Credentialing_Case__c.Board_Certification__c populated.

Given the CAQH API returns HTTP 404 (provider not found),
When the integration subscriber processes the response,
Then Credentialing_Case__c must still be created with Status__c = 'CAQH Not Found – Manual Entry Required' and a Task assigned to the Credentialing Specialist queue with Subject = 'CAQH Lookup Failed – Manual Credentialing Required'.

Given a Credentialing_Case__c is created with Status__c = 'Document Collection',
When 5 business days have elapsed without Status__c changing,
Then an escalation email must be sent to the assigned Credentialing Specialist and Credentialing_Case__c.Escalated__c must be set to TRUE.

-Credentialing cases must be mirrored to the legacy CredentialStream system via SOAP callout- (deprecated: CredentialStream decommissioned as of Phase 2 go-live)

Given any Credentialing_Case__c status change,
When the record is updated,
Then Provider__c.Credentialing_Status__c must be kept in sync via a Record-Triggered Flow.

## Solution

Implement a Record-Triggered Flow on Provider__c (after insert, filter: NPI__c IS NOT NULL AND Record_Source__c = 'Network Onboarding') to publish the Provider_Onboarding_Initiated__e Platform Event. Build a Platform Event-Triggered Flow subscribing to this event that invokes an Apex invocable method 'CAQH_ProView_Callout' using the Named Credential 'CAQH_ProView_NC'. The Apex class handles HTTP callout to CAQH REST API, parses the JSON response, and performs DML to create Credentialing_Case__c. A separate Record-Triggered Flow on Credentialing_Case__c (after update) syncs status back to Provider__c. Escalation handled via a Scheduled Path on the Credentialing_Case__c flow (5 business days after Created Date). Permission Set Group 'Credentialing_Specialist_PSG' must have CRUD on Credentialing_Case__c.

## Description

As a Credentialing Specialist, I need an automated alerting and re-credentialing initiation workflow that identifies Credentialing_Case__c records expiring within 90, 60, and 30 days and launches a guided re-credentialing OmniScript so that provider network compliance is maintained and we avoid gaps in coverage that could result in claims payment errors or regulatory violations.

## Acceptance Criteria

Given a Scheduled Flow runs daily at 6:00 AM EST and queries all Credentialing_Case__c records where Status__c = 'Active' and Expiration_Date__c <= TODAY+90,
When expiring records are identified,
Then Credentialing_Case__c.Expiration_Alert_Stage__c must be set to '90 Day Warning', '60 Day Warning', or '30 Day Warning' based on days remaining, and a custom notification must be sent to the assigned Credentialing_Case__c.Owner.

Given Credentialing_Case__c.Expiration_Alert_Stage__c = '60 Day Warning' and no active re-credentialing case exists (no related Credentialing_Case__c with Type__c = 'Re-Credentialing' and Status__c NOT IN ('Closed','Withdrawn')),
When the daily flow runs,
Then a new Credentialing_Case__c record must be auto-created with Type__c = 'Re-Credentialing', Status__c = 'Initiated', Parent_Credentialing_Case__c = expiring record Id, and assigned to the same owner.

Given the re-credentialing case is created,
When the Credentialing Specialist opens the record,
Then the 'Start Re-Credentialing' OmniScript button must be visible on the page layout and launch the 'ProviderReCredentialing' OmniScript with pre-populated provider NPI, TIN, and CAQH_ID__c values from the parent Provider__c record.

Given the OmniScript completes the CAQH data fetch step,
When CAQH API returns provider attestation data,
Then Credentialing_Case__c.CAQH_Last_Attested__c and Credentialing_Case__c.CAQH_Status__c must be updated via DataRaptor Load.

Given Credentialing_Case__c.Expiration_Date__c < TODAY and Status__c = 'Active',
When the daily flow runs,
Then Credentialing_Case__c.Status__c must be auto-updated to 'Expired', Provider__c.Credentialing_Status__c must be set to 'Expired', and all active PNA records (Provider_Network_Agreement__c.Status__c = 'Active') for that provider must be flagged with PNA.Credentialing_Hold__c = true.

## Solution

Build a Scheduled Flow ('DailyCredentialingExpirationCheck') that runs at 6 AM EST, queries Credentialing_Case__c with appropriate date filters, and uses loop + decision elements to set alert stages and create re-credentialing cases. Implement a CAQH integration using a Named Credential and Apex REST callout class (CAQHIntegrationService) invoked from within the OmniScript via an Integration Procedure. The 'ProviderReCredentialing' OmniScript is built in OmniStudio with DataRaptor Extract steps to pre-populate fields and DataRaptor Load steps to persist CAQH results. Add the OmniScript action button to the Credentialing_Case__c Lightning Record Page via App Builder. Sharing Rules ensure Credentialing Specialists can access all provider records in their assigned region.

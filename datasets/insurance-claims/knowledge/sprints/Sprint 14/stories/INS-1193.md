## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard embedded on the Provider__c (Person Account) record page that surfaces the provider's active Provider_Network_Agreement__c, associated Fee_Schedule__c effective dates, and a real-time count of open Claim__c records so that I have a single unified view without navigating to multiple related lists.

## Acceptance Criteria

Given a Provider__c record is open in Salesforce,
When the Provider 360 FlexCard loads,
Then it must display the count of Provider_Network_Agreement__c records where PNA__c.Status__c = 'Active' linked to that provider.

Given the active PNA section of the FlexCard,
When PNA data is rendered,
Then PNA__c.Contract_Effective_Date__c, PNA__c.Contract_Termination_Date__c, PNA__c.Network_Tier__c, and PNA__c.TIN__c must all be visible.

Given the Fee Schedule section,
When the FlexCard loads,
Then Fee_Schedule__c.Effective_Date__c and Fee_Schedule__c.Expiration_Date__c must be shown, with a visual indicator (red badge) if the fee schedule expires within 30 days.

Given the Open Claims summary section,
When the FlexCard renders,
Then the count of Claim__c records where Claim__c.Billing_Provider__c = Provider__c.Id AND Claim__c.Status__c NOT IN ('Adjudicated','Denied','Void') must be displayed.

-Given the FlexCard, a 'Send Message' action button must be present to initiate an in-app Chatter message (removed per UX review, replaced by dedicated action bar)-

## Solution

OmniStudio FlexCard (ProviderSummary360_FC) built with three child cards: PNA Summary, Fee Schedule Summary, and Open Claims Counter. DataRaptor Turbo Extract components retrieve data from Provider_Network_Agreement__c, Fee_Schedule__c, and a SOQL aggregate on Claim__c. The 30-day fee schedule expiry indicator uses a FlexCard conditional state with a formula comparing Fee_Schedule__c.Expiration_Date__c to TODAY(). FlexCard is embedded on the Provider__c Lightning Record Page via the OmniStudio FlexCard component. CRUD/FLS enforced via Permission Set Group (PSG_Provider_Relations).

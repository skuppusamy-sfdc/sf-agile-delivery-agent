## Description

As a Provider Relations Specialist, I need a Provider 360 FlexCard on the Provider__c (Person Account) record page that surfaces real-time credentialing status from Credentialing_Case__c, active Provider_Network_Agreement__c contracts, and a network participation summary so that I can assess a provider's full engagement profile without navigating across multiple objects. The card must also reflect any open Provider_Change_Notification__c items.

## Acceptance Criteria

Given a Provider__c record page is loaded by a user with the 'Provider_Relations' Permission Set
When the Provider 360 FlexCard renders
Then the card displays: NPI, TIN, Credentialing_Case__c.Status__c (most recent active case), count of active PNA__c records, and network tier from Provider_Network_Agreement__c.Network_Tier__c

Given the provider has an open Provider_Change_Notification__c with PCN__c.Status__c = 'Pending Review'
When the FlexCard loads
Then a yellow warning badge is displayed with the PCN count and a deep-link to the PCN list view

Given Credentialing_Case__c.Expiration_Date__c is within 30 days of today
When the FlexCard renders
Then the credentialing status chip displays in amber with label 'Expiring Soon'

Given Credentialing_Case__c.Status__c = 'Lapsed' OR Network_Termination_Case__c is active for this provider
When the FlexCard renders
Then the network participation status displays 'Out of Network' in red
And a 'Reinstate' action button is visible to users with 'Provider_Network_Admin' Permission Set only

-Given CAQH integration is live, pull credentialing data directly from CAQH API- (deferred to Phase 3 per architecture review)

Given no active PNA records exist
When the FlexCard renders
Then the contracts section displays 'No Active Contracts' with a 'Create PNA' quick action button

## Solution

Build OmniStudio FlexCard (ProviderSummary360Card) with three child DataRaptor Extract DataRaptors: DR_GetProviderCredentials (queries Credentialing_Case__c), DR_GetActivePNAs (queries Provider_Network_Agreement__c filtered by Status__c = 'Active'), and DR_GetOpenPCNs (queries Provider_Change_Notification__c). Conditional styling states configured in FlexCard State configuration for amber/red/green credentialing badges. The 'Reinstate' action invokes an OmniScript (ProviderReinstateOS) gated by a Flyout action with Permission Set check via Custom Permission 'Can_Reinstate_Provider'. FlexCard deployed to Provider__c Lightning Record Page via App Builder. DataRaptor caching set to 60-second TTL to balance freshness vs. SOQL limits.

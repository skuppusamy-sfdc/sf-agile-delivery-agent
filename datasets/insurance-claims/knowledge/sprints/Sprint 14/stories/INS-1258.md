## Description

As a Provider Network Specialist, I need the system to automatically detect Provider_Network_Agreement__c records approaching expiration and initiate a structured renewal workflow so that no provider contracts lapse unintentionally, network adequacy is maintained, and compliance reporting reflects accurate in-network provider counts.

Alerts should be sent at 120 days, 60 days, and 30 days before PNA.Termination_Date__c with escalation to a supervisor at the 30-day mark if renewal has not been initiated.

## Acceptance Criteria

Given a Provider_Network_Agreement__c.Termination_Date__c is 120 days from today and PNA.Status__c = 'Active',
When the nightly scheduled Flow (PNA_Expiration_Alert_Scheduler) executes,
Then a Task is created assigned to PNA.Owner with Subject = 'Contract Renewal Due in 120 Days – [Provider Name]', Due_Date = today + 5 days, and PNA.Renewal_Alert_120_Sent__c = true.

Given PNA.Termination_Date__c is 60 days from today and PNA.Renewal_Initiated__c = false,
When the scheduled Flow executes,
Then a Task is created, an email alert is sent to PNA.Owner using Email Template 'PNA_60Day_Renewal_Alert', and PNA.Renewal_Alert_60_Sent__c = true.

Given PNA.Termination_Date__c is 30 days from today and PNA.Renewal_Initiated__c = false,
When the scheduled Flow executes,
Then a Task is created assigned to PNA.Owner's manager (User.ManagerId), PNA.Escalation_Flag__c = true, and an email is sent to both the Owner and the Owner's manager.

Given a Provider Network Specialist clicks 'Initiate Renewal' on the PNA record,
When the action is invoked,
Then PNA.Renewal_Initiated__c = true, PNA.Status__c = 'Renewal In Progress', a new Conga CLM contract draft is created via API callout using PNA.Contract_Template_ID__c, and the Conga envelope ID is stored in PNA.Conga_Renewal_Envelope_ID__c.

Given PNA.Termination_Date__c passes and PNA.Status__c != 'Renewed' and PNA.Status__c != 'Terminated',
When the scheduled Flow executes on the day after expiration,
Then PNA.Status__c = 'Expired', all linked Fee_Schedule__c records have Fee_Schedule__c.Active__c set to false, and a ComplianceReporting_Alert__c record is created with Alert_Type__c = 'Contract Lapse' for network adequacy reporting.

## Solution

Build a Scheduled Flow (PNA_Expiration_Alert_Scheduler) using a Scheduled Path triggered nightly at 2:00 AM. The flow uses Get Records to find PNA records meeting each date threshold (DATEVALUE(TODAY()) + 120, +60, +30) with corresponding flag fields = false. Decision elements branch to sub-flows for each alert tier. Email alerts use Salesforce Email Templates with merge fields. The 'Initiate Renewal' button on the PNA Lightning Record Page triggers a Screen Flow (PNA_Renewal_Initiation_Flow) that performs the Conga CLM callout via an Apex Action (CongaCLMService.createRenewalDraft()) using Named Credential 'Conga_CLM_NC'. The post-expiration lapse logic runs in a separate Scheduled Flow path that bulk-updates Fee_Schedule__c records via a Loop + Update Records element. Custom fields Renewal_Alert_120_Sent__c, Renewal_Alert_60_Sent__c, Renewal_Initiated__c, Escalation_Flag__c, Conga_Renewal_Envelope_ID__c are added to Provider_Network_Agreement__c. Sharing Rules ensure Provider Network Specialists see only PNAs in their assigned region.

## Description

As a Claims Operations Manager, I need an Agentforce AI Triage Agent to automatically score incoming FNOL submissions for claim severity so that high-risk and complex claims are immediately routed to senior adjusters, reducing manual triage time and improving SLA compliance. The agent should evaluate structured claim intake data alongside policy coverage details and historical loss patterns to produce a severity score (Low / Medium / High / Critical) and recommended routing action at the point of FNOL registration. This story extends INS-1261 (FNOL Intake OmniScript) by consuming the structured payload produced during digital intake.

## Acceptance Criteria

Given a new Claim__c record is created with Claim__c.Stage__c = 'FNOL Received' and Claim__c.LOB__c populated,
When the Agentforce Triage Agent is invoked via the AgentAction callout within 30 seconds of record creation,
Then Claim__c.AI_Severity_Score__c must be populated with one of: 'Low', 'Medium', 'High', 'Critical' and Claim__c.AI_Triage_Timestamp__c must be set to the current datetime.

Given Claim__c.AI_Severity_Score__c = 'Critical',
When the severity score is written back to the record,
Then a high-priority Case of RecordType 'Senior_Adjuster_Review' must be auto-created and associated via Claim__c.Triage_Case__c lookup, and a Platform Event Claim_Triage_Alert__e must be published to notify the Claims Operations queue.

Given Claim__c.AI_Severity_Score__c = 'High' or 'Critical',
When triage completes,
Then Claim__c.Assigned_Adjuster_Tier__c must be set to 'Tier 1 - Senior' and Claim__c.SLA_Target_Hours__c must be set to 4.

Given Claim__c.AI_Severity_Score__c = 'Low' or 'Medium',
When triage completes,
Then Claim__c.Assigned_Adjuster_Tier__c must be set to 'Tier 2 - Standard' and Claim__c.SLA_Target_Hours__c must be set to 24 or 48 respectively.

Given any Agentforce callout failure or timeout,
When the agent response is not received within 60 seconds,
Then Claim__c.AI_Triage_Status__c must be set to 'Failed - Manual Review Required' and the record must be added to the Manual_Triage_Queue__c assignment group without blocking claim progression.

-Given Claim__c.LOB__c = 'Workers Compensation', the triage agent shall apply a flat 'High' floor score regardless of model output- (superseded: model now trained on WC-specific loss corpus; dynamic scoring applies to all LOBs as of Sprint 13 model refresh).

Given a Claims Operations Manager views a triaged Claim__c record,
When the Agentforce Triage FlexCard component is rendered on the Claim lightning page,
Then the card must display AI_Severity_Score__c, AI_Triage_Timestamp__c, AI_Confidence_Pct__c (0–100), and a human-readable rationale string from Claim__c.AI_Triage_Rationale__c (max 500 characters).

Given Claim__c.COB_Indicator__c = true,
When the triage agent scores the claim,
Then Claim__c.AI_Triage_Rationale__c must include the token 'COB_FLAG' and Claim__c.AI_Severity_Score__c must be promoted by one tier (e.g., Low → Medium) to account for coordination of benefits complexity.

## Solution

Implement a Record-Triggered Flow on Claim__c (trigger: after insert, condition: Stage__c = 'FNOL Received') that enqueues an Apex Queueable job (AgentforceTriage_Queueable) to perform the Agentforce Agent API callout asynchronously, respecting governor limits. The Agentforce Triage Agent is configured in the Agentforce Agent Builder with a custom AgentTopic 'FNOL_Severity_Triage' and three AgentActions: (1) RetrievePolicyContext — DataRaptor Extract pulling Policy__c coverage, deductible, and prior-loss fields; (2) ScoreClaim — Einstein generative model invocation with a structured prompt template referencing ICD-10 codes, CPT codes if medical LOB, and loss description NLP tokens; (3) WriteTriageResult — Apex invocable method that performs a DML update on Claim__c and conditionally creates the triage Case. A Platform Event (Claim_Triage_Alert__e) is published for Critical claims and consumed by a separate Event-Triggered Flow that posts a Slack/Chatter notification to the Claims Operations Manager group. The Agentforce FlexCard is built in OmniStudio FlexCard Designer, bound to a DataRaptor Extract on Claim__c. Permission Set Group 'Agentforce_Claims_Triage_PSG' controls access to the AI fields and FlexCard component. All AI field values are audit-logged via a custom AI_Audit_Log__c object with a master-detail to Claim__c.
